## 1️⃣ บทนำ


### Start9 คืออะไร?


Start9 เป็นบริษัทที่ก่อตั้งขึ้นในปี 2020 ซึ่งเป็นที่รู้จักดีที่สุดในการพัฒนา [**StartOS**](https://github.com/Start9Labs/start-os) ระบบปฏิบัติการที่ใช้ Linux ออกแบบมาสำหรับเซิร์ฟเวอร์ส่วนบุคคล ช่วยให้ผู้ใช้สามารถโฮสต์ซอฟต์แวร์บริการต่างๆ ได้อย่างง่ายดาย เช่น Bitcoin และโหนด Lightning แอปส่งข้อความ หรือผู้จัดการรหัสผ่าน ในขณะที่ยังคงควบคุมข้อมูลของตนเองได้อย่างเต็มที่และลดการพึ่งพาแพลตฟอร์มเทคโนโลยีแบบรวมศูนย์ StartOS มีอินเทอร์เฟซที่ใช้งานง่ายผ่านเบราว์เซอร์ ตลาดที่คัดสรรมาเพื่อการติดตั้งบริการ และเครื่องมือความเป็นส่วนตัวในตัว เช่น การรวม Tor และการเข้ารหัส HTTPS ทั่วทั้งระบบ Start9 ยังมีอุปกรณ์ฮาร์ดแวร์ที่ติดตั้งระบบปฏิบัติการล่วงหน้า แม้ว่าซอฟต์แวร์จะสามารถติดตั้งบนฮาร์ดแวร์ที่เข้ากันได้หรือเครื่องเสมือน (VMs) ได้ก็ตาม


### มีตัวเลือกอะไรบ้าง?


Start9 มีตัวเลือกการปรับใช้ทั้งแบบสำเร็จรูปและ DIY [**Server One**](https://store.start9.com/collections/servers/products/server-one) และ [**Server Pure**](https://store.start9.com/collections/servers/products/server-pure) เป็นอุปกรณ์ฮาร์ดแวร์อย่างเป็นทางการที่มีส่วนประกอบประสิทธิภาพสูง: Server One ใช้โปรเซสเซอร์ **AMD Ryzen 7 5825U** พร้อม RAM ที่ปรับแต่งได้ (16GB–64GB) และที่เก็บข้อมูล (2TB–4TB NVMe SSD) ในขณะที่ Server Pure ติดตั้ง **Intel i7-10710U** ซึ่งมีตัวเลือก RAM และที่เก็บข้อมูลที่ปรับแต่งได้เช่นกัน ทั้งสองรวมถึง **การสนับสนุนทางเทคนิคตลอดชีพ** เมื่อซื้อโดยตรงจาก Start9 สำหรับผู้ใช้ที่ต้องการความยืดหยุ่น สามารถติดตั้ง StartOS ได้ฟรีบนฮาร์ดแวร์ที่มีอยู่หลากหลายประเภท รวมถึงแล็ปท็อป เดสก์ท็อป มินิพีซี และคอมพิวเตอร์บอร์ดเดี่ยว หรือภายใน VM

### ความแตกต่างจาก Umbrel คืออะไร?


StartOS และ Umbrel ทั้งสองช่วยให้การติดตั้งบริการแบบโฮสต์เองง่ายขึ้น แต่แตกต่างกันในด้านสถาปัตยกรรมและคุณสมบัติ Umbrel ทำงานเป็นชั้นแอปพลิเคชันบนระบบหรือ VM ของ Debian/Ubuntu ในขณะที่ Start9 เป็นระบบปฏิบัติการเฉพาะที่ต้องการการติดตั้งฮาร์ดแวร์หรือ VM โดยตรง Umbrel มีอินเทอร์เฟซที่ขัดเกลาและได้รับแรงบันดาลใจจาก macOS ในขณะที่ Start9 ให้ความสำคัญกับการออกแบบที่ใช้งานได้และเรียบง่าย Umbrel มี [การเลือกแอปพลิเคชัน](https://apps.umbrel.com/) ที่ใหญ่กว่า ในขณะที่ [Start9 Marketplace](https://marketplace.start9.com/) ชดเชยด้วยความสามารถทางเทคนิคขั้นสูง Start9 ทำให้การเข้าถึงการตั้งค่าขั้นสูงง่ายขึ้นผ่านฟอร์ม UI ที่ผ่านการตรวจสอบ ลดความจำเป็นในการโต้ตอบกับบรรทัดคำสั่ง นอกจากนี้ยังโดดเด่นในการจัดการการสำรองข้อมูลด้วยการสำรองข้อมูลระบบที่เข้ารหัสด้วยคลิกเดียว ซึ่งเป็นคุณสมบัติที่ Umbrel ขาดในตัว StartOS มีเครื่องมือการตรวจสอบในตัวและบังคับใช้การเข้ารหัส HTTPS สำหรับการเข้าถึงเครือข่ายท้องถิ่น เพิ่มความปลอดภัย Umbrel ใช้ HTTP ที่ไม่ได้เข้ารหัสในท้องถิ่น แม้ว่าทั้งสองแพลตฟอร์มจะรองรับการเข้าถึงระยะไกลที่ปลอดภัยผ่าน Tor Umbrel เหมาะสำหรับผู้ใช้ที่ให้ความสำคัญกับระบบนิเวศของแอปที่หลากหลายและ UI ที่ขัดเกลา Start9 เป็นตัวเลือกที่แข็งแกร่งสำหรับผู้ที่ให้ความสำคัญกับความปลอดภัย การปรับแต่ง ความน่าเชื่อถือในการสำรองข้อมูล และการจัดการบริการขั้นสูงโดยไม่ต้องใช้ความเชี่ยวชาญในบรรทัดคำสั่ง หากต้องการเรียนรู้เพิ่มเติมเกี่ยวกับ Umbrel และความแตกต่างจาก Start9 โปรดเยี่ยมชมหลักสูตรนี้:

## 2️⃣ ข้อกำหนดเบื้องต้น DIY: สเปคขั้นต่ำและที่แนะนำ


สำหรับการใช้งานพื้นฐานที่มีบริการน้อยที่สุด **สเปคขั้นต่ำ** คือ: **1 vCPU core (2.0GHz+ boost), 4GB RAM, 64GB storage**, และพอร์ต Ethernet อย่างไรก็ตาม ผมแนะนำให้ใช้สเปคที่สูงกว่านั้น โดยเฉพาะถ้าคุณกำลังใช้งาน Bitcoin Node ส่วนตัวผมเริ่มต้นด้วย 1TB และพื้นที่ก็หมดอย่างรวดเร็ว ควรตั้งเป้าไว้ที่ **อย่างน้อย 2TB storage**, พร้อมกับ **quad-core CPU (2.5GHz+)** และ **8GB+ RAM** ซึ่งจะสร้างความแตกต่างอย่างมากในด้านประสิทธิภาพและอายุการใช้งาน หากคุณต้องการเจาะลึก นี่คือกระทู้ชุมชนที่อัปเดตเกี่ยวกับ [Hardware Capable of Running StartOS](https://community.start9.com/t/known-good-hardware-master-list-hardware-capable-of-running-startos/66/229)


## 3️⃣ ดาวน์โหลดและแฟลชเฟิร์มแวร์


ในการเริ่มต้นการตั้งค่า ให้ใช้คอมพิวเตอร์เครื่องอื่นเพื่อเยี่ยมชม [เว็บไซต์ Start9](https://start9.com/) และไปที่ส่วนเอกสารโดยคลิกที่ `DOCS` จากนั้นเข้าถึง `Flashing Guides` เพื่อค้นหาเวอร์ชันที่เหมาะสมของ StartOS มีสองตัวเลือกให้เลือก:



- StartOS (Raspberry Pi)
- StartOS (X86/ARM)


บทแนะนำนี้ครอบคลุมตัวเลือก `x86/ARM`


สามารถดาวน์โหลดเวอร์ชันล่าสุดของระบบปฏิบัติการได้จาก [Github release page](https://github.com/Start9Labs/start-os/releases/latest) นอกจากนี้ยังมีเวอร์ชัน [Pre-release](https://github.com/Start9Labs/start-os/releases) สำหรับผู้ใช้ที่ต้องการทดสอบฟีเจอร์ใหม่ ๆ ที่ด้านล่างของหน้า ภายใต้ `Assets` ให้ดาวน์โหลด `x86_64` หรือ `x86_64-nonfree.iso` ไฟล์ภาพ `x86_64-nonfree.iso` ประกอบด้วยซอฟต์แวร์ที่ไม่ใช่ซอฟต์แวร์เสรี (closed-source) ที่จำเป็นสำหรับ Server One และฮาร์ดแวร์ DIY ส่วนใหญ่ โดยเฉพาะอย่างยิ่งสำหรับการรองรับอุปกรณ์กราฟิกและเครือข่าย


การตรวจสอบความสมบูรณ์ของไฟล์โดยการตรวจสอบค่าแฮช SHA256 กับค่าที่ระบุไว้บน GitHub เป็นสิ่งที่แนะนำ สำหรับ Linux สามารถใช้คำสั่ง `sha256sum startos-0.3.4.2-efc56c0-20230525_x86_64.iso` ได้ โดยมีระบบปฏิบัติการอื่นๆ ครอบคลุมอยู่ในเอกสารประกอบ


หลังจากดาวน์โหลดและตรวจสอบภาพ StartOS แล้ว จะต้องแฟลชลงในไดรฟ์ USB `BalenaEtcher` เป็นซอฟต์แวร์ที่แนะนำสำหรับงานนี้ เป็นเครื่องมือฟรีและโอเพ่นซอร์สสำหรับเขียนไฟล์ภาพ OS ลงในไดรฟ์ USB และการ์ด SD ซึ่งมีให้สำหรับ Windows, macOS และ Linux ดาวน์โหลดเวอร์ชันที่เหมาะสมจาก [เว็บไซต์ Balena Etcher อย่างเป็นทางการ](https://www.balena.io/etcher/) และเรียกใช้ตัวติดตั้ง เชื่อมต่อไดรฟ์ USB หรือการ์ด SD เป้าหมาย เปิด Balena Etcher และคลิก `Flash from file` เพื่อเลือกภาพ OS ที่ดาวน์โหลดมา Etcher จะตรวจจับไดรฟ์ที่เชื่อมต่อโดยอัตโนมัติ เลือกเป้าหมายที่ถูกต้องหากมีหลายตัว คลิก `Flash!` เพื่อเริ่มเขียนภาพ Etcher จะตรวจสอบกระบวนการเขียนโดยอัตโนมัติเมื่อเสร็จสิ้น เมื่อเสร็จแล้ว ให้ถอดไดรฟ์ออกอย่างปลอดภัยและใช้เพื่อบูตอุปกรณ์

## 4️⃣ การตั้งค่าเริ่มต้น


สำหรับการตั้งค่าเริ่มต้น ให้ดูที่หน้า [เอกสารประกอบ](https://docs.start9.com/0.3.5.x/) Start9 ภายใต้ `USER MANUAL` ตามด้วย `Getting Started - Initial Setup` ควรปรึกษาคู่มืออย่างเป็นทางการนี้เพื่อข้อมูลที่ทันสมัยที่สุด


มีการนำเสนอสองตัวเลือก:



- เริ่มต้นใหม่
- ตัวเลือกการกู้คืน


สำหรับการติดตั้งเซิร์ฟเวอร์ใหม่ ให้เลือก `Start Fresh` ก่อนอื่นให้เชื่อมต่อเซิร์ฟเวอร์กับแหล่งจ่ายไฟและสาย Ethernet ตรวจสอบให้แน่ใจว่าคอมพิวเตอร์ที่ใช้สำหรับการตั้งค่าอยู่ในเครือข่ายท้องถิ่นเดียวกัน ถอดไดรฟ์ USB ที่เพิ่งแฟลชออกจากคอมพิวเตอร์และเสียบเข้ากับเซิร์ฟเวอร์


คุณสามารถควบคุมเซิร์ฟเวอร์จากระยะไกลได้จากคอมพิวเตอร์ใดก็ได้ในเครือข่ายเดียวกัน เปิดเว็บเบราว์เซอร์และไปที่ `http://start.local`.


**หมายเหตุ**: หากเกิดปัญหาการเชื่อมต่อกับที่อยู่นี้ มักเกิดจากเครือข่ายภายในบ้านที่ไม่สามารถแก้ไขชื่อโดเมน `.local` ได้ ปัญหานี้สามารถแก้ไขได้โดยการเข้าถึงเซิร์ฟเวอร์โดยตรงผ่านที่อยู่ IP สามารถค้นหา IP ได้โดยการเข้าสู่ระบบอินเทอร์เฟซผู้ดูแลของเราเตอร์ (โดยทั่วไปที่ `192.168.1.1` หรือที่อยู่ที่คล้ายกัน) และค้นหาอุปกรณ์ในรายการลูกค้า DHCP หรือแผนที่เครือข่าย จากนั้นป้อนที่อยู่ IP เต็ม (เช่น `http://192.168.1.105`) ในเบราว์เซอร์ วิธีนี้จะข้ามการแก้ไข DNS หากปัญหายังคงอยู่ ให้ปรึกษา [หน้าปัญหาทั่วไป](https://docs.start9.com/0.3.5.x/support/common-issues.html#setup-troubleshoot) หรือ [ติดต่อฝ่ายสนับสนุนของพวกเขา](https://start9.com/contact/)


หน้าจอติดตั้ง StartOS ควรปรากฏขึ้น คลิก `Start Fresh` เพื่อเริ่มการตั้งค่าเซิร์ฟเวอร์ใหม่


![image](assets/en/03.webp)


ขั้นตอนต่อไปคือการเลือกไดรฟ์จัดเก็บข้อมูลที่ข้อมูล StartOS จะถูกจัดเก็บไว้


![image](assets/en/04.webp)


ตั้งรหัสผ่านที่แข็งแกร่งสำหรับเซิร์ฟเวอร์ บันทึกตามคำแนะนำของ Start9 จากนั้นคลิก `FINISH`


![image](assets/en/05.webp)


หน้าจอจะแสดงว่า StartOS กำลังเริ่มต้นและตั้งค่าเซิร์ฟเวอร์ ขั้นตอนต่อไปคือการ `Download address info` เนื่องจากที่อยู่ `start.local` ใช้สำหรับการตั้งค่าเท่านั้นและจะไม่ทำงานหลังจากนั้น


![image](assets/en/06.webp)


ไฟล์การกำหนดค่าประกอบด้วยที่อยู่การเข้าถึงที่สำคัญสองรายการ: หนึ่งสำหรับ `local network (LAN)` และอีกหนึ่งสำหรับการเข้าถึงที่ปลอดภัยผ่าน `Tor` ทั้งสองที่อยู่ควรถูกบันทึกในตัวจัดการรหัสผ่านที่ปลอดภัย ขั้นตอนต่อไปคือการ `Trust your Root CA` เปิดแท็บเบราว์เซอร์ใหม่และทำตามคำแนะนำเพื่อเชื่อถือ Root CA และเข้าสู่ระบบ ใบรับรอง Root CA สามารถดาวน์โหลดได้โดยคลิกที่ `Download certificate` ในไฟล์ที่ดาวน์โหลดมา


## 5️⃣ เชื่อถือ Root CA ของคุณ


หลังจากดาวน์โหลดใบรับรองแล้ว `Root CA` ของเซิร์ฟเวอร์จะต้องได้รับความไว้วางใจจากระบบปฏิบัติการ คลิก `View Instructions` และค้นหาคำแนะนำสำหรับระบบปฏิบัติการเฉพาะ


![image](assets/en/07.webp)


สำหรับระบบ Linux ใช้คำสั่งต่อไปนี้ ก่อนอื่นให้เปิด Terminal และติดตั้งแพ็คเกจที่จำเป็น:


```shell
sudo apt update

sudo apt install -y ca-certificates p11-kit
```


นำทางไปยังไดเรกทอรีที่ดาวน์โหลดใบรับรอง โดยปกติคือ `~/Downloads` ดำเนินการคำสั่งเหล่านี้เพื่อเพิ่มใบรับรองไปยังที่เก็บความเชื่อถือของระบบปฏิบัติการ เปลี่ยนไปที่โฟลเดอร์ดาวน์โหลดด้วย `cd ~/Downloads` สร้างไดเรกทอรีที่ต้องการด้วย `sudo mkdir -p /usr/share/ca-certificates/start9` คัดลอกไฟล์ใบรับรอง โดยแทนที่ `your-filename.crt` ด้วยชื่อไฟล์จริง โดยใช้ `sudo cp "your-filename.crt" /usr/share/ca-certificates/start9/` ลงทะเบียนใบรับรองอย่างถาวรโดยเพิ่มเส้นทางของมันไปยังการกำหนดค่าระบบด้วย `sudo bash -c "echo 'start9/your-filename.crt' >> /etc/ca-certificates.conf"` สุดท้าย สร้างที่เก็บความเชื่อถือใหม่ด้วย `sudo update-ca-certificates` สิ่งสำคัญคือต้องใช้ชื่อไฟล์ใบรับรองจริงและตรวจสอบเส้นทางทั้งหมดก่อนดำเนินการคำสั่งเหล่านี้ กระบวนการนี้สร้างความเชื่อถือถาวรสำหรับการเชื่อมต่อ HTTPS ของเซิร์ฟเวอร์ Start9


การติดตั้งที่สำเร็จจะแสดงผลลัพธ์ว่า `1 added` แอปพลิเคชันส่วนใหญ่จะสามารถเชื่อมต่อได้อย่างปลอดภัยผ่าน `https` หากใช้ Firefox จะต้องมี [ขั้นตอนสุดท้าย](https://docs.start9.com/0.3.5.x/misc-guides/ca-ff.html#ca-ff) เพิ่มเติม สำหรับ Chrome หรือ Brave จะต้องมี [ขั้นตอนสุดท้าย](https://docs.start9.com/0.3.5.x/misc-guides/ca-chrome.html#ca-chrome) ที่แตกต่างกันเพื่อกำหนดค่าเบราว์เซอร์ให้เคารพ Root CA ทดสอบการเชื่อมต่อโดยการรีเฟรชหน้า หากปัญหายังคงอยู่ ให้ปิดและเปิดเบราว์เซอร์ใหม่ก่อนที่จะกลับไปที่หน้าอีกครั้ง

## 7️⃣ ระบบ


แท็บระบบใน StartOS ให้การเข้าถึงฟังก์ชันระบบหลักสำหรับการจัดการเซิร์ฟเวอร์ส่วนบุคคล มันมีเครื่องมือสำหรับการบำรุงรักษาระบบ ความปลอดภัย การวินิจฉัย และการกำหนดค่าโดยไม่ต้องใช้ความเชี่ยวชาญในบรรทัดคำสั่ง


ส่วน `Backups` อนุญาตให้สร้างการสำรองข้อมูลระบบทั้งหมด รวมถึงบริการ การกำหนดค่า และข้อมูล ซึ่งสามารถกู้คืนได้ในภายหลัง สิ่งนี้จำเป็นสำหรับการกู้คืนจากภัยพิบัติหรือการย้ายไปยังฮาร์ดแวร์ใหม่ การสำรองข้อมูลสามารถจัดเก็บไว้ในไดรฟ์ภายนอกและถูกเข้ารหัสโดยใช้รหัสผ่านหลัก


ส่วน `Manage` ในแท็บ Systems ช่วยให้ควบคุมฟังก์ชันหลักของระบบได้ ผู้ใช้สามารถตรวจสอบและใช้การอัปเดต StartOS ด้วยตนเอง เพื่อควบคุมกระบวนการอัปเดตระบบ สามารถติดตั้งบริการที่กำหนดเองหรือของบุคคลที่สามที่ไม่มีในตลาดอย่างเป็นทางการได้ หากเซิร์ฟเวอร์ไม่ได้เชื่อมต่อผ่าน Ethernet สามารถกำหนดค่า Wi-Fi ได้จากส่วนนี้ ผู้ใช้ขั้นสูงสามารถเปิดใช้งานการเข้าถึง SSH สำหรับการจัดการระบบในระดับเทอร์มินัลได้


![image](assets/en/10.webp)


ส่วน `Insights` ให้การตรวจสอบประสิทธิภาพและสุขภาพของเซิร์ฟเวอร์แบบเรียลไทม์ โดยแสดงการใช้งาน CPU, RAM และดิสก์ผ่านกราฟ นอกจากนี้ยังแสดงอุณหภูมิของระบบ ซึ่งมีประโยชน์สำหรับอุปกรณ์อย่าง Raspberry Pi ที่ไม่มีการระบายความร้อนแบบแอคทีฟ เมตริก Uptime และค่าเฉลี่ยโหลดช่วยประเมินความเสถียรของระบบ และมีบันทึกสดสำหรับการแก้ไขปัญหาบริการหรือปัญหาของระบบ


ส่วน `Support` ให้การเข้าถึงคำถามที่พบบ่อยในตัว เอกสารทางการ และช่องทางสนับสนุนจากชุมชน สามารถดาวน์โหลดบันทึกการดีบักจากส่วนนี้เพื่อแชร์กับการสนับสนุน Start9 เพื่อการแก้ไขปัญหาที่รวดเร็วยิ่งขึ้น


![image](assets/en/11.webp)


## 8️⃣ มาร์เก็ตเพลส


`Marketplace` ใช้เพื่อค้นหา ติดตั้ง และจัดการบริการบนเซิร์ฟเวอร์ส่วนตัว มันให้การเข้าถึงซอฟต์แวร์เช่น Bitcoin Core, BTCPay Server และ electrs StartOS รองรับตลาดหลายแห่ง รวมถึง Start9 Registry อย่างเป็นทางการและรีจิสทรีที่ดำเนินการโดยชุมชน สามารถเพิ่มได้โดยคลิก `CHANGE` และสลับไปที่ `Community Registry` ซึ่งให้การเข้าถึงบริการที่หลากหลายมากขึ้น


![image](assets/en/12.webp)


## 9️⃣ การติดตั้ง Bitcoin Full Node


การติดตั้ง Bitcoin full node บน StartOS มอบอำนาจอธิปไตยเต็มรูปแบบเหนือประสบการณ์ของ Bitcoin มันช่วยให้การตรวจสอบธุรกรรมและเพิ่มความเป็นส่วนตัวและความปลอดภัยโดยการลบการพึ่งพาบริการภายนอกที่อาจบันทึกกิจกรรม การควบคุมธุรกรรมอย่างเต็มที่สามารถทำได้ ทำให้สามารถกระจายไปยังเครือข่ายได้โดยตรง ตัวเลือกเริ่มต้นคือ `Bitcoin Core` ซึ่งผสานรวมกับ StartOS โดยตรงและอนุญาตให้เชื่อมต่อกับกระเป๋าเงินเช่น Specter, Sparrow หรือ Electrum สำหรับการตั้งค่าการดูแลตนเอง อีกทางเลือกหนึ่ง `Bitcoin Knots` ก็มีให้ผ่าน Community Registry เช่นกัน


เมื่อต้องการติดตั้ง Bitcoin Core ให้ไปที่ Marketplace ภายใต้รีจิสทรีเริ่มต้น ค้นหาและติดตั้งบริการ Bitcoin Core หลังการติดตั้ง จะมีข้อความแจ้ง `Needs Config` ปรากฏขึ้น ซึ่งต้องการให้ตั้งค่าก่อนที่บริการจะสามารถทำงานได้ โดยปกติจะเกิดขึ้นหลังจากการอัปเดตหรือการติดตั้งใหม่ และจะมีการแจ้งให้ตรวจสอบ `RPC settings` ดำเนินการด้วยการตั้งค่าเริ่มต้นและคลิก `Save`


![image](assets/en/13.webp)


เมื่อซิงโครไนซ์เสร็จสมบูรณ์ โหนดของคุณสามารถทำหน้าที่เป็นแบ็กเอนด์ส่วนตัวสำหรับกระเป๋าเงินเช่น Sparrow โดยให้ความเป็นส่วนตัวที่เพิ่มขึ้นและการตรวจสอบธุรกรรม อย่างไรก็ตาม สำหรับผู้ใช้ที่เก็บเงินจำนวนมาก เป็นสิ่งสำคัญที่จะต้องเข้าใจการแลกเปลี่ยนด้านความปลอดภัยของการเชื่อมต่อโดยตรงนี้ เมื่อ wallet เชื่อมต่อโดยตรงกับ Bitcoin Core อาจเปิดเผยข้อมูลที่ละเอียดอ่อน เนื่องจาก Bitcoin Core เก็บคีย์สาธารณะ (xpubs) และยอดคงเหลือของ wallet โดยไม่เข้ารหัสบนเครื่องโฮสต์ ระบบที่ถูกบุกรุกอาจทำให้ผู้โจมตีค้นพบทรัพย์สินของคุณและเป็นเป้าหมายคุณได้


เพื่อลดความเสี่ยงนี้และสร้างรูปแบบความปลอดภัยที่แข็งแกร่งยิ่งขึ้น คุณสามารถตั้งค่า Private Electrum Server ได้ เซิร์ฟเวอร์นี้ทำหน้าที่เป็นตัวกลาง โดยทำการจัดทำดัชนีบล็อกเชนโดยไม่เก็บข้อมูลเฉพาะของ wallet การเชื่อมต่อ wallet ของคุณกับเซิร์ฟเวอร์ Electrum ของคุณเองแทนที่จะเชื่อมต่อโดยตรงกับ Bitcoin Core จะช่วยป้องกันไม่ให้ wallet เข้าถึงข้อมูลที่ละเอียดอ่อนของโหนด


![image](assets/en/14.webp)


## 🔟 ตั้งค่า electrs


`electrs` (Electrum Rust Server) เป็นตัวจัดทำดัชนีที่รวดเร็วและมีประสิทธิภาพที่เชื่อมต่อกับโหนด Bitcoin Core ของคุณและช่วยให้กระเป๋าเงินที่เข้ากันได้กับ Electrum สามารถสอบถามประวัติการทำธุรกรรมและยอดคงเหลือได้แบบเรียลไทม์ โดยการรัน electrs บน StartOS คุณจะไม่ต้องพึ่งพาเซิร์ฟเวอร์ Electrum ของบุคคลที่สาม ซึ่งช่วยปรับปรุงความเป็นส่วนตัวและความปลอดภัยอย่างมาก—การสอบถาม wallet ของคุณจะไปยังโหนดที่คุณโฮสต์เองโดยตรง


ในการตั้งค่า ให้ติดตั้งบริการ electrs จาก StartOS Marketplace ก่อน ระบบจะต้องการให้ Bitcoin Core ซิงค์อย่างสมบูรณ์ก่อนดำเนินการต่อ หลังการติดตั้ง ยืนยันการตั้งค่า `Needs Config` ด้วยค่าเริ่มต้นที่แนะนำ และ electrs จะเริ่มทำการจัดทำดัชนีบล็อกเชน ซึ่งอาจใช้เวลาถึงหนึ่งวันขึ้นอยู่กับฮาร์ดแวร์ของคุณ


![image](assets/en/15.webp)


เมื่อเสร็จสิ้น คุณสามารถเชื่อมต่อกระเป๋าเงินเช่น Sparrow หรือ Specter การเชื่อมต่อที่สำเร็จจะทำให้ wallet ของคุณสามารถซิงค์โดยตรงกับโหนดของคุณ มอบประสบการณ์ Bitcoin ที่ปลอดภัย เป็นส่วนตัว และโฮสต์ด้วยตนเอง


## 1️⃣1️⃣ เชื่อมต่อ Sparrow Wallet


ในการเชื่อมต่อ `Sparrow Wallet` กับโหนด StartOS ของคุณโดยใช้การใช้งาน electrs ก่อนอื่นให้แน่ใจว่า Bitcoin Core ซิงค์เสร็จสมบูรณ์และ electrs ได้รับการติดตั้งและทำงานอยู่ เปิด Sparrow Wallet บนอุปกรณ์ของคุณและไปที่ `File` -> `Settings` -> `Server` จากนั้นเลือก `Private Electrum Server` ในช่อง URL ให้ป้อน `Tor hostname` และ `Port` สำหรับ electrs ซึ่งคุณสามารถหาได้ใน StartOS ภายใต้ `Services` -> `electrs` -> `Properties` (โดยทั่วไปจะลงท้ายด้วย `.onion:50001`)


![image](assets/en/16.webp)


ถัดไป เปิดใช้งาน Tor โดยการเลือก `Use Proxy`, ตั้งค่า proxy address เป็น `127.0.0.1` และ port เป็น `9050` คลิก `Test Connection` และรอสักครู่ การเชื่อมต่อที่สำเร็จจะแสดงข้อความยืนยัน เช่น `Connected to electrs` เมื่อยืนยันแล้ว ปิดการตั้งค่าและดำเนินการสร้างหรือกู้คืน wallet ของคุณ การตั้งค่านี้จะทำให้ wallet ของคุณสอบถามข้อมูลจากโหนดของคุณเองผ่าน electrs ซึ่งให้ความเป็นส่วนตัวเต็มรูปแบบและการทำงานที่ไม่ต้องเชื่อถือ


![image](assets/en/17.webp)


## 🎯 Conclusion


StartOS โดย Start9 เป็นแพลตฟอร์มที่ใช้งานง่ายและเน้นความเป็นส่วนตัว ออกแบบมาเพื่อการโฮสต์บริการที่จำเป็นด้วยตัวเอง เช่น Bitcoin และโหนด Lightning, กระเป๋าเงิน, และแอปส่วนตัว มันขจัดความจำเป็นในการใช้ทักษะบรรทัดคำสั่งด้วยการเสนออินเทอร์เฟซกราฟิกที่สะอาด, การสำรองข้อมูลอัตโนมัติ, การตรวจสอบสุขภาพ, และการเข้าถึง Tor ที่ปลอดภัย ทำให้เหมาะสำหรับผู้ใช้ที่ไม่เชี่ยวชาญด้านเทคนิค สถาปัตยกรรมแบบโมดูลาร์ของมันสนับสนุนการผสานรวมอย่างราบรื่นกับเครื่องมือเช่น electrs หรือ Sparrow ให้คุณควบคุมเต็มที่เหนืออธิปไตยทางการเงินและดิจิทัลของคุณ ด้วยการเน้นที่ความโปร่งใส, การควบคุมในท้องถิ่น, และการขยายตัว StartOS ช่วยให้ผู้ใช้สามารถเรียกคืนความเป็นเจ้าของจากแพลตฟอร์มที่รวมศูนย์และดำเนินการโครงสร้างพื้นฐานส่วนตัวที่ยืดหยุ่นของตนเอง
