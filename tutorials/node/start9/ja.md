## 1️⃣ はじめに


### Start9とは何ですか？


Start9は2020年に設立された企業で、個人サーバー向けに設計されたLinuxベースのオペレーティングシステム[**StartOS**,](https://github.com/Start9Labs/start-os)を開発したことで知られている。StartOSは、ユーザーがBitcoinやLightningノード、メッセージング・アプリ、パスワード・マネージャーなど、幅広いソフトウェア・サービスを簡単にセルフ・ホストできるようにする。StartOSは、ユーザーフレンドリーなブラウザベースのインターフェイス、サービスをインストールするための厳選されたマーケットプレイス、Tor統合やシステム全体のHTTPS暗号化などのビルトインプライバシーツールを特徴としています。Start9はまた、OSがプリインストールされたハードウェアデバイスを提供しているが、ソフトウェアは互換性のあるハードウェアや仮想マシン（VM）にインストールすることもできる。


### どのようなオプションがありますか？


Start9は、構築済みとDIYの両方の導入オプションを提供している。Server Oneは**AMD Ryzen 7 5825U**プロセッサーを搭載し、RAM (16GB-64GB)とストレージ (2TB-4TB NVMe SSD)の設定が可能です。Server Pureは**Intel i7-10710U**を搭載し、RAMとストレージの設定が可能です。Start9から直接購入した場合、どちらも**生涯テクニカルサポート**が含まれます。柔軟性を重視するユーザーのために、StartOSはラップトップ、デスクトップ、ミニPC、シングルボードコンピュータなどの幅広い既存のハードウェア、またはVM内に無料でインストールできます。

### アンブレルとの違いは？


StartOSとUmbrelはどちらもセルフホスト型サービスのインストールを簡素化しますが、アーキテクチャと機能が異なります。UmbrelはDebian/UbuntuシステムまたはVM上のアプリケーションレイヤーとして動作しますが、Start9はハードウェアまたはVMへの直接インストールを必要とする専用オペレーティングシステムです。Umbrelは洗練されたmacOSにインスパイアされたインターフェースを特徴としていますが、Start9は機能的で最小限のデザインを優先しています。Umbrelはより多くの[アプリケーションの選択](https://apps.umbrel.com/)を提供し、[Start9 Marketplace](https://marketplace.start9.com/)は高度な技術機能でそれを補います。Start9 は、有効な UI フォームを通じて高度な設定へのアクセスを簡素化し、コマンドラインの必要性を低減します。また、ワンクリックで暗号化されたシステムバックアップによるバックアップ管理にも優れており、Umbrel にはネイティブで欠けている機能です。StartOS は、ビルトイン監視ツールを提供し、ローカルネットワークアクセスに HTTPS 暗号化を適用し、セキュリティを強化します。Umbrel はローカルで暗号化されていない HTTP を使用しますが、どちらのプラットフォームも Tor 経由の安全なリモートアクセスをサポートしています。Umbrelは、豊富なアプリ・エコシステムと洗練されたUIを優先するユーザーに適している。Start9 は、コマンドラインの専門知識を必要とせず、セキュリティ、設定可能性、バックアップの信頼性、および高度なサービス管理を重視するユーザーにとって強力な選択肢です。Umbrel の詳細と Start9 との違いについては、このコースをご覧ください：


https://planb.academy/courses/3cd9cb94-82e8-417a-9c5a-02afc2589426

## 2️⃣ DIYの前提条件：最低＆推奨スペック


最小限のサービスで基本的に使用する場合、**最低スペック**は以下の通りです： **vCPU 1コア（2.0GHz+ブースト）、4GB RAM、64GBストレージ**、イーサネットポートです。とはいえ、特にBitcoinノードを運用するのであれば、それ以上にすることをお勧めします。個人的には1TBで始めたが、すぐに容量が足りなくなった。クアッドコアCPU（2.5GHz以上）**と**8GB以上のRAM**とともに、**少なくとも2TBのストレージ**を目指したほうがいい。パフォーマンスと寿命に大きな違いが出る。StartOSを実行できるハードウェア](https://community.start9.com/t/known-good-hardware-master-list-hardware-capable-of-running-startos/66/229)についての最新コミュニティスレッドがあります。


## 3️⑯ ファームウェアのダウンロードとフラッシュ


セットアップを開始するには、別のコンピュータを使用して [Start9 ウェブサイト](https://start9.com/) にアクセスし、`DOCS` をクリックしてドキュメントセクションに移動します。そこから`Flashing Guides`にアクセスし、StartOSの適切なバージョンを探します。2つのオプションがあります：



- StartOS (Raspberry Pi)
- スタートOS（X86/ARM）


このチュートリアルでは `x86/ARM` オプションについて説明する。


最新のOSバージョンは[Githubリリースページ](https://github.com/Start9Labs/start-os/releases/latest)からダウンロードできます。新機能をテストしたいユーザーのために、[プレリリース](https://github.com/Start9Labs/start-os/releases) バージョンも用意されています。ページ下部の `Assets` から `x86_64` または `x86_64-nonfree.iso` をダウンロードしてください。  x86_64-nonfree.iso`イメージには、Server OneとほとんどのDIYハードウェア、特にグラフィックとネットワークデバイスのサポートに必要なフリーではない（クローズドソース）ソフトウェアが含まれています。


GitHubに掲載されているSHA256ハッシュと照合して、ファイルの完全性を確認することを推奨する。Linuxの場合は、`sha256sum startos-0.3.4.2-efc56c0-20230525_x86_64.iso`コマンドを使用することができます。


StartOSイメージをダウンロードして確認した後、USBドライブにフラッシュする必要があります。この作業には`BalenaEtcher`が推奨される。これはUSBドライブやSDカードにOSイメージファイルを書き込むための無料のオープンソースツールで、Windows、macOS、Linuxで利用できる。Balena Etcher公式サイト](https://www.balena.io/etcher/)から適切なバージョンをダウンロードし、インストーラーを実行してください。ターゲットのUSBドライブやSDカードを接続し、Balena Etcherを開き、`Flash from file`をクリックしてダウンロードしたOSイメージを選択します。Etcherは接続されたドライブを自動的に検出します。複数のドライブが存在する場合は正しいターゲットを選択してください。Flash!`をクリックしてイメージの書き込みを開始します。書き込みが完了すると、Etcherは自動的に書き込みプロセスを検証します。完了後、ドライブを安全に取り外し、デバイスの起動に使用します。

選択肢は2つある：



- フレッシュなスタート
- リカバリー・オプション


サーバーを新規にインストールする場合は、`Start Fresh`を選択する。まず、サーバーの電源とイーサネットケーブルを接続します。セットアップに使用するコンピュータが同じローカルネットワーク上にあることを確認します。新しくフラッシュしたUSBドライブをコンピュータから取り出し、サーバーに挿入します。


同じネットワーク上のどのコンピューターからでもサーバーを遠隔操作することができます。ウェブブラウザを開き、`http://start.local`に移動する。

## 5️⃣ ルート CA を信頼する


証明書をダウンロードした後、サーバーの `Root CA` がオペレーティングシステムから信頼されていなければならない。View Instructions`をクリックし、特定のOSのガイドラインを見つける。


![image](assets/en/07.webp)


Linuxシステムの場合、以下のコマンドを使用する。まず、ターミナルを開き、必要なパッケージをインストールする：


```shell
sudo apt update

sudo apt install -y ca-certificates p11-kit
```


証明書がダウンロードされたディレクトリ（通常は `~/Downloads` ）に移動する。以下のコマンドを実行して、証明書をOSのトラストストアに追加する。cd ~/Downloads` で downloads フォルダに移動する。sudo mkdir -p /usr/share/ca-certificates/start9` で必要なディレクトリを作成する。sudo cp "your-filename.crt" /usr/share/ca-certificates/start9/` を使用して、`your-filename.crt` を実際のファイル名に置き換えて証明書ファイルをコピーする。sudo bash -c "echo 'start9/your-filename.crt' >> /etc/ca-certificates.conf"`でシステム設定にパスを追加して、証明書を恒久的に登録する。最後に、`sudo update-ca-certificates`でトラストストアを再構築する。これらのコマンドを実行する前に、実際の証明書ファイル名を使用し、すべてのパスを確認することが重要である。このプロセスにより、Start9サーバーのHTTPS接続に対する恒久的な信頼が確立される。


インストールに成功すると、`1 added` と出力される。ほとんどのアプリケーションは `https` 経由で安全に接続できるようになる。Firefoxを使用している場合は、追加の[最終ステップ](https://docs.start9.com/0.3.5.x/misc-guides/ca-ff.html#ca-ff)が必要です。ChromeやBraveの場合は、ルートCAを尊重するようにブラウザを設定するために、別の[最終ステップ](https://docs.start9.com/0.3.5.x/misc-guides/ca-chrome.html#ca-chrome)が必要です。ページを更新して接続をテストしてください。問題が解決しない場合は、ページを再度表示する前にブラウザを終了し、再度開いてください。

## 7️⑭システム


StartOS の [システム] タブでは、パーソナル サーバーを管理するためのコア システム機能にアクセスできます。システム メンテナンス、セキュリティ、診断、および構成のためのツールを、コマンドラインの専門知識を必要とせずに提供します。


バックアップ`セクションでは、サービス、設定、データを含む完全なシステムバックアップを作成することができます。これはディザスタリカバリや新しいハードウェアへの移行に不可欠です。バックアップは外付けドライブに保存でき、マスターパスワードを使って暗号化される。


システム] タブの [管理] セクションでは、主要なシステム機能を制御できます。ユーザーは手動で StartOS アップデートをチェックして適用し、システム アップデート プロセスを制御できます。公式マーケットプレイスでは利用できないカスタムサービスやサードパーティサービスをサイドロードすることも可能です。サーバーがイーサネット経由で接続されていない場合、Wi-Fi 設定はこのセクションから構成できます。上級ユーザーは、ターミナルレベルのシステム管理のためにSSHアクセスを有効にすることができます。


![image](assets/en/10.webp)


Insights`セクションはサーバーのパフォーマンスと健全性をリアルタイムで監視し、CPU、RAM、ディスクの使用状況をグラフで表示する。また、システム温度も表示されるため、Raspberry Pi のようにアクティブな冷却機能を持たないデバイスに便利です。稼働時間と負荷平均メトリクスはシステムの安定性を評価するのに役立ち、ライブログはサービスやシステムの問題のトラブルシューティングに利用できます。


サポート`セクションは、組み込みのFAQ、公式ドキュメント、コミュニティサポートチャンネルへのアクセスを提供します。デバッグログはこのセクションからダウンロードすることができ、Start9 のサポートと共有することで、問題を迅速に解決することができます。

## 9️⃣ Bitcoin フルノードのインストール


StartOS に Bitcoin full node をインストールすることで、Bitcoin の体験に対する完全な主権を提供します。トランザクションの検証を可能にし、アクティビティを記録する可能性のある外部サービスへの依存を排除することで、プライバシーとセキュリティを強化します。トランザクションを完全に制御し、ネットワークに直接ブロードキャストすることができます。デフォルトのオプションは`Bitcoin Core`で、StartOSとネイティブに統合され、Specter、Sparrow、またはElectrumのようなウォレットと接続してセルフカストディを設定することができます。代替の`Bitcoin Knots`もコミュニティレジストリを通じて利用可能です。


Bitcoin Coreをインストールするには、Marketplaceに移動します。デフォルトのレジストリで、Bitcoin Core サービスを見つけてインストールする。インストール後、`Needs Config`プロンプトが表示され、サービスを実行する前に設定を完了する必要があります。これは通常、アップデートや新規インストールの後に発生し、`RPC settings`の確認を促します。デフォルトの設定のまま進み、`Save`をクリックしてください。


![image](assets/en/13.webp)


完全に同期されると、あなたのノードはSparrowのようなウォレットのプライベート・バックエンドとして機能し、より強化されたプライバシーとトランザクションの検証を提供することができます。walletがBitcoin Coreに直接接続する場合、機密データが流出する可能性があります。wallet が Bitcoin Core に直接接続する場合、Bitcoin Core は公開鍵（xpub）と wallet の残高をホストマシン上で暗号化せずに保存するため、機密データを公開する可能性があります。BitcoinCoreは公開鍵(xpubs)とwalletの残高をホストマシンに暗号化されずに保存します。


このリスクを軽減し、より強固なセキュリティ・モデルを実現するために、プライベートElectrum Serverを設定することができる。このサーバーは仲介役として機能し、wallet固有の情報を保存することなくブロックチェーンのインデックスを作成します。walletをBitcoin Coreに直接接続するのではなく、自分のElectrumサーバーに接続することで、walletがノードの機密データにアクセスするのを防ぐことができます。


![image](assets/en/14.webp)


## エレクトーンのセットアップ


electrs`（ElectrumRustサーバー）は、Bitcoinコアノードに接続し、Electrum互換ウォレットがリアルタイムで取引履歴と残高を照会できるようにする、高速で効率的なインデクサーです。StartOS上でelectrsを実行することで、サードパーティのElectrumサーバーへの依存を排除し、プライバシーとセキュリティを大幅に改善します。


設定するには、まずStartOS Marketplaceからelectrsサービスをインストールします。システムを進める前に、Bitcoin Coreが完全に同期されている必要があります。インストール後、`Needs Config`設定を推奨デフォルトで確認すると、electrsがブロックチェーンのインデックス作成を開始します。


![image](assets/en/15.webp)


接続が完了すると、Sparrow や Specter などのウォレットに接続できます。接続に成功すると、walletがあなたのノードと直接同期し、安全でプライベートなBitcoin体験を提供します。


## 1️⃣ 1️⃣ コネクト Sparrow Wallet


SparrowのWallet`をelectrsの実装を使用してStartOSノードに接続するには、まずBitcoin Coreが完全に同期され、electrsがインストールされ、実行されていることを確認します。デバイスで Sparrow Wallet を開き、`File` -> `Settings` -> `Server` に移動します。そして`Private Electrum Server`を選択する。URLフィールドにelectrsの`Tor hostname`と`Port`を入力する。electrsはStartOSの`Services` -> `electrs` -> `Properties`にある（通常は末尾が`.onion:50001`）。


![image](assets/en/16.webp)


次に、`Use Proxy`にチェックを入れてTorを有効にし、プロキシアドレスを`127.0.0.1`に、ポートを`9050`に設定する。Test Connection`をクリックし、しばらく待つ。接続に成功すると、`Connected to electrs`のような確認メッセージが表示される。確認ができたら設定を閉じて、walletの作成またはリストアに進みます。この設定により、walletはelectrsを経由して自分のノードに問い合わせを行い、完全なプライバシーと信頼のおける操作を提供します。


![image](assets/en/17.webp)


## 結論


Start9のStartOSは、BitcoinやLightningノード、ウォレット、個人アプリなどの重要なサービスをセルフホストするために設計された、ユーザーフレンドリーでプライバシーに重点を置いたプラットフォームです。クリーンなグラフィカルインターフェース、自動バックアップ、ヘルスモニタリング、セキュアなTorアクセスを提供することで、コマンドラインスキルの必要性を排除し、非技術系ユーザーにとって理想的なものとなっている。そのモジュラーアーキテクチャは、electrsやSparrowのようなツールとのシームレスな統合をサポートし、あなたの財政とデジタル主権を完全に制御することができます。透明性、ローカルコントロール、拡張性に重点を置くStartOSは、ユーザーが中央集権的なプラットフォームから所有権を取り戻し、独自のプライベートで回復力のあるインフラストラクチャを運用することを可能にします。
