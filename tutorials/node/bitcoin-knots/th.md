Bitcoin Knots เป็นการใช้งานทางเลือกของโปรโตคอล Bitcoin ที่ได้มาจาก Bitcoin Core ออกแบบและดูแลโดย Luke Dashjr มันมีคุณสมบัติเพิ่มเติมและการปรับกฎบางอย่างจาก Mempool ในขณะที่ยังคงความเข้ากันได้กับโหนดอื่นๆ ในเครือข่าย Bitcoin Knots ผสานรวม Bitcoin wallet แต่ยังสามารถใช้เป็นโหนด Bitcoin ธรรมดาควบคู่ไปกับซอฟต์แวร์ wallet อื่นๆ ได้เช่นกัน


## ทำไมต้องใช้ Knots แทนที่จะเป็น Core?


ปัจจุบัน Core เป็นการใช้งานส่วนใหญ่ของโปรโตคอล Bitcoin บนเครือข่าย โปรโตคอล Bitcoin เป็นเพียงชุดของกฎ มันต้องการซอฟต์แวร์ในการนำไปใช้ เครื่องที่รันซอฟต์แวร์ที่ใช้งานโปรโตคอล Bitcoin เรียกว่าโหนด และโหนดทั้งหมดนี้รวมกันเป็นเครือข่าย Bitcoin


ตลอดประวัติศาสตร์ของ Bitcoin มีลูกค้าจำนวนมากที่เกิดจากซอฟต์แวร์เริ่มต้นที่พัฒนาโดย Satoshi Nakamoto ปรากฏขึ้น ปัจจุบัน (มีนาคม 2025) Bitcoin Core เป็นส่วนใหญ่ที่ท่วมท้น โดยมีเกือบ 98% ของโหนดบนเครือข่าย Bitcoin ใช้ลูกค้านี้


อย่างไรก็ตาม ซอฟต์แวร์ทางเลือกก็มีให้ใช้งานเช่นกัน ซึ่งไม่ใช่โหนดที่เชื่อมโยงกับ altcoin เช่น Bitcoin Cash แต่เป็นไคลเอนต์ทางเลือกที่เข้ากันได้กับเครือข่าย Bitcoin จริง ในบรรดาเหล่านี้ Bitcoin Knots เป็นที่รู้จักมากที่สุด ปัจจุบันมีสัดส่วนประมาณ 1.4% ของเครือข่าย ลูกค้าทางเลือกอื่น ๆ ยังคงเป็นส่วนน้อยมาก


![Image](assets/fr/01.webp)


มีสองเหตุผลหลักในการใช้ไคลเอนต์ทางเลือกอย่าง Knots แทนที่จะเป็น Core:




- Technical**: ลูกค้าเหล่านี้มักจะเสนอทางเลือกที่แตกต่างจาก Core โดยเฉพาะในแง่ของการจัดการ Mempool โดยการกำหนดว่าธุรกรรมใดจะได้รับการยอมรับและกระจายโดยโหนดของคุณ
- นโยบาย**: บางคนชอบใช้ไคลเอนต์ทางเลือกเช่น Knots ด้วยเหตุผลที่ไม่เกี่ยวกับเทคนิค โดยเฉพาะเพื่อสนับสนุนทางเลือกแทน Core และลดการผูกขาดของมัน หาก Core ถูกประนีประนอม มันจะมีประโยชน์ไม่เพียงแต่มีไคลเอนต์ทางเลือกที่มั่นคงและได้รับการดูแลอย่างดีเท่านั้น แต่ยังรู้วิธีการใช้งานพวกมันด้วย คนอื่นๆ ใช้ Knots เพื่อประท้วง เพราะพวกเขาสูญเสียความเชื่อมั่นในนักพัฒนาของ Core หรือไม่เห็นด้วยกับการจัดการของไคลเอนต์ส่วนใหญ่

## เปิดตัว IBD


ครั้งแรกที่คุณเปิดตัว Bitcoin Knots คุณจะสามารถเลือกไดเรกทอรีท้องถิ่นที่ข้อมูลโหนดของคุณ (รวมถึงชุดและพารามิเตอร์ Blockchain, UTXO) จะถูกจัดเก็บได้


![Image](assets/fr/04.webp)


คุณยังสามารถเลือกที่จะตัดแต่งข้อมูล Blockchain เพื่อเก็บเฉพาะบล็อกที่ใหม่ที่สุดเท่านั้น ตัวเลือกนี้ช่วยให้โหนดของคุณสามารถตรวจสอบแต่ละบล็อกได้อย่างครบถ้วนภายในขีดจำกัดการจัดเก็บที่กำหนด โดยจะค่อยๆ ลบบล็อกที่เก่าที่สุดออกไป หากคุณมีพื้นที่ดิสก์เพียงพอ (ปัจจุบันประมาณ 650 GB แต่ตัวเลขนี้กำลังเพิ่มขึ้น) ให้ปล่อยตัวเลือกนี้ไว้ไม่ต้องเลือก หากพื้นที่ดิสก์ของคุณมีจำกัด ให้เปิดใช้งานการตัดแต่งและระบุความจุสูงสุดที่อนุญาต


โปรดทราบ: หากโหนดของคุณคือ pruned และคุณใช้มันเพื่อซิงโครไนซ์ wallet ที่กู้คืน คุณจะไม่สามารถดึงธุรกรรมก่อนบล็อกที่เก็บไว้ในเครื่องที่เก่าที่สุดได้


![Image](assets/fr/05.webp)


อีกตัวเลือกหนึ่งที่มีให้คือ "*Assume Valid*". มันช่วยเร่งการซิงโครไนซ์เริ่มต้นโดยการข้ามการตรวจสอบลายเซ็นสำหรับธุรกรรมที่รวมอยู่ในบล็อกก่อนหน้าบล็อกที่ระบุ


จุดประสงค์ของ "*Assume Valid*" คือการเร่งการซิงโครไนซ์ครั้งแรกของโหนดโดยไม่ลดความปลอดภัยลงอย่างมีนัยสำคัญ โดยการสมมติว่าธุรกรรมเหล่านี้ได้รับการตรวจสอบอย่างกว้างขวางโดยเครือข่ายล่วงหน้าแล้ว การประนีประนอมที่สำคัญเพียงอย่างเดียวคือโหนดของคุณจะไม่ตรวจจับการขโมยบิตคอยน์ก่อนหน้านี้ แต่ยังคงรับประกันความถูกต้องของจำนวนบิตคอยน์ที่ออกทั้งหมด โหนดของคุณจะตรวจสอบลายเซ็นของธุรกรรมทั้งหมดหลังจากบล็อกที่ระบุ วิธีการนี้อิงตามสมมติฐานว่าธุรกรรมที่ได้รับการยอมรับจากเครือข่ายมาเป็นเวลานานโดยไม่มีการท้าทายมีแนวโน้มที่จะถูกต้อง


ตัวอย่างเช่น ที่นี่ "*Assume Valid*" ถูกตั้งค่าให้บล็อกหมายเลข 855 000 `00000000000000000000000233ea80aa10d38aa4486cd7033fffc2c4df556d0b9138` ซึ่งเผยแพร่เมื่อวันที่ 1 สิงหาคม 2024 ดังนั้นในระหว่าง IBD โหนดของฉันจะเริ่มการตรวจสอบลายเซ็นเต็มรูปแบบจากบล็อกนี้เท่านั้น


![Image](assets/fr/06.webp)


จากนั้นคลิกที่ปุ่ม "*OK*" เพื่อเริ่มต้น *Initial Block Download* คุณจะต้องอดทนในระหว่างการซิงโครไนซ์โหนดครั้งแรก หากคุณต้องการกลับมาซิงโครไนซ์ในภายหลัง เพียงแค่ปิดซอฟต์แวร์และปิดคอมพิวเตอร์ของคุณ การซิงโครไนซ์จะกลับมาโดยไม่มีปัญหาในครั้งถัดไปที่คุณเปิดโปรแกรม


![Image](assets/fr/07.webp)


## กำลังตั้งค่า Bitcoin Knot ของคุณ


คลิกที่แท็บ "*Settings*" จากนั้นเลือก "*Options*"


![Image](assets/fr/08.webp)


ในแท็บ "*Main*" คุณสามารถเข้าถึงพารามิเตอร์หลักของโหนด:




- "*Start...*" จะเปิดใช้งานโหนดโดยอัตโนมัติเมื่อคอมพิวเตอร์ของคุณเริ่มทำงานเพื่อเริ่มการซิงโครไนซ์ทันที;
- "*Prune...*" ปรับขีดจำกัดการจัดเก็บหากคุณเลือกที่จะตัดแต่ง Blockchain ;
- "*Database cache...*" กำหนดจำนวน RAM สูงสุดที่อนุญาตให้กับโหนดของคุณ;
- สุดท้าย ให้เปิดใช้งาน "*Enable RPC server*" หากคุณต้องการเชื่อมต่อโหนด Bitcoin Knots ของคุณกับซอฟต์แวร์ wallet อื่นๆ เช่น Sparrow Wallet หรือ Liana เป็นต้น


![Image](assets/fr/09.webp)


ในแท็บ "*Wallet*" คุณจะพบการตั้งค่าสำหรับ wallet ที่รวมอยู่ซึ่งคุณสามารถสร้างได้ในภายหลังบน Knots ฉันแนะนำให้คุณเปิดใช้งาน RBF และการควบคุมเหรียญ คุณยังสามารถกำหนดประเภทของสคริปต์ที่จะใช้ได้อีกด้วย


![Image](assets/fr/10.webp)


แท็บ "*Network*" ประกอบด้วยพารามิเตอร์เครือข่ายที่คุณสามารถปรับให้เหมาะสมกับความต้องการเฉพาะของคุณได้


![Image](assets/fr/11.webp)


แท็บ "*Mempool*" ให้คุณกำหนดค่า *Memory Pool* นั่นคือการจัดการธุรกรรมที่ยังไม่ได้ยืนยันที่เก็บไว้ในหน่วยความจำ และขนาดสูงสุดที่จัดสรรให้กับฟังก์ชันนี้ (ค่าเริ่มต้นคือ 300 MB)


![Image](assets/fr/12.webp)


แท็บ "การกรองสแปม" เป็นฟีเจอร์ของ Bitcoin Knots ที่นี่คุณจะพบกับการตั้งค่าหลายอย่างที่ช่วยให้คุณเลือกได้ว่าธุรกรรมใดที่คุณจะยอมรับหรือปฏิเสธที่จะเผยแพร่ วัตถุประสงค์หลักคือการจำกัดการใช้งานบางอย่างของ Bitcoin โดยเฉพาะเมตาโปรโตคอล เพื่อที่จะต่อสู้กับการปฏิบัติเหล่านี้ในขณะที่หลีกเลี่ยงการทำให้โหนดของคุณทำงานหนักเกินไป นี่เป็นการเลือกทางการเมือง ขึ้นอยู่กับวิสัยทัศน์ส่วนตัวของคุณเกี่ยวกับ Bitcoin


คุณจะพบพารามิเตอร์คลาสสิกเช่นเดียวกับการกำหนดเกณฑ์ "*Dust*"


อย่างไรก็ตาม พารามิเตอร์เหล่านี้มีผลเฉพาะต่อกฎการมาตรฐานเท่านั้น โหนดของคุณจะยังคงยอมรับธุรกรรมที่ยังไม่ได้ยืนยันเฉพาะเมื่อมันถูกรวมอยู่ในบล็อก เพื่อให้ยังคงเข้ากันได้กับเครือข่าย Bitcoin ที่เหลือ การตั้งค่าเหล่านี้เพียงแค่ปรับเปลี่ยนวิธีที่โหนดของคุณประมวลผลและแจกจ่ายธุรกรรมที่ยังไม่ได้ยืนยันไปยังเพื่อนร่วมเครือข่าย ในทางปฏิบัติ เนื่องจาก Knots อยู่ในกลุ่มน้อย กฎที่ถูกกำหนดโดยค่าเริ่มต้นบน Bitcoin Core จึงเป็นตัวกำหนดมาตรฐานบนเครือข่าย


![Image](assets/fr/13.webp)


แท็บ "*Mining*" ช่วยให้คุณกำหนดค่าการเข้าร่วมที่เป็นไปได้ของโหนดของคุณใน mining หากคุณต้องการเปิดใช้งานฟังก์ชันนี้


![Image](assets/fr/14.webp)


ในที่สุด แท็บ "*Display*" เกี่ยวข้องกับพารามิเตอร์ที่เกี่ยวข้องกับกราฟิก Interface รวมถึงภาษาของซอฟต์แวร์


![Image](assets/fr/15.webp)


## สร้าง Bitcoin wallet


เมื่อการซิงโครไนซ์เริ่มต้นเสร็จสิ้น โหนด Bitcoin Knots ของคุณจะทำงานได้อย่างเต็มที่ ขณะนี้คุณมีตัวเลือกในการเชื่อมต่อโหนดนี้กับซอฟต์แวร์ wallet อื่น ๆ หรือใช้ wallet ในตัวโดยตรง ในการทำเช่นนั้น ให้คลิกที่ปุ่ม "*สร้าง Wallet ใหม่*"


![Image](assets/fr/16.webp)


ตั้งชื่อ wallet ของคุณ คุณยังสามารถป้องกันด้วย passphrase BIP39 โดยคลิกที่ "*เข้ารหัส Wallet*". เมื่อพร้อมแล้ว คลิกที่ปุ่ม "*สร้าง*".


![Image](assets/fr/17.webp)


passphrase BIP39 เป็นรหัสผ่านเสริมที่คุณสามารถเลือกได้อย่างอิสระ นอกเหนือจากวลีช่วยจำของคุณ เพื่อเพิ่มความปลอดภัยให้กับ wallet ของคุณ ก่อนที่จะตั้งค่าฟีเจอร์นี้ เราขอแนะนำอย่างยิ่งให้คุณอ่านบทความต่อไปนี้ ซึ่งอธิบายอย่างละเอียดว่า passphrase ทำงานอย่างไรในทางทฤษฎี และวิธีหลีกเลี่ยงข้อผิดพลาดที่อาจนำไปสู่การสูญเสียบิตคอยน์ของคุณอย่างถาวร:


https://planb.academy/tutorials/wallet/backup/passphrase-a26a0220-806c-44b4-af14-bafdeb1adce7
หากคุณได้เปิดใช้งานตัวเลือก passphrase แล้ว ให้เลือกตัวที่แข็งแกร่งและบันทึกไว้อย่างระมัดระวังในสื่อทางกายภาพที่ปลอดภัยหนึ่งหรือมากกว่า

## เชื่อมต่อโหนดของคุณกับโปรแกรมอื่น


Bitcoin Knots' รวม Interface สำหรับการจัดการ Bitcoin wallet ของคุณอาจไม่ใช่วิธีที่เข้าใจง่ายที่สุด และฟังก์ชันการทำงานยังคงค่อนข้างจำกัด อย่างไรก็ตาม คุณสามารถเชื่อมต่อโหนด Bitcoin Knots ของคุณกับซอฟต์แวร์การจัดการ wallet เฉพาะทางเพื่อเข้าถึงข้อมูล Blockchain Bitcoin ได้อย่างง่ายดายและกระจายธุรกรรมของคุณ


ขั้นตอนจะขึ้นอยู่กับซอฟต์แวร์ที่ใช้ แต่มีสองสถานการณ์หลัก: ไม่ว่า Bitcoin Knots จะติดตั้งบนคอมพิวเตอร์เดียวกับซอฟต์แวร์ wallet ของคุณ หรือมันทำงานบนเครื่องแยกต่างหาก


### ด้วย Bitcoin Knots ในท้องถิ่น :


หาก Bitcoin Knots ถูกติดตั้งบนคอมพิวเตอร์ของคุณ ให้ค้นหาไฟล์ `Bitcoin.conf` ในบรรดาไฟล์ซอฟต์แวร์ หากไฟล์นี้ไม่มีอยู่ คุณสามารถสร้างมันขึ้นมาได้ เปิดไฟล์ด้วยโปรแกรมแก้ไขข้อความและใส่บรรทัดต่อไปนี้:


```ini
server=1
```


จากนั้นบันทึกการเปลี่ยนแปลงของคุณ


คุณยังสามารถทำสิ่งนี้ผ่านกราฟิก Bitcoin-QT ของ Interface โดยไปที่ "*Settings*" > "*Options...*" และเปิดใช้งานตัวเลือก "*Enable RPC server*"


อย่าลืมรีสตาร์ทซอฟต์แวร์หลังจากทำการเปลี่ยนแปลงเหล่านี้


![Image](assets/fr/34.webp)


จากนั้นไปที่ซอฟต์แวร์การจัดการ wallet ของคุณ (เช่น Sparrow Wallet หรือ Liana) และป้อนเส้นทางไปยังไฟล์คุกกี้ของคุณ ซึ่งมักจะอยู่ในโฟลเดอร์เดียวกับ `Bitcoin.conf` ขึ้นอยู่กับระบบปฏิบัติการของคุณ:



| **macOS**   | ~/Library/Application Support/Bitcoin |
| ----------- | ------------------------------------- |
| **Windows** | %APPDATA%\Bitcoin                     |
| **Linux**   | ~/.bitcoin                            |

![Image](assets/fr/35.webp)


ปล่อยพารามิเตอร์อื่น ๆ ไว้ตามค่าเริ่มต้น, URL `127.0.0.1` และพอร์ต `8332`, จากนั้นคลิกที่ "*Test Connection*".


![Image](assets/fr/36.webp)


### ด้วยรีโมท Bitcoin Knots :


หากติดตั้ง Bitcoin Knots บนเครื่องอื่นที่เชื่อมต่อกับเครือข่ายเดียวกัน ให้ค้นหาไฟล์ `Bitcoin.conf` ในบรรดาไฟล์ซอฟต์แวร์ก่อน หากไฟล์นี้ยังไม่มีอยู่ คุณสามารถสร้างมันขึ้นมาได้ เปิดไฟล์นี้ด้วยโปรแกรมแก้ไขข้อความและเพิ่มบรรทัดต่อไปนี้:


```ini
server=1
```


หลังจากแก้ไขไฟล์แล้ว ให้แน่ใจว่าคุณบันทึกไฟล์ในโฟลเดอร์ที่เหมาะสมสำหรับระบบปฏิบัติการของคุณ:



|**macOS**|~/Library/Application Support/Bitcoin|
|---|---|
|**Windows**|%APPDATA%\Bitcoin|
|**Linux**|~/.bitcoin|

การดำเนินการนี้สามารถทำได้ผ่านกราฟิก Bitcoin-QT ของ Interface ไปที่เมนู "*Settings*" จากนั้นเลือก "*Options...*" และเปิดใช้งานตัวเลือก "*Enable RPC server*" โดยการทำเครื่องหมายในช่องที่สอดคล้องกัน หากไฟล์ `Bitcoin.conf` ไม่มีอยู่ คุณสามารถสร้างไฟล์ได้โดยตรงจาก Interface นี้โดยคลิกที่ "*Open Configuration File*"


![Image](assets/fr/37.webp)


ค้นหาที่อยู่ IP ของเครื่องที่โฮสต์ Bitcoin Knots บนเครือข่ายท้องถิ่นของคุณ ในการทำเช่นนี้ คุณสามารถใช้เครื่องมือเช่น [Angry IP Scanner](https://angryip.org/) สมมติว่าเพื่อประโยชน์ในการโต้แย้ง ที่อยู่ IP ของโหนดของคุณคือ `192.168.1.18`


ในไฟล์ `Bitcoin.conf` เพิ่มบรรทัดต่อไปนี้ โดยตั้งค่า `rpcbind=192.168.1.18` ให้ตรงกับที่อยู่ IP ของโหนดของคุณ


```ini
[main]
rpcbind=127.0.0.1
rpcbind=192.168.1.18
rpcallowip=127.0.0.1
rpcallowip=192.168.1.0/24
```


![Image](assets/fr/38.webp)


เพิ่มชื่อผู้ใช้และรหัสผ่านสำหรับการเชื่อมต่อระยะไกลไปยังไฟล์ `Bitcoin.conf` อย่าลืมแทนที่ `loic` ด้วยชื่อผู้ใช้ของคุณและ `my_password` ด้วยรหัสผ่านที่แข็งแกร่ง:


```ini
rpcuser=loic
rpcpassword=my_password
```


![Image](assets/fr/39.webp)


หลังจากแก้ไขและบันทึกไฟล์แล้ว ให้รีสตาร์ท Bitcoin Knots.


คุณสามารถไปที่ซอฟต์แวร์การจัดการ wallet ของคุณ (เช่น Sparrow Wallet หรือ Liana) บน Sparrow ไปที่แท็บ "*User / Pass*" ป้อนชื่อผู้ใช้และรหัสผ่านที่คุณได้กำหนดค่าไว้ในไฟล์ `Bitcoin.conf` ปล่อยให้พารามิเตอร์อื่นๆ เป็นค่าเริ่มต้น เช่น URL `127.0.0.1` และพอร์ต `8332` จากนั้นคลิกที่ "*Test Connection*"


![Image](assets/fr/40.webp)


การเชื่อมต่อได้รับการจัดตั้งขึ้นแล้ว


ตอนนี้คุณรู้ทุกอย่างเกี่ยวกับการใช้งาน Bitcoin Knots ทางเลือกแล้ว
