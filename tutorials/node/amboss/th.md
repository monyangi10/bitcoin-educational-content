Lightning Network เป็นเลเยอร์ของโปรโตคอล Bitcoin ซึ่งพัฒนาขึ้นมาเพื่อส่งเสริมการนำการชำระเงิน Bitcoin มาใช้ในชีวิตประจำวันโดยการเพิ่มความเร็วในการประมวลผลของแต่ละธุรกรรม โดยอิงตามหลักการของการกระจายศูนย์ Lightning Network ประกอบด้วยโหนด (คอมพิวเตอร์ที่รันการใช้งาน Lightning) ที่สื่อสารแบบเพียร์ทูเพียร์ ส่งต่อข้อมูล (การชำระเงินและการยืนยันการชำระเงิน) ให้กันและกัน



https://planb.academy/tutorials/node/lightning-network/lightning-network-daemon-linux-59d777e9-72c8-4b32-8c50-e86cdae8f2f9

เช่นเดียวกับในเชนหลัก มันกลายเป็นสิ่งสำคัญที่จะทำให้ผู้ใช้สามารถทราบข้อมูลและสถานะของเครือข่าย เพื่ออำนวยความสะดวกในการเชื่อมต่อระหว่างโหนดและลดปัญหาสภาพคล่องที่มักเกิดขึ้นในเครือข่าย อันที่จริง บน Lightning Network เราแนะนำการชำระเงินขนาดเล็กที่มีจำนวนค่อนข้างน้อยกว่าการทำธุรกรรมบนเชนหลัก Bitcoin



สิ่งสำคัญที่ควรทราบคือไม่ใช่ทุกโหนด Lightning ที่พร้อมใช้งานบนแพลตฟอร์ม Amboss



เหมือนกับ [Mempool Space](https://Mempool.space) ซึ่งให้ข้อมูลที่เป็นประโยชน์เกี่ยวกับเครือข่ายหลักของโปรโตคอล Bitcoin ตั้งแต่ปี 2022 [Amboss](https://amboss.space) ให้ข้อมูลเกี่ยวกับ :





- โหนดบน Lightning Network
- ช่องทางการชำระเงินและความสามารถในการชำระเงินของพวกเขา
- วิวัฒนาการของ Lightning Network ตลอดเวลา
- สถิติการเรียกเก็บเงินไปยังโหนดรีเลย์สำหรับการชำระเงินของคุณ



https://planb.academy/tutorials/privacy/explorer/mempool-space-f3e468a1-92f1-43ce-b2e4-c3298fa0e02f

ในบทแนะนำนี้ เราจะพาคุณทัวร์แพลตฟอร์มนี้ ซึ่งเป็นแหล่งข้อมูลที่จำเป็นสำหรับผู้ใช้ Lightning Network ผู้ที่ต้องการเชื่อมต่อโหนดของตนเพื่อขยายเครือข่าย เป็นต้น




## ค้นหาคู่



หนึ่งในเป้าหมายของแพลตฟอร์ม Amboss คือการทำให้โหนดต่าง ๆ ในเครือข่ายสามารถเชื่อมต่อและสื่อสารข้อมูลระหว่างกันได้ บนหน้าแรกของแพลตฟอร์ม คุณสามารถค้นหาชื่อของโหนดที่คุณรู้จักอยู่แล้วโดยตรง หรือค้นหาโหนดจากกระเป๋าเงิน Lightning ที่คุณใช้บ่อยที่สุด

บนหน้าแรก คุณจะพบโหนดที่จัดประเภทตาม:




- วิวัฒนาการของความจุ: ปริมาณของ Bitcoin ที่เชื่อมโยงกับคีย์สาธารณะของโหนดและปริมาณทั้งหมดที่มีอยู่ในช่องทางทั้งหมดของมัน
- วิวัฒนาการของช่องทาง: จำนวนการเชื่อมต่อใหม่กับโหนดอื่นในเครือข่าย
- ความนิยมของโหนด: ความถี่ที่โหนดถูกใช้งานบ่อยแค่ไหน



![nodes](assets/fr/03.webp)



การเลือกโหนดที่เหมาะสมในการเชื่อมต่อจึงขึ้นอยู่กับการตรวจสอบเกณฑ์ต่อไปนี้:





- ตรวจสอบให้แน่ใจว่าโหนดมีปริมาณบิตคอยน์เพียงพอ; ยิ่งความจุของโหนดมากเท่าใด คุณก็สามารถจ่ายบิตคอยน์ได้มากเท่านั้น





- ตรวจสอบให้แน่ใจว่าโหนดมีการเชื่อมต่อจำนวนมากและเปิดช่องทางกับโหนดอื่นๆ ในเครือข่าย





- ตรวจสอบให้แน่ใจว่าโหนดนั้นยังคงใช้งานอยู่และยังคงได้รับการยอมรับจากเพื่อนๆ โดยการตรวจสอบจำนวนช่องทางใหม่; ยิ่งโหนดนี้มีช่องทางใหม่เปิดมากเท่าไหร่ ก็ยิ่งได้รับการยอมรับจากโหนดอื่นๆ ในเครือข่ายมากขึ้นเท่านั้น



เมื่อคุณพบโหนดที่ถูกต้องแล้ว ให้คลิกที่ชื่อของมันเพื่อไปยังหน้าข้อมูลโหนด



บน Interface นี้ โดยการตรวจสอบเวลาประทับของช่องที่สร้างขึ้นใหม่ คุณจะได้รับเบาะแสเกี่ยวกับกิจกรรมของโหนดนี้ คุณยังจะพบข้อมูลเกี่ยวกับความจุของช่องของโหนดนี้: ข้อมูลนี้มีความสำคัญหากคุณต้องการใช้โหนดนี้ในการชำระเงินของคุณอย่างจริงจัง




![node_info](assets/fr/04.webp)



ในส่วนด้านซ้าย คุณจะพบรายละเอียดเพิ่มเติมเกี่ยวกับตำแหน่งของโหนดนี้ ตัวอย่างเช่น คุณสามารถดู:




- คีย์สาธารณะ: ตัวระบุที่อยู่ด้านล่างชื่อโหนด
- ตำแหน่งทางภูมิศาสตร์ของโหนดนี้




![channels](assets/fr/05.webp)



Interface นี้บอกที่อยู่การเชื่อมต่อสำหรับโหนดนี้: มีรูปแบบเป็น `pubkey@ip:port` ในตัวอย่างของเรา เราต้องการเชื่อมต่อกับโหนดที่:




- คีย์สาธารณะ `pubkey` คือ: `035e4ff418fc8b5554c5d9eea66396c227bd429a3251c8cbc711002ba215bfc226`
- ที่อยู่ iP: `170.75.163.209`
- พอร์ต: `9735`



![geoinfo](assets/fr/06.webp)



ในส่วนของ **Channels** คุณจะเห็นรายการของช่องทางที่เปิดอยู่และการเชื่อมต่อของโหนดกับโหนดอื่นๆ ในเครือข่าย บน Interface นี้ มีข้อมูลหลายชิ้นที่สำคัญในการยืนยันว่าโหนดนี้ตรงตามความต้องการของเราหรือมีความน่าเชื่อถือ:





- Incoming ratio**: จำนวนที่โหนดจะเรียกเก็บจากคุณสำหรับทุก ๆ ล้าน Satoshi ที่ได้รับ ขึ้นอยู่กับช่องทางที่เลือก
- อัตราส่วน (ส่วนในล้าน)** : ซึ่งแสดงถึงจำนวน Satoshi ต่อหนึ่งล้านหน่วยที่โหนดจะเรียกเก็บจากคุณเมื่อคุณตัดสินใจที่จะชำระเงินผ่านช่องทางใดช่องทางหนึ่งของมัน สมมติว่าคุณตัดสินใจที่จะชำระเงิน `10_000 Sats` ผ่านช่องทางที่มีอัตราส่วน ppm เป็น `500 Sats` คุณจะต้องจ่ายให้โหนด `10_000 * 500 / 1_000_000` satoshis ซึ่งเทียบเท่ากับ `5 Sats`
- [HTLC](https://planb.academy/resources/glossary/htlc) สูงสุด** : จำนวนสูงสุดที่โหนดนี้อนุญาตให้คุณส่งผ่านหนึ่งในช่องทางเหล่านี้



โดยการปรึกษาตารางใน Interface นี้ คุณยังสามารถค้นหาข้อมูลทั้งหมดนี้บนโหนดที่ตรงกันได้



![channels_info](assets/fr/07.webp)



ในส่วนของ **แผนที่ช่องสัญญาณ** คุณสามารถดูการกระจายและความจุของช่องสัญญาณต่างๆ บนโหนดนี้ได้ คุณสามารถเปลี่ยนเกณฑ์การกระจายที่แสดงได้โดยเลือกหนึ่งในตัวเลือกจากรายการแบบเลื่อนลงทางด้านขวา



![cha_maps](assets/fr/08.webp)



ส่วน **ช่องทางที่ปิดแล้ว** จะจัดกลุ่มช่องทางเดิมทั้งหมดของโหนดตามประเภทของการปิด:




- การปิดร่วมกัน**: แสดงถึงข้อตกลงของทั้งสองฝ่ายที่ใช้กุญแจส่วนตัวของพวกเขาในการลงนามธุรกรรมที่ระบุการปิดช่องทางและการกระจายยอดคงเหลือภายใน
- การ **ปิดบังคับ**: หมายถึงการปิดช่องทางด้านหนึ่งอย่างกะทันหันและฝ่ายเดียว การปิดประเภทนี้ไม่แนะนำ เนื่องจาก Lightning Network เป็นโปรโตคอลที่มีพื้นฐานจากการลงโทษ: เมื่อคุณพยายามฉ้อโกงยอดคงเหลือของช่องทาง คุณเสี่ยงที่จะสูญเสียยอดคงเหลือทั้งหมดที่มีในช่องทางนั้น



![closed](assets/fr/09.webp)



รับข้อมูลเกี่ยวกับค่าธรรมเนียมการขนส่งสำหรับการส่งผ่านการชำระเงินของคุณผ่านช่องทางบนโหนดที่คุณใช้



![fees](assets/fr/10.webp)



## ข้อมูลเครือข่าย



Amboss มุ่งเน้นไม่เพียงแค่ข้อมูลสมาชิกเครือข่ายเท่านั้น แต่ยังรวมถึงสถานะของเครือข่ายเองด้วย



ในส่วน **สถิติ** ภายใต้เมนู "การจำลอง" ทางด้านซ้าย คุณจะพบกราฟที่แสดงความน่าจะเป็นของการชำระเงินที่สำเร็จเป็นฟังก์ชันของจำนวนเงินที่ชำระ



ในความเป็นจริง คุณจะสังเกตเห็นว่าเส้นโค้งกำลังลดลง เพราะเมื่อจำนวนเงินที่คุณจ่ายเพิ่มขึ้น โอกาสที่การชำระเงินนั้นจะผ่านก็จะน้อยลง นี่สะท้อนถึงปัญหาสภาพคล่องที่แท้จริงบน Lightning Network ตัวอย่างเช่น การชำระเงินของคุณที่ `10_000` ซาโตชิ มีโอกาส `79%` ที่จะสำเร็จ ในทางกลับกัน สำหรับการชำระเงิน `3_000_000` ซาโตชิ คุณมีโอกาสสำเร็จน้อยกว่า `13%`



![network](assets/fr/11.webp)



เมนู **สถิติของเครือข่าย** ช่วยให้คุณสามารถแสดงสถิติในรูปแบบกราฟิกสำหรับ :




- ช่องทางการชำระเงิน
- โหนด
- ความจุ
- ค่าธรรมเนียม
- วิวัฒนาการของช่องทาง



![network_stat](assets/fr/12.webp)



ในเมนู **สถิติตลาด** ตัวเลือก **รายละเอียดคำสั่งซื้อ** ช่วยให้คุณดูความต้องการสภาพคล่องบน Lightning Network กราฟนี้ยังสามารถแสดงให้เห็นว่าช่องทางใดมีความต้องการมากที่สุดและ/หรือช่องทางใดมีความจุที่มากพอสมควร



![markets](assets/fr/13.webp)




## เครื่องมือ



Amboss มีเครื่องมือหลายอย่างที่จะช่วยให้คุณปรับปรุงการค้นหาและการดำเนินการของคุณให้มีประสิทธิภาพมากขึ้น



### Lightning Network decoder



เครื่องมือนี้มีหน้าที่หลักในการให้รายละเอียดเกี่ยวกับโครงสร้างของใบแจ้งหนี้ Lightning, ที่อยู่ Lightning หรือ URL ของ Lightning



ในหน้าแรก ในส่วนของ **เครื่องมือ** ให้ส่งที่อยู่ Lightning ของคุณ ตัวอย่างเช่น



![decoder](assets/fr/14.webp)



จากตัวถอดรหัสนี้ คุณสามารถรับข้อมูลเช่น :




- คีย์สาธารณะของโหนดที่เชื่อมโยงกับที่อยู่ Lightning ของคุณ;
- เวลาหมดอายุของใบแจ้งหนี้จากที่อยู่ของคุณ;
- จำนวนขั้นต่ำและสูงสุดที่คุณสามารถส่งได้;
- โหนด Nostr ที่เชื่อมโยงกับที่อยู่ของคุณ หาก Nostr ถูกเปิดใช้งานบนโหนดนี้



![decode](assets/fr/15.webp)



### Magma IA



ค้นพบเครื่องมือใหม่ล่าสุดที่เปิดตัวโดย Amboss เพื่อจัดการการเชื่อมต่อของคุณกับโหนด Lightning Network อย่างมีประสิทธิภาพ Magma AI ใช้ปัญญาประดิษฐ์เฉพาะทางเพื่อมุ่งเน้นไปที่ปัญหาสำคัญ: การเลือกโหนดที่ดีในการเชื่อมต่อด้วย



![magma](assets/fr/16.webp)
