การติดตั้ง Bitcoin Core บนคอมพิวเตอร์ทั่วไปของคุณสามารถทำได้ แต่ไม่ใช่ตัวเลือกที่ดีที่สุด หากคุณไม่รังเกียจที่จะเปิดคอมพิวเตอร์ตลอด 24 ชั่วโมงทุกวัน นี่ก็จะทำงานได้ดี แต่ถ้าคุณจำเป็นต้องปิดคอมพิวเตอร์ มันจะน่ารำคาญที่ต้องรอให้ซอฟต์แวร์ซิงค์ทุกครั้งที่คุณเปิดเครื่องใหม่


คำแนะนำเหล่านี้สำหรับผู้ใช้ Mac หรือ Windows ผู้ใช้ Linux อาจไม่ต้องการความช่วยเหลือจากฉัน แต่คำแนะนำสำหรับ Linux นั้นคล้ายกับ Mac มาก


## เริ่มต้นใหม่


ในอุดมคติ คุณต้องการใช้คอมพิวเตอร์ที่สะอาด ปราศจากมัลแวร์ แม้ว่าคุณจะใช้ฮาร์ดแวร์ wallet มัลแวร์ก็ยังสามารถหลอกเอาเหรียญของคุณได้


คุณสามารถล้างข้อมูลคอมพิวเตอร์เก่าให้สะอาด และใช้เป็นคอมพิวเตอร์ Bitcoin โดยเฉพาะ หรือซื้อคอมพิวเตอร์/แล็ปท็อปเฉพาะทางก็ได้


## ฮาร์ดไดรฟ์


Bitcoin Core จะใช้พื้นที่ประมาณ 400 กิกะไบต์บนไดรฟ์ของคุณ และจะเพิ่มขึ้นเรื่อยๆ คุณสามารถใช้ไดรฟ์ภายในได้ แต่คุณก็สามารถต่อฮาร์ดไดรฟ์ภายนอกได้เช่นกัน ฉันจะอธิบายทั้งสองตัวเลือกนี้ อุดมคติแล้ว คุณควรใช้ไดรฟ์แบบโซลิดสเตต หากคุณมีคอมพิวเตอร์เก่า มันอาจจะไม่มีไดรฟ์แบบนี้ภายใน แค่ซื้อ SSD ภายนอกขนาด 1 หรือ 2 เทราไบต์มาใช้ ไดรฟ์แบบปกติอาจจะใช้งานได้ แต่คุณอาจจะเจอปัญหาและมันจะช้ากว่ามาก

## ยืนยันการดาวน์โหลด (ส่วนที่ 1)


คุณต้องการไฟล์ที่มี hashes ของการเปิดตัวต่างๆ ไฟล์นี้เคยอยู่ในหน้าดาวน์โหลดของ bitcoin.org แต่ตอนนี้ย้ายไปที่ bitcoincore.org/en/download:


![image](assets/fr/05.webp)


คุณต้องการไฟล์ SHA256 binary hashes ไฟล์นี้ประกอบด้วย SHA256 hashes ของแพ็คเกจดาวน์โหลดต่างๆ ของ Bitcoin Core.


ถัดไป เราจำเป็นต้อง hash ดาวน์โหลดแกน Bitcoin และเปรียบเทียบกับสิ่งที่ไฟล์บอกว่า hash ควรจะเป็น จากนั้นเราจะรู้ว่าการดาวน์โหลดนั้นเหมือนกับที่คาดไว้ ตามที่ bitcoincore.org กล่าว


ไปที่ไดเรกทอรี Downloads อีกครั้งและรันคำสั่งนี้ (แทนที่ X ด้วยชื่อไฟล์ดาวน์โหลด full node bitcoin อย่างถูกต้อง):


```bash
shasum -a 256 XXXXXXXXXXXX # <--- FOR MAC
certutil -hashfile XXXXXXXXXXX SHA256 # <--- FOR WINDOWS
```


คุณจะได้รับเอาต์พุต hash จดบันทึกไว้ และเปรียบเทียบกับ hash ที่อยู่ในไฟล์ SHA256SUMS


หากผลลัพธ์เหมือนกัน แสดงว่าคุณได้ยืนยันแล้วว่าไม่มีบิตของข้อมูลถูกดัดแปลง...เกือบจะใช่ เรายังต้องมั่นใจว่าไฟล์ SHA256SUMS ไม่เป็นอันตราย


ในการดำเนินการขั้นตอนถัดไป เราต้องติดตั้ง gpg บนคอมพิวเตอร์ของเรา


ในการทำเช่นนั้น ให้ดูคู่มือ SHA256/gpg ของฉัน และเลื่อนประมาณครึ่งทางไปยังส่วน “Download gpg” และมองหาหัวข้อย่อยของระบบปฏิบัติการของคุณ จากนั้นกลับมาที่นี่


## รับคีย์สาธารณะ


กลับไปที่หน้าดาวน์โหลด รับไฟล์ลายเซ็น SHA256 hash


![image](assets/fr/06.webp)


คลิกที่นี่และบันทึกไฟล์ลงในดิสก์ โดยควรบันทึกในไดเรกทอรีดาวน์โหลด


ไฟล์นี้มีลายเซ็นของบุคคลต่างๆ ของไฟล์ SHA256SUMS


เราต้องการคีย์สาธารณะของนักพัฒนาหลัก, Wladimir J. van der Laan ในพวงกุญแจของคอมพิวเตอร์ของเรา รหัสคีย์สาธารณะของเขาคือ:

1 - 01EA 5486 DE18 A882 D4C2 6845 90C8 019E 36C2 E964


คัดลอกข้อความนั้นลงในคำสั่งต่อไปนี้:


```bash
gpg --keyserver hkp://keyserver.ubuntu.com --recv-keys 01EA5486DE18A882D4C2684590C8019E36C2E964
```


ด้วยความสนใจ คุณสามารถดูว่ามีคีย์อะไรอยู่ในพวงกุญแจของคอมพิวเตอร์ได้ตลอดเวลาด้วยคำสั่งนี้:


```bash
gpg --list-keys
```


## ยืนยันการดาวน์โหลด (ส่วนที่ 2)


เรามีคีย์สาธารณะแล้ว ดังนั้นเราสามารถตรวจสอบไฟล์ SHA256SUMS ซึ่งมี hashes ของการดาวน์โหลด Bitcoin Core และลายเซ็นสำหรับ hashes เหล่านั้นได้แล้ว


เปิด Terminal หรือ CMD อีกครั้ง และตรวจสอบให้แน่ใจว่าคุณอยู่ในไดเรกทอรี Downloads จากนั้นให้รันคำสั่งนี้:


```bash
gpg –verify SHA256SUMS.asc SHA256SUMS
```


ไฟล์ที่ระบุเป็นไฟล์แรกคือการสะกดชื่อไฟล์ลายเซ็นอย่างถูกต้อง ไฟล์ที่ระบุเป็นไฟล์ที่สองควรเป็นการสะกดชื่อไฟล์ข้อความที่มี hashes อย่างถูกต้อง ทั้งสองไฟล์ควรอยู่ในไดเรกทอรีเดียวกันและคุณต้องอยู่ในไดเรกทอรีของไฟล์เหล่านั้น มิฉะนั้นคุณจะต้องพิมพ์เส้นทางเต็มสำหรับแต่ละไฟล์


นี่คือผลลัพธ์ที่คุณควรได้รับ


![image](assets/fr/07.webp)


เป็นการปลอดภัยที่จะเพิกเฉยต่อข้อความเตือน – นั่นเป็นเพียงการเตือนคุณว่าคุณยังไม่ได้พบกับ Wladimir ในส่วนสำคัญและถามเขาเป็นการส่วนตัวว่าคีย์สาธารณะของเขาคืออะไร จากนั้นบอกคอมพิวเตอร์ของคุณให้เชื่อถือคีย์นี้อย่างสมบูรณ์


หากคุณได้รับข้อความนี้ คุณจะทราบว่าไฟล์ SHA256SUMS.asc ไม่ได้ถูกดัดแปลงหลังจากที่ Wladimir ลงนามแล้ว

ที่นี่คุณสามารถเลือกใช้เส้นทางเริ่มต้นสำหรับที่ที่ blockchain จะถูกดาวน์โหลดไป หรือคุณสามารถเลือกไดรฟ์ภายนอกของคุณได้ ฉันแนะนำว่าไม่ควรเปลี่ยนเส้นทางเริ่มต้นหากคุณจะใช้ไดรฟ์ภายใน เพราะมันทำให้การตั้งค่าง่ายขึ้นเมื่อคุณติดตั้งซอฟต์แวร์อื่นเพื่อสื่อสารกับ Bitcoin Core


คุณสามารถเลือกที่จะรันโหนด pruned ได้ มันช่วยประหยัดพื้นที่ แต่จะจำกัดสิ่งที่คุณสามารถทำได้กับโหนดของคุณ ไม่ว่าจะอย่างไรก็ตาม คุณจะต้องดาวน์โหลด blockchain ทั้งหมดและตรวจสอบมันอยู่ดี ดังนั้นถ้าคุณมีพื้นที่ ให้เก็บสิ่งที่คุณดาวน์โหลดไว้ และอย่าตัดแต่งถ้าคุณสามารถหลีกเลี่ยงได้


เมื่อคุณยืนยันแล้ว blockchain จะเริ่มดาวน์โหลด ซึ่งจะใช้เวลาหลายวัน
