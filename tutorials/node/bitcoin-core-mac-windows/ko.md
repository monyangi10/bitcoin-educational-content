일반 컴퓨터에 Bitcoin core를 설치하는 것도 가능하지만 이상적이지는 않습니다. 컴퓨터를 24시간 내내 켜두어도 괜찮다면 이 방법도 괜찮습니다. 컴퓨터를 꺼야 하는 경우 컴퓨터를 다시 켤 때마다 소프트웨어가 동기화될 때까지 기다리는 것이 번거롭습니다.


이 안내는 Mac 또는 Windows 사용자를 위한 것입니다. Linux 사용자는 제 도움이 필요하지 않을 가능성이 높지만 Linux용 지침은 Mac과 매우 유사합니다.


## 정리 시작


이상적으로는 멀웨어가 없는 깨끗한 컴퓨터를 사용하는 것이 좋습니다. Hardware Wallet를 사용하더라도 멀웨어에 속아 코인을 빼앗길 수 있습니다.


기존 컴퓨터를 깨끗이 닦아내고 Bitcoin 전용 컴퓨터로 사용하거나 전용 컴퓨터/노트북을 구입할 수 있습니다.


## Hard 드라이브


Bitcoin core는 드라이브에서 약 400기가바이트의 데이터를 차지하며 계속 늘어날 것입니다. 내장 드라이브를 사용할 수도 있지만 외장 Hard 드라이브를 연결할 수도 있습니다. 두 가지 옵션 모두 설명해 드리겠습니다. 이상적으로는 솔리드 스테이트 드라이브를 사용하는 것이 좋습니다. 구형 컴퓨터를 사용 중이라면 내부에 이러한 드라이브가 없을 수도 있습니다. 1테라바이트 또는 2테라바이트 외장형 SSD를 구입하여 사용하세요. 일반 드라이브도 작동할 수 있지만 문제가 발생할 수 있고 속도가 훨씬 느릴 수 있습니다.

## 다운로드 확인(1부)


다양한 릴리스의 해시가 포함된 파일이 필요합니다. 이 파일은 Bitcoin.org의 다운로드 페이지에 있었지만 이제 bitcoincore.org/en/download로 이동했습니다:


![image](assets/fr/05.webp)


SHA256 바이너리 해시 파일이 필요합니다. 이 파일에는 Bitcoin core의 다양한 다운로드 패키지의 SHA256 해시가 포함되어 있습니다.


다음으로, Bitcoin core 다운로드를 Hash로 변환하고 파일에 표시된 것과 비교해야 합니다. 그러면 bitcoincore.org에 따르면 다운로드가 예상과 동일하다는 것을 알 수 있습니다.


다운로드 디렉터리로 다시 이동하여 이 명령을 실행합니다(X를 Full node Bitcoin 다운로드 파일 이름으로 정확히 바꿉니다):


```bash
shasum -a 256 XXXXXXXXXXXX # <--- FOR MAC
certutil -hashfile XXXXXXXXXXX SHA256 # <--- FOR WINDOWS
```


Hash 출력을 받게 됩니다. 이를 메모해 두었다가 SHA256SUMS 파일에 포함된 Hash과 비교합니다.


출력이 동일하다면, 데이터 비트가 변조되지 않았다는 것을 거의 확인하신 것입니다. 여전히 SHA256SUMS 파일이 악의적이지 않은지 확인해야 합니다.


다음 단계를 진행하려면 컴퓨터에 gpg가 설치되어 있어야 합니다.


그러려면 SHA256/gpg 가이드를 참조하여 'gpg 다운로드' 섹션까지 절반 정도 스크롤한 후 운영 체제의 소제목을 찾으세요. 그런 다음 여기로 돌아옵니다.


## 공개 키 받기


다운로드 페이지로 돌아가서 SHA256 Hash 서명 파일을 받습니다


![image](assets/fr/06.webp)


파일을 클릭하고 디스크(가급적 다운로드 디렉터리)에 저장합니다.


이 파일에는 SHA256SUMS 파일에 대한 다양한 사람들의 서명이 포함되어 있습니다.


컴퓨터의 키고리에 있는 수석 개발자의 공개 키인 Wladimir J. van der Laan이 필요합니다. 그의 공개 키 ID는 다음과 같습니다:

1 - 01EA 5486 DE18 A882 D4C2 6845 90C8 019E 36C2 E964


해당 텍스트를 다음 명령에 복사합니다:


```bash
gpg --keyserver hkp://keyserver.ubuntu.com --recv-keys 01EA5486DE18A882D4C2684590C8019E36C2E964
```


이 명령으로 언제든지 컴퓨터의 키링에 어떤 키가 있는지 확인할 수 있습니다:


```bash
gpg --list-keys
```


## 다운로드 확인(2부)


공개키가 있으므로 이제 Bitcoin core 다운로드의 해시가 포함된 SHA256SUMS 파일과 해당 해시에 대한 서명을 확인할 수 있습니다.


터미널 또는 CMD를 다시 열고 다운로드 디렉터리에 있는지 확인합니다. 거기에서 다음 명령을 실행합니다:


```bash
gpg –verify SHA256SUMS.asc SHA256SUMS
```


첫 번째 나열된 파일은 서명 파일의 정확한 철자법입니다. 두 번째로 나열된 파일은 해시가 포함된 텍스트 파일의 정확한 철자이어야 합니다. 두 파일은 모두 같은 디렉터리에 있어야 하며, 그렇지 않은 경우 각 파일의 전체 경로를 입력해야 합니다.


출력은 다음과 같습니다


![image](assets/fr/07.webp)


경고 메시지는 무시해도 안전합니다. 이는 단지 주요 부분에서 블라디미르를 만나 공개 키가 무엇인지 개인적으로 물어본 다음 컴퓨터에 이 키를 완전히 신뢰하라고 말한 것을 상기시켜주는 것입니다.


이 메시지가 표시되었다면 Wladimir가 서명한 후 SHA256SUMS.asc 파일이 변조되지 않았다는 것을 알 수 있습니다.

여기에서 Blockchain을 다운로드할 기본 경로를 사용하거나 외장 드라이브를 선택할 수 있습니다. 내장 드라이브를 사용하려는 경우 기본 경로를 변경하지 않는 것이 좋습니다. 그래야 Bitcoin core와 통신하기 위해 다른 소프트웨어를 설치할 때 설정하기가 더 쉬워집니다.


pruned 노드를 실행하도록 선택하면 공간을 절약할 수 있지만 노드로 수행할 수 있는 작업이 제한됩니다. 어느 쪽을 선택하든 Blockchain 전체를 다운로드하여 확인하게 되므로 공간이 있다면 다운로드한 내용을 그대로 유지하고 피할 수 있다면 가지치기를 하지 마세요.
