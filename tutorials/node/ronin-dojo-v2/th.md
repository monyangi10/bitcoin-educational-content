## RoninDojo คืออะไร?

Dojo เริ่มต้นเป็นการใช้งานโหนด Bitcoin แบบเต็มรูปแบบ โดยอิงจาก Bitcoin Core และพัฒนาโดยทีม Samourai Wallet โซลูชันนี้สามารถติดตั้งบนอุปกรณ์ใดก็ได้ แตกต่างจากการใช้งาน Core อื่น ๆ Dojo ได้รับการปรับแต่งเป็นพิเศษเพื่อรวมเข้ากับสภาพแวดล้อมแอปพลิเคชัน Android ของ Samourai Wallet สำหรับ RoninDojo นั้นเป็นเครื่องมือที่ออกแบบมาเพื่ออำนวยความสะดวกในการติดตั้งและการจัดการ Dojo รวมถึงเครื่องมือเสริมอื่น ๆ อีกมากมาย กล่าวโดยย่อ RoninDojo ช่วยเพิ่มการใช้งานพื้นฐานของ Dojo โดยการรวมเครื่องมือเพิ่มเติมหลากหลาย พร้อมทั้งทำให้การติดตั้งและการจัดการง่ายขึ้น


Ronin ยังมี [โซลูชัน node-in-box ที่เรียกว่า "*Tanto*"](https://ronindojo.io/en/products) ซึ่งเป็นอุปกรณ์ที่ติดตั้ง RoninDojo ไว้แล้วบนระบบที่ประกอบโดยทีมงานของพวกเขา Tanto เป็นตัวเลือกที่ต้องชำระเงิน ซึ่งอาจน่าสนใจสำหรับผู้ที่ต้องการหลีกเลี่ยงความซับซ้อนทางเทคนิค แต่เนื่องจากซอร์สโค้ดของ RoninDojo เป็นแบบเปิด จึงสามารถติดตั้งบนฮาร์ดแวร์ของคุณเองได้เช่นกัน ทางเลือกนี้ที่ประหยัดกว่า อย่างไรก็ตามต้องการการจัดการเพิ่มเติมบางอย่าง ซึ่งเราจะครอบคลุมในบทแนะนำนี้

RoninDojo คือ Dojo ดังนั้นจึงช่วยให้สามารถรวม Whirlpool CLI เข้ากับโหนด Bitcoin ของคุณได้อย่างง่ายดายเพื่อมอบประสบการณ์ coinjoin ที่ดีที่สุด ด้วย Whirlpool CLI ทำให้สามารถรีมิกซ์บิตคอยน์ของคุณได้อย่างต่อเนื่องตลอด 24 ชั่วโมง 7 วันต่อสัปดาห์ โดยไม่จำเป็นต้องเปิดคอมพิวเตอร์ส่วนตัวของคุณไว้


นอกเหนือจาก Whirlpool CLI, RoninDojo รวมเครื่องมือต่างๆ เพื่อเพิ่มประสิทธิภาพการทำงานของ Dojo ของคุณ ในบรรดาเครื่องมือเหล่านี้ เครื่องคิดเลข Boltzmann จะวิเคราะห์ระดับความเป็นส่วนตัวของธุรกรรมของคุณ, เซิร์ฟเวอร์ Electrum ช่วยให้สามารถเชื่อมต่อกระเป๋าเงิน Bitcoin ของคุณกับโหนดของคุณได้, และเซิร์ฟเวอร์ Mempool ช่วยให้คุณสามารถดูธุรกรรมของคุณในเครื่องได้ โดยไม่รั่วไหลข้อมูล.


เมื่อเปรียบเทียบกับโซลูชันโหนดอื่น ๆ เช่น Umbrel, RoninDojo มุ่งเน้นไปที่โซลูชัน on-chain และเครื่องมือความเป็นส่วนตัวอย่างชัดเจน ไม่เหมือนกับ Umbrel, RoninDojo ไม่รองรับการตั้งค่าโหนด Lightning หรือการรวมแอปพลิเคชันเซิร์ฟเวอร์ทั่วไปเพิ่มเติม แม้ว่า RoninDojo จะมีเครื่องมือที่หลากหลายน้อยกว่า Umbrel แต่ก็มีฟังก์ชันการทำงานที่จำเป็นทั้งหมดสำหรับการจัดการกิจกรรม on-chain ของคุณ


หากคุณไม่ต้องการฟังก์ชันทั่วไปหรือฟังก์ชันที่เกี่ยวข้องกับ Lightning Network ที่เสนอโดย Umbrel และคุณกำลังมองหาโหนดที่เรียบง่ายและเสถียรพร้อมเครื่องมือพื้นฐานเช่น Whirlpool หรือ Mempool, RoninDojo อาจเป็นทางออกที่เหมาะสม ในขณะที่ Umbrel มักจะกลายเป็นเซิร์ฟเวอร์มัลติทาสกิ้งขนาดเล็กที่มุ่งเน้นไปที่ Lightning Network และความหลากหลาย, RoninDojo สอดคล้องกับปรัชญาของ Samourai Wallet โดยมุ่งเน้นไปที่เครื่องมือพื้นฐานสำหรับความเป็นส่วนตัวของผู้ใช้


ตอนนี้ที่เราได้ร่าง RoninDojo แล้ว มาดูกันว่าจะตั้งค่าโหนดนี้อย่างไร


## ควรเลือกฮาร์ดแวร์อะไรสำหรับการติดตั้ง RoninDojo v2?

RoninDojo มีภาพสำหรับการติดตั้งซอฟต์แวร์อัตโนมัติบน [RockPro64](https://ronindojo.io/en/download) อย่างไรก็ตาม คู่มือของเรามุ่งเน้นไปที่ขั้นตอนการติดตั้งด้วยตนเองบน Raspberry Pi 4 แม้ว่า Raspberry Pi 5 จะเพิ่งเปิดตัวเมื่อไม่นานมานี้ และคู่มือนี้ควรจะสามารถใช้งานร่วมกับรุ่นใหม่นี้ได้ในทางทฤษฎี แต่ฉันยังไม่มีโอกาสทดสอบด้วยตัวเอง และยังไม่ได้รับข้อเสนอแนะจากชุมชน เมื่อฉันได้ Pi 5 และส่วนประกอบที่เข้ากันได้แล้ว ฉันจะอัปเดตคู่มือนี้เพื่อให้คุณทราบ ในระหว่างนี้ ฉันแนะนำให้ให้ความสำคัญกับ Pi 4 เนื่องจากมันทำงานได้อย่างสมบูรณ์แบบสำหรับโหนดของฉัน

สำหรับฉัน ฉันใช้ RoninDojo บน Raspberry Pi ที่มี RAM ขนาด 8 GB แม้ว่าสมาชิกบางคนในชุมชนจะสามารถทำให้มันทำงานบนอุปกรณ์ที่มี RAM เพียง 4 GB ได้ แต่ฉันยังไม่ได้ทดสอบการตั้งค่านี้ด้วยตัวเอง เนื่องจากความแตกต่างของราคาที่ไม่มากนัก ดูเหมือนว่าจะเป็นการฉลาดที่จะเลือกใช้รุ่นที่มี RAM ขนาด 8 GB ซึ่งอาจมีประโยชน์หากคุณวางแผนที่จะนำ Raspberry Pi ของคุณไปใช้ประโยชน์อื่นในอนาคต

สิ่งสำคัญคือต้องทราบว่าทีม RoninDojo ได้รายงานปัญหาบ่อยครั้งที่เกี่ยวข้องกับเคสและอะแดปเตอร์ SSD ฉันเองก็เคยประสบปัญหาเหล่านี้ **ดังนั้น ขอแนะนำอย่างยิ่งให้หลีกเลี่ยงเคสที่มีสาย USB สำหรับ SSD ของโหนดของคุณ** ให้เลือกใช้การ์ดขยายพื้นที่จัดเก็บข้อมูลที่ออกแบบมาโดยเฉพาะสำหรับ Raspberry Pi ของคุณ:


![storage expansion card RPI4](assets/notext/1.webp)


ในการจัดเก็บบล็อกเชน Bitcoin คุณจะต้องมี SSD ที่เข้ากันได้กับการ์ดขยายพื้นที่จัดเก็บที่คุณเลือก ปัจจุบัน (กุมภาพันธ์ 2024) เราอยู่ในช่วงการเปลี่ยนแปลง คาดว่าในอีกไม่กี่เดือนข้างหน้า ดิสก์ขนาด 1 TB จะไม่เพียงพอที่จะรองรับขนาดที่เพิ่มขึ้นของบล็อกเชน โดยเฉพาะอย่างยิ่งเมื่อพิจารณาถึงแอปพลิเคชันต่างๆ ที่คุณวางแผนจะรวมเข้ากับโหนดของคุณ ดังนั้นบางคนจึงแนะนำให้ลงทุนใน SSD ขนาด 2 TB เพื่อความสบายใจในระยะยาว อย่างไรก็ตาม ด้วยแนวโน้มราคาของ SSD ที่ลดลงทุกปี บางคนแนะนำให้ใช้ดิสก์ขนาด 1 TB ซึ่งควรจะเพียงพอสำหรับหนึ่งหรือสองปี โดยให้เหตุผลว่าเมื่อถึงเวลาที่มันล้าสมัย ราคาของรุ่น 2 TB อาจจะลดลง การเลือกจึงขึ้นอยู่กับความชอบส่วนบุคคลของคุณ หากคุณวางแผนที่จะเก็บ RoninDojo ของคุณไว้เป็นระยะเวลานานและต้องการหลีกเลี่ยงการจัดการทางเทคนิคในอีกไม่กี่ปีข้างหน้า ตัวเลือก SSD ขนาด 2 TB ดูเหมือนจะเป็นทางเลือกที่รอบคอบที่สุด เนื่องจากมันมอบขอบเขตที่สะดวกสบายสำหรับอนาคต


นอกจากนี้ คุณจะต้องมีส่วนประกอบขนาดเล็กต่างๆ:


- เคสที่ติดตั้งพัดลมเพื่อรองรับ Raspberry Pi ของคุณและการ์ดขยายพื้นที่จัดเก็บข้อมูล ชุดที่รวมทั้งการ์ดขยาย SSD และเคสที่เข้ากันได้มีจำหน่ายออนไลน์;
- สายไฟสำหรับ Raspberry Pi ของคุณ;
- การ์ด micro SD ขนาดอย่างน้อย 16 GB (แม้ว่า 8 GB อาจเพียงพอในทางเทคนิค แต่ความแตกต่างของราคาระหว่างการ์ด 8 และ 16 GB มักจะน้อยมาก);
- สายเคเบิลอีเธอร์เน็ต RJ45 สำหรับการเชื่อมต่อเครือข่าย

## วิธีประกอบ Raspberry Pi 4?

การประกอบโหนดของคุณจะแตกต่างกันไปตามฮาร์ดแวร์ที่เลือก โดยเฉพาะประเภทของเคส อย่างไรก็ตาม โครงร่างทั่วไปของขั้นตอนที่ต้องปฏิบัติตามยังคงคล้ายคลึงกันในการประกอบ

เริ่มต้นด้วยการติดตั้ง SSD ของคุณบนการ์ดขยายพื้นที่จัดเก็บ โดยระมัดระวังในการยึดสกรูล็อคสองตัวที่ด้านหลังให้แน่น


![assembly1](assets/notext/3.webp)


จากนั้นเชื่อมต่อ Raspberry Pi ของคุณเข้ากับการ์ดขยาย


![assembly2](assets/notext/4.webp)


นอกจากนี้ ให้ติดตั้งพัดลมเข้ากับ Raspberry Pi.


![assembly3](assets/notext/5.webp)


เชื่อมต่อส่วนประกอบต่าง ๆ โดยให้ความสนใจในการใช้พินที่ถูกต้อง โดยอ้างอิงจากคู่มือของเคสของคุณ ผู้ผลิตเคสมักจะมีวิดีโอสอนเพื่อช่วยคุณในการประกอบ ในกรณีของฉัน ฉันมีการ์ดขยายเพิ่มเติมที่ติดตั้งปุ่มเปิด/ปิด ซึ่งไม่จำเป็นสำหรับการสร้างโหนด Bitcoin ฉันใช้มันหลัก ๆ เพื่อมีปุ่มเปิด/ปิด


หากคุณมีการ์ดขยายที่มีปุ่มเปิด/ปิด อย่าลืมติดตั้งจัมเปอร์ "Auto Power On" ขนาดเล็ก ซึ่งจะช่วยให้โหนดของคุณเริ่มทำงานโดยอัตโนมัติทันทีที่เปิดเครื่อง ฟีเจอร์นี้มีประโยชน์อย่างยิ่งในกรณีที่เกิดไฟฟ้าดับ เนื่องจากช่วยให้โหนดของคุณเริ่มทำงานใหม่ได้เอง โดยไม่ต้องมีการแทรกแซงด้วยตนเองจากคุณ

ก่อนที่จะใส่อุปกรณ์ฮาร์ดแวร์ทั้งหมดลงในเคส สิ่งสำคัญคือต้องตรวจสอบการทำงานที่ถูกต้องของ Raspberry Pi การ์ดขยายพื้นที่จัดเก็บ และพัดลม โดยการเปิดเครื่องพวกมัน


![assembly5](assets/notext/7.webp)


สุดท้าย ติดตั้ง Raspberry Pi ของคุณในเคส โปรดทราบว่าในขั้นตอนต่อไปจะต้องใส่การ์ด micro SD ลงในพอร์ตที่เหมาะสมบน Raspberry Pi หากเคสของคุณมีช่องเปิดที่ให้คุณใส่การ์ด SD ได้โดยไม่ต้องเปิด (เช่นเดียวกับเคสของฉันที่แสดงในภาพ) คุณสามารถปิดเคสได้เลย อย่างไรก็ตาม หากเคสของคุณไม่มีช่องเข้าถึงพอร์ต micro SD โดยตรง คุณจะต้องรอจนกว่าคุณจะเตรียมการ์ด micro SD เสร็จแล้วจึงใส่ก่อนที่จะประกอบให้เสร็จสิ้น


![assembly6](assets/notext/8.webp)

### ขั้นตอนที่ 3: สร้างการเชื่อมต่อ SSH กับโหนด

ก่อนอื่น จำเป็นต้องค้นหา IP address ของโหนดของคุณ คุณมีตัวเลือกในการใช้เครื่องมือเช่น _[Advanced IP Scanner](https://www.advanced-ip-scanner.com/)_ หรือ _[Angry IP Scanner](https://angryip.org/)_ หรือเช็คที่อินเตอร์เฟซการจัดการของเราเตอร์ของคุณ IP address ควรอยู่ในรูปแบบ `192.168.1.??` **สำหรับคำสั่งทั้งหมดต่อไปนี้ ให้แทนที่ `[IP]` ด้วย IP address ที่แท้จริงของโหนดของคุณ** (ลบวงเล็บออก)


เปิดเทอร์มินัล


ในการลบคีย์ที่อาจเชื่อมโยงกับที่อยู่ IP ของโหนดของคุณ ให้ดำเนินการคำสั่ง:

`ssh-keygen -R [IP]`.


ข้อผิดพลาดที่ตามมาหลังจากคำสั่งนี้ไม่ใช่เรื่องร้ายแรง; มันหมายความว่า key นั้นไม่มีอยู่ในรายการโฮสต์ที่รู้จักของคุณ (ซึ่งค่อนข้างเป็นไปได้) ตัวอย่างเช่น หาก IP ของโหนดของคุณคือ `192.168.1.40` คำสั่งจะกลายเป็น: `ssh-keygen -R 192.168.1.40`.


ถัดไป, สร้างการเชื่อมต่อ SSH กับโหนดของคุณโดยการรันคำสั่ง:

`ssh pi@[IP]`.

ข้อความจะปรากฏเกี่ยวกับความถูกต้องของโฮสต์: `The authenticity of host '[IP]' can't be established.` ซึ่งบ่งบอกว่าความถูกต้องของอุปกรณ์ที่คุณพยายามเชื่อมต่อไม่สามารถตรวจสอบได้เนื่องจากขาดคีย์สาธารณะที่รู้จัก เมื่อเชื่อมต่อผ่าน SSH ไปยังโฮสต์ใหม่เป็นครั้งแรก ข้อความนี้จะปรากฏเสมอ คุณต้องตอบ `yes` เพื่อเพิ่มคีย์สาธารณะของมันไปยังไดเรกทอรีท้องถิ่นของคุณ ซึ่งจะป้องกันไม่ให้ข้อความเตือนนี้ปรากฏในระหว่างการเชื่อมต่อ SSH ในอนาคตกับโหนดนี้ ดังนั้น พิมพ์ `yes` และกด `enter` เพื่อยืนยัน

จากนั้นคุณจะถูกขอให้ป้อนรหัสผ่านของคุณ ซึ่งเป็นรหัสผ่านชั่วคราวที่ตั้งไว้ก่อนหน้านี้ในขั้นตอนที่ 1 ยืนยันด้วย `enter` จากนั้นคุณจะเชื่อมต่อกับโหนดของคุณผ่าน SSH


โดยสรุป นี่คือคำสั่งที่ต้องดำเนินการ:


- `ssh-keygen -R [IP]`
- `ssh pi@[IP]`
- `ใช่`
- ป้อนรหัสผ่านชั่วคราวและยืนยัน


### ขั้นตอนที่ 4: การอัปเดตและการเตรียมการ

คุณเชื่อมต่อกับโหนดของคุณผ่านเซสชัน SSH แล้ว บนเทอร์มินัลของคุณ, พร้อมรับคำสั่งควรเป็น: `pi@RoninDojo:~ $`. เพื่อเริ่มต้น, อัปเดตรายการแพ็กเกจที่มีอยู่และติดตั้งการอัปเดตสำหรับแพ็กเกจที่มีอยู่ด้วยคำสั่งต่อไปนี้:

`sudo apt update && sudo apt upgrade -y`


เมื่อติดตั้งการอัปเดตเสร็จสิ้น ให้ดำเนินการติดตั้ง *Git* และ *Dialog* โดยใช้คำสั่ง:

`sudo apt install git dialog -y`


ถัดไป โคลนสาขา `master` ของที่เก็บ Git _RoninOS_ โดยการดำเนินการ:

`sudo git clone --branch master https://code.samourai.io/ronindojo/RoninOS.git /opt/RoninOS`


เรียกใช้สคริปต์ `customize-image.sh` ด้วยคำสั่ง:

`cd /opt/RoninOS/ && sudo ./customize-image.sh`


**สิ่งสำคัญคือต้องปล่อยให้สคริปต์ทำงานโดยไม่ขัดจังหวะและรออย่างอดทนจนกว่าจะสิ้นสุดกระบวนการ**, ซึ่งใช้เวลาประมาณ 10 นาที เมื่อข้อความ `Setup is complete` ปรากฏขึ้น คุณสามารถไปยังขั้นตอนถัดไปได้


### ขั้นตอนที่ 5: การเปิดตัว RoninOS

เปิดตัว RoninOS ด้วยคำสั่ง:

`sudo systemctl start ronin-setup`


แสดงบรรทัดของไฟล์บันทึกด้วยคำสั่ง:

`tail -f /home/ronindojo/.logs/setup.logs`


ในขั้นตอนนี้ **สิ่งสำคัญคือต้องปล่อยให้ RoninOS เริ่มทำงานและรอให้มัน** ทำงานจนเสร็จ ซึ่งใช้เวลาประมาณ 40 นาที เมื่อข้อความ `All RoninDojo feature installations complete!` ปรากฏขึ้น คุณสามารถดำเนินการต่อไปยังขั้นตอนที่ 6 ได้


### ขั้นตอนที่ 6: การเข้าถึง RoninUI และการเปลี่ยนแปลงข้อมูลรับรอง

หลังจากติดตั้งเสร็จสิ้น เพื่อเชื่อมต่อกับโหนดของคุณผ่านเบราว์เซอร์ ตรวจสอบให้แน่ใจว่าคอมพิวเตอร์ส่วนบุคคลของคุณเชื่อมต่อกับเครือข่ายท้องถิ่นเดียวกันกับโหนดของคุณ หากคุณกำลังใช้ VPN บนเครื่องของคุณ ให้ปิดการใช้งานชั่วคราว เพื่อเข้าถึงอินเทอร์เฟซของโหนดในเบราว์เซอร์ของคุณ ให้ป้อนในแถบ URL:


- โดยตรงที่อยู่ IP ของโหนดของคุณ เช่น `192.168.1.??`;
- หรือ พิมพ์ `ronindojo.local`.


เมื่ออยู่ที่หน้าแรกของ RoninUI คุณจะได้รับการแจ้งให้เริ่มการตั้งค่า ในการทำเช่นนี้ ให้คลิกที่ปุ่ม `Let's start`


![lets start](assets/notext/25.webp)


ในขั้นตอนนี้ RoninUI จะแสดงรหัสผ่าน `root` ของคุณ จำเป็นต้องเก็บรักษาไว้อย่างปลอดภัย คุณสามารถเลือกสำรองข้อมูลทางกายภาพบนกระดาษ หรือบันทึกไว้ใน [ตัวจัดการรหัสผ่าน](https://planb.academy/courses/99c46148-7080-4915-a7e0-9df0e145cd47/0b3c69b2-522c-56c8-9fb8-1562bd55930f)


![root password](assets/notext/26.webp)


หลังจากบันทึกรหัสผ่าน `root` แล้ว ให้ทำเครื่องหมายในช่อง `I have backed up Root user credentials` และคลิกที่ `Continue` เพื่อดำเนินการต่อ


![confirm root password](assets/notext/27.webp)


ขั้นตอนถัดไปเกี่ยวข้องกับการสร้างรหัสผ่านผู้ใช้ ซึ่งจะใช้ทั้งในการเข้าถึงอินเทอร์เฟซเว็บ RoninUI และสำหรับการสร้างเซสชัน SSH กับโหนดของคุณ เลือกรหัสผ่านที่แข็งแกร่งและอย่าลืมบันทึกไว้อย่างปลอดภัย คุณจะต้องป้อนรหัสผ่านนี้สองครั้งก่อนคลิกที่ `Finish` เพื่อยืนยัน สำหรับชื่อผู้ใช้ แนะนำให้ใช้ตัวเลือกเริ่มต้น `ronindojo` หากคุณตัดสินใจเปลี่ยน อย่าลืมปรับคำสั่งในขั้นตอนต่อไปให้เหมาะสม


![user credentials](assets/notext/28.webp)


เมื่อดำเนินการเหล่านี้เสร็จสิ้น รอให้โหนดของคุณเริ่มต้นทำงาน จากนั้นคุณจะสามารถเข้าถึงอินเทอร์เฟซเว็บ RoninUI ได้ คุณเกือบจะถึงจุดสิ้นสุดของกระบวนการแล้ว เหลือเพียงไม่กี่ขั้นตอนเล็ก ๆ เท่านั้น!

![Ronin UI](assets/notext/29.webp)


### ขั้นตอนที่ 7: ลบข้อมูลรับรองชั่วคราว

เปิดเทอร์มินัลใหม่บนคอมพิวเตอร์ส่วนตัวของคุณและสร้างการเชื่อมต่อ SSH กับโหนดของคุณโดยใช้คำสั่งต่อไปนี้:

`SSH ronindojo@[IP]`


หากตัวอย่างเช่น ที่อยู่ IP ของโหนดของคุณคือ `192.168.1.40` คำสั่งที่เหมาะสมจะเป็น:

`SSH ronindojo@192.168.1.40`


หากคุณเปลี่ยนชื่อผู้ใช้ของคุณในขั้นตอนก่อนหน้า โดยแทนที่ชื่อผู้ใช้เริ่มต้น (`ronindojo`) ด้วยชื่ออื่น ให้แน่ใจว่าได้ใช้ชื่อนี้ในคำสั่ง ตัวอย่างเช่น หากคุณเลือก `planb` เป็นชื่อผู้ใช้และที่อยู่ IP คือ `192.168.1.40` คำสั่งที่จะป้อนคือ:

`SSH planb@192.168.1.40`

คุณจะถูกขอให้ป้อนรหัสผ่านของผู้ใช้ ป้อนรหัสผ่านแล้วกด `enter` เพื่อยืนยัน จากนั้นคุณจะเข้าสู่หน้าต่าง RoninCLI ใช้ปุ่มลูกศรบนแป้นพิมพ์ของคุณเพื่อไปที่ตัวเลือก `Exit RoninDojo` และกด `enter` เพื่อเลือก

![RoninCLI](assets/notext/30.webp)


ณ จุดนี้ คุณอยู่ที่เทอร์มินัลของโหนดของคุณ โดยมีพรอมต์คำสั่งที่คล้ายกับ: `ronindojo@RoninDojo:~ $` เพื่อที่จะลบผู้ใช้ชั่วคราวที่สร้างขึ้นระหว่างการกำหนดค่าของการ์ด micro SD ที่สามารถบู๊ตได้ ให้ป้อนคำสั่งต่อไปนี้และกด `enter`:

`sudo deluser --remove-home pi`


คุณจะได้รับการแจ้งให้ยืนยันรหัสผ่านผู้ใช้ของคุณ ป้อนรหัสผ่านและยืนยันโดยกด `enter` รอให้การดำเนินการเสร็จสิ้น จากนั้นใช้คำสั่ง `exit` เพื่อออกจากเทอร์มินัล


ขอแสดงความยินดี! โหนด RoninDojo v2 ของคุณได้รับการกำหนดค่าและพร้อมใช้งานแล้ว มันจะเริ่มต้น IBD (*Initial Block Download*) โดยดำเนินการดาวน์โหลดและตรวจสอบบล็อกเชน Bitcoin จากบล็อก Genesis ขั้นตอนนี้เกี่ยวข้องกับการดึงธุรกรรม Bitcoin ทั้งหมดที่ทำตั้งแต่วันที่ 3 มกราคม 2009 และใช้เวลาสักครู่ เมื่อบล็อกเชนถูกดาวน์โหลดเสร็จสมบูรณ์แล้ว ตัวจัดทำดัชนีจะดำเนินการบีบอัดฐานข้อมูล ระยะเวลาของ IBD อาจแตกต่างกันไป โหนด RoninDojo ของคุณจะพร้อมใช้งานอย่างเต็มที่เมื่อกระบวนการนี้เสร็จสิ้น


**หากคุณกำลังย้ายจากโหนด RoninDojo v1 เก่า** ไปยังเวอร์ชันใหม่นี้ด้วยบทแนะนำนี้ในขณะที่ใช้ SSD เดิม โหนดของคุณควรตรวจจับและใช้ข้อมูลที่มีอยู่บนดิสก์โดยอัตโนมัติ ช่วยให้คุณไม่จำเป็นต้องทำ IBD อีกครั้ง ในกรณีนี้ คุณเพียงแค่ต้องรอให้โหนดของคุณซิงโครไนซ์กับบล็อกล่าสุดอีกครั้ง


### ขั้นตอนที่ 8: "veth fix"

หากคุณพบข้อบกพร่องกับ RoninDojo v2 บน Raspberry Pi ซึ่งหลังจากการติดตั้งที่ไม่มีปัญหาใด ๆ แล้ว โหนดของคุณกลับไม่สามารถเข้าถึงได้ผ่าน SSH แต่กลับมาทำงานได้หลังจากรีสตาร์ทง่าย ๆ คุณจำเป็นต้องทำตามขั้นตอนที่ 8 ข้อบกพร่องทั่วไปนี้สามารถแก้ไขได้ง่ายด้วยวิธีการที่พัฒนาขึ้นโดยชุมชน: "_veth fix_" การแก้ไขเล็กน้อยนี้จะช่วยแก้ปัญหาการตัดการเชื่อมต่ออย่างกะทันหันได้อย่างถาวร นี่คือวิธีการนำไปใช้


เปิดเทอร์มินัลใหม่บนคอมพิวเตอร์ส่วนตัวของคุณและสร้างการเชื่อมต่อ SSH กับโหนดของคุณโดยใช้คำสั่งต่อไปนี้:

`SSH ronindojo@[IP]`


หากที่อยู่ IP ของโหนดของคุณคือ `192.168.1.40` คำสั่งที่เหมาะสมจะเป็น:

`SSH ronindojo@192.168.1.40`


คุณจะได้รับการแจ้งให้ป้อนรหัสผ่านของผู้ใช้ ป้อนรหัสผ่านและกด `enter` เพื่อยืนยัน จากนั้นคุณจะเข้าสู่หน้าต่าง RoninCLI ใช้ลูกศรบนแป้นพิมพ์ของคุณเพื่อไปที่ตัวเลือก `Exit RoninDojo` และกด `enter` เพื่อเลือก


ณ จุดนี้ คุณอยู่ที่เทอร์มินัลของโหนดของคุณ โดยมีพรอมต์คำสั่งที่คล้ายกับ: `ronindojo@RoninDojo:~ $` เพื่อใช้การแก้ไข **veth** ให้พิมพ์คำสั่งต่อไปนี้และกด `enter`:

`sudo nano /etc/dhcpcd.conf`


ยืนยันรหัสผ่านของคุณอีกครั้งและกด `enter`


คุณจะไปถึงไฟล์ `dhcpcd.conf` คุณจำเป็นต้องคัดลอกข้อความต่อไปนี้ โดยต้องแน่ใจว่าได้รวมเครื่องหมายดอกจันไว้ด้วย และเพิ่มลงไปที่ด้านล่างของไฟล์:

`denyinterfaces veth*`


ในการทำเช่นนี้ ให้เลื่อนไปที่ด้านล่างของไฟล์โดยใช้ลูกศรลงบนแป้นพิมพ์ของคุณ จากนั้นใช้การคลิกขวาของเมาส์เพื่อวางข้อความบนบรรทัดที่แยกออกมา


หลังจากเพิ่มข้อความแล้ว ให้กด `ctrl X` เพื่อเริ่มออก ตามด้วย `ctrl Y` เพื่อยืนยันการบันทึกการเปลี่ยนแปลง และกด `enter` เพื่อเสร็จสิ้นและกลับไปที่พรอมต์คำสั่ง เพื่อให้แน่ใจว่าการแก้ไขได้ถูกนำไปใช้อย่างถูกต้อง ให้เปิดไฟล์ `dhcpcd.conf` อีกครั้งโดยใช้คำสั่งที่เหมาะสม


ในการดำเนินการแก้ไขให้เสร็จสมบูรณ์ ให้รีสตาร์ทโหนดของคุณโดยการดำเนินการ:

`sudo reboot now`


ณ จุดนี้ คุณสามารถปิดเทอร์มินัลของคุณได้ รอเวลาที่จำเป็นสำหรับ RoninDojo ของคุณในการรีสตาร์ท หลังจากนั้นคุณควรจะสามารถเชื่อมต่อใหม่ผ่านอินเทอร์เฟซกราฟิกของเบราว์เซอร์ของคุณได้ กระบวนการนี้ควรจะแก้ไขบั๊กที่พบเจอได้


## วิธีการใช้โหนด RoninDojo v2 ของคุณ?


### เชื่อมต่อซอฟต์แวร์ wallet ของคุณกับ Electrs

การใช้งานครั้งแรกของโหนดที่คุณติดตั้งและซิงโครไนซ์ใหม่จะเป็นการกระจายธุรกรรมของคุณไปยังเครือข่าย Bitcoin คุณอาจต้องการเชื่อมต่อกระเป๋าสตางค์ต่างๆ ของคุณกับโหนดของคุณเพื่อกระจายธุรกรรมของคุณอย่างเป็นความลับ คุณสามารถทำได้ผ่านเซิร์ฟเวอร์ Electrum Rust (electrs) แอปพลิเคชันนี้มักจะติดตั้งล่วงหน้าในโหนด RoninDojo ของคุณ หากไม่ คุณสามารถติดตั้งด้วยตนเองผ่านอินเทอร์เฟซ RoninCLI ใน `Applications > Manage Applications > Install Electrum Server`


ในการรับที่อยู่ Tor ของ Electrum Server ของคุณ จากอินเทอร์เฟซเว็บ RoninUI ให้ไปที่:

`จับคู่ > เซิร์ฟเวอร์ Electrum > จับคู่ตอนนี้`

![Pairing](assets/notext/31.webp)

![Electrs](assets/notext/32.webp)

จากนั้นคุณจะต้องป้อนที่อยู่ `Hostname` ที่ลงท้ายด้วย `.onion` ในซอฟต์แวร์ wallet ของคุณ พร้อมด้วยพอร์ต `50001`. ![hostname](assets/notext/33.webp)

ตัวอย่างเช่น บน Sparrow Wallet เพียงไปที่แท็บ:

`ไฟล์ > การตั้งค่า > เซิร์ฟเวอร์ > ไพรเวท Electrum`


![Sparrow](assets/notext/34.webp)


### เชื่อมต่อซอฟต์แวร์ wallet ของคุณกับ Samourai Dojo

เป็นทางเลือกแทนการใช้ Electrs, Dojo ช่วยให้คุณสามารถเชื่อมต่อซอฟต์แวร์ที่เข้ากันได้ wallet ของคุณโดยตรงกับโหนด RoninDojo ของคุณ กระเป๋าเงินอย่าง Samourai Wallet และ Sentinel มีฟังก์ชันนี้


ในการสร้างการเชื่อมต่อ คุณเพียงแค่สแกน QR code ของ Dojo ของคุณ เพื่อเข้าถึง QR code นี้ผ่าน RoninUI ให้ไปที่:

`จับคู่ > Samourai Dojo > จับคู่ตอนนี้`

![Samourai Dojo](assets/notext/35.webp)

ในการเชื่อมต่อ Samourai Wallet ของคุณกับ Dojo เพียงแค่สแกน QR โค้ดนี้ระหว่างการติดตั้งแอป:


![Samourai Wallet connection](assets/notext/36.webp)


หากคุณมี Samourai Wallet อยู่แล้วก่อนที่จะตั้งค่า Ronin Dojo จำเป็นต้องสำรองข้อมูล wallet ของคุณ ถอนการติดตั้งแล้วติดตั้งแอป Samourai Wallet ใหม่ ก่อนที่จะกู้คืน wallet ของคุณ เมื่อเปิดแอปที่ติดตั้งใหม่ คุณจะมีตัวเลือกในการเชื่อมต่อกับ Dojo ใหม่ **ระวัง กระบวนการนี้มีความเสี่ยงที่จะสูญเสียบิตคอยน์ของคุณหากดำเนินการไม่ถูกต้อง!** ตรวจสอบให้แน่ใจว่าคุณมีสำเนาสำรองของ Samourai wallet ในไฟล์ของคุณและตรวจสอบความถูกต้องของ passphrase ของคุณผ่าน `Settings > Troubleshoot > Passphrase` นอกจากนี้ยังสำคัญที่จะต้องมีสำเนาสำรองที่อ่านได้ของวลีการกู้คืนและ passphrase ของคุณ สำหรับความแม่นยำในการดำเนินการนี้ ขอแนะนำให้ทำตามบทแนะนำโดยละเอียดนี้: [https://wiki.ronindojo.io/en/setup/v2_0_0-upgrade/reconnectsamourai](https://wiki.ronindojo.io/en/setup/v2_0_0-upgrade/reconnectsamourai)


### ใช้ตัวสำรวจบล็อก Mempool.space ของคุณเอง

ตัวสำรวจบล็อกจะแปลงข้อมูลดิบจากบล็อกเชน Bitcoin ให้เป็นรูปแบบที่มีโครงสร้างและอ่านได้ง่าย ด้วยเครื่องมืออย่าง *Mempool.space* สามารถวิเคราะห์ธุรกรรม ค้นหาที่อยู่เฉพาะ หรือแม้กระทั่งตรวจสอบอัตราค่าธรรมเนียมเฉลี่ยของเมมพูลของเครือข่ายแบบเรียลไทม์


อย่างไรก็ตาม การใช้ตัวสำรวจบล็อกออนไลน์มีความเสี่ยงต่อความเป็นส่วนตัวของคุณและเกี่ยวข้องกับความไว้วางใจในข้อมูลที่จัดหาโดยบุคคลที่สาม อันที่จริง การใช้บริการเหล่านี้โดยไม่ผ่านโหนดของคุณเอง คุณอาจเปิดเผยข้อมูลเกี่ยวกับธุรกรรมของคุณโดยไม่ตั้งใจและต้องพึ่งพาความถูกต้องของข้อมูลที่นำเสนอโดยเจ้าของเว็บไซต์

เพื่อลดความเสี่ยงเหล่านี้ ขอแนะนำให้ใช้อินสแตนซ์ของคุณเองบน *Mempool.space* ผ่านเครือข่าย Tor ซึ่งโฮสต์โดยตรงบนโหนดของคุณ วิธีแก้ปัญหานี้ช่วยให้มั่นใจได้ถึงการรักษาความเป็นส่วนตัวของคุณและความเป็นอิสระของข้อมูลของคุณ

ในการทำเช่นนี้ ให้เริ่มต้นด้วยการติดตั้ง *Mempool Space Visualizer* จาก RoninUI บนเว็บอินเตอร์เฟซ ไปที่แท็บ `Dashboard` และคลิกที่ `Manage` ใต้ `Mempool Space`:

`แดชบอร์ด > Mempool สเปซ > จัดการ`

![Manage mempool](assets/notext/37.webp)

จากนั้นคลิกที่ปุ่ม `Install Mempool visualizer`:

![install mempool](assets/notext/38.webp)

ยืนยันรหัสผ่านผู้ใช้ของคุณ:

![password mempool](assets/notext/39.webp)

รอให้การติดตั้งเสร็จสิ้น จากนั้นคลิกที่ปุ่ม `จัดการ` อีกครั้ง:

![Mempool Manage](assets/notext/40.webp)

คุณจะได้รับลิงก์ `.onion` เพื่อเข้าถึงอินสแตนซ์ของคุณเองบน *Mempool.space* ผ่านเครือข่าย Tor

![Mempool link](assets/notext/41.webp)

ฉันแนะนำให้คุณบันทึกลิงก์นี้ในรายการโปรดของคุณบนเบราว์เซอร์ Tor หรือเพิ่มลงในแอป Tor Browser บนสมาร์ทโฟนของคุณเพื่อการเข้าถึงที่ง่ายและปลอดภัยจากทุกที่ หากคุณยังไม่มีเบราว์เซอร์ Tor คุณสามารถดาวน์โหลดได้ที่นี่: [https://www.torproject.org/download/](https://www.torproject.org/download/)

การทำ coinjoin ผ่าน Whirlpool ต้องการให้แอปพลิเคชันที่ใช้เปิดใช้งานเพื่อดำเนินการรีมิกซ์ เงื่อนไขนี้อาจเป็นข้อจำกัดสำหรับผู้ที่ต้องการบรรลุระดับความเป็นนิรนามสูง อันที่จริง อุปกรณ์ที่โฮสต์แอปพลิเคชันที่รวม Whirlpool ต้องเปิดใช้งานอย่างต่อเนื่อง ซึ่งหมายความว่าเพื่อเข้าร่วมในการรีมิกซ์ตลอด 24 ชั่วโมง คอมพิวเตอร์หรือสมาร์ทโฟนของคุณต้องเปิดใช้งาน Samourai หรือ Sparrow อย่างต่อเนื่อง วิธีแก้ปัญหาข้อจำกัดนี้คือการใช้ _WhirlpoolCLI_ บนเครื่องที่เปิดใช้งานตลอดเวลา เช่น โหนด Bitcoin ทำให้เหรียญของคุณสามารถรีมิกซ์ได้โดยไม่หยุดชะงัก และไม่จำเป็นต้องเปิดอุปกรณ์อื่นทิ้งไว้


กำลังเตรียมบทแนะนำอย่างละเอียดเพื่อแนะนำคุณทีละขั้นตอนผ่านกระบวนการ coinjoining กับ Samourai Wallet และ RoninDojo v2 ตั้งแต่ต้นจนจบ


สำหรับความเข้าใจที่ลึกซึ้งยิ่งขึ้นเกี่ยวกับ coinjoin และการใช้งานบน Bitcoin ฉันขอเชิญคุณอ่านบทความอื่นนี้: การทำความเข้าใจและการใช้ coinjoin บน Bitcoin ซึ่งฉันได้อธิบายทุกสิ่งที่คุณจำเป็นต้องรู้เกี่ยวกับเทคนิคนี้




### การใช้เครื่องมือสถิติ Whirlpool (WST)


หลังจากทำการ coinjoins กับ Whirlpool แล้ว การประเมินระดับความเป็นส่วนตัวที่ได้รับสำหรับ UTXOs ที่ผสมของคุณอย่างแม่นยำเป็นสิ่งที่มีประโยชน์ ในการทำเช่นนี้ คุณสามารถใช้เครื่องมือ Python ที่ชื่อว่า *Whirlpool Stat Tool* เครื่องมือนี้ช่วยให้คุณสามารถวัดคะแนนทั้งในเชิงคาดการณ์และย้อนหลังของ UTXOs ของคุณ ในขณะที่วิเคราะห์อัตราการแพร่กระจายในพูล


เพื่อเพิ่มความเข้าใจในกลไกการคำนวณของ anonsets เหล่านี้ ฉันขอแนะนำให้อ่านบทความ: REMIX - WHIRLPOOL ซึ่งอธิบายการทำงานของดัชนีเหล่านี้อย่างละเอียด


https://planb.academy/tutorials/privacy/on-chain/remix-whirlpool-2b887bd9-8a6a-4dca-8aa9-a1c33682b0aa



ในการเข้าถึงเครื่องมือ WST ให้ไปที่ RoninCLI ในการทำเช่นนี้ ให้เปิดเทอร์มินัลบนคอมพิวเตอร์ส่วนบุคคลของคุณและสร้างการเชื่อมต่อ SSH กับโหนดของคุณโดยใช้คำสั่งต่อไปนี้:

`SSH ronindojo@[IP]`


หากที่อยู่ IP ของโหนดของคุณคือ `192.168.1.40` คำสั่งที่เหมาะสมจะเป็น:

`SSH ronindojo@192.168.1.40`


หากคุณเปลี่ยนชื่อผู้ใช้ของคุณในขั้นตอนที่ 6 โดยแทนที่ชื่อผู้ใช้เริ่มต้น (`ronindojo`) ด้วยชื่ออื่น ให้แน่ใจว่าได้ใช้ชื่อนี้ในคำสั่ง ตัวอย่างเช่น หากคุณเลือก `planb` เป็นชื่อผู้ใช้ของคุณและที่อยู่ IP คือ `192.168.1.40` คำสั่งที่ต้องป้อนจะเป็น:

`SSH planb@192.168.1.40`


คุณจะถูกขอให้ป้อนรหัสผ่านของผู้ใช้ ป้อนรหัสผ่านและกด `enter` เพื่อยืนยัน จากนั้นคุณจะเข้าสู่หน้าต่าง RoninCLI ใช้ปุ่มลูกศรบนแป้นพิมพ์ของคุณเพื่อไปที่เมนู `Samourai Toolkit` และกด `enter` เพื่อเลือก:

ก่อนอื่น จำเป็นต้องกำหนดค่า proxy เพื่อใช้ Tor เพื่อให้แน่ใจว่ามีความเป็นส่วนตัวเมื่อดึงข้อมูลจาก OXT ป้อนคำสั่ง:

`socks5 127.0.0.1:9050`


จากนั้น ดำเนินการดาวน์โหลดข้อมูลพูลที่มีธุรกรรมของคุณ:

`ดาวน์โหลด 0001`

แทนที่ `0001` ด้วยรหัสสกุลเงินของพูลที่คุณสนใจ รหัสสกุลเงินมีดังนี้บน WST:

- พูล 0.05 บิตคอยน์: `005`
- พูล 0.01 บิตคอยน์: `001`
- พูล 0.001 บิตคอยน์: `0001`


หลังจากดาวน์โหลด ให้โหลดข้อมูลโดยแทนที่ `0001` ด้วยรหัสพูลของคุณในคำสั่งนี้: `load 0001`


![WST loading](assets/notext/46.webp)


รอให้การโหลดเสร็จสิ้น ซึ่งอาจใช้เวลาสักครู่ เมื่อข้อมูลโหลดเสร็จแล้ว เพื่อทราบคะแนน anonset ของเหรียญของคุณ ให้ดำเนินการคำสั่ง `score` ตามด้วย TXID ของคุณ (โดยไม่ต้องใส่วงเล็บ):

`score [TXID]`


![WST score](assets/notext/47.webp)


จากนั้น WST จะแสดงคะแนนย้อนหลัง (_Backward-looking metrics_) ตามด้วยคะแนนคาดการณ์ (_Forward-looking metrics_) นอกเหนือจากคะแนน anonset แล้ว WST จะแสดงอัตราการแพร่กระจายของธุรกรรมของคุณภายในพูลเมื่อเทียบกับ anonset ของมันด้วย


**สิ่งสำคัญคือต้องทราบว่าคะแนนที่คาดหวังของเหรียญของคุณควรถูกคำนวณจาก TXID ของการผสมครั้งแรกของคุณ ไม่ใช่จากการผสมครั้งล่าสุด ในทางกลับกัน คะแนนย้อนหลังของ UTXO จะถูกคำนวณจาก TXID ของรอบสุดท้าย**


### การใช้เครื่องคิดเลขโบลต์ซมันน์

เครื่องคิดเลข Boltzmann เป็นเครื่องมือสำหรับวิเคราะห์ธุรกรรม Bitcoin โดยมีความสามารถในการวัดระดับเอนโทรปีของมันพร้อมกับเมตริกขั้นสูงอื่น ๆ ข้อมูลเหล่านี้ให้การประเมินเชิงปริมาณเกี่ยวกับความเป็นส่วนตัวของธุรกรรมและช่วยระบุข้อบกพร่องที่อาจเกิดขึ้น เครื่องมือนี้ได้ถูกรวมเข้ากับโหนด RoninDojo ของคุณแล้ว ทำให้ง่ายต่อการเข้าถึงและใช้งาน


ก่อนที่จะอธิบายขั้นตอนการใช้เครื่องคิดเลข Boltzmann สิ่งสำคัญคือต้องเข้าใจความหมายของตัวชี้วัดเหล่านี้ วิธีการคำนวณ และประโยชน์ของมัน แม้ว่าจะสามารถใช้กับธุรกรรม Bitcoin ใดๆ ก็ได้ แต่ตัวชี้วัดเหล่านี้มีประโยชน์อย่างยิ่งในการประเมินคุณภาพของธุรกรรม coinjoin


**ตัวบ่งชี้แรก** ที่ซอฟต์แวร์คำนวณคือจำนวนรวมของการผสมผสานที่เป็นไปได้ทั้งหมด ซึ่งแสดงภายใต้ `nb combinations` ในเครื่องมือ โดยอิงจากค่าของ UTXOs ที่เกี่ยวข้อง ตัวบ่งชี้นี้จะคำนวณจำนวนวิธีที่อินพุตสามารถเชื่อมโยงกับเอาต์พุตได้ กล่าวอีกนัยหนึ่ง มันจะกำหนดจำนวนการตีความที่เป็นไปได้ที่ธุรกรรมสามารถ generate ได้ ตัวอย่างเช่น coinjoin ที่มีโครงสร้างตามโมเดล Whirlpool 5x5 จะมี `1496` การผสมผสานที่เป็นไปได้:

![combinations](assets/notext/50.webp)

เครดิต: KYCP


**ตัวบ่งชี้ที่สอง** ที่คำนวณคือเอนโทรปีของธุรกรรม ซึ่งกำหนดโดย `Entropy` เมื่อธุรกรรมมีจำนวนการผสมผสานที่เป็นไปได้สูง มักจะมีความเกี่ยวข้องมากกว่าที่จะอ้างถึงเอนโทรปีของมัน ซึ่งนิยามเป็นลอการิทึมฐานสองของจำนวนการผสมผสานที่เป็นไปได้ นี่คือสูตรที่ใช้:


- $E$: เอนโทรปีของธุรกรรม;
- $C$: จำนวนของการจัดชุดที่เป็นไปได้สำหรับการทำธุรกรรม

$$E = \log_2(C)$$


ในทางคณิตศาสตร์ ลอการิทึมฐานสอง (ลอการิทึมฐาน 2) สอดคล้องกับการดำเนินการผกผันของการยก 2 เป็นกำลัง กล่าวอีกนัยหนึ่ง ลอการิทึมฐานสองของ $x$ คือเลขชี้กำลังที่ 2 ต้องถูกยกขึ้นเพื่อให้ได้ $x$ ดังนั้น ตัวบ่งชี้นี้จะแสดงในบิต ลองพิจารณาตัวอย่างของการคำนวณเอนโทรปีสำหรับธุรกรรม coinjoin ที่มีโครงสร้างตามโมเดล Whirlpool 5x5 ซึ่งตามที่กล่าวไว้ก่อนหน้านี้ มีจำนวนของการจัดเรียงที่เป็นไปได้ `1496`:

$$ C = 1496 $$

$$ E = \log_2(1496) $$

$$ E \approx 10.5469 \text{ bits}$$


ดังนั้น การทำธุรกรรม coinjoin นี้แสดงให้เห็นถึงเอนโทรปีที่ 10.5469 บิต ซึ่งถือว่าน่าพอใจมาก ยิ่งค่านี้สูงเท่าใด การตีความที่แตกต่างกันที่การทำธุรกรรมนี้ยอมรับก็ยิ่งมากขึ้นเท่านั้น ซึ่งจะช่วยเพิ่มระดับความเป็นส่วนตัวของมัน


มาดูตัวอย่างเพิ่มเติมกับธุรกรรมที่เป็นแบบทั่วไปมากขึ้น โดยมีหนึ่งอินพุตและสองเอาท์พุต: [1b1b0c3f0883a99f1161c64da19471841ed12a1f78e77fab128c69a5f578ccce](https://mempool.space/en/tx/1b1b0c3f0883a99f1161c64da19471841ed12a1f78e77fab128c69a5f578ccce)

ในกรณีของธุรกรรมนี้ การตีความที่เป็นไปได้เพียงอย่างเดียวคือ: `(inp 0) > (Outp 0 ; Outp 1)` ดังนั้น ความเอนโทรปีของมันจึงถูกกำหนดไว้ที่ `0`:

$$ C = 1 $$

$$ E = \log_2(1) $$

$$ E \approx 0 \text{ บิต}$$

**ตัวบ่งชี้ที่สาม** ที่ให้โดย Boltzmann Calculator มีชื่อว่า `Wallet Efficiency` ตัวบ่งชี้นี้ประเมินประสิทธิภาพของการทำธุรกรรมโดยเปรียบเทียบกับการทำธุรกรรมที่เหมาะสมที่สุดที่สามารถเกิดขึ้นได้ในสถานการณ์ที่เหมือนกัน ซึ่งนำเราไปสู่การพูดคุยเกี่ยวกับแนวคิดของเอนโทรปีสูงสุด ซึ่งสอดคล้องกับเอนโทรปีสูงสุดที่โครงสร้างการทำธุรกรรมเฉพาะสามารถบรรลุได้ในทางทฤษฎี ดังนั้น สำหรับโครงสร้าง coinjoin Whirlpool 5x5 เอนโทรปีสูงสุดถูกตั้งไว้ที่ `10.5469` ประสิทธิภาพของการทำธุรกรรมจะถูกคำนวณโดยการเปรียบเทียบเอนโทรปีสูงสุดนี้กับเอนโทรปีจริงของการทำธุรกรรมที่วิเคราะห์ สูตรที่ใช้คือดังนี้:


- $ER$: เอนโทรปีที่แท้จริงของธุรกรรม แสดงเป็นบิต;
- $EM$: ค่าความเอนโทรปีสูงสุดที่เป็นไปได้สำหรับโครงสร้างธุรกรรมที่กำหนด ซึ่งวัดเป็นบิต;
- $Ef$: ประสิทธิภาพของธุรกรรม เป็นบิต

$$Ef = ER - EM$$ $$Ef = 10.5469 - 10.5469$$

$$Ef = 0 \text{ bits}$$


ตัวบ่งชี้นี้ยังแสดงเป็นเปอร์เซ็นต์ สูตรของมันคือ:


- $CR$: จำนวนของการจัดเรียงที่เป็นไปได้จริง;
- $CM$: จำนวนชุดค่าผสมสูงสุดที่เป็นไปได้ที่มีโครงสร้างเดียวกัน;
- $Ef$: ประสิทธิภาพที่แสดงเป็นเปอร์เซ็นต์

$$Ef = \frac{CR}{CM}$$

$$Ef = \frac{1496}{1496}$$

$$Ef = 100\%$$


ประสิทธิภาพที่ `100%` จึงบ่งบอกว่าธุรกรรมนั้นเพิ่มศักยภาพสูงสุดสำหรับความเป็นส่วนตัวตามโครงสร้างของมัน


**ตัวบ่งชี้ที่สี่**, ความหนาแน่นของเอนโทรปี, หรือ `Entropy Density`, ให้มุมมองเกี่ยวกับเอนโทรปีที่สัมพันธ์กับแต่ละอินพุตหรือเอาต์พุตของธุรกรรม ตัวบ่งชี้นี้มีประโยชน์สำหรับการประเมินและเปรียบเทียบประสิทธิภาพของธุรกรรมที่มีขนาดต่างกัน ในการคำนวณ เพียงแค่แบ่งเอนโทรปีทั้งหมดของธุรกรรมด้วยจำนวนรวมของอินพุตและเอาต์พุตที่เกี่ยวข้อง ยกตัวอย่างเช่น Whirlpool 5x5 coinjoin:


- $ED$: ความหนาแน่นของเอนโทรปีที่แสดงในบิต;
- $E$: เอนโทรปีของธุรกรรมที่แสดงในบิต;
- $T$: จำนวนอินพุตและเอาต์พุตทั้งหมดในธุรกรรม

$$T = 5 + 5 = 10$$

$$ED = \frac{E}{T}$$

$$ED = \frac{10.5469}{10}$$

$$ED = 1.054 \text{ bits}$$

**ชิ้นส่วนข้อมูลที่ห้า** ที่ส่งมอบโดย Boltzmann Calculator คือ ตารางความน่าจะเป็นของการจับคู่ระหว่างอินพุตและเอาต์พุต ตารางนี้แสดงให้เห็นผ่าน `Boltzmann score` ความน่าจะเป็นที่อินพุตเฉพาะจะเชื่อมโยงกับเอาต์พุตที่กำหนด โดยใช้ตัวอย่างของ Whirlpool coinjoin ตารางความน่าจะเป็นจะแสดงโอกาสของการเชื่อมโยงระหว่างแต่ละอินพุตและเอาต์พุต โดยให้การวัดเชิงปริมาณของความคลุมเครือหรือความสามารถในการคาดเดาของการเชื่อมโยงในธุรกรรม:


| %       | Output 0 | Output 1 | Output 2 | Output 3 | Output 4 |
|---------|----------|----------|----------|----------|----------|
| Input 0 | 34%      | 34%      | 34%      | 34%      | 34%      |
| Input 1 | 34%      | 34%      | 34%      | 34%      | 34%      |
| Input 2 | 34%      | 34%      | 34%      | 34%      | 34%      |
| Input 3 | 34%      | 34%      | 34%      | 34%      | 34%      |
| Input 4 | 34%      | 34%      | 34%      | 34%      | 34%      |


ที่นี่ เห็นได้ชัดว่าแต่ละอินพุตมีโอกาสเท่าเทียมกันที่จะเชื่อมโยงกับเอาต์พุตใด ๆ ซึ่งเสริมสร้างความคลุมเครือและความเป็นส่วนตัวของธุรกรรม อย่างไรก็ตาม ในกรณีของธุรกรรมง่าย ๆ ที่มีอินพุตเดียวและเอาต์พุตสองรายการ สถานการณ์จะแตกต่างออกไป:


| %       | Output 0 | Output 1 |
|---------|----------|----------|
| Input 0 | 100%     | 100%     |

ที่นี่ เราเห็นว่าความน่าจะเป็นสำหรับแต่ละผลลัพธ์ที่จะมาจากอินพุต 0 คือ 100% ความน่าจะเป็นที่ต่ำกว่าจึงแปลเป็นความเป็นส่วนตัวที่มากขึ้น โดยการเจือจางการเชื่อมโยงโดยตรงระหว่างอินพุตและผลลัพธ์


**ข้อมูลชิ้นที่หก** ที่ให้มาคือจำนวนของลิงก์ที่กำหนดแน่นอน ซึ่งเสริมด้วยอัตราส่วนของลิงก์เหล่านี้ ตัวบ่งชี้นี้แสดงให้เห็นว่ามีการเชื่อมต่อระหว่างอินพุตและเอาต์พุตในธุรกรรมที่วิเคราะห์แล้วกี่รายการที่ไม่สามารถโต้แย้งได้ ด้วยความน่าจะเป็น 100% ในทางกลับกัน อัตราส่วนนี้ให้มุมมองเกี่ยวกับน้ำหนักของลิงก์ที่กำหนดแน่นอนเหล่านี้ภายในลิงก์ทั้งหมดของธุรกรรม


ตัวอย่างเช่น การทำธุรกรรมแบบ coinjoin ประเภท Whirlpool ไม่มีการเชื่อมโยงที่กำหนดได้แน่นอน ดังนั้นจึงแสดงตัวบ่งชี้และอัตราส่วนที่ 0% ในทางกลับกัน ในการทำธุรกรรมที่เราตรวจสอบครั้งที่สอง (มีหนึ่งอินพุตและสองเอาต์พุต) ตัวบ่งชี้ถูกตั้งค่าไว้ที่ 2 และอัตราส่วนถึง 100% ดังนั้น ตัวบ่งชี้ที่เป็นศูนย์บ่งบอกถึงความเป็นส่วนตัวที่ยอดเยี่ยมเนื่องจากไม่มีการเชื่อมโยงโดยตรงและที่ปฏิเสธไม่ได้ระหว่างอินพุตและเอาต์พุต


**วิธีการเข้าถึงเครื่องคิดเลข Boltzmann บน RoninDojo?**

ในการเข้าถึงเครื่องมือ *Boltzmann Calculator* ให้ไปที่ RoninCLI ในการทำเช่นนี้ ให้เปิดเทอร์มินัลบนคอมพิวเตอร์ส่วนบุคคลของคุณและสร้างการเชื่อมต่อ SSH กับโหนดของคุณโดยใช้คำสั่งต่อไปนี้: `SSH ronindojo@[IP]`


หากตัวอย่างเช่น ที่อยู่ IP ของโหนดของคุณคือ `192.168.1.40` คำสั่งที่เหมาะสมจะเป็น:

`SSH ronindojo@192.168.1.40`


หากคุณเปลี่ยนชื่อผู้ใช้ของคุณในขั้นตอนที่ 6 โดยแทนที่ชื่อผู้ใช้เริ่มต้น (`ronindojo`) ด้วยชื่ออื่น ให้แน่ใจว่าได้ใช้ชื่อนี้ในคำสั่ง ตัวอย่างเช่น หากคุณเลือก `planb` เป็นชื่อผู้ใช้ของคุณและที่อยู่ IP คือ `192.168.1.40` คำสั่งที่จะป้อนคือ:

`SSH planb@192.168.1.40`


คุณจะถูกขอให้ป้อนรหัสผ่านผู้ใช้ ป้อนรหัสผ่านแล้วกด `enter` เพื่อยืนยัน จากนั้นคุณจะเข้าสู่หน้าต่าง RoninCLI ใช้ลูกศรบนแป้นพิมพ์ของคุณเพื่อนำทางไปยังเมนู `Samourai Toolkit` และกด `enter` เพื่อเลือก:

### คุณสมบัติอื่น ๆ ของ RoninDojo v2 ของคุณ

โหนด RoninDojo ของคุณรวมคุณสมบัติต่างๆ ไว้มากมาย โดยเฉพาะอย่างยิ่ง คุณมีความสามารถในการสแกนข้อมูลเฉพาะเพื่อพิจารณา ตัวอย่างเช่น บางครั้ง Samourai wallet ของคุณที่เชื่อมต่อกับ RoninDojo อาจไม่แสดงบิตคอยน์ที่คุณถืออยู่จริง หากยอดคงเหลือแสดง 0 ในขณะที่คุณมั่นใจว่ามีบิตคอยน์ใน wallet นี้ อาจมีหลายสาเหตุที่อธิบายสถานการณ์นี้ได้ เช่น ข้อผิดพลาดในเส้นทางการอนุพันธ์ แต่หนึ่งในสาเหตุอาจเป็นเพราะโหนดของคุณไม่ได้ติดตามที่อยู่ของคุณอย่างถูกต้อง ในการแก้ปัญหานี้ คุณสามารถตรวจสอบให้แน่ใจว่าโหนดของคุณกำลังติดตาม `xpub` ของคุณโดยใช้ _xpub tool_ ในการเข้าถึงเครื่องมือนี้ผ่าน RoninUI ให้ทำตามเส้นทาง:

`การบำรุงรักษา > เครื่องมือ XPUB`


ป้อน `xpub` ที่กำลังทำให้เกิดปัญหาและคลิกที่ปุ่ม `Check` เพื่อตรวจสอบข้อมูลนี้:

![xpub tool](assets/notext/54.webp)

ตรวจสอบให้แน่ใจว่าธุรกรรมทั้งหมดถูกระบุไว้อย่างถูกต้อง นอกจากนี้ยังสำคัญที่จะต้องตรวจสอบว่าประเภทการอนุพันธ์ที่ใช้ตรงกับของ wallet ของคุณหรือไม่ หากไม่เป็นเช่นนั้น ให้คลิกที่ `Retype` จากนั้นเลือกจาก `BIP44`, `BIP49`, หรือ `BIP84` ตามความต้องการของคุณ

นอกเหนือจากเครื่องมือนี้แล้ว แท็บ `Maintenance` ของ RoninUI ยังเต็มไปด้วยฟีเจอร์ที่มีประโยชน์อื่นๆ:


- เครื่องมือธุรกรรม*: ช่วยให้ตรวจสอบรายละเอียดของธุรกรรมที่กำหนด;
- Address Tool*: ช่วยยืนยันการติดตามที่อยู่ที่กำหนดโดย Dojo ของคุณ;
- Rescan Blocks*: บังคับให้โหนดของคุณทำการสแกนใหม่ในช่วงบล็อกที่ระบุ


แท็บ `Push Tx` เป็นอีกหนึ่งฟีเจอร์ที่น่าสนใจของ RoninUI ซึ่งช่วยให้สามารถกระจายธุรกรรมที่ลงนามแล้วบนเครือข่าย Bitcoin ได้ ธุรกรรมต้องถูกป้อนในรูปแบบเลขฐานสิบหก


เกี่ยวกับแท็บอื่น ๆ ที่มีอยู่บนแดชบอร์ด RoninUI ของคุณ:


- `Apps`: โฮสต์แอปพลิเคชัน Whirlpool และจะถูกใช้ในการรวมแอปพลิเคชันใหม่ในอนาคตอย่างแน่นอน;
- `Logs`: ให้การเข้าถึงบันทึกเหตุการณ์ของซอฟต์แวร์ของคุณแบบเรียลไทม์;
- `System Info`: ให้ข้อมูลทั่วไปเกี่ยวกับโหนดของคุณ เช่น อุณหภูมิของ CPU การใช้งานพื้นที่จัดเก็บ หรือข้อมูล RAM คุณยังจะพบตัวเลือก `Reboot` และ `Shut down` เพื่อรีสตาร์ทหรือปิดโหนดของคุณ;
- `การตั้งค่า`: อนุญาตให้คุณเปลี่ยนรหัสผ่านผู้ใช้ของคุณ
