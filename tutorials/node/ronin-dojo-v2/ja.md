## RoninDojoとは？

Dojoは当初、Bitcoin coreをベースにした完全なBitcoinノードの実装であり、Samourai Walletチームによって開発された。このソリューションはどのような機器にもインストールできます。他のCore実装とは異なり、Dojoは特にSamourai WalletのAndroidアプリケーション環境と統合するために最適化されています。RoninDojoに関しては、Dojoのインストールと管理を容易にするために設計されたユーティリティであり、他の様々な補完ツールでもあります。つまり、RoninDojoは、Dojoのインストールと管理を簡素化しながら、多数の追加ツールを統合することによって、Dojoの基本的な実装を豊かにします。


Roninはまた、[「*Tanto*」と呼ばれるノード・イン・ボックス・ソリューション](https://ronindojo.io/en/products)も提供している。これは、彼らのチームが組み立てたシステムにRoninDojoがすでにインストールされたデバイスである。Tantoは有料オプションで、技術的な複雑さを避けたい人には興味深いかもしれない。しかし、RoninDojoのソースコードはオープンなので、自分のハードウェアに導入することも可能だ。この選択肢は経済的ですが、いくつかの追加操作が必要です。

RoninDojo は道場であるため、Whirlpool CLI を Bitcoin ノードに簡単に統合することができ、CoinJoin で可能な限り最高の体験を提供します。Whirlpool CLIを使えば、24時間365日、パソコンの電源を入れたままで、ビットコインをリミックスし続けることが可能になります。


Whirlpool CLI 以外にも、RoninDojo にはあなたの Dojo の機能を強化する様々なツールが含まれています。その中でも、ボルツマン計算機はあなたの取引のプライバシーレベルを分析し、ElectrumサーバーはBitcoinウォレットをあなたのノードに接続することを可能にし、Mempoolサーバーは情報を漏らすことなくあなたの取引をローカルで見ることを可能にします。


Umbrelのような他のノード・ソリューションと比較して、RoninDojoは明らかにOn-Chainソリューションとプライバシー・ツールに焦点を当てている。Umbrel とは異なり、RoninDojo は Lightning ノードのセットアップや、より汎用的なサーバーアプリケーションの統合をサポートしていません。RoninDojo は Umbrel よりも多機能ではありませんが、On-Chain の活動を管理するために必要な機能はすべて備えています。


Umbrelが提供するような汎用的な機能やLightning Networkに関連する機能を必要とせず、WhirlpoolやMempoolのような必須ツールを備えたシンプルで安定したノードをお探しなら、RoninDojoは理想的なソリューションとなるでしょう。UmbrelがLightning Networkや多用途性を指向したミニ・マルチタスク・サーバーになりがちなのに対して、RoninDojoはSamourai Walletの理念に沿って、ユーザーのプライバシーのための基本的なツールに焦点を当てています。


RoninDojoの概要がわかったところで、このノードをどのようにセットアップするか一緒に見てみましょう。


## RoninDojo v2をインストールするには、どのハードウェアを選べばよいですか？

RoninDojoは[RockPro64](https://ronindojo.io/en/download)にソフトウェアを自動インストールするためのイメージを提供しています。しかし、このチュートリアルでは、Raspberry Pi 4への手動インストール手順に焦点を当てます。Raspberry Pi 5が最近発売され、このチュートリアルは理論的にはこの新しいモデルに対応しているはずですが、私はまだ個人的にテストする機会がなく、コミュニティからのフィードバックも見当たりません。私がPi 5と互換性のあるコンポーネントを入手し次第、このチュートリアルを更新し、情報をお伝えします。それまでは、Pi 4を優先することをお勧めします。

私自身は、8GBのRAMを搭載したRaspberry PiでRoninDojoを動かしています。コミュニティーのメンバーの中には、4GBのRAMしか搭載していないデバイスでRoninDojoを動作させた人もいますが、私自身はこの構成をテストしたことはありません。わずかな価格差を考えると、8GB RAMバージョンを選ぶのが賢明だと思われる。また、将来Raspberry Piを他の用途に再利用する予定がある場合にも便利です。

重要なことは、RoninDojoチームから、ケースとSSDアダプターに関する問題が頻繁に報告されていることだ。私自身もこれらの問題に直面しました。 **したがって、ノードのSSD用のUSBケーブルを備えたケースは避けることを強くお勧めします：


![storage expansion card RPI4](assets/notext/1.webp)


Bitcoin Blockchainを保存するには、選択したストレージ拡張カードと互換性のあるSSDが必要です。現在（2024年2月）、私たちは移行期にあります。特に、ノードに統合する予定のさまざまなアプリケーションを考慮すると、数カ月後には、1 TB ディスクでは、Blockchain のサイズが大きくなるのを格納するのに十分ではなくなると予想されます。そのため、長期的な安心のために2TBのSSDに投資することを勧める人もいる。しかし、SSDの価格は年々下落傾向にあるため、1～2年は十分な1TBディスクで我慢することを勧める人もいる。このように、選択は個人の好みによる。RoninDojoをかなりの期間使用する予定があり、今後数年間の技術的な対応を避けたいのであれば、2 TBのSSDを選択するのが最も賢明であると思われる。


さらに、さまざまな小さな部品が必要になる：


- Raspberry Piとストレージ拡張カードを収納するファン付きケース。SSD拡張カードと互換性のあるケースの両方を含むキットをオンラインで入手できます；
- Raspberry Pi用の電源ケーブル；
- 少なくとも16GBのマイクロSDカード（技術的には8GBでも十分だが、8GBと16GBのカードの価格差はごくわずかであることが多い）；
- ネットワーク接続用のRJ45イーサネットケーブル。


![node material](assets/notext/2.webp)


## Raspberry Pi 4の組み立て方は？

ノードの組み立ては、選択したハードウェア、特にケースのタイプによって異なります。しかし、従うべきステップの大まかな概要は、組み立てにおいて概ね同様であることに変わりはありません。

まず、ストレージ拡張カードにSSDを取り付け、背面にある2つのロックネジに注意してください。


![assembly1](assets/notext/3.webp)


次にRaspberry Piを拡張カードに取り付けます。


![assembly2](assets/notext/4.webp)


また、ファンをRaspberry Piに取り付ける。


![assembly3](assets/notext/5.webp)


ケースのマニュアルを参照し、正しいピンを使用するように注意しながら、さまざまなコンポーネントを接続します。ケースメーカーは、組み立てをサポートするビデオチュートリアルを提供していることが多い。私の場合、オン/オフボタンを備えた拡張カードを追加しています。これはBitcoinノードを作るのに必須ではありません。主に電源ボタンとして使っています。


私のように拡張カードにオン／オフボタンが付いている場合は、小さな「Auto Power On」ジャンパを忘れずに取り付けてください。これにより、電源が入るとすぐにノードが自動的に起動するようになります。この機能は特に停電時に便利で、手動で操作しなくてもノードが勝手に再起動します。


![assembly4](assets/notext/6.webp)


すべてのハードウェアをケースに挿入する前に、Raspberry Pi、ストレージ拡張カード、ファンの電源を入れ、正しく機能することを確認することが重要です。


![assembly5](assets/notext/7.webp)


最後に、Raspberry Piをケースに取り付けます。この後、マイクロSDカードをRaspberry Piの適切なポートに挿入する必要があります。ケースを開けなくてもSDカードを挿入できる開口部がある場合は（写真の私のケースのように）、今すぐケースを閉じてください。しかし、あなたのケースがマイクロSDポートに直接アクセスできない場合は、マイクロSDカードを挿入する準備ができるまで待ってから、最終的な組み立てを行う必要があります。


![assembly6](assets/notext/8.webp)

### ステップ1：ブータブルマイクロSDを準備する

- マック用：https://downloads.raspberrypi.org/imager/imager_latest.dmg

### ステップ3：ノードとのSSH接続の確立

まず、ノードのIP Addressを見つける必要があります。Advanced IP Scanner](https://www.advanced-ip-scanner.com/)_や[Angry IP Scanner](https://angryip.org/)_のようなツールを使うか、ルーターの管理Interfaceをチェックする方法がある。IPのAddressは、`192.168.1.??`という形式になっているはずです。 **以下のコマンドはすべて、`[IP]`をお使いのノード**の実際のIP Addressに置き換えてください（括弧を削除してください）。


ターミナルを起動する。


ノードのIP Addressに関連付けられている可能性のあるキーを削除するには、コマンドを実行します：

ssh-keygen -R [IP]`.


このコマンドに続くエラーは深刻なものではなく、単にそのキーが既知のホストのリストに存在しない（可能性が高い）ことを意味する。たとえば、ノードの IP アドレスが `192.168.1.40` の場合、コマンドは次のようになります：ssh-keygen -R 192.168.1.40`となります。


次に、コマンドを実行してノードとのSSH接続を確立する：

ssh pi@[IP]`。

ホストの真正性に関して次のようなメッセージが表示されます: `The authenticity of host '[IP]' can't be established.` これは接続しようとしているデバイスの真正性が、既知の公開鍵がないために 検証できないことを示しています。初めて SSH で新しいホストに接続するときは、常にこのメッセージが表示されます。yes`と答えて、その公開鍵をローカルディレクトリに追加してください。そうすれば、今後このノードにSSH接続するときにこの警告メッセージが表示されなくなります。そのため、`yes`と入力し、`enter`を押して認証してください。

ステップ1で仮パスワードとして設定したパスワードを入力するよう求められます。Enter` で確認する。SSHでノードに接続されます。


まとめると、実行するコマンドは以下の通り：


- ssh-keygen -R [IP]`.
- ssh pi@[IP]`」。
- はい
- 仮パスワードを入力し、認証する。


### ステップ4：更新と準備

これでSSHセッションでノードに接続されました。ターミナルでは、コマンドプロンプトが表示されているはずです：pi@RoninDojo:~ $`.開始するには、以下のコマンドで利用可能なパッケージのリストを更新し、既存のパッケージのアップデートをインストールします：

sudo apt update && sudo apt upgrade -y`.


アップデートが完了したら、コマンドを使って*Git*と*Dialog*をインストールする：

sudo apt install git dialog -y`.


次に、_RoninOS_ Git リポジトリの `master` ブランチをクローンします：

`sudo git clone --branch master https://code.samourai.io/ronindojo/RoninOS.git /opt/RoninOS`


スクリプト `customize-image.sh` をコマンドで実行する：

cd /opt/RoninOS/ && sudo ./customize-image.sh`.


**スクリプトを中断することなく実行させ、プロセスの終了を辛抱強く待つことが重要です。セットアップが完了しました」というメッセージが表示されたら、次のステップに進むことができます。


### ステップ 5: RoninOS の起動

コマンドで RoninOS を起動する：

sudo systemctl start ronin-setup`.


コマンドでログファイルの行を表示する：

`tail -f /home/ronindojo/.logs/setup.logs`.


この段階で、** RoninOS を起動させ、** 実行が終了するのを待つことが重要です。これには約 40 分かかります。すべての RoninDojo 機能のインストールが完了しました!

### ステップ 7: 一時的な認証情報の削除

パソコンで新しいターミナルを開き、以下のコマンドを使ってノードとのSSH接続を確立する：

SSH ronindojo@[IP]`」。


例えば、ノードのIP Addressが`192.168.1.40`の場合、適切なコマンドは次のようになる：

前のステップでユーザー名を変更し、デフォルトのユーザー名(`ronindojo`)を別のユーザー名に置き換えた場合は、コマンドでこの新しい名前を使うようにしてください。例えば、ユーザー名に `planb` を選び、IP Address を `192.168.1.40` とした場合、入力するコマンドは次のようになる：

ユーザーパスワードの入力を求められます。パスワードを入力し、`enter` を押して認証します。RoninCLI Interface にアクセスします。キーボードの矢印キーを使用して `Exit RoninDojo` オプションに移動し、`enter` を押して選択します。

### ボルツマン計算機の使用

ボルツマン計算機は、Bitcoinトランザクションを分析するためのツールであり、他の高度なメトリクスの中でエントロピーのレベルを測定する機能を提供します。これらのデータは、トランザクションのプライバシーを定量的に評価し、潜在的な欠陥を特定するのに役立ちます。このツールはすでにあなたの RoninDojo ノードに統合されており、簡単にアクセスして使用することができます。


ボルツマン計算機の使用手順を詳しく説明する前に、これらの指標の意味、計算方法、および有用性を理解することが重要である。どの Bitcoin 取引にも適用できるが、これらの指標は CoinJoin 取引の質を評価する際に特に有用である。


**ソフトウェアが計算する最初の指標**は、可能な組み合わせの総数である。関係するUTXOの値に基づいて、この指標は入力が出力に関連付けられ得る方法の数を定量化する。言い換えれば、これはトランザクションがgenerateで可能なもっともらしい解釈の数を決定する。例えば、Whirlpool 5x5モデルに従って構造化されたCoinJoinは、`1496`の可能な組み合わせを提示する：

- E$：トランザクションのエントロピー；
- C$：トランザクションの可能な組み合わせの数。

|---------|----------|----------|
| Input 0 | 100%     | 100%     |

ここで、各アウトプットがインプット0に由来する確率は100％であることがわかる。確率が低ければ低いほど、インプットとアウトプットの直接的な結びつきが希薄になるため、プライバシーがより守られることになる。


**提供される情報**の6つ目は、決定論的リンクの数であり、これらのリンクの比率によって補完される。この指標は、分析されたトランザクションのインプットとアウトプットの間に、100％の確率で、どれだけのつながりがあるかを明らかにする。この比率は、トランザクションの全リンクの中で、これらの決定論的リンクがどの程度のウェイトを占めているかを示すものである。


例えば、WhirlpoolタイプのCoinJoinトランザクションは決定論的リンクを示さないため、インジケータと比率は0％である。一方、2 番目に調査したトランザクション（入力が 1 つ、出力が 2 つ）では、インジケータは 2 に設定され、比率は 100％に達する。このように、インプットとアウトプットの間に直接的で議論の余地のないリンクがないため、インジケータがNULLであれば優れたプライバシーを示すことになる。


**RoninDojoでボルツマン計算機にアクセスするには？

ボルツマン電卓* ツールにアクセスするには、RoninCLI にアクセスしてください。これを行うには、パソコンでターミナルを開き、以下のコマンドを使用してノードとのSSH接続を確立します：SSH ronindojo@[IP]`。


例えば、ノードのIP Addressが`192.168.1.40`の場合、適切なコマンドは次のようになる：

SSH ronindojo@192.168.1.40`

SSH planb@192.168.1.40`

### RoninDojo v2 のその他の機能

あなたの RoninDojo ノードには、他にもさまざまな機能が統合されています。特に、特定の情報をスキャンして考慮する機能があります。例えば、RoninDojo に接続している Samourai Wallet が、実際に保有しているビットコインを表示しないことがあります。このWalletにビットコインがあるはずなのに残高が0と表示される場合、派生パスのエラーなど、いくつかの原因が考えられます。しかし、あなたのノードがあなたのアドレスを適切に監視していないことも原因の一つです。この問題を解決するには、_xpub tool_ を使って、ノードがあなたの `xpub` に従っていることを確認します。このツールに RoninUI 経由でアクセスするには、以下のパスに従います：

メンテナンス > XPUBツール


問題の原因となっている`xpub`を入力し、`Check`ボタンをクリックしてこの情報を確認する：

![xpub tool](assets/notext/54.webp)

すべてのトランザクションが適切にリストされていることを確認する。また、使用されている派生タイプが Wallet のものと一致していることを確認することも重要である。そうでない場合は、`Retype`をクリックし、`BIP44`、`BIP49`、`BIP84`から必要なものを選択してください。

このツール以外にも、RoninUIの「メンテナンス」タブには便利な機能が満載です：


- 取引ツール*：指定されたトランザクションの詳細を調べることができます；
- Addressツール*：道場でAddressのトラッキングを確認できる；
- ブロックの再スキャン*：指定されたブロック範囲の新しいスキャンをノードに実行させる。


Push Tx` タブは RoninUI のもう一つの興味深い機能で、Bitcoin ネットワーク上で署名されたトランザク ションのブロードキャストを可能にします。トランザクションは16進数で入力しなければならない。


RoninUI ダッシュボードで利用可能な他のタブについて：


- Apps`：Whirlpoolのアプリケーションをホストし、将来的に新しいアプリケーションを統合するために使用される；
- ログ`：ソフトウェアのイベントログにリアルタイムでアクセスできる；
- システム情報`：CPU温度、ストレージ使用量、RAMデータなど、ノードに関する一般的な情報を提供します。ノードを再起動したり電源を切ったりするための `Reboot` と `Shut down` オプションもあります；
- 設定`：ユーザーパスワードを変更できます。
