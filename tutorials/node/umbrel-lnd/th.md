## บทนำ



บทแนะนำขั้นสูงนี้จะนำคุณไปทีละขั้นตอนผ่านการติดตั้ง การกำหนดค่า และการใช้งานแอปพลิเคชัน Lightning Node (LND) บนโหนด Umbrel ของคุณ คุณจะได้เรียนรู้วิธีเปิดช่องทาง จัดการสภาพคล่องของคุณ และซิงโครไนซ์โหนดของคุณกับแอปพลิเคชันมือถือ



## 1. ข้อกำหนดเบื้องต้น: โหนด Bitcoin Umbrel ที่ใช้งานได้



ก่อนที่จะปรับใช้ Lightning คุณจำเป็นต้องมีโหนด Bitcoin ที่ทำงานได้อย่างสมบูรณ์บน Umbrel ซึ่งรวมถึงการติดตั้ง Umbrel (บน Raspberry Pi, NAS หรือเครื่องอื่น ๆ) และการซิงโครไนซ์ Blockchain Bitcoin อย่างเต็มที่



ในการติดตั้ง Umbrel และกำหนดค่าโหนด Bitcoin ของคุณ เราขอแนะนำให้คุณทำตามบทแนะนำเฉพาะของเรา :



https://planb.academy/tutorials/node/bitcoin/umbrel-8b0e3b5b-d3cf-4a1e-8bb8-1ad2db4dd848

ตรวจสอบให้แน่ใจว่าโหนด Bitcoin ของคุณเป็นปัจจุบันและทำงานอย่างถูกต้อง เนื่องจาก Lightning Network ต้องพึ่งพามันสำหรับธุรกรรม off-chain ทั้งหมด



## 2. บทนำสู่ Lightning Network



Lightning Network เป็นโปรโตคอลชั้นที่สองที่ออกแบบมาเพื่อเร่งความเร็วและลดต้นทุนของธุรกรรม Bitcoin โดยดำเนินการนอก Blockchain หลัก



ในเชิงรูปธรรม Lightning ใช้เครือข่ายของช่องทางการชำระเงินระหว่างโหนด: ผู้ใช้สองคนเปิดช่องทางโดยการบล็อก On-Chain BTC (ธุรกรรมเริ่มต้น) จากนั้นสามารถแลกเปลี่ยนการชำระเงินภายในช่องทางนี้ได้ทันที ธุรกรรมเหล่านี้ off-chain จะไม่ถูกบันทึกบน Blockchain ดังนั้นจึงมีความเร็วและค่าใช้จ่ายแทบจะเป็นศูนย์



การชำระเงินสามารถส่งผ่านช่องทางหลายช่องทาง (ขอบคุณโหนดตัวกลาง) เพื่อเข้าถึงผู้รับใด ๆ ในเครือข่าย ทำให้สามารถทำธุรกรรมได้ในระดับที่เกือบจะไม่จำกัด Lightning จึงเสนอธุรกรรมที่รวดเร็วมากและมีต้นทุนต่ำ ซึ่งเหมาะสำหรับการชำระเงินในชีวิตประจำวันหรือธุรกรรมขนาดเล็ก ในขณะที่ลดภาระบน Blockchain Bitcoin



ในการทำงาน โหนด Lightning จะต้องเชื่อมต่อกับเครือข่ายอย่างถาวรและโต้ตอบกับโหนด Lightning อื่น ๆ มีการใช้งานซอฟต์แวร์หลายแบบ (LND, Core Lightning, Eclair, ฯลฯ) ซึ่งทั้งหมดสามารถทำงานร่วมกันได้ Umbrel ใช้ LND (Lightning Network Daemon) เป็นส่วนหนึ่งของแอปพลิเคชันโหนด Lightning อย่างเป็นทางการ บทแนะนำนี้เน้นที่ LND



สำหรับการแนะนำทฤษฎีเกี่ยวกับ Lightning Network อย่างครบถ้วน เราขอแนะนำให้คุณเข้าร่วมหลักสูตรเฉพาะของเรา :



https://planb.academy/courses/34bd43ef-6683-4a5c-b239-7cb1e40a4aeb

หลักสูตรนี้จะให้ความรู้พื้นฐานที่ครอบคลุมเกี่ยวกับแนวคิดพื้นฐานของ Lightning Network ก่อนที่จะไปฝึกปฏิบัติกับโหนด LND ของคุณ



## 3. ทำไมต้องโฮสต์ LND ด้วยตัวเอง?



การดำเนินการโหนด Lightning ของคุณเอง (LND) บน Umbrel ให้คุณมีอำนาจอธิปไตยเต็มที่เหนือเงินทุนและช่องทางของคุณ เมื่อเปรียบเทียบกับโซลูชันที่มีการดูแลหรือกึ่งดูแล



### การเปรียบเทียบโซลูชัน Lightning :



**โซลูชันการดูแลรักษา (เช่น: Wallet of Satoshi)** :




- บิตคอยน์ Lightning ของคุณได้รับการจัดการโดยบุคคลที่สามที่เชื่อถือได้
- ใช้งานง่าย ไม่มีความซับซ้อนทางเทคนิค
- ผู้ดำเนินการถือเงินของคุณและสามารถติดตามธุรกรรมของคุณได้
- คุณเสียสละการควบคุมและความเป็นส่วนตัว



**กระเป๋าเงินผู้บริโภคที่ไม่ใช่สินค้าโภคภัณฑ์ (เช่น Phoenix, Breez)** :




- ผู้ใช้เก็บรักษาคีย์ส่วนตัวของตนและด้วยเหตุนี้จึงเป็นเจ้าของ BTC ของตน
- ไม่มีการจัดการโหนดอย่างสมบูรณ์ - แอปพลิเคชันจัดการช่องทางในพื้นหลัง
- ประนีประนอมระหว่างความเรียบง่ายและอธิปไตย
- การพึ่งพาโครงสร้างพื้นฐานของผู้จัดหาสำหรับสภาพคล่อง
- ข้อจำกัดทางเทคนิค (สมาร์ทโฟนเครื่องหนึ่งไม่สามารถกำหนดเส้นทางการชำระเงินสำหรับเครื่องอื่นได้)



**โหนด LND ที่โฮสต์เอง (Umbrel)** :




- อธิปไตยสูงสุด: On-Chain และ off-chain BTC ของคุณอยู่ภายใต้การควบคุมของคุณทั้งหมด
- ไม่มีบุคคลที่สามเข้ามาเกี่ยวข้องในการเปิดช่องทางหรือจัดการการชำระเงินของคุณ
- ความเป็นส่วนตัวที่เพิ่มขึ้น (ช่องทางและธุรกรรมของคุณจะเป็นที่รู้จักเฉพาะคุณและเพื่อนร่วมงานโดยตรงของคุณเท่านั้น)
- อิสระในการใช้งาน: เชื่อมต่อกับบริการและกระเป๋าเงินของคุณเอง
- ความเป็นไปได้ในการกำหนดเส้นทางธุรกรรมสำหรับผู้อื่น (ค่าตอบแทนขนาดเล็ก)
- ความรับผิดชอบทางเทคนิคที่เพิ่มขึ้น (การบำรุงรักษา, การจัดการสภาพคล่อง, การสำรองข้อมูล)



โดยสรุป การโฮสต์ LND ด้วยตัวเองจะให้การควบคุมสูงสุดแก่คุณ แต่ต้องการทักษะทางเทคนิคมากขึ้น เป็นการแลกเปลี่ยนระหว่างความสะดวกสบายและอธิปไตย



## 4. บทแนะนำทีละขั้นตอน



### 4.1 การติดตั้งและกำหนดค่าการใช้งานแอปพลิเคชัน Lightning Node บน Umbrel



เมื่อโหนด Umbrel ของคุณ (Bitcoin) ได้รับการซิงโครไนซ์แล้ว ให้ทำตามขั้นตอนเหล่านี้ :



![Installation de Lightning Node depuis l'App Store Umbrel](assets/fr/01.webp)



ติดตั้งแอปพลิเคชัน Lightning Node จากส่วน "App Store" ของ Interface Umbrel.



![Avertissement sur la nature expérimentale de Lightning](assets/fr/02.webp)



LND (Lightning Network Daemon) จะถูกติดตั้งบน Umbrel ของคุณในฐานะโปรแกรมประยุกต์ เมื่อคุณเปิดมันครั้งแรก คุณจะเห็นข้อความเตือนที่แจ้งให้คุณทราบว่า Lightning เป็นเทคโนโลยีทดลอง



![Création ou restauration d'un nœud LND](assets/fr/03.webp)



คุณสามารถเลือกได้ระหว่างการสร้างโหนดใหม่หรือการกู้คืนจากการสำรองข้อมูล/seed สำหรับการติดตั้งครั้งแรก ให้เลือกสร้างโหนดใหม่ แอป Lightning Node จะ generate วลีช่วยจำ 24 คำ (seed Lightning ของคุณ): จดบันทึกไว้อย่างระมัดระวัง (ควรทำแบบออฟไลน์ บนกระดาษ) เนื่องจากจะใช้ในการกู้คืนเงิน Lightning ของคุณหากจำเป็น



**หมายเหตุ:** ในเวอร์ชันล่าสุดของ Umbrel การติดตั้งแอป Lightning จะให้คำ 24 คำของ seed (ตัวโหนด Bitcoin Umbrel เองไม่มี)



![Interface principale de Lightning Node](assets/fr/04.webp)



หลังจากการเริ่มต้น คุณจะเข้าถึง Interface หลักของ Lightning Node



![Paramètres de l'application](assets/fr/05.webp)



ในการตั้งค่าแอปพลิเคชัน คุณจะพบตัวเลือกที่สำคัญหลายอย่าง:




   - ปรึกษา Node ID ของคุณ (ตัวระบุเฉพาะของโหนดของคุณ)
   - เชื่อมต่อ wallet ภายนอก (เชื่อมต่อ Wallet)
   - ดูคำลับ
   - เข้าถึงการตั้งค่าขั้นสูง
   - กู้คืนช่อง
   - ดาวน์โหลดไฟล์สำรองข้อมูลช่อง
   - เปิดใช้งานการสำรองข้อมูลอัตโนมัติ
   - กำหนดค่าการสำรองข้อมูลผ่าน Tor (สำรองข้อมูลผ่าน Tor)



ตัวเลือกเหล่านี้มีความสำคัญต่อความปลอดภัยและการจัดการโหนด Lightning ของคุณ อย่าลืมเปิดใช้งานการสำรองข้อมูลอัตโนมัติและเก็บรักษาคำลับของคุณให้ปลอดภัย



**แหล่งข้อมูลที่มีประโยชน์:**




- [Umbrel Community](https://community.umbrel.com) - ฟอรัมสนทนาสำหรับผู้ใช้เพื่อแบ่งปันปัญหาและวิธีแก้ไขเกี่ยวกับ Umbrel และระบบนิเวศของมัน


> - [Umbrel App Store - Lightning Node (LND)](https://apps.umbrel.com/app/lightning) - คำอธิบายคุณสมบัติของแอป Lightning Node บน Umbrel
> - [LND Docs - Quickstart](https://docs.lightning.engineering/lightning-network-tools/LND/run-LND) - เอกสารทางการของ LND

### 4.2 การเปิดช่องทาง Lightning



เมื่อ LND ทำงานแล้ว คุณสามารถเปิดช่องทาง Lightning แรกของคุณได้ ในการค้นหาโหนดคุณภาพเพื่อเชื่อมต่อ:



![Page d'accueil Amboss.space](assets/fr/06.webp)



[Amboss.space](https://amboss.space/) เป็นเครื่องมือสำรวจสำหรับค้นหาโหนดที่เชื่อถือได้เพื่อเปิดช่องทาง



![Exemple de nœud ACINQ sur Amboss](assets/fr/07.webp)



ตัวอย่างเช่น [ACINQ node](https://amboss.space/node/03864ef025fde8fb587d989186ce6a4a186895ee44a926bfc370e2c366597a3f8f) เป็นโหนดที่ได้รับการยอมรับด้วยสถิติความพร้อมใช้งานและสภาพคล่องที่ยอดเยี่ยม



![Informations de connexion Swiss Bitcoin Pay](assets/fr/08.webp)



สำหรับบทแนะนำนี้ เราจะเปิดช่องทางกับ [Swiss Bitcoin Pay](https://amboss.space/node/03c181e13a09a649c13f60ea3ddbeefc66123c43280da8eebc19f54445f35173ca) ข้อมูลที่จำเป็นสำหรับการเชื่อมต่อ (pubkey@ip:port) มีให้ในหน้า Amboss ของพวกเขา



ในการเปิดช่อง :



![Bouton d'ouverture de canal](assets/fr/09.webp)



บนหน้าแรกของ Lightning Node ให้คลิกที่ปุ่ม "+ OPEN CHANNEL"



![Configuration du canal](assets/fr/10.webp)



ในหน้าการกำหนดค่าช่อง :




   - วางรหัสโหนดที่คัดลอกจาก Amboss (รูปแบบ: pubkey@ip:port)
   - กำหนดจำนวนความจุของช่องสัญญาณ (บางโหนดเช่น ACINQ มีขั้นต่ำ เช่น 400k Sats)
   - ปรับค่าธรรมเนียมการทำธุรกรรมเปิดหากจำเป็น



![Canal en cours d'ouverture](assets/fr/11.webp)



เมื่อทำการส่งธุรกรรมแล้ว ช่องทางจะปรากฏเป็น "กำลังเปิด" บนหน้าแรก รอการยืนยันธุรกรรม On-Chain



![Détails du canal](assets/fr/12.webp)



คลิกที่ช่องเพื่อดูรายละเอียด:




   - สถานะปัจจุบัน
   - ความจุรวม
   - การแยกย่อยของสภาพคล่อง (ขาเข้า/ขาออก)
   - คีย์สาธารณะของโหนดระยะไกล
   - และข้อมูลทางเทคนิคอื่น ๆ



### การใช้ Lightning Network+ เพื่อรับสภาพคล่องขาเข้า



![Lightning Network+ dans l'App Store](assets/fr/13.webp)

![Interface principale de LN+](assets/fr/14.webp)

- "Open For Me": กรองการแลกเปลี่ยนที่คุณมีสิทธิ์
- "To Docs": เข้าถึงเอกสาร



![Message d'erreur LN+](assets/fr/15.webp)



หมายเหตุ: หากคุณยังไม่ได้เปิดช่อง คุณจะเห็นข้อความแสดงข้อผิดพลาดนี้เมื่อคุณคลิกที่ "เปิดสำหรับฉัน"



![Liste des swaps disponibles](assets/fr/16.webp)



หน้า "การแลกเปลี่ยนสภาพคล่อง" แสดงข้อเสนอการแลกเปลี่ยนทั้งหมดที่มีอยู่ในเครือข่าย



![Swaps éligibles](assets/fr/17.webp)



"Open For Me" จะกรองเฉพาะการสลับที่โหนดของคุณตรงตามเงื่อนไขที่กำหนดเท่านั้น



![Détails d'un swap](assets/fr/18.webp)



ตัวอย่างรายละเอียดการสวอป :




- การกำหนดค่าห้าเหลี่ยม (ผู้เข้าร่วม 5 คน)
- ความจุ 300k Sats ต่อช่อง
- ข้อกำหนดเบื้องต้น: ช่องเปิดขั้นต่ำ 10 ช่องที่มีความจุรวม 1M Sats
- สถานที่ว่าง: 4/5

   - ในพารามิเตอร์ เลือก "สร้างหรือเชื่อมต่อ Wallet"

ใน Zeus:




   - เลือก "LND (REST)" เป็นประเภทการเชื่อมต่อ
   - คุณสามารถสแกน QR โค้ด (วิธีที่แนะนำ) หรือป้อนข้อมูลด้วยตนเอง (อย่าลังเลที่จะซูมเข้าไปที่ QR โค้ด Umbrel เนื่องจากมีความหนาแน่นมาก)
   - สำคัญ: เปิดใช้งานตัวเลือก "Use Tor" หาก Umbrel ของคุณสามารถเข้าถึงได้ผ่าน Tor เท่านั้น (ค่าเริ่มต้น)
   - บันทึกการกำหนดค่า



Zeus ของคุณเชื่อมต่อกับโหนด Umbrel ของคุณแล้วและช่วยให้คุณสามารถทำการชำระเงินด้วย Lightning ดูช่องทาง ยอดคงเหลือ ฯลฯ ในขณะที่ยังคงจัดการด้วยตนเองอย่างสมบูรณ์



**ตัวเลือกการเชื่อมต่อขั้นสูง:**



ตามค่าเริ่มต้น การเชื่อมต่อ Zeus ↔ Umbrel จะผ่าน Tor สำหรับการเชื่อมต่อที่เร็วขึ้น มีสองทางเลือก:



**Lightning Node Connect (LNC)** :




   - กลไกการเชื่อมต่อเข้ารหัสของ Lightning Labs
   - ติดตั้งแอป Lightning Terminal บน Umbrel (รวมถึงการเข้าถึง LNC)
   - สร้างรหัส QR การเชื่อมต่อใน Lightning Terminal (เชื่อมต่อ → เชื่อมต่อ Zeus ผ่าน LNC)
   - สแกนเข้า Zeus (เลือก "LNC" เป็นประเภทการเชื่อมต่อ)



**VPN Tailscale**:




   - VPN แบบตาข่ายที่กำหนดค่าได้ง่าย
   - ติดตั้ง Tailscale บน Umbrel (App Store) และบนโทรศัพท์มือถือของคุณ
   - เชื่อมต่อ Zeus ผ่าน IP ส่วนตัว Tailscale แทนที่อยู่ Tor



ตัวเลือกเหล่านี้ไม่จำเป็น และโซลูชัน Tor + Zeus ใช้งานได้ดีในกรณีส่วนใหญ่

## 5. ความปลอดภัยและแนวทางปฏิบัติที่ดีที่สุด



การจัดการโหนด Lightning ที่โฮสต์เองต้องให้ความสำคัญเป็นพิเศษกับความปลอดภัย



### การสำรองข้อมูลและความปลอดภัยสำหรับโหนดของคุณ



**ประเภทสำคัญของการสำรองข้อมูล**



โหนด Lightning Umbrel ของคุณต้องการการสำรองข้อมูลสองประเภท:



**ประโยค seed (24 คำ)**




- กู้คืนเงินทุน On-Chain
- จำเป็นต้องสร้าง Wallet Lightning ของคุณใหม่
- สำหรับการจัดเก็บที่ปลอดภัยสูงสุด (ออฟไลน์, บนกระดาษ)



ไฟล์ **Static Channel Backup (SCB)**




- ประกอบด้วยข้อมูลช่องทาง Lightning
- เปิดใช้งานการปิดช่องทางบังคับในกรณีที่เกิดการขัดข้อง
- สำคัญ:** ห้ามบันทึกไฟล์ `channel.db` ด้วยตนเอง (เสี่ยงต่อการถูกลงโทษ)



**ขั้นตอนการสำรองข้อมูลด้วยตนเอง**



ในการบันทึกช่องของคุณด้วยตนเอง :


1. เข้าถึงเมนู Lightning Node (จุดสามจุด "⋮" ถัดจาก "+ Open Channel")


2. ดาวน์โหลดไฟล์สำรองข้อมูลช่อง


3. ส่งออก SCB ใหม่หลังจากการแก้ไขช่องแต่ละครั้ง


4. เก็บรักษา SCB อย่างปลอดภัย (สื่อที่เข้ารหัส, สำเนานอกสถานที่)



**Umbrel** ระบบสำรองข้อมูลอัตโนมัติ



Umbrel มีระบบสำรองข้อมูลอัตโนมัติที่ซับซ้อนซึ่งรับประกันว่า :



*การปกป้องข้อมูล:*




- การเข้ารหัสฝั่งไคลเอ็นต์ก่อนการส่ง
- ส่งผ่านเครือข่าย Tor
- ข้อมูลที่เสริมด้วยการเติมแบบสุ่ม
- คีย์การเข้ารหัสที่ไม่ซ้ำกันสำหรับอุปกรณ์ของคุณ



*การรักษาความปลอดภัยที่เพิ่มขึ้น:*




- การสำรองข้อมูลทันทีเมื่อมีการเปลี่ยนแปลงสถานะ
- "สำรองข้อมูลหลอก" ในช่วงเวลาสุ่ม
- ซ่อนการเปลี่ยนแปลงขนาดสำรองข้อมูล
- การป้องกันการวิเคราะห์เวลา



*กระบวนการฟื้นฟู:*




- ตัวระบุและคีย์ที่ได้มาจาก seed Umbrel
- การกู้คืนอย่างสมบูรณ์เป็นไปได้ด้วยวลีช่วยจำเท่านั้น
- การกู้คืนอัตโนมัติของข้อมูลสำรองล่าสุด
- คืนค่าการตั้งค่าและข้อมูลของช่อง



### การกู้คืนจากการขัดข้อง



หากโหนดของคุณสูญหาย (ฮาร์ดแวร์ล้มเหลว, การ์ด SD เสียหาย) :




- ติดตั้งใหม่ Umbrel
- ป้อนคำ 24 คำของคุณ seed ในแอป Lightning
- นำเข้าไฟล์ SCB ระหว่างการกู้คืน



LND จะติดต่อพันธมิตรแต่ละรายของช่องเก่าของคุณเพื่อปิดช่องเหล่านั้นและกู้คืนส่วนแบ่งของคุณจากกองทุน On-Chain ช่องจะถูกปิดอย่างถาวร (สามารถเปิดใหม่ได้หากจำเป็น)



### ความพร้อมใช้งานและการป้องกันการฉ้อโกง



ในอุดมคติ ควรปล่อยให้เครือข่ายของคุณออนไลน์บ่อยที่สุดเท่าที่จะเป็นไปได้ ในกรณีที่ต้องขาดการเชื่อมต่อเป็นเวลานาน:




- เพียร์ที่เป็นอันตรายอาจพยายามกระจายสถานะช่องเก่าที่ผ่านมา
- Lightning มี "ช่วงเวลาประท้วง" (ประมาณ 2 สัปดาห์บน LND)
- หากคุณจะไม่อยู่เป็นเวลานาน ให้ตั้งค่า Watchtower



**Watchtower configuration:**




- ในการตั้งค่าขั้นสูงของ LND ให้เพิ่ม URL ของเซิร์ฟเวอร์ Watchtower ที่เชื่อถือได้
- คุณสามารถใช้บริการสาธารณะหรือทำการติดตั้ง Watchtower ของคุณเอง




หากต้องการทราบข้อมูลเพิ่มเติมเกี่ยวกับการกำหนดค่าและการใช้หอคอยเฝ้าระวัง เราขอแนะนำให้คุณดูบทแนะนำเฉพาะของเรา :



https://planb.academy/tutorials/node/lightning-network/watch-tower-26937006-dfe5-404e-9ee4-e82e422c5cf2
### แนวปฏิบัติที่ดีที่สุดอื่น ๆ





- อัปเดตซอฟต์แวร์:** รักษา Umbrel และ LND ให้ทันสมัย (การแก้ไขความปลอดภัย)
- การป้องกันฮาร์ดแวร์:** ใช้ระบบที่เสถียร (Raspberry Pi พร้อม SSD, mini-PC) และ UPS
- ความปลอดภัยของเครือข่าย:** รักษาค่าการตั้งค่าเริ่มต้นของ Tor เปลี่ยนรหัสผ่านผู้ดูแลระบบ Umbrel (ค่าเริ่มต้น: "moneyprintergobrrr")
- การเข้ารหัส:** เปิดใช้งานการเข้ารหัสดิสก์หากเป็นไปได้



## 6. เครื่องมือเพิ่มเติม



แอป Lightning Node ของ Umbrel มอบสิ่งจำเป็นสำหรับการจัดการช่องของคุณ แต่เครื่องมือของบุคคลที่สามมีฟังก์ชันการทำงานขั้นสูง



### ThunderHub



Interface ระบบการจัดการโหนด Lightning บนเว็บสมัยใหม่ที่สามารถติดตั้งได้ผ่าน Umbrel App Store.



**คุณสมบัติ:**




- การแสดงผลแบบเรียลไทม์ของช่องทาง (ความจุ, ยอดคงเหลือ)
- เครื่องมือปรับสมดุลแบบบูรณาการ
- รองรับการเรียกเก็บเงินแบบหลายเส้นทาง (MPP)
- การสร้างรหัส QR LNURL
- การจัดการธุรกรรม On-Chain



ThunderHub เหมาะอย่างยิ่งสำหรับการตรวจสอบโหนดการกำหนดเส้นทางที่ใช้งานอยู่และการปรับสมดุลอย่างง่าย



### ขี่สายฟ้า (RTL)



Interface เว็บที่เข้ากันได้กับการใช้งาน Lightning หลายรูปแบบ (LND, Core Lightning, Eclair)



**ไฮไลท์:**




- การจัดการหลายโหนด
- แดชบอร์ดที่ตอบสนองตามบริบท
- การสนับสนุนสำหรับการแลกเปลี่ยนใต้น้ำ (Lightning Loop)
- การยืนยันตัวตนแบบสองปัจจัย
- ส่งออก/นำเข้าสำรองข้อมูลช่องทาง



RTL เป็น "มีดพับสวิส" ที่ครบครันสำหรับการจัดการโหนด Lightning ด้วยวิธีการที่เน้นผู้เชี่ยวชาญมากขึ้น



### เครื่องมือที่มีประโยชน์อื่นๆ





- Lightning Shell**: บรรทัดคำสั่ง (lncli) ผ่านเบราว์เซอร์
- BTC RPC Explorer & Mempool**: Monitoring Blockchain
- LNmetrics & Torq**: การวิเคราะห์ประสิทธิภาพการกำหนดเส้นทาง
- Amboss & 1ML**: "การจัดการทางสังคม" ของโหนดของคุณ (นามแฝง, รายชื่อผู้ติดต่อ, การวิเคราะห์เครือข่าย)



เครื่องมือเหล่านี้สามารถติดตั้งได้เพียงไม่กี่คลิกผ่าน Umbrel App Store โดยไม่ต้องมีการกำหนดค่าที่ซับซ้อนใดๆ

## บทสรุป



การรันโหนด LND ของคุณเองบน Umbrel เป็นขั้นตอนสำคัญสู่ความเป็นอิสระทางการเงิน แม้ว่าจะต้องมีการมีส่วนร่วมทางเทคนิคมากกว่าการใช้โซลูชันแบบผู้ดูแล แต่ประโยชน์ในแง่ของการควบคุม ความเป็นส่วนตัว และการมีส่วนร่วมอย่างแข็งขันในเครือข่าย Lightning นั้นมีความสำคัญอย่างมาก



โดยทำตามคู่มือนี้ คุณควรจะสามารถติดตั้ง LND เปิดช่องทาง จัดการสภาพคล่องของคุณ และเข้าถึงโหนดของคุณจากระยะไกลได้แล้ว อย่าลังเลที่จะค่อยๆ สำรวจฟีเจอร์ขั้นสูงและเครื่องมือเพิ่มเติมเมื่อคุณคุ้นเคยกับระบบนิเวศมากขึ้น



โปรดจำไว้ว่าความปลอดภัยของเงินทุนของคุณขึ้นอยู่กับการป้องกันและการปฏิบัติของคุณ ใช้เวลาในการทำความเข้าใจทุกแง่มุมก่อนที่จะลงทุนจำนวนมาก
