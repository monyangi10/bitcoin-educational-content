## Запуск собственного узла с помощью Bitcoin core


Введение в Bitcoin и концепцию узла, дополненное исчерпывающим руководством по установке на Linux.


Одним из самых интересных аспектов Bitcoin является возможность самостоятельно запускать программу и таким образом участвовать на гранулированном уровне в работе сети и проверке публичной транзакции Ledger.


Bitcoin, как проект с открытым исходным кодом, находится в свободном доступе и публично распространяется с 2009 года. Спустя почти 17 лет после своего создания Bitcoin превратилась в надежную и неудержимую цифровую денежную сеть, пользующуюся мощным органическим сетевым эффектом. За свои усилия и видение Satoshi Накамото заслуживает нашей благодарности. Кстати, мы разместили документ о Bitcoin здесь, на Agora 256 (примечание: также на сайте университета).


### Станьте собственным банком


Запуск собственного узла стал необходимым для приверженцев этики Bitcoin. Не спрашивая ничьего разрешения, можно загрузить Blockchain с самого начала и таким образом проверить все транзакции от А до Я в соответствии с протоколом Bitcoin.


Программа также включает в себя собственный Wallet. Таким образом, мы контролируем транзакции, которые отправляем в остальную сеть, без каких-либо посредников или третьих лиц. Вы сами являетесь своим банком.


Поэтому остальная часть этой статьи представляет собой руководство по установке Bitcoin core - наиболее распространённой версии ПО Bitcoin - на совместимые с Debian дистрибутивы Linux, такие как Ubuntu и Pop!OS. Следуя этому руководству, вы сделаете ещё один шаг к своему индивидуальному суверенитету.


## Руководство по установке Bitcoin core для Debian/Ubuntu


**Пререквизиты**


- Минимум 6 ГБ памяти для хранения данных (узел pruned) - 1 ТБ памяти для хранения данных (Full node)
- Ожидайте, что операция *Initial Block Download* (IBD) займет не менее 24 часов. Эта операция обязательна даже для узла pruned.
- Обеспечьте ~600 ГБ пропускной способности для IBD, даже для узла pruned.


**Примечание:💡** следующие команды предопределены для Bitcoin core версии 24.1.


### Загрузка и проверка файлов



- [Скачать](https://bitcoincore.org/en/download/) `Bitcoin-24.1-x86_64-linux-gnu.tar.gz`, а также файлы `SHA256SUMS` и `SHA256SUMS.asc` (очевидно, вам нужно скачать последнюю версию программы).



- Откройте терминал в каталоге, где находятся загруженные файлы. Пример: `cd ~/Downloads/`.



- Убедитесь, что контрольная сумма файла версии указана в файле контрольных сумм, используя команду `sha256sum --ignore-missing --check SHA256SUMS`.



- В выводе этой команды должно быть имя загруженного файла версии, за которым следует `OK`. Example: `Bitcoin-24.0.1-x86_64-linux-gnu.tar.gz: OK`.



- Установите git с помощью команды `sudo apt install git`. Затем клонируйте репозиторий, содержащий PGP-ключи подписчиков Bitcoin core, с помощью команды `git clone https://github.com/Bitcoin-core/guix.sigs`.



- Импортируйте PGP-ключи всех подписантов с помощью команды `gpg --import guix.sigs/builder-keys/*`



- Убедитесь, что файл контрольной суммы подписан PGP-ключами подписавших его лиц, с помощью команды `gpg --verify SHA256SUMS.asc`.



Каждая правильная подпись будет отображать строку, начинающуюся со слов: `gpg: Хорошая подпись` и другая строка, заканчивающаяся на: `Отпечаток первичного ключа: 133E AC17 9436 F14A 5CF1 B794 860F EB80 4E66 9320` (пример отпечатка PGP-ключа Питера Вуйля).


**Примечание💡:** не обязательно, чтобы все ключи подписи возвращали "OK". Фактически, может потребоваться только один. Пользователь может сам определить порог проверки PGP.


Вы можете игнорировать предупреждения:



- `Этот ключ не заверен доверенной подписью!



- нет никаких признаков того, что подпись принадлежит владельцу


### Установка графического Bitcoin core Interface



- В терминале, все еще находясь в директории, где расположен файл версии Bitcoin core, используйте команду `tar xzf Bitcoin-24.1-x86_64-linux-gnu.tar.gz` для извлечения файлов, содержащихся в архиве.



- Установите ранее извлеченные файлы с помощью команды `sudo install -m 0755 -o root -g root -t /usr/local/bin Bitcoin-24.1/bin/*`



- Установите необходимые зависимости с помощью команды `sudo apt-get install libqt5gui5 libqt5core5a libqt5dbus5 qttools5-dev qttools5-dev-tools qtwayland5 libqrencode-dev`



- Запустите _bitcoin-qt_ (графический Bitcoin core, графический Interface) с помощью команды `Bitcoin-qt`.



- Чтобы выбрать узел pruned, установите флажок _Limit Blockchain storage_ и настройте предельное количество данных для хранения:

## Настройка Tor для узла Bitcoin core


**Примечание💡:** данное руководство предназначено для Bitcoin core 24.0.1 на Ubuntu/Debian совместимых дистрибутивах Linux.


### Установка и настройка Tor для Bitcoin core


Сначала нам нужно установить сервис Tor (The Onion Router) - сеть, используемую для анонимного общения, которая позволит нам анонимизировать взаимодействие с сетью Bitcoin. Для ознакомления с инструментами защиты конфиденциальности в Интернете, включая Tor, обратитесь к нашей статье на эту тему.


Чтобы установить Tor, откройте терминал и введите `sudo apt -y install tor`. После завершения установки служба будет автоматически запущена в фоновом режиме. Проверьте его корректную работу командой `sudo systemctl status tor`. В ответ должно появиться сообщение `Active: active (exited)`. Нажмите `Ctrl+C`, чтобы выйти из этой функции.


**Совет:** В любом случае вы можете использовать следующие команды в терминале для запуска, остановки или перезапуска Tor:


```shell
sudo systemctl start tor
sudo systemctl stop tor
sudo systemctl restart tor
```


Далее запустим графический Bitcoin core командой `Bitcoin-qt`. Затем включим автоматическую функцию программы для маршрутизации наших соединений через прокси-сервер Tor: _Настройки > Сеть_, а затем отметьте _Подключаться через прокси SOCKS5 (прокси по умолчанию)_, а также _Использовать отдельный прокси SOCKS5 для связи с пирами через луковые сервисы Tor_.


![option](assets/fr/04.webp)


Bitcoin core автоматически определяет, установлен ли Tor, и, если установлен, по умолчанию создает исходящие соединения с другими узлами, также использующими Tor, в дополнение к соединениям с узлами, использующими сети IPv4/IPv6 (clearnet).


**Примечание💡:** чтобы изменить язык отображения на французский, перейдите на вкладку Дисплей в Настройках.


### Расширенная настройка Tor (опционально)


Можно настроить Bitcoin core так, чтобы он использовал только сеть Tor для соединения с пирами, оптимизируя таким образом анонимность через наш узел. Поскольку в графическом Interface нет встроенной функции для этого, нам придется вручную создать файл конфигурации. Перейдите в раздел "Настройки", затем "Параметры".


![option 2](assets/fr/05.webp)


Здесь нажмите на кнопку _Открыть файл конфигурации_. Находясь в текстовом файле `Bitcoin.conf`, просто добавьте строку `onlynet=onion` и сохраните файл. Чтобы эта команда вступила в силу, необходимо перезапустить Bitcoin core.


Затем мы настроим службу Tor таким образом, чтобы Bitcoin core мог принимать входящие соединения через прокси, что позволит другим узлам сети использовать наш узел для загрузки данных Blockchain без ущерба для безопасности нашей машины.


В терминале введите `sudo nano /etc/tor/torrc`, чтобы получить доступ к файлу конфигурации сервиса Tor. В этом файле найдите строку `#ControlPort 9051` и удалите `#`, чтобы включить ее. Теперь добавьте в файл две новые строки:


```
HiddenServiceDir /var/lib/tor/bitcoin-service/
HiddenServicePort 8333 127.0.0.1:8334
```


Чтобы выйти из файла во время его сохранения, нажмите `Ctrl+X > Y > Enter`. Вернувшись в терминал, перезапустите Tor, введя команду ``sudo systemctl restart tor`.


При такой конфигурации Bitcoin core сможет устанавливать входящие и исходящие соединения с другими узлами сети только через сеть Tor (Onion). Чтобы убедиться в этом, перейдите на вкладку _Окно_, затем _Peers_.


![Nodes Window](assets/fr/06.webp)


### Дополнительные ресурсы


В конечном итоге использование только сети Tor (`onlynet=onion`) может сделать вас уязвимым для Sybil Attack. Поэтому некоторые рекомендуют использовать многосетевую конфигурацию для снижения такого риска. Кроме того, все соединения IPv4/IPv6 будут направляться через прокси-сервер Tor после его настройки, как было указано ранее.


В качестве альтернативы, чтобы оставаться исключительно в сети Tor и снизить риск Sybil Attack, вы можете добавить Address другого доверенного узла в файл `Bitcoin.conf`, добавив строку `addnode=trusted_address.onion`. Вы можете добавить эту строку несколько раз, если хотите подключиться к нескольким доверенным узлам.


Чтобы просмотреть журналы узла Bitcoin, связанные с его взаимодействием с Tor, добавьте `debug=tor` в файл `Bitcoin.conf`. Теперь у вас будет соответствующая информация о Tor в отладочном журнале, который можно просмотреть в окне _Информация_ с помощью кнопки _Файл отладки_. Также можно просмотреть эти журналы непосредственно в терминале с помощью команды `bitcoind -debug=tor`.
