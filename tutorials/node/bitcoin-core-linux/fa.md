## اجرای نود خود با Bitcoin core


مقدمه‌ای بر Bitcoin و مفهوم یک نود، همراه با یک راهنمای جامع نصب بر روی لینوکس.


یکی از هیجان‌انگیزترین جنبه‌های Bitcoin توانایی اجرای برنامه به‌صورت مستقل است و به این ترتیب، مشارکت در سطح جزئی در شبکه و تأیید تراکنش عمومی Ledger امکان‌پذیر می‌شود.


Bitcoin، به عنوان یک پروژه متن‌باز، از سال 2009 به صورت رایگان در دسترس و به طور عمومی توزیع شده است. تقریباً 17 سال پس از آغاز به کار، Bitcoin اکنون به یک شبکه پولی دیجیتال قوی و غیرقابل توقف تبدیل شده است که از یک اثر شبکه‌ای ارگانیک قدرتمند بهره می‌برد. برای تلاش‌ها و دیدگاهشان، Satoshi Nakamoto شایسته قدردانی ماست. به هر حال، ما وایت‌پیپر Bitcoin را اینجا در Agora 256 میزبانی می‌کنیم (توجه: همچنین در دانشگاه).


### بانک خود شدن


اجرای نود خودتان برای پیروان اصول Bitcoin ضروری شده است. بدون نیاز به اجازه از کسی، می‌توانید Blockchain را از ابتدا دانلود کرده و بدین ترتیب تمام تراکنش‌ها را از A تا Z طبق پروتکل Bitcoin تأیید کنید.


این برنامه همچنین شامل Wallet خود است. بنابراین، ما کنترل کاملی بر تراکنش‌هایی که به بقیه شبکه ارسال می‌کنیم داریم، بدون هیچ واسطه یا شخص ثالثی. شما بانک خودتان هستید.


بنابراین، ادامه این مقاله راهنمای نصب Bitcoin core — پرکاربردترین نسخه نرم‌افزار Bitcoin — به‌طور خاص بر روی توزیع‌های لینوکس سازگار با دبیان مانند اوبونتو و Pop!OS است. با دنبال کردن این راهنما، یک قدم به حاکمیت فردی خود نزدیک‌تر شوید.


## راهنمای نصب Bitcoin core برای Debian/Ubuntu


**پیش‌نیازها**


- حداقل 6 گیگابایت فضای ذخیره‌سازی داده (گره pruned) — 1 ترابایت فضای ذخیره‌سازی داده (Full node)
- انتظار داشته باشید که *دانلود اولیه بلاک* (IBD) حداقل 24 ساعت طول بکشد. این عملیات حتی برای یک نود pruned نیز الزامی است.
- اجازه دهید ~600GB پهنای باند برای IBD، حتی برای یک نود pruned.


**توجه:💡** دستورات زیر برای نسخه Bitcoin core ورژن 24.1 از پیش تعریف شده‌اند.


### دانلود و تأیید فایل‌ها



- [Download](https://bitcoincore.org/en/download/) `Bitcoin-24.1-x86_64-linux-gnu.tar.gz`، و همچنین فایل‌های `SHA256SUMS` و `SHA256SUMS.asc` (بدیهی است که باید آخرین نسخه نرم‌افزار را دانلود کنید).



- یک ترمینال را در دایرکتوری که فایل‌های دانلود شده در آن قرار دارند باز کنید. مثال: `cd ~/Downloads/`.



- تأیید کنید که مجموع کنترلی فایل نسخه در فایل مجموع کنترلی فهرست شده است با استفاده از دستور `sha256sum --ignore-missing --check SHA256SUMS`.



- خروجی این دستور باید شامل نام فایل نسخه دانلود شده به دنبال `OK` باشد. مثال: `Bitcoin-24.0.1-x86_64-linux-gnu.tar.gz: OK`.



- با استفاده از دستور `sudo apt install git`، گیت را نصب کنید. سپس، مخزن حاوی کلیدهای PGP امضاکنندگان Bitcoin core را با استفاده از دستور `git clone https://github.com/Bitcoin-core/guix.sigs` کلون کنید.



- کلیدهای PGP همه امضاکنندگان را با استفاده از دستور `gpg --import guix.sigs/builder-keys/*` وارد کنید.



- تأیید کنید که فایل چکسام با کلیدهای PGP امضاکنندگان امضا شده است با استفاده از دستور `gpg --verify SHA256SUMS.asc`.



هر امضای معتبر خطی را نمایش می‌دهد که با: `gpg: Good signature` شروع می‌شود و خط دیگری که با: `Primary key fingerprint: 133E AC17 9436 F14A 5CF1 B794 860F EB80 4E66 9320` (مثالی از اثر انگشت کلید PGP پیتر ویلی) به پایان می‌رسد.


**توجه💡:** لازم نیست که همه کلیدهای امضاکننده "OK" برگردانند. در واقع، ممکن است فقط یکی لازم باشد. تعیین آستانه اعتبارسنجی برای تأیید PGP بر عهده کاربر است.


می‌توانید هشدارها را نادیده بگیرید:



- `این کلید با یک امضای معتبر تأیید نشده است!`



- `هیچ نشانه‌ای وجود ندارد که امضا متعلق به مالک باشد.`


### نصب Bitcoin core گرافیکی Interface



- در ترمینال، همچنان در دایرکتوری که فایل نسخه Bitcoin core قرار دارد، از دستور `tar xzf Bitcoin-24.1-x86_64-linux-gnu.tar.gz` برای استخراج فایل‌های موجود در آرشیو استفاده کنید.



- فایل‌های استخراج‌شده قبلی را با استفاده از دستور `sudo install -m 0755 -o root -g root -t /usr/local/bin Bitcoin-24.1/bin/*` نصب کنید.



- نصب وابستگی‌های لازم با استفاده از دستور `sudo apt-get install libqt5gui5 libqt5core5a libqt5dbus5 qttools5-dev qttools5-dev-tools qtwayland5 libqrencode-dev`



- شروع _bitcoin-qt_ (Bitcoin core گرافیکی Interface) با استفاده از فرمان `Bitcoin-qt`.



- برای انتخاب یک گره pruned، گزینه _Limit Blockchain storage_ را بررسی کرده و محدودیت داده‌ای که باید ذخیره شود را پیکربندی کنید:

## پیکربندی Tor برای یک گره Bitcoin core


**توجه💡:** این راهنما برای Bitcoin core 24.0.1 بر روی توزیع‌های لینوکس سازگار با اوبونتو/دبیان طراحی شده است.


### نصب و پیکربندی Tor برای Bitcoin core


ابتدا، باید سرویس تور (The Onion Router) را نصب کنیم، شبکه‌ای که برای ارتباط ناشناس استفاده می‌شود و به ما اجازه می‌دهد تعاملات خود با شبکه Bitcoin را ناشناس کنیم. برای مقدمه‌ای بر ابزارهای حفاظت از حریم خصوصی آنلاین، از جمله تور، به مقاله ما در این زمینه مراجعه کنید.


برای نصب Tor، یک ترمینال باز کرده و دستور `sudo apt -y install tor` را وارد کنید. پس از اتمام نصب، سرویس معمولاً به‌طور خودکار در پس‌زمینه اجرا می‌شود. با دستور `sudo systemctl status tor` بررسی کنید که به‌درستی در حال اجرا است. پاسخ باید نشان دهد `Active: active (exited)`. برای خروج از این عملکرد `Ctrl+C` را فشار دهید.


**نکته:** در هر صورت، می‌توانید از دستورات زیر در ترمینال برای شروع، توقف یا راه‌اندازی مجدد Tor استفاده کنید:


```shell
sudo systemctl start tor
sudo systemctl stop tor
sudo systemctl restart tor
```


سپس، بیایید Bitcoin core گرافیکی Interface را با فرمان `Bitcoin-qt` راه‌اندازی کنیم. سپس، ویژگی خودکار نرم‌افزار را برای مسیریابی اتصالات ما از طریق یک پروکسی Tor فعال کنید: _Settings > Network_، و از آنجا _Connect through SOCKS5 proxy (default proxy)_ و همچنین _Use a separate SOCKS5 proxy to reach peers via Tor onion services_ را بررسی کنید.


![option](assets/fr/04.webp)


‏Bitcoin core به‌طور خودکار تشخیص می‌دهد که آیا Tor نصب شده است و در صورت نصب، به‌طور پیش‌فرض علاوه بر اتصال به نودهایی که از شبکه‌های IPv4/IPv6 (clearnet) استفاده می‌کنند، اتصالات خروجی به نودهای دیگری که از Tor استفاده می‌کنند نیز ایجاد می‌کند.


**توجه💡:** برای تغییر زبان نمایش به فرانسوی، به برگه نمایش در تنظیمات بروید.


### پیکربندی پیشرفته Tor (اختیاری)


امکان پیکربندی Bitcoin core برای استفاده انحصاری از شبکه Tor جهت اتصال به همتایان وجود دارد، که از این طریق می‌توانیم ناشناس بودن خود را از طریق نودمان بهینه کنیم. از آنجا که در Interface گرافیکی این قابلیت به صورت داخلی وجود ندارد، نیاز است که به صورت دستی یک فایل پیکربندی ایجاد کنیم. به تنظیمات بروید، سپس گزینه‌ها را انتخاب کنید.


![option 2](assets/fr/05.webp)


اینجا، روی _Open configuration file_ کلیک کنید. وقتی در فایل متنی `Bitcoin.conf` هستید، به سادگی یک خط `onlynet=onion` اضافه کنید و فایل را ذخیره کنید. برای اعمال این دستور، نیاز است که Bitcoin core را مجدداً راه‌اندازی کنید.


سپس سرویس Tor را پیکربندی خواهیم کرد تا Bitcoin core بتواند از طریق یک پروکسی اتصالات ورودی را دریافت کند و به سایر نودهای شبکه اجازه دهد تا از نود ما برای دانلود داده‌های Blockchain استفاده کنند، بدون اینکه امنیت دستگاه ما به خطر بیفتد.


در ترمینال، دستور `sudo nano /etc/tor/torrc` را وارد کنید تا به فایل پیکربندی سرویس Tor دسترسی پیدا کنید. در این فایل، به دنبال خط `#ControlPort 9051` بگردید و `#` را حذف کنید تا فعال شود. حالا دو خط جدید به فایل اضافه کنید:


```
HiddenServiceDir /var/lib/tor/bitcoin-service/
HiddenServicePort 8333 127.0.0.1:8334
```


برای خروج از فایل و ذخیره آن، `Ctrl+X > Y > Enter` را فشار دهید. در ترمینال، برای راه‌اندازی مجدد Tor، فرمان `sudo systemctl restart tor` را وارد کنید.


با این پیکربندی، Bitcoin core قادر خواهد بود تا اتصالات ورودی و خروجی با سایر گره‌های شبکه را تنها از طریق شبکه تور (Onion) برقرار کند. برای تأیید این موضوع، روی زبانه _Window_ کلیک کنید، سپس _Peers_ را انتخاب کنید.


![Nodes Window](assets/fr/06.webp)


### منابع اضافی


در نهایت، استفاده از شبکه تور به تنهایی (`onlynet=onion`) می‌تواند شما را در معرض خطر Sybil Attack قرار دهد. به همین دلیل برخی توصیه می‌کنند که یک پیکربندی چند شبکه‌ای را حفظ کنید تا این نوع ریسک را کاهش دهید. علاوه بر این، تمام اتصالات IPv4/IPv6 پس از پیکربندی از طریق پروکسی تور هدایت خواهند شد، همان‌طور که قبلاً اشاره شد.


به طور متناوب، برای باقی ماندن صرفاً در شبکه Tor و کاهش خطر Sybil Attack، می‌توانید Address یک نود مورد اعتماد دیگر را به فایل `Bitcoin.conf` خود اضافه کنید با افزودن خط `addnode=trusted_address.onion`. اگر می‌خواهید به چندین نود مورد اعتماد متصل شوید، می‌توانید این خط را چندین بار اضافه کنید.


برای مشاهده لاگ‌های نود Bitcoin خود که به‌طور خاص به تعامل آن با Tor مربوط می‌شود، `debug=tor` را به فایل `Bitcoin.conf` خود اضافه کنید. اکنون اطلاعات مربوط به Tor را در لاگ دیباگ خود خواهید داشت که می‌توانید آن را در پنجره _Information_ با دکمه _Debug File_ مشاهده کنید. همچنین می‌توانید این لاگ‌ها را مستقیماً در ترمینال با فرمان `bitcoind -debug=tor` مشاهده کنید.
