## การรันโหนดของคุณเองด้วย Bitcoin Core


บทนำสู่ Bitcoin และแนวคิดของโหนด พร้อมด้วยคู่มือการติดตั้งที่ครอบคลุมบน Linux


หนึ่งในแง่มุมที่น่าตื่นเต้นที่สุดของ Bitcoin คือความสามารถในการรันโปรแกรมด้วยตนเอง และด้วยเหตุนี้จึงสามารถมีส่วนร่วมในระดับที่ละเอียดในเครือข่ายและการตรวจสอบบัญชีธุรกรรมสาธารณะ


Bitcoin ในฐานะโครงการโอเพ่นซอร์ส ได้เปิดให้ใช้งานฟรีและเผยแพร่สู่สาธารณะตั้งแต่ปี 2009 เกือบ 17 ปีหลังจากการเริ่มต้น Bitcoin ได้กลายเป็นเครือข่ายการเงินดิจิทัลที่แข็งแกร่งและไม่หยุดยั้ง โดยได้รับประโยชน์จากผลกระทบของเครือข่ายอินทรีย์ที่ทรงพลัง สำหรับความพยายามและวิสัยทัศน์ของพวกเขา Satoshi Nakamoto สมควรได้รับความขอบคุณของเรา อย่างไรก็ตาม เราโฮสต์เอกสารไวท์เปเปอร์ของ Bitcoin ไว้ที่นี่บน Agora 256 (หมายเหตุ: ยังมีที่มหาวิทยาลัยด้วย)


### การเป็นธนาคารของตัวเอง


การรันโหนดของคุณเองได้กลายเป็นสิ่งสำคัญสำหรับผู้ที่ยึดมั่นในแนวคิด Bitcoin โดยไม่ต้องขออนุญาตจากใคร คุณสามารถดาวน์โหลดบล็อกเชนตั้งแต่ต้นและตรวจสอบธุรกรรมทั้งหมดจาก A ถึง Z ตามโปรโตคอล Bitcoin ได้


โปรแกรมยังรวมถึง wallet ของตัวเองด้วย ดังนั้นเราจึงสามารถควบคุมธุรกรรมที่เราส่งไปยังเครือข่ายที่เหลือได้โดยไม่มีตัวกลางหรือบุคคลที่สาม คุณคือธนาคารของคุณเอง


ส่วนที่เหลือของบทความนี้จึงเป็นคู่มือในการติดตั้ง Bitcoin Core — ซึ่งเป็นเวอร์ชันซอฟต์แวร์ Bitcoin ที่ใช้กันอย่างแพร่หลายที่สุด — โดยเฉพาะบนการแจกจ่าย Linux ที่เข้ากันได้กับ Debian เช่น Ubuntu และ Pop!OS ทำตามคู่มือนี้เพื่อก้าวไปอีกขั้นสู่ความเป็นอิสระส่วนบุคคลของคุณ


## คู่มือการติดตั้งแกน Bitcoin สำหรับ Debian/Ubuntu


**ข้อกำหนดเบื้องต้น**


- ขั้นต่ำ 6GB ของพื้นที่จัดเก็บข้อมูล (pruned node) — 1TB ของพื้นที่จัดเก็บข้อมูล (full node)
- คาดว่า *Initial Block Download* (IBD) จะใช้เวลาอย่างน้อย 24 ชั่วโมง การดำเนินการนี้เป็นสิ่งจำเป็นแม้กระทั่งสำหรับโหนด pruned
- อนุญาตแบนด์วิธ ~600GB สำหรับ IBD แม้กระทั่งสำหรับโหนด pruned


**หมายเหตุ:💡** คำสั่งต่อไปนี้ถูกกำหนดไว้ล่วงหน้าสำหรับ Bitcoin Core เวอร์ชัน 24.1.


### กำลังดาวน์โหลดและตรวจสอบไฟล์



- [Download](https://bitcoincore.org/en/download/) `bitcoin-24.1-x86_64-linux-gnu.tar.gz`, as well as the `SHA256SUMS` and `SHA256SUMS.asc` files (คุณจำเป็นต้องดาวน์โหลดเวอร์ชันล่าสุดของซอฟต์แวร์อย่างชัดเจน).



- เปิดเทอร์มินัลในไดเรกทอรีที่ไฟล์ที่ดาวน์โหลดถูกจัดเก็บอยู่ ตัวอย่าง: `cd ~/Downloads/`.



- ยืนยันว่า checksum ของไฟล์เวอร์ชันถูกระบุในไฟล์ checksum โดยใช้คำสั่ง `sha256sum --ignore-missing --check SHA256SUMS`



- เอาต์พุตของคำสั่งนี้ควรรวมชื่อของไฟล์เวอร์ชันที่ดาวน์โหลดตามด้วย `OK` ตัวอย่าง: `bitcoin-24.0.1-x86_64-linux-gnu.tar.gz: OK`.



- ติดตั้ง git โดยใช้คำสั่ง `sudo apt install git` จากนั้น โคลนที่เก็บข้อมูลที่มีคีย์ PGP ของผู้ลงนาม Bitcoin Core โดยใช้คำสั่ง `git clone https://github.com/bitcoin-core/guix.sigs`.



- นำเข้า PGP keys ของผู้ลงนามทั้งหมดโดยใช้คำสั่ง `gpg --import guix.sigs/builder-keys/*`



- ยืนยันว่าไฟล์ checksum ได้รับการลงนามด้วยกุญแจ PGP ของผู้ลงนามโดยใช้คำสั่ง `gpg --verify SHA256SUMS.asc`.



ลายเซ็นที่ถูกต้องแต่ละรายการจะแสดงบรรทัดที่ขึ้นต้นด้วย: `gpg: Good signature` และอีกบรรทัดที่ลงท้ายด้วย: `Primary key fingerprint: 133E AC17 9436 F14A 5CF1 B794 860F EB80 4E66 9320` (ตัวอย่างของลายนิ้วมือกุญแจ PGP ของ Pieter Wuille)


**หมายเหตุ💡:** ไม่จำเป็นที่คีย์ของผู้ลงนามทั้งหมดจะต้องแสดงผลเป็น "OK" ในความเป็นจริง อาจจำเป็นเพียงแค่หนึ่งคีย์เท่านั้น ขึ้นอยู่กับผู้ใช้ในการกำหนดเกณฑ์การตรวจสอบของตนเองสำหรับการยืนยัน PGP


คุณสามารถเพิกเฉยต่อคำเตือนได้:



- `คีย์นี้ไม่ได้รับการรับรองด้วยลายเซ็นที่เชื่อถือได้!`



- `ไม่มีข้อบ่งชี้ว่าลายเซ็นนั้นเป็นของเจ้าของ.`


### การติดตั้งอินเทอร์เฟซกราฟิก Bitcoin Core



- ในเทอร์มินัล ยังคงอยู่ในไดเรกทอรีที่ไฟล์เวอร์ชัน Bitcoin Core ตั้งอยู่ ใช้คำสั่ง `tar xzf bitcoin-24.1-x86_64-linux-gnu.tar.gz` เพื่อแยกไฟล์ที่อยู่ในไฟล์อาร์ไคฟ์ออกมา



- ติดตั้งไฟล์ที่แยกออกมาก่อนหน้านี้โดยใช้คำสั่ง `sudo install -m 0755 -o root -g root -t /usr/local/bin bitcoin-24.1/bin/*`



- ติดตั้ง dependencies ที่จำเป็นโดยใช้คำสั่ง `sudo apt-get install libqt5gui5 libqt5core5a libqt5dbus5 qttools5-dev qttools5-dev-tools qtwayland5 libqrencode-dev`



- เริ่ม _bitcoin-qt_ (อินเทอร์เฟซกราฟิก Bitcoin Core) โดยใช้คำสั่ง `bitcoin-qt`.



- ในการเลือกโหนด pruned ให้ตรวจสอบ _จำกัดการจัดเก็บบล็อกเชน_ และกำหนดขีดจำกัดข้อมูลที่จะจัดเก็บ:

## กำลังตั้งค่า Tor สำหรับโหนดหลัก Bitcoin


**หมายเหตุ💡:** คู่มือนี้ออกแบบมาสำหรับ Bitcoin Core 24.0.1 บนการแจกจ่าย Linux ที่รองรับ Ubuntu/Debian


### การติดตั้งและกำหนดค่า Tor สำหรับ Bitcoin Core


ก่อนอื่น เราจำเป็นต้องติดตั้งบริการ Tor (The Onion Router) ซึ่งเป็นเครือข่ายที่ใช้สำหรับการสื่อสารแบบไม่ระบุตัวตน ซึ่งจะช่วยให้เราสามารถทำให้การโต้ตอบกับเครือข่าย Bitcoin เป็นแบบไม่ระบุตัวตนได้ สำหรับการแนะนำเครื่องมือป้องกันความเป็นส่วนตัวออนไลน์ รวมถึง Tor โปรดดูบทความของเราในหัวข้อนี้


ในการติดตั้ง Tor ให้เปิดเทอร์มินัลและป้อน `sudo apt -y install tor` เมื่อการติดตั้งเสร็จสิ้น บริการจะถูกเปิดใช้งานโดยอัตโนมัติในพื้นหลังตามปกติ ตรวจสอบว่ามันทำงานอย่างถูกต้องด้วยคำสั่ง `sudo systemctl status tor` การตอบสนองควรแสดง `Active: active (exited)` กด `Ctrl+C` เพื่อออกจากฟังก์ชันนี้


**เคล็ดลับ:** ในทุกกรณี คุณสามารถใช้คำสั่งต่อไปนี้ในเทอร์มินัลเพื่อเริ่ม หยุด หรือเริ่มใหม่ Tor:


```shell
sudo systemctl start tor
sudo systemctl stop tor
sudo systemctl restart tor
```


ต่อไป ให้เราเปิดตัวอินเทอร์เฟซกราฟิก Bitcoin Core ด้วยคำสั่ง `bitcoin-qt` จากนั้นเปิดใช้งานฟีเจอร์อัตโนมัติของซอฟต์แวร์เพื่อกำหนดเส้นทางการเชื่อมต่อของเราผ่านพร็อกซี Tor: _Settings > Network_ และจากนั้นให้เลือก _Connect through SOCKS5 proxy (default proxy)_ รวมถึง _Use a separate SOCKS5 proxy to reach peers via Tor onion services_


![option](assets/fr/04.webp)


Bitcoin Core จะตรวจจับโดยอัตโนมัติว่ามีการติดตั้ง Tor หรือไม่ และหากมี จะสร้างการเชื่อมต่อขาออกไปยังโหนดอื่นที่ใช้ Tor โดยค่าเริ่มต้น นอกเหนือจากการเชื่อมต่อกับโหนดที่ใช้เครือข่าย IPv4/IPv6 (clearnet)


**หมายเหตุ💡:** หากต้องการเปลี่ยนภาษาการแสดงผลเป็นภาษาฝรั่งเศส ให้ไปที่แท็บการแสดงผลในการตั้งค่า


### การกำหนดค่า Tor ขั้นสูง (ไม่บังคับ)


สามารถกำหนดค่า Bitcoin Core ให้ใช้เฉพาะเครือข่าย Tor ในการเชื่อมต่อกับเพื่อนร่วมเครือข่ายได้ ซึ่งจะช่วยเพิ่มความเป็นนิรนามของเราผ่านโหนดของเรา เนื่องจากไม่มีฟังก์ชันนี้ในอินเทอร์เฟซกราฟิก เราจำเป็นต้องสร้างไฟล์การกำหนดค่าด้วยตนเอง ไปที่ การตั้งค่า แล้วเลือก ตัวเลือก


![option 2](assets/fr/05.webp)


ที่นี่ คลิกที่ _Open configuration file_ เมื่ออยู่ในไฟล์ข้อความ `bitcoin.conf` เพียงแค่เพิ่มบรรทัด `onlynet=onion` และบันทึกไฟล์ คุณจำเป็นต้องรีสตาร์ท Bitcoin Core เพื่อให้คำสั่งนี้มีผล


จากนั้นเราจะกำหนดค่าเซอร์วิส Tor เพื่อให้ Bitcoin Core สามารถรับการเชื่อมต่อขาเข้าผ่านพร็อกซีได้ ทำให้โหนดอื่นๆ ในเครือข่ายสามารถใช้โหนดของเราในการดาวน์โหลดข้อมูลบล็อกเชนโดยไม่กระทบต่อความปลอดภัยของเครื่องของเรา


ในเทอร์มินัล ให้ป้อน `sudo nano /etc/tor/torrc` เพื่อเข้าถึงไฟล์การกำหนดค่าบริการ Tor ในไฟล์นี้ ให้มองหาบรรทัด `#ControlPort 9051` และลบ `#` ออกเพื่อเปิดใช้งาน ตอนนี้เพิ่มสองบรรทัดใหม่ลงในไฟล์:


```
HiddenServiceDir /var/lib/tor/bitcoin-service/
HiddenServicePort 8333 127.0.0.1:8334
```


หากต้องการออกจากไฟล์พร้อมบันทึก ให้กด `Ctrl+X > Y > Enter` กลับไปที่เทอร์มินัล ให้รีสตาร์ท Tor โดยป้อนคำสั่ง `sudo systemctl restart tor`


ด้วยการกำหนดค่านี้ Bitcoin Core จะสามารถสร้างการเชื่อมต่อขาเข้าและขาออกกับโหนดอื่นๆ บนเครือข่ายได้เฉพาะผ่านเครือข่าย Tor (Onion) เท่านั้น เพื่อยืนยันสิ่งนี้ ให้คลิกที่แท็บ _Window_ จากนั้นเลือก _Peers_


![Nodes Window](assets/fr/06.webp)


### ทรัพยากรเพิ่มเติม


ในที่สุด การใช้เฉพาะเครือข่าย Tor (`onlynet=onion`) อาจทำให้คุณเสี่ยงต่อการถูกโจมตีแบบ Sybil นั่นคือเหตุผลที่บางคนแนะนำให้รักษาการกำหนดค่าหลายเครือข่ายเพื่อลดความเสี่ยงประเภทนี้ นอกจากนี้ การเชื่อมต่อ IPv4/IPv6 ทั้งหมดจะถูกส่งผ่านพร็อกซี Tor เมื่อมีการกำหนดค่า ตามที่ระบุไว้ก่อนหน้านี้


ในทางกลับกัน เพื่อที่จะอยู่บนเครือข่าย Tor เท่านั้นและลดความเสี่ยงของการโจมตี Sybil คุณสามารถเพิ่มที่อยู่ของโหนดที่เชื่อถือได้อีกโหนดหนึ่งไปยังไฟล์ `bitcoin.conf` ของคุณโดยการเพิ่มบรรทัด `addnode=trusted_address.onion` คุณสามารถเพิ่มบรรทัดนี้หลายครั้งหากคุณต้องการเชื่อมต่อกับโหนดที่เชื่อถือได้หลายโหนด


ในการดูบันทึกของโหนด Bitcoin ของคุณที่เกี่ยวข้องกับการทำงานร่วมกับ Tor โดยเฉพาะ ให้เพิ่ม `debug=tor` ลงในไฟล์ `bitcoin.conf` ของคุณ ตอนนี้คุณจะมีข้อมูล Tor ที่เกี่ยวข้องในบันทึกการดีบักของคุณ ซึ่งคุณสามารถดูได้ในหน้าต่าง _Information_ ด้วยปุ่ม _Debug File_ นอกจากนี้ยังสามารถดูบันทึกเหล่านี้ได้โดยตรงในเทอร์มินัลด้วยคำสั่ง `bitcoind -debug=tor`
