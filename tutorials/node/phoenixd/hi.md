वित्तीय स्वायत्तता का अर्थ आपके लाइटनिंग इंफ्रास्ट्रक्चर को नियंत्रित करना भी है। अपने अनुप्रयोगों में Bitcoin लाइटनिंग को एकीकृत करने के इच्छुक डेवलपर्स और कंपनियों के लिए, **फीनिक्सडी** एक आदर्श समाधान प्रस्तुत करता है: स्वचालित तरलता प्रबंधन वाला एक न्यूनतम, विशिष्ट लाइटनिंग नोड।



फीनिक्सड, ACINQ द्वारा विकसित एक लाइटनिंग सर्वर है, जिसे विशेष रूप से HTTP API के माध्यम से लाइटनिंग भुगतान भेजने और प्राप्त करने के लिए डिज़ाइन किया गया है। LND या कोर लाइटनिंग जैसे पूर्ण-विशेषताओं वाले कार्यान्वयनों के विपरीत, फीनिक्सड आपके धन की स्व-सुरक्षा को बनाए रखते हुए चैनल प्रबंधन की सभी जटिलताओं को दूर करता है।



इस ट्यूटोरियल में, हम देखेंगे कि स्व-होस्टेड बुनियादी ढांचे और उपयोग में आसान API के साथ लाइटनिंग अनुप्रयोगों को विकसित करने के लिए फीनिक्सड को कैसे स्थापित, कॉन्फ़िगर और उपयोग किया जाए।



## फीनिक्स्ड क्या है?



फीनिक्सड, ACINQ द्वारा विकसित एक न्यूनतम, विशिष्ट लाइटनिंग नोड है। यह उन डेवलपर्स और उद्यमों के लिए डिज़ाइन किया गया एक समाधान है जो Full node की प्रबंधन जटिलता के बिना लाइटनिंग को अपने अनुप्रयोगों में एकीकृत करना चाहते हैं।



### संचालन सिद्धांत



**फीनिक्सडी एक न्यूनतम लाइटनिंग नोड है जो स्वचालित तरलता के लिए ACINQ को अपने LSP (लाइटनिंग सेवा प्रदाता) के रूप में उपयोग करता है। जब आप लाइटनिंग भुगतान प्राप्त करते हैं, तो यह आवश्यक इनकमिंग क्षमता आवंटित करने के लिए ACINQ नोड्स के साथ स्वचालित रूप से चैनल खोल देता है। यह "ऑन-द-फ्लाई" तरलता तत्काल होती है, लेकिन प्राप्त राशि का ठीक **1% + Mining शुल्क** लिया जाता है।



**स्वचालित प्रबंधन:** यह प्रणाली तीन प्रमुख Elements का प्रबंधन करती है:




- लाइटनिंग** चैनल: आवश्यकतानुसार स्वचालित रूप से खोलें, बंद करें और प्रबंधित करें
- आवक/जावक तरलता**: स्प्लिसिंग और चैनल खोलने के माध्यम से स्वचालित प्रावधान
- शुल्क क्रेडिट**: किसी चैनल को उचित ठहराने के लिए अपर्याप्त छोटे भुगतानों को भविष्य के शुल्कों के प्रावधान के रूप में संग्रहीत किया जाता है



### फीनिक्सड के लाभ



**आप अपनी निजी कुंजियों (12-शब्द seed) और निधियों को नियंत्रित करते हैं। फ़ीनिक्स्ड आपकी कुंजियों को साझा किए बिना ही स्थानीय रूप से आपकी Wallet उत्पन्न करता है।



**व्यक्तिगत बुनियादी ढाँचा:** फीनिक्सडी आपके सर्वर पर चलता है, जिससे आपको विस्तृत लॉग, कॉन्फ़िगरेशन और एपीआई नियंत्रण तक पहुँच मिलती है। अब आप अपने फंड तक पहुँच के लिए किसी तृतीय-पक्ष सेवा पर निर्भर नहीं हैं।



**एकीकृत API:** फीनिक्सड में अन्य सेवाओं के साथ एकीकरण, मूल LNURL समर्थन और कस्टम अनुप्रयोग विकास के लिए HTTP API की सुविधा है।



**एकीकरण में आसानी:** इसके सरल REST API के कारण, फीनिक्स्ड को लाइटनिंग भुगतान की आवश्यकता वाले किसी भी एप्लिकेशन या सेवा में एकीकृत किया जा सकता है।



**महत्वपूर्ण नोट:** स्वचालित तरलता अभी भी ACINQ से LSP (लाइटनिंग सर्विस प्रोवाइडर) के रूप में आती है। फ़ीनिक्सड स्वचालित चैनल प्रबंधन के लिए फ़ीनिक्स मोबाइल जैसी ही प्रणाली का उपयोग करता है।



## फीनिक्सडी स्थापित करना



### आवश्यक शर्तें



फीनिक्सड के लिए एक लिनक्स वातावरण (उबंटू/डेबियन अनुशंसित) और कुछ बुनियादी कमांड-लाइन कौशल की आवश्यकता होती है। सर्वोत्तम प्रदर्शन के लिए, आपको निम्न की आवश्यकता होगी:





- लिनक्स सर्वर**: स्थिर कनेक्शन के साथ VPS या स्थानीय मशीन
- OpenJDK 21** : जावा रनटाइम वातावरण
- स्थिर इंटरनेट कनेक्शन**: Lightning Network के साथ समन्वय के लिए
- डोमेन नाम** (वैकल्पिक): API तक सुरक्षित HTTPS पहुँच के लिए

Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target

## कॉन्फ़िगरेशन और सुरक्षा



### कॉन्फ़िगरेशन फ़ाइल



फीनिक्सड स्वचालित रूप से आवश्यक पैरामीटर के साथ `~/.phoenix/phoenix.conf` बनाता है:



```conf
# Network (mainnet by default)
chain=mainnet

# Size of automatic channels and requested liquidity amount (in satoshis)
auto-liquidity=2000000

# API configuration
http-bind-address=127.0.0.1
http-bind-port=9740
http-password=auto_generated_password
http-password-limited-access=limited_password
```



**मुख्य पैरामीटर:**




- `auto-liquidity`: स्वचालित रूप से खुलने वाले चैनलों का आकार (डिफ़ॉल्ट: 2M Sats)
- http-password`: API के लिए व्यवस्थापक पासवर्ड (Invoice निर्माण और भुगतान प्रेषण)
- http-password-limited-access`: प्रतिबंधित पासवर्ड (केवल Invoice निर्माण)



### HTTPS के साथ सुरक्षित पहुँच



डिफ़ॉल्ट रूप से, फीनिक्सडी एपीआई केवल स्थानीय HTTP (`http://127.0.0.1:9740`) के माध्यम से ही सुलभ है। अपने नोड को बाहर से (मोबाइल एप्लिकेशन, अन्य सर्वर, वेब इंटीग्रेशन) उपयोग करने के लिए, आपको सुरक्षित HTTPS एक्सेस कॉन्फ़िगर करना होगा।



**रिवर्स प्रॉक्सी सिद्धांत:**


```
Internet → nginx (port 443 HTTPS) → Phoenixd (port 9740 HTTP local)
```



**Nginx** एक **रिवर्स प्रॉक्सी** के रूप में कार्य करता है: यह पोर्ट 443 पर इंटरनेट से HTTPS अनुरोधों को सुनता है, उन्हें स्थानीय रूप से Phoenixd (पोर्ट 9740) पर पुनर्निर्देशित करता है, फिर एन्क्रिप्टेड प्रतिक्रियाओं को क्लाइंट को वापस भेजता है।



**SSL/TLS** प्रमाणपत्र एक डिजिटल फ़ाइल है जो:




- अपने सर्वर की पहचान प्रमाणित करें** (मैन-इन-द-मिडिल हमलों को रोकता है)
- HTTPS** एन्क्रिप्शन सक्षम करता है: आपके API पासवर्ड सहित सभी डेटा, परिवहन के दौरान एन्क्रिप्ट किया जाता है
- लेट्स एनक्रिप्ट द्वारा सर्टबॉट टूल के माध्यम से निःशुल्क जारी किया गया**



यह कॉन्फ़िगरेशन आपको इसकी अनुमति देता है:




- इंटरनेट से API तक सुरक्षित पहुंच**
- परिवहन के दौरान अपने API** पासवर्ड को एन्क्रिप्ट करें (ताकि उन्हें स्पष्ट पाठ में प्रेषित होने से रोका जा सके)
- फीनिक्सडी** को HTTPS की आवश्यकता वाले बाहरी अनुप्रयोगों में एकीकृत करें
- वित्तीय API के लिए सुरक्षा मानकों** का अनुपालन



इस HTTPS रिवर्स प्रॉक्सी को nginx के साथ कॉन्फ़िगर करें:



**1. Nginx** कॉन्फ़िगरेशन



```bash
sudo apt install nginx certbot python3-certbot-nginx
sudo nano /etc/nginx/sites-available/phoenixd.conf
```



```nginx
server {
listen 80;
server_name phoenixd.your-domain.com;

location / {
proxy_pass http://127.0.0.1:9740;
proxy_set_header X-Real-IP $remote_addr;
proxy_set_header Host $host;
}
}
```



```bash
sudo ln -s /etc/nginx/sites-available/phoenixd.conf /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx
```



**2. एसएसएल** प्रमाणपत्र



```bash
sudo certbot --nginx -d phoenixd.your-domain.com
```

**मूलभूत संचालन:**


```bash
# Create an invoice
curl -X POST http://localhost:9740/createinvoice \
-u :your_password \
-d description='Test payment' \
-d amountSat=100000

# Send a payment (routing fee 0.4%)
curl -X POST http://localhost:9740/payinvoice \
-u :your_password \
-d invoice='lnbc...'

# Check balance
curl http://localhost:9740/getbalance \
-u :your_password

# Send on-chain funds (in case of channel closure)
./phoenix-cli sendtoaddress \
--address bc1q... \
--amountSat 50000 \
--feerateSatByte 12
```



**लागत पर महत्वपूर्ण:**




- रसीद**: स्वचालित तरलता के लिए 1% + Mining शुल्क
- शिपिंग**: Lightning Network पर 0.4% रूटिंग शुल्क



**वेबहुक:** वेबहुक, फीनिक्सड को आपके एप्लिकेशन को किसी भी घटना (भुगतान प्राप्त, Invoice भुगतान, चैनल खोला गया, आदि) के घटित होने पर **स्वचालित रूप से सूचित** करने में सक्षम बनाते हैं। फीनिक्सड से लगातार अपडेट मांगने के बजाय, आपके एप्लिकेशन को तुरंत HTTP सूचना प्राप्त होती है।



**जब कोई ग्राहक किसी ऑर्डर के लिए भुगतान करता है तो आपके ऑनलाइन स्टोर को स्वचालित रूप से एक सूचना प्राप्त होती है, जिससे लेनदेन का तुरंत सत्यापन संभव हो जाता है।



`phoenix.conf` में कॉन्फ़िगरेशन:


```conf
webhook-url=https://your-app.com/webhook-phoenixd
webhook-secret=votre_secret_de_verification
```



## उन्नत अनुप्रयोग



### LNURL एकीकरण



फीनिक्स्ड उन्नत एकीकरण के लिए मूल रूप से LNURL प्रोटोकॉल का समर्थन करता है:



**LNURL-पे:** LNURL-संगत सेवाओं के लिए भुगतान करें


```bash
curl -X POST http://localhost:9740/lnurlpay \
-u :your_password \
-d lnurl=LNURL1DP68GURN8GHJ7MRWW4EXCTN... \
-d amountSat=100
```



**LNURL-Withdraw :** LNURL सेवाओं से धन प्राप्त करें


```bash
curl -X POST http://localhost:9740/lnurlwithdraw \
-u :your_password \
-d lnurl=lightning:LNURL1DP68GURN8GHJ7MRW...
```



**LNURL-Auth:** सेवाओं तक पहुँचने के लिए लाइटनिंग के माध्यम से प्रमाणीकरण


```bash
curl -X POST http://localhost:9740/lnurlauth \
-u :your_password \
-d lnurl=lnurl1dp68gurn8ghj7um5v93kket...
```



### LNbits के साथ एकीकरण



LNbits अपने [आधिकारिक दस्तावेज़](https://docs.lnbits.org/guide/wallets.html) के अनुसार, फ़ॉक्सिड को फंडिंग स्रोत के रूप में उपयोग कर सकता है:



**LNbits कॉन्फ़िगरेशन:**


```bash
LNBITS_BACKEND_WALLET_CLASS=PhoenixdWallet
PHOENIXD_API_ENDPOINT=http://localhost:9740/
PHOENIXD_API_PASSWORD=your_password_phoenixd
```



यह एकीकरण आपको अपने फीनिक्सड नोड द्वारा संचालित LNbits उप-खाते बनाने की अनुमति देता है, जो कई लाइटनिंग वॉलेट्स के प्रबंधन के लिए एक वेब-आधारित Interface प्रदान करता है।



### कस्टम अनुप्रयोग



इसके व्यापक REST API के लिए धन्यवाद, आप विकसित कर सकते हैं:



**ई-कॉमर्स:** आपके स्टोर में लाइटनिंग भुगतानों का सीधा एकीकरण


**दान सेवाएँ:** चालान और स्वचालित वेबहुक वाली दान प्रणालियाँ


**सोशल नेटवर्किंग बॉट्स:** टिप फ़ंक्शन वाले टेलीग्राम/डिस्कोर्ड बॉट्स


**पेवॉल लाइटनिंग:** प्रीमियम सामग्री लाइटनिंग शुल्क पर उपलब्ध है



## सुरक्षा और सर्वोत्तम प्रथाएँ



### पहुँच सुरक्षा



**एपीआई पासवर्ड:** स्वचालित रूप से जनरेट किए गए पासवर्ड आपके लाइटनिंग खजाने की चाबियाँ हैं। इन्हें कभी भी साझा न करें, और संदेह होने पर इन्हें बदल दें।



**फ़ायरवॉल:** पोर्ट 9740 को कभी भी सीधे इंटरनेट के लिए खुला न छोड़ें। हमेशा HTTPS के साथ nginx का उपयोग करें।



**उन्नत प्रमाणीकरण:** अपने सर्वर तक पहुंच को केवल अधिकृत डिवाइसों तक सीमित करने के लिए VPN या टेलस्केल पर विचार करें।



### आवश्यक बैकअप



**seed रिकवरी:** अपने 12 शब्दों को सर्वर से दूर किसी सुरक्षित जगह पर सेव करें। यही आपकी रिकवरी की एकमात्र गारंटी है।



*~/.phoenix निर्देशिका:** चैनल की स्थिति को सुरक्षित रखने और बहाली में तेजी लाने के लिए इस फ़ोल्डर का नियमित रूप से बैकअप लें (फीनिक्सडी बंद होने के बाद)।



**सेवा पुनर्प्राप्ति कोड:** उन सभी सेवाओं के लिए बैकअप कोड भी रखें जहां आप अपने फीनिक्स के साथ 2FA सक्रिय करते हैं।

- ❌ विस्तृत लॉग तक कोई पहुँच नहीं



**फीनिक्सडी (सर्वर):**

- ✅ लॉग तक पूर्ण पहुँच
- ✅ व्यक्तिगत बुनियादी ढांचा

- ❌ सर्वर रखरखाव आवश्यक

**फीनिक्सडी :**

## सामान्य समस्याओं का समाधान



### API एक्सेस समस्याएँ



**प्रमाणीकरण विफल" त्रुटि:**


1. `~/.phoenix/phoenix.conf` फ़ाइल में पासवर्ड जांचें


2. प्रमाणीकरण प्रारूप `-u:password` की जाँच करें


3. सुनिश्चित करें कि Phoenixd चल रहा है (`./phoenix-CLI getinfo`)



**कनेक्शन टाइमआउट:**




- जाँच करें कि फीनिक्सड सही पोर्ट (9740) पर सुन रहा है
- HTTPS कॉन्फ़िगर करने से पहले स्थानीय पहुँच का परीक्षण करें
- लॉग जांचें: `journalctl -u phoenixd -f`



### तरलता की समस्याएँ



**भुगतान नहीं आ रहा है :**


1. जाँच करें कि राशि न्यूनतम सीमा (~30k Sats) से अधिक है


2. चैनल त्रुटियों की पहचान करने के लिए लॉग देखें


3. यदि आवश्यक हो तो फीनिक्सडी को पुनः आरंभ करें



**"व्यय क्रेडिट" में शेष:**


छोटे भुगतान प्रावधान के रूप में जमा किए जाते हैं। चैनल खोलने और इन निधियों को जारी करने के लिए बड़ी राशि प्राप्त करें।



## निष्कर्ष



फीनिक्स्ड, डेवलपर्स के लिए उपयोग में आसानी और तकनीकी संप्रभुता के बीच एक बेहतरीन समझौता प्रस्तुत करता है। यह स्वचालित तरलता प्रबंधन के साथ एक सरल लेकिन शक्तिशाली लाइटनिंग एपीआई प्रदान करता है, जो पारंपरिक लाइटनिंग नोड्स की जटिलता को समाप्त करता है।



यह समाधान विशेष रूप से उन डेवलपर्स और कंपनियों के लिए उपयुक्त है जो:




- अपने अनुप्रयोगों में Bitcoin लाइटनिंग को एकीकृत करें
- लाइटनिंग चैनल प्रबंधन की जटिलता से बचें
- स्व-होस्टेड बुनियादी ढांचे से लाभ उठाएं
- एक सरल, विश्वसनीय API



फीनिक्सड के साथ, आप आधुनिक REST API और तकनीकी पहलुओं के स्वचालित प्रबंधन के साथ अपना निजी लाइटनिंग इंफ्रास्ट्रक्चर तैयार कर सकते हैं। यह आपकी परियोजनाओं में लाइटनिंग एकीकरण को लोकतांत्रिक बनाने के लिए एक आदर्श समाधान है।
