Dojo - это программа с открытым исходным кодом, предназначенная для работы в качестве внутреннего сервера для некоторых кошельков Bitcoin, ориентированных на приватность, основанных на узле Bitcoin core. Исторически она была разработана для работы с Samourai Wallet, мобильным Wallet, который предлагал расширенные функции приватности, такие как Whirlpool (CoinJoin), Ricochet, Stonewall, PayNym... Samourai Wallet был закрыт после ареста его разработчиков, но его преемник в сообществе, **Ashigaru Wallet**, взял на себя ответственность и продолжает полагаться на Dojo, чтобы предложить полный опыт для пользователей, желающих сохранить контроль над своими данными при использовании Bitcoin.



![Image](assets/fr/01.webp)



С практической точки зрения, Dojo выступает в роли шлюза между вашим Wallet и сетью Bitcoin. Без Dojo легкий мобильный Wallet должен был бы запрашивать сторонние серверы, чтобы получить статус ваших UTXO и вашу историю, или транслировать ваши транзакции. Это подразумевает зависимость и утечку конфиденциальных данных на сторонний сервер (используемые адреса, суммы, частота платежей и т. д.). С помощью Dojo вы сами размещаете этот сервер, напрямую подключенный к вашему собственному узлу Bitcoin. Таким образом, все ваши портфельные запросы проходят через инфраструктуру, которую вы контролируете, без посредников, что укрепляет вашу конфиденциальность и суверенитет.



## Требования для создания додзё



Для создания сервера Dojo не требуется сверхмощная машина. Любой человек, имеющий компьютер начального уровня, стабильное подключение к Интернету и возможность оставлять его включенным постоянно (24 часа в сутки 7 дней в неделю), может создать работающий Dojo.



### Выберите тип машины



Вы можете использовать :




- ноутбук ;
- настольный компьютер;
- мини-ПК (например, Intel NUC, Lenovo Thincentre Tiny...).



У каждого варианта есть свои преимущества и недостатки:




- Цена: отремонтированный мини-ПК или настольный компьютер часто стоит дешевле нового ноутбука.
- Компактность: мини-ПК занимает меньше места.
- Power Supply: ноутбук имеет преимущество в виде аккумулятора, а значит, он не отключится в случае перебоев с электричеством, в отличие от мини-ПК.
- Возможность модернизации: как правило, барбосы позволяют добавить память или легко заменить диск Hard.



Для получения более подробной информации о выборе оборудования я рекомендую вам пройти этот курс:

### Рекомендуемое оборудование



Нет необходимости покупать новую машину. Восстановленный компьютер с приведенными ниже характеристиками будет работать гораздо лучше, чем одноплатная электроника (например, Raspberry Pi).

## 2. Установка пристройки



Для правильной работы Dojo в вашей системе должны присутствовать определенные программные кирпичики. Они используются для управления репозиториями программного обеспечения, коммуникации, распаковки архивов и выполнения Dojo внутри контейнеров Docker. Все эти операции выполняются в терминале.



### 2.1. Подготовка



Следующая команда возвращает вас в личную папку. Это хорошая практика перед запуском серии установок.



```bash
cd ~/
```



Перед установкой любого программного обеспечения убедитесь, что база данных программ, доступных на вашем компьютере, актуальна. Это позволит избежать установки устаревших версий.



```bash
sudo apt-get update
```



![Image](assets/fr/07.webp)



### 2.2. установить утилиты



В систему необходимо добавить несколько инструментов:




- `apt-transport-https`: позволяет безопасно загружать пакеты через HTTPS
- `ca-certificates`: управляет сертификатами, необходимыми для зашифрованных соединений
- `curl`: для получения файлов из Интернета
- `gnupg-agent`: для управления ключами GPG
- software-properties-common`: предоставляет утилиты для работы с репозиториями APT
- `unzip`: распаковывает файлы в формате ZIP



```bash
sudo apt-get install apt-transport-https ca-certificates curl gnupg-agent software-properties-common unzip
```



Во время установки система может запросить подтверждение. Нажмите клавишу "*y*", затем нажмите "*Enter*".



![Image](assets/fr/08.webp)



### 2.3. установите Торсокс



Torsocks позволяет выполнять определенные команды через сеть Tor, что повышает конфиденциальность переписки.



```bash
sudo apt install torsocks
```



![Image](assets/fr/09.webp)



### 2.4. установите Docker и Docker Compose



Dojo работает внутри контейнеров Docker. Это означает, что каждый сервис изолирован в независимой среде, что упрощает обслуживание и безопасность. Для этого необходимо установить Docker и инструмент Docker Compose, который позволяет управлять несколькими контейнерами одновременно.



#### Добавьте ключ подписи Docker



Docker предоставляет собственный ключ цифровой подписи. Его добавление проверяет подлинность загруженных пакетов.



```bash
sudo apt-get update
sudo apt-get install ca-certificates curl
sudo install -m 0755 -d /etc/apt/keyrings
sudo curl -fsSL https://download.docker.com/linux/ubuntu/gpg -o /etc/apt/keyrings/docker.asc
sudo chmod a+r /etc/apt/keyrings/docker.asc
```



![Image](assets/fr/10.webp)



#### Добавлен официальный репозиторий Docker



Далее вам нужно указать системе, где найти официальные пакеты Docker. Эта команда добавляет новый репозиторий в конфигурацию менеджера пакетов.



```bash
echo \
"deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.asc] https://download.docker.com/linux/ubuntu \
$(. /etc/os-release && echo "*$VERSION_CODENAME*") stable" | \
sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

sudo apt-get update
```



![Image](assets/fr/11.webp)



#### Установка Docker и Docker Compose



Теперь можно установить основные компоненты Docker.



```bash
sudo apt-get install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
```



![Image](assets/fr/12.webp)



#### Авторизация пользователей



По умолчанию запустить Docker могут только команды, выполняемые с правами администратора. Для большего удобства я рекомендую добавить вашего текущего пользователя в группу "*docker*". Это позволит вам использовать Docker без необходимости каждый раз вводить `sudo.



```bash
sudo usermod -aG docker $USER
```



![Image](assets/fr/13.webp)



## 3. Создание одного пользователя (опционально)



Если вы хотите повысить безопасность своей системы, я рекомендую вам создать отдельного пользователя исключительно для работы с Dojo. Такое разделение ограничивает риски: если в Dojo возникнут проблемы с безопасностью, они не будут напрямую влиять на вашу основную учетную запись.



### 3.1. создание учетной записи пользователя



Следующая команда создает нового пользователя с именем "*dojo*". Этот пользователь будет иметь домашний каталог `/home/dojo` и доступ к терминалу bash. Он также будет добавлен в группу sudo, чтобы иметь возможность выполнять команды администратора.



```bash
sudo useradd -s /bin/bash -d /home/dojo -m -G sudo dojo
```



### 3.2. Установка пароля



Важно назначить для этой учетной записи надежный пароль. В идеале следует использовать менеджер паролей, например Bitwarden, чтобы generate длинную, Hard угадываемую комбинацию.



```bash
sudo passwd dojo
```



Система попросит вас ввести выбранный пароль, а затем подтвердить его второй раз.



https://planb.academy/tutorials/computer-security/authentication/bitwarden-0532f569-fb00-4fad-acba-2fcb1bf05de9

### 3.3. Разрешите пользователю использовать Docker



Чтобы пользователь "*dojo*" мог запускать контейнеры, необходимые для работы Dojo, его нужно добавить в группу Docker. Это избавит вас от необходимости предварять каждую команду командой `sudo.



```bash
sudo usermod -aG docker dojo
```



### 3.4. Перезапуск системы



Чтобы изменения в группе вступили в силу, необходимо перезагрузить машину.



```bash
sudo reboot
```



### 3.5. Войдите в систему с новым пользователем



После перезапуска системы войдите в систему под логином ***dojo*** и паролем, который вы задали ранее. Все последующие действия должны выполняться из этой учетной записи.



## 4. Загрузите и проверьте Dojo



Перед установкой Dojo необходимо убедиться, что файлы получены от официального разработчика и не были изменены. Этот шаг основан на использовании PGP и хэшей для проверки подлинности и целостности файлов.



### 4.1. импортируйте PGP-ключ разработчика



Скачайте открытый ключ разработчика через Tor и импортируйте его в свою локальную связку ключей. Этот ключ будет использоваться для проверки подписей, связанных с файлами Dojo.



```bash
torsocks wget http://zkaan2xfbuxia2wpf7ofnkbz6r5zdbbvxbunvp5g2iebopbfc4iqmbad.onion/vks/v1/by-fingerprint/E53AD419B242822F19E23C6D3033D463D6E544F6 && gpg --import E53AD419B242822F19E23C6D3033D463D6E544F6
```



![Image](assets/fr/14.webp)



### 4.2. загрузите последнюю версию Dojo



Получите сжатый архив с исходным кодом Dojo. В данном примере последней версией является `1.27.0`: измените команду в соответствии с [последняя версия здесь на официальном репозитории GitHub](https://github.com/Dojo-Open-Source-Project/samourai-dojo/releases).



```bash
torsocks wget -O samourai-dojo-1.27.0.zip https://github.com/Dojo-Open-Source-Project/samourai-dojo/archive/refs/tags/v1.27.0.zip
```



![Image](assets/fr/15.webp)



### 4.3. Загрузите отпечатки пальцев и подпись



Разработчики публикуют файл со списком цифровых отпечатков архивов, а также файл, подписанный их PGP-ключом. Загрузите их, чтобы сравнить свои файлы локально.



```bash
torsocks wget https://github.com/Dojo-Open-Source-Project/samourai-dojo/releases/download/v1.27.0/samourai-dojo-1.27.0-fingerprints.txt && torsocks wget https://github.com/Dojo-Open-Source-Project/samourai-dojo/releases/download/v1.27.0/samourai-dojo-1.27.0-fingerprints.txt.sig
```



![Image](assets/fr/16.webp)



### 4.4. Проверка подписи PGP



Проверьте, что файл отпечатков пальцев был подписан импортированным ключом.



```bash
gpg --verify samourai-dojo-1.27.0-fingerprints.txt.sig
```



Правильный результат показывает действительную подпись с ключом `E53AD419B242822F19E23C6D3033D463D6E544F6` и связанным с ним Address `dojocoder@pm.me`. Может появиться предупреждение о том, что ключ не сертифицирован: его можно проигнорировать.



Если же подпись недействительна, немедленно прекратите процесс установки и начните его с самого начала.



![Image](assets/fr/17.webp)



### 4.5. Проверьте целостность архива



Вычислите отпечаток SHA256 загруженного файла, затем откройте файл отпечатка и сравните два значения.



```bash
sha256sum samourai-dojo-1.27.0.zip
cat samourai-dojo-1.27.0-fingerprints.txt
```



Если отпечатки идентичны, вы можете быть уверены, что архив не был изменен. Если они разные, не идите дальше и удалите файлы.



![Image](assets/fr/18.webp)



### 4.6. Извлечение и упорядочивание файлов



После успешного завершения проверки вы можете распаковать архив и подготовить папку для установки Dojo.



```bash
unzip samourai-dojo-1.27.0.zip -d .
mkdir ~/dojo-app
mv ~/samourai-dojo-1.27.0/* ~/dojo-app/
```



![Image](assets/fr/19.webp)



### 4.7. Очистите ненужные файлы



Удалите временные файлы и архивы, которые больше не нужны, чтобы поддерживать чистоту среды.



```bash
rm -r samourai-dojo-1.27.0 && rm samourai-dojo-1.27.0.zip && rm samourai-dojo-1.27.0-fingerprints.txt && rm samourai-dojo-1.27.0-fingerprints.txt.sig && rm E53AD419B242822F19E23C6D3033D463D6E544F6
```



![Image](assets/fr/20.webp)



## 5. Конфигурация Dojo



Dojo - это бэкэнд-сервер, объединяющий несколько сервисов для взаимодействия с вашим портфелем и управления узлом Bitcoin. Его настройка может быть сложной, но проект предлагает упрощенный метод, который автоматически устанавливает и настраивает следующие компоненты:




- Dojo (основной API)
- Bitcoin core (полный узел Bitcoin)
- BTC-RPC Explorer (web Block explorer)
- Fulcrum Indexer (быстрое индексирование блоков и транзакций)
- Сервер Fulcrum Electrum Server доступен в сети Tor
- Сервер Fulcrum Electrum Server доступен в локальной сети
- Административные полномочия



### 5.1. учетные данные администратора



Чтобы защитить доступ к различным сервисам, вам нужно generate несколько уникальных идентификаторов:




- `BITCOIND_RPC_USER`
- `BITCOIND_RPC_PASSWORD`
- `MYSQL_ROOT_PASSWORD`
- mYSQL_USER
- `MYSQL_PASSWORD`
- nODE_API_KEY`
- `NODE_ADMIN_KEY`
- `NODE_JWT_SECRET`



Эти идентификаторы **должны быть уникальными** (это очень важно: вы не должны использовать один и тот же пароль для нескольких сервисов), состоять исключительно из цифр, заглавных и строчных букв (алфавитно-цифровых) и иметь длину около 40 символов, чтобы гарантировать высокий уровень безопасности. И снова я настоятельно рекомендую использовать менеджер паролей.



### 5.2. Доступ к файлам конфигурации

### 6.5. Понимание *Инициальной загрузки блока* (IBD)



При запуске Bitcoin core должен загрузить и проверить весь Blockchain с 2009 года. Этот шаг называется ***Initial Block Download* (IBD)**. Он очень важен, так как позволяет узлу Dojo проверять каждый блок и транзакцию Bitcoin независимо друг от друга.



Продолжительность этой синхронизации зависит от нескольких факторов:




- мощность вашего процессора и объем доступной оперативной памяти,
- скорость вашего диска,
- количество и качество пиров, с которыми соединяется ваш узел,
- скорость вашего интернет-соединения.



На практике эта операция обычно занимает от **2 до 7 дней**. В течение этого периода вы можете оставить машину работать постоянно. Чем дольше машина будет включена, тем быстрее завершится синхронизация. Я советую вам регулярно проверять статус синхронизации, просматривая журналы Bitcoin core или используя инструмент обслуживания Dojo после установки (см. следующий раздел).



Чтобы углубить свои знания о IBD и, в более общем смысле, о работе и роли узла Bitcoin, я рекомендую вам ознакомиться с этим курсом:



https://planb.academy/courses/3cd9cb94-82e8-417a-9c5a-02afc2589426


## 7. Контроль синхронизации



При первой установке Dojo вам необходимо дождаться полного завершения двух основных операций: полной загрузки Blockchain Bitcoin (*IBD*) и индексации этого Blockchain компанией Fulcrum. В зависимости от вашего соединения и мощности машины это может занять несколько дней. В течение этого времени вы можете следить за ходом процесса, чтобы убедиться, что все идет гладко.



Существует два способа отслеживания состояния синхронизации:




- использование инструмента Dojo Maintenance Tool (или DMT), который прост, но предоставляет мало деталей во время IBD;
- прямое изучение журналов Dojo на вашей машине, более техническое, но гораздо более точное.



### 7.1. Проверка с помощью инструмента обслуживания Dojo (DMT)



Dojo Maintenance Tool - это безопасный веб-инструмент Interface, позволяющий отслеживать состояние вашей установки и выполнять определенные операции. Это самый простой и доступный способ контроля за ходом работы IBD. На начальном этапе синхронизации отображаемая информация может быть ограничена. Например, DMT не показывает подробный ход индексации Fulcrum. С другой стороны, после завершения синхронизации DMT будет четко отображать :




- все огни на Green;
- последний проверенный блок для каждого сервиса (Node, Indexer, Dojo DB).



Чтобы получить к нему доступ, вам нужно знать URL вашего DMT и подключиться к нему [через браузер Tor](https://www.torproject.org/download/). Для этого откройте терминал и перейдите в директорию `/my-dojo`:



```bash
cd ~/dojo-app/docker/my-dojo
```



Затем выполните следующую команду:



```bash
./dojo.sh onion
```



![Image](assets/fr/32.webp)



После этого вы получите доступ ко всей информации, касающейся подключений к вашему Dojo через Tor. В данном случае нас интересует следующий URL:



```
Dojo API and Maintenance Tool =
```



Чтобы получить доступ к DMT с любой машины в любой сети (даже удаленно), откройте Tor Browser и введите этот URL, за которым следует `/admin`. Например, если ваш URL - `wo4zobymdl45gmmzzmpoypeemoukbj74wpibc22rxs2yfgpej62v6dyd.onion`, вам нужно будет ввести в строке [Tor Browser](https://www.torproject.org/download/):



```
wo4zobymdl45gmmzzmpoypeemoukbj74wpibc22rxs2yfgpej62v6dyd.onion/admin
```



⚠️ **Просьба сохранять эту Address в строгой тайне



После этого вы будете перенаправлены на страницу аутентификации. DMT входит в систему, используя пароль `NODE_ADMIN_KEY`, который вы сгенерировали ранее.



![Image](assets/fr/33.webp)



Войдя в систему, вы можете получить доступ к *Инструменту обслуживания Dojo*! Во время IBD вы можете увидеть информацию "*Latest Block*" в меню "*Full node*", которая позволит вам узнать текущий статус вашего узла Bitcoin.



![Image](assets/fr/34.webp)



Не забудьте сделать закладку на этот Address в Tor Browser, чтобы потом легко найти его.



Когда ваш Dojo будет полностью синхронизирован, вы должны увидеть галочку Green ✅ на всех индикаторах на главной странице DMT.



### 7.2. Проверка через журналы Dojo



Второй способ отслеживания прогресса IBD - это непосредственное изучение журналов машины. Этот подход обеспечивает гораздо более точный мониторинг в реальном времени. Вы можете видеть, как Bitcoin core продвигается в загрузке блоков и как Fulcrum их индексирует.



Подключитесь к компьютеру, на котором установлен ваш Dojo, и откройте терминал. Все команды должны выполняться из директории `my-dojo`. Расположитесь в этой папке:



```bash
cd ~/dojo-app/docker/my-dojo
```



![Image](assets/fr/35.webp)



#### Журналы Bitcoin core



Просматривайте журналы Bitcoin core, чтобы отслеживать прогресс IBD:



```bash
./dojo.sh logs bitcoind
```



В начале вы увидите фазу предварительной синхронизации заголовков блоков:



```
bitcoind | Pre-synchronizing blockheader, height : NNNNNN
```



Когда этот показатель достигнет 100%, Bitcoin core начнет полную загрузку Blockchain. Вы увидите прогресс IBD. Чтобы узнать текущую высоту блока, посмотрите на значение, обозначенное `height=`. Вы также можете следить за ключом `progress=`, который указывает на процент выполнения IBD.



![Image](assets/fr/36.webp)



Как обычно, чтобы закрыть журналы и вернуться в командную строку, используйте комбинацию `Ctrl + C`.



#### Бревна Fulcrum



После того как Bitcoin core завершил предварительную синхронизацию заголовков, Fulcrum начинает индексировать Blockchain по мере выполнения. Просматривайте его журналы с помощью :



```bash
./dojo.sh logs fulcrum
```



Вы увидите высоту последнего проиндексированного блока, указанную после `height:`, а также процент прогресса индексации.



![Image](assets/fr/37.webp)



### 7.3. Повреждение базы данных Fulcrum



Fulcrum - очень мощный индексатор, но его установка может быть сложной, не в последнюю очередь из-за тонкой работы с базой данных. В случае отключения электричества или внезапной остановки машины во время начальной синхронизации база данных индексатора может быть повреждена. Вы можете увидеть это, например, если у вас есть следующие журналы:



```bash
fulcrum | The database has been corrupted etc...
```



**Примечание:** Эта ошибка должна быть исправлена в ближайшем выпуске Fulcrum 2.0.



Если это произойдет с вами, это никак не повлияет на bitcoind (ваш узел Bitcoin): его IBD будет продолжать выполняться независимо от Fulcrum. Однако вы не сможете использовать Fulcrum, пока не удалите его поврежденные данные и не запустите синхронизацию с самого начала. Вот как это работает:



Остановите Додзё:



```bash
cd ~/dojo-app/docker/my-dojo
./dojo.sh stop
```



Удалите только контейнер и объем Fulcrum:



```bash
docker rm -f fulcrum || true
docker volume ls | grep -i fulcrum
docker volume rm my-dojo_data-fulcrum
```



Обычно это имя - `my-dojo_data-fulcrum`, если для вас это не так, адаптируйте имя, возвращаемое предыдущей командой.



Затем перезапустите Dojo и перестройте Fulcrum с нуля:



```bash
./dojo.sh upgrade
```



Затем вы можете убедиться, что Fulcrum работает правильно, просмотрев журналы:



```bash
docker logs -f fulcrum
```




## 8. Использование инструмента обслуживания Dojo



Как только ваш узел Bitcoin будет синхронизирован с головкой основы с помощью самого Proof of Work, а Blockchain будет на 100% проиндексирован Fulcrum, вы сможете начать использовать свой Dojo.



Ваш Dojo предлагает широкий спектр возможностей, которые регулярно расширяются с каждой новой версией. На мой взгляд, наиболее важны две из них:




- возможность подключения Ashigaru Wallet для использования собственного узла для получения данных Blockchain и трансляции транзакций,
- и Block explorer, что позволяет получить доступ к информации о Blockchain Bitcoin, не открывая свои данные внешнему экземпляру, который вы не контролируете.



Давайте узнаем, как их использовать.


### 8.1. Подключите Ashigaru к своему додзё



Подключить Ashigaru Wallet к Dojo очень просто: на DMT откройте меню "*Pairing*". Появится QR-код, который вы можете отсканировать прямо с помощью приложения Ashigaru.

![Image](assets/fr/39.webp)



Затем нажмите на кнопку "*Продолжить*".



![Image](assets/fr/40.webp)



Теперь вы подключены к своему додзё.



![Image](assets/fr/41.webp)



### 8.2. Использование Block explorer



Dojo автоматически устанавливает Block explorer, [BTC-RPC Explorer](https://github.com/janoside/btc-RPC-explorer), который опирается непосредственно на данные вашего собственного узла Bitcoin. Эксплорер позволяет вам получить доступ к необработанной информации из Blockchain и вашего Mempool через понятный веб Interface. Таким образом, вы можете, например, проверить статус ожидающей транзакции, просмотреть баланс Address или изучить состав блока.



Чтобы получить к нему доступ, просто извлеките Tor Address из вашего браузера. Для этого выполните ту же команду, которую вы использовали для получения Address вашего DMT:



```bash
./dojo.sh onion
```



![Image](assets/fr/42.webp)



Вы получите доступ ко всей информации о подключении к Dojo через Tor. В данном случае нас интересует следующий URL:



```
Block Explorer =
```



Если вы уже подключены к DMT, вы также можете найти этот Address в меню "*Pairing*", внутри JSON соединения:



![Image](assets/fr/43.webp)



Чтобы получить доступ к браузеру с любой машины в любой сети (даже удаленно), откройте [Tor Browser](https://www.torproject.org/download/) и введите URL, который вы только что получили.



⚠️ **Просьба хранить этот Address в строгой тайне



После этого вы получите доступ к своему собственному Block explorer.



![Image](assets/fr/44.webp)



*Image credit: https://ashigaru.rs/.*



Чтобы отследить транзакцию, просто введите ее txid в строку поиска в правом верхнем углу.



![Image](assets/fr/45.webp)



*Image credit: https://ashigaru.rs/.*



Чтобы проверить движения, связанные с Address, действуйте аналогичным образом, введя Address в строку поиска.



![Image](assets/fr/46.webp)



*Image credit: https://ashigaru.rs/.*



Вы также можете ввести Hash или высоту блока в строке поиска, чтобы отобразить подробную информацию.




### 9.2 Обновите свой додзё



Dojo регулярно развивается, и выходят новые версии, в которых исправляются ошибки, повышается стабильность и добавляются новые возможности. Поэтому важно [регулярно проверять наличие обновлений](https://github.com/Dojo-Open-Source-Project/samourai-dojo/releases) и обновлять свой Dojo. Процесс похож на первоначальную установку, но требует замены некоторых файлов на последнюю доступную версию, сохраняя при этом свою конфигурацию. Вот подробные шаги, которые необходимо выполнить для чистого и безопасного обновления:



Чтобы узнать текущую версию вашего Dojo, выполните команду :



```bash
./dojo.sh version
```



Хотя этот шаг необязателен, я рекомендую начать с обновления вашей ОС. Это снизит риск несовместимости и обеспечит актуальность зависимостей, используемых Dojo:



```bash
sudo apt-get update
sudo apt-get upgrade
```



Перейдите в каталог Dojo и остановите текущие службы:



```bash
cd ~/dojo-app/docker/my-dojo
./dojo.sh stop
```



Затем перезагрузите систему, чтобы получить чистый лист:



```bash
sudo reboot
```



Dojo поставляется с файлами, подписанными цифровой подписью. Эти PGP-подписи гарантируют, что файлы получены от разработчика и не подвергались никаким изменениям. Важно проверять их каждый раз, когда вы обновляете Dojo, так же как и при первой установке. Начните с загрузки открытого ключа разработчика через Tor, затем импортируйте его :



```bash
torsocks wget http://zkaan2xfbuxia2wpf7ofnkbz6r5zdbbvxbunvp5g2iebopbfc4iqmbad.onion/vks/v1/by-fingerprint/E53AD419B242822F19E23C6D3033D463D6E544F6 && gpg --import E53AD419B242822F19E23C6D3033D463D6E544F6
```



Вернитесь в личный каталог:



```bash
cd ~/
```



Загрузите последнюю версию Dojo с GitHub через Tor. В данном примере это версия `1.28.0` (которая на момент написания статьи еще не существует: это просто для примера). Не забудьте заменить файл и ссылку [на версию, которую вы хотите установить](https://github.com/Dojo-Open-Source-Project/samourai-dojo/releases):



```bash
torsocks wget -O samourai-dojo-1.28.0.zip https://github.com/Dojo-Open-Source-Project/samourai-dojo/archive/refs/tags/v1.28.0.zip
```



Также загрузите файл, содержащий отпечатки пальцев и подпись PGP (еще раз не забудьте настроить версию в команде):



```bash
torsocks wget https://github.com/Dojo-Open-Source-Project/samourai-dojo/releases/download/v1.28.0/samourai-dojo-1.28.0-fingerprints.txt && torsocks wget https://github.com/Dojo-Open-Source-Project/samourai-dojo/releases/download/v1.28.0/samourai-dojo-1.28.0-fingerprints.txt.sig
```



Убедитесь, что загруженный файл отпечатков пальцев подписан ключом разработчика:



```bash
gpg --verify samourai-dojo-1.28.0-fingerprints.txt.sig
```



Правильный результат должен отображаться :



```
gpg: Signature made [date + time]
gpg: using EDDSA key E53AD419B242822F19E23C6D3033D463D6E544F6
gpg: Good signature from "dojocoder@pm.me" <dojocoder@pm.me> [unknown]
```



Может появиться предупреждение о том, что ключ несертифицирован, но это не имеет значения. С другой стороны, если подпись недействительна или соответствует другому ключу, не идите дальше и начните сначала, проверяя ссылки.



Затем вычислите SHA256-отпечаток архива и сравните его с официальным файлом отпечатков пальцев:



```bash
sha256sum samourai-dojo-1.28.0.zip
cat samourai-dojo-1.28.0-fingerprints.txt
```



Если отпечатки пальцев совпадают, значит, архив подлинный и не поврежден. Если они отличаются, немедленно удалите файлы и не продолжайте работу.



Распакуйте архив в своем домашнем каталоге:



```bash
unzip samourai-dojo-1.28.0.zip -d .
```



Затем скопируйте содержимое в каталог Dojo, перезаписав старый файл :



```bash
cp -a samourai-dojo-1.28.0/. dojo-app/
```



Эта операция сохраняет ваши конфигурационные файлы, расположенные в `~/dojo-app/docker/my-dojo/conf`, но заменяет все остальные файлы обновленными версиями.



Чтобы поддерживать чистоту среды, удаляйте ненужные файлы:



```bash
rm -r samourai-dojo-1.28.0 && rm samourai-dojo-1.28.0.zip && rm samourai-dojo-1.28.0-fingerprints.txt && rm samourai-dojo-1.28.0-fingerprints.txt.sig && rm E53AD419B242822F19E23C6D3033D463D6E544F6
```



Вернитесь в каталог скриптов Dojo и запустите команду update:



```bash
cd ~/dojo-app/docker/my-dojo
./dojo.sh upgrade -y
```



Эта команда дает команду Docker пересобрать образы с новыми файлами, а затем автоматически перезапустить все сервисы. В конце процесса проверьте журналы, чтобы убедиться, что Bitcoin core, Fulcrum и Dojo работают правильно:



```bash
./dojo.sh logs bitcoind
./dojo.sh logs fulcrum
```



Если сервисы запускаются без ошибок и в журналах видны обрабатываемые блоки, значит, ваш Dojo обновлен и работает.
