Dojo е програма с отворен код, създадена да действа като бекенд сървър за някои ориентирани към поверителността портфейли Bitcoin, базиран на възел Bitcoin core. В исторически план тя е разработена за работа със Samourai Wallet, мобилен wallet, който предлага усъвършенствани функции за защита на личните данни като Whirlpool (CoinJoin), Ricochet, Stonewall, PayNym... Samourai Wallet вече е спрян след ареста на разработчиците му, но неговият наследник в общността, **Ashigaru Wallet**, пое функциите и продължава да разчита на Dojo, за да предложи пълноценно преживяване на потребителите, които искат да запазят контрола върху данните си, когато използват Bitcoin.



![Image](assets/fr/01.webp)



На практика Dojo действа като шлюз между вашата мрежа wallet и Bitcoin. Без Dojo лекият мобилен wallet би трябвало да се допитва до сървъри на трети страни, за да получи информация за състоянието на вашите UTXOs и вашата история или да излъчва вашите транзакции. Това предполага зависимост и изтичане на чувствителни данни към сървър на трета страна (използвани адреси, суми, честота на плащанията и т.н.). С Dojo вие сами хоствате този сървър, директно свързан с вашия собствен възел Bitcoin. По този начин всички ваши заявки по wallet преминават през инфраструктура, която вие контролирате, без посредници, което засилва вашата поверителност и суверенитет.



## Изисквания за създаване на Dojo



За създаването на сървър Dojo не е необходима изключително мощна машина. Всеки, който разполага с компютър от начално ниво, стабилна интернет връзка и възможност да го остави включен непрекъснато (24 часа в денонощието, 7 дни в седмицата), може да създаде работещ Dojo.



### Изберете типа на машината си



Можете да използвате :




- лаптоп;
- настолен компютър;
- миникомпютър (напр. Intel NUC, Lenovo Thincentre Tiny...).



Всеки вариант има своите предимства и недостатъци:




- Цена: един ремонтиран мини компютър или настолен компютър често е по-евтин от нов лаптоп.
- Площ: миникомпютърът заема по-малко място.
- Захранване: предимството на лаптопа е, че има батерия, което означава, че няма да се изключи в случай на прекъсване на електрозахранването, за разлика от мини компютъра.
- Възможност за надграждане: барбоните обикновено позволяват да добавите памет или лесно да замените твърдия диск.

## 2. Монтаж на стопанска сграда



За да работи Dojo правилно, в системата ви трябва да има определени софтуерни елементи. Те се използват за управление на софтуерни хранилища, комуникация, декомпресия на архиви и изпълнение на Dojo в контейнери Docker. Всички тези операции се извършват в терминала.



### 2.1. Подготовка



Следната команда ви връща в личната ви папка. Това е добра практика, преди да стартирате поредица от инсталации.



```bash
cd ~/
```



Преди да инсталирате какъвто и да е софтуер, се уверете, че базата данни с наличния на вашата машина софтуер е актуална. Така ще избегнете инсталирането на остарели версии.



```bash
sudo apt-get update
```



![Image](assets/fr/07.webp)



### 2.2. инсталиране на помощни програми



Към системата трябва да се добавят няколко инструмента:




- `apt-transport-https`: позволява ви да изтегляте пакети сигурно чрез HTTPS
- `ca-certificates`: управлява сертификатите, необходими за криптирани връзки
- `curl`: за извличане на файлове от интернет
- `gnupg-agent`: за управление на GPG ключове
- software-properties-common`: предоставя помощни програми за манипулиране на хранилищата на APT
- `unzip`: разархивира файлове във формат ZIP



```bash
sudo apt-get install apt-transport-https ca-certificates curl gnupg-agent software-properties-common unzip
```



По време на инсталацията системата може да ви попита за потвърждение. Натиснете клавиша "*y*", след което натиснете "*Enter*".



![Image](assets/fr/08.webp)



### 2.3. инсталирайте Torsocks



Torsocks позволява изпълнението на определени команди чрез мрежата Tor, като по този начин се подобрява поверителността на комуникациите.



```bash
sudo apt install torsocks
```



![Image](assets/fr/09.webp)



### 2.4. инсталирайте Docker и Docker Compose



Dojo работи в контейнери Docker. Това означава, че всяка услуга е изолирана в независима среда, което опростява поддръжката и сигурността. За целта е необходимо да инсталирате Docker и инструмента Docker Compose, който ви позволява да управлявате няколко контейнера едновременно.



#### Добавяне на ключ за подписване на Docker



Docker предоставя свой собствен ключ за цифров подпис. С добавянето му се проверява автентичността на изтеглените пакети.



```bash
sudo apt-get update
sudo apt-get install ca-certificates curl
sudo install -m 0755 -d /etc/apt/keyrings
sudo curl -fsSL https://download.docker.com/linux/ubuntu/gpg -o /etc/apt/keyrings/docker.asc
sudo chmod a+r /etc/apt/keyrings/docker.asc
```



![Image](assets/fr/10.webp)



#### Добавено е официално хранилище на Docker



След това трябва да укажете на системата къде да намери официалните пакети на Docker. Тази команда добавя ново хранилище към конфигурацията на вашия мениджър на пакети.



```bash
echo \
"deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.asc] https://download.docker.com/linux/ubuntu \
$(. /etc/os-release && echo "*$VERSION_CODENAME*") stable" | \
sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

sudo apt-get update
```



![Image](assets/fr/11.webp)



#### Инсталиране на Docker и Docker Compose



Основните компоненти на Docker вече могат да бъдат инсталирани.



```bash
sudo apt-get install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
```



![Image](assets/fr/12.webp)



#### Оторизация на потребителя



По подразбиране само команди, изпълнени с администраторски права, могат да стартират Docker. За по-голямо удобство препоръчвам да добавите текущия си потребител към групата "*docker*". Това ви позволява да използвате Docker, без да се налага да въвеждате `sudo` всеки път.



```bash
sudo usermod -aG docker $USER
```



![Image](assets/fr/13.webp)



## 3. Създаване на един потребител (по избор)



Ако искате да подобрите сигурността на вашата система, препоръчвам ви да създадете отделен потребител, който да работи само с Dojo. Това разделяне ограничава рисковете: ако в Dojo възникне проблем със сигурността, той няма да компрометира директно основния ви акаунт.



### 3.1. създаване на потребителски акаунт



Следната команда създава нов потребител с име "*dojo*". Този потребител ще има домашна директория `/home/dojo` и достъп до терминала bash. Той ще бъде добавен и към групата sudo, за да може да изпълнява администраторски команди.



```bash
sudo useradd -s /bin/bash -d /home/dojo -m -G sudo dojo
```



### 3.2. Задаване на парола



Важно е да зададете силна парола на този акаунт. В идеалния случай трябва да използвате мениджър на пароли, като например Bitwarden до generate дълга, трудна за отгатване комбинация.



```bash
sudo passwd dojo
```



След това системата ще ви помоли да въведете избраната от вас парола и да я потвърдите втори път.



https://planb.academy/tutorials/computer-security/authentication/bitwarden-0532f569-fb00-4fad-acba-2fcb1bf05de9

### 3.3. Упълномощаване на потребителя да използва Docker



За да може потребителят "*dojo*" да стартира контейнерите, необходими за стартирането на Dojo, той трябва да бъде добавен към групата Docker. Така се избягва необходимостта всяка команда да се предшества от `sudo`.



```bash
sudo usermod -aG docker dojo
```



### 3.4. Рестартиране на системата



За да влязат в сила груповите промени, машината трябва да се рестартира.



```bash
sudo reboot
```



### 3.5. Влезте с нов потребител



Когато системата се рестартира, влезте в нея с потребителското име ***dojo*** и паролата, която сте определили по-рано. Всички следващи стъпки трябва да се извършват от този специален акаунт.



## 4. Изтегляне и проверка на Dojo



Преди да инсталирате Dojo, е важно да се уверите, че файловете идват от официалния разработчик и че не са модифицирани. Тази стъпка разчита на използването на PGP и хешове за проверка на автентичността и целостта на файловете.



### 4.1. импортиране на PGP ключа на разработчика



Изтеглете публичния ключ на разработчика чрез Tor и го импортирайте в локалния си ключодържател. Този ключ ще се използва за проверка на подписите, свързани с файлове Dojo.



```bash
torsocks wget http://zkaan2xfbuxia2wpf7ofnkbz6r5zdbbvxbunvp5g2iebopbfc4iqmbad.onion/vks/v1/by-fingerprint/E53AD419B242822F19E23C6D3033D463D6E544F6 && gpg --import E53AD419B242822F19E23C6D3033D463D6E544F6
```



![Image](assets/fr/14.webp)



### 4.2. изтеглете най-новата версия на Dojo



Извличане на компресирания архив, съдържащ изходния код на Dojo. В този пример най-новата версия е `1.27.0`: променете командата в съответствие с [последната версия тук, в официалното хранилище на GitHub](https://github.com/Dojo-Open-Source-Project/samourai-dojo/releases).



```bash
torsocks wget -O samourai-dojo-1.27.0.zip https://github.com/Dojo-Open-Source-Project/samourai-dojo/archive/refs/tags/v1.27.0.zip
```



![Image](assets/fr/15.webp)



### 4.3. Изтегляне на пръстови отпечатъци и подпис



Разработчиците публикуват файл със списъка на цифровите отпечатъци на архивите, както и файл, подписан с техния PGP ключ. Изтеглете ги, за да сравните файловете си локално.



```bash
torsocks wget https://github.com/Dojo-Open-Source-Project/samourai-dojo/releases/download/v1.27.0/samourai-dojo-1.27.0-fingerprints.txt && torsocks wget https://github.com/Dojo-Open-Source-Project/samourai-dojo/releases/download/v1.27.0/samourai-dojo-1.27.0-fingerprints.txt.sig
```



![Image](assets/fr/16.webp)



### 4.4. Проверка на PGP подписа



Проверете дали файлът с пръстови отпечатъци е подписан с импортирания ключ.



```bash
gpg --verify samourai-dojo-1.27.0-fingerprints.txt.sig
```



Правилният резултат показва валиден подпис с ключ `E53AD419B242822F19E23C6D3033D463D6E544F6` и свързания с него адрес `dojocoder@pm.me`. Може да се появи предупреждение, че ключът не е сертифициран: можете да го пренебрегнете.



Ако пък подписът е невалиден, незабавно прекратете процеса на инсталиране и започнете отново отначало.



![Image](assets/fr/17.webp)



### 4.5. Проверка на целостта на архива



Изчислете пръстовия отпечатък SHA256 на изтегления файл, след което отворете файла с пръстовия отпечатък, за да сравните двете стойности.



```bash
sha256sum samourai-dojo-1.27.0.zip
cat samourai-dojo-1.27.0-fingerprints.txt
```



Ако двата отпечатъка са идентични, можете да сте сигурни, че архивът не е бил променян. Ако са различни, не продължавайте и изтрийте файловете.



![Image](assets/fr/18.webp)



### 4.6. Извличане и организиране на файлове



След като проверката приключи успешно, можете да разопаковате архива и да подготвите папка, предназначена за инсталацията на Dojo.



```bash
unzip samourai-dojo-1.27.0.zip -d .
mkdir ~/dojo-app
mv ~/samourai-dojo-1.27.0/* ~/dojo-app/
```



![Image](assets/fr/19.webp)



### 4.7. Почистване на ненужните файлове



Изтрийте временни файлове и архиви, които вече не са необходими, за да поддържате средата си чиста.



```bash
rm -r samourai-dojo-1.27.0 && rm samourai-dojo-1.27.0.zip && rm samourai-dojo-1.27.0-fingerprints.txt && rm samourai-dojo-1.27.0-fingerprints.txt.sig && rm E53AD419B242822F19E23C6D3033D463D6E544F6
```

## 5. Конфигурация на Dojo



Dojo е бекенд сървър, който обединява няколко услуги за взаимодействие с вашия wallet и за управление на вашия възел Bitcoin. Неговата конфигурация може да бъде сложна, но проектът предлага опростен метод, който автоматично инсталира и конфигурира следните компоненти:




- Dojo (основен API)
- Bitcoin core (пълен възел Bitcoin)
- BTC-RPC Explorer (изследовател на уеб блокове)
- Fulcrum Indexer (бързо индексиране на блокове и транзакции)
- Fulcrum Electrum Server е достъпен в мрежата Tor
- Fulcrum Electrum Server, наличен в локалната мрежа
- Административни пълномощия



### 5.1. административни пълномощия

### 6.5. Разбиране на *Инициално изтегляне на блок* (IBD)



При стартиране Bitcoin core трябва да изтегли и провери целия Blockchain от 2009 г. насам. Тази стъпка се нарича ***Initial Block Download* (IBD)**. Тя е от съществено значение, тъй като дава възможност на възела Dojo да проверява независимо всеки блок и транзакция на Bitcoin.



Продължителността на тази синхронизация зависи от няколко фактора:




- мощността на вашия процесор и количеството налична RAM памет,
- скоростта на вашия диск,
- броя и качеството на връстниците, с които се свързва вашият възел,
- скоростта на вашата интернет връзка.



На практика тази операция обикновено отнема между **2 и 7 дни**. През този период можете да оставите машината си да работи непрекъснато. Колкото по-дълго машината е включена, толкова по-бързо ще бъде завършена синхронизацията. Препоръчвам ви да проверявате редовно състоянието на синхронизацията, като се консултирате с дневниците на Bitcoin core или като използвате инструмента за поддръжка на Dojo, след като го инсталирате (вж. следващия раздел).



За да задълбочите познанията си за IBD и, по-общо, за работата и ролята на вашия възел Bitcoin, ви препоръчвам да разгледате този курс:



https://planb.academy/courses/3cd9cb94-82e8-417a-9c5a-02afc2589426


## 7. Наблюдение на синхронизацията



Когато инсталирате Dojo за първи път, трябва да изчакате две основни операции да бъдат напълно завършени: пълното изтегляне на Blockchain Bitcoin (*IBD*) и индексирането на този Blockchain от Fulcrum. В зависимост от вашата връзка и мощността на машината това може да отнеме няколко дни. През това време можете да следите напредъка на процеса, за да се уверите, че всичко върви гладко.



Съществуват два начина за наблюдение на състоянието на синхронизацията:




- използване на инструмента за поддръжка на Dojo (или DMT), който е прост, но предоставя малко подробности по време на IBD;
- директна справка с дневниците на Dojo на вашата машина, по-техническа, но много по-прецизна.



### 7.1. Проверка чрез Dojo Maintenance Tool (DMT)



Инструментът за поддръжка Dojo е защитена, уеб-базирана версия на Interface, която ви позволява да наблюдавате състоянието на вашата инсталация и да извършвате определени операции. Това е най-лесният и достъпен начин за наблюдение на напредъка на IBD. По време на първоначалната фаза на синхронизация показваната информация може да е ограничена. Например ДМТ не показва подробен напредък на индексирането на Fulcrum. От друга страна, след като синхронизацията приключи, ДМТ ясно ще покаже :




- всички светлини светят в зелено;
- последния валидиран блок за всяка услуга (възел, индексиращ модул, Dojo DB).



За да получите достъп до него, трябва да знаете URL адреса на вашия DMT и да се свържете с него [чрез браузъра Tor](https://www.torproject.org/download/). За да направите това, отворете терминал и отидете в директорията `/my-dojo`:



```bash
cd ~/dojo-app/docker/my-dojo
```



След това изпълнете следната команда:



```bash
./dojo.sh onion
```



![Image](assets/fr/32.webp)



След това ще имате достъп до цялата информация, свързана с връзките към вашия Dojo чрез Tor. Тук ни интересува следният URL адрес:



```
Dojo API and Maintenance Tool =
```



За да получите достъп до DMT от която и да е машина в която и да е мрежа (дори от разстояние), отворете Tor Browser и въведете този URL адрес, последван от `/admin`. Например, ако вашият URL адрес е `wo4zobymdl45gmmzzmpoypeemoukbj74wpibc22rxs2yfgpej62v6dyd.onion`, ще трябва да въведете в лентата [Tor Browser](https://www.torproject.org/download/):



```
wo4zobymdl45gmmzzmpoypeemoukbj74wpibc22rxs2yfgpej62v6dyd.onion/admin
```



⚠️ **Моля, запазете този адрес в строга тайна



След това ще бъдете пренасочени към страница за удостоверяване. DMT влиза в системата, като използва паролата `NODE_ADMIN_KEY`, която генерирахте по-рано.



![Image](assets/fr/33.webp)



След като влезете в системата, можете да получите достъп до *Инструмента за поддръжка на Dojo*! По време на IBD можете да видите информацията "*Последен блок*" в менюто "*Full node*", която ви позволява да се запознаете с текущото състояние на вашия възел Bitcoin.



![Image](assets/fr/34.webp)



Не забравяйте да отбележите този адрес в Tor Browser, за да го намерите лесно по-късно.



След като вашият Dojo е напълно синхронизиран, трябва да видите проверка на Green ✅ на всички индикатори на началната страница на DMT.



### 7.2. Проверка чрез дневниците на Dojo



Вторият начин за проследяване на напредъка на IBD е да се консултирате директно с протоколите на машината. Този подход предлага много по-прецизно наблюдение в реално време. Можете да видите как Bitcoin core напредва в изтеглянето на блокове и как Fulcrum ги индексира.



Свържете се с компютъра, на който се намира Dojo, и отворете терминал. Всички команди трябва да се изпълняват от директорията `my-dojo`. Разположете се в тази папка:



```bash
cd ~/dojo-app/docker/my-dojo
```



![Image](assets/fr/35.webp)



#### Протоколи Bitcoin core



Преглеждайте дневниците на Bitcoin core, за да проследявате напредъка на IBD:



```bash
./dojo.sh logs bitcoind
```



В началото ще видите фаза на предварителна синхронизация на блоковите заглавия:



```
bitcoind | Pre-synchronizing blockheader, height : NNNNNN
```



Когато тази цифра достигне 100%, Bitcoin core ще започне пълното изтегляне на Blockchain. Ще видите напредъка на IBD. За да разберете текущата височина на блока, погледнете стойността, посочена с `height=`. Можете също така да проследите ключа `progress=`, който показва процента на напредък на IBD.



![Image](assets/fr/36.webp)



Както винаги, за да затворите регистрите и да се върнете към командния ред, използвайте комбинацията `Ctrl + C`.



#### Дървени трупи Fulcrum



След като Bitcoin core завърши предварителната синхронизация на заглавието, Fulcrum започва да индексира Blockchain в процеса на работа. Прегледайте неговите дневници с :



```bash
./dojo.sh logs fulcrum
```



След това ще видите височината на последния индексиран блок, посочена след `height:`, както и процента на напредъка на индексирането.



![Image](assets/fr/37.webp)



### 7.3. Повреждане на базата данни на Fulcrum



Fulcrum е особено мощен индексиращ модул, но инсталацията му може да бъде сложна, не на последно място поради деликатното управление на базата данни. В случай на прекъсване на електрозахранването или внезапно изключване на машината по време на първоначалната синхронизация, базата данни на индексиращия модул може да бъде повредена. Можете да видите това, например, ако имате следните регистри:



```bash
fulcrum | The database has been corrupted etc...
```



**Забележка:** Този тип грешки трябва да бъдат коригирани с предстоящото издание на Fulcrum 2.0.



Ако това се случи с вас, няма да има въздействие върху bitcoind (вашия възел Bitcoin): неговата IBD ще продължи да се развива независимо от Fulcrum. Въпреки това няма да можете да използвате Fulcrum, докато не изтриете повредените му данни и не рестартирате синхронизацията му отначало. Ето как става това:



Спирка Dojo:



```bash
cd ~/dojo-app/docker/my-dojo
./dojo.sh stop
```



Изтрийте само контейнера и обема на Fulcrum:



```bash
docker rm -f fulcrum || true
docker volume ls | grep -i fulcrum
docker volume rm my-dojo_data-fulcrum
```



Обикновено името е `my-dojo_data-fulcrum`, но ако случаят не е такъв, адаптирайте името, върнато от предишната команда.



След това възобновете Dojo и възстановете Fulcrum от нулата:



```bash
./dojo.sh upgrade
```



След това можете да проверите дали Fulcrum работи правилно, като се консултирате с дневниците:



```bash
docker logs -f fulcrum
```




## 8. Използване на инструмента за поддръжка Dojo



След като вашият възел Bitcoin е синхронизиран с главата на основата с най-много proof of work и Blockchain е 100% индексиран от Fulcrum, можете да започнете да използвате вашия Dojo.



Вашият Dojo предлага широк набор от функции, които редовно се подобряват с всяка нова версия. По мое мнение двете най-важни са:




- възможност за свързване на вашия Ashigaru wallet, за да използвате свой собствен възел за справка с данни от Blockchain и излъчване на транзакции,
- и изследователя на блокове, който ви дава достъп до информация за Blockchain Bitcoin, без да излагате данните си на външен екземпляр, който не контролирате.



Нека разберем как да ги използваме.


### 8.1. Свържете Ashigaru към вашия Dojo



Свързването на вашия Ashigaru wallet с вашия Dojo е много просто: след като влезете в DMT, отворете менюто "*Pairing*". Появява се QR код, който можете да сканирате директно с приложението Ashigaru.



![Image](assets/fr/38.webp)



В приложението Ashigaru, когато го стартирате за първи път след създаването или възстановяването на wallet, ще бъдете пренасочени към страницата "*Конфигуриране на вашия сървър Dojo*". Натиснете "*Scan QR*", след което сканирайте QR кода, показан на вашия DMT.



![Image](assets/fr/39.webp)



След това щракнете върху бутона "*Продължи*".



![Image](assets/fr/40.webp)



Вече сте свързани с вашия Dojo.



![Image](assets/fr/41.webp)



### 8.2. Използване на блоковия изследовател



Dojo автоматично инсталира блоков изследовател, [BTC-RPC Explorer](https://github.com/janoside/btc-RPC-explorer), който черпи данни директно от вашия собствен възел Bitcoin. ## 9. Поддръжка на Dojo



### 9.1 Спрете вашия Dojo



Никога не прекъсвайте внезапно захранването на Dojo, тъй като това може да доведе до повреда на някои бази данни, особено на индексиращата програма Fulcrum. Ако все пак се налага да го изключите, винаги извършвайте чисто изключване на Dojo, след което, след като всички процедури са завършени, изключете и машината:



```bash
cd ~/dojo-app/docker/my-dojo
./dojo.sh stop
```



### 9.2 Актуализиране на вашия Dojo



Dojo се развива редовно и се пускат нови версии за отстраняване на грешки, подобряване на стабилността и добавяне на функции. Затова е важно [да проверявате редовно за актуализации](https://github.com/Dojo-Open-Source-Project/samourai-dojo/releases) и да актуализирате своя Dojo. Процесът е подобен на първоначалната инсталация, но изисква да замените определени файлове с последната налична версия, като запазите конфигурацията си. Ето подробните стъпки, които трябва да следвате за чиста и сигурна актуализация:



За да разберете текущата версия на вашия Dojo, изпълнете командата :



```bash
./dojo.sh version
```



Въпреки че тази стъпка не е задължителна, препоръчвам ви да започнете с актуализиране на операционната система. Това намалява риска от несъвместимост и гарантира, че зависимостите, използвани от Dojo, са актуални:



```bash
sudo apt-get update
sudo apt-get upgrade
```



Отидете в директорията Dojo и спрете текущите услуги:



```bash
cd ~/dojo-app/docker/my-dojo
./dojo.sh stop
```



След това рестартирайте системата, за да получите чиста версия:



```bash
sudo reboot
```



Dojo се предлага с цифрово подписани файлове. Тези PGP подписи гарантират, че файловете произхождат от разработчика и не са били променяни по никакъв начин. Важно е да ги проверявате всеки път, когато актуализирате Dojo, точно както сте направили, когато сте го инсталирали за първи път. Започнете, като изтеглите публичния ключ на разработчика чрез Tor, след което го импортирайте :



```bash
torsocks wget http://zkaan2xfbuxia2wpf7ofnkbz6r5zdbbvxbunvp5g2iebopbfc4iqmbad.onion/vks/v1/by-fingerprint/E53AD419B242822F19E23C6D3033D463D6E544F6 && gpg --import E53AD419B242822F19E23C6D3033D463D6E544F6
```



Върнете се в личната си директория:



```bash
cd ~/
```



Изтеглете най-новата версия на Dojo от GitHub чрез Tor. В този пример това е версия `1.28.0` (която все още не съществува към момента на писане на статията: това е само за пример). Не забравяйте да замените файла и линка [с версията, която искате да инсталирате](https://github.com/Dojo-Open-Source-Project/samourai-dojo/releases):



```bash
torsocks wget -O samourai-dojo-1.28.0.zip https://github.com/Dojo-Open-Source-Project/samourai-dojo/archive/refs/tags/v1.28.0.zip
```



Изтеглете и файла, съдържащ пръстовите отпечатъци и подписа на PGP (отново не забравяйте да настроите версията в командата):



```bash
torsocks wget https://github.com/Dojo-Open-Source-Project/samourai-dojo/releases/download/v1.28.0/samourai-dojo-1.28.0-fingerprints.txt && torsocks wget https://github.com/Dojo-Open-Source-Project/samourai-dojo/releases/download/v1.28.0/samourai-dojo-1.28.0-fingerprints.txt.sig
```



Проверете дали изтегленият файл с пръстови отпечатъци е подписан с ключа на разработчика:



```bash
gpg --verify samourai-dojo-1.28.0-fingerprints.txt.sig
```



Правилният резултат трябва да се покаже :



```
gpg: Signature made [date + time]
gpg: using EDDSA key E53AD419B242822F19E23C6D3033D463D6E544F6
gpg: Good signature from "dojocoder@pm.me" <dojocoder@pm.me> [unknown]
```



Възможно е да се появи предупреждение, че ключът не е сертифициран, но това е без значение. От друга страна, ако подписът е невалиден или съответства на друг ключ, не продължавайте по-нататък и започнете отново, като проверите връзките.



След това изчислете SHA256 пръстовия отпечатък на архива и го сравнете с официалния файл с пръстови отпечатъци :



```bash
sha256sum samourai-dojo-1.28.0.zip
cat samourai-dojo-1.28.0-fingerprints.txt
```



Ако двата пръстови отпечатъка съвпадат напълно, архивът е истински и непокътнат. Ако се различават, незабавно изтрийте файловете и не продължавайте.



Разкомпресирайте архива в домашната си директория:



```bash
unzip samourai-dojo-1.28.0.zip -d .
```



След това копирайте съдържанието в директорията Dojo, като презапишете стария :



```bash
cp -a samourai-dojo-1.28.0/. dojo-app/
```



Тази операция запазва конфигурационните файлове, намиращи се в `~/dojo-app/docker/my-dojo/conf`, но заменя всички останали файлове с актуализираните версии.



За да поддържате средата си чиста, изтривайте ненужните файлове :



```bash
rm -r samourai-dojo-1.28.0 && rm samourai-dojo-1.28.0.zip && rm samourai-dojo-1.28.0-fingerprints.txt && rm samourai-dojo-1.28.0-fingerprints.txt.sig && rm E53AD419B242822F19E23C6D3033D463D6E544F6
```



Върнете се в директорията със скриптове на Dojo и изпълнете командата за актуализация:



```bash
cd ~/dojo-app/docker/my-dojo
./dojo.sh upgrade -y
```



Тази команда инструктира Docker да възстанови образите с новите файлове, след което автоматично да рестартира всички услуги. В края на процеса проверете дневниците, за да се уверите, че Bitcoin core, Fulcrum и Dojo работят правилно:



```bash
./dojo.sh logs bitcoind
./dojo.sh logs fulcrum
```



Ако услугите се стартират без грешка и дневниците показват, че се обработват блокове, вашият Dojo вече е актуализиран и функционира.
