Dojo เป็นโปรแกรมโอเพ่นซอร์สที่ออกแบบมาเพื่อทำหน้าที่เป็นเซิร์ฟเวอร์แบ็กเอนด์สำหรับกระเป๋าเงิน Bitcoin ที่เน้นความเป็นส่วนตัวบางตัว โดยอิงจากโหนด Bitcoin core ในอดีตมันถูกพัฒนาขึ้นเพื่อทำงานร่วมกับ Samourai Wallet ซึ่งเป็น wallet บนมือถือที่มีคุณสมบัติความเป็นส่วนตัวขั้นสูงเช่น Whirlpool (CoinJoin), Ricochet, Stonewall, PayNym... Samourai Wallet ได้ถูกปิดตัวลงแล้วหลังจากการจับกุมผู้พัฒนา แต่ผู้สืบทอดจากชุมชนของมัน **Ashigaru Wallet** ได้เข้ามาแทนที่และยังคงพึ่งพา Dojo เพื่อมอบประสบการณ์ที่สมบูรณ์ให้กับผู้ใช้ที่ต้องการควบคุมข้อมูลของตนเมื่อใช้ Bitcoin



![Image](assets/fr/01.webp)



ในทางปฏิบัติ Dojo ทำหน้าที่เป็นเกตเวย์ระหว่าง wallet ของคุณและเครือข่าย Bitcoin โดยไม่มี Dojo wallet ที่มีน้ำหนักเบาจะต้องสอบถามเซิร์ฟเวอร์ของบุคคลที่สามเพื่อรับสถานะของ UTXOs และประวัติของคุณ หรือเพื่อกระจายธุรกรรมของคุณ ซึ่งหมายถึงการพึ่งพาและการรั่วไหลของข้อมูลที่ละเอียดอ่อนไปยังเซิร์ฟเวอร์ของบุคคลที่สาม (ที่อยู่ที่ใช้ จำนวนเงิน ความถี่ในการชำระเงิน ฯลฯ) ด้วย Dojo คุณโฮสต์เซิร์ฟเวอร์นี้ด้วยตัวเอง เชื่อมต่อโดยตรงกับโหนด Bitcoin ของคุณเอง ด้วยวิธีนี้ คำขอ wallet ทั้งหมดของคุณจะผ่านโครงสร้างพื้นฐานที่คุณควบคุมได้โดยไม่มีตัวกลาง เสริมสร้างความลับและอธิปไตยของคุณ



## ข้อกำหนดสำหรับการตั้งค่า Dojo



การตั้งค่าเซิร์ฟเวอร์ Dojo ไม่จำเป็นต้องใช้เครื่องที่มีประสิทธิภาพสูงมาก ใครก็ตามที่มีคอมพิวเตอร์ระดับเริ่มต้น การเชื่อมต่ออินเทอร์เน็ตที่เสถียร และความสามารถในการเปิดเครื่องต่อเนื่อง (24/7) สามารถตั้งค่า Dojo ที่ใช้งานได้

คุณสามารถใช้ :




- แล็ปท็อป ;
- คอมพิวเตอร์ตั้งโต๊ะ ;
- มินิพีซี (เช่น Intel NUC, Lenovo Thincentre Tiny...)



แต่ละตัวเลือกมีข้อดีและข้อเสีย:




- ราคา: มินิพีซีหรือเดสก์ท็อปที่ได้รับการตกแต่งใหม่มักจะถูกกว่าคอมพิวเตอร์แล็ปท็อปเครื่องใหม่
- ขนาด: มินิพีซีใช้พื้นที่น้อยลง
- แหล่งจ่ายไฟ: แล็ปท็อปมีข้อได้เปรียบของแบตเตอรี่ ซึ่งหมายความว่ามันจะไม่ปิดตัวลงในกรณีที่ไฟฟ้าดับ ต่างจากมินิพีซี
- อัปเกรดได้: โดยทั่วไปแล้ว barbones จะอนุญาตให้คุณเพิ่มหน่วยความจำหรือเปลี่ยนฮาร์ดดิสก์ได้อย่างง่ายดาย



สำหรับข้อมูลเพิ่มเติมเกี่ยวกับการเลือกอุปกรณ์ของคุณ ฉันขอแนะนำให้คุณเข้าร่วมหลักสูตรนี้:

## 1 - การติดตั้ง Ubuntu



*หากคุณต้องการติดตั้ง Dojo บนอุปกรณ์ที่ได้ตั้งค่าไว้แล้ว คุณสามารถข้ามขั้นตอนนี้และไปยังขั้นตอนที่ 2 ได้โดยตรง*



เมื่อเตรียมฮาร์ดแวร์ที่เลือกไว้แล้ว ก็ถึงเวลาติดตั้งระบบปฏิบัติการ คุณสามารถใช้การแจกจ่าย Debian ใด ๆ ก็ได้ แต่ฉันแนะนำให้คุณเลือกใช้เวอร์ชัน LTS ของ Ubuntu เนื่องจากเหมาะสมอย่างยิ่งกับวัตถุประสงค์ของเรา นี่คือขั้นตอนที่ต้องปฏิบัติตาม:



### 1.1. สร้าง USB key ที่สามารถบูตได้



จากคอมพิวเตอร์ที่ใช้งานอยู่แล้ว (เครื่องที่คุณใช้ประจำ) ดาวน์โหลดภาพ ISO ของ Ubuntu LTS [จากเว็บไซต์ทางการ](https://ubuntu.com/download/desktop) (`24.04` ณ เวลาที่เขียนนี้ แต่ให้เลือกเวอร์ชันล่าสุดหากมีเวอร์ชันอื่นที่พร้อมใช้งาน)



![Image](assets/fr/02.webp)



ใส่ USB key ที่มีความจุอย่างน้อย 8 GB ลงในคอมพิวเตอร์นี้ จากนั้นสร้าง USB key ที่สามารถบูตได้โดยใช้ซอฟต์แวร์เช่น [Balena Etcher](https://etcher.balena.io/). เลือกภาพ ISO ของ Ubuntu ที่คุณเพิ่งดาวน์โหลด เลือก USB key เป็นอุปกรณ์เป้าหมาย จากนั้นเริ่มกระบวนการสร้าง (โปรดอดทน อาจใช้เวลาหลายนาที).



![Image](assets/fr/03.webp)



ใส่ USB key ที่สามารถบูตได้เข้าไปในคอมพิวเตอร์ที่ปิดอยู่ (เครื่องที่คุณต้องการรัน Dojo) เริ่มต้นเครื่องและกด **F12** หรือ **F10** บนแป้นพิมพ์ของคุณทันที (ขึ้นอยู่กับรุ่น) เพื่อเข้าสู่เมนูบูต จากนั้นเลือก USB key ของคุณเป็นอุปกรณ์ลำดับความสำคัญในการบูตของคอมพิวเตอร์



![Image](assets/fr/04.webp)



### 1.2. ติดตั้งระบบปฏิบัติการ



หน้าจอหลักของ Ubuntu ปรากฏขึ้น เลือก "ลองหรือ ติดตั้ง Ubuntu*"



![Image](assets/fr/05.webp)



จากนั้นทำตามขั้นตอนการติดตั้ง Ubuntu แบบคลาสสิก:




- เลือกภาษา
- เลือกประเภทแป้นพิมพ์
- หากคุณเชื่อมต่อผ่านสาย RJ45 ไม่จำเป็นต้องตั้งค่า Wi-Fi
- คลิกที่ "*ติดตั้ง Ubuntu*" และเลือกตัวเลือกเพื่อติดตั้งซอฟต์แวร์ของบุคคลที่สาม (ไดรเวอร์ Wi-Fi, โค้ดมัลติมีเดีย ฯลฯ)
- เมื่อพ่อมดถามถึงประเภทของการติดตั้ง ให้เลือก "*ลบข้อมูลในดิสก์และติดตั้ง Ubuntu*". **คำเตือน**: การดำเนินการนี้จะลบเนื้อหาทั้งหมดในดิสก์ ตรวจสอบให้แน่ใจว่าดิสก์ที่คุณเลือกตรงกับ NVMe SSD ที่ตั้งใจไว้สำหรับ Dojo.
- สร้างชื่อผู้ใช้แบบง่าย (เช่น "*loic*")
- กำหนดชื่อให้กับเครื่อง (เช่น "*dojo-node*").
- ตั้งรหัสผ่านที่แข็งแกร่งและเก็บรักษาให้ปลอดภัย
- เปิดใช้งานตัวเลือก "*ขอรหัสผ่านของฉันเพื่อเข้าสู่ระบบ*" เพื่อเสริมความปลอดภัย
- ระบุเขตเวลาของคุณ จากนั้นคลิกที่ "*ติดตั้ง*"
- รอให้การติดตั้งเสร็จสิ้น เมื่อเสร็จสิ้นแล้ว ระบบจะรีสตาร์ทโดยอัตโนมัติ
- ถอดกุญแจติดตั้ง USB ออกเมื่อรีสตาร์ทคอมพิวเตอร์



สำหรับรายละเอียดเพิ่มเติมเกี่ยวกับกระบวนการติดตั้ง Ubuntu โปรดดูบทแนะนำเฉพาะของเรา :

## 2. การติดตั้งอาคารเสริม



สำหรับ Dojo ที่จะทำงานได้อย่างถูกต้อง จำเป็นต้องมีซอฟต์แวร์อิฐบางตัวในระบบของคุณ ซอฟต์แวร์เหล่านี้ใช้ในการจัดการที่เก็บซอฟต์แวร์ การสื่อสาร การคลายการบีบอัดไฟล์ และการดำเนินการของ Dojo ภายใน Docker containers การดำเนินการทั้งหมดนี้ทำในเทอร์มินัล



### 2.1. การเตรียมการ



คำสั่งต่อไปนี้จะนำคุณกลับไปยังโฟลเดอร์ส่วนตัวของคุณ นี่เป็นวิธีปฏิบัติที่ดีที่ควรทำก่อนที่จะทำการติดตั้งหลายๆ ครั้ง



```bash
cd ~/
```



ก่อนติดตั้งซอฟต์แวร์ใด ๆ ตรวจสอบให้แน่ใจว่าฐานข้อมูลของซอฟต์แวร์ที่มีอยู่ในเครื่องของคุณเป็นปัจจุบัน วิธีนี้จะช่วยหลีกเลี่ยงการติดตั้งเวอร์ชันที่ล้าสมัย



```bash
sudo apt-get update
```



![Image](assets/fr/07.webp)



### 2.2. ติดตั้งยูทิลิตี้



เครื่องมือหลายอย่างจำเป็นต้องเพิ่มในระบบ:




- `apt-transport-https`: ช่วยให้คุณดาวน์โหลดแพ็กเกจได้อย่างปลอดภัยผ่าน HTTPS
- `ca-certificates`: จัดการใบรับรองที่จำเป็นสำหรับการเชื่อมต่อที่เข้ารหัส
- `curl`: เพื่อดึงไฟล์จากอินเทอร์เน็ต
- `gnupg-agent`: สำหรับการจัดการคีย์ GPG
- ซอฟต์แวร์-properties-common`: ให้ยูทิลิตี้สำหรับการจัดการที่เก็บ APT
- `unzip`: คลายไฟล์ในรูปแบบ ZIP



```bash
sudo apt-get install apt-transport-https ca-certificates curl gnupg-agent software-properties-common unzip
```



ในระหว่างการติดตั้ง ระบบอาจขอให้คุณยืนยัน กดปุ่ม "*y*" จากนั้นกด "*Enter*"



![Image](assets/fr/08.webp)



### 2.3. ติดตั้ง Torsocks



Torsocks ช่วยให้สามารถดำเนินการคำสั่งบางอย่างผ่านเครือข่าย Tor ซึ่งช่วยเพิ่มความลับของการสื่อสาร



```bash
sudo apt install torsocks
```

### 2.4. ติดตั้ง Docker และ Docker Compose



Dojo ทำงานภายใน Docker containers ซึ่งหมายความว่าแต่ละบริการจะถูกแยกออกในสภาพแวดล้อมที่เป็นอิสระ ทำให้การบำรุงรักษาและความปลอดภัยง่ายขึ้น ในการทำเช่นนี้ คุณจำเป็นต้องติดตั้ง Docker และเครื่องมือ Docker Compose ซึ่งช่วยให้คุณจัดการหลาย containers ได้ในเวลาเดียวกัน



#### เพิ่มคีย์การลงนามของ Docker



Docker ให้คีย์ลายเซ็นดิจิทัลของตัวเอง การเพิ่มคีย์นี้ช่วยยืนยันความถูกต้องของแพ็กเกจที่ดาวน์โหลด



```bash
sudo apt-get update
sudo apt-get install ca-certificates curl
sudo install -m 0755 -d /etc/apt/keyrings
sudo curl -fsSL https://download.docker.com/linux/ubuntu/gpg -o /etc/apt/keyrings/docker.asc
sudo chmod a+r /etc/apt/keyrings/docker.asc
```



![Image](assets/fr/10.webp)



#### ที่เก็บ Docker อย่างเป็นทางการถูกเพิ่มแล้ว



ถัดไป คุณจำเป็นต้องบอกระบบว่าจะหาชุดแพ็กเกจ Docker อย่างเป็นทางการได้ที่ไหน คำสั่งนี้จะเพิ่มที่เก็บใหม่ไปยังการกำหนดค่าตัวจัดการแพ็กเกจของคุณ



```bash
echo \
"deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.asc] https://download.docker.com/linux/ubuntu \
$(. /etc/os-release && echo "*$VERSION_CODENAME*") stable" | \
sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

sudo apt-get update
```



![Image](assets/fr/11.webp)



#### การติดตั้ง Docker และ Docker Compose



ขณะนี้สามารถติดตั้งส่วนประกอบหลักของ Docker ได้แล้ว



```bash
sudo apt-get install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
```

## 3. การสร้างผู้ใช้เดี่ยว (ไม่บังคับ)



หากคุณต้องการปรับปรุงความปลอดภัยของระบบของคุณ ฉันขอแนะนำให้คุณสร้างผู้ใช้แยกต่างหากเพื่อใช้รัน Dojo โดยเฉพาะ การแยกนี้จะจำกัดความเสี่ยง: หากเกิดปัญหาด้านความปลอดภัยใน Dojo จะไม่ส่งผลกระทบโดยตรงต่อบัญชีหลักของคุณ



### 3.1. การสร้างบัญชีผู้ใช้



คำสั่งต่อไปนี้จะสร้างผู้ใช้ใหม่ชื่อ "*dojo*". ผู้ใช้นี้จะมีไดเรกทอรีบ้าน `/home/dojo` และสามารถเข้าถึงเทอร์มินัล bash ได้ นอกจากนี้ยังจะถูกเพิ่มไปยังกลุ่ม sudo เพื่อให้สามารถดำเนินการคำสั่งผู้ดูแลระบบได้



```bash
sudo useradd -s /bin/bash -d /home/dojo -m -G sudo dojo
```



### 3.2. การตั้งรหัสผ่าน



สิ่งสำคัญคือต้องกำหนดรหัสผ่านที่แข็งแกร่งให้กับบัญชีนี้ โดยในอุดมคติแล้ว คุณควรใช้ตัวจัดการรหัสผ่าน เช่น Bitwarden ถึง generate เพื่อสร้างชุดรหัสผ่านที่ยาวและยากต่อการคาดเดา



```bash
sudo passwd dojo
```



ระบบจะขอให้คุณป้อนรหัสผ่านที่คุณเลือก จากนั้นยืนยันอีกครั้งหนึ่ง



https://planb.academy/tutorials/computer-security/authentication/bitwarden-0532f569-fb00-4fad-acba-2fcb1bf05de9

### 3.3. อนุญาตให้ผู้ใช้ใช้ Docker



ในการอนุญาตให้ผู้ใช้ "*dojo*" เปิดตัวคอนเทนเนอร์ที่จำเป็นสำหรับการรัน Dojo ผู้ใช้จะต้องถูกเพิ่มเข้าไปในกลุ่ม Docker ซึ่งจะช่วยหลีกเลี่ยงการต้องใส่ `sudo` นำหน้าทุกคำสั่ง

## 5. การกำหนดค่า Dojo



Dojo เป็นเซิร์ฟเวอร์แบ็คเอนด์ที่รวบรวมบริการหลายอย่างเพื่อโต้ตอบกับ wallet ของคุณและจัดการโหนด Bitcoin ของคุณ การกำหนดค่าของมันอาจซับซ้อน แต่โครงการนี้มีวิธีการที่ง่ายขึ้นที่ติดตั้งและกำหนดค่าคอมโพเนนต์ต่อไปนี้โดยอัตโนมัติ:




- Dojo (main API)
- Bitcoin core (complete Bitcoin node)
- BTC-RPC Explorer (เว็บบล็อกเอ็กซ์พลอเรอร์)
- Fulcrum Indexer (การจัดทำดัชนีบล็อกและธุรกรรมอย่างรวดเร็ว)
- Fulcrum Electrum Server มีจำหน่ายบนเครือข่าย Tor
- Fulcrum Electrum Server พร้อมใช้งานบนเครือข่ายท้องถิ่น
- ข้อมูลประจำตัวของผู้ดูแลระบบ



### 5.1. ข้อมูลประจำตัวของผู้ดูแลระบบ



ในการรักษาความปลอดภัยในการเข้าถึงบริการต่าง ๆ คุณจำเป็นต้อง generate ตัวระบุเฉพาะหลายรายการ:




- `BITCOIND_RPC_USER`
- `BITCOIND_RPC_PASSWORD`
- `MYSQL_ROOT_PASSWORD`
- mYSQL_USER
- `MYSQL_PASSWORD`
- nODE_API_KEY`
- `NODE_ADMIN_KEY`
- `NODE_JWT_SECRET`



ตัวระบุเหล่านี้ **ต้องไม่ซ้ำกัน** (นี่เป็นสิ่งสำคัญมาก: คุณต้องไม่ใช้รหัสผ่านเดียวกันสำหรับหลายบริการ) ประกอบด้วยตัวเลข ตัวอักษรตัวใหญ่ และตัวอักษรตัวเล็ก (อักขระตัวเลข) และควรมีความยาวประมาณ 40 ตัวอักษรเพื่อรับประกันระดับความปลอดภัยที่สูง อีกครั้งหนึ่ง ฉันขอแนะนำอย่างยิ่งให้ใช้โปรแกรมจัดการรหัสผ่าน



### 5.2. เข้าถึงไฟล์การกำหนดค่า



ไฟล์การกำหนดค่าของ Dojo จะอยู่ในโฟลเดอร์ `conf/` ย้ายไปที่ไดเรกทอรีนี้:



```bash
cd ~/dojo-app/docker/my-dojo/conf/
```



![Image](assets/fr/21.webp)



### 5.3. การกำหนดค่า Bitcoin core



เปิดไฟล์การกำหนดค่า Bitcoin core ด้วยโปรแกรมแก้ไขข้อความ nano:



```bash
nano docker-bitcoind.conf.tpl
```



![Image](assets/fr/22.webp)



ในไฟล์นี้ ให้ป้อนตัวระบุที่สร้างขึ้น:



```
BITCOIND_RPC_USER=your-ID-here
BITCOIND_RPC_PASSWORD=your-password-here
```



⚠️ ***แทนที่ `your-ID-here` และ `your-password-here` ด้วยข้อมูลการเข้าสู่ระบบของคุณเอง (ด้วยรหัสผ่านที่แข็งแกร่ง)***



คุณยังสามารถปรับขนาดของหน่วยความจำแคชที่ใช้โดย Bitcoin core เพื่อปรับปรุงประสิทธิภาพ (คุณสามารถใช้มากขึ้นได้หากมี RAM มาก):



```
BITCOIND_DB_CACHE=2048
```



ในการบันทึกการเปลี่ยนแปลงของคุณและปิดตัวแก้ไข :




- กด `Ctrl + X
- พิมพ์ `y`
- จากนั้นกด "*Enter*"



### 5.4. การกำหนดค่า MySQL



จากนั้นเปิดการกำหนดค่าฐานข้อมูล MySQL:



```bash
nano docker-mysql.conf.tpl
```



กรุณาใส่รายละเอียดการเข้าสู่ระบบของคุณ:



```
MYSQL_ROOT_PASSWORD=your-password-here
MYSQL_USER=your-ID-here
MYSQL_PASSWORD=your-password-here
```



⚠️ ***แทนที่ `your-ID-here` และ `your-password-here` ด้วยข้อมูลการเข้าสู่ระบบของคุณเอง (ด้วยรหัสผ่านที่แข็งแกร่งและไม่ซ้ำกัน)***



บันทึกในลักษณะเดียวกัน (`Ctrl + X`, `y`, "*Enter*").



![Image](assets/fr/23.webp)



### 5.5. การกำหนดค่า Fulcrum indexer



เปิดไฟล์ต่อไปนี้:



```bash
nano docker-indexer.conf.tpl
```



เพิ่มพารามิเตอร์เพื่อเปิดใช้งาน Fulcrum และผสานรวมเข้ากับ Dojo อย่างถูกต้อง:



```
INDEXER_INSTALL=on
INDEXER_TYPE=fulcrum
INDEXER_BATCH_SUPPORT=active
INDEXER_EXTERNAL=on
```



![Image](assets/fr/24.webp)



ต่อไปนี้มี 2 ความเป็นไปได้ ขึ้นอยู่กับการกำหนดค่าของคุณ หาก Dojo ติดตั้งอยู่บนเครื่องที่แยกจากคอมพิวเตอร์ที่คุณใช้ทุกวัน (บนเครื่องเฉพาะ, เซิร์ฟเวอร์...) ให้ป้อนที่อยู่ IP ของมันในเครือข่ายท้องถิ่นของคุณ ตัวอย่างเช่น :



```
INDEXER_EXTERNAL_IP=192.168.1.157
```



![Image](assets/fr/25.webp)



หากต้องการค้นหาที่อยู่ IP ภายในเครื่องของคุณ ให้เปิดเทอร์มินัลอีกหน้าต่างหนึ่งแล้วป้อนคำสั่งต่อไปนี้:



```bash
hostname -I
```



ความเป็นไปได้ที่สอง: หาก Dojo ทำงานโดยตรงบนคอมพิวเตอร์ส่วนตัวที่คุณใช้ทุกวัน ให้คงค่าดีฟอลต์ที่มีอยู่แล้วในไฟล์การกำหนดค่า :



```
INDEXER_EXTERNAL_IP=127.0.0.1
```



บันทึกและออกจากโปรแกรมแก้ไข (`Ctrl + X`, `y`, "*Enter*").



### 5.6. การกำหนดค่าบริการโหนด



สุดท้ายนี้ เปิดการกำหนดค่าของบริการหลัก Dojo:



```bash
nano docker-node.conf.tpl
```



กรุณาใส่รายละเอียดการเข้าสู่ระบบของคุณ:



```
NODE_API_KEY=your-password-here
NODE_ADMIN_KEY=your-password-here
NODE_JWT_SECRET=your-password-here
```



⚠️ ***แทนที่ `your-password-here` ด้วยข้อมูลรับรองของคุณเอง (ด้วยรหัสผ่านที่แข็งแกร่งและไม่ซ้ำกัน)***



![Image](assets/fr/26.webp)



จากนั้นเปิดใช้งานตัวจัดทำดัชนีในเครื่อง:



```
NODE_ACTIVE_INDEXER=local_indexer
```



บันทึกและออกจากโปรแกรมแก้ไข (`Ctrl + X`, `y`, "*Enter*").



### 5.7. การจัดการเข้าสู่ระบบ



เมื่อการกำหนดค่าเสร็จสมบูรณ์ ไม่จำเป็นต้องบันทึกตัวระบุทั้งหมดที่สร้างขึ้น ตัวเดียวที่จำเป็นต้องบันทึกคือ :



```
NODE_ADMIN_KEY
```



การเข้าสู่ระบบนี้จะช่วยให้คุณสามารถเข้าสู่ระบบเครื่องมือบำรุงรักษา Dojo ได้ในภายหลัง การเข้าสู่ระบบอื่น ๆ ทั้งหมดสามารถลบออกจากตัวจัดการรหัสผ่านหรือบันทึกที่เขียนด้วยมือของคุณได้ พวกมันยังคงสามารถเข้าถึงได้จากไฟล์การกำหนดค่า Dojo หากคุณจำเป็นต้องเรียกคืนในอนาคต



## 6. การติดตั้ง Dojo



ในขั้นตอนนี้ Dojo จะถูกติดตั้งและเริ่มต้นบนเครื่องของคุณ การดำเนินการจะเปิดบริการหลายอย่าง (Bitcoin core, the Fulcrum indexer, the Dojo backend, ฯลฯ) และเริ่มการซิงโครไนซ์เต็มรูปแบบของ Blockchain Bitcoin ซึ่งอาจใช้เวลาหลายวัน ขึ้นอยู่กับฮาร์ดแวร์และการเชื่อมต่ออินเทอร์เน็ตของคุณ

### 6.5. การทำความเข้าใจ *Initial Block Download* (IBD)



เมื่อเริ่มต้นใช้งาน Bitcoin core จะต้องดาวน์โหลดและตรวจสอบ Blockchain ทั้งหมดตั้งแต่ปี 2009 ขั้นตอนนี้เรียกว่า ***Initial Block Download* (IBD)** ซึ่งเป็นสิ่งสำคัญเพราะช่วยให้โหนด Dojo ของคุณสามารถตรวจสอบแต่ละบล็อกและธุรกรรมของ Bitcoin ได้อย่างอิสระ



ระยะเวลาของการซิงโครไนซ์นี้ขึ้นอยู่กับหลายปัจจัย:




- พลังของโปรเซสเซอร์ของคุณและปริมาณหน่วยความจำ RAM ที่มีอยู่,
- ความเร็วของดิสก์ของคุณ,
- จำนวนและคุณภาพของเพียร์ที่โหนดของคุณเชื่อมต่อถึง,
- ความเร็วของการเชื่อมต่ออินเทอร์เน็ตของคุณ



ในการปฏิบัติจริง การดำเนินการนี้มักใช้เวลาระหว่าง **2 ถึง 7 วัน** ในช่วงเวลานี้ คุณสามารถเปิดเครื่องของคุณให้ทำงานต่อเนื่องได้ ยิ่งเครื่องเปิดนานเท่าไร การซิงโครไนซ์ก็จะเสร็จเร็วขึ้นเท่านั้น ฉันแนะนำให้คุณตรวจสอบสถานะการซิงโครไนซ์เป็นประจำโดยการตรวจสอบบันทึก Bitcoin core หรือโดยการใช้เครื่องมือบำรุงรักษา Dojo เมื่อทำการติดตั้งแล้ว (ดูในส่วนถัดไป)



เพื่อเพิ่มพูนความรู้ของคุณเกี่ยวกับ IBD และโดยทั่วไปเกี่ยวกับการทำงานและบทบาทของโหนด Bitcoin ของคุณ ฉันขอแนะนำให้คุณดูหลักสูตรนี้:



https://planb.academy/courses/3cd9cb94-82e8-417a-9c5a-02afc2589426


## 7. การตรวจสอบการซิงโครไนซ์



เมื่อติดตั้ง Dojo เป็นครั้งแรก คุณจำเป็นต้องรอให้การดำเนินการหลักสองอย่างเสร็จสมบูรณ์: การดาวน์โหลด Blockchain Bitcoin (*IBD*) อย่างสมบูรณ์และการจัดทำดัชนีของ Blockchain นี้โดย Fulcrum ขึ้นอยู่กับการเชื่อมต่อและพลังของเครื่องของคุณ อาจใช้เวลาหลายวัน ในระหว่างนี้ คุณสามารถตรวจสอบความคืบหน้าของกระบวนการเพื่อให้แน่ใจว่าทุกอย่างทำงานได้อย่างราบรื่น



มีสองวิธีในการตรวจสอบสถานะของการซิงโครไนซ์:




- การใช้เครื่องมือบำรุงรักษา Dojo (หรือ DMT) ซึ่งใช้งานง่ายแต่ให้รายละเอียดน้อยในระหว่าง IBD;
- การปรึกษาโดยตรงของบันทึก Dojo บนเครื่องของคุณ มีความเทคนิคมากกว่าแต่แม่นยำกว่ามาก



### 7.1. ตรวจสอบผ่านเครื่องมือบำรุงรักษา Dojo (DMT)



เครื่องมือบำรุงรักษา Dojo เป็น Interface ที่ปลอดภัยและใช้ผ่านเว็บที่ช่วยให้คุณตรวจสอบสถานะของโรงงานของคุณ และดำเนินการบางอย่างได้ เป็นวิธีที่ง่ายที่สุดและเข้าถึงได้มากที่สุดในการตรวจสอบความคืบหน้าของ IBD ในระยะเริ่มต้นของการซิงโครไนซ์ ข้อมูลที่แสดงอาจมีจำกัด ตัวอย่างเช่น DMT ไม่แสดงความคืบหน้าโดยละเอียดของการจัดทำดัชนี Fulcrum ในทางกลับกัน เมื่อการซิงโครไนซ์เสร็จสมบูรณ์ DMT จะแสดงอย่างชัดเจน:




- ไฟทั้งหมดเป็นสีเขียว;
- บล็อกที่ตรวจสอบแล้วล่าสุดสำหรับแต่ละบริการ (Node, Indexer, Dojo DB)



ในการเข้าถึง คุณจำเป็นต้องทราบ URL ของ DMT ของคุณและเชื่อมต่อกับมัน [ผ่านทางเบราว์เซอร์ Tor](https://www.torproject.org/download/) เพื่อทำเช่นนี้ ให้เปิดเทอร์มินัลและไปที่ไดเรกทอรี `/my-dojo`:



```bash
cd ~/dojo-app/docker/my-dojo
```



จากนั้นรันคำสั่งต่อไปนี้:



```bash
./dojo.sh onion
```



![Image](assets/fr/32.webp)



จากนั้นคุณจะสามารถเข้าถึงข้อมูลทั้งหมดที่เกี่ยวข้องกับการเชื่อมต่อกับ Dojo ของคุณผ่าน Tor สิ่งที่เราสนใจที่นี่คือ URL ต่อไปนี้:



```
Dojo API and Maintenance Tool =
```



ในการเข้าถึง DMT จากเครื่องใด ๆ บนเครือข่ายใด ๆ (แม้แต่ระยะไกล) ให้เปิด Tor Browser และป้อน URL นี้ตามด้วย `/admin` ตัวอย่างเช่น หาก URL ของคุณคือ `wo4zobymdl45gmmzzmpoypeemoukbj74wpibc22rxs2yfgpej62v6dyd.onion` คุณจะต้องป้อนในแถบ [Tor Browser](https://www.torproject.org/download/):



```
wo4zobymdl45gmmzzmpoypeemoukbj74wpibc22rxs2yfgpej62v6dyd.onion/admin
```



⚠️ **โปรดเก็บที่อยู่นี้ไว้เป็นความลับอย่างเคร่งครัด



จากนั้นคุณจะถูกเปลี่ยนเส้นทางไปยังหน้าการยืนยันตัวตน DMT จะเข้าสู่ระบบโดยใช้รหัสผ่าน `NODE_ADMIN_KEY` ที่คุณสร้างขึ้นก่อนหน้านี้



![Image](assets/fr/33.webp)



เมื่อเข้าสู่ระบบแล้ว คุณสามารถเข้าถึง *Dojo Maintenance Tool* ได้! ในระหว่าง IBD คุณสามารถดูข้อมูล "*Latest Block*" ในเมนู "*Full node*" ซึ่งช่วยให้คุณทราบสถานะปัจจุบันของโหนด Bitcoin ของคุณ



![Image](assets/fr/34.webp)



อย่าลืมบุ๊กมาร์กที่อยู่นี้ใน Tor Browser เพื่อการเรียกดูที่ง่ายขึ้นในภายหลัง



เมื่อ Dojo ของคุณซิงโครไนซ์เสร็จสมบูรณ์ คุณควรเห็น Green เช็ค ✅ บนตัวบ่งชี้ทั้งหมดบนหน้าแรกของ DMT



### 7.2. การตรวจสอบผ่านบันทึก Dojo



วิธีที่สองในการติดตามความคืบหน้าของ IBD ของคุณคือการตรวจสอบบันทึกของเครื่องโดยตรง วิธีนี้ให้การตรวจสอบที่แม่นยำและทันเวลามากขึ้น คุณสามารถดูได้ว่า Bitcoin core กำลังดาวน์โหลดบล็อกอย่างไร และ Fulcrum กำลังทำดัชนีบล็อกเหล่านั้นอย่างไร



เชื่อมต่อกับเครื่องที่โฮสต์ Dojo ของคุณและเปิดเทอร์มินัล คำสั่งทั้งหมดควรถูกดำเนินการจากไดเรกทอรี `my-dojo` วางตำแหน่งตัวเองในโฟลเดอร์นี้:



```bash
cd ~/dojo-app/docker/my-dojo
```



![Image](assets/fr/35.webp)



#### บันทึก Bitcoin core



ดูบันทึก Bitcoin core เพื่อติดตามความคืบหน้า IBD:



```bash
./dojo.sh logs bitcoind
```



ในตอนเริ่มต้น คุณจะเห็นเฟสก่อนการซิงโครไนซ์ของส่วนหัวบล็อก:



```
bitcoind | Pre-synchronizing blockheader, height : NNNNNN
```



เมื่อค่าตัวเลขนี้ถึง 100% Bitcoin core จะเริ่มการดาวน์โหลด Blockchain อย่างสมบูรณ์ คุณจะเห็นความคืบหน้าของ IBD ในการตรวจสอบความสูงของบล็อกปัจจุบัน ให้ดูที่ค่าที่ระบุโดย `height=` คุณยังสามารถติดตามคีย์ `progress=` ซึ่งแสดงเปอร์เซ็นต์ของความคืบหน้า IBD



![Image](assets/fr/36.webp)



ตามปกติ เพื่อปิดบันทึกและกลับไปยังพรอมต์คำสั่ง ให้ใช้การกดปุ่ม `Ctrl + C` ร่วมกัน



#### บันทึกฟัลครัม



เมื่อ Bitcoin core เสร็จสิ้นการซิงโครไนซ์หัวเรื่องล่วงหน้า Fulcrum จะเริ่มทำดัชนี Blockchain ไปพร้อมกัน ดูบันทึกของมันด้วย :



```bash
./dojo.sh logs fulcrum
```



จากนั้นคุณจะเห็นความสูงของบล็อกสุดท้ายที่ถูกจัดทำดัชนี ซึ่งระบุหลังจาก `height:` รวมถึงเปอร์เซ็นต์ความคืบหน้าของการจัดทำดัชนี



![Image](assets/fr/37.webp)



### 7.3. การเสียหายของฐานข้อมูล Fulcrum



Fulcrum เป็นตัวจัดทำดัชนีที่ทรงพลังเป็นพิเศษ แต่การติดตั้งอาจซับซ้อน โดยเฉพาะอย่างยิ่งเนื่องจากการจัดการฐานข้อมูลที่ละเอียดอ่อน ในกรณีที่ไฟฟ้าดับหรือเครื่องปิดกะทันหันระหว่างการซิงโครไนซ์ครั้งแรก ฐานข้อมูลของตัวจัดทำดัชนีอาจเสียหายได้ คุณสามารถเห็นสิ่งนี้ได้ ตัวอย่างเช่น หากคุณมีบันทึกต่อไปนี้:



```bash
fulcrum | The database has been corrupted etc...
```



**หมายเหตุ:** ข้อผิดพลาดประเภทนี้ควรได้รับการแก้ไขในการเปิดตัว Fulcrum 2.0 ที่จะมาถึง



หากสิ่งนี้เกิดขึ้นกับคุณ จะไม่มีผลกระทบต่อ bitcoind (โหนด Bitcoin ของคุณ): IBD ของมันจะดำเนินต่อไปอย่างอิสระจาก Fulcrum อย่างไรก็ตาม คุณจะไม่สามารถใช้ Fulcrum ได้จนกว่าคุณจะลบข้อมูลที่เสียหายและเริ่มการซิงโครไนซ์ใหม่ตั้งแต่ต้น นี่คือวิธีการทำงาน:



หยุด Dojo:



```bash
cd ~/dojo-app/docker/my-dojo
./dojo.sh stop
```



ลบเฉพาะคอนเทนเนอร์และวอลุ่มของ Fulcrum:



```bash
docker rm -f fulcrum || true
docker volume ls | grep -i fulcrum
docker volume rm my-dojo_data-fulcrum
```



โดยปกติชื่อจะเป็น `my-dojo_data-fulcrum` หากไม่ใช่กรณีนี้สำหรับคุณ ให้ปรับชื่อที่ส่งคืนโดยคำสั่งก่อนหน้า



จากนั้นเปิดตัว Dojo อีกครั้งและสร้าง Fulcrum ขึ้นใหม่ตั้งแต่ต้น:



```bash
./dojo.sh upgrade
```



จากนั้นคุณสามารถตรวจสอบว่า Fulcrum ทำงานอย่างถูกต้องโดยการตรวจสอบบันทึก:



```bash

```




## 8. การใช้เครื่องมือบำรุงรักษา Dojo



เมื่อปม Bitcoin ของคุณถูกซิงโครไนซ์กับหัววาร์ปที่มี proof of work มากที่สุด และ Blockchain ถูกจัดทำดัชนีโดย Fulcrum 100% คุณสามารถเริ่มใช้ Dojo ของคุณได้



Dojo ของคุณมีฟีเจอร์หลากหลายที่ได้รับการปรับปรุงอย่างสม่ำเสมอในแต่ละเวอร์ชันใหม่ ในความคิดของฉัน 2 อย่างที่สำคัญที่สุดคือ :




- ความเป็นไปได้ในการเชื่อมต่อ Ashigaru wallet ของคุณเพื่อใช้โหนดของคุณเองในการปรึกษาข้อมูล Blockchain และกระจายธุรกรรมของคุณ,
- และตัวสำรวจบล็อก ซึ่งให้คุณเข้าถึงข้อมูลเกี่ยวกับ Blockchain Bitcoin โดยไม่ต้องเปิดเผยข้อมูลของคุณไปยังอินสแตนซ์ภายนอกที่คุณไม่สามารถควบคุมได้

## 9. การบำรุงรักษา Dojo



### 9.1 หยุด Dojo ของคุณ



อย่าตัดไฟ Dojo ของคุณอย่างกะทันหัน เพราะอาจทำให้ฐานข้อมูลบางอย่างเสียหาย โดยเฉพาะ Fulcrum indexer หากคุณจำเป็นต้องปิดเครื่อง ควรทำการปิดระบบ Dojo อย่างถูกต้องก่อน จากนั้นเมื่อขั้นตอนทั้งหมดเสร็จสิ้นแล้ว ให้ปิดเครื่องด้วยเช่นกัน:



```bash
cd ~/dojo-app/docker/my-dojo
./dojo.sh stop
```



### 9.2 อัปเดต Dojo ของคุณ



Dojo มีการพัฒนาอย่างสม่ำเสมอและมีการปล่อยเวอร์ชันใหม่เพื่อแก้ไขข้อบกพร่อง ปรับปรุงความเสถียร และเพิ่มคุณสมบัติใหม่ ดังนั้นจึงเป็นสิ่งสำคัญ [ที่จะตรวจสอบการอัปเดตเป็นประจำ](https://github.com/Dojo-Open-Source-Project/samourai-dojo/releases) และอัปเดต Dojo ของคุณ กระบวนการนี้คล้ายกับการติดตั้งครั้งแรก แต่ต้องการให้คุณแทนที่ไฟล์บางไฟล์ด้วยเวอร์ชันล่าสุดที่มีอยู่ ในขณะที่ยังคงการกำหนดค่าของคุณไว้ นี่คือขั้นตอนโดยละเอียดที่ต้องปฏิบัติตามเพื่อการอัปเดตที่สะอาดและปลอดภัย:



ในการตรวจสอบเวอร์ชันปัจจุบันของ Dojo ของคุณ ให้รันคำสั่ง :



```bash
./dojo.sh version
```



แม้ว่าขั้นตอนนี้จะเป็นทางเลือก แต่ฉันขอแนะนำให้คุณเริ่มต้นด้วยการอัปเดตระบบปฏิบัติการของคุณก่อน สิ่งนี้จะช่วยลดความเสี่ยงของความไม่เข้ากันและทำให้แน่ใจว่าการพึ่งพาที่ใช้โดย Dojo นั้นเป็นปัจจุบัน:



```bash
sudo apt-get update
sudo apt-get upgrade
```



ไปที่ไดเรกทอรี Dojo และหยุดบริการปัจจุบัน:



```bash
cd ~/dojo-app/docker/my-dojo
./dojo.sh stop
```



จากนั้นรีบูตระบบของคุณเพื่อเริ่มต้นใหม่:



```bash
sudo reboot
```



Dojo มาพร้อมกับไฟล์ที่มีการลงนามแบบดิจิทัล ลายเซ็น PGP เหล่านี้ช่วยให้มั่นใจว่าไฟล์มาจากนักพัฒนาและไม่ได้ถูกแก้ไขในทางใดทางหนึ่ง สิ่งสำคัญคือต้องตรวจสอบทุกครั้งที่คุณอัปเดต Dojo เช่นเดียวกับที่คุณทำเมื่อคุณติดตั้งครั้งแรก เริ่มต้นด้วยการดาวน์โหลดคีย์สาธารณะของนักพัฒนาผ่าน Tor จากนั้นนำเข้ามัน :



```bash
torsocks wget http://zkaan2xfbuxia2wpf7ofnkbz6r5zdbbvxbunvp5g2iebopbfc4iqmbad.onion/vks/v1/by-fingerprint/E53AD419B242822F19E23C6D3033D463D6E544F6 && gpg --import E53AD419B242822F19E23C6D3033D463D6E544F6
```



กลับไปที่ไดเรกทอรีส่วนตัวของคุณ:



```bash
cd ~/
```



ดาวน์โหลดเวอร์ชันล่าสุดของ Dojo จาก GitHub ผ่าน Tor ในตัวอย่างนี้คือเวอร์ชัน `1.28.0` (ซึ่งยังไม่มีอยู่ในขณะที่เขียน: นี่เป็นเพียงตัวอย่าง) อย่าลืมแทนที่ไฟล์และลิงก์ [ด้วยเวอร์ชันที่คุณต้องการติดตั้ง](https://github.com/Dojo-Open-Source-Project/samourai-dojo/releases):



```bash
torsocks wget -O samourai-dojo-1.28.0.zip https://github.com/Dojo-Open-Source-Project/samourai-dojo/archive/refs/tags/v1.28.0.zip
```



ดาวน์โหลดไฟล์ที่มีลายนิ้วมือ PGP และลายเซ็น (อีกครั้ง, อย่าลืมปรับเวอร์ชันในคำสั่ง):



```bash
torsocks wget https://github.com/Dojo-Open-Source-Project/samourai-dojo/releases/download/v1.28.0/samourai-dojo-1.28.0-fingerprints.txt && torsocks wget https://github.com/Dojo-Open-Source-Project/samourai-dojo/releases/download/v1.28.0/samourai-dojo-1.28.0-fingerprints.txt.sig
```



ตรวจสอบว่าไฟล์ลายนิ้วมือที่ดาวน์โหลดได้รับการลงนามโดยคีย์ของนักพัฒนา:



```bash
gpg --verify samourai-dojo-1.28.0-fingerprints.txt.sig
```



ผลลัพธ์ที่ถูกต้องควรแสดง:



```
gpg: Signature made [date + time]
gpg: using EDDSA key E53AD419B242822F19E23C6D3033D463D6E544F6
gpg: Good signature from "dojocoder@pm.me" <dojocoder@pm.me> [unknown]
```



คำเตือนว่าไม่มีการรับรองคีย์อาจปรากฏขึ้น แต่สิ่งนี้ไม่มีความสำคัญ อย่างไรก็ตาม หากลายเซ็นไม่ถูกต้องหรือสอดคล้องกับคีย์อื่น อย่าดำเนินการต่อและเริ่มใหม่อีกครั้ง โดยตรวจสอบลิงก์ต่างๆ



จากนั้นคำนวณลายนิ้วมือ SHA256 ของไฟล์เก็บถาวรและเปรียบเทียบกับไฟล์ลายนิ้วมืออย่างเป็นทางการ :



```bash
sha256sum samourai-dojo-1.28.0.zip
cat samourai-dojo-1.28.0-fingerprints.txt
```



หากลายนิ้วมือทั้งสองตรงกันอย่างสมบูรณ์ แสดงว่าเอกสารเก็บถาวรนั้นเป็นของแท้และสมบูรณ์ หากแตกต่างกัน ให้ลบไฟล์ทันทีและอย่าดำเนินการต่อ



แตกไฟล์ที่เก็บไว้ในไดเรกทอรีบ้านของคุณ:



```bash
unzip samourai-dojo-1.28.0.zip -d .
```



จากนั้นคัดลอกเนื้อหาไปยังไดเรกทอรี Dojo โดยเขียนทับของเก่า :



```bash
cp -a samourai-dojo-1.28.0/. dojo-app/
```



การดำเนินการนี้จะเก็บไฟล์การกำหนดค่าของคุณที่อยู่ใน `~/dojo-app/docker/my-dojo/conf` ไว้ แต่จะแทนที่ไฟล์อื่น ๆ ทั้งหมดด้วยเวอร์ชันที่อัปเดตแล้ว



เพื่อรักษาสภาพแวดล้อมของคุณให้สะอาด ลบไฟล์ที่ไม่จำเป็น :



```bash
rm -r samourai-dojo-1.28.0 && rm samourai-dojo-1.28.0.zip && rm samourai-dojo-1.28.0-fingerprints.txt && rm samourai-dojo-1.28.0-fingerprints.txt.sig && rm E53AD419B242822F19E23C6D3033D463D6E544F6
```



กลับไปที่ไดเรกทอรีสคริปต์ Dojo และรันคำสั่งอัปเดต:



```bash
cd ~/dojo-app/docker/my-dojo
./dojo.sh upgrade -y
```



คำสั่งนี้สั่งให้ Docker สร้างภาพใหม่ด้วยไฟล์ใหม่ จากนั้นเริ่มบริการทั้งหมดใหม่โดยอัตโนมัติ เมื่อสิ้นสุดกระบวนการ ให้ตรวจสอบบันทึกเพื่อให้แน่ใจว่า Bitcoin core, Fulcrum และ Dojo ทำงานถูกต้อง:



```bash
./dojo.sh logs bitcoind
./dojo.sh logs fulcrum
```
