Dojo یک برنامه متن‌باز است که به‌عنوان یک سرور بک‌اند برای برخی کیف‌پول‌های Bitcoin با محوریت حریم خصوصی، بر اساس یک نود Bitcoin core طراحی شده است. به‌طور تاریخی، این برنامه برای کار با Samourai Wallet توسعه یافته بود، یک Wallet موبایل که ویژگی‌های پیشرفته حریم خصوصی مانند Whirlpool (CoinJoin)، Ricochet، Stonewall، PayNym را ارائه می‌داد... Samourai Wallet اکنون پس از دستگیری توسعه‌دهندگانش تعطیل شده است، اما جانشین جامعه‌ای آن، **Ashigaru Wallet**، جایگزین شده و همچنان به Dojo متکی است تا تجربه کاملی را برای کاربرانی که می‌خواهند کنترل داده‌های خود را هنگام استفاده از Bitcoin حفظ کنند، ارائه دهد.



![Image](assets/fr/01.webp)



به‌طور عملی، دوجو به عنوان یک دروازه بین Wallet شما و شبکه Bitcoin عمل می‌کند. بدون دوجو، یک Wallet موبایل سبک باید برای به‌دست آوردن وضعیت UTXOهای شما و تاریخچه‌تان یا برای پخش تراکنش‌هایتان به سرورهای شخص ثالث مراجعه کند. این به معنای وابستگی و نشت داده‌های حساس به یک سرور شخص ثالث است (آدرس‌های استفاده شده، مقادیر، فرکانس پرداخت و غیره). با دوجو، شما این سرور را خودتان میزبانی می‌کنید و مستقیماً به نود Bitcoin خود متصل می‌شوید. به این ترتیب، تمام درخواست‌های پورتفولیوی شما از طریق زیرساختی که خودتان کنترل می‌کنید و بدون واسطه‌ها عبور می‌کند، که محرمانگی و حاکمیت شما را تقویت می‌کند.



## الزامات برای راه‌اندازی یک دوجو



راه‌اندازی یک سرور Dojo نیاز به یک ماشین فوق‌العاده قدرتمند ندارد. هر کسی با یک کامپیوتر سطح ابتدایی، یک اتصال اینترنت پایدار و توانایی روشن نگه‌داشتن آن به‌صورت مداوم (24/7) می‌تواند یک Dojo کارآمد راه‌اندازی کند.



### نوع دستگاه خود را انتخاب کنید



می‌توانید استفاده کنید:




- یک لپ‌تاپ ؛
- یک رایانه رومیزی ;
- یک مینی‌پی‌سی (مثلاً Intel NUC، Lenovo Thincentre Tiny...).



هر گزینه مزایا و معایب خاص خود را دارد:




- قیمت: یک مینی‌پی‌سی یا دسکتاپ بازسازی‌شده اغلب ارزان‌تر از یک لپ‌تاپ جدید خواهد بود.
- اثر پا: یک مینی‌پی‌سی فضای کمتری اشغال می‌کند.
- پاور Supply: یک لپ‌تاپ دارای مزیت باتری است، به این معنی که در صورت قطع برق خاموش نمی‌شود، برخلاف یک مینی‌پی‌سی.
- قابلیت ارتقاء: باربون‌ها عموماً به شما اجازه می‌دهند تا حافظه اضافه کنید یا به‌راحتی یک دیسک Hard را جایگزین کنید.

### تجهیزات پیشنهادی



نیازی به خرید یک ماشین جدید نیست. یک کامپیوتر بازسازی‌شده با مشخصات زیر عملکرد بسیار بهتری نسبت به الکترونیک تک‌برد (مانند Raspberry Pi) ارائه خواهد داد.



**حداقل مشخصات:**

- پردازنده دو هسته‌ای ۲ گیگاهرتز یا سریع‌تر.

**سیستم عامل پیشنهادی: **

## ۱ - نصب اوبونتو



*اگر می‌خواهید Dojo را روی دستگاهی که قبلاً پیکربندی شده است نصب کنید، می‌توانید این مرحله را رد کرده و مستقیماً به مرحله ۲ بروید.*



پس از آماده‌سازی سخت‌افزار انتخابی، زمان نصب یک سیستم‌عامل فرا رسیده است. شما می‌توانید از تقریباً هر توزیع دبیان استفاده کنید، اما من توصیه می‌کنم که یک نسخه LTS از اوبونتو را انتخاب کنید، زیرا به‌خوبی برای اهداف ما مناسب است. در اینجا مراحل دنبال کردن آمده است:



### 1.1. ایجاد یک کلید USB قابل بوت



از یک کامپیوتر که در حال حاضر کار می‌کند (ماشین معمولی شما)، تصویر ISO اوبونتو LTS را [از سایت رسمی](https://ubuntu.com/download/desktop) دانلود کنید (`24.04` در زمان نوشتن، اما اگر نسخه جدیدتری موجود است، آن را بگیرید).



![Image](assets/fr/02.webp)



یک کلید USB با حداقل ظرفیت ۸ گیگابایت را به این کامپیوتر وصل کنید، سپس با استفاده از نرم‌افزاری مانند [Balena Etcher](https://etcher.balena.io/) یک کلید بوت‌پذیر ایجاد کنید. تصویر ISO اوبونتو که به‌تازگی دانلود کرده‌اید را انتخاب کنید، کلید USB را به‌عنوان دستگاه هدف انتخاب کنید، سپس فرآیند ایجاد را شروع کنید (صبور باشید، ممکن است چند دقیقه طول بکشد).



![Image](assets/fr/03.webp)



کلید USB قابل بوت را در رایانه خاموش (رایانه‌ای که می‌خواهید Dojo را روی آن اجرا کنید) وارد کنید. دستگاه را روشن کنید و بلافاصله **F12** یا **F10** را روی صفحه‌کلید خود فشار دهید (بسته به مدل) تا به منوی بوت دسترسی پیدا کنید. سپس کلید USB خود را به عنوان دستگاه اولویت‌دار در ترتیب بوت رایانه انتخاب کنید.



![Image](assets/fr/04.webp)



### 1.2. نصب سیستم عامل



صفحه اصلی اوبونتو ظاهر می‌شود. گزینه "Try or Install Ubuntu*" را انتخاب کنید.



![Image](assets/fr/05.webp)



سپس فرآیند نصب کلاسیک اوبونتو را دنبال کنید:




- زبان را انتخاب کنید.
- نوع صفحه‌کلید را انتخاب کنید.
- اگر از طریق کابل RJ45 متصل هستید، نیازی به پیکربندی وای‌فای نیست.
- روی "*Install Ubuntu*" کلیک کنید و گزینه نصب نرم‌افزارهای شخص ثالث (درایورهای Wi-Fi، کدک‌های چندرسانه‌ای، و غیره) را انتخاب کنید.
- وقتی جادوگر نوع نصب را می‌پرسد، گزینه "*Erase disk and install Ubuntu*" را انتخاب کنید. **هشدار**: این عملیات محتوای دیسک را به‌طور کامل پاک خواهد کرد. با دقت بررسی کنید که دیسکی که انتخاب کرده‌اید با NVMe SSD مورد نظر برای Dojo مطابقت داشته باشد.
- یک نام کاربری ساده ایجاد کنید (مثلاً "*loic*").
- به ماشین یک نام اختصاص دهید (مثلاً "*dojo-node*").
- یک رمز عبور قوی تنظیم کنید و آن را ایمن نگه دارید.
- گزینه "*درخواست رمز عبور من برای ورود*" را فعال کنید تا امنیت را تقویت کنید.
- منطقه زمانی خود را مشخص کنید، سپس روی "*نصب*" کلیک کنید.
- منتظر بمانید تا نصب کامل شود. پس از تکمیل، سیستم به‌طور خودکار راه‌اندازی مجدد خواهد شد.
- هنگام راه‌اندازی مجدد رایانه، کلید نصب USB را بردارید.



برای جزئیات بیشتر در مورد فرآیند نصب اوبونتو، لطفاً به آموزش اختصاصی ما مراجعه کنید:



https://planb.academy/tutorials/computer-security/operating-system/ubuntu-78a3be56-5d51-4ec3-8629-0dd27c352ab5

## ۲. نصب ساختمان جانبی



برای عملکرد صحیح Dojo، برخی از نرم‌افزارهای ضروری باید بر روی سیستم شما موجود باشند. این نرم‌افزارها برای مدیریت مخازن نرم‌افزاری، ارتباطات، استخراج آرشیو و اجرای Dojo در داخل کانتینرهای Docker استفاده می‌شوند. تمامی این عملیات‌ها در ترمینال انجام می‌شوند.



### ۲.۱. آماده‌سازی



دستور زیر شما را به پوشه شخصی‌تان بازمی‌گرداند. این یک تمرین خوب قبل از اجرای یک سری نصب‌ها است.



```bash
cd ~/
```



قبل از نصب هر نرم‌افزاری، مطمئن شوید که پایگاه داده نرم‌افزارهای موجود در دستگاه شما به‌روز است. این کار از نصب نسخه‌های منسوخ جلوگیری می‌کند.



```bash
sudo apt-get update
```



![Image](assets/fr/07.webp)



### ۲.۲. نصب ابزارها



چندین ابزار باید به سیستم اضافه شود:




- `apt-transport-https`: به شما امکان می‌دهد بسته‌ها را به‌صورت امن از طریق HTTPS دانلود کنید
- `ca-certificates`: گواهی‌های مورد نیاز برای اتصالات رمزگذاری شده را مدیریت می‌کند
- `curl`: برای بازیابی فایل‌ها از اینترنت
- `gnupg-agent`: برای مدیریت کلید GPG
- software-properties-common`: ابزارهایی برای مدیریت مخازن APT فراهم می‌کند
- `unzip`: فایل‌ها را از فرمت ZIP خارج می‌کند



```bash
sudo apt-get install apt-transport-https ca-certificates curl gnupg-agent software-properties-common unzip
```



در طول نصب، ممکن است سیستم از شما تأیید بخواهد. کلید "*y*" را فشار دهید، سپس "*Enter*" را فشار دهید.



![Image](assets/fr/08.webp)

### ۲.۴. نصب Docker و Docker Compose



دوجو داخل کانتینرهای داکر اجرا می‌شود. این بدان معناست که هر سرویس در یک محیط مستقل ایزوله شده است، که نگهداری و امنیت را ساده‌تر می‌کند. برای انجام این کار، شما نیاز به نصب داکر و ابزار Docker Compose دارید، که به شما امکان مدیریت چندین کانتینر به طور همزمان را می‌دهد.



#### کلید امضای Docker را اضافه کنید



داکر کلید امضای دیجیتال خود را ارائه می‌دهد. افزودن آن اصالت بسته‌های دانلود شده را تأیید می‌کند.



```bash
sudo apt-get update
sudo apt-get install ca-certificates curl
sudo install -m 0755 -d /etc/apt/keyrings
sudo curl -fsSL https://download.docker.com/linux/ubuntu/gpg -o /etc/apt/keyrings/docker.asc
sudo chmod a+r /etc/apt/keyrings/docker.asc
```



![Image](assets/fr/10.webp)



#### مخزن رسمی Docker اضافه شد



سپس، باید به سیستم بگویید که بسته‌های رسمی Docker را کجا پیدا کند. این دستور یک مخزن جدید به پیکربندی مدیر بسته شما اضافه می‌کند.



```bash
echo \
"deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.asc] https://download.docker.com/linux/ubuntu \
$(. /etc/os-release && echo "*$VERSION_CODENAME*") stable" | \
sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

sudo apt-get update
```



![Image](assets/fr/11.webp)



#### نصب Docker و Docker Compose



اکنون می‌توان اجزای اصلی Docker را نصب کرد.



```bash
sudo apt-get install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
```



![Image](assets/fr/12.webp)



#### مجوز کاربر



به طور پیش‌فرض، تنها دستورات اجرا شده با حقوق مدیر می‌توانند Docker را راه‌اندازی کنند. برای راحتی بیشتر، توصیه می‌کنم کاربر فعلی خود را به گروه "*docker*" اضافه کنید. این به شما اجازه می‌دهد تا بدون نیاز به تایپ `sudo` هر بار از Docker استفاده کنید.



```bash
sudo usermod -aG docker $USER
```

## ۳. ایجاد کاربر واحد (اختیاری)



اگر می‌خواهید امنیت سیستم خود را بهبود بخشید، توصیه می‌کنم که یک کاربر جداگانه به‌طور انحصاری برای اجرای Dojo ایجاد کنید. این جداسازی خطرات را محدود می‌کند: اگر مشکلی امنیتی در Dojo رخ دهد، به‌طور مستقیم حساب اصلی شما را به خطر نمی‌اندازد.



### ۳.۱. ایجاد حساب کاربری



دستور زیر یک کاربر جدید با نام "*dojo*" ایجاد می‌کند. این کاربر دارای یک دایرکتوری خانگی به نام `/home/dojo` و دسترسی به ترمینال bash خواهد بود. همچنین به گروه sudo اضافه می‌شود تا امکان اجرای دستورات مدیریتی را داشته باشد.



```bash
sudo useradd -s /bin/bash -d /home/dojo -m -G sudo dojo
```



### ۳.۲. تنظیم یک رمز عبور



مهم است که یک رمز عبور قوی به این حساب اختصاص دهید. ایده‌آل این است که از یک مدیر رمز عبور مانند Bitwarden استفاده کنید تا یک ترکیب طولانی و سخت برای حدس زدن ایجاد کنید.



```bash
sudo passwd dojo
```



سپس سیستم از شما می‌خواهد که رمز عبور انتخابی خود را وارد کرده و برای بار دوم تأیید کنید.



https://planb.academy/tutorials/computer-security/authentication/bitwarden-0532f569-fb00-4fad-acba-2fcb1bf05de9

### ۳.۳. مجوز دادن به کاربر برای استفاده از Docker



برای اینکه کاربر "*dojo*" بتواند کانتینرهای مورد نیاز برای اجرای Dojo را راه‌اندازی کند، باید به گروه Docker اضافه شود. این کار از نیاز به استفاده از `sudo` قبل از هر دستور جلوگیری می‌کند.



```bash
sudo usermod -aG docker dojo
```



### ۳.۴. راه‌اندازی مجدد سیستم



برای اعمال تغییرات گروه، دستگاه باید مجدداً راه‌اندازی شود.



```bash
sudo reboot
```



### ۳.۵. ورود با کاربر جدید



هنگامی که سیستم راه‌اندازی مجدد می‌شود، با ورود ***dojo*** و رمز عبوری که قبلاً تعریف کرده‌اید وارد شوید. تمام مراحل بعدی باید از این حساب اختصاصی انجام شود.



## ۴. دانلود و بررسی دوجو

```bash
torsocks wget http://zkaan2xfbuxia2wpf7ofnkbz6r5zdbbvxbunvp5g2iebopbfc4iqmbad.onion/vks/v1/by-fingerprint/E53AD419B242822F19E23C6D3033D463D6E544F6 && gpg --import E53AD419B242822F19E23C6D3033D463D6E544F6
```

### 4.2. دانلود آخرین نسخه Dojo



بایگانی فشرده حاوی کد منبع Dojo را بازیابی کنید. در این مثال، جدیدترین نسخه `1.27.0` است: فرمان را مطابق با [آخرین نسخه اینجا در مخزن رسمی GitHub](https://github.com/Dojo-Open-Source-Project/samourai-dojo/releases) تغییر دهید.



```bash
torsocks wget -O samourai-dojo-1.27.0.zip https://github.com/Dojo-Open-Source-Project/samourai-dojo/archive/refs/tags/v1.27.0.zip
```

### ۴.۳. اثر انگشت‌ها و امضا را دانلود کنید



توسعه‌دهندگان فایلی را منتشر می‌کنند که شامل اثر انگشت دیجیتال آرشیوها است، همچنین فایلی که با کلید PGP آن‌ها امضا شده است. آن‌ها را دانلود کنید تا فایل‌های خود را به صورت محلی مقایسه کنید.



```bash
torsocks wget https://github.com/Dojo-Open-Source-Project/samourai-dojo/releases/download/v1.27.0/samourai-dojo-1.27.0-fingerprints.txt && torsocks wget https://github.com/Dojo-Open-Source-Project/samourai-dojo/releases/download/v1.27.0/samourai-dojo-1.27.0-fingerprints.txt.sig
```

### ۴.۴. بررسی امضای PGP



بررسی کنید که فایل اثر انگشت توسط کلید وارد شده امضا شده است.



```bash

```



یک نتیجه صحیح یک امضای معتبر با کلید `E53AD419B242822F19E23C6D3033D463D6E544F6` و Address مرتبط `dojocoder@pm.me` را نمایش می‌دهد. ممکن است هشداری ظاهر شود که کلید تأیید نشده است: می‌توانید آن را نادیده بگیرید.



اگر، از سوی دیگر، امضا نامعتبر باشد، بلافاصله فرآیند نصب را متوقف کرده و از ابتدا شروع کنید.

## ۵. پیکربندی دوجو



دوجو یک سرور بک‌اند است که چندین سرویس را برای تعامل با پورتفولیوی شما و مدیریت نود Bitcoin شما گرد هم می‌آورد. پیکربندی آن می‌تواند پیچیده باشد، اما پروژه یک روش ساده‌شده ارائه می‌دهد که به‌طور خودکار اجزای زیر را نصب و پیکربندی می‌کند:




- Dojo (API اصلی)
- Bitcoin core (کامل Bitcoin گره)
- BTC-RPC Explorer (وب Block explorer)
- ایندکسر فولکروم (ایندکس‌گذاری سریع بلوک‌ها و تراکنش‌ها)
- سرور Fulcrum Electrum در شبکه Tor در دسترس است
- سرور Fulcrum Electrum در شبکه محلی موجود است
- اعتبارات مدیریت



### ۵.۱. مدارک مدیریت



برای دسترسی به خدمات مختلف، باید چندین شناسه منحصر به فرد را generate کنید:




- `BITCOIND_RPC_USER`
- `BITCOIND_RPC_PASSWORD`
- `MYSQL_ROOT_PASSWORD`
- mYSQL_USER
- `MYSQL_PASSWORD`
- nODE_API_KEY`
- `NODE_ADMIN_KEY`
- `NODE_JWT_SECRET`



این شناسه‌ها **باید منحصر به فرد باشند** (این بسیار مهم است: شما نباید از یک رمز عبور برای چندین سرویس استفاده کنید)، و فقط از اعداد، حروف بزرگ و حروف کوچک (الفبایی-عددی) تشکیل شده و حدود ۴۰ کاراکتر طول داشته باشند تا سطح امنیتی بالایی را تضمین کنند. بار دیگر، من به شدت توصیه می‌کنم از یک مدیر رمز عبور استفاده کنید.



### ۵.۲. دسترسی به فایل‌های پیکربندی



فایل‌های پیکربندی Dojo در پوشه `conf/` قرار دارند. به این دایرکتوری بروید:



```bash
cd ~/dojo-app/docker/my-dojo/conf/
```



![Image](assets/fr/21.webp)



### ۵.۳. پیکربندی Bitcoin core



فایل پیکربندی Bitcoin core را با ویرایشگر متن nano باز کنید:



```bash
nano docker-bitcoind.conf.tpl
```



![Image](assets/fr/22.webp)



در این فایل، شناسه‌های تولید شده را وارد کنید:



```
BITCOIND_RPC_USER=your-ID-here
BITCOIND_RPC_PASSWORD=your-password-here
```



⚠️ ***`your-ID-here` و `your-password-here` را با اطلاعات ورود خودتان (با یک رمز عبور قوی) جایگزین کنید.***



شما همچنین می‌توانید اندازه حافظه کش استفاده شده توسط Bitcoin core را برای بهبود عملکرد تنظیم کنید (حتی می‌توانید بیشتر استفاده کنید اگر RAM زیادی در دسترس دارید):



```
BITCOIND_DB_CACHE=2048
```



برای ذخیره تغییرات و بستن ویرایشگر:




- دکمه `Ctrl + X` را فشار دهید
- `y`
- سپس "*Enter*" را فشار دهید



### ۵.۴. پیکربندی MySQL

لطفاً جزئیات ورود خود را وارد کنید:




⚠️ ***`your-ID-here` و `your-password-here` را با اطلاعات ورود خودتان (با رمزهای عبور قوی و منحصر به فرد) جایگزین کنید.***



به همان روش ذخیره کنید (`Ctrl + X`, `y`, "*Enter*").



![Image](assets/fr/23.webp)



### ۵.۵. پیکربندی ایندکسر Fulcrum



فایل زیر را باز کنید:



```bash
nano docker-indexer.conf.tpl
```



پارامترها را برای فعال‌سازی Fulcrum اضافه کنید و آن را به‌درستی در Dojo ادغام کنید:



```
INDEXER_INSTALL=on
INDEXER_TYPE=fulcrum
INDEXER_BATCH_SUPPORT=active
INDEXER_EXTERNAL=on
```



![Image](assets/fr/24.webp)



سپس، بسته به پیکربندی شما، ۲ امکان وجود دارد. اگر Dojo روی ماشینی جدا از کامپیوتر روزمره شما نصب شده است (روی یک ماشین اختصاصی، یک سرور...)، IP آن را به صورت Address در شبکه محلی خود وارد کنید، به عنوان مثال:



```
INDEXER_EXTERNAL_IP=192.168.1.157
```



![Image](assets/fr/25.webp)



برای پیدا کردن IP محلی Address دستگاه خود، یک ترمینال دیگر باز کرده و فرمان زیر را وارد کنید:



```bash
hostname -I
```



امکان دوم: اگر Dojo مستقیماً بر روی رایانه شخصی روزمره شما اجرا می‌شود، مقدار پیش‌فرض موجود در فایل پیکربندی را حفظ کنید:



```
INDEXER_EXTERNAL_IP=127.0.0.1
```



ذخیره و خروج از ویرایشگر (`Ctrl + X`, `y`, "*Enter*").

![Image](assets/fr/26.webp)



سپس ایندکسر محلی را فعال کنید:

ذخیره و خروج از ویرایشگر (`Ctrl + X`, `y`, "*Enter*").



### ۵.۷. مدیریت ورود به سیستم



پس از اتمام پیکربندی، نیازی به ذخیره همه شناسه‌های تولید شده نیست. تنها چیزی که حتماً باید ذخیره شود عبارت است از:



```

```



این ورود به شما امکان می‌دهد تا بعداً به ابزار نگهداری Dojo وارد شوید. تمام ورودهای دیگر را می‌توان از مدیر رمز عبور یا یادداشت‌های دست‌نویس شما حذف کرد. در صورت نیاز به بازیابی آن‌ها در آینده، از فایل‌های پیکربندی Dojo قابل دسترسی باقی می‌مانند.



## ۶. نصب دوجو



در این مرحله، Dojo بر روی دستگاه شما نصب و راه‌اندازی خواهد شد. این عملیات چندین سرویس (Bitcoin core، ایندکسر Fulcrum، بک‌اند Dojo و غیره) را راه‌اندازی کرده و همگام‌سازی کامل Blockchain Bitcoin را آغاز خواهد کرد. این ممکن است بسته به سخت‌افزار و اتصال اینترنت شما چند روز طول بکشد.



### 6.1. بررسی کنید که Docker به درستی کار می‌کند



قبل از شروع نصب، مطمئن شوید که Docker عملیاتی است. فرمان زیر را اجرا کنید:



```bash
docker run hello-world
```



این دستور یک کانتینر تست کوچک را دانلود و اجرا می‌کند. اگر همه چیز به درستی کار کند، باید پیامی مشابه زیر را ببینید:



```
Hello from Docker!
This message shows that your installation appears to be working correctly...
```



![Image](assets/fr/27.webp)



اگر این پیام نمایش داده نمی‌شود، با راه‌اندازی مجدد دستگاه خود شروع کنید با:



```bash
sudo reboot
```



سپس دوباره وارد حساب **dojo** خود شوید و دستور تست را دوباره اجرا کنید. اگر خطا همچنان ادامه داشت، Docker به درستی نصب نشده است. در این صورت، به مرحله `2.4.` نصب Docker برگردید و هر دستور را با دقت بررسی کنید.



### ۶.۲. به دایرکتوری نصب Dojo بروید



اسکریپت‌های مورد نیاز برای نصب در پوشه `my-dojo` قرار دارند. به این دایرکتوری بروید:



```bash
cd ~/dojo-app/docker/my-dojo
```



![Image](assets/fr/28.webp)



از دستور `ls` استفاده کنید تا بررسی کنید که فایل `dojo.sh` موجود است. این اسکریپت اصلی است که نصب Dojo و راه‌اندازی تمامی سرویس‌های آن را خودکار می‌کند.



![Image](assets/fr/29.webp)



### 6.3. شروع نصب



نصب را با اجرای :



```bash
./dojo.sh install
```



نصب را با فشار دادن `y` و سپس "*Enter*" تأیید کنید.



![Image](assets/fr/30.webp)



این اسکریپت خواهد:




- دانلود و راه‌اندازی کانتینرهای Docker ضروری،
- راه‌اندازی Bitcoin core و شروع همگام‌سازی Blockchain،
- شروع شاخص Fulcrum برای ردیابی تراکنش‌ها و آدرس‌ها،
- پشتیبان Dojo و APIهای آن را فعال کنید.



شما یک جریان پیوسته از لاگ‌ها را خواهید دید که با ارجاعات رنگارنگی مانند `bitcoind`، `soroban`، `nodejs` یا `fulcrum` در حال اسکرول هستند. این اسکرول نشان می‌دهد که Dojo در حال اجرا است و شروع به اجرای خدمات مختلف کرده است.



![Image](assets/fr/31.webp)



### 6.4. نمایش گزارش خروجی



ورود به سیستم‌ها به صورت لحظه‌ای در ترمینال شما نمایش داده می‌شوند. برای بازگشت به خط فرمان در حالی که Dojo در پس‌زمینه در حال اجرا است، تایپ کنید:



```
Ctrl + C
```



نگران نباشید: متوقف کردن نمایش لاگ، خدمات را متوقف نمی‌کند. داکر به اجرای Dojo در پس‌زمینه ادامه می‌دهد (واضح است که اگر می‌خواهید IBD ادامه یابد، نیازی به خاموش کردن کامپیوتر ندارید).



### ۶.۵. درک *دانلود اولیه بلاک* (IBD)



در هنگام راه‌اندازی، Bitcoin core باید کل Blockchain را از سال 2009 دانلود و تأیید کند. این مرحله به نام ***دانلود اولیه بلاک* (IBD)** شناخته می‌شود. این مرحله ضروری است، زیرا به نود Dojo شما امکان می‌دهد تا هر بلاک و تراکنش Bitcoin را به‌طور مستقل تأیید کند.



مدت زمان این همگام‌سازی به چندین عامل بستگی دارد:




- قدرت پردازنده شما و میزان حافظه RAM موجود،
- سرعت دیسک شما،
- تعداد و کیفیت همتایانی که گره شما به آن‌ها متصل می‌شود،
- سرعت اتصال اینترنت شما.



در عمل، این عملیات معمولاً بین **۲ تا ۷ روز** طول می‌کشد. در این مدت، می‌توانید دستگاه خود را به‌طور مداوم روشن نگه دارید. هرچه دستگاه بیشتر روشن باشد، همگام‌سازی سریع‌تر انجام خواهد شد. توصیه می‌کنم وضعیت همگام‌سازی را به‌طور منظم با بررسی لاگ‌های Bitcoin core یا با استفاده از ابزار نگهداری Dojo پس از نصب (به بخش بعدی مراجعه کنید) بررسی کنید.



برای تعمیق دانش خود در مورد IBD و به طور کلی، عملکرد و نقش گره Bitcoin خود، توصیه می‌کنم به این دوره نگاهی بیندازید:



https://planb.academy/courses/3cd9cb94-82e8-417a-9c5a-02afc2589426


## ۷. نظارت بر همگام‌سازی



هنگام نصب Dojo برای اولین بار، باید منتظر بمانید تا دو عملیات اصلی به طور کامل انجام شوند: دانلود کامل Blockchain Bitcoin (*IBD*) و ایندکس‌گذاری این Blockchain توسط Fulcrum. بسته به سرعت اتصال و قدرت دستگاه شما، این ممکن است چندین روز طول بکشد. در این مدت، می‌توانید پیشرفت فرآیند را نظارت کنید تا مطمئن شوید همه چیز به خوبی پیش می‌رود.



دو روش برای نظارت بر وضعیت همگام‌سازی وجود دارد:




- استفاده از ابزار نگهداری دوجو (یا DMT)، که ساده است اما جزئیات کمی در طول IBD ارائه می‌دهد؛
- مشاوره مستقیم از لاگ‌های Dojo روی دستگاه شما، فنی‌تر اما بسیار دقیق‌تر.



### ۷.۱. از طریق ابزار نگهداری دوجو (DMT) بررسی کنید



ابزار نگهداری دوجو یک Interface امن و مبتنی بر وب است که به شما اجازه می‌دهد وضعیت کارخانه خود را نظارت کرده و عملیات خاصی را انجام دهید. این ساده‌ترین و در دسترس‌ترین راه برای نظارت بر پیشرفت IBD است. در طول مرحله همگام‌سازی اولیه، اطلاعات نمایش داده شده ممکن است محدود باشد. به عنوان مثال، DMT پیشرفت دقیق ایندکس‌گذاری Fulcrum را نشان نمی‌دهد. از سوی دیگر، هنگامی که همگام‌سازی کامل شد، DMT به وضوح نمایش خواهد داد:




- همه چراغ‌ها روی Green؛
- آخرین بلوک معتبر برای هر سرویس (Node، Indexer، Dojo DB).



برای دسترسی به آن، باید URL مربوط به DMT خود را بدانید و از طریق [مرورگر Tor](https://www.torproject.org/download/) به آن متصل شوید. برای انجام این کار، یک ترمینال باز کنید و به دایرکتوری `/my-dojo` بروید:



```bash
cd ~/dojo-app/docker/my-dojo
```



سپس فرمان زیر را اجرا کنید:



```bash
./dojo.sh onion
```



![Image](assets/fr/32.webp)



سپس به تمام اطلاعات مربوط به اتصالات به دوجوی خود از طریق تور دسترسی خواهید داشت. آدرسی که در اینجا به آن علاقه‌مندیم به شرح زیر است:



```
Dojo API and Maintenance Tool =
```



برای دسترسی به DMT از هر دستگاهی در هر شبکه‌ای (حتی به صورت از راه دور)، مرورگر Tor را باز کرده و این URL را به همراه `/admin` وارد کنید. به عنوان مثال، اگر URL شما `wo4zobymdl45gmmzzmpoypeemoukbj74wpibc22rxs2yfgpej62v6dyd.onion` است، باید در نوار [مرورگر Tor](https://www.torproject.org/download/) وارد کنید:



```
wo4zobymdl45gmmzzmpoypeemoukbj74wpibc22rxs2yfgpej62v6dyd.onion/admin
```



⚠️ **لطفاً این Address را کاملاً محرمانه نگه دارید**



سپس به صفحه احراز هویت هدایت خواهید شد. DMT با استفاده از رمز عبور `NODE_ADMIN_KEY` که قبلاً ایجاد کرده‌اید وارد می‌شود.



![Image](assets/fr/33.webp)



پس از ورود به سیستم، می‌توانید به *ابزار نگهداری دوجو* دسترسی پیدا کنید! در طول IBD، می‌توانید اطلاعات "*آخرین بلوک*" را در منوی "*Full node*" مشاهده کنید، که به شما وضعیت فعلی نود Bitcoin شما را نشان می‌دهد.



![Image](assets/fr/34.webp)



به خاطر داشته باشید که این Address را در مرورگر Tor برای بازیابی آسان‌تر بعداً نشانک‌گذاری کنید.



هنگامی که دوجوی شما به‌طور کامل همگام‌سازی شد، باید علامت Green ✅ را بر روی تمام شاخص‌ها در صفحه اصلی DMT مشاهده کنید.



### ۷.۲. تأیید از طریق لاگ‌های دوجو



راه دوم برای پیگیری پیشرفت IBD این است که مستقیماً به گزارش‌های ماشین خود مراجعه کنید. این روش نظارت دقیق‌تر و در زمان واقعی را ارائه می‌دهد. شما می‌توانید ببینید که Bitcoin core چگونه در حال پیشرفت در دانلود بلوک‌ها است و Fulcrum چگونه آن‌ها را فهرست‌بندی می‌کند.



به ماشینی که Dojo شما را میزبانی می‌کند متصل شوید و یک ترمینال باز کنید. تمام دستورات باید از دایرکتوری `my-dojo` اجرا شوند. خود را در این پوشه قرار دهید:



```bash
cd ~/dojo-app/docker/my-dojo
```



![Image](assets/fr/35.webp)



#### گزارش‌های Bitcoin core



مشاهده گزارش‌های Bitcoin core برای پیگیری پیشرفت IBD:



```bash
./dojo.sh logs bitcoind
```



در ابتدا، شما یک مرحله پیش‌همگام‌سازی از سرآیندهای بلوک را مشاهده خواهید کرد:



```
bitcoind | Pre-synchronizing blockheader, height : NNNNNN
```



وقتی این رقم به 100٪ برسد، Bitcoin core دانلود کامل Blockchain را آغاز خواهد کرد. پیشرفت IBD را مشاهده خواهید کرد. برای اطلاع از ارتفاع بلوک فعلی، به مقداری که با `height=` نشان داده شده نگاه کنید. همچنین می‌توانید کلید `progress=` را دنبال کنید که درصد پیشرفت IBD را نشان می‌دهد.



![Image](assets/fr/36.webp)



همان‌طور که همیشه، برای بستن لاگ‌ها و بازگشت به اعلان فرمان، از ترکیب `Ctrl + C` استفاده کنید.



#### گزارش‌های فولکروم



هنگامی که Bitcoin core پیش‌همگام‌سازی سرآیند را به پایان رساند، Fulcrum شروع به فهرست‌بندی Blockchain در حین پیشرفت می‌کند. لاگ‌های آن را با مشاهده کنید:



```bash
./dojo.sh logs fulcrum
```



سپس ارتفاع آخرین بلوک ایندکس شده را مشاهده خواهید کرد که پس از `height:` نشان داده شده است، همچنین درصد پیشرفت ایندکس را خواهید دید.



![Image](assets/fr/37.webp)



### ۷.۳. خرابی پایگاه داده Fulcrum



فولکروم یک ایندکسر بسیار قدرتمند است، اما نصب آن می‌تواند پیچیده باشد، به‌ویژه به دلیل مدیریت حساس پایگاه داده آن. در صورت قطع برق یا خاموش شدن ناگهانی دستگاه در طول همگام‌سازی اولیه، ممکن است پایگاه داده ایندکسر خراب شود. می‌توانید این را ببینید، به عنوان مثال، اگر لاگ‌های زیر را داشته باشید:



```bash
fulcrum | The database has been corrupted etc...
```



**توجه:** این نوع خطا باید با انتشار نسخه 2.0 Fulcrum اصلاح شود.



اگر این اتفاق برای شما بیفتد، هیچ تأثیری بر bitcoind (گره Bitcoin شما) نخواهد داشت: IBD آن به طور مستقل از Fulcrum به پیشرفت خود ادامه خواهد داد. با این حال، تا زمانی که داده‌های خراب آن را حذف نکرده و همگام‌سازی آن را از ابتدا شروع نکنید، نمی‌توانید از Fulcrum استفاده کنید. در اینجا نحوه کار آن آمده است:



توقف دوجو:



```bash
cd ~/dojo-app/docker/my-dojo
./dojo.sh stop
```



فقط کانتینر و حجم Fulcrum را حذف کنید:



```bash
docker rm -f fulcrum || true
docker volume ls | grep -i fulcrum
docker volume rm my-dojo_data-fulcrum
```



به طور معمول نام `my-dojo_data-fulcrum` است، اگر این مورد برای شما صدق نمی‌کند، نامی که توسط فرمان قبلی بازگردانده شده است را تطبیق دهید.



سپس Dojo را مجدداً راه‌اندازی کنید و Fulcrum را از ابتدا بازسازی کنید:



```bash
./dojo.sh upgrade
```



سپس می‌توانید با بررسی لاگ‌ها اطمینان حاصل کنید که Fulcrum به درستی کار می‌کند:



```bash
docker logs -f fulcrum
```




## ۸. استفاده از ابزار نگهداری دوجو



هنگامی که گره Bitcoin شما با سر وارپ با بیشترین Proof of Work همگام‌سازی شد و Blockchain به طور 100٪ توسط Fulcrum فهرست‌بندی شد، می‌توانید از دوجوی خود استفاده کنید.



دوجوی شما طیف گسترده‌ای از ویژگی‌ها را ارائه می‌دهد که با هر نسخه جدید به‌طور منظم بهبود می‌یابند. به نظر من، ۲ ویژگی مهم عبارتند از:




- امکان اتصال Ashigaru Wallet شما برای استفاده از نود خودتان جهت مشاوره داده‌های Blockchain و پخش تراکنش‌های شما،
- و Block explorer، که به شما امکان دسترسی به اطلاعات Blockchain و Bitcoin را می‌دهد بدون اینکه داده‌های شما را به یک نهاد خارجی که کنترل نمی‌کنید، افشا کند.



بیایید بفهمیم چگونه از آنها استفاده کنیم.


### ۸.۱. آشیکارو را به دوجوی خود متصل کنید



اتصال Ashigaru Wallet شما به Dojo بسیار ساده است: پس از ورود به DMT خود، منوی "*Pairing*" را باز کنید. یک کد QR ظاهر می‌شود که می‌توانید مستقیماً با برنامه Ashigaru آن را اسکن کنید.



![Image](assets/fr/38.webp)



در برنامه Ashigaru، اولین باری که آن را پس از ایجاد یا بازیابی Wallet خود اجرا می‌کنید، به صفحه "*پیکربندی سرور Dojo خود*" هدایت خواهید شد. دکمه "*اسکن QR*" را فشار دهید، سپس کد QR نمایش داده شده روی DMT خود را اسکن کنید.



![Image](assets/fr/39.webp)



سپس روی دکمه "*ادامه*" کلیک کنید.



![Image](assets/fr/40.webp)



شما اکنون به دوجوی خود متصل شده‌اید.



![Image](assets/fr/41.webp)



### 8.2. استفاده از Block explorer



دوجو به‌طور خودکار یک Block explorer نصب می‌کند، [BTC-RPC Explorer](https://github.com/janoside/btc-RPC-explorer)، که به‌طور مستقیم از داده‌های نود Bitcoin شما استفاده می‌کند. یک اکسپلورر به شما اجازه می‌دهد تا از طریق یک وب Interface که به‌راحتی قابل درک است، به اطلاعات خام از Blockchain و Mempool خود دسترسی پیدا کنید. بنابراین می‌توانید، به عنوان مثال، وضعیت یک تراکنش در حال انتظار را بررسی کنید، موجودی یک Address را مشاهده کنید یا به‌راحتی ترکیب یک بلاک را بررسی کنید.



برای دسترسی به آن، به سادگی Tor Address را از مرورگر خود بازیابی کنید. برای انجام این کار، همان دستوری را که برای به‌دست‌آوردن Address از DMT خود استفاده کردید، اجرا کنید:



```bash
./dojo.sh onion
```



![Image](assets/fr/42.webp)



شما از طریق Tor به تمام اطلاعات اتصال Dojo خود دسترسی خواهید داشت. موردی که در اینجا به آن علاقه‌مندیم، URL زیر است:



```
Block Explorer =
```



اگر قبلاً به DMT خود متصل شده‌اید، می‌توانید این Address را نیز در منوی "*Pairing*" داخل JSON اتصال پیدا کنید:



![Image](assets/fr/43.webp)



برای دسترسی به مرورگر خود از هر دستگاهی در هر شبکه‌ای (حتی از راه دور)، [مرورگر تور](https://www.torproject.org/download/) را باز کرده و URLی که به‌تازگی دریافت کرده‌اید را وارد کنید.



⚠️ **لطفاً این Address را کاملاً محرمانه نگه دارید**



سپس به Block explorer خود دسترسی خواهید داشت.



![Image](assets/fr/44.webp)



*اعتبار تصویر: https://ashigaru.rs/.*



برای پیگیری یک تراکنش، به سادگی txid آن را در نوار جستجو در بالا سمت راست وارد کنید.



![Image](assets/fr/45.webp)



*اعتبار تصویر: https://ashigaru.rs/.*



برای بررسی حرکات مرتبط با Address، به همان روش با وارد کردن Address در نوار جستجو اقدام کنید.



![Image](assets/fr/46.webp)



*اعتبار تصویر: https://ashigaru.rs/.*



همچنین می‌توانید Hash یا ارتفاع یک بلوک را در نوار جستجو وارد کنید تا جزئیات نمایش داده شود.



![Image](assets/fr/47.webp)



*اعتبار تصویر: https://ashigaru.rs/.*



## ۹. نگهداری دوجو



### ۹.۱ دوجوی خود را متوقف کنید



هرگز برق Dojo خود را به طور ناگهانی قطع نکنید، زیرا این کار ممکن است برخی از پایگاه‌های داده، به ویژه ایندکسر Fulcrum را خراب کند. اگر مجبور به خاموش کردن آن هستید، همیشه یک خاموش کردن تمیز از Dojo انجام دهید، سپس، پس از تکمیل تمام مراحل، دستگاه را نیز خاموش کنید:



```bash
cd ~/dojo-app/docker/my-dojo
./dojo.sh stop
```



### 9.2 دوجوی خود را به‌روزرسانی کنید



دوجو به طور منظم تکامل می‌یابد و نسخه‌های جدید برای رفع اشکالات، بهبود پایداری و افزودن ویژگی‌ها منتشر می‌شوند. بنابراین مهم است که [به طور منظم برای به‌روزرسانی‌ها بررسی کنید](https://github.com/Dojo-Open-Source-Project/samourai-dojo/releases) و دوجوی خود را به‌روزرسانی کنید. این فرآیند مشابه نصب اولیه است، اما نیاز دارد که برخی فایل‌ها را با آخرین نسخه موجود جایگزین کنید، در حالی که پیکربندی خود را حفظ می‌کنید. در اینجا مراحل دقیق برای یک به‌روزرسانی تمیز و امن آورده شده است:



برای اطلاع از نسخه فعلی Dojo خود، فرمان زیر را اجرا کنید:



```bash
./dojo.sh version
```



اگرچه این مرحله اختیاری است، من توصیه می‌کنم که با به‌روزرسانی سیستم‌عامل خود شروع کنید. این کار خطر ناسازگاری‌ها را کاهش می‌دهد و اطمینان حاصل می‌کند که وابستگی‌های استفاده‌شده توسط Dojo به‌روز هستند:



```bash
sudo apt-get update
sudo apt-get upgrade
```



به دایرکتوری Dojo بروید و سرویس‌های فعلی را متوقف کنید:



```bash
cd ~/dojo-app/docker/my-dojo
./dojo.sh stop
```



سپس سیستم خود را راه‌اندازی مجدد کنید تا یک صفحه تمیز داشته باشید:



```bash
sudo reboot
```



دوجو با فایل‌های امضاشده دیجیتالی ارائه می‌شود. این امضاهای PGP تضمین می‌کنند که فایل‌ها از سوی توسعه‌دهنده هستند و به هیچ وجه تغییر نکرده‌اند. مهم است که هر بار که دوجو را به‌روزرسانی می‌کنید، آن‌ها را بررسی کنید، همان‌طور که هنگام نصب اولیه انجام دادید. ابتدا کلید عمومی توسعه‌دهنده را از طریق تور دانلود کرده و سپس آن را وارد کنید:



```bash
torsocks wget http://zkaan2xfbuxia2wpf7ofnkbz6r5zdbbvxbunvp5g2iebopbfc4iqmbad.onion/vks/v1/by-fingerprint/E53AD419B242822F19E23C6D3033D463D6E544F6 && gpg --import E53AD419B242822F19E23C6D3033D463D6E544F6
```



به دایرکتوری شخصی خود بازگردید:



```bash
cd ~/
```



نسخه‌ی جدید Dojo را از GitHub از طریق Tor دانلود کنید. در این مثال، نسخه `1.28.0` است (که در زمان نوشتن هنوز وجود ندارد: این فقط برای ارائه یک مثال است). به یاد داشته باشید که فایل و لینک را [با نسخه‌ای که می‌خواهید نصب کنید](https://github.com/Dojo-Open-Source-Project/samourai-dojo/releases) جایگزین کنید:



```bash
torsocks wget -O samourai-dojo-1.28.0.zip https://github.com/Dojo-Open-Source-Project/samourai-dojo/archive/refs/tags/v1.28.0.zip
```



همچنین فایل حاوی اثر انگشت‌ها و امضای PGP را دانلود کنید (دوباره به یاد داشته باشید که نسخه را در فرمان تنظیم کنید):



```bash
torsocks wget https://github.com/Dojo-Open-Source-Project/samourai-dojo/releases/download/v1.28.0/samourai-dojo-1.28.0-fingerprints.txt && torsocks wget https://github.com/Dojo-Open-Source-Project/samourai-dojo/releases/download/v1.28.0/samourai-dojo-1.28.0-fingerprints.txt.sig
```



بررسی کنید که فایل اثر انگشت دانلود شده توسط کلید توسعه‌دهنده امضا شده است:



```bash
gpg --verify samourai-dojo-1.28.0-fingerprints.txt.sig
```



نتیجه صحیح باید نمایش داده شود :



```
gpg: Signature made [date + time]
gpg: using EDDSA key E53AD419B242822F19E23C6D3033D463D6E544F6
gpg: Good signature from "dojocoder@pm.me" <dojocoder@pm.me> [unknown]
```



هشداری مبنی بر اینکه کلید تأیید نشده است ممکن است ظاهر شود، اما این موضوع اهمیتی ندارد. از سوی دیگر، اگر امضا نامعتبر باشد یا با کلید دیگری مطابقت داشته باشد، ادامه ندهید و دوباره شروع کنید و لینک‌ها را بررسی کنید.



سپس اثر انگشت SHA256 آرشیو را محاسبه کرده و آن را با فایل اثر انگشت رسمی مقایسه کنید:



```bash
sha256sum samourai-dojo-1.28.0.zip
cat samourai-dojo-1.28.0-fingerprints.txt
```



اگر دو اثر انگشت کاملاً مطابقت داشته باشند، آرشیو معتبر و دست‌نخورده است. اگر تفاوت داشته باشند، فایل‌ها را فوراً حذف کنید و ادامه ندهید.



آرشیو را در دایرکتوری خانگی خود از حالت فشرده خارج کنید:



```bash
unzip samourai-dojo-1.28.0.zip -d .
```



سپس محتویات را به دایرکتوری Dojo کپی کنید و جایگزین قدیمی کنید:



```bash
cp -a samourai-dojo-1.28.0/. dojo-app/
```



این عملیات فایل‌های پیکربندی شما را که در `~/dojo-app/docker/my-dojo/conf` قرار دارند حفظ می‌کند، اما تمام فایل‌های دیگر را با نسخه‌های به‌روز شده جایگزین می‌کند.



برای تمیز نگه داشتن محیط خود، فایل‌های غیرضروری را حذف کنید :



```bash
rm -r samourai-dojo-1.28.0 && rm samourai-dojo-1.28.0.zip && rm samourai-dojo-1.28.0-fingerprints.txt && rm samourai-dojo-1.28.0-fingerprints.txt.sig && rm E53AD419B242822F19E23C6D3033D463D6E544F6
```



به دایرکتوری اسکریپت‌های Dojo برگردید و فرمان به‌روزرسانی را اجرا کنید:



```bash
cd ~/dojo-app/docker/my-dojo
./dojo.sh upgrade -y
```



این دستور به Docker می‌گوید تا تصاویر را با فایل‌های جدید بازسازی کند و سپس به‌طور خودکار تمام سرویس‌ها را مجدداً راه‌اندازی کند. در پایان فرآیند، لاگ‌ها را بررسی کنید تا مطمئن شوید که Bitcoin core، Fulcrum و Dojo همگی به‌درستی کار می‌کنند:



```bash
./dojo.sh logs bitcoind
./dojo.sh logs fulcrum
```



اگر خدمات بدون خطا شروع شوند و لاگ‌ها نشان دهند که بلوک‌ها در حال پردازش هستند، دوجوی شما اکنون به‌روز و عملیاتی است.
