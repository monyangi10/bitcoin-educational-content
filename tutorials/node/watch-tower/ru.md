## Как работают сторожевые башни?



Являясь неотъемлемой частью экосистемы Lightning Network, _Watchtowers_ обеспечивают дополнительный уровень защиты для каналов Lightning пользователей. Их основная задача - следить за состоянием канала и вмешиваться, если одна сторона канала пытается обмануть другую.



Как Watchtower может определить, был ли взломан канал? Он получает от клиента (одной из сторон канала) информацию, необходимую для правильной идентификации и устранения последствий взлома. Эта информация включает детали последней транзакции, текущий статус канала и Elements, необходимый для создания штрафных транзакций. Перед передачей этих данных в Watchtower клиент может зашифровать их, чтобы сохранить конфиденциальность. Таким образом, даже если Watchtower получит данные, он не сможет их расшифровать до тех пор, пока не произойдет нарушение. Этот механизм шифрования защищает конфиденциальность клиента и не позволяет Watchtower получить несанкционированный доступ к конфиденциальной информации.



В этом уроке мы рассмотрим 3 способа использования **Watchtower** :




- во-первых, классический сырой метод через LND,
- затем еще один подход с Eye of Satoshi,
- и, наконец, упрощенная конфигурация Watchtower на узле Lightning, размещенном в Umbrel.



## 1 - Настройка Watchtower или клиента через LND



*Это руководство взято из [официальной документации по LND](https://github.com/lightningnetwork/LND/blob/master/docs/Watchtower.md). В первоначальную версию могли быть внесены некоторые изменения



Начиная с версии 0.7.0, `LND` поддерживает выполнение частного альтруистического Watchtower в качестве полностью интегрированной подсистемы `LND`. Сторожевые башни обеспечивают вторую линию защиты от сценариев злонамеренного или случайного взлома, когда узел-клиент находится в автономном режиме или не может ответить в момент взлома, предлагая повышенную степень безопасности для средств канала.



В отличие от _вознаграждающей сторожевой башни_, которая требует долю средств канала в обмен на свои услуги, _альтруистическая сторожевая башня_ возвращает все средства жертвы (за вычетом платы за On-Chain) без взимания комиссии. Наградные сторожевые башни будут активированы в более поздней версии; они все еще находятся на стадии тестирования и улучшения.



Кроме того, `LND` теперь может быть настроен на работу в качестве _клиента сторожевой башни_, сохраняя зашифрованные транзакции по устранению бреши (так называемые "транзакции справедливости") от других альтруистических сторожевых башен. Watchtower хранит зашифрованные блобы фиксированного размера и может расшифровать и опубликовать транзакцию правосудия только после того, как нарушитель передаст отмененное состояние Commitment. Коммуникации между клиентом и Watchtower шифруются и аутентифицируются с помощью эфемерных пар ключей, что ограничивает возможности Watchtower по отслеживанию своих клиентов с помощью долгосрочных учетных данных.



Обратите внимание, что мы решили включить в этот выпуск ограниченный набор функций, уже обеспечивающих значительную безопасность пользователей `LND`. Многие другие функции, связанные с Watchtower, либо близки к завершению, либо находятся на стадии разработки; мы продолжим их развертывание по мере тестирования и как только они будут признаны безопасными.



примечание: на данный момент сторожевые башни сохраняют только `to_local` и `to_remote` выходные данные отозванных обязательств; сохранение выходных данных HTLC будет развернуто в будущей версии, поскольку протокол может быть расширен для включения дополнительных данных подписи в зашифрованные блобы._



### Настройка Watchtower



Чтобы настроить Watchtower, пользователям командной строки необходимо скомпилировать дополнительный подсервер `watchtowerrpc`, который позволяет взаимодействовать с Watchtower через gRPC или `lncli`. Публикуемые двоичные файлы по умолчанию включают в себя подсервер `watchtowerrpc`.



Минимальная конфигурация для активации Watchtower - `Watchtower.active=1`.



Вы можете получить информацию о конфигурации Watchtower с помощью команды `lncli tower info` :



```shell
$  lncli tower info
{
"pubkey": "03281d603b2c5e19b8893a484eb938d7377179a9ef1a6bca4c0bcbbfc291657b63",
"listeners": [
"[::]:9911"
],
"uris": [
],
}
```



Полный набор опций конфигурации Watchtower доступен через `LND -h` :



```shell
$  lnd -h
...
watchtower:
--watchtower.active                                     If the watchtower should be active or not
--watchtower.towerdir=                                  Directory of the watchtower.db (default: $HOME/.lnd/data/watchtower)
--watchtower.listen=                                    Add interfaces/ports to listen for peer connections
--watchtower.externalip=                                Add interfaces/ports where the watchtower can accept peer connections
--watchtower.readtimeout=                               Duration the watchtower server will wait for messages to be received before hanging up on client connections
--watchtower.writetimeout=                              Duration the watchtower server will wait for messages to be written before hanging up on client connections
...
```



#### Интерфейсы прослушивания



По умолчанию Watchtower прослушивает `:9911`, что соответствует порту `9911` на всех доступных интерфейсах. Пользователи могут задать свои собственные прослушивающие интерфейсы с помощью опции `--Watchtower.listen=`. Вы можете проверить свою конфигурацию в поле `"listeners"` в `lncli tower info`. Если у вас возникли проблемы с подключением к Watchtower, убедитесь, что `<порт>` открыт или что ваш прокси-сервер правильно настроен на активный Interface.



#### Внешние IP-адреса



Пользователи также могут указать внешний IP Address(es) Watchtower с помощью `Watchtower.externalip=`, который раскрывает полный URI Watchtower (pubkey@host:port) через RPC или `lncli tower info` :



```shell
$  lncli tower info
...
"uris": [
"03281d603b2c5e19b8893a484eb938d7377179a9ef1a6bca4c0bcbbfc291657b63@1.2.3.4:9911"
]
```



URI Watchtower могут быть переданы клиентам для подключения и использования с помощью следующей команды:



```shell
$  lncli wtclient add 03281d603b2c5e19b8893a484eb938d7377179a9ef1a6bca4c0bcbbfc291657b63@1.2.3.4:9911
```



Если клиентам Watchtower требуется удаленный доступ, убедитесь, что :




- Откройте порт 9911 (или порт, определенный с помощью `Watchtower.listen`).
- Используйте прокси-сервер для перенаправления трафика с открытого порта на прослушивающий Watchtower порт Address.



#### Скрытые услуги Tor



Watchtowers поддерживает скрытые сервисы Tor и может автоматически запускать generate со следующими опциями:



```shell
$  lnd --tor.active --tor.v3 --watchtower.active
```



После этого .onion Address появляется в поле `"uris"` при запросе `lncli tower info`:



```shell
$  lncli tower info
...
"uris": [
"03281d603b2c5e19b8893a484eb938d7377179a9ef1a6bca4c0bcbbfc291657b63@bn2kxggzjysvsd5o3uqe4h7655u7v2ydhxzy7ea2fx26duaixlwuguad.onion:9911"
]
```



примечание: открытый ключ Watchtower отличается от открытого ключа узла `LND`. На данный момент он действует как "белый список Soft", поскольку клиентам необходимо знать открытый ключ Watchtower, чтобы использовать его в качестве резервного, в ожидании более продвинутых механизмов белого списка. Мы не рекомендуем открыто раскрывать этот открытый ключ, если только вы не готовы выставить свой Watchtower на всеобщее обозрение в Интернете._



#### Каталог базы данных Watchtower



Базу данных Watchtower можно переместить с помощью опции `Watchtower.towerdir=`. Обратите внимание, что к выбранному пути будет добавлен суффикс `/Bitcoin/Mainnet/Watchtower.db`, чтобы изолировать базы данных по строкам. Таким образом, установка `Watchtower.towerdir=/path/to/towerdir` приведет к созданию базы данных по адресу `/path/to/towerdir/Bitcoin/Mainnet/Watchtower.db`.



В Linux, например, база данных Watchtower по умолчанию находится по адресу :


`/home/$USER/.LND/data/Watchtower/Bitcoin/Mainnet/Watchtower.db`



### Настройка клиента Watchtower



Для настройки клиента Watchtower вам понадобятся два предмета:





- Активируйте клиент Watchtower с помощью опции `--wtclient.active`.



```shell
$  lnd --wtclient.active
```





- URI активного Watchtower.



```shell
$  lncli wtclient add 03281d603b2c5e19b8893a484eb938d7377179a9ef1a6bca4c0bcbbfc291657b63@1.2.3.4:9911
```



Таким образом можно настроить несколько сторожевых башен.



#### Ставки комиссионных за юридические операции



Пользователи могут опционально установить ставку комиссии за транзакции правосудия с помощью опции `wtclient.sweep-fee-rate`, которая принимает значения в сат/байт. По умолчанию значение равно 10 сат/байт, но можно установить более высокую ставку, чтобы добиться более высокого приоритета во время пиковых нагрузок. Изменение `sweep-fee-rate` применяется ко всем новым обновлениям после перезапуска daemon.



#### Надзор



С помощью команды `lncli wtclient` пользователи теперь могут напрямую взаимодействовать с клиентом Watchtower, чтобы получить или изменить информацию обо всех зарегистрированных сторожевых башнях.



Например, с помощью `lncli wtclient tower` можно узнать количество сессий, которые в данный момент согласованы с Watchtower, добавленным выше, и определить, используется ли он для резервного копирования, благодаря полю `active_session_candidate`.



```shell
$  lncli wtclient tower 03281d603b2c5e19b8893a484eb938d7377179a9ef1a6bca4c0bcbbfc291657b63
{
"pubkey": "03281d603b2c5e19b8893a484eb938d7377179a9ef1a6bca4c0bcbbfc291657b63",
"addresses": [
"1.2.3.4:9911"
],
"active_session_candidate": true,
"num_sessions": 1,
"sessions": []
}
```



Чтобы получить информацию о сеансах Watchtower, используйте опцию `--include_sessions`.



```shell
$  lncli wtclient tower --include_sessions 03281d603b2c5e19b8893a484eb938d7377179a9ef1a6bca4c0bcbbfc291657b63
{
"pubkey": "03281d603b2c5e19b8893a484eb938d7377179a9ef1a6bca4c0bcbbfc291657b63",
"addresses": [
"1.2.3.4:9911"
],
"active_session_candidate": true,
"num_sessions": 1,
"sessions": [
{
"num_backups": 0,
"num_pending_backups": 0,
"max_backups": 1024,
"sweep_sat_per_vbyte": 10
}
]
}
```



Все параметры конфигурации клиента Watchtower доступны через `lncli wtclient -h` :



```shell
$  lncli wtclient -h
NAME:
lncli wtclient - Interact with the watchtower client.

USAGE:
lncli wtclient command [command options] [arguments...]

COMMANDS:
add     Register a watchtower to use for future sessions/backups.
remove  Remove a watchtower to prevent its use for future sessions/backups.
towers  Display information about all registered watchtowers.
tower   Display information about a specific registered watchtower.
stats   Display the session stats of the watchtower client.
policy  Display the active watchtower client policy configuration.

OPTIONS:
--help, -h  show help
```




## 2 - Установка собственного глаза Satoshi



*Этот туториал частично взят из статьи на [Summer of Bitcoin Blog](https://blog.summerofbitcoin.org/). В оригинальную версию были внесены изменения*



The Eye of Satoshi ([Rust-TEOS](https://github.com/talaia-labs/Rust-teos)) - это бездепозитный Watchtower Lightning, соответствующий [Bolt 13](https://github.com/sr-gi/bolt13/blob/master/13-watchtowers.md?ref=blog.summerofbitcoin.org). Он состоит из двух основных компонентов:





- teos**: включает в себя командную строку Interface (CLI) и основные серверные функции Watchtower. При компиляции этого _крафта_ создаются два двоичных файла - **teosd** и **teos-CLI**.





- teos-common**: включает общую функциональность на стороне сервера и клиента (полезно для создания клиента).



Для корректной работы Watchtower необходимо запустить команду **bitcoind** перед запуском Watchtower с помощью команды **teosd**. Перед выполнением этих двух команд необходимо настроить файл **Bitcoin.conf**. Вот как это сделать:





- Установите Bitcoin core из исходного кода или скачайте его. После загрузки поместите файл **Bitcoin.conf** в каталог пользователя Bitcoin core. Смотрите эту ссылку для получения дополнительной информации о том, куда поместить файл, поскольку это зависит от используемой операционной системы.





- После того как местоположение будет определено, добавьте следующие параметры:



```shell
# RPC
server=1
rpcuser=<your-user>
rpcpassword=<your-password>

# chaîne
regtest=1
```





- сервер**: для запросов RPC





- rpcuser** и **rpcpassword**: аутентификация клиентов RPC на сервере





- regtest**: не обязательно, но полезно, если вы планируете разработку.



Значения для **rpcuser** и **rpcpassword** выбираете вы сами. Они должны быть написаны без кавычек. Например:



```shell
rpcuser=aniketh
rpcpassword=strongpassword
```



Теперь, если вы запустите **bitcoind**, узел должен запуститься.





- Для части Watchtower необходимо сначала установить **teos** из исходников. Следуйте инструкциям, приведенным в этой ссылке.





- После того как вы успешно установили **teos** на свою систему и провели тесты, можно переходить к последнему шагу: настройке файла **teos.toml** в каталоге пользователя teos. Файл должен быть помещен в папку с именем **.teos** (обратите внимание на точку) в вашей домашней директории. Например, **/home//.teos** в Linux. После того как местоположение найдено, создайте файл **teos.toml** и установите эти параметры в соответствии с изменениями, внесенными в **bitcoind** :



```shell
# bitcoind
btc_network = "regtest"
btc_rpc_user = <your-user>
btc_rpc_password = <your-password>
```



Обратите внимание, что здесь имя пользователя и пароль должны быть написаны **в кавычках**. Используя предыдущий пример :



```shell
btc_rpc_user = "aniketh"
btc_rpc_password = "strongpassword"
```



После этого вы должны быть готовы к запуску Watchtower. Поскольку мы работаем на **regtest**, скорее всего, при первом подключении Watchtower в нашей тестовой сети Bitcoin не было добыто ни одного блока (если было, значит, что-то не так). Watchtower создает внутренний кэш из последних 100 блоков **bitcoind**; поэтому при первом запуске вы можете получить следующую ошибку:



```shell
ERROR [teosd] Not enough blocks to start the tower (required: 100). Mine at least 100 more
```



Поскольку мы используем **regtest**, мы можем получить Miner блоков, подав команду RPC, без необходимости ждать средней 10-минутной задержки, которая наблюдается в других сетях (например, Mainnet или Testnet). Подробности о том, как создавать блоки Miner, см. в справке **bitcoin-cli**.



![Image](assets/fr/01.webp)



Вот и все: вы успешно запустили Watchtower. Поздравляем. 🎉




## 3 - Настройка Watchtower на Umbrel



На Umbrel подключение к Watchtower для защиты узла Lightning очень простое, так как все делается через графический Interface. После удаленного подключения к узлу откройте приложение "**Lightning Node**".



![Image](assets/fr/02.webp)



Нажмите на три маленькие точки в правом верхнем углу Interface, затем выберите "**Дополнительные настройки**".



![Image](assets/fr/03.webp)



В меню "**Watchtower**" доступны две опции:





- Watchtower Service**: эта опция позволяет вам управлять Watchtower, то есть сервисом, который отслеживает каналы других узлов для выявления попыток мошенничества. В случае взлома ваш Watchtower публикует транзакцию на Blockchain, позволяя пользователям вернуть свои заблокированные средства. После активации появляется URI вашего Watchtower, который может быть передан другим узлам, чтобы они могли добавить его в свой клиент Watchtower;





- Watchtower Client**: эта опция позволяет подключаться к внешним сторожевым башням для защиты собственных каналов. После активации вы можете добавить сервисы Watchtower, которым ваш узел будет передавать необходимую информацию о своих каналах. Эти сторожевые башни будут следить за их состоянием и вмешиваться в случае попыток мошенничества.



Приоритетом для вас, конечно, является активация *Watchtower Client* для защиты вашего узла, но я также рекомендую вам активировать *Watchtower Service*, чтобы в свою очередь внести вклад в безопасность других пользователей.



![Image](assets/fr/04.webp)



Затем нажмите на кнопку Green "**Сохранить и перезапустить узел**". Ваш LND перезапустится.



В этом же меню вы найдете URI вашей службы Watchtower, если вы ее активировали. Вы также можете добавить URI внешнего Watchtower для защиты ваших каналов. Нажмите на "**ADD**" для подтверждения.



![Image](assets/fr/05.webp)



В сети доступно несколько Сторожевых башен. Например, [LN+ и Voltage предлагают альтруистическую Watchtower](https://lightningnetwork.plus/Watchtower), к которой вы можете подключиться:



```
023bad37e5795654cecc69b43599da8bd5789ac633c098253f60494bde602b60bf@iiu4epqzm6cydqhezueenccjlyzrqeruntlzbx47mlmdgfwgtrll66qd.onion:9911
```



![Image](assets/fr/06.webp)



Другой вариант - передать свой URI Exchange своим товарищам по биткойну, чтобы каждый защищал узел другого.



Я также рекомендую установить несколько Сторожевых башен, чтобы снизить риски в случае, если одна из них окажется недоступной.



Наконец, вы можете настроить параметр "**Watchtower Client Sweep Fee Rate**". Он определяет максимальную ставку комиссии, которую вы готовы заплатить за включение в блок транзакции наказания за трансляцию Watchtower. Убедитесь, что вы установили достаточно высокое значение, соответствующее суммам, заблокированным в ваших каналах.
