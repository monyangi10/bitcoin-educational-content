## 簡介



Lightning Network 是建構在 Bitcoin 之上的快速、低成本支付解決方案，可進行即時、安全的交易。觀察這個網路對於瞭解其運作方式、其拓樸結構以及組成它的節點的狀態至關重要。Lightning explorer（如 1ML）用於可視化網絡的公共數據，包括活動節點、開放通道和可用容量，為用戶、開發者和節點運營商提供有價值的概覽。

這些數字會定期更新，以反映網路的現況。從這些數字可以了解網路的規模、成長和穩定性。

## 探索 Lightning 節點



要探索 1ML 上的節點，請從頁面頂端的搜尋列開始。您可以輸入**節點 ID**，即節點的唯一識別碼，或其**別名**，這是一個較容易記憶的名稱。



搜尋完成後，按一下對應的節點，即可進入其詳細頁面。



![capture](assets/fr/07.webp)



此頁面會顯示幾項重要資訊：





- 節點 ID**：這個唯一的識別碼是一長串字符，可讓節點在整個網路中被精確識別。



![capture](assets/fr/08.webp)





- 別名**：這是節點所有者選擇用來公開表示節點的名稱。



![capture](assets/fr/09.webp)





- 通道數量**表示節點與其他節點開啟了多少連接，而總容量**表示這些通道中可用的比特幣總和。擁有大量通道和高容量的節點一般都有良好的連線，並能夠在網路上快速轉移大量資金。



![capture](assets/fr/10.webp)





- **uptime** 或可用性衡量節點在線上保持活躍和可存取的時間，反映其可靠性。另一方面，節點的**年齡**表示該節點在網路中存在的時間，反映其在 Lightning Network 中的穩定性和經驗。



![capture](assets/fr/11.webp)



這些資料可協助您瞭解節點在 Lightning Network 中的重要性和可靠性。舉例來說，擁有大量頻道、高容量和高正常運作時間的節點通常是網路中的重要角色。



## 探索閃電通道



### 什麼是 Lightning 通道？



Lightning 通道是兩個網路節點之間的直接連接。它使這兩個節點能夠交換即時、低成本的付款，而無需每次交易都經過 Bitcoin 主鏈。通道是 Lightning Network 快速和可擴展的基礎。



### 閱讀 1ML 的頻道資訊



在 1ML 上，每個通道都有自己的頁面或描述表，其中包含許多重要的資料：



您可從節點頁面存取其頻道清單。按一下頻道，1ML 會顯示一個專用頁面，其中包含幾項關鍵資訊。



![capture](assets/fr/12.webp)



![capture](assets/fr/13.webp)



主要可見資料如下：





- 夥伴節點**：每個通道連結兩個節點。1ML 會顯示這兩個識別碼及其各自的別名。



![capture](assets/fr/14.webp)





- 通道容量**：這是鎖定在此通道的比特幣總額。此容量代表可透過此通道支付的最大限額。



![capture](assets/fr/15.webp)





- 通道年齡**：表示通道開啟的時間。老頻道通常代表兩個節點之間的關係穩定。



![capture](assets/fr/16.webp)



### 航道能見度限制



重要的是要了解 1ML 只顯示 Lightning Network 的**部分**。探索器只顯示**公開頻道**，也就是那些已在網路中公佈的頻道。因保密或策略原因而經常使用的私人頻道則無法顯示。此外，1ML 不會顯示頻道每一邊的確實資金分佈，也不會顯示已支付的款項或某一特定時刻實際可用的流動資金。因此，顯示的資訊可用於分析網路的**一般結構，但無法分析其實際財務活動或詳細的流動性狀態。



## 依地點探索 Lightning Network



### 按國家和地區劃分的節點



1ML 可讓您根據**地理細分**探索 Lightning Network。從首頁或透過專屬區段，可依國家或地區顯示節點。此功能基於節點經營者所申報的位置資訊。


在導覽列中，您會看到 **Location** 連結。在頁面上，節點依大洲、國家和城市分類。



![capture](assets/fr/17.webp)



選擇一個國家後，1ML 會顯示相關節點的清單，以及該地理區域的節點數量和可見總容量等彙總統計資料。



#### 這說明了當地領養的情況





- 技術採用**：一個地區有大量節點，表示 Bitcoin 用戶、公司或服務正積極採用 Lightning Network。這表明技術成熟，並願意利用 Lightning 的優點（交易快速、成本較低）。
- 經濟生態系統** ：密集的節點也代表 Bitcoin 周邊的當地經濟結構：接受 Lightning 的商家、開發工具的新創公司、社區活動等。
- 安全性與彈性**：多樣化的地理分佈可增強網路在面對局部中斷或限制時的彈性。節點越分散，網路就越分散、越難審查。
- 政策與法規**：有些國家可能會因為有利的法規框架或積極主動的社群而加快採用速度。相反地，在法規嚴格或充滿敵意的地區，節點數量會較少。



#### 地理資料的限制



但請記住，Lightning 節點的地理位置有其限制和偏差：





- 近似 IP 位置**：1ML 通常使用節點的公開 IP 位址來估計其位置。然而，此方法可能會因為使用 VPN、雲端伺服器 (AWS、Google Cloud) 或託管在與節點擁有者實際居住地不同的國家而失真。
- 虛擬節點**：有些運營商基於可靠性和可用性的理由，將其節點託管在遠端伺服器上，這可能會造成實際位置的錯覺。
- 使用者移動性**：節點操作者可能會變更位置、移動基礎設施，或在不同區域開設數個節點，使得資料讀取變得更加複雜。
- 隱藏私人節點**：有些節點不公開其 IP 位址或使用方法隱藏其位置，使得地理定位無法進行。

這些觀察為 Lightning Network 的成長、成熟度和趨勢提供了有趣的角度。



## 1ML 最佳實作與限制



### 公開資料≠完整現實



1ML 僅基於 Lightning Network 的**公佈**資料。這意味著該工具只顯示了部分現實。許多頻道可能是私人頻道，一些節點可能未公開，而且頻道中的實際可用流動性也不可見。因此，必須將 1ML 用作全局分析工具，而非網路的詳盡表達。



### 隱私權與閃電



Lightning Network 的設計非常注重**支付隱私**。個別交易在 1ML 上不可見，確切的頻道餘額也不公開。此限制並非探索者的過失，而是 Lightning Network 的基本功能，旨在保護使用者的隱私。



### 不要急於下結論



容量大或通道多的節點不一定在所有情況下都更 「可靠 」或更 「有效率」。同樣地，整體網路容量的暫時下降也不一定代表結構性問題。對於數字的詮釋，總是應該以事後觀察的方式，並將其與實際情況相結合。



### 與其他工具的互補性



1ML 是探索 Lightning Network 的絕佳起點，但最好與 Lightning 組合、節點管理介面和其他探索器等其他工具結合使用。此方法可提供更完整、更細微的網路檢視。



## 1ML 連線選項 (進階功能)



1ML 提供**登入/建立帳戶**選項，在網站上可見，但檢視 Lightning Network 資料**不需要此選項。本教學到目前為止所討論的所有功能，**無需帳號**即可使用。



此連線主要針對**Lightning 節點操作者**。特別是，它可讓節點與 1ML 帳戶相關聯，以便管理某些公開資訊，例如節點的簡報、連絡連結及其他元資料。此功能旨在改善節點在探索器中的可見性和識別。



必須注意的是，1ML 並非**託管服務。建立帳戶並不會允許存取節點的資金、通道或付款。它只用於從聲明和資訊的角度與探索者互動。



因此，在學習或探索 Lightning Network 的情況下，此選項可視為**選項**，並保留給更進階的用途。
