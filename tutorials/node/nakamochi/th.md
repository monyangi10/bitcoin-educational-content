การใช้งาน Bitcoin และ Lightning full node ของคุณเองไม่จำเป็นต้องเป็นงานที่ซับซ้อนจำกัดเฉพาะผู้เชี่ยวชาญด้านเทคนิคอีกต่อไป โดยปกติแล้ว การตั้งค่าและการจัดการโหนดต้องการความรู้เชิงลึกเกี่ยวกับการเข้ารหัสลับ เครือข่าย และการพัฒนาซอฟต์แวร์ Nakamochi เปลี่ยนแปลงสิ่งนั้นโดยทำให้โหนดสามารถเข้าถึงได้สำหรับทุกคน ไม่ว่าจะมีพื้นฐานทางเทคนิคหรือไม่ก็ตาม


ด้วย Nakamochi ใครๆ ก็สามารถตั้งค่าและดำเนินการโหนดจากที่บ้านได้ ทำให้มีความเป็นส่วนตัวและอิสระทางการเงินอย่างเต็มที่ การใช้งาน full node ไม่เพียงแต่จะรักษาความปลอดภัยให้กับธุรกรรมของคุณเอง แต่ยังช่วยเสริมความแข็งแกร่งให้กับเครือข่าย Bitcoin อีกด้วย เครือข่าย Bitcoin ที่กระจายอำนาจและยืดหยุ่นต้องพึ่งพาการกระจายโหนดอย่างกว้างขวางเพื่อให้มั่นใจในความปลอดภัยและอิสระของมัน

## Nakamochi คืออะไรและทำงานอย่างไร?


Nakamochi เป็น Bitcoin-only full node ที่รองรับทั้งเครือข่าย Bitcoin และ Lightning รวมถึงมีการรวม Bitcoin และ Lightning wallet ทำให้ผู้ใช้สามารถรันโหนด Bitcoin ที่ปลอดภัยและมีอำนาจอธิปไตยในตัวเองได้ ในขณะที่ได้รับประโยชน์จากความเร็วและต้นทุนการทำธุรกรรมที่ต่ำของ Lightning Network


โหนด Nakamochi ของคุณถูกจัดการผ่านแอปพลิเคชันมือถือ, [BitBanana (Android)](https://bitbanana.app) และ [Zeus (iOS)](https://bitbanana.app), ช่วยให้คุณควบคุมได้อย่างสะดวกจากทุกที่ แอปเหล่านี้ทำหน้าที่เป็นรีโมทคอนโทรลสำหรับโหนดของคุณ ช่วยให้คุณชำระเงินโดยตรงด้วย Bitcoin หรือ Lightning, จัดการธุรกรรม, เปิดหรือปิดช่องทาง, ตรวจสอบยอดคงเหลือ, และติดตามประสิทธิภาพของโหนดของคุณได้อย่างง่ายดาย

## เกี่ยวกับ Lightning Network


https://planb.academy/courses/34bd43ef-6683-4a5c-b239-7cb1e40a4aeb

Bitcoin Lightning Network ปฏิวัติการทำธุรกรรม Bitcoin โดยทำให้รวดเร็วขึ้น ถูกลง และมีประสิทธิภาพมากขึ้น เหมาะสำหรับการใช้งานในชีวิตประจำวัน ช่วยให้การชำระเงินเกือบจะทันทีด้วยค่าธรรมเนียมที่น้อยที่สุด เหมาะสำหรับการทำธุรกรรมขนาดเล็ก เช่น การซื้อกาแฟหรือการจัดการการซื้อขนาดเล็กบ่อยๆ

โดยการดำเนินงาน off-chain, Lightning ถูกออกแบบมาเพื่อรองรับการขยายตัว, สนับสนุนการทำธุรกรรมหลายพันรายการต่อวินาทีโดยไม่ทำให้บล็อกเชนหลัก Bitcoin เกิดการโอเวอร์โหลด. สิ่งนี้ทำให้มันเป็นผู้เล่นสำคัญในการพัฒนา Bitcoin ให้กลายเป็นระบบการชำระเงินระดับโลกที่ใช้งานได้จริง.

ความเป็นส่วนตัวเป็นข้อได้เปรียบอีกประการหนึ่ง เนื่องจากการทำธุรกรรมบน Lightning ถูกส่งผ่านช่องทางการชำระเงินส่วนตัวแทนที่จะถูกบันทึกโดยตรงบนบล็อกเชน ซึ่งช่วยให้การทำธุรกรรมเป็นไปอย่างรอบคอบมากขึ้นในขณะที่ยังคงรักษาความปลอดภัยที่แข็งแกร่งของ Bitcoin



#### ช่องทางการชำระเงินที่อธิบายแล้ว


Lightning Network ทำงานผ่านช่องทางการชำระเงิน ซึ่งเป็นการเชื่อมต่อระหว่างสองฝ่ายที่อนุญาตให้ทำธุรกรรมหลายรายการโดยไม่ต้องโต้ตอบโดยตรงกับบล็อกเชน เมื่อช่องทางเปิดอยู่ ยอดคงเหลือระหว่างสองฝ่ายจะได้รับการอัปเดตบนโซลูชัน Lightning ชั้นที่สองสำหรับทุกธุรกรรม เพื่อให้มั่นใจว่าการชำระเงินรวดเร็วและมีต้นทุนต่ำ เฉพาะการเปิดและปิดช่องทางเท่านั้นที่ถูกบันทึก on-chain ลดความแออัดบนบล็อกเชน Bitcoin การออกแบบนี้ช่วยให้มั่นใจได้ถึงความสามารถในการขยายตัวและความเป็นส่วนตัว เนื่องจากธุรกรรมแต่ละรายการจะไม่ได้รับการบันทึกในบัญชีแยกประเภทสาธารณะ


**ตัวอย่าง:** Alice และ Bob เปิดช่องทางโดยการคอมมิต Bitcoin ไปยังช่องทางนั้น Alice ส่ง Bitcoins ไปยัง Bob และยอดคงเหลือ off-chain ของพวกเขาจะอัปเดตทันทีโดยไม่มีบันทึก on-chain หาก Alice จ่ายเงินให้กับ Charlie และ Alice ไม่มีช่องทางตรงไปยัง Charlie การชำระเงินสามารถถูกส่งผ่านช่องทางของ Bob เพื่อไปถึง Charlie การส่งผ่านโหนดตัวกลางช่วยให้มั่นใจว่าการชำระเงินสามารถทำได้แม้ไม่มีการเชื่อมต่อโดยตรง ทำให้เครือข่ายมีประสิทธิภาพสูง



## การเปิดช่องทางและการทำธุรกรรมใน Lightning Network


เมื่อคุณตั้งค่า Nakamochi และเชื่อมต่อกับแอปจัดการโหนดเรียบร้อยแล้ว คุณสามารถเริ่มใช้งาน Lightning Network ได้โดยการเปิดช่องทางและทำธุรกรรม


### การเปิดช่องบน Zeus (iOS):


1. ไปที่แท็บ **“Channels”** (เมนูด้านล่าง)

2. คลิกที่ **“+”** เพื่อเปิดช่องใหม่

3. สแกนหรือป้อน pubkey ของโหนดที่คุณต้องการเชื่อมต่อด้วย

4. ป้อนจำนวนที่ล็อกไว้ (เลือกกับเพื่อนของคุณหรือใช้จำนวนขั้นต่ำที่กำหนดไว้สำหรับโหนดที่เป็นที่รู้จักดี)

5. คลิกที่ **“Open Channel”**.

### การเปิดช่องทางบน BitBanana (Android):


1. เปิดเมนูแฮมเบอร์เกอร์ (ด้านซ้าย)

2. ไปที่ **“Channels”**.

3. คลิกที่ **“+”** เพื่อเพิ่ม/เปิดช่องใหม่

4. สแกนหรือวางคีย์สาธารณะ (pubkey)

5. ป้อนจำนวนที่ล็อกไว้ (เลือกกับเพื่อนของคุณหรือใช้จำนวนขั้นต่ำที่กำหนดไว้สำหรับโหนดที่เป็นที่รู้จัก)

เมื่อช่องของคุณเปิดแล้ว การชำระเงินสามารถส่งผ่านช่องนั้นไปยังผู้เข้าร่วมคนอื่น ๆ ในเครือข่าย ยอดคงเหลือจะปรับ off-chain เพื่อให้แน่ใจว่าการทำธุรกรรมเกือบจะทันทีและมีค่าธรรมเนียมต่ำสุด

หากคุณไม่ต้องการใช้ช่องทางอีกต่อไป คุณสามารถปิดมันได้ การกระทำนี้จะทำการชำระยอดคงเหลือสุดท้ายระหว่างคุณและคู่สัญญาของคุณและบันทึกไว้ที่ on-chain โดยทั่วไปแล้วทั้งสองฝ่ายควรเห็นด้วยและออนไลน์สำหรับการ "ปิดแบบร่วมมือ" (รวดเร็วกว่าและมีค่าธรรมเนียมน้อยกว่าการ "ปิดแบบบังคับ" กับคู่สัญญาที่ไม่ตอบสนอง/ออฟไลน์)

โดยทั่วไป เราแนะนำให้เปิดช่องทางไว้เพื่อลดค่าใช้จ่ายและเพิ่มความน่าเชื่อถือและประสิทธิภาพของเครือข่าย การเปิดช่องทางไว้จะช่วยลดค่าธรรมเนียมการทำธุรกรรม on-chain หลีกเลี่ยงการหยุดทำงานสำหรับการเชื่อมต่อช่องทางใหม่ และรักษาความสามารถในการกำหนดเส้นทางที่เสถียรสำหรับการประมวลผลการชำระเงินที่ราบรื่น วิธีการนี้ส่งเสริมเครือข่ายที่แข็งแกร่งและยืดหยุ่นในขณะที่เพิ่มประสบการณ์ผู้ใช้โดยรวมและลดค่าใช้จ่ายในการดำเนินงาน
