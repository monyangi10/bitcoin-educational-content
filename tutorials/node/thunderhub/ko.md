## 소개



ThunderHub는 라이트닝 노드(LND)**를 위한 **오픈 소스 관리자로, 모든 기기나 브라우저에서 액세스할 수 있는 직관적인 Interface를 제공합니다.



**주요 기능:**




- 모니터링**: 잔액, 채널, 트랜잭션, 라우팅 통계에 대한 글로벌 보기
- 관리**: 채널 열기/닫기, 수신/발신 결제, 채널 밸런싱
- 통합**: LNURL 지원, Boltz를 통한 스왑, Amboss 백업
- Interface 반응형**: 어두운/밝은 테마의 모바일, 태블릿 및 데스크톱 장치와 호환 가능



ThunderHub는 **엄브렐**, **볼티지**, **라스피블리츠**, **마이노드**와 쉽게 통합되어 노드의 일상적인 관리를 간소화합니다.



**특히 채널 관리, 유동성 제어(리밸런싱), 거래 모니터링, 앰보스와 같은 타사 서비스 통합을 위해 인체공학적인 Interface를 찾는 운영자에게 적합한 서비스입니다. 로컬 또는 토르 연결을 통해 보안이 보장됩니다.**



아직 라이트닝 노드가 없는 경우 LND 엄브렐 튜토리얼을 따르는 것이 좋습니다:



https://planb.academy/tutorials/node/lightning-network/umbrel-lnd-b12e0b5b-12ff-45f1-978e-62f4b4a8ba16

## 설치



ThunderHub는 라이트닝 노드 호스팅 환경에 따라 여러 가지 방법으로 설치할 수 있습니다. 턴키 솔루션(엄브렐, 전압, 라스피블리츠, 마이노드, 스타트9 등)을 사용하든 수동 설치를 사용하든, 큰 노력 없이 ThunderHub를 사용할 수 있는 경우가 많습니다. 아래에서는 Umbrel 앱 스토어를 통한 방법과 수동 설치(서버 또는 자체 호스팅 배포에 적용 가능)의 두 가지 일반적인 접근 방식을 설명합니다.



### 엄브렐을 통한 설치



Umbrel은 **앱 스토어**에 ThunderHub를 통합하여 설치가 매우 간단합니다. 명령줄이나 수동 구성이 필요 없습니다. 모든 것이 Interface Umbrel을 통해 이루어집니다. 다음 단계를 따르기만 하면 됩니다:





- 엄브렐 대시보드를 엽니다**: 엄브렐 노드의 Interface 웹에 연결합니다(예: 로컬 네트워크의 경우 `http://umbrel.local`, Tor를 사용하는 경우 `.onion` Address를 통해).
- 앱 스토어**에 액세스합니다: 엄브렐의 메인 메뉴에서 "앱 스토어"(또는 "앱"을 클릭합니다.) 사용 가능한 애플리케이션 목록에서 **ThunderHub**를 검색합니다.



![Écran d'installation de ThunderHub via Umbrel](assets/fr/01.webp)





- ThunderHub**를 설치합니다: ThunderHub 애플리케이션을 클릭한 다음 설치 버튼을 클릭합니다. 필요한 경우 확인합니다. 엄브렐이 자동으로 ThunderHub를 다운로드하여 노드에 배포합니다.





- 애플리케이션을 실행합니다**: 설치가 완료되면(수십 초 정도 소요) ThunderHub가 홈 페이지에 나타납니다. 아이콘을 클릭하여 엽니다. 브라우저에서 ThunderHub가 실행됩니다.



![Identifiants par défaut pour ThunderHub](assets/fr/02.webp)



**중요:** ThunderHub를 처음 열면 로그인에 필요한 **기본 비밀번호**가 자동으로 표시됩니다. "다시 표시하지 않음" 옵션을 사용하면 나중에 연결할 때 이 표시를 숨길 수 있습니다. **강력히 권장하는 사항:**




- 비밀번호 관리자에 이 비밀번호를 즉시 저장**하세요
- 다음 단계에서 사용하기 위해 **복사**합니다
- 비밀번호가 저장되면 '다시 표시하지 않음'에 체크합니다



![Page de connexion de ThunderHub](assets/fr/03.webp)



그러면 로그인 페이지로 이동하여 이전 단계에서 복사한 비밀번호를 입력해야 Interface의 잠금을 해제할 수 있습니다.



![Interface de connexion complète à ThunderHub](assets/fr/04.webp)



Umbrel은 백그라운드에서 LND 연결 정보(TLS 인증서, 관리 마카롱 등)를 ThunderHub에 제공하므로 추가 구성이 필요하지 않습니다. 단 몇 번의 클릭만으로 Umbrel 노드에서 ThunderHub를 실행할 수 있습니다.



### 수동 설치(자체 호스팅, 엄브렐 제외)



엄브렐 외부의 사용자(예: 개인 서버, 라즈베리파이에 라즈비블리츠 또는 *독립형* 설치)의 경우, ThunderHub를 설치하려면 몇 가지 추가 단계가 필요합니다. 아래에서는 [공식 ThunderHub 문서](https://docs.thunderhub.io)에 따라 소스에서 설치하는 방법과 구성에 대해 설명합니다.



#### 설치



**전제 조건: ** 시스템이 [문서 설정](https://docs.thunderhub.io/setup)에 따른 최소 요구 사항을 충족하는지 확인하세요:




- Node.js** 버전 18 이상
- npm** 설치
- LND 인증 파일에 액세스 :
  - LND TLS 인증서(`tls.cert`)
  - LND 관리 마카롱(`admin.macaroon`)
  - LND gRPC 서비스 Address(호스트명:포트)(기본값 `127.0.0.1:10009` 로컬)



**1. ThunderHub 코드 가져오기: ** [설치 설명서](https://docs.thunderhub.io/installation)에 설명된 대로 프로젝트의 GitHub 리포지토리를 복제합니다:



```bash
git clone https://github.com/apotdevin/thunderhub.git
cd thunderhub
```



**2. 종속성 설치 및 애플리케이션 빌드:**



```bash
npm install
npm run build
```



이 명령은 필요한 모든 모듈을 설치한 다음 애플리케이션을 컴파일합니다(ThunderHub는 TypeScript/React로 작성됨).



**3. 추후 업데이트:** ThunderHub는 향후 업데이트를 위한 여러 가지 방법을 제공합니다:



```bash
# Méthode rapide
npm run update

# Ou méthode manuelle
git pull
npm install
npm run build
```



#### 구성(설정)



**1. 기본 구성 파일: ThunderHub 폴더의 루트에 '.env.local' 파일을 생성하여 구성을 사용자 지정합니다(이렇게 하면 업데이트 중에 설정을 덮어쓰는 것을 방지할 수 있습니다). 주요 변수는 [설정 문서](https://docs.thunderhub.io/setup)에 따릅니다:



```bash
# -----------
# Server Configs
# -----------
LOG_LEVEL='info' # 'error' | 'warn' | 'info' | 'http' | 'verbose' | 'debug' | 'silly'
PORT=3000
NODE_ENV=production

# -----------
# Interface Configs
# -----------
THEME='dark' # 'dark' | 'light' | 'night'
CURRENCY='sat' # 'sat' | 'btc' | 'fiat'

# -----------
# Privacy Configs
# -----------
FETCH_PRICES=true # Récupération des prix BTC/fiat depuis Blockchain.com
FETCH_FEES=true # Récupération des frais on-chain depuis Earn.com
DISABLE_LINKS=false # Liens vers 1ml.com et Blockchain.com
NO_VERSION_CHECK=false # Vérification de version depuis GitHub

# -----------
# TOR (optionnel)
# -----------
TOR_PROXY_SERVER='socks://127.0.0.1:9050' # Pour proxifier via TOR

# -----------
# Account Configs
# -----------
ACCOUNT_CONFIG_PATH='/chemin/vers/thubConfig.yaml' # Fichier de comptes
```



**2. 서버 계정 구성:** `ACCOUNT_CONFIG_PATH`에 지정된 YAML 파일을 생성합니다:



```yaml
masterPassword: 'votre_mot_de_passe_principal'
accounts:
- name: 'Mon Nœud LND'
serverUrl: '127.0.0.1:10009'
macaroonPath: '/home/user/.lnd/data/chain/bitcoin/mainnet/admin.macaroon'
certificatePath: '/home/user/.lnd/tls.cert'
password: 'mot_de_passe_compte_specifique'
# Optionnel : compte avec macaroon en hexadécimal
- name: 'Nœud Distant'
serverUrl: 'ip.distante:10009'
macaroon: '0201056c6e6402f8...' # Macaroon en HEX ou Base64
certificate: '0202045c7365...' # Certificat en HEX ou Base64
```



**3. 원격 액세스:** 원격 LND 노드에 연결하려면 `LND.conf`에 추가합니다:



```bash
# Option 1 : accès par IP
tlsextraip=<ip-externe-accessible>
rpclisten=0.0.0.0:10009

# Option 2 : accès par domaine
tlsextradomain=<domaine-externe-accessible>
rpclisten=0.0.0.0:10009
```



**4. 보안:** 처음 시작할 때 ThunderHub는 서버에 암호가 일반 텍스트로 표시되지 않도록 '마스터 비밀번호'와 모든 계정 비밀번호를 YAML 파일에 **자동으로** 숨깁니다.



**5. ThunderHub 시작:**



```bash
npm start
```



기본적으로 서버는 포트 3000에서 수신 대기합니다. 로컬 네트워크에서 `http://localhost:3000`(또는 `http://ip-serveur:3000`)에 액세스합니다.



**6. 도커 대안:** ThunderHub는 공식 도커 이미지를 제공합니다:



```bash
# Image standard
docker pull apotdevin/thunderhub:latest
docker run --rm -it -p 3000:3000/tcp apotdevin/thunderhub:latest

# Image avec base path /thub
docker pull apotdevin/thunderhub:base-v0.11.1
```



ThunderHub 로그인 페이지가 나타납니다. 구성된 계정을 선택하고 비밀번호를 입력하여 대시보드에 액세스합니다.



**다른 배포판에 설치: ** 사전 패키지된 노드 배포판(RaspiBlitz, MyNode, Start9 등)은 일반적으로 해당 관리 인터페이스를 통해 기본 ThunderHub 지원을 제공합니다.



**자세한 내용은 공식 문서 전문을 참조하세요:




- 설치:** [docs.thunderhub.io/installation](https://docs.thunderhub.io/installation)
- 구성:** [docs.thunderhub.io/setup](https://docs.thunderhub.io/setup)



이 리소스에서는 SSO 계정, 암호화된 마카롱, TOR 구성 등과 같은 고급 옵션에 대해 자세히 설명합니다.



ThunderHub를 설치하고 액세스할 수 있게 되면 모든 기능을 활용할 준비가 된 것입니다. 다음 섹션에서는 Interface ThunderHub와 다양한 탭을 살펴보고 사용 방법을 안내해 드리겠습니다.



## Interface 프레젠테이션



Interface ThunderHub는 몇 가지 주요 섹션으로 구성된 메인 메뉴(일반적으로 왼쪽 열에 표시됨)를 중심으로 구성되어 있습니다. 각 섹션은 라이트닝 노드 관리의 한 측면에 해당합니다. 하나씩 살펴보겠습니다:





- 홈** - 일반 대시보드(노드 개요 및 빠른 작업)가 있는 홈 탭입니다.
- 대시보드** - 위젯과 고급 메트릭이 포함된 사용자 지정 가능한 대시보드입니다.
- 피어** - 라이트닝 피어 관리(다른 노드에 대한 연결).
- 채널** - 라이트닝 채널의 세부 관리.
- 리밸런싱** - 채널 밸런싱 도구(순환 결제).
- 거래** - 라이트닝 결제 내역(LN 거래).
- 포워드** - 라우팅 통계(노드에서 중계한 결제).
- 체인** - 노드의 On-Chain Wallet(On-Chain BTC: UTXO, 트랜잭션).
- Amboss** - Amboss와 통합(노드 모니터링, 백업 등).
- 도구** - 기타 도구(백업, 서명된 메시지, 마카롱, 보고서 등).
- 스왑** - Boltz를 통한 On-Chain/Lightning 스왑 기능.
- 통계** - 고급 통계 및 노드 성능 메트릭.



*(참고: ThunderHub 버전에 따라 일부 섹션의 제목이나 추가 기능이 약간 다를 수 있지만 일반적인 원칙은 동일하게 유지됩니다)*



### 홈(일반 제어판)



ThunderHub의 **홈** 탭은 로그인 후 표시되는 홈 페이지입니다. 여기에는 **일반 대시보드**가 포함되어 있으며, 여기에서는 라이트닝 노드 상태에 대한 **전체 개요**를 제공하고 일상적인 작업을 위한 **빠른 작업**을 제안합니다. 일반적으로 다음과 같은 내용을 확인할 수 있습니다:



![Tableau de bord principal de ThunderHub](assets/fr/05.webp)





- 잔액 및 용량: ** 페이지 상단에 ThunderHub는 사용 가능한 잔액을 표시합니다. 여기에는 일반적으로 On-Chain 잔액(노드의 Bitcoin에 있는 On-Chain, Anchor ⚓로 표시)과 라이트닝 잔액(채널의 용량, 라이트닝 Bolt ⚡로 표시)을 볼 수 있습니다. 이를 통해 On-Chain와 라이트닝에 남아 있는 자금을 즉시 파악할 수 있습니다. 계정 또는 채널이 여러 개 있는 경우 올바른 계정 또는 채널을 사용하고 있는지 확인하세요(예: Mainnet 대 Testnet).





- 주요 통계: ** 대시보드에는 노드에 대한 몇 가지 글로벌 지표(예: 열린 채널 수, 연결된 피어 수, 라우팅 수수료 수입(해당되는 경우) 등)가 표시됩니다. 이는 노드의 최근 활동과 상태를 요약한 것입니다.





- 빠른 작업: ** 대시보드에는 메뉴를 탐색할 필요 없이 가장 일반적인 작업을 빠르게 실행할 수 있는 버튼이 있습니다. 이러한 빠른 작업에는 다음이 포함됩니다:





- Ghost**: 앰보스를 통해 커스텀 라이트닝 Address를 설정했습니다.
- 기부**: Lightning을 통해 기부하기.
- 로그인/이동**: 앰보스 계정에 연결(빠른 연결)하고 Amboss.space로 바로 이동하여 노드 정보를 확인합니다.
- Address**: 결제하려면 라이트닝 Address을 입력하세요.
- 열기**: 새 라이트닝 채널을 엽니다. 클릭하면 채널을 열 원격 노드의 URI, 금액 및 해당되는 경우 사용할 최대 On-Chain 요금을 입력할 수 있는 양식이 열립니다.
- 디코딩**: 결제하기 전에 세부 정보를 보려면 Lightning Invoice 또는 LNURL을 디코딩하세요.
- LNURL**: 라이트닝 결제 또는 인출을 위한 LNURL을 처리합니다.
- LnMarkets 로그인**: 거래를 위해 LnMarkets에 로그인합니다.



이러한 빠른 작업을 통해 Interface의 다양한 탭을 탐색할 필요 없이 홈 페이지에서 바로 가장 자주 사용하는 작업을 수행할 수 있습니다.



간단히 말해, ThunderHub 대시보드를 통해 노드를 **빠르게** 살펴볼 수 있으며, 한 번의 클릭으로 가장 자주 사용하는 작업(송수신, 채널 열기)을 수행할 수 있습니다. 일상적인 관리를 위한 이상적인 시작점입니다.



### 대시보드



대시보드** 섹션은 홈 탭과 분리되어 있으며 보다 고급의 사용자 지정 가능한 대시보드를 제공합니다. 이 섹션에서는 노드 운영자의 필요에 따라 특정 위젯을 사용하여 사용자 지정 보기를 만들 수 있습니다.





- 사용자 지정 가능한 위젯: ** 고정된 레이아웃이 있는 홈 페이지와 달리 대시보드에서는 표시할 Elements과 구성 방법을 정확하게 선택할 수 있습니다.



![Dashboard sans widgets activés](assets/fr/06.webp)



활성화된 위젯이 없는 경우 사용자 지정 매개변수에 액세스할 수 있는 '설정' 버튼과 함께 "활성화된 위젯이 없습니다!"라는 메시지가 표시됩니다.



![Paramètres des widgets du Dashboard](assets/fr/07.webp)



설정에서 카테고리별로 구성된 다양한 위젯 중에서 선택할 수 있습니다: "라이트닝 - 정보", "라이트닝 - 표", "라이트닝 - 그래프" 등입니다. 각 위젯은 '표시/숨기기' 버튼으로 개별적으로 활성화 또는 비활성화할 수 있습니다.



![Bas de la page des paramètres](assets/fr/08.webp)



설정 하단에는 대시보드로 돌아가는 '대시보드로 이동' 버튼과 기본 표시를 재설정하는 '위젯 재설정' 버튼이 있습니다.



![Dashboard personnalisé avec widgets](assets/fr/09.webp)



구성이 완료되면 대시보드에 다양한 그래프와 메트릭을 표시할 수 있습니다: 라이트닝 결제 그래프, 발행된 송장 수, 전달 통계, 상세 잔액 등 다양한 그래프와 지표를 표시할 수 있습니다.





- 고급 지표: ** 그래프와 실시간 데이터를 통해 노드 성능에 대한 자세한 통계에 액세스하세요.





- 구성 가능한 개요:** 일반 사용자 또는 여러 라우팅 채널을 관리하는 전문 운영자 등 사용자에 맞게 디스플레이를 맞춤 설정할 수 있습니다.





- 모듈형 Interface:** 필요에 따라 위젯을 추가하거나 제거합니다: 포워드 차트, 유동성 지표, 노드 상태 알림 등.



이 섹션은 특정 메트릭을 모니터링하거나 라이트닝 노드에 대한 자세한 기술 개요를 확인하려는 고급 사용자에게 특히 유용합니다. 홈 섹션을 보완하여 정보 표시 방식을 더욱 유연하게 제어할 수 있습니다.



### 동료



피어** 섹션에는 현재 내 라이트닝 노드에 피어로 연결된 모든 라이트닝 노드가 나열됩니다. 피어**는 Lightning Network에서 노드 간 직접 연결입니다. 노드는 오픈 채널 없이도 피어에 연결될 수 있으며(예: 네트워크의 Exchange 가십 정보에만 연결), 물론 모든 오픈 채널은 연결된 피어를 자동으로 의미합니다.



![Vue de l'onglet Peers avec les pairs connectés](assets/fr/10.webp)



피어 탭에 표시됩니다:





- 정보 열:** Interface에는 동기화 상태, 연결 유형(클리어넷 또는 Tor), 핑, 사토시 수신/발신 및 교환된 데이터 양과 같은 유용한 세부 정보가 표시됩니다.
- 피어를 추가합니다: ThunderHub에서는 오른쪽 상단의 **"추가"** 버튼을 통해 새 피어를 수동으로 연결할 수 있습니다. 노드의 URI를 입력해야 합니다(형식은 `<public_key>@<소켓>`). 유효성이 확인되면 ThunderHub는 해당 `lncli connect` 명령을 전송합니다. 노드가 온라인 상태이고 액세스할 수 있으면 피어 목록에 추가됩니다.



### 채널



채널** 탭은 라이트닝 채널 관리의 핵심입니다. 아마도 가장 자주 참조하게 될 섹션일 것입니다. 여기에는 **모든 라이트닝 채널**의 세부 정보가 표시되며, 이러한 채널에 대한 관리 작업을 수행할 수 있습니다.



![Vue d'ensemble des channels ouverts](assets/fr/11.webp)



채널 페이지에서 확인할 수 있는 내용은 다음과 같습니다:





- 채널 목록 보기: ** 열려 있는(또는 열림/닫힘) 각 채널이 나열되며, 일반적으로 원격 노드의 별칭, 총 채널 용량, 로컬 대 원격 유동성의 분포를 나타내는 색상 막대가 함께 표시됩니다. ThunderHub는 색상 코드(보통 파란색/Green) 또는 백분율로 채널 잔액을 표시합니다(예: 로컬 점유율은 파란색, 원격 점유율은 Green). 채널이 완벽하게 균형 잡힌 경우(50/50) 막대는 각 색상의 절반이 됩니다. 이렇게 하면 어떤 채널이 불균형 상태인지 한눈에 파악할 수 있습니다(파란색 모두 = 거의 모든 로컬, Green 모두 = 거의 모든 원격).





- 정보 열:** Interface은 상태, 사용 가능한 작업, 피어 정보, 채널 ID, 용량, 활동, 수수료 및 잔액 등의 세부 열을 그래픽 유동성 표시와 함께 표시합니다.





- 디스플레이 구성: ** 오른쪽 상단의 톱니바퀴를 이용해 원하는 대로 채널 디스플레이를 맞춤 설정할 수 있습니다.





- 상태:** '활성'(채널이 열려 있고 작동 중), '오프라인'(피어 연결이 끊어져 일시적으로 채널을 사용할 수 없음), '보류 중'(On-Chain 확인을 기다리는 개방 또는 폐쇄의 경우) 등의 상태 표시도 볼 수 있습니다.





- 채널에 대한 작업: ** 각 채널에 대해 ThunderHub는 작업 버튼(주로 아이콘 형태)을 제공합니다:



![Actions de gestion des canaux - Modifier et Fermer](assets/fr/12.webp)





- 수수료 수정: ** Interface "채널 정책 업데이트"를 통해 모든 채널 매개변수를 조정할 수 있습니다: 기본 수수료, 수수료율(ppm), CLTV 델타, 최대 HTLC 및 최소 HTLC. 이를 통해 라우팅 트래픽을 유치(또는 억제)하기 위해 채널별로 수수료 정책을 개별적으로 조정할 수 있습니다. *(참고: ThunderHub는 자동 수수료 관리 도구를 대체할 수는 없지만 수동 조정에는 매우 효과적입니다)*
- 채널 닫기(**닫기**): Interface "채널 닫기"는 (Sats/vByte 단위로) 수수료를 정의하여 **협동 닫기**(기본값) 또는 **강제 닫기**(**강제 닫기**) 중에서 선택할 수 있습니다. **중요:** On-Chain 정산 지연과 높은 수수료를 피하려면 가능하면 항상 협력 청산을 선호하세요. ThunderHub는 피어가 온라인 상태인지(협력 가능) 여부를 알려줍니다. 강제 종료의 경우, 이는 되돌릴 수 없으며 타임락(일반적으로 144블록 또는 Bitcoin Mainnet의 경우 ~1일)으로 전면 거래를 트리거하므로 반드시 확인하시기 바랍니다.
- 새 채널 열기: ** 새 채널을 열려면 채널 페이지의 오른쪽 상단에 있는 톱니바퀴를 클릭한 다음 '열기'를 선택합니다. 그러면 신규 또는 기존 피어에게 채널을 시작할 수 있습니다. 이 페이지를 사용하면 기존 채널 목록이 바로 앞에 표시되므로 새 채널을 개설할 위치를 결정하는 데 도움이 된다는 장점이 있습니다.



간단히 말해, 채널 섹션에서는 각 채널을 세밀하게 제어할 수 있습니다. 여기에서 유동성 할당을 추진하고, 유지 또는 폐쇄할 채널을 결정하고, 채널별 라우팅 매개변수를 설정할 수 있습니다. ThunderHub는 이러한 중요한 노드 관리 작업을 위한 명확한 Interface를 제공합니다.



### 재조정



리밸런싱** 탭은 **채널 밸런싱** 전용 탭입니다. 밸런싱(또는 *재조정*)은 Lightning Network을 통해 채널 중 하나에서 다른 채널로 **순환 결제**를 하여 나가는 채널과 들어오는 채널 간의 자금 분배를 재조정하는 것을 포함합니다. 이를 통해 새로운 자금을 유입하지 않고도 너무 꽉 찬 채널에서 너무 비어 있는 채널로 유동성을 이동하여 채널을 더 유용하게 만들 수 있습니다(균형 잡힌 채널은 송금과 수신이 모두 가능합니다).



![Interface de rebalance des canaux](assets/fr/13.webp)



명령줄에서는 지루할 수 있는 이 작업을 ThunderHub를 사용하면 매우 쉽게 수행할 수 있습니다. 재조정 섹션을 사용하는 방법은 다음과 같습니다:





- 초기 채널 보기: ** 리밸런스에 들어가면 ThunderHub는 채널 목록과 각 채널에 대한 밸런스 표시기를 표시합니다(채널 페이지의 표시기와 유사). 어떤 채널이 균형이 맞지 않는지 바로 확인할 수 있습니다. ThunderHub는 밸런스가 높은 순서대로 채널을 정렬할 수 있으므로 가장 밸런스가 맞지 않는 채널이 목록의 맨 위에 표시됩니다(0.0은 완전히 로컬 또는 원격임을 의미).





- 피어 선택:** Interface를 사용하면 리밸런싱을 위해 발신 및 수신 피어를 쉽게 선택할 수 있습니다.





- 매개변수 설정:** 설정할 수 있습니다:
  - 지불할 의향이 있는 **최대 수수료**(Sats 및 ppm 단위)
  - "고정" 또는 "목표" 옵션으로 **재조정할 금액**을 선택합니다
- 라우팅 시 피해야 할 노드**
- 경로 탐색을 위한 최대 체험 시간**





- 소스** 채널을 선택합니다: 먼저 **발신(소스)** 채널, 즉 로컬 유동성이 너무 많아 이동하기 어려운 채널을 선택합니다. 실제로는 로컬 점유율이 높은 채널(50% 이상)을 의미합니다. 1,000,000개의 Sats가 있고 그 중 90만 개가 로컬인 A 채널이 다른 곳으로 Sats를 전송하기에 좋은 채널이라고 가정해 보겠습니다. 이 A 채널을 "발신"으로 클릭하면 ThunderHub에서 해당 채널을 소스로 표시합니다.





- 타겟 채널**을 선택합니다: 다음으로 유동성을 공급받아야 하는 **유입(타겟)** 채널을 선택합니다. 일반적으로 이 채널은 대부분의 자금이 먼 쪽에 있는 채널이 될 것입니다(예: 1,000,000개 중 100,000개의 로컬 Sats만). 소스 채널을 선택하면 ThunderHub는 다른 채널을 역순으로 정렬(잔액 감소)하여 가장 상호 보완적인 채널을 식별하는 데 도움을 줍니다. 로컬 쪽에 여유가 있는 B 채널을 선택합니다. 그러면 ThunderHub는 선택한 두 채널(소스 A와 타겟 B)을 명확하게 표시합니다.





- 수수료 금액 및 허용 한도 설정: ** 양식을 통해 입력할 수 있습니다:





  - 재조정할 **금액**(Sats 단위)입니다. 보통 두 채널의 밸런스를 \~50/50으로 맞추는 것과 같은 양을 선택합니다. 예를 들어 ThunderHub는 소스 채널의 초과 용량의 절반을 미리 채울 수 있습니다.
  - 이 작업에 대해 지불할 의향이 있는 **최대 수수료**(선택 사항)입니다. 이 수수료는 Sats(순환 라우팅의 총 비용)로 표시됩니다. 이 값을 비워두면 ThunderHub는 비용에 관계없이 경로를 검색하므로 일반적으로 바람직하지 않습니다(예: 작은 리밸런싱의 경우 10 Sats 또는 최대 ppm과 같이 한도를 설정하는 것이 좋습니다).





- 경로 찾기: ** 버튼을 클릭하면 경로를 찾을 수 있습니다. ThunderHub는 LND를 쿼리하여 소스 채널에서 네트워크를 통해 자신의 타겟 채널로 가는 경로를 계산합니다. 수수료 기준을 충족하는 가능한 경로를 찾으면 홉과 수수료 비용에 대한 세부 정보와 함께 경로를 표시합니다. 예를 들어, 총 2Sats의 요금이 부과되는 3홉 경로를 찾았다고 표시할 수 있습니다.





- 재조정 시작: 제안된 경로가 마음에 들면 **재조정 채널**을 클릭합니다. 그러면 썬더허브가 LND을 통해 순환 결제를 시작합니다. 결제가 성공하면 성공 알림이 표시되고 채널 A와 B의 잔액이 실시간으로 수정됩니다. ThunderHub는 이 채널의 잔액 표시기를 업데이트합니다(이상적으로는 이전보다 녹색으로 표시되어 더 나은 잔액을 나타냅니다).





- 조정 및 반복: ** 첫 번째 시도에서 경로를 찾을 수 없거나 너무 비싼 경우 매개변수를 조정할 수 있습니다:





  - 적은 양으로 시도해 보세요(때로는 많은 양이 실패하는 동안 부분적으로 재조정되는 경우도 있습니다).
  - 최대 수수료를 점진적으로 인상하되, 적정 수준보다 많은 수수료를 지불하지 않도록 주의하세요.
  - 가능한 경우 **다른 경로 찾기** 버튼을 사용하여 다른 경로를 시도하세요.
  - 문제가 해결되지 않으면 다른 채널을 사용해 보세요.



ThunderHub는 프로세스를 매우 **직관적이고 시각적으로** 만듭니다. 복잡한 수동 명령이 필요했던 작업을 단 4단계(발신 채널 선택, 수신 채널 선택, 금액, 유효성 검사)만으로 수행할 수 있습니다. 이 도구는 균형 잡힌 노드를 유지하고 라우팅 성능과 경험을 개선하는 데 매우 유용합니다(모든 채널에서 결제를 더 쉽게 보내고 받을 수 있음).



마지막으로, 리밸런싱은 라우팅 비용(중간 노드에 지불)을 소모하므로 노드를 더 유동적으로 만들기 위한 **투자**라는 점에 유의하세요. 예를 들어 자주 사용하는 서비스에 대한 채널을 지원하거나(인바운드 유동성) 대규모 라우팅 채널의 균형을 맞추는 등 현명하게 사용하세요. 썬더허브를 사용하면 이를 **간단하고 효율적으로** 수행할 수 있습니다.



### 거래



썬더허브의 **거래** 섹션은 노드의 **라이트닝** 거래 내역, 즉 채널을 통해 지불하거나 받은 결제 및 송장에 해당합니다. 이는 LN 운영에 대한 일종의 계정 명세서입니다.



![Historique des transactions Lightning](assets/fr/14.webp)



이 탭에서 찾을 수 있는 항목은 :





- Invoice 그래프:** 오른쪽 상단에는 시간 경과에 따른 송장 수신 추이를 그래프로 표시하여 노드의 활동을 시각화할 수 있습니다.





- 내 노드에서 *발신* 또는 *수신*한 모든 라이트닝 트랜잭션의 시간순 목록입니다. 각 항목은 을 표시할 수 있습니다:





  - 작업 유형: **송금**(발신 결제) 또는 **수령**(인바운드, 유료 Invoice을 통한 결제).
  - Sats 단위의 금액입니다.
  - 날짜/시간.
  - 결제 ID(Hash 또는 RHash 사전 이미지) 또는 댓글(Invoice에 메모를 추가한 경우).
  - 상태: **완료** 또는 **진행 중**/*실패*(예: 결제가 해결되기를 기다리는 중이지만 일반적으로 LND은 이를 빠르게 처리하므로 On-Chain 거래에 비해 '보류 중'이라는 표시가 거의 없습니다).



간단히 말해, 거래 섹션은 **LN 활동 로그** 역할을 합니다. 결제가 진행되었는지, 얼마나 많은 수수료가 발생했는지 확인하거나 라이트닝 거래 내역을 추적하는 데 매우 유용합니다. 포워드 섹션(다음에 설명)과 함께 사용하면 노드를 통과한 자금을 전체적으로 확인할 수 있습니다.



### 포워드



전달** 탭은 노드의 **라우팅** 활동, 즉 채널을 통해 **전달**되는 결제(Lightning Network에서 중개 노드 역할을 하는 경우)에 대한 전용 탭입니다. 노드를 라우팅 노드로 운영하는 경우 이 탭은 성과를 추적하는 데 중요한 섹션입니다.



![Statistiques de routage Lightning](assets/fr/15.webp)



포워드에서 ThunderHub는 :





- 필터 및 표시 옵션: 오른쪽 상단의 필터를 사용하면 일/주/월/연도별로 데이터를 정렬하고 그래픽 또는 표 형식의 표시 중에서 선택할 수 있습니다.





- 활동 메시지: ** 선택한 기간 동안 라우팅이 수행되지 않은 경우 이 예와 같이 Interface에 "이 기간 동안 포워딩 없음"이 표시됩니다.





- 최근 전달 표**: 각 항목은 노드를 통해 **전달**된 결제에 해당합니다. 각 전달에 대해 일반적으로 :





  - Timestamp,
  - 라우팅된 금액(Sats 기준),
  - 이 전달에 대한 **수수료**(Sats에서는 수신 채널에서 받은 금액과 발신 채널에서 보낸 금액의 차액)를 의미합니다,
  - 사용된 수신 및 발신 채널(종종 피어 별칭 또는 채널 ID로 식별됨).
  - 상태(일반적으로 *완료*, 전달이 도중에 실패한 경우 실패)를 반환합니다.





- 집계된 통계**: ThunderHub는 주어진 기간(예: 지난 24시간 또는 7일 등, 때로는 구성 가능) 동안의 총합과 통계를 계산하여 페이지 상단에 표시합니다.



간단히 말해, 포워드 섹션은 라이트닝 노드의 라우팅 활동에 대한 **실시간 개요**를 제공합니다. 채널 및 리밸런싱 섹션과 함께 노드 최적화를 위한 완벽한 패키지를 구성합니다: 유동성을 위한 채널/리밸런싱, 결과(흐름과 수익) 관찰을 위한 포워드.



### 체인



체인** 섹션은 On-Chain Wallet 노드의 LND 관리에 해당합니다. 이 Interface에서는 채널을 열거나 닫힌 채널에서 자금을 받는 데 사용되는 Bitcoin 자금을 보고 관리할 수 있습니다.



![Gestion du portefeuille on-chain](assets/fr/16.webp)



체인에서는 을 찾을 수 있습니다:





- 잔액 On-Chain:** Wallet LND에서 사용 가능한 총 BTC 잔액을 표시합니다.





- 미사용 출력 목록:** 각 출력의 금액, 확인, Address 및 형식과 함께 미사용 출력(UTXO)을 모두 확인합니다.





- 거래 내역: ** 유형(입/출금), 날짜, 금액, 요금, 확인, 포함 블록, 주소 및 txid이 포함된 모든 Bitcoin 거래의 상세 표입니다.



### Amboss



썬더허브는 라이트닝 노드에 대한 자세한 정보, 유동성 마켓플레이스, 암호화된 채널 백업 및 가용성 모니터링과 같은 유용한 기능을 제공하는 **Amboss** 플랫폼(amboss.space)과 통합되어 있습니다.



![Intégration Amboss avec ses options](assets/fr/17.webp)



썬더허브의 앰보스 섹션에서는 노드를 앰보스 계정에 **연결**할 수 있습니다:





- 고스트 Address:** 노드에 **개인화된 라이트닝 Address**을 설정하여 수신 결제를 용이하게 합니다.





- 자동 채널 백업: **암보스의 암호화된 채널 백업**(SCB 파일)을 위한 플래그십 기능입니다. 설정에서 **암보스 자동 백업 = 예**를 활성화하면 채널을 변경할 때마다 암호화된 백업 업데이트를 자동으로 전송합니다. 장애가 발생하면 이 외부 백업 덕분에 자금을 복구할 수 있습니다.





- 상태 확인:** **암보스 상태 확인 = 예**를 활성화하여 노드가 암보스에 정기적으로 핑을 보내도록 합니다. 노드가 오프라인으로 표시되면 알림을 받게 됩니다.





- 기타 기능: **자동 잔액 푸시, **마그마/하이드로** 통합(유동성 마켓플레이스), 상세 실적 통계 액세스.



Amboss 통합은 자동 외부 백업 및 가용성 모니터링 기능을 갖춘 필수 **보안 Layer**을 추가하며, ThunderHub에서 직접 액세스할 수 있습니다.



### 도구



도구** 섹션에는 노드 관리를 위한 다양한 고급 도구가 모여 있습니다. 주요 Elements는 다음과 같습니다:



![Interface des outils disponibles](assets/fr/18.webp)





- 백업:** 채널 백업(SCB)을 수동으로 관리합니다. ThunderHub를 사용하면 채널의 전체 백업 파일을 **다운로드**할 수 있습니다("모든 채널 백업 -> 다운로드" 옵션). 이 `channel-all.bak` 파일을 안전한 곳에 보관하세요. 이 파일은 충돌이 발생했을 때 자금을 복구하는 데 필수적입니다. 노드를 재배포할 때 백업 파일을 **임포트**할 수도 있습니다.





- 회계: ** 특정 기간 동안 벌어들인/지불한 수수료와 라우팅된 볼륨을 포함한 재무 보고서를 내보내기 위한 도구입니다.
- 서명된 메시지: ** 암호화 서명을 통해 라이트닝 노드의 Ownership을 증명하기 위해 노드로 메시지에 서명하거나 확인합니다.
- 마카롱(베이커리 섹션): ** LND** 마카롱을 관리하여 맞춤형 액세스를 생성합니다. Interface "베이커리"를 사용하면 각 권한을 정확하게 선택할 수 있습니다: "피어 추가 또는 제거", "체인 주소 생성", "송장 생성", "마카롱 생성", "키 파생", "액세스 키 가져오기", "체인 트랜잭션 가져오기", "송장 가져오기", "Wallet 정보 가져오기", "결제 받기", "피어 받기", "송장 지불", "액세스 ID 취소", "체인 주소로 보내기", "바이트 서명", "메시지 서명", "daemon 중지", "바이트 서명 확인", "메시지 확인" 등입니다. 각 권한은 '예/아니요' 버튼으로 개별적으로 활성화하여 맞춤형 마카롱을 만들 수 있습니다.
- 시스템 정보: ** Wallet 버전 및 활성화된 RPC 표시.



간단히 말해, 도구 섹션은 백업, 계정, 보안 및 액세스 관리와 같은 고급 관리 기능을 통합된 Interface에 통합합니다.



### 스왑



ThunderHub의 **스왑** 탭을 사용하면 Boltz 서비스를 통해 라이트닝 사토시를 Bitcoin On-Chain로 교환할 수 있습니다. 이 기능은 채널을 닫지 않고 초과 라이트닝 유동성을 채널에 '덤핑'하는 데 유용합니다.



![Interface de swap via Boltz](assets/fr/19.webp)



절차는 간단합니다:





- 금액**: 교환할 금액을 정의합니다
- Address**: Bitcoin 수신 Address 입력
- 실행**: ThunderHub가 Boltz와 통신하여 Exchange을 자동으로 처리합니다



**혜택:**




- 비수탁 서비스(현금 보관 없음)
- 기존 채널 유지
- 사용하기 쉬운 통합 Interface



Boltz는 소액의 수수료를 부과하고 귀하는 표준 Bitcoin 거래 수수료를 지불합니다. ThunderHub는 확인 전에 모든 비용을 표시합니다.



### 통계



ThunderHub의 **통계** 섹션에서는 성능을 분석하고 라우팅을 최적화할 수 있도록 라이트닝 노드에 대한 **고급 통계**를 제공합니다.



![Statistiques du nœud via Amboss](assets/fr/20.webp)



이 섹션은 비용을 최적화하고 성공적인 채널을 파악하며 노드 확장을 계획하는 데 필수적입니다.



## 결론



**ThunderHub**는 라이트닝 **LND** 노드를 쉽게 관리할 수 있는 필수 도구로 자리매김했습니다. 이 최신 Interface는 채널 관리, 결제, 모니터링 등 모든 필수 기능을 제공하며, 자동 리밸런싱 및 Amboss 통합과 같은 고급 기능도 제공합니다.



**주요 혜택:**




- 세련되고 직관적인 Interface
- 강력한 도구(리밸런싱, 볼츠 스왑, 자동 백업)
- 엄브렐, 전압, 라스피블리츠 및 기타 배포판과 호환 가능



ThunderHub는 고급 라이트닝 노드 관리를 대중화하여 이전에는 복잡한 기술 명령이 필요했던 기능을 쉽게 이용할 수 있게 해줍니다. 초보자이든 숙련된 운영자이든, ThunderHub를 사용하면 최신의 포괄적인 Interface을 통해 Lightning 노드를 효율적으로 관리할 수 있습니다.
