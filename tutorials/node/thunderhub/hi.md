## परिचय



थंडरहब लाइटनिंग नोड्स (LND) के लिए एक ओपन-सोर्स प्रबंधक है, जो किसी भी डिवाइस या ब्राउज़र से सुलभ एक सहज Interface प्रदान करता है।



**मुख्य विशेषताएं:**




- **निगरानी**: शेष राशि, चैनल, लेनदेन, रूटिंग सांख्यिकी का वैश्विक दृश्य
- **प्रबंधन**: चैनल खोलना/बंद करना, आने/जाने वाले भुगतान, चैनल संतुलन
- **एकीकरण**: LNURL समर्थन, बोल्ट्ज़ के माध्यम से स्वैप, एम्बॉस बैकअप
- **Interface रेस्पॉन्सिव**: डार्क/लाइट थीम वाले मोबाइल, टैबलेट और डेस्कटॉप डिवाइस के साथ संगत



थंडरहब आसानी से **अम्ब्रेल**, **वोल्टेज**, **रास्पीब्लिट्ज** और **माईनोड** के साथ एकीकृत हो जाता है, जिससे आपके नोड का दिन-प्रतिदिन का प्रबंधन सरल हो जाता है।



**थंडरहब खास तौर पर उन ऑपरेटरों के लिए उपयुक्त है जो अपने चैनलों को प्रबंधित करने, लिक्विडिटी (पुनर्संतुलन) को नियंत्रित करने, लेन-देन की निगरानी करने और एम्बॉस जैसी थर्ड-पार्टी सेवाओं को एकीकृत करने के लिए एर्गोनोमिक Interface की तलाश कर रहे हैं। स्थानीय या टोर कनेक्शन के माध्यम से सुरक्षा सुनिश्चित की जाती है।**



यदि आपके पास अभी तक लाइटनिंग नोड नहीं है, तो हम आपको हमारे LND अम्ब्रेल ट्यूटोरियल का पालन करने की सलाह देते हैं:



https://planb.network/tutorials/node/lightning-network/umbrel-lnd-b12e0b5b-12ff-45f1-978e-62f4b4a8ba16

## इंस्टालेशन



थंडरहब को आपके लाइटनिंग नोड होस्टिंग वातावरण के आधार पर कई अलग-अलग तरीकों से इंस्टॉल किया जा सकता है। चाहे आप टर्नकी समाधान (अम्ब्रेल, वोल्टेज, रास्पिब्लिट्ज, मायनोड, स्टार्ट9, आदि) का उपयोग करें या मैन्युअल इंस्टॉलेशन, थंडरहब अक्सर बिना किसी बड़े प्रयास के उपलब्ध होता है। नीचे, हम दो सामान्य तरीकों का वर्णन करते हैं: अम्ब्रेल ऐप स्टोर के माध्यम से, और मैन्युअल इंस्टॉलेशन के माध्यम से (सर्वर या स्व-होस्टेड वितरण पर लागू)।



### अम्ब्रेल के माध्यम से स्थापना



अम्ब्रेल थंडरहब को अपने **ऐप स्टोर** में एकीकृत करता है, जिससे इंस्टॉलेशन बेहद सरल हो जाता है। कमांड लाइन या मैन्युअल कॉन्फ़िगरेशन की कोई ज़रूरत नहीं: सब कुछ Interface अम्ब्रेल के ज़रिए किया जाता है। बस इन चरणों का पालन करें:





- **अम्ब्रेल डैशबोर्ड खोलें**: अपने अम्ब्रेल नोड के Interface वेब से कनेक्ट करें (उदाहरण के लिए अपने स्थानीय नेटवर्क पर `http://umbrel.local`, या यदि आप Tor का उपयोग कर रहे हैं तो इसके `.onion` Address के माध्यम से)।
- **ऐप स्टोर तक पहुँचें**: अम्ब्रेल के मुख्य मेनू में, "ऐप स्टोर" (या "ऐप") पर क्लिक करें। उपलब्ध एप्लिकेशन की सूची में **थंडरहब** खोजें।



![Écran d'installation de ThunderHub via Umbrel](assets/fr/01.webp)





- **थंडरहब इंस्टॉल करें**: थंडरहब एप्लीकेशन पर क्लिक करें, फिर इंस्टॉल बटन पर क्लिक करें। यदि आवश्यक हो तो पुष्टि करें। अम्ब्रेल स्वचालित रूप से आपके नोड पर थंडरहब को डाउनलोड और तैनात करेगा।





- **एप्लिकेशन लॉन्च करें**: इंस्टॉलेशन पूरा होने के बाद (कुछ सेकंड में), थंडरहब आपके होम पेज पर दिखाई देगा। इसे खोलने के लिए आइकन पर क्लिक करें। थंडरहब आपके ब्राउज़र में लॉन्च हो जाएगा।



![Identifiants par défaut pour ThunderHub](assets/fr/02.webp)



**महत्वपूर्ण:** जब थंडरहब पहली बार खोला जाता है, तो यह स्वचालित रूप से लॉग इन करने के लिए आवश्यक **डिफ़ॉल्ट पासवर्ड** प्रदर्शित करता है। "इसे फिर से न दिखाएँ" विकल्प आपको भविष्य के कनेक्शन के लिए इस डिस्प्ले को छिपाने देता है। **हम आपको दृढ़ता से सलाह देते हैं कि:**




- इस पासवर्ड को तुरंत अपने पासवर्ड मैनेजर में सेव करें
- अगले चरण में उपयोग के लिए इसे कॉपी करें
- पासवर्ड सहेजे जाने के बाद "इसे दोबारा न दिखाएं" चेक करें



![Page de connexion de ThunderHub](assets/fr/03.webp)



इसके बाद आपको लॉगिन पेज पर ले जाया जाएगा, जहां आपको Interface को अनलॉक करने के लिए पिछले चरण में कॉपी किया गया पासवर्ड दर्ज करना होगा।



![Interface de connexion complète à ThunderHub](assets/fr/04.webp)



Umbrel बैकग्राउंड में ThunderHub को LND कनेक्शन जानकारी (TLS प्रमाणपत्र, एडमिनिस्ट्रेशन मैकरून, आदि) प्रदान करने का ध्यान रखता है, इसलिए आपको कोई अतिरिक्त कॉन्फ़िगरेशन करने की आवश्यकता नहीं है। बस कुछ ही क्लिक में, आपके Umbrel नोड पर ThunderHub चालू हो जाएगा।



### मैनुअल इंस्टॉलेशन (स्वयं-होस्टेड, अम्ब्रेल को छोड़कर)



अम्ब्रेल के बाहर के उपयोगकर्ताओं के लिए (जैसे कि किसी व्यक्तिगत सर्वर पर, RaspiBlitz के साथ Raspberry Pi या *स्टैंड-अलोन* इंस्टॉलेशन), ThunderHub इंस्टॉलेशन के लिए कुछ अतिरिक्त चरणों की आवश्यकता होती है। नीचे हम [आधिकारिक ThunderHub दस्तावेज़](https://docs.thunderhub.io) के अनुसार स्रोत से इंस्टॉलेशन और कॉन्फ़िगरेशन का वर्णन करते हैं।



#### इंस्टालेशन



**पूर्वापेक्षाएँ:** सुनिश्चित करें कि आपका सिस्टम [दस्तावेज़ीकरण सेटअप](https://docs.thunderhub.io/setup) के अनुसार न्यूनतम आवश्यकताओं को पूरा करता है:




- **Node.js** संस्करण 18 या उच्चतर
- **एनपीएम** स्थापित
- LND प्रमाणीकरण फ़ाइलों तक पहुंच :
  - LND TLS प्रमाणपत्र (`tls.cert`)
  - LND प्रशासन मैकरून (`admin.macaroon`)
  - LND gRPC सेवा Address (होस्टनाम:पोर्ट) (स्थानीय रूप से डिफ़ॉल्ट `127.0.0.1:10009`)



**1. थंडरहब कोड प्राप्त करें:** [इंस्टॉलेशन डॉक्यूमेंटेशन](https://docs.thunderhub.io/installation) में बताए अनुसार प्रोजेक्ट के GitHub रिपॉजिटरी को क्लोन करें:



```bash
git clone https://github.com/apotdevin/thunderhub.git
cd thunderhub
```



**2. निर्भरताएं स्थापित करें और एप्लिकेशन बनाएं:**



```bash
npm install
npm run build
```



ये कमांड सभी आवश्यक मॉड्यूल स्थापित करते हैं और फिर एप्लिकेशन को संकलित करते हैं (थंडरहब टाइपस्क्रिप्ट/रिएक्ट में लिखा गया है)।



**3. बाद में अपडेट करें:** थंडरहब भविष्य के अपडेट के लिए कई तरीके प्रदान करता है:



```bash
# Méthode rapide
npm run update

# Ou méthode manuelle
git pull
npm install
npm run build
```



#### कॉन्फ़िगरेशन (सेटअप)



**1. मुख्य कॉन्फ़िगरेशन फ़ाइल:** कॉन्फ़िगरेशन को कस्टमाइज़ करने के लिए थंडरहब फ़ोल्डर की रूट पर `.env.local` फ़ाइल बनाएँ (यह अपडेट के दौरान आपकी सेटिंग्स को ओवरराइट होने से रोकेगा)। [सेटअप डॉक्यूमेंटेशन](https://docs.thunderhub.io/setup) के अनुसार मुख्य वैरिएबल:



```bash
# -----------
# Server Configs
# -----------
LOG_LEVEL='info' # 'error' | 'warn' | 'info' | 'http' | 'verbose' | 'debug' | 'silly'
PORT=3000
NODE_ENV=production

# -----------
# Interface Configs
# -----------
THEME='dark' # 'dark' | 'light' | 'night'
CURRENCY='sat' # 'sat' | 'btc' | 'fiat'

# -----------
# Privacy Configs
# -----------
FETCH_PRICES=true # Récupération des prix BTC/fiat depuis Blockchain.com
FETCH_FEES=true # Récupération des frais on-chain depuis Earn.com
DISABLE_LINKS=false # Liens vers 1ml.com et Blockchain.com
NO_VERSION_CHECK=false # Vérification de version depuis GitHub

# -----------
# TOR (optionnel)
# -----------
TOR_PROXY_SERVER='socks://127.0.0.1:9050' # Pour proxifier via TOR

# -----------
# Account Configs
# -----------
ACCOUNT_CONFIG_PATH='/chemin/vers/thubConfig.yaml' # Fichier de comptes
```



**2. सर्वर खाता कॉन्फ़िगरेशन:** `ACCOUNT_CONFIG_PATH` में निर्दिष्ट YAML फ़ाइल बनाएँ:



```yaml
masterPassword: 'votre_mot_de_passe_principal'
accounts:
- name: 'Mon Nœud LND'
serverUrl: '127.0.0.1:10009'
macaroonPath: '/home/user/.lnd/data/chain/bitcoin/mainnet/admin.macaroon'
certificatePath: '/home/user/.lnd/tls.cert'
password: 'mot_de_passe_compte_specifique'
# Optionnel : compte avec macaroon en hexadécimal
- name: 'Nœud Distant'
serverUrl: 'ip.distante:10009'
macaroon: '0201056c6e6402f8...' # Macaroon en HEX ou Base64
certificate: '0202045c7365...' # Certificat en HEX ou Base64
```



**3. दूरस्थ पहुँच:** दूरस्थ LND नोड से कनेक्ट करने के लिए, `LND.conf` में जोड़ें:



```bash
# Option 1 : accès par IP
tlsextraip=<ip-externe-accessible>
rpclisten=0.0.0.0:10009

# Option 2 : accès par domaine
tlsextradomain=<domaine-externe-accessible>
rpclisten=0.0.0.0:10009
```



**4. सुरक्षा:** पहली बार शुरू होने पर, थंडरहब **स्वचालित रूप से** YAML फ़ाइल में `masterPassword` और सभी खाता पासवर्ड को छिपा देता है, ताकि सर्वर पर पासवर्ड स्पष्ट पाठ में न दिखाई दें।



**5. थंडरहब शुरू करें:**



```bash
npm start
```



डिफ़ॉल्ट रूप से, सर्वर पोर्ट 3000 पर सुनता है। `http://localhost:3000` (या स्थानीय नेटवर्क से `http://ip-serveur:3000`) तक पहुँचें।



**6. डॉकर विकल्प:** थंडरहब आधिकारिक डॉकर छवियां प्रदान करता है:



```bash
# Image standard
docker pull apotdevin/thunderhub:latest
docker run --rm -it -p 3000:3000/tcp apotdevin/thunderhub:latest

# Image avec base path /thub
docker pull apotdevin/thunderhub:base-v0.11.1
```



थंडरहब लॉगिन पेज दिखाई देगा। कॉन्फ़िगर किए गए खाते का चयन करें और डैशबोर्ड तक पहुँचने के लिए पासवर्ड दर्ज करें।



**अन्य वितरणों पर स्थापना:** पूर्व-पैक नोड वितरण (रास्पिब्लिट्ज़, मायनोड, स्टार्ट9, आदि) आम तौर पर अपने संबंधित प्रशासन इंटरफेस के माध्यम से मूल थंडरहब समर्थन प्रदान करते हैं।



**अधिक जानकारी के लिए:** पूर्ण आधिकारिक दस्तावेज देखें:




- **स्थापना:** [docs.thunderhub.io/installation](https://docs.thunderhub.io/installation)
- **कॉन्फ़िगरेशन:** [docs.thunderhub.io/setup](https://docs.thunderhub.io/setup)



ये संसाधन उन्नत विकल्पों जैसे कि SSO खाते, एन्क्रिप्टेड मैकरून, TOR कॉन्फ़िगरेशन और बहुत कुछ के बारे में विस्तार से बताते हैं।



एक बार थंडरहब इंस्टॉल हो जाने और उस तक पहुँच पाने के बाद, आप इसकी सभी सुविधाओं का लाभ उठाने के लिए तैयार हैं। अगले भाग में, हम Interface थंडरहब और इसके विभिन्न टैब पर नज़र डालेंगे, ताकि आपको इसके उपयोग के बारे में मार्गदर्शन मिल सके।



## जी.डब्लू.-19 प्रस्तुति



Interface थंडरहब एक मुख्य मेनू (आमतौर पर बाएं हाथ के कॉलम में प्रदर्शित) के आसपास संरचित है जिसमें कई प्रमुख अनुभाग शामिल हैं। प्रत्येक आपके लाइटनिंग नोड को प्रबंधित करने के एक पहलू से मेल खाता है। आइए एक-एक करके उन पर नज़र डालें:





- **होम** - सामान्य डैशबोर्ड वाला होम टैब (आपके नोड और त्वरित कार्रवाइयों का अवलोकन).
- **डैशबोर्ड** - विजेट और उन्नत मैट्रिक्स के साथ अनुकूलन योग्य डैशबोर्ड।
- **पीयर्स** - लाइटनिंग पीयर प्रबंधन (अन्य नोड्स से कनेक्शन).
- **चैनल** - लाइटनिंग चैनलों का विस्तृत प्रबंधन।
- **पुनर्संतुलन** - चैनल संतुलन उपकरण (परिपत्र भुगतान)।
- **लेन-देन** - लाइटनिंग भुगतान इतिहास (LN लेनदेन)।
- **फॉरवर्ड** - रूटिंग सांख्यिकी (आपके नोड द्वारा रिले किए गए भुगतान)।
- **चेन** - नोड का On-Chain पोर्टफोलियो (On-Chain BTC: UTXOs, लेनदेन)।
- **एम्बॉस** - एम्बॉस के साथ एकीकरण (नोड मॉनिटरिंग, बैकअप, आदि)।
- **उपकरण** - विविध उपकरण (बैकअप, हस्ताक्षरित संदेश, मैकरून, रिपोर्ट, आदि).
- **स्वैप** - On-Chain/लाइटनिंग स्वैप कार्य बोल्ट्ज़ के माध्यम से।
- **आँकड़े** - उन्नत आँकड़े और नोड प्रदर्शन मेट्रिक्स।



*(नोट: थंडरहब संस्करण के आधार पर, कुछ अनुभागों में थोड़ा अलग शीर्षक या अतिरिक्त विशेषताएं हो सकती हैं, लेकिन सामान्य सिद्धांत समान रहते हैं)*



### होम (सामान्य नियंत्रण पैनल)



थंडरहब का **होम** टैब वह होम पेज है जो आपके लॉग इन करने के बाद दिखाई देता है। इसमें **सामान्य डैशबोर्ड** शामिल है, जो आपके लाइटनिंग नोड की स्थिति का **वैश्विक अवलोकन** प्रदान करता है और नियमित संचालन के लिए **त्वरित क्रियाएँ** सुझाता है। यहाँ आपको आमतौर पर क्या मिलेगा:



![Tableau de bord principal de ThunderHub](assets/fr/05.webp)





- **शेष राशि और क्षमता:** पृष्ठ के शीर्ष पर, थंडरहब आपके उपलब्ध शेष राशि को प्रदर्शित करता है। यहाँ आपको आमतौर पर On-Chain शेष राशि (नोड के Wallet में Bitcoin On-Chain, जिसे Anchor ⚓ द्वारा दर्शाया गया है) और लाइटनिंग शेष राशि (आपके चैनल की क्षमता, जिसे लाइटनिंग Bolt ⚡ द्वारा दर्शाया गया है) दिखाई देगी। यह आपको On-Chain और लाइटनिंग में आपके पास मौजूद फंड का तुरंत अंदाजा देता है। यदि आपके पास कई खाते या चैनल हैं, तो सुनिश्चित करें कि आप सही खाते या चैनल पर





- **मुख्य आँकड़े:** डैशबोर्ड आपके नोड के लिए कुछ वैश्विक मीट्रिक दिखा सकता है - उदाहरण के लिए, खुले चैनलों की संख्या, कनेक्टेड पीयर की संख्या, अर्जित रूटिंग शुल्क (यदि लागू हो), आदि। यह नोड की हाल की गतिविधि और स्वास्थ्य का सारांश है।





- **त्वरित क्रियाएँ:** डैशबोर्ड में सबसे आम कार्यों को जल्दी से निष्पादित करने के लिए बटन हैं, बिना मेनू के माध्यम से नेविगेट किए। इन त्वरित क्रियाओं में शामिल हैं:





- **भूत**: एम्बॉस के माध्यम से एक कस्टम लाइटनिंग Address स्थापित करें।
- **दान करें**: लाइटनिंग के माध्यम से दान करें।
  - लॉग इन करें/यहां जाएं: अपने एम्बॉस खाते से कनेक्ट करें (त्वरित कनेक्ट) और अपने नोड की जानकारी देखने के लिए सीधे एम्बॉस.स्पेस पर जाएं।
- **Address** : भुगतान करने के लिए लाइटनिंग Address दर्ज करें।
- **खोलें**: एक नया लाइटनिंग चैनल खोलें। क्लिक करने पर रिमोट नोड का URI दर्ज करने के लिए एक फॉर्म खुलता है, जिस पर चैनल खोलना है, राशि और, यदि लागू हो, तो उपयोग की जाने वाली अधिकतम On-Chain फीस।
- **डिकोड**: भुगतान से पहले विवरण देखने के लिए लाइटनिंग Invoice या LNURL को डिकोड करें।
- **LNURL**: लाइटनिंग भुगतान या निकासी के लिए LNURLs की प्रक्रिया करें।
- **LnMarkets लॉगिन**: ट्रेडिंग के लिए LnMarkets पर लॉगिन करें।



ये त्वरित क्रियाएं आपको Interface के विभिन्न टैबों पर नेविगेट किए बिना, सीधे होम पेज से सबसे अधिक बार किए जाने वाले कार्यों को करने की अनुमति देती हैं।



संक्षेप में, थंडरहब डैशबोर्ड आपको अपने नोड पर एक **त्वरित नज़र** देता है और आपको एक क्लिक के साथ सबसे अधिक बार किए जाने वाले ऑपरेशन (भेजना/प्राप्त करना, चैनल खोलना) करने देता है। यह दिन-प्रतिदिन के प्रशासन के लिए आदर्श शुरुआती बिंदु है।



### डैशबोर्ड



**डैशबोर्ड** अनुभाग होम टैब से अलग है और अधिक उन्नत, अनुकूलन योग्य डैशबोर्ड प्रदान करता है। यह अनुभाग आपको नोड ऑपरेटर के रूप में अपनी आवश्यकताओं के अनुसार विशिष्ट विजेट के साथ एक अनुकूलित दृश्य बनाने की अनुमति देता है।





- **अनुकूलन योग्य विजेट:** होम पेज के विपरीत, जिसमें एक निश्चित लेआउट होता है, डैशबोर्ड आपको यह चुनने की सुविधा देता है कि कौन सा Elements प्रदर्शित करना है और उन्हें कैसे व्यवस्थित करना है।



![Dashboard sans widgets activés](assets/fr/06.webp)



यदि कोई विजेट सक्षम नहीं है, तो आपको अनुकूलन मापदंडों तक पहुंचने के लिए "सेटिंग्स" बटन के साथ "कोई विजेट सक्षम नहीं है!" संदेश दिखाई देगा।



![Paramètres des widgets du Dashboard](assets/fr/07.webp)



सेटिंग्स में, आप श्रेणियों में व्यवस्थित विजेट की एक विस्तृत श्रृंखला से चुन सकते हैं: "लाइटनिंग - जानकारी", "लाइटनिंग - तालिका", "लाइटनिंग - ग्राफ़", और इसी तरह। प्रत्येक विजेट को "दिखाएँ/छिपाएँ" बटन के साथ व्यक्तिगत रूप से सक्रिय या निष्क्रिय किया जा सकता है।



![Bas de la page des paramètres](assets/fr/08.webp)



सेटिंग्स के नीचे, आपको डैशबोर्ड पर वापस जाने के लिए "डैशबोर्ड पर जाएं" बटन मिलेगा और डिफ़ॉल्ट डिस्प्ले को रीसेट करने के लिए "विजेट रीसेट करें" बटन मिलेगा।



![Dashboard personnalisé avec widgets](assets/fr/09.webp)



एक बार कॉन्फ़िगर हो जाने पर, आपका डैशबोर्ड विभिन्न ग्राफ और मेट्रिक्स प्रदर्शित कर सकता है: लाइटनिंग भुगतान ग्राफ, जारी किए गए चालानों की संख्या, अग्रेषण आँकड़े, विस्तृत शेष राशि, आदि।





- **उन्नत मीट्रिक्स:** ग्राफ़ और वास्तविक समय डेटा के साथ अपने नोड के प्रदर्शन पर अधिक विस्तृत सांख्यिकी तक पहुंचें।





- **विन्यास योग्य अवलोकन:** चाहे आप एक आकस्मिक उपयोगकर्ता हों या एकाधिक रूटिंग चैनलों का प्रबंधन करने वाले पेशेवर ऑपरेटर हों, डिस्प्ले को अपनी सुविधानुसार ढालें।





- **मॉड्यूलर Interface:** आवश्यकतानुसार विजेट जोड़ें या हटाएं: फॉरवर्ड चार्ट, लिक्विडिटी मेट्रिक्स, नोड हेल्थ अलर्ट, आदि।



यह अनुभाग विशेष रूप से उन्नत उपयोगकर्ताओं के लिए उपयोगी है जो विशिष्ट मीट्रिक की निगरानी करना चाहते हैं या अपने लाइटनिंग नोड का अधिक तकनीकी अवलोकन प्राप्त करना चाहते हैं। यह जानकारी प्रदर्शित करने के तरीके पर अधिक लचीलापन और नियंत्रण प्रदान करके होम अनुभाग का पूरक है।



### समकक्ष लोग



**पीयर्स** अनुभाग में वर्तमान में आपके साथ जुड़े सभी लाइटनिंग नोड्स को पीयर्स के रूप में सूचीबद्ध किया गया है। **पीयर** Lightning Network पर एक सीधा नोड-टू-नोड कनेक्शन है। आपका नोड बिना किसी खुले चैनल के भी पीयर्स से कनेक्ट हो सकता है (उदाहरण के लिए, नेटवर्क पर केवल Exchange गॉसिप जानकारी के लिए), या निश्चित रूप से प्रत्येक खुला चैनल स्वचालित रूप से एक कनेक्टेड पीयर का संकेत देता है।



![Vue de l'onglet Peers avec les pairs connectés](assets/fr/10.webp)



पीयर्स टैब में, आप देखेंगे:





- **सूचना कॉलम:** Interface उपयोगी विवरण प्रदर्शित करता है, जैसे कि सिंक्रोनाइजेशन स्थिति, कनेक्शन प्रकार (क्लियरनेट या टोर), पिंग, प्राप्त/भेजे गए सातोशी और आदान-प्रदान किए गए डेटा की मात्रा।
- **एक सहकर्मी जोड़ें:** थंडरहब आपको ऊपरी दाएँ कोने में **"जोड़ें"** बटन के माध्यम से मैन्युअल रूप से एक नए सहकर्मी से कनेक्ट करने देता है। आपको नोड का URI (प्रारूप `<public_key>@<socket>`) दर्ज करना होगा। एक बार मान्य होने के बाद, थंडरहब संबंधित `lncli connect` कमांड भेजता है। यदि नोड ऑनलाइन और सुलभ है, तो इसे आपके सहकर्मियों की सूची में जोड़ दिया जाएगा।



### चैनल



**चैनल** टैब लाइटनिंग चैनल प्रबंधन का दिल है। यह संभवतः वह अनुभाग है जिसे आप सबसे अधिक बार देखेंगे। यह **आपके सभी लाइटनिंग चैनल** को उनके विवरण के साथ प्रस्तुत करता है, और आपको इन चैनलों पर प्रबंधन क्रियाएँ करने की अनुमति देता है।



![Vue d'ensemble des channels ouverts](assets/fr/11.webp)



चैनल पृष्ठ पर आपको यह मिलेगा:





- **चैनल सूची दृश्य:** प्रत्येक खुला (या खुलने/बंद होने वाला) चैनल सूचीबद्ध होता है, आमतौर पर रिमोट नोड के उपनाम, कुल चैनल क्षमता और स्थानीय बनाम रिमोट लिक्विडिटी के वितरण को दर्शाने वाली रंगीन पट्टी के साथ। थंडरहब चैनल बैलेंस को दर्शाने के लिए एक रंग कोड (अक्सर नीला/Green) या प्रतिशत का उपयोग करता है: उदाहरण के लिए, आपके स्थानीय शेयर के लिए नीला, रिमोट शेयर के लिए Green। यदि कोई चैनल पूरी तरह से संतुलित (50/50) है, तो बार प्रत्येक





- **सूचना कॉलम:** Interface ग्राफिकल लिक्विडिटी डिस्प्ले के साथ स्थिति, उपलब्ध क्रियाकलाप, सहकर्मी जानकारी, चैनल आईडी, क्षमता, गतिविधि, शुल्क और शेष राशि सहित विस्तृत कॉलम प्रदर्शित करता है।





- **प्रदर्शन कॉन्फ़िगरेशन:** ऊपरी दाएं कोने में एक कोगव्हील आपको अपनी प्राथमिकताओं के अनुरूप चैनल प्रदर्शन को अनुकूलित करने देता है।





- **स्थिति:** आपको स्थिति संकेतक भी दिखाई देंगे - जैसे `सक्रिय` (चैनल खुला और चालू है), `ऑफ़लाइन` (साथी डिस्कनेक्ट है, इसलिए चैनल कुछ समय के लिए अनुपयोगी है), `लंबित` (On-Chain पुष्टि की प्रतीक्षा में खुलने या बंद होने के लिए)।





- **चैनल पर क्रियाएं:** प्रत्येक चैनल के लिए, थंडरहब क्रिया बटन प्रदान करता है (अक्सर आइकन के रूप में):



![Actions de gestion des canaux - Modifier et Fermer](assets/fr/12.webp)





  - शुल्क संपादित करें:** Interface "चैनल नीति अपडेट करें" आपको सभी चैनल पैरामीटर समायोजित करने देता है: बेस शुल्क, शुल्क दर (पीपीएम में), CLTV डेल्टा, अधिकतम HTLC और न्यूनतम HTLC। यह आपको रूटिंग ट्रैफ़िक को आकर्षित करने (या हतोत्साहित करने) के उद्देश्य से प्रत्येक चैनल के लिए अपनी शुल्क नीतियों को व्यक्तिगत रूप से समायोजित करने की अनुमति देता है। *(नोट: थंडरहब स्वचालित शुल्क प्रबंधन उपकरण का विकल्प नहीं है, लेकिन मैन्युअल समायोजन के लिए यह बहुत प्रभावी है)*
- चैनल बंद करें (**बंद करें**): Interface "चैनल बंद करें" आपको शुल्क (Sats/vByte में) परिभाषित करके **सहकारी बंद** (डिफ़ॉल्ट) या **मजबूर बंद** (*बलपूर्वक बंद*) के बीच विकल्प देता है। **महत्वपूर्ण:** जब भी संभव हो, हमेशा सहकारी बंद को प्राथमिकता दें, ताकि On-Chain निपटान में देरी और उच्च शुल्क से बचा जा सके। थंडरहब आपको बताएगा कि सहकर्मी ऑनलाइन है (सहकारी संभव है) या नहीं। बलपूर्वक बंद होने की स्थिति में, पुष्टि करना सुनिश्चित करें क्योंकि यह अपरि
- **नया चैनल खोलें:** नया चैनल खोलने के लिए, चैनल पेज के ऊपर दाईं ओर स्थित कोगव्हील पर क्लिक करें, फिर "खोलें" चुनें। फिर आप किसी नए या मौजूदा पीयर के लिए चैनल शुरू कर सकते हैं। इस पेज का उपयोग करने का लाभ यह है कि आपके सामने आपके मौजूदा चैनलों की एक सूची होती है, जो आपको यह तय करने में मदद कर सकती है कि नया चैनल कहाँ खोलना है।



संक्षेप में, चैनल अनुभाग आपको **प्रत्येक चैनल पर बढ़िया नियंत्रण** देता है। यह वह जगह है जहाँ आप लिक्विडिटी आवंटन को संचालित करते हैं, तय करते हैं कि कौन से चैनल रखने हैं या बंद करने हैं, और अपने प्रति-चैनल रूटिंग पैरामीटर सेट करते हैं। थंडरहब इन महत्वपूर्ण नोड प्रबंधन कार्यों के लिए एक स्पष्ट Interface प्रदान करता है।



### संतुलित



**रीबैलेंस** टैब **चैनल बैलेंसिंग** के लिए समर्पित है। बैलेंसिंग (या *रीबैलेंसिंग*) में आपके आउटगोइंग और इनकमिंग चैनलों के बीच फंड के वितरण को फिर से समायोजित करना शामिल है, Lightning Network के माध्यम से आपके एक चैनल से दूसरे चैनल में **सर्कुलर भुगतान** करके। यह आपको नए फंड लाए बिना, एक चैनल से लिक्विडिटी को एक ऐसे चैनल में स्थानांतरित करने की अनुमति देता है जो बहुत भरा हुआ है, जो बहुत खाली है, जिससे आपके चैनल अधिक उपयोगी बन जाते हैं (एक संतुलित चैनल भुगतान भेज और प्राप्त दोनों कर सकता है)।



![Interface de rebalance des canaux](assets/fr/13.webp)



थंडरहब इस ऑपरेशन को बहुत आसान बनाता है, जो अन्यथा कमांड लाइन पर थकाऊ होगा। रीबैलेंस सेक्शन का उपयोग करने का तरीका यहां बताया गया है:





- **आरंभिक चैनल दृश्य:** रीबैलेंस में प्रवेश करने पर, थंडरहब आपके चैनलों की एक सूची प्रदर्शित करता है, जिसमें प्रत्येक के लिए एक बैलेंस इंडिकेटर होता है (चैनल पेज पर मौजूद इंडिकेटर के समान)। आप तुरंत देख सकते हैं कि कौन से चैनल बैलेंस से बाहर हैं। थंडरहब चैनलों को बैलेंस बढ़ाने के क्रम में सॉर्ट कर सकता है, ताकि सबसे असंतुलित चैनल सूची में सबसे ऊपर दिखाई दें (0.0 का मतलब पूरी तरह से स्थानीय या दूरस्थ है)।





- **सहकर्मी चयन:** Interface पुनर्संतुलन के लिए आउटगोइंग और इनकमिंग सहकर्मियों का चयन करना आसान बनाता है।





- **पैरामीटर सेटिंग्स:** आप सेट कर सकते हैं:
  - **अधिकतम शुल्क** (Sats और ppm में) जो आप भुगतान करने को तैयार हैं
  - "निश्चित" या "लक्ष्य" विकल्प के साथ **पुनर्संतुलित की जाने वाली राशि**
- **रूटिंग करते समय बचने योग्य नोड्स**
- **मार्ग खोजने के लिए अधिकतम परीक्षण समय**





- **स्रोत चैनल चुनें**: सबसे पहले **आउटगोइंग (स्रोत)** चैनल चुनें, यानी वह चैनल जिससे आपके पास स्थानांतरित करने के लिए बहुत अधिक स्थानीय तरलता है। व्यवहार में, यह एक ऐसा चैनल है जहाँ आपका स्थानीय हिस्सा उच्च (> 50%) है। आइए 1,000,000 सैट्स वाले एक ए चैनल की कल्पना करें, जिनमें से 900,000 स्थानीय हैं - सैट्स को कहीं और भेजने के लिए एक अच्छा उम्मीदवार। इस ए चैनल पर "आउटगोइंग" के रूप में क्लिक करके, थंडरहब इसे एक स्रोत के रूप में च





- **लक्ष्य चैनल** चुनें: इसके बाद, **आने वाला (लक्ष्य)** चैनल चुनें जिसे लिक्विडिटी प्राप्त करने की आवश्यकता है। आम तौर पर, यह एक ऐसा चैनल होगा जहाँ यह उल्टा होगा - अधिकांश फंड दूर की तरफ हैं (उदाहरण के लिए 1,000,000 में से केवल 100,000 स्थानीय सैट्स)। थंडरहब, एक बार स्रोत चैनल का चयन करने के बाद, सबसे अधिक पूरक चैनलों की पहचान करने में मदद करने के लिए अन्य चैनलों को उल्टे क्रम (घटते हुए बैलेंस) में सॉर्ट करेगा। एक B चैनल





- **शुल्क राशि और सहनशीलता निर्धारित करें:** एक फॉर्म आपको निम्नलिखित दर्ज करने की अनुमति देता है:





  - **राशि** जिसे पुनः संतुलित किया जाना है (Sats में)। अक्सर, हम ऐसी राशि चुनते हैं जो दोनों चैनलों को \~50/50 पर संतुलित करेगी। उदाहरण के लिए, थंडरहब स्रोत चैनल की आधी अतिरिक्त क्षमता को पहले से भर सकता है।
  - इस ऑपरेशन के लिए आप जो **अधिकतम शुल्क** चुकाने को तैयार हैं (वैकल्पिक)। यह शुल्क Sats (सर्कुलर रूटिंग की कुल लागत) में व्यक्त किया जाता है। यदि आप इसे खाली छोड़ देते हैं, तो थंडरहब लागत की परवाह किए बिना एक पथ की खोज करेगा, जो आम तौर पर उचित नहीं है (एक सीमा निर्धारित करना बेहतर है, उदाहरण के लिए एक छोटे से पुनर्संतुलन के लिए 10 Sats, या अधिकतम पीपीएम)।





- **मार्ग खोजें:** मार्ग खोजने के लिए बटन पर क्लिक करें। थंडरहब आपके स्रोत चैनल से नेटवर्क के माध्यम से आपके अपने लक्ष्य चैनल तक मार्ग की गणना करने के लिए LND से पूछताछ करता है। यदि उसे कोई संभावित मार्ग मिलता है जो आपके शुल्क मानदंड को पूरा करता है, तो वह उसे हॉप्स और शुल्क लागत के विवरण के साथ प्रदर्शित करता है। उदाहरण के लिए, यह संकेत दे सकता है कि उसे कुल 2 Sats शुल्क के साथ 3-हॉप पथ मिला है।





- **पुनर्संतुलन शुरू करें:** यदि आप प्रस्तावित मार्ग से खुश हैं, तो **शेष चैनल** पर क्लिक करें। थंडरहब तब LND के माध्यम से परिपत्र भुगतान आरंभ करेगा। यदि भुगतान सफल होता है, तो आपको सफलता की सूचना दिखाई देगी, और चैनल A और B में वास्तविक समय में उनके शेष राशि को संशोधित किया जाएगा। थंडरहब इन चैनलों के लिए शेष राशि संकेतक को अपडेट करेगा (आदर्श रूप से वे पहले की तुलना में अधिक हरे होंगे, जो बेहतर संतुलन का संकेत देते हैं)।





- **समायोजन और पुनरावृत्तियाँ:** यदि पहले प्रयास में कोई मार्ग नहीं मिलता (या यदि यह बहुत महंगा है), तो आप पैरामीटर समायोजित कर सकते हैं:





  - छोटी राशि का प्रयास करें (कभी-कभी आंशिक पुनर्संतुलन सफल हो जाता है, जबकि बड़ी राशि विफल हो जाती है)।
  - अधिकतम शुल्क को धीरे-धीरे बढ़ाएं, लेकिन ध्यान रखें कि शुल्क के रूप में इसकी कीमत से अधिक भुगतान न करना पड़े।
  - यदि उपलब्ध हो तो वैकल्पिक मार्ग आजमाने के लिए **अन्य मार्ग प्राप्त करें** बटन का प्रयोग करें।
  - यदि चीजें वास्तव में कठिन हो जाएं तो अन्य चैनल आज़माएँ।



थंडरहब इस प्रक्रिया को बहुत ही सहज और दृश्य बनाता है। केवल 4 चरणों में (आउटगोइंग चैनल चुनें, इनकमिंग चैनल चुनें, राशि, मान्य करें), आप वह कर सकते हैं जो पहले जटिल मैन्युअल कमांड की आवश्यकता होती थी। यह टूल एक संतुलित नोड को बनाए रखने, आपके रूटिंग प्रदर्शन और अनुभव को बेहतर बनाने (आपके सभी चैनलों पर भुगतान भेजना और प्राप्त करना आसान) के लिए अमूल्य है।



अंत में, ध्यान दें कि पुनर्संतुलन रूटिंग लागत (मध्यवर्ती नोड्स को भुगतान) का उपभोग करता है, इसलिए यह आपके नोड को अधिक तरल बनाने के लिए एक **निवेश** है। इसका बुद्धिमानी से उपयोग करें, उदाहरण के लिए किसी ऐसी सेवा के लिए चैनल का समर्थन करने के लिए जिसका आप अक्सर उपयोग करते हैं (इनबाउंड लिक्विडिटी) या किसी बड़े रूटिंग चैनल को संतुलित करने के लिए। थंडरहब आपको यह **सरलतापूर्वक और कुशलतापूर्वक** करने देता है।



### लेनदेन



थंडरहब में **लेनदेन** अनुभाग आपके नोड के **लाइटनिंग** लेन-देन इतिहास से मेल खाता है, यानी चैनलों के माध्यम से भुगतान या प्राप्त किए गए भुगतान और चालान। यह आपके LN संचालन के लिए एक तरह का खाता विवरण है।



![Historique des transactions Lightning](assets/fr/14.webp)



इस टैब में आपको मिलेगा:





- **Invoice ग्राफ:** ऊपरी दाएं कोने में, एक ग्राफ समय के साथ प्राप्त चालानों के विकास को दर्शाता है, जिससे आप अपने नोड की गतिविधि को देख सकते हैं।





- आपके नोड से या आपके नोड को किए गए सभी लाइटनिंग लेनदेन की कालानुक्रमिक सूची। प्रत्येक प्रविष्टि में निम्न दिखाया जा सकता है:





  - ऑपरेशन का प्रकार: **भेजा गया भुगतान** (आउटगोइंग भुगतान) या **प्राप्त भुगतान** (इनबाउंड, सशुल्क Invoice के माध्यम से)।
  - राशि Sats में.
  - दिनांक समय।
  - भुगतान आईडी (Hash या RHash प्री-इमेज) या टिप्पणी (यदि आपने Invoice में कोई मेमो जोड़ा है)।
  - स्थिति: **पूर्ण**, या संभवतः **प्रगति पर**/*विफल* (उदाहरण के लिए, समाधान की प्रतीक्षा में भुगतान, लेकिन आम तौर पर LND इसे शीघ्रता से संसाधित करता है, इसलिए On-Chain लेनदेन की तुलना में यहां बहुत कम "लंबित" है)।



संक्षेप में, लेन-देन अनुभाग आपके **LN गतिविधि लॉग** के रूप में कार्य करता है। यह जाँचने के लिए बहुत उपयोगी है कि कोई भुगतान हुआ है या नहीं, इस पर कितनी फीस लगी है, या आपके लाइटनिंग एक्सचेंज के इतिहास का पता लगाना। फ़ॉरवर्ड अनुभाग (जिसका वर्णन आगे किया गया है) के साथ, आपके पास अपने नोड से गुज़रे पैसे का पूरा दृश्य होगा।



### आगे



**फॉरवर्ड** टैब आपके नोड की **रूटिंग** गतिविधि के लिए समर्पित है, यानी आपके चैनलों के माध्यम से **ट्रांजिट** करने वाले भुगतान (जब आप Lightning Network पर मध्यस्थ नोड के रूप में कार्य करते हैं)। यदि आप अपने नोड को रूटिंग नोड के रूप में संचालित करते हैं, तो यह आपके प्रदर्शन को ट्रैक करने के लिए एक महत्वपूर्ण अनुभाग है।



![Statistiques de routage Lightning](assets/fr/15.webp)



फॉरवर्ड्स में, थंडरहब प्रस्तुत करता है:





- **फ़िल्टर और प्रदर्शन विकल्प:** ऊपर दाईं ओर, फ़िल्टर आपको दिन/सप्ताह/माह/वर्ष के अनुसार डेटा को क्रमबद्ध करने और ग्राफ़िकल या सारणीबद्ध प्रदर्शन के बीच चयन करने की सुविधा देते हैं।





- **गतिविधि संदेश:** यदि चयनित अवधि के दौरान कोई रूटिंग नहीं की गई है, तो Interface "इस अवधि के लिए कोई अग्रेषण नहीं" प्रदर्शित करता है, जैसा कि इस उदाहरण में दिखाया गया है।





- हाल ही में किए गए फ़ॉरवर्ड की तालिका: प्रत्येक प्रविष्टि आपके नोड के माध्यम से किए गए भुगतान से मेल खाती है। प्रत्येक फ़ॉरवर्ड के लिए, हम आम तौर पर देखते हैं:





  - जीडब्ल्यू-71,
  - भेजी गई राशि (Sats में),
  - इस अग्रेषण पर अर्जित शुल्क (Sats में, यह आपके द्वारा इनकमिंग चैनल पर प्राप्त और आउटगोइंग चैनल पर भेजे गए शुल्क के बीच का अंतर है),
  - उपयोग किए जाने वाले इनकमिंग और आउटगोइंग चैनल (अक्सर पीयर उपनाम या चैनल आईडी द्वारा पहचाने जाते हैं)।
  - स्थिति (सामान्यतः *पूर्ण*, या विफलता यदि मार्ग में अग्रेषित विफल हुआ)।





- **समेकित आंकड़े**: थंडरहब एक निश्चित अवधि (जैसे पिछले 24 घंटे, या 7 दिन, आदि, कभी-कभी विन्यास योग्य) के लिए कुल और आंकड़ों की गणना करता है और पृष्ठ के शीर्ष पर प्रदर्शित करता है।



संक्षेप में, फॉरवर्ड सेक्शन आपके लाइटनिंग नोड की रूटिंग गतिविधि का **वास्तविक समय का अवलोकन** प्रदान करता है। चैनल और रीबैलेंस सेक्शन के साथ मिलकर, यह आपके नोड को अनुकूलित करने के लिए एक पूर्ण पैकेज बनाता है: लिक्विडिटी के लिए चैनल/रीबैलेंस, परिणाम (प्रवाह और लाभ) देखने के लिए फॉरवर्ड।



### जंजीर



**चेन** अनुभाग आपके LND नोड के Bitcoin On-Chain पोर्टफोलियो प्रबंधन से मेल खाता है। यह Interface आपको Bitcoin फंड देखने और प्रबंधित करने की अनुमति देता है, जिसका उपयोग चैनल खोलने या बंद चैनलों से फंड प्राप्त करने के लिए किया जाता है।



![Gestion du portefeuille on-chain](assets/fr/16.webp)



चेन में आपको मिलेगा:





- **बैलेंस On-Chain:** Wallet LND में उपलब्ध कुल BTC बैलेंस प्रदर्शित करता है।





- **UTXO की सूची:** सभी अप्रयुक्त आउटपुट (UTXO) को राशि, पुष्टिकरण, Address और प्रत्येक आउटपुट के प्रारूप के साथ देखें।





- **लेन-देन इतिहास:** सभी Bitcoin लेन-देन की विस्तृत तालिका, प्रकार (इन/आउट), तिथि, राशि, शुल्क, पुष्टिकरण, समावेशन ब्लॉक, पते और txid के साथ।



### अम्बॉस



थंडरहब एम्बॉस प्लेटफॉर्म (amboss.space) के साथ एकीकृत है, जो लाइटनिंग नोड्स, एक लिक्विडिटी मार्केटप्लेस और एन्क्रिप्टेड चैनल बैकअप और उपलब्धता मॉनिटरिंग जैसी उपयोगी सुविधाओं पर विस्तृत जानकारी प्रदान करता है।



![Intégration Amboss avec ses options](assets/fr/17.webp)



थंडरहब में, एम्बॉस अनुभाग आपको अपने नोड को अपने एम्बॉस खाते से लिंक करने की अनुमति देता है:





- **घोस्ट Address:** अपने नोड के लिए एक **व्यक्तिगत लाइटनिंग Address** सेट करें, जिससे आने वाले भुगतानों में सुविधा होगी।





- स्वचालित चैनल बैकअप:** एम्बॉस पर एन्क्रिप्टेड चैनल बैकअप** (एससीबी फ़ाइलें) के लिए प्रमुख सुविधा। हर बार चैनल बदलने पर एन्क्रिप्टेड बैकअप अपडेट स्वचालित रूप से भेजने के लिए सेटिंग्स में ** एम्बॉस ऑटो बैकअप = हां ** सक्रिय करें। विफलता की स्थिति में, आप इस बाहरी बैकअप की बदौलत अपने फंड को पुनर्प्राप्त करने में सक्षम होंगे।





- **स्वास्थ्य जाँच:** अपने नोड को नियमित रूप से एम्बॉस को पिंग भेजने के लिए **एम्बॉस स्वास्थ्य जाँच = हाँ** सक्रिय करें। यदि आपका नोड ऑफ़लाइन दिखाई देता है, तो आपको अलर्ट प्राप्त होंगे।





- अन्य विशेषताएं: स्वचालित बैलेंस पुश, मैग्मा/हाइड्रो एकीकरण (तरलता बाज़ार), और विस्तृत प्रदर्शन आंकड़ों तक पहुंच।



एम्बॉस एकीकरण स्वचालित बाह्य बैकअप और उपलब्धता निगरानी के साथ एक आवश्यक **सुरक्षा Layer** जोड़ता है, जो थंडरहब से सीधे सुलभ है।



### औजार



**टूल्स** अनुभाग आपके नोड को प्रबंधित करने के लिए विभिन्न उन्नत उपकरणों को एक साथ समूहित करता है। यहाँ मुख्य Elements हैं:



![Interface des outils disponibles](assets/fr/18.webp)





- **बैकअप:** अपने चैनल बैकअप (SCB) को मैन्युअल रूप से प्रबंधित करें। थंडरहब आपको अपने चैनलों की **पूरी बैकअप फ़ाइल डाउनलोड करने देता है** (विकल्प "सभी चैनलों का बैकअप लें -> डाउनलोड करें")। इस `channel-all.bak` फ़ाइल को सुरक्षित स्थान पर रखें - क्रैश की स्थिति में अपने फंड को पुनर्प्राप्त करने के लिए यह आवश्यक है। आप नोड को फिर से तैनात करते समय बैकअप फ़ाइल को **आयात** भी कर सकते हैं।





- **लेखांकन:** किसी निश्चित अवधि में अर्जित/भुगतान किए गए शुल्क और प्रेषित मात्रा सहित वित्तीय रिपोर्टों के लिए निर्यात उपकरण।
- **हस्ताक्षरित संदेश:** क्रिप्टोग्राफिक हस्ताक्षर के माध्यम से अपने लाइटनिंग नोड के Ownership को प्रमाणित करने के लिए अपने नोड के साथ संदेशों पर हस्ताक्षर करें या उन्हें सत्यापित करें।
- मैकरून (बेकरी अनुभाग):** अनुकूलित पहुँच बनाने के लिए LND** मैकरून प्रबंधित करें। Interface "बेकरी" आपको प्रत्येक अनुमति को सटीक रूप से चुनने की अनुमति देता है: "पीयर जोड़ें या निकालें", "चेन पते बनाएँ", "चालान बनाएँ", "मैकरून बनाएँ", "कुंजी प्राप्त करें", "पहुँच कुंजियाँ प्राप्त करें", "चेन लेनदेन प्राप्त करें", "चालान प्राप्त करें", "Wallet जानकारी प्राप्त करें", "भुगतान प्राप्त करें", "पीयर प्राप्त करें", "चालान का भुगतान करें", "पहुँच आईडी रद्द करें", "चेन पतों पर भेजें", "बाइट्स पर हस्ताक्षर करें", "संदेशों पर हस्ताक्षर करें", "daemon रोकें", "बाइट्स हस्ताक्षर सत्यापित करें", "संदेश सत्यापित करें", और इसी तरह। प्रत्येक अनुमति को "हां/नहीं" बटन के साथ व्यक्तिगत रूप से सक्रिय किया जा सकता है ताकि एक दर्जी मैकरून बनाया जा सके।
- **सिस्टम जानकारी:** Wallet संस्करण और सक्रिय RPC का प्रदर्शन।



संक्षेप में, उपकरण अनुभाग उन्नत प्रशासन कार्यों - बैकअप, लेखांकन, सुरक्षा और पहुंच प्रबंधन - को एकीकृत Interface में एक साथ लाता है।



### बदलना



थंडरहब का **स्वैप** टैब आपको बोल्ट्ज सेवा के माध्यम से लाइटनिंग सैटोशी को Bitcoin On-Chain में स्वैप करने की सुविधा देता है। यह सुविधा चैनल को बंद किए बिना चैनल में अतिरिक्त लाइटनिंग लिक्विडिटी को "डंप" करने के लिए उपयोगी है।



![Interface de swap via Boltz](assets/fr/19.webp)



प्रक्रिया सरल है:





- **राशि**: विनिमय की जाने वाली राशि निर्धारित करें
- **Address** : Bitcoin रिसेप्शन Address दर्ज करें
- निष्पादन: थंडरहब बोल्ट्ज़ के साथ संचार करके Exchange को स्वचालित रूप से संसाधित करता है



**फ़ायदे:**




- गैर-हिरासत सेवा (नकद हिरासत नहीं)
- अपने मौजूदा चैनलों को सुरक्षित रखें
- उपयोग में आसान एकीकृत Interface



बोल्ट्ज़ एक छोटा कमीशन लेता है और आप मानक Bitcoin लेनदेन शुल्क का भुगतान करते हैं। थंडरहब पुष्टि से पहले सभी लागतों को प्रदर्शित करता है।



### आँकड़े



थंडरहब का **आँकड़े** अनुभाग आपके लाइटनिंग नोड पर प्रदर्शन का विश्लेषण करने और रूटिंग को अनुकूलित करने के लिए **उन्नत आँकड़े** प्रदान करता है।



![Statistiques du nœud via Amboss](assets/fr/20.webp)



यह अनुभाग आपकी लागतों को अनुकूलित करने, सफल चैनलों की पहचान करने और आपके नोड के विस्तार की योजना बनाने के लिए आवश्यक है।



## निष्कर्ष



**थंडरहब** ने खुद को लाइटनिंग **LND** नोड के आसान प्रशासन के लिए आवश्यक उपकरण के रूप में स्थापित किया है। यह आधुनिक Interface सभी आवश्यक कार्य प्रदान करता है: चैनल प्रबंधन, भुगतान, निगरानी, ​​​​स्वचालित पुनर्संतुलन और एम्बॉस एकीकरण जैसी उन्नत सुविधाओं के साथ।



**मुख्य लाभ:**




- Interface चिकना और सहज
- शक्तिशाली उपकरण (पुनर्संतुलन, बोल्ट्ज़ स्वैप, स्वचालित बैकअप)
- अम्ब्रेल, वोल्टेज, रास्पिब्लिट्ज़ और अन्य वितरणों के साथ संगत



थंडरहब उन्नत लाइटनिंग नोड प्रबंधन को लोकतांत्रिक बनाता है, जो पहले जटिल तकनीकी आदेशों की आवश्यकता होती थी उसे सुलभ बनाता है। चाहे आप एक शुरुआती या एक अनुभवी ऑपरेटर हों, थंडरहब आपको एक आधुनिक, व्यापक Interface के माध्यम से अपने लाइटनिंग नोड को कुशलतापूर्वक प्रबंधित करने देता है।
