## Введение



ThunderHub - это **опенсорс-менеджер для узлов Lightning (LND)**, предлагающий интуитивно понятный Interface, доступный с любого устройства или браузера.



**Основные характеристики:**




- **Мониторинг**: Глобальный обзор балансов, каналов, транзакций, статистики маршрутизации
- **Управление**: Открытие/закрытие каналов, входящие/исходящие платежи, балансировка каналов
- **Интеграции**: Поддержка LNURL, свопы через Boltz, резервное копирование Amboss
- **Interface отзывчивый**: Совместимость с мобильными, планшетными и настольными устройствами с темными/светлыми темами



ThunderHub легко интегрируется с **Umbrel**, **Voltage**, **RaspiBlitz** и **MyNode**, упрощая повседневное управление вашим узлом.



**ThunderHub особенно подходит операторам, которым нужен эргономичный Interface для управления каналами, контроля ликвидности (ребалансировки), мониторинга транзакций и интеграции сторонних сервисов, таких как Amboss.** Безопасность обеспечивается с помощью локального или Tor-соединения.



Если у вас еще нет узла Lightning, мы рекомендуем вам следовать нашему руководству по LND Umbrel:



https://planb.network/tutorials/node/lightning-network/umbrel-lnd-b12e0b5b-12ff-45f1-978e-62f4b4a8ba16

## Установка



ThunderHub можно установить разными способами, в зависимости от среды хостинга узла Lightning. Независимо от того, используете ли вы готовое решение (Umbrel, Voltage, RaspiBlitz, MyNode, Start9 и т. д.) или устанавливаете вручную, ThunderHub часто доступен без особых усилий. Ниже мы опишем два распространенных подхода: через магазин приложений Umbrel и через ручную установку (применимо к серверу или самостоятельному хостингу).



### Инсталляция через Umbrel



Umbrel интегрирует ThunderHub в свой **App Store**, что делает установку чрезвычайно простой. Нет необходимости в командной строке или ручной настройке: все делается через Interface Umbrel. Просто выполните следующие шаги:





- Откройте **панель управления Umbrel**: Подключитесь к сети Interface вашего узла Umbrel (например, `http://umbrel.local` в локальной сети или через его `.onion` Address, если вы используете Tor).
- Войдите в магазин приложений: В главном меню Umbrel нажмите на "App Store" (или "App"). Найдите **ThunderHub** в списке доступных приложений.



![Écran d'installation de ThunderHub via Umbrel](assets/fr/01.webp)





- **Установите ThunderHub**: Нажмите на приложение ThunderHub, затем на кнопку установки. При необходимости подтвердите установку. Umbrel автоматически загрузит и развернет ThunderHub на вашем узле.





- **Запустите приложение**: После завершения установки (несколько десятков секунд) ThunderHub появится на вашей домашней странице. Нажмите на значок, чтобы открыть его. ThunderHub запустится в вашем браузере.



![Identifiants par défaut pour ThunderHub](assets/fr/02.webp)



**Важно:** При первом открытии ThunderHub автоматически отображается **пароль по умолчанию**, необходимый для входа в систему. Опция "Не показывать это снова" позволяет скрыть это отображение для последующих подключений. **Мы настоятельно рекомендуем вам:**




- **Сохраните этот пароль немедленно** в менеджере паролей
- Скопируйте его для использования в следующем шаге
- Установите флажок "Не показывать это снова" после сохранения пароля



![Page de connexion de ThunderHub](assets/fr/03.webp)



Вы попадете на страницу входа в систему, где необходимо ввести пароль, скопированный на предыдущем шаге, чтобы разблокировать Interface.



![Interface de connexion complète à ThunderHub](assets/fr/04.webp)



Umbrel позаботится о предоставлении ThunderHub информации о подключении LND (сертификат TLS, административный макарон и т. д.) в фоновом режиме, поэтому вам не нужно выполнять никаких дополнительных настроек. Всего за несколько кликов ThunderHub будет запущен на вашем узле Umbrel.



### Ручная установка (самостоятельное размещение, за исключением Umbrel)



Для пользователей вне Umbrel (например, на персональном сервере, на Raspberry Pi с RaspiBlitz или при *одиночной* установке) установка ThunderHub требует нескольких дополнительных шагов. Ниже мы опишем установку из исходного кода и настройку в соответствии с [официальной документацией ThunderHub](https://docs.thunderhub.io).



#### Установка



**Пререквизиты:** Убедитесь, что ваша система соответствует минимальным требованиям, указанным в [документации setup](https://docs.thunderhub.io/setup):




- **Node.js** версии 18 или выше
- **npm** установлен
- Доступ к файлам аутентификации LND :
  - LND Сертификат TLS (`tls.cert`)
  - LND административный макаронник (`admin.macaroon`)
  - LND gRPC-сервис Address (имя хоста:порт) (по умолчанию `127.0.0.1:10009` локально)



**1. Получите код ThunderHub:** Клонируйте репозиторий GitHub проекта, как описано в [документации по установке](https://docs.thunderhub.io/installation):



```bash
git clone https://github.com/apotdevin/thunderhub.git
cd thunderhub
```



**2. Установите зависимости и соберите приложение:**



```bash
npm install
npm run build
```



Эти команды устанавливают все необходимые модули, а затем компилируют приложение (ThunderHub написан на TypeScript/React).



**3. Обновление позже:** ThunderHub предлагает несколько способов обновления в будущем:



```bash
# Méthode rapide
npm run update

# Ou méthode manuelle
git pull
npm install
npm run build
```



#### Конфигурация (настройка)



**1. Основной файл конфигурации:** Создайте файл `.env.local` в корне папки ThunderHub для настройки конфигурации (это предотвратит перезапись ваших настроек при обновлении). Основные переменные в соответствии с [документацией по настройке](https://docs.thunderhub.io/setup):



```bash
# -----------
# Server Configs
# -----------
LOG_LEVEL='info' # 'error' | 'warn' | 'info' | 'http' | 'verbose' | 'debug' | 'silly'
PORT=3000
NODE_ENV=production

# -----------
# Interface Configs
# -----------
THEME='dark' # 'dark' | 'light' | 'night'
CURRENCY='sat' # 'sat' | 'btc' | 'fiat'

# -----------
# Privacy Configs
# -----------
FETCH_PRICES=true # Récupération des prix BTC/fiat depuis Blockchain.com
FETCH_FEES=true # Récupération des frais on-chain depuis Earn.com
DISABLE_LINKS=false # Liens vers 1ml.com et Blockchain.com
NO_VERSION_CHECK=false # Vérification de version depuis GitHub

# -----------
# TOR (optionnel)
# -----------
TOR_PROXY_SERVER='socks://127.0.0.1:9050' # Pour proxifier via TOR

# -----------
# Account Configs
# -----------
ACCOUNT_CONFIG_PATH='/chemin/vers/thubConfig.yaml' # Fichier de comptes
```



**2. Конфигурация учетных записей сервера:** Создайте YAML-файл, указанный в `ACCOUNT_CONFIG_PATH` :



```yaml
masterPassword: 'votre_mot_de_passe_principal'
accounts:
- name: 'Mon Nœud LND'
serverUrl: '127.0.0.1:10009'
macaroonPath: '/home/user/.lnd/data/chain/bitcoin/mainnet/admin.macaroon'
certificatePath: '/home/user/.lnd/tls.cert'
password: 'mot_de_passe_compte_specifique'
# Optionnel : compte avec macaroon en hexadécimal
- name: 'Nœud Distant'
serverUrl: 'ip.distante:10009'
macaroon: '0201056c6e6402f8...' # Macaroon en HEX ou Base64
certificate: '0202045c7365...' # Certificat en HEX ou Base64
```



**3. Удаленный доступ:** Чтобы подключиться к удаленному узлу LND, добавьте в файл `LND.conf` :



```bash
# Option 1 : accès par IP
tlsextraip=<ip-externe-accessible>
rpclisten=0.0.0.0:10009

# Option 2 : accès par domaine
tlsextradomain=<domaine-externe-accessible>
rpclisten=0.0.0.0:10009
```



**4. Безопасность:** При первом запуске ThunderHub **автоматически** скрывает `masterPassword` и все пароли учетных записей в YAML-файле, чтобы избежать наличия паролей в открытом виде на сервере.



**5. Запустите ThunderHub:**



```bash
npm start
```



По умолчанию сервер прослушивает порт 3000. Зайдите на `http://localhost:3000` (или `http://ip-serveur:3000` из локальной сети).



**6. Альтернатива Docker:** ThunderHub предоставляет официальные образы Docker:



```bash
# Image standard
docker pull apotdevin/thunderhub:latest
docker run --rm -it -p 3000:3000/tcp apotdevin/thunderhub:latest

# Image avec base path /thub
docker pull apotdevin/thunderhub:base-v0.11.1
```



Откроется страница входа в ThunderHub. Выберите настроенную учетную запись и введите пароль для доступа к приборной панели.



**Установка на другие дистрибутивы:** Готовые дистрибутивы узлов (RaspiBlitz, MyNode, Start9 и т. д.) обычно предлагают встроенную поддержку ThunderHub через соответствующие интерфейсы администрирования.



**Для получения дополнительной информации:** Обратитесь к полной официальной документации :




- **Установка:** [docs.thunderhub.io/installation](https://docs.thunderhub.io/installation)
- **Конфигурация:** [docs.thunderhub.io/setup](https://docs.thunderhub.io/setup)



На этих ресурсах подробно описаны такие расширенные возможности, как учетные записи SSO, шифрованные макароны, настройка TOR и многое другое.



Когда ThunderHub установлен и доступен, вы готовы использовать все его возможности. В следующем разделе мы рассмотрим Interface ThunderHub и его различные вкладки, чтобы помочь вам разобраться с его использованием.



## Презентация Interface



Структура Interface ThunderHub включает в себя главное меню (обычно отображается в левой колонке), состоящее из нескольких ключевых разделов. Каждый из них соответствует определенному аспекту управления узлом Lightning. Давайте рассмотрим их по порядку:





- **Главная** - Главная вкладка с общей панелью (обзор вашего узла и быстрые действия).
- **Приборная панель** - настраиваемая приборная панель с виджетами и расширенными показателями.
- **Peers** - управление пирами Lightning (соединения с другими узлами).
- **Каналы** - Детальное управление каналами Lightning.
- **Rebalance** - инструмент балансировки каналов (круговые платежи).
- **Транзакции** - история молниеносных платежей (LN транзакция).
- **Forwards** - статистика маршрутизации (платежи, переданные вашим узлом).
- **Цепочка** - портфель On-Chain Node (On-Chain BTC: UTXOs, транзакции).
- **Amboss** - Интеграция с Amboss (мониторинг узлов, резервное копирование и т. д.).
- **Инструменты** - Различные инструменты (резервное копирование, подписанные сообщения, макары, отчеты и т. д.).
- **Подмена** - функции подмены On-Chain/Lightning через Boltz.
- **Stats** - расширенная статистика и показатели производительности узлов.



*(Примечание: в зависимости от версии ThunderHub некоторые разделы могут иметь несколько иные заголовки или дополнительные функции, но общие принципы остаются неизменными)*



### Главная (общая панель управления)



Вкладка **Home** в ThunderHub - это главная страница, которая появляется после входа в систему. Она содержит **Общую приборную панель**, которая предлагает **глобальный обзор** состояния вашего узла Lightning и предлагает **быстрые действия** для выполнения рутинных операций. Вот что вы обычно найдете:



![Tableau de bord principal de ThunderHub](assets/fr/05.webp)





- **Балансы и мощности:** В верхней части страницы ThunderHub отображает ваши доступные балансы. Здесь вы обычно видите баланс On-Chain (Bitcoin On-Chain в Wallet узла, обозначенный Anchor ⚓) и баланс Lightning (мощности ваших каналов, обозначенные молнией Bolt ⚡). Это дает вам представление о том, какими средствами вы располагаете в On-Chain и Lightning. Если у вас несколько аккаунтов или каналов, убедитесь, что вы находитесь на правильном (например, Mainnet против Testnet).





- **Ключевая статистика:** На приборной панели могут отображаться некоторые глобальные показатели для вашего узла - например, количество открытых каналов, количество подключенных пиров, заработанная плата за маршрутизацию (если применимо) и т. д. Это краткая информация о недавней активности и состоянии узла.





- **Быстрые действия:** На приборной панели имеются кнопки для быстрого выполнения наиболее распространенных задач без необходимости перемещения по меню. Эти быстрые действия включают в себя :





- **Призрак**: Установите пользовательский Lightning Address через Amboss.
- **Пожертвовать**: Сделайте пожертвование через Lightning.
- **Вход/переход**: Подключитесь к своему аккаунту Amboss (Quick Connect) и перейдите непосредственно на сайт Amboss.space, чтобы просмотреть информацию о своем узле.
- **Address**: Введите молнию Address, чтобы произвести оплату.
- **Открыть**: Открыть новый канал Lightning. При нажатии открывается форма для ввода URI удаленного узла, к которому нужно открыть канал, суммы и, если применимо, максимальной комиссии On-Chain, которая будет использоваться.
- **Декодирование**: Декодируйте Lightning Invoice или LNURL, чтобы просмотреть детали перед оплатой.
- **LNURL**: Обрабатывайте LNURL для платежей или снятия средств с помощью Lightning.
- **Вход в систему LnMarkets**: Войдите в систему LnMarkets для торговли.



Эти быстрые действия позволяют выполнять наиболее частые операции прямо с главной страницы, без необходимости перемещаться по различным вкладкам Interface.



Одним словом, приборная панель ThunderHub дает вам возможность **быстро взглянуть** на ваш узел и выполнить самые частые операции (отправить/получить, открыть канал) одним щелчком мыши. Это идеальная отправная точка для повседневного администрирования.



### Приборная панель



Раздел **Панель** находится отдельно от вкладки "Главная" и предлагает более продвинутую, настраиваемую панель. Этот раздел позволяет создать индивидуальное представление с определенными виджетами в соответствии с вашими потребностями как оператора узла.





- **Настраиваемые виджеты:** В отличие от главной страницы, которая имеет фиксированный макет, панель Dashboard позволяет вам выбрать, какие именно Elements будут отображаться и как их организовать.



![Dashboard sans widgets activés](assets/fr/06.webp)



Если виджеты не включены, вы увидите сообщение "No Widgets Enabled!" с кнопкой "Settings" для доступа к параметрам настройки.



![Paramètres des widgets du Dashboard](assets/fr/07.webp)



В настройках можно выбрать из широкого спектра виджетов, разбитых на категории: "Молния - информация", "Молния - таблица", "Молния - график" и так далее. Каждый виджет можно активировать или деактивировать отдельно с помощью кнопок "Показать/скрыть".



![Bas de la page des paramètres](assets/fr/08.webp)



В нижней части настроек находится кнопка "На приборную панель" для возврата на приборную панель и кнопка "Сбросить виджеты" для сброса отображения по умолчанию.



![Dashboard personnalisé avec widgets](assets/fr/09.webp)



После настройки приборная панель может отображать различные графики и метрики: Графики молниеносных платежей, количество выставленных счетов, статистика форвардов, подробные балансы и т.д.





- **Расширенные метрики:** Доступ к более подробной статистике производительности вашего узла с графиками и данными в реальном времени.





- **Настраиваемый обзор:** Настройте дисплей так, чтобы он подходил как для обычного пользователя, так и для профессионального оператора, управляющего несколькими каналами маршрутизации.





- **Модульный Interface:** Добавляйте и удаляйте виджеты по мере необходимости: графики форвардов, метрики ликвидности, оповещения о состоянии узлов и т. д.



Этот раздел особенно полезен для опытных пользователей, которые хотят отслеживать конкретные показатели или получить более подробный технический обзор своего узла Lightning. Он дополняет раздел "Главная", предлагая большую гибкость и контроль над отображением информации.



### Ровесники



В разделе **Пиры** перечислены все узлы Lightning, подключенные к вашему узлу в качестве пиров. **Пир** - это прямое соединение между узлами на Lightning Network. Ваш узел может быть связан с равными даже без открытого канала (например, просто для того, чтобы Exchange сплетничал о сети), или, конечно, каждый открытый канал автоматически подразумевает наличие связанного равного.



![Vue de l'onglet Peers avec les pairs connectés](assets/fr/10.webp)



На вкладке Peers вы увидите :





- **Информационные колонки:** На Interface отображаются такие полезные сведения, как статус синхронизации, тип соединения (clearnet или Tor), пинг, полученные/отправленные сатоши и объем обмениваемых данных.
- **Добавление пира:** ThunderHub позволяет вручную подключиться к новому пиру с помощью кнопки **"Добавить"** в правом верхнем углу. Вам нужно будет ввести URI узла (формат `<public_key>@<socket>`). После проверки ThunderHub отправляет соответствующую команду `lncli connect`. Если узел находится в сети и доступен, он будет добавлен в список ваших пиров.



### Каналы



Вкладка **Channels** - это сердце управления каналами Lightning. Вероятно, именно к этому разделу вы будете обращаться чаще всего. Здесь представлены **все ваши каналы Lightning** с их подробной информацией, и вы можете выполнять действия по управлению этими каналами.



![Vue d'ensemble des channels ouverts](assets/fr/11.webp)



Вот что вы найдете на странице "Каналы":





- **Вид списка каналов:** Каждый открытый (или открывающийся/закрывающийся) канал отображается в списке, обычно с указанием псевдонима удаленного узла, общей емкости канала и цветной полосой, иллюстрирующей распределение локальной и удаленной ликвидности. ThunderHub использует цветовой код (чаще всего синий/Green) или процентное соотношение для обозначения баланса канала: например, синий - ваша локальная доля, Green - удаленная доля. Если канал идеально сбалансирован (50/50), полоса будет наполовину состоять из каждого цвета. Это позволяет с первого взгляда определить, какие каналы несбалансированы (все синие = почти все локальные, все Green = почти все удаленные).





- **Информационные колонки:** Interface отображает подробные колонки, включая статус, доступные действия, информацию о коллегах, идентификатор канала, емкость, активность, плату и баланс с графическим отображением ликвидности.





- **Конфигурация дисплея:** Зубчатое колесо в правом верхнем углу позволяет настроить отображение каналов в соответствии с вашими предпочтениями.





- **Статус:** Вы также увидите индикаторы статуса - например, `Active` (канал открыт и работает), `Offline` (пир отключен, поэтому канал временно недоступен), `Pending` (для открытий или закрытий, ожидающих подтверждения On-Chain).





- **Действия на канале:** Для каждого канала ThunderHub предоставляет кнопки действий (часто в виде иконок):



![Actions de gestion des canaux - Modifier et Fermer](assets/fr/12.webp)





- **Редактирование сборов:** "Обновить политику канала" Interface позволяет настроить все параметры канала: Base Fee, Fee Rate (в ppm), CLTV Delta, Max HTLC и Min HTLC. Это позволяет настраивать политику тарифов индивидуально для каждого канала с целью привлечения (или отталкивания) маршрутизируемого трафика. *(Примечание: ThunderHub не является заменой автоматического инструмента управления комиссиями, но для ручной настройки он очень эффективен)*
- Канал закрытия (*Close*): Interface "Close Channel" дает вам возможность выбора между **кооперативным закрытием** (по умолчанию) и **принудительным закрытием** (*Force Close*), определяя сборы (в Sats/vByte). **Важно:** всегда предпочитайте кооперативное закрытие, когда это возможно, чтобы избежать задержек в расчетах On-Chain и более высоких комиссий. ThunderHub сообщит вам, находится ли пир в сети (возможно кооперативное закрытие) или нет. В случае принудительного закрытия обязательно подтвердите его, так как это необратимо и вызовет транзакцию с задержкой по времени (обычно 144 блока или ~1 день на Bitcoin Mainnet).
- **Чтобы открыть новый канал:** Чтобы открыть новый канал, нажмите на шестеренку в правом верхнем углу страницы "Каналы" и выберите "Открыть". Вы можете открыть канал для нового или существующего коллеги. Преимущество использования этой страницы в том, что перед вами список существующих каналов, который поможет вам решить, где открыть новый канал.



Одним словом, раздел "Каналы" дает вам **тонкий контроль над каждым каналом**. Именно здесь вы управляете распределением ликвидности, решаете, какие каналы оставить или закрыть, и задаете параметры маршрутизации каждого канала. ThunderHub предлагает четкий Interface для этих важнейших операций по управлению узлами.



### Перебалансировка



Вкладка **Ребаланс** посвящена **балансировке каналов**. Балансировка (или *ребалансировка*) включает в себя изменение распределения средств между вашими исходящими и входящими каналами путем **кругового платежа** из одного вашего канала в другой, через Lightning Network. Это позволяет вам, не вводя новых средств, переместить ликвидность из слишком переполненного канала в слишком пустой, сделав ваши каналы более полезными (сбалансированный канал может как отправлять, так и получать платежи).



![Interface de rebalance des canaux](assets/fr/13.webp)



ThunderHub значительно облегчает эту операцию, которая в противном случае была бы утомительной в командной строке. Вот как использовать раздел Rebalance:





- **Первоначальный вид каналов:** При входе в Rebalance ThunderHub отображает список ваших каналов с индикатором баланса для каждого из них (аналогично индикатору на странице "Каналы"). Вы можете сразу увидеть, какие каналы находятся вне баланса. ThunderHub может сортировать каналы в порядке возрастания баланса, так что самые несбалансированные каналы будут выделяться в верхней части списка (0.0 означает полностью локальный или удаленный).





- **Выбор пиров:** Interface позволяет легко выбирать исходящие и входящие пиры для ребалансировки.





- **Настройки параметров:** Вы можете установить :
  - **максимальная плата** (в Sats и ppm), которую вы готовы заплатить
  - **сумма для ребалансировки** при использовании опции "Фиксированная" или "Целевая"
- **Узлы, которые следует избегать** при маршрутизации
- **Максимальное время испытания** для поиска маршрута





- Выберите **исходный канал**: Сначала выберите **исходящий (исходный)** канал, т.е. канал, из которого у вас слишком много локальной ликвидности для перемещения. На практике это канал, где ваша локальная доля высока (> 50 %). Представим себе канал А с 1 000 000 сатов, 900 000 из которых локальные - хороший кандидат для отправки сатов в другое место. Нажав на этот канал A как на "исходящий", ThunderHub отмечает его как источник.





- Выберите **целевой канал**: Далее выберите **входящий (целевой)** канал, который должен получать ликвидность. Как правило, это канал, где все наоборот - большинство средств находится на дальней стороне (например, только 100 000 местных сатов из 1 000 000). После выбора исходного канала ThunderHub отсортирует остальные каналы в обратном порядке (по убыванию баланса), чтобы помочь определить наиболее взаимодополняемые каналы. Выберите канал B, на котором есть место на местной стороне. ThunderHub четко покажет, какие два канала были выбраны (источник A и цель B).





- Установите размер платы и допуск: **Форма позволяет ввести:**





- **Сумма**, которую нужно перебалансировать (в Sats). Часто мы выбираем сумму, равную той, которая уравновесит оба канала при \~50/50. ThunderHub может предварительно заполнить половину избыточной емкости канала-источника, например.
  - **максимальная плата**, которую вы готовы заплатить за эту операцию (необязательно). Эта плата выражается в Sats (общая стоимость круговой маршрутизации). Если оставить это значение пустым, ThunderHub будет искать путь независимо от стоимости, что, как правило, не рекомендуется (лучше установить ограничение, например 10 Sats за небольшую ребалансировку или максимальное значение ppm).





- **Найти маршрут:** Нажмите на кнопку, чтобы найти маршрут. ThunderHub запрашивает LND, чтобы рассчитать маршрут от вашего исходного канала через сеть к вашему целевому каналу. Если он находит возможный маршрут, который соответствует вашим критериям оплаты, он отображает его с подробным описанием переходов и стоимости оплаты. Например, он может указать, что найден 3-хоповый маршрут с общей стоимостью 2 Sats.





- Начните ребалансировку: Если вас устраивает предложенный маршрут, нажмите на **Канал ребалансировки**. ThunderHub инициирует циркулярный платеж через LND. Если платеж пройдет успешно, вы увидите уведомление об успехе, а балансы каналов A и B будут изменены в режиме реального времени. ThunderHub обновит индикатор баланса для этих каналов (в идеале он будет более зеленым, чем раньше, что свидетельствует об улучшении баланса).





- **Корректировки и итерации:** Если маршрут не найден с первой попытки (или он слишком дорог), вы можете изменить параметры:





  - Попробуйте меньшую сумму (иногда частичная ребалансировка проходит, а большая - нет).
  - Постепенно увеличивайте максимальную плату, но будьте осторожны, чтобы не заплатить больше, чем это стоит.
  - Воспользуйтесь кнопкой **Get Another Route**, если она доступна, чтобы попробовать альтернативный вариант.
  - Попробуйте использовать другую пару каналов, если ситуация действительно затягивается.



ThunderHub делает этот процесс очень **интуитивным и наглядным**. Всего за 4 шага (выбор исходящего канала, выбор входящего канала, сумма, подтверждение) вы можете сделать то, что раньше требовало сложных ручных команд. Этот инструмент неоценим для поддержания сбалансированного узла, повышения производительности маршрутизации и удобства работы (проще отправлять и получать платежи по всем каналам).



Наконец, обратите внимание, что при ребалансировке расходуются средства на маршрутизацию (оплачиваются промежуточным узлам), поэтому это **инвестиция**, чтобы сделать ваш узел более подвижным. Используйте ее с умом, например, для поддержки канала к сервису, которым вы часто пользуетесь (входящая ликвидность), или для балансировки большого канала маршрутизации. ThunderHub позволяет сделать это **просто и эффективно**.



### Транзакции



Раздел **Транзакции** в ThunderHub соответствует истории транзакций **Lightning** вашего узла, то есть платежам и счетам, оплаченным или полученным по каналам. Это своего рода выписка по счету для ваших операций LN.



![Historique des transactions Lightning](assets/fr/14.webp)



На этой вкладке вы найдете :





- График **Invoice:** В правом верхнем углу находится график, отображающий динамику полученных счетов-фактур с течением времени, что позволяет наглядно представить активность вашего узла.





- Хронологический список всех транзакций Lightning, совершенных *с* или *к* вашей ноде. Каждая запись может показывать :





  - Тип операции: **отправленный платеж** (исходящий платеж) или **полученный платеж** (входящий, через оплаченный Invoice).
  - Сумма в Sats.
  - Дата/время.
  - Идентификатор платежа (Hash или предварительный образ RHash) или комментарий (если вы добавили памятку в Invoice).
  - Статус: **завершено**, или, возможно, **в процессе**/*не удалось* (например, платеж, ожидающий разрешения, но обычно LND обрабатывает его быстро, поэтому "ожидающих" здесь немного по сравнению с транзакциями On-Chain).



Одним словом, раздел "Транзакции" служит для вас **журналом действий LN**. Он очень полезен для проверки того, прошел ли платеж, сколько комиссий он стоил, или для отслеживания истории ваших обменов Lightning. В сочетании с разделом Forwards (описанным далее) вы получите полное представление о деньгах, прошедших через вашу ноду.



### Форварды



Вкладка **Forwards** посвящена **маршрутизации** вашего узла, то есть платежам, которые **транзитируются** через ваши каналы (когда вы выступаете в качестве узла-посредника на Lightning Network). Если вы используете свой узел в качестве маршрутизирующего узла, этот раздел очень важен для отслеживания результатов вашей работы.



![Statistiques de routage Lightning](assets/fr/15.webp)



В разделе "Форварды" ThunderHub представляет:





- **Фильтры и параметры отображения:** В правом верхнем углу расположены фильтры, позволяющие сортировать данные по дням/неделям/месяцам/годам, а также выбирать между графическим и табличным отображением.





- **Сообщение об активности:** Если в течение выбранного периода маршрутизация не выполнялась, Interface отображает "No forwards for this period", как показано в данном примере.





- Таблица **последних форвардов**: каждая запись соответствует платежу, который был **передан** через ваш узел. Для каждого форварда мы обычно видим :





  - Timestamp,
  - количество маршрутизации (в Sats),
  - **fee earned** on this forward (в Sats это разница между тем, что вы получили на входящем канале и отправили на исходящем),
  - используемые входящие и исходящие каналы (часто идентифицируются псевдонимом пира или идентификатором канала).
  - статус (обычно *завершено*, или неудача, если пересылка не удалась в пути).





- **Агрегированная статистика**: ThunderHub рассчитывает и отображает в верхней части страницы итоговые показатели и статистику за определенный период (например, за последние 24 часа, 7 дней и т. д., иногда настраивается).



Одним словом, раздел Forwards предлагает **обзор маршрутизации узла Lightning в режиме реального времени**. В сочетании с разделами Channels и Rebalance это формирует полный пакет для оптимизации узла: Каналы/Ребаланс для обеспечения ликвидности, Форварды для наблюдения за результатами (потоками и прибылью).



### Цепь



Раздел **Chain** соответствует управлению портфелем Bitcoin On-Chain вашего узла LND. Этот раздел Interface позволяет просматривать и управлять средствами Bitcoin, которые используются для открытия каналов или получения средств из закрытых каналов.



![Gestion du portefeuille on-chain](assets/fr/16.webp)



В цепочке вы найдете :





- Баланс On-Chain: **Отображает общий баланс BTC, доступный в Wallet LND.**





- **Список UTXO:** Просмотреть все неизрасходованные средства (UTXO) с указанием суммы, подтверждений, Address и формата для каждого средства.





- **История транзакций:** Подробная таблица всех транзакций Bitcoin с указанием типа (ввод/вывод), даты, суммы, начислений, подтверждений, блока включения, адресов и txid.



### Амбосс



ThunderHub интегрируется с платформой **Amboss** (amboss.space), которая предлагает подробную информацию об узлах Lightning, рынок ликвидности и такие полезные функции, как резервное копирование зашифрованных каналов и мониторинг доступности.



![Intégration Amboss avec ses options](assets/fr/17.webp)



В ThunderHub раздел Amboss позволяет **связать** ваш узел с учетной записью Amboss:





- **Ghost Address:** Установите **персонализированный Lightning Address** для вашей ноды, чтобы облегчить входящие платежи.





- Автоматическое резервное копирование каналов:** Флагманская функция для зашифрованных резервных копий каналов** (SCB-файлы) на Amboss. Активируйте **Amboss Auto Backup = Yes** в настройках, чтобы автоматически отправлять обновления зашифрованных резервных копий при каждом изменении каналов. В случае сбоя вы сможете восстановить свои средства благодаря этой внешней резервной копии.





- **Проверка здоровья:** Активируйте **Amboss Healthcheck = Yes**, чтобы ваш узел регулярно отправлял пинги в Amboss. Вы будете получать предупреждения, если ваш узел окажется в автономном режиме.





- **Другие возможности:** Автоматическое пополнение баланса, интеграция **Magma/Hydro** (рынок ликвидности) и доступ к подробной статистике работы.



Интеграция с Amboss добавляет существенную **безопасность Layer** с автоматическим внешним резервным копированием и мониторингом доступности, доступным непосредственно из ThunderHub.



### Инструменты



В разделе **Инструменты** собраны различные расширенные инструменты для управления узлом. Вот основные из них Elements:



![Interface des outils disponibles](assets/fr/18.webp)





- **Резервные копии:** Ручное управление резервными копиями ваших каналов (SCB). ThunderHub позволяет **скачать полный файл резервной копии** ваших каналов (опция "Резервная копия всех каналов -> Скачать"). Храните этот файл `channel-all.bak` в надежном месте - он необходим для восстановления ваших средств в случае сбоя. Вы также можете **импортировать** файл резервной копии при переразвертывании узла.





- **Бухгалтерский учет:** Инструмент экспорта финансовых отчетов, включая заработанные/оплаченные гонорары и объемы маршрутизации за определенный период.
- **Подписанные сообщения:** Подписывайте или проверяйте сообщения с помощью своего узла, чтобы подтвердить Ownership вашего узла Lightning с помощью криптографической подписи.
- Макароны (раздел "Пекарня"):** Управляйте макаронами LND** для создания индивидуального доступа. "Пекарня" Interface позволяет точно выбрать каждое разрешение: "Добавлять или удалять пиров", "Создавать адреса цепочки", "Создавать счета", "Создавать макароны", "Выводить ключи", "Получать ключи доступа", "Получать транзакции цепочки", "Получать счета", "Получать информацию Wallet", "Получать платежи", "Получать пиров", "Оплачивать счета", "Отзывать идентификаторы доступа", "Отправлять на адреса цепочки", "Подписывать байты", "Подписывать сообщения", "Останавливать daemon", "Проверять подпись байтов", "Проверять сообщения" и так далее. Каждое разрешение можно активировать по отдельности с помощью кнопок "Да/Нет", чтобы создать индивидуальный макаронник.
- **Информация о системе:** Отображение версии Wallet и активированных RPC.



Одним словом, раздел "Инструменты" объединяет расширенные функции администрирования - резервное копирование, учет, безопасность и управление доступом - в едином Interface.



### Обмен



Вкладка **Swap** на ThunderHub позволяет обменивать сатоши Lightning на Bitcoin On-Chain через сервис Boltz. Эта функция полезна для "сброса" избыточной ликвидности Lightning в канал без закрытия канала.



![Interface de swap via Boltz](assets/fr/19.webp)



Процесс прост:





- **Сумма**: Определите сумму для обмена
- **Address**: Введите Bitcoin прием Address
- **Исполнение**: ThunderHub взаимодействует с Boltz для автоматической обработки Exchange



**Преимущества:**




- Некастодиальное обслуживание (без хранения наличных)
- Сохраните существующие каналы
- Простой в использовании интегрированный Interface



Boltz взимает небольшую комиссию, а вы платите стандартную комиссию за транзакцию Bitcoin. ThunderHub отображает все расходы до подтверждения.

![Statistiques du nœud via Amboss](assets/fr/20.webp)

**ThunderHub** зарекомендовал себя как незаменимый инструмент для удобного администрирования узла Lightning **LND**. Этот современный Interface предлагает все основные функции: управление каналами, платежи, мониторинг, а также расширенные возможности, такие как автоматическая ребалансировка и интеграция с Amboss.



**Ключевые преимущества:**




- Interface элегантный и интуитивно понятный
- Мощные инструменты (ребалансировка, свопы Boltz, автоматическое резервное копирование)
- Совместимость с Umbrel, Voltage, RaspiBlitz и другими дистрибутивами



ThunderHub демократизирует управление узлом Lightning, делая доступным то, что раньше требовало сложных технических команд. Независимо от того, новичок вы или опытный оператор, ThunderHub позволит вам эффективно управлять узлом Lightning с помощью современного, всеобъемлющего Interface.
