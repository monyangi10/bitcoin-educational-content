## บทนำ



ThunderHub เป็น **ตัวจัดการโอเพ่นซอร์สสำหรับโหนด Lightning (LND)** ที่มี Interface ที่ใช้งานง่ายและเข้าถึงได้จากอุปกรณ์หรือเบราว์เซอร์ใด ๆ



**คุณสมบัติหลัก:**




- การตรวจสอบ**: มุมมองทั่วโลกของยอดคงเหลือ, ช่องทาง, ธุรกรรม, สถิติการกำหนดเส้นทาง
- การจัดการ**: เปิด/ปิดช่องทาง, การชำระเงินขาเข้า/ขาออก, การปรับสมดุลช่องทาง
- การผสานรวม**: รองรับ LNURL, การแลกเปลี่ยนผ่าน Boltz, การสำรองข้อมูล Amboss
- Interface responsive**: เข้ากันได้กับอุปกรณ์มือถือ แท็บเล็ต และเดสก์ท็อป พร้อมธีมสีเข้ม/สว่าง



ThunderHub ผสานรวมได้อย่างง่ายดายกับ **Umbrel**, **Voltage**, **RaspiBlitz** และ **MyNode** ทำให้การจัดการโหนดของคุณในแต่ละวันง่ายขึ้น



**ThunderHub เหมาะอย่างยิ่งสำหรับผู้ปฏิบัติงานที่มองหา Interface ที่มีการออกแบบตามหลักสรีรศาสตร์เพื่อจัดการช่องทางของพวกเขา ควบคุมสภาพคล่อง (การปรับสมดุล) ตรวจสอบธุรกรรม และผสานรวมบริการของบุคคลที่สาม เช่น Amboss ความปลอดภัยได้รับการรับรองผ่านการเชื่อมต่อในพื้นที่หรือ Tor**



หากคุณยังไม่มีโหนด Lightning เราขอแนะนำให้คุณทำตามบทแนะนำ LND Umbrel ของเรา:



https://planb.academy/tutorials/node/lightning-network/umbrel-lnd-b12e0b5b-12ff-45f1-978e-62f4b4a8ba16

## การติดตั้ง



ThunderHub สามารถติดตั้งได้หลายวิธี ขึ้นอยู่กับสภาพแวดล้อมการโฮสต์โหนด Lightning ของคุณ ไม่ว่าคุณจะใช้โซลูชันสำเร็จรูป (Umbrel, Voltage, RaspiBlitz, MyNode, Start9, ฯลฯ) หรือการติดตั้งด้วยตนเอง ThunderHub มักจะพร้อมใช้งานโดยไม่ต้องใช้ความพยายามมาก ด้านล่างนี้ เราจะอธิบายสองวิธีที่พบบ่อย: ผ่านทาง Umbrel App Store และผ่านการติดตั้งด้วยตนเอง (ใช้ได้กับเซิร์ฟเวอร์หรือการกระจายแบบโฮสต์เอง)



### การติดตั้งผ่าน Umbrel



Umbrel รวม ThunderHub เข้ากับ **App Store** ทำให้การติดตั้งง่ายมาก ไม่จำเป็นต้องใช้บรรทัดคำสั่งหรือการกำหนดค่าด้วยตนเอง: ทุกอย่างทำผ่าน Interface Umbrel เพียงทำตามขั้นตอนเหล่านี้:





- เปิดแดชบอร์ด Umbrel**: เชื่อมต่อกับเว็บ Interface ของโหนด Umbrel ของคุณ (เช่น `http://umbrel.local` บนเครือข่ายท้องถิ่นของคุณ หรือผ่านที่อยู่ `.onion` หากคุณใช้ Tor)
- เข้าถึง App Store**: ในเมนูหลักของ Umbrel ให้คลิกที่ "App Store" (หรือ "App") ค้นหา **ThunderHub** ในรายการแอปพลิเคชันที่มีอยู่



![Écran d'installation de ThunderHub via Umbrel](assets/fr/01.webp)





- ติดตั้ง ThunderHub**: คลิกที่แอปพลิเคชัน ThunderHub จากนั้นคลิกที่ปุ่มติดตั้ง ยืนยันหากจำเป็น Umbrel จะดาวน์โหลดและติดตั้ง ThunderHub บนโหนดของคุณโดยอัตโนมัติ





- เปิดแอปพลิเคชัน**: เมื่อการติดตั้งเสร็จสมบูรณ์ (ใช้เวลาประมาณไม่กี่สิบวินาที) ThunderHub จะปรากฏบนหน้าแรกของคุณ คลิกที่ไอคอนเพื่อเปิด ThunderHub จะเปิดในเบราว์เซอร์ของคุณ



![Identifiants par défaut pour ThunderHub](assets/fr/02.webp)



**สำคัญ:** เมื่อเปิด ThunderHub ครั้งแรก ระบบจะแสดง **รหัสผ่านเริ่มต้น** ที่จำเป็นสำหรับการเข้าสู่ระบบโดยอัตโนมัติ ตัวเลือก "ไม่ต้องแสดงอีก" ช่วยให้คุณซ่อนการแสดงนี้สำหรับการเชื่อมต่อในอนาคต **เราขอแนะนำอย่างยิ่งให้คุณ:**




- บันทึกรหัสผ่านนี้ทันที**ในตัวจัดการรหัสผ่านของคุณ
- คัดลอก **เพื่อใช้ในขั้นตอนถัดไป**
- เลือก "อย่าแสดงอีกครั้ง" เมื่อบันทึกรหัสผ่านแล้ว



![Page de connexion de ThunderHub](assets/fr/03.webp)



จากนั้นคุณจะถูกนำไปยังหน้าล็อกอิน ซึ่งคุณต้องป้อนรหัสผ่านที่คุณคัดลอกไว้ในขั้นตอนก่อนหน้าเพื่อปลดล็อก Interface



![Interface de connexion complète à ThunderHub](assets/fr/04.webp)



Umbrel ดูแลการให้ข้อมูลการเชื่อมต่อ LND กับ ThunderHub (ใบรับรอง TLS, administration macaroon, ฯลฯ) ในเบื้องหลัง ดังนั้นคุณไม่จำเป็นต้องทำการกำหนดค่าเพิ่มเติมใดๆ เพียงไม่กี่คลิก คุณก็จะมี ThunderHub ทำงานบนโหนด Umbrel ของคุณแล้ว



### การติดตั้งแบบแมนนวล (โฮสต์เอง, ยกเว้น Umbrel)



สำหรับผู้ใช้นอก Umbrel (เช่น บนเซิร์ฟเวอร์ส่วนตัว, Raspberry Pi ที่ใช้ RaspiBlitz หรือการติดตั้ง *แบบสแตนด์อโลน*) การติดตั้ง ThunderHub ต้องการขั้นตอนเพิ่มเติมเล็กน้อย ด้านล่างนี้เราจะอธิบายการติดตั้งจากซอร์สและการกำหนดค่า ตาม [เอกสารทางการของ ThunderHub](https://docs.thunderhub.io)



#### การติดตั้ง



**ข้อกำหนดเบื้องต้น:** ตรวจสอบให้แน่ใจว่าระบบของคุณตรงตามข้อกำหนดขั้นต่ำตาม [การตั้งค่าเอกสาร](https://docs.thunderhub.io/setup):




- Node.js** เวอร์ชัน 18 หรือสูงกว่า
- npm** ติดตั้งแล้ว
- การเข้าถึงไฟล์การตรวจสอบสิทธิ์ LND :
  - LND ใบรับรอง TLS (`tls.cert`)
  - การจัดการ LND มาการูน (`admin.macaroon`)
  - LND gRPC service address (hostname:port) (default `127.0.0.1:10009` locally)



**1. ดึงรหัส ThunderHub:** โคลนที่เก็บ GitHub ของโปรเจ็กต์ตามที่อธิบายไว้ใน [เอกสารการติดตั้ง](https://docs.thunderhub.io/installation):



```bash
git clone https://github.com/apotdevin/thunderhub.git
cd thunderhub
```



**2. ติดตั้ง dependencies และสร้างแอปพลิเคชัน:**



```bash
npm install
npm run build
```



คำสั่งเหล่านี้จะติดตั้งโมดูลที่จำเป็นทั้งหมดและจากนั้นคอมไพล์แอปพลิเคชัน (ThunderHub เขียนด้วย TypeScript/React)



**3. อัปเดตภายหลัง:** ThunderHub มีวิธีการหลายอย่างสำหรับการอัปเดตในอนาคต:



```bash
# Méthode rapide
npm run update

# Ou méthode manuelle
git pull
npm install
npm run build
```



#### การกำหนดค่า (การตั้งค่า)



**1. Main configuration file:** สร้างไฟล์ `.env.local` ที่รูทของโฟลเดอร์ ThunderHub เพื่อปรับแต่งการตั้งค่า (วิธีนี้จะป้องกันไม่ให้การตั้งค่าของคุณถูกเขียนทับระหว่างการอัปเดต) ตัวแปรหลักตาม [เอกสารการตั้งค่า](https://docs.thunderhub.io/setup):



```bash
# -----------
# Server Configs
# -----------
LOG_LEVEL='info' # 'error' | 'warn' | 'info' | 'http' | 'verbose' | 'debug' | 'silly'
PORT=3000
NODE_ENV=production

# -----------
# Interface Configs
# -----------
THEME='dark' # 'dark' | 'light' | 'night'
CURRENCY='sat' # 'sat' | 'btc' | 'fiat'

# -----------
# Privacy Configs
# -----------
FETCH_PRICES=true # Récupération des prix BTC/fiat depuis Blockchain.com
FETCH_FEES=true # Récupération des frais on-chain depuis Earn.com
DISABLE_LINKS=false # Liens vers 1ml.com et Blockchain.com
NO_VERSION_CHECK=false # Vérification de version depuis GitHub

# -----------
# TOR (optionnel)
# -----------
TOR_PROXY_SERVER='socks://127.0.0.1:9050' # Pour proxifier via TOR

# -----------
# Account Configs
# -----------
ACCOUNT_CONFIG_PATH='/chemin/vers/thubConfig.yaml' # Fichier de comptes
```



**2. Server Accounts configuration:** สร้างไฟล์ YAML ที่ระบุใน `ACCOUNT_CONFIG_PATH` :



```yaml
masterPassword: 'votre_mot_de_passe_principal'
accounts:
- name: 'Mon Nœud LND'
serverUrl: '127.0.0.1:10009'
macaroonPath: '/home/user/.lnd/data/chain/bitcoin/mainnet/admin.macaroon'
certificatePath: '/home/user/.lnd/tls.cert'
password: 'mot_de_passe_compte_specifique'
# Optionnel : compte avec macaroon en hexadécimal
- name: 'Nœud Distant'
serverUrl: 'ip.distante:10009'
macaroon: '0201056c6e6402f8...' # Macaroon en HEX ou Base64
certificate: '0202045c7365...' # Certificat en HEX ou Base64
```



**3. Remote Access:** To connect to a remote LND node, add to `LND.conf` :



```bash
# Option 1 : accès par IP
tlsextraip=<ip-externe-accessible>
rpclisten=0.0.0.0:10009

# Option 2 : accès par domaine
tlsextradomain=<domaine-externe-accessible>
rpclisten=0.0.0.0:10009
```



**4. ความปลอดภัย:** เมื่อเริ่มต้นใช้งานครั้งแรก ThunderHub **ซ่อน** `masterPassword` และรหัสผ่านบัญชีทั้งหมดในไฟล์ YAML โดยอัตโนมัติเพื่อหลีกเลี่ยงการมีรหัสผ่านในรูปแบบข้อความธรรมดาบนเซิร์ฟเวอร์



**5. เริ่มต้น ThunderHub:**



```bash
npm start
```



โดยค่าเริ่มต้น เซิร์ฟเวอร์จะฟังที่พอร์ต 3000 เข้าถึง `http://localhost:3000` (หรือ `http://ip-serveur:3000` จากเครือข่ายภายใน)



**6. ทางเลือกของ Docker:** ThunderHub มีภาพ Docker อย่างเป็นทางการ:



```bash
# Image standard
docker pull apotdevin/thunderhub:latest
docker run --rm -it -p 3000:3000/tcp apotdevin/thunderhub:latest

# Image avec base path /thub
docker pull apotdevin/thunderhub:base-v0.11.1
```



หน้าเข้าสู่ระบบ ThunderHub ปรากฏขึ้น เลือกบัญชีที่กำหนดค่าไว้และป้อนรหัสผ่านเพื่อเข้าถึงแดชบอร์ด



**การติดตั้งบนดิสทริบิวชันอื่น ๆ:** ดิสทริบิวชันของโหนดที่บรรจุไว้ล่วงหน้า (RaspiBlitz, MyNode, Start9, เป็นต้น) โดยทั่วไปจะมีการรองรับ ThunderHub แบบเนทีฟผ่านทางอินเทอร์เฟซการจัดการที่เกี่ยวข้องของพวกเขา



**สำหรับข้อมูลเพิ่มเติม:** ปรึกษาเอกสารทางการฉบับสมบูรณ์ :




- การติดตั้ง:** [docs.thunderhub.io/installation](https://docs.thunderhub.io/installation)
- การกำหนดค่า:** [docs.thunderhub.io/setup](https://docs.thunderhub.io/setup)



ทรัพยากรเหล่านี้มีรายละเอียดเกี่ยวกับตัวเลือกขั้นสูง เช่น บัญชี SSO, มาการูนที่เข้ารหัส, การกำหนดค่า TOR และอื่น ๆ อีกมากมาย



เมื่อ ThunderHub ติดตั้งและเข้าถึงได้แล้ว คุณก็พร้อมที่จะใช้ประโยชน์จากคุณสมบัติทั้งหมดของมัน ในส่วนต่อไปนี้ เราจะมาดู Interface ThunderHub และแท็บต่างๆ เพื่อแนะนำคุณในการใช้งาน



## การนำเสนอ Interface



Interface ThunderHub ถูกจัดโครงสร้างรอบเมนูหลัก (มักจะแสดงในคอลัมน์ด้านซ้ายมือ) ซึ่งประกอบด้วยหลายส่วนสำคัญ แต่ละส่วนสอดคล้องกับแง่มุมของการจัดการโหนด Lightning ของคุณ เรามาดูทีละส่วน:





- หน้าแรก** - แท็บหน้าแรกพร้อมแดชบอร์ดทั่วไป (ภาพรวมของโหนดของคุณและการดำเนินการด่วน)
- Dashboard** - แดชบอร์ดที่ปรับแต่งได้ด้วยวิดเจ็ตและเมตริกขั้นสูง
- เพียร์** - การจัดการเพียร์ของ Lightning (การเชื่อมต่อกับโหนดอื่น ๆ)
- Channels** - การจัดการรายละเอียดของช่องทาง Lightning.
- Rebalance** - เครื่องมือปรับสมดุลช่องทาง (การชำระเงินแบบวงกลม)
- ธุรกรรม** - ประวัติการชำระเงินด้วย Lightning (ธุรกรรม LN)
- Forwards** - สถิติการส่งต่อ (การชำระเงินที่ส่งผ่านโหนดของคุณ)
- เชน** - โหนด On-Chain wallet (On-Chain BTC: UTXOs, ธุรกรรม).
- Amboss** - การผสานรวมกับ Amboss (การตรวจสอบโหนด, การสำรองข้อมูล, ฯลฯ)
- เครื่องมือ** - เครื่องมือต่างๆ (การสำรองข้อมูล, ข้อความที่ลงนาม, มาการูน, รายงาน, ฯลฯ)
- สลับ** - On-Chain/ฟังก์ชันการสลับ Lightning ผ่าน Boltz.
- สถิติ** - สถิติขั้นสูงและเมตริกประสิทธิภาพของโหนด



*(หมายเหตุ: ขึ้นอยู่กับรุ่นของ ThunderHub บางส่วนอาจมีหัวข้อที่แตกต่างกันเล็กน้อยหรือคุณสมบัติเพิ่มเติม แต่หลักการทั่วไปยังคงเหมือนเดิม)*



### หน้าแรก (แผงควบคุมทั่วไป)



แท็บ **Home** ของ ThunderHub คือหน้าแรกที่ปรากฏหลังจากที่คุณเข้าสู่ระบบ มันประกอบด้วย **General Dashboard** ซึ่งให้ภาพรวม **ทั่วโลก** ของสถานะของ Lightning node ของคุณและแนะนำ **การดำเนินการด่วน** สำหรับการปฏิบัติการตามปกติ นี่คือสิ่งที่คุณจะพบโดยทั่วไป:



![Tableau de bord principal de ThunderHub](assets/fr/05.webp)





- ยอดคงเหลือและความจุ:** ที่ด้านบนของหน้า ThunderHub จะแสดงยอดคงเหลือที่คุณมีอยู่ โดยปกติคุณจะเห็นยอดคงเหลือ On-Chain (Bitcoin On-Chain ในโหนด Wallet ซึ่งแสดงด้วยสัญลักษณ์สมอ ⚓) และยอดคงเหลือ Lightning (ความจุของช่องของคุณ ซึ่งแสดงด้วยสัญลักษณ์สายฟ้า ⚡) ซึ่งจะทำให้คุณทราบทันทีถึงเงินทุนที่คุณมีใน On-Chain และ Lightning หากคุณมีหลายบัญชีหรือหลายช่อง ให้แน่ใจว่าคุณอยู่ในบัญชีที่ถูกต้อง (เช่น Mainnet เทียบกับ Testnet)





- สถิติสำคัญ:** แดชบอร์ดสามารถแสดงเมตริกทั่วโลกบางอย่างสำหรับโหนดของคุณ - เช่น จำนวนช่องทางที่เปิดอยู่ จำนวนเพียร์ที่เชื่อมต่อ ค่าธรรมเนียมการกำหนดเส้นทางที่ได้รับ (ถ้ามี) เป็นต้น เป็นสรุปของกิจกรรมล่าสุดและสุขภาพของโหนด





- การดำเนินการด่วน:** แดชบอร์ดมีปุ่มสำหรับการดำเนินการที่พบบ่อยที่สุดอย่างรวดเร็ว โดยไม่ต้องผ่านเมนู การดำเนินการด่วนเหล่านี้รวมถึง:





- Ghost**: ตั้งค่าที่อยู่ Lightning แบบกำหนดเองผ่าน Amboss.
- บริจาค**: บริจาคผ่าน Lightning.
- เข้าสู่ระบบ/ไปที่**: เชื่อมต่อกับบัญชี Amboss ของคุณ (เชื่อมต่อด่วน) และไปที่ Amboss.space โดยตรงเพื่อดูข้อมูลของโหนดของคุณ
- Address**: ป้อนที่อยู่ Lightning เพื่อทำการชำระเงิน
- เปิด**: เปิดช่องทาง Lightning ใหม่ การคลิกจะเปิดแบบฟอร์มสำหรับป้อน URI ของโหนดระยะไกลที่ต้องการเปิดช่องทาง จำนวนเงิน และหากมี ให้ป้อนค่าธรรมเนียม On-Chain สูงสุดที่จะใช้
- ถอดรหัส**: ถอดรหัสใบแจ้งหนี้ Lightning หรือ LNURL เพื่อดูรายละเอียดก่อนการชำระเงิน
- LNURL**: ประมวลผล LNURLs สำหรับการชำระเงินหรือการถอนเงินด้วย Lightning
- LnMarkets Login**: เข้าสู่ระบบ LnMarkets เพื่อการซื้อขาย



การดำเนินการอย่างรวดเร็วเหล่านี้ช่วยให้คุณสามารถดำเนินการที่พบบ่อยที่สุดได้โดยตรงจากหน้าแรก โดยไม่ต้องนำทางผ่านแท็บต่างๆ ของ Interface



โดยสรุปแล้ว แดชบอร์ด ThunderHub ให้คุณได้ **ดูอย่างรวดเร็ว** เกี่ยวกับโหนดของคุณและให้คุณดำเนินการที่พบบ่อยที่สุด (ส่ง/รับ, เปิดช่องทาง) ด้วยการคลิกเพียงครั้งเดียว เป็นจุดเริ่มต้นที่เหมาะสมสำหรับการบริหารจัดการในแต่ละวัน



### แดชบอร์ด



ส่วน **แดชบอร์ด** แยกออกจากแท็บหน้าแรกและมีแดชบอร์ดที่ปรับแต่งได้ขั้นสูงมากขึ้น ส่วนนี้ช่วยให้คุณสร้างมุมมองที่ปรับแต่งได้ด้วยวิดเจ็ตเฉพาะตามความต้องการของคุณในฐานะผู้ดำเนินการโหนด





- วิดเจ็ตที่ปรับแต่งได้:** ต่างจากหน้าแรกที่มีรูปแบบคงที่ แดชบอร์ดให้คุณเลือกได้อย่างอิสระว่าจะให้แสดงองค์ประกอบใดบ้างและจัดเรียงอย่างไร



![Dashboard sans widgets activés](assets/fr/06.webp)



หากไม่มีวิดเจ็ตใดเปิดใช้งาน คุณจะเห็นข้อความ "No Widgets Enabled!" พร้อมปุ่ม "Settings" เพื่อเข้าถึงพารามิเตอร์การปรับแต่ง



![Paramètres des widgets du Dashboard](assets/fr/07.webp)



ในหน้าการตั้งค่า คุณสามารถเลือกจากวิดเจ็ตหลากหลายประเภทที่จัดเป็นหมวดหมู่: "Lightning - Info", "Lightning - Table", "Lightning - Graph" และอื่นๆ แต่ละวิดเจ็ตสามารถเปิดหรือปิดการใช้งานได้ด้วยปุ่ม "แสดง/ซ่อน"



![Bas de la page des paramètres](assets/fr/08.webp)



ที่ด้านล่างของการตั้งค่า คุณจะพบปุ่ม "ไปที่แดชบอร์ด" เพื่อกลับไปยังแดชบอร์ดและปุ่ม "รีเซ็ตวิดเจ็ต" เพื่อรีเซ็ตการแสดงผลเริ่มต้น



![Dashboard personnalisé avec widgets](assets/fr/09.webp)



เมื่อกำหนดค่าแล้ว แดชบอร์ดของคุณสามารถแสดงกราฟและเมตริกต่างๆ ได้: กราฟการชำระเงิน Lightning, จำนวนใบแจ้งหนี้ที่ออก, สถิติการส่งต่อ, ยอดคงเหลือละเอียด, เป็นต้น





- เมตริกขั้นสูง:** เข้าถึงสถิติโดยละเอียดเพิ่มเติมเกี่ยวกับประสิทธิภาพของโหนดของคุณ พร้อมกราฟและข้อมูลแบบเรียลไทม์





- ภาพรวมที่ปรับแต่งได้:** ปรับแต่งการแสดงผลให้เหมาะสมไม่ว่าคุณจะเป็นผู้ใช้ทั่วไปหรือผู้ปฏิบัติงานมืออาชีพที่จัดการช่องทางการกำหนดเส้นทางหลายช่องทาง





- โมดูลาร์ Interface:** เพิ่มหรือลบวิดเจ็ตตามต้องการ: แผนภูมิการส่งต่อ, เมตริกสภาพคล่อง, การแจ้งเตือนสุขภาพของโหนด, เป็นต้น



ส่วนนี้มีประโยชน์เป็นพิเศษสำหรับผู้ใช้ขั้นสูงที่ต้องการติดตามเมตริกเฉพาะหรือรับภาพรวมทางเทคนิคเพิ่มเติมของโหนด Lightning ของพวกเขา มันช่วยเสริมส่วนหน้าแรกโดยการเสนอความยืดหยุ่นและการควบคุมที่มากขึ้นเกี่ยวกับวิธีการแสดงข้อมูล



### เพื่อนร่วมงาน



ส่วน **Peers** แสดงรายการโหนด Lightning ทั้งหมดที่เชื่อมต่อกับโหนดของคุณในฐานะเพียร์ **เพียร์** คือการเชื่อมต่อโหนดต่อโหนดโดยตรงบนเครือข่าย Lightning โหนดของคุณสามารถเชื่อมต่อกับเพียร์ได้แม้ไม่มีช่องทางเปิด (เช่น เพียงเพื่อแลกเปลี่ยนข้อมูลซุบซิบบนเครือข่าย) หรือแน่นอนว่าทุกช่องทางที่เปิดอยู่จะหมายถึงเพียร์ที่เชื่อมต่อโดยอัตโนมัติ



![Vue de l'onglet Peers avec les pairs connectés](assets/fr/10.webp)



ในแท็บเพื่อน คุณจะเห็น :





- คอลัมน์ข้อมูล:** Interface แสดงรายละเอียดที่มีประโยชน์ เช่น สถานะการซิงโครไนซ์, ประเภทการเชื่อมต่อ (clearnet หรือ Tor), ping, satoshis ที่ได้รับ/ส่ง และปริมาณข้อมูลที่แลกเปลี่ยน
- เพิ่มเพียร์: ThunderHub ให้คุณเชื่อมต่อกับเพียร์ใหม่ด้วยตนเองผ่านปุ่ม **"Add"** ที่มุมขวาบน คุณจะต้องป้อน URI ของโหนด (รูปแบบ `<public_key>@<socket>`) เมื่อได้รับการตรวจสอบแล้ว ThunderHub จะส่งคำสั่ง `lncli connect` ที่สอดคล้องกัน หากโหนดออนไลน์และเข้าถึงได้ มันจะถูกเพิ่มไปยังรายการเพียร์ของคุณ



### ช่องทาง



แท็บ **ช่องทาง** เป็นหัวใจของการจัดการช่องทาง Lightning นี่อาจเป็นส่วนที่คุณจะเข้ามาดูบ่อยที่สุด มันแสดง **ช่องทาง Lightning ทั้งหมดของคุณ** พร้อมรายละเอียด และให้คุณดำเนินการจัดการในช่องทางเหล่านี้ได้



![Vue d'ensemble des channels ouverts](assets/fr/11.webp)



นี่คือสิ่งที่คุณจะพบในหน้าช่อง:





- มุมมองรายการช่อง:** แต่ละช่องที่เปิด (หรือกำลังเปิด/ปิด) จะถูกแสดงรายการ โดยปกติจะมีนามแฝงของโหนดระยะไกล ความจุช่องทั้งหมด และแถบสีที่แสดงการกระจายของสภาพคล่องในท้องถิ่นเทียบกับระยะไกล ThunderHub ใช้รหัสสี (มักเป็นสีน้ำเงิน/เขียว) หรือเปอร์เซ็นต์เพื่อระบุยอดคงเหลือของช่อง: ตัวอย่างเช่น สีน้ำเงินสำหรับส่วนแบ่งในท้องถิ่นของคุณ สีเขียวสำหรับส่วนแบ่งระยะไกล หากช่องมีความสมดุลอย่างสมบูรณ์ (50/50) แถบจะเป็นครึ่งหนึ่งของแต่ละสี สิ่งนี้ช่วยให้คุณระบุได้อย่างรวดเร็วว่าช่องใดไม่สมดุล (สีน้ำเงินทั้งหมด = เกือบทั้งหมดในท้องถิ่น, สีเขียวทั้งหมด = เกือบทั้งหมดระยะไกล)





- คอลัมน์ข้อมูล:** Interface แสดงคอลัมน์รายละเอียดรวมถึง สถานะ, การดำเนินการที่มี, ข้อมูลเพื่อน, รหัสช่อง, ความจุ, กิจกรรม, ค่าธรรมเนียม และยอดคงเหลือ พร้อมการแสดงสภาพคล่องแบบกราฟิก.





- การกำหนดค่าการแสดงผล:** สัญลักษณ์รูปเฟืองที่มุมขวาบนช่วยให้คุณปรับแต่งการแสดงผลของช่องให้เหมาะกับความต้องการของคุณ





- สถานะ:** คุณจะเห็นตัวบ่งชี้สถานะด้วย - เช่น `Active` (ช่องเปิดและใช้งานได้), `Offline` (เพียร์ถูกตัดการเชื่อมต่อ ดังนั้นช่องจึงไม่สามารถใช้งานได้ชั่วคราว), `Pending` (สำหรับการเปิดหรือปิดที่รอการยืนยัน On-Chain)





- การดำเนินการบนช่องทาง:** สำหรับแต่ละช่องทาง ThunderHub มีปุ่มการดำเนินการ (มักอยู่ในรูปแบบของไอคอน):



![Actions de gestion des canaux - Modifier et Fermer](assets/fr/12.webp)





- ค่าธรรมเนียมการแก้ไข:** Interface "Update Channel Policy" ช่วยให้คุณปรับพารามิเตอร์ของช่องทั้งหมด: ค่าธรรมเนียมพื้นฐาน, อัตราค่าธรรมเนียม (ใน ppm), CLTV Delta, Max HTLC และ Min HTLC สิ่งนี้ช่วยให้คุณปรับนโยบายค่าธรรมเนียมของคุณเป็นรายช่องได้อย่างอิสระ โดยมีเป้าหมายเพื่อดึงดูด (หรือกีดกัน) การจราจรในการกำหนดเส้นทาง *(หมายเหตุ: ThunderHub ไม่ใช่เครื่องมือจัดการค่าธรรมเนียมอัตโนมัติ แต่สำหรับการปรับด้วยตนเองนั้นมีประสิทธิภาพมาก)*
- ปิดช่องทาง (**Close**): Interface "ปิดช่องทาง" ให้คุณเลือกระหว่าง **ปิดแบบร่วมมือ** (ค่าเริ่มต้น) หรือ **ปิดแบบบังคับ** (**Force Close**) โดยกำหนดค่าธรรมเนียม (ใน Sats/vByte) **สำคัญ:** ควรเลือกปิดแบบร่วมมือเมื่อเป็นไปได้เสมอ เพื่อหลีกเลี่ยงความล่าช้าในการชำระ On-Chain และค่าธรรมเนียมที่สูงขึ้น ThunderHub จะแจ้งให้คุณทราบว่าเพียร์ออนไลน์อยู่หรือไม่ (สามารถปิดแบบร่วมมือได้) ในกรณีของการปิดแบบบังคับ โปรดตรวจสอบให้แน่ใจเนื่องจากไม่สามารถย้อนกลับได้และจะทำให้เกิดการทำธุรกรรมกวาดล้างพร้อมกับการล็อคเวลา (โดยทั่วไป 144 บล็อกหรือประมาณ 1 วันบน Bitcoin Mainnet)
- เปิดช่องใหม่:** ในการเปิดช่องใหม่ ให้คลิกที่รูปเฟืองที่มุมขวาบนของหน้าช่อง จากนั้นเลือก "เปิด" คุณสามารถเริ่มช่องไปยังเพื่อนใหม่หรือเพื่อนที่มีอยู่แล้ว ข้อดีของการใช้หน้านี้คือคุณมีรายการช่องที่มีอยู่แล้วอยู่ตรงหน้าคุณ ซึ่งสามารถช่วยให้คุณตัดสินใจว่าจะเปิดช่องใหม่ที่ไหน



โดยสรุป ส่วนของช่องทางจะให้คุณ **ควบคุมแต่ละช่องทางได้อย่างละเอียด** นี่คือที่ที่คุณจัดการการจัดสรรสภาพคล่อง ตัดสินใจว่าจะเก็บหรือปิดช่องทางใด และตั้งค่าพารามิเตอร์การกำหนดเส้นทางต่อช่องทาง ThunderHub เสนอ Interface ที่ชัดเจนสำหรับการดำเนินการจัดการโหนดที่สำคัญเหล่านี้



### ปรับสมดุลใหม่



แท็บ **Rebalance** ถูกออกแบบมาเพื่อการ **ปรับสมดุลช่องทาง** การปรับสมดุล (หรือ *การปรับสมดุลใหม่*) เกี่ยวข้องกับการปรับการกระจายของเงินทุนระหว่างช่องทางขาออกและขาเข้าของคุณ โดยการทำ **การชำระเงินแบบวงกลม** จากช่องทางหนึ่งของคุณไปยังอีกช่องทางหนึ่งของคุณ ผ่านเครือข่าย Lightning ซึ่งช่วยให้คุณสามารถย้ายสภาพคล่องจากช่องทางที่เต็มเกินไปไปยังช่องทางที่ว่างเกินไปได้โดยไม่ต้องนำเงินทุนใหม่เข้ามา ทำให้ช่องทางของคุณมีประโยชน์มากขึ้น (ช่องทางที่สมดุลสามารถส่งและรับการชำระเงินได้)



![Interface de rebalance des canaux](assets/fr/13.webp)



ThunderHub ช่วยให้การดำเนินการนี้ง่ายขึ้นมาก ซึ่งหากไม่ใช้จะเป็นงานที่น่าเบื่อบนบรรทัดคำสั่ง นี่คือวิธีการใช้ส่วน Rebalance:





- มุมมองช่องเริ่มต้น:** เมื่อเข้าสู่ Rebalance, ThunderHub จะแสดงรายการช่องของคุณ พร้อมตัวบ่งชี้สมดุลสำหรับแต่ละช่อง (คล้ายกับที่อยู่ในหน้าช่อง) คุณสามารถเห็นได้ทันทีว่าช่องใดไม่สมดุล ThunderHub สามารถจัดเรียงช่องตามลำดับของการเพิ่มสมดุล เพื่อให้ช่องที่ไม่สมดุลที่สุดโดดเด่นอยู่ด้านบนสุดของรายการ (0.0 หมายถึงท้องถิ่นหรือระยะไกลทั้งหมด)





- การเลือกเพียร์:** Interface ทำให้ง่ายต่อการเลือกเพียร์ขาออกและขาเข้าสำหรับการปรับสมดุลใหม่





- การตั้งค่าพารามิเตอร์:** คุณสามารถตั้งค่า :
  - **ค่าธรรมเนียมสูงสุด** (ใน Sats และ ppm) ที่คุณยินดีจ่าย
  - จำนวนเงินที่จะปรับสมดุลด้วยตัวเลือก "คงที่" หรือ "เป้าหมาย"
- โหนดที่ควรหลีกเลี่ยง** เมื่อทำการกำหนดเส้นทาง
- เวลาทดลองสูงสุด**สำหรับการค้นหาเส้นทาง





- เลือก **ช่องทางต้นทาง**: เริ่มต้นโดยการเลือก **ช่องทางขาออก (ต้นทาง)** ซึ่งเป็นช่องทางที่คุณมีสภาพคล่องในท้องถิ่นมากเกินไปที่จะย้าย ในทางปฏิบัติ นี่คือช่องทางที่ส่วนแบ่งในท้องถิ่นของคุณสูง (> 50%) ลองจินตนาการถึงช่องทาง A ที่มี 1,000,000 Sats โดยที่ 900,000 เป็นของท้องถิ่น - เป็นตัวเลือกที่ดีสำหรับการส่ง Sats ไปที่อื่น โดยการคลิกที่ช่องทาง A นี้เป็น "ขาออก" ThunderHub จะทำเครื่องหมายว่าเป็นต้นทาง





- เลือก **ช่องเป้าหมาย**: ต่อไป, เลือกช่อง **ขาเข้า (เป้าหมาย)** ที่ต้องการรับสภาพคล่อง โดยทั่วไปแล้วจะเป็นช่องที่ตรงกันข้าม - ส่วนใหญ่ของเงินทุนจะอยู่ฝั่งไกล (เช่น มีเพียง 100,000 Satss ในท้องถิ่นจาก 1,000,000) ThunderHub, เมื่อได้เลือกช่องต้นทางแล้ว, จะจัดเรียงช่องอื่น ๆ ในลำดับย้อนกลับ (ยอดคงเหลือลดลง) เพื่อช่วยระบุช่องที่เสริมกันมากที่สุด เลือกช่อง B ที่มีพื้นที่ในฝั่งท้องถิ่น ThunderHub จะทำการแสดงอย่างชัดเจนว่าช่องสองช่องใดที่ได้เลือกแล้ว (ต้นทาง A และเป้าหมาย B)





- ตั้งค่าจำนวนค่าธรรมเนียมและความคลาดเคลื่อน:** แบบฟอร์มอนุญาตให้คุณป้อน :





  - **จำนวน**ที่จะปรับสมดุลใหม่ (ใน Sats) บ่อยครั้งที่เราเลือกจำนวนที่เท่ากับสิ่งที่จะทำให้ทั้งสองช่องทางสมดุลที่ \~50/50 ThunderHub สามารถเติมความจุส่วนเกินของช่องทางต้นทางได้ครึ่งหนึ่ง ตัวอย่างเช่น
  - **ค่าธรรมเนียมสูงสุด** ที่คุณยินดีจ่ายสำหรับการดำเนินการนี้ (ไม่บังคับ) ค่าธรรมเนียมนี้แสดงในรูปแบบ Sats (ต้นทุนรวมของการกำหนดเส้นทางแบบวงกลม) หากคุณปล่อยว่างไว้ ThunderHub จะค้นหาเส้นทางโดยไม่คำนึงถึงค่าใช้จ่าย ซึ่งโดยทั่วไปไม่แนะนำ (ควรกำหนดขีดจำกัด เช่น 10 Sats สำหรับการปรับสมดุลเล็กน้อย หรือ ppm สูงสุด)





- ค้นหาเส้นทาง:** คลิกที่ปุ่มเพื่อค้นหาเส้นทาง ThunderHub จะสอบถาม LND เพื่อคำนวณเส้นทางจากช่องทางต้นทางของคุณผ่านเครือข่ายไปยังช่องทางเป้าหมายของคุณเอง หากพบเส้นทางที่เป็นไปได้ที่ตรงตามเกณฑ์ค่าธรรมเนียมของคุณ มันจะแสดงรายละเอียดของการกระโดดและค่าใช้จ่ายค่าธรรมเนียม ตัวอย่างเช่น อาจระบุว่าพบเส้นทาง 3 กระโดดที่มีค่าใช้จ่ายรวม 2 Sats ในค่าธรรมเนียม





- เริ่มปรับสมดุล:** หากคุณพอใจกับเส้นทางที่เสนอ ให้คลิกที่ **ปรับสมดุลช่องทาง** จากนั้น ThunderHub จะเริ่มการชำระเงินแบบวงกลมผ่าน LND หากการชำระเงินสำเร็จ คุณจะเห็นการแจ้งเตือนความสำเร็จ และช่องทาง A และ B จะมีการปรับสมดุลในเวลาจริง ThunderHub จะอัปเดตตัวบ่งชี้สมดุลสำหรับช่องทางเหล่านี้ (ในอุดมคติจะมีสีเขียวมากขึ้นกว่าเดิม ซึ่งบ่งบอกถึงสมดุลที่ดีขึ้น)





- การปรับและการทำซ้ำ:** หากไม่พบเส้นทางในครั้งแรก (หรือหากมีค่าใช้จ่ายสูงเกินไป) คุณสามารถปรับพารามิเตอร์:





  - ลองจำนวนที่น้อยลง (บางครั้งการปรับสมดุลบางส่วนสามารถทำได้ในขณะที่จำนวนมากล้มเหลว)
  - เพิ่มค่าธรรมเนียมสูงสุดอย่างค่อยเป็นค่อยไป แต่ระวังอย่าจ่ายค่าธรรมเนียมมากเกินกว่าที่มันคุ้มค่า
  - ใช้ปุ่ม **รับเส้นทางอื่น** หากมี เพื่อทดลองเส้นทางอื่น
  - ลองใช้ช่องทางอื่น ๆ หากสถานการณ์เริ่มตึงเครียดจริง ๆ



ThunderHub ทำให้กระบวนการนี้เป็นเรื่อง **ง่ายและมองเห็นได้ชัดเจน** เพียงแค่ 4 ขั้นตอน (เลือกช่องทางขาออก, เลือกช่องทางขาเข้า, จำนวนเงิน, ยืนยัน) คุณสามารถทำสิ่งที่เคยต้องใช้คำสั่งที่ซับซ้อนด้วยตนเองได้ เครื่องมือนี้มีคุณค่าอย่างยิ่งในการรักษาโหนดให้สมดุลดีขึ้น ปรับปรุงประสิทธิภาพการกำหนดเส้นทางและประสบการณ์ของคุณ (ง่ายต่อการส่งและรับการชำระเงินผ่านทุกช่องทางของคุณ)



สุดท้ายนี้ โปรดทราบว่าการปรับสมดุลจะมีค่าใช้จ่ายในการกำหนดเส้นทาง (จ่ายให้กับโหนดกลาง) ดังนั้นจึงเป็น **การลงทุน** เพื่อทำให้โหนดของคุณมีความคล่องตัวมากขึ้น ใช้อย่างชาญฉลาด เช่น เพื่อสนับสนุนช่องทางไปยังบริการที่คุณใช้บ่อย (สภาพคล่องขาเข้า) หรือเพื่อปรับสมดุลช่องทางการกำหนดเส้นทางขนาดใหญ่ ThunderHub ช่วยให้คุณทำสิ่งนี้ได้อย่าง **ง่ายดายและมีประสิทธิภาพ**



### ธุรกรรม



ส่วน **ธุรกรรม** ใน ThunderHub สอดคล้องกับประวัติธุรกรรม **Lightning** ของโหนดของคุณ เช่น การชำระเงินและใบแจ้งหนี้ที่ชำระหรือรับผ่านช่องทางต่างๆ มันเป็นเหมือนใบแจ้งยอดบัญชีสำหรับการดำเนินงาน LN ของคุณ



![Historique des transactions Lightning](assets/fr/14.webp)



ในแท็บนี้คุณจะพบ:





- Invoice กราฟ:** ในมุมขวาบนของกราฟ แสดงการพัฒนาของใบแจ้งหนี้ที่ได้รับตามเวลา ช่วยให้คุณมองเห็นกิจกรรมของโหนดของคุณ





- รายการตามลำดับเวลาของธุรกรรม Lightning ทั้งหมดที่ทำ *จาก* หรือ *ไปยัง* โหนดของคุณ แต่ละรายการสามารถแสดง:





  - ประเภทของการดำเนินการ: **ส่งการชำระเงิน** (การชำระเงินขาออก) หรือ **รับการชำระเงิน** (ขาเข้า, ผ่าน Invoice ที่ชำระแล้ว)
  - จำนวนใน Sats.
  - วันที่/เวลา
  - รหัสการชำระเงิน (Hash หรือ RHash pre-image) หรือความคิดเห็น (หากคุณเพิ่มบันทึกช่วยจำใน Invoice)
  - สถานะ: **เสร็จสมบูรณ์**, หรืออาจจะ **กำลังดำเนินการ**/*ล้มเหลว* (เช่น การชำระเงินที่รอการแก้ไข แต่โดยทั่วไปแล้ว LND จะดำเนินการอย่างรวดเร็ว ดังนั้นจึงมีสถานะ "รอดำเนินการ" น้อยกว่าที่นี่เมื่อเทียบกับธุรกรรม On-Chain)



โดยสรุป ส่วนของธุรกรรมทำหน้าที่เป็น **บันทึกกิจกรรม LN** ของคุณ มีประโยชน์มากในการตรวจสอบว่าการชำระเงินได้ดำเนินการไปแล้วหรือไม่ มีค่าธรรมเนียมเท่าใด หรือการติดตามประวัติการแลกเปลี่ยน Lightning ของคุณ เมื่อใช้ร่วมกับส่วนของการส่งต่อ (ที่จะอธิบายต่อไป) คุณจะมีมุมมองที่ครบถ้วนเกี่ยวกับเงินที่ผ่านเข้ามาในโหนดของคุณ



### ส่งต่อ



แท็บ **Forwards** อุทิศให้กับกิจกรรม **routing** ของโหนดของคุณ เช่น การชำระเงินที่ **transit** ผ่านช่องทางของคุณ (เมื่อคุณทำหน้าที่เป็นโหนดตัวกลางบนเครือข่าย Lightning) หากคุณดำเนินการโหนดของคุณเป็นโหนด routing นี่เป็นส่วนสำคัญสำหรับการติดตามประสิทธิภาพของคุณ



![Statistiques de routage Lightning](assets/fr/15.webp)



ใน Forwards, ThunderHub นำเสนอ :





- ตัวกรองและตัวเลือกการแสดงผล:** ที่มุมขวาบน ตัวกรองช่วยให้คุณจัดเรียงข้อมูลตามวัน/สัปดาห์/เดือน/ปี และเลือกการแสดงผลระหว่างแบบกราฟิกหรือแบบตาราง





- ข้อความกิจกรรม:** หากไม่มีการกำหนดเส้นทางในช่วงเวลาที่เลือก Interface จะแสดง "ไม่มีการส่งต่อสำหรับช่วงเวลานี้" ตามที่แสดงในตัวอย่างนี้





- **ตารางการส่งต่อล่าสุด**: แต่ละรายการสอดคล้องกับการชำระเงินที่ถูก **ส่งต่อ** ผ่านโหนดของคุณ สำหรับการส่งต่อแต่ละครั้ง เรามักจะเห็น:





  - timestamp,
  - ปริมาณที่ส่งผ่าน (ใน Sats),
  - **ค่าธรรมเนียมที่ได้รับ** จากการส่งต่อครั้งนี้ (ใน Sats นี่คือความแตกต่างระหว่างสิ่งที่คุณได้รับจากช่องทางขาเข้าและส่งออกในช่องทางขาออก)
  - ช่องทางขาเข้าและขาออกที่ใช้ (มักจะระบุโดยชื่อเล่นของเพื่อนหรือรหัสช่องทาง)
  - สถานะ (ปกติจะเป็น *completed* หรือ failure หากการส่งต่อไม่สำเร็จระหว่างทาง)





- สถิติรวม**: ThunderHub คำนวณและแสดงยอดรวมและสถิติที่ด้านบนของหน้าในช่วงเวลาที่กำหนด (เช่น 24 ชั่วโมงที่ผ่านมา หรือ 7 วัน เป็นต้น ซึ่งบางครั้งสามารถกำหนดค่าได้)



โดยสรุป ส่วนของ Forwards นำเสนอ **ภาพรวมแบบเรียลไทม์ของกิจกรรมการกำหนดเส้นทางของโหนด Lightning ของคุณ** เมื่อรวมกับส่วนของ Channels และ Rebalance จะเป็นแพ็คเกจที่สมบูรณ์สำหรับการเพิ่มประสิทธิภาพโหนดของคุณ: Channels/Rebalance สำหรับสภาพคล่อง, Forwards สำหรับการสังเกตผลลัพธ์ (การไหลและกำไร)



### โซ่



ส่วน **Chain** สอดคล้องกับการจัดการ Bitcoin On-Chain wallet ของโหนด LND ของคุณ Interface นี้ช่วยให้คุณสามารถดูและจัดการกองทุน Bitcoin ซึ่งใช้ในการเปิดช่องทางหรือรับกองทุนจากช่องทางที่ปิดแล้ว



![Gestion du portefeuille on-chain](assets/fr/16.webp)



ใน Chain, คุณจะพบ:





- ยอดคงเหลือ On-Chain:** แสดงยอดคงเหลือ BTC ทั้งหมดที่มีอยู่ใน Wallet LND.





- รายการ UTXOs:** ดูเอาต์พุตที่ยังไม่ได้ใช้ทั้งหมด (UTXO) พร้อมจำนวนเงิน การยืนยัน ที่อยู่ และรูปแบบสำหรับแต่ละเอาต์พุต





- ประวัติการทำธุรกรรม:** ตารางรายละเอียดของธุรกรรม Bitcoin ทั้งหมดพร้อมประเภท (เข้า/ออก), วันที่, จำนวนเงิน, ค่าธรรมเนียม, การยืนยัน, บล็อกที่รวม, ที่อยู่ และ txid.



### Amboss



ThunderHub ผสานรวมกับแพลตฟอร์ม **Amboss** (amboss.space) ซึ่งมีข้อมูลรายละเอียดเกี่ยวกับโหนด Lightning ตลาดสภาพคล่อง และฟีเจอร์ที่มีประโยชน์ เช่น การสำรองข้อมูลช่องทางที่เข้ารหัสและการตรวจสอบความพร้อมใช้งาน



![Intégration Amboss avec ses options](assets/fr/17.webp)



ใน ThunderHub ส่วน Amboss อนุญาตให้คุณ **เชื่อมโยง** โหนดของคุณกับบัญชี Amboss ของคุณ:





- Ghost Address:** ตั้งค่าที่อยู่ Lightning **ส่วนบุคคลสำหรับโหนดของคุณ เพื่ออำนวยความสะดวกในการรับชำระเงิน





- การสำรองข้อมูลช่องอัตโนมัติ:** ฟีเจอร์หลักสำหรับการสำรองข้อมูลช่องแบบเข้ารหัส** (ไฟล์ SCB) บน Amboss เปิดใช้งาน **Amboss Auto Backup = Yes** ในการตั้งค่าเพื่อส่งการอัปเดตการสำรองข้อมูลแบบเข้ารหัสโดยอัตโนมัติทุกครั้งที่คุณเปลี่ยนช่อง ในกรณีที่เกิดความล้มเหลว คุณจะสามารถกู้คืนเงินของคุณได้ด้วยการสำรองข้อมูลภายนอกนี้





- การตรวจสุขภาพ:** เปิดใช้งาน **Amboss Healthcheck = Yes** เพื่อให้โหนดของคุณส่งสัญญาณพิงเป็นประจำไปยัง Amboss คุณจะได้รับการแจ้งเตือนหากโหนดของคุณดูเหมือนจะออฟไลน์





- คุณสมบัติอื่นๆ:** การผลักดันยอดคงเหลืออัตโนมัติ, **การผสานรวม Magma/Hydro** (ตลาดสภาพคล่อง), และการเข้าถึงสถิติประสิทธิภาพโดยละเอียด.



การผสานรวม Amboss เพิ่ม **ชั้นความปลอดภัย** ที่สำคัญด้วยการสำรองข้อมูลภายนอกอัตโนมัติและการตรวจสอบความพร้อมใช้งาน ซึ่งสามารถเข้าถึงได้โดยตรงจาก ThunderHub



### เครื่องมือ



ส่วน **เครื่องมือ** รวมเครื่องมือต่างๆ ที่มีความก้าวหน้าสำหรับการจัดการโหนดของคุณ นี่คือองค์ประกอบหลัก:



![Interface des outils disponibles](assets/fr/18.webp)





- การสำรองข้อมูล:** จัดการการสำรองข้อมูลช่องของคุณด้วยตนเอง (SCB) ThunderHub ให้คุณ **ดาวน์โหลดไฟล์สำรองข้อมูลทั้งหมด** ของช่องของคุณ (ตัวเลือก "สำรองข้อมูลทุกช่อง -> ดาวน์โหลด") เก็บไฟล์ `channel-all.bak` นี้ไว้ในที่ปลอดภัย - มันสำคัญสำหรับการกู้คืนเงินของคุณในกรณีที่เกิดความผิดพลาด คุณยังสามารถ **นำเข้า** ไฟล์สำรองข้อมูลเมื่อทำการปรับใช้โหนดใหม่ได้อีกด้วย





- การบัญชี:** เครื่องมือส่งออกสำหรับรายงานทางการเงินรวมถึงค่าธรรมเนียมที่ได้รับ/จ่ายและปริมาณที่ส่งผ่านในช่วงเวลาที่กำหนด
- ข้อความที่ลงนาม:** ลงนามหรือยืนยันข้อความด้วยโหนดของคุณเพื่อพิสูจน์ความเป็นเจ้าของโหนด Lightning ของคุณผ่านลายเซ็นทางเข้ารหัสลับ
- มาการอง (ส่วนเบเกอรี่):** จัดการ LND** มาการองเพื่อสร้างการเข้าถึงที่ปรับแต่งได้ Interface "เบเกอรี่" ช่วยให้คุณสามารถเลือกสิทธิ์แต่ละรายการได้อย่างแม่นยำ: "เพิ่มหรือลบเพื่อน", "สร้างที่อยู่เชน", "สร้างใบแจ้งหนี้", "สร้างมาการอง", "สร้างคีย์", "รับคีย์การเข้าถึง", "รับธุรกรรมเชน", "รับใบแจ้งหนี้", "รับข้อมูล Wallet", "รับการชำระเงิน", "รับเพื่อน", "ชำระใบแจ้งหนี้", "เพิกถอนรหัสการเข้าถึง", "ส่งไปยังที่อยู่เชน", "ลงนามไบต์", "ลงนามข้อความ", "หยุด daemon", "ตรวจสอบลายเซ็นไบต์", "ตรวจสอบข้อความ" และอื่นๆ แต่ละสิทธิ์สามารถเปิดใช้งานได้ทีละรายการด้วยปุ่ม "ใช่/ไม่ใช่" เพื่อสร้างมาการองที่ปรับแต่งได้ตามต้องการ
- ข้อมูลระบบ:** การแสดงเวอร์ชัน Wallet และ RPCs ที่เปิดใช้งานแล้ว



โดยสรุป ส่วนเครื่องมือรวบรวมฟังก์ชันการจัดการขั้นสูง - การสำรองข้อมูล การบัญชี การรักษาความปลอดภัย และการจัดการการเข้าถึง - ไว้ใน Interface ที่รวมเป็นหนึ่งเดียว



### สวอป



แท็บ **Swap** ของ ThunderHub ช่วยให้คุณแลกเปลี่ยน Lightning satoshis เป็น Bitcoin On-Chain ผ่านบริการ Boltz ฟีเจอร์นี้มีประโยชน์สำหรับการ "ทิ้ง" สภาพคล่องของ Lightning ที่เกินไปยังช่องโดยไม่ต้องปิดช่อง



![Interface de swap via Boltz](assets/fr/19.webp)



กระบวนการนั้นง่าย:





- จำนวน**: กำหนดจำนวนที่จะแลกเปลี่ยน
- Address**: ป้อนที่อยู่รับ Bitcoin
- การดำเนินการ**: ThunderHub สื่อสารกับ Boltz เพื่อประมวลผลการแลกเปลี่ยนโดยอัตโนมัติ



**ประโยชน์:**




- บริการที่ไม่รับฝากทรัพย์สิน (ไม่มีการรับฝากเงินสด)
- รักษาช่องทางที่มีอยู่ของคุณ
- ใช้งานง่ายรวม Interface



Boltz เรียกเก็บค่าคอมมิชชั่นเล็กน้อยและคุณจ่ายค่าธรรมเนียมการทำธุรกรรมมาตรฐาน Bitcoin ThunderHub แสดงค่าใช้จ่ายทั้งหมดก่อนการยืนยัน



### สถิติ



ส่วน **สถิติ** ของ ThunderHub นำเสนอ **สถิติขั้นสูง** เกี่ยวกับโหนด Lightning ของคุณเพื่อวิเคราะห์ประสิทธิภาพและเพิ่มประสิทธิภาพการกำหนดเส้นทาง



![Statistiques du nœud via Amboss](assets/fr/20.webp)



ส่วนนี้มีความสำคัญสำหรับการเพิ่มประสิทธิภาพค่าใช้จ่ายของคุณ การระบุช่องทางที่ประสบความสำเร็จ และการวางแผนการขยายตัวของโหนดของคุณ



## บทสรุป



**ThunderHub** ได้กลายเป็นเครื่องมือที่จำเป็นสำหรับการจัดการโหนด Lightning **LND** อย่างง่ายดาย Interface ที่ทันสมัยนี้มีฟังก์ชันที่จำเป็นทั้งหมด: การจัดการช่องทาง, การชำระเงิน, การตรวจสอบ, พร้อมคุณสมบัติขั้นสูงเช่น การปรับสมดุลอัตโนมัติและการรวม Amboss



**ประโยชน์หลัก:**




- Interface เพรียวบางและใช้งานง่าย
- เครื่องมือทรงพลัง (rebalance, Boltz swaps, automatic backups)
- เข้ากันได้กับ Umbrel, Voltage, RaspiBlitz และการแจกจ่ายอื่น ๆ



ThunderHub ทำให้การจัดการโหนด Lightning ขั้นสูงเป็นเรื่องที่ทุกคนเข้าถึงได้ ซึ่งก่อนหน้านี้ต้องใช้คำสั่งทางเทคนิคที่ซับซ้อน ไม่ว่าคุณจะเป็นผู้เริ่มต้นหรือผู้ดำเนินการที่มีประสบการณ์ ThunderHub ช่วยให้คุณจัดการโหนด Lightning ของคุณได้อย่างมีประสิทธิภาพผ่าน Interface ที่ทันสมัยและครอบคลุม
