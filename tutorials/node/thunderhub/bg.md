## Въведение



ThunderHub е **мениджър с отворен код за възли Lightning (LND)**, който предлага интуитивен Interface, достъпен от всяко устройство или браузър.



**Основни характеристики:**




- Мониторинг**: Глобален преглед на салдата, каналите, транзакциите и статистиката за маршрутизиране
- Управление**: Отваряне/затваряне на канали, входящи/изходящи плащания, балансиране на каналите
- Интеграции**: Поддръжка на LNURL, размяна чрез Boltz, резервно копие на Amboss
- Interface реагира**: Съвместим с мобилни, таблетни и настолни устройства с тъмни/светли теми



ThunderHub се интегрира лесно с **Umbrel**, **Voltage**, **RaspiBlitz** и **MyNode**, като опростява ежедневното управление на вашия възел.



**ThunderHub е особено подходящ за оператори, които търсят ергономичен Interface за управление на своите канали, контрол на ликвидността (ребалансиране), наблюдение на транзакциите и интегриране на услуги на трети страни, като например Amboss. Сигурността е гарантирана чрез локална или Tor връзка.**



Ако все още нямате възел Lightning, препоръчваме ви да следвате нашия урок LND Umbrel:



https://planb.academy/tutorials/node/lightning-network/umbrel-lnd-b12e0b5b-12ff-45f1-978e-62f4b4a8ba16

## Инсталация



ThunderHub може да се инсталира по няколко различни начина в зависимост от хостинг средата на Lightning node. Независимо дали използвате готово решение (Umbrel, Voltage, RaspiBlitz, MyNode, Start9 и т.н.) или ръчна инсталация, ThunderHub често е наличен без големи усилия. По-долу описваме два често срещани подхода: чрез магазина за приложения Umbrel и чрез ръчна инсталация (приложимо за сървър или самостоятелно хоствана дистрибуция).



### Монтаж чрез Umbrel



Umbrel интегрира ThunderHub в своя **App Store**, което прави инсталацията изключително лесна. Няма нужда от команден ред или ръчно конфигуриране: всичко се извършва чрез Interface Umbrel. Просто следвайте тези стъпки:





- Отворете таблото за управление на Umbrel**: Свържете се с уебсайта Interface на вашия възел Umbrel (напр. `http://umbrel.local` в локалната мрежа или чрез адреса `.onion`, ако използвате Tor).
- Достъп до App Store**: В главното меню на Umbrel щракнете върху "App Store" (или "App"). Потърсете **ThunderHub** в списъка с наличните приложения.



![Écran d'installation de ThunderHub via Umbrel](assets/fr/01.webp)





- Инсталирайте ThunderHub**: Кликнете върху приложението ThunderHub и след това върху бутона за инсталиране. Потвърдете, ако е необходимо. Umbrel автоматично ще изтегли и разположи ThunderHub на вашия възел.





- Стартирайте приложението**: След като инсталацията приключи (няколко десетки секунди), ThunderHub се появява на началната ви страница. Щракнете върху иконата, за да го отворите. ThunderHub се стартира във вашия браузър.



![Identifiants par défaut pour ThunderHub](assets/fr/02.webp)



**Важно:** Когато ThunderHub се отваря за първи път, той автоматично показва **паролата по подразбиране**, необходима за влизане в системата. Опцията "Не показвай това отново" ви позволява да скриете това показване за бъдещи връзки. **Настоятелно ви съветваме да:**




- Запазете тази парола незабавно** в мениджъра си за пароли
- Копирайте го **за използване в следващата стъпка**
- Поставете отметка на "Не показвай това отново", след като паролата бъде запазена



![Page de connexion de ThunderHub](assets/fr/03.webp)



След това ще бъдете прехвърлени към страницата за вход, където трябва да въведете паролата, която копирахте в предишната стъпка, за да отключите Interface.



![Interface de connexion complète à ThunderHub](assets/fr/04.webp)



Umbrel се грижи за предоставянето на ThunderHub на информация за връзката с LND (TLS сертификат, административен макарон и т.н.) във фонов режим, така че не е необходимо да правите допълнителна конфигурация. Само с няколко щраквания на мишката ThunderHub ще бъде пуснат в експлоатация във вашия възел Umbrel.



### Ръчно инсталиране (самостоятелно, с изключение на Umbrel)



За потребители извън Umbrel (например на личен сървър, Raspberry Pi с RaspiBlitz или *самостоятелна* инсталация), инсталирането на ThunderHub изисква няколко допълнителни стъпки. По-долу описваме инсталацията от изходния код и конфигурацията, съгласно [официалната документация на ThunderHub](https://docs.thunderhub.io).



#### Инсталация



**Предпоставки:** Уверете се, че системата ви отговаря на минималните изисквания съгласно [документация за настройка](https://docs.thunderhub.io/setup):




- Node.js** версия 18 или по-висока
- npm** инсталиран
- Достъп до файловете за удостоверяване на LND :
  - LND TLS сертификат (`tls.cert`)
  - Административен макарон LND (`admin.macaroon`)
  - LND адрес на услугата gRPC (име на хост:порт) (по подразбиране `127.0.0.1:10009` локално)



**1. Извличане на кода ThunderHub:** Клонирайте хранилището на проекта в GitHub, както е описано в [документацията за инсталиране](https://docs.thunderhub.io/installation):



```bash
git clone https://github.com/apotdevin/thunderhub.git
cd thunderhub
```



**2. инсталиране на зависимостите и изграждане на приложението:**



```bash
npm install
npm run build
```



Тези команди инсталират всички необходими модули и след това компилират приложението (ThunderHub е написано на TypeScript/React).



**3. Актуализиране по-късно:** ThunderHub предлага няколко метода за бъдещи актуализации:



```bash
# Méthode rapide
npm run update

# Ou méthode manuelle
git pull
npm install
npm run build
```



#### Конфигурация (Настройка)



**1. Основен конфигурационен файл:** Създайте файл `.env.local` в корена на папката ThunderHub, за да персонализирате конфигурацията (това ще предотврати презаписването на вашите настройки по време на актуализации). Основни променливи, както е посочено в [документацията за настройка](https://docs.thunderhub.io/setup):



```bash
# -----------
# Server Configs
# -----------
LOG_LEVEL='info' # 'error' | 'warn' | 'info' | 'http' | 'verbose' | 'debug' | 'silly'
PORT=3000
NODE_ENV=production

# -----------
# Interface Configs
# -----------
THEME='dark' # 'dark' | 'light' | 'night'
CURRENCY='sat' # 'sat' | 'btc' | 'fiat'

# -----------
# Privacy Configs
# -----------
FETCH_PRICES=true # Récupération des prix BTC/fiat depuis Blockchain.com
FETCH_FEES=true # Récupération des frais on-chain depuis Earn.com
DISABLE_LINKS=false # Liens vers 1ml.com et Blockchain.com
NO_VERSION_CHECK=false # Vérification de version depuis GitHub

# -----------
# TOR (optionnel)
# -----------
TOR_PROXY_SERVER='socks://127.0.0.1:9050' # Pour proxifier via TOR

# -----------
# Account Configs
# -----------
ACCOUNT_CONFIG_PATH='/chemin/vers/thubConfig.yaml' # Fichier de comptes
```



**2. Конфигуриране на акаунти на сървъра:** Създайте файла YAML, посочен в `ACCOUNT_CONFIG_PATH` :



```yaml
masterPassword: 'votre_mot_de_passe_principal'
accounts:
- name: 'Mon Nœud LND'
serverUrl: '127.0.0.1:10009'
macaroonPath: '/home/user/.lnd/data/chain/bitcoin/mainnet/admin.macaroon'
certificatePath: '/home/user/.lnd/tls.cert'
password: 'mot_de_passe_compte_specifique'
# Optionnel : compte avec macaroon en hexadécimal
- name: 'Nœud Distant'
serverUrl: 'ip.distante:10009'
macaroon: '0201056c6e6402f8...' # Macaroon en HEX ou Base64
certificate: '0202045c7365...' # Certificat en HEX ou Base64
```



**3. Отдалечен достъп:** За да се свържете с отдалечен възел на LND, добавете в `LND.conf` :



```bash
# Option 1 : accès par IP
tlsextraip=<ip-externe-accessible>
rpclisten=0.0.0.0:10009

# Option 2 : accès par domaine
tlsextradomain=<domaine-externe-accessible>
rpclisten=0.0.0.0:10009
```



**4. Сигурност:** При първото стартиране ThunderHub **автоматично** скрива `masterPassword` и всички пароли на акаунти в YAML файла, за да се избегне наличието на пароли в свободен текст на сървъра.



**5. Стартиране на ThunderHub:**



```bash
npm start
```



По подразбиране сървърът слуша на порт 3000. Осъществете достъп до `http://localhost:3000` (или `http://ip-serveur:3000` от локалната мрежа).



**6. Алтернатива на Docker:** ThunderHub предоставя официални образи на Docker:



```bash
# Image standard
docker pull apotdevin/thunderhub:latest
docker run --rm -it -p 3000:3000/tcp apotdevin/thunderhub:latest

# Image avec base path /thub
docker pull apotdevin/thunderhub:base-v0.11.1
```



Появява се страницата за вход в ThunderHub. Изберете конфигурирания акаунт и въведете паролата за достъп до таблото за управление.



**Инсталиране в други дистрибуции:** Предварително подготвените дистрибуции на възли (RaspiBlitz, MyNode, Start9 и др.) обикновено предлагат собствена поддръжка на ThunderHub чрез съответните си административни интерфейси.



**За повече информация:** Обърнете се към пълната официална документация :




- Инсталация:** [docs.thunderhub.io/installation](https://docs.thunderhub.io/installation)
- Конфигурация:** [docs.thunderhub.io/setup](https://docs.thunderhub.io/setup)



В тези ресурси са описани подробно разширените опции, като SSO акаунти, криптирани макарони, конфигурация на TOR и много други.

## Представяне на Interface



Interface ThunderHub е структуриран около главно меню (обикновено се показва в лявата странична колона), което се състои от няколко ключови раздела. Всеки от тях съответства на определен аспект от управлението на вашия възел Lightning. Нека да ги разгледаме един по един:





- Начало** - Начален раздел с общо табло за управление (преглед на възела и бързи действия).
- Табло за управление** - Табло за управление с възможност за персонализиране с уиджети и разширени показатели.
- Връстници** - Управление на връстници (връзки с други възли).
- Канали** - Подробно управление на каналите на Lightning.
- Rebalance** - инструмент за балансиране на каналите (кръгови плащания).
- Транзакции** - история на плащанията с мълния (LN транзакции).
- Forwards** - Статистика за маршрутизацията (плащания, предадени от вашия възел).
- Верига** - On-Chain wallet на възела (On-Chain BTC: UTXOs, транзакции).
- Amboss** - Интеграция с Amboss (наблюдение на възли, архивиране и др.).
- Инструменти** - Различни инструменти (резервни копия, подписани съобщения, макарони, отчети и др.).
- Размяна** - Функциите за размяна на On-Chain и Lightning се осъществяват чрез Boltz.
- Статистики** - Разширени статистики и показатели за производителността на възлите.



*(Забележка: В зависимост от версията на ThunderHub някои раздели могат да имат малко по-различни заглавия или допълнителни функции, но общите принципи остават същите)*



### Начало (Общ контролен панел)



Табът **Home** на ThunderHub е началната страница, която се появява, след като влезете в системата. Тя съдържа **Общото табло**, което предлага **глобален преглед** на състоянието на вашия възел Lightning и предлага **бързи действия** за рутинни операции. Ето какво обикновено ще намерите:



![Tableau de bord principal de ThunderHub](assets/fr/05.webp)





- Баланси и капацитети:** В горната част на страницата ThunderHub показва наличните ви баланси. Тук обикновено виждате баланса On-Chain (Bitcoin On-Chain в Wallet на възела, символизиран от котва ⚓) и баланса Светкавица (капацитетите на вашите канали, символизирани от светкавица ⚡). Това ви дава непосредствена представа за средствата, с които разполагате в On-Chain и Светкавицата. Ако имате няколко акаунта или канала, уверете се, че сте на правилния от тях (например Mainnet срещу Testnet).





- Основни статистически данни:** Таблото за управление може да показва някои глобални показатели за вашия възел - например брой отворени канали, брой свързани колеги, получени такси за маршрутизация (ако е приложимо) и др. Това е обобщение на неотдавнашната дейност и състоянието на възела.





- Бързи действия:** Таблото за управление разполага с бутони за бързо изпълнение на най-често срещаните задачи, без да се налага да преминавате през менюта. Тези бързи действия включват:





- Призрак**: Настройте персонализиран адрес на Lightning чрез Amboss.
- Дарете**: Направете дарение чрез Lightning.
- Влизане в системата/влизане в системата**: Свържете се с вашия акаунт в Amboss (Quick Connect) и отидете директно в Amboss.space, за да видите информацията за вашия възел.
- Address**: Въведете адрес на светкавица, за да извършите плащане.
- Отворено**: Отворете нов канал Lightning. Щракването отваря формуляр за въвеждане на URI на отдалечения възел, към който да се отвори каналът, сумата и, ако е приложимо, максималната такса On-Chain, която да се използва.
- Декодиране**: Декодирайте фактура Lightning или LNURL, за да видите подробностите преди плащане.
- LNURL**: Обработване на LNURL за плащания или тегления от Lightning.
- Вход за LnMarkets**: Влезте в LnMarkets, за да търгувате.



Тези бързи действия ви позволяват да извършвате най-често срещаните операции директно от началната страница, без да се налага да навигирате из различните раздели на Interface.



Накратко, таблото за управление на ThunderHub ви дава **бърз поглед** върху вашия възел и ви позволява да извършвате най-често срещаните операции (изпращане/приемане, отваряне на канал) с едно кликване. Това е идеалната отправна точка за ежедневна администрация.

- Персонализируеми джаджи:** За разлика от началната страница, която има фиксирано оформление, таблото за управление ви позволява да избирате точно кои елементи да се показват и как да бъдат организирани.

![Paramètres des widgets du Dashboard](assets/fr/07.webp)

![Dashboard personnalisé avec widgets](assets/fr/09.webp)



Веднъж конфигурирано, таблото за управление може да показва различни графики и показатели: Графики за светкавични плащания, брой издадени фактури, статистически данни за нападателите, подробни баланси и др.





- Разширени показатели:** Получете достъп до по-подробни статистически данни за производителността на вашия възел с графики и данни в реално време.





- Конфигурируем преглед:** Приспособете дисплея, за да отговаря на нуждите на обикновения потребител или на професионалния оператор, който управлява множество канали за маршрутизация.





- Модулен Interface:** Добавяйте или премахвайте уиджети, както е необходимо: графики за напредъка, показатели за ликвидност, предупреждения за състоянието на възлите и др.



Този раздел е особено полезен за напреднали потребители, които искат да наблюдават конкретни показатели или да получат по-технически преглед на своя възел Lightning. Той допълва раздела Home (Начало), като предлага по-голяма гъвкавост и контрол върху начина, по който се показва информацията.



### Връстници



В раздела **Пеърс** са изброени всички възли Lightning, които в момента са свързани с вашия като равностойни. **Пеър** е директна връзка между възли в мрежата на Lightning. Вашият възел може да бъде свързан с връстници дори без отворен канал (например само за обмен на информация за клюки в мрежата) или, разбира се, всеки отворен канал автоматично предполага свързан връстник.



![Vue de l'onglet Peers avec les pairs connectés](assets/fr/10.webp)



В раздела Партньори ще видите :





- Информационни колони:** Interface показва полезни данни, като например състояние на синхронизация, тип на връзката (clearnet или Tor), ping, получени/изпратени сатоши и обем на обменените данни.
- Добавяне на партньор: ThunderHub ви позволява ръчно да се свържете с нов партньор чрез бутона **"Добавяне "** в горния десен ъгъл. Ще трябва да въведете URI на възела (формат `<публичен_ключ>@<сокет>`). След като бъде валидиран, ThunderHub изпраща съответната команда `lncli connect`. Ако възелът е онлайн и достъпен, той ще бъде добавен към вашия списък с равнопоставени потребители.



### Канали



Табът **Канали** е сърцето на управлението на каналите на Lightning. Вероятно това е разделът, с който ще се консултирате най-често. В него са представени **всичките ви Lightning канали** с подробна информация за тях и ви позволява да извършвате действия по управление на тези канали.



![Vue d'ensemble des channels ouverts](assets/fr/11.webp)



Ето какво ще намерите на страницата Канали:





- Изглед на списъка с канали:** Всеки отворен (или отварящ/затварящ се) канал се показва в списък, обикновено с псевдонима на отдалечения възел, общия капацитет на канала и цветна лента, илюстрираща разпределението на локалната спрямо отдалечената ликвидност. ThunderHub използва цветен код (често син/зелен) или процент за обозначаване на баланса на канала: например синьо за вашия локален дял, зелено за отдалечения дял. Ако каналът е напълно балансиран (50/50), лентата ще бъде наполовина от всеки цвят. Това ви позволява да определите с един поглед кои канали са небалансирани (всички сини = почти всички местни, всички зелени = почти всички отдалечени).





- Информационни колони:** Interface показва подробни колони, включително статус, налични действия, информация за партньора, идентификатор на канала, капацитет, активност, такси и баланс с графичен дисплей на ликвидността.





- Конфигуриране на дисплея:** Колелцето в горния десен ъгъл ви позволява да персонализирате дисплея на канала според предпочитанията си.





- Състояние:** Ще видите и индикатори за състояние - например `Активен` (каналът е отворен и функционира), `Офлайн` (партньорът е изключен, така че каналът е временно неизползваем), `Очакващ` (за отваряне или затваряне, което очаква потвърждение от On-Chain).





- Действия в канал:** За всеки канал ThunderHub предоставя бутони за действие (често под формата на икони):



![Actions de gestion des canaux - Modifier et Fermer](assets/fr/12.webp)





- Такси за редактиране:** Interface "Актуализиране на политиката на канала" ви позволява да регулирате всички параметри на канала: Базова такса, ставка на таксата (в ppm), CLTV Delta, Max HTLC и Min HTLC. Това ви позволява да регулирате политиките за таксите индивидуално за всеки канал, с цел привличане (или обезкуражаване) на маршрутизиращ трафик. *(Забележка: ThunderHub не е заместител на инструмент за автоматично управление на таксите, но за ръчно регулиране е много ефективен)*
- Затваряне на канала (**Close**): "Канал за затваряне" на Interface ви дава възможност да избирате между **кооперативно затваряне** (по подразбиране) или **принудително затваряне** (**Force Close**), като определяте таксите (в Sats/vByte). **Важно:** винаги предпочитайте кооперативно затваряне, когато е възможно, за да избегнете забавяне на сетълмента на On-Chain и по-високи такси. ThunderHub ще ви покаже дали партньорът е онлайн (възможно е кооперативно затваряне) или не. В случай на принудително затваряне, не забравяйте да потвърдите, тъй като това е необратимо и ще задейства транзакция за разчистване с времеви блокаж (обикновено 144 блока или ~1 ден при Bitcoin Mainnet).
- Отваряне на нов канал:** За да отворите нов канал, щракнете върху зъбното колело в горния десен ъгъл на страницата "Канали", след което изберете "Отвори". След това можете да инициирате канал към нов или съществуващ партньор. Предимството на използването на тази страница е, че имате пред себе си списък със съществуващите канали, което може да ви помогне да решите къде да отворите нов канал.



Накратко, разделът "Канали" ви дава възможност за **прецизно управление на всеки канал**. Това е мястото, където управлявате разпределението на ликвидността, решавате кои канали да запазите или закриете и задавате параметрите на маршрутизацията за всеки канал. ThunderHub предлага ясен Interface за тези важни операции по управление на каналите.



### Възстановяване на баланса



Табът **Ребалансиране** е посветен на **балансирането на каналите**. Балансирането (или *ребалансирането*) включва коригиране на разпределението на средствата между вашите изходящи и входящи канали, като се извършва **кръгово плащане** от един от вашите канали към друг от вашите канали чрез мрежата Lightning. Това ви позволява, без да внасяте нови средства, да прехвърлите ликвидност от твърде пълен канал към твърде празен, като по този начин каналите ви стават по-полезни (балансираният канал може както да изпраща, така и да получава плащания).



![Interface de rebalance des canaux](assets/fr/13.webp)



ThunderHub значително улеснява тази операция, която иначе би била досадна за командния ред. Ето как да използвате раздела Ребалансиране:





- Първоначален изглед на каналите:** При влизане в режим Rebalance (Повторно балансиране) ThunderHub показва списък на каналите ви с индикатор за баланс за всеки от тях (подобен на този на страницата Channels (Канали)). Можете веднага да видите кои канали са извън баланс. ThunderHub може да подреди каналите по реда на увеличаване на баланса, така че най-небалансираните канали да се открояват в горната част на списъка (0,0 означава изцяло местен или отдалечен).





- Избор на равнопоставени потребители:** Interface улеснява избора на изходящи и входящи равнопоставени потребители за ребалансиране.





- Настройки на параметрите:** Можете да зададете :
  - **максималната такса** (в Sats и ppm), която сте готови да платите
  - Сумата за ребалансиране** с опция "Fixed" или "Target"
- Възли, които да се избягват** при маршрутизиране
- Максимално време за изпитание** за намиране на маршрут





- Изберете канал **източник**: Първо изберете **изходящия (източник)** канал, т.е. канала, от който имате твърде много локална ликвидност за преместване. На практика това е канал, в който вашият местен дял е висок (> 50 %). Нека си представим канал А с 1 000 000 сата, 900 000 от които са локални - добър кандидат за изпращане на сата на друго място. Като щракнете върху този канал А като "изходящ", ThunderHub го отбелязва като източник.





- Изберете **целеви канал**: След това изберете **входящия (целевия)** канал, който трябва да получи ликвидност. Обикновено това ще бъде канал, при който е обратното - повечето средства са от далечната страна (напр. само 100 000 местни сайтове от 1 000 000). ThunderHub, след като бъде избран каналът източник, ще подреди останалите канали в обратен ред (намаляващ баланс), за да помогне за идентифицирането на най-допълващите се канали. Изберете канал B, който има място на локалната страна. След това ThunderHub ясно ще покаже кои два канала са избрани (източник А и целеви В).





- Задаване на размер на таксата и допустимо отклонение:** Формуляр ви позволява да въведете :





  - Сумата**, която трябва да се ребалансира (в Sats). Често избираме сума, равна на тази, която би балансирала двата канала при \~50/50. ThunderHub може да запълни предварително половината от излишния капацитет на изходния канал, например.
  - Максималната такса**, която сте готови да платите за тази операция (по избор). Тази такса е изразена в Sats (обща цена на кръговото маршрутизиране). Ако я оставите празна, ThunderHub ще търси път независимо от разходите, което по принцип не е препоръчително (по-добре е да зададете лимит, например 10 Sats за малък ребаланс, или максимален ppm).





- Намиране на маршрут:** Щракнете върху бутона, за да намерите маршрут. ThunderHub се обръща към LND, за да изчисли маршрут от вашия канал източник през мрежата до вашия собствен целеви канал. Ако намери възможен маршрут, който отговаря на вашите критерии за такса, той го показва с подробности за скоковете и цената на таксата. Например може да посочи, че е намерил маршрут с 3 скока с общо 2 Sats в такси.





- Започнете ребалансирането:** Ако сте доволни от предложения маршрут, кликнете върху **Канал за балансиране**. След това ThunderHub ще инициира кръгово плащане чрез LND. Ако плащането е успешно, ще видите известие за успех, а балансите на каналите A и B ще бъдат променени в реално време. ThunderHub ще актуализира индикатора за баланс за тези канали (в идеалния случай те ще бъдат по-зелени от преди, което показва по-добър баланс).





- Корекции и итерации:** Ако при първия опит не бъде намерен маршрут (или ако той е твърде скъп), можете да коригирате параметрите:





  - Опитайте с по-малка сума (понякога частичното ребалансиране преминава успешно, докато голямото количество не успява).
  - Постепенно увеличавайте максималния размер на таксата, но внимавайте да не платите повече такси, отколкото си заслужава.
  - Използвайте бутона **Получаване на друг маршрут**, ако е наличен, за да опитате алтернативен маршрут.
  - Опитайте с друг чифт канали, ако нещата станат наистина лепкави.



ThunderHub прави процеса много **интуитивен и нагледен**. Само в 4 стъпки (избор на изходящ канал, избор на входящ канал, сума, валидиране) можете да направите това, което преди изискваше сложни ръчни команди. Инструментът е безценен за поддържане на добре балансиран възел, подобряване на ефективността на маршрутизацията и опита (по-лесно изпращане и получаване на плащания по всички ваши канали).



И накрая, имайте предвид, че при ребалансирането се изразходват разходи за маршрутизация (заплащани на междинните възли), така че това е **инвестиция**, за да направите възела си по-плавен. Използвайте го разумно, например за поддържане на канал към услуга, която използвате често (входяща ликвидност), или за балансиране на голям канал за маршрутизация. ThunderHub ви позволява да направите това **просто и ефективно**.



### Транзакции



Разделът **Транзакции** в ThunderHub съответства на историята на транзакциите на вашия възел **Lightning**, т.е. на плащанията и фактурите, платени или получени по каналите. Това е нещо като извлечение от сметката за вашите операции в LN.



![Historique des transactions Lightning](assets/fr/14.webp)



В този раздел ще намерите :





- Графика Invoice:** В горния десен ъгъл графиката показва развитието на получените фактури с течение на времето, което ви позволява да визуализирате дейността на вашия възел.





- Хронологичен списък на всички транзакции със светкавици, извършени *от* или *към* вашия възел. Всеки запис може да показва :





  - Вид на операцията: **изпратено плащане** (изходящо плащане) или **получено плащане** (входящо, чрез платена Invoice).
  - Сумата в Sats.
  - Дата/час.
  - Идентификатор на плащането (Hash или RHash pre-image) или коментар (ако сте добавили бележка към Invoice).
  - Статус: **завършено** или евентуално **в процес**/*неуспешно* (например плащане, което очаква решение, но обикновено LND обработва това бързо, така че тук има малко "чакащи" в сравнение с транзакциите On-Chain).



Накратко, разделът "Транзакции" служи като дневник на дейността на вашия **LN**. Той е много полезен, за да проверите дали дадено плащане е преминало, колко такси е струвало или да проследите историята на вашите светкавични обмени. В комбинация със секцията Forwards (описана по-нататък) ще имате пълен поглед върху парите, които са преминали през вашия възел.



### Нападатели



Табът **Напред** е посветен на дейността на вашия възел по **маршрутизиране**, т.е. на плащанията, които **преминават** през вашите канали (когато действате като междинен възел в мрежата на "Светкавица"). Ако управлявате възела си като маршрутизиращ възел, това е важен раздел за проследяване на ефективността ви.



![Statistiques de routage Lightning](assets/fr/15.webp)



В "Напред" ThunderHub представя :





- Филтри и опции за показване:** В горния десен ъгъл филтрите ви позволяват да сортирате данните по ден/седмица/месец/година и да избирате между графично или таблично показване.





- Съобщение за дейност:** Ако през избрания период не е извършено маршрутизиране, Interface показва "No forwards for this period" (Няма препратки за този период), както е показано в този пример.





- Таблица с последните препращания**: всеки запис съответства на плащане, което е било **препратено** през вашия възел. За всяко препращане обикновено виждаме :





  - времеви печат,
  - маршрутизираната сума (в Sats),
  - спечелената такса** за това препращане (в Sats това е разликата между полученото по входящия канал и изпратеното по изходящия),
  - използваните входящи и изходящи канали (често идентифицирани чрез псевдоним на партньор или идентификатор на канал).
  - състояние (обикновено *завършено* или неуспешно, ако препращането е неуспешно по пътя).





- Обобщени статистически данни**: ThunderHub изчислява и показва в горната част на страницата общи суми и статистически данни за даден период (напр. последните 24 часа, 7 дни и т.н., които понякога могат да се конфигурират).



Накратко, разделът Forwards предлага **преглед в реално време на маршрутизиращата дейност на вашия възел Lightning**. В съчетание със секциите Канали и Ребалансиране това представлява пълен пакет за оптимизиране на вашия възел: Канали/Ребаланс за ликвидност, Forwards за наблюдение на резултатите (потоци и печалби).



### Верига



Разделът **Chain** съответства на управлението на Bitcoin On-Chain wallet на вашия възел LND. Този раздел Interface ви позволява да преглеждате и управлявате средства Bitcoin, които се използват за отваряне на канали или получаване на средства от затворени канали.



![Gestion du portefeuille on-chain](assets/fr/16.webp)



Във веригата ще намерите :





- Баланс On-Chain:** Показва общия баланс на BTC, наличен в Wallet LND.





- Списък на UTXOs:** Вижте всички неизползвани средства (UTXO) със сумата, потвържденията, адреса и формата на всяко средство.





- История на транзакциите:** Подробна таблица на всички транзакции Bitcoin с вид (в/из), дата, сума, такси, потвърждения, блок за включване, адреси и txid.



### Amboss



ThunderHub се интегрира с платформата **Amboss** (amboss.space), която предлага подробна информация за възлите Lightning, пазар на ликвидност и полезни функции като криптирано архивиране на канали и наблюдение на наличността.



![Intégration Amboss avec ses options](assets/fr/17.webp)



В ThunderHub разделът Amboss ви позволява да **свържете** възела си с вашия акаунт в Amboss:





- Ghost Address:** Настройте **персонализиран Lightning адрес** за вашия възел, за да улесните входящите плащания.





- Автоматични резервни копия на канали:** Водеща функция за криптирани резервни копия на канали** (SCB файлове) в Amboss. Активирайте **Автоматично архивиране на Amboss = Да** в настройките, за да изпращате автоматично актуализации на криптирани резервни копия при всяка смяна на канала. В случай на повреда ще можете да възстановите средствата си благодарение на това външно резервно копие.





- Проверки на състоянието:** Активирайте **Amboss Healthcheck = Yes**, за да може възелът ви да изпраща редовно пингове към Amboss. Ще получавате предупреждения, ако възелът ви изглежда офлайн.





- Други функции:** Автоматично прехвърляне на баланс, интеграция с **Magma/Hydro** (пазар на ликвидност) и достъп до подробна статистика за ефективността.



Интеграцията с Amboss добавя важно ниво на **сигурност** с автоматично външно архивиране и наблюдение на наличността, достъпни директно от ThunderHub.



### Инструменти



Разделът **Инструменти** обединява различни усъвършенствани инструменти за управление на вашия възел. Ето основните елементи:



![Interface des outils disponibles](assets/fr/18.webp)





- Резервни копия:** Управлявайте ръчно резервните копия на канала (SCB). ThunderHub ви позволява **да изтеглите пълния резервен файл** на вашите канали (опция "Архивиране на всички канали -> Изтегляне"). Съхранявайте този файл `channel-all.bak` на сигурно място - той е от съществено значение за възстановяването на средствата ви в случай на срив. Можете също така да **импортирате** резервен файл, когато пренасочвате възел.





- Счетоводство:** Инструмент за експортиране на финансови отчети, включващи получени/платени такси и маршрутизирани обеми за даден период.
- Подписани съобщения:** Подписвайте или проверявайте съобщения с вашия възел, за да докажете собствеността си върху вашия възел Lightning чрез криптографски подпис.
- Макарони (раздел "Пекарна"):** Управлявайте макароните LND**, за да създадете персонализиран достъп. Разделът Interface "Пекарна" ви позволява прецизно да изберете всяко разрешение: "Добавяне или премахване на колеги", "Създаване на верижни адреси", "Създаване на фактури", "Създаване на макарони", "Извеждане на ключове", "Получаване на ключове за достъп", "Получаване на верижни транзакции", "Получаване на фактури", "Получаване на информация за Wallet", "Get Payments" (Получаване на плащания), "Get Peers" (Получаване на партньори), "Pay Invoices" (Плащане на фактури), "Revoke Access Ids" (Отмяна на идентификатори за достъп), "Send to Chain Addresses" (Изпращане на верижни адреси), "Sign bytes" (Подписване на байтове), "Sign Messages" (Подписване на съобщения), "Stop daemon" (Спиране на daemon), "Verify bytes signature" (Проверка на байтове с подпис), "Verify messages" (Проверка на съобщения) и т.н. Всяко разрешение може да се активира поотделно с бутоните "Да/Не", за да се създаде макара по поръчка.
- Системна информация:** Показване на версията на Wallet и активираните RPC.



Накратко, разделът "Инструменти" обединява разширени функции за администриране - архивиране, счетоводство, сигурност и управление на достъпа - в единен Interface.



### Размяна на



Разделът **Swap** на ThunderHub ви позволява да подменяте светкавични сатоши към Bitcoin On-Chain чрез услугата Boltz. Тази функция е полезна за "изхвърляне" на излишната ликвидност на Lightning към канала, без да се затваря канал.



![Interface de swap via Boltz](assets/fr/19.webp)



Процесът е лесен:





- Сума**: Определете сумата, която трябва да бъде обменена
- Address**: Въведете адреса за приемане на Bitcoin
- Изпълнение**: ThunderHub се свързва с Boltz, за да обработи автоматично обмена



**Придобивки:**




- Услуга, която не е свързана с попечителство (без попечителство в брой)
- Запазване на съществуващите канали
- Лесен за използване интегриран Interface



Boltz начислява малка комисиона, а вие плащате стандартната такса за транзакция на Bitcoin. ThunderHub показва всички разходи преди потвърждение.



### Статистики



Разделът **Stats** на ThunderHub предлага **разширени статистически данни** за вашия възел Lightning, за да анализирате производителността и да оптимизирате маршрутизацията.



![Statistiques du nœud via Amboss](assets/fr/20.webp)



Този раздел е от съществено значение за оптимизиране на разходите, идентифициране на успешни канали и планиране на разширяването на вашия възел.



## Заключение



**ThunderHub** се утвърди като основен инструмент за лесно администриране на възел Lightning **LND**. Този съвременен Interface предлага всички основни функции: управление на канали, плащания, мониторинг, с разширени функции, като автоматизирано ребалансиране и интеграция с Amboss.



**Ключови предимства:**




- Interface - елегантен и интуитивен
- Мощни инструменти (ребалансиране, размяна на Boltz, автоматично архивиране)
- Съвместимост с Umbrel, Voltage, RaspiBlitz и други дистрибуции



ThunderHub демократизира усъвършенстваното управление на възли Lightning, като прави достъпно това, което преди изискваше сложни технически команди. Независимо дали сте начинаещ, или опитен оператор, ThunderHub ви позволява ефективно да управлявате вашия Lightning възел чрез модерния, всеобхватен Interface.
