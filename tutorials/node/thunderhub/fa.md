## مقدمه



ThunderHub یک **مدیر متن‌باز برای نودهای لایتنینگ (LND)** است که یک Interface کاربرپسند ارائه می‌دهد و از هر دستگاه یا مرورگری قابل دسترسی است.



**ویژگی‌های اصلی:**




- **نظارت**: نمای کلی از موجودی‌ها، کانال‌ها، تراکنش‌ها، آمار مسیریابی
- **مدیریت**: باز کردن/بستن کانال‌ها، پرداخت‌های ورودی/خروجی، متعادل‌سازی کانال
- **یکپارچه‌سازی‌ها**: پشتیبانی از LNURL، مبادلات از طریق Boltz، پشتیبان‌گیری Amboss
- **Interface پاسخگو**: سازگار با دستگاه‌های موبایل، تبلت و دسکتاپ با تم‌های تیره/روشن



ThunderHub به‌راحتی با **Umbrel**، **Voltage**، **RaspiBlitz** و **MyNode** یکپارچه می‌شود و مدیریت روزمره نود شما را ساده‌تر می‌کند.



**ThunderHub به‌ویژه برای اپراتورهایی مناسب است که به دنبال یک Interface ارگونومیک برای مدیریت کانال‌های خود، کنترل نقدینگی (متعادل‌سازی مجدد)، نظارت بر تراکنش‌ها و ادغام خدمات شخص ثالث مانند Amboss هستند. امنیت از طریق یک اتصال محلی یا Tor تضمین می‌شود.**



اگر هنوز یک نود لایتنینگ ندارید، توصیه می‌کنیم آموزش LND Umbrel ما را دنبال کنید:



https://planb.network/tutorials/node/lightning-network/umbrel-lnd-b12e0b5b-12ff-45f1-978e-62f4b4a8ba16

## نصب و راه‌اندازی



ThunderHub می‌تواند به روش‌های مختلفی نصب شود، بسته به محیط میزبانی نود لایتنینگ شما. چه از یک راه‌حل آماده (Umbrel, Voltage, RaspiBlitz, MyNode, Start9, و غیره) استفاده کنید یا نصب دستی انجام دهید، ThunderHub اغلب بدون تلاش عمده‌ای در دسترس است. در زیر، دو روش معمول را توضیح می‌دهیم: از طریق فروشگاه اپلیکیشن Umbrel و از طریق نصب دستی (قابل اجرا بر روی یک سرور یا توزیع خودمیزبان).



### نصب از طریق Umbrel



آمبرل ThunderHub را در **فروشگاه برنامه** خود ادغام می‌کند و نصب را بسیار ساده می‌سازد. نیازی به خط فرمان یا پیکربندی دستی نیست: همه چیز از طریق Interface Umbrel انجام می‌شود. فقط این مراحل را دنبال کنید:





- **داشبورد Umbrel را باز کنید**: به وب Interface نود Umbrel خود متصل شوید (مثلاً `http://umbrel.local` در شبکه محلی شما، یا از طریق `.onion` Address اگر از Tor استفاده می‌کنید).
- دسترسی به فروشگاه برنامه: در منوی اصلی Umbrel، روی "فروشگاه برنامه" (یا "برنامه") کلیک کنید. **ThunderHub** را در لیست برنامه‌های موجود جستجو کنید.



![Écran d'installation de ThunderHub via Umbrel](assets/fr/01.webp)





- نصب **ThunderHub**: بر روی برنامه ThunderHub کلیک کنید، سپس بر روی دکمه نصب کلیک کنید. در صورت لزوم تأیید کنید. Umbrel به طور خودکار ThunderHub را بر روی نود شما دانلود و اجرا خواهد کرد.





- **برنامه را راه‌اندازی کنید**: پس از تکمیل نصب (چند ده ثانیه)، ThunderHub در صفحه اصلی شما ظاهر می‌شود. برای باز کردن آن روی آیکون کلیک کنید. ThunderHub در مرورگر شما راه‌اندازی می‌شود.



![Identifiants par défaut pour ThunderHub](assets/fr/02.webp)



**مهم:** هنگامی که ThunderHub برای اولین بار باز می‌شود، به‌طور خودکار **رمز عبور پیش‌فرض** مورد نیاز برای ورود را نمایش می‌دهد. گزینه "دیگر این را نشان نده" به شما اجازه می‌دهد این نمایش را برای اتصالات آینده پنهان کنید. **ما به شدت توصیه می‌کنیم که:**




- این رمز عبور را **فوراً** در مدیر رمز عبور خود ذخیره کنید
- آن را **کپی کنید** برای استفاده در مرحله بعدی
- پس از ذخیره شدن رمز عبور، "دیگر این را نشان نده" را انتخاب کنید.



![Page de connexion de ThunderHub](assets/fr/03.webp)



سپس به صفحه ورود هدایت می‌شوید، جایی که باید رمز عبوری را که در مرحله قبل کپی کرده‌اید وارد کنید تا Interface را باز کنید.



![Interface de connexion complète à ThunderHub](assets/fr/04.webp)



آمبرل به صورت خودکار اطلاعات اتصال LND (گواهی TLS، ماکارون مدیریت و غیره) را برای ThunderHub فراهم می‌کند، بنابراین نیازی به انجام تنظیمات اضافی ندارید. تنها با چند کلیک، ThunderHub را بر روی نود آمبرل خود راه‌اندازی خواهید کرد.



### نصب دستی (میزبانی شخصی، به‌جز Umbrel)



برای کاربرانی که خارج از Umbrel هستند (مثلاً روی یک سرور شخصی، یک Raspberry Pi با RaspiBlitz یا یک نصب *مستقل*)، نصب ThunderHub نیاز به چند مرحله اضافی دارد. در زیر نصب از منبع و پیکربندی را طبق [مستندات رسمی ThunderHub](https://docs.thunderhub.io) توضیح می‌دهیم.



#### نصب



**پیش‌نیازها:** اطمینان حاصل کنید که سیستم شما حداقل نیازمندی‌ها را مطابق با [مستندات راه‌اندازی](https://docs.thunderhub.io/setup) برآورده می‌کند:




- **Node.js** نسخه 18 یا بالاتر
- **npm** نصب شد
- دسترسی به فایل‌های احراز هویت LND :
  - گواهینامه TLS LND (`tls.cert`)
  - LND مدیریت ماکارون (`admin.macaroon`)
  - سرویس LND gRPC Address (hostname:port) (به طور پیش‌فرض `127.0.0.1:10009` به صورت محلی)



**1. کد ThunderHub را بازیابی کنید:** مخزن GitHub پروژه را همانطور که در [مستندات نصب](https://docs.thunderhub.io/installation) توضیح داده شده است، کلون کنید:



```bash
git clone https://github.com/apotdevin/thunderhub.git
cd thunderhub
```



**2. نصب وابستگی‌ها و ساخت برنامه:**



```bash
npm install
npm run build
```



این دستورات تمام ماژول‌های مورد نیاز را نصب کرده و سپس برنامه را کامپایل می‌کنند (ThunderHub با TypeScript/React نوشته شده است).



**3. به‌روزرسانی بعدی:** ThunderHub چندین روش برای به‌روزرسانی‌های آینده ارائه می‌دهد:



```bash
# Méthode rapide
npm run update

# Ou méthode manuelle
git pull
npm install
npm run build
```



#### پیکربندی (راه‌اندازی)



**1. فایل پیکربندی اصلی:** یک فایل `.env.local` در ریشه پوشه ThunderHub ایجاد کنید تا پیکربندی را سفارشی کنید (این کار از بازنویسی تنظیمات شما در طول به‌روزرسانی‌ها جلوگیری می‌کند). متغیرهای اصلی طبق [مستندات راه‌اندازی](https://docs.thunderhub.io/setup):



```bash
# -----------
# Server Configs
# -----------
LOG_LEVEL='info' # 'error' | 'warn' | 'info' | 'http' | 'verbose' | 'debug' | 'silly'
PORT=3000
NODE_ENV=production

# -----------
# Interface Configs
# -----------
THEME='dark' # 'dark' | 'light' | 'night'
CURRENCY='sat' # 'sat' | 'btc' | 'fiat'

# -----------
# Privacy Configs
# -----------
FETCH_PRICES=true # Récupération des prix BTC/fiat depuis Blockchain.com
FETCH_FEES=true # Récupération des frais on-chain depuis Earn.com
DISABLE_LINKS=false # Liens vers 1ml.com et Blockchain.com
NO_VERSION_CHECK=false # Vérification de version depuis GitHub

# -----------
# TOR (optionnel)
# -----------
TOR_PROXY_SERVER='socks://127.0.0.1:9050' # Pour proxifier via TOR

# -----------
# Account Configs
# -----------
ACCOUNT_CONFIG_PATH='/chemin/vers/thubConfig.yaml' # Fichier de comptes
```



**۲. پیکربندی حساب‌های سرور:** فایل YAML مشخص شده در `ACCOUNT_CONFIG_PATH` را ایجاد کنید:



```yaml
masterPassword: 'votre_mot_de_passe_principal'
accounts:
- name: 'Mon Nœud LND'
serverUrl: '127.0.0.1:10009'
macaroonPath: '/home/user/.lnd/data/chain/bitcoin/mainnet/admin.macaroon'
certificatePath: '/home/user/.lnd/tls.cert'
password: 'mot_de_passe_compte_specifique'
# Optionnel : compte avec macaroon en hexadécimal
- name: 'Nœud Distant'
serverUrl: 'ip.distante:10009'
macaroon: '0201056c6e6402f8...' # Macaroon en HEX ou Base64
certificate: '0202045c7365...' # Certificat en HEX ou Base64
```



**3. دسترسی از راه دور:** برای اتصال به یک گره راه دور LND، به `LND.conf` اضافه کنید:



```bash
# Option 1 : accès par IP
tlsextraip=<ip-externe-accessible>
rpclisten=0.0.0.0:10009

# Option 2 : accès par domaine
tlsextradomain=<domaine-externe-accessible>
rpclisten=0.0.0.0:10009
```



**۴. امنیت:** در اولین راه‌اندازی، ThunderHub به‌طور **خودکار** `masterPassword` و تمام رمزهای عبور حساب‌ها را در فایل YAML مخفی می‌کند تا از وجود رمزهای عبور به‌صورت متن ساده روی سرور جلوگیری شود.



**5. شروع ThunderHub:**



```bash
npm start
```



به طور پیش‌فرض، سرور به پورت 3000 گوش می‌دهد. به `http://localhost:3000` (یا `http://ip-serveur:3000` از شبکه محلی) دسترسی پیدا کنید.



**6. جایگزین Docker:** ThunderHub تصاویر رسمی Docker ارائه می‌دهد:



```bash
# Image standard
docker pull apotdevin/thunderhub:latest
docker run --rm -it -p 3000:3000/tcp apotdevin/thunderhub:latest

# Image avec base path /thub
docker pull apotdevin/thunderhub:base-v0.11.1
```



صفحه ورود ThunderHub ظاهر می‌شود. حساب کاربری تنظیم‌شده را انتخاب کرده و رمز عبور را وارد کنید تا به داشبورد دسترسی پیدا کنید.



**نصب بر روی توزیع‌های دیگر:** توزیع‌های از پیش بسته‌بندی شده نود (RaspiBlitz، MyNode، Start9 و غیره) عموماً پشتیبانی بومی از ThunderHub را از طریق رابط‌های مدیریت مربوطه خود ارائه می‌دهند.



**برای اطلاعات بیشتر:** به مستندات رسمی کامل مراجعه کنید :




- **نصب:** [docs.thunderhub.io/installation](https://docs.thunderhub.io/installation)
- **پیکربندی:** [docs.thunderhub.io/setup](https://docs.thunderhub.io/setup)



این منابع گزینه‌های پیشرفته‌ای مانند حساب‌های SSO، ماکارون‌های رمزگذاری‌شده، پیکربندی TOR و بسیاری موارد دیگر را شرح می‌دهند.



پس از نصب و دسترسی به ThunderHub، آماده‌اید تا از تمامی ویژگی‌های آن بهره‌برداری کنید. در بخش زیر، نگاهی به Interface ThunderHub و زبانه‌های مختلف آن خواهیم داشت تا شما را در استفاده از آن راهنمایی کنیم.



## ارائه Interface



Interface ThunderHub حول یک منوی اصلی (که معمولاً در ستون سمت چپ نمایش داده می‌شود) سازماندهی شده است که شامل چندین بخش کلیدی است. هر کدام به جنبه‌ای از مدیریت نود لایتنینگ شما مربوط می‌شود. بیایید آن‌ها را یکی یکی مرور کنیم:





- **خانه** - زبانه خانه با داشبورد کلی (نمای کلی از نود شما و اقدامات سریع).
- **داشبورد** - داشبورد قابل تنظیم با ویجت‌ها و معیارهای پیشرفته.
- **Peers** - مدیریت همتاهای لایتنینگ (اتصالات به نودهای دیگر).
- **کانال‌ها** - مدیریت دقیق کانال‌های لایتنینگ.
- **توازن مجدد** - ابزار متعادل‌سازی کانال (پرداخت‌های دایره‌ای).
- **تراکنش‌ها** - تاریخچه پرداخت لایتنینگ (تراکنش‌های LN).
- **فورواردها** - آمار مسیریابی (پرداخت‌هایی که توسط نود شما منتقل شده‌اند).
- **زنجیره** - پرتفوی On-Chain نود (On-Chain BTC: UTXOs، تراکنش‌ها).
- **Amboss** - یکپارچه‌سازی با Amboss (نظارت بر نود، پشتیبان‌گیری، و غیره).
- **ابزارها** - ابزارهای متفرقه (پشتیبان‌گیری، پیام‌های امضا شده، ماکارون‌ها، گزارش‌ها، و غیره).
- **مبادله** - On-Chain/عملکردهای مبادله رعد و برق از طریق Boltz.
- **آمار** - آمار پیشرفته و معیارهای عملکرد نود.



*(توجه: بسته به نسخه ThunderHub، برخی بخش‌ها ممکن است عناوین کمی متفاوت یا ویژگی‌های اضافی داشته باشند، اما اصول کلی یکسان باقی می‌مانند)*



### خانه (صفحه کنترل عمومی)



زبانه **Home** در ThunderHub صفحه اصلی است که پس از ورود به سیستم ظاهر می‌شود. این زبانه شامل **داشبورد عمومی** است که یک **نمای کلی جهانی** از وضعیت نود لایتنینگ شما ارائه می‌دهد و **اقدامات سریع** برای عملیات روتین پیشنهاد می‌کند. در اینجا معمولاً آنچه پیدا خواهید کرد:



![Tableau de bord principal de ThunderHub](assets/fr/05.webp)





- **موجودی‌ها و ظرفیت‌ها:** در بالای صفحه، ThunderHub موجودی‌های در دسترس شما را نمایش می‌دهد. در اینجا معمولاً موجودی On-Chain (Bitcoin On-Chain در Wallet نود، که با یک Anchor ⚓ نمادگذاری شده است) و موجودی Lightning (ظرفیت کانال‌های شما، که با یک Bolt ⚡ نمادگذاری شده است) را مشاهده خواهید کرد. این به شما ایده‌ای فوری از وجوهی که در On-Chain و Lightning دارید می‌دهد. اگر چندین حساب یا کانال دارید، مطمئن شوید که در حساب یا کانال صحیح هستید (مثلاً Mainnet در مقابل Testnet).





- **آمار کلیدی:** داشبورد می‌تواند برخی از معیارهای جهانی برای نود شما را نشان دهد - به عنوان مثال، تعداد کانال‌های باز، تعداد همتایان متصل، کارمزدهای مسیریابی کسب‌شده (در صورت وجود)، و غیره. این یک خلاصه‌ای از فعالیت‌های اخیر و سلامت نود است.





- **اقدامات سریع:** داشبورد دارای دکمه‌هایی برای اجرای سریع‌ترین وظایف رایج است، بدون نیاز به پیمایش در منوها. این اقدامات سریع شامل:





- **Ghost**: یک Lightning Address سفارشی از طریق Amboss راه‌اندازی کنید.
- **اهدا**: از طریق لایتنینگ کمک مالی کنید.
- **ورود/برو به**: به حساب Amboss خود متصل شوید (اتصال سریع) و مستقیماً به Amboss.space بروید تا اطلاعات نود خود را مشاهده کنید.
- **Address** : وارد کردن یک Lightning Address برای انجام پرداخت.
- **باز کردن**: یک کانال جدید لایتنینگ باز کنید. با کلیک کردن، فرمی برای وارد کردن URI نود راه دور که می‌خواهید کانال را به آن باز کنید، مقدار و در صورت لزوم، حداکثر کارمزد On-Chain که باید استفاده شود، باز می‌شود.
- **رمزگشایی**: یک Lightning Invoice یا LNURL را رمزگشایی کنید تا جزئیات را قبل از پرداخت مشاهده کنید.
- **LNURL**: پردازش LNURLها برای پرداخت‌ها یا برداشت‌های لایتنینگ.
- ورود به **LnMarkets**: برای معامله به LnMarkets وارد شوید.



این اقدامات سریع به شما امکان می‌دهند تا عملیات‌های پرتکرار را مستقیماً از صفحه اصلی انجام دهید، بدون اینکه نیاز باشد از طریق تب‌های مختلف Interface جابجا شوید.



به طور خلاصه، داشبورد ThunderHub به شما یک **نگاه سریع** به نودتان می‌دهد و به شما اجازه می‌دهد تا متداول‌ترین عملیات‌ها (ارسال/دریافت، باز کردن یک کانال) را با یک کلیک انجام دهید. این نقطه شروع ایده‌آل برای مدیریت روزمره است.



### داشبورد



بخش **داشبورد** از زبانه خانه جدا است و یک داشبورد پیشرفته‌تر و قابل تنظیم ارائه می‌دهد. این بخش به شما اجازه می‌دهد تا با استفاده از ویجت‌های خاص، نمای سفارشی‌سازی شده‌ای را بر اساس نیازهای خود به عنوان یک اپراتور نود ایجاد کنید.





- **ابزارک‌های قابل تنظیم:** برخلاف صفحه اصلی که دارای یک چیدمان ثابت است، داشبورد به شما اجازه می‌دهد تا دقیقاً انتخاب کنید کدام Elements را نمایش دهید و چگونه آن‌ها را سازماندهی کنید.



![Dashboard sans widgets activés](assets/fr/06.webp)



اگر هیچ ویجتی فعال نباشد، پیغام "No Widgets Enabled!" را با دکمه "Settings" برای دسترسی به پارامترهای سفارشی‌سازی مشاهده خواهید کرد.



![Paramètres des widgets du Dashboard](assets/fr/07.webp)



در تنظیمات، می‌توانید از میان طیف گسترده‌ای از ویجت‌ها که به دسته‌بندی‌های مختلفی مانند: "Lightning - Info"، "Lightning - Table"، "Lightning - Graph" و غیره سازماندهی شده‌اند، انتخاب کنید. هر ویجت می‌تواند به صورت جداگانه با دکمه‌های "نمایش/پنهان" فعال یا غیرفعال شود.



![Bas de la page des paramètres](assets/fr/08.webp)



در پایین تنظیمات، دکمه "به داشبورد" را برای بازگشت به داشبورد و دکمه "بازنشانی ویجت‌ها" را برای بازنشانی نمایش پیش‌فرض پیدا خواهید کرد.

پس از پیکربندی، داشبورد شما می‌تواند نمودارها و معیارهای مختلفی را نمایش دهد: نمودارهای پرداخت لایتنینگ، تعداد فاکتورهای صادر شده، آمار انتقالات، ترازهای دقیق، و غیره.





- **معیارهای پیشرفته:** به آمار دقیق‌تری از عملکرد نود خود دسترسی پیدا کنید، با نمودارها و داده‌های لحظه‌ای.





- **نمای کلی قابل تنظیم:** نمایش را متناسب با نیازهای خود تنظیم کنید، چه یک کاربر عادی باشید یا یک اپراتور حرفه‌ای که چندین کانال مسیریابی را مدیریت می‌کند.





- **ماژولار Interface:** ویجت‌ها را بر اساس نیاز اضافه یا حذف کنید: نمودارهای پیش‌رو، معیارهای نقدینگی، هشدارهای سلامت نود و غیره.



این بخش به‌ویژه برای کاربران پیشرفته که مایل به نظارت بر معیارهای خاص یا دریافت یک نمای کلی فنی‌تر از نود لایتنینگ خود هستند، مفید است. این بخش با ارائه انعطاف‌پذیری و کنترل بیشتر بر نحوه نمایش اطلاعات، بخش خانه را تکمیل می‌کند.



### همسالان



بخش **Peers** تمام نودهای لایتنینگی که در حال حاضر به نود شما به عنوان همتا متصل هستند را فهرست می‌کند. یک **همتا** یک اتصال مستقیم نود به نود در Lightning Network است. نود شما می‌تواند حتی بدون یک کانال باز به همتاها متصل باشد (مثلاً فقط برای اطلاعات شایعه Exchange در شبکه)، یا البته هر کانال باز به طور خودکار یک همتای متصل را نشان می‌دهد.



![Vue de l'onglet Peers avec les pairs connectés](assets/fr/10.webp)



در برگه Peers، شما خواهید دید:





- **ستون‌های اطلاعات:** Interface جزئیات مفیدی مانند وضعیت همگام‌سازی، نوع اتصال (clearnet یا Tor)، پینگ، ساتوشی‌های دریافت/ارسال شده و حجم داده‌های مبادله شده را نمایش می‌دهد.
- افزودن یک همتا: ThunderHub به شما اجازه می‌دهد تا به صورت دستی به یک همتای جدید از طریق دکمه **"افزودن"** در گوشه بالا سمت راست متصل شوید. شما باید URI نود را وارد کنید (فرمت `<public_key>@<socket>`). پس از اعتبارسنجی، ThunderHub فرمان `lncli connect` مربوطه را ارسال می‌کند. اگر نود آنلاین و قابل دسترسی باشد، به لیست همتایان شما اضافه خواهد شد.



### کانال‌ها



زبانه **کانال‌ها** قلب مدیریت کانال لایتنینگ است. احتمالاً این بخشی است که بیشتر به آن مراجعه خواهید کرد. این بخش **تمام کانال‌های لایتنینگ شما** را با جزئیاتشان نمایش می‌دهد و به شما اجازه می‌دهد تا اقدامات مدیریتی را بر روی این کانال‌ها انجام دهید.



![Vue d'ensemble des channels ouverts](assets/fr/11.webp)

به طور خلاصه، بخش کانال‌ها به شما **کنترل دقیق بر هر کانال** را می‌دهد. اینجا جایی است که تخصیص نقدینگی را هدایت می‌کنید، تصمیم می‌گیرید کدام کانال‌ها را نگه دارید یا ببندید، و پارامترهای مسیریابی هر کانال را تنظیم می‌کنید. ThunderHub یک Interface واضح برای این عملیات مهم مدیریت نود ارائه می‌دهد.



### متعادل‌سازی مجدد



زبانه **Rebalance** به **متعادل‌سازی کانال** اختصاص دارد. متعادل‌سازی (یا *متعادل‌سازی مجدد*) شامل تنظیم مجدد توزیع وجوه بین کانال‌های خروجی و ورودی شما است، با انجام یک **پرداخت دایره‌ای** از یکی از کانال‌های شما به کانال دیگر شما، از طریق Lightning Network. این به شما اجازه می‌دهد، بدون وارد کردن وجوه جدید، نقدینگی را از کانالی که بیش از حد پر است به کانالی که بیش از حد خالی است منتقل کنید، و کانال‌های شما را مفیدتر کنید (یک کانال متعادل می‌تواند هم پرداخت‌ها را ارسال و هم دریافت کند).



![Interface de rebalance des canaux](assets/fr/13.webp)



ThunderHub به طور قابل توجهی این عملیات را تسهیل می‌کند، که در غیر این صورت در خط فرمان خسته‌کننده خواهد بود. در اینجا نحوه استفاده از بخش Rebalance آمده است:





- **نمای اولیه کانال:** با ورود به Rebalance، ThunderHub لیستی از کانال‌های شما را نمایش می‌دهد، همراه با یک نشانگر تعادل برای هر کدام (مشابه آنچه در صفحه کانال‌ها وجود دارد). شما می‌توانید بلافاصله ببینید کدام کانال‌ها نامتعادل هستند. ThunderHub می‌تواند کانال‌ها را به ترتیب افزایش تعادل مرتب کند، به طوری که نامتعادل‌ترین کانال‌ها در بالای لیست برجسته شوند (0.0 به معنای کاملاً محلی یا از راه دور است).





- **انتخاب همتا:** Interface انتخاب همتایان خروجی و ورودی برای متعادل‌سازی مجدد را آسان می‌کند.





- **تنظیمات پارامتر:** شما می‌توانید تنظیم کنید:
  - حداکثر هزینه (در Sats و ppm) که مایل به پرداخت آن هستید
  - مقدار **برای متعادل‌سازی** با گزینه "ثابت" یا "هدف"
  - گره‌هایی که باید **اجتناب شود** هنگام مسیریابی
- **حداکثر زمان آزمایش** برای یافتن مسیر





- انتخاب **کانال منبع**: ابتدا **کانال خروجی (منبع)** را انتخاب کنید، یعنی کانالی که در آن نقدینگی محلی زیادی برای انتقال دارید. در عمل، این کانالی است که سهم محلی شما در آن بالا است (> 50%). فرض کنیم یک کانال A با 1,000,000 ساتس داریم که 900,000 از آن محلی است - یک گزینه خوب برای ارسال ساتس به جای دیگر. با کلیک بر روی این کانال A به عنوان "خروجی"، ThunderHub آن را به عنوان منبع علامت‌گذاری می‌کند.





- انتخاب **کانال هدف**: سپس، **کانال ورودی (هدف)** را انتخاب کنید که نیاز به دریافت نقدینگی دارد. معمولاً این کانالی خواهد بود که وضعیت برعکس است - بیشتر وجوه در سمت دور قرار دارند (مثلاً فقط 100,000 سات محلی از 1,000,000). ThunderHub، پس از انتخاب کانال منبع، سایر کانال‌ها را به ترتیب معکوس (کاهش موجودی) مرتب می‌کند تا به شناسایی کانال‌های مکمل کمک کند. یک کانال B را انتخاب کنید که در سمت محلی جا داشته باشد. سپس ThunderHub به وضوح نشان می‌دهد که کدام دو کانال انتخاب شده‌اند (منبع A و هد





- **مقدار هزینه و تحمل را تنظیم کنید:** یک فرم به شما اجازه می‌دهد وارد کنید:





  - مقدار **برای** متعادل‌سازی مجدد (در Sats). اغلب، مقداری را انتخاب می‌کنیم که هر دو کانال را در حدود \~50/50 متعادل کند. ThunderHub می‌تواند نیمی از ظرفیت اضافی کانال منبع را به‌طور خودکار پر کند، به عنوان مثال.
  - حداکثر هزینه‌ای که مایل به پرداخت برای این عملیات هستید (اختیاری). این هزینه به صورت Sats (هزینه کل مسیریابی دایره‌ای) بیان می‌شود. اگر آن را خالی بگذارید، ThunderHub به دنبال مسیری بدون توجه به هزینه خواهد گشت، که معمولاً توصیه نمی‌شود (بهتر است یک حد تعیین کنید، مثلاً 10 Sats برای یک تعادل کوچک، یا حداکثر ppm).





- یافتن مسیر: روی دکمه کلیک کنید تا مسیری پیدا کنید. ThunderHub از LND برای محاسبه مسیری از کانال مبدا شما از طریق شبکه به کانال هدف خودتان استفاده می‌کند. اگر مسیری ممکن پیدا کند که با معیارهای هزینه شما مطابقت داشته باشد، آن را با جزئیات پرش‌ها و هزینه نمایش می‌دهد. به عنوان مثال، ممکن است نشان دهد که یک مسیر 3 پرشی با مجموع 2 Sats در هزینه‌ها پیدا کرده است.





- شروع توازن‌دهی: اگر از مسیر پیشنهادی راضی هستید، روی **توازن کانال** کلیک کنید. سپس ThunderHub پرداخت دایره‌ای را از طریق LND آغاز خواهد کرد. اگر پرداخت موفقیت‌آمیز باشد، یک اعلان موفقیت مشاهده خواهید کرد و توازن کانال‌های A و B به‌صورت لحظه‌ای تغییر خواهد کرد. ThunderHub نشانگر توازن این کانال‌ها را به‌روزرسانی خواهد کرد (به‌طور ایده‌آل، آن‌ها سبزتر از قبل خواهند بود که نشان‌دهنده توازن بهتر است).





- **تنظیمات و تکرارها:** اگر در تلاش اول مسیری پیدا نشد (یا اگر خیلی گران بود)، می‌توانید پارامترها را تنظیم کنید :





  - مقدار کمتری را امتحان کنید (گاهی اوقات یک تراز مجدد جزئی انجام می‌شود در حالی که مقدار زیادی شکست می‌خورد).
  - حداکثر هزینه را به تدریج افزایش دهید، اما مراقب باشید که بیشتر از ارزش آن هزینه نکنید.
  - از دکمه **مسیر دیگری بگیر**، در صورت موجود بودن، برای امتحان یک مسیر جایگزین استفاده کنید.
  - اگر اوضاع واقعاً پیچیده شد، یک جفت کانال دیگر را امتحان کنید.



ThunderHub فرآیند را بسیار **بصری و شهودی** می‌کند. تنها در ۴ مرحله (انتخاب کانال خروجی، انتخاب کانال ورودی، مقدار، اعتبارسنجی)، می‌توانید کاری را انجام دهید که قبلاً نیاز به دستورات پیچیده دستی داشت. این ابزار برای حفظ یک نود متعادل، بهبود عملکرد و تجربه مسیریابی شما (ارسال و دریافت پرداخت‌ها در تمام کانال‌های شما آسان‌تر می‌شود) بی‌نظیر است.



در نهایت، توجه داشته باشید که توازن مجدد هزینه‌های مسیریابی (پرداخت به گره‌های واسطه) را مصرف می‌کند، بنابراین این یک **سرمایه‌گذاری** برای روان‌تر کردن گره شماست. از آن به‌طور هوشمندانه استفاده کنید، مثلاً برای پشتیبانی از یک کانال به سرویسی که اغلب استفاده می‌کنید (نقدینگی ورودی) یا برای توازن یک کانال مسیریابی بزرگ. ThunderHub به شما اجازه می‌دهد این کار را **به‌سادگی و کارآمد** انجام دهید.



### تراکنش‌ها



بخش **تراکنش‌ها** در ThunderHub به تاریخچه تراکنش‌های **لایتنینگ** نود شما مربوط می‌شود، یعنی پرداخت‌ها و فاکتورهایی که از طریق کانال‌ها پرداخت یا دریافت شده‌اند. این یک نوع صورت‌حساب برای عملیات LN شما است.



![Historique des transactions Lightning](assets/fr/14.webp)



در این زبانه شما خواهید یافت :





- نمودار **Invoice:** در گوشه بالا سمت راست، نموداری نشان داده شده است که تکامل فاکتورهای دریافتی در طول زمان را نمایش می‌دهد و به شما امکان می‌دهد فعالیت نود خود را مشاهده کنید.





- فهرست زمانی از تمام تراکنش‌های Lightning که *از* یا *به* نود شما انجام شده‌اند. هر ورودی می‌تواند نشان دهد:





  - نوع عملیات: **پرداخت ارسال‌شده** (پرداخت خروجی) یا **پرداخت دریافت‌شده** (ورودی، از طریق Invoice پرداخت‌شده).
  - مقدار در Sats.
  - تاریخ/زمان.
  - شناسه پرداخت (Hash یا تصویر اولیه RHash) یا نظر (اگر یادداشتی به Invoice اضافه کرده‌اید).
  - وضعیت: **تکمیل شده**، یا احتمالاً **در حال انجام**/*ناموفق* (به عنوان مثال، یک پرداخت در انتظار حل و فصل، اما به طور کلی LND این را به سرعت پردازش می‌کند، بنابراین در مقایسه با تراکنش‌های On-Chain، "در انتظار" کمی در اینجا وجود دارد).



به طور خلاصه، بخش تراکنش‌ها به عنوان **دفترچه فعالیت LN** شما عمل می‌کند. این بخش برای بررسی اینکه آیا یک پرداخت انجام شده است، هزینه‌های آن چقدر بوده، یا ردیابی تاریخچه مبادلات لایتنینگ شما بسیار مفید است. همراه با بخش انتقالات (که در ادامه توضیح داده می‌شود)، شما دید کاملی از پولی که از طریق نود شما عبور کرده است خواهید داشت.



### به جلو



زبانه **Forwards** به فعالیت **مسیریابی** نود شما اختصاص دارد، یعنی پرداخت‌هایی که از طریق کانال‌های شما **عبور** می‌کنند (زمانی که شما به عنوان یک نود واسطه در Lightning Network عمل می‌کنید). اگر نود خود را به عنوان یک نود مسیریابی اداره می‌کنید، این بخش برای پیگیری عملکرد شما مهم است.



![Statistiques de routage Lightning](assets/fr/15.webp)



در Forwards، ThunderHub ارائه می‌دهد :





- **فیلترها و گزینه‌های نمایش:** در بالا سمت راست، فیلترها به شما اجازه می‌دهند داده‌ها را بر اساس روز/هفته/ماه/سال مرتب کنید و بین نمایش گرافیکی یا جدولی انتخاب کنید.





- **پیام فعالیت:** اگر در دوره انتخاب‌شده هیچ مسیریابی انجام نشده باشد، Interface پیام "هیچ انتقالی برای این دوره وجود ندارد" را نمایش می‌دهد، همان‌طور که در این مثال نشان داده شده است.





- یک **جدول از فورواردهای اخیر**: هر ورودی مربوط به پرداختی است که از طریق نود شما **فوروارد** شده است. برای هر فوروارد، معمولاً می‌بینیم:





  - Timestamp,
  - مقدار مسیریابی شده (در Sats)،
  - **کارمزد کسب‌شده** در این فوروارد (در Sats، این تفاوت بین آنچه در کانال ورودی دریافت کرده‌اید و آنچه در کانال خروجی ارسال کرده‌اید است)،
  - کانال‌های ورودی و خروجی استفاده شده (اغلب با نام مستعار همتا یا شناسه کانال شناسایی می‌شوند).
  - وضعیت (معمولاً *تکمیل شده*، یا شکست اگر یک ارسال در مسیر شکست خورده باشد).





- **آمار تجمیعی**: ThunderHub مجموع‌ها و آمارها را در بالای صفحه برای یک دوره زمانی مشخص (مثلاً 24 ساعت گذشته، یا 7 روز، و غیره، که گاهی قابل تنظیم است) محاسبه و نمایش می‌دهد.



به طور خلاصه، بخش Forwards یک **نمای کلی در زمان واقعی از فعالیت مسیریابی نود لایتنینگ شما** ارائه می‌دهد. همراه با بخش‌های Channels و Rebalance، این یک بسته کامل برای بهینه‌سازی نود شما تشکیل می‌دهد: Channels/Rebalance برای نقدینگی، Forwards برای مشاهده نتایج (جریان‌ها و سودها).



### زنجیر



بخش **Chain** مربوط به مدیریت پورتفولیو Bitcoin On-Chain از نود LND شما است. این Interface به شما اجازه می‌دهد تا وجوه Bitcoin را مشاهده و مدیریت کنید، که برای باز کردن کانال‌ها یا دریافت وجوه از کانال‌های بسته شده استفاده می‌شوند.



![Gestion du portefeuille on-chain](assets/fr/16.webp)



در زنجیره، شما خواهید یافت:





- موجودی On-Chain: **موجودی کل BTC موجود در Wallet LND را نمایش می‌دهد.**





- **لیست UTXOها:** مشاهده تمام خروجی‌های خرج‌نشده (UTXO) با مقدار، تأییدیه‌ها، Address و فرمت برای هر خروجی.





- **سابقه تراکنش:** جدول دقیق از تمام تراکنش‌های Bitcoin با نوع (ورودی/خروجی)، تاریخ، مبلغ، هزینه‌ها، تاییدیه‌ها، بلوک گنجانده شده، آدرس‌ها و txid.



### آمبوس



ThunderHub با پلتفرم **Amboss** (amboss.space) یکپارچه می‌شود، که اطلاعات دقیقی درباره نودهای Lightning، یک بازار نقدینگی و ویژگی‌های مفیدی مانند پشتیبان‌گیری رمزگذاری‌شده کانال و نظارت بر دسترسی ارائه می‌دهد.



![Intégration Amboss avec ses options](assets/fr/17.webp)



در ThunderHub، بخش Amboss به شما اجازه می‌دهد تا **گره** خود را به حساب Amboss خود متصل کنید:





- **شبح Address:** یک **Lightning Address شخصی‌سازی‌شده** برای نود خود راه‌اندازی کنید و پرداخت‌های ورودی را تسهیل کنید.





- پشتیبان‌گیری خودکار کانال‌ها:** ویژگی برجسته برای پشتیبان‌گیری رمزگذاری‌شده کانال‌ها** (فایل‌های SCB) در Amboss. در تنظیمات، **Amboss Auto Backup = Yes** را فعال کنید تا هر بار که کانال‌ها را تغییر می‌دهید، به‌طور خودکار به‌روزرسانی‌های پشتیبان رمزگذاری‌شده ارسال شود. در صورت بروز مشکل، به لطف این پشتیبان‌گیری خارجی، قادر خواهید بود وجوه خود را بازیابی کنید.





- **بررسی‌های سلامت:** فعال‌سازی **Amboss Healthcheck = Yes** برای ارسال پینگ‌های منظم از نود شما به Amboss. اگر نود شما آفلاین به نظر برسد، هشدار دریافت خواهید کرد.





- **ویژگی‌های دیگر:** انتقال خودکار موجودی، **ادغام Magma/Hydro** (بازار نقدینگی)، و دسترسی به آمار عملکرد دقیق.



ادغام Amboss یک **امنیت Layer** ضروری با پشتیبان‌گیری خارجی خودکار و نظارت بر دسترسی را اضافه می‌کند که مستقیماً از ThunderHub قابل دسترسی است.



### ابزارها



بخش **ابزارها** مجموعه‌ای از ابزارهای پیشرفته برای مدیریت نود شما را گرد هم می‌آورد. در اینجا اصلی‌ترین Elements آورده شده است:



![Interface des outils disponibles](assets/fr/18.webp)





- **پشتیبان‌گیری‌ها:** پشتیبان‌گیری‌های کانال خود را به صورت دستی مدیریت کنید (SCB). ThunderHub به شما اجازه می‌دهد تا **فایل پشتیبان کامل** کانال‌های خود را دانلود کنید (گزینه "پشتیبان‌گیری از همه کانال‌ها -> دانلود"). این فایل `channel-all.bak` را در مکانی امن نگه دارید - این فایل برای بازیابی وجوه شما در صورت وقوع خرابی ضروری است. همچنین می‌توانید هنگام استقرار مجدد یک نود، یک فایل پشتیبان را **وارد** کنید.





- **حسابداری:** ابزار صادرات برای گزارش‌های مالی شامل هزینه‌های کسب‌شده/پرداخت‌شده و حجم‌های مسیریابی‌شده در یک دوره معین.
- **پیام‌های امضا شده:** پیام‌ها را امضا یا تأیید کنید با نود خود تا Ownership نود لایتنینگ خود را از طریق امضای رمزنگاری ثابت کنید.
- ماکارون‌ها (بخش نانوایی):** مدیریت ماکارون‌های LND** برای ایجاد دسترسی سفارشی. "نانوایی" Interface به شما اجازه می‌دهد تا هر مجوز را به دقت انتخاب کنید: "افزودن یا حذف همتایان"، "ایجاد آدرس‌های زنجیره‌ای"، "ایجاد فاکتورها"، "ایجاد ماکارون‌ها"، "مشتق‌گیری کلیدها"، "دریافت کلیدهای دسترسی"، "دریافت تراکنش‌های زنجیره‌ای"، "دریافت فاکتورها"، "دریافت اطلاعات Wallet"، "دریافت پرداخت‌ها"، "دریافت همتایان"، "پرداخت فاکتورها"، "لغو شناسه‌های دسترسی"، "ارسال به آدرس‌های زنجیره‌ای"، "امضای بایت‌ها"، "امضای پیام‌ها"، "توقف daemon"، "تأیید امضای بایت‌ها"، "تأیید پیام‌ها"، و غیره. هر مجوز می‌تواند به صورت جداگانه با دکمه‌های "بله/خیر" فعال شود تا یک ماکارون سفارشی ساخته شود.
- **اطلاعات سیستم:** نمایش نسخه Wallet و RPCهای فعال شده.



به طور خلاصه، بخش ابزارها عملکردهای پیشرفته مدیریت - پشتیبان‌گیری، حسابداری، امنیت و مدیریت دسترسی - را در یک Interface یکپارچه گرد هم می‌آورد.



### تعویض



زبانه **Swap** در ThunderHub به شما امکان می‌دهد ساتوشی‌های Lightning را از طریق سرویس Boltz به Bitcoin On-Chain تبدیل کنید. این ویژگی برای "تخلیه" نقدینگی اضافی Lightning به کانال بدون بستن کانال مفید است.



![Interface de swap via Boltz](assets/fr/19.webp)



فرآیند ساده است:





- **مقدار**: مقدار قابل مبادله را تعریف کنید
- **Address** : وارد Bitcoin پذیرش Address
- **اجرا**: ThunderHub با Boltz ارتباط برقرار می‌کند تا به‌طور خودکار Exchange را پردازش کند



**مزایا:**




- خدمات غیر حضانتی (بدون نگهداری وجه نقد)
- کانال‌های موجود خود را حفظ کنید
- یکپارچه Interface با استفاده آسان



بولتز کمیسیون کمی دریافت می‌کند و شما هزینه تراکنش استاندارد Bitcoin را پرداخت می‌کنید. ThunderHub تمام هزینه‌ها را قبل از تأیید نمایش می‌دهد.



### آمار



بخش **آمار** ThunderHub آمار **پیشرفته‌ای** را در مورد نود لایتنینگ شما ارائه می‌دهد تا عملکرد را تحلیل کرده و مسیریابی را بهینه‌سازی کنید.



![Statistiques du nœud via Amboss](assets/fr/20.webp)



این بخش برای بهینه‌سازی هزینه‌ها، شناسایی کانال‌های موفق و برنامه‌ریزی گسترش نود شما ضروری است.



## نتیجه‌گیری



**ThunderHub** خود را به عنوان ابزار ضروری برای مدیریت آسان یک نود لایتنینگ **LND** تثبیت کرده است. این Interface مدرن تمامی عملکردهای اساسی را ارائه می‌دهد: مدیریت کانال، پرداخت‌ها، نظارت، با ویژگی‌های پیشرفته‌ای مانند متعادل‌سازی خودکار و یکپارچه‌سازی Amboss.



**مزایای کلیدی:**




- Interface براق و شهودی
- ابزارهای قدرتمند (متعادل‌سازی مجدد، مبادلات بولتز، پشتیبان‌گیری خودکار)
- سازگار با Umbrel، Voltage، RaspiBlitz و سایر توزیع‌ها



ThunderHub مدیریت پیشرفته نود لایتنینگ را دموکراتیزه می‌کند و دسترسی به آنچه قبلاً نیاز به دستورات فنی پیچیده داشت را فراهم می‌سازد. چه یک مبتدی باشید و چه یک اپراتور با تجربه، ThunderHub به شما اجازه می‌دهد تا نود لایتنینگ خود را به‌طور کارآمد از طریق Interface مدرن و جامع مدیریت کنید.
