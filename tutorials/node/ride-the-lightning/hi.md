## 1 परिचय



**राइड द लाइटनिंग (RTL)** एक Lightning Network नोड को प्रबंधित करने के लिए एक पूर्ण Interface वेब एप्लिकेशन है। यह स्व-होस्टेड वेब एप्लिकेशन आपके ब्राउज़र से सुलभ लाइटनिंग **कॉकपिट** प्रदान करता है। RTL सभी प्रमुख लाइटनिंग कार्यान्वयन (LND, कोर लाइटनिंग/CLN और Eclair) के साथ काम करता है और आपको अपने नोड और चैनलों पर पूर्ण नियंत्रण देता है। ओपन-सोर्स (MIT लाइसेंस) और निःशुल्क, RTL कई टर्नकी नोड समाधानों (रास्पिब्लिट्ज, मायनोड, अम्ब्रे



**ग्राफ़िकल Interface के बिना, लाइटनिंग नोड्स को केवल उपयोगकर्ता-अनुकूल CLI कमांड के माध्यम से प्रबंधित किया जा सकता है। RTL इन कार्यों को एर्गोनोमिक Interface के साथ सरल बनाता है। यहाँ** मुख्य अनुप्रयोग **दिए गए हैं:**





- **अपने चैनल और नोड देखें** - डैशबोर्ड On-Chain बैलेंस, लाइटनिंग लिक्विडिटी (स्थानीय/दूरस्थ), सिंक्रोनाइज़ेशन स्थिति, नोड उपनाम और बहुत कुछ प्रदर्शित करता है। आप अपनी चैनल सूची, क्षमता, स्थानीय/दूरस्थ वितरण और स्थिति देख सकते हैं। RTL विभिन्न कोणों से गतिविधि का विश्लेषण करने के लिए संदर्भ-संवेदनशील डैशबोर्ड प्रदान करता है।





- **लाइटनिंग चैनल प्रबंधन** - कुछ क्लिक से चैनल खोलें/बंद करें। RTL आपको किसी सहकर्मी से कनेक्ट होने और बिना किसी आदेश के चैनल खोलने की सुविधा देता है। आप रूटिंग शुल्क समायोजित कर सकते हैं, बैलेंस स्कोर देख सकते हैं, या चैनलों के बीच फंड को पुनर्संतुलित करने के लिए सर्कुलर रीबैलेंस शुरू कर सकते हैं।





- **भुगतान ट्रैक करें और करें** - RTL लाइटनिंग लेनदेन का प्रबंधन करता है: चालान के माध्यम से भुगतान भेजें, generate चालान प्राप्त करें, विस्तृत इतिहास के साथ लेनदेन (भुगतान, रूटिंग) को ट्रैक करें। Interface यह देखने के लिए रूटिंग का विश्लेषण भी करता है कि कौन से भुगतान आपके नोड से गुजर रहे हैं।





- **Wallet On-Chain प्रबंधन और बैकअप** - On-Chain टैब आपको generate पते और लेनदेन भेजने की सुविधा देता है। RTL प्रत्येक चैनल संशोधन के लिए स्वचालित अपडेट के साथ LND के लिए SCB फ़ाइल निर्यात करके चैनलों को सहेजना आसान बनाता है।



संक्षेप में, RTL **Lightning Network के लिए एक शक्तिशाली डैशबोर्ड है**, जो आपके नोड को दैनिक आधार पर संचालित करने के लिए एक शैक्षिक Interface प्रदान करता है। यह ट्यूटोरियल आपको अपने चैनल और भुगतान प्रबंधित करने के लिए इसकी स्थापना और उपयोग के माध्यम से मार्गदर्शन करेगा।



## 2. आरटीएल का तकनीकी संचालन



![Schéma de l'architecture RTL : interface responsive compatible avec tous les appareils, frontend Angular, serveur Node (port 3000), connecté aux API REST de LND](assets/fr/01.webp)



बारीकियों पर जाने से पहले, यह समझना उपयोगी है कि **RTL आपके लाइटनिंग नोड के साथ तकनीकी स्तर पर कैसे इंटरैक्ट करता है**।



**सामान्य आर्किटेक्चर:** RTL एक वेब एप्लिकेशन है जिसे Node.js (बैकएंड) और Angular (फ्रंटएंड) के साथ बनाया गया है। ठोस शब्दों में, RTL एक छोटे स्थानीय वेब सर्वर (डिफ़ॉल्ट रूप से पोर्ट 3000 पर) के रूप में चलता है जो आपके लाइटनिंग कार्यान्वयन के साथ अपने API के माध्यम से संवाद करता है। प्रकार के आधार पर:





- **LND** के साथ, RTL लाइटनिंग कमांड निष्पादित करने के लिए LND के REST API (पोर्ट 8080) का उपयोग करता है। कनेक्शन TLS द्वारा सुरक्षित है और प्रमाणीकरण के लिए LND की **admin macaroon** फ़ाइल की आवश्यकता होती है।





- **कोर लाइटनिंग (CLN)** के साथ, RTL या तो *c-lightning-REST* द्वारा प्रदान की गई REST API का उपयोग करता है, या `कमांडो` प्लगइन के माध्यम से **एक्सेस रूण** का उपयोग करता है। Umbrel जैसे समाधान स्वचालित रूप से इन Elements को कॉन्फ़िगर करते हैं।





- **Eclair** के साथ, RTL कॉन्फ़िगर किए गए प्रमाणीकरण पासवर्ड का उपयोग करके Eclair REST API से जुड़ता है।



**कॉन्फ़िगरेशन और सुरक्षा:** RTL को JSON फ़ाइल (`RTL-Config.json`) के ज़रिए कॉन्फ़िगर किया जाता है, जहाँ आप वेब पोर्ट, एक्सेस पासवर्ड और अपने नोड से कनेक्शन की जानकारी परिभाषित करते हैं। Interface वेब एक लॉगिन/पासवर्ड (डिफ़ॉल्ट `पासवर्ड` जिसे बदला जाना है) द्वारा सुरक्षित है और 2FA का समर्थन करता है। डिफ़ॉल्ट रूप से, RTL स्थानीय HTTP (`http://localhost:3000`) के रूप में चलता है, लेकिन रिमोट एक्सेस के लिए, हमेशा एक सुरक्षित कनेक्शन (रिवर्स-प्रॉक्सी, टोर या VPN के माध्यम से HTTPS) का उपयोग करें।



संक्षेप में, RTL एक अतिरिक्त घटक है जो सुरक्षित API के माध्यम से आपके नोड से जुड़ता है, उचित एक्सेस टोकन की आवश्यकता होती है, और अपनी स्वयं की सुरक्षा Layer प्रदान करता है। यह मॉड्यूलर आर्किटेक्चर आपको **एकल RTL इंस्टेंस के साथ कई लाइटनिंग नोड्स** को प्रबंधित करने की भी अनुमति देता है।



## 3. आरटीएल स्थापना



चूंकि RTL को ओपन-सोर्स सॉफ़्टवेयर के रूप में वितरित किया जाता है, इसलिए इसे आपके सिस्टम पर इंस्टॉल करने के कई तरीके हैं। इस अनुभाग में, हम दो मुख्य तरीकों को कवर करेंगे: मैन्युअल इंस्टॉलेशन और Umbrel के माध्यम से इंस्टॉलेशन।



### मैनुअल विधि



यदि आप बारीक नियंत्रण रखना चाहते हैं, या यदि आप RTL को कस्टम कॉन्फ़िगरेशन में एकीकृत कर रहे हैं, तो मैन्युअल इंस्टॉलेशन उपयुक्त है। नीचे दिए गए चरण LND नोड के लिए हैं जो Linux चला रहे हैं (उदाहरण के लिए रास्पबेरी पाई या VPS जो Ubuntu/Debian चला रहे हैं), लेकिन CLN/Eclair के लिए समान हैं।



**पूर्वापेक्षा:** सुनिश्चित करें कि आपके पास मशीन पर एक **सिंक्रोनाइज़्ड** Bitcoin नोड और एक कार्यशील लाइटनिंग नोड (LND, CLN या Eclair) है। RTL में लाइटनिंग नोड नहीं होता है, यह किसी मौजूदा नोड से जुड़ता है। आपको **Node.js** भी इंस्टॉल करना होगा (संस्करण 14+ अनुशंसित)। आप `node -v` से जाँच कर सकते हैं या आधिकारिक साइट (https://nodejs.org/en/download/) या अपने पैकेज मैनेजर से Node इंस्टॉल कर सकते हैं।



स्थापना के मुख्य चरण इस प्रकार हैं:



**RTL कोड डाउनलोड करें**: अपनी पसंद की डायरेक्टरी में आधिकारिक RTL GitHub रिपॉजिटरी को क्लोन करें। उदाहरण के लिए:



```bash
git clone https://github.com/Ride-The-Lightning/RTL.git
cd RTL
```



**निर्भरताएँ स्थापित करें**: RTL एक Node.js अनुप्रयोग है, इसलिए आपको इसके आवश्यक मॉड्यूल स्थापित करने होंगे। RTL फ़ोल्डर में, चलाएँ:



```bash
npm install --omit=dev --legacy-peer-deps
```



यह कमांड आवश्यक NPM पैकेज स्थापित करता है (विकास निर्भरताओं को अनदेखा करते हुए)। संभावित नोड निर्भरता संघर्षों से बचने के लिए `--legacy-peer-deps` विकल्प की अनुशंसा की जाती है।



**RTL-Config**: निर्भरताएँ स्थापित होने के बाद, कॉन्फ़िगरेशन फ़ाइल तैयार करें। प्रोजेक्ट रूट में `Sample-RTL-Config.json` फ़ाइल को कॉपी/नाम बदलकर `RTL-Config.json` करें। इसे अपने में खोलें:





- **यूआई पासवर्ड**: एक सुरक्षित पासवर्ड चुनें और उसे `मल्टीपास` में दर्ज करें (डिफ़ॉल्ट `"पासवर्ड"` के बजाय)।
- **पोर्ट**: डिफ़ॉल्ट `3000`. यदि यह पोर्ट आपकी मशीन पर पहले से ही लिया गया है तो आप इसे बदल सकते हैं.
- **नोड**: `nodes[0]` अनुभाग में, अपने नोड के लिए पैरामीटर समायोजित करें:
     - `lnNode`: आपके नोड के लिए एक वर्णनात्मक नाम (उदाहरणार्थ `"LND Node Maison"`).
     - lnImplementation`: `"LND"` (या `"CLN"`/`"ECL"` जैसा उपयुक्त हो).
     - `प्रमाणीकरण` के अंतर्गत:
       - macaroonPath`: LND के macaroon admin वाले फ़ोल्डर का पूर्ण पथ निर्दिष्ट करें।
       - `configPath`: नोड की कॉन्फ़िगरेशन फ़ाइल का पथ (LND के लिए `LND.conf`).
     - `सेटिंग्स` के अंतर्गत:
       - `fiatConversion`: यदि आप फिएट मुद्रा रूपांतरण चाहते हैं तो इसे `true` पर सेट करें।
       - `अघोषित चैनल`: अघोषित चैनल देखने के लिए `सत्य` पर सेट करें।
       - themeColor` और `themeMode`: Interface अनुकूलन.
       - channelBackupPath`: LND चैनल बैकअप के लिए पथ.
       - `lnServerUrl`: आपके लाइटनिंग नोड का URL (उदाहरणार्थ `https://127.0.0.1:8080`).



**RTL सर्वर प्रारंभ करें**: RTL फ़ोल्डर में, चलाएँ:



```bash
node rtl
```



एप्लिकेशन प्रारंभ हो जाता है और इसे `http://localhost:3000` पर एक्सेस किया जा सकता है।



**(वैकल्पिक) RTL को सेवा के रूप में चलाएँ**: स्वचालित स्टार्टअप के लिए, एक systemd बनाएँ:



यह करने के लिए:




- अपनी मशीन पर टर्मिनल खोलें.
- निम्नलिखित कमांड के साथ एक नई सेवा फ़ाइल बनाएं (`nano` को अपने पसंदीदा एडिटर से बदलें):


```bash
sudo nano /etc/systemd/system/RTL.service
```




- नीचे दी गई सामग्री को कॉपी करके इस फ़ाइल में पेस्ट करें:



```ini
[Unit]
Description=Ride The Lightning web UI
Wants=lnd.service
After=lnd.service

[Service]
ExecStart=/usr/bin/node /chemin/vers/RTL/rtl
User=<votre_user>
Restart=always
TimeoutSec=120
RestartSec=30

[Install]
WantedBy=multi-user.target
```





- `/path/to/RTL/rtl` को अपनी मशीन पर `rtl` फ़ाइल के वास्तविक पथ से तथा `<your_user>` को अपने Linux उपयोगकर्ता नाम से प्रतिस्थापित करें।
- फ़ाइल को सहेजें और बंद करें।
- systemd कॉन्फ़िगरेशन पुनः लोड करें:


```bash
sudo systemctl daemon-reload
```




- RTL सेवा को सक्रिय और प्रारंभ करें:


```bash
sudo systemctl enable RTL
sudo systemctl start RTL
```



RTL अब हर बार मशीन के रीबूट होने पर अपने आप शुरू हो जाएगा। आप इसकी स्थिति की जाँच इस प्रकार कर सकते हैं:


```bash
sudo systemctl status RTL
```



### अम्ब्रेल के माध्यम से स्थापना



यदि आप [Umbrel](https://getumbrel.com) का उपयोग करते हैं, तो RTL इंस्टॉलेशन बहुत सरल है:





- Interface अम्ब्रेल तक पहुंच (आमतौर पर `http://umbrel.local` के माध्यम से)
- **ऐप स्टोर** पर जाएं
- "राइड द लाइटनिंग" को खोजें



**महत्वपूर्ण नोट: अम्ब्रेल ऐप स्टोर में दो अलग-अलग RTL अनुप्रयोग हैं:**




- **राइड द लाइटनिंग** (LND के लिए): अम्ब्रेल के डिफ़ॉल्ट लाइटनिंग नोड (LND) के साथ उपयोग के लिए।
- **राइड द लाइटनिंग (कोर लाइटनिंग)**: केवल तभी उपयोग करें जब आपने अम्ब्रेल पर *कोर लाइटनिंग* एप्लिकेशन इंस्टॉल किया हो और इस नोड को RTL के साथ प्रबंधित करना चाहते हों।



*प्रत्येक RTL संस्करण को संबंधित कार्यान्वयन (LND या कोर लाइटनिंग) के साथ संवाद करने के लिए पूर्व-कॉन्फ़िगर किया गया है। यदि आपने अपना लाइटनिंग नोड नहीं बदला है, तो बस क्लासिक LND संस्करण चुनें*



![Fiche de l'application Ride The Lightning dans Umbrel : présentation de l'app avec captures d'écran et bouton violet "Install" en haut à droite](assets/fr/02.webp)





- **इंस्टॉल करें** पर क्लिक करें



![Fenêtre d'affichage du mot de passe par défaut après installation de RTL dans Umbrel, avec bouton "Open Ride The Lightning"](assets/fr/03.webp)



**महत्वपूर्ण:** स्थापना के बाद, RTL डिफ़ॉल्ट पासवर्ड के साथ एक स्क्रीन प्रदर्शित करता है। **इस पासवर्ड को कॉपी करें और सेव करें** - आपको Interface RTL पर लॉग ऑन करने के लिए इसकी आवश्यकता होगी। यह पासवर्ड हर बार RTL के शुरू होने पर प्रदर्शित किया जाएगा जब तक कि आप "इसे फिर से न दिखाएँ" विकल्प को चेक न करें।



अम्ब्रेल स्वचालित रूप से इनका ध्यान रखता है:




- RTL डाउनलोड करें और कॉन्फ़िगर करें
- लाइटनिंग नोड तक पहुंच को कॉन्फ़िगर करना
- अपडेट प्रबंधित करें
- Interface तक पहुंच सुरक्षित करना



एक बार इंस्टॉल हो जाने पर, एप्लिकेशन आपके अम्ब्रेल पर *ऐप्स* मेनू में दिखाई देता है। इसे लॉन्च करने के लिए **राइड द लाइटनिंग** पर क्लिक करें।



![Écran de connexion RTL via Umbrel : champ de mot de passe avec logo du cheval en haut à gauche, bouton "Login"](assets/fr/04.webp)



लॉगिन स्क्रीन पर, वह पासवर्ड डालें जिसे आपने पहले कॉपी किया था। लॉग इन करने के बाद, Interface वेब RTL को सीधे अम्ब्रेल डैशबोर्ड से एक्सेस किया जा सकेगा, इसके लिए किसी अतिरिक्त कॉन्फ़िगरेशन की आवश्यकता नहीं होगी!



## 4. Interface RTL का परिचय और उपयोग



अब जबकि RTL चालू हो गया है, आइए इसके Interface वेब और इसकी मुख्य विशेषताओं को देखें। हम आपको संपूर्ण जानकारी देने के लिए एप्लिकेशन के विभिन्न अनुभागों पर नज़र डालेंगे।



### मुख्य नियंत्रण पैनल



![Tableau de bord RTL : solde Lightning, solde on-chain, capacité de liquidité entrante/sortante et création de facture](assets/fr/05.webp)



जैसे ही आप लॉग इन करते हैं, आप **मुख्य डैशबोर्ड** तक पहुँच जाते हैं, जो आपको अपने लाइटनिंग नोड का अवलोकन देता है। यह पृष्ठ आवश्यक जानकारी को केंद्रीकृत करता है:




- आपका कुल लाइटनिंग बैलेंस
- On-Chain बैलेंस उपलब्ध
- आपकी तरलता का विवरण (आवक/जावक)
- अपने लाइटनिंग नोड के माध्यम से सैट्स भेजने और प्राप्त करने के लिए त्वरित पहुँच



### फंड प्रबंधन On-Chain



![Onglet "On-chain" actif dans RTL : solde Bitcoin (en sats, BTC, USD), et liste des transactions avec type d'adresse Taproot](assets/fr/06.webp)



**On-Chain** टैब आपको अपने बिटकॉइन को सीधे मुख्य श्रृंखला पर प्रबंधित करने देता है:




- विभिन्न इकाइयों में शेष राशि का प्रदर्शन (Sats, BTC, USD)
- लेन-देन की पूरी सूची
- Address पीढ़ी Taproot (P2TR), P2SH (NP2WKH) या Bech32 (P2WKH)
- UTXO प्रबंधन, बिटकॉइन प्राप्त करना और भेजना



### लाइटनिंग: सबमेनू की विस्तृत प्रस्तुति



Interface RTL में Lightning Network के लिए समर्पित एक साइड मेनू है, जो आपके नोड को प्रबंधित करने के लिए सभी आवश्यक कार्यों को एक साथ लाता है। स्क्रीनशॉट के क्रम में प्रत्येक उप-मेनू का विवरण यहां दिया गया है:



#### 1. सहकर्मी/चैनल



![Vue de gestion des canaux Lightning (onglet "Peers/Channels" ouvert)](assets/fr/07.webp)



यह उप-मेनू आपको यह करने की अनुमति देता है:




- अपने कनेक्टेड पीयर्स और खुले या प्रतीक्षारत लाइटनिंग चैनलों की सूची देखें।
- एक नया पीयर जोड़ें (किसी अन्य लाइटनिंग नोड से कनेक्ट करें).
- मौजूदा चैनल खोलें, बंद करें या प्रबंधित करें.
- प्रत्येक चैनल का विवरण देखें: क्षमता, स्थानीय/दूरस्थ शेष, स्थिति, ट्रेडिंग इतिहास, शेष स्कोर, आदि।



#### 2. लेन-देन



![Historique des transactions Lightning (onglet "Transactions" > "Payments")](assets/fr/08.webp)



इस सबमेनू में, आप यह कर सकते हैं:




- लाइटनिंग भुगतान भेजें (Invoice के माध्यम से).
- generate और भुगतान प्राप्त करने के लिए चालान प्रबंधित करें।
- भेजे गए और प्राप्त भुगतानों का पूरा इतिहास देखें, विवरण सहित (राशि, दिनांक, स्थिति, शुल्क, छोड़े गए भुगतानों की संख्या, आदि)।



#### 3. रूटिंग



यह सबमेनू प्रदर्शित करता है:




- आपके नोड द्वारा अन्य Lightning Network उपयोगकर्ताओं के लिए किए गए भुगतान।
- रूटिंग आँकड़े: रिले किए गए लेनदेन की संख्या, अर्जित शुल्क, आई त्रुटियाँ।
- उन्नत विश्लेषण के लिए निर्यात योग्य इतिहास.



#### 4. आस्थगन



यह उप-मेनू प्रदान करता है:




- आपके लाइटनिंग नोड की गतिविधि पर विस्तृत रिपोर्ट।
- चैनल, भुगतान, शुल्क, क्षमता आदि पर चार्ट और तालिकाएं।
- आपके नोड के प्रदर्शन को बेहतर ढंग से समझने के लिए उपकरण.



#### 5. ग्राफ लुकअप



यह उप-मेनू आपको यह करने की अनुमति देता है:




- Lightning Network की टोपोलॉजी का अन्वेषण करें।
- विशिष्ट नोड्स या चैनल खोजें.
- कनेक्टिविटी और समग्र नेटवर्क क्षमता के बारे में जानकारी प्राप्त करें।



#### 6. हस्ताक्षर/सत्यापन करें



यह सबमेनू प्रदान करता है:




- अपने नोड की कुंजी के साथ संदेश पर हस्ताक्षर करने की क्षमता (Ownership का प्रमाण)।
- अन्य नोड्स से संदेशों को प्रमाणित करने के लिए हस्ताक्षर सत्यापन।



#### 7. बैकअप



यह उप-मेनू बैकअप के लिए समर्पित है:




- चैनल बैकअप फ़ाइल निर्यात करें (LND के लिए SCB).
- यदि आवश्यक हो तो कॉन्फ़िगरेशन या चैनल पुनर्स्थापित करें.
- अपने बैकअप को सुरक्षित करने के लिए सुझाव.



#### 8. नोड/नेटवर्क



![Vue d'ensemble du nœud Lightning (onglet "Node/Network")](assets/fr/09.webp)



इस सबमेनू में आपको मिलेगा:




- आपके लाइटनिंग नोड की स्थिति का संपूर्ण सारांश: उपनाम, संस्करण, रंग, सिंक्रनाइज़ेशन स्थिति।
- चैनलों (सक्रिय, प्रतीक्षारत, बंद), कुल क्षमता, कनेक्टिविटी पर आंकड़े।
- वैश्विक Lightning Network और उसमें आपके नोड की स्थिति के बारे में जानकारी।



### सेवाएँ: बोल्ट्ज़ स्वैप्स



![Interface de gestion des swaps avec Boltz (onglet "Services" > "Boltz")](assets/fr/10.webp)



बोल्ट्ज़ एक गोपनीयता-अनुकूल, गैर-कस्टोडियल Exchange सेवा है जो Lightning Network और Blockchain Bitcoin (On-Chain) के बीच बिटकॉइन को परिवर्तित करती है। यह दो प्रकार के ऑपरेशन प्रदान करता है: रिवर्स सबमरीन स्वैप (**स्वैप आउट**) और सबमरीन स्वैप (**स्वैप इन**)।



#### स्वैप आउट (रिवर्स सबमरीन स्वैप)



स्वैप आउट Lightning Network पर उपलब्ध सैट्स को On-Chain बिटकॉइन में परिवर्तित करता है। यह तंत्र तब उपयोगी होता है जब नोड की आने वाली क्षमता समाप्त हो जाती है, या जब आप On-Chain Address से फंड रिकवर करना चाहते हैं। प्रक्रिया इस प्रकार काम करती है:




- उपयोगकर्ता विनिमय की जाने वाली राशि का चयन करता है
- नोड बोल्ट्ज़ को लाइटनिंग भुगतान भेजता है
- Exchange में, बोल्ट्ज़ On-Chain बिटकॉइन में समतुल्य राशि को ब्लॉक करता है
- एक बार लेन-देन की पुष्टि हो जाने पर, उपयोगकर्ता स्वैप में प्रयुक्त रहस्य का खुलासा करके धन का दावा कर सकता है



यह प्रक्रिया गैर-संरक्षक है, जिसमें बोल्ट्ज़ कभी भी उपयोगकर्ता के धन को अपने पास नहीं रखता है।


![Double capture des étapes de configuration d'un swap-out](assets/fr/11.webp)



#### स्वैप इन (पनडुब्बी स्वैप)



दूसरी ओर, स्वैप इन, On-Chain फंड को Lightning Network में पुनः इंजेक्ट करने की अनुमति देता है। यह आपके चैनलों पर आउटपुट क्षमता को बहाल करने के लिए विशेष रूप से उपयोगी है। प्रक्रिया इस प्रकार है:




- उपयोगकर्ता बोल्ट्ज़ द्वारा उत्पन्न एक विशिष्ट Address को बिटकॉइन भेजता है
- ये फंड बोल्ट्ज़ द्वारा तभी जारी किए जा सकते हैं जब वह उपयोगकर्ता के नोड द्वारा उत्पन्न लाइटनिंग Invoice का भुगतान करते हैं
- एक बार Invoice का भुगतान हो जाने के बाद, फंड लाइटनिंग चैनल पर उपलब्ध हो जाते हैं, जिससे नोड की आउटपुट क्षमता बढ़ जाती है



![Configuration d'un swap-in](assets/fr/12.webp)



ये दोनों तंत्र लाइटनिंग चैनल लिक्विडिटी को कुशलतापूर्वक प्रबंधित करने में सक्षम बनाते हैं, जबकि उपयोगकर्ता की अपने फंड पर संप्रभुता बनाए रखते हैं।



### कॉन्फ़िगरेशन और अनुकूलन



![Écran de configuration du nœud (onglet "Node Config")](assets/fr/13.webp)



**नोड कॉन्फ़िगरेशन** टैब आपको अपना अनुभव अनुकूलित करने देता है:




- अघोषित चैनलों का सक्रियण
- बिक्री प्रदर्शन को अनुकूलित करें
- Block explorer विन्यास
- Interface को समायोजित करना



### दस्तावेज़ीकरण और सहायता



![Section d'aide de RTL (onglet "Help")](assets/fr/14.webp)



अंत में, **सहायता** अनुभाग निम्नलिखित पर व्यापक दस्तावेज प्रदान करता है:




- प्रारंभिक कॉन्फ़िगरेशन
- सहकर्मी और चैनल प्रबंधन
- भुगतान और लेनदेन
- बैकअप और पुनर्स्थापना
- रिपोर्ट और आंकड़े
- हस्ताक्षर और सत्यापन
- नोड और अनुप्रयोग पैरामीटर



यह व्यापक Interface आपको अपने लाइटनिंग नोड को कुशलतापूर्वक प्रबंधित करने की सुविधा देता है, बुनियादी संचालन से लेकर उन्नत सुविधाओं तक, सभी एक सहज, सुव्यवस्थित Interface में।



## 5. उन्नत उपयोग के मामले और सुरक्षा



इस अनुभाग में, आपको RTL के साथ आगे बढ़ने और अपने लाइटनिंग नोड को सुरक्षित करने के लिए जो कुछ जानना आवश्यक है, वह बताया गया है:



### निगरानी और समस्या निवारण



अपने नोड की निगरानी करने के लिए, आप RTL डेटा (लॉग, CSV) निर्यात कर सकते हैं और उन्हें Grafana जैसे टूल में देख सकते हैं। किसी समस्या (अवरुद्ध भुगतान, निष्क्रिय चैनल) की स्थिति में, LND/CLN लॉग देखें और डायग्नोस्टिक कमांड (`lncli listchannels`, `lncli pendingchannels`, आदि) का उपयोग करें। RTL रूटिंग ईवेंट की निगरानी के लिए Interface लॉग भी प्रदान करता है।



### सुरक्षित दूरस्थ पहुँच



RTL को कभी भी सीधे इंटरनेट पर प्रदर्शित न करें। इन्हें प्राथमिकता दें:




- निजी, एन्क्रिप्टेड पहुंच के लिए **VPN** (जैसे टेलस्केल)
- सुरक्षित, अनाम पहुंच के लिए **Tor**
- रिवर्स प्रॉक्सी **HTTPS** (Nginx/Caddy) केवल तभी जब आप इसे कॉन्फ़िगर करना जानते हों



https://planb.network/tutorials/computer-security/communication/tailscale-9acbd7de-04d9-40f6-ab80-35f0dfedb632

### अच्छे सुरक्षा अभ्यास





- **अपनी पहुँच सुरक्षित रखें**: admin.macaroon या अपना RTL पासवर्ड कभी भी साझा न करें। संवेदनशील फ़ाइलों पर अनुमतियाँ सीमित करें।
- **नियमित बैकअप**: प्रत्येक संशोधन के बाद चैनल बैकअप फ़ाइल (एससीबी) को निर्यात करें और इसे नोड के बाहर संग्रहीत करें।
- **अपडेट**: RTL, अपने नोड और अम्ब्रेल को नवीनतम सुरक्षा सुधारों के साथ अद्यतन रखें।
- **गोपनीयता**: लॉग और स्क्रीनशॉट को शेयर करने से पहले उन्हें गुमनाम कर दें। अपने बैलेंस या पीयर लिस्ट को कभी भी सार्वजनिक रूप से शेयर न करें।
- **एकल पहुँच**: RTL बहु-उपयोगकर्ता नहीं है। व्यवस्थापक पहुँच साझा न करें। केवल पढ़ने के लिए पहुँच के लिए, यदि आवश्यक हो तो एक समर्पित मैकरून का उपयोग करें।



इन सिद्धांतों को लागू करके, आप जोखिमों को काफी हद तक सीमित कर सकते हैं और अपने लाइटनिंग नोड पर नियंत्रण बनाए रख सकते हैं।



## 7. निष्कर्ष



**राइड द लाइटनिंग** Bitcoin/लाइटनिंग नोड को प्रभावी ढंग से प्रबंधित करने के लिए एक आवश्यक उपकरण है, चाहे आप शुरुआती हों या उन्नत उपयोगकर्ता। यह आपके चैनल, भुगतान और नोड स्वास्थ्य को नियंत्रित करने के लिए एक स्पष्ट Interface प्रदान करता है, जबकि Lightning Network की आपकी समझ को गहरा करता है।



RTL अपनी बहु-कार्यान्वयन अनुकूलता, अपने उन्नत कार्यों (पुनर्संतुलन, स्वैप, रिपोर्ट) और अपने शैक्षणिक दृष्टिकोण के लिए जाना जाता है। सरल आवश्यकताओं के लिए, Interface अम्ब्रेल या Wallet मोबाइल पर्याप्त होगा, लेकिन RTL सक्रिय, अनुकूलित नोड प्रबंधन के लिए एकदम सही है।
