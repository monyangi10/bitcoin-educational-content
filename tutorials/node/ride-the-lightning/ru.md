## 1. Введение



**Ride The Lightning (RTL)** - это полноценное веб-приложение Interface для управления узлом Lightning Network. Это самодостаточное веб-приложение предлагает "кокпит" **Lightning**, доступный из вашего браузера. RTL работает со всеми основными реализациями Lightning (LND, Core Lightning/CLN и Eclair) и предоставляет вам полный контроль над узлом и каналами. С открытым исходным кодом (лицензия MIT) и бесплатно, RTL по умолчанию интегрирован во многие готовые решения для узлов (RaspiBlitz, MyNode, Umbrel и т. д.).



**Без графического интерфейса управление узлами Lightning может осуществляться только с помощью удобных команд CLI. RTL упрощает эти операции с помощью эргономичного интерфейса. Вот основные приложения:**





- **Просмотр каналов и узла** - На приборной панели отображается баланс On-Chain, ликвидность Lightning (локальная/удаленная), статус синхронизации, псевдоним узла и многое другое. Вы можете просмотреть список каналов, их емкость, локальное/дистанционное распределение и статус. RTL предлагает контекстно-зависимые приборные панели для анализа деятельности с разных точек зрения.





- **Молниеносное управление каналами** - открывайте/закрывайте каналы несколькими щелчками мыши. RTL позволяет подключаться к аналогу и открывать канал без команды. Вы можете настроить комиссию за маршрутизацию, просмотреть баланс или инициировать круговую ребалансировку, чтобы перераспределить средства между каналами.





- **Отслеживайте и проводите платежи** - RTL управляет транзакциями Lightning: отправляйте платежи по счетам, generate выставляет счета для получения, отслеживает транзакции (платежи, маршрутизацию) с подробной историей. Interface также анализирует маршрутизацию, чтобы увидеть, какие платежи проходят через ваш узел.





- Управление и резервное копирование **Wallet On-Chain** - Вкладка On-Chain позволяет сохранять адреса generate и отправлять транзакции. RTL позволяет легко сохранять каналы, экспортируя SCB-файл для LND, с автоматическим обновлением при каждом изменении канала.



Вкратце, RTL - это **мощная приборная панель для Lightning Network**, предлагающая образовательный Interface для ежедневного пилотирования вашего узла. В этом руководстве вы узнаете, как его установить и использовать для управления каналами и платежами.



## 2. Техническая эксплуатация RTL



![Schéma de l'architecture RTL : interface responsive compatible avec tous les appareils, frontend Angular, serveur Node (port 3000), connecté aux API REST de LND](assets/fr/01.webp)



Прежде чем перейти к деталям, полезно вкратце понять, **как RTL взаимодействует с узлом Lightning** на техническом уровне.



**Общая архитектура:** RTL - это веб-приложение, построенное на Node.js (backend) и Angular (frontend). Говоря конкретнее, RTL работает как небольшой локальный веб-сервер (по умолчанию на порту 3000), который взаимодействует с вашей реализацией Lightning через ее API. В зависимости от типа :





- При использовании **LND** RTL использует REST API LND (порт 8080) для выполнения команд Lightning. Соединение защищено TLS и требует для аутентификации файл **admin macaroon** от LND.





- В **Core Lightning (CLN)** RTL использует либо REST API, предоставляемый *c-lightning-REST*, либо **access rune** через плагин `commando`. Такие решения, как Umbrel, автоматически настраивают эти Elements.





- При использовании **Eclair** RTL подключается к Eclair REST API, используя настроенный пароль аутентификации.



**Конфигурация и безопасность:** RTL настраивается через JSON-файл (`RTL-Config.json`), в котором вы определяете веб-порт, пароль доступа и информацию о подключении к узлу. Веб-порт Interface защищен логином/паролем (по умолчанию `пароль` может быть изменен) и поддерживает 2FA. По умолчанию RTL работает как локальный HTTP (`http://localhost:3000`), но для удаленного доступа всегда используйте защищенное соединение (HTTPS через reverse-proxy, Tor или VPN).



В двух словах, RTL - это дополнительный компонент, который подключается к узлу через защищенные API, требует соответствующих токенов доступа и предлагает собственную защиту Layer. Эта модульная архитектура даже позволяет управлять **несколькими узлами Lightning с помощью одного экземпляра RTL**.



## 3. Установка RTL



Поскольку RTL распространяется как программное обеспечение с открытым исходным кодом, существует несколько способов установить его на вашу систему. В этом разделе мы рассмотрим два основных подхода: ручная установка и установка через Umbrel.



### Ручной метод



Ручная установка подходит, если вы хотите сохранить тонкий контроль или если вы интегрируете RTL в пользовательскую конфигурацию. Приведенные ниже шаги предназначены для узла LND под управлением Linux (например, Raspberry Pi или VPS под управлением Ubuntu/Debian), но аналогичны и для CLN/Eclair.



**Пререквизиты:** убедитесь, что на машине есть **синхронизированный** узел Bitcoin и работающий узел Lightning (LND, CLN или Eclair). RTL не содержит узла Lightning как такового, он подключается к существующему узлу. Вам также необходим установленный **Node.js** (рекомендуется версия 14+). Вы можете проверить это с помощью `node -v` или установить Node с официального сайта (https://nodejs.org/en/download/) или с помощью менеджера пакетов.



Основными этапами установки являются :



**Скачайте код RTL**: Клонируйте официальный репозиторий RTL на GitHub в выбранную вами директорию. Например:



```bash
git clone https://github.com/Ride-The-Lightning/RTL.git
cd RTL
```



**Установка зависимостей**: RTL - это приложение Node.js, поэтому вам нужно установить его необходимые модули. В папке RTL выполните команду :



```bash
npm install --omit=dev --legacy-peer-deps
```



Эта команда устанавливает необходимые пакеты NPM (игнорируя зависимости от разработки). Рекомендуется использовать опцию `--legacy-peer-deps`, чтобы избежать возможных конфликтов зависимостей Node.



**RTL-Config**: Когда все зависимости будут установлены, подготовьте файл конфигурации. Скопируйте/переименуйте файл `Sample-RTL-Config.json` в корне проекта в `RTL-Config.json`. Откройте его в папке :





- **UI password**: выберите надежный пароль и введите его в `multiPass` (вместо стандартного `"password"`).
- **Порт**: по умолчанию `3000`. Вы можете изменить его, если этот порт уже занят на вашей машине.
- **Node**: в разделе `nodes[0]` настройте параметры для вашего узла:
     - `lnNode`: описательное имя для вашего узла (например, `"LND Node Maison"`).
     - lnImplementation`: `"LND"` (или `"CLN"`/ `"ECL"` в зависимости от ситуации).
     - В разделе `Аутентификация`:
       - macaroonPath`: укажите полный путь к папке, содержащей админку макарон LND.
       - `configPath`: путь к файлу конфигурации узла (`LND.conf` для LND).
     - В разделе "Настройки":
       - `fiatConversion`: установите значение `true`, если вы хотите конвертировать фиатную валюту.
       - `unannouncedChannels`: установите значение `true`, чтобы видеть необъявленные каналы.
       - themeColor` и `themeMode`: Настройка Interface.
       - channelBackupPath`: путь для резервных копий каналов LND.
       - `lnServerUrl`: URL вашего узла Lightning (например, `https://127.0.0.1:8080`).



**Запустите сервер RTL**: В папке RTL запустите :



```bash
node rtl
```



Приложение запускается, и доступ к нему можно получить по адресу `http://localhost:3000`.



**(Необязательно) Запустите RTL в качестве службы**: Для автоматического запуска создайте systemd :



Для этого:




- Откройте терминал на вашем компьютере.
- Создайте новый служебный файл с помощью следующей команды (замените `nano` на ваш любимый редактор):


```bash
sudo nano /etc/systemd/system/RTL.service
```




- Скопируйте и вставьте в этот файл содержимое, приведенное ниже:



```ini
[Unit]
Description=Ride The Lightning web UI
Wants=lnd.service
After=lnd.service

[Service]
ExecStart=/usr/bin/node /chemin/vers/RTL/rtl
User=<votre_user>
Restart=always
TimeoutSec=120
RestartSec=30

[Install]
WantedBy=multi-user.target
```





- Замените `/path/to/RTL/rtl` реальным путем к файлу `rtl` на вашей машине, а `<ваш_пользователь>` - вашим именем пользователя Linux.
- Сохраните и закройте файл.
- Перезагрузите конфигурацию systemd:


```bash
sudo systemctl daemon-reload
```




- Активируйте и запустите службу RTL:


```bash
sudo systemctl enable RTL
sudo systemctl start RTL
```



Теперь RTL будет запускаться автоматически при каждой перезагрузке машины. Вы можете проверить его состояние с помощью :


```bash
sudo systemctl status RTL
```

### Услуги: Boltz Swaps



![Interface de gestion des swaps avec Boltz (onglet "Services" > "Boltz")](assets/fr/10.webp)



Boltz - это безопасный для конфиденциальности сервис, который конвертирует биткоины между Lightning Network и Blockchain Bitcoin (On-Chain). Он предлагает два типа операций: Reverse Submarine Swap (**Swap Out**) и Submarine Swap (**Swap In**).



#### Swap Out (Reverse Submarine Swap)



Swap Out конвертирует саты, доступные на Lightning Network, в биткоины On-Chain. Этот механизм полезен, когда у узла заканчивается входящая мощность, или когда вы хотите вернуть средства с On-Chain Address. Процесс работает следующим образом:




- Пользователь выбирает сумму для обмена
- Узел отправляет платеж Lightning в Boltz
- В Exchange Больц блокирует эквивалентную сумму в биткоинах On-Chain
- После подтверждения транзакции пользователь может получить средства, раскрыв секрет, использованный при обмене



Этот процесс не связан с лишением свободы, Boltz никогда не держит средства пользователя.


![Double capture des étapes de configuration d'un swap-out](assets/fr/11.webp)



#### Подмена (Подмена подводной лодки)



Swap In, с другой стороны, позволяет повторно вводить средства On-Chain в Lightning Network. Это особенно полезно для восстановления выходной мощности ваших каналов. Процедура выполняется следующим образом:




- Пользователь отправляет биткоины на определенный Address, сгенерированный Boltz
- Эти средства могут быть освобождены Больцем только в том случае, если он заплатит Lightning Invoice, генерируемый узлом пользователя
- После оплаты Invoice средства становятся доступны на канале "Молния", увеличивая производственную мощность узла



![Configuration d'un swap-in](assets/fr/12.webp)



Эти два механизма позволяют эффективно управлять ликвидностью канала Lightning, сохраняя при этом суверенитет пользователя над своими средствами.

## 5. Расширенные возможности использования и безопасность



В этом разделе мы расскажем о том, что нужно знать, чтобы продвинуться в RTL и обеспечить безопасность узла Lightning:



### Мониторинг и устранение неисправностей



Для мониторинга узла вы можете экспортировать данные RTL (журналы, CSV) и просматривать их в таких инструментах, как Grafana. В случае возникновения проблемы (заблокированный платеж, неактивный канал) обратитесь к журналам LND/CLN и используйте диагностические команды (`lncli listchannels`, `lncli pendingchannels` и т. д.). RTL также предлагает журналы Interface для мониторинга событий маршрутизации.



### Безопасный удаленный доступ



Никогда не открывайте RTL непосредственно в Интернете. Отдавайте предпочтение :




- **VPN** (например, Tailscale) для частного, зашифрованного доступа
- **Tor** для безопасного анонимного доступа
- Обратный прокси **HTTPS** (Nginx/Caddy), только если вы знаете, как его настроить



https://planb.academy/tutorials/computer-security/communication/tailscale-9acbd7de-04d9-40f6-ab80-35f0dfedb632

### Надлежащая техника безопасности





- **Защитите свой доступ**: никогда не сообщайте admin.macaroon или свой RTL-пароль. Ограничьте права доступа к важным файлам.
- **Регулярное резервное копирование**: экспортируйте файл резервного копирования канала (SCB) после каждой модификации и храните его вне узла.
- **Обновления**: поддерживайте RTL, ваш узел и Umbrel в актуальном состоянии с помощью последних исправлений безопасности.
- **Конфиденциальность**: анонимизируйте журналы и скриншоты перед тем, как поделиться ими. Никогда не публикуйте свои балансы или списки сверстников.
- **Однократный доступ**: RTL не является многопользовательским. Не используйте общий доступ администратора. Для доступа только для чтения при необходимости используйте специальный макаронник.



Применяя эти принципы, вы значительно ограничиваете риски и сохраняете контроль над своим узлом Lightning.
