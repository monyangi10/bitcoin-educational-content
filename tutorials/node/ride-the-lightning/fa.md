## ۱. مقدمه



**Ride The Lightning (RTL)** یک برنامه وب کامل Interface برای مدیریت یک نود Lightning Network است. این برنامه وب خودمیزبان یک "کاکپیت" لایتنینگ ارائه می‌دهد که از مرورگر شما قابل دسترسی است. RTL با تمام پیاده‌سازی‌های اصلی لایتنینگ (LND، Core Lightning/CLN و Eclair) کار می‌کند و کنترل کامل بر نود و کانال‌های شما را فراهم می‌کند. RTL که متن‌باز (با مجوز MIT) و رایگان است، به‌طور پیش‌فرض در بسیاری از راه‌حل‌های نود آماده به کار (RaspiBlitz، MyNode، Umbrel و غیره) یکپارچه شده است.



**بدون یک Interface گرافیکی، نودهای Lightning تنها می‌توانند از طریق دستورات کاربرپسند CLI مدیریت شوند. RTL این عملیات را با یک Interface ارگونومیک ساده می‌کند. در اینجا **کاربردهای اصلی** آمده است:**





- **کانال‌ها و نود خود را مشاهده کنید** - داشبورد موجودی On-Chain، نقدینگی لایتنینگ (محلی/راه دور)، وضعیت همگام‌سازی، نام مستعار نود و موارد دیگر را نمایش می‌دهد. شما می‌توانید لیست کانال‌های خود، ظرفیت، توزیع محلی/راه دور و وضعیت را مشاهده کنید. RTL داشبوردهای حساس به زمینه ارائه می‌دهد تا فعالیت را از زوایای مختلف تحلیل کنید.





- **مدیریت کانال لایتنینگ** - باز و بسته کردن کانال‌ها با چند کلیک. RTL به شما اجازه می‌دهد بدون نیاز به فرمان، به یک همتا متصل شده و یک کانال باز کنید. شما می‌توانید هزینه‌های مسیریابی را تنظیم کنید، امتیاز تراز را مشاهده کنید یا برای تراز کردن وجوه بین کانال‌ها، یک تراز مجدد دایره‌ای را آغاز کنید.





- **پیگیری و انجام پرداخت‌ها** - RTL تراکنش‌های لایتنینگ را مدیریت می‌کند: ارسال پرداخت‌ها از طریق فاکتورها، فاکتورهای generate برای دریافت، پیگیری تراکنش‌ها (پرداخت‌ها، مسیریابی) با تاریخچه‌ی دقیق. Interface همچنین مسیریابی را تحلیل می‌کند تا ببیند کدام پرداخت‌ها از نود شما عبور می‌کنند.





- مدیریت و پشتیبان‌گیری **Wallet On-Chain** - زبانه On-Chain به شما اجازه می‌دهد تا آدرس‌های generate را مدیریت کرده و تراکنش‌ها را ارسال کنید. RTL با صادرات فایل SCB برای LND، ذخیره کانال‌ها را آسان می‌کند و با هر تغییر کانال به‌طور خودکار به‌روزرسانی می‌شود.



به طور خلاصه، RTL یک **داشبورد قدرتمند برای Lightning Network** است که یک Interface آموزشی برای هدایت نود شما به صورت روزانه ارائه می‌دهد. این آموزش شما را در نصب و استفاده از آن برای مدیریت کانال‌ها و پرداخت‌ها راهنمایی خواهد کرد.



## ۲. عملیات فنی RTL



![Schéma de l'architecture RTL : interface responsive compatible avec tous les appareils, frontend Angular, serveur Node (port 3000), connecté aux API REST de LND](assets/fr/01.webp)



قبل از پرداختن به جزئیات، مفید است که به طور خلاصه بفهمیم **چگونه RTL به صورت فنی با نود لایتنینگ شما تعامل دارد**.



**معماری کلی:** RTL یک برنامه وب است که با Node.js (بک‌اند) و Angular (فرانت‌اند) ساخته شده است. به طور مشخص، RTL به عنوان یک سرور وب محلی کوچک (به طور پیش‌فرض روی پورت 3000) اجرا می‌شود که از طریق APIهای خود با پیاده‌سازی Lightning شما ارتباط برقرار می‌کند. بسته به نوع:





- با **LND**، RTL از REST API مربوط به LND (پورت 8080) برای اجرای دستورات Lightning استفاده می‌کند. این اتصال توسط TLS ایمن شده و برای احراز هویت به فایل **admin macaroon** مربوط به LND نیاز دارد.





- با **Core Lightning (CLN)**، RTL از API REST ارائه شده توسط *c-lightning-REST* یا یک **دسترسی رون** از طریق پلاگین `commando` استفاده می‌کند. راه‌حل‌هایی مانند Umbrel به‌طور خودکار این Elements را پیکربندی می‌کنند.





- با **Eclair**، RTL با استفاده از رمز عبور احراز هویت تنظیم‌شده به Eclair REST API متصل می‌شود.



**پیکربندی و امنیت:** RTL از طریق یک فایل JSON (`RTL-Config.json`) پیکربندی می‌شود که در آن پورت وب، رمز عبور دسترسی و اطلاعات اتصال به نود خود را تعریف می‌کنید. وب Interface با یک ورود/رمز عبور (پیش‌فرض `password` که باید تغییر کند) محافظت می‌شود و از 2FA پشتیبانی می‌کند. به طور پیش‌فرض، RTL به صورت HTTP محلی (`http://localhost:3000`) اجرا می‌شود، اما برای دسترسی از راه دور، همیشه از یک اتصال امن (HTTPS از طریق معکوس پروکسی، تور، یا VPN) استفاده کنید.



به طور خلاصه، RTL یک مؤلفه اضافی است که از طریق APIهای امن به نود شما متصل می‌شود، نیاز به توکن‌های دسترسی مناسب دارد و امنیت خاص خود را با Layer ارائه می‌دهد. این معماری مدولار حتی به شما اجازه می‌دهد **چندین نود لایتنینگ را با یک نمونه RTL مدیریت کنید**.



## ۳. نصب RTL



از آنجا که RTL به عنوان نرم‌افزار متن‌باز توزیع می‌شود، روش‌های مختلفی برای نصب آن بر روی سیستم شما وجود دارد. در این بخش، دو روش اصلی را پوشش خواهیم داد: نصب دستی و نصب از طریق Umbrel.



### روش دستی



نصب دستی مناسب است اگر می‌خواهید کنترل دقیق‌تری داشته باشید، یا اگر در حال ادغام RTL در یک پیکربندی سفارشی هستید. مراحل زیر برای یک نود LND که لینوکس اجرا می‌کند (مثلاً Raspberry Pi یا VPS که Ubuntu/Debian اجرا می‌کند) ارائه شده است، اما برای CLN/Eclair نیز مشابه است.



**پیش‌نیاز:** اطمینان حاصل کنید که یک نود Bitcoin **همگام‌سازی‌شده** و یک نود لایتنینگ فعال (LND، CLN یا Eclair) روی دستگاه دارید. RTL به خودی خود شامل یک نود لایتنینگ نیست، بلکه به یک نود موجود متصل می‌شود. همچنین نیاز به نصب **Node.js** دارید (نسخه 14+ توصیه می‌شود). می‌توانید با `node -v` بررسی کنید یا Node را از سایت رسمی (https://nodejs.org/en/download/) یا مدیر بسته خود نصب کنید.



مراحل اصلی نصب عبارتند از:



**دانلود کد RTL**: مخزن رسمی RTL را از GitHub در دایرکتوری دلخواه خود کلون کنید. برای مثال:



```bash
git clone https://github.com/Ride-The-Lightning/RTL.git
cd RTL
```



**نصب وابستگی‌ها**: RTL یک برنامه Node.js است، بنابراین شما باید ماژول‌های مورد نیاز آن را نصب کنید. در پوشه RTL، اجرا کنید:



```bash
npm install --omit=dev --legacy-peer-deps
```



این دستور بسته‌های ضروری NPM را نصب می‌کند (با نادیده گرفتن وابستگی‌های توسعه). گزینه `--legacy-peer-deps` برای جلوگیری از تعارضات احتمالی وابستگی‌های Node توصیه می‌شود.



**RTL-Config**: پس از آماده شدن وابستگی‌ها، فایل پیکربندی را آماده کنید. فایل `Sample-RTL-Config.json` را در ریشه پروژه کپی/تغییر نام دهید به `RTL-Config.json`. آن را در :





- **رمز عبور UI**: یک رمز عبور امن انتخاب کنید و آن را در `multiPass` وارد کنید (به جای مقدار پیش‌فرض `"password"`).
- **پورت**: پیش‌فرض `3000`. اگر این پورت در دستگاه شما قبلاً استفاده شده است، می‌توانید آن را تغییر دهید.
- **گره**: در بخش `nodes[0]`، پارامترهای گره خود را تنظیم کنید:
     - `lnNode`: یک نام توصیفی برای نود شما (مثلاً `"LND Node Maison"`)
     - lnImplementation`: `"LND"` (or `"CLN"`/`"ECL"` as appropriate).
     - تحت `authentication`:
       - macaroonPath`: مسیر کامل پوشه‌ای که حاوی ماکارون ادمین LND است را مشخص کنید.
       - `configPath`: مسیر فایل پیکربندی نود (`LND.conf` برای LND).
     - در زیر `تنظیمات`:
       - `fiatConversion`: در صورتی که می‌خواهید تبدیل ارز فیات انجام شود، به `true` تنظیم کنید.
       - `unannouncedChannels`: روی `true` تنظیم کنید تا کانال‌های اعلام‌نشده را ببینید.
       - رنگ‌تم` و `حالت‌تم`: سفارشی‌سازی Interface.
       - channelBackupPath`: مسیر پشتیبان‌گیری کانال LND.
       - `lnServerUrl`: آدرس URL نود لایتنینگ شما (مثلاً `https://127.0.0.1:8080`).



**سرور RTL را شروع کنید**: در پوشه RTL، اجرا کنید:



```bash
node rtl
```



برنامه راه‌اندازی می‌شود و می‌توان به آن در `http://localhost:3000` دسترسی داشت.



**(اختیاری) اجرای RTL به عنوان یک سرویس**: برای راه‌اندازی خودکار، یک systemd ایجاد کنید:



برای انجام این کار:




- یک ترمینال روی دستگاه خود باز کنید.
- با استفاده از فرمان زیر یک فایل سرویس جدید ایجاد کنید (به جای `nano` از ویرایشگر مورد علاقه خود استفاده کنید):


```bash
sudo nano /etc/systemd/system/RTL.service
```




- متن را در فایل زیر کپی و جای‌گذاری کنید:



```ini
[Unit]
Description=Ride The Lightning web UI
Wants=lnd.service
After=lnd.service

[Service]
ExecStart=/usr/bin/node /chemin/vers/RTL/rtl
User=<votre_user>
Restart=always
TimeoutSec=120
RestartSec=30

[Install]
WantedBy=multi-user.target
```





- جایگزین کردن `/path/to/RTL/rtl` با مسیر واقعی فایل `rtl` در دستگاه خود، و `<your_user>` با نام کاربری لینوکس خود.
- فایل را ذخیره کرده و ببندید.
- پیکربندی systemd را مجدداً بارگذاری کنید:


```bash
sudo systemctl daemon-reload
```




- سرویس RTL را فعال و شروع کنید:


```bash
sudo systemctl enable RTL
sudo systemctl start RTL
```



اکنون RTL به‌طور خودکار هر بار که دستگاه راه‌اندازی مجدد می‌شود، شروع به کار خواهد کرد. می‌توانید وضعیت آن را با :


```bash
sudo systemctl status RTL
```



### نصب از طریق Umbrel



اگر از [Umbrel](https://getumbrel.com) استفاده می‌کنید، نصب RTL بسیار ساده‌تر است:





- دسترسی به Interface Umbrel (معمولاً از طریق `http://umbrel.local`)
- به **App Store** بروید
- جستجو برای "Ride The Lightning"



**توجه مهم: دو برنامه RTL جداگانه در فروشگاه برنامه Umbrel وجود دارد:**




- **Ride The Lightning** (برای LND): برای استفاده با نود لایتنینگ پیش‌فرض Umbrel (LND).
- **سوار بر رعد (Core Lightning)**: فقط در صورتی استفاده کنید که برنامه *Core Lightning* را روی Umbrel نصب کرده‌اید و می‌خواهید این نود را با RTL مدیریت کنید.



*هر نسخه RTL به‌طور پیش‌فرض برای گفتگو با پیاده‌سازی مربوطه (LND یا Core Lightning) پیکربندی شده است. اگر گره لایتنینگ خود را تغییر نداده‌اید، به‌سادگی نسخه کلاسیک LND را انتخاب کنید*



![Fiche de l'application Ride The Lightning dans Umbrel : présentation de l'app avec captures d'écran et bouton violet "Install" en haut à droite](assets/fr/02.webp)





- روی **نصب** کلیک کنید



![Fenêtre d'affichage du mot de passe par défaut après installation de RTL dans Umbrel, avec bouton "Open Ride The Lightning"](assets/fr/03.webp)



**مهم:** پس از نصب، RTL صفحه‌ای با رمز عبور پیش‌فرض نمایش می‌دهد. **این رمز عبور را کپی و ذخیره کنید** - برای ورود به Interface RTL به آن نیاز خواهید داشت. این رمز عبور هر بار که RTL راه‌اندازی می‌شود نمایش داده خواهد شد تا زمانی که گزینه "دیگر نمایش نده" را انتخاب کنید.



آمبرل به‌طور خودکار مراقبت می‌کند از:




- دانلود و پیکربندی RTL
- پیکربندی دسترسی به نود Lightning
- مدیریت به‌روزرسانی‌ها
- ایمن‌سازی دسترسی به Interface



پس از نصب، برنامه در منوی *Apps* شما در Umbrel ظاهر می‌شود. برای اجرای آن روی **Ride The Lightning** کلیک کنید.



![Écran de connexion RTL via Umbrel : champ de mot de passe avec logo du cheval en haut à gauche, bouton "Login"](assets/fr/04.webp)



در صفحه ورود، رمز عبوری که قبلاً کپی کرده‌اید را وارد کنید. پس از ورود، وب RTL Interface به‌طور مستقیم از داشبورد Umbrel قابل دسترسی خواهد بود و نیازی به پیکربندی اضافی نیست!



## ۴. معرفی و استفاده از Interface RTL



اکنون که RTL راه‌اندازی شده است، بیایید به بررسی وب Interface و ویژگی‌های کلیدی آن بپردازیم. ما بخش‌های مختلف برنامه را مرور خواهیم کرد تا یک نمای کلی کامل به شما ارائه دهیم.



### پنل کنترل اصلی



![Tableau de bord RTL : solde Lightning, solde on-chain, capacité de liquidité entrante/sortante et création de facture](assets/fr/05.webp)



به محض ورود به سیستم، به **داشبورد اصلی** دسترسی پیدا می‌کنید که نمای کلی از نود لایتنینگ شما را ارائه می‌دهد. این صفحه اطلاعات ضروری را متمرکز می‌کند:




- موجودی کل لایتنینگ شما
- موجودی On-Chain
- شکست نقدینگی شما (ورودی/خروجی)
- دسترسی سریع برای ارسال و دریافت ساتس از طریق نود لایتنینگ شما



### مدیریت صندوق On-Chain



![Onglet "On-chain" actif dans RTL : solde Bitcoin (en sats, BTC, USD), et liste des transactions avec type d'adresse Taproot](assets/fr/06.webp)



زبانه **On-Chain** به شما امکان می‌دهد بیت‌کوین‌های خود را مستقیماً روی زنجیره اصلی مدیریت کنید:




- نمایش موجودی در واحدهای مختلف (Sats، BTC، USD)
- لیست کامل تراکنش‌ها
- Address نسل Taproot (P2TR)، P2SH (NP2WKH) یا Bech32 (P2WKH)
- مدیریت UTXO، دریافت و ارسال بیت‌کوین‌ها



### رعد و برق: ارائه دقیق زیرمنوها



ویژگی‌های Interface RTL شامل یک منوی جانبی اختصاصی برای Lightning Network است که تمامی عملکردهای ضروری برای مدیریت نود شما را گرد هم می‌آورد. در اینجا جزئیات هر زیرمنو به ترتیب اسکرین‌شات آمده است:



#### 1. همتاها/کانال‌ها



![Vue de gestion des canaux Lightning (onglet "Peers/Channels" ouvert)](assets/fr/07.webp)



این زیرمنو به شما اجازه می‌دهد تا:




- لیست همتایان متصل و کانال‌های لایتنینگ باز یا در انتظار خود را مشاهده کنید.
- یک همتا جدید اضافه کنید (به یک نود لایتنینگ دیگر متصل شوید).
- باز کردن، بستن یا مدیریت کانال‌های موجود.
- جزئیات هر کانال را مشاهده کنید: ظرفیت، موجودی‌های محلی/راه دور، وضعیت، تاریخچه معاملات، امتیاز موجودی، و غیره.



#### ۲. تراکنش‌ها



![Historique des transactions Lightning (onglet "Transactions" > "Payments")](assets/fr/08.webp)



در این زیرمنو، می‌توانید:




- ارسال پرداخت‌های Lightning (از طریق Invoice).
- generate و مدیریت فاکتورها برای دریافت پرداخت‌ها.
- مشاهده تاریخچه کامل پرداخت‌های ارسال و دریافت شده، با جزئیات (مبلغ، تاریخ، وضعیت، هزینه‌ها، تعداد دفعات رد شدن، و غیره).



#### ۳. مسیریابی



این زیرمنو نمایش می‌دهد :




- پرداخت‌هایی که توسط نود شما برای سایر کاربران Lightning Network هدایت می‌شود.
- آمار مسیریابی: تعداد تراکنش‌های منتقل‌شده، کارمزدهای کسب‌شده، خطاهای مواجه‌شده.
- تاریخچه قابل صادرات برای تحلیل پیشرفته.



#### ۴. تعویق‌ها



این زیرمنو ارائه می‌دهد :




- گزارش‌های دقیق درباره فعالیت نود لایتنینگ شما.
- نمودارها و جداول در مورد کانال‌ها، پرداخت‌ها، هزینه‌ها، ظرفیت، و غیره.
- ابزارهایی برای درک بهتر عملکرد نود شما.



#### ۵. جستجوی گراف



این زیرمنو به شما اجازه می‌دهد تا:




- توپولوژی Lightning Network را بررسی کنید.
- جستجو برای گره‌ها یا کانال‌های خاص.
- اطلاعاتی در مورد اتصال و ظرفیت کلی شبکه به دست آورید.



#### ۶. امضاء/تأیید



این زیرمنو ارائه می‌دهد:




- توانایی امضای یک پیام با کلید نود شما (اثبات Ownership).
- تأیید امضا برای احراز هویت پیام‌ها از سایر گره‌ها.



#### ۷. پشتیبان‌گیری



این زیرمنو به پشتیبان‌گیری اختصاص دارد:




- فایل پشتیبان کانال را صادر کنید (SCB برای LND).
- پیکربندی یا کانال‌ها را در صورت لزوم بازیابی کنید.
- نکاتی برای ایمن‌سازی نسخه‌های پشتیبان شما.



#### ۸. نود/شبکه



![Vue d'ensemble du nœud Lightning (onglet "Node/Network")](assets/fr/09.webp)



در این زیرمنو شما خواهید یافت :




- خلاصه کامل وضعیت نود لایتنینگ شما: نام مستعار، نسخه، رنگ، وضعیت همگام‌سازی.
- آمار کانال‌ها (فعال، در انتظار، بسته‌شده)، ظرفیت کل، اتصال.
- اطلاعاتی درباره‌ی Lightning Network جهانی و موقعیت نود شما در آن.



### خدمات: مبادلات بولتز



![Interface de gestion des swaps avec Boltz (onglet "Services" > "Boltz")](assets/fr/10.webp)



بولتز یک سرویس Exchange غیرامانی و حریم‌خصوصی‌محور است که بیت‌کوین‌ها را بین Lightning Network و Blockchain Bitcoin (On-Chain) تبدیل می‌کند. این سرویس دو نوع عملیات ارائه می‌دهد: تعویض معکوس زیرآبی (**Swap Out**) و تعویض زیرآبی (**Swap In**).



#### تعویض معکوس (تعویض زیردریایی معکوس)



مکانیزم Swap Out ساتس‌های موجود در Lightning Network را به بیت‌کوین‌های On-Chain تبدیل می‌کند. این مکانیزم زمانی مفید است که یک نود ظرفیت ورودی‌اش تمام شده باشد، یا زمانی که بخواهید وجوه را از یک On-Chain Address بازیابی کنید. فرآیند به صورت زیر عمل می‌کند:




- کاربر مبلغی را برای مبادله انتخاب می‌کند
- گره یک پرداخت لایتنینگ به Boltz ارسال می‌کند.
- در Exchange، بولتز مقدار معادل را در بیت‌کوین‌های On-Chain مسدود می‌کند.
- به محض تأیید تراکنش، کاربر می‌تواند با افشای یک راز استفاده شده در مبادله، وجوه را مطالبه کند.



این فرآیند غیر حضانتی است و Boltz هرگز وجوه کاربر را نگه نمی‌دارد.


![Double capture des étapes de configuration d'un swap-out](assets/fr/11.webp)



#### تعویض درون (تعویض زیردریایی)



از سوی دیگر، Swap In اجازه می‌دهد تا وجوه On-Chain مجدداً به Lightning Network تزریق شوند. این امر به‌ویژه برای بازیابی ظرفیت خروجی در کانال‌های شما مفید است. روش به شرح زیر است:




- کاربر بیت‌کوین‌ها را به یک Address خاص که توسط Boltz تولید شده است ارسال می‌کند.
- این وجوه تنها در صورتی توسط بولتز آزاد می‌شوند که او یک Lightning Invoice تولید شده توسط نود کاربر را پرداخت کند.
- پس از پرداخت Invoice، وجوه در کانال Lightning در دسترس قرار می‌گیرند و ظرفیت خروجی نود را افزایش می‌دهند.



![Configuration d'un swap-in](assets/fr/12.webp)



این دو مکانیزم امکان مدیریت کارآمد نقدینگی کانال لایتنینگ را فراهم می‌کنند، در حالی که حاکمیت کاربر بر وجوه خود را حفظ می‌کنند.



### پیکربندی و سفارشی‌سازی

زبانه **پیکربندی نود** به شما امکان می‌دهد تجربه خود را سفارشی کنید:

- نمایش فروش را سفارشی کنید

![Section d'aide de RTL (onglet "Help")](assets/fr/14.webp)



در نهایت، بخش **راهنما** مستندات جامعی را ارائه می‌دهد در مورد:

## ۵. موارد استفاده پیشرفته و امنیت



در این بخش، آنچه برای پیشرفت با RTL و ایمن‌سازی نود لایتنینگ خود باید بدانید:



### نظارت و عیب‌یابی



برای نظارت بر نود خود، می‌توانید داده‌های RTL (لاگ‌ها، CSV) را صادر کرده و آن‌ها را در ابزارهایی مانند Grafana مشاهده کنید. در صورت بروز مشکل (پرداخت مسدود شده، کانال غیرفعال)، لاگ‌های LND/CLN را بررسی کرده و از دستورات تشخیصی (`lncli listchannels`, `lncli pendingchannels`, و غیره) استفاده کنید. RTL همچنین لاگ‌های Interface را برای نظارت بر رویدادهای مسیریابی ارائه می‌دهد.



### دسترسی امن از راه دور



هرگز RTL را به‌طور مستقیم در اینترنت قرار ندهید. اولویت را به :




- **VPN** (به عنوان مثال Tailscale) برای دسترسی خصوصی و رمزگذاری شده
- **تور** برای دسترسی امن و ناشناس
- پراکسی معکوس **HTTPS** (Nginx/Caddy) فقط اگر می‌دانید چگونه آن را پیکربندی کنید



https://planb.network/tutorials/computer-security/communication/tailscale-9acbd7de-04d9-40f6-ab80-35f0dfedb632

### روش‌های ایمنی خوب





- **از دسترسی خود محافظت کنید**: هرگز admin.macaroon یا رمز عبور RTL خود را به اشتراک نگذارید. مجوزها را بر روی فایل‌های حساس محدود کنید.
- **پشتیبان‌گیری منظم**: فایل پشتیبان کانال (SCB) را پس از هر تغییر صادر کرده و آن را خارج از نود ذخیره کنید.
- **به‌روزرسانی‌ها**: RTL، نود و Umbrel خود را با آخرین اصلاحات امنیتی به‌روز نگه دارید.
- **محرمانگی**: قبل از به اشتراک‌گذاری، لاگ‌ها و اسکرین‌شات‌ها را ناشناس کنید. هرگز موجودی‌ها یا لیست‌های همتایان خود را به‌صورت عمومی به اشتراک نگذارید.
- **دسترسی تک‌کاربره**: RTL چندکاربره نیست. دسترسی ادمین را به اشتراک نگذارید. برای دسترسی فقط خواندنی، در صورت لزوم از یک ماکارون اختصاصی استفاده کنید.



با اعمال این اصول، به طور قابل توجهی ریسک‌ها را محدود کرده و کنترل بر نود لایتنینگ خود را حفظ می‌کنید.



## ۷. نتیجه‌گیری



**Ride The Lightning** ابزاری ضروری برای مدیریت مؤثر یک نود Bitcoin/Lightning است، چه شما یک کاربر مبتدی باشید یا پیشرفته. این ابزار یک Interface واضح برای کنترل کانال‌ها، پرداخت‌ها و سلامت نود شما فراهم می‌کند، در حالی که درک شما از Lightning Network را عمیق‌تر می‌سازد.



RTL به دلیل سازگاری با چندین پیاده‌سازی، عملکردهای پیشرفته‌اش (متعادل‌سازی مجدد، مبادلات، گزارش‌ها) و رویکرد آموزشی‌اش برجسته است. برای نیازهای ساده، Interface Umbrel یا یک Wallet موبایل کافی خواهد بود، اما RTL برای مدیریت فعال و بهینه‌سازی شده‌ی نود کاملاً منطقی است.
