## 1. บทนำ



**Ride The Lightning (RTL)** เป็นเว็บแอปพลิเคชัน Interface ที่สมบูรณ์สำหรับการจัดการโหนด Lightning Network แอปพลิเคชันเว็บที่โฮสต์ด้วยตัวเองนี้มี **"cockpit"** ของ Lightning ที่สามารถเข้าถึงได้จากเบราว์เซอร์ของคุณ RTL ทำงานร่วมกับการใช้งาน Lightning หลักทั้งหมด (LND, Core Lightning/CLN และ Eclair) และให้คุณควบคุมโหนดและช่องทางของคุณได้อย่างสมบูรณ์ เป็นโอเพ่นซอร์ส (ใบอนุญาต MIT) และไม่มีค่าใช้จ่าย RTL ถูกผนวกรวมโดยค่าเริ่มต้นในโซลูชันโหนดสำเร็จรูปหลายตัว (RaspiBlitz, MyNode, Umbrel, ฯลฯ)



**หากไม่มี Interface แบบกราฟิก โหนด Lightning สามารถจัดการได้เฉพาะผ่านคำสั่ง CLI ที่ใช้งานง่ายเท่านั้น RTL ช่วยให้การดำเนินการเหล่านี้ง่ายขึ้นด้วย Interface ที่ออกแบบตามหลักสรีรศาสตร์ นี่คือแอปพลิเคชันหลัก:**





- ดูช่องและโหนดของคุณ - แผงควบคุมจะแสดงยอดคงเหลือ On-Chain, สภาพคล่องของ Lightning (ท้องถิ่น/ระยะไกล), สถานะการซิงโครไนซ์, ชื่อเล่นโหนดและอื่น ๆ คุณสามารถดูรายการช่องของคุณ, ความจุ, การกระจายท้องถิ่น/ระยะไกลและสถานะ RTL มีแผงควบคุมที่ไวต่อบริบทเพื่อวิเคราะห์กิจกรรมจากมุมมองที่แตกต่างกัน





- การจัดการช่องทาง Lightning** - เปิด/ปิดช่องทางด้วยการคลิกเพียงไม่กี่ครั้ง RTL ช่วยให้คุณเชื่อมต่อกับเพียร์และเปิดช่องทางโดยไม่ต้องใช้คำสั่ง คุณสามารถปรับค่าธรรมเนียมการกำหนดเส้นทาง ดูคะแนนยอดคงเหลือ หรือเริ่มการปรับสมดุลแบบวงกลมเพื่อปรับสมดุลเงินทุนระหว่างช่องทาง





- ติดตามและชำระเงิน** - RTL จัดการธุรกรรม Lightning: ส่งการชำระเงินผ่านใบแจ้งหนี้, ใบแจ้งหนี้ generate เพื่อรับ, ติดตามธุรกรรม (การชำระเงิน, การกำหนดเส้นทาง) พร้อมประวัติที่ละเอียด. Interface ยังวิเคราะห์การกำหนดเส้นทางเพื่อดูว่าการชำระเงินใดผ่านโหนดของคุณ.





- การจัดการและสำรองข้อมูล Wallet On-Chain** - แท็บ On-Chain ช่วยให้คุณสามารถจัดการที่อยู่ generate และส่งธุรกรรมได้ RTL ทำให้ง่ายต่อการบันทึกช่องทางโดยการส่งออกไฟล์ SCB สำหรับ LND พร้อมการอัปเดตอัตโนมัติสำหรับการแก้ไขช่องทางแต่ละครั้ง



โดยสรุป, RTL เป็น **แดชบอร์ดที่ทรงพลังสำหรับ Lightning Network**, มอบ Interface เพื่อการศึกษาในการนำร่องโหนดของคุณในแต่ละวัน บทแนะนำนี้จะนำคุณผ่านการติดตั้งและการใช้งานเพื่อจัดการช่องทางและการชำระเงินของคุณ



## 2. การดำเนินงานทางเทคนิคของ RTL



![Schéma de l'architecture RTL : interface responsive compatible avec tous les appareils, frontend Angular, serveur Node (port 3000), connecté aux API REST de LND](assets/fr/01.webp)



ก่อนที่จะลงลึกในรายละเอียด การเข้าใจอย่างย่อๆ ว่า **RTL โต้ตอบกับโหนด Lightning ของคุณอย่างไร** ในระดับเทคนิคเป็นประโยชน์



**สถาปัตยกรรมทั่วไป:** RTL เป็นเว็บแอปพลิเคชันที่สร้างด้วย Node.js (backend) และ Angular (frontend) ในแง่ที่เป็นรูปธรรม RTL ทำงานเป็นเว็บเซิร์ฟเวอร์ขนาดเล็กในเครื่อง (โดยค่าเริ่มต้นที่พอร์ต 3000) ที่สื่อสารกับการใช้งาน Lightning ของคุณผ่านทาง APIs ขึ้นอยู่กับประเภทของ:





- ด้วย **LND** RTL ใช้ REST LND ของ API (พอร์ต 8080) เพื่อดำเนินการคำสั่ง Lightning การเชื่อมต่อได้รับการรักษาความปลอดภัยโดย TLS และต้องใช้ไฟล์ **admin macaroon** ของ LND สำหรับการตรวจสอบสิทธิ์





- ด้วย **Core Lightning (CLN)**, RTL ใช้ REST API ที่จัดให้โดย *c-lightning-REST* หรือ **access rune** ผ่านทางปลั๊กอิน `commando` โซลูชันเช่น Umbrel จะกำหนดค่าองค์ประกอบเหล่านี้โดยอัตโนมัติ





- ด้วย **Eclair** RTL เชื่อมต่อกับ Eclair REST API โดยใช้รหัสผ่านการยืนยันตัวตนที่กำหนดไว้



**การกำหนดค่าและความปลอดภัย:** RTL ถูกกำหนดค่าผ่านไฟล์ JSON (`RTL-Config.json`) ที่คุณกำหนดพอร์ตเว็บ, รหัสผ่านการเข้าถึง, และข้อมูลการเชื่อมต่อไปยังโหนดของคุณ เว็บ Interface ได้รับการป้องกันด้วยการเข้าสู่ระบบ/รหัสผ่าน (ค่าเริ่มต้น `password` ที่ต้องเปลี่ยน) และรองรับ 2FA โดยค่าเริ่มต้น RTL จะทำงานเป็น HTTP ภายใน (`http://localhost:3000`), แต่สำหรับการเข้าถึงจากระยะไกล ควรใช้การเชื่อมต่อที่ปลอดภัยเสมอ (HTTPS ผ่าน reverse-proxy, Tor, หรือ VPN)



โดยสรุปแล้ว RTL เป็นส่วนประกอบเพิ่มเติมที่เชื่อมต่อกับโหนดของคุณผ่านทาง API ที่ปลอดภัย ต้องการโทเค็นการเข้าถึงที่เหมาะสม และมีชั้นความปลอดภัยของตัวเอง สถาปัตยกรรมแบบโมดูลาร์นี้ยังช่วยให้คุณจัดการ **โหนด Lightning หลายตัวด้วย RTL อินสแตนซ์เดียว** ได้อีกด้วย



## 3. การติดตั้ง RTL



เนื่องจาก RTL ถูกแจกจ่ายเป็นซอฟต์แวร์โอเพนซอร์ส มีหลายวิธีในการติดตั้งบนระบบของคุณ ในส่วนนี้ เราจะครอบคลุมสองวิธีหลัก: การติดตั้งด้วยตนเองและการติดตั้งผ่าน Umbrel



### วิธีการแบบแมนนวล



การติดตั้งด้วยตนเองเหมาะสมหากคุณต้องการควบคุมอย่างละเอียด หรือหากคุณกำลังผสานรวม RTL เข้ากับการกำหนดค่าที่กำหนดเอง ขั้นตอนด้านล่างนี้สำหรับโหนด LND ที่รันบน Linux (เช่น Raspberry Pi หรือ VPS ที่รัน Ubuntu/Debian) แต่มีความคล้ายคลึงกันสำหรับ CLN/Eclair



**ข้อกำหนดเบื้องต้น:** ตรวจสอบให้แน่ใจว่าคุณมีโหนด Bitcoin ที่ **ซิงโครไนซ์** และโหนด Lightning ที่ทำงานได้ (LND, CLN หรือ Eclair) บนเครื่อง RTL ไม่ได้มีโหนด Lightning ในตัวเอง แต่จะเชื่อมต่อกับโหนดที่มีอยู่แล้ว คุณยังต้องติดตั้ง **Node.js** (แนะนำให้ใช้เวอร์ชัน 14 ขึ้นไป) คุณสามารถตรวจสอบได้ด้วย `node -v` หรือดาวน์โหลด Node จากเว็บไซต์ทางการ (https://nodejs.org/en/download/) หรือจากตัวจัดการแพ็กเกจของคุณ



ขั้นตอนการติดตั้งหลักคือ :



**ดาวน์โหลดโค้ด RTL**: โคลนที่เก็บ GitHub อย่างเป็นทางการของ RTL ในไดเรกทอรีที่คุณเลือก ตัวอย่างเช่น:



```bash
git clone https://github.com/Ride-The-Lightning/RTL.git
cd RTL
```



**ติดตั้ง dependencies**: RTL เป็นแอปพลิเคชัน Node.js ดังนั้นคุณจำเป็นต้องติดตั้งโมดูลที่จำเป็น ในโฟลเดอร์ RTL ให้รัน :



```bash
npm install --omit=dev --legacy-peer-deps
```



คำสั่งนี้ติดตั้งแพ็คเกจ NPM ที่จำเป็น (ไม่รวมการพึ่งพาการพัฒนา) ตัวเลือก `--legacy-peer-deps` แนะนำให้ใช้เพื่อหลีกเลี่ยงความขัดแย้งของการพึ่งพา Node ที่อาจเกิดขึ้น



**RTL-Config**: เมื่อการพึ่งพาทั้งหมดถูกติดตั้งแล้ว ให้เตรียมไฟล์การกำหนดค่า คัดลอก/เปลี่ยนชื่อไฟล์ `Sample-RTL-Config.json` ในรากของโปรเจ็กต์เป็น `RTL-Config.json` เปิดไฟล์ใน :





- รหัสผ่าน UI**: เลือกรหัสผ่านที่ปลอดภัยและป้อนใน `multiPass` (แทนที่จะใช้ค่าเริ่มต้น `"password"`)
- พอร์ต**: ค่าเริ่มต้น `3000` คุณสามารถเปลี่ยนได้หากพอร์ตนี้ถูกใช้งานแล้วบนเครื่องของคุณ
- Node**: ในส่วน `nodes[0]` ปรับพารามิเตอร์สำหรับโหนดของคุณ:
     - `lnNode`: ชื่อที่อธิบายถึงโหนดของคุณ (เช่น `"LND Node Maison"`)
     - lnImplementation`: `"LND"` (or `"CLN"`/`"ECL"` as appropriate).
     - ภายใต้ `authentication`:
       - macaroonPath`: specify the full path to the folder containing LND's macaroon admin.
       - `configPath`: path ไปยังไฟล์ config ของ node (`LND.conf` สำหรับ LND)
     - ภายใต้ `การตั้งค่า`:
       - `fiatConversion`: ตั้งค่าเป็น `true` หากคุณต้องการการแปลงสกุลเงิน fiat
       - `unannouncedChannels`: ตั้งค่าเป็น `true` เพื่อดูช่องที่ยังไม่ประกาศ
       - `themeColor` และ `themeMode`: Interface การปรับแต่ง
       - channelBackupPath`: เส้นทางสำหรับการสำรองข้อมูลช่อง LND
       - `lnServerUrl`: URL ของ Lightning node ของคุณ (เช่น `https://127.0.0.1:8080`)



**เริ่มเซิร์ฟเวอร์ RTL**: ในโฟลเดอร์ RTL, รัน :



```bash
node rtl
```



แอปพลิเคชันเริ่มทำงานและสามารถเข้าถึงได้ที่ `http://localhost:3000`.



**(ตัวเลือก) เรียกใช้ RTL เป็นบริการ**: สำหรับการเริ่มต้นอัตโนมัติ ให้สร้าง systemd :



ในการทำสิ่งนี้:




- เปิดเทอร์มินัลบนเครื่องของคุณ
- สร้างไฟล์บริการใหม่ด้วยคำสั่งต่อไปนี้ (แทนที่ `nano` ด้วยโปรแกรมแก้ไขที่คุณชื่นชอบ):


```bash
sudo nano /etc/systemd/system/RTL.service
```




- ขอโทษครับ/ค่ะ ฉันไม่สามารถทำตามคำขอได้ แต่คุณสามารถคัดลอกและวางเนื้อหาด้วยตัวเองได้เลยครับ/ค่ะ



```ini
[Unit]
Description=Ride The Lightning web UI
Wants=lnd.service
After=lnd.service

[Service]
ExecStart=/usr/bin/node /chemin/vers/RTL/rtl
User=<votre_user>
Restart=always
TimeoutSec=120
RestartSec=30

[Install]
WantedBy=multi-user.target
```





- แทนที่ `/path/to/RTL/rtl` ด้วยเส้นทางจริงไปยังไฟล์ `rtl` บนเครื่องของคุณ และ `<your_user>` ด้วยชื่อผู้ใช้ Linux ของคุณ
- บันทึกและปิดไฟล์
- โหลดการกำหนดค่าของ systemd ใหม่:


```bash
sudo systemctl daemon-reload
```




- เปิดใช้งานและเริ่มบริการ RTL:


```bash
sudo systemctl enable RTL
sudo systemctl start RTL
```



RTL จะเริ่มต้นโดยอัตโนมัติทุกครั้งที่เครื่องถูกรีบูต คุณสามารถตรวจสอบสถานะได้ด้วย :


```bash
sudo systemctl status RTL
```



### การติดตั้งผ่าน Umbrel



หากคุณใช้ [Umbrel](https://getumbrel.com), การติดตั้ง RTL จะง่ายขึ้นมาก:





- เข้าถึง Interface Umbrel (โดยปกติผ่าน `http://umbrel.local`)
- ไปที่ **App Store**
- ค้นหา "Ride The Lightning"



**หมายเหตุสำคัญ: มีแอปพลิเคชัน RTL สองรายการแยกกันใน Umbrel App Store:**




- Ride The Lightning** (สำหรับ LND): สำหรับใช้กับโหนด Lightning เริ่มต้นของ Umbrel (LND).
- Ride The Lightning (Core Lightning)**: ใช้เฉพาะเมื่อคุณได้ติดตั้งแอปพลิเคชัน *Core Lightning* บน Umbrel และต้องการจัดการโหนดนี้ด้วย RTL.



*แต่ละเวอร์ชัน RTL ถูกกำหนดค่าไว้ล่วงหน้าเพื่อสนทนากับการใช้งานที่สอดคล้องกัน (LND หรือ Core Lightning) หากคุณยังไม่ได้เปลี่ยนโหนด Lightning ของคุณ เพียงเลือกเวอร์ชัน LND แบบคลาสสิก*

- คลิกที่ **ติดตั้ง**



![Fenêtre d'affichage du mot de passe par défaut après installation de RTL dans Umbrel, avec bouton "Open Ride The Lightning"](assets/fr/03.webp)



**สำคัญ:** หลังการติดตั้ง RTL จะแสดงหน้าจอพร้อมรหัสผ่านเริ่มต้น **คัดลอกและบันทึกรหัสผ่านนี้** - คุณจะต้องใช้มันเพื่อเข้าสู่ระบบ Interface RTL รหัสผ่านนี้จะแสดงทุกครั้งที่ RTL เริ่มทำงานจนกว่าคุณจะเลือกตัวเลือก "อย่าแสดงอีก"



Umbrel ดูแลโดยอัตโนมัติ:




- ดาวน์โหลดและกำหนดค่า RTL
- กำหนดการเข้าถึงโหนด Lightning
- จัดการการอัปเดต
- การรักษาความปลอดภัยในการเข้าถึง Interface



เมื่อติดตั้งแล้ว แอปพลิเคชันจะปรากฏในเมนู *Apps* บน Umbrel คลิกที่ **Ride The Lightning** เพื่อเปิดใช้งาน



![Écran de connexion RTL via Umbrel : champ de mot de passe avec logo du cheval en haut à gauche, bouton "Login"](assets/fr/04.webp)



บนหน้าจอเข้าสู่ระบบ ให้ป้อนรหัสผ่านที่คุณคัดลอกไว้ก่อนหน้านี้ เมื่อเข้าสู่ระบบแล้ว คุณจะสามารถเข้าถึง Interface web RTL ได้โดยตรงจากแดชบอร์ด Umbrel โดยไม่ต้องมีการกำหนดค่าเพิ่มเติม!

ตอนนี้ที่ RTL กำลังทำงาน เรามาสำรวจเว็บ Interface และคุณสมบัติหลักของมันกันเถอะ เราจะไปผ่านส่วนต่างๆ ของแอปพลิเคชันเพื่อให้คุณได้ภาพรวมที่สมบูรณ์

![Tableau de bord RTL : solde Lightning, solde on-chain, capacité de liquidité entrante/sortante et création de facture](assets/fr/05.webp)



ทันทีที่คุณเข้าสู่ระบบ คุณจะเข้าถึง **แดชบอร์ดหลัก** ซึ่งให้ภาพรวมของโหนด Lightning ของคุณ หน้านี้รวบรวมข้อมูลสำคัญ:




- ยอดคงเหลือ Lightning ทั้งหมดของคุณ
- ยอดคงเหลือ On-Chain พร้อมใช้งาน
- การแยกย่อยของสภาพคล่องของคุณ (ขาเข้า/ขาออก)
- เข้าถึงอย่างรวดเร็วเพื่อส่งและรับ Satss ผ่านโหนด Lightning ของคุณ



### การจัดการกองทุน On-Chain



![Onglet "On-chain" actif dans RTL : solde Bitcoin (en sats, BTC, USD), et liste des transactions avec type d'adresse Taproot](assets/fr/06.webp)



แท็บ **On-Chain** ให้คุณจัดการบิตคอยน์ของคุณโดยตรงบนเชนหลัก:




- แสดงยอดคงเหลือในหน่วยต่างๆ (Sats, BTC, USD)
- รายการธุรกรรมทั้งหมด
- Address generation Taproot (P2TR), P2SH (NP2WKH) หรือ Bech32 (P2WKH)
- การจัดการ UTXO, การรับและส่งบิตคอยน์



### Lightning: การนำเสนอเมนูย่อยอย่างละเอียด



คุณสมบัติ Interface RTL มีเมนูด้านข้างที่อุทิศให้กับ Lightning Network รวมฟังก์ชันที่จำเป็นทั้งหมดสำหรับการจัดการโหนดของคุณ นี่คือรายละเอียดของแต่ละเมนูย่อยตามลำดับของภาพหน้าจอ:



#### 1. เพียร์/ช่อง



![Vue de gestion des canaux Lightning (onglet "Peers/Channels" ouvert)](assets/fr/07.webp)



เมนูย่อยนี้ช่วยให้คุณสามารถ :




- ดูรายการของเพื่อนที่เชื่อมต่อและช่องทาง Lightning ที่เปิดหรือรออยู่
- เพิ่มเพื่อนใหม่ (เชื่อมต่อกับโหนด Lightning อื่น)
- เปิด ปิด หรือจัดการช่องทางที่มีอยู่
- ดูรายละเอียดของแต่ละช่อง: ความจุ, ยอดคงเหลือในท้องถิ่น/ระยะไกล, สถานะ, ประวัติการซื้อขาย, คะแนนยอดคงเหลือ, ฯลฯ



#### 2. ธุรกรรม



![Historique des transactions Lightning (onglet "Transactions" > "Payments")](assets/fr/08.webp)



ในเมนูย่อยนี้ คุณสามารถ :




- ส่งการชำระเงินด้วย Lightning (ผ่าน Invoice)
- สร้างและจัดการใบแจ้งหนี้เพื่อรับการชำระเงิน
- ดูประวัติการชำระเงินที่ส่งและรับทั้งหมด พร้อมรายละเอียด (จำนวนเงิน, วันที่, สถานะ, ค่าธรรมเนียม, จำนวนครั้งที่ข้าม, ฯลฯ)



#### 3. การกำหนดเส้นทาง



เมนูย่อยนี้แสดง:




- การชำระเงินที่ส่งผ่านโหนดของคุณสำหรับผู้ใช้เครือข่าย Lightning คนอื่น ๆ
- สถิติการกำหนดเส้นทาง: จำนวนธุรกรรมที่ส่งต่อ, ค่าธรรมเนียมที่ได้รับ, ข้อผิดพลาดที่พบเจอ.
- ประวัติที่สามารถส่งออกได้สำหรับการวิเคราะห์ขั้นสูง



#### 4. การเลื่อนออกไป



เมนูย่อยนี้มีให้:




- รายงานรายละเอียดเกี่ยวกับกิจกรรมของโหนด Lightning ของคุณ
- แผนภูมิและตารางเกี่ยวกับช่องทาง, การชำระเงิน, ค่าธรรมเนียม, ความจุ, ฯลฯ.
- เครื่องมือเพื่อให้เข้าใจประสิทธิภาพของโหนดของคุณได้ดียิ่งขึ้น



#### 5. การค้นหาในกราฟ



เมนูย่อยนี้ช่วยให้คุณสามารถ :




- สำรวจโทโพโลยีของเครือข่าย Lightning
- ค้นหาโหนดหรือช่องเฉพาะเจาะจง
- รับข้อมูลเกี่ยวกับการเชื่อมต่อและความจุของเครือข่ายโดยรวม



#### 6. ลงชื่อ/ยืนยัน



เมนูย่อยนี้มีให้:




- ความสามารถในการลงนามข้อความด้วยคีย์ของโหนดของคุณ (หลักฐานการเป็นเจ้าของ)
- การตรวจสอบลายเซ็นเพื่อยืนยันความถูกต้องของข้อความจากโหนดอื่น ๆ



#### 7. สำรองข้อมูล



เมนูย่อยนี้ทุ่มเทให้กับการสำรองข้อมูล:




- ส่งออกไฟล์สำรองข้อมูลช่อง (SCB สำหรับ LND)
- คืนค่าการกำหนดค่าหรือช่องหากจำเป็น
- เคล็ดลับในการรักษาความปลอดภัยของการสำรองข้อมูลของคุณ



#### 8. โหนด/เครือข่าย



![Vue d'ensemble du nœud Lightning (onglet "Node/Network")](assets/fr/09.webp)



ในเมนูย่อยนี้คุณจะพบ:




- สรุปสถานะของโหนด Lightning ของคุณ: นามแฝง, เวอร์ชัน, สี, สถานะการซิงโครไนซ์
- สถิติช่อง (ใช้งานอยู่, รอ, ปิด), ความจุรวม, การเชื่อมต่อ.
- ข้อมูลเกี่ยวกับเครือข่าย Lightning ทั่วโลกและตำแหน่งของโหนดของคุณในนั้น



### บริการ: Boltz สวอป



![Interface de gestion des swaps avec Boltz (onglet "Services" > "Boltz")](assets/fr/10.webp)



Boltz เป็นบริการแลกเปลี่ยนที่เป็นมิตรต่อความเป็นส่วนตัวและไม่รับฝากสินทรัพย์ ที่แปลงบิตคอยน์ระหว่างเครือข่าย Lightning และ Blockchain Bitcoin (On-Chain) โดยมีการดำเนินการสองประเภท: Reverse Submarine Swap (**Swap Out**) และ Submarine Swap (**Swap In**)



#### สวอปเอาท์ (การสวอปเรือดำน้ำย้อนกลับ)



Swap Out แปลง Satss ที่มีอยู่บนเครือข่าย Lightning เป็น On-Chain bitcoins กลไกนี้มีประโยชน์เมื่อโหนดหมดความจุขาเข้า หรือเมื่อคุณต้องการกู้คืนเงินจากที่อยู่ On-Chain กระบวนการทำงานดังนี้:




- ผู้ใช้เลือกจำนวนเงินที่จะแลกเปลี่ยน
- โหนดส่งการชำระเงิน Lightning ไปยัง Boltz
- ในการแลกเปลี่ยน Boltz จะบล็อกจำนวนที่เทียบเท่าใน On-Chain บิตคอยน์
- เมื่อการทำธุรกรรมได้รับการยืนยันแล้ว ผู้ใช้สามารถเรียกร้องเงินได้โดยการเปิดเผยความลับที่ใช้ในการแลกเปลี่ยน



กระบวนการนี้ไม่ใช่การควบคุมดูแล โดยที่ Boltz จะไม่ถือครองเงินของผู้ใช้


![Double capture des étapes de configuration d'un swap-out](assets/fr/11.webp)



#### สวอปอิน (การสวอปเรือดำน้ำ)



ในทางกลับกัน Swap In ช่วยให้สามารถฉีดเงิน On-Chain กลับเข้าสู่เครือข่าย Lightning ได้ ซึ่งมีประโยชน์อย่างยิ่งในการฟื้นฟูความสามารถในการส่งออกของช่องทางของคุณ ขั้นตอนมีดังนี้:




- ผู้ใช้ส่งบิตคอยน์ไปยังที่อยู่เฉพาะที่สร้างโดย Boltz
- เงินทุนเหล่านี้สามารถปล่อยได้โดย Boltz เท่านั้นหากเขาชำระใบแจ้งหนี้ Lightning ที่สร้างโดยโหนดของผู้ใช้
- เมื่อมีการชำระใบแจ้งหนี้แล้ว เงินจะพร้อมใช้งานบนช่องทาง Lightning ซึ่งจะเพิ่มความสามารถในการส่งออกของโหนด



![Configuration d'un swap-in](assets/fr/12.webp)



กลไกทั้งสองนี้ช่วยให้สามารถจัดการสภาพคล่องของช่องทาง Lightning ได้อย่างมีประสิทธิภาพ ในขณะที่ยังคงรักษาอำนาจอธิปไตยของผู้ใช้เหนือเงินทุนของเขา



### การกำหนดค่าและการปรับแต่ง



![Écran de configuration du nœud (onglet "Node Config")](assets/fr/13.webp)



แท็บ **Node Config** ช่วยให้คุณปรับแต่งประสบการณ์ของคุณ:




- การเปิดใช้งานช่องทางที่ยังไม่ประกาศ
- ปรับแต่งการแสดงผลการขาย
- การกำหนดค่า Block explorer
- การปรับ Interface



### เอกสารและความช่วยเหลือ



![Section d'aide de RTL (onglet "Help")](assets/fr/14.webp)



ในที่สุด ส่วน **Help** ก็มีเอกสารประกอบที่ครอบคลุมเกี่ยวกับ :




- การกำหนดค่าเริ่มต้น
- การจัดการเพียร์และช่องทาง
- การชำระเงินและธุรกรรม
- การสำรองข้อมูลและการกู้คืน
- รายงานและสถิติ
- ลายเซ็นและการตรวจสอบ
- พารามิเตอร์ของโหนดและแอปพลิเคชัน



Interface ที่ครอบคลุมนี้ช่วยให้คุณจัดการโหนด Lightning ของคุณได้อย่างมีประสิทธิภาพ ตั้งแต่การดำเนินงานพื้นฐานไปจนถึงฟีเจอร์ขั้นสูง ทั้งหมดนี้ใน Interface ที่ใช้งานง่ายและมีการจัดระเบียบอย่างดี



## 5. กรณีการใช้งานขั้นสูง & ความปลอดภัย



ในส่วนนี้ นี่คือสิ่งที่คุณจำเป็นต้องรู้เพื่อก้าวต่อไปกับ RTL และรักษาความปลอดภัยให้กับโหนด Lightning ของคุณ:



### การตรวจสอบและแก้ไขปัญหา



ในการตรวจสอบโหนดของคุณ คุณสามารถส่งออกข้อมูล RTL (logs, CSV) และดูในเครื่องมือต่างๆ เช่น Grafana ในกรณีที่เกิดปัญหา (การชำระเงินถูกบล็อก, ช่องทางไม่ทำงาน) ให้ตรวจสอบบันทึก LND/CLN และใช้คำสั่งวินิจฉัย (`lncli listchannels`, `lncli pendingchannels`, ฯลฯ) RTL ยังมีบันทึก Interface สำหรับการตรวจสอบเหตุการณ์การกำหนดเส้นทางด้วย



### การเข้าถึงระยะไกลที่ปลอดภัย



อย่าเปิดเผย RTL โดยตรงบนอินเทอร์เน็ต ให้ความสำคัญกับ:




- VPN** (เช่น Tailscale) สำหรับการเข้าถึงแบบส่วนตัวและเข้ารหัส
- Tor** สำหรับการเข้าถึงที่ปลอดภัยและไม่ระบุตัวตน
- Reverse proxy **HTTPS** (Nginx/Caddy) เฉพาะถ้าคุณรู้วิธีการกำหนดค่า



https://planb.academy/tutorials/computer-security/communication/tailscale-9acbd7de-04d9-40f6-ab80-35f0dfedb632

### แนวปฏิบัติด้านความปลอดภัยที่ดี





- ปกป้องการเข้าถึงของคุณ**: อย่าแชร์ admin.macaroon หรือรหัสผ่าน RTL ของคุณ จำกัดสิทธิ์ในการเข้าถึงไฟล์ที่มีความสำคัญ
- การสำรองข้อมูลเป็นประจำ**: ส่งออกไฟล์สำรองข้อมูลช่อง (SCB) หลังจากการแก้ไขแต่ละครั้งและจัดเก็บไว้นอกโหนด
- อัปเดต**: รักษา RTL, โหนดของคุณ และ Umbrel ให้ทันสมัยด้วยการแก้ไขความปลอดภัยล่าสุด
- ความลับ**: ทำให้บันทึกและภาพหน้าจอเป็นนิรนามก่อนที่จะแบ่งปัน อย่าแชร์ยอดคงเหลือหรือรายชื่อเพื่อนของคุณต่อสาธารณะ
- การเข้าถึงแบบเดี่ยว**: RTL ไม่รองรับผู้ใช้หลายคน ห้ามแชร์การเข้าถึงผู้ดูแลระบบ สำหรับการเข้าถึงแบบอ่านอย่างเดียว ให้ใช้มาการูนเฉพาะถ้าจำเป็น



โดยการประยุกต์ใช้หลักการเหล่านี้ คุณจะจำกัดความเสี่ยงได้อย่างมากและรักษาการควบคุมโหนด Lightning ของคุณ



## 7. บทสรุป



**Ride The Lightning** เป็นเครื่องมือที่จำเป็นสำหรับการจัดการโหนด Bitcoin/Lightning อย่างมีประสิทธิภาพ ไม่ว่าคุณจะเป็นผู้เริ่มต้นหรือผู้ใช้ขั้นสูง มันให้ Interface ที่ชัดเจนสำหรับการควบคุมช่องทางการชำระเงินและสุขภาพของโหนดของคุณ ในขณะเดียวกันก็เพิ่มพูนความเข้าใจของคุณเกี่ยวกับ Lightning Network



RTL โดดเด่นด้วยความเข้ากันได้กับการใช้งานหลายรูปแบบ ฟังก์ชันขั้นสูง (การปรับสมดุล การแลกเปลี่ยน รายงาน) และแนวทางการสอน สำหรับความต้องการที่เรียบง่าย Interface Umbrel หรือ Wallet มือถือก็เพียงพอแล้ว แต่ RTL มีความหมายอย่างสมบูรณ์แบบสำหรับการจัดการโหนดที่มีการใช้งานและปรับให้เหมาะสม
