# 파만의 라스피블리츠 설정 가이드


라스피블리츠는 Bitcoin 노드와 관련 앱을 실행하기 위한 훌륭한 시스템입니다. 저는 대부분의 사용자에게 이 노드와 마이노드 노드를 추천합니다(이중화를 위해 두 개의 노드를 사용하는 것이 이상적입니다.) 한 가지 큰 장점은 마이노드나 엄브렐과 달리 라스피블리츠 노드는 "무료 오픈 소스 소프트웨어"라는 점입니다. (https://Bitcoin-takeover.com/why-Bitcoin-free-open-source-software-matters/amp/?__twitter_impression=true) 이더넷이 아닌 WiFi 연결로도 Raspiblitz를 실행할 수 있으며, 이에 대한 [보충 가이드](https://armantheparman.com/headless-wifi/)가 있습니다. (MyNode에서는 이 방법을 찾지 못했습니다).


미니 스크린이 부착된 기성품 노드를 구입하거나 직접 제작할 수 있습니다(스크린이 필요 없음).


깃허브](https://github.com/rootzoll/raspiblitz) 페이지의 가이드는 훌륭하지만, 경험이 어느 정도 있는 사용자에게는 너무 자세할 수 있습니다. 제 안내는 좀 더 간결하고 따라하기 쉬울 것입니다.


기본적으로 이 과정은 라즈베리파이 4로 [마이노드 노드](https://armantheparman.com/mynode-Bitcoin-node-easy-setup-guide-raspberry-pi/)를 설정하는 과정과 매우 유사합니다. 라스피블리츠 가이드에서는 모니터를 구입할 것을 권장하지만, 실제로는 모니터가 필요하지 않으며 권장하지 않습니다. 별도의 키보드나 마우스도 필요하지 않습니다. 같은 홈 네트워크에 있는 컴퓨터를 통해 디바이스의 터미널 메뉴에 액세스하고 터미널을 사용하여 ssh 명령을 사용하기만 하면 됩니다. Linux/Mac에서는 쉽고, Windows에서는 조금 더 어렵습니다.


## 1단계: 장비 구매하기.


마이노드 노드를 실행하는 데 필요한 장비와 정확히 동일한 장비가 필요합니다. 마이크로 SD 카드의 데이터만 다를 뿐 둘 중 하나를 사용해 볼 수 있습니다.



- 라즈베리 파이 4, 4Gb 메모리 또는 8Gb(4Gb면 충분)
- 공식 라즈베리 파이 파워(매우 중요! 일반적이지 않게, 진지하게)
- 파이를 위한 케이스. (FLIRC 케이스는 정말 멋집니다. 케이스 전체가 방열판으로 되어 있어 시끄러울 수 있는 팬이 필요 없습니다.)
- 32Gb microSD 카드(하나만 있으면 되지만 몇 개만 있으면 편리합니다.)
- 메모리 카드 리더기(대부분의 컴퓨터에는 microSD 카드용 슬롯이 없습니다).
- 외장 SSD 1Tb Hard 드라이브.
- 이더넷 케이블(가정용 라우터 연결용).


모니터(또는 키보드나 마우스)가 필요하지 않습니다


참고: 잘못된 Hard 드라이브입니다: 이것은 휴대용 외장형 Hard 드라이브입니다. SSD가 아닙니다. SSD가 중요합니다. 이것이 바로 저렴한 이유입니다(호주 달러 기준 가격)

## 3단계: 이미지 확인


계속 진행하기 전에 명령줄에서 파일 시스템을 사용하는 방법을 모른다면 쉽게 배울 수 있으며, 배워야 합니다.


다음은 [Linux에 유용하지만 Mac에도 적용되는 동영상](https://youtu.be/id3DGvljhT4?list=PLtK75qxsQaMLZSo7KL-PmiRarU7hrpnwK)입니다.


Windows의 경우 [간단한 튜토리얼](https://www.youtube.com/watch?v=MBBWVgE0ewk&t=1s)을 참조하세요.


_업데이트: 이제 PGP/GPG 인증이 가능합니다. Openoms의 공개키가 필요합니다. [여기](http://parman.org/downloadable/openoms.txt)에 있습니다(링크가 작동하려면 시크릿 모드가 필요할 수 있습니다 - https가 아닌 http)_


Mac/Linux


파일 다운로드가 완료될 때까지 기다린 다음(중요!) 터미널을 연 상태에서 파일을 다운로드한 위치로 이동하여 다음 명령을 입력합니다:

```bash
shasum -a 256 xxxxxxxxxxxxxx
```


여기서 `xxxxxxxxxx`는 방금 다운로드한 파일의 이름입니다. 해당 파일이 있는 디렉터리에 없는 경우 전체 경로를 입력해야 합니다.


컴퓨터가 20초 정도 생각합니다. 출력 해시파일이 이전 단계에서 웹사이트에서 다운로드한 해시파일과 일치하는지 확인합니다. 동일하면 계속 진행해도 됩니다.

Windows


명령 프롬프트를 열고 파일이 다운로드된 위치로 이동한 다음 다음 명령을 입력합니다:


```bash
certUtil -hashfile xxxxxxxxxxxxxxx SHA256
```


여기서 `xxxxxxxxxx`는 방금 다운로드한 파일의 이름입니다. 해당 파일이 있는 디렉터에 없는 경우 전체 경로를 입력해야 합니다.


컴퓨터가 20초 정도 생각합니다. 출력 해시파일이 이전 단계에서 웹사이트에서 다운로드한 해시파일과 일치하는지 확인합니다. 동일하면 계속 진행해도 됩니다.

## 6단계: Pi의 IP Address 찾기


라스피블리츠에는 모니터가 필요하지 않습니다. 하지만 홈 네트워크에 다른 컴퓨터가 필요합니다. Pi가 이더넷으로 연결되어 있지 않고 WiFi를 사용하려는 경우 IP를 찾으려면 약간의 컴퓨터 기술이 필요합니다. 도와드릴 수 없습니다. 이더넷 연결이 필요합니다. (WiFi를 연결하고 비밀번호를 입력하려면 모니터와 운영 체제에 액세스해야 하기 때문에 문제가 발생합니다.)


라우터에서 연결된 모든 디바이스의 모든 IP 목록을 확인하세요.


브라우저에 192.168.0.1(라우터와 함께 제공된 지침)을 입력하고 로그인했더니 IP 192.168.0.191로 제 기기를 확인할 수 있었습니다. 이 IP 주소는 인터넷에 공개적으로 표시되지 않으며(라우터를 먼저 통과해야 함), 홈 네트워크에 있는 디바이스의 식별자일 뿐이라는 점에 유의하세요.


IP를 찾는 것이 중요합니다.


**참고: Mac 또는 Linux 컴퓨터의 터미널에서 "arp -a" 명령을 사용하여 홈 네트워크에 연결된 모든 이더넷 연결 장치의 IP Address를 찾을 수 있습니다. 출력은 라우터에 표시되는 것만큼 예쁘지는 않지만 필요한 모든 정보가 있습니다. 어느 것이 파이인지 명확하지 않은 경우 시행착오를 수행하세요.


## 7단계: Pi로 SSH 접속


전원을 켜기 전에 SD 카드를 Pi에 넣는 것을 잊지 마세요. 몇 분 정도 기다린 다음 다른 Linux/Mac에서 터미널을 엽니다.


Mac/Linux의 경우 터미널 유형에서:


```bash
ssh admin@You_Pi's_IP_address
```


Windows의 경우, [putty](http://putty.org/)를 설치하여 Pi에 ssh를 실행해야 합니다. 위와 동일한 명령을 입력합니다.


이 작업을 처음 수행하거나 SD 카드를 전환하여 Pi의 OS를 변경할 때마다 이 오류가 발생할 수도 있고 발생하지 않을 수도 있습니다..


![image](assets/15.webp)


이 문제를 해결하는 방법은 "known_hosts" 파일이 있는 위치(오류 메시지에 표시됨)로 이동하여 해당 파일을 삭제하는 것입니다. 명령은 "rm known_hosts"입니다


그런 다음 ssh 명령을 반복하여 로그인합니다. 이렇게 하면..


![image](assets/16.webp)


계속하려면 예(전체 단어)를 입력합니다.


성공하면 비밀번호를 입력하라는 메시지가 표시됩니다. 이 비밀번호는 컴퓨터용이 아니라 raspiblitz용입니다. 일반적인 비밀번호는 "raspiblitz"이며 나중에 변경할 수 있습니다. 터미널 창이 파란색으로 바뀌고 이전 DOS 메뉴와 같은 메뉴 옵션이 표시됩니다. 화살표 키 또는 마우스로 탐색합니다.


![image](assets/17.webp)


지시에 따라 비밀번호를 설정하면 Hard 드라이브를 감지하고 필요한 경우 포맷할 수 있는 옵션이 표시됩니다.


그러면 다른 소스에서 Blockchain 데이터를 복사할 것인지 아니면 다시 다운로드할 것인지 묻는 메시지가 표시됩니다. 복사하는 것은 학습 과정이며 지침은 매우 훌륭하고 편리하게 보관하는 것이 좋습니다....


![image](assets/18.webp)


간단하지만 느린 방법은 전체 체인을 처음부터 다운로드하는 것입니다..


![image](assets/19.webp)


단말기 화면에 많은 텍스트가 깜박입니다. Blockchain 다운로드 과정으로 착각할 수도 있지만, 제가 보기에는 통신을 위한 개인 키를 생성하는 것처럼 보입니다.


그러면 번개 옵션이 나타납니다.


![image](assets/20.webp)


조명 Wallet을 잠그기 위해 새 비밀번호를 만들면 새 Wallet이 생성되고 24개의 단어를 적을 수 있습니다..


![image](assets/21.webp)


반드시 적어두고 안전하게 보관하세요. 라이트닝을 사용할 계획이 없어서 사용하지 않다가 1년 후에 사용하기로 결정하고 채널을 개설한 사람에 대해 들었습니다. 그러다가 자신의 말이 백업되지 않았고, 기기에서 다시 단어를 추출할 수 없다는 사실을 깨닫고 모든 채널을 닫고 다시 시작해야 했던 기억이 납니다. 그는 운이 좋았지만 다른 사람들은 그렇지 않을 수도 있습니다.


그 후 몇 분 동안 터미널 창 아래로 텍스트가 스크롤됩니다. 그런 다음..


![image](assets/22.webp)


Ssh 세션에서 로그아웃됩니다. 이번에는 새 비밀번호인 비밀번호 A를 사용하여 다시 로그인합니다. 로그인하면 라이트닝 Wallet의 잠금을 해제하기 위해 비밀번호 C를 입력하라는 메시지가 표시됩니다.


이제 기다립니다. 2주 후에 뵙겠습니다. 터미널을 닫아도 됩니다. 터미널은 Pi에 아무런 영향을 미치지 않으며 단지 통신 창일 뿐입니다.


![image](assets/23.webp)


어떤 이유로든 Blockchain의 다운로드가 완료되기 전에 Pi를 종료하려는 경우, 제대로 종료하기만 하면 괜찮습니다. 다시 연결하면 Blockchain은 중단된 부분부터 계속 다운로드됩니다.


CTRL+c를 눌러 블루 스크린을 종료합니다. Pi의 Linux 터미널에 액세스하게 됩니다. 여기에서 "메뉴"를 입력하면 다음 화면이 로드되고 여기에서 Pi의 전원을 끌 수 있습니다.


![image](assets/24.webp)
