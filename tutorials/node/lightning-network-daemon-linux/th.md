เครือข่าย Lightning เป็นชั้นที่สองของ Bitcoin ช่วยให้สามารถรับมิติของสายฟ้าได้ โดยเฉพาะอย่างยิ่งด้วยความเร็วและต้นทุนต่ำของการทำธุรกรรมที่เสนอ



ในบทแนะนำนี้ เราจะติดตั้งการใช้งาน Lightning Network Daemon บนเครื่อง Linux ของเรา (การแจกจ่าย Ubuntu 24.04)



## Lightning Network Daemon คืออะไร?



Lightning Network Daemon เป็นการนำเสนอ Go ที่สมบูรณ์ของเครือข่าย Lightning มันถูกสร้างขึ้นโดย Lightning Labs และช่วยให้คุณสามารถรันอินสแตนซ์เต็มรูปแบบของโหนด Lightning บนเครื่องของคุณ


กล่าวอีกนัยหนึ่ง ด้วยการใช้งานนี้ คุณสามารถ :





- โต้ตอบกับเครือข่าย Lightning**: คุณสามารถใช้บรรทัดคำสั่งเพื่อสร้างกระเป๋าเงิน Lightning จัดการช่องทางการชำระเงินและเส้นทาง และอื่นๆ อีกมากมายได้โดยตรงจากเทอร์มินัลเครื่องของคุณ
- การเชื่อมโยงโหนด Bitcoin ระยะไกลหรืออินสแตนซ์ Bitcoin Core ของคุณเอง**: LND ช่วยให้คุณเชื่อมโยงอินสแตนซ์ Bitcoin และใช้เป็นแบ็กเอนด์ของคุณได้ ในการใช้งานนี้ คุณไม่จำเป็นต้องรันอินสแตนซ์ Bitcoin Core บนเครื่องของคุณ




https://planb.academy/fr/tutorials/node/bitcoin/bitcoin-core-linux-568c13a6-8746-4d63-8e95-f4a61c5ae0ed

## ทำไมต้องมี Lightning node ของตัวเอง?


Lightning เป็นการซ้อนทับ Bitcoin ที่เพิ่มประสิทธิภาพกระบวนการโอนและลดต้นทุนการทำธุรกรรม



ด้วยการหมุนเวียนโหนด Lightning ของคุณ คุณจะได้รับอธิปไตยและความเป็นอิสระ คุณควบคุมเงินทุนของคุณเอง ดังนั้นโปรดจำไว้ว่า:



"ไม่ใช่กุญแจของคุณ ไม่ใช่บิตคอยน์ของคุณ"



ในแง่นี้ การรันโหนด Lightning จะเพิ่มความปลอดภัยและความสมบูรณ์ของข้อมูลของคุณในวิธีต่อไปนี้:





- การควบคุมทั้งหมด**: จัดการช่องทางการชำระเงินของคุณเอง กลายเป็นธนาคารของคุณเอง และเป็นเจ้านายของทรัพย์สินของคุณ
- ความลับ**: ทำธุรกรรมโดยไม่ต้องพึ่งพาบุคคลที่สามในการปกป้องความเป็นส่วนตัวของคุณ
- การเรียนรู้และความเป็นอิสระ**: ด้วยคำสั่ง `lncli` คุณสามารถเข้าใจกระบวนการพื้นฐานของ Lightning ได้ดีขึ้นโดยการฝึกฝนจากเทอร์มินัลของคุณเอง
- การกระจายอำนาจ**: มีส่วนร่วมอย่างแข็งขันในการเสริมสร้างและกระจายอำนาจเครือข่าย Bitcoin / Lightning



https://planb.academy/courses/65c138b0-4161-4958-bbe3-c12916bc959c


คุณมีสองตัวเลือกสำหรับการรันอินสแตนซ์ของการใช้งาน LND บนเครื่องของเรา เราสามารถตั้งค่าสภาพแวดล้อมบนเครื่องของเราเองเพื่อรันในเครื่อง หรือจะติดตั้ง LND จาก Docker container ก็ได้ ที่นี่เราจะมุ่งเน้นไปที่ตัวเลือกแรก และดูวิธีการดำเนินการกับ Docker ในบทเรียนถัดไป


## ติดตั้ง LND จากซอร์สโค้ด



### ข้อกำหนดเบื้องต้น


เนื่องจาก LND ถูกเขียนด้วยภาษา Go คุณจำเป็นต้องตรวจสอบให้แน่ใจว่าคุณมีสภาพแวดล้อม GoLang และการพึ่งพาที่จำเป็นบนเครื่อง Linux ของคุณ





- ข้อกำหนดฮาร์ดแวร์:**


สำหรับประสบการณ์ที่ราบรื่นและไร้รอยต่อ เครื่องของคุณจะต้องมีความจุที่จำเป็นในการรันโหนด LND Lightning ของคุณ



คุณจะต้องใช้ :


1. **8 GB RAM** สำหรับความลื่นไหลที่ดีที่สุด,


2. **โปรเซสเซอร์แบบหลายคอร์ (ควอดคอร์หรือมากกว่า)** เพื่อจัดการการทำงานของโหนดของคุณอย่างมีประสิทธิภาพ,


3. **พื้นที่ดิสก์อย่างน้อย 5GB** สำหรับโหมดโหนด pruned และ 1TB เพื่อรัน Bitcoin Core (ไม่จำเป็นหากใช้โหนดระยะไกล)





- ติดตั้งการพึ่งพาที่มีประโยชน์:


คำสั่งด้านล่างนี้จะช่วยให้คุณติดตั้งเครื่องมือที่จำเป็นสำหรับการรัน LND บนเครื่องของคุณ นอกจากสิ่งอื่น ๆ แล้ว คุณจะต้องติดตั้ง `Git` ซึ่งเป็นเครื่องมือจัดการเวอร์ชัน และ `make` ซึ่งสามารถดำเนินการและสร้างการใช้งาน LND จากซอร์สโค้ด



```bash
sudo apt install -y build-essential git make
```



![installation-dependances](assets/fr/01.webp)





- ติดตั้ง GoLang บนเครื่อง Linux ของคุณ**



ณ วันที่ของบทแนะนำนี้ LND ต้องการเวอร์ชัน 1.23.6 ของ **Go** สำหรับการติดตั้ง



หากคุณมีเวอร์ชันก่อนหน้านี้ติดตั้งอยู่แล้ว ให้ลบออกเพื่อการติดตั้ง Go ใหม่


```bash
# Suppression d'une ancienne version de Go
sudo rm -rf /usr/local/go

# Installation de la version 1.23.6 de Go
wget https://golang.org/dl/go1.23.6.linux-amd64.tar.gz

# Decompression du package

sudo tar -C /usr/local -xzf go1.23.6.linux-amd64.tar.gz

```



![go-install](assets/fr/02.webp)





- การกำหนดค่าของสภาพแวดล้อม Go**


ในไฟล์ `~/.bashrc` ของคุณ ให้เริ่มต้นตัวแปรสภาพแวดล้อมต่อไปนี้เพื่อเพิ่ม Go ให้กับระบบ Linux ของคุณ



```bash
# Configuration de l'environnement Go
export GOROOT=/usr/local/go
export GOPATH=~/gocode
export PATH=$PATH:$GOROOT/bin

# Appliquer les modifications

source ~/.bashrc
```





- กำลังตรวจสอบ **installation** (ในภาษาฝรั่งเศส)


```bash
go version
```



![go-version](assets/fr/03.webp)


### คัดลอกที่เก็บ GitHub LND



ใช้ git เพื่อคัดลอกซอร์สโค้ดของ LND มายังเครื่องของคุณ

### สร้างและติดตั้ง



เครื่องมือ `make` ที่ติดตั้งไว้ก่อนหน้านี้จะช่วยให้คุณสร้างไฟล์ปฏิบัติการจากซอร์สโค้ด LND และดำเนินการติดตั้งต่อไปได้



```bash

cd lnd

# construire LND
make
```



ติดตั้ง LND บนเครื่องของคุณ



```bash
# installer LND
make install
```



![make-lnd](assets/fr/06.webp)




- การตรวจสอบการติดตั้งของคุณ** (เป็นภาษาฝรั่งเศส)



เพื่อให้แน่ใจว่าทุกอย่างเป็นไปอย่างราบรื่น การรันคำสั่งนี้ควรจะแสดงเวอร์ชันของ LND ที่คุณกำลังใช้งานอยู่



```bash
# Version de LND
lnd --version

# Version  de LNCLI
lncli --version
```


![lnd-version](assets/fr/05.webp)




- การบำรุงรักษาและการอัปเกรด



```bash
cd lnd
git pull
make clean && make && make install
```


⚠️ **IMPORTANT**: การอัปเดต LND อาจต้องการเวอร์ชันที่ใหม่กว่าของ Go ดังนั้นโปรดอัปเดตระบบของคุณเพื่อหลีกเลี่ยงปัญหาการพึ่งพาในระหว่างการติดตั้ง


### กำลังกำหนดค่า Lightning Network Daemon



การกำหนดค่าของโหนด Lightning LND นั้นคล้ายกับของ Bitcoin และดำเนินการในไฟล์การกำหนดค่าที่มีพารามิเตอร์ทั้งหมดของโหนดของคุณ ในการทำเช่นนี้ ที่รูทของเครื่องของคุณ คุณสามารถสร้างโฟลเดอร์ที่ซ่อนอยู่ `.LND` และจากนั้นสร้างไฟล์การกำหนดค่า `LND.conf` ในโฟลเดอร์นี้



```bash
# Création du ficher
mkdir -p ~/.lnd

cd ~/.lnd

# Fichier de configuration
touch lnd.conf
```





ในไฟล์การกำหนดค่า คุณสามารถตั้งค่าโหนด LND ของคุณได้



```
noseedbackup=0
debuglevel=debug

[Bitcoin]
bitcoin.active=1
bitcoin.mainnet=1
bitcoin.node=bitcoind

[Bitcoind]
bitcoind.rpcuser=<UTILISATEUR_BITCOIN>
bitcoind.rpcpassword=<MOT_DE_PASSE_BITCOIN>
bitcoind.zmqpubrawblock=tcp://127.0.0.1:28332
bitcoind.zmqpubrawtx=tcp://127.0.0.1:28333

```



## ทำความเข้าใจการกำหนดค่าของคุณ



สิ่งสำคัญคือคุณต้องเข้าใจการกำหนดค่าขั้นต่ำที่คุณต้องการสำหรับการติดตั้งโหนด LND ของคุณให้ถูกต้องและสมบูรณ์



จากเนื้อหาของไฟล์ `~/.LND/LND.conf` นี่คือรายละเอียดของฟิลด์:





- noseedbackup**: อนุญาตให้คุณเลือกว่าต้องการให้ LND ทำการสำรองข้อมูลกระเป๋าเงินของคุณโดยอัตโนมัติหรือไม่ การตั้งค่านี้เป็น `0` จะอนุญาตให้คุณบันทึกข้อมูลการกู้คืนด้วยตนเองในสถานที่ที่ปลอดภัยที่คุณเลือกเอง





- debuglevel**: ช่วยให้คุณกำหนดระดับความละเอียดของข้อผิดพลาดและบันทึกในกรณีที่เกิดข้อผิดพลาดระหว่างการดำเนินการ





- Bitcoin.active**: สั่งให้ LND ทำงานเป็นโหนด Bitcoin และโต้ตอบกับเครือข่าย Bitcoin





- Bitcoin.Mainnet**: ระบุให้ LND เชื่อมต่อกับเครือข่ายหลักของ Bitcoin (Mainnet) คุณสามารถตั้งค่าค่า `bitcoind.signet` และ `bitcoind.regtest` สำหรับเครือข่าย Bitcoin Signet และ Bitcoin Regtest ตามลำดับ





- Bitcoin.node**: ระบุประเภทของโหนด Bitcoin ที่ LND ควรเชื่อมต่อ





- Bitcoin.rpcuser** และ **Bitcoin.rpcpassword** : แทน.


ตามลำดับการเข้าสู่ระบบ (ผู้ใช้, รหัสผ่าน) เพื่อเชื่อมต่อกับโหนด Bitcoin ของคุณ





- bitcoind.zmqpubrawblock** และ **bitcoind.zmqpubrawtx**: กำหนด ZeroMQ endpoints เพื่อรับการแจ้งเตือนเกี่ยวกับบล็อกและธุรกรรมใหม่บนเครือข่าย Bitcoin ตามลำดับ

⚠️ **สำคัญ**: โปรดจดบันทึก 24 คำ **seed phrase** ของคุณอย่างระมัดระวัง คุณจะต้องใช้มันเพื่อกู้คืนเงินของคุณในกรณีที่เกิดปัญหา



นอกจากนี้ ให้บันทึกรหัสผ่าน wallet ของคุณเพื่อที่คุณจะสามารถปลดล็อกมันด้วยคำสั่ง `lncli unlock` เมื่อคุณรีสตาร์ทโหนด LND ของคุณ

### การจัดการช่องทาง



ช่อง Lightning ช่วยให้คุณมี**การเชื่อมต่อแบบคู่ต่อคู่โดยตรงกับโหนดอื่นบนเครือข่าย Lightning** ในช่องนี้ คุณสามารถแลกเปลี่ยน Satoshis ได้อย่างอิสระจนถึงความจุของช่อง



### เชื่อมต่อกับโหนด



การเชื่อมต่อกับโหนด Lightning อื่น ๆ เป็นการกระทำพื้นฐานหากคุณต้องการมีส่วนร่วมอย่างแข็งขันและได้รับประโยชน์จากพลังของ Lightning



ในการเชื่อมต่อกับเพียร์ (Lightning node) คุณจะต้องมีข้อมูลสามส่วน:




- คีย์สาธารณะของโหนด**: นี่คือรหัสประจำตัวเฉพาะของโหนดในเครือข่าย Bitcoin;
- IP**: IP ของเครื่องที่ติดตั้งโหนด;
- PORT**: พอร์ตที่เปิดบนเครื่องที่อนุญาตให้สื่อสารกับโหนดนี้



คุณสามารถค้นหาโหนดเพื่อเชื่อมต่อได้ที่ [amboss](https://amboss.space/), แพลตฟอร์มที่แสดงข้อมูลเกี่ยวกับโหนด Lightning.



```bash
# Se connecter à un noeud
lncli connect <ID_PUBKEY>@<IP>:<PORT>

# Un exemple  : Connexion au noeud de Wallet of Satoshi
lncli connect 035e4ff418fc8b5554c5d9eea66396c227bd429a3251c8cbc711002ba215bfc226@170.75.163.209:9735
```




ตรวจสอบให้แน่ใจว่าคุณเชื่อมต่อกับ **โหนดที่เชื่อถือได้** เพื่อรักษาความสมบูรณ์ของระบบของคุณเอง นี่คือคำแนะนำบางประการในการเลือกการเชื่อมต่อที่เหมาะสม





- การกระจายความหลากหลายทางภูมิศาสตร์**: เชื่อมต่อกับโหนดในภูมิภาคต่างๆ





- ชื่อเสียง**: เลือกโหนดที่มีความพร้อมใช้งานดี





- Capacity**: เลือกปมที่มีสภาพคล่องดี





- ค่าธรรมเนียม**: ตรวจสอบค่าธรรมเนียมการส่งผ่าน.


### เปิดช่องทางการชำระเงิน


ในการเปิดช่องทางการชำระเงิน ให้แน่ใจว่าคุณได้ **เชื่อมต่อ** กับโหนดเพียร์แล้ว จากนั้นกำหนด **ความจุ** (จำนวน satoshis) ที่คุณต้องการบล็อกในช่องทางนี้



```bash
lncli openchannel --node_key=<ID_PUBKEY> --local_amt=<AMOUNT_SATOSHIS>
```


### สร้างใบแจ้งหนี้ Lightning



ใบแจ้งหนี้ Lightning แสดงถึงสตริงของอักขระที่แสดงความต้องการของคุณในการรับ satoshis ใน Lightning wallet ของคุณ


การสร้างใบแจ้งหนี้ Lightning ด้วยโหนดของคุณเองช่วยให้คุณปกป้องข้อมูลของคุณ (ทั้งทางภูมิศาสตร์และส่วนบุคคล) และให้คุณมีอิสระ 100% ในการจัดการเงินทุนของคุณ



```bash
# Générer une facture de 1000 sats

lncli addinvoice --amt=1000 --memo="Facture de 1000 sats"
```



### การชำระใบแจ้งหนี้ Lightning



```bash
lncli payinvoice <FACTURE>
```


### ปิดช่อง



มีสองวิธีในการปิดช่องทางที่ใช้งานอยู่บนโหนดปัจจุบันของคุณ





- การปิดร่วมมือ**: สิ่งนี้ส่งสัญญาณถึงความต้องการของโหนดของคุณที่จะถอนตัวจากช่องทางการชำระเงิน โดยรับรองว่างานที่กำลังดำเนินอยู่จะเสร็จสมบูรณ์และข้อมูลจะถูกสำรองเพื่อหลีกเลี่ยงการสูญเสียเงินทุน


```
lncli closechannel <ID_CANAL>
```




- การปิดแบบบังคับ**: ⚠️ ควรหลีกเลี่ยงหากเป็นไปได้ การดำเนินการนี้จะขัดจังหวะกระบวนการที่กำลังดำเนินอยู่ในช่องทางการชำระเงินของคุณและเพิ่มความเสี่ยงในการสูญเสียเงินทุน


```
lncli closechannel --force <ID_CANAL>
```


## แนวทางปฏิบัติที่ดีที่สุดและความปลอดภัยสำหรับโหนด LND ของคุณ


ความปลอดภัยเป็นสิ่งสำคัญที่สุดเมื่อใช้ Bitcoin/ Lightning node นี่คือบางจุดเพื่อเสริมสร้างความปลอดภัยในการติดตั้งของคุณ:





- เก็บ `seed phrase` ของคุณไว้ในที่ปลอดภัยและออฟไลน์





- ทำการสำรองข้อมูลไฟล์ `~/.LND/channel.backup` เป็นประจำ: ไฟล์นี้จะบันทึกสถานะของช่องทุกครั้งที่มีการเปิดช่องใหม่ (หรือปิดช่องเก่า) บนโหนดของคุณ


⚠️ ช่วยให้คุณสามารถกู้คืนช่องทางและกู้คืนเงินที่ถูกบล็อกในช่องทางการชำระเงินในกรณีที่ข้อมูลสูญหายหรือโหนดล้มเหลว



คุณสามารถกู้คืนเงินของคุณด้วยคำสั่งด้านล่างโดยระบุเส้นทางสำรองของไฟล์นี้:


```
lncli restorechanbackup <CHEMIN_DU_FICHIER>
```




- ตรวจสอบให้แน่ใจว่าคุณได้บันทึกคำกู้คืนและรหัสผ่านของ Lightning wallet ของคุณแล้ว
- อัปเดตระบบของคุณให้ทันสมัยอยู่เสมอ
