Lightning Network, Bitcoin का दूसरा Layer है, जो इसे तीव्र गति से आगे बढ़ने में सक्षम बनाता है, विशेष रूप से इसके द्वारा प्रदान की जाने वाली लेनदेन की गति और कम लागत के कारण।



इस ट्यूटोरियल में, हम अपने Linux मशीन (Ubuntu 24.04 वितरण) पर Lightning Network Daemon कार्यान्वयन स्थापित करेंगे।



## Lightning Network Daemon क्या है?



Lightning Network Daemon, Lightning Network का पूर्ण Go कार्यान्वयन है। इसे लाइटनिंग लैब्स द्वारा बनाया गया था और यह आपको अपनी मशीन पर लाइटनिंग नोड का पूर्ण इंस्टेंस चलाने की सुविधा देता है।


दूसरे शब्दों में, इस कार्यान्वयन के साथ, आप यह कर सकते हैं:





- **Lightning Network** के साथ इंटरैक्ट करें: आप सीधे अपने मशीन टर्मिनल से लाइटनिंग पोर्टफोलियो बनाने, भुगतान चैनल और रूट प्रबंधित करने और बहुत कुछ करने के लिए कमांड लाइनों का उपयोग कर सकते हैं।
- **रिमोट Bitcoin नोड या अपने खुद के Bitcoin कोर इंस्टेंस को लिंक करना**: LND आपको Bitcoin इंस्टेंस को लिंक करने और इसे अपने बैकएंड के रूप में उपयोग करने देता है। इस कार्यान्वयन का उपयोग करने के लिए, आपको अपनी मशीन पर Bitcoin कोर इंस्टेंस चलाने की आवश्यकता नहीं है।




https://planb.network/fr/tutorials/node/bitcoin/bitcoin-core-linux-568c13a6-8746-4d63-8e95-f4a61c5ae0ed

## अपना स्वयं का लाइटनिंग नोड क्यों रखें?


लाइटनिंग एक Bitcoin ओवरले है जो स्थानांतरण प्रक्रिया को अनुकूलित करता है और लेनदेन लागत को कम करता है।



अपने लाइटनिंग नोड को घुमाकर, आप संप्रभुता और स्वायत्तता प्राप्त करते हैं। आप अपने फंड पर नियंत्रण रखते हैं, इसलिए ध्यान रखें:



"न आपकी चाबियाँ, न आपके बिटकॉइन।"



इस अर्थ में, लाइटनिंग नोड चलाने से आपके डेटा की सुरक्षा और अखंडता निम्नलिखित तरीकों से बढ़ जाती है:





- **संपूर्ण नियंत्रण**: अपने भुगतान चैनलों का प्रबंधन स्वयं करें, अपना स्वयं का बैंक बनें और अपनी परिसंपत्तियों के स्वामी बनें।
- **गोपनीयता**: अपनी गोपनीयता की सुरक्षा के लिए तीसरे पक्ष पर निर्भर हुए बिना लेनदेन करें।
- **सीखना और स्वायत्तता**: `lncli` कमांड की बदौलत, आप अपने टर्मिनल से स्वयं को लागू करके लाइटनिंग की अंतर्निहित प्रक्रियाओं को बेहतर ढंग से समझ सकते हैं।
- **विकेंद्रीकरण**: Bitcoin / Lightning Network को मजबूत और विकेंद्रीकृत करने में सक्रिय भूमिका निभाएं।



https://planb.network/courses/65c138b0-4161-4958-bbe3-c12916bc959c


हमारे मशीन पर LND कार्यान्वयन का इंस्टेंस चलाने के लिए आपके पास दो विकल्प हैं। हम या तो स्थानीय रूप से चलाने के लिए अपने मशीन पर वातावरण सेट कर सकते हैं, या Docker कंटेनर से LND इंस्टॉल कर सकते हैं। यहाँ, हम पहले विकल्प पर ध्यान केंद्रित करेंगे, और बाद के ट्यूटोरियल में Docker के साथ आगे बढ़ने का तरीका देखेंगे।


## स्रोत कोड से LND स्थापित करें



### आवश्यक शर्तें


चूंकि LND गो भाषा में लिखा गया है, इसलिए आपको यह सुनिश्चित करना होगा कि आपके पास GoLang वातावरण और आपके Linux मशीन पर आवश्यक निर्भरताएं मौजूद हों।





- **हार्डवेयर आवश्यकताएँ:**


सुचारू, निर्बाध अनुभव के लिए, आपकी मशीन में LND लाइटनिंग नोड को चलाने के लिए आवश्यक क्षमता होनी चाहिए।



आपको चाहिये होगा :


1. **इष्टतम तरलता के लिए 8 जीबी रैम**,


2. **एक मल्टी-कोर प्रोसेसर (क्वाड-कोर या अधिक)** आपके नोड की क्रियाओं को कुशलतापूर्वक प्रबंधित करने के लिए,


3. **प्रून्ड नोड मोड के लिए कम से कम 5GB डिस्क स्थान** और Bitcoin कोर चलाने के लिए 1TB (रिमोट नोड का उपयोग करने पर वैकल्पिक)





- **उपयोगी निर्भरताएँ स्थापित करें:**


नीचे दिया गया कमांड आपको अपनी मशीन पर LND चलाने के लिए आवश्यक उपकरण इंस्टॉल करने की अनुमति देगा। अन्य चीजों के अलावा, आपको `Git`, एक वर्जनिंग टूल, और `make` इंस्टॉल करना होगा, जो सोर्स कोड से LND कार्यान्वयन को निष्पादित और बना सकता है।



```bash
sudo apt install -y build-essential git make
```



![installation-dependances](assets/fr/01.webp)





- **अपने Linux मशीन पर GoLang स्थापित करें**



इस ट्यूटोरियल की तिथि के अनुसार, LND को स्थापना के लिए **Go** के संस्करण 1.23.6 की आवश्यकता है।



यदि आपके पास पहले से ही कोई पुराना संस्करण स्थापित है, तो नए Go इंस्टॉलेशन के लिए उसे हटा दें।


```bash
# Suppression d'une ancienne version de Go
sudo rm -rf /usr/local/go

# Installation de la version 1.23.6 de Go
wget https://golang.org/dl/go1.23.6.linux-amd64.tar.gz

# Decompression du package

sudo tar -C /usr/local -xzf go1.23.6.linux-amd64.tar.gz

```



![go-install](assets/fr/02.webp)





- **जाओ** पर्यावरण विन्यास


अपने Linux सिस्टम में Go जोड़ने के लिए अपनी `~/.bashrc` फ़ाइल में, निम्नलिखित पर्यावरण चरों को आरंभीकृत करें।



```bash
# Configuration de l'environnement Go
export GOROOT=/usr/local/go
export GOPATH=~/gocode
export PATH=$PATH:$GOROOT/bin

# Appliquer les modifications

source ~/.bashrc
```





- **स्थापना की जाँच** (फ्रेंच में)


```bash
go version
```

### Lightning Network Daemon को कॉन्फ़िगर करना



लाइटनिंग LND नोड का कॉन्फ़िगरेशन Bitcoin के समान है, और इसे आपके नोड के सभी पैरामीटर वाली कॉन्फ़िगरेशन फ़ाइल में किया जाता है। ऐसा करने के लिए, अपनी मशीन के रूट पर आप एक छिपा हुआ फ़ोल्डर `.LND` बना सकते हैं और फिर इस फ़ोल्डर में अपनी कॉन्फ़िगरेशन फ़ाइल `LND.conf` बना सकते हैं।



```bash
# Création du ficher
mkdir -p ~/.lnd

cd ~/.lnd

# Fichier de configuration
touch lnd.conf
```





कॉन्फ़िगरेशन फ़ाइल में, आप अपना LND नोड सेट कर सकते हैं।



```
noseedbackup=0
debuglevel=debug

[Bitcoin]
bitcoin.active=1
bitcoin.mainnet=1
bitcoin.node=bitcoind

[Bitcoind]
bitcoind.rpcuser=<UTILISATEUR_BITCOIN>
bitcoind.rpcpassword=<MOT_DE_PASSE_BITCOIN>
bitcoind.zmqpubrawblock=tcp://127.0.0.1:28332
bitcoind.zmqpubrawtx=tcp://127.0.0.1:28333

```



## अपने कॉन्फ़िगरेशन को समझना



आपके लिए यह समझना महत्वपूर्ण है कि आपके LND नोड की सही और पूर्ण स्थापना के लिए आपको किस न्यूनतम कॉन्फ़िगरेशन की आवश्यकता है।



`~/.LND/LND.conf` फ़ाइल की सामग्री के आधार पर, फ़ील्ड का विवरण यहां दिया गया है:





- **नोजेडबैकअप**: आपको यह चुनने की अनुमति देता है कि क्या आप चाहते हैं कि LND आपके पोर्टफ़ोलियो का स्वचालित बैकअप करे। इस प्रॉपर्टी को `0` पर सेट करने से आप व्यक्तिगत रूप से चुने गए सुरक्षित स्थान पर मैन्युअल रूप से पुनर्स्थापना जानकारी सहेज सकते हैं।





- **debuglevel**: आपको किसी कार्रवाई के दौरान होने वाली त्रुटियों की स्थिति में त्रुटियों और लॉग के विवरण के स्तर को परिभाषित करने की अनुमति देता है।





- **Bitcoin.active**: LND को Bitcoin नोड के रूप में संचालित करने और Bitcoin नेटवर्क के साथ अंतःक्रिया करने का निर्देश देता है।





- **Bitcoin.Mainnet**: LND को Bitcoin के मुख्य नेटवर्क (Mainnet) से कनेक्ट करने के लिए निर्दिष्ट करता है, आप Bitcoin Signet और Bitcoin Regtest नेटवर्क के लिए क्रमशः `bitcoind.signet` और `bitcoind.regtest` मान सेट कर सकते हैं





- **Bitcoin.node**: Bitcoin नोड का प्रकार निर्दिष्ट करता है जिससे LND को कनेक्ट होना चाहिए।





- **Bitcoin.rpcuser** और **Bitcoin.rpcpassword** : प्रतिनिधित्व करते हैं।


आपके Bitcoin नोड से कनेक्ट करने के लिए क्रमशः लॉगिन (उपयोगकर्ता, पासवर्ड)





- **bitcoind.zmqpubrawblock** और **bitcoind.zmqpubrawtx**: क्रमशः Bitcoin नेटवर्क पर नए ब्लॉक और लेनदेन के बारे में सूचनाएं प्राप्त करने के लिए ZeroMQ एंडपॉइंट को परिभाषित करते हैं।




## LND के साथ अपनी स्थापना की जाँच करना



आप शायद यह सुनिश्चित करना चाहेंगे कि प्रक्रिया सफल रही है, और आप अपनी नोड जानकारी को अद्यतन रखने के लिए Lightning Network के साथ समन्वय कर रहे हैं।



LND कार्यान्वयन शुरू करने और अपने नोड के बारे में जानकारी प्राप्त करने के लिए, बस कमांड टाइप करें:


```bash
lnd getinfo
```


![lnd-getinfo](assets/fr/07.webp)


## LND पर कार्रवाई करना



एक बार आपकी स्थापना पूरी हो जाने और जांच हो जाने के बाद, आप इसका उपयोग शुरू कर सकते हैं।


यहां आपको आरंभ करने के लिए आवश्यक आदेश दिए गए हैं।



### पोर्टफोलियो बनाएं


आपका लाइटनिंग पोर्टफोलियो आपके फंड को प्रबंधित करने की किसी भी कार्रवाई में पहला कदम है।



⚠️ **महत्वपूर्ण**: अपने 24-शब्द **seed वाक्यांश** को ध्यान से नोट करें। समस्याओं की स्थिति में आपको अपने धन की वसूली के लिए इसकी आवश्यकता होगी।



इसके अलावा अपना Wallet पासवर्ड भी सुरक्षित रखें ताकि जब आप अपना LND नोड पुनः आरंभ करें तो आप `lncli unlock` कमांड से उसे अनलॉक कर सकें।



```bash
lncli create
```


![créer-portefeuille](assets/fr/08.webp)

अपने टर्मिनल से सीधे अपने खातों की जानकारी प्राप्त करें:



```bash

```

### आपके नोड के बारे में जानकारी



आपके नोड पर कौन से चैनल सक्रिय हैं, यह जानने के लिए नीचे दिए गए कमांड का उपयोग करें।



```bash
lncli listchannels
```



आप उन नोड्स की सूची भी प्राप्त कर सकते हैं जिनसे आप जुड़े हुए हैं।



```bash
lncli listpeers
```



### चैनल प्रबंधन



लाइटनिंग चैनल आपको Lightning Network पर किसी अन्य नोड के साथ **सीधा, जोड़ी-दर-जोड़ी कनेक्शन** बनाने की अनुमति देता है। इस चैनल में, आप चैनल की क्षमता तक स्वतंत्र रूप से Exchange सतोशी कर सकते हैं।



### नोड से कनेक्ट करें



यदि आप सक्रिय रूप से भाग लेना चाहते हैं और लाइटनिंग की शक्ति से लाभ उठाना चाहते हैं तो अन्य लाइटनिंग नोड्स से जुड़ना एक मौलिक कार्य है।



किसी सहकर्मी (लाइटनिंग नोड) से जुड़ने के लिए, आपको तीन प्रकार की जानकारी की आवश्यकता होगी:




- **नोड की सार्वजनिक कुंजी**: यह Bitcoin नेटवर्क में नोड का विशिष्ट पहचानकर्ता है;
- **आईपी** : उस मशीन का आईपी जिस पर नोड स्थापित है;
- **पोर्ट** : मशीन पर खुला पोर्ट जो इस नोड के साथ संचार की अनुमति देता है।



आप [amboss](https://amboss.space/) पर कनेक्ट करने के लिए नोड्स पा सकते हैं, यह एक ऐसा प्लेटफ़ॉर्म है जो लाइटनिंग नोड्स पर जानकारी सूचीबद्ध करता है।



```bash
# Se connecter à un noeud
lncli connect <ID_PUBKEY>@<IP>:<PORT>

# Un exemple  : Connexion au noeud de Wallet of Satoshi
lncli connect 035e4ff418fc8b5554c5d9eea66396c227bd429a3251c8cbc711002ba215bfc226@170.75.163.209:9735
```




सुनिश्चित करें कि आप अपने सिस्टम की अखंडता को बनाए रखने के लिए **विश्वसनीय नोड्स** से कनेक्ट हों। सही कनेक्शन चुनने के लिए यहाँ कुछ सुझाव दिए गए हैं।





- **भौगोलिक विविधीकरण**: विभिन्न क्षेत्रों में नोड्स से जुड़ें।





- **प्रतिष्ठा**: अच्छी उपलब्धता वाले नोड्स चुनें।





- **क्षमता**: अच्छी तरलता वाली गांठें चुनें।





- **शुल्क**: रूटिंग शुल्क की जांच करें।


### भुगतान चैनल खोलें


भुगतान चैनल खोलने के लिए, सुनिश्चित करें कि आप पीयर नोड से जुड़े हुए हैं, फिर उस क्षमता (सातोशी की मात्रा) को परिभाषित करें जिसे आप इस चैनल में ब्लॉक करना चाहते हैं।



```bash
lncli openchannel --node_key=<ID_PUBKEY> --local_amt=<AMOUNT_SATOSHIS>
```


### लाइटनिंग Invoice बनाएं



लाइटनिंग Invoice, आपके लाइटनिंग Wallet में सातोशी प्राप्त करने की आपकी इच्छा को व्यक्त करने वाले वर्णों की एक स्ट्रिंग का प्रतिनिधित्व करता है।


अपने स्वयं के नोड के साथ लाइटनिंग इनवॉइस बनाने से आप अपने डेटा (भौगोलिक और व्यक्तिगत) की सुरक्षा कर सकते हैं और आपको अपने फंड के प्रबंधन पर 100% स्वायत्तता मिलती है।



```bash
# Générer une facture de 1000 sats

lncli addinvoice --amt=1000 --memo="Facture de 1000 sats"
```



### लाइटनिंग Invoice का भुगतान करना



```bash
lncli payinvoice <FACTURE>
```


### चैनल बंद करें



आपके वर्तमान नोड पर सक्रिय चैनल को बंद करने के दो तरीके हैं।





- **सहकारी समापन**: यह आपके नोड की भुगतान चैनल से हटने की इच्छा को दर्शाता है, जिससे यह सुनिश्चित होता है कि चल रहे कार्य पूरे हो जाएं और धन की हानि से बचने के लिए डेटा का बैकअप ले लिया जाए।


```
lncli closechannel <ID_CANAL>
```




- **जबरन बंद करना**: ⚠️ यदि संभव हो तो इससे बचें, क्योंकि यह कार्रवाई आपके भुगतान चैनल में चल रही प्रक्रियाओं को बाधित करती है और धन खोने का जोखिम बढ़ाती है।


```
lncli closechannel --force <ID_CANAL>
```


## आपके LND नोड के लिए सर्वोत्तम अभ्यास और सुरक्षा।


Bitcoin/ लाइटनिंग नोड का उपयोग करते समय सुरक्षा सर्वोपरि है। आपकी स्थापना की सुरक्षा को सुदृढ़ करने के लिए यहां कुछ बिंदु दिए गए हैं:





- अपने `seed वाक्यांश` को सुरक्षित, ऑफ-लाइन स्थान पर रखें।





- `~/.LND/channel.backup` फ़ाइल का नियमित बैकअप बनाएं: यह फ़ाइल आपके चैनल की स्थिति को हर बार सहेजती है जब आपके नोड पर कोई नया चैनल खोला जाता है (या कोई पुराना चैनल बंद किया जाता है)।


⚠️ डेटा हानि या नोड विफलता की स्थिति में आपको चैनलों को पुनर्स्थापित करने और भुगतान चैनलों में अवरुद्ध धन को पुनर्प्राप्त करने की अनुमति देता है



आप इस फ़ाइल का बैकअप पथ निर्दिष्ट करके नीचे दिए गए आदेश से अपने फंड को पुनर्स्थापित कर सकते हैं:


```
lncli restorechanbackup <CHEMIN_DU_FICHIER>
```




- सुनिश्चित करें कि आपने अपने लाइटनिंग Wallet के पुनर्स्थापना शब्द और पासवर्ड को सहेज लिया है।
- अपने सिस्टम को अद्यतन रखें.
