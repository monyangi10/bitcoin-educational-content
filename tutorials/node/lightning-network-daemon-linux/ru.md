Lightning Network - это второй Layer от Bitcoin, что позволяет ему приобретать молниеносные размеры, в частности, благодаря скорости и низкой стоимости операций, которые он предлагает.



В этом руководстве мы установим реализацию Lightning Network Daemon на нашу Linux-машину (дистрибутив Ubuntu 24.04).



## Что такое Lightning Network Daemon?



Lightning Network Daemon - это полная реализация Lightning Network на языке Go. Она была создана Lightning Labs и позволяет запускать полный экземпляр узла Lightning на вашей машине.


Другими словами, с помощью этой реализации вы можете :





- **Взаимодействуйте с Lightning Network**: Вы можете использовать командные строки для создания портфелей Lightning, управления каналами и маршрутами платежей и многого другого прямо с терминала вашего компьютера.
- **Подключение удаленного узла Bitcoin или собственного экземпляра Bitcoin Core**: LND позволяет связать экземпляр Bitcoin и использовать его в качестве бэкенда. Чтобы использовать эту реализацию, вам не нужно запускать экземпляр Bitcoin Core на своей машине.




https://planb.network/fr/tutorials/node/bitcoin/bitcoin-core-linux-568c13a6-8746-4d63-8e95-f4a61c5ae0ed

## Зачем нужен собственный узел Lightning?


Lightning - это оверлей Bitcoin, который оптимизирует процесс перевода средств и снижает транзакционные издержки.



Вращая свой узел Lightning, вы обретаете суверенитет и автономию. Вы контролируете свои средства, поэтому имейте в виду:



"Не ваши ключи, не ваши биткоины"



В этом смысле работа узла Lightning повышает безопасность и целостность ваших данных следующим образом:





- **Полный контроль**: Управляйте своими платежными каналами, станьте собственным банком и будьте хозяином своих активов.
- **Конфиденциальность**: Совершайте сделки, не полагаясь на третьих лиц, которые защитят вашу конфиденциальность.
- **Обучение и автономность**: Благодаря командам `lncli` вы можете лучше понять процессы, лежащие в основе Lightning, применяя их самостоятельно из своего терминала.
- **Децентрализация**: Примите активное участие в укреплении и децентрализации Bitcoin / Lightning Network.



https://planb.network/courses/65c138b0-4161-4958-bbe3-c12916bc959c

## Установите LND из исходного кода



### Пререквизиты


Поскольку LND написан на языке Go, вам необходимо убедиться, что на вашей машине Linux установлено окружение GoLang и необходимые зависимости.





- **Требования к оборудованию:**


Для плавной и бесперебойной работы ваша машина должна обладать необходимой мощностью для работы узла LND Lightning.



Вам понадобится :


1. **8 ГБ оперативной памяти** для оптимальной скорости работы,


2. **Многоядерный процессор (четырехъядерный или более)** для эффективного управления действиями вашего узла,


3. **Не менее 5 ГБ дискового пространства** для режима обрезанного узла и 1 ТБ для запуска Bitcoin Core (необязательно при использовании удаленного узла)





- Установите полезные зависимости:


Приведенная ниже команда позволит вам установить на машину инструменты, необходимые для работы LND. Среди прочего, вам нужно будет установить `Git`, инструмент версионирования, и `make`, который может выполнять и собирать реализацию LND из исходного кода.



```bash
sudo apt install -y build-essential git make
```

- Установите GoLang на компьютер с Linux



На момент публикации этого руководства для установки LND требуется версия 1.23.6 **Go**.



Если у вас уже была установлена предыдущая версия, удалите ее для новой установки Go.


```bash

sudo rm -rf /usr/local/go

# Installation de la version 1.23.6 de Go
wget https://golang.org/dl/go1.23.6.linux-amd64.tar.gz

# Decompression du package

sudo tar -C /usr/local -xzf go1.23.6.linux-amd64.tar.gz

```



![go-install](assets/fr/02.webp)





- **Конфигурация среды Go**


В файле `~/.bashrc` инициализируйте следующие переменные окружения, чтобы добавить Go в вашу систему Linux.



```bash
# Configuration de l'environnement Go
export GOROOT=/usr/local/go
export GOPATH=~/gocode
export PATH=$PATH:$GOROOT/bin

# Appliquer les modifications

source ~/.bashrc
```

### Настройка Lightning Network Daemon



Конфигурация узла Lightning LND аналогична конфигурации узла Bitcoin и выполняется в конфигурационном файле, содержащем все параметры вашего узла. Для этого в корне машины можно создать скрытую папку `.LND`, а затем создать в ней конфигурационный файл `LND.conf`.



```bash
# Création du ficher
mkdir -p ~/.lnd

cd ~/.lnd

# Fichier de configuration
touch lnd.conf
```





В файле конфигурации вы можете настроить узел LND.



```
noseedbackup=0
debuglevel=debug

[Bitcoin]
bitcoin.active=1
bitcoin.mainnet=1
bitcoin.node=bitcoind

[Bitcoind]
bitcoind.rpcuser=<UTILISATEUR_BITCOIN>
bitcoind.rpcpassword=<MOT_DE_PASSE_BITCOIN>
bitcoind.zmqpubrawblock=tcp://127.0.0.1:28332
bitcoind.zmqpubrawtx=tcp://127.0.0.1:28333

```



## Понимание конфигурации



Важно, чтобы вы понимали, какая минимальная конфигурация необходима для правильной и полной установки узла LND.



Исходя из содержимого файла `~/.LND/LND.conf`, вот подробная информация о полях:





- **noseedbackup**: Позволяет выбрать, хотите ли вы, чтобы LND выполнял автоматическое резервное копирование ваших портфелей.  Установив это свойство в `0`, вы можете вручную сохранять информацию о восстановлении в выбранном лично вами безопасном месте.





- **debuglevel**: Позволяет определить уровень детализации ошибок и журналов в случае возникновения ошибок во время выполнения действия.





- **Bitcoin.active**: Указывает LND работать в качестве узла Bitcoin и взаимодействовать с сетью Bitcoin.





- **Bitcoin.Mainnet**: Указывает LND для подключения к основной сети Bitcoin (Mainnet), вы можете установить значения `bitcoind.signet` и `bitcoind.regtest` соответственно для сетей Bitcoin Signet и Bitcoin Regtest





- **Bitcoin.node**: Указывает тип узла Bitcoin, к которому должен подключиться LND.





- **Bitcoin.rpcuser** и **Bitcoin.rpcpassword** : Представляют.


соответственно, логины (пользователь, пароль) для подключения к узлу Bitcoin





- **bitcoind.zmqpubrawblock** и **bitcoind.zmqpubrawtx**: соответственно определяют конечные точки ZeroMQ для получения уведомлений о новых блоках и транзакциях в сети Bitcoin.

## Выполнение действий на LND



После завершения установки и проверки можно приступать к работе.


Вот основные команды, которые помогут вам начать работу.



### Создайте портфолио


Портфель Lightning - это первый шаг в управлении вашими средствами.



⚠️ **ВАЖНО**: Внимательно запишите свою фразу из 24 слов **seed**. Она понадобится вам для восстановления средств в случае возникновения проблем.



Также сохраните пароль Wallet, чтобы разблокировать его командой `lncli unlock` при перезапуске узла LND.



```bash
lncli create
```


![créer-portefeuille](assets/fr/08.webp)


### Проверьте свой баланс



Обращайтесь к своим счетам прямо с терминала:



```bash
lncli walletbalance
```


![solde](assets/fr/09.webp)


### Информация о вашем узле



Используйте приведенную ниже команду, чтобы узнать, какие каналы активны на вашем узле.



```bash
lncli listchannels
```



Вы также можете получить список узлов, к которым вы подключены.



```bash
lncli listpeers
```



### Управление каналами



Канал Lightning позволяет вам иметь **прямое, парное соединение с другим узлом на Lightning Network**. В этом канале вы можете свободно использовать сатоши Exchange в пределах емкости канала.



### Подключение к узлу



Подключение к другим узлам Lightning является основополагающим действием, если вы хотите активно участвовать и пользоваться преимуществами Lightning.



Чтобы подключиться к аналогу (узлу Lightning), вам понадобятся три части информации:




- **Открытый ключ узла**: Это уникальный идентификатор узла в сети Bitcoin;
- **IP** : IP-адрес машины, на которой установлен узел;
- **ПОРТ** :  Порт, открытый на машине, который позволяет взаимодействовать с этим узлом.



Вы можете найти узлы для подключения на [amboss](https://amboss.space/), платформе, содержащей информацию об узлах Lightning.



```bash
# Se connecter à un noeud
lncli connect <ID_PUBKEY>@<IP>:<PORT>

# Un exemple  : Connexion au noeud de Wallet of Satoshi
lncli connect 035e4ff418fc8b5554c5d9eea66396c227bd429a3251c8cbc711002ba215bfc226@170.75.163.209:9735
```




Убедитесь, что вы подключаетесь к **надежным узлам**, чтобы сохранить целостность вашей собственной системы. Вот несколько рекомендаций по выбору правильных соединений.





- **Географическая диверсификация**: Подключайтесь к узлам в разных регионах.





- **Репутация**: Выбирайте узлы с хорошей доступностью.





- **Вместимость**: Выбирайте узлы с хорошей ликвидностью.





- **Сборы**: Плата за маршрутизацию чека.


### Откройте канал оплаты


Чтобы открыть платежный канал, убедитесь, что вы **подключены** к узлу-аналогу, затем определите **емкость** (количество сатоши), которую вы хотите заблокировать в этом канале.



```bash
lncli openchannel --node_key=<ID_PUBKEY> --local_amt=<AMOUNT_SATOSHIS>
```


### Создайте молнию Invoice



Lightning Invoice представляет собой строку символов, выражающих ваше желание получить сатоши в Lightning Wallet.


Создание счетов-фактур Lightning с собственным узлом позволяет защитить ваши данные (географические и личные) и дает вам 100%-ную автономию в управлении вашими средствами.



```bash
# Générer une facture de 1000 sats

lncli addinvoice --amt=1000 --memo="Facture de 1000 sats"
```



### Оплата молнии Invoice



```bash
lncli payinvoice <FACTURE>
```


### Закрыть канал



Есть два способа закрыть активный канал на текущем узле.





- **Закрытие кооператива**: Это сигнал о том, что ваша нода хочет выйти из канала платежей, обеспечивая выполнение текущих задач и резервное копирование данных во избежание потери средств.


```
lncli closechannel <ID_CANAL>
```




- **Принудительное закрытие**: ⚠️ По возможности избегайте этого действия: оно прерывает текущие процессы в вашем платежном канале и повышает риск потери средств.


```
lncli closechannel --force <ID_CANAL>
```


## Лучшие практики и техника безопасности для узла LND.


Безопасность имеет первостепенное значение при использовании узла Bitcoin/ Lightning. Вот несколько пунктов, которые помогут повысить безопасность вашей установки:





- Храните свою фразу `seed` в надежном месте, вне сети.





- Регулярно создавайте резервные копии файла `~/.LND/channel.backup`: Этот файл сохраняет состояние ваших каналов каждый раз, когда на вашем узле открывается новый канал (или закрывается старый).


⚠️ Позволяет восстанавливать каналы и возвращать средства, заблокированные в платежных каналах, в случае потери данных или отказа узла



Вы можете восстановить свои средства с помощью приведенной ниже команды, указав путь к резервной копии этого файла:


```
lncli restorechanbackup <CHEMIN_DU_FICHIER>
```




- Убедитесь, что вы сохранили слова восстановления и пароль для Lightning Wallet.
- Поддерживайте систему в актуальном состоянии.



## Текущее устранение неполадок


### Частые проблемы




- **Ошибка подключения bitcoind**: Проверьте данные для входа в систему RPC
- **Синхронизация заблокирована**: Проверьте подключение к Интернету
- **Ошибка разрешения**: Проверьте права на папку `~/.LND`
