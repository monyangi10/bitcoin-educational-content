การโอนเงินระหว่าง Bitcoin on-chain และ Lightning Network โดยทั่วไปต้องการการเปิดช่องทาง Lightning ด้วยตนเอง (ซึ่งมีความซับซ้อนและค่าใช้จ่ายสูง) หรือการใช้แพลตฟอร์มแลกเปลี่ยนแบบรวมศูนย์ที่ต้องมี KYC SwapMarket เสนอทางเลือก: การแลกเปลี่ยนอะตอมแบบไม่ต้องเชื่อถือผ่านผู้ให้บริการที่มีการแข่งขัน โดยไม่ต้องมี KYC



นวัตกรรม: แม้ว่าผู้ให้บริการจะเป็นตัวกลาง แต่ HTLC (*Hash Time Locked Contracts*) รับประกันทางคณิตศาสตร์ว่าเงินทุนของคุณยังคงอยู่ภายใต้การควบคุมของคุณ การรวมตัวของผู้ให้บริการหลายราย (Boltz, ZEUS Swaps, Eldamar, Middle Way) สร้างการแข่งขันด้านราคา Interface เว็บโอเพ่นซอร์สที่สามารถโฮสต์เองได้



## SwapMarket คืออะไร?



ตัวรวบรวมโอเพ่นซอร์สที่เปิดตัวในปี 2024, SwapMarket ทำหน้าที่เป็นตัวเปรียบเทียบของผู้ให้บริการสวอป Bitcoin/Lightning ผู้ใช้สามารถเปรียบเทียบเงื่อนไข (ค่าธรรมเนียม, สภาพคล่อง, ขีดจำกัด) ได้ทันทีและเลือกผู้ให้บริการที่เหมาะสมที่สุด



### สถาปัตยกรรมทางเทคนิค



**Frontend client-side**: แอปพลิเคชันฝั่งไคลเอนต์ 100% (fork Boltz Web App) โฮสต์บน GitHub Pages โค้ดทำงานในเบราว์เซอร์โดยไม่มีเซิร์ฟเวอร์แบ็กเอนด์ ประวัติถูกเก็บไว้ในเครื่อง (คุกกี้/แคช) ซอร์สโค้ดเป็นสาธารณะและสามารถตรวจสอบได้



**Provider discovery** : รายการที่ถูกฮาร์ดโค้ดใน `src/configs/mainnet.ts` ผู้ให้บริการใหม่เพิ่มผ่าน Pull Request หรืออีเมล



**Independent backends**: ผู้ให้บริการแต่ละรายดำเนินการแบ็กเอนด์ Boltz ของตนเอง อินเทอร์เฟซจะเรียกใช้ API แบบเรียลไทม์เพื่อเปรียบเทียบใบเสนอราคาในทันที



**HTLC Atomic Swaps**: Hash Time Locked Contracts รับประกันความเป็นอะตอม: การแลกเปลี่ยนจะดำเนินการหรือแต่ละฝ่ายจะได้รับเงินคืน ความเสี่ยงของคู่สัญญาถูกกำจัดด้วยคณิตศาสตร์



### ปรัชญา



SwapMarket ลดการรวมศูนย์โดยการสร้างการแข่งขันระหว่างผู้ให้บริการสำหรับค่าธรรมเนียมและสภาพคล่อง ไม่มี KYC, โค้ดโอเพ่นซอร์สที่สามารถโฮสต์เองได้, การเพิ่มจำนวนผู้ดำเนินการอิสระเพื่อหลีกเลี่ยงจุดล้มเหลวเดียว



## คุณสมบัติหลัก



### ตลาดผู้ให้บริการ



อินเทอร์เฟซแสดงผู้ให้บริการที่ใช้งานอยู่ทั้งหมด: ชื่อของผู้ให้บริการ, ค่าธรรมเนียมที่ใช้ (เปอร์เซ็นต์และ/หรือคงที่), จำนวนเงินขั้นต่ำ/สูงสุดที่มีอยู่, และประเภทการแลกเปลี่ยนที่รองรับ แอปพลิเคชันจะสอบถาม API ของผู้ให้บริการแต่ละรายที่อ้างอิงในไฟล์การกำหนดค่าโดยตรงเพื่อดึงข้อมูลอัตราแลกเปลี่ยนแบบเรียลไทม์ การแข่งขันระหว่างผู้ให้บริการรับประกันอัตราที่เหมาะสมที่สุด โดยทั่วไปประมาณ 0.5% สำหรับการแลกเปลี่ยนมาตรฐาน



### การแลกเปลี่ยนแบบสองทิศทาง



**Swap-in (on-chain → Lightning)**: แปลง on-chain BTCs เป็น Lightning satoshis กรณีการใช้งาน: ใช้งาน wallet Lightning บนมือถือ, ได้รับความจุขาเข้าบนโหนด, หรือมีสภาพคล่องทันที



**Swap-out (Lightning → on-chain)**: แปลง Lightning satoshis เป็น on-chain BTC กรณีการใช้งาน: ทำให้ wallet Lightning ว่างเปล่าเพื่อเก็บใน cold storage หรือปรับสมดุลสภาพคล่องระหว่างเลเยอร์



### ความปลอดภัยและการฟื้นตัว



**Trustless atomic swaps**: HTLCs รับประกันว่าการแลกเปลี่ยนจะเสร็จสมบูรณ์ทั้งหมด หรือแต่ละฝ่ายจะได้รับเงินคืน ความเสี่ยงของคู่สัญญาถูกกำจัดด้วยคณิตศาสตร์



**กลไกการชำระคืน**: การแลกเปลี่ยนแต่ละครั้งมีการล็อกเวลา หากการแลกเปลี่ยนล้มเหลว เงินทุนจะสามารถขอคืนได้โดยอัตโนมัติหลังจากหมดอายุ ผู้ใช้ยังคงมีตัวเลือกในการเรียกคืนบิตคอยน์ของตนเสมอ



**Recovery keys**: SwapMarket ช่วยให้คุณสามารถส่งออก recovery keys สำหรับการแลกเปลี่ยนที่กำลังดำเนินการอยู่ ในกรณีที่เกิดปัญหา คีย์เหล่านี้สามารถใช้เพื่อสรุปหรือยกเลิกการแลกเปลี่ยนจากอุปกรณ์ใดก็ได้

### การกำหนดค่าเริ่มต้น



**Wallet Lightning**: ตรวจสอบให้แน่ใจว่าคุณมี wallet Lightning ที่ใช้งานได้ (Phoenix, Zeus, BlueWallet, ฯลฯ) สำหรับการสลับเข้า คุณจะ generate ใบแจ้งหนี้ Lightning สำหรับการสลับออก คุณจะชำระใบแจ้งหนี้ Lightning



**Wallet on-chain**: สำหรับการสลับเข้า คุณจะต้องมี wallet Bitcoin on-chain เพื่อส่งเงิน สำหรับการสลับออก เตรียมที่อยู่รับ Bitcoin



**การกำหนดค่าทางเลือก**: SwapMarket เก็บประวัติการแลกเปลี่ยนและการตั้งค่าที่ชื่นชอบในคุกกี้ของเบราว์เซอร์ ไม่จำเป็นต้องสร้างบัญชี

## สร้างการแลกเปลี่ยนทีละขั้นตอน



### สลับออก: Lightning → Bitcoin



ตัวอย่างแรกนี้แสดงวิธีการแปลง Lightning satoshis เป็น on-chain bitcoins.



**ขั้นตอนที่ 1: สลับการกำหนดค่า



จากหน้าหลัก เลือกแบบฟอร์มการแลกเปลี่ยน :




- LIGHTNING** (upper field): ป้อนจำนวนเงินที่คุณต้องการส่งใน sats Lightning (ตัวอย่าง: 30,000 sats)
- BITCOIN** (lower field): จำนวนเงินที่คุณจะได้รับจะแสดงโดยอัตโนมัติหลังจากหักค่าธรรมเนียมแล้ว (ตัวอย่าง: 29,320 sats)



ในช่องด้านล่าง วางที่อยู่ **Bitcoin ที่จะรับ** ของคุณที่คุณต้องการรับเงิน ตรวจสอบที่อยู่นี้อย่างรอบคอบ



ผู้ให้บริการเริ่มต้นมักจะเป็น Boltz Exchange ค่าธรรมเนียมเครือข่ายและค่าธรรมเนียมผู้ให้บริการจะแสดงอย่างชัดเจน



![Configuration swap-out](assets/fr/03.webp)



**ขั้นตอนที่ 2: การเลือกผู้ให้บริการ**



คลิกที่เมนูแบบเลื่อนลงของผู้ให้บริการ (ค่าเริ่มต้น: "Boltz Exchange") เพื่อแสดงผู้ให้บริการสภาพคล่องทั้งหมดที่มีอยู่



หน้าต่างโมดอลเปิดขึ้น แสดงตารางเปรียบเทียบ:




- สถานะ**: ตัวบ่งชี้ Green หากผู้ให้บริการใช้งานอยู่
- นามแฝง**: ชื่อผู้ให้บริการ (Boltz Exchange, Middle Way, Eldamar, ZEUS Swaps)
- ค่าธรรมเนียม**: ค่าบริการที่เรียกเก็บโดยผู้ให้บริการ (โดยทั่วไปอยู่ระหว่าง 0.49% ถึง 0.5%)
- Max Swap**: จำนวนสูงสุดที่ยอมรับสำหรับการแลกเปลี่ยน



เปรียบเทียบค่าธรรมเนียมและจำนวนเงินสูงสุด จากนั้นเลือกผู้ให้บริการที่คุณต้องการ



**โปรดทราบ**: อินเทอร์เฟซการเลือกผู้ให้บริการจะไม่แสดง **จำนวนขั้นต่ำ** สำหรับแต่ละผู้ให้บริการ ข้อมูลนี้จะแสดงเฉพาะในอินเทอร์เฟซการสร้างการแลกเปลี่ยน หลังจากที่ได้เลือกผู้ให้บริการแล้ว จำนวนขั้นต่ำและสูงสุดอาจแตกต่างกันไปตามผู้ให้บริการแต่ละราย และอาจเปลี่ยนแปลงได้ตามเวลา **ตรวจสอบขีดจำกัดเหล่านี้เสมอในขณะที่คุณทำการแลกเปลี่ยน**: หากจำนวนที่คุณต้องการแลกเปลี่ยนอยู่นอกขีดจำกัดของผู้ให้บริการ คุณสามารถเลือกผู้ให้บริการอื่นที่เหมาะสมกว่าสำหรับการทำธุรกรรมของคุณได้



![Sélection du provider](assets/fr/04.webp)



**ขั้นตอนที่ 3: การสร้างการแลกเปลี่ยนและ Lightning** การชำระเงิน



คลิกที่ปุ่มสีเหลือง **"CREATE ATOMIC SWAP "** SwapMarket จะ generate **Lightning invoice** (BOLT11) สำหรับคุณเพื่อชำระจาก wallet Lightning ของคุณ



หน้าจอแสดง:




- รหัสการแลกเปลี่ยน**: ตัวระบุการแลกเปลี่ยนที่ไม่ซ้ำกัน (ตัวอย่าง: J4ymFIMVR6Hm)
- สถานะ**: "swap.created" (สร้างการแลกเปลี่ยนแล้ว, รอการชำระเงิน)
- QR code**: สแกนด้วย wallet Lightning ของคุณ
- Invoice Lightning**: Character string starting with "lnbc" (example: lnbc300u1p50whiv...gn5dk2szgqkvfkzc)



ชำระใบแจ้งหนี้นี้จาก wallet Lightning ของคุณ (Phoenix, Zeus, BlueWallet, ฯลฯ) จำนวนเงินที่ต้องชำระจะแสดงไว้อย่างชัดเจน (ตัวอย่าง: 30,000 sats)



![Paiement Lightning](assets/fr/05.webp)



**ขั้นตอนที่ 4: การยืนยันและการยอมรับ**



เมื่อการชำระเงินผ่าน Lightning ได้รับการยืนยันแล้ว SwapMarket จะได้รับการชำระเงินของคุณทันทีและผู้ให้บริการจะกระจายธุรกรรม Bitcoin ไปยังที่อยู่ของคุณ



สถานะเปลี่ยนเป็น **"invoice.settled "** (ใบแจ้งหนี้ชำระแล้ว) และมีการแสดงข้อความยืนยัน



on-chain bitcoins ของคุณจะพร้อมใช้งานทันทีที่การทำธุรกรรมได้รับการยืนยัน (โดยปกติจะใช้เวลาตั้งแต่ไม่กี่นาทีถึงไม่กี่ชั่วโมง ขึ้นอยู่กับค่าธรรมเนียม mining ที่ผู้ให้บริการเลือก)



![Confirmation swap-out](assets/fr/06.webp)



คุณสามารถคลิกที่ **"OPEN CLAIM TRANSACTION "** เพื่อดูธุรกรรม Bitcoin บนเบราว์เซอร์บล็อกเชน

เลือกไฟล์กุญแจช่วยเหลือที่ดาวน์โหลดมาก่อนหน้านี้ หลังจากการตรวจสอบสำเร็จ อินเทอร์เฟซจะเปลี่ยนไปยังขั้นตอนถัดไปโดยอัตโนมัติ



**ขั้นตอนที่ 3: Bitcoin** ที่อยู่สำหรับฝากเงิน



SwapMarket ตอนนี้สร้าง **ที่อยู่ Bitcoin ที่ไม่ซ้ำกัน** ซึ่งมีสัญญา HTLC ที่เชื่อมโยงกับใบแจ้งหนี้ Lightning ของคุณ



หน้าจอแสดง:




- รหัสการแลกเปลี่ยน**: ตัวระบุเฉพาะ (ตัวอย่าง: 1kGmB6JyGqU4)
- สถานะ**: "invoice.set" (ตั้งค่าใบแจ้งหนี้, รอการชำระเงิน Bitcoin)
- รหัส QR**: ที่อยู่คลังสินค้า Bitcoin
- Bitcoin** ที่อยู่: มักจะเริ่มต้นด้วย "bc1p..." (ตัวอย่าง: bc1p5mvtwxapjkds...9d4n9f)
- คำเตือนในสีเหลือง** : "ตรวจสอบให้แน่ใจว่าธุรกรรมของคุณได้รับการยืนยันภายใน ~24 ชั่วโมงหลังจากสร้างการแลกเปลี่ยนนี้!"



ช่วงเวลาประมาณ ~24 ชั่วโมงนี้คือ **timeout** ของสัญญา HTLC หากการทำธุรกรรม Bitcoin ของคุณไม่ได้รับการยืนยันภายในกรอบเวลานี้ การแลกเปลี่ยนจะล้มเหลวและคุณจะต้องใช้กุญแจช่วยเหลือเพื่อกู้คืนเงินของคุณ



![Adresse de dépôt Bitcoin](assets/fr/09.webp)



คุณสามารถคัดลอกที่อยู่ได้โดยคลิกที่ปุ่ม **"ADDRESS "** หรือสแกน QR โค้ดโดยตรงจาก wallet on-chain ของคุณ



**ขั้นตอนที่ 4: การส่งบิตคอยน์**



จาก wallet Bitcoin on-chain ของคุณ ส่ง **จำนวนที่แน่นอน** ที่ระบุไว้ (เช่น 63,400 sats) ไปยังที่อยู่ที่สร้างขึ้น



**สำคัญ**: ใช้ค่าธรรมเนียม mining ที่เหมาะสมเพื่อรับประกันการยืนยันที่รวดเร็ว หากค่าธรรมเนียมต่ำเกินไปและธุรกรรมยังคงอยู่ใน mempool เกินเวลาที่กำหนด (~24 ชั่วโมง) การแลกเปลี่ยนจะล้มเหลว



เมื่อทำการส่งธุรกรรมแล้ว SwapMarket ตรวจพบว่ามันอยู่ใน mempool และแสดง:




- สถานะ** : "transaction.mempool
- ข้อความ**: "ธุรกรรมอยู่ใน mempool - รอการยืนยันเพื่อทำการสวอปให้เสร็จสิ้น"

## ข้อดีและข้อจำกัด



### ประโยชน์



**การแข่งขันด้านอัตรา**: การรวมตัวของผู้ให้บริการสร้างการแข่งขันตามธรรมชาติที่ดึงค่าธรรมเนียมลง (0.49% ถึง 0.5%)



**ความลับ**: ไม่มี KYC, อินเทอร์เฟซฝั่งลูกค้า 100% (ไม่มีการส่งข้อมูลส่วนบุคคล), เข้ากันได้กับ Tor Browser.



**ไม่ใช่การคุมขัง**: HTLC รับประกันการควบคุมเงินของคุณโดยเฉพาะด้วยคณิตศาสตร์ ไม่ว่าการแลกเปลี่ยนจะสำเร็จหรือไม่ คุณจะได้รับบิตคอยน์ของคุณคืน



**โอเพนซอร์สที่โฮสต์เองได้**: โค้ดสาธารณะที่ตรวจสอบได้, สามารถปรับใช้ในเครื่องเพื่อความต้านทานต่อการเซ็นเซอร์สูงสุด



### ข้อจำกัด



**สภาพคล่องจำกัด**: จำนวนผู้ให้บริการที่ใช้งานอยู่มีจำกัด (Boltz, Eldamar, MiddleWay ขึ้นอยู่กับช่วงเวลา) จำนวนสูงสุดอาจถูกจำกัด



**เวลาหมดอายุ**: หมดเวลาตั้งแต่ 24 ชั่วโมงถึง 48 ชั่วโมง หากธุรกรรม on-chain ไม่ได้รับการยืนยันก่อนหมดอายุ จำเป็นต้องกู้คืนด้วยตนเอง



**Interface centralization**: แม้ว่าจะสามารถโฮสต์เองได้ แต่ส่วนติดต่ออย่างเป็นทางการนั้นโฮสต์บน GitHub Pages หาก GitHub เซ็นเซอร์ repo การเข้าถึงผ่าน swapmarket.github.io จะถูกบล็อก (วิธีแก้ไข: โฮสต์เอง)



**on-chain Traces**: สคริปต์ HTLC อาจถูกระบุได้โดยการวิเคราะห์บล็อกเชนขั้นสูง



## แนวทางปฏิบัติที่ดีที่สุด



### การกำหนดค่าที่ปลอดภัย



**ดาวน์โหลดกุญแจช่วยเหลือของคุณ**: ก่อนการแลกเปลี่ยนครั้งแรกของคุณ ดาวน์โหลดกุญแจช่วยเหลือจากการตั้งค่า (ดูส่วนเฉพาะด้านบน) กุญแจที่ไม่ซ้ำกันนี้จะใช้ได้กับการแลกเปลี่ยนในอนาคตทั้งหมดของคุณ ช่วยให้คุณกู้คืนเงินของคุณในกรณีที่เกิดปัญหา



**ใช้ Tor Browser**: เพื่อความเป็นส่วนตัวสูงสุด เข้าถึง SwapMarket ผ่าน Tor Browser เพื่อซ่อนที่อยู่ IP ของคุณ



**พิจารณาการโฮสต์ด้วยตนเอง**: สำหรับผู้ใช้ที่มีความเชี่ยวชาญทางเทคนิค การรันอินสแตนซ์ SwapMarket ของคุณเองจะช่วยขจัดการพึ่งพาโดเมน GitHub Pages อย่างเป็นทางการ



### การเพิ่มประสิทธิภาพการสลับ



**จับตาดู mempool**: ตรวจสอบ mempool.space ก่อนการ swap-in เลือกช่วงเวลาที่มีกิจกรรมต่ำเพื่อลดค่าใช้จ่าย mining ให้เหลือน้อยที่สุด



**ตรวจสอบที่อยู่**: สำหรับการเปลี่ยนออก ให้ตรวจสอบที่อยู่รับของคุณอย่างละเอียด ใช้การคัดลอกและวางและตรวจสอบ 5 ตัวอักษรแรกและ 5 ตัวอักษรสุดท้าย



**ทดสอบด้วยปริมาณน้อย**: เริ่มต้นด้วยขั้นต่ำที่อนุญาต (25,000 ถึง 50,000 sats) เพิ่มขึ้นทีละน้อยเมื่อคุณเชี่ยวชาญกระบวนการแล้ว



**บันทึกการแลกเปลี่ยนของคุณ**: จดบันทึก ID ของการแลกเปลี่ยน ที่อยู่การแลก และวันหมดอายุ ข้อมูลนี้ช่วยในการติดตามและกู้คืนในกรณีที่เกิดปัญหาทางเทคนิค



### กลยุทธ์การใช้งาน



**ปรับสมดุลกระแสเงินสดของคุณ**: ใช้ SwapMarket เพื่อปรับการจัดสรรระหว่าง on-chain (การออม, ความมั่นคงระยะยาว) และ Lightning (ค่าใช้จ่ายประจำวัน, การชำระเงินทันที) ตามความต้องการที่แท้จริงของคุณ



**คำนวณความสามารถในการทำกำไร**: สำหรับความต้องการสภาพคล่องของ Lightning แบบถาวร เปรียบเทียบต้นทุนสะสมของการแลกเปลี่ยนซ้ำ ๆ กับการเปิดช่องทาง Lightning โดยตรง SwapMarket เหมาะสำหรับการปรับเปลี่ยนครั้งเดียว ไม่จำเป็นสำหรับการไหลขนาดใหญ่ที่เกิดขึ้นเป็นประจำ



## SwapMarket กับ Boltz: มีความแตกต่างกันอย่างไร?



### Boltz: เทคโนโลยี vs. การบริการ



**Boltz เป็นเทคโนโลยีโอเพ่นซอร์ส** (`boltz-backend` บน GitHub) ที่ดำเนินการแลกเปลี่ยนอะตอมผ่าน HTLC ระหว่าง Bitcoin, Lightning และ Liquid.



**Critical point**: ผู้ให้บริการ SwapMarket ทั้งหมด (Boltz Exchange, ZEUS Swaps, Eldamar, Middle Way) ใช้งานอินสแตนซ์ของ Boltz backend ของตนเอง ดังนั้นเทคโนโลยีพื้นฐานจึงเหมือนกัน การมีช่องโหว่ใน Boltz backend อาจส่งผลกระทบต่อผู้ให้บริการทั้งหมด แต่ลักษณะโอเพนซอร์สของระบบช่วยให้ชุมชนสามารถตรวจสอบได้



**Boltz Exchange** เป็นบริการเดียวที่ดำเนินการโดยทีม Boltz ในขณะที่ **SwapMarket** รวบรวมผู้ให้บริการหลายรายที่ใช้เทคโนโลยี Boltz สร้างสภาพแวดล้อมการกำหนดราคาที่มีการแข่งขัน



ดูบทแนะนำ Boltz และ Zeus Swap ของเราสำหรับรายละเอียดเพิ่มเติม:

### ความแตกต่างที่สำคัญ



| Aspect       | Boltz Exchange       | SwapMarket                           |
| ------------ | -------------------- | ------------------------------------ |
| Nature        | Unique service       | Multi-provider aggregator             |
| Providers     | Boltz only           | Boltz, ZEUS, Eldamar, Middle Way     |
| Competition   | Fixed fees           | Open competition                     |
| Interface     | boltz.exchange       | swapmarket.github.io (self-hostable) |
| Security      | Non-custodial (HTLC) | Non-custodial (HTLC)                 |

**SwapMarket benefits**: การแข่งขันด้านราคา, การกระจายความหลากหลายของ backend instances, การเปรียบเทียบแบบเรียลไทม์.



**ทางเลือกทางเทคโนโลยี** (ไม่รองรับ SwapMarket): Lightning Loop (Lightning Labs), Muun Wallet, NLoop, Breez Wallet. โซลูชันเหล่านี้ใช้การดำเนินการของตนเองสำหรับการแลกเปลี่ยนใต้น้ำ



**คำแนะนำ**: ใช้ Boltz Exchange เพื่อความเรียบง่ายหรือ SwapMarket เพื่อเพิ่มประสิทธิภาพด้านต้นทุนผ่านการแข่งขัน ทั้งสองมีความปลอดภัยเทียบเท่ากัน (HTLC แบบไม่รับฝาก)



## บทสรุป



SwapMarket อำนวยความสะดวกในการแลกเปลี่ยน Bitcoin/Lightning โดยการรวมผู้ให้บริการหลายรายเข้ากับอินเทอร์เฟซเดียว สถาปัตยกรรม HTLC รับประกันลักษณะที่ไม่ต้องมีการควบคุมของการแลกเปลี่ยน การไม่มี KYC รักษาความลับ และโค้ดโอเพ่นซอร์สที่สามารถโฮสต์เองได้ช่วยเสริมความต้านทานต่อการเซ็นเซอร์



การแข่งขันระหว่างผู้ให้บริการช่วยปรับปรุงอัตราและเพิ่มแหล่งสภาพคล่อง เพื่อเพิ่มประสิทธิภาพการจัดการสองชั้น (การออม on-chain, ค่าใช้จ่าย Lightning) SwapMarket เป็นเครื่องมือที่ใช้งานได้จริงที่รักษาอธิปไตยทางการเงินและความลับ
