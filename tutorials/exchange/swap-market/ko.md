Bitcoin on-chain와 Lightning Network 간에 자금을 이체하려면 일반적으로 라이트닝 채널을 수동으로 열거나(기술 및 비용이 많이 듭니다) KYC를 통해 중앙화된 스왑 플랫폼을 사용해야 합니다. 스왑마켓은 KYC 없이 경쟁 공급자를 통한 무신뢰 아토믹 스왑이라는 대안을 제공합니다.



혁신: 제공자는 중개자이지만, HTLC(*Hash 시간 고정 계약*)은 회원님의 자금이 회원님의 통제하에 있다는 것을 수학적으로 보장합니다. 여러 제공자(Boltz, ZEUS 스왑, Eldamar, Middle Way)가 모여 가격 경쟁을 유발합니다. Interface 웹 오픈소스 자체 호스팅 가능.



## 스왑마켓이란 무엇인가요?



2024년에 출시된 오픈 소스 애그리게이터인 스왑마켓은 Bitcoin/라이트닝 스왑 제공업체를 비교하는 기능을 합니다. 사용자는 조건(수수료, 유동성, 한도)을 즉시 비교하고 최적의 공급자를 선택할 수 있습니다.



### 기술 아키텍처



**프론트엔드 클라이언트 측**: 100% 클라이언트 측 애플리케이션(fork Boltz 웹 앱)으로 GitHub 페이지에서 호스팅됩니다. 백엔드 서버 없이 브라우저에서 코드가 실행됩니다. 로컬에 기록이 저장됩니다(쿠키/캐시). 공개 및 감사 가능한 소스 코드.



**공급자 검색** : Src/configs/mainnet.ts`에 하드코딩된 목록. 풀 리퀘스트 또는 이메일을 통해 새로운 공급자가 추가되었습니다.



**독립 백엔드**: 각 제공업체는 자체 Boltz 백엔드를 운영합니다. 인터페이스는 실시간으로 API를 쿼리하여 견적을 즉시 비교합니다.



**HTLC 아토믹 스왑**: Hash 시간 고정 계약은 원자성을 보장합니다: 스왑이 실행되거나 각 당사자가 자금을 회수합니다. 거래 상대방 리스크가 수학적으로 제거됩니다.



### 철학



스왑마켓은 공급자 간 수수료와 유동성 경쟁을 유도해 중앙 집중화를 줄입니다. KYC가 없고, 자체 호스팅이 가능한 오픈소스 코드를 사용하며, 독립적인 운영자가 많아 단일 장애 지점을 피할 수 있습니다.



## 주요 기능



### 공급자 마켓플레이스



인터페이스는 모든 활성 공급자를 표시합니다: 공급자 이름, 적용된 수수료(퍼센트 및/또는 고정), 사용 가능한 최소/최대 금액, 지원되는 스왑 유형. 애플리케이션은 구성 파일에 참조된 각 공급자의 API를 직접 쿼리하여 실시간으로 호가를 검색합니다. 공급자 간의 경쟁을 통해 일반적으로 표준 스왑의 경우 0.5% 내외의 최적의 금리가 보장됩니다.



### 양방향 스왑



**스왑인(on-chain → 라이트닝)**: on-chain BTC를 라이트닝 사토시로 전환합니다. 사용 사례: 모바일 wallet 라이트닝에 전력을 공급하거나, 노드에서 들어오는 용량을 확보하거나, 즉각적인 유동성을 확보하세요.



**스왑아웃(라이트닝 → on-chain)**: 라이트닝 사토시를 on-chain BTC로 변환합니다. 사용 사례: wallet 라이트닝을 비워 콜드 스토리지에 보관하거나 레이어 간 유동성을 재조정합니다.



### 안전 및 복구



**Trustless 아토믹 스왑**: HTLC는 교환이 완전히 완료되거나 각 당사자가 지분을 회수할 수 있도록 보장합니다. 거래 상대방 리스크는 수학적으로 제거됩니다.



**상환 메커니즘**: 각 스왑에는 타임락이 있습니다. 스왑이 실패하면 만기 후 자금은 자동으로 환불됩니다. 사용자는 항상 자신의 비트코인을 회수할 수 있는 옵션을 보유합니다.



**복구 키**: 스왑마켓에서는 진행 중인 스왑에 대한 복구 키를 내보낼 수 있습니다. 문제가 발생하면 이 키를 사용하여 모든 장치에서 스왑을 완료하거나 취소할 수 있습니다.



## 설치 및 액세스



### Interface 웹



스왑마켓은 설치가 필요하지 않습니다. 브라우저를 통해 https://swapmarket.github.io 에 접속할 수 있습니다. 기밀성을 극대화하려면 Brave, 추적 방지 확장 기능이 있는 Firefox 또는 LibreWolf를 사용하세요. 네트워크 익명성을 위해 Tor 브라우저를 권장합니다.



등록, 이메일 또는 본인 인증이 필요하지 않습니다.



### 셀프 호스팅(선택 사항)



공식 GitHub 페이지 도메인에 대한 종속성을 없애고자 하는 기술적인 사용자의 경우, SwapMarket을 로컬에서 실행할 수 있습니다:



**엔피엠을 통해** :


```
git clone https://github.com/SwapMarket/swapmarket.github.io.git
cd swapmarket.github.io
npm install
npm run dev
```



**Docker를 통해** :


```
docker run -p 3000:80 ghcr.io/swapmarket/swapmarket:latest
```



애플리케이션은 `http://localhost:3000`에서 액세스할 수 있습니다. 자체 호스팅은 인터페이스에 대한 완전한 제어를 보장하고 공식 도메인의 검열 위험을 제거하며 실행 전에 소스 코드를 감사할 수 있습니다.



### 초기 구성



**Wallet 라이트닝**: 작동 중인 wallet 라이트닝(Phoenix, 제우스, 블루월렛 등)이 있는지 확인하세요. 스왑 인의 경우, 라이트닝 인보이스를 generate에 발행합니다. 스왑 아웃의 경우, 라이트닝 인보이스를 지불하게 됩니다.



**Wallet on-chain**: 스왑인의 경우 자금을 보내려면 wallet Bitcoin on-chain가 필요합니다. 스왑아웃의 경우 Bitcoin 수취 주소를 준비하세요.



**선택적 구성**: 스왑마켓은 브라우저 쿠키에 스왑 내역과 환경설정을 저장합니다. 계정을 만들 필요가 없습니다.

**중요 포인트** :

- 이 키를 **안전하고 영구적인** 장소(비밀번호 관리자, 디지털 금고)에 보관하세요
- 스왑 문제(타임아웃, 기술적 오류)가 발생한 경우 이 키를 통해 자금을 복구할 수 있습니다



## 단계별 스왑 만들기



### 교체: 라이트닝 → Bitcoin



이 첫 번째 예시는 라이트닝 사토시를 on-chain 비트코인으로 변환하는 방법을 보여줍니다.



**1단계: 스왑 구성



메인 페이지에서 스왑 양식 을 선택합니다:




- 라이트닝**(상단 필드): sats 라이트닝으로 송금할 금액을 입력합니다(예: 30,000 sats)
- 비트코인**(아래 필드): 수수료가 공제된 후 받게 될 금액이 자동으로 표시됩니다(예: 29,320 sats)



하단 필드에 자금을 수령할 **수령 Bitcoin 주소**를 입력합니다. 이 주소를 주의 깊게 확인하세요.



기본 제공업체는 일반적으로 Boltz Exchange입니다. 네트워크 요금과 공급자 수수료가 명확하게 표시됩니다.



![Configuration swap-out](assets/fr/03.webp)



**2단계: 제공업체 선택**



공급자 드롭다운 메뉴(기본값: "Boltz Exchange")를 클릭하면 사용 가능한 모든 유동성 공급자가 표시됩니다.



모달 창이 열리고 비교 표가 표시됩니다:




- 상태**: 공급자가 활성 상태인 경우 Green 표시기
- 별칭**: 제공업체 이름(Boltz Exchange, Middle Way, Eldamar, ZEUS 스왑)
- 수수료**: 제공업체가 적용하는 수수료(일반적으로 0.49%~0.5% 사이)
- 최대 스왑**: 스왑에 허용되는 최대 금액



수수료와 최대 금액을 비교한 다음 원하는 제공업체를 선택하세요.



**참고**: 공급자 선택 인터페이스에는 각 공급자에 대한 **최소 금액**이 표시되지 않습니다. 이 정보는 제공업체를 선택한 후 스왑 생성 인터페이스에만 표시됩니다. 최소 및 최대 금액은 제공업체마다 다를 수 있으며 시간이 지남에 따라 변경될 수 있습니다. **스왑할 때 항상 한도를 확인하세요**: 스왑하려는 금액이 제공업체의 한도를 초과하는 경우 거래에 더 적합한 다른 제공업체를 선택할 수 있습니다.



![Sélection du provider](assets/fr/04.webp)



**3단계: 스왑 생성 및 라이트닝** 결제



노란색 **"원자 스왑 생성"** 버튼을 클릭합니다. 스왑마켓에서 generate 라이트닝에서 결제할 수 있도록 **라이트닝 인보이스**(BOLT11)를 wallet로 전송합니다.



페이지가 표시됩니다:




- 스왑 ID**: 고유 스왑 식별자(예: J4ymFIMVR6Hm)
- 상태**: "swap.created"(스왑 생성, 결제 대기 중)
- QR코드**: wallet 라이트닝으로 스캔하세요
- Invoice Lightning**: "lnbc"로 시작하는 문자 문자열(예: lnbc300u1p50whiv...gn5dk2szgqkvfkzc)



wallet Lightning(Phoenix, Zeus, BlueWallet 등)에서 이 청구서를 결제합니다. 결제할 정확한 금액이 표시됩니다(예: 30,000 sats).



![Paiement Lightning](assets/fr/05.webp)



**4단계: 확인 및 수락**



라이트닝 결제가 확인되면 스왑마켓은 즉시 결제를 수령하고 공급자는 Bitcoin 거래를 회원님의 주소로 브로드캐스트합니다.



상태가 **"인보이스 결제 완료"**(인보이스 결제됨)로 변경되고 확인 메시지가 표시됩니다.



on-chain 비트코인은 거래가 확인되는 즉시 사용할 수 있습니다(일반적으로 공급자가 선택한 mining 수수료에 따라 몇 분에서 몇 시간 정도 소요됨).



![Confirmation swap-out](assets/fr/06.webp)



"클레임 트랜잭션 열기"**를 클릭하면 블록체인 브라우저에서 Bitcoin 트랜잭션을 볼 수 있습니다.



### 스왑인: Bitcoin → 라이트닝



두 번째 예시는 on-chain 비트코인을 라이트닝 사토시로 변환하는 방법을 보여줍니다.



**1단계: 스왑 구성



메인 페이지에서 스왑 양식 을 선택합니다:




- 비트코인**(상단 필드): 송금하고자 하는 금액을 sats Bitcoin로 입력합니다(예: 63,400 sats)
- 라이트닝**(하단 필드): 수수료 공제 후 받게 될 금액이 자동으로 표시됩니다(예: 62 884 sats)



하단 필드에 wallet 라이트닝에서 생성된 라이트닝** 인보이스(BOLT11)를 붙여넣거나 wallet에서 지원하는 경우 LNURL 주소를 사용합니다.



![Configuration swap-in](assets/fr/07.webp)



**2단계: 구조 키 확인**



"원자 스왑 생성"**을 클릭하면 구조 키를 인증하라는 모달 창이 나타납니다.



![Modal Rescue Key](assets/fr/08.webp)



**볼츠 복구 키**: 초기 구성 시 복구 키를 이미 업로드했으므로(이전 섹션 참조) **"기존 키 확인"** 버튼을 클릭하여 저장한 키를 가져옵니다.



이전에 다운로드한 구조 키 파일을 선택합니다. 인증에 성공하면 인터페이스가 자동으로 다음 단계로 전환됩니다.



**3단계: Bitcoin** 입금 주소



스왑마켓은 이제 라이트닝 인보이스에 연결된 HTLC 계약이 포함된 **고유 Bitcoin 주소**를 생성합니다.



페이지가 표시됩니다:




- 스왑 ID**: 고유 식별자(예: 1kGmB6JyGqU4)
- 상태**: "invoice.set"(인보이스 세트, 결제 대기 중 Bitcoin)
- QR코드**: Bitcoin 창고 주소
- Bitcoin** 주소: 보통 "bc1p..."로 시작합니다 (예: bc1p5mvtwxapjkds...9d4n9f)
- 노란색 경고** : "이 스왑 생성 후 ~24시간 이내에 거래가 확인되었는지 확인하세요!"



최대 24시간은 HTLC 계약의 **타임아웃** 기간입니다. 이 기간 내에 Bitcoin 거래가 확인되지 않으면 스왑이 실패하며, 자금을 복구하려면 구조 키를 사용해야 합니다.



![Adresse de dépôt Bitcoin](assets/fr/09.webp)



"주소"** 버튼을 클릭하여 주소를 복사하거나 wallet on-chain에서 직접 QR 코드를 스캔할 수 있습니다.



**4단계: 비트코인 보내기**



wallet Bitcoin on-chain에서 표시된 금액(예: 63,400 sats)을 생성된 주소로 **정확하게** 송금하세요.



**중요**: 빠른 확인을 위해 적절한 mining 수수료를 사용하세요. 수수료가 너무 낮고 트랜잭션이 타임아웃(~24시간) 이후에도 멤풀에 남아 있으면 스왑이 실패합니다.



트랜잭션이 전송되면 스왑마켓은 트랜잭션이 멤풀에 있음을 감지하고 을 표시합니다:




- Status** : "트랜잭션.mempool

## 장점과 한계



### 혜택



**요금 경쟁**: 제공업체가 모이면 자연스러운 경쟁이 발생하여 수수료가 낮아집니다(0.49%~0.5%).



**기밀성**: KYC 없음, 100% 클라이언트 측 인터페이스(개인 데이터 전송 없음), 토르 브라우저 호환.



**비수탁**: HTLC은 여러분의 자금에 대한 독점적 통제권을 수학적으로 보장합니다. 스왑이 성공하면 비트코인을 돌려받을 수 있습니다.



**자체 호스팅 가능한 오픈 소스**: 검열에 최대한 저항할 수 있도록 로컬에 배포할 수 있는 감사 가능한 공개 코드.



### 제한 사항



**유동성 제한**: 활성 공급자 수가 제한되어 있습니다(기간에 따라 Boltz, Eldamar, MiddleWay). 최대 금액이 제한될 수 있습니다.



**만료 시간**: 24시간에서 48시간까지의 타임아웃. 만료 전에 on-chain 트랜잭션이 확인되지 않으면 수동 복구가 필요합니다.



**Interface 중앙 집중화**: 자체 호스팅이 가능하지만 공식 인터페이스는 GitHub 페이지에서 호스팅됩니다. GitHub에서 리포지토리를 검열하면 swapmarket.github.io를 통한 액세스가 차단됩니다(해결 방법: 자체 호스팅).



**on-chain 추적**: HTLC 스크립트는 고급 블록체인 분석을 통해 잠재적으로 식별할 수 있습니다.



## 모범 사례



### 보안 구성



**구조 키 다운로드**: 첫 스왑을 시작하기 전에 설정에서 구조 키를 다운로드하세요(위의 전용 섹션 참조). 이 고유 키는 향후 모든 스왑에서 작동하며, 문제가 발생했을 때 자금을 복구할 수 있도록 도와줍니다.



**토르 브라우저 사용**: 기밀성을 극대화하려면 토르 브라우저를 통해 스왑마켓에 접속하여 IP 주소를 숨기세요.



**자체 호스팅을 고려하세요**: 기술적인 사용자의 경우, 자체 SwapMarket 인스턴스를 실행하면 공식 GitHub 페이지 도메인에 대한 종속성을 제거할 수 있습니다.



### 스왑 최적화



**멤풀을 주시하세요**: 스왑인 전에 mempool.space를 확인하세요. 활동량이 적은 시간을 선택하여 mining 비용을 최소화하세요.



**주소 확인**: 스왑아웃의 경우 수령 주소를 꼼꼼하게 확인하세요. 복사 및 붙여넣기를 사용하여 처음 5자와 마지막 5자를 확인하세요.



**소량으로 테스트**: 허용되는 최소량(25,000~50,000 sats)으로 시작하세요. 프로세스를 숙달한 후 점차적으로 양을 늘립니다.



**스왑을 문서화하세요**: 각 스왑의 ID, 상환 주소, 만료일을 기록해 두세요. 이 정보는 기술적 문제 발생 시 추적과 복구를 용이하게 해줍니다.



### 사용 전략



**현금 흐름의 균형을 맞추세요**: 스왑마켓을 사용하여 실제 필요에 따라 on-chain(저축, 장기 보안)과 라이트닝(일일 경비, 즉시 결제) 간의 할당을 조정하세요.



**수익성 계산하기**: 영구적인 라이트닝 유동성이 필요한 경우, 반복 스왑의 누적 비용과 라이트닝 채널을 직접 개설하는 비용을 비교해보세요. 스왑마켓은 일회성 조정에 탁월하지만, 대규모 정기적 흐름에는 적합하지 않습니다.



## 스왑마켓과 볼츠: 차이점은 무엇인가요?



### Boltz: 기술 대 서비스



**볼츠는 오픈 소스 기술**(GitHub의 '볼츠 백엔드')로, HTLC을 통해 Bitcoin, 라이트닝, Liquid 간의 아토믹 스왑을 구현합니다.



**중요 포인트**: 모든 스왑마켓 제공자(Boltz Exchange, 제우스 스왑, 엘다마르, 미들웨이)는 자체적인 Boltz 백엔드 인스턴스를 배포합니다. 따라서 기본 기술은 동일합니다. 볼츠 백엔드의 취약점은 잠재적으로 모든 공급자에게 영향을 미칠 수 있지만, 시스템의 오픈 소스 특성으로 인해 커뮤니티 감사가 가능합니다.



**볼츠 Exchange**은 볼츠 팀이 운영하는 단일 서비스이며, **스왑마켓**은 볼츠 기술을 사용하는 여러 공급자를 한데 모아 경쟁력 있는 가격 환경을 조성합니다.

## 결론



스왑마켓은 여러 공급자를 단일 인터페이스로 통합하여 Bitcoin/라이트닝 교환을 용이하게 합니다. HTLC 아키텍처는 스왑의 비수탁성을 보장하고, KYC가 없어 기밀성을 유지하며, 자체 호스팅 가능한 오픈 소스 코드로 검열에 대한 저항력을 강화합니다.



공급자 간의 경쟁은 금리를 개선하고 유동성 공급원을 늘립니다. 스왑마켓은 2계층 관리(on-chain 절감, 라이트닝 비용)를 최적화하기 위해 재정 주권과 기밀을 유지하는 실용적인 도구입니다.
