KaleidoSwap は、RGB プロトコルと Lightning Network のギャップを埋める洗練されたオープンソースのデスクトップ・アプリケーションです。RGBライトニング・ノードの管理、LSPS1仕様によるRGBライトニング・サービス・プロバイダー（LSP）とのやりとり、RGBアセットのアトミック・スワップ実行のための包括的なインターフェースとして機能します。


非保管型ソリューションとして、KaleidoSwapはユーザーが鍵とデータを完全に管理できるようにする。RGBのクライアント側検証パラダイムを活用することで、Bitcoinの上にプライベートでスケーラブルなスマート・コントラクトを実現します。このチュートリアルでは、KaleidoSwapの高度な機能を掘り下げ、「カラー」UTXO管理の複雑さ、特定資産のチャネル流動性、テイカー・メイカー取引モデルについて説明し、この強力な分散型取引所プロトコルを十分に活用できるようにします。


## インストール


KaleidoSwap は主要なオペレーティング・システム用のビルド済みバイナリを提供しますが、上級ユーザはソースからビルドすることで、最新のコードを特定の設定で実行することができます。


### バイナリのダウンロード


お使いのOSの最新リリースは、[公式ウェブサイト](https://kaleidoswap.com/) または [GitHub リリースページ](https://github.com/kaleidoswap/desktop-app/releases) からダウンロードできます。


### 設置方法


1.  **Windows**：.exeインストーラをダウンロードして実行してください。

2.  **macOS**：.dmg`ファイルをダウンロードして開き、KaleidoSwapをアプリケーションフォルダにドラッグしてください。

3.  **Linux**：.AppImage` または `.deb` ファイルをダウンロードし、インストール/実行してください。



## ノード設定オプション


KaleidoSwapを初めて起動すると、**接続画面**が表示されます。これが環境設定の第一歩です。


![Node Selection Screen](assets/en/01.webp)


RGB Lightning Network への接続方法を選択する必要があります。この選択は、コントロールとプライバシーのレベルに影響します。


### オプション1：ローカルノード（セルフカストディに推奨）


**完全なコントロールとプライバシー**のためには、あなたのマシン上で直接ノードを実行してください：


- 自己所有**：あなたが鍵を握っています。第三者があなたの資金を凍結したり、取引を検閲することはできません。
- プライバシー**：あなたのデータはあなたのデバイスに残ります。
- 独立性**：外部のサービスプロバイダーに依存しない。


ローカル・ノード**を選択すると、walletの新規作成または既存のノードのリストアができるセットアップ画面が表示されます。


![Local Node Setup Screen](assets/en/02.webp)


### オプション2：リモート・ノード


リモートのRGB Lightning Nodeに接続する（VPSまたはホスティングプロバイダーでセルフホスト）。


- メリット**：ローカルリソースを使用せず、24時間365日利用可能。
- トレードオフ**：ホストを信頼するか、リモートサーバーを管理する必要がある。


![Remote Node Setup Screen](assets/en/03.webp)


**KaleidoSwapはセルフ・カストディを強化するように設計されています。** BitcoinとRGBの検閲に強い特性を完全に活用するために、ローカル（オプション1）またはリモート・ノードをセルフ・ホストして、あなた自身のノードを実行することを強くお勧めします。


## Walletの作成


KaleidoSwapはBitcoinとRGBの両方のアセットを管理する。walletの作成プロセスでは、on-chainの資金とLightningチャネルの状態を保護するキーストアを初期化します。


詳しいプロセスはこうだ：

1.KaleidoSwapを開き、**ローカルノード**を選択します。

2.新しいWalletを作成する**をクリックしてください。

3. **アカウント設定**：アカウント名**を入力し、**ネットワーク**（例：Bitcoin Mainnet Testnet Signet）を選択します。

4. **詳細設定** (オプション)：パワーユーザーであれば、"Advanced Settings "の下でカスタムRPCエンドポイント、Indexer URL、またはProxy設定を構成することができます。

5.Continue**をクリックしてください。

6. **パスワード**：walletファイルをローカルで暗号化するための強力なパスワードを設定します。

7. **リカバリーフレーズseedのフレーズを書き留め、安全に保管してください。


    - Critical**：このフレーズはon-chainの資金とノードのIDを回復するために必要です。
    - 警告**： **ライトニング・チャンネルの状態は、seed だけでは完全に回復することはできません**。また、チャンネルにロックされた資金を回復するために、スタティック・チャンネル・バックアップ（SCB）を維持する必要があります。


![Wallet Creation Screen](assets/en/04.webp)

## Walletの資金調達


RGB資産で運用するには、walletに資金を入金する必要があります。KaleidoSwapは、on-chain取引またはLightning Networkを介してBitcoinとRGBの両方の資産の預け入れをサポートしています。


### Bitcoinを預ける


1.クイックアクションメニューの**預金**をクリックします。

2.資産リストから**BTC**を選択します。


![Select BTC Asset](assets/en/06.webp)


3.入金方法をお選びください： **オンチェーン**または**ライトニング**。


![BTC Deposit Options](assets/en/07.webp)



- オンチェーン**：別のwalletからBitcoinを送信するには、QRコードをスキャンするか、アドレスをコピーします。
- ライトニング**：希望金額の請求書を作成します。


![BTC On-chain Deposit](assets/en/08.webp)


### RGBアセットとカラーUTXOを預ける


RGBアセット（USDTのような）を受け取るには、特定のUTXOを「色付け」（アセットを割り当てる）する必要があります。


1.Deposit**をクリックし、RGBアセット（例：USDT）を選択します。 **重要**です：ノードがこの特定のアセットを受け取るのが**初めて**の場合、**アセットIDフィールドは空のままにしておいてください。不明なアセットのIDを入力すると、ノードはそのアセットをまだ認識していないため、エラーを返します。

2.オンチェーン**または**ライトニング**を選択する。


![USDT Deposit Options](assets/en/09.webp)


#### オンチェーン受信モードウィットネス対ブラインド


RGB資産on-chainを受信する場合、2つのプライバシーモードがあります：



- ブラインド受信（プライバシー保護のため推奨）**：あなたは送信者に「blinded」UTXOを提供します。あなたは送信者に、あなたが所有する既存のUTXOにアセットを送信するよう依頼しますが、実際のUTXOの識別子を難読化します。これはより良いプライバシーを提供します。
- 証人受理**：あなたは標準的な Bitcoin のアドレスを提供します。あなたはそのアドレスにアセットを送信することで、あなたのために*新しい*UTXOを作成するよう送信者に依頼します。これにより、RGBアセットがトランザクションによって作成された新しいUTXOに直接アタッチされます。


#### ライトニング・デポジット


ライトニング入金の場合は、generateに請求書を発行するだけです。RGBのアセットがオープン・チャネルを通じてあなたに送られます。


![USDT Lightning Invoice](assets/en/10.webp)



## RGBアセットでチャンネルを開く


RGB の資産を Lightning Network でルーティングするには、十分な流動性と資産配分を持つチャネルが必要です。最も簡単な方法は、LSP（ライトニング・サービス・プロバイダー）から**チャネル**を購入することです。


### カレイドLSPからのチャンネル購入


1.Channels**タブを開く。Open Channel"(手動)または "Buy Channel"(LSP)のオプションが表示されます。

2.チャンネル購入**をクリック。


![Channels Dashboard](assets/en/11.webp)


3. **LSP**に接続します：アプリはデフォルトのカレイドLSPに接続します。このプロバイダーはインバウンドの流動性を提供し、RGB アセット・ルーティングをサポートします。


![Connect to LSP](assets/en/12.webp)


4. **チャンネルを設定する：


    - 容量**：チャンネルの合計Bitcoin容量を選択します。
    - RGB アロケーション**：受信または送信できるようにしたいRGBアセット（USDTなど）を選択します。LSPは、チャネルがこのアセットをサポートするように設定されていることを確認します。


![Configure Channel](assets/en/13.webp)



    - RGB アロケーション**：受信または送信できるようにしたい RGB アセット（USDT など）を選択します。LSP は、チャネルがこのアセットをサポートするように設定されていることを確認します。


![RGB Allocation](assets/en/14.webp)


5.  **支払い**：チャネルを開設し流動性を提供するために、LSPに手数料を支払う必要があります。支払いは**Lightning**または**On-chain** Bitcoinを使って行うことができます。支払いは内部 KaleidoSwap wallet または外部 wallet から行うことができます。


![Complete Payment](assets/en/15.webp)


6.支払いが確認されると、LSPはチャネル開設トランザクションを開始します。注文完了**画面が表示されます。


![Order Completed](assets/en/16.webp)


7.ブロックチェーン上で確認されると、あなたのチャンネルはアクティブになり、RGB送金の準備が整います。



## トレーディングテイカー・メイカー・モデル


KaleidoSwapの取引エンジンは**Taker-Makerモデル**で動作します。あなたはピアと手動で資産を交換したり、マーケットメーカー（LSP）を使用することができます。


### マーケットメーカー（LSP）とのスワッピング


これは最も一般的な取引方法です。お客様は**テイカー**として、LSP（**メイカー**）が提供する利用可能な流動性に対して注文を執行します。


1.Trade**タブに移動し、**Market Maker**を選択します。

2. **Enter Amount**：送信したいBitcoinの金額（または受信したい資産）を入力します。インターフェイスには、推定為替レートと手数料が表示されます。


![Market Maker Swap](assets/en/17.webp)


3. **スワップを確認する為替レートや正確な受取額などの詳細を確認してください。スワップ確認**をクリックします。


![Confirm Swap](assets/en/18.webp)


4. **処理**：スワップはLightning Network上でアトミックに実行される。スワップが保留中であることを示すステータス画面が表示されます。


![Swap Pending](assets/en/19.webp)


5. **成功**：HTLCが決済されると、スワップは完了し、アセットがあなたのチャネルに入ります。


![Swap Success](assets/en/20.webp)



## デベロッパー API


KaleidoSwapの上に構築する開発者のために、アプリケーションはサポートするAPIを公開する：



- RGB LSPS1**：自動流動性サービス用。
- スワップAPI**：プログラマティック取引およびマーケットメイク用。
- WebSocket**：リアルタイムのマーケットデータ購読用。


エンドポイントや仕様の詳細については[APIドキュメント](https://docs.kaleidoswap.com/api/introduction)を参照。

KaleidoSwapは、Lightning Network上でRGBアセットのプライバシーとスケーラビリティを活用することを可能にします。Colored UTXOとチャネルアセットアロケーションを理解することで、この強力な分散型交換プロトコルを十分に活用することができます。
