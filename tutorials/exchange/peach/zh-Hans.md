## 导言



无 KYC 点对点（P2P）交易所对于保护用户的保密性和金融自主权至关重要。它们实现了个人之间的直接交易，无需身份验证，这对于那些重视隐私的人来说至关重要。如需更深入地了解理论概念，请参阅 BTC204 课程：



https://planb.academy/courses/65c138b0-4161-4958-bbe3-c12916bc959c

### 1.什么是 Peach？



Peach 是一个 P2P 交易所平台，允许用户在无需 KYC 的情况下买卖比特币。它提供直观的界面和先进的安全功能。与 Bisq、HodlHodl 和 Robosat 等其他解决方案相比，Peach 以其易用性脱颖而出。


multisignature 托管系统（2-2）保证了交易过程中的资金安全。Peach 支持各种付款方式，并设有信誉系统，为交易者的行为提供指导。因此，与 P2P 平台一样，保持良好的声誉对于维护在其他交易者心目中的信誉非常重要。



### 2.隐私和收集的数据



**Peach 收集哪些信息？



Peach 致力于将用户数据存储到最低限度。以下是存储在我们服务器上的数据概览：





- 您的唯一应用程序标识符 (AdID) 的 hash
- 付款数据 hash
- 您的加密对话
- 确保匿名用户不超过交易限额的交易数据（使用的支付方式类型、买卖金额）
- 用于发送和接收代管账户信息的 Addresses
- 使用数据（Firebase 和 Google Analytics），仅在您同意的情况下使用



需要提醒的是，hash 是一种无法识别的数据，类似于加密。相同的数据总是会产生相同的 hash，因此可以在不知道原始数据的情况下检测出重复数据。



*有关 hashing 的更详细解释，请参阅本课程：* 。



https://planb.academy/courses/46b0ced2-9028-4a61-8fbc-3b005ee8d70f

**谁可以查看我的付款详情？





- 只有您的交易对手可以看到您的付款详情
- 数据通过 Peach 服务器传输，但从头到尾完全加密
- 如果发生争议，指定的 Peach 调解员将看到您的付款详情和对话历史记录

### 4.如何保护您的投资组合



**了解您的 Peach 账户



Peach 账户没有用户名和密码。它是一个存储在您手机本地的文件，这意味着 Peach 不需要存储您的数据，也不需要知道您的身份：您可以掌控一切。该文件包含您的所有数据：包括 12 个比特币恢复词、PGP 密钥、支付详情等。因此，保存这个文件并使用__robust__密码保护它是至关重要的。



这种方法保证了一定程度的保密性，并将数据和备份管理的责任交给了用户。在没有备份的情况下丢失手机，就意味着无法访问 Peach 账户和资金。



**创建备份**






- 从主屏幕右下角的选项卡访问设置
- 在设置菜单中选择 "备份 "选项



![Processus de sauvegarde](assets/fr/06.webp)



有两种备份方式可供选择：



**保存账户文件（图像 14）**




- 点击 "创建新备份
- 创建**强**密码来加密备份文件
- 将该文件发送到一个位置，以确保手机丢失时的冗余性。



文件备份可恢复完整的 Peach 账户，包括 .NET 和 .NET 文件：




- 您的投资组合
- 您的付款方式
- 付款数据
- 交易历史，包括交易对手的详细信息以及与他们的对话



**保存恢复短语（图像 15）**




- 按照说明显示恢复短语
- 按正确顺序认真写下单词
- 将备份存储在一个安全的位置，最好与账户文件不同



恢复短语允许您恢复 .NET Framework 3.0：




- 你的声誉，你的行业
- 您的比特币资金



但**不**以下内容：




- 您当前和过去的对话
- 付款数据

## 优缺点



### Peach 的优势





- 无需 KYC**：为用户保密。
- 无法访问银行详情**：Peach 无法访问您的银行详情或您的身份。
- Interface 直观**：便于中级用户使用。
- 开放源代码** ：源代码公开，可由社区验证。



### Peach 的缺点





- 有限的 Liquidity**：交易量低于更成熟的平台。
- 监管风险** ：应用程序由一家瑞士公司管理。因此，它受瑞士法规的约束，这些法规可能会不断变化，并有可能对应用程序进行审查。
