볼츠만 계산기는 다른 고급 지표와 함께 엔트로피 수준을 측정하여 Bitcoin 트랜잭션을 분석하는 도구입니다. 이는 트랜잭션의 입력과 출력 사이의 연결에 대한 인사이트를 제공합니다. 이러한 지표는 거래의 개인정보 보호에 대한 정량화된 평가를 제공하고 잠재적인 오류를 식별하는 데 도움이 됩니다.


이 파이썬 도구는 사무라이 Wallet와 OXT 팀에서 개발했지만, 모든 Bitcoin 트랜잭션에서 사용할 수 있습니다.


## 볼츠만 계산기는 어떻게 사용하나요?

볼츠만 계산기를 사용하려면 두 가지 옵션을 사용할 수 있습니다. 첫 번째는 컴퓨터에 직접 Python 도구를 설치하는 것입니다. 또는 간소화된 사용 플랫폼을 제공하는 KYCP.org(_Know Your Coin Privacy_) 웹사이트를 선택할 수도 있습니다. 로닌도조 사용자의 경우, 이 도구가 이미 노드에 통합되어 있다는 점에 유의하세요.


KYCP 사이트 이용 방법은 매우 간단합니다. 검색창에 분석하고자 하는 거래 식별자(txid)를 입력한 후 '엔터'를 누르기만 하면 됩니다.

## 볼츠만 계산기의 지표는 무엇인가요?

### 조합 / 해석:

소프트웨어가 계산하는 첫 번째 지표는 도구의 'nb 조합' 또는 '해석' 아래에 표시된 가능한 조합의 총 개수입니다.


이 지표는 트랜잭션에 관련된 UTXO의 값을 고려하여 입력과 출력을 연관시킬 수 있는 방법의 수를 계산합니다. 즉, 트랜잭션을 분석하는 외부 관찰자의 관점에서 트랜잭션이 도출할 수 있는 그럴듯한 해석의 수를 결정합니다.

예를 들어, Whirlpool 5x5 모델에 따라 구조화된 CoinJoin는 '1,496개'의 가능한 조합을 제시합니다: ![KYCP](assets/4.webp)

반면에 Whirlpool 서지 사이클 8x8 CoinJoin은 '9,934,563'의 가능한 해석을 제시합니다: ![KYCP](assets/5.webp)

반면, 입력이 1개이고 출력이 2개인 보다 전통적인 트랜잭션은 ![KYCP](assets/6.webp)라는 단일 해석만 표시합니다


### 엔트로피:

두 번째로 계산되는 지표는 트랜잭션의 엔트로피이며, '엔트로피'로 지정됩니다.


암호화 및 정보의 일반적인 맥락에서 엔트로피는 데이터 소스 또는 임의의 프로세스와 관련된 불확실성 또는 예측 불가능성을 정량적으로 측정하는 척도입니다. 다시 말해, 엔트로피는 정보를 예측하거나 추측하기가 얼마나 어려운지를 측정하는 방법입니다.


체인 분석의 특정 맥락에서 엔트로피는 섀넌 엔트로피에서 파생된 지표의 이름이기도 하며, 볼츠만 도구로 계산되는 [로랑MT가 개발한](https://gist.github.com/LaurentMT/e758767ca4038ac40aaf) 지표입니다.


트랜잭션에 가능한 조합의 수가 많을 때는 엔트로피를 참조하는 것이 더 적절할 때가 많습니다. 이 지표를 통해 거래의 정확한 구성에 대한 분석가의 지식 부족을 측정할 수 있습니다. 즉, 엔트로피가 높을수록 분석가가 입력과 출력 사이의 Bitcoin 움직임을 식별하는 작업이 더 어려워집니다.


실제로 엔트로피는 외부 관찰자의 관점에서 거래가 다른 외부 또는 내부 패턴과 휴리스틱을 고려하지 않고 입력과 출력의 양에만 근거하여 여러 가지 해석이 가능한지 여부를 나타냅니다. 따라서 엔트로피가 높을수록 트랜잭션의 프라이버시가 더 잘 보호된다는 뜻입니다.


엔트로피는 가능한 조합의 수에 대한 2진 로그로 정의됩니다. 다음은 사용된 공식입니다:

```plaintext
E: the entropy of the transaction
C: the number of possible combinations for the transaction

E = log2(C)
```


수학에서 이항로그(기저-2 로그)는 2를 지수화하는 역연산에 해당합니다. 즉, `x`의 이항 로그는 `2`를 올리면 `x`를 얻어야 하는 지수입니다. 따라서 이 지표는 비트로 표현됩니다.


앞서 언급했듯이 `1,496`의 가능한 조합을 제공하는 Whirlpool 5x5 모델에 따라 구조화된 CoinJoin 트랜잭션의 엔트로피를 계산하는 예를 들어보겠습니다:

```plaintext
C = 1,496
E = log2(1,496)
E = 10.5469 bits
```

따라서 이 CoinJoin 트랜잭션의 엔트로피는 '10.5469 비트'로 매우 만족스러운 것으로 간주됩니다. 이 값이 높을수록 트랜잭션이 더 많은 다른 해석을 허용하므로 프라이버시 수준이 강화됩니다.

'9,934,563'의 해석을 제시하는 8x8 CoinJoin 트랜잭션의 경우 엔트로피는 다음과 같습니다:

```plaintext
C = 9,934,563
E = log2(9,934,563)
E = 23.244 bits
```


하나의 입력과 두 개의 출력이 있는 좀 더 일반적인 트랜잭션의 다른 예를 살펴봅시다: [1b1b0c3f0883a99f1161c64da19471841ed12a1f78e77fab128c69a5f578ccce](https://Mempool.space/tx/1b1b0c3f0883a99f1161c64da19471841ed12a1f78e77fab128c69a5f578ccce) 이 트랜잭션의 경우 가능한 유일한 해석은 '(In.0) > (Out.0 ; Out.1)`뿐입니다. 따라서 엔트로피는 `0`으로 설정됩니다:

```plaintext
C = 1
E = log2(1)
E = 0 bits
```


### 효율성:

볼츠만 계산기가 제공하는 세 번째 지표는 'Wallet 효율성'입니다. 이 지표는 동일한 구성에서 생각할 수 있는 최적의 트랜잭션과 비교하여 트랜잭션의 효율성을 평가합니다.


이를 통해 특정 트랜잭션 구조가 이론적으로 달성할 수 있는 가장 높은 엔트로피에 해당하는 최대 엔트로피의 개념에 대해 논의할 수 있습니다. 그런 다음 이 최대 엔트로피와 분석된 트랜잭션의 실제 엔트로피를 비교하여 트랜잭션의 효율성을 계산합니다.


사용된 공식은 다음과 같습니다:

```plaintext
ER: the actual entropy of the transaction expressed in bits
EM: the maximum possible entropy for a given transaction structure expressed in bits
Ef: the efficiency of the transaction in bits

Ef = ER - EM
```


예를 들어 Whirlpool 5x5 타입 CoinJoin 구조의 경우 최대 엔트로피는 '10.5469'로 설정됩니다:

```plaintext
ER = 10.5469
EM = 10.5469
Ef = 10.5469 - 10.5469 = 0 bits
```


이 지표는 백분율로도 표시되며, 그 공식은 다음과 같습니다:

```plaintext
CR: the actual number of possible combinations
CM: the maximum number of possible combinations with the same structure
Ef: the efficiency expressed as a percentage

Ef = CR / CM
Ef = 1,496 / 1,496
Ef = 100%
```


따라서 효율성이 '100%'라는 것은 트랜잭션의 구조에 따라 프라이버시 잠재력이 극대화되었음을 의미합니다.


### 엔트로피 밀도:

네 번째 지표는 '엔트로피 밀도' 도구에 표시된 엔트로피 밀도입니다. 이는 트랜잭션의 각 입력 또는 출력과 관련된 엔트로피에 대한 관점을 제공합니다. 이 지표는 다양한 규모의 트랜잭션의 효율성을 평가하고 비교하는 데 유용합니다. 이를 계산하려면 트랜잭션의 총 엔트로피를 관련된 총 입력 및 출력 수로 나누기만 하면 됩니다:

```plaintext
ED: the entropy density expressed in bits
E: the entropy of the transaction expressed in bits
T: the total number of inputs and outputs in the transaction

ED = E / T
```


Whirlpool 5x5 CoinJoin의 예를 들어 보겠습니다:

```plaintext
T = 5 + 5 = 10
E = 10.5469
ED = 10.5469 / 10 = 1.054 bits
```


또한 Whirlpool 8x8 CoinJoin의 엔트로피 밀도를 계산해 보겠습니다:

```plaintext
T = 8 + 8 = 16
E = 23.244
ED = 23.244 / 16 = 1.453 bits
```


### 볼츠만 점수:

볼츠만 계산기가 제공하는 다섯 번째 정보는 입력과 출력 간의 일치 확률 표입니다. 이 표는 볼츠만 점수를 통해 특정 입력이 주어진 출력과 관련되어 있을 조건부 확률을 나타냅니다.


따라서 일련의 해석에서 이 이벤트의 유리한 발생 횟수와 가능한 총 발생 횟수의 비율을 기준으로 거래에서 입력과 출력 간의 연관성이 발생할 조건부 확률을 정량적으로 측정한 것입니다.


Whirlpool CoinJoin의 예를 다시 살펴보면 조건부 확률 표는 각 입력과 출력 간의 연결 가능성을 강조하여 거래에서 연관성의 모호성을 정량적으로 측정할 수 있습니다:


| %       | Output 0 | Output 1 | Output 2 | Output 3 | Output 4 |
| ------- | -------- | -------- | -------- | -------- | -------- |
| Input 0 | 34%      | 34%      | 34%      | 34%      | 34%      |
| Input 1 | 34%      | 34%      | 34%      | 34%      | 34%      |
| Input 2 | 34%      | 34%      | 34%      | 34%      | 34%      |
| Input 3 | 34%      | 34%      | 34%      | 34%      | 34%      |
| Input 4 | 34%      | 34%      | 34%      | 34%      | 34%      |

여기서 각 입력이 모든 출력과 연결될 확률이 동일하다는 것을 분명히 알 수 있으며, 이는 트랜잭션의 프라이버시를 향상시킵니다.

볼츠만 점수를 계산하려면 특정 이벤트가 발생한 해석의 수를 사용 가능한 총 해석의 수로 나누어야 합니다. 따라서 입력 번호 0과 출력 번호 3(`512` 해석)을 연결하는 점수를 결정하기 위해 다음 절차가 사용됩니다:

```plaintext
Interpretations (IN.0 > OUT.3) = 512
Total Interpretations = 1496
Score = 512 / 1496 = 34%
```


Whirlpool 8x8 CoinJoin(서지 주기)의 예를 들어 볼츠만 테이블은 다음과 같습니다:


|       | OUT.0 | OUT.1 | OUT.2 | OUT.3 | OUT.4 | OUT.5 | OUT.6 | OUT.7 |
|-------|-------|-------|-------|-------|-------|-------|-------|-------|
| IN.0  | 23%   | 23%   | 23%   | 23%   | 23%   | 23%   | 23%   | 23%   |
| IN.1  | 23%   | 23%   | 23%   | 23%   | 23%   | 23%   | 23%   | 23%   |
| IN.2  | 23%   | 23%   | 23%   | 23%   | 23%   | 23%   | 23%   | 23%   |
| IN.3  | 23%   | 23%   | 23%   | 23%   | 23%   | 23%   | 23%   | 23%   |
| IN.4  | 23%   | 23%   | 23%   | 23%   | 23%   | 23%   | 23%   | 23%   |
| IN.5  | 23%   | 23%   | 23%   | 23%   | 23%   | 23%   | 23%   | 23%   |
| IN.6  | 23%   | 23%   | 23%   | 23%   | 23%   | 23%   | 23%   | 23%   |
| IN.7  | 23%   | 23%   | 23%   | 23%   | 23%   | 23%   | 23%   | 23%   |

그러나 입력이 하나이고 출력이 둘인 간단한 트랜잭션의 경우에는 상황이 다릅니다:


| %       | Output 0 | Output 1 |
|---------|----------|----------|
| Input 0 | 100%     | 100%     |

여기서 각 출력이 입력 0번에서 시작될 확률은 '100%'입니다. 따라서 확률이 낮을수록 입력과 출력 사이의 직접적인 연결이 희석되어 프라이버시가 강화됩니다.


### 결정론적 링크:

여섯 번째 정보는 결정적 링크의 수이며, 이러한 링크의 비율로 보완됩니다. 이 지표는 분석된 트랜잭션에서 입력과 출력 사이에 얼마나 많은 연결이 '100%'의 확률로 확실한지를 나타냅니다. 반면, 비율은 전체 트랜잭션 링크 세트 내에서 이러한 결정적 링크의 가중치에 대한 관점을 제공합니다.

예를 들어, Whirlpool 유형의 CoinJoin 트랜잭션에는 결정론적 링크가 없으므로 지표와 비율이 `0%`로 표시됩니다. 반대로 두 번째로 살펴본 단순 트랜잭션(입력 1개, 출력 2개)에서는 지표가 `2`로 설정되고 비율은 `100%`에 이릅니다. 따라서, null 지표는 입력과 출력 사이에 직접적이고 확실한 링크가 없기 때문에 우수한 프라이버시를 나타냅니다.
