## ธุรกรรม Stonewall x2 คืออะไร?



Stonewall x2 เป็นรูปแบบเฉพาะของธุรกรรม Bitcoin ที่ออกแบบมาเพื่อเพิ่มความเป็นส่วนตัวของผู้ใช้เมื่อใช้จ่าย โดยการร่วมมือกับบุคคลที่สามที่ไม่เกี่ยวข้องกับการใช้จ่าย วิธีนี้จำลองการทำ mini-coinjoin ระหว่างผู้เข้าร่วมสองคน ในขณะที่ทำการชำระเงินให้กับบุคคลที่สาม ธุรกรรม Stonewall x2 สามารถใช้ได้บนแอปพลิเคชัน Ashigaru ซึ่งเป็น fork จาก Samourai Wallet (ทีมที่อยู่เบื้องหลังการสร้างธุรกรรมประเภทนี้)



https://planb.academy/tutorials/wallet/mobile/ashigaru-9f903b55-2e55-4b06-9627-80f8e178158f

วิธีการทำงานค่อนข้างง่าย: คุณใช้ UTXO ที่คุณมีอยู่เพื่อทำการชำระเงิน และขอความช่วยเหลือจากบุคคลที่สามที่มี UTXO ของตนเองเข้าร่วมด้วย การทำธุรกรรมจะสิ้นสุดด้วยผลลัพธ์สี่รายการ: สองรายการในจำนวนที่เท่ากัน หนึ่งรายการส่งไปยังที่อยู่ของผู้รับเงิน อีกหนึ่งรายการส่งไปยังที่อยู่ที่เป็นของผู้ร่วมมือ UTXO ที่สามจะถูกส่งกลับไปยังที่อยู่อื่นที่เป็นของผู้ร่วมมือ ทำให้เขาสามารถกู้คืนจำนวนเงินเริ่มต้นได้ (เป็นการกระทำที่เป็นกลางสำหรับเขา ยกเว้นค่าใช้จ่ายของ mining) และ UTXO สุดท้ายจะถูกส่งกลับไปยังที่อยู่ที่เป็นของเรา ซึ่งถือเป็นการแลกเปลี่ยนการชำระเงิน



บทบาทที่แตกต่างกันสามบทบาทถูกกำหนดไว้ในธุรกรรม Stonewall x2:




- ผู้ออกหลักทรัพย์, ผู้ที่ทำการชำระเงินจริง ;
- ผู้ร่วมมือซึ่งทำให้บิตคอยน์พร้อมใช้งานเพื่อปรับปรุงความไม่เปิดเผยตัวตนของการทำธุรกรรม ในขณะที่กู้คืนเงินทุนของเขาเต็มจำนวนในตอนท้าย (การกระทำที่เป็นกลางสำหรับเขา โมดูลค่าใช้จ่าย mining);
- ผู้รับซึ่งอาจไม่ทราบลักษณะเฉพาะของธุรกรรมและเพียงแค่คาดหวังการชำระเงินจากผู้ส่ง



ลองมาดูตัวอย่างกัน Alice อยู่ที่ร้านเบเกอรี่เพื่อซื้อบาแกตต์ ซึ่งมีราคา `4,000 sats` เธอต้องการจ่ายด้วยบิตคอยน์ โดยยังคงรักษาความลับบางอย่างเกี่ยวกับการชำระเงินของเธอ ดังนั้นเธอจึงเรียกเพื่อนของเธอ Bob มาช่วยเธอในเรื่องนี้



![image](assets/fr/01.webp)



ในการวิเคราะห์ธุรกรรมนี้ เราจะเห็นว่าช่างทำขนมปังได้รับ `4,000 sats` เป็นการชำระเงินสำหรับบาแกตต์จริง ๆ Alice ใช้ `10,000 sats` เป็นอินพุตและกู้คืน `6,000 sats` เป็นเอาต์พุต ทำให้มีดุลสุทธิ `-4,000 sats` ซึ่งสอดคล้องกับราคาของบาแกตต์ สำหรับ Bob นั้น ได้จัดหา `15,000 sats` เป็นอินพุตและได้รับเอาต์พุตสองรายการ: หนึ่งคือ `4,000 sats` และอีกหนึ่งคือ `11,000 sats` ทำให้มีดุล `0`



ในตัวอย่างนี้ ฉันได้ละเว้นค่าธรรมเนียม mining โดยเจตนาเพื่อให้ง่ายต่อการเข้าใจ ในความเป็นจริง ค่าธรรมเนียมการทำธุรกรรมจะถูกแบ่งเท่า ๆ กันระหว่างผู้ออกการชำระเงินและผู้ร่วมมือ



## ความแตกต่างระหว่าง Stonewall และ Stonewall x2 คืออะไร?



ธุรกรรม StonewallX2 ทำงานในลักษณะเดียวกับธุรกรรม Stonewall ทุกประการ ยกเว้นว่าแบบแรกเป็นการทำงานร่วมกัน ในขณะที่แบบหลังไม่ใช่ ดังที่เราได้เห็น ธุรกรรม Stonewall x2 เกี่ยวข้องกับการมีส่วนร่วมของบุคคลที่สาม ซึ่งอยู่นอกเหนือการชำระเงิน และจะทำให้บิตคอยน์ของเขาหรือเธอพร้อมใช้งานเพื่อเพิ่มความเป็นส่วนตัวของธุรกรรม ในธุรกรรม Stonewall แบบคลาสสิก บทบาทของผู้ร่วมมือจะถูกทำโดยผู้ส่ง



กลับไปที่ตัวอย่างของ Alice ที่ร้านเบเกอรี่กันเถอะ ถ้าเธอไม่สามารถหาคนอย่าง Bob มาร่วมเดินทางช้อปปิ้งกับเธอได้ เธอก็สามารถทำ Stonewall ด้วยตัวเองได้ ด้วยวิธีนั้น UTXOs สองรายการที่เข้ามาจะเป็นของเธอ และเธอจะได้รับ 3 รายการเมื่อออกไป



![image](assets/fr/02.webp)




จากมุมมองของคนนอก การทำธุรกรรมจะยังคงเหมือนเดิม



![image](assets/fr/05.webp)



ตรรกะควรเป็นดังนี้เมื่อคุณต้องการใช้เครื่องมือการใช้จ่าย Ashigaru:




- หากร้านค้าไม่รองรับ Payjoin Stowaway คุณสามารถทำธุรกรรมร่วมกับบุคคลอื่นนอกเหนือจากการชำระเงินได้ด้วย Stonewall x2 ;
- หากคุณไม่สามารถหาผู้ใดทำธุรกรรม Stonewall x2 ได้ คุณสามารถทำธุรกรรมแบบ Stonewall เท่านั้น ซึ่งจะเลียนแบบพฤติกรรมของธุรกรรม Stonewall x2

## จุดประสงค์ของการทำธุรกรรม Stonewall x2 คืออะไร?



โครงสร้าง Stonewall x2 เพิ่มปริมาณเอนโทรปีอย่างมหาศาลให้กับธุรกรรม ทำให้การวิเคราะห์เชนสับสน เมื่อมองจากภายนอก ธุรกรรมดังกล่าวอาจถูกตีความว่าเป็น Coinjoin ขนาดเล็กระหว่างสองคน แต่ในความเป็นจริง มันคือการชำระเงิน วิธีนี้จึงสร้างความไม่แน่นอนในการวิเคราะห์เชน หรือแม้กระทั่งนำไปสู่เบาะแสที่ผิดพลาด



ลองดูตัวอย่างของ Alice, Bob และ Boulanger การทำธุรกรรมบนบล็อกเชนจะมีลักษณะดังนี้:



![image](assets/fr/03.webp)



ผู้สังเกตการณ์ภายนอกที่พึ่งพาวิธีการวิเคราะห์เชนทั่วไปอาจสรุปผิดว่า "*Alice และ Bob ได้ทำ coinjoin ขนาดเล็ก โดยมี UTXO หนึ่งรายการเข้าและ UTXO สองรายการออกแต่ละรายการ*".



![image](assets/fr/04.webp)



การตีความนี้ไม่ถูกต้องเพราะว่า ตามที่คุณทราบ UTXO ถูกส่งไปยัง Boulanger, Alice มีเพียงหนึ่งเอาท์พุตแลกเปลี่ยน และ Bob มีสองเอาท์พุตแลกเปลี่ยน



![image](assets/fr/01.webp)



แม้ว่าผู้สังเกตการณ์ภายนอกจะสามารถระบุรูปแบบของการทำธุรกรรม Stonewall x2 ได้ แต่เขาก็จะไม่มีข้อมูลทั้งหมด เขาจะไม่สามารถระบุได้ว่า UTXO สองรายการที่มีจำนวนเท่ากันนั้นรายการใดที่สอดคล้องกับการชำระเงิน นอกจากนี้เขายังจะไม่สามารถระบุได้ว่า Alice หรือ Bob เป็นผู้ทำการชำระเงิน สุดท้ายเขาจะไม่สามารถระบุได้ว่า UTXO สองรายการที่นำเข้านั้นมาจากสองคนที่แตกต่างกันหรือไม่ หรือว่าพวกเขาเป็นของบุคคลเดียวที่ได้รวมเข้าด้วยกัน ข้อนี้เกิดจากข้อเท็จจริงที่ว่าการทำธุรกรรม Stonewall แบบคลาสสิกที่กล่าวถึงข้างต้นนั้นมีรูปแบบเดียวกันกับการทำธุรกรรม Stonewall x2 เมื่อมองจากภายนอกและไม่มีข้อมูลบริบทเพิ่มเติม จะเป็นไปไม่ได้ที่จะจำแนกการทำธุรกรรม Stonewall จากการทำธุรกรรม Stonewall x2 การทำธุรกรรมแบบแรกไม่ใช่การทำธุรกรรมแบบร่วมมือกัน ในขณะที่การทำธุรกรรมแบบหลังเป็นการทำธุรกรรมแบบร่วมมือกัน ซึ่งเพิ่มความสงสัยให้กับการใช้จ่ายมากยิ่งขึ้น

## ฉันจะสร้างการเชื่อมต่อระหว่าง Paynyms ได้อย่างไร?



เช่นเดียวกับธุรกรรมร่วมมืออื่น ๆ บน Ashigaru (*Cahoots*), Stonewall x2 เกี่ยวข้องกับการแลกเปลี่ยนธุรกรรมที่ลงนามบางส่วนระหว่างผู้ส่งและผู้ร่วมมือ การแลกเปลี่ยนนี้สามารถดำเนินการได้ด้วยตนเอง หากคุณอยู่กับผู้ร่วมมือของคุณ หรือโดยอัตโนมัติด้วยการใช้โปรโตคอลการสื่อสาร Soroban



หากคุณเลือกตัวเลือกที่สอง คุณจะต้องสร้างการเชื่อมต่อระหว่าง Paynyms ก่อนที่คุณจะสามารถทำ Stonewall x2 ได้ ในการทำเช่นนี้ Paynym ของคุณต้อง "*ติดตาม*" Paynym ของผู้ร่วมงานของคุณ และในทางกลับกัน หากต้องการทราบวิธีการทำเช่นนี้ คุณสามารถติดตามจุดเริ่มต้นของบทแนะนำอื่นนี้:

## ฉันจะทำธุรกรรม Stonewall x2 บน Ashigaru ได้อย่างไร?



หากต้องการดำเนินการธุรกรรม Stonewall x2 ให้คลิกที่ภาพ Paynym ของคุณที่มุมซ้ายบนของหน้าจอ จากนั้นเปิดเมนู `Collaborate` บุคคลที่เข้าร่วมธุรกรรมกับคุณต้องทำเช่นเดียวกัน ยกเว้นว่าคุณกำลังแลกเปลี่ยนรหัส QR ด้วยตนเอง



![Image](assets/fr/06.webp)



คุณมีสองตัวเลือก: เลือก `Initiate` หากคุณเป็นผู้ส่งการชำระเงิน หรือเลือก `Participate` หากคุณเป็นบุคคลที่ร่วมมือในธุรกรรมแต่ไม่ได้เป็นผู้ชำระเงินหรือผู้รับจริง



![Image](assets/fr/07.webp)



หากคุณมีบทบาทเป็นผู้ร่วมมือ ขั้นตอนจะง่ายมาก สำหรับการทำงานร่วมกันระยะไกลผ่านเครือข่าย Soroban ให้คลิกที่ `Participate` เลือกบัญชีที่คุณต้องการใช้ จากนั้นกด `LISTEN FOR CAHOOTS REQUESTS` เพื่อรอคำขอที่ส่งโดยผู้ชำระเงิน



![Image](assets/fr/08.webp)



ในทางกลับกัน สำหรับการทำงานร่วมกันแบบตัวต่อตัวผ่านการสแกน QR code ให้ไปที่หน้าแรกของ wallet ของคุณ กดไอคอน QR code ที่ด้านบนของหน้าจอ จากนั้นสแกน QR code ที่ผู้ชำระเงินให้มาเพื่อเริ่มการทำธุรกรรม



![Image](assets/fr/09.webp)



หากคุณอยู่ในบทบาทผู้ชำระเงิน เช่น ผู้ที่เริ่มต้นการทำธุรกรรม ให้ไปที่เมนู `Collaborate` จากนั้นเลือก `Initiate`.



![Image](assets/fr/10.webp)



สำหรับประเภทการทำธุรกรรม เนื่องจากเราต้องการทำ Stonewall x2 ให้เลือกตัวเลือกนี้



![Image](assets/fr/11.webp)



จากนั้นคุณสามารถเลือกได้ระหว่างการทำงานร่วมกันออนไลน์ (*Cahoots* ผ่าน *Soroban*) หรือการทำงานร่วมกันแบบพบหน้ากัน โดยใช้การแลกเปลี่ยน QR code



![Image](assets/fr/12.webp)



### Cahoots ออนไลน์



หากคุณเลือกตัวเลือก `ออนไลน์` ให้เลือกผู้ร่วมงานของคุณจาก Paynyms ที่คุณกำลังติดตามอยู่



![Image](assets/fr/13.webp)



คลิกที่ `ตั้งค่าธุรกรรม` จากนั้นเลือกบัญชีที่คุณต้องการใช้ในการใช้จ่าย



![Image](assets/fr/14.webp)



ในหน้าถัดไป ให้ป้อนรายละเอียดการทำธุรกรรม: ที่อยู่ของผู้รับเงินที่แท้จริง จำนวนเงินที่จะส่ง และอัตราค่าธรรมเนียม จากนั้นคลิกที่ `ตรวจสอบการตั้งค่าธุรกรรม`



![Image](assets/fr/15.webp)



ตรวจสอบข้อมูลอย่างรอบคอบ ตรวจสอบให้แน่ใจว่าผู้ร่วมงานของคุณกำลังฟังคำขอ *Cahoots* จากนั้นคลิกที่ปุ่มสีเขียว `BEGIN TRANSACTION` เพื่อเริ่มการแลกเปลี่ยน PSBTs ผ่าน Soroban.



![Image](assets/fr/16.webp)



รอจนกว่าผู้เข้าร่วมทั้งสองจะลงนามในธุรกรรม จากนั้นจึงเผยแพร่บนเครือข่าย Bitcoin



![Image](assets/fr/17.webp)
