Payjoin เป็นโครงสร้างธุรกรรม Bitcoin ที่ออกแบบมาเพื่อเพิ่มความเป็นส่วนตัวของผู้ใช้โดยการมีส่วนร่วมโดยตรงกับผู้รับชำระเงิน มีการใช้งานหลายรูปแบบเพื่ออำนวยความสะดวกในการใช้งานและทำให้กระบวนการเป็นอัตโนมัติ สิ่งที่รู้จักกันดีที่สุดคือ Stowaway ซึ่งพัฒนาโดยทีม Samurai Wallet ในตอนแรกและตอนนี้ได้ถูกรวมเข้ากับ fork Ashigaru ของพวกเขาแล้ว



## Stowaway ทำงานอย่างไร?



ตามที่กล่าวไว้ก่อนหน้านี้ Ashigaru ผสานรวมเครื่องมือ PayJoin ที่เรียกว่า `Stowaway` ซึ่งมีให้ในแอป Ashigaru บน Android สำหรับการดำเนินการ Payjoin ผู้รับ (ซึ่งรับบทบาทเป็นผู้ร่วมมือด้วย) จะต้องใช้ซอฟต์แวร์ที่เข้ากันได้กับ Stowaway นั่นคือเฉพาะ Ashigaru ในขณะนี้



Stowaway อ้างอิงจากหมวดหมู่ของธุรกรรมที่ Samurai เรียกว่า "Cahoots" Cahoot คือธุรกรรมที่ทำร่วมกันระหว่างผู้ใช้หลายคน ซึ่งเกี่ยวข้องกับการแลกเปลี่ยนข้อมูลนอกบล็อกเชน Bitcoin ปัจจุบัน Ashigaru มีเครื่องมือ Cahoots สองอย่าง: Stowaway (Payjoins) และ StonewallX2.



https://planb.academy/tutorials/privacy/on-chain/ashigaru-stonewall-x2-05120280-f6f9-4e14-9fb8-c9e603f73e5b

ธุรกรรม Cahoots ต้องการการแลกเปลี่ยนธุรกรรมที่ลงนามบางส่วนระหว่างผู้ใช้ กระบวนการนี้อาจยาวนานและน่าเบื่อ โดยเฉพาะเมื่อดำเนินการจากระยะไกล อย่างไรก็ตาม ยังสามารถทำได้ด้วยตนเอง หากผู้ร่วมมืออยู่ในสถานที่เดียวกัน ในทางปฏิบัติ นี่หมายถึงการสแกน QR โค้ดห้ารายการติดต่อกัน ซึ่งแลกเปลี่ยนระหว่างผู้เข้าร่วมสองคน



เมื่ออยู่ในระยะไกล วิธีนี้จะซับซ้อนเกินไป เพื่อแก้ไขปัญหานี้ Samourai ได้พัฒนาโปรโตคอลการสื่อสารเข้ารหัสที่ใช้ Tor ชื่อว่า "*Soroban*" ด้วย Soroban การแลกเปลี่ยนที่จำเป็นสำหรับ Payjoin จะถูกทำให้เป็นอัตโนมัติและเกิดขึ้นในพื้นหลัง



การสื่อสารที่เข้ารหัสเหล่านี้ต้องการการเชื่อมต่อและการยืนยันตัวตนระหว่างผู้เข้าร่วม Cahoot นี่คือเหตุผลที่ Soroban พึ่งพา Paynyms ของผู้ใช้ หากคุณยังไม่คุ้นเคยกับวิธีการทำงานของ Paynyms ลองดูที่บทแนะนำเฉพาะนี้เพื่อเรียนรู้เพิ่มเติม:



https://planb.academy/tutorials/privacy/on-chain/paynym-bip47-a492a70b-50eb-4f95-a766-bae2c5535093

โดยสรุปแล้ว Paynym คือรหัสประจำตัวที่ไม่ซ้ำกันที่เชื่อมโยงกับ wallet ของคุณ ซึ่งช่วยให้คุณสามารถเปิดใช้งานฟังก์ชันต่างๆ รวมถึงการแลกเปลี่ยนข้อมูลที่เข้ารหัส มันมีรูปแบบเป็นรหัสประจำตัวที่มาพร้อมกับภาพประกอบ นี่คือตัวอย่างที่ฉันใช้บน Testnet:

**สรุป:**





- payjoin" = โครงสร้างธุรกรรมแบบร่วมมือเฉพาะ;





- `Stowaway` = การใช้งาน Payjoin ที่มีอยู่บน Ashigaru ;





- `Cahoots` = ชื่อที่ Samourai ตั้งให้กับธุรกรรมแบบร่วมมือทุกประเภทของพวกเขา โดยเฉพาะอย่างยิ่ง Payjoin `Stowaway` ซึ่งถูกนำมาใช้ในวันนี้บน Ashigaru ;





- soroban = โปรโตคอลการสื่อสารเข้ารหัสที่จัดตั้งขึ้นบน Tor ซึ่งอนุญาตให้มีการทำงานร่วมกันกับผู้ใช้รายอื่นในธุรกรรม `Cahoots`;





- paynym" = ตัวระบุ wallet ที่ไม่ซ้ำกันซึ่งใช้ในการสื่อสารกับผู้ใช้อื่นบน "Soroban" เพื่อดำเนินการทำธุรกรรม "Chahoots"

## ฉันจะทำ Payjoin บน Ashigaru ได้อย่างไร?



ในการดำเนินการธุรกรรม Stowaway ให้คลิกที่ภาพ Paynym ของคุณที่มุมซ้ายบนของหน้าจอ จากนั้นเปิดเมนู `Collaborate` บุคคลที่เข้าร่วมในการทำธุรกรรมกับคุณต้องทำเช่นเดียวกัน ยกเว้นว่าคุณกำลังแลกเปลี่ยนรหัส QR ด้วยตนเอง



![Image](assets/fr/02.webp)



คุณมีสองตัวเลือก: เลือก `Initiate` หากคุณเป็นผู้ส่งการชำระเงิน หรือ `Participate` หากคุณเป็นผู้รับเงินของ payjoin นี้



![Image](assets/fr/03.webp)



หากคุณเป็นผู้รับ ขั้นตอนนั้นง่ายมาก สำหรับการทำงานร่วมกันระยะไกลผ่านเครือข่าย Soroban ให้คลิกที่ `Participate` เลือกบัญชีที่คุณต้องการใช้ จากนั้นกด `LISTEN FOR CAHOOTS REQUESTS` เพื่อรอคำขอที่ส่งโดยผู้ชำระเงิน



![Image](assets/fr/04.webp)



ในทางกลับกัน สำหรับการทำงานร่วมกันแบบตัวต่อตัวผ่านการสแกน QR code ให้ไปที่หน้าแรกของ wallet ของคุณ กดไอคอน QR code ที่ด้านบนของหน้าจอ จากนั้นสแกน QR code ที่ผู้ชำระเงินเริ่มต้นธุรกรรมให้มา



![Image](assets/fr/05.webp)



หากคุณอยู่ในบทบาทผู้ชำระเงิน เช่น ผู้ที่เริ่มการทำธุรกรรม ให้ไปที่เมนู `Collaborate` จากนั้นเลือก `Initiate`.



![Image](assets/fr/06.webp)



สำหรับประเภทการทำธุรกรรม เนื่องจากเราต้องการทำ Payjoin Stowaway ให้เลือกตัวเลือกนี้



![Image](assets/fr/07.webp)



จากนั้นคุณสามารถเลือกได้ระหว่างการทำงานร่วมกันออนไลน์ (*Cahoots* ผ่าน *Soroban*) หรือการทำงานร่วมกันแบบพบหน้ากัน โดยใช้การแลกเปลี่ยน QR code



![Image](assets/fr/08.webp)



### Cahoots ออนไลน์



หากคุณเลือกตัวเลือก `ออนไลน์` ให้เลือกผู้รับจาก Paynyms ที่คุณกำลังติดตามอยู่



![Image](assets/fr/09.webp)



คลิกที่ `Set up transaction` จากนั้นเลือกบัญชีที่คุณต้องการใช้ในการใช้จ่าย



![Image](assets/fr/10.webp)



ในหน้าถัดไป ให้ป้อนรายละเอียดการทำธุรกรรม: จำนวนเงินที่จะส่งให้ผู้รับและอัตราค่าธรรมเนียม ไม่จำเป็นต้องป้อนที่อยู่ผู้รับ เนื่องจากผู้รับจะส่งต่อเองระหว่างการแลกเปลี่ยน PSBT



จากนั้นคลิกที่ `Review transaction setup`.



![Image](assets/fr/11.webp)



ตรวจสอบข้อมูลอย่างรอบคอบ, ให้แน่ใจว่าผู้ร่วมงานของคุณกำลังฟังคำขอจาก *Cahoots*, จากนั้นคลิกที่ปุ่มสีเขียว `BEGIN TRANSACTION` เพื่อเริ่มการแลกเปลี่ยน PSBTs ผ่าน Soroban.



![Image](assets/fr/12.webp)



รอจนกว่าผู้เข้าร่วมทั้งสองจะลงนามในธุรกรรม จากนั้นจึงเผยแพร่บนเครือข่าย Bitcoin



![Image](assets/fr/13.webp)



### การสนทนาแบบเผชิญหน้า



หากคุณต้องการดำเนินการแลกเปลี่ยนด้วยตนเอง ให้เลือกประเภทธุรกรรม `STONEWALL X2` จากนั้นเลือกตัวเลือก `In Person / Manual`



![Image](assets/fr/14.webp)



คลิกที่ `Set up transaction` จากนั้นเลือกบัญชีที่คุณต้องการใช้ในการใช้จ่าย



![Image](assets/fr/15.webp)



ในหน้าถัดไป ให้ป้อนรายละเอียดการทำธุรกรรม: จำนวนเงินที่จะส่งให้ผู้รับและอัตราค่าธรรมเนียม ไม่จำเป็นต้องป้อนที่อยู่ผู้รับ เนื่องจากผู้รับจะส่งต่อเองระหว่างการแลกเปลี่ยน PSBT



จากนั้นคลิกที่ `Review transaction setup`.



![Image](assets/fr/16.webp)



ตรวจสอบรายละเอียด จากนั้นกดปุ่ม `BEGIN TRANSACTION` สีเขียวเพื่อเริ่มการแลกเปลี่ยน PSBTs ผ่านการสแกน QR code



![Image](assets/fr/17.webp)



การแลกเปลี่ยนจะทำโดยการสลับการสแกนกับผู้ร่วมงาน: คลิกที่ `SHOW QR CODE` เพื่อแสดง QR code ของคุณให้ผู้ร่วมงานของคุณสแกน จากนั้นเขาจะคลิกที่ `SHOW QR CODE` เพื่อแสดงของเขา และคุณจะสแกนด้วย `LAUNCH QR Scanner` ทำซ้ำขั้นตอนนี้จนกว่าการแลกเปลี่ยนทั้งห้าขั้นตอนจะเสร็จสิ้น



![Image](assets/fr/18.webp)



เมื่อการแลกเปลี่ยนทั้งหมดเสร็จสิ้น ตรวจสอบรายละเอียดการทำธุรกรรม จากนั้นปล่อยโดยการลากลูกศรสีเขียวที่ด้านล่างของหน้าจอ



![Image](assets/fr/19.webp)

หากเราวิเคราะห์ธุรกรรมนี้ เราจะเห็น UTXO ของฉันที่ `164,211 sats` ในด้านอินพุต เช่นเดียวกับ UTXO ของ `190,002 sats` ที่เป็นของผู้รับเงินที่แท้จริง ในด้านเอาต์พุต ฉันได้รับการแลกเปลี่ยน UTXO ของ `63,995 sats` ในขณะที่ผู้รับได้รับ UTXO ของ `290,002 sats` เมื่อเปรียบเทียบอินพุตและเอาต์พุต เราจะเห็นว่าผู้รับได้รับ `100,000 sats` ซึ่งสอดคล้องกับจำนวนเงินที่ฉันจ่ายจริง และฉันได้สูญเสีย `100,000 sats` ซึ่งฉันได้เพิ่มค่าใช้จ่าย mining เข้าไปด้วย



เห็นได้ชัดว่าฉันสามารถอธิบายโครงสร้างนี้ได้เพราะฉันสร้างธุรกรรมนี้ด้วยตัวเอง แต่สำหรับผู้สังเกตการณ์ภายนอก โดยทั่วไปแล้วจะเป็นไปไม่ได้ที่จะระบุว่า UTXOs ใดเป็นของผู้เข้าร่วมคนใด ไม่ว่าจะเป็นในส่วนของอินพุตหรือเอาต์พุต
