ความโปร่งใสของบล็อกเชน Bitcoin ทำให้สามารถติดตามประวัติการทำธุรกรรมได้ CoinJoins ทำลายลิงก์เหล่านี้โดยการผสมธุรกรรมหลายรายการพร้อมกัน ฟื้นฟูระดับความลับที่เทียบเท่ากับเงินสด



อย่างไรก็ตาม โซลูชันแบบดั้งเดิมต้องพึ่งพาผู้ประสานงานกลางซึ่งเป็นจุดล้มเหลวเพียงจุดเดียว Wasabi และ Samourai หยุดดำเนินการในปี 2024 ภายใต้แรงกดดันด้านกฎระเบียบ



**Joinstr** กำจัดจุดอ่อนนี้โดยใช้เครือข่าย Nostr แบบกระจายอำนาจสำหรับการประสานงาน แอปพลิเคชันโอเพ่นซอร์สนี้ช่วยให้เกิด CoinJoins ที่มีอธิปไตยอย่างแท้จริง ซึ่งไม่มีหน่วยงานกลางสามารถเซ็นเซอร์ ตรวจสอบ หรือขัดจังหวะการบริการได้



## Joinstr คืออะไร?



Joinstr เป็นเครื่องมือโอเพ่นซอร์สที่ปฏิวัติวิธีการ CoinJoins โดยใช้โปรโตคอล Nostr เป็นโครงสร้างพื้นฐานในการประสานงาน แตกต่างจากโซลูชันแบบรวมศูนย์ที่ต้องการเซิร์ฟเวอร์เฉพาะ Joinstr อาศัยเครือข่ายกระจายของรีเลย์ Nostr เพื่อให้ผู้เข้าร่วมสามารถประสานงานกันโดยตรงระหว่างเพื่อนร่วมงาน



**สถาปัตยกรรมแบบกระจายศูนย์** : เครือข่าย Nostr แทนที่ผู้ประสานงานกลาง ผู้เข้าร่วมสร้างหรือเข้าร่วม "พูล" โดยการโพสต์ประกาศที่เข้ารหัสผ่านรีเลย์ Nostr ข้อมูลนี้ (จำนวน ผู้เข้าร่วม ที่อยู่) ยังคงไม่สามารถเข้าใจได้สำหรับรีเลย์ เพื่อให้มั่นใจว่าไม่มีหน่วยงานกลางใดสามารถตรวจสอบ เซ็นเซอร์ หรือกรอง CoinJoins ได้



**SIGHASH_ALL|ANYONECANPAY mechanism**: Joinstr ใช้ประโยชน์จาก Bitcoin signature flag นี้ โดยอนุญาตให้ผู้เข้าร่วมแต่ละคนเซ็นเฉพาะอินพุตของตนเองในขณะที่ตรวจสอบความถูกต้องของเอาต์พุตทั้งหมด ผู้ใช้แต่ละคนสร้าง PSBT ของตนเองในเครื่อง จากนั้นแจกจ่ายผ่าน Nostr ผู้ใช้แต่ละคนสร้าง PSBT ของตนเองในเครื่อง เซ็นชื่อ จากนั้นแจกจ่ายผ่าน Nostr บิตคอยน์ของคุณยังคงอยู่ภายใต้การควบคุมของคุณแต่เพียงผู้เดียวจนกว่าจะมีการเซ็นชื่อครั้งสุดท้าย



**ปรัชญา**: Joinstr ปฏิบัติตามจริยธรรมของ Bitcoin cypherpunk โดยมีเป้าหมายสามประการ: **การต่อต้านการเซ็นเซอร์** (ไม่มีอำนาจใดสามารถหยุดบริการได้), **การไม่ถือครองโดยสมบูรณ์** (คุณเก็บกุญแจส่วนตัวของคุณเอง), และ **การปฏิบัติอย่างเท่าเทียม** (ไม่มี UTXO ใดที่ถูกเลือกปฏิบัติ).



### คุณสมบัติหลัก



**Flexible pools**: ไม่เหมือนกับสกุลเงินที่กำหนดไว้ล่วงหน้า ผู้ใช้สามารถสร้างพูลด้วยจำนวนที่ต้องการและจำนวนผู้เข้าร่วมเป้าหมายได้ ซึ่งช่วยเพิ่มประสิทธิภาพการใช้ UTXO โดยไม่ต้องแบ่งแยกอย่างไม่จำเป็น



**ค่าธรรมเนียมโปร่งใส**: ไม่มีค่าธรรมเนียมการประสานงาน มีเพียงค่าธรรมเนียมการทำธุรกรรม Bitcoin ที่จะแบ่งกันระหว่างผู้เข้าร่วม (ประมาณไม่กี่พันซาโตชิต่อคน)



**ชั่วคราว**: ไม่มีการเก็บข้อมูลผู้ใช้ ข้อมูลจะถูกส่งผ่านข้อความ Nostr ที่เข้ารหัสและชั่วคราว ซึ่งจะถูกลืมทันทีหลังจากการทำธุรกรรม

### ข้อกำหนดของเครือข่าย



#### ค้นหาที่อยู่ IP ภายในเครื่องของคุณ



โทรศัพท์ Android ของคุณต้องสามารถเข้าถึงโหนด Bitcoin ของคุณบนเครือข่ายท้องถิ่นได้ ค้นหาที่อยู่ IP ของคอมพิวเตอร์ของคุณ:



**บน macOS** :



```bash
ifconfig | grep "inet " | grep -v "127.0.0.1" | awk '{print $2}' | head -n 1
```



ทางเลือกที่ง่าย:



```bash
ipconfig getifaddr en0
# or for WiFi: ipconfig getifaddr en1
```



**บน Linux** :



```bash
hostname -I | awk '{print $1}'
```



**บน Windows** :



```cmd
ipconfig
```



ค้นหาที่อยู่ IPv4 (รูปแบบ `192.168.x.x` หรือ `10.0.x.x`)



### การกำหนดค่า RPC



#### แก้ไข bitcoin.conf



![LOCALISATION BITCOIN.CONF](assets/fr/01.webp)



ค้นหาไฟล์ `bitcoin.conf` ของคุณ:




- Linux**: `~/.bitcoin/bitcoin.conf
- macOS**: `~/Library/Application Support/Bitcoin/bitcoin.conf
- Windows**: `%APPDATA%\Bitcoin\bitcoin.conf`



![CONTENU BITCOIN.CONF](assets/fr/02.webp)



เปิดไฟล์ด้วยโปรแกรมแก้ไขข้อความที่คุณชื่นชอบและเพิ่มการกำหนดค่านี้เพื่อเปิดใช้งานเซิร์ฟเวอร์ RPC



**การกำหนดค่าที่แนะนำสำหรับการเริ่มต้น (บุ๊กมาร์ก)** :



```conf
# Enable signet (test network)
signet=1
prune=550

# Enable the RPC server
server=1
rpcbind=0.0.0.0

# Allow connections from your local network
# Adjust according to your network (192.168.x.0/24 or 10.0.x.0/24)
rpcallowip=192.168.1.0/24

# RPC Credentials (CHANGE THESE VALUES!)
rpcuser=your_username
rpcpassword=your_strong_password

# Specific signet configuration
[signet]
rpcport=38332
```



การกำหนดค่า **mainnet** (สำหรับการใช้งานในผลิต):



```conf
# RPC Server
server=1
rpcbind=0.0.0.0
rpcallowip=192.168.1.0/24

# RPC Credentials
rpcuser=your_username
rpcpassword=your_strong_password

# Mainnet Port
rpcport=8332
```



**Important** : สำคัญ




- Signet ได้รับการแนะนำอย่างสูง** สำหรับการทดสอบครั้งแรกของคุณ: แอปพลิเคชันยังอยู่ในระหว่างการพัฒนา (เบต้า) และอาจยังมีข้อบกพร่องอยู่ Signet ให้คุณทดสอบได้ฟรี โดยไม่ต้องเสี่ยงกับเงินทุนจริง
- แทนที่ `192.168.1.0/24` ด้วยซับเน็ตเครือข่ายของคุณ (เช่น หาก IP ของคุณคือ `192.168.68.57` ให้ใช้ `192.168.68.0/24`)



**Security**: สร้างรหัสผ่านที่แข็งแกร่ง :



```bash
openssl rand -base64 32
```

![CONFIGURATION SETTINGS](assets/fr/13.webp)



กำหนดค่าการตั้งค่าในแท็บ "การตั้งค่า":



**1. Nostr Relay**: ที่อยู่รีเลย์ Nostr สำหรับการประสานงานพูล




- ตัวอย่าง: `wss://relay.damus.io`
- รีเลย์ที่แนะนำอื่นๆ: `wss://nos.lol`, `wss://relay.nostr.band`, `wss://nostr.fmt.wiz.biz`
- ⚠️ **สำคัญ**: ผู้เข้าร่วมทุกคนต้องใช้ **Nostr relay เดียวกัน** เพื่อดูและเข้าร่วมกลุ่มเดียวกัน หากคุณใช้ relay ที่แตกต่างกัน คุณจะไม่เห็นกลุ่มที่สร้างบน relay อื่น



**2. Node URL**: ที่อยู่ IP ของโหนด Bitcoin ของคุณ (ไม่รวมพอร์ต)




- รูปแบบ: `http://VOTRE_IP_LOCALE`
- ตัวอย่าง: `http://192.168.68.57`



**3. RPC Username** : ชื่อผู้ใช้ที่กำหนดใน `rpcuser=` บน bitcoin.conf ของคุณ




- ตัวอย่าง: `satoshi



**4. RPC Password** : รหัสผ่านที่ตั้งไว้ใน `rpcpassword=` บน bitcoin.conf ของคุณ



**5. RPC Port** : พอร์ต RPC ขึ้นอยู่กับเครือข่าย




- Mainnet** : `8332`
- บุ๊กมาร์ก**: `38332`



**6. Wallet**: เลือกพอร์ตโฟลิโอ Bitcoin Core ที่มี UTXOs ที่จะผสม




- ตัวอย่าง: `tuto_joinstr_signet`



**7. เกตเวย์ VPN**: เลือกเซิร์ฟเวอร์ VPN ของ Riseup




- (Paris) vpn07-par.riseup.net
- อื่นๆ ที่มี: อัมสเตอร์ดัม, ซีแอตเทิล, เป็นต้น
- ⚠️ **Important**: ผู้เข้าร่วมทั้งหมดในกลุ่มเดียวกันต้องใช้ **VPN Gateway เดียวกัน** เพื่อเข้าร่วมใน CoinJoin ในระหว่างรอบการผสม ผู้เข้าร่วมทั้งหมดต้องปรากฏด้วยที่อยู่ IP ออกเดียวกันเพื่อป้องกันการวิเคราะห์เครือข่ายจากการเชื่อมโยงผู้เข้าร่วม



แอปพลิเคชัน Joinstr **ผสานรวมโดยตรง** กับ Riseup VPN ทำให้การประสานงานระหว่างผู้เข้าร่วมง่ายขึ้น



**สำคัญ** :




- ตรวจสอบให้แน่ใจว่าโทรศัพท์และคอมพิวเตอร์ของคุณอยู่ในเครือข่าย WiFi เดียวกันในพื้นที่เดียวกัน

- คลิกที่ **"บันทึก"** เมื่อคุณตั้งค่าพารามิเตอร์ทั้งหมดแล้ว

คลิกที่ "Create New Pool" ในแท็บ "Pools" ระบุสกุลเงินใน BTC (เช่น 0.002 BTC) และจำนวนผู้เข้าร่วมที่ต้องการ (ขั้นต่ำ 2, แนะนำ 3-5 เพื่อความไม่เปิดเผยตัวตนที่มากขึ้น) จากนั้นแอปพลิเคชันจะรอให้ผู้ใช้คนอื่นเข้าร่วมพูลของคุณ เมื่อถึงจำนวนที่ต้องการแล้ว กระบวนการลงทะเบียนเอาต์พุตจะเริ่มต้นโดยอัตโนมัติ และคุณจะต้องเลือก UTXO ของคุณเพื่อทำการผสมและคลิกที่ "Register"



**⏱️ Important**: สระจะหมดอายุหลังจากไม่มีการใช้งาน **10 นาที** หากจำนวนผู้เข้าร่วมที่ต้องการไม่ครบ สระจะถูกปิดโดยอัตโนมัติ



**ตัวเลือกที่ 2: เข้าร่วมพูลที่มีอยู่**



ในแท็บ "ดูพูลอื่น ๆ" ให้เรียกดูพูลที่มีอยู่ซึ่งสร้างโดยผู้ใช้รายอื่น เลือกพูลที่ตรงกับจำนวนของคุณและเข้าร่วม ตรวจสอบให้แน่ใจว่าคุณได้กำหนดค่ารีเลย์ Nostr และเกตเวย์ VPN เดียวกันกับผู้เข้าร่วมคนอื่น ๆ (ดูส่วนการกำหนดค่า)



**กฎการเลือก UTXO**: UTXO ของคุณต้องสูงกว่าค่าของสระเล็กน้อย (ระหว่าง +500 และ +5000 ส่วนเกิน sats)



**ตัวอย่าง**: สำหรับพูล 0.002 BTC (200,000 sats), ใช้ UTXO ระหว่าง 200,500 และ 205,000 sats.



![PROCESSUS COINJOIN](assets/fr/11.webp)



กระบวนการนี้ **เป็นอัตโนมัติอย่างเต็มที่**: เมื่อข้อมูลของคุณได้รับการลงทะเบียนแล้ว แอปพลิเคชันจะรอให้ผู้เข้าร่วมทุกคนลงทะเบียนข้อมูลของตน เมื่อข้อมูลทั้งหมดได้รับการลงทะเบียนแล้ว PSBT จะถูกสร้างขึ้น ลงนามโดยอัตโนมัติด้วยกุญแจของคุณ จากนั้นรวมกับลายเซ็นของผู้เข้าร่วมคนอื่น ๆ สุดท้าย การทำธุรกรรมที่สมบูรณ์จะถูกกระจายบนเครือข่าย Bitcoin คุณจะได้รับ TXID (ตัวระบุการทำธุรกรรม) เมื่อการกระจายเสร็จสมบูรณ์ ไม่จำเป็นต้องมีการจัดการ PSBT ด้วยตนเองบน Android



### การตรวจสอบ on-chain



![TRANSACTION MEMPOOL](assets/fr/12.webp)



เมื่อการทำธุรกรรมถูกกระจายออกไป คุณจะได้รับ TXID (ตัวระบุธุรกรรม) ดูได้ที่ [mempool.space](https://mempool.space) หรือเบราว์เซอร์ที่คุณชื่นชอบ หากต้องการทดสอบบนบุ๊คมาร์ค ให้ใช้ [mempool.space/signet](https://mempool.space/signet)



คุณสามารถสังเกตได้ :





- N entries**: หนึ่งรายการต่อผู้เข้าร่วม (ในตัวอย่างของเรา, 2 รายการ)
- เอาต์พุตที่เหมือนกัน N**: จำนวนที่แน่นอนของสกุลเงิน (ที่นี่, 2 เอาต์พุตของ 0.00199800 BTC แต่ละอัน)
- ผังงาน**: mempool.space แสดงการผสมของอินพุตและเอาต์พุตอย่างชัดเจน
- คุณสมบัติ** : ธุรกรรมสามารถระบุได้เป็น SegWit, Taproot, RBF, เป็นต้น



เนื่องจากผลลัพธ์หลักทั้งหมดมีจำนวนเท่ากัน จึงเป็น **ไปไม่ได้ที่จะระบุว่าเป็นของใคร** นี่คือหลักการพื้นฐานของ CoinJoin: ความไม่สามารถแยกแยะได้ของผลลัพธ์



**Exemple de transaction signet** : [404a6a58a1682341c1197655fa492fa160bf63fca8c3b29be125e7360dbec44c](https://mempool.space/signet/tx/404a6a58a1682341c1197655fa492fa160bf63fca8c3b29be125e7360dbec44c)



**โปรดทราบ**: หาก UTXO ของคุณมีขนาดใหญ่กว่าสกุลเงินของพูล คุณจะมีเอาต์พุตแลกเปลี่ยนเงินตราต่างประเทศ (ส่วนเกิน) UTXO แลกเปลี่ยนเหล่านี้ยังคงสามารถติดตามได้และต้องจัดการแยกต่างหากจากเอาต์พุตที่ไม่ระบุตัวตนของคุณ อย่ารวมพวกมันกับเอาต์พุตที่ผสมของคุณ



## CoinJoin แพ็คเกจคุณภาพและการไม่เปิดเผยตัวตน



ประสิทธิภาพของ CoinJoin ถูกวัดโดย **anonset**: จำนวนของผลลัพธ์ที่มีค่าเหมือนกันที่ผลิตโดยธุรกรรม ยิ่งจำนวนนี้สูงเท่าใด ก็ยิ่งยากที่จะระบุได้ว่าข้อมูลนำเข้าใดตรงกับผลลัพธ์ใด



Joinstr ปัจจุบันสร้างกลุ่มที่มีผู้เข้าร่วม **2 ถึง 5 คน** โดยเฉลี่ย ตัวเลขเหล่านี้ต่ำกว่าผู้ประสานงานแบบรวมศูนย์แบบดั้งเดิมอย่าง Wasabi (50-100 คน) หรือ Whirlpool (5-10 คน) แต่นั่นคือราคาของการกระจายอำนาจ



💡 **หากต้องการทำความเข้าใจเกี่ยวกับชุดนิรนามและการคำนวณอย่างละเอียด** ดูหลักสูตรเต็มของเรา: [ชุดนิรนาม](https://planb.academy/fr/courses/65c138b0-4161-4958-bbe3-c12916bc959c/les-ensembles-danonymat-be1093dc-1a74-40e5-9545-2b97a7d7d431).

## ข้อดีและข้อจำกัด



### ไฮไลท์



**ความเป็นส่วนตัวที่เพิ่มขึ้น**: การผสมผสานระหว่างการเข้ารหัสการสื่อสาร Nostr, VPN ที่ใช้ร่วมกันระหว่างผู้เข้าร่วม, และการแนะนำให้ใช้ Tor สร้างการป้องกันหลายชั้นที่ยากต่อการหลีกเลี่ยง



**โปร่งใส, ต้นทุนต่ำสุด**: ไม่มีต้นทุนการประสานงาน, มีเพียงต้นทุน mining ที่แบ่งปันอย่างเท่าเทียมระหว่างผู้เข้าร่วม ไม่มีการเรียกเก็บเปอร์เซ็นต์โดยผู้ดำเนินการใด ๆ



### ข้อจำกัดที่ต้องพิจารณา





- สภาพคล่องที่เปลี่ยนแปลงได้**: กลุ่มที่เล็กกว่า สามารถรอให้ผู้เข้าร่วมมารวมตัวกัน
- โครงการที่กำลังดำเนินการ**: แอปพลิเคชันอยู่ในช่วงเบต้า อาจมีข้อบกพร่อง ทดสอบก่อนด้วยจำนวนเล็กน้อยบนบุ๊กมาร์ก
- การโจมตีแบบ Sybil**: ความเป็นไปได้ที่คู่ต่อสู้คนหนึ่งจะควบคุมผู้เข้าร่วมพูลหลายคน



## แนวทางปฏิบัติที่ดีที่สุด



**UTXO isolation**: ห้ามผสม UTXO ที่ผสมแล้วกับที่ยังไม่ผสม ใช้พอร์ตโฟลิโอแยกสำหรับผลลัพธ์ที่ไม่ระบุตัวตนของคุณ



**หลายรอบเป็นสิ่งจำเป็น**: ดำเนินการอย่างน้อย 3 รอบติดต่อกันกับผู้เข้าร่วมที่แตกต่างกัน ปรับเปลี่ยนจำนวนและเวลาเพื่อหลีกเลี่ยงรูปแบบ จัดรอบให้ห่างกันหลายชั่วโมง



**การป้องกันเครือข่าย**: ใช้ Tor อย่างเป็นระบบควบคู่กับ VPN ที่มีอยู่ในตัว สร้างคีย์ Nostr ชั่วคราวสำหรับแต่ละเซสชันที่สำคัญ



**ความก้าวหน้าอย่างระมัดระวัง**: เริ่มบันทึกที่คั่นหน้าด้วยจำนวนเล็กน้อย
