Прозрачность блокчейна Bitcoin позволяет отслеживать историю транзакций. CoinJoins разрывает эти связи, смешивая несколько одновременных транзакций, восстанавливая уровень конфиденциальности, сравнимый с наличными.



Однако традиционные решения опираются на центрального координатора как единую точку отказа. Компании Wasabi и Samourai прекратили свою деятельность в 2024 году под давлением регулирующих органов.



**Joinstr** устраняет эту слабость, используя для координации децентрализованную сеть Nostr. Это приложение с открытым исходным кодом позволяет создавать по-настоящему суверенные CoinJoins, где ни один центральный орган не может цензурировать, контролировать или прерывать работу сервиса.



## Что такое Joinstr?



Joinstr - это инструмент с открытым исходным кодом, который революционизирует подход CoinJoins за счет использования протокола Nostr в качестве инфраструктуры координации. В отличие от централизованных решений, требующих наличия выделенного сервера, Joinstr опирается на распределенную сеть ретрансляторов Nostr, позволяющую участникам координировать действия непосредственно между пирами.



**Децентрализованная архитектура** : Сеть Nostr заменяет центрального координатора. Участники создают или присоединяются к "пулам", публикуя зашифрованные объявления через ретрансляторы Nostr. Эта информация (суммы, участники, адреса) остается неразборчивой для ретрансляторов, что гарантирует, что ни одна центральная организация не сможет контролировать, цензурировать или фильтровать CoinJoins.



*механизм *SIGHASH_ALL|ANYONECANPAY**: Joinstr использует этот флаг подписи Bitcoin, позволяя каждому участнику подписывать только свои входные данные, проверяя при этом все выходные. Каждый пользователь генерирует свой PSBT локально, а затем распространяет его через Nostr. Каждый пользователь генерирует свой PSBT локально, подписывает его, а затем распространяет через Nostr. Ваши биткоины остаются под вашим исключительным контролем до финальной подписи.



**Философия**: Joinstr следует этике киберпанка Bitcoin, стремясь к трем целям: **устойчивость к цензуре** (ни один орган власти не может остановить работу сервиса), **полное отсутствие лишения свободы** (вы храните свои приватные ключи) и **равное отношение** (ни один UTXO не может быть дискриминирован).



### Основные характеристики



**Гибкие пулы**: В отличие от фиксированных номиналов, любой пользователь может создать пул с точной суммой и целевым количеством участников, оптимизируя использование UTXO без искусственного дробления.



**Прозрачные цены**: Никаких координационных сборов. Только плата за транзакции Bitcoin делится поровну между участниками (несколько тысяч сатоши на человека).



**Эфемерность**: Никаких пользовательских данных не сохраняется. Информация передается через зашифрованные эфемерные сообщения Nostr, которые сразу же забываются после транзакции.



### Доступные платформы



В этом руководстве рассматривается приложение **Android**, единственное стабильное и рекомендуемое решение на данный момент. Существует плагин для Electrum, но у него есть проблемы с совместимостью, которые делают его нестабильным. Веб-интерфейс находится в стадии разработки.

### Требования к сети



#### Найдите свой локальный IP-адрес



Ваш телефон Android должен иметь возможность связаться с узлом Bitcoin в локальной сети. Найдите IP-адрес вашего компьютера:



**На macOS** :



```bash
ifconfig | grep "inet " | grep -v "127.0.0.1" | awk '{print $2}' | head -n 1
```



Простая альтернатива:



```bash
ipconfig getifaddr en0
# or for WiFi: ipconfig getifaddr en1
```



**В Linux** :



```bash
hostname -I | awk '{print $1}'
```



**На Windows** :



```cmd
ipconfig
```



Найдите IPv4-адрес (формат `192.168.x.x` или `10.0.x.x`)



### Конфигурация RPC



#### Редактировать bitcoin.conf



![LOCALISATION BITCOIN.CONF](assets/fr/01.webp)



Найдите свой файл `bitcoin.conf`:




- Linux**: `~/.bitcoin/bitcoin.conf
- macOS**: `~/Library/Application Support/Bitcoin/bitcoin.conf
- Windows**: `%APPDATA%\Bitcoin\bitcoin.conf`



![CONTENU BITCOIN.CONF](assets/fr/02.webp)



Откройте файл с помощью вашего любимого текстового редактора и добавьте эту конфигурацию, чтобы активировать сервер RPC.



**Рекомендуемая конфигурация для начала работы (закладка)** :



```conf
# Enable signet (test network)
signet=1
prune=550

# Enable the RPC server
server=1
rpcbind=0.0.0.0

# Allow connections from your local network
# Adjust according to your network (192.168.x.0/24 or 10.0.x.0/24)
rpcallowip=192.168.1.0/24

# RPC Credentials (CHANGE THESE VALUES!)
rpcuser=your_username
rpcpassword=your_strong_password

# Specific signet configuration
[signet]
rpcport=38332
```



*конфигурация *mainnet** (для производственного использования) :



```conf
# RPC Server
server=1
rpcbind=0.0.0.0
rpcallowip=192.168.1.0/24

# RPC Credentials
rpcuser=your_username
rpcpassword=your_strong_password

# Mainnet Port
rpcport=8332
```



**Важно** :




- Signet настоятельно рекомендуется** для первых тестов: приложение все еще находится в стадии разработки (бета-версия), и в нем еще могут быть ошибки. Signet позволяет тестировать бесплатно, не рискуя реальными средствами
- Замените `192.168.1.0/24` на подсеть вашей сети (например, если ваш IP - `192.168.68.57`, используйте `192.168.68.0/24`)



**Безопасность**: Создайте надежный пароль:



```bash
openssl rand -base64 32
```

### Конфигурация приложения



![PERMISSIONS VPN](assets/fr/08.webp)



При первом запуске приложение Joinstr запросит разрешения на управление встроенным VPN. Примите оба запроса на разрешение: Управление OpenVPN и Подключение VPN. Эти разрешения необходимы для интеграции VPN, которая защищает конфиденциальность вашей сети.



![INTERFACE APPLICATION](assets/fr/09.webp)



Приложение Joinstr состоит из трех основных вкладок:




- Главная**: Главный экран
- Пулы**: Создание и управление пулами CoinJoin
- Настройки**: Конфигурация приложения



![CONFIGURATION SETTINGS](assets/fr/13.webp)



Настройте параметры на вкладке "Настройки":



**1. Nostr Relay**: Адрес ретранслятора Nostr для координации пула




- Пример: `wss://relay.damus.io`
- Другие рекомендуемые реле: `wss://nos.lol`, `wss://relay.nostr.band`, `wss://nostr.fmt.wiz.biz`
- ⚠️ **Важно**: Все участники должны использовать **один и тот же релей Nostr**, чтобы видеть и присоединяться к одним и тем же пулам. Если вы используете другой ретранслятор, вы не увидите пулы, созданные на других ретрансляторах



**2. URL-адрес узла**: IP-адрес вашего узла Bitcoin (без указания порта)




- Формат: `http://VOTRE_IP_LOCALE`
- Example: `http://192.168.68.57`



**3. Имя пользователя RPC** : Имя пользователя, настроенное в `rpcuser=` на вашем bitcoin.conf




- Пример: `satoshi



**4. Пароль RPC** : Пароль, установленный в `rpcpassword=` на вашем bitcoin.conf



**5. Порт RPC** : Порт RPC в зависимости от сети




- Mainnet** : `8332`
- Закладка**: `38332`



**6. Wallet**: Выберите портфель Bitcoin Core, содержащий UTXO для смешивания




- Пример: `tuto_joinstr_signet`



**7. VPN-шлюз**: Выберите VPN-сервер Riseup




- Пример: `(Париж) vpn07-par.riseup.net`
- Другие доступные: Амстердам, Сиэтл и др.
- ⚠️ **Важно**: Все участники одного пула должны использовать **один и тот же VPN-шлюз** для участия в CoinJoin. Во время раунда смешивания все участники должны появляться с одним и тем же IP-адресом выхода, чтобы сетевой анализ не смог сопоставить участников



Приложение Joinstr **интегрируется в Riseup VPN, упрощая координацию между участниками.



**Важно** :




- Убедитесь, что ваш телефон и компьютер находятся в одной локальной сети WiFi
- VPN-соединение будет активировано автоматически при участии в пуле
- Нажмите на **"Сохранить "**, когда установите все параметры



## Практическое использование



### Создайте пул или присоединитесь к нему



![CRÉATION POOL ANDROID](assets/fr/10.webp)



У вас есть два варианта участия в CoinJoin:



**Вариант 1: Создайте новый пул**



Нажмите на кнопку "Создать новый пул" на вкладке "Пулы". Укажите номинал в BTC (например, 0,002 BTC) и желаемое количество участников (минимум 2, рекомендуется 3-5 для большей анонимности). Затем приложение ожидает, пока другие пользователи присоединятся к вашему пулу. Как только необходимое количество участников будет достигнуто, процесс регистрации вывода начнется автоматически, и вам нужно будет выбрать свой UTXO для смешивания и нажать кнопку "Зарегистрироваться".



**⏱️ Важно**: Срок действия пулов истекает через **10 минут** бездействия. Если необходимое количество участников не будет набрано, пул будет автоматически закрыт.



**Вариант 2: Присоединиться к существующему пулу**



На вкладке "Просмотр других пулов" просмотрите доступные пулы, созданные другими пользователями. Выберите пул, соответствующий вашей сумме, и присоединитесь к нему. Убедитесь, что вы настроили те же ретранслятор Nostr и VPN-шлюз, что и другие участники (см. раздел "Конфигурация").



**Правило выбора UTXO**: Ваш UTXO должен быть немного выше номинала пула (от +500 до +5000 sats в плюсе).



**Пример**: Для пула в 0,002 BTC (200 000 sats) используйте UTXO между 200 500 и 205 000 sats.



![PROCESSUS COINJOIN](assets/fr/11.webp)



Далее процесс идет **полностью автоматически**: после регистрации вашего вклада приложение ждет, пока все участники зарегистрируют свои вклады. Как только все входы зарегистрированы, создается PSBT, автоматически подписывается вашими ключами, а затем объединяется с подписями других участников. Наконец, вся транзакция транслируется в сеть Bitcoin. После завершения трансляции вы получаете TXID (идентификатор транзакции). Никаких ручных манипуляций с PSBT на Android не требуется.



### Проверка on-chain



![TRANSACTION MEMPOOL](assets/fr/12.webp)



Как только транзакция будет транслирована, вы получите TXID (идентификатор транзакции). Просмотрите его на [mempool.space](https://mempool.space) или в вашем любимом браузере. Чтобы проверить на закладке, используйте [mempool.space/signet](https://mempool.space/signet).



Вы можете наблюдать :





- N записей**: По одной на участника (в нашем примере - 2 заявки)
- N одинаковых выходов**: точная сумма деноминации (здесь 2 выхода по 0,00199800 BTC)
- Блок-схема**: mempool.space наглядно отображает сочетание входов и выходов
- Особенности** : Сделка может быть идентифицирована как SegWit, Taproot, RBF и т.д.



Поскольку все основные выходы имеют одинаковое количество, **невозможно определить, какой из них кому принадлежит**. Это фундаментальный принцип CoinJoin: неразличимость выходов.



**Exemple de transaction signet** : [404a6a58a1682341c1197655fa492fa160bf63fca8c3b29be125e7360dbec44c](https://mempool.space/signet/tx/404a6a58a1682341c1197655fa492fa160bf63fca8c3b29be125e7360dbec44c)



**Обратите внимание**: Если ваши UTXO были больше, чем номинал пула, у вас будут валютные выходы (излишки). Эти обменные UTXO остаются отслеживаемыми и должны управляться отдельно от ваших анонимизированных выпусков. Никогда не объединяйте их с вашими смешанными выходами.



## Пакеты CoinJoin для обеспечения качества и анонимности



Эффективность CoinJoin измеряется ее **анонсет**: количество выходов одинаковой ценности, произведенных в результате операции. Чем выше это число, тем сложнее определить, какой вход соответствует какому выходу.



В настоящее время Joinstr генерирует пулы в среднем от **2 до 5 участников**. Эти показатели ниже, чем у традиционных централизованных координаторов вроде Wasabi (50-100 участников) или Whirlpool (5-10 участников), но такова цена децентрализации.

## Преимущества и ограничения



### Основные моменты



**Усиленная конфиденциальность**: Сочетание шифрования связи Nostr, общей VPN между участниками и рекомендуемого использования Tor создает многоуровневую защиту, которую трудно обойти.



**Прозрачность, минимальные затраты**: Никаких расходов на координацию, только расходы mining, которые справедливо распределяются между участниками. Ни один оператор не взимает процент.



### Ограничения, которые необходимо учитывать





- Переменная ликвидность**: Небольшие пулы, могут ждать, пока участники соберутся вместе
- Проект в разработке**: Приложение в бета-версии, возможны ошибки. Сначала протестируйте небольшие суммы на закладке
- Атаки сибилов**: Возможность контроля одним противником нескольких участников пула



## Лучшие практики



**Изоляция UTXO**: Никогда не объединяйте смешанный UTXO с несмешанным. Используйте отдельный портфель для анонимизированных результатов.



**Много раундов обязательно**: Выполните минимум 3 последовательных раунда с разными участниками. Варьируйте количество и время, чтобы избежать шаблонов. Проводите раунды с разницей в несколько часов.



**Защита сети**: Систематически используйте Tor в дополнение к встроенному VPN. Генерируйте эфемерные ключи Nostr для каждой важной сессии.



**Осторожное продвижение**: Начинайте делать закладки с небольших объемов.



## Заключение



Joinstr представляет собой по-настоящему децентрализованное решение для обеспечения конфиденциальности Bitcoin. Используя Nostr для координации, он устраняет зависимость от центральных координаторов, сохраняя суверенитет пользователей.



**Для пользователей, которые ценят устойчивость к цензуре и готовы выполнить несколько раундов CoinJoin, чтобы компенсировать меньшие пулы.



На фоне растущего финансового контроля децентрализованные инструменты для защиты конфиденциальности становятся крайне необходимыми.
