## پیش‌نیازها


پیش‌نیازها قبل از شروع آموزش:




- یک [حساب GitHub](https://github.com/signup) داشته باشید؛

- [یک پروفایل معلم در Plan ₿ Network داشته باشید](https://planb.network/professors) (فقط اگر یک آموزش کامل ارائه می‌دهید).


اگر برای تهیه این پیش‌نیازها به کمک نیاز دارید، آموزش‌های دیگر من کمک خواهند کرد:

هنگامی که همه چیز در جای خود قرار گرفت و شما Fork از مخزن Plan ₿ Network خود را دارید، می‌توانید اضافه کردن آموزش را شروع کنید.


## ۱ - ایجاد یک شاخه جدید


مرورگر خود را باز کرده و به صفحه Fork خود در مخزن Plan ₿ Network بروید. این همان Fork است که شما در GitHub ایجاد کرده‌اید. URL مربوط به Fork شما باید به این شکل باشد: `https://github.com/[your-username]/Bitcoin-educational-content`:


![GITHUB](assets/fr/01.webp)


مطمئن شوید که در شاخه اصلی `dev` هستید، سپس روی دکمه "*Sync Fork*" کلیک کنید. اگر Fork شما به‌روز نیست، GitHub از شما می‌خواهد که شاخه خود را به‌روزرسانی کنید. با این به‌روزرسانی ادامه دهید:


![GITHUB](assets/fr/02.webp)


روی شاخه `dev` کلیک کنید، سپس شاخه کاری خود را طوری نام‌گذاری کنید که عنوان آن به وضوح هدف آن را منعکس کند و از خط تیره برای جدا کردن کلمات استفاده کنید. به عنوان مثال، اگر هدف ما نوشتن یک آموزش در مورد استفاده از Green Wallet باشد، شاخه می‌تواند به این صورت نام‌گذاری شود: `tuto-Green-Wallet-loic`. پس از وارد کردن یک نام مناسب، روی "*Create branch*" کلیک کنید تا ایجاد شاخه جدید خود بر اساس `dev` را تأیید کنید:


![GITHUB](assets/fr/03.webp)


اکنون باید در شاخه جدید کاری خود باشید:


![GITHUB](assets/fr/04.webp)


این بدان معناست که هر تغییری که ایجاد کنید فقط در آن شاخه خاص ذخیره خواهد شد.


برای هر مقاله جدیدی که قصد انتشار دارید، یک شاخه جدید از `dev` ایجاد کنید.


یک شاخه در Git نمایانگر نسخه‌ای موازی از پروژه است که به شما اجازه می‌دهد روی تغییرات کار کنید بدون اینکه شاخه اصلی تحت تأثیر قرار گیرد، تا زمانی که کار شما آماده ادغام شود.


## ۲ - افزودن فایل‌های آموزشی


اکنون که شاخه کاری ایجاد شده است، زمان آن رسیده که آموزش جدید خود را ادغام کنید.


در فایل‌های شاخه خود، باید زیرپوشه مناسب برای قرار دادن آموزش خود را پیدا کنید. سازماندهی پوشه‌ها بخش‌های مختلف وب‌سایت Plan ₿ Network را منعکس می‌کند. در مثال ما، از آنجا که در حال اضافه کردن یک آموزش در Green Wallet هستیم، به مسیر زیر بروید: `Bitcoin-educational-content\tutorials\Wallet` که به بخش `Wallet` وب‌سایت مربوط می‌شود:


![GITHUB](assets/fr/05.webp)


در پوشه `Wallet`، یک دایرکتوری جدید به‌طور خاص برای آموزش خود ایجاد کنید. نام این پوشه باید به‌وضوح نرم‌افزار پوشش داده‌شده در آموزش را نشان دهد و از خط تیره برای اتصال کلمات استفاده کند. برای مثال من، پوشه به نام `Green-Wallet` خواهد بود. روی "*Add File*" کلیک کنید و سپس روی "*Create new file*":


![GITHUB](assets/fr/06.webp)


نام پوشه را وارد کنید و به دنبال آن یک اسلش `/` بگذارید تا ایجاد آن به عنوان یک پوشه تأیید شود.


![GITHUB](assets/fr/07.webp)


در این زیرپوشه جدید که به آموزش شما اختصاص داده شده است، باید چندین مورد اضافه کنید:




- یک پوشه به نام `assets` ایجاد کنید تا تمام تصاویر مورد نیاز برای آموزش خود را در آن نگه دارید؛
- در این پوشه `assets`، یک زیرپوشه با نام کد زبان اصلی آموزش ایجاد کنید. به عنوان مثال، اگر آموزش به زبان انگلیسی نوشته شده است، این زیرپوشه باید `en` نامگذاری شود. تمام تصاویر آموزشی (نمودارها، تصاویر، اسکرین‌شات‌ها و غیره) را در این پوشه قرار دهید.
- یک فایل `tutorial.yml` باید ایجاد شود تا جزئیات آموزش شما را ثبت کند؛
- یک فایل مارک‌داون باید ایجاد شود تا محتوای واقعی آموزش شما نوشته شود. این فایل باید بر اساس کد زبانی که در آن نوشته شده است نام‌گذاری شود. به عنوان مثال، برای یک آموزش نوشته شده به زبان فرانسوی، فایل باید `fr.md` نامیده شود.


خلاصه اینکه، در اینجا سلسله مراتب فایل‌ها آمده است (ما به ایجاد آنها در بخش بعدی ادامه خواهیم داد):


```
bitcoin-educational-content/
└── tutorials/
└── wallet/ (à modifier avec la bonne catégorie)
└── green-wallet/ (à modifier avec le nom du tuto)
├── assets/
│   ├── fr/ (à modifier selon le code de langue approprié)
├── tutorial.yml
└── fr.md (à modifier selon le code de langue approprié)
```


## ۳ - فایل YAML را پر کنید


بیایید با فایل YAML شروع کنیم. در کادر ایجاد فایل جدید، `tutorial.yml` را وارد کنید:


![GITHUB](assets/fr/08.webp)


`tutorial.yml` فایل را با کپی کردن قالب زیر پر کنید:


```
id:

project_id:

tags:
-
-
-

category:

level:

professor_id:

# Proofreading metadata

original_language:
proofreading:
- language:
last_contribution_date:
urgency:
contributor_names:
-
reward:
```


در اینجا فیلدهای مورد نیاز آمده است:



- شناسه**: یک UUID (_شناسه‌ی یکتا به صورت جهانی_) که به‌طور منحصربه‌فرد آموزش را شناسایی می‌کند. می‌توانید آن را با استفاده از [یک ابزار آنلاین](https://www.uuidgenerator.net/version4) به generate تبدیل کنید. تنها شرط این است که این UUID تصادفی باشد تا از تداخل با UUID دیگری در پلتفرم جلوگیری شود؛



- project_id**: شناسه UUID شرکت یا سازمان پشت ابزار ارائه‌شده در آموزش [از لیست پروژه‌ها](https://github.com/PlanB-Network/Bitcoin-educational-content/tree/dev/resources/projects). به عنوان مثال، اگر در حال ایجاد یک آموزش درباره نرم‌افزار Green Wallet هستید، می‌توانید این `project_id` را در فایل زیر پیدا کنید: `Bitcoin-educational-content/resources/projects/blockstream/project.yml`. این اطلاعات به فایل YAML آموزش شما اضافه می‌شود زیرا Plan ₿ Network یک پایگاه داده از تمام شرکت‌ها و سازمان‌هایی که بر روی Bitcoin یا پروژه‌های مرتبط فعالیت می‌کنند، نگهداری می‌کند. با افزودن `project_id` موجودیت مرتبط با آموزش خود، یک پیوند بین دو Elements ایجاد می‌کنید؛



- برچسب‌ها**: 2 یا 3 کلمه کلیدی مرتبط با محتوای آموزش، به‌طور انحصاری انتخاب شده [از لیست برچسب‌های Plan ₿ Network](https://github.com/PlanB-Network/Bitcoin-educational-content/blob/dev/docs/50-planb-tags.md)؛



- دسته‌بندی**: زیرمجموعه مربوط به محتوای آموزشی، مطابق با ساختار وب‌سایت Plan ₿ Network (برای مثال، برای کیف‌پول‌ها: `desktop`، `hardware`، `mobile`، `backup`);



- سطح**: سطح دشواری آموزش، انتخاب شده از:
    - `مبتدی`
    - `متوسط`
    - `پیشرفته`
    - `کارشناس`



- شناسه_استاد**: `شناسه_استاد` شما (UUID) همانطور که در [پروفایل استاد شما](https://github.com/PlanB-Network/Bitcoin-educational-content/tree/dev/professors) نمایش داده شده است؛



- original_language**: زبان اصلی آموزش (به عنوان مثال، `fr`، `en`، و غیره)؛



- بازخوانی**: اطلاعاتی درباره فرآیند بازخوانی. بخش اول را کامل کنید، زیرا بازخوانی آموزش خودتان به عنوان اولین اعتبارسنجی محسوب می‌شود:
    - زبان**: کد زبان برای تصحیح (مثلاً، `fr`، `en`، و غیره).
    - last_contribution_date**: تاریخ روز.
    - فوریت**: 1
    - نام‌های_مشارکت‌کننده**: شناسه GitHub شما.
    - پاداش**: 0


برای جزئیات بیشتر در مورد شناسه معلم خود، لطفاً به آموزش مربوطه مراجعه کنید:


https://planb.network/tutorials/contribution/others/create-teacher-profile-8ba9ba49-8fac-437a-a435-c38eebc8f8a4

## ۵ - ایجاد فایل Markdown


حالا ما قصد داریم فایلی ایجاد کنیم که میزبان آموزش شما باشد، نام‌گذاری شده بر اساس کد زبان شما، برای مثال `fr.md` اگر به زبان فرانسوی می‌نویسیم. به پوشه آموزش خود بروید:


![GITHUB](assets/fr/16.webp)


روی "افزودن فایل*" کلیک کنید، سپس روی "ایجاد فایل جدید*" کلیک کنید.


![GITHUB](assets/fr/17.webp)


نام فایل را با کد زبان خود نام‌گذاری کنید. در مورد من، چون آموزش به زبان فرانسوی نوشته شده است، فایل خود را `fr.md` نام‌گذاری می‌کنم. پسوند `.md` نشان می‌دهد که فایل در فرمت Markdown است.


![GITHUB](assets/fr/18.webp)


ما با پر کردن بخش `Properties` در بالای سند شروع می‌کنیم. به صورت دستی بلوک کد زیر را اضافه کرده و پر کنید (کلیدهای `name:` و `description:` باید به زبان انگلیسی حفظ شوند، اما مقادیر آنها باید به زبانی که در آموزش شما استفاده می‌شود نوشته شوند):


```
---
name: [Titre]
description: [Description]
---
```


![GITHUB](assets/fr/19.webp)


نام آموزش خود را وارد کنید و توضیح کوتاهی بنویسید:


![GITHUB](assets/fr/20.webp)

## ۶ - افزودن لوگو و کاور


در پوشه `assets`، شما باید فایلی به نام `logo.webp` اضافه کنید که به عنوان تصویر بندانگشتی برای مقاله شما عمل خواهد کرد. این تصویر باید در فرمت `.webp` باشد و باید به صورت مربع باشد تا با کاربر Interface مطابقت داشته باشد.


شما آزاد هستید که لوگوی نرم‌افزار استفاده شده در آموزش، یا هر تصویر مرتبط دیگری را انتخاب کنید، به شرطی که بدون حق امتیاز باشد. علاوه بر این، یک تصویر با عنوان `cover.webp` در همان مکان اضافه کنید. این تصویر در بالای آموزش شما نمایش داده خواهد شد. اطمینان حاصل کنید که این تصویر، مانند لوگو، حقوق استفاده را رعایت کرده و با محتوای آموزش شما مناسب است.


برای افزودن تصاویر به پوشه‌ی `/assets`، می‌توانید آن‌ها را از فایل‌های محلی خود بکشید و رها کنید. مطمئن شوید که در پوشه‌ی `/assets` و روی شاخه‌ی درست هستید، سپس روی "*Commit changes*" کلیک کنید.


![GITHUB](assets/fr/26.webp)


اکنون باید تصاویر در پوشه ظاهر شوند.


![GITHUB](assets/fr/27.webp)


## ۷ - نوشتن آموزش


ادامه نوشتن آموزش خود را با یادداشت کردن محتوای خود در فایل Markdown با کد زبان (در مثال من، به زبان فرانسوی، فایل `fr.md` است) انجام دهید. به فایل بروید و روی آیکون مداد کلیک کنید:


![GITHUB](assets/fr/28.webp)


نوشتن آموزش خود را شروع کنید. هنگام افزودن زیرعنوان، از قالب‌بندی مناسب Markdown با پیشوند کردن متن با `##` استفاده کنید:

## ۸ - افزودن تصاویر


زیرپوشه زبان در پوشه `/assets` (در مثال من: `/assets/en`) برای ذخیره‌سازی نمودارها و تصاویر بصری که همراه با آموزش شما خواهند بود، استفاده می‌شود. تا حد امکان، از قرار دادن متن در تصاویر خودداری کنید تا محتوای شما برای مخاطبان بین‌المللی قابل دسترس باشد. البته، نرم‌افزار ارائه شده شامل متن خواهد بود، اما اگر شما طرح‌ها یا نشانه‌های اضافی روی اسکرین‌شات‌های نرم‌افزار اضافه می‌کنید، این کار را بدون متن انجام دهید یا اگر ضروری است، از زبان انگلیسی استفاده کنید.


برای نام‌گذاری تصاویر خود، به سادگی از اعدادی استفاده کنید که با ترتیب ظاهر شدن آن‌ها در آموزش مطابقت دارند و به صورت دو رقمی (یا سه رقمی اگر آموزش شما بیش از 99 تصویر دارد) قالب‌بندی شده‌اند. به عنوان مثال، اولین تصویر خود را `01.webp`، دومین را `02.webp` و به همین ترتیب نام‌گذاری کنید.


تصاویر شما باید فقط در قالب `.webp` باشند. در صورت لزوم، می‌توانید از [نرم‌افزار تبدیل تصویر من](https://github.com/LoicPandul/ImagesConverter) استفاده کنید.


![GITHUB](assets/fr/32.webp)


اکنون که تصاویر خود را به زیرپوشه اضافه کرده‌اید، می‌توانید فایل ساختگی `.gitkeep` را حذف کنید. این فایل را باز کنید، روی سه نقطه کوچک در گوشه بالا سمت راست کلیک کنید، سپس روی "*Delete file*" کلیک کنید.


![GITHUB](assets/fr/33.webp)


تغییرات خود را با کلیک بر روی "*Commit changes...*" ذخیره کنید.


![GITHUB](assets/fr/34.webp)


برای وارد کردن یک نمودار از زیرپوشه خود به سند ویرایشی، از فرمان Markdown زیر استفاده کنید و دقت کنید که متن جایگزین مناسب و مسیر تصویر صحیح برای زبان خود را مشخص کنید:


```

```

- فونت**: از [IBM Plex Sans](https://fonts.google.com/specimen/IBM+Plex+Sans) استفاده کنید؛
- رنگ‌ها**:
 - نارنجی: #FF5C00

**ضروری است که تمام تصاویر موجود در آموزش‌های شما عاری از حق کپی‌رایت باشند یا به مجوز فایل منبع احترام بگذارند**. بنابراین، تمام نمودارهایی که در Plan ₿ Network منتشر می‌شوند تحت مجوز CC-BY-SA در دسترس قرار می‌گیرند، به همان شیوه‌ای که متن قرار دارد.


**-> نکته:** هنگام به اشتراک‌گذاری فایل‌ها به‌صورت عمومی، مانند تصاویر، مهم است که فراداده‌های اضافی را حذف کنید. این اطلاعات می‌تواند شامل اطلاعات حساس مانند داده‌های مکانی، تاریخ‌های ایجاد و جزئیات نویسنده باشد. برای حفاظت از حریم خصوصی خود، ایده خوبی است که این فراداده‌ها را حذف کنید. برای ساده‌سازی این عملیات، می‌توانید از ابزارهای تخصصی مانند [Exif Cleaner](https://exifcleaner.com/) استفاده کنید که به شما امکان می‌دهد فراداده‌های یک سند را با کشیدن و رها کردن ساده پاک کنید.


## ۹ - پیشنهاد آموزش


پس از اتمام نوشتن آموزش خود به زبان دلخواه، مرحله بعدی ارسال یک **Pull Request** است. سپس مدیر، ترجمه‌های ناقص را با استفاده از روش ترجمه خودکار ما و بازبینی انسانی به آموزش شما اضافه خواهد کرد.


برای ادامه با Pull Request، پس از ذخیره تمام تغییرات خود، روی دکمه "*Contribute*" کلیک کنید، سپس روی "*Open pull request*":


![GITHUB](assets/fr/36.webp)


یک درخواست Pull Request درخواستی است برای ادغام تغییرات از شاخه شما به شاخه اصلی مخزن Plan ₿ Network، که امکان بررسی و بحث در مورد تغییرات را قبل از ادغام فراهم می‌کند.


قبل از ادامه، با دقت در پایین Interface بررسی کنید که این تغییرات همان چیزی است که انتظار داشتید:


![GITHUB](assets/fr/37.webp)


مطمئن شوید که در بالای Interface، شاخه کاری شما با شاخه `dev` مخزن Plan ₿ Network (که شاخه اصلی است) ادغام شده است.


عنوانی وارد کنید که به طور خلاصه تغییراتی که می‌خواهید با مخزن منبع ادغام کنید را خلاصه کند. یک نظر کوتاه توصیف‌کننده این تغییرات اضافه کنید (اگر شماره مسئله‌ای با ایجاد آموزش شما مرتبط است، به یاد داشته باشید که `Closes #{issue number}` را به عنوان یک نظر یادداشت کنید)، سپس روی دکمه Green "*ایجاد درخواست کشش*" کلیک کنید تا درخواست ادغام را تأیید کنید:


![GITHUB](assets/fr/38.webp)


سپس درخواست کشش شما در زبانه "*Pull Request*" مخزن اصلی Plan ₿ Network قابل مشاهده خواهد بود. اکنون تنها کاری که باید انجام دهید این است که منتظر بمانید تا یک مدیر با شما تماس بگیرد تا تأیید کند که مشارکت شما ادغام شده است یا درخواست هرگونه تغییرات بیشتری را بدهد.


![GITHUB](assets/fr/39.webp)


پس از ادغام PR خود با شاخه اصلی، توصیه می‌کنیم شاخه کاری خود را حذف کنید (در مثال من: `tuto-Green-Wallet`) تا تاریخچه تمیزی از Fork خود داشته باشید. GitHub به‌طور خودکار این گزینه را در صفحه PR شما ارائه می‌دهد:


![GITHUB](assets/fr/40.webp)


اگر می‌خواهید پس از ارسال PR خود تغییراتی در مشارکت خود ایجاد کنید، مراحل مورد نیاز بستگی به وضعیت فعلی PR شما دارد:




- اگر درخواست کشش شما هنوز باز است و ادغام نشده است، تغییرات را در همان شاخه کاری اعمال کنید. تغییرات تعهدی به درخواست کشش باز شما اضافه خواهد شد؛
- در صورتی که PR شما قبلاً با شاخه اصلی ادغام شده است، باید فرآیند را از ابتدا با ایجاد یک شاخه جدید و سپس ارسال یک PR جدید تکرار کنید. اطمینان حاصل کنید که Fork شما با مخزن منبع Plan ₿ Network در شاخه `dev` همگام‌سازی شده است قبل از ادامه.


اگر در ارسال آموزش خود با مشکلات فنی مواجه هستید، لطفاً در درخواست کمک از [گروه تلگرام اختصاصی ما برای مشارکت‌ها](https://t.me/PlanBNetwork_ContentBuilder) تردید نکنید. بسیار متشکریم!
