## 1 - สร้างสาขาใหม่


เปิดเบราว์เซอร์ของคุณและไปที่หน้า fork ของคุณในที่เก็บ Plan ₿ Academy นี่คือ fork ที่คุณสร้างขึ้นบน GitHub URL ของ fork ของคุณควรมีลักษณะดังนี้: `https://github.com/[your-username]/bitcoin-educational-content`:


![GITHUB](assets/fr/01.webp)


ตรวจสอบให้แน่ใจว่าคุณอยู่บนสาขาหลัก `dev` จากนั้นคลิกที่ปุ่ม "*Sync fork*" หาก fork ของคุณไม่เป็นปัจจุบัน GitHub จะขอให้คุณอัปเดตสาขาของคุณ ดำเนินการอัปเดตนี้:


![GITHUB](assets/fr/02.webp)


คลิกที่สาขา `dev` จากนั้นตั้งชื่อสาขาการทำงานของคุณเพื่อให้ชื่อของมันสะท้อนถึงวัตถุประสงค์อย่างชัดเจน โดยใช้ขีดกลางเพื่อแยกคำ ตัวอย่างเช่น หากเป้าหมายของเราคือการเขียนบทแนะนำเกี่ยวกับการใช้ Green Wallet สาขาอาจถูกเรียกว่า: `tuto-green-wallet-loic` หลังจากป้อนชื่อที่เหมาะสมแล้ว คลิกที่ "*Create branch*" เพื่อยืนยันการสร้างสาขาใหม่ของคุณจาก `dev`:


![GITHUB](assets/fr/03.webp)


คุณควรอยู่ในสาขางานใหม่ของคุณแล้ว:


![GITHUB](assets/fr/04.webp)


ซึ่งหมายความว่าการเปลี่ยนแปลงใด ๆ ที่คุณทำจะถูกบันทึกไว้เฉพาะในสาขานั้นเท่านั้น


สำหรับแต่ละบทความใหม่ที่คุณวางแผนจะเผยแพร่ ให้สร้างสาขาใหม่จาก `dev`.


สาขาใน Git แทนเวอร์ชันคู่ขนานของโปรเจกต์ ช่วยให้คุณทำงานบนการแก้ไขโดยไม่กระทบต่อสาขาหลัก จนกว่างานของคุณจะพร้อมที่จะถูกรวมเข้าด้วยกัน


## 2 - เพิ่มไฟล์สอน


ตอนนี้ที่ได้สร้างสาขาการทำงานแล้ว ถึงเวลาที่จะรวมบทแนะนำใหม่ของคุณเข้าไป


ภายในไฟล์สาขาของคุณ คุณจะต้องค้นหาโฟลเดอร์ย่อยที่เหมาะสมสำหรับการวางตำแหน่งบทแนะนำของคุณ การจัดระเบียบของโฟลเดอร์สะท้อนถึงส่วนต่างๆ ของเว็บไซต์ Plan ₿ Academy ในตัวอย่างของเรา เนื่องจากเรากำลังเพิ่มบทแนะนำใน Green Wallet ให้ไปที่เส้นทางต่อไปนี้: `bitcoin-educational-content\tutorials\wallet` ซึ่งสอดคล้องกับส่วน `WALLET` ของเว็บไซต์:


![GITHUB](assets/fr/05.webp)


ในโฟลเดอร์ `wallet` ให้สร้างไดเรกทอรีใหม่ที่อุทิศให้กับบทเรียนของคุณโดยเฉพาะ ชื่อของโฟลเดอร์นี้ควรระบุซอฟต์แวร์ที่ครอบคลุมในบทเรียนอย่างชัดเจน โดยใช้ขีดกลางเชื่อมคำ ตัวอย่างของฉัน โฟลเดอร์จะถูกตั้งชื่อว่า `green-wallet` คลิกที่ "*Add File*" จากนั้นคลิกที่ "*Create new file*":


![GITHUB](assets/fr/06.webp)


ป้อนชื่อโฟลเดอร์ตามด้วยเครื่องหมายทับ `/` เพื่อยืนยันการสร้างเป็นโฟลเดอร์


![GITHUB](assets/fr/07.webp)


ในโฟลเดอร์ย่อยใหม่ที่อุทิศให้กับบทเรียนของคุณ คุณจำเป็นต้องเพิ่มรายการหลายรายการ:




- สร้างโฟลเดอร์ `assets` เพื่อเก็บภาพประกอบทั้งหมดที่จำเป็นสำหรับบทแนะนำของคุณ;
- ภายในโฟลเดอร์ `assets` นี้ ให้สร้างโฟลเดอร์ย่อยที่ตั้งชื่อตามรหัสภาษาต้นฉบับของบทแนะนำ ตัวอย่างเช่น หากบทแนะนำเขียนเป็นภาษาอังกฤษ โฟลเดอร์ย่อยนี้ควรตั้งชื่อว่า `en` วางภาพประกอบทั้งหมดของบทแนะนำ (แผนภาพ รูปภาพ ภาพหน้าจอ ฯลฯ) ในโฟลเดอร์นี้
- ต้องสร้างไฟล์ `tutorial.yml` เพื่อบันทึกรายละเอียดของบทแนะนำของคุณ;
- ไฟล์มาร์กดาวน์ต้องถูกสร้างขึ้นเพื่อเขียนเนื้อหาจริงของบทแนะนำของคุณ ไฟล์นี้ต้องถูกตั้งชื่อตามรหัสของภาษาที่เขียนตัวอย่างเช่น สำหรับบทแนะนำที่เขียนเป็นภาษาฝรั่งเศส ไฟล์ควรถูกเรียกว่า `fr.md`.


โดยสรุป นี่คือโครงสร้างไฟล์ (เราจะสร้างเพิ่มเติมในส่วนถัดไป):


```
bitcoin-educational-content/
└── tutorials/
└── wallet/ (à modifier avec la bonne catégorie)
└── green-wallet/ (à modifier avec le nom du tuto)
├── assets/
│   ├── fr/ (à modifier selon le code de langue approprié)
├── tutorial.yml
└── fr.md (à modifier selon le code de langue approprié)
```


## 3 - กรอกข้อมูลในไฟล์ YAML


มาเริ่มกันที่ไฟล์ YAML ในช่องสำหรับสร้างไฟล์ใหม่ ให้ป้อน `tutorial.yml`:


![GITHUB](assets/fr/08.webp)


กรอกไฟล์ `tutorial.yml` โดยคัดลอกเทมเพลตต่อไปนี้:


```
id:

project_id:

tags:
-
-
-

category:

level:

professor_id:

# Proofreading metadata

original_language:
proofreading:
- language:
last_contribution_date:
urgency:
contributor_names:
-
reward:
```


นี่คือช่องที่จำเป็น:



- id**: A UUID (_Universally Unique Identifier_) ที่ระบุบทเรียนอย่างเฉพาะเจาะจง คุณสามารถ generate มันได้โดยใช้ [เครื่องมือออนไลน์](https://www.uuidgenerator.net/version4) ข้อกำหนดเดียวคือ UUID นี้ต้องเป็นแบบสุ่มเพื่อหลีกเลี่ยงความขัดแย้งกับ UUID อื่นบนแพลตฟอร์ม;



- project_id**: UUID ของบริษัทหรือองค์กรที่อยู่เบื้องหลังเครื่องมือที่นำเสนอในบทแนะนำ [จากรายการโครงการ](https://github.com/PlanB-Network/bitcoin-educational-content/tree/dev/resources/projects) ตัวอย่างเช่น หากคุณกำลังสร้างบทแนะนำเกี่ยวกับซอฟต์แวร์ Green Wallet คุณสามารถค้นหา `project_id` นี้ได้ในไฟล์ต่อไปนี้: `bitcoin-educational-content/resources/projects/blockstream/project.yml` ข้อมูลนี้จะถูกเพิ่มลงในไฟล์ YAML ของบทแนะนำของคุณเพราะ Plan ₿ Academy รักษาฐานข้อมูลของบริษัทและองค์กรทั้งหมดที่ดำเนินการบน Bitcoin หรือโครงการที่เกี่ยวข้อง โดยการเพิ่ม `project_id` ของหน่วยงานที่เชื่อมโยงกับบทแนะนำของคุณ คุณจะสร้างลิงก์ระหว่างสององค์ประกอบนี้;



- tags**: 2 or 3 relevant keywords related to the tutorial content, exclusively chosen [from the Plan ₿ Academy tag list](https://github.com/PlanB-Network/bitcoin-educational-content/blob/dev/docs/50-planb-tags.md);



- หมวดหมู่**: หมวดหมู่ย่อยที่สอดคล้องกับเนื้อหาการสอน ตามโครงสร้างเว็บไซต์ Plan ₿ Academy (ตัวอย่างเช่น สำหรับกระเป๋าเงิน: `desktop`, `hardware`, `mobile`, `backup`);



- ระดับ**: ระดับความยากของบทแนะนำ เลือกจาก:
    - `ผู้เริ่มต้น`
    - `ระดับกลาง`
    - `ขั้นสูง`
    - `ผู้เชี่ยวชาญ`



- professor_id**: `professor_id` (UUID) ของคุณตามที่แสดงใน [โปรไฟล์ศาสตราจารย์ของคุณ](https://github.com/PlanB-Network/bitcoin-educational-content/tree/dev/professors);



- original_language**: ภาษาต้นฉบับของบทแนะนำ (เช่น `fr`, `en`, เป็นต้น);



- การพิสูจน์อักษร**: ข้อมูลเกี่ยวกับกระบวนการพิสูจน์อักษร การทำส่วนแรกให้เสร็จสิ้น เนื่องจากการพิสูจน์อักษรบทเรียนของคุณเองถือเป็นการตรวจสอบครั้งแรก:
    - language**: รหัสภาษาของการพิสูจน์อักษร (เช่น `fr`, `en`, เป็นต้น)
    - last_contribution_date**: วันที่ของวัน
    - ความเร่งด่วน**: 1
    - contributor_names**: Your GitHub ID.
    - reward**: 0


สำหรับรายละเอียดเพิ่มเติมเกี่ยวกับรหัสประจำตัวครูของคุณ โปรดดูที่บทแนะนำที่เกี่ยวข้อง:


https://planb.academy/tutorials/contribution/others/create-teacher-profile-8ba9ba49-8fac-437a-a435-c38eebc8f8a4

```

project_id: 3b2f45e6-d612-412c-95ba-cf65b49aa5b8

tags:
- wallets
- software
- keys

category: mobile

level: beginner

professor_id: 6516474c-c190-41f2-b2ab-3d452ce7bdf0

# Proofreading metadata

## 5 - การสร้างไฟล์ Markdown


ตอนนี้เราจะสร้างไฟล์ที่จะเป็นที่เก็บบทเรียนของคุณ โดยตั้งชื่อตามรหัสภาษาของคุณ เช่น `fr.md` หากเรากำลังเขียนเป็นภาษาฝรั่งเศส ไปที่โฟลเดอร์บทเรียนของคุณ:


![GITHUB](assets/fr/16.webp)


คลิกที่ "เพิ่มไฟล์*" จากนั้นคลิกที่ "สร้างไฟล์ใหม่*"


![GITHUB](assets/fr/17.webp)


ตั้งชื่อไฟล์โดยใช้รหัสภาษาของคุณ ในกรณีของฉัน เนื่องจากบทแนะนำเขียนเป็นภาษาฝรั่งเศส ฉันจึงตั้งชื่อไฟล์ของฉันว่า `fr.md` ส่วนขยาย `.md` บ่งบอกว่าไฟล์อยู่ในรูปแบบ Markdown


![GITHUB](assets/fr/18.webp)


เราเริ่มต้นด้วยการกรอกส่วน `Properties` ที่ด้านบนของเอกสาร เพิ่มและกรอกบล็อกโค้ดต่อไปนี้ด้วยตนเอง (คีย์ `name:` และ `description:` ต้องเก็บไว้เป็นภาษาอังกฤษ แต่ค่าของพวกมันต้องเขียนเป็นภาษาที่ใช้ในบทแนะนำของคุณ):


```
---
name: [Titre]
description: [Description]

## 6 - เพิ่มโลโก้และหน้าปก


ภายในโฟลเดอร์ `assets` คุณจำเป็นต้องเพิ่มไฟล์ชื่อ `logo.webp` ซึ่งจะใช้เป็นภาพขนาดย่อสำหรับบทความของคุณ ภาพนี้ต้องอยู่ในรูปแบบ `.webp` และต้องมีขนาดเป็นสี่เหลี่ยมจัตุรัสเพื่อให้ตรงกับส่วนติดต่อผู้ใช้

## 8 - เพิ่มภาพประกอบ

## 9 - เสนอการสอน


เมื่อคุณเขียนบทแนะนำในภาษาที่คุณเลือกเสร็จแล้ว ขั้นตอนต่อไปคือการส่ง **Pull Request** ผู้ดูแลระบบจะเพิ่มการแปลที่ขาดหายไปในบทแนะนำของคุณ โดยใช้วิธีการแปลอัตโนมัติของเราพร้อมการพิสูจน์อักษรโดยมนุษย์


ในการดำเนินการต่อกับคำขอดึง หลังจากบันทึกการเปลี่ยนแปลงทั้งหมดของคุณแล้ว ให้คลิกที่ปุ่ม "*Contribute*" จากนั้นคลิกที่ "*Open pull request*":


![GITHUB](assets/fr/36.webp)


คำขอดึง (Pull Request) คือคำขอที่ทำเพื่อรวมการเปลี่ยนแปลงจากสาขาของคุณเข้าสู่สาขาหลักของที่เก็บ Plan ₿ Academy ซึ่งช่วยให้สามารถตรวจสอบและอภิปรายการเปลี่ยนแปลงก่อนที่จะรวมเข้าด้วยกัน


ก่อนดำเนินการต่อ โปรดตรวจสอบอย่างละเอียดที่ด้านล่างของอินเทอร์เฟซว่าการเปลี่ยนแปลงเหล่านี้เป็นไปตามที่คุณคาดหวัง:


![GITHUB](assets/fr/37.webp)


ตรวจสอบให้แน่ใจว่า ที่ด้านบนของอินเทอร์เฟซ สาขาที่คุณกำลังทำงานอยู่ได้ถูกรวมเข้ากับสาขา `dev` ของที่เก็บ Plan ₿ Academy (ซึ่งเป็นสาขาหลัก) แล้ว


ป้อนชื่อเรื่องที่สรุปการเปลี่ยนแปลงที่คุณต้องการรวมเข้ากับที่เก็บต้นทาง เพิ่มความคิดเห็นสั้น ๆ ที่อธิบายการเปลี่ยนแปลงเหล่านี้ (หากคุณมีหมายเลขปัญหาที่เกี่ยวข้องกับการสร้างบทแนะนำของคุณ อย่าลืมระบุ `Closes #{issue number}` เป็นความคิดเห็น) จากนั้นคลิกที่ปุ่ม "*Create pull request*" สีเขียวเพื่อยืนยันคำขอรวม:


![GITHUB](assets/fr/38.webp)


PR ของคุณจะปรากฏในแท็บ "*Pull Request*" ของที่เก็บหลัก Plan ₿ Academy สิ่งที่คุณต้องทำตอนนี้คือรอจนกว่าผู้ดูแลระบบจะติดต่อคุณเพื่อยืนยันว่าการมีส่วนร่วมของคุณได้ถูกรวมเข้าด้วยกันแล้ว หรือเพื่อขอการแก้ไขเพิ่มเติมใดๆ


![GITHUB](assets/fr/39.webp)


หลังจากรวม PR ของคุณเข้ากับสาขาหลักแล้ว เราแนะนำให้ลบสาขาการทำงานของคุณ (ในตัวอย่างของฉัน: `tuto-green-wallet`) เพื่อรักษาประวัติที่สะอาดของ fork ของคุณ GitHub จะเสนอทางเลือกนี้ให้คุณโดยอัตโนมัติในหน้าของ PR ของคุณ:


![GITHUB](assets/fr/40.webp)


หากคุณต้องการแก้ไขการมีส่วนร่วมของคุณหลังจากที่คุณได้ส่ง PR แล้ว ขั้นตอนที่ต้องปฏิบัติตามจะขึ้นอยู่กับสถานะปัจจุบันของ PR ของคุณ:




- หาก PR ของคุณยังเปิดอยู่และยังไม่ได้ถูกรวมเข้าด้วยกัน ให้ทำการเปลี่ยนแปลงบน workbranch เดิม การเปลี่ยนแปลงการ commit จะถูกเพิ่มเข้าไปใน PR ที่ยังเปิดอยู่ของคุณ;
- ในกรณีที่ PR ของคุณได้ถูกรวมเข้ากับสาขาหลักแล้ว คุณจะต้องเริ่มกระบวนการใหม่ตั้งแต่ต้นโดยการสร้างสาขาใหม่ จากนั้นส่ง PR ใหม่ ตรวจสอบให้แน่ใจว่า fork ของคุณถูกซิงโครไนซ์กับที่เก็บต้นทาง Plan ₿ Academy บนสาขา `dev` ก่อนดำเนินการต่อ


หากคุณประสบปัญหาทางเทคนิคในการส่งบทเรียนของคุณ โปรดอย่าลังเลที่จะขอความช่วยเหลือใน [กลุ่ม Telegram สำหรับการมีส่วนร่วมของเรา](https://t.me/PlanBNetwork_ContentBuilder) ขอบคุณมาก!
