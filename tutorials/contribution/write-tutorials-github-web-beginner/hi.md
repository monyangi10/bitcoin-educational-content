## 1 - एक नई शाखा बनाएँ


अपना ब्राउज़र खोलें और Plan ₿ Network रिपॉजिटरी में अपने Fork पेज पर जाएँ। यह वही Fork है जिसे आपने GitHub पर स्थापित किया है। आपके Fork का URL इस तरह दिखना चाहिए: `https://github.com/[your-username]/Bitcoin-educational-content` :


![GITHUB](assets/fr/01.webp)


सुनिश्चित करें कि आप मुख्य `dev` शाखा पर हैं, फिर "*Sync Fork*" बटन पर क्लिक करें। यदि आपका Fork अद्यतित नहीं है, तो GitHub आपसे अपनी शाखा को अपडेट करने के लिए कहेगा। इस अपडेट के साथ आगे बढ़ें:


![GITHUB](assets/fr/02.webp)


`dev` शाखा पर क्लिक करें, फिर अपनी कार्यशील शाखा को नाम दें ताकि उसका शीर्षक स्पष्ट रूप से उसके उद्देश्य को दर्शाए, शब्दों को अलग करने के लिए डैश का उपयोग करें। उदाहरण के लिए, यदि हमारा उद्देश्य Green Wallet का उपयोग करने पर एक ट्यूटोरियल लिखना है, तो शाखा को इस प्रकार कहा जा सकता है: `tuto-Green-Wallet-loic`। उपयुक्त नाम दर्ज करने के बाद, `dev` पर आधारित अपनी नई शाखा के निर्माण की पुष्टि करने के लिए "*शाखा बनाएँ*" पर क्लिक करें:


![GITHUB](assets/fr/03.webp)


अब आप अपने कार्य की नई शाखा पर होंगे:


![GITHUB](assets/fr/04.webp)


इसका अर्थ यह है कि आपके द्वारा किये गए कोई भी परिवर्तन केवल उसी विशिष्ट शाखा पर ही सहेजे जायेंगे।


प्रत्येक नए आलेख के लिए जिसे आप प्रकाशित करना चाहते हैं, `dev` से एक नई शाखा बनाएं।


Git में एक शाखा परियोजना के समानांतर संस्करण का प्रतिनिधित्व करती है, जो आपको मुख्य शाखा को प्रभावित किए बिना संशोधनों पर काम करने की अनुमति देती है, जब तक कि आपका काम एकीकृत होने के लिए तैयार न हो जाए।


## 2 - ट्यूटोरियल फ़ाइलें जोड़ें


अब चूंकि कार्यशील शाखा बन गई है, तो अब आपके नए ट्यूटोरियल को एकीकृत करने का समय आ गया है।


अपनी शाखा फ़ाइलों के भीतर, आपको अपने ट्यूटोरियल की नियुक्ति के लिए उपयुक्त सबफ़ोल्डर ढूँढ़ना होगा। फ़ोल्डरों का संगठन Plan ₿ Network वेबसाइट के विभिन्न अनुभागों को दर्शाता है। हमारे उदाहरण में, चूँकि हम Green Wallet पर एक ट्यूटोरियल जोड़ रहे हैं, इसलिए निम्न पथ पर जाएँ: `Bitcoin-educational-content\tutorials\Wallet` जो वेबसाइट के `Wallet` अनुभाग से मेल खाता है:


![GITHUB](assets/fr/05.webp)


`Wallet` फ़ोल्डर में, अपने ट्यूटोरियल के लिए विशेष रूप से समर्पित एक नई निर्देशिका बनाएँ। इस फ़ोल्डर का नाम ट्यूटोरियल में शामिल सॉफ़्टवेयर को स्पष्ट रूप से इंगित करना चाहिए, शब्दों को जोड़ने के लिए हाइफ़न का उपयोग करना चाहिए। मेरे उदाहरण के लिए, फ़ोल्डर का नाम `Green-Wallet` होगा। "*फ़ाइल जोड़ें*" पर क्लिक करें और फिर "*नई फ़ाइल बनाएँ*" पर क्लिक करें:


![GITHUB](assets/fr/06.webp)


फ़ोल्डर के रूप में इसके निर्माण की पुष्टि करने के लिए फ़ोल्डर का नाम और उसके बाद स्लैश `/` दर्ज करें।


![GITHUB](assets/fr/07.webp)


आपके ट्यूटोरियल को समर्पित इस नए सबफ़ोल्डर में, आपको कई आइटम जोड़ने होंगे:




- अपने ट्यूटोरियल के लिए आवश्यक सभी चित्रों को रखने के लिए एक `assets` फ़ोल्डर बनाएं;
- इस `एसेट्स` फ़ोल्डर में, ट्यूटोरियल के मूल भाषा कोड के अनुसार नाम वाला एक सबफ़ोल्डर बनाएँ। उदाहरण के लिए, यदि ट्यूटोरियल अंग्रेजी में लिखा गया है, तो इस सबफ़ोल्डर का नाम `en` होना चाहिए। ट्यूटोरियल के सभी विज़ुअल (आरेख, चित्र, स्क्रीनशॉट, आदि) इस फ़ोल्डर में रखें।
- आपके ट्यूटोरियल का विवरण रिकॉर्ड करने के लिए एक `tutorial.yml` फ़ाइल बनाई जानी चाहिए;
- आपके ट्यूटोरियल की वास्तविक सामग्री लिखने के लिए एक मार्कडाउन फ़ाइल बनाई जानी चाहिए। इस फ़ाइल का नाम उस भाषा के कोड के अनुसार होना चाहिए जिसमें इसे लिखा गया है। उदाहरण के लिए, फ्रेंच में लिखे गए ट्यूटोरियल के लिए, फ़ाइल को `fr.md` कहा जाना चाहिए।


संक्षेप में, यहां फ़ाइल पदानुक्रम है (हम अगले अनुभाग में उनका निर्माण जारी रखेंगे):


```
bitcoin-educational-content/
└── tutorials/
└── wallet/ (à modifier avec la bonne catégorie)
└── green-wallet/ (à modifier avec le nom du tuto)
├── assets/
│   ├── fr/ (à modifier selon le code de langue approprié)
├── tutorial.yml
└── fr.md (à modifier selon le code de langue approprié)
```


## 3 - YAML फ़ाइल भरें


आइए YAML फ़ाइल से शुरू करें। नई फ़ाइल बनाने के लिए बॉक्स में, `tutorial.yml` दर्ज करें:


![GITHUB](assets/fr/08.webp)


निम्नलिखित टेम्पलेट की प्रतिलिपि बनाकर `tutorial.yml` फ़ाइल भरें:


```
id:

project_id:

tags:
-
-
-

category:

level:

professor_id:

# Proofreading metadata

original_language:
proofreading:
- language:
last_contribution_date:
urgency:
contributor_names:
-
reward:
```


आवश्यक फ़ील्ड इस प्रकार हैं:



- id** : एक UUID (_यूनिवर्सली यूनिक आइडेंटिफ़ायर_) जो ट्यूटोरियल को विशिष्ट रूप से पहचानता है। आप इसे [ऑनलाइन टूल](https://www.uuidgenerator.net/version4) का उपयोग करके generate कर सकते हैं। एकमात्र आवश्यकता यह है कि यह UUID प्लेटफ़ॉर्म पर किसी अन्य UUID के साथ टकराव से बचने के लिए यादृच्छिक हो;



- project_id** : ट्यूटोरियल में प्रस्तुत टूल के पीछे कंपनी या संगठन का UUID [प्रोजेक्ट सूची से](https://github.com/PlanB-Network/Bitcoin-educational-content/tree/dev/resources/projects)। उदाहरण के लिए, यदि आप Green Wallet सॉफ़्टवेयर के बारे में एक ट्यूटोरियल बना रहे हैं, तो आप इस `project_id` को निम्न फ़ाइल में पा सकते हैं: `Bitcoin-educational-content/resources/projects/blockstream/project.yml`। यह जानकारी आपके ट्यूटोरियल की YAML फ़ाइल में इसलिए जोड़ी जाती है क्योंकि Plan ₿ Network Bitcoin या संबंधित प्रोजेक्ट पर काम करने वाली सभी कंपनियों और संगठनों का डेटाबेस बनाए रखता है। अपने ट्यूटोरियल से जुड़ी इकाई का `project_id` जोड़कर, आप दो Elements के बीच एक लिंक बनाते हैं;



- टैग**: ट्यूटोरियल सामग्री से संबंधित 2 या 3 प्रासंगिक कीवर्ड, विशेष रूप से [Plan ₿ Network टैग सूची से](https://github.com/PlanB-Network/Bitcoin-educational-content/blob/dev/docs/50-planb-tags.md) चुने गए;



- श्रेणी** : Plan ₿ Network वेबसाइट संरचना के अनुसार ट्यूटोरियल सामग्री के अनुरूप उप-श्रेणी (उदाहरण के लिए, वॉलेट के लिए: `डेस्कटॉप`, `हार्डवेयर`, `मोबाइल`, `बैकअप`);



- स्तर** : ट्यूटोरियल का कठिनाई स्तर, निम्न में से चुना गया:
    - `शुरुआती`
    - `मध्यवर्ती`
    - `उन्नत`
    - `विशेषज्ञ`



- professor_id** : आपका `professor_id` (UUID) जैसा कि [आपके प्रोफेसर प्रोफ़ाइल](https://github.com/PlanB-Network/Bitcoin-educational-content/tree/dev/professors) पर प्रदर्शित है;



- original_language** : ट्यूटोरियल की मूल भाषा (जैसे, `fr`, `en`, आदि);



- प्रूफ़रीडिंग** : प्रूफ़रीडिंग प्रक्रिया के बारे में जानकारी। पहला भाग पूरा करें, क्योंकि अपने स्वयं के ट्यूटोरियल को प्रूफ़रीडिंग करना प्रथम सत्यापन के रूप में गिना जाता है:
    - भाषा** : प्रूफ़रीडिंग का भाषा कोड (जैसे, `fr`, `en`, आदि).
    - last_contribution_date** : दिन की तारीख.
    - तात्कालिकता** : 1
    - योगदानकर्ता_नाम** : आपकी GitHub आईडी.
    - इनाम** : 0


अपने शिक्षक आईडी के बारे में अधिक जानकारी के लिए कृपया संबंधित ट्यूटोरियल देखें:


https://planb.network/tutorials/contribution/others/create-teacher-profile-8ba9ba49-8fac-437a-a435-c38eebc8f8a4

```
id: e84edaa9-fb65-48c1-a357-8a5f27996143

project_id: 3b2f45e6-d612-412c-95ba-cf65b49aa5b8

tags:
- wallets
- software
- keys

category: mobile

level: beginner

professor_id: 6516474c-c190-41f2-b2ab-3d452ce7bdf0

# Proofreading metadata

original_language: fr
proofreading:
- language: fr
last_contribution_date: 2024-11-20
urgency: 1
contributor_names:
- LoicPandul
reward: 0
```


एक बार जब आप अपनी `tutorial.yml` फ़ाइल को संशोधित करना समाप्त कर लें, तो "*Commit changes...*" बटन पर क्लिक करके अपने दस्तावेज़ को सहेजें:


![GITHUB](assets/fr/09.webp)


शीर्षक और विवरण जोड़ें, और सुनिश्चित करें कि इस ट्यूटोरियल की शुरुआत में आपके द्वारा बनाई गई शाखा में कमिट किया गया है। फिर "*कमिट परिवर्तन*" पर क्लिक करके पुष्टि करें।


![GITHUB](assets/fr/10.webp)


## 4 - छवियों के लिए सबफ़ोल्डर्स बनाना


पुनः "*फ़ाइल जोड़ें*" पर क्लिक करें और फिर "*नई फ़ाइल बनाएँ*" पर क्लिक करें:


![GITHUB](assets/fr/11.webp)


फ़ोल्डर बनाने के लिए `assets` के बाद स्लैश `/` दर्ज करें:


![GITHUB](assets/fr/12.webp)


भाषा सबफ़ोल्डर बनाने के लिए `/assets` फ़ोल्डर में इस चरण को दोहराएं, उदाहरण के लिए `fr` यदि आपका ट्यूटोरियल फ्रेंच में है:


![GITHUB](assets/fr/13.webp)


इस फ़ोल्डर में, GitHub को आपका फ़ोल्डर रखने के लिए बाध्य करने के लिए एक डमी फ़ाइल बनाएँ (जो अन्यथा खाली होगा)। इस फ़ाइल का नाम `.gitkeep` रखें। फिर "*Commit changes...*" पर क्लिक करें।


![GITHUB](assets/fr/14.webp)


पुनः जाँच लें कि आप सही शाखा पर हैं, फिर "*परिवर्तन प्रतिबद्ध करें*" पर क्लिक करें।

## 6 - लोगो और कवर जोड़ें


`assets` फ़ोल्डर में, आपको `logo.webp` नामक फ़ाइल जोड़नी होगी, जो आपके लेख के लिए थंबनेल के रूप में काम करेगी। यह छवि `.webp` प्रारूप में होनी चाहिए, और उपयोगकर्ता Interface से मेल खाने के लिए आकार में चौकोर होनी चाहिए।


आप ट्यूटोरियल में इस्तेमाल किए गए सॉफ़्टवेयर लोगो या किसी अन्य प्रासंगिक छवि को चुनने के लिए स्वतंत्र हैं, बशर्ते वह रॉयल्टी-मुक्त हो। इसके अलावा, उसी स्थान पर `cover.webp` नामक एक छवि जोड़ें। यह आपके ट्यूटोरियल के शीर्ष पर प्रदर्शित होगी। सुनिश्चित करें कि यह छवि, लोगो की तरह, उपयोग के अधिकारों का सम्मान करती है और आपके ट्यूटोरियल के संदर्भ के लिए उपयुक्त है।


`/assets` फ़ोल्डर में छवियाँ जोड़ने के लिए, आप उन्हें अपनी स्थानीय फ़ाइलों से खींचकर छोड़ सकते हैं। सुनिश्चित करें कि आप `/assets` फ़ोल्डर में हैं और दाईं शाखा पर हैं, फिर "*Commit changes*" पर क्लिक करें।

## 8 - दृश्य जोड़ें


`/assets` फ़ोल्डर में भाषा सबफ़ोल्डर (मेरे उदाहरण में: `/assets/en`) का उपयोग आपके ट्यूटोरियल के साथ आने वाले आरेखों और दृश्यों को संग्रहीत करने के लिए किया जाता है। जहाँ तक संभव हो, अपनी सामग्री को अंतर्राष्ट्रीय दर्शकों के लिए सुलभ बनाने के लिए अपनी छवियों में पाठ शामिल करने से बचें। बेशक, प्रस्तुत सॉफ़्टवेयर में पाठ होगा, लेकिन यदि आप सॉफ़्टवेयर स्क्रीनशॉट पर योजनाबद्ध या अतिरिक्त संकेत जोड़ते हैं, तो ऐसा बिना पाठ के करें या, यदि आवश्यक हो, तो अंग्रेजी का उपयोग करें।


अपनी छवियों को नाम देने के लिए, ट्यूटोरियल में उनकी उपस्थिति के क्रम के अनुरूप संख्याओं का उपयोग करें, जिन्हें दो अंकों के रूप में प्रारूपित किया गया है (या तीन अंक यदि आपके ट्यूटोरियल में 99 से अधिक छवियां हैं)। उदाहरण के लिए, अपनी पहली छवि का नाम `01.webp`, अपनी दूसरी छवि का नाम `02.webp`, और इसी तरह रखें।


आपकी छवियाँ केवल `.webp` प्रारूप में होनी चाहिए। यदि आवश्यक हो, तो आप [मेरे छवि रूपांतरण सॉफ़्टवेयर](https://github.com/LoicPandul/ImagesConverter) का उपयोग कर सकते हैं।


![GITHUB](assets/fr/32.webp)


अब जब आपने अपनी छवियों को सबफ़ोल्डर में जोड़ दिया है, तो आप डमी फ़ाइल `.gitkeep` को हटा सकते हैं। इस फ़ाइल को खोलें, ऊपरी दाएँ कोने में तीन छोटे बिंदुओं पर क्लिक करें, फिर "*फ़ाइल हटाएँ*" पर क्लिक करें।


![GITHUB](assets/fr/33.webp)


"*परिवर्तन प्रतिबद्ध करें...*" पर क्लिक करके अपने परिवर्तन सहेजें।


![GITHUB](assets/fr/34.webp)


अपने सबफ़ोल्डर से अपने संपादकीय दस्तावेज़ में आरेख सम्मिलित करने के लिए, निम्नलिखित मार्कडाउन कमांड का उपयोग करें, अपनी भाषा के लिए उपयुक्त वैकल्पिक पाठ और सही छवि पथ निर्दिष्ट करने का ध्यान रखें:


![GITHUB](assets/fr/35.webp)

## 9 - ट्यूटोरियल का प्रस्ताव करें


एक बार जब आप अपनी पसंद की भाषा में अपना ट्यूटोरियल लिखना समाप्त कर लेते हैं, तो अगला चरण **पुल रिक्वेस्ट** सबमिट करना होता है। फिर व्यवस्थापक मानव प्रूफ़रीडिंग के साथ हमारी स्वचालित अनुवाद पद्धति का उपयोग करके आपके ट्यूटोरियल में अनुपलब्ध अनुवाद जोड़ देगा।


पुल अनुरोध के साथ आगे बढ़ने के लिए, अपने सभी परिवर्तनों को सहेजने के बाद, "*योगदान करें*" बटन पर क्लिक करें, फिर "*पुल अनुरोध खोलें*" पर क्लिक करें:


![GITHUB](assets/fr/36.webp)


पुल अनुरोध आपके शाखा से परिवर्तनों को Plan ₿ Network रिपोजिटरी की मुख्य शाखा में एकीकृत करने के लिए किया गया अनुरोध है, जो विलय से पहले परिवर्तनों की समीक्षा और चर्चा की अनुमति देता है।


आगे बढ़ने से पहले, Interface के निचले भाग को ध्यानपूर्वक जांच लें कि क्या ये परिवर्तन वही हैं जिनकी आपको अपेक्षा थी:


![GITHUB](assets/fr/37.webp)


सुनिश्चित करें कि Interface के शीर्ष पर आपकी कार्यशील शाखा Plan ₿ Network रिपोजिटरी (जो कि मुख्य शाखा है) की `dev` शाखा में विलीन हो गई है।


एक शीर्षक दर्ज करें जो उन परिवर्तनों का संक्षिप्त सारांश देता है जिन्हें आप स्रोत रिपॉजिटरी के साथ मर्ज करना चाहते हैं। इन परिवर्तनों का वर्णन करते हुए एक संक्षिप्त टिप्पणी जोड़ें (यदि आपके पास अपने ट्यूटोरियल के निर्माण से संबंधित कोई समस्या संख्या है, तो टिप्पणी के रूप में `Closes #{issue number}` को नोट करना याद रखें), फिर मर्ज अनुरोध की पुष्टि करने के लिए Green "*पुल अनुरोध बनाएँ*" बटन पर क्लिक करें:


![GITHUB](assets/fr/38.webp)


फिर आपका PR मुख्य Plan ₿ Network रिपॉजिटरी के "*पुल रिक्वेस्ट*" टैब में दिखाई देगा। अब आपको बस इतना करना है कि जब तक कोई व्यवस्थापक आपसे संपर्क करके यह पुष्टि न कर दे कि आपका योगदान मर्ज हो गया है, या कोई और संशोधन करने का अनुरोध न कर दे, तब तक प्रतीक्षा करें।


![GITHUB](assets/fr/39.webp)


अपने PR को मुख्य शाखा के साथ मर्ज करने के बाद, हम आपके Fork का साफ़ इतिहास बनाए रखने के लिए आपकी कार्यशील शाखा (मेरे उदाहरण में: `tuto-Green-Wallet`) को हटाने की सलाह देते हैं। GitHub स्वचालित रूप से आपको आपके PR पृष्ठ पर यह विकल्प प्रदान करेगा:


![GITHUB](assets/fr/40.webp)


यदि आप अपना पी.आर. जमा करने के बाद अपने योगदान में परिवर्तन करना चाहते हैं, तो निम्नलिखित चरण आपके पी.आर. की वर्तमान स्थिति पर निर्भर करेंगे:




- यदि आपका PR अभी भी खुला है और अभी तक मर्ज नहीं हुआ है, तो उसी वर्कब्रांच पर बदलाव करें। कमिट किए गए बदलाव आपके अभी भी खुले PR में जोड़े जाएँगे;
- यदि आपका PR पहले ही मुख्य शाखा के साथ मर्ज हो चुका है, तो आपको एक नई शाखा बनाकर, फिर एक नया PR सबमिट करके प्रक्रिया को शुरू से ही फिर से करना होगा। आगे बढ़ने से पहले सुनिश्चित करें कि आपका Fork `dev` शाखा पर Plan ₿ Network स्रोत रिपॉजिटरी के साथ सिंक्रोनाइज़ है।
