## 1. Что такое груши?


Pears - это одновременно среда выполнения, инструмент разработки и платформа для развертывания одноранговых приложений. Этот инструмент с открытым исходным кодом позволяет создавать, совместно использовать и запускать программное обеспечение без серверов или инфраструктуры, напрямую между пользователями. На практике это означает, что вместо размещения приложения на центральном сервере каждый пользователь становится узлом в сети: он делится частью приложения и данными с другими пользователями. Вся система образует распределенную сеть, где каждый экземпляр сотрудничает, чтобы поддерживать доступ к сервису.


![Image](assets/fr/01.webp)


Этот подход основан на наборе модульных программных компонентов, разработанных компанией Holepunch:



- Hypercore**: распределенный журнал, обеспечивающий согласованность и безопасность данных без центральной базы данных.
- Hyperbee**: индекс, построенный на основе Hypercore, который позволяет эффективно организовывать и запрашивать данные.
- Hyperdrive**: распределенная файловая система, используемая для хранения и синхронизации файлов приложений между пирами.
- Hyperswarm** и **HyperDHT**: сетевые уровни, позволяющие находить пиров и устанавливать соединения по всему миру без центрального сервера.
- Secretstream**: протокол сквозного шифрования, обеспечивающий безопасность общения между пирами.


Объединяя эти компоненты, Pears позволяет создавать автономные, зашифрованные и распределенные приложения, в которых каждый пользователь активно участвует в работе сети. Эта децентрализованная архитектура исключает затраты на инфраструктуру, риски цензуры и SPOF (*одиночные точки отказа*).


Pears разработана компанией Holepunch, основанной Матиасом Буусом и Паоло Ардойно (CEO Tether и CTO Bitfinex) с целью расширить логику пиринга за пределы Bitcoin. Их цель - построить "*Интернет равных*", где каждое приложение может работать без разрешения, без серверов и без посредников. Holepunch уже стоит за **Keet**, приложением для видеоконференций и обмена сообщениями, полностью соответствующим P2P.


https://planb.academy/tutorials/computer-security/communication/keet-efdb759d-5e94-4bbf-b28c-5fa8669c809b

## 2. Как установить Pears на Linux (Debian)?


Установка Pears на Debian относительно проста, но требует нескольких предварительных условий, о которых мы расскажем в этом разделе.


### 2.1. Обновите систему


Прежде всего, необходимо убедиться в том, что ваша система находится в актуальном состоянии.


```bash
sudo apt update && sudo apt upgrade -y
```


![Image](assets/fr/02.webp)


### 2.2. Установите зависимости


Pears использует некоторые системные библиотеки, в том числе `libatomic1`, используемую движком Bare JavaScript runtime. Установите ее с помощью следующей команды:


```bash
sudo apt install -y libatomic1 curl git
```


![Image](assets/fr/03.webp)


### 2.3. Установите Node.js и npm через NVM


Pears распространяется через *npm*, менеджер пакетов *Node.js*. Хотя работа Pears напрямую не зависит от *Node.js*, он необходим для установки. Рекомендуемый способ установки *Node.js* в Linux - через *NVM* (*Node Version Manager*), который позволяет вам управлять несколькими версиями Node бок о бок.


```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/master/install.sh | bash
```


![Image](assets/fr/04.webp)


Затем перезагрузите терминал, чтобы активировать *NVM*:


```bash
source ~/.bashrc
```


![Image](assets/fr/05.webp)


Проверьте правильность установки *NVM*:


```bash
nvm --version
```


![Image](assets/fr/06.webp)


Затем установите стабильную версию *Node.js* (например, текущую версию LTS):


```bash
nvm install --lts
```


![Image](assets/fr/07.webp)


Убедитесь, что *Node.js* и *npm* установлены правильно:


```bash
node -v
npm -v
```


![Image](assets/fr/08.webp)


### 2.4. Установите Pears с помощью npm


Как только *npm* станет доступен, вы сможете глобально установить Pears CLI в своей системе. Это позволит вам запускать команду `pear` из любой директории.


```bash
npm install -g pear
```


![Image](assets/fr/09.webp)


### 2.5. Инициализация груш


После установки просто выполните следующую команду в терминале:


```bash
pear
```


При первом запуске Pears подключится к пиринговой сети, чтобы загрузить необходимые компоненты. Этот процесс не зависит от какого-либо центрального сервера - файлы получаются напрямую от других пиров.


![Image](assets/fr/10.webp)


После завершения загрузки запустите команду еще раз, чтобы убедиться, что все работает:


```bash
pear
```


![Image](assets/fr/11.webp)


Если все установлено правильно, появится меню помощи Pears со списком доступных команд.


### 2.6. Тестовые груши с китом


Чтобы убедиться в работоспособности Pears, можно запустить существующее в сети приложение P2P, например Keet, программное обеспечение для обмена сообщениями и видеоконференций с открытым исходным кодом, разработанное компанией Holepunch.


```bash
pear run pear://keet
```


Эта команда загружает приложение Keet непосредственно из сети Pears, без использования центрального сервера. Если Keet запускается корректно, это означает, что ваша установка Pears полностью исправна.


![Image](assets/fr/12.webp)


Теперь ваша система Linux готова к запуску и размещению одноранговых приложений с помощью Pears.


## 3. Как установить Pears на Windows


Установка Pears на Windows так же проста, как и на Linux, но требует нескольких специальных инструментов.


*Если вы используете Linux и уже установили Pears, вы можете сразу перейти к **Шагу 5**.*


### 3.1. Откройте PowerShell от имени администратора


Сначала запустите PowerShell с правами администратора:



- Нажмите на меню "Пуск";
- Введите "PowerShell";
- Щелкните правой кнопкой мыши на "*Windows PowerShell*";
- Выберите "*Запуск от имени администратора*".


![Image](assets/fr/15.webp)


### 3.2. Скачать NVS


Pears устанавливается с помощью *npm*, менеджера пакетов *Node.js*. На Windows метод, рекомендуемый Holepunch, заключается в использовании *NVS* (*Node Version Switcher*), который более стабилен, чем *NVM* на этой системе.


В PowerShell выполните следующую команду, чтобы установить последнюю версию *NVS*:


```PowerShell
winget install jasongin.nvs
```


![Image](assets/fr/16.webp)


### 3.3. Установите Node.js


После установки перезапустите PowerShell, а затем введите следующую команду:


```powershell
nvs
```


Вы увидите список доступных версий *Node.js*. Выберите первую из них, нажав клавишу `a` на клавиатуре.


![Image](assets/fr/17.webp)


*Node.js* теперь установлен.


![Image](assets/fr/18.webp)


### 3.4. Проверьте установку


Убедитесь, что *Node.js* и *npm* доступны:


```powershell
node -v
npm -v
```


Обе команды должны вернуть номер версии.


![Image](assets/fr/19.webp)


### 3.5. Установите Pears с помощью npm


Как только *Node.js* и *npm* будут доступны, установите **Pears CLI** глобально на вашу систему:


```powershell
npm install -g pear
```


Это установит бинарник `pear` в вашу глобальную директорию *npm*.


![Image](assets/fr/20.webp)


### 3.6. Проверка и инициализация груш


После завершения установки запустите его:


```powershell
pear
```


При первом запуске Pears автоматически загрузит необходимые компоненты из пиринговой сети. Этот процесс может занять несколько минут.


![Image](assets/fr/21.webp)


Если все прошло успешно, вы должны увидеть меню помощи Pears CLI со списком доступных подкоманд (run, seed, info...).


### 3.7. Тестовые груши с китом


Чтобы убедиться в работоспособности Pears, можно запустить существующее приложение P2P, доступное в сети, например Keet - программное обеспечение для обмена сообщениями и видеоконференций с открытым исходным кодом, разработанное компанией Holepunch.


```bash
pear run pear://keet
```


Эта команда загружает приложение Keet непосредственно из сети Pears, без использования какого-либо центрального сервера. Если Keet успешно запускается, это означает, что ваша установка Pears полностью функциональна.


![Image](assets/fr/22.webp)


Теперь ваша система Windows готова к запуску и размещению одноранговых приложений с помощью Pears.


## 4. Как установить Pears на macOS


Установка Pears на macOS аналогична Linux, но требует нескольких настроек, характерных для среды Apple. Давайте пройдемся по этим шагам вместе.


*Если вы используете Linux или Windows и уже установили Pears, вы можете перейти непосредственно к **Шагу 5**.*


### 4.1. Проверьте предварительные условия системы


Перед установкой убедитесь, что в вашей системе установлены *Xcode Command Line Tools*. Этот пакет предоставляет необходимые инструменты сборки для *Node.js* и его зависимостей.


Для этого откройте терминал с помощью сочетания клавиш `Cmd + Пробел`, введите `Terminal` и нажмите `Enter`. Затем выполните в терминале следующую команду для установки:


```bash
xcode-select --install
```


Если инструменты уже установлены в вашей системе, macOS уведомит вас об этом.


### 4.2. Установите NVM


Pears распространяется через *npm*, менеджер пакетов *Node.js*. Хотя работа Pears напрямую не зависит от *Node.js*, он необходим для установки. Рекомендуемый метод установки *Node.js* на macOS - *NVM* (*Node Version Manager*), который позволяет управлять несколькими версиями Node одновременно.


```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/master/install.sh | bash
```


Затем перезагрузите терминал, чтобы активировать *NVM*:


```bash
source ~/.zshrc
```


Если вы используете *bash* вместо *zsh*, выполните команду:


```bash
source ~/.bashrc
```


Затем проверьте правильность установки *NVM*:


```bash
nvm --version
```


Ваш терминал должен отобразить версию установленной *NVM*.


### 4.3. Установите Node.js и npm


Затем установите стабильную версию *Node.js* (например, текущую версию LTS):


```bash
nvm install --lts
```


После завершения установки проверьте установленные версии:


```bash
node -v
npm -v
```


Обе команды должны вернуть номер версии.


### 4.4. Установите Pears с помощью npm


Как только *npm* станет доступен, вы сможете глобально установить Pears CLI в своей системе. Это позволит вам выполнять команду `pear` из любой директории.


```bash
npm install -g pear
```


### 4.5. Инициализация груш


После установки просто выполните следующую команду в терминале:


```bash
pear
```


При первом запуске Pears подключается к пиринговой сети, чтобы загрузить необходимые компоненты. Этот процесс не требует наличия центрального сервера - файлы берутся напрямую у других пиров.


После завершения загрузки повторно выполните команду, чтобы убедиться, что все работает:


```bash
pear
```


Если все установлено правильно, появится справочное меню Pears со списком доступных команд.


### 4.6. Тестовые груши с китом


Чтобы убедиться в работоспособности Pears, можно запустить уже имеющееся в сети приложение P2P, например Keet, программное обеспечение для обмена сообщениями и видеоконференций с открытым исходным кодом от компании Holepunch.


```bash
pear run pear://keet
```


Эта команда загружает приложение Keet непосредственно из сети Pears, без использования центрального сервера. Если Keet успешно запустится, это будет означать, что ваша установка Pears полностью функциональна.


Теперь ваша система macOS готова к запуску и размещению одноранговых приложений с помощью Pears.

## 6. Как посеять семена ₿ Академия на грушах


В сети Pears *seed* приложение означает перераспределение его среди других пользователей с вашего собственного компьютера. На практике, когда вы seed Plan ₿ Academy, ваш компьютер становится источником данных, который позволяет другим пользователям загружать приложение, не полагаясь на центральный сервер.


Этот механизм повышает устойчивость и цензуроустойчивость нашего приложения в сети Pears. Чем больше пиров seed у приложения, тем более доступным и децентрализованным оно становится, даже если некоторые исходные узлы уйдут в офлайн.


Чтобы помочь распространить Plan ₿ Academy, просто выполните следующую команду:


```bash
pear seed pear://k9cawqdsan3bkobkigesuyfeqjcasi49ikjaru5cipap835t7nwy
```


Пока эта команда остается активной, ваше устройство будет участвовать в распространении файлов приложения. Если вы закроете терминал, процесс обмена прекратится.


Чтобы продолжить посев после перезапуска, можно запустить команду в фоновом режиме или создать системную службу - например, службу systemd в Linux, LaunchAgent в macOS или запланированную задачу в Windows. Эти методы гарантируют, что приложение Plan ₿ Academy автоматически возобновит посев при запуске системы.
