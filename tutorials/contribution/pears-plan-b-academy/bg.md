## 1. Какво е Pears?


Pears е едновременно среда за изпълнение, инструмент за разработка и платформа за разгръщане на peer-to-peer приложения. Този инструмент с отворен код ви позволява да създавате, споделяте и изпълнявате софтуер без сървъри или инфраструктура, директно между потребителите. На практика това означава, че вместо да хоства приложението на централен сървър, всеки потребител се превръща във възел в мрежата: той споделя част от приложението и данните с други равноправни потребители. Цялата система образува разпределена мрежа, в която всеки екземпляр си сътрудничи, за да поддържа услугата достъпна.


![Image](assets/fr/01.webp)


Този подход се основава на набор от модулни софтуерни компоненти, разработени от Holepunch:



- Hypercore**: разпределен дневник, който осигурява последователност и сигурност на данните без централна база данни.
- Hyperbee**: индекс, изграден върху Hypercore, който позволява ефективно организиране и търсене на данни.
- Hyperdrive**: разпределена файлова система, използвана за съхраняване и синхронизиране на файлове на приложения между равнопоставени потребители.
- Hyperswarm** и **HyperDHT**: мрежови слоеве, които позволяват откриването на равнопоставени потребители и свързването им по целия свят без централен сървър.
- Secretstream**: протокол за криптиране "от край до край", който защитава комуникацията между равнопоставени потребители.


Чрез комбинирането на тези компоненти Pears позволява създаването на автономни, криптирани и разпределени приложения, в които всеки потребител участва активно в мрежата. Тази децентрализирана архитектура елиминира инфраструктурните разходи, рисковете от цензура и SPOF (*Single Points of Failure*).


Pears се разработва от Holepunch, компания, основана от Матиас Буус и Паоло Ардоино (главен изпълнителен директор на Tether и главен технически директор на Bitfinex), с мисията да разшири логиката на peer-to-peer отвъд Bitcoin. Тяхната амбиция е да изградят "*Интернет на връстниците*", където всяко приложение може да работи без разрешение, без сървъри и без посредници. Holepunch вече стои зад **Keet**, изцяло P2P приложение за видеоконференции и съобщения.


https://planb.academy/tutorials/computer-security/communication/keet-efdb759d-5e94-4bbf-b28c-5fa8669c809b

## 2. Как да инсталирам Pears под Linux (Debian)?


Инсталирането на Pears на Debian е сравнително лесно, но изисква няколко предварителни условия, които ще опишем подробно в този раздел.


### 2.1. Актуализиране на системата


Преди всичко друго е важно да се уверите, че системата ви е актуализирана.


```bash
sudo apt update && sudo apt upgrade -y
```


![Image](assets/fr/02.webp)


### 2.2. Инсталиране на зависимости


Pears разчита на някои системни библиотеки, включително `libatomic1`, използвани от изпълнителния механизъм на Bare JavaScript. Инсталирайте я със следната команда:


```bash
sudo apt install -y libatomic1 curl git
```


![Image](assets/fr/03.webp)


### 2.3. Инсталиране на Node.js и npm чрез NVM


Pears се разпространява чрез *npm*, мениджъра на пакети *Node.js*. Въпреки че работата на Pears не зависи пряко от *Node.js*, той е необходим за инсталиране. Препоръчителният начин за инсталиране на *Node.js* под Linux е чрез *NVM* (*Node Version Manager*), който ви позволява да управлявате няколко версии на Node една до друга.


```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/master/install.sh | bash
```


![Image](assets/fr/04.webp)


След това презаредете терминала, за да активирате *NVM*:


```bash
source ~/.bashrc
```


![Image](assets/fr/05.webp)


Проверете дали *NVM* е правилно инсталиран:


```bash
nvm --version
```


![Image](assets/fr/06.webp)


След това инсталирайте стабилна версия на *Node.js* (например текущата версия LTS):


```bash
nvm install --lts
```


![Image](assets/fr/07.webp)


Проверете дали *Node.js* и *npm* са правилно инсталирани:


```bash
node -v
npm -v
```


![Image](assets/fr/08.webp)


### 2.4. Инсталирайте Pears с npm


След като *npm* е наличен, можете да инсталирате Pears CLI на вашата система. Това ви позволява да изпълнявате командата `pear` от всяка директория.


```bash
npm install -g pear
```


![Image](assets/fr/09.webp)


### 2.5. Иницииране на Pears


След инсталацията просто изпълнете следната команда в терминала:


```bash
pear
```


При първото стартиране Pears ще се свърже с партньорската мрежа, за да изтегли необходимите компоненти. Този процес не разчита на никакъв централен сървър - файловете се изтеглят директно от други равноправни потребители.


![Image](assets/fr/10.webp)


След като изтеглянето завърши, изпълнете командата отново, за да потвърдите, че всичко работи:


```bash
pear
```


![Image](assets/fr/11.webp)


Ако всичко е инсталирано правилно, ще се появи помощното меню на Pears със списък на наличните команди.


### 2.6. Изпитване на Pears с Keet


За да проверите дали Pears работи пълноценно, можете да стартирате съществуващо приложение P2P, налично в мрежата, като например Keet, софтуер за съобщения и видеоконференции с отворен код, разработен от Holepunch.


```bash
pear run pear://keet
```


Тази команда зарежда приложението Keet директно от мрежата на Pears, без да използва централен сървър. Ако Keet се стартира правилно, това означава, че вашата инсталация на Pears е напълно функционална.


![Image](assets/fr/12.webp)


Вашата Linux система вече е готова да стартира и хоства peer-to-peer приложения с Pears.


## 3. Как да инсталирате Pears в Windows


Инсталирането на Pears в Windows е също толкова лесно, колкото и в Linux, но изисква няколко специфични инструмента.


*Ако използвате Linux и вече сте инсталирали Pears, можете да преминете директно към **Стъпка 5**.*


### 3.1. Отворете PowerShell като администратор


Първо, стартирайте PowerShell с привилегии на администратор:



- Кликнете върху менюто Start;
- Въведете "PowerShell";
- Кликнете с десния бутон на мишката върху "*Windows PowerShell*";
- Изберете "*Задействай като администратор*".


![Image](assets/fr/15.webp)


### 3.2. Изтегляне на NVS


Pears се инсталира чрез *npm*, мениджъра на пакети *Node.js*. В Windows методът, препоръчан от Holepunch, е да се използва *NVS* (*Node Version Switcher*), който е по-стабилен от *NVM* в тази система.


В PowerShell изпълнете следната команда, за да инсталирате най-новата версия на *NVS*:


```PowerShell
winget install jasongin.nvs
```


![Image](assets/fr/16.webp)


### 3.3. Инсталиране на Node.js


След инсталацията рестартирайте PowerShell, след което въведете следната команда:


```powershell
nvs
```


Трябва да видите списък с наличните версии на *Node.js*. Изберете първата от тях, като натиснете клавиша `a` на клавиатурата си.


![Image](assets/fr/17.webp)


*Node.js* вече е инсталиран.


![Image](assets/fr/18.webp)


### 3.4. Проверка на инсталациите


Уверете се, че *Node.js* и *npm* са достъпни:


```powershell
node -v
npm -v
```


И двете команди трябва да върнат номер на версията.


![Image](assets/fr/19.webp)


### 3.5. Инсталиране на Pears с npm


След като *Node.js* и *npm* са налични, инсталирайте **Pears CLI** глобално на вашата система:


```powershell
npm install -g pear
```


Това инсталира бинарната програма `pear` в глобалната директория *npm*.


![Image](assets/fr/20.webp)


### 3.6. Проверка и инициализация на Pears


След като инсталацията приключи, стартирайте:


```powershell
pear
```


При първото стартиране Pears автоматично изтегля необходимите компоненти от партньорската мрежа. Този процес може да отнеме няколко минути.


![Image](assets/fr/21.webp)


Ако всичко е минало добре, трябва да видите помощното меню Pears CLI със списъка на наличните подкоманди (run, seed, info...).


### 3.7. Изпитване на Pears с Keet


За да проверите дали Pears работи пълноценно, можете да стартирате съществуващо приложение P2P, налично в мрежата, например Keet - софтуер за съобщения и видеоконференции с отворен код, разработен от Holepunch.


```bash
pear run pear://keet
```


Тази команда зарежда приложението Keet директно от мрежата на Pears, без да използва централен сървър. Ако Keet се стартира успешно, това означава, че вашата инсталация на Pears е напълно функционална.


![Image](assets/fr/22.webp)


Вашата система Windows вече е готова да изпълнява и хоства peer-to-peer приложения с Pears.


## 4. Как да инсталирате Pears на macOS


Инсталирането на Pears в macOS е подобно на това в Linux, но изисква няколко настройки, специфични за средата на Apple. Нека преминем през тези стъпки заедно.


*Ако използвате Linux или Windows и вече сте инсталирали Pears, можете да преминете директно към **Стъпка 5**.*


### 4.1. Проверка на предварителните условия на системата


Преди да инсталирате, уверете се, че в системата ви са инсталирани *Xcode Command Line Tools*. Този пакет предоставя необходимите инструменти за изграждане на *Node.js* и неговите зависимости.


За да направите това, отворете терминал, като използвате бързата клавишна комбинация `Cmd + Space bar`, въведете `Terminal` и натиснете `Enter`. След това изпълнете следната команда в терминала, за да я инсталирате:


```bash
xcode-select --install
```


Ако инструментите вече са инсталирани в системата ви, macOS ще ви уведоми за това.


### 4.2. Инсталиране на NVM


Pears се разпространява чрез *npm*, мениджъра на пакети *Node.js*. Въпреки че функционирането на Pears не зависи пряко от *Node.js*, той е необходим за инсталиране. Препоръчителният метод за инсталиране на *Node.js* на macOS е *NVM* (*Node Version Manager*), който ви позволява да управлявате няколко версии на Node едновременно.



След това презаредете терминала си, за да активирате *NVM*:


```bash

```


Ако използвате *bash* вместо *zsh*, изпълнете:


```bash

```


След това проверете дали *NVM* е инсталиран правилно:


```bash
nvm --version
```


Терминалът ви трябва да покаже инсталираната версия на *NVM*.

## 6. Как да посеете план ₿ Академия на Pears


В мрежата Pears да *seed* дадено приложение означава да го преразпределите към други равнопоставени потребители от собствената си машина. На практика, когато seed Plan ₿ Academy, вашият компютър се превръща в източник на данни, който позволява на другите потребители да изтеглят приложението, без да разчитат на централен сървър.


Този механизъм засилва устойчивостта и цензурата на нашето приложение в мрежата Pears. Колкото повече връстници seed има едно приложение, толкова по-достъпно и децентрализирано става то, дори ако някои оригинални възли излязат от мрежата.


За да подпомогнете разпространението на Plan ₿ Academy, просто изпълнете следната команда:


```bash
pear seed pear://k9cawqdsan3bkobkigesuyfeqjcasi49ikjaru5cipap835t7nwy
```


Докато тази команда е активна, устройството ви ще участва в разпространението на файловете на приложението. Ако затворите терминала, процесът на споделяне ще спре.


За да продължите посяването след рестартиране, можете да стартирате командата във фонов режим или да създадете системна услуга - например systemd услуга в Linux, LaunchAgent в macOS или планирана задача в Windows. Тези методи гарантират, че приложението Plan ₿ Academy автоматично възобновява посявката при стартиране на системата.
