आप शायद पहले से ही जानते होंगे कि प्लान ₿ अकादमी, Bitcoin को समर्पित सबसे बड़ा शैक्षिक डेटाबेस है, जो पाठ्यक्रम, ट्यूटोरियल और हज़ारों ओपन-सोर्स संसाधनों को एक साथ लाता है। मूल रूप से, प्लान ₿ अकादमी एक वेबसाइट थी। लेकिन क्या होगा अगर आप इसे सामान्य रूप से एक्सेस नहीं कर पाएँ, उदाहरण के लिए सेंसरशिप की स्थिति में?


इस ट्यूटोरियल में, हम सीखेंगे कि **Plan ₿ Academy** प्लेटफॉर्म को **Pears** का उपयोग करके वास्तव में सेंसरशिप-प्रतिरोधी तरीके से कैसे चलाया जाए, जो कि **Holepunch** द्वारा विकसित और **Tether** द्वारा समर्थित एक पीयर-टू-पीयर (P2P) तकनीक है।


पीयर्स एक ऐसा सॉफ़्टवेयर है जो हमें किसी केंद्रीकृत वेबसाइट पर निर्भर हुए बिना प्लान ₿ अकादमी प्लेटफ़ॉर्म चलाने की सुविधा देता है। इस ट्यूटोरियल में, हम आपके कंप्यूटर पर पीयर्स इंस्टॉल करेंगे ताकि आप पीयर्स के ज़रिए प्लान ₿ अकादमी तक पहुँच सकें।


पीयर्स का लक्ष्य सरल है: किसी भी केंद्रीकृत बुनियादी ढाँचे (बिना सर्वर, बिना होस्ट, बिना मध्यस्थ) पर निर्भर हुए वेब एप्लिकेशन का वितरण और उपयोग संभव बनाना। दूसरे शब्दों में, भले ही कोई क्लाउड प्रदाता बंद हो जाए या कोई देश किसी डोमेन को ब्लॉक कर दे, एप्लिकेशन नेटवर्क के समकक्षों के बीच बना रहता है। यह दृष्टिकोण हमारे शैक्षिक प्लेटफ़ॉर्म, प्लान ₿ अकादमी, को बिना किसी रुकावट के, दुनिया भर में सुलभ बनाए रखने में मदद करता है।

## 1. नाशपाती क्या है?


पीयर्स एक रनटाइम वातावरण, एक विकास उपकरण और पीयर-टू-पीयर अनुप्रयोगों के लिए एक परिनियोजन प्लेटफ़ॉर्म दोनों है। यह ओपन-सोर्स टूल आपको बिना सर्वर या बुनियादी ढाँचे के, सीधे उपयोगकर्ताओं के बीच सॉफ़्टवेयर बनाने, साझा करने और चलाने की अनुमति देता है। व्यावहारिक रूप से, इसका अर्थ है कि किसी एप्लिकेशन को केंद्रीय सर्वर पर होस्ट करने के बजाय, प्रत्येक उपयोगकर्ता नेटवर्क में एक नोड बन जाता है: वे एप्लिकेशन का एक हिस्सा और डेटा अन्य पीयर के साथ साझा करते हैं। पूरा सिस्टम एक वितरित नेटवर्क बनाता है जहाँ प्रत्येक इंस्टेंस सेवा को सुलभ बनाए रखने के लिए सहयोग करता है।


![Image](assets/fr/01.webp)


यह दृष्टिकोण होलपंच द्वारा विकसित मॉड्यूलर सॉफ्टवेयर घटकों के एक सेट पर आधारित है:



- हाइपरकोर**: एक वितरित लॉग जो केंद्रीय डेटाबेस के बिना डेटा की स्थिरता और सुरक्षा सुनिश्चित करता है।
- हाइपरबी**: हाइपरकोर के शीर्ष पर निर्मित एक सूचकांक जो डेटा को कुशलतापूर्वक व्यवस्थित और क्वेरी करने की अनुमति देता है।
- हाइपरड्राइव**: एक वितरित फ़ाइल सिस्टम जिसका उपयोग समकक्षों के बीच अनुप्रयोग फ़ाइलों को संग्रहीत और सिंक्रनाइज़ करने के लिए किया जाता है।
- हाइपरस्वार्म** और **हाइपरडीएचटी**: नेटवर्क परतें जो केंद्रीय सर्वर के बिना दुनिया भर में सहकर्मी खोज और कनेक्शन को सक्षम बनाती हैं।
- सीक्रेटस्ट्रीम**: एक अंत-से-अंत एन्क्रिप्शन प्रोटोकॉल जो साथियों के बीच संचार को सुरक्षित करता है।


इन घटकों को मिलाकर, पीयर्स स्वायत्त, एन्क्रिप्टेड और वितरित अनुप्रयोगों के निर्माण को सक्षम बनाता है, जहाँ प्रत्येक उपयोगकर्ता नेटवर्क में सक्रिय रूप से भाग लेता है। यह विकेन्द्रीकृत आर्किटेक्चर बुनियादी ढाँचे की लागत, सेंसरशिप जोखिमों और एसपीओएफ (*विफलता के एकल बिंदु*) को समाप्त करता है।


पीयर्स को होलपंच द्वारा विकसित किया गया है, जिसकी स्थापना मैथियास बुस और पाओलो अर्दोइनो (टीथर के सीईओ और बिटफाइनेक्स के सीटीओ) ने की थी, जिसका लक्ष्य पीयर-टू-पीयर लॉजिक को Bitcoin से आगे बढ़ाना है। उनकी महत्वाकांक्षा "पीयर्स का इंटरनेट" बनाने की है, जहाँ हर एप्लिकेशन बिना अनुमति, बिना सर्वर और बिना किसी मध्यस्थ के काम कर सके। होलपंच पहले से ही कीट (Keet) नामक एक पूर्ण P2P वीडियो कॉन्फ्रेंसिंग और मैसेजिंग ऐप बना रहा है।


https://planb.academy/tutorials/computer-security/communication/keet-efdb759d-5e94-4bbf-b28c-5fa8669c809b

*यह पीयर्स इंस्टॉलेशन ट्यूटोरियल आपके ऑपरेटिंग सिस्टम के आधार पर कई खंडों में विभाजित है। उपयुक्त निर्देशों का पालन करने के लिए सीधे अपने परिवेश से संबंधित अनुभाग पर जाएँ:*



- लिनक्स (डेबियन)** → अनुभाग **2**
- विंडोज़** → अनुभाग **3**
- macOS** → अनुभाग **4**


## 2. लिनक्स (डेबियन) पर पीयर्स कैसे स्थापित करें?


डेबियन पर पीयर्स को स्थापित करना अपेक्षाकृत सरल है, लेकिन इसके लिए कुछ पूर्वापेक्षाएँ आवश्यक हैं, जिनके बारे में हम इस अनुभाग में विस्तार से बताएंगे।


### 2.1. सिस्टम अपडेट करें


किसी भी अन्य कार्य से पहले, यह सुनिश्चित करना महत्वपूर्ण है कि आपका सिस्टम अद्यतन है।


```bash
sudo apt update && sudo apt upgrade -y
```


![Image](assets/fr/02.webp)


### 2.2. निर्भरताएँ स्थापित करें


पीयर्स कुछ सिस्टम लाइब्रेरीज़ पर निर्भर करता है, जिनमें `libatomic1` भी शामिल है, जिसका इस्तेमाल बेयर जावास्क्रिप्ट रनटाइम इंजन द्वारा किया जाता है। इसे निम्न कमांड से इंस्टॉल करें:


```bash
sudo apt install -y libatomic1 curl git
```


![Image](assets/fr/03.webp)


### 2.3. NVM के माध्यम से Node.js और npm स्थापित करें


Pears को *npm*, *Node.js* पैकेज मैनेजर के माध्यम से वितरित किया जाता है। हालाँकि Pears चलने के लिए सीधे *Node.js* पर निर्भर नहीं है, लेकिन इंस्टॉलेशन के लिए यह आवश्यक है। Linux पर *Node.js* को इंस्टॉल करने का अनुशंसित तरीका *NVM* (*Node Version Manager*) है, जो आपको Node के कई संस्करणों को एक साथ प्रबंधित करने की अनुमति देता है।


```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/master/install.sh | bash
```


![Image](assets/fr/04.webp)


फिर, *NVM* को सक्रिय करने के लिए अपने टर्मिनल को पुनः लोड करें:


```bash
source ~/.bashrc
```


![Image](assets/fr/05.webp)


जाँच करें कि *NVM* ठीक से स्थापित है:


```bash
nvm --version
```


![Image](assets/fr/06.webp)


इसके बाद, *Node.js* का एक स्थिर संस्करण स्थापित करें (उदाहरण के लिए, वर्तमान LTS संस्करण):


```bash
nvm install --lts
```


![Image](assets/fr/07.webp)


सत्यापित करें कि *Node.js* और *npm* ठीक से स्थापित हैं:


```bash
node -v
npm -v
```


![Image](assets/fr/08.webp)


### 2.4. NPM के साथ Pears इंस्टॉल करें


*npm* उपलब्ध होने के बाद, आप अपने सिस्टम पर Pears CLI को वैश्विक रूप से इंस्टॉल कर सकते हैं। इससे आप किसी भी डायरेक्टरी से `pear` कमांड चला सकते हैं।


```bash
npm install -g pear
```


![Image](assets/fr/09.webp)


### 2.5. नाशपाती को आरंभ करें


स्थापना के बाद, अपने टर्मिनल में निम्नलिखित कमांड चलाएँ:


```bash
pear
```


पहली बार लॉन्च होने पर, Pears आवश्यक घटकों को डाउनलोड करने के लिए पीयर-टू-पीयर नेटवर्क से जुड़ जाएगा। यह प्रक्रिया किसी केंद्रीय सर्वर पर निर्भर नहीं करती है - फ़ाइलें सीधे अन्य पीयर सर्वरों से प्राप्त की जाती हैं।


![Image](assets/fr/10.webp)


डाउनलोड पूरा हो जाने पर, सब कुछ ठीक से काम कर रहा है, इसकी पुष्टि के लिए कमांड को पुनः चलाएँ:


```bash
pear
```


![Image](assets/fr/11.webp)


यदि सब कुछ सही ढंग से स्थापित है, तो Pears सहायता मेनू उपलब्ध कमांडों की सूची के साथ दिखाई देगा।


### 2.6. कीट के साथ नाशपाती का परीक्षण करें


यह सत्यापित करने के लिए कि पीयर्स पूरी तरह से चालू है, आप नेटवर्क पर उपलब्ध मौजूदा P2P एप्लिकेशन को लॉन्च कर सकते हैं, जैसे कि कीट, जो होलपंच द्वारा विकसित ओपन-सोर्स मैसेजिंग और वीडियो कॉन्फ्रेंसिंग सॉफ्टवेयर है।


```bash
pear run pear://keet
```


यह कमांड किसी केंद्रीय सर्वर का उपयोग किए बिना, सीधे Pears नेटवर्क से Keet एप्लिकेशन लोड करता है। यदि Keet सही ढंग से लॉन्च होता है, तो इसका मतलब है कि आपका Pears इंस्टॉलेशन पूरी तरह कार्यात्मक है।


![Image](assets/fr/12.webp)


आपका लिनक्स सिस्टम अब Pears के साथ पीयर-टू-पीयर अनुप्रयोगों को चलाने और होस्ट करने के लिए तैयार है।


## 3. विंडोज़ पर पीयर्स कैसे स्थापित करें


विंडोज़ पर पीयर्स को स्थापित करना लिनक्स की तरह ही सरल है, लेकिन इसके लिए कुछ विशिष्ट उपकरणों की आवश्यकता होती है।


*यदि आप लिनक्स का उपयोग कर रहे हैं और पहले से ही पीयर्स स्थापित कर चुके हैं, तो आप सीधे **चरण 5** पर जा सकते हैं।*


### 3.1. PowerShell को व्यवस्थापक के रूप में खोलें


सबसे पहले, व्यवस्थापक विशेषाधिकारों के साथ PowerShell लॉन्च करें:



- स्टार्ट मेनू पर क्लिक करें;
- “PowerShell” टाइप करें;
- "*Windows PowerShell*" पर राइट-क्लिक करें;
- "*व्यवस्थापक के रूप में चलाएँ*" का चयन करें।


![Image](assets/fr/15.webp)


### 3.2. एनवीएस डाउनलोड करें


पीयर्स को *npm*, *Node.js* पैकेज मैनेजर के ज़रिए इंस्टॉल किया जाता है। विंडोज़ पर, होलपंच द्वारा सुझाई गई विधि *NVS* (*नोड वर्जन स्विचर*) का उपयोग करना है, जो इस सिस्टम पर *NVM* से ज़्यादा स्थिर है।


PowerShell में, *NVS* का नवीनतम संस्करण स्थापित करने के लिए निम्न आदेश चलाएँ:


```PowerShell
winget install jasongin.nvs
```


![Image](assets/fr/16.webp)


### 3.3. Node.js इंस्टॉल करें


स्थापना के बाद, PowerShell को पुनः आरंभ करें, फिर निम्न आदेश दर्ज करें:


```powershell
nvs
```


आपको उपलब्ध *Node.js* संस्करणों की एक सूची दिखाई देगी। अपने कीबोर्ड पर `a` कुंजी दबाकर पहला संस्करण चुनें।


![Image](assets/fr/17.webp)


*Node.js* अब स्थापित है.


![Image](assets/fr/18.webp)


### 3.4. इंस्टॉलेशन सत्यापित करें


सुनिश्चित करें कि *Node.js* और *npm* सुलभ हैं:


```powershell
node -v
npm -v
```


दोनों कमांडों को एक संस्करण संख्या लौटानी चाहिए।


![Image](assets/fr/19.webp)


### 3.5. NPM के साथ Pears इंस्टॉल करें


एक बार *Node.js* और *npm* उपलब्ध हो जाने पर, अपने सिस्टम पर **Pears CLI** को वैश्विक रूप से स्थापित करें:


```powershell
npm install -g pear
```


यह आपकी वैश्विक *npm* निर्देशिका में `pear` बाइनरी स्थापित करता है।


![Image](assets/fr/20.webp)


### 3.6. नाशपाती को सत्यापित और आरंभ करें


एक बार स्थापना पूर्ण हो जाने पर, चलाएँ:


```powershell
pear
```


पहली बार लॉन्च होने पर, Pears पीयर-टू-पीयर नेटवर्क से आवश्यक घटकों को स्वचालित रूप से डाउनलोड कर लेगा। इस प्रक्रिया में कुछ समय लग सकता है।


![Image](assets/fr/21.webp)


यदि सब कुछ ठीक रहा, तो आपको उपलब्ध उप-कमांडों (रन, seed, जानकारी...) की सूची के साथ Pears CLI सहायता मेनू दिखाई देगा।


### 3.7. कीट के साथ नाशपाती का परीक्षण करें


यह सत्यापित करने के लिए कि पीयर्स पूरी तरह से चालू है, आप नेटवर्क पर उपलब्ध मौजूदा P2P एप्लिकेशन लॉन्च कर सकते हैं, जैसे कि कीट - होलपंच द्वारा विकसित ओपन-सोर्स मैसेजिंग और वीडियो कॉन्फ्रेंसिंग सॉफ्टवेयर।


```bash
pear run pear://keet
```


यह कमांड किसी भी केंद्रीय सर्वर का उपयोग किए बिना, सीधे Pears नेटवर्क से Keet एप्लिकेशन लोड करता है। यदि Keet सफलतापूर्वक लॉन्च होता है, तो इसका मतलब है कि आपका Pears इंस्टॉलेशन पूरी तरह कार्यात्मक है।


![Image](assets/fr/22.webp)


आपका विंडोज सिस्टम अब Pears के साथ पीयर-टू-पीयर एप्लिकेशन चलाने और होस्ट करने के लिए तैयार है।


## 4. macOS पर Pears कैसे इंस्टॉल करें


MacOS पर Pears इंस्टॉल करना Linux जैसा ही है, लेकिन इसके लिए Apple के परिवेश के अनुसार कुछ समायोजनों की आवश्यकता होती है। आइए इन चरणों को एक साथ देखें।


*यदि आप लिनक्स या विंडोज का उपयोग कर रहे हैं और पहले से ही पीयर्स स्थापित कर चुके हैं, तो आप सीधे **चरण 5** पर जा सकते हैं।*


### 4.1. सिस्टम पूर्वापेक्षाएँ जांचें


इंस्टॉलेशन से पहले, सुनिश्चित करें कि आपके सिस्टम पर *Xcode कमांड लाइन टूल्स* इंस्टॉल हैं। यह पैकेज *Node.js* और उसकी निर्भरताओं के लिए आवश्यक बिल्ड टूल्स प्रदान करता है।


ऐसा करने के लिए, `Cmd + Space bar` शॉर्टकट का उपयोग करके टर्मिनल खोलें, `Terminal` टाइप करें और `Enter` दबाएँ। फिर, इसे इंस्टॉल करने के लिए टर्मिनल में निम्न कमांड चलाएँ:


```bash
xcode-select --install
```


यदि उपकरण आपके सिस्टम पर पहले से ही स्थापित हैं, तो macOS आपको सूचित करेगा।


### 4.2. NVM स्थापित करें


Pears को *npm*, *Node.js* पैकेज मैनेजर के ज़रिए वितरित किया जाता है। हालाँकि Pears काम करने के लिए सीधे *Node.js* पर निर्भर नहीं है, लेकिन इंस्टॉलेशन के लिए यह ज़रूरी है। macOS पर *Node.js* इंस्टॉल करने के लिए सुझाई गई विधि *NVM* (*Node Version Manager*) है, जो आपको एक साथ कई Node वर्जन प्रबंधित करने की सुविधा देती है।


```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/master/install.sh | bash
```


फिर *NVM* को सक्रिय करने के लिए अपने टर्मिनल को पुनः लोड करें:


```bash
source ~/.zshrc
```


यदि आप *zsh* के स्थान पर *bash* का उपयोग करते हैं, तो चलाएँ:


```bash
source ~/.bashrc
```


इसके बाद, जांचें कि *NVM* सही तरीके से स्थापित है:


```bash
nvm --version
```


आपके टर्मिनल को स्थापित *NVM* संस्करण प्रदर्शित करना चाहिए।


### 4.3. Node.js और npm इंस्टॉल करें


इसके बाद, *Node.js* का एक स्थिर संस्करण स्थापित करें (उदाहरण के लिए, वर्तमान LTS संस्करण):


```bash
nvm install --lts
```


स्थापना पूर्ण हो जाने पर, स्थापित संस्करणों को सत्यापित करें:


```bash
node -v
npm -v
```


दोनों कमांडों को एक संस्करण संख्या लौटानी चाहिए।


### 4.4. NPM के साथ Pears इंस्टॉल करें


*npm* उपलब्ध होने के बाद, आप अपने सिस्टम पर Pears CLI को वैश्विक रूप से इंस्टॉल कर सकते हैं। इससे आप किसी भी डायरेक्टरी से `pear` कमांड निष्पादित कर सकेंगे।


```bash
npm install -g pear
```


### 4.5. नाशपाती को आरंभ करें


स्थापना के बाद, अपने टर्मिनल में निम्नलिखित कमांड चलाएँ:


```bash
pear
```


पहली बार लॉन्च होने पर, Pears आवश्यक घटकों को डाउनलोड करने के लिए पीयर-टू-पीयर नेटवर्क से जुड़ता है। इस प्रक्रिया के लिए किसी केंद्रीय सर्वर की आवश्यकता नहीं होती है - फ़ाइलें सीधे अन्य पीयर से प्राप्त की जाती हैं।


डाउनलोड पूरा हो जाने पर, यह सत्यापित करने के लिए कि सब कुछ काम कर रहा है, कमांड को पुनः चलाएँ:


```bash
pear
```


यदि सब कुछ सही ढंग से स्थापित है, तो उपलब्ध कमांडों की सूची के साथ पीयर्स सहायता मेनू दिखाई देगा।


### 4.6. कीट के साथ नाशपाती का परीक्षण करें


यह सत्यापित करने के लिए कि पीयर्स पूरी तरह से चालू है, आप नेटवर्क पर पहले से उपलब्ध P2P एप्लिकेशन लॉन्च कर सकते हैं, जैसे कि कीट, होलपंच का ओपन-सोर्स मैसेजिंग और वीडियो कॉन्फ्रेंसिंग सॉफ्टवेयर।


```bash
pear run pear://keet
```


यह कमांड किसी केंद्रीय सर्वर का उपयोग किए बिना, सीधे Pears नेटवर्क से Keet ऐप लोड करता है। अगर Keet सफलतापूर्वक लॉन्च हो जाता है, तो इसका मतलब है कि आपका Pears इंस्टॉलेशन पूरी तरह से काम कर रहा है।


आपका macOS सिस्टम अब Pears के साथ पीयर-टू-पीयर एप्लिकेशन चलाने और होस्ट करने के लिए तैयार है।

## 6. नाशपाती पर बीज योजना ₿ अकादमी कैसे बनाएं


पीयर्स नेटवर्क में, किसी एप्लिकेशन को *seed* करने का मतलब है उसे अपनी मशीन से दूसरे पीयर्स को पुनर्वितरित करना। व्यवहार में, जब आप seed Plan ₿ Academy का उपयोग करते हैं, तो आपका कंप्यूटर एक डेटा स्रोत बन जाता है जो अन्य उपयोगकर्ताओं को किसी केंद्रीय सर्वर पर निर्भर हुए बिना एप्लिकेशन डाउनलोड करने की अनुमति देता है।


यह तंत्र पीयर्स नेटवर्क पर हमारे एप्लिकेशन की लचीलापन और सेंसरशिप प्रतिरोध को मज़बूत करता है। किसी एप्लिकेशन के जितने ज़्यादा सहकर्मी seed होंगे, वह उतना ही ज़्यादा उपलब्ध और विकेन्द्रीकृत होगा, भले ही कुछ मूल नोड्स ऑफ़लाइन हो जाएँ।


प्लान ₿ अकादमी को वितरित करने में सहायता के लिए, बस निम्नलिखित कमांड चलाएँ:


```bash
pear seed pear://k9cawqdsan3bkobkigesuyfeqjcasi49ikjaru5cipap835t7nwy
```


जब तक यह कमांड सक्रिय रहेगा, आपका डिवाइस एप्लिकेशन की फ़ाइलों को वितरित करने में भाग लेगा। यदि आप टर्मिनल बंद कर देते हैं, तो साझाकरण प्रक्रिया रुक जाती है।


रीस्टार्ट के बाद सीडिंग जारी रखने के लिए, आप बैकग्राउंड में कमांड चला सकते हैं या सिस्टम सर्विस बना सकते हैं—उदाहरण के लिए, Linux पर systemd सर्विस, macOS पर LaunchAgent, या Windows पर शेड्यूल्ड टास्क। ये तरीके सुनिश्चित करते हैं कि Plan ₿ Academy एप्लिकेशन सिस्टम स्टार्टअप पर सीडिंग को अपने आप फिर से शुरू कर दे।


पीयर्स पर प्लान ₿ अकादमी के विकेन्द्रीकृत वितरण में योगदान देने और Bitcoin शिक्षा को वास्तव में सेंसरशिप-प्रतिरोधी बनाने में मदद करने के लिए धन्यवाद!
