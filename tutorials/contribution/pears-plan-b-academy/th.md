ในบทแนะนำนี้ เราจะเรียนรู้วิธีการใช้งานแพลตฟอร์ม **Plan ₿ Academy** ในรูปแบบที่ทนทานต่อการเซ็นเซอร์อย่างแท้จริง โดยใช้ **Pears** ซึ่งเป็นเทคโนโลยีแบบเพียร์ทูเพียร์ (P2P) ที่พัฒนาโดย **Holepunch** และได้รับการสนับสนุนโดย **Tether**.


Pears เป็นซอฟต์แวร์ที่ช่วยให้เราสามารถใช้งานแพลตฟอร์ม Plan ₿ Academy ได้โดยไม่ต้องพึ่งพาเว็บไซต์ศูนย์กลาง ในบทแนะนำนี้ เราจะติดตั้ง Pears บนคอมพิวเตอร์ของคุณเพื่อเข้าถึง Plan ₿ Academy ผ่าน Pears


เป้าหมายของ Pears นั้นง่ายมาก: เพื่อทำให้สามารถแจกจ่ายและใช้แอปพลิเคชันเว็บได้โดยไม่ต้องพึ่งพาโครงสร้างพื้นฐานแบบรวมศูนย์ (ไม่มีเซิร์ฟเวอร์, ไม่มีโฮสต์, ไม่มีตัวกลาง) กล่าวอีกนัยหนึ่ง แม้ว่าผู้ให้บริการคลาวด์จะปิดตัวลงหรือประเทศใดประเทศหนึ่งบล็อกโดเมน แอปพลิเคชันก็ยังคงมีชีวิตอยู่ท่ามกลางเพื่อนร่วมเครือข่าย วิธีการนี้ทำให้แพลตฟอร์มการศึกษาของเรา Plan ₿ Academy ยังคงเข้าถึงได้ทั่วโลก โดยไม่มีจุดล้มเหลวเพียงจุดเดียว

## 1. Pears คืออะไร?


Pears เป็นทั้งสภาพแวดล้อมการรันไทม์ เครื่องมือพัฒนา และแพลตฟอร์มการปรับใช้สำหรับแอปพลิเคชันแบบเพียร์ทูเพียร์ เครื่องมือโอเพ่นซอร์สนี้ช่วยให้คุณสร้าง แบ่งปัน และรันซอฟต์แวร์โดยไม่ต้องใช้เซิร์ฟเวอร์หรือโครงสร้างพื้นฐาน โดยตรงระหว่างผู้ใช้ ในทางปฏิบัติ หมายความว่าแทนที่จะโฮสต์แอปพลิเคชันบนเซิร์ฟเวอร์กลาง ผู้ใช้แต่ละคนจะกลายเป็นโหนดในเครือข่าย: พวกเขาแบ่งปันส่วนหนึ่งของแอปพลิเคชันและข้อมูลกับเพียร์อื่น ๆ ระบบทั้งหมดจะสร้างเครือข่ายแบบกระจายที่แต่ละอินสแตนซ์ร่วมมือกันเพื่อให้บริการสามารถเข้าถึงได้


![Image](assets/fr/01.webp)


วิธีการนี้อิงตามชุดของส่วนประกอบซอฟต์แวร์แบบโมดูลาร์ที่พัฒนาโดย Holepunch:



- Hypercore**: บันทึกแบบกระจายที่รับประกันความสอดคล้องและความปลอดภัยของข้อมูลโดยไม่ต้องใช้ฐานข้อมูลกลาง
- Hyperbee**: ดัชนีที่สร้างขึ้นบน Hypercore ซึ่งช่วยให้ข้อมูลถูกจัดระเบียบและสืบค้นได้อย่างมีประสิทธิภาพ
- Hyperdrive**: ระบบไฟล์แบบกระจายที่ใช้ในการจัดเก็บและซิงโครไนซ์ไฟล์แอปพลิเคชันระหว่างเพียร์
- Hyperswarm** และ **HyperDHT**: เลเยอร์เครือข่ายที่ช่วยให้การค้นหาและเชื่อมต่อเพียร์ทั่วโลกโดยไม่ต้องใช้เซิร์ฟเวอร์กลาง
- Secretstream**: โปรโตคอลการเข้ารหัสแบบ end-to-end ที่รักษาความปลอดภัยการสื่อสารระหว่างเพียร์


โดยการรวมส่วนประกอบเหล่านี้ Pears ช่วยให้สามารถสร้างแอปพลิเคชันที่เป็นอิสระ เข้ารหัส และกระจายตัวได้ ซึ่งผู้ใช้ทุกคนมีส่วนร่วมอย่างแข็งขันในเครือข่าย สถาปัตยกรรมแบบกระจายศูนย์นี้ช่วยขจัดค่าใช้จ่ายด้านโครงสร้างพื้นฐาน ความเสี่ยงจากการเซ็นเซอร์ และ SPOFs (*Single Points of Failure*)


Pears ได้รับการพัฒนาโดย Holepunch บริษัทที่ก่อตั้งโดย Mathias Buus และ Paolo Ardoino (CEO ของ Tether และ CTO ของ Bitfinex) โดยมีภารกิจในการขยายตรรกะ peer-to-peer ให้เกินกว่า Bitcoin ความทะเยอทะยานของพวกเขาคือการสร้าง “*Internet of peers*” ที่ทุกแอปพลิเคชันสามารถทำงานได้โดยไม่ต้องมีการอนุญาต ไม่มีเซิร์ฟเวอร์ และไม่มีตัวกลาง Holepunch อยู่เบื้องหลัง **Keet** ซึ่งเป็นแอปประชุมทางวิดีโอและส่งข้อความแบบ P2P อย่างเต็มรูปแบบแล้ว


https://planb.academy/tutorials/computer-security/communication/keet-efdb759d-5e94-4bbf-b28c-5fa8669c809b

*คู่มือการติดตั้ง Pears นี้แบ่งออกเป็นหลายส่วนขึ้นอยู่กับระบบปฏิบัติการของคุณ ไปยังส่วนที่ตรงกับสภาพแวดล้อมของคุณเพื่อทำตามคำแนะนำที่เหมาะสม:*



- ลินุกซ์ (Debian)** → ส่วน **2**
- Windows** → Section **3**
- macOS** → Section **4**


## 2. วิธีการติดตั้ง Pears บน Linux (Debian)


การติดตั้ง Pears บน Debian นั้นค่อนข้างง่ายแต่ต้องมีข้อกำหนดเบื้องต้นบางประการ ซึ่งเราจะอธิบายในส่วนนี้


### 2.1. อัปเดตระบบ


ก่อนอื่น สิ่งสำคัญคือต้องตรวจสอบให้แน่ใจว่าระบบของคุณเป็นปัจจุบัน


```bash
sudo apt update && sudo apt upgrade -y
```


![Image](assets/fr/02.webp)


### 2.2. ติดตั้งการพึ่งพา


Pears อาศัยไลบรารีระบบบางตัว รวมถึง `libatomic1` ซึ่งใช้โดย Bare JavaScript runtime engine ติดตั้งด้วยคำสั่งต่อไปนี้:


```bash
sudo apt install -y libatomic1 curl git
```


![Image](assets/fr/03.webp)


### 2.3. ติดตั้ง Node.js และ npm ผ่าน NVM


Pears ถูกแจกจ่ายผ่าน *npm* ซึ่งเป็นตัวจัดการแพ็คเกจของ *Node.js* แม้ว่า Pears จะไม่จำเป็นต้องพึ่งพา *Node.js* โดยตรงในการทำงาน แต่จำเป็นต้องใช้สำหรับการติดตั้ง วิธีที่แนะนำในการติดตั้ง *Node.js* บน Linux คือผ่าน *NVM* (*Node Version Manager*) ซึ่งช่วยให้คุณจัดการหลายเวอร์ชันของ Node ได้พร้อมกัน


```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/master/install.sh | bash
```


![Image](assets/fr/04.webp)


จากนั้น โหลดเทอร์มินัลของคุณใหม่เพื่อเปิดใช้งาน *NVM*:


```bash
source ~/.bashrc
```


![Image](assets/fr/05.webp)


ตรวจสอบว่า *NVM* ติดตั้งอย่างถูกต้อง:


```bash
nvm --version
```


![Image](assets/fr/06.webp)


ถัดไป ติดตั้งเวอร์ชันที่เสถียรของ *Node.js* (เช่น เวอร์ชัน LTS ปัจจุบัน):


```bash
nvm install --lts
```


![Image](assets/fr/07.webp)


ยืนยันว่า *Node.js* และ *npm* ถูกติดตั้งอย่างถูกต้อง:


```bash
node -v
npm -v
```


![Image](assets/fr/08.webp)


### 2.4. ติดตั้ง Pears ด้วย npm


เมื่อ *npm* พร้อมใช้งาน คุณสามารถติดตั้ง Pears CLI บนระบบของคุณได้ทั่วโลก ซึ่งจะช่วยให้คุณสามารถเรียกใช้คำสั่ง `pear` จากไดเรกทอรีใดก็ได้


```bash
npm install -g pear
```


![Image](assets/fr/09.webp)


### 2.5. เริ่มต้น Pears


หลังการติดตั้ง เพียงแค่รันคำสั่งต่อไปนี้ในเทอร์มินัลของคุณ:


```bash
pear
```


เมื่อเปิดตัวครั้งแรก Pears จะเชื่อมต่อกับเครือข่ายเพียร์ทูเพียร์เพื่อดาวน์โหลดส่วนประกอบที่จำเป็น กระบวนการนี้ไม่ต้องพึ่งพาเซิร์ฟเวอร์กลางใด ๆ — ไฟล์จะถูกดึงมาโดยตรงจากเพียร์อื่น ๆ


![Image](assets/fr/10.webp)


เมื่อการดาวน์โหลดเสร็จสิ้น ให้รันคำสั่งอีกครั้งเพื่อยืนยันว่าทุกอย่างทำงานได้:


```bash
pear
```


![Image](assets/fr/11.webp)


หากทุกอย่างติดตั้งอย่างถูกต้อง เมนูช่วยเหลือ Pears จะปรากฏขึ้นพร้อมรายการคำสั่งที่ใช้ได้


### 2.6. ทดสอบ Pears กับ Keet


ในการยืนยันว่า Pears ทำงานได้อย่างเต็มที่ คุณสามารถเปิดแอปพลิเคชัน P2P ที่มีอยู่บนเครือข่าย เช่น Keet ซึ่งเป็นซอฟต์แวร์ส่งข้อความและการประชุมทางวิดีโอแบบโอเพนซอร์สที่พัฒนาโดย Holepunch


```bash
pear run pear://keet
```


คำสั่งนี้จะโหลดแอปพลิเคชัน Keet โดยตรงจากเครือข่าย Pears โดยไม่ต้องใช้เซิร์ฟเวอร์กลาง หาก Keet เปิดตัวได้อย่างถูกต้อง หมายความว่าการติดตั้ง Pears ของคุณทำงานได้อย่างสมบูรณ์


![Image](assets/fr/12.webp)


ระบบ Linux ของคุณพร้อมที่จะรันและโฮสต์แอปพลิเคชันแบบเพียร์ทูเพียร์ด้วย Pears แล้ว


## 3. วิธีการติดตั้ง Pears บน Windows


การติดตั้ง Pears บน Windows นั้นง่ายพอๆ กับบน Linux แต่ต้องใช้เครื่องมือเฉพาะบางอย่าง


*หากคุณใช้ Linux และได้ติดตั้ง Pears แล้ว คุณสามารถข้ามไปที่ **ขั้นตอนที่ 5** ได้เลย*


### 3.1. เปิด PowerShell ในฐานะผู้ดูแลระบบ


ก่อนอื่น ให้เปิด PowerShell ด้วยสิทธิ์ของผู้ดูแลระบบ:



- คลิกที่เมนูเริ่ม;
- พิมพ์ “PowerShell”;
- คลิกขวาที่ "*Windows PowerShell*";
- เลือก "*เรียกใช้ในฐานะผู้ดูแลระบบ*"


![Image](assets/fr/15.webp)


### 3.2. ดาวน์โหลด NVS


Pears ถูกติดตั้งผ่าน *npm* ซึ่งเป็นตัวจัดการแพ็กเกจของ *Node.js* บน Windows วิธีที่ Holepunch แนะนำคือการใช้ *NVS* (*Node Version Switcher*) ซึ่งมีความเสถียรกว่า *NVM* บนระบบนี้


ใน PowerShell ให้รันคำสั่งต่อไปนี้เพื่อติดตั้งเวอร์ชันล่าสุดของ *NVS*:


```PowerShell
winget install jasongin.nvs
```


![Image](assets/fr/16.webp)


### 3.3. ติดตั้ง Node.js


หลังการติดตั้ง ให้รีสตาร์ท PowerShell จากนั้นป้อนคำสั่งต่อไปนี้:


```powershell
nvs
```


คุณควรเห็นรายการของเวอร์ชัน *Node.js* ที่มีอยู่ เลือกอันแรกโดยกดปุ่ม `a` บนแป้นพิมพ์ของคุณ


![Image](assets/fr/17.webp)


*Node.js* ได้รับการติดตั้งแล้ว


![Image](assets/fr/18.webp)


### 3.4. ตรวจสอบการติดตั้ง


ตรวจสอบให้แน่ใจว่า *Node.js* และ *npm* สามารถเข้าถึงได้:


```powershell
node -v
npm -v
```


คำสั่งทั้งสองควรส่งคืนหมายเลขเวอร์ชัน


![Image](assets/fr/19.webp)


### 3.5. ติดตั้ง Pears ด้วย npm


เมื่อ *Node.js* และ *npm* พร้อมใช้งานแล้ว ให้ติดตั้ง **Pears CLI** ทั่วทั้งระบบของคุณ:


```powershell
npm install -g pear
```


สิ่งนี้จะติดตั้งไบนารี `pear` ในไดเรกทอรี *npm* ทั่วโลกของคุณ


![Image](assets/fr/20.webp)


### 3.6. ตรวจสอบและเริ่มต้น Pears


เมื่อติดตั้งเสร็จสิ้น ให้เรียกใช้:


```powershell
pear
```


เมื่อเปิดตัวครั้งแรก Pears จะดาวน์โหลดส่วนประกอบที่จำเป็นจากเครือข่ายเพียร์ทูเพียร์โดยอัตโนมัติ กระบวนการนี้อาจใช้เวลาสักครู่


![Image](assets/fr/21.webp)


หากทุกอย่างเป็นไปด้วยดี คุณควรเห็นเมนูช่วยเหลือ Pears CLI พร้อมรายการคำสั่งย่อยที่มีอยู่ (run, seed, info...)


### 3.7. ทดสอบ Pears กับ Keet


ในการยืนยันว่า Pears ทำงานได้อย่างสมบูรณ์ คุณสามารถเปิดแอปพลิเคชัน P2P ที่มีอยู่บนเครือข่าย เช่น Keet — ซอฟต์แวร์ส่งข้อความและการประชุมทางวิดีโอแบบโอเพ่นซอร์สที่พัฒนาโดย Holepunch


```bash
pear run pear://keet
```


คำสั่งนี้จะโหลดแอปพลิเคชัน Keet โดยตรงจากเครือข่าย Pears โดยไม่ต้องใช้เซิร์ฟเวอร์กลาง หาก Keet เปิดตัวได้สำเร็จ หมายความว่าการติดตั้ง Pears ของคุณทำงานได้อย่างสมบูรณ์


![Image](assets/fr/22.webp)


ระบบ Windows ของคุณพร้อมที่จะรันและโฮสต์แอปพลิเคชันแบบเพียร์ทูเพียร์ด้วย Pears แล้ว


## 4. วิธีการติดตั้ง Pears บน macOS


การติดตั้ง Pears บน macOS คล้ายกับ Linux แต่ต้องมีการปรับเปลี่ยนบางอย่างที่เฉพาะเจาะจงสำหรับสภาพแวดล้อมของ Apple มาดำเนินการตามขั้นตอนเหล่านี้ไปด้วยกัน


*หากคุณใช้ Linux หรือ Windows และได้ติดตั้ง Pears แล้ว คุณสามารถข้ามไปที่ **ขั้นตอนที่ 5** ได้เลย.*


### 4.1. ตรวจสอบข้อกำหนดเบื้องต้นของระบบ


ก่อนการติดตั้ง ตรวจสอบให้แน่ใจว่าได้ติดตั้ง *Xcode Command Line Tools* บนระบบของคุณแล้ว แพ็กเกจนี้จะให้เครื่องมือที่จำเป็นสำหรับการสร้าง *Node.js* และการพึ่งพาของมัน


ในการทำเช่นนี้ ให้เปิดเทอร์มินัลโดยใช้ทางลัด `Cmd + Space bar` พิมพ์ `Terminal` และกด `Enter` จากนั้น รันคำสั่งต่อไปนี้ในเทอร์มินัลเพื่อติดตั้ง:


```bash
xcode-select --install
```


หากเครื่องมือได้รับการติดตั้งบนระบบของคุณแล้ว macOS จะแจ้งให้คุณทราบ


### 4.2. ติดตั้ง NVM


Pears ถูกแจกจ่ายผ่าน *npm* ซึ่งเป็นตัวจัดการแพ็กเกจของ *Node.js* แม้ว่า Pears จะไม่ขึ้นอยู่กับ *Node.js* โดยตรงในการทำงาน แต่จำเป็นต้องใช้สำหรับการติดตั้ง วิธีที่แนะนำสำหรับการติดตั้ง *Node.js* บน macOS คือ *NVM* (*Node Version Manager*) ซึ่งช่วยให้คุณจัดการหลายเวอร์ชันของ Node ได้พร้อมกัน


```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/master/install.sh | bash
```


จากนั้นโหลดเทอร์มินัลของคุณใหม่เพื่อเปิดใช้งาน *NVM*:


```bash
source ~/.zshrc
```


หากคุณใช้ *bash* แทนที่จะเป็น *zsh* ให้รัน:


```bash
source ~/.bashrc
```


ถัดไป ตรวจสอบว่า *NVM* ติดตั้งอย่างถูกต้อง:


```bash
nvm --version
```


เทอร์มินัลของคุณควรแสดงเวอร์ชัน *NVM* ที่ติดตั้งอยู่


### 4.3. ติดตั้ง Node.js และ npm


ถัดไป ติดตั้งเวอร์ชันที่เสถียรของ *Node.js* (เช่น เวอร์ชัน LTS ปัจจุบัน):


```bash
nvm install --lts
```


เมื่อติดตั้งเสร็จสิ้น ให้ตรวจสอบเวอร์ชันที่ติดตั้ง:


```bash
node -v
npm -v
```


ทั้งสองคำสั่งควรส่งคืนหมายเลขเวอร์ชัน


### 4.4. ติดตั้ง Pears ด้วย npm


เมื่อ *npm* พร้อมใช้งาน คุณสามารถติดตั้ง Pears CLI บนระบบของคุณได้ทั่วโลก ซึ่งจะช่วยให้คุณสามารถเรียกใช้คำสั่ง `pear` จากไดเรกทอรีใดก็ได้


```bash
npm install -g pear
```


### 4.5. เริ่มต้น Pears


หลังการติดตั้ง เพียงแค่รันคำสั่งต่อไปนี้ในเทอร์มินัลของคุณ:


```bash
pear
```


เมื่อเปิดตัวครั้งแรก Pears จะเชื่อมต่อกับเครือข่ายเพียร์ทูเพียร์เพื่อดาวน์โหลดส่วนประกอบที่จำเป็น กระบวนการนี้ไม่ต้องการเซิร์ฟเวอร์กลางใด ๆ — ไฟล์จะถูกดึงโดยตรงจากเพียร์อื่น ๆ


เมื่อการดาวน์โหลดเสร็จสิ้น ให้รันคำสั่งอีกครั้งเพื่อยืนยันว่าทุกอย่างทำงานได้:


```bash
pear
```


หากทุกอย่างติดตั้งถูกต้อง เมนูช่วยเหลือ Pears จะปรากฏขึ้นพร้อมรายการคำสั่งที่ใช้ได้


### 4.6. ทดสอบ Pears กับ Keet


ในการยืนยันว่า Pears ทำงานได้อย่างสมบูรณ์ คุณสามารถเปิดแอปพลิเคชัน P2P ที่มีอยู่แล้วบนเครือข่าย เช่น Keet ซึ่งเป็นซอฟต์แวร์ส่งข้อความและการประชุมทางวิดีโอแบบโอเพนซอร์สจาก Holepunch


```bash
pear run pear://keet
```


คำสั่งนี้จะโหลดแอป Keet โดยตรงจากเครือข่าย Pears โดยไม่ต้องใช้เซิร์ฟเวอร์กลาง หาก Keet เปิดตัวได้สำเร็จ หมายความว่าการติดตั้ง Pears ของคุณทำงานได้อย่างสมบูรณ์


ระบบ macOS ของคุณพร้อมแล้วที่จะรันและโฮสต์แอปพลิเคชันแบบเพียร์ทูเพียร์ด้วย Pears

## 6. วิธีการเพาะเมล็ด Plan ₿ Academy บน Pears


ในเครือข่าย Pears, การ *seed* แอปพลิเคชันหมายถึงการแจกจ่ายใหม่ให้กับเพื่อนอื่น ๆ จากเครื่องของคุณเอง ในทางปฏิบัติ เมื่อคุณ seed Plan ₿ Academy คอมพิวเตอร์ของคุณจะกลายเป็นแหล่งข้อมูลที่อนุญาตให้ผู้ใช้รายอื่นดาวน์โหลดแอปพลิเคชันโดยไม่ต้องพึ่งพาเซิร์ฟเวอร์กลาง


กลไกนี้ช่วยเสริมสร้างความยืดหยุ่นและความต้านทานการเซ็นเซอร์ของแอปพลิเคชันของเราบนเครือข่าย Pears ยิ่งมีเพื่อนร่วมเครือข่าย seed มากเท่าใด แอปพลิเคชันก็จะยิ่งมีความพร้อมใช้งานและกระจายศูนย์มากขึ้นเท่านั้น แม้ว่าบางโหนดดั้งเดิมจะออฟไลน์ก็ตาม


ในการช่วยกระจาย Plan ₿ Academy เพียงแค่รันคำสั่งต่อไปนี้:


```bash
pear seed pear://k9cawqdsan3bkobkigesuyfeqjcasi49ikjaru5cipap835t7nwy
```


ตราบใดที่คำสั่งนี้ยังคงทำงานอยู่ อุปกรณ์ของคุณจะมีส่วนร่วมในการแจกจ่ายไฟล์ของแอปพลิเคชัน หากคุณปิดเทอร์มินัล กระบวนการแชร์จะหยุดลง


หากต้องการดำเนินการปล่อยข้อมูลต่อหลังจากรีสตาร์ท คุณสามารถรันคำสั่งในพื้นหลังหรือสร้างบริการระบบ — ตัวอย่างเช่น บริการ systemd บน Linux, LaunchAgent บน macOS หรือกำหนดงานตามตารางเวลาใน Windows วิธีการเหล่านี้จะช่วยให้แอปพลิเคชัน Plan ₿ Academy กลับมาปล่อยข้อมูลโดยอัตโนมัติเมื่อเริ่มต้นระบบใหม่


ขอบคุณที่มีส่วนร่วมในการกระจาย Plan ₿ Academy แบบกระจายศูนย์บน Pears และช่วยทำให้การศึกษา Bitcoin ปลอดจากการเซ็นเซอร์อย่างแท้จริง!
