## 1.梨とは？


Pears はピアツーピアアプリケーションのための実行環境であり、開発ツールであり、デプロイメントプラットフォームです。このオープンソースツールにより、サーバーやインフラを使わずに、ユーザー間で直接ソフトウェアを構築、共有、実行することができます。つまり、アプリケーションを中央サーバーでホスティングする代わりに、各ユーザーがネットワークのノードとなり、アプリケーションの一部やデータを他のピアと共有する。システム全体が分散ネットワークを形成し、各インスタンスが協力してサービスにアクセスし続ける。


![Image](assets/fr/01.webp)


このアプローチは、Holepunchが開発した一連のモジュラー・ソフトウェア・コンポーネントに基づいている：



- ハイパーコア**：中央データベースなしでデータの一貫性とセキュリティを確保する分散ログ。
- Hyperbee**：Hypercoreの上に構築されたインデックスで、データを整理して効率的に照会できる。
- Hyperdrive**：ピア間でのアプリケーションファイルの保存と同期に使用される分散ファイルシステム。
- Hyperswarm**と**HyperDHT**：中央サーバーなしで世界中のピア発見と接続を可能にするネットワークレイヤー。
- Secretstream**：ピア間の通信を保護するエンドツーエンドの暗号化プロトコル。


これらのコンポーネントを組み合わせることで、Pearsは自律的で暗号化された分散型アプリケーションの作成を可能にし、すべてのユーザーが積極的にネットワークに参加します。この分散型アーキテクチャにより、インフラコスト、検閲リスク、SPOF（*Single Points of Failure*）が排除されます。


PearsはMathias BuusとPaolo Ardoino（Tether CEOとBitfinex CTO）によって設立されたHolepunch社によって開発され、Bitcoinを超えてピアツーピアのロジックを拡張することを使命としている。彼らの野望は、「ピアのインターネット*」を構築することであり、そこではあらゆるアプリケーションが許可なく、サーバーなしで、仲介者なしで動作することができる。Holepunchはすでに、完全なP2Pビデオ会議・メッセージングアプリである**Keet**を開発している。


https://planb.academy/tutorials/computer-security/communication/keet-efdb759d-5e94-4bbf-b28c-5fa8669c809b

## 2.Pears を Linux (Debian) にインストールするには？


Debian への Pears のインストールは比較的簡単ですが、いくつかの前提条件があります。


### 2.1.システムの更新


何よりもまず、システムが最新であることを確認することが重要だ。


```bash
sudo apt update && sudo apt upgrade -y
```


![Image](assets/fr/02.webp)


### 2.2.依存関係のインストール


Pears は Bare JavaScript ランタイムエンジンで使用される `libatomic1` を含むいくつかのシステムライブラリに依存しています。以下のコマンドでインストールしてください：


```bash
sudo apt install -y libatomic1 curl git
```


![Image](assets/fr/03.webp)


### 2.3.NVM経由でNode.jsとnpmをインストールする。


Pears は *Node.js* のパッケージマネージャである *npm* から配布されています。Pears は直接 *Node.js* に依存して動作するわけではありませんが、 インストールの際には *Node.js* が必要です。Linux 上で *Node.js* をインストールするには、*NVM* (*Node Version Manager*) を使うことをお勧めします。


```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/master/install.sh | bash
```


![Image](assets/fr/04.webp)


その後、端末をリロードして*NVM*を起動する：


```bash
source ~/.bashrc
```


![Image](assets/fr/05.webp)


NVM* が正しくインストールされていることを確認する：


```bash
nvm --version
```


![Image](assets/fr/06.webp)


次に、*Node.js* の安定版（例えば、現在の LTS バージョン）をインストールします：


```bash
nvm install --lts
```


![Image](assets/fr/07.webp)


Node.js*と*npm*が正しくインストールされていることを確認する：


```bash
node -v
npm -v
```


![Image](assets/fr/08.webp)


### 2.4.npm で Pears をインストールする


npm* が利用できるようになると、Pears CLI をグローバルにインストールすることができます。これにより、どのディレクトリからでも `pear` コマンドを実行できるようになります。


```bash
npm install -g pear
```


![Image](assets/fr/09.webp)


### 2.5.梨の初期化


インストールが終わったら、ターミナルで以下のコマンドを実行するだけだ：


```bash
pear
```


最初の起動時に、Pears はピアツーピアネットワークに接続し、必要なコンポーネントをダウンロードします。このプロセスは中央のサーバーに依存せず、他のピアから直接ファイルを取得します。


![Image](assets/fr/10.webp)


ダウンロードが完了したら、もう一度コマンドを実行し、すべてがうまくいくことを確認する：


```bash
pear
```


![Image](assets/fr/11.webp)


すべてが正しくインストールされていれば、Pears のヘルプメニューが表示され、使用可能なコマンドのリストが表示されます。


### 2.6.洋梨のキート添え


Pears が完全に動作していることを確認するために、Holepunch 社が開発したオープンソースのメッセージングおよびビデオ会議ソフトウェアである Keet など、ネットワーク上で利用可能な既存の P2P アプリケーションを起動することができます。


```bash
pear run pear://keet
```


このコマンドは Pears ネットワークから直接 Keet アプリケーションを読み込みます。Keet が正しく起動すれば、Pears が完全に動作していることを意味します。


![Image](assets/fr/12.webp)


これであなたの Linux システムは Pears を使ってピアツーピアアプリケーションを実行し、ホストする準備ができました。


## 3.WindowsにPearsをインストールする方法


Windows への Pears のインストールは Linux と同様に簡単ですが、いくつかの特別なツールが必要です。


*Linux を使っていて、すでに Pears をインストールしている場合は、そのまま **Step 5**.* に進んでください。


### 3.1.PowerShellを管理者として開く


まず、PowerShellを管理者権限で起動する：



- スタートメニューをクリックする；
- PowerShell」と入力する；
- Windows PowerShell*」を右クリック；
- 管理者として実行*」を選択する。


![Image](assets/fr/15.webp)


### 3.2.NVSのダウンロード


Pears は *Node.js* のパッケージマネージャである *npm* を使ってインストールします。Windows の場合、Holepunch が推奨する方法は *NVS* (*Node Version Switcher*) を使う方法で、このシステムでは *NVM* よりも安定しています。


PowerShellで以下のコマンドを実行し、*NVS*の最新バージョンをインストールする：


```PowerShell
winget install jasongin.nvs
```


![Image](assets/fr/16.webp)


### 3.3.Node.jsのインストール


インストール後、PowerShellを再起動し、以下のコマンドを入力する：


```powershell
nvs
```


利用可能な *Node.js* バージョンのリストが表示されるはずです。キーボードの `a` キーを押して最初のものを選択してください。


![Image](assets/fr/17.webp)


*Node.js*がインストールされました。


![Image](assets/fr/18.webp)


### 3.4.インストールの確認


Node.js*と*npm*にアクセスできることを確認する：


```powershell
node -v
npm -v
```


どちらのコマンドもバージョン番号を返すはずだ。


![Image](assets/fr/19.webp)


### 3.5.npm で Pears をインストールする


Node.js* と *npm* が利用可能になったら、 **Pears CLI** をシステムにグローバルにインストールします：


```powershell
npm install -g pear
```


これは、グローバルな *npm* ディレクトリに `pear` バイナリをインストールします。


![Image](assets/fr/20.webp)


### 3.6.ベリファイと初期化


インストールが完了したら、実行する：


```powershell
pear
```


初回起動時に、Pears はピアツーピアネットワークから必要なコンポーネントを自動的にダウンロードします。このプロセスには少し時間がかかります。


![Image](assets/fr/21.webp)


すべてがうまくいっていれば、Pears CLI のヘルプメニューが表示され、使用可能なサブコマンド（run、seed、info...）のリストが表示されるはずです。


### 3.7.洋梨のキート添え


例えば、Holepunch社が開発したオープンソースのメッセージング・ビデオ会議ソフトウェアKeetなどです。


```bash
pear run pear://keet
```


このコマンドは Pears ネットワークから直接 Keet アプリケーションを読み込みます。Keet が正常に起動すれば、Pears が完全に動作していることを意味します。


![Image](assets/fr/22.webp)


これであなたの Windows システムは Pears を使ってピアツーピアアプリケーションを実行し、ホストする準備が整いました。


## 4.PearsをmacOSにインストールする方法


macOSへのPearsのインストールはLinuxと似ていますが、Appleの環境特有の調整がいくつか必要です。これらの手順を一緒に見ていきましょう。


*Linux または Windows を使用していて、すでに Pears をインストールしている場合は、そのまま **ステップ 5** に進んでください。


### 4.1.システムの前提条件を確認する


インストールする前に、*Xcode Command Line Tools* がシステムにインストールされていることを確認してください。本パッケージは、*Node.js* とその依存関係に必要なビルドツールを提供します。


これを行うには、ショートカット `Cmd + スペースバー` を使ってターミナルを開き、`Terminal` と入力して `Enter` を押します。次に、ターミナルで以下のコマンドを実行してインストールする：


```bash

```


ツールがすでにシステムにインストールされている場合は、macOSが通知する。

```


その後、端末をリロードして*NVM*をアクティブにする：

```

次に、*NVM*が正しくインストールされていることを確認する：

ダウンロードが完了したら、コマンドを再実行してすべてがうまくいくことを確認する：

## 6.シード・プランの立て方 ₿ アカデミー・オン・ナシ


Pearsのネットワークでは、アプリケーションを*seed*することは、自分のマシンから他のピアに再配布することを意味します。実際には、あなたがseedプラン₿アカデミーを利用するとき、あなたのコンピュータがデータソースとなり、他のユーザーが中央サーバーに依存することなくアプリケーションをダウンロードできるようになります。


このメカニズムにより、Pearsネットワーク上のアプリケーションの耐障害性と検閲耐性が強化されます。seedのピア数が多ければ多いほど、元のノードがオフラインになったとしても、アプリケーションはより利用しやすくなり、分散化されます。


プラン₿アカデミーを配布するには、以下のコマンドを実行するだけです：


```bash
pear seed pear://k9cawqdsan3bkobkigesuyfeqjcasi49ikjaru5cipap835t7nwy
```


このコマンドが有効である限り、デバイスはアプリケーションのファイル配布に参加します。ターミナルを閉じると、共有プロセスは停止します。


再起動後もシーディングを継続するには、バックグラウンドでコマンドを実行するか、システム サービスを作成します。たとえば、Linux の systemd サービス、macOS の LaunchAgent、Windows のスケジュール タスクなどです。これらの方法により、Plan ↪Sc_20Academy アプリケーションがシステム起動時に自動的にシーディングを再開します。
