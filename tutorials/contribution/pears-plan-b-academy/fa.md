احتمالاً شما قبلاً می‌دانید که آکادمی Plan ₿ بزرگترین پایگاه داده آموزشی اختصاص داده شده به Bitcoin است که دوره‌ها، آموزش‌ها و هزاران منبع متن‌باز را گرد هم می‌آورد. در ابتدا، آکادمی Plan ₿ یک وب‌سایت بود. اما چه اتفاقی می‌افتد اگر دیگر نتوانید به طور عادی به آن دسترسی پیدا کنید، مثلاً در صورت سانسور؟

پیرز نرم‌افزاری است که به ما امکان می‌دهد پلتفرم آکادمی Plan ₿ را بدون وابستگی به یک وب‌سایت متمرکز اجرا کنیم. در این آموزش، ما پیرز را بر روی کامپیوتر شما نصب خواهیم کرد تا از طریق پیرز به آکادمی Plan ₿ دسترسی پیدا کنید.


هدف Pears ساده است: امکان توزیع و استفاده از برنامه‌های وب بدون وابستگی به هرگونه زیرساخت متمرکز (بدون سرورها، بدون میزبان‌ها، بدون واسطه‌ها). به عبارت دیگر، حتی اگر یک ارائه‌دهنده ابری تعطیل شود یا یک کشور دامنه‌ای را مسدود کند، برنامه همچنان در میان همتایان شبکه به زندگی خود ادامه می‌دهد. این رویکرد به پلتفرم آموزشی ما، Plan ₿ Academy، اجازه می‌دهد تا بدون نقطه شکست واحدی، در سراسر جهان قابل دسترسی باقی بماند.

## 1. گلابی چیست؟


Pears به طور همزمان یک محیط زمان اجرا، یک ابزار توسعه و یک پلتفرم استقرار برای برنامه‌های همتا به همتا است. این ابزار متن‌باز به شما اجازه می‌دهد تا نرم‌افزار را بدون سرورها یا زیرساخت‌ها، مستقیماً بین کاربران بسازید، به اشتراک بگذارید و اجرا کنید. به زبان عملی، این بدان معناست که به جای میزبانی یک برنامه بر روی یک سرور مرکزی، هر کاربر به یک گره در شبکه تبدیل می‌شود: آن‌ها بخشی از برنامه و داده‌ها را با دیگر همتایان به اشتراک می‌گذارند. کل سیستم یک شبکه توزیع‌شده را تشکیل می‌دهد که در آن هر نمونه برای حفظ دسترسی به سرویس همکاری می‌کند.


![Image](assets/fr/01.webp)


این رویکرد بر اساس مجموعه‌ای از اجزای نرم‌افزاری مدولار توسعه‌یافته توسط Holepunch است:



- Hypercore**: یک لاگ توزیع‌شده که بدون نیاز به یک پایگاه داده مرکزی، انسجام و امنیت داده‌ها را تضمین می‌کند.
- Hyperbee**: یک شاخص ساخته شده بر روی Hypercore که اجازه می‌دهد داده‌ها به صورت کارآمد سازماندهی و جستجو شوند.
- هایپردرایو**: یک سیستم فایل توزیع‌شده که برای ذخیره و همگام‌سازی فایل‌های برنامه بین همتایان استفاده می‌شود.
- Hyperswarm** و **HyperDHT**: لایه‌های شبکه‌ای که امکان کشف همتا و اتصال در سراسر جهان را بدون نیاز به سرور مرکزی فراهم می‌کنند.
- Secretstream**: یک پروتکل رمزگذاری سرتاسری که ارتباط بین همتایان را امن می‌کند.


با ترکیب این اجزا، Pears امکان ایجاد برنامه‌های خودمختار، رمزگذاری‌شده و توزیع‌شده را فراهم می‌کند، جایی که هر کاربر به‌طور فعال در شبکه شرکت می‌کند. این معماری غیرمتمرکز هزینه‌های زیرساخت، خطرات سانسور و SPOFها (*نقاط شکست منفرد*) را از بین می‌برد.


Pears توسط Holepunch توسعه یافته است، شرکتی که توسط Mathias Buus و Paolo Ardoino (مدیر عامل Tether و مدیر فناوری Bitfinex) تأسیس شده است، با مأموریت گسترش منطق همتا به همتا فراتر از Bitcoin. هدف آن‌ها ساختن "اینترنت همتاها" است، جایی که هر برنامه‌ای می‌تواند بدون نیاز به مجوز، بدون سرورها و بدون واسطه‌ها عمل کند. Holepunch در حال حاضر پشت **Keet**، یک برنامه کنفرانس ویدیویی و پیام‌رسانی کاملاً P2P است.


https://planb.academy/tutorials/computer-security/communication/keet-efdb759d-5e94-4bbf-b28c-5fa8669c809b

*این آموزش نصب Pears به بخش‌های مختلفی تقسیم شده است که بسته به سیستم‌عامل شما متفاوت است. مستقیماً به بخشی بروید که با محیط شما مطابقت دارد تا دستورالعمل‌های مناسب را دنبال کنید:*



- لینوکس (دبیان)** → بخش **2**
- ویندوز** → بخش **3**
- macOS** → بخش **4**


## 2. چگونه Pears را روی لینوکس (دبیان) نصب کنیم؟


نصب Pears بر روی Debian نسبتاً ساده است اما نیاز به چند پیش‌نیاز دارد که در این بخش به تفصیل خواهیم پرداخت.


### ۲.۱. به‌روزرسانی سیستم


قبل از هر چیز، مهم است که مطمئن شوید سیستم شما به‌روز است.


```bash
sudo apt update && sudo apt upgrade -y
```


![Image](assets/fr/02.webp)


### ۲.۲. نصب وابستگی‌ها


گلابی به برخی از کتابخانه‌های سیستم متکی است، از جمله `libatomic1` که توسط موتور زمان اجرای Bare JavaScript استفاده می‌شود. آن را با فرمان زیر نصب کنید:


```bash
sudo apt install -y libatomic1 curl git
```


![Image](assets/fr/03.webp)


### 2.3. نصب Node.js و npm از طریق NVM


گلابی‌ها از طریق *npm*، مدیر بسته *Node.js* توزیع می‌شوند. اگرچه گلابی‌ها به طور مستقیم برای اجرا به *Node.js* وابسته نیستند، اما برای نصب لازم است. روش توصیه شده برای نصب *Node.js* بر روی لینوکس از طریق *NVM* (*مدیر نسخه Node*) است که به شما امکان می‌دهد چندین نسخه از Node را در کنار هم مدیریت کنید.


```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/master/install.sh | bash
```


![Image](assets/fr/04.webp)


سپس، ترمینال خود را مجدداً بارگذاری کنید تا *NVM* فعال شود:


```bash
source ~/.bashrc
```


![Image](assets/fr/05.webp)


بررسی کنید که *NVM* به درستی نصب شده است:


```bash
nvm --version
```


![Image](assets/fr/06.webp)


سپس، یک نسخه پایدار از *Node.js* را نصب کنید (برای مثال، نسخه LTS فعلی):


```bash
nvm install --lts
```


![Image](assets/fr/07.webp)


تأیید کنید که *Node.js* و *npm* به‌درستی نصب شده‌اند:


```bash
node -v
npm -v
```


![Image](assets/fr/08.webp)


### ۲.۴. نصب Pears با npm


هنگامی که *npm* در دسترس باشد، می‌توانید Pears CLI را به صورت جهانی روی سیستم خود نصب کنید. این به شما امکان می‌دهد تا فرمان `pear` را از هر دایرکتوری اجرا کنید.


```bash
npm install -g pear
```


![Image](assets/fr/09.webp)


### ۲.۵. مقداردهی اولیه گلابی‌ها


پس از نصب، به سادگی فرمان زیر را در ترمینال خود اجرا کنید:


```bash
pear
```


در اولین راه‌اندازی، Pears به شبکه همتا به همتا متصل می‌شود تا اجزای لازم را دانلود کند. این فرآیند به هیچ سرور مرکزی وابسته نیست — فایل‌ها به‌طور مستقیم از دیگر همتاها بازیابی می‌شوند.


![Image](assets/fr/10.webp)


پس از اتمام دانلود، فرمان را دوباره اجرا کنید تا تأیید کنید که همه چیز کار می‌کند:


```bash
pear
```


![Image](assets/fr/11.webp)


اگر همه چیز به درستی نصب شده باشد، منوی راهنمای Pears با فهرستی از دستورات موجود ظاهر خواهد شد.


### ۲.۶. آزمایش گلابی‌ها با کیت


برای تأیید اینکه Pears به‌طور کامل عملیاتی است، می‌توانید یک برنامه P2P موجود در شبکه، مانند Keet، نرم‌افزار پیام‌رسانی و ویدئو کنفرانس متن‌باز توسعه‌یافته توسط Holepunch را اجرا کنید.


```bash
pear run pear://keet
```


این دستور برنامه Keet را مستقیماً از شبکه Pears بارگذاری می‌کند، بدون استفاده از یک سرور مرکزی. اگر Keet به درستی اجرا شود، به این معنی است که نصب Pears شما به طور کامل کار می‌کند.


![Image](assets/fr/12.webp)


سیستم لینوکس شما اکنون آماده است تا برنامه‌های همتا به همتا را با Pears اجرا و میزبانی کند.


## 3. چگونه Pears را روی ویندوز نصب کنیم


نصب Pears بر روی ویندوز به همان سادگی لینوکس است اما به چند ابزار خاص نیاز دارد.


*اگر از لینوکس استفاده می‌کنید و Pears را قبلاً نصب کرده‌اید، می‌توانید مستقیماً به **مرحله ۵** بروید.*


### 3.1. پاورشل را به عنوان مدیر باز کنید


ابتدا، PowerShell را با دسترسی‌های مدیر اجرا کنید:



- روی منوی شروع کلیک کنید؛
- عبارت "PowerShell" را تایپ کنید؛
- روی "*Windows PowerShell*" راست‌کلیک کنید؛
- گزینه "*Run as administrator*" را انتخاب کنید.


![Image](assets/fr/15.webp)


### ۳.۲. دانلود NVS


گلابی‌ها از طریق *npm*، مدیر بسته *Node.js* نصب می‌شوند. در ویندوز، روش توصیه شده توسط Holepunch استفاده از *NVS* (*Node Version Switcher*) است که نسبت به *NVM* در این سیستم پایدارتر است.


در پاورشل، فرمان زیر را برای نصب آخرین نسخه *NVS* اجرا کنید:


```PowerShell
winget install jasongin.nvs
```


![Image](assets/fr/16.webp)


### ۳.۳. نصب Node.js


پس از نصب، PowerShell را مجدداً راه‌اندازی کنید، سپس فرمان زیر را وارد کنید:


```powershell
nvs
```


شما باید لیستی از نسخه‌های موجود *Node.js* را ببینید. با فشار دادن کلید `a` روی صفحه‌کلید خود، اولین نسخه را انتخاب کنید.


![Image](assets/fr/17.webp)


*Node.js* اکنون نصب شده است.


![Image](assets/fr/18.webp)


### ۳.۴. تأیید نصب‌ها


مطمئن شوید که *Node.js* و *npm* در دسترس هستند:


```powershell
node -v
npm -v
```


هر دو دستور باید یک شماره نسخه را برگردانند.


![Image](assets/fr/19.webp)


### ۳.۵. نصب Pears با npm


هنگامی که *Node.js* و *npm* در دسترس هستند، **Pears CLI** را به صورت جهانی بر روی سیستم خود نصب کنید:


```powershell
npm install -g pear
```


این کار باینری `pear` را در دایرکتوری جهانی *npm* شما نصب می‌کند.


![Image](assets/fr/20.webp)


### ۳.۶. تأیید و راه‌اندازی گلابی‌ها


پس از اتمام نصب، اجرا کنید:


```powershell
pear
```


در اولین اجرا، Pears به‌طور خودکار اجزای مورد نیاز را از شبکه همتا به همتا دانلود خواهد کرد. این فرآیند ممکن است چند لحظه طول بکشد.


![Image](assets/fr/21.webp)


اگر همه چیز به خوبی پیش رفته باشد، باید منوی راهنمای Pears CLI را با لیست زیرفرمان‌های موجود (run، seed، info...) ببینید.


### ۳.۷. آزمایش گلابی‌ها با کیت


برای اطمینان از اینکه Pears به‌طور کامل عملیاتی است، می‌توانید یک برنامه P2P موجود در شبکه، مانند Keet — نرم‌افزار پیام‌رسانی و ویدئو کنفرانس متن‌باز توسعه‌یافته توسط Holepunch را اجرا کنید.


```bash

```


این دستور برنامه Keet را مستقیماً از شبکه Pears بارگذاری می‌کند، بدون استفاده از هیچ سرور مرکزی. اگر Keet با موفقیت راه‌اندازی شود، به این معنی است که نصب Pears شما به‌طور کامل عملیاتی است.


![Image](assets/fr/22.webp)


سیستم ویندوز شما اکنون آماده است تا برنامه‌های همتا به همتا را با Pears اجرا و میزبانی کند.


## ۴. نحوه نصب Pears روی macOS


نصب Pears بر روی macOS مشابه لینوکس است اما نیاز به چند تنظیم خاص برای محیط اپل دارد. بیایید این مراحل را با هم مرور کنیم.


*اگر از لینوکس یا ویندوز استفاده می‌کنید و Pears را قبلاً نصب کرده‌اید، می‌توانید مستقیماً به **مرحله ۵** بروید.*


### ۴.۱. بررسی پیش‌نیازهای سیستم


قبل از نصب، مطمئن شوید که *Xcode Command Line Tools* بر روی سیستم شما نصب شده است. این بسته ابزارهای ساخت لازم برای *Node.js* و وابستگی‌های آن را فراهم می‌کند.


برای انجام این کار، یک ترمینال را با استفاده از میانبر `Cmd + Space bar` باز کنید، `Terminal` را تایپ کنید و `Enter` را فشار دهید. سپس، فرمان زیر را در ترمینال اجرا کنید تا آن را نصب کنید:


```bash
xcode-select --install
```


اگر ابزارها از قبل روی سیستم شما نصب شده باشند، macOS به شما اطلاع خواهد داد.


### ۴.۲. نصب NVM


گلابی‌ها از طریق *npm*، مدیر بسته *Node.js* توزیع می‌شوند. اگرچه گلابی‌ها به طور مستقیم به *Node.js* برای عملکرد وابسته نیستند، اما برای نصب لازم است. روش توصیه شده برای نصب *Node.js* بر روی macOS استفاده از *NVM* (*مدیر نسخه Node*) است که به شما امکان می‌دهد چندین نسخه Node را به طور همزمان مدیریت کنید.


```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/master/install.sh | bash
```


سپس ترمینال خود را مجدداً بارگذاری کنید تا *NVM* فعال شود:


```bash
source ~/.zshrc
```


اگر از *bash* به جای *zsh* استفاده می‌کنید، اجرا کنید:


```bash
source ~/.bashrc
```


سپس، بررسی کنید که *NVM* به درستی نصب شده است:


```bash
nvm --version
```


ترمینال شما باید نسخه نصب شده *NVM* را نمایش دهد.


### ۴.۳. نصب Node.js و npm


سپس، یک نسخه پایدار از *Node.js* را نصب کنید (برای مثال، نسخه LTS فعلی):


```bash
nvm install --lts
```


پس از اتمام نصب، نسخه‌های نصب شده را بررسی کنید:


```bash
node -v

```


هر دو دستور باید یک شماره نسخه را برگردانند.

## 6. چگونه آکادمی Seed Plan ₿ را روی گلابی‌ها بکاریم


در شبکه Pears، *seed* کردن یک برنامه به معنای توزیع مجدد آن به سایر همتایان از دستگاه خودتان است. در عمل، وقتی Plan ₿ Academy را seed می‌کنید، رایانه شما به یک منبع داده تبدیل می‌شود که به سایر کاربران اجازه می‌دهد برنامه را بدون وابستگی به یک سرور مرکزی دانلود کنند.


این مکانیزم مقاومت و مقاومت در برابر سانسور برنامه ما را در شبکه Pears تقویت می‌کند. هرچه همتایان بیشتری برنامه seed را اجرا کنند، برنامه در دسترس‌تر و غیرمتمرکزتر می‌شود، حتی اگر برخی از گره‌های اصلی آفلاین شوند.


برای کمک به توزیع Plan ₿ Academy، به سادگی فرمان زیر را اجرا کنید:


```bash
pear seed pear://k9cawqdsan3bkobkigesuyfeqjcasi49ikjaru5cipap835t7nwy
```


تا زمانی که این دستور فعال باقی بماند، دستگاه شما در توزیع فایل‌های برنامه مشارکت خواهد کرد. اگر ترمینال را ببندید، فرآیند اشتراک‌گذاری متوقف می‌شود.


برای ادامه دادن به seeding پس از راه‌اندازی مجدد، می‌توانید فرمان را در پس‌زمینه اجرا کنید یا یک سرویس سیستم ایجاد کنید — به عنوان مثال، یک سرویس systemd در لینوکس، یک LaunchAgent در macOS، یا یک وظیفه زمان‌بندی‌شده در ویندوز. این روش‌ها تضمین می‌کنند که برنامه Plan ₿ Academy به‌طور خودکار در شروع سیستم به seeding ادامه می‌دهد.
