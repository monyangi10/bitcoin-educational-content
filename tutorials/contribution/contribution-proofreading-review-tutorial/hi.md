## सारांश


यदि आप Plan ₿ Network के लिए सामग्री का प्रूफ़रीड या समीक्षा करना चाहते हैं, तो योगदान के लिए समर्पित [हमारे टेलीग्राम समूह](https://t.me/PlanBNetwork_ContentBuilder) से जुड़ें। अपना संक्षिप्त परिचय दें, फिर वह सामग्री निर्दिष्ट करें जिसकी आप समीक्षा करना चाहते हैं, और भाषा। अपनी समीक्षा पूरी होने के बाद किसी कार्यशील शाखा पर अपने सुधार करें और PR सबमिट करें।


यदि आपको उपरोक्त भाग समझ में नहीं आया, तो चिंता न करें - यह ट्यूटोरियल आपके लिए है!

## जीडब्ल्यू-2 क्या है?


Plan ₿ Network सभी के लिए कई पाठ्यक्रमों और ट्यूटोरियल के माध्यम से Bitcoin के बारे में जानने का एक मंच है। वर्तमान में, हमारी सभी सामग्री ओपन-सोर्स है, GitHub रिपॉजिटरी पर होस्ट की गई है, और समीक्षा और सामग्री निर्माण के लिए योगदान के लिए खुली है।


Plan ₿ Network का लक्ष्य Bitcoin साइबरस्पेस का सार्वजनिक केंद्र बनना है, जो विश्व भर में Bitcoin समुदायों को जोड़ेगा और प्रोत्साहित करेगा।


## योगदान क्यों करें?


Plan ₿ Network मूल रूप से बहुभाषी है, क्योंकि हमारा लक्ष्य Bitcoin संसाधनों को दुनिया भर में सभी के लिए सुलभ बनाना है। हमारा मानना ​​है कि भाषा संप्रभु धन के लिए बाधा नहीं होनी चाहिए।


इसे प्राप्त करने के लिए, हम मानव x AI दृष्टिकोण का उपयोग करते हैं। हमारे [प्रोग्राम](https://github.com/Asi0Flammeus/LLM-Translator) के माध्यम से, हम अपनी सामग्री को स्वचालित रूप से कई भाषाओं में अनुवाद कर सकते हैं। हालाँकि, हमें यह सुनिश्चित करने के लिए कि सामग्री समझने योग्य है और अनुचित शब्दों से मुक्त है, इसकी समीक्षा करने के लिए हमें एक धाराप्रवाह वक्ता की आवश्यकता है।


यदि आप अपने समुदाय में Bitcoin शिक्षा को बढ़ावा देने के लिए इस शैक्षिक परियोजना को बनाने में मदद करना चाहते हैं, तो आप किसी पाठ्यक्रम या ट्यूटोरियल की समीक्षा करके शुरुआत कर सकते हैं।


इसके अतिरिक्त, जैसा कि हम *Value4Value* मॉडल को महत्व देते हैं, हम समीक्षा योगदान को पुरस्कृत करेंगे। यह पुरस्कार सामग्री की लंबाई, हमारे ऑटो-अनुवाद की भाषाई सटीकता और सामग्री की कठिनाई के आधार पर दिया जाएगा।


## क्या मुझे यह जानने की ज़रूरत है कि GitHub कैसे काम करता है?


आपको कंटेंट की समीक्षा करने के लिए GitHub विशेषज्ञ होने की आवश्यकता नहीं है। हालाँकि यह समझना हमेशा बेहतर होता है कि आप क्या कर रहे हैं (और क्यों), आप योगदान देने के लिए बस नीचे दिए गए निर्देशों का पालन कर सकते हैं।


हालाँकि, यदि आप Git और GitHub से परिचित नहीं हैं और अधिक जानना चाहते हैं, तो आप इन उपकरणों पर हमारा परिचयात्मक लेख देख सकते हैं:

## 3. GitHub के Interface के माध्यम से सामग्री की समीक्षा और संपादन करें



- इस अंक में एक विवरण (cf. \**A\**) दिया गया है, जिसमें मूल सामग्री और समीक्षा हेतु फ़ाइल के लिंक दिए गए हैं।
- यदि आप समीक्षा का प्रभार लेना चाहते हैं, तो समस्या पर एक टिप्पणी छोड़ें (cf. \**B\**) यह इंगित करने के लिए कि आप इस पर काम करेंगे। यह दोहराए गए प्रयासों को रोकता है।
- सामग्री तक पहुंचने के लिए समीक्षा हेतु फ़ाइल के आगे दिए गए लिंक पर क्लिक करें (cf. \**C\**)।


![REVIEW](assets/fr/03.webp)



- अब आप समीक्षा के लिए फ़ाइल का GitHub Interface देख सकते हैं.
- बायां पैनल आपको अपनी शाखा (cf. \**A\**) और फ़ाइलों (cf. \**B\**) के माध्यम से नेविगेट करने की अनुमति देता है।
- समीक्षा शुरू करने के लिए पेंसिल आइकन पर क्लिक करें (cf. \**C\**).


![REVIEW](assets/fr/04.webp)



- यदि यह आपका पहला योगदान है, तो GitHub आपको स्रोत रिपॉजिटरी को Fork करने के लिए संकेत देगा। बस बटन पर क्लिक करें।


![REVIEW](assets/fr/05.webp)



- फिर आप संपादक Interface (cf. \**A\**) पर पहुंचेंगे, जहां आप फ़ाइल पथ और संबंधित शाखा (cf. \**B\**) देख सकते हैं और संपादन और पूर्वावलोकन मोड (cf. \**C\**) के बीच स्विच कर सकते हैं।


![REVIEW](assets/fr/06.webp)


ध्यान दें। कोड विंडो में समीक्षा करने की सलाह दी जाती है, जिसमें मूल सामग्री एक अलग विंडो में प्रदर्शित होती है। उदाहरण के लिए, आप Plan ₿ Network वेबसाइट पर या ब्राउज़र टैब पर GitHub पूर्वावलोकन में पाठ देख सकते हैं। कोड दृश्य में त्रुटियों का पता लगाने और सुधार करने के लिए `ctrl + F` का उपयोग करें।


## 4. परिवर्तन के लिए प्रतिबद्ध रहें



- आप आवश्यकतानुसार सामग्री की समीक्षा और सुधार शुरू कर सकते हैं।
- जब आप अपना सत्र रोकना चाहते हैं, तो आप अपने परिवर्तनों को कमिट कर सकते हैं, भले ही समीक्षा पूरी न हुई हो। यह GitHub पर अपनी प्रगति को सहेजने जैसा है।
- "परिवर्तन प्रतिबद्ध करें..." बटन पर क्लिक करें।


![REVIEW](assets/fr/07.webp)



- अपने परिवर्तनों के लिए एक वर्णनात्मक संदेश जोड़ें (cf. \**A\**).
- यदि आवश्यक हो तो विवरण में अधिक संदर्भ जोड़ें (cf. \**B\**).
- "परिवर्तन प्रस्तावित करें" बटन पर क्लिक करें (cf. \**C\**).


![REVIEW](assets/fr/08.webp)



- यदि यह आपका पहला योगदान नहीं है, तो GitHub आपको एक नई शाखा बनाने के लिए संकेत देगा। आपको कभी भी मुख्य `dev` शाखा पर सीधे प्रतिबद्ध नहीं होना चाहिए। यदि आपने पहले से ही किसी मौजूदा कार्यशील शाखा पर प्रतिबद्धताएँ की हैं, तो उस शाखा का चयन करें। अन्यथा, एक नई शाखा बनाने के लिए "एक नई शाखा बनाएँ" विकल्प चुनें।


![REVIEW](assets/fr/13.webp)



- फिर आप अपने सभी बदलावों का सारांश देने वाले पेज पर पहुंचेंगे। इसे स्क्रॉल करके, आप सटीक संशोधन देख सकते हैं: हटाए गए आइटम लाल रंग में दिखाई देते हैं, जबकि जोड़े गए आइटम Green में दिखाई देते हैं।


![REVIEW](assets/fr/09.webp)



- यदि आपने अपनी समीक्षा पूरी नहीं की है और बाद में अपना काम जारी रखना चाहते हैं, तो आप आसानी से बाहर निकल सकते हैं।
- जब तक आपने अपने परिवर्तन प्रतिबद्ध कर दिए हैं, वे आपके GitHub Fork पर सहेजे जाते हैं।
- अपना काम फिर से शुरू करने के लिए, अपने GitHub प्रोफ़ाइल के माध्यम से Plan ₿ Network सामग्री रिपॉजिटरी के Fork पर जाएं।


![REVIEW](assets/fr/14.webp)



- अपनी प्रगति वाली शाखा का चयन करें (cf. \**A\**), अपनी सामग्री फ़ाइल का पता लगाएं (cf. \**B\**), फिर समीक्षा जारी रखने के लिए पेंसिल आइकन पर क्लिक करें।


![REVIEW](assets/fr/15.webp)



## 5. समर्पित शाखा के साथ एक पीआर बनाएं



- यदि सब कुछ ठीक लगता है और आपने अपने परिवर्तन पूरे कर लिए हैं, तो Green "पुल अनुरोध बनाएँ" बटन पर क्लिक करें।


![REVIEW](assets/fr/10.webp)



- फिर आप पीआर पेज पर पहुंचेंगे। पुल रिक्वेस्ट एक रिमोट रिपोजिटरी की शाखा पर किए गए परिवर्तनों को समीक्षा और मुख्य शाखा में संभावित विलय के लिए प्रस्तुत करने का अनुरोध है।
- अपने PR में एक शीर्षक और संक्षिप्त विवरण जोड़ें। शीर्षक उस मुद्दे से मेल खाना चाहिए जिस पर आप काम कर रहे हैं। उदाहरण के लिए, यदि मुद्दे का शीर्षक `[अनुवाद] कोर्स CYP201 - FR` है, तो अपने PR के लिए उसी शीर्षक का उपयोग करें (cf. \**A\**)। विवरण में, संबंधित मुद्दे का उल्लेख उसके नंबर का उपयोग करके करें, उसके पहले "बंद हो जाता है"। PR के मर्ज हो जाने पर यह स्वचालित रूप से मुद्दे को बंद कर देगा (cf. \**B\**)।
- यदि सब कुछ ठीक लगे, तो Green "पुल अनुरोध बनाएं" बटन पर क्लिक करें (cf. \**C\**).


![REVIEW](assets/fr/11.webp)


## 6. सामग्री की प्रूफरीडिंग करते समय पालन किए जाने वाले नियम



- नया प्रूफ़रीडिंग शुरू करने से पहले, [टेलीग्राम ग्रुप](https://t.me/PlanBNetwork_ContentBuilder) में इसकी घोषणा करें या अपने Plan ₿ Network समन्वयक को बताएं। वे एक [मुद्दा](https://docs.github.com/en/issues/tracking-your-work-with-issues/about-issues) खोलेंगे, जहाँ आप टिप्पणी कर सकते हैं कि आप काम शुरू कर रहे हैं।
- प्रूफ़रीडिंग करते समय हमेशा कोर्स/ट्यूटोरियल के मूल अंग्रेज़ी (या फ़्रेंच) संस्करण की जाँच करना महत्वपूर्ण है। प्रवाह को बढ़ाने के लिए समायोजन करने के लिए स्वतंत्र महसूस करें, लेकिन समग्र मूल अर्थ को न बदलें।
- एलएलएम उपकरण Bitcoin से संबंधित कुछ शब्दों का शाब्दिक अनुवाद कर सकते हैं, जैसे Lightning Network, जो इतालवी में "रेटे फुलमाइन" बन जाएगा। इस तरह के मामलों में, बेहतर स्पष्टता के लिए मूल अंग्रेजी शब्द का उपयोग करना उचित है, जब तक कि आपके भाषा नियम आपको हर एक शब्द का अनुवाद करने के लिए बाध्य न करें।
- यदि आपको उद्धरण मिलें, तो अपनी भाषा में सही अनुवाद खोजने के लिए ऑनलाइन कुछ शोध अवश्य करें।
- पाठ के आरंभ में दिए गए स्थिर शब्दों का अनुवाद न करें, जिनका प्रयोग उसकी विषय-वस्तु का वर्णन करने के लिए किया जाता है: "नाम", "वर्णन", "उद्देश्य"।
- यदि आपको पाठ में "Plan ₿ Network" मिलता है, तो इसे इसी मूल रूप में छोड़ दें। "योजना" या "नेटवर्क" शब्द का अनुवाद न करें। इसके अलावा, Plan ₿ Network का परिचय देते समय "The" लेख का उपयोग न करें, और इसे एक ब्रांड के रूप में मानें। यही बात "₿-CERT", "BIZ SCHOOL", "TECH SCHOOL" पर भी लागू होती है, जिन्हें भी मूल रूप में रखा जाना चाहिए।
- `\*\*\` जैसे अजीब विराम चिह्नों पर ध्यान दें जो बोल्ड प्रतीक ** के खराब रेंडरिंग को दर्शा सकते हैं। हमेशा मूल अंग्रेजी पाठ की जाँच करें कि क्या बोल्ड टेक्स्ट होना चाहिए। सामान्य तौर पर, बोल्ड रेंडर करने के लिए, कृपया शब्द/वाक्य से पहले और बाद में दो तारांकन चिह्न ** डालें। आप मार्कडाउन भाषा के बारे में अधिक जानकारी [यहाँ](https://www.markdownguide.org/basic-syntax/) पा सकते हैं।
- जब आप अपनी प्रूफरीडिंग पूरी कर लें, तो आप समन्वयक द्वारा खोले गए मूल अंक पर वापस जा सकते हैं, "प्रूफरीडिंग पूर्ण" के साथ टिप्पणी कर सकते हैं, और वहां अपना पीआर लिंक डाल सकते हैं।
