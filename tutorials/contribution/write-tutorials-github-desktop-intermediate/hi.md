## 1 - एक नई शाखा बनाएँ


अपना ब्राउज़र खोलें और Plan ₿ Network रिपॉजिटरी के अपने Fork के पेज पर जाएँ। यह वह Fork है जिसे आपने GitHub पर स्थापित किया है। आपके Fork का URL इस तरह दिखना चाहिए: `https://github.com/[your-username]/Bitcoin-educational-content`:


![TUTO](assets/fr/03.webp)


सुनिश्चित करें कि आप मुख्य शाखा `dev` पर हैं, फिर `Sync Fork` बटन पर क्लिक करें। यदि आपका Fork अद्यतित नहीं है, तो GitHub आपकी शाखा को अपडेट करने की पेशकश करेगा। इस अपडेट के साथ आगे बढ़ें। यदि, इसके विपरीत, आपकी शाखा पहले से ही अद्यतित है, तो GitHub आपको सूचित करेगा:


![TUTO](assets/fr/04.webp)


GitHub डेस्कटॉप सॉफ्टवेयर खोलें और सुनिश्चित करें कि आपका Fork विंडो के ऊपरी बाएं कोने में सही ढंग से चयनित है:


![TUTO](assets/fr/05.webp)


`Fetch origin` बटन पर क्लिक करें। यदि आपका स्थानीय रिपॉजिटरी पहले से ही अद्यतित है, तो GitHub Desktop कोई अतिरिक्त कार्रवाई का सुझाव नहीं देगा। अन्यथा, `Pull origin` विकल्प दिखाई देगा। अपने स्थानीय रिपॉजिटरी को अपडेट करने के लिए इस बटन पर क्लिक करें:


![TUTO](assets/fr/06.webp)


सत्यापित करें कि आप वास्तव में मुख्य शाखा `dev` पर हैं:


![TUTO](assets/fr/07.webp)


इस शाखा पर क्लिक करें, फिर `नई शाखा` बटन पर क्लिक करें:


![TUTO](assets/fr/08.webp)


सुनिश्चित करें कि नई शाखा स्रोत रिपोजिटरी, अर्थात् `PlanB-Network/Bitcoin-educational-content` पर आधारित है।


अपनी शाखा का नाम इस तरह रखें कि शीर्षक से उसका उद्देश्य स्पष्ट हो, प्रत्येक शब्द को अलग करने के लिए डैश का उपयोग करें। उदाहरण के लिए, मान लें कि हमारा लक्ष्य स्पैरो Wallet सॉफ़्टवेयर का उपयोग करने पर एक ट्यूटोरियल लिखना है। इस मामले में, इस ट्यूटोरियल को लिखने के लिए समर्पित कार्यशील शाखा का नाम हो सकता है: `tuto-sparrow-Wallet-loic`। एक बार उपयुक्त नाम दर्ज हो जाने के बाद, शाखा के निर्माण की पुष्टि करने के लिए `शाखा बनाएँ` पर क्लिक करें:


![TUTO](assets/fr/09.webp)


अब अपनी नई कार्यशील शाखा को GitHub पर अपने ऑनलाइन Fork में सहेजने के लिए `शाखा प्रकाशित करें` बटन पर क्लिक करें:

![TUTORIAL](assets/fr/10.webp)

अब, GitHub डेस्कटॉप पर, आपको खुद को अपनी नई शाखा पर पाना चाहिए। इसका मतलब है कि आपके कंप्यूटर पर स्थानीय रूप से किए गए सभी परिवर्तन विशेष रूप से इस विशिष्ट शाखा पर सहेजे जाएंगे। साथ ही, जब तक यह शाखा GitHub डेस्कटॉप पर चयनित रहती है, तब तक आपकी मशीन पर स्थानीय रूप से दिखाई देने वाली फ़ाइलें इस शाखा (`tuto-sparrow-Wallet-loic`) से मेल खाती हैं, न कि मुख्य शाखा (`dev`) से।


![TUTORIAL](assets/fr/11.webp)


प्रत्येक नए लेख के लिए जिसे आप प्रकाशित करना चाहते हैं, आपको `dev` से एक नई शाखा बनाने की आवश्यकता होगी। Git में एक शाखा परियोजना का एक समानांतर संस्करण है, जो आपको मुख्य शाखा को प्रभावित किए बिना परिवर्तन करने की अनुमति देता है, जब तक कि कार्य विलय के लिए तैयार न हो जाए।


## 2 - ट्यूटोरियल फ़ाइलें जोड़ना


अब जब वर्किंग ब्रांच बन गई है, तो अपने नए ट्यूटोरियल को एकीकृत करने का समय आ गया है। आपके पास दो विकल्प हैं: मेरी पायथन स्क्रिप्ट का उपयोग करें, जो आवश्यक दस्तावेज़ों के निर्माण को स्वचालित करती है, या प्रत्येक फ़ाइल को मैन्युअल रूप से बनाएँ। हम प्रत्येक विकल्प के लिए अनुसरण करने के चरणों को देखेंगे।


### मेरी पायथन स्क्रिप्ट के साथ


आपको अपनी मशीन पर यह स्थापित करना होगा:


- पायथन 3.8 या उच्चतर.


स्क्रिप्ट का उपयोग करने के लिए, उस फ़ोल्डर पर जाएँ जहाँ यह संग्रहीत है। स्क्रिप्ट Plan ₿ Network डेटा रिपॉजिटरी में इस पथ पर स्थित है: `Bitcoin-educational-content/scripts/tutorial-related/data-creator`.


एक बार फ़ोल्डर में, निर्भरताएँ स्थापित करें:


```
pip install -r requirements.txt
```


फिर निम्न आदेश के साथ सॉफ्टवेयर लॉन्च करें:


```
python3 main.py
```


एक ग्राफ़िकल यूजर Interface (GUI) खुलेगा। पहली बार, आपको सभी ज़रूरी जानकारी दर्ज करनी होगी, लेकिन बाद के इस्तेमाल में, स्क्रिप्ट आपकी व्यक्तिगत जानकारी याद रखेगी, इसलिए आपको इसे दोबारा दर्ज नहीं करना पड़ेगा।


![DATA-CREATOR-PY](assets/fr/37.webp)


अपने क्लोन किए गए रिपॉजिटरी (`.../Bitcoin-educational-content/tutorials/`) में `/tutorials` फ़ोल्डर का स्थानीय पथ दर्ज करके शुरू करें। आप इसे मैन्युअल रूप से दर्ज कर सकते हैं या अपने फ़ाइल एक्सप्लोरर का उपयोग करके नेविगेट करने के लिए "ब्राउज़ करें" बटन पर क्लिक कर सकते हैं।


![DATA-CREATOR-PY](assets/fr/38.webp)


वह भाषा चुनें जिसमें आप अपना ट्यूटोरियल लिखेंगे।


![DATA-CREATOR-PY](assets/fr/39.webp)


"योगदानकर्ता की GitHub ID" फ़ील्ड में, अपना GitHub उपयोगकर्ता नाम दर्ज करें.


![DATA-CREATOR-PY](assets/fr/40.webp)


इसके बाद, आपको अपना प्रोफेसर प्रोफाइल भरना होगा। आपके पास कई विकल्प उपलब्ध हैं:


- "प्रोफ़ेसर का नाम" फ़ील्ड में अपने नाम के पहले अक्षर दर्ज करें। फिर आपका नाम नीचे "प्रोफ़ेसर सुझाव" ड्रॉपडाउन सूची में दिखाई देगा। उस पर क्लिक करके उसे चुनें;
- वैकल्पिक रूप से, आप सीधे "प्रोफेसर सुझाव" ड्रॉपडाउन सूची पर क्लिक कर सकते हैं और अपने प्रोफेसर का नाम चुन सकते हैं।


यह क्रिया स्वचालित रूप से आपके प्रोफेसर का UUID संबंधित फ़ील्ड में भर देगी।



![DATA-CREATOR-PY](assets/fr/41.webp)


यदि आपके पास अभी तक कोई प्रोफेसर प्रोफ़ाइल नहीं है, तो यह ट्यूटोरियल देखें:


https://planb.network/tutorials/contribution/others/create-teacher-profile-8ba9ba49-8fac-437a-a435-c38eebc8f8a4

फिर "नया ट्यूटोरियल" बटन पर क्लिक करें।


![DATA-CREATOR-PY](assets/fr/42.webp)


अपने ट्यूटोरियल के लिए एक मुख्य श्रेणी चुनें। फिर, अपनी चुनी हुई मुख्य श्रेणी के आधार पर एक प्रासंगिक उपश्रेणी चुनें।


![DATA-CREATOR-PY](assets/fr/43.webp)


ट्यूटोरियल का कठिनाई स्तर निर्धारित करें।


![DATA-CREATOR-PY](assets/fr/44.webp)


अपने ट्यूटोरियल के लिए विशेष रूप से बनाई गई निर्देशिका के लिए एक नाम चुनें। इस फ़ोल्डर का नाम ट्यूटोरियल में शामिल सॉफ़्टवेयर को दर्शाना चाहिए, शब्दों को अलग करने के लिए हाइफ़न का उपयोग करना चाहिए। उदाहरण के लिए, फ़ोल्डर का नाम `red-Wallet` रखा जा सकता है:


![DATA-CREATOR-PY](assets/fr/45.webp)


`project_id` ट्यूटोरियल में शामिल टूल के पीछे की कंपनी या संगठन का UUID है, जो [प्रोजेक्ट की सूची में](https://github.com/PlanB-Network/Bitcoin-educational-content/tree/dev/resources/projects) उपलब्ध है। उदाहरण के लिए, स्पैरो Wallet पर एक ट्यूटोरियल के लिए, आप इसकी `project_id` फ़ाइल में पा सकते हैं: `Bitcoin-educational-content/resources/projects/sparrow/project.yml`। यह जानकारी आपके ट्यूटोरियल की YAML फ़ाइल में इसलिए जोड़ी जाती है क्योंकि Plan ₿ Network Bitcoin या संबंधित प्रोजेक्ट में सक्रिय कंपनियों और संगठनों का डेटाबेस बनाए रखता है। संबंधित `project_id` जोड़कर, आप अपनी सामग्री को संबंधित इकाई से लिंक करते हैं।


***अपडेट:*** स्क्रिप्ट के नए संस्करण में, अब आपको `project_id` को मैन्युअल रूप से दर्ज करने की आवश्यकता नहीं है। नाम से प्रोजेक्ट को खोजने और संबंधित `project_id` को स्वचालित रूप से प्राप्त करने के लिए एक खोज फ़ंक्शन जोड़ा गया है। इसे खोजने के लिए "प्रोजेक्ट नाम" फ़ील्ड में प्रोजेक्ट के नाम की शुरुआत टाइप करें, फिर ड्रॉपडाउन मेनू से वांछित कंपनी का चयन करें। `project_id` नीचे दिए गए फ़ील्ड में स्वचालित रूप से पॉप्युलेट हो जाएगा। यदि आवश्यक हो तो आप इसे मैन्युअल रूप से भी दर्ज कर सकते हैं।


![DATA-CREATOR-PY](assets/fr/46.webp)


टैग के लिए, अपने ट्यूटोरियल कंटेंट से संबंधित 2 या 3 प्रासंगिक कीवर्ड चुनें, विशेष रूप से [Plan ₿ Network टैग सूची](https://github.com/PlanB-Network/Bitcoin-educational-content/blob/dev/docs/50-planb-tags.md) से चुनें। सॉफ़्टवेयर ड्रॉपडाउन सूची के साथ कीवर्ड खोज फ़ंक्शन भी प्रदान करता है।


![DATA-CREATOR-PY](assets/fr/47.webp)

- आपके ट्यूटोरियल से संबंधित विवरण रिकॉर्ड करने के लिए एक `tutorial.yml` फ़ाइल बनाई जानी चाहिए;

## 3 - YAML फ़ाइल भरें


निम्नलिखित टेम्पलेट की प्रतिलिपि बनाकर `tutorial.yml` फ़ाइल भरें:


```
id:

project_id:

tags:
-
-
-

category:

level:

professor_id:

# Proofreading metadata

original_language:
proofreading:
- language:
last_contribution_date:
urgency:
contributor_names:
-
reward:
```


आवश्यक फ़ील्ड इस प्रकार हैं:



- id** : एक UUID (_यूनिवर्सली यूनिक आइडेंटिफ़ायर_) जो ट्यूटोरियल को विशिष्ट रूप से पहचानता है। आप इसे [ऑनलाइन टूल](https://www.uuidgenerator.net/version4) का उपयोग करके generate कर सकते हैं। एकमात्र आवश्यकता यह है कि यह UUID प्लेटफ़ॉर्म पर किसी अन्य UUID के साथ टकराव से बचने के लिए यादृच्छिक हो;



- project_id** : ट्यूटोरियल में प्रस्तुत टूल के पीछे कंपनी या संगठन का UUID [प्रोजेक्ट सूची से](https://github.com/PlanB-Network/Bitcoin-educational-content/tree/dev/resources/projects)। उदाहरण के लिए, यदि आप Green Wallet सॉफ़्टवेयर के बारे में एक ट्यूटोरियल बना रहे हैं, तो आप इस `project_id` को निम्न फ़ाइल में पा सकते हैं: `Bitcoin-educational-content/resources/projects/blockstream/project.yml`। यह जानकारी आपके ट्यूटोरियल की YAML फ़ाइल में इसलिए जोड़ी जाती है क्योंकि Plan ₿ Network Bitcoin या संबंधित प्रोजेक्ट पर काम करने वाली सभी कंपनियों और संगठनों का डेटाबेस बनाए रखता है। अपने ट्यूटोरियल से जुड़ी इकाई का `project_id` जोड़कर, आप दो Elements के बीच एक लिंक बनाते हैं;



- टैग**: ट्यूटोरियल सामग्री से संबंधित 2 या 3 प्रासंगिक कीवर्ड, विशेष रूप से [Plan ₿ Network टैग सूची से](https://github.com/PlanB-Network/Bitcoin-educational-content/blob/dev/docs/50-planb-tags.md) चुने गए;



- श्रेणी** : Plan ₿ Network वेबसाइट संरचना के अनुसार ट्यूटोरियल सामग्री के अनुरूप उप-श्रेणी (उदाहरण के लिए, वॉलेट के लिए: `डेस्कटॉप`, `हार्डवेयर`, `मोबाइल`, `बैकअप`);



- स्तर** : ट्यूटोरियल का कठिनाई स्तर, निम्न में से चुना गया:
    - `शुरुआती`
    - `मध्यवर्ती`
    - `उन्नत`
    - `विशेषज्ञ`



- professor_id** : आपका `professor_id` (UUID) जैसा कि [आपके प्रोफेसर प्रोफ़ाइल](https://github.com/PlanB-Network/Bitcoin-educational-content/tree/dev/professors) पर प्रदर्शित है;



- original_language** : ट्यूटोरियल की मूल भाषा (जैसे, `fr`, `en`, आदि);



- प्रूफ़रीडिंग** : प्रूफ़रीडिंग प्रक्रिया के बारे में जानकारी। पहला भाग पूरा करें, क्योंकि अपने स्वयं के ट्यूटोरियल की प्रूफ़रीडिंग को प्रथम सत्यापन के रूप में गिना जाता है:
    - भाषा** : प्रूफ़रीडिंग का भाषा कोड (जैसे, `fr`, `en`, आदि).
    - last_contribution_date** : दिन की तारीख.
    - तात्कालिकता** : 1
    - योगदानकर्ता_नाम** : आपकी GitHub आईडी.
    - इनाम** : 0


अपने शिक्षक आईडी के बारे में अधिक जानकारी के लिए कृपया संबंधित ट्यूटोरियल देखें:


https://planb.network/tutorials/contribution/others/create-teacher-profile-8ba9ba49-8fac-437a-a435-c38eebc8f8a4

## 5 - लोगो और कवर जोड़ें

एक बार जब आप अपनी पसंद की भाषा में अपना ट्यूटोरियल लिखना समाप्त कर लेते हैं, तो अगला चरण **पुल रिक्वेस्ट** सबमिट करना होता है। फिर एडमिनिस्ट्रेटर आपके ट्यूटोरियल के किसी भी छूटे हुए अनुवाद को जोड़ने का ध्यान रखेगा, यह काम मानवीय समीक्षा के साथ हमारी स्वचालित अनुवाद पद्धति के ज़रिए किया जाता है।


पुल रिक्वेस्ट के साथ आगे बढ़ने के लिए, GitHub डेस्कटॉप सॉफ़्टवेयर खोलें। सॉफ़्टवेयर को मूल रिपॉजिटरी की तुलना में आपकी शाखा पर स्थानीय रूप से आपके द्वारा किए गए परिवर्तनों का स्वचालित रूप से पता लगाना चाहिए। आगे बढ़ने से पहले, Interface के बाईं ओर ध्यान से जाँच करें कि ये परिवर्तन आपकी अपेक्षा से मेल खाते हैं:
