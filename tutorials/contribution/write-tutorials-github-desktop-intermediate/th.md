## ข้อกำหนดเบื้องต้น


ซอฟต์แวร์ที่จำเป็นสำหรับการติดตามบทแนะนำนี้:



- [GitHub Desktop](https://desktop.github.com/);
- โปรแกรมแก้ไขไฟล์มาร์กดาวน์เช่น [Obsidian](https://obsidian.md/);
- โปรแกรมแก้ไขโค้ด ([VSC](https://code.visualstudio.com/) หรือ [Sublime Text](https://www.sublimetext.com/))


![TUTO](assets/fr/01.webp)


ข้อกำหนดเบื้องต้นก่อนเริ่มการสอน:



- มี[บัญชี GitHub](https://github.com/signup);
- มี fork ของ [ที่เก็บต้นทาง Plan ₿ Academy](https://github.com/PlanB-Network/bitcoin-educational-content);
- มี[โปรไฟล์อาจารย์บน Plan ₿ Academy](https://planb.academy/professors) (เฉพาะเมื่อคุณเสนอการสอนแบบสมบูรณ์)


หากคุณต้องการความช่วยเหลือในการรับข้อกำหนดเบื้องต้นเหล่านี้ บทแนะนำอื่น ๆ ของฉันจะช่วยคุณได้:



เมื่อทุกอย่างถูกจัดเตรียมเรียบร้อยและสภาพแวดล้อมในเครื่องของคุณถูกตั้งค่าอย่างถูกต้องด้วย fork ของคุณเองจาก Plan ₿ Academy คุณสามารถเริ่มเพิ่มบทแนะนำได้


## 1 - สร้างสาขาใหม่


เปิดเบราว์เซอร์ของคุณและไปที่หน้าของ fork ในที่เก็บ Plan ₿ Academy ของคุณ นี่คือ fork ที่คุณได้สร้างขึ้นบน GitHub URL ของ fork ของคุณควรมีลักษณะดังนี้: `https://github.com/[your-username]/bitcoin-educational-content`:


![TUTO](assets/fr/03.webp)


ตรวจสอบให้แน่ใจว่าคุณอยู่บนสาขาหลัก `dev` จากนั้นคลิกที่ปุ่ม `Sync fork` หาก fork ของคุณไม่ทันสมัย GitHub จะเสนอให้คุณอัปเดตสาขาของคุณ ดำเนินการอัปเดตนี้ หากในทางตรงกันข้าม สาขาของคุณทันสมัยแล้ว GitHub จะแจ้งให้คุณทราบ:


![TUTO](assets/fr/04.webp)


เปิดซอฟต์แวร์ GitHub Desktop และตรวจสอบให้แน่ใจว่า fork ของคุณถูกเลือกอย่างถูกต้องที่มุมซ้ายบนของหน้าต่าง:


![TUTO](assets/fr/05.webp)


คลิกที่ปุ่ม `Fetch origin` หากที่เก็บข้อมูลในเครื่องของคุณเป็นปัจจุบันแล้ว GitHub Desktop จะไม่แนะนำการดำเนินการเพิ่มเติมใดๆ มิฉะนั้น ตัวเลือก `Pull origin` จะปรากฏขึ้น คลิกที่ปุ่มนี้เพื่ออัปเดตที่เก็บข้อมูลในเครื่องของคุณ:


![TUTO](assets/fr/06.webp)


ยืนยันว่าคุณอยู่บนสาขาหลัก `dev`:


![TUTO](assets/fr/07.webp)


คลิกที่สาขานี้ จากนั้นคลิกที่ปุ่ม `New Branch`:


![TUTO](assets/fr/08.webp)


ตรวจสอบให้แน่ใจว่าสาขาใหม่อิงตามที่เก็บต้นทาง คือ `Plan ₿ Academy/bitcoin-educational-content`


ตั้งชื่อสาขาของคุณในลักษณะที่ชื่อมีความชัดเจนเกี่ยวกับวัตถุประสงค์ โดยใช้ขีดกลางเพื่อแยกแต่ละคำ ตัวอย่างเช่น สมมติว่าเป้าหมายของเราคือการเขียนบทแนะนำเกี่ยวกับการใช้ซอฟต์แวร์ Sparrow Wallet ในกรณีนี้ สาขาการทำงานที่อุทิศให้กับการเขียนบทแนะนำนี้อาจถูกตั้งชื่อว่า: `tuto-sparrow-wallet-loic` เมื่อป้อนชื่อที่เหมาะสมแล้ว ให้คลิกที่ `Create branch` เพื่อยืนยันการสร้างสาขา:


![TUTO](assets/fr/09.webp)


ตอนนี้คลิกที่ปุ่ม `Publish branch` เพื่อบันทึกสาขาการทำงานใหม่ของคุณไปยัง fork ออนไลน์ของคุณบน GitHub:

![TUTORIAL](assets/fr/10.webp)

ตอนนี้ บน GitHub Desktop คุณควรจะอยู่บนสาขาใหม่ของคุณ ซึ่งหมายความว่าการเปลี่ยนแปลงทั้งหมดที่ทำในเครื่องคอมพิวเตอร์ของคุณจะถูกบันทึกไว้เฉพาะในสาขานี้เท่านั้น นอกจากนี้ ตราบใดที่สาขานี้ยังคงถูกเลือกบน GitHub Desktop ไฟล์ที่มองเห็นได้ในเครื่องของคุณจะสอดคล้องกับไฟล์ของสาขานี้ (`tuto-sparrow-wallet-loic`) และไม่ใช่ไฟล์ของสาขาหลัก (`dev`)


![TUTORIAL](assets/fr/11.webp)


สำหรับบทความใหม่แต่ละบทความที่คุณต้องการเผยแพร่ คุณจะต้องสร้างสาขาใหม่จาก `dev` สาขาใน Git คือเวอร์ชันคู่ขนานของโปรเจ็กต์ ซึ่งช่วยให้คุณสามารถทำการเปลี่ยนแปลงได้โดยไม่กระทบต่อสาขาหลัก จนกว่างานจะพร้อมที่จะถูกรวมเข้าด้วยกัน


## 2 - เพิ่มไฟล์สอน


ตอนนี้ที่สร้างสาขาการทำงานแล้ว ก็ถึงเวลาที่จะรวมบทแนะนำใหม่ของคุณ คุณมีสองตัวเลือก: ใช้สคริปต์ Python ของฉัน ซึ่งจะทำการสร้างเอกสารที่จำเป็นโดยอัตโนมัติ หรือสร้างแต่ละไฟล์ด้วยตนเอง เราจะดูขั้นตอนที่ต้องปฏิบัติตามสำหรับแต่ละตัวเลือก


### ด้วยสคริปต์ Python ของฉัน


คุณจำเป็นต้องติดตั้งบนเครื่องของคุณ:


- Python 3.8 หรือสูงกว่า


ในการใช้สคริปต์ ให้ไปที่โฟลเดอร์ที่เก็บสคริปต์ไว้ สคริปต์นี้อยู่ในที่เก็บข้อมูล Plan ₿ Academy ที่เส้นทาง: `bitcoin-educational-content/scripts/tutorial-related/data-creator`.


เมื่ออยู่ในโฟลเดอร์ ให้ติดตั้ง dependencies:


```
pip install -r requirements.txt
```


จากนั้นเปิดซอฟต์แวร์ด้วยคำสั่ง:


```
python3 main.py
```


ส่วนติดต่อผู้ใช้แบบกราฟิก (GUI) จะเปิดขึ้น ในครั้งแรกคุณจะต้องป้อนข้อมูลที่จำเป็นทั้งหมด แต่ในการใช้งานครั้งถัดไป สคริปต์จะจดจำข้อมูลส่วนตัวของคุณ ดังนั้นคุณจะไม่ต้องป้อนอีกครั้ง


![DATA-CREATOR-PY](assets/fr/37.webp)


เริ่มต้นโดยการป้อนเส้นทางในเครื่องไปยังโฟลเดอร์ `/tutorials` ในที่เก็บที่คุณโคลนไว้ (`.../bitcoin-educational-content/tutorials/`) คุณสามารถป้อนด้วยตนเองหรือคลิกปุ่ม "Browse" เพื่อสำรวจโดยใช้ตัวสำรวจไฟล์ของคุณ


![DATA-CREATOR-PY](assets/fr/38.webp)


เลือกภาษาที่คุณจะใช้เขียนบทแนะนำของคุณ


![DATA-CREATOR-PY](assets/fr/39.webp)

![DATA-CREATOR-PY](assets/fr/40.webp)


ถัดไป คุณจำเป็นต้องกรอกโปรไฟล์อาจารย์ของคุณ มีตัวเลือกหลายอย่างที่คุณสามารถเลือกได้:

- นอกจากนี้ คุณสามารถคลิกที่รายการแบบเลื่อนลง "Prof. Suggestions" โดยตรงและเลือกชื่อศาสตราจารย์ของคุณได้


การดำเนินการนี้จะกรอก UUID ของศาสตราจารย์ของคุณในช่องที่เกี่ยวข้องโดยอัตโนมัติ



![DATA-CREATOR-PY](assets/fr/41.webp)

`project_id` คือ UUID ของบริษัทหรือองค์กรที่อยู่เบื้องหลังเครื่องมือที่ครอบคลุมในบทแนะนำ ซึ่งสามารถหาได้ [ในรายการโครงการ](https://github.com/PlanB-Network/bitcoin-educational-content/tree/dev/resources/projects) ตัวอย่างเช่น สำหรับบทแนะนำเกี่ยวกับ Sparrow Wallet คุณสามารถหา `project_id` ได้ในไฟล์: `bitcoin-educational-content/resources/projects/sparrow/project.yml` ข้อมูลนี้ถูกเพิ่มลงในไฟล์ YAML ของบทแนะนำของคุณเพราะ Plan ₿ Academy ดูแลฐานข้อมูลของบริษัทและองค์กรที่มีความเคลื่อนไหวใน Bitcoin หรือโครงการที่เกี่ยวข้อง โดยการเพิ่ม `project_id` ที่เกี่ยวข้อง คุณจะเชื่อมโยงเนื้อหาของคุณกับหน่วยงานที่เกี่ยวข้อง


***อัปเดต:*** ในเวอร์ชันใหม่ของสคริปต์ คุณไม่จำเป็นต้องป้อน `project_id` ด้วยตนเองอีกต่อไป มีการเพิ่มฟังก์ชันการค้นหาเพื่อค้นหาโปรเจกต์ตามชื่อและดึง `project_id` ที่สอดคล้องกันโดยอัตโนมัติ พิมพ์ชื่อโปรเจกต์ในช่อง "Project Name" เพื่อค้นหา จากนั้นเลือกบริษัทที่ต้องการจากเมนูแบบเลื่อนลง `project_id` จะถูกกรอกในช่องด้านล่างโดยอัตโนมัติ คุณยังสามารถป้อนด้วยตนเองหากจำเป็น

![DATA-CREATOR-PY](assets/fr/47.webp)


เมื่อป้อนและตรวจสอบข้อมูลทั้งหมดแล้ว ให้คลิก "Create Tutorial" เพื่อยืนยันการสร้างไฟล์บทแนะนำของคุณ สิ่งนี้จะ generate โฟลเดอร์บทแนะนำของคุณและไฟล์ที่จำเป็นทั้งหมดในหมวดหมู่ที่เลือกไว้ในเครื่อง

สำหรับข้อมูลเพิ่มเติมเกี่ยวกับสคริปต์ Python นี้ คุณสามารถตรวจสอบได้ที่ [README](https://github.com/PlanB-Network/bitcoin-educational-content/blob/dev/scripts/tutorial-related/new-tutorial-creation/README.md)

## 3 - กรอกข้อมูลในไฟล์ YAML


กรอกไฟล์ `tutorial.yml` โดยคัดลอกเทมเพลตต่อไปนี้:


```
id:

project_id:

tags:
-
-
-

category:

level:

professor_id:

# Proofreading metadata

original_language:
proofreading:
- language:
last_contribution_date:
urgency:
contributor_names:
-
reward:
```


นี่คือฟิลด์ที่จำเป็น:



- id**: UUID (_Universally Unique Identifier_) ที่ระบุบทเรียนอย่างเฉพาะเจาะจง คุณสามารถ generate มันได้โดยใช้ [เครื่องมือออนไลน์](https://www.uuidgenerator.net/version4) ข้อกำหนดเดียวคือ UUID นี้ต้องเป็นแบบสุ่มเพื่อหลีกเลี่ยงความขัดแย้งกับ UUID อื่นบนแพลตฟอร์ม;



- project_id**: UUID ของบริษัทหรือองค์กรที่อยู่เบื้องหลังเครื่องมือที่นำเสนอในบทแนะนำ [จากรายการโครงการ](https://github.com/PlanB-Network/bitcoin-educational-content/tree/dev/resources/projects) ตัวอย่างเช่น หากคุณกำลังสร้างบทแนะนำเกี่ยวกับซอฟต์แวร์ Green Wallet คุณสามารถค้นหา `project_id` นี้ได้ในไฟล์ต่อไปนี้: `bitcoin-educational-content/resources/projects/blockstream/project.yml` ข้อมูลนี้จะถูกเพิ่มลงในไฟล์ YAML ของบทแนะนำของคุณเพราะ Plan ₿ Academy รักษาฐานข้อมูลของบริษัทและองค์กรทั้งหมดที่ดำเนินการบน Bitcoin หรือโครงการที่เกี่ยวข้อง โดยการเพิ่ม `project_id` ของหน่วยงานที่เชื่อมโยงกับบทแนะนำของคุณ คุณจะสร้างลิงก์ระหว่างสององค์ประกอบนี้



- tags**: 2 หรือ 3 คำสำคัญที่เกี่ยวข้องกับเนื้อหาการสอน, เลือกเฉพาะ [จากรายการแท็ก Plan ₿ Academy](https://github.com/PlanB-Network/bitcoin-educational-content/blob/dev/docs/50-planb-tags.md);



- หมวดหมู่**: หมวดหมู่ย่อยที่สอดคล้องกับเนื้อหาการสอน ตามโครงสร้างเว็บไซต์ Plan ₿ Academy (ตัวอย่างเช่น สำหรับกระเป๋าเงิน: `desktop`, `hardware`, `mobile`, `backup`);



- ระดับ**: ระดับความยากของบทแนะนำ เลือกจาก:
    - `ผู้เริ่มต้น`
    - `ระดับกลาง`
    - `ขั้นสูง`
    - `ผู้เชี่ยวชาญ`



- professor_id**: `professor_id` (UUID) ของคุณตามที่แสดงใน [โปรไฟล์ศาสตราจารย์ของคุณ](https://github.com/PlanB-Network/bitcoin-educational-content/tree/dev/professors);



- original_language**: ภาษาต้นฉบับของบทแนะนำ (เช่น `fr`, `en`, เป็นต้น);



- การพิสูจน์อักษร**: ข้อมูลเกี่ยวกับกระบวนการพิสูจน์อักษร การทำส่วนแรกให้เสร็จสิ้น เนื่องจากการพิสูจน์อักษรบทเรียนของคุณเองถือเป็นการตรวจสอบครั้งแรก:
    - language**: รหัสภาษาของการพิสูจน์อักษร (เช่น `fr`, `en`, เป็นต้น)
    - last_contribution_date**: วันที่ของวัน
    - ความเร่งด่วน**: 1
    - contributor_names**: Your GitHub ID.
    - reward**: 0


สำหรับรายละเอียดเพิ่มเติมเกี่ยวกับรหัสประจำตัวครูของคุณ โปรดดูที่บทแนะนำที่เกี่ยวข้อง:

## 6 - การเขียนบทแนะนำและการเพิ่มภาพประกอบ

## 7 - บันทึกและส่งบทแนะนำ


เมื่อคุณเขียนบทแนะนำในภาษาที่คุณเลือกเสร็จแล้ว ขั้นตอนต่อไปคือการส่ง **Pull Request** ผู้ดูแลระบบจะดูแลการเพิ่มการแปลที่ขาดหายไปของบทแนะนำของคุณด้วยวิธีการแปลอัตโนมัติของเราที่มีการตรวจสอบโดยมนุษย์


ในการดำเนินการ Pull Request ให้เปิดซอฟต์แวร์ GitHub Desktop ซอฟต์แวร์ควรตรวจจับการเปลี่ยนแปลงที่คุณได้ทำในสาขาของคุณเมื่อเทียบกับที่เก็บต้นฉบับโดยอัตโนมัติ ก่อนดำเนินการต่อ โปรดตรวจสอบอย่างรอบคอบทางด้านซ้ายของอินเทอร์เฟซว่าการเปลี่ยนแปลงเหล่านี้ตรงกับที่คุณคาดหวัง:


![TUTO](assets/fr/28.webp)


เพิ่มชื่อสำหรับการคอมมิตของคุณ จากนั้นคลิกที่ปุ่มสีน้ำเงิน `Commit to [your branch]` เพื่อยืนยันการเปลี่ยนแปลงเหล่านี้:


![TUTO](assets/fr/29.webp)


คอมมิตคือการบันทึกการเปลี่ยนแปลงที่ทำกับสาขา พร้อมด้วยข้อความอธิบาย ซึ่งช่วยให้ติดตามวิวัฒนาการของโครงการตามกาลเวลาได้ มันเหมือนกับจุดตรวจสอบระหว่างทาง


จากนั้นคลิกที่ปุ่ม `Push origin` สิ่งนี้จะส่งคอมมิตของคุณไปยัง fork ของคุณ:


![TUTO](assets/fr/30.webp)


หากคุณยังทำบทเรียนไม่เสร็จ คุณสามารถกลับมาทำต่อได้ในภายหลังและทำการ commit ใหม่ หากคุณทำการเปลี่ยนแปลงสำหรับสาขานี้เสร็จแล้ว ให้คลิกที่ปุ่ม `Preview Pull Request` ตอนนี้:


![TUTO](assets/fr/31.webp)


คุณสามารถตรวจสอบครั้งสุดท้ายว่าการแก้ไขของคุณถูกต้องแล้ว จากนั้นคลิกที่ปุ่ม `Create pull request`:


![TUTO](assets/fr/32.webp)


คำขอดึง (Pull Request) คือคำขอที่ทำขึ้นเพื่อรวมการเปลี่ยนแปลงจากสาขาของคุณไปยังสาขาหลักของที่เก็บ Plan ₿ Academy ซึ่งช่วยให้สามารถตรวจสอบและอภิปรายเกี่ยวกับการเปลี่ยนแปลงก่อนที่จะรวมเข้าด้วยกัน


คุณจะถูกเปลี่ยนเส้นทางไปยังเบราว์เซอร์ของคุณบน GitHub ไปยังหน้าการเตรียมการของคำขอดึงของคุณ:


![TUTO](assets/fr/33.webp)

ระบุชื่อเรื่องที่สรุปการเปลี่ยนแปลงที่คุณต้องการรวมเข้ากับที่เก็บข้อมูลต้นทาง เพิ่มความคิดเห็นสั้น ๆ ที่อธิบายการเปลี่ยนแปลงเหล่านี้ (หากคุณมีหมายเลขปัญหาที่เกี่ยวข้องกับการสร้างบทแนะนำของคุณ อย่าลืมระบุในความคิดเห็น `Closes #{issue number}`) จากนั้นคลิกที่ปุ่มสีเขียว `Create pull request` เพื่อยืนยันคำขอรวม:

![TUTO](assets/fr/34.webp)


PR ของคุณจะปรากฏในแท็บ `Pull Request` ของที่เก็บหลัก Plan ₿ Academy จากนั้นคุณเพียงแค่รอจนกว่าผู้ดูแลระบบจะติดต่อคุณเพื่อยืนยันการรวมการมีส่วนร่วมของคุณหรือขอการแก้ไขเพิ่มเติมใดๆ


![TUTO](assets/fr/35.webp)


หลังจากที่ PR ของคุณถูกรวมเข้ากับสาขาหลักแล้ว ขอแนะนำให้ลบสาขาการทำงานของคุณ (`tuto-sparrow-wallet`) เพื่อรักษาประวัติที่สะอาดบน fork ของคุณ GitHub จะเสนอทางเลือกนี้ให้คุณโดยอัตโนมัติบนหน้าของ PR ของคุณ:


![TUTO](assets/fr/36.webp)


บนซอฟต์แวร์ GitHub Desktop คุณสามารถสลับกลับไปที่สาขาหลักของ fork ของคุณ (`dev`)


![TUTO](assets/fr/07.webp)


หากคุณต้องการแก้ไขการมีส่วนร่วมของคุณหลังจากที่คุณได้ส่ง PR แล้ว ขั้นตอนจะขึ้นอยู่กับสถานะปัจจุบันของ PR ของคุณ:


- หาก PR ของคุณยังเปิดอยู่และยังไม่ได้ถูกรวมเข้าด้วยกัน ให้ทำการเปลี่ยนแปลงในเครื่องในขณะที่ยังคงอยู่บนสาขาเดียวกัน เมื่อการแก้ไขเสร็จสิ้น ให้ใช้ปุ่ม `Push origin` เพื่อเพิ่มคอมมิตใหม่ไปยัง PR ที่ยังเปิดอยู่ของคุณ
- หาก PR ของคุณได้ถูกรวมเข้ากับสาขาหลักแล้ว คุณจะต้องเริ่มกระบวนการใหม่โดยการสร้างสาขาใหม่ จากนั้นส่ง PR ใหม่ ตรวจสอบให้แน่ใจว่าที่เก็บข้อมูลในเครื่องของคุณซิงโครไนซ์กับที่เก็บข้อมูลต้นทาง Plan ₿ Academy ก่อนดำเนินการต่อ


หากคุณพบปัญหาทางเทคนิคในการส่งบทแนะนำของคุณ อย่าลังเลที่จะขอความช่วยเหลือใน [กลุ่ม Telegram สำหรับการมีส่วนร่วมของเรา](https://t.me/PlanBNetwork_ContentBuilder) ขอบคุณ!
