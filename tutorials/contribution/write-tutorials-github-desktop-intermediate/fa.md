- مبتدیان (وب Interface)**:

## پیش‌نیازها


نرم‌افزار مورد نیاز برای دنبال کردن این آموزش:



- [GitHub Desktop](https://desktop.github.com/);
- یک ویرایشگر فایل مارک‌داون مانند [Obsidian](https://obsidian.md/);
- یک ویرایشگر کد ([VSC](https://code.visualstudio.com/) یا [Sublime Text](https://www.sublimetext.com/)).


![TUTO](assets/fr/01.webp)


پیش‌نیازها قبل از شروع آموزش:



- یک [حساب GitHub](https://github.com/signup) داشته باشید؛
- دارای Fork از [مخزن منبع Plan ₿ Network](https://github.com/PlanB-Network/Bitcoin-educational-content) باشید؛
- [یک پروفایل استاد در Plan ₿ Network داشته باشید](https://planb.network/professors) (فقط اگر یک آموزش کامل پیشنهاد می‌دهید).


اگر برای تهیه این پیش‌نیازها به کمک نیاز دارید، آموزش‌های دیگر من به شما کمک خواهند کرد:



پس از اینکه همه چیز در جای خود قرار گرفت و محیط محلی شما به درستی با Fork خودتان از Plan ₿ Network تنظیم شد، می‌توانید شروع به افزودن آموزش کنید.


## ۱ - ایجاد یک شاخه جدید


مرورگر خود را باز کنید و به صفحه Fork از مخزن Plan ₿ Network خود بروید. این همان Fork است که شما در GitHub ایجاد کرده‌اید. URL مربوط به Fork شما باید به این شکل باشد: `https://github.com/[your-username]/Bitcoin-educational-content`:


![TUTO](assets/fr/03.webp)


مطمئن شوید که در شاخه اصلی `dev` هستید، سپس روی دکمه `Sync Fork` کلیک کنید. اگر Fork شما به‌روز نیست، GitHub پیشنهاد به‌روزرسانی شاخه شما را خواهد داد. با این به‌روزرسانی ادامه دهید. اگر برعکس، شاخه شما از قبل به‌روز است، GitHub به شما اطلاع خواهد داد:


![TUTO](assets/fr/04.webp)


نرم‌افزار GitHub Desktop را باز کنید و اطمینان حاصل کنید که Fork شما به‌درستی در گوشه بالا سمت چپ پنجره انتخاب شده است:


![TUTO](assets/fr/05.webp)


روی دکمه `Fetch origin` کلیک کنید. اگر مخزن محلی شما از قبل به‌روز باشد، GitHub Desktop هیچ اقدام اضافی پیشنهاد نخواهد داد. در غیر این صورت، گزینه `Pull origin` ظاهر خواهد شد. برای به‌روزرسانی مخزن محلی خود روی این دکمه کلیک کنید:


![TUTO](assets/fr/06.webp)


تأیید کنید که واقعاً در شاخه اصلی `dev` هستید:


![TUTO](assets/fr/07.webp)


روی این شاخه کلیک کنید، سپس روی دکمه `New Branch` کلیک کنید:


![TUTO](assets/fr/08.webp)


اطمینان حاصل کنید که شاخه جدید بر اساس مخزن منبع، یعنی `PlanB-Network/Bitcoin-educational-content` باشد.


نام شاخه خود را به گونه‌ای انتخاب کنید که عنوان آن به وضوح هدف آن را نشان دهد و از خط تیره برای جدا کردن هر کلمه استفاده کنید. به عنوان مثال، فرض کنید هدف ما نوشتن یک آموزش در مورد استفاده از نرم‌افزار Sparrow wallet است. در این صورت، شاخه کاری اختصاص داده شده به نوشتن این آموزش می‌تواند به این صورت نامگذاری شود: `tuto-Sparrow-Wallet-loic`. پس از وارد کردن نام مناسب، روی `Create branch` کلیک کنید تا ایجاد شاخه تأیید شود:


![TUTO](assets/fr/09.webp)


اکنون روی دکمه `Publish branch` کلیک کنید تا شاخه کاری جدید خود را در Fork آنلاین خود در GitHub ذخیره کنید:

![TUTORIAL](assets/fr/10.webp)

اکنون، در GitHub Desktop، باید خود را در شاخه جدیدتان بیابید. این بدان معناست که تمام تغییراتی که به صورت محلی روی کامپیوترتان انجام می‌دهید، به طور انحصاری در این شاخه خاص ذخیره خواهند شد. همچنین، تا زمانی که این شاخه در GitHub Desktop انتخاب شده باقی بماند، فایل‌هایی که به صورت محلی روی دستگاه شما قابل مشاهده هستند، با فایل‌های این شاخه (`tuto-Sparrow-Wallet-loic`) مطابقت دارند و نه با فایل‌های شاخه اصلی (`dev`).


![TUTORIAL](assets/fr/11.webp)


برای هر مقاله جدیدی که می‌خواهید منتشر کنید، نیاز دارید یک شاخه جدید از `dev` ایجاد کنید. یک شاخه در Git نسخه‌ای موازی از پروژه است که به شما اجازه می‌دهد تغییراتی ایجاد کنید بدون اینکه بر شاخه اصلی تأثیر بگذارد، تا زمانی که کار آماده ادغام شود.


## ۲ - اضافه کردن فایل‌های آموزشی


اکنون که شاخه کاری ایجاد شده است، زمان آن رسیده که آموزش جدید خود را ادغام کنید. شما دو گزینه دارید: از اسکریپت پایتون من استفاده کنید که ایجاد اسناد لازم را خودکار می‌کند، یا هر فایل را به صورت دستی ایجاد کنید. ما به مراحل لازم برای هر گزینه خواهیم پرداخت.


### با اسکریپت پایتون من


شما باید روی دستگاه خود نصب کنید:


- پایتون 3.8 یا بالاتر.


برای استفاده از اسکریپت، به پوشه‌ای که در آن ذخیره شده است بروید. اسکریپت در مخزن داده Plan ₿ Network در مسیر زیر قرار دارد: `Bitcoin-educational-content/scripts/tutorial-related/data-creator`.


پس از ورود به پوشه، وابستگی‌ها را نصب کنید:


```
pip install -r requirements.txt
```


سپس نرم‌افزار را با دستور زیر اجرا کنید:


```
python3 main.py
```


یک رابط کاربری گرافیکی Interface (GUI) باز خواهد شد. در اولین بار، شما باید تمام اطلاعات لازم را وارد کنید، اما در استفاده‌های بعدی، اسکریپت اطلاعات شخصی شما را به خاطر خواهد سپرد، بنابراین نیازی به وارد کردن مجدد آن نخواهید داشت.


![DATA-CREATOR-PY](assets/fr/37.webp)


با وارد کردن مسیر محلی به پوشه `/tutorials` در مخزن کلون شده خود (`.../Bitcoin-educational-content/tutorials/`) شروع کنید. می‌توانید آن را به صورت دستی وارد کنید یا با کلیک بر روی دکمه "Browse" از طریق مرورگر فایل خود پیمایش کنید.


![DATA-CREATOR-PY](assets/fr/38.webp)


زبانی را انتخاب کنید که می‌خواهید آموزش خود را به آن بنویسید.


![DATA-CREATOR-PY](assets/fr/39.webp)


در قسمت "شناسه GitHub مشارکت‌کننده"، نام کاربری GitHub خود را وارد کنید.


![DATA-CREATOR-PY](assets/fr/40.webp)


سپس، باید پروفایل استاد خود را تکمیل کنید. چندین گزینه در دسترس شماست:


- حروف اول نام خود را در قسمت "نام استاد" وارد کنید. سپس نام شما در لیست کشویی "پیشنهادات استاد" در زیر ظاهر می‌شود. با کلیک بر روی آن، آن را انتخاب کنید؛
- به طور جایگزین، می‌توانید مستقیماً روی لیست کشویی "پیشنهادات استاد" کلیک کرده و نام استاد خود را انتخاب کنید.


این اقدام به طور خودکار UUID استاد شما را در فیلد مربوطه پر می‌کند.



![DATA-CREATOR-PY](assets/fr/41.webp)


اگر هنوز پروفایل استادی ندارید، این آموزش را بررسی کنید:


https://planb.network/tutorials/contribution/others/create-teacher-profile-8ba9ba49-8fac-437a-a435-c38eebc8f8a4

سپس روی دکمه "آموزش جدید" کلیک کنید.


![DATA-CREATOR-PY](assets/fr/42.webp)


یک دسته‌بندی اصلی برای آموزش خود انتخاب کنید. سپس، یک زیرمجموعه مرتبط بر اساس دسته‌بندی اصلی انتخاب شده انتخاب کنید.


![DATA-CREATOR-PY](assets/fr/43.webp)


سطح دشواری آموزش را تعیین کنید.


![DATA-CREATOR-PY](assets/fr/44.webp)


یک نام برای دایرکتوری ایجاد شده مخصوص آموزش خود انتخاب کنید. نام این پوشه باید نرم‌افزار پوشش داده شده در آموزش را منعکس کند و از خط تیره برای جدا کردن کلمات استفاده کند. به عنوان مثال، پوشه می‌تواند به نام `red-Wallet` باشد:


![DATA-CREATOR-PY](assets/fr/45.webp)


`project_id` شناسه UUID شرکت یا سازمانی است که ابزار مورد بحث در آموزش را پشتیبانی می‌کند و [در لیست پروژه‌ها](https://github.com/PlanB-Network/Bitcoin-educational-content/tree/dev/resources/projects) موجود است. به عنوان مثال، برای یک آموزش در مورد Sparrow wallet، می‌توانید `project_id` آن را در فایل: `Bitcoin-educational-content/resources/projects/Sparrow/project.yml` پیدا کنید. این اطلاعات به فایل YAML آموزش شما اضافه می‌شود زیرا Plan ₿ Network یک پایگاه داده از شرکت‌ها و سازمان‌های فعال در Bitcoin یا پروژه‌های مرتبط را نگهداری می‌کند. با افزودن `project_id` مرتبط، محتوای خود را به نهاد مربوطه متصل می‌کنید.


***به‌روزرسانی:*** در نسخه جدید اسکریپت، دیگر نیازی به وارد کردن دستی `project_id` ندارید. یک تابع جستجو اضافه شده است تا پروژه را بر اساس نام پیدا کرده و به‌طور خودکار `project_id` مربوطه را بازیابی کند. برای جستجو، ابتدای نام پروژه را در فیلد "نام پروژه" تایپ کنید، سپس شرکت مورد نظر را از منوی کشویی انتخاب کنید. `project_id` به‌طور خودکار در فیلد زیر پر خواهد شد. در صورت نیاز می‌توانید آن را به‌صورت دستی نیز وارد کنید.


![DATA-CREATOR-PY](assets/fr/46.webp)


برای برچسب‌ها، ۲ یا ۳ کلمه کلیدی مرتبط با محتوای آموزشی خود را انتخاب کنید و به‌طور انحصاری از [لیست برچسب‌های Plan ₿ Network](https://github.com/PlanB-Network/Bitcoin-educational-content/blob/dev/docs/50-planb-tags.md) انتخاب کنید. نرم‌افزار همچنین یک تابع جستجوی کلمه کلیدی با یک لیست کشویی ارائه می‌دهد.


![DATA-CREATOR-PY](assets/fr/47.webp)


پس از وارد کردن و تأیید تمام اطلاعات، روی "Create Tutorial" کلیک کنید تا ایجاد فایل‌های آموزشی خود را تأیید کنید. این کار پوشه آموزشی شما و تمام فایل‌های لازم را در دسته‌بندی انتخاب‌شده به صورت محلی generate خواهد کرد.


![DATA-CREATOR-PY](assets/fr/48.webp)


اکنون می‌توانید زیر بخش "بدون اسکریپت پایتون من" و همچنین مرحله 3، "پر کردن فایل YAML" را نادیده بگیرید، زیرا اسکریپت قبلاً این اقدامات را برای شما انجام داده است. مستقیماً به مرحله 4 بروید و شروع به نوشتن آموزش خود کنید.


برای کسب اطلاعات بیشتر در مورد این اسکریپت پایتون، می‌توانید [README](https://github.com/PlanB-Network/Bitcoin-educational-content/blob/dev/scripts/tutorial-related/new-tutorial-creation/README.md) را نیز بررسی کنید.


### بدون اسکریپت پایتون من


مدیر فایل خود را باز کرده و به پوشه `Bitcoin-educational-content` بروید، که نمایانگر کلون محلی مخزن شما است. معمولاً باید آن را تحت مسیر `Documents\GitHub\Bitcoin-educational-content` پیدا کنید.


در این فهرست، شما باید زیرپوشه مناسب برای قرار دادن آموزش خود را پیدا کنید. سازماندهی پوشه‌ها بخش‌های مختلف وب‌سایت Plan ₿ Network را منعکس می‌کند. در مثال ما، از آنجا که می‌خواهیم آموزشی درباره Sparrow wallet اضافه کنیم، باید به مسیر زیر برویم: `Bitcoin-educational-content\tutorials\Wallet`، که به بخش `Wallet` در وب‌سایت مربوط می‌شود:


![TUTO](assets/fr/12.webp)


در پوشه `Wallet`، شما باید یک دایرکتوری جدید ایجاد کنید که به طور خاص به آموزش شما اختصاص داده شده باشد. نام این پوشه باید نرم‌افزار پوشش داده شده در آموزش را تداعی کند و مطمئن شوید که کلمات را با خط تیره به هم متصل کنید. برای مثال من، پوشه با عنوان `Sparrow-Wallet` خواهد بود:


![TUTO](assets/fr/13.webp)


در این زیرپوشه جدید که به آموزش شما اختصاص داده شده است، چندین Elements باید اضافه شوند:


- یک پوشه به نام `assets` ایجاد کنید که قرار است تمام تصاویر لازم برای آموزش شما را دریافت کند؛
- در این پوشه `assets`، شما باید یک زیرپوشه با نام کد زبان اصلی آموزش ایجاد کنید. به عنوان مثال، اگر آموزش به زبان انگلیسی نوشته شده است، این زیرپوشه باید با نام `en` نامگذاری شود. تمام تصاویر آموزشی (نمودارها، تصاویر، اسکرین‌شات‌ها و غیره) را در آنجا قرار دهید.
- یک فایل `tutorial.yml` باید ایجاد شود تا جزئیات مربوط به آموزش شما را ثبت کند؛
- یک فایل با فرمت مارک‌داون باید ایجاد شود تا محتوای واقعی آموزش شما نوشته شود. این فایل باید بر اساس کد زبان نوشتار نام‌گذاری شود. به عنوان مثال، برای یک آموزش نوشته شده به زبان فرانسوی، فایل باید `fr.md` نامیده شود.


![TUTO](assets/fr/14.webp)


به طور خلاصه، در اینجا سلسله مراتب فایل‌هایی که باید ایجاد شوند آمده است:


```
bitcoin-educational-content/
└── tutorials/
└── wallet/ (to be modified with the correct category)
└── sparrow-wallet/ (to be modified with the name of the tutorial)
├── assets/
│   ├── en/ (to be modified according to the appropriate language code)
├── tutorial.yml
└── en.md (to be modified according to the appropriate language code)
```


## 3 - فایل YAML را پر کنید


الگوی زیر را در فایل `tutorial.yml` کپی کنید:


```
id:

project_id:

tags:
-
-
-

category:

level:

professor_id:

# Proofreading metadata

original_language:
proofreading:
- language:
last_contribution_date:
urgency:
contributor_names:
-
reward:
```


در اینجا فیلدهای مورد نیاز آمده است:



- شناسه**: یک UUID (_شناسه منحصربه‌فرد جهانی_) که به‌طور منحصربه‌فرد آموزش را شناسایی می‌کند. می‌توانید آن را با استفاده از [یک ابزار آنلاین](https://www.uuidgenerator.net/version4) به generate تبدیل کنید. تنها شرط این است که این UUID تصادفی باشد تا از تداخل با UUID دیگری در پلتفرم جلوگیری شود؛



- project_id**: شناسه UUID شرکت یا سازمان پشت ابزار ارائه شده در آموزش [از لیست پروژه](https://github.com/PlanB-Network/Bitcoin-educational-content/tree/dev/resources/projects). به عنوان مثال، اگر در حال ایجاد یک آموزش درباره نرم‌افزار Green Wallet هستید، می‌توانید این `project_id` را در فایل زیر پیدا کنید: `Bitcoin-educational-content/resources/projects/blockstream/project.yml`. این اطلاعات به فایل YAML آموزش شما اضافه می‌شود زیرا Plan ₿ Network یک پایگاه داده از تمام شرکت‌ها و سازمان‌هایی که بر روی Bitcoin یا پروژه‌های مرتبط فعالیت می‌کنند را نگهداری می‌کند. با افزودن `project_id` موجودیت مرتبط با آموزش خود، شما یک پیوند بین دو Elements ایجاد می‌کنید؛



- برچسب‌ها**: 2 یا 3 کلمه کلیدی مرتبط با محتوای آموزشی، به‌طور انحصاری انتخاب شده [از لیست برچسب‌های Plan ₿ Network](https://github.com/PlanB-Network/Bitcoin-educational-content/blob/dev/docs/50-planb-tags.md)؛



- دسته‌بندی**: زیرمجموعه‌ای که با محتوای آموزش مطابقت دارد، بر اساس ساختار وب‌سایت Plan ₿ Network (برای مثال، برای کیف پول‌ها: `desktop`, `hardware`, `mobile`, `backup`);



- سطح**: سطح دشواری آموزش، انتخاب شده از:
    - `مبتدی`
    - `متوسط`
    - `پیشرفته`
    - `کارشناس`



- professor_id**: `professor_id` شما (UUID) همانطور که در [پروفایل استاد شما](https://github.com/PlanB-Network/Bitcoin-educational-content/tree/dev/professors) نمایش داده شده است؛



- زبان_اصلی**: زبان اصلی آموزش (مثلاً، `fr`، `en`، و غیره)؛



- بازخوانی**: اطلاعاتی درباره فرآیند بازخوانی. بخش اول را کامل کنید، زیرا بازخوانی آموزش خودتان به عنوان اولین اعتبارسنجی محسوب می‌شود:
    - زبان**: کد زبان برای تصحیح (مثلاً، `fr`، `en`، و غیره).
    - last_contribution_date**: تاریخ روز.
    - فوریت**: 1
    - نام‌های_مشارکت‌کننده**: شناسه GitHub شما.
    - پاداش**: 0

## ۷ - آموزش را ذخیره و ارسال کنید


پس از اتمام نوشتن آموزش خود به زبان دلخواه، مرحله بعدی ارسال یک **Pull Request** است. سپس مدیر مسئول اضافه کردن هرگونه ترجمه‌ی ناقص آموزش شما خواهد بود، به لطف روش ترجمه‌ی خودکار ما با بازبینی انسانی.


برای ادامه با Pull Request، نرم‌افزار GitHub Desktop را باز کنید. نرم‌افزار باید به‌طور خودکار تغییراتی که به‌صورت محلی در شاخه خود نسبت به مخزن اصلی ایجاد کرده‌اید را شناسایی کند. قبل از ادامه، به دقت در سمت چپ Interface بررسی کنید که این تغییرات با آنچه انتظار داشتید مطابقت داشته باشد:


![TUTO](assets/fr/28.webp)


برای تعهد خود یک عنوان اضافه کنید، سپس روی دکمه آبی `Commit to [your branch]` کلیک کنید تا این تغییرات تأیید شوند:


![TUTO](assets/fr/29.webp)


یک کامیت ذخیره‌ای از تغییرات اعمال‌شده بر روی شاخه است که همراه با یک پیام توصیفی ارائه می‌شود و امکان دنبال کردن تکامل یک پروژه در طول زمان را فراهم می‌کند. این به نوعی یک نقطه بازرسی میانی است.


سپس روی دکمه `Push origin` کلیک کنید. این کار کامیت شما را به Fork شما ارسال می‌کند:
