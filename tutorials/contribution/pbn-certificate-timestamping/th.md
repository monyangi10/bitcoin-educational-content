ในบทแนะนำนี้ เราจะค้นพบว่า Plan ₿ Academy ออกหลักฐานที่ตรวจสอบได้สำหรับใบรับรองการทดสอบ ₿-CERT ของคุณหรือประกาศนียบัตรใด ๆ เกี่ยวกับการสำเร็จหลักสูตรได้อย่างไร จากนั้นในส่วนที่สองเราจะอธิบายวิธีการตรวจสอบความถูกต้องของหลักฐานเหล่านี้


## กลไกการพิสูจน์ Plan ₿ Academy


ที่ Plan ₿ Academy เราเซ็นรับรองใบรับรองและประกาศนียบัตรด้วยการเข้ารหัส และประทับเวลาโดยใช้ Timechain (เช่น บล็อกเชน Bitcoin) ผ่านกลไกการพิสูจน์ที่อาศัยการดำเนินการเข้ารหัสสองแบบ:


1. ลายเซ็น GPG บนไฟล์ข้อความที่สังเคราะห์ความสำเร็จของคุณ

2. การประทับเวลาของไฟล์ที่ลงนามเดียวกันผ่าน [opentimestamps](https://opentimestamps.org/)


การดำเนินการแรกช่วยให้คุณยืนยันผู้ที่ออกใบรับรอง (หรือประกาศนียบัตร) ในขณะที่การดำเนินการที่สองช่วยให้คุณตรวจสอบวันที่ออกใบรับรองได้

เราเชื่อว่า กลไกการพิสูจน์ที่เรียบง่ายนี้ช่วยให้เราสามารถออกใบรับรองและประกาศนียบัตรพร้อมหลักฐานที่ปฏิเสธไม่ได้ซึ่งใคร ๆ ก็สามารถตรวจสอบได้อย่างอิสระ


![image](./assets/proof-mechanism.webp)


ด้วยกลไกการพิสูจน์นี้ ความพยายามใด ๆ ที่จะเปลี่ยนแปลงรายละเอียดเล็ก ๆ น้อย ๆ ของใบรับรองหรือประกาศนียบัตรของคุณจะส่งผลให้เกิดค่าแฮช SHA-256 ที่แตกต่างกันโดยสิ้นเชิงของไฟล์ที่ลงนาม ซึ่งจะเผยให้เห็นการปลอมแปลงในทันที เนื่องจากทั้งลายเซ็นและการประทับเวลาจะไม่ถูกต้องอีกต่อไป นอกจากนี้ หากใครพยายามปลอมแปลงใบรับรองหรือประกาศนียบัตรในนามของ Plan ₿ Academy การตรวจสอบลายเซ็นอย่างง่ายจะเปิดเผยการฉ้อโกง


### GPG-signature ทำงานอย่างไร?


ลายเซ็น GPG ถูกสร้างขึ้นโดยใช้ซอฟต์แวร์โอเพ่นซอร์สที่เรียกว่า GNU Privacy Guard ซอฟต์แวร์นี้ช่วยให้ผู้ใช้สามารถสร้างกุญแจส่วนตัวได้อย่างง่ายดาย ลงนามและตรวจสอบลายเซ็น และเข้ารหัสและถอดรหัสไฟล์ สำหรับวัตถุประสงค์ของบทแนะนำนี้ สิ่งสำคัญคือต้องทราบว่า Plan ₿ Academy ใช้ GPG ในการสร้างกุญแจส่วนตัว/สาธารณะและลงนามในใบรับรอง ₿-CERT และประกาศนียบัตรการจบหลักสูตรทั้งหมด


ในทางกลับกัน หากมีคนต้องการตรวจสอบความถูกต้องของไฟล์ที่ลงนามแล้ว พวกเขาสามารถใช้ GPG เพื่อนำเข้าคีย์สาธารณะของผู้ออกและตรวจสอบได้


สำหรับผู้ที่อยากรู้อยากเห็นและต้องการเรียนรู้เพิ่มเติมเกี่ยวกับซอฟต์แวร์ที่ยอดเยี่ยมนี้ คุณสามารถอ้างอิงได้ที่ ["The GNU Privacy Handbook"](https://www.gnupg.org/gph/en/manual/x135.html)


### การทำงานของการประทับเวลาเป็นอย่างไร?


ใครๆ ก็สามารถใช้ OpenTimestamps เพื่อประทับเวลาไฟล์และรับหลักฐานที่ตรวจสอบได้เกี่ยวกับการมีอยู่ของไฟล์นั้น กล่าวอีกนัยหนึ่ง มันไม่ได้ให้หลักฐานว่าไฟล์ถูกสร้างขึ้นเมื่อใด แต่ให้หลักฐานว่าไฟล์นั้นมีอยู่ไม่เกินช่วงเวลาหนึ่งในเวลา

OpenTimestamps ให้บริการนี้ฟรีโดยใช้วิธีที่มีประสิทธิภาพสูงในการจัดเก็บหลักฐานในบล็อกเชน Bitcoin โดยใช้ SHA-256 hash algorithm เพื่อสร้างตัวระบุที่ไม่ซ้ำกันสำหรับไฟล์ของคุณ และสร้าง Merkle tree โดยใช้แฮชของไฟล์ที่ส่งโดยผู้ใช้คนอื่น ๆ เฉพาะแฮชของโครงสร้าง Merkle tree เท่านั้นที่ถูกยึดในธุรกรรม OP_RETURN เพื่อให้มั่นใจในวิธีการที่ปลอดภัยและกระชับในการยืนยันการมีอยู่ของไฟล์

เมื่อธุรกรรมนี้เข้าสู่บล็อกแล้ว ใครก็ตามที่มีไฟล์เริ่มต้นและไฟล์ `.ots` ที่เกี่ยวข้องสามารถตรวจสอบความถูกต้องของการประทับเวลาได้ ในส่วนที่สองของบทแนะนำ เราจะดูวิธีการตรวจสอบใบรับรอง Bitcoin ของคุณหรือประกาศนียบัตรการจบหลักสูตรใด ๆ ผ่านทางเทอร์มินัลและผ่านทางอินเทอร์เฟซกราฟิกบนเว็บไซต์ของ OpenTimestamps


## วิธีการตรวจสอบใบรับรองหรือประกาศนียบัตร Plan ₿ Academy ₿-CERT


### ขั้นตอนที่ 1. ดาวน์โหลดใบรับรองหรือประกาศนียบัตรของคุณ


เข้าสู่ระบบแดชบอร์ดส่วนตัว/นักเรียนของคุณที่ planb.network.


![image](./assets/login.webp)


ไปที่ "Credentials" โดยคลิกที่เมนูด้านซ้ายมือ และเลือกเซสชันการสอบหรือประกาศนียบัตรของคุณ


![image](./assets/credential.webp)


ดาวน์โหลดไฟล์ zip


![image](./assets/download.webp)


คลายเนื้อหาโดยคลิกขวาที่ไฟล์ `.zip` และเลือก "Extract" คุณจะพบไฟล์สามไฟล์ที่แตกต่างกัน:



- ไฟล์ข้อความที่ลงนามแล้ว (เช่น certificate.txt)
- ไฟล์ Open timestamp (OTS) (เช่น certificate.txt.ots)
- ใบรับรอง PDF (เช่น certificate.pdf)


### ขั้นตอนที่ 2: คุณจะตรวจสอบลายเซ็นของไฟล์ข้อความได้อย่างไร?


ก่อนอื่น ให้ไปที่โฟลเดอร์ที่คุณแยกไฟล์ออกมาแล้วเปิดเทอร์มินัล (คลิกขวาที่หน้าต่างโฟลเดอร์แล้วคลิกที่ "Open in Terminal") จากนั้นทำตามคำแนะนำด้านล่างนี้


1. นำเข้า PGP key สาธารณะ Plan ₿ Academy ด้วยคำสั่งต่อไปนี้:


```bash
curl -s https://raw.githubusercontent.com/Asi0Flammeus/pgp-public-keys/master/planb-network-pk.asc | gpg --import
```


คุณควรเห็นข้อความดังต่อไปนี้หากคุณนำเข้า PGP Key สำเร็จ


```
gpg: key 8F12D0C63B1A606E: public key "Plan ₿ Academy (used for Plan ₿ Academy platform) <admin@planb.network>" imported
gpg: Total number processed: 1
gpg:               imported: 1
```


หมายเหตุ: หากคุณเห็นว่ามีการประมวลผล 1 คีย์และนำเข้า 0 คีย์ นั่นอาจหมายความว่าคุณได้นำเข้าคีย์เดียวกันมาก่อนแล้ว ซึ่งถือว่าเป็นเรื่องปกติ


2. ตรวจสอบลายเซ็นของใบรับรองหรือประกาศนียบัตรโดยใช้คำสั่งต่อไปนี้:


```bash
gpg --verify certificate.txt
```


คำสั่งนี้จะแสดงรายละเอียดเกี่ยวกับลายเซ็นต์ให้คุณ รวมถึง:



- ใครเป็นผู้ลงนาม (Plan ₿ Academy)
- เมื่อมีการลงนาม
- ไม่ว่าลายเซ็นจะถูกต้องหรือไม่


นี่คือตัวอย่างของผลลัพธ์:


```
gpg: Signature made lun 11 nov 2024, 00:39:04 CET
gpg:                using RSA key 5720CD577E7894C98DBD580E8F12D0C63B1A606E
gpg:                issuer "admin@planb.network"
gpg: Good signature from "Plan ₿ Academy (used for Plan ₿ Academy platform) <admin@planb.network>" [unknown]
```


หากคุณเห็นข้อความเช่น "BAD signature" นั่นหมายความว่าไฟล์ถูกดัดแปลง


### ขั้นตอนที่ 3: การตรวจสอบ Open Timestamp


#### การตรวจสอบผ่านกราฟิก Interface


1. เยี่ยมชมเว็บไซต์ OpenTimestamps: https://opentimestamps.org/

2. คลิกที่แท็บ "Stamp & Verify"

3. ลากและวางไฟล์ OTS (เช่น `certificate.txt.ots`) ลงในพื้นที่ที่กำหนด.

4. ลากและวางไฟล์ที่มีการประทับเวลา (เช่น `certificate.txt`) ลงในพื้นที่ที่กำหนด.

5. เว็บไซต์จะตรวจสอบเวลาเปิดโดยอัตโนมัติและแสดงผลลัพธ์


หากคุณเห็นข้อความดังต่อไปนี้ แสดงว่าเวลาประทับถูกต้อง:


![cover](assets/opentimestamp_wegui_verified.webp)


#### วิธีการ CLI


หมายเหตุ: ขั้นตอนนี้ **จะต้องมีการรันโหนด Bitcoin ในเครื่อง**


1. ติดตั้ง OpenTimestamps client จาก [repository](https://github.com/opentimestamps/opentimestamps-client) อย่างเป็นทางการโดยรันคำสั่งต่อไปนี้:


```
pip install opentimestamps-client
```


2. ไปที่ไดเรกทอรีที่มีไฟล์ใบรับรองที่แยกออกมา


3. รันคำสั่งต่อไปนี้เพื่อตรวจสอบเวลาประทับเปิด:


```
ots verify certificate.txt.ots
```


คำสั่งนี้จะ:



- ตรวจสอบเวลาประทับเทียบกับบล็อกเชนของ Bitcoin
- แสดงให้คุณเห็นอย่างชัดเจนว่าไฟล์ถูกประทับเวลาเมื่อใด
- ยืนยันความถูกต้องของการประทับเวลา


#### ผลลัพธ์สุดท้าย


การยืนยันจะสำเร็จหาก **ทั้งสอง** ข้อความต่อไปนี้แสดงขึ้น:


1. ลายเซ็น GPG รายงานว่าเป็น **"Good signature from Plan ₿ Academy"**

2. การยืนยัน OpenTimestamps แสดงเวลาประทับบล็อก Bitcoin เฉพาะและรายงาน **"สำเร็จ! บล็อก Bitcoin [blockheight] ยืนยันว่าข้อมูลมีอยู่ตั้งแต่ [timestamp]"**


ตอนนี้ที่คุณรู้แล้วว่า Plan ₿ Academy ออกหลักฐานที่ตรวจสอบได้สำหรับใบรับรองและประกาศนียบัตร ₿-CERT อย่างไร คุณสามารถตรวจสอบความถูกต้องของพวกมันได้อย่างง่ายดาย
