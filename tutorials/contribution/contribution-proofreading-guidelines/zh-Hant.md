## 什麼是校對？


校對是書面文章的最後審查程序，目的是找出並更正文法、拼寫、標點和格式上的錯誤。在出版或投稿前，可確保文字清晰、連貫且沒有錯誤。


當您進行這種類型的任務時，重要的是要遵循原始語言（EN 或 FR）的意義，但要確保最終語言的文字對於母語使用者而言盡可能流暢。


## 在 Plan ₿ Network 上進行校對之前的第一步


在開始新的校對工作之前，請在 [Telegram 群組](https://t.me/PlanBNetwork_ContentBuilder) 中公佈，或通知您的 Plan ₿ Network 協調員，他們會開啟專屬的 [issue](https://github.com/orgs/PlanB-Network/projects/3) 。當您收到問題連結時，只要註明您正開始該內容的校對工作即可。

此系統可讓協調員追蹤 repo 內的進度，也可讓校對員「認領」內容，避免他人重複工作。

在問題本身上，您會發現直接帶您到要檢查的內容的連結。您可以直接點擊它們，或者更好的方法是回到您自己的 forked repo，直接從那裡開始工作。


首先，請務必記得將您的 repo 同步到「dev」分支。如此一來，在您開始任何類型的工作之前，內容總是會被更新，而且您也不會在新舊資料之間產生任何衝突。請務必點選「Sync Fork」和「Update branch」。



![REVIEW](assets/en/1.webp)



同步成功後，您可以直接存取感興趣的內容，並在新的分支上提交，如本 [教學](https://planb.network/tutorials/contribution/content/contribution-proofreading-review-tutorial-1ee068ca-ddaf-4bec-b44e-b41a9abfdef6) 所示。否則，您可以點選「分支」開啟一個新的分支，如下圖所示。



![REVIEW](assets/en/2.webp)



在這個新頁面中，您可以在標題「您的分支」下找到您已經開啟的所有分支。這個部分非常有用，因為它可以讓您輕鬆找到您修改某些內容的地方。如果您想開啟新的分支，可以按一下頁面右上角的「新分支」。


![REVIEW](assets/en/3.webp)



然後，您會看到一個彈出視窗，您需要插入新分支的名稱。在下面的例子中，我選擇稱它為 "BTC101-FR"。這樣，我就會永遠記得，這個特定的分支需要用於 BTC101 法語課程的校對，**我不會將它用於任何其他任務**。


我建議您也這麼做：確保在需要開始新任務時，隨時開啟新的分支。



![REVIEW](assets/en/4.webp)



建立這個新的分支後，請務必從上一頁的 "Your Branches「 點選它，然後開始處理與特定內容相關的 *.md* 檔案 (在我的例子中，我會點選 」courses" 資料夾，然後選取 BTC101 子資料夾，搜尋 fr.md)。所有與特定檔案相關的提交都必須在同一分支中提交（儲存）。


## 原文還是譯文？


在進行一些內容校對（proofreading）時，一定要檢查其英文（或法文）原文。請注意，我們使用 AI 語言工具進行翻譯，因此對最終讀者而言，目標語言的呈現可能不夠流暢或難以理解。


因此，如有需要，請隨意調整文字和修改句子。我們的目標是提高流暢性，但始終遵循原意。如果對如何處理特定詞語有疑問，請務必詢問翻譯協調員。


LLM 工具可能會從字面上翻譯一些與 Bitcoin 相關的字詞，例如 Lightning Network，在義大利語中會變成「Rete Fulmine」。


尤其是涉及到非常技術性的字詞時。在這種情況下，除非您的語言規則強制您翻譯每個字詞，否則建議您在目標語言上保留原始英文字詞，以提高清晰度。在第二種情況下，請務必做一些研究，看看在您的 Bitcoin 社群中是否有人已經翻譯過這個字詞，而且這個字詞現在已被廣泛使用。



- 一個解決方案可以是在您的目標語言[BitcoinWiki](https://en.Bitcoin.it/wiki/Main_Page)上查看該詞是否被翻譯。如果沒有，你就保留這個英語單詞。



- 在任何情況下，我的建議是先插入 EN 單詞，然後在圓括號內插入目標語言的對應含義，遵循方案 EN (LANG)，反之亦然。例如Address (indirizzo) 或 indirizzo (Address)。



- 另一個好的解決方案是保留 EN 原有的字/詞，然後建立一個超連結，重定向到 planb.network 上的 [詞彙表](https://planb.network/en/resources/glossary)。要做到這一點，您需要在方括號內插入單詞/短語，並在圓括弧內插入鏈接，就像您在下面的示例中看到的那樣：


```
[UTXO](https://planb.network/resources/glossary/utxo)
```


在最後的結果（如下圖）中，您不會看到整個連結，而單字會變成可點選。


![REVIEW](assets/en/5.webp)



請注意，您從網站取得的詞彙連結，在「network」一詞後面含有語言代碼 (例如： ``https://planb.network/en/resources/glossary/UTXO`-> 在這裡您可以讀到語言代碼 "en")。在這種情況下，請移除連結中的語言代碼，就像您在上面方塊中看到的一樣。這樣，系統就會自動將讀者帶到他們指定的語言。


repo 上的內容充滿了像上面這些超連結。現在您知道它們的意思了，請確保不要刪除任何由作者插入的連結。


與字彙詮釋相關的另一件事如下。如果您在文本中找到 "Plan ₿ Network"，請保留其原始形式。請勿翻譯「計畫」或「網路」。此外，在介紹 Plan ₿ Network 時，請勿使用「The」一詞，並**將其視為品牌**。


同樣地，「₿-CERT」、「BIZ SCHOOL」、「TECH SCHOOL」也應該保留原來的形式。


本段的最後一點說明：如上文所述，我們使用 AI 工具來翻譯內容，然後請「人工」介入，以確保一切流暢且校對良好。


當您使用 AI 校對大部分的文字時，我們一定會注意到，因為我們熟悉 AI 所產生的典型句子結構。如果我們發現您僅僅依賴 AI 進行校對，而沒有做出重大修改，Sats 中的最終獎勵可能會減少一半！


## 標頭的結構


在 markdown 語言中，標題（和段落標題）都以 Hash 符號 ``#`` 開頭。Hash 符號的數量對應於標題的層級。例如，第三級的標題在文字前有三個數字符號 (例如：`### 我的標題`)。


在課程中，最重要的部分會以一個 Hash 標誌來介紹，而子部分則可以有兩到四個 Hash 標誌。在教學中，我們通常只使用兩個 Hash 標誌的標頭。



![REVIEW](assets/en/6.webp)


請務必**永遠**刪除標題前的 Hash 符號，否則會造成文字結構上的問題。


同時，**不要**變更您在上圖中看到的 chapterID 部分，``<chapterId>d668fdf6-fb4c-4bbf-82e1-afcb95c122e0</chapterId>``或影片引用，如``::video id=ba99951f-81d2-418f-b5e7-4b8c9f8b8cc8:::``。


當我們在標題前插入 ``#``` 時，標題在課程預覽中會自動變成粗體，因此請避免在修正時將標題變成粗體。


此外，在 EN 版本的課程中，有一個或兩個 ``#``` 的標題會以大寫字母開頭，而以三個或四個字母開頭的標題通常不遵循此規則。如果可能的話，請確保您的目標語言的標題也遵循此結構。


## 課程的初始部分


在任何內容的開頭，您會發現下列靜態小寫字詞：「名稱」、「說明」、「目標」。這些詞彙是網站用來解碼內容本身的，而且**始終保留為 EN**。因此，請勿翻譯它們，否則內容會產生同步問題。請確保只校對冒號之後的部分，該部分會由 AI 自動翻譯。



![REVIEW](assets/en/7.webp)



在這個相同的開頭部分，保持原來的格式。不要在文字開頭添加任何東西。例如：避免在破折號前加上「tt」，就像下面的圖片一樣！

## 格式建議


以下是一些校對目標語言內容時需要注意的格式問題範例。



- 注意一些奇怪的標點符號，如`\*\*`或``**``，它們可能代表粗體符號的不良渲染。在下面的圖片中，您可以看到星號只出現在單字的右邊，看起來很奇怪。


![REVIEW](assets/en/9.webp)



請務必檢查英文原文，看看是否應該有粗體字。在這種情況下，只要在字首加上兩個星號，就可以讓它正確地顯示在網站上。事實上，在 markdown 語言中，若要顯示粗體，您必須在單字/句子的前後插入兩個星號 ``**`` (請參閱以下範例)。



![REVIEW](assets/en/10.webp)




- 同樣的問題也可能發生在 $ 和 ``` `` 等符號上。

請務必檢查原始語言檔案（通常是 EN 或 FR），以瞭解這些符號應該放在哪裡。您可以隨時向協調員請求協助。



- 如果您發現引號，請務必在網上做一些研究，以找到您語言的正確翻譯。引號通常插在符號 ``>``` 之後。



![REVIEW](assets/en/11.webp)


## 測驗校對


您知道您也可以校對每門課程的測驗題嗎？舉例來說，如果您想要校對 IT 科目 BTC101 的測驗題，您可以開啟一個專屬的分支，並遵循下列路徑："courses" -> "BTC101" -> "quiz"。在那裡，您會找到所有專門針對每個問題的資料夾，以及 _yml_ 格式的相關語言檔案。


再次確認您是在專門為此目的而開設的專用分行，並務必通知協調員。


檢閱問題後，請確認您已將「已檢閱」狀態從「false」變更為「true」，如下圖所示。

## 其他最佳實踐


如果您需要在文字中搜尋特定的字詞，您可以按一下 ``CTRL+F`` 並顯示尋找取代部分。當您需要跳到文字的特定部分，或需要批次取代特定的字詞/句子，而不需要捲動完整內容時，這部分非常有用。



![REVIEW](assets/en/13.webp)



使用「取代所有」功能時，請務必仔細檢查結果，以確保連結也沒有被修改。例如，如果您要將「Bitcoin」改成「Bitkoin」（在某些語言中可能是必要的），使用「取代所有」功能可以有效地更新文字中的所有實例。不過，請注意此工具也會修改任何包含該單字的連結，可能會導致重定向問題。


在下面的例子中，校對人員使用上述功能將 "Satoshi 「更換為 」Satoshi(Sats)"，同時也將連結變更為包含該詞本身的教程。結果，連結變得無效。


![REVIEW](assets/en/14.webp)


關於連結的主題，如果作者插入指向 Plan ₿ Network 教學課程的連結，網站會自動建立一個顯示相關縮圖的 「卡片」。因此，請務必在文字與連結本身之間留出空格，否則您可能會在網站上看到以下錯誤。


![REVIEW](assets/en/15.webp)




最後，當您完成校對工作並傳送 PR 時，另一個最佳做法是回到協調員開啟的原始問題，並註解「校對完成」。確保也插入您的 PR 連結。
