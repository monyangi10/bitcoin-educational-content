## 校正とは何か？


校正とは、文法、スペル、句読点、書式などの誤りを特定し、修正することである。出版や投稿の前に、文章が明確で一貫性があり、間違いがないことを確認する。


この種の作業を行う場合、原語（ENまたはFR）の意味に従うことが重要ですが、最終的な言語のテキストがネイティブスピーカーにとって可能な限り流動的であることを確認してください。


## Plan ₿ Networkで校正する前の最初のステップ


新しい校正タスクを開始する前に、[Telegram グループ](https://t.me/PlanBNetwork_ContentBuilder) でお知らせいただくか、Plan ₿ Network コーディネータにお知らせください。コーディネータが専用の [issue](https://github.com/orgs/PlanB-Network/projects/3) を開設します。issueのリンクを受け取ったら、その内容の校正タスクを開始するとコメントしてください。

このシステムにより、コーディネーターはレポ内の進捗状況を把握することができ、校正者によってコンテンツが「クレーム」され、他の誰かによる重複作業を防ぐことができる。

issue自体には、チェックすべきコンテンツに直接移動できるリンクがあります。直接クリックすることもできますし、自分のフォークしたレポに戻ってそこから直接作業することもできます。


まず第一に、"dev "ブランチのレポをSYNCすることを常に忘れないでください。こうすることで、何らかの作業を始める前に常に内容が更新され、古いものと新しいものの間でコンフリクトが発生しなくなります。必ず "Sync Fork "と "Update branch "をクリックしてください。



![REVIEW](assets/en/1.webp)



同期に成功したら、この[チュートリアル](https://planb.network/tutorials/contribution/content/contribution-proofreading-review-tutorial-1ee068ca-ddaf-4bec-b44e-b41a9abfdef6)にあるように、目的のコンテンツに直接アクセスして新しいブランチにコミットできます。そうでなければ、下図のように "Branches" をクリックして新しいブランチを開き、そこで作業することができます。



![REVIEW](assets/en/2.webp)



この新しいページでは、「Your Branches（あなたのブランチ）」というタイトルの下に、すでに開いたブランチがすべて表示されます。このセクションは、あなたがコンテンツを修正した場所を簡単に見つけることができるので、とても便利です。新しいブランチを開きたい場合は、ページの右上にある "New Branch "をクリックしてください。


![REVIEW](assets/en/3.webp)



すると、新しいブランチの名前を入力するポップアップが表示されます。下の例では、「BTC101-FR」としました。こうすることで、この特定のブランチをBTC101フランス語コースの校正に使用する必要があることを常に覚えておくことができます。


新しい仕事を始める必要があるときは、いつでも新しいブランチを開くようにしてください。



![REVIEW](assets/en/4.webp)



新しいブランチを作成したら、前のページの「Your Branches」からそのブランチをクリックし、特定のコンテンツに関連する *.md* ファイルの作業を開始します（私の場合、フォルダ「courses」とサブフォルダ「BTC101」をクリックして、fr.md を探します）。特定のファイルに関連するすべてのコミットは、同じブランチ内でコミット（保存）する必要があります。


## 原語か翻訳か？


コンテンツの校正を行う際には、必ず元の英語（またはフランス語）版を確認することが重要です。私たちはAI言語ツールを使って翻訳しているため、ターゲット言語でのレンダリングが流動的であったり、最終的な読者にとって理解しやすいものでなかったりする可能性があることに注意してください。


したがって、必要であれば、自由に文章を調整し、文章を修正してください。私たちの目的は流動性を高めることですが、常に原文の意味に従います。特定の単語の扱いについて疑問がある場合は、必ず翻訳コーディネーターにお尋ねください。


LLMツールは、Bitcoinに関連するいくつかの単語を、Lightning Networkのように文字通りに翻訳するかもしれない。


非常に専門的な単語を扱う場合は特にそうです。このような場合、原語の英単語をそのまま訳した方がわかりやすいでしょう。このような場合は、Bitcoinのコミュニティで誰かがすでにその単語を翻訳し、広く使われていないかどうかを常に調査してください。



- 一つの解決策は、ターゲット言語の[BitcoinWiki](https://en.Bitcoin.it/wiki/Main_Page)で、その単語が翻訳されているかどうかを確認することです。もし翻訳されていなければ、その単語は英語のままにしておきます。



- いずれにせよ、私のアドバイスとしては、EN(LANG)のスキームに従って、EN単語を挿入し、丸括弧の中にターゲット言語の対応する意味を挿入すること、またはその逆を行うことである。例Address（indirizzo）またはindirizzo（Address）。



- もう一つの良い解決策は、ENオリジナルの単語/フレーズをそのままにして、planb.networkの[glossary](https://planb.network/en/resources/glossary)にリダイレクトするハイパーリンクを作成することです。これを行うには、下の例のように、単語/フレーズを角括弧の中に、リンクを丸括弧の中に挿入する必要があります：


```
[UTXO](https://planb.network/resources/glossary/utxo)
```


最終結果（下の画像）では、リンク全体が視覚化されず、単語がクリック可能になる。


![REVIEW](assets/en/5.webp)



ウェブサイトから取得する用語集リンクには、「network」という単語の後に言語コードが含まれていることに注意してください（例：``https://planb.network/en/resources/glossary/UTXO``->ここでは言語コード「en」を読むことができます）。この場合、上のボックスのように、リンクから言語コードを削除してください。こうすることで、システムは自動的に読者を指定された言語に誘導します。


レポ上のコンテンツは、上記のようなハイパーリンクでいっぱいだ。リンクの意味がわかったところで、作者によって張られたリンクを削除しないように注意してください。


もう一つ、語釈に関連することとして、次のようなものがある。本文中に「Plan ₿ Network」とあったら、この原文のままにしておくこと。プラン」や「ネットワーク」は訳さないこと。また、Plan ₿ Networkを紹介するときに冠詞の "The "を使わない。


また、「⃻-CERT」、「BIZ SCHOOL」、「TECH SCHOOL」についても同様で、これらは原型をとどめておく必要がある。


この段落の最後の注釈：上で述べたように、私たちはAIツールを使ってコンテンツを翻訳し、その後、すべてが流動的で校正が行き届いていることを確認するために「人間」の介入を求めます。


文章の大部分をAIで校正した場合、AIが生成する典型的な文章構造を熟知している私たちは必ず気づきます。もし、大幅な変更を加えることなく、AIだけに頼って校正したことが判明した場合、Satsの最終報酬は半減する可能性があります！


## ヘッダーの構造


マークダウン言語では、ヘッダー（および段落タイトル）はすべてHash記号 ``#`` で始まります。Hash記号の数は見出しレベルに対応しています。例えば、レベル3の見出しはテキストの前に3つの数字記号があります（例：`### My Header`）。


コースでは、最も重要な部分は1つのHash記号で紹介され、サブパートは2つから4つのHash記号を持つことができます。チュートリアルでは、通常2つのHash記号を持つヘッダーのみを使用します。



![REVIEW](assets/en/6.webp)


タイトルの前のHash記号は絶対に削除しないでください。


同時に、上の画像にあるチャプターIDの部分、``<chapterId>d668fdf6-fb4c-4bbf-82e1-afcb95c122e0</chapterId>``や、``::video id=ba99951f-81d2-418f-b5e7-4b8c9f8b8cc8:::`` のようなビデオ参照は変更しないでください。


タイトルの前に ``#`` を挿入した場合、コースプレビューでは自動的に太字になりますので、修正中にタイトルを太字にすることは避けてください。


また、EN版のコースでは、1つまたは2つの``#``で始まるタイトルは、すべての単語が大文字で始まりますが、3つまたは4つで始まるタイトルは、通常このルールに従いません。可能であれば、ターゲット言語のタイトルがこの構造に従っていることを確認してください。


## コースの初期セクション


どのコンテンツの冒頭にも、次のような静的な小文字の単語がある：「名前」、「説明」、「目的」。これらはウェブサイトがコンテンツをデコードするために使用され、**常にEN**のままです。そのため、これらを翻訳しないでください。そうしないと、コンテンツに同期の問題が生じます。AIが自動的に翻訳するコロン以降の部分のみを校正してください。



![REVIEW](assets/en/7.webp)



この同じ最初のセクションでは、書式をそのままにする。テキストの先頭に何も追加しないでください。例：下の画像のように、ダッシュの前に "tt "を付けない！



![REVIEW](assets/en/8.webp)

- や ``**`` のような奇妙な句読点に注意してください。これは、太字記号のレンダリングが間違っている可能性があります。下の画像では、アスタリスクが単語の右側にしかなく、奇妙に見える。


![REVIEW](assets/en/9.webp)

![REVIEW](assets/en/10.webp)




- や `` `` のような記号でも同じ問題が起こるかもしれない。

これらの記号がどこにあるべきかを確認するために、元の言語ファイル（多くの場合、ENまたはFR）を確認してください。この件に関しては、いつでもコーディネーターに助けを求めることができます。



- 引用符を見つけたら、必ずネットで調べて、あなたの言語での正しい翻訳を見つけましょう。引用符は通常、記号 ``>`` の後に挿入されます。

## クイズ校正


各コースの小テスト問題も校正できることをご存知ですか？例えば、ITのBTC101の小テストを校正したい場合、専用のブランチを開き、以下の手順に従ってください："コース" -> "BTC101" -> "小テスト".そこには各問題専用のフォルダと_yml_フォーマットの関連言語ファイルがあります。


もう一度言うが、この目的のために開設した専用支店であることを確認し、必ずコーディネーターに報告すること。


質問を確認した後、下の画像のように「reviewed」のステータスを「false」から「true」に変更してください。

## その他のベストプラクティス


テキスト内の特定の単語を検索したい場合は、``CTRL+F``をクリックすると、検索置換セクションが表示されます。この部分は、テキストの特定の部分にジャンプする必要がある場合や、コンテンツ全体をスクロールすることなく、特定の単語や文章を一括で置換する必要がある場合に非常に便利です。



![REVIEW](assets/en/13.webp)



全置換」機能を使用する場合、リンクも変更されていないか、結果をダブルチェックすることが重要です。例えば、"Bitcoin "という単語を "Bitkoin "に変更したい場合（言語によっては必要な場合もあります）、"全置換 "機能を使用すると、テキスト内のすべてのインスタンスを効率的に更新することができます。ただし、このツールを使用すると、その単語を含むリンクも変更されるため、リダイレクトの問題が発生する可能性があることに注意してください。


下の例では、校正者が上記の関数を使って「Satoshi」を「Satoshi(Sats)」に置き換え、さらにその単語そのものを含むチュートリアルへのリンクを変更した。その結果、リンクは無効となった。


![REVIEW](assets/en/14.webp)


チュートリアルのPlan ₿ Networkコースを参照するリンクを挿入すると、ウェブサイトは自動的に関連するサムネイルを表示する「カード」を作成します。そのため、テキストとリンクの間には必ずスペースを入れてください。


![REVIEW](assets/en/15.webp)




最後に、校正作業を終えてPRを送信する際のもう1つのベストプラクティスは、コーディネーターが開いた元のissueに戻り、「校正完了」とコメントすることです。その際、あなたのPRリンクも必ず挿入してください。
