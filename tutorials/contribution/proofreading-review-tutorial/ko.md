## Plan ₿ Network란 무엇인가요?


Plan ₿ Network는 누구나 다양한 강좌와 튜토리얼을 통해 Bitcoin에 대해 배울 수 있는 플랫폼입니다. 현재 모든 콘텐츠는 오픈 소스이며, GitHub 리포지토리에서 호스팅되며, 검토 및 콘텐츠 제작을 위한 기여에 열려 있습니다.


Plan ₿ Network는 전 세계 Bitcoin 커뮤니티를 연결하고 격려하는 Bitcoin 사이버 공간의 광장이 되는 것을 목표로 합니다.


## 기여하는 이유는 무엇인가요?


Plan ₿ Network은 기본적으로 다국어를 지원하며, 이는 전 세계 모든 사람이 Bitcoin 리소스에 액세스할 수 있도록 하는 것이 목표이기 때문입니다. 저희는 언어가 주권 화폐의 장벽이 되어서는 안 된다고 믿습니다.


이를 위해 인간과 인공지능의 접근 방식을 사용합니다. 프로그램](https://github.com/Asi0Flammeus/LLM-Translator)을 통해 콘텐츠를 여러 언어로 자동 번역할 수 있습니다. 하지만 콘텐츠가 이해하기 쉽고 부적절한 용어가 없는지 확인하기 위해 유창한 화자가 검토해야 합니다.


커뮤니티에서 Bitcoin 교육을 홍보하기 위해 이 교육 프로젝트를 구축하는 데 도움을 주고 싶다면 강좌 또는 튜토리얼을 검토하는 것부터 시작할 수 있습니다.


또한 *가치 4 가치* 모델을 소중히 여기기 때문에 리뷰 기여에 대한 보상을 제공합니다. 보상은 콘텐츠의 길이, 자동 번역의 언어적 정확도, 콘텐츠의 난이도에 따라 결정됩니다.


## GitHub의 작동 방식을 알아야 하나요?


콘텐츠를 검토하기 위해 GitHub 전문가가 될 필요는 없습니다. 자신이 하는 일과 그 이유를 이해하는 것이 가장 좋지만, 아래 지침에 따라 간단히 기여할 수 있습니다.


하지만 Git과 GitHub에 익숙하지 않아 더 자세히 알고 싶다면 이러한 도구에 대한 소개글을 참조하세요:

## 3. GitHub의 Interface을 통해 콘텐츠를 검토하고 편집하세요



- 해당 이슈에는 원본 콘텐츠와 검토할 파일에 대한 링크가 포함된 설명(참조: \**A\**)이 포함되어 있습니다.
- 검토를 담당하고 싶으면 해당 이슈에 댓글(예: \**B\**)을 남겨서 검토할 것임을 표시하세요. 이렇게 하면 중복 작업을 방지할 수 있습니다.
- 검토할 파일 옆의 링크(참조: \**C\**)를 클릭하여 콘텐츠에 액세스합니다.


![REVIEW](assets/fr/03.webp)


*이제 검토할 파일의 GitHub Interface을 볼 수 있습니다*


- 왼쪽 패널에서는 브랜치(참조: \**A\**) 및 파일(참조: \**B\**)을 탐색할 수 있습니다.

*연필 아이콘을 클릭하여 검토를 시작합니다(**C** 참조)


![REVIEW](assets/fr/04.webp)



- 처음 기여하는 경우 GitHub에서 소스 리포지토리를 Fork로 설정하라는 메시지가 표시됩니다. 버튼을 클릭하기만 하면 됩니다.


![REVIEW](assets/fr/05.webp)



- 그러면 파일 경로와 해당 브랜치(\**B\** 참조)를 확인하고 편집 및 미리보기 모드(\**C\** 참조)로 전환할 수 있는 편집기 Interface(\**A\** 참조)이 표시됩니다.


![REVIEW](assets/fr/06.webp)


참고. 코드 창에서 검토하는 것이 좋으며, 원본 콘텐츠는 별도의 창에 표시되는 것이 좋습니다. 예를 들어 Plan ₿ Network 웹사이트 또는 브라우저 탭의 GitHub 미리 보기에서 텍스트를 볼 수 있습니다. 코드 보기에서 오류를 찾아 수정하려면 `ctrl + F`를 사용하여 이동하면서 수정하세요.


## 4. 변경 사항 커밋



- 필요에 따라 콘텐츠를 검토하고 수정할 수 있습니다.
- 세션을 일시 중지하고 싶을 때는 검토가 끝나지 않았더라도 변경 내용을 커밋할 수 있습니다. 마치 GitHub에 진행 상황을 저장하는 것과 같습니다.
- "변경 사항 커밋..." 버튼을 클릭합니다.


![REVIEW](assets/fr/07.webp)



- 변경 사항에 대한 설명 메시지를 추가합니다(참조: \**A\**).
- 필요한 경우 설명에 추가 컨텍스트를 추가합니다(참조: \**B\**).
- '변경 제안' 버튼을 클릭합니다(참조: \**C\**).


![REVIEW](assets/fr/08.webp)



- 처음 기여하는 것이 아니라면 GitHub에서 새 브랜치를 만들라는 메시지를 표시합니다. 메인 `dev` 브랜치에 직접 커밋해서는 안 됩니다. 이미 작업 중인 브랜치에 커밋을 했다면 해당 브랜치를 선택하세요. 그렇지 않으면 "새 브랜치 만들기" 옵션을 선택하여 새 브랜치를 만듭니다.


![REVIEW](assets/fr/13.webp)



- 그러면 모든 변경 사항을 요약한 페이지에 액세스하게 됩니다. 스크롤을 통해 정확한 수정 사항을 확인할 수 있습니다. 삭제된 항목은 빨간색으로 표시되고 추가된 항목은 Green로 표시됩니다.


![REVIEW](assets/fr/09.webp)



- 검토를 완료하지 않았고 나중에 작업을 계속하려면 간단히 종료하면 됩니다.
- 변경 사항을 커밋하면 GitHub Fork에 저장됩니다.
- 작업을 재개하려면 GitHub 프로필을 통해 Plan ₿ Network 콘텐츠 리포지토리의 Fork로 이동하세요.


![REVIEW](assets/fr/14.webp)



- 진행 상황이 포함된 브랜치를 선택하고(\**A\** 참조), 콘텐츠 파일을 찾은 다음(\**B\** 참조) 연필 아이콘을 클릭하여 검토를 계속합니다.


![REVIEW](assets/fr/15.webp)



## 5. 전담 브랜치로 PR 만들기



- 모든 것이 정상이고 변경을 완료했다면 Green "풀 리퀘스트 만들기" 버튼을 클릭합니다.


![REVIEW](assets/fr/10.webp)



- 그러면 PR 페이지로 이동합니다. 풀 리퀘스트는 원격 리포지토리의 브랜치에서 변경한 내용을 검토하고 메인 브랜치에 병합할 수 있도록 제출하는 요청입니다.
- PR에 제목과 간단한 설명을 추가합니다. 제목은 작업 중인 이슈와 일치해야 합니다. 예를 들어, 문제 제목이 '[번역] 코스 CYP201 - FR'인 경우 PR에도 동일한 제목을 사용합니다(예: \**A\**). 설명에서 관련 이슈의 번호를 사용하여 그 앞에 '종료'를 붙여 언급합니다. 이렇게 하면 PR이 병합되면 자동으로 이슈가 닫힙니다(참조: \**B\**).
- 모든 것이 정상으로 보이면 Green "풀 리퀘스트 만들기" 버튼을 클릭합니다(참조: \**C\**).


![REVIEW](assets/fr/11.webp)
