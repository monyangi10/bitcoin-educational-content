## In sintesi

Se vuoi aiutare facendo proofreading o correggendo i contenuti sul sito Plan ₿ Academy, unisciti al [nostro gruppo Telegram](https://t.me/PlanBNetwork_ContentBuilder) dedicato alle collaborazioni. Presentati brevemente, poi specifica il contenuto che vorresti revisionare e la lingua. Apporta le tue correzioni su un 'branch' di lavoro e invia una PR una volta completata la revisione.

Se non hai capito quello che hai appena letto, non preoccuparti: questo tutorial è per te!

---
## Che cos'è Plan ₿ Academy?

Plan ₿ Academy è una piattaforma dedicata allo studio di Bitcoin attraverso numerosi corsi e tutorial per tutti. Attualmente, tutti i nostri contenuti sono open-source, e si trovano su un repository GitHub, aperto a collaborazioni esterne per la revisione e la creazione di contenuti.

Plan ₿ Academy mira a diventare uno spazio pubblico nel cyberspazio Bitcoin, collegando le community Bitcoin sparse in tutto il mondo.

## Perché collaborare con Plan ₿ Academy?

Plan ₿ Academy è fondamentalmente multilingue, poiché il nostro obiettivo è quello di rendere le risorse su Bitcoin accessibili a tutti, in tutto il mondo. Crediamo che la lingua non debba essere un ostacolo per l'utilizzo della moneta libera.

Per raggiungere questo obiettivo, utilizziamo un approccio che chiamiamo "Human x AI". Attraverso il nostro [programma](https://github.com/Asi0Flammeus/LLM-Translator), possiamo tradurre automaticamente i nostri contenuti in molte lingue. Tuttavia, abbiamo bisogno di una persona che parli correntemente la lingua target per verificare che il contenuto sia comprensibile e privo di termini inappropriati.

Se vuoi contribuire a costruire questo progetto educativo per promuovere l'educazione su Bitcoin nella tua community, puoi iniziare revisionando un corso o un tutorial.

Inoltre, poiché abbiamo a cuore il modello *Value4Value*, avremo cura di inviare un "reward" a coloro che contribuiscono. La ricompensa si baserà sulla lunghezza del testo, sull'accuratezza linguistica della nostra traduzione automatica e sulla difficoltà del contenuto.

## Devo sapere come funziona GitHub?

Non è necessario essere esperti di GitHub per revisionare i contenuti del repository. Sebbene sia sempre meglio capire cosa si stia facendo (e perché), è sufficiente seguire le istruzioni riportate di seguito per cambiare qualcosa e inviare una Pull Request.

Ad ogni modo, se non conosci Git (e GitHub) e vuoi saperne di più, puoi consultare il nostro tutorial introduttivo su questi strumenti:

https://planb.academy/tutorials/contribution/others/create-github-account-a75fc39d-f0d0-44dc-9cd5-cd94aee0c07c

## 1. Crea un account

- Crea un account su [GitHub](https://github.com/) per iniziare a collaborare con Plan ₿ Academy. Se ti serve una guida sulla creazione del tuo account GitHub, puoi cliccare sul tutorial menzionato qui sopra.

- Avrai anche bisogno di un account sull'app di messaggistica [Telegram](https://telegram.org/).

## 2. Unisciti al gruppo Telegram


- Unisciti al [gruppo Plan ₿ Academy Content Builder](https://t.me/PlanBNetwork_ContentBuilder) su Telegram.

![REVIEW](assets/fr/01.webp)


- Presentati brevemente, specifica il contenuto che vuoi revisionare e la lingua di riferimento, mandando un messaggio nella sezione "General".
- Un coordinatore creerà quindi una 'issue' su GitHub dedicata al lavoro che ti verrà assegnato: lì troverai tutte le informazioni necessarie per iniziare.

![REVIEW](assets/fr/02.webp)

## 3. Rivedi e modifica il contenuto tramite l'interfaccia di GitHub


- La 'issue' contiene una descrizione (cfr. **A**) con i link al contenuto originale e al file da revisionare.
- Se vuoi occuparti del proofreading, lascia un commento sulla 'issue' (cfr. **B**) per indicare che te ne occuperai. In questo modo si evita che altre persone prendano in carico lo stesso compito.
- Clicca sul link accanto al file da cambiare (cfr. **C**) per accedere al contenuto.

![REVIEW](assets/fr/03.webp)


- Ora puoi visualizzare l'interfaccia GitHub del file da revisionare.
- Il pannello di sinistra consente di navigare nel tuo branch (cfr. **A**) e nei file (cfr. **B**).
- Clicca sull'icona a forma di matita per avviare la revisione (cfr. **C**).

![REVIEW](assets/fr/04.webp)


- Se questo è il tuo primo contributo, GitHub ti chiederà di creare un fork del repository sorgente. Basta cliccare sul pulsante che vedi nell'immagine.

![REVIEW](assets/fr/05.webp)


- Arriverai poi all'interfaccia dell'editor (cfr. **A**), dove potrai vedere il percorso del file e il 'branch' corrispondente (cfr. **B**). Potrai inoltre passare dalla modalità di modifica a quella di anteprima (cfr. **C**).

![REVIEW](assets/fr/06.webp)

NB. Ti consiglio di effettuare la revisione nella finestra `code`, con il contenuto originale visualizzato in una finestra separata. Ad esempio, puoi visualizzare il testo sul sito web planb.academy o nell'anteprima GitHub in una scheda del browser. Puoi anche utilizzare `ctrl + F` per individuare gli errori comuni e apportare correzioni multiple nella visualizzazione `code`.

## 4. Conferma le modifiche


- Puoi iniziare a rivedere e correggere i contenuti, se necessario.
- Quando vuoi mettere in pausa la sessione, puoi confermare le modifiche, anche se la revisione non è ancora terminata. In questo modo, puoi salvare i tuoi progressi su GitHub.
- Per farlo, clicca sul pulsante "Commit Changes...".

![REVIEW](assets/fr/07.webp)


- Inserisci un messaggio descrittivo per le tue modifiche (cfr. **A**).
- Se necessario, aggiungi più dettagli nella descrizione (cfr. **B**).
- Clicca sul pulsante "Propose changes" (cfr. **C**).

![REVIEW](assets/fr/08.webp)


- Se questo **non** è il tuo primo contributo, GitHub ti chiederà di creare un nuovo branch. Non devi **mai** salvare un commit direttamente sul branch principale `dev`. Se hai gia effettuato dei commit su un branch di lavoro esistente, seleziona proprio quello. Altrimenti, scegli l'opzione "Create a new branch" per crearne uno nuovo.

![REVIEW](assets/fr/13.webp)


- Accederai poi ad una pagina che riassume tutte le modifiche apportate. Scorrendo giù, potrai vedere con maggiore dettaglio ciò che hai cambiato: le voci eliminate appaiono in rosso, mentre quelle aggiunte saranno visualizzate in verde.

![REVIEW](assets/fr/09.webp)


- Se non hai completato la tua revisione e desideri continuare il lavoro in un secondo momento, è sufficiente uscire da questa pagina.
- Nel momento in cui clicchi su "commit changes", le modifiche vengono salvate sul tuo fork GitHub.
- Per riprendere il lavoro, accedi al tuo fork tramite il tuo profilo GitHub.

![REVIEW](assets/fr/14.webp)


- Seleziona il branch che contiene i tuoi progressi (cfr. **A**), individua il contenuto che hai già modificato (cfr. **B**), quindi clicca sull'icona della matita per continuare la revisione.

![REVIEW](assets/fr/15.webp)

## 5. Creare una PR tramite il branch apposito

- Se tutto ti sembra a posto e le modifiche sono state completate, clicca sul pulsante verde "Create Pull Request".

![REVIEW](assets/fr/10.webp)


- A questo punto, accedi alla pagina per creare la PR: una Pull Request è una richiesta di integrazione delle modifiche effettuate su un branch remoto, all'interno di un repository di un progetto. Tramite la PR, questi cambiamenti vengono revisionati ed eventualmente approvati dagli admin del progetto principale.
- Aggiungi un titolo e una breve descrizione alla tua PR. Il primo deve corrispondere alla issue su cui si sta lavorando. Ad esempio, se il titolo della issue è `[PROOFREADING] Course CYP201 - FR`, usa lo stesso titolo per la tua PR (cfr. **A**). Nella descrizione, inserisci il numero della `issue` associata, preceduto da "Closes #". In questo modo, la issue verrà chiusa automaticamente una volta che la PR sarà accettata e integrata (cfr. **B**).
- Se tutto ti sembra a posto, clicca sul pulsante verde "Create Pull Request" (cfr. **C**).

![REVIEW](assets/fr/11.webp)

## 7. In attesa di revisione

- Congratulazioni, la tua PR è stata inviata! puoi seguirne l'avanzamento nella sezione "Pull requests" del [repository GitHub di Plan ₿ Academy](https://github.com/PlanB-Network/bitcoin-educational-content/pulls):

![REVIEW](assets/fr/12.webp)


- Grazie mille per il tuo aiuto! Dopo l'approvazione, un coordinatore del repo di Plan ₿ Academy ti contatterà per inviati la tua ricompensa. \**I pagamenti vengono effettuati esclusivamente tramite la rete Lightning\**.
- Se vuoi effettuare una nuova revisione, torna al punto 2 e invia un nuovo messaggio sul gruppo Telegram. In questo modo, potrai iniziare a lavorare su un altro contenuto. Assicurati di sincronizzare il tuo fork prima di creare un nuovo branch di lavoro. Per farlo, vai nella homepage del tuo fork, seleziona il branch principale chiamato `dev` (cfr. **A**), quindi clicca sul pulsante "Sync fork" (cfr. **B**). Se sono disponibili aggiornamenti per il tuo fork, apparirà un pulsante "Update branch". Ti basterà cliccarlo per eseguire l'aggiornamento.

![REVIEW](assets/fr/16.webp)


- Se hai osservazioni, domande o commenti su questo progetto, sentiti libero di inviare un DM [al gruppo Telegram dei contributors](https://t.me/PlanBNetwork_ContentBuilder).
