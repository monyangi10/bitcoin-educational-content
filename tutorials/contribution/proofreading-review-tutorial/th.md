## สรุป


หากคุณต้องการพิสูจน์อักษรหรือทบทวนเนื้อหาสำหรับ Plan ₿ Academy เข้าร่วม [กลุ่ม Telegram ของเรา](https://t.me/PlanBNetwork_ContentBuilder) ที่อุทิศให้กับการมีส่วนร่วม แนะนำตัวเองสั้นๆ จากนั้นระบุเนื้อหาที่คุณต้องการทบทวนและภาษา ทำการแก้ไขของคุณบนสาขาการทำงานและส่ง PR เมื่อการทบทวนของคุณเสร็จสมบูรณ์


หากคุณไม่เข้าใจส่วนด้านบน ไม่ต้องกังวล—บทแนะนำนี้เหมาะสำหรับคุณ!

## Plan ₿ Academy คืออะไร?


Plan ₿ Academy เป็นแพลตฟอร์มสำหรับการเรียนรู้เกี่ยวกับ Bitcoin ผ่านหลักสูตรและบทเรียนมากมายสำหรับทุกคน ปัจจุบัน เนื้อหาทั้งหมดของเราเป็นโอเพ่นซอร์ส โฮสต์บนที่เก็บ GitHub และเปิดให้มีการมีส่วนร่วมสำหรับการตรวจสอบและการสร้างเนื้อหา


Plan ₿ Academy มีเป้าหมายที่จะกลายเป็นจัตุรัสสาธารณะของไซเบอร์สเปซ Bitcoin โดยเชื่อมโยงและส่งเสริมชุมชน Bitcoin ทั่วโลก


## ทำไมถึงมีส่วนร่วม?


Plan ₿ Academy มีความเป็นพหุภาษาพื้นฐาน เนื่องจากเป้าหมายของเราคือการทำให้ทรัพยากร Bitcoin เข้าถึงได้สำหรับทุกคนทั่วโลก เราเชื่อว่าภาษาไม่ควรเป็นอุปสรรคต่อเงินอธิปไตย


ในการบรรลุเป้าหมายนี้ เราใช้วิธีการ Human x AI ผ่าน [โปรแกรมของเรา](https://github.com/Asi0Flammeus/LLM-Translator) เราสามารถแปลเนื้อหาเป็นหลายภาษาได้โดยอัตโนมัติ อย่างไรก็ตาม เราต้องการผู้ที่พูดภาษาได้อย่างคล่องแคล่วเพื่อตรวจสอบเพื่อให้แน่ใจว่าเนื้อหานั้นเข้าใจได้และไม่มีคำที่ไม่เหมาะสม


หากคุณต้องการช่วยสร้างโครงการการศึกษานี้เพื่อส่งเสริมการศึกษา Bitcoin ในชุมชนของคุณ คุณสามารถเริ่มต้นด้วยการตรวจสอบหลักสูตรหรือบทแนะนำ


นอกจากนี้ เนื่องจากเราชื่นชมโมเดล *Value4Value* เราจะให้รางวัลสำหรับการมีส่วนร่วมในการรีวิว รางวัลจะขึ้นอยู่กับความยาวของเนื้อหา ความถูกต้องทางภาษาของการแปลอัตโนมัติของเรา และความยากของเนื้อหา


## ฉันจำเป็นต้องรู้วิธีการทำงานของ GitHub หรือไม่?


คุณไม่จำเป็นต้องเป็นผู้เชี่ยวชาญ GitHub เพื่อรีวิวเนื้อหา แม้ว่าจะเป็นการดีกว่าที่จะเข้าใจสิ่งที่คุณกำลังทำ (และทำไม) คุณสามารถทำตามคำแนะนำด้านล่างเพื่อมีส่วนร่วมได้เลย


อย่างไรก็ตาม หากคุณไม่คุ้นเคยกับ Git และ GitHub และต้องการเรียนรู้เพิ่มเติม คุณสามารถดูบทความเบื้องต้นเกี่ยวกับเครื่องมือเหล่านี้ได้ที่:


https://planb.academy/tutorials/contribution/others/create-github-account-a75fc39d-f0d0-44dc-9cd5-cd94aee0c07c

## 1. สร้างบัญชี



- สร้างบัญชีบน [GitHub](https://github.com/) เพื่อเริ่มต้นการมีส่วนร่วม หากคุณต้องการคำแนะนำในการสร้างบัญชี GitHub ของคุณ ให้ไปที่บทแนะนำที่กล่าวถึงข้างต้น



- คุณจะต้องมีบัญชีบนแอปส่งข้อความ [Telegram](https://telegram.org/) ด้วยเช่นกัน


## 2. เข้าร่วมกลุ่ม Telegram



- เข้าร่วม [กลุ่ม Plan ₿ Academy Content Builder](https://t.me/PlanBNetwork_ContentBuilder) บน Telegram.


![REVIEW](assets/fr/01.webp)



- แนะนำตัวเองสั้นๆ และระบุเนื้อหาที่คุณต้องการมีส่วนร่วมและภาษาที่ใช้ในข้อความไปยังช่อง "General"
- สมาชิกในทีมจะสร้างปัญหาใน GitHub สำหรับการมีส่วนร่วมในอนาคตของคุณ คุณจะพบข้อมูลทั้งหมดที่จำเป็นในการเริ่มการตรวจสอบของคุณที่นั่น


![REVIEW](assets/fr/02.webp)


## 3. ตรวจสอบและแก้ไขเนื้อหาผ่านอินเทอร์เฟซของ GitHub



- ปัญหานี้มีคำอธิบาย (cf. \**A\**) พร้อมลิงก์ไปยังเนื้อหาต้นฉบับและไฟล์ที่ต้องตรวจสอบ
- หากคุณต้องการรับผิดชอบการตรวจสอบ ให้แสดงความคิดเห็นในปัญหา (cf. \**B\**) เพื่อระบุว่าคุณจะทำงานในเรื่องนี้ วิธีนี้จะป้องกันความพยายามที่ซ้ำซ้อน
- คลิกที่ลิงก์ถัดจากไฟล์เพื่อทบทวน (cf. \**C\**) เพื่อเข้าถึงเนื้อหา


![REVIEW](assets/fr/03.webp)


*คุณสามารถดูอินเทอร์เฟซของ GitHub ของไฟล์เพื่อทำการตรวจสอบได้แล้ว*


- แผงด้านซ้ายช่วยให้คุณนำทางผ่านสาขาของคุณ (cf. \**A\**) และไฟล์ (cf. \**B\**)

*คลิกไอคอนดินสอเพื่อเริ่มการตรวจสอบ (cf. **C**).*


![REVIEW](assets/fr/04.webp)



- หากนี่เป็นการมีส่วนร่วมครั้งแรกของคุณ GitHub จะเตือนให้คุณ fork ที่ที่เก็บต้นทาง เพียงคลิกปุ่ม


![REVIEW](assets/fr/05.webp)



- จากนั้นคุณจะเข้าสู่ส่วนติดต่อของ Editor (cf. \**A\**), ที่ซึ่งคุณสามารถดูเส้นทางของไฟล์และสาขาที่สอดคล้องกัน (cf. \**B\**) และสลับระหว่างโหมดแก้ไขและโหมดแสดงตัวอย่าง (cf. \**C\**).


![REVIEW](assets/fr/06.webp)


หมายเหตุ ขอแนะนำให้ตรวจสอบในหน้าต่างโค้ด โดยแสดงเนื้อหาต้นฉบับในหน้าต่างแยก ตัวอย่างเช่น คุณสามารถดูข้อความบนเว็บไซต์ Plan ₿ Academy หรือในตัวอย่าง GitHub บนแท็บเบราว์เซอร์ ใช้ `ctrl + F` เพื่อค้นหาข้อผิดพลาดและทำการแก้ไขในมุมมองโค้ดขณะที่คุณดำเนินการ


## 4. ยืนยันการเปลี่ยนแปลง



- คุณสามารถเริ่มตรวจสอบและแก้ไขเนื้อหาได้ตามต้องการ
- เมื่อคุณต้องการหยุดพักการทำงาน คุณสามารถบันทึกการเปลี่ยนแปลงของคุณได้ แม้ว่าการตรวจสอบจะยังไม่เสร็จสิ้นก็ตาม มันเหมือนกับการบันทึกความคืบหน้าของคุณบน GitHub
- คลิกปุ่ม "Commit Changes..."


![REVIEW](assets/fr/07.webp)



- เพิ่มข้อความอธิบายสำหรับการเปลี่ยนแปลงของคุณ (cf. \**A\**).
- เพิ่มบริบทในคำอธิบายหากจำเป็น (cf. \**B\**)
- คลิกปุ่ม "เสนอการเปลี่ยนแปลง" (cf. \**C\**).


![REVIEW](assets/fr/08.webp)



- หากนี่ไม่ใช่การมีส่วนร่วมครั้งแรกของคุณ GitHub จะเตือนให้คุณสร้างสาขาใหม่ คุณไม่ควรคอมมิตโดยตรงไปยังสาขา `dev` หลัก หากคุณได้ทำการคอมมิตในสาขาการทำงานที่มีอยู่แล้ว ให้เลือกสาขานั้น มิฉะนั้น ให้เลือกตัวเลือก "Create a new branch" เพื่อสร้างสาขาใหม่


![REVIEW](assets/fr/13.webp)



- จากนั้นคุณจะเข้าถึงหน้าที่สรุปการเปลี่ยนแปลงทั้งหมดของคุณ โดยการเลื่อนผ่าน คุณสามารถดูการปรับเปลี่ยนที่แม่นยำ: รายการที่ถูกลบจะปรากฏเป็นสีแดง ในขณะที่รายการที่เพิ่มจะปรากฏเป็นสีเขียว


![REVIEW](assets/fr/09.webp)



- หากคุณยังไม่ได้ทำการตรวจสอบให้เสร็จสิ้นและต้องการทำงานต่อในภายหลัง คุณสามารถออกได้เลย
- ตราบใดที่คุณได้ทำการ commit การเปลี่ยนแปลงของคุณแล้ว การเปลี่ยนแปลงเหล่านั้นจะถูกบันทึกไว้บน GitHub fork ของคุณ
- ในการกลับมาทำงานของคุณ ให้ไปที่ fork ของที่เก็บเนื้อหา Plan ₿ Academy ผ่านโปรไฟล์ GitHub ของคุณ


![REVIEW](assets/fr/14.webp)



- เลือกสาขาที่มีความก้าวหน้าของคุณ (ดู \**A\**), ค้นหาไฟล์เนื้อหาของคุณ (ดู \**B\**), จากนั้นคลิกที่ไอคอนดินสอเพื่อดำเนินการตรวจสอบต่อไป


![REVIEW](assets/fr/15.webp)



## 5. สร้าง PR ด้วยสาขาที่กำหนด



- หากทุกอย่างดูดีและคุณได้ทำการเปลี่ยนแปลงเสร็จสิ้นแล้ว ให้คลิกปุ่ม "Create Pull Request" สีเขียว


![REVIEW](assets/fr/10.webp)



- จากนั้นคุณจะไปถึงหน้าของ PR คำขอ Pull Request คือคำขอเพื่อส่งการเปลี่ยนแปลงที่ทำบนสาขาของที่เก็บระยะไกลเพื่อการตรวจสอบและอาจรวมเข้ากับสาขาหลัก
- เพิ่มชื่อเรื่องและคำอธิบายสั้น ๆ ให้กับ PR ของคุณ ชื่อเรื่องควรตรงกับปัญหาที่คุณกำลังทำงานอยู่ ตัวอย่างเช่น หากชื่อปัญหาคือ `[TRANSLATION] Course CYP201 - FR` ให้ใช้ชื่อเดียวกันสำหรับ PR ของคุณ (ดู \**A\**) ในคำอธิบาย ให้กล่าวถึงปัญหาที่เกี่ยวข้องโดยใช้หมายเลขของมัน นำหน้าด้วยคำว่า "Closes" ซึ่งจะปิดปัญหาโดยอัตโนมัติเมื่อ PR ถูกรวม (ดู \**B\**)
- หากทุกอย่างดูดี ให้คลิกปุ่ม "Create Pull Request" สีเขียว (cf. \**C\**)


![REVIEW](assets/fr/11.webp)


## 6. กฎที่ต้องปฏิบัติตามเมื่อแก้ไขเนื้อหา


หากต้องการทราบข้อมูลเพิ่มเติมเกี่ยวกับแนวทางการใช้เมื่อพิสูจน์อักษรบน Plan ₿ Academy ในแง่ของกฎภาษา หรือรูปแบบ โปรดดูที่บทแนะนำ "contribution-proofreading-guidelines" ในส่วนนี้!


## 7. รอการตรวจสอบ



- ยินดีด้วย! PR ของคุณได้ถูกส่งเรียบร้อยแล้ว! คุณสามารถติดตามความคืบหน้าได้ในแท็บ "Pull requests" บน [ที่เก็บ GitHub ของ Plan ₿ Academy](https://github.com/PlanB-Network/bitcoin-educational-content/pulls):


![REVIEW](assets/fr/12.webp)



- ขอบคุณมากสำหรับการมีส่วนร่วมของคุณ! หลังจากการอนุมัติ สมาชิกทีม Plan ₿ Academy จะติดต่อคุณเพื่อส่งรางวัลของคุณ \**การชำระเงินทำผ่าน Lightning Network เท่านั้น\**.
- หากคุณต้องการเริ่มใหม่ ให้กลับไปที่ขั้นตอนที่ 2 และส่งข้อความใหม่ในกลุ่ม Telegram เพื่อดำเนินการมีส่วนร่วมกับเนื้อหาอื่น ๆ ต่อไป อย่าลืมซิงค์ fork ของคุณก่อนที่จะสร้างสาขาการทำงานใหม่ ในการทำเช่นนี้ ให้ไปที่หน้าแรกของ fork ของคุณ เลือกสาขาหลักที่ชื่อว่า `dev` (ดู \**A\**) จากนั้นคลิกที่ปุ่ม "Sync fork" (ดู \**B\**) หากมีการอัปเดตสำหรับ fork ของคุณ ปุ่ม "Update branch" จะปรากฏขึ้น เพียงคลิกเพื่อทำการอัปเดต


![REVIEW](assets/fr/16.webp)



- หากคุณมีข้อคิดเห็น คำถาม หรือความคิดเห็นเกี่ยวกับโครงการนี้ สามารถส่ง DM [ไปยังกลุ่ม Telegram ของผู้ร่วมงาน](https://t.me/PlanBNetwork_ContentBuilder) ได้เลย
