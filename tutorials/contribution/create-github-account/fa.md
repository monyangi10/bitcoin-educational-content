مأموریت Plan ₿ ارائه منابع آموزشی درجه یک درباره Bitcoin است که در بیشترین تعداد زبان ممکن در دسترس باشد. تمام محتوای منتشر شده در سایت منبع‌باز است و در GitHub میزبانی می‌شود، که به هر کسی فرصت می‌دهد تا به غنی‌سازی این پلتفرم کمک کند. مشارکت‌ها می‌توانند به اشکال مختلفی باشند: تصحیح و ویرایش متون موجود، ترجمه به زبان‌های دیگر، به‌روزرسانی اطلاعات، یا ایجاد آموزش‌های جدیدی که هنوز در سایت ما موجود نیستند.


اگر می‌خواهید به Plan ₿ Academy کمک کنید، باید از Git و GitHub استفاده کنید، بنابراین، اگر این ابزارها برای شما ناآشنا هستند یا عملکرد آن‌ها مبهم به نظر می‌رسد، نگران نباشید، این مقاله برای شماست! ما با هم اصول Git و GitHub و همچنین اصطلاحات فنی مرتبط را مرور خواهیم کرد تا بتوانید پس از آن‌ها به طور مؤثر از آن‌ها استفاده کنید.


## گیت چیست؟


گیت یک سیستم کنترل نسخه است که به‌طور خاص برای مدیریت پروژه‌های نرم‌افزاری طراحی شده است. گیت که در سال ۲۰۰۵ توسط لینوس توروالدز ایجاد شد، به سرعت به استانداردی در صنعت توسعه نرم‌افزار برای کنترل نسخه تبدیل شده است. اما دقیقاً به چه معناست؟

![git](assets/11.webp)

در اصل، Git به توسعه‌دهندگان اجازه می‌دهد تا تغییرات ایجاد شده در کد منبع یک پروژه را در طول زمان پیگیری کنند. این بدان معناست که با هر تغییر در کد، Git یک نسخه جدید از پروژه را ثبت می‌کند. اگر خطایی رخ دهد یا اگر یک ویژگی آزمایشی به‌طور مورد انتظار کار نکند، امکان بازگشت به حالت قبلی کد وجود دارد، مانند یک ماشین زمان، اما برای فایل‌ها.


یکی از ویژگی‌های کلیدی گیت مدیریت شاخه است. شاخه یک نوع خط موازی است که در آن توسعه‌دهندگان می‌توانند به‌طور مستقل از بقیه پروژه کار کنند. این امر به‌طور قابل‌توجهی افزودن ویژگی‌های جدید یا اصلاح اشکالات را بدون ایجاد اختلال در کد اصلی تسهیل می‌کند. هنگامی که تغییرات آزمایش و تأیید شدند، می‌توانند با شاخه اصلی ادغام شوند.


یکی از ویژگی‌های خاص Git توانایی آن در عمل به صورت توزیع‌شده است. هر توسعه‌دهنده یک نسخه کامل از پروژه را بر روی درایو Hard کامپیوتر خود دارد، که به آن‌ها اجازه می‌دهد به صورت آفلاین کار کنند و تغییرات را بعداً زمانی که اتصال به اینترنت موجود است، ادغام کنند. این امر خطر بروز تعارضات را کاهش می‌دهد و به چندین توسعه‌دهنده اجازه می‌دهد به طور همزمان بر روی یک پروژه کار کنند بدون اینکه مزاحم یکدیگر شوند.

در ابتدا، Git به‌طور عمده برای پروژه‌های توسعه نرم‌افزار طراحی شده بود. با این حال، می‌توان از آن برای مدیریت پروژه‌های نوشتن محتوا نیز استفاده کرد. به جای همکاری بر روی کد، ما بر روی متن همکاری می‌کنیم. و دقیقاً این روش است که Plan ₿ Academy برای مدیریت محتوای خود اتخاذ کرده است! Git همکاری در نوشتن دوره‌ها و آموزش‌ها را تسهیل می‌کند، زیرا امکان ردیابی دقیق تغییرات، مدیریت نسخه کارآمد و همچنین بررسی و بهبود محتوا توسط دیگر مشارکت‌کنندگان را فراهم می‌کند.


## گیت‌هاب چیست؟


GitHub یک پلتفرم مدیریت و میزبانی کد منبع است که بر اساس سیستم کنترل نسخه Git که به تازگی درباره آن صحبت کردیم، ساخته شده است. GitHub که در سال ۲۰۰۸ راه‌اندازی شد، به سرعت محبوبیت پیدا کرد و به یک مرجع ضروری برای توسعه‌دهندگان در سراسر جهان تبدیل شده است. اما GitHub چگونه با Git تفاوت دارد و چرا در روش تولید محتوای ما این‌قدر مهم است؟

![github](assets/12.webp)

اول، مهم است که بفهمیم GitHub بر پایه Git ساخته شده است (که در بخش قبلی درباره آن صحبت کردیم). در حالی که Git ابزاری است که تغییرات کد را پیگیری می‌کند، GitHub سرویس آنلاین است که این کد را میزبانی، به اشتراک‌گذاری و مدیریت می‌کند.


تصور کنید Git به عنوان نوعی دفترچه ثبت است که هر توسعه‌دهنده از آن در رایانه خود برای ثبت تمام تغییرات در پروژه‌شان استفاده می‌کند. از سوی دیگر، GitHub مانند یک کتابخانه عمومی است که در آن همه این دفترچه‌های ثبت می‌توانند به اشتراک گذاشته شوند، مقایسه شوند و ترکیب شوند.


تفاوت اساسی بین Git و GitHub در عملکرد آنها نهفته است: Git ابزاری است که به صورت محلی توسط هر توسعه‌دهنده برای مدیریت نسخه‌های کد خود استفاده می‌شود، در حالی که GitHub پلتفرم آنلاین است که این نسخه‌ها را میزبانی کرده و همکاری را تسهیل می‌کند.


GitHub چیزی فراتر از یک سرویس میزبانی کد است. این یک پلتفرم همکاری است که به توسعه‌دهندگان اجازه می‌دهد به‌طور مؤثر با یکدیگر کار کنند. و در واقع، Plan ₿ Academy از این پلتفرم برای میزبانی نه تنها تمام کدی که وب‌سایت را قدرت می‌بخشد، بلکه، و این چیزی است که ما را علاقه‌مند می‌کند، تمام محتوا (آموزش‌ها، تمرین‌ها، منابع...) استفاده می‌کند.


## برخی اصطلاحات فنی


در Git و GitHub، با دستورات و ویژگی‌هایی مواجه خواهید شد که نام‌هایشان ممکن است پیچیده به نظر برسند. در این بخش آخر، تعاریف ساده‌ای ارائه خواهم داد تا اصطلاحات فنی که ممکن است هنگام استفاده از Git و GitHub با آن‌ها روبرو شوید را بهتر درک کنید:



- **دریافت مبدا:** دستوری که اطلاعات و تغییرات اخیر را از یک مخزن راه دور بازیابی می‌کند بدون اینکه آن‌ها را با کار محلی شما ادغام کند. این دستور مخزن محلی شما را با شاخه‌ها و تعهدات جدید موجود در مخزن راه دور به‌روزرسانی می‌کند.



- **کشیدن از مبدا:** دستوری که به‌روزرسانی‌ها را از یک مخزن راه دور دریافت کرده و بلافاصله آن‌ها را در شاخه محلی شما ادغام می‌کند تا آن را همگام‌سازی کند. این دستور مراحل دریافت و ادغام را در یک دستور واحد ترکیب می‌کند.
- **همگام‌سازی Fork:** یک ویژگی در GitHub که به شما اجازه می‌دهد Fork پروژه خود را با آخرین تغییرات از مخزن منبع به‌روزرسانی کنید. این اطمینان را می‌دهد که نسخه شما از پروژه با توسعه اصلی به‌روز باقی می‌ماند.
- **Push origin:** دستوری که برای ارسال تغییرات محلی شما به یک مخزن راه دور استفاده می‌شود.



- **درخواست کشش:** درخواستی که توسط یک مشارکت‌کننده ارسال می‌شود تا نشان دهد که تغییراتی را به یک شاخه در یک مخزن راه دور ارسال کرده‌اند و می‌خواهند این تغییرات بررسی شده و به‌طور بالقوه در شاخه اصلی مخزن ادغام شوند.



- **تعهد:** ذخیره تغییرات شما. یک تعهد مانند یک عکس فوری از کار شما در یک لحظه خاص است که امکان نگهداری تاریخچه تغییرات را فراهم می‌کند. هر تعهد شامل یک پیام توصیفی است که توضیح می‌دهد چه چیزی تغییر کرده است.



- **شاخه:** نسخه‌ای موازی از مخزن که به شما اجازه می‌دهد بدون تأثیر بر شاخه اصلی (که اغلب "main" یا "master" نامیده می‌شود) روی تغییرات کار کنید. شاخه‌ها توسعه ویژگی‌های جدید و اصلاح اشکالات را بدون خطر اختلال در کد پایدار تسهیل می‌کنند.



- **ادغام:** ادغام شامل یکپارچه‌سازی تغییرات از یک شاخه به شاخه دیگر است. به عنوان مثال، برای افزودن تغییرات از یک شاخه کاری به شاخه اصلی استفاده می‌شود که این امکان را می‌دهد تا مشارکت‌های مختلف اضافه شوند.



- **Fork:** فورک کردن یک مخزن به معنای ایجاد یک کپی از آن مخزن در حساب GitHub خودتان است که به شما اجازه می‌دهد روی پروژه کار کنید بدون اینکه مخزن اصلی را تحت تأثیر قرار دهید. Fork می‌تواند راه خود را برود و به یک پروژه متفاوت از اصلی تبدیل شود، یا می‌تواند به طور منظم با پروژه اصلی همگام‌سازی شود تا به آن کمک کند.



- **کلون:** کلون کردن یک مخزن به معنای ایجاد یک کپی محلی بر روی کامپیوتر شما است که به شما دسترسی به تمام فایل‌ها و تاریخچه را می‌دهد. این به شما اجازه می‌دهد تا به‌طور مستقیم به‌صورت محلی بر روی پروژه کار کنید.



- **مخزن:** فضای ذخیره‌سازی برای یک پروژه در GitHub. یک مخزن شامل تمام فایل‌های پروژه و همچنین تاریخچه تمام تغییراتی است که بر روی آن انجام شده است. این اساس ذخیره‌سازی و همکاری در GitHub است.



- **مشکل:** ابزاری برای پیگیری وظایف و باگ‌ها در GitHub. مشکلات امکان گزارش مشکلات، پیشنهاد بهبودها یا بحث در مورد ویژگی‌های جدید را فراهم می‌کنند. هر مشکل می‌تواند اختصاص داده شود، برچسب‌گذاری شود و نظراتی بر روی آن گذاشته شود.


این فهرست به وضوح کامل نیست. اصطلاحات فنی بسیاری دیگری وجود دارند که به Git و GitHub اختصاص دارند. با این حال، آن‌هایی که در اینجا ذکر شده‌اند، اصلی‌ترین‌هایی هستند که به‌طور مکرر با آن‌ها مواجه خواهید شد.


پس از خواندن این مقاله، ممکن است برخی جنبه‌های Git و GitHub هنوز برای شما نامشخص باشد، بنابراین شما را تشویق می‌کنم که خودتان شروع به استفاده از این ابزارها کنید. تمرین اغلب بهترین راه برای درک چگونگی کارکرد ماشین است! و برای شروع، می‌توانید این ۲ آموزش دیگر را کشف کنید:


## چگونه یک حساب GitHub ایجاد کنیم


اگر می‌خواهید به شبکه PlanB کمک کنید، به یک حساب GitHub نیاز خواهید داشت. در این آموزش، ما شما را گام به گام راهنمایی خواهیم کرد که چگونه حساب خود را ایجاد کنید، آن را تنظیم کنید و به درستی ایمن کنید.



- به [https://github.com/signup](https://github.com/signup) بروید.
- ایمیل خود را وارد کنید Address، سپس روی دکمه `ادامه` Green کلیک کنید:

![github](assets/1.webp)


- یک رمز عبور قوی انتخاب کنید، سپس روی دکمه `ادامه` Green کلیک کنید:

![github](assets/12.webp)


- سپس، نام کاربری خود را انتخاب کنید. می‌توانید هویت واقعی خود را فاش کنید یا از یک نام مستعار استفاده کنید. سپس، روی دکمه `ادامه` Green کلیک کنید:

![github](assets/3.webp)


- کپچا را کامل کنید:

![github](assets/4.webp)


- یک ایمیل حاوی کد تأیید برای شما ارسال خواهد شد؛ شما باید آن را وارد کنید تا ایجاد حساب کاربری خود را نهایی کنید:

![github](assets/5.webp)


- سوالات را پر کنید اگر می‌خواهید GitHub شما را به سمت ابزارهای خاصی هدایت کند، یا روی `skip personalization` کلیک کنید تا از این مرحله بگذرید:

![github](assets/6.webp)


- با کلیک بر روی دکمه `ادامه به صورت رایگان`، طرح رایگان را انتخاب کنید:

![github](assets/7.webp)


- سپس به داشبورد خود هدایت خواهید شد.
- اگر مایلید، می‌توانید حساب کاربری خود را با کلیک بر روی تصویر پروفایل خود که در بالای سمت راست صفحه قرار دارد، سفارشی کنید و سپس به منوی `تنظیمات` دسترسی پیدا کنید:

![github](assets/8.webp)


- در این بخش، شما این امکان را دارید که یک عکس پروفایل جدید اضافه کنید، نامی انتخاب کنید، بیوگرافی خود را سفارشی کنید، یا لینکی به وب‌سایت شخصی خود اضافه کنید:

![github](assets/9.webp)
