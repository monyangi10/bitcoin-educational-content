Plan ₿의 사명은 Bitcoin에 대한 최고 수준의 교육 리소스를 가능한 한 많은 언어로 제공하는 것입니다. 이 사이트에 게시된 모든 콘텐츠는 오픈 소스이며 GitHub에서 호스팅되므로 누구나 플랫폼 강화에 기여할 수 있는 기회를 제공합니다. 기여는 기존 텍스트의 수정 및 교정, 다른 언어로 번역, 정보 업데이트, 아직 제공되지 않는 새로운 튜토리얼 제작 등 다양한 형태로 이루어질 수 있습니다.


Plan ₿ Network에 기여하려면 Git과 GitHub를 사용해야 하므로 이 도구가 익숙하지 않거나 기능이 모호해 보이더라도 당황하지 마세요. 이 글은 여러분을 위한 것입니다! 나중에 효과적으로 사용할 수 있도록 Git과 GitHub의 기본 사항과 관련 기술 전문 용어를 함께 살펴볼 것입니다.


## Git이란 무엇인가요?


Git은 소프트웨어 프로젝트를 관리하기 위해 특별히 고안된 버전 관리 시스템입니다. 2005년 라이너스 토발즈가 만든 Git은 소프트웨어 개발 업계에서 버전 관리를 위한 표준으로 빠르게 자리 잡았습니다. 하지만 정확히 무엇을 의미할까요?

![git](assets/11.webp)

Git의 핵심은 개발자가 시간이 지남에 따라 프로젝트의 소스 코드에 대한 변경 사항을 추적할 수 있도록 하는 것입니다. 즉, 코드가 변경될 때마다 Git은 프로젝트의 새 버전을 기록합니다. 오류가 발생하거나 실험적인 기능이 예상대로 작동하지 않는 경우 일종의 타임머신처럼 코드의 이전 상태로 되돌릴 수 있지만 파일에 대해서는 불가능합니다.


Git의 핵심 기능 중 하나는 브랜치 관리입니다. 브랜치는 개발자가 프로젝트의 나머지 부분과 독립적으로 작업할 수 있는 일종의 평행선입니다. 따라서 메인 코드를 방해하지 않고 새로운 기능을 추가하거나 버그를 수정하는 데 매우 용이합니다. 수정 사항을 테스트하고 검증한 후에는 메인 브랜치에 병합할 수 있습니다.


Git의 특징 중 하나는 분산된 방식으로 작동한다는 점입니다. 각 개발자는 자신의 컴퓨터의 Hard 드라이브에 프로젝트의 전체 사본을 가지고 있으므로 오프라인에서 작업하고 나중에 인터넷 연결이 가능할 때 변경 사항을 병합할 수 있습니다. 이렇게 하면 충돌의 위험이 줄어들고 여러 개발자가 서로의 발끝을 밟지 않고도 동일한 프로젝트에서 동시에 작업할 수 있습니다.

처음에 Git은 주로 소프트웨어 개발 프로젝트를 위해 설계되었습니다. 하지만 콘텐츠 작성 프로젝트를 관리하는 데에도 사용할 수 있습니다. 코드 공동 작업 대신 텍스트 공동 작업을 하죠. Plan ₿ Network가 콘텐츠 관리를 위해 채택한 방법이 바로 이 방법입니다! Git은 변경 사항을 정확하게 추적하고 효율적인 버전 관리가 가능하며 다른 기여자들이 콘텐츠를 검토하고 개선할 수 있기 때문에 강좌 및 튜토리얼 작성에 있어 협업을 용이하게 해줍니다.


## GitHub란 무엇인가요?


깃허브는 방금 설명한 Git 버전 관리 시스템을 기반으로 하는 소스 코드 관리 및 호스팅 플랫폼입니다. 2008년에 출시된 GitHub는 빠르게 인기를 얻으며 전 세계 개발자들의 필수 참고 자료가 되었습니다. 그렇다면 GitHub는 Git과 어떻게 다르며, 콘텐츠 제작 방식에서 왜 그렇게 중요한가요?

![github](assets/12.webp)

먼저, GitHub는 이전 섹션에서 설명한 Git을 기반으로 구축되었다는 점을 이해하는 것이 중요합니다. Git이 코드 변경 사항을 추적하는 도구라면, GitHub는 이 코드를 호스팅, 공유 및 관리하는 온라인 서비스입니다.


각 개발자가 자신의 컴퓨터에서 프로젝트의 모든 변경 사항을 기록하기 위해 사용하는 일종의 로그북이라고 상상해 보세요. 반면에 GitHub는 이러한 모든 로그북을 공유하고, 비교하고, 결합할 수 있는 공용 라이브러리와 같습니다.


Git과 GitHub의 근본적인 차이점은 기능에 있습니다: Git은 각 개발자가 코드 버전을 관리하기 위해 로컬에서 사용하는 도구인 반면, GitHub는 이러한 버전을 호스팅하고 협업을 촉진하는 온라인 플랫폼입니다.


GitHub는 단순한 코드 호스팅 서비스 그 이상입니다. 개발자들이 효율적으로 협업할 수 있는 협업 플랫폼입니다. 실제로 Plan ₿ Network는 이 플랫폼을 사용하여 웹사이트를 구동하는 모든 코드뿐만 아니라 모든 콘텐츠(튜토리얼, 교육, 리소스 등)도 호스팅합니다.


## 일부 기술 용어


Git과 GitHub에서는 이름이 복잡해 보일 수 있는 명령어와 기능을 만나게 됩니다. 이 마지막 파트에서는 Git과 GitHub를 사용할 때 접할 수 있는 기술 용어를 이해하기 위해 몇 가지 간단한 정의를 제공하겠습니다:



- Fetch origin:** 로컬 작업과 병합하지 않고 원격 리포지토리에서 최근 정보와 변경 내용을 검색하는 명령입니다. 원격 리포지토리에 있는 새 브랜치 및 커밋으로 로컬 리포지토리를 업데이트합니다.



- 원본 가져오기:** 원격 리포지토리에서 업데이트를 검색하여 로컬 브랜치에 즉시 통합하여 동기화하는 명령입니다. 이 명령은 가져오기와 병합 단계를 하나의 명령으로 결합합니다.
- Fork 동기화: ** 소스 리포지토리의 최신 변경 사항으로 프로젝트의 Fork를 업데이트할 수 있는 GitHub의 기능입니다. 이렇게 하면 프로젝트의 복사본이 주요 개발 내용을 최신 상태로 유지할 수 있습니다.
- 푸시 원본:** 로컬 변경 내용을 원격 리포지토리로 보내는 데 사용되는 명령입니다.



- 풀 리퀘스트: ** 기여자가 원격 리포지토리의 브랜치에 변경 사항을 푸시했으며 이러한 변경 사항을 검토하여 리포지토리의 메인 브랜치에 병합되기를 원한다는 의미로 보내는 요청입니다.



- 커밋:** 변경 내용을 저장합니다. 커밋은 특정 시점의 작업 내용을 즉시 스냅샷으로 저장하는 것으로, 변경 내역을 보관할 수 있습니다. 각 커밋에는 수정된 내용을 설명하는 설명 메시지가 포함됩니다.



- 브랜치: ** 리포지토리의 병렬 버전으로, 메인 브랜치(흔히 "메인" 또는 "마스터"라고 함)에 영향을 주지 않고 변경 작업을 할 수 있습니다. 브랜치를 사용하면 안정적인 코드를 방해할 위험 없이 새로운 기능을 개발하고 버그를 수정할 수 있습니다.



- 병합:** 병합은 한 브랜치의 변경 내용을 다른 브랜치에 통합하는 작업입니다. 예를 들어 작업 중인 브랜치의 변경 내용을 메인 브랜치에 추가하여 다양한 기여를 추가할 수 있도록 하는 데 사용됩니다.



- Fork: ** 리포지토리를 포크한다는 것은 자신의 GitHub 계정에 해당 리포지토리의 복사본을 만들어 원본 리포지토리에 영향을 주지 않고 프로젝트 작업을 할 수 있도록 하는 것을 의미합니다. Fork는 독자적인 방식으로 진행하여 원본과 다른 프로젝트가 되거나, 원본 프로젝트와 정기적으로 동기화하여 프로젝트에 기여할 수 있습니다.



- 복제:** 리포지토리를 복제한다는 것은 컴퓨터에 로컬 복사본을 만들어 모든 파일과 기록에 액세스할 수 있다는 뜻입니다. 이렇게 하면 로컬에서 직접 프로젝트를 작업할 수 있습니다.



- 리포지토리:** GitHub의 프로젝트 저장 공간입니다. 리포지토리에는 모든 프로젝트 파일과 프로젝트에 적용된 모든 변경 내역이 포함됩니다. 리포지토리는 GitHub에서 저장 및 협업의 기본입니다.



- 이슈:** GitHub에서 작업과 버그를 추적하기 위한 도구입니다. 이슈를 통해 문제를 보고하고, 개선 사항을 제안하거나, 새로운 기능에 대해 논의할 수 있습니다. 각 이슈는 할당하고, 레이블을 지정하고, 댓글을 달 수 있습니다.


물론 이 목록이 전부는 아닙니다. Git 및 GitHub와 관련된 다른 많은 기술 용어가 있습니다. 하지만 여기에 언급된 용어는 자주 접하게 될 주요 용어입니다.


이 글을 읽은 후에도 Git과 GitHub의 일부 측면이 아직 명확하지 않을 수 있으므로 이러한 도구를 직접 사용해 보는 것이 좋습니다. 연습은 종종 기계의 작동 방식을 이해하는 가장 좋은 방법입니다! 시작하려면 다음 두 가지 튜토리얼을 참조하세요:


## GitHub 계정을 만드는 방법


PlanB 네트워크에 기여하려면 GitHub 계정이 필요합니다. 이 튜토리얼에서는 계정을 만들고, 설정하고, 올바르게 보호하는 방법을 단계별로 안내해 드립니다.



- Https://github.com/signup](https://github.com/signup)로 이동합니다.
- 이메일 Address을 입력한 후 Green `계속` 버튼을 클릭합니다:

![github](assets/1.webp)


- 강력한 비밀번호를 선택한 다음 Green `계속` 버튼을 클릭합니다:

![github](assets/12.webp)


- 다음으로 사용자 아이디를 선택합니다. 실제 신원을 밝히거나 가명을 사용할 수 있습니다. 그런 다음 Green `계속` 버튼을 클릭합니다:

![github](assets/3.webp)


- 보안 문자를 입력합니다:

![github](assets/4.webp)


- 확인 코드가 포함된 이메일이 전송되며, 이를 입력해야 계정 생성을 완료할 수 있습니다:

![github](assets/5.webp)


- GitHub에서 특정 도구에 대한 안내를 받으려면 질문을 입력하거나 '개인화 건너뛰기'를 클릭하여 건너뛰세요:

![github](assets/6.webp)


- '무료 계속하기' 버튼을 클릭하여 무료 요금제를 선택하세요:

![github](assets/7.webp)


- 그러면 대시보드로 리디렉션됩니다.
- 원하는 경우 화면 오른쪽 상단에 있는 프로필 사진을 클릭한 다음 '설정' 메뉴에 액세스하여 계정을 맞춤 설정할 수 있습니다:

![github](assets/8.webp)


- 이 섹션에서는 새 프로필 사진을 추가하고, 이름을 선택하고, 약력을 맞춤 설정하거나, 개인 웹사이트 링크를 추가할 수 있습니다:

![github](assets/9.webp)


- 또한 '비밀번호 및 인증' 메뉴를 방문하여 최소 2단계 인증을 설정하는 것이 좋습니다:

![github](assets/10.webp)
