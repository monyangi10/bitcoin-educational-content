Tutakagua pamoja misingi ya Git na GitHub, pamoja na jargon husika ya kiufundi, ili kukuwezesha kuzitumia kwa ufanisi baadaye.


## Git ni nini?


Git ni mfumo wa kudhibiti toleo, iliyoundwa mahsusi kudhibiti miradi ya programu. Iliyoundwa mnamo 2005 na Linus Torvalds, Git imekuwa kawaida katika tasnia ya ukuzaji wa programu kwa udhibiti wa toleo. Lakini ina maana gani hasa?

![git](assets/11.webp)

Kwa msingi wake, Git inaruhusu wasanidi programu kufuatilia mabadiliko yaliyofanywa kwa msimbo wa chanzo wa mradi kwa wakati. Hii inamaanisha kuwa kwa kila mabadiliko katika nambari, Git hurekodi toleo jipya la mradi. Ikiwa hitilafu itatokea au ikiwa kipengele cha majaribio hakifanyi kazi inavyotarajiwa, inawezekana kurejesha hali ya awali ya msimbo, kama aina ya mashine ya saa, lakini kwa faili.


Moja ya sifa kuu za Git ni usimamizi wa tawi. Tawi ni aina ya mstari sambamba ambapo watengenezaji wanaweza kufanya kazi kwa kujitegemea kutoka kwa mradi wote. Hii inawezesha sana uongezaji wa vipengele vipya au urekebishaji wa hitilafu bila kusumbua msimbo kuu. Mara tu marekebisho yamejaribiwa na kuthibitishwa, yanaweza kuunganishwa na tawi kuu.


Moja ya sifa za Git ni uwezo wake wa kufanya kazi kwa njia iliyosambazwa. Kila msanidi ana nakala kamili ya mradi kwenye kiendeshi cha Hard cha kompyuta yake, kinachowaruhusu kufanya kazi nje ya mtandao na kuunganisha mabadiliko baadaye muunganisho wa Mtandao utakapopatikana. Hii inapunguza hatari ya migogoro na inaruhusu watengenezaji wengi kufanya kazi kwa wakati mmoja kwenye mradi huo bila kukanyaga vidole vyake.

Hapo awali, Git iliundwa kimsingi kwa miradi ya ukuzaji wa programu. Hata hivyo, inaweza pia kutumika kusimamia miradi ya uandishi wa maudhui. Badala ya kushirikiana kwenye msimbo, tunashirikiana kwenye maandishi. Na ni njia hii ambayo Plan ₿ Network imepitisha kudhibiti yaliyomo! Git hurahisisha ushirikiano katika uandishi wa kozi na mafunzo, kwani inaruhusu ufuatiliaji sahihi wa mabadiliko, usimamizi bora wa toleo, na pia kuwezesha ukaguzi na uboreshaji wa yaliyomo na wachangiaji wengine.


## GitHub ni nini?


GitHub ni usimamizi wa msimbo wa chanzo na jukwaa la kukaribisha kulingana na mfumo wa udhibiti wa toleo la Git ambao tumejadili hivi punde. Ilizinduliwa mnamo 2008, GitHub ilipata umaarufu haraka na imekuwa rejeleo muhimu kwa watengenezaji ulimwenguni kote. Lakini GitHub inatofautiana vipi na Git, na kwa nini ni muhimu sana katika njia yetu ya utengenezaji wa yaliyomo?

![github](assets/12.webp)

Kwanza, ni muhimu kuelewa kuwa GitHub imejengwa kwenye Git (ambayo tulijadili katika sehemu iliyopita). Wakati Git ni zana inayofuatilia mabadiliko ya msimbo, GitHub ni huduma ya mtandaoni ambayo inapangisha, kushiriki, na kudhibiti msimbo huu.


Fikiria Git kama aina ya kitabu cha kumbukumbu ambacho kila msanidi hutumia kwenye kompyuta yake kurekodi mabadiliko yote katika mradi wao. GitHub, kwa upande mwingine, ni kama maktaba ya umma ambapo vitabu hivi vyote vya kumbukumbu vinaweza kushirikiwa, kulinganishwa, na kuunganishwa.


Tofauti ya kimsingi kati ya Git na GitHub iko katika utendaji wao: Git ni zana inayotumiwa ndani na kila msanidi programu kudhibiti matoleo yao ya misimbo, huku GitHub ni jukwaa la mtandaoni linalopangisha matoleo haya na kuwezesha ushirikiano.


GitHub ni zaidi ya huduma ya mwenyeji wa nambari. Ni jukwaa la ushirikiano ambalo huruhusu wasanidi programu kufanya kazi pamoja kwa ufanisi. Na hakika, Plan ₿ Network hutumia jukwaa hili kupangisha sio tu msimbo wote unaowezesha tovuti lakini pia, na hii ndiyo inatuvutia, maudhui yote (mafunzo, mafunzo, rasilimali ...).


## Baadhi ya Masharti ya Kiufundi


Kwenye Git na GitHub, utakutana na amri na vipengele ambavyo majina yake yanaweza kuonekana kuwa magumu. Katika sehemu hii ya mwisho, nitatoa ufafanuzi rahisi kuelewa maneno ya kiufundi ambayo unaweza kupata unapotumia Git na GitHub:



- Leta asili:** Amri inayorejesha maelezo ya hivi majuzi na mabadiliko kutoka kwa hazina ya mbali bila kuyaunganisha na kazi yako ya karibu. Inasasisha hazina yako ya karibu na matawi mapya na kujitolea sasa kwenye hazina ya mbali.



- Vuta asili:** Amri inayorejesha masasisho kutoka kwa hazina ya mbali na kuyaunganisha mara moja kwenye tawi lako la karibu ili kusawazisha. Hii inachanganya hatua za kuchota na kuunganisha katika amri moja.
- Sawazisha Fork:** Kipengele kwenye GitHub kinachokuruhusu kusasisha Fork yako ya mradi na mabadiliko ya hivi punde kutoka hazina ya chanzo. Hii inahakikisha kwamba nakala yako ya mradi inasalia kusasishwa na usanidi mkuu.
- Asili ya push:** Amri inayotumiwa kutuma mabadiliko yako ya ndani kwenye hazina ya mbali.



- Ombi la Kuvuta:** Ombi lililotumwa na mchangiaji kuashiria kwamba wamesukuma mabadiliko kwenye tawi katika hazina ya mbali na wanataka mabadiliko haya yakaguliwe na uwezekano wa kuunganishwa katika tawi kuu la hazina.



- Ahadi:** Inahifadhi mabadiliko yako. Ahadi ni kama picha ya papo hapo ya kazi yako kwa wakati fulani, ambayo inaruhusu kuweka historia ya mabadiliko. Kila ahadi inajumuisha ujumbe wa maelezo unaoelezea kile ambacho kimerekebishwa.



- Tawi:** Toleo sambamba la hazina, linalokuruhusu kufanya kazi kwenye mabadiliko bila kuathiri tawi kuu (mara nyingi huitwa "kuu" au "bwana"). Matawi huwezesha uundaji wa vipengele vipya na urekebishaji wa hitilafu bila hatari ya kuvuruga msimbo thabiti.



- Unganisha:** Kuunganisha kunajumuisha kuunganisha mabadiliko kutoka tawi moja hadi jingine. Inatumika, kwa mfano, kuongeza mabadiliko kutoka kwa tawi la kufanya kazi hadi tawi kuu, ambayo inaruhusu kuongeza michango mbalimbali.



- Fork:** Kuunda hazina inamaanisha kuunda nakala ya hazina hiyo kwenye akaunti yako ya GitHub, ambayo hukuruhusu kufanya kazi kwenye mradi bila kuathiri hazina asili. Fork inaweza kwenda kwa njia yake yenyewe na kuwa mradi tofauti na asili, au inaweza kusawazisha mara kwa mara na mradi wa asili ili kuchangia.



- Clone:** Kuweka hazina kunamaanisha kutengeneza nakala ya ndani kwenye kompyuta yako, ambayo hukupa ufikiaji wa faili zote na historia. Hii hukuruhusu kufanya kazi kwenye mradi moja kwa moja ndani ya nchi.



- Hazina:** Nafasi ya kuhifadhi kwa mradi kwenye GitHub. Hifadhi ina faili zote za mradi na historia ya mabadiliko yote ambayo yamefanywa kwake. Ni msingi wa uhifadhi na ushirikiano kwenye GitHub.



- Tatizo:** Zana ya kufuatilia kazi na hitilafu kwenye GitHub. Masuala huruhusu matatizo ya kuripoti, kupendekeza maboresho, au kujadili vipengele vipya. Kila toleo linaweza kugawiwa, kuwekewa lebo, na kutoa maoni.


Orodha hii ni wazi sio kamili. Kuna maneno mengine mengi ya kiufundi maalum kwa Git na GitHub. Walakini, wale waliotajwa hapa ndio kuu ambao utakutana nao mara kwa mara.
