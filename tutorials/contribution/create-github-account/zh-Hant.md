## 什麼是 Git？


Git 是一個版本控制系統，專門用於管理軟體專案。Git 於 2005 年由 Linus Torvalds 創建，並迅速成為軟體開發產業中版本控制的標準。但它到底是什麼意思呢？

![git](assets/11.webp)

Git 的核心功能是讓開發人員可以追蹤項目源代碼隨時間所做的變更。這意味著程式碼的每一次變更，Git 都會記錄下專案的新版本。如果發生錯誤，或者實驗性的功能無法達到預期的效果，就有可能還原到之前的程式碼狀態，就像是一種時間機器，不過是針對檔案而言。


Git 的關鍵功能之一是分支管理。分支是一種平行線，開發人員可以獨立於專案的其他部分進行工作。這大大方便了在不影響主程式碼的情況下增加新功能或修正錯誤。一旦修改經過測試和驗證，就可以與主分支合併。


Git 的特點之一是它能以分散式的方式運作。每個開發者在自己電腦的 Hard 磁碟機上都有一份完整的專案副本，讓他們可以離線工作，並在有網際網路連線時再合併變更。這樣可以降低衝突的風險，並允許多個開發人員同時在同一個專案上工作，而不會互相踩到對方的腳趾。

最初，Git 主要是為軟體開發專案而設計。不過，它也可以用來管理內容撰寫專案。與其在程式碼上合作，不如在文字上合作。而 Plan ₿ Network 也正是採用了這種方法來管理其內容！Git 可以精確追蹤變更、進行有效的版本管理，還能讓其他貢獻者對內容進行審查和改進，因此有助於撰寫課程和教學的協作。


## 什麼是 GitHub？


GitHub 是基於我們剛剛討論過的 Git 版本控制系統的源代碼管理和託管平台。GitHub 於 2008 年推出，很快就受到歡迎，並成為全球開發人員必備的參考工具。但 GitHub 與 Git 有何不同，為何它在我們的內容製作方法中如此重要？

![github](assets/12.webp)

首先，我們必須瞭解 GitHub 是建基於 Git（我們在上一節已經討論過）。Git 是追蹤程式碼變更的工具，而 GitHub 則是託管、分享和管理這些程式碼的線上服務。


把 Git 想像成一種日誌，每個開發者都在自己的電腦上記錄自己專案中的所有變更。另一方面，GitHub 就像是一個公共圖書館，所有這些日誌都可以在這裡分享、比較和合併。


Git 和 GitHub 的根本區別在於它們的功能：Git 是每個開發人員在本機使用的工具，用來管理他們的程式碼版本，而 GitHub 則是一個線上平台，用來託管這些版本並促進合作。


GitHub 不只是代碼託管服務。它是一個協作平台，能讓開發人員有效率地合作。事實上，Plan ₿ Network 不僅使用這個平台來託管為網站提供動力的所有程式碼，也使用這個平台來託管我們感興趣的所有內容（教學、訓練、資源......）。


## 一些技術術語


在 Git 和 GitHub 上，您會遇到一些命令和功能，它們的名稱看起來可能很複雜。在最後這一部分，我會提供一些簡單的定義，讓您了解在使用 Git 和 GitHub 時可能會遇到的專業術語：



- Fetch origin:** 從遠端套件庫擷取最新資訊和變更的指令，不需要將它們與您的本機工作合併。它會使用遠端套件庫中的新分支和提交更新您的本地套件庫。



- Pull origin：** 從遠端套件庫擷取更新，並立即將其整合到您的本機分支，使其同步化的指令。這將 fetch 與 merge 的步驟結合為單一指令。
- 同步 Fork：** GitHub 上的一項功能，可讓您使用源代碼倉庫的最新變更更新專案的 Fork。這可確保您的專案副本與主開發保持同步。
- Push origin:** 用來將您的本機變更傳送到遠端套件庫的指令。



- 拉取請求：** 由貢獻者發出的請求，表示他們已將變更推送到遠端儲存庫的分支，並希望這些變更能被審核及合併到儲存庫的主分支。



- 提交：** 儲存您的變更。提交就像是您在某個特定時刻工作的即時快照，允許保留變更的歷史。每次提交都會包含一則描述性訊息，說明修改了什麼。



- 分支：** 儲存庫的平行版本，允許您在不影響主分支（通常稱為「main」或「master」）的情況下進行變更。分支有助於開發新功能和修正錯誤，而不會有破壞穩定程式碼的風險。



- 合併：** 合併包括將一個分支的變更整合到另一個分支。例如，它用來將工作分支的變更加入主分支，這樣就可以加入各種貢獻。



- Fork:** Forking 倉庫意味著在您自己的 GitHub 帳戶上建立該倉庫的副本，這樣您就可以在不影響原始倉庫的情況下進行專案工作。Fork 既可以我行我素，成為與原始專案不同的專案，也可以定期與原始專案同步，為原始專案做出貢獻。



- 複製：** 複製套件庫表示在您的電腦上製作本機複本，讓您可以存取所有檔案和歷史記錄。這可讓您直接在本機上處理專案。



- 儲存庫：** GitHub 上專案的儲存空間。儲存庫包含所有專案檔案，以及所有變更的歷史記錄。它是 GitHub 上儲存和協作的基礎。



- Issue:** 在 GitHub 上追蹤任務和 Bug 的工具。問題允許報告問題、提出改進建議或討論新功能。每個問題都可以被指派、標籤和評論。


這份清單顯然並不詳盡。Git 和 GitHub 還有許多其他特定的技術術語。不過，這裡提到的都是您會經常遇到的主要術語。

## 如何建立 GitHub 帳戶


如果您想為 PlanB Network 做出貢獻，您需要一個 GitHub 帳戶。在本教程中，我們將逐步指導您如何創建自己的帳戶、設定它並妥善保護它。



- 前往 [https://github.com/signup](https://github.com/signup)。
- 輸入您的電子郵件 Address，然後按一下 Green「繼續」按鈕：

![github](assets/1.webp)


- 選擇一個強大的密碼，然後按一下 Green `Continue`按鈕：

![github](assets/12.webp)


- 接下來，選擇您的使用者名稱。您可以透露您的真實身份，或使用假名。然後按一下 Green `Continue`按鈕：

![github](assets/3.webp)


- 完成 Captcha：

![github](assets/4.webp)


- 一封包含確認代碼的電子郵件將發送給您，您需要輸入該代碼以完成帳戶的建立：

![github](assets/5.webp)


- 如果您希望 GitHub 引導您使用某些工具，請填寫問題，或點選 `skip personalization` 跳過：

![github](assets/6.webp)


- 按一下「繼續免費」按鈕，選擇免費計劃：

![github](assets/7.webp)


- 接著您會被重新導向到您的儀表板。
- 如果您願意，可以按一下螢幕右上方的個人資料圖片，然後進入「設定」功能表，自訂您的帳戶：

![github](assets/8.webp)


- 在此部分中，您可以選擇新增個人檔案圖片、選擇名稱、自訂個人簡歷或新增個人網站連結：

![github](assets/9.webp)


- 我也建議您造訪「密碼與驗證」功能表，至少設定雙重驗證：

![github](assets/10.webp)
