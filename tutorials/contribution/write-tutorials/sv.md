Plan ₿ Network:s uppdrag är att göra ledande utbildningsresurser om Bitcoin tillgängliga på så många språk som möjligt. Allt innehåll som publiceras på webbplatsen är öppen källkod och finns på GitHub, vilket ger alla möjlighet att delta i arbetet med att berika plattformen. Bidrag kan ta många former: korrigering och korrekturläsning av befintliga texter, översättning till andra språk, uppdatering av information eller till och med skapande av nya handledningar som ännu inte finns tillgängliga på vår webbplats.


I den här handledningen tittar vi på hur man lägger till nytt innehåll, till exempel en handledning eller kurs, till Plan ₿ Network-plattformen.


## Vad är skillnaden mellan en handledning och en kurs?


De två huvudavsnitten i vår plattform är kurser och handledning. Kurser ger teoretiska utbildningsresurser, medan handledningar ger praktiskt innehåll som visar hur man använder ett specifikt verktyg (t.ex. en Hardware Wallet eller programvara) eller en viss metod (t.ex. att säkra en Mnemonic-fras eller kontrollera programvarans äkthet).


Kurser är naturligtvis längre och mer strukturerade än självstudier. De bör utforska ett specifikt ämne relaterat till Bitcoin eller dess ekosystem på djupet, med precision och i detalj.


[Upptäck Plan ₿ Network-kurser](https://planb.network/courses)

Handledningar är i allmänhet kortare. De är avsedda som steg-för-steg-guider för hur man utför en specifik Bitcoin-relaterad uppgift, med bilder som stöd.


[Upptäck Plan ₿ Network handledning](https://planb.network/tutorials)

## Hur föreslår jag en ny kurs?


Om du har en idé för en Bitcoin-kurs som ännu inte finns på plattformen och vill skriva den, kan du kontakta oss på följande Address:


contact@planb.network


I ditt meddelande ber vi dig att kort presentera dig själv och beskriva din kursidé. Om du redan har en idé om hur din kurs ska se ut kan du även bifoga de enskilda kapitlen. Vi återkommer till dig så snart som möjligt för att förklara hur du ska gå vidare och vid behov diskutera kursens upplägg.


## Hur skickar jag in en ny handledning?


För att lägga till en ny handledning till Plan ₿ Network kan du göra det direkt på GitHub genom att skicka in en Pull Request. I handledningen nedan och de medföljande handledningarna tar jag dig steg för steg genom denna process och informerar dig om de riktlinjer som ska följas för att säkerställa att din handledning är kompatibel med plattformen, samtidigt som du upprätthåller en tydlig bidragshistorik.


## 1 - Välj ett tema för din handledning


Vi letar främst efter självstudier om verktyg relaterade till Bitcoin eller dess ekosystem. Detta innehåll kan delas in i sex huvudkategorier:




- Portfölj ;
- Nod ;
- Mining ;
- Köpman;
- Exchange ;
- Konfidentialitet.


Utöver dessa ämnen som är specifikt relaterade till Bitcoin letar Plan ₿ Network också efter bidrag om ämnen som betonar individuell suveränitet, till exempel:




- Verktyg med öppen källkod ;
- IT ;
- Kryptografi ;
- Energi ;
- Matematik;
- Ekonomi;
- GÖR DET SJÄLV;
- LifeHacking...


Till exempel har vi för närvarande handledning om Tails, Nostr och GrapheneOS. Dessa verktyg är inte direkt relaterade till Bitcoin, men de är system som kan vara av intresse för oss som en del av en rörelse mot suveränitet i den digitala världen. Detta innehåll kan integreras i en underkategori i avsnittet "Övriga".


Du kan antingen utforma en handledning från grunden eller ta en handledning som tidigare har publicerats på din webbplats (förutsatt att du äger upphovsrätten) och dela den på Plan ₿ Network också, och lägga till en länk till originalartikeln.


Oavsett vad du väljer, kom ihåg att allt innehåll som publiceras på Plan ₿ Network är licensierat under en fri [CC-BY-SA](https://creativecommons.org/licenses/by-sa/4.0/) licens. Denna licens tillåter vem som helst att kopiera och eventuellt modifiera ditt innehåll, på det enda villkoret att originalkällan vederbörligen krediteras.


## 2 - Kontakta Plan ₿ Network-teamet


När du har valt tema för din handledning är nästa steg att informera oss om din avsikt att lägga till detta innehåll på plattformen. Det enklaste sättet, om du har ett Telegram-konto, är att [gå med i vår grupp] (https://t.me/PlanBNetwork_ContentBuilder).


![TUTO](assets/fr/39.webp)


Presentera dig själv kort och ange det specifika innehåll du vill skriva, samt vilket språk du vill skriva på, genom att skicka ett meddelande i kanalen "General". En teammedlem kommer sedan att skapa en fråga på GitHub som motsvarar ditt framtida bidrag.


Om du inte har ett Telegram-konto och föredrar att inte skapa ett, kan du också kontakta oss via e-post på följande Address


paolo@planb.network


## 3 - Välja rätt verktyg för att bidra


För att bidra till Plan ₿ Network har du 3 alternativ beroende på din erfarenhetsnivå med GitHub:




- Erfarna användare**: Fortsätt med dina vanliga metoder och konsultera helt enkelt den korta handledningen nedan som sammanfattar Plan ₿ Network-arkivets filstruktur, specifika krav och arbetsmetod:


https://planb.network/tutorials/contribution/content/write-tutorials-git-expert-0ce1e490-c28f-4c51-b7e0-9a6ac9728410


- Mellanliggande (GitHub Desktop)**: Om du inte är bekant med att använda Git är det första alternativet att enkelt ställa in din egen lokala miljö för att bidra till Plan ₿ Network. Detta tillvägagångssätt rekommenderas för betydande bidrag, till exempel att skriva en fullständig handledning. För att göra detta, följ steg-för-steg-handledningen nedan:


https://planb.network/tutorials/contribution/content/write-tutorials-github-desktop-intermediate-4a36a052-1000-4191-890a-9a1dc65f8957


- Nybörjare (webb-Interface)**: Du kan också välja att använda GitHubs webb-Interface direkt, utan att behöva sätta upp en komplett lokal miljö. Den här metoden kan övervägas för mindre bidrag. Men för större bidrag, som att lägga till en helt ny handledning, kan det här alternativet visa sig vara mer komplext än att sätta upp en lokal miljö. Jag kommer att förklara hur man gör detta i denna handledning:


https://planb.network/tutorials/contribution/content/write-tutorials-github-web-beginner-e64f8fed-4c0b-4225-9ebb-7fc5f1c01a79

Välj ett av dessa tre alternativ beroende på din kunskapsnivå och sätt sedan igång med att skriva din första Plan ₿ Network-handledning!
