ภารกิจของ PlanB คือการจัดหาทรัพยากรการศึกษาชั้นยอดเกี่ยวกับ Bitcoin ในหลายภาษามากที่สุด เนื้อหาทั้งหมดที่เผยแพร่บนเว็บไซต์เป็นโอเพ่นซอร์สและโฮสต์บน GitHub ซึ่งอนุญาตให้ทุกคนมีส่วนร่วมในการเพิ่มคุณค่าให้กับแพลตฟอร์ม การมีส่วนร่วมสามารถทำได้หลายรูปแบบ: การแก้ไขและพิสูจน์อักษรข้อความที่มีอยู่ การแปลเป็นภาษาอื่น การอัปเดตข้อมูล หรือการสร้างบทเรียนใหม่ที่ยังไม่มีในเว็บไซต์ของเรา


หากคุณต้องการมีส่วนร่วมใน Plan ₿ Academy คุณจะต้องใช้ GitHub เพื่อเสนอการเปลี่ยนแปลง ในการทำเช่นนี้ คุณมีสองตัวเลือก:


- มีส่วนร่วมโดยตรงผ่านอินเทอร์เฟซเว็บของ GitHub**: นี่เป็นวิธีที่ง่ายที่สุด หากคุณเป็นมือใหม่หรือหากคุณวางแผนที่จะทำการมีส่วนร่วมเพียงเล็กน้อย ตัวเลือกนี้อาจเป็นตัวเลือกที่ดีที่สุดสำหรับคุณ;
- ใช้ Git ในการมีส่วนร่วมในเครื่องของคุณ**: วิธีนี้เหมาะสมกว่าหากคุณวางแผนที่จะมีส่วนร่วมกับ Plan ₿ Academy อย่างสม่ำเสมอหรือมีนัยสำคัญ แม้ว่าการตั้งค่าสภาพแวดล้อม Git ในเครื่องของคุณอาจดูซับซ้อนในตอนแรก แต่วิธีนี้มีประสิทธิภาพมากกว่าในระยะยาว มันช่วยให้การจัดการการเปลี่ยนแปลงมีความยืดหยุ่นมากขึ้น หากคุณเป็นมือใหม่ ไม่ต้องกังวล **เราจะอธิบายกระบวนการทั้งหมดในการตั้งค่าสภาพแวดล้อมของคุณในบทเรียนนี้** (สัญญาเลยว่าคุณจะไม่ต้องพิมพ์คำสั่งใด ๆ ^^)


หากคุณไม่รู้ว่า GitHub คืออะไร หรือหากคุณต้องการเรียนรู้เพิ่มเติมเกี่ยวกับคำศัพท์ทางเทคนิคที่เกี่ยวข้องกับ Git และ GitHub ฉันขอแนะนำให้คุณอ่านบทความเบื้องต้นของเราเพื่อทำความคุ้นเคยกับแนวคิดเหล่านี้

## ขั้นตอนที่ 1: ติดตั้ง GitHub Desktop



- ไปที่ https://desktop.github.com/ เพื่อดาวน์โหลดซอฟต์แวร์ GitHub Desktop ซอฟต์แวร์นี้ช่วยให้คุณโต้ตอบกับ GitHub ได้อย่างง่ายดายโดยไม่ต้องใช้เทอร์มินัล:

![github-desktop](assets/1.webp)


- เมื่อคุณเปิดซอฟต์แวร์ครั้งแรก คุณจะถูกขอให้เชื่อมต่อบัญชี GitHub ของคุณ ในการทำเช่นนี้ ให้คลิกที่ `Sign in to GitHub.com`:

![github-desktop](assets/2.webp)


- หน้าการยืนยันตัวตนจะเปิดในเบราว์เซอร์ของคุณ ป้อนที่อยู่อีเมลและรหัสผ่านที่คุณเลือกเมื่อสร้างบัญชีของคุณ จากนั้นคลิกที่ปุ่ม `Sign in`:

![github-desktop](assets/3.webp)


- คลิกที่ `Authorize desktop` เพื่อยืนยันการเชื่อมต่อระหว่างบัญชีของคุณกับซอฟต์แวร์:

![github-desktop](assets/4.webp)


- คุณจะถูกเปลี่ยนเส้นทางไปยังซอฟต์แวร์ GitHub Desktop โดยอัตโนมัติ คลิกที่ `Finish`: ![github-desktop](assets/5.webp)
- หากคุณเพิ่งสร้างบัญชี GitHub ของคุณ คุณจะถูกเปลี่ยนเส้นทางไปยังหน้าที่ระบุว่าคุณยังไม่ได้สร้างที่เก็บใด ๆ ในตอนนี้ ให้พักซอฟต์แวร์ GitHub Desktop ไว้ก่อน; เราจะกลับมาที่มันในภายหลัง: ![github-desktop](assets/6.webp)


## ขั้นตอนที่ 2: ติดตั้ง Obsidian


มาดำเนินการติดตั้งซอฟต์แวร์สำหรับการเขียนกันเถอะ ที่นี่คุณมีตัวเลือกหลายอย่าง คุณจะต้องใช้ซอฟต์แวร์ที่รองรับการแก้ไขไฟล์ Markdown เนื่องจาก Plan ₿ Academy ใช้รูปแบบนี้สำหรับไฟล์ข้อความในที่เก็บข้อมูลของมัน


มีซอฟต์แวร์มากมายที่เชี่ยวชาญในการแก้ไขไฟล์ Markdown เช่น Typora ที่ออกแบบมาโดยเฉพาะสำหรับการเขียน แม้ว่าจะไม่ใช่ตัวเลือกที่ดีที่สุด แต่ก็สามารถเลือกใช้โปรแกรมแก้ไขโค้ด เช่น Visual Studio Code (VSC) หรือ Sublime Text ได้ อย่างไรก็ตาม ในฐานะนักเขียน ฉันชอบใช้ซอฟต์แวร์ Obsidian มากกว่า มาดูกันว่าติดตั้งและเริ่มต้นใช้งานอย่างไร



- ไปที่ https://obsidian.md/download และดาวน์โหลดซอฟต์แวร์: ![github-desktop](assets/7.webp)
- ติดตั้ง Obsidian เปิดซอฟต์แวร์ เลือกภาษาของคุณ แล้วคลิกที่ `Quick Start`: ![github-desktop](assets/8.webp)
- คุณจะมาถึงที่ซอฟต์แวร์ Obsidian ขณะนี้คุณยังไม่มีไฟล์ที่เปิดอยู่: ![github-desktop](assets/9.webp)


## ขั้นตอนที่ 3: Fork ที่ที่เก็บ Plan ₿ Academy



- ไปที่ที่เก็บข้อมูล Plan ₿ Academy ที่อยู่ต่อไปนี้: [https://github.com/PlanB-Network/bitcoin-educational-content](https://github.com/PlanB-Network/bitcoin-educational-content): ![github-desktop](assets/10.webp)
- จากหน้านี้ ให้คลิกที่ปุ่ม `Fork` ที่มุมขวาบนของหน้าต่าง: ![github-desktop](assets/11.webp)
- ในเมนูการสร้าง คุณสามารถปล่อยการตั้งค่าเริ่มต้นไว้ได้ ตรวจสอบให้แน่ใจว่าได้ทำเครื่องหมายที่ช่อง `Copy the dev branch only` แล้วคลิกที่ปุ่ม `Create fork`: ![github-desktop](assets/12.webp)
- จากนั้นคุณจะไปถึง fork ของคุณเองในที่เก็บ Plan ₿ Academy: ![github-desktop](assets/13.webp)

fork นี้ถือเป็นที่เก็บข้อมูลแยกต่างหากจากต้นฉบับ แม้ว่าปัจจุบันจะมีข้อมูลเดียวกันก็ตาม ตอนนี้คุณจะทำงานกับที่เก็บข้อมูลใหม่นี้


เราได้ทำสำเนาของที่เก็บข้อมูลต้นฉบับ Plan ₿ Academy ในทางหนึ่ง fork ของคุณ (สำเนา) และที่เก็บข้อมูลต้นฉบับจะพัฒนาอย่างอิสระจากกันและกัน ในที่เก็บข้อมูลต้นฉบับ ผู้ร่วมให้ข้อมูลอื่น ๆ อาจเพิ่มข้อมูลใหม่ ในขณะที่คุณจะดำเนินการแก้ไขของคุณเองบน fork ของคุณ

เพื่อรักษาความสอดคล้องระหว่างที่เก็บข้อมูลทั้งสองนี้ จะจำเป็นต้องซิงโครไนซ์เป็นระยะๆ เพื่อให้พวกเขาดึงข้อมูลเดียวกันได้ ในการส่งการเปลี่ยนแปลงของคุณไปยังที่เก็บข้อมูลต้นทาง คุณจะใช้สิ่งที่เรียกว่า **Pull Request** และในการรวมการเปลี่ยนแปลงจากที่เก็บข้อมูลต้นทางเข้าสู่ fork ของคุณ คุณจะใช้คำสั่ง **Sync fork** ที่มีอยู่บนอินเทอร์เฟซเว็บของ GitHub

![github-desktop](assets/14.webp)


## ขั้นตอนที่ 4: โคลน fork



- กลับไปที่ซอฟต์แวร์ GitHub Desktop ตอนนี้ fork ของคุณควรปรากฏในส่วน `Your repositories` หากคุณไม่เห็นทันที ให้ใช้ปุ่มลูกศรคู่เพื่อรีเฟรชรายการ เมื่อ fork ของคุณปรากฏขึ้น ให้คลิกเพื่อเลือก:

![github-desktop](assets/15.webp)


- จากนั้นคลิกที่ปุ่มสีน้ำเงิน: `Clone [username]/bitcoin-educational-content`:

![github-desktop](assets/16.webp)


- เก็บเส้นทางเริ่มต้นไว้ เพื่อยืนยัน ให้คลิกที่ปุ่ม `Clone` สีน้ำเงิน:

![github-desktop](assets/17.webp)


- โปรดรอในขณะที่ GitHub Desktop ทำการโคลน fork ของคุณในเครื่อง:

![github-desktop](assets/18.webp)


- หลังจากโคลนที่เก็บแล้ว ซอฟต์แวร์จะเสนอทางเลือกสองทางให้คุณ คุณต้องเลือกทางเลือกแรก: `To contribute to the parent project` การเลือกนี้จะช่วยให้คุณนำเสนอผลงานในอนาคตของคุณเป็นการมีส่วนร่วมในโครงการหลัก (`Plan ₿ Academy/bitcoin-educational-content`) และไม่ใช่เพียงการแก้ไขของ fork ส่วนตัวของคุณ (`[username]/bitcoin-educational-content`) เมื่อเลือกตัวเลือกแล้ว ให้คลิกที่ `Continue`:

![github-desktop](assets/19.webp)


- GitHub Desktop ของคุณได้รับการกำหนดค่าอย่างถูกต้องแล้ว ตอนนี้คุณสามารถเปิดซอฟต์แวร์ไว้ในพื้นหลังเพื่อติดตามการแก้ไขที่เราจะทำได้

![github-desktop](assets/20.webp)

สิ่งที่เราประสบความสำเร็จในขั้นตอนนี้คือการสร้างสำเนาท้องถิ่นของที่เก็บข้อมูลของคุณ ซึ่งโฮสต์อยู่บน GitHub เพื่อเป็นการเตือนความจำ ที่เก็บข้อมูลนี้คือ fork ของที่เก็บข้อมูลต้นทางของ Plan ₿ Academy คุณจะสามารถทำการแก้ไขสำเนาท้องถิ่นนี้ได้ เช่น การเพิ่มบทแนะนำ การแปล หรือการแก้ไข เมื่อทำการแก้ไขเหล่านี้เสร็จแล้ว คุณจะใช้คำสั่ง **Push origin** เพื่อส่งการแก้ไขท้องถิ่นของคุณไปยัง fork ที่โฮสต์บน GitHub ของคุณ


คุณยังสามารถดึงการแก้ไขจาก fork ได้ เช่น ระหว่างการซิงโครไนซ์กับที่เก็บ Plan ₿ Academy สำหรับสิ่งนี้ คุณจะใช้คำสั่ง **Fetch origin** เพื่อดาวน์โหลดการแก้ไขไปยังสำเนาท้องถิ่นของคุณ (โคลนของคุณ) จากนั้นใช้คำสั่ง **Pull origin** เพื่อรวมเข้ากับงานของคุณ วิธีนี้ช่วยให้คุณทันสมัยกับการพัฒนาล่าสุดของโครงการในขณะที่มีส่วนร่วมอย่างมีประสิทธิภาพ

## ขั้นตอนที่ 5: สร้าง Obsidian vault ใหม่



- เปิดซอฟต์แวร์ Obsidian และคลิกที่ไอคอนห้องนิรภัยเล็ก ๆ ที่มุมล่างซ้ายของหน้าต่าง:

![github-desktop](assets/22.webp)


- คลิกที่ปุ่ม `Open` เพื่อเปิดโฟลเดอร์ที่มีอยู่แล้วเป็น vault: ![github-desktop](assets/23.webp)
- ตัวสำรวจไฟล์ของคุณจะเปิดขึ้น คุณจำเป็นต้องค้นหาและเลือกโฟลเดอร์ที่มีชื่อว่า `GitHub` ซึ่งควรอยู่ในไดเรกทอรี `Documents` ของคุณท่ามกลางไฟล์ต่างๆ เส้นทางนี้สอดคล้องกับที่คุณได้ตั้งค่าไว้ในขั้นตอนที่ 4 หลังจากเลือกโฟลเดอร์แล้ว ให้ยืนยันการเลือก การสร้างคลังข้อมูลของคุณบน Obsidian จะเริ่มต้นบนหน้าต่างใหม่ของซอฟต์แวร์:


![github-desktop](assets/24.webp)

-> **Attention**, it is important not to choose the `bitcoin-educational-content` folder when creating a new vault in Obsidian. Instead, select the parent folder, `GitHub`. If you select the `bitcoin-educational-content` folder, the configuration folder `.obsidian`, containing your local Obsidian settings, will automatically be integrated into the repository. We want to avoid this, as it is not necessary to transfer your Obsidian configurations to the Plan ₿ Academy repository. An alternative is to add the `.obsidian` folder to the `.gitignore` file, but this method would also modify the `.gitignore` file of the source repository, which is not desirable.



- ทางด้านซ้ายของหน้าต่าง คุณสามารถเห็นโครงสร้างไฟล์ที่มีที่เก็บ GitHub ต่างๆ ของคุณที่ถูกโคลนมาไว้ในเครื่องแล้ว
- โดยการคลิกที่ลูกศรถัดจากชื่อโฟลเดอร์ คุณสามารถขยายเพื่อเข้าถึงโฟลเดอร์ย่อยของที่เก็บข้อมูลและเอกสารของพวกเขา:

![github-desktop](assets/25.webp)


- อย่าลืมตั้งค่า Obsidian เป็นโหมดมืด: "แสงดึงดูดแมลง" ;)


## ขั้นตอนที่ 6: ติดตั้งโปรแกรมแก้ไขโค้ด


การแก้ไขส่วนใหญ่ของคุณจะอยู่ในไฟล์รูปแบบ Markdown (`.md`) ในการแก้ไขเอกสารเหล่านี้ คุณสามารถใช้ Obsidian ซึ่งเป็นซอฟต์แวร์ที่เราได้พูดคุยกันก่อนหน้านี้ อย่างไรก็ตาม Plan ₿ Academy ใช้รูปแบบไฟล์อื่น ๆ และคุณจะต้องแก้ไขบางไฟล์เหล่านั้นด้วย


ตัวอย่างเช่น เมื่อสร้างบทแนะนำใหม่ คุณจะต้องสร้างไฟล์ YAML (`.yml`) เพื่อป้อนแท็กสำหรับบทแนะนำของคุณ ชื่อเรื่อง และรหัสครูของคุณ Obsidian ไม่ได้มีความสามารถในการแก้ไขไฟล์ประเภทนี้ ดังนั้นคุณจะต้องใช้โปรแกรมแก้ไขโค้ด


สำหรับสิ่งนี้ มีตัวเลือกหลายอย่างที่คุณสามารถใช้ได้ แม้ว่าจะสามารถใช้โปรแกรมโน้ตแพดมาตรฐานของคอมพิวเตอร์ของคุณสำหรับการปรับเปลี่ยนเหล่านี้ได้ แต่โซลูชันนี้ไม่เหมาะสำหรับงานที่ต้องการความเรียบร้อย ฉันขอแนะนำให้เลือกซอฟต์แวร์ที่ออกแบบมาเฉพาะสำหรับวัตถุประสงค์นี้ เช่น [VS Code](https://code.visualstudio.com/download) หรือ [Sublime Text](https://www.sublimetext.com/download) Sublime Text ซึ่งมีน้ำหนักเบาเป็นพิเศษ จะเพียงพอมากกว่าสำหรับความต้องการของเรา
