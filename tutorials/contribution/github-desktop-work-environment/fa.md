مأموریت PlanB ارائه منابع آموزشی درجه یک در مورد Bitcoin به بیشترین تعداد زبان ممکن است. تمام محتوای منتشر شده در سایت منبع باز است و در GitHub میزبانی می‌شود، که به هر کسی اجازه می‌دهد در غنی‌سازی این پلتفرم مشارکت کند. مشارکت‌ها می‌توانند به اشکال مختلفی باشند: تصحیح و ویرایش متون موجود، ترجمه به زبان‌های دیگر، به‌روزرسانی اطلاعات، یا ایجاد آموزش‌های جدیدی که هنوز در سایت ما موجود نیستند.


اگر می‌خواهید به شبکه PlanB کمک کنید، باید از GitHub برای پیشنهاد تغییرات استفاده کنید. برای این کار، دو گزینه دارید:


- **مستقیماً از طریق Interface وب GitHub مشارکت کنید**: این ساده‌ترین روش است. اگر شما یک مبتدی هستید یا قصد دارید فقط چند مشارکت جزئی انجام دهید، این گزینه احتمالاً بهترین گزینه برای شماست؛
- **با استفاده از Git به صورت محلی مشارکت کنید**: این روش بیشتر مناسب است اگر قصد دارید به طور منظم یا قابل توجهی به شبکه PlanB کمک کنید. اگرچه راه‌اندازی محیط Git محلی بر روی کامپیوتر شما در ابتدا ممکن است پیچیده به نظر برسد، اما این روش در درازمدت کارآمدتر است. این امکان را فراهم می‌کند تا تغییرات را به صورت انعطاف‌پذیرتری مدیریت کنید. اگر در این زمینه تازه‌کار هستید، نگران نباشید، **ما در این آموزش کل فرآیند راه‌اندازی محیط شما را توضیح می‌دهیم** (قول می‌دهیم، نیازی به تایپ هیچ خط فر

## مرحله 1: GitHub Desktop را نصب کنید



- به https://desktop.github.com/ بروید تا نرم‌افزار GitHub Desktop را دانلود کنید. این نرم‌افزار به شما اجازه می‌دهد به راحتی با GitHub تعامل داشته باشید، بدون اینکه نیاز به استفاده از ترمینال داشته باشید:

![github-desktop](assets/1.webp)


- هنگامی که برای اولین بار نرم‌افزار را اجرا می‌کنید، از شما خواسته می‌شود که حساب GitHub خود را متصل کنید. برای انجام این کار، روی `Sign in to GitHub.com` کلیک کنید:

![github-desktop](assets/2.webp)


- یک صفحه تأیید هویت در مرورگر شما باز خواهد شد. ایمیل Address و رمز عبوری که هنگام ایجاد حساب خود انتخاب کرده‌اید را وارد کنید، سپس روی دکمه `Sign in` کلیک کنید:

![github-desktop](assets/3.webp)


- برای تأیید اتصال بین حساب کاربری شما و نرم‌افزار، روی `Authorize desktop` کلیک کنید:

![github-desktop](assets/4.webp)


- شما به‌طور خودکار به نرم‌افزار GitHub Desktop هدایت خواهید شد. روی `Finish` کلیک کنید: ![github-desktop](assets/5.webp)
- اگر به‌تازگی حساب GitHub خود را ایجاد کرده‌اید، به صفحه‌ای هدایت می‌شوید که نشان می‌دهد هنوز هیچ مخزنی ایجاد نکرده‌اید. در این مرحله، نرم‌افزار GitHub Desktop را کنار بگذارید؛ بعداً به آن باز خواهیم گشت: ![github-desktop](assets/6.webp)


## مرحله 2: نصب Obsidian


بیایید به نصب نرم‌افزار نوشتن بپردازیم. در اینجا، چندین گزینه دارید. شما به نرم‌افزاری نیاز دارید که از ویرایش فایل‌های Markdown پشتیبانی کند، زیرا شبکه PlanB از این فرمت برای فایل‌های متنی در مخزن خود استفاده می‌کند.


تعداد زیادی نرم‌افزار تخصصی برای ویرایش فایل‌های Markdown وجود دارد، مانند Typora که به‌طور خاص برای نوشتن طراحی شده است. اگرچه ایده‌آل نیست، اما می‌توان یک ویرایشگر کد مانند Visual Studio Code (VSC) یا Sublime Text را نیز انتخاب کرد. با این حال، به عنوان یک نویسنده، من ترجیح می‌دهم از نرم‌افزار Obsidian استفاده کنم. بیایید با هم ببینیم چگونه آن را نصب کرده و شروع به کار کنیم.



- به https://obsidian.md/download بروید و نرم‌افزار را دانلود کنید: ![github-desktop](assets/7.webp)
- نصب Obsidian، نرم‌افزار را اجرا کنید، زبان خود را انتخاب کنید، و سپس روی `Quick Start` کلیک کنید: ![github-desktop](assets/8.webp)
- شما به نرم‌افزار Obsidian خواهید رسید. در حال حاضر، هیچ فایلی باز نیست: ![github-desktop](assets/9.webp)


## مرحله 3: Fork مخزن شبکه PlanB



- به مخزن داده‌های شبکه PlanB در Address زیر بروید: [https://github.com/PlanB-Network/Bitcoin-educational-content](https://github.com/PlanB-Network/Bitcoin-educational-content): ![github-desktop](assets/10.webp)
- از این صفحه، روی دکمه `Fork` در بالای سمت راست پنجره کلیک کنید: ![github-desktop](assets/11.webp)
- در منوی ایجاد، می‌توانید تنظیمات پیش‌فرض را حفظ کنید. اطمینان حاصل کنید که گزینه `فقط شاخه dev را کپی کن` تیک خورده باشد، سپس روی دکمه `Create Fork` کلیک کنید: ![github-desktop](assets/12.webp)
- سپس به Fork خود در مخزن شبکه PlanB خواهید رسید: ![github-desktop](assets/13.webp)

این Fork یک مخزن جداگانه از نسخه اصلی است، اگرچه در حال حاضر حاوی همان داده‌ها است. اکنون شما بر روی این مخزن جدید کار خواهید کرد.


ما به نوعی یک نسخه از مخزن منبع شبکه PlanB ایجاد کرده‌ایم. Fork شما (نسخه کپی) و مخزن اصلی اکنون به طور مستقل از یکدیگر تکامل خواهند یافت. در مخزن اصلی، سایر مشارکت‌کنندگان ممکن است داده‌های جدیدی اضافه کنند، در حالی که شما در Fork خود با اصلاحات خود ادامه خواهید داد.

برای حفظ هماهنگی بین این دو مخزن، لازم است که آنها را به صورت دوره‌ای همگام‌سازی کنید تا اطلاعات یکسانی را بازیابی کنند. برای ارسال تغییرات خود به مخزن منبع، از چیزی که به آن **Pull Request** گفته می‌شود استفاده خواهید کرد. و برای ادغام تغییرات از مخزن منبع به Fork خود، از فرمان **Sync Fork** که در وب GitHub Interface موجود است، استفاده خواهید کرد.

![github-desktop](assets/14.webp)


## مرحله 4: کلون کردن Fork



- به نرم‌افزار GitHub Desktop بازگردید. اکنون باید Fork شما در بخش `Your repositories` ظاهر شود. اگر بلافاصله آن را نمی‌بینید، از دکمه پیکان دوتایی برای تازه‌سازی لیست استفاده کنید. وقتی Fork شما ظاهر شد، روی آن کلیک کنید تا انتخاب شود:

![github-desktop](assets/15.webp)


- سپس روی دکمه آبی کلیک کنید: `Clone [username]/Bitcoin-educational-content`:

![github-desktop](assets/16.webp)


- مسیر پیش‌فرض را نگه دارید. برای تأیید، روی دکمه آبی `Clone` کلیک کنید:

![github-desktop](assets/17.webp)


- صبر کنید تا GitHub Desktop پروژه Fork شما را به صورت محلی کلون کند:

![github-desktop](assets/18.webp)


- پس از کلون کردن مخزن، نرم‌افزار دو گزینه به شما ارائه می‌دهد. شما باید گزینه اول را انتخاب کنید: `برای مشارکت در پروژه والد`. این انتخاب به شما اجازه می‌دهد تا کارهای آینده خود را به عنوان یک مشارکت در پروژه والد (`PlanB-Network/Bitcoin-educational-content`) ارائه دهید و نه صرفاً به عنوان یک تغییر در Fork شخصی خودتان (`[username]/Bitcoin-educational-content`). پس از انتخاب گزینه، روی `ادامه` کلیک کنید:

![github-desktop](assets/19.webp)


- گیت‌هاب دسکتاپ شما اکنون به‌درستی پیکربندی شده است. حالا می‌توانید نرم‌افزار را در پس‌زمینه باز نگه دارید تا تغییراتی که ایجاد خواهیم کرد را دنبال کنید.

![github-desktop](assets/20.webp)

آنچه که در این مرحله به دست آورده‌ایم، ایجاد یک نسخه محلی از مخزن شما است که بر روی GitHub میزبانی می‌شود. به عنوان یادآوری، این مخزن یک Fork از مخزن منبع PlanB Network است. شما قادر خواهید بود تا تغییراتی در این نسخه محلی ایجاد کنید، مانند افزودن آموزش‌ها، ترجمه‌ها یا اصلاحات. پس از انجام این تغییرات، از فرمان **Push origin** استفاده خواهید کرد تا تغییرات محلی خود را به Fork خود که بر روی GitHub میزبانی می‌شود، ارسال کنید.


شما همچنین می‌توانید تغییرات را از Fork بازیابی کنید، به عنوان مثال، در طول همگام‌سازی با مخزن شبکه PlanB. برای این کار، از فرمان **Fetch origin** برای دانلود تغییرات به نسخه محلی خود (کلون شما) استفاده خواهید کرد و سپس از فرمان **Pull origin** برای ادغام آن‌ها با کار خود استفاده می‌کنید. این به شما اجازه می‌دهد تا با آخرین پیشرفت‌های پروژه به‌روز بمانید و به طور مؤثر مشارکت کنید.


![github-desktop](assets/21.webp)

## مرحله ۵: یک مخزن جدید Obsidian ایجاد کنید



- نرم‌افزار Obsidian را باز کرده و روی آیکون کوچک گاوصندوق در پایین سمت چپ پنجره کلیک کنید:

![github-desktop](assets/22.webp)


- روی دکمه `Open` کلیک کنید تا یک پوشه موجود را به عنوان یک گاوصندوق باز کنید: ![github-desktop](assets/23.webp)
- مرورگر فایل شما باز خواهد شد. شما باید پوشه‌ای با عنوان `GitHub` را پیدا کرده و انتخاب کنید، که باید در دایرکتوری `Documents` شما در میان فایل‌هایتان باشد. این مسیر با مسیری که در مرحله ۴ ایجاد کردید مطابقت دارد. پس از انتخاب پوشه، انتخاب آن را تأیید کنید. ایجاد خزانه شما در Obsidian سپس در یک صفحه جدید از نرم‌افزار آغاز خواهد شد:


![github-desktop](assets/24.webp)

-> **توجه**، مهم است که هنگام ایجاد یک خزانه جدید در Obsidian، پوشه `Bitcoin-educational-content` را انتخاب نکنید. در عوض، پوشه والد، `GitHub` را انتخاب کنید. اگر پوشه `Bitcoin-educational-content` را انتخاب کنید، پوشه پیکربندی `.obsidian` که شامل تنظیمات محلی Obsidian شما است، به طور خودکار در مخزن ادغام خواهد شد. ما می‌خواهیم از این کار اجتناب کنیم، زیرا انتقال تنظیمات Obsidian شما به مخزن PlanB Network ضروری نیست. یک جایگزین این است که پوشه `.obsidian` را به فایل `.gitignore` اضافه کنید، اما این روش نیز فایل `.gitignore` مخزن منبع را تغییر می‌دهد که مطلوب نیست.



- در سمت چپ پنجره، می‌توانید درخت فایل را با مخازن مختلف GitHub خود که به صورت محلی کلون شده‌اند، مشاهده کنید.
- با کلیک بر روی فلش‌های کنار نام پوشه‌ها، می‌توانید آن‌ها را گسترش دهید تا به زیرپوشه‌های مخازن و اسناد آن‌ها دسترسی پیدا کنید:

![github-desktop](assets/25.webp)


- فراموش نکنید که حالت تاریک را در Obsidian فعال کنید: "نور حشرات را جذب می‌کند" ;)


## مرحله 6: نصب یک ویرایشگر کد


بیشتر تغییرات شما بر روی فایل‌هایی با فرمت Markdown (`.md`) خواهد بود. برای ویرایش این اسناد، می‌توانید از Obsidian، نرم‌افزاری که قبلاً درباره آن صحبت کردیم، استفاده کنید. با این حال، شبکه PlanB از فرمت‌های فایل دیگری استفاده می‌کند و شما نیاز خواهید داشت که برخی از آن‌ها را نیز ویرایش کنید.


به عنوان مثال، هنگام ایجاد یک آموزش جدید، شما نیاز به ایجاد یک فایل YAML (`.yml`) دارید تا برچسب‌ها برای آموزش خود، عنوان آن و شناسه معلم خود را وارد کنید. Obsidian امکان ویرایش این نوع فایل‌ها را ارائه نمی‌دهد، بنابراین شما به یک ویرایشگر کد نیاز خواهید داشت.


برای این کار، چندین گزینه در دسترس شماست. اگرچه می‌توان از نوت‌پد استاندارد کامپیوتر شما برای این تغییرات استفاده کرد، اما این راه‌حل برای کارهای مرتب ایده‌آل نیست. من توصیه می‌کنم نرم‌افزاری را انتخاب کنید که به‌طور خاص برای این منظور طراحی شده باشد، مانند [VS Code](https://code.visualstudio.com/download) یا [Sublime Text](https://www.sublimetext.com/download). Sublime Text، به‌ویژه به دلیل سبک بودن، برای نیازهای ما بیش از حد کافی خواهد بود.


- یکی از این نرم‌افزارها را نصب کنید و برای تغییرات آینده خود کنار بگذارید. ![github-desktop](assets/26.webp)
