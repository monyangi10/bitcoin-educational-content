प्लानबी का मिशन Bitcoin पर यथासंभव अधिक से अधिक भाषाओं में शीर्ष स्तरीय शैक्षिक संसाधन उपलब्ध कराना है। साइट पर प्रकाशित सभी सामग्री ओपन-सोर्स है और GitHub पर होस्ट की गई है, जो किसी को भी प्लेटफ़ॉर्म को समृद्ध बनाने में भाग लेने की अनुमति देती है। योगदान विभिन्न रूपों में हो सकते हैं: मौजूदा पाठों को सही करना और प्रूफ़रीडिंग करना, अन्य भाषाओं में अनुवाद करना, जानकारी अपडेट करना, या हमारी साइट पर अभी तक उपलब्ध नहीं होने वाले नए ट्यूटोरियल बनाना।


यदि आप PlanB नेटवर्क में योगदान देना चाहते हैं, तो आपको परिवर्तनों का प्रस्ताव देने के लिए GitHub का उपयोग करना होगा। ऐसा करने के लिए, आपके पास दो विकल्प हैं:


- GitHub के वेब **Interface** के माध्यम से सीधे योगदान करें: यह सबसे सरल तरीका है। यदि आप एक शुरुआती हैं या यदि आप केवल कुछ छोटे योगदान करने की योजना बनाते हैं, तो यह विकल्प संभवतः आपके लिए सबसे अच्छा है;
- **Git का उपयोग करके स्थानीय रूप से योगदान करें**: यदि आप PlanB नेटवर्क में नियमित या महत्वपूर्ण योगदान देने की योजना बनाते हैं तो यह विधि अधिक उपयुक्त है। हालाँकि आपके कंप्यूटर पर अपना स्थानीय Git वातावरण सेट करना पहली बार में जटिल लग सकता है, लेकिन यह दृष्टिकोण लंबे समय में अधिक कुशल है। यह परिवर्तनों के अधिक लचीले प्रबंधन की अनुमति देता है। यदि आप इसके लिए नए हैं, तो चिंता न करें, **हम इस ट्यूटोरियल में आपके वातावरण को सेट करने की पूरी प्रक्रिया को समझाते हैं**


यदि आपको नहीं पता कि GitHub क्या है, या यदि आप Git और GitHub से संबंधित तकनीकी शब्दों के बारे में अधिक जानना चाहते हैं, तो मेरा सुझाव है कि आप इन अवधारणाओं से परिचित होने के लिए हमारा परिचयात्मक लेख पढ़ें।


https://planb.network/tutorials/contribution/others/create-github-account-a75fc39d-f0d0-44dc-9cd5-cd94aee0c07c




- शुरू करने के लिए, आपको निश्चित रूप से एक GitHub खाते की आवश्यकता होगी। यदि आपके पास पहले से ही एक है, तो आप लॉग इन कर सकते हैं, अन्यथा, आप एक नया खाता बनाने के लिए हमारे ट्यूटोरियल का उपयोग कर सकते हैं।

## चरण 1: GitHub डेस्कटॉप स्थापित करें



- GitHub डेस्कटॉप सॉफ़्टवेयर डाउनलोड करने के लिए https://desktop.github.com/ पर जाएँ। यह सॉफ़्टवेयर आपको टर्मिनल का उपयोग किए बिना, GitHub के साथ आसानी से इंटरैक्ट करने की अनुमति देता है:

![github-desktop](assets/1.webp)


- जब आप पहली बार सॉफ़्टवेयर लॉन्च करेंगे, तो आपसे अपना GitHub खाता कनेक्ट करने के लिए कहा जाएगा। ऐसा करने के लिए, `GitHub.com पर साइन इन करें` पर क्लिक करें:

![github-desktop](assets/2.webp)


- आपके ब्राउज़र में एक प्रमाणीकरण पृष्ठ खुलेगा। अपना ईमेल Address और अपना खाता बनाते समय चुना गया पासवर्ड दर्ज करें, फिर `साइन इन` बटन पर क्लिक करें:

![github-desktop](assets/3.webp)


- अपने खाते और सॉफ्टवेयर के बीच कनेक्शन की पुष्टि करने के लिए `डेस्कटॉप अधिकृत करें` पर क्लिक करें:

![github-desktop](assets/4.webp)


- आपको स्वचालित रूप से GitHub डेस्कटॉप सॉफ़्टवेयर पर रीडायरेक्ट कर दिया जाएगा। `Finish` पर क्लिक करें: ![github-desktop](assets/5.webp)
- यदि आपने अभी-अभी अपना GitHub खाता बनाया है, तो आपको एक ऐसे पृष्ठ पर पुनः निर्देशित किया जाएगा जो यह दर्शाता है कि आपने अभी तक कोई रिपॉजिटरी नहीं बनाई है। इस बिंदु पर, GitHub डेस्कटॉप सॉफ़्टवेयर को अलग रखें; हम बाद में इस पर वापस आएंगे: ![github-desktop](assets/6.webp)


## चरण 2: ओब्सीडियन स्थापित करें


चलिए लेखन सॉफ़्टवेयर स्थापित करने के लिए आगे बढ़ते हैं। यहाँ, आपके पास कई विकल्प हैं। आपको ऐसे सॉफ़्टवेयर की आवश्यकता होगी जो मार्कडाउन फ़ाइलों को संपादित करने का समर्थन करता हो, क्योंकि प्लानबी नेटवर्क अपने रिपॉजिटरी में टेक्स्ट फ़ाइलों के लिए इस प्रारूप का उपयोग करता है।


मार्कडाउन फ़ाइलों को संपादित करने में विशेषज्ञता रखने वाले कई सॉफ़्टवेयर हैं, जैसे कि टाइपोरा, जिसे विशेष रूप से लिखने के लिए डिज़ाइन किया गया है। हालाँकि यह आदर्श नहीं है, लेकिन विज़ुअल स्टूडियो कोड (VSC) या सबलाइम टेक्स्ट जैसे कोड एडिटर को चुनना भी संभव है। हालाँकि, एक लेखक के रूप में, मैं ओब्सीडियन सॉफ़्टवेयर का उपयोग करना पसंद करता हूँ। आइए एक साथ देखें कि इसे कैसे इंस्टॉल करें और इसके साथ शुरुआत करें।



- https://obsidian.md/download पर जाएं और सॉफ्टवेयर डाउनलोड करें: ![github-desktop](assets/7.webp)
- ओब्सीडियन स्थापित करें, सॉफ्टवेयर लॉन्च करें, अपनी भाषा चुनें, और फिर `त्वरित प्रारंभ` पर क्लिक करें: ![github-desktop](assets/8.webp)
- आप ओब्सीडियन सॉफ्टवेयर पर पहुंच जाएंगे। अभी के लिए, आपके पास कोई फ़ाइल खुली नहीं है: ![github-desktop](assets/9.webp)


## चरण 3: Fork प्लानबी नेटवर्क रिपोजिटरी



- निम्नलिखित Address पर PlanB नेटवर्क डेटा रिपॉजिटरी पर जाएं: [https://github.com/PlanB-Network/Bitcoin-educational-content](https://github.com/PlanB-Network/Bitcoin-educational-content): ![github-desktop](assets/10.webp)
- इस पृष्ठ से, विंडो के शीर्ष दाईं ओर स्थित `Fork` बटन पर क्लिक करें: ![github-desktop](assets/11.webp)
- क्रिएशन मेनू में, आप डिफ़ॉल्ट सेटिंग को छोड़ सकते हैं। सुनिश्चित करें कि `केवल डेव शाखा की प्रतिलिपि बनाएँ` बॉक्स चेक किया गया है, फिर `Fork बनाएँ` बटन पर क्लिक करें: ![github-desktop](assets/12.webp)
- फिर आप PlanB नेटवर्क रिपोजिटरी के अपने Fork पर पहुंचेंगे: ![github-desktop](assets/13.webp)

यह Fork मूल से अलग रिपॉजिटरी है, हालाँकि इसमें वर्तमान में वही डेटा है। अब आप इस नए रिपॉजिटरी पर काम करेंगे।


हमने एक तरह से PlanB Network स्रोत रिपोजिटरी की एक प्रति बना ली है। आपका Fork (प्रतिलिपि) और मूल रिपोजिटरी अब एक दूसरे से स्वतंत्र रूप से विकसित होंगे। मूल रिपोजिटरी पर, अन्य योगदानकर्ता नया डेटा जोड़ सकते हैं, जबकि आप अपने Fork पर अपने स्वयं के संशोधनों के साथ आगे बढ़ेंगे।

इन दो रिपॉजिटरी के बीच एकरूपता बनाए रखने के लिए, उन्हें समय-समय पर सिंक्रोनाइज़ करना आवश्यक होगा ताकि वे एक ही जानकारी प्राप्त कर सकें। अपने परिवर्तनों को स्रोत रिपॉजिटरी में भेजने के लिए, आप **पुल रिक्वेस्ट** नामक चीज़ का उपयोग करेंगे। और स्रोत रिपॉजिटरी से अपने Fork में परिवर्तनों को एकीकृत करने के लिए, आप GitHub वेब Interface पर उपलब्ध **Sync Fork** कमांड का उपयोग करेंगे।

![github-desktop](assets/14.webp)


## चरण 4: Fork का क्लोन बनाएं



- GitHub डेस्कटॉप सॉफ़्टवेयर पर वापस जाएँ। अब तक, आपका Fork `आपकी रिपॉजिटरी` अनुभाग में दिखाई देना चाहिए। यदि आप इसे तुरंत नहीं देखते हैं, तो सूची को ताज़ा करने के लिए डबल एरो बटन का उपयोग करें। जब आपका Fork दिखाई दे, तो इसे चुनने के लिए उस पर क्लिक करें:

![github-desktop](assets/15.webp)


- फिर नीले बटन पर क्लिक करें: `Clone [username]/Bitcoin-educational-content`:

![github-desktop](assets/16.webp)


- डिफ़ॉल्ट पथ रखें। पुष्टि करने के लिए, नीले `क्लोन` बटन पर क्लिक करें:

![github-desktop](assets/17.webp)


- GitHub डेस्कटॉप द्वारा आपके Fork को स्थानीय रूप से क्लोन किए जाने तक प्रतीक्षा करें:

![github-desktop](assets/18.webp)


- रिपॉजिटरी को क्लोन करने के बाद, सॉफ्टवेयर आपको दो विकल्प प्रदान करता है। आपको पहला विकल्प चुनना होगा: `मूल परियोजना में योगदान करने के लिए`। यह विकल्प आपको अपने भविष्य के काम को मूल परियोजना (`PlanB-Network/Bitcoin-educational-content`) में योगदान के रूप में प्रस्तुत करने की अनुमति देगा, न कि केवल आपके व्यक्तिगत Fork (`[उपयोगकर्ता नाम]/Bitcoin-educational-content`) के संशोधन के रूप में। एक बार विकल्प चुनने के बाद, `जारी रखें` पर क्लिक करें:

![github-desktop](assets/19.webp)


- आपका GitHub डेस्कटॉप अब सही तरीके से कॉन्फ़िगर हो गया है। अब, आप हमारे द्वारा किए जाने वाले संशोधनों का पालन करने के लिए सॉफ़्टवेयर को पृष्ठभूमि में खुला छोड़ सकते हैं।

![github-desktop](assets/20.webp)

इस चरण में हमने जो हासिल किया है वह आपके रिपॉजिटरी की एक स्थानीय प्रति का निर्माण है, जिसे GitHub पर होस्ट किया गया है। याद दिला दें कि यह रिपॉजिटरी PlanB Network के स्रोत रिपॉजिटरी का Fork है। आप इस स्थानीय प्रति में संशोधन कर सकेंगे, जैसे ट्यूटोरियल, अनुवाद या सुधार जोड़ना। एक बार ये संशोधन हो जाने के बाद, आप अपने स्थानीय संशोधनों को GitHub पर होस्ट किए गए अपने Fork पर भेजने के लिए **Push origin** कमांड का उपयोग करेंगे।


आप Fork से संशोधनों को भी पुनः प्राप्त कर सकते हैं, उदाहरण के लिए, PlanB नेटवर्क रिपॉजिटरी के साथ सिंक्रोनाइज़ेशन के दौरान। इसके लिए, आप संशोधनों को अपनी स्थानीय प्रतिलिपि (अपने क्लोन) में डाउनलोड करने के लिए **Fetch origin** कमांड का उपयोग करेंगे, और फिर उन्हें अपने काम के साथ मर्ज करने के लिए **Pull origin** कमांड का उपयोग करेंगे। यह आपको प्रभावी रूप से योगदान करते हुए परियोजना के नवीनतम विकास के साथ अद्यतित रहने की अनुमति देता है।


![github-desktop](assets/21.webp)

## चरण 5: एक नया ओब्सीडियन वॉल्ट बनाएं



- ओब्सीडियन सॉफ्टवेयर खोलें और विंडो के नीचे बाईं ओर छोटे वॉल्ट आइकन पर क्लिक करें:

![github-desktop](assets/22.webp)


- किसी मौजूदा फ़ोल्डर को वॉल्ट के रूप में खोलने के लिए `Open` बटन पर क्लिक करें: ![github-desktop](assets/23.webp)
- आपका फ़ाइल एक्सप्लोरर खुल जाएगा। आपको `GitHub` नामक फ़ोल्डर ढूँढ़ना और चुनना होगा, जो आपकी फ़ाइलों के बीच आपकी `Documents` निर्देशिका में होना चाहिए। यह पथ चरण 4 के दौरान आपके द्वारा स्थापित पथ से मेल खाता है। फ़ोल्डर चुनने के बाद, इसके चयन की पुष्टि करें। फिर ओब्सीडियन पर आपके वॉल्ट का निर्माण सॉफ़्टवेयर के एक नए पृष्ठ पर लॉन्च होगा:


![github-desktop](assets/24.webp)

-> **ध्यान दें**, ओब्सीडियन में नया वॉल्ट बनाते समय `Bitcoin-educational-content` फ़ोल्डर का चयन न करना महत्वपूर्ण है। इसके बजाय, पैरेंट फ़ोल्डर, `GitHub` का चयन करें। यदि आप `Bitcoin-educational-content` फ़ोल्डर का चयन करते हैं, तो कॉन्फ़िगरेशन फ़ोल्डर `.obsidian`, जिसमें आपकी स्थानीय ओब्सीडियन सेटिंग्स शामिल हैं, स्वचालित रूप से रिपॉजिटरी में एकीकृत हो जाएगा। हम इससे बचना चाहते हैं, क्योंकि आपके ओब्सीडियन कॉन्फ़िगरेशन को प्लानबी नेटवर्क रिपॉजिटरी में स्थानांतरित करना आवश्यक नहीं है। एक विकल्प `.obsidian` फ़ोल्डर को `.gitignore` फ़ाइल में जोड़ना है, लेकिन यह विधि स्रोत रिपॉजिटरी की `.gitignore` फ़ाइल को भी संशोधित करेगी, जो वांछनीय नहीं है।

## चरण 6: कोड संपादक स्थापित करें


आपके अधिकांश संशोधन मार्कडाउन प्रारूप (`.md`) में फ़ाइलों पर होंगे। इन दस्तावेज़ों को संपादित करने के लिए, आप ओब्सीडियन का उपयोग कर सकते हैं, जिस सॉफ़्टवेयर पर हमने पहले चर्चा की थी। हालाँकि, प्लानबी नेटवर्क अन्य फ़ाइल प्रारूपों का उपयोग करता है, और आपको उनमें से कुछ को संशोधित करने की आवश्यकता होगी।


उदाहरण के लिए, एक नया ट्यूटोरियल बनाते समय, आपको अपने ट्यूटोरियल, उसके शीर्षक और अपने शिक्षक आईडी के लिए टैग दर्ज करने के लिए एक YAML (`.yml`) फ़ाइल बनाने की आवश्यकता होगी। ओब्सीडियन इस प्रकार की फ़ाइलों को संशोधित करने की संभावना प्रदान नहीं करता है, इसलिए आपको एक कोड संपादक की आवश्यकता होगी।


इसके लिए आपके पास कई विकल्प उपलब्ध हैं। हालाँकि आपके कंप्यूटर के मानक नोटपैड का उपयोग इन संशोधनों के लिए किया जा सकता है, लेकिन यह समाधान साफ-सुथरे काम के लिए आदर्श नहीं है। मैं इस उद्देश्य के लिए विशेष रूप से डिज़ाइन किए गए सॉफ़्टवेयर को चुनने की सलाह देता हूँ, जैसे कि [VS Code](https://code.visualstudio.com/download) या [Sublime Text](https://www.sublimetext.com/download)। Sublime Text, विशेष रूप से हल्का होने के कारण, हमारी ज़रूरतों के लिए पर्याप्त से अधिक होगा।


- इनमें से किसी एक सॉफ्टवेयर प्रोग्राम को इंस्टॉल करें, और इसे अपने भविष्य के संशोधनों के लिए अलग रखें। ![github-desktop](assets/26.webp)
