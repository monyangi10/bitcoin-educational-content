![cover](assets/cover.webp)


Ако възнамерявате да допринесете за Plan ₿ Academy, като напишете нов урок или курс, ще ви е необходим профил на учител. Този профил ще ви позволи да получавате съответните кредити за съдържанието, което предоставяте на платформата.


Тези от вас, които вече са участвали в създаването на образователно съдържание в Plan ₿ Academy, вероятно вече имат профил на учител. Можете да го намерите в папката `/professors` [в нашето хранилище в GitHub](https://github.com/PlanB-Network/Bitcoin-educational-content/tree/dev/professors). Ако профилът ви вече съществува, намерете потребителското си име във файла `professor.yml`.


За да направите промени в профила си, отидете на раздела "Редактиране на профила на учителя" в края на този урок.


## Добавяне на нов учител с нашия софтуер


Най-лесният начин за създаване на профил на учител в Plan ₿ Academy е да използвате нашия интегриран инструмент Python. Ето как работи той.


### 1 - Конфигуриране на локалната среда


Трябва да разполагате със собствен Fork от [хранилище Plan ₿ Academy в GitHub](https://github.com/PlanB-Network/Bitcoin-educational-content).


Синхронизирайте главния клон (`dev`) на вашия Fork с хранилището на изходния код.


Актуализирайте локалния си клонинг.


```bash
# Cloner votre fork (si ce n'est pas déjà fait)
git clone https://github.com/<username>/bitcoin-educational-content.git
cd bitcoin-educational-content
# Ajouter le dépôt source en tant que remote upstream
git remote add upstream https://github.com/PlanB-Network/bitcoin-educational-content.git
# Récupérer les dernières modifications depuis le dépôt source
git fetch upstream
# Se positionner sur la branche principale 'dev'
git checkout dev
# Fusionner les modifications de la branche 'dev' du dépôt source dans votre fork
git merge upstream/dev
# Pousser les mises à jour vers votre fork sur GitHub
git push origin dev
```


### 2 - Създаване на нов клон


Уверете се, че сте в клона `dev`. Създайте нов клон с описателно име (напр. `add-professor-loic-morel`).


Публикувайте този клон във вашия Fork онлайн.


```bash
# Assurez-vous d’être sur la branche 'dev'
git checkout dev
# Créez une nouvelle branche avec un nom descriptif
git checkout -b add-professor-loic-morel
# Publiez cette branche sur votre fork en ligne
git push -u origin add-professor-loic-morel
```


### 3 - Създаване на профил на учителя


Отидете в папката `scripts/tutorial-related/data-creator/` на вашия локален клонинг. Уверете се, че сте инсталирали всички зависимости, необходими за софтуера, като преди това сте инсталирали Python:


```bash
pip install -r requirements.txt
```


След това стартирайте софтуера с командата:


```bash
python3 main.py
```


След като се озовете на началната страница, въведете локалния път до вашия клонинг на хранилище, езика, на който пишете, и идентификатора си в GitHub. Ако създавате този профил за някой друг и вече имате профил на професор, въведете идентификатора си в полето "*Plan ₿ Academy Professor's ID*". Ако създавате свой собствен профил, все още няма да имате идентификатор на професор, тъй като сте в процес на създаване на такъв, затова оставете това поле празно.


След това кликнете върху бутона "*Нов професор*".


![Image](assets/fr/01.webp)


Попълнете необходимата информация (моля, имайте предвид, че цялата тази информация ще бъде публична в нашата платформа, както и в GitHub):




- Име на досието на учителя (използвайте име и фамилия или псевдоним, с малки букви) ;
- Вашето име или прякор ;
- Вашият уебсайт и профил X (по избор) ;
- Адрес на светкавица за получаване на дарения от читатели (по избор) ;
- Изберете 2 или 3 етикета от списъка;
- Щракнете върху "*Избор на изображение*", за да изберете профилно изображение от локалните папки (за изображението може да се използва всяко име и формат и софтуерът ще го адаптира автоматично. Просто се уверете, че изображението е квадратно);
- Напишете кратко описание на профила си.


Завършете създаването, като кликнете върху "*Създаване на професор*". Това автоматично ще generate всички файлове, необходими за вашия профил.


![Image](assets/fr/02.webp)


Запазете промените си локално, като създадете коммит с обяснително съобщение. Изпратете промените във вашия Fork GitHub.


```bash
# Créez un commit avec un message descriptif
git commit -m "*new professor Loïc Morel*"
# Poussez vos modifications sur votre fork
git push origin add-professor-loic-morel
```


След като приключите, създайте заявка за изтегляне (PR) в GitHub, за да предложите интегрирането на вашите модификации. Добавете заглавие и кратко описание към PR.


### 4 - Корекция и обединяване


Изчакайте потвърждение или обратна връзка от администратор. Ако е необходимо, направете корекции и изпратете нови корекции.


```bash
# Créez un commit décrivant les corrections apportées
git commit -m "*Corrections suite à la revue du tutoriel green-wallet*"
# Poussez les corrections sur votre fork
git push origin add-professor-loic-morel
```


След като PR-ът бъде обединен, можете да изтриете работния клон.


## Промяна на вашия профил на учител


Ако сте усвоили използването на Git, модифицирайте профила на учителя си, като създадете нов клон и редактирате съответния файл директно в съществуващата папка. Промените могат да бъдат направени или във файла `professor.yml`, или във файла markdown, в зависимост от информацията, която трябва да се коригира. След като направите локалните си промени, ги изпратете в Fork и изпратете PR.


За начинаещите препоръчвам да направят модификацията директно чрез Interface в GitHub. Уверете се, че имате акаунт в GitHub. Ако не знаете как да създадете такъв, следвайте този урок:


https://planb.academy/tutorials/contribution/others/create-github-account-a75fc39d-f0d0-44dc-9cd5-cd94aee0c07c
Отидете в [хранилището на Plan ₿ Academy в GitHub, посветено на данните](https://github.com/PlanB-Network/Bitcoin-educational-content/graphs/contributors).


![Image](assets/fr/03.webp)


Кликнете върху папката "*професори*", след което отидете в личната си папка.


![Image](assets/fr/04.webp)


За да промените метаданните на профила си, като например адрес на Lightning, име или връзки, изберете файла "*professor.yml*". За да промените описанието си, кликнете върху YAML файла за вашия език (например "*en.yml*" или "*fr.yml*").


Ако променяте описанието си, не забравяйте да премахнете всички остарели преводи. След това можете или да се погрижите за превода на описанието си на другите езици с помощта на LLM, или да оставите само описанието на родния си език и да споменете в заявката си за изтегляне, че описанието ви изисква превод от нашия екип.


![Image](assets/fr/05.webp)


След като се озовете върху файла, който искате да промените, щракнете върху иконата на молив.


![Image](assets/fr/06.webp)


Ако все още нямате Fork от хранилището Plan ₿ Academy, GitHub ще ви предложи да създадете такъв. Щракнете върху "*Fork this repository*".


![Image](assets/fr/07.webp)


Направете желаните промени във файла. Когато приключите, щракнете върху "*Commit changes*".


![Image](assets/fr/08.webp)


Въведете съобщение, описващо вашата промяна, след което изберете "*Предлагане на промени*".


![Image](assets/fr/09.webp)


Ще бъде показано обобщение на промените. Ако желаете да направите допълнителни промени в профила си, можете да се върнете в папките и да извършите допълнителни промени. Когато приключите, щракнете върху "*Създаване на заявка за изтегляне*".


Заявката за изтегляне е заявка за интегриране на промени от вашия клон в основния клон на хранилището Plan ₿ Academy, което позволява преглед и обсъждане на промените, преди те да бъдат обединени.


![Image](assets/fr/10.webp)


Уверете се, че в началото на Interface работният ви клон е слят с клона `dev` на хранилището Plan ₿ Academy (който е основният клон).


Въведете заглавие, което накратко обобщава промените, които искате да слеете с изходното хранилище. Добавете кратък коментар, описващ тези промени, след което щракнете върху зеления бутон "*Създай заявка за изтегляне*", за да потвърдите заявката за изтегляне:


![Image](assets/fr/11.webp)


След това вашият PR ще бъде видим в раздела "*Заявка за изтегляне*" на главното хранилище Plan ₿ Academy. Всичко, което трябва да направите сега, е да изчакате администратор да обедини вашата промяна.


![Image](assets/fr/12.webp)


Ако срещнете някакви технически трудности при подаването на промяната си, моля, не се колебайте да поискате помощ в [нашата група Telegram, посветена на приноса](https://t.me/PlanBNetwork_ContentBuilder). Много ви благодаря!
