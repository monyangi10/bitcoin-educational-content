## 校正とは何か？


校正とは、文法、スペル、句読点、書式などの誤りを特定し、修正することである。出版や投稿の前に、文章が明確で一貫性があり、間違いがないことを確認する。


この種の作業を行う場合、原語（ENまたはFR）の意味に従うことが重要ですが、最終的な言語のテキストは、ネイティブ・スピーカーにとって可能な限り流暢なものにしてください。


翻訳・校正はEDUCATIONであることを常に忘れないでください！


実際、私たちの共通の目標は、Bitcoinについてできるだけ多くの人々に知ってもらうことであり、そのためには、彼らが読む資料がスムーズで明快であることが基本である。

その意味で、Plan ₿ Networkに貢献するすべての人は教育者である！


## Plan ₿ Networkで校正する前の最初のステップ


新しい校正タスクを開始する前に、[Telegram グループ](https://t.me/PlanBNetwork_ContentBuilder) でお知らせいただくか、Plan ₿ Networkコーディネーターにお知らせください。コーディネーターが専用の [issue](https://github.com/orgs/Plan ₿ Academy/projects/3) を開設します。issueのリンクを受け取ったら、**その内容の校正タスクを開始**するとコメントしてください。


このシステムは、コーディネーターがレポ内の進捗状況を把握するのに役立ち、校正者によってコンテンツが「クレーム」され、他の誰かによる重複作業を防ぐことができる。

issue自体には、チェックすべきコンテンツにリダイレクトするリンクがあります。それをクリックしてもいいし、自分のフォークしたリポジトリに戻ってそこから直接作業することもできる。では、その方法を見てみよう！


まず第一に、**常に「dev」ブランチのレポを同期することを忘れないでください**。こうすることで、何らかの作業を始める前にコンテンツが常に更新され、古いものと新しいものの間でコンフリクトが発生しなくなります。必ず「Forkを同期」と「ブランチを更新」をクリックしてください。



![REVIEW](assets/en/1.webp)



同期に成功したら、この[チュートリアル](https://planb.academy/tutorials/contribution/content/proofreading-review-tutorial-28236c98-23b2-4efd-9563-953f08707017)にあるように、目的のコンテンツに直接アクセスして新しいブランチにコミットできます。そうでなければ、下図のように "Branches" をクリックして新しいブランチを開き、そこで作業することができます。



![REVIEW](assets/en/2.webp)



この新しいページでは、「Your Branches（あなたのブランチ）」というタイトルの下に、すでに開いたブランチがすべて表示されます。このセクションはとても便利です。新しいブランチを開きたい場合は、ページの右上にある "新しいブランチ "をクリックしてください。



![REVIEW](assets/en/3.webp)



すると、新しいブランチの名前を入力するポップアップが表示されます。下の例では、「BTC101-FR」としました。こうすることで、この特定のブランチをBTC101フランス語コースの校正に使用する必要があることを常に覚えておくことができます。


新しい仕事を始める必要があるときは、いつでも新しいブランチを開くようにしてください。



![REVIEW](assets/en/4.webp)



新しいブランチを作成したら、前のページの「Your Branches」からそのブランチをクリックし、特定のコンテンツに関連する *.md* ファイル（私の場合、"courses" -> "BTC101" -> "fr.md" をクリックします）の作業を開始します。特定のファイルに関連するすべてのコミットは、同じブランチ内でコミット（保存）する必要があります。



## 原語か翻訳か？


コンテンツの校正を行う際には、必ずオリジナルの英語（またはフランス語）**版**を確認することが重要です。私たちはAI言語ツールを使って翻訳しているため、ターゲット言語でのレンダリングが流動的であったり、最終的な読者にとって理解しやすいものではないかもしれないことに注意してください。


したがって、必要であれば、自由に文章を調整し、文章を修正してください。私たちの目的は流動性を高めることですが、常に原文の意味に従います。特定の単語の扱いについて疑問がある場合は、必ず翻訳コーディネーターにお尋ねください。


LLMツールは、Lightning Networkのように、Bitcoinに関連する単語を文字通りに翻訳することがあります。特に、非常に専門的な単語を扱う場合はそうです。このような場合、あなたの言語規則がすべての単語を翻訳することを強制しない限り、より明確にするために、あなたのターゲット言語で元の英語の単語を維持することをお勧めします。


この2番目のケースでは、**Bitcoinのコミュニティで誰かがその単語をすでに訳していて**、現在広く使われていないかどうか、必ず調べてください。



- 一つの解決策は、**ターゲット言語の[BitcoinWiki](https://en.Bitcoin.it/wiki/Main_Page)**で、その単語が翻訳されているかどうかをチェックすることです。もし翻訳されていなければ、その単語は英語のままにしておきます。



- いずれにせよ、私のアドバイスとしては、EN (LANG)のスキームに従って、丸括弧の中に目標言語の対応する意味を加えながら、**それにもかかわらず**というEN語を挿入すること、あるいはその逆をすることである。例Address（indirizzo）、またはindirizzo（Address）。



- もう一つの良い解決策は、ENオリジナルの単語/フレーズをそのままにして、planb.networkの[glossary](https://planb.academy/en/resources/glossary)にリダイレクトするハイパーリンクを作成することです。これを行うには、下の例のように、単語/フレーズを角括弧の中に、リンクを丸括弧の中に挿入する必要があります：


```
[UTXO](https://planb.academy/resources/glossary/utxo)
```


最終結果（下の画像）では、リンク全体が視覚化されず、単語がクリック可能になる。



![REVIEW](assets/en/5.webp)



ウェブサイトから取得する用語集のリンクには、「network」という単語の後に言語コードが含まれていることにご注意ください（例：``https://planb.academy/en/resources/glossary/utxo``->ここでは言語コード「en」を読むことができます）。この場合、上のボックスのように、**リンクから言語コードを削除してください**。そうすれば、システムは自動的に読者を指定された言語に誘導します。


リポジトリ上のコンテンツは、上記のようなハイパーリンクでいっぱいです。リンクが何を意味するのかわかったところで、**原作者が挿入したリンク**を削除しないようにしてください。



- もうひとつ、言葉の表記に関連することとして、次のようなものがある。本文中に「Plan ₿ Academy」があったら、**この原形のままにしておく**。plan や network は訳さないこと。また、Plan ₿ Networkを紹介するときに「The」という冠詞を使わないこと： **ブランドとして考えてください**。



- また、「⃻-CERT」、「BIZ SCHOOL」、「TECH SCHOOL」についても同様で、これらは原型をとどめておく必要がある。


この段落の最後の注釈として、上で述べたように、私たちはAIツールを使ってコンテンツを翻訳し、その後、投稿者に介入してもらって、すべてが流動的で校正が行き届いていることを確認しています。


文章の大部分をAIで校正した場合、私たちはAIが生成する典型的な文章構造を熟知しているので、必ず気づきます。もし、あなたが大きな変更を加えることなく、校正をAIだけに頼っていたことが分かれば、Satsでの最終報酬は半減するかもしれません！



## ヘッダーの構造


マークダウン言語では、見出し（および段落のタイトル）はすべてHash記号 ``#`` で始まります。Hash記号の数は見出しレベルに対応しています。例えば、レベル3の見出しはテキストの前に3つの数字記号があります（例：`### My Header`）。


コースでは、最も重要な部分は1つのHash記号で紹介され、サブパートは2つから4つのHash記号を持つことができます。チュートリアルでは、通常2つのHash記号を持つヘッダーのみを使用します。



![REVIEW](assets/en/6.webp)



タイトルの前のHash記号**は絶対に削除しないでください**。


同時に、上の画像にあるチャプターIDの部分、``<chapterId>d668fdf6-fb4c-4bbf-82e1-afcb95c122e0</chapterId>``や、``::video id=ba99951f-81d2-418f-b5e7-4b8c9f8b8cc8:::`` のようなビデオ参照は変更しないでください。


タイトルの前に ``#`` を挿入した場合、コースプレビューでは自動的に太字になりますので、**修正中に太字にすることは避けてください**。


余談ですが、EN版のコースでは、**1つまたは2つの``#``で始まるタイトルは、すべての単語が大文字で始まります**が、3つまたは4つの``#``で始まるタイトルは、通常この規則に従いません。可能であれば、ターゲット言語のタイトルがこの構造に従っていることを確認してください。



## コースの初期セクション


どのコンテンツの冒頭にも、次のような静的な小文字の単語がある：「名前」、「説明」、「目的」。これらはウェブサイトがコンテンツ自体をデコードするために使用され、**常にEN**のままです。そのため、これらを翻訳しないでください。そうしないと、コンテンツに同期の問題が生じます。AIが自動的に翻訳するコロン以降の部分のみを校正してください。



![REVIEW](assets/en/7.webp)



この同じ最初のセクションでは、書式をそのままにする。テキストの先頭には何も追加しないでください。例えば、下の画像のように、ダッシュの前に "tt "を追加するのは避けましょう！



![REVIEW](assets/en/8.webp)



## 推奨フォーマット


以下に、ターゲット言語のコンテンツを校正する際に注意すべき書式の例をいくつか挙げます。



- や ``**`` のような奇妙な句読点に注意してください。これは、太字記号のレンダリングが間違っている可能性があります。下の画像では、アスタリスクが単語の右側にしかなく、奇妙に見える。



![REVIEW](assets/en/9.webp)



従って、太字のテキストがあるはずかどうか、常に英語の原文をチェックしてください。この場合、単語の頭にアスタリスクを2つ追加するだけで、ウェブサイト上で正しく表示されるようになります。実際、マークダウン言語では、**太字を表示するには、単語/文の前後に2つのアスタリスク ``**`` を挿入する必要があります**（以下の例を参照）。



![REVIEW](assets/en/10.webp)




- や `` `` のような記号でも同じ問題が起こる可能性がある。

これらの記号がどこにあるべきかを確認するために、元の言語ファイル（多くの場合、ENまたはFR）を確認してください。この件に関しては、いつでもコーディネーターに助けを求めることができます。



- 引用符を見つけたら、必ずネットで調べて、あなたの言語での正しい翻訳を見つけましょう。引用符は通常、記号 ``>`` の後に挿入されます。



![REVIEW](assets/en/11.webp)

## クイズ校正


各コースの小テスト問題も校正できることをご存知ですか？例えば、ITのBTC101の小テストを校正したい場合、専用のブランチを開き、以下の手順に従ってください：「courses」 -> 「BTC101」 -> 「quiz」。そこには各問題専用のフォルダと_yml_フォーマットの関連言語ファイルがあります。


もう一度言うが、この目的のために開設した専用支店であることを確認し、必ずコーディネーターに報告すること。


この種の_yml_ファイルを校正するときに注意すべき重要なことは、テキストの中にコロン ``:`` を加えないようにすることです。実際、コロンは "wrong_answers "のようなキーとバリューのペアを区切るためだけに使われます。下の画像に例があります：


![REVIEW](assets/en/12.webp)


問題を見直したら、下の画像のように「reviewed」のステータスを「false」から「true」に変更してください。どの言語で作業している場合でも、これらのステータスの単語は必ず英語にしてください！



![REVIEW](assets/en/13.webp)


もし「reviewed:true」というステータス行がない場合は、クイズの最後に追加してください。


## 用語集の校正


クイズと同じように、用語集も校正することができます。元の用語集はフランス語で書かれているので、次のような文章があります：「フランス語では、この表現は...と訳せます。


このような場合、文章をターゲット言語または英語に直してください。例えば、"In English, this expression... "というように。

タイトルが英語のままなら、その文章をあなたの言語に合わせることができる："スワヒリ語では、この表現は..."


また、タイトルは必ず大文字で書くこと。


![REVIEW](assets/en/14.webp)


## PRのタイトルと説明


PRを送る際、次のような形式で名前をつけると素晴らしいでしょう：[校正] 内容名 - 言語：


```
[PROOFREADING] BTC101 - ENGLISH
```


また、PR**の**コメント欄には、校正作業を開始したときにコーディネーターから送られてきたissueの番号と "closes "を、その前に``#``と書いてください。

例えば、cyp201の校正＋クイズのPRを送ったばかりなら、「closes [#2934](https://github.com/PlanB-Network/Bitcoin-educational-content/issues/2934)」と書けばいい。


こうすることで、PRとイシューがつながり、公開されているGithubリポジトリを誰が読んでも簡単に情報を見つけることができる。



## その他のベストプラクティス



- テキスト内の特定の単語を検索したい場合は、``CTRL+F``をクリックすると、検索置換セクションが表示されます。この部分は、テキストの特定の部分にジャンプする必要がある場合や、コンテンツ全体をスクロールすることなく、特定の単語や文章を一括で置換する必要がある場合に非常に便利です。



![REVIEW](assets/en/15.webp)



全置換」機能を使用する場合、リンクも変更されていないか、結果をダブルチェックすることが重要です。例えば、"Bitcoin "という単語を "Bitkoin "に変更したい場合（言語によっては必要な場合もあります）、"全置換 "機能を使用すると、テキスト内のすべてのインスタンスを効率的に更新することができます。ただし、このツールを使用すると、その単語を含むリンクも変更されるため、リダイレクトの問題につながる可能性があることに注意してください。


下の例では、校正者は上記の関数を使って「Satoshi」を「Satoshi(Sats)」に置き換え、さらにその単語そのものを含むチュートリアルへのリンクを変更した。その結果、リンクは無効となりました。


本文中のハイパーリンクが正しいかどうか、必ず再確認すること。



![REVIEW](assets/en/16.webp)




- Plan ₿ Networkのコースやチュートリアルにリンクを貼る場合（括弧の中ではなく）、ウェブサイトは自動的に関連するサムネイルを表示する "カード "を作成します。その結果、テキストとリンクの間に改行を入れるようにしてください。



![REVIEW](assets/en/17.webp)


このような「画像コード」でも同じことが起こります ``[IMAGE](asset/fr/001.webp)``: 画像コードとテキストの間には必ず改行を入れてください。以下に例を示します：


```
WRONG CONFIGURATION:
- to start translating, click on the button `Translate`: ![language](assets/08.webp)
To save, click on `save`!


RIGHT CONFIGURATION:

- to start translating, click on the button `Translate`:

![language](assets/08.webp)

To save, click on `save`!
```
