## 什麼是校對？


校對是書面文章的最後審查程序，目的是找出並更正文法、拼寫、標點和格式上的錯誤。在出版或投稿前，可確保文字清晰、連貫且沒有錯誤。


當您進行這類任務時，重要的是要遵循原始語言（EN 或 FR）的意義，但要確保最終語言的文字對於母語使用者而言盡可能流暢。


永遠記住翻譯/校對是 EDUCATION！


事實上，我們的共同目標是讓盡可能多的人了解 Bitcoin，因此最基本的是讓他們讀到的資料流暢、清晰。

從這個意義上說，Plan ₿ Academy 上的所有貢獻者都是教育者！


## 在 Plan ₿ Academy 上進行校對之前的第一步


在開始新的校對工作之前，請在 [Telegram 群組](https://t.me/PlanBNetwork_ContentBuilder) 中公佈，或通知您的 Plan ₿ Academy 協調員，他們會開啟專屬的 [issue] (https://github.com/orgs/Plan ₿ Academy/projects/3)。當您收到問題連結時，只需**註明您正開始**該內容的校對工作。


此系統可協助協調員追蹤 repo 內的進度，並允許校對員「認領」內容，以防止其他人重複工作。

在問題本身，您會找到將您重定向至要檢查的內容的連結。您可以直接點擊它們，或者，更好的方法是回到您自己的 fork 版本，直接從那裡開始工作。讓我們看看您可以如何做！


首先，**永遠記得同步您的 repo，在「dev」分支上**。如此一來，在您開始任何類型的工作之前，內容總是會被更新，而且您也不會在新舊資料之間產生任何衝突。請務必點選「Sync Fork」和「Update branch」。



![REVIEW](assets/en/1.webp)



同步成功後，您可以直接存取感興趣的內容，並在新的分支上提交，如本 [教學](https://planb.academy/tutorials/contribution/content/proofreading-review-tutorial-28236c98-23b2-4efd-9563-953f08707017) 所示。否則，您可以點選「分支」開啟一個新的分支，如下圖所示。



![REVIEW](assets/en/2.webp)



在這個新頁面中，您可以在標題「您的分支」下找到您已經開啟的所有分支。這部分非常有用，因為它可以讓您輕鬆找到您修改某些內容的位置。如果您想開啟一個新的分支，可以按一下頁面右上角的「新分支」。



![REVIEW](assets/en/3.webp)



然後，您會看到一個彈出視窗，您需要插入新分支的名稱。在下面的例子中，我選擇稱它為 "BTC101-FR"。這樣，我就會永遠記得，這個特定的分支需要用於 BTC101 法語課程的校對，**我不會將它用於任何其他任務**。


我建議您也這麼做：確保在需要開始新任務時，隨時開啟新的分支。



![REVIEW](assets/en/4.webp)



建立這個新的分支後，請務必從上一頁的 "Your Branches「 點選它，然後開始處理與特定內容相關的 *.md* 檔案 (在我的例子中，我會點選 」courses" -> "BTC101" -> "fr.md")。所有與特定檔案相關的提交都必須在同一個分支中提交（儲存）。



## 原文還是譯文？


在進行一些內容校對（proofreading）時，**重要的是一定要檢查其原始英文（或法文）**版本。請注意，我們使用 AI 語言工具進行翻譯，因此對最終讀者而言，目標語言的呈現可能不夠流暢或難以理解。


因此，如有需要，請隨意調整文字和修改句子。我們的目標是提高流暢性，但始終遵循原意。如果對如何處理特定詞語有疑問，請務必詢問翻譯協調員。


LLM 工具可能會從字面上翻譯一些與 Bitcoin 相關的字詞，就像 Lightning Network。尤其是涉及到非常技術性的字詞時。在這種情況下，除非您的語言規則強制您翻譯每個字詞，否則建議您在目標語言中保留原始英文字詞，以提高清晰度。


在第二種情況下，**總要做一些研究，看看在您的 Bitcoin 社群中是否有人已經翻譯過這個字**，而且現在已經被廣泛使用。



- 一個解決方案可以是**在您的目標語言的[BitcoinWiki](https://en.Bitcoin.it/wiki/Main_Page)**上查看該詞是否被翻譯。如果沒有，您可以保留這個英文單詞。



- 無論如何，我的建議是**插入 EN 單詞 nonetheless**，在圓括號內添加目標語言的相應含義，遵循方案 EN (LANG)，反之亦然。例如Address (indirizzo)，或 indirizzo (Address)。



- 另一個好的解決方案是保留 EN 原有的字/詞，然後**建立一個超連結**，重定向到 planb.network 上的 [詞彙表](https://planb.academy/en/resources/glossary)。要做到這一點，您需要在方括號內插入單詞/短語，並在圓括弧內插入鏈接，就像您在下面的示例中看到的那樣：


```
[UTXO](https://planb.academy/resources/glossary/utxo)
```


在最後的結果（如下圖）中，您不會看到整個連結，而單字會變成可點選。



![REVIEW](assets/en/5.webp)



請注意，您將從網站取得的詞彙連結，在「network」一詞後面含有語言代碼（例如： ``https://planb.academy/en/resources/glossary/utxo`-> 此處您可以讀取語言代碼「en」）。在這種情況下，**從連結中移除語言代碼**，就像您在上面方塊中看到的那樣。這樣，系統就會自動將讀者帶到他們指定的語言。


資源庫上的內容充滿了像上面這些超連結。現在您知道它們的意思了，**請確定不要刪除原作者插入的任何連結**。



- 與字彙詮釋相關的另一件事如下。如果您在文中找到「Plan ₿ Academy」，請**保留此原始形式**。請勿翻譯「計畫」或「網路」。此外，在介紹 Plan ₿ Academy 時，請勿使用「The」一詞： **將其視為一個品牌**。



- 同樣地，「₿-CERT」、「BIZ SCHOOL」、「TECH SCHOOL」也應該保留原來的形式。


本段的最後一點說明：如上文所述，我們使用 AI 工具來翻譯內容，然後請撰稿人介入，以確保一切流暢且校對良好。


如果您使用 AI 來校對大部分的文字，我們一定會注意到，因為我們熟悉 AI 所產生的典型句子結構。如果我們發現您僅僅依賴 AI 進行校對，而沒有進行重大修改，Sats 中的最終獎勵可能會減少一半！



## 標頭的結構


在 markdown 語言中，標題（和段落標題）都以 Hash 符號 ``#`` 開頭。Hash 符號的數量對應於標題的層級。例如，第三級的標題在文字前有三個數字符號 (例如，`### 我的標題`)。


在課程中，最重要的部分會以一個 Hash 標誌來介紹，而子部分則可以有兩到四個 Hash 標誌。在教學中，我們通常只使用兩個 Hash 標誌的標頭。



![REVIEW](assets/en/6.webp)



請務必**切勿刪除標題前的 Hash 符號**，否則會造成文字結構上的問題。


與此同時，***請勿變更***您在上圖中看到的 chapterID 部分，``<chapterId>d668fdf6-fb4c-4bbf-82e1-afcb95c122e0</chapterId>``或影片引用，例如``::video id=ba99951f-81d2-418f-b5e7-4b8c9f8b8cc8:::``。


當我們在標題前插入 ``#``` 時，它會在課程預覽中自動變成粗體，因此**避免在修正時將標題變成粗體**。


附帶一提的是，在 EN 版本的課程中，由一個或兩個 ``#`` 開頭的**標題，所有單字都以大寫字母**開頭，而由三個或四個 ``#`` 開頭的標題，通常不遵循此規則。如果可能的話，請確保您目標語言的標題也遵循此結構。



## 課程的初始部分


在任何內容的開頭，您會發現下列靜態小寫字詞：「名稱」、「說明」、「目標」。這些詞彙是網站用來解碼內容本身的，而且**始終保留為 EN**。因此，請勿翻譯它們，否則內容會產生同步問題。請確保只校對冒號之後的部分，該部分會由 AI 自動翻譯。



![REVIEW](assets/en/7.webp)



在這個相同的開頭部分，保持原來的格式。不要在文字開頭添加任何東西。例如，避免在破折號前加上「tt」，就像下面的圖片一樣！



![REVIEW](assets/en/8.webp)



## 格式建議


以下是一些校對目標語言內容時需要注意的格式問題範例。



- 注意一些奇怪的標點符號，如`**`或`**`，它們可能代表粗體符號的不良渲染。在下面的圖片中，您可以看到星號只出現在單字的右邊，看起來很奇怪。



![REVIEW](assets/en/9.webp)



因此，請務必檢查英文原文，看看是否應該有粗體字。在這種情況下，只要在字首加上兩個星號，就可以讓它正確地顯示在網站上。事實上，在 Markdown 語言中，**要顯示粗體，您必須在字/句**前後插入兩個星號``**``（請參閱以下範例）。



![REVIEW](assets/en/10.webp)




- 同樣的問題也可能發生在 $ 和 ``` ``` 等符號上。

請務必檢查原始語言檔案（通常是 EN 或 FR），以瞭解這些符號應該放在哪裡。您可以隨時向協調員請求協助。



- 如果您發現引號，請務必在網上做一些研究，以找到您語言的正確翻譯。引號通常插在符號 ``>``` 之後。



![REVIEW](assets/en/11.webp)

## 测验校对


您知道您还可以校对每门课程的测验题吗？例如，如果您想校对信息技术 BTC101 的测验，您可以打开一个专门的分支，然后按照以下路径进行操作："课程" -> "BTC101" -> "测验"。在那里，您可以找到每个问题的所有专用文件夹，以及 _yml_ 格式的相关语言文件。


再次，确保您在专门为此目的开设的专用分行，并始终通知协调员。


在校对这类 _yml_ 文件时，要注意避免在文本中添加冒号 ``:``。事实上，冒号仅**用于分隔 "wrong_answers "等键值对和其他键值对**。下图就是一个例子：


![REVIEW](assets/en/12.webp)


审阅问题后，确保将 "已审阅 "状态从 "假 "更改为 "真"，如下图所示。无论您使用哪种语言，都要确保这些状态词使用英语！



![REVIEW](assets/en/13.webp)


如果缺少状态行 "reviewed:true"，请确保**在测验**的结尾处添加。


## 词汇校对


与测验一样，您也可以校对词汇表。最初的词汇表是用法语写的，所以你会发现像这样的句子："在法语中，这个表达可以翻译成......"


在这种情况下，请根据目标语言或英语改写句子。例如，您可以写 "在英语中，这个表达......"。

如果标题为英文，您可以根据自己的语言调整句子："在斯瓦希里语中，这个表达......"


此外，确保用大写字母书写标题。


![REVIEW](assets/en/14.webp)


## PR 的标题和描述


当您发送公关稿时，如果能使用以下格式为其命名，那将会非常棒：[校对] 内容名称 - 语言：


```
[PROOFREADING] BTC101 - ENGLISH
```


此外，在 PR**的**评论部分，您可以写上 "已关闭 "+协调人在您开始校对任务时发送给您的问题编号，前面加上 "#"。

例如，如果您刚刚发送了校对 cyp201 + quizzes 的 PR，您可以写 "closes [#2934](https://github.com/PlanB-Network/Bitcoin-educational-content/issues/2934)" 。


这样，PR 和问题就会连接起来，无论谁阅读 Github 公共仓库，都能轻松找到相关信息。



## 其他最佳做法



- 如果需要搜索文本中的特定词语，可以点击 ``CTRL+F`` ，然后会出现查找-替换部分。当你需要跳转到文本的特定部分，或者需要批量替换特定的单词/句子，而不需要滚动整个内容时，这部分非常有用。



![REVIEW](assets/en/15.webp)



使用 "全部替换 "功能时，一定要仔细检查结果，确保链接没有被修改。例如，如果您想将单词 "Bitcoin "改为 "Bitkoin"（在某些语言中可能是必要的），使用 "全部替换 "功能可以有效地更新文本中的所有实例。但要注意的是，该工具也会修改包含该单词的任何链接，可能会导致重定向问题。


在下面的例子中，校对员使用上述功能将 "Satoshi "替换为 "Satoshi(Sats)"，并将链接改为包含该词本身的教程。结果，链接失效。


务必仔细检查文本中的所有超链接，确保其正确无误。



![REVIEW](assets/en/16.webp)




- 顺着主题，如果作者插入了指向 Plan ₿ Academy 课程或教程的链接（**不**在括号内），网站会自动创建一个 "卡片"，显示相关缩略图。因此，请务必在文本和链接本身之间**加一行新线**，否则网站可能会出现以下错误。



![REVIEW](assets/en/17.webp)


同样的情况也会发生在 "图像代码 "中，例如``[IMAGE](asset/fr/001.webp)``：请确保在图像代码和文本之间添加新行。下面是一个例子：


```
WRONG CONFIGURATION:
- to start translating, click on the button `Translate`: ![language](assets/08.webp)
To save, click on `save`!


RIGHT CONFIGURATION:

- to start translating, click on the button `Translate`:

![language](assets/08.webp)

To save, click on `save`!
```
