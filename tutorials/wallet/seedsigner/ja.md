SeedSignerは、オープンソースのwallet Bitcoinハードウェアで、安価な汎用電子部品を使って誰でも自作することができます。Ledger、Coldcard、Trezorのような商用製品とは異なり、これは企業によって製造されたすぐに使えるデバイスではありません：これはコミュニティ・プロジェクトであり、誰もがすべてのステップを制御しながら自分自身のデバイスを作成することができます。



SeedSignerは100％***エアギャップ***で設計されています：インターネットに接続することはなく、Wi-FiやBluetooth（Raspberry Pi Zero v1.3の場合）もなく、データを交換するためにコンピュータに接続することもありません。通信はもっぱらQRコード交換システムを介して行われる。具体的には、ポートフォリオ管理ソフトウェア（Sparrow Walletなど）が署名すべき取引をQRコードで表示し、それをSeedSignerのカメラでスキャンすると、デバイスがRAMに一時的に保存された秘密鍵を使って取引に署名する。最後に、SeedSignerは署名されたトランザクションを含むQRコードを生成し、それをソフトウェアでスキャンしてBitcoinネットワークに送信する。



![Image](assets/fr/001.webp)



また、SeedSignerは***ステートレス***です。つまり、他のハードウェアウォレットとは異なり、seedや秘密鍵を永久に保存しません。microSDカードに設定を保存するようにデバイスを設定しない限り、再起動するたびにメモリは完全に空になります。したがって、seedを使用するたびに再入力する必要があります。最も実用的な方法は、QRコードの形で保存し、SeedSignerのカメラを使って起動時にスキャンすることです。この動作モードは、攻撃対象範囲を大幅に縮小します。たとえ泥棒があなたのデバイスを盗んだとしても、デフォルトでは常に空なので、泥棒はデバイス上の情報を見つけることはできません。



seedを保管し、SeedSignerで使用するもう一つの方法は、*SeedKeeper*スマートカードと互換性のあるリーダーを使用することです。これにより、seedを保管するための非常に堅牢な*Secure Element*ができ、同時にSeedSignerの画面を使ってトランザクションに署名することができます。しかし、この特殊な構成については、別の専用チュートリアルの対象です。ここでは、SeedSignerの基本的な使い方を説明します：



https://planb.academy/tutorials/wallet/hardware/seedkeeper-seedsigner-45cca4c4-1f22-46bb-87ae-9cddb68aa579

SeedSignerプロジェクトは、Bitcoinエコシステムにおいて重要な位置を占めている。世界中の誰もが、ビットコインを保護するための高度なセキュリティの恩恵を受ける可能性を提供するからだ。その主な利点は、必要なハードウェアが50ドル未満で購入できるというアクセスのしやすさにある。さらに、制限のある国に住む人々は、入手が容易で規制の制約を受けにくい標準的なコンピュータ部品から、walletハードウェアを自作することができる。



SeedSignerはオープンソースで、ステートレスで、エアギャップで動作し、walletハードウェアのサプライチェーンに関連する攻撃ベクトルを減らすことができる。

## 2.ソフトウェアの準備



ハードウェアが揃ったら、microSDカードにSeedSignerシステムをインストールして準備する必要があります。これを行うには、普段お使いのパソコンに行き、SeedSigner用のmicroSDを差し込んでください。



### 2.1.ダウンロード



プロジェクトの公式GitHubリポジトリ](https://github.com/SeedSigner/seedsigner/releases)にアクセスする。ソフトウェアの最新バージョンでは、.NET Frameworkをダウンロードしてください：




- あなたのPiモデルに対応する `.img` イメージ。
- .sha256.txt`ファイル。
- .sha256.txt.sig`ファイル。



![Image](assets/fr/006.webp)



インストールを始める前に、ソフトウェアをチェックしよう。



### 2.2 LinuxとmacOSでの検証



まず、SeedSignerプロジェクトの公式公開鍵をKeybaseから直接インポートします：



```
gpg --fetch-keys https://keybase.io/seedsigner/pgp_keys.asc
```



![Image](assets/fr/007.webp)



鍵がインポートまたは更新されたことがターミナルに表示されるはずです。次に、署名ファイルに対して検証コマンドを実行する（ここでは`0.8.6.`）：



```
gpg --verify seedsigner.0.8.6.sha256.txt.sig
```



![Image](assets/fr/008.webp)



すべてが正しければ、`Good signature`と出力されるはずである。これは、ファイル `.sha256.txt` がインポートした鍵によって署名され、その署名が有効であることを意味する。警告メッセージ `WARNING: This key is not certified with a trusted signature` は無視してください。



これを行うには、表示されたフィンガープリントの最後の16文字を[Keybase.io/SeedSigner](https://keybase.io/seedsigner)、彼らの[公式Twitter](https://twitter.com/SeedSigner/status/1530555252373704707)、または[SeedSigner.com](https://seedsigner.com/keybase.txt)で公開されているファイル内のものと比較します。これらの識別子が完全に一致すれば、その鍵が本当にそのプロジェクトのものであると確信できます。疑わしい場合は、直ちに停止し、SeedSignerコミュニティ（Telegram、X、GitHub...）に助けを求めてください。



キーが認証されたら、ダウンロードしたイメージが変更されていないことを確認できます（バージョンに応じてコマンドを変更することを忘れないでください、ここでは `0.8.6.`）：



```
shasum -a 256 --ignore-missing --check seedsigner.0.8.6.sha256.txt
```



![Image](assets/fr/009.webp)





- Linuxでは、このコマンドはビルトインされている。
- 警告: `Big Sur (11)` より前の macOS のバージョンは `--ignore-missing` オプションを認識しません。この場合、このオプションを削除し、ファイルが見つからないという警告を無視してください。



期待される結果は、`.img`ファイルの横に`OK`が表示されることです。これは、アップロードされた画像がプロジェクトによって公開されたものと同一であり、変更されていないことを確認するものです。



### 2.3 ウィンドウズ認証



Windowsでは、手順は似ているがコマンドが異なる。まず[Gpg4win](https://www.gpg4win.org/)をインストールし、*Kleopatra*アプリケーションを開きます。URLのKeybaseからSeedSignerプロジェクトの公開鍵をインポートする：



```
https://keybase.io/seedsigner/pgp_keys.asc
```



![Image](assets/fr/010.webp)



次に、ダウンロードしたファイルがあるフォルダで PowerShell を開きます（`Shift` + 右クリック > `ここで PowerShell を開く`）。以下のコマンドを実行して、マニフェストの署名を確認します（ここでは `0.8.6.` というように、バージョンに応じてコマンドを変更することを忘れないでください）：



```
gpg --verify seedsigner.0.8.6.sha256.txt.sig
```



![Image](assets/fr/011.webp)



すべてが正しければ、`Good signature`と出力されるはずである。これは、ファイル `.sha256.txt` がインポートした鍵によって署名され、その署名が有効であることを意味する。警告メッセージ `WARNING: This key is not certified with a trusted signature` は無視してください。



これを行うには、表示されたフィンガープリントの最後の16文字を[Keybase.io/SeedSigner](https://keybase.io/seedsigner)、彼らの[公式Twitter](https://twitter.com/SeedSigner/status/1530555252373704707)、または[SeedSigner.com](https://seedsigner.com/keybase.txt)で公開されているファイル内のものと比較します。これらの識別子が完全に一致すれば、その鍵が本当にそのプロジェクトのものであると確信できます。疑わしい場合は、直ちに停止し、SeedSignerコミュニティ（Telegram、X、GitHub...）に助けを求めてください。



キーが検証されたら、イメージ・ファイルが壊れていないことを確認する必要がある。これを行うには、PowerShellで以下のコマンドを使用する：



```
CertUtil -hashfile seedsigner_os.0.8.6.[your-Pi-model].img SHA256
```



Raspberry Pi Zero 2の例（ここでは`0.8.6.`）：



```
CertUtil -hashfile seedsigner_os.0.8.6.pi02w.img SHA256
```



![Image](assets/fr/012.webp)



その後、PowerShellは画像ファイルのSHA256ハッシュを計算します。このハッシュを `seedsigner.0.8.6.sha256.txt` の対応する値と比較します。




- 2つの値が厳密に同じであれば、チェックは成功し、続行できる。
- 異なる場合は、ファイルが壊れているか、破損しています。使用せず、ダウンロードをやり直してください。



![Image](assets/fr/013.webp)



検証に成功すると、`.img`ファイルが本物であり（SeedSignerによって署名されている）、かつ変更されていない（改変されていない）ことが保証されます。これで安心して次のステップに進むことができます。



### 2.4.画像をフラッシュする



まだお持ちでない方は、[Balena Etcher]ソフトウェア(https://etcher.balena.io/)をダウンロードしてください：




- microSDカードをコンピュータに挿入します。
- ローンチ・エッチャー
- ダウンロードして確認した `.img` ファイルを選択する。
- ターゲットとしてmicroSDカードを選択します。
- フラッシュ！」をクリックする。



![Image](assets/fr/014.webp)



プロセスが完了するまでお待ちください。いよいよ組み立てです！



## 3.SeedSignerアセンブリ



microSDカードを準備し、SeedSignerソフトウェアでフラッシュしたら、最終的な組み立てに進むことができます。壊れやすい部品もあるので（特にテーブルクロス、カメラ、GPIOピン）、時間をかけてください。



### 3.1 ハウジングの準備



まず、ケースを開けてください。清潔であること、3Dプリントのプラスチックが内部の留め具の邪魔になっていないことを確認してください。以下のものに注意してください：




- カメラの位置（正面の小さな丸い穴）。
- スクリーンのオープニング。
- Raspberry Pi ZeroのマイクロUSBポートとmicroSDスロットのカットアウト。



### 3.2 カメラの設置



Raspberry Pi Zeroのカメラリボンコネクタを探します：基板の側面にある細い黒い帯で、少し持ち上げると開きます。数ミリ傾くだけです。



![Image](assets/fr/015.webp)



カメラカバーを挿入します。茶色/銅の部分が下を向くようにしてください。ねじれないように、しっかりとコネクターに収まっていることを確認してください。



![Image](assets/fr/016.webp)



黒いバーを閉じてテーブルクロスをロックします（わずかにカチッと音がします）。テーブルクロスが動かないことをゆっくり確認してください。



![Image](assets/fr/017.webp)



次に、カメラモジュールをハウジングの適切な穴に入れます。モデルによっては、直接はめ込むか、小さな粘着ストリップで固定する必要があります。レンズは外側に向け、完全に位置が合っている必要があります。



### 3.3 Raspberry Pi Zeroのインストール



ケースを使用している場合は、Raspberry Pi Zeroボードを中に挿入します。ポートの位置を慎重に合わせます。



次にWaveshareディスプレイをRaspberry Pi Zeroの上に置きます。PiのGPIOピンがディスプレイのメスコネクターにぴったり合うようにします。ディスプレイをピンの上にゆっくりと押し付け、ピンが曲がらないように両側に均等に圧力をかけます。



![Image](assets/fr/018.webp)



ケースを持っている場合は、フロントパネルとジョイスティックを追加して組み立てを完了する。



最後に、フラッシュしたソフトウェアの入ったmicroSDカードをRaspberry Pi Zeroの端にあるスロットに挿入します。カチッと音がすることを確認してください。



### 3.4 最初のスタートアップ



マイクロUSB電源ケーブルを専用ポートに接続する。約1分待ちます。SeedSignerのロゴが表示され、ホーム画面が表示されます。



![Image](assets/fr/019.webp)



まずはじめに、`Settings > I/O test`メニューで、様々なコンポーネントが正しく動作していることを確認する。



![Image](assets/fr/020.webp)



すべてのボタンをテストし、正しく反応することを確認してください。次に`KEY1`ボタンをクリックし、カメラが期待通りに動作していることを確認してください。これで写真が撮れます。



![Image](assets/fr/021.webp)



### 3.5 カメラの調整



SeedSignerの取り付け方によっては、カメラの画像が反転して表示されることがあります。これを修正するには、`設定 > 詳細 > カメラの回転`に進み、必要に応じて180°の回転を選択します。



![Image](assets/fr/022.webp)



カメラの向きを変更したり、他の設定（インターフェース言語など）を後日変更したい場合は、microSDの設定の永続化を有効にする必要があります。そうしないと、Raspberry Pi Zeroには永続メモリがないため、再起動するたびに設定がデフォルトに戻ってしまいます。



これを行うには、`Settings > Persistent settings`メニューを開き、`Enabled`を選択します。



![Image](assets/fr/023.webp)



すべてが機能していれば、SeedSignerは使用可能です！

次に`Advanced`メニューに進みます。ここにはいくつかの便利なオプションがあります：

## 5.seedの作成と保存



seed（またはニーモニック・フレーズ）は、あなたのBitcoinポートフォリオの基礎を形成します。seedはあなたの秘密鍵とアドレスを導き出し、資金にアクセスするために使用されます。SeedSignerはseedを生成するためのいくつかの方法を提供しています。



その前に、いくつかの重要な注意事項を：




- このフレーズは、あなたのすべてのビットコインに完全かつ無制限にアクセスできるようにします。** このフレーズを所持する者は、あなたのSeedSignerに物理的にアクセスしなくても、あなたの資金を盗むことができます；
- 通常、walletのハードウェアを紛失または盗難された場合、12語のフレーズを使用してwalletを復元します。しかしSeedSignerは*ステートレス*デバイスなので、seedを登録することはありません。ですから、あなたの物理的バックアップは単なるバックアップコピーではなく、**あなたのwalletを使用する唯一の方法**なのです。もしこれらのバックアップを失ったら、あなたのビットコインは永久に失われます。ですから、慎重に、複数のメディアに、安全な場所にバックアップしてください；
- まだ始めたばかりであれば、このチュートリアルを読んで、ニーモニック・フレーズを管理する上でのリスクを詳しく理解することを強くお勧めする：

### 5.6 「ステートレス・デバイス」の概念を理解する



SeedSigner は永久メモリーを持たないデバイスです。つまり、Ledger、Trezor、Coldcard などとは異なり、seed がデバイス内に保存されることはありません。電源を切ると同時に、seed は RAM から完全に消えます。再起動すると、SeedSignerは空白の状態に戻ります。その後、あなたのseedを再びSeedSignerに渡し、あなたの取引に署名できるようにする必要があります。



これは本質的な保護を提供します。他のハードウェアウォレットとは異なり、SeedSignerはRaspberry Pi Zeroをベースにしており、*Secure Element*を含む物理的な保護はありません。しかし、機密データが保存されていないため、物理的に侵害されたデバイスであっても、攻撃者があなたの秘密鍵を抜き取ったり、ビットコインを使ったりすることはできません。



その一方で、このアーキテクチャはさらなる責任を意味します：バックアップがなければ、資金は決定的に失われます。だからこそ、私は**二重バックアップ**を推奨しているのです。リカバリ・フレーズはすでにお持ちでしょう：これは安全な場所に保管する長期的なメイン・バックアップです。次に、このフレーズのコピーを**QRコード**の形で作成します。



SeedSignerを使用するたびに、デバイスのカメラでこのQRコードをスキャンし、取引に署名している間、seedを一時的にメモリにロードします。このQRコードを持っている人は、あなたのビットコインに完全にアクセスすることができます。


また、QRコードとニーモニックフレーズを別々の場所に保管し、クレームの際にすべてを失わないようにすることをお勧めする。



最後に、より高度で安全な方法として、seedをsecure elementに保存する**SeedKeeper**と一緒にSeedSignerを使用する方法があります。詳しくはこちらのチュートリアルをご覧ください：



https://planb.academy/tutorials/wallet/hardware/seedkeeper-seedsigner-45cca4c4-1f22-46bb-87ae-9cddb68aa579

![Image](assets/fr/068.webp)



すべてが正しければ、`Approve PSBT` を選択して、対応する秘密鍵を使用してトランザク ションに署名する。



![Image](assets/fr/069.webp)



署名されると、SeedSignerは署名されたトランザクションを含む新しいQRコードを生成し、Sparrowでスキャンできるようにする。



![Image](assets/fr/070.webp)



#### 7.2.3 Sparrowからのトランザクションのブロードキャスト



この取引は有効であるため、Bitcoinネットワーク上でブロードキャストされ、ブロックに追加するマイナーに届く必要がある。



Sparrowで「QRスキャン」をクリックする。



![Image](assets/fr/071.webp)



SeedSignerが表示したQRコード（署名された取引のQRコード）をウェブカメラに提示してください。Sparrowが署名をデコードし、取引の詳細を表示します。すべての情報が正しいことを最終確認し、「Broadcast Transaction」をクリックしてBitcoinネットワークにブロードキャストします。



![Image](assets/fr/072.webp)



あなたの取引は Bitcoin ネットワークに送信されました。Sparrow Walletの`Transactions`タブで進行状況を確認できます。



![Image](assets/fr/073.webp)
