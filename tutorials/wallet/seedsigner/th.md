![cover](assets/cover.webp)



SeedSigner เป็นฮาร์ดแวร์ wallet Bitcoin แบบโอเพนซอร์สที่ใครๆ ก็สามารถสร้างขึ้นเองได้โดยใช้ส่วนประกอบอิเล็กทรอนิกส์ทั่วไปที่มีราคาถูก แตกต่างจากผลิตภัณฑ์เชิงพาณิชย์เช่น Ledger, Coldcard หรือ Trezor นี่ไม่ใช่อุปกรณ์พร้อมใช้ที่ผลิตโดยบริษัท: มันเป็นโครงการของชุมชนที่อนุญาตให้ใครก็ตามสร้างอุปกรณ์ของตนเอง ควบคุมทุกขั้นตอน



SeedSigner ได้รับการออกแบบให้เป็น ***air-gapped*** 100%: มันไม่เคยเชื่อมต่อกับอินเทอร์เน็ต ไม่มี Wi-Fi หรือ Bluetooth (ในกรณีของ Raspberry Pi Zero v1.3) และไม่เคยเชื่อมต่อกับคอมพิวเตอร์เพื่อแลกเปลี่ยนข้อมูล การสื่อสารเป็นแบบเฉพาะผ่านระบบแลกเปลี่ยน QR code ในทางปฏิบัติ ซอฟต์แวร์การจัดการ wallet ของคุณ (เช่น Sparrow Wallet) จะแสดงธุรกรรมที่จะลงนามในรูปแบบของ QR codes; คุณสแกนด้วยกล้องของ SeedSigner จากนั้นอุปกรณ์จะลงนามในธุรกรรมโดยใช้กุญแจส่วนตัวของคุณที่เก็บไว้ชั่วคราวใน RAM สุดท้ายมันจะสร้าง QR codes ที่มีธุรกรรมที่ลงนามแล้ว ซึ่งคุณสแกนด้วยซอฟต์แวร์ของคุณเพื่อส่งไปยังเครือข่าย Bitcoin



![Image](assets/fr/001.webp)



SeedSigner ยังเป็น ***stateless*** อีกด้วย กล่าวอีกนัยหนึ่งคือ มันจะไม่บันทึก seed หรือกุญแจส่วนตัวของคุณอย่างถาวร แตกต่างจากกระเป๋าเงินฮาร์ดแวร์อื่น ๆ ทุกครั้งที่คุณรีบูต หน่วยความจำของมันจะว่างเปล่าโดยสมบูรณ์ เว้นแต่คุณจะตั้งค่าอุปกรณ์ให้บันทึกการตั้งค่าของคุณลงในการ์ด microSD ดังนั้นคุณจึงต้องป้อน seed ของคุณใหม่ทุกครั้งที่ใช้งาน วิธีที่สะดวกที่สุดคือการเก็บไว้ในรูปแบบของรหัส QR เพื่อสแกนเมื่อเริ่มต้นใช้งานโดยใช้กล้องของ SeedSigner โหมดการทำงานนี้ช่วยลดพื้นผิวการโจมตีได้อย่างมาก: แม้ว่าขโมยจะขโมยอุปกรณ์ของคุณไป เขาก็จะไม่พบข้อมูลใด ๆ บนอุปกรณ์ เนื่องจากมันจะว่างเปล่าเสมอโดยค่าเริ่มต้น



อีกทางเลือกหนึ่งสำหรับการจัดเก็บ seed ของคุณและการใช้งานร่วมกับ SeedSigner คือการใช้ *SeedKeeper* สมาร์ทการ์ดร่วมกับเครื่องอ่านที่เข้ากันได้ ซึ่งจะทำให้คุณมี *Secure Element* ที่แข็งแกร่งมากในการจัดเก็บ seed ของคุณ ในขณะที่ใช้หน้าจอ SeedSigner เพื่อเซ็นชื่อธุรกรรม แต่การกำหนดค่านี้เป็นหัวข้อของบทเรียนเฉพาะอื่น ที่นี่เราจะมุ่งเน้นไปที่การใช้งานพื้นฐานของ SeedSigner:



https://planb.academy/tutorials/wallet/hardware/seedkeeper-seedsigner-45cca4c4-1f22-46bb-87ae-9cddb68aa579

โครงการ SeedSigner มีความสำคัญในระบบนิเวศ Bitcoin เนื่องจากมอบโอกาสให้ทุกคนทั่วโลกได้รับประโยชน์จากความปลอดภัยขั้นสูงในการปกป้องบิตคอยน์ของพวกเขา ข้อได้เปรียบหลักของมันคือความสามารถในการเข้าถึง: ฮาร์ดแวร์ที่จำเป็นสามารถซื้อได้ในราคาต่ำกว่า $50 นอกจากนี้ยังช่วยให้ผู้ที่อาศัยอยู่ในประเทศที่มีข้อจำกัดสามารถสร้างฮาร์ดแวร์ wallet ของตนเองจากส่วนประกอบคอมพิวเตอร์มาตรฐาน ซึ่งหาซื้อได้ง่ายและมีข้อจำกัดด้านกฎหมายน้อยกว่า



แต่อยู่ภายนอกบริบทเฉพาะเหล่านี้ SeedSigner อาจเป็นตัวเลือกที่น่าสนใจสำหรับคุณ: มันเป็นโอเพ่นซอร์ส ทำงานแบบไร้สถานะและแยกจากเครือข่าย และลดเวกเตอร์การโจมตีที่เชื่อมโยงกับห่วงโซ่อุปทานของฮาร์ดแวร์ wallet ของคุณ



## 1. อุปกรณ์ที่จำเป็น



ในการสร้าง SeedSigner ของคุณ คุณจะต้องใช้ส่วนประกอบดังต่อไปนี้:





- Raspberry Pi Zero
    - เวอร์ชัน 1.3 เป็นที่แนะนำ เนื่องจากไม่มี Wi-Fi หรือ Bluetooth ทำให้มั่นใจได้ถึงการแยกตัวอย่างสมบูรณ์
 - W และ v2 เวอร์ชันก็เข้ากันได้เช่นกัน แต่มีการรวมชิป Wi-Fi/Bluetooth ไว้ด้วย ดังนั้นจึงแนะนำให้ปิดการใช้งานทางกายภาพโดยการถอดโมดูลวิทยุออกจากการ์ด การดำเนินการนี้ค่อนข้างง่าย แต่ต้องการความแม่นยำ (คีมขนาดเล็กเพียงพอสำหรับ Zero W ในขณะที่ต้องใช้ปากกาความร้อนสำหรับ v2 เพื่อถอดแผ่นโลหะที่ซ่อนโมดูล) ฉันจะไม่ลงรายละเอียดในบทแนะนำนี้ แต่คุณจะพบคำแนะนำทั้งหมดในเอกสารนี้: *[Disabling WiFi/Bluetooth by hardware](https://github.com/DesobedienteTecnologico/rpi_disable_wifi_and_bt_by_hardware)*.
 - โปรดทราบ: บางรุ่นของ Raspberry Pi Zero จำหน่ายโดยไม่มีขา GPIO ที่บัดกรีไว้ล่วงหน้า คุณสามารถซื้อรุ่นที่มีขาในตัวโดยตรง (วิธีที่ง่ายที่สุด) หรือซื้อขาแยกต่างหากและบัดกรีเอง (วิธีที่ซับซ้อนกว่า)
 - อย่าลืมรวมแหล่งจ่ายไฟแบบ micro-USB ด้วย

## 2. การเตรียมซอฟต์แวร์



เมื่อคุณมีฮาร์ดแวร์พร้อมแล้ว คุณจำเป็นต้องเตรียม microSD card โดยการติดตั้งระบบ SeedSigner ลงไป ในการทำเช่นนี้ ให้ไปที่คอมพิวเตอร์ส่วนตัวที่คุณใช้ทุกวัน และเสียบ microSD ที่ตั้งใจใช้สำหรับ SeedSigner เข้าไป



### 2.1. ดาวน์โหลด



ไปที่ [ที่เก็บ GitHub อย่างเป็นทางการของโครงการ](https://github.com/SeedSigner/seedsigner/releases) ในเวอร์ชันล่าสุดของซอฟต์แวร์ ดาวน์โหลด :




- ไฟล์ภาพ `.img` ที่ตรงกับรุ่น Pi ของคุณ
- ไฟล์ `.sha256.txt`
- ไฟล์ `.sha256.txt.sig`



![Image](assets/fr/006.webp)



ก่อนเริ่มการติดตั้ง มาตรวจสอบซอฟต์แวร์กันก่อน



### 2.2 การตรวจสอบภายใต้ Linux และ macOS



เริ่มต้นโดยการนำเข้าคีย์สาธารณะอย่างเป็นทางการของโครงการ SeedSigner โดยตรงจาก Keybase :



```
gpg --fetch-keys https://keybase.io/seedsigner/pgp_keys.asc
```



![Image](assets/fr/007.webp)



เทอร์มินัลควรแจ้งให้คุณทราบว่ามีการนำเข้าหรืออัปเดตคีย์แล้ว จากนั้นให้รันคำสั่งการตรวจสอบบนไฟล์ลายเซ็น (อย่าลืมแก้ไขคำสั่งตามเวอร์ชันของคุณ ที่นี่ `0.8.6.`):



```
gpg --verify seedsigner.0.8.6.sha256.txt.sig
```



![Image](assets/fr/008.webp)



หากทุกอย่างถูกต้อง ผลลัพธ์ควรแสดงว่า `Good signature` ซึ่งหมายความว่าไฟล์ `.sha256.txt` ได้รับการลงนามโดยคีย์ที่คุณเพิ่งนำเข้า และลายเซ็นนั้นถูกต้อง ให้ละเว้นข้อความเตือน `WARNING: This key is not certified with a trusted signature`: นี่เป็นเรื่องปกติ เนื่องจากตอนนี้ขึ้นอยู่กับคุณที่จะตรวจสอบว่าคีย์ที่ใช้เป็นของโครงการ SeedSigner



ในการทำเช่นนี้ ให้เปรียบเทียบอักขระ 16 ตัวสุดท้ายของลายนิ้วมือที่แสดงกับที่มีอยู่บน [Keybase.io/SeedSigner](https://keybase.io/seedsigner), บน [Twitter อย่างเป็นทางการของพวกเขา](https://twitter.com/SeedSigner/status/1530555252373704707), หรือในไฟล์ที่เผยแพร่บน [SeedSigner.com](https://seedsigner.com/keybase.txt) หากตัวระบุเหล่านี้ตรงกันอย่างแม่นยำ คุณสามารถมั่นใจได้ว่าคีย์นั้นเป็นของโครงการจริง หากมีข้อสงสัย ให้หยุดทันทีและขอความช่วยเหลือจากชุมชน SeedSigner (Telegram, X, GitHub...)



เมื่อคีย์ได้รับการตรวจสอบแล้ว คุณสามารถตรวจสอบได้ว่าภาพที่ดาวน์โหลดมาไม่ได้ถูกแก้ไข (อย่าลืมแก้ไขคำสั่งตามเวอร์ชันของคุณ ที่นี่ `0.8.6.`):



```
shasum -a 256 --ignore-missing --check seedsigner.0.8.6.sha256.txt
```



![Image](assets/fr/009.webp)





- ภายใต้ Linux คำสั่งนี้เป็นคำสั่งในตัว
- คำเตือน: macOS เวอร์ชันก่อนหน้า `Big Sur (11)` ไม่รู้จักตัวเลือก `--ignore-missing` ในกรณีนี้ ให้ลบออกและเพิกเฉยต่อคำเตือนเกี่ยวกับไฟล์ที่หายไป



ผลลัพธ์ที่คาดหวังคือ `OK` ถัดจากไฟล์ `.img` ซึ่งยืนยันว่าไฟล์ภาพที่อัปโหลดนั้นเหมือนกับไฟล์ที่เผยแพร่โดยโครงการและไม่ได้ถูกแก้ไข



### 2.3 การยืนยัน Windows



บน Windows ขั้นตอนจะคล้ายกันแต่คำสั่งจะแตกต่างกัน เริ่มต้นด้วยการติดตั้ง [Gpg4win](https://www.gpg4win.org/) และเปิดแอปพลิเคชัน *Kleopatra* นำเข้าคีย์สาธารณะของโครงการ SeedSigner จาก URL Keybase :



```
https://keybase.io/seedsigner/pgp_keys.asc
```



![Image](assets/fr/010.webp)



ถัดไป เปิด PowerShell ในโฟลเดอร์ที่ไฟล์ที่คุณดาวน์โหลดอยู่ (`Shift` + คลิกขวา > `Open PowerShell here`) รันคำสั่งต่อไปนี้เพื่อตรวจสอบลายเซ็นของ manifest (อย่าลืมแก้ไขคำสั่งตามเวอร์ชันของคุณ ที่นี่คือ `0.8.6.`):



```
gpg --verify seedsigner.0.8.6.sha256.txt.sig
```



![Image](assets/fr/011.webp)



หากทุกอย่างถูกต้อง ผลลัพธ์ควรอ่านว่า `Good signature` ซึ่งหมายความว่าไฟล์ `.sha256.txt` ได้รับการลงนามโดยคีย์ที่คุณเพิ่งนำเข้า และลายเซ็นนั้นถูกต้อง ให้ละเว้นข้อความเตือน `WARNING: This key is not certified with a trusted signature`: นี่เป็นเรื่องปกติ เนื่องจากตอนนี้ขึ้นอยู่กับคุณที่จะตรวจสอบว่าคีย์ที่ใช้เป็นของโครงการ SeedSigner



ในการทำเช่นนี้ ให้เปรียบเทียบอักขระ 16 ตัวสุดท้ายของลายนิ้วมือที่แสดงกับที่มีอยู่บน [Keybase.io/SeedSigner](https://keybase.io/seedsigner), บน [Twitter อย่างเป็นทางการของพวกเขา](https://twitter.com/SeedSigner/status/1530555252373704707), หรือในไฟล์ที่เผยแพร่บน [SeedSigner.com](https://seedsigner.com/keybase.txt) หากตัวระบุเหล่านี้ตรงกันอย่างสมบูรณ์ คุณสามารถมั่นใจได้ว่าคีย์นั้นเป็นของโครงการจริง หากมีข้อสงสัย ให้หยุดทันทีและขอความช่วยเหลือจากชุมชน SeedSigner (Telegram, X, GitHub...)



เมื่อคีย์ได้รับการตรวจสอบแล้ว คุณจำเป็นต้องตรวจสอบว่าไฟล์ภาพไม่ได้เสียหาย ในการทำเช่นนี้ ให้ใช้คำสั่งต่อไปนี้ใน PowerShell :



```
CertUtil -hashfile seedsigner_os.0.8.6.[your-Pi-model].img SHA256
```



ตัวอย่างสำหรับ Raspberry Pi Zero 2 (อย่าลืมแก้ไขคำสั่งตามเวอร์ชันของคุณ ที่นี่ `0.8.6.`):



```
CertUtil -hashfile seedsigner_os.0.8.6.pi02w.img SHA256
```



![Image](assets/fr/012.webp)



PowerShell จะคำนวณค่าแฮช SHA256 ของไฟล์ภาพของคุณ จากนั้นเปรียบเทียบค่าแฮชนี้กับค่าที่สอดคล้องกันใน `seedsigner.0.8.6.sha256.txt`




- หากค่าสองค่ามีความเหมือนกันอย่างเคร่งครัด การตรวจสอบจะสำเร็จและคุณสามารถดำเนินการต่อได้
- หากแตกต่างกัน ไฟล์นั้นเสียหายหรือถูกทำลาย อย่าใช้มัน และเริ่มดาวน์โหลดใหม่อีกครั้ง



![Image](assets/fr/013.webp)



การยืนยันที่สำเร็จรับรองว่าไฟล์ `.img` ของคุณเป็นของแท้ (ลงนามโดย SeedSigner) และไม่ถูกแก้ไข (ไม่มีการดัดแปลง) จากนั้นคุณสามารถดำเนินการไปยังขั้นตอนถัดไปได้อย่างปลอดภัย



### 2.4. Flash the image

## 3. การประกอบ SeedSigner



เมื่อคุณได้เตรียมและแฟลชการ์ด microSD ด้วยซอฟต์แวร์ SeedSigner แล้ว คุณสามารถดำเนินการประกอบขั้นสุดท้ายได้ ใช้เวลาให้เต็มที่ เนื่องจากบางส่วนมีความเปราะบาง (โดยเฉพาะผ้าปูโต๊ะ กล้อง และขา GPIO)



### 3.1 การเตรียมที่อยู่อาศัย



ก่อนอื่น ให้เปิดเคสของคุณ ตรวจสอบให้แน่ใจว่ามันสะอาดและไม่มีเศษพลาสติกจากการพิมพ์ 3 มิติขวางทางยึดภายใน ระวัง:




- ตำแหน่งกล้อง (รูวงกลมเล็กที่ด้านหน้า)
- การเปิดหน้าจอ
- ช่องตัดสำหรับพอร์ต micro-USB และช่องใส่ microSD ของ Raspberry Pi Zero



### 3.2 การติดตั้งกล้อง



ค้นหาตัวเชื่อมต่อริบบิ้นกล้องบน Raspberry Pi Zero: มันเป็นแถบสีดำบาง ๆ ที่ด้านข้างของบอร์ด ซึ่งสามารถยกขึ้นเล็กน้อยเพื่อเปิด ยกขึ้นอย่างระมัดระวังโดยไม่ต้องบังคับ: มันควรจะเอียงเพียงไม่กี่มิลลิเมตรเท่านั้น



![Image](assets/fr/015.webp)



ใส่ฝาครอบกล้อง ส่วนที่เป็นสีน้ำตาล/ทองแดงควรหันลงด้านล่าง ตรวจสอบให้แน่ใจว่าได้ติดตั้งอย่างแน่นหนาในตัวเชื่อมต่อ โดยไม่บิดเบือน



![Image](assets/fr/016.webp)



ปิดแถบสีดำเพื่อล็อคผ้าปูโต๊ะ (คุณจะรู้สึกถึงการคลิกเล็กน้อย) ตรวจสอบอย่างเบามือว่ามันอยู่ในตำแหน่งและไม่ขยับ



![Image](assets/fr/017.webp)



จากนั้นวางโมดูลกล้องในช่องที่เหมาะสมในตัวเรือน ขึ้นอยู่กับรุ่น อาจจะสามารถกดเข้าไปได้โดยตรงหรืออาจต้องใช้แถบกาวเล็กๆ เพื่อยึดให้อยู่กับที่ เลนส์ต้องจัดแนวอย่างสมบูรณ์ โดยหันออกด้านนอก



### 3.3 การติดตั้ง Raspberry Pi Zero



หากคุณใช้เคส ให้ใส่บอร์ด Raspberry Pi Zero เข้าไปข้างใน จัดพอร์ตให้ตรงกับช่องเปิดที่มีให้



จากนั้นวางหน้าจอ Waveshare ไว้ด้านบนของ Raspberry Pi Zero ขาของ GPIO ของ Pi ควรจะตรงกับตัวเชื่อมต่อเพศหญิงของหน้าจออย่างสมบูรณ์แบบ กดหน้าจอลงบนขาอย่างช้าๆ โดยใช้แรงกดที่เท่ากันในแต่ละด้านเพื่อหลีกเลี่ยงการงอของขา



![Image](assets/fr/018.webp)



หากคุณมีเคส ให้ประกอบให้เสร็จโดยการเพิ่มแผงด้านหน้าและจอยสติ๊ก



สุดท้าย ใส่การ์ด microSD ที่มีซอฟต์แวร์ที่แฟลชแล้วลงในช่องที่ติดตั้งอยู่ขอบของ Raspberry Pi Zero ตรวจสอบให้แน่ใจว่ามันคลิกเข้าที่



### 3.4 การเริ่มต้นครั้งแรก



เชื่อมต่อสายไฟ micro-USB เข้ากับพอร์ตเฉพาะ รอประมาณหนึ่งนาที โลโก้ SeedSigner ควรปรากฏขึ้น ตามด้วยหน้าจอหลัก



![Image](assets/fr/019.webp)



ในการเริ่มต้น ให้ตรวจสอบว่าชิ้นส่วนต่างๆ ทำงานอย่างถูกต้องโดยไปที่เมนู `Settings > I/O test`



![Image](assets/fr/020.webp)



ทดสอบปุ่มทั้งหมดและตรวจสอบให้แน่ใจว่าตอบสนองอย่างถูกต้อง จากนั้นคลิกที่ปุ่ม `KEY1` เพื่อตรวจสอบว่ากล้องทำงานตามที่คาดไว้หรือไม่ ซึ่งจะถ่ายภาพ



![Image](assets/fr/021.webp)



### 3.5 การปรับกล้อง



ขึ้นอยู่กับวิธีที่คุณติดตั้ง SeedSigner ของคุณ กล้องอาจแสดงภาพกลับหัวได้ หากต้องการแก้ไข ให้ไปที่ `การตั้งค่า > ขั้นสูง > การหมุนกล้อง` และเลือกการหมุน 180° หากจำเป็น



![Image](assets/fr/022.webp)



หากคุณได้เปลี่ยนการวางแนวของกล้องหรือประสงค์จะเปลี่ยนการตั้งค่าอื่น ๆ (เช่น ภาษาของอินเทอร์เฟซ) ในภายหลัง คุณจะต้องเปิดใช้งานการคงอยู่ของการตั้งค่าบน microSD มิฉะนั้น การตั้งค่าของคุณจะกลับไปเป็นค่าเริ่มต้นทุกครั้งที่คุณรีบูต เนื่องจาก Raspberry Pi Zero ไม่มีหน่วยความจำถาวร



ในการทำเช่นนี้ ให้เปิดเมนู `Settings > Persistent settings` จากนั้นเลือก `Enabled`.



![Image](assets/fr/023.webp)



หากทุกอย่างทำงานได้ดี SeedSigner ของคุณพร้อมใช้งานแล้ว!



## 4. การตั้งค่า SeedSigner



ก่อนที่จะสร้าง Bitcoin wallet ของคุณ ให้ตั้งค่า SeedSigner ก่อน เนื่องจากมันทำงานบน Raspberry Pi Zero ที่ไม่มีหน่วยความจำถาวร การตั้งค่าของมันจะไม่ถูกบันทึกโดยอัตโนมัติ เว้นแต่คุณจะบันทึกลงใน microSD card ดังนั้นให้แน่ใจว่าคุณได้เปิดใช้งานตัวเลือกนี้แล้ว มิฉะนั้นการตั้งค่าเหล่านี้จะหายไปเมื่อรีบูต (ดูขั้นตอน 3.5)



### 4.1 การเข้าถึงเมนูพารามิเตอร์



เริ่มต้น SeedSigner ของคุณและรอให้หน้าจอหลักปรากฏขึ้น ใช้จอยสติ๊กนำทางไปยังตัวเลือก `Settings` จากนั้นยืนยันโดยกดปุ่มตรงกลาง ตอนนี้คุณเข้าสู่เมนูการตั้งค่าหลัก

### 4.2 การเลือกซอฟต์แวร์การจัดการ wallet



จากนั้นเข้าถึงเมนู `Coordinator software`



![Image](assets/fr/025.webp)



`Coordinator` หมายถึงซอฟต์แวร์การจัดการ wallet ที่ SeedSigner ของคุณจะสื่อสารผ่านรหัส QR ซอฟต์แวร์นี้ติดตั้งได้ทั้งบนคอมพิวเตอร์หรือสมาร์ทโฟนของคุณ มันจะช่วยให้คุณจัดการบิตคอยน์ของคุณได้ แต่จะไม่มีการเข้าถึงกุญแจส่วนตัวของคุณ SeedSigner ยังคงเป็นอุปกรณ์เดียวที่สามารถลงนามในธุรกรรมของคุณได้



เฟิร์มแวร์เวอร์ชันปัจจุบันรองรับโปรแกรมหลายตัว: Sparrow, Specter, BlueWallet, Nunchuk และ Keeper ในกรณีของฉัน ฉันใช้ **Sparrow Wallet** ซึ่งฉันแนะนำเป็นพิเศษสำหรับความเรียบง่ายและฟังก์ชันการทำงานที่หลากหลาย



หากคุณไม่ทราบวิธีการติดตั้ง คุณสามารถทำตามบทแนะนำนี้ได้:

- เครือข่าย gW-17`: ให้แก้ไขเฉพาะเมื่อคุณต้องการใช้ SeedSigner บน Testnet.
- ความหนาแน่นของรหัส qR`: ปรับปริมาณข้อมูลที่บรรจุในแต่ละรหัส QR คุณสามารถปล่อยค่าเริ่มต้นไว้ได้ เว้นแต่คุณจะพบว่ามันยากที่จะอ่านเมื่อสแกน
- `Xpub export`: เปิดหรือปิดการส่งออกคีย์สาธารณะที่ขยายของคุณ (`xpub`, `ypub`, `zpub`) ไปยังซอฟต์แวร์การจัดการ wallet ผ่านรหัส QR (ฟังก์ชันที่เราจะใช้ในภายหลัง ดังนั้นให้เปิดใช้งานไว้ในตอนนี้)
- `Script types`: กำหนดประเภทสคริปต์ที่อนุญาตให้ล็อคบิตคอยน์ของคุณ คุณไม่จำเป็นต้องแก้ไขพารามิเตอร์นี้ เนื่องจากประเภทสคริปต์จะถูกตั้งค่าโดยตรงเป็น Sparrow ที่นี่จะเกี่ยวข้องเฉพาะสคริปต์ที่ SeedSigner ได้รับอนุญาตให้จัดการเท่านั้น

## 5. การสร้างและบันทึก seed



seed (หรือวลีช่วยจำ) เป็นพื้นฐานของ Bitcoin wallet ของคุณ มันถูกใช้ในการสร้างคีย์ส่วนตัวและที่อยู่ของคุณ และให้การเข้าถึงเงินทุนของคุณ SeedSigner มีวิธีการหลายอย่างในการสร้างมัน ซึ่งเราจะพิจารณาในส่วนนี้



ก่อนที่เราจะเริ่มต้น มีข้อเตือนใจสำคัญบางประการ:




- วลีนี้ให้การเข้าถึงบิตคอยน์ของคุณอย่างเต็มที่และไม่มีข้อจำกัด** ใครก็ตามที่มีวลีนี้สามารถขโมยเงินของคุณได้ แม้ไม่มีการเข้าถึงทางกายภาพไปยัง SeedSigner ของคุณ ;
- โดยปกติแล้ว วลี 12 คำจะถูกใช้เพื่อกู้คืน wallet ในกรณีที่สูญหายหรือถูกขโมยฮาร์ดแวร์ wallet แต่เนื่องจาก SeedSigner เป็นอุปกรณ์ *stateless* มันจึงไม่เคยลงทะเบียน seed ของคุณ ดังนั้นการสำรองข้อมูลทางกายภาพของคุณจึงไม่ใช่แค่สำเนาสำรอง แต่เป็น **วิธีเดียวในการใช้ wallet ของคุณ** หากคุณสูญเสียการสำรองข้อมูลเหล่านี้ บิตคอยน์ของคุณจะสูญหายอย่างถาวร ดังนั้นให้สำรองข้อมูลอย่างระมัดระวัง บนสื่อหลายชนิดและในที่ปลอดภัย
- หากคุณเพิ่งเริ่มต้น ฉันขอแนะนำอย่างยิ่งให้คุณอ่านบทแนะนำนี้เพื่อทำความเข้าใจรายละเอียดเกี่ยวกับความเสี่ยงที่เกี่ยวข้องกับการจัดการวลีช่วยจำ :

### 5.1 การเข้าถึงเครื่องมือสร้าง seed



จากหน้าจอหลัก SeedSigner ไปที่เมนู `Tools`



![Image](assets/fr/029.webp)



คุณจะทำการ generate กับ seed ของคุณ seed เป็นเพียงแค่ตัวเลขสุ่มขนาดใหญ่ ยิ่งสร้างแบบสุ่มมากเท่าไหร่ ก็ยิ่งปลอดภัยมากขึ้นเท่านั้น SeedSigner มีวิธีการทำเช่นนี้สองวิธี:




- กล้อง": seed ถูกสร้างขึ้นจากสัญญาณรบกวนทางภาพของภาพถ่าย คุณถ่ายภาพของสภาพแวดล้อมแบบสุ่ม (วัตถุ, ภูมิทัศน์, ใบหน้า, ฯลฯ) ซึ่งความแปรปรวนของพิกเซลถูกใช้เพื่อสร้าง generate entropy เป็นวิธีที่รวดเร็ว แต่ไม่สามารถทำซ้ำได้

### 5.6 ความเข้าใจในแนวคิดของ "อุปกรณ์ไร้สถานะ"



SeedSigner เป็นอุปกรณ์ที่ไม่มีหน่วยความจำถาวร ซึ่งหมายความว่า seed ของคุณจะไม่ถูกเก็บไว้ในอุปกรณ์ (ต่างจาก Ledger, Trezor หรือ Coldcard เป็นต้น) ทันทีที่คุณปิดเครื่อง seed จะหายไปจาก RAM ของมันอย่างสมบูรณ์ เมื่อคุณเริ่มต้นใหม่ SeedSigner จะกลับสู่สถานะว่างเปล่า: คุณจะต้องให้ seed ของคุณอีกครั้ง เพื่อให้มันสามารถลงนามในธุรกรรมของคุณได้



สิ่งนี้ให้การป้องกันที่จำเป็น แตกต่างจากกระเป๋าเงินฮาร์ดแวร์อื่น ๆ SeedSigner มีพื้นฐานมาจาก Raspberry Pi Zero ซึ่งไม่มีการป้องกันทางกายภาพ รวมถึง *Secure Element* แต่เนื่องจากไม่มีการจัดเก็บข้อมูลที่ละเอียดอ่อน แม้ว่าอุปกรณ์จะถูกบุกรุกทางกายภาพ ผู้โจมตีก็ไม่สามารถดึงกุญแจส่วนตัวของคุณหรือใช้จ่ายบิตคอยน์ของคุณได้



ในทางกลับกัน สถาปัตยกรรมนี้หมายถึงความรับผิดชอบเพิ่มเติม: หากไม่มีการสำรองข้อมูล เงินของคุณจะสูญหายอย่างถาวร นั่นคือเหตุผลที่ฉันแนะนำให้มี **การสำรองข้อมูลสองชั้น** คุณมีวลีการกู้คืนของคุณแล้ว: นี่คือการสำรองข้อมูลระยะยาวหลักของคุณ ควรเก็บไว้ในที่ปลอดภัย ตอนนี้เราจะสร้างสำเนาของวลีนี้ในรูปแบบของ **QR code**



ทุกครั้งที่คุณใช้ SeedSigner คุณจะสแกน QR โค้ดนี้ด้วยกล้องของอุปกรณ์เพื่อให้มันโหลด seed ของคุณเข้าสู่หน่วยความจำชั่วคราวในขณะที่คุณเซ็นชื่อธุรกรรมของคุณ การสำรองข้อมูลครั้งที่สองนี้ ซึ่งมีไว้สำหรับการใช้งานในชีวิตประจำวัน จะต้องเก็บรักษาอย่างระมัดระวังที่สุด: ใครก็ตามที่ครอบครอง QR โค้ดนี้จะสามารถเข้าถึงบิตคอยน์ของคุณได้อย่างเต็มที่


ฉันยังแนะนำให้คุณเก็บ QR code และวลีช่วยจำของคุณไว้ในสองสถานที่แยกกัน เพื่อหลีกเลี่ยงการสูญเสียทุกอย่างในกรณีที่มีการเรียกร้อง.



ในที่สุด ทางเลือกที่ก้าวหน้าและปลอดภัยยิ่งขึ้นคือการใช้ SeedSigner ร่วมกับ **SeedKeeper** ซึ่งเก็บ seed ไว้ใน secure element หากต้องการทราบข้อมูลเพิ่มเติม โปรดดูที่บทแนะนำนี้:



https://planb.academy/tutorials/wallet/hardware/seedkeeper-seedsigner-45cca4c4-1f22-46bb-87ae-9cddb68aa579

### 5.8 การบันทึก seed ในรูปแบบ QR (*SeedQR*)



SeedSigner ช่วยให้คุณแปลง seed ของคุณให้เป็น QR code บนกระดาษที่เรียกว่า *SeedQR* วิธีนี้ช่วยให้การโหลด wallet ของคุณง่ายขึ้น เนื่องจากไม่ต้องพิมพ์แต่ละคำด้วยตนเองอีกต่อไป



ในการทำเช่นนี้ คุณจะต้องมีแผ่นกระดาษเปล่าหรือ QR code โลหะที่สอดคล้องกับความยาวของวลีช่วยจำของคุณ หากคุณซื้อชุดแพ็คเกจครบชุดสำหรับ SeedSigner ของคุณ โดยปกติแล้วจะมีแม่แบบรวมอยู่ด้วย หากไม่ คุณสามารถดาวน์โหลดและพิมพ์ออกมา (หรือทำซ้ำด้วยมือ) ได้ที่นี่ :

## 6. นำเข้า wallet ไปยัง Sparrow



เมื่อ SeedSigner ของคุณถูกตั้งค่าเรียบร้อยแล้วและ seed ของคุณถูกสร้างและบันทึกอย่างถูกต้อง ขั้นตอนต่อไปคือการเชื่อมโยง wallet นี้กับซอฟต์แวร์การจัดการเช่น Sparrow Wallet seed ของคุณจะยังคงออฟไลน์เสมอ เนื่องจากมีเพียงส่วนสาธารณะของ wallet ของคุณเท่านั้นที่จะถูกส่งไปยัง Sparrow ซึ่งจะทำให้ซอฟต์แวร์สามารถแสดงที่อยู่ของคุณ การทำธุรกรรม และสร้างธุรกรรมใหม่ โดยไม่สามารถใช้จ่ายบิตคอยน์ของคุณได้เลย ในการใช้จ่ายบิตคอยน์ของคุณ SeedSigner ของคุณจะต้องเซ็นชื่อในธุรกรรมที่เตรียมโดย Sparrow เสมอ

สิ่งนี้จะนำคุณไปยังเมนูสำหรับ wallet ของคุณ ในกรณีของฉันชื่อ `d4149b27` หากคุณกลับไปที่หน้าหลัก ให้เลือก `Seeds` จากนั้นเลือกการพิมพ์ที่ตรงกับ wallet ของคุณ จากนั้นคลิกที่ `Export Xpub`.



![Image](assets/fr/048.webp)



เลือกประเภท wallet ในกรณีของเรา มันคือ wallet เดี่ยว: เลือก `Single Sig`.



![Image](assets/fr/049.webp)



ถัดไปคือการเลือกมาตรฐานการเขียนสคริปต์ มาตรฐานล่าสุดและประหยัดที่สุดในแง่ของต้นทุนการทำธุรกรรมคือ `Taproot` ดังนั้นฉันขอแนะนำให้คุณเลือกมาตรฐานนี้



![Image](assets/fr/050.webp)



ข้อความเตือนจะปรากฏขึ้น นี่เป็นเรื่องปกติ: คีย์สาธารณะที่ขยายออกนี้ (`xpub`) ช่วยให้คุณสามารถดูที่อยู่ทั้งหมดที่ได้มาจาก seed ของคุณ (ในบัญชีแรก) มันไม่อนุญาตให้คุณใช้จ่ายเงินของคุณ แต่จะเปิดเผยโครงสร้างของ wallet ของคุณ หากมันรั่วไหล มันจะเป็นปัญหาสำหรับความเป็นส่วนตัวของคุณ แต่ไม่ใช่สำหรับความปลอดภัยของบิตคอยน์ของคุณ: มันช่วยให้คุณเห็นพวกมัน แต่ไม่สามารถใช้จ่ายได้



คลิก `I Understand` จากนั้น `Export Xpub` หากคุณพอใจกับข้อมูลที่แสดง



SeedSigner จะสร้าง xpub ของคุณในรูปแบบของ QR code แบบไดนามิกที่มีข้อมูลทั้งหมดที่คุณต้องการเพื่อจัดการ wallet ของคุณใน Sparrow Wallet



![Image](assets/fr/051.webp)



คุณสามารถใช้จอยสติ๊กเพื่อปรับความสว่างของหน้าจอให้ง่ายต่อการสแกน QR โค้ด



### 6.2 การนำเข้า wallet ใหม่เข้าสู่ Sparrow Wallet



ตรวจสอบให้แน่ใจว่าคุณได้ติดตั้งซอฟต์แวร์ Sparrow Wallet บนคอมพิวเตอร์ของคุณแล้ว หากคุณไม่ทราบวิธีการดาวน์โหลด ตรวจสอบและติดตั้งให้ถูกต้อง ดูบทแนะนำฉบับเต็มของเราในหัวข้อนี้:



https://planb.academy/tutorials/wallet/desktop/sparrow-c674e2ac-d46f-4c82-92a7-7d1b0e262f5d

บนคอมพิวเตอร์ของคุณ เปิด Sparrow Wallet จากนั้นในแถบเมนู คลิก `File → Import Wallet`.



![Image](assets/fr/052.webp)



เลื่อนลงไปที่ `SeedSigner` จากนั้นเลือก `Scan...` เว็บแคมของคุณจะเปิดขึ้น: สแกน QR โค้ดแบบไดนามิกที่แสดงบนหน้าจอ SeedSigner ของคุณ



![Image](assets/fr/053.webp)



กำหนดชื่อให้กับ wallet ของคุณ จากนั้นคลิกที่ `Create Wallet` Sparrow จะขอให้คุณตั้งรหัสผ่านเพื่อป้องกันการเข้าถึงในพื้นที่ของ wallet นี้ เลือกรหัสผ่านที่แข็งแกร่ง: มันจะปกป้องการเข้าถึงข้อมูล wallet ของคุณใน Sparrow (กุญแจสาธารณะ, ที่อยู่, ป้ายกำกับ และประวัติการทำธุรกรรม) รหัสผ่านนี้ไม่จำเป็นสำหรับการกู้คืน wallet ในภายหลัง: มีเพียงวลีช่วยจำของคุณ (และอาจจะเป็น passphrase ของคุณ) ที่จำเป็นสำหรับวัตถุประสงค์นี้



ฉันขอแนะนำให้คุณบันทึกรหัสผ่านนี้ในตัวจัดการรหัสผ่านเพื่อหลีกเลี่ยงการสูญหาย



![Image](assets/fr/054.webp)



ที่เก็บคีย์ของคุณได้ถูกนำเข้าเรียบร้อยแล้ว



![Image](assets/fr/055.webp)



จากนั้นตรวจสอบว่า `Master fingerprint` ที่แสดงใน Sparrow ตรงกับที่บันทึกไว้ก่อนหน้านี้ใน SeedSigner



SeedSigner และ Sparrow Wallet ของคุณเชื่อมโยงกันอย่างปลอดภัยแล้ว Sparrow ทำหน้าที่เป็นอินเทอร์เฟซการจัดการที่สมบูรณ์ ในขณะที่ SeedSigner ยังคงเป็นอุปกรณ์เดียวที่สามารถลงนามธุรกรรมของคุณได้ ตอนนี้คุณพร้อมที่จะรับและส่งบิตคอยน์ในรูปแบบที่แยกออกจากกันโดยสิ้นเชิงแล้ว



## 7. การรับและส่งบิตคอยน์



SeedSigner และ Sparrow Wallet ของคุณได้รับการกำหนดค่าให้ทำงานร่วมกันแล้ว ในส่วนสุดท้ายนี้ เราจะมาดูวิธีการรับและส่งบิตคอยน์โดยใช้การกำหนดค่านี้



### 7.1 การรับบิตคอยน์



#### 7.1.1 การสร้างที่อยู่รับสัญญาณ



บนคอมพิวเตอร์ของคุณ เปิด Sparrow Wallet และปลดล็อก SeedSigner wallet โดยใช้รหัสผ่านของคุณ ตรวจสอบให้แน่ใจว่าซอฟต์แวร์เชื่อมต่อกับเซิร์ฟเวอร์ (สัญลักษณ์ที่มุมล่างขวา) ในแถบด้านข้าง คลิกที่ `Receive`.



![Image](assets/fr/056.webp)



มีการแสดงที่อยู่ Bitcoin ใหม่ คุณจะเห็น :




- ที่อยู่ข้อความ (เริ่มต้นด้วย `bc1p...` ถ้าคุณใช้ P2TR เหมือนที่ฉันใช้)
- รหัส QR ที่สอดคล้องกัน,
- ฟิลด์ `Label` สำหรับติดตามธุรกรรมของคุณ



ฉันขอแนะนำอย่างยิ่งให้คุณเพิ่มป้ายกำกับให้กับใบเสร็จรับเงินบิตคอยน์แต่ละใบใน wallet ของคุณ สิ่งนี้จะช่วยให้คุณสามารถระบุแหล่งที่มาของแต่ละ UTXO ได้อย่างง่ายดายและปรับปรุงการจัดการความเป็นส่วนตัวของคุณ หากต้องการเจาะลึกในหัวข้อสำคัญนี้ คุณสามารถตรวจสอบการฝึกอบรมเฉพาะทางใน Plan ₿ Academy :



https://planb.academy/courses/65c138b0-4161-4958-bbe3-c12916bc959c

หากต้องการเพิ่มป้ายกำกับ เพียงป้อนชื่อในช่อง `Label` แล้วกดยืนยัน



ตัวอย่างเช่น:



```txt
Label : Sale of the Raspberry Pi Zero
```



ที่อยู่ของคุณได้ถูกเชื่อมโยงกับป้ายนี้ในทุกส่วนของ Sparrow แล้ว



![Image](assets/fr/057.webp)



#### 7.1.2 การตรวจสอบ Address บน SeedSigner



ก่อนที่จะแชร์ที่อยู่รับของคุณ สิ่งสำคัญมากคือต้องตรวจสอบว่ามันเป็นของ seed ของคุณ ขั้นตอนนี้จะช่วยให้แน่ใจว่า SeedSigner ของคุณจะสามารถลงนามในธุรกรรมที่เกี่ยวข้องกับที่อยู่นี้ได้ นอกจากนี้ยังป้องกันการโจมตีที่อาจเกิดขึ้นซึ่ง Sparrow แสดงที่อยู่ปลอม อย่าลืมว่า Sparrow ทำงานในสภาพแวดล้อมที่ไม่ปลอดภัย (คอมพิวเตอร์ของคุณ) ซึ่งมีพื้นผิวการโจมตีที่ใหญ่กว่ามากเมื่อเทียบกับ SeedSigner ของคุณซึ่งถูกแยกออกอย่างสมบูรณ์ นั่นคือเหตุผลที่คุณไม่ควรเชื่อที่อยู่รับที่แสดงบน Sparrow โดยไม่ตรวจสอบกับฮาร์ดแวร์ wallet ของคุณก่อน



บน Sparrow คลิกที่ QR โค้ดของที่อยู่เพื่อขยาย: จากนั้นมันจะแสดงผลเต็มหน้าจอ



![Image](assets/fr/058.webp)



บน SeedSigner ของคุณ จากเมนูหลัก เลือก `Scan` สแกน QR code ที่แสดงบนหน้าจอคอมพิวเตอร์ของคุณ จากนั้นเลือก seed ที่ตรงกับ wallet ของคุณ (ในกรณีของฉันคือ ลายนิ้วมือ `d4149b27`)



![Image](assets/fr/059.webp)



หากที่อยู่ที่สแกนตรงกับที่ได้จาก seed หน้าจอ SeedSigner จะแสดงข้อความ: `Address Verified`.



![Image](assets/fr/060.webp)



ข้อความนี้ยืนยันว่าที่อยู่นี้เป็นของ wallet ของคุณและคุณสามารถรับบิตคอยน์จากที่อยู่นี้ได้อย่างมั่นใจ



#### 7.1.3 การรับเงิน



คุณสามารถสื่อสารที่อยู่นี้ (ในรูปแบบข้อความหรือ QR code) ไปยังบุคคลหรือแผนกที่ต้องการส่ง satss ให้คุณได้ เมื่อการทำธุรกรรมถูกกระจายบนเครือข่ายแล้ว มันจะแสดงในแท็บ `Transactions` ของ Sparrow Wallet



![Image](assets/fr/061.webp)



### 7.2 ส่งบิตคอยน์



การส่งบิตคอยน์ด้วย SeedSigner เป็นกระบวนการ 3 ขั้นตอน:




- การสร้างธุรกรรมใน Sparrow ;
- ลายเซ็นของธุรกรรมบน SeedSigner ;
- การกระจายขั้นสุดท้ายของธุรกรรมผ่าน Sparrow.



การแลกเปลี่ยนทั้งหมดระหว่างอุปกรณ์ทั้งสองทำโดยใช้ QR codes เท่านั้น



#### 7.2.1 การสร้างธุรกรรมใน Sparrow



ใน Sparrow Wallet คุณสามารถคลิกที่แท็บ `Send` ในแถบด้านข้างซ้าย อย่างไรก็ตาม ฉันชอบใช้แท็บ `UTXOs` ซึ่งช่วยให้คุณฝึก "*Coin Control*" วิธีนี้ให้คุณควบคุม UTXOs ที่ใช้ได้อย่างแม่นยำ ดังนั้นคุณสามารถควบคุมข้อมูลที่คุณเปิดเผยระหว่างการทำธุรกรรมได้



ในแท็บ `UTXOs` ให้เลือกเหรียญที่คุณต้องการใช้จ่าย จากนั้นคลิก `Send Selected`.



![Image](assets/fr/062.webp)



จากนั้นกรอกข้อมูลในช่องธุรกรรม:




- ใน `Pay to` วางที่อยู่ของผู้รับหรือคลิกที่ไอคอนกล้องเพื่อสแกน QR โค้ด;
- ใน `Label` ให้เพิ่มป้ายกำกับเพื่อติดตามค่าใช้จ่ายนี้;
- ใน `Amount` ป้อนจำนวนเงินที่จะส่ง;
- สุดท้ายนี้ เลือกอัตราค่าธรรมเนียมตามสภาพตลาดปัจจุบัน (สามารถดูประมาณการได้ที่ [mempool.space](https://mempool.space/))



เมื่อกรอกข้อมูลในช่องต่างๆ เสร็จแล้ว ให้ตรวจสอบข้อมูลอย่างละเอียด จากนั้นคลิกที่ `Create Transaction >>`.



![Image](assets/fr/063.webp)



ตรวจสอบรายละเอียดการทำธุรกรรมเพื่อให้แน่ใจว่าทุกอย่างถูกต้อง จากนั้นคลิกที่ `Finalize Transaction for Signing`.



![Image](assets/fr/064.webp)



ธุรกรรมพร้อมแล้ว แต่ยังไม่ได้ลงนาม หากต้องการแสดง [PSBT (*Partially Signed Bitcoin Transaction*)](https://planb.academy/en/resources/glossary/psbt) เป็นรหัส QR ให้คลิกที่ `Show QR`.



![Image](assets/fr/065.webp)



#### 7.2.2 การลงนามธุรกรรมด้วย SeedSigner



เปิด SeedSigner ของคุณและสแกน SeedQR ของคุณเพื่อเข้าถึง wallet ตามปกติ จากหน้าจอหลัก เลือก `Scan` จากนั้นสแกนรหัส QR ที่แสดงบน Sparrow



![Image](assets/fr/066.webp)



จากนั้นเลือก seed ให้ตรงกับ wallet ของคุณ



![Image](assets/fr/067.webp)



SeedSigner ตรวจจับโดยอัตโนมัติว่านี่คือ PSBT และแสดงสรุปของธุรกรรม:




   - จำนวนเงินที่ส่ง,
   - เอาต์พุตที่อยู่
   - ต้นทุนการทำธุรกรรมที่เกี่ยวข้อง



คลิกที่ `Review Details` และตรวจสอบข้อมูลทั้งหมดอย่างละเอียดบนหน้าจอ SeedSigner รายการที่สำคัญที่สุดที่ต้องตรวจสอบคือจำนวนเงินที่ส่ง ที่อยู่ของผู้รับ และจำนวนค่าธรรมเนียมที่เรียกเก็บ



![Image](assets/fr/068.webp)



หากทุกอย่างถูกต้อง ให้เลือก `Approve PSBT` เพื่อเซ็นชื่อธุรกรรมโดยใช้คีย์ส่วนตัวที่เกี่ยวข้อง



![Image](assets/fr/069.webp)



เมื่อเซ็นแล้ว SeedSigner จะสร้างรหัส QR ใหม่ที่มีธุรกรรมที่เซ็นแล้ว พร้อมที่จะถูกสแกนโดย Sparrow



![Image](assets/fr/070.webp)



#### 7.2.3 การกระจายธุรกรรมจาก Sparrow



ตอนนี้ที่ธุรกรรมถูกต้องแล้ว จำเป็นต้องกระจายไปยังเครือข่าย Bitcoin เพื่อให้ถึงนักขุดที่จะเพิ่มมันลงในบล็อก



บน Sparrow คลิกที่ `QR Scan`.



![Image](assets/fr/071.webp)



แสดง QR code ที่แสดงโดย SeedSigner ของคุณ (ซึ่งเป็นของธุรกรรมที่ลงนามแล้ว) ไปที่เว็บแคม Sparrow จะถอดรหัสลายเซ็นและแสดงรายละเอียดธุรกรรมทั้งหมด ตรวจสอบครั้งสุดท้ายว่าข้อมูลทั้งหมดถูกต้อง จากนั้นคลิกที่ Broadcast Transaction เพื่อกระจายมันบนเครือข่าย Bitcoin



![Image](assets/fr/072.webp)



ธุรกรรมของคุณได้ถูกส่งไปยังเครือข่าย Bitcoin แล้ว คุณสามารถติดตามความคืบหน้าได้ในแท็บ `Transactions` ของ Sparrow Wallet



![Image](assets/fr/073.webp)



คุณได้เชี่ยวชาญพื้นฐานของการใช้ SeedSigner แล้ว เพื่อเพิ่มพูนความรู้และสำรวจการใช้งานขั้นสูงเพิ่มเติม ฉันขอเชิญคุณศึกษาบทแนะนำต่อไปนี้:
