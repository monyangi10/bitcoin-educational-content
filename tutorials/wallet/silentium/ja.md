Bitcoinアドレスの再利用は、ユーザーの機密性に対する最も直接的な脅威の一つである。受取人が複数の支払いを受けるために1つのアドレスを共有する場合、観察者であれば誰でも、関連するすべての取引を追跡し、その財務履歴を再構築することができる。この問題は、特に、プライバシーを損なうことなく寄付アドレスを公に表示したいコンテンツ制作者、慈善団体、活動家に影響する。



Silentiumは、ブラウザから直接アクセスできるエレガントなソリューションでこの問題に対応します。このオープンソースのプログレッシブ・ウェブ・アプリケーション（PWA）は、ルイ・シンガーによって2024年5月に発表され、サイレント・ペイメント（BIP-352）を実装している。再利用可能な静的アドレスで、各支払いは別のblockchainアドレスで終わり、事前のやり取りや取引間の観察可能なリンクはない。



**重要なお知らせSilentiumはサイレント・ペイメンツの軽量walletのコンセプト実証のための実験的プロジェクトです。Silentiumは、サイレント・ペイメンツの軽量walletの概念実証のための実験的プロジェクトです。開発者はこう明言しています：



> 自己責任で使用すること。

なお、このwalletはテストネットとしてもレグテストとしても使用できる。



## Silentiumとは？



### 理念と目標



Silentiumは、軽量なwalletブラウザにサイレント・ペイメントを実装するための技術的なデモンストレーションの役割を果たす。従来のBitcoinアドレスもサポートしているが、ユーザーがこのプライバシー技術をわかりやすく試すことができるよう、サイレント・ペイメントに重点を置いている。



### サイレント・ペイメントの仕組み



サイレント・ペイメント(BIP-352)は楕円曲線ディフィー・ヘ ルマン鍵Exchange(ECDH)を使用する。受信者は2つの公開鍵からなる静的アドレス(mainnetでは `sp1...`、testnetでは `tsp1...`)を生成する。



送信者は、自分の入力秘密鍵と受信者のスキャン鍵を組み合わせて、暗号的な「微調整」を生成する共有秘密を計算する。この微調整を支出鍵に加えることで、各取引に固有のTaprootアドレスが作成される。受信者はこの計算を自分のプライベート・スキャン・キーで再現し、事前のやりとりなしに資金を検出して使用する。



利点：送信者と受信者の機密性が向上、サードパーティ・サーバーが不要、従来のTaproot決済と見分けがつかない取引。主な欠点：支払いを検知するためにblockchainを集中的にスキャンすること。



サイレント・ペイメントの理論的な仕組みについては、BTC,204コースのPlan ₿ Academyの最後の部分をご覧ください：



https://planb.academy/courses/65c138b0-4161-4958-bbe3-c12916bc959c

## 対応プラットフォーム



Silentiumは、あらゆるモダンブラウザ（モバイルまたはデスクトップ）からアクセス可能なプログレッシブWebアプリ（PWA）です。app.silentium.dev`で直接使用したり、ブラウザ経由でネイティブアプリケーションとしてインストールしたり、ローカルにデプロイしたりできます。インストールは、公式ストアを経由せずにブラウザから直接行います。



## ウェブアプリの使用



### アクセスと設置



[ブラウザから`https://app.silentium.dev/`にアクセス](https://app.silentium.dev/)。アプリケーションは即座にロードされ、ホーム画面が表示されます。



iOSでネイティブアプリケーションとしてインストールするには、共有ボタン（四角に上向き矢印）を押し、「ホーム画面に追加」を選択します。Androidの場合、通常はブラウザから直接「ホーム画面に追加」の通知が表示されます。インストールされると、Silentiumは専用のアイコンで表示され、ネイティブアプリケーションとして動作しますが、取引を同期するにはインターネット接続が必要です。



![Installation de Silentium comme PWA sur iOS](assets/fr/01.webp)



### ポートフォリオの作成



初回起動時に、新しいwalletを生成する場合は「Walletを作成」、既存のリカバリフレーズから復元する場合は「Walletを復元」を選択します。



Walletを作成」を選択する。アプリケーションは、あなたが注意深く書き留めなければならない12語のフレーズを生成します。このフレーズが資金を回復する唯一の方法です。テストネット上であっても、バックアップはしっかりと行ってください。フレーズを保存したら「Continue」を押してください。



アプリケーションは、walletへのアクセスを保護するためのパスワードを設定するよう求めます。強力なパスワードを選択し、確認してください。



![Création d'un nouveau wallet avec phrase de récupération](assets/fr/02.webp)



フレーズが確認され、パスワードが設定されると、メイン・インターフェースに移動します。



### Interface 主要諸元



メイン・インターフェースは、残高をsatoshi（最初は0sats）で表示し、下部に3つのボタンがある：




- 同期**：walletとblockchainを同期させる
- Receive**：資金を受け取る
- Send**：ビットコインを送る



右上のアイコン（3本の横棒）から「設定」にアクセスします。設定メニューにはいくつかのオプションがあります：





- について**: 募集要項
- バックアップ**：回復フレーズのバックアップ
- エクスプローラ**：blockchainエクスプローラ（デフォルトはMempool）とSilentiumdサーバを選択する。
- ネットワーク**：ネットワーク選択（mainnet/testnet）
- パスワード**: パスワードの変更
- リロード**：walletの再装填
- リセット**：完全なリセット
- テーマ**：テーマの変更



![Interface principale et paramètres avec Explorer](assets/fr/03.webp)



Explorer**セクションは特に重要で、使用するblockchainエクスプローラ（デフォルトはMempool）を選択し、SilentiumdサーバのURLを表示します（mainnetの場合は`https://bitcoin.silentium.dev/v1`）。Silentiumdサーバーをホストしている場合、またはtestnetを使用したい場合は、ここでこれらのパラメータを設定します。



### 資金の受け取り



メイン・インターフェースから「受信」ボタンを押す。デフォルトでは、SilentiumはQRコードとともにサイレントペイメントのアドレスを表示する。アドレスはmainnetでは`sp1...`、testnetでは`tsp1...`で始まる。



画面下部の「クラシックアドレスに切り替える」/「サイレントアドレスに切り替える」ボタンで、サイレント決済とクラシックBitcoinアドレスを切り替えることができます。



![Réception de fonds avec Silent Payment et adresse classique](assets/fr/04.webp)




トランザクションがブロードキャストされたら、数分間お待ちください。サイレント・ペイメントでは、Silentiumが自動的にblockchainをスキャンし、お客様宛の取引を確認します。取引は「未確認」というステータスで表示され、その後徐々に確認されます。



### 支払いを送る



メイン・インターフェースから「送信」ボタンを押す。送信画面が表示されます：



1. **Address**: サイレント・ペイメント・アドレス(`sp1...`または`tsp1...`)またはクラシックBitcoinアドレスを貼り付ける。QRスキャンアイコンを使ってアドレスをスキャンできます。


2. **金額**：送信する金額をsatoshiで入力します。テンキーで簡単に入力できます。上部に利用可能残高が表示されますので、参考にしてください。



![Envoi de fonds depuis Silentium](assets/fr/05.webp)



住所と金額を入力したら、"Proceed "を押して次に進みます。取引を確定する前に、アプリケーションから希望の手数料レベルを選択するよう求められます。



## walletセルフ・ホスティング



### なぜセルフホストなのか？



Silentiumのローカル・ホスティングは、完全な主権、コード検証、開発環境、そして公式サイトの障害に対する回復力を提供します。



### 前提条件



Node.js（バージョン14以上）、npmまたはyarn、Git、500MB程度のディスクスペース。



### 現地設置



リポジトリをクローンし、.NET Frameworkをインストールする：



```bash
git clone https://github.com/louisinger/silentium.git
cd silentium
yarn install
```



### 打ち上げと使用



開発モードでアプリケーションを起動する：



```bash
yarn start
```



ブラウザで`http://localhost:3000`にアクセスしてください。最適化された製品版については：



```bash
yarn build
```



build/`に生成されたファイルは、nginx、Apache、または任意のウェブサーバで提供することができる。デフォルトでは、Silentium は公開されている `bitcoin.silentium.dev` サーバーに接続します。testnetまたは独自のサーバーを使用する場合は、パラメータでこの設定を変更してください。



## Silentiumdサーバー



### 役割と運営



Silentiumは、**Silentiumd**インデックスサーバーを使用し、支払い検出を最適化します。Taprootのすべてのトランザクションをスキャンすることは、ブラウザや携帯電話では面倒すぎる。



SilentiumdはTaprootの各トランザクションの中間データ（Tweak）を事前に計算する。walletはこれらの微調整データ（トランザクションごとに数バイト）をダウンロードし、最終的な計算をローカルで行い、支払いの所有権を確認する。従来のElectrumサーバーとは異なり、サーバーはあなたのキーを知ることも、あなたのトランザクションを特定することもできない。



コンパクトなBIP158フィルターにより、walletはアドレスを明かすことなくスキャンするブロックを決定することができ、機密性を保持することができます。



### 公開サーバー



Vulpem Venturesがスポンサーを務める公開サーバー`bitcoin.silentium.dev`（mainnet）は、シンプルで即時性の高い体験を提供する。機密性は保持されるが、このアプローチはサードパーティのインフラに対する相対的な信頼を意味する。



### あなた自身のSilentiumdサーバーをホストする



完全な主権を得るには、あなた自身のSilentiumdサーバーをホストしてください。前提条件txindex=1`と`blockfilterindex=1`を持つBitcoin Coreの非エラグノード、Go 1.21+、10-20 GBのディスクスペース、システム管理スキル。



*インストール：***。



```bash
git clone https://github.com/louisinger/silentiumd.git
cd silentiumd
make build
./build/silentium-[OS]-[ARCH]
```



環境変数を使って設定する（詳細はリポジトリの `config.md` を参照）。サーバーはblockchainのインデックスを作成し、walletから問い合わせ可能なAPIを公開する。



現在、UmbrelやStart9のパッケージ・ソリューションは存在しないため、非技術系ユーザーへのアクセスが制限されている。



## 利点と限界



### ハイライト





- 最大限のアクセシビリティ**：どのブラウザからでも使用でき、重いインストールは不要
- マルチプラットフォーム**：PWAテクノロジーにより、モバイル（Android/iOS）とデスクトップで動作します。
- シンプルなセルフホスティング**：いくつかのコマンドでローカルにインストール可能
- オープンソース**：GitHub上で完全に監査可能なコード
- プライバシー重視**: 追跡なし、分析なし、ローカル暗号計算
- セパレート・アーキテクチャ**：wallet（クライアント）とインデックス・サーバーの明確な分離
- アカウントなし**：登録も個人情報も不要



### 考慮すべき制約





- 実験的プロジェクト**：コンセプトの実証のみ。
- 保証なし**：バグや脆弱性のリスク、資金損失の可能性
- 限定的なサポート**：ユーザードキュメントが少ない、コミュニティが小さい、公式サポートがない
- サーバー依存**：Silentiumdサーバー（パブリックまたはセルフホスト）が必要です。
- 集中スキャン**：サイレント・ペイメント検出で帯域幅を消費
- 機能低下**：コインコントロール、ライトニング、multi-signatureなし



## ベストプラクティス



### seedの安全性



テストネットでも、リカバリーフレーズを真剣に扱うこと。それを書き留め、安全な場所に保管してください。テストネット用とmainnet用のwalletは別々に保管すること。実資金用のwalletにテスト用のseedを決して使用しないこと。



### ソースコードの検証



セルフホスティングの利点の一つは、実行する前にソースコードをチェックできることです。実際の資金でSilentiumを使用する予定がある場合は、時間をかけてコードを監査するか、信頼できる開発者に監査してもらいましょう。また、`app.silentium.dev`にデプロイされたコードのhashとGitHubリポジトリのコードを比較し、信憑性を確認してください。



### バックアップと復元



サイレント・ペイメントの資金回収には、BIP-352プロトコルと互換性のあるwalletが必要です。標準的なwalletでは、blockchainをスキャンしてUTXOサイレント・ペイメントを回収することはできません。Silentiumをインストールしておくか、必要に応じて他の互換性のあるwallet（Cake Walletまたは他の将来の実装など）にアクセスして資金を復元できるようにしてください。



## 結論



Silentiumは、技術的な摩擦なしにサイレント・ペイメントを理解するための利用しやすい試験場を提供します。コンセプトの実証として、このプライバシー技術をwalletブラウザーに統合し、セルフ・カストディを維持する方法を示しています。testnetで実験して、on-chainプライバシーのためのこの有望なブレークスルーを発見してください。
