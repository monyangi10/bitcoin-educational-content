Повторное использование адресов Bitcoin - одна из самых прямых угроз конфиденциальности пользователей. Когда получатель использует один адрес для получения нескольких платежей, любой наблюдатель может отследить все связанные с ним транзакции и восстановить его финансовую историю. Эта проблема особенно затрагивает создателей контента, благотворительные организации или активистов, которые хотят публично показать адрес для пожертвований, не нарушая при этом конфиденциальности.



Silentium отвечает на эту проблему элегантным решением, доступным прямо из браузера. Это прогрессивное веб-приложение с открытым исходным кодом (PWA), запущенное в мае 2024 года Луисом Сингером, реализует Silent Payments (BIP-352): многоразовый статический адрес, где каждый платеж попадает на отдельный адрес блокчейна, без предварительного взаимодействия или наблюдаемой связи между транзакциями.



**Важное предупреждение**: Silentium - это экспериментальный проект, служащий в качестве *доказательства концепции* для легких кошельков Silent Payments. Его не следует использовать в качестве ежедневного wallet или для хранения значительных сумм. Разработчики недвусмысленно заявляют:



> Используйте на свой страх и риск.

Обратите внимание, что этот wallet можно использовать в качестве тестовой сети или регтеста.



## Что такое Silentium?



### Философия и цели



Silentium служит технической демонстрацией реализации Silent Payments в легком браузере wallet. Хотя он также поддерживает обычные адреса Bitcoin, акцент сделан на Silent Payments, чтобы дать пользователям возможность экспериментировать с этой технологией конфиденциальности в простой форме.



### Как работают "тихие платежи"?



Silent Payments (BIP-352) используют эллиптический кривой ключ Диффи-Хеллмана Exchange (ECDH). Получатель генерирует статический адрес (`sp1...` на mainnet, `tsp1...` на testnet), состоящий из двух открытых ключей: ключа сканирования для обнаружения платежей и ключа траты для их использования.



Отправитель объединяет свои закрытые ключи ввода с ключом сканирования получателя для вычисления общего секрета, генерирующего криптографический "твик". Этот твик, добавленный к ключу траты, создает уникальный адрес Taproot для каждой транзакции. Получатель воспроизводит это вычисление с помощью своего закрытого ключа сканирования, чтобы обнаружить и потратить средства без какого-либо предварительного взаимодействия.



Преимущества: повышенная конфиденциальность для отправителя и получателя, не требуется сторонний сервер, транзакции неотличимы от обычных платежей Taproot. Основной недостаток: интенсивное сканирование блокчейна для обнаружения платежей.

## Поддерживаемые платформы



Silentium - это прогрессивное веб-приложение (PWA), доступное из любого современного браузера (мобильного или настольного). Используйте его непосредственно на `app.silentium.dev`, установите как нативное приложение через браузер или разверните его локально. Установка осуществляется прямо из браузера, без обращения к официальным магазинам.



## Использование веб-приложения



### Доступ и установка



[Перейдите на `https://app.silentium.dev/` из вашего браузера](https://app.silentium.dev/). Приложение мгновенно загружается и отображает главный экран.



Чтобы установить его как родное приложение на iOS, нажмите кнопку поделиться (квадрат со стрелкой вверх) и выберите "На домашний экран". На Android браузер обычно предлагает уведомление "Добавить на домашний экран" напрямую. После установки Silentium отображается в виде специального значка и работает как родное приложение, но требует подключения к интернету для синхронизации транзакций.



![Installation de Silentium comme PWA sur iOS](assets/fr/01.webp)



### Создание портфолио



При первом запуске выберите "Создать Wallet" для создания нового портфеля generate или "Восстановить Wallet" для восстановления из существующей фразы восстановления.



Выберите "Создать Wallet". Приложение генерирует фразу из 12 слов, которую вы должны тщательно записать. Эта фраза - единственный способ восстановить ваши средства. Даже в тестовой сети используйте хорошие методы резервного копирования. После сохранения фразы нажмите "Продолжить".



Затем приложение попросит вас установить пароль для защиты доступа к wallet. Выберите надежный пароль и подтвердите его.



![Création d'un nouveau wallet avec phrase de récupération](assets/fr/02.webp)



После подтверждения фразы и ввода пароля вы попадете в основной интерфейс.



### Основные и параметры Interface



Основной интерфейс отображает ваш баланс в сатоши (изначально 0 sats), а внизу расположены три кнопки:




- Sync**: синхронизирует wallet с блокчейном
- Получать**: получать средства
- Отправить**: отправить биткоины



Войдите в Настройки с помощью значка в правом верхнем углу (три горизонтальные полоски). Меню "Настройки" предлагает несколько опций:





- О**: информация о применении
- Резервная копия**: резервная копия фразы восстановления
- Проводник**: выберите проводник блокчейна (по умолчанию Mempool) и сервер Silentiumd
- Сеть**: выбор сети (mainnet/testnet)
- Пароль**: изменить пароль
- Перезарядка**: перезарядка wallet
- Сброс**: полный сброс
- Тема**: изменить тему



![Interface principale et paramètres avec Explorer](assets/fr/03.webp)



Раздел **Explorer** особенно важен: он позволяет выбрать используемый блокчейн-проводник (по умолчанию Mempool), а также отображает URL-адрес сервера Silentiumd (`https://bitcoin.silentium.dev/v1` для mainnet). Если вы размещаете свой собственный сервер Silentiumd или хотите использовать testnet, здесь вы можете настроить эти параметры.



### Получение средств



В главном интерфейсе нажмите кнопку "Получить". По умолчанию Silentium отображает адрес Silent Payment с его QR-кодом. Адрес начинается с `sp1...` на mainnet или `tsp1...` на testnet.



Вы можете переключаться между "Тихим платежом" и классическими адресами Bitcoin с помощью кнопки "Переключиться на классический адрес" / "Переключиться на тихий адрес" в нижней части экрана.



![Réception de fonds avec Silent Payment et adresse classique](assets/fr/04.webp)




После того как транзакция будет передана, пожалуйста, подождите несколько минут. Для Silent Payments Silentium автоматически сканирует блокчейн на предмет транзакций, предназначенных для вас. Транзакции появляются со статусом "Неподтвержденные", а затем постепенно подтверждаются.



### Отправить платеж



В главном интерфейсе нажмите кнопку "Отправить". На экране отправки появится запрос:



1. **Address**: вставьте адрес "Тихой оплаты" (`sp1...` или `tsp1...`) или классический адрес Bitcoin. Для сканирования адреса можно использовать значок QR-сканирования.


2. **Сумма**: введите сумму в сатоши, которую необходимо отправить. Для удобства ввода отображается цифровая клавиатура. Ваш доступный баланс отображается вверху для справки.



![Envoi de fonds depuis Silentium](assets/fr/05.webp)



После ввода адреса и суммы нажмите "Приступить", чтобы продолжить. Перед подтверждением транзакции приложение попросит вас выбрать желаемый уровень комиссии.



## Самостоятельный хостинг wallet



### Почему именно хостинг?



Локальный хостинг Silentium обеспечивает полный суверенитет, проверку кода, среду разработки и устойчивость к сбоям в работе официального сайта.



### Пререквизиты



Node.js (версия 14+), npm или yarn, Git и около 500 МБ дискового пространства.



### Локальная установка



Клонируйте репозиторий и установите файл :



```bash
git clone https://github.com/louisinger/silentium.git
cd silentium
yarn install
```



### Запуск и использование



Запустите приложение в режиме разработки:



```bash
yarn start
```



Перейдите по адресу `http://localhost:3000` в вашем браузере. Для оптимизированной производственной версии :



```bash
yarn build
```



Файлы, созданные в `build/`, можно обслуживать с помощью nginx, Apache или любого другого веб-сервера. По умолчанию Silentium подключается к публичному серверу `bitcoin.silentium.dev`. Измените эту настройку в параметрах, чтобы использовать testnet или свой собственный сервер.



## Сервер Silentiumd



### Роль и функционирование



Silentium использует индексирующий сервер **Silentiumd** для оптимизации обнаружения платежей. Сканирование всех транзакций Taproot было бы слишком громоздким для браузера или мобильного телефона.



Silentiumd предварительно вычисляет промежуточные данные (твики) для каждой транзакции Taproot. Ваш wallet загружает эти твики (несколько байт на транзакцию) и выполняет окончательный расчет локально, проверяя право собственности на платеж. В отличие от обычных серверов Electrum, сервер никогда не знает ваших ключей и не может идентифицировать ваши транзакции.



Компактные фильтры BIP158 позволяют вашему wallet определять блоки для сканирования без раскрытия ваших адресов, что сохраняет конфиденциальность.



### Публичный сервер



Публичный сервер `bitcoin.silentium.dev` (mainnet), спонсируемый Vulpem Ventures, предлагает простой и непосредственный опыт. Хотя конфиденциальность сохраняется, такой подход подразумевает относительное доверие к сторонней инфраструктуре.



### Разместите свой собственный сервер Silentiumd



Для полного суверенитета разместите свой собственный сервер Silentiumd. Необходимые условия: Нода Bitcoin Core без лагов с `txindex=1` и `blockfilterindex=1`, Go 1.21+, 10-20 Гб дискового пространства, навыки системного администрирования.



**Установка:*



```bash
git clone https://github.com/louisinger/silentiumd.git
cd silentiumd
make build
./build/silentium-[OS]-[ARCH]
```



Настройте через переменные окружения (подробнее см. в файле `config.md` репозитория). Сервер индексирует блокчейн и выдает API, к которому может обращаться ваш wallet.



В настоящее время не существует готовых решений для Umbrel и Start9, что ограничивает их доступность для нетехнических пользователей.



## Преимущества и ограничения



### Основные моменты





- Максимальная доступность**: использование из любого браузера, не требуется установка тяжелых систем
- Мультиплатформенность**: работает на мобильных (Android/iOS) и настольных компьютерах благодаря технологии PWA
- Простой хостинг**: локальная установка возможна с помощью нескольких команд
- Открытый исходный код**: полностью проверяемый код на GitHub
- Ориентированность на конфиденциальность**: никакого слежения, никакой аналитики, локальные криптографические вычисления
- Раздельная архитектура**: четкое разделение между wallet (клиентом) и сервером индексирования
- Без аккаунта**: не требуется регистрация или личные данные



### Ограничения, которые необходимо учитывать





- Экспериментальный проект**: только доказательство концепции, не предназначен для ежедневного использования или производства
- Никаких гарантий**: риск ошибок, уязвимостей, возможной потери средств
- Ограниченная поддержка**: мало пользовательской документации, небольшое сообщество, отсутствие официальной поддержки
- Зависимость от сервера**: требуется работающий сервер Silentiumd (публичный или самостоятельно размещенный)
- Интенсивное сканирование**: Бесшумное обнаружение платежей потребляет полосу пропускания
- Урезанная функциональность**: нет контроля монет, нет Lightning, нет мультиподписей



## Лучшие практики



### Безопасность seed



Даже в тестовой сети относитесь к фразе восстановления серьезно. Запишите ее и храните в надежном месте. Держите отдельные кошельки для testnet и mainnet: никогда не используйте тестовый seed на wallet, предназначенном для реальных средств.



### Проверка исходного кода



Одно из преимуществ самостоятельного хостинга - возможность проверить исходный код перед запуском. Если вы планируете использовать Silentium с реальными средствами, потратьте время на аудит кода или поручите это проверенному разработчику. Также сравните хэш кода, размещенного на `app.silentium.dev`, с хэшем репозитория GitHub, чтобы убедиться в подлинности.



### Резервное копирование и восстановление



Для восстановления средств Silent Payments требуется wallet, совместимый с протоколом BIP-352. Стандартный wallet не может просканировать блокчейн, чтобы получить ваши UTXO Silent Payments. Держите Silentium установленным или убедитесь, что у вас есть доступ к другому совместимому wallet (например, Cake Wallet или другим будущим реализациям), чтобы восстановить свои средства в случае необходимости.



## Заключение



Silentium представляет собой доступную тестовую площадку для понимания Silent Payments без технических сложностей. В качестве доказательства концепции он демонстрирует, как эта технология конфиденциальности может быть интегрирована в браузер wallet при сохранении самоохраны. Экспериментируйте на testnet, чтобы узнать об этом многообещающем прорыве в области конфиденциальности on-chain.
