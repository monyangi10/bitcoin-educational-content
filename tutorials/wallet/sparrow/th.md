Sparrow Wallet เป็นซอฟต์แวร์จัดการ Bitcoin wallet ที่ผู้ใช้เก็บรักษาเอง พัฒนาโดย Craig Raw ซอฟต์แวร์โอเพนซอร์สนี้ได้รับการชื่นชมจากผู้ใช้บิตคอยน์เนื่องจากมีคุณสมบัติมากมายและ Interface ที่ใช้งานง่าย


มีสองวิธีในการใช้ Sparrow:




- เหมือนกับ wallet ร้อนแรง ที่เก็บคีย์ส่วนตัวของคุณไว้ในพีซีของคุณ
- ในฐานะผู้จัดการ wallet แบบเย็น ที่ซึ่งคีย์ส่วนตัวถูกเก็บไว้บน Hardware Wallet ในโหมดนี้ Sparrow จะจัดการเฉพาะข้อมูลสาธารณะของ wallet เท่านั้น ติดตามเงินทุน สร้างที่อยู่ และสร้างธุรกรรม แต่ต้องการลายเซ็นของ Hardware Wallet เพื่อทำให้ธุรกรรมเหล่านี้ถูกต้อง ดังนั้นจึงสามารถแทนที่แอปพลิเคชันเช่น Ledger Live หรือ Trezor Suite ได้


Sparrow รองรับกระเป๋าเงินแบบลายเซ็นเดียวและหลายลายเซ็น และช่วยให้การจัดการกระเป๋าเงินหลายใบเป็นไปอย่างราบรื่น ตัวอย่างเช่น คุณสามารถควบคุม wallet หนึ่งตัวที่เชื่อมต่อกับ Ledger อีกตัวหนึ่งกับ Trezor และยังมี wallet แบบร้อนได้พร้อมกัน


ซอฟต์แวร์ยังมีฟีเจอร์การควบคุมเหรียญขั้นสูง ช่วยให้คุณเลือก UTXOs ที่จะใช้ในธุรกรรมของคุณได้อย่างแม่นยำเพื่อเพิ่มความเป็นส่วนตัวของคุณให้สูงสุด


ในแง่ของการเชื่อมต่อ Sparrow ให้คุณเชื่อมต่อกับโหนด Bitcoin ของคุณเอง ไม่ว่าจะจากระยะไกลผ่าน Electrum Server หรือด้วย Bitcoin Core นอกจากนี้ยังสามารถใช้โหนดสาธารณะได้หากคุณยังไม่มีของตัวเอง การเชื่อมต่อระยะไกลทำผ่าน Tor

## กำลังเชื่อมต่อกับเครือข่าย Bitcoin


ในการโต้ตอบกับเครือข่าย Bitcoin และกระจายธุรกรรมของคุณ Sparrow จะต้องเชื่อมต่อกับโหนด Bitcoin มีสามวิธีหลักในการสร้างการเชื่อมต่อนี้:



- 🟡 การใช้โหนดสาธารณะ เช่น การเชื่อมต่อกับโหนดของบุคคลที่สามที่อนุญาตให้มีการเชื่อมต่อเช่นนี้ หากคุณไม่มีโหนด Bitcoin ของคุณเอง ตัวเลือกนี้จะช่วยให้คุณเริ่มต้นกับ Sparrow ได้อย่างรวดเร็ว อย่างไรก็ตาม โหนดที่คุณเชื่อมต่อจะเห็นธุรกรรมทั้งหมดของคุณ ซึ่งอาจทำให้ความลับของคุณถูกเปิดเผย การควบคุมกุญแจของคุณเป็นสิ่งสำคัญ แต่การมีโหนดของคุณเองจะดียิ่งขึ้น ดังนั้นใช้ตัวเลือกนี้เฉพาะเมื่อคุณเพิ่งเริ่มต้น โดยตระหนักถึงความเสี่ยงต่อความเป็นส่วนตัวของคุณ
- 🟢 กำลังเชื่อมต่อกับโหนด Bitcoin Core หากคุณมีโหนด Bitcoin Core ของคุณเอง คุณสามารถเชื่อมต่อมันกับ Sparrow Wallet ได้ทั้งในเครื่องเดียวกันหากติดตั้ง Bitcoin Core ไว้บนเครื่องเดียวกัน หรือจากระยะไกล
- 🔵 การเชื่อมต่อผ่านเซิร์ฟเวอร์ Electrum หากโหนด Bitcoin ของคุณติดตั้ง Electrs เช่นเดียวกับโซลูชัน node-in-a-box เช่น Umbrel หรือ Start9 คุณสามารถเชื่อมต่อกับมันจากระยะไกลผ่าน Sparrow ได้


**ควรใช้การเชื่อมต่อผ่าน Electrs หรือ Bitcoin Core บนโหนดของคุณเองเพื่อลดความจำเป็นในการไว้วางใจบุคคลที่สามและเพิ่มประสิทธิภาพความเป็นส่วนตัวของคุณ**

### กำลังเชื่อมต่อกับ Bitcoin Core 🟢


วิธีที่สองในการเชื่อมต่อกับโหนด Bitcoin คือการเชื่อมโยง Sparrow กับ Bitcoin Core หาก Bitcoin Core ติดตั้งอยู่บนเครื่องเดียวกัน การยืนยันตัวตนจะทำผ่านไฟล์คุกกี้ หาก Bitcoin Core อยู่บนเครื่องระยะไกล คุณจะต้องใช้รหัสผ่านที่กำหนดไว้ในไฟล์ `Bitcoin.conf`


โปรดทราบว่าหากคุณใช้โหนด pruned Bitcoin Core คุณจะไม่สามารถกู้คืน wallet ที่มีธุรกรรมที่เกิดขึ้นก่อนบล็อกที่เก็บไว้ในเครื่องได้ อย่างไรก็ตาม สำหรับ wallet ใหม่ที่สร้างบน Sparrow จะไม่มีปัญหานี้: ธุรกรรมใหม่ของคุณจะมองเห็นได้ แม้จะใช้โหนด pruned ก็ตาม


ในการกำหนดค่าโหนด Bitcoin Core คุณสามารถดูหนึ่งในบทแนะนำต่อไปนี้ ขึ้นอยู่กับระบบปฏิบัติการของคุณ:


https://planb.academy/tutorials/node/bitcoin/bitcoin-core-mac-windows-9684ab02-e0af-41c9-8102-86ac7c7727f3

https://planb.academy/tutorials/node/bitcoin/bitcoin-core-linux-568c13a6-8746-4d63-8e95-f4a61c5ae0ed

บน Sparrow ไปที่แท็บ "*Bitcoin Core*"


![Image](assets/fr/07.webp)


**ด้วย Bitcoin Core ท้องถิ่น:**


หากติดตั้ง Bitcoin Core บนคอมพิวเตอร์ของคุณ ให้ค้นหาไฟล์ `Bitcoin.conf` ในบรรดาไฟล์ซอฟต์แวร์ หากไฟล์นี้ไม่มีอยู่ คุณสามารถสร้างมันขึ้นมาได้ เปิดไฟล์ด้วยโปรแกรมแก้ไขข้อความและใส่บรรทัดต่อไปนี้:


```ini
server=1
```


จากนั้นบันทึกการเปลี่ยนแปลงของคุณ


คุณยังสามารถทำสิ่งนี้ผ่านกราฟิก Bitcoin-QT ของ Interface โดยไปที่ "*Settings*" > "*Options...*" และเปิดใช้งานตัวเลือก "*Enable RPC server*"


อย่าลืมรีสตาร์ทซอฟต์แวร์หลังจากทำการเปลี่ยนแปลงเหล่านี้


![Image](assets/fr/08.webp)


จากนั้นกลับไปที่ Sparrow Wallet และป้อนเส้นทางไปยังไฟล์คุกกี้ของคุณ ซึ่งมักจะอยู่ในโฟลเดอร์เดียวกับ `Bitcoin.conf` ขึ้นอยู่กับระบบปฏิบัติการของคุณ:


| **macOS** | ~/Library/Application Support/Bitcoin |
| ----------- | ------------------------------------- |
| **Windows** | %APPDATA%\Bitcoin |
| **Linux** | ~/.Bitcoin |

![Image](assets/fr/09.webp)


ปล่อยพารามิเตอร์อื่นๆ ไว้ตามค่าเริ่มต้น, URL `127.0.0.1` และพอร์ต `8332`, จากนั้นคลิกที่ "*Test Connection*".


![Image](assets/fr/10.webp)


การเชื่อมต่อได้รับการจัดตั้งขึ้นแล้ว เครื่องหมายถูกสีเขียวจะปรากฏที่มุมล่างขวาเพื่อแสดงว่าคุณเชื่อมต่อกับโหนด Bitcoin Core แล้ว


![Image](assets/fr/11.webp)


**ด้วยรีโมท Bitcoin Core:**


หากติดตั้ง Bitcoin Core บนเครื่องอื่นที่เชื่อมต่อกับเครือข่ายเดียวกัน ให้ค้นหาไฟล์ `Bitcoin.conf` ในบรรดาไฟล์ซอฟต์แวร์ก่อน หากไฟล์นี้ยังไม่มีอยู่ คุณสามารถสร้างมันขึ้นมาได้ เปิดไฟล์นี้ด้วยโปรแกรมแก้ไขข้อความและเพิ่มบรรทัดต่อไปนี้:


```ini
server=1
```


หลังจากแก้ไขไฟล์แล้ว ให้แน่ใจว่าคุณบันทึกไฟล์ในโฟลเดอร์ที่เหมาะสมสำหรับระบบปฏิบัติการของคุณ:


| **macOS** | ~/Library/Application Support/Bitcoin |
| ----------- | ------------------------------------- |
| **Windows** | %APPDATA%\Bitcoin |
| **Linux** | ~/.Bitcoin |

การดำเนินการนี้สามารถทำได้ผ่านทางอินเทอร์เฟซกราฟิก Bitcoin-QT Interface เช่นกัน ไปที่เมนู "*Settings*" จากนั้นเลือก "*Options...*" และเปิดใช้งานตัวเลือก "*Enable RPC server*" โดยการทำเครื่องหมายในช่องที่เกี่ยวข้อง หากไฟล์ `Bitcoin.conf` ไม่มีอยู่ คุณสามารถสร้างไฟล์นี้ได้โดยตรงจาก Interface โดยคลิกที่ "*Open Configuration File*".


![Image](assets/fr/12.webp)


ค้นหาที่อยู่ IP ของเครื่องที่โฮสต์ Bitcoin Core บนเครือข่ายท้องถิ่นของคุณ ในการทำเช่นนี้ คุณสามารถใช้เครื่องมือเช่น [Angry IP Scanner](https://angryip.org/) สมมติว่า เพื่อประโยชน์ในการโต้แย้ง ที่อยู่ IP ของโหนดของคุณคือ `192.168.1.18`


ในไฟล์ `Bitcoin.conf` เพิ่มบรรทัดต่อไปนี้ โดยตั้งค่า `rpcbind=192.168.1.18` ให้ตรงกับที่อยู่ IP ของโหนดของคุณ


```ini
[main]
rpcbind=127.0.0.1
rpcbind=192.168.1.18
rpcallowip=127.0.0.1
rpcallowip=192.168.1.0/24
```


![Image](assets/fr/13.webp)


ในไฟล์ `Bitcoin.conf` ให้เพิ่มชื่อผู้ใช้และรหัสผ่านสำหรับการเชื่อมต่อระยะไกล อย่าลืมแทนที่ `loic` ด้วยชื่อผู้ใช้ของคุณและ `my_password` ด้วยรหัสผ่านที่แข็งแกร่ง:


```ini
rpcuser=loic
rpcpassword=my_password
```


![Image](assets/fr/14.webp)


หลังจากแก้ไขและบันทึกไฟล์แล้ว ให้รีสตาร์ทซอฟต์แวร์ Bitcoin-QT ใหม่


คุณสามารถกลับไปที่ Sparrow Wallet ได้แล้ว ไปที่แท็บ "*User / Pass*". ป้อนชื่อผู้ใช้และรหัสผ่านที่คุณกำหนดไว้ในไฟล์ `Bitcoin.conf`. ปล่อยให้พารามิเตอร์อื่นๆ เป็นค่าเริ่มต้น เช่น URL `127.0.0.1` และพอร์ต `8332`. จากนั้นคลิกที่ "*Test Connection*".


![Image](assets/fr/15.webp)


การเชื่อมต่อได้รับการจัดตั้งขึ้นแล้ว เครื่องหมายถูกสีเขียวจะปรากฏที่มุมล่างขวาเพื่อแสดงว่าคุณเชื่อมต่อกับโหนด Bitcoin Core แล้ว


![Image](assets/fr/16.webp)

### กำลังเชื่อมต่อกับเซิร์ฟเวอร์ Electrum 🔵


ตัวเลือกสุดท้ายสำหรับการเชื่อมต่อคือการใช้เซิร์ฟเวอร์ Electrum ระยะไกล วิธีนี้ช่วยให้คุณเชื่อมต่อกับโหนดของคุณผ่าน Tor จากอุปกรณ์อื่น และใช้ประโยชน์จากตัวจัดทำดัชนีเพื่อเรียกดูวอลเล็ตของคุณบน Sparrow ได้รวดเร็วยิ่งขึ้น เหมาะอย่างยิ่งหากคุณมีโซลูชันโหนดในกล่อง เช่น Umbrel หรือ Start9 ซึ่งช่วยให้คุณติดตั้ง Electrs ได้ด้วยการคลิกเพียงครั้งเดียว


ในการทำเช่นนี้ ให้รับที่อยู่ Tor `.onion' ของเซิร์ฟเวอร์ Electrum ของคุณ สำหรับ Umbrel ตัวอย่างเช่น คุณจะพบมันในแอปพลิเคชัน Electrs


![Image](assets/fr/17.webp)


บน Sparrow Wallet เข้าถึงแท็บ "*Private Electrum*"


![Image](assets/fr/18.webp)


ป้อนที่อยู่ Tor ของคุณในช่องที่ให้ไว้ การตั้งค่าอื่นๆ สามารถคงค่าเริ่มต้นไว้ได้ จากนั้นคลิกที่ "*ทดสอบการเชื่อมต่อ*"


![Image](assets/fr/19.webp)


การเชื่อมต่อได้รับการยืนยันแล้ว หากคุณปิดหน้าต่างนี้ จะมีเครื่องหมายถูกสีน้ำเงินปรากฏขึ้นที่มุมล่างขวา แสดงว่าคุณเชื่อมต่อกับเซิร์ฟเวอร์ Electrum แล้ว

## สร้าง wallet ร้อน


ตอนนี้ที่ Sparrow Wallet ถูกกำหนดค่าให้สื่อสารกับเครือข่าย Bitcoin แล้ว คุณพร้อมที่จะสร้าง wallet ตัวแรกของคุณแล้ว ส่วนนี้จะแนะนำคุณผ่านการสร้าง wallet แบบร้อน หรือก็คือ wallet ที่มีคีย์ส่วนตัวถูกเก็บไว้ในคอมพิวเตอร์ของคุณ เนื่องจากคอมพิวเตอร์ของคุณเป็นเครื่องจักรที่ซับซ้อนและเชื่อมต่อกับอินเทอร์เน็ต มันจึงมีพื้นผิวการโจมตีที่ใหญ่มาก ดังนั้น wallet แบบร้อนควรใช้สำหรับจำนวนบิตคอยน์ที่จำกัดเท่านั้น หากต้องการเก็บจำนวนที่มากขึ้น ควรเลือกใช้ wallet ที่ปลอดภัยด้วย Hardware Wallet หากนี่คือสิ่งที่คุณกำลังมองหา คุณสามารถข้ามไปยังส่วนถัดไปได้เลย


ในการสร้าง wallet ร้อน จากหน้าจอหลัก Sparrow Wallet ให้คลิกที่แท็บ "*File*" แล้วคลิกที่ "*New Wallet*"


![Image](assets/fr/21.webp)


ป้อนชื่อสำหรับ wallet ของคุณและคลิกที่ "*Create Wallet*".


![Image](assets/fr/22.webp)


ที่ด้านบนของ Interface คุณสามารถเลือกได้ว่าจะสร้าง "*Single Signature*" หรือ "*Multi Signature*" wallet ด้านล่างให้เลือกประเภทของสคริปต์เพื่อล็อค UTXOs ของคุณ ฉันแนะนำให้คุณใช้มาตรฐานล่าสุด: "*Taproot (P2TR)*"


![Image](assets/fr/23.webp)


จากนั้นคลิกที่ "*New or Imported Software Wallet*"


![Image](assets/fr/24.webp)


เลือกมาตรฐาน BIP39 เนื่องจากได้รับการสนับสนุนจากซอฟต์แวร์ Bitcoin wallet เกือบทั้งหมด จากนั้นเลือกความยาวของวลีการกู้คืนของคุณ ปัจจุบัน วลี 12 คำก็เพียงพอแล้ว เนื่องจากทั้งสองมีความปลอดภัยที่คล้ายกัน แต่การบันทึกวลี 12 คำนั้นง่ายกว่า


![Image](assets/fr/25.webp)


คลิกที่ปุ่ม "*generate New*" เพื่อ generate วลีช่วยจำของ wallet ของคุณ วลีนี้ให้การเข้าถึงบิตคอยน์ทั้งหมดของคุณอย่างเต็มที่และไม่มีข้อจำกัด ใครก็ตามที่มีวลีนี้สามารถขโมยเงินของคุณได้ แม้ไม่มีการเข้าถึงทางกายภาพไปยังคอมพิวเตอร์ของคุณ


วลี 12 คำจะช่วยกู้คืนการเข้าถึงบิตคอยน์ของคุณในกรณีที่สูญหาย ถูกขโมย หรือคอมพิวเตอร์ของคุณเสียหาย ดังนั้นจึงเป็นสิ่งสำคัญมากที่จะต้องบันทึกไว้อย่างระมัดระวังและเก็บไว้ในที่ปลอดภัย


คุณสามารถจารึกมันลงบนกระดาษหรือเพื่อความปลอดภัยเพิ่มเติม สลักมันลงบนสแตนเลสเพื่อป้องกันจากไฟไหม้ น้ำท่วม หรือการพังทลาย การเลือกสื่อสำหรับการจดจำของคุณจะขึ้นอยู่กับกลยุทธ์ความปลอดภัยของคุณ แต่ถ้าคุณใช้ Sparrow เป็นการใช้จ่ายที่อบอุ่น wallet ที่มีปริมาณปานกลาง กระดาษก็ควรเพียงพอแล้ว


สำหรับข้อมูลเพิ่มเติมเกี่ยวกับวิธีการที่ถูกต้องในการบันทึกและจัดการวลีช่วยจำของคุณ ฉันขอแนะนำให้ติดตามบทแนะนำอื่นนี้ โดยเฉพาะอย่างยิ่งหากคุณเป็นมือใหม่:


https://planb.academy/tutorials/wallet/backup/backup-mnemonic-22c0ddfa-fb9f-4e3a-96f9-46e2a7954270

![Image](assets/fr/26.webp)


**เห็นได้ชัดว่าคุณต้องไม่แชร์คำเหล่านี้บนอินเทอร์เน็ต เช่นเดียวกับที่ฉันทำในบทแนะนำนี้ ตัวอย่าง wallet นี้จะใช้เฉพาะกับ Testnet และจะถูกลบเมื่อสิ้นสุดบทแนะนำ**


คุณยังสามารถเลือกเพิ่ม passphrase BIP39 ได้โดยคลิกที่กล่อง "*Use passphrase*". คำเตือน: การใช้ passphrase สามารถมีประโยชน์มาก แต่ถ้าคุณไม่เข้าใจวิธีการทำงาน มันอาจมีความเสี่ยงสูง นั่นคือเหตุผลที่ฉันขอแนะนำอย่างยิ่งให้คุณอ่านบทความทฤษฎีสั้น ๆ เกี่ยวกับเรื่องนี้:


https://planb.academy/tutorials/wallet/backup/passphrase-a26a0220-806c-44b4-af14-bafdeb1adce7

เมื่อคุณบันทึกคำช่วยจำและ passphrase ใดๆ ลงในสื่อทางกายภาพแล้ว ให้คลิกที่ "*ยืนยันการสำรองข้อมูล*"

ป้อนคำ 12 คำของคุณอีกครั้งเพื่อยืนยันว่าบันทึกถูกต้องแล้ว จากนั้นคลิกที่ "*Create Keystore*"


![Image](assets/fr/28.webp)


จากนั้นคลิกที่ "*Import Keystore*" เพื่อ generate คีย์ wallet ของคุณจากวลีช่วยจำ


![Image](assets/fr/29.webp)


คลิกที่ "*Apply*" เพื่อเสร็จสิ้นการสร้าง wallet


![Image](assets/fr/30.webp)


ตั้งรหัสผ่านที่แข็งแกร่งเพื่อรักษาความปลอดภัยในการเข้าถึง Sparrow wallet ของคุณ เป็นความคิดที่ดีที่จะเก็บรหัสผ่านนี้ไว้ในตัวจัดการรหัสผ่าน เพื่อที่คุณจะได้ไม่ลืม โปรดทราบว่ารหัสผ่านนี้ไม่มีส่วนในการสร้างคีย์ของคุณ มันใช้เพียงเพื่อเข้าถึง wallet ของคุณผ่าน Sparrow Wallet ดังนั้น แม้ไม่มีรหัสผ่านนี้ วลีช่วยจำของคุณก็เพียงพอที่จะเข้าถึงบิตคอยน์ของคุณจากแอปพลิเคชันที่รองรับ BIP39 ใดๆ


![Image](assets/fr/31.webp)


wallet ของคุณถูกสร้างขึ้นแล้ว คุณสามารถข้ามไปยังส่วน *การรับ Bitcoins* ของบทช่วยสอนนี้ได้หากคุณไม่วางแผนที่จะใช้ Hardware Wallet กับ Sparrow


## การจัดการอาการหวัด wallet


วิธีที่สองในการใช้ Sparrow Wallet คือการตั้งค่าให้เป็นผู้จัดการ wallet กับ Hardware Wallet ในการกำหนดค่านี้ คีย์ส่วนตัวของ Bitcoin wallet ของคุณจะยังคงอยู่บน Hardware Wallet เท่านั้น ในขณะที่ Sparrow จะเข้าถึงเฉพาะข้อมูลสาธารณะ วิธีการนี้ให้ระดับความปลอดภัยที่สูงกว่ากระเป๋าเงินร้อนที่กล่าวถึงข้างต้น เนื่องจากคีย์ส่วนตัวถูกเก็บไว้บนอุปกรณ์เฉพาะทาง ซึ่งมักจะมีชิปที่ปลอดภัย และไม่ได้เชื่อมต่อกับอินเทอร์เน็ต ดังนั้นจึงมีพื้นผิวการโจมตีที่เล็กกว่าคอมพิวเตอร์แบบดั้งเดิมมาก


มีสองวิธีหลักในการเชื่อมต่อ Hardware Wallet ของคุณกับ Sparrow:




- โดยสายเคเบิล มักใช้กับรุ่นระดับเริ่มต้น เช่น Trezor Safe 3 หรือ Ledger Nano S Plus ;
- ในโหมด Air-Gap คือไม่มีการเชื่อมต่อผ่านสายโดยตรง ผ่านการแลกเปลี่ยน MicroSD card หรือ QR code


Sparrow รองรับวิธีการสื่อสารทั้งหมดเหล่านี้และเข้ากันได้กับกระเป๋าเงินฮาร์ดแวร์ส่วนใหญ่ในตลาด


สำหรับบทแนะนำนี้ ฉันจะใช้ Ledger Nano S พร้อมสายเคเบิล แต่ขั้นตอนจะคล้ายกันในโหมด Air-Gap คุณจะพบรายละเอียดเฉพาะสำหรับ Hardware Wallet ของคุณในบทแนะนำเฉพาะบน Plan ₿ Academy


ก่อนเริ่มต้น ตรวจสอบให้แน่ใจว่า wallet ได้รับการกำหนดค่าแล้วบน Hardware Wallet ของคุณ หากคุณใช้การเชื่อมต่อแบบมีสาย ให้เชื่อมต่อกับคอมพิวเตอร์ของคุณผ่านสายเคเบิล


ในการนำเข้า "*Keystore*" (ข้อมูลสาธารณะที่จำเป็นในการจัดการ wallet) ไปยัง Sparrow Wallet ให้คลิกที่แท็บ "*File*" จากนั้นคลิกที่ "*New Wallet*"


![Image](assets/fr/32.webp)


ตั้งชื่อ wallet ของคุณและคลิกที่ "*Create Wallet*". ฉันแนะนำให้คุณใส่ชื่อของ Hardware Wallet ของคุณเพื่อให้ง่ายต่อการระบุในภายหลัง.


![Image](assets/fr/33.webp)


ที่ด้านบนของ Interface เลือกระหว่าง "*Single Signature*" หรือ "*Multi Signature*" wallet สำหรับตัวอย่างของเรา เราจะกำหนดค่า single-sig wallet


ด้านล่างนี้ เลือกประเภทสคริปต์เพื่อล็อค UTXOs ของคุณ หาก Hardware Wallet ของคุณรองรับ ฉันขอแนะนำให้คุณเลือก "*Taproot (P2TR)*"


![Image](assets/fr/34.webp)


ถัดไป ขั้นตอนจะแตกต่างกันไปตามวิธีการเชื่อมต่อของคุณ หากคุณใช้วิธี Air-Gap ให้เลือก "*Airgapped Hardware Wallet*" จากนั้นทำตามคำแนะนำเฉพาะสำหรับอุปกรณ์ของคุณ


![Image](assets/fr/35.webp)


หากคุณใช้การเชื่อมต่อผ่านสายเคเบิล เช่นเดียวกับในกรณีของฉัน ให้เลือก "*Connected Hardware Wallet*"


![Image](assets/fr/36.webp)


คลิกที่ "*Scan*" เพื่อให้ Sparrow ตรวจจับอุปกรณ์ของคุณ ตรวจสอบให้แน่ใจว่าได้เสียบและปลดล็อกแล้ว สำหรับบางรุ่น เช่น Ledger คุณจะต้องเปิดแอปพลิเคชัน "*Bitcoin*" เพื่อเปิดใช้งานการตรวจจับ


![Image](assets/fr/37.webp)


เลือก "*นำเข้า Keystore*"


![Image](assets/fr/38.webp)


คลิกที่ "*Apply*" เพื่อเสร็จสิ้นการสร้าง wallet


![Image](assets/fr/39.webp)


ตั้งรหัสผ่านที่แข็งแกร่งเพื่อรักษาความปลอดภัยในการเข้าถึง Sparrow wallet ของคุณ รหัสผ่านนี้จะปกป้องคีย์สาธารณะ ที่อยู่ และประวัติการทำธุรกรรมของคุณ เราแนะนำให้คุณบันทึกไว้ในตัวจัดการรหัสผ่าน โปรดทราบว่ารหัสผ่านนี้ไม่มีส่วนเกี่ยวข้องในการสร้างคีย์ของคุณ แม้ไม่มีรหัสผ่านนี้ คุณยังสามารถกู้คืนการเข้าถึงบิตคอยน์ของคุณด้วยการใช้มินโมนิกผ่านซอฟต์แวร์ที่รองรับ BIP39 ได้

## รับบิตคอยน์


ตอนนี้ที่ wallet ของคุณตั้งค่าอยู่บน Sparrow คุณสามารถรับบิตคอยน์ได้แล้ว เพียงแค่เข้าถึงเมนู "*Receive*"


![Image](assets/fr/42.webp)


Sparrow จะแสดงที่อยู่ที่ยังไม่ได้ใช้งานแรกใน wallet ของคุณ คุณสามารถเพิ่ม "*Label*" ให้กับที่อยู่นี้เพื่อเตือนความจำเกี่ยวกับแหล่งที่มาของ satoshis เหล่านี้ในอนาคต


![Image](assets/fr/43.webp)


หากคุณกำลังใช้ wallet ที่ร้อนอยู่ ที่อยู่ที่แสดงสามารถใช้งานได้ทันที ไม่ว่าจะโดยการคัดลอกหรือโดยการสแกน QR code ที่เกี่ยวข้อง


หากคุณใช้ Hardware Wallet สิ่งสำคัญมากคือต้องตรวจสอบที่อยู่บนหน้าจออุปกรณ์ก่อนใช้งาน สำหรับอุปกรณ์ที่ใช้สาย ให้เชื่อมต่อและปลดล็อก Hardware Wallet ของคุณ จากนั้นใน Sparrow ให้คลิกที่ "*Display Address*" ตรวจสอบให้แน่ใจว่าที่อยู่ที่แสดงบน Hardware Wallet ของคุณตรงกับที่แสดงบน Sparrow


![Image](assets/fr/44.webp)


สำหรับผู้ใช้ Hardware Wallet Air-Gap การยืนยันที่อยู่จะแตกต่างกันไปตามรุ่นของอุปกรณ์ ดูบทแนะนำ Plan ₿ Academy เฉพาะสำหรับคำแนะนำที่แม่นยำ


เมื่อการทำธุรกรรมถูกส่งออกโดยผู้ชำระเงิน คุณจะเห็นมันปรากฏในแท็บ "*Transactions*" คุณสามารถคลิกที่มันเพื่อดูรายละเอียดเพิ่มเติม เช่น txid


![Image](assets/fr/45.webp)


ในแท็บ "*Addresses*" คุณจะพบรายการที่อยู่กล่องจดหมายทั้งหมดของคุณ คุณสามารถดูได้ว่ามีการใช้งานแล้วหรือยังและมีการเพิ่มป้ายกำกับหรือไม่ ที่อยู่ "*Receive*" คือที่อยู่ที่ Sparrow แสดงเมื่อคุณคลิกที่ "*Receive*" และมีไว้สำหรับการรับชำระเงิน ที่อยู่ "*Change*" ใช้สำหรับการแลกเปลี่ยนในธุรกรรมของคุณ เช่น การกู้คืนส่วนที่ไม่ได้ใช้ของ UTXOs ในอินพุตของคุณ


![Image](assets/fr/46.webp)


แท็บ "*UTXOs*" จะแสดง UTXOs ทั้งหมดของคุณ นั่นคือเศษส่วนของบิตคอยน์ที่คุณถืออยู่ คุณสามารถดูจำนวนของแต่ละ UTXO และป้ายกำกับที่เกี่ยวข้องได้


![Image](assets/fr/47.webp)


## ส่งบิตคอยน์


ตอนนี้ที่คุณมีซาโตชิอยู่ใน wallet ของคุณแล้ว คุณยังมีตัวเลือกในการส่งบางส่วนด้วย แม้ว่าจะมีหลายวิธีในการทำเช่นนี้ แต่ฉันแนะนำให้คุณใช้เมนู "*UTXOs*" เพื่อควบคุมเหรียญที่คุณใช้จ่ายได้อย่างแม่นยำมากขึ้น (*การควบคุมเหรียญ*) แทนที่จะไปที่เมนู "*Send*" โดยตรง (แม้ว่าวิธีหลังอาจเพียงพอหากคุณเป็นมือใหม่)


![Image](assets/fr/48.webp)


เลือก UTXOs ที่คุณต้องการใช้เป็นอินพุตสำหรับธุรกรรมนี้ จากนั้นคลิกที่ "*Send Selected*". วิธีนี้ช่วยให้คุณสามารถเลือกแหล่งที่เหมาะสมที่สุดจาก UTXOs ของคุณ ตามค่าใช้จ่ายและป้ายกำกับที่ใช้เมื่อได้รับ เพื่อเพิ่มความเป็นส่วนตัวของการชำระเงินของคุณให้สูงสุด ตรวจสอบให้แน่ใจว่าผลรวมของ UTXOs ที่เลือกมีค่ามากกว่าจำนวนที่คุณต้องการส่ง


![Image](assets/fr/49.webp)


ป้อนที่อยู่ของผู้รับในช่อง "*Pay to*". คุณยังสามารถสแกนที่อยู่ด้วยเว็บแคมของคุณโดยคลิกที่ไอคอนกล้อง. ปุ่ม "*+Add*" ช่วยให้คุณชำระเงินหลายที่อยู่ในธุรกรรมเดียว.


![Image](assets/fr/50.webp)


เพิ่มป้ายกำกับให้กับธุรกรรมของคุณเพื่อเตือนความจำเกี่ยวกับวัตถุประสงค์ของมัน ป้ายกำกับนี้จะถูกเชื่อมโยงกับการแลกเปลี่ยนของคุณในที่สุดด้วย


![Image](assets/fr/51.webp)


กรอกจำนวนเงินที่จะส่งไปยังที่อยู่นี้


![Image](assets/fr/52.webp)


ปรับอัตราค่าธรรมเนียมตามสภาพตลาดปัจจุบัน คุณสามารถทำได้โดยการป้อนค่าธรรมเนียมแบบสัมบูรณ์หรือโดยการปรับอัตราค่าธรรมเนียมด้วยแถบเลื่อน


![Image](assets/fr/53.webp)


ที่ด้านล่างของ Interface คุณสามารถเลือกได้ระหว่าง "*Efficiency*" และ "*Privacy*". ในกรณีของฉัน ตัวเลือก "*Privacy*" ไม่สามารถใช้ได้ เนื่องจากฉันมีเพียง UTXO หนึ่งตัวใน wallet นี้ "*Efficiency*" สอดคล้องกับธุรกรรมแบบคลาสสิก ในขณะที่ "*Privacy*" เป็นธุรกรรมประเภท Stonewall ซึ่งเป็นโครงสร้างธุรกรรมที่เสริมความลับของคุณโดยการจำลอง mini-CoinJoin ซึ่งทำให้การวิเคราะห์เชนซับซ้อนมากขึ้น


![Image](assets/fr/54.webp)


Sparrow แสดงแผนภาพสรุปที่แสดงข้อมูลนำเข้า ข้อมูลส่งออก และค่าธรรมเนียมการทำธุรกรรมของคุณ (โปรดทราบว่าค่าธรรมเนียมไม่ใช่ข้อมูลส่งออกจริง ๆ ตรงกันข้ามกับที่แผนภาพนี้แสดง) หากคุณพอใจกับทุกอย่างแล้ว ให้คลิกที่ "*Create Transaction*"


![Image](assets/fr/55.webp)


ข้อความนี้จะนำคุณไปยังหน้าที่แสดงรายละเอียดของธุรกรรมของคุณ ตรวจสอบให้แน่ใจว่าข้อมูลทั้งหมดถูกต้อง จากนั้นคลิกที่ "*Finalize Transaction for Signing*"


![Image](assets/fr/56.webp)


สิ่งสำคัญคือต้องรักษา Sighash เริ่มต้นไว้ หากต้องการเข้าใจเหตุผล โปรดดูที่หลักสูตรการฝึกอบรมนี้ ซึ่งฉันอธิบายทุกสิ่งที่คุณจำเป็นต้องรู้เกี่ยวกับ Sighash:


https://planb.academy/courses/46b0ced2-9028-4a61-8fbc-3b005ee8d70f

บนหน้าจอถัดไป ตัวเลือกจะแตกต่างกันไปตามประเภทของ wallet ที่คุณกำลังใช้งาน:




- สำหรับ Hardware Wallet Air-Gap ให้คลิกที่ "*Show QR*" เพื่อแสดง PSBT ที่คุณสามารถเซ็นด้วยอุปกรณ์ของคุณ จากนั้นโหลด PSBT ที่เซ็นแล้วเข้าไปใน Sparrow โดยใช้ "*Scan QR*". ตัวเลือก "*Save Transaction*" ทำงานในลักษณะเดียวกัน แต่ใช้การแลกเปลี่ยนบน microSD ;
- สำหรับ wallet ที่ร้อนแรง เพียงคลิกที่ "*Sign*" และป้อนรหัสผ่าน wallet เพื่อเซ็นชื่อ ;
- สำหรับ Hardware Wallet แบบมีสาย ให้คลิกที่ "*Sign*" เพื่อส่งธุรกรรมที่ยังไม่ได้ลงนามไปยังอุปกรณ์ของคุณ


![Image](assets/fr/57.webp)


บน Hardware Wallet ของคุณ ตรวจสอบที่อยู่ของผู้รับ จำนวนเงินที่ส่ง และค่าธรรมเนียม หากทุกอย่างถูกต้อง ให้ดำเนินการลงนาม


เมื่อทำการลงนามธุรกรรมแล้ว ธุรกรรมนั้นจะปรากฏอีกครั้งใน Sparrow พร้อมที่จะถูกกระจายบนเครือข่าย Bitcoin เพื่อรวมในบล็อกต่อไป หากทุกอย่างถูกต้อง ให้คลิกที่ "*Broadcast Transaction*"


![Image](assets/fr/58.webp)


ธุรกรรมของคุณได้รับการเผยแพร่แล้วและกำลังรอการยืนยัน

## การจัดการและกำหนดค่ากระเป๋าเงินบน Sparrow


ในแท็บ "*การตั้งค่า*" คุณจะพบข้อมูลรายละเอียดเกี่ยวกับ wallet ของคุณ เช่น :



- ประเภทพอร์ตโฟลิโอ (single-sig หรือ multi-sig) ;
- ประเภทของสคริปต์ที่ใช้ ;
- ชื่อที่คุณได้กำหนดให้กับ wallet ;
- รอยประทับกุญแจหลัก;
- เส้นทางเลี่ยง ;
- คีย์สาธารณะขยายของบัญชี


![Image](assets/fr/60.webp)


ปุ่ม "*Export*" ช่วยให้คุณสามารถส่งออกข้อมูล wallet ของคุณเพื่อให้คุณสามารถใช้ในซอฟต์แวร์อื่นได้ในขณะที่ยังคงรักษาการตั้งค่าข้อมูลใน Sparrow ไว้


ปุ่ม "*Add Account*" ช่วยให้คุณเพิ่มบัญชีเพิ่มเติมใน wallet ของคุณ บัญชีหนึ่งสอดคล้องกับชุดที่อยู่กล่องจดหมายแยกต่างหาก ฟีเจอร์นี้มีประโยชน์ เช่น หากคุณต้องการแยกบัญชีส่วนตัวและบัญชีธุรกิจ โดยใช้วลีช่วยจำเดียวกัน


ปุ่ม "*ขั้นสูง*" ให้การเข้าถึงการตั้งค่าขั้นสูง เช่น การปรับแต่งการค้นหาที่อยู่ของ Sparrow และการเปลี่ยนรหัสผ่านของ wallet


![Image](assets/fr/61.webp)


เมื่อคุณปิด Sparrow Wallet, wallet ของคุณจะล็อคโดยอัตโนมัติ ครั้งต่อไปที่คุณเปิดซอฟต์แวร์ จะมีหน้าต่างแจ้งให้คุณปลดล็อค wallet ด้วยรหัสผ่านของมัน


![Image](assets/fr/62.webp)


หากหน้าต่างนี้ไม่เปิด หรือหากคุณต้องการเปิด wallet อีกอันบน Sparrow ให้คลิกที่แท็บ "*File*" และเลือก "*Open Wallet*".


![Image](assets/fr/63.webp)


สิ่งนี้จะเปิดตัวจัดการไฟล์ของคุณไปยังโฟลเดอร์ที่ Sparrow เก็บกระเป๋าเงินของคุณไว้ เพียงเลือก wallet ที่คุณต้องการเปิดและป้อนรหัสผ่านเพื่อปลดล็อก


![Image](assets/fr/64.webp)


ในเมนู "*ไฟล์*" ภายใต้ "*การตั้งค่า*", คุณจะพบพารามิเตอร์การเชื่อมต่อเครือข่าย Bitcoin ที่ได้สำรวจไว้ในส่วนก่อนหน้าแล้ว คุณยังสามารถปรับพารามิเตอร์ต่างๆ เช่น หน่วยที่ใช้, สกุลเงินสำหรับการแปลง, และแหล่งข้อมูลได้อีกด้วย


![Image](assets/fr/65.webp)


แท็บ "*View*" มีตัวเลือกการปรับแต่งและการเข้าถึงคำสั่งที่มีประโยชน์บางอย่าง เช่น "*Refresh Wallet*" ซึ่งจะรีเฟรชการค้นหาธุรกรรมสำหรับ wallet ของคุณ


![Image](assets/fr/66.webp)


แท็บ "*เครื่องมือ*" รวมเครื่องมือขั้นสูงหลายอย่างไว้ด้วยกัน รวมถึง :



- "*Sign/Verify Message*" ช่วยให้คุณพิสูจน์การครอบครองที่อยู่รับหรือยืนยันลายเซ็นได้
- "*Send To Many*" เสนอ Interface ที่ง่ายขึ้นสำหรับการทำธุรกรรมไปยังที่อยู่ผู้รับหลายแห่งพร้อมกัน ซึ่งสะดวกสำหรับการใช้จ่ายแบบกลุ่ม
- "*กวาดคีย์ส่วนตัว*" ช่วยให้คุณสามารถดึงบิตคอยน์ที่ถูกเก็บรักษาโดยคีย์ส่วนตัวแบบง่ายและโอนไปยัง Sparrow wallet ของคุณได้ สิ่งนี้อาจมีประโยชน์เป็นพิเศษสำหรับผู้ที่มีบิตคอยน์ตั้งแต่ช่วงต้นปี 2010 ก่อนยุคของกระเป๋าเงิน HD
- "Verify Download" ตรวจสอบความสมบูรณ์และความถูกต้องของซอฟต์แวร์ที่ดาวน์โหลดก่อนที่จะติดตั้งลงในอุปกรณ์ของคุณ
- "*Restart In*" ช่วยให้คุณสลับไปยังวอลเล็ตของคุณบนเครือข่าย Testnet หรือ Signet ได้ ซึ่งอาจมีประโยชน์หากคุณต้องการเข้าถึงเครือข่ายทดสอบที่มีเหรียญที่ไม่มีมูลค่าจริง


![Image](assets/fr/67.webp)


ตอนนี้คุณรู้ทุกอย่างเกี่ยวกับซอฟต์แวร์ Sparrow Wallet ซึ่งเป็นเครื่องมือที่ยอดเยี่ยมสำหรับการจัดการกระเป๋าเงิน Bitcoin ของคุณในแต่ละวัน


หากคุณพบว่าบทแนะนำนี้มีประโยชน์ ฉันจะรู้สึกขอบคุณมากหากคุณจะกดนิ้วโป้งสีเขียวด้านล่างนี้ อย่าลังเลที่จะแบ่งปันบนเครือข่ายสังคมของคุณ ขอบคุณมาก!
