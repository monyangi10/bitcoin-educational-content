wallet หลายลายเซ็น (มักเรียกว่า "*Multisig*") เป็นโครงสร้าง Bitcoin wallet ที่ต้องการลายเซ็นเข้ารหัสหลายรายการจากคีย์ต่าง ๆ เพื่ออนุมัติการใช้จ่าย แตกต่างจาก wallet แบบดั้งเดิม ("*singlesig*") ที่คีย์ส่วนตัวเพียงคีย์เดียวก็เพียงพอที่จะปลดล็อก UTXO ได้ Multisig จะขึ้นอยู่กับโมเดล **m-of-n**: จากคีย์ _n_ ที่เชื่อมโยงกับ wallet จะต้องมี _m_ ที่ร่วมลงนามในแต่ละธุรกรรมอย่างหลีกเลี่ยงไม่ได้



กลไกนี้ช่วยให้การควบคุม wallet สามารถแบ่งปันระหว่างหลายหน่วยงานหรืออุปกรณ์ได้ ตัวอย่างเช่น ในการกำหนดค่า 2-of-3 จะมีการสร้างชุดกุญแจอิสระสามชุด แต่ต้องใช้เพียงสองชุดในการปล่อยเงินทุน สถาปัตยกรรมนี้ลดความเสี่ยงที่เกี่ยวข้องกับการถูกโจมตีหรือการสูญเสียกุญแจอย่างมาก: โจรที่เข้าถึงได้เพียงกุญแจเดียวไม่สามารถทำให้ wallet ว่างเปล่าได้ และผู้ใช้ที่สูญเสียกุญแจหนึ่งชุดยังคงสามารถเข้าถึงเงินทุนของเขาได้ด้วยกุญแจที่เหลืออีกสองชุด



![Image](assets/fr/01.webp)



อย่างไรก็ตาม ความปลอดภัยที่มากขึ้นนี้มาพร้อมกับความซับซ้อนที่มากขึ้น การตั้งค่า Multisig wallet ต้องมีการรักษาความปลอดภัยของวลีช่วยจำหลายชุด (หนึ่งชุดต่อปัจจัยลายเซ็น) และคีย์สาธารณะที่ขยายออก ("*xpub*") จริง ๆ แล้ว หากคุณกำลังใช้ Multisig 2-of-3 wallet ในการดึงข้อมูล wallet คุณต้องมีวลีช่วยจำทั้งสามชุด หรืออย่างน้อยสองในสามชุด แต่ถ้าคุณมีเพียงสองในสามชุด คุณยังต้องเข้าถึง *xpubs* ทั้งสามชุด ซึ่งหากไม่มีจะไม่สามารถดึงคีย์สาธารณะที่จำเป็นในการเข้าถึงบิตคอยน์ที่พวกเขาปกป้องได้



โดยสรุปแล้ว ในการกู้คืน Multisig wallet คุณต้อง :




- หรือเข้าถึงวลีช่วยจำทั้งหมดที่เกี่ยวข้องกับแต่ละปัจจัยลายเซ็น;
- มีจำนวนวลีช่วยจำขั้นต่ำตามที่กำหนดโดยเกณฑ์เพื่อให้สามารถลงนามได้ และยังต้องเข้าถึง xpubs ของปัจจัยทั้งหมดเพื่อดึงคีย์สาธารณะที่จำเป็นออกมา



![Image](assets/fr/02.webp)



การจัดการการสำรองข้อมูล Multisig wallet นี้ได้รับการอำนวยความสะดวกโดย *Output Script Descriptors* ซึ่งรวมกลุ่มข้อมูลสาธารณะที่จำเป็นทั้งหมดเพื่อเข้าถึงเงินทุน อย่างไรก็ตาม ฟังก์ชันนี้ยังไม่ได้รับการใช้งานในซอฟต์แวร์การจัดการ wallet ทั้งหมด



Multisig เหมาะอย่างยิ่งสำหรับผู้ใช้บิทคอยน์ที่มองหาความปลอดภัยที่เพิ่มขึ้นหรือการจัดการเงินร่วมกัน: บริษัท สมาคม ครอบครัว หรือผู้ใช้รายบุคคลที่ถือบิทคอยน์จำนวนมาก สามารถใช้เพื่อสร้างรูปแบบการปกครองแบบกระจายศูนย์ เช่น การกระจายอำนาจการลงนามในหมู่ผู้จัดการหรือสมาชิกทีมหลายคน



ในบทแนะนำนี้ เราจะเรียนรู้วิธีการสร้างและใช้ wallet แบบหลายลายเซ็นคลาสสิกกับ **Sparrow Wallet** หากคุณต้องการสร้าง wallet แบบหลายลายเซ็นที่ปรับแต่งได้พร้อมการล็อกเวลา ฉันขอแนะนำให้ใช้ Liana แทน:



https://planb.academy/tutorials/wallet/desktop/liana-306ef457-700c-4fdd-b07a-8fb7a8a29f04

## ข้อกำหนดเบื้องต้น



สำหรับบทแนะนำนี้ ฉันจะแสดงวิธีการสร้าง Multisig ด้วย [ซอฟต์แวร์จัดการ Sparrow Wallet wallet](https://sparrowwallet.com/download/) หากคุณยังไม่ได้ติดตั้งซอฟต์แวร์นี้ กรุณาทำการติดตั้งตอนนี้ หากคุณต้องการความช่วยเหลือ เราก็มีบทแนะนำที่ละเอียดเกี่ยวกับการกำหนดค่า Sparrow Wallet :



https://planb.academy/tutorials/wallet/desktop/sparrow-c674e2ac-d46f-4c82-92a7-7d1b0e262f5d)

ในการตั้งค่า wallet แบบหลายลายเซ็น คุณจะต้องใช้กระเป๋าเงินฮาร์ดแวร์ที่แตกต่างกัน สำหรับ Multisig 2-de-3 ตัวอย่างเช่น คุณสามารถใช้ :




- A Trezor Model One;
- Ledger Flex;
- Coldcard MK3



![Image](assets/fr/03.webp)



เป็นความคิดที่ดีที่จะใช้รุ่นต่างๆ ของ Hardware Wallet ในการกำหนดค่า Multisig ของคุณ สิ่งนี้จะช่วยให้มั่นใจได้ว่าหากรุ่นใดรุ่นหนึ่งพบปัญหาร้ายแรง จะไม่ส่งผลกระทบต่อความปลอดภัยโดยรวมของ Multisig ของคุณ ยิ่งไปกว่านั้น ยังช่วยให้คุณได้รับประโยชน์จากข้อดีเฉพาะของแต่ละอุปกรณ์อีกด้วย ตัวอย่างเช่น ในการกำหนดค่าของฉัน :





- Trezor Model One เป็นโอเพ่นซอร์สอย่างสมบูรณ์ ซึ่งทำให้สามารถตรวจสอบการผลิต seed ได้ อย่างไรก็ตาม เนื่องจากมันไม่ได้ติดตั้ง Secure Element จึงยังคงเสี่ยงต่อการโจมตีทางกายภาพ;





- Ledger Flex ในทางกลับกัน ได้รับประโยชน์จากเฟิร์มแวร์ที่เป็นกรรมสิทธิ์ที่ไม่สามารถตรวจสอบได้ แต่รวมถึง Secure Element ที่ให้การป้องกันทางกายภาพที่ยอดเยี่ยม;





- Coldcard มาพร้อมกับ Secure Element และโค้ดของมันสามารถค้นหาได้ เป็นตัวเลือกที่น่าสนใจสำหรับการกำหนดค่าของเรา เนื่องจากมีฟีเจอร์การตรวจสอบที่ไม่มีในรุ่นอื่น ๆ



ก่อนที่จะกำหนดค่า Multisig wallet ของคุณ ตรวจสอบให้แน่ใจว่าแต่ละ Hardware Wallet ได้รับการกำหนดค่าอย่างถูกต้อง (การสร้างและบันทึกคำช่วยจำ การกำหนด PIN) สำหรับคำแนะนำโดยละเอียด คุณสามารถดูบทแนะนำของเราสำหรับแต่ละ Hardware Wallet ได้ เช่น:



https://planb.academy/tutorials/wallet/hardware/trezor-model-one-5c250c49-ce3b-4c63-bd05-4600d7c11a02

https://planb.academy/tutorials/wallet/hardware/ledger-flex-3728773e-74d4-4177-b39f-bd923700c76a

https://planb.academy/tutorials/wallet/hardware/coldcard-q-73e86d1a-6fe6-4d8b-bb15-8690298020e3

ตามที่เราจะได้เห็นในภายหลังในบทแนะนำนี้ ยังสามารถรวมเข้ากับการกำหนดค่า Multisig ของคุณ ปัจจัยที่ไม่ได้เชื่อมโยงกับ Hardware Wallet แต่มีคีย์ส่วนตัวที่เก็บไว้ในพีซีของคุณ วิธีนี้ชัดเจนว่ามีความปลอดภัยน้อยกว่าการใช้กระเป๋าเงินฮาร์ดแวร์เพียงอย่างเดียว แต่ก็อาจมีความเกี่ยวข้องในบางกรณี ตัวอย่างเช่น สำหรับ Multisig 2-de-3 คุณอาจเลือกใช้กระเป๋าเงินฮาร์ดแวร์สองใบและหนึ่ง Software Wallet



## สร้าง Multisig wallet



เปิด Sparrow Wallet คลิกที่แท็บ "*File*" จากนั้นเลือก "*New Wallet*"



![Image](assets/fr/04.webp)



กำหนดชื่อให้กับ wallet หลายลายเซ็นของคุณ จากนั้นคลิกที่ "*สร้าง Wallet*" เพื่อยืนยัน



![Image](assets/fr/05.webp)



ในเมนูแบบเลื่อนลง "*Policy Type*" ให้เลือกตัวเลือก "*Multi Signature*"



![Image](assets/fr/06.webp)



ที่มุมขวาบน คุณสามารถกำหนดจำนวนคีย์ทั้งหมดใน Multisig ของคุณ รวมถึงจำนวนผู้ร่วมลงนามที่จำเป็นในการอนุมัติค่าใช้จ่าย ในตัวอย่างของฉัน นี่คือรูปแบบ 2-of-3



![Image](assets/fr/07.webp)



ที่ด้านล่างของหน้าต่าง Sparrow Wallet จะแสดง "*Keystore*" สามรายการ แต่ละรายการแสดงถึงชุดของกุญแจ ที่นี่ฉันกำลังใช้กระเป๋าเงินฮาร์ดแวร์สามใบ ดังนั้นแต่ละ "*Keystore*" จึงสอดคล้องกับหนึ่งในนั้น ตอนนี้เราจะทำการกำหนดค่าพวกมัน



ฉันเริ่มต้นด้วย Coldcard ในแท็บ "*Keystore 1*" ฉันเลือกตัวเลือก "*Airgapped Hardware Wallet*"



![Image](assets/fr/08.webp)



บน Coldcard เมื่อปลดล็อกอุปกรณ์แล้ว ฉันไปที่เมนู "*Settings*" จากนั้นไปที่ "*Multisig Wallets*"



![Image](assets/fr/09.webp)



เมนูนี้ช่วยให้คุณจัดการกระเป๋าเงิน multisig ที่ Coldcard เข้าร่วมได้ ฉันต้องการสร้างใหม่ ดังนั้นฉันเลือก "*Export XPUB*"



![Image](assets/fr/10.webp)



สำหรับช่อง "*หมายเลขบัญชี*" หากคุณจัดการเพียงบัญชีเดียว คุณสามารถเว้นว่างไว้และยืนยันได้โดยตรงโดยกดปุ่มยืนยัน



![Image](assets/fr/11.webp)



Coldcard จะทำการ generate ไฟล์ที่มี xpub ของคุณ บันทึกลงบนการ์ด Micro SD



![Image](assets/fr/12.webp)



ใส่ Micro SD นี้ลงในคอมพิวเตอร์ของคุณ ใน Sparrow Wallet คลิกที่ปุ่ม "*Import File...*" ถัดจาก "*Coldcard Multisig*" จากนั้นเลือกไฟล์ที่สร้างโดย Coldcard บนการ์ด



![Image](assets/fr/13.webp)



xpub ของคุณถูกนำเข้าเรียบร้อยแล้ว ตอนนี้เราจะทำขั้นตอนเดียวกันกับฮาร์ดแวร์วอลเล็ตอีกสองตัว



![Image](assets/fr/14.webp)



สำหรับ Ledger Flex ฉันเลือก "*Keystore 2*" จากนั้นคลิกที่ "*Connected Hardware Wallet*". ตรวจสอบให้แน่ใจว่า Ledger เชื่อมต่อกับคอมพิวเตอร์ ปลดล็อกแล้ว และแอปพลิเคชัน Bitcoin เปิดอยู่



![Image](assets/fr/15.webp)



จากนั้นคลิกที่ปุ่ม "*Scan...*"



![Image](assets/fr/16.webp)



ถัดจากชื่อฮาร์ดแวร์ของคุณ wallet ให้คลิกที่ "*Import Keystore*"



![Image](assets/fr/17.webp)



ผู้ลงนามคนที่สองได้ลงทะเบียนอย่างถูกต้องใน Sparrow Wallet แล้ว



![Image](assets/fr/18.webp)



ฉันทำตามขั้นตอนเดิมกับ Trezor One เพื่อสรุปการตั้งค่า Multisig



![Image](assets/fr/19.webp)



ในการกำหนดค่าของฉัน เราไม่ได้ครอบคลุมกรณีนี้ แต่ถ้าคุณต้องการรวมลายเซ็นผ่านซอฟต์แวร์ wallet ใน Sparrow (wallet ร้อน) ภายใน Multisig ของคุณ เพียงคลิกที่ปุ่ม "*New or Imported Software Wallet*"



ตอนนี้ที่อุปกรณ์ลายเซ็นทั้งหมดของคุณถูกนำเข้าไปยัง Sparrow Wallet แล้ว คุณสามารถสรุปการสร้าง Multisig ได้โดยคลิกที่ "*Apply*".



![Image](assets/fr/20.webp)



เลือกใช้รหัสผ่านที่แข็งแกร่งเพื่อรักษาความปลอดภัยในการเข้าถึง Sparrow Wallet wallet ของคุณ รหัสผ่านนี้จะปกป้องกุญแจสาธารณะ ที่อยู่ ป้ายกำกับ และประวัติการทำธุรกรรมของคุณจากการเข้าถึงโดยไม่ได้รับอนุญาต



อย่าลืมบันทึกรหัสผ่านนี้ในที่ปลอดภัย เช่น โปรแกรมจัดการรหัสผ่าน เพื่อหลีกเลี่ยงการสูญหาย



![Image](assets/fr/21.webp)



## สำรองข้อมูล Multisig wallet



เรากำลังจะบันทึก *Output Script Descriptor* ของเราใน Coldcard (สิ่งนี้ใช้ได้เฉพาะกับผู้ใช้ที่มี Coldcard ใน Multisig ของพวกเขา) และที่สำคัญที่สุด เราจะเก็บสำรองข้อมูลไว้ในสื่ออิสระ



*Descriptor* ประกอบด้วย xpubs ทั้งหมดใน Multisig wallet ของคุณ รวมถึงเส้นทางการอนุพันธ์ที่ใช้ในการ generate คีย์ จำสิ่งที่เราเห็นในส่วนที่ 1: เพื่อกู้คืน Multisig wallet คุณต้องมีทั้ง **วลีช่วยจำทั้งหมด** หรือเพียงจำนวนขั้นต่ำที่จำเป็นเพื่อให้ถึงเกณฑ์ลายเซ็น อย่างไรก็ตาม ในกรณีหลังนี้ จำเป็นต้องมี **xpubs** ของผู้ลงนามที่ขาดหายไปด้วย *Descriptor* ประกอบด้วย xpubs ทั้งหมดของ Multisig ของคุณ



หากสิ่งนี้ไม่ชัดเจน เพียงจำไว้ว่าการดึงข้อมูล Multisig คุณจำเป็นต้องมีจำนวนวลีช่วยจำขั้นต่ำสำหรับแต่ละ Hardware Wallet ที่ใช้ ขึ้นอยู่กับเกณฑ์ (ในกรณีของฉัน: 2 วลี) รวมถึง *Descriptor* ด้วย



*Descriptor* นี้ไม่มีคีย์ส่วนตัว มีแต่คีย์สาธารณะเท่านั้น ซึ่งหมายความว่ามันไม่ให้การเข้าถึงเงินทุน ดังนั้นจึงไม่สำคัญเท่ากับวลีช่วยจำที่ให้การเข้าถึงบิตคอยน์ของคุณอย่างเต็มที่ ความเสี่ยงกับ *Descriptor* เกี่ยวข้องกับความลับเท่านั้น: ในกรณีที่ถูกละเมิด บุคคลที่สามอาจสังเกตเห็นธุรกรรมทั้งหมดของคุณ แต่ไม่สามารถใช้จ่ายเงินของคุณได้



ฉันขอแนะนำอย่างยิ่งให้คุณสร้างสำเนาหลายชุดของ *Descriptor* และเก็บไว้กับอุปกรณ์เซ็นชื่อแต่ละตัวบน Multisig ของคุณ ตัวอย่างเช่น ในกรณีของฉัน ฉันพิมพ์ *Descriptor* ลงบนกระดาษและเก็บสำเนาหนึ่งชุดไว้กับ Coldcard อีกชุดกับ Trezor และอีกชุดกับ Ledger ฉันยังบันทึก *Descriptor* นี้เป็นไฟล์ PDF บน USB sticks สามอัน แต่ละอันเก็บไว้กับกระเป๋าเงินฮาร์ดแวร์หนึ่งอัน ด้วยวิธีนี้ ฉันเพิ่มโอกาสสูงสุดที่จะไม่สูญเสีย *Descriptor* นี้ และฉันมั่นใจว่ามีสำเนาสองชุด (หนึ่งชุดเป็นแบบกายภาพและอีกชุดเป็นแบบดิจิทัล) กับอุปกรณ์แต่ละตัว



เมื่อสร้าง Multisig wallet ของคุณแล้ว Sparrow จะให้ *Descriptor* นี้แก่คุณโดยอัตโนมัติ คลิกที่ปุ่ม "*Save PDF...*" เพื่อบันทึกทั้งในรูปแบบข้อความและรหัส QR



![Image](assets/fr/22.webp)



จากนั้นคุณสามารถพิมพ์ PDF นี้และคัดลอกไปยัง USB ของคุณได้



![Image](assets/fr/23.webp)



เราจะลงทะเบียน *Descriptor* นี้ใน Coldcard ด้วย (หากคุณใช้ในการตั้งค่าของคุณ) ซึ่งจะช่วยให้ Coldcard ตรวจสอบได้ว่าแต่ละธุรกรรมที่ลงนามในภายหลังสอดคล้องกับ wallet ดั้งเดิม: xpubs ที่ถูกต้อง, รูปแบบที่อยู่ที่ถูกต้อง, เส้นทางการอนุพันธ์ที่ถูกต้อง... หากไม่มี *Descriptor* ที่นำเข้านี้ Coldcard จะไม่สามารถยืนยันได้ว่าที่อยู่แลกเปลี่ยนไม่ได้ถูกแฮ็กหรือว่า PSBT ไม่ได้ถูกดัดแปลง



นี่คือสิ่งที่ทำให้ Coldcard น่าสนใจใน Multisig: มันมีการตรวจสอบเพิ่มเติมเพื่อต่อต้านการโจมตีที่ซับซ้อนบางอย่าง ซึ่งกระเป๋าเงินฮาร์ดแวร์อื่นๆ ไม่อนุญาต (แน่นอนว่าต้องใช้มันในการเซ็นชื่อ)



ใน Sparrow ให้เข้าถึงเมนู "*Settings*" จากนั้นคลิกที่ "*Export...*"



![Image](assets/fr/24.webp)



ถัดจากตัวเลือก "*Coldcard Multisig*" ให้คลิกที่ "*Export File...*" และบันทึกไฟล์ข้อความลงในการ์ด Micro SD.



![Image](assets/fr/25.webp)

เลือกไฟล์ที่เหมาะสมและยืนยันการนำเข้า

ตรวจสอบพารามิเตอร์การกำหนดค่าของ Multisig จากนั้นยืนยันการลงทะเบียน



![Image](assets/fr/29.webp)



Multisig ของคุณถูกบันทึกใน Coldcard ของคุณเรียบร้อยแล้ว หากคุณมี Coldcard หลายตัวใน Multisig เดียวกัน ให้ทำตามขั้นตอนนี้ซ้ำสำหรับแต่ละตัว



นอกเหนือจากการบันทึก *Descriptor* อย่าลืมให้ความสำคัญเป็นพิเศษกับการบันทึกวลีช่วยจำสำหรับอุปกรณ์ลายเซ็นของคุณแต่ละเครื่อง หากคุณเพิ่งเริ่มต้น ฉันขอแนะนำอย่างยิ่งให้คุณปรึกษาบทแนะนำอื่นนี้เพื่อเรียนรู้วิธีบันทึกและจัดการอย่างถูกต้อง:



https://planb.academy/tutorials/wallet/backup/backup-mnemonic-22c0ddfa-fb9f-4e3a-96f9-46e2a7954270

ก่อนที่จะรับบิตคอยน์ครั้งแรกบน Multisig ของคุณ **ฉันขอแนะนำอย่างยิ่งให้คุณทำการทดสอบการกู้คืนแบบว่างเปล่า** จดบันทึกข้อมูลอ้างอิงบางอย่าง เช่น ที่อยู่รับแรก จากนั้นรีเซ็ตกระเป๋าเงินฮาร์ดแวร์ของคุณในขณะที่ wallet ยังคงว่างเปล่า จากนั้นลองกู้คืน Multisig wallet ของคุณบนกระเป๋าเงินฮาร์ดแวร์โดยใช้การสำรองข้อมูลกระดาษวลีช่วยจำของคุณ จากนั้นบน Sparrow โดยใช้ *Descriptor* ตรวจสอบว่าที่อยู่แรกที่สร้างขึ้นหลังการกู้คืนตรงกับที่คุณเขียนไว้ในตอนแรกหรือไม่ หากตรงกัน คุณสามารถมั่นใจได้ว่าการสำรองข้อมูลกระดาษของคุณเชื่อถือได้



หากต้องการเรียนรู้เพิ่มเติมเกี่ยวกับวิธีการทดสอบการกู้คืน ฉันขอแนะนำให้คุณดูบทแนะนำอื่นนี้:

## การส่งบิตคอยน์ด้วย Multisig ของคุณ



ตอนนี้ที่คุณได้รับ Satss แรกของคุณบน Multisig wallet แล้ว คุณก็สามารถใช้จ่ายได้เช่นกัน! ใน Sparrow ไปที่แท็บ "*Send*" เพื่อสร้างธุรกรรมใหม่



![Image](assets/fr/36.webp)



หากคุณต้องการใช้ *Coin Control* ซึ่งหมายถึงการเลือก UTXOs ที่คุณต้องการใช้ด้วยตนเอง ให้ไปที่แท็บ "*UTXOs*" เลือก UTXOs ที่คุณต้องการใช้ จากนั้นคลิกที่ "*Send Selected*" คุณจะถูกนำไปยังแท็บ "*Send*" โดยอัตโนมัติ โดยมี UTXOs ที่เลือกไว้ล่วงหน้าแล้ว



![Image](assets/fr/37.webp)



ป้อนที่อยู่ปลายทาง สามารถเพิ่มที่อยู่หลายแห่งได้โดยคลิกที่ "*+ เพิ่ม*"



![Image](assets/fr/38.webp)



เพิ่ม "*ป้ายกำกับ*" เพื่ออธิบายวัตถุประสงค์ของค่าใช้จ่ายนี้ เพื่อให้ง่ายต่อการติดตามธุรกรรมของคุณ



![Image](assets/fr/39.webp)



ป้อนจำนวนเงินที่จะส่งไปยังที่อยู่ที่เลือก



![Image](assets/fr/40.webp)



ปรับอัตราการชาร์จตามสภาพเครือข่ายปัจจุบัน ตัวอย่างเช่น ปรึกษา [Mempool.space](https://Mempool.space/) เพื่อเลือกระดับการชาร์จที่เหมาะสม



หลังจากตรวจสอบพารามิเตอร์การทำธุรกรรมทั้งหมดแล้ว ให้คลิกที่ "*Create Transaction*"



![Image](assets/fr/41.webp)



หากคุณพอใจกับทุกอย่าง ให้คลิกที่ "*Finalize Transaction for Signing*"



![Image](assets/fr/42.webp)



ที่ด้านล่างของหน้าจอ คุณจะเห็นว่า Sparrow กำลังรอ 2 ลายเซ็น นี่เป็นเรื่องปกติ: wallet ที่ใช้ที่นี่คือ Multisig 2-de-3



![Image](assets/fr/43.webp)



ฉันเริ่มเซ็นด้วย Coldcard ของฉัน ในการทำเช่นนี้ ฉันใส่การ์ด Micro SD ลงในคอมพิวเตอร์ จากนั้นคลิกที่ "*Save Transaction*"



![Image](assets/fr/44.webp)



มี 3 วิธีในการส่งธุรกรรมที่จะลงนามไปยัง Hardware Wallet แล้วดึงข้อมูลจาก Sparrow วิธีแรกคือการใช้ Micro SD card ซึ่งเราจะใช้สำหรับ Coldcard ที่นี่ วิธีที่สองคือผ่านการเชื่อมต่อสายเคเบิล ซึ่งเราจะใช้สำหรับลายเซ็นที่สอง (Ledger และ Trezor) สุดท้ายสามารถใช้การสื่อสารผ่าน QR code สำหรับอุปกรณ์ที่มีกล้อง เช่น Coldcard Q, Jade Plus หรือ Passport V2



เมื่อบันทึก PSBT (*Partially Signed Bitcoin Transaction*) ลงใน Micro SD แล้ว ฉันใส่มันเข้าไปใน Coldcard MK3 จากนั้นเลือกเมนู "*Ready to Sign*"



![Image](assets/fr/45.webp)



บนหน้าจอ Hardware Wallet ของคุณ ตรวจสอบพารามิเตอร์การทำธุรกรรมอย่างละเอียด: ที่อยู่ของผู้รับ จำนวนเงินที่ส่ง และค่าธรรมเนียม เมื่อการทำธุรกรรมได้รับการยืนยันแล้ว ให้ตรวจสอบเพื่อดำเนินการลงนามต่อไป



![Image](assets/fr/46.webp)



จากนั้นใส่ Micro SD กลับเข้าไปในคอมพิวเตอร์ของคุณ และคลิกที่ "*Load Transaction*" ใน Sparrow เลือก PSBT ที่เซ็นโดย Coldcard จากไฟล์ของคุณ



![Image](assets/fr/47.webp)



คุณสามารถเห็นได้ว่าลายเซ็นของ Coldcard ได้ถูกเพิ่มเข้ามาแล้ว ตอนนี้ฉันจะใช้เครื่องที่สอง ในกรณีนี้คือ Ledger เพื่อทำลายเซ็นที่สองที่จำเป็น ฉันเชื่อมต่อ ปลดล็อก แล้วคลิกที่ "*Sign*" บน Sparrow



![Image](assets/fr/48.webp)



คลิกที่ "*Sign*" ถัดจากชื่อ Hardware Wallet ของคุณ



![Image](assets/fr/49.webp)



ครั้งแรกที่คุณใช้ Ledger ของคุณกับ Multisig นี้, Sparrow จะขอให้คุณยืนยันคีย์สาธารณะขยาย (xpubs) ของผู้ร่วมลงนาม เช่นเดียวกับ Coldcard ขั้นตอนนี้จะป้องกันไม่ให้คุณลงนามโดยไม่รู้ตัวในภายหลัง ในการตรวจสอบข้อมูลนี้ ให้เปรียบเทียบ xpub ที่แสดงบนหน้าจอ Ledger กับที่ให้โดยตรงจากกระเป๋าเงินฮาร์ดแวร์อื่น ๆ ของคุณ



![Image](assets/fr/50.webp)



ตรวจสอบที่อยู่ของผู้รับ จำนวนเงินที่โอน และค่าธรรมเนียมการทำธุรกรรม จากนั้นลงนามในธุรกรรม
