Coinkiteの主力製品であるColdCardQの**キーテレポート**機能とは何ですか。



**Key Teleport** を使用すると、2 台の ColdCardQ 間で機密データを安全に転送できます。転送チャネルは暗号化する必要もなく、公開することもできます。



これは移籍に使える：





- gW-0フレーズ**（コールドカードQのseedマスター、またはコールドカードQの[seed Vault](https://coldcard.com/docs/temporary-seeds/#seed-vault)に保存されている秘密）**
- **機密メモとパスワード**: ColdCardQ の [機密メモとパスワード] ディレクトリ (https://coldcard.com/docs/secure_notes/) 全体を指定できます。
- **ColdCardQ 全体のバックアップ**: このバックアップを受け取る ColdCardQ は、seed マスターを持っていない必要があります。
- マルチ署名スキーム**の一部としてのgW-3（*部分署名Bitcoin取引*）。



このためには、[デバイスのファームウェアをバージョンv1.3.2Q](https://coldcard.com/docs/upgrade/)以上にアップグレードしている必要があります。



## キー・テレポートの使い方は？



### 1- あらゆる種類のデータを転送する



ここでは、seed の文章、メモ、パスワード、ColdCardQ バックアップ全体の転送について説明します。マルチシグネチャトランザクションの PSBT 転送のケースは後で扱う。



#### 秘密を受け取るためにデバイスを準備する



ColdCardQ の **"Advanced / Tools**" メニューで、**"Key Teleport (start) "** を選択します。


次の画面では、8桁のパスワードが提案されます（ここでは「20420219」）。このパスワードを送信者に伝える必要があります。このパスワードを送信するには、例えばSMSを使うか、お気に入りのセキュア・メッセージング・システム、あるいは音声通話を使う。



その後、ColdCardQ の **"Enter**" ボタンをクリックして次のステップに進みます。




![CCQ-key-teleport](assets/fr/01.webp)




画面上にQRコードが生成されます。もう一度、このQRコードをColdCardQの「送信者」に伝える必要がある。最も簡単な方法はビデオ通話です。



**この qr コードは、前の 8 桁のパスワードの送信に使用したのと同じ送信チャネルで送信しないでください。**



![CCQ-key-teleport](assets/fr/02.webp)



*興味のある方のために、安全でないチャンネル*で秘密が転送される根本的なメカニズムを理解してみよう。



**ここで実際に行っているのは、ディフィー・ヘルマン方式による秘密の転送を開始することだ。**

*私たちは現在*




- エフェメラル鍵ペア（公開鍵/秘密鍵それぞれKaとka、Ka=G.ka、GはECDH生成ポイント）と8桁のパスワードを生成。
- このパスワードを使用して公開鍵（Ka）をAES-256-CTRで暗号化し、このパスワードを通信チャネルAを介して「送信側」のColdCardQ.*に送信する。
- 最後に、暗号化されたパケットを、上記QRコードを介して、1番目とは異なる2番目の通信チャネルBを使用して送信者に送信した。



#### 秘密を送信するデバイスを準備する



送信側機器から、**"QR "**ボタンをクリックして、受信側機器から送られてきたQRコードをスキャンし、前のステップで通知された8桁のパスワードを別のチャンネルで入力します。これで「送信側」機器からのデータ送信を開始する準備が整いました。



**エラーメッセージは表示されず、プロセスは続行されます。ただし、最終的なデータ転送は失敗し、もう一度やり直す必要があります。**



![CCQ-key-teleport](assets/fr/03.webp)



*好奇心旺盛な諸君のために、暗号と秘密転送に関して我々が何をしようとしているのか、もう一度見てみよう：*。




- 受信デバイスでQRコードをスキャンすることで、暗号化されたデータを取り込みました。
- その後、セカンダリ・チャンネル（※）を介して送信された8桁のパスワードを使って復号化した。
- したがって、受信者が最初に生成した公開鍵（Ka）を所有していることになる。
- 次に、送信側デバイスで新しいエフェメラル鍵ペア(Kb/kb、Kb=G.kb)をgenerateし、KaにECDHを適用するために使用する。したがって、我々はkb.Ka=Ksという操作を実行する。




ここで、2 つの ColdCardQ 間で転送する秘密の性質 (機密メモ、パスワード、フルバックアップ、保管庫に含まれるシード、seed デバイスマスター) を選択するよう求められます。



![CCQ-key-teleport](assets/fr/04.webp)



ここで私たちの秘密は、**"クイックテキストメッセージ "**を選択することにより、短いメッセージになります。あなたのメッセージ（私たちの場合は "PlanB Network rocks"）を入力し、**"ENTER "**を押します。


このデバイスは次に、「NE XG BT SK」の例では、**"テレポート・パスワード "** と呼ばれる新しいランダムなパスワードを生成します。



![CCQ-key-teleport](assets/fr/05.webp)



ENTER "**を押すと、新しいQRコードが表示されます。それを受信デバイスでスキャンしてください。そして別の通信チャンネルで、**"テレポートパスワード "**を受信機に送信してください。



![CCQ-key-teleport](assets/fr/06.webp)



**好奇心旺盛な方のために、この段階での話をもう一度。**




- 送信する秘密を選択した後、「テレポートパスワード」***と呼ばれる新しいランダムなパスワードをgenerateに入力する。
- 次に、前のステップで生成された**"セッション鍵 "**、"Ks "を使用して、AES-256-CTRで秘密を暗号化する。
- で暗号化されたパケットの前に、我々のKb公開鍵で**"セッション鍵 "**を付加し、さらに**"テレポートパスワード "**でAES-256-CTR暗号化のLayerを付加する。そして全体がQRコードとしてエンコードされる。




#### 受信デバイスへの秘密の転送を確定する



QR」**ボタンを押して、visioチャンネルを通じて送信機器から提示されたQRコードをスキャンします。NE XG BT SK "用の**"テレポートパスワード "**"の入力を求められます。



![CCQ-key-teleport](assets/fr/07.webp)





その後、データは復号化され、受信デバイスで理解できるようになる。受信したメッセージは確かに「PlanB Network rocks」である。以上です。



![CCQ-key-teleport](assets/fr/08.webp)



**この最後のステージで実際に何が起こったのか。**




- 送信者が**"Teleport Password "**を使って送信したデータを解読した。
- したがって、私たちは公開鍵Kbと、「セッション鍵」である「Ks」によって暗号化された秘密のメッセージを持つことになる。しかし、受信者である私たちは、送信者が作成したKsを知らないので、どのようにすればよいのでしょうか?
- 最初のステップ**「データを受信するデバイスを準備する」**で使用した秘密鍵 "ka "を、公開鍵**Kb**に適用する必要がある。
- 実際、ka.Kb=ka.kb.G=kb.ka.G=kb.Ka=Ksを計算することで、Ksが求められる。これが最終的に秘密メッセージの解読に使われる。



### 2- PSBTをMultisigに移行する（上級編）



これは、Wallet Multisig がすでに作成され、ColdCardQ デバイスがマルチシグネチャトランザクションを実行できるようにプリセットされていることを前提としています。そうでない場合は、Coinkite ウェブサイトの [こちら](https://coldcard.com/docs/Multisig/) に説明があります。



マルチシグネチャーのWallet（Multisig）とは何か、簡単に覚えておこう。



通常、Walletの資金を使うには、アドレスに関連するUTXOのロックを解除するための秘密鍵が1つだけ必要です。


Wallet、Multisigの場合、資金を使用するために最大15個の秘密鍵、したがって15個の 署名が必要となる可能性がある。これは "M out of N "ポートフォリオと呼ばれ、Nは1～15、Mは資金を使用するために必要な署名の数である。例えば、Wallet Multisig 3 out of 5 の場合、5人中少なくとも3人の署名が必要となる。



Partially Signed Bitcoin Transaction "に "PSBT "を順番に署名するために、署名者間で調整することが課題である。その際、"**Key Teleport**"を使えば、簡単かつ秘密裏に共同署名者間でPSBTを送信することができる。簡単なビデオ通話で済みます。



Multisigの3/4はこうだ。



**署名者1:**



署名者1はPSBTをインポートし署名する。最後に、彼は**"T "**をクリックし、**"Key Teleport "**を使ってPSBTを署名者2に送信する。



![CCQ-key-teleport](assets/fr/09.webp)




ENTER "**をクリックして署名者2を選択すると、"TELEPORT PASSWORD"（ここではJJ YC AB 6A）が提供される。例えば、SMSや音声通話である。



もう一度**"ENTER "**を押すと、1が署名し、"TELEPORT PASSWORD "で暗号化されたPSBTを表すQRコードが表示されます。



![CCQ-key-teleport](assets/fr/10.webp)



**署名者2:**



署名者2は、署名者1から提示されたQRコードを読み取る。その後、二次通信路で送信された「TELEPORT PASSWORD」を入力し、送信データを復号化する。



署名者 2 は取引に署名し、**"T "**をクリックして PSBT を "Key Teleport "を介して署名者 3 に送信する。


明らかに、2つの署名はすでに適用されている。トランザクションを有効にするには、署名者3の署名が必要です。**ENTER**をクリックして署名者3を選択する。



![CCQ-key-teleport](assets/fr/11.webp)



そして、新しい「TELEPORT PASSWORD」が作成され、その後に1と2によって署名されたPSBTをエンコードしたQRコードが再び続き、さらにこの新しい「TELEPORT PASSWORD」によって暗号化される（GW YQ K3 LL）。



![CCQ-key-teleport](assets/fr/12.webp)



**署名者3：**



上記と同じ手順を繰り返す。


署名者 3 は、署名者 2 から提示された QR コードをスキャンする。その後、二次通信チャネルから送信された「TELEPORT PASSWORD」を入力する。



署名者3はトランザクションに署名し、今回は4つの署名のうち3つが適用されたので、トランザクションは確定したと表示され、様々なメディア（SDカード、NFC、QRなど）を介して配布する準備が整った。



![CCQ-key-teleport](assets/fr/13.webp)



ColdCardQ の「Push Tx」機能が有効になっている場合は、ColdCardQ を NFC 対応のインターネット接続機器 (スマートフォン/タブレット) の背面に貼り付けるだけで、Bitcoin ネットワーク経由で取引がブロードキャストされます。



![CCQ-key-teleport](assets/fr/14.webp)



*ある署名者から別の署名者へPSBTを転送する場合、"Key Teleport "は単に各段階で "Teleport Password "を介して使用され、PSBTはある署名者から別の署名者へ転送される際に暗号化される。送信されたデータは資金を盗むために使用することができないため、機密性の高い秘密（seed、パスワードなど...）を送信する場合のように、ディフィー・ヘルマン（Diffie-Hellman）を使用する必要はない*。
