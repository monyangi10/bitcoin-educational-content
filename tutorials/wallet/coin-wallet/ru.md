## О Coin Wallet


**Coin Wallet** - это самостоятельный / несамостоятельный wallet с открытым исходным кодом, созданный командой энтузиастов Bitcoin в 2015 году. Он был создан как веб-приложение, затем в 2017 году появилось приложение для iOS, а в 2020 году - приложение для Android, доступное в Google Play, Samsung Galaxy Store и Huawei AppGallery.


Ключевые преимущества:


- Архитектура, не связанная с лишением свободы
- Полностью [открытый код](https://github.com/CoinSpace/CoinSpace/blob/master/LICENSE)
- Простой и чистый дизайн
- Сосредоточены на основной цели - безопасном хранении криптовалюты, без лишних функций
- Кроссплатформенная поддержка: мобильные устройства (iOS и Android), настольные компьютеры, веб
- Поддержка RBF (Replace-By-Fee) - ускорение затянувшихся транзакций в любое время
- Аппаратная 2FA с помощью [YubiKey](https://www.yubico.com/works-with-yubikey/catalog/coin-wallet/) / ключа FIDO2
- Встроенная поддержка Tor - направляйте весь трафик через сеть Tor для максимальной конфиденциальности

## 2️⃣ Создание Wallet и установка PIN-кода


wallet создается на основе passphrase - случайной последовательности из 12 слов, разделенных пробелами, сгенерированной из [списка из 2048 слов](https://github.com/paulmillr/scure-bip39/blob/main/src/wordlists/english.ts).

Coin Wallet поддерживает кодовые фразы из 12, 15, 18, 21 или 24 слов, импортированные из других кошельков.


passphrase - это человекочитаемая форма главного закрытого ключа. Он должен быть сохранен в безопасном месте. passphrase - это все, что необходимо для доступа или восстановления wallet. Если passphrase будет утерян, wallet и все средства будут безвозвратно потеряны. passphrase ни в коем случае нельзя передавать друг другу. Coin Wallet не хранит ключи ни на одном сервере или в базе данных.


**Безопасен ли passphrase из 12 слов?

При 2048 возможных словах в каждой позиции существует 2048¹² ≈ 10³⁹ комбинаций, что обеспечивает ~128 бит защиты, сравнимых с закрытыми ключами Bitcoin. Этот уровень широко считается достаточным.


![image](assets/en/02.webp)


После того как passphrase будет записан и подтвержден, приложение попросит ввести **4-значный PIN-код** для повседневного доступа. Для большего удобства можно включить биометрическую аутентификацию (распознавание отпечатков пальцев или лица) вместо использования PIN-кода.


![image](assets/en/03.webp)



Нет ни учетной записи, ни восстановления ключей, ни сброса passphrase, ни отмены транзакций. Ответственность за безопасность полностью лежит на пользователе.


Подробные рекомендации по сохранению мнемонической фразы:


https://planb.academy/tutorials/wallet/backup/backup-mnemonic-22c0ddfa-fb9f-4e3a-96f9-46e2a7954270


## 3️⃣ Пассфраза + PIN-код. Когда и как они используются


**Когда требуется passphrase?

Только в этих редких случаях:


- Настройка wallet на новом устройстве
- Переустановка приложения Coin Wallet
- Очистка данных приложения/браузера (локальное хранилище)
- Удаление аппаратного ключа из учетной записи
- Трижды введите неправильный PIN-код (приложение блокируется в целях безопасности)


При повседневном использовании для разблокировки wallet достаточно 4-значного PIN-кода.


**Базовая фраза + PIN-код: как это работает**

passphrase - это настоящий главный закрытый ключ, который работает на любом устройстве.

Поскольку набирать каждый раз 12-24 слова было бы неудобно, в Coin Wallet используется 4-значный PIN-код для быстрого и повседневного доступа на текущем устройстве.

Простой PIN-код недостаточно надежен для защиты мастер-ключа напрямую, поэтому он никогда не используется для шифрования. Вместо этого:



- PIN-код отправляется на сервер и обменивается на длинный криптографический token.
- Этот token расшифровывает зашифрованный мастер-ключ, хранящийся только на устройстве.


Если PIN-код введен неверно три раза, сервер навсегда удаляет token. Хранящийся локально ключ становится непригодным для использования, и единственным способом восстановить wallet является ввод оригинального passphrase.

Такая конструкция обеспечивает удобство и надежную защиту от обратного удара.



## 4️⃣ Получение ₿иткоина - Address Типы и конфиденциальность


Coin Wallet поддерживает все три формата адресов Bitcoin:



- Родной SegWit (Bech32)** - начинается с **bc1q** - самая низкая плата, рекомендуется
- Вложенный SegWit (P2SH)** - начинается с **3**
- Наследие (P2PKH)** - начинается с **1**


![image](assets/en/04.webp)


**Почему адрес меняется после каждого вклада?**

Это сделано намеренно и защищает конфиденциальность. Каждый раз, когда поступают монеты, Coin Wallet автоматически генерирует новый неиспользуемый адрес. Если бы один и тот же адрес использовался повторно (например, для ежемесячной зарплаты), любой человек мог бы легко просуммировать все входящие транзакции в блокчейн-проводнике и узнать общий доход.


Старые адреса остаются в силе навсегда - вы по-прежнему можете получать на них сообщения, но рекомендуется использовать каждый раз новый адрес для обеспечения конфиденциальности.


**Как получить Bitcoin:**

1. Откройте монету

2. Нажмите **Получить**

3. Выберите желаемый тип адреса (предпочтительно **bc1q** - `Native SegWit`)

4. Покажите QR-код или скопируйте адрес и отправьте его плательщику


**Дополнительно - Mecto (для оплаты при личной встрече):**

На том же экране получения есть кнопка `Mecto`.

Когда вы включаете его:


- Вам будет предложено ввести **никнейм** (граватар)
- Ваше текущее местоположение + адрес получения временно передаются другим пользователям Coin Wallet, у которых также включена функция Mecto
- Они могут обнаружить вас на небольшой карте и отправить монеты без ввода или сканирования


Данные видны только другим пользователям Mecto и автоматически удаляются через 1 час (или сразу после выключения).

Mecto не является обязательным - отключите его, если вы предпочитаете максимальную конфиденциальность.

## 5️⃣ Отправка ₿itcoin


Чтобы отправить Bitcoin:


1. Откройте монету → нажмите **Отправить**

2. Вставьте адрес или отсканируйте QR-код

3. Введите сумму (или нажмите **Макс**)

4. Выберите скорость транзакции:



| Скорость   | Приблизительное время подтверждения | Уровень комиссии     |
|---------|---------------------------|---------------|
| **Медленно**    | ~120 минут              | Низший
| **По умолчанию** | ~60 минут               | Средний
| **Быстро**    | ~20 минут               | Выше

5. Подтвердите свой 4-значный PIN-код → транзакция транслируется


### Как ускорить ожидающую транзакцию ₿itcoin (RBF)


Если вы выбрали медленную комиссию, а транзакция застряла:


1. Перейдите на вкладку **История**

2. Нажмите на ожидающую транзакцию

3. Нажмите **Ускорить** (замена за плату)

4. Подтвердить → транзакция будет повторно транслироваться с более высокой комиссией


Ускорение RBF в настоящее время поддерживается для:

Bitcoin - Avalanche - Binance Smart Chain - Ethereum - Ethereum Classic - Polygon


Подробнее о Replace-by-fee (RBF): https://bitcoinops.org/en/topics/replace-by-fee/


## 6️⃣ Экспорт закрытых ключей


**Когда вам действительно нужен закрытый ключ?

(99 % пользователей никогда этого не делают - достаточно 12 слов passphrase)



| Ситуация                                      | Почему вам нужен приватный ключ                     |
|------------------------------------------------|--------------------------------------------------|
| Очистка старого бумажного кошелька                   | Перемещение средств в текущий кошелек             |
| Импорт в аппаратный сигнер (например, Coldcard) | Для автономного подписания                              |
| Аварийное восстановление (потеря семени, но приложение все еще открыто) | Спасти монеты перед тем, как приложение исчезнет           |
| Использование инструментов, которые не принимают фразы-семена     | Некоторые утилиты только для мониторинга или подписания             |

### Как экспортировать закрытые ключи в Coin Wallet


1. Открыть **Bitcoin (BTC)**

2. Прокрутите страницу в самый низ - нажмите **Экспорт закрытых ключей**

3. Приложение показывает каждый адрес с балансом + его закрытый ключ в формате **WIF** (начинается с 5, K или L) и QR-код.


Отображаются только непустые адреса.


**Пример ключа WIF**

`L2v1eK4i9j3k3j4k3j4k3j4k3j4k3j4k3j4k3j4k3j4k3j4k3j4k3j4k`


**Что делать дальше (рекомендуется)**


- Откройте Electrum, Sparrow, BlueWallet или любое другое оборудование wallet
- Импорт/приобретение закрытого ключа
- Все средства мгновенно переводятся на новый безопасный адрес под вашим текущим seed


Никогда не храните закрытый ключ в цифровом виде. После очистки его можно безопасно удалить.


Полное руководство по закрытым ключам и путям деривации: [How to Work with Private Keys: The Ultimate Guide](https://coin.space/how-to-work-with-private-keys-the-ultimate-guide/)



## 7️⃣ Технические детали - BIP39, BIP32 и пути деривации


Coin Wallet строго следует официальным стандартам Bitcoin, которые используют практически все серьезные кошельки.


`BIP39` - как passphrase становится главным закрытым ключом


- Количество слов по умолчанию: 12
- Необязательный passphrase/пароль: нет ("")
- Начальная энтропия: 128 бит (12 слов) → 256 бит (24 слова)
- Реализация с открытым исходным кодом: https://github.com/paulmillr/scure-bip39
- Список слов: стандартный английский список из 2048 слов
- Поддерживает импорт 12, 15, 18, 21 и 24-словных фраз из любого другого BIP39 wallet


`BIP32 + BIP44/BIP49/BIP84` - детерминированная генерация всех адресов

С одного мастер-ключа wallet может generate миллиарды адресов в строго определенном порядке. Именно поэтому одни и те же 12 слов, введенные в Electrum, Sparrow, Trezor, Ledger, BlueWallet и т. д., покажут абсолютно одинаковые адреса и балансы.





**Пути вывода, используемые в Coin Wallet для Bitcoin**

| Тип адреса              | Стандарт | Путь вывода       | Начинается с | Комментарий                              |
|---------------------------|----------|-----------------------|-------------|--------------------------------------|
| Собственный SegWit (Bech32)    | BIP84    | `m/84'/0'/0'`         | bc1q…       | Современный формат, самые низкие комиссии           |
| Вложенный SegWit (P2SH)      | BIP49    | `m/49'/0'/0'`         | 3…          | Оболочка совместимости для старых сервисов |
| Старый (P2PKH)            | BIP44    | `m/44'/0'/0'`         | 1…          | Самый старый формат, самые высокие комиссии          |

Внутри каждого пути:


- `/0` - внешняя цепочка (адреса, которые вы указываете для получения платежей)
- `/1` - внутренняя цепочка (изменение адресов, которые использует сам wallet)


Поскольку Coin Wallet следует этим государственным стандартам без каких-либо изменений, ваши средства останутся пригодными для возврата в любой другой совместимый wallet даже через 10-20-30 лет.


## 8️⃣ Повышение анонимности с помощью Tor


**Зачем использовать Tor в Coin Wallet**

Tor скрывает ваш реальный IP-адрес от узлов Bitcoin, бирж и наблюдателей.

Весь трафик (баланс, транзакции, обмен) проходит через сеть Tor - никаких прямых соединений, никаких утечек IP-адресов.

Это реализовано непосредственно в исходном коде приложения (см. [.env configuration](https://github.com/CoinSpace/CoinSpace/blob/master/web/.env#L31)).


Coin Wallet имеет скрытый .onion-адрес и, начиная с версии 6.6.3 (декабрь 2024 года), **встроенную поддержку Tor непосредственно в мобильном приложении**.


### Как включить Tor на Android и iOS


1. **Установите Orbot** - официальное приложение проекта Tor (бесплатно)


   - Android → [Google Play](https://play.google.com/store/apps/details?id=org.torproject.android) / [F-Droid](https://orbot.app/en/)
   - iPhone / iPad → [App Store](https://apps.apple.com/us/app/orbot/id1609461599)


2. **Откройте Orbot → нажмите Start**

Выберите **Coin Wallet** из списка, чтобы только wallet использовал Tor (необязательно, но рекомендуется)

Подождите, пока не появится надпись **"Подключено "** (10-30 секунд)


3. **Откройте Coin Wallet → Настройки → Сеть**

Включите **Использовать Tor**


4. **Проверьте статус**

В верхней панели появится **пурпурный значок лука Tor** → весь трафик теперь направляется через Tor
