การเริ่มต้นกับ Bitcoin ดูเหมือนจะเป็นความท้าทายใหญ่สำหรับผู้ที่สงสัยเกี่ยวกับความเรียบง่ายในการใช้งาน การค้นหาเครื่องมือที่เหมาะสมเพื่อรับประกันความเรียบง่ายนี้จึงกลายเป็นสิ่งสำคัญยิ่งสำหรับการยอมรับ Bitcoin ในฐานะสื่อกลางในการแลกเปลี่ยนและไม่ใช่เพียงแค่การเก็บรักษามูลค่าเท่านั้น



ในบทแนะนำนี้ เราจะมาดู Blue Wallet ซึ่งเป็น Bitcoin wallet ที่เรียบง่ายแต่มีประสิทธิภาพสูงที่ช่วยให้คุณจัดการบิตคอยน์ของคุณเองและยังสามารถสร้างสหกรณ์การจัดการตาม [Multisig](https://planb.academy/resources/glossary/multisig) (ไม่ต้องกังวล เราจะกลับมาที่นี่)

### Multisig Portfolio



Multisig (multi-signature) wallet แสดงถึง wallet ที่สร้างขึ้นจากการรวมกลุ่มของกระเป๋าเงิน Bitcoin จำนวนหนึ่ง (ขั้นต่ำ 2) ในประเภทของ wallet นี้ ขึ้นอยู่กับการกำหนดค่าและวิธีการที่เลือก การใช้จ่ายบิตคอยน์จะกลายเป็นการกระทำร่วมกันและร่วมมือกัน



ใน Blue Wallet คุณสามารถสร้างกระเป๋าเงินหลายลายเซ็นสำหรับสมาคมของคุณ ครอบครัวของคุณ บริษัทของคุณ ตลอดทั้งส่วนนี้ เราจะสำรวจทุกแง่มุมของ wallet ประเภทพิเศษนี้



เพิ่ม wallet ใหม่และเลือกประเภท **Multisig Vault** เพื่อสร้าง wallet แบบหลายลายเซ็น



![multisig-vault](assets/fr/10.webp)



กำหนดค่าการกำหนดค่า m-de-n ในองค์กรหลายลายเซ็นของคุณโดยคลิกที่ **การตั้งค่าห้องนิรภัย**.



⚠️ ในการกำหนดค่าแบบ m-of-n **m** แทนจำนวนลายเซ็นขั้นต่ำที่ต้องการเพื่ออนุมัติการทำธุรกรรม และ **n** แทนจำนวนกระเป๋าเงินในองค์กรของคุณ



อย่าลืมกำหนดจำนวนลายเซ็นขั้นต่ำ (m) สำหรับส่วนใหญ่ขององค์กรของคุณ ตัวอย่างเช่น การกำหนดค่าแบบหลายลายเซ็น 2 จาก 3 ต้องการให้กระเป๋าเงินสองใบในองค์กรของคุณลงนามในธุรกรรมก่อนที่จะสามารถดำเนินการได้



❗การกำหนดค่ารูปแบบ m-of-n ที่ n เท่ากับ m เป็นความเสี่ยงใหญ่ เมื่อสมาชิกสูญเสียการเข้าถึง wallet คุณจะสูญเสียการอนุญาตในการใช้จ่ายบิตคอยน์ใน wallet



นี่คือตัวอย่างบางส่วนของการกำหนดค่าที่เหมาะสมเพื่อให้มั่นใจในความปลอดภัยและการเข้าถึงบิตคอยน์:





- 2-de-3 multi signature.





- 5-de-7 multi signature.



![vault-settings](assets/fr/11.webp)



ปฏิบัติตามแนวทางที่ดีที่สุดโดยการเลือกใช้รูปแบบ P2WSH



❗ **[P2WSH](https://planb.academy/resources/glossary/p2wsh) หรือ Pay to Witness Script Hash** เป็นวิธีการล็อกที่ล็อกบิตคอยน์ที่ออกจากธุรกรรมของคุณ (Outputs) ไปยัง Hash ของสคริปต์ที่กำหนดเองซึ่ง Blue Wallet ตั้งค่าไว้ ข้อได้เปรียบหลักของการล็อกประเภทนี้คือช่วยลดขนาดข้อมูลของธุรกรรมและโดยนัยช่วยให้คุณจ่ายค่าธรรมเนียมการทำธุรกรรมที่ต่ำลง



สร้างหรือนำเข้าแต่ละกระเป๋าเงิน **n** ในการตั้งค่าของคุณ ในบทแนะนำของเรา เราจะใช้การตั้งค่าแบบหลายลายเซ็น 2-of-3 อย่าลืมบันทึกคำกู้คืนสำหรับแต่ละ wallet แยกกัน



![vault-keys](assets/fr/12.webp)





- รับบิตคอยน์



ในหน้าของคุณ Multisig wallet คุณจะพบประวัติการทำธุรกรรมและปุ่มรับและส่ง



การรับบิตคอยน์ใน wallet แบบหลายลายเซ็นเป็นกระบวนการเดียวกันกับเมื่อคุณอยู่ใน Bitcoin wallet มาตรฐาน





- ส่ง**บิตคอยน์**:



โดยการจัดการ wallet หลายลายเซ็น การใช้จ่ายบิตคอยน์กลายเป็นการกระทำที่ซับซ้อน ไม่ว่าจะกับคนอื่นหรือ wallet ที่สองของคุณเอง ลายเซ็นเดียวของ wallet ของคุณไม่เพียงพออีกต่อไป สิ่งนี้เพิ่มชั้นความปลอดภัยให้กับบิตคอยน์ของคุณ เพราะบุคคลที่ประสงค์ร้ายจะไม่สามารถใช้จ่ายบิตคอยน์เหล่านั้นได้เมื่อเขาได้ครอบครองเพียงหนึ่งในกุญแจส่วนตัวของคุณ



เช่นเดียวกับมาตรฐาน Bitcoin wallet ของ Blue Wallet คุณสามารถกำหนดผู้รับหลายคนในตัวเลือก **เพิ่มผู้รับ**



เมื่อทำการตรวจสอบธุรกรรมของคุณ คุณจะต้องใช้ลายเซ็นที่สองเพื่ออนุมัติการใช้จ่ายบิตคอยน์ อย่าลืมว่าเราอยู่ในรูปแบบลายเซ็นหลายตัว 2-จาก-3



ผู้ลงนาม wallet คนที่สอง หากเขาหรือเธอเป็นผู้ใช้ ก็สามารถลงนามในธุรกรรมได้แม้ว่าเขาหรือเธอจะอยู่นอกอินเทอร์เน็ต (ไม่มี Wi-Fi, ไม่มีข้อมูลมือถือ) โดยการสแกน QR code ของ [ธุรกรรมที่ลงนามบางส่วน](https://planb.academy/resources/glossary/psbt) ที่คุณเพิ่งสร้างขึ้น



![mutisig-send](assets/fr/13.webp)





- ก้าวไปอีกขั้นกับ **Multi signature** wallet:



บน Interface ของ wallet แบบหลายลายเซ็นของคุณ ให้คลิกที่ปุ่ม **จัดการคีย์**



โดยการลืมหนึ่งในคำกู้คืนของหนึ่งในกระเป๋าเงินที่ลงนาม (**ลืม seed นี้...**), คุณจะแจ้งให้ Blue Wallet ลบการสำรองข้อมูลของคำเหล่านี้ออกจากหน่วยความจำของมัน ดังนั้นคุณจะได้ทำการสำรองข้อมูลภายนอกแล้ว



![revoke-key](assets/fr/14.webp)



โดยการดำเนินการนี้ คุณจะเก็บเฉพาะกุญแจสาธารณะที่เชื่อมโยงกับคำกู้คืนเหล่านี้เท่านั้น



⚠️ การเก็บเฉพาะคีย์สาธารณะ (XPUB) ช่วยให้คุณเพิ่มระดับความปลอดภัยให้กับการกำหนดค่าลายเซ็นหลายตัวแบบ 2-of-3 ได้จริง ๆ แล้ว การเก็บคำกู้คืนทั้งหมดไว้ในที่เดียวเมื่อโทรศัพท์ของคุณถูกโจมตีอาจเป็นอันตรายได้ ผู้โจมตีที่เข้าถึงได้เพียง **VAULT** (คำสำคัญ) ที่คุณใช้ในการลงนามธุรกรรมของคุณ จะไม่สามารถขโมยบิตคอยน์ของคุณได้ (ต้องการลายเซ็นอย่างน้อย 02 ลายเซ็น) เพราะคีย์สาธารณะไม่สามารถใช้ในการลงนามธุรกรรมได้

## เปิดใช้งาน Lightning Network



Lightning Network ไม่ได้รับการสนับสนุนโดยตรงในแอปพลิเคชัน Blue Wallet อีกต่อไป



ใน **การตั้งค่า > การตั้งค่า Lightning** คุณสามารถเชื่อมโยง Lightning wallet ของคุณด้วยตนเองเมื่อรันโหนด Lightning Network Daemon (LND) ติดตั้ง LND Hub จากนั้นเชื่อมโยง wallet ของคุณโดยป้อนลิงก์ที่สร้างโดย Hub
