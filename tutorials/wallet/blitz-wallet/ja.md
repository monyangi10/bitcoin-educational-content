Bitcoinウォレットを選ぶ際、ユーザー体験は最も重要な判断基準のひとつです。このチュートリアルでは、シンプルさを設計の中心に据えたウォレット「Blitz Wallet」をご紹介します。**Spark**プロトコルの統合により、Blitzは市場で最もシンプルかつ多機能なBitcoinウォレットのひとつを実現し、技術的な管理を必要とせずに即時決済を可能にします。

## Blitz Walletとは？

Blitz Walletは、**self-custodial**かつ**open source**のBitcoinウォレットであり、あなたの主権と、直感的でスムーズなユーザー体験を重視しています。

[Blitz Wallet](https://blitz-wallet.com/)は、Android（Play Store）およびiOS（App Store）で利用可能なモバイルアプリケーションです。

従来のLightningウォレットでは決済チャネルや受信流動性の管理が必要ですが、Blitz Walletは**Sparkプロトコル**を活用してこれらの技術的な複雑さを解消しています。その結果、最初のサトシを受け取った瞬間から事前の設定なしに即時決済が可能になります。Blitzの目標は、資金のself-custodyを一切損なうことなく、Bitcoinでの支払いを一般的な決済アプリと同じくらいシンプルにすることです。

Blitz Walletは`ユーザー名@ドメイン.com`形式の**人間が読めるアドレス**（LNURL経由）にも対応しており、毎回長いinvoiceやQR codeを扱う必要なく、メールを送るのと同じくらい簡単にビットコインを送ることができます。

## Sparkプロトコルを理解する

実際の操作に入る前に、Blitz Walletの内部で動作している技術、**Sparkプロトコル**について理解しておくと役立ちます。

### Sparkとは？

Sparkは、Lightsparkチームが開発したBitcoin上に構築されたopen sourceのレイヤープロトコルです。ビットコインのself-custodyを維持しながら、即時かつ低コストの取引を可能にします。

二者間の**決済チャネル**に依存するLightning Networkとは異なり、Sparkは**statechain**（状態チェーン）と呼ばれる技術を使用します。基本原理は次の通りです：各取引ごとにブロックチェーン上でビットコインを移動させる代わりに、Sparkはあるユーザーから別のユーザーへ**支出権**を移転し、on-chainでの移動は発生しません。

### どのように機能するのか？

Sparkを直感的に理解するために、Spark上でビットコインを使うには2ピースのパズルを完成させる必要があると想像してみましょう：
- 1つのピースはユーザー（例えばAlice）が保持しています。
- もう1つのピースは**Spark Entity (SE)**と呼ばれるオペレーターのグループが保持しています。

対応する2つのピースの組み合わせだけが、ビットコインの支出を可能にします。

AliceがBobにビットコインを送りたい場合、次のようなことが起こります：BobとSEの間で新しいパズルが共同で作成されます。パズルの形は同じですが、構成するピースが変わります。今度はBobのピースがSEのピースに対応するようになります。SEが対応するピースを破棄したため、Aliceの古いピースは使用不能になります。ビットコインの所有権が移転しましたが、**ブロックチェーン上での取引は一切発生していません**。

Bobはその後、同じプロセスを繰り返してこれらのビットコインをCarolに送ることができ、以降も同様です。各転送はパズルのピースの置き換えによって機能し、on-chainでの資金移動は伴いません。

### なぜ安全なのか？

正当な疑問が生じます：SEが実際には古いピースを破棄しなかった場合はどうなるのでしょうか？

Spark Entityは単一の主体ではなく、独立したオペレーターのグループです。SEのピースは決して単一のオペレーターが保持することはありません。パズルの置き換えには複数のオペレーターの協力が必要です。転送時に**たった1人のオペレーターが誠実であれば**、古いパズルの再有効化を防ぐことができます。単独のオペレーターが秘密裏に古い有効なパズルを保持したり、後から再作成したりすることはできません。

さらに、Sparkには一方的な出金メカニズムが組み込まれています：SEの協力なしに、いつでもビットコインをon-chainで回収できます。このフォールバックメカニズムはSparkのアーキテクチャの不可欠な部分であり、資金へのアクセスにおいてネットワークに依存することがないことを保証します。

### Spark vs Lightning Network

SparkとLightningは競合関係ではなく、**補完的**な存在です。Blitz Walletは両方を統合しており、それぞれの利点を享受できます。

|                               | **Spark**                    | **Lightning Network** |
| ----------------------------- | ---------------------------- | --------------------- |
| **技術**                      | Statechains（状態チェーン）  | 決済チャネル          |
| **チャネル管理**              | 不要                         | 必要                  |
| **受信流動性**                | 不要                         | 必要                  |
| **即時取引**                  | はい                         | はい                  |
| **Self-custody**              | はい                         | はい                  |
| **Lightning互換性**           | はい（atomic swaps経由）     | ネイティブ            |

Lightning Networkは即時決済のための優れたプロトコルですが、初心者にとっては障壁となりうる技術的な制約（チャネル管理、受信流動性、オンラインノード等）があります。SparkはLightningとの互換性を維持しながら、これらの制約を取り除きます。

## インストールと設定

このチュートリアルではBlitz WalletのAndroid版を基に説明しますが、以下で紹介するすべてのプロセスはiOSでも同様に有効です。

![installation](assets/fr/01.webp)

Blitz Walletはself-custodyのウォレットであるため、**新しいウォレットを作成する**か、既存のウォレットの**リカバリーフレーズをインポートする**（12語または24語）かを選択できます。

ここでは、新しいウォレットの作成を行います。リカバリーフレーズのバックアップに関する推奨事項については、以下をご覧ください：

https://planb.academy/tutorials/wallet/backup/backup-mnemonic-22c0ddfa-fb9f-4e3a-96f9-46e2a7954270

❗ **重要**：これらの12語または24語のリカバリーワードは、**あなたのビットコインにアクセスするための唯一の鍵**です。Blitzはself-custodialウォレットです：BlitzもSparkも、あなたのリカバリーフレーズや資金にアクセスすることはできません。これらの単語を紛失すると、ビットコインへのアクセスを永久に失います。誰にも共有しないでください：これらの単語を持つ人は誰でもあなたのビットコインを使うことができます。

次に、ウォレットへのアクセスを保護するための**PINコード**を作成してください。

![setup-wallet](assets/fr/02.webp)

## Blitzを使い始める

Blitzでの取引は、他のほとんどのBitcoinウォレットよりも直感的です。インターフェースはミニマルで、送金、スキャン、受取の3つの主要アクションを中心に構成されています。

### ビットコインを受け取る

Blitzウォレットでビットコインを受け取るには、**「下矢印」**アイコン（↓）をクリックし、受け取りたいサトシの金額を入力し、任意の説明を追加すると、ウォレットが送金者に共有できるinvoice（請求書）を生成します。

⚠️ **注意**：サトシ（または「sat」）はビットコインの最小単位です：**1ビットコイン = 1億サトシ**。

Blitz Walletの特徴のひとつは、Bitcoinエコシステムの複数のネットワークとプロトコルに対応していることです：

- **Lightning Network**：非常に低い手数料で即時のマイクロペイメントを可能にするBitcoinのレイヤーのひとつ。日常の少額決済に最適です。

- **Bitcoin (on-chain)**：Bitcoinプロトコルのメインブロックチェーン。最大限のセキュリティとファイナリティが優先される高額取引に適しています。

- **Liquid Network**：Blockstreamが開発したBitcoinのsidechain（並行チェーン）で、Liquid Bitcoin（L-BTC）を使用した高速かつ秘匿性の高い取引を可能にします。

https://planb.academy/tutorials/wallet/mobile/blockstream-app-liquid-b3e4fb82-902e-4782-ad2b-a61ab05a543a

デフォルトではBlitzはLightning invoiceを生成しますが、**「Choose format」**（フォーマットを選択）ボタンをクリックして、サトシを受け取るネットワークを選択できます。

![receive-sats](assets/fr/03.webp)

### 連絡先を作成する

Blitz Walletは、連絡先システムによって繰り返しのビットコイン送金を簡素化します。

**Contacts**メニューでは、頻繁にやり取りするBlitzユーザー名やLightningアドレス（LNURL）を登録できます。

これにより、毎回QR codeをスキャンしたりアドレスを手動で入力したりすることなく、数クリックでこれらのアドレスにサトシを送ることができます。

### ビットコインを送る

一般的なビットコイン送金方法（QR codeのスキャン、アドレスの手動入力）に加えて、Blitzはいくつかの便利なオプションを提供しています：

- **画像から**（*From Image*）：フォトギャラリーからQR codeをインポートします。
- **クリップボードから**（*From Clipboard*）：以前コピーしたアドレスやinvoiceを貼り付けます。
- **手動入力**（*Manual Input*）：Bitcoinアドレス、Lightning invoice、または人間が読めるアドレス（例：`ユーザー名@walletofsatoshi.com`）を直接入力します。
- **連絡先から**：事前に登録した受取人を選択して、数クリックで送金します。

**Wallet**メニューで**「上矢印」**ボタン（↑）をクリックし、送金方法を選択し、送金額を入力し、説明を追加して、取引を確認します。

送金の最低金額は現在**1,000サトシ**です。

![send-bitcoin](assets/fr/05.webp)

## Blitzストア

ビットコインの送受金操作に加えて、Blitz Walletにはアプリ内で直接ビットコインを使ってデジタルサービスを購入できるストアが統合されています。

メインメニューから**Store**タブをクリックしてストアにアクセスします。ビットコインで直接、世界中の何千もの加盟店でギフトカードを購入できる**Bitrefill**プラットフォームへのアクセスも用意されています。

- **人工知能**：最新の生成AIモデル（Claude 3.5 Sonnet、GPT-4o、GPT-4o-mini、Gemini Flash 1.5）にアクセスし、クレジットをサトシで直接支払えます。ニーズに応じて複数のプラン（Casual、Pro、Power）が利用可能です。

![ia-credits](assets/fr/06.webp)

- **匿名SMS**：個人の電話番号を明かすことなく、世界中でSMSの送受信が可能です。送信メッセージごとにサトシで課金されます。

![sms-credit](assets/fr/07.webp)

- **VPN WireGuard**：Blitzストアから直接ビットコインで支払い、VPN WireGuardサブスクリプション（週間、月間、四半期）に加入してオンラインプライバシーを保護できます。お使いのデバイスにWireGuardクライアントアプリをダウンロードするだけで利用できます。

![wireguard](assets/fr/08.webp)

https://planb.academy/tutorials/exchange/centralized/bitrefill-8c588412-1bfc-465b-9bca-e647a647fbc1

## Blitz Walletの裏側：さらに深く

Blitz Walletの使いやすさの背後には、Bitcoinエコシステムの複数のレイヤーを組み合わせた、よく考えられた技術アーキテクチャがあります。

### 残高の分配

Blitz Walletは、必要に応じて異なるプロトコル間で資金を分配し、残高を透過的に管理します。残高が500,000サトシ未満の場合、Blitzは**Liquid Network**とatomic swapsを使用して、スムーズな体験を提供し、少額でもLightning Networkでの取引を可能にします。

このアプローチにより、新規ユーザーが背後のメカニズムを理解する必要なく、簡単に使い始めることができます。**設定 > Balance Info**メニューで、各レイヤー間の残高分配を確認できます。

![balance](assets/fr/09.webp)

### Lightningモード（オプション）

デフォルトでは、Blitz WalletはLiquid NetworkとSparkプロトコルを使用して、技術的な設定なしにスムーズな体験を提供します。しかし、Blitzでは**Lightningモード**を有効にすることもでき、残高が**500,000サトシ**（0.005 BTC）に達すると、自動的に決済チャネルが開設されます。

Lightningモードを有効にするには、**設定**に移動し、**技術設定**セクションの**Node Info**オプションをクリックします。

![enable-lightning](assets/fr/10.webp)

Sparkプロトコルのおかげで、この有効化は**オプション**です：Sparkはチャネルを開設したり受信流動性を管理したりすることなく、Lightning互換の支払いをすでに行うことができます。ネイティブLightningモードは、アプリケーション内に自身のLightningノードを持ちたい上級ユーザーにとって有用です。

### ポイントオブセール（PoS）

Blitz Walletには**ポイントオブセール**機能が統合されており、加盟店がアプリから直接ビットコインでの支払いを受け付けることができます。

**設定 > Point-of-sale**メニューで、以下を設定できます：

- 店舗の固有識別子
- 価格を表示するための現地法定通貨
- 従業員向けの指示
- 顧客向けのチップオプション

顧客は生成されたQR codeをスキャンするだけで、即座にビットコインで支払いを完了できます。
