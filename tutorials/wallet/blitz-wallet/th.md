## Blitz Wallet คืออะไร?

Blitz Wallet เป็นกระเป๋าเงิน Bitcoin แบบ **self-custodial** และ **open source** ที่เน้นอำนาจอธิปไตยของคุณ พร้อมประสบการณ์ผู้ใช้ที่ลื่นไหลและใช้งานง่าย

[Blitz Wallet](https://blitz-wallet.com/) เป็นแอปพลิเคชันมือถือที่ใช้ได้บน Android (Play Store) และ iOS (App Store)

ต่างจากกระเป๋าเงิน Lightning แบบดั้งเดิมที่ต้องจัดการช่องทางชำระเงินและสภาพคล่องขาเข้า Blitz Wallet ใช้ **โปรโตคอล Spark** เพื่อขจัดความซับซ้อนทางเทคนิคเหล่านี้ ผลลัพธ์คือ การชำระเงินแบบทันทีตั้งแต่ satoshi แรกที่ได้รับ โดยไม่ต้องตั้งค่าใดๆ ล่วงหน้า เป้าหมายของ Blitz คือทำให้การชำระเงินด้วย bitcoin ง่ายเหมือนแอปชำระเงินทั่วไป โดยไม่เคยลดทอนความเป็น self-custody ของเงินทุนคุณ

Blitz Wallet ยังรองรับ **ที่อยู่แบบอ่านได้** ในรูปแบบ `user@domain.com` (ผ่าน LNURL) ซึ่งช่วยให้ส่ง bitcoin ได้ง่ายเหมือนส่งอีเมล โดยไม่ต้องจัดการกับ invoice ยาวๆ หรือ QR code ในทุกธุรกรรม

## ทำความเข้าใจโปรโตคอล Spark

ก่อนเริ่มลงมือปฏิบัติ จะเป็นประโยชน์หากเข้าใจเทคโนโลยีที่ทำให้ Blitz Wallet ทำงานอยู่เบื้องหลัง นั่นคือ **โปรโตคอล Spark**

### Spark คืออะไร?

Spark เป็นโปรโตคอลเลเยอร์เสริมแบบ open source ที่สร้างบน Bitcoin พัฒนาโดยทีม Lightspark ช่วยให้ทำธุรกรรมได้ทันทีด้วยต้นทุนต่ำ ในขณะที่ยังคงรักษาความเป็น self-custody ของ bitcoin ของคุณ

ต่างจาก Lightning Network ที่อาศัย **ช่องทางชำระเงิน** ระหว่างสองฝ่าย Spark ใช้เทคโนโลยีที่เรียกว่า **statechain** (ห่วงโซ่สถานะ) หลักการพื้นฐานคือ แทนที่จะเคลื่อนย้าย bitcoin บน blockchain ในทุกธุรกรรม Spark จะโอน **สิทธิ์ในการใช้จ่าย** จากผู้ใช้คนหนึ่งไปยังอีกคนหนึ่ง โดยไม่มีการเคลื่อนไหว on-chain

### มันทำงานอย่างไร?

เพื่อทำความเข้าใจ Spark อย่างเข้าใจง่าย ลองจินตนาการว่าการใช้จ่าย bitcoin บน Spark ต้องต่อจิ๊กซอว์สองชิ้นให้สมบูรณ์:
- ชิ้นหนึ่งถือโดยผู้ใช้ (เช่น Alice)
- อีกชิ้นหนึ่งถือโดยกลุ่มผู้ดำเนินการที่เรียกว่า **Spark Entity (SE)**

เฉพาะการรวมกันของชิ้นจิ๊กซอว์ที่ตรงกันทั้งสองชิ้นเท่านั้นที่จะใช้จ่าย bitcoin ได้

เมื่อ Alice ต้องการส่ง bitcoin ให้ Bob สิ่งที่เกิดขึ้นคือ จิ๊กซอว์ใหม่ถูกสร้างร่วมกันระหว่าง Bob และ SE จิ๊กซอว์ยังคงรูปทรงเดิม แต่ชิ้นส่วนที่ประกอบขึ้นเปลี่ยนไป ตอนนี้ชิ้นของ Bob ตรงกับชิ้นของ SE ชิ้นเก่าของ Alice ใช้ไม่ได้แล้ว เพราะ SE ได้ทำลายชิ้นที่ตรงกับของ Alice ไปแล้ว ความเป็นเจ้าของ bitcoin ได้เปลี่ยนมือ **โดยไม่มีธุรกรรมใดๆ บน blockchain**

Bob สามารถทำซ้ำกระบวนการเดียวกันเพื่อส่ง bitcoin เหล่านี้ให้ Carol และต่อไปเรื่อยๆ การโอนแต่ละครั้งทำงานโดยการเปลี่ยนชิ้นจิ๊กซอว์ ไม่ใช่โดยการเคลื่อนย้ายเงินทุนแบบ on-chain

### ทำไมจึงปลอดภัย?

คำถามที่สมเหตุสมผลเกิดขึ้น: จะเกิดอะไรขึ้นถ้า SE ไม่ได้ทำลายชิ้นเก่าจริงๆ?

Spark Entity ไม่ใช่หน่วยงานเดียว แต่เป็นกลุ่มผู้ดำเนินการอิสระ ชิ้นของ SE ไม่เคยถือโดยผู้ดำเนินการคนเดียว การเปลี่ยนจิ๊กซอว์ต้องอาศัยความร่วมมือจากผู้ดำเนินการหลายราย เพียงแค่ **ผู้ดำเนินการคนเดียวที่ซื่อสัตย์** ในการโอนก็เพียงพอที่จะป้องกันการเปิดใช้งานจิ๊กซอว์เก่าอีกครั้ง ไม่มีผู้ดำเนินการคนใดสามารถเก็บจิ๊กซอว์เก่าที่ยังใช้งานได้อย่างลับๆ หรือสร้างมันขึ้นมาใหม่ในภายหลัง

นอกจากนี้ Spark ยังมีกลไกการถอนฝ่ายเดียว: คุณสามารถเรียกคืน bitcoin ของคุณแบบ on-chain ได้เสมอโดยไม่ต้องอาศัยความร่วมมือจาก SE กลไกสำรองนี้เป็นส่วนสำคัญของสถาปัตยกรรม Spark และรับประกันว่าคุณไม่เคยต้องพึ่งพาเครือข่ายในการเข้าถึงเงินทุนของคุณ

### Spark เทียบกับ Lightning Network

Spark และ Lightning ไม่ได้แข่งขันกัน แต่เป็น **ส่วนเสริมกัน** Blitz Wallet ผสานทั้งสอง ช่วยให้คุณได้รับประโยชน์จากข้อดีของแต่ละโปรโตคอล

|                               | **Spark**                    | **Lightning Network**       |
| ----------------------------- | ---------------------------- | --------------------------- |
| **เทคโนโลยี**                 | Statechains (ห่วงโซ่สถานะ)   | ช่องทางชำระเงิน              |
| **การจัดการช่องทาง**           | ไม่จำเป็น                     | จำเป็น                      |
| **สภาพคล่องขาเข้า**           | ไม่จำเป็น                     | จำเป็น                      |
| **ธุรกรรมแบบทันที**            | ใช่                           | ใช่                         |
| **Self-custody**              | ใช่                           | ใช่                         |
| **ความเข้ากันได้กับ Lightning** | ใช่ (ผ่าน atomic swaps)      | ดั้งเดิม                     |

Lightning Network ยังคงเป็นโปรโตคอลที่ยอดเยี่ยมสำหรับการชำระเงินแบบทันที แต่มีข้อจำกัดทางเทคนิค (การจัดการช่องทาง สภาพคล่องขาเข้า โหนดต้องออนไลน์...) ที่อาจเป็นอุปสรรคสำหรับผู้เริ่มต้น Spark ขจัดข้อจำกัดเหล่านี้ในขณะที่ยังคงเข้ากันได้กับ Lightning

## เริ่มต้นใช้งาน Blitz

การทำธุรกรรมด้วย Blitz นั้นใช้งานง่ายกว่ากระเป๋าเงิน Bitcoin ส่วนใหญ่ อินเทอร์เฟซเป็นแบบมินิมอลและเน้นสามการกระทำหลัก: ส่ง สแกน และรับ

### รับ bitcoin

หากต้องการรับ bitcoin ในกระเป๋าเงิน Blitz ของคุณ ให้คลิกที่ไอคอน **"ลูกศรลง"** (↓) ใส่จำนวน satoshi ที่คุณต้องการรับ เพิ่มคำอธิบายเพิ่มเติม (ไม่บังคับ) จากนั้นกระเป๋าเงินจะสร้าง invoice ที่คุณสามารถแชร์ให้ผู้ส่งได้

⚠️ **หมายเหตุ** : Satoshi (หรือ "sat") คือหน่วยที่เล็กที่สุดของ bitcoin: **1 bitcoin = 100,000,000 satoshi**

หนึ่งในจุดเด่นของ Blitz Wallet คือรองรับหลายเครือข่ายและโปรโตคอลในระบบนิเวศ Bitcoin:

- **Lightning Network** : หนึ่งในเลเยอร์เสริมของ Bitcoin ที่ช่วยให้ทำการชำระเงินขนาดเล็กได้ทันทีด้วยค่าธรรมเนียมต่ำมาก เหมาะสำหรับจำนวนเงินเล็กๆ ในชีวิตประจำวัน

- **Bitcoin (on-chain)** : blockchain หลักของโปรโตคอล Bitcoin เหมาะสำหรับธุรกรรมจำนวนเงินมากกว่าที่ความปลอดภัยสูงสุดและความสิ้นสุดเป็นสิ่งสำคัญ

- **Liquid Network** : sidechain (เชนคู่ขนาน) ของ Bitcoin ที่พัฒนาโดย Blockstream ช่วยให้ทำธุรกรรมได้รวดเร็วและเป็นส่วนตัวโดยใช้ Liquid Bitcoin (L-BTC)

https://planb.academy/tutorials/wallet/mobile/blockstream-app-liquid-b3e4fb82-902e-4782-ad2b-a61ab05a543a

โดยค่าเริ่มต้น Blitz จะสร้าง invoice แบบ Lightning แต่คุณสามารถเลือกเครือข่ายที่ต้องการรับ satoshi ได้โดยคลิกที่ปุ่ม **"Choose format"** (เลือกรูปแบบ)

![receive-sats](assets/fr/03.webp)

### สร้างรายชื่อผู้ติดต่อ

Blitz Wallet ทำให้การส่ง bitcoin ซ้ำๆ ง่ายขึ้นด้วยระบบรายชื่อผู้ติดต่อ

ในเมนู **Contacts** คุณสามารถบันทึกชื่อผู้ใช้ Blitz หรือที่อยู่ Lightning (LNURL) ที่คุณมีปฏิสัมพันธ์บ่อยๆ ได้

คุณจึงสามารถส่ง satoshi ไปยังที่อยู่เหล่านี้ได้ในไม่กี่คลิก โดยไม่ต้องสแกน QR code หรือกรอกที่อยู่ด้วยตนเองทุกครั้ง

### ส่ง bitcoin

นอกเหนือจากวิธีส่ง bitcoin แบบมาตรฐาน (สแกน QR code, กรอกที่อยู่ด้วยตนเอง) Blitz ยังมีตัวเลือกที่สะดวกหลายอย่าง:

- **จากรูปภาพ** (*From Image*) : นำเข้า QR code จากแกลเลอรีรูปภาพของคุณ
- **จากคลิปบอร์ด** (*From Clipboard*) : วางที่อยู่หรือ invoice ที่คัดลอกไว้ก่อนหน้านี้
- **กรอกด้วยตนเอง** (*Manual Input*) : กรอกที่อยู่ Bitcoin, invoice แบบ Lightning หรือที่อยู่แบบอ่านได้โดยตรง (เช่น `user@walletofsatoshi.com`)
- **จากรายชื่อผู้ติดต่อ** : เลือกผู้รับที่บันทึกไว้ล่วงหน้าเพื่อส่งในไม่กี่คลิก

ในเมนู **Wallet** คลิกที่ปุ่ม **"ลูกศรขึ้น"** (↑) เลือกวิธีการส่ง ใส่จำนวนเงินที่ต้องการส่ง เพิ่มคำอธิบาย แล้วยืนยันธุรกรรม

จำนวนขั้นต่ำสำหรับการส่งในปัจจุบันคือ **1,000 satoshi**

![send-bitcoin](assets/fr/05.webp)

## ร้านค้า Blitz

นอกเหนือจากการโอน bitcoin แล้ว Blitz Wallet ยังมีร้านค้าที่คุณสามารถใช้จ่าย bitcoin เพื่อซื้อบริการดิจิทัลได้โดยตรงจากแอปพลิเคชัน

จากเมนูหลัก คลิกที่แท็บ **Store** เพื่อเข้าถึงร้านค้า คุณยังจะพบการเข้าถึงแพลตฟอร์ม **Bitrefill** ที่ช่วยให้ซื้อบัตรของขวัญจากร้านค้าหลายพันแห่งทั่วโลกด้วย bitcoin โดยตรง

- **ปัญญาประดิษฐ์** : เข้าถึงโมเดล AI รุ่นล่าสุด (Claude 3.5 Sonnet, GPT-4o, GPT-4o-mini, Gemini Flash 1.5) และจ่ายเครดิตด้วย satoshi โดยตรง มีแพ็กเกจหลายระดับตามความต้องการของคุณ (Casual, Pro, Power)

![ia-credits](assets/fr/06.webp)

- **SMS แบบไม่ระบุตัวตน** : ส่งและรับ SMS ได้ทั่วโลกโดยไม่ต้องเปิดเผยหมายเลขโทรศัพท์ส่วนตัว บริการคิดค่าบริการเป็น satoshi สำหรับแต่ละข้อความที่ส่ง

![sms-credit](assets/fr/07.webp)

- **VPN WireGuard** : ปกป้องความเป็นส่วนตัวออนไลน์ของคุณด้วยการสมัครสมาชิก VPN WireGuard (รายสัปดาห์ รายเดือน หรือรายไตรมาส) ชำระเงินด้วย bitcoin ได้โดยตรงจากร้านค้า Blitz เพียงดาวน์โหลดแอป WireGuard client บนอุปกรณ์ของคุณเพื่อเริ่มใช้งาน

เบื้องหลังความง่ายในการใช้งานของ Blitz Wallet คือสถาปัตยกรรมทางเทคนิคที่ออกแบบมาอย่างดี ซึ่งผสานหลายเลเยอร์ของระบบนิเวศ Bitcoin

### การกระจายยอดคงเหลือของคุณ

Blitz Wallet จัดการยอดคงเหลือของคุณอย่างโปร่งใสโดยกระจายเงินทุนระหว่างโปรโตคอลต่างๆ ตามความจำเป็น เมื่อยอดคงเหลือของคุณน้อยกว่า 500,000 satoshi Blitz จะใช้ **Liquid Network** และ atomic swaps เพื่อมอบประสบการณ์ที่ลื่นไหลและเปิดใช้งานธุรกรรมบน Lightning Network แม้จะมีจำนวนเงินน้อย

แนวทางนี้รับประกันการเริ่มต้นใช้งานที่ง่ายสำหรับผู้ใช้ใหม่ โดยไม่จำเป็นต้องเข้าใจกลไกพื้นฐาน คุณสามารถดูการกระจายยอดคงเหลือระหว่างเลเยอร์ต่างๆ ได้ในเมนู **ตั้งค่า > Balance Info**

![balance](assets/fr/09.webp)

### โหมด Lightning (ไม่บังคับ)

โดยค่าเริ่มต้น Blitz Wallet ใช้ Liquid Network และโปรโตคอล Spark เพื่อมอบประสบการณ์ที่ลื่นไหลโดยไม่ต้องตั้งค่าทางเทคนิค อย่างไรก็ตาม Blitz ให้คุณสามารถเปิดใช้งาน **โหมด Lightning** ซึ่งจะเปิดช่องทางชำระเงินโดยอัตโนมัติเมื่อคุณมียอดคงเหลือถึง **500,000 satoshi** (0.005 BTC)

หากต้องการเปิดใช้งานโหมด Lightning ไปที่ **ตั้งค่า** จากนั้นในส่วน **ตั้งค่าทางเทคนิค** คลิกที่ตัวเลือก **Node Info**

![enable-lightning](assets/fr/10.webp)

ด้วยโปรโตคอล Spark การเปิดใช้งานนี้เป็น **ทางเลือก**: Spark ช่วยให้ทำการชำระเงินที่เข้ากันได้กับ Lightning ได้อยู่แล้วโดยไม่ต้องเปิดช่องทางหรือจัดการสภาพคล่องขาเข้า โหมด Lightning แบบดั้งเดิมยังคงมีประโยชน์สำหรับผู้ใช้ขั้นสูงที่ต้องการมีโหนด Lightning ของตัวเองภายในแอปพลิเคชัน

### จุดขาย (PoS)

Blitz Wallet มีฟีเจอร์ **จุดขาย** ที่ช่วยให้ผู้ค้าสามารถรับชำระเงินด้วย bitcoin ได้โดยตรงจากแอปพลิเคชัน

ในเมนู **ตั้งค่า > Point-of-sale** คุณสามารถกำหนดค่า:

- รหัสเฉพาะของร้านค้าของคุณ
- สกุลเงินท้องถิ่นสำหรับแสดงราคา
- คำแนะนำสำหรับพนักงานของคุณ
- ตัวเลือกทิปสำหรับลูกค้าของคุณ

ลูกค้าของคุณเพียงแค่สแกน QR code ที่สร้างขึ้นเพื่อทำการชำระเงินด้วย bitcoin ได้ทันที
