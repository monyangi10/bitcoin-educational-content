## Wallet ที่ทรงพลัง


Nunchuk มาถึงในช่วงปลายปี 2020 พร้อมด้วยปรัชญาที่ชัดเจน: เพื่อทำให้การใช้ลายเซ็นหลายตัวเป็นมาตรฐาน ดังนั้นจึงถูกออกแบบมาเพื่อทำหน้าที่ขั้นสูงมาก โดยมีทางเลือกที่มีคุณค่าในการสร้างการออกแบบโดยตรงบน Bitcoin Core ซึ่งเป็นซอฟต์แวร์อ้างอิงสำหรับระบบนิเวศ Bitcoin



หลังจากการพัฒนาและใช้งานมากกว่า 4 ปี ตอนนี้มันพร้อมที่จะทดลองใช้ในขนาดใหญ่แล้ว หากคุณเป็นมือใหม่และไม่คุ้นเคยกับ Nunchuk คู่มือนี้จะช่วยให้คุณก้าวแรกและค้นพบซอฟต์แวร์นี้ ซึ่งคุณจะสามารถเรียนรู้เกี่ยวกับฟังก์ชันขั้นสูงหลังจากที่คุณผ่านการใช้งานเบื้องต้นไปแล้ว บทแนะนำนี้มุ่งเน้นไปที่ผู้ใช้ระดับกลางที่มีทักษะที่จำเป็นในการทำตามขั้นตอนทั้งหมด แต่ก็สามารถเป็นแรงบันดาลใจให้ทุกคนค้นหาวิธีเพิ่มทักษะได้ เราจะเริ่มต้นด้วยเวอร์ชันมือถือ และการชี้แจงนี้เป็นสิ่งจำเป็น เนื่องจาก Nunchuk มีซอฟต์แวร์ที่สามารถทำงานบนคอมพิวเตอร์ได้เช่นกัน



## ดาวน์โหลด


ขั้นตอนแรกคือการตัดสินใจว่าจะดาวน์โหลดแอปจากที่ไหน ไปที่ [เว็บไซต์ทางการ](https://nunchuk.io/) ซึ่งคุณสามารถหาข้อมูลเอกสารบางส่วน (ไม่มากแต่ก็เป็นการเริ่มต้น) การนำเสนอคุณสมบัติ และที่ส่วนท้ายของหน้า ลิงก์ดาวน์โหลดทั้งหมด



📌 สำหรับบทแนะนำนี้ ฉันตัดสินใจที่จะแสดงวิธีการดาวน์โหลด Software Wallet จากที่เก็บ Github และวิธีการตรวจสอบการปล่อยเวอร์ชันก่อนที่จะติดตั้งบนโทรศัพท์มือถือของคุณ **ขั้นตอนต่อไปนี้สามารถทำได้จากคอมพิวเตอร์ของคุณเท่านั้น** ดังนั้นฉันขอแนะนำให้คุณทำตามขั้นตอนเหล่านี้ทั้งหมดจากเดสก์ท็อปหรือแล็ปท็อปของคุณ และ - หลังจากการตรวจสอบทั้งหมด - โอนไฟล์ `.apk` ไปยังโทรศัพท์มือถือของคุณ



![image](assets/en/01.webp)



หากทักษะของคุณยังไม่ก้าวหน้ามากนัก คุณอาจตัดสินใจดาวน์โหลด `.apk` จากร้านค้าอย่างเป็นทางการและข้ามไปยังส่วนการกำหนดค่าในบทเรียนนี้โดยตรง แต่ถ้าคุณต้องการก้าวกระโดด ให้ติดตามขั้นตอนต่อไปทีละขั้นตอน

## การตั้งค่า


Nunchuk นำเสนอด้วยหน้าต่าง _Home_ ซึ่งเข้าใจปรัชญาการทำงานได้ง่าย และเราจะอธิบายเพิ่มเติมในอีกสักครู่



ที่ด้านล่างคุณสามารถพบเมนูต่างๆ และในขั้นตอนแรก ให้เลือก _Profile_ เพื่อเข้าถึงการตั้งค่า



![image](assets/en/10.webp)



จากนั้นเลือก _การตั้งค่าการแสดงผล_ โดยยังคงไม่สนใจคำเชิญให้สร้างบัญชี



![image](assets/en/14.webp)



ในหน้าจอด้านล่างนี้ คุณสามารถตรวจสอบได้ว่า Wallet ออนไลน์หรือไม่ และคุณสามารถเชื่อมต่อเซิร์ฟเวอร์ของคุณได้ โดยให้ความสนใจเป็นพิเศษกับคำแนะนำที่ลิงก์ที่เสนอโดยการคลิกที่ _คู่มือนี้_



![image](assets/en/15.webp)



บันทึกการตั้งค่าด้วยคำสั่ง _Save network settings_ กลับไปที่เมนู _Profile_ และเลือก _Security settings_



![image](assets/en/16.webp)



จากเมนูนี้คุณสามารถตั้งค่าการป้องกันการเปิดแอปได้ เพื่อป้องกันการเข้าถึงที่ไม่พึงประสงค์ คุณสามารถป้องกัน Nunchuk ด้วยไบโอเมตริกของโทรศัพท์ และ/หรือเพิ่มรหัส PIN ความปลอดภัย



![image](assets/en/17.webp)



โปรดดูที่เมนู _About_ ซึ่งคุณจะพบได้เสมอในหน้าต่าง _Profile_



![image](assets/en/18.webp)



ซึ่งจะช่วยให้คุณตรวจสอบเวอร์ชันของแอป หรือ ติดต่อผู้พัฒนาได้หากจำเป็น



![image](assets/en/19.webp)



## การสร้างคีย์และ Wallet


ตามที่สามารถเดาได้ง่ายจากปรัชญาของ Nunchuk ซอฟต์แวร์นี้ถูกออกแบบมาเพื่อเป็นเครื่องมือที่มีประโยชน์สำหรับการจัดการกระเป๋าเงินหลายลายเซ็น เพื่อทำหน้าที่นี้ Nunchuk อนุญาตให้สร้าง Wallet โดยแยกออกจากกุญแจที่จำเป็นในการจัดเตรียมลายเซ็นดิจิทัล



ในความเป็นจริง การดำเนินงานที่เหมาะสมของ Nunchuk เกี่ยวข้องกับการสร้างกระเป๋าเงินที่สามารถดูได้เท่านั้น ซึ่งขึ้นอยู่กับกุญแจที่สามารถเป็น "Colds" ได้



ในหน้าจอก่อนหน้านี้คุณอาจสังเกตเห็นว่ามีเมนูที่ด้านล่างที่เรียกว่า _Keys_ หากคุณเพิ่งดาวน์โหลด Nunchuk ในทั้ง _Home_ และ _Keys_ คุณจะเห็นปุ่มใหญ่ที่เชิญชวนให้คุณเพิ่มคีย์, _Add Key_.



![image](assets/en/20.webp)


![image](assets/en/21.webp)



**นี่คือวิธีการทำงานของ Nunchuk:** ก่อนอื่นคุณต้อง generate/นำเข้ากุญแจ จากนั้นคุณสร้าง Wallet โดยกำหนดค่าให้เลือกว่ากุญแจใดจะอนุญาตการปลดล็อกเงินที่เก็บไว้ในนั้น



แม้ในกรณีของ Wallet singlesig คุณจะสร้างคีย์ก่อนและหลังจากนั้นจึงสร้าง Wallet และนั่นคือสิ่งที่เราจะทำตอนนี้ โดยเริ่มจาก Wallet singlesig เพื่อทำลายน้ำแข็งและค้นพบฟังก์ชันของ Nunchuk



คลิก _Add Key_



![image](assets/en/22.webp)



Nunchuk แสดงอุปกรณ์ลายเซ็นที่รองรับหลายรายการ แต่เพื่อเริ่มต้น ให้เลือก _Software_



![image](assets/en/23.webp)



Nunchuk จะ generate คำช่วยจำที่จะถูกเก็บไว้ในอุปกรณ์ จากนั้นคุณจำเป็นต้องจดลำดับของคำสำหรับการสำรองข้อมูล สร้างสภาพแวดล้อมที่ดีที่สุดและมั่นใจว่าคุณมีเวลาที่จะทำอย่างดีและเงียบ ซอฟต์แวร์จะแสดงคำช่วยจำเพียงครั้งเดียว ไม่ว่าคุณจะเลือกแสดงตอนนี้หรือตอนหลัง ดังนั้นเลือก _สร้างและสำรองข้อมูลตอนนี้_



![image](assets/en/24.webp)



Nunchuk สร้างประโยคช่วยจำ 24 คำ ซึ่งจะปรากฏบนหน้าจอถัดไปทันที



![image](assets/en/25.webp)



จากนั้นดำเนินการตรวจสอบอย่างรวดเร็ว โดยขอให้คุณเลือกคำที่ถูกต้องจาก 3 ตัวเลือก ซึ่งตรงกับหมายเลขในลำดับช่วยจำ


หากคุณเขียนคำช่วยจำถูกต้องแล้ว ปุ่ม _Continue_ จะสามารถใช้งานได้ กดเพื่อดำเนินการต่อ



![image](assets/en/26.webp)



ตั้งชื่อคีย์ของคุณและกด _Continue_



![image](assets/en/27.webp)



เมื่อสิ้นสุดขั้นตอนเหล่านี้ คุณจะถูกถามว่าต้องการเพิ่ม [passphrase](https://planb.academy/en/resources/glossary/passphrase-bip39) ลงในวลีช่วยจำของคุณหรือไม่ หากคุณไม่มีความรู้ที่จำเป็นเกี่ยวกับวิธีการใช้ passphrase การจัดการสำรองข้อมูล หรือวิธีการทำงานของมัน ฉันขอแนะนำให้คุณเลือก _ฉันไม่ต้องการรหัสผ่านเพิ่มเติม_



![image](assets/en/28.webp)



คีย์ถูกสร้างขึ้นแล้วและแสดงให้คุณเห็นในเมนู:




- ด้วย _Key Spec_ จะมีการระบุลายนิ้วมือหลัก
- คุณมีการตั้งค่า จุดสามจุดที่มุมขวาบน ที่คุณสามารถลบคีย์หรือเซ็นข้อความ
- ถัดจากชื่อของคีย์ คุณจะพบไอคอนรูปปากกา เมื่อคลิกที่ไอคอนนี้ คุณสามารถแก้ไขชื่อของคีย์ได้ เช่น เพื่อจัดระเบียบคีย์ของคุณในอนาคต
- ในฐานะคำสั่งสุดท้าย คุณสามารถตรวจสอบสถานะสุขภาพของคีย์: โดยการกด _Run health check_ คุณสามารถให้แอปตรวจสอบว่าคีย์ถูกบุกรุกหรือไม่



เมื่อคุณพร้อมแล้ว ให้คลิก _Done_



![image](assets/en/29.webp)



ในเมนู _Keys_ คุณจะเห็นคีย์แรกของคุณปรากฏขึ้น



![image](assets/en/30.webp)



โดยไปที่เมนู _Home_ จะมีตัวเลือกให้สร้าง Wallet ปรากฏขึ้น คลิก _Create new wallet_



![image](assets/en/31.webp)



Nunchuk แสดงให้คุณเห็นถึงความเป็นไปได้หลายประการที่เกี่ยวข้องกับบริการที่บริษัทนำเสนอ ซึ่งส่วนใหญ่ไม่ใช่หัวข้อของบทแนะนำนี้



ในคู่มือนี้เราจะสร้าง _Hot Wallet และ _Custom wallet_ โดยการอธิบายรายละเอียดต่างๆ


มาเริ่มกันที่ _กระเป๋าสตางค์แบบกำหนดเอง_



![image](assets/en/32.webp)



ในวิธีที่ง่าย ๆ แอปจะขอให้คุณตั้งชื่อ Wallet ใหม่นี้และเลือกสคริปต์สำหรับที่อยู่ สำหรับบทแนะนำนี้ฉันเลือกที่จะปล่อยให้เป็นการตั้งค่าเริ่มต้น, _Native segwit_ เมื่อคุณเสร็จแล้ว เลือก _Continue_



![image](assets/en/33.webp)



การกำหนดค่า Wallet จะขอให้คุณตั้งค่าด้วยคีย์ใดที่เงินทุนของ Wallet นี้จะถูกปลดล็อก หากมีคีย์หลายตัว คุณจะเห็นรายการให้เลือก เราในขณะนี้ได้สร้างเพียงหนึ่งเดียว ดังนั้นเราจึงเลือกทำเครื่องหมายถูกที่ตัวนั้น ในมุมขวาล่างคุณสามารถเห็นว่า Nunchuk จะขอให้คุณตั้งค่าลายเซ็นหลายตัวของ Wallet ในอนาคตของคุณ เพิ่มจำนวน _คีย์ที่จำเป็น_



![image](assets/en/34.webp)



เนื่องจากเรากำลังสร้าง singlesig เราจึงปล่อยให้เป็น `1` และคลิก _Continue_



สุดท้าย หน้าจอยืนยันจะปรากฏขึ้น ซึ่งคุณสามารถตรวจสอบคุณสมบัติของ Wallet ได้:




- ชื่อ
- `1/1 Multisig` tage, ซึ่งเป็นวิธีที่ Nunchuk ตั้งชื่อ Wallet singlesig
- ประเภทสคริปต์, `Native SegWit`
- คีย์ `Keys` พร้อมด้วยลายนิ้วมือและเส้นทางการสืบทอด



เมื่อคุณพอใจแล้ว ให้กด _Create wallet_



![image](assets/en/35.webp)



Wallet ได้ถูกสร้างขึ้นแล้ว และคุณสามารถดาวน์โหลดไฟล์ [.BSMS](https://github.com/Bitcoin/bips/blob/master/bip-0129.mediawiki) เป็นการสำรองข้อมูลได้ หากต้องการกลับไปที่เมนูหลัก ให้คลิกที่ลูกศรที่มุมซ้ายบน



![image](assets/en/36.webp)



คุณอยู่ใน _Home_ ซึ่งคุณจะเห็นรายงาน Wallet ที่สร้างขึ้นใหม่แสดงยอดคงเหลือและสถานะของการเชื่อมต่อ โดยการคลิกในพื้นที่สีน้ำเงิน คุณสามารถเข้าถึงฟังก์ชันหลักของ Wallet ได้



![image](assets/en/37.webp)





- ไอคอนรูปเลนส์ที่มุมขวาบนช่วยให้คุณค้นหาธุรกรรมได้;
- `View Wallet config` ให้การเข้าถึงเมนูการกำหนดค่า ซึ่งคุณสามารถแก้ไขชื่อของ Wallet และเปิดใช้งานตัวเลือกขั้นสูง ที่มุมขวาบน (ซึ่งคุณไม่สามารถถ่ายภาพหน้าจอได้) ที่นี่คุณสามารถส่งออกการกำหนดค่า Wallet, ป้ายกำกับ, แทนที่คีย์, เปลี่ยน [gap limit](https://planb.academy/en/resources/glossary/gap-limit) และอื่น ๆ

เราได้รับ UTXO มาถึงที่อยู่แรกนี้แล้ว,

### การใช้จ่ายธุรกรรมด้วยการควบคุมเหรียญ



เมื่อ UTXO ตัวที่สองนี้มาถึงแล้ว ให้กลับไปที่หน้าจอหลักของ Wallet เพื่อตรวจสอบสถานะของธุรกรรมที่เข้ามาทั้งสองรายการ และที่สำคัญที่สุด คลิกที่ตัวเลือก _View coins_



![image](assets/en/42.webp)



ที่นี่คุณจะเห็น UTXOs แต่ละรายการ คุณสามารถเลือกดูรายการใดรายการหนึ่งโดยเฉพาะได้โดยคลิกที่ลูกศรถัดจากจำนวนเงิน



![image](assets/en/43.webp)



และตรวจสอบเมื่อมาถึง, คำอธิบาย, บล็อก UTXO เพื่อไม่ให้ใช้จ่ายและอื่น ๆ



![image](assets/en/44.webp)



แต่ถ้าคุณกลับไปที่เมนู _Coins_ โดยการคลิกที่ลูกศรที่มุมขวาบน คุณสามารถเปิด "Coin Control" เพื่อใช้จ่าย UTXOs ของคุณในวิธีที่ควบคุมได้มากขึ้น



ในตัวอย่างต่อไปนี้ ฉันเลือกที่จะเลือก UTXO จาก 21,000 Sats แล้วคลิกสัญลักษณ์ที่มุมล่างซ้าย



![image](assets/en/45.webp)



Nunchuk เปิดหน้าต่าง _ธุรกรรมใหม่_ โดยอัตโนมัติเพื่อใช้จ่าย UTXO นี้ ในธุรกรรมการใช้จ่าย ก่อนอื่นคุณต้องตั้งค่าจำนวนเงินด้วยตนเองหรือโดยการเลือก _ส่งทั้งหมดที่เลือก_ เพื่อส่งยอดคงเหลือการควบคุมเหรียญทั้งหมด โดยไม่สร้างเศษเหลือ เมื่อกำหนดจำนวนเงินแล้ว ให้เลือก _ดำเนินการต่อ_



![image](assets/en/46.webp)



ตอนนี้ Nunchuk แสดงตำแหน่งที่วางที่อยู่เพื่อโอนเงินเหล่านี้ รายละเอียดคำอธิบาย และสรุปการทำธุรกรรม



![image](assets/en/47.webp)



การเลือก _สร้างธุรกรรม_ จะมอบหมายการจัดการค่าธรรมเนียมและธุรกรรมอัตโนมัติให้กับแอป ฉันแนะนำให้เลือก _ธุรกรรมที่กำหนดเอง_ เพื่อการควบคุมที่มากขึ้น



ในหน้าจอใหม่นี้ สิ่งสำคัญคือต้องเลือก




- _หักค่าธรรมเนียมจากจำนวนเงินที่ส่ง_ เพื่อป้องกันไม่ให้ค่าธรรมเนียมถูกจ่ายโดย UTXO อื่นที่มีอยู่ใน Wallet ใช้จ่ายและสร้างเศษเหลือ (ซึ่งเป็นการสูญเสียความเป็นส่วนตัวที่สามารถหลีกเลี่ยงได้);
- จากนั้นตั้งค่าธรรมเนียมด้วยตนเองหลังจากตรวจสอบบน explorer แล้ว



เมื่อทำตามขั้นตอนเหล่านี้เสร็จแล้ว ให้คลิกที่ _Continue_



![image](assets/en/48.webp)



หน้าจอถัดไปคือสรุปการทำธุรกรรมทั้งหมด หากทุกอย่างถูกต้อง ให้ยืนยันโดยเลือก _ยืนยันและสร้างธุรกรรม_



![image](assets/en/49.webp)



ด้วย _Pending signatures_ Nunchuk แจ้งเตือนคุณว่าธุรกรรมกำลังรอการลงนามของคุณเพื่ออนุมัติการใช้จ่าย ซึ่งคุณสามารถลงนามได้โดยคลิกที่ _Sign_



![image](assets/en/50.webp)



คำสั่ง _Broadcast_ ปรากฏที่ด้านล่างเพื่อเผยแพร่ธุรกรรมที่เสร็จสิ้นและลงนามแล้ว



![image](assets/en/51.webp)



### การใช้จ่ายจากเมนู _Send_



ในขณะที่อยู่บนหน้าหลักของ Wallet เราเห็นธุรกรรมกำลังออกและรอการยืนยัน เราใช้เมนู _Send_ เพื่อจำลองค่าใช้จ่ายรายวัน



![image](assets/en/52.webp)



การคลิก _Send_ จะนำหน้าจอสำหรับส่งธุรกรรมขึ้นมา ซึ่งเหมือนกับหน้าจอที่เพิ่งเห็น แต่ไม่ต้องผ่านการควบคุมเหรียญ



ในตัวอย่างที่สองนี้ ฉันตัดสินใจเลือก _Custom transaction_ และส่งจำนวนเงินทั้งหมด แต่ฉันสามารถตั้งค่าได้ด้วยตนเอง เมื่อคุณตัดสินใจเกี่ยวกับจำนวนเงินที่จะส่งแล้ว กด _Continue_



![image](assets/en/53.webp)



จากนั้นให้สร้างกรณีเสมอว่าค่าธรรมเนียมถูกหักออกจาก UTXO ที่เป็นปัญหา (ในตัวอย่างนี้ตัวเลือกถูกบังคับ เพราะมีเพียงหนึ่งเดียว) ปรับค่าธรรมเนียมด้วยตนเองตามสถานการณ์ในขณะนั้นใน Mempool และกด _Continue_



![image](assets/en/54.webp)



หากหน้าจอสรุปเป็นที่น่าพอใจ ให้เลือก _ยืนยันและสร้างธุรกรรม_



![image](assets/en/55.webp)



ลงนามธุรกรรมด้วย _Sign_



![image](assets/en/56.webp)



และเผยแพร่ไปยังเครือข่าย



![image](assets/en/57.webp)



Wallet อยู่ในจุดนี้โดยมียอดคงเหลือเป็นศูนย์และประวัติกำลังถูกอัปเดต

## การสร้าง "Hot Wallet"

ในเมนู _Home_ ของ Nunchuk ที่แสดงรายการของ Wallets ให้คลิกที่ `+` ที่มุมขวาบน

![image](assets/en/60.webp)

![image](assets/en/61.webp)



หลังจากผ่านไปไม่กี่อึดใจ Wallet จะถูกสร้างขึ้นและปรากฏในรายการด้วยสีออกน้ำตาล นี่คือสีที่ Nunchuk ใช้เตือนคุณว่ายังไม่ได้สำรองข้อมูล Wallet



![image](assets/en/62.webp)



คลิกที่ชื่อของ Wallet เพื่อเข้าถึงการกำหนดค่า และคุณอาจสังเกตเห็นคำเชิญให้สำรองวลีช่วยจำทันที

ขั้นตอนเหมือนกับที่เราเคยเห็นมาก่อน ดังนั้นเราจะไม่ย้ำอีกครั้ง เมื่อเสร็จแล้ว Nunchuk จะพาคุณไปยังหน้าคีย์ที่เกี่ยวข้อง ซึ่งคุณสามารถแก้ไขได้เหมือนกับที่คุณสร้างด้วยขั้นตอน _Custom_

## โปรดจำไว้เพื่อดำเนินการต่ออย่างอิสระ


เช่นเดียวกับที่มีลำดับสำหรับการสร้าง นั่นคือ การสร้างคีย์ก่อนแล้วจึงสร้าง Wallet คุณจะต้องรักษาลำดับย้อนกลับสำหรับการลบรายการเหล่านี้ออกจากแอปของคุณ



หากคุณจำเป็นต้องลบหนึ่งในคีย์ คุณควรมีวิสัยทัศน์ล่วงหน้าในการลบ Wallet หรือ Wallets ซึ่งใช้หนึ่งในคีย์ลายเซ็นสำหรับการทำธุรกรรม: ก่อนอื่นคุณลบ Wallets และหลังจากนั้นจึงลบคีย์ หากคุณไม่ปฏิบัติตามลำดับนี้ คุณจะพบว่าตัวเองไม่สามารถลบคีย์ได้



ตอนนี้ที่คุณรู้วิธีเริ่มต้นกับ Nunchuk แล้ว คุณสามารถศึกษาต่อเกี่ยวกับแอปนี้และค้นพบความลับของมันได้ ในบทแนะนำนี้เราได้เพียงแค่ก้าวแรกเท่านั้น แต่ยังมีการใช้งานที่ซับซ้อนมากขึ้นและความต้องการขั้นสูงที่ Software Wallet นี้สามารถช่วยคุณตอบสนองได้
