บทแนะนำทีละขั้นตอนนี้แสดงวิธีการเขียนข้อความบน Blockchain Bitcoin โดยใช้ Wallet Electrum การดำเนินการนี้ใช้คำสั่ง OP_RETURN เพื่อแทรกข้อความลงในธุรกรรม ซึ่งสามารถมองเห็นได้สาธารณะบน Blockchain ทำตามขั้นตอนง่าย ๆ เหล่านี้เพื่อการลงทะเบียนที่ประสบความสำเร็จ

## ข้อกำหนดเบื้องต้น





- คอมพิวเตอร์ (Windows, macOS หรือ Linux)
- การเชื่อมต่ออินเทอร์เน็ต
- ไม่กี่ซาโตชิ (Sats) หรือบิตคอยน์ (BTC) ใน Wallet ของคุณเพื่อครอบคลุมจำนวนธุรกรรมและค่าธรรมเนียม
- ตัวแปลงข้อความเป็นเลขฐานสิบหก (เช่น เว็บไซต์ออนไลน์) หรือเครื่องมือเฉพาะ เช่น [ตัวสร้างสคริปต์ OP_RETURN นี้](https://resources.davidcoen.it/opreturnelectrum/)

1. เปิดตัว Electrum.


2. เลือก สร้าง Wallet ใหม่ หรือ กู้คืน Wallet ที่มีอยู่แล้ว หากคุณมีวลี seed (วลีกู้คืน)


3. ทำตามคำแนะนำเพื่อกำหนดค่า Wallet ของคุณ :

4. ตั้งรหัสผ่านเพื่อรักษาความปลอดภัยให้กับ Wallet ของคุณ

## ขั้นตอนที่ 4: เตรียมข้อความที่จะจารึก



เลือกข้อความที่คุณต้องการป้อน (เช่น `Thanks Satoshi`) หมายเหตุ: ข้อความ OP_RETURN จำกัดที่ 80 ไบต์ หรือประมาณ 80 อักขระ ASCII



*ใช้เวลาสักครู่ในการคิด: สิ่งที่คุณเขียนบน Blockchain Bitcoin นั้นเป็นนิรันดร์และเข้าถึงได้สำหรับทุกคน ดังนั้น :*




- ทิ้งไว้ซึ่งการแสดงออกที่งดงามของความเป็นมนุษย์ของเรา,
- หลีกเลี่ยงการใส่เนื้อหาที่คุณอาจเสียใจในภายหลัง



*Blockchain พื้นที่และบิตคอยน์ของคุณมีค่า ใช้อย่างชาญฉลาดและด้วยความตั้งใจ*



แปลงข้อความของคุณเป็นเลขฐานสิบหก :




- คุณสามารถใช้[เครื่องมือออนไลน์](https://www.rapidtables.com/convert/number/ascii-to-hex.html) แต่ระวังอย่าประมวลผลข้อมูลที่ละเอียดอ่อนที่นั่น (แม้ว่าในหลักการแล้ว ข้อมูลที่ตั้งใจจะเผยแพร่บน Blockchain Bitcoin ผ่าน OP_RETURN จะไม่มีปัญหาด้านความลับใด ๆ);
- เพื่อความเป็นส่วนตัวที่มากขึ้น ให้ทำการแปลงในเครื่องโดยใช้ Python ขนาดเล็ก:



```py
#!/usr/bin/env python3

def main():
ascii_text = input("Enter ASCII text: ")
try:
hex_output = ascii_text.encode('ascii').hex()
print(hex_output)
except UnicodeEncodeError:
print("Error: Input contains non-ASCII characters.", file=sys.stderr)

if __name__ == "__main__":
import sys
main()
```



ตัวอย่าง: `Thanks Satoshi` ใน ASCII ให้ `5468616e6b73205361746f736869` ในเลขฐานสิบหก



เตรียมสคริปต์การทำธุรกรรม นี่คือตัวอย่างของรูปแบบที่คาดหวัง:



```script
bc1q879cv4p5q6s9537orange3zss33d3turzad8, 0.00001
script(OP_RETURN 5468616e6b73205361746f736869), 0
```



ซึ่งประกอบด้วย :





- ที่อยู่ปลายทาง**: ที่อยู่ Bitcoin ที่ถูกต้อง ที่นี่, `bc1q879cv4p5q6s9537orange3zss33d3turzad8`. นี่อาจเป็นที่อยู่ของคุณเอง หากคุณต้องการคืนเงินที่โอนไปให้ตัวคุณเอง;
- จำนวนที่โอน**: จำนวนของธุรกรรม, ที่นี่ `0.00001` BTC. **โปรดทราบ**: เนื่องจากหน่วยที่ใช้ใน Electrum คือ BTC, จำนวนที่ระบุในสคริปต์ธุรกรรมต้องแสดงเป็น BTC เช่นกัน, และไม่ใช่ใน Sats ;
- สคริปต์ **OP_RETURN**: ข้อความที่ถูกแปลงเป็นเลขฐานสิบหกนำหน้าด้วยสคริปต์ (`OP_RETURN <messsage>), 0` ที่นี่ `5468616e6b73205361746f736869` สำหรับข้อความในเลขฐานสิบหก



⚠️ **คำเตือน**: เป็นสิ่งสำคัญมากที่จะต้องระบุ `0` หลังจาก OP_RETURN เนื่องจาก opcode นี้จะทำให้สคริปต์ไม่ถูกต้อง ทำให้ผลลัพธ์ไม่สามารถใช้จ่ายได้อย่างถาวร โหนดเครือข่ายจะลบผลลัพธ์นี้ออกจากชุด UTXO ของพวกเขา หากคุณป้อนค่าที่ไม่ใช่ `0` ค่านั้นจะสูญหายไปอย่างไม่สามารถกู้คืนได้: บิตคอยน์ของคุณจะถือว่าเผาไปแล้ว ดังนั้นคุณควรป้อน `0` เสมอพร้อมกับ OP_RETURN เพื่อบันทึกข้อมูลที่ต้องการ แต่ไม่ควรเชื่อมโยงเงินทุนกับมัน ซึ่งจะสูญหายไป



เคล็ดลับ: ใช้เครื่องมือ [OP_RETURN Generator] (https://resources.davidcoen.it/opreturnelectrum/) เพื่อ generate สคริปต์โดยอัตโนมัติ แม้ว่าเครื่องมือนี้จะแนะนำให้ป้อนจำนวนใน BTC ให้คงหน่วยที่กำหนดค่าใน Electrum ไว้

## เคล็ดลับและแนวทางปฏิบัติที่ดีที่สุด





- ทดสอบด้วยจำนวนเล็กน้อย: เริ่มต้นด้วยการทำธุรกรรมขนาดเล็ก (เช่น Sats 1000) เพื่อหลีกเลี่ยงข้อผิดพลาดที่มีค่าใช้จ่ายสูง
- ตรวจสอบให้แน่ใจว่าผลลัพธ์ที่มี OP_RETURN เท่ากับศูนย์ มิฉะนั้นบิตคอยน์ของคุณจะสูญหายอย่างถาวร
- ตรวจสอบหน่วย: ตรวจสอบให้แน่ใจว่าจำนวนที่ป้อนตรงกับหน่วยที่แสดงใน Electrum (BTC, mBTC หรือ Sats)
- ค่าธรรมเนียมการทำธุรกรรม: หากเครือข่ายมีความแออัด ให้เพิ่มค่าธรรมเนียมเพื่อการยืนยันที่รวดเร็วขึ้น
- ข้อความสั้น: รายการ OP_RETURN จำกัดไว้ที่ 80 ไบต์ วางแผนข้อความของคุณให้เหมาะสม
