## 概念入门（15 分钟）


### 什么是自我监护，为什么它很重要？


Bitcoin 的诞生是为了让我们的货币系统不再需要银行和公司等可信赖的第三方。比特币不需要信任，而是利用数学、物理学和密码学，让任何人都有能力拥有和控制自己的钱，而不需要任何人的许可。


其工作原理是，比特币存在于一个名为区块链（又称比特币时间链）的全球数字账本上，这是一个由计算机运行的公开透明账本，而不是像银行账户那样的中央账本。


重要的一点是，要把比特币从一个地方转移到另一个地方，你必须用所谓的私钥来签署交易。这就好比用密码打开一个保险库，然后把比特币转移到别人的保险库里。Bitcoin 让你有能力自己掌握金库的钥匙，而不是依赖银行为你转移资金。


权力越大，责任越大，钥匙丢了，资金就会永远消失。因此，你可以把金库的钥匙想象成钱本身。虽然钥匙和比特币不是一回事，但它们是转移资金的机制，因此对它们的保护极为重要。这就是为什么我们说 "不是你的钥匙，也不是你的币"。


自我保管一词听起来可能会让人困惑，但它的意思就是持有自己的私钥，控制自己的比特币。如果你不持有私钥，你就是在委托别人替你保管。如果你的比特币在 ETF 或交易所（Mt. Gox、FTX、Coinbase、Binance 等），你就不是拥有比特币，而是拥有对比特币的所有权。这就带来了各种风险，比如交易所被黑客攻击，丢失你的比特币，或者公司把你的钱借出去，只给你一小部分储备。此外，受信任的第三方可以完全控制你的钱，并限制或冻结取款。


![image](assets/fr/01.webp)


有了自我托管，您就不再需要信任。没有人可以冻结你的资金或拒绝交易，你可以跨越国界，随时随地向任何人汇款，你不需要银行账户、身份证或任何人的批准。没有人可以阻止你、审查你或窃取你的资金，这就释放了比特币作为自由货币的全部力量。这就是为什么我们说，有了比特币，你就可以成为自己的银行。


Bitcoin 的诞生是为了解决操纵信任和金钱的问题，是我们现行制度的一种选择，但只有当你拿走钥匙时，出口才会起作用。这就是自我监护如此重要的原因。


### 什么是 Wallet？


wallet 这个词有点名不副实，因此可能会引起混淆。没错，比特币 wallet 和实物 wallet 一样，都可以存储价值。但主要区别在于，比特币钱包实际上并不存储任何比特币。


Bitcoin 只作为公共区块链上的分类账条目存在，或存在于网络空间的隐喻金库中。记住，要移动比特币，你必须用你的钥匙打开金库的锁，然后把币移动到其他地方，私钥就是用来消费比特币的。当你用你的 wallet 进行交易时，你实际上只是用你的密钥来签署交易。这就是你证明你拥有这笔钱并有权使用这些币的方式。


Bitcoin 钱包实际上只是存储你的私钥，所以称其为钥匙链更准确。


### Hot 与 Cold 钱包


热wallet是手机或电脑上的一个软件应用程序。它连接到互联网，因此使用更方便，签署交易更快捷，但这也意味着它更容易受到黑客、恶意软件和网络钓鱼的攻击。之所以称其为 "热"，是因为它连接了互联网，插上了插头并接通了电源。例如手机 wallet 或浏览器 wallet。


另一方面，冷 wallet 或硬件 wallet 是一种离线创建和存储密钥的设备。这就消除了他人入侵您的资金的可能性，对于长期储蓄来说要安全得多，但这是一种需要在每笔交易中签名的设备，可能不太方便。


### Hardware Wallet 威胁模型


硬件钱包的存在是为了解决一个基本问题：如何签署 Bitcoin 交易，而又不将私钥暴露在可能被恶意软件或远程攻击者入侵的联网计算机上？核心威胁模型假定您日常使用的笔记本电脑或手机具有潜在的敌意。硬件 wallet 创造了一个隔离环境，私钥永远不会离开设备，交易签名在 secure element 或微控制器中进行，只将签名传回主机，而不是私钥本身。即使您的电脑完全被入侵，攻击者也无法在无法实际接触设备和您的 PIN 码的情况下窃取您的 Bitcoin。


不过，硬件钱包也有自己的威胁。你必须相信制造商没有引入后门，供应链没有被篡改，随机数生成是真正随机的。物理攻击者可能会通过侧信道攻击或芯片操纵来提取密钥，而拥有临时访问权限的人可能会修改你的设备。构建自己的硬件 wallet 可以帮助你理解这些权衡--你将就安全元件与通用微控制器、如何验证显示屏上的交易以及如何防范远程和物理威胁做出决定。我们的目标并不是实现完美的安全，而是了解您正在防范哪些威胁，以及哪些威胁仍然存在。


### 关键概念



- 熵和 seed 短语：** wallet 的安全性取决于它的随机性。我们将把设备的随机数发生器与掷骰子等对人类友好的技巧混合在一起，把熵转换成 12 或 24 个字的[BIP39 短语](https://github.com/bitcoin/bips/blob/master/bip-0039.mediawiki)，然后带着您信任的书面或金属备份离开房间。
- 种子短语的卫生：** 把 seed 当作你储蓄的万能钥匙。切勿在手机或电脑上输入这些词语--键盘记录程序、屏幕截图和云备份都可能永远泄露它。将短语离线保存，存放在只有你能访问的地方，并在离开前练习大声回读。
- 安全元件 + 微控制器：** 将 secure element 视为金库，将微控制器视为大脑。secure element 用于保护具有防篡改功能的私人密钥，而微控制器则负责处理屏幕、按钮和固件逻辑。请注意，我们今天构建的硬件钱包没有 secure element。这并不意味着它不安全，只是少了一级保护。
- 信任固件：** 固件是 wallet 的隐形操作系统。请务必从已标记的版本中下载，检查已发布的哈希值，并了解可重现构建可让多人编译相同的代码并获得完全相同的二进制文件。如果校验和不匹配，则不能签名。

## 简单方法还是 Hard 方法


硬件 wallet 固件闪存主要有两种方法：简单方法和困难方法。简单方法是利用预配置工具或基于网络的闪存程序，只需少量输入就能自动将固件加载到设备上。这种方法非常适合希望速战速决或希望避免调试和命令行交互等复杂问题的初学者。它简化了过程，让您更快地启动和运行，使嵌入式开发或硬件钱包新手也能使用。


另一方面，困难的方法是使用命令行工具手动闪存固件。这种方法需要验证固件签名和校验和，以确保真实性和完整性，让您更深入地了解闪存过程以及固件与硬件的交互方式。虽然需要付出更多努力并熟悉终端命令，但它提供了更强的控制力、透明度和对设备安全性的信心。


每种方法都有其利弊：简单的方法为了速度和方便而牺牲了一定程度的信任和理解，而困难的方法需要更多的时间和技术技能，但却能让你获得灵活性和对底层技术更强的掌握。教师应鼓励学生选择最符合自己舒适度和好奇心的方法，培养自信和探索精神。

## 设置 Jade Wallet


固件刷新成功后，您的 LilyGO T-Display 现在就是一个功能齐全的 Jade 硬件 wallet。本节将指导您完成初始设置过程，从生成 seed 短语到使用 wallet 软件（如 Sparrow 或 Blockstream Green 移动应用程序）连接设备。


### 初始启动和设备设置



- 打开设备电源：** LilyGO 仍通过 USB-C 与笔记本电脑连接，Jade 固件会自动启动。您会看到显示屏上出现 Jade 徽标。



- 进入设置模式：** 设备将显示初始菜单。使用电路板上的两个物理按钮进行导航：
 - 左键：** 上移/后退
 - 右键：** 向下/向前移动
 - 同时按下两个按钮：** 选择/确认



- 选择 "设置"：** 导航至设置选项并按下两个按钮确认。设备将引导您完成初始配置过程。


### 创建您的 Wallet



- 选择 "开始设置"：** 设备将提示您开始 wallet 创建过程。确认您的选择。



- 选择 "创建新 Wallet"：** 您将看到两个选项：
 - 创建新的 Wallet：** 生成一个新的 seed 短语（为研讨会选择此短语）。
 - 恢复 Wallet：** 从 seed 短语中恢复现有的 wallet（面向高级用户）



- 选择 "创建新 Wallet "并确认。



- 生成熵：** 设备将使用随机数生成器生成加密熵。这个过程需要几秒钟，因为设备会从多个来源收集随机数。


### 记录您的种子词组



- 写下您的 seed 短语：** 设备现在将显示 12 个字符的 BIP39 seed 短语，每次显示一个字。这是整个过程中最关键的一步。


**重要的安全实践：**


- 将每个单词清楚地写在纸上（如果有 seed 短语卡片，请使用）。
- 在书写时仔细检查每个单词
- 切勿用手机拍摄 seed 短语
- 切勿在任何电脑或手机上输入这些词语
- 保持 seed 短语的私密性--不要共享屏幕或向他人展示



- 验证您的 seed 短语：** 在写下所有 12 个单词后，设备会要求您确认短语中的几个单词，以确保您记录的单词正确无误。使用按钮为每个提示选择正确的单词。


**专业建议：** 在继续之前，先练习大声回读自己的 seed 短语（小声，不要让别人听到）。这有助于发现任何笔迹错误或含糊不清之处。


### 连接方法



- 选择连接类型：** Jade 固件支持两种连接方式：
 - USB：** 通过 USB-C 电缆进行有线连接（建议用于此研讨会）
 - 蓝牙：** 与移动设备无线连接



- 暂时选择 **USB**，因为它是桌面 wallet 软件最直接的选项，而且不会引入无线攻击向量。



- 设备命名：** Jade 会显示一个唯一的标识符，如 "Connect Jade A7D924"。如果您最终构建了多个硬件钱包，该标识符可帮助您区分多个硬件钱包。如果需要，请记下这个标识符。


### 连接到 Wallet 软件


现在，您有两个主要选择来连接新创建的硬件 wallet：Blockstream Green 移动应用程序（用于移动使用）或 Sparrow Wallet（用于具有更高级功能的桌面使用）。在本讲座中，我们将重点介绍 Sparrow Wallet，因为它能更好地了解 Bitcoin 交易的技术细节。


#### 选项 1：Blockstream Green 移动应用程序（快速启动）


如果您想使用移动设备快速测试您的设备：



- 从 App Store（iOS）或 Google Play（Android）下载 **Blockstream Green** 应用程序
- 打开应用程序并选择 "连接 Hardware Wallet
- 从支持的设备列表中选择 "Jade
- 使用 USB-C 至 USB-C 连接线（或用于 iPhone 15+ 的 USB-C 至 Lightning 适配器）将 Jade 插入手机
- 按照屏幕提示连接并创建第一个 wallet


**关于 Liquid 的注意事项：** Blockstream Green 应用程序同时支持 Bitcoin 和 Liquid（Bitcoin 侧链）。如果您正在使用 Liquid 功能，系统可能会提示您 "导出主屏蔽密钥"--这允许应用程序查看 Liquid 网络上的交易金额，否则这些金额将被保密。在本次研讨会中，您可以跳过 Liquid 功能，专注于标准 Bitcoin 交易。


#### 方案 2：Sparrow Wallet（建议用于车间）


Sparrow Wallet 是一款功能强大的桌面应用程序，可对 Bitcoin 交易进行细化控制，并与 Jade 硬件 wallet 无缝连接。


**安装：**



- 从官方网站下载 Sparrow Wallet：[sparrowwallet.com](https://sparrowwallet.com)
- 验证下载签名（详见 Sparrow 文档）
- 安装并启动应用程序


**将您的翡翠连接到 Sparrow:**



- 在 Sparrow 中，转到 ** 文件 → 新 Wallet**
- 给 wallet 起个名字（例如 "我的 Jade Wallet"）。
- 点击 **连接 Hardware Wallet**
- Sparrow 会自动检测插入的 Jade 设备
- 如果出现提示，请按下 Jade 显示屏上的两个按钮确认连接
- 选择所需的脚本类型：
 - 本地 Segwit (P2WPKH)：** 建议初学者使用，费用最低，与现代钱包的兼容性最强
 - 嵌套式 Segwit (P2SH-P2WPKH)：** 用于兼容旧版服务
 - Taproot (P2TR)：** 最先进，提供最佳隐私保护和最低费用，但需要最先进的 wallet 支持
- 单击**导入密钥存储**完成连接


**配置 Sparrow 的服务器连接：**


在查看余额或广播交易之前，Sparrow 需要连接到 Bitcoin 节点以获取区块链数据。您有几种选择，每种选择都要在便利性、隐私和信任之间做出不同的权衡：



- 公共 Electrum Server（最简单、最不保密）：** 连接到由第三方运营的公共服务器。设置快捷，但服务器可以看到您的 wallet 的地址，并有可能将它们与您的 IP 地址联系起来。适合在测试网上进行测试。
 - 在 Sparrow 中，转到 **工具 → 首选项 → 服务器**
 - 选择 **公共服务器**，然后从列表中选择一个服务器
 - 单击**测试连接**进行验证



- Bitcoin 核心或节点（最私密、最有效）：** 运行您自己的完整 Bitcoin 节点。这是隐私和验证的黄金标准，因为你要自己验证每一笔交易，而不是相信别人的服务器。不过，这需要下载整个区块链（约 600GB）并保持节点同步。
 - 安装和同步 Bitcoin 核心或绳结
 - 在 Sparrow 中，转到 **工具 → 首选项 → 服务器**
 - 选择 **Bitcoin 核心或节点**，然后输入节点的连接详情



- 私有 Electrum Server（良好平衡）：** 托管您自己的 Electrum 服务器（如 Fulcrum 或 Electrs），并连接到您的 Bitcoin 核心或 Knots 节点。无需在与您的节点相同的机器上运行 Sparrow，即可提供完全的隐私保护。
 - 设置指向 Bitcoin 核心或 Knots 节点的 Electrum 服务器
 - 在 Sparrow 中，转到**工具→首选项→服务器**
 - 选择 ** 私有 Electrum**，然后输入服务器的 URL


在本次研讨会中，使用**公共 Electrum Server** 进行测试网络交易是完全没问题的。在拥有真实资金的生产环境中，您需要考虑运行自己的节点或使用受信任的私人服务器，以最大限度地保护隐私。
