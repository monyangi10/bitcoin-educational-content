## 概念入門 (15 分鐘)


### 什麼是自我監護？為什麼這很重要？


Bitcoin 的創建是為了從我們的貨幣系統中消除對銀行和公司等可信第三方的需求。比特幣沒有使用信任，而是使用數學、物理學和密碼學，讓任何人都有能力擁有和控制自己的錢，而不需要任何人的允許。


其運作方式是，比特幣存在於一個稱為區塊鏈又名比特幣時間鏈的全球數位總帳上，這是一個由電腦運行的公開透明總帳，而不是像銀行帳戶那樣的中央總帳。


最重要的一點是，要將比特幣從一個地方轉移到另一個地方，你必須用所謂的私鑰來簽署該交易。想想看，就像是用密碼解開一個保險庫，然後把比特幣移到別人的保險庫。Bitcoin 賦予您自己持有金庫鑰匙的權力，而不是依賴銀行為您移動資金。


有多大的權力就有多大的責任，失去鑰匙，您的資金就會永遠消失。這樣一來，你就可以把金庫的鑰匙看作是錢本身。儘管鑰匙與比特幣並不是一回事，但它們是移動資金的機制，因此保護它們極為重要。這就是為什麼我們說 「不是你的鑰匙，不是你的錢幣」。


自我託管這個詞聽起來可能會讓人摸不著頭腦，但它的意思就是持有您自己的私人鑰匙，並控制您自己的 bitcoin。如果您不持有該密鑰，您就等於信任別人為您持有。如果您的比特幣在 ETF 或交易所（Mt. Gox、FTX、Coinbase、Binance 等），您擁有的不是比特幣，而是對比特幣的權利要求。這會帶來各種風險，比如交易所被黑客入侵，導致您的比特幣丟失，或者公司借出您的錢，只給您一小部分儲備。此外，受信任的第三方可以完全控制你的錢，並可以限制或凍結提款。


![image](assets/fr/01.webp)


有了自我託管，您就不再需要信任。沒有人可以凍結您的資金或拒絕交易，您可以跨越國界，隨時向任何人發送資金，您不需要銀行帳戶、身份證或任何人的批准。沒有人可以阻止你、審查你、或從你身邊盜竊，釋放比特幣作為自由貨幣的全部力量。這就是為什麼我們說，有了比特幣，你可以做自己的銀行。


Bitcoin 的設立是為了解決信託與金錢被操控的問題，這是我們現行體系的一個選擇，但出口只有在您拿走鑰匙時才會起作用。這就是自我保管如此重要的原因。


### 什麼是 Wallet？


wallet 這個名詞有點名不副實，因此可能會引起混淆。是的，比特币wallet确实和实体wallet一样，存储价值。但主要的區別是，比特幣錢包並不實際儲存任何比特幣。


Bitcoin只存在於公共區塊鏈上的分類帳目中，或者在網絡空間的隱喻保險庫中。請記住，要移動比特幣，你必須用你的鑰匙來解鎖金庫，然後把幣移動到別的地方，私人鑰匙就是用來花比特幣的。當你用 wallet 進行交易時，你其實只是用你的鑰匙來簽署交易。這就是你如何證明你擁有這些錢，並有權花這些錢。


Bitcoin 錢包其實只是儲存您的私人金鑰，所以稱之為鑰匙鏈會比較準確。


### Hot 與 Cold 皮夾


Hot wallet 是手機或電腦上的軟體應用程式。它連接到網際網路，因此使用起來更方便，簽署交易也更快速，但這也意味著它更容易受到駭客、惡意軟體和網路釣魚的攻擊。它之所以被稱為「熱」，是因為它連接到網際網路、插上電源並開機。電話 wallet 或瀏覽器 wallet 就是一個例子。


另一方面，冷凍 wallet 或硬體 wallet 是一種可以離線建立並儲存您的金鑰的裝置。這可以避免有人入侵您的資金，對於長期儲蓄而言更為安全，但它是需要簽署每筆交易的裝置，可能不太方便。


### Hardware Wallet 威脅模型


硬體錢包的存在是為了解決一個基本問題：如何簽署 Bitcoin 交易，而又不會讓您的私密金鑰暴露在與網際網路連線的電腦中，讓惡意軟體或遠端攻擊者入侵？核心威脅模型假設您日常使用的筆記型電腦或手機有潛在敵意。硬體 wallet 可創造一個隔離的環境，讓私密金鑰永遠不會離開裝置，而交易簽章會在 secure element 或微控制器中進行，secure element 或微控制器只會將簽章傳回主機電腦，而非金鑰本身。即使您的電腦完全受到攻擊，攻擊者若無法實體存取裝置和您的 PIN 碼，也無法竊取您的 Bitcoin。


不過，硬體錢包也有自己的威脅。您必須相信製造商沒有開啟後門、供應鏈沒有被竄改，以及隨機數生成是真正隨機的。物理攻擊者可能會透過側邊通道攻擊或晶片竄改來提取金鑰，而擁有臨時存取權限的人可能會修改您的裝置。建立您自己的硬體 wallet 可協助您瞭解這些權衡取捨 - 您會就安全元件與一般用途微控制器、如何驗證顯示器上的交易，以及如何防範遠端與實體威脅做出決定。我們的目標不是完美的安全性，而是了解您正在防範哪些威脅，以及哪些威脅仍然存在。


### 關鍵概念



- 熵和 seed 短語：** 您的 wallet 的安全性僅取決於誕生它的隨機性。我們會將裝置的隨機數字產生器與擲骰子等人性化的技巧混合，將這種熵轉換成 12 或 24 個字的 [BIP39 短語](https://github.com/bitcoin/bips/blob/master/bip-0039.mediawiki)，然後帶著您信任的書面或金屬備份離開房間。
- 種子詞組的衛生：** 把 seed 當成您儲蓄的萬能鑰匙。切勿在手機或電腦中輸入這些字句 - 紀錄程式、螢幕截圖和雲端備份都可能會永遠洩漏這些字句。請將短語離線保存，存放在只有您可以存取的地方，並在離開之前練習大聲回讀。
- 安全元件 + 微控制器：** 將 secure element 視為金庫，微控制器則是大腦。secure element 保護具有防篡改功能的私人金鑰，而微控制器則處理螢幕、按鈕和韌體邏輯。請注意，我們今天所建立的硬體錢包並沒有 secure element。這並不表示它不安全，只是少了一層保護。
- 信任韌體：** 韌體是 wallet 的隱形作業系統。請務必從已標籤的版本下載，檢查已公佈的雜湊值，並瞭解可重複的建立可讓多人編譯相同的程式碼，並得出完全相同的二進位檔。如果校驗和不匹配，您就不要簽名。

## 簡易方法或 Hard 方法


更新硬體 wallet 韌體有兩種主要方式：簡易方式和困難方式。簡單的方法是使用預先設定的工具或網頁型快閃記憶體，只需最少的輸入就能自動將韌體載入您的裝置。此方法非常適合想要速戰速決或避免複雜的除錯及指令列互動的初學者。它簡化了流程，讓您更快上手，讓嵌入式開發或硬件錢包的新手也能輕鬆上手。


另一方面，困難的方法是使用命令列工具手動更新韌體。這種方法需要驗證韌體簽章和校驗和，以確保真實性和完整性，讓您更深入瞭解快閃程序以及韌體如何與硬體互動。雖然這種方法需要更多的努力和熟悉終端機指令，但它提供了更大的控制力、透明度和對裝置安全性的信心。


每種方法都有其優劣之分：簡易的方法需要犧牲某種程度的信任和理解，以換取速度和便利性，而困難的方法則需要更多的時間和技術技能，但回報是靈活性和對底層技術更強的掌握。導師應該鼓勵學生選擇最符合他們的舒適度和好奇心的方法，培養他們的信心和探索精神。

## 設定 Jade Wallet


一旦韌體成功更新，您的 LilyGO T-Display 即成為功能完整的 Jade 硬體 wallet。本節將引導您完成初始設定程序，從產生您的 seed 詞組到使用 wallet 軟體 (如 Sparrow 或 Blockstream Green 行動應用程式) 連接裝置。


### 初始開機與裝置設定



- 開啟裝置電源：** 當 LilyGO 仍透過 USB-C 連接到您的筆記型電腦時，Jade 韌體應該會自動開機。您會看到 Jade 標誌出現在顯示器上。



- 進入設定模式：** 裝置會為您呈現初始功能表。使用電路板上的兩個實體按鈕進行瀏覽：
 - 左鍵：** 上移/後移
 - 右鍵：** 向下/向前移動
 - 兩個按鈕同時按下：** 選擇/確認



- 選擇「設定」：** 導覽至「設定」選項，然後按下兩個按鈕確認。裝置會引導您完成初始設定程序。


### 建立您的 Wallet



- 選擇「開始設定」：** 裝置會提示您開始 wallet 建立程序。確認您的選擇。



- 選擇「建立新 Wallet」：** 您會看到兩個選項：
 - 建立新的 Wallet：** 產生新的 seed 詞組（選擇此項用於工作坊）
 - 還原 Wallet：** 從 seed 樂句復原現有的 wallet（進階使用者適用）



- 選擇「建立新 Wallet」並確認。



- 產生熵:** 裝置會使用其隨機數生成器來產生加密熵。這個過程需要幾秒鐘，因為裝置會從多個來源收集隨機性。


### 記錄您的種子詞組



- 寫下您的 seed 詞組：** 裝置現在會顯示 12 個字的 BIP39 seed 詞組，每次一個字。這是整個過程中最關鍵的一步。


**重要的安全作法：**


- 將每個單字清楚寫在紙上（若有提供 seed 詞組卡片，請使用）。
- 書寫時仔細檢查每個字
- 切勿使用手機拍攝 seed 詞組
- 切勿在任何電腦或手機上輸入這些字詞
- 保持您的 seed 詞組私密 - 不要分享您的螢幕或顯示給其他人看



- 確認您的 seed 詞組：** 寫下所有 12 個單字後，裝置會要求您確認詞組中的幾個單字，以確保您正確記錄。使用按鈕為每個提示選取正確的單字。


**專業建議：** 在繼續之前，先練習大聲回讀您的 seed 詞組（小聲，以免其他人聽到）。這有助於捕捉任何筆跡錯誤或含糊不清的地方。


### 連接方法



- 選擇連線類型：** Jade 韌體支援兩種連線方式：
 - USB:** 透過 USB-C 連接線進行有線連接 (建議用於本工作坊)
 - 藍牙：** 無線連接至行動裝置



- 暫時選擇 **USB**，因為這是桌上型 wallet 軟體最直接的選項，而且不會引入無線攻擊媒介。



- 裝置命名：** Jade 會顯示一個獨特的識別碼，如「Connect Jade A7D924」。如果您最終建立了多個硬體錢包，這個識別碼可以幫助您區分它們。如果需要，請記下此識別碼。


### 連接至 Wallet 軟體


現在您有兩個主要選項可與新建立的硬體 wallet 連接：Blockstream Green 行動應用程式 (適用於隨身使用) 或 Sparrow Wallet (適用於桌面使用，具備更先進的功能)。在這個研討會中，我們將專注於 Sparrow Wallet，因為它能提供更好的 Bitcoin 交易技術細節的可視性。


#### 選項 1：Blockstream Green 行動應用程式 (快速入門)


如果您想使用行動裝置快速測試您的裝置：



- 從 App Store (iOS) 或 Google Play (Android) 下載 **Blockstream Green** 應用程式
- 開啟應用程式並選擇「連接 Hardware Wallet」。
- 從支援的裝置清單中選擇「Jade
- 使用 USB-C 至 USB-C 連接線 (或適用於 iPhone 15+ 的 USB-C 至 Lightning 轉接器) 將您的 Jade 插入手機。
- 按照螢幕上的提示連接並建立您的第一個 wallet


**關於 Liquid 的注意事項：** Blockstream Green 應用程式同時支援 Bitcoin 和 Liquid（Bitcoin 的側鏈）。如果您使用 Liquid 功能，可能會提示您「匯出主遮蔽密鑰」- 這可讓應用程式看到 Liquid 網路上的交易金額，否則這些交易金額是保密的。在本研討會中，您可以跳過 Liquid 功能，專注於標準的 Bitcoin 交易。


#### 選項 2：Sparrow Wallet（建議用於工作坊）


Sparrow Wallet 是一款功能強大的桌面應用程式，可讓您對 Bitcoin 交易進行細粒度控制，並與您的 Jade 硬體 wallet 進行無縫連接。


**安裝：**



- 從官方網站下載 Sparrow Wallet：[sparrowwallet.com](https://sparrowwallet.com)
- 驗證下載簽章（詳情請參閱 Sparrow 說明文件）
- 安裝並啟動應用程式


**將您的 Jade 連接到 Sparrow:**



- 在 Sparrow 中，移至 ** 檔案 → 新增 Wallet**
- 為您的 wallet 命名（例如，「我的 Jade Wallet」）。
- 按一下 **連接 Hardware Wallet**
- Sparrow 應該會自動偵測您插入的 Jade 裝置
- 如果出現提示，請按下 Jade 顯示器上的兩個按鈕，以確認連線。
- 選擇您所需的腳本類型：
 - 原生 Segwit (P2WPKH)：** 建議初學者使用，費用最低，與現代錢包的相容性最廣。
 - Nested Segwit (P2SH-P2WPKH):** 與舊版服務相容
 - Taproot (P2TR):** 最先進，提供最佳隱私和最低費用，但需要最尖端的 wallet 支援
- 按一下 ** 匯入金鑰儲存庫** 以完成連線


**設定 Sparrow 的伺服器連線：**


在您可以看到您的餘額或廣播交易之前，Sparrow 需要連接到 Bitcoin 節點來取得區塊鏈資料。您有幾種選擇，每種選擇都在便利性、隱私和信任之間有不同的取捨：



- 公共 Electrum Server（最簡單、最不隱私）：** 連接到由第三方操作的公共伺服器。設定快速，但伺服器可以看到您的 wallet 位址，並可能將其連結到您的 IP 位址。適合在 testnet 上進行測試。
 - 在 Sparrow 中，移至 ** 工具 → 偏好設定 → 伺服器***
 - 選擇 **公共伺服器**，然後從清單中選擇伺服器
 - 按一下 ** 測試連線** 以驗證



- Bitcoin 核心或 Knots 節點（最隱私、最有效）：** 運行您自己的完整 Bitcoin 節點。這是隱私和驗證的黃金標準，因為您要自己驗證每筆交易，而不信任其他人的伺服器。但是，這需要下載整個區塊鏈 (~600GB) 並保持您的節點同步。
 - 安裝和同步 Bitcoin 核心或結點
 - 在 Sparrow 中，移至 ** 工具 → 偏好設定 → 伺服器***
 - 選擇 **Bitcoin Core 或 Knots**，然後輸入節點的連線詳細資訊



- 私人 Electrum Server (良好平衡):**託管您自己的 Electrum 伺服器 (如 Fulcrum 或 Electrs)，連接至您的 Bitcoin 核心或 Knots 節點。提供完全隱私，無需在與您節點相同的機器上執行 Sparrow。
 - 設定 Electrum 伺服器，指向您的 Bitcoin 核心或 Knots 節點
 - 在 Sparrow 中，移至 ** 工具 → 偏好設定 → 伺服器**。
 - 選擇 ** 私人 Electrum**，並輸入伺服器的 URL


在這個研討會中，使用 ** 公共 Electrum Server** 進行測試網路交易是完全沒有問題的。在有實際資金的生產環境中，您應該考慮運行自己的節點或使用可信賴的私人伺服器，以獲得最大的隱私。
