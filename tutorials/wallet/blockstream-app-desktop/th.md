## 1. บทนำ



### 1.1 วัตถุประสงค์ของบทเรียน





- บทแนะนำนี้อธิบายวิธีการใช้ **Blockstream App** บนคอมพิวเตอร์เพื่อจัดการ Bitcoin **onchain** wallet ด้วย **Hardware Wallet** เพื่อให้สามารถทำธุรกรรมที่ปลอดภัยซึ่งบันทึกไว้บน Blockchain Bitcoin หลัก
- ครอบคลุมการติดตั้ง การกำหนดค่าเริ่มต้น การเชื่อมต่อ Hardware Wallet และการรับและส่งบิตคอยน์บนเชน
- หมายเหตุ: บทแนะนำอื่นๆ ในภาคผนวกครอบคลุม Liquid, Watch-Only และแอปพลิเคชันมือถือ



### 1.2 กลุ่มเป้าหมาย





- ผู้เริ่มต้น**: ผู้ใช้ที่ต้องการจัดการบิตคอยน์ของตนด้วยซอฟต์แวร์เดสก์ท็อปที่ปลอดภัยและ Hardware Wallet.
- ผู้ใช้ระดับกลาง**: ผู้ที่ต้องการเข้าใจวิธีการใช้ Hardware Wallet สำหรับการทำธุรกรรมบนเชนและตัวเลือกความเป็นส่วนตัวเช่น Tor หรือ SPV.



### 1.3. พื้นหลังเกี่ยวกับกระเป๋าเงินฮาร์ดแวร์





- Hardware Wallet**, **Cold Wallet**: อุปกรณ์ทางกายภาพที่เก็บคีย์ส่วนตัวแบบออฟไลน์ ให้ความปลอดภัยในระดับสูงต่อการโจมตีทางไซเบอร์ ซึ่งแตกต่างจาก **Hot wallets** (กระเป๋าเงินซอฟต์แวร์บนอุปกรณ์ที่เชื่อมต่อ)
- การใช้งานที่แนะนำ**:
    - เหมาะสำหรับการรักษาความปลอดภัยจำนวนมากหรือการออมระยะยาว
    - เหมาะสำหรับผู้ใช้ที่เน้นความปลอดภัยที่ต้องการปกป้องเงินทุนจากความเสี่ยงที่เกี่ยวข้องกับอุปกรณ์ที่เชื่อมต่อ
- ข้อจำกัด**: ต้องใช้ซอฟต์แวร์เช่น Blockstream App เพื่อดูยอดคงเหลือ, ที่อยู่ generate, และกระจายธุรกรรมที่ลงนามโดย Hardware Wallet



## 2. แนะนำแอป Blockstream





- Blockstream App** เป็นแอปพลิเคชันมือถือ (iOS, Android) และเดสก์ท็อปสำหรับจัดการกระเป๋าเงิน Bitcoin และสินทรัพย์บนเครือข่าย Liquid ถูกซื้อกิจการโดย Blockstream ในปี 2016 เดิมเรียกว่า *GreenAddress* ถูกเปลี่ยนชื่อเป็น *Blockstream Green* (2019) และปัจจุบันเรียกว่า *Blockstream app* (2025)
- คุณสมบัติหลัก**:
- ธุรกรรม** บนเชนบน Blockchain Bitcoin.
    - ธุรกรรมบนเครือข่าย **Liquid** (Sidechain สำหรับการแลกเปลี่ยนที่รวดเร็วและเป็นความลับ)
- กระเป๋าเงินแบบ** Watch-only สำหรับการตรวจสอบเงินโดยไม่ต้องเข้าถึงกุญแจ.
    - ตัวเลือกความเป็นส่วนตัว: การเชื่อมต่อผ่าน **Tor**, การเชื่อมต่อกับ **โหนดส่วนตัว** ผ่าน Electrum, หรือการตรวจสอบ **SPV** เพื่อลดการพึ่งพาโหนดของบุคคลที่สาม
    - ฟังก์ชัน **Replace-by-fee (RBF)** เพื่อเร่งความเร็วของธุรกรรมที่ยังไม่ได้รับการยืนยัน
- ความเข้ากันได้**: ผสานรวมกระเป๋าเงินฮาร์ดแวร์เช่น **Blockstream Jade**.
- Interface**: ใช้งานง่ายสำหรับผู้เริ่มต้น พร้อมตัวเลือกขั้นสูงสำหรับผู้เชี่ยวชาญ
- หมายเหตุ**: คู่มือนี้เน้นการใช้งานบนเชนกับ Hardware Wallet บนเวอร์ชันเดสก์ท็อป บทแนะนำอื่น ๆ ที่ให้ไว้ในภาคผนวกครอบคลุมการใช้งานบนแอปพลิเคชันมือถือ สำหรับบนเชน, Liquid, และฟีเจอร์ Watch-Only

- ในเมนูการตั้งค่า ให้คลิกที่ "**เครือข่าย**"

- ทำไม?**: ซ่อนที่อยู่ IP ของคุณและปกป้องความเป็นส่วนตัวของคุณ เหมาะอย่างยิ่งหากคุณไม่ไว้วางใจเครือข่ายของคุณ (เช่น Wi-Fi สาธารณะ)
- ข้อเสีย**: อาจทำให้แอปพลิเคชันช้าลงเนื่องจากการเข้ารหัส
- คำแนะนำ**: เปิดใช้งาน Tor หากความลับเป็นสิ่งสำคัญ แต่ทดสอบความเร็วในการเชื่อมต่อ

#### 3.2.4. การตรวจสอบ SPV





- ในเมนูการตั้งค่า ให้คลิกที่ "**เซิร์ฟเวอร์ที่กำหนดเองและการตรวจสอบความถูกต้อง**"
- ฟังก์ชัน**: ใช้ **Simplified Payment Verification (SPV)** ซึ่งดาวน์โหลดเฉพาะบล็อกเฮดเดอร์และยืนยันธุรกรรมของคุณโดยการพิสูจน์การรวม (Merkle) โดยไม่ต้องจัดเก็บ Blockchain ทั้งหมด
- ทำไม?**: ลดการพึ่งพาโหนดเริ่มต้นของ Blockstream ในขณะที่ยังคงความเบาสำหรับอุปกรณ์
- ข้อเสีย**: มีความปลอดภัยน้อยกว่า full node เนื่องจากต้องพึ่งพาโหนดของบุคคลที่สามสำหรับข้อมูลบางอย่าง
- คำแนะนำ**: เปิดใช้งาน SPV หากคุณไม่สามารถใช้โหนดส่วนตัวได้ แต่ควรเลือกใช้ full node เพื่อความปลอดภัยสูงสุด

## 4. การเชื่อมต่อ Hardware Wallet กับแอป Blockstream



### 4.1. ข้อพิจารณาเบื้องต้น



#### 4.1.1. สำหรับผู้ใช้ Ledger





- Blockstream Green รองรับเฉพาะแอปพลิเคชัน **Bitcoin Legacy** บนอุปกรณ์ Ledger (Nano S, Nano X)
- ขั้นตอนที่ต้องปฏิบัติใน **Ledger Live** ก่อนเชื่อมต่ออุปกรณ์ของคุณ :


1. ไปที่ **"การตั้งค่า"** → **"คุณสมบัติทดลอง"** และเปิดใช้งาน **โหมดนักพัฒนา**.


2. ไปที่ **"My Ledger"** → **"App Catalogue "** จากนั้นติดตั้งแอปพลิเคชัน **Bitcoin Legacy**


3. เปิดแอปพลิเคชัน **Bitcoin Legacy** บน Ledger ของคุณก่อนเปิดตัว Blockstream Green เพื่อสร้างการเชื่อมต่อ




- หมายเหตุ**: ตรวจสอบให้แน่ใจว่า Ledger ของคุณปลดล็อกด้วยรหัส PIN ของคุณและแอปพลิเคชัน Bitcoin Legacy กำลังทำงานเมื่อคุณเชื่อมต่อ



#### 4.1.2 การเริ่มต้นใช้งาน Hardware Wallet





- หาก Hardware Wallet (Ledger, Trezor, หรือ Blockstream Jade) ของคุณไม่เคยถูกใช้งานมาก่อน (ไม่ว่าจะกับ Blockstream Green หรือซอฟต์แวร์อื่นๆ เช่น Ledger Live) คุณจะต้องทำการเริ่มต้นใช้งานก่อน ขั้นตอนนี้เกี่ยวข้องกับการทำในสภาพแวดล้อมที่ปลอดภัย โดยไม่มีการใช้กล้องหรือมีผู้สังเกตการณ์:


1. **seed การสร้างวลี / วลีช่วยจำ** (12, 18 หรือ 24 คำ): จดบันทึกลงบนกระดาษอย่างระมัดระวัง


2. **การตรวจสอบวลี seed**: ทดสอบการนำเข้า Wallet จากคำที่ระบุ เช่น โดยการตรวจสอบคีย์สาธารณะที่ขยายออกไป ดำเนินการก่อนที่จะส่งเงินไปยัง Wallet และรักษาความปลอดภัยวลี seed อย่างถาวร


3. **การรักษาความปลอดภัยวลี seed**: เก็บวลีนี้ไว้ในสื่อทางกายภาพ (กระดาษหรือโลหะ) และในที่ปลอดภัย ห้ามเก็บไว้ในรูปแบบดิจิทัล (ไม่มีการถ่ายภาพหน้าจอ, บนคลาวด์, หรืออีเมล)




- สำคัญ**: วลี seed เป็นวิธีเดียวที่คุณสามารถกู้คืนเงินของคุณได้หากอุปกรณ์สูญหายหรือทำงานผิดปกติ ใครก็ตามที่เข้าถึงได้สามารถขโมยบิตคอยน์ของคุณได้

- การตรวจสอบ**: ตรวจสอบที่อยู่ที่แชร์อย่างรอบคอบเพื่อหลีกเลี่ยงข้อผิดพลาดหรือการโจมตี (เช่น มัลแวร์ที่แก้ไขคลิปบอร์ด)
- เมื่อธุรกรรมถูกกระจายบนเครือข่ายแล้ว จะปรากฏใน wallet ของคุณ รอการยืนยัน 1 ถึง 6 ครั้งเพื่อพิจารณาว่าธุรกรรมนั้นไม่สามารถเปลี่ยนแปลงได้

### 5.2. ส่งบิตคอยน์





- จากหน้าจอหลัก wallet ให้คลิกที่ "**Send**".



![image](assets/fr/29.webp)





- ป้อนรายละเอียด:
    - (1) ตรวจสอบว่าสินทรัพย์ที่เลือกคือ **Bitcoin** (onchain)
    - (2) ป้อน**ที่อยู่ของผู้รับ**โดยการวางหรือสแกนรหัส QR ด้วยเว็บแคมของคุณ
    - (3) ระบุ**จำนวน**ที่จะส่ง (ในรูปแบบ BTC, satoshis, หรือหน่วยอื่น ๆ)




![image](assets/fr/16.webp)





- เลือก **ค่าธรรมเนียมการทำธุรกรรม** (ไม่บังคับ) :
 - เลือกจากตัวเลือกที่แนะนำ (เร็ว, ปานกลาง, ช้า) ตามความเร่งด่วน พร้อมเวลายืนยันโดยประมาณ
 - สำหรับค่าธรรมเนียมที่กำหนดเอง ปรับจำนวนซาโตชิต่อ vbyte ด้วยตนเอง ซึ่งจะแสดงบนหน้าจอหลัก ดูเพิ่มเติมที่ [Mempool.space](https://Mempool.space/)



![image](assets/fr/17.webp)





- การเลือก UTXO ด้วยตนเอง** (ไม่บังคับ): คลิกที่ "**การเลือก Coin ด้วยตนเอง**" เพื่อเลือก UTXO เฉพาะที่จะใช้ในธุรกรรม



![image](assets/fr/18.webp)





- การตรวจสอบเบื้องต้น**: ตรวจสอบที่อยู่ จำนวนเงิน และค่าธรรมเนียมบนหน้าจอสรุป จากนั้นคลิกที่ "**ยืนยันการทำธุรกรรม**" ในความเป็นจริง การทำธุรกรรมจะไม่ถูกปล่อยไปยังเครือข่ายจนกว่าคุณจะได้ลงนามด้วย Hardware Wallet ของคุณ ซึ่งมีเพียงอย่างเดียวที่มีคีย์ลับที่เกี่ยวข้องกับที่อยู่ที่ UTXOs (satoshis) จะถูกหักออกจากบัญชี



![image](assets/fr/19.webp)





- ตรวจสอบขั้นสุดท้ายและลงนาม**: ตรวจสอบให้แน่ใจว่าพารามิเตอร์การทำธุรกรรมทั้งหมดถูกต้อง **บนหน้าจอ Hardware Wallet ของคุณ** จากนั้นลงนามในธุรกรรมโดยใช้มัน ข้อผิดพลาดของที่อยู่อาจส่งผลให้สูญเสียเงินทุนอย่างไม่สามารถย้อนกลับได้





- Broadcast**: เมื่อเซ็นแล้ว Blockstream App จะทำการกระจายธุรกรรมโดยอัตโนมัติบนเครือข่าย Bitcoin



![image](assets/fr/20.webp)





- ติดตาม**:
 - ธุรกรรมจะแสดงในหน้าจอหลัก Wallet เป็น "รอดำเนินการ" จนกว่าจะได้รับการยืนยัน
 - ตราบใดที่ธุรกรรมยังไม่ได้รับการยืนยัน ฟังก์ชัน **Replace-by-fee (RBF)** สามารถใช้เพื่อเร่งการยืนยันโดยการเพิ่มค่าธรรมเนียม (ดูภาคผนวก)



![image](assets/fr/21.webp)

### A2. คำอธิบายของ Replace-by-fee (RBF)





- Definition**: Replace-by-fee (RBF) เป็นคุณสมบัติของเครือข่าย Bitcoin ที่ช่วยให้ผู้ส่งสามารถเร่งการยืนยันของธุรกรรม **onchain** โดยการเพิ่มค่าธรรมเนียม
- ขีดจำกัด**:
    - RBF ไม่พร้อมใช้งานสำหรับธุรกรรม Liquid หรือ Lightning
    - ธุรกรรมเริ่มต้นต้องถูกทำเครื่องหมายว่าเข้ากันได้กับ RBF ซึ่ง Blockstream App ทำโดยอัตโนมัติ
- สำหรับข้อมูลเพิ่มเติม ดู [อภิธานศัพท์ของเรา](https://planb.academy/resources/glossary/rbf-replacebyfee)
