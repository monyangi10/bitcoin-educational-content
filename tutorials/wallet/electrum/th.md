## ทำไมต้อง Electrum?


นี่คือคู่มือรายละเอียดเกี่ยวกับวิธีการใช้ Electrum Bitcoin Wallet พร้อมด้วยวิธีแก้ปัญหากับกับดักและความซับซ้อนทั้งหมด - สิ่งที่ฉันได้พัฒนาขึ้นหลังจากการใช้งานหลายปี และการสอนนักเรียนเกี่ยวกับความปลอดภัย/ความเป็นส่วนตัวของ Bitcoin Electrum ไม่ใช่ตัวเลือกที่ดีที่สุดสำหรับผู้ที่ต้องการให้ทุกอย่างเรียบง่ายที่สุด และชอบที่จะอยู่ในระดับเริ่มต้น แต่เป็นสำหรับผู้ที่เป็น หรือมุ่งหวังที่จะเป็นผู้ใช้ระดับ "พลัง"


สำหรับผู้เริ่มต้นใช้งาน Bitcoin จะเป็นการดีหากมีผู้มีประสบการณ์คอยดูแลเพื่อแนะนำวิธีการใช้งาน หากเรียนรู้การใช้งานด้วยตนเองจะปลอดภัยหากพวกเขาใช้เวลาและทดลองใช้งานในสภาพแวดล้อมการทดสอบโดยใช้ sats จำนวนเล็กน้อยในตอนแรก คู่มือนี้สนับสนุนความพยายามนั้น แต่ก็เป็นข้อมูลอ้างอิงที่ดีสำหรับผู้อื่นด้วยเช่นกัน


**คำเตือน:** คู่มือนี้ค่อนข้างยาว ดังนั้นอย่าพยายามทำทุกขั้นตอนที่อธิบายไว้ในวันเดียว ขอแนะนำให้เก็บคู่มือนี้ไว้ใกล้ตัวและทำความก้าวหน้าอย่างต่อเนื่องตามเวลา


**หมายเหตุ:** คุณยังสามารถรับชมวิดีโอแนะนำ Electrum ฉบับเต็มต่อไปนี้ (โปรดทราบว่าวิดีโอนี้ไม่ได้แทนที่บทแนะนำที่เป็นลายลักษณ์อักษร แต่เป็นการเสริมเพิ่มเติม):

## กำลังดาวน์โหลด Electrum


ในอุดมคติ ควรใช้คอมพิวเตอร์ Bitcoin เฉพาะสำหรับการทำธุรกรรม Bitcoin ของคุณ (คู่มือของฉันสำหรับสิ่งนี้ https://armantheparman.com/mint/) _(มีให้ในส่วนความเป็นส่วนตัวด้วย)_. สามารถฝึกฝนด้วยจำนวนเงินเล็กน้อยบนคอมพิวเตอร์ “สกปรก” เมื่อคุณเริ่มเรียนรู้ (ใครจะรู้ว่ามัลแวร์ที่ซ่อนอยู่มากแค่ไหนที่คอมพิวเตอร์ปกติของคุณสะสมมาตลอดหลายปี – คุณไม่ต้องการเปิดเผยกระเป๋าเงิน Bitcoin ของคุณให้กับพวกมัน)


รับ Electrum จาก https://electrum.org/.


คลิกที่แท็บดาวน์โหลดที่ด้านบน


คลิกที่ลิงก์ดาวน์โหลดที่ตรงกับคอมพิวเตอร์ของคุณ คอมพิวเตอร์ Linux หรือ Mac ใด ๆ สามารถใช้ลิงก์ Python (วงกลมสีแดง) คอมพิวเตอร์ Linux ที่มีชิป Intel หรือ AMD สามารถใช้ Appimage (วงกลมสีเขียว; นี่เหมือนกับไฟล์ปฏิบัติการของ Windows) อุปกรณ์ Raspberry Pi มีไมโครโปรเซสเซอร์ ARM และสามารถใช้ได้เฉพาะเวอร์ชัน Python (วงกลมสีแดง) ไม่ใช่ Appimage แม้ว่า Pi จะรัน Linux วงกลมสีน้ำเงินสำหรับ Windows และวงกลมสีดำสำหรับ Mac

## กำลังตรวจสอบ Electrum


วัตถุประสงค์ของการ "ตรวจสอบ" การดาวน์โหลดคือเพื่อให้แน่ใจว่าไม่มีข้อมูลแม้แต่บิตเดียวที่ถูกดัดแปลง มันป้องกันไม่ให้คุณใช้ซอฟต์แวร์เวอร์ชันที่ถูก "แฮ็ก" ที่เป็นอันตราย คุณสามารถข้ามขั้นตอนนี้ได้หากคุณใช้สำเนาที่ดาวน์โหลดมาเพื่อฝึกฝนเท่านั้น เช่น ไม่ใช้กระเป๋าเงินที่ถือเงินจำนวนมาก จากนั้น เมื่อคุณพร้อมที่จะใช้ Electrum สำหรับเงินทุนจริงของคุณ คุณควรลบสำเนาของคุณและเริ่มต้นใหม่ คราวนี้ตรวจสอบการดาวน์โหลดของคุณ


ในการทำเช่นนี้ เราใช้เครื่องมือการเข้ารหัสคีย์สาธารณะ/คีย์ส่วนตัว – gpg ซึ่งได้เขียนคู่มือเกี่ยวกับที่นี่ (https://armantheparman.com/gpg/) เครื่องมือ gpg มาพร้อมกับระบบปฏิบัติการ Linux ทั้งหมด สำหรับ Mac และ Windows ดูลิงก์ gpg สำหรับคำแนะนำในการดาวน์โหลด


นอกเหนือจากการดาวน์โหลดซอฟต์แวร์ Electrum แล้ว เพื่อความปลอดภัย คุณยังต้องการลายเซ็นดิจิทัลของซอฟต์แวร์ด้วย นี่คือสตริงของข้อความ (จริงๆ แล้วมันคือตัวเลขที่เข้ารหัสโดยใช้ข้อความ) ที่ผู้พัฒนาได้สร้างขึ้นด้วยกุญแจ gpg ส่วนตัวของเขา โดยใช้โปรแกรม gpg เราสามารถ "ทดสอบ" ลายเซ็นกับกุญแจสาธารณะของเขา (สร้างจากกุญแจส่วนตัวของผู้พัฒนา) ซึ่งทุกคนสามารถเข้าถึงได้ เปรียบเทียบกับไฟล์ที่ดาวน์โหลด


กล่าวอีกนัยหนึ่ง ด้วยอินพุตสามอย่าง (ลายเซ็น, กุญแจสาธารณะ, และไฟล์ข้อมูล) เราจะได้รับผลลัพธ์เป็นจริงหรือเท็จเพื่อยืนยันว่าไฟล์นั้นไม่ได้ถูกดัดแปลง


หากต้องการรับลายเซ็น ให้คลิกที่ลิงก์ที่ตรงกับไฟล์ที่คุณดาวน์โหลด (ดูตามลูกศรสี):


![image](assets/2.webp)


การคลิกที่ลิงก์อาจดาวน์โหลดไฟล์ไปยังโฟลเดอร์ดาวน์โหลดของคุณโดยอัตโนมัติ หรืออาจเปิดในเบราว์เซอร์ หากเปิดในเบราว์เซอร์ คุณจำเป็นต้องบันทึกไฟล์ คุณสามารถคลิกขวาและเลือก "บันทึกเป็น" ขึ้นอยู่กับระบบปฏิบัติการหรือเบราว์เซอร์ คุณอาจต้องคลิกขวาบนพื้นที่ว่างสีขาว ไม่ใช่บนข้อความ


ด้านล่างนี้คือหน้าตาของข้อความที่ดาวน์โหลด คุณจะเห็นว่ามีลายเซ็นหลายรายการ – เหล่านี้คือลายเซ็นจากบุคคลต่างๆ คุณสามารถตรวจสอบแต่ละรายการหรือทั้งหมดก็ได้ ฉันจะแสดงวิธีการตรวจสอบเฉพาะของนักพัฒนาให้คุณดู


![image](assets/3.webp)


ถัดไป คุณจำเป็นต้องได้รับกุญแจสาธารณะของ ThomasV – เขาเป็นนักพัฒนาหลัก คุณสามารถรับได้โดยตรงจากเขา บัญชี Keybase ของเขา Github หรือจากคนอื่น จากเซิร์ฟเวอร์กุญแจ หรือจากเว็บไซต์ Electrum


การรับข้อมูลจากเว็บไซต์ Electrum เป็นวิธีที่ปลอดภัยน้อยที่สุดจริง ๆ เพราะถ้าเว็บไซต์นี้มีเจตนาร้าย (ซึ่งเป็นสิ่งที่เรากำลังตรวจสอบ) ทำไมเราถึงรับกุญแจสาธารณะจากมัน (กุญแจสาธารณะอาจเป็นของปลอมได้)?


เพื่อให้มันง่ายในตอนนี้ ฉันจะแสดงวิธีการรับมันจากเว็บไซต์ให้คุณดู แต่โปรดจำไว้ในใจ นี่คือสำเนา (https://github.com/spesmilo/electrum/blob/master/pubkeys/ThomasV.asc ) บน GitHub ที่คุณสามารถเปรียบเทียบได้


เลื่อนลงมาที่หน้าหน่อยเพื่อหาลิงก์ไปยังคีย์สาธารณะของ ThomasV (วงกลมสีแดงด้านล่าง) คลิกเพื่อดาวน์โหลด หรือถ้าเปิดเป็นข้อความในเบราว์เซอร์ ให้คลิกขวาเพื่อบันทึก


![image](assets/4.webp)


ตอนนี้คุณมีไฟล์ใหม่ 3 ไฟล์ ซึ่งอาจจะอยู่ในโฟลเดอร์ดาวน์โหลด ไม่สำคัญว่าไฟล์จะอยู่ที่ไหน แต่จะทำให้กระบวนการง่ายขึ้นถ้าคุณนำไฟล์ทั้งหมดมาไว้ในโฟลเดอร์เดียวกัน


ไฟล์ทั้งสาม:


1. การดาวน์โหลด Electrum

2. ไฟล์ลายเซ็น (โดยปกติจะเป็นชื่อไฟล์เดียวกันกับการดาวน์โหลด Electrum โดยเพิ่ม “.asc” เข้าไป

3. กุญแจสาธารณะของ ThomasV


เปิดเทอร์มินัลใน Mac หรือ Linux หรือพรอมต์คำสั่ง (CMD) ใน Windows.


ไปที่ไดเรกทอรี Downloads (หรือที่ใดก็ตามที่คุณเก็บไฟล์ทั้งสามไว้) หากคุณไม่รู้ว่านี่หมายถึงอะไร เรียนรู้จากวิดีโอสั้นนี้สำหรับ Linux/Mac (https://www.youtube.com/watch?v=AO0jzD1hpXc) และวิดีโอนี้สำหรับ Windows (https://www.youtube.com/watch?v=9zMWXD-xoxc) จำไว้ว่าบนคอมพิวเตอร์ Linux ชื่อไดเรกทอรีจะคำนึงถึงตัวพิมพ์เล็กและใหญ่ด้วย


ในเทอร์มินัล ให้พิมพ์สิ่งนี้เพื่อที่จะนำเข้าคีย์สาธารณะของ ThomasV ไปยัง "keyring" ของคอมพิวเตอร์ของคุณ (keyring เป็นแนวคิดเชิงนามธรรม – จริงๆ แล้วมันเป็นเพียงไฟล์บนคอมพิวเตอร์ของคุณ):


```bash
gpg --import ThomasV.asc
```


ตรวจสอบให้แน่ใจว่าชื่อไฟล์ตรงกับที่คุณได้ดาวน์โหลดมา นอกจากนี้ สังเกตว่ามันเป็นขีดคู่ไม่ใช่ขีดเดี่ยว และมีช่องว่างก่อนและหลัง "–import" จากนั้นกด <enter>


ไฟล์ควรนำเข้าได้ หากคุณได้รับข้อผิดพลาด ให้ตรวจสอบว่าคุณอยู่ในไดเรกทอรีที่ไฟล์นั้นมีอยู่จริงหรือไม่ เพื่อตรวจสอบว่าคุณอยู่ในไดเรกทอรีใด (บน Mac หรือ Linux) ให้พิมพ์ pwd เพื่อดูว่าไฟล์ใดอยู่ในไดเรกทอรีที่คุณอยู่ (บน Mac หรือ Linux) ให้พิมพ์ ls คุณควรเห็นไฟล์ข้อความ “ThomasV.asc” แสดงอยู่ อาจจะอยู่ท่ามกลางไฟล์อื่นๆ


จากนั้นเรารันคำสั่งเพื่อตรวจสอบลายเซ็น


```bash
gpg –verify Electrum-4.1.5.tar.gz.asc Electrum-4.1.5.tar.gz
```


โปรดทราบว่ามี "องค์ประกอบ" 4 อย่างที่นี่ แต่ละอย่างแยกจากกันด้วยช่องว่าง ฉันได้ทำตัวหนาข้อความสลับกันเพื่อช่วยให้คุณเห็น องค์ประกอบทั้งสี่คือ:


1. โปรแกรม gpg

2. ตัวเลือก –verify

3. ชื่อไฟล์ของลายเซ็น

4. ชื่อไฟล์ของโปรแกรม


ที่น่าสนใจ บางครั้งคุณสามารถละองค์ประกอบที่ 4 และคอมพิวเตอร์จะเดาว่าคุณหมายถึงอะไร ฉันไม่แน่ใจ แต่ฉันเชื่อว่ามันใช้ได้เฉพาะเมื่อชื่อไฟล์แตกต่างกันเฉพาะที่ "asc" ท้ายสุดเท่านั้น


อย่าคัดลอกชื่อไฟล์ที่ฉันแสดงไว้ที่นี่เท่านั้น – ตรวจสอบให้แน่ใจว่าตรงกับชื่อไฟล์ที่คุณมีในระบบของคุณ


กด <enter> เพื่อรันคำสั่ง คุณควรเห็นข้อความ “good signature from ThomasV” เพื่อบ่งบอกถึงความสำเร็จ จะมีข้อผิดพลาดบางอย่างเพราะเราไม่มี public keys สำหรับลายเซ็นของคนอื่น ๆ ที่อยู่ในไฟล์ลายเซ็น (ระบบการรวมลายเซ็นในไฟล์เดียวนี้อาจเปลี่ยนแปลงในเวอร์ชันถัดไป) นอกจากนี้ยังมีคำเตือนที่ด้านล่างที่เราสามารถละเลยได้ (นี่คือการแจ้งเตือนว่าเรายังไม่ได้บอกคอมพิวเตอร์อย่างชัดเจนว่าเราเชื่อถือ public key ของ ThomasV)


ตอนนี้เรามีสำเนาที่ตรวจสอบแล้วของ Electrum ซึ่งปลอดภัยในการใช้งาน

### รัน Electrum หากใช้ Python


หากคุณดาวน์โหลดเวอร์ชัน Python นี่คือวิธีทำให้มันทำงาน คุณจะเห็นสิ่งนี้ในหน้าดาวน์โหลด:


![image](assets/5.webp)


สำหรับ Linux ควรอัปเดตระบบของคุณก่อน:


```bash
sudo apt-get update
sudo apt-get upgrade
```


คัดลอกข้อความที่ไฮไลต์สีเหลือง วางลงในเทอร์มินัล แล้วกด <enter> คุณจะถูกขอรหัสผ่าน อาจจะมีการยืนยันเพื่อดำเนินการต่อ จากนั้นมันจะติดตั้งไฟล์เหล่านั้น (“dependencies”)


คุณจะต้องแยกไฟล์ที่ถูกบีบอัดไปยังไดเรกทอรีที่คุณเลือก คุณสามารถทำได้ผ่านทางอินเทอร์เฟซผู้ใช้แบบกราฟิก หรือจากบรรทัดคำสั่ง (คำสั่งที่ไฮไลต์ด้วยสีชมพู) - จำไว้ว่าชื่อไฟล์ของคุณอาจแตกต่างออกไป (โปรดทราบว่าเมื่อเราตรวจสอบการดาวน์โหลดในส่วนก่อนหน้า เราได้ตรวจสอบไฟล์ zip ไม่ใช่ไดเรกทอรีที่ถูกแยกออกมา)


มีตัวเลือกในการ "ติดตั้ง" โดยใช้โปรแกรม PIP แต่ไม่จำเป็น และเพิ่มขั้นตอนและการติดตั้งไฟล์เพิ่มเติม เพียงแค่รันโปรแกรมโดยใช้เทอร์มินัลเพื่อข้ามสิ่งเหล่านั้นทั้งหมด


ขั้นตอน (Linux) คือ:


1. นำทางไปยังไดเรกทอรีที่ไฟล์ถูกแยกออก

2. พิมพ์: ./run_electrum


บน Mac ขั้นตอนจะเหมือนกัน แต่คุณอาจต้องติดตั้ง Python3 ก่อน และใช้คำสั่งนี้เพื่อรัน:


```bash
python3 ./run_electrum
```


เมื่อ Electrum กำลังทำงานอยู่ หน้าต่างเทอร์มินัลจะยังคงเปิดอยู่ หากคุณปิดมัน โปรแกรม Electrum จะหยุดทำงาน เพียงแค่ย่อหน้าต่างขณะใช้งาน Electrum


### เรียกใช้ Electrum ด้วย Appimage


นี่ง่ายกว่าเล็กน้อย แต่ไม่ง่ายเท่าไฟล์ปฏิบัติการของ Windows ขึ้นอยู่กับเวอร์ชันของ Linux ที่คุณใช้งาน โดยค่าเริ่มต้น ไฟล์ Appimage อาจมีการตั้งค่าคุณสมบัติที่ทำให้ระบบไม่อนุญาตให้ดำเนินการ เราต้องเปลี่ยนสิ่งนี้ หาก Appimages ของคุณทำงานได้ คุณสามารถข้ามขั้นตอนนี้ได้ นำทางไปยังตำแหน่งที่ไฟล์อยู่ โดยใช้ terminal จากนั้นรันคำสั่งนี้:


```bash
sudo chmod ug+x Electrum-4.1.5-x86_64.AppImage
```


"sudo" ให้สิทธิ์ผู้ใช้ระดับสูงสุด; "chmod" เป็นคำสั่งเพื่อเปลี่ยนโหมด, เปลี่ยนแปลงว่าใครสามารถอ่าน, เขียน, หรือดำเนินการ; "ug+x" หมายถึงเรากำลังแก้ไขผู้ใช้และกลุ่มเพื่ออนุญาตให้ดำเนินการได้


ปรับชื่อไฟล์ให้ตรงกับเวอร์ชันของคุณ


จากนั้น Electrum จะทำงานโดยการดับเบิลคลิกที่ไอคอน Appimage


### กำลังรัน Electrum ด้วย Mac


เพียงดับเบิลคลิกที่ไฟล์ที่ดาวน์โหลด (มันคือ “ไดรฟ์”) หน้าต่างจะเปิดขึ้น ลากไอคอน Electrum ในหน้าต่างไปยังเดสก์ท็อปของคุณ หรือที่ใดก็ตามที่คุณต้องการเก็บโปรแกรม จากนั้นคุณสามารถ “นำออก” ไดรฟ์ และลบไดรฟ์ (ไฟล์ที่ดาวน์โหลด) ได้


ในการรันโปรแกรม เพียงดับเบิลคลิก คุณอาจพบข้อผิดพลาดแบบ "nanny" ที่เฉพาะเจาะจงกับ Mac ซึ่งจำเป็นต้องข้ามผ่านไป


### กำลังรัน Electrum ด้วย Windows


แม้ว่าฉันจะเกลียด Windows มากที่สุด แต่นี่เป็นวิธีที่ง่ายที่สุด เพียงดับเบิลคลิกที่ไฟล์ปฏิบัติการเพื่อรัน


## เริ่มต้นด้วย wallet จำลอง


เมื่อคุณโหลด Electrum ครั้งแรก หน้าต่างจะเปิดขึ้นดังนี้:


![image](assets/6.webp)


เราจะเลือกเซิร์ฟเวอร์ของคุณด้วยตนเองในภายหลัง แต่ตอนนี้ให้ปล่อยค่าเริ่มต้นและเชื่อมต่ออัตโนมัติ


ถัดไป, สร้าง wallet จำลอง – ห้ามใส่เงินใน wallet นี้เด็ดขาด วัตถุประสงค์ของ wallet จำลองนี้คือเพื่อดำเนินการผ่านซอฟต์แวร์และตรวจสอบให้แน่ใจว่าทุกอย่างทำงานได้ดี ก่อนที่คุณจะโหลด wallet จริงของคุณ เรากำลังพยายามหลีกเลี่ยงการละเมิดความเป็นส่วนตัวโดยไม่ได้ตั้งใจด้วย wallet จริง หากคุณเพียงแค่ฝึกฝน wallet ที่คุณสร้างสามารถถือว่าเป็น wallet จำลองได้อยู่แล้ว


คุณสามารถปล่อยชื่อเป็น “default_wallet” หรือเปลี่ยนเป็นชื่อที่คุณต้องการ และคลิกถัดไป ต่อมา หากคุณมีหลายกระเป๋าเงิน คุณสามารถค้นหาและเปิดมันได้ในขั้นตอนนี้โดยคลิกที่ “เลือก…”


![image](assets/7.webp)


เลือก "Standard wallet" และ <Next>:


![image](assets/8.webp)


จากนั้น เลือก “ฉันมี seed อยู่แล้ว” ฉันไม่ต้องการให้คุณเริ่มสร้าง Electrum seed เนื่องจากมันใช้โปรโตคอลของตัวเองที่ไม่เข้ากันกับกระเป๋าเงินอื่น ๆ – นี่คือเหตุผลที่เราไม่คลิก “สร้าง seed ใหม่”


![image](assets/9.webp)


ไปที่ https://iancoleman.io/bip39/ และสร้าง seed จำลอง ก่อนอื่นให้เปลี่ยนจำนวนคำเป็น 12 (ซึ่งเป็นวิธีปฏิบัติทั่วไป) จากนั้นคลิก “generate” และคัดลอกคำในกล่องไปยังคลิปบอร์ดของคุณ


![image](assets/10.webp)


จากนั้นวางคำลงใน Electrum นี่คือตัวอย่าง:


![image](assets/11.webp)


Electrum จะค้นหาคำที่ตรงกับโปรโตคอลของตัวเอง เราต้องหลีกเลี่ยงสิ่งนั้น คลิกที่ตัวเลือก และเลือก BIP39 Seed:


![image](assets/12.webp)


seed จึงกลายเป็นที่ถูกต้อง (ก่อนที่จะทำเช่นนี้ Electrum คาดหวัง Electrum seed ดังนั้น seed นี้จึงถูกมองว่าไม่ถูกต้อง) ก่อนที่คุณจะคลิกถัดไป สังเกตข้อความที่ระบุว่า “Checksum OK” สิ่งนี้สำคัญ (สำหรับ wallet จริงที่คุณอาจใช้ในภายหลัง) ที่คุณเห็นสิ่งนี้ก่อนดำเนินการต่อ เนื่องจากยืนยันความถูกต้องของ seed ที่คุณใส่เข้าไป คำเตือนใกล้ด้านล่างสามารถละเลยได้ มันเป็นการบ่นของนักพัฒนา Electrum เกี่ยวกับ BIP39 และข้อกล่าวหา “FUD’ey” ของพวกเขาว่ารุ่นของพวกเขา (ซึ่งไม่เข้ากันกับกระเป๋าเงินอื่น ๆ) ดีกว่า


**การเบี่ยงเบนเล็กน้อยเพื่อคำเตือนที่สำคัญ:** วัตถุประสงค์ของการตรวจสอบเช็คซัมคือเพื่อให้แน่ใจว่าคุณได้ป้อน seed ของคุณโดยไม่มีข้อผิดพลาดในการพิมพ์ เช็คซัมเป็นส่วนสุดท้ายของ seed (คำที่ 12 จะเป็นคำเช็คซัม) ซึ่งถูกกำหนดทางคณิตศาสตร์โดยส่วนแรกของ seed (11 คำ) หากคุณพิมพ์บางอย่างผิดในตอนเริ่มต้น คำเช็คซัมจะไม่ตรงกันทางคณิตศาสตร์ และซอฟต์แวร์ wallet จะแจ้งเตือนคุณด้วยคำเตือน นี่ไม่ได้หมายความว่า seed ไม่สามารถใช้สร้าง Bitcoin Wallet ที่ใช้งานได้ ลองนึกภาพการสร้าง wallet ด้วยข้อผิดพลาดในการพิมพ์ โหลด wallet ด้วยบิทคอยน์ แล้ววันหนึ่งคุณอาจต้องการกู้คืน wallet แต่เมื่อคุณทำ คุณไม่ได้สร้างข้อผิดพลาดในการพิมพ์ซ้ำ – คุณจะกู้คืน wallet ที่ผิด! มันค่อนข้างอันตรายที่ Electrum จะให้คุณดำเนินการสร้าง wallet ได้หากเช็คซัมของคุณไม่ถูกต้อง ดังนั้นโปรดระวัง มันเป็นความรับผิดชอบของคุณที่จะต้องแน่ใจ กระเป๋าเงินอื่น ๆ จะไม่ให้คุณดำเนินการต่อ ซึ่งปลอดภัยกว่ามาก นี่คือหนึ่งในสิ่งที่ฉันหมายถึงเมื่อฉันบอกว่า Electrum ใช้ได้ดีเมื่อคุณเรียนรู้ที่จะใช้มันอย่างถูกต้อง (นักพัฒนา Electrum ควรแก้ไขสิ่งนี้)


โปรดทราบว่าหากคุณต้องการเพิ่ม passphrase โอกาสในการเลือกนั้นอยู่ในหน้าต่างตัวเลือกนี้ ที่ด้านบนสุด


หลังจากคลิกตกลง คุณจะถูกนำกลับไปยังที่ที่คุณพิมพ์วลี seed หากคุณเลือกตัวเลือก passphrase คุณจะไม่ต้องป้อนมันพร้อมกับคำ seed (คำแนะนำสำหรับสิ่งนั้นจะมาถัดไป)


หากคุณไม่ได้ขอ passphrase คุณจะเห็นหน้าจอนี้ถัดไป – ตัวเลือกเพิ่มเติมสำหรับประเภทสคริปต์ wallet และเส้นทางการอนุพันธ์ซึ่งคุณสามารถเรียนรู้เพิ่มเติมได้ที่นี่ (https://armantheparman.com/public-and-private-keys/) แต่เพียงแค่ปล่อยค่าเริ่มต้นและดำเนินการต่อไป


![image](assets/13.webp)


**สำหรับข้อมูลเพิ่มเติม** ตัวเลือกแรกช่วยให้คุณเลือกได้ระหว่าง:



- legacy (ที่อยู่ที่ขึ้นต้นด้วย “1”),
- pay-to-script-hash (addresses starting with “3”),
- bech32/native segwit (ที่อยู่ที่ขึ้นต้นด้วย “bc1q”)


ณ เวลาที่เขียน Electrum ยังไม่รองรับ taproot (ที่อยู่ที่เริ่มต้นด้วย "bc1p") ตัวเลือกที่สองในหน้าต่างนี้ช่วยให้คุณปรับเปลี่ยนเส้นทางการสืบทอดได้ ฉันแนะนำว่าอย่าแก้ไขสิ่งนี้ โดยเฉพาะอย่างยิ่งก่อนที่จะเข้าใจว่ามันหมายถึงอะไร ผู้คนจะเน้นย้ำถึงความสำคัญของการจดบันทึกเส้นทางการสืบทอดเพื่อให้คุณสามารถกู้คืน wallet ของคุณได้หากจำเป็น แต่ถ้าคุณปล่อยให้เป็นค่าเริ่มต้น คุณก็น่าจะโอเค ดังนั้นอย่าตื่นตระหนก - แต่มันยังคงเป็นการปฏิบัติที่ดีในการจดบันทึกเส้นทางการสืบทอดไว้


ถัดไป คุณจะได้รับตัวเลือกในการเพิ่มรหัสผ่าน อย่าสับสนกับ "PASSPHRASE" รหัสผ่านจะล็อกไฟล์บนคอมพิวเตอร์ของคุณ passphrase เป็นส่วนหนึ่งของการสร้างคีย์ส่วนตัว เนื่องจากนี่เป็น wallet จำลอง คุณสามารถปล่อยให้รหัสผ่านว่างไว้และดำเนินการต่อได้


![image](assets/14.webp)


คุณจะได้รับป๊อปอัพเกี่ยวกับการแจ้งเตือนเวอร์ชันใหม่ (ฉันแนะนำให้คุณเลือกไม่) จากนั้น wallet จะ generate ตัวเองและพร้อมใช้งาน (แต่จำไว้ว่าตัว wallet นี้ถูกกำหนดให้ลบ มันเป็นเพียง wallet จำลอง)


![image](assets/15.webp)


มีบางสิ่งที่ฉันแนะนำให้คุณทำเพื่อการตั้งค่าสภาพแวดล้อมซอฟต์แวร์ (จำเป็นต้องทำเพียงครั้งเดียว):


### เปลี่ยนหน่วยเป็น BTC


ไปที่เมนูด้านบน, เครื่องมือ –> การตั้งค่า electrum, และที่นั่นภายใต้แท็บทั่วไป, คุณจะพบตัวเลือกในการเปลี่ยน "หน่วยฐาน" เป็น BTC.

เปิดใช้งานแท็บที่อยู่และเหรียญ


ไปที่เมนูด้านบน เลือก "แสดงที่อยู่" จากนั้นกลับไปที่มุมมองและเลือก "แสดงเหรียญ"


### เปิดใช้งาน Oneserver


ตามค่าเริ่มต้น Electrum จะเชื่อมต่อกับโหนดแบบสุ่ม นอกจากนี้ยังเชื่อมต่อกับโหนดรองอื่นๆ อีกมากมาย ฉันไม่แน่ใจว่ามีการแลกเปลี่ยนข้อมูลอะไรกับโหนดรองเหล่านั้น แต่เราไม่ต้องการให้มันเกิดขึ้น เพื่อความเป็นส่วนตัว แม้ว่าคุณจะระบุโหนด เช่น โหนดของคุณเอง โหนดอื่นๆ หลายโหนดก็จะถูกเชื่อมต่อด้วย และฉันไม่แน่ใจว่ามีการแชร์ข้อมูลอะไรบ้าง อย่างไรก็ตาม มันง่ายที่จะป้องกัน ก่อนที่ฉันจะแสดงวิธีระบุโหนดของคุณเอง เราจะบังคับให้ Electrum เชื่อมต่อกับเซิร์ฟเวอร์เพียงตัวเดียวในแต่ละครั้ง


**หมายเหตุ:** มีวิธีการทำเช่นนี้โดยการระบุ “oneserver” จากบรรทัดคำสั่ง แต่ฉันไม่แนะนำวิธีนี้ ฉันจะแสดงวิธีทางเลือกที่ฉันคิดว่าง่ายกว่าในระยะยาว และมีโอกาสน้อยที่จะทำให้คุณเชื่อมต่อกับโหนดอื่นโดยไม่ได้ตั้งใจ


เหตุผลที่เราใช้ wallet จำลองคือถ้าเราโหลด wallet จริงของเรา พร้อมกับบิทคอยน์จริงของเรา เราจะเชื่อมต่อกับโหนดแบบสุ่มไปแล้ว (แม้ว่าเราจะเลือก "ตั้งค่าเซิร์ฟเวอร์ด้วยตนเอง" ในตอนเริ่มต้น มันก็ยังเชื่อมต่อกับโหนดรองอื่น ๆ ด้วยเหตุผลบางอย่าง (เฮ้ นักพัฒนา Electrum คุณควรแก้ไขเรื่องนี้) ถ้า wallet ของเราเป็นส่วนตัว นี่จะเป็นหายนะ


เราไม่สามารถทำขั้นตอนที่จะแสดงให้คุณเห็นด้านล่างได้โดยไม่ต้องโหลด wallet บางประเภทก่อน (เรากำลังจะแก้ไขไฟล์การกำหนดค่าที่จะถูกเติมข้อมูลและพร้อมสำหรับการแก้ไขเมื่อโหลด wallet แล้วเท่านั้น)


ตอนนี้ปิด Electrum **(สำคัญ: หากคุณไม่ทำเช่นนี้ การเปลี่ยนแปลงต่อไปที่คุณทำจะถูกลบ)**.


### ไฟล์การกำหนดค่าของ LINUX/MAC


เปิดเทอร์มินัลใน Linux หรือ Mac (คำแนะนำสำหรับ Windows จะตามมาทีหลัง):


คุณควรอยู่ในโฟลเดอร์หลักโดยอัตโนมัติ จากนั้นไปที่โฟลเดอร์ตั้งค่าที่ซ่อนอยู่ของ electrum (ซึ่งแตกต่างจากที่ที่แอปพลิเคชันอยู่)


```bash
cd .electrum
```


สังเกตจุดก่อน “electrum” ซึ่งบ่งบอกว่าเป็นโฟลเดอร์ที่ซ่อนอยู่


อีกวิธีหนึ่งในการไปที่นั่นคือการพิมพ์:


```bash
cd ~/.electrum
```


ที่ “~” แทนที่เส้นทางของไดเรกทอรีบ้านของคุณ คุณสามารถดูเส้นทางเต็มของไดเรกทอรีปัจจุบันของคุณด้วยคำสั่ง “pwd“


เมื่ออยู่ในไดเรกทอรี “.electrum” ให้พิมพ์ “nano config” แล้วกด <enter>.


ตัวแก้ไขข้อความจะเปิดขึ้น (เรียกว่า nano) พร้อมกับไฟล์การกำหนดค่าเปิดอยู่ เมาส์จะใช้งานได้ไม่มากที่นี่ ใช้ปุ่มลูกศรเพื่อไปยังบรรทัดที่ระบุว่า:


```json
"oneserver": false,
```


เปลี่ยน "false" เป็น "true"; และอย่ารบกวนไวยากรณ์ (อย่าลบเครื่องหมายจุลภาคหรือเครื่องหมายอัฒภาค)


กด <ctrl> x เพื่อออก จากนั้นกด “y” เพื่อบันทึก แล้วกด <enter> เพื่อยืนยันการเปลี่ยนแปลงโดยไม่แก้ไขชื่อไฟล์


ตอนนี้รัน Electrum อีกครั้ง จากนั้นคลิกที่วงกลมด้านล่างขวา ซึ่งจะเปิดการตั้งค่าเครือข่าย จากนั้น ใกล้ด้านบนในแท็บภาพรวม คุณจะเห็น “connected to 1 node” – นี่แสดงถึงความสำเร็จ


ถัดลงมาด้านล่าง คุณจะเห็นช่องข้อความและที่อยู่ของเซิร์ฟเวอร์อยู่ในนั้น ขณะนี้คุณเชื่อมต่อกับโหนดแบบสุ่มนั้นอยู่ รายละเอียดเพิ่มเติมเกี่ยวกับการเชื่อมต่อกับโหนดจะอยู่ในส่วนถัดไป


### ไฟล์กำหนดค่าของ Windows


ไฟล์กำหนดค่าของ Windows ค่อนข้างหายากกว่าเล็กน้อย ไดเรกทอรีคือ: `C:/Users/Parman/AppData/Roaming/Electrum`


เห็นได้ชัดว่าคุณต้องเปลี่ยน "Parman" เป็นชื่อผู้ใช้ของคุณเองสำหรับคอมพิวเตอร์


ในโฟลเดอร์นั้นคุณจะพบไฟล์ config เปิดด้วยโปรแกรมแก้ไขข้อความและแก้ไขบรรทัด:


```json
"oneserver": false,
```


เปลี่ยน “false” เป็น “true”; อย่ารบกวนไวยากรณ์ (อย่าลบเครื่องหมายจุลภาคหรือเครื่องหมายอัฒภาค)


จากนั้นบันทึกไฟล์และออกจากโปรแกรม


## เชื่อมต่อ Electrum กับโหนด


ต่อไป เราต้องการเชื่อมต่อ wallet จำลองของเราเข้ากับโหนดที่เราเลือก หากคุณยังไม่พร้อมที่จะรันโหนด คุณสามารถทำอย่างใดอย่างหนึ่งต่อไปนี้:


1. เชื่อมต่อกับโหนดส่วนตัวของเพื่อน (ต้องใช้ Tor)

2. เชื่อมต่อกับโหนดของบริษัทที่เชื่อถือได้

3. เชื่อมต่อกับโหนดแบบสุ่ม (ไม่แนะนำ)


อนึ่ง นี่คือคำแนะนำในการรันโหนดของคุณเอง และนี่คือเหตุผลที่คุณควรทำ (ดูบทแนะนำในส่วนโหนดหรือในคอร์สฟรีของเรา)


### เชื่อมต่อกับโหนดของเพื่อนผ่าน Tor (คู่มือกำลังจะมาเร็ว ๆ นี้)


### เชื่อมต่อกับโหนดของบริษัทที่เชื่อถือได้


เหตุผลเดียวที่จะทำเช่นนี้คือถ้าคุณต้องเข้าถึงบล็อกเชนและคุณไม่มีโหนดของคุณเอง (หรือของเพื่อน) ให้ใช้งาน


มาลองเชื่อมต่อกับโหนดของ Bitaroo กัน – เราได้รับข้อมูลว่าพวกเขาไม่ได้เก็บข้อมูล พวกเขาเป็นการแลกเปลี่ยนแบบ Bitcoin เท่านั้น ดำเนินการโดยผู้ที่หลงใหลใน Bitcoin การเชื่อมต่อกับพวกเขาต้องอาศัยความไว้วางใจเล็กน้อย แต่ก็ดีกว่าการเชื่อมต่อกับโหนดแบบสุ่ม ซึ่งอาจเป็นบริษัทเฝ้าระวังได้


ไปที่การตั้งค่าเครือข่ายโดยคลิกที่วงกลมในส่วนล่างขวาของหน้าต่าง Wallet (สีแดงหมายถึงไม่ได้เชื่อมต่อ, สีเขียวหมายถึงเชื่อมต่อแล้ว, และสีน้ำเงินหมายถึงเชื่อมต่อผ่าน Tor)


![image](assets/16.webp)


เมื่อคุณคลิกที่ไอคอนวงกลม หน้าต่างป๊อปอัปจะปรากฏขึ้น: wallet ของคุณจะแสดงว่า “เชื่อมต่อกับ 1 โหนด” เนื่องจากเราได้บังคับให้เป็นเช่นนั้นก่อนหน้านี้


ยกเลิกการเลือกช่อง “เลือกเซิร์ฟเวอร์โดยอัตโนมัติ” จากนั้นในช่องเซิร์ฟเวอร์ ให้พิมพ์รายละเอียดของ Bitaroo ตามที่แสดง:


![image](assets/17.webp)


ปิดหน้าต่าง และตอนนี้เราควรจะเชื่อมต่อกับโหนดของ Bitaroo แล้ว เพื่อยืนยัน วงกลมควรจะเป็นสีเขียว คลิกอีกครั้งและตรวจสอบว่ารายละเอียดเซิร์ฟเวอร์ไม่ได้เปลี่ยนกลับไปเป็นโหนดแบบสุ่ม


### เชื่อมต่อกับโหนดของคุณเอง


หากคุณมีโหนดของตัวเอง นั่นเยี่ยมมาก หากคุณมีเฉพาะ Bitcoin Core และไม่มี Electrum SERVER ด้วย คุณจะยังไม่สามารถเชื่อมต่อ Electrum WALLET กับโหนดของคุณได้


**โปรดทราบว่า Electrum Server และ Electrum Wallet เป็นสิ่งที่แตกต่างกัน:** เซิร์ฟเวอร์เป็นซอฟต์แวร์ที่จำเป็นสำหรับ Electrum Wallet เพื่อให้สามารถสื่อสารกับบล็อกเชน Bitcoin ได้ – ฉันไม่รู้ว่าทำไมมันถึงถูกออกแบบมาแบบนี้ – อาจจะเพื่อความเร็วแต่ฉันอาจจะผิดก็ได้


หากคุณใช้งานซอฟต์แวร์แพ็คเกจโหนดเช่น MyNode (ซึ่งเป็นตัวที่ฉันแนะนำให้เริ่มต้น), Raspiblitz (แนะนำเมื่อคุณมีความชำนาญมากขึ้น), หรือ Umbrel (ฉันยังไม่แนะนำเพราะพบปัญหามากเกินไป), คุณจะสามารถเชื่อมต่อ wallet ของคุณได้ง่ายๆ โดยการป้อนที่อยู่ IP ของคอมพิวเตอร์ (Raspberry Pi) ที่รันโหนด, ตามด้วยเครื่องหมายจุดคู่ และ 50002, ตามที่แสดงในภาพในส่วนก่อนหน้า (ต่อไปฉันจะแสดงวิธีค้นหาที่อยู่ IP ของโหนดของคุณ)


เปิดการตั้งค่าเครือข่าย (คลิกที่วงกลมสีเขียวหรือสีแดงที่มุมล่างขวา) ยกเลิกการเลือกช่อง “เลือกเซิร์ฟเวอร์โดยอัตโนมัติ” จากนั้นป้อนที่อยู่ IP ของคุณตามที่ฉันได้ทำไว้ ของคุณอาจจะแตกต่างกัน แต่เครื่องหมายทวิภาคและ “50002” ควรจะเหมือนกัน


![image](assets/18.webp)


ปิดหน้าต่าง และตอนนี้เราควรเชื่อมต่อกับโหนดของคุณแล้ว เพื่อยืนยัน ให้คลิกที่วงกลมอีกครั้งและตรวจสอบว่ารายละเอียดเซิร์ฟเวอร์ไม่ได้เปลี่ยนกลับไปเป็นโหนดแบบสุ่ม


บางครั้ง แม้ว่าจะทำทุกอย่างถูกต้องแล้วก็ตาม แต่ดูเหมือนว่ามันจะไม่ยอมเชื่อมต่อ นี่คือสิ่งที่ควรลองทำ...



- อัปเกรดเป็นเวอร์ชันใหม่ของ Electrum และซอฟต์แวร์โหนดของคุณ;
- ลองลบโฟลเดอร์แคชในไดเรกทอรี “.electrum”;
- ลองเปลี่ยนพอร์ตจาก 50002 เป็น 50001 ในการตั้งค่าเครือข่าย;
- ใช้[คู่มือนี้](https://armantheparman.com/tor/) เพื่อเชื่อมต่อโดยใช้ Tor เป็นทางเลือก;
- ติดตั้ง Electrum Server ใหม่บนโหนด


## ค้นหา IP ของโหนดของคุณ Address


ที่อยู่ IP ไม่ใช่สิ่งที่ผู้ใช้ทั่วไปมักจะรู้วิธีค้นหาและใช้งาน ฉันได้ช่วยหลายคนในการรันโหนด และจากนั้นเชื่อมต่อกระเป๋าเงินของพวกเขากับโหนด – อุปสรรคที่มักจะพบคือการหาที่อยู่ IP ของมัน


สำหรับ MyNode คุณสามารถพิมพ์ในหน้าต่างเบราว์เซอร์: `mynode.local`


บางครั้ง “mynode.local” อาจไม่ทำงาน (ตรวจสอบให้แน่ใจว่าคุณไม่ได้พิมพ์ในแถบค้นหาของ Google เพื่อบังคับให้แถบนำทางรับรู้ข้อความของคุณเป็นที่อยู่และไม่ใช่การค้นหา ให้ใส่ `http://` นำหน้าข้อความ เช่นนี้: `http://mynode.local` หากยังไม่ทำงาน ลองใส่ “s” เช่นนี้: `https://mynode.local`


สิ่งนี้จะเข้าถึงอุปกรณ์ และคุณสามารถคลิกที่ลิงก์การตั้งค่า (ดู “วงกลม” สีน้ำเงินของฉันด้านล่าง) เพื่อแสดงหน้าจอนี้ซึ่งที่อยู่ IP ตั้งอยู่:


![image](assets/19.webp)


หน้านี้จะโหลดและคุณจะเห็น IP ของโหนด (“วงกลม” สีน้ำเงิน)


![image](assets/20.webp)


จากนั้น ในอนาคต คุณสามารถพิมพ์ 192.168.0.150 หรือ http://192.168.0.150 ลงในเบราว์เซอร์ของคุณได้


สำหรับ Raspiblitz (เมื่อไม่ได้เชื่อมต่อหน้าจอ) คุณจำเป็นต้องใช้วิธีการที่แตกต่างออกไป (ซึ่งใช้ได้กับ MyNode ด้วยเช่นกัน):


เข้าสู่ระบบหน้าเว็บของเราเตอร์ของคุณ – ที่นี่เราจะพบที่อยู่ IP ของอุปกรณ์ทั้งหมดที่เชื่อมต่อของคุณ หน้าเว็บของเราเตอร์จะเป็นที่อยู่ IP ที่คุณป้อนในเว็บเบราว์เซอร์ ของฉันดูเหมือนว่า:


http://192.168.0.1


ในการรับข้อมูลการเข้าสู่ระบบของเราเตอร์ คุณสามารถดูได้ในคู่มือผู้ใช้หรือบางครั้งอาจอยู่บนสติกเกอร์ที่ติดอยู่บนตัวเราเตอร์เอง มองหาชื่อผู้ใช้และรหัสผ่าน หากคุณหาไม่พบ ลองใช้ ชื่อผู้ใช้: "admin" รหัสผ่าน: "password"


หากคุณสามารถเข้าสู่ระบบได้ คุณจะเห็นอุปกรณ์ที่เชื่อมต่อของคุณและจากชื่อของพวกมัน อาจจะชัดเจนว่าอันไหนคือโหนดของคุณ ที่อยู่ IP จะอยู่ที่นั่น


### หากสองวิธีแรกล้มเหลว วิธีสุดท้ายจะได้ผลแต่ยุ่งยาก:


ขั้นแรก ค้นหาที่อยู่ IP ของอุปกรณ์ใดๆ บนเครือข่ายของคุณ (คอมพิวเตอร์ปัจจุบันก็ใช้ได้)


บน Mac คุณจะพบมันในการตั้งค่าเครือข่าย:


![image](assets/21.webp)


เราสนใจใน 4 องค์ประกอบแรก (192.168.0) ไม่ใช่องค์ประกอบที่ 4 คือ “166” ที่คุณเห็นในภาพ (ของคุณจะแตกต่างออกไป)


สำหรับ Linux ให้ใช้บรรทัดคำสั่ง:


```bash
ifconfig | grep inet
```


เส้นแนวตั้งนั้นคือสัญลักษณ์ “pipe” และคุณจะพบมันอยู่ใต้ปุ่ม <delete> คุณจะเห็นผลลัพธ์บางอย่างและที่อยู่ IP (ไม่ต้องสนใจ 127.0.0.1 มันไม่ใช่ และไม่ต้องสนใจ netmask)


สำหรับ Windows ให้เปิดพรอมต์คำสั่ง (cmd) และพิมพ์:


```bash
ipconfig/all
```


และกด Enter ที่อยู่ IP สามารถพบได้ในผลลัพธ์


นั่นเป็นส่วนที่ง่าย ส่วนที่ยากคือการหาที่อยู่ IP ของโหนดของคุณ – เราจำเป็นต้องเดาแบบ brute-force สมมติว่าที่อยู่ IP ของคอมพิวเตอร์ของคุณเริ่มต้นด้วย 192.168.0.xxx จากนั้นสำหรับโหนดของคุณ ในเบราว์เซอร์ ให้ลอง: `https://192.168.0.2`


ตัวเลขที่เล็กที่สุดที่เป็นไปได้คือ 2 (0 หมายถึงอุปกรณ์ใด ๆ และ 1 เป็นของเราเตอร์) และตัวเลขที่ใหญ่ที่สุดที่ฉันเชื่อคือ 255 (ซึ่งบังเอิญเป็น 11111111 ในระบบเลขฐานสอง ซึ่งเป็นตัวเลขที่ใหญ่ที่สุดที่เก็บโดย 1 ไบต์)


ทีละขั้นตอน ทำงานของคุณขึ้นไปจนถึง 255 ในที่สุด คุณจะหยุดที่หมายเลขที่ถูกต้องที่โหลดหน้าของคุณ MyNode (หรือหน้าของ RaspiBlitz) จากนั้นคุณจะรู้ว่าต้องใส่หมายเลขใดในการตั้งค่าเครือข่าย Electrum ของคุณเพื่อเชื่อมต่อกับโหนดของคุณ

## ลบหุ่นจำลอง wallet


ตอนนี้เราได้เชื่อมต่อกับโหนดเพียงหนึ่งเดียวสำเร็จแล้ว นี่คือวิธีที่ Electrum จะโหลดโดยค่าเริ่มต้นจากนี้ไป คุณควรลบ wallet ที่เป็นตัวอย่างออก (เมนู: ไฟล์ –> ลบ) เผื่อว่าคุณส่งเงินไปยัง wallet ที่ไม่ปลอดภัยนี้โดยไม่ตั้งใจ (มันไม่ปลอดภัยเพราะเราไม่ได้สร้างมันในวิธีที่ปลอดภัย)


## สร้างแบบฝึกหัด wallet


หลังจากลบ wallet ตัวอย่างออกแล้ว ให้เริ่มใหม่และสร้างขึ้นมาใหม่ในลักษณะเดียวกัน คราวนี้ให้จดคำของ seed และเก็บไว้ให้ค่อนข้างปลอดภัย


เป็นความคิดที่ดีที่จะเรียนรู้ว่า Electrum ทำงานอย่างไรกับการฝึกฝน wallet นี้ โดยไม่ต้องใช้ฮาร์ดแวร์ wallet ที่ยุ่งยาก (จำเป็นสำหรับความปลอดภัยสูง) ใส่บิตคอยน์เพียงเล็กน้อยใน wallet นี้ – สมมติว่าคุณจะสูญเสียเงินนี้ เมื่อเชี่ยวชาญแล้ว ให้เรียนรู้การใช้ Electrum กับฮาร์ดแวร์ wallet


ใน wallet ใหม่ที่คุณสร้างขึ้น คุณจะเห็นรายการที่อยู่ ที่อยู่สีเขียวเรียกว่า "ที่อยู่รับ" พวกมันเป็นผลลัพธ์ของ 3 สิ่ง:



- The seed phrase
- passphrase
- เส้นทางการอนุพันธ์


wallet ใหม่ของคุณมีชุดที่อยู่รับที่สามารถสร้างขึ้นได้ทางคณิตศาสตร์และสามารถสร้างซ้ำได้โดยซอฟต์แวร์ wallet ใด ๆ ที่มี seed, passphrase และเส้นทางการอนุพันธ์ มีทั้งหมด 4.3 พันล้านที่อยู่! มากกว่าที่คุณจะต้องการ Electrum จะแสดงให้คุณเห็นเพียง 20 ที่อยู่แรก และจะแสดงเพิ่มเติมเมื่อคุณใช้ที่อยู่แรกหมดแล้ว


สามารถค้นหาข้อมูลเพิ่มเติมเกี่ยวกับคีย์ส่วนตัวของบิตคอยน์ได้ในคู่มือนี้

เนื่องจากคุณป้อนวลี seed ในการสร้าง wallet, Electrum มีคีย์ส่วนตัวสำหรับแต่ละที่อยู่ และสามารถใช้จ่ายจากที่อยู่เหล่านี้ได้

## ทดสอบ Wallet (เรียนรู้วิธีการใช้งาน)


ที่นี่ฉันจะแสดงวิธีรับ UTXO ไปยัง wallet ของคุณและจากนั้นย้ายมัน (ใช้จ่าย) ไปยังที่อยู่อื่นภายใน wallet นี่เป็นจำนวนเงินที่น้อยมากที่เราจะไม่กังวลหากสูญเสียไป


สิ่งนี้มีวัตถุประสงค์หลายประการ



- มันจะพิสูจน์ว่าคุณมีพลังในการใช้จ่ายเหรียญใน wallet ใหม่
- มันจะแสดงวิธีการใช้ซอฟต์แวร์ Electrum เพื่อทำการใช้จ่าย (และคุณสมบัติบางอย่าง) ก่อนที่เราจะเพิ่มความซับซ้อนเพิ่มเติมเพื่อความปลอดภัย (โดยใช้ฮาร์ดแวร์ wallet หรือคอมพิวเตอร์ที่แยกออกจากเครือข่าย)
- มันจะเสริมความคิดที่ว่าคุณมีที่อยู่มากมายให้เลือกเพื่อรับและใช้จ่าย ภายใน wallet เดียวกัน

ตอนนี้ที่เรามี UTXO ใน wallet ของเรา เราควรติดป้ายกำกับมัน ป้ายกำกับนี้มีเพียงเราที่สามารถเห็นได้ มันไม่มีส่วนเกี่ยวข้องกับบัญชีแยกประเภทสาธารณะ ป้ายกำกับ Electrum ทั้งหมดของเราจะมองเห็นได้เฉพาะบนคอมพิวเตอร์ที่เราใช้งานอยู่ เราสามารถบันทึกไฟล์และใช้มันเพื่อกู้คืนป้ายกำกับทั้งหมดของเราไปยังคอมพิวเตอร์เครื่องอื่นที่ใช้ wallet เดียวกันได้จริงๆ

ในอุดมคติ คุณควรหลีกเลี่ยงการสะสมเหรียญหลายเหรียญในที่อยู่เดียวกัน หากคุณตัดสินใจที่จะทำเช่นนั้น (อย่า) คุณสามารถติดป้ายกำกับแต่ละเหรียญแทนที่จะใช้ป้ายกำกับเดียวกันทั้งหมดโดยใช้วิธีที่อยู่ คุณไม่สามารถไปที่แท็บ "เหรียญ" และแก้ไขป้ายกำกับที่นั่นได้จริงๆ (ไม่ นั่นจะเป็นการใช้งานที่ง่ายเกินไป!) คุณต้องไปที่แท็บประวัติ ค้นหาธุรกรรม ติดป้ายกำกับนั้น แล้วคุณจะเห็นป้ายกำกับในส่วนเหรียญ ป้ายกำกับใดๆ ที่คุณเห็นในส่วนเหรียญมาจากป้ายกำกับ Address หรือป้ายกำกับประวัติ แต่ป้ายกำกับประวัติใดๆ จะมีผลเหนือกว่าป้ายกำกับที่อยู่ใดๆ ในการสำรองข้อมูลป้ายกำกับของคุณไปยังไฟล์ คุณสามารถส่งออกจากเมนูที่ด้านบน wallet–>lables–>export


ต่อไป, มาลองใช้เหรียญจากที่อยู่แรกไปยังที่อยู่ที่สองกัน คลิกขวาที่อยู่แรกแล้วเลือก “spend from” (จริงๆ แล้วในสถานการณ์นี้ไม่จำเป็นต้องทำ แต่ลองนึกภาพว่าเรามีเหรียญจำนวนมากในหลายๆ ที่อยู่; การใช้ฟีเจอร์นี้ เราสามารถบังคับให้ wallet ใช้เฉพาะเหรียญที่เราต้องการได้ หากเราต้องการเลือกเหรียญหลายๆ เหรียญในหลายๆ ที่อยู่ เราสามารถเลือกที่อยู่เหล่านั้นด้วยการคลิกซ้ายขณะกดปุ่มคำสั่ง, จากนั้นคลิกขวา และเลือก “spend from”:

คุณยังสามารถใช้เหรียญแต่ละเหรียญภายในที่อยู่หนึ่งและยกเว้นเหรียญอื่นๆ ในที่อยู่เดียวกันได้ แต่ไม่แนะนำให้ทำเช่นนี้เพราะคุณกำลังทิ้งเหรียญไว้ในที่อยู่ที่ถูกทำให้อ่อนแอลงทางการเข้ารหัสเนื่องจากการใช้เหรียญหนึ่งเหรียญ (อีกเหตุผลหนึ่งที่ไม่ควรใส่เหรียญหลายเหรียญในที่อยู่เดียวกัน นอกจากเหตุผลด้านความเป็นส่วนตัว คือเมื่อคุณควรใช้เหรียญทั้งหมดหากคุณใช้เหรียญหนึ่งเหรียญ ซึ่งจะทำให้มีค่าใช้จ่ายสูงโดยไม่จำเป็น) นี่คือวิธีการเลือกเหรียญเดียวจากที่อยู่ที่ใช้ร่วมกัน แต่ไม่ควรทำ:

เราจะเห็นกล่อง/หน้าต่างสีขาวภายในสองกล่อง กล่องด้านบนคือหน้าต่างอินพุต (เหรียญที่ถูกใช้จ่าย) และกล่องด้านล่างคือเอาต์พุต (ที่ที่เหรียญกำลังไป)


โปรดทราบว่า สถานะ (บนซ้าย) คือ “unsigned” ในขณะนี้ “Amount sent” คือ 0 เพราะเหรียญกำลังถูกโอนภายใน wallet ค่าธรรมเนียมคือ 481 sats โปรดทราบว่าหากเป็น 480 sats ศูนย์สุดท้ายจะถูกตัดออก เช่นนี้ 0.0000048 และสำหรับตาที่เหนื่อยล้า อาจดูเหมือน 48 sats – ระวังด้วย (บางสิ่งที่นักพัฒนา Electrum ควรแก้ไข)


ขนาดของธุรกรรมหมายถึงขนาดของข้อมูลในหน่วยไบต์ ไม่ใช่จำนวนของบิตคอยน์ “replace by fee” ถูกเปิดใช้งานโดยค่าเริ่มต้น และมันช่วยให้คุณสามารถส่งธุรกรรมอีกครั้งด้วยค่าธรรมเนียมที่สูงขึ้นหากจำเป็น LockTime ช่วยให้คุณปรับเมื่อธุรกรรมมีผล – ฉันยังไม่ได้ลองใช้มัน แต่แนะนำว่าอย่าใช้มันเว้นแต่คุณจะเข้าใจอย่างถ่องแท้ในสิ่งที่คุณกำลังทำและมีการฝึกฝนกับจำนวนเงินเล็กน้อยแล้ว


ที่ด้านล่าง เรามีเครื่องมือปรับค่าธรรมเนียม mining ที่หรูหรา สิ่งที่คุณต้องทำสำหรับการโอนภายในคือการตั้งค่าธรรมเนียมขั้นต่ำที่ 1 sat/byte เพียงพิมพ์ตัวเลขลงในช่องค่าธรรมเนียมเป้าหมายด้วยตนเอง สำหรับการตรวจสอบค่าธรรมเนียมที่เหมาะสมสำหรับการชำระเงินภายนอก คุณสามารถปรึกษา https://mempool.space เพื่อดูว่ามีความหนาแน่นของ mempool อย่างไร และมีการแสดงค่าธรรมเนียมที่แนะนำบางส่วน


![image](assets/41.webp)


ฉันได้เลือก 1 sat/byte.


ในหน้าต่างอินพุต เราเห็นสองรายการ รายการแรกคือการบริจาค 5000 sat เราเห็นทางซ้ายเป็นแฮชธุรกรรม (ซึ่งเราสามารถค้นหาได้บนบล็อกเชน) ถัดจากนั้นมี “21” – ซึ่งบ่งบอกว่าเป็นเอาต์พุตที่ติดป้ายกำกับ 21 ในธุรกรรมนั้น (จริงๆ แล้วเป็นเอาต์พุตที่ 22 เพราะรายการแรกติดป้ายกำกับเป็นศูนย์)


สิ่งที่ควรทราบที่นี่: UTXOs มีอยู่เฉพาะภายในธุรกรรมเท่านั้น ในการใช้จ่าย UTXO เราต้องอ้างอิงถึงมัน และใส่อ้างอิงนั้นลงในอินพุตของธุรกรรมใหม่ จากนั้นเอาต์พุตจะกลายเป็น UTXOs ใหม่และ UTXO เก่าจะกลายเป็น STXO (เอาต์พุตธุรกรรมที่ใช้แล้ว)


บรรทัดที่สองคือการบริจาค 10,000 sat มี “0” ถัดจากแฮชธุรกรรมที่มาจากเพราะเป็นเอาต์พุตแรก (และอาจเป็นเพียงเอาต์พุตเดียว) สำหรับธุรกรรมนั้น


ในหน้าต่างด้านล่าง เราจะเห็นที่อยู่ของเรา สังเกตว่าจำนวนรวมของบิตคอยน์ในอินพุตไม่ค่อยตรงกับจำนวนรวมของเอาต์พุต ความแตกต่างนี้จะไปที่นักขุด นักขุดจะดูความคลาดเคลื่อนในธุรกรรมทั้งหมดในบล็อกที่พยายามขุด และเพิ่มตัวเลขนั้นเข้าไปในรางวัลของตน (ค่าธรรมเนียม Mining ในลักษณะนี้จะถูกตัดขาดจากห่วงโซ่ของธุรกรรมและเริ่มชีวิตใหม่)


หากเราปรับค่าธรรมเนียม mining ค่าผลลัพธ์จะเปลี่ยนแปลงโดยอัตโนมัติ


**มันคุ้มค่าที่จะชี้ให้เห็นที่นี่:** สังเกตสีของที่อยู่ในหน้าต่างธุรกรรม จำไว้ว่าที่อยู่สีเขียวจะถูกแสดงในแท็บที่อยู่ของคุณ หากที่อยู่ถูกเน้นเป็นสีเขียว (หรือสีเหลือง) ในหน้าต่างธุรกรรม นั่นหมายความว่า Electrum ได้รับรู้ที่อยู่นั้นว่าเป็นของตัวเอง หากที่อยู่ไม่มีการเน้นสี นั่นคือที่อยู่นอก (นอก wallet ที่เปิดอยู่ในปัจจุบัน) และคุณควรตรวจสอบด้วยความระมัดระวังเป็นพิเศษ


เมื่อคุณตรวจสอบทุกอย่างในธุรกรรมและมั่นใจว่าคุณพอใจกับเหรียญที่คุณใช้จ่าย และสถานที่ที่เหรียญจะไป คุณสามารถคลิก "เสร็จสิ้น"


![image](assets/42.webp)


หลังจากที่คุณคลิก "finalise" คุณจะไม่สามารถแก้ไขได้อีกต่อไป - หากคุณจำเป็นต้องแก้ไข คุณต้องปิดหน้านี้และเริ่มใหม่ สังเกตว่าปุ่ม "finalise" ได้เปลี่ยนเป็น "export" และมีปุ่มใหม่ปรากฏขึ้น: "save", "combine", "sign" และ "broadcast" ปุ่ม "broadcast" จะเป็นสีเทาเพราะธุรกรรมยังไม่ได้ลงนามและจึงไม่ถูกต้องในขั้นตอนนี้


เมื่อคุณคลิกเซ็นชื่อ หากคุณมีรหัสผ่านสำหรับ wallet คุณจะถูกขอให้ใส่รหัสผ่านนั้น จากนั้นสถานะ (ด้านบนขวา) จะเปลี่ยนจาก “Unsigned” เป็น “Signed” จากนั้นปุ่ม “Broadcast” จะพร้อมใช้งาน


หลังจากที่คุณออกอากาศแล้ว คุณสามารถปิดหน้าต่างการทำธุรกรรมได้ หากคุณไปที่แท็บที่อยู่ คุณจะเห็นว่าที่อยู่แรกว่างเปล่า และที่อยู่ที่สองมี 1 UTXO.


หมายเหตุ: คุณจะเห็นการเปลี่ยนแปลงทั้งหมดนี้แม้กระทั่งก่อนที่ธุรกรรมจะถูกขุดลงในบล็อก หรือ "ยืนยัน" แล้ว นี่เป็นเพราะว่า Electrum อัปเดตยอดคงเหลือ/ธุรกรรมโดยอิงจากไม่เพียงแค่ข้อมูลบล็อกเชนเท่านั้น แต่ยังรวมถึงข้อมูลเมมพูลด้วย ไม่ใช่ว่ากระเป๋าเงินทั้งหมดจะทำเช่นนี้


สิ่งที่ควรชี้ให้เห็นคือแทนที่จะทำการกระจายข้อมูล เราสามารถบันทึกธุรกรรมไว้ใช้ในภายหลังได้ สามารถบันทึกได้ทั้งในสถานะที่ยังไม่ได้ลงนามหรือที่ลงนามแล้ว


คลิกปุ่ม "export" (ขัดแย้งกัน, อย่าคลิกปุ่ม "save") แล้วคุณจะเห็นตัวเลือกหลายอย่าง การทำธุรกรรมถูกเข้ารหัสด้วยข้อความ ดังนั้นจึงสามารถบันทึกได้หลายวิธี

การโหลดธุรกรรมที่บันทึกไว้อาจไม่ใช่เรื่องที่เข้าใจได้ง่าย ดังนั้นควรให้ความสนใจเป็นพิเศษ การโหลดธุรกรรมไม่ใช่ "เครื่องมือ" แต่ตัวเลือกนี้ถูกซ่อนไว้ในเมนูเครื่องมือ (อีกสิ่งหนึ่งที่นักพัฒนา Electrum ควรแก้ไข)


กระบวนการที่คล้ายกันสามารถทำได้กับธุรกรรมที่บันทึกเป็นไฟล์ ลองฝึกฝนด้วยวิธีใดวิธีหนึ่งภายใน wallet เดียวกัน ฉันจะไม่อธิบายที่นี่ แต่คุณสามารถใช้ฟีเจอร์นี้เพื่อส่งต่อธุรกรรมระหว่าง wallet เดียวกันบนคอมพิวเตอร์ต่าง ๆ ระหว่างกระเป๋าเงินหลายลายเซ็น และไปยังและจากกระเป๋าเงินฮาร์ดแวร์ นี่คือคำแนะนำบางส่วน


ตอนนี้ กลับมาที่ปุ่ม “บันทึก” – นี่ไม่ใช่วิธีการบันทึกข้อความธุรกรรม สิ่งที่ปุ่มนี้ทำจริง ๆ คือบอกให้ Electrum wallet รับรู้ว่าธุรกรรมนี้บนคอมพิวเตอร์ท้องถิ่นถูกส่งเป็นการชำระเงิน หากคุณทำเช่นนั้นโดยบังเอิญ คุณจะเห็นธุรกรรมพร้อมไอคอนคอมพิวเตอร์เล็ก ๆ คุณสามารถคลิกขวาและลบธุรกรรมได้ – ไม่ต้องกังวล คุณไม่สามารถลบ bitcoin ด้วยวิธีนี้ Electrum จะลืมว่าธุรกรรมนี้เคยเกิดขึ้น และจะ “คืนเงิน” sats กลับมาและแสดง sats ในตำแหน่งที่ถูกต้องที่พวกมันมีอยู่จริง


### เปลี่ยนที่อยู่


ที่อยู่สำหรับเปลี่ยนมีความน่าสนใจ คุณจำเป็นต้องเข้าใจ UTXOs เพื่อที่จะเข้าใจคำอธิบายนี้ หากคุณกำลังใช้จ่ายไปยังที่อยู่ในจำนวนที่น้อยกว่า UTXO แล้ว บิทคอยน์ที่เหลือจะไปยังผู้ขุดเว้นแต่จะมีการระบุเอาท์พุตสำหรับเปลี่ยน


คุณอาจมี 6.15 bitcoin UTXO และต้องการใช้ 0.15 bitcoin เพื่อบริจาคให้กับผู้ประท้วงที่ถูกกดขี่โดยรัฐบาล “ประชาธิปไตย” เผด็จการที่ไหนสักแห่งในโลก จากนั้นคุณจะนำ 6.15 bitcoin (โดยใช้ฟังก์ชัน “spend from” ใน Electrum) และใส่มันในธุรกรรม


คุณจะวางที่อยู่ของผู้ประท้วงในช่อง "จ่ายให้" บางทีคุณอาจใส่ "EndTheFed & WEF" ในช่อง "คำอธิบาย" และสำหรับจำนวนเงิน คุณจะใส่ 0.15 bitcoin แล้วคลิก "จ่าย" จากนั้น "ขั้นสูง"


ในหน้าจอธุรกรรม สำหรับหน้าต่างป้อนข้อมูล คุณจะเห็น 6.15 bitcoin UTXO สำหรับหน้าต่างผลลัพธ์ คุณจะเห็นที่อยู่ที่ไม่มีการเน้น (นี่คือที่อยู่ของผู้ประท้วง) พร้อมกับ 0.15 bitcoin ข้างๆ นอกจากนี้คุณยังจะเห็นที่อยู่สีเหลืองที่มี bitcoin น้อยกว่า 6.0 เล็กน้อย นี่คือที่อยู่เปลี่ยนที่ถูกเลือกโดยอัตโนมัติโดย wallet จากหนึ่งในที่อยู่เปลี่ยนสีเหลืองของตัวเอง วัตถุประสงค์ของที่อยู่เปลี่ยนคือเพื่อให้ wallet สามารถใส่เหรียญเปลี่ยนในนั้นโดยไม่ทำให้ความพร้อมใช้งานของที่อยู่รับที่คุณอาจมีแผนสำหรับ หรือส่งใบแจ้งหนี้สำหรับ หากพวกเขาจะถูกใช้ในภายหลังโดยลูกค้า ตัวอย่างเช่น คุณไม่ต้องการให้ wallet ของคุณใช้พวกเขาโดยอัตโนมัติและเติมเต็มพวกเขา มันยุ่งเหยิงและไม่ดีต่อความเป็นส่วนตัว


โปรดทราบว่าเมื่อคุณปรับค่าธรรมเนียม mining จำนวนเงินที่เปลี่ยนแปลงจะปรับโดยอัตโนมัติ ไม่ใช่จำนวนเงินที่ชำระ

## กระเป๋าสตางค์


ฉันต้องการสาธิตการดูเท่านั้น Wallet โดยใช้ Electrum ในการทำเช่นนั้น ฉันต้องกำหนด "wallet" ก่อน มีสองวิธีที่ "wallet" ถูกใช้ใน Bitcoin:



- ประเภท A, “wallet” – หมายถึงซอฟต์แวร์ที่แสดงที่อยู่และยอดคงเหลือของคุณ เช่น Electrum, Blue Wallet, Sparrow Wallet เป็นต้น



- ประเภท B, “wallet” – หมายถึงคอลเล็กชันที่ไม่ซ้ำกันของที่อยู่ที่เชื่อมโยงกับการรวมกันของ seed_phrase/passphrase/derivation_path ของเรา มีที่อยู่ 8.6 พันล้านที่อยู่ใน wallet ใด ๆ (ที่อยู่รับ 4.3 พันล้าน และที่อยู่เปลี่ยน 4.3 พันล้าน) หากคุณเปลี่ยนแปลงสิ่งใดในวลี seed, passphrase หรือเส้นทางการอนุพันธ์ คุณจะได้รับ wallet ที่ไม่ได้ใช้พร้อมที่อยู่ว่างใหม่ทั้งหมด 8.6 พันล้านที่อยู่ที่ไม่ซ้ำกันทั้งหมด


ประเภทใดที่ใครบางคนหมายถึงเมื่อใช้คำว่า "wallet" นั้นชัดเจนในบริบท


## กำลังดู Wallet – การฝึกซ้อม


ยังไม่ชัดเจนอย่างสมบูรณ์ว่า wallet สำหรับการดูคืออะไร แต่ฉันจะเริ่มโดยการอธิบายว่ามันคืออะไร วิธีการทำแบบฝึกหัด และจากนั้นเราจะกลับมาที่วัตถุประสงค์ของมันในภายหลังเมื่อฉันอธิบายเพิ่มเติมเกี่ยวกับกระเป๋าเงินฮาร์ดแวร์ (สำหรับการรีวิวเชิงลึกเกี่ยวกับวิธีการใช้ฮาร์ดแวร์ wallet และแบรนด์เฉพาะต่างๆ ดูที่นี่)


เราจะสร้าง wallet ปกติแบบจำลอง (คราวนี้เพิ่มความซับซ้อนเล็กน้อยด้วย passphrase) และจากนั้นสร้าง wallet ที่สอดคล้องกัน หากคุณต้องการ คุณสามารถคัดลอกแบบที่ฉันทำไว้ได้เลย หรือสร้างของคุณเองก็ได้ – wallet นี้จะถูกทิ้ง; อย่าใช้จริง เริ่มต้นด้วยการสร้าง seed ที่มี 12 คำ โดยใช้เว็บไซต์ Ian Coleman


โปรดสังเกตคำสุ่ม 12 คำในภาพหน้าจอด้านล่าง และฉันได้ป้อน passphrase ในช่อง passphrase:


PASSPHRASE: “Craig Wright เป็นคนโกหกและฉ้อโกงและควรถูกจำคุก นอกจากนี้ Ross Ulbricht ควรถูกปล่อยตัวจากเรือนจำทันที”


passphrase สามารถยาวได้ถึง 100 ตัวอักษร และควรจะไม่กำกวมและไม่สั้นเกินไป – ที่ฉันใช้เป็นเพียงเพื่อความสนุก – ฉันมักจะแนะนำให้หลีกเลี่ยงการใช้ตัวอักษรพิมพ์ใหญ่และสัญลักษณ์เพื่อลดความเครียดในการลองใช้หลายๆ แบบหากคุณมีปัญหาในการจำ passphrase ของคุณ

![image](assets/49.webp)


ขั้นตอนต่อไปที่คุณควรคุ้นเคยแล้ว ดังนั้นฉันจะระบุโดยไม่มีรูปภาพ:



- มาตรฐาน wallet
- ฉันมี seed อยู่แล้ว
- คัดลอกและวางคำ 12 คำในกล่อง หรือพิมพ์ด้วยตนเอง

- ป้อน passphrase ของคุณให้เหมือนกับที่คุณทำในหน้า Ian Coleman

- ไม่จำเป็นต้องเพิ่มรหัสผ่าน (ล็อค wallet)

คุณจะเห็น "BIP32 extended private/public" keys ด้วย – ให้ละเว้นสิ่งเหล่านี้ไปก่อนในตอนนี้


คีย์ขยายส่วนตัวของบัญชีสามารถใช้ในการสร้าง wallet ของคุณใหม่ได้อย่างสมบูรณ์ อย่างไรก็ตาม คีย์ขยายสาธารณะของบัญชีสามารถผลิตเวอร์ชันจำกัดของ wallet เดียวกันได้เท่านั้น (ดู wallet) – หากคุณใส่คีย์นี้ใน Electrum มันจะยังคงผลิตที่อยู่ทั้งหมด 8.6 พันล้านที่ seed หรือคีย์ขยายส่วนตัวจะมี แต่จะไม่มีคีย์ส่วนตัวที่ใช้ได้กับ Electrum ดังนั้นจึงไม่สามารถใช้จ่ายได้ ลองทำตอนนี้เพื่อแสดงให้เห็นถึงประเด็น:


คัดลอก “account extended public key” ไปยังคลิปบอร์ด


จากนั้นไปที่ Electrum ทิ้ง wallet ปัจจุบันที่เราทำไว้เปิดอยู่ แล้วไปที่ ไฟล์–>ใหม่/คืนค่า กระบวนการทำ wallet จะแตกต่างจากเดิมเล็กน้อย:



- มาตรฐาน wallet
- ใช้กุญแจหลัก
- วางคีย์สาธารณะที่ขยายในช่องแล้วดำเนินการต่อ
- ไม่จำเป็นต้องป้อน passphrase; มันเป็นส่วนหนึ่งของคีย์สาธารณะที่ขยายแล้ว
- ไม่จำเป็นต้องป้อนความหมายของสคริปต์และเส้นทางการสืบทอด
- ไม่จำเป็นต้องเพิ่มรหัสผ่าน (ล็อค wallet)


เมื่อ wallet โหลด คุณควรสังเกตว่าแอดเดรสที่โหลดนั้นเหมือนกับที่เคยโหลดเมื่อ seed ถูกป้อน คุณควรสังเกตด้วยว่าที่ด้านบนสุดในแถบชื่อเรื่อง มันบอกว่า "watching wallet" wallet นี้สามารถแสดงแอดเดรสและยอดคงเหลือของคุณ (โดยการตรวจสอบยอดคงเหลือผ่านโหนด) แต่คุณไม่สามารถลงนามธุรกรรมได้ (เพราะ watching wallet ไม่มีคีย์ส่วนตัว)

เหตุผลหนึ่ง และไม่ใช่เหตุผลหลัก คือคุณสามารถสังเกต wallet ของคุณและยอดคงเหลือบนคอมพิวเตอร์ได้โดยไม่ต้องเปิดเผยกุญแจส่วนตัวของคุณต่อมัลแวร์ใด ๆ บนคอมพิวเตอร์


อีกประการหนึ่งคือ จำเป็นต้องใช้เพื่อทำการชำระเงินหากคุณเลือกที่จะเก็บคีย์ส่วนตัวของคุณออกจากคอมพิวเตอร์; ฉันจะอธิบาย:


**กระเป๋าเงินฮาร์ดแวร์ (HWW)** ถูกสร้างขึ้นเพื่อให้มีอุปกรณ์ที่สามารถเก็บกุญแจส่วนตัวของคุณอย่างปลอดภัย (ล็อคด้วย PIN) โดยไม่เคยเปิดเผยกุญแจให้กับคอมพิวเตอร์ (แม้จะเชื่อมต่อกับคอมพิวเตอร์ผ่านสายเคเบิล) และตัวอุปกรณ์เองไม่สามารถเชื่อมต่อกับอินเทอร์เน็ตได้ อุปกรณ์ดังกล่าวไม่สามารถทำธุรกรรมได้ด้วยตัวเองเพราะธุรกรรมบิตคอยน์ทั้งหมดเริ่มต้นด้วยการอ้างอิง UTXO(s) บนบล็อกเชน (ซึ่งอยู่บนโหนด) wallet ต้องระบุว่า transaction ID ใดที่ UTXO อยู่ในนั้น และเอาต์พุตของธุรกรรมใดที่เป็นอันที่จะใช้จ่ายได้ เฉพาะหลังจากระบุอินพุตแล้วเท่านั้นที่สามารถสร้างธุรกรรมใหม่ได้ในตอนแรก ยังไม่ต้องพูดถึงการเซ็นชื่อ กระเป๋าเงินฮาร์ดแวร์ไม่สามารถสร้างธุรกรรมได้เพราะพวกเขาไม่มีการเข้าถึง UTXOs ใด ๆ – พวกเขาไม่ได้เชื่อมต่อกับอะไรเลย!


คีย์สาธารณะที่ขยายออกมามักจะถูกดึงออกมาจาก HWW และที่อยู่จะถูกแสดงบนคอมพิวเตอร์ – หลายคนจะคุ้นเคยกับซอฟต์แวร์ Ledger หรือ Trezor Suite ที่แสดงที่อยู่และยอดคงเหลือบนคอมพิวเตอร์ของพวกเขา – นี่คือการดู wallet โปรแกรมเหล่านี้สามารถสร้างธุรกรรมได้ แต่ไม่สามารถเซ็นได้ พวกเขาสามารถให้ธุรกรรมถูกเซ็นโดย HWWs ที่เชื่อมต่อกับพวกเขาเท่านั้น HWW จะรับธุรกรรมที่สร้างขึ้นใหม่จากการดู wallet เซ็นมัน แล้วส่งกลับไปยังคอมพิวเตอร์เพื่อกระจายไปยังโหนด **HWW ไม่สามารถกระจายได้ด้วยตัวเอง** การดู wallet ที่เกี่ยวข้องจะทำเช่นนั้น ด้วยวิธีนี้ กระเป๋าเงินทั้งสอง (คีย์สาธารณะ wallet บนคอมพิวเตอร์ และคีย์ส่วนตัว wallet ใน HWW) จะร่วมมือกันเพื่อ generate เซ็น และกระจาย ทั้งหมดนี้ในขณะที่มั่นใจว่าคีย์ส่วนตัวจะถูกแยกออกและห่างจากอุปกรณ์ที่เชื่อมต่ออินเทอร์เน็ต


## ธุรกรรม Bitcoin ที่ลงนามบางส่วน (PSBTs)


เป็นไปได้ที่ธุรกรรมจะถูกสร้างและบันทึกลงในไฟล์ จากนั้นโหลดใหม่ ลงนาม บันทึก โหลดใหม่อีกครั้ง และสุดท้ายก็เผยแพร่ – ฉันไม่ได้บอกว่าใครจะต้องทำแบบนี้; มันก็แค่สิ่งที่เป็นไปได้เท่านั้น


ตอนนี้พิจารณา 3 จาก 5 มัลติซิกเนเจอร์ wallet – คีย์ส่วนตัว 5 คีย์สร้าง wallet และต้องการ 3 คีย์เพื่อเซ็นชื่อธุรกรรมอย่างสมบูรณ์ (ดูที่นี่เพื่อเรียนรู้เพิ่มเติมเกี่ยวกับคีย์มัลติซิกเนเจอร์ wallet) เป็นไปได้ที่จะมีคอมพิวเตอร์ 5 เครื่องที่แตกต่างกันแต่ละเครื่องมีคีย์ส่วนตัวหนึ่งในห้าคีย์


คอมพิวเตอร์หนึ่งสามารถ generate การทำธุรกรรมและเซ็นชื่อได้ จากนั้นสามารถบันทึกลงไฟล์และส่งทางอีเมลไปยังคอมพิวเตอร์ 2 คอมพิวเตอร์ 2 สามารถเซ็นชื่อได้ และอาจบันทึกไฟล์เป็น QR โค้ด และส่ง QR ผ่านการโทร Zoom ไปยังคอมพิวเตอร์ 3 ที่อยู่อีกฝั่งของโลก คอมพิวเตอร์ 3 สามารถจับภาพ QR โหลดการทำธุรกรรมเข้าสู่ electrum และเซ็นชื่อการทำธุรกรรม หลังจากเซ็นชื่อครั้งแรก 2 ครั้ง การทำธุรกรรมเป็น PSBT (การทำธุรกรรมบิตคอยน์ที่เซ็นชื่อบางส่วน) หลังจากเซ็นชื่อครั้งที่ 3 การทำธุรกรรมกลายเป็นเซ็นชื่อครบถ้วนและถูกต้อง พร้อมสำหรับการกระจายสัญญาณ


หากต้องการเรียนรู้เพิ่มเติมเกี่ยวกับ PSBTS โปรดดูคู่มือนี้ (https://armantheparman.com/psbt/)


## การใช้กระเป๋าเงินฮาร์ดแวร์กับ Electrum


ฉันมีคู่มือเกี่ยวกับการใช้กระเป๋าเงินฮาร์ดแวร์ทั่วไป ซึ่งฉันคิดว่าสำคัญสำหรับผู้ที่เพิ่งเริ่มใช้ HWWs ที่จะอ่าน (https://armantheparman.com/using-hwws/)


นอกจากนี้ยังมีคำแนะนำเกี่ยวกับแบรนด์ต่างๆ ของ HWWs ที่เชื่อมต่อกับ Sparrow Bitcoin Wallet ที่พบได้ที่นี่ (https://armantheparman.com/hwws/)


นี่จะเป็นคู่มือแรกของฉันที่แสดงวิธีการใช้ฮาร์ดแวร์ wallet กับ Electrum – ฉันจะใช้ฮาร์ดแวร์ ColdCard wallet เพื่อสาธิต นี่ไม่ใช่คู่มือรายละเอียดเกี่ยวกับ ColdCard โดยเฉพาะ คู่มือนั้นอยู่ที่นี่ ฉันแค่แสดงจุดที่เฉพาะเจาะจงของ Electrum (https://armantheparman.com/cc/)


### เชื่อมต่อผ่านการ์ด micro SD (แยกอากาศ)


ก่อนที่จะเชื่อมต่อ wallet จริงของคุณผ่าน ColdCard ฉันหวังว่าคุณได้ทำตามขั้นตอนก่อนหน้านี้ในการโหลด Electrum ดัมมี่ wallet และตั้งค่าพารามิเตอร์เครือข่ายแล้ว


จากนั้น ใส่การ์ด SD ลงใน ColdCard ฉันสมมติว่าคุณได้สร้าง seed ของคุณแล้ว หากคุณกำลังเข้าถึง wallet ด้วย passphrase ให้ใช้มันตอนนี้ เลื่อนลงและเลือกเมนู Advanced/Tools –>Export Wallet –> Electrum Wallet.


คุณสามารถเลื่อนลงและอ่านข้อความได้ มันจะเสนอให้คุณเลือก "1" เพื่อป้อนหมายเลขบัญชีที่ไม่ใช่ศูนย์ (บางสิ่งที่เป็นส่วนหนึ่งของเส้นทางการอนุพันธ์) แต่ถ้าคุณทำตามคำแนะนำของฉัน คุณจะไม่ยุ่งกับเส้นทางการอนุพันธ์เริ่มต้น ดังนั้นคุณจะไม่ต้องการหมายเลขบัญชีที่ไม่ใช่ศูนย์ เพียงกดเครื่องหมายถูกเพื่อดำเนินการต่อ


จากนั้นเลือกความหมายของสคริปต์ คนส่วนใหญ่จะเลือก “Native Segwit”


มันจะบอกว่า “Electrum wallet file written” และจะแสดงชื่อไฟล์ ให้จดจำไว้ในใจ


จากนั้นนำการ์ด micro SD ออกและใส่ในคอมพิวเตอร์ที่มี Electrum


ระบบปฏิบัติการบางระบบจะเปิดโปรแกรมสำรวจไฟล์โดยอัตโนมัติเมื่อคุณใส่ microSD หลายคนจะเห็นไฟล์ wallet ใหม่และดับเบิลคลิกที่มัน และสงสัยว่าทำไมมันถึงไม่ทำงาน มันไม่ใช่การออกแบบที่ดีนัก คุณต้องไม่สนใจโปรแกรมสำรวจไฟล์ (ปิดมัน) และเปิดไฟล์ wallet โดยใช้ Electrum:


เปิด Electrum หากมีการเปิด wallet อื่นอยู่แล้ว ให้เลือกไฟล์ –> ใหม่ เรากำลังมองหาหน้าต่างนี้:


![image](assets/52.webp)


นี่คือเคล็ดลับ มันไม่ใช่สิ่งที่เข้าใจได้ทันที คลิก "เลือก" จากนั้นนำทางระบบไฟล์บนการ์ด microSD และค้นหาไฟล์ wallet แล้วเปิดมัน


ตอนนี้คุณได้เปิดฮาร์ดแวร์ wallet ที่สอดคล้องกับการดู wallet ของคุณแล้ว เยี่ยมมาก


### เชื่อมต่อผ่านสาย USB


วิธีนี้ควรจะง่ายกว่า แต่สำหรับคอมพิวเตอร์ Linux มันยากกว่ามาก สิ่งที่เรียกว่า "Udev rules" จำเป็นต้องได้รับการอัปเดต นี่คือวิธีการ (คู่มือรายละเอียด https://armantheparman.com/gpg-articles/ ) และโดยสังเขป:


เป็นความคิดที่ดีที่จะตรวจสอบให้แน่ใจว่าระบบเป็นปัจจุบัน จากนั้น:


```bash
sudo apt-get install libusb-1.0-0-dev libudev-dev
```


แล้ว...


```bash
python3 -m pip install ckcc-protocol
```


หากคุณได้รับข้อผิดพลาด ตรวจสอบให้แน่ใจว่าได้ติดตั้ง pip แล้ว คุณสามารถตรวจสอบได้ด้วย (pip –version) และคุณสามารถติดตั้งได้ด้วย (sudo apt install python3-pip)


สร้างหรือแก้ไขไฟล์ที่มีอยู่, /etc/udev/rules.d/


เช่นนี้:


```bash
sudo nano /etc/udev/rules.d
```


โปรแกรมแก้ไขข้อความจะเปิดขึ้น คัดลอกข้อความจากที่นี่และวางลงในไฟล์ rules.d บันทึกและออก


![image](assets/53.webp)


จากนั้นรันคำสั่งเหล่านี้ทีละคำสั่ง:


```bash
sudo groupadd plugdev
sudo usermod -aG plugdev $(whoami)
sudo udevadm control –reload-rules && sudo udevadm trigger
```


หากคุณได้รับข้อความ “group plugdev” มีอยู่แล้ว ไม่เป็นไร ดำเนินการต่อ หลังจากคำสั่งที่สอง คุณจะไม่ได้รับข้อเสนอแนะ/การตอบกลับ เพียงดำเนินการต่อไปยังคำสั่งที่สาม


คุณอาจจำเป็นต้องตัดการเชื่อมต่อและเชื่อมต่อ ColdCard กับคอมพิวเตอร์ใหม่อีกครั้ง


หากหลังจากทั้งหมดนี้คุณยังไม่สามารถเชื่อมต่อ ColdCard ได้ ฉันขอแนะนำให้ลองอัปเดตเฟิร์มแวร์ (คู่มือจะมาเร็วๆ นี้ แต่ตอนนี้คุณสามารถหาคำแนะนำได้บนเว็บไซต์ของผู้ผลิต)


ถัดไป, สร้าง wallet ใหม่:



- มาตรฐาน wallet
- ใช้ฮาร์ดแวร์อุปกรณ์
- มันจะสแกนและตรวจจับ ColdCard ของคุณ ดำเนินการต่อ
- เลือกความหมายของสคริปต์และเส้นทางการสืบทอด
- ตัดสินใจว่าควรเข้ารหัสไฟล์ wallet หรือไม่ (แนะนำ)


### ธุรกรรมโดยใช้ ColdCard


เมื่อเชื่อมต่อสายเคเบิลแล้ว การทำธุรกรรมจะง่าย การลงนามธุรกรรมจะราบรื่น


หากใช้เครื่องในลักษณะที่แยกออกจากเครือข่าย คุณต้องถ่ายโอนธุรกรรมที่บันทึกไว้ระหว่างอุปกรณ์ด้วยตนเองโดยใช้การ์ด microSD มีเคล็ดลับบางอย่าง


หลังจากสร้างธุรกรรมและทำให้เสร็จสิ้นแล้ว คุณต้องคลิกปุ่มส่งออกที่มุมล่างซ้าย คุณจะเห็น “save to file” ซึ่งขัดกับความคาดหมาย ไม่ใช่สิ่งที่เราต้องการ คุณต้องไปที่ตัวเลือกเมนูสุดท้ายที่ระบุว่า “for hardware wallets” ก่อน แล้วจากนั้น ในการเลือกนั้น ให้หาตัวเลือก “save to file” อื่นและเลือก จากนั้นบันทึกไฟล์ลงใน microSD นำการ์ดออกและใส่ใน ColdCard จำไว้ว่าคุณอาจต้องใช้ passphrase เพื่อเลือก wallet ที่ถูกต้อง หน้าจอจะแสดงว่าพร้อมที่จะเซ็นชื่อ คลิกเครื่องหมายถูก ตรวจสอบธุรกรรม และดำเนินการต่อโดยยืนยันด้วยเครื่องหมายถูก เมื่อเสร็จแล้ว ให้นำการ์ดออกและใส่กลับเข้าไปในคอมพิวเตอร์


จากนั้นเราจำเป็นต้องเปิดธุรกรรมโดยใช้ electrum ฟังก์ชันนี้ซ่อนอยู่ในเมนู tools –> load transaction นำทางไปยังระบบไฟล์และค้นหาไฟล์ จะมีไฟล์สามไฟล์ทุกครั้งที่คุณเซ็น ไฟล์ที่บันทึกต้นฉบับที่ Watching Wallet สร้างขึ้น และสองไฟล์ที่สร้างโดย ColdCard (ฉันไม่รู้ว่าทำไมมันถึงทำเช่นนี้) หนึ่งจะบอกว่า “signed” และอีกหนึ่งจะบอกว่า “final” มันไม่ใช่เรื่องที่เข้าใจง่ายแต่ไฟล์ “signed” นั้นไม่มีประโยชน์ เราจำเป็นต้องเปิดธุรกรรม “final”


เมื่อคุณโหลดเสร็จแล้ว คุณสามารถคลิก "ออกอากาศ" และการชำระเงินจะถูกดำเนินการ



แมค:


```bash
/Users/your_user_name_goes_here/.electrum
```


ลินุกซ์:



หากคุณลบโปรแกรม Electrum ออกจากคอมพิวเตอร์ของคุณ ไดเรกทอรีที่ซ่อนอยู่จะยังคงอยู่ เว้นแต่ว่าคุณจะลบมันด้วยตัวเอง


ในการอัปเกรด electrum คุณจะต้องทำตามขั้นตอนเดียวกันกับที่ฉันได้อธิบายไว้ในตอนต้นเพื่อดาวน์โหลดและตรวจสอบ จากนั้นคุณจะมีสำเนาของโปรแกรมสองชุดในคอมพิวเตอร์ของคุณ และคุณสามารถเรียกใช้ได้ทั้งสองชุด – แต่ละโปรแกรมจะเข้าถึงโฟลเดอร์ electrum ที่ซ่อนอยู่เดียวกันสำหรับการกำหนดค่าและการเข้าถึง wallet ทุกสิ่งที่เราบันทึกไว้ เช่น หน่วยฐาน โหนดเริ่มต้นที่จะเชื่อมต่อ การตั้งค่าอื่น ๆ และการเข้าถึงกระเป๋าเงิน จะยังคงอยู่เพราะทั้งหมดนั้นถูกบันทึกไว้ในโฟลเดอร์นั้น
