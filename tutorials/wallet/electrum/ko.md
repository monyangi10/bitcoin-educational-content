## 왜 일렉트럼인가요?


이 글은 제가 몇 년 동안 사용하면서 개발한 모든 함정과 단점에 대한 해결책을 제시하고 학생들에게 Bitcoin 보안/개인정보 보호에 대해 가르치면서 일렉트럼 Wallet을 사용하는 방법에 대한 자세한 안내서입니다. Electrum은 모든 것을 가능한 한 단순하게 유지하고 초보자 수준에 머물기를 원하는 사람에게는 최고의 Bitcoin Wallet이 아닙니다. 대신 '파워' 사용자이거나 파워 유저가 되고자 하는 분들을 위한 제품입니다.


초보 비트코인의 경우, 숙련된 사용자의 감독 하에 길을 알려주는 것이 좋습니다. 혼자서 사용법을 배우는 경우, 처음에는 적은 수의 Sats만 있는 테스트 환경에서 천천히 사용한다면 안전할 것입니다. 이 가이드는 이러한 노력을 지원하지만 다른 사람에게도 좋은 참고 자료가 될 수 있습니다.


**경고: 이 가이드는 상당히 길기 때문에 하루에 설명된 모든 단계를 수행하려고 하지 마세요. 가이드를 잘 보관하고 시간이 지남에 따라 꾸준히 진행하는 것이 좋습니다.

## 일렉트럼 다운로드


Bitcoin 거래에는 전용 Bitcoin 컴퓨터를 사용하는 것이 가장 이상적입니다(이에 대한 저의 가이드 https://armantheparman.com/mint/) _(개인정보 보호 섹션에서도 확인 가능)_. 처음 배울 때는 '더러운' 컴퓨터에서 소량으로 연습하는 것도 괜찮습니다(일반 컴퓨터에 수년 동안 얼마나 많은 숨겨진 멀웨어가 축적되어 있는지 누가 알겠습니까? Bitcoin 지갑을 노출하고 싶지 않으시겠죠).


Electrum은 https://electrum.org/ 에서 다운로드하세요.


상단의 다운로드 탭을 클릭합니다.


자신의 컴퓨터에 해당하는 다운로드 링크를 클릭합니다. 모든 Linux 또는 Mac 컴퓨터는 Python 링크(빨간색 원)를 사용할 수 있습니다. 인텔 또는 AMD 칩이 탑재된 Linux 컴퓨터는 Appimage(Green 원, Windows 실행 파일과 유사)를 사용할 수 있습니다. 라즈베리 파이 디바이스에는 ARM 마이크로프로세서가 탑재되어 있으며, 파이가 Linux를 실행하더라도 Appimage가 아닌 Python 버전(빨간색 원)만 사용할 수 있습니다. 파란색 원은 Windows용이고 검은색 원은 Mac용입니다.

## 일렉트럼 확인


다운로드 '확인'의 목적은 단 한 비트의 데이터도 변조되지 않았는지 확인하는 것입니다. 이는 "해킹된" 악성 버전의 소프트웨어를 사용하는 것을 방지합니다. 다운로드한 사본을 연습용으로만 사용한다면, 즉 중요한 돈이 들어 있는 지갑을 사용하지 않는다면 이 단계를 건너뛰어도 괜찮습니다. 그런 다음 실제 자금으로 Electrum을 사용할 준비가 되면 사본을 삭제하고 다시 시작해야 하며, 이번에는 다운로드를 확인해야 합니다.


이를 위해 공개/개인 키 암호화 도구인 gpg를 사용하며, 이에 대한 가이드는 여기(https://armantheparman.com/gpg/)에서 확인할 수 있습니다. Gpg 도구는 모든 Linux 운영 체제와 함께 제공됩니다. Mac 및 Windows의 경우 gpg 링크에서 다운로드 지침을 참조하세요.


보안을 위해 Electrum 소프트웨어를 다운로드하는 것 외에도 소프트웨어의 디지털 서명이 필요합니다. 이는 개발자가 자신의 비공개 gpg 키로 생성한 텍스트 문자열(실제로는 텍스트를 사용하여 인코딩된 숫자)입니다. 그런 다음 gpg 프로그램을 사용하여 모든 사람이 액세스할 수 있는 공개 키(개발자의 개인 키로 생성)와 다운로드 파일에 대해 서명을 "테스트"할 수 있습니다.


즉, 세 가지 입력(서명, 공개 키, 데이터 파일)을 통해 파일이 변조되지 않았는지 확인하기 위해 참 또는 거짓 출력을 얻습니다.


서명을 받으려면 다운로드한 파일에 해당하는 링크를 클릭하세요(색 화살표 참조):


![image](assets/2.webp)


링크를 클릭하면 다운로드 폴더에 파일이 자동으로 다운로드되거나 브라우저에서 열릴 수 있습니다. 브라우저에서 열리면 파일을 저장해야 합니다. 마우스 오른쪽 버튼을 클릭하고 '다른 이름으로 저장'을 선택하면 됩니다. 운영 체제나 브라우저에 따라 텍스트가 아닌 공백 영역을 마우스 오른쪽 버튼으로 클릭해야 할 수도 있습니다.


아래는 다운로드한 텍스트의 모습입니다. 서명이 여러 개 있는 것을 볼 수 있는데, 이는 여러 사람이 서명한 것입니다. 각각 또는 모든 서명을 확인할 수 있습니다. 여기서는 개발자의 서명만 확인하는 방법을 보여드리겠습니다.


![image](assets/3.webp)


다음으로, 메인 개발자인 ThomasV의 공개키를 받아야 합니다. 이 키는 본인, 키베이스 계정, Github 또는 다른 사람, 키서버 또는 Electrum 웹사이트에서 직접 받을 수 있습니다.


왜냐하면 이 웹사이트가 악의적인 웹사이트(우리가 확인하는 바로 그 웹사이트)라면 왜 거기서 공개키를 받는지(공개키가 가짜일 수 있습니다), 사실 가장 안전하지 않은 방법이기 때문입니다


지금은 간단하게 설명하기 위해 웹사이트에서 다운로드하는 방법을 보여드리겠지만 이 점을 염두에 두세요. 다음은 GitHub에 있는 사본(https://github.com/spesmilo/electrum/blob/master/pubkeys/ThomasV.asc )과 비교할 수 있습니다.


페이지를 조금 아래로 스크롤하면 ThomasV의 공개 키 링크(아래 빨간색 원)를 찾을 수 있습니다. 이를 클릭하여 다운로드하거나 브라우저에서 텍스트가 열리면 마우스 오른쪽 버튼으로 클릭하여 저장합니다.


![image](assets/4.webp)


이제 다운로드 폴더에 3개의 새 파일이 생겼을 것입니다. 파일이 어디에 있든 상관없지만 모두 같은 폴더에 넣으면 프로세스가 더 쉬워집니다.


세 개의 파일입니다:


1. 일렉트럼 다운로드

2. 서명 파일(일반적으로 일렉트럼 다운로드와 동일한 파일 이름에 ".asc"가 추가된 파일 이름입니다

3. ThomasV의 공개 키입니다.


Mac 또는 Linux에서는 터미널을, Windows에서는 명령 프롬프트(CMD)를 엽니다.


다운로드 디렉토리(또는 세 개의 파일을 저장한 위치)로 이동합니다. 이것이 무엇을 의미하는지 잘 모르겠다면 이 짧은 Linux/Mac용 동영상(https://www.youtube.com/watch?v=AO0jzD1hpXc)과 이 Windows용 동영상(https://www.youtube.com/watch?v=9zMWXD-xoxc)을 참조하세요. Linux 컴퓨터에서는 디렉터리 이름이 대소문자를 구분한다는 점을 기억하세요.


터미널에서 이것을 입력해 ThomasV의 공개 키를 컴퓨터의 '키링'으로 가져옵니다(키링은 추상적인 개념으로, 실제로는 컴퓨터에 있는 파일에 불과합니다):


```bash
gpg --import ThomasV.asc
```


파일 이름이 다운로드한 파일과 일치하는지 확인하세요. 또한 단일 대시가 아닌 이중 대시인지 확인하세요. 또한 "-import" 앞뒤에 공백이 있다는 점에 유의하세요. 그런 다음 <입력>을 누릅니다.


파일을 가져와야 합니다. 오류가 발생하면 파일이 실제로 존재하는 디렉터리에 있는지 확인하세요. 현재 디렉터리를 확인하려면(Mac 또는 Linux의 경우) pwd를 입력합니다. 현재 디렉터리에 어떤 파일이 있는지 확인하려면(Mac 또는 Linux의 경우) ls를 입력합니다. 여러 파일 중 "ThomasV.asc" 텍스트 파일이 나열되어 있을 것입니다.


그런 다음 명령을 실행하여 서명을 확인합니다.


```bash
gpg –verify Electrum-4.1.5.tar.gz.asc Electrum-4.1.5.tar.gz
```


여기에는 4개의 "Elements"가 공백으로 구분되어 있습니다. 쉽게 볼 수 있도록 텍스트를 굵게 표시했습니다. 4개의 Elements는 다음과 같습니다:


1. gPG 프로그램

2. 확인 옵션

3. 서명의 파일 이름

4. 프로그램의 파일 이름


흥미롭게도 때때로 4번째 요소를 빼고 입력하면 컴퓨터가 사용자가 의미하는 바를 추측할 수 있습니다. 확실하지는 않지만 파일 이름이 끝에 "asc"만 다를 경우에만 작동한다고 생각합니다.


여기에 표시된 파일 이름을 그대로 복사하지 말고 시스템에 있는 파일 이름과 일치하는지 확인하세요.


<입력> 키를 눌러 명령을 실행합니다. "ThomasV의 서명이 양호합니다"라는 메시지가 표시되면 성공입니다. 서명 파일에 포함된 다른 사람의 서명에 대한 공개 키가 없기 때문에 약간의 오류가 있을 수 있습니다(하나의 파일에 서명을 결합하는 이 시스템은 이후 버전에서 변경될 수 있습니다). 또한, 하단에 무시해도 되는 경고가 있습니다(이는 컴퓨터에 ThomasV 공개키를 신뢰한다고 명시적으로 말하지 않았음을 알려주는 경고입니다).


이제 안전하게 사용할 수 있는 Electrum의 검증된 사본을 확보했습니다.


## 일렉트럼 실행


### Python을 사용하는 경우 Electrum 실행


Python 버전을 다운로드한 경우 다음과 같이 작동하는 방법입니다. 다운로드 페이지에서 이 내용을 확인할 수 있습니다:


![image](assets/5.webp)


Linux의 경우 먼저 시스템을 업데이트하는 것이 좋습니다:


```bash
sudo apt-get update
sudo apt-get upgrade
```


강조 표시된 노란색 텍스트를 복사하여 터미널에 붙여넣고 <입력>을 누릅니다. 비밀번호를 묻는 메시지가 표시되고 계속하려면 확인 메시지가 표시된 다음 해당 파일("종속성")이 설치됩니다.


또한 압축된 파일을 원하는 디렉터리에 압축을 풀어야 합니다. 그래픽 사용자 Interface 또는 명령줄(분홍색 강조 표시된 명령어)에서 이 작업을 수행할 수 있으며, 파일 이름이 다를 수 있다는 점을 기억하세요. (이전 섹션에서 다운로드를 확인했을 때는 압축이 풀린 디렉터리가 아닌 압축 파일을 확인했습니다.)


PIP 프로그램을 사용하여 "설치"하는 옵션이 있지만 이는 불필요하며 추가 단계와 파일 설치가 추가됩니다. 터미널을 사용하여 프로그램을 실행하면 이 모든 과정을 생략할 수 있습니다.


단계(Linux)는 다음과 같습니다:


1. 파일이 추출된 디렉토리로 이동합니다

2. 유형: ./run_electrum


Mac의 경우 단계는 동일하지만 먼저 Python3을 설치한 다음 이 명령을 사용하여 실행해야 할 수 있습니다:


```bash
python3 ./run_electrum
```


일렉트럼이 실행 중이면 터미널 창이 열려 있는 상태로 유지됩니다. 터미널 창을 닫으면 Electrum 프로그램이 종료됩니다. 일렉트럼을 사용하는 동안에는 최소화하세요.


### 앱이미지로 일렉트럼 실행


이것은 조금 더 쉽지만 Windows 실행 파일만큼 쉽지는 않습니다. 실행 중인 Linux 버전에 따라 기본적으로 Appimage 파일은 시스템에서 실행을 허용하지 않도록 속성이 설정되어 있을 수 있습니다. 이를 변경해야 합니다. 앱이미지가 작동하는 경우 이 단계를 건너뛸 수 있습니다. 터미널을 사용하여 파일이 있는 위치로 이동한 다음 이 명령을 실행합니다:


```bash
sudo chmod ug+x Electrum-4.1.5-x86_64.AppImage
```


"sudo"는 수퍼유저 권한을 부여하고, "chmod"는 모드를 변경하여 읽기, 쓰기 또는 실행할 수 있는 사용자를 변경하는 명령이며, "ug+x"는 실행을 허용하도록 사용자 및 그룹을 수정한다는 의미입니다.


버전에 맞게 파일 이름을 조정합니다.


그런 다음 앱이미지 아이콘을 더블클릭하면 일렉트럼이 실행됩니다.


### Mac에서 Electrum 실행


다운로드한 파일('드라이브' 파일)을 두 번 클릭하기만 하면 됩니다. 창이 열립니다. 창에 있는 일렉트럼 아이콘을 바탕화면 또는 프로그램을 보관하고 싶은 곳으로 드래그합니다. 그런 다음 드라이브를 '꺼내기'하고 드라이브(다운로드한 파일)를 삭제할 수 있습니다.


프로그램을 실행하려면 더블클릭하세요. 우회해야 하는 일부 Mac 전용 '내니' 오류가 발생할 수 있습니다.


### Windows에서 Electrum 실행


저는 Windows를 가장 싫어하지만, 이 방법이 가장 간단한 방법입니다. 실행할 실행 파일을 두 번 클릭하기만 하면 됩니다.

**중요한 경고에 대한 간략한 설명: ** 체크섬의 목적은 입력 오류 없이 seed을 입력했는지 확인하는 것입니다. 체크섬은 seed의 마지막 부분(12번째 단어가 체크섬 단어가 됨)으로, 수학적으로 seed의 첫 번째 부분(11개 단어)에 의해 결정됩니다. 처음에 잘못된 단어를 입력하면 체크섬 단어가 수학적으로 일치하지 않으며 Wallet 소프트웨어가 경고를 표시합니다. 그렇다고 해서 seed을 사용하여 Bitcoin Wallet을 만들 수 없다는 뜻은 아닙니다. 입력 오류가 있는 Wallet을 생성하고 Wallet에 Bitcoin을 로드한 다음 언젠가 Wallet을 복원해야 하는데, 복원할 때 입력 오류를 다시 만들지 않고 잘못된 Wallet을 복원하게 된다고 상상해 보세요! 체크섬이 유효하지 않은 경우 일렉트럼에서 Wallet을 생성하도록 허용하는 것은 매우 위험하므로, 이를 확인하는 것은 사용자의 책임입니다. 다른 지갑은 진행을 허용하지 않으므로 훨씬 안전합니다. 이것이 바로 제가 일렉트럼을 제대로 사용하는 법을 배우면 사용하기에 괜찮다고 말하는 이유 중 하나입니다(일렉트럼 개발자가 이 문제를 해결해야 합니다).


passphrase를 추가하려는 경우 이 옵션 창에서 바로 상단에 해당 옵션을 선택할 수 있는 기회가 있습니다.


확인을 클릭하면 seed 문구를 입력한 위치로 돌아갑니다. passphrase 옵션을 선택한 경우 seed 단어와 함께 입력하지 않습니다(해당 메시지가 다음에 표시됩니다).


passphrase를 요청하지 않은 경우 다음 화면이 표시되며, Wallet 스크립트 유형 및 파생 경로에 대한 추가 옵션은 여기(https://armantheparman.com/public-and-private-keys/)에서 확인할 수 있지만 기본값을 그대로 두고 계속 진행하세요.


![image](assets/13.webp)


**추가 정보** 첫 번째 옵션은 다음 중에서 선택할 수 있습니다:



- 레거시("1"로 시작하는 주소),
- Pay-to-Script-Hash("3"으로 시작하는 주소),
- bech32/네이티브 SegWit("bc1q"로 시작하는 주소).


이 글을 쓰는 시점에서 Electrum은 아직 Taproot("bc1p"로 시작하는 주소)을 지원하지 않습니다. 이 창의 두 번째 옵션에서는 파생 경로를 수정할 수 있습니다. 특히 이 옵션의 의미를 이해하기 전에는 절대 수정하지 않는 것이 좋습니다. 사람들은 필요한 경우 Wallet을 복구할 수 있도록 파생 경로를 기록하는 것이 중요하다고 강조하지만, 기본값으로 두어도 괜찮을 테니 당황하지 마시고 파생 경로를 기록해 두는 것이 좋은 습관입니다.


다음으로 비밀번호를 추가할 수 있는 옵션이 제공됩니다. 이것은 "passphrase"과 혼동하지 마세요. 비밀번호는 컴퓨터의 파일을 잠급니다. passphrase은 개인 키 구성의 일부입니다. 이것은 더미 Wallet이므로 비밀번호를 비워두고 계속 진행할 수 있습니다.

### 원서버 사용


기본적으로 일렉트럼은 임의의 노드에 연결합니다. 또한 다른 많은 보조 노드에도 연결됩니다. 이러한 보조 노드와 어떤 데이터가 교환되는지는 잘 모르겠지만, 개인 정보 보호를 위해 이러한 데이터 교환을 원하지 않습니다. 내 노드와 같은 노드를 지정하더라도 다른 여러 노드도 연결되며 어떤 정보가 공유되는지 잘 모르겠습니다. 어쨌든 쉽게 방지할 수 있습니다. 자신만의 노드를 지정하는 방법을 보여드리기 전에 일렉트럼이 한 번에 하나의 서버에만 연결하도록 강제 설정하겠습니다.


**참고: 명령줄에서 "oneserver"를 지정하여 이 작업을 수행하는 방법이 있지만 이 방법은 권장하지 않습니다. 장기적으로 볼 때 더 쉽고 실수로 다른 노드에 연결되지 않을 가능성이 더 높은 대안을 보여드리겠습니다.


더미 Wallet을 사용하는 이유는 실제 Bitcoin에 실제 Wallet을 로드했다면 지금쯤 실수로 임의의 노드에 연결되었을 것이기 때문입니다(처음에 "서버 수동 설정"을 선택했더라도 어떤 이유로 다른 보조 노드에 연결됩니다(Electrum 개발자들, 이 문제를 해결해야 합니다). Wallet이 비공개였다면 큰일 날 뻔했습니다.


또한 아래에서 보여드리는 단계는 먼저 Wallet를 로드하지 않고는 수행할 수 없습니다. (Wallet가 로드되면 편집할 수 있는 구성 파일이 채워지고 편집할 준비가 됩니다).


이제 Electrum을 종료하세요 **(중요: 이 작업을 수행하지 않으면 다음 변경 사항이 지워집니다)**.


### LINUX/MAC 구성 파일


Linux 또는 Mac에서 터미널을 엽니다(Windows 지침은 나중에 안내):


자동으로 홈 폴더로 이동합니다. 거기에서 숨겨진 일렉트럼 설정 폴더로 이동합니다(애플리케이션이 있는 위치와 다릅니다).


```bash
cd .electrum
```


'electrum' 앞에 있는 점은 숨겨진 폴더임을 나타냅니다.


또 다른 방법은 입력하는 것입니다:


```bash
cd ~/.electrum
```


여기서 "~"는 홈 디렉터리 경로를 나타냅니다. "pwd" 명령으로 현재 디렉터리의 전체 경로를 확인할 수 있습니다.


".electrum" 디렉토리에 들어가면 "nano config"를 입력하고 <enter>를 누릅니다.


구성 파일이 열린 상태에서 텍스트 편집기(나노라고 함)가 열립니다. 여기서는 마우스가 잘 작동하지 않습니다. 화살표 키를 사용하여 다음과 같은 줄로 이동합니다:


```json
"oneserver": false,
```


"false"를 "true"로 변경하고 구문을 어지럽히지 않도록 합니다(쉼표나 세미콜론을 삭제하지 마세요).

1. 친구의 개인 노드에 연결 (토르 필요)

2. 신뢰할 수 있는 회사의 노드에 연결하기

비타루의 노드에 연결해 봅시다 - 데이터를 수집하지 않는다고 합니다. 열정적인 비트코인이 운영하는 Bitcoin 전용 Exchange입니다. 이 노드에 연결하려면 약간의 신뢰가 필요하지만 감시 회사일 수 있는 임의의 노드에 연결하는 것보다는 낫습니다.

### 내 노드에 연결


자체 노드가 있다면 다행입니다. 일렉트럼 서버가 아닌 Bitcoin core만 보유하고 있는 경우에는 아직 일렉트럼 Wallet를 노드에 연결할 수 없습니다.


**Electrum 서버와 Electrum Wallet은 다른 것임에 유의하세요: ** 서버는 Electrum Wallet이 Bitcoin과 통신하기 위해 필요한 소프트웨어입니다 - 왜 이런 식으로 설계되었는지 모르겠지만 아마도 속도 때문일 수 있지만 제가 틀릴 수도 있습니다.


MyNode(처음 시작하는 분들에게 추천하는 패키지), Raspiblitz(고급으로 갈수록 추천하는 패키지) 또는 Umbrel(개인적으로 너무 많은 문제를 경험했기 때문에 아직 추천하지 않음)과 같은 노드 소프트웨어 패키지를 실행하는 경우 이전 섹션의 이미지와 같이 노드를 실행하는 컴퓨터(라즈베리파이)의 IP Address와 콜론, 50002를 입력하기만 하면 Wallet을 연결할 수 있습니다. (아래에서 노드의 IP Address를 찾는 방법을 자세히 설명해 드리겠습니다.)


네트워크 설정을 엽니다(오른쪽 하단의 Green 또는 빨간색 원 클릭). "서버 자동 선택" 확인란을 선택 취소한 다음 IP Address을 입력합니다. 여러분의 IP는 다르지만 콜론과 "50002"는 동일해야 합니다.

## 노드 IP 찾기 Address


IP Address는 일반 사용자가 조회하고 사용하는 방법을 일반적으로 알고 있는 것이 아닙니다. 저는 많은 사람이 노드를 실행하고 지갑을 노드에 연결하는 것을 도왔는데, 걸림돌이 되는 것은 종종 IP Address를 찾는 것 같았습니다.


마이노드의 경우 브라우저 창에 입력할 수 있습니다: `mynode.local`


간혹 'mynode.local'이 작동하지 않는 경우가 있습니다(Google 검색창에 입력하지 않았는지 확인하세요). 탐색 모음에서 텍스트를 검색이 아닌 Address로 인식하도록 하려면 다음과 같이 텍스트 앞에 `http://`을 추가합니다: `http://mynode.local`. 그래도 문제가 해결되지 않으면 다음과 같이 's'를 추가해 보세요: `https://mynode.local`.


그러면 장치에 액세스하고 설정 링크(아래 파란색 '원' 참조)를 클릭하면 IP Address이 위치한 이 화면이 표시됩니다:


![image](assets/19.webp)


이 페이지가 로드되고 노드의 IP(파란색 "원")가 표시됩니다


![image](assets/20.webp)


그런 다음 나중에 브라우저에 192.168.0.150 또는 http://192.168.0.150 을 입력하면 됩니다.


라스피블리츠(화면을 연결하지 않은 경우)의 경우 다른 방법이 필요합니다(마이노드에서도 작동함):


라우터 웹페이지에 로그인하세요 - 여기에서 연결된 모든 디바이스의 IP Address을 찾을 수 있습니다. 라우터의 웹 페이지는 웹 브라우저에 입력한 IP Address이 됩니다. 제 모습은 이렇습니다:


http://192.168.0.1


라우터 로그인 자격 증명은 사용자 설명서에서 확인하거나 라우터 자체의 스티커에서 찾을 수 있습니다. 사용자 이름과 비밀번호를 찾습니다. 찾을 수 없는 경우 사용자: "admin" 비밀번호: "password"


로그인할 수 있으면 연결된 디바이스가 표시되고 이름에서 어떤 노드인지 알 수 있습니다. IP Address이 표시됩니다.


### 처음 두 가지 방법이 실패하면 마지막 방법이 작동하지만 지루합니다:


먼저 네트워크에 있는 모든 디바이스의 IP Address를 찾습니다(현재 컴퓨터가 가능합니다).


Mac에서는 네트워크 환경설정에서 찾을 수 있습니다:


![image](assets/21.webp)


우리는 이미지에 표시된 4번째 요소인 "166"이 아닌 첫 번째 4 Elements(192.168.0)에 관심이 있습니다(귀하의 것은 다를 수 있습니다).


Linux의 경우 명령줄을 사용합니다:


```bash
ifconfig | grep inet
```


세로줄은 "파이프" 기호이며 <삭제> 키 아래에서 찾을 수 있습니다. 일부 출력과 IP Address이 표시됩니다. (127.0.0.1은 무시하고 넷마스크는 무시하세요.)


Windows의 경우 명령 프롬프트(cmd)를 열고 입력합니다:


```bash
ipconfig/all
```


를 입력하고 Enter 키를 누릅니다. 출력에서 IP Address를 찾을 수 있습니다.


이는 쉬운 부분입니다. 이제 Hard 부분은 노드의 IP Address을 찾는 것으로, 무차별 대입을 통해 추측해야 합니다. 예를 들어 컴퓨터의 IP Address이 192.168.0.xxx로 시작한다고 가정하고 노드에 대해 브라우저에서 다음을 입력해 보세요: `https://192.168.0.2`


가능한 가장 작은 숫자는 2(0은 모든 장치를 의미하고 1은 라우터에 속함)이고 가장 큰 숫자는 255(1바이트로는 11111111, 1바이트가 보유한 가장 큰 숫자)입니다.


하나씩 하나씩 255를 향해 올라가세요. 결국 마이노드 페이지(또는 RaspiBlitz 페이지)를 로드하는 올바른 번호에서 멈추게 됩니다. 그러면 일렉트럼 네트워크 설정에서 노드에 연결하기 위해 어떤 번호를 입력해야 하는지 알 수 있습니다.

## Wallet 테스트(사용 방법 알아보기)


여기서는 UTXO을 Wallet로 받은 다음 Wallet 내에서 다른 Address로 이동(사용)하는 방법을 보여드리겠습니다. 이것은 손실 위험을 감수해도 괜찮을 정도로 아주 적은 금액입니다.


여기에는 여러 가지 목적이 있습니다.



- 새로운 Wallet에서 코인을 사용할 수 있는 힘이 있다는 것을 증명할 것입니다.
- 안전을 위해 더 복잡하게 추가하기 전에 Electrum 소프트웨어를 사용하여 지출하는 방법(및 일부 기능)을 시연합니다(Hardware Wallet 또는 에어 갭 컴퓨터 사용)
- 동일한 Wallet 내에서 수신과 지출을 위해 선택할 수 있는 주소가 많다는 점을 강조할 수 있습니다.


테스트용 일렉트럼 Wallet을 열고 주소 탭을 클릭한 다음 첫 번째 Address을 마우스 오른쪽 버튼으로 클릭하고 복사 -> Address을 선택합니다:


![image](assets/30.webp)


이제 Address가 컴퓨터 메모리에 저장되었습니다.


이제 Bitcoin가 있는 Exchange으로 가서 이 Address로 소량, 예를 들어 50,000 Sats을 인출해 봅시다. 코인베이스는 Bitcoin의 적이지만 가장 일반적으로 사용되는 Exchange이기 때문에 예를 들어 사용하려고 하며, 이러한 목적으로 오래된 버려진 계정에 로그인하는 것은 역겨운 일입니다.


로그인한 후 보내기/받기 버튼을 클릭합니다(현재 웹페이지 오른쪽 상단에 있음).


![image](assets/31.webp)


코인베이스에 자금이 없는 것은 분명하지만 여기에 자금이 있다고 상상하고 따라하세요: 제가 했던 것처럼 '받는 사람' 필드에 Electrum의 Address를 붙여넣습니다. 또한 금액을 선택해야 합니다(50,000 Sats 정도를 추천합니다). "선택 메시지"를 입력하지 마세요. 코인베이스에서 충분한 데이터를 수집(및 판매)하고 있으므로 도와줄 필요가 없습니다. 마지막으로 "계속"을 클릭합니다. 그 후에 어떤 다른 팝업이 표시될지 모르겠지만 모든 거래소에서 방법은 비슷합니다.


![image](assets/32.webp)


Exchange에 따라 Wallet에 Sats이 즉시 표시되거나 몇 시간/일 후에 표시될 수도 있습니다.


일렉트럼은 Blockchain에서 코인이 확인되지 않은 경우에도 받은 코인을 표시합니다. 보유하고 있는 코인은 Bitcoin 노드의 대기 목록, 즉 "Mempool"에서 읽혀지고 있습니다. 블록에 올라가면 자금이 확정된 것으로 표시됩니다.


이제 Wallet에 UTXO가 있으므로 라벨을 붙여야 합니다. 이 라벨은 우리만 볼 수 있으며 공용 Ledger와는 아무 관련이 없습니다. 모든 일렉트럼 라벨은 우리가 사용 중인 컴퓨터에서만 볼 수 있습니다. 실제로 파일을 저장하고 이를 사용하여 동일한 Wallet을 실행하는 다른 컴퓨터로 모든 라벨을 복원할 수 있습니다.

이상적으로는 동일한 Address에 여러 개의 코인을 쌓아두지 않는 것이 좋습니다. 그렇게 하기로 결정했다면(하지 않기로 결정했다면) Address 방법을 사용하여 모든 Coin에 동일한 라벨을 붙이는 대신 각 Address에 라벨을 붙일 수 있습니다. 실제로 "코인" 탭으로 이동하여 레이블을 편집할 수는 없습니다(너무 직관적이지 않아요!). 거래 내역 탭으로 이동하여 거래를 찾아 라벨을 지정한 다음 Coin 섹션에 라벨을 표시해야 합니다. 코인 섹션에 표시되는 레이블은 Address 레이블 또는 내역 레이블에서 가져온 것이지만, 내역 레이블은 Address 레이블보다 우선합니다. 라벨을 파일로 백업하려면 상단 메뉴의 Wallet->레이블->내보내기에서 내보낼 수 있습니다.

![image](assets/40.webp)


두 개의 내부 흰색 상자/창이 보입니다. 위쪽은 입력 창(코인이 사용되는 곳)이고 아래쪽은 출력 창(코인이 이동하는 곳)입니다.


현재 상태(왼쪽 상단)는 "서명되지 않음"입니다. 코인이 Wallet 내에서 전송되고 있기 때문에 "전송된 금액"은 0입니다. 수수료는 481 Sats입니다. 480 Sats이라면 마지막 0이 0.0000048처럼 떨어져서 피곤한 눈에는 48 Sats처럼 보일 수 있으니 주의하세요(일렉트럼 개발자가 수정해야 할 사항).


트랜잭션의 크기는 바이트 단위의 데이터 크기를 의미하며, Bitcoin의 양을 의미하지는 않습니다. "Replace by fee"은 기본적으로 켜져 있으며, 필요한 경우 더 높은 수수료로 트랜잭션을 다시 전송할 수 있습니다. 잠금 시간을 사용하면 트랜잭션이 유효한 시간을 조정할 수 있습니다. 아직 사용해 보지는 않았지만, 이 기능을 완전히 이해하고 소량으로 연습해 본 적이 없다면 사용하지 않는 것이 좋습니다.


하단에는 멋진 Mining 수수료 조정 도구가 있습니다. 내부 전송의 경우 최소 수수료인 1샛/바이트로 설정하기만 하면 됩니다. 대상 수수료 필드에 숫자를 수동으로 입력하기만 하면 됩니다. 외부 결제에 대한 적절한 수수료를 확인하려면 https://Mempool.space 을 참조하여 Mempool의 사용량을 확인하면 몇 가지 권장 수수료가 표시됩니다.


![image](assets/41.webp)


1 sat/바이트를 선택했습니다.


입력 창에 두 개의 항목이 표시됩니다. 첫 번째는 5000 SAT 기부입니다. 왼쪽에 트랜잭션 Hash가 표시됩니다(Blockchain에서 조회할 수 있음). 그 옆에는 "21"이 있는데, 이는 해당 트랜잭션에서 21로 레이블이 지정된 출력임을 나타냅니다(첫 번째는 0으로 레이블이 지정되었으므로 실제로는 22번째 출력입니다).


여기서 주의할 점이 있습니다: UTXO는 트랜잭션 내부에만 존재합니다. UTXO을 사용하려면 이를 참조하고 새 트랜잭션의 입력에 해당 참조를 넣어야 합니다. 그러면 출력은 새로운 UTXO가 되고 이전 UTXO은 STXO(소비된 트랜잭션 출력)가 됩니다.


두 번째 줄은 10,000 Sat 기부금입니다. 해당 트랜잭션의 첫 번째(그리고 아마도 유일한) 출력이기 때문에 트랜잭션 Hash 옆에 "0"이 표시됩니다.


아래쪽 창에 Address가 표시됩니다. Bitcoin의 입력 총합이 출력 총합과 일치하지 않는 것을 알 수 있습니다. 그 차이는 Miner로 이동합니다. Miner은 채굴하려는 블록에 있는 모든 트랜잭션의 불일치를 살펴보고 그 수치를 보상에 더합니다. (이러한 방식으로 Mining 수수료는 거래 체인에서 완전히 분리되어 새로운 삶을 시작합니다).


Mining 수수료를 조정하면 출력 값이 자동으로 변경됩니다.


**거래 창에 표시되는 주소의 색상에 주목하세요. Green 주소는 Address 탭에 나열된다는 점을 기억하세요. 거래 창에서 Address이 Green(또는 노란색)로 강조 표시되어 있으면 Electrum이 Address을 자체 주소로 인식한 것입니다. Address에 하이라이트가 없는 경우 외부 Address(현재 열려 있는 Wallet의 외부)이므로 각별히 주의하여 확인해야 합니다.


거래의 모든 사항을 확인하고 어떤 코인을 사용하고 있는지, 코인이 어디로 가는지 만족스럽다면 "완료"를 클릭할 수 있습니다


![image](assets/42.webp)


"완료"를 클릭하면 더 이상 수정할 수 없으며, 수정이 필요한 경우 이 버튼을 닫고 다시 시작해야 합니다. '마무리' 버튼이 '내보내기'로 변경되고 새로운 버튼이 나타난 것을 확인할 수 있습니다: "저장", "결합", "서명" 및 "방송" 버튼이 새로 생겼습니다. "브로드캐스트" 버튼은 트랜잭션이 서명되지 않았으므로 이 단계에서는 유효하지 않으므로 회색으로 표시되어 있습니다.


서명을 클릭하면 Wallet의 비밀번호가 있는 경우 비밀번호를 입력하라는 메시지가 표시되고 상태(오른쪽 상단)가 "서명되지 않음"에서 "서명됨"으로 변경됩니다. 그러면 "생방송" 버튼을 사용할 수 있습니다.


생방송을 한 후에는 트랜잭션 창을 닫을 수 있습니다. Address 탭으로 이동하면 이제 첫 번째 Address는 비어 있고 두 번째 Address에는 UTXO가 1개 있는 것을 볼 수 있습니다.


참고: 트랜잭션이 블록으로 채굴되거나 "확정"되기 전에도 이러한 모든 변경 사항을 볼 수 있습니다. 이는 일렉트럼이 Blockchain 데이터뿐만 아니라 Mempool 데이터도 기반으로 잔액/거래를 업데이트하기 때문입니다. 모든 지갑이 이렇게 하는 것은 아닙니다.


한 가지 주목할 점은 브로드캐스팅하는 대신 트랜잭션을 나중에 저장할 수 있다는 점입니다. 서명되지 않은 상태 또는 서명된 상태로 저장할 수 있습니다.

이제 다시 '저장' 버튼으로 돌아와서, 이것은 거래 텍스트를 저장하는 방법이 아닙니다. 실제로 이 버튼이 하는 일은 일렉트럼 Wallet이 로컬 컴퓨터에서 이 거래를 결제 제출로 인식하도록 지시하는 것입니다. 실수로 그렇게 하면 작은 컴퓨터 아이콘과 함께 트랜잭션이 표시됩니다. 트랜잭션을 마우스 오른쪽 버튼으로 클릭하여 삭제할 수 있습니다. 이 방법으로 Bitcoin을 삭제할 수는 없으니 걱정하지 마세요. 그러면 Electrum은 이 거래가 발생한 사실을 잊고 Sats를 다시 '환불'하고 실제 존재하는 올바른 위치에 Sats를 표시합니다.


### 주소 변경


주소 변경은 흥미롭습니다. 이 설명을 이해하려면 UTXO를 이해해야 합니다. UTXO보다 작은 금액을 Address으로 지출하는 경우 변경 출력을 지정하지 않는 한 남은 Bitcoin는 Miner로 이동합니다.


6.15 Bitcoin UTXO을 가지고 있는데, 전 세계 어딘가에서 폭압적인 '민주' 정부에 의해 억압받는 시위대를 위해 0.15 Bitcoin을 기부하고 싶다고 가정해 보겠습니다. 그런 다음 Electrum의 "보내기" 기능을 사용하여 6.15 Bitcoin을 트랜잭션에 넣을 수 있습니다.


'지불 대상' 필드에 시위대의 Address를 붙여넣고 '설명' 필드에 'EndTheFed & WEF'를 입력한 후 금액은 0.15Bitcoin을 입력하고 '지불'을 클릭한 다음 '진행'을 클릭합니다.


트랜잭션 화면에서 입력 창에는 6.15 Bitcoin UTXO가 표시됩니다. 출력 창에는 하이라이트가 없는 Address(시위대의 Address)이 표시되고 그 옆에 0.15 Bitcoin가 표시됩니다. 또한 6.0 Bitcoin보다 약간 낮은 노란색 Address도 표시됩니다. 이것은 Wallet이 노란색 변경 주소 중 하나에서 자동으로 선택한 변경 Address입니다. 변경 Address의 목적은 Wallet이 계획 중이거나 송장을 보낸 수신 주소의 가용성을 방해하지 않고 거스름돈을 넣을 수 있도록 하기 위한 것입니다. 예를 들어 고객이 나중에 사용할 경우 Wallet이 자동으로 동전을 사용하여 동전을 채우는 것을 원하지 않을 것입니다. 지저분하고 개인정보 보호에 좋지 않으니까요.


Mining 수수료를 조정하면 결제 금액이 아니라 변경 출력 금액이 자동으로 조정된다는 점에 유의하세요.

## 지갑


일렉트럼을 사용하여 Wallet만 보기를 시연하고 싶습니다. 그러기 위해서는 먼저 "Wallet"를 정의해야 합니다. Bitcoin에서 "Wallet"를 사용하는 방법에는 두 가지가 있습니다:



- 유형 A, "Wallet" - 주소와 잔액을 표시하는 소프트웨어(예: 일렉트럼, 블루 Wallet, Sparrow wallet 등)를 말합니다.



- 유형 B, "Wallet"은 시드_문구/passphrase/유래_경로의 조합과 연관된 고유한 주소 모음을 나타냅니다. Wallet에는 86억 개의 주소가 있습니다(수신 주소 43억 개, 변경 주소 43억 개). seed 구문, passphrase 또는 파생 경로에서 아무것도 변경하지 않으면 사용되지 않는 Wallet에 새롭고 모두 고유한 86억 개의 빈 주소가 생깁니다.


"Wallet"이라는 단어를 사용할 때 어떤 유형을 가리키는지는 문맥상 명확합니다.

그렇다면 그 이유는 무엇일까요?


주된 이유는 아니지만 한 가지 이유는 컴퓨터의 멀웨어에 개인 키를 노출하지 않고도 컴퓨터에서 Wallet와 그 잔액을 잠재적으로 관찰할 수 있다는 점입니다.


다른 하나는 개인 키를 컴퓨터에 보관하지 않기로 선택한 경우 결제를 위해 반드시 필요하다는 점입니다:


**하드웨어 지갑(HWW)**은 기기가 개인 키를 안전하게 보관하고(PIN으로 잠김), 컴퓨터에 키를 노출하지 않으며(케이블로 컴퓨터에 연결되어 있어도), 그 자체로 인터넷에 연결할 수 없도록 만들어졌습니다. 모든 Bitcoin 트랜잭션은 노드에 있는 Blockchain의 UTXO을 참조하여 시작하기 때문에 이러한 장치는 자체적으로 트랜잭션을 수행할 수 없습니다. Wallet는 UTXO이 어느 transaction ID에 있는지, 트랜잭션의 어떤 출력이 소비될 것인지 지정해야 합니다. 입력을 지정한 후에야 서명은 물론이고 새로운 트랜잭션이 처음에 생성될 수 있습니다. 하드웨어 지갑은 UTXO에 액세스할 수 없기 때문에 트랜잭션을 생성할 수 없으며, 어떤 것과도 연결되지 않습니다!


확장된 공개 키는 일반적으로 HWW에서 추출되며, 주소는 컴퓨터에 표시됩니다 - 많은 분이 컴퓨터에서 주소와 잔액을 보여주는 Ledger 소프트웨어 또는 Trezor Suite에 익숙하실 텐데요, 이는 Wallet을 보는 것입니다. 이러한 프로그램은 트랜잭션을 생성할 수는 있지만 서명할 수는 없습니다. 연결된 HWW에 의해서만 트랜잭션에 서명할 수 있습니다. HWW는 감시 중인 Wallet에서 새로 생성된 트랜잭션을 가져와서 서명한 다음 노드로 브로드캐스팅하기 위해 컴퓨터로 다시 보냅니다. **HWW는 자체적으로 브로드캐스팅을 할 수 없으며, 연결된 워칭 Wallet이 이를 수행합니다. 이러한 방식으로 두 지갑(컴퓨터의 공개 키 Wallet과 HWW의 개인 키 Wallet)이 협력하여 generate에 서명하고 방송하며, 개인 키가 인터넷에 연결된 장치에서 격리된 채로 유지되도록 합니다.


## 부분적으로 서명된 Bitcoin 거래(PSBT)


트랜잭션을 생성하여 파일에 저장하고, 나중에 다시 로드하고, 서명하고, 저장하고, 나중에 다시 로드한 다음 마지막으로 브로드캐스트하는 것은 누구나 할 수 있는 일이며, 단지 가능한 일입니다.


이제 5개 중 3개의 다중 서명 Wallet를 예로 들어보겠습니다. 5개의 개인 키가 Wallet를 생성하고 트랜잭션에 완전히 서명하려면 3개가 필요합니다(다중 서명 Wallet 키에 대해 자세히 알아보려면 여기를 참조하세요). 5개의 개인 키 중 하나를 가지고 있는 5개의 서로 다른 컴퓨터를 각각 5대씩 사용할 수 있습니다.


컴퓨터 1은 트랜잭션을 generate하고 서명할 수 있습니다. 그런 다음 파일에 저장하고 이메일로 컴퓨터 2로 보낼 수 있습니다. 그러면 컴퓨터 2는 서명하고 파일을 QR 코드에 저장한 후 지구 반대편에 있는 컴퓨터 3에게 Zoom 통화를 통해 QR을 전송할 수 있습니다. 컴퓨터 3은 QR을 캡처하고 트랜잭션을 일렉트럼에 로드한 후 트랜잭션에 서명할 수 있습니다. 처음 두 번의 서명 후 트랜잭션은 PSBT(Partially Signed Bitcoin Transaction)이 되었습니다. 세 번째 서명이 끝나면 거래는 완전히 서명되고 유효하며 방송할 준비가 되었습니다.


PSBTS에 대해 자세히 알아보려면 이 가이드를 참조하세요. (https://armantheparman.com/PSBT/)

### 콜드카드를 사용한 거래


케이블을 연결하면 거래가 쉬워집니다. 거래에 원활하게 서명할 수 있습니다.


에어 갭 방식으로 디바이스를 사용하는 경우, 마이크로SD 카드를 사용하여 디바이스 간에 저장된 트랜잭션을 수동으로 전달해야 합니다. 몇 가지 요령이 있습니다.


거래를 생성하고 완료한 후에는 왼쪽 하단에 있는 내보내기 버튼을 클릭해야 합니다. "파일에 저장"이라고 표시되는데, 이는 직관적으로 볼 때 저희가 원하는 것이 아닙니다. 실제로는 맨 마지막 메뉴 옵션인 "하드웨어 지갑용"으로 이동한 다음, 그 선택 항목에서 다른 "파일에 저장"을 찾아서 선택해야 합니다. 그런 다음 파일을 microSD에 저장하고 카드를 꺼내서 콜드카드에 넣습니다. 올바른 Wallet를 선택하려면 passphrase을 적용해야 할 수도 있다는 점을 기억하세요. 화면에 서명할 준비가 되었다는 메시지가 표시됩니다. 확인 표시를 클릭하고 거래를 검사한 다음 확인 표시로 확인하여 계속 진행합니다. 완료되면 카드를 꺼내서 컴퓨터에 다시 넣습니다.


그런 다음 일렉트럼을 사용하여 트랜잭션을 열어야 합니다. 이 기능은 메뉴 도구 -> 트랜잭션 로드에 숨겨져 있습니다. 파일 시스템을 탐색하고 파일을 찾습니다. 서명할 때마다 세 개의 파일이 있습니다. 워칭 Wallet이 만든 원본 저장 파일과 콜드카드가 만든 두 개의 파일입니다(왜 이렇게 하는지 모르겠습니다). 하나는 "서명됨"이라고 표시되고 하나는 "최종"이라고 표시됩니다. 직관적이지는 않지만 "서명된" 파일은 유용하지 않으므로 "최종" 트랜잭션을 열어야 합니다.


이를 로드한 후 '생방송'을 클릭하면 결제가 이루어집니다.


## 일렉트럼 및 숨겨진 ".electrum" 디렉토리 업데이트하기


Electrum은 컴퓨터의 두 곳에 존재합니다. 애플리케이션 자체와 숨겨진 구성 폴더가 있습니다. 이 폴더는 운영 체제에 따라 다른 위치에 있습니다:


Windows:


```bash
C:/Users/your_user_name_goes_here/AppData/Roaming/Electrum
```


Mac:


```bash
/Users/your_user_name_goes_here/.electrum
```


Linux:


```bash
/home/your_user_name_goes_here/.electrum
```


Linux와 Mac에서 `.` 앞에 `electrum`이 오는 것은 디렉터가 숨겨져 있음을 나타냅니다. 또한, 이 디렉터리는 일렉트럼을 처음 실행할 때만 (자동으로) 생성된다는 점에 유의하세요. 이 디렉터리에는 일렉트럼 구성 파일과 저장한 지갑이 있는 디렉터리도 포함됩니다.


일렉트럼 프로그램을 컴퓨터에서 삭제하면 해당 디렉터리도 적극적으로 삭제하지 않는 한 숨겨진 디렉터리는 그대로 유지됩니다.


일렉트럼을 업그레이드하려면 처음에 설명한 것과 동일한 절차를 거쳐 다운로드하고 확인합니다. 그러면 컴퓨터에 두 개의 프로그램 사본이 생성되며, 각 프로그램은 구성 및 Wallet 액세스를 위해 동일한 숨겨진 일렉트럼 폴더에 액세스할 수 있습니다. 기본 장치, 연결할 기본 노드, 기타 환경설정, 지갑 액세스 등 우리가 저장한 모든 항목은 해당 폴더에 저장되므로 그대로 유지됩니다.


### 일렉트럼과 지갑을 다른 컴퓨터로 옮기기


이렇게 하려면 프로그램 파일을 USB 드라이브에 복사하고 .electrum 디렉터리도 복사할 수 있습니다. 그런 다음 새 컴퓨터로 복사하거나 이동합니다. 프로그램을 다시 인증할 필요는 없습니다. .electrum 디렉터리를 기본 위치에 복사해야 합니다(해당 컴퓨터에서 Electrum을 처음 실행하기 전에 복사해야 합니다. 그렇지 않으면 빈 기본 .electrum 폴더가 채워져 혼동을 일으킬 수 있습니다).
