## 為何選擇 Electrum？


這是一份關於如何使用 Electrum Bitcoin Wallet 的詳細指南，其中包含了所有陷阱和怪癖的解決方案 - 這是我經過幾年的使用和教授學生關於 Bitcoin 安全/隱私方面的知識而開發的。Electrum 並不是最適合希望一切盡可能簡單，並希望保持在初學者水準的人使用的 Bitcoin Wallet。相反，它是為了那些 「強大 」的使用者，或渴望成為 「強大 」的使用者的人準備的。


對於新的 Bitcoiner 來說，只有在有經驗的用戶指導下使用才是最好的。如果單獨學習使用，只要他們慢慢來，一開始只在有少量 Sats 的測試環境中使用，就會很安全。本指南支持這種努力，但對其他人來說也是很好的參考。


**警告：** 本指南篇幅很大。不要嘗試在一天內完成所有工作。最好先儲存這份指南，然後慢慢來。

## 驗證 Electrum


驗證 "下載的目的是確保沒有一丁點資料被篡改。它可以防止您使用 「黑客 」惡意版本的軟體。如果您只將下載的副本用於練習，即不使用持有大量金錢的錢包，則可以跳過此步驟。然後，當您準備好使用 Electrum 儲存您的真實資金時，您應該刪除您的拷貝並重新開始，這一次要驗證您的下載。


為此，我們使用公鑰/私鑰加密工具 - gpg，我們已在此撰寫指南 (https://armantheparman.com/gpg/)。gpg 工具隨附於所有 Linux 作業系統。對於 Mac 和 Windows，請參閱 gpg 連結的下載說明。


除了下載 Electrum 軟體之外，為了安全起見，您也需要該軟體的數位簽章。這是開發者用他的私人 gpg 金鑰製作的一串文字（它實際上是一個用文字編碼的數字）。使用 gpg 程式，我們可以將 SIGNATURE 與他的公開金鑰 (由開發者的私人金鑰所建立) (每個人都可以存取) 相對於下載檔案來進行「測試」。


換句話說，有了三個輸入（簽章、公開金鑰和資料檔案），我們就可以得到真或假的輸出，確認檔案未被竄改。


若要取得簽章，請按一下與您下載的檔案對應的連結 (請參閱彩色箭頭)：


![image](assets/2.webp)


按一下連結可能會自動將檔案下載到您的下載資料夾，也可能會在瀏覽器中開啟。如果在瀏覽器中開啟，您需要儲存檔案。您可以用滑鼠右鍵按一下，然後選擇「另存新檔」。視作業系統或瀏覽器而定，您可能需要在空白區域而非文字上按一下滑鼠右鍵。


以下是下載的文字。您可以看到有多個簽名 - 這些是不同人的簽名。您可以驗證每一個或任何一個。我將向您展示如何僅驗證開發人員的簽名。


![image](assets/3.webp)


接下來，您需要取得 ThomasV 的公開金鑰 - 他是主要的開發者。您可以直接從他、他的 Keybase 帳戶、Github 或其他人、鑰匙伺服器或 Electrum 網站取得。


從 Electrum 網站取得金鑰其實是最不安全的方式，因為如果這個網站是惡意的（這正是我們要檢查的），為什麼我們要從它取得公開金鑰呢（公開金鑰可能是假的）？


為了暫時保持簡單，無論如何我會告訴您如何從網站上取得，但請記住這一點。以下是 GitHub 上的副本 (https://github.com/spesmilo/electrum/blob/master/pubkeys/ThomasV.asc )，您可以與之比較。


向下捲動頁面，找到 ThomasV 公開金鑰的連結 (如下紅圈)。按一下並下載，或者如果在瀏覽器中開啟一些文字，按滑鼠右鍵儲存。


![image](assets/4.webp)


現在您有 3 個新檔案，可能都在下載資料夾中。它們在哪個資料夾並不重要，但如果您把它們都放在同一個資料夾中，會使處理過程更容易。


三個檔案：


1.電子琴下載

2.簽章檔案（通常與 Electrum 下載的檔案名稱相同，並加上「.asc

3.ThomasV 的公開金鑰。


在 Mac 或 Linux 中開啟終端機，或在 Windows 中開啟指令提示 (CMD)。


導覽到 Downloads 目錄 (或您放置這三個檔案的地方)。如果您不知道這是什麼意思，請參考此 Linux/Mac 短片 (https://www.youtube.com/watch?v=AO0jzD1hpXc) 和此 Windows 短片 (https://www.youtube.com/watch?v=9zMWXD-xoxc)。請記住，在 Linux 電腦上，目錄名是區分大小寫的。


在終端機中，鍵入以下指令，將 ThomasV 的公開密碼匙匯入您電腦上的「密碼匙環」（密碼匙環是一個抽象概念，實際上只是您電腦上的一個檔案）：


```bash
gpg --import ThomasV.asc
```


確保檔案名稱與您所下載的相符。此外，請注意是雙破折號而非單破折號。此外，請注意「-import」前後有一個空格。然後按 <enter>。


檔案應該會匯入。如果出現錯誤，請檢查您是否在檔案實際存在的目錄中。若要檢查您所在的目錄 (在 Mac 或 Linux 上)，請輸入 pwd。若要查看您所在的目錄中有哪些檔案 (在 Mac 或 Linux 上)，請鍵入 ls。您應該會看到「ThomasV.asc」文字檔被列在其中，可能還包括其他檔案。


然後，我們執行指令來驗證簽章。


```bash
gpg –verify Electrum-4.1.5.tar.gz.asc Electrum-4.1.5.tar.gz
```


請注意這裡有 4 個「Elements」，每個都以空格隔開。為了幫助您看清楚，我用粗體字交替標示。這四個 Elements 是


1. GPG 計畫

2. 驗證選項

3. 簽章的檔案名稱

4. 程式的檔案名稱


有趣的是，有時候您可以省略第 4 個元素，電腦就會猜到您的意思。我不太確定，但我相信只有當檔案名稱只差末尾的「asc」時才會有效。


不要只是複製我在這裡顯示的檔名 - 確保它們與您系統上的檔案名稱相符。


按下 <enter> 執行指令。您應該會看到 "good signature from ThomasV" 表示成功。會有一些錯誤，因為我們沒有其他人簽章的公開金鑰，而這些公開金鑰是包含在簽章檔中的（這個將簽章合併在一個檔案中的系統在以後的版本中可能會改變）。此外，底部還有一個警告，我們可以忽略（這是提醒我們，我們還沒有明確地告訴電腦我們信任 ThomasV 公鑰）。


現在我們有一個可安全使用的 Electrum 驗證副本。


## 運行電子幣


### 使用 Python 執行 Electrum


如果您下載的是 Python 版本，這就是讓它運作的方法。您會在下載頁面上看到這樣的內容：


![image](assets/5.webp)


對於 Linux，最好先更新您的系統：


```bash
sudo apt-get update
sudo apt-get upgrade
```


複製反白的黃色文字，貼到終端機，然後按 <enter>。您會被要求輸入密碼，可能還需要確認才能繼續，接著它就會安裝這些檔案 ("dependencies")。


您還需要將壓縮檔解壓縮到您選擇的目錄中。您可以使用圖形化使用者 Interface，或從命令列 (粉紅色反白的命令) 執行此操作 - 記得您的檔案名稱可能不同。(請注意，當我們在上一節驗證下載時，我們驗證的是壓縮檔，而不是解壓縮後的目錄）。


有一個使用 PIP 程式「安裝」的選項，但這是不必要的，會增加額外的步驟和安裝檔案。只要使用終端機執行程式，就可以繞過所有這些步驟。


步驟 (Linux) 為


1. 導覽到擷取檔案的目錄

2. 類型： ./run_electrum


在 Mac 上，步驟相同，但您可能需要先安裝 Python3，並使用此指令執行：


```bash
python3 ./run_electrum
```


一旦 Electrum 開始執行，終端視窗將會保持開啟。如果您關閉它，它會終止 Electrum 程式。只要在使用 Electrum 時最小化它即可。


### 使用 Appimage 執行 Electrum


這會比較容易，但不像 Windows 可執行檔案那麼容易。根據您執行的 Linux 版本，Appimage 檔案的預設屬性可能設定為系統不允許執行。我們必須更改這一設定。如果您的 Appimages 能正常運作，您可以跳過此步驟。使用終端機導航到檔案所在的位置，然後執行此指令：


```bash
sudo chmod ug+x Electrum-4.1.5-x86_64.AppImage
```


"sudo 「給予超級使用者權限；」chmod 「是變更模式的指令，改變誰可以讀、寫或執行；」ug+x "表示我們修改使用者和群組，允許執行。


調整檔案名稱以符合您的版本。


接著，雙擊 Appimage 圖示即可執行 Electrum。


### 在 Mac 上執行 Electrum


只要按兩下下載的檔案（它是一個「磁碟機」）。一個視窗將會打開。將視窗中的 Electrum 圖示拖曳到您的桌面，或任何您想保留程式的地方。然後您就可以「彈出」磁碟機，並刪除磁碟機 (下載的檔案)。


若要執行程式，只需按兩下程式。您可能會遇到一些 Mac 特有的「保姆」錯誤，需要繞過這些錯誤。


### 使用 Windows 執行 Electrum


儘管我最討厭 Windows，但這是最簡單的方法。只要按兩下可執行檔案即可執行。


## 從假的 Wallet 開始


當您第一次載入 Electrum 時，會開啟一個類似這樣的視窗：


![image](assets/6.webp)


我們稍後會手動選擇您的伺服器，但目前請保留預設值並自動連線。


接下來，建立一個虛擬 Wallet - 切勿將資金存入這個 Wallet。這個虛擬 Wallet 的目的是讓您在載入真正的 Wallet 之前，先進行軟體測試並確定一切運作良好。我們試著避免不小心洩露真正 Wallet 的隱私。如果您只是在練習，您所建立的 Wallet 無論如何都可以視為虛擬 Wallet。


您可以將名稱保留為「default_wallet」，或變更為任何您喜歡的名稱，然後按下一步。之後，如果您有多個錢包，您可以在此階段找到並打開它們，首先點擊 "Choose..."（選擇...）。


![image](assets/7.webp)


選擇「標準 Wallet」，然後選取 <下一步>：


![image](assets/8.webp)


然後，選擇「我已經有一個 seed」。我不希望您養成創建一個 Electrum seed 的習慣，因為它使用自己的協議，與其他錢包不兼容 - 這就是為什麼我們不點擊 "new seed"。


![image](assets/9.webp)


前往 https://iancoleman.io/bip39/ 並建立一個假的 seed。首先，將字號改為 12（這是常見的做法），然後按一下「generate」，並將方塊中的字複製到剪貼簿。


![image](assets/10.webp)


然後將單字貼到 Electrum 中。以下是一個範例：


![image](assets/11.webp)


Electrum 會尋找符合其本身協定的字詞。我們必須繞過這一點。點選選項，選擇 BIP39 seed：


![image](assets/12.webp)


這樣 seed 就變得有效了。(在這樣做之前，Electrum 期望的是一個 Electrum seed，所以這個 seed 被視為無效)。在您按下一步之前，請注意 "Checksum OK "的文字。重要的是（對於您稍後可能會使用的真正 Wallet），您必須在繼續之前看到它，因為它確認了您輸入的 seed 的有效性。底部的警告可以忽略，這是 Electrum 開發者對 BIP39 的抱怨，以及他們的 "FUD'ey "聲稱，他們的版本（與其他錢包不兼容）更優越。


**快速繞過一個重要的警告：** 檢查和的目的是確保您輸入的 seed 沒有輸入錯誤。校驗和是 seed 的最後部分 (第 12 個字最後是校驗和字)，在數學上由 seed 的第一部分 (11 個字) 決定。如果您在開始時輸入錯誤的字元，校驗和字元在數學上將不匹配，Wallet 軟體會警告您。這並不表示 seed 不能用來建立一個功能正常的 Bitcoin Wallet。試想一下，建立一個有打字錯誤的 Wallet，用 Bitcoin 載入 Wallet，然後有一天您可能需要還原 Wallet，但是當您還原時，您不會重新建立打字錯誤 - 您會還原錯誤的 Wallet！如果您的 Checksum 無效，Electrum 會讓您繼續製作 Wallet，這是相當危險的，所以請注意，這是您的責任。其他的錢包不會讓您繼續，這會安全得多。這就是我說 Electrum 很好用的原因之一，只要您學會正確使用它（Electrum 開發人員應該修正這一點）。


請注意，如果您想要新增 passphrase，選擇的機會就在這個選項視窗中，就在頂端。


按一下「確定」後，您會回到輸入 seed 詞組的位置。如果您選擇了 passphrase 選項，請勿與 seed 詞組一起輸入 (接下來會有相關提示)。


如果您沒有要求 passphrase，接下來您會看到這個畫面 - 您的 Wallet 腳本類型和衍生路徑的更多選項，您可以在這裡瞭解 (https://armantheparman.com/public-and-private-keys/)，但只要保留預設值就可以繼續。


![image](assets/13.webp)


**For extra info** 第一個選項允許您在兩者之間進行選擇：



- 傳統 (位址以「1」開頭)、
- Pay-to-Script-Hash（地址以「3」開頭）、
- bech32/native SegWit（位址以「bc1q」開頭）。


在撰寫本文時，Electrum 尚未支援 Taproot（位址以 "bc1p" 開頭）。此視窗的第二個選項允許您修改衍生路徑。我建議您永遠不要修改，尤其是在了解它的意義之前。人們會強調寫下衍生路徑的重要性，以便您可以在需要時恢復您的 Wallet，但如果您將其保留為預設值，您可能會沒事，所以不要驚慌 - 但寫下衍生路徑仍然是個好習慣。


接下來，您可以選擇加入 PASSWORD。這與「passphrase」不能混淆。密碼可以鎖定電腦上的檔案。passphrase 是私密金鑰組成的一部分。由於這是一個虛擬的 Wallet，您可以將密碼留空並繼續。


![image](assets/14.webp)


您會收到一個關於新版本通知的彈出視窗（我建議您選擇否）。之後，Wallet 就會自行 generate 並準備好使用 (但請記住，這個 Wallet 注定會被刪除，它只是一個假的 Wallet)。


![image](assets/15.webp)


我建議您做一些事情來設定軟體環境 (只需要一次)：


### 將單位變更為 BTC


進入頂部功能表，工具 -> electrum 偏好設定，在一般標籤下，您會找到將「基本單位」改成 BTC 的選項。

啟用位址和硬幣索引標籤


進入頂部功能表，檢視，然後選擇「顯示地址」。然後回到檢視，選擇「顯示硬幣」。


### 啟用單一伺服器


預設情況下，Electrum 會連接到一個隨機節點。它也會連接到許多其他次要節點。我不確定這些次要節點會交換什麼資料，但為了隱私起見，我們不希望發生這種情況。即使您指定一個節點，例如您自己的節點，這些多個其他節點也會連線，我不確定有哪些資訊會被分享。無論如何，這是很容易防止的。在我告訴您如何指定您自己的節點之前，我們會強制 Electrum 一次只連線到一個伺服器。


**Note:** 有一種方法可以從命令列指定「oneerver」，但我不推薦這種方式。我將展示一個我認為從長遠來看比較容易的替代方法，而且更有可能不會讓您意外連線到其他節點。


我們使用假的 Wallet 的原因是，如果我們載入了我們真正的 Wallet，加上我們真正的 Bitcoin，我們現在可能已經不小心連接到隨機節點了（即使我們一開始就選擇了 「手動設置服務器」，它還是會因為某些原因連接到這些其他的次要節點（嘿，Electrum 開發人員，你們應該修正這個問題）。如果我們的 Wallet 是私有的，這將是一個災難。


如果不先載入某種類型的 Wallet，我們也無法執行以下的步驟。(我們要編輯的設定檔只有在載入 Wallet 後才會被填入並可供編輯）。

您的新 Wallet 有一組接收位址，可由任何擁有 seed、passphrase 及衍生路徑的 Software Wallet 以數學方式重製。共有 43 億個！比您需要的還要多。Electrum 只會顯示前 20 個，當你用完前 20 個後，就會顯示更多。

## 測試 Wallet (學習使用)

這樣做有許多目的。



- 這將證明您有能力在新的 Wallet 中花費硬幣。
- 它將示範如何使用 Electrum 軟體製作花費（以及一些功能），在我們為了安全而增加額外的複雜性之前（使用 Hardware Wallet 或空氣封閉電腦）
- 它會強化您的想法，讓您知道在同一個 Wallet 內，有許多地址可供您選擇接收和消費。


開啟您的測試 Electrum Wallet，按一下 Addresses (位址) 索引標籤，然後用滑鼠右鍵按一下第一個 Address，選擇 Copy (複製) -> Address：

![image](assets/36.webp)

## 皮夾


我想使用 Electrum 演示一個 Watching Only Wallet。為此，我需要先定義「Wallet」。在 Bitcoin 中，"Wallet "有兩種用法：



- A 類，「Wallet」- 指顯示您的地址和餘額的軟體，例如 Electrum、Blue Wallet、Sparrow Wallet 等。



- B 類，「Wallet」- 是指與我們的 seed_phrase/passphrase/derivation_path 組合相關的獨特地址集合。任何 Wallet 中都有 86 億個位址 (43 億個接收位址，以及 43 億個變更位址)。如果您變更 seed 詞組、passphrase 或衍生路徑中的任何內容，您就會得到一個未使用的 Wallet，其中包含新的、且都是唯一的 86 億個空位址。


任何人在使用「Wallet」一詞時，所指的是哪一種類型，在上下文中是顯而易見的。

那還有什麼意義？


其中一個原因，但不是最主要的原因，是您有可能在電腦上觀察您的 Wallet 及其餘額，而不會讓您的私密金鑰暴露在電腦上的任何惡意軟體中。


另一個原因是，如果您選擇將您的私人金鑰放在電腦以外的地方，則必須使用它才能進行付款；我將向您解釋：


**硬體錢包 (HWW)** 的創造是為了讓裝置能安全地保存您的私人金鑰 (以 PIN 鎖定)，絕不將金鑰暴露給電腦 (即使透過纜線連接至電腦)，而且裝置本身無法連上網際網路。這樣的裝置無法自行進行交易，因為所有 Bitcoin 交易都是從參照 Blockchain (位於節點上) 的 UTXO 開始。Wallet 必須指定 UTXO 在哪個 transaction ID，以及交易的輸出是要花費的。只有在指定輸入後，才能首先建立新的交易，更不用說簽名了。硬體錢包無法建立交易，因為它們無法存取任何 UTXO - 它們沒有連接任何東西！


通常會從 HWW 擷取延伸的公開金鑰，然後在電腦上顯示地址 - 很多人都會熟悉在電腦上顯示地址和餘額的 Ledger 軟體或 Trezor Suite - 這是觀看 Wallet。這些程式可以建立交易，但無法簽署。它們只能讓與之相連的 HWW 簽署交易。HWW 從觀察的 Wallet 取得新產生的交易，簽署之後送回電腦，再廣播到節點。 **HWW 不能自己進行廣播**，而是由其相關的觀看 Wallet 進行廣播。如此一來，兩個錢包 (電腦上的公開金鑰 Wallet 和 HWW 中的私人金鑰 Wallet) 合作進行 generate、簽署和廣播，同時確保私人金鑰與網際網路連線裝置隔離。


## 部分簽署的 Bitcoin 交易 (PSBT)


交易有可能被建立並儲存到檔案中，之後重新載入、簽署、儲存、之後重新載入，最後再廣播 - 我不是說任何人都需要這麼做；這只是有可能發生的事。


現在考慮 3 對 5 多重簽章 Wallet - 5 個私密金鑰建立一個 Wallet，並且需要 3 個私密金鑰才能完全簽章交易 (請參閱此處以瞭解更多關於多重簽章 Wallet 金鑰的資訊)。可以有 5 台不同的電腦，每台電腦擁有 5 個私人金鑰中的一個。


電腦 1 可以 generate 進行交易並簽署。然後它可以將其儲存為檔案，並透過電子郵件傳送給電腦 2。電腦 2 可以簽名，並可能將檔案儲存為 QR 代碼，然後透過 Zoom Call 將 QR 傳送至世界另一端的電腦 3。電腦 3 可以擷取 QR，將交易載入 electrum，並簽署交易。前 2 次簽名之後，交易就變成 PSBT (Partially Signed Bitcoin Transaction)。第 3 次簽名之後，交易變成完全簽名且有效，可以進行廣播。


若要進一步瞭解 PSBTS，請參閱本指南。(https://armantheparman.com/PSBT/)

您可以向下捲動並閱讀該訊息。它總是讓您選擇「1」來輸入非零的帳號（衍生路徑的一部分），但如果您遵循我的建議，您就沒有亂用預設的衍生路徑，所以您不會想要非零的帳號；只要按下核取標記繼續即可。


然後選擇腳本語意。大多數人會選擇「Native SegWit」。


會顯示「已寫入 Electrum Wallet 檔案」，並顯示檔案名稱。在腦海中記下它。


然後取出 micro SD 卡，將其與 Electrum 一起放入電腦中。


有些作業系統會在您插入 microSD 時自動開啟檔案總管。很多人會看到新的 Wallet 檔案，然後按兩下，不知道為什麼不能用。這樣的設計並不好。實際上您必須忽略檔案總管（關閉），並使用 Electrum 開啟 Wallet 檔案：


開啟 Electrum。如果它已經用其他 Wallet 開啟，請選擇檔案 -> 新增。我們要找的是這個視窗：


![image](assets/52.webp)


訣竅在此，這並不直覺。按一下「選擇」。然後瀏覽 microSD 卡上的檔案系統，找到 Wallet 檔案並開啟。


現在您開啟了您的 Hardware Wallet 相對應的觀看 Wallet。不錯。


### 透過 USB 傳輸線連接。


這種方式應該比較容易，但對 Linux 電腦來說，就難多了。有一種叫做「Udev 規則」的東西需要更新。以下是方法 (詳細指南 https://armantheparman.com/gpg-articles/ )，簡單介紹一下：


這是一個好主意，以確保系統是最新的。然後


```bash
sudo apt-get install libusb-1.0-0-dev libudev-dev
```


那麼


```bash
python3 -m pip install ckcc-protocol
```


如果出現錯誤，請確定已安裝 pip。您可以使用 (pip -version) 檢查，也可以使用 (sudo apt install pythron3-pip) 安裝。


建立或修改現有檔案 /etc/udev/rules.d/


像這樣


```bash
sudo nano /etc/udev/rules.d
```


將會開啟文字編輯器。複製此處的文字並貼到 rules.d 檔案中，儲存並退出。


![image](assets/53.webp)


然後逐一執行這些指令：


```bash
sudo groupadd plugdev
sudo usermod -aG plugdev $(whoami)
sudo udevadm control –reload-rules && sudo udevadm trigger
```


如果您收到「group plugdev」已存在的訊息，沒問題，繼續。在第二個指令之後，您將不會收到任何回應，只要繼續執行第三個指令即可。


您可能需要中斷 ColdCard 與電腦的連線，然後再重新連線。

### 使用 ColdCard 進行交易


連接電纜後，交易變得輕而易舉。簽名交易將暢通無阻。


如果以空間封鎖的方式使用裝置，您必須手動使用 microSD 卡在裝置之間傳遞儲存的交易。有一些小技巧。


建立交易並完成後，您需要按一下左下角的匯出按鈕。您會看到「儲存到檔案」，但這並不是我們想要的。實際上，您必須先移至最後的選單選項「針對硬體錢包」，然後從該選項中找到另一個「儲存至檔案」並選擇該選項。然後將檔案儲存到 microSD，取出卡片並放入 ColdCard。請記住，您可能需要套用 passphrase 來選擇正確的 Wallet。螢幕會顯示準備簽署。按一下核取標記，檢查交易，然後用核取標記確認即可繼續。完成後，取出卡片，放回電腦。


然後，我們需要使用 electrum 開啟交易。該功能隱藏在菜單 tools -> load transaction 中。瀏覽檔案系統，找到檔案。每次簽約都會有三個檔案。守望 Wallet 製作的原始儲存檔案，以及兩個由 ColdCard 製作的檔案（我不知道它為何會這樣做）。其中一個會寫著 「已簽署」，另一個會寫著 「最終」。這並不直觀，但 「已簽署 」的那個沒有用，我們需要打開 「最終 」的交易。


載入之後，您可以按一下「廣播」，付款就會完成。


## 更新 Electrum 和隱藏的 ".electrum "目錄


Electrum 存儲在您電腦的兩個地方。一個是應用程式本身，另一個是隱藏的設定資料夾。根據您的作業系統，該資料夾存放在不同的地方：


視窗：


```bash
C:/Users/your_user_name_goes_here/AppData/Roaming/Electrum
```


Mac：


```bash
/Users/your_user_name_goes_here/.electrum
```


Linux：


```bash
/home/your_user_name_goes_here/.electrum
```


請注意在 Linux 和 Mac `electrum` 前面的 `.` - 這表示 director 是隱藏的。另外，請注意這個目錄只有在您第一次執行Electrum時才會（自動）建立。該目錄包含了Electrum的配置文件，也包含了您保存的任何錢包目錄。


如果您從電腦中刪除 Electrum 程式，除非您主動刪除，否則隱藏目錄仍會保留。


要升級電子琴，您必須經過我在一開始所描述的相同程序來下載和驗證。然後您的電腦上會有兩個程式的複本，您可以執行其中一個 - 每個程式都會存取相同的隱藏電子琴資料夾，以進行設定和存取 Wallet。所有我們儲存的東西，像是基本裝置、預設連線節點、其他偏好設定以及錢包存取權限，都會保留下來，因為這些東西都儲存在該資料夾中。


### 將您的電子琴和錢包移動到另一台電腦上


要做到這一點，您可以將程式檔案複製到 USB 磁碟機，同時複製 .electrum 目錄。然後將它們複製或移動到新電腦上。您不需要再次驗證程式。確保將 .electrum 目錄複製到預設位置（記得在該電腦首次運行 Electrum 之前複製，否則會產生一個空的預設 .electrum 資料夾，您可能會感到困惑）。
