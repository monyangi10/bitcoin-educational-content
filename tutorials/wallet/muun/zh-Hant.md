## Muun Wallet - 完整使用指南


Muun 應用程式的完整使用者指南 (含螢幕截圖)；方便使用的行動版 Bitcoin Wallet，可讓您在 Lightning Network 上進行交易。


![image](assets/0.5.webp)


### 下載 Muun 並建立您的 Wallet


首先，您需要下載 iOS 和 Android 版本的行動應用程式。請務必下載正確的版本。市面上有時候會出現假冒的複製品。我建議找到 Muun Wallet 的官方網站 https://muun.com/，並使用您選擇的系統（iOS/Android）的連結，從那裡您可以確定您使用的是官方應用程式。


![image](assets/1.webp)


開啟應用程式時，您可以選擇建立新的 Wallet 或復原現有的 Wallet。讓我們先建立一個新的 Wallet。之後我將示範復原 Wallet 的步驟。按下「建立新的 Wallet」。


![image](assets/2.webp)


Muun Wallet 會要求您建立一個四位數的個人識別碼 (PIN)。有了 PIN 碼，您就可以提高 Wallet 的安全性，以防萬一，例如惡意竊取您的手機，進而盜取您的比特幣。


![image](assets/3.webp)


就這樣，應用程式已產生全新的 Wallet，現在將成為首頁。現在，您需要先確保相關的 Wallet 復原資訊，然後再向其發送任何資金，因為這是一種有風險的做法。


![image](assets/4.webp)


### 關鍵備份


按下「備份您的 Wallet」方塊，您會被重新導向到「安全性」標籤。Muun 的備份程序方便地分為三個步驟。並非必須完成所有三個步驟，但結合三個步驟可提供最大的預防。


![image](assets/5.webp)


第一個選項允許您將 Wallet 連接到電子郵件 Address，此外還可以使用密碼保護。這個選項是可選的，跳過也沒有問題。如果您想使用它，請按下 「1：備份您的 Wallet」，然後在下一個畫面按下 "Start"，並輸入一個電子郵件 Address。下一個畫面會告訴您，您需要存取您的信箱並按一下 Muun 電子郵件中提供的連結來驗證電子郵件。


![image](assets/6.webp)


驗證電子郵件後，系統會提示您建立密碼。然後，您需要勾選兩個方塊，表示您瞭解 Wallet 復原（如有必要）需要使用您剛才選擇的電子郵件和密碼。這與傳統程式不同，傳統程式允許您在遺失或忘記密碼時重新設置密碼，因此請確保您已記下所有事項。


![image](assets/7.webp)

![image](assets/8.webp)


安全」標籤現在顯示您有基本備份。現在您可以返回「Wallet」標籤，使用應用程式完成交易（這些功能將在本指南稍後詳述），因為您知道 Wallet 現在是可以復原的。但是，我建議使用安全選項 #2 來 generate 額外的備份密碼，以防在選項 #1 中輸入的密碼被洩露，或者如果您不想使用電子郵件復原選項。


![image](assets/9.webp)


Muun 的「另類備份」選項類似於許多 Bitcoin 使用者熟悉的多個 Wallet 應用程式所使用的 Mnemonic 用語。按「開始」顯示您的復原代碼，並將其寫在紙上（應用程式會刪除顯示代碼頁面上的截圖）。記下後，請與螢幕上顯示的代碼比較，因為您需要在應用程式中重新輸入代碼，以確認其有效性。


Muun 再次要求您確認對問題的了解，也就是您需要這個 32 個字元的代碼，以防遺失先前建立的密碼。


根據我們所知的現代標準，Wallet 備份現在已經有顯著的安全性。不過，Muun 應用程式有第三個安全選項，稱為「緊急套件」。建立「緊急資料包」可讓您復原 Wallet，而無需透過 Muun。換句話說，只要使用不同於 Muun 的 Bitcoin Wallet 軟體即可。


![image](assets/10.webp)


按下「建立緊急應變工具包」後，您會聽到說明，此工具包以 PDF 文件的形式提供，其中包含有關獨立轉移資金的資訊與說明。由於需要使用您的 「恢復代碼」（文件中未包含），因此可以放心地將該工具包儲存到雲端。輕掃螢幕即可進入套件建立頁面。


![image](assets/11.webp)


有三種選項可供您選擇：



- 儲存至 Google 帳戶的雲端。
- 向自己的 Address 發送電子郵件，備份您的套件並進行存取。
- 使用裝置上的本機應用程式手動備份。


![image](assets/12.webp)


當您將套件寄送到所選的備份目的地後，請確定您可以存取套件，因為 Muun 會要求您輸入套件中的六位數代碼，以進行驗證。


![image](assets/13.webp)


完成最後一個步驟後，您的 Wallet 安全與復原設定就完成了。現在我們將探討使用新建立的備份來復原 Wallet 的不同方式。

Wallet 回收


![image](assets/14.webp)


在許多情況下，使用者可能暫時無法存取 Wallet 及資金；裝置遺失、未安裝/遺失應用程式、忘記個人識別號碼、Wallet 斷線等。因此，必須知道如何重新獲得此存取權。透過 Muun 應用程式恢復時，請按開啟畫面中的 "I Already Have A Wallet "選項。


![image](assets/15.webp)


### 使用電子郵件 Address 復原


如果您使用了 Muun 的備份選項 #1，請輸入當時選擇的電子郵件 Address。由於此選項是可選的，您也可以選擇繼續使用復原碼，這正好是沐恩提供的選項 #2。我們先來看看電子郵件選項。


![image](assets/15.webp)


輸入您的電子郵件 Address 後，Muun 會通知您有一封電子郵件寄給您，您需要存取該電子郵件以授權 Wallet 復原。檢查您的信箱（包括垃圾郵件區），並使用 Muun 電子郵件中提供的連結。您會被重新導向到應用程式，此時螢幕會要求您輸入與註冊電子郵件 Address 相關聯的密碼。


![image](assets/16.webp)


最後一步是建立個人識別號碼，然後您就會回到熟悉的 Wallet 首頁，顯示與之相關的餘額。


![image](assets/17.webp)


### 使用 「復原碼」


在復原現有 Wallet 的存取權限時，如果您選擇備份選項 #2，您可以選擇使用之前註明的復原代碼 (「復原代碼」，由 Muun 指定)。


此程序與上一節中詳述的類似；通過電子郵件進行復原。只要選擇使用「使用復原碼進行復原」選項，並在螢幕上顯示的適當欄位中輸入即可。如果您的 Wallet 除了有復原碼外，也有電子郵件備份，Muun 會要求您檢查信箱以授權復原程序，點選提供的連結後回到應用程式即可完成復原。再次，您必須建立個人識別號碼。就這樣，您就可以再次存取您的 Wallet。


### 使用緊急應變工具箱進行復原


若要在不使用 Muun Wallet 應用程式的情況下恢復您的 Wallet，您將需要您的緊急工具包，這是 Muun 提供的第三個恢復選項。此選項允許您將您的 Muun Wallet 中持有的資金發送至任何其他 Bitcoin Address。因此，請確保您有一個可以 generate 的替代 Wallet，而您要將資金傳送至該 Address。


存取您在建立套件時儲存的 PDF 文件。此文件包含復原 Wallet 所需的說明。請注意，此功能需要使用桌上型電腦或筆記型電腦，因為您需要下載由 Muun 開發團隊建立的腳本。該連結已包含在電子郵件中，但我還是要在此分享： https://github.com/muun/recovery


緊急工具包內有一個驗證碼，您已經用來確認工具包的建立，以及兩組鑰匙。在啟動 Muun 復原指令碼時，鑰匙是必需的。因此，請確保您在操作期間手邊有這些鑰匙。

## 接收交易


### Bitcoin 標籤


現在我們將介紹 Muun Wallet 的「接收」部分及其各種功能。您應用程式的首頁是「Wallet」標籤。您的餘額會顯示在中央，您可以點擊它來切換隱藏和顯示金額。我們將在本文稍後時間介紹所有的應用程式設定。現在，讓我們按「接收」來探索這項功能。


![image](assets/22.webp)


在此頁面，您可以選擇在 Bitcoin 和 Lightning 網路上接收交易。與所需網路相對應的新 Address (以及相關 QR 代碼) 將會顯示。預設情況下，當您到達 「接收 」畫面時，會顯示一個 Bitcoin Address。點選 QR 碼，Address 就會複製到裝置的剪貼簿。您可以輕鬆地使用「分享」按鈕直接分享 Address 到其他應用程式，也可以使用「複製」按鈕複製 Address。按下 Address 末端的眼睛圖示，就會顯示完整的 Address，讓您可以與分享時複製到剪貼簿的 Address 進行比較。


![image](assets/23.webp)


此資訊包含您在 Bitcoin 網路上接收交易所需的一切。此外，Muun 在「Address 設定」功能表下提供您一些自訂選項。首先，您可以在 Address 描述中加入金額。其次，您可以選擇使用 SegWit Address（預設選項）或傳統 Address（傳統）。


![image](assets/24.webp)


按 "Add +"，您可以在 Address 中加入特定金額，讓發送方更方便。這個選項是可選的。請注意，一旦輸入了金額，上一頁的 「複製 」按鈕就會在複製後的 Address 中加入資訊（"Bitcoin: "為前綴，金額為後綴）。為了避免臨時調整，請直接按下 QR 碼來複製 Address。金額資訊仍會隨之固定。此外，應用程式允許您選擇以您所選擇的貨幣輸入金額，簡化了兌換成 BTC 的過程。


![image](assets/25.webp)


關於 Address 類型、SegWit 或 Legacy 的選擇，我建議保留 SegWit。這種類型的 Address（以「bc1」開頭）可減少交易資料的大小，進而降低相關的交易費用。然而，在 Wallet 或軟體與 SegWit 位址不相容的情況下，您有可能需要使用「傳統」系統 (以「3」開頭的 Address)。因此，了解如何區分這兩種類型是很重要的。


![image](assets/26.webp)


## 閃電標籤


若要透過 Lightning Network 接收交易，您需要點選螢幕上方的同名標籤。此時會顯示一個包含 Lightning Address 的 QR 碼，您可以複製並分享這個 QR 碼，方法與本指南前面提到的 Bitcoin 位址相同。我提醒您，Lightning Network 除了讓您享受幾乎立竿見影的交易速度外，交易費用也只是 Bitcoin 鏈上交易費用的一小部分。


![image](assets/27.webp)


可在「Invoice 設定」功能表下找到自訂選項。在這裡，您可以點選「Add +」來修改與 Address 相關的金額。根據我使用 Lightning Network 的經驗，我認為在建立交易時輸入金額會比較好，因為有幾個錢包對空白的發票反應不好。您也會注意到在這個選單中有一個到期計時器。在此應用程式中，計時器設定為 60 分鐘，60 分鐘後 Address 將無效。請注意，Muun 會在您每次變更金額或離開又回到頁面時，產生新的 Lightning Address。


![image](assets/28.webp)


## 使用 LNURL 函式


Muun Wallet 提供使用 LNURL 接收交易的功能。點選位於頁面右上方的方形掃描符號即可啟動此功能，此功能有多項優點，包括可避免共用 Invoice 來接收交易。取而代之的是，您需要掃描 QR 碼來接收付款資訊，然後您可以驗證該 QR 碼來確認交易流程。


![image](assets/29.webp)


一開始，Muun 會向您顯示一個說明網頁（見上圖），然後請您啟用裝置的相機輸入，這是使用應用程式的必要步驟。請注意，目前並非所有 Lightning 電子錢包都支援 LNURL 位址。那些支持它的錢包通常只提供使用 LNURL 接收交易的功能，而不提供發送交易的功能。


![image](assets/30.webp)


## 傳送交易


### 透過 Bitcoin 網路


既然我們已經瞭解了如何使用 Muun 接收比特幣，現在讓我們來探討一下如何發送比特幣。回到主頁的 "Wallet 「標籤下，您需要按 」發送"。現在會出現一個簡單的頁面，您可以選擇將 Bitcoin 或 Lightning Address 複製到指定欄位，或按此欄位右側的 QR 碼圖示啟動相機，以 QR 碼形式掃描 Address。


![image](assets/31.webp)

![image](assets/32.webp)


當您到達「發送」頁面時，如果您的裝置上已經有複製的 Address，Muun 會識別 Address 格式 (Bitcoin 或 Lightning)，並建議使用它透過框架訊息廣播交易。


![image](assets/33.webp)

![image](assets/34.webp)


準備 Bitcoin 交易時，您需要輸入要發送的金額。確保螢幕上方顯示的目的地 Address 與之前複製的 Address 相符。在要發送的金額下方，Muun 會顯示您 Wallet 的餘額，並給您使用所有資金的選擇，如果您想完全清空您的 Wallet，避免留下 "Dust"（幾薩托希），這是一個非常有用的功能。


![image](assets/35.webp)


確認要寄送的金額後，Muun 會在下一頁要求您寫一則便條。這可作為額外的驗證，您可以隨意寫任何相關或不相關的內容。


![image](assets/36.webp)


在最終廣播之前，有必要對交易細節進行最後審查。驗證輸入的 Address 和金額，然後根據需要按「網路費用」右側的藍色鉛筆圖示自訂交易費用。了解 Bitcoin 交易池 (Mempool) 運作的基本原理是一個很好的教育經驗，隨著時間的推移，可以為您節省許多 Satoshis！


![image](assets/37.webp)


Muun的軟件預設實現了一種算法，可以在30分鐘或更短的時間內計算出確認所需的交易費用。這就是當您嘗試修改交易費用時會顯示的內容。手動輸入費用 "按鈕允許您自己定制這個細節，如果您需要更快的確認，或者相反，如果您有很大的操作空間，這個功能會非常有用。


![image](assets/38.webp)

選擇自行輸入交易費用金額後，您會被帶到一個新的頁面，顯示以 sat/vbyte (satoshis per virtual byte) 為單位輸入的金額。Muun 甚至會顯示與所選金額相關的預估確認時間，以及您所選擇的 BTC 和法定貨幣的費用。

![image](assets/39.webp)


回到交易詳細資訊總覽頁面，然後按「傳送」。這樣，您的交易就會在 Bitcoin 網路上廣播！您會被轉到 Wallet 的首頁，在這裡您會看到從您的餘額中扣除的金額。在螢幕底部，有一個箭頭，您可以按下來檢視您的交易歷史。您剛剛進行的交易將被添加到其發行的初始時刻。


![image](assets/40.webp)


按下項目可檢視特定交易的詳細資訊。當 Miner 將包含該交易的新區塊加入到鏈中時，您的交易將被確認。Muun 在螢幕底部分享 transaction ID，讓您在 Block explorer 上查看交易狀態。


![image](assets/41.webp)


## 透過 Lightning Network


現在讓我們使用 Bolt 11 Invoice（傳統/預設 Lightning Invoice）來進行交易。在 「發送 」頁面上複製或掃描 Lightning Address。您會被轉到一個新的頁面，顯示目前 Invoice 的詳細資料。交易金額會顯示出來（包括網路費用），還有寫在 Invoice 上的備注或說明，以及底部的到期時間。請注意，Lightning 交易的交易費用無法修改。它們是由它們到達收件人所需的通道路徑決定的。


![image](assets/42.webp)


(這是使用空白 Invoice 時螢幕上顯示的警告，意思是它沒有預先填入的金額。有些錢包支援這種類型的 Invoice，並允許您自訂金額。但 Muun 並非如此)。


![image](assets/43.webp)


按下 「眼睛 」圖示，就會顯示您在此交易中處理的Lightning節點的詳細資訊。您甚至可以選擇檢視網頁瀏覽器，以獲得更多資訊。這是 Muun 實現技術抽象的好例子。


![image](assets/44.webp)


一旦您按下「傳送」，您的交易就會啟動，通常在幾分之一秒內就會完成。支付的金額會從您的餘額中扣除，並在應用程式首頁顯示。回到您的交易記錄，即可看到付款的即時確認。


![image](assets/45.webp)


請注意，在歷史記錄中，Lightning 和 Bitcoin 交易會以不同的符號區分。若要檢視 Lightning 交易的詳細資訊，請點選歷史畫面中的交易。


![image](assets/46.webp)


## 應用程式設定


首頁的第三個標籤「設定」是您可以找到應用程式設定的地方。這個頁面短得出奇，尤其是與其他流行的手機錢包相比。在我看來，這並不是一個缺點；相反，我認為這是簡潔的優勢。

在一般類別中，您可以選擇偏好的帳戶單位和貨幣，以及應用程式的外觀主題 (深色或淺色)，最初會根據您的裝置系統決定。

要理解所謂的 Turbo 通道，我們必須先知道 Lightning 交易是透過一個使用者到另一個使用者的通道來進行的，而這些通道最初必須由 Bitcoin Blockchain 上的交易來提供資金。


渦輪通道允許您甚至在任何 On-Chain 交易被確認之前就開始在 Lightning Network 上交易。停用此功能將意味著您在 Lightning Network 上交易時，為了增加資金的安全性，您必須在 Exchange 上等待更長的時間，否則您必須相信 Muun 在等待您的交易在 Blockchain 上被確認時，不會有惡意的行為 (非常公開的雙重花費)。
