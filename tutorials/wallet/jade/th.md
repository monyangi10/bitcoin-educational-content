เสียบ Jade เข้ากับเดสก์ท็อปของคุณโดยใช้สาย USB ที่ให้มา


**หมายเหตุ:** หาก Jade ไม่ถูกจดจำโดยคอมพิวเตอร์ของคุณ ตรวจสอบให้แน่ใจว่ามีการติดตั้งไดรเวอร์ที่จำเป็นแล้ว รวมถึงตรวจสอบว่ามีปัญหาเกี่ยวกับสิทธิ์การใช้งาน USB หรือไม่


เมื่อ Jade ของคุณปรากฏใน Green ให้ทำการอัปเดต Jade โดยคลิกที่ ตรวจสอบการอัปเดต และเลือกเวอร์ชันเฟิร์มแวร์ล่าสุด ใช้ล้อเลื่อนหรือสวิตช์บน Jade เพื่อยืนยันและดำเนินการอัปเดตต่อไป ตรวจสอบให้แน่ใจว่า Jade ของคุณยังคงแสดงปุ่ม "Initialize" มิฉะนั้นคุณจะต้องรอจนกว่าจะตั้งค่า Jade เสร็จสิ้นเพื่ออัปเกรด ใช้ปุ่มย้อนกลับเพื่อกลับไปยังหน้าจอนี้หากจำเป็น

หลังจากที่คุณได้อัปเดตเฟิร์มแวร์ของ Jade แล้ว ให้เลือกการตั้งค่า Jade บนเครือข่ายและนโยบายความปลอดภัยที่คุณต้องการใช้


**เคล็ดลับ:** นโยบายความปลอดภัยจะแสดงอยู่ภายใต้ประเภทบนหน้าจอเข้าสู่ระบบที่แสดงด้านล่าง หากคุณไม่แน่ใจว่าจะเลือก Singlesig หรือ Multisig Shield โปรดตรวจสอบคู่มือของเรา [ที่นี่](https://help.blockstream.com/hc/en-us/articles/4403642609433)

### ภาคผนวก A - การตรวจสอบไฟล์ดาวน์โหลด Green Wallet


การตรวจสอบการดาวน์โหลดหมายถึงการตรวจสอบว่าไฟล์ที่คุณดาวน์โหลดไม่ได้ถูกแก้ไขตั้งแต่ถูกปล่อยโดยนักพัฒนา


เราทำเช่นนี้โดยการตรวจสอบว่าลายเซ็น (ที่สร้างโดยกุญแจส่วนตัวของนักพัฒนา) พร้อมกับไฟล์ที่ดาวน์โหลดและกุญแจสาธารณะของนักพัฒนาคืนค่าผลลัพธ์เป็น TRUE เมื่อผ่านฟังก์ชัน gpg –verify ฉันจะแสดงวิธีการทำเช่นนั้นให้คุณดูต่อไป


ก่อนอื่น เราจะได้รับคีย์การลงนาม:


สำหรับ Linux ให้เปิดเทอร์มินัลและรันคำสั่งนี้ (คุณควรคัดลอกและวางข้อความ และรวมเครื่องหมายอัญประกาศด้วย):


```bash
gpg --keyserver keyserver.ubuntu.com --recv-keys "04BE BF2E 35A2 AF2F FDF1 FA5D E7F0 54AA 2E76 E792"
```


สำหรับ Mac คุณทำสิ่งเดียวกัน ยกเว้นว่าคุณจะต้องดาวน์โหลดและติดตั้ง GPG Suite ก่อน


สำหรับ Windows คุณทำสิ่งเดียวกัน ยกเว้นว่าคุณจะต้องดาวน์โหลดและติดตั้ง GPG4Win ก่อน


คุณจะได้รับข้อความแสดงว่าคีย์สาธารณะได้ถูกนำเข้าแล้ว


![image](assets/9.webp)


รูปภาพนี้มีแอตทริบิวต์ alt ว่างเปล่า; ชื่อไฟล์ของมันคือ image-3-1024x162.webp


ถัดไป เราจำเป็นต้องได้รับไฟล์ที่มีแฮชของซอฟต์แวร์ มันถูกเก็บไว้ในหน้าของ Blockstream บน GitHub ก่อนอื่นไปที่หน้าข้อมูลของพวกเขาที่นี่ และคลิกที่ลิงก์สำหรับ "desktop" มันจะพาคุณไปยังหน้าปล่อยล่าสุดบน GitHub และที่นั่นคุณจะเห็นลิงก์ไปยังไฟล์ SHA256SUMS.asc ซึ่งเป็นเอกสารข้อความที่มีแฮชที่ Blockstream เผยแพร่ของโปรแกรมที่เราดาวน์โหลด


![image](assets/10.webp)


GitHub:


![image](assets/11.webp)


ไม่จำเป็นต้องทำ แต่หลังจากบันทึกลงดิสก์แล้ว ฉันได้เปลี่ยนชื่อ “SHA256SUMS.asc” เป็น “SHA256.txt” เพื่อให้ง่ายต่อการเปิดไฟล์บน Mac โดยใช้โปรแกรมแก้ไขข้อความ นี่คือเนื้อหาของไฟล์:


![image](assets/12.webp)


ข้อความที่เราต้องการอยู่ด้านบน ขึ้นอยู่กับไฟล์ที่เราดาวน์โหลด มีค่าแฮชที่สอดคล้องกันซึ่งเราจะเปรียบเทียบในภายหลัง


ส่วนล่างของเอกสารมีลายเซ็นที่ทำบนข้อความข้างต้น – เป็นไฟล์สองในหนึ่งเดียว


ลำดับไม่สำคัญ แต่ก่อนที่จะตรวจสอบแฮช เราจะตรวจสอบว่าแฮชข้อความนั้นเป็นของแท้ (เช่น ไม่ได้ถูกแก้ไข)


เปิดเทอร์มินัล คุณต้องอยู่ในไดเรกทอรีที่ถูกต้องที่ไฟล์ SHA256SUMS.asc ถูกดาวน์โหลดมา สมมติว่าคุณดาวน์โหลดมันไปที่ไดเรกทอรี “Downloads” สำหรับ Linux และ Mac ให้เปลี่ยนไปที่ไดเรกทอรีแบบนี้ (คำนึงถึงตัวพิมพ์เล็กใหญ่):


```bash
cd Downloads
```


แน่นอน คุณต้องกด <enter> หลังจากคำสั่งเหล่านี้ สำหรับ Windows ให้เปิด CMD (command prompt) และพิมพ์สิ่งเดียวกัน (แม้ว่าจะไม่คำนึงถึงตัวพิมพ์ใหญ่หรือตัวพิมพ์เล็กก็ตาม)


สำหรับ Windows และ Mac คุณจำเป็นต้องดาวน์โหลด GPG4Win และ GPG Suite ตามลำดับ ตามที่ได้แนะนำไว้ก่อนหน้านี้ สำหรับ Linux gpg มาพร้อมกับระบบปฏิบัติการ จาก Terminal (หรือ CMD สำหรับ Windows) ให้พิมพ์คำสั่งนี้:


```bash
gpg --verify SHA256SUMS.asc
```


การสะกดชื่อไฟล์ (เป็นสีแดง) อาจแตกต่างกันในวันที่คุณดึงไฟล์มา ดังนั้นให้แน่ใจว่าคำสั่งตรงกับชื่อไฟล์ที่ดาวน์โหลด คุณควรได้รับผลลัพธ์นี้ และให้ละเว้นคำเตือนเกี่ยวกับลายเซ็นที่เชื่อถือได้ – นั่นหมายความว่าคุณยังไม่ได้บอกคอมพิวเตอร์ด้วยตนเองว่าคุณเชื่อถือกุญแจสาธารณะที่เรานำเข้าไว้ก่อนหน้านี้


![image](assets/13.webp)


รูปภาพนี้มีแอตทริบิวต์ alt ว่างเปล่า; ชื่อไฟล์ของมันคือ image-4-1024x165.webp


ข้อความนี้ยืนยันว่าลายเซ็นถูกต้อง และเรามั่นใจว่าคีย์ส่วนตัวของ “info@greenaddress.it” ได้ลงนามในข้อมูล (รายงานแฮช)


ตอนนี้เราควรแฮชไฟล์ zip ที่ดาวน์โหลดมาและเปรียบเทียบผลลัพธ์ตามที่เผยแพร่ไว้ โปรดทราบว่าในไฟล์ SHA256SUMS.asc มีข้อความบางส่วนที่ระบุว่า “Hash: SHA512” ซึ่งทำให้ฉันสับสน เนื่องจากไฟล์นี้มีผลลัพธ์ของ SHA256 อย่างชัดเจน ดังนั้นฉันจะไม่สนใจส่วนนั้น


สำหรับ Mac และ Linux ให้เปิดเทอร์มินัล นำทางไปยังตำแหน่งที่ไฟล์ zip ถูกดาวน์โหลด (อาจจะต้องพิมพ์ “cd Downloads” อีกครั้ง เว้นแต่ว่าคุณยังไม่ได้ปิดเทอร์มินัลตั้งแต่ครั้งก่อน) อีกอย่างหนึ่ง คุณสามารถตรวจสอบได้เสมอว่าคุณอยู่ในไดเรกทอรีใดโดยพิมพ์ PWD (“print working directory”) และถ้าทั้งหมดนี้ดูแปลกใหม่ การดูวิดีโอ YouTube อย่างรวดเร็วโดยการค้นหา “how to navigate the Linux/Mac/Windows file system” จะเป็นประโยชน์


ในการรับไฟล์ ให้พิมพ์สิ่งนี้:


```bash
shasum -a 256 BlockstreamGreen_MacOS_x86_64.zip
```


คุณควรตรวจสอบว่าชื่อไฟล์ของคุณคืออะไร และแก้ไขข้อความในสีน้ำเงินด้านบนหากจำเป็น

### การใช้งานบน Sparrow

**หมายเหตุ:** หากคุณเคยใช้ Blockstream Jade เป็น Singlesig wallet กับ Blockstream Green และต้องการดูธุรกรรมของคุณใน Sparrow โปรดตรวจสอบให้แน่ใจว่าประเภทสคริปต์ตรงกับประเภทบัญชีที่มีเงินของคุณใน Green คุณจะต้องให้เส้นทางการอนุพันธ์ตรงกันด้วยเช่นกัน


![image](assets/18.webp)

**เคล็ดลับ:** ก่อนเชื่อมต่อ Jade ของคุณ ตรวจสอบให้แน่ใจว่าแอป Blockstream Green ไม่ได้เปิดอยู่ หาก Green เปิดอยู่ อาจทำให้เกิดปัญหาในการตรวจจับ Jade ภายใน Sparrow ได้
