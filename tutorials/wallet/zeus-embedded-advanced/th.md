## บทนำสู่ ZEUS Wallet


ZEUS เป็นแอปพลิเคชันสำหรับมือถือ Bitcoin wallet และการจัดการโหนดที่มีฟังก์ชันการทำงานเต็มรูปแบบของ bitcoin lightning wallet ที่ทำให้การชำระเงินด้วย bitcoin เป็นเรื่องง่าย ให้ผู้ใช้ควบคุมการเงินของตนเองได้อย่างสมบูรณ์ และอนุญาตให้ผู้ใช้ขั้นสูงจัดการโหนด Lightning ของพวกเขาจากฝ่ามือของพวกเขา


### ใครคือ ZEUS สำหรับ?


ปัจจุบัน ZEUS เหมาะสำหรับผู้ที่ดำเนินการ [Lightning Network Daemon (LND)](https://lightning.engineering/) หรือ [Core Lightning lightning (CLN)](https://blockstream.com/lightning/) โหนดที่บ้าน / ธุรกิจของตนเองและจัดการผ่าน Zeus จากระยะไกล


ผู้ค้าใช้ [BTCPay](https://btcpayserver.org/) หรือ [LNBits](https://lnbits.com/) หรือ [Alby](https://getalby.com/) (หรือบัญชี LNDhub อื่น ๆ) สามารถเชื่อมต่อ ใช้งาน และจัดการโหนด / บัญชีของพวกเขาจาก ZEUS ได้เช่นกัน


[เริ่มต้นใน v0.8](https://blog.zeusln.com/zeus-v0-8-0-open-beta/), ZEUS จะเริ่มให้บริการแก่ผู้ใช้ทั่วไปที่ต้องการวิธีง่ายๆ ในการทำธุรกรรมบิตคอยน์ที่รวดเร็วและราคาถูกจากอุปกรณ์มือถือของพวกเขา โดยมี [โหนด Lightning บนมือถือในตัว](https://docs.zeusln.app/category/embedded-node) พร้อมด้วย [ผู้ให้บริการ Lightning (LSP)](https://docs.zeusln.app/lsp/intro) ที่ผสานรวมอยู่ในตัว.

### คุณสมบัติของ Zeus

#### คุณสมบัติทั่วไป:


- Self-custodial, Bitcoin และ Lightning only wallet
- ไม่มีค่าธรรมเนียมการดำเนินการ, ไม่มี KYC
- โอเพ่นซอร์สอย่างเต็มที่ (APGLv3)
- รองรับหลายโหนด / บัญชี (คุณสามารถจัดการโหนดบ้านของคุณเอง, รันโหนด LND แบบฝัง, เชื่อมต่อกับหลายบัญชี LNDhub)
- เมนูกิจกรรมที่ใช้งานง่าย
- PIN หรือการเข้ารหัส passphrase, โหมดความเป็นส่วนตัว - ซ่อนข้อมูลที่ละเอียดอ่อนของคุณ
- สมุดติดต่อ, หลายธีม, หลายภาษา


#### คุณสมบัติทางเทคนิค


- เชื่อมต่อผ่าน Tor
- รองรับ LNURL อย่างเต็มรูปแบบ (ชำระเงิน, ถอน, ยืนยันตัวตน, ช่องทาง), ส่งไปยังที่อยู่ Lightning
- การจัดการช่องทางแสงสว่างโดยละเอียด, การสนับสนุน MPP/AMP, Keysend, การจัดการค่าธรรมเนียมการกำหนดเส้นทาง
- Replace-by-fee (RBF) และ Child-pays-for-parent (CPFP) สนับสนุน
- การชำระเงินและการร้องขอผ่าน NFC, ลงชื่อและยืนยันข้อความ
- Segwit และ Taproot สนับสนุน
- ช่อง Taproot แบบง่าย
- ที่อยู่ Lightning แบบดูแลตนเอง (@zeuspay.com)
- Point of Sale by Square (เร็วๆ นี้เปิด PoS)

## คำแนะนำทีละขั้นตอนเกี่ยวกับวิธีเริ่มต้นใช้งาน Zeus LN embedded node บนอุปกรณ์มือถือของคุณ


![Image](assets/en/01.webp)


ฉันขอมอบคู่มือนี้ให้กับผู้ใช้ Lightning Network (LN) ใหม่ทุกคนที่ต้องการเริ่มต้นการเดินทางอธิปไตยใหม่โดยใช้โหนดการดูแลตนเอง wallet บนอุปกรณ์เคลื่อนที่ของพวกเขา


ลองพิจารณาว่าคุณได้ผ่านกระเป๋าเงิน LN ที่มีการดูแลทั้งหมดแล้ว แต่คุณยังไม่พร้อมที่จะเริ่มใช้งานโหนด LN สำหรับการกำหนดเส้นทางสาธารณะ คุณแค่ต้องการสะสม sats เพิ่มเติมบน LN ในวิธีที่คุณดูแลเองมากขึ้นและทำการชำระเงินปกติของคุณบน LN


นี่คือ Zeus ที่มาพร้อมกับ [เวอร์ชัน v0.8.0 ที่ประกาศบนบล็อกของพวกเขา](https://blog.zeusln.com/new-release-zeus-v0-8-0/), ตอนนี้มีการฝังโหนด LND เข้าไปในแอปแล้ว จนถึงตอนนี้ Zeus เป็นแอปจัดการโหนดระยะไกล + บัญชี LNDhub แต่ตอนนี้... โหนดอยู่ในโทรศัพท์แล้ว!

### สรุปคุณสมบัติหลักของ Zeus Node:



- โหนดส่วนตัว LND** - หมายความว่าโหนดนี้จะไม่ทำการส่งต่อการชำระเงินของผู้อื่นผ่านโหนดของคุณ โหนดและช่องทางจะไม่ได้ประกาศ (เป็นส่วนตัว ไม่ปรากฏบนกราฟสาธารณะ LN) การรับและทำการชำระเงินจะทำผ่านเพื่อน LSP ที่คุณเชื่อมต่ออยู่ จำไว้ว่า: โหนด Zeus Embedded จะไม่ทำการส่งต่อสาธารณะ!
- บริการ LND แบบถาวร** - ผู้ใช้สามารถเปิดใช้งานคุณสมบัตินี้และรักษาบริการ LND ให้ทำงานอย่างต่อเนื่องเหมือนกับโหนด LN ปกติ แอปไม่จำเป็นต้องเปิด บริการแบบถาวรจะรักษาการสื่อสารทั้งหมดให้เชื่อมต่อออนไลน์ตลอดเวลา
- Neutrino block filters** - block sync is done using [block filters and the Neutrino protocol](https://bitcoinops.org/en/topics/compact-block-filters/) (given no information about our users' on-chain funds). Reminder: for high-latency / slow internet connections this block sync based on neutrino, sometimes could fail. Trying to switch to a near neutrino server could help restore the sync. Without this sync your LND node could not start!
- Simple Taproot Channels** - เมื่อปิดช่องทางเหล่านี้ ผู้ใช้จะเสียค่าธรรมเนียมน้อยลงและได้รับความเป็นส่วนตัวมากขึ้น เนื่องจากพวกเขาจะปรากฏเหมือนการใช้จ่าย Taproot อื่น ๆ เมื่อพิจารณาจากร่องรอย on-chain ของพวกเขา
- Integrated LSP** - Olympus เป็นโหนด LSP ใหม่สำหรับ Zeus ผู้ใช้สามารถรับ sats ผ่าน LN ได้ทันที โดยไม่จำเป็นต้องตั้งค่า LN ล่วงหน้า เพียงแค่สร้างใบแจ้งหนี้ LN และชำระเงินจาก LN wallet อื่น ๆ ด้วยบริการช่องทาง 0-conf ของ Zeus อ่านเพิ่มเติมเกี่ยวกับ Zeus LSP ได้ที่นี่ LSP ยังเพิ่มความเป็นส่วนตัวให้กับผู้ใช้ของเราโดยการให้ใบแจ้งหนี้ที่ห่อหุ้มซึ่งปกปิดคีย์สาธารณะของโหนดจากผู้ชำระเงิน
- สมุดรายชื่อ** - คุณสามารถบันทึกรายชื่อด้วยตนเองหรือ นำเข้าจาก NOSTR เพื่อส่งการชำระเงินไปยังปลายทางที่คุณใช้บ่อยได้อย่างง่ายดาย
- รองรับ LNURL, LN Address การส่งและรับ** - ตอนนี้คุณสามารถตั้งค่า LN Address ที่คุณดูแลเองได้ที่ @zeuspay.com เตือนความจำ: คุณยังสามารถใช้ Zeus สำหรับ LN-auth บนเว็บไซต์ที่คุณสามารถเข้าสู่ระบบด้วยการยืนยันตัวตน LN ได้อีกด้วย สะดวกมาก
- จุดขาย** - ตอนนี้ผู้ค้าใช้งานสามารถตั้งค่าสินค้าของตนเองและขายโดยตรงจาก Zeus ด้วยระบบ PoS ที่รวมอยู่ด้วย ขณะนี้มีฟีเจอร์พื้นฐานแต่ในอนาคตจะมีฟีเจอร์เพิ่มเติม
- LND logs** - ผู้ใช้สามารถอ่านบันทึกการให้บริการ LND แบบเรียลไทม์และใช้เพื่อแก้ไขปัญหาที่อาจเกิดขึ้น (ส่วนใหญ่สำหรับการเชื่อมต่อที่ไม่ดี)
- การสำรองข้อมูลอัตโนมัติ** - ช่องสัญญาณของโหนด LN จะถูกสำรองข้อมูลอัตโนมัติบนเซิร์ฟเวอร์ Olympus การสำรองข้อมูลอัตโนมัตินี้ถูกเข้ารหัสด้วยโหนด wallet seed ของคุณ (หากไม่มี seed จะไม่มีประโยชน์เลย) ผู้ใช้ยังสามารถส่งออก SCB (การสำรองข้อมูลช่องสัญญาณแบบสถิต) ด้วยตนเองสำหรับการกู้คืนจากภัยพิบัติได้


### วิธีการเข้าร่วมกับ Zeus LN Node (LND embedded)


ในคู่มือนี้ ฉันจะพูดถึงเฉพาะโหนด LND ที่ฝังอยู่เท่านั้น และจะไม่พูดถึงวิธีการอื่นๆ ในการใช้แอปที่ยอดเยี่ยมนี้ (การจัดการโหนดระยะไกลและบัญชี LNDhub) สำหรับประเภทการเชื่อมต่ออื่นๆ โปรดดูที่ [หน้าคู่มือ Zeus](https://docs.zeusln.app/category/getting-started) ซึ่งอธิบายได้ดีมากและไม่จำเป็นต้องเขียนคู่มือเฉพาะเพิ่มเติม


#### ขั้นตอนที่ 1 - การตั้งค่าเริ่มต้น


เนื่องจาก Zeus เป็นโหนด LND เต็มรูปแบบ ฉันจึงมีคำแนะนำเบื้องต้นบางประการ:



- อย่าใช้เครื่องเก่า เพราะอาจส่งผลต่อการใช้แอปที่ทรงพลังนี้ได้ โดยเฉพาะในช่วงที่ซิงค์ แอปอาจใช้ CPU และ RAM อย่างเข้มข้น การขาดแคลนสิ่งเหล่านี้อาจทำให้ไม่สามารถใช้แอป Zeus ได้เลย
- ใช้ระบบปฏิบัติการ Android 11 ขึ้นไปและอัปเดตให้มากที่สุดเท่าที่จะเป็นไปได้ สำหรับ iOS ก็เช่นกัน พยายามใช้เวอร์ชันของ OS ที่สูงขึ้นมาก
- คุณจะต้องมีพื้นที่ดิสก์อย่างน้อย 1GB สำหรับการจัดเก็บข้อมูล ในเวลาต่อมาอาจเพิ่มขึ้นได้ แต่มีฟังก์ชันในการบีบอัดฐานข้อมูลให้เหลือระดับ MBs
- ไม่มีความจำเป็นที่จะต้องใช้ Zeus กับบริการ Tor หรือ Orbot โปรดอย่าทำให้เรื่องซับซ้อนเกินความจำเป็น ในกรณีนี้ Tor จะไม่ให้ความเป็นส่วนตัวมากขึ้น แต่จะทำให้การซิงค์เริ่มต้นแย่ลงเท่านั้น นอกจากนี้ควรระวัง VPN ที่คุณใช้งานและตรวจสอบความหน่วงของการเชื่อมต่อไปยังเซิร์ฟเวอร์ neutrino โปรดทราบว่า Neutrino block filter ไม่ได้รั่วไหลหรือสืบค้นตัวตนอุปกรณ์ของคุณ เพียงแค่ให้บริการบล็อกเท่านั้น การจราจรของ LN ก็อยู่หลัง LSP กับช่องทางส่วนตัว ดังนั้นข้อมูลที่ออกมามีน้อยมาก ไม่มีเหตุผลที่จะต้องกังวลเกี่ยวกับความเป็นส่วนตัว
- โปรดอดทนรอสำหรับการซิงค์เริ่มต้น อาจใช้เวลาหลายนาที พยายามเชื่อมต่อกับอินเทอร์เน็ตบรอดแบนด์ที่มีความหน่วงต่ำ หากคุณใช้งานโหนด Bitcoin ของคุณเอง [คุณสามารถเปิดใช้งานบริการนิวตริโน](https://docs.lightning.engineering/lightning-network-tools/lnd/enable-neutrino-mode-in-bitcoin-core) และเชื่อมต่อ Zeus ของคุณกับโหนดของคุณเอง แม้กระทั่งการใช้ LAN ภายใน เพื่อให้คุณมีความเร็วสูงสุด


เมื่อคุณตั้งค่าประเภทการเชื่อมต่อ “Embedded node” แอปจะเริ่มซิงค์สักครู่ รออย่างอดทนจนกว่าส่วนนั้นจะเสร็จสิ้น จากนั้นเข้าสู่หน้าการตั้งค่าหลัก


![Image](assets/en/03.webp)


ก่อนอื่น มาดูแต่ละส่วนของการตั้งค่าและทำความเข้าใจคุณสมบัติหลักบางประการ ก่อนที่คุณจะเริ่มใช้ Zeus:


**A - การตั้งค่า**


นี่คือส่วนที่มีการตั้งค่าทั่วไปสำหรับแอปทั้งหมด


**1 - ผู้ให้บริการสายฟ้า (LSP)**


นี่คือการนำเสนอบริการ LSP สองประเภท:



- _Just in time channels_ - เมื่อคุณไม่มีช่องทางเปิดหรือไม่มีสภาพคล่องขาเข้า หากบริการนี้ถูกเปิดใช้งาน มันจะเปิดช่องทางให้คุณทันที ตัวเลือกนี้สามารถปิดใช้งานได้หากคุณไม่ต้องการเปิดช่องทางเพิ่มเติมในลักษณะนี้
- _ขอช่องทางล่วงหน้า_ - คุณสามารถซื้อช่องทางขาเข้าจาก Olympus LSP ได้โดยตรงในแอปพร้อมตัวเลือกและจำนวนเงินที่หลากหลาย (สำหรับขาเข้าและขาออก)


LSP ช่วยเชื่อมต่อผู้ใช้กับเครือข่าย Lightning โดยการเปิดช่องทางการชำระเงินไปยังโหนดของพวกเขา [อ่านเพิ่มเติมเกี่ยวกับ LSP ที่นี่](https://medium.com/breez-technology/envisioning-lsps-in-the-lightning-economy-832b45871992) ZEUS มี LSP ใหม่ที่รวมเข้าด้วยกันเรียกว่า [OLYMPUS by ZEUS](https://mempool.space/lightning/node/031b301307574bbe9b9ac7b79cbe1700e31e544513eae0b5d7497483083f99e581) ซึ่งพร้อมใช้งานสำหรับผู้ใช้ทุกคนที่ใช้โหนดฝังตัวใหม่


ในส่วนนี้ โดยค่าเริ่มต้นคือ Olympus LSP (https://0conf.lnolymp.us) แต่เร็วๆ นี้คุณสามารถตั้งค่า 0conf LSP อื่นที่รองรับโปรโตคอลนี้ได้เช่นกัน


_โปรดทราบ:_

_เมื่อคุณเปิดช่องทางกับ Olympus LSP โดยใช้ใบแจ้งหนี้ LN ที่ห่อหุ้มไว้ คุณยังได้รับสภาพคล่องขาเข้าจำนวน 100k ด้วย! นี่เป็นตัวเลือกที่ดีมากในกรณีที่คุณต้องการรับ sats เพิ่มทันที_

_ตัวอย่าง: คุณฝาก 400k sats เพื่อเปิดช่องทาง LSP จากนั้น LSP จะเปิดช่องทางความจุ 500k sats ไปยังโหนด Zeus ของคุณและผลักดัน 400k sats ที่คุณฝากไปยังฝั่งของคุณ._

_"Inbound liquidity" = มี “พื้นที่” ในช่องของคุณเพื่อรับมากขึ้น._


ในอนาคตเราหวังว่าเราจะมี LSP อื่นๆ อีกมากมายที่สามารถรวมเข้ากับ Zeus และใช้สลับกันได้ เป็นเพียงเรื่องของเวลาจนกว่า LSP ใหม่ๆ จะนำมาตรฐานเปิดมาใช้สำหรับช่องทาง 0conf ประเภทนี้


หากคุณไม่ต้องการเปิดช่องทางใหม่ “ทันที” คุณสามารถปิดใช้งานตัวเลือกนี้ได้


ในส่วนเดียวกันนี้ คุณยังมีตัวเลือกในการเลือก "ขอช่องทาง Simple Taproot" เมื่อ LSP จะเปิดช่องทางไปยังโหนด Zeus ของคุณ ช่องทาง Simple Taproot เหล่านี้เสนอความเป็นส่วนตัวของ on-chain ที่ดียิ่งขึ้นและค่าธรรมเนียมที่ต่ำกว่าในการปิดช่องทาง มีเพียงสองเหตุผลที่คุณอาจไม่ต้องการใช้พวกมัน:



- พวกมันเป็นของใหม่ และอาจยังมีข้อบกพร่องใน LND เมื่อใช้งาน
- คู่สัญญาของคุณไม่รองรับพวกเขา แม้แต่โหนด LND ก็ต้องเลือกเข้าร่วมอย่างชัดเจนในตอนนี้


**2 - การตั้งค่าการชำระเงิน**


ฟีเจอร์นี้จะมอบวิธีให้คุณตั้งค่าธรรมเนียมที่คุณต้องการสำหรับการชำระเงิน ผ่าน LN หรือบนเชน นอกจากนี้ยังมีตัวเลือกในการเพิ่มหรือลดเวลาหมดอายุสำหรับใบแจ้งหนี้ของคุณ


หากการชำระเงิน LN บางรายการของคุณล้มเหลว คุณสามารถเพิ่มค่าธรรมเนียมเพื่อค้นหาเส้นทางที่ดีกว่าได้ นอกจากนี้ หากคุณกำลังทำธุรกรรมบนเชน คุณสามารถตั้งค่าธรรมเนียมเฉพาะเพื่อให้ธุรกรรมของคุณไม่ติดอยู่ในเมมพูลเป็นเวลานาน ในกรณีที่มีช่วงค่าธรรมเนียมสูง


**3 - การตั้งค่าใบแจ้งหนี้**


ในส่วนนี้มีตัวเลือกบางอย่างสำหรับใบแจ้งหนี้ generate:



- ตั้งค่าบันทึกมาตรฐานให้แสดงในใบแจ้งหนี้คุณ generate
- เวลาหมดอายุเป็นวินาที ในกรณีที่คุณต้องการเวลาที่เฉพาะเจาะจง ยาวขึ้นหรือน้อยลงสำหรับการชำระเงินใบแจ้งหนี้ของคุณ
- รวมคำแนะนำเส้นทาง - ให้ข้อมูลเพื่อค้นหาช่องทางที่ไม่ได้โฆษณาหรือช่องทางส่วนตัว ซึ่งช่วยให้การชำระเงินสามารถส่งไปยังโหนดที่ไม่ปรากฏต่อสาธารณะบนเครือข่ายได้ คำแนะนำเส้นทางจะให้เส้นทางบางส่วนระหว่างโหนดส่วนตัวของผู้รับและโหนดสาธารณะ จากนั้นคำแนะนำเส้นทางนี้จะถูกรวมอยู่ในใบแจ้งหนี้ที่สร้างโดยผู้รับและมอบให้กับผู้ชำระเงิน ฉันแนะนำให้เปิดใช้งานโดยค่าเริ่มต้น มิฉะนั้นการชำระเงินขาเข้าอาจล้มเหลว (ไม่พบเส้นทาง)
- ใบแจ้งหนี้ AMP - การชำระเงินแบบหลายเส้นทางเชิงอะตอมเป็นประเภทใหม่ของการชำระเงินผ่าน Lightning ที่ดำเนินการโดย LND ซึ่งอนุญาตให้รับ sats โดยไม่ต้องมีใบแจ้งหนี้เฉพาะ โดยใช้ [keysend](https://docs.lightning.engineering/lightning-network-tools/lnd/send-messages-with-keysend) ซึ่งเป็นรหัสการชำระเงินแบบคงที่ [อ่านเพิ่มเติมที่นี่](https://docs.lightning.engineering/lightning-network-tools/lnd/amp)
- แสดงฟิลด์ preimage แบบกำหนดเอง - ใช้ตัวเลือกนี้เฉพาะในกรณีที่เฉพาะเจาะจงมากเมื่อคุณต้องการใช้ฟิลด์แบบกำหนดเองใน preimage จริงๆ [อ่านเพิ่มเติมที่นี่](https://bitcoin.stackexchange.com/questions/90797/how-can-i-generate-preimage-for-lightning-network-invoice-should-i).


อีกตัวเลือกหนึ่งในส่วนนี้คือวิธีการตั้งค่าประเภทของที่อยู่บนเชนที่คุณต้องการใช้: segwit nested, segwit, taproot.


![Image](assets/en/04.webp)


คลิกที่ปุ่มล้อด้านบนและหน้าจอป๊อปอัพจะปรากฏขึ้นเพื่อเลือกประเภทที่อยู่ที่ต้องการ เมื่อคุณตั้งค่านั้นแล้ว ครั้งต่อไปที่คุณกดปุ่มรับสำหรับ onchain มันจะ generate ประเภทที่อยู่ที่เลือกไว้ คุณสามารถเปลี่ยนได้ตลอดเวลา


**4 - การตั้งค่าช่อง**


ในส่วนนี้คุณสามารถตั้งค่าคุณสมบัติของช่องเปิดบางอย่างได้ เช่น:



- จำนวนการยืนยัน
- ประกาศช่อง (ค่าเริ่มต้นคือปิด) ซึ่งหมายความว่าจะเป็นช่องที่ไม่ได้ประกาศ
- ช่อง Taproot แบบง่าย
- แสดงปุ่มซื้อช่อง


**5 - การตั้งค่าความเป็นส่วนตัว**


ที่นี่คุณจะพบการตั้งค่าพื้นฐานบางอย่างเพื่อเพิ่มความเป็นส่วนตัวโดยใช้แอป Zeus:



- Block explorer เพื่อเปิดรายละเอียด tx (mempool.space, blockstream.info หรือแบบกำหนดเอง)
- อ่านคลิปบอร์ด - เปิด/ปิดสลับหากคุณต้องการให้ Zeus อ่านคลิปบอร์ดของอุปกรณ์ของคุณ
- โหมดซุ่มดู - เปิด/ปิดสลับได้หากคุณต้องการซ่อนข้อมูลที่ละเอียดอ่อนจากแอป Zeus ของคุณ เป็นตัวเลือกที่ดีเมื่อคุณทำการสาธิตหรือถ่ายภาพหน้าจอ
- ข้อเสนอค่าธรรมเนียม Mempool - เปิดใช้งานตัวเลือกนี้หากคุณต้องการใช้ระดับค่าธรรมเนียมที่แนะนำจาก [mempool.space](https://mempool.space/)


**6 - ความปลอดภัย**


ส่วนนี้มีเพียงสองตัวเลือกสำหรับการรักษาความปลอดภัยของแอปเมื่อเปิด: ตั้งรหัสผ่านหรือ PIN


เมื่อคุณตั้ง PIN เพื่อเปิดแอป คุณยังสามารถตั้ง “duress PIN” ได้อีกด้วย PIN ลับนี้จะใช้เฉพาะในกรณีที่คุณถูกบังคับเท่านั้น หากคุณใส่ PIN นี้ การตั้งค่าทั้งหมดจะถูกลบออก ดังนั้นคุณควรอัปเดตข้อมูลสำรองของคุณให้ทันสมัยอยู่เสมอ การสำรองข้อมูลอัตโนมัติเปิดใช้งานโดยค่าเริ่มต้น แต่ก็ควรมีข้อมูลสำรองของคุณเองด้วย นอกอุปกรณ์


**7 - สกุลเงิน**


เปิดหรือปิดตัวเลือกเพื่อแสดงการแปลงสกุลเงินเฟียตในการใช้งานแอป Zeus ปัจจุบันรองรับสกุลเงินเฟียตมากกว่า 30 สกุลทั่วโลก


**8 - ภาษา**


คุณสามารถสลับระหว่างหลายภาษาการแปล ซึ่งได้รับการตรวจสอบโดยชุมชน Zeus ที่มีเจ้าของภาษาพูด


**9 - แสดงผล**


ในส่วนนี้คุณสามารถปรับแต่งการแสดงผล Zeus ของคุณได้ โดยเลือกธีมสีต่างๆ หน้าจอเริ่มต้น (แป้นพิมพ์หรือยอดคงเหลือ) แสดงชื่อเล่นของโหนด เปิดใช้งานปุ่มแป้นพิมพ์ขนาดใหญ่ แสดงตำแหน่งทศนิยมเพิ่มเติม


**10 - จุดขาย**


นี่คือฟีเจอร์พิเศษสำหรับเปิด / ปิดระบบ PoS ที่ผสานรวมเข้ากับ Zeus คุณสามารถใช้งาน PoS แบบสแตนด์อโลนหรือเชื่อมต่อกับระบบ Square PoS ได้ ปัจจุบันรองรับฟังก์ชันพื้นฐานในฐานะ PoS แต่เพียงพอสำหรับการเริ่มต้นที่ดีและสามารถช่วยเหลือผู้ค้ารายย่อย (บาร์/ร้านอาหาร, ร้านขายของชำ) ให้เริ่มรับ BTC ได้ในรูปแบบที่เป็นธรรมชาติ


ภายในการตั้งค่านี้ คุณจะพบตัวเลือกต่างๆ เพื่อการตั้งค่า PoS ของคุณ:



- ประเภทการยืนยันการชำระเงิน: LN เท่านั้น, 0-conf, 1-conf
- เปิด / ปิดคำแนะนำสำหรับพนักงานที่ใช้งาน PoS
- แสดง / ซ่อนแป้นพิมพ์
- เปอร์เซ็นต์ภาษีที่จะใช้กับตั๋ว
- สร้างผลิตภัณฑ์และหมวดหมู่ของผลิตภัณฑ์
- รายการขายทั้งหมดอย่างง่าย

**B - สำรองข้อมูล wallet**


โหนดที่ฝังอยู่ใน ZEUS ใช้พื้นฐานจาก LND และใช้ [รูปแบบ aezeed seed](https://github.com/lightningnetwork/lnd/blob/master/aezeed/README.md) ซึ่งแตกต่างจาก [รูปแบบ BIP39](https://github.com/bitcoin/bips/blob/master/bip-0039.mediawiki) ที่คุณเห็นในกระเป๋าเงิน Bitcoin ส่วนใหญ่ แม้ว่ามันอาจจะดูคล้ายกันก็ตาม Aezeed รวมข้อมูลเพิ่มเติมบางอย่างรวมถึงวันเกิดของ wallet ที่จะช่วยให้การสแกนใหม่ในระหว่างการกู้คืนเกิดขึ้นได้อย่างมีประสิทธิภาพมากขึ้น


รูปแบบคีย์ aezeed ควรจะเข้ากันได้กับกระเป๋าเงินมือถือดังต่อไปนี้: Blixt, BlueWallet และ Breez. โปรดทราบว่า seed เพียงอย่างเดียวจะไม่เพียงพอในการกู้คืนยอดเงินทั้งหมดของคุณหากคุณมีช่องทางที่เปิดหรือกำลังปิดอยู่!


เรียนรู้เพิ่มเติมเกี่ยวกับกระบวนการสำรองและกู้คืนได้ที่ [หน้าคู่มือ Zeus](https://docs.zeusln.app/for-users/embedded-node/backup-and-recovery)


คำแนะนำด้านพลังงาน: เมื่อคุณบันทึก seed ของคุณ กรุณาบันทึก pubkey ของ node ด้วย! บางครั้งการมีมันไว้ในมือพร้อมกับ seed และ SCB (Static Channels Backup) ของคุณก็เป็นสิ่งที่ดีในกรณีที่คุณต้องการตรวจสอบการกู้คืน.


SCB จำเป็นเฉพาะเมื่อคุณมีช่องทาง LN เปิดอยู่ ในกรณีที่คุณมีเฉพาะเงินทุนบนเชน ไม่จำเป็น


หากคุณเห็นว่าหลังจากเวลาผ่านไปนานแล้วยังไม่แสดงประวัติ txs เก่า ให้ไปที่ Embedded node - Peers และปิดตัวเลือกการใช้รายการของ peers ที่เลือกไว้ (โดยค่าเริ่มต้นคือ btcd.lnolymp.us) นั่นจะทำให้เกิดการรีสตาร์ทและจะเชื่อมต่อกับ neutrino node ที่พร้อมใช้งานตัวแรกที่มีการตอบสนองเวลาที่ดีกว่า หรือใช้ neutrino peers อื่น ๆ ที่เป็นที่รู้จักดีตามที่กล่าวถึงด้านล่าง


หากคุณต้องการดูตัวเลือกการกู้คืนเพิ่มเติมสำหรับโหนด LND [โปรดอ่านคู่มือก่อนหน้าของฉัน](https://darth-coin.github.io/nodes/shtf-restore-lnd-node-en.html) ซึ่งคุณสามารถหาขั้นตอนวิธีการนำเข้า aezeed seed ไปยัง Sparrow wallet หรือวิธีอื่น ๆ ได้


**C - โหนดฝังตัว**


ในส่วนนี้เราจะพบเครื่องมือพื้นฐานบางอย่างในการจัดการโหนดแบบบูรณาการ:



- _การกู้คืนจากภัยพิบัติ_ - การสำรองข้อมูลอัตโนมัติและด้วยตนเองสำหรับช่อง LN โปรดอ่านเพิ่มเติมเกี่ยวกับวิธีการใช้คุณลักษณะนี้ในหน้าของ Zeus Docs
- _Express Graph Sync_ - แอป Zeus จะดาวน์โหลดกราฟข้อมูลซุบซิบ LN จากเซิร์ฟเวอร์เฉพาะ เพื่อการซิงโครไนซ์ที่รวดเร็วและดียิ่งขึ้น เสนอเส้นทางการชำระเงินที่ดีที่สุด คุณยังสามารถเลือกที่จะล้างข้อมูลกราฟก่อนหน้าเมื่อเริ่มต้นได้อีกด้วย
- _Peers_ - ส่วนสำหรับจัดการ neutrino peers และ 0-conf peers หากคุณมีปัญหากับการซิงค์เริ่มต้น ช่องทางไม่ออนไลน์ เป็นเพราะอุปกรณ์ของคุณมีความหน่วงสูงกับ neutrino peer ที่กำหนดไว้ ลองสลับรายการของ peers ที่ต้องการหรือเพิ่ม peer เฉพาะที่คุณรู้ว่ามีความหน่วงที่ดีกว่าสำหรับการซิงค์ เซิร์ฟเวอร์ neutrino ที่รู้จักกันดีคือ:



 - btcd1.lnolymp.us | btcd2.lnolymp.us - สำหรับภูมิภาคสหรัฐอเมริกา
 - sg.lnolymp.us - สำหรับภูมิภาคเอเชีย
 - btcd-mainnet.lightning.computer - สำหรับภูมิภาคสหรัฐอเมริกา
 - uswest.blixtwallet.com (Seattle) - สำหรับภูมิภาคสหรัฐอเมริกา
 - europe.blixtwallet.com (Germany) - สำหรับภูมิภาค EU
 - asia.blixtwallet.com - สำหรับภูมิภาคเอเชีย
 - node.eldamar.icu - สำหรับภูมิภาคสหรัฐอเมริกา
 - noad.sathoarder.com - สำหรับภูมิภาคสหรัฐอเมริกา
 - bb1.breez.technology | bb2.breez.technology - สำหรับภูมิภาคสหรัฐอเมริกา
 - neutrino.shock.network - ภูมิภาคสหรัฐอเมริกา



- _LND logs_ - เครื่องมือที่มีประโยชน์มากในการแก้ไขปัญหาโหนด LN ของคุณและควบคุมในเชิงลึกว่าเกิดอะไรขึ้นในระดับที่มีความเทคนิคมากขึ้น
- _การตั้งค่าขั้นสูง_ - เครื่องมือเพิ่มเติมเพื่อควบคุมการใช้งานโหนด LND ของคุณ:



 - _โหมดการค้นหาเส้นทาง_ - แบบสองโหมดหรือแบบล่วงหน้า วิธีการค้นหาเส้นทางที่ดีกว่าสำหรับการชำระเงิน LN ของคุณและการรีเซ็ตข้อมูลการกำหนดเส้นทางก่อนหน้า โปรดอ่านคำแนะนำที่ดีมากเกี่ยวกับการค้นหาเส้นทาง: [Pathfinding](https://docs.lightning.engineering/lightning-network-tools/lnd/pathfinding) - โดย Docs Lightning Engineering และ [LN Payment Pathfinding](https://voltage.cloud/blog/lightning-network-faq/understanding-payment-pathfinding-between-nodes-on-lightning-network/) - โดย Voltage
 - _Persistent LND_ - เปิดใช้งานโหมดนี้หากคุณต้องการให้บริการ LND ทำงานต่อเนื่องในพื้นหลังและทำให้โหนดของคุณออนไลน์ตลอด 24/7 สิ่งนี้มีประโยชน์มากหากคุณใช้ Zeus เป็น PoS ในร้านค้าขนาดเล็กหรือคุณกำลังรับทิป LN จำนวนมากผ่าน LN Address
 - _Rescan wallet_ - ตัวเลือกนี้จะเริ่มการสแกนเต็มรูปแบบของธุรกรรมทั้งหมดบนเชนของ wallet ของคุณเมื่อรีสตาร์ท เปิดใช้งานเฉพาะในกรณีที่คุณขาดธุรกรรมบางรายการใน wallet ของคุณ งานการสแกนใหม่จะใช้เวลา หลายนาที ดังนั้นโปรดอดทนและตรวจสอบบันทึกเพื่อดูรายละเอียดเพิ่มเติมเกี่ยวกับความคืบหน้าเสมอ
 - _Compact Database_ - ตัวเลือกนี้มีประโยชน์มากหากแอป Zeus ของคุณใช้พื้นที่ในอุปกรณ์มาก (ดูรายละเอียดแอปในการตั้งค่าอุปกรณ์ของคุณ) หากคุณมีกิจกรรมมากมายที่ใช้ Zeus ฉันขอแนะนำให้ทำการบีบอัดนี้บ่อยขึ้น เมื่อคุณเห็นว่าคุณมีข้อมูลมากกว่า 1-1.5GB สำหรับแอป Zeus ให้ทำการบีบอัด มันจะรีสตาร์ทและใช้เวลาสักครู่ ดังนั้นโปรดอดทนรอ
 - _ลบไฟล์ Neutrino_ - ตัวเลือกนี้เพื่อลบไฟล์นิวทริโน (พร้อมรีสตาร์ท) จะช่วยลดการใช้พื้นที่จัดเก็บข้อมูลได้มาก การลดการใช้ข้อมูลยังส่งผลกระทบอย่างมากต่อการใช้แบตเตอรี่ ลดการใช้แบตเตอรี่ โดยเฉพาะอย่างยิ่งหากคุณใช้ Zeus ในโหมดถาวร


**D - ข้อมูลโหนด**


ในส่วนนี้ คุณจะพบรายละเอียดเพิ่มเติมเกี่ยวกับสถานะของโหนด Zeus ของคุณดังนี้:



- นามแฝง - รหัสประจำตัวโหนดสั้น
- คีย์สาธารณะ - คีย์สาธารณะเต็มรูปแบบสำหรับโหนดของคุณที่จำเป็นสำหรับโหนดอื่นๆ ในการค้นหาเส้นทางไปยังโหนดของคุณ จำไว้ว่าคีย์สาธารณะนี้จะไม่ปรากฏบน LN Explorers ปกติ (Mempool, Amboss, 1ML เป็นต้น) คีย์สาธารณะนี้สามารถเข้าถึงได้ผ่านทางเพื่อนและช่องทางที่เชื่อมต่อกับ LN ของคุณเท่านั้น
- LN เวอร์ชันการใช้งาน
- เวอร์ชันแอป Zeus
- ซิงค์กับสถานะเชนและซิงค์กับสถานะแผนภูมิ - เป็นสิ่งที่สำคัญมาก แสดงสถานะที่ถูกต้องของโหนดของคุณ หากทั้งสองนี้ไม่ได้แสดงว่า “true” หมายความว่าโหนดของคุณยังคงกำลังซิงค์หรือมีปัญหาในการซิงค์ ดังนั้นแนะนำให้ตรวจสอบบันทึก LND ของคุณหรือรอสักครู่
- ความสูงของบล็อกและแฮช - แสดงบล็อกและแฮชล่าสุดที่โหนดของคุณเห็นและซิงค์แล้ว


**E - ข้อมูลเครือข่าย**


ส่วนนี้แสดงรายละเอียดเพิ่มเติมเกี่ยวกับสถานะทั่วไปของเครือข่าย Lightning ที่ดึงมาจากข้อมูลการซิงค์กราฟของคุณ: จำนวนช่องสาธารณะที่มีอยู่, จำนวนโหนด, จำนวนช่องซอมบี้ (ออฟไลน์หรือเสียชีวิต), เส้นผ่านศูนย์กลางของกราฟ, ค่าเฉลี่ยและค่าสูงสุดของ out degree สำหรับกราฟ


ข้อมูลนี้อาจมีประโยชน์ในการดีบักหรือใช้สำหรับสถิติเท่านั้น


**F - Lightning Address**


ในส่วนนี้ผู้ใช้สามารถตั้งค่าการดูแลตนเอง LN Address @zeuspay.com ได้เอง


ZEUS PAY ใช้ประโยชน์จากแฮชภาพล่วงหน้าที่ผู้ใช้สร้างขึ้น, ใบแจ้งหนี้ hodl และโครงการรับรอง Zaplocker Nostr เพื่อให้ผู้ใช้ที่อาจไม่ได้ออนไลน์ตลอด 24/7 สามารถรับการชำระเงินไปยังที่อยู่สายฟ้าคงที่ได้ ผู้ใช้เพียงแค่ต้องเข้าสู่ระบบกระเป๋าเงิน ZEUS ของพวกเขาภายใน 24 ชั่วโมงเพื่อรับการชำระเงิน มิฉะนั้นจะถูกส่งคืนไปยังผู้ส่ง


หากคุณเปิดใช้งาน "โหมดถาวร" การชำระเงินทั้งหมดไปยัง LN Address ของคุณจะได้รับทันที


เรียนรู้เกี่ยวกับการทำงานของการชำระเงิน [Zaplocker](https://github.com/supertestnet/zaplocker#how-it-works) และข้อมูลเพิ่มเติมเกี่ยวกับ [ค่าธรรมเนียม ZeusPay ที่นี่](https://docs.zeusln.app/lightning-address/fees)


**G - ที่อยู่บนเชน**


ในส่วนนี้คุณสามารถตรวจสอบที่อยู่บนเชนที่สร้างขึ้นเพื่อการควบคุมเหรียญที่ดียิ่งขึ้น


**H - รายชื่อผู้ติดต่อ**


มีการแนะนำสมุดติดต่อใหม่ใน Zeus v 0.8.0 ที่คุณสามารถใช้เพื่อส่งการชำระเงินให้กับเพื่อนและครอบครัวของคุณได้อย่างรวดเร็ว นอกจากนี้ยังมีความสามารถในการนำเข้ารายชื่อติดต่อของคุณจาก Nostr


เพียงแค่ป้อน npub Nostr ของคุณหรือที่อยู่ NIP-05 ที่มนุษย์อ่านได้ ZEUS จะทำการค้นหา Nostr สำหรับผู้ติดต่อทั้งหมดของคุณ จากนั้นคุณสามารถส่งการชำระเงินอย่างรวดเร็วไปยังผู้ติดต่อ หรือเลือกนำเข้าผู้ติดต่อทั้งหมดหรือบางส่วนไปยังสมุดรายชื่อผู้ติดต่อในเครื่องของคุณ


นี่คือวิดีโอสั้น ๆ เกี่ยวกับวิธีการตั้งค่าและใช้งานรายชื่อ Zeus ของคุณ:


**I - เครื่องมือ**


ที่นี่เรามีหมวดหมู่ย่อยต่างๆ พร้อมเครื่องมือเพิ่มเติม:



- _Accounts_ - ที่นี่คุณสามารถนำเข้าบัญชีภายนอก / กระเป๋าเงิน, กระเป๋าเงินเย็น, กระเป๋าเงินร้อน เพื่อควบคุมหรือใช้เป็นแหล่งเงินทุนภายนอกสำหรับช่องโหนด Zeus ของคุณ ฟีเจอร์นี้ยังอยู่ในขั้นทดลอง
- _เร่งความเร็วในการทำธุรกรรม_ - ฟีเจอร์นี้อาจมีประโยชน์เมื่อคุณมีธุรกรรมที่ติดอยู่ใน mempool และต้องการเพิ่มค่าธรรมเนียม คุณจะต้องให้ข้อมูลเอาต์พุตของธุรกรรมจากรายละเอียดธุรกรรมและเลือกค่าธรรมเนียมใหม่ที่ต้องการใช้ ต้องสูงกว่าค่าธรรมเนียมก่อนหน้าและต้องมีเงินทุนเพิ่มเติมใน onchain wallet ของคุณ


![Image](assets/en/05.webp)


คุณต้องไปที่ธุรกรรมที่ค้างอยู่ของคุณและคัดลอก txID outpoint จากนั้นมาที่ส่วนนี้และวางมัน แล้วเลือกค่าธรรมเนียมใหม่ที่คุณต้องการใช้เพื่อเพิ่มมัน จะมีหน้าจอใหม่ปรากฏขึ้นพร้อมค่าธรรมเนียมที่แนะนำในขณะนั้น หรือคุณสามารถตั้งค่าที่กำหนดเองได้ จำไว้ว่าต้องสูงกว่าค่าก่อนหน้า


การเก็บ UTXO ที่มี sats สูงสุด 100k ใน Zeus onchain wallet ของคุณเสมอจะดีกว่า เพื่อให้สามารถใช้ในการเพิ่มค่าธรรมเนียมเมื่อจำเป็น



- _ลงชื่อหรือยืนยัน_ - ด้วยคุณสมบัตินี้ คุณสามารถลงชื่อในข้อความเฉพาะด้วยกุญแจ wallet ของคุณ นอกจากนี้ยังสามารถใช้เพื่อยืนยันข้อความเพื่อพิสูจน์ว่ามาจากกุญแจ wallet เฉพาะเจาะจง
- _ตัวแปลงสกุลเงิน_ - เครื่องมือที่ง่ายสำหรับคำนวณอัตราแลกเปลี่ยนระหว่าง BTC และสกุลเงินเฟียตอื่น ๆ


**J - สินค้าและการสนับสนุน**


ที่นี่คุณจะพบข้อมูลเพิ่มเติมและลิงก์เกี่ยวกับ Zeus, ร้านค้าออนไลน์, ผู้สนับสนุน, โซเชียลมีเดีย


**K - ช่วยเหลือ**


ในส่วนสุดท้ายนี้ คุณจะพบลิงก์ไปยังหน้าการเอกสารของ Zeus, ปัญหาใน Github (หากคุณต้องการโพสต์บั๊กหรือคำขอโดยตรงถึงนักพัฒนาแอป), การสนับสนุนทางอีเมล



### ขั้นตอนที่ 2 - เริ่มใช้งาน ZEUS NODE



โปรดจำไว้ว่า Zeus ถูกใช้เป็นหลักในฐานะ LN wallet สำหรับการชำระเงินที่ง่ายและรวดเร็วผ่าน LN แน่นอนว่ามันยังมี wallet บนเชน แต่ควรใช้เฉพาะสำหรับการเปิด/ปิดช่องทาง LN และไม่ควรใช้สำหรับการชำระเงินทั่วไปเช่นการซื้อกาแฟ


โปรดอ่านคู่มืออื่นของฉันเกี่ยวกับ [วิธีการเป็นธนาคารของคุณเองโดยใช้ 3 ระดับของการสะสม](https://darth-coin.github.io/beginner/be-your-own-bank-en.html)


ในขณะนี้ผู้ใช้มี 2 วิธีในการเริ่มใช้ Zeus:



- ตรงไปที่ LN โดยใช้ช่อง 0-conf จาก Olympus LSP
- ฝากครั้งแรกใน onchain wallet แล้วเปิดช่องทาง LN ปกติกับเพียร์ที่คุณต้องการ


#### วิธี A - การใช้ Olympus LSP


นี่เป็นวิธีที่ง่ายและตรงไปตรงมาในการเริ่มต้นใช้งานผู้ใช้ LN ใหม่กับ Zeus อาจเป็นผู้ใช้บิทคอยน์ใหม่ทั้งหมดที่ไม่มี sats เลย ถูกแนะนำโดยเพื่อน หรือพ่อค้าใหม่ที่เริ่มต้นด้วยการชำระเงิน LN ครั้งแรกของเขา


ตามค่าเริ่มต้น Zeus จะใช้ LSP ของตัวเองคือ Olympus แต่ต่อมาคุณสามารถสลับไปใช้ LSP อื่น ๆ ที่รองรับโปรโตคอล 0-conf นี้เพื่อเปิดช่องทางได้เช่นกัน


เพียงแค่สร้างใบแจ้งหนี้บน Zeus ของคุณ (ใส่จำนวนเงินและคลิกปุ่ม “request”) คุณจะสามารถรับ sats เหล่านั้นได้ทันที

#### วิธีที่ B - การใช้ยอดคงเหลือบนเชนของคุณ


ด้วยวิธีนี้คุณสามารถเปิดช่องทางไปยังโหนด LN อื่น ๆ ได้ รวมถึง Olympus LSP เดียวกันด้วย แต่ถ้าคุณมีช่องทางกับ Olympus อยู่แล้ว แนะนำให้มีช่องทางกับโหนดอื่นด้วย เพื่อความน่าเชื่อถือที่มากขึ้นและยังสามารถใช้ MPP (การชำระเงินหลายส่วน) ได้อีกด้วย

ด้านบนเป็นตัวอย่างของการชำระใบแจ้งหนี้ LN โดยใช้ MPP ตามที่คุณเห็นที่ด้านล่างของหน้าจอคุณมี "การตั้งค่า" และกำลังเปิดหน้าแบบเลื่อนลงพร้อมรายละเอียดเพิ่มเติมสำหรับการชำระเงินที่คุณกำลังจะทำ ในหน้าจอนั้น หากคุณมีช่องทางเปิดอย่างน้อย 2 ช่อง ฟีเจอร์ MPP จะเปิดใช้งานโดยค่าเริ่มต้น นอกจากนี้คุณยังสามารถเปิดใช้งาน AMP (atomic multi-path) และตั้งค่าส่วนที่คุณต้องการได้ นี่เป็นฟีเจอร์ที่ทรงพลัง!


สำหรับโหนดส่วนตัวอย่าง Zeus ฉันขอแนะนำให้มีช่องทางที่ดี 2-3 ช่อง (สูงสุด 4-5 ช่อง) พร้อมกับ LSPs ที่ดีและสภาพคล่องที่ดีเพื่อครอบคลุมความต้องการทั้งหมดของคุณในการจ่ายหรือรับ sats ผ่าน LN [ดูคำแนะนำเพิ่มเติมเกี่ยวกับสภาพคล่องของโหนด LN ในคู่มือนี้](/nodes/managing-lightning-node-liquidity-en.html) นอกจากนี้ยังมี [คู่มือทั่วไปเกี่ยวกับสภาพคล่องของ LN](https://bitcoin.design/guide/how-it-works/liquidity/) จากทีมออกแบบ Bitcoin


การเลือกเพียร์ที่เหมาะสม ฉันรู้ว่าไม่ใช่เรื่องง่าย แม้แต่สำหรับผู้ใช้ที่มีประสบการณ์ [ดังนั้นฉันจะให้ตัวเลือกบางอย่างเพื่อเริ่มต้น](https://github.com/ZeusLN/zeus/discussions/2265) นี่คือเพียร์โหนดที่ฉันทดสอบด้วยตัวเองโดยใช้ Zeus (ฉันพยายามเชื่อมต่อเฉพาะกับโหนด LND เพื่อหลีกเลี่ยงปัญหาความเข้ากันไม่ได้)


นี่คือรายชื่อของเพื่อนร่วมโหนดที่ได้รับการรับรองสำหรับ Zeus หากคุณรู้จักโหนดที่ดี คุณสามารถเพิ่มเข้าไปในรายชื่อนั้นได้

คุณจะต้องตรวจสอบให้แน่ใจว่าช่องทางนั้นไม่ได้ประกาศ (เป็นส่วนตัว) โดยค่าเริ่มต้นตัวเลือกนี้จะปิดสำหรับช่องทางที่ประกาศแล้ว ไม่แนะนำให้เปิดใช้งานตัวเลือกนี้สำหรับโหนดที่ฝังอยู่ใน Zeus มีประโยชน์เฉพาะเมื่อคุณใช้ Zeus กับโหนดระยะไกลของคุณ ในฐานะโหนดการกำหนดเส้นทางสาธารณะเท่านั้น


ต่างจากช่องทาง LSP ที่ต้องชำระเงิน คุณจะไม่ได้รับประโยชน์จากการกำหนดเส้นทางที่ไม่มีค่าธรรมเนียมโดยการเปิดช่องทางด้วยวิธีนี้


และเสร็จสิ้น เพียงคลิกที่ปุ่ม “Open Channel” รอให้ tx ได้รับการยืนยันโดยนักขุด เมื่อช่องเปิดแล้วคุณสามารถทำธุรกรรมตามที่คุณต้องการกับ sats จากช่องของคุณ


โปรดทราบว่าช่องทางเหล่านี้จะมีสมดุลทั้งหมดอยู่ในฝั่งของคุณ ดังนั้นคุณจะไม่มีสภาพคล่องขาเข้า อย่างที่ฉันบอกไปก่อนหน้านี้ ให้แลกเปลี่ยนหรือใช้จ่าย sats บางส่วนในการซื้อของผ่าน LN เพื่อ "สร้างพื้นที่เพิ่มเติม" สำหรับการรับ


คิดถึงช่อง LN ของคุณเหมือนแก้วน้ำ คุณเทน้ำบางส่วน (sats) ลงในแก้วเปล่า (ช่องของคุณ) จนเต็ม คุณไม่สามารถเทน้ำเพิ่มได้จนกว่าคุณจะดื่มบางส่วน (ใช้จ่าย / สลับออก) เมื่อแก้วเกือบว่างเปล่า ให้เทน้ำเพิ่ม (sats) ลงไปโดยใช้การสลับเข้า [อ่านเพิ่มเติมเกี่ยวกับบริการสลับภายนอกได้ที่นี่](https://darth-coin.github.io/nodes/lightning-submarine-swaps-en.html).


มีบริการ LSP อื่น ๆ ที่ขายช่องทางขาเข้าให้คุณ: LNBig หรือ Bitrefill ฉันคิดว่ายังมีบริการอื่น ๆ แบบนี้อีกแต่จำไม่ได้ในตอนนี้


ดังนั้นหากคุณต้องการช่อง LN ที่แทบจะว่างเปล่า (ยอดคงเหลืออยู่ที่ฝั่งเพียร์ 100% ตั้งแต่เริ่มต้น) เพื่อรับการชำระเงินมากกว่าที่คุณสามารถจัดการได้ในช่องที่มีอยู่ที่เต็มแล้ว นี่อาจเป็นตัวเลือกที่ดีมาก คุณจะต้องจ่ายค่าธรรมเนียมเฉพาะสำหรับการเปิดช่องเหล่านี้และคุณจะได้รับพื้นที่ขาเข้ามากมาย



## เคล็ดลับและเทคนิค


### ขีดจำกัดการสำรองขาเข้า


ขณะนี้ เนื่องจากข้อจำกัดของรหัส LN จึงไม่สามารถรับจำนวนที่แสดงใน "Inbound" ได้อย่างแม่นยำเสมอไป ควรจำไว้เสมอว่าคุณควรทำใบแจ้งหนี้ของคุณด้วยจำนวนที่น้อยกว่าเล็กน้อย ซึ่งก็คือจำนวน "Channel Local Reserve"


![Image](assets/en/13.webp)


ตามที่คุณเห็นในภาพด้านบน "inbound" แสดงว่าฉันยังสามารถรับ 5101 sats ได้ แต่ในความเป็นจริงในขณะนี้เป็นไปไม่ได้ที่จะรับเพิ่มเติม และคุณสามารถสังเกตได้ว่ามีจำนวนเท่ากับ "Local reserve"


ดังนั้นโปรดจำไว้ว่า เมื่อคุณทำใบแจ้งหนี้เพื่อรับเงิน ให้ตรวจสอบสภาพคล่องของช่องทางของคุณและหักเงินสำรองในท้องถิ่นออกจากจำนวนนั้น หากคุณต้องการผลักดันจำนวนเงินที่สามารถรับได้ถึงขีดจำกัด

- คว้าโอกาสในช่องทางใหม่ของคุณอย่างถูกต้อง


ตัวอย่างเช่น หากคุณทราบว่าคุณจะได้รับ sats จำนวน 1M ในสัปดาห์หน้า ให้เปิดช่องทาง sats จำนวน 2M และแลกเปลี่ยนออกไปเป็น wallet บนเชนหรือเข้าสู่บัญชีผู้ดูแล LN (ชั่วคราว) 50-60% ของสภาพคล่องขาออกของคุณ ควรเตรียมพร้อมด้วยสภาพคล่องเพิ่มเติมเสมอ เมื่อคุณต้องการสภาพคล่องกลับคืนในช่องทาง Zeus ของคุณ คุณสามารถย้ายกลับจากบัญชีผู้ดูแลได้


หากคุณรู้ว่าคุณจะส่ง sats จำนวน 500k/สัปดาห์ ให้เปิดช่อง sats ขนาด 1M ด้วยวิธีนี้คุณจะยังคงมีสำรองจนกว่าคุณจะเติมมันอีกครั้ง



- หากคุณเป็นพ่อค้าและคุณจะได้รับมากกว่าที่คุณใช้จ่ายเป็นประจำเสมอ ซื้อช่องทางขาเข้าที่เฉพาะเจาะจง เป็นวิธีที่ถูกที่สุด คุณจ่ายค่าธรรมเนียมขั้นต่ำและคุณจะได้รับช่องทาง "ว่างเปล่า"



- อย่าเปิดช่องเล็ก ๆ ที่ไม่มีความหมายขนาด 50-100-300-500k sats คุณจะเติมเต็มพวกมันในเวลาไม่กี่วัน แม้ว่าคุณจะใช้พวกมันเพียงเพื่อ zaps ก็ตาม เปิดช่องที่ใหญ่กว่าและแตกต่างกัน ไม่ใช่แค่ช่องเดียว


เมื่อคุณเปิดช่องทางที่ใหญ่ขึ้น คุณสามารถใช้การแลกเปลี่ยนใต้น้ำภายนอกเพื่อย้าย sats ไปยังวอลเล็ตบนเชนของคุณได้เสมอ (รวมถึงกลับไปยัง Zeus บนเชน) การรักษาสมดุลระหว่างสภาพคล่องเข้าและออกเป็นสิ่งที่ดี และคุณยังสามารถ "ใช้ซ้ำ" sats เหล่านั้นเพื่อเปิดช่องทางเพิ่มเติมได้หากคุณต้องการ


### ใบแจ้งหนี้ที่ห่อหุ้ม


หากคุณต้องการเพิ่มความเป็นส่วนตัวเมื่อคุณได้รับ คุณสามารถใช้วิธี “wrapped invoice” ได้ การเตือน: เพื่อที่จะทำเช่นนี้ได้ คุณจำเป็นต้องมีช่องทางกับ Olympus LSP Wrapped invoices จะ “ซ่อน” ปลายทางสุดท้าย (โหนด Zeus ของคุณ) และแสดงโหนด LSP ของคุณเป็นปลายทางให้กับผู้ชำระเงิน


ในการรับใบแจ้งหนี้แบบห่อ ให้ไปที่หน้าจอแป้นพิมพ์หลัก ใส่จำนวนเงินและกดขอ จะปรากฏรหัส QR ปกติสำหรับใบแจ้งหนี้ของคุณ ตอนนี้ คลิกที่ปุ่ม “X” ด้านขวาบนและคุณจะถูกเปลี่ยนเส้นทางไปยังตัวเลือกเพิ่มเติมสำหรับใบแจ้งหนี้


![Image](assets/en/14.webp)


ตอนนี้คุณจะต้องเปิดใช้งานตัวเลือกนั้นด้านบน “Enable LSP” และกดปุ่ม “Create invoice” ตัวเลือกนั้นจะสร้างใบแจ้งหนี้ที่ห่อหุ้มและจำไว้ว่าจะมีการคิดค่าธรรมเนียมเล็กน้อย


### ใบแจ้งหนี้พร้อมคำแนะนำเส้นทาง


นี่เป็นคุณสมบัติที่มีประโยชน์มากหากคุณต้องการจัดการสภาพคล่องของช่องทางขาเข้าหลายช่องทาง ในทางปฏิบัติคุณสามารถระบุได้ว่าต้องการรับ sats จากใบแจ้งหนี้ไปยังช่องทางขาเข้าช่องทางใด


คุณสมบัตินี้สามารถใช้สำหรับการปรับสมดุลแบบวงกลมได้เช่นกัน เมื่อคุณต้องการย้ายสภาพคล่องจากช่องทางที่เต็มไปยังช่องทางที่ขาดแคลน


วิธีสร้างใบแจ้งหนี้พร้อมคำแนะนำเส้นทาง?



- บนหน้าจอหลัก เลื่อนไปทางขวาที่ลิ้นชัก LN และคลิกที่ "รับ"
- ในหน้าการตั้งค่าใบแจ้งหนี้ ให้เลื่อนไปที่ส่วนล่างและเปิดใช้งานปุ่ม "แทรกคำแนะนำเส้นทาง" จากนั้นเลือกแท็บ "กำหนดเอง" มันจะเปิดหน้าจอที่มีช่องทางทั้งหมดที่คุณมีอยู่ เลือกช่องทางที่คุณต้องการรับ
- กรอกข้อมูลใบแจ้งหนี้อื่นๆ ทั้งหมด จำนวนเงิน บันทึกช่วยจำ ฯลฯ และคลิกที่ "สร้างใบแจ้งหนี้"
- การชำระใบแจ้งหนี้นั้นจะนำ sats เข้าสู่ช่องทางที่ระบุ


หากคุณต้องการชำระเงินให้กับตัวเองในใบแจ้งหนี้นั้น (การปรับสมดุลแบบวงกลม) เมื่อคุณชำระเงินจากโหนด Zeus เดียวกัน ในหน้าจอการชำระเงิน ให้เลือกช่องทางขาออก (ช่องทางที่มีสภาพคล่องมากกว่า) ที่คุณต้องการใช้เป็นการส่งการชำระเงิน


### ชำระเงินด้วย Keysend


Keysend เป็นคุณสมบัติของ LN ที่ถูกประเมินค่าต่ำเกินไป และผู้ใช้ควรใช้งานบ่อยขึ้น


[Keysend](https://docs.lightning.engineering/lightning-network-tools/lnd/send-messages-with-keysend) อนุญาตให้ผู้ใช้ในเครือข่าย Lightning ส่งการชำระเงินไปยังผู้อื่นโดยตรงไปยัง public key ของพวกเขา ตราบใดที่โหนดของพวกเขามีช่องทางสาธารณะและเปิดใช้งาน keysend แล้ว Keysend ไม่จำเป็นต้องให้ผู้รับออกใบแจ้งหนี้


แล้วคุณจะทำอย่างนั้นกับ Zeus ได้อย่างไร?


เพียงแค่สแกนหรือคัดลอก nodeID ปลายทาง (หรือใช้ Zeus contacts เพื่อบันทึก node ปลายทางที่ใช้ประจำเป็นรายชื่อผู้ติดต่อ) จากนั้นจากหน้าจอหลักของ Zeus ให้คลิกที่ปุ่ม “ส่ง” ในหน้าจอนั้นให้วาง nodeID หรือเลือกจากรายชื่อผู้ติดต่อของคุณ


ใส่จำนวน sats ข้อความถ้าจำเป็น (ใช่ คุณสามารถใช้มันเป็นการแชทลับผ่าน LN ได้เช่นกัน) แล้วคลิกที่ปุ่ม “ส่ง” เสร็จสิ้น!


![Image](assets/en/15.webp)


หากคุณมีช่องทางโดยตรงกับเพียร์ปลายทาง จะไม่มีค่าธรรมเนียมใด ๆ เกี่ยวข้อง


หากคุณไม่มีช่องทางโดยตรงกับเพียร์ปลายทาง การชำระเงิน keysend จะจ่ายค่าธรรมเนียมเหมือนการชำระเงินใบแจ้งหนี้ LN ปกติ ซึ่งจะถูกส่งผ่านเส้นทางปกติเหมือนการชำระเงินอื่น ๆ เพียงแต่จำไว้ว่าจะไม่มีการทิ้งร่องรอยใด ๆ ในฐานะใบแจ้งหนี้ LN
