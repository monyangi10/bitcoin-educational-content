เมื่อสร้าง Bitcoin wallet คุณจะถูกขอให้จดวลีช่วยจำ ซึ่งมักประกอบด้วย 12 หรือ 24 คำ วลีนี้ช่วยให้คุณกู้คืนการเข้าถึงบิตคอยน์ของคุณในกรณีที่อุปกรณ์ที่โฮสต์ wallet ของคุณสูญหาย เสียหาย หรือถูกขโมย ก่อนที่คุณจะเริ่มใช้ Bitcoin wallet ใหม่ของคุณ สิ่งสำคัญมากคือต้องตรวจสอบความถูกต้องของวลีช่วยจำนี้ วิธีที่ดีที่สุดในการทำเช่นนี้คือการทดสอบการกู้คืนแบบแห้ง


การทดสอบนี้เกี่ยวข้องกับการจำลองการกู้คืน wallet ก่อนที่จะฝากบิตคอยน์ใด ๆ ลงไป ตราบใดที่ wallet ว่างเปล่า เราจะจำลองสถานการณ์ที่อุปกรณ์ที่เก็บกุญแจของเราหายไป และสิ่งที่เหลืออยู่คือวลีช่วยจำของเราเพื่อพยายามกู้คืนบิตคอยน์ของเรา


![RECOVERY TEST](assets/notext/01.webp)


## วัตถุประสงค์คืออะไร?


กระบวนการทดสอบนี้ช่วยให้คุณสามารถตรวจสอบได้ว่าการสำรองข้อมูลทางกายภาพของวลีช่วยจำของคุณ ไม่ว่าจะเป็นบนกระดาษหรือโลหะนั้นใช้งานได้หรือไม่ การล้มเหลวในระหว่างการทดสอบการกู้คืนนี้บ่งชี้ถึงข้อผิดพลาดในการสำรองข้อมูลของวลี ซึ่งอาจทำให้บิตคอยน์ของคุณตกอยู่ในความเสี่ยง ในทางกลับกัน หากการทดสอบประสบความสำเร็จ จะยืนยันได้ว่าวลีช่วยจำของคุณสามารถใช้งานได้อย่างเต็มที่ และคุณสามารถรักษาความปลอดภัยของบิตคอยน์ได้อย่างสบายใจโดยใช้ wallet นี้


การทดสอบการกู้คืนแบบแห้งมีข้อดีสองประการ ไม่เพียงแต่ช่วยให้คุณตรวจสอบความถูกต้องของวลีช่วยจำของคุณ แต่ยังเปิดโอกาสให้คุณได้ทำความคุ้นเคยกับกระบวนการกู้คืน wallet ด้วย วิธีนี้คุณจะค้นพบความยากลำบากที่อาจเกิดขึ้นก่อนที่สถานการณ์จริงจะเกิดขึ้น ในวันที่คุณต้องกู้คืน wallet จริง ๆ คุณจะเครียดน้อยลง เพราะคุณจะรู้กระบวนการแล้ว ลดความเสี่ยงของข้อผิดพลาด นั่นคือเหตุผลที่สำคัญที่ไม่ควรละเลยขั้นตอนการทดสอบนี้และใช้เวลาที่จำเป็นในการทำให้ถูกต้อง


## การทดสอบการกู้คืนคืออะไร?


กระบวนการทดสอบนั้นค่อนข้างง่าย:


- หลังจากสร้าง Bitcoin wallet ใหม่ของคุณ และก่อนที่จะฝาก satoshis ครั้งแรกของคุณ ให้จดบันทึกข้อมูลพยานเช่น xpub ที่อยู่รับแรก หรือแม้กระทั่งลายนิ้วมือคีย์หลัก;
- จากนั้น ลบ wallet ที่ยังว่างอยู่โดยเจตนา เช่น โดยการรีเซ็ตฮาร์ดแวร์ wallet ของคุณไปที่การตั้งค่าจากโรงงาน;
- ถัดไป, จำลองการกู้คืน wallet ของคุณโดยใช้เฉพาะการสำรองข้อมูลบนกระดาษของวลีช่วยจำและ passphrase ของคุณหากคุณใช้;
- สุดท้าย ตรวจสอบว่าข้อมูลพยานตรงกับข้อมูลของ wallet ที่สร้างใหม่หรือไม่ หากข้อมูลตรงกัน คุณสามารถมั่นใจได้ในความน่าเชื่อถือของการสำรองข้อมูลทางกายภาพของคุณ และคุณสามารถส่งบิตคอยน์แรกของคุณไปยัง wallet นี้ได้

โปรดระวัง ในระหว่างการทดสอบการกู้คืน **คุณต้องใช้เครื่องเดียวกันที่ตั้งใจจะใช้สำหรับ wallet สุดท้ายของคุณ** เพื่อไม่ให้เพิ่มพื้นผิวการโจมตีของ wallet ของคุณ ตัวอย่างเช่น หากคุณสร้าง wallet บน Trezor Safe 5 ให้แน่ใจว่าทำการทดสอบการกู้คืนบน Trezor Safe 5 เดียวกันนี้ สิ่งสำคัญคือต้องไม่ป้อนวลีการกู้คืนของคุณลงในซอฟต์แวร์อื่นใด เนื่องจากจะทำให้ความปลอดภัยที่ฮาร์ดแวร์ wallet ของคุณมอบให้ลดลง แม้ว่า wallet จะยังว่างอยู่ก็ตาม


## วิธีการทดสอบการกู้คืน?


ในบทแนะนำนี้ ฉันจะอธิบายวิธีการทดสอบการกู้คืนบนซอฟต์แวร์ Bitcoin wallet โดยใช้ Sparrow Wallet (สำหรับ wallet ที่ร้อน) อย่างไรก็ตาม กระบวนการยังคงเหมือนเดิมสำหรับอุปกรณ์ประเภทอื่น ๆ อีกครั้ง **หากคุณกำลังใช้ฮาร์ดแวร์ wallet อย่าทำการทดสอบการกู้คืนบน Sparrow Wallet** (ดูส่วนก่อนหน้า)


ฉันเพิ่งสร้าง wallet ใหม่บน Sparrow Wallet ขึ้นมา ตอนนี้ฉันยังไม่ได้ส่งบิตคอยน์ใด ๆ ไปยังมัน มันยังว่างเปล่า


![RECOVERY TEST](assets/notext/02.webp)


ฉันได้จดวลีช่วยจำ 12 คำของฉันลงบนกระดาษอย่างระมัดระวัง และเนื่องจากฉันต้องการเพิ่มความปลอดภัยให้กับ wallet นี้ ฉันจึงได้ตั้งค่า BIP39 passphrase ที่ฉันได้บันทึกไว้บนกระดาษอีกแผ่นหนึ่ง:


```txt
1. shield
2. brass
3. sentence
4. cube
5. marble
6. glad
7. satoshi
8. door
9. project
10. panic
11. prepare
12. general
```


```text
Passphrase: YfaicGzXH9t5C#g&47Kzbc$JL
```


***เห็นได้ชัดว่าคุณไม่ควรแชร์วลีช่วยจำและ passphrase ของคุณบนอินเทอร์เน็ต ซึ่งต่างจากที่ฉันกำลังทำในบทแนะนำนี้ wallet ตัวอย่างนี้จะไม่ถูกใช้งานและจะถูกลบเมื่อสิ้นสุดบทแนะนำนี้***


ฉันจะจดบันทึกข้อมูลพยานจาก wallet ของฉันในร่าง คุณสามารถเลือกข้อมูลชิ้นส่วนต่าง ๆ ได้ เช่น ที่อยู่รับแรก, xpub, หรือรอยนิ้วมือของคีย์หลัก ส่วนตัวแล้ว ฉันแนะนำให้เลือกที่อยู่รับแรก ซึ่งจะช่วยให้คุณตรวจสอบได้ว่าคุณสามารถหาวิถีการแตกแขนงแรกที่สมบูรณ์ซึ่งนำไปสู่ที่อยู่นี้ได้


บน Sparrow คลิกที่แท็บ "*Addresses*"


![RECOVERY TEST](assets/notext/03.webp)


จากนั้น จดที่อยู่รับแรกสุดของ wallet ของคุณลงบนกระดาษ ในตัวอย่างของฉัน ที่อยู่นั้นคือ:


```txt
tb1qxv56mma5x5r7uhdkn0ldvcx6m0gj6f3kre0gwd
```


หลังจากจดบันทึกข้อมูลแล้ว ให้ไปที่เมนู "*File*" จากนั้นเลือก "*Delete Wallet*". ฉันขอเตือนคุณอีกครั้งว่า Bitcoin wallet ของคุณต้องว่างเปล่าก่อนที่จะดำเนินการนี้ต่อไป


![RECOVERY TEST](assets/notext/04.webp)


หาก wallet ของคุณว่างเปล่าจริง ๆ โปรดยืนยันการลบ wallet ของคุณ

![RECOVERY TEST](assets/notext/16.webp)


หากที่อยู่ที่ได้รับตรงกัน การทดสอบการกู้คืนของคุณประสบความสำเร็จ และคุณสามารถใช้ Bitcoin wallet ใหม่ของคุณได้ หากไม่ตรงกัน อาจบ่งชี้ถึงข้อผิดพลาดในการเลือกประเภทสคริปต์ ซึ่งทำให้เส้นทางการอนุมานไม่ถูกต้อง หรือปัญหากับการสำรองข้อมูลวลีช่วยจำหรือ passphrase ของคุณ ในทั้งสองกรณี ฉันขอแนะนำอย่างยิ่งให้เริ่มต้นใหม่ตั้งแต่ต้นและสร้าง Bitcoin wallet ใหม่เพื่อหลีกเลี่ยงความเสี่ยงใดๆ ในครั้งนี้ ให้จดบันทึกวลีช่วยจำโดยไม่มีข้อผิดพลาด

ขอแสดงความยินดี ตอนนี้คุณพร้อมแล้วในการทำการทดสอบการกู้คืน! ฉันแนะนำให้คุณทำให้กระบวนการนี้เป็นมาตรฐานสำหรับการสร้างกระเป๋าเงิน Bitcoin ทั้งหมดของคุณ หากคุณพบว่าบทแนะนำนี้มีประโยชน์ ฉันจะขอบคุณมากหากคุณสามารถกดไลค์ด้านล่างได้ อย่าลังเลที่จะแบ่งปันบทความนี้บนเครือข่ายสังคมของคุณ ขอบคุณมาก!
