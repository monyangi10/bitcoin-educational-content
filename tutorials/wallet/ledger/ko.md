# Ledger을 Sparrow에 연결 Bitcoin Wallet(쓰기 가이드)


다른 글인 "Bitcoin 하드웨어 지갑 사용하기"를 먼저 읽어보시기 바랍니다. 여기서는 몇 가지 단계를 훑어보고 Ledger과 관련된 내용을 위주로 설명하겠습니다.


## 디바이스 설정


Ledger에는 자체 USB 케이블이 함께 제공됩니다. 아무 케이블이나 사용하지 말고 반드시 이 케이블을 사용하세요. 일부 USB 케이블은 전원 전용입니다. 이 케이블은 데이터와 전원을 모두 전송합니다. 휴대폰 충전용 USB 케이블을 주변에 두고 디바이스를 사용했을 때 디바이스 연결에 실패했습니다.


컴퓨터에 연결하면 기기의 전원이 켜집니다.


![image](assets/1.webp)


옵션을 순환합니다. 다음이 표시됩니다


1. 새 디바이스로 설정

2. 복구 문구에서 복원


기본적으로 디바이스에서 seed을 생성해 줄 것인지, 아니면 이미 사용하고 싶은 seed이 있는지 묻는 것입니다. seed을 직접 만드는 것이 가장 좋지만 안전하게 만드는 방법은 매우 고급이며 이 도움말의 범위를 벗어납니다. "새 디바이스로 설정"을 선택합니다


그러면 PIN을 선택하라는 메시지가 표시됩니다. 이는 Bitcoin seed에는 포함되지 않으며 이 장치에만 해당됩니다. 그러면 디바이스가 잠깁니다.


그러면 순환하며 적어야 하는 24개의 단어가 표시됩니다.


이상하게도 마지막에 도달하면 "왼쪽을 눌러 단어를 확인"이라고 표시됩니다. 이는 진행을 확인하는 방법이 아니라 뒤로 돌아가서 단어를 다시 볼 수 있다는 의미입니다. 대신 오른쪽을 누르고 왼쪽과 오른쪽을 동시에 눌러 확인합니다.


다음 단계는 매우 성가신 부분입니다. 24개의 단어가 섞여 있고 각 선택에 대해 모든 단어를 순환하여 1~24개의 단어를 각각 확인해야 합니다. 완료되면 버튼 두 개를 눌러 확인하고 계속할 수 있습니다.


![image](assets/2.webp)


대시보드에 설정 버튼과 앱을 설치할 수 있는 더하기 기호 버튼이 있는 것을 볼 수 있습니다. 하지만 먼저 Ledger Live에 연결해야 합니다. 다음에 설명하겠습니다..

상단에는 운영 체제에 따라 필요한 소프트웨어 패키지를 선택할 수 있는 링크가 있습니다. 적절한 것을 클릭하여 다운로드합니다.


다음으로 보안을 강화하기 위해 다운로드의 Hash을 확인합니다. Ledger는 사용 가능한 각 파일의 Hash을 게시합니다. 이제 다운로드의 Hash을 생성하고 출력을 비교합니다. 파일이 변조되지 않았는지 확인하려면 동일해야 합니다.


Mac의 경우 터미널을 열고, Windows의 경우 CMD를 열고 다음 명령을 입력한 후 Enter 키를 누릅니다:


```bash
cd Downloads
```


```bash
shasum -a 512 ledger-live-desktop-2.32.2-mac.dmg # <--- For Mac
certutil -hashfile ledger-live-desktop-2.32.2-win.exe SHA512 # <--- For Windows
```


화살표 뒤에 명령이 시작되는 것이 분명하시길 바랍니다. 이 문서가 오래된 경우 명령어의 파일 이름을 다운로드한 파일 이름으로 정확히 변경해야 합니다. 각 명령 뒤에 <Enter> 키를 눌러야 합니다. 여기에 표시된 명령은 웹 브라우저에서 한 줄에 맞지 않을 수 있습니다. 모두 한 줄에 입력되었음을 참고하세요.


Hash의 출력을 살펴보고 GitHub에 게시된 것과 동일한지 확인합니다.


이상적으로는 게시된 해시가 가짜가 아닌지 확인하고 싶을 것입니다. GPG 서명으로 이를 수행할 수 있지만 이 글의 범위를 벗어납니다. 이에 대해 자세히 알아보고 싶다면(그리고 언젠가는 알아볼 것을 권장합니다) 이 글을 살펴보세요.

## passphrase 추가


이제 Bitcoin 앱이 생겼으니 seed 문구에 passphrase을 추가할 수 있습니다. seed이 처음 만들어졌을 때는 Bitcoin 앱이 없었고 Ledger 라이브에 연결해야 했기 때문에 이전에는 그렇게 할 수 없었습니다.


디바이스 내의 '설정' 메뉴로 이동한 다음 '보안' 하위 메뉴로 이동합니다. 그런 다음 passphrase을 선택합니다. "고급 기능"이 표시됩니다. 오른쪽 버튼을 클릭하면 "매뉴얼 읽기..."가 표시되고 오른쪽 버튼을 한 번 더 클릭하면 "뒤로"가 표시됩니다. 하지만 그게 끝이 아닙니다. 직관적으로 그렇게 생각할 수 있지만 오른쪽 버튼을 다시 클릭하세요. "passphrase 설정"이 표시됩니다.


"비밀번호에 연결" 또는 "임시로 설정"을 선택할 수 있습니다. 저는 'PIN에 연결'을 추천합니다. 이렇게 하면 장치를 처음 켤 때 입력한 PIN에 따라 다른 지갑에 액세스할 수 있습니다. "임시로 설정"을 선택하면 Wallet에 액세스할 때마다 passphrase를 입력해야 하지만 항상 기본 PIN을 사용하게 됩니다.


passphrase를 입력하고 확인합니다.


"현재 PIN"을 입력하라는 메시지가 표시됩니다. 이것은 새 passphrase에 연결하려는 PIN이 아닙니다. 이 세션을 위해 디바이스의 전원을 켤 때 입력한 PIN입니다.


이제 뒤로 옵션을 몇 번 선택하면 메인 메뉴로 나갈 수 있습니다.


## Wallet 시청하기


이전 글에서 Sparrow wallet을 다운로드하고 확인하는 방법과 자체 노드 또는 퍼블릭 노드에 연결하는 방법에 대해 설명했습니다. 이 가이드를 따라야 합니다:



- Bitcoin core 설치 ( https://armantheparman.com/bitcoincore/ )



- Sparrow Bitcoin Wallet 설치(https://armantheparman.com/download-Sparrow/)



- Sparrow Bitcoin Wallet를 Bitcoin core에 연결(https://armantheparman.com/sparrowcore/)


Sparrow Bitcoin Wallet을 사용하는 대안으로 Electrum Desktop Wallet이 있지만 대부분의 사람들에게 가장 적합하다고 판단하여 Sparrow Bitcoin Wallet에 대해 설명하겠습니다. 고급 사용자는 일렉트럼을 대안으로 사용할 수 있습니다.


이제 Ledger을 로드하고 Wallet에 passphrase를 연결합니다. 이 Wallet은 Ledger Live에 디바이스를 연결한 후에 생성되었기 때문에 Ledger Live에 노출된 적이 없습니다. 새로운 비공개 Wallet을 노출하지 않으려면 다시는 Ledger Live에 연결하지 마세요.

## 수신


Bitcoin을 받으려면 왼쪽의 주소 탭으로 이동하여 수신할 주소 중 하나를 선택하세요. 원하는 Address를 마우스 오른쪽 버튼으로 클릭하고 "Address 복사"를 선택합니다. 그런 다음 송금할 Exchange로 이동하여 그곳에 붙여넣습니다. 또는 Address를 결제에 사용할 수 있는 고객에게 줄 수도 있습니다.


Wallet를 처음 사용할 때는 아주 적은 양을 받아 Wallet 내에서 다른 Address에 사용하거나 다시 Exchange에 사용하는 연습을 통해 Wallet가 예상대로 작동하는지 확인해야 합니다.


그렇게 한 후에는 적어 놓은 단어를 백업해야 합니다. 사본 한 장만으로는 충분하지 않습니다. 최소한 두 장의 종이 사본(금속 사본이 더 좋음)을 준비하여 보안이 잘 갖춰진 두 곳에 보관하세요. 이렇게 하면 자연재해로 인해 한 번의 사고로 HWW와 종이가 파괴될 위험을 줄일 수 있습니다. 이에 대한 자세한 내용은 "Bitcoin 하드웨어 지갑 사용"을 참조하세요.


## 보내기


![image](assets/22.webp)


결제할 때 '지불 대상' 필드에 결제할 Address을 붙여넣어야 합니다. 라벨은 지갑의 기록을 위한 것이므로 비워둘 수는 없지만 Sparrow에서는 이를 허용하지 않으므로 그냥 입력하세요(본인만 볼 수 있음). 금액을 입력하면 원하는 수수료를 수동으로 조정할 수도 있습니다.


Wallet는 HWW가 연결되지 않으면 트랜잭션에 서명할 수 없습니다. 트랜잭션을 수신하여 서명하고 서명된 상태로 돌려주는 것이 HWW의 역할입니다. 장치에서 서명할 때 지불하려는 Address과 컴퓨터 화면에서 받는 Invoice가 동일한지 육안으로 확인하세요(예: 특정 Address을 지불하라는 이메일을 받았을 수 있음).


또한 결제 금액보다 큰 Coin를 사용하도록 선택하면 나머지는 지갑의 변경 주소 중 하나로 다시 전송된다는 점에 유의하세요. 일부 사람들은 이 사실을 모르고 공개 Blockchain에서 자신의 거래를 조회한 후 일부 Bitcoin이 공격자의 Address로 전송되었다고 생각했지만 실제로는 자신의 잔돈 Address이었습니다.


## 펌웨어


펌웨어를 업데이트하려면 Ledger Live에 연결해야 합니다. 이렇게 하려면 먼저 장치를 초기화하고 장치를 복원할 수 있는 백업 단어와 passphrase이 있는지 확인해야 합니다. 제가 디바이스를 먼저 초기화하는 것을 선호하는 이유는 펌웨어를 업데이트하려면 디바이스를 Ledger Live에 연결해야 하고, 새 Wallet(passphrase이 있는 디바이스)를 Ledger Live에 노출하고 싶지 않기 때문입니다. Ledger Live에 연결할 때 Ledger이 장치에서 제 공개 키 정보를 추출하지 않는다고 믿지 못합니다. 그들은 그렇지 않다고 주장하지만 코드를 읽고 내부 하드웨어를 이해하지 않는 한 직접 확인할 수 없습니다.
