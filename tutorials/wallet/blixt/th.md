คู่มือนี้อุทิศให้กับผู้ใช้ใหม่ทุกคนที่ต้องการเริ่มใช้ Bitcoin Lightning Network (LN) ในรูปแบบที่เปิดเผยฟรีและไม่ต้องมีการควบคุมดูแลอย่างเต็มที่


ใช้ [Blixt Wallet](https://blixtwallet.com/), โหนด LN เต็มรูปแบบบนมือถือของคุณ ไม่ว่าคุณจะอยู่ที่ไหนก็ตาม


หากคุณไม่เคยใช้ Bitcoin Lightning Network มาก่อน [กรุณาอ่านคำอธิบายง่ายๆ เกี่ยวกับ Lightning Network (LN)](https://darth-coin.github.io/beginner/ln-airport-analogy-en.html) ก่อนเริ่มต้น


## แง่มุมที่สำคัญ:



- Blixt เป็นโหนดส่วนตัว ไม่ใช่โหนดการกำหนดเส้นทาง! โปรดจำไว้ว่า: นั่นหมายความว่า ช่องทาง LN ทั้งหมดใน Blixt จะไม่ถูกประกาศในกราฟ LN (เรียกว่าช่องทางส่วนตัว) นั่นหมายความว่า โหนดนี้จะไม่ทำการกำหนดเส้นทางการชำระเงินของผู้อื่นผ่านโหนด Blixt โหนด Blixt นี้ไม่ใช่สำหรับการกำหนดเส้นทาง ขอย้ำอีกครั้ง ใช้เพื่อจัดการช่องทาง LN ของคุณเองและทำการชำระเงิน LN ของคุณเองอย่างเป็นส่วนตัวเมื่อคุณต้องการ โหนด Blixt นี้จำเป็นต้องออนไลน์และซิงค์เฉพาะก่อนที่คุณจะทำธุรกรรม นั่นคือเหตุผลที่คุณจะเห็นไอคอนด้านบนที่แสดงสถานะการซิงค์ ใช้เวลาเพียงไม่กี่อึดใจ ขึ้นอยู่กับระยะเวลาที่คุณเก็บมันออฟไลน์



- Blixt กำลังใช้ LND (aezeed) เป็น backend ของ wallet ดังนั้นอย่าพยายามนำเข้ากระเป๋าเงินบิตคอยน์ประเภทอื่นเข้าไป [ที่นี่คุณสามารถอธิบายประเภทของ wallet mnemonic seeds](https://coldbit.com/what-types-of-mnemonic-seeds-are-used-in-bitcoin/) และนี่คือ [รายการที่ครอบคลุมมากขึ้นของกระเป๋าเงินทุกประเภท](https://walletsrecovery.org/) ดังนั้นหากคุณเคยมีโหนด LND มาก่อน คุณสามารถนำเข้า seed และ backup.channels เข้าไปใน Blixt [ตามที่อธิบายไว้ในคู่มือนี้](https://darth-coin.github.io/nodes/shtf-restore-lnd-node-en.html)



- เมื่อสิ้นสุดคู่มือนี้ คุณจะพบส่วนพิเศษที่มี ["เคล็ดลับและเทคนิค"](https://darth-coin.github.io/wallets/getting-started-blixt-wallet-en.html#tips)



- ลิงก์สำคัญของ Blixt - ดูได้ที่ท้ายคู่มือนี้ กรุณาบุ๊กมาร์กไว้

## Blixt - การติดต่อครั้งแรก


ดังนั้น... แม่ของดาร์ธตัดสินใจเริ่มใช้ LN กับ Blixt การตัดสินใจที่ยาก แต่ฉลาด Blixt เหมาะสำหรับคนฉลาดและผู้ที่ต้องการเรียนรู้เพิ่มเติมเกี่ยวกับการใช้ LN อย่างลึกซึ้ง


![blixt](assets/en/01.webp)


ดาร์ธเตือนแม่ของเขา:


"*แม่คะ ถ้าแม่เริ่มใช้ Blixt LN Node แม่จะต้องรู้ก่อนว่า Lightning Network คืออะไรและทำงานอย่างไร อย่างน้อยในระดับพื้นฐาน [นี่คือรายการทรัพยากรเกี่ยวกับ Lightning Network ที่หนูรวบรวมไว้](https://blixtwallet.github.io/faq#what-is-ln) กรุณาอ่านก่อนนะคะ*"


แม่ของดาร์ธอ่านแหล่งข้อมูลและทำขั้นตอนแรกของเธอ: ติดตั้ง Blixt บนอุปกรณ์ Android เครื่องใหม่ของเธอ Blixt ยังมีให้สำหรับ iOS และ macOS (เดสก์ท็อป) แต่ไม่ใช่สำหรับแม่ของดาร์ธ... อย่างไรก็ตาม แนะนำให้ใช้เวอร์ชันใหม่ของ Android อย่างน้อย 9 หรือ 10 เพื่อความเข้ากันได้และประสบการณ์ที่ดียิ่งขึ้น การรันโหนด LN เต็มรูปแบบบนอุปกรณ์เคลื่อนที่ไม่ใช่เรื่องง่ายและอาจใช้พื้นที่ (ขั้นต่ำ 600MB) และหน่วยความจำ


เมื่อคุณเปิด Blixt หน้าจอ “ยินดีต้อนรับ” จะแสดงตัวเลือกบางอย่างให้คุณ:


![blixt](assets/en/02.webp)


ที่มุมขวาบน คุณจะเห็นจุด 3 จุดที่เปิดเมนูพร้อมด้วย:



- "เปิดใช้งาน Tor" - ผู้ใช้สามารถเริ่มต้นด้วยเครือข่าย Tor โดยเฉพาะหากต้องการกู้คืนโหนด LND เก่าที่เคยทำงานร่วมกับเพื่อน Tor เท่านั้น
- "ตั้งค่าโหนด Bitcoin" - หากผู้ใช้ต้องการเชื่อมต่อกับโหนดของตนเองโดยตรง เพื่อซิงค์บล็อกผ่าน Neutrino สามารถทำได้ทันทีจากหน้าจอต้อนรับ ตัวเลือกนี้ยังดีในกรณีที่การเชื่อมต่ออินเทอร์เน็ตหรือ Tor ของคุณไม่เสถียรพอที่จะเชื่อมต่อกับโหนดบิตคอยน์เริ่มต้น (node.blixtwallet.com)
- ในไม่ช้าจะมีการเพิ่มภาษาที่นั่น ดังนั้นผู้ใช้สามารถเริ่มต้นด้วยภาษาที่สะดวกสบายได้ทันที หากคุณต้องการมีส่วนร่วมในโครงการโอเพ่นซอร์สนี้ด้วยการแปลเป็นภาษาอื่น ๆ [กรุณาเข้าร่วมที่นี่](https://explore.transifex.com/blixt-wallet/blixt-wallet/)


### ตัวเลือก A - สร้าง wallet ใหม่


หากคุณเลือกที่จะ "สร้าง wallet ใหม่" คุณจะถูกเปลี่ยนเส้นทางไปยังหน้าจอหลักของ Blixt Wallet โดยตรง


นี่คือ "cockpit" ของคุณและยังเป็น "Main LN Wallet" ดังนั้นโปรดทราบว่ามันจะแสดงเฉพาะยอดคงเหลือของ LN wallet ของคุณเท่านั้น wallet บนเชนจะแสดงแยกต่างหาก (ดู C)


![blixt](assets/en/03.webp)


A - ไอคอนตัวบ่งชี้การซิงค์ของ Blixt blocks นี่คือสิ่งที่สำคัญที่สุดสำหรับโหนด LN: การซิงค์กับเครือข่าย หากไอคอนนั้นยังคงทำงานอยู่ หมายความว่าโหนดของคุณยังไม่พร้อม! ดังนั้นจงอดทน โดยเฉพาะอย่างยิ่งสำหรับการซิงค์เริ่มต้นครั้งแรก อาจใช้เวลาถึง 6-8 นาที ขึ้นอยู่กับอุปกรณ์และการเชื่อมต่ออินเทอร์เน็ตของคุณ


คุณสามารถคลิกมันและดูสถานะของการซิงค์:


![blixt](assets/en/04.webp)


นอกจากนี้ คุณสามารถคลิกที่ปุ่ม “Show LND Log” (A) หากคุณต้องการดูและอ่านรายละเอียดทางเทคนิคเพิ่มเติมของบันทึก LND แบบเรียลไทม์ ซึ่งมีประโยชน์มากสำหรับการดีบักและเรียนรู้เพิ่มเติมเกี่ยวกับการทำงานของ LN


B - ที่นี่คุณสามารถเข้าถึงการตั้งค่าทั้งหมดของ Blixt และมีมากมาย! Blixt นำเสนอฟีเจอร์และตัวเลือกที่หลากหลายเพื่อจัดการโหนด LN ของคุณอย่างมืออาชีพ ตัวเลือกทั้งหมดนั้นอธิบายอย่างละเอียดใน “[Blixt Features Page](https://blixtwallet.github.io/features#blixt-options) - เมนูตัวเลือก”


C - ที่นี่คุณมีเมนู “Magic Drawer” [อธิบายรายละเอียดเพิ่มเติมที่นี่](https://blixtwallet.github.io/features#blixt-drawer) นี่คือ “Onchain Wallet” (B), Lightning Channels (C), รายชื่อผู้ติดต่อ, ไอคอนสถานะช่อง (A), Keysend (D)


![blixt](assets/en/05.webp)


D - คือเมนูช่วยเหลือ พร้อมลิงก์ไปยังหน้าคำถามที่พบบ่อย / คู่มือ ติดต่อผู้พัฒนา หน้า Github และกลุ่มสนับสนุน Telegram


E - ระบุที่อยู่ BTC แรกของคุณ ที่คุณสามารถฝาก sats สำหรับการทดสอบครั้งแรกของคุณได้ นี่เป็นทางเลือก! หากคุณฝากตรงไปยังที่อยู่นั้น จะเป็นการเปิดช่อง LN ไปยัง Blixt Node ซึ่งหมายความว่าคุณจะเห็น sats ที่คุณฝากเข้าไปในธุรกรรมบนเชน (tx) อีกครั้ง เพื่อเปิดช่อง LN นั้น คุณสามารถตรวจสอบได้ใน Blixt onchain wallet (ดูที่จุด C) โดยคลิกที่เมนู TX ด้านขวาบน


![blixt](assets/en/06.webp)


ตามที่คุณเห็นในบันทึกธุรกรรม Onchain ขั้นตอนมีความละเอียดมากซึ่งบ่งบอกว่า sats กำลังไปที่ไหน (ฝาก เปิด ปิดช่องทาง)


คำแนะนำ:


หลังจากทดสอบหลายสถานการณ์ เราได้ข้อสรุปว่ามีประสิทธิภาพมากกว่าที่จะเปิดช่องระหว่าง 1 ถึง 5 M sats ช่องที่เล็กกว่ามักจะหมดเร็วและต้องจ่ายค่าธรรมเนียมในอัตราที่สูงกว่าเมื่อเปรียบเทียบกับช่องที่ใหญ่กว่า


F - ระบุยอดคงเหลือหลักของ Lightning wallet ของคุณ นี่ไม่ใช่ยอดคงเหลือรวมของ Blixt wallet ของคุณ มันแสดงเฉพาะ sats ที่คุณมีใน Lightning Channels ที่พร้อมสำหรับการส่ง ตามที่ได้ระบุไว้ก่อนหน้านี้ Onchain wallet แยกออกจากกัน โปรดจำไว้ว่าแง่มุมนี้ Onchain wallet แยกออกจากกันด้วยเหตุผลสำคัญ: มันถูกใช้หลักๆ สำหรับการเปิด/ปิด LN channels


ตกลง ตอนนี้แม่ของดาร์ธได้ฝาก sats เข้าไปในที่อยู่บนเชนที่แสดงบนหน้าจอหลักแล้ว ขอแนะนำว่าเมื่อคุณทำเช่นนั้น ให้เปิดแอป Blixt ของคุณออนไลน์และใช้งานได้สักพัก จนกว่า BTC tx จะถูกนำโดยนักขุดเข้าสู่บล็อกแรก


หลังจากนั้นอาจใช้เวลาถึง 20-30 นาทีจนกว่าจะได้รับการยืนยันอย่างสมบูรณ์และช่องทางจะเปิดและคุณจะเห็นใน Magic Drawer - Lightning Channels ว่าเป็นสถานะใช้งาน นอกจากนี้จุดสีเล็ก ๆ บนยอดของลิ้นชัก หากเป็นสีเขียวจะแสดงว่าช่องทาง LN ของคุณออนไลน์และพร้อมที่จะใช้ส่ง sats ผ่าน LN แล้ว


ที่อยู่และข้อความต้อนรับที่แสดงจะหายไป ไม่จำเป็นต้องเปิดช่องอัตโนมัติอีกต่อไป คุณยังสามารถปิดใช้งานตัวเลือกในเมนูการตั้งค่าได้


ถึงเวลาที่จะก้าวต่อไป ทดสอบคุณสมบัติและตัวเลือกอื่นๆ เพื่อเปิดช่อง LN


ตอนนี้ มาเปิดช่องทางอีกช่องทางหนึ่งกับโหนดเพียร์อีกตัวหนึ่ง ชุมชน Blixt ได้รวบรวม[รายชื่อโหนดที่ดีเพื่อเริ่มใช้งานกับ Blixt](https://github.com/hsjoberg/blixt-wallet/issues/1033)


**ขั้นตอนการเปิดช่อง LN ใน Blixt**


นี่เป็นเรื่องง่ายมาก เพียงแค่ทำตามขั้นตอนเล็กน้อยและใช้ความอดทนสักนิด:



- ไปที่ [Blixt Community](https://github.com/hsjoberg/blixt-wallet/issues/1033) รายการของเพื่อนร่วมงาน
- เลือกโหนดและคลิกที่ลิงก์ชื่อเรื่องของมัน จะเปิดหน้า Amboss ของมัน
- คลิกเพื่อแสดงรหัส QR สำหรับที่อยู่ URI ของโหนด


![blixt](assets/en/07.webp)


เปิด Blixt และไปที่ลิ้นชักบนสุด - ช่องทาง Lightning และคลิกที่ปุ่ม “+”


![blixt](assets/en/08.webp)


ตอนนี้ คลิกที่ (A) กล้องเพื่อสแกน QR โค้ดจากหน้าของ Amboss และรายละเอียดของโหนดจะถูกกรอกโดยอัตโนมัติ เพิ่มจำนวนของ sats สำหรับช่องที่คุณต้องการและเลือกอัตราค่าธรรมเนียมสำหรับ tx คุณสามารถปล่อยให้เป็นอัตโนมัติ (B) เพื่อการยืนยันที่เร็วขึ้นหรือปรับเองโดยการเลื่อนปุ่ม คุณยังสามารถกดค้างที่ตัวเลขและแก้ไขตามที่คุณต้องการได้


อย่าใส่น้อยกว่า 1 sat/vbyte! โดยปกติแล้วควรตรวจสอบ [ค่าธรรมเนียม mempool](https://mempool.space/) ก่อนเปิดช่องและเลือกค่าธรรมเนียมที่สะดวก.


เสร็จแล้ว ตอนนี้เพียงคลิกที่ปุ่ม “open channel” และรอการยืนยัน 3 ครั้ง ซึ่งปกติจะใช้เวลาประมาณ 30 นาที (ประมาณ 1 บล็อกทุกๆ 10 นาที)


เมื่อได้รับการยืนยันแล้ว คุณจะเห็นช่องทางที่ใช้งานอยู่ในส่วน "Lightning Channels" ของคุณ

## Blixt - Second Contact


ตกลง ตอนนี้เรามีช่องทาง LN ที่มีเฉพาะสภาพคล่องขาออกเท่านั้น ซึ่งหมายความว่าเราสามารถส่งได้เท่านั้น เรายังไม่สามารถรับ sats ผ่าน LN ได้


![blixt](assets/en/09.webp)


ทำไม? คุณได้อ่านคู่มือที่ระบุไว้ในตอนต้นหรือยัง? ยังไม่ได้อ่าน? กลับไปอ่านพวกมัน มันสำคัญมากที่จะต้องเข้าใจว่า LN channels ทำงานอย่างไร


![blixt](assets/en/10.webp)


ตามที่คุณเห็นในตัวอย่างนี้ ช่องทางเปิดด้วยการฝากครั้งแรก ไม่มีสภาพคล่องขาเข้า ("สามารถรับ") มากนัก แต่มีสภาพคล่องขาออก ("สามารถส่ง") มากมาย


ดังนั้นคุณมีตัวเลือกอะไรบ้าง หากคุณต้องการรับ sats มากกว่า LN?



- ใช้จ่าย sats จากช่องทางที่มีอยู่ ใช่, LN เป็นเครือข่ายการชำระเงินของ Bitcoin ซึ่งใช้หลักในการใช้จ่าย sats ของคุณให้เร็วขึ้น, ถูกกว่า, เป็นส่วนตัวและง่าย LN ไม่ใช่วิธีการถือครอง สำหรับนั้นคุณมี onchain wallet



- แลกเปลี่ยนบางส่วนของ sats กลับไปยัง wallet บนเชนของคุณ โดยใช้บริการแลกเปลี่ยนแบบซับมารีน ด้วยวิธีนี้คุณจะไม่ใช้จ่าย sats ของคุณ แต่จะคืนมันกลับไปยัง wallet บนเชนของคุณเอง ที่นี่คุณสามารถดูรายละเอียดบางวิธีได้ใน [หน้าคู่มือ Blixt](https://blixtwallet.github.io/guides)



- เปิดช่องทาง INBOUND จากผู้ให้บริการ LSP ใดก็ได้ นี่คือวิดีโอสาธิตเกี่ยวกับวิธีการใช้ LNBig LSP สำหรับการเปิดช่องทาง inbound ซึ่งหมายความว่าคุณจะต้องจ่ายค่าธรรมเนียมเล็กน้อยสำหรับช่องทางที่ว่างเปล่า (ในฝั่งของคุณ) และคุณจะสามารถรับ sats เพิ่มเติมในช่องทางนั้นได้ หากคุณเป็นพ่อค้าที่รับมากกว่าที่ใช้ นั่นเป็นตัวเลือกที่ดี นอกจากนี้หากคุณกำลังซื้อ sats ผ่าน LN โดยใช้ Robosats หรือการแลกเปลี่ยน LN อื่น ๆ



- เปิดช่อง Dunder ด้วย Blixt node หรือผู้ให้บริการ Dunder LSP รายอื่น ช่อง Dunder เป็นวิธีง่ายๆ ในการรับสภาพคล่องขาเข้า แต่ในขณะเดียวกันคุณฝาก sats บางส่วนลงในช่องนั้นด้วย นอกจากนี้ยังดีเพราะจะเปิดช่องด้วย [UTXO](https://en.bitcoin.it/wiki/UTXO) ที่ไม่ใช่จาก Blixt wallet ของคุณ ซึ่งเพิ่มความเป็นส่วนตัว นอกจากนี้ยังดีเพราะหากคุณไม่มี sats ใน wallet บนเชนเพื่อเปิดช่อง LN ปกติ แต่คุณมีใน LN wallet อื่น คุณสามารถชำระเงินจาก wallet อื่นผ่าน LN การเปิดและการฝาก (ในฝั่งของคุณ) ของช่อง Dunder นั้น [รายละเอียดเพิ่มเติมเกี่ยวกับวิธีการทำงานของ Dunder และวิธีการเรียกใช้เซิร์ฟเวอร์ของคุณเองที่นี่](https://github.com/hsjoberg/dunder-lsp)


![blixt](assets/en/11.webp)


นี่คือขั้นตอนในการเปิดใช้งานช่อง Dunder:



- ไปที่การตั้งค่า ในส่วน "Experiments" เปิดใช้งานช่องสำหรับ "Enable Dunder LSP"
- เมื่อคุณทำเช่นนั้นแล้ว ให้กลับขึ้นไปที่ส่วน “Lightning Network” และคุณจะเห็นว่ามีตัวเลือก “Set Dunder LSP Server” ปรากฏขึ้น ที่นั่น โดยค่าเริ่มต้นจะตั้งค่าเป็น “https://dunder.blixtwallet.com” แต่คุณสามารถเปลี่ยนเป็นที่อยู่ผู้ให้บริการ Dunder LSP อื่น ๆ ได้ [นี่คือรายการชุมชน Blixt](https://github.com/hsjoberg/blixt-wallet/issues/1033) ที่มีโหนดที่สามารถให้ช่อง Dudner LSP สำหรับ Blixt ของคุณ
- ตอนนี้คุณสามารถไปที่หน้าจอหลักและคลิกที่ปุ่ม “รับ” จากนั้นทำตามขั้นตอนนี้ [อธิบายไว้ในคู่มือนี้](https://blixtwallet.github.io/guides#guide-lsp).


ตกลง หลังจากช่อง Dunder ได้รับการยืนยัน (จะใช้เวลาสองสามนาที) คุณจะมีช่อง LN สองช่อง: ช่องหนึ่งเปิดในตอนแรกด้วย autopilot (ช่อง A) และอีกช่องหนึ่งที่มีสภาพคล่องขาเข้ามากกว่า เปิดด้วย Dunder (ช่อง B)


![blixt](assets/en/12.webp)


ดี ตอนนี้คุณพร้อมที่จะส่งและรับ sats มากพอผ่าน LN แล้ว!

## Blixt - Third Contact


โปรดจำไว้ว่า ในบทที่หนึ่ง “การติดต่อครั้งแรก” มี 2 ตัวเลือกในหน้าจอต้อนรับ:


- [Option A](https://darth-coin.github.io/wallets/getting-started-blixt-wallet-en.html#option-a) - สร้าง wallet ใหม่
- ตัวเลือก B - กู้คืน wallet


ตอนนี้เรามาพูดคุยเกี่ยวกับวิธีการกู้คืน Blixt wallet หรือโหนด LND ที่ล่มกันเถอะ นี่อาจจะดูเทคนิคเล็กน้อย แต่โปรดให้ความสนใจ มันไม่ยากขนาดนั้น


### ตัวเลือก B - กู้คืน wallet


ในอดีตฉันได้เขียนคู่มือเฉพาะเกี่ยวกับ [วิธีการกู้คืนโหนด Umbrel ที่ล่ม](https://darth-coin.github.io/nodes/shtf-restore-lnd-node-en.html) ซึ่งฉันได้กล่าวถึงวิธีการใช้ Blixt เป็นกระบวนการกู้คืนอย่างรวดเร็ว โดยใช้ไฟล์ seed + channel.backup จาก Umbrel ด้วย


ฉันยังเขียนคู่มือเกี่ยวกับวิธีการกู้คืน Blixt node ของคุณหรือย้าย Blixt ของคุณไปยังอุปกรณ์อื่น [ที่นี่](https://blixtwallet.github.io/faq#blixt-restore)


![blixt](assets/en/13.webp)


แต่ลองอธิบายขั้นตอนนี้อย่างง่าย ๆ ตามที่คุณเห็นในภาพด้านบน มี 2 สิ่งที่คุณควรทำเพื่อกู้คืนโหนด Blixt/LND ก่อนหน้านี้ของคุณ:



- กล่องด้านบนคือที่ที่คุณต้องกรอกคำทั้ง 24 คำจาก seed ของคุณ (โหนดเก่า / โหนดที่ตายแล้ว)
- ด้านล่างมีตัวเลือกปุ่มสองปุ่มสำหรับแทรก / อัปโหลดไฟล์ channel.backup ที่บันทึกไว้ก่อนหน้านี้จากโหนด Blixt/LND เก่าของคุณ ไฟล์นี้สามารถมาจากไฟล์ในเครื่อง (คุณอัปโหลดลงในอุปกรณ์ของคุณก่อนหน้านี้) หรือสามารถมาจากตำแหน่งระยะไกลของ Google drive / iCloud Blixt มีตัวเลือกนี้ในการบันทึกการสำรองข้อมูลช่องของคุณโดยตรงลงใน Google / iCloud drive ดูรายละเอียดเพิ่มเติมได้ที่ [Blixt Features Page](https://blixtwallet.github.io/features#blixt-options)


อย่างไรก็ตาม หากคุณไม่เคยมีช่อง LN เปิดมาก่อน ก็ไม่จำเป็นต้องอัปโหลดไฟล์ channels.backup เพียงแค่ใส่คำ 24 คำของ seed แล้วกดปุ่มกู้คืน


อย่าลืมเปิดใช้งาน Tor จากเมนูจุดสามจุดด้านบน ตามที่เราอธิบายในส่วนตัวเลือก A นั่นคือกรณีที่คุณมีเพียงเพื่อน Tor และไม่สามารถติดต่อผ่าน clearnet (โดเมน/IP) ได้ มิฉะนั้นไม่จำเป็นต้องทำ


คุณสมบัติที่มีประโยชน์อีกอย่างหนึ่งคือการตั้งค่าโหนด Bitcoin เฉพาะจากเมนูด้านบน โดยค่าเริ่มต้นมันจะซิงค์บล็อกจาก node.blixtwallet.com (โหมด neutrino) แต่คุณสามารถตั้งค่าโหนด Bitcoin อื่นๆ ที่ให้บริการซิงค์ neutrino ได้


ดังนั้นเมื่อคุณกรอกตัวเลือกเหล่านั้นแล้ว กดปุ่มกู้คืน Blixt จะเริ่มซิงค์บล็อกผ่าน Neutrino ก่อนตามที่เราอธิบายในบท First Contact ดังนั้นโปรดอดทนและดูขั้นตอนการกู้คืนในหน้าจอหลัก โดยการคลิกที่ไอคอนซิงค์


![blixt](assets/en/14.webp)


ตามที่คุณเห็นในตัวอย่างนี้ แสดงให้เห็นว่าบล็อกของบิตคอยน์ซิงค์ 100% (A) และกระบวนการกู้คืนกำลังดำเนินการอยู่ (B) นั่นหมายความว่า ช่องทาง LN ที่คุณมีมาก่อนหน้านี้จะถูกปิดและเงินทุนจะถูกคืนเข้าสู่ Blixt onchain wallet ของคุณ


กระบวนการนี้ต้องใช้เวลา! ดังนั้นโปรดอดทนและพยายามให้ Blixt ของคุณทำงานและออนไลน์อยู่เสมอ การซิงค์ครั้งแรกอาจใช้เวลาถึง 6-8 นาที และการปิดช่องทางอาจใช้เวลาถึง 10-15 นาที ดังนั้นคุณควรชาร์จอุปกรณ์ให้เต็มที่


เมื่อกระบวนการนี้เริ่มต้นขึ้น คุณสามารถตรวจสอบใน Magic Drawer - Lightning Channels สถานะของแต่ละช่องทางก่อนหน้าของคุณ ซึ่งจะแสดงว่าอยู่ในสถานะ "pending to close" เมื่อแต่ละช่องทางถูกปิด คุณสามารถดูการปิด tx ใน onchain wallet (ดู Magic Drawer - Onchain) และเปิดบันทึกเมนู tx ได้


![blixt](assets/en/15.webp)


นอกจากนี้จะเป็นการดีที่จะตรวจสอบและเพิ่มเพื่อนร่วมงานที่คุณเคยมีในโหนด LN เก่าของคุณหากยังไม่มี ดังนั้นไปที่เมนูการตั้งค่า เลื่อนลงไปที่ “Lightning Network” และเข้าสู่ตัวเลือก “แสดงเพื่อนร่วมสายฟ้า”


![blixt](assets/en/16.webp)


ภายในส่วนนี้คุณจะเห็นเพื่อนที่คุณเชื่อมต่อในขณะนั้นและคุณสามารถเพิ่มได้อีก ควรเพิ่มเพื่อนที่คุณเคยมีช่องทางด้วย เพียงไปที่ [Amboss page](https://amboss.space/), ค้นหาชื่อเล่นของเพื่อนหรือ nodeID ของคุณและสแกน node URI ของพวกเขา


![blixt](assets/en/17.webp)


ตามที่คุณเห็นในภาพด้านบน มี 3 แง่มุม:


A - แทนที่อยู่ URI ของโหนด clearnet (โดเมน/IP)


B - แทนที่อยู่โหนดหัวหอมของ Tor URI (.onion)


C - คือรหัส QR ที่จะสแกนด้วยกล้อง Blixt ของคุณหรือปุ่มคัดลอก


ที่อยู่โหนด URI นี้คุณต้องเพิ่มเข้าไปในรายการเพื่อนของคุณ ดังนั้นโปรดทราบว่าไม่เพียงพอแค่ชื่อเล่นของโหนดหรือ nodeID เท่านั้น


ตอนนี้คุณสามารถไปที่ Magic Drawer (เมนูด้านบนซ้าย) - Lightning Channels และคุณสามารถดูได้ว่าที่บล็อกความสูงของการครบกำหนดใดที่เงินจะถูกส่งคืนไปยังที่อยู่ onchain ของคุณ


![blixt](assets/en/18.webp)


หมายเลขบล็อก 764272 คือเมื่อเงินจะสามารถใช้งานได้ในที่อยู่บิตคอยน์ออนเชนของคุณ และอาจใช้เวลาถึง 144 บล็อกจากบล็อกยืนยันแรกจนกว่าจะถูกปล่อยออกมา [ดังนั้นตรวจสอบในเมมพูล](https://mempool.space/)


และนั่นคือทั้งหมด รออย่างอดทนจนกว่าช่องทางทั้งหมดจะปิดและเงินกลับเข้าสู่ onchain wallet ของคุณ

### B - Lightning Box


เริ่มต้นด้วย v0.6.9-420 ใหม่ [ประกาศเมื่อเร็วๆ นี้](https://github.com/hsjoberg/blixt-wallet/releases/tag/v0.6.9-420) Blixt ได้แนะนำฟีเจอร์ใหม่ที่ทรงพลังสำหรับ Lightning Address ใน Blixt


ฟีเจอร์ใหม่นี้เป็นการเลือกเข้าร่วมแบบออปชั่น ไม่ได้เปิดใช้งานโดยค่าเริ่มต้น!


ในขณะนี้ กล่อง LN เริ่มต้นถูกดำเนินการโดยเซิร์ฟเวอร์ Blixt และเสนอ @blixtwallet.com LN Address แต่ใครก็ตามที่มีโหนดสาธารณะ LND สามารถดำเนินการเซิร์ฟเวอร์ Lightning Box และเสนอ LN Address สำหรับโดเมนของตนเอง การดูแลตนเอง


ขณะนี้ เซิร์ฟเวอร์ Blixt กำลังส่งต่อการชำระเงินที่ส่งไปยังที่อยู่ LN @blixtwallet.com ให้กับผู้ใช้ Blixt ที่ตั้งค่าที่อยู่ LN ของพวกเขา ผู้ใช้ต้องตั้งค่าโหนด Blixt wallet ของพวกเขาใน "โหมดถาวร" เพื่อรับการชำระเงินเหล่านี้ไปยังที่อยู่ LN @blixtwallet.com ของพวกเขา


ดูในบันทึกการเปิดตัววิดีโอสาธิตเกี่ยวกับวิธีการตั้งค่า LN Address ของคุณใน Blixt


ที่อยู่ LN นี้ที่นำไปใช้ในแอป Blixt wallet นั้นเหมือนกับการแชทผ่าน LN ทันทีและสนุกสนาน อีกทั้งยังรองรับ [LUD-18](https://github.com/lnurl/luds/blob/luds/18.md) (การเพิ่มชื่อเล่นให้กับการชำระเงิน) คุณสามารถเพิ่มในรายชื่อผู้ติดต่อที่อยู่ LN ทั้งหมดที่คุณใช้บ่อย ๆ และมีไว้สำหรับการแชท ตอนนี้ Blixt สามารถถือว่าเป็นแอปแชท LN เต็มรูปแบบ 😂😂


คุณสมบัติที่มีประโยชน์อีกอย่างคือการสนับสนุนเต็มรูปแบบสำหรับ LUD-18 (ซึ่ง [Stacker.News](https://stacker.news/r/DarthCoin) และอื่น ๆ กำลังสนับสนุนมัน)


![blixt](assets/en/22.webp)

### C - สำรองข้อมูลช่อง LN และคำ seed


นี่เป็นคุณสมบัติที่สำคัญมาก !


หลังจากเปิดหรือปิดช่อง LN คุณควรทำการสำรองข้อมูล สามารถทำได้ด้วยตนเองโดยการบันทึกไฟล์ขนาดเล็กลงในอุปกรณ์ท้องถิ่น (มักจะเป็นโฟลเดอร์ดาวน์โหลด) หรือใช้บัญชี Google Drive หรือ iCloud


![blixt](assets/en/23.webp)


ไปที่การตั้งค่า Blixt - ส่วน Wallet ที่นั่นคุณมีตัวเลือกในการบันทึกข้อมูลสำคัญทั้งหมดสำหรับ Blixt wallet:



- "แสดงคำช่วยจำ" - จะแสดง 24 คำ seed เพื่อเขียนลงไป
- "ลบคำช่วยจำออกจากอุปกรณ์" - นี่เป็นทางเลือกและใช้เฉพาะเมื่อคุณต้องการลบคำ seed ออกจากอุปกรณ์ของคุณจริงๆ สิ่งนี้จะไม่ลบ wallet ของคุณ มีเพียง seed เท่านั้น แต่โปรดระวัง! ไม่มีวิธีการกู้คืนหากคุณไม่ได้จดบันทึกไว้ก่อน
- "Export channel backup" - ตัวเลือกนี้จะบันทึกไฟล์ขนาดเล็กลงในอุปกรณ์ของคุณ โดยปกติจะอยู่ในโฟลเดอร์ "ดาวน์โหลด" ซึ่งคุณสามารถนำออกและย้ายไปยังที่อื่นนอกอุปกรณ์ของคุณเพื่อเก็บรักษาอย่างปลอดภัย
- "ยืนยันการสำรองข้อมูลช่อง" - นี่เป็นตัวเลือกที่ดีหากคุณใช้ Google Drive หรือ iCloud เพื่อตรวจสอบความสมบูรณ์ของการสำรองข้อมูลที่ทำจากระยะไกล
- "การสำรองข้อมูลช่องทาง Google Drive" - จะบันทึกไฟล์สำรองข้อมูลลงใน Google Drive ส่วนตัวของคุณ ไฟล์นี้ถูกเข้ารหัสและถูกจัดเก็บในที่เก็บแยกต่างหากจากไฟล์ Google ปกติของคุณ ดังนั้นจึงไม่มีความกังวลว่าจะถูกอ่านโดยใครก็ตาม อย่างไรก็ตามไฟล์นั้นจะไม่มีประโยชน์เลยหากไม่มีคำ seed ดังนั้นไม่มีใครสามารถนำเงินของคุณจากไฟล์นั้นได้เพียงอย่างเดียว


ฉันขอแนะนำสำหรับส่วนนี้ดังต่อไปนี้:



- ใช้โปรแกรมจัดการรหัสผ่านเพื่อเก็บรักษา seed และไฟล์สำรองของคุณอย่างปลอดภัย KeePass หรือ Bitwarden เป็นตัวเลือกที่ดีมากสำหรับการใช้งานนี้ และสามารถใช้ได้บนหลายแพลตฟอร์มและสามารถโฮสต์เองหรือใช้งานแบบออฟไลน์ได้
- ทำการสำรองข้อมูลทุกครั้งที่คุณเปิดหรือปิดช่องทาง ไฟล์นั้นจะได้รับการอัปเดตด้วยข้อมูลของช่องทาง ไม่จำเป็นต้องทำหลังจากแต่ละธุรกรรมที่คุณทำบน LN การสำรองข้อมูลช่องทางไม่ได้เก็บข้อมูลนั้น แต่เก็บเฉพาะสถานะของช่องทางเท่านั้น

## Blixt - เคล็ดลับและเทคนิค


### กรณีที่ 1 - ปัญหาการซิงค์


"_Blixt ของฉันไม่ซิงค์... Blixt ของฉันไม่แสดงยอดคงเหลือ... Blixt ของฉันไม่สามารถเปิดช่องทางได้... ฉันพยายามกู้คืนในอุปกรณ์อื่น... เป็นต้น_"


ปัญหาทั้งหมดนี้เริ่มต้นเพราะ อุปกรณ์ของคุณไม่ได้ซิงค์อย่างถูกต้อง โปรดเข้าใจในแง่มุมสำคัญนี้: Blixt เป็นโหนด LND บนมือถือ ที่ใช้ Neutrino สำหรับการซิงค์ / อ่านบล็อก



- นี่คือคำอธิบายที่ไม่ซับซ้อนจาก [Bitcoin Magazine](https://bitcoinmagazine.com/technical/why-bitcoin-wallets-need-block-filters)
- นี่คือแหล่งข้อมูลทางเทคนิคเพิ่มเติมจาก [Bitcoin Optech](https://bitcoinops.org/en/topics/compact-block-filters/)
- นี่คือวิธีที่คุณสามารถเปิดใช้งาน Neutrino บนโหนดบ้านของคุณเองและให้บริการตัวกรองบล็อกสำหรับโหนดมือถือของคุณ จาก [Docs Lightning Engineering](https://docs.lightning.engineering/lightning-network-tools/lnd/enable-neutrino-mode-in-bitcoin-core)


การแจ้งเตือน: การใช้ Neutrino ผ่าน clearnet นั้นปลอดภัยอย่างสมบูรณ์ IP หรือ xpub ของคุณจะไม่ถูกเปิดเผย คุณเพียงแค่อ่านบล็อกจากโหนดระยะไกลด้วย neutrino เท่านั้น ที่เหลือทั้งหมดจะทำบนอุปกรณ์ท้องถิ่นของคุณ


ดังนั้นจึงไม่จำเป็นต้องใช้ร่วมกับ Tor Tor จะเพิ่มความหน่วงอย่างมากในกระบวนการซิงค์ของคุณและจะทำให้ Blixt ของคุณไม่เสถียรมาก หากคุณต้องการใช้ผ่าน Tor จริงๆ ให้แน่ใจว่าคุณรู้ว่ากำลังทำอะไรอยู่และมีการเชื่อมต่อที่ดีและความอดทน กรณีเดียวกันสำหรับการใช้ VPN ระวังว่าความหน่วงที่ได้รับจาก VPN นั้นเป็นอย่างไร


คุณสามารถทดสอบความหน่วงของเซิร์ฟเวอร์นิวตริโนได้โดยการพิงมันจากพีซีหรือจากมือถือของคุณ


![blixt](assets/en/25.webp)


นี่คือการ ping ปกติไปยังเซิร์ฟเวอร์นิวตริโน europe.blixtwallet.com ซึ่งแสดงให้เห็นว่าการเชื่อมต่อดีมากด้วยเวลาในการตอบสนองเฉลี่ย 50ms และ TTL ที่ 51 เวลาในการตอบสนองอาจแตกต่างกันได้แต่ไม่มาก TTL ต้องคงที่


หากค่าต่างๆ เหล่านี้สูงกว่า 100-150ms กระบวนการซิงค์ของคุณจะล่าช้าหรือแย่กว่านั้น อาจทำให้ช่องทางถูกปิดโดยเพื่อนร่วมเครือข่าย! อย่ามองข้ามประเด็นนี้เด็ดขาด


หากไม่มีการซิงค์ที่เหมาะสม คุณจะไม่สามารถเห็นยอดคงเหลือที่ถูกต้องหรือช่อง LN ของคุณจะไม่ออนไลน์และทำงานได้ ไม่สำคัญว่าคุณจะมีความเร็วในการดาวน์โหลดกี่กิกะอัลตร้าเทรา Mbps มันไม่สำคัญ สิ่งที่สำคัญคือการตอบสนองเวลาและ TTL (time to live)


นี่เป็นกรณีทั่วไปสำหรับผู้ใช้ในภูมิภาคละตินอเมริกา ฉันไม่รู้ว่าเกิดอะไรขึ้นที่นั่น แต่พวกคุณมีการเชื่อมต่อที่แย่มากด้วยค่า ping ที่มากกว่า 200ms ซึ่งสามารถรบกวนการซิงค์ได้


แล้วทางออกสำหรับผู้ใช้ที่สิ้นหวังเหล่านี้คืออะไร?



- หยุดใช้ Blixt กับ Tor มันไร้ประโยชน์โดยสิ้นเชิง
- คุณสามารถใช้ VPN ได้ แต่เลือกอย่างชาญฉลาดและตรวจสอบ ping ตลอดเวลา ใช้ตัวที่อยู่ใกล้กับตำแหน่งทางภูมิศาสตร์ของคุณ ระยะทางหมายถึงเวลาตอบสนองที่มากขึ้นในหน่วยมิลลิวินาที จำไว้
- เลือกเพื่อนร่วมงานนิวตริโนของคุณอย่างชาญฉลาด นี่คือรายชื่อเซิร์ฟเวอร์นิวตริโนสาธารณะที่เป็นที่รู้จักกันดี:


```txt
For US region
btcd1.lnolymp.us | btcd2.lnolymp.us
btcd-mainnet.lightning.computer
swest.blixtwallet.com (Seattle)
node.eldamar.icu
noad.sathoarder.com
bb1.breez.technology | bb2.breez.technology
neutrino.shock.network
For EU region
europe.blixtwallet.com (Germany)
For Asia region
sg.lnolymp.us
asia.blixtwallet.com
```


อีกวิธีหนึ่งคือการเลือกจากรายการโหนดที่ประกาศ "compact filters" (BIP157 / neutrino) - [Bitnodes Page Neutrino filter](https://bitnodes.io/nodes/?q=NODE_COMPACT_FILTERS) เลือกโหนดที่อยู่ใกล้กับตำแหน่งทางภูมิศาสตร์ของคุณมากกว่า


อีกวิธีหนึ่ง (วิธีที่ดีที่สุด) คือการเชื่อมต่อกับโหนดชุมชนท้องถิ่นที่ดำเนินการโดยเพื่อนหรือกลุ่มที่คุณรู้จัก และเสนอการเชื่อมต่อแบบนิวตริโน [นี่คือคำแนะนำวิธีการทำ](https://docs.lightning.engineering/lightning-network-tools/lnd/enable-neutrino-mode-in-bitcoin-core) โหนดของพวกเขาจะไม่ได้รับผลกระทบใดๆ เพียงแค่ต้องการการเชื่อมต่อที่เสถียรและเป็นสาธารณะ


มีความจำเป็นต้องมีเซิร์ฟเวอร์นิวตริโนเพิ่มเติมในภูมิภาค LATAM เพื่อการซิงค์ที่ดียิ่งขึ้นและรวดเร็วขึ้น ดังนั้นโปรดจัดระเบียบตัวเองกับชุมชน Bitcoin ในท้องถิ่นของคุณและตัดสินใจว่าใครและที่ไหนจะรัน Bitcoin Core + Neutrino สำหรับการใช้งานของคุณเอง เพียงแค่มี IP สาธารณะก็เพียงพอแล้ว หากคุณไม่มีการเข้าถึง IP สาธารณะ คุณสามารถใช้ VPS IP และทำ wireguard tunnel ไปยังโหนดที่บ้านของคุณ วิธีนี้คุณจะเปลี่ยนเส้นทางการจราจรทั้งหมดไปยัง VPS IP ในท้องถิ่นของคุณ โดยไม่เปิดเผยข้อมูลส่วนตัวใดๆ เกี่ยวกับโหนดที่บ้านของคุณ


### กรณีที่ 2 - ไม่เคยซิงค์เสร็จสิ้น


"_Blixt ของฉันมีการเชื่อมต่อที่ดีกับเซิร์ฟเวอร์นิวตริโนแต่ติดอยู่ที่การซิงค์._"


#### เซิร์ฟเวอร์เวลา


บางครั้งผู้คนใช้เครื่องมือเก่าหรือไม่ได้เชื่อมต่อกับเซิร์ฟเวอร์เวลาอย่างถูกต้อง Neutrino กำลังซิงค์ได้ดีจนกระทั่งถึงบล็อกที่ไม่ตรงกับเวลาท้องถิ่นจริง คุณจะเห็นในบันทึก Blixt lnd ข้อความแสดงข้อผิดพลาดที่บอกว่า "block time stamp is far from future" หรือบางอย่างที่เกี่ยวข้องกับ "header don't pass sanity check"


การแก้ไขอย่างรวดเร็ว: ตั้งเวลาและวันที่ที่ถูกต้องสำหรับอุปกรณ์ของคุณและรีสตาร์ท Blixt.


#### พื้นที่ในอุปกรณ์เหลือน้อย


บางครั้งการใช้เครื่องเก่า ที่มีพื้นที่น้อย อาจถึงขีดจำกัดและค้างได้ จริงๆ แล้ว ยิ่งคุณใช้มือถือ LND node นี้มากเท่าไหร่ ไฟล์ neutrino ก็จะใหญ่ขึ้นและไฟล์ channel.db ก็เช่นกัน


Quick fix: ไปที่ Blixt Options - ส่วน Debug - เลือก "stop LND and delete neutrino files" มันจะรีสตาร์ทแอปและเริ่มการซิงค์ใหม่ บางครั้งการแก้ไขด่วนนี้สามารถซ่อมแซมข้อมูลที่เสียหายได้ด้วย โปรดทราบว่าจะใช้เวลาระหว่าง 1 ถึง 3 นาทีในการซิงค์ใหม่ทั้งหมด มันจะไม่ลบเงินหรือช่องทางที่มีอยู่ แต่ใช่ หลังจากการซิงค์ใหม่อาจทำให้เกิดการสแกนใหม่ของที่อยู่บิตคอยน์ของคุณและอาจใช้เวลามากขึ้น


ขั้นตอนต่อไปคือการตรวจสอบว่ามีข้อมูลที่ยังคงถูกครอบครองอยู่เท่าใด คุณสามารถดูได้ในข้อมูลแอป Android - ข้อมูล หากยังคงมีขนาดใหญ่กว่า 400-500MB คุณสามารถบีบอัดไฟล์ lnd ได้ ดังนั้นไปที่ Blixt Options - ส่วน Debug - เลือก "Compact DB LND" รีสตาร์ทแอป Blixt หากไม่ได้ทำโดยอัตโนมัติ การบีบอัดจะเกิดขึ้นเมื่อเริ่มต้นและมีเพียงครั้งเดียว ตอนนี้คุณจะเห็นว่าข้อมูล Blixt ถูกครอบครองน้อยลงมาก


#### โหมดถาวร


บางครั้งผู้คนไม่ได้เปิด Blixt เป็นเวลานาน ดังนั้นการซิงค์จึงเก่าเกินไป แต่พวกเขาคาดหวังว่าจะซิงค์ทันทีเมื่อเปิดใช้งาน


โปรดอดทน และดูที่วงล้อหมุนด้านบน คุณสามารถไปที่ ตัวเลือก - ดูข้อมูลโหนด และตรวจสอบว่ามีการซิงค์กับเชนและกราฟที่ถูกทำเครื่องหมายว่า "true" หรือไม่ หากไม่มีการทำเครื่องหมาย "true" คุณจะไม่สามารถใช้ Blixt ได้อย่างถูกต้อง คุณจะไม่สามารถดูยอดเงินได้อย่างถูกต้อง คุณจะไม่สามารถดูช่อง LN ออนไลน์ได้ และคุณจะไม่สามารถทำการชำระเงินได้


Quick fix: มีตัวเลือกที่ทรงพลังในการ "รักษาการทำงาน" ของ Blixt node ของคุณ ไปที่ Options - Experiments - เลือก "Activate Persistent Mode" นั่นจะรีสตาร์ท Blixt ของคุณและจะตั้งค่า lnd service ในโหมดถาวร หรือก็คือจะทำงานอยู่เสมอและรักษาการเชื่อมต่อออนไลน์ของคุณ แม้ว่าคุณจะสลับไปใช้แอปอื่นหรือปิด Blixt (ไม่ใช่การปิดแบบบังคับหรือฆ่างาน) คุณสามารถตั้งค่าแบบนี้ได้ตลอดทั้งวันหากคุณมีการเชื่อมต่อที่เสถียรและต้องการใช้ Blixt หลายครั้ง มันจะไม่กินแบตเตอรี่มากนัก


### กรณีที่ 3 - ฉันต้องการย้ายไปยังอุปกรณ์อื่น


ตกลงเกี่ยวกับสถานการณ์นี้ ฉันได้เขียนคู่มืออย่างละเอียดใน [หน้าคำถามที่พบบ่อย](https://blixtwallet.github.io/faq#blixt-restore): โดยมี 2 ตัวเลือก, เร็ว (ปิดช่องทางร่วมมือก่อนการย้าย) และช้า (บังคับปิดช่องทางเพราะอุปกรณ์เก่าเสีย)


แต่ฉันต้องการย้ำถึงแง่มุมสำคัญบางประการที่นี่และเพิ่ม "ขั้นตอนลับ" ใหม่


เตือนความจำ:



- ควรสำรองข้อมูลสถานะช่องของคุณ (SCB) ทุกครั้งหลังจากที่คุณเปิดหรือปิดช่อง ใช้เวลาเพียงไม่กี่วินาทีในการทำ
- อย่าเก็บไฟล์ SCB เก่าไว้ เพื่อไม่ให้เกิดความสับสนและกู้คืนไฟล์เหล่านั้น ไฟล์เหล่านี้ไม่มีประโยชน์และอาจทำให้เกิดกระบวนการลงโทษได้หากคุณใช้ไฟล์เหล่านั้นเสมอ ใช้เวอร์ชันล่าสุดของไฟล์ SCB หากคุณดำเนินการกู้คืน
- บันทึกไฟล์ SCB (เป็นข้อความเข้ารหัสที่มีนามสกุล .bin) ออกจากอุปกรณ์ของคุณในที่ปลอดภัย คุณสามารถใช้ [LocalSend](https://github.com/localsend/localsend) สำหรับการย้ายไฟล์นี้ไปยังพีซีหรืออุปกรณ์อื่น ๆ ได้
- บันทึก seed ของ Blixt wallet ของคุณในที่ปลอดภัย เช่น ตัวจัดการรหัสผ่านออฟไลน์ / USB ที่เข้ารหัส


วิธีลับ: วิธีการย้ายโหนด Blixt โดยไม่ต้องปิดช่องทางที่มีอยู่ สำหรับสิ่งนี้คุณจะต้องอ่านส่วน "Third Contact" ในคู่มือนี้เกี่ยวกับ "Restore wallet" อย่างละเอียด


ขั้นตอนนี้ไม่เหมาะสำหรับมือใหม่ มันเหมาะสำหรับผู้ใช้ขั้นสูงเท่านั้น! นั่นคือเหตุผลที่ไม่ได้เปิดเผยอย่างกว้างขวางและฉันแนะนำให้ทำเฉพาะเมื่อมีความช่วยเหลือจากนักพัฒนา Blixt หรือการสนับสนุนจากฉันเท่านั้น กรุณาอย่ามองข้ามคำแนะนำนี้


### กรณีที่ 4 - ใช้เพียร์ใดในการเปิดช่องทาง?


ตามที่ฉันเขียนไว้ใน [หน้าคำแนะนำของ Blixt](https://blixtwallet.github.io/guides) มีหลายวิธีในการเปิดช่องทางกับโหนด LND บนมือถือเครื่องนี้ แต่มีบางประเด็นสำคัญที่ฉันอยากจะเตือนคุณที่นี่:



- เปิดด้วยโหนด LSP ที่เป็นที่รู้จักและกับเพื่อนที่ชุมชนรับรองแล้ว [ดูรายการที่นี่](https://github.com/hsjoberg/blixt-wallet/issues/1033)
- อย่าเปิดด้วยโหนด Tor แบบสุ่มเท่านั้น โหนดเหล่านั้นไม่มีค่าและคุณจะพบปัญหาในการชำระเงิน ไม่สำคัญว่าเพื่อนของคุณ "ผู้รันโหนด" จะเก่งแค่ไหนกับโหนด Tor ที่ไม่ดีในป่า มันจะไม่ให้เส้นทางที่ดีที่สุดสำหรับโหนดส่วนตัวบนมือถือ คุณไม่เปิดช่องทางกับใครเพราะเขาเป็นเพื่อนของคุณ นี่ไม่ใช่ Facebook! คุณเปิดช่องทางเพื่อ: เส้นทางที่ดี ค่าธรรมเนียมต่ำ ความพร้อมใช้งาน
- ไม่มีความจำเป็นต้องเปิดช่องเล็กๆ จำนวนมาก 2-3 หรือสูงสุด 4 ช่องก็พอ แต่ต้องมี sats ในปริมาณที่ดี อย่าเปิดช่องเล็กๆ เพราะไม่มีประโยชน์ ช่องที่เล็กกว่า 200k สำหรับมือถือไม่มีประโยชน์มากนัก
- โปรดคำนึงถึง LSPs ที่มีช่องทาง inbound และช่องทาง JIT (just in time) ซึ่งมีประโยชน์มากเพราะคุณไม่จำเป็นต้องใช้ UTXOs ของคุณเอง คุณสามารถชำระค่าช่องทางเปิดด้วยเงินที่คุณมีอยู่แล้วในกระเป๋า LN อื่น ๆ โดยการสะสมและเตรียมพร้อมสำหรับการเปิดช่องทางที่ใหญ่ขึ้น คุณควรใช้ช่องทาง JIT เหล่านี้ให้เป็นประโยชน์ [ฉันได้อธิบายในคู่มือนี้](https://darth-coin.github.io/nodes/managing-lightning-node-liquidity-en.html) ตัวเลือกเพิ่มเติมสำหรับเพื่อนร่วมเครือข่ายสำหรับโหนดส่วนตัวเช่น Blixt นอกจากนี้ [ในคู่มือนี้ที่โพสต์บน SN](https://stacker.news/items/679242/r/DarthCoin) ฉันได้อธิบายวิธีการจัดการสภาพคล่องของโหนดมือถือส่วนตัว

ตกลง ยังมีฟีเจอร์ที่น่าทึ่งอื่นๆ อีกมากมายที่ Blixt นำเสนอ ฉันจะให้คุณค้นพบทีละอย่างและสนุกไปกับมัน


แอปนี้ถูกประเมินต่ำเกินไปจริง ๆ ส่วนใหญ่เป็นเพราะไม่ได้รับการสนับสนุนจากเงินทุนของ VC ใด ๆ ขับเคลื่อนโดยชุมชน สร้างขึ้นด้วยความรักและความหลงใหลใน Bitcoin และ Lightning Network


LN node แบบพกพานี้, Blixt เป็นเครื่องมือที่ทรงพลังมากในมือของผู้ใช้หลายคน หากพวกเขารู้วิธีใช้งานอย่างถูกต้อง ลองจินตนาการดูว่าคุณกำลังเดินรอบโลกพร้อมกับ LN node ในกระเป๋าของคุณเองและไม่มีใครจะรู้เลย


และไม่ต้องพูดถึงคุณสมบัติที่หลากหลายอื่น ๆ ที่มาพร้อมกับมัน ซึ่งมีเพียงไม่กี่แอปหรือไม่มีแอป wallet อื่นใดที่สามารถเสนอได้
