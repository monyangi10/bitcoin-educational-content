當您創建一個新的 Bitcoin Wallet 時，無論是通過軟件還是 Hardware Wallet，您都會收到一個由 12 或 24 個單詞組成的 Mnemonic 短語。這個短語非常重要，因為它是確保您的比特幣安全的所有私鑰的來源。因此必須小心保護，以保證在您的 Wallet 媒介破損、被盜或丟失的情況下，您的資金能夠復原。


在本教程中，我們將探討安全保存 Mnemonic 詞組的最佳做法，這樣您就不會失去對比特幣的存取權。


## 風險意識


Mnemonic 讓您可以完全不受限制地存取您所有的 bitcoins。任何擁有此短語的人都可以竊取您的資金，即使沒有實體存取權限，也可以存取託管您 Wallet 的媒介。


這表示，舉例來說，如果您在 Ledger 上使用 Bitcoin Wallet，任何可以存取您的 Mnemonic 詞組的人都可以竊取您所有的 bitcoins，即使沒有存取 Ledger 本身。這就是為什麼無論在什麼情況下，**您都不應該分享您的 Mnemonic 詞組**。


因此，當 Wallet 的媒體遺失、遭竊或損壞時，這個短語是讓您能夠恢復存取比特幣的唯一資訊。讓我們再以 Ledger 為例：如果您遺失了裝置，只要在新的 Ledger 或任何其他相容的 Wallet（無論是軟體或硬件）上輸入您的 Mnemonic 詞組，就可以恢復您的資金。


因此，請務必小心保存此短語，並將其存放在安全的地方，我們將在以下各節中詳細說明。


** 您的 Mnemonic 短語因此面臨兩大風險：盜竊和遺失**


盜竊主要有兩種方式：




- 有人實際取得您的備份，例如在入室行竊時或透過朋友取得；
- 您自願或非自願與他人分享您的刑期。


為避免您的 Mnemonic 短語備份遭竊，將其存放在安全的地方非常重要。我們會在下面的章節詳細說明這一點。


當遇到遠端竊取嘗試時，請永遠記住，無論在任何情況下，都不要分享您的 Mnemonic 詞組。網路釣魚嘗試很常見：欺詐電子郵件、仿冒官方錢包的網站或透過各種通訊管道直接要求。如果有人向您索取您的 passphrase，即使在緊急情況下也是騙局！盜賊冒充您的 Hardware Wallet 製造商的員工是很常見的，但請注意，無論在任何情況下，這些公司都不會向您索要您的 passphrase。因此，對於您收到的任何通訊，無論是透過電子郵件、電話、郵寄、社交網路或甚至當面，都要保持高度警覺。


當您在初始支援出現問題後，需要將您的短語輸入到 Hardware Wallet 或軟體中以恢復 Wallet 的存取權時，請花時間檢查您所使用的硬體或軟體的真實性和完整性。不要慌張，有條理地進行。


此外，在處理您的 Mnemonic 短語時要小心。確保您沒有被其他人或攝影機觀察到。


就遺失風險而言，主要有三種原因：備份媒體遺失、退化或評級錯誤。我們將在以下各節中詳細探討如何避免或最小化這三種風險。


## 支援


要儲存您的復原短語，您需要將它寫在實體媒體上，例如紙張或金屬。切勿使用數位媒體：不要將它儲存到文字檔、拍攝照片或儲存到密碼管理器中。與實體媒體相比，這些方法大大增加了攻擊面。因此，規則很清楚：使用紙張、硬紙板或金屬來儲存您的短語。


雖然將您的詞彙寫在一張簡單的紙上已經是很好的做法，但選擇金屬夾具（如不銹鋼）可增加安全性。這種固定器可以保護您的 Mnemonic 詞彙，避免因火災、水災或倒塌而影響儲存位置。


對於那些正在尋找經濟實惠的金屬詞句備份方案的人來說，[「*SAFU 忍者*」的 DIY 方法](https://jlopp.github.io/metal-Bitcoin-storage-reviews/reviews/safu-ninja/) 是一個極佳的解決方案。您只需在商店購買金屬墊圈、螺絲和螺帽。然後，您可以在每個墊圈上刻上您的句子，並確保對它們進行編號，然後將它們與螺帽一起組裝在螺絲上。這種低成本的方法已經具有很好的耐磨性。

如果您偏好投資於完整的金屬後備裝置，我建議您看看 [Jameson Lopp 的電阻測試](https://jlopp.github.io/metal-Bitcoin-storage-reviews/)，其中評估了市面上大部分的解決方案。我建議您選擇一體成型的支架，例如用來雕刻、沖壓或打孔的金屬板。這些裝置的阻力通常比使用獨立字母組合的系統大得多。


如果您選擇紙質 Wallet，您有幾種選擇：一張簡單的白紙、Hardware Wallet 隨附的紙板 Wallet，或是我們提供的可下載範本，您可以 [點選此處](https://github.com/PlanB-Network/Bitcoin-educational-content/blob/dev/resources/bet/Wallet-backup-sheet/assets/Mnemonic-sheet.pdf) 列印出來。

## 備份


現在您已經選擇了您的實體媒介，是時候寫下您的復原用語了！為了避免遺失您的 bitcoins，請遵循這些最佳實務。


首先，在您寫下句子時，請確定沒有其他人或攝影機在觀察您。


然後，花點時間把每個字寫得清楚易讀。多年之後，您可能需要重讀您的句子，或者其他人可能需要這樣做，作為繼承的一部分。因此，仔細書寫是非常重要的。


理論上，您可以只寫每個單字的前 4 個字母，因為在 BIP39 的 2048 個單字清單中，沒有兩個單字的前 4 個字母順序相同。但是，如果您的媒體有足夠的空間，我建議您完整地保存每個單詞。這在部分媒體損壞時會很有用。舉例來說，如果您只為單字 `accuse` 記下 `accu` 而字母 `u` 消失了，您可能必須在 `accuse`, `access`, `accident` 或 `account` 之間做選擇。另一方面，對於整個單詞，即使少了一個字母，還是很容易認出正確的單詞。


以正確的順序書寫單字也很重要。如果您有 24 個單字，但不知道它們的確切順序，就不可能恢復 Wallet 了。為字詞編碼也同樣重要：如果媒體損壞或字詞變得雜亂無章，編碼可以讓您輕鬆地將它們放回正確的順序中。


![SEED](assets/fr/03.webp)


此外，重要的是要了解，理論上，光是 Mnemonic 詞組並不總是足以讓您重新獲得比特幣的存取權。您還需要知道用來 generate 金鑰的衍生路徑。如果您使用的是具有標準路徑的 SingleSig Wallet，則恢復金鑰會相對簡單。但是，如果使用非標準路徑，即使擁有 Mnemonic 詞組，這也可能變得不可能。為了避免這個問題，我強烈建議您在媒體上記下您使用的帳戶的衍生路徑。您可以在 Wallet 軟體的設定中找到此資訊。例如，對於標準的 Taproot Wallet，預設的衍生路徑是 ：


```txt
m / 86' / 0' / 0' /
```


![SEED](assets/fr/04.webp)


如果您使用的是 Multisig Wallet 或 Wallet 包含復原金鑰的複雜腳本，例如 Liana 軟體提供的腳本，保存您的 ** 輸出腳本描述符** 是非常必要的。這些描述符除了復原短語之外，還包含您重新取得比特幣所需的所有資訊。


您也可以用與 Wallet 支援相關的其他資訊來豐富您的備份。例如，記下用於解鎖您的 Hardware Wallet 的 PIN 碼，或者如果您使用 COLDCARD，記下反釣魚字樣。


![SEED](assets/fr/05.webp)


另一方面，如果您使用 passphrase，請確保不要將其寫在與您的 Mnemonic 詞組相同的媒介上。passphrase 的目的是在您的 Wallet 被盜時保護您的 Wallet。要瞭解有關如何使用 passphrase 的更多資訊，請參閱本補充教學：


https://planb.network/tutorials/wallet/backup/passphrase-a26a0220-806c-44b4-af14-bafdeb1adce7

當您將 Mnemonic 詞組儲存至實體媒體後，強烈建議您在新建立的 Wallet 仍為空時進行復原測試。此測試包括寫下一則資訊樣本，故意刪除空的 Wallet，然後僅使用 Mnemonic 短語的實體備份嘗試還原。這可讓您檢查您的備份是否完整且沒有輸入錯誤。這也可讓您熟悉還原過程。如此一來，如果您將來需要進行復原，您就可以準備得更充分，並避免在真實情況下初次嘗試時的壓力。要瞭解有關如何進行此測試的更多資訊，請參閱此其他教程 ：


https://planb.network/tutorials/wallet/backup/recovery-test-5a75db51-a6a1-4338-a02a-164a8d91b895

最後，還有備份數量的問題。這個選擇完全取決於您的個人情況。限制備份份數，例如在媒體上只寫一次您的 Mnemonic 詞組，可以降低盜竊風險，但會增加遺失風險。相反地，複製數份可降低遺失的風險，但卻會增加盜竊的風險。因此，您必須根據自己的需求找出適當的平衡點，並決定您認為最適當的複本數量。


## 儲存


現在您已仔細備份 Mnemonic 詞組，是時候選擇合適的儲存位置了。這將取決於您的安全策略。無論如何，請選擇一個不引人注意的地方，不容易被人發現，但又可以定期檢查。也要確保它不會受到 Elements 的破壞，以防止基板受損。


我也建議您不要將 Mnemonic 存放在您沒有主權的地方，例如公證處或銀行的保險箱。這些選項看似安全，但卻意味著您必須仰賴第三方才能存取您的備份，這與 Bitcoin 的基本原則背道而馳。


為了增加安全性，我建議使用防偽塑膠袋或類似的密封系統。這可讓您檢查是否有人取得您的詞組。舉例來說，如果您將您的詞組存放在家中並接待客人，可能無法知道是否有人看過、記下或拍攝過您的詞組。防偽袋可讓這種驗證變得簡單：如果它完好無損，您就可以確定您的詞組仍然保密。這些完全不透明的保護套可在線上或 Bitcoin 專門店購買。


![SEED](assets/fr/06.webp)


最後，當您的短語存放在防偽信封中時，別忘了記下信封的唯一識別碼。這可讓您在檢查時驗證其真偽。


## 時間管理


既然您的詞組已小心儲存，定期監控就很重要了。定期檢查您的詞組是否仍在其儲存位置中，且其不透明封套未被打開。


在這些檢查過程中，您也可以打開信封檢查底稿的狀況。請確定它沒有損壞，而且句子仍然清晰可辨。如果您發現任何損壞的跡象，最好從 Hardware Wallet 建立新的副本。檢查這份新的複本是否正常運作，然後將損壞的備份乾淨銷毀，以避免任何洩漏的風險。
