Arch Linux - это дистрибутив, известный своей надежностью, производительностью и адаптивностью, особенно для целей разработки. Он предлагает отличную стабильность и среду, благоприятную для настройки, поддерживаемую чрезвычайно быстрым и надежным менеджером пакетов. Философия Arch Linux основана на принципе **KISS** (*Keep It Simple, Stupid*): предложить легкий, простой, быстрый и незагроможденный дистрибутив, оставляя при этом большую свободу пользователю.



## Почему стоит выбрать Arch Linux?





- Свободный и открытый исходный код**: Как и большинство дистрибутивов Linux, Arch Linux полностью бесплатен. Лицензионные платежи не взимаются, что делает его отличным выбором для студентов, фрилансеров и энтузиастов.
- Философия KISS**: Arch разработан, чтобы быть простым, легким и эффективным. Он предоставляет только самое необходимое, позволяя вам создавать свое окружение по меню.
- Пакетный менеджер Pacman**: Pacman - это быстрый, надежный и хорошо продуманный менеджер пакетов. Он обеспечивает эффективную установку и обновление программ, а также точно управляет зависимостями.
- Исчерпывающая документация и активное сообщество**: [Arch Wiki](https://wiki.archlinux.org) - это, пожалуй, одна из лучших технических документаций в мире Linux. Это золотая жила для понимания того, что вы делаете. Сообщество, состоящее в основном из опытных профи, очень активно и может помочь вам, если вы застряли, при условии, что вы предварительно провели небольшое исследование.

## Установка Arch Linux



## Загрузка с USB-носителя





- Полностью выключите компьютер
- Подключите загрузочный USB-носитель
- Перезагрузитесь и войдите в BIOS/UEFI, нажав **F1**, **Escape**, **F9** и т.д. (в зависимости от модели)
- В меню загрузки выберите USB-носитель в качестве загрузочного устройства. Если все настроено правильно, вы попадете на экран загрузки Arch Linux Interface.



## Запуск установки



На экране загрузки выберите первый вариант для запуска установки. Обратите внимание, что Arch Linux не предлагает графического установщика. После запуска вы перейдете в терминал в режиме root.

### Конфигурация клавиатуры



Вы можете отобразить доступные макеты с помощью:



```shell
localectl list-keymaps
```



![0_6](assets/fr/06.webp)



Затем загрузите макет с помощью:



```shell
loadkeys nom-disposition
```



По умолчанию клавиатура английская (qwerty), но вы можете переключиться на `azerty` с помощью `loadkeys fr`.



### Установка даты и времени



Arch Linux использует инструмент `timedatectl` для управления системными часами.



![0_7](assets/fr/07.webp)





- Установите часовой пояс с помощью:


```shell
timedatectl set-timezone Europe/Paris
```





- Убедитесь, что автоматическая синхронизация включена с помощью:


```shell
timedatectl status
```





- При необходимости активируйте его:


```shell
timedatectl set-ntp true
```




Это активирует NTP, протокол для автоматической синхронизации с серверами времени. Этот шаг важен, чтобы избежать ошибок с датой при установке пакетов или настройке сертификатов SSL в дальнейшем.



### Разбиение диска на разделы





- Проверьте, загружается ли ваша система в **UEFI** или **BIOS** с:



```shell
ls /sys/firmware/efi
```



Если файл существует, вы находитесь в **UEFI**. В противном случае вы находитесь в **BIOS (Legacy)**.




- Перечислите доступные диски:


```shell
lsblk
```



![0_8](assets/fr/08.webp)





- Запуск диспетчера разделов:



```shell
cfdisk /dev/nom-du-disque
```



Выберите **GPT**, если вы находитесь в UEFI, **DOS**, если вы находитесь в BIOS.



![0_9](assets/fr/09.webp)



#### Баллы для создания




- В режиме UEFI**



| Point de montage sur le système installé | Partition                 | Type de partition       | Taille suggérée |
| ---------------------------------------- | ------------------------- | ----------------------- | --------------- |
| /boot1                                   | /dev/efi_system_partition | Partition système EFI   | 1 Go            |
| [SWAP]                                   | /dev/swap_partition       | Espace d’échange (swap) | Au moins 4 Go   |
| /                                        | /dev/root_partition       | Racine Linux x86-64 (/) | Reste du disque |

- В BIOS



| Point de montage sur le système installé | Partition           | Type de partition       | Taille suggérée |
| ---------------------------------------- | ------------------- | ----------------------- | --------------- |
| [SWAP]                                   | /dev/swap_partition | Espace d’échange (swap) | Au moins 4 Go   |
| /                                        | /dev/root_partition | Linux                   | Reste du disque |

![0_10](assets/fr/10.webp)



Выберите **Запись**, введите **Да**, затем **Выход**.



### Форматирование разделов





- UEFI**:



```shell
mkfs.fat -F32 /dev/sda1
mkswap /dev/sda2
swapon /dev/sda2
mkfs.ext4 /dev/sda3
```





- BIOS**:



```shell
mkswap /dev/sda1
swapon /dev/sda1
mkfs.ext4 /dev/sda2
```



![0_11](assets/fr/11.webp)



### Базовая установка системы



Смонтируйте раздел **root**:





- В BIOS:


```shell
mount /dev/sda2 /mnt
```




- на UEFI:


```shell
mount /dev/sda3 /mnt
```



Затем установите необходимые пакеты:



```shell
pacstrap -K /mnt base linux linux-firmware
```



![0_12](assets/fr/12.webp)



generate файл **fstab**, который позволяет операционной системе автоматически управлять монтированием разделов при каждой загрузке без ручного вмешательства:



```shell
genfstab -U /mnt >> /mnt/etc/fstab
```



Войдите в среду **Chroot**:



```shell
arch-chroot /mnt
```

### Конфигурация системы





- Установите текстовый редактор для редактирования:



```shell
pacman -S vim
```





- Установите язык:


Отредактируйте `/etc/locale.gen`, затем удалите строку `en_US.UTF-8 UTF-8`



![0_14](assets/fr/14.webp)





- Установите имя машины:



```shell
echo nom_machine > /etc/hostname
```





- Установите пароль root:



```shell
passwd
```



![0_15](assets/fr/15.webp)



### Установка GRUB



Установите:



```shell
pacman -S grub
```



![0_16](assets/fr/16.webp)



После загрузки необходимо установить его в соответствии с форматом раздела диска.




- Для **BIOS**:



```shell
grub-install --target=i386-pc /dev/sda
grub-mkconfig -o /boot/grub/grub.cfg
```



![0_17](assets/fr/17.webp)





- Для **UEFI**:



```shell
pacman -S efibootmgr
mkdir /boot/efi
mount /dev/sda1 /boot/efi
grub-install --target=x86_64-efi --efi-directory=/boot/efi --bootloader-id=GRUB
grub-mkconfig -o /boot/grub/grub.cfg
```



### Доработка





- Выйдите из среды chroot:


```shell
exit
```





- Удалите разделы:


```shell
umount -R /mnt
```





- Перезапуск:


```shell
reboot
```



При запуске войдите в систему, используя логин и пароль **root**.

## Подключение к сети после перезагрузки



Может случиться так, что при перезагрузке ни одно сетевое соединение не будет активным. Список интерфейсов можно составить с помощью:



```shell
ip link
```



Затем настройте сеть Interface, введя в терминале следующий текст



```shell
cat <<EOF > /etc/systemd/network/20-wired.network
[Match]
Name=nom_de_l_interface

[Network]
DHCP=yes
EOF
```



## Графика Interface (GNOME)



По умолчанию **Arch Linux** не содержит графического Interface. Чтобы добавить его:



Обновите систему:



```shell
pacman -Syu
```



Установите **Xorg** с помощью следующей команды и каждый раз нажимайте клавишу Enter, чтобы сохранить выбор по умолчанию:



```shell
pacman -S xorg
```



![0_19](assets/fr/19.webp)



Установите **GNOME** с помощью следующей команды и каждый раз вводите enter, чтобы сохранить выбор по умолчанию:



```shell
pacman -S gnome gnome-extra
```



![0_20](assets/fr/20.webp)



Активируйте **менеджер сеансов**:



```shell
systemctl enable gdm
systemctl start gdm
```



Система автоматически перезагрузится, и вы получите графический вход в систему Interface. Войдите в систему, используя имя пользователя root и пароль.



![0_21](assets/fr/21.webp)



## Создание пользователя



Попав в **Interface GNOME**, вам нужно будет создать нового пользователя для большей безопасности и безопасного, безрискового использования. Войдите в приложения и выберите опцию "консоль", чтобы запустить терминал.





- Добавить пользователя:



```shell
useradd -m -G wheel -s /bin/bash nom_utilisateur
passwd nom_utilisateur
```



![0_22](assets/fr/22.webp)





- Установите **sudo**:


```shell
pacman -S sudo
```





- Активируйте права **sudo**:



```shell
EDITOR=nano visudo
```





- Затем откомментируйте строку:



```shell
%wheel ALL=(ALL:ALL) ALL
```





- Перезапустите систему и войдите в нее под своим именем пользователя.



![0_23](assets/fr/23.webp)



![0_24](assets/fr/24.webp)



## Установка программного обеспечения



Поскольку Arch Linux минималистичен, многие программы не устанавливаются по умолчанию. Чтобы добавить все необходимое, введите следующую команду:



```shell
pacman -S nom_du_paquet_a_installe
```



Например, чтобы установить текстовый редактор **nano**, вы можете набрать:



```shell
pacman -S nano
```



Чтобы установить легкий веб-браузер, например `firefox`, используйте:



```shell
pacman -S firefox
```



Наконец, если вы хотите добавить необходимые сетевые инструменты, такие как `net-tools`, команда будет выглядеть так:



```shell
pacman -S net-tools
```



Не забывайте, что вы можете установить несколько пакетов одной командой, перечислив их по отдельности:



```shell
pacman -S vim firefox net-tools
```

Предоставляя только самое необходимое, она обеспечивает легкую и высокопроизводительную основу, которую легко настроить под ваши нужды.
