![cover](assets/cover.webp)



آرچ لینوکس یک توزیع مشهور به خاطر استحکام، عملکرد و سازگاری‌اش است، به‌ویژه برای اهداف توسعه. این توزیع پایداری عالی و محیطی مناسب برای سفارشی‌سازی ارائه می‌دهد که توسط یک مدیر بسته فوق‌العاده سریع و قابل اعتماد پشتیبانی می‌شود. فلسفه آن بر اساس اصل **KISS** (*Keep It Simple, Stupid*) است: ارائه یک توزیع سبک، ساده، سریع و بدون پیچیدگی، در حالی که آزادی زیادی به کاربر می‌دهد.



## چرا Arch Linux را انتخاب کنیم؟





- **رایگان و متن‌باز**: مانند اکثر توزیع‌های لینوکس، آرچ لینوکس کاملاً رایگان است. هیچ هزینه‌ی مجوزی وجود ندارد، که آن را به انتخابی عالی برای دانشجویان، فریلنسرها یا علاقه‌مندان تبدیل می‌کند.
- **فلسفه KISS**: آرچ به گونه‌ای طراحی شده است که ساده، سبک و کارآمد باشد. این تنها ملزومات را فراهم می‌کند و به شما اجازه می‌دهد محیط خود را به صورت سفارشی بسازید.
- مدیر بسته **Pacman**: Pacman یک مدیر بسته سریع، قابل اعتماد و خوش‌طراحی است. این امکان را فراهم می‌کند تا نرم‌افزارها به‌طور کارآمد نصب و به‌روزرسانی شوند و وابستگی‌ها را با دقت مدیریت می‌کند.
- **مستندات جامع و یک جامعه فعال**: [ویکی آرچ](https://wiki.archlinux.org) احتمالاً یکی از بهترین مستندات فنی در دنیای لینوکس است. این یک معدن طلا برای درک آنچه انجام می‌دهید است. جامعه، که عمدتاً از پروفایل‌های با تجربه تشکیل شده، بسیار فعال است و می‌تواند به شما کمک کند اگر گیر کردید، به شرطی که قبلاً کمی تحقیق کرده باشید.

## نصب آرچ لینوکس



## راه‌اندازی از طریق کلید USB





- رایانه خود را به طور کامل خاموش کنید
- کلید USB قابل بوت را وصل کنید
- راه‌اندازی مجدد و ورود به BIOS/UEFI با فشردن **F1**، **Escape**، **F9** و غیره (بسته به مدل شما)
- در منوی بوت، کلید USB را به عنوان دستگاه بوت انتخاب کنید. اگر همه چیز به درستی تنظیم شده باشد، به صفحه بوت Arch Linux Interface هدایت خواهید شد.



## راه‌اندازی نصب



در صفحه بوت، گزینه اول را برای شروع نصب انتخاب کنید. توجه داشته باشید که Arch Linux نصب‌کننده گرافیکی ارائه نمی‌دهد. پس از شروع، به یک ترمینال در حالت ریشه منتقل خواهید شد.



![0_3](assets/fr/03.webp)



![0_4](assets/fr/04.webp)



![0_5](assets/fr/05.webp)



### پیکربندی صفحه‌کلید



می‌توانید طرح‌بندی‌های موجود را با: نمایش دهید



```shell
localectl list-keymaps
```



![0_6](assets/fr/06.webp)



سپس یک چیدمان بارگذاری کنید با:



```shell
loadkeys nom-disposition
```



به طور پیش‌فرض، صفحه‌کلید به زبان انگلیسی (qwerty) است، اما می‌توانید با استفاده از `loadkeys fr` به `azerty` تغییر دهید.



### تنظیم تاریخ و زمان



آرچ لینوکس از ابزار `timedatectl` برای مدیریت ساعت سیستم استفاده می‌کند.



![0_7](assets/fr/07.webp)





- منطقه زمانی خود را تنظیم کنید با:


```shell
timedatectl set-timezone Europe/Paris
```





- بررسی کنید که همگام‌سازی خودکار با: فعال است


```shell
timedatectl status
```





- در صورت لزوم آن را فعال کنید:


```shell
timedatectl set-ntp true
```




این NTP را فعال می‌کند، پروتکلی برای همگام‌سازی خودکار با سرورهای زمان. این مرحله برای جلوگیری از خطاهای تاریخ هنگام نصب بسته‌ها یا پیکربندی گواهینامه‌های SSL در مراحل بعدی مهم است.



### پارتیشن‌بندی دیسک





- بررسی کنید که سیستم شما در **UEFI** یا **BIOS** بوت می‌شود با:



```shell
ls /sys/firmware/efi
```



اگر فایل وجود داشته باشد، شما در **UEFI** هستید. در غیر این صورت، شما در **BIOS (Legacy)** هستید.




- لیست دیسک‌های موجود:


```shell
lsblk
```



![0_8](assets/fr/08.webp)





- مدیر پارتیشن را شروع کنید:



```shell
cfdisk /dev/nom-du-disque
```



اگر در UEFI هستید، **GPT** را انتخاب کنید، اگر در BIOS هستید، **DOS** را انتخاب کنید.



![0_9](assets/fr/09.webp)



#### امتیازات برای ایجاد




- در حالت **UEFI**



| Point de montage sur le système installé | Partition                 | Type de partition       | Taille suggérée |
| ---------------------------------------- | ------------------------- | ----------------------- | --------------- |
| /boot1                                   | /dev/efi_system_partition | Partition système EFI   | 1 Go            |
| [SWAP]                                   | /dev/swap_partition       | Espace d’échange (swap) | Au moins 4 Go   |
| /                                        | /dev/root_partition       | Racine Linux x86-64 (/) | Reste du disque |

- در BIOS



| Point de montage sur le système installé | Partition           | Type de partition       | Taille suggérée |
| ---------------------------------------- | ------------------- | ----------------------- | --------------- |
| [SWAP]                                   | /dev/swap_partition | Espace d’échange (swap) | Au moins 4 Go   |
| /                                        | /dev/root_partition | Linux                   | Reste du disque |

![0_10](assets/fr/10.webp)



انتخاب کنید **Write**، تایپ کنید **yes**، سپس **Quit**.



### پارتیشن‌بندی فرمت‌بندی





- **UEFI**:



```shell
mkfs.fat -F32 /dev/sda1
mkswap /dev/sda2
swapon /dev/sda2
mkfs.ext4 /dev/sda3
```





- **BIOS**:



```shell
mkswap /dev/sda1
swapon /dev/sda1
mkfs.ext4 /dev/sda2
```



![0_11](assets/fr/11.webp)



### نصب سیستم پایه



پارتیشن **ریشه** را مونت کنید:





- در BIOS:


```shell
mount /dev/sda2 /mnt
```




- در UEFI:


```shell
mount /dev/sda3 /mnt
```



سپس بسته‌های ضروری را نصب کنید:



```shell
pacstrap -K /mnt base linux linux-firmware
```



![0_12](assets/fr/12.webp)



generate فایل **fstab**، که به سیستم‌عامل امکان می‌دهد تا بدون دخالت دستی، مدیریت خودکار مونت پارتیشن‌ها را در هر بار راه‌اندازی انجام دهد:



```shell
genfstab -U /mnt >> /mnt/etc/fstab
```



وارد محیط **Chroot** شوید:



```shell
arch-chroot /mnt
```



![0_13](assets/fr/13.webp)



### پیکربندی سیستم





- یک ویرایشگر متن برای ویرایش نصب کنید:



```shell
pacman -S vim
```





- زبان را تنظیم کنید:


فایل `/etc/locale.gen` را ویرایش کنید و سپس خط `en_US.UTF-8 UTF-8` را از حالت کامنت خارج کنید.



![0_14](assets/fr/14.webp)





- نام ماشین را تنظیم کنید:



```shell
echo nom_machine > /etc/hostname
```





- رمز عبور ریشه را تنظیم کنید:



```shell
passwd
```



![0_15](assets/fr/15.webp)



### نصب GRUB



نصب کنید:



```shell
pacman -S grub
```



![0_16](assets/fr/16.webp)



پس از دانلود، باید آن را مطابق با فرمت پارتیشن دیسک نصب کنید.




- برای **BIOS**:



```shell
grub-install --target=i386-pc /dev/sda
grub-mkconfig -o /boot/grub/grub.cfg
```



![0_17](assets/fr/17.webp)





- برای **UEFI**:



```shell
pacman -S efibootmgr
mkdir /boot/efi
mount /dev/sda1 /boot/efi
grub-install --target=x86_64-efi --efi-directory=/boot/efi --bootloader-id=GRUB
grub-mkconfig -o /boot/grub/grub.cfg
```



### نهایی‌سازی





- از محیط chroot خارج شوید:


```shell
exit
```





- پارتیشن‌ها را بردارید:


```shell
umount -R /mnt
```





- راه‌اندازی مجدد:


```shell
reboot
```



در هنگام راه‌اندازی، با ورود **root** و رمز عبور خود وارد شوید.



![0_18](assets/fr/18.webp)


## اتصال شبکه پس از راه‌اندازی مجدد



ممکن است هنگام راه‌اندازی مجدد، هیچ اتصال شبکه‌ای فعال نباشد. می‌توانید رابط‌ها را با استفاده از:



```shell
ip link
```



سپس با وارد کردن متن زیر در ترمینال، شبکه Interface را پیکربندی کنید.



```shell
cat <<EOF > /etc/systemd/network/20-wired.network
[Match]
Name=nom_de_l_interface

[Network]
DHCP=yes
EOF
```



## گرافیک Interface (GNOME)



به طور پیش‌فرض، **Arch Linux** شامل هیچ Interface گرافیکی نیست. برای اضافه کردن یکی:



سیستم را به‌روزرسانی کنید:



```shell
pacman -Syu
```



**Xorg** را با استفاده از فرمان زیر نصب کنید و هر بار برای حفظ تنظیمات پیش‌فرض، کلید Enter را فشار دهید:



```shell
pacman -S xorg
```



![0_19](assets/fr/19.webp)



**GNOME** را با استفاده از فرمان زیر نصب کنید و هر بار برای حفظ انتخاب‌های پیش‌فرض، Enter را فشار دهید:



```shell
pacman -S gnome gnome-extra
```



![0_20](assets/fr/20.webp)



فعال کردن **مدیر جلسه**:



```shell
systemctl enable gdm
systemctl start gdm
```



سیستم به‌طور خودکار راه‌اندازی مجدد می‌شود و شما صفحه ورود گرافیکی Interface را مشاهده می‌کنید. با نام کاربری و رمز عبور root وارد شوید.



![0_21](assets/fr/21.webp)



## ایجاد کاربر



هنگامی که در **Interface GNOME** هستید، برای امنیت بیشتر و استفاده ایمن و بدون ریسک، باید یک کاربر جدید ایجاد کنید. وارد برنامه‌ها شوید و گزینه "کنسول" را برای راه‌اندازی ترمینال انتخاب کنید.





- یک کاربر اضافه کنید:



```shell
useradd -m -G wheel -s /bin/bash nom_utilisateur
passwd nom_utilisateur
```



![0_22](assets/fr/22.webp)





- نصب **sudo**:


```shell
pacman -S sudo
```





- فعال کردن **sudo** حقوق:



```shell
EDITOR=nano visudo
```





- سپس خط را از حالت توضیح خارج کنید:



```shell
%wheel ALL=(ALL:ALL) ALL
```





- سیستم را مجدداً راه‌اندازی کرده و با نام کاربری خود وارد شوید.



![0_23](assets/fr/23.webp)

## نصب نرم‌افزار



از آنجا که Arch Linux مینیمالیستی است، بسیاری از نرم‌افزارها به‌طور پیش‌فرض نصب نشده‌اند. برای افزودن آنچه نیاز دارید، دستور زیر را تایپ کنید:



```shell
pacman -S nom_du_paquet_a_installe
```



به عنوان مثال، برای نصب ویرایشگر متن **nano**، می‌توانید تایپ کنید:



```shell
pacman -S nano
```



برای نصب یک مرورگر وب سبک مانند `firefox`، از:



```shell
pacman -S firefox
```



در نهایت، اگر می‌خواهید ابزارهای شبکه ضروری مانند `net-tools` را اضافه کنید، دستور به صورت زیر خواهد بود:



```shell
pacman -S net-tools
```



فراموش نکنید که می‌توانید چندین بسته را با یک فرمان نصب کنید، با فهرست کردن آن‌ها به‌صورت جداگانه:



```shell
pacman -S vim firefox net-tools
```



آرچ لینوکس به دلیل پایداری فوق‌العاده، فلسفه مینیمالیستی و استحکام خود برجسته است و آن را به انتخابی ایده‌آل برای محیط‌های توسعه تبدیل می‌کند. با ارائه تنها ضروریات، پایه‌ای سبک و با عملکرد بالا فراهم می‌کند که به راحتی می‌توان آن را بر اساس نیازهای خاص خود سفارشی کرد. این رویکرد مینیمالیستی همچنین کنترل بیشتری بر سیستم را ترجیح می‌دهد و با محدود کردن سطح حمله، امنیت را تقویت می‌کند. به لطف جامعه فعال و مستندات جامع آن، آرچ لینوکس می‌تواند به شما در ایجاد محیطی امن و انعطاف‌پذیر که برای توسعه حرفه‌ای بهینه شده است، کمک کند.
