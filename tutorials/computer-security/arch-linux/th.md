Arch Linux เป็นดิสทริบิวชันที่มีชื่อเสียงในด้านความทนทาน ประสิทธิภาพ และความสามารถในการปรับตัว โดยเฉพาะอย่างยิ่งสำหรับวัตถุประสงค์ในการพัฒนา มันมีความเสถียรที่ยอดเยี่ยมและสภาพแวดล้อมที่เอื้อต่อการปรับแต่ง โดยได้รับการสนับสนุนจากตัวจัดการแพ็กเกจที่รวดเร็วและเชื่อถือได้อย่างยิ่ง ปรัชญาของมันอิงตามหลักการ **KISS** (*Keep It Simple, Stupid*): เพื่อเสนอการกระจายที่เบา เรียบง่าย รวดเร็ว และไม่รก ในขณะที่ปล่อยให้ผู้ใช้มีอิสระอย่างมาก



## ทำไมถึงเลือก Arch Linux?





- ฟรีและโอเพ่นซอร์ส**: เช่นเดียวกับลีนุกซ์ดิสทริบิวชันส่วนใหญ่ Arch Linux เป็นซอฟต์แวร์ที่ฟรีทั้งหมด ไม่มีค่าธรรมเนียมใบอนุญาต ทำให้เป็นตัวเลือกที่ยอดเยี่ยมสำหรับนักเรียน ฟรีแลนซ์ หรือผู้ที่ชื่นชอบ
- ปรัชญา KISS**: Arch ถูกออกแบบมาให้เรียบง่าย เบา และมีประสิทธิภาพ มันให้เพียงสิ่งจำเป็นเท่านั้น ทำให้คุณสามารถสร้างสภาพแวดล้อมของคุณเองได้ตามต้องการ
- Pacman** package manager: Pacman เป็นตัวจัดการแพ็คเกจที่รวดเร็ว เชื่อถือได้ และออกแบบมาอย่างดี ช่วยให้การติดตั้งและอัปเดตซอฟต์แวร์เป็นไปอย่างมีประสิทธิภาพ และจัดการการพึ่งพาได้อย่างแม่นยำ
- เอกสารที่ครอบคลุมและชุมชนที่มีความเคลื่อนไหว**: [Arch Wiki](https://wiki.archlinux.org) เป็นหนึ่งในเอกสารทางเทคนิคที่ดีที่สุดในโลกของ Linux เป็นแหล่งข้อมูลที่มีค่าในการทำความเข้าใจสิ่งที่คุณกำลังทำอยู่ ชุมชนซึ่งส่วนใหญ่ประกอบด้วยผู้มีประสบการณ์มีความเคลื่อนไหวมากและสามารถช่วยคุณได้หากคุณติดขัด โดยมีเงื่อนไขว่าคุณได้ทำการค้นคว้าล่วงหน้ามาบ้างแล้ว



## การติดตั้งและการกำหนดค่า



### ข้อกำหนดเบื้องต้น



วัสดุที่ต้องใช้:





- คีย์ USB ขนาดอย่างน้อย **8 GB**
- RAM ขั้นต่ำ 2 GB**
- คอมพิวเตอร์ที่มีพื้นที่ว่างบนดิสก์อย่างน้อย 20 GB



### ดาวน์โหลด



![0_1](assets/fr/01.webp)



ตั้งแต่ปี 2017 เป็นต้นมา Arch Linux ไม่รองรับสถาปัตยกรรม 32 บิตอีกต่อไป มีเฉพาะเวอร์ชัน 64 บิตเท่านั้นที่มีให้ใช้งาน





- เยี่ยมชม[เว็บไซต์ทางการ](https://mir.archlinux.fr/iso/latest/)เพื่อดาวน์โหลดเวอร์ชันล่าสุดของไฟล์ ISO อย่างเป็นทางการ



### สร้างคีย์ที่สามารถบูตได้



ในการสร้าง USB แฟลชไดรฟ์ที่สามารถบูตได้ คุณสามารถใช้เครื่องมืออย่าง **Balena Etcher**:





- ดาวน์โหลด Balena Etcher จาก[เว็บไซต์ทางการ](https://etcher.balena.io)
- เปิดซอฟต์แวร์ เลือกภาพ ISO ของ Arch Linux
- เลือกคีย์ USB ของคุณเป็นอุปกรณ์เป้าหมาย
- คลิกที่ **Flash** เพื่อเริ่มสร้างคีย์ที่สามารถบูตได้

## การบูตจาก USB key





- ปิดคอมพิวเตอร์ของคุณอย่างสมบูรณ์
- เสียบกุญแจ USB ที่สามารถบู๊ตได้
- รีบูตและเข้าสู่ BIOS/UEFI โดยกด **F1**, **Escape**, **F9**, เป็นต้น (ขึ้นอยู่กับรุ่นของคุณ)
- ในเมนูบูต ให้เลือก USB key เป็นอุปกรณ์บูต หากทุกอย่างตั้งค่าอย่างถูกต้อง คุณจะถูกนำไปยังหน้าจอบูต Arch Linux Interface

### การกำหนดค่าคีย์บอร์ด



คุณสามารถแสดงเลย์เอาต์ที่มีอยู่ด้วย:



```shell
localectl list-keymaps
```



![0_6](assets/fr/06.webp)



จากนั้นโหลดเลย์เอาต์ด้วย:



```shell
loadkeys nom-disposition
```



ตามค่าเริ่มต้น แป้นพิมพ์จะเป็นภาษาอังกฤษ (qwerty) แต่คุณสามารถเปลี่ยนเป็น `azerty` ด้วยคำสั่ง `loadkeys fr`.



### การตั้งค่าวันที่และเวลา



Arch Linux ใช้เครื่องมือ `timedatectl` เพื่อจัดการนาฬิกาของระบบ



![0_7](assets/fr/07.webp)





- ตั้งเขตเวลาของคุณด้วย:


```shell
timedatectl set-timezone Europe/Paris
```





- ตรวจสอบว่าการซิงโครไนซ์อัตโนมัติเปิดใช้งานด้วย:


```shell
timedatectl status
```





- เปิดใช้งานหากจำเป็น:


```shell
timedatectl set-ntp true
```




สิ่งนี้จะเปิดใช้งาน NTP ซึ่งเป็นโปรโตคอลสำหรับการซิงโครไนซ์อัตโนมัติกับเซิร์ฟเวอร์เวลา ขั้นตอนนี้มีความสำคัญเพื่อหลีกเลี่ยงข้อผิดพลาดของวันที่เมื่อทำการติดตั้งแพ็คเกจหรือกำหนดค่าใบรับรอง SSL ในภายหลัง



### การแบ่งพาร์ติชันดิสก์





- ตรวจสอบว่าระบบของคุณบูตใน **UEFI** หรือ **BIOS** ด้วย:



```shell
ls /sys/firmware/efi
```



หากไฟล์มีอยู่ แสดงว่าคุณอยู่ใน **UEFI** มิฉะนั้น คุณจะอยู่ใน **BIOS (Legacy)**




- แสดงรายการดิสก์ที่มีอยู่:


```shell
lsblk
```



![0_8](assets/fr/08.webp)





- เริ่มตัวจัดการพาร์ติชัน:



```shell
cfdisk /dev/nom-du-disque
```



เลือก **GPT** หากคุณอยู่ใน UEFI, **DOS** หากคุณอยู่ใน BIOS.



![0_9](assets/fr/09.webp)



#### คะแนนที่จะสร้าง




- ในโหมด **UEFI**




| จุดติดตั้งในระบบที่ติดตั้งแล้ว | พาร์ติชัน                 | ประเภทพาร์ติชัน       | ขนาดที่แนะนำ |
| ---------------------------------------- | ------------------------- | ----------------------- | --------------- |
| /boot1                                   | /dev/efi_system_partition | พาร์ติชันระบบ EFI   | 1 GB            |
| [SWAP]                                   | /dev/swap_partition       | พื้นที่ swap (swap) | อย่างน้อย 4 GB   |
| /                                        | /dev/root_partition       | Linux x86-64 root (/) | ส่วนที่เหลือของดิสก์ |

- ใน BIOS




| จุดติดตั้งในระบบที่ติดตั้งแล้ว | พาร์ติชัน           | ประเภทพาร์ติชัน       | ขนาดที่แนะนำ |
| ---------------------------------------- | ------------------- | ----------------------- | --------------- |
| [SWAP]                                   | /dev/swap_partition | พื้นที่ swap (swap) | อย่างน้อย 4 GB   |
| /                                        | /dev/root_partition | Linux                   | ส่วนที่เหลือของดิสก์ |

![0_10](assets/fr/10.webp)



เลือก **เขียน**, พิมพ์ **ใช่**, จากนั้น **ออก**.



### การฟอร์แมตพาร์ติชัน





- UEFI**:



```shell
mkfs.fat -F32 /dev/sda1
mkswap /dev/sda2
swapon /dev/sda2
mkfs.ext4 /dev/sda3
```





- BIOS**:



```shell
mkswap /dev/sda1
swapon /dev/sda1
mkfs.ext4 /dev/sda2
```



![0_11](assets/fr/11.webp)



### การติดตั้งระบบพื้นฐาน



เมานต์พาร์ติชัน **root**:





- บน BIOS:


```shell
mount /dev/sda2 /mnt
```




- บน UEFI:


```shell
mount /dev/sda3 /mnt
```



จากนั้นติดตั้งแพ็คเกจที่จำเป็น:



```shell
pacstrap -K /mnt base linux linux-firmware
```



![0_12](assets/fr/12.webp)



สร้างไฟล์ **fstab** ซึ่งช่วยให้ระบบปฏิบัติการจัดการการเมานต์พาร์ติชันโดยอัตโนมัติในแต่ละการบูต โดยไม่ต้องมีการแทรกแซงด้วยตนเอง:



```shell
genfstab -U /mnt >> /mnt/etc/fstab
```



เข้าสู่สภาพแวดล้อม **Chroot**:



```shell
arch-chroot /mnt
```



![0_13](assets/fr/13.webp)



### การกำหนดค่าระบบ





- ติดตั้งโปรแกรมแก้ไขข้อความเพื่อแก้ไข:



```shell
pacman -S vim
```





- ตั้งค่าภาษา:


แก้ไข `/etc/locale.gen` จากนั้นยกเลิกการคอมเมนต์บรรทัด `en_US.UTF-8 UTF-8`



![0_14](assets/fr/14.webp)





- ตั้งชื่อเครื่อง:



```shell
echo nom_machine > /etc/hostname
```





- ตั้งรหัสผ่านรูท:



```shell
passwd
```



![0_15](assets/fr/15.webp)



### การติดตั้ง GRUB



ติดตั้ง:



```shell
pacman -S grub
```



![0_16](assets/fr/16.webp)



เมื่อดาวน์โหลดแล้ว คุณจำเป็นต้องติดตั้งตามรูปแบบพาร์ทิชันดิสก์




- สำหรับ **BIOS**:



```shell
grub-install --target=i386-pc /dev/sda
grub-mkconfig -o /boot/grub/grub.cfg
```



![0_17](assets/fr/17.webp)





- สำหรับ **UEFI**:



```shell
pacman -S efibootmgr
mkdir /boot/efi
mount /dev/sda1 /boot/efi
grub-install --target=x86_64-efi --efi-directory=/boot/efi --bootloader-id=GRUB
grub-mkconfig -o /boot/grub/grub.cfg
```



### การสรุปผล





- ออกจากสภาพแวดล้อม chroot:


```shell
exit
```





- ลบพาร์ติชัน:


```shell
umount -R /mnt
```





- เริ่มใหม่:


```shell
reboot
```



เมื่อเริ่มต้นระบบ ให้เข้าสู่ระบบด้วย **root** ล็อกอินและรหัสผ่านของคุณ

## การเชื่อมต่อเครือข่ายหลังจากรีบูต



อาจเกิดขึ้นได้ว่าไม่มีการเชื่อมต่อเครือข่ายที่ใช้งานอยู่เมื่อรีบูต คุณสามารถแสดงรายการอินเทอร์เฟซได้ด้วย:



```shell
ip link
```



จากนั้นกำหนดค่าเครือข่าย Interface โดยป้อนข้อความต่อไปนี้ในเทอร์มินัล



```shell
cat <<EOF > /etc/systemd/network/20-wired.network
[Match]
Name=nom_de_l_interface

[Network]
DHCP=yes
EOF
```



## Interface กราฟิก (GNOME)



ตามค่าเริ่มต้น **Arch Linux** ไม่มี Interface กราฟิก ในการเพิ่ม:



อัปเดตระบบ:



```shell
pacman -Syu
```



ติดตั้ง **Xorg** ด้วยคำสั่งต่อไปนี้และกด Enter ทุกครั้งเพื่อเลือกค่าปริยาย:



```shell
pacman -S xorg
```



![0_19](assets/fr/19.webp)



ติดตั้ง **GNOME** ด้วยคำสั่งต่อไปนี้และกด Enter ทุกครั้งเพื่อเลือกค่าปริยาย:



```shell
pacman -S gnome gnome-extra
```



![0_20](assets/fr/20.webp)



เปิดใช้งาน **ตัวจัดการเซสชัน**:



```shell
systemctl enable gdm
systemctl start gdm
```



ระบบจะรีบูตโดยอัตโนมัติและคุณจะได้รับการเข้าสู่ระบบกราฟิก Interface ให้เข้าสู่ระบบด้วยชื่อผู้ใช้ root และรหัสผ่าน



![0_21](assets/fr/21.webp)



## การสร้างผู้ใช้



เมื่ออยู่ใน **Interface GNOME** คุณจะต้องสร้างผู้ใช้ใหม่เพื่อความปลอดภัยที่มากขึ้นและการใช้งานที่ปลอดภัย ปราศจากความเสี่ยง เข้าไปที่แอปพลิเคชันและเลือกตัวเลือก "console" เพื่อเปิดเทอร์มินัล





- เพิ่มผู้ใช้:



```shell
useradd -m -G wheel -s /bin/bash nom_utilisateur
passwd nom_utilisateur
```



![0_22](assets/fr/22.webp)





- ติดตั้ง **sudo**:


```shell
pacman -S sudo
```





- เปิดใช้งานสิทธิ์ **sudo**:



```shell
EDITOR=nano visudo
```





- จากนั้นยกเลิกการคอมเมนต์บรรทัด:



```shell
%wheel ALL=(ALL:ALL) ALL
```





- รีสตาร์ทระบบและเข้าสู่ระบบด้วยชื่อผู้ใช้ของคุณ



![0_23](assets/fr/23.webp)

## การติดตั้งซอฟต์แวร์



เนื่องจาก Arch Linux เป็นแบบมินิมอล ซอฟต์แวร์จำนวนมากจึงไม่ได้ติดตั้งโดยค่าเริ่มต้น หากต้องการเพิ่มสิ่งที่คุณต้องการ ให้พิมพ์คำสั่งต่อไปนี้:



```shell
pacman -S nom_du_paquet_a_installe
```



ตัวอย่างเช่น ในการติดตั้งโปรแกรมแก้ไขข้อความ **nano** คุณสามารถพิมพ์:



```shell
pacman -S nano
```



ในการติดตั้งเว็บเบราว์เซอร์ที่มีน้ำหนักเบา เช่น `firefox` ให้ใช้:



```shell
pacman -S firefox
```



สุดท้ายนี้ หากคุณต้องการเพิ่มเครื่องมือเครือข่ายที่จำเป็น เช่น `net-tools` คำสั่งจะเป็น:



```shell
pacman -S net-tools
```



อย่าลืมว่าคุณสามารถติดตั้งแพ็กเกจหลายตัวในคำสั่งเดียวได้โดยการแยกรายการ:



```shell
pacman -S vim firefox net-tools
```



Arch Linux โดดเด่นด้วยความเสถียรที่น่าทึ่ง ปรัชญามินิมอล และความแข็งแกร่ง ทำให้เป็นตัวเลือกที่เหมาะสมสำหรับสภาพแวดล้อมการพัฒนา ด้วยการให้เฉพาะสิ่งจำเป็นเท่านั้น มันจึงมอบพื้นฐานที่มีน้ำหนักเบาและมีประสิทธิภาพสูงที่ง่ายต่อการปรับแต่งให้ตรงกับความต้องการเฉพาะของคุณ แนวทางมินิมอลนี้ยังช่วยให้ควบคุมระบบได้มากขึ้น เสริมสร้างความปลอดภัยโดยการจำกัดพื้นผิวการโจมตี ด้วยชุมชนที่มีความเคลื่อนไหวและเอกสารที่ครอบคลุม Arch Linux สามารถช่วยคุณสร้างสภาพแวดล้อมที่ปลอดภัย ยืดหยุ่น และปรับให้เหมาะสมสำหรับการพัฒนาระดับมืออาชีพ
