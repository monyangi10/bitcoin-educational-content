Arch Linux はその堅牢性と安定性により、多くの分野で人気のあるオペレーティングシステムです。しかし、初心者が使いこなすのは難しいかもしれません。この問題を解決するために、**Manjaro** は作られました: Arch Linux のパワーを提供しつつ、直感的で学習しやすい Interface に基づいて、よりシンプルでアクセスしやすい体験を提供するためです。



## マンジャロ入門



Manjaroの最大の資産の1つは、**シンプルで効率的なアップデートシステム**です。手動で管理する必要はありません：Manjaroがあなたの代わりにアップデートを行います！アップデートが利用可能になると、通知領域のアイコン（場所はエディションによって異なります）が知らせてくれます。あとは指示に従うだけです。



Manjaro はアプリケーションの**豊富なカタログ**も提供します。Manjaro は Arch Linux をベースにしているので、プロプライエタリなアプリケーションを含む様々なソフトウェアが豊富な公式リポジトリに直接アクセスすることができます。Manjaro は追加のテストのためにいくつかの Arch Linux のアップデートを少し遅らせます; これは新しいリリースを少し遅らせる代償として安定性を向上させます。もしこの選択だけでは不十分なら、コミュニティによって管理されている巨大なライブラリである **AUR (*Arch User Repository*) ** にもアクセスできます。公式リポジトリに存在しないプログラムでも、AURにあれば利用できる可能性があります。



Manjaroのもう一つの利点は、WindowsやmacOSのようなシステムよりも**リソース消費が少ない**ことです。より少ないRAMとコンピューティングパワーを消費するため、古いコンピューターやパワーのないコンピューターに理想的な選択です。



その上、Manjaroは**完全に無料**です。WindowsやmacOSとは異なり、インストールやその機能を最大限に活用するためにお金を払う必要はありません。最後に、定期的で迅速なアップデートのおかげで**強化されたセキュリティ**を提供し、脆弱性への露出を制限します。また、活発なコミュニティにより、いかなる問題も迅速に修正され、効果的な解決策が提案される。



## マンジャロOSを発見する



Manjaroのインストールを始める前に、このディストリビューションにはいくつかのエディションがあることを知っておくことが重要です。それぞれのエディションは、ワークフローとシステムリソースの消費の両方に影響を与えるユニークなデスクトップ環境を提供します。Manjaroには3つの公式エディションがあります。



![0_01](assets/fr/01.webp)





- **KDE Plasma**エディションは最もカスタマイズ性に優れています。視覚的にエレガントで高性能なシステムをお探しなら、**KDE Plasma**が最適です。この安定したデスクトップ環境は、完全なコントロールとカスタマイズされたエクスペリエンスを求めるユーザーに最適です。





- リソースの限られたマシンには、**Xfce**エディションが理想的なソリューションです。そのInterfaceは軽量かつ直感的で、古いコンピュータでもスムーズな操作が保証されている。さらに、そのレイアウトはWindowsを彷彿とさせ、新規ユーザーにとってLinuxへの移行を容易にする。





- 最後に、**GNOME**エディションはまったく異なるアプローチを提供します。その合理的なデザインは、仮想ワークスペースを通した生産性と整理整頓に重点を置いている。このアクティビティ重視のワークフローは、すでにLinuxに慣れ親しんでいて、ミニマルで非常に効率的な環境を探しているユーザーには特に魅力的です。



### その他のマンジャロ・エディション



![0_02](assets/fr/02.webp)



公式エディションに加えて、Manjaroコミュニティは他のバージョンも提供しています。これらの代替エディションは、特定のニーズを満たすように設計されており、さまざまなデスクトップ環境を提供しています。



**Cinnamon**エディションは、使い慣れたInterfaceをお探しの方に最適です。伝統的なオフィス環境の慣習を残しつつ、使いやすく設計されています。



上級ユーザー向けには、**i3 Window Manager**や**Sway**といったエディションがある。より軽量で高速であるにもかかわらず、これらの環境を完全に設定し利用するには、コマンドラインを使いこなす必要がある。これらの環境は、システムを完全にコントロールしたい人や、何よりも効率を重視する人に最適である。



## 技術的要件



Manjaroが最適に動作するためには、お使いのコンピュータがいくつかの最低要件を満たしている必要があります。少なくとも：





- 64ビット（x86_64）**プロセッサ**
- 4 GB RAM推奨（最低2 GB）**（下記参照）**
- 30 GB のディスク容量（`/home` 専用パーティションを作成する場合はそれ以上）**。**



## マンジャロのインストール



ダウンロードするには、[Manjaro公式ウェブサイト](https://manjaro.org/)にアクセスし、あなたのニーズに最も適したエディションを選択してください。ファイルをダウンロードしたら、Manjaro ISOイメージでブータブルUSBキーを作成する必要があります。



そして[Rufus]ソフトウェアのウェブサイト(https://rufus.ie/fr/)に行き、ダウンロードしてください。プログラムを実行し、USBキーを差し込み、Manjaro ISOイメージを選択し、フラッシュを開始します。プロセスが終了するまで待ってからキーを取り外してください。その後、コンピュータを再起動できます。



![0_03](assets/fr/03.webp)



コンピュータに Manjaro をインストールするには、まずコンピュータの電源を完全に切ります。それから USB キーを差し込み、マシンの電源を入れ直し、**F2, F10, F12, Escape** または **Delete** (メーカーによる) を押してブートメニューや UEFI/BIOS ファームウェアにアクセスします。



次に、USBキーをブートデバイスとして選択し、OSのインストールプロセスを開始する。



### 起動画面



初めてUSBキーからManjaroを起動すると、直接インストールメニューに移動します。インストールを開始する前に、キーボードレイアウトやシステム言語を変更できます。



次に、**オープンソースドライバで起動する**オプションを選択して、Manjaroのインストールを開始します。これらのオープンソースドライバはほとんどのハードウェアと互換性があり、ほとんどのケースで十分です。例えばNVIDIAのグラフィックカードを持っていたり、より高いグラフィック性能が必要な場合は、プロプライエタリドライバを使用する**プロプライエタリドライバで起動**を選択できます。



![0_04](assets/fr/04.webp)



システムは**デフォルトのライブモード**で起動します。これにより、Manjaroの機能をテストし、永久にインストールする前にニーズに合うかどうかを確認できます。準備ができたら、**Install Manjaro Linux** オプションをクリックしてください。



インストールする言語を選択し、**Next**をクリックします。



![0_06](assets/fr/06.webp)



次に、お住まいの地域を選択し、正しいタイムゾーンを設定します。この段階で、言語と日付フォーマットを変更することもできます。



![0_07](assets/fr/07.webp)



キーボードレイアウトを選択します。入力されたキーが期待される文字に対応しているかどうかをチェックするためのテストフィールドが用意されています。



![0_08](assets/fr/08.webp)



### ディスク・パーティショニング


ディスクのパーティション分割には2つのオプションがあります。





- 最初の、そして最も簡単な方法は、ディスク全体を消去し、そこにManjaroをインストールすることです。
- もうひとつは、**手動によるパーティション分割**を可能にするものだ。



![0_09](assets/fr/09.webp)



クリーンインストールのためには、少なくとも3つのパーティションを作成することをお勧めします：





- 最初のパーティションは**516MB**（プライマリ）で、**ブート**用です。
- 2GB**（論理）パーティションは**スワップ**用です。
- 個人データ用の第3パーティション。



別のシステムを並行してインストールしたい場合は、この最後のパーティションを分割し、その一部だけを Manjaro に割り当てる必要があります（システムの正常な動作を保証するため、少なくとも **15 GB** 必要）。


### ユーザーアカウントの作成



必要な情報を入力して、システムのユーザーアカウントを作成する。最後に、管理者のパスワード(**root**)を設定する。この管理者は、システムの全権限を持ち、高度なコマンドを実行できる **スーパーユーザー** である。



![0_10](assets/fr/10.webp)



### 適切なオフィス・スイートを選ぶ



Manjaroでは、**FreeOffice**と**LibreOffice**を選択できます。





- **LibreOffice**は、より幅広いソフトウェアと高度な機能を備え、より完全です。
- 一方、**FreeOffice**はより軽量で、必要なものだけが含まれています： **TextMaker**、**PlanMaker**、そして**Presentations**です。



![0_11](assets/fr/11.webp)



### 設定概要



サマリー画面には、設定したすべてのパラメーターが表示されます。必要であれば、すでに行った他の設定に影響を与えることなく、戻って変更することができます。



その後、**インストール**をクリックし、確認して実際のインストールを開始します。



![0_12](assets/fr/12.webp)



![0_13](assets/fr/13.webp)



### インストール終了



プロセスの最後に、**Restart now**ボックスをチェックし、**Done**をクリックしてください。システムが再起動し、**Manjaroが使えるようになります**。



![0_13](assets/fr/14.webp)



## マンジャロの第一歩



Manjaroのインストールは最初の一歩に過ぎません。システムを最大限に活用するために、知っておくと便利なことがいくつかあります。



### システムの更新



Manjaroはアップデートを大幅に簡素化します。パッケージをアップデートするには ：



```shell
sudo pacman -Syu
```



このコマンドは、利用可能な最新バージョンをダウンロードしてインストールします。システムを**安全かつ安定**に保つため、定期的に実行することをお勧めします。



### 開発環境の設定



ManjaroでWebアプリケーションを開発するには、1つのコマンドで必須ツールをインストールします：



```shell
sudo pacman -S nodejs npm git yarn
```



このコマンドは、JavaScriptとTypeScriptプロジェクトの実行と管理のためにNode.jsとnpmを、バージョン管理のためにGitを、代替パッケージ・マネージャーとしてYarnをインストールする。



### Bitcoin Walletの取り付け



Manjaro上でビットコインを管理するには、軽量で安全なWallet .Netである**Electrum**をインストールすることができます：



```shell
sudo pacman -S electrum  # Install Electrum
```



Electrumは、複数のWallet管理やpassphrase保護などの高度な機能を提供しながら、**ビットコイン**を簡単に送受信することができます。Electrumの使い方については、Walletの作成方法、資金の確保方法、取引の実行方法を説明した専用チュートリアルをご覧ください。



https://planb.network/tutorials/wallet/desktop/electrum-efec9166-46b5-4937-8cee-6bc310975177

## Manjaroシステムのセキュリティ



Linuxのインストールにおいて、セキュリティは非常に重要な要素です。データの機密性と完全性を保護することが重要です。



### ファイアウォールの設定



ManjaroにはUFW (*Uncomplicated Firewall*)というネットワークフィルターのファイアウォールを管理するプログラムが含まれていますが、手動で起動する必要があります：



```bash
# Installation if not present
sudo pacman -S ufw

# Firewall activation
sudo systemctl enable ufw.enable

sudo ufw enable

# Allow SSH connections (optional)
sudo ufw allow ssh

# Check the status
sudo ufw status verbose
```



![verbose](assets/fr/15.webp)



### ユーザー権限の管理



1. **非特権ユーザーを作成する**。



```shell
sudo useradd -m username
sudo passwd username
```



2. **Sudoersファイルの設定**



```shell
sudo EDITOR=nano visudo
```
