## 1. บทนำ



Tailscale เป็น VPN รุ่นถัดไปที่สร้างเครือข่ายเมชที่เข้ารหัสระหว่างอุปกรณ์ของคุณ ช่วยให้คุณเชื่อมต่อเครื่องระยะไกลราวกับว่าอยู่ในเครือข่ายท้องถิ่นเดียวกัน โดยไม่ต้องมีการกำหนดค่าที่ซับซ้อนหรือการเปิดพอร์ต



สำหรับการโฮสต์ด้วยตนเอง Tailscale จะกำหนด IP ส่วนตัวแบบคงที่ให้กับแต่ละอุปกรณ์ในเครือข่ายเสมือน ซึ่งช่วยให้เข้าถึงบริการของคุณได้อย่างเสถียรแม้ว่า IP สาธารณะของคุณจะเปลี่ยนแปลงก็ตาม ซึ่งหมายความว่าคุณสามารถจัดการเซิร์ฟเวอร์ของคุณจากระยะไกลโดยไม่ต้องเปิดเผยบริการของคุณโดยตรงต่ออินเทอร์เน็ต



**แอปพลิเคชันหลัก:**




- จัดการเซิร์ฟเวอร์ส่วนตัวจากภายนอก
- จัดการโหนด Umbrel/Lightning ได้เร็วกว่าที่ Tor
- การเข้าถึง Raspberry Pi หรือ NAS อย่างปลอดภัย
- เชื่อมต่อกับบริการของคุณผ่าน SSH หรือ HTTP โดยไม่ต้องกำหนดค่าเครือข่ายที่ซับซ้อน



แนวทางที่เน้นความเรียบง่ายนี้ช่วยให้ผู้ที่โฮสต์เองสามารถเข้าถึงบริการของตนได้อย่างปลอดภัย หลีกเลี่ยงข้อผิดพลาดของ VPN แบบดั้งเดิม



## 2. วิธีการทำงานของ Tailscale



ซึ่งแตกต่างจาก VPN แบบดั้งเดิมที่ส่งข้อมูลทั้งหมดผ่านเซิร์ฟเวอร์กลาง Tailscale สร้างเครือข่ายเมชที่อุปกรณ์สามารถสื่อสารกันโดยตรง เซิร์ฟเวอร์กลางจะจัดการเฉพาะการตรวจสอบสิทธิ์และการแจกจ่ายคีย์ โดยไม่เห็นข้อมูลของผู้ใช้



![VPN traditionnel (hub-and-spoke)](assets/fr/01.webp)


*รูปที่ 1: VPN แบบดั้งเดิมที่มีสถาปัตยกรรมแบบฮับและสโปก ซึ่งทราฟฟิกทั้งหมดผ่านเซิร์ฟเวอร์กลาง*



จาก WireGuard อุปกรณ์แต่ละเครื่องจะสร้างคีย์การเข้ารหัสของตัวเอง เซิร์ฟเวอร์ประสานงานจะกระจายคีย์สาธารณะไปยังโหนด ซึ่งจากนั้นจะสร้างช่องทางการเข้ารหัสแบบ end-to-end โดยตรงระหว่างกันเอง เพื่อผ่านไฟร์วอลล์ Tailscale ใช้การเดินทางผ่าน NAT และในกรณีสุดท้ายใช้ DERP relays ที่ยังคงรักษาการเข้ารหัสไว้



![VPN maillé (mesh)](assets/fr/02.webp)


*รูปที่ 2: เครือข่ายตาข่าย Tailscale ที่อุปกรณ์สื่อสารกันโดยตรง*



การสื่อสารทั้งหมดถูกเข้ารหัสด้วย WireGuard. Tailscale จะเห็นเฉพาะเมตาดาต้า (การเชื่อมต่อ) แต่ไม่เคยเห็นเนื้อหาของการแลกเปลี่ยนข้อมูล เพื่อความเป็นอิสระที่มากขึ้น Headscale ช่วยให้เซิร์ฟเวอร์ประสานงานสามารถโฮสต์เองได้



**ความปลอดภัยและการรักษาความลับ:** ขอบคุณ WireGuard การสื่อสารทั้งหมดบน Tailscale ได้รับการเข้ารหัสแบบ end-to-end Tailscale ไม่สามารถอ่านข้อมูลการใช้งานของคุณได้ - มีเพียงอุปกรณ์ของคุณเท่านั้นที่มีคีย์ส่วนตัวที่จำเป็น บริการนี้จะเห็นเพียงข้อมูลเมตา: ที่อยู่ IP, ชื่ออุปกรณ์, เวลาประทับการเชื่อมต่อ และบันทึกการเชื่อมต่อแบบ peer-to-peer (โดยไม่มีเนื้อหา)



อย่างไรก็ตาม สถาปัตยกรรมนี้ขึ้นอยู่กับ Tailscale Inc. สำหรับการประสานงานเครือข่าย เพื่อกำจัดการพึ่งพานี้ Headscale เสนอทางเลือกโอเพ่นซอร์สที่ช่วยให้คุณสามารถโฮสต์เซิร์ฟเวอร์ควบคุมด้วยตัวเองในขณะที่ใช้ไคลเอนต์ Tailscale อย่างเป็นทางการ ซึ่งรับประกันอธิปไตยอย่างสมบูรณ์เหนือโครงสร้างพื้นฐานเครือข่ายของคุณ โดยมีค่าใช้จ่ายในการกำหนดค่าทางเทคนิคที่มากขึ้น



**สำหรับคำอธิบายโดยละเอียดเกี่ยวกับการทำงานภายในของ Tailscale รวมถึงการจัดการ control plane, NAT traversal และ DERP relays เราขอแนะนำบทความที่ยอดเยี่ยม [How Tailscale Works](https://tailscale.com/blog/how-tailscale-works) บนบล็อกทางการ บทความนี้อธิบายแนวคิดทางเทคนิคอย่างลึกซึ้งที่ทำให้ Tailscale ทรงพลังมาก**



## 3. การติดตั้ง Tailscale

### 3.2 การติดตั้งบนแพลตฟอร์มต่าง ๆ





- บน Windows และ macOS: เพียงดาวน์โหลดแอปพลิเคชันกราฟิกจากเว็บไซต์ทางการของ Tailscale และติดตั้ง (.msi ไฟล์บน Windows, .dmg ไฟล์บน Mac) เมื่อทำการติดตั้งแล้ว แอปพลิเคชันจะเปิดตัว Interface กราฟิกที่ให้คุณเชื่อมต่อ (ผ่านเบราว์เซอร์) ไปยังบัญชี Tailscale ของคุณเพื่อยืนยันตัวตนของเครื่อง



![Connexion d'un appareil macOS](assets/fr/08.webp)


*เชื่อมต่อ MacBook กับ tailnet*



![Connexion réussie](assets/fr/09.webp)


*ยืนยันว่าอุปกรณ์เชื่อมต่อกับเครือข่าย Tailscale*





- บน Linux (Debian, Ubuntu, ฯลฯ): คุณมีหลายตัวเลือก วิธีที่ง่ายที่สุดคือการเรียกใช้สคริปต์การติดตั้งอย่างเป็นทางการ: ตัวอย่างเช่น บน Debian/Ubuntu:



```bash
curl -fsSL https://tailscale.com/install.sh | sh
```



สคริปต์นี้จะเพิ่มที่เก็บข้อมูล Tailscale อย่างเป็นทางการและติดตั้งแพ็คเกจ คุณยังสามารถ [เพิ่มที่เก็บ APT ด้วยตนเอง](https://pkgs.tailscale.com) หรือใช้แพ็คเกจ Snap หรือ apt ปกติได้ เมื่อทำการติดตั้งแล้ว daemon `tailscaled` จะทำงานในพื้นหลัง จากนั้นคุณจะต้อง **ยืนยันตัวตนของโหนด** (ดู Interface CLI เทียบกับเว็บด้านล่าง) บนการกระจายอื่น ๆ (Fedora, Arch...) แพ็คเกจนี้ยังมีให้ผ่านที่เก็บข้อมูลมาตรฐานหรือสคริปต์การติดตั้งสากล สำหรับเซิร์ฟเวอร์ที่ไม่มีหัวใช้ CLI: ตัวอย่างเช่น `sudo tailscale up --auth-key <key>` หากใช้คีย์ยืนยันตัวตนที่สร้างไว้ล่วงหน้า หรือเพียงแค่ `tailscale up` สำหรับการเข้าสู่ระบบแบบโต้ตอบ (ซึ่งจะให้ URL เพื่อเยี่ยมชมเพื่อยืนยันตัวตนอุปกรณ์)





- บนระบบที่ใช้ ARM (Raspberry Pi, ฯลฯ): โดยทั่วไปเราจะใช้ Linux ดังนั้นวิธีการเดียวกันกับข้างต้น (สคริปต์หรือแพ็คเกจ) โปรดทราบว่า Tailscale รองรับสถาปัตยกรรม ARM32/ARM64 โดยไม่มีปัญหาใด ๆ ผู้ใช้หลายคนติดตั้ง Tailscale บน Raspberry Pi OS ผ่าน apt หรือบนดิสทริบิวชันที่มีน้ำหนักเบา (DietPi, ฯลฯ) เพื่อเข้าถึง Pi ของพวกเขาได้ทุกที่





- บน iOS และ Android: Tailscale มีแอปพลิเคชันมือถือ **อย่างเป็นทางการ** เพียงติดตั้ง *Tailscale* จาก [App Store](https://apps.apple.com/us/app/tailscale/id1470499037?ls=1) (iOS) หรือ [Play Store](https://play.google.com/store/apps/details?id=com.tailscale.ipn) (Android)



![Installation sur iPhone](assets/fr/11.webp)


*ขั้นตอนการติดตั้ง Tailscale บน iPhone: ยินดีต้อนรับ, ความเป็นส่วนตัว, การแจ้งเตือน, VPN*



![Connexion sur iPhone](assets/fr/12.webp)


*เชื่อมต่อกับ tailnet เลือกบัญชีและยืนยันบน iPhone*



เมื่อติดตั้งแอปแล้ว ในการเปิดใช้งานครั้งแรก แอปจะขอให้คุณยืนยันตัวตนผ่านผู้ให้บริการที่เลือกไว้ (Google, Apple ID, Microsoft, ฯลฯ ขึ้นอยู่กับสิ่งที่คุณใช้สำหรับ Tailscale) - เป็นขั้นตอนเดียวกันกับแพลตฟอร์มอื่น ๆ โดยปกติจะเป็นการเปลี่ยนเส้นทางไปยังหน้าเว็บ OAuth หลังจากนั้น แอปมือถือจะสร้าง VPN (บน iOS คุณจะต้องยอมรับการเพิ่มการกำหนดค่า VPN) แอปสามารถทำงานในพื้นหลังได้ ทำให้คุณเข้าถึง tailnet ของคุณจากที่ใดก็ได้ *โปรดทราบ:* บนมือถือ คุณสามารถมี **VPN ที่ใช้งานอยู่ได้เพียงหนึ่งตัวเท่านั้น** - ดังนั้นให้แน่ใจว่าคุณไม่ได้เชื่อมต่อ VPN อื่นในเวลาเดียวกัน มิฉะนั้น Tailscale จะไม่สามารถสร้างการเชื่อมต่อของตัวเองได้ บน Android คุณสามารถตั้งค่าโปรไฟล์การทำงานแยกต่างหากได้หากคุณต้องการแยกการใช้งานบางอย่าง (เช่น โปรไฟล์ที่มี Tailscale ใช้งานสำหรับแอปที่กำหนด)

*รายการอุปกรณ์ที่เชื่อมต่อพร้อมลักษณะและสถานะของพวกเขา*



**Interface Web vs Interface CLI:** Tailscale มีสองวิธีที่เสริมกันในการโต้ตอบกับเครือข่าย: **การจัดการเว็บ Interface** และ **ไคลเอนต์บรรทัดคำสั่ง (CLI)**





- Interface Web (Admin Console)**: เข้าถึงได้ที่ [https://login.tailscale.com](https://login.tailscale.com), คอนโซลเว็บนี้เป็นแดชบอร์ดหลักสำหรับเครือข่าย Tailscale ของคุณ มันจะแสดงรายการอุปกรณ์ทั้งหมด (*Machines*), สถานะออนไลน์/ออฟไลน์, ที่อยู่ IP ของ Tailscale และอื่นๆ ที่นี่คุณสามารถ **จัดการอุปกรณ์** (เปลี่ยนชื่อ, หมดอายุคีย์, อนุญาตเส้นทาง, ปิดใช้งานโหนด), **จัดการผู้ใช้** (ในบริบทขององค์กร), และกำหนดกฎความปลอดภัย (ACLs) นี่คือที่ที่คุณกำหนดค่าตัวเลือกทั่วโลก เช่น MagicDNS, แท็ก, หรือคีย์การยืนยันตัวตน (คีย์การยืนยันตัวตนก่อน generate สำหรับการเพิ่มอุปกรณ์อัตโนมัติ) Interface เว็บมีประโยชน์มากสำหรับการดูภาพรวมและการปรับเปลี่ยนที่จะถูกเผยแพร่ผ่านเซิร์ฟเวอร์ประสานงานไปยังโหนดทั้งหมด *ตัวอย่าง:* การเปิดใช้งาน **เส้นทางซับเน็ต** หรือ **โหนดออก** ทำได้ด้วยการคลิกเพียงครั้งเดียวในคอนโซล เมื่อโหนดที่เกี่ยวข้องได้ประกาศตัวเองเป็นเช่นนั้นแล้ว





- Interface command line (CLI):** คำสั่ง `tailscale` มีให้ใช้งานใน CLI บนอุปกรณ์ทุกเครื่องที่ติดตั้ง Tailscale แล้ว CLI นี้ช่วยให้คุณทำทุกอย่างได้ในเครื่อง: เชื่อมต่อ (`tailscale up`), ตรวจสอบสถานะ (`tailscale status` เพื่อดูว่าเพื่อนคนไหนเชื่อมต่ออยู่), แก้ไขข้อบกพร่อง (`tailscale ping <ip>`), และอื่นๆ บางฟีเจอร์ยัง **เฉพาะสำหรับ CLI** หรือมีความก้าวหน้ามากกว่า เช่น:





  - `tailscale up --advertise-routes=192.168.0.0/24` เพื่อโฆษณาเส้นทางซับเน็ต,
  - `tailscale up --advertise-exit-node` เพื่อเสนอเครื่องของคุณเป็น exit node,
  - `tailscale set --accept-routes=true` (or `--exit-node=<IP>`) to consume a route or use an exit node,
  - `tailscale ip -4` เพื่อแสดง IP Tailscale ของอุปกรณ์,
  - `tailscale lock/unlock` (ถ้าใช้ *tailnet-lock*, ฟีเจอร์ความปลอดภัยขั้นสูง),
  - หรือ `tailscale file send <node>` เพื่อใช้ **Taildrop** (การถ่ายโอนไฟล์ระหว่างอุปกรณ์)


CLI มีประโยชน์มากบนเซิร์ฟเวอร์ที่ไม่มีกราฟิก Interface และสำหรับการสคริปต์การกระทำบางอย่าง **ความแตกต่างในการใช้งาน:** การกำหนดค่าพื้นฐานส่วนใหญ่สามารถทำได้ทั้งผ่านเว็บหรือผ่าน CLI ตัวอย่างเช่น การเพิ่มอุปกรณ์สามารถทำได้โดยการแจ้งผ่านคอนโซล หรือโดยการรัน `tailscale up` บนอุปกรณ์และยืนยันผ่านเว็บ ในทำนองเดียวกัน การเปลี่ยนชื่ออุปกรณ์สามารถทำได้ผ่านคอนโซลหรือด้วย `tailscale set --hostname` **สรุป**, คอนโซลเว็บเหมาะสำหรับการจัดการเครือข่ายทั่วโลก (โดยเฉพาะกับเครื่อง/ผู้ใช้หลายคน) ในขณะที่ CLI เหมาะสำหรับการควบคุมอย่างละเอียดบนเครื่องที่กำหนด, สคริปต์อัตโนมัติ, หรือการใช้งานบนระบบที่ไม่มี GUI

เมื่อ Umbrel เชื่อมต่อกับ Tailscale **คุณสามารถเข้าถึง Interface Umbrel และแอปพลิเคชันที่ทำงานอยู่บนมันจากที่ใดก็ได้ ราวกับว่าคุณอยู่ในเครือข่ายท้องถิ่น** นี่เป็นหนึ่งในข้อได้เปรียบหลักเหนือ Tor.



การเข้าถึงนั้นง่ายมาก: แทนที่จะใช้ `umbrel.local` (ซึ่งใช้ได้เฉพาะในเครือข่ายท้องถิ่นของคุณ) คุณใช้ที่อยู่ IP ของ Tailscale ของ Umbrel (`http://100.x.y.z`) โดยตรงจากอุปกรณ์ใดๆ ที่เชื่อมต่อกับ tailnet ของคุณ วิธีนี้ใช้ได้ไม่ว่าคุณจะอยู่ที่ไหนหรือใช้อินเทอร์เน็ตการเชื่อมต่อแบบใด (4G, Wi-Fi สาธารณะ, เครือข่ายองค์กร)



**ตัวอย่างของแอปพลิเคชันที่โฮสต์โดย Umbrel ที่สามารถเข้าถึงได้ผ่าน Tailscale:**

- Bitcoin node**: จัดการโหนด Bitcoin ของคุณโดยไม่มีความล่าช้า ดูการซิงโครไนซ์และสถิติ

- noStrudel**: เข้าถึงบริการ Nostr ของคุณที่โฮสต์บน Umbrel

### 4.3 ข้อดีเหนือกว่า Tor



Umbrel มีการเข้าถึงระยะไกลผ่าน Tor โดยตรง (โดยให้ที่อยู่ `.onion` สำหรับบริการเว็บของมัน) ในขณะที่ Tor มีข้อดีในเรื่องของการรักษาความลับ (การไม่เปิดเผยตัวตน) และไม่ต้องการการลงทะเบียน ผู้ใช้หลายคนพบว่า **Tor ช้าและไม่เสถียร** สำหรับการใช้งานในชีวิตประจำวัน (หน้าเว็บโหลดช้า, การหมดเวลา, ฯลฯ) - *"Umbrel ผ่าน Tor ช้ามาก "* บางคนบ่น.



Tailscale ให้การเชื่อมต่อที่ **เร็วกว่าและมีความหน่วงต่ำ** โดยทั่วไป เนื่องจากการรับส่งข้อมูลผ่านโดยตรง (หรือผ่านรีเลย์ที่รวดเร็ว) แทนที่จะผ่าน 3 โหนดของ Tor นอกจากนี้ Tailscale ยังมอบประสบการณ์ "เครือข่ายท้องถิ่น": ใช้ IP ส่วนตัว และสามารถแมปแอปพลิเคชันได้โดยตรง (เช่น ไดรฟ์เครือข่าย SMB บน \100.x.y.z)



ที่กล่าวมานั้น Tor มีข้อได้เปรียบในการเป็นระบบกระจายศูนย์และ "พร้อมใช้งาน" บน Umbrel ในขณะที่ Tailscale ต้องอาศัยความไว้วางใจในบุคคลที่สาม (หรือโฮสต์ headscale) Tor ยังมีประโยชน์สำหรับการเข้าถึงแบบไม่ต้องใช้ไคลเอนต์ (จากเบราว์เซอร์ Tor ใด ๆ คุณสามารถเข้าถึง UI ของ Umbrel ได้ ในขณะที่ Tailscale ต้องการให้ติดตั้งไคลเอนต์บนอุปกรณ์ที่เข้าถึง)



**สรุป**, สำหรับการใช้งานแบบโต้ตอบ (กระเป๋าเงิน Lightning, อินเทอร์เฟซเว็บที่ใช้บ่อย) Tailscale มอบความสะดวกสบายและความเร็วที่น่าพอใจเมื่อเทียบกับ Tor โดยมีราคาของการพึ่งพาภายนอกเล็กน้อย หลายคนเลือกที่จะใช้ *ทั้งสองอย่าง*: Tailscale ในการใช้งานประจำวัน และ Tor เป็นทางเลือกสำรองหรือเพื่อแบ่งปันการเข้าถึงกับผู้อื่นโดยไม่ต้องเชิญพวกเขาเข้าสู่ VPN ของตน



### 4.4 ความปลอดภัย



โดยการใช้ Tailscale ร่วมกับ Umbrel คุณจะหลีกเลี่ยงการเปิดเผย Umbrel ต่ออินเทอร์เน็ต โหนด Umbrel สามารถเข้าถึงได้เฉพาะอุปกรณ์อื่นๆ ที่ได้รับการยืนยันใน tailnet ของคุณเท่านั้น ซึ่งช่วยลดพื้นผิวการโจมตีได้อย่างมาก (ไม่มีพอร์ตเปิดให้คนแปลกหน้า ไม่มีความเสี่ยงต่อการโจมตีบริการเว็บผ่านอินเทอร์เน็ต)



การสื่อสารถูกเข้ารหัส (WireGuard) นอกเหนือจากการเข้ารหัสที่บริการของคุณทำอยู่แล้ว (เช่น แม้แต่ HTTP ภายในก็อยู่ในอุโมงค์) คุณยังสามารถใช้ Tailscale ACLs ได้ เช่น เพื่อป้องกันอุปกรณ์ tailnet เฉพาะจากการเข้าถึง Umbrel หากคุณต้องการแบ่งเครือข่าย Umbrel เองไม่เห็นความแตกต่าง - มันคิดว่ามันกำลังให้บริการในท้องถิ่น

เพื่อสรุปในส่วนนี้ การรวม Tailscale บน Umbrel ใช้เพียงไม่กี่คลิกและ **ช่วยปรับปรุงการเข้าถึงได้อย่างมาก** ของโหนดที่โฮสต์ด้วยตนเอง คุณจะสามารถจัดการ Umbrel และบริการต่างๆ ได้จากทุกที่ อย่างปลอดภัยและมีประสิทธิภาพ ราวกับว่าคุณอยู่ที่บ้าน นี่เป็นโซลูชันที่มีประโยชน์อย่างยิ่งสำหรับแอปพลิเคชันเรียลไทม์ (Lightning) ที่ประสบปัญหาความล่าช้าของ Tor หรือโดยทั่วไปสำหรับผู้ที่โฮสต์ด้วยตนเองที่มองหาการเชื่อมต่อส่วนตัวที่ง่ายดาย ทั้งหมดนี้โดยไม่ต้องเปิดพอร์ตใดๆ บนกล่องของคุณ และไม่ต้องมีการกำหนดค่าเครือข่ายที่ซับซ้อน



## 5. การจัดการขั้นสูงและกรณีการใช้งาน



### 5.1 Tailscale คุณสมบัติขั้นสูง



**การจัดการเครือข่าย:** คอนโซลการจัดการ (login.tailscale.com) ช่วยให้คุณจัดการอุปกรณ์ของคุณ ดูการเชื่อมต่อ และกำหนดค่ากฎการเข้าถึง



**MagicDNS:** แก้ไขชื่ออุปกรณ์โดยอัตโนมัติ (เช่น `raspberrypi.tailnet.ts.net`) เพื่อหลีกเลี่ยงการจำที่อยู่ IP



**ACL และการควบคุมการเข้าถึง:** กำหนดอย่างแม่นยำว่าใครสามารถเข้าถึงอะไรในเครือข่ายของคุณผ่านกฎ JSON ซึ่งเหมาะสำหรับการแยกอุปกรณ์บางอย่างหรือจำกัดการเข้าถึงบริการเฉพาะ



**การแชร์อุปกรณ์** ช่วยให้คุณเชิญบุคคลเข้าถึงเครื่องเฉพาะโดยไม่ต้องให้พวกเขาเข้าถึงเครือข่ายทั้งหมดของคุณ



**Subnet Router:** เครื่อง Tailscale สามารถทำหน้าที่เป็นเกตเวย์สำหรับซับเน็ตทั้งหมด ทำให้สามารถเข้าถึงอุปกรณ์ที่ไม่ใช่ Tailscale (IoT, เครื่องพิมพ์ ฯลฯ) ผ่านเครื่องที่กำหนดค่าเพียงเครื่องเดียวได้



**Exit Node:** ใช้เครื่องเป็นเกตเวย์อินเทอร์เน็ตเพื่อออกด้วย IP ของมัน มีประโยชน์สำหรับ Wi-Fi สาธารณะหรือเพื่อข้ามข้อจำกัดทางภูมิศาสตร์



**Taildrop:** ทางเลือกที่ปลอดภัยสำหรับ AirDrop ช่วยให้คุณสามารถถ่ายโอนไฟล์ระหว่างอุปกรณ์ Tailscale ของคุณได้ ไม่ว่าจะเป็นแพลตฟอร์มหรือสถานที่ใดก็ตาม ต่างจาก AirDrop ที่จำกัดอยู่ในระบบนิเวศของ Apple และต้องอยู่ใกล้กัน Taildrop ทำงานระหว่างอุปกรณ์ทั้งหมดของคุณ (Windows, Mac, Linux, Android, iOS) แม้ว่าจะอยู่ในประเทศต่างๆ ไฟล์จะถูกถ่ายโอนโดยตรงระหว่างอุปกรณ์ด้วยการเข้ารหัสแบบ end-to-end โดยไม่ผ่านเซิร์ฟเวอร์กลาง ใช้บรรทัดคำสั่ง `tailscale file cp` หรือแอปพลิเคชันกราฟิก Interface ขึ้นอยู่กับระบบของคุณ



### 5.2 การเปรียบเทียบกับโซลูชันอื่น ๆ



**เทียบกับ OpenVPN:** Tailscale นั้นง่ายต่อการตั้งค่ามากกว่า (ไม่ต้องเปิดพอร์ต ไม่ต้องจัดการใบรับรอง) แต่ต้องพึ่งพาบริการจากบุคคลที่สาม OpenVPN สามารถควบคุมได้อย่างเต็มที่ แต่ต้องการความเชี่ยวชาญมากกว่า



**ในฐานะคู่แข่งโดยตรง ZeroTier ทำงานที่ Layer 2 (Ethernet) ซึ่งรองรับการส่งข้อมูลแบบ broadcast/multicast ในขณะที่ Tailscale ทำงานที่ Layer 3 (IP) ZeroTier มอบความยืดหยุ่นของเครือข่ายที่มากกว่า ในขณะที่ Tailscale เน้นความง่ายในการใช้งาน**



**เทียบกับ Tor:** Tor มอบความเป็นนิรนามที่ Tailscale ไม่มี แต่ช้ากว่ามาก Tor เป็นแบบกระจายศูนย์และไม่ต้องการบัญชี ในขณะที่ Tailscale เร็วกว่าและมอบประสบการณ์คล้าย LAN



**คู่มือ Vs WireGuard:** Tailscale ทำการจัดการคีย์และการเชื่อมต่อทั้งหมดโดยอัตโนมัติ ซึ่ง WireGuard แบบดิบต้องให้คุณจัดการด้วยตนเอง มันคือ WireGuard + ชั้นการจัดการที่ง่ายขึ้น



โดยสรุปแล้ว Tailscale วางตำแหน่งตัวเองเป็นโซลูชันที่ทันสมัยและเน้นความเรียบง่าย เหมาะสำหรับการใช้งานส่วนบุคคลและทีมขนาดเล็ก สำหรับผู้ที่ชื่นชอบการควบคุมอย่างสมบูรณ์ Headscale เสนอทางเลือกในการโฮสต์เอง



## 6. บทสรุป



**Tailscale benefits:** Tailscale มีข้อดีหลายประการสำหรับการโฮสต์ด้วยตนเอง:





- ความเรียบง่ายและประสิทธิภาพ** - การติดตั้งที่รวดเร็วบนทุกแพลตฟอร์มโดยไม่ต้องมีการกำหนดค่าเครือข่ายที่ซับซ้อน การจราจรจะตามเส้นทางที่ตรงที่สุดระหว่างเครื่องของคุณ (P2P mesh) ด้วยประสิทธิภาพของโปรโตคอล WireGuard และไม่มีเซิร์ฟเวอร์กลางที่จำกัดอัตราการส่งข้อมูล
- ความปลอดภัยและความยืดหยุ่น** - การเข้ารหัสแบบ end-to-end, ลดพื้นผิวการโจมตี, และคุณสมบัติขั้นสูง (การรับรองความถูกต้อง ACL, SSO/MFA) ทำงานได้แม้หลัง NATs หรือขณะเคลื่อนที่, ด้วยเราเตอร์ซับเน็ตและโหนดทางออกเพื่อปรับเครือข่ายให้ตรงกับความต้องการของคุณ



**ขีดจำกัด:** โปรดจำไว้ว่า:





- การพึ่งพาภายนอก** - ในเวอร์ชันมาตรฐาน บริการนี้พึ่งพาโครงสร้างพื้นฐานของ Tailscale Inc. การพึ่งพานี้สามารถหลีกเลี่ยงได้ผ่านทาง Headscale (ทางเลือกสำหรับการโฮสต์ด้วยตนเอง)
- ข้อจำกัดอื่นๆ** - โค้ดบางส่วนเป็นแบบปิด, ข้อจำกัดของเวอร์ชันฟรีสำหรับการใช้งานขั้นสูงบางอย่าง, ไม่มีการสนับสนุนสำหรับ Layer 2 (broadcast/multicast), และต้องการการเชื่อมต่ออินเทอร์เน็ตเพื่อสร้างการเชื่อมต่อ



**Tailscale เหมาะสำหรับผู้ที่โฮสต์เองและทีมขนาดเล็ก นักพัฒนาที่ต้องการเข้าถึงทรัพยากรกระจายตัว ผู้เริ่มต้นใช้ VPN และผู้ใช้มือถือ สำหรับบริษัทที่ต้องการการควบคุมอย่างสมบูรณ์ โซลูชันอื่นๆ เช่น Headscale หรือ WireGuard อาจเป็นที่ต้องการมากกว่า**



**สำรวจ Headscale สำหรับการโฮสต์ด้วยตนเองเต็มรูปแบบ, API และการผสานรวม DevOps (Terraform), หรือทางเลือกอื่นๆ เช่น Innernet (คล้ายกันแต่โฮสต์ด้วยตนเองเต็มรูปแบบ) และ Netmaker.**



Tailscale เป็นเครื่องมือที่จำเป็นสำหรับการโฮสต์ด้วยตนเอง ด้วยความเรียบง่ายและประสิทธิภาพ ทำให้โครงสร้างพื้นฐานส่วนตัวของคุณเข้าถึงได้ง่ายราวกับว่าอยู่ในคลาวด์ ในขณะที่ยังคงควบคุมได้ที่บ้าน
