## 1. Введение



Tailscale - это VPN нового поколения, создающий зашифрованную ячеистую сеть между вашими устройствами. Он позволяет подключать удаленные машины так, как будто они находятся в одной локальной сети, без сложной настройки или открытия портов.



При самостоятельном хостинге Tailscale присваивает каждому устройству фиксированный частный IP-адрес в виртуальной сети, обеспечивая стабильный доступ к вашим услугам даже при изменении публичного IP-адреса. Это означает, что вы можете управлять своими серверами удаленно, не открывая доступ к своим услугам напрямую через Интернет.



**Основные приложения:**




- Управление персональным сервером извне
- Управление узлами Umbrel/Lightning быстрее, чем Tor
- Безопасный доступ к Raspberry Pi или NAS
- Подключение к вашим сервисам через SSH или HTTP без сложной настройки сети



Такой подход, ориентированный на простоту, позволяет самостоятельным хостерам получать безопасный доступ к своим услугам, избегая "подводных камней" традиционных VPN.



## 2. Как работает Tailscale



В отличие от традиционных VPN, которые направляют весь трафик через центральный сервер, Tailscale создает ячеистую сеть, в которой устройства общаются друг с другом напрямую. Центральный сервер занимается только аутентификацией и распределением ключей, не видя данных пользователя.



![VPN traditionnel (hub-and-spoke)](assets/fr/01.webp)


*Рисунок 1. Традиционная VPN с архитектурой "хаб и спица", где весь трафик проходит через центральный сервер*



На основе WireGuard каждое устройство генерирует собственные криптографические ключи. Координационный сервер распределяет открытые ключи между узлами, которые затем устанавливают сквозные зашифрованные туннели непосредственно между собой. Чтобы преодолеть брандмауэры, Tailscale использует обход NAT и, в крайнем случае, DERP-реле, которые поддерживают шифрование.



![VPN maillé (mesh)](assets/fr/02.webp)


*Рисунок 2. Масштабная ячеистая сеть, в которой устройства напрямую общаются друг с другом*



Все соединения шифруются с помощью WireGuard. Tailscale видит только метаданные (соединения), но никогда не видит содержимого обменов. Для большей суверенности Headscale позволяет самостоятельно размещать координационный сервер.



**Безопасность и конфиденциальность:** Благодаря WireGuard все коммуникации в Tailscale шифруются из конца в конец. Tailscale не может прочитать ваш трафик - только ваши устройства имеют необходимые закрытые ключи. Сервис видит только метаданные: IP-адреса, имена устройств, временные метки соединений и журналы одноранговых соединений (без содержимого).



Однако эта архитектура зависит от Tailscale Inc. в плане координации сети. Чтобы устранить эту зависимость, Headscale предлагает альтернативу с открытым исходным кодом, которая позволяет вам самостоятельно размещать сервер управления, используя официальные клиенты Tailscale, гарантируя тем самым полный контроль над вашей сетевой инфраструктурой, но за счет более сложной технической конфигурации.



**Для подробного объяснения внутренней работы Tailscale, включая управление плоскостью управления, обход NAT и ретрансляцию DERP, мы рекомендуем отличную статью** [How Tailscale Works](https://tailscale.com/blog/how-tailscale-works) **в официальном блоге. В этой статье подробно объясняются технические концепции, которые делают Tailscale таким мощным.**

### 3.4 Панель администрирования



Веб-консоль администрирования позволяет просматривать и управлять всеми подключенными устройствами:



![Tableau de bord des machines](assets/fr/15.webp)


*Список подключенных устройств с указанием их характеристик и статуса*



**Interface Web vs Interface CLI:** Tailscale предлагает два взаимодополняющих способа взаимодействия с сетью: **Interface web-администрирование** и **клиент командной строки (CLI)**.





- Interface Web (Admin Console)**: доступная по адресу [https://login.tailscale.com](https://login.tailscale.com), эта веб-консоль является центральной панелью управления вашей сетью Tailscale. Здесь перечислены все устройства (*машины*), их статус онлайн/оффлайн, их IP-адреса Tailscale и многое другое. Здесь вы можете **управлять устройствами** (переименовывать, истекать срок действия ключей, авторизировать маршруты, отключать узел), **управлять пользователями** (в организационном контексте) и определять правила безопасности (ACL). Здесь также настраиваются глобальные параметры, такие как MagicDNS, теги или ключи авторизации (ключи авторизации до generate для автоматического добавления устройств). Interface web очень удобен для получения общего представления и применения изменений, которые будут распространяться через координационный сервер на все узлы. *Пример:* Активация **маршрута подсети** или **выходного узла** выполняется одним щелчком мыши в консоли, как только узел объявит себя таковым.





- **Командная строка Interface (CLI):** Команда `tailscale` доступна в CLI на каждом устройстве, где установлена Tailscale. С помощью этой команды CLI можно делать все локально: подключаться (`tailscale up`), проверять статус (`tailscale status`, чтобы увидеть, какие пиры подключены), отлаживать (`tailscale ping <ip>`) и так далее. Некоторые функции даже **эксклюзивны для CLI** или более продвинуты, например:





  - `tailscale up --advertise-routes=192.168.0.0/24` для рекламы маршрута подсети,
  - `tailscale up --advertise-exit-node`, чтобы предложить вашу машину в качестве узла выхода,
  - `tailscale set --accept-routes=true` (или `--exit-node=<IP>`), чтобы использовать маршрут или узел выхода,
  - `tailscale ip -4` для отображения Tailscale IP устройства,
  - `Блокировка/разблокировка хвостовой шкалы` (при использовании *tailnet-lock*, расширенной функции безопасности),
  - или `tailscale file send <node>` для использования **Taildrop** (передача файлов между устройствами).


CLI очень полезен на серверах, где нет графики Interface, и для скриптинга определенных действий. **Различия в использовании:** Большинство базовых конфигураций может быть выполнено либо через Web, либо через CLI. Например, добавление устройства осуществляется либо запросом через консоль, либо запуском `tailscale up` на устройстве и проверкой через веб. Аналогично, переименование устройства может быть выполнено через консоль или с помощью `tailscale set --hostname`. **В целом**, веб-консоль идеально подходит для глобального администрирования сети (особенно при работе с несколькими машинами/пользователями), а CLI удобна для тонкого контроля над конкретной машиной, сценариев автоматизации или использования в системе без графического интерфейса.

### 4.3 Преимущества перед Tor



Umbrel предлагает удаленный доступ через Tor (предоставляя `.onion` адреса для своих веб-сервисов). Хотя Tor обладает преимуществом конфиденциальности (анонимности) и не требует регистрации, многие пользователи находят **Tor медленным и нестабильным** для повседневного использования (страницы загружаются медленно, таймауты и т.д.) - *"Umbrel через Tor такой медленный "* некоторые жалуются.



Tailscale обеспечивает в целом **быстрое соединение с низкой задержкой**, поскольку трафик проходит напрямую (или через быстрый ретранслятор), а не скачет через 3 узла Tor. Более того, Tailscale обеспечивает работу в "локальной сети": используются частные IP-адреса, и приложения могут быть отображены напрямую (например, сетевой диск SMB на \100.x.y.z).



При этом преимущество Tor заключается в том, что он децентрализован и работает "из коробки" на Umbrel, тогда как Tailscale требует доверия к третьей стороне (или хостингу headscale). Tor также полезен для бесклиентского доступа (из любого браузера Tor можно обратиться к Umbrel UI, в то время как Tailscale требует установки клиента на устройство доступа).



**Подводя итог**, можно сказать, что для интерактивного использования (кошельки Lightning, частые веб-интерфейсы) Tailscale предлагает ощутимое удобство и скорость по сравнению с Tor, ценой небольшой внешней зависимости. Многие люди предпочитают использовать и то, и другое: Tailscale на повседневной основе, а Tor - в качестве запасного варианта или для того, чтобы поделиться доступом с кем-то, не приглашая его в свою VPN.



### 4.4 Безопасность



Используя Tailscale вместе с Umbrel, вы не подвергаете Umbrel воздействию Интернета. Узел Umbrel доступен только для других аутентифицированных устройств в сети tailnet, что значительно сокращает площадь атаки (нет портов, открытых для посторонних, нет риска атаки на веб-сервис через Интернет).



Коммуникации шифруются (WireGuard) в дополнение к любому шифрованию, которое уже выполняют ваши службы (например, даже внутренний HTTP находится в туннеле). Вы все еще можете применять Tailscale ACL, чтобы, например, запретить определенному устройству tailnet доступ к Umbrel, если вы хотите разделить сеть. Сам Umbrel не видит разницы - он думает, что обслуживает локальную сеть.



---

В заключение этого раздела отметим, что интеграция Tailscale в Umbrel занимает всего несколько кликов и **значительно повышает доступность** вашего узла, размещенного самостоятельно. Вы сможете управлять Umbrel и его сервисами из любого места, безопасно и эффективно, как будто вы находитесь дома. Это особенно полезное решение для приложений реального времени (Lightning), которые страдают от задержек Tor, или вообще для любого хостера, которому нужно простое частное соединение. И все это **без открытия ни одного порта** на вашем компьютере и без сложной настройки сети.



## 5. Расширенное управление и варианты использования



### 5.1 Дополнительные возможности Tailscale



**Управление сетью:** Консоль администрирования (login.tailscale.com) позволяет управлять устройствами, просматривать подключения и настраивать правила доступа.



**MagicDNS:** Автоматически разрешает имена устройств (например, `raspberrypi.tailnet.ts.net`), чтобы не запоминать IP-адреса.



**ACL и контроль доступа:** Точно определяйте, кто и к чему может получить доступ в вашей сети, с помощью правил JSON, идеально подходящих для изоляции определенных устройств или ограничения доступа к конкретным службам.



**Общий доступ к устройствам позволяет пригласить кого-либо получить доступ к определенной машине, не предоставляя ему доступ ко всей сети.**



**Роутер подсети:** Машина Tailscale может выступать в качестве шлюза для целой подсети, обеспечивая доступ к устройствам, не относящимся к Tailscale (IoT, принтерам и т. д.), через одну настроенную машину.



**Узел выхода:** Используйте машину в качестве интернет-шлюза для выхода с ее IP. Полезно для публичного Wi-Fi или для обхода географических ограничений.



**Taildrop:** Безопасная альтернатива AirDrop, позволяющая передавать файлы между устройствами Tailscale, независимо от их платформы и местоположения. В отличие от AirDrop, который ограничен экосистемой Apple и физической близостью, Taildrop работает между всеми вашими устройствами (Windows, Mac, Linux, Android, iOS), даже если они находятся в разных странах. Файлы передаются напрямую между устройствами со сквозным шифрованием, не проходя через центральный сервер. Используйте командную строку `tailscale file cp` или графическое приложение Interface в зависимости от вашей системы.



### 5.2 Сравнение с другими решениями



**Vs OpenVPN:** Tailscale гораздо проще в настройке (не нужно открывать порты, управлять сертификатами), но предполагает зависимость от сторонней службы. OpenVPN полностью управляем, но требует большего опыта.



**В качестве прямого конкурента ZeroTier работает на Layer 2 (Ethernet), обеспечивая широковещательную/многоадресную рассылку, в то время как Tailscale работает на Layer 3 (IP). ZeroTier предлагает большую гибкость сети, в то время как Tailscale предпочитает простоту использования.**



**Vs Tor:** Tor обеспечивает анонимность, которой нет у Tailscale, но работает гораздо медленнее. Tor децентрализован и не требует учетной записи, в то время как Tailscale работает быстрее и предлагает работу в локальной сети.



**В сравнении с WireGuard manual:** Tailscale автоматизирует все управление ключами и соединениями, которые WireGuard raw требует выполнять вручную. По сути, это WireGuard + упрощенное управление Layer.



В заключение можно сказать, что Tailscale позиционирует себя как современное, ориентированное на простоту решение, идеально подходящее для личного использования и небольших команд. Для приверженцев полного контроля Headscale предлагает альтернативу в виде самостоятельного хостинга.



## 6. Заключение



**Преимущества Tailscale:** Tailscale предлагает несколько преимуществ для самостоятельного хостинга:





- **Простота и производительность** - Быстрая установка на всех платформах без сложной настройки сети. Трафик идет по самому прямому пути между вашими машинами (сетка P2P), с производительностью протокола WireGuard и без центрального сервера, ограничивающего пропускную способность.
- **Безопасность и гибкость** - сквозное шифрование, уменьшенная площадь атаки и расширенные функции (ACL, аутентификация SSO/MFA). Работает даже за NAT или в движении, с маршрутизаторами подсети и выходными узлами для адаптации сети к вашим потребностям.



**Ограничения:** Также имейте в виду:





- **Внешняя зависимость** - В своей стандартной версии сервис полагается на инфраструктуру Tailscale Inc. Эту зависимость можно обойти с помощью Headscale (альтернатива самостоятельному хостингу).
- **Другие ограничения** - частично закрытый исходный код, ограничения бесплатной версии для некоторых расширенных применений, отсутствие поддержки Layer 2 (широковещательная/мультикастовая рассылка), необходимость доступа в Интернет для установления соединения.



**Tailscale идеально подходит для индивидуальных хостеров и небольших команд, разработчиков, нуждающихся в доступе к разрозненным ресурсам, начинающих пользователей VPN и мобильных пользователей. Для компаний, требующих полного контроля, могут быть предпочтительны другие решения, такие как Headscale или WireGuard напрямую.**



**Изучите Headscale для полного самостоятельного хостинга, API и DevOps-интеграции (Terraform), или альтернативные варианты, такие как Innernet (аналогичный, но полностью самостоятельный хостинг) и Netmaker.**



Tailscale - это незаменимый инструмент для самостоятельного хостинга, благодаря своей простоте и эффективности делающий вашу частную инфраструктуру такой же доступной, как если бы она находилась в облаке, при этом управление остается дома.
