SimpleX 於 2021 年推出，是一款免費的即時通訊應用程式，以完全不同的方式處理隱私。與 WhatsApp、Signal 及其他集中式訊息服務不同，SimpleX 的突出之處在於其使用者管理：沒有使用者識別碼、假名、號碼或可見的公開金鑰。由於完全沒有識別碼，因此幾乎不可能對使用者進行關聯，保證了高度的隱私性。



與大多數需要帳號或電話號碼的應用程式不同，SimpleX 可讓您透過分享連結或短暫 QR 代碼來啟動對話。每個連結都會建立獨一無二的加密頻道，聯絡人若沒有明確的 Exchange 就無法找到或再次聯絡寄件者。訊息從頭到尾都經過加密，並會經過中繼伺服器，這些伺服器會在發送訊息後刪除訊息，而且不會看到寄件者或收件者，也不會看到他們的金鑰。



![Image](assets/fr/01.webp)



網路架構是完全分散且無封鎖的：伺服器之間互不認識，也不保留全球目錄，也不寄存任何使用者檔案。更棒的是，每個使用者都可以部署和使用自己的中繼伺服器，同時與公共網路中的伺服器保持互通。



SimpleX 完全開放原始碼 (用戶端、協定和伺服器)，適用於 Android、iOS、Linux、Windows 和 macOS。它的本機儲存是加密且可攜式的，因此您可以將設定檔從一台裝置傳輸到另一台裝置，而無需依賴集中式伺服器。



SimpleX 整合了訊息應用程式的所有經典功能。不過，它的人體工學設計仍不如 WhatsApp 或 Signal 流暢。它的使用限制也較多，尤其是在新增連絡人時。因此，在我看來，它是 WhatsApp 或 Signal 的替代品，適合將保密性放在首位的使用者，以及願意為此在日常使用舒適度上做出少許讓步的使用者。

## 安裝 SimpleX Chat 應用程式



SimpleX Chat 適用於所有平台。您可以直接從手機的應用程式商店下載應用程式：




- [Google Play](https://play.google.com/store/apps/details?id=chat.simplex.app)；
- [App Store](https://apps.apple.com/us/app/simplex-chat-secure-messenger/id1605771084)；
- [F-Droid](https://simplex.chat/fdroid/).



在 Android 上，也可以 [透過 APK 安裝](https://github.com/simplex-chat/simplex-chat/releases)。



在本教程中，我們將專注於行動版本，但請注意 [桌上型電腦版本也可用](https://simplex.chat/downloads/) (MacOS, Linux and Windows)。可以將桌面版本連結至現有的行動使用者設定檔，但為了讓此同步化生效，兩台裝置必須連線至相同的區域網路。



![Image](assets/fr/02.webp)



## 在 SimpleX Chat 上建立帳號



首次啟動應用程式時，按一下「*建立您的個人資料*」按鈕。



![Image](assets/fr/03.webp)



選擇使用者名稱，可以是您的真名或假名，然後按一下「*建立*」。



![Image](assets/fr/04.webp)



接下來，設定應用程式檢查新訊息的頻率。如果您的手機電池壽命沒有問題，請選擇「**即時**」，即可即時接收訊息。如果您希望節省電池，並防止應用程式在背景執行，請選擇「**當應用程式執行時**」： 這樣您只會在應用程式開啟時收到訊息。一個可能的折衷辦法是選擇每 10 分鐘定期檢查一次。



做出選擇後，按一下「*使用聊天*」。



![Image](assets/fr/05.webp)



現在您已連線到 SimpleX Chat，可以開始聊天了。



![Image](assets/fr/06.webp)



## 設定 SimpleX 聊天



首先，按一下右下角的個人檔案照片，然後按一下「*設定*」，即可存取設定。



![Image](assets/fr/07.webp)



預設設定一般適合大多數使用者。不過，我建議您前往「*資料庫 passphrase & 匯出*」功能表。在這裡您可以匯出您的 SimpleX 帳戶資料庫，以便備份。



您也可以修改用於加密此資料庫的 passphrase。預設情況下，它是隨機產生並儲存在您裝置的本機中。如果您願意，可以定義自己的 passphrase，並刪除本機 passphrase 備份：如此一來，每次開啟應用程式以及遷移到其他裝置時，都必須手動輸入 passphrase。



**請注意**：如果您選擇此選項，passphrase 的遺失會導致您所有的 SimpleX 設定檔和訊息永久遺失。



![Image](assets/fr/08.webp)



我還建議您進入「*隱私與安全*」功能表，在這裡您可以啟動「*SimpleX 鎖*」選項。這樣就可以使用鎖碼保護對應用程式的存取。



![Image](assets/fr/09.webp)



最後，「*通知*」和「*外觀*」功能表可讓您自訂 SimpleX Chat，以符合您的喜好。



![Image](assets/fr/10.webp)



## 使用 SimpleX Chat 傳送訊息



若要在 SimpleX 上與他人連線，您有兩個選項：




- 使用單次連結；
- 使用可重複使用的靜態 Address。



靜態 Address 允許任何知道它的人在 SimpleX 上與您聯絡。這是一個持久性的 Address，不同的人可以多次使用，沒有時間限制。正是這種持久性使它更容易受到垃圾郵件的攻擊。不過，與其他訊息應用程式不同的是，刪除您的 SimpleX Address 就足以阻止所有垃圾郵件，而不會影響現有的對話。事實上，這個 Address 只用於建立初始連線，一旦 Exchange 開始後就不再需要了。



另一方面，單次使用連結只能由任何使用者使用一次。一旦連絡人使用，連結就會失效。您需要為每個新連線 generate 一個新的連結。



### 使用靜態 Address



如果您希望使用 Address，請按一下 Interface 左下方的個人資料圖片，然後選擇「*建立 SimpleX Address*」。然後再按一下「*建立 SimpleX Address*」。



![Image](assets/fr/11.webp)



您可重複使用的 Address 已經建立。您可以與想要與您聯繫的人分享，您可以向他們展示 QR 代碼，或是傳送連結給他們。



![Image](assets/fr/12.webp)



按一下「*Address 設定*」按鈕。您可以在此設定與 Address 相關的權限。與連絡人分享*"選項可讓您的 Address 在您的 SimpleX 個人資料上可見。這樣您的聯絡人就可以參考它，並將它轉寄給希望與您聯絡的其他人。



自動接受*"選項會自動接受透過您的 Address 傳入的連線。除非您啟動 「*隱身接受*」選項，否則任何擁有您的 Address 的人都可以看到您的個人資料並發送訊息給您。這會在建立新連線時隱藏您的姓名和個人檔案照片，並以隨機假名取代，每個對話者的假名都不同。



![Image](assets/fr/13.webp)



### 附有可重複使用的連結



第二種連線方式是建立一次性連結。要這樣做，請按一下 Interface 右下角的鉛筆圖示，然後選擇「*建立一次性連結*」。



如果您的聯絡人已傳送連結給您，請按一下「*掃描/貼上連結*」來掃描或貼上。



![Image](assets/fr/14.webp)



然後，SimpleX 會產生一個單次使用的連結。您可以透過任何方式將它轉寄給您的連絡人：實體 Exchange、其他訊息等。



![Image](assets/fr/15.webp)



您也可以選擇將哪個個人資料與此邀請連結相關聯。要這樣做，請按一下 QR 代碼下方的您的個人資料。接下來您就可以 ：




- 選擇您現有的設定檔之一（我們將在下一節了解如何建立多個設定檔）；
- 或選擇「*隱身*」模式，該模式會隱藏您的姓名和個人檔案照片，並隨機為您的通信者生成假名。



在這裡，我選擇「*隱身*」模式。



![Image](assets/fr/16.webp)



我的聯絡人使用了這個連結。就他而言，他沒有啟動 「*隱身*」模式，所以我看到他的個人資料名稱 "*Bob*"。另一方面，Bob 看到的不是我的真名 "*Loïc Morel*「，而是一個隨機的假名，在這種情況下是 」*RealSynergy*"。



![Image](assets/fr/17.webp)



我可以馬上開始聊天，但首先我要確認我是在和 Bob 聊天，而不是和某些可能截取連結或進行 MITM 攻擊的惡意使用者聊天。



要做到這一點，我們要在應用程式**外**檢查我們的安全連結。這一點很重要，因為一旦發生 MITM 攻擊，內部驗證將會被洩露。因此，請使用其他安全的訊息系統，或者親自檢查密碼會更好。



在聊天中，按一下 Bob 的照片，然後按一下「*驗證安全碼*」。鮑勃這邊也要這麼做。



![Image](assets/fr/18.webp)



如果您是遠端工作，請比較另一個安全訊息系統上的代碼 (它們必須完全相同)。或者更好的方法是，如果您可以面對面，按一下「*掃描碼*」來掃描聯絡人的 QR 代碼。



![Image](assets/fr/19.webp)



如果驗證成功，您的連絡人姓名旁會出現一個打勾的盾牌圖示。這是您與 Bob 進行交換的保證。如果驗證不成功，則會出現「*安全代碼不正確！*」的警示。



![Image](assets/fr/20.webp)



現在您可以與 Bob 自由地 Exchange 訊息、通話和檔案，這取決於您為此對話設定的權限。



## 自訂您的 SimpleX 聊天設定檔



SimpleX 最強大的功能之一，就是可以管理數個完全獨立的使用者設定檔，所有這些設定檔都可以從相同的本機帳號存取。這可讓您整齊地區分不同的身分，而不會使訊息管理變得複雜。



例如，您可以建立 .NET Framework 2.0：




- 有真實姓名和真實照片的個人資料，以便進行專業交流；
- 一個有您真實姓名的個人檔案，以及一張有趣的照片，供您的家人交流；
- 使用虛假照片和假名的個人資料，用於您的個人對話；
- 另一個用來與陌生人聊天的假名檔案。



這就是我們要做的。我首先設定我的主要個人檔案，也就是與我的真實身分連結的那個。點選右下角的個人檔案照片，然後點選使用者名稱。



![Image](assets/fr/21.webp)



然後我會按一下我的個人檔案照片，以變更照片並新增新照片。



![Image](assets/fr/22.webp)



若要新增其他設定檔，請按一下「**您的聊天設定檔**」功能表。



![Image](assets/fr/23.webp)



在這裡您會看到您所有的個人資料。按一下「*新增個人資料*」以建立新的個人資料。



![Image](assets/fr/24.webp)



然後選擇新個人資料的資訊：姓名、照片等。在此，我使用假名和不同的圖像，以便在某些交流中隱藏我的真實身份。



![Image](assets/fr/25.webp)



用手指按住個人資料，就可以隱藏它。這會讓您在應用程式中看不到它，以及所有相關的對話。您也可以選擇「*靜音*」來停止接收通知。



![Image](assets/fr/26.webp)



建立個人資料後，您可以獨立管理它們。在首頁中，只要選擇要顯示的作用中個人資料即可。



![Image](assets/fr/27.webp)



當您建立邀請連結或靜態 Address 時，現在您可以選擇與之相關聯的個人資料。例如，如果我選擇檔案「*Satoshi Nakamoto*」來 generate 連結並寄送給 Alice，她只會看到我的偽名身分「*Satoshi Nakamoto*」，而不會知道我的真實身分「*Loïc Morel*」。相反，如果我從我的真實檔案中提供一個連結給她，她就無法連結到我的偽匿名檔案。



![Image](assets/fr/28.webp)
