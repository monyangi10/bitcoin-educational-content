## I. การนำเสนอ



ในบทแนะนำนี้ เราจะมาดูที่ไฟร์วอลล์โอเพ่นซอร์ส OPNsense เราจะดูคุณสมบัติหลัก ข้อกำหนดเบื้องต้น และวิธีการติดตั้งโซลูชันที่ใช้ FreeBSD นี้



ก่อนเริ่มต้น คุณควรรู้ว่า **OPNsense และ pfSense เป็นไฟร์วอลล์โอเพ่นซอร์สทั้งคู่** ที่พัฒนาบน FreeBSD เราสามารถกล่าวได้ว่า pfSense เป็นเหมือนพี่ใหญ่ของ OPNsense เนื่องจากตัวหลังเป็น Fork ที่สร้างขึ้นในปี 2015 สุดท้ายนี้ สิ่งสำคัญที่ควรชี้ให้เห็นคือ ตั้งแต่ปี 2017 เป็นต้นมา **OPNsense ได้เปลี่ยนไปใช้ HardenedBSD แทน FreeBSD** HardenedBSD เป็นเวอร์ชันที่ปรับปรุงของ FreeBSD พร้อมด้วยคุณสมบัติด้านความปลอดภัยขั้นสูง



OPNsense โดดเด่นด้วยการใช้งานที่ทันสมัยกว่า Interface และ **ความถี่ในการอัปเดตที่มากกว่า** จริง ๆ แล้ว ตารางการอัปเดตของ OPNsense รวมถึงการเปิดตัวหลักสองครั้งต่อปี ซึ่งจะมีการอัปเดตทุก ๆ สองสัปดาห์หรือประมาณนั้น (ส่งผลให้มีการเปิดตัวรอง) การติดตามผลนี้น่าสนใจมากเมื่อเปรียบเทียบกับ pfSense หากเราดูที่เวอร์ชันชุมชนของโซลูชันเหล่านี้



![Image](assets/fr/050.webp)



## II. คุณสมบัติของ OPNsense



OPNsense เป็นระบบปฏิบัติการที่ออกแบบมาเพื่อทำหน้าที่เป็นไฟร์วอลล์และเราเตอร์ แม้ว่าจะมีคุณสมบัติมากมายและสามารถขยายได้โดยการติดตั้งแพ็กเกจเพิ่มเติม เหมาะสำหรับการใช้งานในสภาพแวดล้อมการผลิต โดยส่วนใหญ่จะใช้เพื่อความปลอดภัยของเครือข่ายและการจัดการการไหลของข้อมูล



### A. คุณสมบัติหลัก



นี่คือคุณสมบัติหลักบางประการของ OPNsense:





- ไฟร์วอลล์และ NAT**: OPNsense มอบฟังก์ชันไฟร์วอลล์ขั้นสูงที่มีการกรองแบบ stateful รวมถึงความสามารถในการแปลที่อยู่เครือข่าย (NAT)





- DNS/DHCP**: OPNsense สามารถกำหนดค่าเพื่อจัดการบริการ DNS และ DHCP บนเครือข่ายได้ มันสามารถทำหน้าที่เป็นเซิร์ฟเวอร์ DHCP แต่ยังสามารถใช้เป็น DNS resolver สำหรับเครื่องในเครือข่ายท้องถิ่นได้อีกด้วย Dnsmasq ยังถูกรวมเข้ามาโดยค่าเริ่มต้น





- VPN**: OPNsense รองรับโปรโตคอล VPN หลายประเภท รวมถึง IPsec, OpenVPN และ WireGuard ช่วยให้การเชื่อมต่อที่ปลอดภัยสำหรับการเข้าถึงระยะไกลไปยังเวิร์กสเตชันเคลื่อนที่หรือการเชื่อมต่อระหว่างไซต์





- พร็อกซีเว็บ**: OPNsense รวมถึงพร็อกซีเว็บเพื่อควบคุมและกรองการเข้าถึงอินเทอร์เน็ต นอกจากนี้ยังสามารถใช้กรองเนื้อหาและจัดการการเข้าถึงเครือข่ายได้อีกด้วย





- การจัดการแบนด์วิดท์ (QoS)**: OPNsense มีคุณสมบัติการจัดการคุณภาพการบริการ (QoS) เพื่อจัดลำดับความสำคัญของการจราจรในเครือข่ายและจัดการแบนด์วิดท์ของเครือข่ายได้ดียิ่งขึ้น





- พอร์ทัลเชลย**: ฟีเจอร์นี้ช่วยให้คุณจัดการการเข้าถึงเครือข่ายของผู้ใช้ผ่านหน้าการยืนยันตัวตน (ฐานข้อมูลท้องถิ่น, บัตรกำนัล, ฯลฯ) เป็นฟีเจอร์ที่มักจะถูกใช้งานในเครือข่าย Wi-Fi สาธารณะ





- IDS/IPS**: OPNsense ผสานรวม Suricata เพื่อให้บริการฟังก์ชันการตรวจจับและป้องกันการบุกรุก (IDS/IPS) เพื่อปกป้องเครือข่ายจากการโจมตี





- ความพร้อมใช้งานสูง (CARP)**: OPNsense รองรับ CARP (*Common Address Redundancy Protocol*) เพื่อความพร้อมใช้งานสูงระหว่างไฟร์วอลล์ OPNsense หลายตัว เพื่อให้มั่นใจว่าบริการยังคงทำงานอยู่แม้ในกรณีที่ฮาร์ดแวร์ล้มเหลว





- การรายงานและการตรวจสอบ**: OPNsense มีเครื่องมือรายงานและตรวจสอบแบบเรียลไทม์เพื่อการติดตามประสิทธิภาพของเครือข่าย (ด้วย NetFlow) และตรวจจับปัญหาที่อาจเกิดขึ้นได้ ด้วยการสร้างบันทึก ซึ่งรวมถึงกราฟิก เครื่องมือ Monit ถูกผนวกรวมเข้ากับ OPNsense และช่วยให้สามารถดูแลไฟร์วอลล์ได้เอง



### ข. แพ็คเกจเพิ่มเติม



นี่เป็นเพียงภาพรวมของคุณสมบัติที่เสนอโดย OPNsense นอกจากนี้ **แคตตาล็อกแพ็คเกจ** ที่เข้าถึงได้จากการจัดการ OPNsense Interface ยังช่วยให้คุณสามารถ **เพิ่มความสามารถของไฟร์วอลล์ด้วยฟังก์ชันเพิ่มเติม** ซึ่งรวมถึง ACME client, Wazuh agent, บริการ NTP Chrony และ Caddy ในฐานะ reverse proxy

## III. ข้อกำหนดเบื้องต้นของ OPNsense



ก่อนอื่น คุณต้องตัดสินใจว่าจะติดตั้ง OPNsense ที่ไหน มีหลายวิธีที่เป็นไปได้ รวมถึงการติดตั้งบน:





- ไฮเปอร์ไวเซอร์เป็นเครื่องเสมือน ไม่ว่าจะเป็น Hyper-V, Proxmox, VMware ESXi เป็นต้น
- เครื่องจักรในฐานะระบบ *bare-metal* นี่อาจเป็นมินิพีซีที่ทำหน้าที่เป็นไฟร์วอลล์



คุณยังสามารถซื้อ **อุปกรณ์ติดตั้งบนชั้นวาง OPNsense** ผ่านร้านค้าออนไลน์ของเราได้อีกด้วย



คุณจำเป็นต้องพิจารณาทรัพยากรฮาร์ดแวร์ที่จำเป็นในการรัน OPNsense ซึ่งมีรายละเอียดอยู่ใน [หน้าคู่มือเอกสารนี้](https://docs.opnsense.org/manual/hardware.html)



**ทรัพยากรขั้นต่ำและที่แนะนำสำหรับการผลิต:**




| คุณสมบัติ | ต่ำสุด | คำแนะนำ |
| --- | --- | --- |
| ตัวประมวลผล | 1 GHz - 2 คอร์ | 1.5 GHz - มัลติคอร์ |
| RAM | 2 GB | 8 GB |
| พื้นที่จัดเก็บสำหรับระบบ | ฮาร์ดไดรฟ์ SD การ์ด SSD หรือ (4 GB) | 120 GB บน SSD |

ในที่สุด, **ความต้องการทรัพยากรของคุณขึ้นอยู่กับจำนวนการเชื่อมต่อที่จะต้องจัดการเป็นหลัก**, และดังนั้นจึงขึ้นอยู่กับ **ความต้องการแบนด์วิดท์ของคุณ**. นอกจากนี้, คุณจำเป็นต้อง **คำนึงถึงบริการที่จะถูกเปิดใช้งานและใช้งาน** (พร็อกซี่, การตรวจจับการบุกรุก, ฯลฯ...) เนื่องจากอาจใช้ CPU และ/หรือ RAM มาก.



คุณจะต้องมีไฟล์ภาพติดตั้ง OPNsense ISO ซึ่งสามารถดาวน์โหลดได้จาก [เว็บไซต์ทางการ](https://opnsense.org/download/) สำหรับการติดตั้งบน VM ให้เลือกประเภทภาพเป็น "**dvd**" เพื่อรับไฟล์ภาพ ISO (และทำตามที่คุณต้องการ...) สำหรับการติดตั้งผ่าน USB ที่สามารถบูตได้ ให้เลือกตัวเลือก "**vga**" เพื่อรับไฟล์ "**.img**"



![Image](assets/fr/048.webp)



คุณจะต้องมีคอมพิวเตอร์สำหรับการบริหารและทดสอบ OPNsense ด้วยเช่นกัน



## IV. การกำหนดค่าเป้าหมาย



เป้าหมายของเราคือการ





- สร้างเครือข่ายเสมือนภายใน (192.168.10.0/24 - LAN) ซึ่งสามารถเข้าถึงอินเทอร์เน็ตผ่านไฟร์วอลล์ OPNsense สำหรับการใช้งานในสภาพแวดล้อมการผลิต นี่อาจเป็นเครือข่ายท้องถิ่นของคุณ สายเคเบิลและ/หรือ Wi-Fi
- เปิดใช้งานและกำหนดค่า **NAT** เพื่อให้ VM ในเครือข่ายเสมือนภายในสามารถเข้าถึงอินเทอร์เน็ตได้
- เปิดใช้งานและกำหนดค่าเซิร์ฟเวอร์ DHCP บน **OPNsense** เพื่อแจกจ่ายการกำหนดค่า IP ให้กับเครื่องในอนาคตที่เชื่อมต่อกับเครือข่ายเสมือนภายใน
- กำหนดค่า **firewall** เพื่ออนุญาตเฉพาะการไหลออกจาก LAN ไปยัง WAN ใน HTTP (80) และ HTTPS (443) เท่านั้น
- กำหนดค่า **firewall** เพื่อให้ virtual LAN ใช้ OPNsense เป็น DNS resolver (53)



หากคุณกำลังใช้แพลตฟอร์ม Hyper-V สิ่งนี้จะให้การแสดงผลดังต่อไปนี้:

## V. การติดตั้งไฟร์วอลล์ OPNsense



### A. การเตรียม USB Key ที่สามารถบูตได้



ขั้นตอนแรกคือการเตรียมสื่อการติดตั้ง: **USB key ที่สามารถบูตได้พร้อมกับ OPNsense** แน่นอนว่านี่เป็นทางเลือกหากคุณทำงานในสภาพแวดล้อมเสมือนจริง แต่ไม่ว่าในกรณีใดคุณจำเป็นต้องดาวน์โหลดภาพ ISO การติดตั้ง OPNsense



หลังจากดาวน์โหลด คุณจะได้รับ **ไฟล์อาร์ไคฟ์ที่มีภาพในรูปแบบ ".img"** คุณสามารถ **สร้าง USB ที่สามารถบูตได้** ด้วยแอปพลิเคชันต่างๆ รวมถึง **balenaEtcher**: ใช้งานง่ายมาก นอกจากนี้ แอปพลิเคชันจะรู้จักภาพในไฟล์อาร์ไคฟ์ ดังนั้นคุณไม่จำเป็นต้องแยกไฟล์ออกมาก่อน





- [ดาวน์โหลด balenaEtcher](https://etcher.balena.io/)



เมื่อติดตั้งแอปพลิเคชันแล้ว ให้เลือกภาพของคุณ กุญแจ USB ของคุณ จากนั้นคลิกที่ "Flash! รอสักครู่



![Image](assets/fr/049.webp)



ตอนนี้คุณพร้อมที่จะติดตั้งแล้ว



### B. การติดตั้งระบบ OPNsense



เริ่มต้นเครื่องที่โฮสต์ OPNsense คุณควรเห็นหน้าต้อนรับที่คล้ายกับด้านล่างนี้ หน้าจอที่แสดงจะปรากฏในหน้าต่างเป็นเวลาสองสามวินาที ปล่อยให้กระบวนการดำเนินไปตามปกติ...



![Image](assets/fr/019.webp)



ภาพ OPNsense ถูกโหลดเข้าสู่เครื่องแล้ว เพื่อให้ระบบสามารถเข้าถึงในโหมด "**live**" ได้ กล่าวคือ ถูกจัดเก็บชั่วคราวในหน่วยความจำ



![Image](assets/fr/025.webp)



จากนั้นคุณจะพบกับ Interface ที่คล้ายกับด้านล่างนี้ เข้าสู่ระบบด้วยชื่อผู้ใช้ "**installer**" และรหัสผ่าน "**opnsense**" โปรดทราบว่าแป้นพิมพ์เป็นแบบ **QWERTY** ในขณะนี้ ในขั้นตอนนี้ เราจะ **เริ่มกระบวนการติดตั้ง OPNsense**



![Image](assets/fr/026.webp)



มีวิซาร์ดใหม่ปรากฏบนหน้าจอ ขั้นตอนแรกคือการเลือกการจัดวางแป้นพิมพ์ที่ตรงกับการกำหนดค่าของคุณ สำหรับแป้นพิมพ์ AZERTY ให้เลือกตัวเลือก **"French (accent keys)"** จากรายการ จากนั้นดับเบิลคลิก



![Image](assets/fr/027.webp)



ขั้นตอนที่สองคือการเลือกงานที่จะดำเนินการ ที่นี่เราจะทำการติดตั้งโดยใช้ **ระบบไฟล์ ZFS** วางตำแหน่งตัวเองบนบรรทัด "**Install (ZFS)**" และยืนยันด้วยการกด **Enter**



![Image](assets/fr/028.webp)



ในขั้นตอนที่สาม ให้เลือก "**stripe**" เนื่องจากเครื่องของเรามี **ดิสก์เพียงหนึ่งเดียว**: ไม่มีความซ้ำซ้อนที่เป็นไปได้ในการรักษาความปลอดภัยของที่เก็บไฟร์วอลล์และข้อมูลของมัน สิ่งนี้มีความสำคัญเป็นพิเศษเมื่อทำการติดตั้งบนเครื่องจริงเพื่อป้องกันความล้มเหลวของฮาร์ดแวร์ของดิสก์ ผ่านหลักการของ RAID



![Image](assets/fr/029.webp)



ในขั้นตอนที่สี่ เพียงกด **Enter** เพื่อยืนยัน



![Image](assets/fr/030.webp)



สุดท้าย ยืนยันโดยเลือก "**YES**" แล้วกดปุ่ม **Enter**



![Image](assets/fr/031.webp)



ตอนนี้คุณจะต้องรอในขณะที่กำลังติดตั้ง OPNsense... กระบวนการนี้ใช้เวลาประมาณ 5 นาที



![Image](assets/fr/032.webp)



เมื่อติดตั้งเสร็จสิ้น เราสามารถเปลี่ยนรหัสผ่าน "**root**" ก่อนรีบูตได้ เลือก "**Root Password**", กด **Enter** และป้อนรหัสผ่าน "**root**" สองครั้ง



![Image](assets/fr/020.webp)



สุดท้าย เลือก "**Complete Install**" และกด **Enter** ใช้โอกาสนี้ในการ **นำดิสก์ออกจากไดรฟ์ DVD ของ VM** ในการตั้งค่า VM คุณยังสามารถตั้งค่าการบูตครั้งแรกให้เป็นดิสก์ได้



![Image](assets/fr/021.webp)



เครื่องเสมือนจะรีบูตและโหลดระบบ OPNsense จากดิสก์ เนื่องจากเราเพิ่งติดตั้งมัน เข้าสู่ระบบด้วยบัญชี "root" ในคอนโซล และรหัสผ่านใหม่ของคุณ (มิฉะนั้น รหัสผ่านเริ่มต้นคือ "**opnsense**")



### ง. การเชื่อมโยงอินเทอร์เฟซเครือข่าย



หน้าจอที่แสดงด้านล่างจะปรากฏขึ้น เลือก "**1**" และกด **Enter** เพื่อเชื่อมโยงการ์ดเครือข่ายของเครื่องกับอินเทอร์เฟซ OPNsense



![Image](assets/fr/022.webp)



ก่อนอื่น พ่อมดจะขอให้คุณกำหนดค่าการรวมลิงก์และ VLANs ระบุ "**n**" เพื่อปฏิเสธ และทุกครั้งให้ยืนยันคำตอบของคุณด้วย **Enter** ต่อไป คุณต้องกำหนดอินเทอร์เฟซสองตัว "**hn0**" และ "**hn1**" ให้กับ **WAN** และ **LAN** โดยหลักการแล้ว "**hn0**" สอดคล้องกับ WAN และอีกตัว Interface กับ LAN



นี่คือวิธีการทำงาน:



![Image](assets/fr/023.webp)



เรามีแล้ว:





- Interface **LAN** ที่เชื่อมโยงกับการ์ดเครือข่าย "**hn1**" และมีที่อยู่ IP เริ่มต้นของ OPNsense คือ **192.168.1.1/24**.
- Interface **WAN** ที่เชื่อมโยงกับการ์ดเครือข่าย "**hn0**" และมีที่อยู่ IP ที่ดึงมาผ่าน **DHCP** บนเครือข่ายท้องถิ่น (ขอบคุณสวิตช์เสมือนภายนอกของเรา)



ตามค่าเริ่มต้น การจัดการ OPNsense Interface สามารถเข้าถึงได้เฉพาะจาก LAN Interface เท่านั้น ด้วยเหตุผลด้านความปลอดภัยที่ชัดเจน ดังนั้นคุณต้องเชื่อมต่อกับ LAN ของไฟร์วอลล์ Interface เพื่อทำการจัดการ หากไม่สามารถทำได้ คุณสามารถจัดการ OPNsense ชั่วคราวจาก WAN ซึ่งเกี่ยวข้องกับการปิดใช้งานฟังก์ชันไฟร์วอลล์



ในการทำเช่นนี้ ให้เปลี่ยนไปที่โหมดเชลล์ผ่านตัวเลือก "**8**" และรันคำสั่งต่อไปนี้:



```
pfctl -d
```

### E. การเข้าถึงระบบการจัดการ OPNsense Interface



OPNsense Administration Interface สามารถเข้าถึงได้ผ่าน HTTPS โดยใช้ที่อยู่ IP ของ **LAN** Interface (หรือ WAN) เบราว์เซอร์ของคุณจะนำคุณไปยังหน้าล็อกอิน เข้าสู่ระบบด้วยบัญชี "root" และรหัสผ่านที่คุณเลือกไว้ก่อนหน้านี้



![Image](assets/fr/034.webp)



รอสักครู่... คุณจะได้รับการแจ้งให้ทำตามวิซาร์ดเพื่อทำการตั้งค่าพื้นฐาน คลิก "**ถัดไป**" เพื่อดำเนินการต่อ



![Image](assets/fr/036.webp)



ขั้นตอนแรกคือการกำหนดชื่อโฮสต์ ชื่อโดเมน เลือกภาษา และกำหนดเซิร์ฟเวอร์ DNS ที่จะใช้สำหรับการแก้ไขชื่อ การเปิดใช้งานตัวเลือก "**Enable Resolver**" จะทำให้ไฟร์วอลล์สามารถใช้เป็น DNS resolver ได้ ซึ่งจะเป็นประโยชน์สำหรับเครื่องในเครือข่ายเสมือนของเรา



![Image](assets/fr/037.webp)



ดำเนินการไปยังขั้นตอนถัดไป วิซาร์ดจะให้คุณเลือก **กำหนดเซิร์ฟเวอร์ NTP สำหรับการซิงโครไนซ์วันที่และเวลา** แม้ว่าจะมีเซิร์ฟเวอร์ที่กำหนดไว้ล่วงหน้าแล้วก็ตาม นอกจากนี้ การเลือกเขตเวลาที่ตรงกับตำแหน่งทางภูมิศาสตร์ของคุณเป็นสิ่งสำคัญ (เว้นแต่คุณจะมีข้อกำหนดพิเศษ)



![Image](assets/fr/038.webp)



จากนั้นมาถึงขั้นตอนสำคัญ: **การกำหนดค่า Interface WAN** ปัจจุบันถูกกำหนดค่าใน DHCP และจะยังคงอยู่ในโหมดการกำหนดค่านี้ เว้นแต่คุณต้องการตั้งค่า IP address แบบคงที่



![Image](assets/fr/039.webp)



ยังคงอยู่ในหน้าการกำหนดค่า Interface WAN คุณจำเป็นต้องยกเลิกการเลือกตัวเลือก "**Block access to private networks via WAN**" หากเครือข่ายด้าน WAN ใช้การกำหนดที่อยู่แบบส่วนตัว ซึ่งอาจเป็นกรณีนี้หากคุณกำลังใช้งานห้องปฏิบัติการ และอาจป้องกันคุณจากการเข้าถึงอินเทอร์เน็ต



![Image](assets/fr/040.webp)



ถัดไป คุณสามารถ **กำหนด "รหัสผ่าน root"** ได้ แต่สิ่งนี้เป็นทางเลือกเพราะเราได้ทำไปแล้ว



![Image](assets/fr/041.webp)



ดำเนินการต่อจนสิ้นสุดเพื่อเริ่มการโหลดการกำหนดค่าใหม่ หากคุณต้องการควบคุมผ่าน WAN ต่อไป ให้เริ่มคำสั่งข้างต้นใหม่เมื่อกระบวนการนี้เสร็จสิ้น

### อี. การกำหนดค่า DHCP



เป้าหมายของเราคือการใช้เซิร์ฟเวอร์ DHCP OPNsense เพื่อแจกจ่ายที่อยู่ IP บนเครือข่ายเสมือน เพื่อทำสิ่งนี้ เราจำเป็นต้องเข้าถึงตำแหน่งเมนูนี้:



```
Services > ISC DHCPv4 > [LAN]
```



**ตามที่คุณเห็น DHCP ถูกเปิดใช้งานโดยค่าเริ่มต้นบน LAN ** หากคุณไม่สนใจในบริการนี้ คุณควรปิดการใช้งาน แม้ว่ามันจะถูกเปิดใช้งานแล้วและเราต้องการใช้มัน แต่ก็สำคัญที่จะตรวจสอบการกำหนดค่าของมัน



หากจำเป็น คุณสามารถเปลี่ยนช่วงของที่อยู่ IP ที่จะกระจายได้: **192.168.10.10** ถึง **192.168.10.245** ขึ้นอยู่กับการตั้งค่าปัจจุบัน



![Image](assets/fr/046.webp)



เรายังสามารถเห็นได้ว่า ฟิลด์ "**DNS servers**", "**Gateway**", "**Domain name**", เป็นต้น ว่างเปล่าโดยค่าเริ่มต้น ในความเป็นจริง มีการสืบทอดอัตโนมัติของตัวเลือกบางอย่างและค่าเริ่มต้นสำหรับฟิลด์ต่างๆ เหล่านี้ ตัวอย่างเช่น สำหรับเซิร์ฟเวอร์ DNS จะมีการแจกจ่ายที่อยู่ IP ของ Interface LAN ทำให้สามารถใช้ไฟร์วอลล์ OPNsense เป็น DNS resolver ได้



บันทึกการกำหนดค่าหลังจากทำการเปลี่ยนแปลงใด ๆ หากจำเป็น



![Image](assets/fr/047.webp)



ในการทดสอบเซิร์ฟเวอร์ DHCP คุณจำเป็นต้องเชื่อมต่อเครื่องเข้ากับเครือข่าย LAN ของไฟร์วอลล์ของคุณ



เครื่องนี้ต้องได้รับที่อยู่ IP จากเซิร์ฟเวอร์ DHCP OPNsense และเครื่องของเราต้องสามารถเข้าถึงเครือข่ายได้ การเข้าถึงอินเทอร์เน็ตต้องทำงานได้ โปรดทราบว่าหากคุณปิดการทำงานของไฟร์วอลล์เพื่อเข้าถึง OPNsense จาก WAN จะทำให้ NAT ถูกปิดใช้งาน ซึ่งจะป้องกันไม่ให้คุณเข้าถึงเว็บได้



**หมายเหตุ**: การเช่า DHCP ที่ออกในปัจจุบันสามารถดูได้จากการจัดการ OPNsense ที่ Interface ในการทำเช่นนี้ ให้ไปที่ตำแหน่งต่อไปนี้: **Services > ISC DHCPv4 > Leases**.



![Image](assets/fr/045.webp)



### ฉ. การกำหนดค่า NAT และกฎไฟร์วอลล์



ข่าวดีคือเราสามารถเข้าถึงการจัดการ OPNsense Interface จาก LAN ได้แล้ว



```
https://192.168.1.10
```



หลังจากเข้าสู่ระบบ OPNsense แล้ว มาค้นหาการตั้งค่า NAT กัน ไปที่ตำแหน่งนี้: **Firewall > NAT > Outbound** ที่นี่คุณสามารถเลือกได้ระหว่างการสร้างกฎ NAT ขาออกแบบอัตโนมัติ (ค่าเริ่มต้น) และแบบแมนนวล



เลือกโหมดอัตโนมัติผ่านตัวเลือก "**การสร้างกฎ NAT ขาออกอัตโนมัติ**" และคลิกที่ "**บันทึก**" (โดยหลักการแล้ว การกำหนดค่านี้เป็นการกำหนดค่าที่ใช้งานอยู่แล้ว) ในโหมดอัตโนมัติ OPNsense จะสร้างกฎ NAT สำหรับแต่ละเครือข่ายของคุณเอง



![Image](assets/fr/043.webp)



ในขณะนี้ คอมพิวเตอร์ทั้งหมดที่เชื่อมต่อกับเครือข่ายเสมือน "**192.168.10.0/24**" สามารถเข้าถึงอินเทอร์เน็ตได้โดยไม่มีข้อจำกัด อย่างไรก็ตาม เป้าหมายของเราคือการจำกัดการเข้าถึง WAN ให้เฉพาะโปรโตคอล HTTP และ HTTPS รวมถึง DNS บน LAN ของไฟร์วอลล์ Interface



ดังนั้นเราจำเป็นต้องสร้างกฎไฟร์วอลล์... เรียกดูเมนูดังนี้: **Firewall > Rules > LAN**.



**ตามค่าเริ่มต้น จะมีสองกฎในการอนุญาตให้ทราฟฟิก LAN ขาออกทั้งหมด ใน IPv4 และ IPv6** ปิดใช้งานกฎทั้งสองนี้โดยคลิกที่ลูกศรสีเขียวทางด้านซ้ายสุด ที่จุดเริ่มต้นของแต่ละบรรทัด



จากนั้นสร้างกฎใหม่สามข้อเพื่ออนุญาตให้ **LAN network** (เช่น "**LAN net**") เพื่อ:





- เข้าถึงทุกจุดหมายโดยใช้ **HTTP**.
- เข้าถึงทุกจุดหมายด้วย **HTTPS**.
- ขอ **OPNsense** บน **Interface LAN** (เช่น "**LAN address**") ผ่าน **DNS protocol** (ซึ่งหมายถึงการใช้ไฟร์วอลล์เป็น DNS) มิฉะนั้นให้อนุญาต DNS resolver ของคุณผ่านที่อยู่ IP ของมัน



สิ่งนี้ให้ผลลัพธ์ดังต่อไปนี้:



![Image](assets/fr/044.webp)



สิ่งที่เหลืออยู่คือการคลิกที่ "**Apply changes**" เพื่อเปลี่ยนกฎไฟร์วอลล์ใหม่ไปยังการผลิต **โปรดทราบว่าโฟลว์ทั้งหมดที่ไม่ได้รับอนุญาตอย่างชัดเจนจะถูกบล็อกโดยค่าเริ่มต้น**



เครื่อง LAN สามารถเข้าถึงอินเทอร์เน็ตได้โดยใช้ HTTP และ HTTPS โปรโตคอลอื่น ๆ ทั้งหมดจะถูกบล็อก



![Image](assets/fr/018.webp)



## IV. บทสรุป



โดยทำตามคู่มือนี้ คุณจะสามารถติดตั้ง OPNsense และเริ่มต้นใช้งานได้ทันที OPNsense มีฟีเจอร์หลากหลายเพื่อรักษาความปลอดภัยและจัดการการจราจรเครือข่ายของคุณอย่างมีประสิทธิภาพ และเหมาะสำหรับการใช้งานในสภาพแวดล้อมมืออาชีพ



การติดตั้งนี้เป็นเพียงจุดเริ่มต้น: อย่าลังเลที่จะสำรวจเมนูและกำหนดค่าฟีเจอร์อื่นๆ เพื่อปรับให้ OPNsense ตรงตามความต้องการของคุณ
