## I. ارائه



در این آموزش، ما یاد خواهیم گرفت که چگونه یک VPN بر اساس WireGuard، یک راه‌حل VPN رایگان و متن‌باز که عملکرد را بدون نادیده گرفتن امنیت افزایش می‌دهد، پیکربندی کنیم.



WireGuard یک راه‌حل نسبتاً جدید است که از مارس 2020 به‌عنوان یک نسخه پایدار در دسترس بوده و افتخار این را داشته که به‌طور مستقیم در **هسته لینوکس از نسخه 5.6** ادغام شود. این امر مانع از دسترسی آن از توزیع‌های قدیمی‌تر لینوکس که از نسخه‌های مختلف هسته استفاده می‌کنند، نمی‌شود. در مقایسه با راه‌حل‌هایی مانند OpenVPN و IPSec، WireGuard ساده‌تر برای استفاده و بسیار سریع‌تر است، همان‌طور که در پایان این مقاله خواهیم دید.



برخی نکات کلیدی درباره WireGuard:





- ساده، سبک و فوق‌العاده کارآمد!
- عملیات فقط UDP (که می‌تواند هنگام عبور از برخی فایروال‌ها یک نقطه ضعف باشد)
- بر اساس مدل همتا به همتا به جای مدل مشتری-سرور کار می‌کند
- احراز هویت توسط کلید Exchange، بر اساس همان اصل SSH با کلیدهای خصوصی/عمومی
- استفاده از چندین الگوریتم: رمزنگاری متقارن با ChaCha20، احراز هویت پیام با Poly1305، و همچنین دیگر الگوریتم‌ها مانند Curve25519، BLAKE2 و SipHash24
- از هر دو IPv4 و IPv6 پشتیبانی می‌کند
- چند پلتفرم: ویندوز، لینوکس، BSD، macOS، اندروید، iOS، OpenWRT (و پیاده‌سازی شده در ProtonVPN)
- فقط ۴۰۰۰ خط کد، در مقایسه با صدها هزار خط برای راه‌حل‌های دیگر



برای بخش رمزنگاری، الگوریتم‌های مختلفی که استفاده می‌شوند به‌صورت دستی انتخاب شده، به روش‌های مختلف حسابرسی شده و توسط پژوهشگران امنیتی متخصص در این زمینه بررسی می‌شوند.



وب‌سایت رسمی پروژه: [wireguard.com](https://www.wireguard.com/)



آیا پس از خواندن این مقدمه، از این راه‌حل قانع شده‌اید؟ تنها کاری که باقی مانده، ادامه خواندن است!



## II. نمودار آزمایشگاه WireGuard



قبل از اینکه آزمایشگاه خود را برای راه‌اندازی WireGuard ارائه دهم، باید بدانید که می‌توانید **از WireGuard برای اتصال دو زیرساخت از راه دور استفاده کنید**، اما همچنین برای **اتصال یک مشتری از راه دور به زیرساختی مانند یک شبکه شرکتی یا شبکه خانگی خود**. این در همان روحیه‌ای است که با OpenVPN، همانطور که اخیراً در آموزش "[OpenVPN روی Synology](https://www.it-connect.fr/tuto-vpn-configurer-openvpn-server-sur-un-nas-synology/)" دیدیم.



اگر WireGuard به‌طور مستقیم روی روتر یا فایروال تنظیم نشده باشد، همان‌طور که با OpenWRT قابل تصور است، باید انتقال پورت را تنظیم کنید تا جریان به جفت WireGuard برسد.



![Image](assets/fr/034.webp)



حالا درباره آزمایشگاهم و آنچه امروز قصد داریم راه‌اندازی کنیم به شما خواهم گفت.



من قصد دارم از یک ماشین Debian 11 به عنوان سرور WireGuard و از یک کلاینت ویندوز به عنوان کلاینت VPN WireGuard استفاده کنم. اگرچه صحبت کردن درباره یک رابطه کلاینت-سرور کمی گمراه‌کننده است، زیرا **WireGuard بر اساس مدل "نظیر به نظیر" کار می‌کند.** این زمانی که شما در حال تلاش برای راه‌اندازی یک اتصال "کلاینت به سایت" هستید، کمی اشتباه‌نام است. بیایید بگوییم که من قصد دارم دو جفت یا **دو نقطه اتصال WireGuard** داشته باشم اگر ترجیح می‌دهید: یکی تحت Debian 11 و دیگری تحت ویندوز.



این دو جفت می‌توانند در هر دو جهت با یکدیگر ارتباط برقرار کنند، به این معنی که از دستگاه ویندوز من می‌توانم به شبکه محلی دستگاه Debian 11 دسترسی پیدا کنم و بالعکس: همه چیز به پیکربندی تونل و فایروال همتای WireGuard بستگی دارد.



در این مثال، من بر روی مورد زیر تمرکز خواهم کرد: **از ویندوز Peer 1 خود که به شبکه خانگی‌ام متصل است، می‌خواهم به شبکه شرکتی‌ام دسترسی پیدا کنم تا بتوانم از طریق تونل VPN WireGuard به سرورهای شرکت دسترسی داشته باشم**. این دیاگرام زیر را به ما می‌دهد:



![Image](assets/fr/035.webp)



از نظر آدرس‌های IP، این به صورت زیر است:





- **شبکه خانگی**: 192.168.1.0/24
- **شبکه شرکتی**: 192.168.100.0/24
- **شبکه تونل WireGuard**: 192.168.110.0/24


+ IP Address از Peer 1 (ویندوز) در تونل: 192.168.110.2/24


+ IP Address از Peer 2 (Debian) در تونل: 192.168.110.121/24



همین است! بیایید به پیکربندی بپردازیم!



**توجه: به طور پیش‌فرض، WireGuard در حالت UDP روی پورت 51820 عمل می‌کند.**



## III نصب و پیکربندی سرور WireGuard



در این آموزش، من از اصطلاحات "کلاینت" برای ماشین ویندوز و "سرور" برای ماشین دبیان استفاده خواهم کرد، زیرا حتی اگر این ارتباط همتا به همتا باشد، به نظر می‌رسد که این اصطلاحات معنادارتر هستند.



### A. نصب WireGuard روی Debian 11



بسته نصب WireGuard در مخازن رسمی Debian 11 موجود است، بنابراین تنها کاری که باید انجام دهیم این است که حافظه پنهان بسته را به‌روزرسانی کرده و آن را نصب کنیم:



```
sudo apt-get update
```



```
sudo apt-get install wireguard
```



![Image](assets/fr/022.webp)



قسمت سرور WireGuard نصب خواهد شد، همراه با ابزارهای مختلفی که دسترسی به دستورات مفید برای مدیریت پیکربندی را فراهم می‌کنند.



### B. نصب Interface WireGuard



با استفاده از **دستور "wg "** ما نیاز داریم که یک کلید خصوصی و یک کلید عمومی generate کنیم: که برای احراز هویت بین دو جفت، یعنی سرور و یک کلاینت (که او نیز به یک جفت کلید نیاز خواهد داشت) ضروری است.



ما کلید خصوصی "**/etc/wireguard/wg-private.key**" و کلید عمومی "**/etc/wireguard/wg-public.key**" را با این دنباله فرمان ایجاد خواهیم کرد:



```
wg genkey | sudo tee /etc/wireguard/wg-private.key | wg pubkey | sudo tee /etc/wireguard/wg-public.key
```



![Image](assets/fr/023.webp)



مقدار کلید عمومی در کنسول بازگردانده خواهد شد. در فایل پیکربندی WireGuard، باید **مقدار کلید خصوصی خود را اضافه کنیم**. برای بازیابی این مقدار، فرمان زیر را وارد کرده و مقدار را کپی کنید:



```
sudo cat /etc/wireguard/wg-private.key
```



زمان آن رسیده است که یک فایل پیکربندی در "**/etc/wireguard/**" ایجاد کنیم. به عنوان مثال، می‌توانیم این فایل را "**wg0.conf**" نام‌گذاری کنیم، اگر فکر می‌کنیم که شبکه Interface مرتبط با WireGuard "wg0" خواهد بود، بر اساس همان اصلی که برای "eth0" استفاده می‌شود.



```
sudo nano /etc/wireguard/wg0.conf
```



در این فایل، ابتدا باید محتوای زیر را اضافه کنیم (بعداً برای تکمیل آن برمی‌گردیم):



```
[Interface]
Address = 192.168.110.121/24
SaveConfig = true
ListenPort = 51820
PrivateKey = <clé privée du serveur>
```



بخش `[Interface]` برای اعلام قسمت سرور استفاده می‌شود. در اینجا اطلاعاتی آمده است:





- **Address**: آدرس IP Address از Interface WireGuard درون تونل VPN (زیرشبکه‌ای متفاوت از LAN راه دور)
- **ذخیره‌پیکربندی**: پیکربندی تا زمانی که Interface فعال است ذخیره (و محافظت) می‌شود
- **ListenPort**: پورت شنود WireGuard. در اینجا، 51820 پورت پیش‌فرض است، اما شما می‌توانید آن را به دلخواه تغییر دهید.
- **کلید خصوصی**: مقدار کلید خصوصی سرور ما (*wg-private.key*)



فایل را ذخیره کرده و آن را ببندید. با استفاده از دستور "**wg-quick**"، می‌توانیم این Interface را با مشخص کردن نام آن (wg0، همانطور که فایل به نام wg0.conf است) شروع کنیم:



```
sudo wg-quick up wg0
```



اگر آدرس‌های IP سرور Debian 11 خود را فهرست کنید، یک Interface جدید به نام "wg0" با IP Address که در فایل پیکربندی تعریف شده است، مشاهده خواهید کرد:



```
ip a
```



![Image](assets/fr/027.webp)



به همین ترتیب، می‌توانیم پیکربندی Interface "wg0" را از طریق دستور "wg show" نمایش دهیم:



```
sudo wg show wg0
```



![Image](assets/fr/024.webp)



در نهایت، ما نیاز داریم که راه‌اندازی خودکار Interface wg0 WireGuard خود را فعال کنیم:



```
sudo systemctl enable wg-quick@wg0.service
```



فعلاً، پیکربندی سمت سرور WireGuard را کنار می‌گذاریم.



### C. فعال‌سازی IP Forwarding



برای اینکه ماشین Debian 11 ما بتواند **بسته‌ها را بین شبکه‌های مختلف (مانند یک روتر) مسیریابی کند**، یعنی بین شبکه VPN و شبکه محلی، نیاز داریم تا [IP Forwarding](https://www.it-connect.fr/activer-lip-forwarding-sous-linux-ipv4ipv6/) را فعال کنیم. به طور پیش‌فرض، این ویژگی غیرفعال است.



پیکربندی این فایل را تغییر دهید:



```
sudo nano /etc/sysctl.conf
```



این دستور را به انتهای فایل اضافه کرده و ذخیره کنید:



```
net.ipv4.ip_forward = 1
```



همه‌اش همین است.



### D. فعال‌سازی IP Masquerade



برای اینکه سرور ما بسته‌ها را به‌درستی مسیریابی کند و شبکه محلی راه دور برای ماشین ویندوز قابل دسترسی باشد، نیاز داریم که IP Masquerade را روی سرور دبیان خود فعال کنیم. این نوعی فعال‌سازی NAT است. من قصد دارم این پیکربندی را روی فایروال لینوکس از طریق UFW انجام دهم، که به استفاده از آن عادت دارم ([آموزش ufw روی دبیان](https://www.it-connect.fr/configurer-un-pare-feu-local-sous-debian-11-avec-ufw/)).



اگر قبلاً UFW را ندارید و می‌خواهید آن را راه‌اندازی کنید (می‌توانید از Nftables نیز استفاده کنید)، با نصب شروع کنید:



```
sudo apt install ufw
```



اول از همه، باید SSH را فعال کنید تا کنترل سرور راه دور را از دست ندهید (شماره پورت را تطبیق دهید):



```
sudo ufw allow 22/tcp
```



پورت 51820 در UDP نیز باید مجاز شود، زیرا ما از آن برای WireGuard استفاده می‌کنیم (دوباره، شماره پورت را تطبیق دهید):



```
sudo ufw allow 51820/udp
```



سپس، ما با پیکربندی برای فعال‌سازی IP masquerade ادامه خواهیم داد. برای انجام این کار، باید نام Interface متصل به شبکه محلی را بازیابی کنیم. اگر نام را نمی‌دانید، "ip a" را اجرا کنید تا نام کارت را ببینید. در مورد من، کارت "**ens192**" است.



![Image](assets/fr/036.webp)



ما قصد داریم از این اطلاعات استفاده کنیم. فایل زیر را ویرایش کنید:



```
sudo nano /etc/ufw/before.rules
```



این خطوط را در انتهای فایل اضافه کنید تا **IP masquerade را بر روی Interface ens192 فعال کنید** (نام Interface را تطبیق دهید) درون رشته POSTROUTING جدول NAT فایروال محلی ما:



```
# NAT - IP masquerade
*nat*
:POSTROUTING ACCEPT [0:0]
-A POSTROUTING -o ens192 -j MASQUERADE

# End each table with the 'COMMIT' line or these rules won't be processed
COMMIT
```



تصویر نشان می‌دهد:



![Image](assets/fr/037.webp)



این فایل پیکربندی را باز نگه دارید و به مرحله بعدی بروید. 😉



### E. پیکربندی فایروال لینوکس برای WireGuard



همچنان در همان فایل پیکربندی، قصد داریم شبکه شرکتی "192.168.100.0/24" را اعلام کنیم تا بتوانیم با آن ارتباط برقرار کنیم. در اینجا دو قانونی که باید اضافه شوند (بهتر است بعد از بخش "*# ok icmp code for FORWARD*" برای گروه‌بندی قوانین با هم) آورده شده است:



```
# autoriser le forwarding pour le réseau distant de confiance (+ le réseau du VPN)
-A ufw-before-forward -s 192.168.100.0/24 -j ACCEPT
-A ufw-before-forward -d 192.168.100.0/24 -j ACCEPT
-A ufw-before-forward -s 192.168.110.0/24 -j ACCEPT
-A ufw-before-forward -d 192.168.110.0/24 -j ACCEPT
```



اگر می‌خواهید فقط یک میزبان را مجاز کنید، به عنوان مثال "192.168.100.11"، این کار آسان است:



```
-A ufw-before-forward -s 192.168.100.11/32 -j ACCEPT
-A ufw-before-forward -d 192.168.100.11/32 -j ACCEPT
```



اکنون می‌توانید فایل را ذخیره کرده و آن را ببندید. تنها کاری که باقی مانده است فعال‌سازی UFW و راه‌اندازی مجدد سرویس برای اعمال تغییرات ما است:



```
sudo ufw enable
```



```
sudo systemctl restart ufw
```



اولین بخش از پیکربندی سرور دبیان اکنون کامل شده است.



## IV. کلاینت WireGuard برای ویندوز



کلاینت WireGuard برای ویندوز، macOS، اندروید و غیره در دسترس است... این خبر بسیار خوبی است. تمامی دانلودها در این صفحه موجود است: [WireGuard Client](https://www.wireguard.com/install/). در این مثال، من قصد دارم کلاینت را روی ویندوز راه‌اندازی کنم. برای راه‌اندازی کلاینت WireGuard روی لینوکس، همان اصول ایجاد فایل wg0.conf روی ماشین دبیان را دنبال کنید (بدون تمامی تنظیمات NAT و غیره).



### الف. نصب کلاینت WireGuard ویندوز



پس از دانلود فایل اجرایی یا بسته MSI، نصب به سادگی انجام می‌شود: فقط نصب‌کننده را اجرا کنید، و... voilà، تمام شد! 🙂



![Image](assets/fr/038.webp)



### B. ایجاد یک پروفایل WireGuard



با باز کردن نرم‌افزار برای ایجاد یک تونل جدید شروع کنید. برای انجام این کار، روی فلش سمت راست دکمه "**Add tunnel**" کلیک کنید و سپس روی دکمه "**Add empty tunnel**" کلیک کنید.



![Image](assets/fr/028.webp)



یک پنجره پیکربندی باز خواهد شد. هر بار که یک پیکربندی تونل جدید ایجاد می‌شود، WireGuard یک جفت کلید خصوصی/عمومی خاص برای این پیکربندی تولید می‌کند. **در این پیکربندی، ما باید "همتا" را اعلام کنیم، یعنی سرور راه دور:**



```
[Interface]
PrivateKey = <la clé privée du PC>
```



ما نیاز داریم این پیکربندی را کامل کنیم، به‌ویژه برای اعلام IP Address بر روی این Interface (*Address*)، اما همچنین برای اعلام سرور WireGuard از راه دور از طریق یک بلوک [Peer]. تصویر زیر باید شما را به یاد فایل پیکربندی که در سمت سرور لینوکس ایجاد کردیم بیندازد.



بیایید با بلوک `[Interface]` شروع کنیم و IP Address "**192.168.110.2**" را اضافه کنیم؛ به یاد داشته باشید که سرور دارای IP Address "**192.168.110.121**" در این بخش شبکه است. این به ما می‌دهد:



```
[Interface]
PrivateKey = <la clé privée du PC>
Address = 192.168.110.2/24
```



سپس، باید بلوک "Peer" را با سه ویژگی اعلام کنیم که منجر به این پیکربندی می‌شود:



```
[Peer]
PublicKey = 1D/Gf5yd3hUDoFyYQ3P1zayBHUJhJZq+k6Sv03HnJCQ=
AllowedIPs = 192.168.110.0/24, 192.168.100.0/24
Endpoint = <ip-serveur-debian>:51820
```



در تصاویر:



![Image](assets/fr/029.webp)



**چند توضیح درباره‌ی بلوک [Peer]:**





- **PublicKey**: این کلید عمومی سرور WireGuard Debian 11 است (می‌توانید مقدار آن را با دستور "*sudo wg*" به دست آورید)
- **AllowedIPs**: این‌ها آدرس‌های IP / زیرشبکه‌هایی هستند که از طریق این شبکه VPN WireGuard قابل دسترسی هستند، در این مورد زیرشبکه خاص VPN WireGuard من (*192.168.110.0/24*) و شبکه LAN راه دور من (*192.168.100.0/24*)
- **نقطه پایانی**: این IP Address میزبان Debian 11 است، زیرا این نقطه اتصال WireGuard ما است (شما باید IP عمومی Address را مشخص کنید)



در نهایت، یک نام در قسمت "**Name**" وارد کنید (بدون فاصله) و کلید عمومی مشتری را کپی و جای‌گذاری کنید، زیرا باید آن را در سرور اعلام کنیم. روی "**Register**" کلیک کنید.



### C. اعلام مشتری در سرور WireGuard



وقت آن است که به سرور دبیان بازگردیم تا "**Peer**"، یعنی رایانه ویندوزی خود را در پیکربندی WireGuard اعلام کنیم. ابتدا باید **Interface "wg0" را متوقف کنیم** تا بتوانیم پیکربندی آن را تغییر دهیم:



```
sudo wg-quick down wg0
# ou
sudo wg-quick down /etc/wireguard/wg0.conf
```



سپس، فایل پیکربندی ایجاد شده قبلی را تغییر دهید:



```
sudo nano /etc/wireguard/wg0.conf
```



در این فایل، پس از بلوک `[Interface]`، باید یک بلوک `[Peer]` اعلام کنیم:



```
[Peer]
PublicKey = 0i2Pg8nwDW2j7yOG09ZXht18o8l8Erb9Y5F4xyAyQV8=
AllowedIPs = 192.168.110.2/32
```



این بلوک [Peer] شامل کلید عمومی رایانه ویندوز 10 (**PublicKey**) و IP Address از Interface رایانه (**AllowedIPs**) است: سرور تنها برای ارتباط با کلاینت ویندوز در این تونل WireGuard ارتباط برقرار خواهد کرد، بنابراین مقدار "**192.168.110.2/32**".



تنها کاری که باقی مانده ذخیره فایل است (*CTRL+O سپس Enter سپس CTRL+X از طریق Nano*). Interface "wg0" را دوباره راه‌اندازی کنید:



```
sudo wg-quick up wg0
# ou
sudo wg-quick up /etc/wireguard/wg0.conf
```



برای بررسی اینکه اعلامیه همتا کار می‌کند، می‌توانید از این دستور استفاده کنید:



```
sudo wg show
```



هنگامی که میزبان راه دور اتصال WireGuard خود را تنظیم کرد، IP آن Address به مقدار "endpoint" منتقل خواهد شد.



![Image](assets/fr/033.webp)



در نهایت، فایل‌های پیکربندی را ایمن می‌کنیم تا دسترسی ریشه را محدود کنیم:



```
sudo chmod 600 /etc/wireguard/ -R
```



### D. اولین اتصال WireGuard



اکنون که پیکربندی آماده است، می‌توانیم آن را از رایانه ویندوزی آغاز کنیم. برای انجام این کار، در کلاینت "**WireGuard**"، روی دکمه "**Activate**" کلیک کنید: اتصال از "**Off" به "On "** تغییر خواهد کرد، اما این به معنای عملکرد صحیح آن نیست. همه چیز بستگی به این دارد که آیا پیکربندی شما درست است یا خیر. **وقتی اتصال برقرار شد، دو دستگاه ما از طریق Interface WireGuard که در هر طرف پیکربندی شده است، ارتباط برقرار می‌کنند!**



![Image](assets/fr/030.webp)



در صورت بروز مشکل، این مورد در زبانه "**Logbook**" قابل مشاهده خواهد بود. دو میزبان به طور منظم بسته‌های Exchange را برای بررسی وضعیت اتصال ارسال می‌کنند، از این رو پیام‌های "*Receiving keepalive packet from peer 1*" وجود دارد.



![Image](assets/fr/031.webp)



اگر زبانه "**ژورنال**" در WireGuard پیامی مانند پیام زیر را نمایش می‌دهد، باید بررسی کنید که کلیدهای عمومی اعلام‌شده در هر دو طرف صحیح هستند.



```
Handshake for peer 1 (<ip>:51820) did not complete after 5 seconds, retrying (try 2)
```



از رایانه شخصی از راه دورم، می‌توانم IP Address از Interface WireGuard خود را در سمت سرور پینگ کنم، همچنین یک میزبان در شبکه محلی از راه دورم.



![Image](assets/fr/032.webp)



### E. عملکرد: OpenVPN در مقابل WireGuard



از رایانه شخصی از راه دورم که به VPN WireGuard متصل است، توانستم به یک سرور فایل دسترسی پیدا کنم و یک فایل را از طریق [SMB](https://www.it-connect.fr/le-protocole-smb-pour-les-debutants/) منتقل کنم تا نرخ انتقال را ببینم. **با WireGuard، حداکثر سرعت من حدود 45 مگابیت بر ثانیه است، که عالی است، زیرا من از WiFi استفاده می‌کنم.**



![Image](assets/fr/025.webp)



تحت شرایط مشابه، اما این بار از طریق یک اتصال OpenVPN (در UDP)، با همان عملیات، توان عملیاتی کاملاً متفاوت است: حدود 3 مگابایت بر ثانیه. تفاوت واضح است!



![Image](assets/fr/026.webp)



این جالب است، زیرا اگر، به عنوان مثال، از یک اتصال WiFi به یک اتصال 4G/5G تغییر دهید، اتصال مجدد آنقدر سریع خواهد بود که وقفه قابل مشاهده نخواهد بود.



### اف. پاداش: تونل کامل WireGuard



با پیکربندی فعلی، بخشی از ترافیک از طریق VPN عبور می‌کند و بقیه از طریق اتصال اینترنت مشتری، از جمله مرور اینترنت. اگر بخواهیم به **حالت تونل کامل** WireGuard سوئیچ کنیم تا همه چیز از طریق تونل VPN عبور کند، باید چند تغییر در پیکربندی خود ایجاد کنیم....



ابتدا باید بسته "resolvconf" را بر روی: نصب کنید.



```
sudo apt-get update
sudo apt-get install resolvconf
```



پس از انجام این کار، باید پروفایل "wg0.conf" را روی ماشین دبیان تغییر دهید: Interface را متوقف کنید، آن را تغییر دهید و مجدداً راه‌اندازی کنید.



```
sudo wg-quick down /etc/wireguard/wg0.conf
```



سپس، **در بلوک `[Interface]`، سرور DNS که باید استفاده شود را اضافه می‌کنیم**. در مورد من، این کنترل‌کننده دامنه آزمایشگاه من است، اما می‌توانیم Bind9 را روی ماشین دبیان نصب کنیم تا یک حل‌کننده محلی داشته باشیم.



```
DNS = 192.168.100.11
```



فایل را ذخیره کنید، سپس Interface را مجدداً راه‌اندازی کنید:



```
sudo wg-quick up /etc/wireguard/wg0.conf
```



در نهایت، در پیکربندی تونل بر روی ورک‌استیشن ویندوز 10، باید بخش "AllowedIPs" را تغییر دهید تا نشان دهید که همه چیز باید از طریق تونل عبور کند. جایگزین کنید:



```
AllowedIPs = 192.168.110.0/24, 192.168.100.0/24
```



توسط:



```
AllowedIPs = 0.0.0.0/0
```



شما می‌توانید ببینید که این همچنین گزینه "**Kill switch**" را فعال می‌کند.



![Image](assets/fr/040.webp)



در نهایت، از این تونل کامل استفاده کردم تا یک آزمایش جریان کوچک انجام دهم که نتایج آن در زیر نشان داده شده است:



![Image](assets/fr/039.webp)



پیکربندی WireGuard بسیار ساده و قابل فهم است و مهم‌تر از همه، نگهداری آن آسان است. **WireGuard به عنوان آینده VPN‌ها در نظر گرفته می‌شود**، بنابراین بهتر است توجه ویژه‌ای به آن داشته باشیم! همچنین می‌توانیم ببینیم که مزیت قابل توجهی از نظر عملکرد وجود دارد، که این یک مزیت بزرگ برای WireGuard در مقایسه با OpenVPN است.
