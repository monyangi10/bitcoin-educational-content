## I. Презентация



В этом руководстве мы узнаем, как настроить VPN на основе WireGuard, бесплатного VPN-решения с открытым исходным кодом, которое повышает производительность, не пренебрегая безопасностью.



WireGuard - относительно недавнее решение, стабильный релиз которого доступен с марта 2020 года, и имеет честь быть интегрированным непосредственно в ядро **Linux начиная с версии 5.6**. Это не мешает ему быть доступным из старых дистрибутивов Linux, использующих другую версию ядра. По сравнению с такими решениями, как OpenVPN и IPSec, WireGuard проще в использовании и намного быстрее, как мы увидим в конце этой статьи.



Некоторые ключевые моменты о WireGuard:





- Простой, легкий и сверхэффективный!
- Работа только по протоколу UDP (что может быть недостатком при прохождении некоторых брандмауэров)
- Работает по одноранговой, а не клиент-серверной модели
- Аутентификация по ключу Exchange, по тому же принципу, что и SSH с приватными/публичными ключами
- Использование нескольких алгоритмов: симметричное шифрование с помощью ChaCha20, аутентификация сообщений с помощью Poly1305, а также других, таких как Curve25519, BLAKE2 и SipHash24
- Поддержка протоколов IPv4 и IPv6
- Мультиплатформенность: Windows, Linux, BSD, macOS, Android, iOS, OpenWRT (и реализовано в ProtonVPN)
- Всего 4 000 строк кода по сравнению с сотнями тысяч у других решений



Что касается криптографической части, то различные используемые алгоритмы отбираются вручную, проверяются несколькими способами и изучаются исследователями безопасности, специализирующимися в этой области.



Официальный сайт проекта: [wireguard.com](https://www.wireguard.com/)



Убедились ли вы в правильности такого решения после прочтения этого вступления? Остается только читать дальше!



## II. Лабораторная схема WireGuard



Прежде чем я представлю свою лабораторную работу по настройке WireGuard, вы должны знать, что вы можете представить себе **использование WireGuard для соединения двух удаленных инфраструктур**, а также для **подключения удаленного клиента к инфраструктуре, такой как корпоративная сеть или ваша домашняя сеть**. Это происходит в том же духе, что и с OpenVPN, как мы недавно видели в руководстве "[OpenVPN на Synology](https://www.it-connect.fr/tuto-vpn-configurer-openvpn-server-sur-un-nas-synology/)".



Если WireGuard не настроен непосредственно на маршрутизаторе или брандмауэре, как это возможно в OpenWRT, вам нужно будет настроить проброс портов, чтобы поток достигал пары WireGuard.



![Image](assets/fr/034.webp)



Теперь я расскажу вам о своей лаборатории и о том, что мы сегодня будем делать.



Я собираюсь использовать машину Debian 11 в качестве сервера WireGuard, а клиент Windows - в качестве VPN-клиента WireGuard. Хотя говорить о клиент-серверных отношениях немного неправильно, потому что **WireGuard работает по модели "peer-to-peer "**. Это немного неверное название, когда вы пытаетесь установить соединение "клиент-сайт". Предположим, что у меня будет две пары или **две точки подключения WireGuard**, если хотите: одна под Debian 11, а другая под Windows.



Эти две пары могут взаимодействовать друг с другом в обоих направлениях, то есть с моей Windows-машины я могу получить доступ к удалённой локальной сети машины Debian 11, и наоборот: всё зависит от конфигурации туннеля и брандмауэра WireGuard peer.



В этом примере я рассмотрю следующий случай: **с моего Windows Peer 1, подключенного к домашней сети, я хочу получить доступ к корпоративной сети, чтобы получить доступ к серверам компании через VPN-туннель WireGuard**. Это дает следующую схему:



![Image](assets/fr/035.webp)



В пересчете на IP-адреса это дает:





- Домашняя сеть**: 192.168.1.0/24
- Корпоративная сеть**: 192.168.100.0/24
- Туннельная сеть WireGuard**: 192.168.110.0/24


+ IP Address пира 1 (Windows) в туннеле: 192.168.110.2/24


+ IP Address Пира 2 (Debian) в туннеле: 192.168.110.121/24



Вот и все! Приступаем к настройке!



**Примечание: по умолчанию WireGuard работает в режиме UDP на **порту 51820**.



## III Установка и настройка сервера WireGuard



В этом руководстве я буду использовать термины "клиент" для машины Windows и "сервер" для машины Debian, потому что, хотя это одноранговая система, это кажется более понятным.



### A. Установка WireGuard на Debian 11



Установочный пакет WireGuard доступен в официальных репозиториях Debian 11, поэтому всё, что нам нужно сделать, это обновить кэш пакетов и установить его:



```
sudo apt-get update
```



```
sudo apt-get install wireguard
```



![Image](assets/fr/022.webp)



Будет установлена серверная часть WireGuard, а также различные инструменты, предоставляющие доступ к полезным командам для управления конфигурацией.



### B. Установка устройства Interface WireGuard



С помощью **команды "wg "** нам нужно получить generate закрытый и открытый ключи: это необходимо для аутентификации двух пар, то есть сервера и клиента (которому также понадобится пара ключей).



Мы создадим закрытый ключ "**/etc/wireguard/wg-private.key**" и открытый ключ "**/etc/wireguard/wg-public.key**" с помощью такой последовательности команд:



```
wg genkey | sudo tee /etc/wireguard/wg-private.key | wg pubkey | sudo tee /etc/wireguard/wg-public.key
```



![Image](assets/fr/023.webp)



Значение открытого ключа будет возвращено в консоли. В файле конфигурации WireGuard нам нужно **добавить значение нашего закрытого ключа**. Чтобы получить это значение, введите команду ниже и скопируйте значение:



```
sudo cat /etc/wireguard/wg-private.key
```



Пришло время создать конфигурационный файл в папке "**/etc/wireguard/**". Например, мы можем назвать этот файл "**wg0.conf**", если считаем, что сеть Interface, связанная с WireGuard, будет "wg0", по тому же принципу, что и "eth0", например.



```
sudo nano /etc/wireguard/wg0.conf
```



В этот файл сначала нужно добавить следующее содержимое (мы вернемся к его заполнению позже):



```
[Interface]
Address = 192.168.110.121/24
SaveConfig = true
ListenPort = 51820
PrivateKey = <clé privée du serveur>
```



Секция `[Interface]` используется для объявления серверной части. Вот некоторая информация:





- Address**: IP Address устройства Interface WireGuard в VPN-туннеле (отличная подсеть от удаленной локальной сети)
- SaveConfig**: конфигурация сохраняется (и защищается) до тех пор, пока Interface активен
- ListenPort**: Порт прослушивания WireGuard. В данном случае по умолчанию используется порт 51820, но вы можете настроить его по своему усмотрению
- PrivateKey**: значение закрытого ключа нашего сервера (*wg-private.key*)



Сохраните файл и закройте его. С помощью команды "**wg-quick**" мы можем запустить этот Interface, указав его имя (wg0, так как файл называется wg0.conf):



```
sudo wg-quick up wg0
```



Если вы перечислите IP-адреса вашего сервера Debian 11, вы увидите новый Interface с именем "wg0" и IP Address, определённым в файле конфигурации:



```
ip a
```



![Image](assets/fr/027.webp)



В том же духе мы можем отобразить конфигурацию Interface "wg0" с помощью команды "wg show":



```
sudo wg show wg0
```



![Image](assets/fr/024.webp)



Наконец, нам нужно активировать автоматический запуск нашего Interface wg0 WireGuard:



```
sudo systemctl enable wg-quick@wg0.service
```



На данный момент мы оставим в стороне настройку серверной части WireGuard.



### C. Включить переадресацию IP-адресов



Чтобы наша машина Debian 11 могла **направлять пакеты между различными сетями (как маршрутизатор)**, то есть между сетью VPN и локальной сетью, нам нужно включить [IP Forwarding](https://www.it-connect.fr/activer-lip-forwarding-sous-linux-ipv4ipv6/). По умолчанию эта функция отключена.



Измените этот файл конфигурации:



```
sudo nano /etc/sysctl.conf
```



Добавьте следующую директиву в конец файла и сохраните:



```
net.ipv4.ip_forward = 1
```



Вот и все.



### D. Включить IP-маскарад



Чтобы наш сервер правильно маршрутизировал пакеты и удаленная локальная сеть была доступна для машины Windows, нам нужно активировать IP Masquerade на нашем сервере Debian. Это своего рода активация NAT. Я собираюсь выполнить эту настройку на брандмауэре Linux через UFW, который я привык использовать ([ufw tutorial on Debian](https://www.it-connect.fr/configurer-un-pare-feu-local-sous-debian-11-avec-ufw/)).



Если у вас еще нет UFW и вы хотите его установить (вы также можете использовать Nftables), начните с установки:



```
sudo apt install ufw
```



Прежде всего, необходимо включить SSH, чтобы не потерять контроль над удаленным сервером (адаптируйте номер порта):



```
sudo ufw allow 22/tcp
```



Порт 51820 в UDP также должен быть авторизован, так как мы используем его для WireGuard (опять же, адаптируйте номер порта):



```
sudo ufw allow 51820/udp
```



Далее мы продолжим настройку, чтобы включить IP-маскарад. Для этого нам нужно узнать имя Interface, подключенного к локальной сети. Если вы не знаете имени, выполните команду "ip a", чтобы узнать имя карты. В моем случае это карта "**ens192**".



![Image](assets/fr/036.webp)



Мы будем использовать эту информацию. Отредактируйте следующий файл:



```
sudo nano /etc/ufw/before.rules
```



Добавьте эти строки в конец файла, чтобы **включить IP-маскарад для Interface ens192** (адаптируйте имя Interface) в строке POSTROUTING таблицы NAT нашего локального брандмауэра:



```
# NAT - IP masquerade
*nat
:POSTROUTING ACCEPT [0:0]
-A POSTROUTING -o ens192 -j MASQUERADE

# End each table with the 'COMMIT' line or these rules won't be processed
COMMIT
```



На картинке изображено:



![Image](assets/fr/037.webp)



Сохраните этот файл конфигурации открытым и перейдите к следующему шагу. 😉



### E. Конфигурация брандмауэра Linux для WireGuard



Все в том же файле конфигурации мы объявим корпоративную сеть "192.168.100.0/24", чтобы мы могли с ней связаться. Вот два правила, которые нужно добавить (в идеале после секции "*# ok icmp code for FORWARD*", чтобы сгруппировать правила вместе):



```
# autoriser le forwarding pour le réseau distant de confiance (+ le réseau du VPN)
-A ufw-before-forward -s 192.168.100.0/24 -j ACCEPT
-A ufw-before-forward -d 192.168.100.0/24 -j ACCEPT
-A ufw-before-forward -s 192.168.110.0/24 -j ACCEPT
-A ufw-before-forward -d 192.168.110.0/24 -j ACCEPT
```



Если вы хотите авторизовать только один хост, например "192.168.100.11", это несложно:



```
-A ufw-before-forward -s 192.168.100.11/32 -j ACCEPT
-A ufw-before-forward -d 192.168.100.11/32 -j ACCEPT
```



Теперь вы можете сохранить файл и закрыть его. Осталось только активировать UFW и перезапустить службу, чтобы применить наши изменения:



```
sudo ufw enable
```



```
sudo systemctl restart ufw
```



Первая часть настройки сервера Debian завершена.



## IV. Клиент WireGuard для Windows



Клиент WireGuard доступен для Windows, macOS, Android и т.д.. Это отличная новость. Все загрузки доступны на этой странице: [WireGuard Client](https://www.wireguard.com/install/). В этом примере я собираюсь установить клиент на Windows. Для настройки клиента WireGuard на Linux следуйте тому же принципу, что и для создания файла wg0.conf на машине Debian (без всех NAT и т.д.).



### A. Установка клиента WireGuard для Windows



После загрузки исполняемого файла или MSI-пакета установка не составит труда: просто запустите инсталлятор и... вуаля, готово! 🙂 



![Image](assets/fr/038.webp)



### B. Создайте профиль WireGuard



Начните с открытия программы для создания нового туннеля. Для этого нажмите на стрелку справа от кнопки "**Добавить туннель**" и нажмите на кнопку "**Добавить пустой туннель**".



![Image](assets/fr/028.webp)



Откроется окно конфигурации. Каждый раз, когда создается новая конфигурация туннеля, WireGuard генерирует пару закрытый/открытый ключ, специфичную для этой конфигурации. **В этой конфигурации нам нужно объявить "peer", то есть удаленный сервер:



```
[Interface]
PrivateKey = <la clé privée du PC>
```



Нам нужно завершить эту конфигурацию, в частности, объявить IP Address на этом Interface (*Address*), а также объявить удаленный сервер WireGuard через блок [Peer]. Изображение ниже должно напомнить вам конфигурационный файл, который мы создали на стороне сервера Linux.



Начнем с блока `[Interface]`, добавив IP Address "**192.168.110.2**"; помните, что сервер имеет IP Address "**192.168.110.121**" в этом сегменте сети. Это дает:



```
[Interface]
PrivateKey = <la clé privée du PC>
Address = 192.168.110.2/24
```



Далее нам нужно объявить блок "Peer" с тремя свойствами, в результате чего мы получим такую конфигурацию:



```
[Peer]
PublicKey = 1D/Gf5yd3hUDoFyYQ3P1zayBHUJhJZq+k6Sv03HnJCQ=
AllowedIPs = 192.168.110.0/24, 192.168.100.0/24
Endpoint = <ip-serveur-debian>:51820
```



На фотографиях:



![Image](assets/fr/029.webp)



** Несколько пояснений о блоке [Peer]:





- PublicKey**: это открытый ключ сервера WireGuard Debian 11 (вы можете получить его значение с помощью команды "*sudo wg*")
- AllowedIPs**: это IP-адреса/подсети, доступные через данную сеть WireGuard VPN, в данном случае подсеть, предназначенная для моей WireGuard VPN (*192.168.110.0/24*) и моей удаленной локальной сети (*192.168.100.0/24*)
- Конечная точка**: это IP Address хоста Debian 11, так как это наша точка подключения WireGuard (вам нужно будет указать публичный IP Address)



Наконец, введите имя в поле "**Имя**" (без пробелов) и скопируйте и вставьте открытый ключ клиента, так как нам нужно будет объявить его на сервере. Нажмите на кнопку "**Регистрация**".



### C. Объявление клиента на сервере WireGuard



Пришло время вернуться на сервер Debian, чтобы объявить "**Peer**", т.е. наш ПК с Windows, в конфигурации WireGuard. Прежде всего, нам нужно **остановить Interface "wg0"**, чтобы изменить его конфигурацию:



```
sudo wg-quick down wg0
# ou
sudo wg-quick down /etc/wireguard/wg0.conf
```



Затем измените ранее созданный файл конфигурации:



```
sudo nano /etc/wireguard/wg0.conf
```



В этом файле после блока `[Interface]` нужно объявить блок `[Peer]`:



```
[Peer]
PublicKey = 0i2Pg8nwDW2j7yOG09ZXht18o8l8Erb9Y5F4xyAyQV8=
AllowedIPs = 192.168.110.2/32
```



Этот блок [Peer] содержит открытый ключ ПК с Windows 10 (**PublicKey**) и IP Address ПК с Interface (**AllowedIPs**): сервер будет взаимодействовать в этом туннеле WireGuard только для связи с клиентом Windows, отсюда значение "**192.168.110.2/32**".



Осталось только сохранить файл (*CTRL+O, затем Enter, затем CTRL+X через Nano*). Перезапустите Interface "wg0":



```
sudo wg-quick up wg0
# ou
sudo wg-quick up /etc/wireguard/wg0.conf
```



Чтобы проверить, работает ли декларация равных, можно воспользоваться этой командой:



```
sudo wg show
```



После того как удаленный хост установит соединение с WireGuard, его IP Address будет перемещен в значение "конечная точка".



![Image](assets/fr/033.webp)



Наконец, мы защитим конфигурационные файлы, чтобы ограничить доступ root:



```
sudo chmod 600 /etc/wireguard/ -R
```



### D. Первое подключение WireGuard



Теперь, когда конфигурация готова, мы можем запустить ее с компьютера под управлением Windows. Для этого в клиенте "**WireGuard**" нажмите на кнопку "**Activate**": соединение **изменится с "Off" на "On "**, но это не значит, что оно будет работать. Все зависит от того, правильна ли ваша конфигурация. **Когда соединение установлено, наши две машины обмениваются данными через Interface WireGuard, настроенный на каждой стороне!



![Image](assets/fr/030.webp)



В случае возникновения проблем это будет видно на вкладке "**Logbook**". Оба узла будут регулярно отправлять пакеты Exchange, чтобы проверить состояние соединения, отсюда и сообщения "*Получение пакета keepalive от peer 1*".



![Image](assets/fr/031.webp)



Если на вкладке "**Журнал**" WireGuard отображается сообщение, подобное приведенному ниже, вам необходимо проверить правильность открытых ключей, объявленных обеими сторонами.



```
Handshake for peer 1 (<ip>:51820) did not complete after 5 seconds, retrying (try 2)
```



С моего удаленного ПК я могу пинговать IP Address моего Interface WireGuard на стороне сервера, а также хост в моей удаленной локальной сети.



![Image](assets/fr/032.webp)



### E. Производительность: OpenVPN против WireGuard



С моего удаленного ПК, подключенного к WireGuard VPN, я смог получить доступ к файловому серверу и передать файл через [SMB](https://www.it-connect.fr/le-protocole-smb-pour-les-debutants/), чтобы посмотреть скорость передачи. **С WireGuard максимальная скорость составляет около 45 Мб/с, что очень хорошо, поскольку я пользуюсь WiFi



![Image](assets/fr/025.webp)



При тех же условиях, но через OpenVPN-соединение (в UDP), на этот раз с теми же операциями, пропускная способность совершенно другая: около 3 МБ/с. Разница очевидна!



![Image](assets/fr/026.webp)



Это интересно, потому что если, например, вы переключитесь с WiFi-соединения на 4G/5G, то переподключение будет настолько быстрым, что прерывание не будет заметно.



### F. Бонус: полный туннель WireGuard



При текущей конфигурации часть трафика проходит через VPN, а остальной - через интернет-соединение клиента, включая просмотр интернет-страниц. Если мы хотим переключиться на режим WireGuard **полный туннель**, чтобы все проходило через VPN-туннель, нам нужно внести несколько изменений в нашу конфигурацию.....



Во-первых, вам нужно установить пакет "resolvconf" на диск:



```
sudo apt-get update
sudo apt-get install resolvconf
```



После этого вам нужно изменить профиль "wg0.conf" на машине Debian: остановите Interface, измените его и перезапустите.



```
sudo wg-quick down /etc/wireguard/wg0.conf
```



Далее, **в блоке `[Interface]` мы добавим DNS-сервер, который будет использоваться**. В моём случае это контроллер домена моей лаборатории, но мы также можем установить Bind9 на машину Debian, чтобы иметь локальный резолвер.



```
DNS = 192.168.100.11
```



Сохраните файл, затем перезапустите Interface:



```
sudo wg-quick up /etc/wireguard/wg0.conf
```



Наконец, в конфигурации туннеля на рабочей станции Windows 10 нужно изменить раздел "AllowedIPs", чтобы указать, что все должно проходить через туннель. Заменить



```
AllowedIPs = 192.168.110.0/24, 192.168.100.0/24
```



По:



```
AllowedIPs = 0.0.0.0/0
```



Вы можете увидеть, что это также включает опцию "**Убить переключатель*".
