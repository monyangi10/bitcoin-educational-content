## I. 프레젠테이션



Windows 네트워크에서 연결된 컴퓨터를 빠르고 쉽게 스캔하려면 어떻게 해야 하나요? 정답은 Angry IP Scanner입니다. 이 오픈 소스 프로젝트를 사용하면 사용하기 쉬운 그래픽 Interface을 사용하여 네트워크를 쉽게 스캔할 수 있습니다.



이 도구는 개인이 **로컬 네트워크를 스캔**하는 데 사용할 수 있지만, IT 전문가도 같은 목적으로 사용할 수 있습니다. 이 도구가 매우 실용적이라는 증거는 이미 **일부 사이버 범죄 그룹**에서 기업 네트워크를 스캔하는 데 사용되었다는 것입니다(Nmap과 동일한 방식으로). 좋은 예가 [랜섬웨어 그룹 랜섬허브](https://www.it-connect.fr/deja-210-victimes-pour-le-groupe-de-ransomware-ransomhub-lance-en-fevrier-2024/)입니다. 여전히 건전한 소프트웨어이지만 다른 네트워크 및 보안 지향 도구와 마찬가지로 악용될 수 있습니다.



여기서는 **윈도우 11**에서 사용하지만, 다른 버전의 **윈도우**는 물론 **리눅스**와 **macOS**에서도 완벽하게 사용할 수 있습니다.



Nmap보다는 덜 포괄적인 **Angry IP** 스캐너는 빠르고 기본적인 네트워크 분석에 유용할 뿐만 아니라 누구나 쉽게 사용할 수 있다는 점에서 흥미롭습니다. 네트워크에 연결된 호스트를 **감지**하고 **호스트 이름**과 **열린 포트**를 식별합니다.



더 자세히 알아보려면 Nmap 튜토리얼을 참조하세요:



https://planb.network/tutorials/computer-security/communication/nmap-862300d7-6dfb-4660-970d-f56a9f58f60d

## II. Angry IP 스캐너 시작하기



### A. Angry IP 스캐너 다운로드 및 설치



애플리케이션의 공식 웹사이트 또는 GitHub에서 최신 버전의 Angry IP Scanner를 다운로드할 수 있습니다. 여기서는 후자의 옵션을 사용하겠습니다. 아래 링크를 클릭하고 EXE 버전을 다운로드하세요: "**ipscan-3.9.1-setup.exe**".





- [앵그리 IP 스캐너 깃허브](https://github.com/angryip/ipscan/releases/latest)



![Image](assets/fr/016.webp)



실행 파일을 실행하여 설치를 진행합니다. 설치하는 동안 특별히 해야 할 일은 없습니다.



![Image](assets/fr/013.webp)



### B. 초기 네트워크 검사 실행



처음 시작할 때 "**시작하기**" 창에 있는 지침을 읽어보고 도구의 작동 방식에 대해 자세히 알아보세요. 참고로 고려해야 할 몇 가지 용어가 있습니다:





- 피더**: 임의의 IP 범위 또는 IP 주소 목록이 포함된 파일에서 스캔할 IP 주소 목록을 생성하는 모듈입니다.
- 페처**: 네트워크의 호스트에 대한 정보를 검색하기 위한 모듈 세트입니다. 예를 들어, MAC 주소 검색, 포트 검색, 호스트 이름 검색 또는 HTTP 요청 전송을 위한 페처가 있습니다.



![Image](assets/fr/018.webp)



IP 서브넷을 스캔하려면 "**IP 범위**" 필드에 **시작 IP Address**과 **끝 IP Address**을 입력하세요(그렇지 않으면 오른쪽에서 유형을 변경하세요). 그런 다음 "**시작**" 버튼을 클릭합니다.



![Image](assets/fr/019.webp)



수십 초 후, 소프트웨어의 Interface에 결과가 표시됩니다. **분석된 범위의 각 IP Address에 대해 Angry IP Scanner가 호스트를 감지했는지 여부가 표시됩니다.** 활성 호스트 수(이 경우 6개)와 포트가 열려 있는 호스트 수를 나타내는 요약도 화면에 나타납니다.



![Image](assets/fr/020.webp)



여기에서 "**192.168.10.1**" IP Address와 연결되고 **포트 80** 및 **443**(HTTP 및 HTTPS)에서 액세스할 수 있는 "**OPNsense.local.domain**"이라는 호스트가 있는 것을 확인할 수 있습니다. 호스트를 마우스 오른쪽 버튼으로 클릭하면 이 IP Address를 통해 핑, 경로 추적, 브라우저 열기 등의 추가 명령에 액세스할 수 있습니다.



![Image](assets/fr/012.webp)



### C. 스캔 포트 추가



기본적으로 **Angry IP 스캐너**는 3개의 포트를 스캔합니다: **80**(HTTP), **443**(HTTPS), **8080**. 애플리케이션 환경설정에서 스캔할 포트를 더 추가할 수 있습니다. "**도구**" 메뉴를 클릭한 다음 "**포트**" 탭을 클릭합니다.



여기에서 "**포트 선택**" 옵션을 통해 포트 목록을 수정할 수 있습니다. 쉼표로 구분된 특정 포트 번호 또는 포트 범위를 **표시**할 수 있습니다. 아래 예에서는 두 개의 포트를 추가합니다: **445**(SMB) 및 **389**(LDAP). 1에서 1000까지의 포트를 스캔하려면 "**1-1000**"을 입력합니다. 포트 스캔을 TCP, UDP 또는 둘 다에서 수행할지 여부는 지정되지 않습니다.



![Image](assets/fr/021.webp)



스캔을 다시 실행하면 새로운 정보를 얻을 수 있습니다. 아래 예제에서, 검사할 포트의 새로운 구성 덕분에 "**SRV-ADDS-01**" 및 "**SRV-ADDS-02**" 호스트에서 포트 389와 445가 열려 있다고 Angry IP Scanner가 알려줍니다.



![Image](assets/fr/022.webp)



**참고**: '**스캐너**' 메뉴에서 스캔 결과를 텍스트 파일로 내보낼 수 있습니다.



스캔을 더 진행하려면 "**도구**" 메뉴를 클릭한 다음 "**페처**"를 클릭합니다. 그러면 스캔에 "기능"이 추가됩니다. 페처를 선택하고 왼쪽으로 이동해 활성화하기만 하면 됩니다. 이렇게 하면 스캔 결과에 열이 하나 더 추가됩니다.



![Image](assets/fr/014.webp)



아래 예는 "**NetBIOS 정보**" 및 "**웹 탐지**" 기능을 보여줍니다. 전자는 컴퓨터의 MAC Address 및 도메인 이름과 같은 추가 정보를 제공하고, 후자는 웹 페이지 제목(웹 서버 또는 애플리케이션의 유형을 알려줄 수 있음)을 표시합니다.



![Image](assets/fr/011.webp)



마지막으로, 환경설정에서 "**ping**"에 사용되는 방법, 즉 호스트가 활성 상태인지 여부를 고려하는 방법을 변경할 수도 있습니다. 일부 호스트는 핑에 응답하지 않으므로 다른 방법(UDP 패킷, TCP 포트 프로브, ARP, UDP + TCP 조합 등)을 시도해 볼 수 있습니다.



## III. 결론



간단하고 효과적입니다: Angry IP Scanner는 네트워크에 연결된 호스트를 감지하고 포트 스캔을 구성할 수 있습니다. 옵션 측면에서 Nmap만큼 유연하지는 않고 범위가 넓지는 않지만 빠른 스캔을 위한 좋은 시작점입니다.



그래픽 Interface에서 **Nmap**을 사용하려면 **젠맵 애플리케이션**을 사용하면 됩니다(아래 개요 참조).
