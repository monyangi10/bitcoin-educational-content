## I.プレゼンテーション



Windowsネットワークに接続されているマシンを素早く簡単にスキャンするには？答えはAngry IP Scannerです。このオープンソースプロジェクトでは、使いやすいグラフィカルなInterfaceを使って、簡単にネットワークをスキャンすることができます。



このツールは、個人が自分のローカルネットワーク**をスキャンするために使うことができるだけでなく、IT専門家が同じ目的で使うこともできる。このツールが非常に実用的**であることの証明として、**一部のサイバー犯罪グループ**が企業ネットワークをスキャンするために既に使用している(Nmapと同じ方法で)。その良い例が[ランサムウェアグループRansomHub](https://www.it-connect.fr/deja-210-victimes-pour-le-groupe-de-ransomware-ransomhub-lance-en-fevrier-2024/)だ。今でも健全なソフトウェアではあるが、他のネットワークやセキュリティ指向のツールと同様に、その使用は悪用される可能性がある。



ここでは**Windows 11**で使用するが、**Windows**の他のバージョンや、**Linux**や**macOS**で使用することも完全に可能である。



Nmap よりも包括的ではないが、**Angry IP** Scanner は素早く基本的なネットワーク分析を行うにはまだ面白い。ネットワークに接続されているホストを**検出**し、**ホスト名**と**開いているポート**を特定する。



さらに詳しく知りたい場合は、Nmapのチュートリアルを参照されたい：



https://planb.network/tutorials/computer-security/communication/nmap-862300d7-6dfb-4660-970d-f56a9f58f60d

## II.Angry IP Scannerを使い始める



### A.Angry IP Scannerをダウンロードしてインストールする



Angry IP Scannerの最新バージョンは、アプリケーションの公式ウェブサイトまたはGitHubからダウンロードできる。ここでは後者を使用します。以下のリンクをクリックし、EXEバージョンをダウンロードしてください："**ipscan-3.9.1-setup.exe**".





- [Angry IP Scanner GitHub](https://github.com/angryip/ipscan/releases/latest)



![Image](assets/fr/016.webp)



実行ファイルを実行してインストールを進めます。インストール中に行うことは特にありません。



![Image](assets/fr/013.webp)



### B.初期ネットワークスキャンの実行



初回起動時には、"**Getting Started**"ウィンドウの説明をじっくり読んで、ツールの動作について詳しく学んでください。ところで、考慮すべき用語がいくつかある：





- **Feeder**：ランダムなIP範囲またはIPアドレスのリストを含むファイルから、スキャンするIPアドレスのリストを生成するモジュール。
- **フェッチャー**：ネットワーク上のホストに関する情報を取得するためのモジュール群。例えば、MACアドレスの検出、ポートのスキャン、ホスト名の検出、HTTPリクエストの送信などのフェッチャーがある。



![Image](assets/fr/018.webp)



IPサブネットをスキャンするには、"**IP range**"フィールドに**開始IP Address**と**終了IP Address**を入力するだけです（そうでない場合は、右側のタイプを変更してください）。その後、"**Start**"ボタンをクリックしてください。



![Image](assets/fr/019.webp)



数十秒後、ソフトウェアのInterfaceに結果が表示されます。 **解析された範囲内の各IP Addressについて、Angry IP Scannerがホストを検出したかどうかが表示されます。**また、アクティブなホストの数（この場合は6）とポートが開いているホストの数を示すサマリーも画面に表示されます。



![Image](assets/fr/020.webp)



ここでは、IP Address "**192.168.10.1**"に関連付けられた "**OPNsense.local.domain**"という名前のホストが存在し、**ポート80**と**443**（HTTPとHTTPS）でアクセスできることがわかる。ホストを右クリックすると、このIP Address経由でping、トレースルート、ブラウザを開くなどの追加コマンドにアクセスできます。



![Image](assets/fr/012.webp)



### C.スキャンポートを追加する



デフォルトでは、**Angry IP Scanner**は3つのポートをスキャンします： **80**（HTTP）、**443**（HTTPS）、および**8080**です。アプリケーションの環境設定からスキャンするポートを追加できます。**ツール**メニューから**ポート**タブをクリックしてください。



ここでは、**ポート選択**オプションを使ってポートリストを変更することができます。コンマで区切られた特定のポート番号、またはポート範囲を指定できます。以下の例では、2つのポートを追加しています： **445**（SMB）と**389**（LDAP）。1～1000のポートをスキャンするには、**1-1000**と入力する。ポート・スキャンをTCP、UDP、またはその両方で実行するかどうかは指定しない。



![Image](assets/fr/021.webp)



再度スキャンを実行すると、新しい情報が得られる可能性が高い。以下の例では、スキャンするポートの新しい設定のおかげで、ポート389と445がホスト「**SRV-ADDS-01**」と「**SRV-ADDS-02**」で開いていることをAngry IP Scannerが教えてくれています。



![Image](assets/fr/022.webp)



**注**："**Scanner**"メニューから、スキャン結果をテキストファイルにエクスポートできます。



さらにスキャンを進めたい場合は、"**Tools**"メニューをクリックし、"**Fetchers**"をクリックしてください。これにより、スキャンに「機能」が追加されます。フェッチャーを選択し、左に移動させるだけで有効になります。これでスキャン結果に列が追加されます。



![Image](assets/fr/014.webp)



以下の例では、"**NetBIOS info**"と "**Web detection**"機能を示している。前者はマシンのMAC Addressやドメイン名などの追加情報を提供し、後者はウェブページのタイトルを表示する(ウェブサーバーやアプリケーションの種類を示す可能性がある)。



![Image](assets/fr/011.webp)



最後に、環境設定から「**ping**」に使用する方法、つまりホストがアクティブかどうかを考慮する方法を変更することもできます。Pingに応答しないホストもあるので、他の方法（UDPパケット、TCPポート・プローブ、ARP、UDPとTCPの組み合わせなど）を試すことができます。



## III.結論



シンプルで効果的：Angry IP Scannerはネットワークに接続されたホストを検出し、ポートスキャンを設定できる。オプションに関しては、Nmapほど柔軟ではないし、それほど遠くまでは行けないが、素早くスキャンするには良いスタートだ。
