## I. การนำเสนอ



คุณจะสแกนเครือข่าย Windows เพื่อค้นหาเครื่องที่เชื่อมต่อได้อย่างรวดเร็วและง่ายดายได้อย่างไร? คำตอบคือ Angry IP Scanner โครงการโอเพ่นซอร์สนี้ช่วยให้คุณสแกนเครือข่ายได้อย่างง่ายดาย โดยใช้กราฟิก Interface ที่ใช้งานง่าย



เครื่องมือนี้สามารถใช้โดยบุคคลทั่วไปเพื่อ **สแกนเครือข่ายท้องถิ่นของตน** แต่ยังสามารถใช้โดยผู้เชี่ยวชาญด้าน IT เพื่อวัตถุประสงค์เดียวกัน หลักฐานว่า **เครื่องมือนี้มีความเป็นประโยชน์อย่างมาก** คือมันได้ถูกใช้โดย **กลุ่มอาชญากรไซเบอร์บางกลุ่ม** เพื่อสแกนเครือข่ายองค์กร (ในลักษณะเดียวกับ Nmap) ตัวอย่างที่ดีคือ [กลุ่มแรนซัมแวร์ RansomHub](https://www.it-connect.fr/deja-210-victimes-pour-le-groupe-de-ransomware-ransomhub-lance-en-fevrier-2024/) มันยังคงเป็นซอฟต์แวร์ที่ดี แต่เช่นเดียวกับเครื่องมือที่เน้นเครือข่ายและความปลอดภัยอื่น ๆ การใช้งานของมันสามารถถูกนำไปใช้ในทางที่ผิดได้



ที่นี่ เราจะใช้มันบน **Windows 11** แต่ก็สามารถใช้ได้บนเวอร์ชันอื่นๆ ของ **Windows** เช่นเดียวกับบน **Linux** และ **macOS**



น้อยกว่าที่ครอบคลุมกว่า Nmap, **Angry IP** Scanner ยังคงน่าสนใจสำหรับการวิเคราะห์เครือข่ายพื้นฐานอย่างรวดเร็ว แต่ก็เพราะว่ามันอยู่ในมือของทุกคน มันจะ **ตรวจจับโฮสต์ที่เชื่อมต่อกับเครือข่าย** และระบุ **ชื่อโฮสต์** และ **พอร์ตที่เปิดอยู่**



หากคุณต้องการไปต่อ ดูบทแนะนำใน Nmap:

## II. เริ่มต้นใช้งาน Angry IP Scanner



### A. ดาวน์โหลดและติดตั้ง Angry IP Scanner



คุณสามารถดาวน์โหลดเวอร์ชันล่าสุดของ Angry IP Scanner ได้จากเว็บไซต์ทางการของแอปพลิเคชันหรือจาก GitHub เราจะใช้ตัวเลือกหลัง คลิกที่ลิงก์ด้านล่างและดาวน์โหลดเวอร์ชัน EXE: "**ipscan-3.9.1-setup.exe**".





- [Angry IP Scanner GitHub](https://github.com/angryip/ipscan/releases/latest)



![Image](assets/fr/016.webp)



เรียกใช้ไฟล์ปฏิบัติการเพื่อดำเนินการติดตั้งต่อไป ไม่มีสิ่งพิเศษที่ต้องทำระหว่างการติดตั้ง



![Image](assets/fr/013.webp)



### B. เรียกใช้การสแกนเครือข่ายเบื้องต้น



เมื่อเปิดใช้งานครั้งแรก ใช้เวลาอ่านคำแนะนำในหน้าต่าง "**เริ่มต้นใช้งาน**" เพื่อเรียนรู้เพิ่มเติมเกี่ยวกับวิธีการทำงานของเครื่องมือนี้ นอกจากนี้ยังมีหลายคำศัพท์ที่ควรพิจารณา:





- Feeder**: โมดูลที่รับผิดชอบในการสร้างรายการที่อยู่ IP ที่จะสแกน จากช่วง IP แบบสุ่มหรือไฟล์ที่มีรายการที่อยู่ IP
- Fetcher**: ชุดของโมดูลสำหรับดึงข้อมูลเกี่ยวกับโฮสต์บนเครือข่าย ตัวอย่างเช่น fetchers สำหรับตรวจจับ MAC addresses, สแกนพอร์ต, ตรวจจับชื่อโฮสต์ หรือส่งคำขอ HTTP



![Image](assets/fr/018.webp)



ในการสแกน IP subnet เพียงแค่ป้อน **ที่อยู่ IP เริ่มต้น** และ **ที่อยู่ IP สิ้นสุด** ในช่อง "**ช่วง IP**" (มิฉะนั้นให้เปลี่ยนประเภททางด้านขวา) จากนั้นคลิกที่ปุ่ม "**เริ่ม**"



![Image](assets/fr/019.webp)



ไม่กี่สิบวินาทีต่อมา ผลลัพธ์จะปรากฏในซอฟต์แวร์ Interface **สำหรับแต่ละที่อยู่ IP ในช่วงที่วิเคราะห์ คุณจะเห็นว่า Angry IP Scanner ตรวจพบโฮสต์หรือไม่** สรุปจะปรากฏบนหน้าจอด้วย โดยระบุจำนวนโฮสต์ที่ใช้งานอยู่ (ในกรณีนี้คือ 6) และจำนวนโฮสต์ที่มีพอร์ตเปิด



![Image](assets/fr/020.webp)



ที่นี่ เราสามารถเห็นการมีอยู่ของโฮสต์ชื่อ "**OPNsense.local.domain**" ซึ่งเชื่อมโยงกับที่อยู่ IP "**192.168.10.1**" และสามารถเข้าถึงได้บน **พอร์ต 80** และ **443** (HTTP และ HTTPS) การคลิกขวาที่โฮสต์จะให้การเข้าถึงคำสั่งเพิ่มเติม เช่น การ ping, trace route และการเปิดเบราว์เซอร์ผ่านที่อยู่ IP นี้



![Image](assets/fr/012.webp)



### ค. เพิ่มพอร์ตสแกน



ตามค่าเริ่มต้น **Angry IP Scanner** จะสแกน 3 พอร์ต: **80** (HTTP), **443** (HTTPS) และ **8080** คุณสามารถเพิ่มพอร์ตเพิ่มเติมเพื่อให้สแกนได้จากการตั้งค่าของแอปพลิเคชัน คลิกที่เมนู "**Tools**" จากนั้นคลิกที่แท็บ "**Ports**"



ที่นี่ คุณสามารถแก้ไขรายการพอร์ตผ่านตัวเลือก "**Port selection**" คุณสามารถ **ระบุหมายเลขพอร์ตเฉพาะที่คั่นด้วยเครื่องหมายจุลภาค หรือช่วงของพอร์ต** ตัวอย่างด้านล่างเพิ่มสองพอร์ต: **445** (SMB) และ **389** (LDAP) หากต้องการสแกนพอร์ตตั้งแต่ 1 ถึง 1000 ให้ป้อน "**1-1000**" ไม่ได้ระบุว่าการสแกนพอร์ตดำเนินการใน TCP, UDP หรือทั้งสองอย่าง



![Image](assets/fr/021.webp)



หากคุณทำการสแกนอีกครั้ง คุณอาจจะได้รับข้อมูลใหม่ ในตัวอย่างด้านล่าง Angry IP Scanner บอกฉันว่าพอร์ต 389 และ 445 เปิดอยู่บนโฮสต์ "**SRV-ADDS-01**" และ "**SRV-ADDS-02**" ขอบคุณการกำหนดค่าพอร์ตใหม่ที่จะถูกสแกน



![Image](assets/fr/022.webp)



**หมายเหตุ**: จากเมนู "**Scanner**" คุณสามารถส่งออกผลการสแกนไปยังไฟล์ข้อความได้



หากคุณต้องการดำเนินการสแกนต่อไป ให้คลิกที่เมนู "**Tools**" จากนั้นคลิกที่ "**Fetchers**" สิ่งนี้จะเพิ่ม "ความสามารถ" ให้กับการสแกน เพียงเลือก fetcher และย้ายไปทางซ้ายเพื่อเปิดใช้งาน สิ่งนี้จะเพิ่มคอลัมน์พิเศษให้กับผลลัพธ์การสแกน



![Image](assets/fr/014.webp)



ตัวอย่างด้านล่างแสดงฟังก์ชัน "**NetBIOS info**" และ "**Web detection**" ฟังก์ชันแรกให้ข้อมูลเพิ่มเติมเช่น MAC address ของเครื่องและชื่อโดเมน ในขณะที่ฟังก์ชันหลังแสดงชื่อหน้าเว็บ (ซึ่งอาจบ่งบอกถึงประเภทของเว็บเซิร์ฟเวอร์หรือแอปพลิเคชัน)



![Image](assets/fr/011.webp)



สุดท้ายนี้ จากการตั้งค่าคุณยังสามารถเปลี่ยนวิธีที่ใช้สำหรับ "**ping**" ได้อีกด้วย นั่นคือการพิจารณาว่าโฮสต์นั้นทำงานอยู่หรือไม่ เนื่องจากบางโฮสต์ไม่ตอบสนองต่อการ ping วิธีนี้จึงช่วยให้คุณลองใช้วิธีอื่นๆ ได้ (แพ็กเก็ต UDP, การตรวจสอบพอร์ต TCP, ARP, การผสมผสานระหว่าง UDP + TCP เป็นต้น)



## III. บทสรุป



เรียบง่ายและมีประสิทธิภาพ: Angry IP Scanner ตรวจจับโฮสต์ที่เชื่อมต่อกับเครือข่าย และให้คุณกำหนดการสแกนพอร์ตได้ ในแง่ของตัวเลือก มันไม่ยืดหยุ่นเท่า Nmap และไม่ครอบคลุมเท่า แต่เป็นการเริ่มต้นที่ดีสำหรับการสแกนอย่างรวดเร็ว



หากคุณต้องการใช้ **Nmap** กับ Interface แบบกราฟิก คุณสามารถใช้ **แอปพลิเคชัน Zenmap** (ดูภาพรวมด้านล่าง)
