## Tox คืออะไร?



Tox เป็นโปรโตคอลการสื่อสารแบบกระจายศูนย์ที่เปิดให้ใช้งานฟรี (โอเพ่นซอร์ส) ซึ่งใช้เทคโนโลยี *Networking and Cryptography Library* (NaCl) รวมกับการเข้ารหัสแบบต่างๆ เพื่อรับประกันความปลอดภัยและความลับของผู้ใช้ Tox ช่วยให้คุณแลกเปลี่ยนข้อความ ส่งเสียงและวิดีโอคอล แชร์ไฟล์ และแชร์หน้าจอกับเพื่อนๆ ได้อย่างปลอดภัย แบบกระจายศูนย์ และไม่มีตัวกลาง



เทคโนโลยีที่โปรโตคอล Tox ใช้นั้นคล้ายกับเครือข่ายแบบเพียร์ทูเพียร์ เช่น บล็อกเชน ซึ่งสนับสนุนการกระจายอำนาจของโครงสร้างพื้นฐานของโปรโตคอล แตกต่างจากเครือข่ายสังคมและแอปพลิเคชันส่งข้อความที่เข้ารหัสแบบปลายทางถึงปลายทาง แอปพลิเคชัน Tox Chat ถูกสร้างขึ้นบนโปรโตคอลแบบกระจายอำนาจที่ไม่มีเซิร์ฟเวอร์กลาง ผู้ใช้ทุกคนสื่อสารกันในเครือข่ายแบบเพียร์ทูเพียร์ที่ไม่มีตัวกลางและทนต่อการเซ็นเซอร์ ในการสื่อสาร ผู้ใช้แต่ละคนจะถูกระบุด้วย ID ที่ไม่ซ้ำกัน (ToxID) ซึ่งได้มาจากคีย์สาธารณะของเขาหรือเธอ ซึ่งถูกเก็บไว้ในตาราง Hash ที่กระจายอยู่



## เข้าร่วม Tox



คุณสามารถใช้โปรโตคอล Tox ผ่านไคลเอนต์ส่งข้อความโต้ตอบแบบทันทีที่คุณสามารถดาวน์โหลดได้จาก [Tox Chat site](https://tox.chat)



![download](assets/fr/01.webp)



ขึ้นอยู่กับระบบปฏิบัติการของคุณ คุณสามารถดาวน์โหลดและติดตั้งไคลเอนต์ Tox ที่ตรงกับ:





- aTox: [aTox](https://github.com/evilcorpltd/aTox), ไคลเอนต์ Tox ที่เขียนด้วย Kotlin ใช้งานได้เฉพาะบน Android



![aTox](assets/fr/02.webp)





- qTox: ไคลเอนต์ Tox จาก [open source](https://github.com/TokTok/qTox) ที่พัฒนาบน Qt Framework (C++) ใช้งานได้บน Windows, Linux, MacOs.



![qTox](assets/fr/03.webp)





- Toxic: [Toxic](https://github.com/JFreegman/toxic) เป็นไคลเอนต์ Tox ที่เขียนด้วยภาษา C และทำงานบนระบบที่มีอินเทอร์เฟซบรรทัดคำสั่ง



![Toxic](assets/fr/04.webp)



ในบทแนะนำนี้ เราจะใช้ qTox clients บน Windows และ aTox เพื่อสื่อสารกัน



## เริ่มต้นกับ qTox



เมื่อติดตั้งเสร็จแล้ว ให้ติดตั้งไคลเอนต์ Tox ของคุณและสร้างโปรไฟล์ของคุณ



![qTox-acount](assets/fr/05.webp)



ยินดีด้วย คุณเพิ่งเข้าร่วมโปรโตคอล Tox บนซอฟต์แวร์ qTox คุณสามารถค้นหาข้อมูลโปรไฟล์ของคุณได้โดยคลิกที่ชื่อผู้ใช้ของคุณ



![profil](assets/fr/06.webp)



โดยเฉพาะอย่างยิ่ง คุณจะพบ ID Tox ของคุณ ซึ่งคุณสามารถบันทึกเป็น QR code และแชร์กับผู้ที่ต้องการติดต่อกับคุณ



ส่งออกไฟล์โปรไฟล์ Tox ของคุณเพื่อให้คุณมีข้อมูลสำรองของโปรไฟล์และข้อมูลติดต่อที่คุณสามารถใช้ในการกู้คืน คลิกที่ปุ่ม **ส่งออก** จากนั้นเลือกเส้นทางไปยังไฟล์สำรองของคุณ



![export](assets/fr/07.webp)



จากเมนู **เพิ่มเติม** เพิ่มเพื่อน นำเข้ารายชื่อผู้ติดต่อ และจัดการคำขอเป็นเพื่อนที่คุณได้รับ

### กลุ่มเพียร์ทูเพียร์



ลูกค้า Tox ของคุณยังช่วยให้คุณสามารถสื่อสารกับกลุ่มคนในรูปแบบที่กระจายศูนย์อย่างสมบูรณ์: สิ่งเหล่านี้เรียกว่าการประชุม ในเมนู **กลุ่ม** สร้างการประชุมใหม่หรือปรึกษารายชื่อคำเชิญเข้าร่วมการประชุมที่คุณได้รับ



![conferences](assets/fr/11.webp)



เมื่อสร้างการประชุมแล้ว คุณสามารถเชิญเพื่อนของคุณเข้าร่วมการประชุมบนไคลเอนต์ Tox ของคุณได้ ในรายการเพื่อนของคุณ ให้คลิกขวาที่ชื่อผู้ใช้ของเพื่อนที่คุณต้องการเชิญ คลิกที่ตัวเลือก **เชิญเข้าร่วมการประชุม** จากนั้นเลือกชื่อการประชุมที่คุณได้สร้างขึ้น คุณยังสามารถเชิญเพื่อนโดยการสร้างการประชุมใหม่โดยใช้ตัวเลือก **สร้างการประชุมใหม่** ได้อีกด้วย



⚠️ ลูกค้า Tox ยังคงอยู่ระหว่างการพัฒนา ดังนั้นคุณอาจพบข้อผิดพลาดเมื่อใช้งานซอฟต์แวร์ ฟังก์ชันการประชุมและการโทรผ่านวิดีโอยังไม่พร้อมใช้งานบนลูกค้า Tox Android (aTox)

### การถ่ายโอนไฟล์



ในเมนู **การถ่ายโอนไฟล์** คุณจะพบประวัติของไฟล์ที่คุณได้ส่งและไฟล์ที่คุณได้รับจากผู้ติดต่อของคุณ



![files](assets/fr/13.webp)



คุณยังสามารถปรับแต่งการกำหนดค่าการแชร์ไฟล์ของคุณสำหรับการสนทนาแต่ละครั้งได้อีกด้วย คลิกขวาที่ชื่อผู้ใช้ของผู้รับของคุณแล้วเลือก "แสดงรายละเอียดเพิ่มเติม"



![details](assets/fr/14.webp)



จากรายละเอียด Interface คุณสามารถจัดการการอนุญาตที่คุณให้แก่ผู้รับของคุณสำหรับ:





- การยอมรับคำเชิญประชุมโดยอัตโนมัติ





- การยอมรับไฟล์อัตโนมัติ





- เส้นทางสำรองสำหรับไฟล์การสนทนาของคุณ

### อะไรที่รับประกันความลับบน Tox?



โปรโตคอล Tox ใช้ตาราง Hash แบบกระจายเพื่อสร้างเครือข่ายของโหนดแบบกระจายศูนย์ ลูกค้า Tox แต่ละรายเป็นส่วนหนึ่งของเครือข่าย DHT และจัดเก็บข้อมูลเกี่ยวกับโหนดอื่น ๆ ในกรณีของ Tox, DHT จะจัดเก็บที่อยู่ IP เป็นค่าที่เชื่อมโยงกับคีย์สาธารณะ Tox (Tox ID) ซึ่งทำให้ง่ายต่อการค้นหาอุปกรณ์ลูกค้า Tox โดยไม่ต้องสอบถามเซิร์ฟเวอร์กลาง



ลองจินตนาการถึงการเขียนถึงผู้ใช้ของเรา `EBC5604D9386E594CCC32943A03F96A96687FBD46788F1CD4F3337EB703C16BE3DC2CA6D9F` ซึ่งเราจะเรียกว่า **ผู้ใช้ B** ไคลเอนต์ Tox ของคุณจะค้นหาตัวระบุนี้ในตารางกระจาย Hash และดึงที่อยู่ IP ที่เกี่ยวข้อง เมื่อพบที่อยู่ IP แล้ว ไคลเอนต์ Tox ของคุณจะสร้างช่องทางการสื่อสารที่เข้ารหัสโดยตรงกับเครื่องของ **ผู้ใช้ B** หรือใช้โหนดอื่นเป็นตัวกลางเพื่อเข้าถึง **ผู้ใช้ B** อัลกอริธึมการเข้ารหัสจะรับประกันว่า ไม่ว่าจะเป็นช่องทางการสื่อสารใด มีเพียง **ผู้ใช้ B** เท่านั้นที่จะสามารถอ่านเนื้อหาของข้อความของคุณได้



หากคุณสนุกกับการค้นพบ Tox และสามารถเข้าใจได้ว่ามันมีประโยชน์อย่างไรในการเสริมสร้างความเป็นส่วนตัวของคุณ โปรดอย่าลังเลที่จะให้คะแนนบทแนะนำนี้ด้วยการกดถูกใจ เรายังแนะนำบทแนะนำของเราเกี่ยวกับ Simple Login ซึ่งเป็นเครื่องมือที่ช่วยให้คุณรับและส่งอีเมลโดยไม่เปิดเผยตัวตน
