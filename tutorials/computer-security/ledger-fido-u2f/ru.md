Устройства Ledger - это аппаратные кошельки, изначально предназначенные для защиты Bitcoin Wallet, но при этом обладающие расширенными возможностями для строгой аутентификации в Интернете. Благодаря совместимости с протоколами **U2F** и **FIDO2** они позволяют защитить доступ к вашим онлайн-аккаунтам, установив второй фактор аутентификации.



Протокол U2F (Universal 2nd Factor) был представлен компаниями Google и Yubico в 2014 году, а затем стандартизирован FIDO Alliance. Он позволяет добавлять второй физический фактор аутентификации (2FA) при входе в систему. После активации, в дополнение к классическому паролю, пользователь должен одобрить каждую попытку подключения к своему аккаунту, нажав кнопку на Ledger. В этом контексте Ledger работает примерно так же, как Yubikey. U2F фактически является подкомпонентом стандарта FIDO2, охватывая его и внося значительные улучшения, включая встроенную поддержку современных браузеров и большую гибкость в управлении ключами аутентификации.



Эти методы основаны на асимметричной криптографии: секретные данные не передаются, что делает фишинговые атаки и атаки перехвата неэффективными. U2F и FIDO2 поддерживаются многими онлайн-сервисами.



В этом руководстве мы покажем вам, как активировать U2F и FIDO2 для двухфакторной аутентификации на вашем Ledger.



**Примечание:** U2F и FIDO2 поддерживаются на всех устройствах Ledger с последней версией прошивки: начиная с версии 2.1.0 для Nano X и Nano S classic, и начиная с версии 1.1.0 для Nano S Plus. Модели Stax и Flex совместимы нативно.



## Установите приложение Ledger Security Key



Если вы пользуетесь устройством Ledger, то наверняка знаете, что одна только прошивка не содержит всех функций, необходимых для управления криптокошельками. Например, чтобы использовать Bitcoin Wallet, вам нужно установить приложение "*Bitcoin*". Аналогично, для управления ключами MFA необходимо установить приложение "*Security Key*".



Прежде чем начать, убедитесь, что вы настроили Bitcoin Wallet на Ledger. Важно правильно сохранить Mnemonic, поскольку ключи, используемые для 2FA, создаются на основе этой фразы Mnemonic. Если ваш Ledger будет утерян или поврежден, вы сможете восстановить доступ к ключам, введя фразу Mnemonic на другом устройстве Ledger (на данный момент идентификаторы FIDO2 в режиме "*без пароля*" пока не поддерживаются на Ledgers, поэтому резидентных идентификаторов нет).



Подключите Ledger к компьютеру и разблокируйте его.



![Image](assets/fr/01.webp)



Чтобы установить приложение, откройте программу [Ledger Live] (https://www.Ledger.com/Ledger-live), затем перейдите на вкладку "*Мой Ledger*". Найдите приложение "*Ключ безопасности*" и установите его на свое устройство.



![Image](assets/fr/02.webp)



Теперь приложение "*Ключ безопасности*" должно появиться рядом с другими приложениями, установленными на вашем Ledger.



![Image](assets/fr/03.webp)



Щелкните по приложению, чтобы оставить его открытым для следующих шагов учебника.



![Image](assets/fr/04.webp)



## Использование U2F/FIDO2 для 2FA с помощью Ledger



Войдите в учетную запись, которую вы хотите защитить с помощью двухфакторной аутентификации. Например, я собираюсь использовать учетную запись Bitwarden. Обычно опцию 2FA можно найти в настройках сервиса, на вкладках "*аутентификация*", "*безопасность*", "*логин*" или "*пароль*".



![Image](assets/fr/05.webp)



В разделе, посвященном двухфакторной аутентификации, выберите опцию "*Passkey*" (термин может отличаться в зависимости от используемого сайта).



![Image](assets/fr/06.webp)



Часто вам будет предложено подтвердить текущий пароль.



![Image](assets/fr/07.webp)



Дайте своему ключу безопасности имя для удобства распознавания, затем нажмите "*Прочитать ключ*".



![Image](assets/fr/08.webp)



На дисплее Ledger появятся данные вашей учетной записи. Для подтверждения нажмите кнопку "*Регистрация*" (или обе кнопки одновременно, в зависимости от используемой модели).



![Image](assets/fr/09.webp)



Ключ доступа успешно зарегистрирован.



![Image](assets/fr/10.webp)



Зарегистрируйте этот ключ безопасности.



![Image](assets/fr/11.webp)



Теперь при входе в аккаунт, помимо обычного пароля, вам будет предложено подключить Ledger.



![Image](assets/fr/12.webp)



Затем вы можете нажать кнопку "*Log in*" на дисплее Ledger, чтобы подтвердить авторизацию (или обе кнопки одновременно, в зависимости от модели, которую вы используете).



![Image](assets/fr/13.webp)



Преимущество использования Hardware Wallet Ledger для двухфакторной аутентификации заключается в том, что вы можете легко восстановить свои ключи благодаря фразе Mnemonic. В дополнение к этому основному резервному копированию вы также можете использовать аварийный код, предоставляемый каждым сервисом, в котором вы активировали 2FA. Этот аварийный код позволяет подключиться к аккаунту, если вы потеряли ключ безопасности. Таким образом, он заменяет 2FA для подключения в случае необходимости.



На Bitwarden, например, вы можете получить этот код, нажав на "*Просмотреть код восстановления*".



![Image](assets/fr/14.webp)



Я рекомендую хранить этот код не там, где хранится ваш основной пароль, чтобы предотвратить их совместную кражу. Например, если ваш пароль хранится в менеджере паролей, храните аварийный код 2FA на бумаге, отдельно.



Такой подход обеспечивает два уровня резервного копирования в случае потери Ledger для аутентификации 2FA: первое резервное копирование с использованием фразы Mnemonic для всех ваших учетных записей и второе резервное копирование для конкретных учетных записей с использованием аварийных кодов. Однако важно **не путать роль Mnemonic с ролью аварийного кода**:




- Фраза Mnemonic из 12 или 24 слов дает вам доступ не только к ключам, используемым для 2FA на всех ваших аккаунтах, но и к вашим биткоинам, защищенным с помощью Ledger;
- Экстренный код позволяет временно обойти запрос 2FA только на соответствующем аккаунте (в данном примере - только на Bitwarden).
