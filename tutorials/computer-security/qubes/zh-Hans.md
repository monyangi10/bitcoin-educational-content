Qubes OS 是一款免费的开源操作系统，专为将安全放在首位的用户设计。它的特殊性基于一个简单而激进的想法：它不把计算机看作一个整体，而是把计算机的使用划分成一个个独立的部分，称为**_qubes_**。



每个 qube 都是一个**隔离的虚拟环境，具有特定的信任级别和功能。因此，即使应用程序受到攻击，攻击也仅限于其 qube，而不会影响系统的其他部分。



> 如果你对安全问题非常重视，Qubes OS 就是当今最好的操作系统。 - **爱德华-斯诺登**。

不过，安装 Qubes OS 需要比安装传统操作系统更多的准备工作。这需要检查某些硬件前提条件，了解虚拟化的基本原理，并接受一种不同的体验，在这种体验中，每项日常任务都要从分离的角度来考虑。但一旦安装完毕，Qubes OS 将提供一个强大的框架，重新定义你与电脑的日常交互方式。在本教程中，我们将介绍 Qubes OS 的工作原理以及如何在系统中轻松安装。



## Qubes OS 如何工作？



Qubes OS 基于分隔原则。它不使用单一系统，而是依靠**Xen**管理程序来创建隔离的虚拟机，称为 qubes。每个 qube 专用于特定任务或信任级别（工作、个人浏览、银行业务等）。这种分离确保了一个 qube 中的任何漏洞都不会扩散到其他 qube 中，就像一台机器中的几台独立计算机一样。



用户 Interface 由一个名为**dom0**的中央安全域管理。该域与互联网完全隔离，是系统的核心。虽然窗口和菜单是由 dom0 显示的，但应用程序的实际执行是在各自的 qubes 中进行的。



## 不同类型的 "qubes



在 dom0 的周围，围绕着不同类型的 qubes，每种 qubes 都发挥着非常特殊的作用。





- **AppVM** 用于运行日常应用程序：因此，用户可以将其专业电子邮件与网络浏览或银行活动分开，每个环境都完全独立于其他环境。





- 这些 AppVM 本身基于**TemplateVM**，作为安装和更新软件的集中模板，无需单独管理每个 qube。



Qubes OS 还集成了专门用于**系统服务**的虚拟机。





- **NetVM** 直接管理**网络设备**，并确保与互联网的连接。它们通常与**防火墙虚拟机**相关联，后者进行干预，以**过滤流量**并限制其他qubes的暴露。





- 服务虚拟机也扮演着类似的角色，例如在 USB 设备管理中，其逻辑始终如一：隔离风险最大的组件，减少攻击面。



最后，Qubes OS还提供**可弃虚拟机**，用于偶尔执行的任务或有风险的任务。在**打开可疑文件**或**访问可疑网站**时，会立即创建这些短暂的虚拟机，然后在关闭时完全消失，清除所有痕迹，防止任何持续性攻击。



### 安全复制机制（qrexec）



qubes之间的交流基于**qrexec**，这是一个专为安全设计的虚拟机间通信系统。qrexec 不允许 qubes 自由交流，而是实施**特定策略**：从一个 AppVM 复制文件到另一个 AppVM，或通过剪贴板传输文本，总是要经过系统监控和验证的通道。这样，即使是简单的复制和粘贴行为也会受到控制，以防止恶意软件的传播。



### 磁盘管理



Qubes OS 采用了一套巧妙的存储管理系统。TemplateVM 包含通用软件基础，而 AppVM 只添加个人数据和特定文件。这样既减少了磁盘空间的使用，又方便了全局更新。另一方面，一次性虚拟机使用的是临时卷，关闭后会完全消失。这种模式不仅能保证更高的安全性，还能有效管理资源。



## 为什么选择 Qubes OS？



Qubes OS 的优势首先在于其独特的安全模式。这种方法的核心是分隔，通过将每个任务隔离在不同的虚拟机中来保护用户。在实际应用中，受感染的电子邮件或恶意网站只能危及单个 Qubes，而系统的其他部分和个人数据则会受到全面保护。这种架构大大限制了复杂的攻击，而在传统系统中，这些攻击可以自由传播。



Qubes OS还提供完全透明的数字环境控制。无论是网络、USB 设备还是其他敏感组件，你都能清楚地知道哪个软件可以访问哪个资源。该系统默认集成了先进的安全功能，如全盘加密，并方便使用 Whonix 操作系统等匿名服务。



https://planb.network/tutorials/computer-security/operating-system/whonix-06f9172c-2962-412e-9487-b665d8ca9f59

Qubes OS 并不追求创建一个坚不可摧的系统，而是注重恢复能力：它封装了系统受损时的破坏，降低了系统其他部分所面临的风险。这种务实的方法使 Qubes OS 成为有高度安全需求或希望最大限度控制数字生活的用户的首选。



## 安装 Qubes 操作系统



在安装 Qubes OS 之前，必须确保您的硬件满足最低要求，因为系统依赖虚拟化来隔离 Qubes。需要检查的主要组件有 ：




- 处理器**：与硬件虚拟化（英特尔 VT-x 或 AMD-V）兼容的 64 位处理器。
- 内存：至少需要 8 GB，但我们建议使用 16 GB 或更大的内存来同时运行多个 qubes。
- 存储空间**：至少 36 GB，最好是 128 GB 的固态硬盘，以获得最佳性能。



要安装 Qubes OS，请从 Qubes OS [官方网站](https://www.qubes-os.org/downloads/) 下载官方 ISO 映像。必须使用提供的 GPG 签名检查 ISO 的完整性，以确保文件未被篡改，并确保下载安全。



https://planb.network/tutorials/computer-security/data/integrity-authenticity-21d0420a-be02-4663-94a3-8d487f23becc

![0_01](assets/fr/01.webp)



ISO 经过验证后，您需要创建一个可启动的安装介质，通常是 U 盘。为此，请下载并安装软件，如 Windows 上的 **Rufus** 或 Windows、Linux 或 macOS 上的 **Etcher**。通过这些工具，您可以将 ISO 复制到 U 盘，使其可以启动。



开始安装前，有必要配置计算机的**BIOS 或 UEFI**，以**启用虚拟化**并允许从 USB 启动。根据计算机型号，可能需要**禁用安全启动**，因为 Qubes OS 在启用该选项后可能无法启动。



![0_02](assets/fr/02.webp)



满足这些条件后，重新启动计算机，并立即按**Esc**、**Del**、**F9**、**F10**、**F11**或**F12**（视制造商而定）进入 BIOS/UEFI。在启动菜单中选择 USB 密钥作为启动设备，以启动 Qubes OS 安装。



**启动屏幕**


从 U 盘启动时，将直接进入 **GRUB**菜单，选择要执行的操作。使用键盘上的方向键，选择 "安装 Qubes OS"，然后按 "Enter "键。



![0_03](assets/fr/03.webp)



**语言选择** ：



安装开始后，第一步是**选择适合您配置的语言**和**地区变量**。这将确保系统和安装选项以您首选的语言正确显示。



![0_04](assets/fr/04.webp)



**参数配置** ：



在此阶段，您需要在机器上启动安装之前配置一些 Elements。



![0_05](assets/fr/05.webp)



**键盘布局** ：



点击**键盘**选项，然后选择适合你电脑的**布局**。做出选择后，点击**终止**进入下一步。



**目的地选择** ：



选择 "安装目的地 "选项，选择要安装 Qubes OS 的磁盘。默认情况下，分区会自动进行，允许你删除磁盘上的所有数据并在上面安装系统。不过，你也可以选择**自定义**或**高级自定义**来执行自定义分区。然后点击 "完成"。系统会要求你设置一个**密码**，作为加密数据的安全Layer。请务必选择一个复杂且唯一的密码。



![0_06](assets/fr/06.webp)



![0_07](assets/fr/07.webp)



**选择日期和时间格式** ：


点击时间和日期选项，然后选择您所在的地理区域。您还可以选择自己喜欢的时间格式：24 小时或 12 小时。



![0_08](assets/fr/08.webp)



**创建用户账户** ：


点击**创建用户**，创建您的**个账户**，这样您就可以登录 Qubes 操作系统了。



![0_09](assets/fr/09.webp)



**激活 root 帐户** ：


您还可以**启用 root 帐户**，为管理设置一个单独的密码。



![0_10](assets/fr/10.webp)



设置完成后，点击**开始安装**开始安装过程。



![0_11](assets/fr/11.webp)



等待**安装结束**，然后点击**重新启动系统**完成安装并启动 Qubes OS。



![0_12](assets/fr/12.webp)



**附加配置** ：


重启后，Qubes OS 会提示您最终确定**默认模板和 qubes 配置**。输入为加密磁盘而定义的密码。



![0_13](assets/fr/13.webp)



接下来，首先选择你要安装的**TemplateVM**。常见的选项包括**Debian 12 Xfce**、**Fedora 41 Xfce**和**Whonix 17**，后者推荐用于需要**匿名性和网络安全**的用途。您还可以定义**默认模板**，作为创建新**应用虚拟机**的基础。



![0_14](assets/fr/14.webp)



在 "主要配置 "部分，您可以选择自动创建基本系统配置，如**系统网络**、**系统防火墙**和**默认一次性虚拟机**。建议启用**系统防火墙和系统 USB 一次性**选项，以便在发生入侵时限制设备和网络风险。您还可以创建默认**应用程序虚拟机**，如**个人**、**工作**、**不信任**和**保险库**，以便根据信任级别组织您的活动。



![0_15](assets/fr/15.webp)



Qubes OS 还允许你创建一个 USB 设备专用的**qube**（sys-usb），并配置**Whonix 网关和工作站 qubes**，以确保通过 Tor 网络通信的安全。对于高级用户，**高级配置**部分可让你创建一个**LVM瘦池**，以有效管理qubes之间的磁盘空间。



配置完所有这些选项后，点击**完成**，然后点击**完成配置**。等待系统应用这些设置。然后，系统会提示您**登录用户账户**，您的 Qubes OS 环境就可以使用了，它安全且适当地隔离了您的各种活动。



![0_17](assets/fr/17.webp)



操作系统现已安装完毕，随时可以使用。



![0_18](assets/fr/18.webp)



## 在系统中创建一个 qube



要创建一个 qube，可通过 "开始 "菜单中的 "**Qube 管理器**"工具管理该过程。在工具窗口中，只需点击**创建qube**图标，即可打开一个新的配置窗口。首先，输入 qube 的名称，如 "perso-web "或 "work"，以确定其用途。



然后，选择其**类型**，通常**AppVM**用于日常任务，**DisposableVM**用于临时使用。至关重要的是，要选择 Qube 所基于的**模板**，如 "fedora-39 "或 "debian-12"，因为这将提供操作系统和软件。您还将指定**NetVM**，它是负责互联网访问的 Qube，默认为**系统防火墙**。



最后，在根据需要调整存储大小和内存后，只需点击**确定**即可启动创建程序。创建过程完成后，新的 Qube 将出现在列表中，随时可供使用。



总之，Qubes OS 不是普通的操作系统，而是重新思考个人电脑架构的尖端安全解决方案。它通过虚拟化实现分隔和隔离，为抵御最复杂的威胁提供了无与伦比的保护。通过将工作环境划分为专门的任务区，它可以确保恶意软件无法扩散并危及整个系统。
