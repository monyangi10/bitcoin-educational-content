Qubes OS 是一套免費的開放原始碼作業系統，專為將安全性放在首位的使用者所設計。它的特點是基於一個簡單但激進的想法：它並非將電腦視為一個整體，而是將電腦的使用分割成獨立的區塊，稱為 **_qubes_**。



每個 qube 都是**獨立的虛擬環境**，具有特定的信任等級和功能。因此，即使應用程式受到攻擊，攻擊仍會局限於其 qube，而不會影響系統的其他部分。



> 如果您重視安全性，Qubes OS 是當今最好的作業系統。 - **Edward Snowden**。

然而，安裝 Qubes OS 需要比安裝傳統作業系統更多的準備工作。這包括檢查某些硬體先決條件、瞭解虛擬化的基本知識，以及接受不同的體驗，在這種體驗中，每項日常工作都是以分離的方式來思考。但是一旦安裝完成，Qubes OS 提供了一個強大的架構，重新定義您每天與電腦互動的方式。在本教程中，我們將解釋 Qubes OS 如何運作，以及如何輕鬆將其安裝到您的系統上。



## Qubes OS 如何運作？



Qubes OS 以區隔原則為基礎。它不使用單一系統，而是依賴 **Xen** Hypervisor 來建立獨立的虛擬機器，稱為 qubes。每個 qube 專用於特定任務或信任等級 (工作、個人瀏覽、銀行業務等)。這種分隔可確保一個 qube 中的任何損害不會擴散到其他 qube，就像單一機器中的多台獨立電腦一樣。



使用者 Interface 由一個稱為 **dom0** 的中央安全網域管理。這個網域與網際網路完全隔離，是系統的核心。雖然視窗和選單是由 dom0 顯示，但應用程式的實際執行是在各自的 qubes 中進行。



## 不同類型的 qubes



圍繞著 dom0 轉動的是不同類型的 qubes，每種都扮演著非常特殊的角色。





- **AppVM** 用於執行日常應用程式：使用者因此可以將專業電子郵件與網頁瀏覽或銀行活動分開，而每個環境仍完全獨立於其他環境。





- 這些 AppVM 本身是基於 **TemplateVM**，作為安裝和更新軟體的集中範本，省去了分開管理每個 qube 的需要。



Qubes OS 也整合了專門用於**系統服務**的虛擬機。





- **NetVM** 直接管理 ** 網路裝置**，並確保連線至網際網路。它們通常與**FirewallVM**相關，FirewallVM**會介入以**過濾流量**並限制其他 qubes 的曝露。





- ServiceVM 也扮演類似的角色，例如在 USB 裝置管理方面，總是有相同的邏輯：隔離最具風險的元件以減少攻擊面。



最後，針對偶爾或有風險的任務，Qubes OS 提供 **DisposableVM** 。這些短暫的 Qubes 會在**開啟可疑文件**或**瀏覽可疑網站**時立即建立，然後在關閉時完全消失，清除所有痕跡，防止任何持續性攻擊。



### 安全複製機制 (qrexec)



Qubes 之間的交換是基於 **qrexec**，一個專為安全設計的 VM 間通訊系統。qrexec 並非讓 qubes 自由通訊，而是強制 ** 特定的政策**：從一個 AppVM 複製檔案到另一個 AppVM，或透過剪貼板傳輸文字，總是會經過系統監控與驗證的通道。如此一來，即使是簡單的複製和貼上行為也受到控制，以防止惡意軟體的散佈。



### 磁碟管理



Qubes OS 使用巧妙的系統進行儲存管理。TemplateVMs 包含共用的軟體基礎，而 AppVMs 只加入個人資料和特定檔案。這可減少磁碟空間的使用，並方便進行全局更新。另一方面，DisposableVMs 使用臨時磁碟區，關閉時會完全消失。這種模式不僅能保證更高的安全性，還能有效管理資源。



## 為何選擇 Qubes OS？



Qubes OS 的優勢首先在於其獨特的安全模型。這種方法的核心是分隔，將每項任務隔離在不同的虛擬機器中，以保護使用者。實際上，受感染的電子郵件或惡意網站只能攻擊單一的 Qubes，而系統的其他部分和您的個人資料則完全受到保護。此架構大大限制了複雜的攻擊，而在傳統系統上，這些攻擊可以自由傳播。



Qubes OS 也提供完全透明的數位環境控制。無論是網路、USB 裝置或其他敏感元件，您都可以清楚知道哪些軟體可以存取哪些資源。系統預設整合了先進的安全功能，例如全磁碟加密，並方便使用 Whonix 作業系統等匿名服務。



https://planb.network/tutorials/computer-security/operating-system/whonix-06f9172c-2962-412e-9487-b665d8ca9f59

Qubes OS 的重點不在於創造一個堅不可摧的系統，而是提高系統的復原能力：當系統受到攻擊時，Qubes OS 會將損害封包起來，以降低系統其他部分所承受的風險。這種務實的方法讓 Qubes OS 成為具有高度安全性需求，或希望對數位生活保留最大控制權的使用者的首選。



## 安裝 Qubes OS



在安裝 Qubes OS 之前，必須確保您的硬體符合最低要求，因為系統依賴虛擬化來隔離 Qubes。需要檢查的主要元件有 ：




- ** 處理器**：與硬體虛擬化 (Intel VT-x 或 AMD-V) 相容的 64 位元處理器。
- 記憶體：至少需要 8 GB，但我們建議使用 16 GB 或更大的記憶體，以便同時執行多個 qubes。
- 儲存空間**：至少 36 GB，最好是 SSD 上的 128 GB，以獲得最佳效能。



要安裝 Qubes OS，請從 Qubes OS [官方網站](https://www.qubes-os.org/downloads/) 下載官方 ISO 映像檔。請務必使用所提供的 GPG 簽署檢查 ISO 的完整性，以確保檔案未被竄改，且下載是安全的。



https://planb.network/tutorials/computer-security/data/integrity-authenticity-21d0420a-be02-4663-94a3-8d487f23becc

![0_01](assets/fr/01.webp)



ISO 經驗證後，您需要建立可開機安裝媒體，通常是 USB 隨身碟。為此，請下載並安裝軟體，例如 Windows 上的 **Rufus** 或 Windows、Linux 或 macOS 上的 **Etcher**。這些工具可讓您將 ISO 複製到 USB 隨身碟，使其可開機。



在開始安裝之前，必須設定您電腦的 **BIOS 或 UEFI** 以 ** 啟用虛擬化** 並允許從 USB 開機。根據您的機器型號，可能需要**停用 Secure Boot**，因為 Qubes OS 在啟用此選項後可能無法開機。



![0_02](assets/fr/02.webp)



滿足這些條件後，請重新啟動電腦，並立即按下 **Esc**、**Del**、**F9**、**F10**、**F11** 或 **F12** (視製造商而定) 存取 BIOS/UEFI。在開機選單中，選擇 USB 鑰匙作為開機裝置，以啟動 Qubes OS 安裝。



**Start-up screen**


從 USB 隨身碟開機時，您會直接進入**GRUB**選單，您可以選擇要執行的動作。使用鍵盤上的方向鍵，選擇「Install Qubes OS」，然後按下「Enter」。



![0_03](assets/fr/03.webp)



** 選擇語言** ：



當安裝開始時，第一步是**選擇適合您設定的語言**和**區域變體**。這可確保系統和安裝選項以您偏好的語言正確顯示。



![0_04](assets/fr/04.webp)



** 參數配置** ：



在此階段，您需要先設定一些 Elements，然後才會在您的機器上啟動安裝。



![0_05](assets/fr/05.webp)



** 鍵盤配置** ：



按一下 ** 鍵盤** 選項，然後針對您的電腦選擇 ** 適當的配置**。做出選擇後，按一下 **終止**，進入下一步。



** 選擇目的地** ：



選擇「安裝目的地」選項，以選擇您希望安裝 Qubes OS 的磁碟。預設情況下，磁碟分割會自動進行，讓您移除磁碟上的所有資料，並在磁碟上安裝系統。不過，您可以選擇 ** 自訂** 或 ** 進階自訂** 來執行自訂的磁碟分割。然後按一下「完成」。系統會要求您設定**密碼**，作為加密資料的安全 Layer。請務必選擇複雜且唯一的密碼。



![0_06](assets/fr/06.webp)



![0_07](assets/fr/07.webp)



**選擇日期和時間格式** ：


按一下時間和日期選項，然後選擇您的地理區域。您也可以選擇偏好的時間格式：24 小時或 12 小時。



![0_08](assets/fr/08.webp)



**User account creation** ：


點選 **Create user** 建立您的**第一個帳號**，這將使您能夠登入 Qubes OS。



![0_09](assets/fr/09.webp)



**Activate root account** ：


您也可以 ** 啟用 root 帳戶**，方法是設定獨立的管理密碼。



![0_10](assets/fr/10.webp)



完成這些設定後，按一下 **Start installation** 開始安裝程序。



![0_11](assets/fr/11.webp)



等待 ** 安裝結束**，然後按一下 ** 重新啟動系統**，以完成安裝並啟動 Qubes OS。



![0_12](assets/fr/12.webp)



**Additional configuration** ：


重新開機後，Qubes OS 會提示您確定**預設範本和 qubes 配置**。輸入為加密磁碟所定義的密碼。



![0_13](assets/fr/13.webp)



接下來，先選擇您想要安裝的 **TemplateVM** 。常見的選項包括**Debian 12 Xfce**、**Fedora 41 Xfce**和**Whonix 17**，後者建議用於需要**匿名性和網路安全性**的用途。您也可以定義 ** 預設範本**，作為建立新 **AppVM 的基礎。



![0_14](assets/fr/14.webp)



在**主要設定**部分，您可以選擇自動建立重要的系統配備，例如**sys-net**、**sys-firewall**和**預設 DisposableVM**。建議您啟用選項，讓 ** 系統防火牆和 sys-usb 成為一次性**，以限制裝置和網路在受到入侵時的風險。您也可以建立預設的 **AppVMs**，例如 **個人**、**工作**、**不信任**和 **儲存庫**，以便根據信任等級組織您的活動。



![0_15](assets/fr/15.webp)



Qubes OS 還可讓您建立專用於 USB 裝置的 **qube** (sys-usb)，並配置 **Whonix Gateway 與 Workstation qubes**，以確保透過 Tor 網路進行通訊的安全性。對於進階使用者，**進階設定**部分可讓您建立**LVM瘦池**，以有效管理 qubes 間的磁碟空間。



完成所有這些選項的設定後，請按一下 **完成**，然後按一下 **完成設定**。等待系統套用這些設定。然後，系統會提示您**登入您的使用者帳號**，您的 Qubes OS 環境就可以使用了，安全並妥善隔離您的各種活動。



![0_17](assets/fr/17.webp)



您的作業系統現已安裝完成，可隨時使用。



![0_18](assets/fr/18.webp)



## 在您的系統上建立一個 qube



若要建立一個 qube，該程序由 **Qube Manager** 工具管理，可從開始功能表存取。在工具視窗中，只需按一下 **Create qube** 圖示，即可開啟新的組態視窗。首先，輸入 qube 的名稱，例如「perso-web」或「work」，以識別其用途。



接下來，您要選擇它的**類型**，通常**AppVM**用於例行任務，或**DisposableVM**用於臨時使用。選擇您的 qube 所基礎的 **Template** 是很重要的，例如「fedora-39」或「debian-12」，因為這將會提供作業系統和軟體。您也要指定 **NetVM**，這是負責網際網路存取的 qube，預設為 **sys-firewall**。



最後，在調整儲存空間大小和記憶體（如有必要）後，按一下 ** 確定** 即可啟動建立程序。程序完成後，您的新 qube 將會出現在清單中，隨時可供使用。



總而言之，Qubes OS 並非一般的作業系統，而是重新思考個人電腦架構的尖端安全解決方案。它的方法是透過虛擬化進行分割和隔離，提供無與倫比的保護，抵禦最複雜的威脅。藉由將工作環境分割為專門處理每項工作的專用區，可確保惡意軟體無法擴散並危及整個系統。



無論您是需要安全上網、保護敏感資訊或在不同信任層級下工作，Qubes OS 都能提供彈性、透明的架構。只要採用良好的作法並充分利用其功能，您就能擁有一個適應現代威脅的**數位堡壘。進一步了解如何保護您的資料和隱私。
