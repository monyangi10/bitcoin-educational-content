В този урок ще научим как да стартираме приложения в **Pears** - технология за равнопоставеност между потребители (P2P), разработена от **Holepunch** и поддържана от **Tether**. Целта е проста: да се направи възможно разпространението и използването на уеб приложения, без да се разчита на каквато и да е централизирана инфраструктура (без сървъри, без хостове, без посредници). С други думи, дори ако доставчикът на облачни услуги спре работа или дадена държава блокира даден домейн, приложението продължава да съществува сред равностойни потребители в мрежата.



## 1. Какво представлява Pears?



Pears е среда за изпълнение, инструмент за разработка и платформа за внедряване на peer-to-peer приложения. Този инструмент с отворен код дава възможност за създаване, споделяне и стартиране на софтуер без сървър или инфраструктура, директно между потребителите. Конкретно това означава, че вместо да хоства приложението на централен сървър, всеки потребител се превръща в мрежов възел, който споделя част от приложението и данните с други равнопоставени потребители. Цялата система образува разпределена мрежа, като всеки екземпляр си сътрудничи, за да поддържа услугата достъпна.



![Image](assets/fr/01.webp)



Този подход се основава на набор от модулни софтуерни тухлички, разработени от Holepunch:




- Hypercore**: разпределен дневник, който гарантира последователност и сигурност на данните без централна база данни.
- Hyperbee**: индексиращ модул върху Hypercore за ефективно организиране на данни и сърфиране.
- Hyperdrive**: разпределена файлова система, използвана за съхраняване и синхронизиране на файлове на приложения между равнопоставени потребители.
- Hyperswarm** и **HyperDHT**: мрежови слоеве, които позволяват откриване и свързване между равнопоставени потребители по целия свят без централен сървър.
- Secretstream**: протокол за криптиране E2E за защита на обмена между двама равнопоставени потребители.



Чрез комбинирането на тези компоненти Pears дава възможност за създаване на автономни, криптирани и разпределени приложения, в които всеки потребител участва активно в мрежата. Тази децентрализирана архитектура елиминира инфраструктурните разходи, рисковете от цензура и SPOF (*Single Point of Failure*).



## 2. Произход и философия на проекта



Pears се разработва от Holepunch, компания, основана от Матиас Буус и Паоло Ардоино (главен изпълнителен директор на Tether и главен технически директор на Bitfinex), със задачата да разшири логиката на peer-to-peer отвъд Bitcoin. Тяхната амбиция е да изградят "Peer-to-Peer интернет", където всяко приложение може да работи без оторизация, без сървъри и без посредници. Holepunch вече стои зад **Keet**, изцяло P2P приложение за видеоконференции и съобщения.



*Това ръководство за инсталиране на Pears е разделено на няколко раздела в зависимост от операционната ви система. Преминете директно към раздела, съответстващ на вашата среда, за да следвате съответните инструкции :*




- Linux (Debian)** → Част **3**
- Windows** → Част **4**
- macOS** → Част **5**



## 3. Как да инсталирате Pears под Linux (Debian)



Инсталирането на Pears на система Debian е сравнително лесно, но изисква няколко предварителни условия, които ще опишем подробно в този раздел.



### 3.1. актуализиране на системата



Преди всичко е важно да се уверите, че системата ви е актуализирана.



```bash
sudo apt update && sudo apt upgrade -y
```



![Image](assets/fr/02.webp)



### 3.2. инсталиране на зависимости



Pears разчита на определени системни библиотеки, а именно `libatomic1`, използвани от изпълнителното време на Bare JavaScript. Инсталирайте я със следната команда:



```bash
sudo apt install -y libatomic1 curl git
```



![Image](assets/fr/03.webp)



### 3.3. инсталирайте Node.js и npm чрез NVM



Pears се разпространява чрез *npm*, мениджъра на пакети *Node.js*. Въпреки че Pears не зависи пряко от *Node.js*, за да функционира, той е необходим за инсталацията. Препоръчителният метод за инсталиране на *Node.js* в Linux е *NVM* (*Node Version Manager*), който ви позволява да управлявате няколко версии на Node паралелно.



```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/master/install.sh | bash
```



![Image](assets/fr/04.webp)



След това презаредете терминала, за да активирате *NVM* :



```bash
source ~/.bashrc
```



![Image](assets/fr/05.webp)



Проверете дали е инсталиран *NVM*:



```bash
nvm --version
```



![Image](assets/fr/06.webp)



След това инсталирайте стабилна версия на *Node.js* (напр. текущата LTS):



```bash
nvm install --lts
```



![Image](assets/fr/07.webp)



Проверете инсталациите на *Node.js* и *npm*:



```bash
node -v
npm -v
```



![Image](assets/fr/08.webp)



### 3.4 Инсталиране на Pears с npm



След като *npm* е наличен, можете да инсталирате Pears CLI глобално на вашата система. Това ще ви позволи да изпълнявате командата `pear` от всяка директория.



```bash
npm install -g pear
```



![Image](assets/fr/09.webp)



### 3.5. Иницииране на Pears



След инсталацията просто изпълнете следната команда в терминала:



```bash
pear
```



При първото стартиране Pears се свързва с равноправната мрежа, за да изтегли необходимите компоненти. Този процес не изисква централен сървър: файловете се получават директно от други равноправни потребители.



![Image](assets/fr/10.webp)



След като изтеглянето приключи, изпълнете командата отново, за да проверите дали всичко работи:



```bash
pear
```



![Image](assets/fr/11.webp)



Ако всичко е инсталирано правилно, ще се появи Pears Help със списък на наличните команди.



### 3.6. Изпитване на Pears с Keet



За да проверите дали Pears работи пълноценно, можете да стартирате приложение P2P, което вече е налично в мрежата, като например Keet - софтуер за съобщения и видеоконференции с отворен код на Holepunch.



```bash
pear run pear://keet
```



Тази команда зарежда приложението Keet директно от мрежата на Pears, без да преминава през централен сървър. Ако Keet се стартира правилно, вашата инсталация на Pears е напълно функционална.



![Image](assets/fr/12.webp)



Вашата Linux система вече е готова да стартира и хоства peer-to-peer приложения с Pears.



## 4. Как да инсталирате Pears в Windows



Инсталирането на Pears в Windows е също толкова лесно, колкото и в Linux, но изисква няколко специални инструмента.



*Ако използвате Linux, можете да прескочите до стъпка 6.*



### 4.1. отворете PowerShell в администраторски режим



Първо, стартирайте PowerShell с администраторски права :




- Кликнете върху менюто Start;
- Тип PowerShell ;
- Кликнете с десния бутон на мишката върху "*Windows PowerShell*" ;
- Изберете "*Задействай като администратор*".



![Image](assets/fr/15.webp)



### 4.2. изтеглете NVS



Pears се инсталира чрез *npm*, мениджъра на пакети *Node.js*. В Windows методът, препоръчан от Holepunch, е да се използва *NVS* (*Node Version Switcher*), който е по-стабилен от *NVM* в тази система.



В PowerShell изпълнете следната команда, за да инсталирате най-новата версия на *NVS* :



```PowerShell
winget install jasongin.nvs
```



![Image](assets/fr/16.webp)



### 4.3. инсталирайте Node.js



След инсталацията рестартирайте PowerShell и въведете следната команда:



```powershell
nvs
```



Трябва да видите списък с наличните версии на *Node.js*. Изберете първата от тях, като натиснете клавиша `a` на клавиатурата си.



![Image](assets/fr/17.webp)



*Инсталиран е Node.js*.



![Image](assets/fr/18.webp)



### 4.4. Проверка на инсталациите



Уверете се, че *Node.js* и *npm* са достъпни:



```powershell
node -v
npm -v
```



И двете команди трябва да връщат номер на версия.



![Image](assets/fr/19.webp)



### 4.5. Инсталиране на Pears с npm



След като *Node.js* и *npm* са налични, инсталирайте глобално **Pears CLI** на вашата система:



```powershell
npm install -g pear
```



Това ще инсталира бинарния файл `pear` в глобалната ви директория *npm*.



![Image](assets/fr/20.webp)



### 4.6. Проверка и инициализация на Pears



След като инсталацията приключи, стартирайте :



```powershell
pear
```



При първото стартиране Pears автоматично ще изтегли необходимите компоненти от мрежата peer-to-peer. Този процес може да отнеме няколко минути.



![Image](assets/fr/21.webp)



Ако всичко е минало добре, трябва да видите помощния екран CLI Pears със списък на наличните подкоманди (run, seed, info...).



### 4.7. Изпитване на Pears с Keet



За да проверите дали Pears работи пълноценно, можете да стартирате приложение P2P, което вече е налично в мрежата, като например Keet - софтуер за съобщения и видеоконференции с отворен код на Holepunch.



```bash
pear run pear://keet
```



Тази команда зарежда приложението Keet директно от мрежата на Pears, без да преминава през централен сървър. Ако Keet се стартира правилно, вашата инсталация на Pears е напълно функционална.



![Image](assets/fr/22.webp)



Вашата система Windows вече е готова да стартира и хоства peer-to-peer приложения с Pears.



## 5. Как да инсталирам Pears на macOS?



Инсталирането на Pears в macOS е подобно на инсталирането му в Linux, но изисква няколко настройки, специфични за средата на Apple. Нека открием тези стъпки заедно.



*Ако използвате Linux или Windows и вече сте инсталирали Pears, можете да преминете директно към **стъпка 6**.*



### 5.1. Проверете системните изисквания



Преди да инсталирате, моля, уверете се, че в системата ви има *Xcode Command Line Tools*. Този пакет предоставя необходимите инструменти за компилиране на _Node.js_ и неговите зависимости.



За да направите това, отворете терминал с клавишната комбинация `Cmd + Space bar`, след което въведете `Terminal` и натиснете клавиша `Enter`. След това можете да въведете тази команда в терминала, за да стартирате инсталацията:



```bash
xcode-select --install
```



Ако инструментите вече са инсталирани в системата ви, macOS ще ви информира за това.



### 5.2. инсталиране на NVM



Pears се разпространява чрез *npm*, мениджъра на пакети *Node.js*. Въпреки че функционирането на Pears не зависи пряко от *Node.js*, той е необходим за инсталацията. Препоръчителният метод за инсталиране на *Node.js* на macOS е *NVM* (*Node Version Manager*), който ви позволява да управлявате няколко версии на Node паралелно.



```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/master/install.sh | bash
```



След това презаредете терминала, за да активирате *NVM* :



```bash
source ~/.zshrc
```



Ако използвате *bash*, а не *zsh*, изпълнете :



```bash
source ~/.bashrc
```



След това проверете дали е инсталиран *NVM*:



```bash
nvm --version
```



Терминалът трябва да върне версията на *NVM*, инсталирана на вашата система.



### 5.3. инсталирайте Node.js и npm



След това инсталирайте стабилна версия на *Node.js* (напр. текущата LTS):



```bash
nvm install --lts
```



След като инсталацията приключи, проверете инсталираните версии:



```bash
node -v
npm -v
```



И двете команди трябва да връщат номер на версия.



### 5.4 Инсталиране на Pears с npm



След като *npm* е наличен, можете да инсталирате Pears CLI глобално на вашата система. Това ще ви позволи да изпълнявате командата `pear` от всяка директория.



```bash
npm install -g pear
```



### 5.5. Иницииране на Pears



След инсталацията просто изпълнете следната команда в терминала:



```bash
pear
```



При първото стартиране Pears ще се свърже с партньорската мрежа, за да изтегли необходимите компоненти. Този процес не изисква централен сървър: файловете се получават директно от други равноправни потребители.



След като изтеглянето приключи, изпълнете командата отново, за да проверите дали всичко работи:



```bash
pear
```



Ако всичко е инсталирано правилно, ще се покаже Pears Help със списък на наличните команди.



### 5.6. Изпитване на Pears с Keet



За да проверите дали Pears работи пълноценно, можете да стартирате приложение P2P, което вече е налично в мрежата, като например Keet, софтуера за съобщения и видеоконференции с отворен код на Holepunch.



```bash
pear run pear://keet
```



Тази команда зарежда приложението Keet директно от мрежата на Pears, без да преминава през централен сървър. Ако Keet се стартира правилно, вашата инсталация на Pears е напълно функционална.



Вашата система macOS вече е готова да стартира и хоства peer-to-peer приложения с Pears.



## 6. Как да използвам дадено приложение в Pears?



След като Pears е готов и работи, можете да стартирате избраното от вас приложение директно със следната команда:



```bash
pear run pear://[KEY]
```



Просто заменете `[KEY]` с ключа на приложението, който искате да използвате.
