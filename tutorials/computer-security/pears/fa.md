در این آموزش، ما یاد خواهیم گرفت که چگونه برنامه‌ها را بر روی **Pears**، یک فناوری همتا به همتا (P2P) توسعه‌یافته توسط **Holepunch** و پشتیبانی‌شده توسط **Tether** اجرا کنیم. هدف ساده است: امکان توزیع و استفاده از برنامه‌های وب بدون تکیه بر هرگونه زیرساخت متمرکز (بدون سرورها، بدون میزبان‌ها، بدون واسطه‌ها) را فراهم کنیم. به عبارت دیگر، حتی اگر یک ارائه‌دهنده ابری تعطیل شود یا یک کشور دامنه‌ای را مسدود کند، برنامه در میان همتایان شبکه به زندگی خود ادامه می‌دهد.



## 1. گلابی چیست؟



پیرز یک محیط زمان اجرا، ابزار توسعه و پلتفرم استقرار برای برنامه‌های همتا به همتا است. این ابزار متن‌باز امکان ساخت، اشتراک‌گذاری و اجرای نرم‌افزار را بدون نیاز به سرور یا زیرساخت، مستقیماً بین کاربران فراهم می‌کند. به‌طور مشخص، این به این معناست که به‌جای میزبانی یک برنامه روی یک سرور مرکزی، هر کاربر به یک گره شبکه تبدیل می‌شود و بخشی از برنامه و داده‌ها را با دیگر همتاها به اشتراک می‌گذارد. کل سیستم یک شبکه توزیع‌شده را تشکیل می‌دهد که هر نمونه با همکاری یکدیگر به حفظ دسترسی‌پذیری سرویس کمک می‌کند.



![Image](assets/fr/01.webp)



این رویکرد بر اساس مجموعه‌ای از بلوک‌های نرم‌افزاری مدولار توسعه‌یافته توسط Holepunch است:




- Hypercore**: یک لاگ توزیع‌شده که بدون نیاز به یک پایگاه‌داده مرکزی، تضمین‌کننده‌ی سازگاری و امنیت داده‌ها است.
- Hyperbee**: یک ایندکسر بر روی Hypercore، برای سازماندهی و مرور کارآمد داده‌ها.
- هایپردرایو**: یک سیستم فایل توزیع‌شده که برای ذخیره و همگام‌سازی فایل‌های برنامه بین همتایان استفاده می‌شود.
- Hyperswarm** و **HyperDHT**: لایه‌های شبکه‌ای که امکان کشف و اتصال بین همتایان در سراسر جهان را بدون نیاز به سرور مرکزی فراهم می‌کنند.
- Secretstream**: یک پروتکل رمزگذاری E2E برای ایمن‌سازی تبادلات بین دو همتا.



با ترکیب این اجزا، Pears امکان ایجاد برنامه‌های خودمختار، رمزگذاری‌شده و توزیع‌شده را فراهم می‌کند، جایی که هر کاربر به‌طور فعال در شبکه مشارکت می‌کند. این معماری غیرمتمرکز هزینه‌های زیرساخت، خطرات سانسور و SPOFها (*نقطه شکست واحد*) را از بین می‌برد.



## ۲. منشأ و فلسفه پروژه



پیرز توسط Holepunch، شرکتی که توسط Mathias Buus و Paolo Ardoino (مدیرعامل Tether و مدیر ارشد فناوری Bitfinex) تأسیس شده است، توسعه می‌یابد. مأموریت این شرکت گسترش منطق همتا به همتا فراتر از Bitcoin است. هدف آن‌ها ساخت "اینترنت همتا به همتا" است، جایی که هر برنامه‌ای می‌تواند بدون مجوز، بدون سرورها و بدون واسطه‌ها اجرا شود. Holepunch پیش از این پشت **Keet**، یک برنامه کامل P2P برای کنفرانس ویدئویی و پیام‌رسانی، بوده است.



*این آموزش نصب Pears به چندین بخش تقسیم شده است که بسته به سیستم عامل شما متفاوت است. مستقیماً به بخشی که با محیط شما مطابقت دارد بروید تا دستورالعمل‌های مناسب را دنبال کنید :*




- لینوکس (دبیان)** → بخش **3**
- ویندوز** → بخش **4**
- macOS** → بخش **5**



## 3. نحوه نصب Pears در لینوکس (دبیان)



نصب Pears بر روی سیستم Debian نسبتاً ساده است، اما نیاز به چند پیش‌نیاز دارد که در این بخش به تفصیل خواهیم پرداخت.



### ۳.۱. به‌روزرسانی سیستم



اول و مهمتر از همه، مهم است که مطمئن شوید سیستم شما به‌روز است.



```bash
sudo apt update && sudo apt upgrade -y
```



![Image](assets/fr/02.webp)



### ۳.۲. نصب وابستگی‌ها



گلابی به برخی از کتابخانه‌های سیستم، به ویژه `libatomic1` که توسط زمان اجرای Bare JavaScript استفاده می‌شود، وابسته است. آن را با فرمان زیر نصب کنید:



```bash
sudo apt install -y libatomic1 curl git
```



![Image](assets/fr/03.webp)



### 3.3. نصب Node.js و npm از طریق NVM



گلابی‌ها از طریق *npm*، مدیر بسته *Node.js* توزیع می‌شوند. اگرچه گلابی‌ها به طور مستقیم به *Node.js* برای عملکرد وابسته نیستند، اما برای نصب ضروری است. روش توصیه شده برای نصب *Node.js* بر روی لینوکس، *NVM* (*مدیر نسخه Node*) است که به شما امکان می‌دهد چندین نسخه از Node را به صورت موازی مدیریت کنید.



```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/master/install.sh | bash
```



![Image](assets/fr/04.webp)



سپس ترمینال خود را مجدداً بارگذاری کنید تا *NVM* فعال شود:



```bash
source ~/.bashrc
```



![Image](assets/fr/05.webp)



بررسی کنید که *NVM* نصب شده است:



```bash
nvm --version
```



![Image](assets/fr/06.webp)



سپس یک نسخه پایدار از *Node.js* (مثلاً LTS فعلی) را نصب کنید:



```bash
nvm install --lts
```



![Image](assets/fr/07.webp)



بررسی نصب *Node.js* و *npm*:



```bash
node -v
npm -v
```



![Image](assets/fr/08.webp)



### ۳.۴ نصب Pears با npm



هنگامی که *npm* در دسترس باشد، می‌توانید Pears CLI را به صورت جهانی روی سیستم خود نصب کنید. این به شما اجازه می‌دهد تا فرمان `pear` را از هر دایرکتوری اجرا کنید.



```bash
npm install -g pear
```



![Image](assets/fr/09.webp)



### ۳.۵. گلابی‌ها را مقداردهی اولیه کنید



پس از نصب، به سادگی فرمان زیر را در ترمینال خود اجرا کنید:



```bash
pear
```



در اولین راه‌اندازی، Pears به شبکه همتا به همتا متصل می‌شود تا اجزای لازم را دانلود کند. این فرآیند به هیچ سرور مرکزی نیاز ندارد: فایل‌ها مستقیماً از سایر همتاها دریافت می‌شوند.



![Image](assets/fr/10.webp)



پس از اتمام دانلود، دوباره فرمان را اجرا کنید تا بررسی کنید که همه چیز درست کار می‌کند:



```bash
pear
```



![Image](assets/fr/11.webp)



اگر همه چیز به درستی نصب شده باشد، راهنمای Pears با فهرستی از دستورات موجود نمایش داده خواهد شد.



### ۳.۶. آزمایش گلابی‌ها با کیت



برای اطمینان از اینکه Pears به‌طور کامل عملیاتی است، می‌توانید یک برنامه P2P که قبلاً در شبکه موجود است، مانند Keet، نرم‌افزار پیام‌رسانی و ویدئوکنفرانس منبع‌باز Holepunch را راه‌اندازی کنید.



```bash
pear run pear://keet
```



این دستور برنامه Keet را مستقیماً از شبکه Pears بارگذاری می‌کند، بدون اینکه از یک سرور مرکزی عبور کند. اگر Keet به درستی راه‌اندازی شود، نصب Pears شما به طور کامل عملیاتی است.



![Image](assets/fr/12.webp)



سیستم لینوکس شما اکنون آماده است تا برنامه‌های نظیر به نظیر را با Pears اجرا و میزبانی کند.



## 4. چگونه Pears را روی ویندوز نصب کنیم



نصب Pears بر روی ویندوز به همان سادگی لینوکس است، اما به چند ابزار ویژه نیاز دارد.



*اگر از لینوکس استفاده می‌کنید، می‌توانید به مرحله ۶ بروید.*



### 4.1. پاورشل را در حالت مدیر باز کنید



ابتدا، PowerShell را با حقوق مدیر اجرا کنید:




- روی منوی شروع کلیک کنید؛
- پاورشل را تایپ کنید ;
- روی "*Windows PowerShell*" راست‌کلیک کنید؛
- گزینه "*Run as administrator*" را انتخاب کنید.



![Image](assets/fr/15.webp)



### ۴.۲. دانلود NVS



گلابی‌ها از طریق *npm*، مدیر بسته *Node.js* نصب می‌شوند. در ویندوز، روشی که توسط Holepunch توصیه می‌شود استفاده از *NVS* (*Node Version Switcher*) است که نسبت به *NVM* در این سیستم پایدارتر است.



در پاورشل، فرمان زیر را برای نصب آخرین نسخه *NVS* اجرا کنید:



```PowerShell
winget install jasongin.nvs
```



![Image](assets/fr/16.webp)



### ۴.۳. نصب Node.js



پس از نصب، PowerShell را مجدداً راه‌اندازی کرده و فرمان زیر را وارد کنید:



```powershell
nvs
```



شما باید لیستی از نسخه‌های موجود *Node.js* را ببینید. اولین نسخه را با فشار دادن کلید `a` روی صفحه‌کلید خود انتخاب کنید.



![Image](assets/fr/17.webp)



*Node.js* نصب شده است.



![Image](assets/fr/18.webp)



### 4.4. بررسی نصب‌ها



مطمئن شوید که *Node.js* و *npm* قابل دسترسی هستند:



```powershell

npm -v
```



هر دو دستور باید یک شماره نسخه برگردانند.



![Image](assets/fr/19.webp)



### ۴.۵. نصب Pears با npm



هنگامی که *Node.js* و *npm* در دسترس هستند، **Pears CLI** را به صورت جهانی روی سیستم خود نصب کنید:



```powershell

```



این کار باینری `pear` را در دایرکتوری *npm* جهانی شما نصب خواهد کرد.

### ۴.۶. بررسی و مقداردهی اولیه گلابی‌ها

## 5. چگونه Pears را روی macOS نصب کنم؟



نصب Pears بر روی macOS مشابه نصب آن بر روی لینوکس است، اما نیاز به چند تنظیم خاص برای محیط اپل دارد. بیایید این مراحل را با هم کشف کنیم.



*اگر از لینوکس یا ویندوز استفاده می‌کنید و Pears را قبلاً نصب کرده‌اید، می‌توانید مستقیماً به **مرحله ۶** بروید.*



### ۵.۱. بررسی الزامات سیستم



قبل از نصب، لطفاً اطمینان حاصل کنید که *Xcode Command Line Tools* بر روی سیستم شما موجود است. این بسته ابزارهای لازم برای کامپایل _Node.js_ و وابستگی‌های آن را فراهم می‌کند.



برای انجام این کار، یک ترمینال را با میانبر صفحه‌کلید `Cmd + Space bar` باز کنید، سپس `Terminal` را تایپ کرده و کلید `Enter` را فشار دهید. سپس می‌توانید این دستور را در ترمینال وارد کنید تا نصب آغاز شود:



```bash
xcode-select --install
```



اگر ابزارها قبلاً روی سیستم شما نصب شده باشند، macOS به شما اطلاع خواهد داد.
