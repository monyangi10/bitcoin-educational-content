В этом уроке мы узнаем, как запускать приложения на **Pears**, одноранговой технологии (P2P), разработанной **Holepunch** и поддерживаемой **Tether**. Цель проста: сделать возможным распространение и использование веб-приложений без опоры на какую-либо централизованную инфраструктуру (без серверов, без хостов, без посредников). Другими словами, даже если облачный провайдер закроется или страна заблокирует домен, приложение будет жить среди сетевых пиров.



## 1. Что такое груши?



Pears - это среда выполнения, инструмент разработки и платформа развертывания для одноранговых приложений. Этот инструмент с открытым исходным кодом позволяет создавать, совместно использовать и запускать программное обеспечение без сервера или инфраструктуры, напрямую между пользователями. Конкретно это означает, что вместо размещения приложения на центральном сервере каждый пользователь становится узлом сети, делясь частью приложения и данными с другими пользователями. Вся система образует распределенную сеть, каждый экземпляр которой сотрудничает для поддержания доступности сервиса.



![Image](assets/fr/01.webp)



Этот подход основан на наборе модульных программных кирпичиков, разработанных компанией Holepunch:




- Hypercore**: распределенный журнал, гарантирующий согласованность и безопасность данных без центральной базы данных.
- Hyperbee**: индексатор поверх Hypercore для эффективной организации и просмотра данных.
- Hyperdrive**: распределенная файловая система, используемая для хранения и синхронизации файлов приложений между пирами.
- Hyperswarm** и **HyperDHT**: сетевые уровни, обеспечивающие обнаружение и соединение между пирами по всему миру без центрального сервера.
- Secretstream**: протокол шифрования E2E для защиты обменов между двумя пирами.



Объединяя эти компоненты, Pears позволяет создавать автономные, зашифрованные и распределенные приложения, где каждый пользователь активно участвует в работе сети. Такая децентрализованная архитектура исключает затраты на инфраструктуру, риски цензуры и SPOF (*одна точка отказа*).



## 2. Истоки и философия проекта



Pears разрабатывается компанией Holepunch, основанной Матиасом Буусом и Паоло Ардойно (CEO Tether и CTO Bitfinex), с целью расширить логику пиринговой сети за пределы Bitcoin. Их цель - построить "одноранговый интернет", где каждое приложение может работать без авторизации, без серверов и без посредников. Holepunch уже стоит за **Keet**, приложением для видеоконференций и обмена сообщениями, полностью соответствующим P2P.



*Это руководство по установке Pears разделено на несколько разделов в зависимости от вашей операционной системы. Перейдите непосредственно к разделу, соответствующему вашей среде, чтобы следовать соответствующим инструкциям :*




- Linux (Debian)** → Часть **3**

## 3. Как установить Pears на Linux (Debian)



Установка Pears на систему Debian относительно проста, но требует нескольких предварительных условий, о которых мы расскажем в этом разделе.



### 3.1. обновить систему



Прежде всего, необходимо убедиться в том, что ваша система находится в актуальном состоянии.



```bash
sudo apt update && sudo apt upgrade -y
```



![Image](assets/fr/02.webp)



### 3.2. установите зависимости



Pears полагается на некоторые системные библиотеки, в частности на `libatomic1`, используемую средой выполнения Bare JavaScript. Установите ее с помощью следующей команды:



```bash
sudo apt install -y libatomic1 curl git
```



![Image](assets/fr/03.webp)



### 3.3. установите Node.js и npm через NVM



Pears распространяется через *npm*, менеджер пакетов *Node.js*. Хотя работа Pears не зависит напрямую от *Node.js*, он необходим для установки. Рекомендуемый метод установки *Node.js* в Linux - *NVM* (*Node Version Manager*), который позволяет управлять несколькими версиями Node параллельно.



```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/master/install.sh | bash
```



![Image](assets/fr/04.webp)



Затем перезагрузите терминал, чтобы активировать *NVM* :



```bash
source ~/.bashrc
```



![Image](assets/fr/05.webp)



Проверьте, установлена ли система *NVM*:



```bash
nvm --version
```



![Image](assets/fr/06.webp)



Затем установите стабильную версию *Node.js* (например, текущую LTS):



```bash
nvm install --lts
```



![Image](assets/fr/07.webp)



Проверьте установки *Node.js* и *npm*:



```bash
node -v
npm -v
```



![Image](assets/fr/08.webp)



### 3.4 Установка Pears с помощью npm



Как только *npm* будет доступен, вы сможете установить Pears CLI глобально в вашей системе. Это позволит вам запускать команду `pear` из любой директории.



```bash
npm install -g pear
```



![Image](assets/fr/09.webp)



### 3.5. Инициализация груш



После установки просто выполните следующую команду в терминале:



```bash
pear
```



При первом запуске Pears подключается к пиринговой сети, чтобы загрузить необходимые компоненты. Этот процесс не требует центрального сервера: файлы получаются напрямую от других пиров.



![Image](assets/fr/10.webp)



После завершения загрузки запустите команду еще раз, чтобы убедиться, что все работает:



```bash
pear
```



![Image](assets/fr/11.webp)



Если все установлено правильно, на экране появится справка Pears со списком доступных команд.



### 3.6. Тестирование груш с китом



Чтобы проверить работоспособность Pears, можно запустить уже имеющееся в сети приложение P2P, например Keet, программное обеспечение Holepunch для обмена сообщениями и видеоконференций с открытым исходным кодом.



```bash
pear run pear://keet
```



Эта команда загружает приложение Keet непосредственно из сети Pears, без прохождения через центральный сервер. Если Keet запускается корректно, значит, ваша установка Pears полностью функциональна.



![Image](assets/fr/12.webp)



Теперь ваша система Linux готова к запуску и размещению одноранговых приложений с помощью Pears.



## 4. Как установить Pears на Windows



Установка Pears на Windows так же проста, как и на Linux, но требует нескольких специальных инструментов.



*Если вы используете Linux, перейдите к шагу 6.*



### 4.1. откройте PowerShell в режиме администратора



Прежде всего, запустите PowerShell с правами администратора:




- Нажмите на меню "Пуск";
- Введите PowerShell ;
- Щелкните правой кнопкой мыши на "*Windows PowerShell*" ;
- Выберите "*Запуск от имени администратора*".



![Image](assets/fr/15.webp)



### 4.2. загрузите NVS



Pears устанавливается с помощью *npm*, менеджера пакетов *Node.js*. На Windows метод, рекомендуемый Holepunch, заключается в использовании *NVS* (*Node Version Switcher*), который более стабилен, чем *NVM* на этой системе.



В PowerShell выполните следующую команду, чтобы установить последнюю версию *NVS* :



```PowerShell
winget install jasongin.nvs
```



![Image](assets/fr/16.webp)



### 4.3. установите Node.js



После установки перезапустите PowerShell и введите следующую команду:



```powershell
nvs
```



Вы увидите список доступных версий *Node.js*. Выберите первую из них, нажав клавишу `a` на клавиатуре.



![Image](assets/fr/17.webp)



*Node.js* установлен.



![Image](assets/fr/18.webp)



### 4.4. Проверьте установку



Убедитесь, что *Node.js* и *npm* доступны:



```powershell
node -v
npm -v
```



Обе команды должны возвращать номер версии.



![Image](assets/fr/19.webp)



### 4.5. Установка Pears с помощью npm



Как только *Node.js* и *npm* будут доступны, установите **Pears CLI** глобально на вашу систему:



```powershell
npm install -g pear
```



Это установит бинарник `pear` в вашу глобальную директорию *npm*.



![Image](assets/fr/20.webp)



### 4.6. Проверка и инициализация груш



После завершения установки запустите команду :



```powershell
pear
```



При первом запуске Pears автоматически загрузит необходимые компоненты из пиринговой сети. Этот процесс может занять несколько минут.



![Image](assets/fr/21.webp)



Если все прошло успешно, вы должны увидеть экран помощи CLI Pears со списком доступных подкоманд (run, seed, info...).



### 4.7. Тестирование груш с китом



Чтобы проверить работоспособность Pears, можно запустить уже имеющееся в сети приложение P2P, например Keet, программное обеспечение Holepunch для обмена сообщениями и видеоконференций с открытым исходным кодом.



```bash
pear run pear://keet
```



Эта команда загружает приложение Keet непосредственно из сети Pears, без прохождения через центральный сервер. Если Keet запускается корректно, значит, ваша установка Pears полностью функциональна.



![Image](assets/fr/22.webp)



Теперь ваша система Windows готова к запуску и размещению одноранговых приложений с помощью Pears.



## 5. Как установить Pears на macOS?



Установка Pears на macOS аналогична установке на Linux, но требует нескольких настроек, характерных для среды Apple. Давайте вместе разберемся в этих шагах.



*Если вы используете Linux или Windows и уже установили Pears, вы можете сразу перейти к **шагу 6**.*



### 5.1. Проверьте системные требования



Перед установкой убедитесь, что в вашей системе присутствует *Xcode Command Line Tools*. Этот пакет предоставляет необходимые инструменты компиляции для _Node.js_ и его зависимостей.



Для этого откройте терминал с помощью сочетания клавиш `Cmd + Пробел`, затем введите `Terminal` и нажмите клавишу `Enter`. Затем введите эту команду в терминале, чтобы запустить установку:



```bash
xcode-select --install
```



Если инструменты уже установлены в вашей системе, macOS сообщит вам об этом.



### 5.2. установите NVM



Pears распространяется через *npm*, менеджер пакетов *Node.js*. Хотя работа Pears не зависит напрямую от *Node.js*, он необходим для установки. Рекомендуемый метод установки *Node.js* на macOS - *NVM* (*Node Version Manager*), который позволяет управлять несколькими версиями Node параллельно.



```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/master/install.sh | bash
```



Затем перезагрузите терминал, чтобы активировать *NVM* :



```bash
source ~/.zshrc
```



Если вы используете *bash*, а не *zsh*, запустите :



```bash
source ~/.bashrc
```



Затем проверьте, что *NVM* установлена:



```bash
nvm --version
```



Терминал должен выдать версию *NVM*, установленную в вашей системе.



### 5.3. установите Node.js и npm



Затем установите стабильную версию *Node.js* (например, текущую LTS):



```bash
nvm install --lts
```



После завершения установки проверьте установленные версии:



```bash
node -v
npm -v
```



Обе команды должны возвращать номер версии.



### 5.4 Установка Pears с помощью npm



Как только *npm* будет доступен, вы сможете установить Pears CLI глобально в вашей системе. Это позволит вам запускать команду `pear` из любой директории.



```bash
npm install -g pear
```



### 5.5. Инициализация груш



После установки просто выполните следующую команду в терминале:



```bash
pear
```



При первом запуске Pears подключается к пиринговой сети, чтобы загрузить необходимые компоненты. Этот процесс не требует центрального сервера: файлы получаются напрямую от других пиров.



После завершения загрузки запустите команду еще раз, чтобы убедиться, что все работает:



```bash
pear
```



Если все установлено правильно, на экране появится справка Pears со списком доступных команд.



### 5.6. Тестирование груш с китом



Чтобы проверить работоспособность Pears, можно запустить уже имеющееся в сети приложение P2P, например Keet, программное обеспечение Holepunch для обмена сообщениями и видеоконференций с открытым исходным кодом.



```bash

```



Эта команда загружает приложение Keet непосредственно из сети Pears, без прохождения через центральный сервер. Если Keet запускается корректно, значит, ваша установка Pears полностью функциональна.

## 6. Как использовать аппликацию на грушах?



После того как Pears будет запущен, вы сможете напрямую запустить выбранное вами приложение с помощью следующей команды:



```bash
pear run pear://[KEY]
```



Просто замените `[KEY]` на ключ приложения, который вы хотите использовать.
