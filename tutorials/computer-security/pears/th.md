ในบทแนะนำนี้ เราจะเรียนรู้วิธีการรันแอปพลิเคชันบน **Pears** ซึ่งเป็นเทคโนโลยีแบบ peer-to-peer (P2P) ที่พัฒนาโดย **Holepunch** และได้รับการสนับสนุนโดย **Tether** เป้าหมายคือเพื่อทำให้สามารถแจกจ่ายและใช้แอปพลิเคชันเว็บได้โดยไม่ต้องพึ่งพาโครงสร้างพื้นฐานแบบรวมศูนย์ (ไม่มีเซิร์ฟเวอร์ ไม่มีโฮสต์ ไม่มีตัวกลาง) กล่าวอีกนัยหนึ่ง แม้ว่าผู้ให้บริการคลาวด์จะปิดตัวลงหรือประเทศใดบล็อกโดเมน แอปพลิเคชันก็ยังคงอยู่ในหมู่เพื่อนร่วมเครือข่าย



## 1. Pears คืออะไร?



Pears เป็นสภาพแวดล้อมรันไทม์ เครื่องมือพัฒนา และแพลตฟอร์มการปรับใช้สำหรับแอปพลิเคชันแบบเพียร์ทูเพียร์ เครื่องมือโอเพ่นซอร์สนี้ทำให้สามารถสร้าง แบ่งปัน และรันซอฟต์แวร์ได้โดยไม่ต้องใช้เซิร์ฟเวอร์หรือโครงสร้างพื้นฐาน โดยตรงระหว่างผู้ใช้ ในแง่ที่เป็นรูปธรรม หมายความว่าแทนที่จะโฮสต์แอปพลิเคชันบนเซิร์ฟเวอร์กลาง ผู้ใช้แต่ละคนจะกลายเป็นโหนดเครือข่าย แบ่งปันส่วนหนึ่งของแอปพลิเคชันและข้อมูลกับเพียร์อื่น ๆ ทั้งระบบจะสร้างเครือข่ายแบบกระจาย โดยแต่ละอินสแตนซ์จะร่วมมือกันเพื่อให้บริการเข้าถึงได้



![Image](assets/fr/01.webp)



วิธีการนี้อิงตามชุดของซอฟต์แวร์โมดูลาร์ที่พัฒนาโดย Holepunch:




- Hypercore**: บันทึกแบบกระจายที่รับประกันความสม่ำเสมอและความปลอดภัยของข้อมูลโดยไม่ต้องใช้ฐานข้อมูลกลาง
- Hyperbee**: ตัวจัดทำดัชนีบน Hypercore สำหรับการจัดระเบียบและเรียกดูข้อมูลอย่างมีประสิทธิภาพ
- Hyperdrive**: ระบบไฟล์แบบกระจายที่ใช้ในการจัดเก็บและซิงโครไนซ์ไฟล์แอปพลิเคชันระหว่างเพียร์
- Hyperswarm** และ **HyperDHT**: เลเยอร์เครือข่ายที่ช่วยให้การค้นหาและเชื่อมต่อระหว่างเพื่อนทั่วโลกเป็นไปได้ โดยไม่ต้องใช้เซิร์ฟเวอร์กลาง
- Secretstream**: โปรโตคอลการเข้ารหัส E2E เพื่อรักษาความปลอดภัยในการแลกเปลี่ยนระหว่างเพียร์สองคน



โดยการรวมส่วนประกอบเหล่านี้ Pears ทำให้สามารถสร้างแอปพลิเคชันที่เป็นอิสระ เข้ารหัส และกระจายได้ ซึ่งผู้ใช้แต่ละคนมีส่วนร่วมอย่างแข็งขันในเครือข่าย สถาปัตยกรรมแบบกระจายนี้ช่วยขจัดค่าใช้จ่ายด้านโครงสร้างพื้นฐาน ความเสี่ยงจากการเซ็นเซอร์ และ SPOFs (*Single Point of Failure*)



## 2. ที่มาและปรัชญาของโครงการ



Pears กำลังพัฒนาโดย Holepunch บริษัทที่ก่อตั้งโดย Mathias Buus และ Paolo Ardoino (CEO ของ Tether และ CTO ของ Bitfinex) โดยมีภารกิจในการขยายตรรกะ peer-to-peer ให้เกินกว่า Bitcoin ความทะเยอทะยานของพวกเขาคือการสร้าง "อินเทอร์เน็ตแบบ Peer-to-Peer" ที่ทุกแอปพลิเคชันสามารถทำงานได้โดยไม่ต้องมีการอนุญาต ไม่มีเซิร์ฟเวอร์ และไม่มีตัวกลาง Holepunch อยู่เบื้องหลัง **Keet** ซึ่งเป็นแอปพลิเคชันวิดีโอคอนเฟอเรนซ์และการส่งข้อความแบบ P2P อย่างเต็มรูปแบบแล้ว



*คู่มือการติดตั้ง Pears นี้แบ่งออกเป็นหลายส่วนขึ้นอยู่กับระบบปฏิบัติการของคุณ ไปยังส่วนที่ตรงกับสภาพแวดล้อมของคุณเพื่อทำตามคำแนะนำที่เหมาะสม :*




- ลินุกซ์ (Debian)** → ส่วนที่ **3**
- Windows** → Part **4**
- macOS** → Part **5**



## 3. วิธีการติดตั้ง Pears บน Linux (Debian)



การติดตั้ง Pears บนระบบ Debian นั้นค่อนข้างตรงไปตรงมา แต่ต้องมีข้อกำหนดเบื้องต้นบางประการ ซึ่งเราจะอธิบายในส่วนนี้



### 3.1. อัปเดตระบบ



ก่อนอื่นเลย สิ่งสำคัญคือต้องตรวจสอบให้แน่ใจว่าระบบของคุณเป็นเวอร์ชันล่าสุด



```bash
sudo apt update && sudo apt upgrade -y
```



![Image](assets/fr/02.webp)



### 3.2. ติดตั้งการพึ่งพา



Pears อาศัยไลบรารีระบบบางตัว โดยเฉพาะ `libatomic1` ซึ่งใช้โดย Bare JavaScript runtime ติดตั้งด้วยคำสั่งต่อไปนี้:



```bash
sudo apt install -y libatomic1 curl git
```



![Image](assets/fr/03.webp)



### 3.3. ติดตั้ง Node.js และ npm ผ่าน NVM



Pears ถูกแจกจ่ายผ่าน *npm* ซึ่งเป็นตัวจัดการแพ็คเกจของ *Node.js* แม้ว่า Pears จะไม่ขึ้นอยู่กับ *Node.js* โดยตรงในการทำงาน แต่ก็จำเป็นสำหรับการติดตั้ง วิธีที่แนะนำสำหรับการติดตั้ง *Node.js* บน Linux คือ *NVM* (*Node Version Manager*) ซึ่งช่วยให้คุณจัดการหลายเวอร์ชันของ Node ได้พร้อมกัน



```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/master/install.sh | bash
```



![Image](assets/fr/04.webp)



จากนั้นโหลดเทอร์มินัลของคุณใหม่เพื่อเปิดใช้งาน *NVM* :



```bash
source ~/.bashrc
```



![Image](assets/fr/05.webp)



ตรวจสอบว่า *NVM* ได้รับการติดตั้งแล้ว:



```bash
nvm --version
```



![Image](assets/fr/06.webp)



จากนั้นติดตั้งเวอร์ชันที่เสถียรของ *Node.js* (เช่น LTS ปัจจุบัน):



```bash
nvm install --lts
```



![Image](assets/fr/07.webp)



ตรวจสอบการติดตั้ง *Node.js* และ *npm*:



```bash
node -v
npm -v
```



![Image](assets/fr/08.webp)



### 3.4 การติดตั้ง Pears ด้วย npm



เมื่อ *npm* พร้อมใช้งาน คุณสามารถติดตั้ง Pears CLI ทั่วทั้งระบบของคุณได้ ซึ่งจะทำให้คุณสามารถเรียกใช้คำสั่ง `pear` จากไดเรกทอรีใดก็ได้



```bash
npm install -g pear
```



![Image](assets/fr/09.webp)



### 3.5. เริ่มต้น Pears



หลังการติดตั้ง เพียงแค่รันคำสั่งต่อไปนี้ในเทอร์มินัลของคุณ:



```bash
pear
```



เมื่อเริ่มต้นครั้งแรก Pears จะเชื่อมต่อกับเครือข่าย peer-to-peer เพื่อดาวน์โหลดส่วนประกอบที่จำเป็น กระบวนการนี้ไม่ต้องการเซิร์ฟเวอร์กลาง: ไฟล์จะถูกดึงมาโดยตรงจากเพียร์อื่น ๆ



![Image](assets/fr/10.webp)



เมื่อการดาวน์โหลดเสร็จสิ้น ให้รันคำสั่งอีกครั้งเพื่อตรวจสอบว่าทุกอย่างทำงานได้:



```bash
pear
```



![Image](assets/fr/11.webp)



หากทุกอย่างติดตั้งอย่างถูกต้อง Pears Help จะแสดงพร้อมรายการคำสั่งที่ใช้ได้



### 3.6. ทดสอบ Pears กับ Keet



ในการตรวจสอบว่า Pears ทำงานได้เต็มที่หรือไม่ คุณสามารถเปิดแอปพลิเคชัน P2P ที่มีอยู่แล้วบนเครือข่าย เช่น Keet ซึ่งเป็นซอฟต์แวร์ส่งข้อความและการประชุมทางวิดีโอแบบโอเพนซอร์สของ Holepunch



```bash
pear run pear://keet
```



คำสั่งนี้จะโหลดแอปพลิเคชัน Keet โดยตรงจากเครือข่าย Pears โดยไม่ต้องผ่านเซิร์ฟเวอร์กลาง หาก Keet เปิดตัวได้อย่างถูกต้อง แสดงว่าการติดตั้ง Pears ของคุณทำงานได้อย่างสมบูรณ์



![Image](assets/fr/12.webp)



ระบบ Linux ของคุณพร้อมแล้วที่จะรันและโฮสต์แอปพลิเคชันแบบเพียร์ทูเพียร์ด้วย Pears



## 4. วิธีการติดตั้ง Pears บน Windows



การติดตั้ง Pears บน Windows นั้นง่ายพอๆ กับบน Linux แต่ต้องใช้เครื่องมือพิเศษบางอย่าง



*หากคุณใช้ Linux คุณสามารถข้ามไปที่ขั้นตอนที่ 6 ได้*



### 4.1. เปิด PowerShell ในโหมดผู้ดูแลระบบ



ก่อนอื่น ให้เรียกใช้ PowerShell ด้วยสิทธิ์ของผู้ดูแลระบบ :

- พิมพ์ PowerShell ;
- คลิกขวาที่ "*Windows PowerShell*" ;
- เลือก "*เรียกใช้ในฐานะผู้ดูแลระบบ*"



![Image](assets/fr/15.webp)



### 4.2. ดาวน์โหลด NVS



Pears ติดตั้งผ่าน *npm* ซึ่งเป็นตัวจัดการแพ็กเกจของ *Node.js* บน Windows วิธีที่ Holepunch แนะนำคือการใช้ *NVS* (*Node Version Switcher*) ซึ่งมีความเสถียรกว่า *NVM* บนระบบนี้



ใน PowerShell ให้รันคำสั่งต่อไปนี้เพื่อติดตั้งเวอร์ชันล่าสุดของ *NVS* :


### 4.3. ติดตั้ง Node.js



หลังการติดตั้ง ให้รีสตาร์ท PowerShell และป้อนคำสั่งต่อไปนี้:



```powershell

```



คุณควรเห็นรายการของเวอร์ชัน *Node.js* ที่มีอยู่ เลือกอันแรกโดยกดปุ่ม `a` บนแป้นพิมพ์ของคุณ

### 4.7. ทดสอบ Pears กับ Keet



ในการตรวจสอบว่า Pears ทำงานได้อย่างเต็มที่หรือไม่ คุณสามารถเปิดแอปพลิเคชัน P2P ที่มีอยู่แล้วบนเครือข่าย เช่น Keet ซึ่งเป็นซอฟต์แวร์ส่งข้อความและการประชุมผ่านวิดีโอแบบโอเพนซอร์สของ Holepunch

## 5. ฉันจะติดตั้ง Pears บน macOS ได้อย่างไร?



การติดตั้ง Pears บน macOS คล้ายกับการติดตั้งบน Linux แต่ต้องมีการปรับเปลี่ยนบางอย่างที่เฉพาะเจาะจงสำหรับสภาพแวดล้อมของ Apple มาค้นพบขั้นตอนเหล่านี้ไปด้วยกัน



*หากคุณใช้ Linux หรือ Windows และได้ติดตั้ง Pears แล้ว คุณสามารถไปที่ **ขั้นตอนที่ 6** ได้โดยตรง*



### 5.1. ตรวจสอบความต้องการของระบบ



ก่อนการติดตั้ง โปรดตรวจสอบให้แน่ใจว่า *Xcode Command Line Tools* มีอยู่ในระบบของคุณ แพ็กเกจนี้จะให้เครื่องมือการคอมไพล์ที่จำเป็นสำหรับ _Node.js_ และการพึ่งพาของมัน



ในการทำเช่นนี้ ให้เปิดเทอร์มินัลด้วยคีย์ลัด `Cmd + Space bar` จากนั้นพิมพ์ `Terminal` และกดปุ่ม `Enter` คุณสามารถป้อนคำสั่งนี้ในเทอร์มินัลเพื่อเริ่มการติดตั้ง:



```bash
xcode-select --install
```



หากเครื่องมือได้รับการติดตั้งบนระบบของคุณแล้ว macOS จะแจ้งให้คุณทราบ



### 5.2. ติดตั้ง NVM



Pears ถูกแจกจ่ายผ่าน *npm* ซึ่งเป็นตัวจัดการแพ็กเกจของ *Node.js* แม้ว่า Pears จะไม่ขึ้นอยู่กับ *Node.js* โดยตรงในการทำงาน แต่ก็จำเป็นสำหรับการติดตั้ง วิธีที่แนะนำสำหรับการติดตั้ง *Node.js* บน macOS คือ *NVM* (*Node Version Manager*) ซึ่งช่วยให้คุณจัดการหลายเวอร์ชันของ Node ได้พร้อมกัน



```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/master/install.sh | bash
```



จากนั้นโหลดเทอร์มินัลของคุณใหม่เพื่อเปิดใช้งาน *NVM* :



```bash
source ~/.zshrc
```



หากคุณใช้ *bash* แทนที่จะเป็น *zsh* ให้รัน :



```bash
source ~/.bashrc
```



จากนั้นตรวจสอบว่า *NVM* ได้รับการติดตั้ง:



```bash
nvm --version
```



เทอร์มินัลควรแสดงเวอร์ชันของ *NVM* ที่ติดตั้งบนระบบของคุณ



### 5.3. ติดตั้ง Node.js และ npm



จากนั้นติดตั้งเวอร์ชันที่เสถียรของ *Node.js* (เช่น LTS ปัจจุบัน):



```bash
nvm install --lts
```



เมื่อติดตั้งเสร็จสิ้น ให้ตรวจสอบเวอร์ชันที่ติดตั้ง:



```bash
node -v
npm -v
```



ทั้งสองคำสั่งต้องส่งคืนหมายเลขเวอร์ชัน



### 5.4 การติดตั้ง Pears ด้วย npm



เมื่อ *npm* พร้อมใช้งาน คุณสามารถติดตั้ง Pears CLI ทั่วทั้งระบบของคุณได้ ซึ่งจะทำให้คุณสามารถเรียกใช้คำสั่ง `pear` จากไดเรกทอรีใดก็ได้



```bash
npm install -g pear
```



### 5.5. เริ่มต้น Pears



หลังการติดตั้ง เพียงแค่รันคำสั่งต่อไปนี้ในเทอร์มินัลของคุณ:



```bash
pear
```



เมื่อเริ่มต้นครั้งแรก Pears จะเชื่อมต่อกับเครือข่าย peer-to-peer เพื่อดาวน์โหลดส่วนประกอบที่จำเป็น กระบวนการนี้ไม่ต้องการเซิร์ฟเวอร์กลาง: ไฟล์จะถูกดึงมาโดยตรงจากเพียร์อื่น ๆ



เมื่อการดาวน์โหลดเสร็จสิ้น ให้รันคำสั่งอีกครั้งเพื่อตรวจสอบว่าทุกอย่างทำงานได้:



```bash
pear
```



หากทุกอย่างติดตั้งอย่างถูกต้อง จะมีการแสดง Pears Help พร้อมรายการคำสั่งที่ใช้ได้



### 5.6. ทดสอบ Pears กับ Keet



ในการตรวจสอบว่า Pears ทำงานได้อย่างสมบูรณ์หรือไม่ คุณสามารถเปิดแอปพลิเคชัน P2P ที่มีอยู่แล้วในเครือข่าย เช่น Keet ซึ่งเป็นซอฟต์แวร์ส่งข้อความและการประชุมทางวิดีโอแบบโอเพนซอร์สของ Holepunch



```bash
pear run pear://keet
```



คำสั่งนี้โหลดแอปพลิเคชัน Keet โดยตรงจากเครือข่าย Pears โดยไม่ผ่านเซิร์ฟเวอร์กลาง หาก Keet เปิดตัวได้อย่างถูกต้อง แสดงว่าการติดตั้ง Pears ของคุณทำงานได้อย่างสมบูรณ์



ระบบ macOS ของคุณพร้อมแล้วที่จะรันและโฮสต์แอปพลิเคชันแบบเพียร์ทูเพียร์ด้วย Pears



## 6. ฉันจะใช้งานแอปพลิเคชันบน Pears ได้อย่างไร?

```bash
pear run pear://[KEY]
```
