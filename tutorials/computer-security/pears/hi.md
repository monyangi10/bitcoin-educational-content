इस ट्यूटोरियल में, हम सीखेंगे कि **Pears** पर एप्लिकेशन कैसे चलाएँ, जो **Holepunch** द्वारा विकसित और **Tether** द्वारा समर्थित एक पीयर-टू-पीयर (P2P) तकनीक है। इसका उद्देश्य सरल है: किसी भी केंद्रीकृत बुनियादी ढाँचे (बिना सर्वर, बिना होस्ट, बिना मध्यस्थ) पर निर्भर हुए वेब एप्लिकेशन को वितरित और उपयोग करना संभव बनाना। दूसरे शब्दों में, भले ही कोई क्लाउड प्रदाता बंद हो जाए या कोई देश किसी डोमेन को ब्लॉक कर दे, एप्लिकेशन नेटवर्क पीयर के बीच बना रहता है।



## 1. नाशपाती क्या है?



पीयर्स पीयर-टू-पीयर अनुप्रयोगों के लिए एक रनटाइम वातावरण, विकास उपकरण और परिनियोजन प्लेटफ़ॉर्म है। यह ओपन-सोर्स उपकरण बिना किसी सर्वर या बुनियादी ढाँचे के, सीधे उपयोगकर्ताओं के बीच सॉफ़्टवेयर बनाना, साझा करना और चलाना संभव बनाता है। ठोस शब्दों में, इसका अर्थ है कि किसी एप्लिकेशन को केंद्रीय सर्वर पर होस्ट करने के बजाय, प्रत्येक उपयोगकर्ता एक नेटवर्क नोड बन जाता है, जो एप्लिकेशन के कुछ हिस्से और डेटा को अन्य पीयर के साथ साझा करता है। पूरा सिस्टम एक वितरित नेटवर्क बनाता है, जिसमें प्रत्येक इंस्टेंस सेवा को सुलभ बनाए रखने के लिए सहयोग करता है।



![Image](assets/fr/01.webp)



यह दृष्टिकोण होलपंच द्वारा विकसित मॉड्यूलर सॉफ्टवेयर ब्रिक्स के एक सेट पर आधारित है:




- हाइपरकोर**: एक वितरित लॉग जो केंद्रीय डेटाबेस के बिना डेटा स्थिरता और सुरक्षा की गारंटी देता है।
- हाइपरबी**: कुशल डेटा संगठन और ब्राउज़िंग के लिए हाइपरकोर के शीर्ष पर एक इंडेक्सर।
- हाइपरड्राइव**: एक वितरित फ़ाइल सिस्टम जिसका उपयोग समकक्षों के बीच अनुप्रयोग फ़ाइलों को संग्रहीत और सिंक्रनाइज़ करने के लिए किया जाता है।
- हाइपरस्वार्म** और **हाइपरडीएचटी**: नेटवर्क परतें जो केंद्रीय सर्वर के बिना, दुनिया भर के साथियों के बीच खोज और कनेक्शन को सक्षम बनाती हैं।
- सीक्रेटस्ट्रीम**: दो साथियों के बीच आदान-प्रदान को सुरक्षित करने के लिए एक E2E एन्क्रिप्शन प्रोटोकॉल।



इन घटकों को मिलाकर, पीयर्स स्वायत्त, एन्क्रिप्टेड और वितरित एप्लिकेशन बनाना संभव बनाता है, जहाँ प्रत्येक उपयोगकर्ता नेटवर्क में सक्रिय रूप से भाग लेता है। यह विकेन्द्रीकृत आर्किटेक्चर बुनियादी ढाँचे की लागत, सेंसरशिप जोखिमों और एसपीओएफ (*एकल विफलता बिंदु*) को समाप्त करता है।



## 2. परियोजना की उत्पत्ति और दर्शन



पीयर्स को होलपंच द्वारा विकसित किया जा रहा है, जिसकी स्थापना मैथियास बुस और पाओलो अर्दोइनो (टीथर के सीईओ और बिटफाइनेक्स के सीटीओ) ने की थी, जिसका लक्ष्य पीयर-टू-पीयर लॉजिक को Bitcoin से आगे बढ़ाना है। उनकी महत्वाकांक्षा "पीयर-टू-पीयर इंटरनेट" बनाने की है, जहाँ हर एप्लिकेशन बिना प्राधिकरण, बिना सर्वर और बिना किसी मध्यस्थ के चल सके। होलपंच पहले से ही **कीट** नामक एक पूर्ण P2P वीडियो कॉन्फ्रेंसिंग और मैसेजिंग एप्लिकेशन पर काम कर रहा है।



*यह पीयर्स इंस्टॉलेशन ट्यूटोरियल आपके ऑपरेटिंग सिस्टम के आधार पर कई खंडों में विभाजित है। उपयुक्त निर्देशों का पालन करने के लिए सीधे अपने परिवेश से संबंधित अनुभाग पर जाएँ:*




- लिनक्स (डेबियन)** → भाग **3**
- विंडोज़** → भाग **4**
- macOS** → भाग **5**



## 3. लिनक्स (डेबियन) पर पीयर्स कैसे स्थापित करें



डेबियन सिस्टम पर पीयर्स को स्थापित करना अपेक्षाकृत सरल है, लेकिन इसके लिए कुछ पूर्वापेक्षाओं की आवश्यकता होती है, जिनके बारे में हम इस अनुभाग में विस्तार से बताएंगे।



### 3.1. सिस्टम को अपडेट करें



सबसे पहले और सबसे महत्वपूर्ण बात यह सुनिश्चित करना है कि आपका सिस्टम अद्यतन है।



```bash
sudo apt update && sudo apt upgrade -y
```



![Image](assets/fr/02.webp)



### 3.2. निर्भरताएँ स्थापित करें



पीयर्स कुछ सिस्टम लाइब्रेरीज़ पर निर्भर करता है, खासकर `libatomic1` पर, जिसका इस्तेमाल बेयर जावास्क्रिप्ट रनटाइम द्वारा किया जाता है। इसे निम्न कमांड से इंस्टॉल करें:



```bash
sudo apt install -y libatomic1 curl git
```



![Image](assets/fr/03.webp)



### 3.3. NVM के माध्यम से Node.js और npm स्थापित करें



Pears को *npm*, *Node.js* पैकेज मैनेजर के माध्यम से वितरित किया जाता है। हालाँकि Pears कार्य करने के लिए सीधे *Node.js* पर निर्भर नहीं है, फिर भी यह इंस्टॉलेशन के लिए आवश्यक है। Linux पर *Node.js* को इंस्टॉल करने के लिए अनुशंसित विधि *NVM* (*Node Version Manager*) है, जो आपको Node के कई संस्करणों को समानांतर रूप से प्रबंधित करने की अनुमति देता है।



```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/master/install.sh | bash
```



![Image](assets/fr/04.webp)



फिर *NVM* को सक्रिय करने के लिए अपने टर्मिनल को पुनः लोड करें:



```bash
source ~/.bashrc
```



![Image](assets/fr/05.webp)



जाँच करें कि *NVM* स्थापित है:



```bash
nvm --version
```



![Image](assets/fr/06.webp)



फिर *Node.js* का एक स्थिर संस्करण स्थापित करें (उदाहरण के लिए वर्तमान LTS):



```bash
nvm install --lts
```



![Image](assets/fr/07.webp)



*Node.js* और *npm* इंस्टॉलेशन की जाँच करें:



```bash
node -v
npm -v
```



![Image](assets/fr/08.webp)



### 3.4 एनपीएम के साथ नाशपाती स्थापित करना



*npm* उपलब्ध होने के बाद, आप अपने सिस्टम पर Pears CLI को वैश्विक रूप से इंस्टॉल कर सकते हैं। इससे आप किसी भी डायरेक्टरी से `pear` कमांड चला सकेंगे।



```bash
npm install -g pear
```



![Image](assets/fr/09.webp)



### 3.5. नाशपाती को आरंभ करें



स्थापना के बाद, अपने टर्मिनल में निम्नलिखित कमांड चलाएँ:



```bash
pear
```



पहली बार शुरू होने पर, पीयर्स आवश्यक घटकों को डाउनलोड करने के लिए पीयर-टू-पीयर नेटवर्क से जुड़ जाएगा। इस प्रक्रिया के लिए किसी केंद्रीय सर्वर की आवश्यकता नहीं होती: फ़ाइलें सीधे अन्य पीयर से प्राप्त की जाती हैं।



![Image](assets/fr/10.webp)



डाउनलोड पूरा हो जाने पर, यह जांचने के लिए कि सब कुछ ठीक से काम कर रहा है, कमांड को पुनः चलाएँ:



```bash
pear
```



![Image](assets/fr/11.webp)



यदि सब कुछ सही ढंग से स्थापित है, तो Pears Help उपलब्ध कमांडों की सूची के साथ प्रदर्शित होगी।



### 3.6. कीट के साथ नाशपाती का परीक्षण



यह जांचने के लिए कि पीयर्स पूरी तरह से चालू है, आप नेटवर्क पर पहले से उपलब्ध P2P एप्लिकेशन लॉन्च कर सकते हैं, जैसे कि कीट, होलपंच का ओपन-सोर्स मैसेजिंग और वीडियोकांफ्रेंसिंग सॉफ्टवेयर।



```bash
pear run pear://keet
```



यह कमांड कीट एप्लिकेशन को सीधे पीयर्स नेटवर्क से लोड करता है, बिना किसी केंद्रीय सर्वर से गुज़रे। अगर कीट सही तरीके से लॉन्च होता है, तो आपका पीयर्स इंस्टॉलेशन पूरी तरह से काम कर रहा है।



![Image](assets/fr/12.webp)



आपका लिनक्स सिस्टम अब Pears के साथ पीयर-टू-पीयर अनुप्रयोगों को चलाने और होस्ट करने के लिए तैयार है।



## 4. विंडोज़ पर पीयर्स कैसे स्थापित करें



विंडोज़ पर पीयर्स को स्थापित करना लिनक्स की तरह ही आसान है, लेकिन इसके लिए कुछ विशेष उपकरणों की आवश्यकता होती है।



*यदि आप लिनक्स का उपयोग कर रहे हैं, तो आप चरण 6 पर जा सकते हैं।*



### 4.1. PowerShell को व्यवस्थापक मोड में खोलें



सबसे पहले, व्यवस्थापक अधिकारों के साथ PowerShell चलाएँ:




- स्टार्ट मेनू पर क्लिक करें;
- प्रकार PowerShell;
- "*Windows PowerShell*" पर राइट-क्लिक करें;
- "*व्यवस्थापक के रूप में चलाएँ*" का चयन करें।



![Image](assets/fr/15.webp)



### 4.2. एनवीएस डाउनलोड करें



पीयर्स को *npm*, *Node.js* पैकेज मैनेजर के ज़रिए इंस्टॉल किया जाता है। विंडोज़ पर, होलपंच द्वारा सुझाई गई विधि *NVS* (*नोड वर्जन स्विचर*) का उपयोग करना है, जो इस सिस्टम पर *NVM* से ज़्यादा स्थिर है।



PowerShell में, *NVS* का नवीनतम संस्करण स्थापित करने के लिए निम्न आदेश चलाएँ:



```PowerShell
winget install jasongin.nvs
```



![Image](assets/fr/16.webp)



### 4.3. Node.js इंस्टॉल करें



स्थापना के बाद, PowerShell को पुनः आरंभ करें और निम्न आदेश दर्ज करें:



```powershell
nvs
```



आपको उपलब्ध *Node.js* संस्करणों की एक सूची दिखाई देगी। अपने कीबोर्ड पर `a` कुंजी दबाकर पहला संस्करण चुनें।



![Image](assets/fr/17.webp)



*Node.js* स्थापित है.



![Image](assets/fr/18.webp)



### 4.4. इंस्टॉलेशन की जाँच करें



सुनिश्चित करें कि *Node.js* और *npm* सुलभ हैं:



```powershell
node -v
npm -v
```



दोनों कमांडों को संस्करण संख्या लौटानी होगी।



![Image](assets/fr/19.webp)



### 4.5. NPM के साथ Pears इंस्टॉल करना



एक बार *Node.js* और *npm* उपलब्ध हो जाने पर, अपने सिस्टम पर **Pears CLI** को वैश्विक रूप से स्थापित करें:



```powershell
npm install -g pear
```



यह आपकी वैश्विक *npm* निर्देशिका में `pear` बाइनरी स्थापित करेगा।



![Image](assets/fr/20.webp)



### 4.6. नाशपाती की जाँच करें और उसे आरंभ करें



एक बार स्थापना पूर्ण हो जाने पर, चलाएँ:



```powershell
pear
```



पहली बार लॉन्च होने पर, Pears पीयर-टू-पीयर नेटवर्क से ज़रूरी कंपोनेंट्स को अपने आप डाउनलोड कर लेगा। इस प्रक्रिया में कुछ समय लग सकता है।



![Image](assets/fr/21.webp)



यदि सब कुछ ठीक रहा है, तो आपको उपलब्ध उप-कमांडों (रन, seed, जानकारी...) की सूची के साथ CLI पीयर्स सहायता स्क्रीन दिखाई देगी।



### 4.7. कीट के साथ नाशपाती का परीक्षण



यह जांचने के लिए कि पीयर्स पूरी तरह से चालू है, आप नेटवर्क पर पहले से उपलब्ध P2P एप्लिकेशन लॉन्च कर सकते हैं, जैसे कि कीट, होलपंच का ओपन-सोर्स मैसेजिंग और वीडियोकांफ्रेंसिंग सॉफ्टवेयर।



```bash
pear run pear://keet
```



यह कमांड कीट एप्लिकेशन को सीधे पीयर्स नेटवर्क से लोड करता है, बिना किसी केंद्रीय सर्वर से गुज़रे। अगर कीट सही तरीके से लॉन्च होता है, तो आपका पीयर्स इंस्टॉलेशन पूरी तरह से काम कर रहा है।



![Image](assets/fr/22.webp)



आपका विंडोज सिस्टम अब Pears के साथ पीयर-टू-पीयर एप्लिकेशन चलाने और होस्ट करने के लिए तैयार है।



## 5. मैं macOS पर Pears कैसे स्थापित करूं?



MacOS पर Pears इंस्टॉल करना Linux पर इंस्टॉल करने जैसा ही है, लेकिन इसके लिए Apple परिवेश के लिए कुछ खास समायोजनों की आवश्यकता होती है। आइए इन चरणों को एक साथ देखें।



*यदि आप लिनक्स या विंडोज का उपयोग कर रहे हैं और आपने पहले ही पीयर्स इंस्टॉल कर लिया है, तो आप सीधे **चरण 6** पर जा सकते हैं।*



### 5.1. सिस्टम आवश्यकताएँ जांचें



इंस्टॉल करने से पहले, कृपया सुनिश्चित करें कि आपके सिस्टम पर *Xcode कमांड लाइन टूल्स* मौजूद हैं। यह पैकेज _Node.js_ और उसकी निर्भरताओं के लिए आवश्यक संकलन उपकरण प्रदान करता है।



ऐसा करने के लिए, कीबोर्ड शॉर्टकट `Cmd + Space bar` से टर्मिनल खोलें, फिर `Terminal` टाइप करें और `Enter` कुंजी दबाएँ। फिर आप इंस्टॉलेशन शुरू करने के लिए टर्मिनल में यह कमांड दर्ज कर सकते हैं:



```bash
xcode-select --install
```



यदि उपकरण आपके सिस्टम पर पहले से ही स्थापित हैं, तो macOS आपको सूचित करेगा।



### 5.2. NVM स्थापित करें



Pears को *npm*, *Node.js* पैकेज मैनेजर के माध्यम से वितरित किया जाता है। हालाँकि Pears कार्य करने के लिए सीधे *Node.js* पर निर्भर नहीं है, फिर भी यह इंस्टॉलेशन के लिए आवश्यक है। macOS पर *Node.js* को इंस्टॉल करने के लिए अनुशंसित तरीका *NVM* (*Node Version Manager*) है, जो आपको Node के कई संस्करणों को समानांतर रूप से प्रबंधित करने की अनुमति देता है।



```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/master/install.sh | bash
```



फिर *NVM* को सक्रिय करने के लिए अपने टर्मिनल को पुनः लोड करें:



```bash
source ~/.zshrc
```



यदि आप *zsh* के बजाय *bash* का उपयोग करते हैं, तो चलाएँ:



```bash
source ~/.bashrc
```



फिर जाँचें कि *NVM* स्थापित है:



```bash
nvm --version
```



टर्मिनल को आपके सिस्टम पर स्थापित *NVM* का संस्करण लौटाना चाहिए।



### 5.3. Node.js और npm इंस्टॉल करें



फिर *Node.js* का एक स्थिर संस्करण स्थापित करें (उदाहरण के लिए वर्तमान LTS):



```bash
nvm install --lts
```



एक बार स्थापना पूर्ण हो जाने पर, स्थापित संस्करणों की जांच करें:



```bash
node -v
npm -v
```



दोनों कमांडों को संस्करण संख्या लौटानी होगी।



### 5.4 एनपीएम के साथ नाशपाती स्थापित करना



*npm* उपलब्ध होने के बाद, आप अपने सिस्टम पर Pears CLI को वैश्विक रूप से इंस्टॉल कर सकते हैं। इससे आप किसी भी डायरेक्टरी से `pear` कमांड चला सकेंगे।



```bash
npm install -g pear
```



### 5.5. नाशपाती को आरंभ करें



स्थापना के बाद, अपने टर्मिनल में निम्नलिखित कमांड चलाएँ:



```bash
pear
```



पहली बार शुरू होने पर, पीयर्स आवश्यक घटकों को डाउनलोड करने के लिए पीयर-टू-पीयर नेटवर्क से जुड़ जाएगा। इस प्रक्रिया के लिए किसी केंद्रीय सर्वर की आवश्यकता नहीं होती: फ़ाइलें सीधे अन्य पीयर से प्राप्त की जाती हैं।



डाउनलोड पूरा हो जाने पर, यह जांचने के लिए कि सब कुछ ठीक से काम कर रहा है, कमांड को पुनः चलाएँ:



```bash
pear
```



यदि सब कुछ सही ढंग से स्थापित है, तो Pears Help उपलब्ध कमांडों की सूची के साथ प्रदर्शित होगी।



### 5.6. कीट के साथ नाशपाती का परीक्षण



यह जांचने के लिए कि पीयर्स पूरी तरह से चालू है, आप नेटवर्क पर पहले से उपलब्ध P2P एप्लिकेशन लॉन्च कर सकते हैं, जैसे कि कीट, होलपंच का ओपन-सोर्स मैसेजिंग और वीडियोकांफ्रेंसिंग सॉफ्टवेयर।



```bash
pear run pear://keet
```



यह कमांड कीट एप्लिकेशन को सीधे पीयर्स नेटवर्क से लोड करता है, बिना किसी केंद्रीय सर्वर से गुज़रे। अगर कीट सही तरीके से लॉन्च होता है, तो आपका पीयर्स इंस्टॉलेशन पूरी तरह से काम कर रहा है।



आपका macOS सिस्टम अब Pears के साथ पीयर-टू-पीयर एप्लिकेशन चलाने और होस्ट करने के लिए तैयार है।



## 6. मैं पीयर्स पर एप्लिकेशन का उपयोग कैसे करूं?

```bash
pear run pear://[KEY]
```
