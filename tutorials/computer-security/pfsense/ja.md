## I.プレゼンテーション



pfSense はフリーでオープンソースのオペレーティングシステムであり、あらゆるコンピューター、専用サーバー、ハードウェアアプライアンスを高性能で高度に設定可能なルーターとファイアウォールに変えます。FreeBSD をベースとし、安定した堅牢なネットワーク アーキテクチャで有名な pfSense は、15 年以上にわたって、企業や地方自治体、要求の厳しいホーム ユーザー向けのオープン ソース ファイアウォールの標準を確立してきました。



pfSenseの主な機能は、何年にもわたり大幅に進化し、新しいバージョンが出るたびに強化されてきました。現在までに、pfSenseは.NET Frameworkを提供しています：





- 最新の直感的で安全なInterfaceウェブInterfaceによる完全な一元管理。
- 高度なNATサポート（NAT-Tを含む）ときめ細かなルール管理を備えた高性能なステートフル・ファイアウォール。
- マルチWANのネイティブサポートにより、インターネット接続のアグリゲーションや冗長化が可能。
- 統合DHCPサーバーとリレー。
- フェイルオーバー用の CARP プロトコルと pfSense クラスタの構成可能性による高可用性。
- 複数の接続またはサーバー間のロードバランシング。
- VPNをフルサポート：IPsec、OpenVPN、WireGuard（L2TPは廃止されました。）
- 設定可能なキャプティブ・ポータルは、特に公共施設やホテル環境でのゲストのアクセス・コントロールに適しています。



pfSenseはまた、Interfaceウェブから直接、透過型プロキシ（Squid）、URLフィルタリング、IDS/IPS（SnortまたはSuricata）などの高度な機能を簡単に追加できる拡張可能なパッケージシステムを備えています。



pfSense は、現在の FreeBSD の勧告に基づき、64 ビットプラットフォーム専用に配布されています。pfSense は、標準的なハードウェア（PC、ラックサーバー）、または Netgate アプライアンスや特定の薄型 x86 ボックスなどの低電力組み込みプラットフォームにインストールできます。



最後に、pfSense は少なくとも 2 つの物理的なネットワーク・インターフェイスを必要とすることを覚えておく価値があります。インフラストラクチャの複雑さ（DMZ、VLAN、複数の WAN）によっては、pfSense の機能を最大限に活用するために、さらに複数のインターフェイスが必要になる場合があります。



## II.画像をダウンロード



このチュートリアルを書いている時点での pfSense の最新安定バージョンは 2.8 (2025 年 6 月リリース) です。あなたのハードウェア環境に合わせた ISO イメージまたはインストールファイルは、公式ウェブサイトから直接ダウンロードできます：





- [pfSenseのダウンロード](https://www.pfsense.org/download/)



ダウンロードポータルでは、：





- アーキテクチャー（一般的に、最新のハードウェアはすべて**AMD64**）。
- イメージタイプ (**Installer USB Memstick** for installation via USB stick, **ISO Installer** for burning or virtual editing).
- 転送速度を最適化するために、最も近いダウンロードミラー。



仮想化環境（Proxmox、VMware ESXi、VirtualBox...）に pfSense を導入したい方のために、**OVA** イメージも用意されています。このすぐに使える仮想マシンはインストールと初期設定を大幅に簡素化します。予想される負荷とネットワーク・トポロジーに応じて、割り当てられたリソース（CPU、RAM、ネットワーク・インターフェース）を調整してください。



インストールする前に、公式ダウンロードページに記載されている**SHA256**を検証して、ダウンロードしたファイルの完全性をチェックすることをお勧めします。これにより、イメージが変更または破損されていないことが保証されます。



## III.インストール



この例では、VirtualBoxを実行する仮想マシン上でインストールを行います。仮想デバイスの管理以外は、物理マシンや他のハイパーバイザー上でも手順はまったく同じです。



### 1.最小ハードウェア要件



標準的なデプロイメントには、.NET Frameworkをお勧めします：





- 1 GB RAM**以上（追加パッケージやZFSサポートを有効にするには2 GB以上を推奨）。
- 8GBのディスクスペース**（プロキシキャッシュ、IDS/IPS、詳細なログをインストールする場合など、より高度な設定には20GB以上が望ましい）。
- 少なくとも2つの仮想ネットワーク・インターフェイス**（1つはWAN用、もう1つはLAN用）。VirtualBoxでは、起動前にVM設定に追加する。



### 2.インストーラー起動



ダウンロードしたISOイメージをVirtualBoxの仮想光学ドライブとしてマウントするか、物理マシンにインストールする場合はUSBキーを挿入します。起動時にブートメニューが表示されます：



オプションを選択しない場合、数秒後に pfSense は自動的にデフォルトのオプションで起動します。Enter**」キーを押すと通常の起動が始まります。



![Image](assets/fr/027.webp)



メインメニューが表示されたら、素早く "**I**"ボタンを押してインストールを開始する。



![Image](assets/fr/001.webp)



### 3.インストーラーの初期設定



最初の画面では、表示フォントや文字エンコードなど、いくつかの地域パラメータを設定できる。これらの設定は特定の場合(非標準キーボード、シリアル画面、東洋言語)に便利です。ほとんどのインストールでは、デフォルト値のままで "**Accept these Settings**"を選択してください。



![Image](assets/fr/002.webp)



### 4.インストール・モードの選択



クイック/簡単インストール**」を選択すると、推奨オプションで自動インストールが実行されます。この方法では、選択したディスクが削除され、デフォルトのパーティショニングで pfSense が設定されます。



![Image](assets/fr/003.webp)



ディスク上のすべてのデータが削除されることを示す警告が表示されます。OK**」で確認してください。



その後、インストーラーが必要なファイルをディスクにコピーします。ハードウェアにもよりますが、数秒から数分かかります。



### 5.コアの選択



インストーラーがカーネル・タイプを選択するよう促したら、"**Standard Kernel**"を選択したままにしてください。この汎用カーネルは、PC、サーバー、VMを問わず、標準的なデプロイメントに最適です。



### 6.インストールの終了と再起動



インストールが完了したら、"**Reboot**"を選択し、新しいpfSenseインスタンスでマシンを再起動します。



**重要**：次回起動時にインストールプログラムの再起動を避けるため、再起動前にISOイメージを削除するか、インストールUSBキーを取り外してください。



## IV.pfSense を初めて起動する



初回起動時、pfSense はネットワークインターフェイス（WAN、LAN、DMZ、VLAN など）を認識し、正しく割り当てるように設定する必要があります。ネットワークカードを注意深く識別することは、Interfaceウェブにアクセスできなくなったり、ファイアウォールが動作しなくなるような設定ミスを避けるために不可欠です。



起動すると、pfSense は利用可能なすべてのネットワーク・インターフェイスを自動的に検出して一覧表示し、それぞれの MAC Address を表示します。これにより、それらの区別が容易になります。



### 1.VLAN



最初の質問は、VLANの設定に関するものです。この段階では、基本的な設定のため、VLANをアクティブにすることはありません。ですから、"**N**"キーを押して、このステップをスキップしてください。



![Image](assets/fr/004.webp)



### 2.WANおよびLAN Interface Assignment



pfSense は、WAN (インターネット・アクセス) に使用する Interface を定義するよう促します。.Interfaceから選択できます：





- Interface名を手動で入力する（仮想環境推奨）。
- A**"を押して自動検出を使用する。このオプションは、ネットワークケーブルが接続され、リンクがアクティブであれば、物理ホスト上で有効です。



![Image](assets/fr/005.webp)



この例では、WANを手動で設定します。Interfaceの正確な名前を入力する。Intel製ボードの場合、FreeBSDでは "**em0**"と表示されることが多いが、ハードウェアによっては異なることがある。例えば、Realtekカードの場合、"**re0**"と表示されることが多い。



![Image](assets/fr/006.webp)



同じ操作を繰り返し、Interface LANを定義する。ここでは "**em1**"とする。



pfSense は、Interface LAN がファイアウォールと NAT の両方をアクティブにして内部ネットワークを保護し、Address 変換を管理することを確認します。



他の物理インターフェイスがある場合は、この段階で追加のインターフェイス（DMZ、Wi-Fi、特定のVLAN）を設定できます。各論理Interfaceには対応するネットワークカードまたは仮想Interfaceが必要です。初期設定ではWANとLANに限定します。



割り当てが完了すると、pfSense は物理インターフェースと割り当てられた役割の対応関係をわかりやすく表示します。Y**」で確認してください。



### 3.PfSenseコンソール



この手順が完了すると、pfSense コンソールのメインメニューが表示されます。このメニューには、ウェブパスワードのリセット、再起動、設定の再読み込み、インターフェースの再割り当てなど、直接管理するための便利なオプションがいくつか用意されています。



![Image](assets/fr/007.webp)



また、Interface LANのデフォルトIP Address（通常は**192.168.1.1**）を含む、現在のネットワーク設定の概要が表示されます。このAddressは、Interfaceの管理ウェブにアクセスするために、ブラウザに入力する必要があります。



**注**：内部ネットワークで異なる Address レンジを使用している場合は、メニューの「**2)** Set Interface(s) IP Address」を選択し、環境に適した IP Address を割り当ててください。



デフォルトでは、Interface WANがDHCP設定されたボックスまたはモデムに接続されている場合、pfSenseは自動的にパブリックIP Addressを取得します。そのため、pfSense Interface LANにクライアントを接続すれば、すぐにインターネットにアクセスすることができます。



## V.Interfaceウェブへの初アクセス



最初のスタートアップが完了し、ネットワーク・インターフェイスが設定されたら、pfSense の Interface ウェブにアクセスして、設定の最終調整と微調整を行うことができます。



### 1.初期接続



LAN ポート（またはハイパーバイザー内の仮想 Interface LAN）にコンピューターを接続し、必要に応じて同じ範囲の IP Address を割り当てます（デフォルトでは、pfSense は LAN 上の DHCP 経由で Address を自動的に配布します）。



ブラウザで、コンソールが示す Address (デフォルトでは `https://192.168.1.1`) にアクセスしてください。pfSense は最初の接続でも HTTPS を要求するので、自己署名証明書の警告が出ると思いますが、例外を追加すれば無視できます。



ログイン画面が表示されます。デフォルトの認証情報は ：




- ユーザー名:** `admin`
- パスワード:** `pfsense`



これらの識別子は、初期設定ウィザード中に変更される。



## VI.セットアップウィザード



初回接続時、pfSenseは**セットアップウィザード**に従うよう促します。すべての重要なパラメーターが正しく定義されていることを確認するために、このウィザードを使用することを強くお勧めします。



### 1.一般パラメーター



あなたができる：




- ホスト名とローカルドメインを指定する（例：`pfsense`と`lan.local`）。
- DNSサーバーを定義し、pfSenseがISPのDNSを使用するか、外部サービス（Cloudflare、OpenDNS、Quad9...）を使用するかを選択します。



### 2.タイムゾーン



ログとスケジュールが一致するように、サイトのタイムゾーンを示してください（例：ヨーロッパ/パリ`）。



### 3.WANコンフィギュレーション



WAN接続の設定 ：




- デフォルトは**DHCP**（ボックスの後ろにいれば十分）。
- 固定IPをお持ちの場合は、パラメータ（固定IP、マスク、ゲートウェイ、DNS）を手動で入力してください。
- 必要に応じて、VLANまたはPPPoE認証（一部のISPで一般的）を定義する。



### 4.LAN設定



ウィザードはデフォルトのLANサブネットの変更を提案します。特定のアドレス指定プランがある場合は、今がそれを適応させるチャンスです。



### 5.管理者パスワードの変更



admin`ユーザーに強力なパスワードを設定して、pfSenseを安全にしましょう。



## VII.検証と更新



ファイアウォールを導入する前に、.NET Frameworkの最新版があることを確認してください：





- システム > アップデート**に移動します。
- 更新チャンネル（通常は**Stable**）を選択します。
- アップデートを確認し、適用する。



セキュリティ・パッチに関する情報を常に入手できるよう、アップデート通知を有効にするのがよいだろう。



## VIII.設定の保存



大きな変更を行う前に、バックアップポリシーを導入する：





- 診断 > バックアップと復元** に進みます。
- 現在のコンフィギュレーション（`config.xml`）のコピーをダウンロードする。
- 安全な場所に保管する（暗号化された外部メディア）。



ミッションクリティカルな環境では、外部サーバーまたはプログラムされたスクリプトによる自動構成バックアップを検討する。



## IX.設置後のベストプラクティス



安心して派遣を終えるために





- ファイアウォール ルールの変更**: デフォルトでは、pfSense は LAN のすべての発信トラフィックを許可し、WAN の受信トラフィックをブロックします。必要に応じてこれらのルールを調整します。
- セキュアリモートアクセスの設定**：必要であれば、WANからVPN経由またはIP制限のみでInterfaceのウェブにアクセスできるようにします。
- 通知を有効にする**：アラート（失敗、更新、エラー）を受信するSMTPサーバーを設定します。
- 便利な拡張機能**をインストールする：例えば、IDS/IPS（Snort、Suricata）、プロキシ（Squid）、DNSフィルタリング（pfBlockerNG）。



pfSense ファイアウォールが稼働し、ネットワークを保護する準備が整いました。その柔軟性と活発なコミュニティのおかげで、将来のニーズ (マルチ WAN、VLAN、サイト間 VPN、キャプティブ ポータルなど) に適応できる強力で拡張性のあるツールを手に入れることができます。



定期的に公式ドキュメント([docs.netgate.com](https://docs.netgate.com/pfsense/en/latest/))を参照し、新機能を発見し、設定が最新で安全であることを確認してください。
