## I.簡報



pfSense 是免費的開放原始碼作業系統，可將任何電腦、專用伺服器或硬體設備轉換成高效能、高配置的路由器與防火牆。pfSense 以 FreeBSD 為基礎，以其穩定、強大的網路架構而聞名，十五年來一直為企業、地方政府及要求嚴苛的家庭使用者設定開放原始碼防火牆的標準。



多年來，其主要功能已大幅演進，並隨著每個新版本而增強。到目前為止，pfSense 提供 .NET 和 .NET 兩種語言：





- 透過現代、直覺且安全的 Interface Web Interface 進行完整的集中化管理。
- 高效能狀態防火牆，具備進階 NAT 支援 (包括 NAT-T) 及細緻的規則管理。
- 本機多重廣域網路支援，可讓網際網路連線聚合或備援。
- 整合式 DHCP 伺服器和中繼。
- 透過 CARP 通訊協定進行故障移轉，並可設定 pfSense 叢集，提供高可用性。
- 在數個連線或伺服器之間進行負載平衡。
- 完整的 VPN 支援：IPsec、OpenVPN 及 WireGuard (取代現已過時的 L2TP)。
- 可設定的擷取式入口網站，用於訪客存取控制，特別是在公共或飯店環境中。



pfSense 還擁有可擴充的套件系統，可輕鬆地直接從 Interface 網路新增進階功能，例如透明代理 (Squid)、URL 過濾或 IDS/IPS (Snort 或 Suricata)。



pfSense 僅發行於 64 位元平台，符合 FreeBSD 目前的建議。它可以安裝在標準硬體 (個人電腦、機架伺服器) 或低功耗的嵌入式平台上，例如 Netgate 裝置或某些低配置的 x86 box，其功能遠比舊有的 Alix box 強大。



最後，值得注意的是 pfSense 至少需要兩個實體網路介面：一個專用於外部區域 (WAN)，另一個專用於內部網路 (LAN)。依據您基礎架構的複雜性 (DMZ、VLAN、多個 WAN)，可能需要數個額外的介面，才能充分利用其功能。



## II.下載圖片



在撰寫本教學時，pfSense 的最新穩定版本是 2.8 (於 2025 年 6 月發行)。您可以直接從官方網站下載 ISO 映像檔或適合您硬體環境的安裝檔案：





- [下載 pfSense](https://www.pfsense.org/download/)



下載入口可讓您選擇 ：





- 架構 (一般而言，**AMD64** 適用於所有現代硬體)。
- 映像類型 (**Installer USB Memstick** 用於透過 USB 隨身碟安裝，**ISO Installer** 用於燒錄或虛擬編輯)。
- 最近的下載鏡像，以最佳化傳輸速度。



對於希望在虛擬化環境 (Proxmox、VMware ESXi、VirtualBox...) 中部署 pfSense 的使用者，我們也提供**OVA**映像檔。這種即開即用的虛擬機器可大幅簡化安裝與初始設定。只需確保根據預期負載和網路拓樸調整分配的資源 (CPU、RAM、網路介面)。



在安裝之前，我們建議透過驗證官方下載頁面上提供的 **SHA256** 來檢查下載檔案的完整性。這可確保映像未被篡改或損壞。



## III.安裝



在本範例中，安裝是在執行 VirtualBox 的虛擬機器上進行。除了虛擬裝置管理之外，安裝程序在實體機器或任何其他管理程序上完全相同。



### 1.最低硬體需求



對於標準部署，我們建議使用 .NET Framework：





- 最低 1 GB RAM**（建議使用 2 GB 或更大的 RAM，以啟用其他套件或 ZFS 支援）。
- 8 GB 磁碟空間** (進階設定最好使用 20 GB 或以上，尤其是安裝代理伺服器快取、IDS/IPS 或詳細記錄時)。
- 至少兩個虛擬網路介面** (一個用於 WAN，一個用於 LAN)。在 VirtualBox 中，啟動前將它們加入虛擬機器設定。



### 2.安裝程式啟動



在 VirtualBox 中將下載的 ISO 映像掛載為虛擬光碟機，如果是在實體機器上安裝，則插入 USB 密鑰。啟動時，會出現開機功能表：



如果您沒有選擇任何選項，pfSense 會在幾秒鐘後以預設選項自動啟動。按下「**Enter**」鍵啟動正常啟動。



![Image](assets/fr/027.webp)



當主功能表出現時，快速按下「**I**」按鈕以啟動安裝。



![Image](assets/fr/001.webp)



### 3.初始安裝程式設定



第一個畫面可讓您設定一些區域參數，例如顯示字型和字元編碼。這些設定在特定情況（非標準鍵盤、序列螢幕、東方語言）下非常有用。對於大多數安裝，請保留預設值，並選擇「**接受這些設定**」。



![Image](assets/fr/002.webp)



### 4.安裝模式選擇



選擇「**快速/簡易安裝**」，以使用建議的選項執行自動安裝。此方法會刪除選取的磁碟，並以預設的磁碟分割設定 pfSense。



![Image](assets/fr/003.webp)



出現警告，表示將刪除磁碟上的所有資料。請以「**OK**」確認。



安裝程式會將必要的檔案複製到磁碟上。視您的硬體而定，這可能需要幾秒到幾分鐘的時間。



### 5.核心選擇



當安裝程式提示您選擇核心類型時，請保留「**標準核心**」選項。此一般核心完全適用於標準部署，無論是在 PC、伺服器或虛擬機器上。



### 6.安裝結束並重啟



安裝完成後，請選擇「**Reboot**」，在新的 pfSense 範例上重新開機。



**重要注意事項**：重新開機前，請移除 ISO 映像或中斷安裝 USB 鑰匙的連接，以避免下次開機時重新啟動安裝程式。



## IV.首次啟動 pfSense



首次啟動時，必須設定 pfSense 以辨識並正確指派其網路介面（WAN、LAN、DMZ、VLAN 等）。小心識別您的網路卡對於避免任何可能導致您無法存取 Interface Web 或防火牆無法運作的設定錯誤是非常重要的。



啟動時，pfSense 會自動偵測並列出所有可用的網路介面，並指出每個介面的 MAC Address。這可讓您輕鬆區分這些介面。



### 1.VLAN



第一個問題是關於 VLAN 的設定。在這個階段，對於基本設定，我們不會啟用任何 VLAN。因此按下「**N**」鍵跳過此步驟。



![Image](assets/fr/004.webp)



### 2.WAN 和 LAN Interface Assignment



pfSense 接著會提示您定義哪個 Interface 會用於 WAN (網際網路存取)。您可以選擇 ：





- 手動輸入 Interface 名稱（建議用於虛擬環境）。
- 按 "**A**"使用自動偵測。此選項對實體主機很有用，前提是您的網路線已連接且連結已啟動。



![Image](assets/fr/005.webp)



在本範例中，我們手動設定 WAN。輸入 Interface 的確切名稱。在 FreeBSD 下，Intel 板卡的名稱通常會是 "**em0**"，但可能會依硬體的不同而有所差異。例如，Realtek 卡通常會顯示為 "**re0**"。



![Image](assets/fr/006.webp)



重複相同的操作，定義 Interface LAN。在此，我們使用「**em1**」。



pfSense 確認 Interface LAN 會同時啟動防火牆和 NAT，以保護您的內部網路，並管理 Address 轉換。



如果您有其他實體介面，可以在此階段設定其他介面 (DMZ、Wi-Fi、特定 VLAN)。每個邏輯 Interface 都需要對應的網卡或虛擬 Interface。對於初始設定，我們將僅限於 WAN 和 LAN。



指派完成後，pfSense 會清楚顯示實體介面與指派角色之間的對應摘要。請以「**Y**」確認。



### 3.PfSense 控制台



此步驟完成後，pfSense 主控台會出現主選單。它提供了多種有用的選項供直接管理，例如重設網頁密碼、重新開機、重新載入設定或重新指定介面。



![Image](assets/fr/007.webp)



您也會看到目前網路設定的摘要，包括 Interface LAN 的預設 IP Address，通常是 **192.168.1.1**。這是您需要在瀏覽器中輸入的 Address，以便存取 Interface 管理網頁。



**註**：如果您的內部網路使用不同的 Address 範圍，請在功能表中選擇「**2)** Set Interface(s) IP Address」，以指定適合您環境的 IP Address。



預設情況下，如果您的 Interface WAN 連接至 DHCP 設定的盒子或數據機，pfSense 會自動擷取一個公共 IP Address。因此，只要將用戶端連接至 pfSense Interface LAN，您就可以立即存取網際網路。



## V.首次進入 Interface 網路



完成初始啟動與網路介面設定後，您就可以存取 pfSense 的 Interface Web 來完成並微調您的設定。



### 1.初始連接



將電腦連接到 LAN 連接埠 (或管理程序中的虛擬 Interface LAN)，必要時為其指定相同範圍內的 IP Address（預設情況下，pfSense 會透過 LAN 上的 DHCP 自動分配 Address）。



在瀏覽器中，前往主控台指示的 Address（預設為 `https://192.168.1.1`）。請注意，即使是第一次連線，pfSense 也要求使用 HTTPS - 因此可能會出現自簽憑證警告，您可以加入例外來忽略它。



出現登入畫面。預設認證為 ：




- 使用者名稱：** `admin`
- 密碼：** `pfsense`



這些識別碼將會在初始組態精靈中修改。



## VI.設定精靈



首次連線時，pfSense 會提示您遵循其 ** 設定精靈**。我們強烈建議您使用它，以確保所有重要參數都已正確定義。



### 1.一般參數



您可以 ：




- 指定主機名稱和本機網域名稱 (例如：`pfsense` 和 `lan.local`)。
- 定義 DNS 伺服器，並選擇 pfSense 應使用 ISP 的 DNS 或外部服務 (Cloudflare、OpenDNS、Quad9...)。



### 2.時區



註明您網站的時區，以便日誌和排程一致 (例如：`歐洲/巴黎`)。



### 3.WAN 設定



設定 WAN 連線 ：




- 預設為 **DHCP**（如果您在方塊後面就足夠）。
- 如果您有固定 IP，請手動輸入參數 (靜態 IP、遮罩、閘道、DNS)。
- 必要時，定義 VLAN 或 PPPoE 驗證（某些 ISP 常見）。



### 4.區域網路配置



精靈建議變更預設 LAN 子網路。如果您有特定的編址計劃，現在正是調整的時候。



### 5.變更管理員密碼



立即為 `admin` 使用者設定強大的密碼，以確保您的 pfSense 安全。



## VII.驗證與更新



在部署防火牆之前，請確定您擁有最新版本的 .NET Framework：





- 移至 ** 系統 > 更新**。
- 選擇更新頻道 (通常為 **穩定**)。
- 檢查更新並套用更新。



啟用更新通知是個好主意，可讓您隨時得知安全修補程式。



## VIII.儲存組態



在進行任何重大變更之前，請先執行備份政策：





- 移至 **診斷 > 備份與還原**。
- 下載目前設定 (`config.xml`)的複本。
- 保存在安全的地方（加密的外部媒體）。



對於關鍵任務環境，可考慮在外部伺服器上或透過程式腳本自動備份組態。



## IX.安裝後的最佳做法



為了讓您安心地結束部署 ：





- 修改防火牆規則**：預設情況下，pfSense 允許 LAN 上的所有傳出流量，並封鎖 WAN 上的傳入流量。請依需要調整這些規則。
- 設定安全遠端存取**：如果需要，請啟用只能從 WAN 透過 VPN 或 IP 限制存取 Interface 網頁。
- 啟用通知**：設定 SMTP 伺服器以接收警示 (故障、更新、錯誤)。
- 安裝有用的擴充套件**：例如 IDS/IPS (Snort, Suricata)、代理 (Squid)、DNS 過濾 (pfBlockerNG)。



您的 pfSense 防火牆現在已開始運作，可隨時保護您的網路。由於其靈活性和活躍的社群，您擁有了一個功能強大、可擴充的工具，能夠適應您未來的需求 (多重廣域網路、VLAN、站點對站點 VPN、captive portal 等)。



請定期參閱官方文件 ([docs.netgate.com](https://docs.netgate.com/pfsense/en/latest/)) 以發現新功能，並確保您的配置是最新且安全的。
