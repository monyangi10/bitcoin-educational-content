## I. 프레젠테이션



pfSense는 무료 오픈 소스 운영 체제로, 모든 컴퓨터, 전용 서버 또는 하드웨어 어플라이언스를 고도로 구성 가능한 고성능 라우터 및 방화벽으로 변환합니다. 안정적이고 강력한 네트워크 아키텍처로 유명한 FreeBSD를 기반으로 하는 pfSense는 15년 이상 기업, 지방 당국 및 까다로운 가정 사용자를 위한 오픈 소스 방화벽의 표준을 정립해 왔습니다.



주요 기능은 수년에 걸쳐 상당히 발전해 왔으며, 새로운 버전이 나올 때마다 기능이 향상되었습니다. 현재까지 pfSense는 다음을 제공합니다:





- 현대적이고 직관적이며 안전한 Interface 웹 Interface을 통해 중앙 집중식으로 완벽하게 관리할 수 있습니다.
- 고급 NAT 지원(NAT-T 포함) 및 세분화된 규칙 관리 기능을 갖춘 고성능 상태 저장 방화벽입니다.
- 기본 멀티-WAN을 지원하여 인터넷 연결의 집계 또는 이중화를 가능하게 합니다.
- 통합 DHCP 서버 및 릴레이.
- 장애 조치를 위한 CARP 프로토콜과 pfSense 클러스터 구성 가능성으로 고가용성을 제공합니다.
- 여러 연결 또는 서버 간의 로드 밸런싱.
- 완벽한 VPN 지원: IPsec, OpenVPN 및 WireGuard(현재 사용되지 않는 L2TP를 대체).
- 특히 공공장소나 호텔 환경에서 게스트 액세스 제어를 위한 구성 가능한 캡티브 포털입니다.



또한 pfSense는 확장 가능한 패키지 시스템을 갖추고 있어 Interface 웹에서 직접 투명 프록시(Squid), URL 필터링 또는 IDS/IPS(Snort 또는 Suricata) 등의 고급 기능을 쉽게 추가할 수 있습니다.



pfSense는 현재 FreeBSD 권장 사항에 따라 64비트 플랫폼 전용으로 배포됩니다. 표준 하드웨어(PC, 랙 서버)에 설치하거나 구형 Alix 박스보다 훨씬 더 강력한 Netgate 어플라이언스 또는 특정 로우 프로파일 x86 박스와 같은 저전력 임베디드 플랫폼에 설치할 수 있습니다.



마지막으로, pfSense에는 외부 영역(WAN) 전용 인터페이스와 내부 네트워크(LAN) 전용 인터페이스 등 최소 두 개의 물리적 네트워크 인터페이스가 필요하다는 점을 기억하세요. 인프라의 복잡성(DMZ, VLAN, 다중 WAN)에 따라 기능을 최대한 활용하려면 몇 가지 추가 인터페이스가 필요할 수 있습니다.



## II. 이미지 다운로드



이 튜토리얼을 작성하는 시점의 최신 안정 버전은 2.8(2025년 6월 출시)입니다. 공식 웹사이트에서 하드웨어 환경에 맞게 조정된 ISO 이미지 또는 설치 파일을 직접 다운로드할 수 있습니다:





- [pfSense 다운로드](https://www.pfsense.org/download/)



다운로드 포털에서 선택할 수 있습니다:





- 아키텍처(일반적으로 모든 최신 하드웨어의 경우 **AMD64**).
- 이미지 유형(USB 스틱을 통한 설치의 경우 **설치 프로그램 USB 멤스틱**, 굽기 또는 가상 편집의 경우 **ISO 설치 프로그램**).
- 가장 가까운 다운로드 미러로 전송 속도를 최적화합니다.



가상화된 환경(Proxmox, VMware ESXi, VirtualBox...)에 pfSense를 배포하려는 경우 **OVA** 이미지도 사용할 수 있습니다. 바로 사용할 수 있는 이 가상 머신은 설치 및 초기 구성을 크게 간소화합니다. 예상되는 부하와 네트워크 토폴로지에 따라 할당된 리소스(CPU, RAM, 네트워크 인터페이스)를 조정하기만 하면 됩니다.



설치하기 전에 공식 다운로드 페이지에 제공된 **SHA256**을 확인하여 다운로드한 파일의 무결성을 확인하는 것이 좋습니다. 이렇게 하면 이미지가 변경되거나 손상되지 않았는지 확인할 수 있습니다.



## III. 설치



이 예에서는 VirtualBox를 실행하는 가상 머신에서 설치가 수행됩니다. 이 절차는 가상 장치 관리를 제외하고는 물리적 시스템이나 다른 하이퍼바이저에서도 완전히 동일하게 유지됩니다.



### 1. 최소 하드웨어 요구 사항



표준 배포의 경우 권장합니다:





- 최소 1GB RAM(**추가 패키지 또는 ZFS 지원을 사용하려면 2GB 이상 권장**).
- 8GB 디스크 공간(특히 프록시 캐시, IDS/IPS 또는 상세 로그를 설치하는 경우 고급 구성의 경우 20GB 이상을 권장합니다).
- 최소 2개의 가상 네트워크 인터페이스(하나는 WAN용, 하나는 LAN용). VirtualBox에서 시작하기 전에 VM 설정에 추가합니다.



### 2. 설치 프로그램 시작



다운로드한 ISO 이미지를 VirtualBox에서 가상 광학 드라이브로 마운트하거나 물리적 머신에 설치하는 경우 USB 키를 삽입합니다. 시작하면 부팅 메뉴가 나타납니다:



옵션을 선택하지 않으면 몇 초 후 기본 옵션으로 pfSense가 자동으로 시작됩니다. "**Enter**" 키를 누르면 정상적으로 시작됩니다.



![Image](assets/fr/027.webp)



메인 메뉴가 나타나면 '**I**' 버튼을 빠르게 눌러 설치를 시작하세요.



![Image](assets/fr/001.webp)



### 3. 초기 설치 프로그램 설정



첫 번째 화면에서는 표시 글꼴 및 문자 인코딩과 같은 몇 가지 지역 매개변수를 설정할 수 있습니다. 이러한 설정은 특정 경우(비표준 키보드, 직렬 화면, 동양 언어)에 유용합니다. 대부분의 설치에서는 기본값을 유지하고 "**이 설정 수락**"을 선택합니다.



![Image](assets/fr/002.webp)



### 4. 설치 모드 선택



권장 옵션으로 자동 설치를 실행하려면 "**빠른/쉬운 설치**"를 선택합니다. 이 방법은 선택한 디스크를 삭제하고 기본 파티셔닝으로 pfSense를 구성합니다.



![Image](assets/fr/003.webp)



디스크의 모든 데이터가 삭제된다는 경고가 표시됩니다. "**OK**"로 확인합니다.



그런 다음 설치 프로그램이 필요한 파일을 디스크에 복사합니다. 하드웨어에 따라 몇 초에서 몇 분 정도 걸릴 수 있습니다.



### 5. 핵심 선택



설치 관리자에서 커널 유형을 선택하라는 메시지가 표시되면 "**표준 커널**"을 선택한 상태로 둡니다. 이 일반 커널은 PC, 서버 또는 가상 머신 등 표준 배포에 완벽하게 적합합니다.



### 6. 설치 종료 후 다시 시작



설치가 완료되면 "**재부팅**"을 선택하여 새 pfSense 인스턴스에서 컴퓨터를 재부팅합니다.



**중요 참고**: 다음에 부팅할 때 설치 프로그램을 다시 시작하지 않으려면 재부팅하기 전에 ISO 이미지를 제거하거나 설치 USB 키를 분리하세요.



## IV. PfSense 처음 시작하기



처음 시작할 때 네트워크 인터페이스(WAN, LAN, DMZ, VLAN 등)를 인식하고 올바르게 할당하도록 pfSense를 구성해야 합니다. 네트워크 카드를 주의 깊게 식별하여 구성 오류로 인해 Interface 웹에 대한 액세스 권한이 박탈되거나 방화벽이 작동하지 않을 수 있는 상황을 방지해야 합니다.



PfSense는 실행 시 사용 가능한 모든 네트워크 인터페이스를 자동으로 감지하여 나열하고 각 인터페이스에 대한 MAC Address을 표시합니다. 따라서 인터페이스를 쉽게 구분할 수 있습니다.



### 1. VLAN



첫 번째 질문은 VLAN 구성에 관한 것입니다. 이 단계에서는 기본 구성의 경우 VLAN을 활성화하지 않습니다. 따라서 이 단계를 건너뛰려면 "**N**" 키를 누르세요.



![Image](assets/fr/004.webp)



### 2. WAN 및 LAN Interface Assignment



그러면 pfSense에서 WAN(인터넷 액세스)에 사용할 Interface을 정의하라는 메시지를 표시합니다. 다음 중에서 선택할 수 있습니다:





- Interface 이름을 수동으로 입력합니다(가상 환경의 경우 권장).
- "**A**"를 눌러 자동 감지를 사용합니다. 이 옵션은 네트워크 케이블이 연결되어 있고 링크가 활성화되어 있는 경우 물리적 호스트에서 유용합니다.



![Image](assets/fr/005.webp)



이 예에서는 WAN을 수동으로 구성합니다. Interface의 정확한 이름을 입력합니다. 인텔 보드의 경우 FreeBSD에서 이름이 "**em0**"인 경우가 많지만 하드웨어에 따라 다를 수 있습니다. 예를 들어, Realtek 카드는 "**re0**"로 표시되는 경우가 많습니다.



![Image](assets/fr/006.webp)



동일한 작업을 반복하여 Interface LAN을 정의합니다. 여기서는 "**em1**"를 사용합니다.



pfSense는 Interface LAN이 방화벽과 NAT를 모두 활성화하여 내부 네트워크를 보호하고 Address 번역을 관리하는지 확인합니다.



다른 물리적 인터페이스가 있는 경우 이 단계에서 추가 인터페이스(DMZ, Wi-Fi, 특정 VLAN)를 구성할 수 있습니다. 각 논리적 Interface에는 해당 네트워크 카드 또는 가상 Interface가 필요합니다. 초기 구성에서는 WAN과 LAN으로 한정하겠습니다.



할당이 완료되면 pfSense는 물리적 인터페이스와 할당된 역할 간의 대응에 대한 명확한 요약을 표시합니다. "**Y**"로 확인합니다.



### 3. PfSense 콘솔



이 단계가 완료되면 pfSense 콘솔 메인 메뉴가 나타납니다. 웹 비밀번호 재설정, 재부팅, 구성 다시 로드 또는 인터페이스 재할당 등 직접 관리를 위한 몇 가지 유용한 옵션을 제공합니다.



![Image](assets/fr/007.webp)



또한 Interface LAN의 기본 IP Address(보통 **192.168.1.1**)를 포함한 현재 네트워크 설정 요약도 확인할 수 있습니다. Interface 관리 웹에 액세스하기 위해 브라우저에 입력해야 하는 Address입니다.



**참고**: 내부 네트워크에서 다른 Address 범위를 사용하는 경우, 메뉴에서 "**2)** Interface(들) IP Address 설정"을 선택하여 사용자 환경에 적합한 IP Address을 할당하세요.



기본적으로 Interface WAN이 DHCP가 구성된 박스 또는 모뎀에 연결되어 있으면 pfSense는 자동으로 공용 IP Address을 검색합니다. 따라서 클라이언트를 pfSense Interface LAN에 연결하면 즉시 인터넷에 액세스할 수 있습니다.



## V. Interface 웹에 처음 액세스



초기 시작이 완료되고 네트워크 인터페이스가 구성되면 pfSense의 Interface 웹에 액세스하여 구성을 마무리하고 미세 조정할 수 있습니다.



### 1. 초기 연결



컴퓨터를 LAN 포트(또는 하이퍼바이저의 가상 Interface LAN)에 연결하고 필요한 경우 동일한 범위의 IP Address를 할당합니다(기본적으로 pfSense는 LAN에서 DHCP를 통해 자동으로 Address를 배포합니다).



브라우저에서 콘솔에 표시된 Address(기본값은 `https://192.168.1.1`)으로 이동합니다. PfSense는 첫 번째 연결 시에도 HTTPS를 요구하므로 자체 서명 인증서 경고가 표시될 수 있으며, 예외를 추가하여 무시할 수 있습니다.



로그인 화면이 나타납니다. 기본 자격 증명은 다음과 같습니다:




- 사용자 이름:** `admin`
- 비밀번호:** `pfsense`



이러한 식별자는 초기 구성 마법사 중에 수정됩니다.



## VI. 설정 마법사



처음 연결하면 pfSense는 **설정 마법사**를 따르라는 메시지를 표시합니다. 모든 필수 매개변수가 올바르게 정의되었는지 확인하기 위해 이 마법사를 사용하는 것을 적극 권장합니다.



### 1. 일반 매개 변수



할 수 있습니다:




- 호스트 이름과 로컬 도메인을 지정합니다(예: `pfsense` 및 `lan.local`).
- DNS 서버를 정의하고 pfSense가 ISP의 DNS를 사용할지 외부 서비스(Cloudflare, OpenDNS, Quad9...)를 사용할지 선택합니다.



### 2. 시간대



로그와 일정이 일관되도록 사이트의 표준 시간대를 표시합니다(예: `유럽/파리`).



### 3. WAN 구성



WAN 연결을 구성합니다:




- 기본값은 **DHCP**입니다(박스 뒤에 있는 경우 이 정도면 충분합니다).
- 고정 IP를 사용하는 경우 매개변수(고정 IP, 마스크, 게이트웨이, DNS)를 수동으로 입력합니다.
- 필요한 경우 VLAN 또는 PPPoE 인증(일부 ISP에서 공통)을 정의합니다.



### 4. LAN 구성



마법사가 기본 LAN 서브넷을 변경할 것을 제안합니다. 특정 주소 지정 계획이 있다면 지금 바로 변경할 수 있습니다.



### 5. 관리자 비밀번호 변경



'admin' 사용자에 대한 강력한 비밀번호를 즉시 설정하여 pfSense를 보호하세요.



## VII. 확인 및 업데이트



방화벽을 배포하기 전에 최신 버전의 방화벽이 있는지 확인하세요:





- 시스템 > 업데이트**로 이동합니다.
- 업데이트 채널을 선택합니다(보통 **안정**).
- 업데이트를 확인하고 적용하세요.



업데이트 알림을 사용 설정하여 보안 패치에 대한 정보를 지속적으로 받는 것이 좋습니다.



## VIII. 구성 저장하기



중요한 변경을 하기 전에 백업 정책을 구현하세요:





- 진단 > 백업 및 복원**으로 이동합니다.
- 현재 구성의 사본(`config.xml`)을 다운로드합니다.
- 안전한 장소(암호화된 외부 미디어)에 보관하세요.



미션 크리티컬한 환경의 경우 외부 서버 또는 프로그래밍된 스크립트를 통한 자동 구성 백업을 고려하세요.



## IX. 설치 후 모범 사례



안심하고 배포를 종료할 수 있습니다:





- 방화벽 규칙 수정**: 기본적으로 pfSense는 LAN에서 나가는 모든 트래픽을 허용하고 WAN에서 들어오는 트래픽을 차단합니다. 필요에 따라 이러한 규칙을 조정하세요.
- 보안 원격 액세스 구성**: 필요한 경우 VPN을 통해서만 또는 IP 제한을 통해 WAN에서 Interface 웹에 액세스할 수 있도록 설정합니다.
- 알림 사용**: 알림(실패, 업데이트, 오류)을 수신하도록 SMTP 서버를 구성합니다.
- 유용한 **확장 프로그램**을 설치하세요: 예: IDS/IPS(Snort, Suricata), 프록시(Squid), DNS 필터링(pfBlockerNG).



이제 pfSense 방화벽이 실행 중이며 네트워크를 보호할 준비가 되었습니다. 유연성과 활발한 커뮤니티 덕분에 향후 요구 사항(멀티-WAN, VLAN, 사이트 간 VPN, 캡티브 포털 등)에 맞게 조정할 수 있는 강력하고 확장 가능한 도구를 갖추게 되었습니다.



공식 문서([docs.netgate.com](https://docs.netgate.com/pfsense/en/latest/)를 정기적으로 참조하여 새로운 기능을 발견하고 구성이 최신 상태이고 안전한지 확인하세요.
