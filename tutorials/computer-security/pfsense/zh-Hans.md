## I.介绍



pfSense 是一款免费的开源操作系统，可将任何计算机、专用服务器或硬件设备转变为高性能、高配置的路由器和防火墙。pfSense 基于 FreeBSD，以其稳定、强大的网络架构而闻名，15 年来一直为企业、地方当局和要求苛刻的家庭用户设定开源防火墙标准。



多年来，它的主要功能有了长足的发展，并在每个新版本中都得到了增强。迄今为止，pfSense 提供了 .NET 和 .NET 两种语言：





- 通过现代、直观和安全的 Interface 网络 Interface 进行完整的集中管理。
- 高性能状态防火墙，支持高级 NAT（包括 NAT-T）和细粒度规则管理。
- 本地多广域网支持，实现互联网连接的聚合或冗余。
- 集成 DHCP 服务器和中继器。
- 通过 CARP 协议进行故障切换，并可配置 pfSense 集群，实现高可用性。
- 在多个连接或服务器之间实现负载平衡。
- 全面支持 VPN：IPsec、OpenVPN 和 WireGuard（取代现已过时的 L2TP）。
- 可配置的专属门户，用于访客访问控制，尤其是在公共场所或酒店环境中。



pfSense 还具有可扩展的软件包系统，可直接从 Interface web 轻松添加透明代理（Squid）、URL 过滤或 IDS/IPS（Snort 或 Suricata）等高级功能。



根据 FreeBSD 目前的建议，pfSense 仅用于 64 位平台。它可以安装在标准硬件（个人电脑、机架式服务器）或低功耗嵌入式平台上，如 Netgate 设备或某些低配置 x86 盒子，它们比旧式的 Alix 盒子功能强大得多。



最后，值得注意的是，pfSense 至少需要两个物理网络接口：一个专用于外部区域（广域网），一个专用于内部网络（局域网）。根据基础设施的复杂程度（DMZ、VLAN、多个广域网），可能还需要几个额外的接口才能充分利用其功能。



## II.下载图片



编写本教程时，pfSense 的最新稳定版本是 2.8（2025 年 6 月发布）。您可以直接从官方网站下载 ISO 映像或适合您硬件环境的安装文件：





- [下载 pfSense](https://www.pfsense.org/download/)



下载门户允许您选择 .NET Framework 3.0 或更高版本：





- 架构（所有现代硬件一般都采用**AMD64**）。
- 映像类型（**安装程序 USB 记忆棒**用于通过 USB 记忆棒安装，**ISO 安装程序**用于刻录或虚拟编辑）。
- 最近的下载镜像，以优化传输速度。



对于希望在虚拟环境（Proxmox、VMware ESXi、VirtualBox......）中部署pfSense的用户，我们还提供了**OVA**镜像。这种随时可用的虚拟机大大简化了安装和初始配置。只需确保根据预期负载和网络拓扑结构调整分配的资源（CPU、内存、网络接口）即可。



安装前，我们建议通过验证官方下载页面提供的 **SHA256** 来检查下载文件的完整性。这样可以确保镜像文件未被篡改或损坏。



## III.安装



在本例中，安装是在运行 VirtualBox 的虚拟机上进行的。除虚拟设备管理外，在物理机或任何其他管理程序上的安装过程完全相同。



### 1.最低硬件要求



对于标准部署，我们建议使用 .NET Framework 2.0：





- 最低 1 GB 内存**（建议使用 2 GB 或更大内存，以支持更多软件包或 ZFS）。
- 8 GB 磁盘空间**（对于更高级的配置，尤其是安装代理缓存、IDS/IPS 或详细日志时，最好使用 20 GB 或更大的空间）。
- 至少两个虚拟网络接口**（一个用于广域网，一个用于局域网）。在 VirtualBox 中，启动前将它们添加到虚拟机设置中。



### 2.安装程序启动



在 VirtualBox 中将下载的 ISO 映像挂载为虚拟光驱，如果是在物理机上安装，则插入 USB 密钥。启动时，会出现一个启动菜单：



如果不选择任何选项，pfSense 将在几秒钟后自动以默认选项启动。按 "**Enter**"键启动正常程序。



![Image](assets/fr/027.webp)



出现主菜单后，快速按下 "**I**"按钮启动安装。



![Image](assets/fr/001.webp)



### 3.初始安装程序设置



在第一个屏幕中，您可以设置一些区域参数，如显示字体和字符编码。这些设置在特定情况下（非标准键盘、串行屏幕、东方语言）非常有用。对于大多数安装，请保留默认值并选择 "**接受这些设置**"。



![Image](assets/fr/002.webp)



### 4.安装模式选择



选择 "**快速/简易安装**"，使用推荐选项运行自动安装。此方法会删除所选磁盘，并使用默认分区配置 pfSense。



![Image](assets/fr/003.webp)



出现警告，表示磁盘上的所有数据将被删除。用 "**OK**"确认。



然后，安装程序会将必要的文件复制到磁盘上。根据硬件情况，这可能需要几秒到几分钟的时间。



### 5.核心选择



当安装程序提示您选择内核类型时，请选择 "**标准内核**"。这种通用内核非常适合标准部署，无论是在个人电脑、服务器还是虚拟机上。



### 6.安装结束并重新启动



安装完成后，选择 "**重新启动**"，在新的 pfSense 实例上重新启动机器。



**重要注意事项**：在重新启动之前，请移除 ISO 映像或拔出安装 USB 密钥，以避免下次启动时重新启动安装程序。



## IV.首次启动 pfSense



首次启动时，必须对 pfSense 进行配置，以识别并正确分配其网络接口（WAN、LAN、DMZ、VLAN 等）。仔细识别网卡对于避免出现任何配置错误至关重要，因为这些错误可能会导致无法访问 Interface Web 或防火墙失效。



启动时，pfSense 会自动检测并列出所有可用的网络接口，同时显示每个接口的 MAC Address。这样就很容易区分它们了。



### 1.VLAN



第一个问题涉及 VLAN 的配置。在此阶段，对于基本配置，我们不会激活任何 VLAN。因此请按 "**N**"键跳过这一步。



![Image](assets/fr/004.webp)



### 2.广域网和局域网 Interface Assignment



然后，pfSense 会提示您定义哪个 Interface 将用于广域网（互联网接入）。您可以在 ：





- 手动输入 Interface 名称（建议用于虚拟环境）。
- 按 "**A**"使用自动检测。该选项在物理主机上非常有用，前提是网线已连接且链路处于活动状态。



![Image](assets/fr/005.webp)



在本例中，我们手动配置广域网。输入 Interface 的确切名称。在 FreeBSD 下，英特尔板卡的名称通常为 "**em0**"，但也可能因硬件而异。例如，Realtek 网卡通常显示为 "**re0**"。



![Image](assets/fr/006.webp)



重复相同操作，定义 Interface 局域网。这里我们使用 "**em1**"。



pfSense 确认 Interface 局域网同时激活防火墙和 NAT，以保护内部网络并管理 Address 转换。



如果有其他物理接口，可以在此阶段配置其他接口（DMZ、Wi-Fi、特定 VLAN）。每个逻辑 Interface 都需要相应的网卡或虚拟 Interface。对于初始配置，我们将仅限于广域网和局域网。



分配完成后，pfSense 会显示物理接口与分配角色之间的清晰对应关系摘要。用 "**Y**"确认。



### 3.PfSense 控制台



完成这一步后，pfSense 控制台主菜单就会出现。它为直接管理提供了几个有用的选项，如重置网络密码、重启、重新加载配置或重新分配接口。



![Image](assets/fr/007.webp)



您还将看到当前网络设置的摘要，包括 Interface 局域网的默认 IP Address，通常为 **192.168.1.1**。这是您需要在浏览器中输入的 Address，以便访问 Interface 管理网络。



**注意**：如果您的内部网络使用不同的 Address 范围，请在菜单中选择 "**2)** 设置 Interface(s) IP Address"，以分配适合您环境的 IP Address。



默认情况下，如果您的 Interface 广域网与 DHCP 配置的盒子或调制解调器相连，pfSense 将自动获取一个公共 IP Address。因此，只要将客户端连接到 pfSense Interface LAN，就能立即访问互联网。



## V.首次访问 Interface 网站



完成初始启动和网络接口配置后，您就可以访问 pfSense 的 Interface 网页，最终确定并微调您的配置。



### 1.初始连接



将计算机连接到 LAN 端口（或管理程序中的虚拟 Interface LAN），必要时为其分配相同范围内的 IP Address（默认情况下，pfSense 会通过 DHCP 在 LAN 上自动分配 Address）。



在浏览器中，转到控制台指示的 Address（默认为 `https://192.168.1.1`）。请注意，即使是首次连接，pfSense 也要求使用 HTTPS，因此可能会出现自签名证书警告，你可以通过添加异常来忽略它。



出现登录屏幕。默认凭据为 ：




- 用户名：** `admin`
- 密码：** `pfsense`



这些标识符将在初始配置向导中进行修改。



## VI.设置向导



首次连接时，pfSense 会提示您按照其**设置向导**进行操作。我们强烈建议您使用该向导，以确保正确定义所有重要参数。



### 1.一般参数



您可以 ：




- 指定主机名和本地域（例如：`pfsense`和`lan.local`）。
- 定义 DNS 服务器，选择 pfSense 应使用 ISP 的 DNS 还是外部服务（Cloudflare、OpenDNS、Quad9......）。



### 2.时区



注明网站时区，以便日志和时间表保持一致（例如 "欧洲/巴黎"）。



### 3.广域网配置



配置广域网连接 ：




- 默认为 **DHCP**（如果您在盒子后面就足够了）。
- 如果使用固定 IP，请手动输入参数（静态 IP、掩码、网关、DNS）。
- 如有必要，请定义 VLAN 或 PPPoE 身份验证（在某些 ISP 中很常见）。



### 4.局域网配置



向导建议更改默认局域网子网。如果你有特定的寻址计划，现在正是调整的时候。



### 5.更改管理员密码



立即为 "admin "用户设置一个强大的密码，以确保 pfSense 的安全。



## VII.核查和更新



在部署防火墙之前，请确保您拥有最新版本的 .NET Framework 4.0：





- 转到 **系统 > 更新**。
- 选择更新通道（通常为 **稳定**）。
- 检查并应用更新。



最好启用更新通知，以便随时了解安全补丁。



## VIII.保存配置



在进行任何重大更改之前，请执行备份策略：





- 转到 ** 诊断 > 备份和还原**。
- 下载当前配置 (`config.xml`) 的副本。
- 将其保存在安全的地方（加密的外部介质）。



对于关键任务环境，可考虑在外部服务器上或通过编程脚本自动备份配置。



## IX.安装后的最佳做法



安心结束部署 ：





- 修改防火墙规则**：默认情况下，pfSense 允许 LAN 上的所有传出流量，并阻止 WAN 上的传入流量。根据需要调整这些规则。
- 配置安全远程访问**：如果需要，只能通过 VPN 或 IP 限制从广域网访问 Interface 网络。
- 启用通知**：配置 SMTP 服务器以接收警报（故障、更新、错误）。
- 安装有用的扩展**：例如，IDS/IPS（Snort、Suricata）、代理（Squid）、DNS 过滤（pfBlockerNG）。



您的 pfSense 防火墙现已启动并开始运行，随时准备保护您的网络。得益于其灵活性和活跃的社区，您将拥有一个功能强大、可扩展的工具，可以适应您未来的需求（多广域网、VLAN、站点到站点 VPN、捕获式门户等）。



定期查阅官方文档（[docs.netgate.com](https://docs.netgate.com/pfsense/en/latest/)）以发现新功能，并确保您的配置是最新和安全的。
