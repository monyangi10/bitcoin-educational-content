Trezorデバイスは、もともとBitcoin Walletを保護するために設計されたハードウェアウォレットですが、ウェブ上での強力な認証のための高度なオプションも備えています。**U2F**および**FIDO2**プロトコルとの互換性により、パスワードのみに依存することなく、オンラインアカウントへのアクセスを保護することができます。



U2F（*Universal 2nd Factor*）プロトコルは、2014年にGoogleとYubicoによって導入され、その後FIDOアライアンスによって標準化された。ログイン時に2つ目の物理的認証要素（2FA）を追加することができる。このプロトコルを有効にすると、従来のパスワードに加えて、ユーザーはアカウントに接続しようとするたびにTrezorのボタンを押して承認する必要がある。この意味で、TrezorはYubikeyと同様の働きをする。



この方式は非対称暗号方式に基づいており、秘密データが送信されないため、フィッシング攻撃や傍受攻撃が効かなくなる。U2Fは現在、多くのオンラインサービスでサポートされている。



二要素認証を可能にするU2Fに加え、TrezorsはU2Fを進化させたFIDO2（*Fast IDentity Online 2.0*）もサポートしている。これは2018年に標準化された認証プロトコルで、U2Fのロジックを拡張し、パスワードを完全に置き換えることを目指している。これは2つのコンポーネントに基づいている： *WebAuthn*（ブラウザ側）と*CTAP2*（物理キー側）である。FIDO2は、"パスワードレス "認証を可能にする。ユーザーは、一意の暗号トークンとして機能するTrezorデバイスのみを介して自分自身を識別し、追加のパスワードは必要ない。このプロトコルは現在、多くのオンラインサービス、特に企業向けサービスと互換性があります。



パスワードレス*」機能に加え、FIDO2はU2Fと同様の方法で二要素認証も可能にする。



FIDO2はまた、常駐認証情報という概念を導入しています。つまり、接続を可能にする秘密鍵とユーザーの識別情報の両方を含む、Trezorに直接保存された識別子です。この仕組みにより、IDもパスワードも入力することなく、Trezorに接続するだけでアクセスを確認できるため、パスワード不要の認証が可能となる。逆に、従来型の非居住者認証情報は、秘密鍵のみをデバイスに保存し、ユーザーIDはサーバー側に保存されたままであるため、接続のたびに入力する必要がある。Trezorでの保存方法は後述する。



このチュートリアルでは、二要素認証のためにU2FまたはFIDO2を有効にする方法と、パスワードなしでTrezorから直接アカウントにアクセスするためのFIDO2の設定方法について説明します。



**注：** U2FはTrezorの全モデルと互換性があるが、FIDO2はSafe 3、Safe 5、Model Tでのみサポートされており、Model Oneではサポートされていない。



## TrezorでU2F/FIDO2を2FAに使用する



始める前に、TrezorでBitcoin Walletをセットアップしたことを確認してください。二要素認証のU2FとFIDO2で使用されるキーはこのMnemonicから派生するため、Mnemonicを正しく保存することが重要です。Trezorを紛失または破損した場合、別のTrezorデバイスでMnemonicのフレーズを入力することで、鍵へのアクセスを回復できます（次のセクションで説明するように、"*パスワードなし*"モードのFIDO2認証情報の場合、seedだけでは不十分であることに注意してください）。



Trezorをコンピュータに接続し、ロックを解除してください。



![Image](assets/fr/01.webp)



二要素認証で保護したいアカウントにアクセスします。例えば、Bitwardenのアカウントを使う。通常、サービス設定の「*authentication*」、「*security*」、「*login*」または「*password*」タブの下に2FAオプションがあります。



![Image](assets/fr/02.webp)



二要素認証専用のセクションで、「*Passkey*」オプションを選択します（用語は使用しているサイトによって異なる場合があります）。



![Image](assets/fr/03.webp)



現在のパスワードの確認を求められることがよくあります。



![Image](assets/fr/04.webp)



セキュリティ・キーに分かりやすいように名前を付け、"*Read Key*"をクリックします。



![Image](assets/fr/05.webp)



Trezorの画面にアカウント詳細が表示されます。画面をタッチするか、ボタンを押して確認します。PINコードの確認も求められます。



![Image](assets/fr/06.webp)



このセキュリティキーを登録する。



![Image](assets/fr/07.webp)



今後、アカウントに接続する際には、通常のパスワードに加えて、Trezorの接続が求められます。



![Image](assets/fr/08.webp)



その後、Trezorの画面を押して認証を確認することができます。



![Image](assets/fr/09.webp)



二要素認証にHardware Wallet Trezorを使用する利点は、Mnemonicフレーズのおかげでキーを簡単に復元できることです。この基本的なバックアップに加えて、2FAを有効化した各サービスから提供される緊急コードを使用することもできます。この緊急コードを使えば、セキュリティ・キーを紛失してもアカウントに接続できる。そのため、必要に応じて2FAの代わりに接続することができます。



例えばBitwardenでは、"*View recovery code*"をクリックすることで、このコードにアクセスすることができます。



![Image](assets/fr/10.webp)



このコードは、一緒に盗まれるのを防ぐため、メインのパスワードを保存している場所とは別の場所に保管することをお勧めします。例えば、パスワードがパスワード・マネージャーに保存されている場合、2FA緊急コードは紙に書いて別に保管してください。



この方法は、2FA認証用のTrezorを紛失した場合に、2つのレベルのバックアップを提供します：すべてのアカウント用のMnemonicフレーズを使用した1つ目のバックアップと、緊急コードを使用した各アカウント固有の2つ目のバックアップです。ただし、Mnemonicと緊急コードの役割を混同しないことが重要です：




- 12語または24語のMnemonicフレーズは、すべてのアカウントの2FAに使用されるキーだけでなく、Trezorでセキュリティで保護されたビットコインにもアクセスできます；
- 緊急コードを使用すると、該当するアカウントのみ（この例ではBitwardenのみ）で一時的に2FAリクエストをバイパスすることができます。



## TrezorでFIDO2を使う



二要素認証に加えて、FIDO2は「パスワードレス」認証、つまりサイトにログオンする際にパスワードを入力する必要のない認証も可能にします。Trezorをコンピュータに接続するだけで、この方法でセキュアアカウントにアクセスできます。この機能の設定方法は以下の通りです。



始める前に、Bitcoin WalletをTrezorにセットアップしたことを確認してください。FIDO2の "*passwordless*"識別子はseedで暗号化されるため、Mnemonicを保存することが重要です（これらの識別子を正しく保存する方法は次のセクションで説明します）。



Trezorをコンピュータに接続し、ロックを解除します。



![Image](assets/fr/01.webp)



安全性を確保したいアカウントに「*パスワードなし*」モードでアクセスします。例としてBitwardenアカウントを使用します。このオプションは通常、サービス設定の「*認証*」「*セキュリティ*」「*パスワード*」タブにあります。



例えばBitwardenでは、このオプションは「*マスターパスワード*」タブの下にあります。FIDO2 による認証を有効にするには、"*Turn on*"をクリックしてください。



![Image](assets/fr/11.webp)



パスワードの確認を求められることがよくあります。



![Image](assets/fr/12.webp)



Trezorの画面にアカウント詳細が表示されます。画面をタッチするか、ボタンを押して確認してください。PINコードの確認も必要です。



![Image](assets/fr/13.webp)



サイト上で、セキュリティ・キーを記憶させるために名前を追加し、「*オンにする*」をクリックする。



![Image](assets/fr/14.webp)



その後、鍵が正しく機能しているかどうかを確認するため、本人確認を求められます。



![Image](assets/fr/15.webp)



今後、アカウントにログインする際、EメールAddressやログインを入力する必要はありません。ログインフォームの物理キーで認証するボタンをクリックするだけです。



![Image](assets/fr/16.webp)



Hardware Wallet暗証番号を入力し、Trezorへの接続を確認します。



![Image](assets/fr/17.webp)



パスワードを入力することなく、アカウントに接続されます。



![Image](assets/fr/18.webp)



**TrezorでFIDO2による "*パスワードレス*"認証を有効にしても、オンラインアカウントのメインパスワードはログイン目的で有効であることにご注意ください**。



## FIDO2クレデンシャルを保存する（クレデンシャル居住者）



二要素認証のためにFIDO2またはU2Fを使用している場合、つまりTrezorによる2FA認証に加えてパスワードが必要なアカウントにログインする場合、Mnemonicフレーズだけで鍵へのアクセスを取得することができます。ただし、前のセクションで説明したように、FIDO2を "*passwordless*"モードで使用している場合は、これらの認証情報を暗号化するMnemonicフレーズのバックアップに加えて、FIDO認証情報のコピーを作成する必要があります。



そのためには、Pythonがインストールされたコンピューターが必要です。ターミナルを開き、以下のコマンドを実行して必要なTrezorソフトウェアをインストールする：



```shell
pip3 install --upgrade trezor
```



TrezorをUSBでコンピュータに接続し、PINコードを使用してロックを解除します。



![Image](assets/fr/01.webp)



Trezorに保存されているFIDO2識別子のリストを取得するには、以下のコマンドを実行します：



```shell
trezorctl fido credentials list
```



Trezorでエクスポートを確認する。



![Image](assets/fr/19.webp)



FIDO2ログイン情報が端末に表示されます。例えば、私のBitwardenアカウントでは、このような情報が表示されました：



```shell
WebAuthn credential at index 0:
Relying party ID:       vault.bitwarden.com
Relying party name:     Bitwarden
User ID:                6e315ebabc8b6945a253b1c50116538d
User name:              tutoplanbnetwork@proton.me

Credential ID:          f1d00200a020a736356d0ceb7ce8b7655b39c399d8111b620bbbbfc78a51add31475e6acd9a68f77f0a6b12a20c7a41412c488787d41e6ee0bdbf3bb99973c9637d21d3a060808143dd228e0831bbb883fb3afedd3f70596a9f6b98f00703244b76260099a9c044346bf6266d3cb9d90db6fc7cde1142b11c5c8ea
```



これらの情報をすべてコピーし、テキストファイルに保存する。あなたがFIDO2でこれらのサービスを使用していることを明らかにする以外、このバックアップに関連する重大なリスクはありません。**クレデンシャルID**はWalletのseedを使用して暗号化されます。つまり、このバックアップを入手した攻撃者は、あなたのアカウントに接続することはできませんが、あなたがこれらのアカウントを使用していることだけはわかります。これらのIDを復号化するには、Walletのseedが必要です。



したがって、このテキストファイルのコピーを複数作成し、コンピュータのローカル、 ファイルホスティングサービス、USB メモリなどの外部メディアなど、さまざまな場所に保存する ことができます。ただし、このバックアップは自動的には更新されないため、Trezorとの新しい「*パスワー ドレス*」接続を設定するたびに更新する必要があることに注意してください。



ここで、Trezorを壊してしまったとしましょう。FIDO2認証情報を復元するには、まず新しいFIDO2対応TrezorデバイスでMnemonicフレーズを使用してWalletを復元する必要があります。



リカバリーが完了したら、新しいデバイスでFIDO2識別子をインポートするには、ターミナルで以下のコマンドを実行する：



```shell
trezorctl fido credentials add <CREDENTIAL_ID>
```



<CREDENTIAL_ID>`をあなたの識別子で置き換えるだけです。例えば、私の場合は、：



```shell
trezorctl fido credentials add f1d00200a020a736356d0ceb7ce8b7655b39c399d8111b620bbbbfc78a51add31475e6acd9a68f77f0a6b12a20c7a41412c488787d41e6ee0bdbf3bb99973c9637d21d3a060808143dd228e0831bbb883fb3afedd3f70596a9f6b98f00703244b76260099a9c044346bf6266d3cb9d90db6fc7cde1142b11c5c8ea
```



TrezorがFIDO2識別子をインポートするよう促します。画面上で を押して確認してください。



![Image](assets/fr/20.webp)



これでTrezorでFIDO2ログインが可能になりました。各識別子についてこの手順を繰り返します。
