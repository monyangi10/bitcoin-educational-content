![cover](assets/cover.webp)



트레저 디바이스는 원래 Bitcoin Wallet을 보호하기 위해 설계된 하드웨어 지갑이지만, 웹에서 강력한 인증을 위한 고급 옵션도 갖추고 있습니다. U2F** 및 **FIDO2** 프로토콜과의 호환성 덕분에 비밀번호에만 의존하지 않고도 온라인 계정에 안전하게 액세스할 수 있습니다.



U2F(*유니버설 2단계*) 프로토콜은 2014년에 구글과 유비코가 도입한 후 FIDO 얼라이언스에 의해 표준화되었습니다. 로그인할 때 두 번째 물리적 인증 요소(2FA)를 추가할 수 있습니다. 활성화되면 기존 비밀번호와 더불어 사용자는 트레저의 버튼을 눌러 계정에 연결하려고 할 때마다 승인을 받아야 합니다. 이러한 맥락에서 트레저는 유비키와 유사한 방식으로 작동합니다.



이 방식은 비대칭 암호화를 기반으로 하므로 비밀 데이터가 전송되지 않아 피싱이나 가로채기 공격에 효과적이지 않습니다. U2F는 현재 많은 온라인 서비스에서 지원되고 있습니다.



트레저는 이중 인증이 가능한 U2F 외에도 U2F의 진화된 버전인 FIDO2(*Fast IDentity Online 2.0*)도 지원합니다. 이는 2018년부터 표준화된 인증 프로토콜로, U2F의 로직을 확장하여 비밀번호를 완전히 대체하는 것을 목표로 합니다. 두 가지 구성 요소를 기반으로 합니다: *WebAuthn*(브라우저 측)과 *CTAP2*(물리적 키 측)입니다. FIDO2는 "비밀번호 없는" 인증을 가능하게 합니다. 사용자는 추가 비밀번호 없이 고유한 암호화 token 역할을 하는 Trezor 장치를 통해서만 자신을 식별할 수 있습니다. 이 프로토콜은 현재 여러 온라인 서비스, 특히 기업용 서비스와 호환됩니다.



"비밀번호 없는" 기능 외에도 FIDO2는 U2F와 유사한 방식으로 2단계 인증을 지원합니다.



FIDO2는 또한 연결을 가능하게 하는 개인 키와 사용자의 신원 정보를 모두 포함하는 Trezor에 직접 저장된 식별자, 즉 상주 자격증명이라는 개념을 도입합니다. 이 메커니즘은 진정한 의미의 비밀번호 없는 인증을 가능하게 합니다. ID나 비밀번호를 입력하지 않고 트레저를 연결하고 접속을 확인하기만 하면 됩니다. 반면, 기존의 방식인 비거주 자격 증명은 디바이스에 개인 키만 저장하고 사용자 ID는 서버 측에 저장되므로 연결할 때마다 입력해야 합니다. 나중에 트레저에 저장하는 방법에 대해 살펴보겠습니다.



이 튜토리얼에서는 2단계 인증을 위해 U2F 또는 FIDO2를 활성화하는 방법과 비밀번호 없이 계정에 액세스할 수 있도록 FIDO2를 구성하는 방법을 Trezor로 직접 알아보세요.



**참고: U2F는 모든 Trezor 모델과 호환되지만 FIDO2는 모델 1이 아닌 Safe 3, Safe 5 및 Model T에서만 지원됩니다.



## 트레저에서 2FA에 U2F/FIDO2 사용



시작하기 전에 트레저에서 Bitcoin Wallet를 설정했는지 확인하세요. 2단계 인증에서 U2F 및 FIDO2에 사용되는 키는 이 Mnemonic에서 파생되므로 Mnemonic을 올바르게 저장하는 것이 중요합니다. Trezor를 분실하거나 손상된 경우, 다른 Trezor 장치에 Mnemonic 구문을 입력하여 키에 대한 액세스를 복구할 수 있습니다(다음 섹션에서 살펴보겠지만 "*암호 없는*" 모드의 FIDO2 자격 증명의 경우, seed만으로는 충분하지 않습니다).



트레저를 컴퓨터에 연결하고 잠금을 해제하세요.



![Image](assets/fr/01.webp)



2단계 인증을 사용하여 보호하려는 계정에 액세스합니다. 예를 들어 Bitwarden 계정을 사용하겠습니다. 일반적으로 서비스 설정의 '*인증*', '*보안*', '*로그인*' 또는 '*비밀번호*' 탭 아래에서 2단계 인증 옵션을 찾을 수 있습니다.



![Image](assets/fr/02.webp)



2단계 인증 전용 섹션에서 '*패스키*' 옵션을 선택합니다(사용 중인 사이트에 따라 용어가 다를 수 있음).



![Image](assets/fr/03.webp)



현재 비밀번호를 확인하라는 메시지가 자주 표시됩니다.



![Image](assets/fr/04.webp)



쉽게 알아볼 수 있도록 보안 키에 이름을 지정한 다음 "*키 읽기*"를 클릭합니다.



![Image](assets/fr/05.webp)



트레저 화면에 계정 세부 정보가 표시됩니다. 화면을 터치하거나 버튼을 눌러 확인합니다. 또한 PIN 코드를 확인하라는 메시지가 표시됩니다.



![Image](assets/fr/06.webp)



이 보안 키를 등록하세요.



![Image](assets/fr/07.webp)



이제부터는 계정에 연결할 때 일반적인 비밀번호 외에 트레저를 연결하라는 메시지가 표시됩니다.



![Image](assets/fr/08.webp)



그런 다음 트레저 화면을 눌러 인증을 확인할 수 있습니다.



![Image](assets/fr/09.webp)



2단계 인증에 Hardware Wallet 트레저를 사용할 때의 장점은 Mnemonic 문구 덕분에 키를 쉽게 복구할 수 있다는 것입니다. 이러한 기본 백업 외에도 2FA를 활성화한 각 서비스에서 제공하는 비상 코드를 사용할 수도 있습니다. 이 비상 코드를 사용하면 보안 키를 분실했을 때 계정에 연결할 수 있습니다. 따라서 필요한 경우 2FA를 대체하여 연결할 수 있습니다.



예를 들어 Bitwarden에서는 "*복구 코드 보기*"를 클릭하여 이 코드에 액세스할 수 있습니다.



![Image](assets/fr/10.webp)



이 코드를 기본 비밀번호와 다른 곳에 보관하여 함께 도용되는 것을 방지하는 것이 좋습니다. 예를 들어 비밀번호를 비밀번호 관리자에 저장하는 경우 2FA 비상 코드를 종이에 따로 보관하세요.



이 접근 방식은 2단계 인증을 위해 Trezor를 분실했을 때 두 가지 수준의 백업을 제공합니다. 모든 계정에 대해 Mnemonic 문구를 사용한 첫 번째 백업과 비상 코드가 포함된 각 계정에 대한 두 번째 백업이 그것입니다. 단, Mnemonic의 역할과 비상 코드의 역할을 혼동하지 않는 것이 중요합니다**:




- 12단어 또는 24단어 Mnemonic 문구를 사용하면 모든 계정의 2FA에 사용되는 키뿐만 아니라 트레저로 보호된 비트코인에도 액세스할 수 있습니다;
- 긴급 코드를 사용하면 해당 계정에 대해서만 일시적으로 2FA 요청을 우회할 수 있습니다(이 예에서는 Bitwarden에서만).



## 트레저에서 FIDO2 사용



FIDO2는 2단계 인증 외에도 사이트에 로그인할 때 비밀번호를 입력할 필요 없이 "비밀번호 없는" 인증도 지원합니다. 이 방법으로 보안 계정에 액세스하려면 트레저를 컴퓨터에 연결하기만 하면 됩니다. 이 기능을 구성하는 방법은 다음과 같습니다.



시작하기 전에 트레저에서 Bitcoin Wallet을 설정했는지 확인하세요. FIDO2 "*암호 없는*" 식별자는 seed로 암호화되므로 Mnemonic을 저장하는 것이 중요합니다(다음 섹션에서 이러한 식별자를 올바르게 저장하는 방법에 대해 알아보겠습니다).



트레저를 컴퓨터에 연결하고 잠금을 해제합니다.



![Image](assets/fr/01.webp)



"*비밀번호 없음*" 모드에서 보호하려는 계정에 액세스합니다. Bitwarden 계정을 예로 들어보겠습니다. 이 옵션은 일반적으로 서비스 설정의 "*인증*", "*보안*" 또는 "*비밀번호*" 탭에서 찾을 수 있습니다.



예를 들어 비트워든에서는 "*마스터 비밀번호*" 탭 아래에 이 옵션이 있습니다. "*켜기*"를 클릭하여 FIDO2를 통한 인증을 활성화합니다.



![Image](assets/fr/11.webp)



비밀번호를 확인하라는 메시지가 자주 표시됩니다.



![Image](assets/fr/12.webp)



트레저 화면에 계정 세부 정보가 표시됩니다. 화면을 터치하거나 버튼을 눌러 확인합니다. 또한 PIN 코드를 확인해야 합니다.



![Image](assets/fr/13.webp)



사이트에서 보안 키를 기억할 이름을 추가한 다음 "*켜기*"를 클릭합니다.



![Image](assets/fr/14.webp)



그런 다음 키가 제대로 작동하는지 확인하기 위해 본인 확인을 요청받게 됩니다.



![Image](assets/fr/15.webp)



이제부터는 계정에 로그인할 때 더 이상 이메일 Address를 입력하거나 로그인할 필요가 없습니다. 로그인 양식에서 버튼을 클릭하여 물리적 키로 본인 인증을 하기만 하면 됩니다.



![Image](assets/fr/16.webp)



Hardware Wallet PIN을 입력하여 트레저와의 연결을 확인합니다.



![Image](assets/fr/17.webp)



비밀번호를 입력하지 않아도 계정에 연결됩니다.



![Image](assets/fr/18.webp)



**트레저에서 FIDO2를 통해 "*비밀번호 없는*" 인증을 활성화하더라도 온라인 계정의 기본 비밀번호는 로그인 용도로 계속 유효합니다**



## FIDO2 자격증명 저장(자격증명 상주자)



2단계 인증에 FIDO2 또는 U2F를 사용하는 경우, 즉 Trezor를 통한 2단계 인증 외에 비밀번호가 필요한 계정에 로그인하는 경우, Mnemonic 문구만으로도 키에 대한 액세스 권한을 검색할 수 있습니다. 그러나 이전 섹션에서 설명한 대로 "*암호 없는*" 모드에서 FIDO2를 사용하는 경우, 이러한 자격 증명을 암호화하는 Mnemonic 문구를 백업하는 것 외에도 FIDO 자격 증명의 사본을 만들어야 합니다.



이렇게 하려면 Python이 설치된 컴퓨터가 필요합니다. 터미널을 열고 다음 명령을 실행하여 필요한 Trezor 소프트웨어를 설치합니다:



```shell
pip3 install --upgrade trezor
```



USB를 통해 Trezor를 컴퓨터에 연결하고 PIN 코드를 사용하여 잠금을 해제합니다.



![Image](assets/fr/01.webp)



Trezor에 저장된 FIDO2 식별자 목록을 검색하려면 다음 명령을 실행하세요:



```shell
trezorctl fido credentials list
```



트레저에서 내보내기를 확인합니다.



![Image](assets/fr/19.webp)



FIDO2 로그인 정보가 단말기에 표시됩니다. 예를 들어, 제 Bitwarden 계정의 경우 이 정보를 받았습니다:



```shell

```



이 모든 정보를 복사하여 텍스트 파일에 저장합니다. 이 백업과 관련된 큰 위험은 없으며, FIDO2로 이러한 서비스를 사용하고 있다는 사실이 드러나는 것 외에는 없습니다. "*자격증명 ID*"는 Wallet의 seed를 사용하여 암호화되므로 공격자가 이 백업을 확보하더라도 계정에 연결할 수 없고 단지 사용자가 이러한 계정을 사용하고 있다는 사실만 알 수 있습니다. 이러한 ID의 암호를 해독하려면 Wallet에 있는 seed가 필요합니다.



따라서 이 텍스트 파일의 사본을 여러 개 만들어 컴퓨터 로컬, 파일 호스팅 서비스, USB 스틱과 같은 외부 미디어 등 여러 곳에 저장할 수 있습니다. 단, 이 백업은 자동으로 업데이트되지 않으므로 트레저에 "*비밀번호 없는*" 연결을 새로 설정할 때마다 갱신해야 한다는 점에 유의하세요.



이제 트레저가 고장났다고 가정해 보겠습니다. FIDO2 자격 증명을 검색하려면 먼저 새로운 FIDO2 호환 Trezor 장치에서 Mnemonic 문구를 사용하여 Wallet을 복구해야 합니다.



복구가 완료되면 새 장치에서 FIDO2 식별자를 가져오려면 터미널에서 다음 명령을 실행합니다:



```shell
trezorctl fido credentials add <CREDENTIAL_ID>
```



'<<CREDENTIAL_ID>>'를 식별자 중 하나로 바꾸기만 하면 됩니다. 예를 들어, 제 경우에는 이렇게 하면 됩니다:



```shell
trezorctl fido credentials add f1d00200a020a736356d0ceb7ce8b7655b39c399d8111b620bbbbfc78a51add31475e6acd9a68f77f0a6b12a20c7a41412c488787d41e6ee0bdbf3bb99973c9637d21d3a060808143dd228e0831bbb883fb3afedd3f70596a9f6b98f00703244b76260099a9c044346bf6266d3cb9d90db6fc7cde1142b11c5c8ea
```



Trezor에서 FIDO2 식별자를 가져오라는 메시지가 표시됩니다. 화면을 눌러 확인합니다.



![Image](assets/fr/20.webp)



이제 트레저에서 FIDO2 로그인이 작동합니다. 각 식별자에 대해 이 절차를 반복합니다.
