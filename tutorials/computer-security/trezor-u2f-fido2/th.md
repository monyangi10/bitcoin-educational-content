อุปกรณ์ Trezor เป็นฮาร์ดแวร์ wallet ที่ออกแบบมาเพื่อรักษาความปลอดภัย Bitcoin wallet แต่ยังมีตัวเลือกขั้นสูงสำหรับการยืนยันตัวตนที่แข็งแกร่งบนเว็บอีกด้วย ด้วยความเข้ากันได้กับโปรโตคอล **U2F** และ **FIDO2** ทำให้สามารถรักษาความปลอดภัยในการเข้าถึงบัญชีออนไลน์ของคุณโดยไม่ต้องพึ่งพารหัสผ่านเพียงอย่างเดียว



โปรโตคอล U2F (*Universal 2nd Factor*) ถูกนำเสนอโดย Google และ Yubico ในปี 2014 จากนั้นได้รับการมาตรฐานโดย FIDO Alliance มันช่วยให้สามารถเพิ่มปัจจัยการยืนยันตัวตนทางกายภาพที่สอง (2FA) เมื่อเข้าสู่ระบบ เมื่อเปิดใช้งานแล้ว นอกจากรหัสผ่านแบบคลาสสิก ผู้ใช้จะต้องอนุมัติแต่ละความพยายามในการเชื่อมต่อกับบัญชีของพวกเขาโดยการกดปุ่มบน Trezor ของพวกเขา ในบริบทนี้ Trezor ทำงานในลักษณะคล้ายกับ Yubikey



วิธีนี้อิงตามการเข้ารหัสแบบอสมมาตร: ไม่มีการส่งข้อมูลลับ ทำให้การโจมตีแบบฟิชชิ่งหรือการดักฟังข้อมูลไม่มีประสิทธิภาพ U2F ได้รับการสนับสนุนจากบริการออนไลน์หลายแห่งในขณะนี้



นอกเหนือจาก U2F ซึ่งช่วยให้สามารถยืนยันตัวตนแบบสองปัจจัยได้แล้ว Trezors ยังรองรับ FIDO2 (*Fast IDentity Online 2.0*) ซึ่งเป็นวิวัฒนาการของ U2F นี่คือโปรโตคอลการยืนยันตัวตนที่ได้มาตรฐานตั้งแต่ปี 2018 ซึ่งขยายตรรกะของ U2F และมุ่งหมายที่จะทดแทนรหัสผ่านอย่างสมบูรณ์ มันขึ้นอยู่กับสององค์ประกอบ: *WebAuthn* (ฝั่งเบราว์เซอร์) และ *CTAP2* (ฝั่งกุญแจทางกายภาพ) FIDO2 ช่วยให้การยืนยันตัวตนแบบ "ไร้รหัสผ่าน": ผู้ใช้ระบุตัวตนผ่านอุปกรณ์ Trezor ของพวกเขาเท่านั้น ซึ่งทำหน้าที่เป็น token เข้ารหัสเฉพาะ โดยไม่ต้องใช้รหัสผ่านเพิ่มเติม โปรโตคอลนี้ตอนนี้เข้ากันได้กับบริการออนไลน์หลายแห่ง โดยเฉพาะอย่างยิ่งที่มุ่งเน้นไปที่องค์กรธุรกิจ



นอกเหนือจากฟังก์ชัน "ไร้รหัสผ่าน" แล้ว FIDO2 ยังรองรับการยืนยันตัวตนแบบสองปัจจัยในลักษณะที่คล้ายกับ U2F อีกด้วย



FIDO2 ยังแนะนำแนวคิดของข้อมูลประจำตัวของผู้พักอาศัย, เช่น ตัวระบุที่เก็บไว้โดยตรงใน Trezor ซึ่งรวมถึงทั้งกุญแจส่วนตัวที่ช่วยให้การเชื่อมต่อและข้อมูลการระบุตัวตนของผู้ใช้ กลไกนี้ช่วยให้การยืนยันตัวตนที่ไม่ต้องใช้รหัสผ่านอย่างแท้จริง: เพียงแค่เสียบ Trezor ของคุณและยืนยันการเข้าถึง โดยไม่ต้องป้อนทั้ง ID หรือรหัสผ่าน ในทางกลับกัน ข้อมูลประจำตัวที่ไม่ใช่ของผู้พักอาศัย ซึ่งเป็นแบบดั้งเดิมมากกว่า จะเก็บเฉพาะกุญแจส่วนตัวในอุปกรณ์; ID ของผู้ใช้ยังคงเก็บไว้ในฝั่งเซิร์ฟเวอร์ และดังนั้นจึงต้องป้อนทุกครั้งที่เชื่อมต่อ เราจะมาดูวิธีการบันทึกข้อมูลเหล่านี้กับ Trezor ของคุณในภายหลัง



ในบทแนะนำนี้ เราจะค้นหาวิธีเปิดใช้งาน U2F หรือ FIDO2 สำหรับการยืนยันตัวตนแบบสองปัจจัย และจากนั้นวิธีการกำหนดค่า FIDO2 เพื่อเข้าถึงบัญชีของคุณโดยไม่ต้องใช้รหัสผ่าน โดยตรงกับ Trezor ของคุณ



**หมายเหตุ:** U2F เข้ากันได้กับทุกรุ่นของ Trezor แต่ FIDO2 รองรับเฉพาะ Safe 3, Safe 5 และ Model T เท่านั้น ไม่รองรับ Model One.



## การใช้ U2F/FIDO2 สำหรับ 2FA บน Trezor



ก่อนที่คุณจะเริ่มต้น ตรวจสอบให้แน่ใจว่าคุณได้ตั้งค่า Bitcoin wallet บน Trezor ของคุณแล้ว สิ่งสำคัญคือต้องบันทึกคำช่วยจำของคุณอย่างถูกต้อง เนื่องจากคีย์ที่ใช้สำหรับ U2F และ FIDO2 ในการยืนยันตัวตนแบบสองปัจจัยนั้นได้มาจากคำช่วยจำนี้ หาก Trezor ของคุณสูญหายหรือเสียหาย คุณสามารถกู้คืนการเข้าถึงคีย์ของคุณได้โดยการป้อนวลีคำช่วยจำของคุณบนอุปกรณ์ Trezor อื่น (โปรดทราบว่าสำหรับข้อมูลรับรอง FIDO2 ในโหมด "*ไม่มีรหัสผ่าน*" เพียงแค่ seed นั้นไม่เพียงพอ ดังที่เราจะเห็นในส่วนถัดไป)



เชื่อมต่อ Trezor ของคุณกับคอมพิวเตอร์และปลดล็อกมัน

![Image](assets/fr/09.webp)



ข้อดีของการใช้ Hardware Wallet Trezor สำหรับการยืนยันตัวตนแบบสองปัจจัยคือคุณสามารถกู้คืนกุญแจของคุณได้อย่างง่ายดายด้วยวลีช่วยจำ นอกเหนือจากการสำรองข้อมูลพื้นฐานนี้ คุณยังสามารถใช้รหัสฉุกเฉินที่ให้โดยแต่ละบริการที่คุณได้เปิดใช้งาน 2FA รหัสฉุกเฉินนี้ช่วยให้คุณเชื่อมต่อกับบัญชีของคุณได้หากคุณสูญเสียกุญแจความปลอดภัย ดังนั้นจึงแทนที่ 2FA สำหรับการเชื่อมต่อหากจำเป็น



ใน Bitwarden ตัวอย่างเช่น คุณสามารถเข้าถึงโค้ดนี้ได้โดยคลิกที่ "*ดูรหัสการกู้คืน*"



![Image](assets/fr/10.webp)

วิธีการนี้มอบการสำรองข้อมูลสองระดับในกรณีที่คุณสูญเสีย Trezor สำหรับการยืนยันตัวตนแบบสองปัจจัย (2FA): การสำรองข้อมูลแรกใช้วลีช่วยจำสำหรับบัญชีทั้งหมดของคุณ และการสำรองข้อมูลที่สองเฉพาะสำหรับแต่ละบัญชีด้วยรหัสฉุกเฉิน อย่างไรก็ตาม สิ่งสำคัญคือ **อย่าสับสนระหว่างบทบาทของวลีช่วยจำกับบทบาทของรหัสฉุกเฉิน**:




- วลีช่วยจำ 12 หรือ 24 คำให้คุณเข้าถึงไม่เพียงแค่กุญแจที่ใช้สำหรับ 2FA ในทุกบัญชีของคุณเท่านั้น แต่ยังรวมถึงบิตคอยน์ของคุณที่ปลอดภัยด้วย Trezor ของคุณด้วย ;
- รหัสฉุกเฉินช่วยให้คุณสามารถข้ามคำขอ 2FA ชั่วคราวได้เฉพาะในบัญชีที่เกี่ยวข้อง (ในตัวอย่างนี้ เฉพาะใน Bitwarden เท่านั้น)



## การใช้ FIDO2 บน Trezor



นอกเหนือจากการยืนยันตัวตนแบบสองปัจจัยแล้ว FIDO2 ยังรองรับการยืนยันตัวตนแบบ "ไม่ใช้รหัสผ่าน" ซึ่งหมายถึงการเข้าสู่ระบบโดยไม่ต้องป้อนรหัสผ่าน เพียงแค่เชื่อมต่อ Trezor ของคุณกับคอมพิวเตอร์เพื่อเข้าถึงบัญชีที่ปลอดภัยของคุณในวิธีนี้ นี่คือวิธีการตั้งค่าฟีเจอร์นี้



ก่อนที่คุณจะเริ่มต้น ตรวจสอบให้แน่ใจว่าคุณได้ตั้งค่า Bitcoin wallet บน Trezor ของคุณแล้ว สิ่งสำคัญคือต้องบันทึก mnemonic เนื่องจาก FIDO2 "*passwordless*" identifiers ถูกเข้ารหัสด้วย seed ของคุณ (เราจะหาวิธีบันทึก identifiers เหล่านี้อย่างถูกต้องในส่วนถัดไป)



เชื่อมต่อ Trezor กับคอมพิวเตอร์ของคุณและปลดล็อกมัน



![Image](assets/fr/01.webp)



เข้าถึงบัญชีที่คุณต้องการรักษาความปลอดภัยในโหมด "*passwordless*". ฉันจะใช้บัญชี Bitwarden เป็นตัวอย่าง ตัวเลือกนี้มักจะพบในการตั้งค่าบริการ โดยทั่วไปจะอยู่ภายใต้แท็บ "*authentication*", "*security*" หรือ "*password*".



บน Bitwarden ตัวอย่างเช่น ตัวเลือกจะอยู่ภายใต้แท็บ "*Master password*" คลิกที่ "*Turn on*" เพื่อเปิดใช้งานการยืนยันตัวตนผ่าน FIDO2



![Image](assets/fr/11.webp)



คุณจะถูกขอให้ยืนยันรหัสผ่านของคุณบ่อยครั้ง



![Image](assets/fr/12.webp)



รายละเอียดบัญชีของคุณจะปรากฏบนหน้าจอ Trezor แตะหน้าจอหรือกดปุ่มเพื่อยืนยัน คุณจะต้องยืนยันรหัส PIN ของคุณด้วย



![Image](assets/fr/13.webp)



บนเว็บไซต์ ให้เพิ่มชื่อเพื่อจดจำกุญแจความปลอดภัยของคุณ จากนั้นคลิกที่ "*เปิดใช้งาน*"



![Image](assets/fr/14.webp)



จากนั้นคุณจะถูกขอให้ระบุตัวตนเพื่อตรวจสอบว่ากุญแจกำลังทำงานอย่างถูกต้องหรือไม่



![Image](assets/fr/15.webp)



ตั้งแต่นี้เป็นต้นไป เมื่อเข้าสู่ระบบบัญชีของคุณ จะไม่จำเป็นต้องป้อนที่อยู่อีเมลหรือชื่อผู้ใช้อีกต่อไป เพียงแค่คลิกที่ปุ่มเพื่อยืนยันตัวตนของคุณด้วยกุญแจทางกายภาพบนแบบฟอร์มเข้าสู่ระบบ



![Image](assets/fr/16.webp)



ยืนยันการเชื่อมต่อกับ Trezor ของคุณโดยป้อนรหัส PIN Hardware Wallet ของคุณ



![Image](assets/fr/17.webp)



คุณจะเชื่อมต่อกับบัญชีของคุณโดยไม่ต้องป้อนรหัสผ่าน



![Image](assets/fr/18.webp)



**โปรดทราบว่าแม้ว่าคุณจะเปิดใช้งานการตรวจสอบสิทธิ์แบบ "*ไม่ใช้รหัสผ่าน*" ผ่าน FIDO2 บน Trezor ของคุณ รหัสผ่านหลักสำหรับบัญชีออนไลน์ของคุณยังคงใช้ได้สำหรับการเข้าสู่ระบบ**



## บันทึกข้อมูลรับรอง FIDO2 ของคุณ (ข้อมูลรับรองผู้อยู่อาศัย)



หากคุณใช้ FIDO2 หรือ U2F สำหรับการยืนยันตัวตนแบบสองปัจจัย เช่น การเข้าสู่ระบบบัญชีที่ต้องใช้รหัสผ่านนอกเหนือจากการยืนยัน 2FA ผ่าน Trezor ของคุณ วลีช่วยจำเพียงอย่างเดียวจะสามารถกู้คืนการเข้าถึงกุญแจของคุณได้ อย่างไรก็ตาม หากคุณใช้ FIDO2 ในโหมด "*ไม่ใช้รหัสผ่าน*" ตามที่อธิบายไว้ในส่วนก่อนหน้า จะจำเป็นต้องทำสำเนาของข้อมูลรับรอง FIDO ของคุณนอกเหนือจากการสำรองวลีช่วยจำของคุณซึ่งเข้ารหัสข้อมูลรับรองเหล่านี้ด้วย



ในการทำเช่นนี้ คุณจะต้องมีคอมพิวเตอร์ที่ติดตั้ง Python เปิดเทอร์มินัลและรันคำสั่งต่อไปนี้เพื่อติดตั้งซอฟต์แวร์ Trezor ที่จำเป็น:



```shell
pip3 install --upgrade trezor
```



เชื่อมต่อ Trezor ของคุณกับคอมพิวเตอร์ผ่าน USB และปลดล็อกโดยใช้รหัส PIN ของคุณ



![Image](assets/fr/01.webp)



ในการดึงรายการของตัวระบุ FIDO2 ที่เก็บไว้ใน Trezor ให้รันคำสั่งต่อไปนี้:



```shell
trezorctl fido credentials list
```

คัดลอกและบันทึกข้อมูลทั้งหมดนี้ในไฟล์ข้อความ ไม่มีความเสี่ยงที่สำคัญที่เกี่ยวข้องกับการสำรองข้อมูลนี้ นอกจากการเปิดเผยว่าคุณกำลังใช้บริการเหล่านี้กับ FIDO2 "*Credential ID*" ถูกเข้ารหัสโดยใช้ seed ของ wallet ของคุณ ซึ่งหมายความว่าผู้โจมตีที่ได้รับการสำรองข้อมูลนี้จะไม่สามารถเชื่อมต่อกับบัญชีของคุณได้ แต่จะสังเกตเห็นว่าคุณกำลังใช้บัญชีเหล่านี้เท่านั้น ในการถอดรหัส ID เหล่านี้ คุณต้องใช้ seed ใน wallet ของคุณ



ดังนั้น คุณสามารถสร้างสำเนาหลายชุดของไฟล์ข้อความนี้และจัดเก็บไว้ในสถานที่ต่างๆ เช่น ในเครื่องคอมพิวเตอร์ของคุณ บนบริการโฮสต์ไฟล์ และบนสื่อภายนอกเช่น USB อย่างไรก็ตาม ควรจำไว้ว่าการสำรองข้อมูลนี้จะไม่อัปเดตโดยอัตโนมัติ ดังนั้นคุณจะต้องต่ออายุทุกครั้งที่คุณตั้งค่าการเชื่อมต่อ "*ไร้รหัสผ่าน*" ใหม่กับ Trezor ของคุณ



ตอนนี้ลองจินตนาการว่าคุณทำ Trezor ของคุณเสียหาย เพื่อกู้คืนข้อมูลรับรอง FIDO2 ของคุณ คุณจะต้องกู้คืน wallet ของคุณก่อนโดยใช้วลีช่วยจำบนอุปกรณ์ Trezor ที่รองรับ FIDO2 ใหม่



เมื่อการกู้คืนเสร็จสิ้น เพื่อที่จะนำเข้า FIDO2 identifiers ของคุณบนอุปกรณ์ใหม่ ให้รันคำสั่งต่อไปนี้ในเทอร์มินัลของคุณ:



```shell
trezorctl fido credentials add <CREDENTIAL_ID>
```



เพียงแทนที่ `<CREDENTIAL_ID>` ด้วยหนึ่งในตัวระบุของคุณ ตัวอย่างเช่น ในกรณีของฉัน จะได้:



```shell
trezorctl fido credentials add f1d00200a020a736356d0ceb7ce8b7655b39c399d8111b620bbbbfc78a51add31475e6acd9a68f77f0a6b12a20c7a41412c488787d41e6ee0bdbf3bb99973c9637d21d3a060808143dd228e0831bbb883fb3afedd3f70596a9f6b98f00703244b76260099a9c044346bf6266d3cb9d90db6fc7cde1142b11c5c8ea
```



Trezor ของคุณจะเตือนให้คุณนำเข้า FIDO2 identifier ของคุณ ยืนยันโดยการกดบนหน้าจอ



![Image](assets/fr/20.webp)



การเข้าสู่ระบบ FIDO2 ของคุณใช้งานได้แล้วบน Trezor ของคุณ ทำซ้ำขั้นตอนนี้สำหรับตัวระบุแต่ละตัวของคุณ



ขอแสดงความยินดี ตอนนี้คุณพร้อมใช้งาน Trezor กับ U2F และ FIDO2 แล้ว! หากคุณพบว่าบทแนะนำนี้มีประโยชน์ ฉันจะรู้สึกขอบคุณมากหากคุณให้คะแนนนิ้วโป้งสีเขียวด้านล่าง โปรดอย่าลังเลที่จะแชร์บทแนะนำนี้บนเครือข่ายสังคมของคุณ ขอบคุณมาก!
