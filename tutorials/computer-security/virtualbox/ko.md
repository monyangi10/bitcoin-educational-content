## I. 프레젠테이션



이 튜토리얼에서는 Windows 10, Windows 11, Windows Server 또는 Linux 배포판(데비안, 우분투, 칼리 리눅스 등)을 실행할 가상 머신을 만들기 위해 Windows 11에 VirtualBox를 설치하는 방법에 대해 알아보세요.



이 VirtualBox 입문 튜토리얼은 오라클에서 개발한 이 오픈 소스 가상화 솔루션을 시작하는 데 도움이 되며, 완전히 무료입니다. 나중에 다른 튜토리얼을 온라인에 게시하여 이 주제에 대해 더 자세히 알아볼 수 있도록 할 예정입니다.



프로젝트의 일부로 테스트 목적으로 또는 IT 공부를 위해 워크스테이션을 가상화할 때 VirtualBox는 분명 좋은 솔루션입니다. 또한 Windows 10 및 Windows 11(및 Windows Server)에 통합된 Hyper-V, VMware Workstation(유료) / VMware Workstation Player(개인용 무료)와 같은 다른 솔루션에 대한 대안이기도 합니다.



내 구성: **VirtualBox를 설치할 Windows 11 워크스테이션**이지만, Windows 10(또는 이전 버전)은 물론 Linux에도 VirtualBox를 설치할 수 있습니다. 가상 머신과 관련하여 VirtualBox는 Windows(예: Windows 10, Windows 11, Windows Server 2022 등), Linux(데비안, 록키 리눅스, 우분투, 페도라 등), BSD(PfSense) 및 macOS를 포함한 다양한 시스템을 지원합니다.



## II. Windows 11용 VirtualBox 다운로드



Windows 머신에 설치하기 위해 VirtualBox를 다운로드하려면 [공식 VirtualBox 웹사이트](https://www.virtualbox.org/wiki/Downloads)의 "**다운로드**" 섹션에 있는 Address만 있으면 됩니다. "Windows 호스트"를 클릭하면 100MB가 조금 넘는 이 실행 파일 다운로드를 시작할 수 있습니다.

## III. 윈도우 11에 가상 박스 설치하기



### A. VirtualBox 설치하기



VirtualBox** 설치는 간단하며, 모든 버전의 Windows에서 동일한 과정을 거칩니다. 먼저 방금 다운로드한 VirtualBox 실행 파일을 실행한 다음 "**다음**"을 클릭합니다.



![Image](assets/fr/026.webp)



이 설치는 사용자 지정할 수 있지만 기본 선택 사항인 모든 기능을 설치하는 것이 좋습니다. 아래 이미지에서 다양한 Elements을 확인할 수 있습니다:





- 버추얼박스 USB 지원**을 통해 버추얼박스가 USB 장치를 지원할 수 있도록 합니다
- "브리지" 모드(가상 머신이 로컬 네트워크에 직접 연결할 수 있음)에서 네트워크 지원을 통합하는 VirtualBox 브리지 네트워크**
- "호스트 전용" 모드에서 네트워크 지원을 통합하는 VirtualBox 호스트 전용 네트워크**(가상 머신은 이 모드에서 Windows 11 물리적 호스트 및 다른 가상 머신과만 통신할 수 있음)



계속하려면 "**다음**"을 클릭합니다.



![Image](assets/fr/001.webp)



Windows 11 컴퓨터에서 **네트워크가 잠시 중단될 것**이라는 점을 염두에 두고 "**예**"를 클릭하면 VirtualBox가 브리지 모드를 비롯한 다양한 네트워크 유형을 지원하기 위해 네트워크 수정을 수행합니다.



![Image](assets/fr/002.webp)



확인을 마치면 설치가 시작됩니다... 그리고 "**이 장치 소프트웨어를 설치하시겠습니까?"라는 알림이 표시됩니다. "**항상 오라클사의 소프트웨어 신뢰**" 옵션을 선택하고 "**설치**"를 클릭합니다. VirtualBox는 실제로 컴퓨터에 여러 드라이버를 설치해야 합니다.



![Image](assets/fr/003.webp)



이제 설치가 완료되었습니다! "**설치 후 Oracle VM VirtualBox 6.1.34 시작**" 옵션을 확인하고 "**클릭**"을 클릭하여 소프트웨어를 시작합니다.



![Image](assets/fr/004.webp)



### B. 확장팩 추가



여전히 공식 VirtualBox 사이트(이전 링크 참조)에서 "**VirtualBox 6.1.34 Oracle VM VirtualBox 확장 팩**" 섹션에서 "**지원되는 모든 플랫폼**" 링크를 클릭하여 공식 확장 팩을 다운로드할 수 있습니다. 이 팩을 사용하면 VirtualBox에 추가 기능을 추가할 수 있습니다. 반드시 추가해야 하는 것은 아니지만 유용할 수 있습니다! 예를 들어, 가상 머신에서 USB 3.0 지원, 웹캠 지원 및 디스크 암호화가 포함됩니다.



VirtualBox를 열고 메뉴에서 "**파일**"을 클릭한 다음 "**설정**"을 클릭합니다.



![Image](assets/fr/005.webp)



왼쪽의 "**확장 프로그램**"을 클릭한 다음(1), 오른쪽의 "**+**" 버튼을 클릭해(2) 방금 다운로드한 VirtualBox** 확장 프로그램 팩을 **로드합니다(3).



![Image](assets/fr/006.webp)



"**설치**" 버튼을 클릭하여 확인합니다.



![Image](assets/fr/007.webp)



"**확인**"을 클릭하면 공식 확장팩이 VirtualBox 인스턴스에 설치됩니다!



![Image](assets/fr/008.webp)



첫 번째 가상 머신 생성으로 넘어가겠습니다.



## IV. 첫 번째 VirtualBox VM 만들기



VirtualBox에서 새 가상 머신을 생성하려면 "**새로 만들기**" 버튼을 클릭하여 VM 생성 마법사를 시작하면 됩니다. VirtualBox를 처음 시작하시는 분들을 위해 Interface 및 기타 버튼에 대해 자세히 설명해드리겠습니다.





- 설정**: 일반 VirtualBox 구성(기본 VM 폴더, 업데이트 관리, 언어, NAT 네트워크, 확장 기능 등)
- 가져오기**: 가상 어플라이언스를 OVF 형식으로 가져옵니다
- 내보내기**: 기존 가상 머신을 OVF 형식으로 내보내 가상 어플라이언스를 생성합니다
- 추가**: 기존 가상 머신을 표준 VirtualBox 형식(.vbox) 또는 XML 형식으로 VirtualBox 인벤토리에 추가합니다



왼쪽의 "**도구**" 섹션에서는 가상 네트워크 관리, 가상 머신 스토리지 관리 등 **고급 기능에 액세스할 수 있습니다. "**도구**" 항목 아래에 가상 머신이 나중에 추가됩니다.



![Image](assets/fr/009.webp)



### A. VM 생성 프로세스



**참고로 VirtualBox는 Windows, Linux 및 BSD를 포함한 다양한 운영 체제를 지원합니다.** 이 예제에서는 Windows 11용 가상 머신을 생성하겠습니다. 몇 가지 필드를 채워야 합니다:





- 이름**: 가상 머신 이름(VirtualBox에 표시되는 이름입니다.)
- 머신 폴더**: 가상 머신을 저장할 위치로, 이 위치에 가상 머신의 이름을 가진 하위 폴더가 생성된다는 것을 알고 있습니다
- 유형**: 설치하려는 운영체제의 유형, 설치하려는 OS에 따라 다릅니다
- 버전**: 설치하려는 시스템의 버전(이 경우 Windows 11, 즉 "**Windows11_64**")



계속하려면 "**다음**"을 클릭합니다.



![Image](assets/fr/010.webp)



이전 단계에서 선택한 운영 체제에 따라 **VirtualBox는 가상 머신에 할당할 리소스를 추천합니다**. 여기서는 가상 머신에 할당할 RAM에 대해 이야기합니다. Windows 11에서는 실제로 권장되므로 4GB를 가정하지만 리소스가 부족한 경우 대신 2GB를 지정합니다. **계속**.



**참고**: 가상 머신에 할당된 리소스는 나중에 수정할 수 있습니다.



![Image](assets/fr/011.webp)



가상 Hard 디스크에 관해서는 처음부터 시작하기 때문에 "**지금 가상 Hard 디스크 만들기**"를 선택하여 VM에 운영 체제를 설치하고 데이터를 저장할 수 있는 스토리지 공간을 확보해야 합니다. "**생성**"을 클릭합니다.



![Image](assets/fr/012.webp)



VirtualBox는 가상 Hard 디스크에 대해 세 가지 형식을 지원하는데, 이는 VMware 및 Hyper-V와 같은 다른 솔루션과 비교했을 때 가장 큰 차이점입니다. 세 가지 포맷 중에서 선택할 수 있습니다:





- VDI**, 공식 VirtualBox 형식
- 공식 Hyper-V 형식인 VHD**를 사용하지만, 최근에는 새로운 VHDX 형식이 더 자주 사용됩니다
- VMDX**는 VMware Workstation과 VMware ESXi 모두에 대한 공식 VMware 형식입니다



이 VirtualBox 인스턴스에서만 사용할 가상 머신을 생성하려면 "**VDI**"를 선택합니다. 반면에 가상 Hard 디스크를 나중에 Hyper-V 호스트에 연결하려는 경우, 변환할 필요가 없도록 VHD 형식으로 시작하는 것이 좋습니다. "**다음**"을 클릭합니다.



![Image](assets/fr/013.webp)



**가상 디스크는 동적이거나 고정된 크기일 수 있습니다**. 동적일 경우 **가상 디스크를 나타내는 파일(여기서는 .vdi 파일)은 최대 크기에 도달할 때까지 데이터가 디스크에 기록됨에 따라 증가하지만, 데이터가 삭제되어도 줄어들지 않습니다. 반대로 고정 크기인 경우 **총 저장 공간이 즉시 할당(따라서 예약)**되므로 성능이 약간 더 높지만 가상 디스크를 처음 만들 때 시간이 더 오래 걸립니다.



개인적으로 VirtualBox를 사용한 테스트 가상 머신의 경우 "**동적으로 할당**" 모드로 충분하다고 생각합니다.



![Image](assets/fr/014.webp)



**다음 단계는 가상 디스크의 크기를 지정하는 것인데, 기본적으로 디스크는 VM 디렉터리에 저장됩니다(이 단계에서 변경할 수 있음). Windows 11 요구 사항을 준수하기 위해 64GB의 크기를 지정했지만 여기에서도 더 작은 크기를 지정할 수 있습니다. "**생성**"을 클릭하여 VM을 생성합니다!



![Image](assets/fr/015.webp)



이 시점에서 VM은 인벤토리에 있고 구성은 완료되었지만 운영 체제는 설치되지 않은 상태입니다. VM을 시작하기 전에 VM의 구성을 마무리해야 합니다.



### B. VirtualBox VM에 ISO 이미지 할당하기



Windows 11 또는 다른 시스템을 설치하려면 설치 소스가 필요합니다. 대부분의 경우 ISO 형식의 디스크 이미지를 사용하여 운영 체제를 설치합니다. **Windows 11 ISO 이미지를 VM의 가상 DVD 드라이브에 로드해야 합니다**



VirtualBox 인벤토리에서 VM을 클릭한 다음(1), 이 가상 머신의 일반 구성에 액세스할 수 있는 "**구성**" 버튼(2)을 클릭합니다. 이 메뉴는 리소스를 관리하는 데 사용됩니다(예: RAM 늘리기, CPU 구성 등). 왼쪽의 "**스토리지**"를 클릭하고(3), "**비우기**"라고 표시된 DVD 드라이브에서(4) DVD 모양의 아이콘(5)과 "**디스크 파일 선택**"을 클릭합니다.



![Image](assets/fr/016.webp)



설치하려는 운영 체제의 ISO 이미지를 선택한 다음 확인을 클릭합니다. 이렇게 표시됩니다:



![Image](assets/fr/017.webp)



VM의 "**구성**" 섹션에 남아서 설명할 내용이 몇 가지 더 있습니다.



### C. VM 네트워크 연결



왼쪽의 "**네트워크**" 섹션으로 이동합니다. 이 섹션에서는 가상 머신의 네트워크를 관리할 수 있습니다: 가상 네트워크 인터페이스 수(VM당 최대 4개), MAC Address, 네트워크 액세스 모드(NAT, 브리지 액세스, 내부 네트워크 등). **가상 머신이 인터넷에 액세스할 수 있도록 하려면 "NAT" 또는 "브리지 액세스"**를 선택하되, 이 두 번째 모드를 사용하려면 네트워크에서 DHCP 서버가 활성화되어 있어야 하거나 IP Address을 수동으로 구성해야 합니다.



참고: 네트워크 부분에 대해서는 전용 문서에서 더 자세히 다룰 예정입니다.



![Image](assets/fr/018.webp)



### D. 가상 프로세서 수



실제 컴퓨터와 마찬가지로 가상 머신이 작동하려면 RAM, Hard 디스크 및 프로세서가 필요합니다. 가상 머신을 생성할 때 마법사에 프로세서 구성이 포함되지 않은 것을 보셨을 것입니다. 그러나 이는 언제든지 "**시스템**" 탭에서 "**프로세서**"를 통해 구성할 수 있으며, 여기서 가상 프로세서 수를 선택할 수 있습니다.



참고: 중첩 가상화에는 "**VT-x/AMD-v 중첩 활성화**" 옵션이 필요합니다.



제 경우에는 가상 머신에 2개의 가상 프로세서가 있습니다:



![Image](assets/fr/019.webp)

### E. 첫 부팅 및 OS 설치



가상 머신을 시작하려면 인벤토리에서 가상 머신을 선택하고 "**시작**" 버튼을 클릭하기만 하면 됩니다. 두 번째 창이 열리고 가상 머신에 대한 시각적 개요가 제공됩니다.



![Image](assets/fr/020.webp)



끔찍한 오류가 발생하고 가상 머신이 시작되지 않습니다! 오류는 "가상 머신에 로그인하지 못했습니다..."이며 자세한 내용은 다음과 같습니다:



```shell
Not in a hypervisor partition (HPV=0)
AMD-V is disabled in the BIOS (or by the host OS)
```



사실 이것은 **내 컴퓨터에서 가상화 기능이 활성화되어 있지 않기 때문에 정상입니다**! 이 문제를 해결하려면 컴퓨터를 재부팅하여 BIOS에 액세스하고 가상화를 활성화해야 합니다.



![Image](assets/fr/021.webp)



기다리지 않고 컴퓨터를 재부팅한 후 "SUPPR" 키를 눌러 ASUS 마더보드의 BIOS**(키는 기기에 따라 다를 수 있으며, 예를 들어 F2일 수 있음)에 액세스했습니다. 가상화를 활성화하려면 제 경우에는 "SVM 모드"를 활성화해야 합니다. 여기서도 마더보드마다, 제조업체마다 이름이 바뀔 수 있습니다. AMD CPU의 경우 **AMD-V** 또는 인텔 CPU의 경우 **Intel VT-x**를 가리키는 기능을 찾아보세요.



![Image](assets/fr/022.webp)



이 작업이 완료되면 수정 사항을 저장하고 실제 머신을 다시 시작합니다... 이제 **VirtualBox**가 가상 머신을 시작하고 Windows ISO 이미지를 로드하여 운영 체제를 설치할 수 있습니다!



![Image](assets/fr/023.webp)



VirtualBox가 설치된 Windows 11 물리적 호스트에서 Windows 11 가상 머신 폴더에 다양한 파일이 들어 있는 것을 볼 수 있습니다.





- VM 구성(RAM, CPU 등)에 해당하는 **VBOX** 파일(XML 형식)
- VBOX-PREV** 파일은 이전 구성의 백업입니다
- 동적 모드에서 **VDI** 파일은 가상 Hard 디스크에 해당하므로 현재 13GB에 불과하지만 최대 크기는 64GB입니다
- NVRAM** 파일에는 물리적 머신의 비휘발성 메모리와 동일한 가상 머신의 BIOS 상태가 포함되어 있습니다
