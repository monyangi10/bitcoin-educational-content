## I. การนำเสนอ



ในบทแนะนำนี้ เราจะเรียนรู้วิธีการติดตั้ง VirtualBox บน Windows 11 เพื่อสร้างเครื่องเสมือน ไม่ว่าจะเป็นการรัน Windows 10, Windows 11, Windows Server หรือการแจกจ่าย Linux (Debian, Ubuntu, Kali Linux, เป็นต้น)



บทแนะนำเบื้องต้นเกี่ยวกับ VirtualBox นี้จะช่วยให้คุณเริ่มต้นใช้งานโซลูชันการจำลองเสมือนแบบโอเพนซอร์สที่พัฒนาโดย Oracle ซึ่งไม่มีค่าใช้จ่ายใดๆ ในภายหลังจะมีบทแนะนำอื่นๆ ออนไลน์เพื่อให้คุณได้ศึกษาในเชิงลึกมากขึ้นในหัวข้อนี้



เมื่อพูดถึงการจำลองเวิร์กสเตชัน ไม่ว่าจะเพื่อวัตถุประสงค์ในการทดสอบเป็นส่วนหนึ่งของโครงการหรือระหว่างการศึกษาด้าน IT ของคุณ VirtualBox เป็นทางเลือกที่ดีอย่างชัดเจน นอกจากนี้ยังเป็นทางเลือกแทนโซลูชันอื่น ๆ เช่น Hyper-V ซึ่งถูกรวมเข้ากับ Windows 10 และ Windows 11 (รวมถึง Windows Server) และ VMware Workstation (มีค่าใช้จ่าย) / VMware Workstation Player (ฟรีสำหรับการใช้งานส่วนตัว)



การกำหนดค่าของฉัน: **เวิร์กสเตชัน Windows 11 ที่ฉันจะติดตั้ง VirtualBox** แต่คุณสามารถติดตั้ง VirtualBox บน Windows 10 (หรือเวอร์ชันเก่า) รวมถึงบน Linux สำหรับเครื่องเสมือน VirtualBox รองรับระบบที่หลากหลาย รวมถึง Windows (เช่น Windows 10, Windows 11, Windows Server 2022, เป็นต้น), Linux (Debian, Rocky Linux, Ubuntu, Fedora, เป็นต้น), BSD (PfSense) และ macOS.

## III. การติดตั้ง VirtualBox บน Windows 11



### A. การติดตั้ง VirtualBox



การติดตั้ง **VirtualBox** นั้นง่ายมาก และขั้นตอนจะเหมือนกันสำหรับทุกเวอร์ชันของ Windows เริ่มต้นด้วยการเปิดไฟล์ปฏิบัติการ VirtualBox ที่คุณเพิ่งดาวน์โหลด จากนั้นคลิกที่ "**Next**"



![Image](assets/fr/026.webp)



การติดตั้งนี้สามารถปรับแต่งได้ แต่ฉันแนะนำให้คุณติดตั้งคุณสมบัติทั้งหมด: ซึ่งเป็นกรณีของการเลือกค่าเริ่มต้น ในภาพด้านล่าง คุณสามารถเห็นองค์ประกอบต่างๆ รวมถึง:





- VirtualBox USB Support** เพื่อเปิดใช้งาน VirtualBox ให้รองรับอุปกรณ์ USB
- VirtualBox Bridged Network** เพื่อรวมการสนับสนุนเครือข่ายในโหมด "Bridge" (เครื่องเสมือนสามารถเชื่อมต่อกับเครือข่ายท้องถิ่นของคุณโดยตรง)
- VirtualBox Host-Only Network** เพื่อรวมการสนับสนุนเครือข่ายในโหมด "Host-Only" (เครื่องเสมือนสามารถสื่อสารได้เฉพาะกับโฮสต์ทางกายภาพ Windows 11 ของคุณและเครื่องเสมือนอื่น ๆ ในโหมดนี้)



คลิก "**ถัดไป**" เพื่อดำเนินการต่อ



![Image](assets/fr/001.webp)



คลิกที่ "**ใช่**", โดยคำนึงว่า **เครือข่ายจะถูกขัดจังหวะชั่วคราวบนเครื่อง Windows 11 ของคุณ**, ในขณะที่ VirtualBox ดำเนินการปรับเปลี่ยนเครือข่ายเพื่อรองรับประเภทเครือข่ายที่แตกต่างกัน รวมถึงโหมด Bridge ด้วย



![Image](assets/fr/002.webp)



เมื่อคุณยืนยันแล้ว การติดตั้งจะเริ่มต้น... และจะมีการแจ้งเตือน "**คุณต้องการติดตั้งซอฟต์แวร์อุปกรณ์นี้หรือไม่?**" ปรากฏขึ้น ให้เลือกตัวเลือก "**เชื่อถือซอฟต์แวร์จาก Oracle Corporation เสมอ**" และคลิก "**ติดตั้ง**" VirtualBox จำเป็นต้องติดตั้งไดรเวอร์หลายตัวบนคอมพิวเตอร์ของคุณจริงๆ



![Image](assets/fr/003.webp)



การติดตั้งเสร็จสมบูรณ์แล้ว! เลือกตัวเลือก "**เริ่ม Oracle VM VirtualBox 6.1.34 หลังการติดตั้ง**" และคลิก "**คลิก**" เพื่อเปิดซอฟต์แวร์



![Image](assets/fr/004.webp)



### B. เพิ่มชุดส่วนขยาย



ยังคงอยู่บนเว็บไซต์ทางการของ VirtualBox (ดูที่ลิงก์ก่อนหน้า) คุณสามารถดาวน์โหลดชุดขยายอย่างเป็นทางการได้ โดยเข้าถึงได้ภายใต้ส่วน "**VirtualBox 6.1.34 Oracle VM VirtualBox Extension Pack**" โดยคลิกที่ลิงก์ "**All supported platforms**" ชุดนี้จะช่วยให้คุณเพิ่มฟังก์ชันการทำงานเพิ่มเติมให้กับ VirtualBox: ไม่จำเป็นต้องเพิ่ม แต่สามารถเป็นประโยชน์ได้! ตัวอย่างเช่น มันรวมถึงการรองรับ USB 3.0 ใน VMs, การรองรับเว็บแคม และการเข้ารหัสดิสก์



เปิด VirtualBox คลิกที่ "**ไฟล์**" แล้วคลิกที่ "**การตั้งค่า**" ในเมนู



![Image](assets/fr/005.webp)



คลิกที่ "**ส่วนขยาย**" ทางด้านซ้าย (1) จากนั้นคลิกที่ปุ่ม "**+**" ทางด้านขวา (2) เพื่อ **โหลดแพ็คส่วนขยาย VirtualBox** ที่คุณเพิ่งดาวน์โหลด (3)



![Image](assets/fr/006.webp)



ยืนยันโดยคลิกที่ปุ่ม "**Installation**"



![Image](assets/fr/007.webp)



คลิก "**OK**": แพ็คส่วนขยายอย่างเป็นทางการได้ติดตั้งบนอินสแตนซ์ VirtualBox ของคุณแล้ว!



![Image](assets/fr/008.webp)



มาเริ่มสร้างเครื่องเสมือนเครื่องแรกของเรากันเถอะ



## IV. การสร้าง VirtualBox VM แรกของคุณ



ในการสร้างเครื่องเสมือนใหม่บน VirtualBox เพียงคลิกที่ปุ่ม "**New**" เพื่อเปิดตัวช่วยสร้างการสร้าง VM เนื่องจากนี่เป็นครั้งแรกที่คุณเริ่มต้น VirtualBox ฉันอยากจะให้รายละเอียดเพิ่มเติมเกี่ยวกับ Interface และปุ่มอื่น ๆ แก่คุณ





- การตั้งค่า**: การกำหนดค่า VirtualBox ทั่วไป (โฟลเดอร์ VM เริ่มต้น, การจัดการอัปเดต, ภาษา, เครือข่าย NAT, ส่วนขยาย, ฯลฯ)
- นำเข้า**: นำเข้าเครื่องใช้ไฟฟ้าเสมือนในรูปแบบ OVF
- ส่งออก**: ส่งออกเครื่องเสมือนที่มีอยู่ในรูปแบบ OVF เพื่อสร้างอุปกรณ์เสมือน
- เพิ่ม**: เพิ่มเครื่องเสมือนที่มีอยู่ลงในสินค้าคงคลัง VirtualBox ของคุณ ในรูปแบบมาตรฐาน VirtualBox (.vbox) หรือรูปแบบ XML



ทางด้านซ้าย ส่วน "**เครื่องมือ**" ให้การเข้าถึง **ฟังก์ชันขั้นสูง โดยเฉพาะสำหรับการจัดการเครือข่ายเสมือน แต่ยังรวมถึงการจัดการที่เก็บข้อมูลของ VM** ภายใต้รายการ "**เครื่องมือ**" เครื่องเสมือนของคุณจะถูกเพิ่มในภายหลัง



![Image](assets/fr/009.webp)



### A. กระบวนการสร้าง VM



**ขอเตือนว่า VirtualBox รองรับระบบปฏิบัติการหลากหลาย รวมถึง Windows, Linux และ BSD** ในตัวอย่างนี้ ฉันจะสร้างเครื่องเสมือนสำหรับ Windows 11 ต้องกรอกหลายช่อง:





- ชื่อ**: ชื่อเครื่องเสมือน (นี่คือชื่อที่จะปรากฏใน VirtualBox)
- โฟลเดอร์เครื่อง**: ที่เก็บเครื่องเสมือน โดยจะมีการสร้างโฟลเดอร์ย่อยที่มีชื่อของ VM ในตำแหน่งนี้
- ประเภท**: ประเภทของระบบปฏิบัติการ ขึ้นอยู่กับว่าคุณต้องการติดตั้ง OS ใด
- เวอร์ชัน**: เวอร์ชันของระบบที่คุณต้องการติดตั้ง ในกรณีนี้คือ Windows 11 ดังนั้น "**Windows11_64**"



คลิก "**ถัดไป**" เพื่อดำเนินการต่อ



![Image](assets/fr/010.webp)



ขึ้นอยู่กับระบบปฏิบัติการที่คุณเลือกในขั้นตอนก่อนหน้า **VirtualBox จะให้คำแนะนำเกี่ยวกับทรัพยากรที่จะจัดสรรให้กับเครื่องเสมือน** ที่นี่เรากำลังพูดถึง RAM ที่คุณต้องการจัดสรรให้กับ VM สมมติว่า 4 GB เพราะนี่คือสิ่งที่แนะนำสำหรับ Windows 11 แต่ถ้าคุณมีทรัพยากรจำกัด ให้ระบุ 2 GB แทน **ดำเนินการต่อ**.



**หมายเหตุ**: สามารถแก้ไขทรัพยากรที่จัดสรรให้กับเครื่องเสมือนได้ในภายหลัง



![Image](assets/fr/011.webp)



เท่าที่เกี่ยวข้องกับดิสก์เสมือน เรากำลังเริ่มต้นจากศูนย์ ดังนั้นเราจำเป็นต้องเลือก "**Create virtual hard disk now**" เพื่อให้ VM มีพื้นที่เก็บข้อมูลสำหรับติดตั้งระบบปฏิบัติการและเก็บข้อมูล คลิกที่ "**Create**".



![Image](assets/fr/012.webp)



VirtualBox รองรับรูปแบบต่าง ๆ สามรูปแบบสำหรับฮาร์ดดิสก์เสมือน ซึ่งเป็นความแตกต่างที่สำคัญเมื่อเทียบกับโซลูชันอื่น ๆ เช่น VMware และ Hyper-V มีรูปแบบให้เลือกสามรูปแบบ:





- VDI**, รูปแบบทางการ VirtualBox
- VHD** ซึ่งเป็นรูปแบบ Hyper-V อย่างเป็นทางการ แม้ว่ารูปแบบ VHDX ใหม่จะถูกใช้งานบ่อยกว่าในปัจจุบัน
- VMDX** เป็นรูปแบบทางการของ VMware สำหรับทั้ง VMware Workstation และ VMware ESXi



ในการสร้างเครื่องเสมือนที่จะใช้เฉพาะบนอินสแตนซ์ VirtualBox นี้ ให้เลือก "**VDI**" ในทางกลับกัน หากต้องการแนบดิสก์เสมือนกับโฮสต์ Hyper-V ในภายหลัง อาจเป็นความคิดที่ดีที่จะเริ่มต้นด้วยรูปแบบ VHD เพื่อหลีกเลี่ยงการต้องแปลง คลิกที่ "**Next**"



![Image](assets/fr/013.webp)



**ดิสก์เสมือนสามารถเป็นแบบไดนามิกหรือขนาดคงที่** เมื่อเป็นแบบไดนามิก ไฟล์ที่แทนดิสก์เสมือน **(ในที่นี้คือไฟล์ .vdi) จะเติบโตเมื่อมีการเขียนข้อมูลลงในดิสก์** จนกว่าจะถึงขนาดสูงสุด แต่จะไม่หดตัวหากลบข้อมูล ในทางกลับกัน เมื่อเป็นขนาดคงที่ **พื้นที่จัดเก็บทั้งหมดจะถูกจัดสรรทันที (และดังนั้นจึงถูกสำรองไว้)** ซึ่งช่วยให้มีประสิทธิภาพสูงขึ้นเล็กน้อย แต่ใช้เวลานานกว่าเมื่อสร้างดิสก์เสมือนครั้งแรก



ส่วนตัวแล้ว สำหรับการทดสอบเครื่องเสมือนด้วย VirtualBox ฉันถือว่าโหมด "**จัดสรรแบบไดนามิก**" เพียงพอแล้ว



![Image](assets/fr/014.webp)



**ขั้นตอนต่อไปคือการระบุขนาดของดิสก์เสมือน** โดยคำนึงว่าตามค่าเริ่มต้น ดิสก์จะถูกเก็บไว้ในไดเรกทอรี VM (สามารถเปลี่ยนแปลงได้ในขั้นตอนนี้) ฉันได้ระบุขนาด 64 GB เพื่อให้เป็นไปตามข้อกำหนดของ Windows 11 แต่ที่นี่อีกครั้ง สามารถกำหนดขนาดที่เล็กกว่าได้ คลิกที่ "**Create**" เพื่อสร้าง VM!



![Image](assets/fr/015.webp)



ณ จุดนี้ VM อยู่ในรายการของเราแล้ว มันถูกกำหนดค่าไว้ แต่ระบบปฏิบัติการยังไม่ได้ติดตั้ง เราจำเป็นต้องทำการกำหนดค่า VM ให้เสร็จสมบูรณ์ก่อนที่จะเริ่มใช้งาน



### B. กำหนดภาพ ISO ให้กับ VM VirtualBox



ในการติดตั้ง Windows 11 หรือระบบอื่น ๆ เราจำเป็นต้องมีแหล่งติดตั้ง ในกรณีส่วนใหญ่ เราใช้ภาพดิสก์ในรูปแบบ ISO เพื่อติดตั้งระบบปฏิบัติการ **จำเป็นต้องโหลดภาพ ISO ของ Windows 11 เข้าไปในไดรฟ์ DVD เสมือนของ VM ของเรา**



คลิกที่ VM ในรายการ VirtualBox (1) จากนั้นคลิกที่ปุ่ม "**Configuration**" (2) ซึ่งจะให้การเข้าถึงการกำหนดค่าทั่วไปของเครื่องเสมือนนี้ เมนูนี้ใช้ในการจัดการทรัพยากร (เช่น เพิ่ม RAM, กำหนดค่า CPU เป็นต้น) คลิกที่ "**Storage**" ทางด้านซ้าย (3) บนไดรฟ์ DVD ที่ระบุว่า "**Empty**" ในขณะนี้ (4) จากนั้นคลิกที่ไอคอนรูป DVD (5) และ "**Choose a disk file**"



![Image](assets/fr/016.webp)



เลือกไฟล์ ISO ของระบบปฏิบัติการที่คุณต้องการติดตั้ง จากนั้นคลิกตกลง นี่คือสิ่งที่ฉันได้รับ:



![Image](assets/fr/017.webp)



อยู่ในส่วน "**Configuration**" ของ VM ฉันยังมีบางสิ่งที่ต้องอธิบายเพิ่มเติม



### C. การเชื่อมต่อเครือข่าย VM



ไปที่ส่วน "**Network**" ทางด้านซ้าย ส่วนนี้ช่วยให้คุณจัดการเครือข่ายของเครื่องเสมือน: จำนวนอินเทอร์เฟซเครือข่ายเสมือน (สูงสุด 4 ต่อ VM), ที่อยู่ MAC, และโหมดการเข้าถึงเครือข่าย (NAT, bridge access, internal network, ฯลฯ) **หากคุณต้องการให้เครื่องเสมือนของคุณเข้าถึงอินเทอร์เน็ตได้ ให้เลือก "NAT" หรือ "Bridge Access"** แต่โหมดที่สองนี้ต้องการให้มีเซิร์ฟเวอร์ DHCP ทำงานอยู่บนเครือข่ายของคุณ หรือคุณจะต้องกำหนดค่า IP address ด้วยตนเอง



หมายเหตุ: ฉันจะกลับมาพูดถึงส่วนของเครือข่ายอย่างละเอียดในบทความเฉพาะ



![Image](assets/fr/018.webp)



### ง. จำนวนโปรเซสเซอร์เสมือน



เช่นเดียวกับคอมพิวเตอร์จริง เครื่องเสมือนต้องการ RAM, ฮาร์ดดิสก์ และโปรเซสเซอร์เพื่อทำงาน เมื่อเราสร้าง VM คุณอาจสังเกตเห็นว่าตัวช่วยสร้างไม่ได้รวมการกำหนดค่าโปรเซสเซอร์ไว้ อย่างไรก็ตาม สามารถกำหนดค่าได้ตลอดเวลาผ่านแท็บ "**System**" จากนั้นเลือก "**Processor**" ซึ่งคุณสามารถเลือกจำนวนโปรเซสเซอร์เสมือนได้



หมายเหตุ: จำเป็นต้องเลือกตัวเลือก "**Enable VT-x/AMD-v nested**" สำหรับการทำงานแบบ nested virtualization.

![Image](assets/fr/019.webp)



**อย่าลังเลที่จะดูส่วนอื่นๆ ของเมนูการกำหนดค่า**



### E. การบูตครั้งแรกและการติดตั้งระบบปฏิบัติการ



ในการเริ่มต้นเครื่องเสมือน เพียงแค่เลือกมันในรายการและคลิกที่ปุ่ม "**Start**" หน้าต่างที่สองจะเปิดขึ้นเพื่อแสดงภาพรวมของ VM



![Image](assets/fr/020.webp)



โอ๊ย ฉันเจอข้อผิดพลาดร้ายแรงและเครื่องเสมือนของฉันไม่สามารถเริ่มทำงานได้! ข้อผิดพลาดคือ "การเข้าสู่ระบบล้มเหลวสำหรับเครื่องเสมือน..." พร้อมรายละเอียดดังนี้:



```shell
Not in a hypervisor partition (HPV=0)
AMD-V is disabled in the BIOS (or by the host OS)
```



ในความเป็นจริง นี่เป็นเรื่องปกติเพราะ **ฟีเจอร์การจำลองเสมือนยังไม่ได้เปิดใช้งานบนคอมพิวเตอร์ของฉัน**! เพื่อแก้ไขปัญหานี้ คุณจำเป็นต้องรีบูตคอมพิวเตอร์ของคุณเพื่อเข้าถึง BIOS และเปิดใช้งานการจำลองเสมือน



![Image](assets/fr/021.webp)



โดยไม่รอช้า ฉันรีบูตคอมพิวเตอร์และ **กดปุ่ม "SUPPR" เพื่อเข้าสู่ BIOS** (ปุ่มอาจแตกต่างกันไปขึ้นอยู่กับเครื่อง อาจเป็น F2 เป็นต้น) ของเมนบอร์ด ASUS ของฉัน เพื่อเปิดใช้งานการจำลองเสมือน "SVM Mode" ต้องถูกเปิดใช้งาน ในกรณีของฉัน ที่นี่อีกครั้ง จากเมนบอร์ดหนึ่งไปยังอีกเมนบอร์ดหนึ่ง จากผู้ผลิตหนึ่งไปยังอีกผู้ผลิตหนึ่ง ชื่ออาจเปลี่ยนไป ค้นหาฟังก์ชันที่อ้างถึง **AMD-V** (สำหรับ CPU ของ AMD) หรือ **Intel VT-x** (สำหรับ CPU ของ Intel)



![Image](assets/fr/022.webp)



เมื่อทำเสร็จแล้ว ให้บันทึกการแก้ไขและรีสตาร์ทเครื่องจริง... คราวนี้, **VirtualBox สามารถเริ่มเครื่องเสมือน** และโหลดภาพ ISO ของ Windows เพื่อติดตั้งระบบปฏิบัติการได้! 🙂



![Image](assets/fr/023.webp)



บนโฮสต์ทางกายภาพ Windows 11 ของเรา ซึ่งติดตั้ง VirtualBox เราสามารถเห็นว่าโฟลเดอร์เครื่องเสมือน Windows 11 มีไฟล์ต่างๆ อยู่





- ไฟล์ **VBOX** (ในรูปแบบ XML) ที่สอดคล้องกับการกำหนดค่า VM (RAM, CPU, ฯลฯ)
- ไฟล์ **VBOX-PREV** เป็นการสำรองข้อมูลของการกำหนดค่าก่อนหน้า
- ไฟล์ **VDI** สอดคล้องกับฮาร์ดดิสก์เสมือนในโหมดไดนามิก ดังนั้นขณะนี้มีขนาดเพียง 13 GB ในขณะที่ขนาดสูงสุดคือ 64 GB
- ไฟล์ **NVRAM** ประกอบด้วยสถานะ BIOS ของเครื่องเสมือน ซึ่งเทียบเท่ากับหน่วยความจำถาวรของเครื่องจริง



![Image](assets/fr/024.webp)

**VirtualBox กำลังทำงานบนเครื่องจริง Windows 11 ของเราแล้ว! สิ่งที่เหลืออยู่คือการใช้ประโยชน์จากมันและสร้างเครื่องเสมือน!** ฉันจะกลับมาติดตั้ง Windows 11 ใน VM ของ VirtualBox ในบทความอื่น สำหรับ Windows 10, Windows Server หรือการแจกจ่าย Linux (Ubuntu, Debian, ฯลฯ) แค่ให้เรานำทางคุณ!
