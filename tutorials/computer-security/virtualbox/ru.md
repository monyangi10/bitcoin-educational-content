## I. Презентация



В этом руководстве мы узнаем, как установить VirtualBox на Windows 11 для создания виртуальных машин, будь то Windows 10, Windows 11, Windows Server или дистрибутив Linux (Debian, Ubuntu, Kali Linux и т. д.).



Это вводное руководство по VirtualBox поможет вам начать работу с этим бесплатным решением для виртуализации с открытым исходным кодом, разработанным компанией Oracle. Позже в сети появятся другие руководства, которые помогут вам глубже изучить эту тему.



Когда речь идет о виртуализации рабочей станции, будь то для тестирования в рамках проекта или во время изучения ИТ, VirtualBox, безусловно, является хорошим решением. Это альтернатива другим решениям, таким как Hyper-V, интегрированный в Windows 10 и Windows 11 (а также Windows Server), и VMware Workstation (платный) / VMware Workstation Player (бесплатный для личного использования).



Моя конфигурация: **рабочая станция с Windows 11, на которую я собираюсь установить VirtualBox**, но вы можете установить VirtualBox на Windows 10 (или более старую версию), а также на Linux. Что касается виртуальных машин, VirtualBox поддерживает широкий спектр систем, включая Windows (например, Windows 10, Windows 11, Windows Server 2022 и т. д.), Linux (Debian, Rocky Linux, Ubuntu, Fedora и т. д.), BSD (PfSense) и macOS.

Чтобы скачать VirtualBox для установки на машину Windows, есть только один хороший Address: [официальный сайт VirtualBox](https://www.virtualbox.org/wiki/Downloads) в разделе "**Загрузки**". Просто щелкните на "Windows hosts", чтобы начать загрузку этого исполняемого файла, размер которого составляет чуть более 100 МБ.

## III. Установка VirtualBox в Windows 11



### A. Установка VirtualBox



Установка VirtualBox** очень проста, и процесс одинаков для всех версий Windows. Начните с запуска исполняемого файла VirtualBox, который вы только что загрузили, затем нажмите "**Next**".



![Image](assets/fr/026.webp)



Эту установку можно настраивать, но я рекомендую установить все функции: что и сделано в случае выбора по умолчанию. На изображении ниже вы можете видеть различные Elements, включая :





- VirtualBox USB Support**, чтобы включить в VirtualBox поддержку USB-устройств
- VirtualBox Bridged Network** для интеграции поддержки сети в режиме "Мост" (виртуальная машина может подключаться непосредственно к локальной сети)
- VirtualBox Host-Only Network** для интеграции поддержки сети в режиме "Host-Only" (в этом режиме виртуальная машина может взаимодействовать только с физическим хостом Windows 11 и другими виртуальными машинами)



Нажмите "**Следующее**", чтобы продолжить.



![Image](assets/fr/001.webp)



Нажмите "**Да**", помня о том, что **сеть будет прервана на некоторое время на вашей машине с Windows 11**, пока VirtualBox будет выполнять модификацию сети для поддержки различных типов сетей, включая режим Bridge.



![Image](assets/fr/002.webp)



После подтверждения начнется установка... Появится уведомление "**Хотите ли вы установить программное обеспечение этого устройства?**". Отметьте опцию "**Всегда доверять программам от Oracle Corporation**" и нажмите "**Установить**". VirtualBox необходимо установить несколько драйверов на ваш компьютер.



![Image](assets/fr/003.webp)



Теперь установка завершена! Отметьте опцию "**Запустить Oracle VM VirtualBox 6.1.34 после установки**" и нажмите "**Клик**", чтобы запустить программу.



![Image](assets/fr/004.webp)



### B. Добавьте пакет расширений



На официальном сайте VirtualBox (см. предыдущую ссылку) вы можете скачать официальный пакет расширений, доступный в разделе "**VirtualBox 6.1.34 Oracle VM VirtualBox Extension Pack**", нажав на ссылку "**Все поддерживаемые платформы**". Этот пакет позволит вам добавить дополнительные функциональные возможности в VirtualBox: это не обязательно, но может быть полезно! Например, он включает поддержку USB 3.0 в виртуальных машинах, поддержку веб-камеры и шифрование дисков.



Откройте VirtualBox, нажмите на "**Файл**", а затем на "**Настройки**" в меню.



![Image](assets/fr/005.webp)



Нажмите на "**Расширения**" слева (1), затем на кнопку "**+**" справа (2), чтобы **загрузить пакет расширений VirtualBox**, который вы только что скачали (3).



![Image](assets/fr/006.webp)



Подтвердите, нажав на кнопку "**Installation**".



![Image](assets/fr/007.webp)



Нажмите "**OK**": официальный пакет расширений теперь установлен на ваш экземпляр VirtualBox!



![Image](assets/fr/008.webp)



Давайте перейдем к созданию нашей первой виртуальной машины.



## IV. Создание первой виртуальной машины VirtualBox



Чтобы создать новую виртуальную машину в VirtualBox, просто нажмите на кнопку "**New**", чтобы запустить мастер создания виртуальной машины. Поскольку вы впервые запускаете VirtualBox, я хотел бы рассказать вам несколько подробностей о Interface и других кнопках.





- Настройки**: общая настройка VirtualBox (папка ВМ по умолчанию, управление обновлениями, язык, сети NAT, расширения и т.д.)
- Импорт**: импорт виртуального устройства в формате OVF
- Экспорт**: экспорт существующей виртуальной машины в формате OVF для создания виртуального устройства
- Добавить**: добавление существующей виртуальной машины в инвентарь VirtualBox в стандартном формате VirtualBox (.vbox) или в формате XML



Слева находится раздел "**Инструменты**", открывающий доступ к **дополнительным функциям, в частности, к управлению виртуальной сетью, а также к управлению хранилищем виртуальных машин**. В разделе "**Инструменты**" позже будут добавлены ваши виртуальные машины.



![Image](assets/fr/009.webp)



### A. Процесс создания виртуальной машины



**Напоминаю, что VirtualBox поддерживает множество операционных систем, включая Windows, Linux и BSD. В этом примере я создам виртуальную машину для Windows 11. Необходимо заполнить несколько полей:





- Name**: имя виртуальной машины (это имя будет отображаться в VirtualBox)
- Папка машины**: где хранить виртуальную машину, зная, что в этом месте будет создана подпапка с именем виртуальной машины
- Тип**: тип операционной системы, в зависимости от того, какую ОС вы хотите установить
- Версия**: версия системы, которую вы хотите установить, в данном случае Windows 11, поэтому "**Windows11_64**"



Нажмите "**Следующее**", чтобы продолжить.



![Image](assets/fr/010.webp)



В зависимости от операционной системы, которую вы выбрали на предыдущем шаге, **VirtualBox дает рекомендации по выделению ресурсов для виртуальной машины**. Здесь мы говорим об оперативной памяти, которую вы хотите выделить для виртуальной машины. Предположим, 4 ГБ, поскольку это действительно рекомендуется для Windows 11, но если у вас не хватает ресурсов, укажите 2 ГБ. **Continue



**Примечание**: ресурсы, выделенные виртуальной машине, могут быть изменены позже.



![Image](assets/fr/011.webp)



Что касается виртуального диска Hard, то мы начинаем с нуля, поэтому нам нужно выбрать "**Создать виртуальный диск Hard сейчас**", чтобы у ВМ было место для установки операционной системы и хранения данных. Нажмите на кнопку "**Создать**".



![Image](assets/fr/012.webp)



VirtualBox поддерживает три различных формата виртуальных дисков Hard, что является существенным отличием от других решений, таких как VMware и Hyper-V. На выбор предлагается три формата:





- VDI**, официальный формат VirtualBox
- VHD**, который является официальным форматом Hyper-V, хотя в последнее время чаще используется новый формат VHDX
- VMDX** - официальный формат VMware для VMware Workstation и VMware ESXi



Чтобы создать виртуальную машину, которая будет использоваться только в этом экземпляре VirtualBox, выберите "**VDI**". С другой стороны, если виртуальный диск Hard будет позже подключен к хосту Hyper-V, целесообразно начать с формата VHD, чтобы избежать необходимости его преобразования. Нажмите на кнопку "**Следующее**".



![Image](assets/fr/013.webp)



**Виртуальный диск может быть как динамическим, так и фиксированным по размеру**. При динамическом размере файл, представляющий **виртуальный диск (здесь это файл .vdi), будет увеличиваться по мере записи данных на диск**, пока не достигнет максимального размера, но не будет уменьшаться при удалении данных. Напротив, при фиксированном размере **все пространство для хранения данных выделяется сразу (и, следовательно, резервируется)**, что обеспечивает несколько более высокую производительность, но занимает больше времени при первом создании виртуального диска.



Лично я считаю, что для тестовых виртуальных машин с VirtualBox достаточно режима "**Динамическое выделение**".



![Image](assets/fr/014.webp)



**На следующем этапе нужно указать размер виртуального диска**, помня о том, что по умолчанию диск будет храниться в каталоге ВМ (это можно изменить на данном этапе). Я указал размер 64 ГБ, чтобы соответствовать требованиям Windows 11, но, опять же, можно назначить и меньший размер. Нажмите кнопку "**Создать**", чтобы создать ВМ!



![Image](assets/fr/015.webp)



На данный момент ВМ находится в нашем инвентаре, она настроена, но операционная система не установлена. Нам нужно завершить конфигурацию ВМ перед ее запуском.



### B. Назначение ISO-образа виртуальной машине VirtualBox



Чтобы установить Windows 11 или любую другую систему, нам нужны исходники для установки. В большинстве случаев для установки операционной системы мы используем образ диска в формате ISO. **Необходимо загрузить ISO-образ Windows 11 в виртуальный DVD-привод нашей виртуальной машины



Щелкните на виртуальной машине в инвентаре VirtualBox (1), затем на кнопке "**Конфигурация**" (2), которая открывает доступ к общей конфигурации этой виртуальной машины. Это меню используется для управления ресурсами (например, увеличение оперативной памяти, настройка процессора и т. д.). Щелкните на "**Storage**" слева (3), на DVD-приводе, где на данный момент написано "**Empty**" (4), затем щелкните на значке в виде DVD (5) и "**Choose a disk file**".



![Image](assets/fr/016.webp)



Выберите ISO-образ операционной системы, которую вы хотите установить, и нажмите OK. Вот что я получаю:



![Image](assets/fr/017.webp)



Оставайтесь в разделе "**Конфигурация**" VM, мне еще нужно кое-что объяснить.



### C. Сетевое подключение ВМ



Перейдите в раздел "**Сеть**" слева. В этом разделе вы можете управлять сетью виртуальной машины: количество виртуальных сетевых интерфейсов (до 4 на ВМ), MAC Address и режим доступа к сети (NAT, мостовой доступ, внутренняя сеть и т. д.). **Если вы хотите, чтобы виртуальная машина имела доступ в Интернет, выберите "NAT" или "Мостовой доступ "**, но этот второй режим требует, чтобы в вашей сети был активен DHCP-сервер, иначе вам придется настраивать IP Address вручную.



Примечание: я вернусь к сетевой части более подробно в отдельной статье.



![Image](assets/fr/018.webp)



### D. Количество виртуальных процессоров



Как и физическому компьютеру, виртуальной машине для работы необходимы оперативная память, диск Hard и процессор. Когда мы создавали виртуальную машину, вы могли заметить, что мастер не включил конфигурацию процессора. Однако его можно настроить в любое время на вкладке "**Система**", затем "**Процессор**", где вы можете выбрать количество виртуальных процессоров.



Примечание: опция "**Enable VT-x/AMD-v nested**" необходима для вложенной виртуализации.

### E. Первая загрузка и установка ОС



Чтобы запустить виртуальную машину, просто выберите ее в инвентаре и нажмите кнопку "**Start**". Откроется второе окно с визуальным обзором виртуальной машины.



![Image](assets/fr/020.webp)



Ай, я получаю неприятную ошибку, и моя виртуальная машина не запускается! Ошибка выглядит так: "Не удалось войти в систему для виртуальной машины..." со следующими подробностями:



```shell
Not in a hypervisor partition (HPV=0)
AMD-V is disabled in the BIOS (or by the host OS)
```



На самом деле, это нормально, потому что **функция виртуализации не включена на моем компьютере**! Чтобы исправить эту проблему, необходимо перезагрузить компьютер, чтобы получить доступ к BIOS и включить виртуализацию.



![Image](assets/fr/021.webp)



Не дожидаясь, я перезагружаю компьютер и **нажимаю клавишу "SUPPR" для доступа к BIOS** (клавиша зависит от машины и может быть, например, F2) моей материнской платы ASUS. Для активации виртуализации в моем случае должен быть включен "SVM Mode". Здесь, опять же, от одной материнской платы к другой, от одного производителя к другому, название может меняться. Ищите функцию, обозначающую **AMD-V** (для процессора AMD) или **Intel VT-x** (для процессора Intel).



![Image](assets/fr/022.webp)



После этого сохраните модификацию и перезапустите физическую машину... На этот раз **VirtualBox сможет запустить виртуальную машину** и загрузить ISO-образ Windows для установки операционной системы! 🙂 



![Image](assets/fr/023.webp)



На нашем физическом хосте Windows 11, где установлена VirtualBox, мы видим, что папка виртуальной машины Windows 11 содержит различные файлы.





- Файл VBOX** (в формате XML), соответствующий конфигурации ВМ (оперативная память, процессор и т. д.)
- Файл VBOX-PREV** представляет собой резервную копию предыдущей конфигурации
- Файл VDI** соответствует виртуальному диску Hard в динамическом режиме, поэтому его размер в настоящее время составляет всего 13 ГБ, в то время как его максимальный размер - 64 ГБ
- Файл NVRAM** содержит состояние BIOS виртуальной машины, которое эквивалентно энергонезависимой памяти физической машины



![Image](assets/fr/024.webp)
