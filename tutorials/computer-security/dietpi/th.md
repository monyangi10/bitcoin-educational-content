ในวงการที่ไม่ใช่เทคนิค แบรนด์อย่าง `Odroid`, `Raspberry Pi`, `Orange Pi` หรือ `Radxa` เป็นที่รู้จักน้อย แต่เพียงแค่ลองมองไปในวงการเทคโนโลยี ก็จะพบว่า **SBC** คอมพิวเตอร์--ที่สร้างขึ้นบนเมนบอร์ดเดียว ซึ่งมักมีขนาดเล็กมากเมื่อเทียบกับคอมพิวเตอร์ที่ใช้กันทั่วไป--กลายเป็นสิ่งที่ขาดไม่ได้ ในการสนับสนุนโครงการส่วนตัวใดๆ



คอมพิวเตอร์เหล่านี้ผลิตในหลากหลายรุ่น พวกมันมักจะใช้ระบบปฏิบัติการลินุกซ์ที่ปรับแต่งให้ทำงานได้อย่างราบรื่นบนเครื่องที่มีประสิทธิภาพต่ำเหล่านี้



**DietPi ก็ไม่ใช่ข้อยกเว้น**: มันเป็นระบบปฏิบัติการที่พัฒนามาจาก Debian โดยได้รับการปรับแต่งให้มีน้ำหนักเบาที่สุดและทำให้ `SBC` ที่มีประสิทธิภาพน้อยที่สุดทำงานได้อย่างรวดเร็ว เปลี่ยนจาก Debian12 Bookworm เป็น Debian13 Trixie ในขณะที่เขียนบทแนะนำนี้ มันยังรองรับ `RISC-V` ที่เป็นโอเพ่นซอร์สซึ่งใช้โปรเซสเซอร์พื้นฐานของ SBCs อีกด้วย DietPi เป็นการค้นพบที่น่าพอใจซึ่งสมควรได้รับการศึกษาเพิ่มเติม



## จุดแข็ง



นี่ไม่ใช่ "สำเนาปกติ" ของ Debian สำหรับบอร์ดขนาดเล็กประเภท Raspberry DietPi คือ:




- ปรับให้เหมาะสมสำหรับความเร็วและความเบา**: [การเปรียบเทียบกับการแจกจ่าย Debian อื่น ๆ สำหรับ SBC](https://dietpi.com/blog/?p=888), DietPi เบากว่าในทุกด้าน ไฟล์ภาพ ISO ของ DietPi มีขนาดน้อยกว่า 1 GB ซึ่งเป็นขนาดที่เล็กที่สุดในบรรดาผู้ที่ทุ่มเทให้กับรุ่นเก่าของ Raspberry หรือ Orange PI (เช่น) ความต้องการทรัพยากร RAM และ CPU ต่ำมาก ทำให้สามารถดึงประสิทธิภาพสูงสุดจากบอร์ดได้เสมอ แม้กระทั่งบอร์ดรุ่นเก่า
- ระบบอัตโนมัติและตัวติดตั้งในตัว**: ชุดคำสั่งเฉพาะช่วยให้ผู้ใช้สามารถตรวจสอบทรัพยากรของระบบ รวมถึงทำงานอัตโนมัติเพื่อติดตั้งและเปิดโปรแกรม อัปเดตเวอร์ชัน สำรองข้อมูล และตรวจสอบบันทึกทั้งหมด
- ชุมชนที่เน้นการทดลองที่แข็งแกร่ง**: [บทแนะนำ](https://dietpi.com/forum/c/community-tutorials/8) และโครงการจากชุมชน DietPi เหมาะอย่างยิ่งสำหรับการได้รับแรงบันดาลใจจากซอฟต์แวร์ที่คุณสามารถติดตั้งได้ด้วยการคลิกเพียงครั้งเดียว ต้องขอบคุณ DietPi



**การใช้ประโยชน์จาก SBC ของคุณให้เต็มที่ไม่เคยง่ายขนาดนี้มาก่อน**.



## ระบบอัตโนมัติสำหรับการโฮสต์ด้วยตนเอง


ต้องการทดลองกับเซิร์ฟเวอร์ของคุณเองเพื่อใช้งานโซลูชันเครือข่ายขั้นสูง หรือเครื่องมือเพื่อพัฒนาความเชี่ยวชาญ Bitcoin ของคุณหรือไม่? DietPi อาจเป็นโซลูชันที่คุณกำลังมองหา แม้ว่าหลายคนจะรู้วิธีจัดการโครงสร้างพื้นฐานของตนเองและใช้งานเซิร์ฟเวอร์ที่กำหนดค่าและป้องกันอย่างสมบูรณ์แบบแล้ว DietPi เป็นขั้นตอนที่เหมาะสมสำหรับผู้ที่ต้องการเริ่มต้นจากศูนย์



แทนที่จะทำงานที่ซับซ้อนทั้งหมดด้วยตนเอง DietPi ช่วยให้คุณสร้างงานเหล่านั้นด้วย `wizard` และบรรทัดคำสั่งของตัวเอง ที่นี่คุณสามารถทดลองกับคลาวด์ส่วนตัวของคุณเอง การจัดการอุปกรณ์ _smart home_ การสำรองข้อมูลอัตโนมัติและ crontab รวมถึงโซลูชันขั้นสูงอื่นๆ

## การติดตั้ง



### ดาวน์โหลด



DietPi มีชุดของไฟล์ ISO มากมาย เพื่อเขียนระบบปฏิบัติการลงในอุปกรณ์ต่าง ๆ บางตัวเท่านั้นที่รองรับ: ไฟล์ ISO สำหรับ Raspberry PI5 ยังคงอยู่ในขั้นตอนการทดสอบ แต่คุณสามารถหาตัวที่เหมาะสมสำหรับบอร์ดของคุณได้อย่างแน่นอน



สำหรับคู่มือนี้ ฉันเลือกที่จะติดตั้งบน thin client ดังนั้นตัวเลือกจึงไปที่ _PC/VM_ และจากนั้นไปที่ _Native PC_ มีภาพ ISO สองภาพสำหรับอุปกรณ์เหล่านี้ ซึ่งแตกต่างกันในรุ่นของ bootloader เครื่องที่ใช้ในบทช่วยสอนค่อนข้างเก่า ดังนั้นตัวเลือกจึงไปที่เวอร์ชัน _BIOS/CSM_ หากคุณมีเครื่องใหม่กว่า เวอร์ชันที่ถูกต้องอาจเป็น `UEFI`



![img](assets/en/02.webp)



คุณจะเข้าสู่หน้าที่มี `ภาพของตัวติดตั้ง`, `sha256` และ `ลายเซ็น`



![img](assets/en/03.webp)



เตรียมไดเรกทอรีใน `home` ของคอมพิวเตอร์ที่คุณใช้ทุกวัน เพื่อให้คุณสามารถดาวน์โหลดไฟล์ที่จำเป็นด้วย `wget`



![img](assets/en/04.webp)



คีย์สาธารณะของนักพัฒนาต้องการการค้นคว้าขั้นต่ำ แต่คุณสามารถค้นหาได้ที่ลิงก์นี้: https://github.com/MichaIng.gpg.



![img](assets/en/05.webp)



ตรวจสอบเนื้อหาของไดเรกทอรีด้วย `ls -la` และนำเข้า MichaIng key เข้าสู่ keyring ของคุณด้วย `gpg --import`



### การยืนยันและแฟลช



สุดท้ายนี้ ส่วนที่ฉันแนะนำมากที่สุด: ตรวจสอบความถูกต้องของระบบปฏิบัติการที่คุณได้ดาวน์โหลดและกำลังจะติดตั้งบน SBC ของคุณ



![img](assets/en/06.webp)



หากคุณได้รับผลลัพธ์ `Good signature` และการควบคุม Hash เดียวกันด้วยคำสั่ง sha256sum คุณสามารถดำเนินการแฟลช ISO ไปยัง USB stick ได้ ใช้แอปพลิเคชันเช่น Whale Etcher เพื่อทำสิ่งนี้

## การติดตั้ง DietPi



![img](assets/en/09.webp)



ถ่ายโอนไดรฟ์แฟลชไปยังอุปกรณ์ที่จะโฮสต์ DietPi และเริ่มการติดตั้งระบบปฏิบัติการสีเขียวมะนาว ในการฝึกนี้เราใช้ไคลเอนต์บางที่มี RAM 16 GB, SSD 128 GB สำหรับระบบปฏิบัติการ, และดิสก์ข้อมูลที่สองขนาด 1 TB การติดตั้งใช้เวลาน้อยกว่า 30 นาที แต่ถ้าคุณจะใช้ Raspberry ตัวอย่างเช่น ทรัพยากรอาจน้อยกว่าและใช้เวลานานกว่า คุณจะเห็นความคืบหน้าระหว่างการติดตั้ง



![img](assets/en/08.webp)



เนื่องจากถูกออกแบบมาสำหรับ Raspberry และอุปกรณ์ที่คล้ายกัน ธรรมชาติที่แท้จริงของ DietPi คือการติดตั้งแบบ `headless` ซึ่งไม่มีสภาพแวดล้อมกราฟิกและมีการเข้าถึง `shsh` โดยตรง ในคู่มือคุณจะเห็นสภาพแวดล้อมกราฟิก ซึ่งก็คือ LXDE อย่างแม่นยำ



ในขั้นตอนนี้ คุณจะถูกขอให้ตัดสินใจเลือกเว็บเบราว์เซอร์ที่คุณต้องการใช้เป็นค่าเริ่มต้น ระหว่าง Chromium และ Firefox ทั้งสองทำงานได้ดี ไม่มีข้อห้ามเฉพาะสำหรับทั้งสอง นอกจากความชอบส่วนตัวของคุณเอง



Towards the end, the installer may ask you if you want to install any programs already, but here **I advise you not to pre-load anything**. You should know that after this step, you will be asked to change the default passwords of the two users, for security reasons. Most importantly you will be required to **set the `Global Software Password (GSP)`**, which will ensure access to the various software in a controlled manner. **If you download any software during OS installation, without the `GSP` set, they will remain virtually inaccessible**. You will have to uninstall and re-install them again after setting the `Global Software Password`: therefore, **don't put anything in so as to avoid double work**. (The inconvenience is probable, not 100% certain).



การติดตั้งสิ้นสุดด้วยคำขอให้เปิดใช้งาน DietPi-Survey ซึ่งเป็นบริการเก็บข้อมูลอัตโนมัติที่ใช้สนับสนุนการพัฒนาระบบปฏิบัติการ ตามข้อมูลจากเว็บไซต์ การเก็บข้อมูลจะเปิดใช้งานเมื่อคุณดาวน์โหลดซอฟต์แวร์ใด ๆ จากระบบอัตโนมัติที่จัดหาโดย DietPi หรือเมื่อคุณอัปเกรดเป็นรุ่นถัดไป ทุกคนมีตัวเลือกในการเข้าร่วม (_Opt IN_) หรือไม่เข้าร่วม (_Opt OUT_)



![img](assets/en/23.webp)



เมื่อติดตั้งเสร็จสิ้นและรีบูตเครื่องใหม่ DietPi จะปรากฏบนหน้าจอขณะที่คุณตั้งค่า สำหรับการสอนตามที่กล่าวไว้ ฉันเลือกสภาพแวดล้อมกราฟิก `LXDE` บนเดสก์ท็อปฉันพบทางลัดเพื่อเริ่ม `htop` และเปิดเทอร์มินัล



![img](assets/en/10.webp)



### "เครื่องมือ" จากระบบปฏิบัติการ



ลืมโปรแกรมส่วนใหญ่ที่คุณใช้ในลีนุกซ์ดิสทริบิวชันของคุณไปได้เลย-DietPi ได้รับการปรับแต่งอย่างมาก คุณจึงละเว้นไปหลายโปรแกรม คุณจะต้องติดตั้งคำสั่งหลายอย่างด้วยตนเอง แต่ถ้าคุณแค่ลองใช้งาน ให้ต้านทานความอยากและลองทดสอบระบบอัตโนมัติของ DietPi ดู



มันคือเทอร์มินัลที่เป็นเครื่องมือที่มีประโยชน์แรกสำหรับการเริ่มต้นใช้งานระบบปฏิบัติการใหม่ของคุณ และมันจะเปิดโดยอัตโนมัติทุกครั้งที่คุณเปิดเครื่อง



![img](assets/en/11.webp)



บนหน้าจอเทอร์มินัล คุณจะพบรายการคำสั่งที่นำหน้าด้วย `dietpi-` ซึ่งแสดงถึง [เครื่องมือ](https://dietpi.com/docs/dietpi_tools/) ของระบบปฏิบัติการนี้:




- `dietpi-launcher`: เพื่อเข้าถึงระบบปฏิบัติการ, ตัวจัดการไฟล์, เป็นต้น
- `dietpi-Software`: ซึ่งเป็นเครื่องมือที่คุณสามารถใช้ติดตั้งซอฟต์แวร์ทั้งหมดที่มีอยู่ในชุด
- `dietpi-config`: เพื่อเข้าถึงการตั้งค่าระบบ แม้กระทั่งการตั้งค่าที่ซับซ้อนที่สุด



![img](assets/en/14.webp)



### สำรองข้อมูล



การสำรองข้อมูลเซิร์ฟเวอร์เป็นกิจวัตรที่ผู้ดูแลระบบควรคาดการณ์ตั้งแต่เริ่มต้นใช้งานครั้งแรก



ด้วย DietPi มีคำสั่ง `dietpi-Backup` ซึ่งฉันแนะนำให้คุณสำรวจเพื่อหาวิธีแก้ปัญหาที่เหมาะสม: มันช่วยให้คุณตั้งค่าการสำรองข้อมูลเป็นประจำ ไม่ว่าจะเป็นแบบเพิ่มขึ้นหรือลดลงขึ้นอยู่กับแอปพลิเคชันที่ใช้ และมีตัวเลือกทั้งหมดเพื่อปรับแต่งกิจวัตรตามที่ต้องการ



![img](assets/en/22.webp)



เลือกปลายทางของการสำรองข้อมูล เช่น ดิสก์อื่น โดยเริ่ม `dietpi-Drive_Manager` เพื่อเมาท์ไดรฟ์ปลายทางและใช้สำหรับฟังก์ชันนี้



## การกำหนดค่า



การโฮสต์ด้วยตนเองเป็นประสบการณ์ที่แนะนำสำหรับทุกคน ไม่ว่าจะเป็นผู้ที่อยากรู้อยากเห็นหรือเพียงแค่มีความกระตือรือร้น อย่างไรก็ตาม การตั้งค่าและกำหนดค่าเซิร์ฟเวอร์นั้นมีความท้าทายทางเทคโนโลยีที่ไม่น้อยเลยทีเดียว นี่คือจุดที่ **ความเรียบง่ายของ DietPi** เข้ามามีบทบาท ช่วยให้คุณสามารถกำหนดค่าระบบที่เหมาะกับความต้องการของคุณได้ในไม่กี่ขั้นตอนง่ายๆ



![img](assets/en/24.webp)



การตั้งค่าพื้นฐานและขั้นสูงทั้งหมดอยู่ที่ปลายนิ้วของคุณในอินเทอร์เฟซเดียวที่มีให้ด้วยคำสั่ง:



```dietpi-config


sudo dietpi-config


```

Che cosa si può fare ora? Automatizzare i processi da avviare all'accensione del server, configurare il `Locale` e tutte le impostazioni correlate alla Time Zone, impostare le schede di rete, le password e le periferiche audio/video, ad esempio.

## I Pacchetti Software

Tra le caratteristiche di semplicità di DietPi, vi è anche la dettagliata pagina dei Software che - oltre all'elenco delle applicazioni - mostra i primi passi da compiere per installarli e interagire con loro. Prendiamo ad esempio il caso di **Docker**:

![img](assets/en/15.webp)

Cliccando sulla sua "icona" compare in alto una finestra, dove è possibile cliccare i link che portano a una spiegazione di massima. La finestra mostra dove si trovano i file più importanti, come accedere all'interfaccia web e tanti altri suggerimenti per un'installazione fluida.

![img](assets/en/16.webp)

Le applicazioni che prevedono l'interazione dell'utente hanno un'interfaccia web pensata per questo scopo, accessibile all'indirizzo IP che va sempre sotto la sintassi `indirizzo-IP-localhost:porta`. Anche l'URL dell'interfaccia web la trovi se hai cliccato _View_.

Tutti [i software disponibili con DietPi](https://dietpi.com/dietpi-software.html), si installano da terminale, digitando:

```


sudo dietpi-software


```
