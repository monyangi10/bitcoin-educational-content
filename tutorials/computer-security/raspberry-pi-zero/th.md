หากคุณอยู่ในหน้า Plan ₿ Academy มาสักระยะหนึ่งแล้ว คุณคงได้เรียนรู้แล้วว่าหนึ่งในการตั้งค่าความปลอดภัยที่ได้รับการสนับสนุนมากที่สุด เกือบจะเป็นสิ่งจำเป็น **คือการจัดการเงินทุนโดยการจัดเก็บคีย์ส่วนตัวของคุณแบบออฟไลน์**



หากคุณยังไม่พบ ในบทแนะนำนี้คุณจะพบลิงก์ไปยังแหล่งข้อมูลโอเพ่นซอร์สเพื่อเรียนรู้เพิ่มเติมเกี่ยวกับมัน



ในการจัดการคีย์ส่วนตัวแบบออฟไลน์ จำเป็นต้องมีอุปกรณ์ที่ตัดการเชื่อมต่อจากเครือข่ายอย่างถาวร ไม่ว่าจะเป็น [hardware wallet](https://planb.academy/resources/glossary/hardware-wallet) หรือคอมพิวเตอร์ที่แยกออกจากเครือข่ายโดยเฉพาะสำหรับฟังก์ชันนี้โดยเฉพาะ



คุณจะทำอย่างไรหากคุณไม่มีความสามารถในการซื้อฮาร์ดแวร์ที่ทำงานเฉพาะงานนี้ แต่คุณไม่ต้องการละทิ้งขั้นตอนความปลอดภัยนี้?

## ข้อกำหนด


**1** [Raspberry PI Zero](https://www.raspberrypi.com/products/raspberry-pi-zero/): PI Zero (ไม่มีคำต่อท้าย) เป็นพื้นฐานสำหรับการสร้างคอมพิวเตอร์ที่มีประสิทธิภาพขั้นต่ำ แต่ที่สำคัญที่สุดคือไม่มีการ์ด Wi-Fi และ Bluetooth ซึ่งเป็นข้อกำหนดที่จำเป็นสำหรับวัตถุประสงค์ของการฝึกหัดนี้





- ค่าใช้จ่าย**: ประมาณ 15.00 ณ เวลาที่เขียนบทช่วยสอนนี้ (สิงหาคม 2025)
- ความต่อเนื่องของการผลิต**: Raspberry รับประกันการผลิตจนถึงเดือนมกราคม 2030.



PI Zero ที่ไม่มี Wi-Fi และ Bluetooth นั้น น่าเสียดายที่แทบจะหาไม่ได้แล้ว คุณอาจจะสามารถหา PI Zero W และ PI Zero 2W ซึ่งเป็นทางเลือกได้ง่ายกว่า ในกรณีนี้ คุณสามารถปิดฟังก์ชันการเชื่อมต่อได้โดยการเปลี่ยนไฟล์ config หลังจากติดตั้งระบบปฏิบัติการแล้ว คุณจะต้องเพิ่มรายการเหล่านี้ลงใน config:



```
dtoverlay=disable-wifi
dtoverlay=disable-bt
```



ส่วนหนึ่งของคู่มือนี้จะแสดงวิธีและสถานที่ในการทำ อย่างไรก็ตาม หากคุณต้องการความมั่นใจจริงๆ คุณสามารถค้นหาบทเรียนหลายๆ บนเว็บเกี่ยวกับการถอดชิป Wi-Fi ด้วยมีดคัตเตอร์ขนาดเล็ก ซึ่งเหมาะสำหรับการทำงานบนแผงวงจรอิเล็กทรอนิกส์



**2** ชุด _starter kit_ สำหรับ Raspberry PI Zero: ตามที่เป็นแนวปฏิบัติสำหรับโลกของ Raspberry, เป็นแบบ bare bones, ไม่มีเคสภายนอก นอกจากนี้ ทรัพยากรที่จำกัดของบอร์ดขนาดเล็กนี้ยังมีผลต่อความเป็นไปได้ในการเชื่อมต่อกับโลกภายนอก



เมื่อฉันตัดสินใจที่จะดำเนินการต่อ ฉันพบ [ชุดนี้](https://www.amazon.it/-/en/GeeekPi-Raspberry-Aluminum-Passive-Heatsink/dp/B0BJ1WWHGF?crid=1NAFFVHG3IFBU&sprefix=raspberry+pi+zero+kit+geeek+pi%2Caps%2C88&sr=8-65) ที่เต็มไปด้วยอุปกรณ์เสริม เพื่อใช้ประโยชน์จากศักยภาพเต็มที่ของ PI Zero ในความเป็นจริง ชุดนี้ประกอบด้วยแหล่งจ่ายไฟ USB A -> micro USB, ฮับ USB ขนาดเล็ก, อะแดปเตอร์ mini-HDMI -> HDMI, ฮีทซิงค์ทองแดง และเคสอลูมิเนียมด้านนอก นอกจากนี้ยังมีสกรูและประแจหกเหลี่ยมที่จำเป็นสำหรับการใส่ PI Zero ในเคสใหม่





- ค่าใช้จ่าย**: 19.99 ยูโร.



**3** บทแนะนำนี้ไม่จำเป็นต้องให้คุณใช้จ่ายงบประมาณจำนวนมากกับคอมพิวเตอร์แบบแอร์แก๊ป อย่างไรก็ตาม คุณควรรู้ว่าคุณจะต้องมีคีย์บอร์ดและเมาส์ USB (แบบมีสายเท่านั้น หลีกเลี่ยง Bluetooth) และจอภาพ ขึ้นอยู่กับอินพุตของจอภาพของคุณ คุณอาจต้องใช้อะแดปเตอร์จาก mini-HDMI ซึ่งเป็นเอาต์พุตเดียวที่มีใน PI Zero สุดท้ายนี้ ลองค้นหาดูดีๆ ว่าพวกเราทุกคนมีคีย์บอร์ดและเมาส์แบบมีสายอยู่ที่บ้านที่ไหนสักแห่ง - ถึงเวลาปัดฝุ่นพวกมันแล้ว

## การติดตั้งระบบปฏิบัติการ


ก่อนที่จะปิด PI Zero ของคุณในเคส ฉันขอแนะนำให้คุณติดตั้งระบบปฏิบัติการก่อน จากนั้นคุณจะสามารถตรวจสอบไฟ LED ที่แสดงการทำงานได้ทันที



ในการเลือกและเบิร์นระบบปฏิบัติการ ฉันเลือกวิธีที่ง่ายที่สุด: ใช้เครื่องมือ `PI Imager` ของ Raspberry



![img](assets/it/01.webp)



ไปที่ [Raspberry Github](https://github.com/raspberrypi/rpi-imager/releases) เพื่อดาวน์โหลดเวอร์ชันล่าสุดของ Imager โดยเลือกเวอร์ชันที่เหมาะสมที่สุดสำหรับระบบปฏิบัติการของคุณ (v. 1.9.6 ณ เวลาที่เขียน) คุณจะสังเกตเห็นว่าถัดจากแต่ละแอสเซทจะมีแฮชของไฟล์ที่เกี่ยวข้องด้วย ซึ่งจะมีประโยชน์สำหรับการตรวจสอบความถูกต้อง



![img](assets/it/02.webp)



ฉันขอแนะนำให้คุณตรวจสอบระบบปฏิบัติการที่คุณจะใช้บนคอมพิวเตอร์ airgap ของคุณ เพื่อความสบายใจส่วนตัวของคุณเอง สิ่งนี้จะทำให้คุณมั่นใจว่าคุณกำลังใช้งานเวอร์ชันที่ถูกต้อง (ไม่เป็นอันตราย) ของ Imager และ Raspi OS



ทำการตรวจสอบทันทีหลังจากดาวน์โหลด โดยใช้เครื่องที่คุณใช้งานปกติเชื่อมต่อกับอินเทอร์เน็ต



จากนั้นเปิดเทอร์มินัล Linux และเตรียมการดาวน์โหลด โดยสร้างไดเรกทอรี `raspios` ที่มีประโยชน์สำหรับการทำงานกับมัน



![img](assets/it/19.webp)



ดาวน์โหลด Imager สำหรับการกระจาย Linux ของคุณด้วย `wget`.



![img](assets/it/20.webp)



สุดท้าย ให้รันคำสั่งไฟล์ `sha256sum` และเปรียบเทียบ Hash กับที่ Raspberry ให้ไว้ใน Github ของมัน



![img](assets/it/21.webp)



หรือหากคุณใช้ Windows ให้เปิด Power Shell และพิมพ์คำสั่งต่อไปนี้:



```
Get-FileHash -Path <yourpath>\imager-1.9.6.exe
```



![img](assets/it/04.webp)



คุณจะได้รับ Hash ซึ่งต้องตรงกับที่เผยแพร่บน Raspberry Github.



เมื่อคุณได้ตรวจสอบการดาวน์โหลดแล้ว คุณสามารถติดตั้ง Imager บนคอมพิวเตอร์ที่คุณใช้ประจำและเปิดมันขึ้นมา Imager เป็นเครื่องมือที่คุณต้องใช้ในการเขียนระบบปฏิบัติการลงใน micro SD ซึ่งจะเป็น "ดิสก์ระบบ" ของ PI Zero



กระบวนการนี้ง่ายมาก: ขั้นแรกเลือกอุปกรณ์ Raspberry ที่คุณจะใช้ (ดังนั้นให้ใส่ใจกับ **รุ่นของคุณ** ของ Raspi Zero) จากนั้นเลือกเวอร์ชันของระบบปฏิบัติการ และสุดท้ายเลือกจุดเมาท์ของการ์ด micro SD ที่จะทำการแฟลชระบบปฏิบัติการลงไป

## การกำหนดค่าครั้งแรก


การเริ่มต้นครั้งแรกของ Raspi OS จะช้ากว่าปกติเล็กน้อยเพราะต้องทำงานติดตั้งหลายอย่าง แต่ถ้าทุกอย่างเป็นไปด้วยดี คุณจะพบกับหน้าจอต้อนรับ



![img](assets/it/14.webp)



คลิกที่ _ถัดไป_ เพื่อกำหนดภูมิภาคทางภูมิศาสตร์ โดยเฉพาะอย่างยิ่งสำหรับการโหลดแป้นพิมพ์ที่ถูกต้อง ให้ใส่ใจเป็นพิเศษกับส่วนหลัง



![img](assets/it/15.webp)



คลิกที่ _Next_ แล้วคุณจะถูกขอให้สร้างผู้ใช้ของคุณ จดบันทึกข้อมูลรับรองของคุณและเก็บรักษาไว้ให้ดี



![img](assets/it/16.webp)



พ่อมดขอให้คุณเลือกเว็บเบราว์เซอร์เริ่มต้น ระหว่าง Chromium และ Firefox; อาจถามคุณเกี่ยวกับการตั้งค่าเครือข่าย Wi-Fi หากคุณกำลังทำงานกับ Zero W หรือ 2W PI ไปข้างหน้าและคลิก _ข้าม_



ในบางจุดคุณจะได้รับการแจ้งให้ปรับปรุงซอฟต์แวร์และระบบปฏิบัติการบนบอร์ด เลือก _ข้าม_: สำหรับวัตถุประสงค์ของการฝึกนี้ เรากำลังสร้างเครื่องออฟไลน์ ซึ่งต้องคงสถานะออฟไลน์ในขณะนี้



ในที่สุด มันอาจขอให้คุณเปิดใช้งาน `ssh` แต่ปฏิเสธขั้นตอนนี้เช่นกัน เนื่องจากเป็น Zero airgap IP.



![img](assets/it/17.webp)



ไม่มีอะไรให้กำหนดค่าเพิ่มเติมมากนัก เมื่อเสร็จสิ้น ให้รีบูต Raspberry เพื่อให้การเปลี่ยนแปลงมีผล

### ขั้นตอนแรกสำหรับ PI Zero W หรือ 2W


หากคุณกำลังทำงานกับ Raspberry PI Zero W หรือ 2W series บอร์ดของคุณมีชิปสำหรับ Wi-Fi และ Bluetooth ในตัว ในระหว่างการตั้งค่าเริ่มต้นคุณข้ามการกำหนดค่าการเชื่อมต่อ ดังนั้น PI Zero ของคุณจึงไม่ได้เชื่อมต่อกับอินเทอร์เน็ต ตอนนี้คุณสามารถปิดใช้งานชิปทั้งสองอย่างถาวรผ่านซอฟต์แวร์ได้



ในความเป็นจริง Raspi OS มีไฟล์ `config.txt` ที่คุณสามารถแก้ไขได้ตามต้องการ ไฟล์ `config` นี้ตั้งอยู่ในพาร์ทิชัน `boot` ในไดเรกทอรี `firmware` และคุณสามารถแก้ไขและบันทึกได้ด้วยสิทธิ์ `root`



เปิดเทอร์มินัลจากไอคอนที่มุมบนซ้ายและมันจะกลายเป็น `root`.(1)



![img](assets/it/23.webp)



(1) หาก Raspi OS ไม่ได้ให้คุณสร้างรหัสผ่าน `root` ในขั้นตอนก่อนหน้านี้ ฉันขอแนะนำให้คุณทำตอนนี้ โดยใช้คำสั่ง `passwd` การให้ผู้ใช้ `root` ทำงานแยกจากผู้ใช้ส่วนตัวของคุณสามารถเป็นประโยชน์อย่างมากในสถานการณ์การกู้คืน



ด้วยเทอร์มินัล ให้ตรวจสอบไฟล์ `config.txt` และเตรียมพร้อมที่จะแก้ไขด้วยโปรแกรมแก้ไข `nano`



![img](assets/it/24.webp)



การแก้ไขควรทำที่ด้านล่างของ `config` ทั้งหมด หลังจากคำว่า `[All]` นี่คือจุดที่คุณจะเพิ่มคำสั่ง `dtoverlay` ที่แสดงในตอนต้นของบทแนะนำ



![img](assets/it/25.webp)



บันทึก, ปิด, และเริ่มใหม่ ในขั้นตอนต่อไปเราจะไปสำรวจ Raspberry เล็กๆ



## สิ่งที่คาดหวังจากอุปกรณ์นี้?


ดูที่ [ข้อมูลจำเพาะทางเทคนิค](https://www.raspberrypi.com/products/raspberry-pi-zero/) จากเว็บไซต์ Raspberry, PI Zero มีโปรเซสเซอร์ BCM2835 แบบ single-core และ RAM ขนาด 512 MB ดังนั้นจึงดูเหมือนไม่มีพลังมากนัก



เนื่องจากเทอร์มินัลมีน้ำหนักเบากว่า เราจะใช้บรรทัดคำสั่งในการสำรวจการกำหนดค่าระบบ

## ใช้


แม้ว่าประสิทธิภาพจะดูจำกัด (บนกระดาษและเมื่อเทียบกับพลังของเครื่องในปัจจุบัน) แต่ PI Zero ก็มีประสิทธิภาพ โดยเฉพาะอย่างยิ่งในฐานะเทอร์มินัล





- ก่อนอื่นคุณสามารถไปที่เมนูหลักและรับแรงบันดาลใจจากแผง _Add/Remove software_ ซึ่งคุณจะพบกับยูทิลิตี้จำนวนมากสำหรับการเขียนโปรแกรมและฝึกฝน อย่าลืมว่าคุณสามารถทำสิ่งนี้ได้จากเทอร์มินัลเช่นกัน แต่ต้องมีสิทธิ์ `root` เสมอ



![img](assets/it/33.webp)





- คุณสามารถ "รับเลี้ยง" อุปกรณ์ออฟไลน์นี้เพื่อเก็บเอกสารลับหลากหลายประเภท ซึ่งจะยังคงเข้าถึงได้เมื่อจำเป็น โดยไม่เคยถูกเปิดเผยต่ออินเทอร์เน็ต
- คุณสามารถใช้การกำหนดค่านี้เพื่อ generate กุญแจ GPG ของคุณอย่างปลอดภัย
- คุณยังสามารถใช้ "ของเล่นชิ้นเล็ก" ใหม่นี้เป็นอุปกรณ์เซ็นชื่อแบบ airgap [โดยทำตามคำแนะนำของ Arman The Parman](https://armantheparman.medium.com/how-to-set-up-a-raspberry-pi-zero-air-gapped-running-latest-version-of-electrum-desktop-wallet-85e59ecaddc0) ได้อีกด้วย



ในบรรดากระเป๋าเงินที่ฉันคุ้นเคย มีเพียง Electrum เท่านั้นที่มีการปล่อยเวอร์ชัน 32 บิต สำหรับ Zero IP ที่เราเตรียมไว้ในบทแนะนำนี้จะช่วยให้คุณเก็บคีย์ส่วนตัวแบบออฟไลน์ การตั้งค่าสำหรับ Wallet airgap ที่เรากล่าวถึงในบทแนะนำนี้:



https://planb.academy/tutorials/wallet/desktop/electrum-airgap-62b5a4c6-a221-4d41-9a62-4618c53d8223

## บทสรุป


การตั้งค่านี้อาจมีจุดอ่อนใหญ่หนึ่งอย่าง: micro SD เป็นสื่อที่อาจก่อให้เกิดปัญหาได้ มันมีความเสี่ยงต่อการใช้งานหนัก; บางทีคุณอาจมีประสบการณ์กับสิ่งนี้แล้วจากการใช้งานกับโทรศัพท์ของคุณ หลังจากติดตั้งซอฟต์แวร์ทั้งหมดที่คุณต้องการใช้บน Zero airgap IP แล้ว ให้ทำการสำรองข้อมูล micro SD ที่มีค่าโดยใช้เครื่องมือ Raspi OS ที่คุณมีอยู่



![img](assets/it/34.webp)



เมื่อความต้องการของคุณเพิ่มขึ้น และงบประมาณของคุณก็เพิ่มขึ้นตาม คุณสามารถสำรวจ Raspberry หรือโซลูชันอื่นๆ ที่คล้ายกันได้ พูดถึงเรื่องหน่วยความจำ ตัวอย่างเช่น PI 5 มีความสามารถในการประกอบไดรฟ์ M2 NVME ซึ่งแน่นอนว่ามีความทนทานมากกว่า microSD



อย่าลืมจดบันทึกและบันทึกทุกขั้นตอนของการติดตั้งซอฟต์แวร์ยูทิลิตี้ที่คุณกำลังจะใช้งานแบบออฟไลน์ **ไม่ช้าก็เร็วจะมี Raspi OS เวอร์ชันอัปเดตออกมา** ซึ่งคุณจะต้องการใช้ประโยชน์อย่างแน่นอน ในตอนนั้นคุณจะต้องลบทุกอย่างและทำใหม่ทั้งหมดอีกครั้ง (อาจจะต้องใช้ micro SD ใหม่ 😂)



การออกกำลังกายที่เราทำไปเมื่อสักครู่ สมมติว่านี่เป็นการทดลองครั้งแรกของคุณเช่นกัน คุณจะจำมันไปอีกนาน ไม่ใช่ทุกครั้งที่สามารถจัดสรรฮาร์ดแวร์ให้กับการปฏิบัติการ `embedded` แบบออฟไลน์ได้ โดยไม่ละเลยการให้ความสนใจกับเครื่องที่แยกออกจากเครือข่ายเพื่อเปิดและตรวจสอบเป็นครั้งคราว
