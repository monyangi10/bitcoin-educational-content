Plan ₿ Network 페이지를 한동안 사용하셨다면, 가장 옹호되는 보안 설정 중 하나이자 거의 필수라고 할 수 있는 **개인키의 오프라인 저장을 통한 자금 관리**에 대해 이미 알고 계실 것입니다.



아직 발견하지 못했다면 이 튜토리얼에서 오픈 소스 리소스에 대한 링크를 통해 자세히 알아볼 수 있습니다.



따라서 오프라인에서 개인 키를 관리하려면 [Hardware Wallet](https://planb.network/resources/glossary/hardware-wallet) 또는 이 특정 기능 전용의 에어 갭 컴퓨터 등 네트워크에서 영구적으로 연결이 끊긴 장치가 필요합니다.



예를 들어 이 작업만 수행하는 하드웨어를 구매할 수 없지만 이 보안 단계를 포기하고 싶지 않은 경우 어떻게 해야 할까요?



## 솔루션


USB 플래시 드라이브 크기와 무게에 35유로짜리 에어 갭 컴퓨터 형태의 오프라인 장치를 만들 수 있다고 하면 어떨까요? 믿기지 않으시나요?



계속 읽어보세요.



자세한 내용은 끝까지 읽어보세요. 제안된 솔루션은 저렴하지만 가장 쉬운 방법은 아닙니다. 먼저 일반적인 아이디어를 얻은 다음 개인적인 조사에 시간을 투자하고 가능한 한 안심하고 진행 여부와 방법을 선택하기로 결정합니다.



## 요구 사항


**1** [라즈베리 파이 제로](https://www.raspberrypi.com/products/raspberry-pi-zero/): (접미사가 없는) 파이 제로는 최소한의 성능으로 컴퓨터를 만들 수 있는 기반이지만, 무엇보다도 이 실습의 목적에 필수적인 Wi-Fi 및 블루투스 카드가 없습니다.





- 비용**: 이 튜토리얼 작성 시점(2025년 8월) 기준 약 15.00달러입니다.
- 생산의 연속성**: 라즈베리는 2030년 1월까지 생산을 보장합니다.



Wi-Fi와 블루투스가 없는 PI Zero는 안타깝게도 사실상 사용할 수 없게 되었습니다. PI Zero W와 PI Zero 2W를 대체할 수 있는 제품을 더 쉽게 찾을 수 있습니다. 이 경우 구성 파일을 변경하여 연결 기능을 비활성화할 수 있습니다. 운영 체제를 설치한 후 이러한 항목을 구성에 추가해야 합니다:



```
dtoverlay=disable-wifi
dtoverlay=disable-bt
```



이 가이드의 섹션에서 방법과 위치를 알려드립니다. 하지만 정말 확실하게 하고 싶다면 웹에서 전자 보드 작업에 적합한 작은 커터로 Wi-Fi 칩을 제거하는 방법에 대한 여러 튜토리얼을 찾을 수 있습니다.



**2** 라즈베리 파이 제로를 위한 _스타터 키트_: 라즈베리 세계의 관행과 마찬가지로 외부 케이스 없이 베어본으로 제공됩니다. 또한 이러한 작은 보드의 제한된 리소스는 외부 세계와의 연결 가능성을 제한합니다.



계속 진행하기로 결정했을 때, [이 키트](https://www.amazon.it/-/en/GeeekPi-Raspberry-Aluminum-Passive-Heatsink/dp/B0BJ1WWHGF?crid=1NAFFVHG3IFBU&sprefix=raspberry+pi+zero+kit+geeek+pi%2Caps%2C88&sr=8-65)를 발견하고 PI Zero의 잠재력을 최대한 활용하기 위한 액세서리로 가득 찬 [이 키트]를 찾았습니다. 실제로 이 키트에는 USB A-> 마이크로 USB 전원 Supply, 소형 USB 허브, 미니 HDMI-> HDMI 어댑터, 구리 히트싱크, 알루미늄 외부 케이스가 포함되어 있습니다. 또한 PI Zero를 새 케이스에 넣는 데 필요한 나사와 알렌 렌치도 키트와 함께 제공됩니다.





- 비용**: 19.99유로.



**3** 이 튜토리얼은 에어갭 컴퓨터에 큰 예산을 들이지 않아도 됩니다. 하지만 USB 키보드와 마우스(반드시 유선, 블루투스는 피해야 함)와 모니터가 필요하다는 점은 알아두세요. 모니터 입력에 따라 PI Zero에서 사용할 수 있는 유일한 출력인 미니 HDMI의 어댑터가 필요할 수도 있습니다. 마지막으로, 집안 어딘가에 무선이 아닌 키보드와 마우스가 있다면 Hard를 살펴보고 이제 Dust를 꺼야 할 때입니다.



## 추가 예산



**4** 라즈베리에서 약 15.00유로의 오리지널 파워 Supply를 구입할 수 있습니다.



**5** 저는 개인적으로 _스타터 키트_에 포함된 전원 Supply을 3.70유로(약 4,000원)의 USBA → miniUSB 소위 '데이터 없음' 케이블과 결합하여 사용하기로 결정했습니다.



**6** 최소 32GB 이상의 대용량 저장공간이 있는 마이크로 SD 카드(산업용 품질/수준이 더 나은 경우).



**7** 사진에 표시된 것과 같은 시스템, USB-마이크로 SD 어댑터가 필요합니다. PI Zero의 운영 체제와 메모리는 실제로 해당 미디어에서 작동합니다.

## 운영 체제 설치


PI Zero를 케이스에 닫기 전에 운영 체제를 설치하는 것이 좋습니다. 그러면 작동을 나타내는 LED를 바로 확인할 수 있습니다.



운영 체제를 선택하고 굽기 위해 가장 쉬운 방법을 선택했는데, 바로 Raspberry의 'PI 이미저' 도구를 사용하는 것이었습니다.



![img](assets/it/01.webp)



라즈베리 깃허브](https://github.com/raspberrypi/rpi-imager/releases)로 이동하여 운영 체제에 가장 적합한 최신 버전을 다운로드합니다(작성 시점에 1.9.6 버전). 각 자산 옆에 해당 파일의 Hash이 있는 것을 볼 수 있습니다. 이는 검증에 유용합니다.



![img](assets/it/02.webp)



개인적으로 안심할 수 있도록 에어갭 컴퓨터에서 사용할 운영 체제를 확인하는 것이 좋습니다. 이렇게 하면 악성 버전이 아닌 합법적인 버전의 이미저와 결과적으로 Raspi OS로 작업하고 있다는 확신을 가질 수 있습니다.



평소 사용하는 컴퓨터가 인터넷에 연결된 상태에서 다운로드 즉시 인증을 수행합니다



그런 다음 Linux 터미널을 열고 다운로드를 준비하여 작업하는 데 유용한 `raspios` 디렉터리를 만듭니다.



![img](assets/it/19.webp)



Wget`을 사용하여 Linux 배포용 이미저를 다운로드합니다.



![img](assets/it/20.webp)



마지막으로 `sha256sum` 명령을 실행하여 Hash와 라즈베리가 Github에서 제공한 파일을 비교합니다.



![img](assets/it/21.webp)



또는 Windows를 사용하는 경우 파워셸을 열고 다음 명령을 입력합니다:



```
Get-FileHash -Path <yourpath>\imager-1.9.6.exe
```



![img](assets/it/04.webp)



라즈베리 깃허브에 게시된 것과 일치해야 하는 Hash을 받게 됩니다.



다운로드를 확인했으면 매일 사용하는 컴퓨터에 Imager를 설치하고 열 수 있습니다. Imager는 PI Zero의 '시스템 디스크'가 될 마이크로 SD에 운영 체제를 구울 때 필요한 도구입니다.



프로세스는 매우 간단합니다. 먼저 사용하려는 Raspberry 장치를 선택한 다음(**사용 중인 Raspi Zero 모델**에 주의하세요), OS 버전을 선택한 다음, 마지막으로 OS를 플래시할 마이크로 SD 카드의 마운트 지점을 선택하면 됩니다.

## 첫 번째 구성


Raspi OS의 첫 시작은 여러 가지 실제 설치 작업을 수행해야 하기 때문에 평소보다 약간 느립니다. 하지만 모든 것이 잘 진행되면 시작 화면이 나타납니다.



![img](assets/it/14.webp)



다음_을 클릭하여 지리적 지역을 설정하고, 특히 올바른 키보드를 로드하려면 _다음_을 클릭합니다. 후자에 특히 주의하세요.



![img](assets/it/15.webp)



다음_을 클릭하면 사용자를 생성하고 자격 증명을 적어 잘 보관하라는 메시지가 표시됩니다.



![img](assets/it/16.webp)



마법사에서는 기본 웹 브라우저를 크롬과 파이어폭스 중에서 선택하라는 메시지가 표시되며, 제로 W 또는 2W PI로 작업하는 경우 Wi-Fi 네트워크 설정에 대해 묻는 메시지가 표시될 수도 있습니다. 계속해서 _스킵_을 클릭합니다.



어느 시점에서 온보드 소프트웨어와 운영 체제를 업그레이드하라는 메시지가 표시됩니다. 건너뛰기_를 선택합니다. 이 연습의 목적을 위해 실제로 오프라인 머신을 구축하고 있으며, 현재로서는 오프라인 상태를 유지해야 합니다.



마지막으로 `ssh`를 활성화하라는 메시지가 표시될 수 있지만, 이 단계도 제로 에어갭 IP이므로 거부하세요.



![img](assets/it/17.webp)



더 이상 구성할 것이 많지 않습니다. 완료되면 변경 사항을 적용하려면 Raspberry를 재부팅합니다.

### PI 제로 W 또는 2W를 위한 첫 단계


라즈베리 파이 제로 W 또는 2W 시리즈로 작업하는 경우 보드에 Wi-Fi 및 블루투스용 칩이 탑재되어 있습니다. 처음 설정하는 동안 연결 구성을 건너뛰었기 때문에 PI Zero가 인터넷에 연결되지 않았습니다. 이제 소프트웨어를 통해 두 칩을 영구적으로 비활성화할 수 있습니다.



실제로 Raspi OS는 원하는 대로 편집할 수 있는 `config.txt` 파일을 제공합니다. Config`는 `boot` 파티션의 `firmware` 디렉토리에 있으며, `root` 권한으로 편집하고 저장할 수 있습니다.



왼쪽 상단의 아이콘에서 터미널을 열면 '루트'가 됩니다.(1)



![img](assets/it/23.webp)



(1) 이전 단계에서 `루트` 비밀번호를 생성하지 않은 경우, 지금 `passwd` 명령을 사용하여 생성하는 것이 좋습니다. 개인 사용자와 독립적으로 `루트` 사용자를 이동하면 복구 상황에서 매우 편리할 수 있습니다.



터미널에서 `config.txt` 파일을 확인하고 `nano` 편집기로 편집할 준비를 합니다.



![img](assets/it/24.webp)



편집은 전체 `config`의 맨 아래, '[All]`이라는 단어 다음에 수행해야 합니다. 이 지점에서 튜토리얼의 시작 부분에 표시된 `dtoverlay` 명령을 추가합니다.



![img](assets/it/25.webp)



저장하고 닫은 후 다시 시작합니다. 다음 단계에서는 작은 라즈베리 탐험으로 이동합니다.



## 이 장치에서 무엇을 기대할 수 있나요?


라즈베리 웹사이트의 [기술 사양](https://www.raspberrypi.com/products/raspberry-pi-zero/)을 보면, 파이 제로는 싱글 코어 BCM2835 프로세서와 512MB RAM을 탑재하고 있어 그다지 강력해 보이지는 않습니다.



터미널이 더 가볍기 때문에 명령줄을 사용하여 시스템 구성을 탐색해 보겠습니다.



전원을 켜면 짧은 무지개색 화면이 표시되고, 이어서 Raspberry의 환영 메시지와 왼쪽 하단에는 부팅과 관련된 커널 메시지가 표시됩니다.



PI OS 데스크톱이 나타나면 터미널을 열고 입력합니다:



```bash
uname -a
```


이 명령은 현재 사용 중인 커널 버전과 기타 정보를 화면에 표시합니다.



![img](assets/it/26.webp)



입력하면 CPU 및 하드웨어에 대한 정보도 확인할 수 있습니다:



```bash
lscpu
```



![img](assets/it/27.webp)



또한 `proc/mem/info`를 참조하세요.



![img](assets/it/28.webp)



Debian 버전과 릴리스 코드명을 확인하려면 다음과 같이 하세요:



``` bash


lsb_release -a


```

![img](assets/it/29.webp)

``` bash

```



![img](assets/it/31.webp)




## 결론


이 설정에는 아마도 한 가지 큰 약점이 있을 것입니다. 마이크로 SD는 문제를 일으킬 수 있는 매체라는 점입니다. 많이 사용하면 문제가 발생할 수 있는 매체이므로 이미 휴대폰에서 사용해 본 경험이 있을 것입니다. 제로 에어갭 IP에 사용할 모든 소프트웨어를 설치한 후, 사용 가능한 Raspi OS 도구를 사용하여 소중한 마이크로 SD를 잘 백업하세요.



![img](assets/it/34.webp)



필요에 따라, 그리고 예산 가능성에 따라 다른 Raspberry 또는 유사한 솔루션을 살펴볼 수 있습니다. 예를 들어 메모리의 경우, PI 5는 M2 NVME 드라이브를 조립할 수 있는 가능성을 제공하는데, 이는 마이크로SD보다 확실히 더 강력합니다.



오프라인에서 사용하려는 유틸리티 소프트웨어 설치의 모든 단계를 메모하고 문서화하는 것을 잊지 마세요. **조만간 업데이트된 Raspi OS가 출시될 예정이니** 꼭 활용하시기 바랍니다. 그 시점에서는 모든 것을 삭제하고 처음부터 다시 해야 합니다(아마도 새 마이크로 SD 😂로).



방금 한 연습은 첫 번째 실험이라고 가정하면 오랫동안 기억할 것입니다. 수시로 전원을 켜고 확인하기 위해 에어 갭 머신에 대한주의를 소홀히하지 않고 오프라인에서 '임베디드'작업에 하드웨어를 전용 할 수있는 것은 항상 가능하지는 않습니다.



그래도 해내셨군요, 축하드립니다! 그리고 예산을 절감해야 하는 모든 사람들에게 이 솔루션을 제안할 수 있을 것입니다.
