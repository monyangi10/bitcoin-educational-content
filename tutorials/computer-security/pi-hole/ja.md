## I.プレゼンテーション



お気に入りのブラウザを立ち上げるとすぐに、**アドブロッカー**（広告ブロック機能）をインストールする、ということは誰もがやったことがあるだろう。しかし、テレビブラウザやアンドロイド端末などを使う場合は...。何か有効なものを見つけるのは少し難しい。また、家に複数のデバイスがある場合は、ブラウザごとに操作を繰り返さなければならない！



このチュートリアルでは、ネットワーク上のすべてのマシンに広告ブロッカーを提供し、それを一元管理する**というシンプルな問題を解決する**。



そのために、この目的のために開発されたツールを使う： **パイホール



Pi-HoleはDNSシンクホールです。あなたのデバイスが行うDNSリクエストを使ってトラフィックを認証または拒否し、広告やマルウェアなどを配信していることが知られているアドレスやドメインからあなたを守ります。



DNSとはDomain Name System（ドメイン・ネーム・システム）の略です。では、ドメイン名とは何か？「it-connect.fr」はその一例です。ドメイン名は、1つまたは複数のリソースに対する一意の識別子であり、通常は単一のエンティティによって管理されます。



マシン名とドメイン名は、*Fully Qualified Domain Name*の略でFQDNと呼ばれる。これにより、特定のマシンを「呼び出す」だけでアクセスできるようになる。例えば、"www.trucmachin.com "と入力すると、"trucmachin.com "というドメインに属する "www "というマシンを呼び出すことになります。



ただし、私たちのコンピューターは人間の言葉を理解せず、理解できるのはバイナリだけなので、ウェブサイトに到達するには電話番号に相当するIP Addressが必要になる。



そのため、ブラウザにウェブサイトの名前を入力したり、リンクをクリックしたりするたびに、コンピューターはまずDNSサーバーにその名前に対応するIP Addressを問い合わせる。



**Pi-Holeは、これらのリクエスト（毎日何百もあります！）を検査し、広告や悪意のあるファイルをホストしていることが分かっているものを自動的にブロックします。



## II.パイホールの設置



Pi-Holeという名前から、Raspberry-Piが必要だと思われるかもしれない。しかし、そうではありません。 **Pi-HoleはどんなLinuxコンピュータ（Debian、Fedora、Rocky、Ubuntuなど）にもインストールできる。



その一方で、**このデバイスは単純な理由で1日24時間稼働しなければならないことを念頭に置く必要がある：DNSがなければインターネットもできない！**したがって、ラズベリーはほとんどエネルギーを消費しないので、良いアイデアだ。



インストールするには、SSH経由でLinuxマシンに接続し、「*root*」として以下のコマンドを入力するだけだ：



```
curl -sSL https://install.pi-hole.net | bash
```



> **注意**：通常、スクリプトが何をするものかを知らずに "ハック "することはお勧めできません。よくわからない場合は、ブラウザでページにアクセスするか、コンテンツをファイルとしてダウンロードしてください。
>


> **注：Debian 11の最小バージョンでは、Curlはインストールされていないので、上記のコマンドを入力する前に、**apt-get install curl**コマンドで手動でインストールする必要があります。

## III.DHCPコンフィグレーション



パイホールが機能するためには、クライアントからのDNSリクエストを "解決 "する必要があります。これを行うにはいくつかの方法があります：





- DHCPサーバー（Boxなど）のDNS設定を変更する。
- このサーバーを無効にし、Pi-Holeが提供するサーバーを使用する。
- DNSとしてPi-Holeを使用するように各デバイスを手動で変更する。



私は個人的には最初の解決策を選びます。あなたがいる場所**にはDHCPサーバーがある可能性があります（通常はあなたのボックス）。だから悩む必要はない。



事業者の異なるボックス（私はすべてについて知らない）や独自のルーターを持っている人の間で、多くの可能性があるので、これらの修正のためのスクリーンショットを提供するつもりはない。いずれにせよ、DHCPの設定で "DNS "パラメーターを変更し、パイホールのIP Addressを含める必要があります。



これが完了すると、デバイスの電源が以前に入っていた場合、古い設定が保持されているため、コンフィギュレーションリクエストを再起動する必要があります。



Windowsのワークステーションでは、コマンドプロンプト：



```
ipconfig /renew
```



Linuxのワークステーションで：



```
dhclient
```



それ以外の機器については、スイッチを切り、再度オンにする必要がある。



だから、彼らは正しいパラメーターを得るべきだ：



```
ipconfig /all
```



DNSフィールドには、パイホールのAddress、私の場合は192.168.1.42を入力します：

## IV.InterfaceウェブPi-Holeを使う



管理を容易にするために、**Pi-Hole**はよく設計されたInterfaceウェブInterfaceの恩恵を受けています。ユーザーフレンドリーでアクセスしやすく、：





- リクエスト数、ブロックされたリクエスト数などをリアルタイムで表示。
- ホワイトリストとブラックリストの管理
- 静的エントリ、エイリアスなどを追加する。
- リストの追加
- その他にも多くの機能がある！



私としては、ブロッキングリストを追加するつもりだ。前述の通り、Softと同時にインストールしたリストは1つだけ。広告サイト用のリストはたくさんあるが、少なくとも住んでいる国に特化したものを選ぶのがベストだ。最も有名なリストの一つは**EasyList**で、そのうちの一つはフランスに特化している：[EasyList-ListFR](https://raw.githubusercontent.com/deathbybandaid/piholeparser/master/Subscribable-Lists/ParsedBlacklists/EasyList-Liste-FR.txt)



追加するには、まず Interface の管理者に接続します： **http://<ip_du_PiHole>/admin**



管理者パスワードはすでに生成されているので（インストール終了のスクリーンショットを参照）、それを入力するだけでInterfaceにアクセスできます：



![Image](assets/fr/030.webp)



パイホールからのInterface



例えば、パイホールに接続している顧客が2人いること、442リクエストを処理したこと、そのうち8リクエストがブロックされたことがわかる。これらのグラフは、特に専門的な文脈では、良い情報源になります。



私たちのリストを追加するには、"**グループ管理**"と "**アドリスト**"メニューに移動します：



![Image](assets/fr/031.webp)



最初のリスト "**StevenBlack**"を見ることができます。私たちのリストを追加するには、上にあげたリンクをコピーして、フィールド "**Address**"に挿入します：



![Image](assets/fr/032.webp)



Pi-Holeにリストを追加する



あとは「**追加**」をクリックして追加するだけです。このリストを有効にするには、Pi-Holeにこのリストを引き継ぐように "警告 "する必要があります。そのためには：





- 内蔵のコマンドライン
- Interfaceウェブ



なぜなら、アップデートを実行するPHPスクリプトへのリンクが、今いるページ（"online "の文字）に直接あるからだ。つまり、それをクリックするだけで、1つの選択肢しかないページに飛ぶことができるのだ：



![Image](assets/fr/033.webp)



ページには、スクリプトが完了すると、リストが考慮されたことを意味する結果が表示されます（もちろん、エラーメッセージが表示されない限り）。



このチュートリアルの冒頭でお知らせしたように、Pi-Holeはマルウェアを配布することが知られているドメインをブロックすることもできます。この機能を強化するために、[このAddress](https://urlhaus.abuse.ch/downloads/hostfile/)から入手できる、Abuse.ch**が配布している定期的に更新されるドメインリストも追加することをお勧めします。



もちろん、関連すると思われるリストを追加したり、ブラックリストメニューから手動でブラックリストを管理することもできます。
