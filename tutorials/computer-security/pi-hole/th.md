## I. การนำเสนอ



เราทุกคนเคยทำมันทันทีที่เราเปิดเบราว์เซอร์ที่เราชื่นชอบ: ติดตั้ง **adblocker** (ตัวบล็อกโฆษณา) อย่างไรก็ตาม เมื่อใช้เบราว์เซอร์บนทีวีหรืออุปกรณ์ Android ฯลฯ... มันจะยากขึ้นเล็กน้อยในการหาสิ่งที่ใช้งานได้ และถ้ามีมากกว่าหนึ่งอุปกรณ์ในบ้าน คุณก็ต้องทำซ้ำขั้นตอนนี้สำหรับแต่ละเบราว์เซอร์!



ในบทแนะนำนี้ เราจะมาแก้ปัญหาง่ายๆ**: จัดหาโปรแกรมบล็อกโฆษณาให้กับเครื่องทั้งหมดในเครือข่ายของเราและจัดการมันจากศูนย์กลาง**



ในการทำเช่นนี้ เราจะใช้เครื่องมือที่พัฒนาขึ้นเพื่อวัตถุประสงค์นี้: **Pi-Hole**



Pi-Hole เป็น DNS sinkhole มันจะใช้คำขอ DNS ที่ทำโดยอุปกรณ์ของคุณเพื่อยืนยันหรือปฏิเสธการจราจร ดังนั้นจึงปกป้องคุณจากที่อยู่และโดเมนที่รู้จักว่ากำลังแจกจ่ายโฆษณา มัลแวร์ และอื่นๆ



DNS ย่อมาจาก Domain Name System แล้วชื่อโดเมนคืออะไร? "it-connect.fr" เป็นเพียงตัวอย่างหนึ่ง ชื่อโดเมนเป็นตัวระบุเฉพาะสำหรับทรัพยากรหนึ่งหรือมากกว่า ซึ่งมักจะถูกจัดการโดยหน่วยงานเดียว



ชื่อเครื่องบวกกับชื่อโดเมนเรียกว่า FQDN สำหรับ *Fully Qualified Domain Name* มันช่วยให้คุณเข้าถึงเครื่องเฉพาะได้เพียงแค่ "เรียก" มัน ตัวอย่างเช่น เมื่อคุณพิมพ์ "www.trucmachin.com" คุณกำลังเรียกเครื่อง "www" ซึ่งเป็นส่วนหนึ่งของโดเมน "trucmachin.com"



ยกเว้นว่าคอมพิวเตอร์ของเราไม่เข้าใจภาษามนุษย์ สิ่งที่พวกมันเข้าใจคือเลขฐานสอง ดังนั้นพวกมันจึงต้องการที่อยู่ IP ซึ่งเทียบเท่ากับหมายเลขโทรศัพท์ เพื่อเข้าถึงเว็บไซต์



ดังนั้นทุกครั้งที่คุณป้อนชื่อเว็บไซต์ในเบราว์เซอร์ของคุณ หรือคลิกที่ลิงก์ คอมพิวเตอร์ของคุณจะถามเซิร์ฟเวอร์ DNS ก่อนเพื่อขอที่อยู่ IP ที่สอดคล้องกับชื่อนั้น



**Pi-Hole จะตรวจสอบคำขอเหล่านี้ (มีหลายร้อยคำขอทุกวัน!) และบล็อกโดยอัตโนมัติสำหรับคำขอที่ทราบว่าเป็นโฮสต์โฆษณาหรือแม้แต่ไฟล์ที่เป็นอันตราย**



## II. การติดตั้ง Pi-Hole



ด้วยชื่ออย่าง Pi-Hole คุณอาจคิดว่าคุณต้องใช้ Raspberry-Pi... แต่จริงๆ แล้วไม่ใช่แบบนั้น **Pi-Hole สามารถติดตั้งบนคอมพิวเตอร์ Linux ใดก็ได้ (Debian, Fedora, Rocky, Ubuntu, ฯลฯ)**



ในทางกลับกัน คุณต้องจำไว้ว่ **าอุปกรณ์นี้จะต้องทำงานตลอด 24 ชั่วโมงต่อวันด้วยเหตุผลง่ายๆ: ไม่มี DNS, ไม่มีอินเทอร์เน็ต!** ดังนั้น Raspberry จึงเป็นความคิดที่ดี เพราะมันแทบไม่ใช้พลังงานเลย



ในการติดตั้ง เพียงเชื่อมต่อกับเครื่อง Linux ของคุณผ่าน SSH และป้อนคำสั่งต่อไปนี้ในฐานะ "*root*":



```
curl -sSL https://install.pi-hole.net | bash
```



> **หมายเหตุ**: โดยปกติแล้ว ไม่แนะนำให้ "แฮ็ก" สคริปต์โดยไม่ทราบก่อนว่ามันทำอะไร หากคุณไม่แน่ใจ ให้ไปที่หน้าเว็บด้วยเบราว์เซอร์หรือดาวน์โหลดเนื้อหาเป็นไฟล์
>


> **หมายเหตุ: ในเวอร์ชันขั้นต่ำของ Debian 11, Curl ไม่ได้ติดตั้งไว้ ดังนั้นคุณจำเป็นต้องติดตั้งด้วยตนเองโดยใช้คำสั่ง** `apt-get install curl` **ก่อนที่จะพิมพ์คำสั่งข้างต้น**

> **หมายเหตุ: หากคุณอยู่ในบริษัท มีโอกาสที่เซิร์ฟเวอร์ DNS ปัจจุบันของคุณจะเป็น Active Directory domain controller แต่ไม่ต้องกังวล คุณสามารถระบุการเปลี่ยนเส้นทางตามเงื่อนไขสำหรับโดเมนที่คุณเลือกได้ในภายหลัง โดยทั่วไป คุณจะสามารถเปลี่ยนเส้นทางคำขอใด ๆ ที่เกี่ยวข้องกับโดเมนท้องถิ่นของคุณไปยังเซิร์ฟเวอร์ DNS ของคุณได้**

คุณจะสังเกตเห็นว่าบางตัวเลือกมีตัวเลือก DNSSEC รวมอยู่ด้วย โดยพื้นฐานแล้ว โปรโตคอล DNS ไม่ปลอดภัย (มันไม่ได้ถูกออกแบบมาโดยคำนึงถึงสิ่งนี้ในตอนแรก) DNSSEC แก้ปัญหานี้โดยการเพิ่มชั้นความปลอดภัยผ่านการเข้ารหัสและการลงนามของการแลกเปลี่ยนข้อมูล ตามที่อธิบายไว้ในบทความที่เกี่ยวข้อง: [DNS Security](https://www.it-connect.fr/securite-dns-doh-quest-ce-le-dns-over-https/)

## III. การกำหนดค่า DHCP



เพื่อให้ทำงานได้ Pi-Hole จำเป็นต้อง "แก้ไข" คำขอ DNS จากลูกค้า ดังนั้นพวกเขาต้องรู้ว่ามันคืออุปกรณ์ที่ต้องส่งคำขอไป มีหลายวิธีในการทำเช่นนี้:





- แก้ไขการตั้งค่า DNS ในเซิร์ฟเวอร์ DHCP ของคุณ (เช่น Box ของคุณ)
- ปิดการใช้งานเซิร์ฟเวอร์นี้และใช้เซิร์ฟเวอร์ที่จัดหาโดย Pi-Hole
- ปรับเปลี่ยนอุปกรณ์แต่ละเครื่องด้วยตนเองเพื่อใช้ Pi-Hole เป็น DNS



ฉันเลือกวิธีแก้ปัญหาแรกเป็นการส่วนตัว มีโอกาสที่ **คุณจะมีเซิร์ฟเวอร์ DHCP ที่ที่คุณอยู่** (โดยปกติคือกล่องของคุณ) ดังนั้นจึงไม่จำเป็นต้องกังวล



เนื่องจากมีความเป็นไปได้จำนวนมาก ระหว่างกล่องของผู้ให้บริการต่างๆ (ซึ่งฉันไม่ทราบทั้งหมด) และผู้ที่มีเราเตอร์ของตนเอง ฉันจะไม่ให้ภาพหน้าจอสำหรับการปรับเปลี่ยนเหล่านี้ ไม่ว่าในกรณีใด คุณจะต้องไปที่การตั้งค่า DHCP และแก้ไขพารามิเตอร์ "DNS" เพื่อรวมที่อยู่ IP ของ Pi-Hole ของคุณ



เมื่อดำเนินการเสร็จสิ้นแล้ว หากมีอุปกรณ์ใดถูกเปิดใช้งานมาก่อน อุปกรณ์เหล่านั้นจะยังคงเก็บการตั้งค่าเดิมไว้ ดังนั้นคุณจะต้องเริ่มคำขอการกำหนดค่าใหม่อีกครั้ง



บนเวิร์กสเตชัน Windows ด้วยพรอมต์คำสั่ง:



```
ipconfig /renew
```



บนเวิร์กสเตชัน Linux:



```
dhclient
```



สำหรับอุปกรณ์อื่น ๆ ทั้งหมด ต้องปิดและเปิดใหม่อีกครั้ง



ดังนั้นพวกเขาควรได้รับพารามิเตอร์ที่ถูกต้อง เพื่อตรวจสอบ:



```
ipconfig /all
```



ในช่อง DNS คุณควรใส่ที่อยู่ของ Pi-Hole ของคุณ ในกรณีของฉันคือ 192.168.1.42:

## IV. การใช้ Interface เว็บ Pi-Hole



เพื่ออำนวยความสะดวกในการบริหารจัดการ **Pi-Hole** ได้รับประโยชน์จากอินเทอร์เฟซเว็บ Interface ที่ออกแบบมาอย่างดี ใช้งานง่ายและเข้าถึงได้ ช่วยให้คุณ:





- ดูจำนวนคำขอ คำขอที่ถูกบล็อก ฯลฯ แบบเรียลไทม์
- จัดการรายชื่อที่อนุญาตและรายชื่อที่ถูกบล็อก
- เพิ่มรายการคงที่, นามแฝง, ฯลฯ
- เพิ่มรายการ
- และฟังก์ชันอื่น ๆ อีกมากมาย!



สำหรับส่วนของฉัน ฉันจะเพิ่มรายการบล็อก ตามที่กล่าวไว้ข้างต้น มีการติดตั้งเพียงรายการเดียวในเวลาเดียวกับ Soft มีรายการมากมายสำหรับเว็บไซต์โฆษณา แต่ควรเลือกอย่างน้อยหนึ่งรายการที่เฉพาะเจาะจงกับประเทศที่คุณอาศัยอยู่ หนึ่งในรายการที่รู้จักกันดีที่สุดคือ **EasyList** และหนึ่งในนั้นเฉพาะสำหรับฝรั่งเศส: [EasyList-ListFR](https://raw.githubusercontent.com/deathbybandaid/piholeparser/master/Subscribable-Lists/ParsedBlacklists/EasyList-Liste-FR.txt)



ในการเพิ่ม ให้เชื่อมต่อกับ Interface admin ก่อน: **http://<ip_du_PiHole>/admin**

ตัวอย่างเช่น เราสามารถเห็นได้ว่ามีลูกค้าสองรายที่เชื่อมต่อกับ Pi-Hole ซึ่งได้ประมวลผลคำขอ 442 รายการ และในจำนวนนี้มี 8 รายการที่ถูกบล็อก กราฟเหล่านี้สามารถเป็นแหล่งข้อมูลที่ดี โดยเฉพาะในบริบททางวิชาชีพ



ในการเพิ่มรายการของเรา ให้ไปที่เมนู "**การจัดการกลุ่ม**" และ "**รายการโฆษณา**":

เพิ่มรายการใน Pi-Hole



สิ่งที่เหลืออยู่คือการคลิกที่ "**เพิ่ม**" เพื่อเพิ่มมันเข้าไป เพื่อเปิดใช้งาน เราจำเป็นต้องทำขั้นตอนเพิ่มเติมเพื่อ "แจ้ง" Pi-Hole ให้เข้าควบคุมรายการนี้ ในการทำเช่นนี้:





- ใช้บรรทัดคำสั่งที่มีอยู่ในตัว
- ไม่ว่าจะเป็นเว็บ Interface



ฉันเลือกอันที่สองเป็นการส่วนตัว เพราะถ้าคุณดูอย่างละเอียด ลิงก์ไปยังสคริปต์ PHP ที่ทำการอัปเดตจะอยู่ตรงหน้าบนหน้าเว็บที่เราอยู่ (คำว่า "ออนไลน์") ดังนั้นสิ่งที่คุณต้องทำคือคลิกที่มัน ซึ่งจะพาคุณไปยังหน้าที่มีเพียงตัวเลือกเดียว:



![Image](assets/fr/033.webp)



หน้าจะแสดงผลลัพธ์ของสคริปต์เมื่อเสร็จสิ้น ซึ่งหมายความว่ารายการได้ถูกนำมาพิจารณาแล้ว (เว้นแต่ว่ามีการแสดงข้อความแสดงข้อผิดพลาด)



ตามที่ได้ประกาศไว้ในตอนต้นของบทแนะนำนี้ Pi-Hole ยังช่วยให้คุณ **บล็อกโดเมนที่ทราบว่ามีการแจกจ่ายมัลแวร์ เพื่อเสริมความแข็งแกร่งให้กับฟีเจอร์นี้ ฉันขอแนะนำให้คุณเพิ่มรายการโดเมนที่อัปเดตเป็นประจำซึ่งแจกจ่ายโดย Abuse.ch** ซึ่งจะช่วยเพิ่มความปลอดภัยของเครือข่ายของคุณได้อย่างมาก สามารถเข้าถึงได้ที่ [ที่อยู่นี้](https://urlhaus.abuse.ch/downloads/hostfile/)



คุณสามารถเพิ่มรายการใด ๆ ที่คุณคิดว่าเกี่ยวข้อง หรือจัดการบัญชีดำของคุณด้วยตนเองผ่านเมนูบัญชีดำ



## V. Pi-Hole tests



ตอนนี้ทุกอย่างพร้อมแล้ว สิ่งที่คุณต้องทำคือทดสอบวิธีแก้ปัญหาเพื่อให้แน่ใจว่ามันทำงานได้อย่างถูกต้อง



ตัวอย่างเช่น ฉันจะพยายามเข้าถึงโดเมน *http://admin.gentbcn.org/* ซึ่งอยู่ในรายการของ Abuse.ch เนื่องจากเป็นที่รู้จักว่าโฮสต์มัลแวร์:



![Image](assets/fr/034.webp)
