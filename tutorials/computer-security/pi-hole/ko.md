## I. 프레젠테이션



즐겨 사용하는 브라우저를 시작하자마자 **애드블로커**(광고 차단기)를 설치하는 것은 누구나 한 번쯤 해봤을 일입니다. 하지만 TV 브라우저나 Android 기기 등을 사용할 때는... 제대로 작동하는 것을 찾기가 조금 더 까다롭습니다. 집에 디바이스가 두 대 이상 있는 경우 각 브라우저에 대해 이 작업을 반복해야 합니다!



이 튜토리얼에서는 네트워크의 모든 디바이스에 광고 차단 기능을 제공하고 중앙에서 관리하는 간단한 문제**를 해결해 보겠습니다



이를 위해 이러한 목적으로 개발된 도구를 사용할 것입니다: **파이홀**



파이홀은 DNS 싱크홀입니다. 사용자의 디바이스에서 생성된 DNS 요청을 사용하여 트래픽을 확인하거나 거부함으로써 광고, 멀웨어 등을 배포하는 것으로 알려진 주소 및 도메인으로 부터 사용자를 보호합니다.



DNS는 도메인 이름 시스템의 약자입니다. 그렇다면 도메인 네임이란 무엇일까요? "it-connect.fr"은 한 가지 예일 뿐입니다. 도메인 네임은 하나 이상의 리소스에 대한 고유 식별자로, 일반적으로 단일 주체가 관리합니다.



컴퓨터 이름에 도메인 이름을 더한 것을 *정격화된 도메인 이름*의 FQDN이라고 합니다. 이를 통해 특정 머신을 "호출"하는 것만으로 해당 머신에 연결할 수 있습니다. 예를 들어 "www.trucmachin.com"를 입력하면 실제로는 "trucmachin.com" 도메인에 속하는 "www"라는 컴퓨터를 호출하는 것입니다.



컴퓨터는 인간의 언어를 이해하지 못하고 2진수만 이해하기 때문에 웹사이트에 접속하려면 전화번호에 해당하는 IP Address이 필요합니다.



따라서 브라우저에 웹사이트 이름을 입력하거나 링크를 클릭할 때마다 컴퓨터는 먼저 해당 이름에 해당하는 IP Address을 DNS 서버에 요청합니다.



**파이홀은 이러한 요청(매일 수백 건에 달합니다!)을 검사하고 광고나 악성 파일을 호스팅하는 것으로 알려진 요청을 자동으로 차단합니다**



## II. 파이홀 설치



파이홀과 같은 이름 때문에 라즈베리파이가 필요할 것이라고 생각할 수 있습니다. 하지만 그건 사실이 아닙니다. **파이홀은 모든 리눅스 컴퓨터(데비안, 페도라, 록키, 우분투 등)**에 설치할 수 있습니다



반면에 **이 장치는 DNS, 인터넷 없음이라는 간단한 이유로 하루 24시간 실행되어야 한다는 점을 명심해야 합니다!** 따라서 에너지를 거의 소비하지 않기 때문에 라즈베리는 좋은 생각입니다.



설치하려면 SSH를 통해 Linux 컴퓨터에 연결하고 다음 명령을 "*root*"로 입력하기만 하면 됩니다:



```
curl -sSL https://install.pi-hole.net | bash
```



> **참고**: 일반적인 상황에서는 스크립트의 기능을 먼저 파악하지 않고 스크립트를 '해킹'하는 것은 바람직하지 않습니다. 확실하지 않은 경우 브라우저로 해당 페이지로 이동하거나 콘텐츠를 파일로 다운로드하세요.
>

## III. DHCP 구성



파이홀이 작동하려면 클라이언트의 DNS 요청을 '확인'해야 하므로 클라이언트는 파이홀이 요청을 보낼 곳이라는 것을 알아야 합니다. 이를 수행하는 방법에는 여러 가지가 있습니다:





- DHCP 서버(예: Box)에서 DNS 설정 수정하기
- 이 서버를 비활성화하고 Pi-Hole에서 제공하는 서버를 사용하세요
- 파이홀을 DNS로 사용하도록 각 장치를 수동으로 수정합니다



저는 개인적으로 첫 번째 솔루션을 선택합니다. 현재 위치(보통 박스)에 DHCP 서버가 있을 가능성이 높습니다. 따라서 귀찮게 할 필요가 없습니다.



여러 통신사 박스(제가 모두 알지 못하는)와 자체 라우터를 사용하는 사람 사이에는 많은 가능성이 있으므로 이러한 수정에 대한 스크린샷은 제공하지 않겠습니다. 어쨌든 DHCP 설정으로 이동하여 파이 홀의 IP Address을 포함하도록 "DNS" 매개변수를 수정해야 합니다.



이 작업이 완료되면 이전에 전원을 켠 적이 있는 디바이스가 있는 경우 이전 설정이 유지되므로 구성 요청을 다시 시작해야 합니다.



Windows 워크스테이션의 경우 명령 프롬프트가 있습니다:



```
ipconfig /renew
```



Linux 워크스테이션에서:



```
dhclient
```



다른 모든 기기의 경우 전원을 껐다가 다시 켜야 합니다.



따라서 올바른 매개 변수를 가져와서 확인해야 합니다:



```
ipconfig /all
```



DNS 필드에 파이 홀의 Address(제 경우에는 192.168.1.42)이 있어야 합니다:

## IV. Interface 웹 파이홀 사용



관리를 용이하게 하기 위해 **Pi-Hole**은 잘 설계된 웹 Interface의 이점을 활용합니다. 사용자 친화적이고 접근성이 뛰어납니다:





- 요청 수, 차단된 요청 등을 실시간으로 확인할 수 있습니다.
- 화이트리스트 및 블랙리스트 관리
- 정적 항목, 별칭 등을 추가합니다.
- 목록 추가
- 그 밖에도 다양한 기능이 있습니다!



저는 차단 목록을 추가해 보겠습니다. 위에서 언급했듯이 Soft과 동시에 하나의 목록만 설치되었습니다. 광고 사이트 차단 목록은 여러 가지가 있지만, 자신이 거주하는 국가와 관련된 목록을 하나 이상 선택하는 것이 가장 좋습니다. 가장 잘 알려진 목록 중 하나는 **EasyList**이며, 그 중 하나는 프랑스에 특화된 목록입니다: [EasyList-ListFR](https://raw.githubusercontent.com/deathbybandaid/piholeparser/master/Subscribable-Lists/ParsedBlacklists/EasyList-Liste-FR.txt)



추가하려면 먼저 Interface 관리자에 접속하세요: **http://<ip_du_PiHole>/admin**



관리자 비밀번호는 이미 생성되어 있으므로(설치 종료 스크린샷 참조), 이 비밀번호를 입력하기만 하면 Interface에 액세스할 수 있습니다:



![Image](assets/fr/030.webp)



파이홀의 Interface



예를 들어 파이홀에 연결된 고객이 두 명이고 442건의 요청을 처리했으며 이 중 8건이 차단되었다는 것을 알 수 있습니다. 이러한 그래프는 특히 전문적인 맥락에서 좋은 정보원이 될 수 있습니다.



목록을 추가하려면 '**그룹 관리**' 및 '**광고 목록**' 메뉴로 이동합니다:



![Image](assets/fr/031.webp)



첫 번째 목록 "**StevenBlack**"을 볼 수 있으며, 우리 목록을 추가하려면 위에서 제공 한 링크를 복사하여 "**Address**"필드에 삽입하고 설명에 목록 이름을 입력하도록 선택합니다:



![Image](assets/fr/032.webp)



파이홀에 목록 추가하기



이제 "**추가**"를 클릭하여 추가하기만 하면 됩니다. 이를 활성화하려면 파이홀이 이 목록을 인수하도록 "경고"하는 추가 단계를 수행해야 합니다. 이렇게 하려면





- 기본 제공 명령줄을 사용하거나
- Interface 웹



저는 개인적으로 두 번째를 선택했는데, 자세히 보셨다면 업데이트를 수행하는 PHP 스크립트 링크가 현재 페이지("온라인"이라는 단어)에 바로 표시되어 있기 때문입니다. 따라서 해당 링크를 클릭하기만 하면 옵션이 하나만 있는 페이지로 이동합니다:



![Image](assets/fr/033.webp)



스크립트가 완료되면 페이지에 스크립트 결과가 표시되며, 이는 목록이 고려되었음을 의미합니다(물론 오류 메시지가 표시되지 않는 한).



이 튜토리얼의 시작 부분에서 설명한 것처럼, Pi-Hole은 멀웨어를 배포하는 것으로 알려진 도메인을 **차단할 수 있습니다. 이 기능을 강화하려면 [이 Address](https://urlhaus.abuse.ch/downloads/hostfile/)에서 제공되는 Abuse.ch**에서 배포하는 정기적으로 업데이트되는 도메인 목록을 추가하여 네트워크의 보안을 크게 강화할 것을 권장합니다.



물론 관련성이 있다고 생각되는 목록을 추가하거나 블랙리스트 메뉴를 통해 수동으로 블랙리스트를 관리할 수도 있습니다.



## V. 파이홀 테스트



이제 모든 것이 준비되었으므로 솔루션을 테스트하여 제대로 작동하는지 확인하기만 하면 됩니다.



예를 들어 악성코드를 호스팅하는 것으로 알려져 있어 Abuse.ch 목록에 있는 *http://admin.gentbcn.org/* 도메인에 접속해 보겠습니다:



![Image](assets/fr/034.webp)



분명히 어딘가에서 차단된 것 같습니다. 파이홀이 작업을 수행했는지 확인하려면 Interface 웹 "쿼리 로그"에서 쿼리 로그를 확인하여 목록 항목에서 차단된 것을 확인할 수 있습니다:



![Image](assets/fr/035.webp)



## VI. 결론



이 튜토리얼에서는 쾌적한 브라우징을 위해 대부분의 광고를 제거할 뿐만 아니라 피싱 및 멀웨어 유포 도메인을 차단하여 **Layer 수준의 보안**을 추가하는 DNS 서버를 설정하는 방법을 보여드리겠습니다.



모두 무료이며, 라즈베리파이에 설치하면 전력 소비 측면에서 경제적입니다.
