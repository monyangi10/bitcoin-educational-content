هنگام دانلود نرم‌افزار، بسیار مهم است که اطمینان حاصل کنید که نرم‌افزار تغییر نکرده و واقعاً از منبع رسمی آمده است. این موضوع به‌ویژه برای نرم‌افزارهای مرتبط با Bitcoin، مانند نرم‌افزار Wallet که به شما امکان می‌دهد کلیدهای دسترسی به وجوه خود را ایمن کنید، صادق است. در این آموزش، خواهیم دید که چگونه قبل از نصب، یکپارچگی و اصالت نرم‌افزار را تأیید کنیم. ما از Sparrow wallet به عنوان مثال استفاده خواهیم کرد، که یک نرم‌افزار محبوب Wallet در میان بیت‌کوینرها است، اما این روش برای هر نرم‌افزار دیگری نیز یکسان خواهد بود.


تأیید یکپارچگی شامل اطمینان از این است که فایل دانلود شده با مقایسه اثر انگشت دیجیتال آن (یعنی Hash آن) با نسخه ارائه شده توسط توسعه‌دهنده رسمی تغییر نکرده است. اگر این دو مطابقت داشته باشند، به این معنی است که فایل با نسخه اصلی یکسان است و توسط یک مهاجم خراب یا تغییر نکرده است.


از سوی دیگر، تأیید اصالت اطمینان حاصل می‌کند که فایل واقعاً از توسعه‌دهنده رسمی است و نه یک جعل‌کننده. این کار با تأیید یک امضای دیجیتال انجام می‌شود. این امضا ثابت می‌کند که نرم‌افزار با کلید خصوصی توسعه‌دهنده قانونی امضا شده است.


اگر این بررسی‌ها انجام نشود، خطر نصب بدافزاری وجود دارد که ممکن است حاوی کدهای تغییر یافته باشد. این کد می‌تواند اطلاعاتی مانند کلیدهای خصوصی شما را سرقت کند یا دسترسی به فایل‌های شما را مسدود کند. این نوع حمله بسیار رایج است، به ویژه در زمینه نرم‌افزارهای متن‌باز که نسخه‌های جعلی می‌توانند توزیع شوند.


برای انجام این تأیید، از دو ابزار استفاده خواهیم کرد: توابع هش برای تأیید یکپارچگی و GnuPG، یک ابزار متن‌باز که پروتکل PGP را پیاده‌سازی می‌کند، برای تأیید اصالت.


## پیش‌نیازها


اگر از **لینوکس** استفاده می‌کنید، GPG در بیشتر توزیع‌ها به‌صورت پیش‌فرض نصب شده است. در غیر این صورت، می‌توانید آن را با استفاده از فرمان زیر نصب کنید:


```bash
sudo apt install gnupg
```


برای **macOS**، اگر هنوز مدیر بسته Homebrew را نصب نکرده‌اید، با استفاده از دستورات زیر این کار را انجام دهید:


```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```


```bash
echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile
```


```bash
eval "$(/opt/homebrew/bin/brew shellenv)"
```


سپس GPG را با این دستور نصب کنید:


```bash
brew install gnupg
```

برای **ویندوز**، اگر GPG ندارید، می‌توانید نرم‌افزار [Gpg4win](https://www.gpg4win.org/) را نصب کنید.

## در حال دانلود اسناد


برای شروع، به مدارک مختلفی نیاز خواهیم داشت. به سایت رسمی [Sparrow wallet در بخش "*دانلود*"](https://sparrowwallet.com/download/) مراجعه کنید. اگر می‌خواهید نرم‌افزار دیگری را تأیید کنید، به وب‌سایت آن نرم‌افزار بروید.


![GnuPG](assets/notext/02.webp)


شما همچنین می‌توانید [به مخزن GitHub پروژه](https://github.com/sparrowwallet/Sparrow/releases) بروید.


![GnuPG](assets/notext/03.webp)


نصب‌کننده نرم‌افزار مربوط به سیستم‌عامل خود را دانلود کنید.


![GnuPG](assets/notext/04.webp)


شما همچنین به فایل Hash نیاز خواهید داشت که اغلب "*SHA256SUMS*" یا "*MANIFEST*" نامیده می‌شود.


![GnuPG](assets/notext/05.webp)


امضای PGP فایل را نیز دانلود کنید. این سند در قالب `.asc` است.


![GnuPG](assets/notext/06.webp)


مطمئن شوید که تمام این فایل‌ها را برای مراحل بعدی در یک پوشه قرار دهید.


در نهایت، شما به کلید عمومی توسعه‌دهنده نیاز خواهید داشت که ما از آن برای تأیید امضای PGP استفاده خواهیم کرد. این کلید اغلب یا در وب‌سایت نرم‌افزار، در مخزن GitHub پروژه، گاهی در شبکه‌های اجتماعی توسعه‌دهنده، یا در سایت‌های تخصصی مانند Keybase در دسترس است. در مورد Sparrow wallet، شما می‌توانید کلید عمومی توسعه‌دهنده، Craig Raw، را [در Keybase](https://keybase.io/craigraw) پیدا کنید. برای دانلود مستقیم آن از طریق ترمینال، فرمان زیر را اجرا کنید:


```bash
curl https://keybase.io/craigraw/pgp_keys.asc | gpg --import
```


![GnuPG](assets/notext/07.webp)


## تأیید امضا


فرآیند تأیید امضا در **ویندوز**، **macOS** و **لینوکس** یکسان است. معمولاً، شما کلید عمومی را در مرحله قبلی وارد کرده‌اید، اما اگر این کار را نکرده‌اید، با استفاده از دستور زیر این کار را انجام دهید:


```bash
gpg --import [key path]
```


جایگزین کردن `[key path]` با مکان فایل کلید عمومی توسعه‌دهنده.


![GnuPG](assets/notext/08.webp)


امضا را با استفاده از فرمان زیر تأیید کنید:


```bash
gpg --verify [file.asc]
```


جایگزین کردن `[file.asc]` با مسیر فایل امضا. در مورد Sparrow، این فایل به نام "*Sparrow-2.0.0-manifest.txt.asc*" برای نسخه 2.0.0 است.


![GnuPG](assets/notext/09.webp)


اگر امضا معتبر باشد، GPG این را به شما نشان خواهد داد. سپس می‌توانید به مرحله بعد بروید، زیرا این تأیید می‌کند که فایل معتبر است.


![GnuPG](assets/notext/10.webp)


## تأیید Hash

اکنون که اصالت نرم‌افزار تأیید شده است، لازم است یکپارچگی آن نیز بررسی شود. ما Hash نرم‌افزار را با Hash ارائه‌شده توسط توسعه‌دهنده مقایسه خواهیم کرد. اگر این دو مطابقت داشته باشند، تضمین می‌کند که کد نرم‌افزار تغییر نکرده است.


روی **ویندوز**، یک ترمینال باز کرده و فرمان زیر را اجرا کنید:


```bash
CertUtil -hashfile [file path] SHA256 | findstr /v "hash"
```


جایگزین کردن `[file path]` با مکان نصب‌کننده.


![GnuPG](assets/notext/11.webp)


ترمینال Hash نرم‌افزار دانلود شده را برمی‌گرداند.


![GnuPG](assets/notext/12.webp)


آگاه باشید که برای برخی نرم‌افزارها، ممکن است لازم باشد از تابع Hash متفاوتی به جای SHA256 استفاده کنید. در این صورت، به سادگی نام تابع Hash را در فرمان جایگزین کنید.


سپس نتیجه را با مقدار مربوطه در فایل "*Sparrow-2.0.0-manifest.txt*" مقایسه کنید.


![GnuPG](assets/notext/13.webp)


در مورد من، می‌بینیم که دو هش به‌طور کامل مطابقت دارند.


در **macOS** و **Linux**، فرآیند تأیید Hash به صورت خودکار انجام می‌شود. لازم نیست که به صورت دستی تطابق بین دو هش را مانند ویندوز بررسی کنید.


به سادگی این دستور را در **macOS** اجرا کنید:


```bash
shasum --check [file name] --ignore-missing
```


جایگزین کردن `[file name]` با نام نصب‌کننده. برای مثال، برای Sparrow wallet:


```bash
shasum --check sparrow-2.0.0-manifest.txt --ignore-missing
```


اگر هش‌ها مطابقت داشته باشند، باید خروجی زیر را ببینید:


```bash
Sparrow-2.0.0.dmg: OK
```


روی **لینوکس**، دستور مشابه است:


```bash
sha256sum --check [file name] --ignore-missing
```


و اگر هش‌ها مطابقت داشته باشند، باید خروجی زیر را ببینید:


```bash
sparrow_2.0.0-1_amd64.deb: OK
```


اکنون اطمینان دارید که نرم‌افزاری که دانلود کرده‌اید هم معتبر و هم سالم است. می‌توانید نصب آن را بر روی دستگاه خود ادامه دهید.


اگر این آموزش برای شما مفید بود، از شما خواهش می‌کنم که در زیر یک لایک بدهید. لطفاً این مقاله را در شبکه‌های اجتماعی خود به اشتراک بگذارید. بسیار متشکرم!
