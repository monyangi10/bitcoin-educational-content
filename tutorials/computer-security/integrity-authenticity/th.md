เมื่อดาวน์โหลดซอฟต์แวร์ สิ่งสำคัญมากคือต้องมั่นใจว่าซอฟต์แวร์นั้นไม่ได้ถูกแก้ไขและมาจากแหล่งที่เป็นทางการจริง ๆ สิ่งนี้สำคัญอย่างยิ่งสำหรับซอฟต์แวร์ที่เกี่ยวข้องกับ Bitcoin เช่น ซอฟต์แวร์ wallet ซึ่งช่วยให้คุณรักษาความปลอดภัยของกุญแจที่ให้การเข้าถึงเงินทุนของคุณ ในบทแนะนำนี้ เราจะดูวิธีการตรวจสอบความสมบูรณ์และความถูกต้องของซอฟต์แวร์ก่อนการติดตั้ง เราจะใช้ Sparrow Wallet เป็นตัวอย่าง ซึ่งเป็นซอฟต์แวร์ wallet ที่ชื่นชอบในหมู่ผู้ใช้บิทคอยน์ แต่ขั้นตอนจะเหมือนกันสำหรับซอฟต์แวร์อื่น ๆ ทุกชนิด


การตรวจสอบความสมบูรณ์เกี่ยวข้องกับการทำให้แน่ใจว่าไฟล์ที่ดาวน์โหลดมาไม่ได้ถูกแก้ไขโดยการเปรียบเทียบลายนิ้วมือดิจิทัลของไฟล์ (เช่น แฮชของไฟล์) กับที่ผู้พัฒนาอย่างเป็นทางการให้ไว้ หากทั้งสองตรงกัน หมายความว่าไฟล์นั้นเหมือนกับต้นฉบับและไม่ได้ถูกทำลายหรือแก้ไขโดยผู้โจมตี


ในทางกลับกัน การตรวจสอบความถูกต้องช่วยให้มั่นใจได้ว่าไฟล์นั้นมาจากนักพัฒนาอย่างเป็นทางการจริง ๆ ไม่ใช่ผู้แอบอ้าง ซึ่งทำได้โดยการตรวจสอบลายเซ็นดิจิทัล ลายเซ็นนี้พิสูจน์ได้ว่าซอฟต์แวร์นั้นถูกลงนามด้วยกุญแจส่วนตัวของนักพัฒนาที่ถูกต้องตามกฎหมาย


หากไม่ได้ดำเนินการตรวจสอบเหล่านี้ มีความเสี่ยงที่จะติดตั้งมัลแวร์ที่อาจมีการแก้ไขโค้ด โค้ดนี้อาจขโมยข้อมูลเช่นคีย์ส่วนตัวของคุณหรือบล็อกการเข้าถึงไฟล์ของคุณ การโจมตีประเภทนี้ค่อนข้างพบได้บ่อย โดยเฉพาะในบริบทของซอฟต์แวร์โอเพ่นซอร์สที่สามารถแจกจ่ายเวอร์ชันปลอมได้


ในการดำเนินการตรวจสอบนี้ เราจะใช้เครื่องมือสองอย่าง: ฟังก์ชันแฮชเพื่อยืนยันความสมบูรณ์ และ GnuPG ซึ่งเป็นเครื่องมือโอเพนซอร์สที่ใช้โปรโตคอล PGP เพื่อยืนยันความถูกต้อง


## ข้อกำหนดเบื้องต้น


หากคุณใช้ **Linux** GPG จะถูกติดตั้งล่วงหน้าในดิสทริบิวชันส่วนใหญ่ หากไม่เป็นเช่นนั้น คุณสามารถติดตั้งได้ด้วยคำสั่งต่อไปนี้:


```bash
sudo apt install gnupg
```


สำหรับ **macOS** หากคุณยังไม่ได้ติดตั้งตัวจัดการแพ็กเกจ Homebrew ให้ทำการติดตั้งด้วยคำสั่งต่อไปนี้:


```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```


```bash
echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile
```


```bash
eval "$(/opt/homebrew/bin/brew shellenv)"
```


จากนั้นติดตั้ง GPG ด้วยคำสั่งนี้:


```bash
brew install gnupg
```

สำหรับ **Windows** หากคุณไม่มี GPG คุณสามารถติดตั้งซอฟต์แวร์ [Gpg4win](https://www.gpg4win.org/) ได้

## กำลังดาวน์โหลดเอกสาร


ในการเริ่มต้น เราจะต้องใช้เอกสารต่างๆ เยี่ยมชมเว็บไซต์ทางการของ [Sparrow Wallet ในส่วน "*Download*"](https://sparrowwallet.com/download/) หากคุณต้องการตรวจสอบซอฟต์แวร์อื่น ให้ไปที่เว็บไซต์ของซอฟต์แวร์นั้น


![GnuPG](assets/notext/02.webp)


คุณยังสามารถไป [ที่ที่เก็บ GitHub ของโครงการ](https://github.com/sparrowwallet/sparrow/releases) ได้เช่นกัน


![GnuPG](assets/notext/03.webp)


ดาวน์โหลดตัวติดตั้งสำหรับซอฟต์แวร์ที่ตรงกับระบบปฏิบัติการของคุณ


![GnuPG](assets/notext/04.webp)


คุณจะต้องใช้แฮชของไฟล์ด้วย ซึ่งมักจะเรียกว่า "*SHA256SUMS*" หรือ "*MANIFEST*"


![GnuPG](assets/notext/05.webp)


ดาวน์โหลดลายเซ็น PGP ของไฟล์ด้วย นี่คือเอกสารในรูปแบบ `.asc`


![GnuPG](assets/notext/06.webp)


ตรวจสอบให้แน่ใจว่าได้วางไฟล์ทั้งหมดนี้ในโฟลเดอร์เดียวกันสำหรับขั้นตอนต่อไปนี้


ในที่สุด คุณจะต้องใช้กุญแจสาธารณะของนักพัฒนา ซึ่งเราจะใช้เพื่อตรวจสอบลายเซ็น PGP กุญแจนี้มักจะมีให้ใช้งานบนเว็บไซต์ของซอฟต์แวร์ บน GitHub repository ของโครงการ บางครั้งบนโซเชียลมีเดียของนักพัฒนา หรือบนเว็บไซต์เฉพาะทางเช่น Keybase ในกรณีของ Sparrow Wallet คุณสามารถหากุญแจสาธารณะของนักพัฒนา Craig Raw ได้ [บน Keybase](https://keybase.io/craigraw) เพื่อดาวน์โหลดโดยตรงจากเทอร์มินัล ให้รันคำสั่ง:


```bash
curl https://keybase.io/craigraw/pgp_keys.asc | gpg --import
```


![GnuPG](assets/notext/07.webp)


## การยืนยันลายเซ็น


กระบวนการตรวจสอบลายเซ็นเหมือนกันบน **Windows**, **macOS**, และ **Linux** โดยปกติแล้ว คุณได้ทำการนำเข้าคีย์สาธารณะในขั้นตอนก่อนหน้าแล้ว แต่ถ้ายังไม่ได้ทำ ให้ดำเนินการด้วยคำสั่ง:


```bash
gpg --import [key path]
```


แทนที่ `[key path]` ด้วยตำแหน่งของไฟล์คีย์สาธารณะของนักพัฒนา


![GnuPG](assets/notext/08.webp)


ยืนยันลายเซ็นด้วยคำสั่งต่อไปนี้:


```bash
gpg --verify [file.asc]
```


แทนที่ `[file.asc]` ด้วยเส้นทางของไฟล์ลายเซ็น ในกรณีของ Sparrow ไฟล์นี้เรียกว่า "*sparrow-2.0.0-manifest.txt.asc*" สำหรับเวอร์ชัน 2.0.0


![GnuPG](assets/notext/09.webp)


หากลายเซ็นถูกต้อง GPG จะแจ้งให้คุณทราบ จากนั้นคุณสามารถดำเนินการต่อไปยังขั้นตอนถัดไปได้ เนื่องจากสิ่งนี้ยืนยันความถูกต้องของไฟล์


![GnuPG](assets/notext/10.webp)


## กำลังตรวจสอบ Hash

ตอนนี้ที่ได้รับการยืนยันความถูกต้องของซอฟต์แวร์แล้ว ก็จำเป็นต้องตรวจสอบความสมบูรณ์ของมันด้วย เราจะเปรียบเทียบแฮชของซอฟต์แวร์กับแฮชที่ผู้พัฒนาจัดหาให้ หากทั้งสองตรงกัน จะเป็นการรับประกันว่ารหัสซอฟต์แวร์ไม่ได้ถูกแก้ไข


บน **Windows** ให้เปิดเทอร์มินัลและรันคำสั่งต่อไปนี้:


```bash
CertUtil -hashfile [file path] SHA256 | findstr /v "hash"
```


แทนที่ `[file path]` ด้วยตำแหน่งที่ตั้งของตัวติดตั้ง


![GnuPG](assets/notext/11.webp)


เทอร์มินัลจะส่งคืนแฮชของซอฟต์แวร์ที่ดาวน์โหลดแล้ว


![GnuPG](assets/notext/12.webp)


โปรดทราบว่า สำหรับซอฟต์แวร์บางตัว อาจจำเป็นต้องใช้ฟังก์ชันแฮชที่แตกต่างจาก SHA256 ในกรณีนี้ เพียงแค่แทนที่ชื่อฟังก์ชันแฮชในคำสั่ง


จากนั้นเปรียบเทียบผลลัพธ์กับค่าที่สอดคล้องกันในไฟล์ "*sparrow-2.0.0-manifest.txt*"


![GnuPG](assets/notext/13.webp)


ในกรณีของฉัน เราเห็นว่าแฮชทั้งสองตรงกันอย่างสมบูรณ์แบบ


บน **macOS** และ **Linux** กระบวนการตรวจสอบแฮชจะเป็นแบบอัตโนมัติ ไม่จำเป็นต้องตรวจสอบการจับคู่ระหว่างแฮชทั้งสองด้วยตนเองเหมือนบน Windows


เพียงแค่รันคำสั่งนี้บน **macOS**:


```bash
shasum --check [file name] --ignore-missing
```


แทนที่ `[file name]` ด้วยชื่อของตัวติดตั้ง ตัวอย่างเช่น สำหรับ Sparrow Wallet:


```bash
shasum --check sparrow-2.0.0-manifest.txt --ignore-missing
```


หากแฮชตรงกัน คุณควรเห็นผลลัพธ์ดังต่อไปนี้:


```bash
Sparrow-2.0.0.dmg: OK
```


บน **Linux** คำสั่งจะคล้ายกัน:


```bash
sha256sum --check [file name] --ignore-missing
```


และหากแฮชตรงกัน คุณควรเห็นผลลัพธ์ดังต่อไปนี้:


```bash
sparrow_2.0.0-1_amd64.deb: OK
```


คุณสามารถมั่นใจได้แล้วว่าซอฟต์แวร์ที่คุณดาวน์โหลดนั้นเป็นของแท้และสมบูรณ์ คุณสามารถดำเนินการติดตั้งบนเครื่องของคุณได้


หากคุณพบว่าบทแนะนำนี้มีประโยชน์ ฉันจะขอบคุณถ้าคุณกดถูกใจด้านล่างนี้ อย่าลังเลที่จะแบ่งปันบทความนี้บนเครือข่ายสังคมของคุณ ขอบคุณมาก!
