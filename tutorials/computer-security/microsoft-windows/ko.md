## 다운로드!


가장 먼저 필요한 것은 설치 파일입니다. 가장 안전하고 신뢰할 수 있는 다운로드 장소는 Microsoft의 공식 웹사이트에서 직접 다운로드하는 것입니다.


아래 링크를 방문하여 안내에 따라 [Windows 11 ISO 파일](https://www.microsoft.com/en-us/software-download/windows11)을 다운로드하세요


![Image](assets/en/02.webp)


다운로드 페이지로 이동한 후 아래로 스크롤하여 ISO 파일 다운로드 섹션으로 이동합니다.


![Image](assets/en/01.webp)


그리고 적절한 버전을 선택합니다.


![Image](assets/en/03.webp)


Windows 11을 선택한 후 확인 버튼을 클릭합니다.


이 단계에서 요청을 처리하는 데 몇 초 정도 소요될 수 있으며, 다음 페이지가 표시됩니다:


![Image](assets/en/04.webp)


요청을 확인한 후 선호하는 언어를 선택해야 합니다.


![Image](assets/en/05.webp)


언어를 선택하고 확인 버튼을 클릭하면 요청이 처리됩니다. 이 단계는 몇 초 정도 걸릴 수 있습니다.


요청이 성공적으로 처리되면 .iso 파일의 다운로드 링크가 있는 페이지가 표시됩니다. 64비트 다운로드 버튼을 클릭하여 다운로드를 시작합니다.


파일 크기는 약 5.5GB이며, 생성된 링크는 24시간 동안 유효합니다.


## 자동화!


이 단계에서는 표준 Windows 설치를 변경해야 합니다. 이 단계에서는 무인 설치를 사용하여 나중에 사용자의 입력 없이 설치할 항목을 결정합니다. 실제로 이 방법에서는 XML 파일을 사용하여 Windows에 설치되는 설치 단계와 서비스를 구성합니다. 즉, Unattended.xml 파일을 사용하면 설치 중에 자동화 프로세스가 생성되어 여러 옵션을 선택할 필요가 없고 일반적으로 설치 중에 필요한 지루한 단계를 피할 수 있습니다. 이 방법은 Microsoft에서 도입한 이례적이지만 표준적인 방법입니다. 자세한 내용은 [Microsoft 공식 웹사이트](https://learn.microsoft.com/en-us/windows-hardware/manufacture/desktop/update-windows-settings-and-scripts-create-your-own-answer-file-sxs?view=windows-11)에서 확인할 수 있습니다.


인터넷에는 무인 파일을 생성하는 데 사용할 수 있는 다양한 도구가 있습니다. 그 중 일부는 온라인에 있고 다른 일부는 오프라인에 있습니다. 이 파일을 생성하는 온라인 도구 중 하나는 [이 웹사이트](https://schneegans.de/windows/unattend-generator)입니다. 이 웹사이트를 열면 다음과 같은 페이지가 표시됩니다:

1. TPM 및 보안 부팅과 같은 Windows 보안 기능을 무시할지 여부를 결정합니다. 보안 부팅 기능은 부팅 프로세스 중에 핵심 Windows 파일이 변조되는 경우 문제를 감지하고 해당 파일의 실행을 방지합니다. 또한 이 기능은 Windows에 악성 업데이트가 설치되지 않도록 시스템을 보호하는 데 도움이 됩니다. 특정 컴퓨터, 특히 구형 모델에서는 이러한 기능을 우회하는 옵션을 활성화하는 것이 불가피한 경우가 있습니다. 그러나 일반적으로 보안 부팅과 같은 기능은 활성화된 상태로 유지하는 것이 좋습니다.

2. 프로세스를 완료하기 위한 인터넷 연결 요구 사항을 무시하세요. 대부분의 경우 Windows 설치 중에 무선 카드가 아직 인식되지 않고 케이블을 통한 인터넷 액세스가 필요하므로 유선 LAN 연결을 사용할 수 없는 상황에서 유용합니다. 이 옵션을 활성화하면 이 단계와 관련된 문제가 해결됩니다.

3. 계정을 만들지 않으려면 모든 계정을 정리하고 이 옵션을 선택합니다. 이 경우 Windows 설치 후 자동으로 Windows 관리자 계정으로 로그인됩니다.

## 보안 설정


이 단계에서는


1. Windows Defender를 사용하거나 사용하지 않도록 설정할 수 있습니다. 이 기능은 Windows의 보안 소프트웨어처럼 작동하며 악성 파일의 실행, 특정 네트워크 공격 등을 방지하는 데 도움이 됩니다.

2. Windows 자동 업데이트를 비활성화할 수 있습니다. 이는 Windows 사용자가 직면하는 일반적인 문제 중 하나입니다!

3. 이 섹션에서는 UAC(사용자 계정 컨트롤)를 활성화 또는 비활성화할 수 있습니다. 이 기능을 사용하면 의심스러운 애플리케이션이 읽기 및 쓰기 권한이 상승된 상태로 실행되는 것을 방지할 수 있습니다.

4. 이 기능은 Windows에서 잠재적으로 유해한 소프트웨어를 탐지하는 데 사용됩니다.

5. PowerShell 등의 Windows 애플리케이션에서 긴 경로에 대한 지원을 사용하거나 사용하지 않도록 설정합니다.

6. 원격으로 시스템에 액세스하기 위해 원격 데스크톱을 사용하거나 사용하지 않도록 설정합니다.


사용 중인 Windows 버전에 따라 이러한 기능 중 일부가 지원되거나 지원되지 않을 수 있습니다.

2. 시작 메뉴 아이콘이 나열되며 요구 사항에 따라 추가하거나 제거할 수도 있습니다.

## 기본 소프트웨어 활성화/비활성화


이 섹션에서는 기본적으로 설치하지 않으려는 애플리케이션을 선택할 수 있습니다. 예를 들어 Cortana 또는 Copilot을 설치하지 않도록 선택할 수 있습니다.


다음 단계는 애플리케이션 실행과 관련된 보안 설정입니다:


![Image](assets/en/19.webp)


WDAC 설정을 적용하면 특정 애플리케이션의 실행을 방지할 수 있습니다.


마지막으로 원하는 설정을 적용한 후 생성된 XML 파일을 다운로드할 수 있습니다:


![Image](assets/en/20.webp)


XML 파일 다운로드를 클릭하면 autounattend.xml 파일이 다운로드됩니다. 이 파일을 사용하려면 다운로드한 ISO를 USB 드라이브에 마운트하고 autounattend.xml 파일을 루트 디렉터리에 넣은 다음 Windows 설치를 진행하기만 하면 됩니다.


부팅 가능한 USB 드라이브를 만드는 데 사용할 수 있는 도구 중 하나는 Rufus입니다. Rufus는 주어진 Windows 설치 ISO 파일로 부팅 가능한 Windows 설치 플래시 드라이브를 만들 수 있습니다. 빠르고 간단하며 [여기](https://rufus.ie/en/#download)에서 다운로드할 수 있습니다


![Image](assets/en/21.webp)


이 소프트웨어에서 원하는 USB 드라이브와 적절한 ISO 파일을 선택한 후 시작을 클릭합니다.


![Image](assets/en/22.webp)


이 단계에서는 모든 옵션을 활성화하면 생성된 무인 파일을 사용할 때 충돌이 발생할 수 있으므로 모든 옵션을 비활성화합니다. 파일이 USB 드라이브에 복사된 후 autounattend.xml 파일을 루트 디렉터리에 배치합니다:


![Image](assets/en/23.webp)


이 시점에서 USB 드라이브는 Windows를 자동으로 설치하는 데 사용할 준비가 되었으며 이 드라이브를 사용하여 설치를 시작할 수 있습니다.


## ISO 편집


가상 머신에 Windows를 설치해야 하는 경우 소프트웨어를 사용하여 ISO 파일을 만들고 편집할 수 있습니다. 그러한 소프트웨어 중 하나가 AnyBurn입니다. Microsoft 웹사이트에서 다운로드한 ISO 파일의 내용을 압축을 푼 후 autounattend.xml 파일을 루트 디렉터리에 배치합니다. 그런 다음 AnyBurn을 사용하여 업데이트된 콘텐츠로 새 ISO를 만듭니다.


AnyBurn은 ISO 파일 작업을 위한 다기능 소프트웨어입니다. ISO 파일 처리를 위한 다양한 기능을 제공하며, 그 중 하나는 부팅 가능한 ISO 이미지 생성입니다(여기](https://www.anyburn.com/download.php)가 원본 웹사이트입니다).


소프트웨어의 메인 페이지에서 '파일/폴더에서 이미지 만들기'를 선택합니다:


![Image](assets/en/24.webp)


다음 페이지에서 ISO에서 추출한 모든 파일을 autounattend.xml 파일과 함께 선택합니다.


![Image](assets/en/25.webp)


이 단계에서는 ISO 파일을 부팅할 수 있도록 설정을 구성합니다:


![Image](assets/en/26.webp)


이 단계에서 ISO를 부팅할 수 있도록 bootfix.bin 파일의 경로를 설정해야 합니다. 이 파일은 ISO의 루트, 부팅 폴더 안에 있습니다. 또한 속성 섹션에서 ISO9660 및 UDF 옵션을 모두 활성화하는 것이 좋습니다.


![Image](assets/en/27.webp)
