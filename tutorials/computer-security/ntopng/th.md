## I. การนำเสนอ



**ไม่ว่าจะเป็นการตรวจสอบความลื่นไหลของเครือข่าย เพื่อให้ได้ภาพที่ชัดเจนของการใช้งาน หรือสำหรับสถิติประสิทธิภาพ การตรวจสอบการไหลของเครือข่ายเป็นส่วนสำคัญของเครือข่ายองค์กร** มันช่วยคาดการณ์การเปลี่ยนแปลงของโครงสร้างพื้นฐานและช่วยให้มั่นใจในคุณภาพการใช้งานสำหรับผู้ใช้ (หรือที่รู้จักกันว่า QoE สำหรับ *Quality of Experience* ซึ่งต่างจาก QoS)



ในการทำเช่นนี้ มีเทคนิคและซอฟต์แวร์/โปรโตคอลหลายอย่างที่สามารถใช้ได้ Netflow ตัวอย่างเช่น พัฒนาโดย Cisco สามารถใช้เพื่อดึงสถิติการไหลของ IP จาก Interface แต่การใช้งานจะจำกัดเฉพาะอุปกรณ์ที่เข้ากันได้เท่านั้น



นั่นคือเหตุผลที่ในบทแนะนำนี้ฉันจะมาแนะนำ **Ntop** และแสดงวิธีการใช้ในโครงสร้างพื้นฐานของคุณเพื่อเฝ้าดูการใช้งานเครือข่ายของคุณ



Ntop เป็นซอฟต์แวร์โอเพ่นซอร์สที่สามารถติดตั้งบนเครื่อง Linux ใดก็ได้ ไม่มีค่าใช้จ่ายและสามารถรวบรวมข้อมูลต่อไปนี้:





- การใช้งานแบนด์วิดท์
- ลูกค้าหลัก
- จุดหมายปลายทางหลัก
- โปรโตคอลที่ใช้
- แอปพลิเคชันที่ใช้
- พอร์ตที่ใช้
- เป็นต้น



ปัจจุบันเปลี่ยนชื่อเป็น **Ntopng (New Generation)** ซึ่งมีฟังก์ชันพื้นฐานหลายอย่างให้ใช้ฟรี นอกจากนี้ยังมีเวอร์ชันเชิงพาณิชย์ที่สามารถส่งออกการแจ้งเตือนที่กำหนดค่าไปยังซอฟต์แวร์ประเภท SIEM หรือกรองการจราจรด้วยกฎที่กำหนดโดยตรงจากโพรบได้



## II. ข้อกำหนดเบื้องต้น



การติดตั้ง Ntop probe จะแตกต่างกันไปตามอุปกรณ์และสภาพแวดล้อม ดังนั้นฉันจะไม่ให้คำแนะนำทีละขั้นตอนที่นี่ แต่จะสรุปความเป็นไปได้ต่างๆ ให้คุณทราบ



### A. โหมดบนเครื่อง



หากคุณมีไฟร์วอลล์ pfSense, OPNSense หรือ Endian ในการผลิต หรือแม้แต่เวิร์กสเตชัน Linux ที่มี NFTables ข่าวดี! คุณสามารถติดตั้ง Ntopng ได้โดยตรงและเริ่มตรวจสอบอินเทอร์เฟซของคุณ



ข้อดีของเทคนิคนี้คือไม่ต้องการฮาร์ดแวร์เพิ่มเติม อย่างไรก็ตาม มันเพิ่มการใช้ทรัพยากร ดังนั้นควรตรวจสอบให้แน่ใจว่าคุณมีฮาร์ดแวร์ที่เพียงพอหรือ VM ที่มีขนาดเพียงพอ (ขั้นต่ำ 2 คอร์ และ RAM 2GB)



### B. โหมด TAP / SPAN



**tap** คือ **อุปกรณ์เครือข่ายที่ทำการคัดลอกข้อมูลที่ผ่านมันและส่งไปยังอุปกรณ์อื่น** ข้อดีของอุปกรณ์นี้คือไม่จำเป็นต้องปรับเปลี่ยนโครงสร้างพื้นฐานที่มีอยู่ และสามารถวางไว้ที่ใดก็ได้เพื่อเฝ้าติดตามส่วนต่างๆ ของเครือข่าย หรือระหว่างสวิตช์หลักและเราเตอร์ขอบเพื่อวิเคราะห์ข้อมูลที่เข้า/ออกจากอินเทอร์เน็ต



ข้อเสียใหญ่ของอุปกรณ์ประเภทนี้คือค่าใช้จ่าย ในเครือข่าย Gigabit ปัจจุบัน ตัวแตะพาสซีฟไม่สามารถจับการจราจรของเครือข่ายได้อย่างถูกต้อง ดังนั้นคุณจำเป็นต้องใช้อุปกรณ์แอคทีฟที่มีราคาประมาณ €200 (ขั้นต่ำ)



โหมด **SPAN** หรือที่รู้จักกันในชื่อ **port mirroring** ใช้โดยสวิตช์เพื่อส่งต่อทราฟฟิกจากพอร์ตหนึ่งไปยังอีกพอร์ตหนึ่ง นี่เป็นวิธีที่ฉันชอบที่สุด เพราะตั้งค่าได้ง่ายและเหมือนกับ tap ที่ช่วยให้คุณสามารถตรวจสอบส่วนหนึ่งของเครือข่ายหรือทั้งเครือข่ายได้ตามต้องการ อย่างไรก็ตาม มีข้อเสียสองประการ: สวิตช์ต้องมีฟังก์ชันนี้ในตัว และการใช้งานอาจเพิ่มภาระการประมวลผลบนสวิตช์



### C. โหมดเราเตอร์



เป็นไปได้อย่างสมบูรณ์ที่จะติดตั้งเราเตอร์ภายใต้ Linux และติดตั้ง ntopng บนมัน ข้อเสียเพียงอย่างเดียวของวิธีนี้คือมันจะเปลี่ยนแปลงเครือข่ายของคุณ ไม่ว่าจะเป็นการกำหนดที่อยู่ภายใน หรือการกำหนดที่อยู่ระหว่างเราเตอร์ "จริง" ของคุณกับตัวที่คุณกำลังเพิ่ม



หมายเหตุ: สำหรับผู้อ่านบทความ [Create a Wifi router with Raspberry Pi and RaspAP](https://www.it-connect.fr/creer-un-routeur-wifi-avec-un-raspberry-pi-et-raspap/) เป็นไปได้อย่างสมบูรณ์ที่จะติดตั้ง ntopng บน Rpi ของคุณเพื่อรับสถิติที่แม่นยำ!



### D. โหมดบริดจ์



ทางเลือกที่น่าสนใจคือการใช้ **เครื่อง Linux ในโหมดบริดจ์** วางไว้ระหว่างอุปกรณ์สองชิ้น ซึ่งจะช่วยให้สามารถจับทราฟฟิกทั้งหมดได้โดยไม่ต้องแทรกแซงการกำหนดค่าของโครงสร้างพื้นฐานหรืออุปกรณ์ของมัน เนื่องจากเครื่องเก่าก็สามารถทำงานได้ ต้นทุนของวิธีนี้จึงน่าสนใจเช่นกัน โปรดทราบว่าเพื่อให้ได้ประสิทธิภาพสูงสุด อุปกรณ์ควรมีการ์ดเครือข่ายสามใบ สองใบในโหมดบริดจ์ หนึ่งใบเพื่อเข้าถึง Interface และ SSH สามารถใช้เพียงสองใบได้ แต่ทราฟฟิกการจัดการอุปกรณ์ก็จะถูกจับด้วย...



ดังนั้นมันคือ **โหมดสุดท้ายนี้ที่ฉันจะใช้**. ด้วยเหตุผลด้านปฏิบัติ ฉันจะใช้เครื่องเสมือนสำหรับการสาธิต แต่ก็ยังคงใช้วิธีเดียวกันนี้กับเครื่องจริง



## III. การเตรียมโพรบด้วยสะพาน Interface



สำหรับโพรบ ฉันเลือกเครื่อง **Debian 11** ในการติดตั้งขั้นต่ำ



ขั้นตอนแรก เหมือนเดิมเสมอ อัปเดต:



```
apt-get update && apt-get upgrade
```



จากนั้นติดตั้งแพ็กเกจ **bridge-utils** ซึ่งจะช่วยให้เราสร้างบริดจ์ของเราได้:



```
apt-get install bridge-utils
```



เมื่อติดตั้งแล้ว สิ่งแรกที่ต้องสังเกตคือชื่อปัจจุบันของการ์ดเครือข่ายของเรา ภายใต้ Debian ชื่อนี้สามารถมีหลายรูปแบบ และเราจะต้องใช้มันสำหรับการกำหนดค่า



คำสั่ง **ip add** แบบง่าย ๆ จะส่งคืนผลลัพธ์พร้อมข้อมูลนี้:



![Image](assets/fr/016.webp)



ที่นี่ ฉันเห็น 3 อินเทอร์เฟซ:





- Lo**: นี่คือ loopback Interface; มันเป็น Interface เสมือนที่ "วนกลับ" ไปยังอุปกรณ์ โดยพื้นฐานแล้ว Interface นี้ ซึ่งมีที่อยู่เป็น 127.0.0.1 (แม้ว่าที่อยู่ใด ๆ ใน 127.0.0.0/8 ก็สามารถใช้ได้ เนื่องจากช่วงนี้ถูกสงวนไว้สำหรับวัตถุประสงค์นี้) ถูกใช้เพื่อติดต่อกับอุปกรณ์เอง หากคุณได้ติดตั้งเว็บไซต์บนเวิร์กสเตชันของคุณ (โดยใช้ WAMPP เป็นต้น) คุณอาจเคยใช้ที่อยู่ "*localhost*" ในบางครั้งเพื่อแสดงเว็บไซต์ที่โฮสต์บนเครื่องของคุณเอง ชื่อโฮสต์นี้เชื่อมโยงกับที่อยู่ 127.0.0.1 และดังนั้นจึงเชื่อมโยงกับ Interface loopback
- ens33**: นี่คือ Interface ตัวแรกของฉัน ซึ่งได้รับที่อยู่ที่นี่จาก DHCP ของฉัน
- ens36**: Interface ที่สองของฉัน



ตอนนี้ที่ฉันมีข้อมูลทั้งหมดแล้ว ฉันสามารถแก้ไขไฟล์ */etc/network/interfaces* เพื่อสร้างบริดจ์ของฉันได้ นี่คือสิ่งที่มันดูเหมือนในปัจจุบัน (อาจแตกต่างกัน):



```
# This file describes the network interfaces available on your system
# and how to activate them. For more information, see interfaces(5).

source /etc/network/interfaces.d/*

# The loopback network interface
auto lo
iface lo inet loopback

# The primary network interface
allow-hotplug ens33
iface ens33 inet dhcp
# This is an autoconfigured IPv6 interface
iface ens33 inet6 auto
```



ส่วนแรกเกี่ยวข้องกับ loopback Interface ของฉัน ซึ่งฉันจะไม่แตะต้อง ตามด้วย Interface ens33 การแก้ไขเกี่ยวข้องกับการเพิ่ม Interface ตัวที่สองของฉัน (ens36) และกำหนดค่า bridge ด้วยสองอินเทอร์เฟซนี้:



```
# The primary network interface
auto ens33
iface ens33 inet manual

#The secondary network interface
auto ens36
iface ens36 inet manual
```



นี่คือคำอธิบายบางส่วนของการเปลี่ยนแปลงแรกเหล่านี้:





- อัตโนมัติ **Interface**: จะ "เริ่ม" Interface โดยอัตโนมัติเมื่อเริ่มระบบ
- iface *Interface* inet **manual**: เพื่อใช้ Interface โดยไม่ต้องมีที่อยู่ IP เช่นเดียวกับคีย์เวิร์ด "static" เพื่อกำหนดที่อยู่ IP แบบคงที่หรือ "dhcp" เพื่อใช้การกำหนดที่อยู่แบบไดนามิก



การปรับเปลี่ยนยังคงดำเนินต่อไป:



```
# Bridge interface
auto br0
iface br0 inet static
address 192.168.1.23
netmask 255.255.255.0
gateway 192.168.1.1
bridge_ports ens33 ens36
bridge_stp off
```



ที่นี่อีกครั้ง, คำอธิบายเล็กน้อย:





- iface br0 inet static**: ที่นี่ฉันได้กำหนดสะพาน Interface ของฉัน (*br0*) ด้วยที่อยู่แบบคงที่
- Address, netmask, gateway**: ข้อมูลการกำหนดที่อยู่ของบอร์ด
- bridge_ports**: อินเทอร์เฟซที่จะรวมอยู่ในบริดจ์
- bridge_stp**: โปรโตคอล Spanning Tree ถูกใช้เมื่อเชื่อมต่อสวิตช์เพื่อค้นหาลิงก์ที่ซ้ำซ้อนและหลีกเลี่ยงลูป เนื่องจากสะพานสามารถถูกแทรกระหว่างสองเส้นทางเครือข่ายได้ มันจึงสามารถเป็นแหล่งของลูปได้ ดังนั้นจึงมีความเป็นไปได้ที่จะเปิดใช้งานโปรโตคอลนี้ ฉันไม่ต้องการมันที่นี่ ดังนั้นฉันจะปิดการใช้งานมัน



บันทึกการเปลี่ยนแปลงและเริ่มเครือข่ายใหม่:



```
systemctl restart networking
```



เพื่อตรวจสอบการเปลี่ยนแปลง ให้แสดงผลลัพธ์ของคำสั่ง **ip** add อีกครั้ง:



![Image](assets/fr/021.webp)


คุณสามารถเห็นได้อย่างชัดเจนว่า **Interface "*br0*" ใหม่ของฉันมีที่อยู่ IP ที่ฉันได้กำหนดให้มัน** นอกจากนี้ คุณยังสามารถเห็นได้ว่าทั้งสองอินเทอร์เฟซทางกายภาพของฉันอยู่ในสถานะ "UP" แต่ไม่มีที่อยู่ IP



## IV. การติดตั้ง NtopNG



ตอนนี้ที่โพรบของเราสามารถดักจับทราฟฟิกระหว่างเครือข่ายของฉันกับเราเตอร์ได้แล้ว สิ่งที่เหลืออยู่คือการติดตั้ง ntopng ในการทำเช่นนี้ ให้แก้ไขไฟล์ */etc/apt/sources.list* และเพิ่ม **contrib** ที่ท้ายบรรทัดแต่ละบรรทัดที่ขึ้นต้นด้วย **deb** หรือ **deb-src**



ตามค่าเริ่มต้น แหล่งแพ็กเกจจะมีเฉพาะแพ็กเกจที่สอดคล้องกับ DFSG (*Debian Free Software Guidelines*) ซึ่งระบุด้วยคีย์เวิร์ด **main** คุณยังสามารถเพิ่มแหล่งเหล่านี้ได้:





- contrib**: แพ็กเกจที่มีซอฟต์แวร์ที่สอดคล้องกับ DFSG แต่ใช้การพึ่งพาที่ไม่ได้เป็นส่วนหนึ่งของสาขา **main**
- non-free**: ประกอบด้วยแพ็กเกจที่ไม่สอดคล้องกับ DFSG



ตัวอย่างของบรรทัดใน /etc/apt/sources.list:



```
deb http://deb.debian.org/debian/ bullseye main
```



ดังนั้นฉันจะเพิ่มคำว่า **contrib** ลงในบรรทัดเหล่านี้



ขั้นตอนที่เหลือจะระบุไว้ในเว็บไซต์ [NtopNG](https://packages.ntop.org/apt/) ซึ่งสำหรับ Debian 11 คุณจำเป็นต้องเพิ่มแหล่งที่มาของ Ntop สำหรับการติดตั้งในอนาคต การเพิ่มนี้จะทำโดยอัตโนมัติด้วยการใช้:



```
wget https://packages.ntop.org/apt/bullseye/all/apt-ntop.deb
apt install ./apt-ntop.deb
```



จากนั้นเข้าสู่ขั้นตอนการติดตั้งจริง:



```
apt-get clean all
apt-get update
apt-get install ntopng
```



คำสั่งแรกจะลบแคชของตัวจัดการแพ็คเกจ apt คำสั่งที่สองจะอัปเดตรายการแพ็คเกจและคำสั่งที่สามจะติดตั้ง NtopNG



หลังการติดตั้ง **ซอฟต์แวร์ NtopNG จะพร้อมใช้งานโดยตรงที่พอร์ต 3000 ของเครื่อง Debian** ดังนั้นสำหรับฉันคือ `http://192.168.1.23:3000`

## VI. การตรวจจับและการแจ้งเตือน



NtopNG ยังสามารถออกการแจ้งเตือนด้านความปลอดภัยในฟีดบางรายการได้อีกด้วย ซึ่งสามารถพบได้ในเมนู **Alerts** บนแถบด้านซ้าย ตัวอย่างเช่น ที่นี่ฉันมีการแจ้งเตือนทั้งหมด 2851 รายการ แบ่งออกเป็นระดับความรุนแรงต่างๆ: Notice, Warning และ Error.



![Image](assets/fr/019.webp)



มาดูการแจ้งเตือนที่มีความสำคัญสูงกันเถอะ ฉันมีทั้งหมด 17 รายการ!



การคลิกที่รูปนี้จะนำไปสู่รายละเอียดของการแจ้งเตือน ไม่มีอะไรที่น่าตกใจที่นี่ มันเป็นการแจ้งเตือนที่ผิดพลาด การดาวน์โหลดอัปเดตถูกจัดประเภทเป็นการโอนย้ายไฟล์ไบนารีในสตรีม HTTP ซึ่งอาจหมายถึงการโจมตีได้จริงๆ



![Image](assets/fr/020.webp)



อย่างไรก็ตาม เนื่องจากฉันใช้เวอร์ชันฟรี ฉันไม่สามารถยกเว้นโดเมนหรือโฮสต์ที่เป็นแหล่งที่มาของการแจ้งเตือนได้ ดังนั้นคุณจะต้องคอยจับตาดูพวกมันเพื่อหลีกเลี่ยงการพลาดสิ่งที่น่ากังวลมากกว่า NtopNG จะ generate การแจ้งเตือนในกรณีของ:





- การดาวน์โหลดไฟล์ไบนารีผ่าน HTTP
- การรับส่งข้อมูล DNS ที่น่าสงสัย
- การใช้พอร์ตที่ไม่เป็นมาตรฐาน
- การตรวจจับการฉีด SQL
- Cross-Site Scripting (XSS) - การโจมตีข้ามไซต์
- เป็นต้น
