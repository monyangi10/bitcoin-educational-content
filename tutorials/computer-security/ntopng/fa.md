## I. ارائه



**چه برای بررسی روانی شبکه، چه برای داشتن تصویری واضح از استفاده یا برای آمار عملکرد، نظارت بر جریان شبکه بخش مهمی از شبکه سازمانی است**. این به پیش‌بینی تغییرات در زیرساخت کمک می‌کند و به تضمین کیفیت استفاده برای کاربران کمک می‌کند (که به عنوان QoE برای *کیفیت تجربه* شناخته می‌شود، در مقابل QoS).



برای انجام این کار، تکنیک‌ها و نرم‌افزار/پروتکل‌های زیادی در دسترس هستند. به عنوان مثال، Netflow که توسط سیسکو توسعه یافته است، می‌تواند برای بازیابی آمار جریان IP از یک Interface استفاده شود، اما استفاده از آن به تجهیزات سازگار محدود است.



به همین دلیل در این آموزش قصد دارم **Ntop** را معرفی کنم و به شما نشان دهم که چگونه از آن در زیرساخت خود برای نظارت بر استفاده از شبکه استفاده کنید.



Ntop نرم‌افزار متن‌بازی است که می‌تواند بر روی هر ماشین لینوکسی نصب شود. این نرم‌افزار رایگان است و می‌تواند داده‌های زیر را جمع‌آوری کند:





- استفاده از پهنای باند
- مشتریان اصلی
- مقاصد اصلی
- پروتکل‌های استفاده شده
- برنامه‌های کاربردی استفاده شده
- پورت‌های استفاده شده
- و غیره.



اکنون با نام جدید **Ntopng (نسل جدید)**، بسیاری از عملکردهای پایه را به‌صورت رایگان ارائه می‌دهد. یک نسخه تجاری نیز موجود است که امکان صادرات هشدارهای پیکربندی‌شده به نرم‌افزارهای نوع SIEM یا فیلتر کردن ترافیک با قوانین تعریف‌شده مستقیماً از پروب را فراهم می‌کند.



## II. پیش‌نیازها



نصب یک پروب Ntop بسته به تجهیزات و محیط متفاوت است. بنابراین، من در اینجا یک راهنمای گام‌به‌گام ارائه نمی‌دهم، بلکه به بررسی امکانات مختلف می‌پردازم.



### A. حالت درون‌بُرد



اگر یک فایروال pfSense، OPNSense یا Endian در حال استفاده دارید، یا حتی یک ورک‌استیشن لینوکس با NFTables، خبر خوب! می‌توانید Ntopng را مستقیماً نصب کرده و شروع به مانیتورینگ رابط‌های خود کنید.



مزیت این تکنیک این است که به هیچ سخت‌افزار اضافی نیاز ندارد. از سوی دیگر، استفاده از منابع را افزایش می‌دهد، بنابراین مطمئن شوید که سخت‌افزار کافی یا یک ماشین مجازی با اندازه مناسب (حداقل ۲ هسته و ۲ گیگابایت رم) دارید.



### B. حالت TAP / SPAN



یک **tap** یک **دستگاه شبکه است که ترافیک عبوری از آن را تکثیر کرده و به دستگاه دیگری ارسال می‌کند.** مزیت این دستگاه این است که نیازی به هیچ گونه تغییری در زیرساخت موجود ندارد و می‌تواند در هر جایی برای نظارت بر بخش‌های خاصی از شبکه قرار گیرد، یا بین سوئیچ مرکزی و روتر لبه برای تحلیل ترافیک به/از اینترنت قرار داده شود.



بزرگترین عیب این نوع تجهیزات هزینه آن است. در شبکه‌های گیگابیتی امروزی، یک تپ غیرفعال نمی‌تواند به درستی ترافیک شبکه را ضبط کند، بنابراین شما به یک دستگاه فعال نیاز دارید که حداقل حدود ۲۰۰ یورو هزینه دارد.



حالت **SPAN**، که به عنوان **port mirroring** نیز شناخته می‌شود، توسط یک سوئیچ برای ارسال ترافیک از یک پورت به پورت دیگر استفاده می‌شود. این روش تا کنون روش مورد علاقه من است، زیرا راه‌اندازی آن ساده است و مانند tap، به شما اجازه می‌دهد بخشی از شبکه یا کل شبکه را به دلخواه نظارت کنید. با این حال، دو اشکال وجود دارد: سوئیچ باید این عملکرد را یکپارچه کند و استفاده از آن می‌تواند بار پردازنده روی سوئیچ را افزایش دهد.



### C. حالت روتر



کاملاً ممکن است که یک روتر را تحت لینوکس نصب کرده و ntopng را روی آن نصب کنید. تنها نقطه ضعف این روش این است که شبکه شما را تغییر می‌دهد، یا آدرس‌دهی داخلی آن، یا آدرس‌دهی بین روتر "واقعی" شما و روتری که اضافه می‌کنید.



توجه: برای خوانندگان مقاله [ایجاد یک روتر وای‌فای با Raspberry Pi و RaspAP](https://www.it-connect.fr/creer-un-routeur-wifi-avec-un-raspberry-pi-et-raspap/) نصب ntopng بر روی Rpi شما برای دریافت آمار دقیق کاملاً ممکن است!



### D. حالت Bridge



یک جایگزین جالب استفاده از **یک ماشین لینوکس در حالت پل** است. قرار دادن آن بین دو قطعه تجهیزات، این امکان را فراهم می‌کند که تمام ترافیک بدون نیاز به دخالت در پیکربندی زیرساخت یا تجهیزات آن، ضبط شود. از آنجا که یک ماشین قدیمی می‌تواند این کار را انجام دهد، هزینه این روش نیز می‌تواند جذاب باشد. توجه داشته باشید که برای بهینه بودن، دستگاه باید سه کارت شبکه داشته باشد، دو کارت در حالت پل و یکی برای دسترسی به Interface و SSH. امکان استفاده از فقط دو کارت وجود دارد، اما در این صورت ترافیک مدیریت دستگاه نیز ضبط خواهد شد...



بنابراین **این حالت آخر است که من قصد دارم از آن استفاده کنم**. به دلایل عملی، من از ماشین‌های مجازی برای نمایش استفاده خواهم کرد، اما روش برای استفاده در ماشین‌های فیزیکی نیز یکسان است.



## III. آماده‌سازی پروب با پل Interface



برای پروب، من یک ماشین **Debian 11** با نصب حداقلی انتخاب می‌کنم.



اولین قدم، همیشه یکسان، به‌روزرسانی:



```
apt-get update && apt-get upgrade
```



سپس بسته **bridge-utils** را نصب کنید، که به ما امکان می‌دهد پل خود را ایجاد کنیم:



```
apt-get install bridge-utils
```



پس از نصب، اولین چیزی که باید به آن توجه کرد نام فعلی کارت‌های شبکه ما است. تحت دبیان، این نام می‌تواند اشکال مختلفی داشته باشد و ما برای پیکربندی به آن نیاز خواهیم داشت.



یک دستور ساده **ip add** خروجی با این اطلاعات را برمی‌گرداند:



![Image](assets/fr/016.webp)



اینجا، من 3 رابط کاربری می‌بینم:





- لو**: این Interface لوپ‌بک است؛ یک Interface مجازی که بر روی تجهیزات "لوپ" می‌کند. اساساً، این Interface که Address آن 127.0.0.1 است (اگرچه هر Address در 127.0.0.0/8 برای این منظور رزرو شده است) برای تماس با خود تجهیزات استفاده می‌شود. اگر یک وب‌سایت روی ورک‌استیشن خود نصب کرده‌اید (برای مثال با استفاده از WAMPP)، احتمالاً در یک زمان یا زمان دیگر از Address "*localhost*" برای نمایش سایت میزبانی شده روی ماشین خودتان استفاده کرده‌اید. این نام میزبان با Address 127.0.0.1 و بنابراین با لوپ‌بک Interface مرتبط است.
- ens33**: این اولین Interface من است که یک Address از DHCP من در اینجا دریافت کرده است.
- ens36**: دومین Interface من



اکنون که تمام اطلاعات را دارم، می‌توانم فایل */etc/network/interfaces* را برای ایجاد بریج خود ویرایش کنم. در حال حاضر به این شکل است (ممکن است متفاوت باشد):



```
# This file describes the network interfaces available on your system
# and how to activate them. For more information, see interfaces(5).

source /etc/network/interfaces.d/*

# The loopback network interface
auto lo
iface lo inet loopback

# The primary network interface
allow-hotplug ens33
iface ens33 inet dhcp
# This is an autoconfigured IPv6 interface
iface ens33 inet6 auto
```



قسمت اول مربوط به loopback Interface من است که به آن دست نخواهم زد، و به دنبال آن Interface ens33 قرار دارد. تغییرات شامل اضافه کردن Interface دوم من (ens36) و پیکربندی پل با این دو رابط است:



```
# The primary network interface
auto ens33
iface ens33 inet manual

#The secondary network interface
auto ens36
iface ens36 inet manual
```



در اینجا برخی توضیحات درباره این تغییرات اولیه آمده است:





- خودکار *Interface***: به‌طور خودکار "Interface" را در هنگام راه‌اندازی سیستم "شروع" خواهد کرد.
- iface *Interface* inet manual**: برای استفاده از Interface بدون هیچ IP Address. مانند کلمه کلیدی "static" برای تعریف یک IP ثابت Address یا "dhcp" برای استفاده از آدرس‌دهی پویا



اصلاحات ادامه دارد:



```
# Bridge interface
auto br0
iface br0 inet static
address 192.168.1.23
netmask 255.255.255.0
gateway 192.168.1.1
bridge_ports ens33 ens36
bridge_stp off
```



در اینجا دوباره، چند توضیح:





- iface br0 inet static**: در اینجا پل Interface خود (*br0*) را با یک Address ثابت تعریف کرده‌ام.
- Address، نت‌ماسک، گیت‌وی**: اطلاعات آدرس‌دهی برد
- bridge_ports**: رابط‌هایی که باید در پل گنجانده شوند
- bridge_stp**: پروتکل Spanning Tree زمانی که سوئیچ‌ها به هم متصل می‌شوند برای شناسایی لینک‌های اضافی و جلوگیری از حلقه‌ها استفاده می‌شود. از آنجا که یک پل می‌تواند بین دو مسیر شبکه قرار گیرد، می‌تواند منبع یک حلقه باشد، بنابراین امکان فعال‌سازی این پروتکل وجود دارد. من به آن در اینجا نیازی ندارم، بنابراین آن را غیرفعال می‌کنم.



تغییرات را ذخیره کرده و شبکه را مجدداً راه‌اندازی کنید:



```
systemctl restart networking
```



برای بررسی تغییرات، نتیجه دستور **ip** add را دوباره نمایش دهید:



![Image](assets/fr/021.webp)


شما می‌توانید به وضوح **Interface "*br0*" جدید من را با IP Address که به آن اختصاص داده‌ام، ببینید.** به طور اتفاقی، شما همچنین می‌توانید ببینید که دو رابط فیزیکی من واقعاً "UP" هستند، اما هیچ IP Address ندارند.



## IV. نصب NtopNG



اکنون که پروب ما قادر به شنود ترافیک بین شبکه من و روتر است، تنها کاری که باقی مانده نصب ntopng است. برای انجام این کار، ابتدا فایل */etc/apt/sources.list* را ویرایش کرده و **contrib** را به انتهای هر خطی که با **deb** یا **deb-src** شروع می‌شود، اضافه کنید.



به طور پیش‌فرض، منابع بسته فقط شامل بسته‌های سازگار با DFSG (*راهنمای نرم‌افزار آزاد دبیان*) هستند که با کلمه کلیدی **main** شناسایی می‌شوند. شما همچنین می‌توانید این منابع را اضافه کنید:





- contrib**: بسته‌هایی که حاوی نرم‌افزارهای سازگار با DFSG هستند، اما از وابستگی‌هایی استفاده می‌کنند که بخشی از شاخه **main** نیستند.
- غیر آزاد**: شامل بسته‌هایی است که با DFSG سازگار نیستند



مثالی از یک خط در /etc/apt/sources.list:



```
deb http://deb.debian.org/debian/ bullseye main
```



بنابراین من فقط کلمه **contrib** را به خطوطی مانند این‌ها اضافه خواهم کرد.



بقیه مراحل در سایت [NtopNG] (https://packages.ntop.org/apt/) فهرست شده‌اند، جایی که برای Debian 11، نیاز دارید منابع Ntop را برای نصب‌های آینده اضافه کنید. این افزودن با استفاده از یک:



```
wget https://packages.ntop.org/apt/bullseye/all/apt-ntop.deb
apt install ./apt-ntop.deb
```



سپس مرحله نصب واقعی فرا می‌رسد:



```
apt-get clean all
apt-get update
apt-get install ntopng
```



اولین دستور حافظه پنهان مدیر بسته apt را حذف می‌کند. دستور دوم فهرست بسته‌ها را به‌روزرسانی می‌کند و دستور سوم NtopNG را نصب می‌کند.



پس از نصب، **نرم‌افزار NtopNG به‌طور مستقیم بر روی پورت 3000 ماشین دبیان در دسترس است**. بنابراین برای من این آدرس است: `http://192.168.1.23:3000`



![Image](assets/fr/022.webp)



صفحه اصلی NtopNG



ورود و رمز عبور پیش‌فرض نمایش داده می‌شوند، بنابراین تنها کاری که باید انجام دهید این است که آن‌ها را وارد کنید!



## V. استفاده از NtopNG



وقتی برای اولین بار وارد سیستم می‌شوید، اولین کاری که از شما خواسته می‌شود انجام دهید، تغییر رمز عبور پیش‌فرض مدیر و زبان است. متأسفانه، فرانسوی یکی از آن‌ها نیست.



سپس به داشبورد می‌رسید:



![Image](assets/fr/023.webp)



داشبورد NtopNG



به این یکی عادت نکنید! اگر به کادر زرد در بالای صفحه توجه کنید، جمله زیر را خواهید دید: "*مجوز در 04:57 منقضی می‌شود*". به‌طور پیش‌فرض، نصب یک نسخه آزمایشی از نسخه کامل NtopNG را راه‌اندازی می‌کند، اما برای مدت زمان (بسیار) محدودی. هنگامی که این "شمارش معکوس" به پایان رسید، نسخه *جامعه* فعال می‌شود و داشبورد تغییر می‌کند:



![Image](assets/fr/024.webp)



داشبورد جدید جامعه NtopNG



اولین کاری که باید انجام دهید این است که **بررسی کنید که Interface صحیح در حال گوش دادن است**. در گوشه بالا سمت چپ، یک لیست کشویی از رابط‌های موجود به شما اجازه می‌دهد تا Interface مورد نظر ما را انتخاب کنید: br0



![Image](assets/fr/025.webp)



انتخاب Interface



پنجره جدید "Top Flaw Talkers" را نمایش می‌دهد، یعنی دستگاه‌هایی که بیشترین ارتباط را دارند. اینجا فقط دو ایستگاه ظاهر می‌شوند:



![Image](assets/fr/026.webp)



**میزبان‌های منبع در سمت چپ ظاهر می‌شوند، مقصدها در سمت راست** این به شما امکان می‌دهد استفاده از پهنای باند کل توسط هر میزبان را مشاهده کنید و دید کلی از ترافیک شبکه داشته باشید. این بد نیست، اما می‌توانیم فراتر برویم...



اگر روی "*Hosts*" کلیک کنم، به عنوان مثال، نموداری را مشاهده می‌کنم که مصرف برق ارسال و دریافت هر میزبان در شبکه من را نشان می‌دهد. اینجا، به عنوان مثال، می‌توانم ببینم که 192.168.1.37 به تنهایی 80٪ از ترافیک من را تشکیل می‌دهد:



![Image](assets/fr/027.webp)



اگر روی IP Address میزبان مورد نظر کلیک کنم، به صفحه خلاصه هدایت می‌شوم:



![Image](assets/fr/028.webp)



می‌توانم ببینم، به عنوان مثال، که این یک ماشین VMWare است (با تشخیص YES از MAC Address من)، که به نام DESKTOP-GHEBGV1 است (پس مطمئناً یک میزبان ویندوزی است) و **آمار بسته‌های دریافت و ارسال شده، و همچنین مقدار داده‌ها** را دارم.



شما همچنین یک منوی جدید در بالای این خلاصه مشاهده خواهید کرد. پیشنهاد می‌کنم روی "**برنامه‌ها**" کلیک کنید تا ببینید چه چیزی باعث این همه ترافیک شده است:



![Image](assets/fr/017.webp)


ها، به نظر می‌رسد که پاسخی داریم! در نمودار سمت چپ، **می‌بینیم که ۷۶.۶٪ از ترافیک آن از ... Windows Update است**، بنابراین این میزبان در حال دانلود به‌روزرسانی‌ها است!



NtopNG از فناوری‌ای به نام DPI برای *بازرسی عمیق بسته‌ها* استفاده می‌کند که به آن امکان می‌دهد هر جریان شبکه را دسته‌بندی کرده و برنامه (یا خانواده برنامه‌ها) پشت آن را تعریف کند.



برای نمایش، یک ویدیو یوتیوب را روی میزبانم اجرا می‌کنم:

**ترافیک بلافاصله شناسایی و دسته‌بندی شد!



توجه: به دلایل واضح، این نوع نرم‌افزار می‌تواند مسائل حریم خصوصی را به وجود آورد، بنابراین مراقب باشید که آن را تحت شرایط مناسب استفاده کنید. همچنین توجه داشته باشید که امکان **ناشناس‌سازی نتایج** وجود دارد، این گزینه را می‌توانید در **تنظیمات > ترجیحات > متفرقه** پیدا کنید و به نام "**Mask Host IP Address**" است.



## VI. شناسایی‌ها و هشدارها



NtopNG همچنین قادر به صدور هشدارهای امنیتی در برخی از فیدها است. این هشدارها را می‌توان در منوی **هشدارها** در نوار سمت چپ یافت. به عنوان مثال، من در اینجا مجموعاً ۲۸۵۱ هشدار دارم که به شدت‌های مختلف تقسیم شده‌اند: اطلاعیه، هشدار و خطا.



![Image](assets/fr/019.webp)



بیایید نگاهی به هشدارهای با اهمیت بالا بیندازیم، من 17 تا از آن‌ها را دارم!



کلیک کردن روی این شکل جزئیات هشدارها را نمایش می‌دهد. اینجا چیزی نگران‌کننده نیست، این یک مثبت کاذب است، دانلود به‌روزرسانی‌ها به‌عنوان انتقال فایل باینری در یک جریان HTTP دسته‌بندی شده است که می‌تواند به‌معنای یک حمله باشد.



![Image](assets/fr/020.webp)



با این حال، از آنجا که من از نسخه رایگان استفاده می‌کنم، نمی‌توانم دامنه‌ها یا میزبان‌هایی که منبع هشدارها هستند را حذف کنم، بنابراین باید خودتان مراقب آن‌ها باشید تا چیزی بسیار نگران‌کننده‌تر را از دست ندهید. NtopNG هشدارهای generate را در صورت وقوع:





- دانلود فایل باینری از طریق HTTP
- ترافیک DNS مشکوک
- استفاده از پورت غیر استاندارد
- تشخیص تزریق SQL
- کراس سایت اسکریپتینگ (XSS)
- و غیره.



## VII. نتیجه‌گیری



در این آموزش، ما **نحوه راه‌اندازی یک پروب با NtopNG** را مشاهده کرده‌ایم که به ما امکان می‌دهد **ترافیک شبکه خود را تحلیل کنیم** تا استفاده از پروتکل‌ها و اشغال هر میزبان را مشاهده کنیم، و همچنین ترافیک مشکوک را شناسایی کنیم.



متأسفانه، نمی‌توانم تمام ویژگی‌ها و امکانات را در این مقاله پوشش دهم، اما با خیال راحت به کاوش بپردازید!



این راه‌حل می‌تواند به صورت دائمی در زیرساخت سازمانی پیاده‌سازی شود. NtopNG همچنین می‌تواند نتایج را به یک پایگاه داده InfluxDB صادر کند و به شما امکان می‌دهد داشبوردهای خود را با استفاده از ابزارهای شخص ثالث مانند Graphana ایجاد کنید.
