## I.プレゼンテーション



**ネットワークの流動性をチェックするためであれ、使用状況を明確に把握するためであれ、パフォーマンス統計のためであれ、ネットワーク・フロー・モニタリングは企業ネットワーク**の重要な部分です。インフラストラクチャの変更を予測し、ユーザーの使用品質を確保するのに役立ちます（QoSとは対照的に、*Quality of Experience*の略でQoEとも呼ばれます）。



そのために、多くの技術やソフトウェア/プロトコルが利用可能である。例えば、Cisco社が開発したNetflowは、InterfaceからIPフロー統計を取得するために使用できるが、その使用は互換性のある機器に限定される。



そこでこのチュートリアルでは、**Ntop**を紹介し、ネットワーク使用状況を監視するためのインフラでの使用方法を紹介する。



Ntopは、Linuxマシンにインストールできるオープンソースソフトウェアである。無料で、以下のデータを収集することができる：





- 帯域幅利用率
- 主要顧客
- 主な目的地
- 使用プロトコル
- 使用アプリケーション
- 使用ポート
- などなど。



現在は**Ntopng（New Generation）**と改名され、多くの基本機能を無料で提供している。また、設定されたアラートをSIEMタイプのソフトウェアにエクスポートしたり、プローブから直接定義されたルールでトラフィックをフィルタリングしたりできる商用バージョンもあります。



## II.前提条件



Ntopプローブの取り付けは、装置や環境によって異なる。そこで、ここではステップ・バイ・ステップのガイドをするつもりはないが、様々な可能性について概説する。



### A.オンボード・モード



pfSense、OPNSense、Endianファイアウォール、あるいはLinuxワークステーションにNFTablesを導入している方に朗報です！Ntopngを直接インストールし、インターフェイスの監視を開始することができます。



このテクニックの利点は、追加のハードウェアを必要としないことだ。その反面、リソースの使用量が増えるので、十分なハードウェアか、十分なサイズのVM（最低2コア、2BG RAM）を用意してください。



### B.TAP / SPANモード



タップ**とは、通過するトラフィックを複製して別のデバイスに送る**ネットワーク・デバイスのことである。このデバイスの利点は、既存のインフラに変更を加える必要がなく、特定のネットワーク・セクションを監視するためにどこにでも設置できること、またはインターネットへのトラフィック/インターネットからのトラフィックを分析するためにコア・スイッチとエッジ・ルーターの間に設置できることである。



このタイプの機器の大きな欠点はコストだ。今日のギガビット・ネットワークでは、パッシブ・タップではネットワーク・トラフィックを適切に捕捉できないため、約200ユーロ（最低）のアクティブ・デバイスが必要となる。



SPAN**モードは、**ポートミラーリング**としても知られ、あるポートから別のポートにトラフィックを転送するためにスイッチによって使用される。セットアップが簡単で、タップと同様、ネットワークの一部または全体を自由に監視することができるため、この方法が圧倒的に好ましい。しかし、2つの欠点がある。スイッチがこの機能を統合しなければならないことと、この機能を使うことでスイッチのプロセッサ負荷が増大する可能性があることだ。



### C.ルーターモード



Linuxでルーターをマウントし、そこにntopngをインストールすることは完全に可能だ。この方法の唯一の欠点は、ネットワークが変更されることで、内部アドレスか、"本当の "ルーターと追加するルーター間のアドレスが変更されることだ。



注：[Raspberry PiとRaspAPでWifiルーターを作る](https://www.it-connect.fr/creer-un-routeur-wifi-avec-un-raspberry-pi-et-raspap/)の記事を読んでくれた読者のために、正確な統計を取るためにRpiにntopngをインストールすることは完全に可能である！



### D.ブリッジモード



2つの機器の間に置くことで、インフラや機器の設定に介入することなく、すべてのトラフィックを捕捉することができる。古いマシンでもできるので、この方法のコストは魅力的である。最適にするには、3枚のネットワークカードが必要で、2枚はブリッジモードで、1枚はInterfaceとSSHにアクセスする。2枚だけでも可能だが、その場合、デバイスの管理トラフィックもキャプチャされてしまう...。



だから、私が使うのは**この最後のモードだ**。実用的な理由から、デモには仮想マシンを使いますが、物理マシンで使う方法は変わりません。



## III.Interfaceブリッジを用いたプローブの準備



プローブには、最小限のインストールで**Debian 11**マシンを選んだ。



最初のステップはいつも同じで、.NET Frameworkをアップデートすることだ：



```
apt-get update && apt-get upgrade
```



次に、ブリッジを作成するための**bridge-utils**パッケージをインストールする：



```
apt-get install bridge-utils
```



インストールが完了したら、まず注意しなければならないのは、現在のネットワークカードの名前だ。Debianでは、この名前はいくつかの形をとることができ、設定にはこの名前が必要になる。



単純な**ip add**コマンドは、この情報を出力する：



![Image](assets/fr/016.webp)



ここには3つのインターフェイスがある：





- Lo**：これはループバックInterfaceであり、機器を「ループ」する仮想Interfaceである。基本的に、このInterfaceのAddressは127.0.0.1（ただし、127.0.0.0/8のどのAddressでも構いません。例えばWAMPPを使って）ワークステーションにウェブサイトをインストールしたことがあれば、おそらく一度は "*localhost*"を使ったことがあるでしょう。Addressを使ったことがあるでしょう。このホスト名はAddressの127.0.0.1、つまりInterfaceのループバックに関連付けられています。
- ens33**: これは私の最初のInterfaceで、DHCPからAddressを受け取った。
- ens36**：2度目のInterface



これですべての情報が揃ったので、*/etc/network/interfaces*ファイルを変更してブリッジを作成することができる。現在はこんな感じだ（異なるかもしれない）：



```
# This file describes the network interfaces available on your system
# and how to activate them. For more information, see interfaces(5).

source /etc/network/interfaces.d/*

# The loopback network interface
auto lo
iface lo inet loopback

# The primary network interface
allow-hotplug ens33
iface ens33 inet dhcp
# This is an autoconfigured IPv6 interface
iface ens33 inet6 auto
```



最初の部分はループバックInterfaceに関するもので、これは触らない。2つ目のInterface（ens36）を追加し、これら2つのインターフェイスでブリッジを設定します：



```
# The primary network interface
auto ens33
iface ens33 inet manual

#The secondary network interface
auto ens36
iface ens36 inet manual
```



以下、最初の変更点について説明する：





- auto *Interface***: システム起動時にInterfaceを自動的に「起動」します。
- iface *Interface* inet manual** : Interface を IP Address なしで使用する。キーワード "static "は静的IP Addressを定義し、"dhcp "は動的アドレッシングを使用します。



修正は続く：



```
# Bridge interface
auto br0
iface br0 inet static
address 192.168.1.23
netmask 255.255.255.0
gateway 192.168.1.1
bridge_ports ens33 ens36
bridge_stp off
```



ここでもう一度、いくつかの説明をしよう：





- iface br0 inet static**: ここではInterfaceブリッジ(*br0*)をスタティックAddressで定義している。
- Address、ネットマスク、ゲートウェイ**：ボードのアドレス情報
- bridge_ports**: ブリッジに含めるインターフェース
- bridge_stp**: スパニング・ツリー・プロトコルは、冗長リンクを検出してループを回避するために、スイッチの相互接続時に使用されます。ブリッジは2つのネットワーク・パスの間に挿入することができるので、ループの原因になる可能性があり、そのためこのプロトコルを有効にする可能性がある。ここでは必要ないので無効にしている。



変更を保存し、ネットワークを再起動します：



```
systemctl restart networking
```



変更を確認するには、**ip** addコマンドの結果をもう一度表示する：



![Image](assets/fr/021.webp)


私の新しいInterface"*br0*"と、それに割り当てたIP Addressがはっきり見えるだろう。**ちなみに、私の2つの物理インターフェースは確かに "UP "しているが、IP Addressを持っていないこともわかるだろう。



## IV.NtopNGのインストール



プローブが私のネットワークとルーター間のトラフィックをスニッフできるようになったので、あとはntopngをインストールするだけだ。これを行うには、まず*/etc/apt/sources.list*ファイルを修正し、**deb**または**deb-src**で始まる各行の末尾に**contrib**を追加する。



デフォルトでは、パッケージソースには DFSG (*Debian Free Sotftware Guidelines*) に準拠したパッケージのみが含まれており、**main** キーワードで識別されます。これらのソースを追加することもできます：





- contrib**：DFSGに準拠したソフトウェアを含むが、**main**ブランチの一部ではない依存関係を使用するパッケージ
- non-free**: DFSGに準拠していないパッケージが含まれています。



etc/apt/sources.list の行の例：



```
deb http://deb.debian.org/debian/ bullseye main
```



だから、このような行には**contrib**という言葉を付け加えることにしている。



残りの手順は[NtopNG]サイト(https://packages.ntop.org/apt/)に記載されています。Debian 11の場合、将来のインストールのためにNtopソースを追加する必要があります。この追加は、.Ntop.exe を使用して自動化されています：



```
wget https://packages.ntop.org/apt/bullseye/all/apt-ntop.deb
apt install ./apt-ntop.deb
```



そして、実際の設置段階に入る：



```
apt-get clean all
apt-get update
apt-get install ntopng
```



最初のコマンドはaptパッケージマネージャーのキャッシュを削除する。2番目はパッケージリストを更新し、3番目はNtopNGをインストールする。



インストール後、**NtopNGソフトウェアはDebianマシンのポート3000**で直接利用できる。つまり、私にとっては `http://192.168.1.23:3000` です。



![Image](assets/fr/022.webp)



NtopNGホームページ



デフォルトのログイン名とパスワードが表示されるので、それを入力するだけだ！

![Image](assets/fr/020.webp)



ただし、私は無料版を使っているので、アラートの発信源であるドメインやホストを除外することはできない。





- HTTPによるバイナリファイルのダウンロード
- 不審なDNSトラフィック

- クロスサイト・スクリプティング (XSS)
- などなど。



## VII.結論



このチュートリアルでは、**NtopNG**でプローブをセットアップする方法**について見てきました。これにより、**ネットワーク・トラフィックを分析**し、プロトコルの使用状況や各ホストの占有率を可視化するだけでなく、不審なトラフィックを検出することができます。



残念ながら、この記事ですべての機能や可能性をカバーすることはできないが、自由に探求してほしい！



このソリューションは、企業インフラ内に恒久的に導入することができます。NtopNGはInfluxDBデータベースに結果をエクスポートすることもでき、Graphanaのようなサードパーティツールで独自のダッシュボードを作成することができます。
