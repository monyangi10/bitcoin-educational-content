## Início de sessão = correio eletrónico = perda de privacidade


No mundo digital, tornou-se prática corrente ter uma conta para cada plataforma a que se pretende aceder.

Cada um destes serviços requer um início de sessão, normalmente associado ao par _nome de utilizador_ e _palavra-passe_. Muitas vezes, o nome de utilizador é o e-mail pessoal do utilizador.


Quando se utiliza o Address do correio eletrónico pessoal para cada início de sessão, é fácil imaginar a primeira consequência: a perda de confidencialidade, que se torna mais grave se o Address for composto por _name.surname@serviceemail.com_.


Os criadores de ferramentas de código aberto criaram uma série de conjuntos de aplicações, nascidos precisamente para que os utilizadores recuperem um pouco de privacidade: continuarão a iniciar sessão, mas utilizando um pseudónimo em vez da ferramenta que revela a sua identidade privada.


O mais simples, entre os que testei pessoalmente e que ainda estou testando, é justamente o [Simple Login](https://simplelogin.io/).


## Apelido


Um alias de correio eletrónico substitui simplesmente a parte _nome.apelido_ do seu Address de correio eletrónico por um nome fictício. Para o utilizador, nada muda; o serviço de alias encaminha as mensagens de correio eletrónico de e para o Address habitual, como habitualmente. Todos podem continuar a usar a sua caixa de entrada como sempre fazem, mas em vez de mostrar o seu nome verdadeiro, mostrará um utilizador irreconhecível. Este serviço precisa de ser eficiente e, até agora, o Simple Login provou ser exatamente isso.


Ao visitar o sítio Simple Login pela primeira vez, deve criar uma conta (também aqui!), utilizando o correio eletrónico "oficial" Address. Aqui, é necessário introduzir o e-mail, uma palavra-passe e resolver um captcha.

## Criação de pseudónimos


No painel, a primeira opção do menu tem a designação _Alias_, que é onde os pode criar. Tem a opção de criar aliases aleatoriamente, clicando no botão Random Alias (Alias aleatório), que é o botão Green apresentado na fotografia seguinte. Esta funcionalidade cria um e-mail único e intrigante Address.


![image](assets/it/24.webp)


Se, no entanto, pretender diferenciar os serviços dando-lhes nomes diferentes, deve escolher _Novo pseudónimo personalizado_. Ao fazê-lo, pode atribuir ao pseudónimo o nome do serviço a que pretende aceder (redes sociais, fornecedores de serviços, eventos em linha, desconhecidos encontrados por acaso, etc.). O resto é tratado pelo Início de sessão simples.

Por brincadeira (mas nem por isso, para dizer a verdade), decidi criar um pseudónimo para o banco e chamei-lhe `BANCO`. Embora seja verdade que o meu banco sabe tudo sobre mim, acho divertido comunicar com eles usando um e-mail Address que é incompreensível para eles. O Simple Login gera de facto um nome aleatório, que é separado daquele que escolhemos por um `.`


Aqui, o novo correio eletrónico Address pode tornar-se:


- bank.breeding123@aleeas.com
- bank.platter456@slmails.com
- bank.preoccupy789@8shield.net
- e assim por diante


É possível escolher mais do que um domínio: os públicos estão disponíveis com o plano gratuito, enquanto outros, indicados como privados (incluindo _@simplelogin.com_), alargam a escolha para os utilizadores que decidam subscrever um plano pago.


![image](assets/it/15.webp)


Uma vez escolhidos o sufixo e o domínio aleatórios, pode definir se este novo (e bizarro) Address deve servir de alias para apenas uma das caixas de correio eletrónico pessoais ou para todas elas. O alias fica pronto e ativo depois de clicar em _Criar_


![image](assets/it/16.webp)


O novo correio eletrónico Address foi criado e está agora visível, pronto a ser enviado (para o banco!), bastando copiá-lo.


![image](assets/it/18.webp)


Nesta altura, pode concentrar-se na criação de um pseudónimo para cada serviço ou plataforma a que pretenda aceder e onde o correio eletrónico seja necessário como parâmetro essencial para criar uma conta.

## Gestão de pseudónimos


A criação de aliases é simples, tal como a sua gestão, que requer apenas um pouco de cuidado e disciplina. Todo o tráfego, de facto, continuará a passar pela caixa de correio eletrónico que definimos, no início, como a oficial. As notificações e as comunicações importantes das plataformas continuarão a chegar ao Gmail, ao Proton ou a qualquer outro fornecedor de correio eletrónico.


O resultado, no entanto, é que preservámos o principal Address que, a partir do momento da criação dos pseudónimos, somos livres de decidir a quem o revelar e a quem não o revelar.


Um pseudónimo funciona tanto para receber como para enviar: outro utilizador receberá efetivamente a resposta de alias.preoccupy789@8shield.net, se este for o pseudónimo escolhido para esse destinatário específico.


## Prós


Em geral, a utilização de pseudónimos é uma forma eficaz de proteger a sua identidade e privacidade. Os endereços de correio eletrónico são frequentemente recolhidos por corretores de dados e sítios Web para seguir os hábitos e comportamentos dos utilizadores. Embora um pseudónimo não o torne completamente indetetável, a utilização consistente de um é um passo positivo para proteger as suas informações. Além disso, na nossa "aldeia digital global", onde a pirataria informática, a venda de dados e as violações de segurança são demasiado comuns, é muito provável que o e-mail que utiliza para se registar em vários Web sites já tenha sido comprometido ou visado.


Um pseudónimo único, utilizado para cada início de sessão, **permite-nos imediatamente perceber que plataforma envia spam (ou pior) para a nossa caixa de correio, porque o e-mail é identificado pelo pseudónimo associado**. Não se faz ideia da quantidade de spam e phishing proveniente de canais ditos fiáveis, porque são institucionais, até se começar a utilizar um pseudónimo para os bancos, um para os serviços postais ou um específico para alguns serviços governamentais obrigatórios. Assim que o remetente do spam (ou pior) for identificado, saberá que esse site foi comprometido, tomando todas as precauções para proteger todos os dados fornecidos (pense nos cartões de crédito!) a esse site específico, que pode aperceber-se da violação semanas mais tarde.


Relativamente ao Simple Login, esta ferramenta tem as seguintes caraterísticas



- aplicação móvel (também da F-Droid) e extensão do navegador, para gerir pseudónimos em qualquer situação;
- autenticação de dois factores para cada novo pseudónimo, o que aumenta o grau de independência em relação ao próprio serviço;
- Suporte PGP (para utilizadores _Premium);
- criação simples de todos os tipos de alias (personalizado, aleatório e UUID);
- entre os planos gratuitos do sector, a possibilidade de utilizar pseudónimos com mais caixas de correio eletrónico "oficiais". Outros concorrentes limitam a apenas uma.


## Contras


- 10 pseudónimos podem não ser suficientes se planear utilizar o início de sessão simples de forma extensiva. Neste caso, o plano pago, que é bastante acessível, é útil para aumentar o número de aliases possíveis disponíveis.
- Não é possível criar um pseudónimo com um nome e um domínio específicos. O sufixo aleatório, adicionado depois de um nome escolhido por nós, gera um Address que pode ser descrito como bizarro, na melhor das hipóteses. As redes sociais tradicionais geralmente se recusam a conceder contas criadas com esse tipo de endereço de e-mail. O Nostr resolve isso!
- Se utilizar um pseudónimo para enviar uma mensagem a alguém, é fácil que esta acabe na pasta de spam do destinatário. Como primeiro passo, é aconselhável utilizar o pseudónimo para receber, tal como no caso da criação de uma conta, da subscrição de uma lista de correio, etc.
