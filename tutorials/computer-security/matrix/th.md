## เมทริกซ์คืออะไร?



Matrix เป็น**โปรโตคอลการสื่อสารแบบกระจายศูนย์**ที่ออกแบบมาเพื่อให้สามารถแลกเปลี่ยนข้อความ ไฟล์ และการโทรด้วยเสียง/วิดีโอระหว่างผู้ใช้และแอปพลิเคชัน โดยไม่ต้องพึ่งพาบริษัทกลาง แตกต่างจากแพลตฟอร์มการส่งข้อความแบบดั้งเดิม Matrix เป็น**โครงสร้างพื้นฐานแบบเปิด** เปรียบเสมือนอีเมล: ใครๆ ก็สามารถเลือกเซิร์ฟเวอร์หรือดำเนินการเซิร์ฟเวอร์เองได้ ในขณะที่ยังคงความสามารถในการแลกเปลี่ยนกับเครือข่ายที่เหลือ



Matrix โดดเด่นด้วยหลักการพื้นฐานสามประการ:



### โปรโตคอล ไม่ใช่แอปพลิเคชัน



Matrix ไม่ใช่แอปพลิเคชันเหมือน WhatsApp หรือ Telegram


มันเป็นภาษามาตรฐานที่แอปพลิเคชันหลายตัวสามารถใช้ได้


กล่าวอีกนัยหนึ่ง ซอฟต์แวร์ Element หลากหลายชนิด, FluffyChat, Cinny, Nheko และอื่น ๆ ให้การเข้าถึงเครือข่าย Matrix เดียวกัน



สิ่งนี้รับประกันอิสรภาพอย่างสมบูรณ์: การเปลี่ยนแอปพลิเคชันโดยไม่สูญเสียรายชื่อผู้ติดต่อ, ความหลากหลายของอินเทอร์เฟซ, ความเป็นอิสระจากผู้จัดจำหน่ายรายเดียว



![capture](assets/fr/03.webp)



### เครือข่ายแบบกระจายศูนย์และสหพันธ์



โครงสร้างของ Matrix ขึ้นอยู่กับ **federation** ซึ่งเป็นรูปแบบที่เซิร์ฟเวอร์อิสระหลายตัวทำงานร่วมกัน


เซิร์ฟเวอร์แต่ละเครื่อง (เรียกว่า _homeserver_) สามารถโฮสต์ผู้ใช้ ห้องแชท และซิงโครไนซ์ข้อความกับเซิร์ฟเวอร์อื่นๆ ในเครือข่ายได้


ดังนี้ :





- ไม่มีหน่วยงานใดหน่วยงานหนึ่งควบคุมระบบทั้งหมด;
- เซิร์ฟเวอร์สามารถหายไปได้โดยไม่ส่งผลกระทบต่อส่วนที่เหลือของเครือข่าย;
- แต่ละองค์กรหรือบุคคลสามารถจัดการพื้นที่ของตนเองได้



โมเดลนี้รับประกัน**ความยืดหยุ่นสูง**และสะท้อนถึงคุณค่าของอธิปไตยทางเทคโนโลยี



![capture](assets/fr/03.webp)



### ระบบที่ปลอดภัยและเข้ารหัส



Matrix รองรับ **การเข้ารหัสแบบปลายทางถึงปลายทาง (E2EE)** สำหรับการแลกเปลี่ยนข้อมูลส่วนตัวและกลุ่มที่เข้ารหัส


ข้อความสามารถอ่านได้โดยผู้เข้าร่วมเท่านั้น ไม่ใช่โดยเซิร์ฟเวอร์กลาง


วิธีการนี้ทำให้สามารถสื่อสารได้โดยไม่เปิดเผยเนื้อหาของการแลกเปลี่ยนให้กับบุคคลที่สาม ในขณะที่ยังคงความโปร่งใสของโปรโตคอลและความเป็นไปได้ในการโฮสต์เซิร์ฟเวอร์ของตนเอง



![capture](assets/fr/05.webp)



### การทำงานร่วมกันที่ไม่เหมือนใคร



หนึ่งในทรัพย์สินที่สำคัญของ Matrix คือความสามารถในการทำหน้าที่เป็น **สะพานเชื่อมระหว่างระบบการสื่อสารที่แตกต่างกัน** ขอบคุณ _สะพาน_ ที่ทำให้สามารถเชื่อมต่อ :





- Telegram
- WhatsApp
- Signal
- เมสเซนเจอร์
- Slack
- ดิสคอร์ด
- IRC, XMPP, ฯลฯ



สิ่งนี้ทำให้สามารถรวมชุมชนที่กระจัดกระจายอยู่บนหลายแพลตฟอร์มเข้าด้วยกัน ในขณะที่ยังคงควบคุมโครงสร้างพื้นฐานได้



![capture](assets/fr/06.webp)



## Matrix ทำงานอย่างไร?



ส่วนนี้นำเสนอโครงสร้างภายในของเครือข่าย Matrix เพื่อทำความเข้าใจว่าผู้ใช้ เซิร์ฟเวอร์ และแอปพลิเคชันมีปฏิสัมพันธ์กันอย่างไรภายในระบบนิเวศแบบกระจายอำนาจนี้ Matrix มีพื้นฐานอยู่บนสามองค์ประกอบที่สำคัญ: _homeservers_, อัตลักษณ์, และ _clients_ ที่ใช้ในการสื่อสาร



### เซิร์ฟเวอร์: homeservers



Matrix ทำงานบนเซิร์ฟเวอร์อิสระที่เรียกว่า _homeservers_


แต่ละโฮมเซิร์ฟเวอร์จัดการ:





- บัญชีผู้ใช้ที่โฮสต์,
- การสนทนาส่วนตัวและเลานจ์ที่ผู้ใช้เหล่านี้เข้าร่วม,
- การซิงโครไนซ์กับเซิร์ฟเวอร์เครือข่ายอื่น ๆ



เซิร์ฟเวอร์บ้านทั้งหมดที่เชื่อมต่อกับเครือข่าย Matrix จะแลกเปลี่ยนข้อความและเหตุการณ์จากห้องนั่งเล่นที่ใช้ร่วมกันโดยอัตโนมัติ ตัวอย่างเช่น:





- ผู้ใช้ที่ลงทะเบียนบนเซิร์ฟเวอร์ A สามารถแชทกับผู้ใช้บนเซิร์ฟเวอร์ B ได้
- ร้านเสริมสวยสามารถกระจายไปยังเซิร์ฟเวอร์หลายสิบเครื่อง,
- ไม่มีใครมีอำนาจควบคุมร้านเสริมสวยหรือชุมชนโดยรวม



โมเดลนี้มีความยืดหยุ่นสูงและอนุญาตให้องค์กรหรือบุคคลแต่ละรายจัดการโครงสร้างพื้นฐานของตนเองได้



### ตัวระบุเมทริกซ์



ผู้ใช้แต่ละคนมีตัวระบุเฉพาะที่เรียกว่า **MXID** (_Matrix ID_) ซึ่งมีลักษณะเหมือนที่อยู่:



```bash
@nomdutilisateur:serveur.xyz
```



ประกอบด้วย :





- ชื่อผู้ใช้ นำหน้าด้วย **@**
- ชื่อของเซิร์ฟเวอร์ที่สร้างบัญชีขึ้นมา นำหน้าด้วย **:**



ตัวอย่าง:





- `@alice:matrix.org`
- `@bened:monserveur.bj`



ตัวระบุนี้ช่วยให้สามารถสื่อสารกับผู้ใช้ Matrix คนอื่น ๆ ได้ ไม่ว่าจะมาจากเซิร์ฟเวอร์ใดก็ตาม



### ไคลเอนต์เมทริกซ์ (แอปพลิเคชัน)



ในการใช้ Matrix คุณจำเป็นต้องเชื่อมต่อกับแอปพลิเคชันที่เรียกว่า **client Matrix**



ที่รู้จักกันดีที่สุดคือ :





- Element (เว็บ, มือถือ, เดสก์ท็อป)
- FluffyChat (มือถือ)
- Cinny (เว็บ/เดสก์ท็อปแบบมินิมอล)
- Nheko (เดสก์ท็อป)



แอปพลิเคชันเหล่านี้เป็นเพียงอินเทอร์เฟซสำหรับ :





- เพื่อดูข้อความ,
- ส่งข้อความ รูปภาพ หรือไฟล์,
- เข้าร่วมหรือสร้างงานแสดงสินค้า,
- โทรเสียง/วิดีโอ



แอปพลิเคชันทั้งหมดสื่อสารกับเซิร์ฟเวอร์ผ่านโปรโตคอลมาตรฐานเดียวกัน



### ห้องและข้อความส่วนตัว (DM)



ใน Matrix การแลกเปลี่ยนเกิดขึ้นใน **ห้อง** :





- ห้องสามารถเป็นสาธารณะหรือส่วนตัวได้
- มันสามารถรองรับได้ทั้งสองคนหรือหลายพันคน
- สามารถแชร์ระหว่างเซิร์ฟเวอร์หลายเครื่องได้
- มันมีตัวระบุเฉพาะที่เริ่มต้นด้วย **!**



ข้อความส่วนตัวคือห้องสนทนาที่มีผู้เข้าร่วมสองคน มักเรียกว่า **DM (Direct Messages)**



การซิงโครไนซ์ซาลอนเกิดขึ้นแบบเรียลไทม์ระหว่างเซิร์ฟเวอร์ที่เข้าร่วม เพื่อให้มั่นใจถึงประสบการณ์ที่ราบรื่นในขณะที่ยังคงการกระจายอำนาจ



## ทำไมต้องใช้ Matrix?



Matrix ไม่ใช่แค่ทางเลือกสำหรับระบบส่งข้อความแบบรวมศูนย์: มันเป็นเทคโนโลยีที่ตอบสนองความต้องการที่แท้จริงในแง่ของอธิปไตยทางดิจิทัล ความปลอดภัย และการทำงานร่วมกัน มีหลายเหตุผลที่ทำให้ผู้คน บริษัท และสถาบันต่างๆ เลือกใช้โปรโตคอลนี้ในการสื่อสารมากขึ้นเรื่อยๆ



### ควบคุมการสื่อสารของคุณอีกครั้ง



แพลตฟอร์มการส่งข้อความส่วนใหญ่ดำเนินการบนโมเดลแบบรวมศูนย์: ผู้เล่นรายเดียวควบคุมเซิร์ฟเวอร์ การเข้าถึง ข้อมูล และกฎการใช้งาน โมเดลนี้บ่งบอกถึงการพึ่งพาผู้ให้บริการอย่างสมบูรณ์


Matrix ใช้วิธีการที่แตกต่างออกไป


ใครๆ ก็สามารถเลือกได้ว่าจะโฮสต์บัญชีของตนที่ไหน หรือแม้กระทั่งติดตั้งเซิร์ฟเวอร์ของตนเอง ไม่มีหน่วยงานใดที่มีอำนาจในการบล็อกผู้ใช้ เรียกร้องการระบุตัวตนที่ล่วงล้ำ หรือบังคับใช้การเปลี่ยนแปลงนโยบาย


สถาปัตยกรรมนี้คืนอำนาจการควบคุมให้กับทั้งบุคคลและองค์กร การสื่อสารไม่ได้ขึ้นอยู่กับความไว้วางใจในบริษัทอีกต่อไป แต่ขึ้นอยู่กับโปรโตคอลที่เปิดกว้าง มีการบันทึกและตรวจสอบได้



### การสื่อสารที่ปลอดภัยและเข้ารหัส



Matrix รองรับการเข้ารหัสแบบ end-to-end สำหรับการสนทนาส่วนตัวและกลุ่ม กลไกนี้ช่วยให้มั่นใจได้ว่ามีเพียงผู้เข้าร่วมเท่านั้นที่สามารถอ่านข้อความได้ แม้ว่าข้อความจะผ่านเซิร์ฟเวอร์ของบุคคลที่สามในสหพันธ์ก็ตาม



โปรโตคอลนี้ใช้อัลกอริธึม Megolm/Olm ซึ่งออกแบบมาโดยเฉพาะเพื่อให้มีความปลอดภัยสูงในสภาพแวดล้อมที่มีการกระจายและใช้งานหลายอุปกรณ์



สิ่งนี้ทำให้สามารถ:





- ปกป้องการสนทนาที่ละเอียดอ่อน,
- ป้องกันการเข้าถึงโดยไม่ได้รับอนุญาต (แม้กระทั่งจากเซิร์ฟเวอร์โฮสต์),
- รักษาความลับในระยะยาว



การเข้ารหัสไม่ใช่ทางเลือก: มันเป็นส่วนประกอบหลักของโปรโตคอล



### ไม่ต้องพึ่งพาแอปพลิเคชันเดียวอีกต่อไป



Matrix ไม่ใช่แอปพลิเคชัน แต่เป็นโปรโตคอล



ความหลากหลายของลูกค้านี้รับประกัน:





- ทางเลือกที่ปรับให้เหมาะสมกับความต้องการของแต่ละบุคคล,
- ความสามารถในการใช้ Matrix บนอุปกรณ์ทุกประเภท,
- ไม่มีการพึ่งพาซอฟต์แวร์เพียงตัวเดียว



หากลูกค้าไม่เหมาะสมหรือหยุดการดูแลรักษา เพียงแค่เลือกอีกคนหนึ่ง: บัญชียังคงดำเนินการตามปกติ



### การรวมและเชื่อมต่อชุมชนต่าง ๆ เข้าด้วยกัน



สหพันธ์ช่วยให้เซิร์ฟเวอร์ต่างๆ ทำงานร่วมกันได้ในขณะที่ยังคงการจัดการอย่างอิสระ


ดังนี้ :





- องค์กรสามารถจัดการโฮมเซิร์ฟเวอร์ของตนเองได้,
- บุคคลสามารถเข้าร่วมเซิร์ฟเวอร์สาธารณะ,
- ทุกคนสามารถสื่อสารกันได้ราวกับว่าอยู่บนแพลตฟอร์มเดียวกัน



ความยืดหยุ่นนี้ทำให้สามารถสร้างพื้นที่การสื่อสารที่ปรับให้เหมาะสมกับทุกความต้องการ: ทีม, สมาคม, ชุมชน, สถาบัน หรือโครงการโอเพ่นซอร์ส



Matrix ได้รับความนิยมเป็นพิเศษในกลุ่มเทคนิค กลุ่มนักเคลื่อนไหว นักวิจัย รัฐบาล และเพิ่มมากขึ้นในชุมชน Bitcoin



### ความสามารถในการทำงานร่วมกันที่เป็นเอกลักษณ์ในภูมิทัศน์การส่งข้อความ



หนึ่งในทรัพย์สินที่สำคัญของ Matrix คือความสามารถในการ **ขยาย** การแลกเปลี่ยนด้วยสะพานที่สามารถเชื่อมโยง :





- WhatsApp
- Telegram
- Signal
- Slack
- ดิสคอร์ด
- ไออาร์ซี
- XMPP
- และแพลตฟอร์มอื่น ๆ อีกมากมาย



Matrix จึงกลายเป็นชั้นรวมสำหรับการสื่อสาร นำชุมชนหลายแห่งที่กระจัดกระจายอยู่ในแอปพลิเคชันต่าง ๆ มารวมกัน



การทำงานร่วมกันนี้ช่วยลดการกระจัดกระจายและทำให้การทำงานร่วมกันง่ายขึ้น



### โปรโตคอลที่เสรี เปิดกว้าง และยั่งยืน



โปรโตคอล Matrix เป็นโอเพ่นซอร์สทั้งหมดและพัฒนาอย่างโปร่งใส


สิ่งนี้รับประกันข้อดีหลายประการ:





- วิวัฒนาการอย่างต่อเนื่องของมาตรฐาน,
- ความสามารถสำหรับใครก็ตามในการตรวจสอบโค้ด
- อิสรภาพจากการเปลี่ยนแปลงทางการค้า หรือการเมือง,
- ความยืดหยุ่นในระยะยาว



ซึ่งแตกต่างจากระบบส่งข้อความที่เป็นกรรมสิทธิ์ อนาคตของ Matrix ไม่ได้ขึ้นอยู่กับบริษัทเดียว แต่ขึ้นอยู่กับชุมชนทั่วโลกและมาตรฐานเปิด



## วิธีสร้างบัญชี Matrix ทำอย่างไร?



การสร้างบัญชี Matrix นั้นง่ายและไม่ต้องการทักษะทางเทคนิค ผู้ใช้สามารถเข้าร่วมเซิร์ฟเวอร์ที่มีอยู่ สร้างการเข้าสู่ระบบและเริ่มแชทได้ทันที ส่วนนี้จะอธิบายขั้นตอนที่จำเป็น



### เลือกเซิร์ฟเวอร์ (สาธารณะหรือส่วนตัว)



Matrix เป็นเครือข่ายแบบกระจายศูนย์: มีเซิร์ฟเวอร์หลายตัว (homeservers) ที่บริหารจัดการโดยองค์กร ชุมชน หรือบุคคลต่าง ๆ การเลือกเซิร์ฟเวอร์จะกำหนดเพียง _ที่_ ที่บัญชีถูกโฮสต์ แต่ไม่เป็นอุปสรรคต่อการสื่อสารกับเครือข่ายทั้งหมด


**มีสองตัวเลือกให้เลือก:**



### - ใช้เซิร์ฟเวอร์สาธารณะ



นี่คือวิธีแก้ปัญหาที่ง่ายที่สุด


ตัวอย่างของเซิร์ฟเวอร์ยอดนิยม:





- _matrix.org_ (ที่รู้จักกันดีที่สุด)
- _envs.net_
- เซิร์ฟเวอร์ชุมชนตามธีม (เทคโนโลยี, ความเป็นส่วนตัว, โอเพ่นซอร์ส...)



เซิร์ฟเวอร์เหล่านี้เหมาะสำหรับผู้ใช้มือใหม่ที่ต้องการลงทะเบียนอย่างรวดเร็ว



### - ใช้เซิร์ฟเวอร์ส่วนตัว



เหมาะสำหรับ :





- องค์กร,
- ครอบครัว,
- โครงการโอเพ่นซอร์ส,
- ทีมงาน,
- หรือสำหรับการใช้งานแบบอธิปไตยและโฮสต์ด้วยตนเอง



ในกรณีนี้ จะต้องมีคนดูแลเซิร์ฟเวอร์ (Synapse, Dendrite, Conduit...)


ไม่ว่าคุณจะเลือกเซิร์ฟเวอร์ใด ผู้ใช้สามารถพูดคุยกันได้ด้วยการเชื่อมต่อแบบเฟเดอเรชั่น



### สร้างบัญชีทีละขั้นตอน



เนื่องจาก Matrix เป็นโปรโตคอลแบบเปิด จึงสามารถเข้าถึงได้โดยหลายแอปพลิเคชัน


ตามที่อธิบายไว้ข้างต้น พวกเขาเสนออินเทอร์เฟซและฟังก์ชันการทำงานที่แตกต่างกันขึ้นอยู่กับความต้องการ:





- Element**: ไคลเอนต์ที่สมบูรณ์ที่สุด ใช้งานได้บนทุกแพลตฟอร์ม
- FluffyChat**: เรียบง่าย ทันสมัย และเหมาะสำหรับมือถือ
- Nheko**: ไคลเอนต์ที่มีน้ำหนักเบาและออกแบบตามหลักสรีรศาสตร์สำหรับพีซี
- SchildiChat**: Element variant with ergonomic improvements.
- NeoChat**: รวมเข้ากับระบบนิเวศ KDE



การเลือกไคลเอนต์ไม่มีผลกระทบต่อบัญชี: ทุกอย่างทำงานร่วมกับเซิร์ฟเวอร์ Matrix ใด ๆ ได้



### ขั้นตอนคลาสสิก :





- เปิดแอปพลิเคชันที่เลือก ในกรณีของเรา เราจะทำสิ่งนี้ด้วย [Element](app.element.io)
- เลือก "สร้างบัญชี"



![cover-kali](assets/fr/10.webp)



เพื่อความง่าย คุณสามารถสร้างบัญชี Matrix โดยใช้ **Google, Facebook, Apple, GitHub หรือ GitLab** บริการเหล่านี้จะทราบเพียงว่าบัญชีของพวกเขาได้ถูกใช้เพื่อเข้าถึง Matrix: สิ่งนี้เรียกว่า **การเชื่อมต่อทางสังคม**



เพื่อความเป็นส่วนตัวที่มากขึ้น คุณยังสามารถลงทะเบียนด้วยตนเองโดยเลือก **ชื่อผู้ใช้**, **รหัสผ่าน** และ **ที่อยู่อีเมล**



ขึ้นอยู่กับเซิร์ฟเวอร์ที่เลือก อาจจำเป็นต้องมี **captcha** รวมถึงการยอมรับ **ข้อกำหนดการใช้งาน**



เมื่อลงทะเบียนเสร็จสิ้นและได้รับการยืนยันแล้ว อีเมลยืนยันจะถูกส่งออกไป


เพียงคลิกที่ลิงก์เพื่อเปิดใช้งานบัญชีของคุณและเข้าถึงเว็บแอปพลิเคชัน (Element) เพื่อเข้าร่วมการสนทนา Matrix ครั้งแรกของคุณ



![cover-kali](assets/fr/11.webp)



คุณมีบัญชีแล้วและใช้เวอร์ชันเว็บของ Element.

ในการสื่อสารกับใครบางคนบน Matrix สิ่งที่คุณจำเป็นต้องรู้คือรหัสประจำตัวเต็มของพวกเขา ซึ่งเรียกว่า **Matrix ID**



ตัวอย่าง:



`@alice:matrix.org @bened:monserveur.bj`



### เพิ่มผู้ติดต่อ



ในการเชิญเพื่อนเข้าร่วมแชทกลุ่มของคุณ ให้คลิกที่วงกลม `i` ที่มุมขวาบน ซึ่งจะเปิดแผงด้านขวา คลิกที่ "People" เพื่อแสดงรายชื่อสมาชิกในห้องนี้: คุณควรจะเป็นคนเดียวที่อยู่ในขณะนี้



![cover-kali](assets/fr/12.webp)

ไม่มีระบบ "เพื่อน": ผู้ติดต่อคือบุคคลที่มีการเปิดการสนทนาด้วยเท่านั้น



![cover-kali](assets/fr/13.webp)



บุคคลที่คุณเชิญสามารถยอมรับหรือปฏิเสธคำเชิญได้ หากพวกเขายอมรับ คุณควรเห็นพวกเขาเข้าร่วมห้อง ยิ่งมากยิ่งสนุก!



![cover-kali](assets/fr/14.webp)



### การตั้งค่าเซิร์ฟเวอร์ของคุณเอง



Matrix จะมีประสิทธิภาพสูงสุดเมื่อใช้ร่วมกับเซิร์ฟเวอร์ส่วนตัว


การปรับใช้เซิร์ฟเวอร์ส่วนตัวของคุณช่วยให้คุณสามารถ:





- รักษาการควบคุมข้อมูลอย่างสมบูรณ์,
- กำหนดกฎการใช้งานของตัวเอง,
- โฮสต์หลายบัญชี (เพื่อน, ทีม, ชุมชน),
- และรับรองความยืดหยุ่นสูงสุดในกรณีที่มีข้อจำกัดหรือการเซ็นเซอร์



**Available solutions:**





- Synapse**: การดำเนินการที่มีประวัติศาสตร์และสมบูรณ์ที่สุด
- Dendrite**: เบากว่า ทรงพลังมากขึ้น และออกแบบเพื่ออนาคตของโปรโตคอล
- Conduit**: รูปแบบมินิมอลที่ติดตั้งได้ง่าย



**ข้อกำหนดเบื้องต้น:**





- ชื่อโดเมน,
- เครื่องหรือ VPS,
- ทักษะการบริหารระบบขั้นต่ำ



แม้ว่าจะต้องมีการกำหนดค่าบ้าง การจัดการเซิร์ฟเวอร์ของคุณเองจะเปลี่ยน Matrix ให้เป็นเครื่องมือที่มีอำนาจอธิปไตยและทนทาน



### การเข้าร่วมงานแสดงสินค้าครั้งแรกของคุณ



Matrix พึ่งพา _rooms_ (ห้องนั่งเล่น) เป็นอย่างมาก


มีงานแสดงสินค้าสาธารณะ เอกชน ชุมชน เทคนิค ท้องถิ่น และนานาชาติ



**สามวิธีในการเข้าร่วมร้านเสริมสวย:**



1. **ผ่านลิงก์เชิญ** (มักอยู่ในรูปแบบของ URL `matrix.to`)


2. **ค้นหาชื่อร้านเสริมสวย** ในแอปพลิเคชัน


3. **โดยการป้อนรหัสการแสดงเต็มรูปแบบ**, เช่น :


`#bitcoin:matrix.org`


`#communauté-bénin:monsrv.bj`



เมื่อเข้าร่วมแล้ว ห้องสนทนาจะทำงานเหมือนกลุ่มข่าวคลาสสิก โดยมีประวัติ การเข้ารหัส ไฟล์ ปฏิกิริยา และการโทรด้วยเสียง/วิดีโอ ขึ้นอยู่กับไคลเอนต์ที่ใช้



![capture](assets/fr/09.webp)



## ก้าวไปข้างหน้า



เมื่อคุณเชี่ยวชาญพื้นฐานแล้ว Matrix มีความเป็นไปได้ขั้นสูงมากมาย ไม่ว่าคุณต้องการเชื่อมต่อระบบส่งข้อความอื่น ๆ โฮสต์เซิร์ฟเวอร์ของคุณเอง หรือจัดระเบียบชุมชน ระบบนิเวศนั้นมีความหลากหลายมาก



### สะพาน (WhatsApp, Telegram, Signal, ฯลฯ)



สะพานเชื่อมต่อ Matrix กับระบบส่งข้อความอื่น ๆ


ด้วยสิ่งนี้ คุณสามารถส่งและรับข้อความจาก :





- WhatsApp
- Telegram**
- Signal**
- เฟซบุ๊ก เมสเซนเจอร์
- ดิสคอร์ด
- Slack**
- IRC** (IRC)
- และอื่น ๆ อีกมากมาย



### สิ่งที่สะพานสามารถทำได้





- รวมการสนทนาทั้งหมดของคุณใน Matrix
- ให้บริการอินเทอร์เฟซแบบเปิดสำหรับการโต้ตอบกับบริการที่เป็นกรรมสิทธิ์
- จัดการชุมชนหลายแพลตฟอร์มจากที่เดียว



สะพานบางแห่งเป็นทางการ ส่วนสะพานอื่นๆ เป็นของชุมชน


ขึ้นอยู่กับแผนก พวกเขาอาจต้องการ :





- เซิร์ฟเวอร์ส่วนตัว,
- การกำหนดค่าเพิ่มเติม,
- หรือการใช้สะพานสาธารณะที่มีอยู่แล้ว



### การใช้ Matrix สำหรับองค์กร Bitcoin, ชุมชน หรือโครงการ



Matrix ไม่ใช่แค่เครื่องมือส่วนบุคคลเท่านั้น


สามารถใช้เพื่อจัดโครงสร้างกลุ่มงาน จัดระเบียบชุมชนท้องถิ่น หรือจัดการการสื่อสารโครงการ



**ตัวอย่างการใช้งาน:**





- ชุมชนโอเพ่นซอร์ส
- Bitcoin และโครงการระบบนิเวศ Lightning
- กลุ่มนักเรียนหรือนักพัฒนา
- องค์กรพลเมือง
- สื่ออิสระ
- กลุ่มและสมาคมท้องถิ่น



**ทำไมถึงน่าสนใจ?**





- เครื่องมือ** โอเพ่นซอร์ส 100%
- การสื่อสารที่มีอธิปไตยและกระจายศูนย์**
- พื้นที่จัดเป็น **เลานจ์**, **กลุ่มย่อย**, **เลานจ์ส่วนตัว**, เป็นต้น
- ผสานรวมกับ Nextcloud, GitLab, Mattermost, หรือบอทที่กำหนดเอง
- การจัดการสิทธิ์และการควบคุมอย่างละเอียด



Matrix จึงกลายเป็นเสาหลักในการสื่อสารสำหรับโครงสร้างใด ๆ ที่ต้องการคงความเป็นอิสระจากแพลตฟอร์มขนาดใหญ่ที่มีการรวมศูนย์



## บทสรุป



Matrix แสดงถึงโซลูชันที่ทันสมัย เปิดกว้าง และปลอดภัยสำหรับการสื่อสารแบบเรียลไทม์ โดยเสนอทางเลือกแบบกระจายศูนย์แทนแพลตฟอร์มแบบดั้งเดิม ด้วยสถาปัตยกรรมแบบเฟเดอเรตและโปรโตคอลการเข้ารหัสขั้นสูง ทำให้ผู้ใช้สามารถควบคุมข้อมูลของตนเองได้ในขณะที่เพลิดเพลินกับประสบการณ์ที่ลื่นไหลและสามารถทำงานร่วมกันได้ ไม่ว่าจะใช้ในด้านส่วนตัว ด้านอาชีพ หรือชุมชน Matrix เสนอกรอบการทำงานที่แข็งแกร่งและสามารถขยายได้สำหรับการสร้างสภาพแวดล้อมการสื่อสารที่ปรับให้เข้ากับความต้องการของวันนี้
