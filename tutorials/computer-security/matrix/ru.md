## Что такое матрица?



Matrix - это **децентрализованный коммуникационный протокол**, предназначенный для обмена сообщениями, файлами и аудио/видеозвонками между пользователями и приложениями, не зависящий от центрального предприятия. В отличие от традиционных платформ обмена сообщениями, Matrix представляет собой **открытую инфраструктуру**, сравнимую с электронной почтой: каждый может выбрать себе сервер или управлять им самостоятельно, сохраняя при этом возможность обмена с остальной частью сети.



Матрицу отличают три основополагающих принципа:



### Протокол, а не приложение



Matrix - это не приложение вроде WhatsApp или Telegram.


Это стандартизированный язык, который могут использовать многие приложения.


Другими словами, самые разные программы Element, FluffyChat, Cinny, Nheko и другие, предоставляют доступ к одной и той же сети Matrix.



Это гарантирует полную свободу: смена применения без потери контактов, разнообразие интерфейсов, независимость от единственного поставщика.



![capture](assets/fr/03.webp)



### Децентрализованная, объединенная сеть



Структура Matrix основана на **федерации** - модели, в которой несколько независимых серверов сотрудничают друг с другом.


Каждый сервер (называемый _homeserver_) может размещать пользователей, чаты и синхронизировать сообщения с другими серверами в сети.


Таким образом





- ни один человек не контролирует всю систему;
- сервер может исчезнуть без ущерба для остальной сети;
- каждая организация или отдельный человек может управлять своим пространством.



Эта модель обеспечивает **высокую устойчивость** и отражает ценности технологического суверенитета.



![capture](assets/fr/03.webp)



### Безопасная, зашифрованная система



Matrix поддерживает **end-to-end шифрование (E2EE)** для частных обменов и зашифрованных групп.


Сообщения могут быть прочитаны только участниками, но не промежуточными серверами.


Такой подход позволяет общаться, не раскрывая содержимое обменов третьей стороне, сохраняя при этом прозрачность протокола и возможность размещения собственного сервера.



![capture](assets/fr/05.webp)



### Уникальная совместимость



Одним из главных достоинств Matrix является его способность выступать в качестве **моста между различными системами связи**. Благодаря _мостам_ можно соединять :





- Telegram
- WhatsApp
- Signal
- Посланник
- Slack
- Дискорд
- IRC, XMPP и т. д.



Это позволяет объединить сообщества, разбросанные по нескольким платформам, сохранив при этом контроль над инфраструктурой.



![capture](assets/fr/06.webp)



## Как работает матрица?



В этом разделе представлена внутренняя структура сети Matrix, чтобы понять, как пользователи, серверы и приложения взаимодействуют в рамках этой децентрализованной экосистемы. В основе Matrix лежат три основных элемента: _домашних серверах_, идентификаторах и _клиентах_, используемых для связи.



### Серверы: homeservers



Matrix работает на независимых серверах, называемых _homeservers_.


Каждый домашний сервер управляет :





- учетные записи пользователей, которые на нем размещены,
- частные беседы и гостиные, в которых участвуют эти пользователи,
- синхронизация с другими сетевыми серверами.



Все домашние серверы, подключенные к сети Matrix, автоматически обмениваются сообщениями и событиями из общих гостиных. Например:





- пользователь, зарегистрированный на сервере A, может общаться с пользователем на сервере B,
- салон может быть распределен по десяткам серверов,
- никто не может контролировать ни салон, ни сообщество в целом.



Эта модель отличается высокой отказоустойчивостью и позволяет каждой организации или отдельному человеку самостоятельно управлять своей инфраструктурой.



### Матричные идентификаторы



Каждый пользователь имеет уникальный идентификатор, называемый **MXID** (_Matrix ID_), который похож на адрес:



```bash
@nomdutilisateur:serveur.xyz
```



Он состоит из :





- имя пользователя, которому предшествует **@**
- имя сервера, на котором была создана учетная запись, которому предшествует **:**



Примеры:





- `@alice:matrix.org`
- `@bened:monserveur.bj`



Этот идентификатор позволяет общаться с любым другим пользователем Matrix, независимо от сервера, с которого он был отправлен.



### Матричные клиенты (приложения)



Чтобы использовать Matrix, вам необходимо подключиться к приложению под названием **клиент Matrix**.



Самые известные из них - :





- Элемент (веб, мобильный, настольный)
- FluffyChat (мобильный)
- Cinny (минималистичный веб/десктоп)
- Nheko (рабочий стол)



Эти приложения - всего лишь интерфейсы для :





- для просмотра сообщений,
- отправляйте текст, изображения или файлы,
- присоединяйтесь к торговым выставкам или создавайте их,
- совершать аудио- и видеозвонки.



Все приложения взаимодействуют с серверами по одному и тому же стандартизированному протоколу.



### Комнаты и личные сообщения (DM)



В Matrix обмен происходит в **комнатах**:





- комната может быть общественной или частной
- он может вместить как двух человек, так и тысячи
- он может быть разделен между несколькими серверами
- у него есть уникальный идентификатор, начинающийся с **!**



Личные сообщения - это просто чат с двумя участниками, часто называемый **DM (Direct Messages)**.



Синхронизация салонов происходит в режиме реального времени между участвующими серверами, обеспечивая бесперебойную работу при сохранении децентрализации.



## Почему стоит использовать Matrix?



Matrix - это не просто альтернатива централизованным системам обмена сообщениями: это технология, которая отвечает реальным потребностям в плане цифрового суверенитета, безопасности и совместимости. Существует множество причин, по которым все больше людей, компаний и учреждений выбирают этот протокол для общения.



### Верните себе контроль над коммуникациями



Большинство платформ обмена сообщениями работают по централизованной модели: один игрок контролирует серверы, доступ, данные и правила использования. Такая модель подразумевает полную зависимость от поставщика.


Matrix использует другой подход.


Каждый может выбрать, где разместить свою учетную запись, или даже развернуть собственный сервер. Ни одна организация не в состоянии заблокировать пользователя, потребовать навязчивой идентификации или навязать изменение политики.


Эта архитектура возвращает автономию как отдельным людям, так и организациям. Коммуникации теперь основаны не на доверии к компании, а на открытом, документированном и проверяемом протоколе.



### Безопасная, зашифрованная связь



Matrix поддерживает сквозное шифрование личных бесед и групп. Этот механизм гарантирует, что только участники могут читать сообщения, даже если они проходят через сторонние серверы в федерации.



Протокол использует алгоритм Megolm/Olm, разработанный специально для обеспечения надежной защиты в распределенных средах с несколькими устройствами.



Это позволяет :





- защищайте конфиденциальные разговоры,
- предотвращение несанкционированного доступа (даже с главного сервера),
- сохранять конфиденциальность в течение длительного времени.



Шифрование - это не опция: это основной компонент протокола.



### Больше не зависит от одного приложения



Matrix - это не приложение, а протокол.



Такое разнообразие клиентов гарантирует :





- выбор, адаптированный к индивидуальным потребностям,
- возможность использовать Matrix на любом типе устройств,
- отсутствие зависимости от одного программного обеспечения.



Если клиент не подходит или перестает обслуживаться, просто выберите другого: счет продолжает работать в обычном режиме.



### Объединение и взаимосвязь различных сообществ



Федерация позволяет различным серверам работать вместе и при этом управляться независимо друг от друга.


Таким образом





- организация может управлять собственным домашним сервером,
- люди могут присоединяться к публичным серверам,
- все могут общаться друг с другом, как будто они находятся на одной платформе.



Такая гибкость позволяет создавать коммуникационные пространства, адаптированные под любые нужды: команды, ассоциации, сообщества, учреждения или проекты с открытым исходным кодом.



Матрица особенно популярна в технических кругах, коллективах активистов, исследователей, правительств и все чаще в сообществах Bitcoin.



### Уникальная функциональная совместимость в сфере обмена сообщениями



Одним из главных достоинств Matrix является возможность **расширения** бирж благодаря мостам, способным соединять :





- WhatsApp
- Telegram
- Signal
- Slack
- Дискорд
- IRC
- XMPP
- и многие другие платформы



Таким образом, Matrix становится объединяющим слоем для коммуникаций, объединяющим несколько сообществ, разбросанных по разным приложениям.



Такая совместимость уменьшает фрагментацию и упрощает сотрудничество.



### Свободный, открытый и устойчивый протокол



Протокол Matrix является полностью открытым и прозрачно разработанным.


Это гарантирует несколько преимуществ:





- непрерывное развитие стандарта,
- возможность любому желающему проверить код,
- независимость от коммерческих или политических изменений,
- долгосрочной устойчивости.



В отличие от проприетарных систем обмена сообщениями, будущее Matrix зависит не от одной компании, а от глобального сообщества и открытого стандарта.



## Как создать учетную запись Matrix?



Создание учетной записи Matrix очень просто и не требует никаких технических навыков. Пользователи могут присоединиться к существующему серверу, создать логин и сразу же начать общение. В этом разделе описаны основные шаги.



### Выберите сервер (публичный или частный)



Matrix - это федеративная сеть: существует множество серверов (хоумсерверов), управляемых различными организациями, сообществами или отдельными людьми. Выбор сервера определяет только _место_ размещения учетной записи, но не препятствует общению со всей сетью.


**Доступны два варианта:**



### - Используйте публичный сервер



Это самое простое решение.


Примеры популярных серверов:





- _matrix.org_ (самый известный)
- _envs.net_
- тематические серверы сообщества (технологии, конфиденциальность, открытый исходный код...)



Эти серверы подходят для начинающих пользователей, которые хотят быстро зарегистрироваться.



### - Используйте частный сервер



Идеально подходит для :





- организация,
- семья,
- проект с открытым исходным кодом,
- рабочая команда,
- или для самостоятельного использования.



В этом случае кто-то должен администрировать сервер (Synapse, Dendrite, Conduit...).


Независимо от того, какой сервер вы выберете, пользователи смогут общаться друг с другом благодаря федерации.



### Создание учетной записи шаг за шагом



Поскольку Matrix является открытым протоколом, доступ к нему могут получить несколько приложений.


Как было описано выше, они предлагают различные интерфейсы и функциональные возможности в зависимости от требований:





- Element**: самый полный клиент, доступный на всех платформах.
- FluffyChat**: простой, современный и идеально подходит для мобильных устройств.
- Nheko**: легкий, эргономичный клиент для ПК.
- SchildiChat**: Вариант Element с эргономическими улучшениями.
- NeoChat**: интегрирован в экосистему KDE.



Выбор клиента не влияет на работу аккаунта: все они работают с любым сервером Matrix.



### Классические шаги :





- Откройте выбранное приложение. В нашем случае мы сделаем это с помощью [Element](app.element.io).
- Выберите "Создать учетную запись".



![cover-kali](assets/fr/10.webp)



Для простоты вы можете создать учетную запись Matrix, используя **Google, Facebook, Apple, GitHub или GitLab**. Эти сервисы будут знать только о том, что их учетная запись использовалась для доступа к Matrix: это называется **социальным подключением**.



Для большей конфиденциальности вы также можете зарегистрироваться вручную, выбрав **имя пользователя**, **пароль** и **адрес электронной почты**.



В зависимости от выбранного сервера может потребоваться **капча**, а также согласие с **условиями использования**.



После того как регистрация будет подтверждена, на электронную почту придет подтверждение.


Просто нажмите на ссылку, чтобы активировать свою учетную запись и получить доступ к веб-приложению (Element), чтобы присоединиться к своим первым беседам в Matrix.



![cover-kali](assets/fr/11.webp)



Теперь у вас есть учетная запись и вы пользуетесь веб-версией Element.



## Добавьте свой первый контакт



Чтобы общаться с кем-то в Matrix, вам нужно знать его полный идентификатор, называемый **Matrix ID**.



Пример:



`@alice:matrix.org @bened:monserveur.bj`



### Добавить контакт



Чтобы пригласить друзей в групповой чат, нажмите на кружок `i` в правом верхнем углу. Откроется правая панель. Нажмите на "Люди", чтобы отобразить список участников в этой комнате: в данный момент вы должны быть единственными присутствующими.



![cover-kali](assets/fr/12.webp)



Нажмите на кнопку "Пригласить в эту комнату" в верхней части списка людей, и откроется окно, в котором вы сможете пригласить своих друзей присоединиться к вам в Matrix. Если они уже вошли в Matrix, введите их идентификатор Matrix ID. Если нет, введите их адрес электронной почты, и они будут приглашены присоединиться к нам.



Здесь нет системы "друзей": контакт - это просто человек, с которым был открыт разговор.



![cover-kali](assets/fr/13.webp)



Приглашенный вами человек может либо принять, либо отклонить приглашение. Если он примет приглашение, вы увидите, как он присоединится к комнате. Чем больше, тем веселее!



![cover-kali](assets/fr/14.webp)



### Создание собственного сервера



Matrix по-настоящему раскрывает свои возможности при использовании в сочетании с персональным сервером.


Развертывание собственного домашнего сервера позволяет вам :





- сохранять полный контроль над данными,
- определить собственные правила использования,
- ведите несколько учетных записей (друзья, команда, сообщество),
- и обеспечить максимальную устойчивость в случае ограничений или цензуры.



**Доступные решения:**





- Synapse**: историческая и наиболее полная реализация.
- Dendrite**: легче, мощнее и предназначен для будущего протокола.
- Conduit**: минималистичный вариант, который легко развернуть.



**Пререквизиты:**





- доменное имя,
- машина или VPS,
- минимальные навыки системного администрирования.



Даже если это требует некоторой настройки, управление собственным сервером превращает Matrix в надежный и долговечный инструмент.



### Участие в ваших первых выставках



Матрица в значительной степени опирается на _комнаты_ (жилые комнаты).


Существуют государственные, частные, общественные, технические, местные и международные выставки.



**Три способа присоединиться к салону:**



1. **По пригласительной ссылке** (часто в виде URL-адреса `matrix.to`).


2. **Поиск названия салона** в приложении.


3. **Введя полный идентификатор шоу**, например :


`#bitcoin:matrix.org`


`#communauté-bénin:monsrv.bj`



После присоединения чат ведет себя как классическая группа новостей, с историей, шифрованием, файлами, реакциями и аудио/видео звонками, в зависимости от используемого клиента.

## Идем дальше



Как только вы освоите основы, Matrix предложит вам множество дополнительных возможностей. Хотите ли вы подключить другие системы обмена сообщениями, разместить свой собственный сервер или организовать сообщество - экосистема очень богата.



### Мосты (WhatsApp, Telegram, Signal и т.д.)



Мост соединяет Matrix с другими системами обмена сообщениями.


С его помощью вы можете отправлять и получать сообщения из :





- WhatsApp
- Telegram**
- Signal**
- Facebook Messenger
- Дискорд
- Slack**
- IRC** (IRC)
- и многие другие

Некоторые мосты являются официальными, другие - общественными.


В зависимости от отдела им может потребоваться :





- персональный сервер,
- дополнительная конфигурация,

- Суверенная и децентрализованная** связь
- Пространства, разделенные на **гостиные**, **подгруппы**, **частные гостиные** и т.д.
- Интеграция с Nextcloud, GitLab, Mattermost или пользовательскими ботами
- Тонкая настройка управления разрешениями и модерацией



Matrix становится коммуникационной опорой для любой структуры, желающей оставаться независимой от крупных централизованных платформ.



## Заключение



Matrix представляет собой современное, открытое и безопасное решение для коммуникации в режиме реального времени, предлагающее децентрализованную альтернативу традиционным платформам. Благодаря федеративной архитектуре и передовым протоколам шифрования, она позволяет пользователям сохранять контроль над своими данными, наслаждаясь при этом гибкостью и совместимостью. Будь то личное, профессиональное или общественное использование, Matrix предлагает надежную и масштабируемую основу для создания коммуникационных сред, адаптированных к современным потребностям.
