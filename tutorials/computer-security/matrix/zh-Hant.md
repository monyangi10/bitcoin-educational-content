## Matrix 是什麼？



Matrix 是一種**分散式通訊協定**，旨在讓使用者與應用程式之間能夠交換訊息、檔案和音訊/視訊通話，而無需依賴中央企業。與傳統的訊息傳輸平台不同，Matrix 是一種**開放式的基礎架構，可與電子郵件相媲美：任何人都可以選擇伺服器或自行操作伺服器，同時保留與其他網路交換的能力。



Matrix 有三個基本原則：



### 協定，而非應用程式



Matrix 不是 WhatsApp 或 Telegram 之類的應用程式。


它是一種標準化的語言，許多應用程式都可以使用。


換句話說，各式各樣的 Element 軟體、FluffyChat、Cinny、Nheko 及其他軟體，都能提供存取同一 Matrix 網路的功能。



這保證了完全的自由度：在不失去接觸點的情況下改變應用、多樣化的介面、獨立於單一供應商。



![capture](assets/fr/03.webp)



### 分散式聯盟網路



Matrix 的結構是以**聯邦**為基礎，這是一種數個獨立伺服器彼此合作的模式。


每台伺服器 (稱為 _homeserver_)都可以寄存使用者、聊天室，並與網路上的其他伺服器同步訊息。


因此：





- 沒有單一實體可以控制整個系統；
- 伺服器可以消失而不影響網路的其他部分；
- 每個組織或個人都可以管理自己的空間。



此模式可確保**高彈性**，並反映技術主權的價值。



![capture](assets/fr/03.webp)



### 安全的加密系統



Matrix 支援私人交換和加密群組的 ** 端對端加密 (E2EE)**。


訊息只能由參與者讀取，不能由中間伺服器讀取。


這種方式使通訊不需向第三方揭露交換的內容，同時保留通訊協定的透明性，並可託管自己的伺服器。



![capture](assets/fr/05.webp)



### 獨特的互操作性



Matrix 的主要優勢之一是它能夠作為不同通訊系統**之間的**橋樑。有了_橋樑_，就可以連接 ：





- Telegram
- WhatsApp
- Signal
- 信使
- 懶惰
- 討論區
- IRC、XMPP 等。



這讓分散在多個平台上的社群得以統一，同時保留對基礎架構的控制。



![capture](assets/fr/06.webp)



## Matrix 如何運作？



本節將介紹 Matrix 網路的內部結構，以瞭解使用者、伺服器和應用程式如何在此分散式生態系統中互動。Matrix 基於三個基本元素：用於溝通的 _homeservers_、身分和 _client_。



### 伺服器：家庭伺服器



Matrix 在稱為 _homeservers_ 的獨立伺服器上執行。


每個家庭伺服器管理 ：





- 它託管的使用者帳戶、
- 這些使用者參與的私人對話和休息室、
- 與其他網路伺服器同步。



所有連接到 Matrix 網路的家庭伺服器會自動交換共用客廳的訊息和事件。舉例來說





- 在伺服器 A 上註冊的使用者可以與伺服器 B 上的使用者聊天、
- 一個沙龍可以分散在數十台伺服器上、
- 任何人都無法控制沙龍或整個社區。



此模式具有高度彈性，可讓每個組織或個人管理自己的基礎結構。



### 矩陣識別碼



每個使用者都有一個獨特的識別碼，稱為 **MXID** (_Matrix ID_)，看起來像一個地址：



```bash
@nomdutilisateur:serveur.xyz
```



它包括 ：





- 使用者名稱，前面加上 **@**
- 建立帳戶的伺服器名稱，前面加上 **:**



範例：





- `@alice:matrix.org`
- `@bened:monserveur.bj`



此識別碼可讓您與任何其他 Matrix 使用者進行通訊，不論發端伺服器為何。



### 矩陣用戶端（應用程式）



若要使用 Matrix，您需要連線一個名為 **client Matrix** 的應用程式。



最著名的有：





- 元素 (網頁、行動裝置、桌上型電腦)
- FluffyChat (行動版)
- Cinny (極簡網頁/桌面)
- Nheko (桌上型電腦)



這些應用程式只是 .NET 的介面：





- 檢視訊息、
- 傳送文字、影像或檔案、
- 加入或建立貿易展、
- 進行音訊/視訊通話。



所有應用程式都透過相同的標準通訊協定與伺服器通訊。



### 房間和私人訊息 (DM)



在 Matrix 中，交換是在 ** 室**進行的：





- 一個房間可以是公共或私人的
- 可容納兩人或上千人
- 可由多台伺服器共用
- 它有一個以 **!** 開頭的唯一識別碼



私人訊息只是兩個參與者的聊天室，通常稱為 **DM（直接訊息）**。



沙龍同步化會在參與的伺服器之間即時進行，以確保無縫的體驗，同時維持分散性。



## 為何使用 Matrix？



Matrix 不只是集中式訊息系統的替代方案：它是一種能滿足數位主權、安全性和互通性等實際需求的技術。越來越多人、公司和機構選擇此通訊協定的原因有很多。



### 重新掌控您的通訊



大多數訊息平台都以集中式模式運作：由單一業者控制伺服器、存取、資料和使用規則。這種模式意味著完全依賴供應商。


Matrix 採取了不同的方法。


任何人都可以選擇在何處託管帳戶，甚至部署自己的伺服器。任何實體都無法封鎖使用者、要求侵入性識別或強制變更政策。


此架構讓個人和組織重新擁有自主權。溝通不再是基於對公司的信任，而是基於開放、有記錄且可驗證的協定。



### 安全、加密的通訊



Matrix 支援私人對話和群組的端對端加密。此機制能確保只有參與者能讀取訊息，即使訊息是經過聯盟中的第三方伺服器。



該通訊協定使用 Megolm/Olm 演算法，專為在分散式多裝置環境中提供強大的安全性而設計。



這使得 ：





- 保護敏感的對話、
- 防止未經授權的存取 (甚至從主機伺服器)、
- 長期保密。



加密不是選項：它是協定的核心元件。



### 不再依賴單一應用程式



Matrix 不是一種應用程式，而是一種通訊協定。



客戶的多樣性保證了 ：





- 適合個人需求的選擇、
- 能夠在任何類型的裝置上使用 Matrix、
- 不依賴單一軟體。



如果客戶不適合或停止維護，只需選擇其他客戶：帳戶繼續正常運作。



### 聯盟不同社群並將其相互連結



聯盟允許不同的伺服器一起工作，同時獨立管理。


因此：





- 組織可以管理自己的家庭伺服器、
- 個人可以加入公共伺服器、
- 所有人都可以像在同一個平台上一樣互相溝通。



這種彈性讓我們可以創造適合各種需求的通訊空間：團隊、協會、社群、機構或開放原始碼專案。



Matrix 在技術圈子、激進集團、研究人員、政府，以及越來越多的 Bitcoin 社群中特別受歡迎。



### 訊息傳輸領域中獨特的互操作性



Matrix 的主要優勢之一，就是能夠透過可連結 .NET 和 .NET Framework 的橋接，**擴展**交換的能力：





- WhatsApp
- Telegram
- Signal
- 懶惰
- 討論區
- IRC
- XMPP
- 和許多其他平台



Matrix 因此成為通訊的統一層，將分散在不同應用程式上的數個社群聚合在一起。



這種互操作性可減少分散性並簡化協作。



### 免費、開放且可持續的協定



Matrix 通訊協定是完全開放原始碼且透明地開發。


這保證了幾個優點：





- 標準的持續演進、
- 任何人都可以檢查程式碼的能力、
- 獨立於商業或政治變遷、
- 長期的適應力。



與專屬訊息系統不同，Matrix 的未來並非取決於單一公司，而是取決於全球社群和開放標準。



## 如何建立 Matrix 帳戶？



建立 Matrix 帳戶非常簡單，不需要任何技術能力。使用者可以加入現有的伺服器、建立登入帳號，然後馬上開始聊天。本節概述了基本步驟。



### 選擇伺服器（公共或私人）



Matrix 是一個聯盟網路：有許多伺服器 (家庭伺服器) 由不同的組織、社群或個人管理。伺服器的選擇只會決定帳號託管的地點，但不會妨礙與整個網路的通訊。


**提供兩種選項：**



### - 使用公共伺服器



這是最簡單的解決方案。


受歡迎伺服器的範例：





- _matrix.org_（最著名的）
- _envs.net_
- 主題社群伺服器（科技、隱私、開放源碼...）



這些伺服器適合想要快速註冊的新手使用者。



### - 使用私人伺服器



適用於 ：





- 一個組織、
- 一個家庭
- 一個開放原始碼專案、
- 一個工作團隊、
- 或主權、自主控使用。



在這種情況下，必須有人管理伺服器 (Synapse、Dendrite、Conduit...)。


無論您選擇哪個伺服器，使用者都可以透過聯盟彼此交談。



### 逐步建立帳戶



由於 Matrix 是開放式通訊協定，因此可被多種應用程式存取。


如上所述，它們會根據需求提供不同的介面和功能：





- Element**: 最完整的用戶端，適用於所有平台。
- FluffyChat**: 簡單、現代，非常適合行動裝置使用。
- Nheko**: 適用於 PC 的輕量、符合人體工學的用戶端。
- SchildiChat**：符合人體工學的元素變體。
- NeoChat**: 整合到 KDE 生態系統中。



用戶端的選擇對帳戶沒有影響：都可以在任何 Matrix 伺服器上使用。



### 經典步驟 ：





- 開啟選取的應用程式。在我們的例子中，我們會使用 [Element](app.element.io)。
- 選擇「建立帳號」。



![cover-kali](assets/fr/10.webp)



為了簡單起見，您可以使用 **Google、Facebook、Apple、GitHub 或 GitLab** 建立 Matrix 帳戶。這些服務只會知道他們的帳號已被用來存取 Matrix：這就是所謂的**社交連線**。



為了提高保密性，您也可以選擇**使用者名稱**、**密碼**和**電子郵件地址**，進行手動註冊。



根據所選擇的伺服器，可能需要**captcha**，以及接受**使用條款**。



註冊一經驗證，我們就會發送一封確認電子郵件。


只需按一下連結即可啟動您的帳戶，並存取網路應用程式 (Element)，加入您的第一次 Matrix 對話。



![cover-kali](assets/fr/11.webp)



您現在擁有一個帳戶，並使用 Element 的 Web 版本。



## 新增您的第一個聯絡人



若要在 Matrix 上與某人通訊，您只需要知道他的完整識別碼，稱為 **Matrix ID**。



範例：



`@alice:matrix.org@bened:monserveur.bj`。



### 新增聯絡人



若要邀請朋友加入群聊，請按一下右上角的 `i` 圓圈。這會開啟右側面板。按一下「人」以顯示此會議室的成員清單：目前應該只有您在場。



![cover-kali](assets/fr/12.webp)



按一下人員清單頂端的「邀請到此房間」，就會開啟一個提示，讓您可以邀請朋友加入 Matrix。如果他們已經登入 Matrix，請輸入他們的 Matrix ID。如果他們尚未登入，請輸入他們的電子郵件地址，他們就會被邀請加入我們。



沒有「朋友」系統：連絡人只是一個已開啟對話的人。



![cover-kali](assets/fr/13.webp)



您所邀請的人可以選擇接受或拒絕邀請。如果他們接受，您應該會看到他們加入房間。人數越多越好！



![cover-kali](assets/fr/14.webp)



### 設定您自己的伺服器



Matrix 在與個人伺服器結合使用時，才真正發揮其功能。


部署您自己的家庭伺服器，可讓您.NET、ASP.NET 及 .NET 的使用者都能使用您的家庭伺服器：





- 保持對資料的完全控制、
- 定義自己的使用規則、
- 託管多個帳號 (朋友、團隊、社群)、
- 並確保在受到限制或審查時，仍能維持最大的彈性。



**可用的解決方案：**





- Synapse**: 歷史悠久且最完整的實作。
- Dendrite**: 更輕、更強大，專為未來通訊協定所設計。
- Conduit**：簡約的變體，容易部署。



**先決條件：**





- 域名、
- 機器或 VPS、
- 最低限度的系統管理技能。



即使需要一點設定，管理您自己的伺服器也能將 Matrix 變成主權和耐用的工具。



### 參加您的首次貿易展



Matrix 非常依賴 _rooms_（客廳）。


有公共、私人、社區、技術、當地和國際貿易展。



**加入美容院的三種方式：**



1. **Via an invitation link** (通常為 `matrix.to` URL 的形式)。


2. **在應用程式中搜尋髮廊名稱**。


3. ** 輸入完整的節目 ID**，例如 ：


`#bitcoin:matrix.org`


`#communauté-bénin:monsrv.bj`



一旦加入，聊天室的行為就像經典的新聞群組一樣，有歷史、加密、檔案、反應和音訊/視訊通話，視使用的用戶端而定。



![capture](assets/fr/09.webp)



## 更進一步



一旦您掌握了基本知識，Matrix 就能提供許多進階的可能性。無論您想連接其他訊息系統、架設自己的伺服器或組織社群，這個生態系統都非常豐富。



### 橋樑 (WhatsApp、Telegram、Signal 等)



橋接器可將 Matrix 連接到其他訊息系統。


有了它，您可以從 .NET 平台傳送和接收訊息：





- WhatsApp
- Telegram**
- Signal**
- Facebook Messenger
- 討論區
- 松弛**
- IRC** (IRC)
- 及其他



### 橋樑可以做什麼





- 在 Matrix 中集中所有對話
- 提供與專屬服務互動的開放介面
- 從單一位置管理多平台社群



有些橋樑是官方的，有些則以社區為基礎。


視部門而定，他們可能會要求 ：





- 個人伺服器、
- 附加設定、
- 或使用現有的公共橋樑。



### 為 Bitcoin 組織、社區或專案使用 Matrix



Matrix 不只是個人工具。


它可用於組織工作群組、組織當地社群或管理專案溝通。



**使用範例：**





- 開放原始碼社群
- Bitcoin 和 Lightning 生態系統專案
- 學生或開發人員團體
- 公民組織
- 獨立媒體
- 當地團體和協會



**為什麼這很有趣？





- 100% 開放原始碼**工具
- 主權與分散**通訊
- 空間組織為**休息室**、**小組**、**私人休息室**等。
- 與 Nextcloud、GitLab、Mattermost 或自訂機器人整合
- 權限和審查的微調管理



Matrix 因此成為任何希望獨立於大型集中式平台的架構的溝通支柱。



## 總結



Matrix 是現代化、開放且安全的即時通訊解決方案，提供傳統平台的分散式替代方案。由於其聯邦架構和先進的加密通訊協定，使用者可以保留對其資料的控制，同時享受流暢、可互操作的體驗。無論是個人、專業或社群使用，Matrix 都能提供強大且可擴充的架構，以建立符合現今需求的通訊環境。
