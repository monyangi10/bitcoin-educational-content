## 什么是 Matrix？



Matrix 是一种**分散的通信协议，旨在实现用户和应用程序之间的信息、文件和音频/视频通话交换，而无需依赖中央企业。与传统的信息平台不同，Matrix 是一种**开放的基础设施，可与电子邮件相媲美：任何人都可以选择一台服务器或自己操作一台服务器，同时保留与网络其他部分交换信息的能力。



Matrix 有三个基本原则：



### 协议而非应用程序



Matrix 不是 WhatsApp 或 Telegram 这样的应用程序。


这是一种标准化语言，许多应用程序都可以使用。


换句话说，FluffyChat、Cinny、Nheko 等各种 Element 软件都可以访问同一个 Matrix 网络。



这就保证了完全的自由度：在不丢失触点的情况下改变应用、接口多样化、独立于单一供应商。



![capture](assets/fr/03.webp)



### 分散式联合网络



Matrix 的结构基于**联邦**，即多个独立服务器相互合作的模式。


每台服务器（称为 _homeserver_）都可以托管用户、聊天室，并与网络上的其他服务器同步信息。


因此 ：





- 没有一个实体可以控制整个系统；
- 服务器可以消失而不影响网络的其他部分；
- 每个组织或个人都可以管理自己的空间。



这种模式确保了**高的恢复能力**，体现了技术主权的价值。



![capture](assets/fr/03.webp)



### 安全的加密系统



Matrix 支持 ** 端到端加密 (E2EE)**，用于私人交换和加密组。


信息只能由参与者读取，不能由中间服务器读取。


这种方法可以在不向第三方暴露交换内容的情况下进行通信，同时保留协议的透明度和托管自己服务器的可能性。



![capture](assets/fr/05.webp)



### 独特的互操作性



Matrix 的主要优势之一是它能够充当不同通信系统之间的**桥梁**。有了桥接器，就可以连接 ：





- Telegram
- WhatsApp
- Signal
- 信使
- 松弛
- 纪和声
- IRC、XMPP 等。



这样就可以将分散在多个平台上的社区统一起来，同时保留对基础设施的控制。



![capture](assets/fr/06.webp)



## Matrix 如何工作？



本节介绍 Matrix 网络的内部结构，以了解用户、服务器和应用程序如何在这个去中心化的生态系统中互动。Matrix 基于三个基本要素：主服务器、身份和用于通信的客户。



### 服务器：家庭服务器



Matrix 在独立的服务器上运行，这些服务器被称为_homeservers_。


每个家庭服务器管理 ：





- 用户账户、
- 这些用户参加的私人谈话和聚会、
- 与其他网络服务器同步。



所有连接到 Matrix 网络的家庭服务器都会自动交换来自共享客厅的信息和事件。例如





- 在服务器 A 上注册的用户可以与服务器 B 上的用户聊天、
- 一个沙龙可以分布在几十台服务器上、
- 没有人能控制沙龙或整个社区。



这种模式具有高度弹性，允许每个组织或个人管理自己的基础设施。



### 矩阵标识符



每个用户都有一个唯一的标识符，称为 **MXID**（_矩阵 ID_），看起来就像一个地址：



```bash
@nomdutilisateur:serveur.xyz
```



它包括 ：





- 用户名，前加 **@**
- 创建账户的服务器名称，前面加上 **:**



例如





- `@alice:matrix.org`
- `@bened:monserveur.bj`



有了这个标识符，就可以与任何其他 Matrix 用户进行通信，而不受发端服务器的限制。



### 矩阵客户端（应用程序）



要使用 Matrix，您需要连接一个名为 **client Matrix** 的应用程序。



最著名的有：





- 元素（网络、手机、桌面）
- FluffyChat (手机)
- Cinny（极简网页/桌面）
- Nheko (桌面)



这些应用程序只是.NET 的接口：





- 查看信息、
- 发送文本、图像或文件
- 加入或创建贸易展览、
- 拨打音频/视频电话



所有应用程序都通过相同的标准化协议与服务器通信。



### 房间和私人信息 (DM)



在 Matrix 系统中，交流是在**房间**进行的：





- 一个房间可以是公共的，也可以是私人的
- 可容纳两人或数千人
- 可由多个服务器共享
- 它有一个以**！**开头的唯一标识符



私人信息只是两个参与者的聊天室，通常被称为 **DM（直接信息）**。



沙龙同步在参与服务器之间实时进行，在确保无缝体验的同时保持分散性。



## 为什么使用 Matrix？



Matrix 不仅仅是集中式信息传输系统的替代品：它还是一种能够满足数字主权、安全性和互操作性等实际需求的技术。越来越多的人、公司和机构选择这种协议进行通信，原因有很多。



### 重新掌控您的通信



大多数信息平台以集中模式运行：由一家公司控制服务器、访问、数据和使用规则。这种模式意味着完全依赖供应商。


Matrix 采用了不同的方法。


任何人都可以选择在哪里托管自己的账户，甚至部署自己的服务器。任何实体都无法阻止用户、要求进行侵入式身份验证或强行更改政策。


这种架构将自主权交还给个人和组织。通信不再基于对公司的信任，而是基于开放、有据可查、可验证的协议。



### 安全的加密通信



Matrix 支持对私人对话和群组进行端到端加密。这种机制可确保只有参与者才能读取信息，即使信息是通过联盟中的第三方服务器发送的。



该协议使用 Megolm/Olm 算法，专为在分布式多设备环境中提供强大的安全性而设计。



这样就可以 ：





- 保护敏感对话、
- 防止未经授权的访问（甚至从主机服务器）、
- 长期保密。



加密不是一个选项，而是协议的核心组成部分。



### 不再依赖单一应用程序



Matrix 不是一个应用程序，而是一个协议。



客户的多样性保证了 ：





- 适合个人需求的选择、
- 能够在任何类型的设备上使用 Matrix、
- 不依赖单一软件。



如果某个客户不合适或不再需要维护，只需选择另一个客户：账户继续正常运行。



### 不同社区的联合与互联



联盟允许不同的服务器在独立管理的同时协同工作。


因此 ：





- 一个组织可以管理自己的家庭服务器、
- 个人可以加入公共服务器、
- 所有人都可以像在同一个平台上一样相互交流。



这种灵活性使其能够创建适应各种需求的交流空间：团队、协会、社区、机构或开源项目。



Matrix 在技术界、活动团体、研究人员、政府中特别流行，在 Bitcoin 社区中也越来越流行。



### 信息传递领域独一无二的互操作性



Matrix 的主要优势之一是它能够**扩展**交换，这要归功于它能够连接 .NET 与 .NET 之间的桥梁：





- WhatsApp
- Telegram
- Signal
- 松弛
- 纪和声
- IRC
- XMPP
- 和许多其他平台



因此，矩阵成为通信的统一层，将分散在不同应用中的多个社区汇聚在一起。



这种互操作性减少了分散，简化了协作。



### 自由、开放和可持续的协议



Matrix 协议完全开源，开发过程透明。


这样做有几个好处：





- 标准的不断发展、
- 任何人都能检查代码、
- 不受商业或政治变革的影响、
- 长期复原力。



与专有消息系统不同，Matrix 的未来并不取决于一家公司，而是取决于全球社区和开放标准。



## 如何创建 Matrix 账户？



创建 Matrix 帐户非常简单，无需任何技术技能。用户可以加入现有服务器，创建登录名并立即开始聊天。本节概述了基本步骤。



### 选择服务器（公共或私人）



Matrix 是一个联合网络：有许多由不同组织、社区或个人管理的服务器（家庭服务器）。服务器的选择只决定账户的托管地点，但并不妨碍与整个网络的通信。


**有两种选择：**



### - 使用公共服务器



这是最简单的解决方案。


常用服务器举例





- _matrix.org_（最著名的）
- _envs.net_____。
- 专题社区服务器（技术、隐私、开源......）



这些服务器适合希望快速注册的新手用户。



### - 使用私人服务器



适用于 ：





- 一个组织、
- 一个家庭
- 是一个开放源码项目、
- 一个工作团队、
- 或供主权国家自行使用。



在这种情况下，必须有人管理服务器（Synapse、Dendrite、Conduit......）。


无论您选择哪个服务器，用户之间都可以通过联盟进行对话。



### 创建账户步骤



由于 Matrix 是一个开放协议，多个应用程序都可以访问它。


如上所述，它们根据需求提供不同的界面和功能：





- Element**：最完整的客户端，适用于所有平台。
- FluffyChat**：简洁、现代，非常适合手机使用。
- Nheko**：适用于个人电脑的轻量级、符合人体工程学的客户端。
- SchildiChat**：改进了人体工程学的元素变体。
- NeoChat**: 集成到 KDE 生态系统中。



客户端的选择对账户没有影响：所有账户都可与任何 Matrix 服务器一起使用。



### 经典步骤 ：





- 打开所选应用程序。在本例中，我们将使用 [Element](app.element.io)。
- 选择 "创建账户"。



![cover-kali](assets/fr/10.webp)



为简单起见，您可以使用**谷歌、Facebook、苹果、GitHub 或 GitLab** 创建 Matrix 账户。这些服务只知道它们的账户被用来访问 Matrix：这就是所谓的**社交连接**。



为提高保密性，您也可以选择**用户名**、**密码**和**电子邮件地址**，进行手动注册。



根据所选服务器的不同，可能需要**验证码**以及接受**使用条款**。



一旦注册通过验证，系统将发送一封确认电子邮件。


只需点击链接激活账户，并访问网络应用程序（Element），即可加入第一次 Matrix 对话。



![cover-kali](assets/fr/11.webp)



您现在拥有一个账户，可以使用网络版的 Element。



## 添加您的第一个联系人



要在 Matrix 上与某人通信，您只需知道他的完整标识符，即 **Matrix ID**。



例如



`@alice:matrix.org@bened:monserveur.bj`。



### 添加联系人



要邀请好友加入群聊，请单击右上角的 "i "圆圈。这将打开右侧面板。点击 "人"，显示该房间的成员名单：目前应该只有你们。



![cover-kali](assets/fr/12.webp)



单击人员列表顶部的 "邀请到此房间"，系统会弹出提示，您可以邀请朋友加入 Matrix。如果他们已经登录 Matrix，请输入他们的 Matrix ID。如果他们还没有登录，请输入他们的电子邮件地址，他们就会被邀请加入我们。



没有 "好友 "系统：联系人只是与之开启对话的人。



![cover-kali](assets/fr/13.webp)



您邀请的人可以接受或拒绝邀请。如果他们接受邀请，你就可以看到他们加入房间。越多越好！



![cover-kali](assets/fr/14.webp)



### 建立自己的服务器



当 Matrix 与个人服务器结合使用时，它才真正发挥了自己的作用。


部署自己的家庭服务器可以让你.NET Framework 3.0.0和.NET Framework 4.0.0更快地运行：





- 保持对数据的完全控制、
- 确定自己的使用规则、
- 托管多个账户（好友、团队、社区）、
- 并确保在受到限制或审查时具有最大的复原力。



**可用的解决方案：**





- Synapse**：历史最悠久、最完整的实施方案。
- Dendrite**：更轻、更强大，专为未来协议而设计。
- Conduit**：易于部署的简约变体。



**前提条件：**





- 域名、
- 机器或 VPS、
- 最低限度的系统管理技能。



即使需要进行一些配置，管理自己的服务器也能将 Matrix 变成一个可靠耐用的工具。



### 参加首次贸易展览



Matrix 在很大程度上依赖于_rooms_（起居室）。


有公共、私人、社区、技术、地方和国际展览。



**加入美容院的三种方式：**



1. **通过邀请链接**（通常采用 "matrix.to "URL 的形式）。


2. **在应用程序中搜索沙龙名称**。


3. **输入完整的节目 ID**，例如 ：


`#bitcoin:matrix.org``


`#communauté-bénin:monsrv.bj`.



一旦加入，聊天室的行为就像一个传统的新闻组，具有历史记录、加密、文件、反应和音频/视频通话功能，具体取决于所使用的客户端。

## 更进一步



一旦掌握了基础知识，Matrix 还能提供大量高级功能。无论您是想连接其他消息系统、托管自己的服务器还是组织一个社区，生态系统都非常丰富。



### 桥梁（WhatsApp、Telegram、Signal 等）



桥接器将 Matrix 与其他报文系统连接起来。


通过它，您可以从 .NET Framework 发送和接收信息：





- WhatsApp
- Telegram**
- Signal**
- Facebook Messenger
- 纪和声
- 松弛**
- IRC** (IRC)
- 及其他



### 桥梁的作用





- 在 Matrix 中集中处理所有对话
- 提供与专有服务交互的开放界面
- 从单一位置管理多平台社区



有些桥梁是官方的，有些则是社区的。


根据不同的部门，他们可能需要.....：





- 个人服务器、
- 附加配置、
- 或使用现有的公共桥梁。



### 为 Bitcoin 组织、社区或项目使用 Matrix



Matrix 不仅仅是一种个人工具。


它可用于构建工作组、组织当地社区或管理项目交流。



**使用示例：**





- 开源社区
- Bitcoin 和闪电生态系统项目
- 学生或开发人员团体
- 公民组织
- 独立媒体
- 地方团体和协会



**为什么这很有趣？





- 100% 的开源**工具
- 主权和分散**通信
- 空间分为**休息室**、**分组**、**私人休息室**等。
- 与 Nextcloud、GitLab、Mattermost 或自定义机器人集成
- 权限和审核的微调管理



Matrix 因此成为任何希望独立于大型中央平台的机构的通信支柱。
