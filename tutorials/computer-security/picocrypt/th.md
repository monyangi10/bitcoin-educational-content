## I. การนำเสนอ



ในบทแนะนำนี้ เราจะมาดู Picocrypt ซึ่งเป็นซอฟต์แวร์เข้ารหัสที่เรียบง่าย เบา และมีประสิทธิภาพสำหรับการเข้ารหัสข้อมูลด้วยระดับความปลอดภัยสูง



เหมาะสำหรับการ **เข้ารหัสไฟล์** คุณสามารถใช้มันเพื่อปกป้อง **ข้อมูลบนคอมพิวเตอร์ของคุณ บน USB stick** แต่ยังรวมถึงข้อมูลที่เก็บไว้ใน Cloud ด้วย ตัวอย่างเช่น คุณสามารถเข้ารหัสข้อมูลและเก็บไว้บน **Microsoft OneDrive, Google Drive, iCloud หรือ Dropbox** แม้ว่าสำหรับวัตถุประสงค์นี้ฉันจะชอบซอฟต์แวร์อีกตัวหนึ่งซึ่งจะถูกนำเสนอในบทความถัดไป



คุณยังสามารถใช้มันเมื่อคุณต้องการ **แชร์ข้อมูลกับบุคคลที่สาม**: ด้วย Picocrypt และกุญแจถอดรหัส พวกเขาจะสามารถถอดรหัสข้อมูลบนเครื่องของพวกเขาได้ ดังนั้นหากบัญชีหรือคอมพิวเตอร์ของคุณถูกบุกรุก ข้อมูลของคุณก็ยังคงได้รับการปกป้อง



ในการติดตามโครงการ Picocrypt มีเพียงที่อยู่เดียว:





- [Picocrypt on GitHub](https://github.com/Picocrypt/Picocrypt)



Totally **free and open source**, PicoCrypt is available for **Windows,** **Linux** and **macOS**. On Windows, you can install it on your own machine or use the portable version.



## II. Picocrypt, ซอฟต์แวร์เข้ารหัสโอเพ่นซอร์ส

Picocrypt มีน้ำหนัก **เบามาก** โดยมีขนาดเพียง **3 MB** และไม่จำเป็นต้องติดตั้ง: เป็น **แอปพลิเคชันแบบพกพา** ที่มีข้อดีคือไม่ต้องการสิทธิ์ของผู้ดูแลระบบ! อย่างไรก็ตาม มันไม่ละเลยความปลอดภัย เนื่องจากอาศัย **อัลกอริธึมที่แข็งแกร่งและเชื่อถือได้**:





- อัลกอริทึมการเข้ารหัส XChaCha20**
- ฟังก์ชันบายพาสคีย์ **Argon2**



เหนือกว่าข้อดีที่กล่าวถึงไปแล้ว สิ่งที่ดึงดูดใจจริง ๆ คือ **ความง่ายในการใช้งาน**!



มันต้องการเพียงสิ่งเดียว: **การตรวจสอบโค้ด**, แต่สิ่งนั้นได้วางแผนไว้แล้ว ดังที่คุณเห็นจากตารางเปรียบเทียบด้านบน (บรรทัดสุดท้าย) แต่เนื่องจากมันเป็นโอเพ่นซอร์ส ไม่มีอะไรที่จะหยุดคุณจากการดูซอร์สโค้ดของมันได้



แม้ว่าจะมีการเปรียบเทียบกับ BitLocker ในตารางข้างต้น **ในความเห็นของฉัน BitLocker และ Picocrypt มีวัตถุประสงค์การใช้งานที่แตกต่างกัน**: BitLocker สำหรับการเข้ารหัสทั้งไดรฟ์ (เช่นของ Windows) และ Picocrypt สำหรับการเข้ารหัสโครงสร้างต้นไม้หรือพื้นที่จัดเก็บประเภท "Drive"



## III. การใช้ Picocrypt



สำหรับการสาธิตนี้ จะใช้เครื่อง Windows 11



### A. การเข้ารหัสข้อมูลด้วย Picocrypt



ก่อนอื่น คุณต้องดาวน์โหลด Picocrypt.exe จาก GitHub อย่างเป็นทางการ ([ดูหน้านี้](https://github.com/Picocrypt/Picocrypt/releases))



เปิดแอปพลิเคชันเพื่อแสดง Interface ที่มีความเรียบง่ายบนหน้าจอ ในการเข้ารหัสข้อมูล ไม่ว่าจะเป็น **ไฟล์เดียว, หลายไฟล์ หรือโฟลเดอร์** เพียงแค่ **ลากและวางลงใน Picocrypt's Interface** ซึ่งจะเลือกข้อมูลที่จะเข้ารหัส



![Image](assets/fr/020.webp)



จากนั้น สำหรับการเข้ารหัสข้อมูล คุณสามารถเข้าถึงตัวเลือกหลายอย่าง รวมถึงกุญแจการเข้ารหัส ซึ่งอาจเป็น **รหัสผ่านที่แข็งแกร่ง** หรือ **กุญแจการเข้ารหัสในรูปแบบของไฟล์กุญแจ** หรือ **ทั้งสองอย่าง** หากเรายกตัวอย่างรหัสผ่าน คุณมีทางเลือกระหว่างการสร้างรหัสผ่านของคุณเอง หรือสร้างรหัสผ่านด้วย Picocrypt



รหัสผ่านนี้ต้องมีความแข็งแกร่ง เนื่องจากสามารถใช้ถอดรหัสข้อมูลได้ ให้ป้อนภายใต้ "**Password**" และ "**Confirm Password**" จากนั้นคลิกที่ "**Encrypt**" เพื่อเข้ารหัสข้อมูล! ก่อนหน้านั้น คุณสามารถคลิกที่ปุ่ม "**Change**" ถัดจาก "**Save output as**" เพื่อระบุไดเรกทอรีเอาต์พุตได้



**หมายเหตุ**: เพื่อใช้คีย์ในรูปแบบไฟล์ ให้คลิกที่ "**สร้าง**" ทางด้านขวาของ "**ไฟล์คีย์**" เพื่อสร้างคีย์ จากนั้นเลือกโดยคลิกที่ "**แก้ไข**" และลากแล้ววางไฟล์คีย์ลงในพื้นที่ที่เหมาะสม



![Image](assets/fr/019.webp)



ไฟล์สองไฟล์ที่เลือกถูก **เข้ารหัสและจัดกลุ่มเข้าด้วยกัน** ในไฟล์ "**Encrypted.zip.pcv**" เนื่องจาก **PCV** เป็นนามสกุลที่ใช้โดย Picocrypt ไฟล์ ZIP นี้ไม่สามารถอ่านได้เนื่องจากการเข้ารหัส เพื่อป้องกันไม่ให้ไฟล์ที่เลือกถูกจัดกลุ่มเข้าด้วยกันในไฟล์ ZIP ที่เข้ารหัสเดียว คุณจำเป็นต้องเลือกตัวเลือก "**Recursively**" เพื่อให้มีไฟล์ที่เข้ารหัสมากเท่ากับจำนวนไฟล์ที่ต้องการเข้ารหัส



ในการเข้าถึงข้อมูลของเรา เราจำเป็นต้องถอดรหัสโดยใช้ Picocrypt



![Image](assets/fr/023.webp)



ก่อนที่เราจะพูดถึงการถอดรหัสข้อมูล นี่คือข้อมูลเกี่ยวกับตัวเลือกบางอย่างที่มีอยู่:





- โหมดหวาดระแวง**: ใช้ระดับความปลอดภัยสูงสุดที่เสนอโดย Picocrypt เครื่องมือนี้จะใช้หลายอัลกอริธึมการเข้ารหัสแบบเรียงซ้อน (XChaCha20 และ Serpent) และ HMAC-SHA3 แทน BLAKE2b สำหรับการยืนยันข้อมูล
- Reed-Solomon**: implement *Reed-Solomon* error correction codes to facilitate error correction on corrupted data. This allows you to support a corruption level of around 3% of your file.
- แบ่งออกเป็นส่วนๆ** หรือ **แบ่งออกเป็นหลายส่วน**: หากคุณกำลังเข้ารหัสไฟล์ขนาดใหญ่ คุณสามารถขอให้ Picocrypt แบ่งไฟล์ออกเป็นหลายส่วนได้ ซึ่งอาจทำให้ไฟล์ง่ายต่อการถ่ายโอนมากขึ้น
- บีบอัดไฟล์** หรือ **บีบอัดไฟล์**: บีบอัดไฟล์เพื่อลดขนาดของไฟล์ที่เข้ารหัส
- ลบไฟล์** หรือ **Fichiers supprimés**: ลบไฟล์ต้นฉบับเพื่อเก็บเฉพาะเวอร์ชันที่เข้ารหัส



### B. ถอดรหัสข้อมูลด้วย Picocrypt



หากคุณต้องการถอดรหัสข้อมูล มันไม่ซับซ้อนไปกว่าการเข้ารหัส เพียงเปิด Picocrypt และ **ลากและวางไฟล์ PCV ที่จะถอดรหัส** จากนั้นป้อนรหัสผ่านและ/หรือเลือกไฟล์คีย์ก่อนคลิกที่ "**Decrypt**"



![Image](assets/fr/021.webp)



เวอร์ชันที่ไม่ได้เข้ารหัสของไฟล์ ZIP "Encrypted.zip" ตอนนี้ทำให้ฉันสามารถกู้คืนไฟล์สองไฟล์ของฉันในรูปแบบข้อความที่ชัดเจนได้แล้ว!



![Image](assets/fr/022.webp)



## IV. บทสรุป



**คุณได้รับการเตือนแล้ว: Picocrypt ใช้งานง่ายมาก และมันได้ผล! แม้ว่า Interface จะมีความเรียบง่าย แต่ก็มีตัวเลือกที่มีประโยชน์มากสำหรับการปรับแต่งการเข้ารหัส! และเนื่องจากมีเวอร์ชันพกพา คุณจึงสามารถพกพาไปได้ทุกที่ที่คุณไป เพื่อให้คุณสามารถถอดรหัสข้อมูลของคุณได้อย่างมั่นใจ**



อย่าลืมใช้รหัสผ่านที่แข็งแกร่งเพื่อปกป้องข้อมูล และหากคุณใช้ไฟล์คีย์ คุณต้องจำสำรองข้อมูลไว้ มิฉะนั้นคุณจะไม่สามารถถอดรหัสคอนเทนเนอร์ PCV ที่สร้างโดย Picocrypt ได้อีกต่อไป สุดท้ายนี้ คุณควรรู้ว่ามี [เวอร์ชัน CLI](https://github.com/Picocrypt/CLI) (ที่มีฟีเจอร์น้อยกว่า) ที่ให้คุณรัน Picocrypt จากบรรทัดคำสั่ง: อีกครั้ง Picocrypt เปิดประตูสู่ความเป็นไปได้ใหม่ๆ
