![cover-bebop](assets/cover.webp)



**be-BOP**은 온라인과 오프라인에서 완전한 자율성을 가지고 판매하고자 하는 기업가를 위해 설계된 이커머스 플랫폼으로, 은행 계좌와 현금을 통해 Bitcoin으로 결제할 수 있습니다. 이 솔루션은 기부금을 모금하거나 다양한 활동으로 수익을 창출하고자 하는 모든 유형의 조직에도 유용합니다.



이 솔루션은 간단하고 가벼우며 자율적입니다. 기존 금융 서비스가 제한적이거나 부재한 환경에서도 온라인 스토어를 만들 수 있습니다. 실제로 **be-BOP**는 Bitcoin을 결제 인프라로 사용하여 은행 이용 여부와 관계없이 효율적으로 운영되도록 설계되었습니다.



이 튜토리얼에서는 단계별로 안내해 드리겠습니다:





- 비밥**으로 첫 온라인 스토어 만들기
- 쇼케이스 및 제품 맞춤 설정
- 사용 가능한 결제 방법 구성
- 비밥**으로 온라인에서 효과적으로 판매하기 위한 모범 사례 이해하기



이 튜토리얼은 고급 기술이 필요하지 않습니다. 개발자는 물론 장인, 상인, 협동조합 또는 기업가들이 주권적이고 탄력적인 방식으로 디지털 상거래를 시작하고자 하는 사람들을 대상으로 합니다.



## 자체 서버에 be-BOP를 설치하기 위한 전제 조건



Be-BOP 설치를 시작하기 전에 다음과 같은 기술 인프라가 있는지 확인하세요. 이러한 Elements는 플랫폼이 올바르게 작동하는 데 필수적입니다:



### S3 호환 스토리지



be-BOP은 스토리지 시스템을 사용하여 파일(예: 제품 이미지)을 관리합니다. 이를 위해서는 다음과 같은 S3 서비스에 대한 액세스가 필요합니다:





- [MinIO](https://min.io/) 자체 호스팅
- Amazon S3(AWS)
- 스케일웨이 오브젝트 스토리지



버킷을 구성하고 다음 정보를 제공해야 합니다:





- S3_BUCKET**: 버킷 이름
- S3_ENDPOINT_URL**: S3 서비스에 대한 액세스 링크
- S3_KEY_ID** 및 **S3_KEY_SECRET**: 액세스 코드
- S3_REGION**: S3 서비스의 지역



### 레플리카셋 모드의 몽고DB 데이터베이스



be-BOP은 몽고DB를 사용하여 스토어, 사용자, 제품 및 기타 데이터를 저장합니다.



두 가지 옵션이 있습니다:





- 복제 설정 모드를 활성화**하여 로컬에 MongoDB를 설치합니다
- 몽고DB 아틀라스**와 같은 온라인 서비스 사용



다음 변수가 필요합니다:





- MONGODB_URL**: 데이터베이스 연결 Address
- MONGODB_DB**: 데이터베이스 이름



## Node.js 환경



be-BOP은 Node.js와 함께 작동합니다. Node.js** 버전 18 이상과 **Corepack**이 활성화되어 있는지 확인하세요(pnpm과 같은 패키지 관리자를 관리하는 데 필요). 실행 명령은 `corepack enable`입니다



### Git LFS 설치



일부 리소스(예: 대용량 이미지)는 Git LFS(대용량 파일 저장소)를 통해 관리됩니다. 'git lfs install` 명령으로 컴퓨터에 Git LFS가 설치되어 있는지 확인하세요. 이러한 전제 조건이 갖추어지면 다음 단계인 be-BOP 다운로드 및 구성으로 넘어갈 준비가 된 것입니다.



**참고: 소프트웨어 배포에 대한 기술 가이드는 별도의 튜토리얼에서 확인할 수 있습니다.



## 슈퍼 관리자 계정 만들기



Be-BOP을 처음 실행하면 **슈퍼 관리자** 계정이 생성됩니다. 이 계정에는 백오피스 기능을 관리하는 데 필요한 모든 권한이 있습니다. 계정을 만들려면 다음 단계를 따르세요:





- 귀하의 사이트웹/관리자/로그인`으로 이동합니다
- 안전한 로그인과 비밀번호로 슈퍼 관리자 계정 만들기



이 계정으로 모든 백오피스 기능에 액세스할 수 있습니다. 계정을 만든 후에는 사용자 아이디와 비밀번호를 입력하여 로그인할 수 있습니다.

## 커뮤니케이션 설정



Be-BOP에서 알림(예: 주문, 등록 또는 시스템 메시지)을 보내도록 설정하려면 하나 이상의 통신 방법을 구성해야 합니다. 이메일(SMTP) 또는 Nostr의 두 가지 옵션을 사용할 수 있습니다.



### SMTP 구성(이메일)



be-BOP은 SMTP 서버를 통해 이메일을 보낼 수 있습니다. 이메일 서비스(예: Mailgun, Gmail 등)에서 제공하는 유효한 SMTP 자격 증명이 필요합니다.



알아야 할 사항은 다음과 같습니다:



SMTP_HOST: SMTP 서버 Address(예: smtp.mailgun.org)



SMTP_PORT: 사용할 포트(보통 587 또는 465)



SMTP_USER: 사용자 이름(일반적으로 이메일 Address)



SMTP_PASSWORD: 비밀번호 또는 API 키



SMTP_FROM: 발신자로 표시될 이메일 Address입니다




### Nostr 구성



be-BOP을 사용하면 탈중앙화 메시징 인프라인 Nostr 프로토콜을 통해 알림을 보낼 수 있습니다. 이렇게 하려면 Nostr 개인 키(NSEC)를 generate 또는 Supply로 변환해야 합니다. 이 키는 Nostr 전용 섹션에서 be-BOP의 Interface을 통해 직접 generate할 수 있습니다. 이러한 Elements가 올바르게 구성되면 be-BOP가 사용자에게 자동으로 메시지와 알림을 보낼 수 있습니다.



## 호환되는 결제 방법



be-BOP은 여러 결제 솔루션과 호환되므로 고객에게 더 큰 유연성을 제공할 수 있습니다. 가장 적합한 결제 방법을 설정하는 데 필요한 사항은 다음과 같습니다.



### Bitcoin 온체인



be-BOP을 사용하면 Blockchain(On-Chain)에서 간단하고 안전하게 Bitcoin 결제를 직접 수락할 수 있습니다.



**구성 단계:**





- 결제 설정** 메뉴로 이동합니다
- On-Chain 결제 매개변수에 액세스하려면 **Bitcoin 노드리스**를 클릭합니다.
- 다음 입력란을 작성합니다:



| Champ                  | Description                                               | Exemple à utiliser                              |
|------------------------|-----------------------------------------------------------|--------------------------------------------------|
| **BIP Standard**       | Le type d’adressage utilisé                               | BIP84 (pour les adresses au format bech32 commençant par `bc1`) |
| **Clé publique étendue** | Votre Zpub (ou Xpub selon le portefeuille utilisé)        | `zpub...` (extrait de votre portefeuille Bitcoin) |
| **Derivation Index**   | L’index de départ pour la génération des adresses         | `1`                                              |
| **Mempool URL**        | L’URL du service mempool utilisé pour suivre les transactions | `https://mempool.space`                         |

![payment-nodeless](assets/fr/005.webp)



**팁: **확장 공개키(Zpub)를 받으려면 Bitcoin Wallet(Sparrow wallet, BlueWallet, Specter 등)의 고급 설정을 참조하세요. 거래 내역을 사용하려면 Wallet이 **읽기 전용이 아닌지** 확인하세요.



### Lightning Network



be-BOP은 Lightning Network를 통해 즉시 Bitcoin 결제도 받을 수 있습니다. 현재 두 가지 구성 옵션을 사용할 수 있습니다:



**Phoenixd**



결제 설정` 메뉴로 이동하여 `Phoenixd`를 클릭합니다



![phoenixd](assets/fr/006.webp)



그런 다음 결제 채널을 복잡하게 관리할 필요 없이 자체 노드로 라이트닝 결제를 관리할 수 있도록 Acinq에서 개발한 백엔드인 Phoenixd 인스턴스에 연결하기 위해 **비밀번호 또는 token 인증**을 입력해야 합니다.



**스위스 Bitcoin Pay**



라이트닝 노드를 직접 관리하고 싶지 않다면, **스위스 Bitcoin Pay**는 복잡한 인프라 없이 라이트닝 결제를 받기 시작하는 데 이상적인 바로 사용 가능하고 구성하기 쉬운 솔루션입니다.



구성 단계:





- '결제 설정' 메뉴에서 '스위스 Bitcoin 페이'를 클릭합니다
- 스위스 Bitcoin Pay 계정에 로그인합니다(아직 계정이 없는 경우 생성하세요).
- 스위스 Bitcoin Pay에서 제공한 API 키를 입력한 후 "저장"을 클릭합니다



설정이 완료되면 be-BOP은 고객에게 자동으로 generate Lightning 인보이스를 발행하고, 스위스 Bitcoin Pay 계정으로 직접 결제를 받게 됩니다. 이 솔루션은 개인 노드의 기술적 복잡성을 피하면서 빠르고 저렴한 비용으로 결제를 받고자 하는 사용자에게 이상적입니다.



![swissbtcpay](assets/fr/007.webp)



### PayPal



Bitcoin 외에도 be-BOP을 사용하면 잘 알려져 있고 널리 사용되는 국제 솔루션인 PayPal을 통해 현금 결제를 받을 수도 있습니다.



구성 단계:





- 결제 설정` 메뉴로 이동합니다
- '페이팔'을 클릭합니다
- Paypal 계정(개발자 섹션)에서 '고객 ID'와 '비밀 번호'를 입력합니다
- 원하는 통화를 선택합니다(예: **USD**, **EUR**, **XOF** 등)
- '저장'을 클릭합니다



![paypal](assets/fr/008.webp)



**참고:** 이 식별자를 generate으로 변경하려면 PayPal 비즈니스 계정이 있어야 합니다. 개발자] 포털(https://developer.paypal.com)을 통해 얻을 수 있습니다



### 요약



이제 소프트웨어에 **SumUp** 결제 솔루션이 통합되어 간편하고 안전하며 효율적으로 신용 카드 결제를 수락할 수 있습니다. 이 기능을 활용하려면 초기 구성이 필요합니다. 다음은 명확하고 점진적으로 구현할 수 있도록 번호가 매겨진 단계입니다:





- 개발자 계정을 만들 때 SumUp에서 제공한 기밀 키인 **API 키**를 입력하는 것으로 시작하세요. 이 키는 SumUp 계정과 소프트웨어 사이에 보안 연결을 설정합니다.
- '판매자 코드' 필드에 SumUp 플랫폼 내에서 비즈니스를 식별하는 고유 코드를 입력합니다. 이 코드는 거래를 비즈니스와 연결하는 데 필수적입니다.
- '통화' 필드에서 거래에 사용하는 기본 통화를 선택합니다(예: **EUR**, **USD**, **CDF** 등).
- 모든 필드를 올바르게 입력했으면 '저장' 버튼을 클릭하여 설정을 저장합니다. 그러면 시스템이 SumUp 계정과의 링크를 설정하고 소프트웨어가 결제를 수락할 준비가 됩니다.



![payment-sumup](assets/fr/009.webp)



이 구성이 완료되면 **SumUp** 통합이 활성화되어 작동하므로 소프트웨어에서 직접 거래를 빠르게 현금화하고 추적할 수 있습니다.



### 스트라이프



be-BOP은 가장 인기 있는 온라인 결제 플랫폼 중 하나인 **Stripe**와도 완벽하게 통합됩니다. Stripe를 사용하면 신용카드, 디지털 Wallet 및 기타 여러 결제 방법을 통해 온라인 결제를 수락할 수 있습니다. 활성화하는 방법은 다음과 같습니다:





- Stripe 대시보드에 제공된 **비밀키**를 입력합니다.
- Stripe에서 제공하는 **공개 키** 필드를 입력합니다.
- 주요 통화**를 선택합니다.
- 구성을 저장한 다음 '저장'을 클릭합니다.



![payment-stripe](assets/fr/010.webp)



⚠️ **주의 사항: **be-BOP**에서 인보이스 발행 옵션을 올바르게 구성하려면 해당 활동에 적용되는 부가가치세 제도(예: 판매자 국가의 부가가치세 적용 판매, 정당한 사유에 따른 면제, 구매자 국가의 부가가치세율로 판매)를 파악하는 것이 필수적입니다.



## 통화 구성



**be-BOP**는 고급 통화 관리 기능을 제공하며 다중 통화 환경과 특정 회계 요구사항에 맞게 조정됩니다. 재무 운영 및 보고의 일관성을 보장하려면 시스템에서 사용되는 다양한 통화를 적절히 구성하는 것이 중요합니다. 이 구성을 위해 따라야 할 단계는 다음과 같습니다:





- 주 통화**(`주 통화`)를 선택합니다
- '보조 통화'를 선택합니다
- 기준 통화** 정의(`가격 기준 통화`)
- '회계 통화'를 표시합니다



모든 통화가 올바르게 구성되면 소프트웨어는 엄격한 회계 일관성을 유지하면서 다중 통화 거래의 자동 변환을 정확하게 보장합니다.



![settings-currencies](assets/fr/011.webp)

## Be-BOP의 Interface 및 신원 구성



**be-BOP**는 디자이너에게 웹사이트 디자인에 필요한 모든 도구를 제공합니다. 첫 번째 단계는 설정에서 `/관리자 > 상품 > 레이아웃` 섹션을 여는 것입니다. 톱바**, 내비바**, 푸터**를 구성하는 것으로 시작하세요.



### 르 탑 바



Top Bar** 구성을 사용하면 Interface의 첫 줄부터 주요 정보를 표시하여 소프트웨어의 시각적 아이덴티티를 맞춤화할 수 있습니다. 이를 통해 브랜드 인지도를 강화하고 사용자에게 명확한 맥락을 제공합니다.



#### 구성 단계:





- '브랜드 이름' 필드에 회사, 조직 또는 제품의 이름을 입력합니다. 이 이름은 Interface 상단에 표시되며 주요 시각적 아이덴티티를 나타냅니다.
- 웹사이트 제목** 표시: 선택한 제목은 플랫폼의 목적을 요약해야 합니다. 이 제목은 헤더 또는 브라우저 탭에 표시할 수 있습니다.
- 웹사이트 설명 추가**: 이니셔티브에 대한 간단한 설명을 입력하는 곳입니다. 이 설명은 사용자를 위해 도구의 맥락을 파악하는 데 도움이 되며 SEO 목적으로도 사용할 수 있습니다.



이 정보를 입력하면 **상단 표시줄**에 명확하고 전문적이며 일관성 있는 솔루션 프레젠테이션이 표시됩니다.



#### 상단 표시줄의 링크



상단 표시줄의 '링크' 섹션에서는 애플리케이션 또는 외부 사이트의 중요한 페이지에 대한 바로가기를 추가할 수 있습니다. 이러한 링크는 상단 표시줄에 바로 표시되므로 사용자가 빠르고 체계적으로 액세스할 수 있습니다.



#### 구성 단계:





- 링크 이름 입력(텍스트)**: '텍스트' 필드에 표시될 링크의 이름 또는 레이블을 그대로 입력합니다(예: 홈, 연락처, 도움말...).
- 링크 Address(URL)** 표시: 'URL' 필드에 대상 페이지(내부 또는 외부)의 전체 Address를 입력합니다.
- 필요한 경우 다른 링크 추가**: 각 구성 줄에서 '텍스트' 및 'URL' 필드를 사용하여 추가 링크를 추가할 수 있습니다.
- 링크 저장**: 모든 링크를 입력한 후 '상단 표시줄 링크 추가' 버튼을 클릭하여 저장합니다.



이 구성을 사용하면 웹사이트의 여러 섹션 또는 보완 리소스를 통해 명확하고 유동적이며 접근하기 쉬운 탐색 기능을 제공할 수 있습니다.



![settings-topbar](assets/fr/014.webp)



### La Nav Bar



Navbar** 섹션에서는 일반적으로 Interface의 측면 또는 상단에 있는 be-BOP의 기본 탐색 메뉴를 구성할 수 있습니다. 이 메뉴는 애플리케이션의 다양한 페이지와 기능으로 사용자를 안내합니다. 링크 구성은 간단하고 직관적입니다. 작동 방식은 다음과 같습니다:





- 링크 이름 입력(`텍스트`): 구성 줄에서 `텍스트` 필드를 입력합니다. 이는 탐색 모음에 표시되는 링크의 이름에 해당합니다(예: *대시보드*, *사용자*, *설정*...).
- 링크 Address(`Url`) 입력: `Text` 필드 옆에 `Url` 필드가 있습니다. 이 필드에 링크가 리디렉션할 페이지의 Address를 입력합니다. 내부 경로 또는 외부 페이지로 연결되는 링크일 수 있습니다.
- 필요한 경우 여러 링크 추가**: 첫 번째 줄 아래에 새로운 '텍스트' 및 'URL' 필드를 추가하여 필요한 만큼의 링크를 추가할 수 있습니다. 각 줄은 추가 탐색 링크를 나타냅니다.
- 링크 저장**: Elements를 모두 입력한 후 '탐색 모음 링크 추가' 버튼을 클릭하여 결과를 저장하고 탐색 모음에 표시합니다.



이 구성을 통해 소프트웨어의 여러 부분에 대한 액세스를 효율적으로 구조화하여 인체공학 및 사용자 경험을 개선할 수 있습니다.



![navbar](assets/fr/015.webp)



### 바닥글



바닥글** 섹션에서는 소프트웨어의 바닥글을 사용자 지정하여 유용한 정보나 링크를 추가할 수 있습니다. 링크를 구성하기 전에 특정 옵션을 활성화하는 것으로 시작하세요:





- "Powered by be-BOP" 라벨 표시 활성화: 바닥글에 이 라벨을 표시하려면 'Powered by be-BOP 표시' 버튼을 활성화합니다.
- 링크 이름 입력(**텍스트**): **텍스트** 필드에 바닥글의 링크 문구에 해당하는 문구를 입력합니다(예: *약관*, *개인정보*, *연락처*...).
- 링크 Address(`Url`) 표시: `Url` 필드에 대상 페이지(내부 또는 외부)의 Address을 입력합니다.
- 필요한 경우 링크 추가**: 추가 줄을 사용하여 원하는 만큼 링크를 만들 수 있습니다.
- 링크 저장**: '푸터 링크 추가' 버튼을 클릭하여 링크를 저장합니다.



![footer](assets/fr/016.webp)

## 제품 관리



### 새 제품 만들기



'관리자 > 판매자 > 제품'으로 이동하여 제품을 추가하거나 수정합니다. 다음 필드를 입력합니다:



#### 기본 정보





- 제품 이름**: 제품 이름(예: *BOP 티셔츠 한정판*).
- 슬러그**: 공백이 없는 URL 식별자(예: `셔츠-밥-에디션-리미티`).
- 별칭** *(선택 사항)*: 전용 필드를 통해 장바구니에 빠르게 추가할 때 유용합니다.



![product-config](assets/fr/028.webp)



#### 가격 책정





- 가격 금액**: 제품 가격(예: `25.00`).
- 가격 통화**: 통화(EUR, USD, BTC 등).
- 특별 상품**:
  - 이 제품은 무료 제품입니다.
  - 이 제품은 종량제 상품입니다.



#### 제품 옵션





- 단일 상품(`독립형`)**: 주문당 하나의 상품(예: 기부금, 입장권)만 추가할 수 있습니다.
- 변형이 있는 제품**:
  - 독립형`에 체크하지 마세요.
  - '제품에 가벼운 변형이 있음(재고 차이 없음)`을 확인합니다.
  - 추가:
- 이름**(예: *사이즈*),
- 값**(예: S, M, L, XL),
- 해당되는 경우 가격 차이**(예: XL의 경우 '+2 USD').

## CMS 페이지 및 위젯의 생성 및 사용자 지정



### 필수 CMS 페이지



'관리자 > 상품 > CMS'로 이동합니다. 기존 페이지 목록이 표시되며 **CMS 페이지 추가**로 새 페이지를 추가할 수 있습니다.



CMS 페이지는 다음과 같은 경우에 중요합니다:





- 방문자에게 정보 제공(예: 이용 약관)
- 법률 준수(예: 개인정보처리방침)
- 특정 스토어 기능 설명(예: IP 수집, VAT 0%)



필요에 따라 다른 페이지를 추가할 수 있습니다:





- 회사 소개 / 회사 소개
- 지원 / 기부
- 자주 묻는 질문
- 연락처
- 등



**팁: 각 링크 또는 아이콘을 클릭하면 각 페이지의 콘텐츠, 제목 또는 검색어 노출을 수정할 수 있습니다**



### 레이아웃 및 그래픽 Elements



이동합니다: '관리자 > 상품 > 레이아웃'으로 이동합니다. 사이트의 시각적 Elements을 사용자 지정할 수 있습니다:



![product-options](assets/fr/032.webp)



#### 상단 바





- 링크 수정 또는 삭제(예: 홈, 회사 소개,...)
- 사이트의 주요 섹션 간 탐색



#### Navbar(기본 탐색 모음)





- 상단 표시줄 아래의 회색 영역에 표시됩니다
- 빠른 액세스가 포함되어 있습니다: 구성`, `결제 설정`, `거래`, `노드 관리`, `위젯` 등에 빠르게 액세스할 수 있습니다.
- 디렉터 전용



#### 푸터





- '관리자 > 상품 > 레이아웃'에서 편집 가능
- 포함 내용: 연락처 정보, 유용한 링크, 법적 고지.



#### 비주얼 사용자 지정



이동합니다: '관리자 > 상품 > 사진'



할 수 있습니다:





- 메인 로고** 변경
- 레이아웃 수정 또는 추가 **이미지**



#### 사이트 설명



'사진'에서도 수정할 수 있으며, 테마에 따라 머리글이나 바닥글에 **요약 또는 슬로건**을 표시할 수 있습니다.



**참고: 교육, 상업 또는 커뮤니티 등 브랜드 아이덴티티에 맞게 외관을 조정할 수 있습니다.



### 위젯을 CMS 페이지에 통합하기



**위젯**은 동적 또는 시각적 Elements로 CMS 페이지를 풍부하게 만들어 줍니다.



#### 위젯 생성



이동합니다: '관리자 > 위젯'



사용 가능한 위젯의 예입니다:





- 챌린지**: 도전 과제 또는 미션
- 태그**: 카테고리 또는 키워드
- 슬라이더**: 이미지 캐러셀
- 사양**: 사양 표
- 양식**: 양식(연락처, 피드백 등)
- 카운트다운**: 타이머
- 갤러리**: 이미지 갤러리
- 리더보드**: 사용자 순위



![widgets](assets/fr/033.webp)



#### CMS 페이지에 통합



CMS 페이지 콘텐츠에 **숏코드**를 사용하세요:



| Objectif                 | Balise à insérer                      |
|--------------------------|---------------------------------------|
| Afficher un produit      | `[Product=slug?display=img-1]`        |
| Afficher une image       | `[Picture=slug width=100 height=100 fit=contain]` |
| Intégrer un slider       | `[Slider=slug?autoplay=3000]`         |
| Ajouter un challenge     | `[Challenge=slug]`                    |
| Ajouter un compte à rebours | `[Countdown=slug]`                 |
| Intégrer un formulaire   | `[Form=slug]`                         |

**현재 매개변수**:





- slug`: 고유 위젯 식별자
- display=img-1`: 제품별 이미지
- 너비`, `높이`, `맞춤`: 이미지 크기 및 스타일

**장점**:





- 간편한 삽입(복사 및 붙여넣기)
- 동적: 위젯에 대한 모든 수정 사항이 자동으로 반영됩니다
- 개발자 필요 없음

## POS(판매 시점) 구성



### POS 역할 할당



사용자에게 POS에 대한 액세스 권한을 부여하려면 `POS(Point of Sale)`에서 역할을 할당합니다: '관리자 > 구성 > ARM'



보안 URL을 통해 연결할 수 있습니다: `/pos` 또는 `/pos/touch`를 통해 연결할 수 있습니다



### POS 관련 기능



Be-BOP은 오프라인 판매(매장, 이벤트 등) 전용 Interface를 제공합니다.



#### 별칭을 통한 빠른 추가



장바구니`에서 제품을 추가할 수 있는 필드입니다:





- 바코드**(ISBN, EAN13)를 스캔합니다
- 제품 별칭**을 수동으로 입력하는 방법



**참고: **상품이 장바구니에 자동으로 추가됩니다.



#### 결제 수단



POS 지원:





- 종
- 신용 카드
- Lightning Network(암호화)
- 기타 구성에 따른 기타



두 가지 고급 옵션을 사용할 수 있습니다:





- 부가가치세 면제**: 정당한 사유가 있는 경우 적용 가능(NGO, 외국인 등)
- 선물 할인**: 필수 댓글을 통한 파격 할인



#### 클라이언트 측 디스플레이



URL `/pos/session`은 **보조 화면**(HDMI, 태블릿...)을 위한 것입니다:



포스터:





- 진행 중인 제품
- 총 금액
- 결제 방법
- 할인 적용



**참고: 고객은 주문을 실시간으로 팔로우하고 판매자는 `/pos`에 주문을 기록합니다.
