**be-BOP**是一个电子商务平台，专为希望在线和离线销售的企业家设计，完全自主，同时接受Bitcoin、银行账户和现金支付。该解决方案还适用于任何希望募集捐款或将其各种活动货币化的组织。



该解决方案简单、轻便、自主。即使在传统金融服务有限或不存在的环境中，也能创建网上商店。事实上，**be-BOP**的设计目的是利用Bitcoin作为支付基础设施，在有银行或没有银行的情况下都能高效运作。



在本教程中，我们将带您逐步学习 .NET 技术：





- 通过**be-BOP**创建您的第一家网店
- 个性化您的展示架和产品
- 配置可用的付款方式
- 通过**be-BOP**了解有效在线销售的最佳做法



本教程不需要高级技术技能。它面向开发人员以及希望以主权和弹性方式开展数字商务的工匠、商人、合作社或企业家。



## 在自己的服务器上安装 be-BOP 的前提条件



在开始安装 be-BOP 之前，请确保您具备以下技术基础设施。这些 Elements 对平台的正常运行至关重要：



### 与 S3 兼容的存储



be-BOP 使用存储系统管理文件（如产品图片）。这需要访问 S3 服务，如 .NET Framework 3.0：





- [MinIO](https://min.io/)自托管
- 亚马逊 S3（AWS）
- Scaleway 对象存储



您需要配置一个存储桶并提供以下信息：





- S3_BUCKET**：水桶名称
- S3_ENDPOINT_URL**：S3 服务的访问链接
- S3_KEY_ID** 和 S3_KEY_SECRET：您的访问密码
- S3_REGION**：S3 服务的区域



### 复制集模式下的 MongoDB 数据库



be-BOP 使用 MongoDB 来存储商店、用户、产品和其他数据。



您有两个选择：





- 在本地安装 MongoDB，并启用 **ReplicaSet 模式**
- 使用在线服务，如 **MongoDB Atlas**



您将需要以下变量：





- MONGODB_URL**：数据库连接 Address
- MONGODB_DB**：数据库名称

## 通信设置



要使 be-BOP 能够发送通知（如订单、注册或系统消息），您需要配置至少一种通信方式。有两种方式可供选择：电子邮件（SMTP）或 Nostr。



### SMTP 配置（电子邮件）



be-BOP 可以通过 SMTP 服务器发送电子邮件。您需要有效的 SMTP 认证，通常由电子邮件服务（如 Mailgun、Gmail 等）提供。



以下是您需要了解的信息：



SMTP_HOST：SMTP 服务器 Address（如 smtp.mailgun.org）。



SMTP_PORT：要使用的端口（通常为 587 或 465）



SMTP_USER：您的用户名（通常是电子邮件 Address）。



SMTP_PASSWORD：您的密码或 API 密钥



SMTP_FROM：将显示为发件人的电子邮件 Address




### Nostr 配置



be-BOP 可让您通过 Nostr 协议（一种去中心化的信息基础设施）发送通知。为此，您需要 generate 或 Supply 一个 Nostr 私钥（NSEC）。您可以通过 be-BOP 的 Interface 直接获取 generate 密钥，请参见 Nostr 专节。正确配置这些 Elements 后，be-BOP 将能够自动向用户发送消息和警报。



## 兼容的付款方式



be-BOP 兼容多种支付解决方案，让您为客户提供更大的灵活性。以下是您需要设置的最适合您的付款方式。



### Bitcoin Onchain



be-BOP 可让您直接在 Blockchain (On-Chain) 上接受 Bitcoin 付款，简单而安全。



**配置步骤：**





- 进入**支付设置**菜单
- 点击 **Bitcoin Nodeless** 访问 On-Chain 支付参数。
- 填写以下字段：



| Champ                  | Description                                               | Exemple à utiliser                              |
|------------------------|-----------------------------------------------------------|--------------------------------------------------|
| **BIP Standard**       | Le type d’adressage utilisé                               | BIP84 (pour les adresses au format bech32 commençant par `bc1`) |
| **Clé publique étendue** | Votre Zpub (ou Xpub selon le portefeuille utilisé)        | `zpub...` (extrait de votre portefeuille Bitcoin) |
| **Derivation Index**   | L’index de départ pour la génération des adresses         | `1`                                              |
| **Mempool URL**        | L’URL du service mempool utilisé pour suivre les transactions | `https://mempool.space`                         |

![payment-nodeless](assets/fr/005.webp)



**提示：** 要获取扩展公钥 (Zpub)，可查阅 Bitcoin Wallet 的高级设置（Sparrow wallet、BlueWallet、Specter 等）。如果您打算使用交易历史记录，请确保您的 Wallet 为**非只读**。



### Lightning Network



be-BOP 还可以通过 Lightning Network 接受 Bitcoin 即时付款。目前有两种配置可供选择：



**Phoenixd**



进入 "支付设置 "菜单，点击 "Phoenixd



![phoenixd](assets/fr/006.webp)



然后，您需要输入**密码或 token 身份验证**，以连接到您的 Phoenixd 实例。Phoenixd 是 Acinq 开发的后端，可让您使用自己的节点管理闪电支付，但无需管理复杂的支付渠道。



**瑞士 Bitcoin 薪酬**



如果您不想自己管理闪电节点，**Swiss Bitcoin Pay**是一个即开即用、易于配置的解决方案，非常适合在没有复杂基础设施的情况下开始接受闪电支付。



配置步骤 ：





- 在 "付款设置 "菜单中，点击 "瑞士 Bitcoin Pay
- 登录您的瑞士 Bitcoin Pay 账户（如果还没有账户，请创建一个）。
- 输入 Swiss Bitcoin Pay 提供的 API 密钥，然后点击 "保存"。



设置完成后，be-BOP 将自动为您的客户开具 generate Lightning 发票，您的瑞士 Bitcoin Pay 账户将直接收到付款。该解决方案非常适合那些既想避免个人节点的技术复杂性，又想接受快速、低成本付款的用户。



![swissbtcpay](assets/fr/007.webp)

该软件现在集成了**SumUp**支付解决方案，使您能够简单、安全、高效地接受信用卡支付。要使用这一功能，需要进行初始配置。以下是需要遵循的步骤，通过编号可清晰、逐步地实施：

## be-BOP中的Interface和身份配置



**be-BOP**为设计师提供设计网站所需的所有工具。第一步是打开设置中的 `/Admin > Merch > Layout` 部分。首先配置**顶栏**、**导航栏**和**页脚**。



### Le Top 酒吧



顶栏***配置可让您在 Interface 的第一行显示关键信息，从而个性化软件的视觉标识。这可以加强品牌识别，并为用户提供清晰的上下文。



#### 配置步骤 ：





- 在 "品牌名称 "字段中，输入贵公司、组织或产品的名称。该名称将出现在 Interface 的顶部，代表您的主要视觉形象。
- 标明网站标题**：所选标题应概括平台的目的。该标题可出现在页眉或浏览器标签中。
- 添加网站描述**：在此输入对您的倡议的简要描述。该描述有助于为用户提供工具的背景信息，也可用于搜索引擎优化。



输入这些信息后，**顶栏**将清晰、专业、连贯地展示您的解决方案。



#### 顶部栏中的链接



顶栏的 "链接 "部分可让您为应用程序或外部网站的重要页面添加快捷方式。这些链接将直接显示在顶栏中，为用户提供快速、有序的访问。



#### 配置步骤 ：





- 输入链接名称（文本）**：在 "文本 "字段中，输入链接的名称或标签（如主页、联系人、帮助......）。
- 指示链接 Address (Url)**：在 "Url "字段中，输入目标页面（内部或外部）的完整 Address。
- 必要时添加其他链接**：每一行配置都允许您使用`Text`和`Url`字段添加其他链接。
- 保存链接**：输入所有链接后，点击 "添加顶部栏链接 "按钮保存链接。



通过这种配置，您可以在网站的不同部分或补充资源中提供清晰、流畅和无障碍的导航。



![settings-topbar](assets/fr/014.webp)



### La Nav 酒吧



导航栏***部分可让您配置 be-BOP 的主导航菜单，通常位于 Interface 的侧面或顶部。该菜单可引导用户访问应用程序的各个页面和功能。链接配置简单直观。具体操作如下





- 输入链接名称 (`Text`)**：在配置行中，首先填写`Text`字段。这相当于导航栏中显示的链接名称（例如：*仪表板*、*用户*、*设置*......）。
- 输入链接 Address (`Url`)**：在`文本`字段旁边，您会发现`Url`字段。在该字段中，输入链接应重定向到的页面的 Address。这可以是内部路径，也可以是指向外部页面的链接。
- 根据需要添加多个链接**：第一行下方有新的 `Text` 和 `Url` 字段，可根据需要添加多个链接。每一行代表一个额外的导航链接。
- 保存链接**：输入所有 Elements 后，点击 "添加导航栏链接 "按钮保存并在导航栏中显示结果。



通过这种配置，可以有效地结构化访问软件的不同部分，改善人体工程学和用户体验。

- 根据需要添加更多链接**：使用附加行创建任意数量的链接。
- 保存链接**：点击 "添加页脚链接 "按钮保存链接。
