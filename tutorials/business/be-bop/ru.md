**be-BOP** - это платформа электронной коммерции, предназначенная для предпринимателей, желающих продавать товары онлайн и офлайн, в полной автономии, принимая при этом платежи в Bitcoin, через банковский счет и наличными. Решение также полезно для организаций любого типа, желающих собирать пожертвования или монетизировать свою разнообразную деятельность.



Решение простое, легкое и автономное. Оно позволяет создать интернет-магазин даже в условиях, когда традиционные финансовые услуги ограничены или отсутствуют. Действительно, **be-BOP** был разработан для эффективной работы как в условиях отсутствия доступа к банкам, так и без него, используя Bitcoin в качестве платежной инфраструктуры.



В этом уроке мы шаг за шагом рассмотрим:





- Создайте свой первый интернет-магазин с помощью **be-BOP**
- Персонализируйте свою витрину и свои продукты
- Настройте доступные способы оплаты
- Поймите лучшие практики эффективных продаж в Интернете с помощью **be-BOP**



Этот учебник не требует развитых технических навыков. Оно предназначено как для разработчиков, так и для ремесленников, торговцев, кооперативов и предпринимателей, желающих приступить к цифровой торговле на суверенной и устойчивой основе.



## Необходимые условия для установки be-BOP на вашем собственном сервере



Прежде чем приступить к установке be-BOP, убедитесь, что у вас есть следующая техническая инфраструктура. Эти Elements необходимы для правильной работы платформы:



### S3-совместимое хранилище



be-BOP использует систему хранения для управления файлами (например, изображениями продуктов). Для этого требуется доступ к сервису S3, например:





- [MinIO](https://min.io/) self-hosted
- Amazon S3 (AWS)
- Объектное хранилище Scaleway



Вам нужно будет настроить ведро и предоставить следующую информацию:





- S3_BUCKET**: имя ведра
- S3_ENDPOINT_URL**: ссылка доступа к вашей службе S3
- S3_KEY_ID** и S3_KEY_SECRET: ваши коды доступа
- S3_REGION**: регион вашего сервиса S3



### База данных MongoDB в режиме ReplicaSet



be-BOP использует MongoDB для хранения данных о магазине, пользователях, продуктах и других данных.



У вас есть два варианта:





- Установите MongoDB локально с включенным режимом **ReplicaSet**
- Используйте онлайн-сервис, например **MongoDB Atlas**



Вам понадобятся следующие переменные:





- MONGODB_URL**: подключение к базе данных Address
- MONGODB_DB**: имя базы данных



## Среда Node.js



be-BOP работает с Node.js. Убедитесь, что у вас есть **Node.js** версии 18 или выше и включен **Corepack** (необходим для управления менеджерами пакетов, такими как pnpm). Команда для запуска - `corepack enable`



### Установлен Git LFS



Некоторые ресурсы (например, большие изображения) управляются с помощью Git LFS (Large File Storage). Убедитесь, что Git LFS установлен на вашей машине с помощью команды `git lfs install`. Как только все эти предварительные условия будут выполнены, вы будете готовы перейти к следующему шагу: загрузке и настройке be-BOP.



**Примечание:** Техническое руководство по развертыванию программного обеспечения доступно в отдельном учебнике.

- Перейдите по адресу `ваш сайт web/admin/login`
- Создайте учетную запись супер-админа с надежным логином и паролем



Эта учетная запись даст вам доступ ко всем функциям бэк-офиса. После создания вы сможете войти в систему, введя имя пользователя и пароль.

## Конфигурация и безопасность бэк-офиса



Прежде чем настраивать подключение к бэк-офису Interface, необходимо создать уникальный Hash. Это обеспечит защиту от злоумышленников, пытающихся украсть ссылку на подключение к вашему администратору Interface.



Чтобы создать Hash, перейдите в раздел `/admin/Settings`. В разделе, посвященном безопасности (например, "Admin Hash"), задайте уникальную строку (Hash). После регистрации URL-адрес бэк-офиса будет изменен (например, `/admin-yourhash/login`), чтобы ограничить доступ для неавторизованных лиц.

## Настройка связи



Чтобы be-BOP мог отправлять уведомления (например, о заказах, регистрациях или системных сообщениях), необходимо настроить хотя бы один способ связи. Доступны два варианта: электронная почта (SMTP) или Nostr.



### Настройка SMTP (электронная почта)



be-BOP может отправлять электронные письма через SMTP-сервер. Вам понадобятся действительные учетные данные SMTP, которые часто предоставляются почтовым сервисом (например, Mailgun, Gmail и т.д.).



Вот что вам нужно знать:



SMTP_HOST: SMTP-сервер Address (например, smtp.mailgun.org)



SMTP_PORT: используемый порт (часто 587 или 465)



SMTP_USER: ваше имя пользователя (обычно это e-mail Address)



SMTP_PASSWORD: ваш пароль или ключ API



SMTP_FROM: электронная почта Address, которая будет отображаться в качестве отправителя




### Конфигурация Nostr



be-BOP позволяет отправлять уведомления по протоколу Nostr, децентрализованной инфраструктуре обмена сообщениями. Для этого вам необходимо получить generate или Supply закрытый ключ Nostr (NSEC). Вы можете generate этот ключ непосредственно через be-BOP's Interface, в разделе, посвященном Nostr. Когда эти Elements правильно настроены, be-BOP сможет автоматически отправлять сообщения и оповещения вашим пользователям.



## Совместимые способы оплаты



be-BOP совместим с несколькими платежными решениями, что позволяет вам предлагать своим клиентам большую гибкость. Вот что вам нужно, чтобы установить способ оплаты, который подходит вам лучше всего.



### Bitcoin Onchain



be-BOP позволяет принимать платежи Bitcoin непосредственно на Blockchain (On-Chain), просто и безопасно.



**Шаги конфигурации:**





- Перейдите в меню **Настройки платежей**
- Нажмите на **Bitcoin Nodeless**, чтобы получить доступ к параметрам оплаты On-Chain.
- Заполните следующие поля:



| Champ                  | Description                                               | Exemple à utiliser                              |
|------------------------|-----------------------------------------------------------|--------------------------------------------------|
| **BIP Standard**       | Le type d’adressage utilisé                               | BIP84 (pour les adresses au format bech32 commençant par `bc1`) |
| **Clé publique étendue** | Votre Zpub (ou Xpub selon le portefeuille utilisé)        | `zpub...` (extrait de votre portefeuille Bitcoin) |
| **Derivation Index**   | L’index de départ pour la génération des adresses         | `1`                                              |
| **Mempool URL**        | L’URL du service mempool utilisé pour suivre les transactions | `https://mempool.space`                         |

![payment-nodeless](assets/fr/005.webp)



**Совет:** Чтобы получить расширенный открытый ключ (Zpub), вы можете обратиться к расширенным настройкам вашего Bitcoin Wallet (Sparrow wallet, BlueWallet, Specter и т.д.). Убедитесь, что ваш Wallet **не доступен только для чтения**, если вы собираетесь использовать историю транзакций.



### Lightning Network



be-BOP также может принимать мгновенные платежи Bitcoin благодаря Lightning Network. В настоящее время доступны два варианта конфигурации:



**Phoenixd**



Перейдите в меню `Настройки платежей`, нажмите на `Phoenixd`



![phoenixd](assets/fr/006.webp)



Затем вам нужно будет ввести **пароль или аутентификацию token**, которая соединит вас с вашим экземпляром Phoenixd - бэкэндом, разработанным компанией Acinq, который позволяет управлять платежами Lightning с помощью собственного узла, но без сложностей управления платежными каналами.



**Швейцария Bitcoin Pay**



Если вы не хотите самостоятельно управлять узлом Lightning, **Swiss Bitcoin Pay** - это готовое к использованию, простое в настройке решение, которое идеально подходит для начала приема платежей Lightning без сложной инфраструктуры.



Этапы конфигурирования:





- В меню "Настройки платежей" нажмите на `Swiss Bitcoin Pay`
- Войдите в свой швейцарский аккаунт Bitcoin Pay (или создайте его, если у вас его еще нет).
- Введите ключ API, предоставленный Swiss Bitcoin Pay, затем нажмите "Сохранить"



После настройки be-BOP будет автоматически выставлять счета generate Lightning для ваших клиентов, а вы будете получать платежи непосредственно на ваш швейцарский счет Bitcoin Pay. Это решение идеально подходит для пользователей, которые хотят избежать технической сложности личного узла, принимая при этом быстрые, недорогие платежи.



![swissbtcpay](assets/fr/007.webp)



### PayPal



Помимо Bitcoin, be-BOP также позволяет принимать наличные платежи через PayPal, известное и широко используемое международное решение.



Этапы конфигурирования:





- Перейдите в меню `Настройки платежей`
- Нажмите на `PayPal
- В своем аккаунте Paypal (раздел разработчика) введите `Client ID` и `Secret`
- Выберите валюту по своему усмотрению (например, **USD**, **EUR**, **XOF** и т.д.)
- Нажмите на кнопку "Сохранить

### SumUp



В программное обеспечение интегрировано платежное решение **SumUp**, позволяющее принимать платежи по кредитным картам просто, безопасно и эффективно. Чтобы воспользоваться этой функциональностью, необходима первоначальная настройка. Ниже приведены шаги, которые необходимо выполнить для четкого и постепенного внедрения:





- Начните с ввода **API Key** - конфиденциального ключа, предоставленного компанией SumUp при создании учетной записи разработчика. Он устанавливает безопасное соединение между вашей учетной записью SumUp и программным обеспечением.
- Заполните поле `Merchant Code` уникальным кодом, который идентифицирует ваш бизнес на платформе SumUp. Этот код необходим для привязки транзакций к вашему бизнесу.
- В поле `Валюта` выберите основную валюту, которую вы используете для своих транзакций (например, **EUR**, **USD**, **CDF** и т.д.).
- После того как все поля будут заполнены правильно, нажмите на кнопку `Save`, чтобы сохранить настройки. После этого система установит связь с вашей учетной записью SumUp, и ваше программное обеспечение будет готово к приему платежей.



![payment-sumup](assets/fr/009.webp)



После этой настройки интеграция с **SumUp** будет активной и рабочей, что позволит вам быстро обналичивать деньги и отслеживать свои операции прямо из программы.



### Полоска



be-BOP также предлагает полную интеграцию с **Stripe**, одной из самых популярных платформ для онлайн-платежей. Stripe позволяет принимать онлайн-платежи с помощью кредитных карт, цифровых Wallet и некоторых других способов оплаты. Вот как ее активировать:





- Введите **секретный ключ**, указанный на приборной панели Stripe.
- Заполните поле **Публичный ключ**, также предоставляемый компанией Stripe.
- Выберите **принципиальную валюту**.
- Сохраните конфигурацию, затем нажмите `Save`.



![payment-stripe](assets/fr/010.webp)

## Interface и конфигурация идентификации в be-BOP



**be-BOP** предоставляет дизайнерам все необходимые инструменты для создания сайта. Первым делом откройте в настройках раздел `/Admin > Merch > Layout`. Начните с настройки **Топ-бара**, **Навбара** и **Футера**.



### Бар Le Top



Конфигурация **Top Bar** позволяет персонализировать визуальный облик вашего программного обеспечения, отображая ключевую информацию прямо в первой строке Interface. Это повышает узнаваемость бренда и обеспечивает четкий контекст для пользователей.



#### Этапы конфигурирования:





- В поле `Название бренда` введите название вашей компании, организации или продукта. Это название будет отображаться в верхней части Interface и будет представлять собой ваш основной визуальный образ.
- Укажите название сайта**: выбранное название должно кратко отражать назначение платформы. Это название может отображаться в заголовке или на вкладке браузера.
- Добавить описание сайта**: здесь вы вводите краткое описание вашей инициативы. Это описание поможет пользователям понять контекст инструмента, а также может быть использовано в целях SEO.



После ввода этой информации на **Верхней панели** появится четкое, профессиональное и последовательное представление вашего решения.



#### Ссылки в верхней панели



Раздел `Ссылки` в верхней панели позволяет добавлять ярлыки на важные страницы в вашем приложении или на внешних сайтах. Эти ссылки отображаются непосредственно на верхней панели, обеспечивая пользователям быстрый и структурированный доступ.



#### Этапы конфигурирования:





- Введите имя ссылки (Текст)**: в поле `Текст` введите название или метку ссылки, как она будет отображаться (например, Главная, Контакт, Помощь...).
- Укажите ссылку Address (Url)**: в поле `Url` введите полный Address целевой страницы (внутренней или внешней).
- Добавьте другие ссылки при необходимости**: каждая строка конфигурации позволяет добавить дополнительную ссылку с помощью полей `Text` и `Url`.
- Сохранить ссылки**: когда все ссылки будут введены, нажмите кнопку "Добавить ссылку на верхнюю панель", чтобы сохранить их.



Такая конфигурация позволит вам предложить четкую, плавную и доступную навигацию по различным разделам вашего сайта или дополнительным ресурсам.

### Бар La Nav



Раздел **Navbar** позволяет настроить главное навигационное меню вашего be-BOP, обычно расположенное сбоку или сверху Interface. Это меню направляет пользователей к различным страницам и функциям приложения. Настройка ссылок проста и интуитивно понятна. Вот как это работает:





- Введите имя ссылки (`Текст`)**: в строке конфигурации начните с заполнения поля `Текст`. Оно соответствует названию ссылки, отображаемой в панели навигации (примеры: *Dashboard*, *Users*, *Settings*...).
- Введите Address ссылки (`Url`)**: рядом с полем `Текст` находится поле `Url`. В этом поле введите Address страницы, на которую должна перенаправлять ссылка. Это может быть внутренний маршрут или ссылка на внешнюю страницу.
- Добавьте несколько ссылок при необходимости**: под первой строкой находятся новые поля `Text` и `Url`, в которые можно добавить столько ссылок, сколько необходимо. Каждая строка представляет собой дополнительную навигационную ссылку.
- Сохранить ссылки**: когда вы введете все Elements, нажмите на кнопку `Добавить ссылку на панель навигации`, чтобы сохранить и отобразить результаты в панели навигации.



Такая конфигурация позволяет эффективно структурировать доступ к различным частям программного обеспечения, улучшая эргономику и удобство использования.

Перейдите в: `Админ > Виджеты`

- Таблицы лидеров**: рейтинги пользователей



![widgets](assets/fr/033.webp)



#### Интеграция в страницы CMS



Используйте **шорткоды** в содержимом страниц CMS:



| Objectif                 | Balise à insérer                      |
|--------------------------|---------------------------------------|
| Afficher un produit      | `[Product=slug?display=img-1]`        |
| Afficher une image       | `[Picture=slug width=100 height=100 fit=contain]` |
| Intégrer un slider       | `[Slider=slug?autoplay=3000]`         |
| Ajouter un challenge     | `[Challenge=slug]`                    |
| Ajouter un compte à rebours | `[Countdown=slug]`                 |
| Intégrer un formulaire   | `[Form=slug]`                         |

**Текущие параметры**:





- `slug`: уникальный идентификатор виджета
- `display=img-1`: изображение конкретного продукта
- `width`, `height`, `fit`: размеры и стиль изображения
- autoplay=3000`: время в мс между двумя слайдами



**Преимущества**:





- Легко вставлять (копировать и вставлять)
- Динамический: любое изменение виджета отражается автоматически
- Разработчик не требуется

## Nostr-bot



Платформа предлагает расширенную интеграцию с протоколом **Nostr** с помощью автоматизированного бота.



Перейдите в раздел: Управление узлами > Ностр



### Основные характеристики



#### Управление реле





- Добавить или удалить **реле**, используемые ботом
- Оптимизация **достижимости** и **надежности** отправляемых сообщений



#### Автоматическое вводное сообщение





- Активируйте автоматическое сообщение при **первом взаимодействии с пользователем**
- Идеально подходит для:
  - Представление вашей услуги
  - Отправьте полезную ссылку (например, FAQ, контакты, заказ)



#### Сертификация вашего `npub





- Добавьте **логотип** и **публичное имя**
- Ссылка на **проверенный веб-домен**
- Повышает доверие и узнаваемость вашей личности Nostr



### Варианты использования бота Nostr





- Отправляем вам **подтверждения заказов**
- Автоматическая реакция на **события (например, новый заказ)**
- Создание **децентрализованного взаимодействия с клиентами**

## Работа в команде и торговые точки (POS)



### Управление правами пользователей и доступа



#### Создание ролей



Перейдите в: `Админ > Настройки > АРМ`



Нажмите на **Создать роль**, чтобы создать роль (например, `Super Admin`, `POS`, `Ticket checker`).



Каждая роль содержит:





- доступ для записи**: доступ для записи
- доступ для чтения**: доступ для чтения
- запрещенный доступ**: секции интердитов

- (необязательно) `recovery npub` для подключения через Nostr



Назначьте ранее определенную роль.

Пользователи, доступные только для чтения**, будут видеть меню, набранное *жирным шрифтом*, и не смогут изменять его содержимое.



## Настройка точек продаж (POS)



### Назначение роли POS



Чтобы предоставить пользователю доступ к кассовому аппарату, назначьте ему роль `Точка продажи (POS)` в разделе: `Админ > Конфигурация > АРМ`



Он может подключиться по защищенному URL-адресу: `/pos` или `/pos/touch`



### Особенности POS-систем



Be-BOP предлагает Interface, посвященный физическим продажам (магазин, мероприятие и т.д.).



#### Быстрое добавление через псевдоним



В `/cart` есть поле, позволяющее добавить товар:





- Отсканировав **штрих-код** (ISBN, EAN13)
- Вводя **продуктовый псевдоним** вручную



**Примечание:** товар автоматически добавляется в корзину.



#### Средства платежа



POS поддерживает:





- Виды
- Кредитная карта
- Lightning Network (криптовалюта)
- Другие в зависимости от конфигурации



Доступны два дополнительных варианта:





- Освобождение от НДС**: применяется при обосновании (НПО, иностранцы...)
- Подарочная скидка**: исключительная скидка с обязательным комментарием



#### Отображение на стороне клиента



URL `/pos/session` предназначен для **вторичного экрана** (HDMI, планшет...):



Плакат:





- Продукция в процессе производства
- Общая сумма
- Способ оплаты
- Применяются скидки



**Примечание:** покупатель следит за выполнением заказа в реальном времени, а продавец записывает его в `/pos`.



### Краткое описание POS



| Fonction                         | Description                                             |
|----------------------------------|---------------------------------------------------------|

| Interface principale             | `/pos` ou `/pos/touch`                                 |

| Paiement                         | Espèces, carte, Lightning, etc.                         |
| Ajout produit                    | Alias ou scan code-barres                              |
