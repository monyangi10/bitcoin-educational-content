**be-BOP** เป็นแพลตฟอร์มอีคอมเมิร์ซที่ออกแบบมาสำหรับผู้ประกอบการที่ต้องการขายสินค้าออนไลน์และออฟไลน์อย่างอิสระ พร้อมทั้งรับชำระเงินในรูปแบบ Bitcoin ผ่านบัญชีธนาคารและเงินสด โซลูชันนี้ยังมีประโยชน์สำหรับองค์กรทุกประเภทที่ต้องการรวบรวมเงินบริจาคหรือสร้างรายได้จากกิจกรรมต่างๆ ของตน



โซลูชันนี้เรียบง่าย เบา และเป็นอิสระ มันช่วยให้การสร้างร้านค้าออนไลน์เป็นไปได้ แม้ในสภาพแวดล้อมที่บริการทางการเงินแบบดั้งเดิมมีจำกัดหรือไม่มีอยู่จริง จริงๆ แล้ว **be-BOP** ได้รับการออกแบบมาให้ทำงานได้อย่างมีประสิทธิภาพไม่ว่าจะมีหรือไม่มีการเข้าถึงธนาคาร โดยใช้ Bitcoin เป็นโครงสร้างพื้นฐานการชำระเงิน



ในบทแนะนำนี้ เราจะพาคุณไปทีละขั้นตอนผ่าน:





- สร้างร้านค้าออนไลน์แห่งแรกของคุณด้วย **be-BOP**
- ปรับแต่งการแสดงผลงานและผลิตภัณฑ์ของคุณให้เป็นส่วนตัว
- กำหนดวิธีการชำระเงินที่มีอยู่
- ทำความเข้าใจแนวทางปฏิบัติที่ดีที่สุดสำหรับการขายออนไลน์อย่างมีประสิทธิภาพด้วย **be-BOP**



บทแนะนำนี้ไม่จำเป็นต้องมีทักษะทางเทคนิคขั้นสูง มุ่งเป้าไปที่นักพัฒนา รวมถึงช่างฝีมือ พ่อค้า สหกรณ์ หรือผู้ประกอบการที่ต้องการเริ่มต้นการค้าดิจิทัลในรูปแบบที่มีอธิปไตยและยืดหยุ่น



## ข้อกำหนดเบื้องต้นสำหรับการติดตั้ง be-BOP บนเซิร์ฟเวอร์ของคุณเอง



ก่อนที่คุณจะเริ่มติดตั้ง be-BOP โปรดตรวจสอบให้แน่ใจว่าคุณมีโครงสร้างพื้นฐานทางเทคนิคดังต่อไปนี้ องค์ประกอบเหล่านี้มีความจำเป็นเพื่อให้แพลตฟอร์มทำงานได้อย่างถูกต้อง:



### ที่จัดเก็บข้อมูลที่เข้ากันได้กับ S3



be-BOP ใช้ระบบจัดเก็บข้อมูลเพื่อจัดการไฟล์ (เช่น รูปภาพสินค้า) ซึ่งต้องการการเข้าถึงบริการ S3 เช่น:





- [MinIO](https://min.io/) โฮสต์ด้วยตนเอง
- Amazon S3 (AWS)
- Scaleway Object Storage



คุณจะต้องกำหนดค่าบัคเก็ตและให้ข้อมูลต่อไปนี้:





- S3_BUCKET**: ชื่อบัคเก็ต
- S3_ENDPOINT_URL**: ลิงก์การเข้าถึงบริการ S3 ของคุณ
- S3_KEY_ID** และ **S3_KEY_SECRET**: รหัสการเข้าถึงของคุณ
- S3_REGION**: ภูมิภาคของบริการ S3 ของคุณ



### ฐานข้อมูล MongoDB ในโหมด ReplicaSet



be-BOP ใช้ MongoDB ในการจัดเก็บข้อมูลร้านค้า, ผู้ใช้, ผลิตภัณฑ์ และข้อมูลอื่น ๆ



คุณมีสองตัวเลือก:





- ติดตั้ง MongoDB ในเครื่องโดยเปิดใช้งาน **ReplicaSet mode**
- ใช้บริการออนไลน์เช่น **MongoDB Atlas**



คุณจะต้องใช้ตัวแปรต่อไปนี้:





- MONGODB_URL**: ที่อยู่การเชื่อมต่อฐานข้อมูล
- MONGODB_DB**: ชื่อฐานข้อมูล



## สภาพแวดล้อม Node.js



be-BOP ทำงานร่วมกับ Node.js ตรวจสอบให้แน่ใจว่าคุณมี **Node.js** เวอร์ชัน 18 หรือสูงกว่าและเปิดใช้งาน **Corepack** (จำเป็นสำหรับการจัดการแพ็คเกจเมเนเจอร์เช่น pnpm) คำสั่งที่ใช้คือ `corepack enable`



### Git LFS ติดตั้งแล้ว



ทรัพยากรบางอย่าง (เช่น รูปภาพขนาดใหญ่) ถูกจัดการผ่าน Git LFS (Large File Storage) ตรวจสอบให้แน่ใจว่าคุณได้ติดตั้ง Git LFS บนเครื่องของคุณด้วยคำสั่ง `git lfs install` เมื่อคุณเตรียมสิ่งที่จำเป็นเหล่านี้เรียบร้อยแล้ว คุณก็พร้อมที่จะไปยังขั้นตอนถัดไป: การดาวน์โหลดและกำหนดค่า be-BOP



**หมายเหตุ:** คู่มือทางเทคนิคสำหรับการปรับใช้ซอฟต์แวร์มีให้ในบทแนะนำแยกต่างหาก



## การสร้างบัญชีซุปเปอร์แอดมิน



ครั้งแรกที่ be-BOP ถูกเปิดตัว บัญชี **Super Admin** จะถูกสร้างขึ้น บัญชีนี้มีการอนุญาตทั้งหมดที่จำเป็นในการจัดการฟังก์ชันแบ็คออฟฟิศ ในการสร้างบัญชี ให้ทำตามขั้นตอนเหล่านี้:





- ไปที่ `yourresiteweb/admin/login`
- สร้างบัญชีซุปเปอร์แอดมินด้วยการเข้าสู่ระบบและรหัสผ่านที่ปลอดภัย



บัญชีนี้จะให้คุณเข้าถึงฟังก์ชันทั้งหมดของสำนักงานหลัง เมื่อสร้างแล้ว คุณสามารถเข้าสู่ระบบโดยป้อนชื่อผู้ใช้และรหัสผ่านของคุณ

## การตั้งค่าการสื่อสาร



ในการเปิดใช้งาน be-BOP เพื่อส่งการแจ้งเตือน (เช่น สำหรับคำสั่งซื้อ การลงทะเบียน หรือข้อความระบบ) คุณจำเป็นต้องกำหนดค่าอย่างน้อยหนึ่งวิธีการสื่อสาร มีสองตัวเลือกที่สามารถใช้ได้: อีเมล (SMTP) หรือ Nostr



### การกำหนดค่า SMTP (อีเมล)



be-BOP สามารถส่งอีเมลผ่านเซิร์ฟเวอร์ SMTP ได้ คุณจะต้องมีข้อมูลประจำตัว SMTP ที่ถูกต้อง ซึ่งมักจะได้รับจากบริการอีเมล (เช่น Mailgun, Gmail, เป็นต้น)



นี่คือสิ่งที่คุณจำเป็นต้องรู้:



SMTP_HOST: ที่อยู่เซิร์ฟเวอร์ SMTP (เช่น smtp.mailgun.org)



SMTP_PORT: พอร์ตที่ใช้ (มักจะเป็น 587 หรือ 465)



SMTP_USER: ชื่อผู้ใช้ของคุณ (โดยปกติจะเป็นที่อยู่อีเมล)



SMTP_PASSWORD: รหัสผ่านของคุณหรือคีย์ API



SMTP_FROM: ที่อยู่อีเมลที่จะปรากฏเป็นผู้ส่ง




### การกำหนดค่า Nostr



be-BOP ช่วยให้คุณสามารถส่งการแจ้งเตือนผ่านโปรโตคอล Nostr ซึ่งเป็นโครงสร้างพื้นฐานการส่งข้อความแบบกระจายศูนย์ ในการทำเช่นนี้ คุณจำเป็นต้อง generate หรือจัดหาคีย์ส่วนตัว Nostr (NSEC) คุณสามารถ generate คีย์นี้ได้โดยตรงผ่าน Interface ของ be-BOP ในส่วนที่อุทิศให้กับ Nostr เมื่อองค์ประกอบเหล่านี้ถูกกำหนดค่าอย่างถูกต้อง be-BOP จะสามารถส่งข้อความและการแจ้งเตือนให้กับผู้ใช้ของคุณโดยอัตโนมัติ



## วิธีการชำระเงินที่รองรับ



be-BOP เข้ากันได้กับโซลูชันการชำระเงินหลายรูปแบบ ช่วยให้คุณมอบความยืดหยุ่นที่มากขึ้นให้กับลูกค้าของคุณ นี่คือสิ่งที่คุณต้องใช้ในการตั้งค่าช่องทางการชำระเงินที่เหมาะสมที่สุดสำหรับคุณ



### Bitcoin Onchain



be-BOP ช่วยให้คุณยอมรับการชำระเงิน Bitcoin ได้โดยตรงบน Blockchain (On-Chain) อย่างง่ายดายและปลอดภัย



**ขั้นตอนการกำหนดค่า:**





- ไปที่เมนู **การตั้งค่าการชำระเงิน**
- คลิกที่ **Bitcoin Nodeless** เพื่อเข้าถึงพารามิเตอร์การชำระเงิน On-Chain
- กรอกข้อมูลในช่องต่อไปนี้:



| Champ                  | Description                                               | Exemple à utiliser                              |
|------------------------|-----------------------------------------------------------|--------------------------------------------------|
| **BIP Standard**       | Le type d’adressage utilisé                               | BIP84 (pour les adresses au format bech32 commençant par `bc1`) |
| **Clé publique étendue** | Votre Zpub (ou Xpub selon le portefeuille utilisé)        | `zpub...` (extrait de votre portefeuille Bitcoin) |
| **Derivation Index**   | L’index de départ pour la génération des adresses         | `1`                                              |
| **Mempool URL**        | L’URL du service mempool utilisé pour suivre les transactions | `https://mempool.space`                         |

![payment-nodeless](assets/fr/005.webp)



**เคล็ดลับ:** ในการรับคีย์สาธารณะที่ขยายของคุณ (Zpub) คุณสามารถตรวจสอบการตั้งค่าขั้นสูงของ Bitcoin wallet (Sparrow wallet, BlueWallet, Specter, ฯลฯ) ตรวจสอบให้แน่ใจว่า wallet ของคุณ **ไม่ได้เป็นแบบอ่านอย่างเดียว** หากคุณตั้งใจจะใช้ประวัติการทำธุรกรรม



### Lightning Network



be-BOP สามารถรับการชำระเงิน Bitcoin ได้ทันทีด้วยความช่วยเหลือจาก Lightning Network ขณะนี้มีตัวเลือกการกำหนดค่าที่พร้อมใช้งานสองแบบ:



**ฟีนิกซ์**



ไปที่เมนู `Payment Settings` แล้วคลิกที่ `Phoenixd`



![phoenixd](assets/fr/006.webp)



จากนั้นคุณจะต้องป้อน **รหัสผ่านหรือการยืนยันตัวตน token** ที่เชื่อมต่อคุณกับอินสแตนซ์ Phoenixd ของคุณ ซึ่งเป็นแบ็กเอนด์ที่พัฒนาโดย Acinq ที่ช่วยให้คุณจัดการการชำระเงิน Lightning ด้วยโหนดของคุณเอง แต่ไม่ต้องยุ่งยากกับการจัดการช่องทางการชำระเงิน



**Swiss Bitcoin Pay**



หากคุณไม่ต้องการจัดการโหนด Lightning ด้วยตัวเอง **Swiss Bitcoin Pay** เป็นโซลูชันที่พร้อมใช้งานและปรับแต่งได้ง่าย ซึ่งเหมาะสำหรับการเริ่มต้นรับชำระเงินผ่าน Lightning โดยไม่ต้องมีโครงสร้างพื้นฐานที่ซับซ้อน



ขั้นตอนการกำหนดค่า:





- ในเมนู "การตั้งค่าการชำระเงิน" คลิกที่ `Swiss Bitcoin Pay`
- เข้าสู่ระบบบัญชี Swiss Bitcoin Pay ของคุณ (หรือสร้างบัญชีใหม่หากคุณยังไม่มี)
- ป้อนรหัส API ที่ได้รับจาก Swiss Bitcoin Pay จากนั้นคลิก "บันทึก"



เมื่อทำการตั้งค่าแล้ว be-BOP จะทำการออกใบแจ้งหนี้ generate Lightning ให้กับลูกค้าของคุณโดยอัตโนมัติ และคุณจะได้รับการชำระเงินโดยตรงเข้าสู่บัญชี Swiss Bitcoin Pay ของคุณ โซลูชันนี้เหมาะสำหรับผู้ใช้ที่ต้องการหลีกเลี่ยงความซับซ้อนทางเทคนิคของโหนดส่วนตัวในขณะที่รับการชำระเงินที่รวดเร็วและมีต้นทุนต่ำ



![swissbtcpay](assets/fr/007.webp)



### เพย์พาล



นอกเหนือจาก Bitcoin แล้ว be-BOP ยังให้คุณรับชำระเงินด้วยเงินสดผ่าน PayPal ซึ่งเป็นโซลูชันระหว่างประเทศที่มีชื่อเสียงและใช้กันอย่างแพร่หลายอีกด้วย



ขั้นตอนการกำหนดค่า:





- ไปที่เมนู `Payment Settings`
- คลิกที่ `PayPal
- ในบัญชี Paypal ของคุณ (ส่วนของนักพัฒนา) ป้อน `Client ID` และ `Secret`
- เลือกสกุลเงินที่คุณต้องการ (เช่น **USD**, **EUR**, **XOF**, เป็นต้น)
- คลิกที่ `save



![paypal](assets/fr/008.webp)



**หมายเหตุ:** คุณต้องมีบัญชีธุรกิจของ PayPal เพื่อ generate ตัวระบุเหล่านี้ คุณสามารถรับได้ผ่านทางพอร์ทัล [developer] (https://developer.paypal.com)



### SumUp



ซอฟต์แวร์นี้ได้รวมการชำระเงินผ่าน **SumUp** ทำให้คุณสามารถรับชำระเงินด้วยบัตรเครดิตได้อย่างง่ายดาย ปลอดภัย และมีประสิทธิภาพ เพื่อใช้ประโยชน์จากฟังก์ชันนี้ จำเป็นต้องมีการตั้งค่าเริ่มต้น นี่คือขั้นตอนที่ต้องปฏิบัติตาม โดยมีการลำดับหมายเลขเพื่อการดำเนินการที่ชัดเจนและเป็นขั้นตอน:





- เริ่มต้นด้วยการป้อน **API Key** ของคุณ ซึ่งเป็นคีย์ลับที่ SumUp จัดหาให้เมื่อคุณสร้างบัญชีนักพัฒนา มันจะสร้างการเชื่อมต่อที่ปลอดภัยระหว่างบัญชี SumUp ของคุณกับซอฟต์แวร์
- กรอกฟิลด์ `Merchant Code` ด้วยรหัสเฉพาะที่ระบุธุรกิจของคุณภายในแพลตฟอร์ม SumUp รหัสนี้มีความสำคัญสำหรับการเชื่อมโยงธุรกรรมกับธุรกิจของคุณ
- ในช่อง `Currency` ให้เลือกสกุลเงินหลักที่คุณใช้สำหรับการทำธุรกรรมของคุณ (เช่น **EUR**, **USD**, **CDF** เป็นต้น)
- เมื่อกรอกข้อมูลในทุกช่องถูกต้องแล้ว ให้คลิกที่ปุ่ม `Save` เพื่อบันทึกการตั้งค่าของคุณ จากนั้นระบบจะเชื่อมโยงกับบัญชี SumUp ของคุณ และซอฟต์แวร์ของคุณจะพร้อมรับการชำระเงิน



![payment-sumup](assets/fr/009.webp)



หลังจากการกำหนดค่านี้ การผสานรวม **SumUp** จะเปิดใช้งานและทำงานได้ ทำให้คุณสามารถถอนเงินสดและติดตามธุรกรรมของคุณได้โดยตรงจากซอฟต์แวร์



### Stripe



be-BOP ยังมีการผสานรวมอย่างเต็มรูปแบบกับ **Stripe** ซึ่งเป็นหนึ่งในแพลตฟอร์มการชำระเงินออนไลน์ที่ได้รับความนิยมมากที่สุด Stripe ช่วยให้คุณสามารถรับชำระเงินออนไลน์ผ่านบัตรเครดิต, ดิจิทัล wallet และวิธีการชำระเงินอื่น ๆ อีกหลายวิธี นี่คือวิธีการเปิดใช้งาน:





- ป้อน **รหัสลับ** ที่ให้ไว้ในแดชบอร์ด Stripe.
- กรอกข้อมูลในช่อง **Public Key** ซึ่ง Stripe จัดเตรียมให้
- เลือก **สกุลเงินหลัก**
- บันทึกการกำหนดค่า จากนั้นคลิก `Save`.



![payment-stripe](assets/fr/010.webp)



⚠️ **โปรดทราบ:** จำเป็นต้องทราบระบอบภาษีมูลค่าเพิ่มที่ใช้กับกิจกรรมของคุณ (เช่น: การขายภายใต้ภาษีมูลค่าเพิ่มในประเทศของผู้ขาย, การยกเว้นภายใต้การพิสูจน์, หรือการขายในอัตราภาษีมูลค่าเพิ่มของประเทศผู้ซื้อ) เพื่อกำหนดค่าตัวเลือกการออกใบแจ้งหนี้ใน **be-BOP** ให้ถูกต้อง



## การกำหนดค่าสกุลเงิน



**be-BOP** นำเสนอการจัดการสกุลเงินขั้นสูงและปรับให้เข้ากับสภาพแวดล้อมหลายสกุลเงินและข้อกำหนดทางบัญชีเฉพาะ เพื่อให้มั่นใจถึงความสม่ำเสมอในการดำเนินงานและการรายงานทางการเงิน จำเป็นต้องกำหนดค่าสกุลเงินต่างๆ ที่ใช้ในระบบอย่างถูกต้อง นี่คือขั้นตอนที่ต้องปฏิบัติตามสำหรับการกำหนดค่านี้:





- เลือก **สกุลเงินหลัก** (`Main currency`)
- เลือก `สกุลเงินรอง`
- กำหนด **reference currency** (`Price reference currency`)
- ระบุ `สกุลเงินบัญชี



เมื่อมีการตั้งค่าสกุลเงินทั้งหมดอย่างถูกต้องแล้ว ซอฟต์แวร์จะรับประกันการแปลงธุรกรรมหลายสกุลเงินโดยอัตโนมัติและแม่นยำ พร้อมทั้งรักษาความสม่ำเสมอทางบัญชีอย่างเข้มงวด

## Interface และการกำหนดค่า Identity ใน be-BOP



**be-BOP** มอบเครื่องมือทั้งหมดที่นักออกแบบต้องการในการออกแบบเว็บไซต์ ขั้นตอนแรกคือการเปิดส่วน `/Admin > Merch > Layout` ในการตั้งค่า เริ่มต้นด้วยการกำหนดค่า **Top Bar**, **Navbar** และ **Footer**



### Le Top Bar



การกำหนดค่า **Top Bar** ช่วยให้คุณปรับแต่งเอกลักษณ์ทางภาพของซอฟต์แวร์ของคุณโดยการแสดงข้อมูลสำคัญตั้งแต่บรรทัดแรกของ Interface ซึ่งจะช่วยเสริมสร้างการจดจำแบรนด์และให้บริบทที่ชัดเจนสำหรับผู้ใช้



#### ขั้นตอนการกำหนดค่า:





- ในช่อง `ชื่อแบรนด์` ให้ป้อนชื่อบริษัท องค์กร หรือผลิตภัณฑ์ของคุณ ชื่อนี้จะปรากฏที่ด้านบนของ Interface และจะเป็นตัวแทนของอัตลักษณ์ทางภาพหลักของคุณ
- ระบุชื่อเว็บไซต์**: ชื่อที่เลือกควรสรุปวัตถุประสงค์ของแพลตฟอร์ม ชื่อนี้สามารถปรากฏในส่วนหัวหรือในแท็บเบราว์เซอร์
- เพิ่มคำอธิบายเว็บไซต์**: นี่คือที่ที่คุณใส่คำอธิบายสั้น ๆ เกี่ยวกับโครงการของคุณ คำอธิบายนี้ช่วยให้ผู้ใช้เข้าใจบริบทของเครื่องมือและยังสามารถใช้เพื่อวัตถุประสงค์ SEO ได้อีกด้วย



เมื่อป้อนข้อมูลนี้แล้ว **แถบด้านบน** จะแสดงการนำเสนอที่ชัดเจน เป็นมืออาชีพ และสอดคล้องกันของโซลูชันของคุณ



#### ลิงก์ในแถบด้านบน



ส่วน `ลิงก์` ของแถบด้านบนช่วยให้คุณเพิ่มทางลัดไปยังหน้าสำคัญในแอปพลิเคชันของคุณหรือบนเว็บไซต์ภายนอก ลิงก์เหล่านี้จะแสดงโดยตรงในแถบด้านบน มอบการเข้าถึงที่รวดเร็วและมีโครงสร้างให้กับผู้ใช้ของคุณ



#### ขั้นตอนการกำหนดค่า:





- ป้อนชื่อของลิงก์ (ข้อความ)**: ในช่อง `ข้อความ` ให้ป้อนชื่อหรือป้ายกำกับของลิงก์ตามที่จะแสดง (เช่น หน้าแรก, ติดต่อ, ช่วยเหลือ...)
- ระบุที่อยู่ลิงก์ (Url)**: ในช่อง `Url` ให้ป้อนที่อยู่เต็มของหน้าที่ต้องการ (ภายในหรือภายนอก)
- เพิ่มลิงก์อื่นๆ หากจำเป็น**: แต่ละบรรทัดการกำหนดค่าช่วยให้คุณเพิ่มลิงก์เพิ่มเติมโดยใช้ฟิลด์ `Text` และ `Url`
- บันทึกลิงก์**: เมื่อป้อนลิงก์ทั้งหมดแล้ว ให้คลิกที่ปุ่ม "เพิ่มลิงก์แถบด้านบน" เพื่อบันทึกพวกมัน



การกำหนดค่านี้ช่วยให้คุณสามารถนำเสนอการนำทางที่ชัดเจน ลื่นไหล และเข้าถึงได้ผ่านส่วนต่างๆ ของเว็บไซต์ของคุณหรือไปยังแหล่งข้อมูลเสริม



![settings-topbar](assets/fr/014.webp)



### ลา นาฟ บาร์



ส่วน **Navbar** ช่วยให้คุณกำหนดค่าเมนูการนำทางหลักของ be-BOP ซึ่งมักจะอยู่ด้านข้างหรือด้านบนของ Interface เมนูนี้จะนำทางผู้ใช้ไปยังหน้าต่างๆ และฟังก์ชันต่างๆ ของแอปพลิเคชัน การกำหนดค่าลิงก์นั้นง่ายและเข้าใจได้ง่าย นี่คือวิธีการทำงาน:





- ป้อนชื่อของลิงก์ (`Text`): บนบรรทัดการกำหนดค่า เริ่มต้นด้วยการกรอกฟิลด์ `Text` ซึ่งสอดคล้องกับชื่อของลิงก์ที่แสดงในแถบนำทาง (ตัวอย่าง: *Dashboard*, *Users*, *Settings*...)
- ป้อนที่อยู่ลิงก์ (`Url`): ถัดจากฟิลด์ `Text` คุณจะพบฟิลด์ `Url` ในฟิลด์นี้ ให้ป้อนที่อยู่ของหน้าที่ลิงก์ควรเปลี่ยนเส้นทางไปยังหน้านั้น ซึ่งอาจเป็นเส้นทางภายในหรือเป็นลิงก์ไปยังหน้าภายนอกก็ได้
- เพิ่มลิงก์หลายรายการหากจำเป็น**: ด้านล่างบรรทัดแรก มีฟิลด์ `Text` และ `Url` ใหม่สำหรับเพิ่มลิงก์นำทางได้ตามต้องการ แต่ละบรรทัดแสดงถึงลิงก์นำทางเพิ่มเติม
- บันทึกลิงก์**: เมื่อคุณป้อนองค์ประกอบทั้งหมดแล้ว ให้คลิกที่ปุ่ม `Add nav bar link` เพื่อบันทึกและแสดงผลในแถบนำทาง



การกำหนดค่านี้ช่วยให้สามารถจัดโครงสร้างการเข้าถึงส่วนต่างๆ ของซอฟต์แวร์ได้อย่างมีประสิทธิภาพ ปรับปรุงการใช้งานและประสบการณ์ของผู้ใช้



![navbar](assets/fr/015.webp)



### ท้ายกระดาษ



ส่วน **Footer** ช่วยให้คุณปรับแต่งส่วนท้ายของซอฟต์แวร์ของคุณ โดยเพิ่มข้อมูลหรือการเชื่อมโยงที่มีประโยชน์ ก่อนที่จะกำหนดค่าการเชื่อมโยง ให้เริ่มต้นด้วยการเปิดใช้งานตัวเลือกเฉพาะ:





- เปิดใช้งานการแสดงป้าย "Powered by be-BOP": เปิดใช้งานปุ่ม `Display Powered by be-BOP` เพื่อแสดงป้ายนี้ในส่วนท้าย.
- ป้อนชื่อของลิงก์ (**Text**): กรอกในช่อง **Text** ซึ่งตรงกับคำของลิงก์ในส่วนท้าย (ตัวอย่าง: *Terms*, *Privacy*, *Contact*...).
- ระบุที่อยู่ลิงก์ (`Url`): ในช่อง `Url` ให้ป้อนที่อยู่ของหน้าปลายทาง (ภายในหรือภายนอก)
- เพิ่มลิงก์เพิ่มเติมหากจำเป็น**: ใช้บรรทัดเพิ่มเติมเพื่อสร้างลิงก์ได้ตามต้องการ
- บันทึกลิงก์**: คลิกที่ปุ่ม "เพิ่มลิงก์ท้ายกระดาษ" เพื่อบันทึกลิงก์

### การตั้งค่าตัวตนผู้ขาย



#### การตั้งค่าตัวตน



สามารถเข้าถึงได้ผ่าน `Admin > Identity` (หรือ `Settings > Identity`) ส่วนนี้ช่วยให้คุณกำหนดค่าข้อมูลการบริหารและกฎหมายของบริษัทของคุณ



#### ข้อมูลทางกฎหมาย





- ชื่อธุรกิจ**: ชื่อบริษัทอย่างเป็นทางการ
- Business ID**: ตัวระบุทางกฎหมายหรือหมายเลขการจดทะเบียน (RCCM, SIRET...).



#### ที่อยู่ธุรกิจ





- ถนน**: ที่อยู่ไปรษณีย์ (ถนน, หมายเลข...).
- ประเทศ**: country.
- รัฐ**: จังหวัดหรือภูมิภาค.
- เมือง**: city.
- รหัสไปรษณีย์**: postal code.



#### ข้อมูลติดต่อ





- อีเมล**: ที่อยู่อีเมลมืออาชีพ
- โทรศัพท์**: หมายเลขโทรศัพท์ของบริษัท



#### บัญชีธนาคาร





- ชื่อผู้ถือบัญชี**: ชื่อของผู้ถือบัญชี
- ผู้ถือบัญชี Address**: ที่อยู่ของผู้ถือบัญชี
- IBAN**: หมายเลขบัญชีธนาคารระหว่างประเทศ.
- BIC**: รหัส SWIFT/BIC.

### การสร้างผลิตภัณฑ์ใหม่

#### ข้อมูลพื้นฐาน

- Slug**: ตัวระบุ URL ที่ไม่มีช่องว่าง (เช่น `tshirt-bop-edition-limitee`)

![product-config](assets/fr/028.webp)

- Price Amount**: ราคาสินค้า (เช่น `25.00`).
- สกุลเงินราคา**: สกุลเงิน (EUR, USD, BTC, ฯลฯ).
- ผลิตภัณฑ์พิเศษ**:
  - นี่คือผลิตภัณฑ์ฟรี

- ผลิตภัณฑ์ที่มีรูปแบบแตกต่างกัน**:
  - อย่าตรวจสอบ `Standalone`.
  - ตรวจสอบ `ผลิตภัณฑ์มีความแตกต่างของแสง (ไม่มีความแตกต่างของสต็อก)`.
  - เพิ่ม:
- ชื่อ** (เช่น *ขนาด*),
- ค่า** (เช่น: S, M, L, XL),
- ความแตกต่างของราคา** ถ้ามี (เช่น: `+2 USD` สำหรับ XL)

## การจัดการสต็อก



### ตัวเลือกขั้นสูงเมื่อสร้างผลิตภัณฑ์ (สต็อก, การจัดส่ง, ตั๋ว, ฯลฯ)



#### ผลิตภัณฑ์ที่มีสต็อกจำกัด



หากสินค้าของคุณไม่มีในปริมาณไม่จำกัด ให้เลือก `สินค้ามีสต็อกจำกัด` ซึ่งจะเปิดใช้งานการติดตามปริมาณที่เหลือโดยอัตโนมัติ เมื่อทำเครื่องหมายในช่องนี้แล้ว จะมีช่องปรากฏขึ้นเพื่อระบุ **สต็อกที่มีอยู่**



ระบบจัดการ:





- สต็อกที่สงวนไว้** → ผลิตภัณฑ์ในตะกร้าที่ลูกค้ายังไม่ได้ชำระเงิน
- สต็อกที่ขายแล้ว** → ผลิตภัณฑ์ที่ซื้อแล้ว



**เวลาการจองตะกร้า**: เมื่อลูกค้าเพิ่มสินค้าในตะกร้า สินค้านั้นจะถูก "จอง" ไว้ในระยะเวลาจำกัด คุณสามารถแก้ไขเวลานี้ได้ที่: `Admin > Config > Cart reservation` (ค่าเป็นนาที)



#### สินค้าที่จะจัดส่ง?



ตรวจสอบ `The product has a physical component that will be shipped to the customer's Address`. สิ่งนี้มีประโยชน์สำหรับสินค้าทั้งหมดที่จะถูกส่งทางกายภาพ (หนังสือ, เสื้อยืด, ฯลฯ)



#### ตัวเลือกอื่น ๆ





- ตั๋ว**: ทำเครื่องหมายหากผลิตภัณฑ์เป็นตั๋วสำหรับงานอีเวนต์
- การจอง**: ตรวจสอบว่านี่คือช่วงเวลาการจองหรือไม่ (เช่น: เซสชัน, การนัดหมาย)

## การสร้างและปรับแต่งหน้าของ CMS และวิดเจ็ต



### หน้าบังคับของ CMS



ไปที่ `Admin > Merch > CMS` คุณจะเห็นรายการของหน้าที่มีอยู่แล้วและสามารถเพิ่มหน้าใหม่ได้ด้วย **Add CMS page**



หน้า CMS มีความสำคัญสำหรับ:





- แจ้งให้ผู้เยี่ยมชมทราบ (เช่น ข้อกำหนดการใช้งาน)
- ปฏิบัติตามกฎหมาย (เช่น นโยบายความเป็นส่วนตัว)
- อธิบายคุณสมบัติบางประการของร้านค้า (เช่น การเก็บ IP, ภาษีมูลค่าเพิ่ม 0%)



คุณสามารถเพิ่มหน้าอื่นๆ ตามที่ต้องการ:

- คำถามที่พบบ่อย
- ติดต่อ
- เป็นต้น



**เคล็ดลับ: คลิกที่ลิงก์หรือไอคอนแต่ละอันเพื่อแก้ไขเนื้อหา ชื่อ หรือการมองเห็น SEO ของแต่ละหน้า**

#### แถบนำทาง (แถบนำทางหลัก)

### การผสานวิดเจ็ตเข้ากับหน้า CMS



**วิดเจ็ต** ช่วยเพิ่มความสมบูรณ์ให้กับหน้า CMS ของคุณด้วยองค์ประกอบที่มีความเคลื่อนไหวหรือมีภาพลักษณ์ที่น่าสนใจ



#### การสร้างวิดเจ็ต



ไปที่: `Admin > Widgets`



ตัวอย่างวิดเจ็ตที่มีอยู่:





- ความท้าทาย**: ความท้าทายหรือภารกิจ
- แท็ก**: หมวดหมู่หรือคำสำคัญ
- สไลเดอร์**: แคโรเซลภาพ
- ข้อกำหนด**: ตารางข้อกำหนด
- ฟอร์ม**: ฟอร์ม (ติดต่อ, ข้อเสนอแนะ, ฯลฯ)
- นับถอยหลัง**: ตัวจับเวลา
- แกลเลอรี**: แกลเลอรีรูปภาพ
- กระดานผู้นำ**: การจัดอันดับผู้ใช้



![widgets](assets/fr/033.webp)



#### การผสานรวมเข้ากับหน้า CMS



ใช้ **shortcodes** ในเนื้อหาของหน้า CMS ของคุณ:



| Objectif                 | Balise à insérer                      |
|--------------------------|---------------------------------------|
| Afficher un produit      | `[Product=slug?display=img-1]`        |
| Afficher une image       | `[Picture=slug width=100 height=100 fit=contain]` |
| Intégrer un slider       | `[Slider=slug?autoplay=3000]`         |
| Ajouter un challenge     | `[Challenge=slug]`                    |
| Ajouter un compte à rebours | `[Countdown=slug]`                 |
| Intégrer un formulaire   | `[Form=slug]`                         |

**Current parameters**:





- `slug`: ตัวระบุวิดเจ็ตที่ไม่ซ้ำกัน
- `display=img-1`: ภาพเฉพาะผลิตภัณฑ์
- `width`, `height`, `fit`: ขนาดและสไตล์ของภาพ
- autoplay=3000`: เวลาในหน่วยมิลลิวินาทีระหว่างสองสไลด์



**ข้อดี**:





- ง่ายต่อการแทรก (คัดลอกและวาง)
- ไดนามิก: การแก้ไขใด ๆ กับวิดเจ็ตจะสะท้อนโดยอัตโนมัติ
- ไม่ต้องการนักพัฒนา



## การจัดการคำสั่งซื้อและการรายงาน



### การติดตามคำสั่งซื้อ



ในการดูและจัดการคำสั่งซื้อที่ผ่านมา ไปที่: `Admin > Transaction > Orders`



ที่นี่คุณจะพบกับ**รายการคำสั่งซื้อทั้งหมด**ที่สั่งบนเว็บไซต์ของคุณ



![orders](assets/fr/034.webp)



#### การสร้างภาพและการค้นหา



Interface ช่วยให้คุณสามารถค้นหาและกรองคำสั่งซื้อตามเกณฑ์หลายประการ:





- หมายเลขคำสั่งซื้อ: order number
- นามแฝงของผลิตภัณฑ์: ตัวระบุหรือชื่อผลิตภัณฑ์
- "payment Mean": วิธีการชำระเงินที่ใช้ (บัตร, คริปโต, ฯลฯ)
- `อีเมล`: อีเมลลูกค้า



ฟิลเตอร์เหล่านี้ช่วยให้การค้นหาและการจัดการเป้าหมายเป็นไปอย่างรวดเร็ว



#### รายละเอียดของแต่ละคำสั่งซื้อ



โดยการคลิกที่คำสั่งซื้อ คุณสามารถเข้าถึงไฟล์ที่สมบูรณ์ซึ่งประกอบด้วย:





- ผลิตภัณฑ์ที่สั่งซื้อ
- ข้อมูลลูกค้า
- ที่อยู่สำหรับจัดส่ง (ถ้ามี)
- มีบันทึกใด ๆ ที่เกี่ยวข้องกับคำสั่งซื้อหรือไม่



#### การดำเนินการที่เป็นไปได้ในคำสั่งซื้อ



คุณสามารถ:





- ยืนยันคำสั่งซื้อ (หากอยู่ระหว่างดำเนินการ)
- ยกเลิกคำสั่งซื้อ (ในกรณีที่มีปัญหาหรือคำขอของลูกค้า)
- เพิ่ม **ป้ายกำกับ** (สำหรับการจัดระเบียบภายใน)
- ปรึกษา / เพิ่ม **บันทึกภายใน**



**หมายเหตุ:** ส่วนนี้มีความสำคัญต่อการจัดการโลจิสติกส์และความสัมพันธ์กับลูกค้า



### การรายงานและการส่งออก



ในการเข้าถึงสถิติการขายและการชำระเงิน:

ที่นี่คุณจะพบภาพรวมของธุรกิจของคุณ ในรูปแบบของ**รายงานรายเดือนและรายปี**



#### รายงานเนื้อหา



รายงานถูกแบ่งออกเป็นส่วน ๆ:





- รายละเอียดการสั่งซื้อ**: จำนวนการสั่งซื้อ, สถานะ (ยืนยันแล้ว, ยกเลิก, รอดำเนินการ), การพัฒนา
- รายละเอียดสินค้า**: สินค้าที่ขาย, ปริมาณ, สินค้ายอดนิยม
- รายละเอียดการชำระเงิน**: จำนวนเงินที่เก็บได้, การแจกแจงตามวิธีการชำระเงิน



#### การส่งออกข้อมูล



แต่ละส่วนประกอบด้วยปุ่ม **Export CSV** ซึ่งช่วยให้คุณสามารถ:





- ดาวน์โหลดข้อมูลในรูปแบบ CSV
- เปิดใน Excel, Google Sheets, เป็นต้น
- การจัดเก็บเอกสารเพื่อการบริหารหรือการบัญชี
- ใช้สำหรับรายงานภายใน



**หมายเหตุ:** เหมาะสำหรับการติดตามประสิทธิภาพ การบัญชี และการนำเสนอ

- ช่วยให้การจัดการรวดเร็วและเป็นมิตรกับอุปกรณ์เคลื่อนที่



**การเชื่อมต่อโดยไม่ใช้รหัสผ่าน**





- เข้าสู่ระบบโดยลิงก์ที่ปลอดภัย (ส่งผ่าน Nostr)
- ความปลอดภัยและความลื่นไหลของผู้ใช้ที่มากขึ้น



## การออกแบบและปรับแต่งธีม



ในการปรับลักษณะของร้านค้าให้เข้ากับกราฟิกชาร์เตอร์ของคุณ ไปที่: `Admin > Merch > Theme`



ที่นี่คุณจะพบตัวเลือกทั้งหมดสำหรับการ**สร้าง**และ**กำหนดค่า**ธีมที่กำหนดเอง



### สร้างธีม



![theme](assets/fr/037.webp)



เมื่อสร้างหรือแก้ไขธีม คุณสามารถกำหนด:





- สี**: สำหรับปุ่ม, พื้นหลัง, ข้อความ, ลิงก์, ฯลฯ.
- แบบอักษร**: การเลือกประเภทตัวอักษรสำหรับหัวข้อ, ย่อหน้า, เมนู
- รูปแบบกราฟิก**: เส้นขอบ, ระยะขอบ, การเว้นระยะ, รูปทรงบล็อก



### ส่วนที่ปรับแต่งได้



แต่ละส่วนของไซต์สามารถปรับได้อย่างอิสระ:





- ส่วนหัว**: แถบนำทางด้านบน
- เนื้อหา**: เนื้อหาหลัก
- ท้ายกระดาษ**: ด้านล่างของหน้า



**หมายเหตุ:** ความละเอียดนี้ช่วยให้มั่นใจได้ถึงความสอดคล้องกันระหว่างภาพของไซต์และอัตลักษณ์ของแบรนด์ของคุณ



### การเปิดใช้งานธีม



เมื่อตั้งค่าธีมเสร็จสิ้น:





- คลิกที่ **บันทึก**
- เปิดใช้งานเป็น **ธีมหลัก** ของร้านค้า



**หมายเหตุ:** ธีมที่ใช้งานอยู่คือธีมที่จะมองเห็นได้โดยผู้เยี่ยมชม



## กำหนดค่าแม่แบบอีเมล



แพลตฟอร์มนี้ให้คุณปรับแต่งอีเมลที่ส่งอัตโนมัติถึงผู้ใช้ ไปที่: `Admin > Settings > Templates`



![emails-templates](assets/fr/038.webp)



### การสร้าง / แก้ไขเทมเพลต



อีเมลแต่ละฉบับ (ยืนยันการสั่งซื้อ, ลืมรหัสผ่าน, ฯลฯ) มี:





- Subject**: หัวข้อของอีเมล (เช่น "คำสั่งซื้อของคุณได้รับการยืนยันแล้ว")
- HTML Body**: เนื้อหา HTML ที่แสดงในอีเมล



**หมายเหตุ:** คุณสามารถแทรกข้อความ รูปภาพ ลิงก์ ฯลฯ ได้ตามต้องการ



### การใช้ตัวแปรแบบไดนามิก



ในการทำให้อีเมลมีความไดนามิก ให้แทรกตัวแปรเช่น:





- `{orderNumber}}`: แทนที่ด้วยหมายเลขคำสั่งซื้อจริง
- `{invoiceLink}}`: ลิงก์ไปยังใบแจ้งหนี้
- `{websiteLink}}`: URL ของเว็บไซต์ของคุณ



**หมายเหตุ:** แท็กเหล่านี้จะถูกแทนที่โดยอัตโนมัติเมื่อส่ง.



### เคล็ดลับขั้นสูง





- สร้างอีเมลที่**ตอบสนอง**เพื่อให้อ่านง่ายบนอุปกรณ์เคลื่อนที่
- เพิ่ม **ปุ่มการทำงาน** (ชำระเงิน, ดาวน์โหลด, ติดตามคำสั่งซื้อ)
- ทดสอบอีเมลของคุณโดยส่งถึงตัวคุณเองก่อนเผยแพร่



## กำหนดค่าแท็กและวิดเจ็ตเฉพาะ



### การจัดการแท็ก



แท็กสามารถใช้เพื่อจัดโครงสร้างและเพิ่มคุณค่าให้กับเนื้อหาของคุณ ในการเข้าถึง: `Admin > Widgets > Tag`



![tags-config](assets/fr/039.webp)



### สร้างแท็ก



กรอกข้อมูลในช่องต่อไปนี้:





- ชื่อแท็ก**: ชื่อแท็กที่แสดง
- Slug**: unique identifier (no spaces or accents)
- แท็กครอบครัว**: จัดกลุ่มแท็กตามหมวดหมู่



![targsconfig](assets/fr/040.webp)



#### ครอบครัวที่มีอยู่:





- ผู้สร้าง`: ผู้เขียนหรือผู้ผลิต
- ผู้ค้าปลีก: พนักงานขายหรือจุดขาย
- `Temporal`: ช่วงเวลา หรือ วันที่
- เหตุการณ์: เหตุการณ์ที่เกี่ยวข้อง



### ฟิลด์ที่ไม่บังคับ



ฟิลด์เหล่านี้สามารถใช้เพื่อเพิ่มข้อมูลให้กับแท็กได้ราวกับว่าเป็นหน้าคอนเทนต์:





- ชื่อเรื่อง
- คำบรรยายใต้ภาพ
- เนื้อหา**สั้น
- เนื้อหาเต็ม** (เป็นภาษาฝรั่งเศส)
- ปุ่มกระตุ้นการตัดสินใจ** (action buttons)



### ใช้แท็ก



แท็กสามารถเป็น:





- จัดสรรให้กับผลิตภัณฑ์
- รวมเข้ากับหน้า CMS ด้วยแท็ก: [Tag=slug?display=var-1]



## การกำหนดค่าของไฟล์ที่สามารถดาวน์โหลดได้



ในการเสนอเอกสารที่ดาวน์โหลดได้ให้กับลูกค้าของคุณ: `Admin > Merch > Files`



### เพิ่มไฟล์



1. คลิกที่ **ไฟล์ใหม่**


2. แจ้ง:




- ชื่อไฟล์** (เช่น *คู่มือการติดตั้ง*)
- ไฟล์ที่จะอัปโหลด** (PDF, รูปภาพ, Word...)



**หมายเหตุ:** เมื่อเพิ่มแล้ว แพลตฟอร์มจะสร้าง **ลิงก์ถาวร** โดยอัตโนมัติ



### ใช้ลิงก์



จากนั้นลิงก์นี้สามารถแทรกลงใน:





- หน้า CMS** (เป็นลิงก์ข้อความหรือปุ่ม)
- ไคลเอนต์ **อีเมล** (ผ่านเทมเพลต)
- แผ่นข้อมูล**ผลิตภัณฑ์** (เช่น ดาวน์โหลดคู่มือ)



เหมาะอย่างยิ่งสำหรับการให้ *คู่มือผู้ใช้, คู่มือทางเทคนิค, แผ่นข้อมูลผลิตภัณฑ์...* โดยไม่จำเป็นต้องมีการโฮสต์ภายนอก



## Nostr-บอท



แพลตฟอร์มนี้มีการผสานรวมขั้นสูงกับโปรโตคอล **Nostr** ผ่านบอทอัตโนมัติ



ไปที่: node Management > Nostr



### คุณสมบัติหลัก



#### การจัดการรีเลย์





- เพิ่มหรือลบ **รีเลย์** ที่ใช้โดยบอท
- ปรับปรุงการ **เข้าถึง** และ **ความน่าเชื่อถือ** ของข้อความที่ส่ง



#### ข้อความแนะนำอัตโนมัติ





- เปิดใช้งานข้อความอัตโนมัติเมื่อ **ผู้ใช้โต้ตอบครั้งแรก**
- เหมาะสำหรับ:
  - นำเสนอบริการของคุณ
  - ส่งลิงก์ที่มีประโยชน์ (เช่น คำถามที่พบบ่อย, ติดต่อ, สั่งซื้อ)



#### การรับรอง `npub` ของคุณ





- เพิ่ม**โลโก้**และ**ชื่อสาธารณะ**
- ลิงก์ไปยัง **โดเมนเว็บที่ผ่านการตรวจสอบแล้ว**
- เพิ่มความน่าเชื่อถือและการยอมรับในตัวตน Nostr ของคุณ



### กรณีการใช้งาน Nostr-bot





- กำลังส่ง**การยืนยันคำสั่งซื้อ**ถึงคุณ
- การตอบสนองอัตโนมัติต่อ **เหตุการณ์ (เช่น คำสั่งซื้อใหม่)**
- สร้าง**การโต้ตอบกับลูกค้าแบบกระจายศูนย์**

## ทีมเวิร์ค & จุดขาย (POS)



### การจัดการผู้ใช้และสิทธิ์การเข้าถึง



#### การสร้างบทบาท



ไปที่: `Admin > Settings > ARM`



คลิกที่ **Create a role** เพื่อสร้างบทบาท (เช่น `Super Admin`, `POS`, `Ticket checker`)



แต่ละบทบาทประกอบด้วย:





- เขียนสิทธิ์การเข้าถึง**: เขียนสิทธิ์การเข้าถึง
- read access**: read access
- การเข้าถึงต้องห้าม**: sections interdites



#### การสร้างผู้ใช้



ในเมนูเดียวกัน `Admin > Settings > ARM` เพิ่มผู้ใช้ด้วย:





- เข้าสู่ระบบ
- นามแฝง
- กู้คืนอีเมล
- (ตัวเลือก) `recovery npub` สำหรับการเชื่อมต่อผ่าน Nostr



กำหนดบทบาทที่กำหนดไว้ก่อนหน้านี้



![pos-users](assets/fr/045.webp)



ผู้ใช้ **Read-only** จะเห็นเมนูในรูปแบบ *ตัวเอียง* และจะไม่สามารถแก้ไขเนื้อหาได้



## การกำหนดค่าจุดขาย (POS)



### กำหนดบทบาท POS



ในการให้สิทธิ์ผู้ใช้เข้าถึง POS ให้กำหนดบทบาท `Point of Sale (POS)` ใน: `Admin > Config > ARM`



เขาสามารถเชื่อมต่อผ่าน URL ที่ปลอดภัย: `/pos` หรือ `/pos/touch`



### คุณสมบัติเฉพาะของ POS



Be-BOP เสนอ Interface ที่ทุ่มเทให้กับการขายทางกายภาพ (ร้านค้า, งานอีเวนต์, ฯลฯ)



#### การบวกอย่างรวดเร็วผ่านนามแฝง



ใน `/cart` มีฟิลด์ที่ให้คุณเพิ่มสินค้า:





- โดยการสแกน **บาร์โค้ด** (ISBN, EAN13)
- โดยการป้อน **ชื่อเล่นผลิตภัณฑ์** ด้วยตนเอง



**หมายเหตุ:** สินค้าจะถูกเพิ่มลงในตะกร้าโดยอัตโนมัติ



#### วิธีการชำระเงิน



POS รองรับ:





- สปีชีส์
- บัตรเครดิต
- Lightning Network (crypto)
- อื่นๆ ตามการกำหนดค่า



มีตัวเลือกขั้นสูงสองตัวเลือก:





- การยกเว้นภาษีมูลค่าเพิ่ม**: ใช้ได้เมื่อมีการชี้แจง (องค์กรไม่แสวงหาผลกำไร, ชาวต่างชาติ...)
- ส่วนลดของขวัญ**: ส่วนลดพิเศษพร้อมความคิดเห็นที่จำเป็น



#### การแสดงผลฝั่งไคลเอ็นต์



URL `/pos/session` ถูกออกแบบมาสำหรับ **หน้าจอรอง** (HDMI, แท็บเล็ต...):



โปสเตอร์:





- ผลิตภัณฑ์ที่กำลังดำเนินการอยู่
- ยอดรวมทั้งหมด
- วิธีการชำระเงิน
- ส่วนลดที่ใช้



**หมายเหตุ:** ลูกค้าติดตามคำสั่งซื้อแบบสด ในขณะที่ผู้ขายบันทึกลงใน `/pos`.
