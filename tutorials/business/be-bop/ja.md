**be-BOP**は、Bitcoin、銀行口座経由、現金での支払いを受け付けながら、完全な自律性で、オンラインおよびオフラインでの販売を希望する起業家のために設計された電子商取引プラットフォームです。このソリューションは、寄付を集めたり、様々な活動を収益化したいあらゆるタイプの組織にも役立ちます。



このソリューションはシンプルで軽く、自律的である。伝統的な金融サービスが限られている、あるいは存在しない環境であっても、オンライン・ストアを作ることができる。実際、**be-BOP**は、Bitcoinを決済インフラとして使用し、銀行へのアクセスの有無にかかわらず効率的に運営できるように設計されている。



このチュートリアルでは、：





- **be-BOP**で初めてのオンラインストアを作ろう
- ショーケースと商品をパーソナライズする
- 利用可能な支払い方法の設定
- **be-BOP**で効果的なオンライン販売のベストプラクティスを理解する。



このチュートリアルには高度な技術的スキルは必要ありません。このチュートリアルは、開発者だけでなく、主権的で弾力的な方法でデジタル・コマースに着手したいと考えている職人、商人、協同組合、起業家を対象としています。



## 自分のサーバーにbe-BOPをインストールするための前提条件



be-BOPのインストールを開始する前に、以下の技術インフラが整っていることを確認してください。これらのElementsはプラットフォームが正しく機能するために不可欠です：



### S3互換ストレージ



be-BOPでは、ファイル（商品画像など）を管理するためにストレージシステムを使用しています。そのため、.NET FrameworkなどのS3サービスにアクセスする必要があります：





- [MinIO](https://min.io/) セルフホスト
- アマゾンS3（AWS）
- Scaleway オブジェクトストレージ



バケットを設定し、以下の情報を提供する必要があります：





- **S3_BUCKET**: バケット名
- **S3_ENDPOINT_URL**: S3サービスへのアクセスリンク
- **S3_KEY_ID**と**S3_KEY_SECRET**：アクセスコード
- **S3_REGION**: S3サービスのリージョン



### ReplicaSet モードの MongoDB データベース



be-BOPはMongoDBを使用して、店舗、ユーザー、商品、その他のデータを保存します。



選択肢は2つある：





- ReplicaSetモードを有効にしてMongoDBをローカルにインストールする。
- **MongoDBアトラス**のようなオンラインサービスを利用する。



以下の変数が必要です：





- **MONGODB_URL**: データベース接続 Address
- **MONGODB_DB**: データベース名



## Node.js環境



be-BOPはNode.jsで動作します。Node.js**のバージョンが18以上で、**Corepack**が有効になっていることを確認してください（pnpmのようなパッケージマネージャを管理するために必要です）。実行するコマンドは `corepack enable` です。



### Git LFSのインストール



一部のリソース (大きなイメージなど) は Git LFS (Large File Storage) で管理されます。あなたのマシンに Git LFS がインストールされていることを `git lfs install` コマンドで確認しましょう。これらの前提条件が整ったら、次のステップに進みましょう。be-BOP のダウンロードと設定です。



**注意:** ソフトウエアデプロイメントに関するテクニカルガイドは、別のチュートリアルでご覧いただけます。

## 通信設定



be-BOPが通知（注文、登録、システムメッセージなど）を送信できるようにするには、少なくとも1つの通信方法を設定する必要があります。電子メール（SMTP）またはNostr.Eメール（SMTP）の2つのオプションがあります。



### SMTP設定（電子メール）



be-BOPはSMTPサーバー経由で電子メールを送信できます。有効なSMTP認証情報が必要で、多くの場合、メールサービス（Mailgun、Gmailなど）から提供されます。



ここで知っておくべきことがある：



SMTP_HOST: SMTPサーバー Address (例: smtp.mailgun.org)



SMTP_PORT: 使用するポート (多くの場合 587 または 465)



SMTP_USER: ユーザー名 (通常は電子メールのAddress)



SMTP_PASSWORD: パスワードまたはAPIキー



SMTP_FROM: 送信者として表示される電子メール Address




### ノストルの構成



be-BOPを使うと、分散型メッセージング基盤であるNostrプロトコルで通知を送ることができます。これを行うには、Nostr秘密鍵（NSEC）をgenerateまたはSupplyする必要があります。この鍵はNostr専用のセクションにあるbe-BOPのInterfaceから直接generateできます。これらのElementsが正しく設定されると、be-BOPはユーザーにメッセージやアラートを自動的に送信できるようになります。



## 対応する支払い方法



be-BOPは複数の決済ソリューションと互換性があるため、お客様に柔軟性を提供することができます。ここでは、お客様に最適なお支払い方法を設定するために必要なものをご紹介します。



### Bitcoin オンチェーン



be-BOPは、Blockchain（On-Chain）で直接Bitcoinの支払いを簡単かつ安全に受け付けることができます。



**設定ステップ**





- **支払い設定**メニューに進む
- On-Chainの支払いパラメーターにアクセスするには、**Bitcoin Nodeless**をクリックしてください。
- 以下のフィールドに記入してください：



| Champ                  | Description                                               | Exemple à utiliser                              |
|------------------------|-----------------------------------------------------------|--------------------------------------------------|
| **BIP Standard**       | Le type d’adressage utilisé                               | BIP84 (pour les adresses au format bech32 commençant par `bc1`) |
| **Clé publique étendue** | Votre Zpub (ou Xpub selon le portefeuille utilisé)        | `zpub...` (extrait de votre portefeuille Bitcoin) |
| **Derivation Index**   | L’index de départ pour la génération des adresses         | `1`                                              |
| **Mempool URL**        | L’URL du service mempool utilisé pour suivre les transactions | `https://mempool.space`                         |

![payment-nodeless](assets/fr/005.webp)



**Bitcoin、Wallet（Sparrow wallet、BlueWallet、Specterなど）の詳細設定を参照してください。取引履歴を使用する場合は、Walletが読み取り専用でないことを確認してください。**



### Lightning Network



be-BOPは、Lightning Networkのおかげで、Bitcoinの即時支払いにも対応できます。現在、2つの設定オプションが用意されている：



**フェニックス**



支払い設定」メニューから「Phoenixd」をクリックします。



![phoenixd](assets/fr/006.webp)



PhoenixdはAcinqが開発したバックエンドで、Lightning決済を独自のノードで管理することができますが、決済チャネルを管理する複雑さはありません。



**スイスBitcoinペイ**。



Lightningノードを自分で管理したくない場合、**Swiss Bitcoin Pay**はすぐに使用でき、設定が簡単なソリューションで、複雑なインフラなしでLightning決済の受け入れを開始するのに理想的です。



設定手順 ：





- 支払い設定」メニューで「スイスBitcoinペイ」をクリックします。
- スイスのBitcoin Payアカウントにログインしてください（まだお持ちでない場合は作成してください）。
- スイスBitcoinペイから提供されたAPIキーを入力し、"保存 "をクリックします。



一度セットアップすれば、be-BOPは自動的にgenerateライトニング請求書を顧客に発行し、スイスBitcoinペイ口座に直接支払いを受けることができます。このソリューションは、個人ノードの技術的な複雑さを避けながら、迅速で低コストの支払いを受け入れたいユーザーに最適です。



![swissbtcpay](assets/fr/007.webp)

### サムアップ



このソフトウェアは、**SumUp**決済ソリューションを統合し、簡単、安全かつ効率的にクレジットカード決済を受け付けることができるようになりました。この機能を利用するには、初期設定が必要です。以下は、明確で漸進的な実装のために番号付けされた、従うべきステップです：





- 開発者アカウントの作成時にSumUpから提供された機密キーである**API Key**を入力して開始します。これは、お客様のSumUpアカウントとソフトウェア間の安全な接続を確立します。
- マーチャントコード欄には、サムアッププラットフォーム内で貴社を識別するための固有のコードを入力してください。このコードはお取引を関連付けるために必要不可欠です。
- Currency`フィールドで、取引に使用する主な通貨を選択します（例：**EUR**、**USD**、**CDF**など）。
- すべてのフィールドが正しく入力されたら、`Save`ボタンをクリックして設定を保存します。その後、システムがお客様のSumUpアカウントとのリンクを確立し、ソフトウェアが支払いを受け付ける準備が整います。



![payment-sumup](assets/fr/009.webp)



この設定を行うと、**SumUp**との統合が有効になり、ソフトウェアから直接、迅速なキャッシュアウトや取引の追跡ができるようになります。

- Stripeダッシュボードで提供された**秘密キー**を入力します。
- 同じくStripeから提供された**公開鍵**フィールドを入力します。
- 主要通貨を選択してください。
- 設定を保存し、`Save`をクリックします。



![payment-stripe](assets/fr/010.webp)



⚠️ **ご注意ください：** **be-BOP**の請求書発行オプションを正しく設定するためには、お客様の活動に適用されるVAT制度（例：売り手の国のVATによる販売、正当な理由による免税、または買い手の国のVAT税率による販売）を知ることが不可欠です。

## Interfaceとbe-BOPのアイデンティティ設定



**be-BOP**はデザイナーがウェブサイトをデザインするために必要なすべてのツールを提供します。最初のステップは、設定の`/Admin > Merch > Layout`セクションを開くことです。**トップバー**、**ナビゲーションバー**、**フッター**の設定から始めます。



### ル・トップ・バー



トップバー**構成**では、Interfaceの最初の行に重要な情報を表示することで、ソフトウェアのビジュアル・アイデンティティをパーソナライズすることができます。これは、ブランド認知を強化し、ユーザーに明確なコンテキストを提供します。



#### 設定手順 ：





- ブランド名」の欄には、会社名、組織名、製品名を入力してください。この名前はInterfaceの上部に表示され、あなたのメインビジュアルアイデンティティを表します。
- ウェブサイトのタイトルを示す：選択したタイトルは、プラットフォームの目的を要約する必要があります。このタイトルは、ヘッダーまたはブラウザーのタブに表示することができます。
- **ウェブサイトの説明を追加**：ここでは、あなたのイニシアチブについての簡単な説明を入力します。この説明は、ユーザーにとってツールの文脈を理解するのに役立ち、SEOの目的にも使用できます。



この情報が入力されると、**トップ・バー**に、あなたのソリューションが明確でプロフェッショナルかつ首尾一貫したプレゼンテーションとして表示されます。



#### トップバーのリンク



トップバーの「リンク」セクションでは、アプリケーションや外部サイトの重要なページへのショートカットを追加することができます。これらのリンクはトップバーに直接表示され、ユーザーに高速で構造化されたアクセスを提供します。



#### 設定手順 ：





- **Enter link name (Text)**: `Text`フィールドに、表示されるリンクの名前またはラベルを入力します（例：Home、Contact、Help...）。
- **リンクAddress (Url)**: `Url`フィールドに、ターゲットページ（内部または外部）の完全なAddressを入力してください。
- **必要であれば他のリンクも追加してください**: 各設定行では `Text` と `Url` フィールドを使って追加のリンクを追加することができます。
- リンクの保存：すべてのリンクを入力したら、「トップバーにリンクを追加」ボタンをクリックして保存します。



この構成により、ウェブサイトのさまざまなセクションや補完的なリソースを通じて、明確で流動的かつアクセスしやすいナビゲーションを提供することができます。



![settings-topbar](assets/fr/014.webp)



### ラ・ナビ・バー



**Navbar**セクションでは、be-BOPのメインナビゲーションメニューを設定することができます。このメニューは、ユーザーをアプリケーションの様々なページや機能に誘導します。リンクの設定は簡単で直感的です。以下はその方法です：





- リンク名の入力 (`Text`): 設定行で、まず `Text` フィールドに入力します。これはナビゲーションバーに表示されるリンクの名前に相当します(例: *Dashboard*、*Users*、*Settings*...)。
- リンクのAddress (`Url`) を入力してください: `Text` フィールドの隣に `Url` フィールドがあります。このフィールドに、リンクのリダイレクト先となるページのAddressを入力してください。これは内部ルートでも外部ページへのリンクでもかまいません。
- 必要であれば複数のリンクを追加してください：最初の行の下に、新しい`テキスト`と`URL`フィールドがあり、必要なだけリンクを追加することができます。各行は追加のナビゲーションリンクを表します。
- **リンクの保存**：すべてのElementsを入力したら、`ナビバーリンクの追加`ボタンをクリックして保存し、結果をナビゲーションバーに表示します。



この構成により、ソフトウェアのさまざまな部分へのアクセスが効率的に構成され、人間工学とユーザーエクスペリエンスが向上します。



![navbar](assets/fr/015.webp)



### フッター



フッター**セクション**では、ソフトウェアのフッターをカスタマイズし、有用な情報やリンクを追加することができます。リンクを設定する前に、特定のオプションを有効にすることから始めます：





- Powered by be-BOP **ラベルの表示を有効にする：フッターにこのラベルを表示するには、`Display Powered by be-BOP`ボタンを有効にします。**
- **リンクの名前(`Text`)を入力する**: フッターのリンクの文言に対応する`Text`フィールドに入力します(例: *Terms*、*Privacy*、*Contact*...)。
- リンクのAddress (`Url`): `Url`フィールドに、ターゲットページ（内部または外部）のAddressを入力する。
- 必要であれば、さらにリンクを追加してください：追加行を使用して、お好きなだけリンクを作成してください。
- リンクの保存：「フッターリンクを追加」ボタンをクリックしてリンクを保存します。
