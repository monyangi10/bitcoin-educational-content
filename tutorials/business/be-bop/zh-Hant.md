**be-BOP**是一個電子商務平台，專為希望完全自主地進行線上和線下銷售的企業家而設計，同時接受 Bitcoin、銀行帳戶和現金付款。此解決方案也適用於希望募集捐款或將其各種活動貨幣化的任何類型組織。



此解決方案簡單、輕便且自主。即使在傳統金融服務有限或不存在的環境中，也能建立線上商店。事實上，**be-BOP** 已被設計成可在有或沒有銀行的情況下有效運作，並使用 Bitcoin 作為支付基礎架構。



在本教程中，我們將逐步帶您瞭解 .NET 的使用方法：





- 使用 **be-BOP** 創建您的第一個網路商店
- 個人化您的展示架和產品
- 設定可用的付款方式
- 瞭解使用 **be-BOP** 進行有效線上銷售的最佳做法



本教學不需要進階的技術技能。它的目標對象是開發人員，以及希望以主權和彈性方式展開數位商務的工匠、商人、合作社或企業家。



## 在您自己的伺服器上安裝 be-BOP 的先決條件



在您開始安裝 be-BOP 之前，請確定您已具備下列技術基礎架構。這些 Elements 是平台正常運作的必要條件：



### 與 S3 相容的儲存空間



be-BOP 使用儲存系統管理檔案 (例如產品圖片)。這需要存取 S3 服務，例如 .NET Framework：





- [MinIO](https://min.io/) 自行管理
- 亞馬遜 S3 (AWS)
- Scaleway 物件儲存



您需要設定一個儲存桶，並提供下列資訊：





- S3_BUCKET**：水桶名稱
- S3_ENDPOINT_URL**：S3 服務的存取連結
- S3_KEY_ID** 和 S3_KEY_SECRET：您的存取密碼
- S3_REGION**：您的 S3 服務的區域



### 在 ReplicaSet 模式下的 MongoDB 資料庫



be-BOP 使用 MongoDB 來儲存商店、使用者、產品和其他資料。



您有兩個選擇：





- 在本機安裝 MongoDB，並啟用 **ReplicaSet 模式**
- 使用線上服務，例如 **MongoDB Atlas**



您將需要下列變數：





- MONGODB_URL**: 資料庫連線 Address
- MONGODB_DB**: 資料庫名稱



## Node.js 環境



be-BOP 可與 Node.js 搭配使用。請確定您的 **Node.js** 版本為 18 或更高，且 **Corepack** 已啟用 (需要用來管理套件管理員，例如 pnpm)。執行的指令是 `corepack enable` 。



### 已安裝 Git LFS



有些資源（例如大圖像）是透過 Git LFS（大檔案儲存）來管理的。請使用 `git lfs install` 指令確認您的機器已經安裝 Git LFS。一旦這些先決條件就緒，您就可以進入下一步：下載並配置 be-BOP。

## 通訊設定



要啟用 be-BOP 傳送通知 (例如訂單、註冊或系統訊息)，您需要設定至少一種通訊方式。有兩種方式可供選擇：電子郵件 (SMTP) 或 Nostr.



### SMTP 設定 (電子郵件)



be-BOP 可以透過 SMTP 伺服器傳送電子郵件。您需要有效的 SMTP 認證，通常由電子郵件服務 (例如 Mailgun、Gmail 等) 提供。



以下是您需要知道的資訊：



SMTP_HOST: SMTP 伺服器 Address (例如 smtp.mailgun.org)



SMTP_PORT: 使用的連接埠（通常為 587 或 465）



SMTP_USER: 您的使用者名稱 (通常是電子郵件 Address)



SMTP_PASSWORD: 您的密碼或 API 金鑰



SMTP_FROM：將顯示為寄件者的電子郵件 Address




### Nostr 設定



be-BOP 可讓您透過 Nostr 通訊協定 (一種分散式訊息基礎架構) 傳送通知。要做到這一點，您需要 generate 或 Supply Nostr 私鑰 (NSEC)。您可以透過 be-BOP 的 Interface 直接 generate 此金鑰，請參閱 Nostr 專區。當這些 Elements 被正確設定後，be-BOP 將能夠自動傳送訊息和警報給您的使用者。



## 相容的付款方式



be-BOP與多種付款解決方案相容，讓您為客戶提供更大的靈活性。以下是您需要設定最適合您的付款方式。



### Bitcoin Onchain



be-BOP 可讓您直接在 Blockchain (On-Chain) 上接受 Bitcoin 付款，簡單又安全。



**設定步驟：**





- 前往 ** 付款設定** 功能表
- 按一下 **Bitcoin Nodeless** 存取 On-Chain 付款參數。
- 完成下列欄位：



| Champ                  | Description                                               | Exemple à utiliser                              |
|------------------------|-----------------------------------------------------------|--------------------------------------------------|
| **BIP Standard**       | Le type d’adressage utilisé                               | BIP84 (pour les adresses au format bech32 commençant par `bc1`) |
| **Clé publique étendue** | Votre Zpub (ou Xpub selon le portefeuille utilisé)        | `zpub...` (extrait de votre portefeuille Bitcoin) |
| **Derivation Index**   | L’index de départ pour la génération des adresses         | `1`                                              |
| **Mempool URL**        | L’URL du service mempool utilisé pour suivre les transactions | `https://mempool.space`                         |

![payment-nodeless](assets/fr/005.webp)



**Tip:** 要取得您的擴充公開金鑰 (Zpub)，您可以參考您的 Bitcoin Wallet (Sparrow wallet、BlueWallet、Specter 等) 的進階設定。如果您打算使用交易記錄，請確定您的 Wallet 是 **非唯讀**。



### Lightning Network



be-BOP也可透過Lightning Network接受即時Bitcoin付款。目前提供兩種配置選項：



**Phoenixd**



前往「付款設定」功能表，按一下「Phoenixd」。



![phoenixd](assets/fr/006.webp)



然後，您需要輸入**密碼或 token 認證**，以連結到您的 Phoenixd 實例，這是 Acinq 開發的後端，可讓您使用自己的節點管理 Lightning 付款，但不需要管理付款管道的複雜性。



**Swiss Bitcoin Pay**



如果您不想自己管理 Lightning 節點，**Swiss Bitcoin Pay** 是一套即開即用、容易設定的解決方案，非常適合在沒有複雜基礎架構的情況下開始接受 Lightning 付款。



設定步驟 ：





- 在「付款設定」功能表中，按一下「瑞士 Bitcoin 付款」。
- 登入您的 Swiss Bitcoin Pay 帳戶（若您沒有帳戶，請建立一個）。
- 輸入 Swiss Bitcoin Pay 提供的 API 金鑰，然後按一下「儲存」。



一旦設定完成，be-BOP 將自動為您的客戶開出 generate Lightning 發票，而您的瑞士 Bitcoin Pay 帳戶則會直接收到付款。此解決方案非常適合想要避免個人節點的複雜技術，同時接受快速、低成本付款的使用者。



![swissbtcpay](assets/fr/007.webp)

### 總結



本軟體目前整合了 **SumUp** 付款解決方案，讓您可以簡單、安全且有效率地接受信用卡付款。若要受惠於此功能，需要進行初始設定。以下是需要遵循的步驟，以編號表示，可讓您清楚且逐步地實作：





- 首先輸入您的 **API Key**，這是 SumUp 在您建立開發者帳號時提供的機密金鑰。它會在您的 SumUp 帳戶和軟體之間建立安全連線。

- 正確填寫所有欄位後，點擊 「保存 」按鈕保存您的設置。然後，系統會與您的 SumUp 帳戶建立連結，您的軟體就可以接受付款了。

完成此設定後，**SumUp** 整合將會啟動並運作，讓您可以直接從軟體快速套現並追蹤交易。

### Le Top 酒吧



**Top Bar** 配置可讓您在 Interface 的第一行直接顯示關鍵資訊，從而個人化軟體的視覺識別。這可強化品牌辨識度，並為使用者提供清晰的情境。
