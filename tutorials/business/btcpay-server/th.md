BTCPay Server เป็นซอฟต์แวร์แพ็คเกจโอเพ่นซอร์สที่ฟรี สร้างขึ้นโดย Nicolas Dorier ที่ช่วยให้การชำระเงินด้วยบิตคอยน์สามารถทำได้โดยไม่ต้องมีตัวกลาง ออกแบบมาเพื่อมอบความเป็นอิสระและอธิปไตยทางการเงิน ติดตั้งบนเซิร์ฟเวอร์ของตัวเองและให้การจัดการธุรกรรมอย่างครบถ้วน ตั้งแต่การออกใบแจ้งหนี้ไปจนถึงการตรวจสอบการชำระเงิน on-chain หรือ Lightning Network และการบัญชี สามารถผสานรวมได้ง่ายกับเว็บไซต์อีคอมเมิร์ซ (WooComerce, Shopify, ฯลฯ) หรือสามารถใช้เป็นเครื่องรับชำระเงินสำหรับร้านค้าจริง (*POS*)



BTCPay Server เป็นโซลูชันที่ทันสมัยที่สุดสำหรับพ่อค้าที่ต้องการรับบิตคอยน์อย่างไม่ต้องสงสัย มันเป็นซอฟต์แวร์ที่ครอบคลุมและแข็งแกร่งที่สุดในแง่ของความปลอดภัย อธิปไตย และความลับ ในทางกลับกัน มันก็เป็นซอฟต์แวร์ที่ซับซ้อนที่สุดในการติดตั้งและบำรุงรักษา นอกจากนี้ยังมีทางเลือกที่ง่ายกว่า: บางตัวเป็นแบบดูแลทั้งหมด เช่น OpenNode ในขณะที่บางตัวเสนอการประนีประนอมที่น่าสนใจระหว่างความง่ายในการใช้งานและอธิปไตย เช่น Swiss Bitcoin Pay :



https://planb.academy/tutorials/business/point-of-sale/swiss-bitcoin-pay-2-a78b057e-ed11-47ac-860c-71019fcb451a

วัตถุประสงค์ของบทแนะนำนี้คือการแนะนำคุณทีละขั้นตอนผ่านการติดตั้ง การกำหนดค่า และการใช้งาน BTCPay Server เพื่อให้คุณสามารถปรับใช้โครงสร้างพื้นฐานการชำระเงินที่ปลอดภัยและเป็นอิสระตามหลักการของ Bitcoin



## คุณสมบัติของ BTCPay Server



โซลูชันจุดขายแบบรวมศูนย์ Bitcoin เช่น *OpenNode* เป็นต้น เสนอความสะดวกในการใช้งาน แต่ต้องพึ่งพาบริษัทบุคคลที่สามเนื่องจากไม่สามารถโฮสต์ด้วยตนเองได้และในกรณีส่วนใหญ่เป็นกรรมสิทธิ์ แม้ว่าจะทำให้การตั้งค่าการชำระเงินง่ายขึ้น แต่ก็มีค่าคอมมิชชั่นและทำให้ผู้ใช้เสี่ยงมากกว่าการใช้โซลูชันอย่าง BTCPay Server ทั้งในแง่ของการดูแลเงินทุนและความลับ



BTCPay Server มุ่งเป้าไปที่พ่อค้าออนไลน์หรือร้านค้าจริง สมาคม และองค์กรไม่แสวงหาผลกำไรที่ต้องการรับบริจาคเป็นบิตคอยน์ นอกจากนี้ยังเป็นโซลูชันที่เหมาะสำหรับเจ้าของโครงการและนักพัฒนาที่ต้องการการสนับสนุนโดยตรงจากชุมชนของพวกเขา



คุณสมบัติพิเศษของ BTCPay Server ได้แก่ :




- การ **ปกครองตนเองอย่างสมบูรณ์**
- การขาดขั้นตอน **KYC**,
- ควบคุมเงินทุนอย่างเต็มที่**,
- การ **กำจัดค่าธรรมเนียมแพลตฟอร์ม**.



ด้วยการเป็นผู้ประมวลผลการชำระเงินของคุณเอง คุณจะขจัดการพึ่งพาบุคคลที่สามที่มีศูนย์กลางระหว่างคุณและลูกค้าของคุณ คุณสามารถรับชำระเงินโดยตรงในรูปแบบบิตคอยน์และใบแจ้งหนี้การชำระเงิน generate ซึ่งจะช่วยให้มั่นใจได้ว่าทั้งคุณและบริษัทของคุณจะไม่ถูกแบนโดยใครก็ตาม คุณทำหน้าที่เป็นทั้งธนาคารและผู้ประมวลผลการชำระเงิน โดยไม่ต้องจ่ายค่าคอมมิชชั่นให้กับคนกลางสำหรับแต่ละธุรกรรม



ค่าธรรมเนียมการทำธุรกรรมสำหรับ **on-chain** ยังคงอยู่ แต่สามารถลดลงได้โดยการใช้เครือข่าย **Liquid** หรือ **Lightning**



นอกจากนี้ :




- อินเทอร์เฟซและใบแจ้งหนี้ที่ปรับแต่งได้อย่างเต็มที่;
- รองรับ **Tor** ในตัวเพื่อเพิ่มความเป็นส่วนตัว;
- รองรับ **การระดมทุนออนไลน์**, **POS** และ **ปุ่มชำระเงิน**;
- เข้ากันได้กับหลายสกุลเงิน ;
- Bitcoin การชำระเงินโดยตรงและแบบเพียร์ทูเพียร์ ;
- ควบคุมคีย์ส่วนตัวของคุณได้อย่างสมบูรณ์;
- ความเป็นส่วนตัวที่เพิ่มขึ้น ;
- การรักษาความปลอดภัยที่เพิ่มขึ้น ;
- ซอฟต์แวร์ที่โฮสต์เอง ;
- รองรับ **SegWit** และ **Lightning network** ;
- พอร์ตโฟลิโอภายในที่ใช้โหนด โดยมีการผสานรวมกับพอร์ตโฟลิโอฮาร์ดแวร์



## การติดตั้งและกำหนดค่า BTCPay Server



### การเลือกโหมดการโฮสต์ของคุณ



BTCPay Server สามารถติดตั้งได้หลายวิธี ขึ้นอยู่กับความต้องการและทรัพยากรของคุณ มีตัวเลือกหลักสามตัวเลือก:




- BTCPay Server โฮสต์โดยบุคคลที่สาม**: คุณใช้แพลตฟอร์มที่โฮสต์บริการให้คุณ มันง่าย แต่โดยปกติจะไม่ฟรี
- BTCPay Server โฮสต์ด้วยตนเองบนเซิร์ฟเวอร์คลาวด์** (เช่น ผ่าน [btcpayprovider](https://btcpayprovider.com/), [Bitcoin People](http://bitcoinpeople.it/) หรือผู้ให้บริการรายอื่นๆ) นี่คือโซลูชันที่แนะนำสำหรับพ่อค้าใหม่ส่วนใหญ่
- BTCPay Server ติดตั้งบนฮาร์ดแวร์ของคุณเอง (ในเครื่อง)** : บนคอมพิวเตอร์, mini-PC หรือ Umbrel วิธีนี้มีความเทคนิคมากกว่า แต่ให้ความเป็นอิสระอย่างสมบูรณ์



https://planb.academy/tutorials/business/point-of-sale/btcpay-server-umbrel-68e1c535-4322-4507-a69c-9dfcbc36dfd1

สำหรับผู้ประกอบการเริ่มต้น **การปรับใช้บนเซิร์ฟเวอร์คลาวด์** เป็นการประนีประนอมที่ดีที่สุดระหว่างความเป็นอิสระ ความเรียบง่าย และความปลอดภัย โดยไม่ต้องจัดการโครงสร้างพื้นฐานทางเทคนิคทั้งหมด



BTCPay Server สามารถปรับใช้ได้อย่างรวดเร็วจากผู้ให้บริการโฮสติ้งหลายราย ในหมู่พวกเขา **Voltage** โดดเด่นในฐานะโซลูชันมาตรฐานสำหรับผู้ใช้ที่ต้องการโครงสร้างพื้นฐานที่ **เชื่อถือได้**, **มืออาชีพ** และ **อธิปไตย**



### สร้างอินสแตนซ์ BTCPay Server บน Voltage



**Voltage** เป็นแพลตฟอร์มโฮสติ้ง Bitcoin และ Lightning แบบครบวงจรที่ช่วยให้คุณสามารถติดตั้งเซิร์ฟเวอร์ BTCPay ของคุณเองได้ทันที โดยไม่ต้องมีการตั้งค่าที่ซับซ้อนหรือการบำรุงรักษาเซิร์ฟเวอร์



เยี่ยมชม[เว็บไซต์ทางการของ Voltage](https://voltage.cloud)



![capture](assets/fr/03.webp)



สร้าง**บัญชีผู้ใช้**ด้วยที่อยู่อีเมลที่ถูกต้องและรหัสผ่านที่แข็งแรง



![capture](assets/fr/04.webp)



Voltage มีการทดลองใช้ฟรี **7 วัน** โปรดทราบว่าหลังจากการทดลองใช้ฟรี 7 วัน คุณจะได้รับเชิญให้สมัครสมาชิกแบบคงที่ในราคา **$25 ต่อเดือน** เพื่อดำเนินการ **รักษาโหนดของคุณให้ทำงานต่อไป**



เมื่อคุณสร้างบัญชี Voltage และเข้าสู่ระบบเป็นครั้งแรก คุณจะถูกเปลี่ยนเส้นทางไปยังหน้าแรก ซึ่งมีสองส่วนหลัก:




- ส่วน **โครงสร้างพื้นฐาน** สำหรับการจัดการโหนด Lightning, Bitcoin Core, BTCPay Server และบริการอื่น ๆ ของ Bitcoin ในระบบคลาวด์;
- และส่วน **Payments** ที่ให้คุณเข้าถึง Voltage's API Lightning เพื่อรวมการชำระเงิน Bitcoin เข้ากับแอปพลิเคชันที่ปรับแต่งเอง



ในการปรับใช้อินสแตนซ์ **BTCPay Server** ของคุณ ให้คลิกที่ **Access infrastructure** ที่นี่คุณสามารถสร้าง จัดการ และตรวจสอบบริการทั้งหมดของคุณ รวมถึงโหนด Bitcoin และ BTCPay Server ของคุณ



#### สร้างกลุ่ม



ก่อนที่คุณจะสามารถปรับใช้บริการได้ แพลตฟอร์มจะเตือนให้คุณ **สร้างกลุ่ม**. **กลุ่ม** (workspace) คือพื้นที่ทำงานที่รวบรวมบริการ Bitcoin และ Lightning ทั้งหมดของคุณ (เช่น โหนด, BTCPay Server, explorer, ฯลฯ) มันคล้ายกับโฟลเดอร์ที่บรรจุโปรเจกต์ต่างๆ ของคุณ.



![capture](assets/fr/05.webp)



สำหรับวัตถุประสงค์ของบทแนะนำนี้ กลุ่มที่เราได้สร้างขึ้นจะเรียกว่า **Genesis** คุณสามารถเพิ่มรูปโปรไฟล์ได้หากต้องการ เมื่อเสร็จแล้ว ให้คลิกที่ปุ่ม **Create** คุณสามารถเชิญผู้ใช้อื่นเข้าร่วมกลุ่ม และแม้กระทั่งเปลี่ยนชื่อกลุ่มได้หากต้องการ



ในหน้าแรกของกลุ่ม จะมีตัวเลือกหลายตัวเลือกปรากฏขึ้น:




- Lightning Nodes** : เพื่อปรับใช้โหนด Lightning แบบสมบูรณ์ (LND) ;
- Bitcoin Core Nodes** : เพื่อเปิดตัวโหนด Bitcoin ที่สมบูรณ์ ;
- BTCPay Server** : เพื่อโฮสต์อินสแตนซ์ BTCPay ที่พร้อมใช้งาน;
- บัญชี Nostr**: เพื่อจัดการตัวตน Nostr



เปิดใช้งาน **การยืนยันตัวตนสองขั้นตอน (2FA)** เพื่อรักษาความปลอดภัยให้กับบัญชีและบริการของคุณ (ปุ่มจะปรากฏเป็นสีแดงหากไม่ได้เปิดใช้งาน)



![capture](assets/fr/06.webp)



คลิกที่ **BTCPay Server** ในบรรดาตัวเลือก จากนั้นคลิกที่ **Launch a BTCPay Store**.



![capture](assets/fr/07.webp)



จากนั้น Voltage จะขอให้คุณ **สร้างและกำหนดค่า BTCPay Server instance ของคุณ** โดยการเลือก **ชื่อบริการ** (1) และเปิดใช้งานการชำระเงินด้วย Lightning (2)



คุณจะต้องมีโหนด Lightning หากคุณตัดสินใจที่จะรับการชำระเงินผ่าน Lightning



หากคุณยังไม่มี Bitcoin หรือโหนด Lightning, Voltage จะแนะนำให้คุณสร้างขึ้นโดยอัตโนมัติ



คลิกที่ **สร้างโหนด Lightning** (3) .



![capture](assets/fr/08.webp)



เมื่ออยู่ในอินเทอร์เฟซการสร้างโหนด คุณจะถูกขอให้เลือกระหว่างรูปแบบ **มาตรฐาน** และ **มืออาชีพ**



คุณสามารถสร้างโหนดจริง (**Mainnet**) หรือโหนดทดสอบ (**Testnet**) จากนั้นคลิกที่ปุ่ม **Continue**



![capture](assets/fr/09.webp)



สำหรับบทแนะนำนี้ ให้เราใช้แผนมาตรฐาน ป้อน **ชื่อโหนด**, **รหัสผ่าน** และกดปุ่ม **สร้าง**



![capture](assets/fr/10.webp)

คุณต้องเลือก **วิธีการนำเข้า** ของคุณ จากตัวเลือกที่มีอยู่ เลือก **ป้อนคีย์สาธารณะขยาย**

เมื่อคุณคลิกที่ปุ่ม **Enter extended public key** คุณจะถูก **เปลี่ยนเส้นทาง** ไปยังหน้าที่คุณต้องระบุคีย์นี้



ตอนนี้ให้เราดึงข้อมูล **extended public key** ออกมา

### สร้างระบบขายหน้าร้าน (PoS)



เมื่อคุณตั้งค่าร้านค้าและพอร์ตโฟลิโอของคุณเรียบร้อยแล้ว คุณสามารถตั้งค่า **จุดขาย (PoS)** เพื่อเริ่มรับการชำระเงิน Bitcoin โดยตรงจากลูกค้าของคุณได้แล้ว



ฟีเจอร์ที่รวมอยู่ใน **BTCPay Server** นี้เหมาะสำหรับ **พ่อค้า, ช่างฝีมือ, เจ้าของร้านอาหาร หรือผู้ให้บริการ** ที่ต้องการรับ Bitcoin **โดยไม่ต้องมีเว็บไซต์** หรือทักษะทางเทคนิคพิเศษ



### จุดขายคืออะไร?



**PoS** คือ **อินเทอร์เฟซ POS ที่เรียบง่าย** ซึ่งทำหน้าที่เป็น **เทอร์มินัลการชำระเงิน Bitcoin**


มันช่วยให้คุณสามารถ:




- สร้าง**เมนูสินค้าและบริการ**พร้อมราคาคงที่
- สร้าง**ใบแจ้งหนี้ทันทีพร้อมรหัส QR** สำหรับสแกน
- แชร์ **URL การชำระเงิน** ที่สามารถเข้าถึงได้ผ่านสมาร์ทโฟน แท็บเล็ต หรือคอมพิวเตอร์



กล่าวอีกนัยหนึ่ง PoS จะเปลี่ยน BTCPay Server ของคุณให้เป็น **อินเทอร์เฟซการขายโดยตรง** ซึ่งทุกการชำระเงินจะได้รับ **ใน Bitcoin wallet ของคุณเอง** โดยไม่มีตัวกลางใดๆ



### กำลังตั้งค่า PoS



ในแดชบอร์ด BTCPay คลิกที่ **PLUGINS** จากนั้นคลิกที่ **Point of sale**.



จากนั้นคลิกที่ **สร้างแอปพลิเคชัน PoS ใหม่** การดำเนินการนี้จะเพิ่ม **แอปพลิเคชันใหม่** ไปยังร้าน BTCPay ของคุณ ราวกับว่าคุณกำลังติดตั้งเว็บไซต์ขายภายในขนาดเล็ก



เปิดหน้าเพื่อสร้างแอปพลิเคชันของคุณ กำหนด **ชื่อแอปพลิเคชัน**: นี่คือชื่อภายในที่มองเห็นได้เฉพาะจากแดชบอร์ดของคุณ (เช่น _Shoe Shop_)



คลิกที่ **Create** เพื่อยืนยัน



![capture](assets/fr/24.webp)



เมื่อสร้างเสร็จแล้ว คุณจะถูกเปลี่ยนเส้นทางไปยัง **หน้าการกำหนดค่ารายละเอียด** ของจุดขาย



### ปรับแต่งอินเทอร์เฟซการขายของคุณ



ในหน้านี้ คุณสามารถกำหนดองค์ประกอบที่จำเป็นของ PoS ของคุณ:





- ชื่อแอปพลิเคชัน** (ชื่อสำหรับการจัดการภายใน สามารถเปลี่ยนแปลงได้ตลอดเวลา)





- แสดงชื่อเรื่อง** (สิ่งที่ลูกค้าของคุณจะเห็นที่ด้านบนของหน้า)





- รูปแบบจุดขาย** (**โหมดคำอธิบาย** เหมาะสำหรับบริการต่างๆ เช่น การตัดผมหรือมื้ออาหาร ในขณะที่ **โหมดแคตตาล็อกสินค้า** เหมาะสำหรับร้านค้าที่มีสินค้าหลายรายการ)





- แสดงสกุลเงิน** (เลือก **XOF**, **EUR** หรือ **USD** ตามราคาปกติของคุณ)





- รายการสินค้า** (เพิ่มสินค้าของคุณ คำอธิบายและราคาไว้ที่นี่)

## ความปลอดภัยและแนวทางปฏิบัติที่ดีที่สุด



อิสระที่ BTCPay Server มอบให้ (อำนาจอธิปไตยเต็มรูปแบบเหนือเงินทุนของคุณ) เป็นจุดแข็งที่แท้จริง แต่ด้วยเสรีภาพนี้มาพร้อมกับความรับผิดชอบที่มากขึ้นในด้านความปลอดภัย โดยการจัดการการชำระเงินของคุณเอง คุณจะรับบทบาทเป็นธนาคารของคุณเอง นั่นคือเหตุผลที่จำเป็นต้องนำแนวปฏิบัติที่ดีที่สุดมาใช้เพื่อปกป้องเงินทุน ข้อมูล และโครงสร้างพื้นฐานของคุณ นี่คือประเด็นหลักที่ควรพิจารณา



### เข้าถึงเซิร์ฟเวอร์ของคุณอย่างปลอดภัย





- ใช้รหัสผ่านที่แข็งแกร่ง: ผสมผสานตัวอักษรพิมพ์ใหญ่และพิมพ์เล็ก ตัวเลข และอักขระพิเศษ หลีกเลี่ยงการใช้รหัสผ่านเดิมซ้ำ
- เปิดใช้งานการยืนยันตัวตนสองปัจจัย (2FA) เพื่อเข้าถึงอินเทอร์เฟซ BTCPay ของคุณ
- อัปเดตระบบปฏิบัติการของคุณ, อินสแตนซ์ BTCPay Server และการพึ่งพาของคุณ (Docker, Bitcoin node, Lightning node...) เป็นประจำ การอัปเดตมักจะแก้ไขช่องโหว่ด้านความปลอดภัย



### การจัดการและสำรองข้อมูลคีย์ส่วนตัว





- บันทึกกุญแจส่วนตัวและวลีเมล็ดพันธุ์ของคุณแบบออฟไลน์ บนสื่อทางกายภาพ (กระดาษหรือโลหะ)
- ควรใช้ฮาร์ดแวร์ wallet wallet.
- เก็บสำเนาสำรองของคุณหลาย ๆ ชุดไว้ในสถานที่ทางกายภาพที่แยกจากกันและได้รับการป้องกัน



### การชำระเงินที่ปลอดภัยและการรักษาความลับ





- ใช้เครือข่าย Tor หรือ VPN เพื่อซ่อนที่อยู่ IP ของเซิร์ฟเวอร์ของคุณและปกป้องความเป็นส่วนตัวของคุณ
- ปิดใช้งานพอร์ตที่ไม่จำเป็นบนเซิร์ฟเวอร์ของคุณและจำกัดการเชื่อมต่อ SSH เฉพาะที่อยู่ที่เชื่อถือได้เท่านั้น
- เปิดใช้งาน HTTPS (ใบรับรอง SSL) สำหรับการเชื่อมต่อทั้งหมดไปยังอินเทอร์เฟซเว็บ BTCPay ของคุณ
- อย่าแชร์อินเทอร์เฟซการจัดการของคุณกับบุคลากรที่ไม่ได้รับการฝึกอบรมในการจัดการพอร์ตโฟลิโอ Bitcoin



### การนำแนวปฏิบัติที่ดีที่สุดสำหรับเครือข่าย Lightning มาใช้



ร้านค้าของคุณเชื่อมต่อกับ **โหนด Lightning ที่โฮสต์บน Voltage** ซึ่งช่วยลดความซับซ้อนในการจัดการทางเทคนิคของเครือข่ายได้อย่างมาก อย่างไรก็ตาม ยังคงมีความสำคัญที่จะต้องนำ **แนวทางการตรวจสอบและความปลอดภัยที่ดี** มาใช้:





- บันทึกข้อมูลการเข้าสู่ระบบและคีย์การเข้าถึง API** ของโหนด Voltage ของคุณ (ซึ่งช่วยให้ BTCPay เชื่อมต่อได้)
- ปกป้องบัญชี Voltage ของคุณ** ด้วยการยืนยันตัวตนสองขั้นตอนและรหัสผ่านที่แข็งแกร่ง
- ตรวจสอบสถานะโหนดและช่อง** จากแดชบอร์ด Voltage ของคุณ: สิ่งนี้ช่วยให้คุณสังเกตเห็นความผิดปกติหรือการไม่ซิงโครไนซ์ใด ๆ
- หลีกเลี่ยงการแชร์ข้อมูลรับรอง API** ของคุณหรือเปิดเผยต่อสาธารณะ (เช่น ในโค้ดของเว็บไซต์)
- ในกรณีที่มีการย้ายถิ่นฐาน เพียงแค่ **กำหนดค่าลิงก์ระหว่าง BTCPay และ Voltage ใหม่**: ไม่จำเป็นต้องปิดช่องทางด้วยตนเอง



ด้วย Voltage คุณจะได้รับประโยชน์จากโครงสร้างพื้นฐานที่**ปลอดภัย มีความพร้อมใช้งานสูง** และ**สำรองข้อมูลอัตโนมัติ** ในขณะที่ยังคง**อำนาจอธิปไตยเต็มรูปแบบเหนือการชำระเงิน Lightning ของคุณ**



### จัดระเบียบและโครงสร้างขั้นตอนภายใน





- กำหนดนโยบายการจัดการการเข้าถึงที่ชัดเจน: ใครสามารถสร้างใบแจ้งหนี้ ดูการชำระเงิน เข้าถึงโหนด ฯลฯ
- จัดทำเอกสารขั้นตอนการสำรองข้อมูลและการกู้คืนของคุณเพื่อให้คุณสามารถตอบสนองได้อย่างรวดเร็วในกรณีที่เกิดเหตุการณ์ไม่คาดคิด
- ทดสอบการกู้คืนข้อมูลสำรองของคุณเป็นประจำเพื่อให้แน่ใจว่าทำงานได้อย่างถูกต้อง
- ฝึกอบรมพนักงานหรือผู้ร่วมงานของคุณในด้านความปลอดภัยในการปฏิบัติงานขั้นพื้นฐาน: การเฝ้าระวังการฟิชชิง, การใช้รหัสผ่านที่ปลอดภัย, การเคารพความลับ.



### ดูแลและจัดตั้งการบำรุงรักษาอย่างต่อเนื่อง





- ตรวจสอบกิจกรรมของเซิร์ฟเวอร์ของคุณอย่างต่อเนื่องโดยใช้เครื่องมือการบันทึกหรือการตรวจสอบ
- กำหนดการตรวจสอบความปลอดภัยเป็นระยะ: ตรวจสอบการอัปเดต การเข้าถึง การสำรองข้อมูล และความสอดคล้องของธุรกรรม
