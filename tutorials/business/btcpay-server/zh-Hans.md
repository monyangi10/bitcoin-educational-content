## BTCPay 服务器功能



集中式 Bitcoin 销售点解决方案（如*OpenNode*）虽然使用方便，但由于无法自托管，而且在大多数情况下是专有的，因此依赖于第三方公司。虽然这些解决方案更容易设置支付，但它们涉及佣金费用，而且在资金保管和保密方面，用户面临的风险比 BTCPay Server 等解决方案更大。



BTCPay 服务器面向希望接收比特币捐款的在线或实体商家、协会和非营利组织。它也是项目所有者和开发者寻求社区直接支持的理想解决方案。



BTCPay 服务器的特殊功能包括 ：




- 其**完全自主**、
- 没有**KYC**程序、
- 完全控制资金**、
- 取消**平台费用**。



通过成为自己的支付处理商，您可以消除您与客户之间对中央第三方的依赖。您可以直接接受比特币付款和 generate 付款发票。这确保了您和您的公司都不会被其他人禁止。您既是银行又是支付处理商，无需为每笔交易向中介支付佣金。



**on-chain**的交易费用仍然存在，但可以通过使用**Liquid**或**Lightning**网络来降低费用。



此外 ：




- 完全可定制的界面和发票；
- 本地**Tor**支持，增强保密性；
- 支持**众筹**、**POS**和**支付按钮**；
- 兼容多种货币；
- Bitcoin 直接支付和点对点支付 ；
- 完全控制私钥
- 加强隐私保护 ；
- 增强安全性 ；
- 自托管软件 ；
- 支持 **SegWit** 和 **Lightning 网络**；
- 基于节点的内部组合，并整合硬件组合。

### 选择托管模式



BTCPay 服务器有多种不同的安装方式。根据您的需求和资源，主要有三种选择：




- 由第三方托管的 BTCPay 服务器**：您使用一个为您托管服务的平台。这很简单，但通常不是免费的。

- BTCPay 服务器安装在您自己的硬件上（本地）**：计算机、迷你 PC 或 Umbrel 上。这种方法技术性更强，但具有完全的独立性。

## 安全和最佳做法



BTCPay 服务器赋予的自主权（对您的资金拥有完全主权）是一个真正的优势。但这种自由也带来了更大的安全责任。通过管理您自己的支付，您就扮演了自己银行的角色。因此，必须采用最佳实践来保护您的资金、数据和基础设施。以下是需要考虑的要点。



### 安全访问服务器





- 使用强大的密码：大小写字母、数字和特殊字符相结合。避免重复使用现有密码。
- 激活双因素身份验证 (2FA)，以访问您的 BTCPay 界面。
- 定期更新您的操作系统、BTCPay 服务器实例和依赖项（Docker、Bitcoin 节点、Lightning 节点......）。更新通常会修正安全漏洞。



### 管理和备份私人密钥





- 将私钥和种子密码离线保存在物理介质（纸张或金属）上。
- 最好使用 wallet 硬件 wallet。
- 在不同的受保护物理位置保存多份备份。



### 安全支付和保密





- 使用 Tor 网络或 VPN 隐藏服务器的 IP 地址，保护你的隐私。
- 禁用服务器上不必要的端口，并将 SSH 连接限制在受信任的地址上。
- 为您的 BTCPay 网页界面的所有连接激活 HTTPS（SSL 证书）。
- 切勿与未接受过 Bitcoin 组合管理培训的人员共享管理界面。



### 为 "闪电 "网络实施最佳做法



您的商店与 Voltage 托管的**Lightning 节点相连，这大大简化了网络的技术管理。不过，采用**良好的监控和安全措施**仍然非常重要：





- 保存您的 Voltage 节点的 API** 登录名和访问密钥（使 BTCPay 能够连接）。
- 使用双因素身份验证和高强度密码保护您的 Voltage 账户**。
- 从 Voltage 面板监控节点和通道状态**：这有助于您发现任何异常或不同步现象。
- 避免共享您的 API** 凭据或将其公开（如在网站代码中）。
- 在迁移时，只需**重新配置 BTCPay 和 Voltage 之间的链接**：无需手动关闭通道。



使用Voltage，您将受益于**安全、高可用性**和**自动备份**的基础设施，同时保留对您的闪电支付**的完全主权。



### 组织和构建内部程序





- 定义明确的访问管理政策：谁可以创建发票、查看付款、访问节点等。
- 记录备份和恢复程序，以便在发生事故时能迅速做出反应。
- 定期测试备份的恢复情况，确保其正常工作。
- 对员工或合作者进行基本操作安全培训：警惕网络钓鱼、使用安全密码、尊重保密性。



### 监督和建立持续维护





- 使用日志或监控工具持续监控服务器的活动。
- 安排定期安全审查：检查更新、访问、备份和事务一致性。
