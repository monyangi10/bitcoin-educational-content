## Въведение



Преобразува класическа машина за дим в устройство, платимо в Bitcoin чрез Lightning Network. Всяко плащане автоматично задейства струя дим!





- Ниво: Междинно ниво
- Очаквано време: 2-3 часа
- Случаи на употреба: Bitcoin събития, артистични изпълнения, демонстрации на светкавици, автоматизирани сценични ефекти



## Предварителни условия



### Знания





 - Основи на електрониката (окабеляване, релета)
 - Заваряване (или използване на съединители Dupont)
 - Конфигуриране на мрежата (WiFi, WebSocket)



### Необходими сметки





- Сървър на BTCPay: Функционална инстанция (самостоятелна или хоствана)
- Blink Wallet : Акаунт + достъп API



### Достъп до





- Администраторски достъп до сървъра на BTCPay
- WiFi връзка за ESP32

4. **Никога не включвайте отново 220V**, докато кутията за дистанционно управление не бъде затворена и обезопасена



Ако не се чувствате комфортно с този вид работа, вземете със себе си човек, който е.

## ЧАСТ 1: Монтаж на хардуера



### Стъпка 1: Подготовка на дистанционното управление



Цел: Свържете релето с бутона ON/OFF на дистанционното управление


1. Отворено дистанционно управление




    - Идентифициране на бутона ON/OFF
    - Отвийте корпуса, за да отворите дистанционното управление


2. Намиране на връзки




 - Намерете клемите + и - на
 - Проверка на непрекъснатостта с мултицет (по избор)



![smoke-machine-remote](assets/fr/8.webp)



3. Окабеляване на бутоните (спойка или съединители)




    - Запояване на черен кабел към клемата - на бутона
    - Запояване на червен кабел към общата + клема



![smoke-machine-remote](assets/fr/9.webp)



### Стъпка 2: Свързване към релейния модул



**Напомняне: Терминология на релето




| **Терминал**         | **Описание**           | **Функция**                        |
| -------------------- | ------------------------- | ----------------------------------- |
| NO (Нормално отворено)   | Верига отворена по подразумеване | Затваря се когато релето е активирано |
| NC (Нормално затворено) | Верига затворена по подразумеване  | Отваря се когато релето е активирано  |
| COM (Общо)         | Централен терминал          | Превключва между NO и NC              |

**Окабеляване от дистанционното управление към релейния модул:**




- Черен проводник от бутона ON/OFF **→** NO (нормално отворен)
- Червен проводник (общ) **→** COM (общ)



**Логика:**


Когато ESP32 активира релето, той свързва COM и NO, което е точно същото като натискането на бутона на дистанционното управление.


Когато ESP32 прекъсне релето, COM и NO се разделят, което е равносилно на освобождаване на бутона.



![remote-relay](assets/fr/10.webp)



### Стъпка 3: Свързване на ESP32 към релейния модул



**Схема на окабеляване:**




| **ESP32** | **→** | **Реле модул** |
| --------- | ----- | ----------------- |
| V5 (5V)   | **→** | VCC               |
| GND       | **→** | GND               |
| GPIO 21   | **→** | IN (Вход)        |

**Проверка:**




- VCC и GND са добре свързани (полярност)
- GPIO 21 се използва за контролен сигнал
- Няма видимо късо съединение



![relay-esp32](assets/fr/11.webp)



**Хечпойнт хардуер**


Преди да преминете към софтуера, проверете :




- Правилно окабелено дистанционно управление
- Релеен модул, свързан към ESP32
- Без оголени проводници, които се допират до други компоненти
- 220V винаги е изключено



![relay-esp32](assets/fr/12.webp)

![btcpay-plugins](assets/fr/15.webp)

### Стъпка 2: Конфигурация на Backend: *BTCPay Server + Blink*



**1 - Създаване на wallet *Blink***




- Посетете https://www.blink.sv
- Създайте акаунта си. Моля, вижте ръководството :



[https://planb.academy/tutorials/wallet/mobile/blink-7ea5f5a4-e728-4ff9-b3f9-cf20aa6fc2bd)



**2 - Генериране на ключ API *Blink***




- Достъп до интерфейса API: **[https://www.blink.sv/en/api](https://www.blink.sv/en/api)** и влезте със същия акаунт, който сте използвали за създаване на wallet *Blink*



![blink-api](assets/fr/18.webp)





   - След като се свържете, отидете в раздела *API Keys*



![blink-api](assets/fr/19.webp)





   - Кликнете върху *" + "* в горния десен ъгъл, за да получите достъп до конфигурацията на вашия ключ API



![blink-api](assets/fr/20.webp)





   - Дайте име на вашия ключ API и оставете настройките по подразбиране. След това, в третата стъпка, запишете своя ключ API - ще го видите само веднъж: `blink_mZ5KxxxxxxxxxxxxxxxNbmX`



![blink-api](assets/fr/21.webp)





   - След като бъде създаден, той трябва да се появи в списъка с активни ключове API.



![blink-api](assets/fr/22.webp)



**3 - Свързване на *Blink* към *BTCPay Server***




- Отворете вашия *BTCPay сървър*
- Навигирайте до : *Wallet* **→** *Мълния*



![btcpay-server](assets/fr/23.webp)





- Щракнете върху *Използване на потребителски възел*
- Поставете следния низ за свързване:



```
type=blink;server=https://api.blink.sv/graphql;api-key=blink_mZ5KxxxxxxxxNbmX;wallet-id=0a3fc465-082xxxxxxxxxx-2545595d856f
```



**⚠️** **Важно** :




- Не променяйте първата част: `type=blink;server=https://api.blink.sv/graphql`;
- Заменете само :
    - api-ключ= *по вашия ключ API Blink*
    - wallet-id= *по вашия wallet Blink* ID
- След това щракнете върху *Тест на връзката*, а след това върху *Спасиране*



![btcpay-server](assets/fr/24.webp)





 - Проверете дали връзката е установена (зелен статус)



![btcpay-server](assets/fr/25.webp)



**4 - Създаване на точка за продажба (PoS)**




- В сървъра на BTCPay отидете в раздела *Плъгини* и кликнете върху *Посока на продажба*



![btcpay-server](assets/fr/26.webp)





- Дайте име на своята PoS и кликнете върху *Създаване*



![btcpay-server](assets/fr/27.webp)





- Конфигурация на PoS :
    - Изберете стил на мястото за продажба = *Принт дисплей*
    - Валута = *SATS*
    - Кликнете върху *SAVE*



![btcpay-server](assets/fr/28.webp)





- Конфигурация на продукта :
    - Изтриване на всички продукти по подразбиране
    - След това щракнете върху *добавяне на елемент*



![btcpay-server](assets/fr/29.webp)



![btcpay-server](assets/fr/30.webp)





- Конфигуриране на продукта:
    - Заглавие : *машина за дим*
    - Цена : *10 sats*
    - Bitcoin Превключвател GPIO : 21
    - Bitcoin Продължителност на превключването (в милисекунди) : 5000
    - Кликнете върху *Затвори* и след това върху *Запиши*, за да запазите новия продукт



![btcpay-server](assets/fr/31.webp)

![bitcoinswitch-lnbits](assets/fr/34.webp)





- Изчакайте качването, процесът е завършен, когато в регистрите се появи *"Leaving... "*


![bitcoinswitch-lnbits](assets/fr/35.webp)





- Изключете ESP32 от електрическата мрежа



**3 - Проверка на инсталацията на фърмуера на BitcoinSwitch




- Презареждане на страницата: [https://bitcoinswitch.lnbits.com/](https://bitcoinswitch.lnbits.com/)
- Свържете отново ESP32 към компютъра с помощта на USB/Micro-USB кабел
- След това щракнете върху *Свързване с устройство
- Изберете USB порта на вашия ESP32, след което щракнете върху *Свързване*, както е описано по-горе
- След като се свържете, натиснете бутона **RESET** на ESP32
- Проверете в регистрите дали последните редове показват :



```
Welcome to BitcoinSwitch! (v1.0.1)
Config file does not exist.
Entering config mode. until we receive /config-done.
```



(Това е нормално, означава, че все още няма конфигурация, но че фърмуерът е инсталиран)



![bitcoinswitch-lnbits](assets/fr/36.webp)



**4 - Генериране на WebSocket LNURL** URL



Очакван окончателен формат :



```
https://XXXXv/apps/46XXXXXXXXXXXXXXXXXXXXwFB/pos
```



Стъпки за генериране :




- Отворете инстанцията на сървъра на BTCPay, след което отидете в PoS, която създадохме по-късно.
- След това щракнете върху "View", за да отворите своя PoS в браузъра



![btcpay-server-https](assets/fr/37.webp)





- Копирайте URL адреса на страницата, например :



![btcpay-server-https](assets/fr/38.webp)



Нека разопаковаме този URL адрес:



```
https://XXXXv/apps/46XXXXXXXXXXXXXXXXXXXXwFB/pos
```





- `XXXXv` → домейнът на вашата инстанция на BTCPay Server
- `46XXXXXXXXXXXXXXXXXXXXwFB` → вашият уникален идентификатор PoS
- `/pos` → указва място за продажба



Преобразувайте го:




- Заменете `https://` с `wss://`
- Добавете `/bitcoinswitch` в края



Резултат:



```
wss://XXXXv/apps/46XXXXXXXXXXXXXXXXXXXXwFB/pos/bitcoinswitch
```



Запазете този URL адрес за бъдеща конфигурация, тъй като той ще позволи на вашия ESP32 да комуникира в реално време със сървъра на BTCPay. Протоколът WebSocket (`wss://`) установява постоянна връзка между тях: веднага щом плащането на Lightning бъде потвърдено на вашия PoS, BTCPay незабавно изпраща информацията на ESP32, който след това може да задейства вашата пушалка.

- В раздела *LNbits Device URL* поставете WebSocket URL адреса, създаден в предишната стъпка

![bitcoinswitch-lnbits](assets/fr/40.webp)





- Изчакайте качването да завърши; в регистрите трябва да се появят параметрите, които току-що сте въвели (SSID, парола и WebSocket URL)



![bitcoinswitch-lnbits](assets/fr/41.webp)

![bitcoinswitch-lnbits](assets/fr/42.webp)

## Тестване и отстраняване на грешки



### Завършване на финалния тест



1. Включете машината за пушене (220V) и я включете


2. Захранване на ESP32 (с батерия или USB)


3. Отворете своята BTCPay PoS в браузъра си


4. Сканиране на елемента "smoke-machine"


5. Плащане с wallet Lightning (Blink или друг wallet)


6. Наблюдавайте:




 - Релето щраква (звуков сигнал и светване на светодиода на релето)
 - Активира се машината за дим
 - Дим!



### Проблеми и решения на справедливостта




| **Проблем**                        | **Вероятна причина**              | **Решение**                                                                                 |
| ----------------------------------- | ------------------------------- | -------------------------------------------------------------------------------------------- |
| ESP32 не се свързва            | Липсва USB драйвер             | Инсталирайте [CH340 drivers](https://learn.sparkfun.com/tutorials/how-to-install-ch340-drivers) |
| Реле не щракне                | Неправилно свързване GPIO            | Проверете GPIO 21 → IN                                                                        |
| Дымовата машина не реагира         | Дистанционното управление неправилно свързано         | Проверете NO/NC/COM                                                                           |
| Време на изтичане на WebSocket                   | Неправилен URL                  | Проверете wss:// и /bitcoinswitch                                                            |
| WiFi не се свързва             | SSID/Password грешен            | Преизберете конфигурацията на WiFi                                                                    |
| Плащане получено, но нищо се не случва | ESP32 не е свързан към WebSocket | Проверете регистрите RESET                                                                      |
