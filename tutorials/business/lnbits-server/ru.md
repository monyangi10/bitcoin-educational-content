LNbits - это веб-интерфейс с открытым исходным кодом, который превращает любой бэкенд Lightning (LND, Core Lightning, Phoenixd) в полноценную сервисную платформу. Это самостоятельное решение позволяет управлять несколькими портфелями Lightning в изоляции, развертывать точки продаж, создавать системы пожертвований или биллинговые сервисы, сохраняя при этом полный контроль над своими средствами.



В этом руководстве рассматриваются два подхода к установке: **VPS Ubuntu с Phoenixd** (легкое решение без полноценного узла Bitcoin) и **Umbrel** (интеграция с существующим узлом LND). В отличие от общего руководства Академии Plan ₿ по LNbits, в котором рассматриваются концепции и расширения, в этом руководстве основное внимание уделяется пошаговым техническим процедурам установки.



## Что такое LNbits?



LNbits - это учетная система Lightning, разработанная на Python (FastAPI) и подключаемая к существующему бэкенду (LND, Core Lightning, Phoenixd). В отличие от традиционных узлов Lightning, LNbits предлагает доступный интерфейс для управления несколькими изолированными портфелями с собственными ключами API. Вы можете создать субсчета для своей семьи, сотрудников или проектов, не предоставляя им доступ ко всем вашим средствам.



Раздельная архитектура хранит информацию в SQLite (по умолчанию) или PostgreSQL (в производстве), а средства остаются под управлением вашего бэкенда Lightning. Такое разделение гарантирует переносимость: вы можете перейти с Phoenixd на LND без потери пользовательских данных.



## Основные характеристики



LNbits предлагает универсальную систему **расширений**: TPoS (точки продаж), Paywall (монетизация контента), Events (продажа билетов), LndHub (сервер для BlueWallet), Bolt Cards (платежи NFC), Split Payments (автоматическое распределение) и User Manager (управление пользователями с аутентификацией).



На **приборной панели** в режиме реального времени отображаются балансы, история операций и инструменты выставления счетов. Каждый wallet имеет уникальный URL-адрес, содержащий его ключи API, что позволяет получить доступ без традиционного входа в систему. Трехуровневая система ключей API** (admin, invoice, read-only) обеспечивает детальный контроль разрешений для безопасных интеграций.



LNbits нативно реализует **LNURL** (LNURL-pay, LNURL-withdraw, LNURL-auth) и поддерживает **Lightning Address**, гарантируя совместимость с современными Lightning-кошельками и облегчая развертывание профессиональных сервисов.

## Установка на Ubuntu VPS с помощью Phoenixd



### Шаг 1: Обеспечение безопасности VPS-сервера



**Перед установкой** вам необходимо защитить ваш сервер Ubuntu VPS в соответствии с правилами искусства. Этот шаг является **критическим** для защиты вашей инфраструктуры и ваших средств Lightning.



Вот подробное руководство, которое поможет вам начать: **[Начальная конфигурация сервера Ubuntu - пошаговое руководство](https://danielpcostas.dev/ubuntu-server-initial-configuration-a-step-by-step-guide/)** Дэниела П. Костаса.



В этом руководстве рассказывается о настройке пользователей, безопасном SSH, брандмауэре (UFW), fail2ban, автоматических обновлениях и надлежащих методах обеспечения безопасности системы.



### Шаг 2: Установка Phoenixd



После того как ваш сервер будет защищен, необходимо установить и настроить Phoenixd. Академия Plan ₿ предлагает подробный специализированный учебник по установке, генерации seed и настройке службы systemd:



https://planb.academy/tutorials/node/lightning-network/phoenixd-beb86edd-f9c0-4bec-ad36-db234c88e7b1

Когда Phoenixd будет запущен (проверьте с помощью `./phoenix-cli getinfo`), запишите **HTTP-пароль** в `~/.phoenix/phoenix.conf` - он понадобится вам для подключения LNbits к Phoenixd.



### Развертывание LNbits



Установите UV и клонируйте LNbits :


```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
export PATH="$HOME/.local/bin:$PATH"
git clone https://github.com/lnbits/lnbits.git && cd lnbits
uv sync --all-extras
```



Настройте бэкэнд Phoenixd:


```bash
cp .env.example .env && nano .env
```



Добавьте в `.env` :


```
LNBITS_BACKEND_WALLET_CLASS=PhoenixdWallet
PHOENIXD_API_ENDPOINT=http://127.0.0.1:9740
PHOENIXD_API_PASSWORD=<mot-de-passe-phoenix.conf>
```



Протестируйте с помощью `uv run lnbits --host 0.0.0.0 --port 5000`, затем создайте службу systemd с `Wants=phoenixd.service`.



## Первоначальная настройка и первое использование



### Активация Суперпользователя



Активируйте интерфейс администратора в файле `.env` :


```
LNBITS_ADMIN_UI=true
```



Перезапустите LNbits (`sudo systemctl restart lnbits`) и получите идентификатор SuperUser:


```bash
cat ~/lnbits/data/.super_user
```



Перейдите по адресу `http://<IP-VPS>:5000/wallet?usr=<SuperUserID>` для получения панели администрирования. В меню "Сервер" вы можете настроить источники финансирования, расширения и учетные записи пользователей.



### Безопасное создание учетной записи



**Важно для публичного размещения**: Если вы размещаете свой экземпляр LNbits на публичном доменном имени, доступном из Интернета, то **критически важно** запретить свободное создание учетных записей пользователей.



В административном интерфейсе SuperUser перейдите в раздел "Настройки", а затем в раздел "Управление пользователями". Там вы найдете опцию "Разрешить создание новых пользователей".



![Gestion des utilisateurs - Sécurité](assets/fr/17.webp)



**Для публичной выставки с доменным именем** :




- Вы должны отключить** опцию "Разрешить создание новых пользователей"
- Без этой защиты любой человек в Интернете может создать учетную запись на вашем экземпляре
- Злоумышленник может создать учетные записи и использовать ликвидность вашего узла Lightning без вашего ведома
- Вам нужно будет создать учетные записи пользователей вручную из интерфейса SuperUser



**Только для местного использования** :




- Этот параметр менее важен, если ваш экземпляр доступен только локально (http://localhost:5000)
- Однако отключение этой опции является хорошей практикой общей безопасности



После настройки только администратор SuperUser может создавать новые учетные записи пользователей через интерфейс "Пользователи". Такой подход гарантирует полный контроль над тем, кто может получить доступ к вашей инфраструктуре Lightning и использовать ваши средства.



### Открытие первого канала



Phoenixd автоматически управляет каналами через автоликвидность. Сгенерируйте Lightning-счет на ~30 000 sats от LNbits и оплатите его с другого wallet. Phoenixd автоматически открывает канал на ACINQ. Плата за открытие (~20-23k sats) списывается, остаток (~7-10k sats) появляется после подтверждения on-chain.



Проверьте состояние с помощью `./phoenix-cli getinfo`. Затем рассмотрите возможность отключения автоликвидности (`auto-liquidity=off` в `phoenix.conf`) для контроля открытия каналов.



### Публичная демонстрация и HTTPS



**Важно**: HTTPS обязателен для публичного показа (безопасность ключа API + совместимость с LNURL). Пропустите этот шаг только для локального использования.



**Caddy (рекомендуется)**: автоматический SSL. `sudo apt install -y caddy`, отредактируйте `/etc/caddy/Caddyfile` :


```
votre-domaine.com {
reverse_proxy 127.0.0.1:5000
}
```


Перезапустите: `sudo systemctl restart caddy`.



**Nginx** : Больше контроля. Установите `nginx certbot python3-certbot-nginx`, создайте `/etc/nginx/sites-available/lnbits` :


```nginx
server {
listen 80;
server_name votre-domaine.com;
location / {
proxy_pass http://127.0.0.1:5000;
proxy_set_header Host $host;
proxy_set_header X-Forwarded-Proto $scheme;
}
}
```


Активируйте: `sudo ln -s /etc/nginx/sites-available/lnbits /etc/nginx/sites-enabled/ && sudo nginx -t && sudo systemctl reload nginx && sudo certbot --nginx -d your-domain.com`



Добавьте в `.env`: `FORWARDED_ALLOW_IPS=*`



## Установка зонта



### Развертывание из App Store



Зайдите в Umbrel App Store, найдите "LNbits" и нажмите "Установить".



![Installation LNbits Umbrel](assets/fr/01.webp)



Umbrel автоматически проверяет наличие необходимых зависимостей. Для работы LNbits требуется узел Lightning Node (LND). Если ваш узел Lightning уже работает, нажмите "Установить LNbits" для подтверждения.



![Dépendances LNbits](assets/fr/02.webp)



Umbrel загружает образ Docker, автоматически настраивает соединения с LND и запускает контейнер (2-5 минут). Установка происходит полностью в фоновом режиме.



### Начальная конфигурация Суперпользователя



При первом запуске LNbits предложит вам создать учетную запись администратора SuperUser. Введите имя пользователя и надежный пароль для защиты доступа к административному интерфейсу.



![Configuration SuperUser](assets/fr/03.webp)



**Важно**: Эта учетная запись SuperUser имеет полные привилегии на вашем экземпляре LNbits. Выберите надежный пароль и храните его в безопасности.



После создания учетной записи вы автоматически перейдете в главный административный интерфейс. Umbrel уже настроил LND в качестве источника финансирования - все платежи Lightning будут проходить через ваши существующие каналы.



### Доступ к интерфейсу администратора



В левом боковом меню нажмите на "Настройки", чтобы получить доступ к полной панели администрирования.



![Interface Settings](assets/fr/04.webp)



В разделе "Управление кошельками" отображается ключевая информация о вашей конфигурации:




- Источник финансирования** : LndBtcRestWallet (прямое подключение к вашему узлу LND Umbrel)
- Баланс узла** : Общая ликвидность, доступная в ваших каналах Lightning
- Баланс LNbits**: Средства, выделенные на систему LNbits (первоначально 0 sats)



Теперь вы можете напрямую использовать ликвидность вашего узла Umbrel для всех созданных вами кошельков LNbits. Никаких дополнительных настроек не требуется - LNbits уже работает.



### Управление пользователями



Одной из самых мощных особенностей LNbits является возможность создания нескольких независимых пользователей, каждый из которых имеет пароль для аутентификации и изолированные кошельки. Такая архитектура позволяет использовать ликвидность узла Umbrel, предлагая при этом полностью изолированные субсчета для различных целей: бизнеса, семьи, сотрудников, проектов и т. д.



В боковом меню нажмите на "Пользователи", чтобы получить доступ к управлению пользователями. Нажмите "Создать учетную запись", чтобы добавить нового пользователя.



![Gestion des utilisateurs](assets/fr/05.webp)



Заполните форму создания пользователя:




- Имя пользователя**: Имя пользователя (пример: "satoshi")
- Установить пароль**: Активируйте этот параметр для установки пароля аутентификации
- Пароль** и **Повтор пароля**: Установите пароль для этого пользователя



![Création utilisateur satoshi](assets/fr/06.webp)



Необязательные поля (открытый ключ Nostr, электронная почта, имя, фамилия) можно оставить пустыми для минимальной настройки. Нажмите кнопку "Создать учетную запись" для подтверждения.



![Confirmation utilisateur créé](assets/fr/07.webp)



Теперь ваш новый пользователь появится в списке пользователей со своим уникальным идентификатором и именем пользователя.



![Liste des utilisateurs](assets/fr/08.webp)



**Важный момент**: Каждый пользователь может входить в систему совершенно независимо, используя свой собственный пароль. Администратор SuperUser сохраняет полный контроль через административный интерфейс.



### Управление пользователем wallet



Теперь, когда пользователь "satoshi" создан, вам нужно назначить ему wallet Lightning. Нажмите на значок wallet (второй значок) для данного пользователя, затем на "CREATE NEW WALLET".



![Gestion des wallets](assets/fr/09.webp)



В диалоговом окне вам будет предложено присвоить имя wallet. Введите описательное имя (например, "Wallet Of Satoshi") и выберите валюту отображения (CUC, USD, EUR и т. д.).



![Création wallet](assets/fr/10.webp)



Нажмите на кнопку "CREATE". LNbits мгновенно сгенерирует рабочую wallet Lightning для этого пользователя.



![Confirmation wallet créé](assets/fr/11.webp)



Теперь вы видите два существующих кошелька: автоматически созданный по умолчанию wallet "LNbits wallet" и новый "Wallet Of Satoshi". Чтобы упростить работу пользователя, вы можете удалить стандартный wallet, нажав на иконку удаления (красная корзина).



![Wallet final unique](assets/fr/12.webp)



У пользователя "satoshi" теперь есть единственный, четко идентифицированный wallet. Каждый пользователь wallet работает полностью автономно, используя ликвидность вашего базового узла LND.



**Ключевая концепция**: Все эти кошельки пользуются глобальной ликвидностью вашего узла Umbrel. Вы не создаете новые каналы Lightning для каждого wallet - LNbits выступает в роли интеллектуального учетного слоя, который управляет распределением средств в рамках существующей инфраструктуры Lightning. В этом и заключается сила системы LNbits с несколькими wallet.



### Вход пользователя в систему



Выйдите из учетной записи SuperUser (значок в правом верхнем углу) и вернитесь на страницу входа в LNbits. Теперь вы можете войти в систему с учетными данными нового пользователя.



![Connexion utilisateur satoshi](assets/fr/13.webp)



Введите имя пользователя ("satoshi") и пароль, заданные ранее, затем нажмите кнопку "LOGIN". Пользователь получает прямой доступ к своему персональному wallet, полностью изолированному от интерфейса администрирования.



### Interface от пользователя wallet



После входа в систему пользователь получает доступ к полному интерфейсу wallet Lightning.



![Interface wallet utilisateur](assets/fr/14.webp)



Интерфейс включает в себя :




- Текущий баланс**: Отображается в sats и в выбранной валюте (в данном примере CUC)
- Основные действия**: ВСТАВИТЬ ЗАПРОС, СОЗДАТЬ СЧЕТ, QR-значок (быстрое сканирование)
- История транзакций** : Полный список всех платежей и поступлений
- Правая боковая панель**: Конфигурация и возможности доступа



### Мобильный доступ к wallet



Правая боковая панель предлагает особенно практичную функцию: мобильный доступ к wallet. Разверните раздел "Мобильный доступ", чтобы открыть доступные опции.



![Mobile Access](assets/fr/15.webp)



LNbits предлагает несколько способов использования этого wallet на смартфоне:



** Вариант 1: Совместимые мобильные приложения




- Загрузите **Zeus** или **BlueWallet** из App Store или Google Play
- Активируйте расширение **LndHub** в LNbits для этого wallet
- Сканируйте QR-код LndHub с помощью мобильного приложения для подключения wallet



**Вариант 2: Прямой доступ через мобильный браузер**




- QR-код, отображаемый в разделе "Экспорт на телефон с помощью QR-кода", содержит полный URL-адрес wallet с интегрированной аутентификацией
- Отсканируйте этот QR-код со своего смартфона, чтобы открыть wallet прямо в мобильном браузере
- Добавьте страницу на главный экран для быстрого доступа



**Важная безопасность**: Этот URL содержит ключи API для полного доступа к wallet. Никогда не сообщайте его публично. Относитесь к этому QR-коду так же, как к своим закрытым ключам Bitcoin - любой, кто отсканирует этот QR-код, получит полный доступ к wallet.



Эта мобильная функция превращает ваш экземпляр LNbits Umbrel в настоящий сервер Lightning wallet для вас и ваших друзей, при этом вы сохраняете полный контроль над своими средствами благодаря самостоятельному хостингу узла.



### Разделение доступа пользователей



Основным вариантом использования этой многопользовательской конфигурации является **обмен кошельками с членами вашей семьи или близким окружением**. Создав пользователя с выделенным wallet (например, "satoshi" в нашем примере), вы можете поделиться этими учетными данными с доверенными членами вашей семьи.



**Защита доступа на Umbrel**: Доступ к вашему экземпляру LNbits на Umbrel естественным образом защищен, так как к нему можно получить доступ только :




- В локальной сети** : Члены вашей семьи, подключенные к той же сети WiFi/Ethernet, могут получить доступ к экземпляру
- Через VPN**: Если вы используете VPN, например Tailscale, настроенный на вашем сервере Umbrel, авторизованные пользователи могут получить безопасный удаленный доступ



Этот двойной уровень защиты (доступ к сети + аутентификация пользователей) делает опцию "Разрешить создание новых пользователей" менее критичной для Umbrel. Только те, у кого уже есть доступ к вашей сети или VPN, могут получить доступ к интерфейсу входа.



**Типичный сценарий**: Вы создаете аккаунт "папа", аккаунт "мама", аккаунт "бизнес" и так далее. У каждого члена семьи есть своя изолированная wallet Lightning, но при этом он пользуется общей ликвидностью вашего узла Umbrel. Просто сообщите имя пользователя и пароль - пользователь сможет подключиться с любого устройства в вашей локальной сети или через VPN в Tailscale. Более подробную информацию вы найдете в нашем специальном руководстве по Tailscale:



https://planb.academy/tutorials/computer-security/communication/tailscale-9acbd7de-04d9-40f6-ab80-35f0dfedb632

### Изучите доступные расширения



Вернитесь в интерфейс SuperUser и откройте меню "Расширения" на левой боковой панели, чтобы открыть для себя всю экосистему расширений LNbits.



![Extensions disponibles](assets/fr/16.webp)



LNbits предлагает богатый каталог расширений, которые превращают ваш экземпляр в настоящую платформу сервисов Lightning:





- Музыкальный автомат**: Музыкальный автомат с питанием от sats (оплата Spotify)
- Тикеты поддержки**: Платная система поддержки (получайте саты за ответы на вопросы)
- TPoS**: Безопасный мобильный терминал для розничной торговли
- User Manager**: расширенное управление пользователями и wallet (которое мы только что использовали)
- События**: Продажа и проверка билетов на мероприятия
- Устройства LNURLD**: Управление точками продаж, банкоматы, подключенные коммутаторы
- SMTP**: Позволяет пользователям отправлять электронные письма и зарабатывать саты
- Boltcards**: Программирование NFC-карт для платежей с использованием технологии Lightning tap-to-pay
- NostrNip5**: Создание NIP5-адресов для ваших доменов
- Раздельные платежи**: Автоматическое распределение платежей между несколькими кошельками



Каждое расширение активируется одним щелчком мыши из этого интерфейса. Расширения с пометкой "FREE" являются бесплатными, а некоторые доступны в виде "PAID" версий. Изучите каталог, чтобы найти те, которые соответствуют вашим потребностям - будь то бизнес, управление семьей или эксперименты с возможностями Lightning Network.



## Преимущества и ограничения



**Преимущества**: Финансовый суверенитет (полный контроль над средствами/ключами/данными), гибкость архитектуры (миграция VPS→full node без потерь), профессиональная система расширения, интуитивно понятный интерфейс.



**Ограничения** : Программное обеспечение в бета-версии (осторожно с суммами), безопасность под ответственностью администратора, URL, содержащие чувствительные ключи API (HTTPS обязателен), многопользовательское управление подразумевает ответственность хранителя.



## Лучшие практики



**Бэкапы**: Учетные данные Phoenixd Seed/LND, база данных LNbits, файлы .env. Автоматизируйте ежедневно, храните вне рабочего сервера, в зашифрованном виде. Регулярно тестируйте восстановление.



**Обслуживание**: Регулярно проверяйте наличие обновлений (LNbits, бэкенд Lightning, операционная система). Всегда проверяйте информацию о выпуске перед основными обновлениями.





- На Umbrel**: App Store автоматически уведомляет вас о выходе новых версий. Синхронизация расширений через "Управление расширениями" > "Обновить все". Проверьте включение базы данных SQLite в автоматические резервные копии Umbrel.
- На VPS**: Обновите вручную с помощью `cd lnbits && git pull && uv sync --all-extras && sudo systemctl restart lnbits`. Следите за системными журналами: `sudo journalctl -u lnbits -f`.



## Заключение



Самостоятельный хостинг LNbits предлагает конкретный путь к финансовому суверенитету Lightning. VPS+Phoenixd предлагает легкое решение для быстрого предоставления услуг, Umbrel полную интеграцию с существующим узлом Bitcoin. Масштабируемая архитектура позволяет перейти от простых многопользовательских wallet к сложным бизнес-кейсам.



Самостоятельный хостинг подразумевает ответственность: резервное копирование семян, защита доступа, начало работы со скромными суммами. Благодаря этим мерам предосторожности LNbits становится надежным решением для молниеносной экономики, сохраняя при этом децентрализацию и автономию.
