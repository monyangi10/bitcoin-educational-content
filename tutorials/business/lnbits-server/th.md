LNbits เป็นเว็บอินเตอร์เฟซโอเพ่นซอร์สที่เปลี่ยนแบ็กเอนด์ Lightning ใด ๆ (LND, Core Lightning, Phoenixd) ให้เป็นแพลตฟอร์มบริการที่สมบูรณ์ โซลูชันที่โฮสต์ด้วยตัวเองนี้ช่วยให้คุณจัดการกระเป๋าเงิน Lightning หลายใบในแบบแยกส่วน, ติดตั้งจุดขาย, สร้างระบบบริจาคหรือบริการเรียกเก็บเงิน, ในขณะที่ยังคงควบคุมเงินของคุณได้อย่างเต็มที่



บทแนะนำนี้ครอบคลุมสองวิธีการติดตั้ง: **VPS Ubuntu กับ Phoenixd** (โซลูชันที่มีน้ำหนักเบาโดยไม่ต้องใช้โหนด Bitcoin เต็มรูปแบบ) และ **Umbrel** (การรวมเข้ากับโหนด LND ที่มีอยู่ของคุณ) แตกต่างจากบทแนะนำ LNbits ทั่วไปของ Plan ₿ Academy ซึ่งครอบคลุมแนวคิดและส่วนขยาย คู่มือนี้มุ่งเน้นไปที่ขั้นตอนการติดตั้งทางเทคนิคทีละขั้นตอน



## LNbits คืออะไร?



LNbits เป็นระบบบัญชี Lightning ที่พัฒนาด้วย Python (FastAPI) ซึ่งเชื่อมต่อกับ backend ที่มีอยู่แล้ว (LND, Core Lightning, Phoenixd) แตกต่างจากโหนด Lightning แบบดั้งเดิม LNbits มีอินเทอร์เฟซที่เข้าถึงได้สำหรับการจัดการกระเป๋าเงินที่แยกออกจากกันหลายใบพร้อมกับกุญแจ API ของตัวเอง คุณสามารถสร้างบัญชีย่อยสำหรับครอบครัว พนักงาน หรือโครงการต่างๆ โดยไม่ต้องให้พวกเขาเข้าถึงเงินทั้งหมดของคุณ



สถาปัตยกรรมที่แยกออกจากกันจะเก็บข้อมูลใน SQLite (ค่าเริ่มต้น) หรือ PostgreSQL (การผลิต) ในขณะที่เงินทุนยังคงถูกจัดการโดยแบ็กเอนด์ Lightning ของคุณ การแยกนี้รับประกันความสามารถในการพกพา: คุณสามารถย้ายจาก Phoenixd ไปยัง LND โดยไม่สูญเสียข้อมูลผู้ใช้ของคุณ



## คุณสมบัติหลัก



LNbits นำเสนอ **ระบบส่วนขยาย** ที่หลากหลาย: TPoS (จุดขาย), Paywall (การสร้างรายได้จากเนื้อหา), Events (การจำหน่ายตั๋ว), LndHub (เซิร์ฟเวอร์สำหรับ BlueWallet), Bolt Cards (การชำระเงินผ่าน NFC), Split Payments (การกระจายอัตโนมัติ), และ User Manager (การจัดการผู้ใช้พร้อมการยืนยันตัวตน)



**แดชบอร์ด** แสดงยอดคงเหลือแบบเรียลไทม์ ประวัติการทำธุรกรรม และเครื่องมือการเรียกเก็บเงิน แต่ละ wallet มี URL เฉพาะที่มีคีย์ API ของตนเอง ทำให้สามารถเข้าถึงได้โดยไม่ต้องใช้การเข้าสู่ระบบแบบดั้งเดิม ระบบคีย์ API สามระดับ** (ผู้ดูแลระบบ, ใบแจ้งหนี้, อ่านอย่างเดียว) มอบการควบคุมสิทธิ์อย่างละเอียดสำหรับการผสานรวมที่ปลอดภัย



LNbits ใช้งาน **LNURL** (LNURL-pay, LNURL-withdraw, LNURL-auth) ได้โดยตรงและรองรับ **Lightning Address** ซึ่งรับประกันความเข้ากันได้กับกระเป๋าเงิน Lightning สมัยใหม่และอำนวยความสะดวกในการปรับใช้บริการระดับมืออาชีพ



## แพลตฟอร์มที่รองรับ

## การติดตั้งบน Ubuntu VPS ด้วย Phoenixd



### ขั้นตอนที่ 1: การรักษาความปลอดภัยเซิร์ฟเวอร์ VPS



**ก่อนการติดตั้งใดๆ**, คุณจำเป็นต้องรักษาความปลอดภัยเซิร์ฟเวอร์ Ubuntu VPS ของคุณตามกฎของศิลปะ ขั้นตอนนี้มีความสำคัญอย่างยิ่งในการปกป้องโครงสร้างพื้นฐานและเงินทุน Lightning ของคุณ



นี่คือคู่มือโดยละเอียดที่จะช่วยให้คุณเริ่มต้นได้: **[Initial Ubuntu server configuration - Step-by-step guide](https://danielpcostas.dev/ubuntu-server-initial-configuration-a-step-by-step-guide/)** โดย Daniel P. Costas.



คู่มือนี้ครอบคลุมการกำหนดค่าผู้ใช้, การรักษาความปลอดภัย SSH, ไฟร์วอลล์ (UFW), fail2ban, การอัปเดตอัตโนมัติ, และแนวปฏิบัติที่ดีในการรักษาความปลอดภัยระบบ



### ขั้นตอนที่ 2: การติดตั้ง Phoenixd



เมื่อเซิร์ฟเวอร์ของคุณปลอดภัยแล้ว คุณจำเป็นต้องติดตั้งและกำหนดค่า Phoenixd. Plan ₿ Academy มีบทแนะนำเฉพาะที่ครอบคลุมการติดตั้ง, การสร้าง seed และการกำหนดค่าบริการ systemd :



https://planb.academy/tutorials/node/lightning-network/phoenixd-beb86edd-f9c0-4bec-ad36-db234c88e7b1

เมื่อ Phoenixd ทำงานแล้ว (ตรวจสอบด้วย `./phoenix-cli getinfo`), จดบันทึก **HTTP password** ใน `~/.phoenix/phoenix.conf` - คุณจะต้องใช้มันเพื่อเชื่อมต่อ LNbits กับ Phoenixd.



### การปรับใช้ LNbits



ติดตั้ง UV และโคลน LNbits :


```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
export PATH="$HOME/.local/bin:$PATH"
git clone https://github.com/lnbits/lnbits.git && cd lnbits
uv sync --all-extras
```



กำหนดค่าฝั่งหลังของ Phoenixd:


```bash
cp .env.example .env && nano .env
```



เพิ่มใน `.env` :


```
LNBITS_BACKEND_WALLET_CLASS=PhoenixdWallet
PHOENIXD_API_ENDPOINT=http://127.0.0.1:9740
PHOENIXD_API_PASSWORD=<mot-de-passe-phoenix.conf>
```



ทดสอบด้วย `uv run lnbits --host 0.0.0.0 --port 5000` จากนั้นสร้าง systemd service ด้วย `Wants=phoenixd.service`.



## การตั้งค่าเริ่มต้นและการใช้งานครั้งแรก



### การเปิดใช้งาน SuperUser



เปิดใช้งานอินเทอร์เฟซผู้ดูแลระบบใน `.env` :


```
LNBITS_ADMIN_UI=true
```



รีสตาร์ท LNbits (`sudo systemctl restart lnbits`) และดึงรหัส SuperUser:


```bash
cat ~/lnbits/data/.super_user
```



ไปที่ `http://<IP-VPS>:5000/wallet?usr=<SuperUserID>` สำหรับแผงการจัดการ เมนู "Server" ช่วยให้คุณกำหนดแหล่งเงินทุน ส่วนขยาย และบัญชีผู้ใช้



### การสร้างบัญชีอย่างปลอดภัย



**สำคัญสำหรับการเปิดเผยต่อสาธารณะ**: หากคุณกำลังเปิดเผยอินสแตนซ์ LNbits ของคุณบนชื่อโดเมนสาธารณะที่เข้าถึงได้จากอินเทอร์เน็ต สิ่งที่ **สำคัญ** คือการปิดการสร้างบัญชีผู้ใช้ฟรี



จากอินเทอร์เฟซการจัดการ SuperUser ไปที่ "Settings" แล้วไปที่ส่วน "User Management" ที่นั่นคุณจะพบตัวเลือก "Allow creation of new users"



![Gestion des utilisateurs - Sécurité](assets/fr/17.webp)



**สำหรับนิทรรศการสาธารณะด้วยชื่อโดเมน** :




- คุณต้องปิดใช้งาน** ตัวเลือก "อนุญาตให้สร้างผู้ใช้ใหม่"
- หากไม่มีการป้องกันนี้ ใครก็ตามบนอินเทอร์เน็ตสามารถสร้างบัญชีบนอินสแตนซ์ของคุณได้
- ผู้โจมตีอาจสร้างบัญชีและใช้สภาพคล่องของโหนด Lightning ของคุณโดยที่คุณไม่รู้ตัว
- คุณจะต้องสร้างบัญชีผู้ใช้ด้วยตนเองจากอินเทอร์เฟซ SuperUser



**สำหรับใช้ในท้องถิ่นเท่านั้น** :




- ตัวเลือกนี้มีความสำคัญน้อยกว่าหากอินสแตนซ์ของคุณสามารถเข้าถึงได้เฉพาะในเครื่องเท่านั้น (http://localhost:5000)
- อย่างไรก็ตาม การปิดใช้งานตัวเลือกนี้เป็นแนวปฏิบัติด้านความปลอดภัยทั่วไปที่ดี



เมื่อกำหนดค่าแล้ว เฉพาะผู้ดูแลระบบ SuperUser เท่านั้นที่สามารถสร้างบัญชีผู้ใช้ใหม่ผ่านอินเทอร์เฟซ "Users" วิธีการนี้รับประกันการควบคุมอย่างสมบูรณ์ว่าใครสามารถเข้าถึงโครงสร้างพื้นฐาน Lightning ของคุณและใช้เงินทุนของคุณได้



### เปิดช่องแรก



Phoenixd จัดการช่องทางโดยอัตโนมัติผ่านการจัดการสภาพคล่องอัตโนมัติ สร้างใบแจ้งหนี้ Lightning ประมาณ ~30,000 sats จาก LNbits และชำระจาก wallet อื่น Phoenixd เปิดช่องทางไปยัง ACINQ โดยอัตโนมัติ ค่าธรรมเนียมการเปิด (~20-23k sats) จะถูกหักออก ยอดคงเหลือที่เหลือ (~7-10k sats) จะปรากฏหลังจากการยืนยัน on-chain



ตรวจสอบสถานะด้วย `./phoenix-cli getinfo` จากนั้นพิจารณาปิดการใช้งาน auto-liquidity (`auto-liquidity=off` ใน `phoenix.conf`) เพื่อควบคุมการเปิดช่องทาง



### การแสดงผลสาธารณะและ HTTPS



**สำคัญ**: จำเป็นต้องใช้ HTTPS สำหรับการแสดงผลสาธารณะ (ความปลอดภัยของคีย์ API + ความเข้ากันได้ของ LNURL) ข้ามขั้นตอนนี้สำหรับการใช้งานเฉพาะในเครื่องเท่านั้น



**Caddy (แนะนำ)**: SSL อัตโนมัติ `sudo apt install -y caddy`, แก้ไข `/etc/caddy/Caddyfile` :


```
votre-domaine.com {
reverse_proxy 127.0.0.1:5000
}
```


รีสตาร์ท: `sudo systemctl restart caddy`.



**Nginx** : ควบคุมได้มากขึ้น ติดตั้ง `nginx certbot python3-certbot-nginx` สร้าง `/etc/nginx/sites-available/lnbits` :


```nginx
server {
listen 80;
server_name votre-domaine.com;
location / {
proxy_pass http://127.0.0.1:5000;
proxy_set_header Host $host;
proxy_set_header X-Forwarded-Proto $scheme;
}
}
```


เปิดใช้งาน: `sudo ln -s /etc/nginx/sites-available/lnbits /etc/nginx/sites-enabled/ && sudo nginx -t && sudo systemctl reload nginx && sudo certbot --nginx -d your-domain.com`



เพิ่มใน `.env`: `FORWARDED_ALLOW_IPS=*`



## การติดตั้ง Umbrel



### การปรับใช้จาก App Store



ไปที่ Umbrel App Store ค้นหา "LNbits" และคลิกที่ "ติดตั้ง"



![Installation LNbits Umbrel](assets/fr/01.webp)



Umbrel ตรวจสอบการพึ่งพาที่จำเป็นโดยอัตโนมัติ LNbits ต้องการ Lightning Node (LND) เพื่อทำงาน หาก Lightning node ของคุณทำงานอยู่แล้ว ให้คลิกที่ "ติดตั้ง LNbits" เพื่อยืนยัน



![Dépendances LNbits](assets/fr/02.webp)



Umbrel ดาวน์โหลดอิมเมจ Docker กำหนดค่าการเชื่อมต่อกับ LND โดยอัตโนมัติ และเริ่มต้นคอนเทนเนอร์ (2-5 นาที) การติดตั้งเกิดขึ้นทั้งหมดในพื้นหลัง



### การกำหนดค่า SuperUser เริ่มต้น



เมื่อเปิดใช้งานครั้งแรก LNbits จะขอให้คุณสร้างบัญชีผู้ดูแลระบบ SuperUser ป้อนชื่อผู้ใช้และรหัสผ่านที่ปลอดภัยเพื่อป้องกันการเข้าถึงอินเทอร์เฟซการจัดการ



![Configuration SuperUser](assets/fr/03.webp)



**สำคัญ**: บัญชี SuperUser นี้มีสิทธิ์เต็มรูปแบบบนอินสแตนซ์ LNbits ของคุณ เลือกรหัสผ่านที่แข็งแกร่งและเก็บรักษาไว้ให้ปลอดภัย



เมื่อคุณสร้างบัญชีแล้ว คุณจะถูกนำไปยังอินเทอร์เฟซการจัดการหลักโดยอัตโนมัติ Umbrel ได้ตั้งค่า LND เป็นแหล่งเงินทุนของคุณแล้ว - การชำระเงินผ่าน Lightning ทั้งหมดจะผ่านช่องทางที่มีอยู่ของคุณ



### การเข้าถึงอินเทอร์เฟซผู้ดูแลระบบ



ในเมนูด้านซ้ายมือ คลิกที่ "Settings" เพื่อเข้าสู่แผงการจัดการทั้งหมด



![Interface Settings](assets/fr/04.webp)



ส่วน "การจัดการกระเป๋าเงิน" แสดงข้อมูลสำคัญเกี่ยวกับการกำหนดค่าของคุณ:




- แหล่งเงินทุน** : LndBtcRestWallet (เชื่อมต่อโดยตรงกับโหนด LND Umbrel ของคุณ)
- ยอดคงเหลือของโหนด** : สภาพคล่องทั้งหมดที่มีอยู่ในช่องทาง Lightning ของคุณ
- LNbits Balance**: เงินที่จัดสรรให้กับระบบ LNbits (เริ่มต้น 0 sats)



คุณสามารถใช้สภาพคล่องของโหนด Umbrel ของคุณได้โดยตรงสำหรับกระเป๋าเงิน LNbits ทั้งหมดที่คุณสร้าง ไม่จำเป็นต้องมีการกำหนดค่าเพิ่มเติม - LNbits พร้อมใช้งานแล้ว



### การจัดการผู้ใช้



หนึ่งในคุณสมบัติที่ทรงพลังที่สุดของ LNbits คือความสามารถในการสร้างผู้ใช้ที่เป็นอิสระหลายคน แต่ละคนมีการยืนยันตัวตนด้วยรหัสผ่านและกระเป๋าเงินที่แยกออกจากกัน สถาปัตยกรรมนี้ทำให้สามารถใช้ประโยชน์จากสภาพคล่องของโหนด Umbrel ของคุณ ในขณะที่เสนอบัญชีย่อยที่แยกออกจากกันอย่างสมบูรณ์สำหรับการใช้งานที่แตกต่างกัน: ธุรกิจ, ครอบครัว, พนักงาน, โครงการ, ฯลฯ



ในเมนูด้านข้าง คลิกที่ "Users" เพื่อเข้าสู่การจัดการผู้ใช้ คลิกที่ "CREATE ACCOUNT" เพื่อเพิ่มผู้ใช้ใหม่



![Gestion des utilisateurs](assets/fr/05.webp)



กรอกแบบฟอร์มการสร้างผู้ใช้:




- ชื่อผู้ใช้**: ชื่อผู้ใช้เข้าสู่ระบบ (ตัวอย่าง: "satoshi")
- ตั้งรหัสผ่าน**: เปิดใช้งานตัวเลือกนี้เพื่อตั้งรหัสผ่านสำหรับการยืนยันตัวตน
- รหัสผ่าน** และ **ยืนยันรหัสผ่าน**: ตั้งรหัสผ่านสำหรับผู้ใช้นี้



![Création utilisateur satoshi](assets/fr/06.webp)



ฟิลด์ที่ไม่บังคับ (Nostr Public Key, อีเมล, ชื่อ, นามสกุล) สามารถเว้นว่างไว้สำหรับการกำหนดค่าขั้นต่ำ คลิกที่ "CREATE ACCOUNT" เพื่อยืนยัน



![Confirmation utilisateur créé](assets/fr/07.webp)



ผู้ใช้ใหม่ของคุณจะปรากฏในรายการผู้ใช้พร้อมกับตัวระบุเฉพาะและชื่อผู้ใช้ของเขาหรือเธอ



![Liste des utilisateurs](assets/fr/08.webp)



**จุดสำคัญ**: ผู้ใช้แต่ละคนสามารถเข้าสู่ระบบได้อย่างอิสระด้วยรหัสผ่านของตนเอง ผู้ดูแลระบบ SuperUser ยังคงควบคุมได้อย่างเต็มที่ผ่านทางอินเทอร์เฟซการจัดการ



### การจัดการผู้ใช้ wallet



ตอนนี้ที่ได้สร้างผู้ใช้ "satoshi" แล้ว คุณจำเป็นต้องมอบหมาย wallet Lightning ให้กับเขา คลิกที่ไอคอน wallet (ไอคอนที่สอง) สำหรับผู้ใช้ที่เกี่ยวข้อง จากนั้นคลิกที่ "CREATE NEW WALLET"



![Gestion des wallets](assets/fr/09.webp)



กล่องโต้ตอบจะขอให้คุณตั้งชื่อ wallet ป้อนชื่อที่อธิบายได้ (เช่น "Wallet ของ Satoshi") และเลือกสกุลเงินที่แสดง (CUC, USD, EUR, ฯลฯ)



![Création wallet](assets/fr/10.webp)



คลิกที่ "CREATE" LNbits สร้าง wallet Lightning ที่ใช้งานได้สำหรับผู้ใช้นี้ทันที



![Confirmation wallet créé](assets/fr/11.webp)



ตอนนี้คุณจะเห็นกระเป๋าเงินสองใบที่มีอยู่: wallet "LNbits wallet" ที่ถูกสร้างขึ้นโดยอัตโนมัติ และ "Wallet Of Satoshi" ใหม่ เพื่อให้ง่ายต่อประสบการณ์ของผู้ใช้ คุณสามารถลบ wallet ที่เป็นค่าเริ่มต้นได้โดยคลิกที่ไอคอนลบ (ถังขยะสีแดง)



![Wallet final unique](assets/fr/12.webp)



ผู้ใช้ "satoshi" ตอนนี้มี wallet ที่ระบุชัดเจนเพียงหนึ่งเดียว ผู้ใช้ wallet แต่ละคนดำเนินการอย่างอิสระโดยสมบูรณ์ในขณะที่ใช้สภาพคล่องของโหนด LND ที่อยู่ภายใต้ของคุณ



**แนวคิดหลัก**: กระเป๋าเงินทั้งหมดนี้ใช้สภาพคล่องทั่วโลกของโหนด Umbrel ของคุณ คุณไม่จำเป็นต้องสร้างช่องทาง Lightning ใหม่สำหรับแต่ละ wallet - LNbits ทำหน้าที่เป็นชั้นบัญชีอัจฉริยะที่จัดการการจัดสรรเงินทุนภายในโครงสร้างพื้นฐาน Lightning ที่มีอยู่ของคุณ นั่นคือพลังของระบบหลาย-wallet ของ LNbits



### เข้าสู่ระบบผู้ใช้



ออกจากระบบบัญชี SuperUser (ไอคอนที่มุมขวาบน) และกลับไปที่หน้าเข้าสู่ระบบ LNbits ตอนนี้คุณสามารถเข้าสู่ระบบด้วยข้อมูลประจำตัวของผู้ใช้ใหม่ได้แล้ว



![Connexion utilisateur satoshi](assets/fr/13.webp)



ป้อนชื่อผู้ใช้ ("satoshi") และรหัสผ่านที่กำหนดไว้ก่อนหน้านี้ จากนั้นคลิกที่ "LOGIN" ผู้ใช้จะเข้าถึง wallet ส่วนตัวของเขาโดยตรง ซึ่งแยกออกจากอินเทอร์เฟซการจัดการโดยสิ้นเชิง



### Interface จาก wallet ผู้ใช้



เมื่อเข้าสู่ระบบแล้ว ผู้ใช้จะเข้าถึงอินเทอร์เฟซ wallet Lightning ของเขาได้อย่างสมบูรณ์



![Interface wallet utilisateur](assets/fr/14.webp)



อินเทอร์เฟซมีคุณสมบัติ :




- ยอดคงเหลือปัจจุบัน**: แสดงใน sats และในสกุลเงินที่เลือก (CUC ในตัวอย่างนี้)
- การดำเนินการหลัก**: วางคำขอ, สร้างใบแจ้งหนี้, ไอคอน QR (สแกนด่วน)
- ประวัติการทำธุรกรรม** : รายการทั้งหมดของการชำระเงินและการรับเงิน
- แผงด้านขวา**: ตัวเลือกการกำหนดค่าและการเข้าถึง



### การเข้าถึง wallet ผ่านมือถือ



แผงด้านขวามือมีคุณสมบัติที่ใช้งานได้จริงเป็นพิเศษ: การเข้าถึง wallet ผ่านมือถือ เปิดส่วน "การเข้าถึงผ่านมือถือ" เพื่อค้นหาตัวเลือกที่มีอยู่



![Mobile Access](assets/fr/15.webp)



LNbits มีหลายวิธีในการใช้ wallet นี้บนสมาร์ทโฟน:



**ตัวเลือกที่ 1: แอปพลิเคชันมือถือที่เข้ากันได้




- ดาวน์โหลด **Zeus** หรือ **BlueWallet** จาก App Store หรือ Google Play
- เปิดใช้งานส่วนขยาย **LndHub** ใน LNbits สำหรับ wallet นี้
- สแกน QR โค้ด LndHub ด้วยแอปมือถือเพื่อเชื่อมต่อ wallet



**ตัวเลือกที่ 2: เข้าถึงโดยตรงผ่านเบราว์เซอร์มือถือ**




- QR โค้ดที่แสดงใน "ส่งออกไปยังโทรศัพท์ด้วย QR โค้ด" มี URL เต็มของ wallet พร้อมการตรวจสอบสิทธิ์ในตัว
- สแกน QR โค้ดนี้จากสมาร์ทโฟนของคุณเพื่อเปิด wallet โดยตรงในเบราว์เซอร์มือถือของคุณ
- เพิ่มหน้าไปยังหน้าจอหลักเพื่อการเข้าถึงที่รวดเร็ว



**ความปลอดภัยที่สำคัญ**: URL นี้ประกอบด้วยกุญแจ API สำหรับการเข้าถึงเต็มรูปแบบของ wallet ห้ามแชร์ต่อสาธารณะ จัดการ QR code นี้เหมือนกับที่คุณจัดการกุญแจส่วนตัว Bitcoin ของคุณ - ใครก็ตามที่สแกน QR code นี้จะได้รับการเข้าถึงเต็มรูปแบบของ wallet



คุณสมบัติมือถือนี้จะเปลี่ยนอินสแตนซ์ LNbits Umbrel ของคุณให้กลายเป็นเซิร์ฟเวอร์ Lightning wallet ที่แท้จริงสำหรับคุณและเพื่อน ๆ ของคุณ ในขณะที่ยังคงรักษาอธิปไตยอย่างสมบูรณ์เหนือเงินทุนของคุณด้วยโหนดที่โฮสต์ด้วยตนเอง



### การแชร์การเข้าถึงของผู้ใช้



กรณีการใช้งานหลักสำหรับการกำหนดค่าผู้ใช้หลายคนนี้คือ **การแชร์กระเป๋าเงินกับครอบครัวหรือคนใกล้ชิดของคุณ** เมื่อคุณสร้างผู้ใช้ด้วย wallet เฉพาะ (เช่น "satoshi" ในตัวอย่างของเรา) คุณสามารถแชร์ข้อมูลการเข้าสู่ระบบเหล่านี้กับสมาชิกในครัวเรือนที่คุณไว้วางใจได้



**การเข้าถึงความปลอดภัยบน Umbrel**: การเข้าถึงอินสแตนซ์ LNbits ของคุณบน Umbrel ได้รับการป้องกันตามธรรมชาติ เนื่องจากสามารถเข้าถึงได้เฉพาะ:




- บนเครือข่ายท้องถิ่นของคุณ** : สมาชิกในครัวเรือนของคุณที่เชื่อมต่อกับเครือข่าย WiFi/Ethernet เดียวกันสามารถเข้าถึงอินสแตนซ์ได้
- ผ่าน VPN**: หากคุณใช้ VPN เช่น Tailscale ที่กำหนดค่าไว้บนเซิร์ฟเวอร์ Umbrel ของคุณ ผู้ใช้ที่ได้รับอนุญาตสามารถเข้าถึงระยะไกลได้อย่างปลอดภัย



การป้องกันสองชั้นนี้ (การเข้าถึงเครือข่าย + การยืนยันตัวตนของผู้ใช้) ทำให้ตัวเลือก "อนุญาตให้สร้างผู้ใช้ใหม่" มีความสำคัญน้อยลงบน Umbrel เฉพาะผู้ที่มีสิทธิ์เข้าถึงเครือข่ายหรือ VPN ของคุณเท่านั้นที่สามารถเข้าถึงอินเทอร์เฟซการเข้าสู่ระบบได้



**สถานการณ์ทั่วไป**: คุณสร้างบัญชี "พ่อ", บัญชี "แม่", บัญชี "ธุรกิจ" และอื่นๆ สมาชิกในครอบครัวแต่ละคนมี wallet Lightning ของตัวเองที่แยกออกมา ในขณะที่ได้รับประโยชน์จากสภาพคล่องที่แชร์ของโหนด Umbrel ของคุณ เพียงแชร์ชื่อผู้ใช้และรหัสผ่าน - ผู้ใช้สามารถเชื่อมต่อจากอุปกรณ์ใดก็ได้ในเครือข่ายท้องถิ่นของคุณหรือผ่าน Tailscale VPN ของคุณ โปรดดูบทแนะนำ Tailscale เฉพาะของเราสำหรับข้อมูลเพิ่มเติม:



https://planb.academy/tutorials/computer-security/communication/tailscale-9acbd7de-04d9-40f6-ab80-35f0dfedb632

### สำรวจส่วนขยายที่มีอยู่



กลับไปที่อินเทอร์เฟซ SuperUser และเข้าถึงเมนู "Extensions" ในแผงด้านซ้ายมือเพื่อค้นพบระบบนิเวศส่วนขยาย LNbits ทั้งหมด



![Extensions disponibles](assets/fr/16.webp)



LNbits นำเสนอแคตตาล็อกส่วนขยายที่หลากหลายซึ่งเปลี่ยนอินสแตนซ์ของคุณให้เป็นแพลตฟอร์มบริการ Lightning ที่แท้จริง:





- Jukebox**: ระบบตู้เพลงที่ใช้พลังงานจาก sats (การชำระเงินผ่าน Spotify)
- ตั๋วสนับสนุน**: ระบบสนับสนุนแบบชำระเงิน (รับ satss เพื่อตอบคำถาม)
- TPoS**: เครื่องรับชำระเงินแบบพกพาที่ปลอดภัยสำหรับผู้ค้าปลีก
- ผู้จัดการผู้ใช้**: การจัดการผู้ใช้ขั้นสูงและ wallet (ซึ่งเราเพิ่งใช้)
- กิจกรรม**: การขายและการตรวจสอบบัตรเข้างาน
- LNURLDevices**: การจัดการจุดขาย, ตู้เอทีเอ็ม, สวิตช์ที่เชื่อมต่อ
- SMTP**: เปิดใช้งานให้ผู้ใช้ส่งอีเมลและรับ satss
- Boltcards**: การตั้งโปรแกรมบัตร NFC สำหรับการชำระเงินแบบแตะเพื่อจ่ายด้วย Lightning
- NostrNip5**: สร้างที่อยู่ NIP5 สำหรับโดเมนของคุณ
- Splitpayments**: การกระจายการชำระเงินอัตโนมัติระหว่างกระเป๋าเงินหลายใบ



ส่วนขยายแต่ละรายการสามารถเปิดใช้งานได้ด้วยการคลิกเพียงครั้งเดียวจากอินเทอร์เฟซนี้ ส่วนขยายที่มีเครื่องหมาย "FREE" นั้นไม่มีค่าใช้จ่าย ในขณะที่บางรายการมีให้ในรูปแบบ "PAID" สำรวจแคตตาล็อกเพื่อระบุรายการที่ตรงกับความต้องการของคุณ - ไม่ว่าจะเพื่อธุรกิจ การจัดการครอบครัว หรือทดลองใช้ความสามารถของ Lightning Network



## ข้อดีและข้อจำกัด



**ประโยชน์**: อธิปไตยทางการเงิน (การควบคุมเงิน/กุญแจ/ข้อมูลทั้งหมด), ความยืดหยุ่นทางสถาปัตยกรรม (การย้าย VPS→full node โดยไม่สูญเสียข้อมูล), ระบบขยายมืออาชีพ, อินเทอร์เฟซที่ใช้งานง่าย.



**ข้อจำกัด** : ซอฟต์แวร์ในรุ่นเบต้า (ระวังเรื่องจำนวนเงิน), ความปลอดภัยอยู่ภายใต้ความรับผิดชอบของผู้ดูแลระบบ, URL ที่มีคีย์ API ที่ละเอียดอ่อน (จำเป็นต้องใช้ HTTPS), การจัดการหลายผู้ใช้หมายถึงความรับผิดชอบในการดูแลรักษา.



## แนวทางปฏิบัติที่ดีที่สุด



**Backups**: Phoenixd Seed/LND credentials, LNbits database, .env files. Automate daily, keep off production server, encrypted. Test restores regularly.



**การบำรุงรักษา**: ตรวจสอบการอัปเดตเป็นประจำ (LNbits, Lightning backend, ระบบปฏิบัติการ) ตรวจสอบบันทึกการเผยแพร่ทุกครั้งก่อนการอัปเดตใหญ่





- บน Umbrel**: App Store จะแจ้งเตือนคุณโดยอัตโนมัติเกี่ยวกับเวอร์ชันใหม่ ซิงโครไนซ์ส่วนขยายผ่าน "จัดการส่วนขยาย" > "อัปเดตทั้งหมด" ตรวจสอบการรวมฐานข้อมูล SQLite ในการสำรองข้อมูลอัตโนมัติของ Umbrel
- บน VPS**: อัปเดตด้วยตนเองโดยใช้ `cd lnbits && git pull && uv sync --all-extras && sudo systemctl restart lnbits`. ตรวจสอบบันทึกของระบบ: `sudo journalctl -u lnbits -f`.

LNbits การโฮสต์ด้วยตนเองเสนอเส้นทางที่เป็นรูปธรรมสู่ความเป็นอิสระทางการเงินของ Lightning VPS+Phoenixd เสนอทางออกที่มีน้ำหนักเบาสำหรับบริการที่รวดเร็ว Umbrel การรวมเข้ากับโหนด Bitcoin ที่มีอยู่ได้อย่างเต็มรูปแบบ สถาปัตยกรรมที่ปรับขนาดได้ช่วยให้พัฒนาจาก wallet แบบหลายผู้ใช้ที่เรียบง่ายไปสู่กรณีการใช้งานทางธุรกิจที่ซับซ้อนได้



การโฮสต์ด้วยตนเองหมายถึงความรับผิดชอบ: สำรองข้อมูลเมล็ดพันธุ์, ปกป้องการเข้าถึง, เริ่มต้นด้วยจำนวนที่พอประมาณ ด้วยการป้องกันเหล่านี้ LNbits กลายเป็นโซลูชันที่แข็งแกร่งสำหรับเศรษฐกิจ Lightning ในขณะที่ยังคงรักษาการกระจายอำนาจและความเป็นอิสระ
