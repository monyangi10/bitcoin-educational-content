![cover](assets/cover.webp)



LNbits یک رابط وب متن‌باز است که هر بک‌اند لایتنینگ (LND، Core Lightning، Phoenixd) را به یک پلتفرم خدمات کامل تبدیل می‌کند. این راه‌حل خودمیزبان به شما امکان می‌دهد تا چندین پورتفولیو لایتنینگ را به‌صورت مجزا مدیریت کنید، نقاط فروش را مستقر کنید، سیستم‌های اهدایی یا خدمات صدور صورت‌حساب ایجاد کنید، در حالی که کنترل کامل بر روی وجوه خود را حفظ می‌کنید.



این آموزش دو روش نصب را پوشش می‌دهد: **VPS Ubuntu با Phoenixd** (راه‌حل سبک بدون یک نود کامل Bitcoin) و **Umbrel** (ادغام با نود LND موجود شما). برخلاف آموزش کلی LNbits آکادمی Plan ₿ که مفاهیم و افزونه‌ها را پوشش می‌دهد، این راهنما بر روی روش‌های نصب فنی گام به گام تمرکز دارد.



## LNbits چیست؟



LNbits یک سیستم حسابداری Lightning است که با استفاده از Python (FastAPI) توسعه یافته و به یک بک‌اند موجود (LND، Core Lightning، Phoenixd) متصل می‌شود. برخلاف نودهای سنتی Lightning، LNbits یک رابط کاربری قابل دسترس برای مدیریت چندین پورتفولیو مجزا با کلیدهای API خود ارائه می‌دهد. شما می‌توانید زیرحساب‌هایی برای خانواده، کارمندان یا پروژه‌های خود ایجاد کنید، بدون اینکه به آن‌ها دسترسی به تمام وجوه خود بدهید.



معماری جدا شده اطلاعات را در SQLite (پیش‌فرض) یا PostgreSQL (تولید) ذخیره می‌کند، در حالی که وجوه توسط بک‌اند لایتنینگ شما مدیریت می‌شود. این جداسازی قابلیت حمل را تضمین می‌کند: شما می‌توانید از Phoenixd به LND مهاجرت کنید بدون اینکه داده‌های کاربری خود را از دست بدهید.



## ویژگی‌های کلیدی



LNbits یک **سیستم افزونه** چندمنظوره ارائه می‌دهد: TPoS (نقطه فروش)، Paywall (کسب درآمد از محتوا)، Events (بلیط‌فروشی)، LndHub (سرور برای BlueWallet)، Bolt Cards (پرداخت‌های NFC)، Split Payments (توزیع خودکار)، و User Manager (مدیریت کاربران با احراز هویت).



**داشبورد** موجودی‌های لحظه‌ای، تاریخچه تراکنش و ابزارهای صدور صورت‌حساب را نمایش می‌دهد. هر wallet دارای یک URL منحصر به فرد است که شامل کلیدهای API آن می‌باشد و امکان دسترسی بدون ورود به سیستم سنتی را فراهم می‌کند. سیستم کلید API سه‌سطحی** (مدیر، صورت‌حساب، فقط‌خواندنی) کنترل دقیق مجوزها را برای یکپارچه‌سازی‌های امن ارائه می‌دهد.



LNbits به صورت بومی **LNURL** (LNURL-pay، LNURL-withdraw، LNURL-auth) را پیاده‌سازی می‌کند و از **Lightning Address** پشتیبانی می‌کند، که سازگاری با کیف‌پول‌های مدرن Lightning را تضمین کرده و استقرار خدمات حرفه‌ای را تسهیل می‌کند.

## نصب بر روی Ubuntu VPS با Phoenixd



### مرحله 1: ایمن‌سازی سرور VPS



**قبل از هرگونه نصب**، باید سرور VPS اوبونتو خود را مطابق با قوانین هنر ایمن کنید. این مرحله برای محافظت از زیرساخت و وجوه Lightning شما **حیاتی** است.



در اینجا یک راهنمای دقیق برای کمک به شما در شروع کار آمده است: **[پیکربندی اولیه سرور اوبونتو - راهنمای گام به گام](https://danielpcostas.dev/ubuntu-server-initial-configuration-a-step-by-step-guide/)** توسط دانیل پی. کوستاس.



این راهنما شامل پیکربندی کاربر، SSH امن، فایروال (UFW)، fail2ban، به‌روزرسانی‌های خودکار و روش‌های خوب امنیت سیستم است.



### مرحله 2: نصب Phoenixd



پس از ایمن‌سازی سرور خود، باید Phoenixd را نصب و پیکربندی کنید. آکادمی Plan ₿ یک آموزش جامع اختصاصی ارائه می‌دهد که نصب، تولید seed و پیکربندی سرویس systemd را پوشش می‌دهد:



https://planb.academy/tutorials/node/lightning-network/phoenixd-beb86edd-f9c0-4bec-ad36-db234c88e7b1

پس از راه‌اندازی Phoenixd (با استفاده از `./phoenix-cli getinfo` بررسی کنید)، **رمز عبور HTTP** را در `~/.phoenix/phoenix.conf` یادداشت کنید - برای اتصال LNbits به Phoenixd به آن نیاز خواهید داشت.



### استقرار LNbits



نصب UV و کلون LNbits :


```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
export PATH="$HOME/.local/bin:$PATH"
git clone https://github.com/lnbits/lnbits.git && cd lnbits
uv sync --all-extras
```



پیکربندی بک‌اند Phoenixd:


```bash
cp .env.example .env && nano .env
```



به `.env` اضافه کنید :


```
LNBITS_BACKEND_WALLET_CLASS=PhoenixdWallet
PHOENIXD_API_ENDPOINT=http://127.0.0.1:9740
PHOENIXD_API_PASSWORD=<mot-de-passe-phoenix.conf>
```



با `uv run lnbits --host 0.0.0.0 --port 5000` تست کنید سپس یک سرویس systemd با `Wants=phoenixd.service` ایجاد کنید.



## راه‌اندازی اولیه و اولین استفاده



### فعال‌سازی SuperUser



رابط کاربری مدیر را در `.env` فعال کنید:


```
LNBITS_ADMIN_UI=true
```



راه‌اندازی مجدد LNbits (`sudo systemctl restart lnbits`) و بازیابی شناسه SuperUser:


```bash
cat ~/lnbits/data/.super_user
```



به `http://<IP-VPS>:5000/wallet?usr=<SuperUserID>` برای پنل مدیریت بروید. منوی "سرور" به شما اجازه می‌دهد منابع مالی، افزونه‌ها و حساب‌های کاربری را پیکربندی کنید.



### ایجاد حساب کاربری امن



**مهم برای نمایش عمومی**: اگر شما در حال نمایش نمونه LNbits خود در یک نام دامنه عمومی قابل دسترسی از اینترنت هستید، **ضروری** است که ایجاد رایگان حساب‌های کاربری را غیرفعال کنید.



از رابط کاربری مدیریت SuperUser، به "تنظیمات" و سپس به بخش "مدیریت کاربران" بروید. در آنجا گزینه "اجازه ایجاد کاربران جدید" را خواهید یافت.



![Gestion des utilisateurs - Sécurité](assets/fr/17.webp)



**برای یک نمایشگاه عمومی با نام دامنه** :




- شما باید گزینه **"اجازه ایجاد کاربران جدید"** را غیرفعال کنید.
- بدون این حفاظت، هر کسی در اینترنت می‌تواند یک حساب کاربری در نمونه شما ایجاد کند.
- یک مهاجم می‌تواند حساب‌هایی ایجاد کند و بدون اطلاع شما از نقدینگی نود لایتنینگ شما استفاده کند.
- شما باید حساب‌های کاربری را به صورت دستی از رابط کاربری SuperUser ایجاد کنید.



**فقط برای استفاده محلی** :




- این گزینه کمتر بحرانی است اگر نمونه شما فقط به صورت محلی قابل دسترسی باشد (http://localhost:5000)
- با این حال، غیرفعال کردن این گزینه یک عمل ایمنی کلی خوب است



پس از پیکربندی، تنها مدیر SuperUser می‌تواند از طریق رابط "کاربران" حساب‌های کاربری جدید ایجاد کند. این رویکرد کنترل کامل بر اینکه چه کسی می‌تواند به زیرساخت Lightning شما دسترسی داشته باشد و از وجوه شما استفاده کند را تضمین می‌کند.



### باز کردن اولین کانال



Phoenixd به‌طور خودکار کانال‌ها را از طریق نقدینگی خودکار مدیریت می‌کند. یک فاکتور Lightning به مبلغ ~30,000 sats از LNbits تولید کنید و آن را از یک wallet دیگر پرداخت کنید. Phoenixd به‌طور خودکار یک کانال به ACINQ باز می‌کند. هزینه باز کردن (~20-23k sats) کسر می‌شود، و موجودی باقی‌مانده (~7-10k sats) پس از تأیید on-chain ظاهر می‌شود.



وضعیت را با `./phoenix-cli getinfo` بررسی کنید. سپس، برای کنترل باز شدن کانال‌ها، غیرفعال کردن خودکار نقدینگی (`auto-liquidity=off` در `phoenix.conf`) را در نظر بگیرید.



### نمایش عمومی و HTTPS



**مهم**: استفاده از HTTPS برای نمایش عمومی الزامی است (امنیت کلید API + سازگاری LNURL). این مرحله را فقط برای استفاده محلی نادیده بگیرید.



**Caddy (توصیه‌شده)**: SSL خودکار. `sudo apt install -y caddy`، فایل `/etc/caddy/Caddyfile` را ویرایش کنید:


```
votre-domaine.com {
reverse_proxy 127.0.0.1:5000
}
```


راه‌اندازی مجدد: `sudo systemctl restart caddy`.



**Nginx** : کنترل بیشتر. نصب `nginx certbot python3-certbot-nginx`، ایجاد `/etc/nginx/sites-available/lnbits` :


```nginx
server {
listen 80;
server_name votre-domaine.com;
location / {
proxy_pass http://127.0.0.1:5000;
proxy_set_header Host $host;
proxy_set_header X-Forwarded-Proto $scheme;
}
}
```


فعال‌سازی: `sudo ln -s /etc/nginx/sites-available/lnbits /etc/nginx/sites-enabled/ && sudo nginx -t && sudo systemctl reload nginx && sudo certbot --nginx -d your-domain.com`



به `.env` اضافه کنید: `FORWARDED_ALLOW_IPS=*`



## نصب Umbrel



### استقرار از اپ استور



به فروشگاه برنامه Umbrel بروید، "LNbits" را جستجو کنید و روی "نصب" کلیک کنید.



![Installation LNbits Umbrel](assets/fr/01.webp)



آمبرل به‌طور خودکار وابستگی‌های مورد نیاز را بررسی می‌کند. LNbits به نود لایتنینگ (LND) برای کار نیاز دارد. اگر نود لایتنینگ شما در حال حاضر عملیاتی است، روی "نصب LNbits" کلیک کنید تا تأیید شود.



![Dépendances LNbits](assets/fr/02.webp)



آمبرل تصویر داکر را دانلود می‌کند، به‌طور خودکار اتصالات با LND را پیکربندی می‌کند و کانتینر را راه‌اندازی می‌کند (۲-۵ دقیقه). نصب به‌طور کامل در پس‌زمینه انجام می‌شود.



### پیکربندی اولیه SuperUser



در اولین راه‌اندازی، LNbits از شما می‌خواهد که حساب کاربری SuperUser مدیر را ایجاد کنید. یک نام کاربری و رمز عبور امن وارد کنید تا دسترسی به رابط مدیریت محافظت شود.



![Configuration SuperUser](assets/fr/03.webp)



**مهم**: این حساب کاربری SuperUser دارای تمامی امتیازات در نمونه LNbits شما است. یک رمز عبور قوی انتخاب کنید و آن را ایمن نگه دارید.



پس از ایجاد حساب کاربری، به‌طور خودکار به رابط اصلی مدیریت هدایت می‌شوید. Umbrel قبلاً LND را به‌عنوان منبع تأمین مالی شما تنظیم کرده است - تمام پرداخت‌های Lightning از طریق کانال‌های موجود شما انجام خواهد شد.



### دسترسی به رابط کاربری مدیر



در منوی سمت چپ، روی "Settings" کلیک کنید تا به پنل کامل مدیریت دسترسی پیدا کنید.



![Interface Settings](assets/fr/04.webp)



بخش "مدیریت کیف‌پول‌ها" اطلاعات کلیدی درباره پیکربندی شما را نمایش می‌دهد:




- منبع تأمین مالی** : LndBtcRestWallet (اتصال مستقیم به نود LND Umbrel شما)
- موجودی نود** : کل نقدینگی موجود در کانال‌های لایتنینگ شما
- موجودی LNbits**: وجوه اختصاص داده شده به سیستم LNbits (در ابتدا 0 sats)



اکنون می‌توانید به‌طور مستقیم از نقدینگی نود Umbrel خود برای تمام کیف‌پول‌های LNbits که ایجاد می‌کنید، بهره‌برداری کنید. هیچ تنظیمات اضافی مورد نیاز نیست - LNbits در حال اجرا است.



### مدیریت کاربران



یکی از قدرتمندترین ویژگی‌های LNbits توانایی آن در ایجاد چندین کاربر مستقل است که هر کدام دارای احراز هویت رمز عبور و کیف پول‌های جداگانه هستند. این معماری امکان بهره‌برداری از نقدینگی نود Umbrel شما را فراهم می‌کند، در حالی که زیرحساب‌های کاملاً جداگانه‌ای برای استفاده‌های مختلف مانند کسب‌وکار، خانواده، کارمندان، پروژه‌ها و غیره ارائه می‌دهد.



در منوی کناری، روی "Users" کلیک کنید تا به مدیریت کاربران دسترسی پیدا کنید. روی "CREATE ACCOUNT" کلیک کنید تا یک کاربر جدید اضافه کنید.



![Gestion des utilisateurs](assets/fr/05.webp)



فرم ایجاد کاربر را پر کنید:




- نام کاربری**: نام کاربری ورود (مثال: "satoshi")
- تنظیم رمز عبور**: این گزینه را فعال کنید تا یک رمز عبور احراز هویت تنظیم کنید
- رمز عبور** و **تکرار رمز عبور**: رمز عبور را برای این کاربر تنظیم کنید



![Création utilisateur satoshi](assets/fr/06.webp)



فیلدهای اختیاری (کلید عمومی Nostr، ایمیل، نام، نام خانوادگی) می‌توانند برای پیکربندی حداقلی خالی بمانند. برای تأیید روی "ایجاد حساب" کلیک کنید.



![Confirmation utilisateur créé](assets/fr/07.webp)



کاربر جدید شما اکنون با شناسه منحصر به فرد و نام کاربری خود در لیست کاربران ظاهر می‌شود.



![Liste des utilisateurs](assets/fr/08.webp)



**نکته مهم**: هر کاربر می‌تواند به‌طور کاملاً مستقل با رمز عبور خود وارد شود. مدیر SuperUser از طریق رابط مدیریت کنترل کامل را حفظ می‌کند.



### مدیریت کاربر wallet



اکنون که کاربر "satoshi" ایجاد شده است، باید به او یک wallet Lightning اختصاص دهید. روی آیکون wallet (آیکون دوم) برای کاربر مربوطه کلیک کنید، سپس روی "CREATE NEW WALLET" کلیک کنید.



![Gestion des wallets](assets/fr/09.webp)



یک کادر محاوره‌ای از شما می‌خواهد که wallet را نام‌گذاری کنید. یک نام توصیفی وارد کنید (مثلاً "Wallet از Satoshi") و ارز نمایش را انتخاب کنید (CUC، USD، EUR، و غیره).



![Création wallet](assets/fr/10.webp)



روی "CREATE" کلیک کنید. LNbits بلافاصله یک wallet Lightning کاربردی برای این کاربر ایجاد می‌کند.



![Confirmation wallet créé](assets/fr/11.webp)



اکنون دو کیف پول موجود را مشاهده می‌کنید: کیف پول پیش‌فرض wallet "LNbits wallet" که به‌صورت خودکار ایجاد شده است، و کیف پول جدید "Wallet Of Satoshi". برای ساده‌سازی تجربه کاربری، می‌توانید کیف پول پیش‌فرض wallet را با کلیک بر روی آیکون حذف (سطل زباله قرمز) حذف کنید.



![Wallet final unique](assets/fr/12.webp)



کاربر "satoshi" اکنون یک wallet مشخص و واحد دارد. هر کاربر wallet به طور کامل به صورت مستقل عمل می‌کند در حالی که از نقدینگی گره LND زیرساختی شما استفاده می‌کند.



**مفهوم کلیدی**: همه این کیف‌پول‌ها نقدینگی جهانی نود Umbrel شما را به اشتراک می‌گذارند. شما برای هر wallet کانال‌های لایتنینگ جدیدی ایجاد نمی‌کنید - LNbits به عنوان یک لایه حسابداری هوشمند عمل می‌کند که تخصیص وجوه را در زیرساخت لایتنینگ موجود شما مدیریت می‌کند. این قدرت سیستم چند wallet LNbits است.

ویژگی‌های رابط:




- موجودی فعلی**: نمایش داده شده در sats و به ارز انتخابی (CUC در این مثال)
- اقدامات اصلی**: چسباندن درخواست، ایجاد فاکتور، آیکون QR (اسکن سریع)
- تاریخچه تراکنش** : لیست کامل تمامی پرداخت‌ها و دریافت‌ها
- پنل سمت راست**: گزینه‌های پیکربندی و دسترسی



### دسترسی موبایل به wallet



پنل سمت راست یک ویژگی بسیار کاربردی ارائه می‌دهد: دسترسی موبایل به wallet. بخش "دسترسی موبایل" را باز کنید تا گزینه‌های موجود را کشف کنید.



![Mobile Access](assets/fr/15.webp)



LNbits چندین روش برای استفاده از این wallet بر روی یک گوشی هوشمند ارائه می‌دهد:



**گزینه 1: برنامه‌های کاربردی موبایل سازگار**




- دانلود **Zeus** یا **BlueWallet** از اپ استور یا گوگل پلی
- افزونه **LndHub** را در LNbits برای این wallet فعال کنید.
- کد QR LndHub را با اپلیکیشن موبایل اسکن کنید تا به wallet متصل شوید.



**گزینه ۲: دسترسی مستقیم از طریق مرورگر موبایل**




- کد QR نمایش داده شده در "صادرات به تلفن با کد QR" حاوی URL کامل wallet با احراز هویت یکپارچه است.
- این کد QR را با گوشی هوشمند خود اسکن کنید تا wallet را مستقیماً در مرورگر موبایل خود باز کنید.
- افزودن صفحه به صفحه اصلی برای دسترسی سریع



**امنیت مهم**: این URL شامل کلیدهای API برای دسترسی کامل به wallet است. هرگز آن را به صورت عمومی به اشتراک نگذارید. با این کد QR همانند کلیدهای خصوصی Bitcoin خود رفتار کنید - هر کسی که این کد QR را اسکن کند به طور کامل به wallet دسترسی پیدا می‌کند.



این ویژگی موبایل، نمونه LNbits Umbrel شما را به یک سرور Lightning wallet واقعی برای شما و دوستانتان تبدیل می‌کند، در حالی که به لطف نود خود میزبان، حاکمیت کامل بر وجوه خود را حفظ می‌کنید.



### اشتراک‌گذاری دسترسی کاربر



مورد اصلی برای این پیکربندی چندکاربره **اشتراک‌گذاری کیف‌پول‌ها با خانواده یا دایره نزدیکان شما** است. هنگامی که یک کاربر با wallet اختصاصی (مانند "satoshi" در مثال ما) ایجاد کردید، می‌توانید این اطلاعات ورود را با اعضای مورد اعتماد خانواده خود به اشتراک بگذارید.



**دسترسی به امنیت در Umbrel**: دسترسی به نمونه LNbits شما در Umbrel به طور طبیعی محافظت شده است، زیرا تنها می‌توان به آن دسترسی داشت :




- در شبکه محلی شما** : اعضای خانواده شما که به همان شبکه WiFi/Ethernet متصل هستند می‌توانند به این نمونه دسترسی داشته باشند
- از طریق VPN**: اگر از VPN مانند Tailscale که بر روی سرور Umbrel شما پیکربندی شده استفاده کنید، کاربران مجاز می‌توانند به‌صورت امن به‌صورت از راه دور دسترسی پیدا کنند.



این لایه دوگانه حفاظت (دسترسی شبکه + احراز هویت کاربر) گزینه "اجازه ایجاد کاربران جدید" را در Umbrel کمتر حیاتی می‌کند. تنها افرادی که از قبل به شبکه یا VPN شما دسترسی دارند می‌توانند به رابط ورود دسترسی پیدا کنند.



**سناریوی معمول**: شما یک حساب "پدر"، یک حساب "مادر"، یک حساب "کسب‌وکار" و غیره ایجاد می‌کنید. هر عضو خانواده دارای wallet Lightning جداگانه خود است، در حالی که از نقدینگی مشترک نود Umbrel شما بهره‌مند می‌شود. به سادگی نام کاربری و رمز عبور را به اشتراک بگذارید - کاربر سپس می‌تواند از هر دستگاهی در شبکه محلی شما یا از طریق VPN Tailscale شما متصل شود. لطفاً برای اطلاعات بیشتر به آموزش اختصاصی Tailscale ما مراجعه کنید:



https://planb.academy/tutorials/computer-security/communication/tailscale-9acbd7de-04d9-40f6-ab80-35f0dfedb632

### کاوش در افزونه‌های موجود



به رابط کاربری SuperUser بازگردید و به منوی "Extensions" در پانل سمت چپ دسترسی پیدا کنید تا اکوسیستم کامل افزونه‌های LNbits را کشف کنید.



![Extensions disponibles](assets/fr/16.webp)



LNbits مجموعه‌ای غنی از افزونه‌ها را ارائه می‌دهد که نمونه شما را به یک پلتفرم واقعی خدمات Lightning تبدیل می‌کند:





- جعبه موسیقی**: سیستم جعبه موسیقی با قدرت sats (پرداخت‌های Spotify)
- تیکت‌های پشتیبانی**: سیستم پشتیبانی پولی (دریافت ساتس برای پاسخ به سوالات)
- TPoS**: ترمینال فروش نقطه‌ای امن و سیار برای خرده‌فروشان
- مدیر کاربر**: مدیریت کاربر پیشرفته و wallet (که به تازگی استفاده کرده‌ایم)
- رویدادها**: فروش و اعتبارسنجی بلیط‌های رویداد
- LNURLDevices**: مدیریت نقطه فروش، دستگاه‌های خودپرداز، سوئیچ‌های متصل
- SMTP**: به کاربران اجازه دهید ایمیل ارسال کنند و ساتس کسب کنند
- بولت‌کارت‌ها**: برنامه‌ریزی کارت‌های NFC برای پرداخت‌های لمسی لایتنینگ
- NostrNip5**: ایجاد آدرس‌های NIP5 برای دامنه‌های شما
- اسپلیت‌پیمنتس**: توزیع خودکار پرداخت‌ها بین چندین کیف پول



هر افزونه با یک کلیک از این رابط فعال می‌شود. افزونه‌هایی که با "FREE" علامت‌گذاری شده‌اند، رایگان هستند، در حالی که برخی به عنوان نسخه‌های "PAID" در دسترس هستند. کاتالوگ را مرور کنید تا آن‌هایی که با نیازهای شما مطابقت دارند را شناسایی کنید - چه برای کسب‌وکار، مدیریت خانواده، یا آزمایش قابلیت‌های Lightning Network.



## مزایا و محدودیت‌ها



**مزایا**: حاکمیت مالی (کنترل کامل بر وجوه/کلیدها/داده‌ها)، انعطاف‌پذیری معماری (مهاجرت بدون از دست دادن VPS→full node)، سیستم توسعه حرفه‌ای، رابط کاربری شهودی.



**محدودیت‌ها** : نرم‌افزار در نسخه بتا (احتیاط در مورد مقادیر)، امنیت تحت مسئولیت مدیر، URLهایی که حاوی کلیدهای حساس API هستند (الزامی بودن HTTPS)، مدیریت چند کاربره به معنای مسئولیت نگهداری است.



## بهترین روش‌ها



**پشتیبان‌گیری‌ها**: اعتبارنامه‌های Phoenixd Seed/LND، پایگاه داده LNbits، فایل‌های .env. به‌صورت روزانه خودکارسازی کنید، از سرور تولید دور نگه دارید، رمزگذاری شده. بازیابی‌ها را به‌طور منظم آزمایش کنید.



**نگهداری**: به‌طور منظم به‌روزرسانی‌ها را بررسی کنید (LNbits، بک‌اند Lightning، سیستم‌عامل). همیشه قبل از به‌روزرسانی‌های عمده، یادداشت‌های انتشار را بررسی کنید.





- روی Umbrel**: فروشگاه برنامه به‌طور خودکار شما را از نسخه‌های جدید مطلع می‌کند. افزونه‌ها را از طریق "مدیریت افزونه‌ها" > "به‌روزرسانی همه" همگام‌سازی کنید. بررسی کنید که آیا پایگاه داده SQLite در پشتیبان‌گیری‌های خودکار Umbrel گنجانده شده است یا خیر.
- روی VPS**: به‌صورت دستی به‌روزرسانی کنید با `cd lnbits && git pull && uv sync --all-extras && sudo systemctl restart lnbits`. لاگ‌های سیستم را نظارت کنید: `sudo journalctl -u lnbits -f`.



## نتیجه‌گیری



میزبانی شخصی LNbits مسیر مشخصی به سوی حاکمیت مالی لایتنینگ ارائه می‌دهد. VPS+Phoenixd یک راه‌حل سبک برای خدمات سریع ارائه می‌دهد، یکپارچگی کامل Umbrel با نود موجود Bitcoin. معماری مقیاس‌پذیر امکان تکامل از wallet چندکاربره ساده به موارد استفاده تجاری پیچیده را فراهم می‌کند.



میزبانی شخصی به معنای مسئولیت است: پشتیبان‌گیری از کلیدها، محافظت از دسترسی، با مقادیر کم شروع کنید. با این احتیاط‌ها، LNbits به یک راه‌حل قوی برای اقتصاد لایتنینگ تبدیل می‌شود، در حالی که تمرکززدایی و خودمختاری را حفظ می‌کند.
