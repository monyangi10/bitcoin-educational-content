## Introduction


Ce tutoriel explique comment utiliser [Divly](https://divly.com/) pour préparer un rapport fiscal Bitcoin (BTC). La préparation d'un rapport fiscal consiste à rassembler toutes les transactions BTC pertinentes au cours d'un exercice financier, à calculer les gains ou les revenus imposables et à exporter un rapport qui peut être soumis à l'administration fiscale locale.


Dans les juridictions où la Bitcoin est soumise à des règles fiscales, vous devez déclarer les bénéfices ou les pertes résultant de la cession de la BTC et les revenus perçus dans la BTC. L'utilisation d'un logiciel de déclaration fiscale vous permet d'organiser ces informations et de générer un rapport fiscal de manière efficace.


## Pour commencer


Pour commencer :



- Créez un compte Divly.
- Définissez votre **pays de résidence fiscale** et **monnaie locale**.
- Confirmez que ces paramètres correspondent à la juridiction dans laquelle vous déclarez vos impôts.


Divly utilise cette configuration pour appliquer les règles fiscales et les conversions de devises appropriées lors de la génération de votre rapport.


![img](assets/en/01.webp)


## Étape 1 - Importer toutes les transactions Bitcoin de vos wallet et de vos échanges


Toutes les transactions Bitcoin de l'année fiscale doivent être importées avant tout calcul de l'impôt.


Divly prend en charge plusieurs méthodes d'importation :



- API ou connexions OAuth** pour les échanges ou les services qui les prennent en charge
- Téléchargement de fichiers CSV** à partir de wallet ou d'échanges
- Saisie manuelle** pour toutes les transactions non couvertes par les méthodes automatisées


Veillez à importer **toutes les entrées et sorties BTC** relatives à la période fiscale que vous déclarez.


![img](assets/en/02.webp)


## Étape 2 - Examen des transactions importées


**Confirmez votre solde de crypto:**


Le mieux est de commencer par vérifier que le nombre total de crypto-monnaies que vous possédez correspond au nombre affiché dans Divly. Divly calcule vos avoirs en additionnant toutes les transactions que vous avez importées.


Pour ce faire, commencez par vous rendre sur la page "Aperçu". Vérifiez que chaque crypto listé correspond bien au nombre que vous possédez. Divly n'affiche pas vos devises fiduciaires dans l'aperçu, veuillez donc les ignorer dans cet exercice.


Vous pouvez filtrer par wallet si vous trouvez des problèmes. Cela vous aide à comprendre quels wallet peuvent être désynchronisés.


![img](assets/en/03.webp)


Après l'importation :



- Allez dans la section **Transactions**.
- Vérifiez que chaque transaction apparaît avec des dates et des montants corrects.
- Résoudre les avertissements de prix ou de base de coûts manquants, le cas échéant.


**Important:** Assurez-vous d'importer **TOUTES** vos transactions cryptographiques dans Divly avant de passer à l'étape suivante. Y compris les wallet froids ! Sinon, il y a un risque que vos impôts ne soient pas corrects.


![img](assets/en/04.webp)


## Étape 3 - Catégoriser les dépôts et les retraits pertinents


Différents types de transactions en crypto-monnaie peuvent avoir des implications fiscales distinctes. Il s'agit notamment d'activités telles que le don de crypto-monnaie, d'actifs perdus, de récompenses mining, de fork, d'airdrops et d'événements similaires. Il est important que toutes les transactions soient catégorisées avec précision.


Dans la plupart des cas, Divly attribue automatiquement les étiquettes correctes. Toutefois, lorsque les données de transaction disponibles sont insuffisantes, la classification automatique peut s'avérer impossible. Dans ce cas, il incombe à l'utilisateur d'attribuer manuellement le libellé approprié. Pour comprendre la signification et le traitement fiscal de chaque étiquette, veuillez vous référer à l'article d'aide correspondant.


Pour étiqueter les transactions, allez à la page Transactions. Sélectionnez une ou plusieurs transactions et choisissez l'étiquette correcte dans le menu déroulant en haut de la page.


![img](assets/en/05.webp)


## Étape 4 - Générer le rapport d'impôt


Une fois les transactions importées et classées :



- Allez à la section **Rapport d'impôt**.
- Sélectionnez l'**année fiscale** concernée.
- Examinez le résumé des gains, des pertes et des catégories de revenus calculés.


Le résumé agrège les événements imposables sur la base des données importées et des classifications.


L'interface des rapports fiscaux de Divly vous permet de confirmer que toutes les transactions ont été saisies avant l'exportation.


![img](assets/en/06.webp)


## Étape 5 - Exporter le rapport


Après examen :



- Exportez la déclaration fiscale BTC finalisée dans le format disponible pour votre pays.
- Enregistrez le fichier exporté ou imprimez-le pour le soumettre à votre administration fiscale.


![img](assets/en/07.webp)


En fonction de votre juridiction, il se peut que vous deviez suivre des instructions de soumission spécifiques ou utiliser des formulaires propres à votre pays pour vos données exportées. les [guides nationaux de Divly] (https://divly.com/en/guides) peuvent vous aider à suivre les étapes de soumission si nécessaire.
