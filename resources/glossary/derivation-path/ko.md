---
term: 파생 경로
---

계층적 결정론(HD) 지갑의 맥락에서 파생 경로는 마스터 키에서 자식 키를 파생하는 데 사용되는 인덱스의 순서를 의미합니다. BIP32에 설명된 이 경로는 자식 키를 도출하기 위한 트리 구조를 식별합니다. 파생 경로는 슬래시로 구분된 일련의 인덱스로 표시되며, 항상 마스터 키(`m/`로 표시)로 시작합니다. 예를 들어, 일반적인 경로는 `m/84'/0'/0'/0/0`일 수 있습니다. 각 파생 수준은 특정 깊이와 연관되어 있습니다:


- m /`은 마스터 개인키를 나타냅니다. 이 키는 Wallet에 고유하며 같은 깊이의 형제를 가질 수 없습니다. 마스터 키는 seed에서 직접 파생됩니다;

'm/목적'/`은 다음 표준을 식별하는 데 도움이 되는 파생 목적을 나타냅니다. 이 필드는 BIP43에 설명되어 있습니다. 예를 들어, Wallet이 BIP84 표준(SegWit V0)을 준수하는 경우 인덱스는 `84'가 됩니다;

**'m/목적' / Coin 유형' /`**는 암호화폐의 유형을 나타냅니다. 이를 통해 멀티 Coin Wallet에서 한 암호화폐 전용 브랜치와 다른 암호화폐 전용 브랜치를 명확하게 구분할 수 있습니다. Bitcoin 전용 인덱스는 **`0'`**입니다;

'm / 목적' / Coin 유형' / 계좌' / '는 계좌 번호를 나타냅니다. 이 깊이를 통해 Wallet을 다른 계정으로 쉽게 구분하고 구성할 수 있습니다. 이러한 계정은 `0'부터 번호가 매겨집니다. 확장 키(`xpub`, `xprv`...)는 이 깊이 수준에서 찾을 수 있습니다;

'm / 목적' / Coin 유형' / 계정' / 변경 /`은 경로를 나타냅니다. 깊이 3에 정의된 각 계정은 깊이 4에 외부 체인과 내부 체인('변경'이라고도 함)의 두 가지 경로가 있습니다. 외부 체인은 공개적으로 공유할 주소, 즉 Wallet 소프트웨어에서 '받기'를 클릭할 때 제공되는 주소를 도출합니다. 내부 체인은 공개적으로 교환할 목적이 아닌 주소, 주로 변경 주소를 도출합니다. 외부 체인은 인덱스 `0`으로 식별되고 내부 체인은 인덱스 `1`로 식별됩니다. 이 깊이부터는 더 이상 강화된 파생이 아닌 일반 파생(아포스트로피가 없음)을 수행한다는 것을 알 수 있습니다. 이 메커니즘 덕분에 모든 자식 공개키를 `xpub`에서 파생할 수 있습니다;



- 'm/목적' / Coin-type' / 계정' / 변경 / Address-index`는 단순히 수신 Address의 번호와 그 키 쌍을 나타내며, 같은 브랜치에서 같은 깊이에 있는 형제들과 구별하기 위해 사용됩니다. 예를 들어, 첫 번째 파생된 Address은 인덱스 `0`, 두 번째 Address은 인덱스 `1`...* 등입니다


예를 들어, 수신 Address의 유도 경로가 `m / 86' / 0' / 0' / 0 / 5`인 경우 다음 정보를 추론할 수 있습니다:


- 86'`은 BIP86(Taproot/SegWit V1)의 파생 표준을 따르고 있음을 나타냅니다;
- 0'`은 Bitcoin Address임을 나타냅니다;
- 0'`은 Wallet의 첫 번째 계정을 사용 중임을 나타냅니다;
- '0'은 외부 Address임을 나타냅니다;

'5'는 이 계정의 여섯 번째 외부 Address임을 나타냅니다.
