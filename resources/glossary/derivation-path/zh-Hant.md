---
term: 導出路徑
---

在 Hierarchical Deterministic (HD) 錢包中，派生路徑是指用於從主密鑰派生子密鑰的索引序列。在 BIP32 中描述，這個路徑識別了推導子金鑰的樹狀結構。派生路徑由一系列用斜線分開的索引表示，並且總是以主密鑰 (表示為 `m/`)開始。例如，一個典型的路徑可能是 `m/84'/0'/0'/0/0`。每個階層的衍生都有特定的深度：


- `m /` 表示主私人密碼匙。它對於 Wallet 是獨一無二的，不能在同一深度有兄弟姊妹。主密鑰直接來自 seed；
- `m / purpose' /` 表示衍生目的，有助於識別所遵循的標準。這個欄位在 BIP43 中有所描述。例如，如果 Wallet 遵循 BIP84 標準 (SegWit V0)，則索引為 `84'`；
- `m / purpose' / coin-type' /` 表示加密貨幣的類型。這樣可以清楚區分在多幣 Wallet 中專用於一種加密貨幣的分支和專用於另一種加密貨幣的分支。專用於 Bitcoin 的索引為 `0'`；
- `m / purpose' / coin-type' / account' /` 表示帳號。此深度可讓您輕鬆將 Wallet 區分為不同的帳戶。這些帳戶的編號從 `0'` 開始。擴充金鑰 (`xpub`、`xprv`...) 可在此深度層級找到；
- `m / purpose' / coin-type' / account' / change /` 表示路徑。深度 3 定義的每個帳戶在深度 4 有兩個路徑：外部鏈和內部鏈（也稱為 「變更」）。外部鏈衍生出打算公開分享的位址，也就是當我們點選 Wallet 軟體中的「接收」時提供給我們的位址。內部鏈衍生出不打算公開交換的位址，主要是變更位址。外部鏈用索引 `0`來識別，內部鏈用索引 `1`來識別。您會注意到，從這個深度開始，我們不再執行加固的衍生，而是正常的衍生 (沒有撇號)。正是由於這個機制，我們才能從它們的 `xpub` 中推導出所有子公鑰；



- `m / purpose' / coin-type' / account' / change / Address-index` 簡單地表示接收 Address 的編號和它的一對鍵值，以區別於同一分支上同一深度的兄弟姊妹。例如，第一個衍生 Address 的索引為 `0`，第二個 Address 的索引為 `1`，以此類推...


舉例來說，如果我的接收 Address 有派生路徑 `m / 86' / 0' / 0' / 0 / 5`，我們可以推導出以下資訊：


- `86'` 表示我們遵循 BIP86 的衍生標準 (Taproot / SegWit V1)；
- `0'`表示是 Bitcoin Address；
- `0'`表示我們在 Wallet 的第一個帳戶；
- `0`表示是外部 Address；
- `5` 表示這是此帳戶的第六個外部 Address。
