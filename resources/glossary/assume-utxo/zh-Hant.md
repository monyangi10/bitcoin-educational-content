---
term: 假設 UTXO
---

領先的 Bitcoin Core 用戶端中的配置參數，允許剛好初始化（但尚未進行 IBD）的節點，將交易和 UTXO 集的驗證延遲到指定的快照。此概念依賴使用 Core 提供的 UTXO 集 (特定時間所有現有 UTXO 的清單)，並假定其準確性，這可讓節點快速與累積工作最多的鏈同步。由於節點省略了冗長的 IBD 步驟，因此對使用者而言，它可以很快地投入運作。假設 UTXO 將同步化 (IBD) 分成兩部分：


- 首先，節點執行 Header First Sync（僅驗證標頭），並將 Core 提供的 UTXO 集視為有效；
- 然後，一旦開始運作，節點就會在背景驗證完整的區塊歷史，更新它自己驗證過的新 UTXO 集。如果這個新的 UTXO 集與 Core 提供的 UTXO 集不符，則會產生錯誤訊息。


因此，假設 UTXO 藉由延遲交易驗證程序和 UTXO 透過 Core 中提供的更新快照集，加速新 Bitcoin 節點的準備。
