---
term: 假設有效
---

大多數用戶端 Bitcoin Core 中的組態參數，允許剛好初始化（但尚未執行 IBD）的節點跳過驗證某個給定區塊之前的區塊中包含的所有交易的簽名。這個著名的區塊是由其標頭（即 Hash）的印記所定義。所選的區塊會隨著每個新版本的 Bitcoin Core 更新。在初始化時，如果節點已啟動此參數，就會檢查區塊標頭鏈，找出累積工作最多的分支。如果節點在其選擇的分支中檢測到 Core 提供的 Hash，則會省略前面區塊的簽章驗證。否則，節點將進行傳統同步 (IBD) 以自行驗證所有內容。


Assume Valid 的目標是在不影響安全性的前提下，加速節點初始同步的過程，假設網路中的大多數人在過去已經對這些交易進行了驗證。對於節點來說，唯一真正的折衷是，如果之前比特幣被盜，它將不會收到通知。但是，它仍然可以確保發行比特幣數量的準確性。節點在 Assume Valid 區塊之後會繼續驗證交易的簽名。這個方法是基於這樣的假設：如果一個交易已經被網絡接受了足夠長的時間而沒有爭議，那麼它就不太可能是欺詐的。
