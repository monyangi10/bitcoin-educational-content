---
term: BITCOIN IMPLEMENTATION
---

Refers to software that enforces and follows the rules defined by the Bitcoin protocol. When people say “Bitcoin”, they usually refer to the electronic cash system itself, a protocol that specifies a set of consensus rules. 
It is represented, in practice, by nodes forming a peer-to-peer network. The protocol itself is not specific code but rather a collection of broadly agreed-upon rules enforced by users through their nodes. 
In contrast, what does have actual source code are the Bitcoin node software clients, which implement those rules. These are independent implementations of the Bitcoin protocol that can connect to the rest of the Bitcoin network.

Among the Bitcoin implementations, Bitcoin Core is by far the most widely used, representing around 99% of the network's nodes. Minority implementations exist as well, such as BTCsuite, Bcoin, and Bitcoin Knots. Despite this diversity, Bitcoin Core’s dominance effectively gives it an almost exclusive influence on how the Bitcoin protocol is applied. As a result, Bitcoin Core is often considered to represent the Bitcoin protocol itself in practice.
