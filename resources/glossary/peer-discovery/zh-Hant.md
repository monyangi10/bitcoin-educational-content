---
term: 同行發現
---

Bitcoin 網路中的節點與其他節點連線以取得資訊的過程。當 Bitcoin 節點第一次啟動時，它沒有網路中其他節點的資訊。然而，它必須建立連線，才能與累積工作最多的 Blockchain 同步。按照優先順序，有幾種機制可用來發現這些對等節點：


- 節點一開始會查閱其本機的「peers.dat」檔案，這個檔案儲存了先前與它互動過的節點的資訊。如果節點是新的，這個檔案將會是空的，而過程將會進入下一步；
- 在 `peers.dat` 檔案中沒有資訊的情況下（這對於新啟動的節點是正常的），節點會向 DNS 種子執行 DNS 查詢。這些伺服器會提供推測為活動節點的 IP 位址清單，以建立連線。DNS 種子的位址是 Bitcoin 核心程式碼中的硬碼。這一步驟通常足以完成發現對等體；
- 如果 DNS 種子在 60 秒內沒有回應，節點就可以求助 seed 節點。這些是公開的 Bitcoin 節點，列在直接整合到 Bitcoin Core 原始碼的靜態清單中，清單上有近千個項目。新節點會使用此清單建立與網路的第一次連線，並取得其他節點的 IP 位址；
- 在前面所有方法都失敗的極小可能情況下，節點操作員總是可以選擇手動新增節點的 IP 位址來建立第一次連線。
