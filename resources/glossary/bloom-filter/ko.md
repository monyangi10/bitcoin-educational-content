---
term: 블룸 필터
---

블룸 필터는 요소가 집합의 멤버인지 여부를 테스트하는 데 사용되는 확률적 데이터 구조입니다.

전체 데이터 세트를 스캔하지 않고도 매우 빠르게 멤버십을 확인할 수 있으므로 메모리 효율성과 속도가 중요하고 작은 오류율을 허용할 수 있는 상황에서 특히 유용합니다.

블룸 필터는 오탐(필터가 세트에 없는 요소라고 표시하면 실제로는 없는 것임)은 없지만 오탐(요소가 있다고 잘못 표시할 수 있음)의 확률은 적습니다.

요소가 필터에 추가되면 여러 개의 Hash 함수가 비트 배열에서 generate 위치를 차지합니다. 멤버십 여부를 확인하기 위해 동일한 Hash 함수가 사용됩니다. 해당 비트가 모두 설정되어 있으면 해당 요소가 집합에 속해 있을 가능성이 높지만 오탐의 위험이 있습니다.

블룸 필터는 데이터베이스와 네트워크 시스템에서 널리 사용됩니다. 예를 들어, *구글의 빅테이블* 데이터베이스 관리 시스템에서 블룸 필터를 사용합니다. Bitcoin 프로토콜에서 블룸 필터는 BIP37에 명시된 대로 SPV 지갑(간편 결제 인증)에 사용됩니다.


> gW-3 트랜잭션의 맥락에서 블룸 필터 사용에 대해 구체적으로 이야기할 때 "트랜잭션 블룸 필터링"이라는 용어가 가끔 등장합니다
