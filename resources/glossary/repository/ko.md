---
term: 리포지토리
---

프로젝트의 버전 관리 정보가 저장되는 Git에서 사용되는 중앙 데이터 구조입니다. 리포지토리에는 모든 변경사항, 브랜치 및 태그의 전체 이력이 포함됩니다. 각 리포지토리는 커밋 기록과 함께 파일 및 폴더의 독립적인 모음으로, 시간 경과에 따른 변경 사항을 추적하고 협업할 수 있습니다. 예를 들어 Bitcoin core 리포지토리는 여기(https://github.com/Bitcoin/Bitcoin)의 GitHub에서 호스팅됩니다.


> 영어에서는 "리포지토리"라고 합니다. 일반적으로 Git 리포지토리를 지칭할 때 "repo"라는 약어를 사용하는 것이 일반적입니다*
