---
term: OUTPUT
---

In the context of Bitcoin, an output within a transaction refers to _Unspent Transaction Outputs_ (UTXOs) which are created as destination funds for payment. More precisely, it is a mechanism by which a transaction distributes funds. A transaction takes UTXOs, i.e. bits of bitcoins, as "inputs" and creates new UTXOs as "outputs". These outputs stipulate a certain quantity of bitcoins, often allocated to a specific address, as well as the conditions under which these funds can be spent at a later date.

The role of the Bitcoin transaction is therefore to consume UTXOs as inputs, and to create new UTXOs as outputs. The difference between the two corresponds to the transaction fees that can be recovered by the winning miner of the block. A UTXO is, in essence, the output of a previous transaction that has not yet been spent. Transaction outputs are therefore the creation of new UTXOs which, in turn, will potentially be used as inputs in future transactions.

From a broader perspective, in computer science, the term "output" generally refers to the data resulting from a function, algorithm or system. For example, when data is passed through a cryptographic hash function, this information is called "input", and the result is called "output".
