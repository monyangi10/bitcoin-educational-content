---
term: БАТАРЕЯ
---

В контексте языка сценариев, используемого для прикрепления условий расходования к Bitcoin UTXO, стек - это структура данных LIFO (*Last In, First Out*), используемая для хранения временных Elements во время выполнения сценария. Каждая операция в сценарии манипулирует этими стеками, где Elements могут быть добавлены (*push*) или удалены (*pop*). Сценарии используют стеки для оценки выражений, хранения временных переменных и управления условиями.


При выполнении скрипта Bitcoin могут использоваться два стека: основной и alt (альтернативный). Основной стек используется для большинства операций сценария. Именно в этом стеке скрипты добавляют, удаляют или манипулируют данными. Альтернативный стек, с другой стороны, используется для временного хранения данных во время выполнения скрипта. Специальные опкоды, такие как `OP_TOALTSTACK` и `OP_FROMALTSTACK`, позволяют переносить Elements из основного стека в альтернативный и наоборот.


Например, при проверке транзакции подписи и открытые ключи помещаются в основной стек и обрабатываются последовательными операционными кодами для проверки соответствия подписей ключам и данным транзакции.
