---
term: 블록 헤더
definition: 블록의 메타데이터(버전, 이전 해시, Merkle 루트, 타임스탬프, 타겟, nonce)를 포함하는 80바이트 구조.
---

블록 헤더는 Bitcoin 블록의 핵심 구성 요소 역할을 하는 데이터 구조입니다. 각 블록은 헤더와 트랜잭션 목록으로 구성됩니다. 블록 헤더에는 Blockchain 내에서 블록의 무결성과 유효성을 보장하는 중요한 정보가 포함되어 있습니다.

블록 헤더의 크기는 80바이트이며 다음 필드를 포함합니다:

*블록 버전;*


- 이전 블록의 Hash입니다;
- 트랜잭션의 Merkle Tree 루트입니다;
- 블록 Timestamp;
- 난이도 목표;

*Nonce.*


예시: 블록 785,530(2023년 4월 15일 미국 파운드리에서 채굴)의 헤더를 16진수 형식으로 표시합니다:


```text
00e0ff3f5ffe3b0d9247dc437e18edc19252e4517cee941752d501000000000000000000206b
de3a10826e2acb2f28fba70463601c789293d0c9c4348d7a0d06711e97c0bcb13a64b2e00517
43f09a40
```


헤더를 분석합니다:

**버전:**


```text
00e0ff3f
```


**이전 Hash:**


```text
5ffe3b0d9247dc437e18edc19252e4517cee941752d501000000000000000000
```


**Merkle Root:**


```text
206bde3a10826e2acb2f28fba70463601c789293d0c9c4348d7a0d06711e97c0
```


**Timestamp:**


```text
bcb13a64
```


**대상:**


```text
b2e00517
```


**Nonce:**


```text
43f09a40
```


블록이 유효하려면 헤더가 `SHA256d`를 사용하여 이중 해시되었을 때 난이도 목표보다 낮거나 같은 Hash을 생성해야 합니다.
