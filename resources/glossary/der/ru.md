---
term: DER
---

Аббревиатура от *Различительные правила кодирования*. Это строгое подмножество правил кодирования ASN.1, определенных в спецификации [ITU-T X.690, 2002.](https://www.itu.int/ITU-T/studygroups/com17/languages/X.690-0207.pdf) и используемых для кодирования любого типа данных в двоичную последовательность. DER в основном используется в специфических областях, таких как криптография, где данные должны быть закодированы стандартным, предсказуемым способом.


На Bitcoin подписи ECDSA кодируются в формате DER. Они состоят из двух 32-байтовых кодированных чисел (`r`,`s`). Формат подписи состоит из следующего Elements (71 байт):


```text
0x30 | length |  0x02 | r-length | r | 0x02 | s-length | s
```


С :




- 0x30` (1 байт): идентификатор составной структуры;
- length` (1 байт): длина следующих данных;
- 0x02` (1 байт): тип идентификатора данных `INTEGER` (целое число) ;
- `r-length` (1 байт): длина следующего значения `r` (32 байта) ;
- r` (32 байта): значение `r` в виде целого числа с большой последовательностью;
- 0x02` (1 байт): тип идентификатора данных `INTEGER` (целое число) ;
- `s-length` (1 байт): длина следующего значения `s` (32 байта) ;
- `s` (32 байта): значение `s` в виде целого числа с большим порядком следования.


В транзакции Bitcoin в конце DER-подписи добавляется байт, указывающий на тип используемого SigHash.
