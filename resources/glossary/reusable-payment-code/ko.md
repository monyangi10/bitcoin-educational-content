---
term: 재사용 가능한 결제 코드
definition: 주소 재사용 없이 고유한 주소를 파생할 수 있는 BIP47의 정적 식별자.
---

BIP47에서 재사용 가능한 결제 코드는 알림 트랜잭션과 고유 주소 도출을 허용하는 Bitcoin Wallet에서 생성된 정적 식별자입니다. 이렇게 하면 결제할 때마다 사용하지 않는 새 주소를 수동으로 생성하여 전송할 필요 없이 개인정보 유출로 이어질 수 있는 주소 재사용을 방지할 수 있습니다. BIP47에서 재사용 가능한 결제 코드는 다음과 같이 구성됩니다:


- 바이트 0은 버전에 해당합니다;
- 바이트 1은 특정 용도의 경우 정보를 추가하기 위한 비트 필드입니다;
- 바이트 2는 공개 키의 'y'의 패리티를 나타냅니다;
- 바이트 3부터 바이트 34까지 공개 키의 'x' 값을 찾을 수 있습니다;
- 바이트 35부터 바이트 66까지는 공개 키와 연결된 chain code가 있습니다;
- 바이트 67부터 바이트 79까지는 패딩이 0입니다.


일반적으로 결제 코드의 시작 부분에 사람이 읽을 수 있는 부분(HRP)을 추가하고 끝 부분에 체크섬을 추가한 다음 Base58로 인코딩합니다. 따라서 결제 코드의 구성은 확장 키의 구성과 매우 유사합니다. 다음은 base58로 된 제 BIP47 결제 코드입니다:


```text
PM8TJSBiQmNQDwTogMAbyqJe2PE2kQXjtgh88MRTxsrnHC8zpEtJ8j7Aj628oUFk8X6P5rJ7P5qD
udE4Hwq9JXSRzGcZJbdJAjM9oVQ1UKU5j2nr7VR5
```


BIP47의 PayNym 구현에서 결제 코드는 로봇의 이미지와 관련된 식별자 형태로도 표현할 수 있습니다. 예를 들어 여기 제 것이 있습니다:


```text
+throbbingpond8B1
```


페이님 구현을 통한 결제 코드 사용은 현재 PC에서는 Sparrow wallet, 모바일에서는 사무라이 Wallet에서 사용할 수 있습니다.
