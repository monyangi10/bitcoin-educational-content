---
term: 壓縮公開密鑰
---

公開金鑰用於腳本 (直接以公開金鑰的形式或以 Address 的形式) 來接收和保護比特幣。一個原始的公開密碼匙是由橢圓曲線上的一個點來表示，由兩個坐標 (`x，y`) 組成，每個坐標有 256 位元。因此，原始格式的公開金鑰有 512 位元，這還不包括用來辨識格式的額外位元組。另一方面，壓縮公開密碼匙是更緊湊的公開密碼匙表示形式。它只使用 `x` 座標和一個前綴 (`02` 或 `03`) 來表示 `y` 座標的奇偶性 (偶數或奇數)。


如果我們將此簡化為實數領域，給定橢圓曲線對 x 軸對稱，對於曲線上的任何一點 $P$ (`x, y`)，存在一個點 $P'$ (`x, -y`) 也會在這同樣的曲線上。這表示對於每一個 `x`, `y` 只有兩種可能的值, 正值和負值.舉例來說，對於給定的斷續點 `x`, 在橢圓曲線上會有兩個點 $P1$ 和 $P2$,共享相同的斷續點但坐標相反：


![](../../dictionnaire/assets/29.webp)

要在曲線上的兩個潛在點之間做選擇，可以在 `x` 中加入一個前綴，指定要選擇的 `y`。這種方法可以將公開密碼匙的大小從 520 位元減至 264 位元 (8 位元的前綴 + 256 位元的 `x`)。這種表示法稱為公開密碼匙的壓縮形式。


然而，在橢圓曲線密碼學的背景下，我們使用的不是實數，而是秩為 `p` (素數) 的有限域。在這種情況下，`y` 的 「符號 」是由其奇偶性決定的，也就是說，`y` 是偶數還是奇數。前綴 `0x02` 表示偶數 `y`，而 `0x03` 表示奇數 `y`。


考慮以下以十六進位表示的原始公開金鑰 (橢圓曲線上的一個點) 的範例：

```plaintext
K = 04678afdb0fe5548271967f1a67130b7105cd6a828e03909a67962e0ea1f61deb649f
6bc3f4cef38c4f35504e51ec112de5c384df7ba0b8d578a4c702b6bf11d5f
```


我們可以隔離前綴、`x` 和`y`：

```plaintext
Prefix = 04
To determine the parity of `y`, we examine the last hexadecimal character of `y`:
```

基數 16（十六進位）：f

基數 10（十進制）：15

y 是奇數

```

The prefix for the compressed public key will be `03`. The compressed public key in hexadecimal becomes:
```

K = 03678afdb0fe5548271967f1a67130b7105cd6a828e03909a67962e0ea1f61deb6

```
