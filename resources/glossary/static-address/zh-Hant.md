---
term: 靜態 Address
---

在無聲付款的情況下，指的是一個獨特的識別碼，它允許在無 Address 重複使用、無互動的情況下接收付款，並且在各種付款和靜態 Address 之間沒有可見的 On-Chain 連結。此技術不需要為每筆交易 generate 新的、未使用的收款地址，因此避免了 Bitcoin 中收款人必須提供新的 Address 給付款人的一般互動。這在某種程度上等同於 BIP47 背景下的可重複使用的付款代碼。


此 Address 由兩個公開金鑰組成：$B_{\text{scan}}$ 用於掃描，$B_{\text{spend}}$ 用於花費，連接起來形成靜態 Address $B = B_{\text{scan}}\文本B_{text{spend}}$。收件人會公佈此 Address，讓寄件者無需與收件人進行任何進一步的互動，即可得出唯一的付款地址。為了管理多個不同的付款來源，可以在 $B_{\text{spend}}$ 中加入標籤，從而從 $B_1$、$B_2$ 等創建多個有標籤的靜態地址。這樣就可以在使用單一基 Address 的同時分隔付款，從而減少 Blockchain 掃描的工作量。然而，由於常用 $B_{text{scan}$，實體的所有靜態地址都可以很容易地聯繫起來。
