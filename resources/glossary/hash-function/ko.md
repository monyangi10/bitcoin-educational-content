---
term: 해시 함수
definition: 가변적인 크기의 입력을 받아 고정된 크기의 출력을 생성하는 수학 함수.
---

가변 크기 입력(메시지라고 함)을 받아 고정 크기 출력(Hash, 해싱, 다이제스트 또는 핑거프린트라고 함)을 생성하는 수학 함수입니다. Hash 함수는 암호화에서 널리 사용되는 프리미티브입니다. 이들은 보안 컨텍스트에서 사용하기에 적합한 특정 속성을 나타냅니다:

**사전 이미지 저항**: 특정 Hash를 생성하는 메시지를 찾는 것, 즉 $h = H(m)$, 여기서 $H$는 Hash 함수인 Hash $h$에 대한 사전 이미지 $m$을 찾는 것은 매우 어려운 일이어야 합니다;

**두 번째 사전 이미지 저항:** 메시지 $m_1$이 주어지면 $H(m_1) = H(m_2)$가 되도록 ($m_1$과 다른) 다른 메시지 $m_2$를 찾기가 매우 어려워야 합니다;

**충돌 저항**: 두 개의 서로 다른 메시지 $m_1$과 $m_2$를 $H(m_1) = H(m_2)$로 찾는 것은 매우 어렵습니다;

**변조 저항성: ** 입력의 작은 변화로 인해 출력에 예측할 수 없는 중대한 변화가 발생해야 합니다.


Bitcoin의 맥락에서 Hash 함수는 Proof-of-Work 메커니즘(*Proof-of-Work*), 트랜잭션 식별자, Address 생성, 체크섬 계산, 머클 트리와 같은 데이터 구조 생성 등 여러 용도로 사용됩니다. 프로토콜 측면에서 Bitcoin는 `HASH256`이라고도 불리는 `SHA256d` 함수를 독점적으로 사용하며, 이는 이중 `SHA256` Hash로 구성됩니다. hASH256`은 특정 체크섬 계산, 특히 확장 키(`xpub`, `xprv`...)에도 사용됩니다. Wallet 쪽에서는 다음도 사용됩니다:


- Mnemonic 구문의 체크섬을 위한 간단한 `SHA256`;
- 결정론적 지갑과 계층적 지갑을 도출하는 과정에서 사용되는 `HMAC` 및 `PBKDF2` 알고리즘 내의 `SHA512`입니다;
- hASH160`은 `SHA256`과 `RIPEMD160`의 연속적인 사용을 설명합니다. hASH160`은 수신 주소를 생성하는 과정(P2PK 및 P2TR 제외)과 확장 키의 부모 키 지문을 계산하는 데 사용됩니다.
