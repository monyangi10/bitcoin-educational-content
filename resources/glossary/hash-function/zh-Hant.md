---
term: Hash 功能
---

一種數學函數，可接受可變大小的輸入（稱為訊息），並產生固定大小的輸出（稱為 Hash、散列、摘要或指紋）。Hash 函數是密碼學中廣泛使用的基元。Hash 函數具有特定的特性，因此適用於安全的情況：


- 前像抵抗：要找到產生特定 Hash 的訊息一定非常困難，也就是為 Hash $h$ 找到前像 $m$，使得 $h = H(m)$，其中 $H$ 是 Hash 函數；
- 第二前像抵抗：給予一個訊息 $m_1$，要找到另一個訊息 $m_2$(不同於 $m_1$)，使 $H(m_1) = H(m_2)$ 一定非常困難；
- 抗碰撞性：要找到兩個不同的訊息 $m_1$ 和 $m_2$，使 $H(m_1) = H(m_2)$ 一定非常困難；
- 抗篡改性：輸入的微小變化必須導致輸出發生顯著且不可預測的變化。


在 Bitcoin 的範圍內，Hash 函式用於多種用途，包括 Proof-of-Work 機制 (*Proof-of-Work*)、交易識別碼、Address 產生、校驗和計算，以及建立 Merkle 樹等資料結構。在通訊協定方面，Bitcoin 獨家使用 `SHA256d` 函式，也稱為 `HASH256`，由雙重 `SHA256` Hash 組成。`HASH256` 也用於計算某些校驗和，特別是擴展金鑰 (`xpub`、`xprv`...)。在 Wallet 方面，也使用了下列內容：


- 簡單的 `SHA256` 用於 Mnemonic 詞組的校驗和；
- SHA512` 內的 `HMAC` 和 `PBKDF2` 演算法，用於推導確定性和分層式錢包的過程中；
- `HASH160`，它描述了 `SHA256` 和 `RIPEMD160` 的連續使用。`HASH160` 用於產生接收位址 (P2PK 和 P2TR 除外) 的過程，以及計算擴充金鑰的父金鑰指紋。


> ► *在英文中，它被稱為「Hash 功能」。
