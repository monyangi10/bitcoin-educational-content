---
term: BIP0042
---

一項改進 Bitcoin Core 的提案，它解決了 Block reward 減少時間表中的一個小錯誤。這個異常情況如果不加以修正，會導致比特幣的總創造量超過計劃中的 2,100 萬個上限。具體來說，在減半結束後，理論上可能會在 2214 年左右觸發新一輪的 Bitcoin 生成。有問題的 Core 程式碼在 Miner 的獎勵值上使用了右二進位移位操作。這個錯誤是因為在 C++ 語言標準未定義的情況下，使用了這個移位操作。將一個 64 位元的整數 (`int64_t`)向右移動超過 63 位元，就屬於這類未定義的行為。在 Bitcoin Core 程式碼中，這種情況可能發生在 64 個 halvings 之後，也就是在區塊高度 13,440,000 處。在此限制之外，位元移位的行為並未定義，這表示不同的編譯方式可能會對程式碼有不同的詮釋。這可能會導致不可預測的結果，包括可能產生無限的 Bitcoin 膨脹。BIP42 糾正了這個問題，強制 Block reward 在 64 次半衰期後設定為零，從而避免在未定義行為的情況下使用右移操作。這項修改是 Soft Fork，因此釐清了 Bitcoin 核心程式碼的行為。儘管這個 BIP42 修正的錯誤相當嚴重，但並非即時危機，因為它要到 2214 年左右才會顯現。BIP42 於 2014 年 4 月 1 日由 Pieter Wuille 發表，因此以幽默的語調而與眾不同。
