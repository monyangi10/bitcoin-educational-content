---
term: BIP0042
---

Een voorstel om Bitcoin core te verbeteren dat een kleine fout in het Block reward reductieschema aanpakt. Als deze anomalie niet gecorrigeerd was, zou de totale creatie van bitcoins de geplande limiet van 21 miljoen overschrijden. Specifiek, na het einde van de halvings, zou een nieuwe cyclus van Bitcoin creatie theoretisch hebben kunnen worden geactiveerd rond het jaar 2214. De Core-code in kwestie gebruikte een binaire shift naar rechts op de waarde van de Miner-beloning. De bug kwam voort uit het gebruik van deze verschuiving in een context waarin het gedrag niet gedefinieerd was volgens de C++ taalstandaarden. Het verschuiven van een 64-bit geheel getal (`int64_t`) meer dan 63 bits naar rechts valt in deze categorie van ongedefinieerd gedrag. In de Bitcoin core code kon dit gebeuren na 64 halvings, op blokhoogte nr. 13.440.000. Voorbij deze grens was het gedrag van de bitverschuiving niet gedefinieerd, wat betekent dat verschillende compilaties de code verschillend konden interpreteren. Dit had tot onvoorspelbare uitkomsten kunnen leiden, waaronder de mogelijkheid om oneindige Bitcoin inflatie te creëren. BIP42 corrigeerde dit probleem door voor te schrijven dat de Block reward op nul gezet moet worden na 64 halvingen, om zo het gebruik van een right shift operatie in een context van ongedefinieerd gedrag te vermijden. Deze wijziging, die een Soft Fork was, verduidelijkte dus het gedrag van de Bitcoin core code. Hoewel vrij ernstig, was deze door BIP42 gecorrigeerde bug niet direct kritiek, omdat hij zich pas rond het jaar 2214 zou hebben gemanifesteerd. Gepubliceerd op 1 april 2014 door Pieter Wuille, onderscheidt BIP42 zich door zijn humoristische toon.
