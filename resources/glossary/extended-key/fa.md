---
term: کلید گسترده
definition: رشته‌ای شامل یک کلید، chain code آن و متادیتا برای اشتقاق در کیف‌پول‌های HD.
---

دنباله‌ای از کاراکترها که یک کلید (عمومی یا خصوصی)، chain code مرتبط با آن، و مجموعه‌ای از فراداده‌ها را ترکیب می‌کند. یک کلید توسعه‌یافته تمام Elements لازم برای استخراج کلیدهای فرزند را به یک شناسه واحد تبدیل می‌کند. آنها در کیف‌پول‌های قطعی و سلسله‌مراتبی استفاده می‌شوند و می‌توانند از دو نوع باشند: یک کلید عمومی توسعه‌یافته (برای استخراج کلیدهای عمومی فرزند استفاده می‌شود) یا یک کلید خصوصی توسعه‌یافته (برای استخراج هر دو کلید خصوصی و عمومی فرزند استفاده می‌شود). بنابراین، یک کلید توسعه‌یافته شامل چندین قطعه داده مختلف است که در BIP32 توصیف شده‌اند، به ترتیب:


- پیشوند: `prv` و `pub` بخش قابل خواندن توسط انسان (HRP) هستند که نشان می‌دهند آیا کلید خصوصی توسعه‌یافته (`prv`) یا کلید عمومی توسعه‌یافته (`pub`) است. حرف اول پیشوند نسخه کلید توسعه‌یافته را مشخص می‌کند: `x` برای Legacy یا SegWit V1 بر روی Bitcoin، `t` برای Legacy یا SegWit V1 بر روی Bitcoin Testnet، `y` برای Nested SegWit بر روی Bitcoin، `u` برای Nested SegWit بر روی Bitcoin Testnet، `z` برای SegWit V0 بر روی Bitcoin، `v` برای SegWit V0 بر روی Bitcoin Testnet.
- عمق، که نشان‌دهنده تعداد مشتقات از کلید اصلی برای رسیدن به کلید توسعه‌یافته است؛
- اثر انگشت والدین. این نشان‌دهنده اولین 4 بایت از `HASH160` کلید عمومی والدین است؛
- شاخص. این شماره جفت در میان همتایانش است که کلید توسعه‌یافته از آن مشتق می‌شود؛
- chain code؛
- یک بایت پدینگ اگر یک کلید خصوصی باشد `0x00`؛
- کلید خصوصی یا عمومی؛
- یک مجموعۀ کنترلی. این مقدار نشان‌دهندۀ اولین ۴ بایت از `HASH256` باقی‌ماندۀ کلید توسعه‌یافته است.


در عمل، کلید عمومی توسعه‌یافته برای آدرس‌های دریافت generate و مشاهده تراکنش‌های یک حساب بدون افشای کلیدهای خصوصی مرتبط استفاده می‌شود. این می‌تواند به عنوان مثال، امکان ایجاد یک Wallet "فقط مشاهده" را فراهم کند. با این حال، مهم است که توجه داشته باشید که کلید عمومی توسعه‌یافته اطلاعات حساسی برای حریم خصوصی کاربر است، زیرا افشای آن می‌تواند به اشخاص ثالث اجازه دهد تا تراکنش‌ها را ردیابی کرده و موجودی حساب مرتبط را ببینند.
