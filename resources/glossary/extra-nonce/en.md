---
term: EXTRA-Nonce
---

Field used in the `scriptSig` of a block's coinbase transaction, which allows a greater number of possibilities to be tested in order to have a hash lower than the difficulty target, in addition to the classic Nonce, which is found directly in the header of each block.

Modifying the extra-Nonce in the coinbase transaction changes the identifier of this transaction, and therefore the Merkle root of all the transactions in the block, which also modifies the block header. This allows the miner to continue searching for hashes when the range of the classic Nonce is already exhausted, without changing the structure of its candidate block.

In mining pools, the extra-Nonce is often divided into two parts: one generated by the pool to identify each chopper, and another modified by the chopper in search of a valid share. This allows the different choppers in the pool to work simultaneously on the same candidate block with the entire nonces range, without duplicating the same work at pool level.
