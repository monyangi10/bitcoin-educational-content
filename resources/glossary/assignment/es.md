---
term: Assignment
---

En la lógica del protocolo RGB, una Assignment es el equivalente de una salida de transacción que modifica, actualiza o crea determinadas propiedades dentro del estado de una Contract. Una Assignment comprende dos Elements:




- Una Seal Definition (referencia a una UTXO específica);
- A Owned State (datos que describen el estado asociado a este nuevo titular).


Por lo tanto, una Assignment indica que una parte del estado (por ejemplo, un activo) se asigna ahora a un titular concreto, identificado mediante una Single-Use Seal vinculada a una UTXO.
