---
term: ASSIGNMENT
---

In the context of the RGB protocol, an Assignment is the equivalent of a transaction output that modifies, updates or creates specific properties within the contract's state. An Assignment consists of two elements:


- A Seal Definition (a reference to a specific UTXO);
- An Owned State (a data describing the state associated with this new holder).

An Assignment therefore indicates that a portion of the state (such as an asset) is now allocated to a particular holder, identified through a Single-Use Seal tied to a UTXO.
