---
term: Assignment
---

In the logic of the RGB protocol, a Assignment is the equivalent of a transaction output that modifies, updates or creates certain properties within the state of a Contract. A Assignment comprises two elements:


- A Seal Definition (reference to a specific UTXO);
- A Owned State (data describing the state associated with this new holder).

A Assignment therefore indicates that a portion of the state (for example, an asset) is now allocated to a particular holder, identified via a Single-Use Seal linked to a UTXO.
