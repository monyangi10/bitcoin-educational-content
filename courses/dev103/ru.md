## Настройка

<chapterId>ba05a290-1782-5268-87c9-62fd09590e05</chapterId>


В этом разделе мы настроим нашу машину для написания и выполнения нашей первой программы на JavaScript.


Программа JavaScript - это просто набор (один или несколько) текстовых файлов, содержащих команды, которые должны быть выполнены средой выполнения JavaScript.


Имена этих текстовых файлов обычно заканчиваются расширением `.js`, например `my_script.js`, `my_program.js` и т.д.


Содержащиеся в них команды написаны на языке программирования JavaScript.


Среда выполнения JavaScript - это специальная программа, которая выполняет эти файлы.


![](assets/en/1.webp)


### Установка NodeJS


Наиболее распространенной средой выполнения JavaScript является NodeJS.


Вы можете установить его, следуя [официальным инструкциям](https://nodejs.org/en/download).


На странице загрузки вы найдете инструкции для всех трех основных ОС (операционных систем): Windows, Linux и MacOS. Предполагается, что вы знаете, как открыть терминал в вашей ОС.


Поскольку NodeJS доступен для всех трех ОС, написанные вами программы смогут выполняться на всех из них (за исключением некоторых крайних случаев).


Это значит, что вы можете, например, написать простую видеоигру на JavaScript на своем компьютере с Windows и передать ее своему другу, чтобы он запустил ее на своем Mac.


![](assets/en/2.webp)


### Редактирование текста


Одна из самых приятных вещей в программировании - это то, что вы можете писать код, используя любой текстовый редактор, даже блокнот по умолчанию в вашей ОС.


Есть текстовые редакторы, которые специализируются на написании кода, но некоторые из них доступны бесплатно, другие требуют оплаты лицензии.


Выбор редактора кода - это огромная кроличья нора, которая выходит за рамки данного курса, поэтому мы не будем говорить об этом здесь. Если вы не знаете, что использовать, то самым распространенным бесплатным редактором является [VSCode](https://code.visualstudio.com/).


Его Interface немного раздут, но в нем есть все, что вам нужно: редактор файлов, проводник файлов (для визуализации файлов и подкаталогов в каталоге, над которым вы работаете) и терминал для запуска кода. Он также поддерживает множество плагинов и по умолчанию поставляется с подсветкой синтаксиса JavaScript.


Если вы хотите быть немного более Cypherpunk-ым, вы можете использовать [VSCodium](https://vscodium.com/) вместо этого.


### Первая программа (hello world)


Традиционно при изучении языка программирования первая программа, которую человек пишет, заключается в печати "hello world!" на консоли.


Создайте каталог `my_js_code/`, внутри которого будет находиться файл `main.js` (эти имена произвольны).


Откройте каталог с помощью VSCode.


Запишите этот код в свой файл:


```javascript
console.log("hello world!")
```


Откройте терминал и выполните эту команду, чтобы запустить программу:


```
node main.js
```


Результат должен быть


```
hello world!
```


### Что случилось


В JavaScript все является "объектом".


`console` - это объект, который используется для отладки программы.


`console.log` - это самый используемый метод `console`. Он просто печатает все аргументы, которые вы ему передали.


Вы передаете аргументы в `console.log` с помощью круглых скобок `()`.


Например, если вы хотите вывести число `1000`, то просто напишите


```javascript
console.log(1000)
```


Затем выполните его, запустив


```
node main.js
```


в терминале (с этого момента в курсе предполагается, что вы знаете, что именно так выполняется программа).


Должно получиться


```
1000
```


Вы можете передать несколько вещей, например


```javascript
console.log(16, 8, 1993)
```


Это позволит напечатать


```
16 8 1993
```


## Переменные и комментарии

<chapterId>23050ab7-343b-5edf-9d37-e4e782e27ce0</chapterId>


Программы обычно выполняют операции над данными.


Переменные - это как именованные ящики, которые мы используем для хранения данных. Они позволяют связать часть данных с определенным именем, чтобы впоследствии получить их по этому имени.


### объявления `let`


Чтобы объявить переменную в JavaScript, мы можем использовать ключевое слово `let`.


После написания `let` мы пишем имя, которое хотим дать переменной, затем знак `=`, а затем значение, которое хотим сохранить.


Например:


```javascript
let age = 25

console.log(age)
```


Имя переменной (технически называемое "идентификатором") обычно может содержать буквы, символы подчеркивания (`_`), знак доллара (`$`) и цифры, хотя первый символ не может быть цифрой.


В приведенном выше коде мы объявили переменную `age` и сохранили в ней значение `25`.


Затем мы вывели значение с помощью `console.log(age)`.


Если запустить этот код с помощью `node main.js`, то на выходе получится:


```
25
```


Идентификаторы чувствительны к регистру, то есть нижний и верхний регистр считаются различиями в идентификаторах, например


```javascript
let age = 25

let Age = 20

console.log(age)
```


выведет 25, потому что это две совершенно разные переменные!


Вы также можете хранить строки (текст) в переменной:


```javascript
let message = "hello again"

console.log(message)
```


Напечатается:


```
hello again
```


Как и раньше, мы использовали `console.log()` для вывода значения, хранящегося в переменной.


Теперь давайте сделаем и то, и другое вместе:


```javascript
let age = 25

let message = "hello again"

console.log(age)

console.log(message)
```


Запустив его, вы получите печать:


```
25
hello again
```

### Перераспределение


Переменные, объявленные с помощью `let`, могут быть изменены после их создания.


Это называется перераспределением.


```javascript
let score = 10

console.log(score)

score = 15

console.log(score)
```


Сначала мы присваиваем `10` к `score`, а затем выводим его на печать.


Затем мы меняем значение `score` на `15` и выводим его снова.


Выходные данные будут такими:


```
10
15
```


Это очень полезно, когда значение меняется со временем, например, в игре, где счет увеличивается.


Добавим еще одну переменную:


```javascript
let score = 10
let player = "Alice"

console.log(score)
console.log(player)

score = 20
player = "Bob"

console.log(score)
console.log(player)
```


Напечатается:


```
10
Alice
20
Bob
```


Как вы можете видеть, и `score`, и `player` были изменены.


### объявления `const`


Однако чаще всего мы не хотим, чтобы переменная изменялась после создания. Для этого мы используем `const`.


`const` - это сокращение от слова "константа". Присвоив переменной `const` значение, вы не сможете его изменить.


```javascript
const pi = 3.14
console.log(pi)
```


Это отпечатки:


```
3.14
```


Но если вы попытаетесь это сделать:


```javascript
const pi = 3.14
console.log(pi)

pi = 99 // this line will cause an error
console.log(pi)
```


JavaScript выдаст ошибку типа:


```
TypeError: Assignment to constant variable.
```


Это происходит потому, что `pi` была объявлена с помощью `const`, и после этого вы не можете изменить ее значение. Вы сообщаете интерпретатору JavaScript, что не хотите, чтобы эта переменная изменялась.


Это полезно, потому что снижает вероятность изменения по ошибке. Когда программы становятся очень большими, с тысячами строк кода, невозможно уследить за всем, что происходит одновременно (это основная причина, по которой мы используем компьютеры, для выполнения сложных процессов, которые мы не можем вычислить своим мозгом), поэтому полезно иметь такие ограничения, как это, которые делают программу более детерминированной.


Считается, что лучше всего всегда объявлять значения как `const`, если только мы не уверены, что захотим изменить их позже.


### Комментарии в JavaScript


Иногда мы хотим написать в коде заметки, которые не выполняются. Такие заметки называются комментариями.


Комментарии игнорируются программой во время ее работы, но они полезны для объяснения самим себе или другим людям.


Чтобы написать однострочный комментарий, используйте `//`


```javascript
// This is a comment
const x = 10 // This is also a comment
console.log(x)
```


Печать будет продолжаться:


```
10
```


Комментарии просто предназначены для чтения людьми.


Вы также можете писать многострочные комментарии, используя `/*` и `*/`


```javascript
/*
This is a multi-line comment.
It can span several lines.
*/
const y = 20
console.log(y)
```


Это позволит напечатать


```
20
```


И комментарий будет проигнорирован.


С помощью комментариев вы можете добавлять небольшие примечания к коду, чтобы не забыть, что он делает и почему написан определенным образом. Это также может помочь другим программистам понять его.


## Основные типы: числа, строки, булевы

<chapterId>cfdb04f6-21a8-5143-bbf9-7aaae04962f0</chapterId>


В JavaScript "тип" говорит о том, к какому типу данных относится то или иное значение.


В Javascript есть несколько основных типов, и в этом разделе мы рассмотрим некоторые из них.


### Числа и арифметические действия


Первый тип, который мы представим, это `number`.


Числа в JavaScript могут быть целыми (например, `5`) или десятичными (например, `3.14`).


С ними можно выполнять арифметические действия: сложение, вычитание, умножение и деление.


Вот базовый пример:


```javascript
const a = 10
const b = 5

const sum = a + b
const difference = a - b
const product = a * b
const quotient = a / b

console.log(sum)
console.log(difference)
console.log(product)
console.log(quotient)
```


Напечатается:


```
15
5
50
2
```


Вы также можете использовать круглые скобки `()` для управления порядком выполнения операций:


```javascript
const result = (2 + 3) * 4
console.log(result)
```


Это отпечатки:


```
20
```


Без скобок это будет `2 + 3 * 4`, то есть:


```javascript
const result = 2 + 3 * 4
console.log(result)
```


Это можно напечатать:


```
14
```


Потому что в обычной математике умножение происходит раньше сложения.


### Строки и интерполяция


Второй тип JavaScript, который мы представим, - это `string`.


Строки - это фрагменты текста. Для их создания можно использовать одинарные кавычки `'...'` или двойные кавычки `"...'`.


```javascript
const greeting = "hello"
const name = 'Bob'
console.log(greeting)
console.log(name)
```


Это отпечатки:


```
hello
Bob
```


Для объединения строк можно использовать оператор `+`:


```javascript
const greeting = "hello"
const space = " "
const name = "Bob"

const fullGreeting = greeting + space + name
console.log(fullGreeting)
```


Напечатается:


```
hello Bob
```


Но есть более приятный способ объединения строк, который называется **интерполяция строк**. Вы используете обратные знаки для объявления строки ```...`` и записываете переменные с помощью `${...}` внутри строки:


```javascript
const greeting = "hello"
const name = "Bob"

const fullGreeting = `${greeting} ${name}`
console.log(fullGreeting)
```


Это также печатается:


```
hello Bob
```


Внутри `${...}` можно включить любое выражение:


```javascript
const age = 30
console.log(`Next year, I will be ${age + 1} years old.`)
```


Это отпечатки:


```
Next year, I will be 31 years old.
```


Интерполяция очень часто встречается в современном JavaScript.


### Булевы числа, сравнение и логические операции


Третий тип, который мы представим, - это `boolean`. Он назван в честь математика Джорджа Буля, который изобрел булеву логику.


Булевы числа просты: только два возможных значения, `true` и `false`.


Вы можете хранить их в переменных:


```javascript
const theSkyIsBlue = true
const thisCourseIsBad = false

console.log(theSkyIsBlue)
console.log(thisCourseIsBad)
```


Это отпечатки:


```
true
false
```


Вы можете комбинировать булевы значения с помощью логических операторов:



- `&&` означает "и", и возвращает `true` только в том случае, если **обои** значения `true`, в противном случае возвращает `false`
- `||` означает "или", и оно вернет `true`, если ** хотя бы одно** из значений будет `true`, в противном случае (если они оба ложные) оно вернет `false`
- `!` означает "не", применяется перед булевым числом и переворачивает его: если булевое число `true`, то оно вернет `false`, и наоборот.


![](assets/en/3.webp)


Примеры:


```javascript
const isSunny = true
const isWarm = true

console.log(isSunny && isWarm)  // true
console.log(isSunny || isWarm)  // true
console.log(!isSunny)           // false
```


В JavaScript можно сравнивать значения с помощью таких операторов, как `>`, `<`, `===` и `!==`. Результатом этих сравнений всегда является булево значение.


```javascript
const first = 10
const second = 5

const firstIsGreater   = (a > b)
const secondIsGreater  = (a < b)
const theyAreEqual     = (a === b)
const theyAreDifferent = (a !== b)

console.log(firstIsGreater)   // true
console.log(secondIsGreater)   // false
console.log(theyAreEqual)  // false
console.log(theyAreDifferent)  // true
```


В Javascript также есть `>=` для обозначения "больше или равно" и `<=` для обозначения "меньше или равно".


Булевы, сравнение и логические операторы часто используются в программах для объявления сложных условий, например, чтобы убедиться, что "письмо пришло И оно содержит нужное мне изображение ИЛИ длина письма больше 10000 символов". Позже вы поймете, что они являются необходимыми строительными блоками для построения логики программы.


## Массивы, null, undefined

<chapterId>7bf18183-5eae-53ed-83d2-b04982145d81</chapterId>


В этом разделе мы рассмотрим еще три типа, которые очень часто встречаются в программах на JavaScript:



- Массивы**: последовательности значений
- undefined**: специальное значение, означающее "ничего не было назначено"
- null**: еще одно специальное значение, означающее "намеренно пустой"


### Массивы и доступ к индексам


Массив** - это тип, который может содержать несколько значений в виде списка.


Вы создаете массив, используя квадратные скобки `[]` и разделяя элементы запятыми.


Вот базовый пример:


```javascript
const numbers = [10, 2, 88]
console.log(numbers)
```


Это отпечатки:


```
[ 10, 2, 88 ]
```


В массиве можно хранить все, что угодно, а не только числа:


```javascript
const things = ["apple", 42, true]
console.log(things)
```


Это отпечатки:


```
[ 'apple', 42, true ]
```


Чтобы получить доступ к определенному элементу массива, мы используем **index**. Индекс - это позиция элемента, начиная с **0**.


Так и в этом массиве:


```javascript
const colors = ["red", "green", "blue"]
```



- `colors[0]` - это `"красный"`
- `Цвета[1]` - это `"Green"`
- `colors[2]` - это `"синий"`


Давайте попробуем:


```javascript
const colors = ["red", "green", "blue"]
console.log(colors[0])
console.log(colors[1])
console.log(colors[2])
```


Напечатается:


```
red
green
blue
```


Вы можете присвоить значение определенному индексу массива


```javascript
const colors = ["red", "green", "blue"]

colors[1] = "yellow"

console.log(colors)
```


Напечатается:


```
[ 'red', 'yellow', 'blue' ]
```


В качестве индекса можно использовать любое натуральное число, даже то, которое хранится в переменной


```javascript

const i = 1
const colors = ["red", "green", "blue"]
console.log(colors[i])
```


Напечатается:


```
green
```


Но если вы попытаетесь обратиться к несуществующему индексу, вы получите `undefined`:


```javascript
const colors = ["red", "green", "blue"]
console.log(colors[3])
```


Это отпечатки:


```
undefined
```


Что это?


### `undefined`


Специальное значение `undefined` означает "значение не было присвоено".


Если вы создадите переменную, но не придадите ей значение, она будет `неопределенной`:


```javascript
const name
console.log(name)
```


Это отпечатки:


```
undefined
```


Поскольку мы ничего не присвоили `name`, JavaScript по умолчанию устанавливает для него значение `undefined`.


Как мы уже видели, вы также можете получить `undefined` при обращении к несуществующему индексу массива:


```javascript
const fruits = ["banana", "apple"]
console.log(fruits[2]) // There is no index 2
```


Это отпечатки:


```
undefined
```


### `null` и как с ним обращаться


`null` - это также особое значение. Оно означает "здесь ничего нет, и я сделал это специально"


В отличие от `undefined`, который работает автоматически, `null` вы задаете сами.


Например:


```javascript
const currentUser = null
console.log(currentUser)
```


Это отпечатки:


```
null
```


Зачем использовать `null`? Возможно, вы ожидаете получить значение позже, но оно еще не готово:


```javascript
let winner = null

// Later in the program:
winner = "Alice"

console.log(winner)
```


Это отпечатки:


```
Alice
```


Поэтому `null` полезен, когда вы хотите сказать, например, "Здесь должно быть что-то позже, но сейчас здесь пусто"


## Блоки и поток управления

<chapterId>be985168-2636-5b0d-a48f-ac1bbfbff8a7</chapterId>


До сих пор мы писали в основном строки кода, которые выполнялись одна за другой.


Но когда мы пишем код, мы можем контролировать порядок его выполнения.


Это называется **поток управления**.


Давайте начнем с понимания блоков и области применения.


### Глобальный масштаб


Каждая переменная, которую мы объявляем, существует в **области**, то есть в той части кода, где переменная известна.


Если вы объявляете переменную вне какого-либо блока, она существует в **глобальной области видимости**.


```javascript
const color = "blue"
console.log(color)
```


Эта переменная `color` находится в глобальной области видимости, поэтому к ней можно обратиться из любой точки файла.


Если вы добавите больше строк:


```javascript
const color = "blue"
console.log(color)

const size = "large"
console.log(color)
console.log(size)
```


И `color`, и `size` являются глобальными переменными. Они доступны везде в файле.


Но что происходит внутри блока?


### Блоки и локальная область


Блок** - это фрагмент кода, окруженный фигурными скобками `{}`.


Переменные, объявленные с помощью `let` или `const` внутри блока, существуют **только** внутри этого блока.


```javascript
{
const message = "inside block"
console.log(message)
}
```


Это отпечатки:


```
inside block
```


Но если вы попробуете это:


```javascript
{
const message = "inside block"
}
console.log(message) // Error!
```


JavaScript выдаст ошибку типа:


```
ReferenceError: message is not defined
```


Это потому, что `message` было объявлено внутри блока и не существует вне его.


Это означает, что мы можем использовать блоки для изоляции частей нашего кода и быть уверенными, что "то, что происходит в блоке, остается в блоке" (что-то вроде Лас-Вегаса).


Организация кода в блоки позволяет нам также структурировать выполнение программы, используя такие конструкции потока управления, как `if`


### `if`, `else`


Иногда мы хотим, чтобы код выполнялся **только если** что-то истинно. Для этого и существует оператор `if`.


```javascript
const myAge = 20

console.log("Am I an adult?")

if (myAge >= 18) {
console.log("Yes I am!")
}
```


Это отпечатки:


```
Am I an adult?
Yes I am!
```


Как вы можете видеть, код сравнивает `myAge` и `18`.

В этом случае оператор `>=` возвращает `true`, поэтому блок будет выполнен.

Если условие не `true`, блок не выполняется.


```javascript
const myAge = 17

console.log("Am I an adult?")

if (myAge >= 18) {
console.log("Yes I am!")
}
```


Это отпечатки:


```
Am I an adult?
```


Вы можете добавить блок `else` для обработки противоположного случая:


```javascript
const myAge = 17

console.log("Am I an adult?")

if (myAge >= 18) {
console.log("Yes I am!")
} else {
console.log("No, I am not.")
}
```


Это отпечатки:


```
Am I an adult?
No, I am not.
```


Блоки `if` и `else` остаются блоками, поэтому переменные, объявленные внутри них, не существуют вне их.


Если мы хотим быть уверены, что что-то **не является правдой, что мы можем сделать?


Как уже говорилось ранее, в JavaScript есть оператор "not", который переворачивает булевы значения. Таким образом, мы можем сделать


```javascript
const myAge = 17

const adult = myAge >= 18

console.log("Am I an adult?")

if (!adult) {
console.log("No, I am not.")
}
```

Он по-прежнему печатается:


```
Am I an adult?
No, I am not.
```

Потому что мы использовали оператор `!` для инвертирования переменной `adult`.


`if (!adult) {...}` следует читать как "if not adult..."


Используя блоки, логику и операторы сравнения, мы можем структурировать выполнение программы, определяя переменные, которые должны быть `истинными` (или `неистинными`), чтобы что-то произошло.


### `время`, `перерыв`, `продолжение`


Цикл `while` повторяет код *пока* условие истинно.


```javascript
let count = 0

while (count < 3) {
console.log("Count is", count)
count = count + 1
}
console.log("the loop is over!")
```


Это отпечатки:


```
Count is 0
Count is 1
Count is 2
the loop is over!
```


Когда `count` становится равным 3, цикл останавливается.


Вы можете остановить цикл раньше, используя `break`:


```javascript
let number = 1 // Start with number 1

while (true) { // This condition is always true, so this loop will run forever unless we stop it
console.log(number) // Print the current number
if (number === 3) { // If the number is 3, stop the loop
break
}
number = number + 1 // Add 1 to the number
}
```


Это отпечатки:


```
0
1
2
```


Потому что когда число становится `3`, выполняется блок `if`, который останавливает цикл.


Вы можете пропустить оставшуюся часть цикла, используя `continue`:


```javascript
let number = 0 // Start with number 0

while (number < 5) { // Keep going while number is less than 5
number = number + 1 // Add 1 to the number

if (number === 3) { // If the number is 3
continue // Skip the rest of the block and go to the next iteration of the loop
}

console.log(number) // Print the number
}
```


Это отпечатки:


```
1
2
4
5
```


Потому что когда число было `3`, `continue` заставляло программу пропустить строку, в которой печаталось число.


### `для ... из ...`


Если у вас есть массив, и вы хотите сделать что-то с каждым элементом в нем, вы можете использовать `for ... of ... {...}`.


```javascript
const fruits = ["apple", "banana", "cherry"]

for (const fruit of fruits) {
console.log(fruit)
}
```


Это отпечатки:


```
apple
banana
cherry
```

Блок будет выполняться один раз для каждого элемента массива.


`fruit` - это новая переменная, которая принимает значение каждого элемента массива, чтобы оперировать им внутри блока.


### `для ... в ...`


Вы можете использовать `for ... in` для перебора ключей (индексов) массива:


```javascript
const fruits = ["apple", "banana", "cherry"]

for (const index in fruits) {
console.log(index)
}
```


Это отпечатки:


```
0
1
2
```


Вы также можете использовать индекс для получения значения:


```javascript
const fruits = ["apple", "banana", "cherry"]

for (const index in fruits) {
console.log(fruits[index])
}
```


Это выводит то же самое, что и `for ... of`:


```
apple
banana
cherry
```


На практике для массивов лучше использовать `for ... of`, так как это проще и чище.


### Ограниченные петли


Иногда мы хотим зациклиться определенное количество раз, или вообще написать кусок кода, который повторяет блок, отслеживая что-то.

Для этого и нужен ограниченный цикл `for`.

Ограниченный цикл обычно состоит из трех условий, разделенных точкой с запятой `;`, как в `(... ; ... ; ....)`.


```javascript
for (let i = 0; i < 3; i = i + 1) {
console.log(i)
}
```


Это отпечатки:


```
0
1
2
```


Давайте объясним это:



- `let i = 0`: объявляет переменную, которая будет использоваться в блоке (в данном случае это счетчик, начинающийся с 0)
- `i < 3`: объявляет условие, которое должно быть `истинным` для выполнения блока (в данном случае это "repeat while `i` is less than 3")
- `i = i + 1`: объявите некоторый код, который будет выполняться после каждого выполнения блока (в данном случае "увеличить `i` на 1")


Как видите, ограниченный цикл позволяет объявить более сложные условия для повторного выполнения фрагмента кода, но чаще всего в этом нет необходимости.


### Этикетки для блоков


Если вам нужно написать более сложный поток управления, JavaScript позволяет назвать блок с помощью **ярлыка**, который может быть использован `break` или `continue` для указания *куда* нужно вернуться.


Пример:


```javascript
outer: {
console.log("We're inside the outer scope.")

inner: {
console.log("We're inside the inner scope.")
break outer
}

console.log("This will not run")
}

console.log("Done")
```


Это отпечатки:


```
Inside outer block
Inside inner block
Done
```


Мы использовали `break outer`, чтобы полностью выйти из блока `outer`.


Вы также можете помечать циклы. Рассмотрим этот пример:


```javascript
// Declare a variable to count the total number of days in a year
let totalDaysInOneYear = 0

// Declare one variable per month, with the number of the month
const january = 1
const february = 2
const march = 3
const april = 4
const may = 5
const june = 6
const july = 7
const august = 8
const september = 9
const october = 10
const november = 11
const december = 12

// Declare an array that holds the months that have 30 days
const monthsWith30Days = [
april, june, september, november
]

// Declare variables to keep track of the month and day we're in
let currentMonth = january
let currentDay = 1

monthsLoop: while (true) {  // Start a loop labeled "monthsLoop" to process each month

daysLoop: while (true) {  // Start a loop labeled "daysLoop" to process each day in the month
totalDaysInOneYear = totalDaysInOneYear + 1  // Increase the total number of days we counted by 1

if (                                                                   // We want to check if we're at the end of the month.
currentDay === 31                                                  // Check if the current day is 31 (for months with 31 days)...
|| currentDay === 30 && (monthsWith30Days.includes(currentMonth))  // ...or 30 if it's among the 30-days months...
|| currentDay === 28 && (currentMonth === february)                // ...or 28 if it's February. If it's any of these three, then:

){
currentMonth = currentMonth + 1  // Move to the next month
currentDay = 1                   // Reset the day to 1 for the new month
break daysLoop                   // Exit the inner loop (which tracks days) and go back to the outer loop (which tracks months)
}
else { currentDay = currentDay + 1 }                                   // Otherwise, we're not at the end of the month, and we just move to the next day

}
if (currentMonth > 12) {  // After processing a month, check if we've gone past December
break monthsLoop  // If so, break the outer loop and stop the day-counting process
}
}

console.log(totalDaysInOneYear)  // Print the total number of days in the year (should be 365)
```

Это был очень скучный пример, но, надеюсь, он прояснил (иногда) необходимость использования ярлыков.


## Внедрение функций

<chapterId>cc324715-09c2-5cf7-9e6f-47a6f16bc04d</chapterId>


По мере роста ваших программ вам часто захочется **использовать** части кода.


Именно для этого и нужны **функции**: они позволяют вам сгруппировать некоторый код, дать ему имя и запускать его, когда захочется.


### Объявление функции


Чтобы объявить функцию, мы можем использовать ключевое слово `function`. Затем мы даем ей имя, пару круглых скобок `()` с аргументами, которые она принимает, и блок кода `{}` для выполнения. Начнем с функции, которая не принимает аргументов:


```javascript
function sayHello () {console.log(`Hello!`) }
```


Этот код **объявляет** функцию, но **не** запускает ее.


### Функциональные вызовы


Чтобы запустить (или "вызвать") функцию, напишите ее имя, заключив его в круглые скобки:


```javascript
function sayHello () {console.log(`Hello!`) }
sayHello()
```


Это отпечатки:


```
Hello!
```


Вы можете вызывать функцию столько раз, сколько захотите:


```javascript
function sayHello() {
console.log("Hello!")
}

sayHello()
sayHello()
```


Это отпечатки:


```
Hello!
Hello!
```


Код внутри функции выполняется только тогда, когда вы ее вызываете.


### Аргументы функций (входные данные для функций)


Иногда требуется, чтобы функция работала с некоторыми входными данными. Это можно сделать, добавив **аргументы** внутри круглых скобок.


Например:


```javascript
function sayHelloTo (friend) {
console.log(`Hello ${friend}!`)
}
```


Теперь эта функция принимает **один аргумент** под названием `friend`.


При вызове функции можно передать значение:


```javascript
sayHelloTo("Tommy")
```


Это отпечатки:


```
Hello Tommy!
```


Вы можете вызвать функцию снова с другим именем:


```javascript
sayHelloTo("Sam")
```


Это отпечатки:


```
Hello Sam!
```


Переданное значение заменяет переменную `friend` внутри функции.


Вы также можете использовать более одного аргумента:


```javascript
function greetTwoPeople(person1, person2) {
console.log(`Hello ${person1} and ${person2}!`)
}

greetTwoPeople("Lina", "Marco")
```


Это отпечатки:


```
Hello Lina and Marco!
```


### `return` (вывод из функций)


Функции также могут **возвращать** значения. Это означает, что они отправляют значение обратно туда, где была вызвана функция.


Вот простой пример:


```javascript
function getNumber() {
return 42
}

const result = getNumber()
console.log(result)
```


Это отпечатки:


```
42
```


Функция `getNumber()` возвращает `42`, и мы сохраняем его в `result`, а затем выводим на печать.


Вы также можете вернуть то, что вычислили:


```javascript
function add(a, b) {
return a + b
}

const result = add(2, 3)
console.log(result)
```


Это отпечатки:


```
5
```


Как только значение `возвращено`, функция останавливается. Все, что происходит после `return` в этом блоке, не происходит.


```javascript
function saySomething() {

return "hi"

console.log("this never runs")

}

const message = saySomething()
console.log(message)
```


Только для печати:


```
hi
```


потому что возвращается только `"привет"`. Строка `console.log("this never runs")` пропускается.


Функции можно рассматривать как небольшие подпрограммы. Каждая функция может принимать входные данные, обрабатывать их и выдавать на выходе некоторую информацию.


Что произойдет, если мы попытаемся использовать возвращаемое значение функции, но эта функция ничего не вернула?


```javascript
function doesNothing () {}

const x = doesNothing()

console.log(x)
```

В результате будет выведено `undefined`. Возвращаемое значение функции, которая ничего не вернула, равно `undefined`.


## Объекты и классы

<chapterId>26689f25-8212-5057-8c21-3a05eee0ac75</chapterId>


JavaScript часто называют объектно-ориентированным языком.


Это значит, что он помогает вам организовать код, группируя значения и функции в **объекты**.


### Что такое `объект`?


Объект может содержать данные и функции, которые работают с этими данными. Когда функция помещается в объект, мы говорим, что это `метод`.


Первым объектом, с которым мы познакомились, был объект `console`. Это объект, который содержит несколько методов для вывода информации на экран и отладки наших программ.


Он даже может печатать сам; вы можете сделать


```javascript
console.log(console)
```


и он выведет список методов, которые в нем содержатся. Например, на моей машине он вывел


```javascript
Object [console] {
log: [Function: log],
warn: [Function: warn],
error: [Function: error],
dir: [Function: dir],
time: [Function: time],
timeEnd: [Function: timeEnd],
timeLog: [Function: timeLog],
trace: [Function: trace],
assert: [Function: assert],
clear: [Function: clear],
count: [Function: count],
countReset: [Function: countReset],
group: [Function: group],
groupEnd: [Function: groupEnd],
table: [Function: table],
debug: [Function: debug],
info: [Function: info],
dirxml: [Function: dirxml],
groupCollapsed: [Function: groupCollapsed],
Console: [Function: Console],
profile: [Function: profile],
profileEnd: [Function: profileEnd],
timeStamp: [Function: timeStamp],
context: [Function: context],
createTask: [Function: createTask]
}
```


Как видите, у него есть множество методов, которые можно использовать для отладки!


Javascript предоставляет нам различные способы создания новых объектов, которые могут делать все, что мы хотим.


### Создание объекта


Самый простой способ создать объект - это сгруппировать данные и функции с помощью **курсивных скобок** `{}`.


Таким образом создается то, что мы называем **анонимным объектом**


```javascript
const cat = {
name: "Whiskers",
age: 3
}
```


Это создает объект и сохраняет его в переменной `cat`.


У объекта есть два **свойства**:



- `имя` со значением `"Усы"`
- `возраст` со значением `3`


Давайте напечатаем его:


```javascript
console.log(cat)
```


Это отпечатки:


```
{ name: 'Whiskers', age: 3 }
```


Вы можете получить свойства из объекта с помощью точки, как в `objectName.propertyName`:


```javascript
console.log(cat.name)  // prints "Whiskers"
console.log(cat.age)   // prints 3
```


Вы также можете **изменить** свойство:


```javascript
cat.age = 4
console.log(cat.age)  // now it prints 4
```


Как видите, даже если объект определен как `const`, вы все равно можете изменять содержащиеся в нем данные.


В случае с объектами `const` не позволит вам переопределить только весь объект:


```javascript
const cat = {
name: "Whiskers",
age: 3
}

cat.age = 5 // this works

cat = 5 // this throws an error, you're trying to reassign the whole object

```


Как уже говорилось, объекты также могут содержать **функции**, и когда функция является частью объекта, мы называем ее **методом**.


Вот пример:


```javascript
const cat = {
name: "Whiskers",
speak () {
console.log("Meow!")
}
}
```


Этот объект имеет:



- Свойство `имя`
- Метод `speak()`


Давайте вызовем метод:


```javascript
cat.speak()
```


Он печатает:


```
Meow!
```


Методы могут использовать данные, которые содержит объект, с помощью ключевого слова `this`.

`this` обозначает текущий объект. В этом примере он будет использоваться для печати имени кошки:


```javascript
const cat = {
name: "Whiskers",
speak () {
console.log(`${this.name} says meow!`)
}
}

cat.speak()
```


Это отпечатки:


```
Whiskers says meow!
```


Слово `this` означает "этот объект"... в данном случае, объект `cat`.



Такие объекты отлично подходят, если вам нужно что-то быстрое и простое. Но если вам нужно создать **много объектов** с одинаковой структурой, есть лучший способ, и именно здесь на помощь приходят **классы**.


### Классы и конструкторы


Класс** - это как чертеж. Он указывает JavaScript, как создать объект определенного типа.


Вы определяете класс с помощью ключевого слова `class`, за которым следует имя класса и блок фигурных скобок `{}`.


```javascript
class Dog {}
```


По традиции классы начинаются с заглавной буквы.


Вы можете создать новый объект класса с помощью `new`:


```javascript
const hachiko = new Dog()
```


Попробуйте распечатать объект:

```javascript
class Dog {}

const myDog = new Dog()

console.log(myDog)
```


Вы получите


```
Dog {}
```


Как вы можете видеть, класс Dog пуст, поэтому объект `myDog` тоже пуст.


Мы можем определить, какие свойства должны содержать объекты Dog, добавив `конструктор`.


Конструктор - это специальная функция, которая запускается при создании (или "конструировании") нового объекта.


```javascript
class Dog {
constructor() { }
}
```


Мы хотим, чтобы у каждой собаки было имя, поэтому добавляем в функцию параметр `имя`:


```javascript
class Dog {
constructor(name) { }
}
```


А затем мы используем `this`, чтобы объявить, что `name` - это `имя` объекта `Dog`, который мы создаем


```javascript
class Dog {
constructor(name) {
this.name = name
}}
```


Давайте попробуем использовать его сейчас:


```javascript
class Dog {
constructor(name) {
this.name = name
}
}

const myDog = new Dog("hachiko")

console.log(myDog)
```


Это выводит что-то вроде:


```
Dog { name: 'hachiko' }
```


Как видите, метод `constructor` принимает аргументы, которые вы передаете классу при выполнении команды `new Dog()`, и использует их для создания объекта.


Давайте разберемся:



- класс "Собака" определяет класс "Собака".
- `constructor(name)` устанавливает объект при его создании.
- `this.name = name` сохраняет значение в новом объекте.
- `new Dog("hachiko")` создает новый объект из класса, со свойством `name`, установленным на `"hachiko"`.


Теперь давайте добавим метод в наш класс:


```javascript
class Dog {
constructor(name) {
this.name = name
}
speak () {
console.log(`${this.name} says barf!`)
}

}

const myDog = new Dog("hachiko")

myDog.speak()
```


Это позволит напечатать


```javascript
hachiko says barf!
```


Если мы сделаем то же самое для двух разных экземпляров Dog



```javascript
class Dog {
constructor(name) {
this.name = name
}
speak () {
console.log(`${this.name} says barf!`)
}

}

const myDog = new Dog("hachiko")

myDog.speak()

const yourDog = new Dog("bobby")

yourDog.speak()
```


мы получаем


```
hachiko says barf!
bobby says barf!
```


метод`speak()` использует свойство `name` собаки, на которой он вызывается.


Это основная причина существования классов: они позволяют нам определить набор методов, которые работают с данными, а затем создать несколько объектов, которые разделяют одну и ту же "форму" данных.


Когда мы вызываем метод одного из этих объектов, он будет работать с данными, хранящимися в этом конкретном объекте.


### Изменение формы объекта


Объекты в JavaScript очень гибкие. Даже создав объект, вы можете добавлять новые свойства или удалять существующие.


Это разрешено, но использовать его нужно осторожно.


Давайте начнем с нашего простого класса `Dog`:


```javascript
class Dog {
constructor(name) {
this.name = name
}

speak() {
console.log(`${this.name} says barf!`)
}
}

const myDog = new Dog("Fido")
```


На данный момент у `myDog` есть только одно свойство: `name`. Мы все еще можем добавить новые свойства после его создания:


```javascript
myDog.age = 5

console.log(myDog.age) // prints 5
```


Мы также можем добавить новый метод:


```javascript
myDog.jump = function () {
console.log(`${this.name} jumps!`)
}

myDog.jump() // Fido jumps!
```


Кроме того, мы можем удалять свойства, используя ключевое слово `delete`.


```javascript
delete myDog.name

console.log(myDog.name) // prints 'undefined'
```


Это работает, но тут важно знать кое-что: Движки JavaScript, такие как V8 (используется в Node.js и в браузере Chrome), работают быстрее, если у объектов всегда сохраняются одни и те же свойства. Если вы добавляете или удаляете свойства после создания объекта, это может замедлить работу.


В небольших программах это не имеет большого значения. Но в больших проектах (например, в играх) лучше с самого начала перечислить все свойства в конструкторе, даже если вы не будете использовать их сразу. Это сохраняет стабильность формы объекта и помогает коду работать быстрее.


Например, вместо этого:


```javascript
class Dog {
constructor(name) {
this.name = name
}
}

const dog = new Dog("Rex")
dog.age = 4
dog.breed = "Labrador"
```


Вы можете сделать


```javascript
class Dog {
constructor(name, age, breed) {
this.name = name
this.age = age
this.breed = breed
}
}

const dog = new Dog("Rex", 4, "Labrador")
```


Оба варианта работают, но второй лучше с точки зрения производительности. Вы заранее сообщаете движку, какие свойства будут у каждого объекта, и он может оптимизировать их соответствующим образом.


JavaScript позволяет свободно изменять форму объектов, но при использовании классов лучше заранее спланировать форму объекта.



### Наследование с помощью `extends` и `super()`


Иногда требуется создать класс, который *почти* такой же, как другой класс, но с некоторыми дополнительными возможностями.


Вместо того чтобы изменять форму объектов (что, как уже говорилось, не лучшим образом сказывается на производительности) или переписывать новый класс с нуля, JavaScript позволяет вам использовать нечто, называемое **наследование**.


Наследование означает, что один класс может **расширять** другой. Новый класс получает все свойства и методы старого, и вы можете добавить еще или изменить то, что вам нужно.


Допустим, у нас есть базовый класс `Vehicle`:


```javascript
class Vehicle {
constructor(brand) {
this.brand = brand
}

start() {
console.log(`${this.brand} vehicle is starting...`)
}
}
```


Теперь мы хотим создать класс `Car`. Автомобиль - это вид транспорта, но мы можем захотеть, чтобы у него были дополнительные функции или другое сообщение при запуске. Вместо того чтобы переписывать все заново, мы можем использовать `extends`:


```javascript
class Car extends Vehicle {
start() {
console.log(`${this.brand} car is ready to drive!`)
}
}
```


Класс `Car` теперь **наследует** все от `Vehicle`. Он получает свойство `brand`, а метод `start()` мы заменили своей собственной версией.


![](assets/en/4.webp)


Давайте попробуем:


```javascript
const myCar = new Car("Toyota")
myCar.start()
```


Это отпечатки:


```
Toyota car is ready to drive!
```


Хотя у `Car` нет собственного конструктора, он все равно использует конструктор из `Vehicle`. Но если мы хотим написать собственный конструктор в `Car`, мы можем это сделать, просто нужно включить вызов конструктора его родителя с помощью `super()`.


Вот как:


```javascript
class Vehicle {
constructor(brand) {
this.brand = brand
}

start() {
console.log(`${this.brand} vehicle is starting...`)
}

}

class Car extends Vehicle {
constructor(brand, model) {
super(brand) // call the parent constructor and passes the brand argument to it
this.model = model
}

start() {
console.log(`${this.brand} ${this.model} is ready to drive!`)
}
}

const myCar = new Car("Toyota", "Corolla")
myCar.start()
```


![](assets/en/5.webp)



Это отпечатки:


```
Toyota Corolla is ready to drive!
```


Итак, подведем итоги



- `extends` означает, что один класс основан на другом.
- `super()` используется для вызова конструктора класса, который вы расширяете.
- Новый класс получает все свойства и методы исходного класса.
- Вы можете **переопределить** методы (например, `start()`), чтобы заставить их делать что-то другое.


Это полезно, когда у вас есть несколько похожих вещей (например, автомобили, грузовики и велосипеды), и вы хотите, чтобы они имели общий код, но при этом вели себя по-своему.


### `instanceof`


Ключевое слово `instanceof` проверяет, был ли объект создан из определенного класса.


Допустим, у нас есть класс под названием `User`:

```javascript
class User {
```

## Обработка ошибок

<chapterId>d0206bc5-d386-5e7f-9917-5803f392448c</chapterId>


По мере написания более сложных программ на JavaScript вы будете сталкиваться с **ошибками**. Это неожиданные ситуации, когда что-то идет не так. Может быть, переменная `не определена`, но вы пытаетесь ее использовать, или какой-то код получает неправильный тип ввода.


Если мы не справимся с этими ошибками должным образом, наша программа может аварийно завершить работу или повести себя непредсказуемым образом. JavaScript предоставляет инструменты для обнаружения и управления этими ошибками, чтобы мы могли обрабатывать их более изящно.


### Общая ошибка: обращение к значению в `undefined`


Вот распространенная ситуация, которая приводит к ошибке:


```javascript
const user = undefined
console.log(user.name)
```


Если вы запустите этот код, то получите ошибку, которая выглядит следующим образом:


```
TypeError: Cannot read properties of undefined (reading 'name')
```


Это JavaScript говорит вам: "Эй, вы пытались получить свойство `name` от чего-то, что является `undefined`, и это не имеет смысла" Как видите, когда происходит подобная ошибка, программа перестает выполняться, если только вы специально не написали код для ее отлова и обработки.


### выбрасывание ошибки


Иногда вы хотите вручную **вызвать ошибку** в своем коде. В этом случае вы используете ключевое слово `throw`.


```javascript
throw new Error("This is a custom error message")
```


Это немедленно останавливает программу и выводит ее на печать:


```
Uncaught Error: This is a custom error message
```


Вы можете использовать `throw` для обеспечения соблюдения правил в вашей программе. Например:


```javascript
function divide(a, b) {
if (b === 0) {
throw new Error("You can't divide by zero")
}
return a / b
}

console.log(divide(10, 2))  // OK: prints 5
console.log(divide(10, 0))  // Error!
```


Второй вызов вызывает ошибку, поскольку деление на ноль в этом примере недопустимо.


### Ловля ошибок с помощью `try...catch`


Если вы не хотите, чтобы ваша программа аварийно завершалась при возникновении ошибки, вы можете перехватить ее с помощью блока `try...catch`. Это полезно, когда вы хотите, чтобы программа **продолжала работать**, даже если что-то не получилось.


```javascript
try {
const user = undefined
console.log(user.name)
console.log("End of the block") // this will never get printed
} catch (error) {
console.log("Oops! Something went wrong.")
}
```


Выход:


```
Oops! Something went wrong.
```


Вот как это работает:



- Код внутри блока `try` выполняется первым.
- Если возникает ошибка, JavaScript **переходит к блоку `catch`**, пропуская остальную часть блока `try`.
- Блок `catch` получает ошибку, и вы можете распечатать ее или обработать другим способом, как, например


```javascript
try {
const user = undefined
console.log(user.name)
console.log("End of the block") // this will never get printed
} catch (error) {
console.log(`The message of the error was: "${error.message}"`)
}
```


Выход:


```
The message of the error was: "Cannot read properties of undefined (reading 'name')"
```


### Блок `finally`


Вы также можете добавить блок `finally`. Это код, который **всегда выполняется**, независимо от того, была ли ошибка или нет.


```javascript
try {
console.log("Trying something risky...")
throw new Error("Uh oh!")
} catch (error) {
console.log("Caught the error:", error.message)
} finally {
console.log("This will run no matter what.")
}
```


Выход:


```
Trying something risky...
Caught the error: Uh oh!
This will run no matter what.
```


## Избегайте жучков

<chapterId>db12d9f6-5806-514c-998e-0ae24805104e</chapterId>


В этой главе рассказывается о некоторых наиболее распространенных подводных камнях в JavaScript и о том, как их избежать.


### `var` и Assignment без объявления


В старом коде JavaScript переменные часто объявлялись с помощью ключевого слова `var`. В отличие от `let` и `const`, о которых вы уже узнали, `var` может вести себя непонятным образом.


Например:


```javascript
{
var message = "hello"
}
console.log(message)
```


Можно было бы ожидать, что `message` будет существовать только внутри блока, но это не так. `var` игнорирует область видимости блока и делает переменную доступной во всей функции или файле.


Это может привести к неожиданному поведению, особенно в больших программах. По этой причине в современном JavaScript-коде всегда следует использовать `let` или `const` вместо `var`.


Что еще хуже, JavaScript позволяет присваивать значения переменным **без их объявления**:


```javascript
function greet() {
user = "Alice"
}
greet()
console.log(user) // prints "Alice"
```


Это создает новую глобальную переменную `name` без какого-либо объявления. Это может произойти незаметно и привести к ошибкам, которые Hard сложно отследить, особенно если это была просто опечатка. Всегда объявляйте переменные с помощью `let` или `const`.


### Слабая система типов


JavaScript слабо типизирован, то есть при необходимости он автоматически преобразует значения из одного типа в другой. Это называется принуждением к типу, и хотя это может быть удобно, часто приводит к запутанным результатам.


Например:


```javascript
console.log("5" + 1)    // "51"
console.log("5" - 1)    // 4
console.log(true + 1)   // 2
console.log(null + 1)   // 1
```


В этих примерах JavaScript пытается угадать, что вы имели в виду. Иногда он превращает строки в числа, или булевы числа в числа, или строки в строки. Это может заставить ваш код вести себя неожиданным образом.


Знать о слабой системе типизации JavaScript очень важно. Если что-то начинает вести себя странно, это может быть связано с неожиданным принуждением к типу.


### `"use strict"`


Вы можете включить более строгий режим, который превратит некоторые тихие ошибки в настоящие и не позволит вам использовать некоторые опасные возможности языка.


Чтобы включить этот более строгий режим, добавьте эту строку в начало вашего файла или функции:


```javascript
"use strict"
```


Например:


```javascript
"use strict"
name = "Alice" // ReferenceError: name is not defined
```


Без строгого режима JavaScript молча создал бы глобальную переменную `name`. Но в строгом режиме это становится реальной ошибкой, помогая вам обнаружить ошибки раньше.


Строгий режим также отключает некоторые устаревшие функции JavaScript и облегчает оптимизацию и поддержку вашего кода.


## Значение по сравнению с эталоном

<chapterId>bb898425-dc2f-5e5c-864b-0cb7a4a9aea9</chapterId>


JavaScript по-разному относится к различным типам значений.


Некоторые значения **копируются**, когда вы присваиваете их переменной.


Другие **обмениваются**, когда вы присваиваете их новой переменной, так что если вы измените одну из них, то изменится и другая.


### Передача по стоимости


Когда значение передается **по значению**, JavaScript создает его **копию**.


Так что если вы измените одну из них, это не повлияет на другую.


Это происходит с примитивными типами, например:



- номера
- строки
- булевые числа (`true` и `false`)
- `null`
- `undefined`


Давайте рассмотрим пример:


```javascript
let a = 5
let b = a

b = 10

console.log(a) // 5
console.log(b) // 10
```


Мы дали `b` значение `a`, но затем изменили `b` на `10`.


Поскольку числа передаются по значению, JavaScript скопировал `5` в `b`. Значение `5` в `b` не зависит от исходного `5` в `a`, поэтому изменение значения `b` не повлияло на `a`.


Давайте попробуем с помощью строки:


```javascript
let name1 = "Alice"
let name2 = name1

name2 = "Bob"

console.log(name1) // Alice
console.log(name2) // Bob
```


Опять же, изменение `name2` не повлияло на `name1`, потому что строки также передаются по значению.


То же самое происходит, когда вы передаете примитив в функцию: он копируется. Поэтому функция не может изменить оригинал.


```javascript
function plusOne(x) {
x = x + 1
}

let number = 5

plusOne(number)

console.log(number) // 5
```


Несмотря на то, что внутри функции `1` было добавлено к `x`, исходное `число` не изменилось.


Это потому, что в функцию была передана только **копия**.


### Передача по ссылке


Объекты передаются **по ссылке**.


Это означает, что вместо того, чтобы копировать их, JavaScript просто передает **ссылку** на них, и если вы измените ее, все остальные переменные, указывающие на нее, тоже увидят изменения.


Например:


```javascript
const person1 = { name: "Alice" }
const person2 = person1

person2.name = "Bob"

console.log(person1.name) // Bob
console.log(person2.name) // Bob
```


И `person1`, и `person2` указывают на один и тот же объект в памяти.


Поэтому, когда мы изменили `person2.name`, мы также изменили `person1.name`, потому что они оба смотрят на одно и то же.


Массивы - это объекты, поэтому попробуем проделать то же самое с массивом:


```javascript
const list1 = [1, 2, 3]
const list2 = list1

list2.push(4)

console.log(list1) // [1, 2, 3, 4]
console.log(list2) // [1, 2, 3, 4]
```


Мы поместили `4` в `список2`, но `список1` тоже пострадал, потому что они оба ссылаются на один и тот же массив.


Давайте посмотрим, что происходит, когда мы передаем объект в функцию:


```javascript
function rename(user) {
user.name = "Charlie"
}

const person = { name: "Dana" }

rename(person)

console.log(person.name) // Charlie
```


Функция изменила объект! Это потому, что она получила **ссылку** на исходный объект `person`.


Он не получил копию. Он получил доступ к оригинальному объекту, а вместе с ним и возможность изменять его.


Важно помнить об этом различии, потому что в противном случае наш код может вести себя не так, как мы ожидаем. Например, мы можем написать функцию с расчетом на то, что она не будет изменять полученные аргументы, а позже выяснить, что на самом деле она их изменяла (поскольку они были объектами и передавались по ссылке).


## Работа с функциями

<chapterId>e0d277a8-c642-5af7-9e53-dee27c811967</chapterId>


Вы уже узнали, как объявлять и использовать функции в JavaScript. Но JavaScript дает вам еще больше инструментов для работы с функциями.


### Функции стрелок


Стрелочные функции - это более короткий способ написания функций. Вместо ключевого слова `функция` мы используем стрелку (`=>`).


Вот обычная функция:


```javascript
function greet(name) {
return `Hello, ${name}!`
}
```


Версия со стрелками выглядит следующим образом:


```javascript
const greet = (name) => {
return `Hello, ${name}!`
}
```


Если функция содержит **только одну строку**, вы можете пропустить скобки и `return`:


```javascript
const greet = (name) => `Hello, ${name}!`
```


Если функция имеет **только один параметр**, можно даже пропустить круглые скобки вокруг параметров:


```javascript
const greet = name => `Hello, ${name}!`
```


Стрелочные функции очень распространены в современном JavaScript, поскольку они позволяют выражать простые функции с гораздо меньшим количеством шаблонов.


### Параметры по умолчанию


Иногда требуется, чтобы функция имела **значение по умолчанию**, если не передан ни один аргумент.


Вы можете сделать это следующим образом:


```javascript
function sayHello(name = "friend") {
console.log(`Hello, ${name}!`)
}

sayHello("Alice") // Hello, Alice!
sayHello()        // Hello, friend!
```


Значение по умолчанию `"friend"` используется, когда ничего не передано.


### Параметры спреда (`...`)


Что, если ваша функция принимает гибкое количество аргументов?


Вы можете использовать оператор **spread** (`...`), чтобы собрать их в массив:


```javascript
function logAll(...items) {
console.log(items)
}

logAll(1, 2, 3) // [1, 2, 3]
logAll("a", "b") // ["a", "b"]
```


Затем вы можете использовать цикл для обработки каждого элемента:


```javascript
function logEach(...items) {
for (const item of items) {
console.log(item)
}
}
```


Оператор spread полезен, когда вы не знаете, сколько аргументов будет передано.


### Функции высшего порядка


Функция **высшего порядка** - это функция, которая:



- принимает на вход другую функцию
- и/или возвращает функцию в качестве выхода


Вот простой пример:


```javascript
function runTwice(action) {
action()
action()
}

function sayHello(name = "friend") {
console.log(`Hello, ${name}!`)
}

runTwice(sayHello)
```


Это отпечатки:


```
Hello, friend!
Hello, friend!
```


Мы можем передать ему функцию стрелки:


```javascript
runTwice(
() => console.log("Hello!")
)
```


Это отпечатки:


```
Hello!
Hello!
```


Вы также можете писать функции, которые **возвращают** другие функции:


```javascript
function makeGreeter(name) {
return () => console.log(`Hi, ${name}`)
}

const greetAlice = makeGreeter("Alice")
const greetBob = makeGreeter("Bob")

greetAlice() // Hi, Alice
greetBob() // Hi, Bob
```


Функция `makeGreeter` - это функция, которая создает другие функции. Она получает строку и возвращает новую функцию, которая использует эту строку в вызове `console.log`.


Этот вид паттерна очень мощный, поскольку позволяет оставлять в функциях "дыры", которые впоследствии можно заполнить нужным вам поведением.


### `map()`, `filter()`, `reduce()`


JavaScript предоставляет вам несколько полезных встроенных методов для работы с массивами.


Эти методы принимают функции в качестве аргументов, поэтому они также являются функциями высшего порядка.


функция `map()` преобразует каждый элемент массива во что-то другое.


Пример:


```javascript
const numbers = [1, 2, 3]
const doubled = numbers.map(x => x * 2)

console.log(doubled) // [2, 4, 6]
```


Каждое число удваивается, и в результате получается новый массив.


`filter()` удаляет элементы из массива, если они не проходят проверку.


Пример:


```javascript
const numbers = [1, 2, 3, 4, 5]
const numbersGreaterThanTwo = numbers.filter(x => x > 2)

console.log(numbersGreaterThanTwo) // [3, 4, 5]
```


Сохраняются только те записи массива, для которых условие `x > 2` возвращает `истину`.


функция `reduce()` используется для объединения всех элементов массива в одно значение.


Пример:


```javascript
const numbers = [1, 2, 3, 4]
const total = numbers.reduce((current, next) => current + next)

console.log(total) // 10
```


`reduce` проходит через массив и, в данном примере, применяет оператор `+` между `1` и `2`, затем между полученными `3` и `3`, затем между полученными `6` и `4`, пока не получит сумму всех записей массива (которая равна 10).


Вы можете использовать `reduce()` для многих вещей, таких как итоги, средние значения или создание новых значений шаг за шагом.


Эти методы (`map`, `filter`, `reduce`) являются мощными инструментами для обработки данных без написания ручных циклов.


Вы даже можете объединить их в цепочку операций, например, так:


```javascript
const numbers = [1, 2, 3, 4, 5]

const result = numbers
.map(n => n * 2)        // Double each entry, obtain [2, 4, 6, 8, 10]
.filter(n => n > 3) // Keep only the entries bigger than 3, so you get [4, 6, 8, 10]
.reduce((n1, n2) => n1 + n2) // Adds them: 4 + 6 + 8 + 10 = 28

console.log(result) // 28
```


## Работа с объектами

<chapterId>7842aada-f009-5518-b8e3-1104e166a035</chapterId>


В этой главе мы познакомимся с некоторыми мощными и немного более продвинутыми инструментами для работы с объектами в JavaScript.


### Частная собственность


Иногда мы хотим скрыть свойство объекта, чтобы его нельзя было изменить или получить к нему доступ извне. JavaScript дает нам возможность сделать это, используя `#` перед именем свойства. Таким образом создается свойство **private**, доступ к которому возможен только изнутри класса.


```javascript
class Person {
#age // this is a private property

constructor(name, age) {
this.name = name
this.#age = age
}

getAge() {
return this.#age
}
}

const alice = new Person("Alice", 30)
console.log(alice.name)      // Alice
console.log(alice.getAge())  // 30, the method can access the private property
console.log(alice.#age)      // ❌ Error! You can't access private properties directly
```


Частные свойства полезны, когда нужно предотвратить случайные изменения.


### свойства `статических` свойств


Иногда нужно, чтобы свойство принадлежало самому классу, а не каждому объекту, который вы создаете на основе этого класса. Для этого и существует `static`. Свойство `static` содержится в классе, и все объекты этого класса будут ссылаться на него.


```javascript
class User {
static counter = 0 // this belongs to the class, not to instances. The same counter will be shared by all objects

constructor() {
User.counter++ // changes the static property every time an object of this class gets initiated
}
}

const a = new User() // the constructor will change the shared counter from 0 to 1
const b = new User() // the constructor will change the shared counter from 1 to 2

console.log(User.counter) //  prints 2
```


Это удобно для хранения общих данных и методов, которые применяются ко всей группе объектов, а не только к одному.


### `get` и `set`


В JavaScript `get` и `set` позволяют создавать свойства, которые *выглядят* как обычные переменные, но на самом деле выполняют специальный код в фоновом режиме.


Метод `get`ter запускается, когда вы пытаетесь *прочитать* свойство. Он объявляется путем написания `get` перед методом с именем свойства.


```javascript
class User {
constructor(firstName, lastName) {
this.firstName = firstName
this.lastName = lastName
}

get fullName() {
return `${this.firstName} ${this.lastName}`
}
}

const user = new User("Jane", "Doe")
console.log(user.fullName) // Jane Doe
```


Хотя `fullName` не является обычным свойством, мы можем использовать его как обычное, и за кулисами оно запускает функцию `get` для создания полного имени.


Метод `set`ter запускается, когда вы *присваиваете* значение свойству. Он позволяет вам контролировать, что произойдет, когда кто-то попытается изменить это значение. Он объявляется путем написания `set` перед методом с именем свойства.


```javascript
class User {
constructor() {
this.firstName = "John"
this.lastName = "Doe"
}

get fullName() {
return `${this.firstName} ${this.lastName}`
}

set fullName(input) {            // gets the name that is passed
const parts = input.split(" ") // breaks it into parts
this.firstName = parts[0]      // uses the first part as first name
this.lastName = parts[1]       // uses the second part as last name
}
}

const user = new User()
user.fullName = "John Smith"

console.log(user.firstName) // John
console.log(user.lastName)  // Smith
```


Когда мы делаем `user.fullName = "John Smith"`, выполняется метод `set` и обновляются значения `firstName` и `lastName`.


И хотя кажется, что мы просто устанавливаем простую переменную, на самом деле мы запускаем логику, которая обновляет другие свойства.


## Ключи и ценности

<chapterId>01a397b8-c12a-5c39-82b3-6d9ebbb72a29</chapterId>


Каждое свойство в объекте JavaScript имеет **ключ** (также называемый именем свойства) и **значение**.


Например:


```javascript
const user = {
name: "Alice",
age: 30
}
```


В этом объекте `"имя"` и `"возраст"` являются ключами, а "Alice" и `30` - их значениями.


### Динамический доступ


Иногда вы не знаете имя свойства заранее... возможно, вы получаете его из пользовательского ввода или читаете из переменной. Вы все равно можете получить к нему доступ, используя **скобочную нотацию**, например `myObject["keyName"]`.


```javascript
const user = {
name: "Alice",
age: 30
}

console.log(user["name"]) // Alice
```


Мы передали объекту строку `name`, чтобы получить соответствующее значение.


Мы можем сохранить ключ в переменной и использовать его для доступа к соответствующему значению позже, например


```javascript
const user = {
name: "Alice",
age: 30
}

const key = "name"

console.log(user[key]) // Alice
```


### Динамический Assignment


Вы также можете создавать или обновлять свойства объектов, используя переменные в качестве ключей.


```javascript
const settings = {}

const key = "theme"
settings[key] = "dark"

console.log(settings) // { theme: "dark" }
```


Это удобно, когда вы хотите построить объект шаг за шагом. Например:


```javascript
const user = {}

user["username"] = "bananaFan123"
user["email"] = "banana@fruit.com"

console.log(user)
// { username: "bananaFan123", email: "banana@fruit.com" }
```


Вы даже можете использовать динамический ключ *в процессе создания* объекта, используя квадратные скобки:


```javascript
const key = "language"
const config = {
[key]: "JavaScript"
}

console.log(config.language) // JavaScript
```


Это называется **вычисленным свойством**. Значение, заключенное в квадратные скобки, оценивается, и результат используется в качестве ключа.


### `Символ` Тип


Помимо строк, JavaScript также позволяет использовать специальный тип `Symbol` в качестве ключа объекта.


Давайте начнем с простого примера:


```javascript
const id = Symbol("userID")

const user = {
name: "Bob",
[id]: 12345
}

console.log(user[id]) // 12345
```


В этом примере `id` - это символ. Это не строка, но она все равно работает как ключ. Если вы попытаетесь вывести в консоль логи `user`, то увидите следующее:


```javascript
console.log(user) // { name: 'Bob', [Symbol(userID)]: 12345 }
```


Еще один важный момент: каждый созданный вами символ уникален, даже если они созданы с использованием одной и той же строки.


```javascript
const a = Symbol("label")
const b = Symbol("label")

console.log(a === b) // false
```


Символы позволяют определять ключи, которые не будут конфликтовать с обычными ключами. Например, допустим, у вас есть объект со свойством `name`, но в будущем этот объект будет настраиваться пользователем так, как вы не можете предсказать, и этот пользователь может добавить свойство `name`. Если первоначальное свойство `name` было задано строкой, то оно будет заменено новым, например, так:


```javascript
const obj = {
name: "John"
}

obj.name = "Jimmy"

console.log(obj) // { name: 'Jimmy' }
```


Если вместо этого мы используем символ:


```javascript
const name = Symbol("name")

const obj = {
[name]: "John"
}

obj.name = "Jimmy"

console.log(obj) // { name: 'Jimmy', [Symbol(name)]: 'John' }
```


Как видите, оригинальное свойство `name` при этом каким-то образом сохраняется. Это может быть полезно в некоторых крайних случаях.

JavaScript предоставляет нам несколько полезных встроенных объектов, которые помогают нам выполнять такие задачи, как отладка и математические операции.


### Другие методы `консоли


Вы уже видели `console.log`, который выводит значения на экран.


В объекте `console` есть еще несколько полезных методов, которые помогут вам отладить ваши программы.


#### `console.warn`


Выводит сообщение желтым цветом (или со значком предупреждения в некоторых средах):

```javascript
console.warn("This is just a warning.")
```

#### `console.error`


Выводит сообщение красным цветом, как при ошибке:


```javascript
console.error("Something went wrong!")
```

#### `console.table`

```javascript
const users = [
```

```javascript
console.time("timer")
```

Пригодится для простого тестирования производительности.


### Объект `Math`


JavaScript предоставляет вам объект `Math` с полезными методами для выполнения вычислений.


#### `Math.random()`


Возвращает случайное число в диапазоне от 0 (включительно) до 1 (исключительно):

```javascript
const r = Math.random()
```

Пример вывода:


```
0.4387429859
```

#### `Math.floor()` и `Math.ceil()`



- `Math.floor(n)` округляет **вниз** до ближайшего целого числа

```javascript
console.log(Math.floor(4.9)) // 4
```

Округляет до ближайшего целого числа:

```javascript
console.log(Math.round(4.4)) // 4
```

```javascript
console.log(Math.max(5, 9, 3)) // 9

```


#### `Math.pow()` и `Math.sqrt()`



- `Math.pow(a, b)` дает вам `a` в степени `b`
- `Math.sqrt(n)` дает квадратный корень из `n`


```javascript
console.log(Math.pow(2, 3))   // 8
console.log(Math.sqrt(16))    // 4
```

## Другие коллекции

<chapterId>a9a70c6d-a343-5a46-a383-e288bc2700e3</chapterId>


JavaScript предоставляет нам несколько специальных типов коллекций, которые выходят за рамки обычных массивов и объектов. К ним относятся `Map` и `Set`.


Они помогают хранить и управлять группами значений, но их работа отличается от того, что вы видели до сих пор.


Карта - это коллекция пар **ключ-значение**, как и объект. Но у него есть несколько важных отличий:



- Ключи могут быть **любыми значениями**, а не только строками.
- Порядок элементов сохраняется.
- В него встроены методы, облегчающие работу с ним.


Вы создаете новую карту следующим образом:


```javascript
const myMap = new Map()
```


Это создаст пустую карту. Чтобы добавить в нее записи, используйте `myMap.set(key, value)`:


```javascript
myMap.set("name", "Alice")
```


Это добавляет ключ `"name"` со значением `"Alice"`.


В качестве ключа можно также использовать номер:


```javascript
myMap.set(42, "The answer")
```


И даже объект в качестве ключа:


```javascript
const objKey = { id: 1 }
myMap.set(objKey, "Object as key")
```


Это не работает с обычными объектами, которые допускают только строковые ключи.


Чтобы **получить значение**, используйте `myMap.get(key)`:


```javascript
console.log(myMap.get("name"))     // Alice
console.log(myMap.get(42))         // The answer
console.log(myMap.get(objKey))     // Object as key
```


Чтобы **проверить, существует ли ключ**, используйте `myMap.has(key)`:


```javascript
console.log(myMap.has("name")) // true
```


Чтобы **удалить ключ**, используйте `myMap.delete(key)`:


```javascript
myMap.delete("name")
```


Чтобы **очистить всю карту**, используйте `myMap.clear()`:


```javascript
myMap.clear()
```


Карты отлично подходят для управления большими коллекциями значений, поскольку доступ к значениям на большой карте обычно намного производительнее, чем к большому объекту.


### `Set`


Набор - это коллекция **значений** (без ключей), где каждое значение должно быть **уникальным**. Это означает:



- Нельзя иметь одну и ту же ценность дважды
- Значения сохраняются в том порядке, в котором вы их добавляете


Вы создаете такой набор:


```javascript
const mySet = new Set()
```


Чтобы **добавить значения**, используйте `mySet.add(value)`:


```javascript
mySet.add(1)
mySet.add(2)
mySet.add(2) // duplicate, will be ignored
```


Даже если мы пытались добавить `2` дважды, набор сохранит только одну копию.


Чтобы **проверить, находится ли значение в наборе**, используйте `mySet.has(value)`:


```javascript
console.log(mySet.has(2)) // true
console.log(mySet.has(3)) // false
```


Чтобы **удалить значение**, используйте `mySet.delete(value)`:


```javascript
mySet.delete(2)
```


Чтобы **очистить все**, используйте `mySet.clear()`:


```javascript
mySet.clear()
```


Набор `Set` полезен, когда вы хотите хранить коллекцию уникальных значений без необходимости вручную проверять их на наличие дубликатов:


```javascript
const numberArray = [1, 2, 2, 3, 4, 4, 4, 5]

const numberSet = new Set(numberArray)

console.log(numberSet) // Set(5) { 1, 2, 3, 4, 5 }
```


С помощью `Set` можно избежать дублирования.


## Итераторы

<chapterId>61d24e5e-b7e4-541a-8322-778f61f26a72</chapterId>


Большинство вещей в JavaScript, по которым можно выполнять циклы (например, массивы, строки, карты, множества), являются **итерируемыми**: они могут предоставлять итераторы для своего содержимого.


**Итератор** - это специальный объект в JavaScript, который помогает вам просматривать список элементов **по одному**.


### итераторы `Object`


В отличие от массивов или карт, обычные объекты **не итерируются** с помощью `for...of`. Если вы попробуете сделать это:


```javascript
const user = {
name: "Alice",
age: 30
}

for (const value of user) {
console.log(value)
}
```


Вы получите ошибку:


```
TypeError: user is not iterable
```


Это потому, что у обычных объектов нет встроенного итератора. Но JavaScript дает вам другие инструменты для работы с ними.


#### `Объект.ключи()`


Вы можете использовать `Object.keys(obj)`, чтобы получить массив **ключей** объекта, а затем выполнить цикл по нему:


```javascript
const user = {
name: "Alice",
age: 30
}

const keys = Object.keys(user)

for (const key of keys) {
console.log(key)
}
```


Это отпечатки:


```
name
age
```


#### `Объект.значения()`


Чтобы перебрать **значения**, используйте `Object.values()`:


```javascript
const user = {
name: "Alice",
age: 30
}

const values = Object.values(user)

for (const value of values) {
console.log(value)
}
```


Это отпечатки:


```
Alice
30
```


#### `Объект.записи()`


Если вам нужны **и ключ, и значение**, используйте `Object.entries()`:


```javascript
const user = {
name: "Alice",
age: 30
}

const entries = Object.entries(user)

for (const [key, value] of entries) {
console.log(`${key} is ${value}`)
}
```


Это отпечатки:


```
name is Alice
age is 30
```


Несмотря на то, что объекты не являются итерируемыми напрямую, эти методы дают вам полный доступ к их содержимому таким образом, который хорошо работает с `for...of`.


Но как работают итераторы?


### `Символ.итератор`


Секрет всех итераторов заключается в специальном **символе** под названием `Symbol.iterator`.


Этот символ является встроенным ключом, который говорит JavaScript: "Этот объект можно итерировать"


Когда вы вызываете `myIterable[Symbol.iterator]()`, JavaScript возвращает вам объект **итератор** с методом `.next()`.


Давайте посмотрим, как это выглядит:


```javascript
const colors = ["red", "green", "blue"]

const iterator = colors[Symbol.iterator]()

console.log(iterator.next()) // { value: 'red', done: false }
```


Каждый вызов `.next()` дает вам следующее значение. По окончании он возвращается:


```javascript
{ value: undefined, done: true }
```


### `next()`


Метод `.next()` используется для получения следующего элемента из последовательности.


Каждый раз, когда вы вызываете `.next()`, вы получаете объект с двумя ключами:



- `значение`: текущий элемент
- `done`: логическое значение, которое сообщает, закончилась ли итерация


Давайте рассмотрим пример:


```javascript
const names = ["Lina", "Tom", "Eva"]      // declare an array
const iterator = names[Symbol.iterator]() // use the Symbol.iterator function to get an iterator for this array

let result = iterator.next()              // get the first element of the array

while (!result.done) {                    // repeat this loop until you reach the last element of the array, which is marked with { done: true }
console.log(result.value)               // print the value of each element
result = iterator.next()                // get the next element of the array
}
```


Это отпечатки:


```
Lina
Tom
Eva
```


Именно так работает цикл `for...of`: он использует этот паттерн с `.next()`.


Вы получите тот же результат, если


```javascript
const names = ["Lina", "Tom", "Eva"]

for (const result of names) {
console.log(result)
}
```


### Создание итерируемого класса


Вы также можете определить свой собственный класс **iterable**, добавив метод `[Symbol.iterator]()`.


Допустим, нам нужен класс, представляющий **диапазон чисел**, например от 1 до 5.


```javascript
class Range {
constructor(start, end) {
this.start = start
this.end = end
}

[Symbol.iterator]() {
let current = this.start
const end = this.end

return {
next() {
if (current <= end) {
const result = { value: current, done: false }
current = current + 1
return result
} else {
return { done: true }
}
}
}
}
}

const myRange = new Range(1, 5)

for (const num of myRange) {
console.log(num)
}
```


Это отпечатки:


```
1
2
3
4
5
```


Вот что происходит:



- Мы определили класс `Range`
- Внутри класса мы реализовали `[Symbol.iterator]()`, чтобы JavaScript знал, как выполнять итерацию
- Метод `next()` возвращает каждое число по очереди
- Когда мы достигаем `конца`, он возвращает `{ done: true }`


Теперь наш класс `Range` работает как массив, и мы можем использовать его в любом цикле, который ожидает итерабельность.


### Функции генератора и `yield`


Чтобы упростить создание итераторов, JavaScript предоставляет вам **генераторные функции**, использующие ключевое слово `function*` (это `function` с `*` в конце) и ключевое слово `yield`.


Давайте попробуем:


```javascript
function* numberGenerator() {
yield 1
yield 2
yield 3
}

const iterator = numberGenerator()

console.log(iterator.next()) // { value: 1, done: false }
console.log(iterator.next()) // { value: 2, done: false }
console.log(iterator.next()) // { value: 3, done: false }
console.log(iterator.next()) // { value: undefined, done: true }
```


Каждый `yield` возвращает значение и **приостанавливает** функцию до следующего вызова `.next()`.


Вы также можете перебирать генераторы с помощью `for...of`:


```javascript
for (const num of numberGenerator()) {
console.log(num)
}
```


Это отпечатки:


```
1
2
3
```


## Взаимодействие с обратными вызовами

<chapterId>f3fc76ca-b3ef-54eb-a06e-501007002054</chapterId>


До сих пор наш код был **синхронным**: он выполнялся по одной строке за раз, по порядку. Но некоторые вещи в реальном мире требуют времени, и мы не хотим, чтобы вся программа приостанавливалась в ожидании.


В этой главе мы познакомимся с новым понятием: **валюта**. Оно позволяет нам управлять порядком выполнения действий. Это полезно при работе с такими вещами, как таймеры, пользовательский ввод или чтение файлов с диска. JavaScript предлагает различные инструменты для работы с параллелизмом.


### `setTimeout`


Функция `setTimeout` позволяет **запустить функцию позже**, по истечении некоторого времени.


Пример:


```javascript
console.log("Start")

setTimeout(
() => console.log("This runs after 2 seconds"),
2000
)

console.log("End")
```


Это отпечатки:


```
Start
End
This runs after 2 seconds
```


Несмотря на то, что `setTimeout` появляется в середине кода, он не блокирует остальной код. Вместо этого он назначает выполнение функции на **позднее** и сразу переходит к следующему действию.


`2000` означает 2000 миллисекунд (что составляет 2 секунды).

Вот более подробное и удобное для начинающих переписывание разделов **Callbacks** и **Promise** с использованием манипуляций с данными и четких аннотаций:


### Обратные вызовы


Обратный вызов - это просто функция, которую мы передаем другой функции, чтобы ее можно было вызвать позже.


Давайте рассмотрим реальный пример на примере чисел. Представьте, что у нас есть список чисел, и мы хотим удвоить каждое из них, а затем применить функцию (обратный вызов) к полученному "удвоенному" массиву, но мы хотим сделать это с небольшой задержкой, как если бы мы ждали чего-то медленного (например, загрузки данных из интернета).


Вот функция, которая делает это с помощью **обратного вызова**:


```javascript
function doubleNumbers(numbersArray, callback) {
// Pretend we're doing a slow operation using setTimeout
setTimeout(() => {
// Use the map method to create a new array where each number is doubled
const doubled = numbersArray.map(n => n * 2)

// When we're done, we call the callback function with the result
callback(doubled)
}, 1000) // Wait 1 second before running the code inside
}
```


Попробуем воспользоваться этой функцией:


```javascript
const input = [1, 2, 3]

doubleNumbers(input, function(result) {
console.log("Here is the doubled array:", result)
})
```


Через 1 секунду появляется надпись:


```
Here is the doubled array: [ 2, 4, 6 ]
```


**Что здесь происходит?


1. Мы передаем `input` как список чисел, которые мы хотим удвоить.

2. Мы также передаем **функцию обратного вызова**, которая сообщает программе, что делать *после* удвоения.

3. Внутри `doubleNumbers` мы имитируем задержку с помощью `setTimeout`, а затем выполняем удвоение.

4. После этого мы вызываем обратный вызов на полученном "удвоенном" массиве.


Эта техника работает, но представьте, что вы хотите сделать **больше шагов** после этого, например, отфильтровать маленькие числа и затем сложить их. Вам придется **вложить** больше обратных вызовов, подобных этому:


```javascript
doubleNumbers(input, function(doubled) {
filterBigNumbers(doubled, function(filtered) {
sumNumbers(filtered, function(total) {
console.log("Final result:", total)
})
})
})
```


Это Hard для чтения и беспорядок. Такой стиль называется **callback hell**, и это именно то, для исправления чего был создан `Promise`.


## Конкуренция с обещаниями

<chapterId>30fddaca-729f-5c8d-bf86-8dfc7b3c9800</chapterId>


Обещание" - это встроенный объект JavaScript, который представляет собой значение, которое **будет готово в будущем**.


Мы можем создать Promise следующим образом:


```javascript
const promise = new Promise((resolve, reject) => {
// Do something that takes time here...

resolve("It worked!") // This means everything went OK
})
```


Часть `new Promise()` создает обещание.


Внутри него мы задаем функцию с двумя параметрами:



- `resolve` - функция, которую мы вызываем, когда все прошло успешно
- `reject` - функция, которую мы вызываем, если что-то пошло не так


В приведенном выше примере мы просто сразу же разрешаем его с сообщением `"Сработало!".


### `.then()`


Чтобы сделать что-то **после** выполнения обещания, мы используем `.then()`:


```javascript
const promise = new Promise((resolve, reject) => {
// Do something that takes time here...

resolve(100) // This means everything went OK
})

promise.then(result => {
console.log("The result is:", result)
})
```


Это отпечатки:


```
The result is: 100
```


Значение, которое мы передали в `resolve()`, будет отправлено в функцию внутри `.then()` как `result`.


Давайте смоделируем задачу, которая займет 2 секунды, используя `setTimeout`:


```javascript
const delayedPromise = new Promise(
(resolve, reject) => {
setTimeout(
() => resolve("Done waiting!"),
2000
)
})

delayedPromise.then(result => console.log(result))
```


Это позволит подождать 2 секунды, а затем напечатать:


```
Done waiting!
```


### `reject()`


Давайте создадим обещание, которое **не работает**:


```javascript
const failingPromise = new Promise((resolve, reject) => {
reject("Something went wrong")
})
```


Теперь, если мы используем `.then()` для этого, ничего не произойдет, потому что `.then()` обрабатывает только успех.


Для обработки ошибок мы используем `.catch()`:


```javascript
const failingPromise = new Promise((resolve, reject) => {
reject("Something went wrong")
})

failingPromise
.then(
result => console.log("This will NOT run:", result)
)
.catch(
error => console.log("Caught an error:", error)
)
```


При этом печатается только


```
Caught an error: Something went wrong
```


Значение, переданное в `reject()`, отправляется в функцию `.catch()`.


Давайте создадим обещание, которое **временами срабатывает, а иногда не срабатывает**, основываясь на некотором условии.


```javascript
function checkNumber(n) {
return new Promise((resolve, reject) => {
if (n > 0) {
resolve("Positive number")
} else {
reject("Not a positive number")
}
})
}
```


Теперь мы можем вызвать это и обработать оба случая:


```javascript
checkNumber(5)
.then(
msg => console.log("Success:", msg)
)
.catch(
err => console.log("Failure:", err)
)
```


Это отпечатки:


```
Success: Positive number
```


А если мы попробуем с другим номером:


```javascript
checkNumber(-1)
.then(
msg => console.log("Success:", msg)
)
.catch(
err => console.log("Failure:", err)
)
```


Он печатает:


```
Failure: Not a positive number
```


### Цепочка операций с использованием `Promise`



Мы можем переписать наш предыдущий пример, используя `Promise`, и он будет выглядеть намного чище.


Давайте начнем с написания новой версии нашей функции удвоения, но на этот раз она возвращает **обещание**:


```javascript
function doubleNumbers(numbers) {
return new Promise(resolve => {
// Wait 1 second before doing the operation
setTimeout(() => {
const doubled = numbers.map(n => n * 2)
resolve(doubled) // Return the result using resolve
}, 1000)
})
}
```


Теперь мы можем использовать `.then()`, чтобы указать JavaScript, что делать с результатом:


```javascript
function doubleNumbers(numbers) {
return new Promise(resolve => {
// Wait 1 second before doing the operation
setTimeout(() => {
const doubled = numbers.map(n => n * 2)
resolve(doubled) // Return the result using resolve
}, 1000)
})
}

const input = [1, 2, 3]

doubleNumbers(input)
.then(
result => console.log("Doubled numbers:", result)
)
```


Это отпечатки:


```
Doubled numbers: [ 2, 4, 6 ]
```


Пока что это работает так же, как и версия с обратным вызовом, но теперь код легче расширять и читать.


Допустим, мы хотим добавить больше шагов:


1. Сначала удвойте все числа

2. Затем удалите числа, меньшие 4

3. Наконец, сложите их все вместе


Мы можем написать по одной функции для каждого шага, используя все обещания:


```javascript
function doubleNumbers(numbers) {
return new Promise(resolve => {
// Wait 1 second before doing the operation
setTimeout(() => {
const doubled = numbers.map(n => n * 2)
resolve(doubled) // Return the result using resolve
}, 1000)
})
}

function filterBigNumbers(numbers) {
return new Promise(resolve => {
setTimeout(() => {
const filtered = numbers.filter(n => n > 3)
resolve(filtered)
}, 1000)
})
}

function sumNumbers(numbers) {
return new Promise(resolve => {
setTimeout(() => {
const total = numbers.reduce((acc, n) => acc + n, 0)
resolve(total)
}, 1000)
})
}
```


Теперь мы можем **соединить** их вместе следующим образом:


```javascript
const input = [1, 2, 3]

doubleNumbers(input)
.then(filterBigNumbers)
.then(sumNumbers)
.then(
result => console.log("Final result after all steps:", result)
)
```


Это отпечатки:


```
Final result after all steps: 10
```


Давайте разберемся, что это дает:


1. `doubleNumbers` удваивает массив: `[2, 4, 6]`

2. `ФильтрБольшиеЧисла` удаляет все, что ≤ 3: `[4, 6]`

3. `sumNumbers` складывает оставшиеся числа: `4 + 6 = 10`

4. Наконец, мы выводим результат.


Каждый `.then()` ожидает завершения предыдущего шага. Таким образом, мы можем построить **цепочку действий** без вложенности. Это делает код более читабельным и облегчает отладку.


## Параллелизм с `async`/`await`

<chapterId>6e93d29f-c8bf-5fd1-a9c9-4e794ee6cbd0</chapterId>


Мы видели, как цепочки `Promise` помогают нам избежать ада обратных вызовов, но они все равно могут стать немного Hard для чтения, когда в них задействовано много шагов.


Вот тут-то и приходят на помощь `async` и `await`. Они позволяют нам писать асинхронный код, который **выглядит как синхронный**, что делает его более понятным.


### Что такое `async`?


Когда вы пишете ключевое слово `async` перед функцией, JavaScript автоматически оборачивает возвращаемое значение функции в Promise.


Рассмотрим базовый пример:


```javascript
async function greet() {
return "hello"
}
```


Если вы вызовете эту функцию:


```javascript
const result = greet()
console.log(result)
```


Вы увидите это:


```
Promise { 'hello' }
```


Несмотря на то, что вы только что вернули строку, JavaScript превращает ее в Promise. Вы можете получить фактическое значение, используя `.then()`, как показано ниже:


```javascript
greet().then( result => console.log(result) ) // prints "hello"
```


Или вы можете использовать `await`...


### Что такое `ожидание`?


Ключевое слово `await` говорит JavaScript: "Подождите, пока это обещание не будет выполнено, и затем выдайте мне результат"


Но использовать `await` можно только **внутри асинхронной функции**.


Давайте перепишем пример, используя `await`:


```javascript
async function greet() {
return "hello"
}

async function greetAndLog() {
const result = await greet()
console.log(result)
}

greetAndLog() // prints "hello"
```


Теперь мы можем использовать результат, как если бы это было обычное значение.


Давайте теперь займемся чем-нибудь более полезным.


### Моделирование задержки с помощью `await`


Мы создадим простую функцию `wait`, которая принимает в качестве аргумента количество миллисекунд и просто разрешается через это количество миллисекунд, не делая ничего другого:


```javascript
function wait(ms) {
return new Promise(resolve => {
setTimeout(resolve, ms)
})
}
```


Давайте попробуем его использовать:


```javascript
async function test() {
console.log("waiting 2 seconds...")
await wait(2000)
console.log("done waiting")
}

test()
```


Это отпечатки:


```
waiting 2 seconds...
done waiting
```


Вы можете думать о `await` как о "паузе здесь, пока обещание не будет выполнено, затем продолжайте"


Это позволяет писать код **сверху вниз**, который ведет себя асинхронно, без цепочки вызовов `.then()`.


### Ожидание данных


Давайте повторим наш предыдущий пример, в котором мы удваивали числа, затем фильтровали, затем суммировали. Но на этот раз мы будем использовать `async`/`await`.


Мы создадим 3 функции, которые имитируют ожидание и возвращают Promises:


```javascript
function doubleNumbers(numbers) {
return new Promise(resolve => {
setTimeout(() => {
const doubled = numbers.map(n => n * 2)
resolve(doubled)
}, 1000)
})
}

function filterBigNumbers(numbers) {
return new Promise(resolve => {
setTimeout(() => {
const filtered = numbers.filter(n => n > 3)
resolve(filtered)
}, 1000)
})
}

function sumNumbers(numbers) {
return new Promise(resolve => {
setTimeout(() => {
const total = numbers.reduce((acc, n) => acc + n, 0)
resolve(total)
}, 1000)
})
}
```


Теперь давайте напишем функцию `async` для их объединения:


```javascript
async function process(numbers) {
const doubled = await doubleNumbers(numbers)
const filtered = await filterBigNumbers(doubled)
const total = await sumNumbers(filtered)

console.log("Final result:", total)
}

const input = [1, 2, 3]
process(input)
```


Это отпечатки:


```
Final result: 10
```


Это гораздо легче читать, чем цепочки `.then()` или вложенные обратные вызовы.


Она выглядит как обычная пошаговая программа, но при этом ведет себя асинхронно.


## Асинхронные итераторы

<chapterId>438b037d-9931-56d7-9052-7b4470f3c75b</chapterId>


Вы уже узнали о **итераторах** и о том, как мы можем использовать `for...of` для обхода массивов и других итерируемых объектов.


Но что делать, если данные, которые мы хотим итерировать, поступают не сразу?


Иногда мы хотим зациклиться на вещах, которые поступают **асинхронно**, например, сообщениях из чата, строках из файла или числах из медленного источника.


Для этого JavaScript предоставляет нам **async-итераторы**.


### Функции асинхронного генератора


Самый простой способ создать асинхронный итератор - использовать **асинхронную функцию-генератор**.


Мы пишем это так:


```javascript
async function* generateNumbers() {
yield 1
yield 2
yield 3
}
```


Это выглядит так же, как обычный генератор, но с `async` перед ним.


Теперь мы можем использовать `for await...of` для потребления значений:


```javascript
async function run() {
for await (const n of generateNumbers()) {
console.log("Got number:", n)
}

console.log("Done!")
}

run()
```


Напечатается:


```
Got number: 1
Got number: 2
Got number: 3
Done!
```


В чем же разница с обычным генератором?


Разница в том, что теперь мы можем использовать `await` внутри генератора.


Давайте снова сделаем помощника по задержке:


```javascript
function wait(ms) {
return new Promise(resolve => setTimeout(resolve, ms))
}
```


Теперь давайте выводить числа **медленно**:


```javascript
async function* generateSlowNumbers() {
await wait(1000)
yield 1

await wait(1000)
yield 2

await wait(1000)
yield 3
}
```


Попробуйте использовать его:


```javascript
async function run() {
for await (const n of generateSlowNumbers()) {
console.log("Got number:", n)
}

console.log("Done!")
}

run()
```


### Зачем использовать асинхронные итераторы?


Асинхронные итераторы полезны, когда:



- Ценности не приходят все сразу.
- Вы хотите справляться с ними по очереди, **по мере их поступления**.
- Вы работаете с Promises и хотите сделать цикл чистым.


Например, если вы хотите загружать сообщения с сервера чата по одному или скачивать большой файл кусками, итераторы async дадут вам возможность написать цикл `for`, который будет работать с отложенными данными.


### `Symbol.asyncIterator`


Мы также можем использовать асинхронные итераторы в пользовательских классах.


Вот пример, который выдает числа с задержкой:


```javascript
function wait(ms) {
return new Promise(resolve => setTimeout(resolve, ms))
}

class DelayedNumbers {
constructor(numbers) {
this.numbers = numbers
}

async *[Symbol.asyncIterator]() {
for (const n of this.numbers) {
await wait(1000)
yield n
}
}
}
```


Теперь мы можем использовать `for await...of`, как и раньше:


```javascript
async function run() {
const source = new DelayedNumbers([10, 20, 30])

for await (const n of source) {
console.log("Received:", n)
}

console.log("All done!")
}

run()
```


Это позволяет создавать объекты, над которыми можно выполнять асинхронные итерации


## Assignment синтаксический сахар

<chapterId>8b1ba7d8-ecfd-5470-b86e-73cb84ccc8b7</chapterId>


"Синтаксический сахар" означает написание чего-либо более коротким или простым способом, не меняя того, что оно делает. Это просто более приятный способ сказать то же самое.


В JavaScript есть встроенный синтаксический сахар, который позволяет нам писать более чистые и короткие объявления. В этой главе мы рассмотрим, как присваивать значения на основе условия, обновлять переменные с помощью математики, извлекать значения из массивов или объектов, а также копировать или комбинировать их с помощью более простого синтаксиса.


### Тернарный оператор


В JavaScript вы можете присваивать значение на основе условия с помощью оператора **ternary**, который представляет собой краткий способ написания `if...else`.


Вместо того чтобы делать:


```javascript
let message

if (isMorning) {
message = "Good morning"
} else {
message = "Hello"
}
```


Вы можете написать:


```javascript
const message = isMorning ? "Good morning" : "Hello"
```


Это означает:



- Если `isMorning` равен true, используйте `"Good morning"`
- В противном случае используйте `"Hello"`


Общая форма такова:


```javascript
condition ? valueIfTrue : valueIfFalse
```


Вы можете использовать его и внутри других выражений:


```javascript
console.log(isSunny ? "Go outside" : "Stay in")
```


Просто убедитесь, что используете его для **простых** решений. Если логика сложная, используйте `if...else`.


### Альтернативные операторы Assignment


В JavaScript есть операторы **shortcut** для выполнения присваиваний в сочетании с операциями.


Давайте рассмотрим обычный способ:


```javascript
let counter = 1
counter = counter + 1
```


Это можно сократить до:


```javascript
let counter = 1
counter += 1 // same as counter = counter + 1
```


Вот самые распространенные из них:


| Operator | Meaning             |
| -------- | ------------------- |
| `+=`     | add and assign      |
| `-=`     | subtract and assign |
| `*=`     | multiply and assign |
| `/=`     | divide and assign   |

Примеры:


```javascript
let score = 10
score += 5  // now score is 15
score *= 2  // now score is 30
score -= 10 // now score is 20
```


Они полезны, когда вы хотите обновить переменную, используя ее собственное значение.


### Деструктуризация


**Деструктуризация** позволяет легко извлекать значения из массивов или объектов и сохранять их в переменных.


#### Массивы


Предположим, у вас есть:


```javascript
const colors = ["red", "green", "blue"]
```


Вместо того чтобы делать:


```javascript
const first = colors[0]
const second = colors[1]
```


Вы можете сделать это:


```javascript
const [first, second] = colors
```


Это назначает:



- `первый` - `красный`
- `Второй` - `"Green"`


Вы также можете пропустить значения:


```javascript
const [,, third] = colors
console.log(third) // blue
```


#### Объекты


Вы также можете извлекать значения из объектов:


```javascript
const user = { name: "Alice", age: 30 }

const { name, age } = user
console.log(name) // Alice
console.log(age)  // 30
```


Если свойство имеет другое имя, чем нужная вам переменная, вы можете переименовать его:


```javascript
const { name: username } = user
console.log(username) // Alice
```


Деструктуризация делает ваш код чище при работе с объектами и массивами.


### Синтаксис спреда


Синтаксис **spread** использует `...` для распаковки или копирования значений.


#### Массивы


Вы можете копировать или объединять массивы:


```javascript
const a = [1, 2]
const b = [3, 4]

const both = [...a, ...b]
console.log(both) // [1, 2, 3, 4]
```


Вы также можете клонировать массив:


```javascript
const original = [10, 20, 30]
const clone = [...original]
```


#### Объекты


То же самое можно сделать с объектами:


```javascript
const a = { x: 1 }
const b = { y: 2 }

const merged = { ...a, ...b }
console.log(merged) // { x: 1, y: 2 }
```


Вы также можете переопределить значения:


```javascript
const user = { name: "Alice", age: 30 }
const updated = { ...user, age: 31 }

console.log(updated) // { name: "Alice", age: 31 }
```


Это очень удобно при обновлении объектов без изменения оригинала.

## Как мы пришли к Node

<chapterId>0da1d60c-06c9-54e6-a181-ae7dabf6e3b8</chapterId>


В этой главе мы узнаем немного исторического контекста о JavaScript и NodeJS.


Исторический контекст очень важен для программного обеспечения, потому что мы часто используем инструменты, созданные другими людьми, и поэтому на нас влияют решения, принятые ими в прошлом.


Понимание причин этих решений и того, как инструменты, которые мы используем, стали такими, какие они есть, поможет нам не так сильно запутаться в том, что мы делаем.


### Происхождение JavaScript


JavaScript начинался как простой язык, предназначенный для создания интерактивных веб-страниц.


В 1990-х годах веб-браузеры, такие как Netscape Navigator, добавили JavaScript, чтобы разработчики могли писать код, выполняющийся непосредственно в браузере.


Изначально предполагалось, что Java будет основным языком для создания сайтов (с так называемыми "Java-апплетами"), а JavaScript - только для второстепенных задач.


Основной дизайн был разработан Бренданом Эйхом, который в то время был сотрудником Netscape, менее чем за 2 недели.


Но большинство людей предпочитали использовать JavaScript, а не Java, к тому же у Java-апплетов в то время были некоторые проблемы с безопасностью, так что в конце концов Java перестали использовать, и JavaScript стал стандартом де-факто для веб-разработки.


### Двигатель V8


JavaScript - интерпретируемый язык, в отличие от компилируемых языков, таких как C.


Код, написанный на компилируемом языке, превращается в двоичный файл, а двоичный файл поступает непосредственно в процессор компьютера.


![](assets/en/6.webp)


Интерпретированные языки, с другой стороны, как правило, более удобны для пользователя и ближе к тому, как мыслит человек ("высокий уровень"), а не к тому, как работают машины ("низкий уровень"); поэтому для выполнения их кода обычно используется виртуальная машина.


Виртуальная машина - это специальная программа, которая находится между написанным вами кодом и процессором и выполняет ваш код (потому что процессор не может его понять).


Это позволяет вам программировать, не зная слишком многого о машине, лежащей в основе, но это также имеет свою цену в плане производительности, потому что компьютер не выполняет только вашу программу; он выполняет программу (виртуальную машину), которая выполняет вашу программу.


По мере того как веб-приложения становились все сложнее и сложнее, возникла потребность в повышении производительности JavaScript. Движок V8 - это интерпретатор, на котором работает JavaScript в Google Chrome. Он был разработан в Google и выпущен в 2008 году.


Если старые движки JavaScript были в основном традиционными виртуальными машинами, то движок V8 представляет собой JIT-компилятор (just-in-time).


Код JavaScript поступает в движок V8, а тот пытается на лету скомпилировать его части в нативные двоичные файлы. Это позволяет вам получить опыт работы с языком высокого уровня, а по производительности он немного ближе к языкам низкого уровня. Это сделало JavaScript самым быстрым языком высокого уровня в мире, своего рода "лучшее из двух миров".


### Время выполнения NodeJS


Несмотря на простоту использования и высокую скорость выполнения, после выхода V8 у JavaScript появилось огромное ограничение: он мог работать только в браузере.


Почему это проблема?


Поскольку браузеры выполняют код, взятый из миллионов различных источников в Интернете, они могут легко превратиться во вредоносное ПО, поэтому их "изолируют" от остальной операционной системы.


![](assets/en/7.webp)


JavaScript не мог получить доступ к файловой системе и другим локальным ресурсам вашего компьютера (по крайней мере, не так легко, как другие языки), поэтому это существенно ограничивало возможности создания приложений на его основе.


В 2009 году Райан Дал опубликовал NodeJS, который представляет собой среду выполнения, позволяющую использовать движок V8 вне браузера, непосредственно на родной операционной системе вашего компьютера. Она также добавляет множество функций, полезных для написания программ на стороне сервера и командной строки. Например, с помощью NodeJS можно создать веб-сервер, читать и записывать файлы или создавать инструменты для автоматизации задач.


![](assets/en/8.webp)


В этом курсе мы изучили возможности JavaScript, которые присутствуют как в браузере, так и в NodeJS. Эти возможности позволили нам определить данные и манипулировать ими абстрактными способами. В следующих нескольких уроках мы рассмотрим функции, характерные для NodeJS и позволяющие взаимодействовать с операционной системой.


## Аргументы командной строки

<chapterId>960d20f3-c424-5d51-a041-ef17d2e94b6d</chapterId>


NodeJS позволяет нам, помимо прочего, создавать CLI (интерфейсы командной строки).


Для этого нам нужен способ получения аргументов командной строки, который в Node осуществляется с помощью встроенного объекта `process`.


### `процесс`


NodeJS предоставляет специальный объект `process`, который представляет текущую запущенную программу.


С его помощью можно проверить окружение, текущий рабочий каталог и даже выйти из программы при необходимости.


Например:


```javascript
console.log(process.platform)
```


Здесь указывается платформа операционной системы, например `win32`, `linux` или `darwin` (Mac).


### `process.argv`


Когда вы запускаете программу NodeJS из терминала, вы можете передать дополнительные слова (аргументы) после имени скрипта. Они хранятся в файле `process.argv`.


Например, если вы выполните эту команду:


```
node my_script.js alpha beta
```


Вы можете вывести аргументы следующим образом:


```javascript
console.log(process.argv)
```


Вывод может выглядеть следующим образом:


```
[ '/path/to/node', '/path/to/my_script.js', 'alpha', 'beta' ]
```


Первые два элемента - это путь к узлу и путь к вашему скрипту. Любые дополнительные слова, которые вы передали скрипту, идут после этого.


Массив `process.argv` может быть разрезан как любой другой массив с помощью метода `.slice()`, поэтому, чтобы получить только переданные аргументы, вы можете сделать следующее


```javascript
const args = process.argv.slice(2)

console.log(args)
```


Доступ к аргументам, передаваемым пользователем, является основой для разработки приложений командной строки.


## Модули

<chapterId>4e1651a5-65fd-50bc-b22a-40313d5659ca</chapterId>


Такие среды выполнения JavaScript, как NodeJS, обычно рассматривают каждый файл JavaScript как отдельный модуль.


Думайте о модуле как о коробке. У коробки есть собственное личное пространство, поэтому переменные и функции, которые вы в ней объявили, не мешают коду в других коробках. По сути, у каждого модуля есть своя область видимости.


Модуль может экспортировать часть своего содержимого, делая его доступным для других модулей, и может импортировать содержимое, которое экспортировали другие модули. JavaScript позволяет экспортировать и импортировать содержимое между модулями, соединять различные файлы.


Программа на JavaScript часто состоит из нескольких модулей, которые связаны друг с другом.


Зачем использовать модули? Разбивая код на модули, вы можете организовать свою программу на более мелкие, понятные и многократно используемые части. Каждый модуль может фокусироваться только на одном типе задач, например на математических вычислениях, работе с файлами или форматировании текста.


### Модули CommonJS


В NodeJS наиболее распространенная система управления модулями называется **CommonJS**.


В этой системе вы можете передавать (экспортировать) код из модуля с помощью `module.exports` и загружать (импортировать) его в другой файл с помощью `require()`.


Чтобы сделать что-то доступным вне модуля, вы назначаете его в `module.exports`:


```javascript
const greeting = "Hello!"

module.exports = greeting
```


Здесь строка `"Hello!"` - это то, что экспортирует данный модуль.


Чтобы использовать экспортированный код из другого файла, вы используете функцию `require()` с указанием пути к этому файлу:


```javascript
const greeting = require("./greeting.js")

console.log(greeting)
```


Это отпечатки:


```
Hello!
```


Вы можете экспортировать несколько вещей, объединив их в анонимный объект, например


```javascript
const greeting1 = "hello"

const greeting2 = "hi"

module.exports = {
greeting1, greeting2
}
```


CommonJS - это система модулей, которая изначально была принята в NodeJS. Позже были добавлены модули ES.


### Модули ES


NodeJS также поддерживает другой тип модулей, называемых **ES Modules**, которые популярны в веб-разработке. В них используются ключевые слова `export` и `import`.


Модули ES были разработаны для браузера и только позже добавлены в Node. Если вы хотите использовать их, вам, возможно, придется использовать `.mjs` в качестве расширения файла вместо `.js`, в зависимости от того, какую версию Node вы используете.


В одном файле под названием `greeting.mjs` мы пишем


```javascript
export const greeting = "Hello!"
```

Как видите, мы экспортируем константу непосредственно туда, где она определяется


В другом файле под названием `index.mjs` мы импортируем его:


```javascript
import { greeting } from "./greeting.mjs"

console.log(greeting)
```


Вы можете экспортировать различные декларации в разные части файла, например


```javascript
export const num = 10

export function double (x) {
return x*2
}
```


### Стандартная библиотека NodeJS


NodeJS также включает в себя множество встроенных модулей. Это готовые модули, предоставляемые NodeJS, которые помогают решать такие распространенные задачи, как чтение файлов, работа с операционной системой или подключение к сети.


Например, модуль `os` предоставляет информацию о вашей операционной системе:


```javascript
const os = require("os")

console.log(os.platform())
```


Вам не нужно устанавливать эти встроенные модули, они поставляются вместе с NodeJS. Они составляют "стандартную библиотеку" среды выполнения и используются большинством приложений Node для выполнения таких действий, как чтение файлов или общение через интернет.


В следующих главах вы найдете несколько полезных примеров их использования.


## Модуль `fs`

<chapterId>911e953a-35ae-5ee7-bd74-372501c32e81</chapterId>


Модуль `fs` (сокращение от **файловая система**) является частью стандартной библиотеки NodeJS. Он позволяет работать с файлами и каталогами на вашем компьютере: вы можете читать файлы, записывать файлы, удалять их, переименовывать и многое другое.


Чтобы использовать его, сначала нужно импортировать его в верхнюю часть вашего файла:


```javascript
const fs = require("fs")
```


### API синхронизации


Самый простой способ использовать `fs` - это его методы **sync**.


Эти методы блокируют выполнение программы до тех пор, пока не завершат работу (поэтому следующая строка кода не будет запущена до завершения операции).


Вот пример синхронного чтения файла:


```javascript
const fs = require("fs")

const data = fs.readFileSync("example.txt", "utf8")

console.log(data)
```


Если в том же каталоге, что и ваш скрипт, есть файл с именем `example.txt`, это выведет его содержимое.


Вы также можете записывать в файл синхронно:


```javascript
const fs = require("fs")

fs.writeFileSync("output.txt", "Hello from Node!")

console.log("File written!")
```


В результате создается (или перезаписывается) файл `output.txt` с текстом.


Вот некоторые общие операции, которые можно выполнить с помощью этого API:


```javascript
const fs = require("fs")

// List files and folders
const items = fs.readdirSync(".")
console.log("Items in current directory:", items)

// Create folder
fs.mkdirSync("my_folder")
console.log("Folder created")

// Delete folder
fs.rmdirSync("my_folder")
console.log("Folder deleted")

// Create & write file
fs.writeFileSync("my_file.txt", "Hello world")
console.log("File created & written")

// Read file
const content = fs.readFileSync("my_file.txt", "utf8")
console.log("File content:", content)

// Delete file
fs.unlinkSync("my_file.txt")
console.log("File deleted")
```


Sync API прост и хорош для небольших скриптов, но поскольку он блокирует работу программы до тех пор, пока она не будет завершена, это может замедлить работу, если вы работаете с большими файлами или сервером.


### API обратного вызова async


API **callback** является неблокирующим: он позволяет NodeJS продолжать выполнять другие действия, пока происходит операция с файлом.


Вместо того чтобы возвращать результат напрямую, он принимает функцию (**callback**), которая вызывается по завершении операции.


```javascript
const fs = require("fs")

fs.readFile("example.txt", "utf8", (err, data) => {
if (err) {
console.error("Error reading file:", err)
} else {
console.log(data)
}
})
```


Вот что происходит:



- `fs.readFile` начинает чтение файла `example.txt`.
- NodeJS не ждет, а переходит к выполнению другого кода, который вы могли написать.
- Когда чтение файла завершено, выполняется обратный вызов:



  - Если произошла ошибка, `err` содержит ее.
  - В противном случае `data` содержит содержимое.


Вот как выполняется запись в файл:


```javascript
const fs = require("fs")

fs.writeFile("output.txt", "Hello async!", (err) => {
if (err) {
console.error("Error writing file:", err)
} else {
console.log("File written!")
}
})
```


Идея та же: программа не останавливается во время записи файла.


Некоторые примеры того, что можно сделать с помощью этого API:


```javascript
const fs = require("fs")

// List files and folders
fs.readdir(".", (err, items) => {
if (err) return console.error(err)
console.log("Items in current directory:", items)
})

// Create folder
fs.mkdir("my_folder", (err) => {
if (err) return console.error(err)
console.log("Folder created")
})

// Delete folder
fs.rmdir("my_folder", (err) => {
if (err) return console.error(err)
console.log("Folder deleted")
})

// Create & write file
fs.writeFile("my_file.txt", "Hello world", (err) => {
if (err) return console.error(err)
console.log("File created & written")
})

// Read file
fs.readFile("my_file.txt", "utf8", (err, content) => {
if (err) return console.error(err)
console.log("File content:", content)
})

// Delete file
fs.unlink("my_file.txt", (err) => {
if (err) return console.error(err)
console.log("File deleted")
})
```


API обратного вызова лучше подходит для серверов и больших задач, потому что не блокирует работу программы, но вложенные обратные вызовы могут стать беспорядочными, если вы выстроите много операций в цепочку. Поэтому был добавлен async API, основанный на обещаниях.


### Promise async API


Promise-based API является современным и отлично работает с `.then()` и `async/await`. Он доступен в виде `fs.promises`.


Вам необходимо импортировать свойство `promises`:


```javascript
const fs = require("fs").promises
```


Использование `.then()`:


```javascript
const fs = require("fs").promises

fs.readFile("example.txt", "utf8")
.then(data => {
console.log(data)
})
.catch(err => {
console.error("Error reading file:", err)
})
```


А еще лучше использовать `async/await`:


```javascript
const fs = require("fs").promises

async function readFile(fileName) {
try {
const data = await fs.readFile(fileName, "utf8")
console.log(data)
} catch (err) {
console.error("Error reading file:", err)
}
}

readFile("example.txt")
```


Запись в файл:


```javascript
const fs = require("fs").promises

async function writeFile(fileName, content) {
try {
await fs.writeFile(fileName, content)
console.log("File written!")
} catch (err) {
console.error("Error writing file:", err)
}
}

writeFile("output.txt", "Hello from promises!")
```


Обычный список примеров для API:


```javascript
const fs = require("fs").promises

// Use an async function to await operations
async function main() {
// List files and folders
const items = await fs.readdir(".")
console.log("Items in current directory:", items)

// Create folder
await fs.mkdir("my_folder")
console.log("Folder created")

// Delete folder
await fs.rmdir("my_folder")
console.log("Folder deleted")

// Create & write file
await fs.writeFile("my_file.txt", "Hello world")
console.log("File created & written")

// Read file
const content = await fs.readFile("my_file.txt", "utf8")
console.log("File content:", content)

// Delete file
await fs.unlink("my_file.txt")
console.log("File deleted")
}

main().catch(err => console.error(err))
```



## NPM

<chapterId>a91d9a75-55cc-51a3-a48f-0c0be6fe6e72</chapterId>


Когда вы пишете код, вам часто приходится использовать код, написанный другими людьми; например, библиотеки, помогающие вам работать с датами, цветами, серверами и почти всем остальным.


Вместо того чтобы загружать и копировать файлы вручную, вы можете воспользоваться **менеджером пакетов**.


Менеджер пакетов - это инструмент, который:



- пакеты загрузок
- отслеживает, какие пакеты нужны вашему проекту
- гарантирует, что все члены вашей команды имеют одинаковые версии пакетов


### Что такое NPM


В мире NodeJS самым популярным менеджером пакетов является **NPM**, что расшифровывается как *Node Package Manager*.


Вы получаете NPM автоматически при установке NodeJS.


Вы можете проверить, есть ли у вас NPM, запустив это в терминале:


```
npm -v
```


Здесь выводится версия NPM, которая у вас установлена, например:


```
10.2.1
```


### Создание пакета


В NodeJS **пакет** - это просто директория с файлом `package.json` в ней.


Давайте создадим его шаг за шагом.


1. Создайте новую папку для своего проекта:


```
mkdir my_project
cd my_project
```


2. Выполните эту команду:


```
npm init
```


Это запустит интерактивную настройку, которая задаст вам такие вопросы, как название вашего проекта, версия, описание и т. д.


Если вы не хотите отвечать на все вопросы и просто принимаете значения по умолчанию, вы можете использовать:


```
npm init -y
```


После его запуска в вашей папке появится новый файл под названием:


```
package.json
```


### `package.json`


Файл `package.json` - это просто JSON-файл, в котором хранятся метаданные о вашем проекте.


Вот пример:


```json
{
"name": "my_project",
"version": "1.0.0",
"description": "",
"main": "index.js",
"scripts": {
"test": "echo \"Error: no test specified\" && exit 1"
},
"keywords": [],
"author": "",
"license": "ISC",
"type": "commonjs"
}

```


Несколько важных полей:



- `name`: название вашего пакета
- `version`: текущая версия
- `main`: файл точки входа (например, `index.js`)
- `scripts`: команды, которые вы можете запускать (например, `npm start`)
- `зависимости`: список всех пакетов, от которых зависит ваш проект


### Установка пакета


Допустим, вы хотите использовать определенный пакет `picocolors` для добавления цветов в вывод терминала.


Вы можете установить его, выполнив команду:


```
npm install picocolors
```


Теперь вы можете использовать его в своем проекте. Создайте файл `index.js` с


```javascript
const pico = require('picocolors')

console.log(
pico.green("This text is green!")
)
```


и попробуйте запустить его. Терминал должен вывести цветную версию текста.


Что сделал NPM?



- Он загрузил пакет и сохранил его в подпапке `node_modules/`
- он добавил запись в `зависимости` в ваш `package.json`
- он обновил файл `package-lock.json`


Что такое `package-lock.json`?


### `package-lock.json`


Этот файл автоматически создается NPM.


Вы можете задаться вопросом, если у нас уже есть `package.json`, зачем нам нужен еще один файл?

Вот причина:



- в `package.json` просто указано, какая версия **диапазона** пакета нужна вашему проекту.

Пример:


```json
"dependencies": {
"picocolors": "^1.1.0"
}
```


`^1.1.0` означает "любая версия, совместимая с 1.1.x", так что это гибкий вариант.



- `package-lock.json` **замораживает** точные версии всех пакетов и их подзависимостей, так что все, кто устанавливает ваш проект, получают точно такую же рабочую установку.


Почему это важно?


Если вы работаете в команде, или развертываете свой проект на сервере, или возвращаетесь к нему в будущем, вы хотите быть уверены, что он по-прежнему работает так же, как и раньше.

Если пакеты были обновлены и вы переустановили их без файла блокировки, вы можете получить немного другую версию, которая будет вести себя по-другому.


Сохраняя `package-lock.json` в вашем проекте, NPM всегда будет устанавливать именно те версии, которые там указаны, гарантируя, что у всех будет одинаковое окружение.


`package-lock.json` блокирует всё до определённой версии, чтобы сделать проект более воспроизводимым на других машинах.


Но если ваш пакет должен использоваться многими людьми, вы можете не фиксировать его, чтобы NPM находил только файл `package.json` и мог автоматически устанавливать последние версии, разрешенные в этом файле.


Но об этом вы будете беспокоиться позже, когда начнете публиковать свой собственный код. Пока же мы познакомили вас с основами NPM, чтобы вы могли находить и использовать в своих проектах библиотеки, опубликованные другими разработчиками.



## Создание сетей в NodeJS

<chapterId>f2cabd8b-754b-5c97-8d6a-8412a9a184c7</chapterId>


NodeJS часто используется как язык для бэкенда: вы можете превратить свой скрипт в сервер, а также использовать его для запросов к другим серверам.


В этой главе мы познакомимся с некоторыми базовыми сетевыми функциями, которые позволят вам это сделать.


### `fetch()`


Если вы хотите, чтобы ваша программа загружала данные с веб-сайта или API, вам нужно сделать **HTTP-запрос**.


В современных версиях NodeJS вы можете использовать встроенную функцию `fetch()`.


Вот пример выполнения GET-запроса к API:


```javascript
const response = await fetch("https://jsonplaceholder.typicode.com/posts/1")
const data = await response.text()

console.log(data)
```


Когда вы запустите эту программу, вы увидите что-то вроде:


```
{
"userId": 1,
"id": 1,
"title": "...",
"body": "..."
}
```


Вот что происходит:


1. в `fetch()` берется URL и выполняется запрос.

2. Он возвращает **Promise**, который разрешается в объект `Response`.

3. `response.text()` считывает тело ответа в виде строки.


Но строка, которую вы получите в ответ, на самом деле является JSON. Что это такое?


### JSON


При работе с веб-интерфейсами API данные часто отправляются и принимаются в виде **JSON**, что расшифровывается как JavaScript Object Notation.


JSON - это просто текстовый формат, который очень похож на объекты JavaScript. Например:


```json
{
"name": "Alice",
"age": 30,
"likes": ["apples", "bananas"]
}
```


Объект `JSON` - это встроенная в JavaScript утилита, которую можно использовать для работы с этим форматом данных.


Вы можете преобразовать объект JavaScript в строку JSON с помощью `JSON.stringify()`:


```javascript
const user = { name: "Alice", age: 30 }

const jsonString = JSON.stringify(user)

console.log(jsonString)
```


Это отпечатки:


```
{"name":"Alice","age":30}
```


Вы также можете преобразовать текст JSON обратно в объект JavaScript с помощью `JSON.parse()`:


```javascript
const jsonText = '{"name":"Bob","age":25}'

const obj = JSON.parse(jsonText)

console.log(obj)
```


Это отпечатки:


```
{ name: 'Bob', age: 25 }
```


Будьте внимательны: `JSON.parse()` выдаст ошибку, если строка не является валидным JSON.


```javascript
JSON.parse("not json") // ❌ Error!
```


Поэтому убедитесь, что строка правильно отформатирована.


### `http` сервер


NodeJS позволяет создать веб-сервер без установки чего-либо еще.


Вы можете использовать встроенный модуль `http` для обработки запросов от клиентов и отправки ответов.


Вот очень простой пример:


```javascript
const http = require("http")

const server = http.createServer((req, res) => {
res.statusCode = 200
res.end("Hello from NodeJS server!")
})

server.listen(3000, () => {
console.log("Server running at http://localhost:3000/")
})
```


Когда вы запустите этот скрипт и откроете `http://localhost:3000` в браузере, вы увидите:


```
Hello from NodeJS server!
```


Вот что происходит в коде:


1. Вы импортировали сервер `http` из стандартной библиотеки Node.

2. `http.createServer()` создает сервер. Вы передали в `http.createServer()` обратный вызов `(req, res) => {...}`, который будет выполняться каждый раз, когда кто-то подключается.

3. Вы присвоили ответу код состояния 200 (что означает успешную операцию). О кодах состояния http можно прочитать [здесь](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Status)

3. `res.end()` отправляет ответ и завершает соединение.

4. `Server.listen(3000)` запускает сервер на порту 3000.


Вы также можете указать `req.url` и `req.method` в запросе, чтобы обрабатывать различные пути или типы запросов.


Пример с маршрутизацией:


```javascript
const server = http.createServer((req, res) => {
if (req.url === "/") { // handle requests for the root of the website
res.statusCode = 200
res.end("Home page")
} else if (req.url === "/about") { // handle requests for the about page
res.statusCode = 200
res.end("About page")
} else {
res.statusCode = 404 // we send a 404 status code to signal that the requested page is missing
res.end("Not Found")
}
})
```


Это очень базовые примеры. Для создания более продвинутых серверов большинство разработчиков, скорее всего, скачают готовую библиотеку бэкенда, например [express](https://www.npmjs.com/package/express).


## Обработка данных: буферы, события, потоки

<chapterId>8c9623f0-a604-51a4-8fe4-871c849d3e3b</chapterId>


В этой главе мы познакомимся, прежде всего, с тремя классами объектов:



- `Буфер`, который представляет собой небольшие фрагменты двоичных данных
- `EventEmitter`, который может быть использован для отслеживания асинхронного процесса путем испускания сигналов, называемых "событиями"
- `Stream`, который позволяет обрабатывать большие порции данных по одному `буферу` за раз, и который отслеживает процесс, испуская события


Они очень часто встречаются в профессиональном коде NodeJS, поэтому, даже если вы не будете использовать их в своих первых проектах, полезно получить базовое представление о том, когда вам придется с ними взаимодействовать


### Буферы


В NodeJS **буфер** - это тип объекта, используемый для работы с двоичными данными.


Буфер можно рассматривать как контейнер фиксированного размера для необработанных байтов.


Вот как создать буфер из строки:


```javascript
const buf = Buffer.from("hello")
console.log(buf)
```


Это выводит что-то вроде:


```
<Buffer 68 65 6c 6c 6f>
```


Эти числа (`68`, `65`, `6c` и т.д.) являются шестнадцатеричными представлениями букв в слове `"hello"`.


Вы можете преобразовать его обратно в строку следующим образом:


```javascript
console.log(buf.toString())
```


Это отпечатки:


```
hello
```


Вы также можете создать буфер определенного размера, заполненный нулями:


```javascript
const buf = Buffer.alloc(10)
console.log(buf)
```


Это выводит что-то вроде:


```
<Buffer 00 00 00 00 00 00 00 00 00 00>
```


Вы можете записывать данные в буфер:


```javascript
buf.write("abc")
console.log(buf)
```


Вы можете получить доступ к отдельным байтам:


```javascript
console.log(buf[0]) // prints the ASCII number for 'a', which is 97
```


Буферы особенно полезны, когда вам нужно манипулировать данными на очень низком уровне.


### События


В JavaScript **событие** - это то, что происходит в вашей программе, на что вы можете отреагировать.


Например:



- завершение загрузки файла
- срабатывает таймер
- пользователь нажимает на кнопку
- сетевой запрос возвращает данные


Событие** - это просто сигнал о том, что что-то произошло, и вы можете написать код для прослушивания этих событий и реагирования на них.


В NodeJS многие объекты могут испускать события. Эти объекты называются **EventEmitters**.


Вот пример:


```javascript
const EventEmitter = require("events")

const emitter = new EventEmitter()

// Listen for an event
emitter.on("greet", () => {
console.log("Hello! An event happened.") // this will get printed when a "greet" event gets fired
})

// Emit the event
emitter.emit("greet")
```


Это отпечатки:


```
Hello! An event happened.
```


Вот что:


1. Мы создаем объект `EventEmitter`.

2. Мы указываем ему запускать обратный вызов каждый раз, когда происходит событие `"greet"`, используя `.on("greet")`.

3. Позже мы запустим событие `"приветствие"` с помощью `.emit()`.

4. Наш обратный вызов будет выполнен


Вместе с событием можно передать данные:


```javascript
emitter.on("greet",
(name) => console.log(`Hello, ${name}!`)
)

emitter.emit("greet", "Alice") // first argument is the type of event, second argument is the data we pass with this event
```


Это отпечатки:


```
Hello, Alice!
```


Вы можете зарегистрировать слушателей и для других событий:


```javascript
emitter.on("goodbye", () => {
console.log("Goodbye!")
})

emitter.emit("goodbye")
```


К одному типу событий можно прикрепить сколько угодно слушателей, а один и тот же эмиттер может вызывать множество различных типов событий.


Многие объекты в NodeJS испускают события, чтобы сообщить остальной части программы о том, что что-то происходит.



### Что такое ручьи?


Потоки объединяют буферы и события, чтобы помочь нам обрабатывать данные.


Когда мы работаем с файлами, данными из сети или даже длинным текстом, нам не всегда нужно (или хочется) загружать в память все сразу. Это может быть медленно и даже привести к сбою программы, если данные слишком велики.


Вместо этого мы можем обрабатывать данные **понемногу**, по мере их поступления или считывания, как будто пьем воду через соломинку, а не пытаемся выпить весь стакан за один раз. Это называется **поток**.


В NodeJS поток - это объект, который позволяет вам читать данные из источника или записывать их в место назначения **по одной штуке за раз**.


В NodeJS существует четыре основных типа потоков:



- Readable**: потоки, из которых можно читать данные (как при чтении файла)
- Writable**: потоки, в которые можно записывать данные (как в файл)
- Дуплекс**: потоки, которые можно и читать, и записывать
- Преобразование**: подобно дуплексным потокам, но они могут изменять (преобразовывать) данные по мере их поступления


### Читаемые потоки


Допустим, у вас есть файл `bigfile.txt`, который нужно обработать. Вы можете создать читаемый поток с помощью модуля `fs`, чтобы читать файл по частям.


```javascript
const fs = require("fs")

const readableStream = fs.createReadStream(
"bigfile.txt"
)

readableStream.on("data", (chunk) => {
console.log("Received chunk:", chunk)
})

readableStream.on("end", () => {
console.log("Finished reading file.")
})

readableStream.on("error", (err) => {
console.error("Error reading file:", err)
})
```


Что здесь происходит?


1. `fs.createReadStream()` создает поток, пригодный для чтения.

2. Всякий раз, когда часть файла готова, поток выдает событие `data` и передает нам "кусок" данных (`буфер`). Мы печатаем этот кусок.

3. Когда весь файл будет прочитан, сработает событие `end`.

4. Если произошла ошибка (например, файл не существует), срабатывает событие `error`.


Таким образом, вы можете читать огромные файлы, не загружая их все сразу в память.


Если мы хотим, чтобы данные поступали в человекочитаемом виде (а не в двоичном), мы можем указать кодировку потока:


```javascript
const fs = require("fs")

const readableStream = fs.createReadStream(
"bigfile.txt",
{ encoding: "utf8" } // we tell NodeJS that the file should be read as utf8
)

readableStream.on("data", (chunk) => {
console.log("Received chunk:", chunk)
})

readableStream.on("end", () => {
console.log("Finished reading file.")
})

readableStream.on("error", (err) => {
console.error("Error reading file:", err)
})
```


Теперь код выведет файл в человекочитаемом виде.


### Записываемые потоки


Поток с возможностью записи позволяет отправлять данные куда-либо, фрагмент за фрагментом.


Вот пример записи в файл `target.txt` с помощью потока:


```javascript
const fs = require("fs")

const stream = fs.createWriteStream("target.txt")

stream.write("First line\n")
stream.write("Second line\n")
stream.end("Finished writing\n")

stream.on("finish", () => {
console.log("All data written.")
})

stream.on("error", err => {
console.error("Error:", err)
})
```


Вот что происходит:


1. `fs.createWriteStream()` создает поток, пригодный для записи.

2. Мы запишем на него текст с помощью `.write()`.

3. Когда мы закончим, мы вызовем `.end()`, чтобы закрыть поток.

4. Когда все данные будут записаны, произойдет событие `finish`.

5. Если что-то идет не так, запускается событие `error`.


Как и потоки с возможностью чтения, потоки с возможностью записи хороши для работы с большими данными, поскольку им не нужно хранить в памяти все и сразу.


### Трубопроводные потоки


Одна из самых замечательных вещей в потоках заключается в том, что их можно **соединить в трубопровод**: соединить поток, доступный для чтения, непосредственно с потоком, доступным для записи.


```javascript
const fs = require("fs")

const readable = fs.createReadStream("bigfile.txt")
const writable = fs.createWriteStream("target.txt")

readable.pipe(writable)
```


Вот:



- Читаемый поток считывается из файла `bigfile.txt`.
- Поток с возможностью записи записывает в файл `copy.txt`.
- `.pipe()` передает данные непосредственно из читаемого в записываемый поток.


### Дуплексные потоки


Дуплексный поток можно и читать, и записывать. Примером может служить сетевой сокет: вы можете отправлять на него данные и получать их с него.


Вот очень простой пример с использованием модуля `net`:


```javascript
const net = require("net")

const server = net.createServer((socket) => {
socket.write("Welcome!\n")

socket.on("data", (chunk) => {
console.log("Received:",
chunk.toString()  // we convert the chunk of data from Buffer to string
)
})
})

server.listen(3000, () => {
console.log("Server listening on port 3000")
})
```


В данном примере:



- Объект `socket` представляет собой дуплексный поток.
- Вы можете `записывать()` в него, а также слушать события `данных` от него.


### Преобразование потоков


Поток преобразования - это дуплексный поток, который также изменяет проходящие через него данные.


Например, вы можете использовать встроенный модуль `zlib` для сжатия или распаковки данных.


Вот как сжать файл с помощью потока преобразований:


```javascript
const fs = require("fs")
const zlib = require("zlib")

const readable = fs.createReadStream("bigfile.txt")     // create a readable stream that reads from a file
const zip = zlib.createGzip()                           // create a transform stream that compresses data
const writable = fs.createWriteStream("bigfile.txt.gz") // create a writable stream that writes to a file

readable          // take the readable stream
.pipe(zip)      // pipe it into the transform stream to compress the data
.pipe(writable) // then pipe it into the writable stream that saves the data to a zipped file

writable.on("finish", () => {
console.log("File compressed.")
})
```


И распаковать его обратно:


```javascript
const readable = fs.createReadStream("bigfile.txt.gz")
const unzip = zlib.createGunzip()
const writable = fs.createWriteStream("bigfile.txt")

readable.pipe(unzip).pipe(writable)

writable.on("finish", () => {
console.log("File decompressed.")
})
```


Потоки преобразования очень полезны для таких задач, как сжатие, шифрование или изменение форматов файлов во время потоковой передачи.


### Противодавление


Иногда поток с возможностью записи медленно обрабатывает данные. Если мы продолжаем передавать данные в записываемый поток быстрее, чем он может их обработать, могут возникнуть проблемы. Это называется **обратное давление**.


Если вы вызываете метод `.write()` для потока с возможностью записи, он возвращает логическое значение, информирующее вас о том, нужна ли потоку пауза; возможно, вам придется проверить возвращаемое значение, как это сделано здесь:


```javascript
const fs = require("fs")

const readable = fs.createReadStream("example.txt")
const writable = fs.createWriteStream("copy.txt")
r
readable.on("data", chunk => {               // each chunk of data we read from the readable stream...

const canContinue = writable.write(chunk)  // ...we send it to the writable, which returns us a boolean to confirm we can continue

if (!canContinue) { readable.pause() }     // ...if we can't, we temporarily pause reading
})

writable.on("drain",                // the writable stream emits a "drain" event when the backpressure is gone

() => { readable.resume() }      // so we resume reading (and writing)

)
```


Это был наглядный пример ручной перекачки данных из считываемого устройства в записываемое и ручного управления противодавлением.


Обычно мы передаем данные с помощью метода `.pipe()`, который автоматически обрабатывает обратное давление.


Так что беспокоиться об обратном давлении нужно только в том случае, если по какой-то причине вы вручную вызываете `.write()`.
