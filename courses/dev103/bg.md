## Настройка

<chapterId>ba05a290-1782-5268-87c9-62fd09590e05</chapterId>


Програма на JavaScript е просто колекция от (един или повече) текстови файлове, които съдържат команди, изпълнявани от изпълнителното време на JavaScript.


Имената на тези текстови файлове обикновено завършват с разширение `.js`, например `my_script.js`, `my_program.js` и т.н.


Съдържащите се в тях команди са написани на езика за програмиране JavaScript.


Изпълнението на JavaScript е специална програма, която изпълнява тези файлове.


![](assets/en/001.webp)


### Изпълнението на NodeJS


Най-разпространената среда за изпълнение на JavaScript е NodeJS.


Вашият IDE може вече да го включва или да се наложи да го изтеглите от [официалния уебсайт](https://nodejs.org/en/download).


Страницата за изтегляне ще ви предостави инструкции за трите основни операционни системи (ОС): Windows, Linux и MacOS. Предполага се, че знаете как да отворите терминал във вашата операционна система.


Тъй като NodeJS е наличен и за трите операционни системи, написаните от вас програми ще могат да се изпълняват на всички тях (с изключение на някои крайни случаи).


Това означава, че можете например да напишете проста видеоигра на JavaScript на своя компютър с Windows и да я предадете на свой приятел, за да я стартира на своя Mac.


![](assets/en/002.webp)


### Първа програма (Здравей, свят)


Традиционно при изучаването на език за програмиране първата програма, която се пише, се състои в отпечатване на "hello world!" на конзолата.


Създайте директория, наречена `my_js_code/`, и в нея файл, наречен `main.js` (тези имена са произволни).


Отворете директорията с редактора на код.


Запишете този код във вашия файл:


```javascript
console.log("hello world!")
```


Отворете терминал и изпълнете тази команда, за да стартирате програмата:


```
node main.js
```


Резултатът трябва да бъде


```
hello world!
```


### Какво се случи


В JavaScript всичко е "обект".


`console` е обект, който се използва за отстраняване на грешки в програмата.


`console.log` е най-използваният метод на `console`. Той просто отпечатва всички аргументи, които му подадете.


Аргументите се предават на `console.log` с помощта на кръгли скоби `()`.


Така например, ако искате да отпечатате числото `1000`, просто ще напишете


```javascript
console.log(1000)
```


След това го изпълнете, като стартирате


```
node main.js
```


в терминала си (оттук нататък в този курс ще се предполага, че знаете, че така се изпълнява програма).


Това трябва да се отпечата


```
1000
```


Можете да предадете няколко неща, като например


```javascript
console.log(16, 8, 1993)
```


Това ще отпечата


```
16 8 1993
```


## Променливи и коментари

<chapterId>23050ab7-343b-5edf-9d37-e4e782e27ce0</chapterId>


Програмите обикновено изпълняват операции върху данни.


Променливите са като именувани кутии, които използваме за съхранение на данни. Те ни позволяват да свържем дадена част от данните с конкретно име, така че да можем да ги извлечем по-късно, използвайки това име.


### нека декларациите


За да декларираме променлива в JavaScript, можем да използваме ключовата дума `let`.


След като напишем `let`, записваме името, което искаме да дадем на променливата, след това знака `=` и след това стойността, която искаме да съхраним.


Например:


```javascript
let age = 25

console.log(age)
```


Името на променливата (наричано технически "идентификатор") обикновено може да съдържа букви, подчертавания (`_`), знака за долар (`$`) и цифри, въпреки че първият символ не може да бъде число.


В горния код декларирахме променлива, наречена `age`, и записахме в нея стойността `25`.


След това отпечатахме стойността с помощта на `console.log(age)`.


Ако стартирате този код с `node main.js`, изходът ще бъде:


```
25
```


Идентификаторите са чувствителни към големи и малки букви, което означава, че малките и големите букви се отчитат като разлики в идентификаторите, така че например


```javascript
let age = 25

let Age = 20

console.log(age)
```


ще се отпечата 25, тъй като това са две напълно отделни променливи!


Можете също така да съхранявате низове (текст) в променлива:


```javascript
let message = "hello again"

console.log(message)
```


Това ще доведе до отпечатване на:


```
hello again
```


Както и преди, използвахме `console.log()`, за да отпечатаме стойността, записана в променливата.


Сега нека направим и двете заедно:


```javascript
let age = 25

let message = "hello again"

console.log(age)

console.log(message)
```


Изпълнението на тази стъпка ще доведе до отпечатване на:


```
25
hello again
```

### Пренасочване


Променливите, декларирани с `let`, могат да бъдат променяни след създаването им.


Това се нарича преназначаване.


```javascript
let score = 10

console.log(score)

score = 15

console.log(score)
```


Първо, присвояваме `10` на `score`, след което го отпечатваме.


След това променяме стойността на `score` на `15` и я отпечатваме отново.


Резултатът ще бъде:


```
10
15
```


Това е много полезно, когато стойността се променя с течение на времето, например в игра, в която резултатът се увеличава.


Нека добавим още една променлива към сместа:


```javascript
let score = 10
let player = "Alice"

console.log(score)
console.log(player)

score = 20
player = "Bob"

console.log(score)
console.log(player)
```


Това ще доведе до отпечатване на:


```
10
Alice
20
Bob
```


Както можете да видите, и `score`, и `player` са променени.


### const декларации


В повечето случаи обаче не искаме променливата да се променя, след като е създадена. За тази цел използваме `const`.


`const` е съкращение от "константа". След като присвоите стойност на променлива `const`, не можете да я промените.


```javascript
const pi = 3.14
console.log(pi)
```


Този печат:


```
3.14
```


Но ако се опитате да го направите:


```javascript
const pi = 3.14
console.log(pi)

pi = 99 // this line will cause an error
console.log(pi)
```


JavaScript ще ви даде грешка като:


```
TypeError: Assignment to constant variable.
```


Това е така, защото `pi` е деклариран с помощта на `const` и не можете да промените стойността му след това. По този начин съобщавате на JavaScript интерпретатора, че не искате тази променлива да се променя.


Това е полезно, защото намалява вероятността да го промените по погрешка. Когато програмите станат много големи, с хиляди редове код, е невъзможно да се следи всичко, което се случва наведнъж (това е основната причина да използваме компютри, за да изпълняваме сложни процеси, които не можем да изчислим с мозъка си), така че става полезно да има ограничения като това, които правят програмата по-детерминирана.


Счита се за най-добра практика винаги да декларираме стойностите си като `const`, освен ако не сме сигурни, че искаме да ги променим по-късно.


### Коментари в JavaScript


Понякога искаме да напишем бележки в нашия код, които не се изпълняват. Те се наричат коментари.


Коментарите се игнорират от програмата, когато тя работи, но са полезни за обяснение на нещата на нас или на други хора.


За да напишете коментар от един ред, използвайте `//`


```javascript
// This is a comment
const x = 10 // This is also a comment
console.log(x)
```


Това все пак ще се отпечата:


```
10
```


Коментарите са просто за четене от хората.


Можете също така да пишете многоредови коментари с помощта на `/*` и `*/`


```javascript
/*
This is a multi-line comment.
It can span several lines.
*/
const y = 20
console.log(y)
```


Това ще отпечата


```
20
```


И коментарът ще бъде игнориран.


Можете да използвате коментарите, за да добавите малки анотации към кода си, така че да запомните какво прави и защо е написан по определен начин. Това може да помогне и на други програмисти да го разберат.


## Основни типове: числа, низове, булеви стойности

<chapterId>cfdb04f6-21a8-5143-bbf9-7aaae04962f0</chapterId>


В JavaScript "типът" показва какъв вид данни представлява дадена стойност.


Javascript има няколко основни типа и в този раздел ще разгледаме някои от тях.


### Числа и аритметични операции


Първият тип, който ще въведем, е `number`.


Числата в JavaScript могат да бъдат цели (като `5`) или десетични (като `3.14`).


С тях можете да правите аритметични действия: събиране, изваждане, умножение и деление.


Ето един основен пример:


```javascript
const a = 10
const b = 5

const sum = a + b
const difference = a - b
const product = a * b
const quotient = a / b

console.log(sum)
console.log(difference)
console.log(product)
console.log(quotient)
```


Това ще доведе до отпечатване на:


```
15
5
50
2
```


Можете също така да използвате скоби `()`, за да контролирате реда на операциите:


```javascript
const result = (2 + 3) * 4
console.log(result)
```


Този печат:


```
20
```


Без скобите то би било `2 + 3 * 4`, което е:


```javascript
const result = 2 + 3 * 4
console.log(result)
```


Това ще се отпечата:


```
14
```


Защото в обикновената математика умножението се извършва преди събирането.


### Низове и интерполация


Вторият тип JavaScript, който ще въведем, е `string`.


Низовете са части от текст. Можете да използвате единични кавички `'...'` или двойни кавички `"..."`, за да ги създадете.


```javascript
const greeting = "hello"
const name = 'Bob'
console.log(greeting)
console.log(name)
```


Този печат:


```
hello
Bob
```


За комбиниране на низове можете да използвате оператора `+`:


```javascript
const greeting = "hello"
const space = " "
const name = "Bob"

const fullGreeting = greeting + space + name
console.log(fullGreeting)
```


Това ще доведе до отпечатване на:


```
hello Bob
```


Но има и по-хубав начин за комбиниране на низове, наречен **интерполация на низове**. Използвате задни тирета, за да декларирате низ `` `...``` и записвате променливи с помощта на `${...}` вътре в низа:


```javascript
const greeting = "hello"
const name = "Bob"

const fullGreeting = `${greeting} ${name}`
console.log(fullGreeting)
```


Това също се отпечатва:


```
hello Bob
```


Можете да включите всеки израз в `${...}`:


```javascript
const age = 30
console.log(`Next year, I will be ${age + 1} years old.`)
```


Този печат:


```
Next year, I will be 31 years old.
```


Интерполацията е много разпространена в съвременния JavaScript.


### Булеви знаци, сравнение и логически операции


Третият тип, който ще въведем, е `boolean`. Той е наречен на името на математика Джордж Бул, който е изобретил булевата логика.


Булевите стойности са прости: само две възможни стойности - `true` и `false`.


Можете да ги съхранявате в променливи:


```javascript
const theSkyIsBlue = true
const thisCourseIsBad = false

console.log(theSkyIsBlue)
console.log(thisCourseIsBad)
```


Този печат:


```
true
false
```


Можете да комбинирате булеви стойности с помощта на логически оператори:



- `&&` означава "и" и ще върне `истина` само ако и двете** стойности са `истина`, в противен случай ще върне `невярно`
- `||` означава "или" и ще върне `истина`, ако **поне една** от стойностите е `истина`, в противен случай (ако и двете са неверни) ще върне `невярно`
- `!` означава "не", прилага се пред булево число и го обръща: ако булевото число е `истина`, то ще върне `невярно`, и обратно.


![](assets/en/003.webp)


Примери:


```javascript
const isSunny = true
const isWarm = true

console.log(isSunny && isWarm)  // true
console.log(isSunny || isWarm)  // true
console.log(!isSunny)           // false
```


Можете да сравнявате стойности в JavaScript, като използвате оператори като `>`, `<`, `===` и `!==`. Резултатът от тези сравнения винаги е булева стойност.


```javascript
const first = 10
const second = 5

const firstIsGreater   = (a > b)
const secondIsGreater  = (a < b)
const theyAreEqual     = (a === b)
const theyAreDifferent = (a !== b)

console.log(firstIsGreater)   // true
console.log(secondIsGreater)   // false
console.log(theyAreEqual)  // false
console.log(theyAreDifferent)  // true
```


В Javascript също така има `>=` за означаване на "по-голям или равен" и `<=` за означаване на "по-малък или равен".


Булевите, операторите за сравнение и логическите оператори често се комбинират в програмите за деклариране на сложни условия, като например да се гарантира, че "имейлът е пристигнал И съдържа изображението, от което се нуждая, ИЛИ дължината на имейла е по-голяма от 10000 символа". По-късно ще разберете, че това са основни градивни елементи за изграждане на логиката на програмата.


## Масиви, нула, неопределено

<chapterId>7bf18183-5eae-53ed-83d2-b04982145d81</chapterId>


В този раздел ще разгледаме още три типа, които се срещат много често в програмите на JavaScript:


**Масиви**: последователности от стойности

**неопределено**: специална стойност, която означава "нищо не е зададено"

**null**: друга специална стойност, която означава "умишлено празен"


### Масиви и достъп до индекси


**Масив** е тип, който може да съдържа множество стойности в списък.


Създавате масив, като използвате квадратни скоби `[]` и разделяте елементите със запетаи.


Ето един основен пример:


```javascript
const numbers = [10, 2, 88]
console.log(numbers)
```


Този печат:


```
[ 10, 2, 88 ]
```


В масив можете да съхранявате всичко, не само числа:


```javascript
const things = ["apple", 42, true]
console.log(things)
```


Този печат:


```
[ 'apple', 42, true ]
```


За достъп до конкретен елемент в масива се използва **индекс**. Индексът е позицията на елемента, като се започне от **0**.


Така че в този масив:


```javascript
const colors = ["red", "green", "blue"]
```


`colors[0]` е `"червено"`

`colors[1]` е `"зелен"`

`цветове[2]` е `"синьо"`


Да опитаме:


```javascript
const colors = ["red", "green", "blue"]
console.log(colors[0])
console.log(colors[1])
console.log(colors[2])
```


Това ще доведе до отпечатване на:


```
red
green
blue
```


Можете да присвоите стойност към определен индекс на масив


```javascript
const colors = ["red", "green", "blue"]

colors[1] = "yellow"

console.log(colors)
```


Това ще доведе до отпечатване на:


```
[ 'red', 'yellow', 'blue' ]
```


Можете да използвате всяко естествено число като индекс, дори такова, което се съхранява в променлива


```javascript

const i = 1
const colors = ["red", "green", "blue"]
console.log(colors[i])
```


Това ще доведе до отпечатване на:


```
green
```


Но ако се опитате да осъществите достъп до несъществуващ индекс, ще получите резултат `undefined`:


```javascript
const colors = ["red", "green", "blue"]
console.log(colors[3])
```


Този печат:


```
undefined
```


Какво е това??


### неопределен


Специалната стойност `undefined` означава "не е зададена стойност".


Ако създадете променлива, но не й зададете стойност, тя ще бъде `неопределена`:


```javascript
const name
console.log(name)
```


Този печат:


```
undefined
```


Тъй като не сме присвоили нищо на `name`, JavaScript го задава като `undefined` по подразбиране.


Както вече видяхте, можете да получите `undefined`, когато получите достъп до несъществуващ индекс на масив:


```javascript
const fruits = ["banana", "apple"]
console.log(fruits[2]) // There is no index 2
```


Този печат:


```
undefined
```


### null и как да го лекуваме


`null` също е специална стойност. Тя означава "тук няма нищо и аз го направих нарочно"


За разлика от `undefined`, което е автоматично, `null` е нещо, което сами задавате.


Например:


```javascript
const currentUser = null
console.log(currentUser)
```


Този печат:


```
null
```


Защо да използвате `null`? Може би очаквате стойност по-късно, но тя все още не е готова:


```javascript
let winner = null

// Later in the program:
winner = "Alice"

console.log(winner)
```


Този печат:


```
Alice
```


Така че `null` е полезен, когато искате да кажете например: "По-късно тук трябва да има нещо, но сега е празно."


## Блокове и поток на управление

<chapterId>be985168-2636-5b0d-a48f-ac1bbfbff8a7</chapterId>


Досега сме писали предимно редове код, които се изпълняват един след друг.


Но когато програмираме, можем да контролираме реда на изпълнение.


Това се нарича **контролен поток**.


Нека започнем с разбирането на блоковете и обхвата.


### Глобален обхват


Всяка променлива, която декларираме, съществува в **scope**, което означава областта от кода, в която променливата е известна.


Ако декларирате променлива извън някой блок, тя съществува в **глобалния обхват**.


```javascript
const color = "blue"
console.log(color)
```


Тази променлива `color` е в глобалния обхват, така че може да бъде достъпна от всяка точка на файла.


Ако добавите повече редове:


```javascript
const color = "blue"
console.log(color)

const size = "large"
console.log(color)
console.log(size)
```


Както `color`, така и `size` са глобални променливи. Те са достъпни навсякъде във файла.


Но какво се случва вътре в блока?


### Блокове и местен обхват


**Блок** е част от кода, заобиколена от къдрави скоби `{}`.


Променливите, декларирани с `let` или `const` в блок, съществуват **само** в този блок.


```javascript
{
const message = "inside block"
console.log(message)
}
```


Този печат:


```
inside block
```


Но ако опитате това:


```javascript
{
const message = "inside block"
}
console.log(message) // Error!
```


JavaScript ще ви даде грешка като:


```
ReferenceError: message is not defined
```


Това е така, защото `message` е декларирано вътре в блока и не съществува извън него.


Това означава, че можем да използваме блокове, за да изолираме части от нашия код и да сме сигурни, че "каквото се случва в блока, остава в блока" (подобно на Лас Вегас).


Организирането на нашия код в блокове ни позволява да структурираме изпълнението на програмата с помощта на конструкции за контрол като `if`


### if, else


Понякога искаме да стартираме код **само ако** нещо е вярно. За това служи операторът `if`.


```javascript
const myAge = 20

console.log("Am I an adult?")

if (myAge >= 18) {
console.log("Yes I am!")
}
```


Този печат:


```
Am I an adult?
Yes I am!
```


Както можете да видите, кодът сравнява `myAge` и `18`.

В този случай операторът `>=` връща `true`, така че блокът се изпълнява.

Ако условието не е `true`, блокът не се изпълнява.


```javascript
const myAge = 17

console.log("Am I an adult?")

if (myAge >= 18) {
console.log("Yes I am!")
}
```


Този печат:


```
Am I an adult?
```


Можете да добавите блок `else`, за да се справите с обратния случай:


```javascript
const myAge = 17

console.log("Am I an adult?")

if (myAge >= 18) {
console.log("Yes I am!")
} else {
console.log("No, I am not.")
}
```


Този печат:


```
Am I an adult?
No, I am not.
```


Блоковете `if` и `else` все още са блокове - така че променливите, декларирани в тях, не съществуват извън тях.


Ако искаме да сме сигурни, че нещо е **не** вярно, какво можем да направим?


Както вече споменахме, JavaScript има оператор "not", който обръща булевите стойности. Така че можем да направим


```javascript
const myAge = 17

const adult = myAge >= 18

console.log("Am I an adult?")

if (!adult) {
console.log("No, I am not.")
}
```

Това все още се отпечатва:


```
Am I an adult?
No, I am not.
```

Защото използвахме оператора `!`, за да инвертираме променливата `adult`.


`if (!adult) {...}` трябва да се чете като "if not adult..."


С помощта на блокове, логически оператори и оператори за сравнение можем да структурираме изпълнението на програмата, като дефинираме променливи, които трябва да бъдат "верни" (или "грешни"), за да се случи нещо.


### while, break, continue


Цикълът `while` повтаря код, докато дадено условие е вярно.


```javascript
let count = 0

while (count < 3) {
console.log("Count is", count)
count = count + 1
}
console.log("the loop is over!")
```


Този печат:


```
Count is 0
Count is 1
Count is 2
the loop is over!
```


Когато `count` стане 3, цикълът спира.


Можете да спрете цикъла по-рано с помощта на `break`:


```javascript
let number = 1 // Start with number 1

while (true) { // This condition is always true, so this loop will run forever unless we stop it
console.log(number) // Print the current number
if (number === 3) { // If the number is 3, stop the loop
break
}
number = number + 1 // Add 1 to the number
}
```


Този печат:


```
0
1
2
```


Защото когато числото стане `3`, блокът `if` се изпълнява и спира цикъла.


Можете да прескочите останалата част от цикъла с помощта на `continue`:


```javascript
let number = 0 // Start with number 0

while (number < 5) { // Keep going while number is less than 5
number = number + 1 // Add 1 to the number

if (number === 3) { // If the number is 3
continue // Skip the rest of the block and go to the next iteration of the loop
}

console.log(number) // Print the number
}
```


Този печат:


```
1
2
4
5
```


Защото когато числото беше `3`, `continue` накара програмата да прескочи реда, който отпечатва числото.


### за ... от ...


Ако имате масив и искате да направите нещо с всеки елемент в него, можете да използвате `for ... of ... {...}`.


```javascript
const fruits = ["apple", "banana", "cherry"]

for (const fruit of fruits) {
console.log(fruit)
}
```


Този печат:


```
apple
banana
cherry
```

Блокът ще се изпълни веднъж за всеки елемент от масива.


`fruit` тук е нова променлива, която приема стойността на всеки елемент от масива, за да оперира с него вътре в блока.


### за ... в ...


Можете да използвате `for ... in`, за да преминете в цикъл през ключовете (индексите) на масив:


```javascript
const fruits = ["apple", "banana", "cherry"]

for (const index in fruits) {
console.log(index)
}
```


Този печат:


```
0
1
2
```


Можете да използвате и индекса, за да получите стойността:


```javascript
const fruits = ["apple", "banana", "cherry"]

for (const index in fruits) {
console.log(fruits[index])
}
```


Отпечатва се същото като `for ... of`:


```
apple
banana
cherry
```


На практика за масиви трябва да предпочитате да използвате `for ... of`, тъй като това е по-просто и по-чисто.


### Ограничени контури


Понякога искаме да зациклим определен брой пъти или като цяло да напишем част от кода, която повтаря даден блок, докато следи нещо.

Именно за това е полезен ограничен цикъл `for`.

Ограниченият цикъл обикновено приема три условия, разделени с точка и запетая `;`, както в `(... ; ... ; ....)`.


```javascript
for (let i = 0; i < 3; i = i + 1) {
console.log(i)
}
```


Този печат:


```
0
1
2
```


Нека го обясним:



- `let i = 0`: декларира променлива, която ще се използва в блока (в този случай това е брояч, който започва от 0)
- `i < 3`: декларира условие, което трябва да бъде `истина`, за да се изпълни блокът (в този случай е "повтори, докато `i` е по-малко от 3")
- `i = i + 1`: декларира някакъв код, който да се изпълнява след всяко изпълнение на блока (в този случай "увеличи `i` с 1")


Както виждате, ограничените цикли ни позволяват да декларираме по-сложни условия за повторното изпълнение на дадена част от кода, но в повечето случаи това не е необходимо.


### Етикети за блокове


Ако трябва да напишете по-сложен поток на управление, JavaScript ви позволява да давате имена на блокове с помощта на **етикет**, който може да се използва от `break` или `continue` за определяне на мястото, където да се върнете.


Пример:


```javascript
outer: {
console.log("We're inside the outer scope.")

inner: {
console.log("We're inside the inner scope.")
break outer
}

console.log("This will not run")
}

console.log("Done")
```


Този печат:


```
Inside outer block
Inside inner block
Done
```


Използвахме `break outer`, за да излезем изцяло от блока `outer`.


Можете също така да поставяте етикети на цикли. Да вземем този пример:


```javascript
// Declare a variable to count the total number of days in a year
let totalDaysInOneYear = 0

// Declare one variable per month, with the number of the month
const january = 1
const february = 2
const march = 3
const april = 4
const may = 5
const june = 6
const july = 7
const august = 8
const september = 9
const october = 10
const november = 11
const december = 12

// Declare an array that holds the months that have 30 days
const monthsWith30Days = [
april, june, september, november
]

// Declare variables to keep track of the month and day we're in
let currentMonth = january
let currentDay = 1

monthsLoop: while (true) {  // Start a loop labeled "monthsLoop" to process each month

daysLoop: while (true) {  // Start a loop labeled "daysLoop" to process each day in the month
totalDaysInOneYear = totalDaysInOneYear + 1  // Increase the total number of days we counted by 1

if (                                                                   // We want to check if we're at the end of the month.
currentDay === 31                                                  // Check if the current day is 31 (for months with 31 days)...
|| currentDay === 30 && (monthsWith30Days.includes(currentMonth))  // ...or 30 if it's among the 30-days months...
|| currentDay === 28 && (currentMonth === february)                // ...or 28 if it's February. If it's any of these three, then:

){
currentMonth = currentMonth + 1  // Move to the next month
currentDay = 1                   // Reset the day to 1 for the new month
break daysLoop                   // Exit the inner loop (which tracks days) and go back to the outer loop (which tracks months)
}
else { currentDay = currentDay + 1 }                                   // Otherwise, we're not at the end of the month, and we just move to the next day

}
if (currentMonth > 12) {  // After processing a month, check if we've gone past December
break monthsLoop  // If so, break the outer loop and stop the day-counting process
}
}

console.log(totalDaysInOneYear)  // Print the total number of days in the year (should be 365)
```

Това беше много скучен пример, но се надявам, че е изяснил (понякога) необходимостта от етикети.


## Въвеждане на функции

<chapterId>cc324715-09c2-5cf7-9e6f-47a6f16bc04d</chapterId>


С разрастването на програмите си често ще искате да **използвате повторно** части от кода.


Именно за това служат **функциите**: те ви позволяват да групирате определен код, да му дадете име и да го стартирате, когато пожелаете.


### Деклариране на функции


За да декларираме функция, можем да използваме ключовата дума `function`. След това й даваме име, двойка скоби `()` с аргументите, които приема, и блок от код `{}`, който трябва да се изпълни. Нека да започнем с функция, която не приема аргументи:


```javascript
function sayHello () {console.log(`Hello!`) }
```


Този код **декларира** функцията, но все още **не я изпълнява**.


### Функционални повиквания


За да стартирате (или да извикате) функцията, трябва да напишете нейното име, последвано от скоби:


```javascript
function sayHello () {console.log(`Hello!`) }
sayHello()
```


Този печат:


```
Hello!
```


Можете да извикате функцията толкова пъти, колкото искате:


```javascript
function sayHello() {
console.log("Hello!")
}

sayHello()
sayHello()
```


Този печат:


```
Hello!
Hello!
```


Кодът във функцията се изпълнява само когато я извикате.


### Аргументи на функциите (входни данни за функциите)


Понякога искате дадена функция да работи с някакъв вход. Можете да направите това, като добавите **аргументи** в скобите.


Например:


```javascript
function sayHelloTo (friend) {
console.log(`Hello ${friend}!`)
}
```


Сега тази функция приема **един аргумент**, наречен `приятел`.


Когато извиквате функцията, можете да подадете стойност:


```javascript
sayHelloTo("Tommy")
```


Този печат:


```
Hello Tommy!
```


Можете да извикате функцията отново с друго име:


```javascript
sayHelloTo("Sam")
```


Този печат:


```
Hello Sam!
```


Предадената от вас стойност замества променливата `friend` във функцията.


Можете да използвате и повече от един аргумент:


```javascript
function greetTwoPeople(person1, person2) {
console.log(`Hello ${person1} and ${person2}!`)
}

greetTwoPeople("Lina", "Marco")
```


Този печат:


```
Hello Lina and Marco!
```


### return (изход от функциите)


Функциите могат също така да **връщат** стойности. Това означава, че те изпращат стойност обратно към мястото, където е била извикана функцията.


Ето един прост пример:


```javascript
function getNumber() {
return 42
}

const result = getNumber()
console.log(result)
```


Този печат:


```
42
```


Функцията `getNumber()` връща `42` и ние го съхраняваме в `result`, след което го отпечатваме.


Можете също така да върнете нещо, което сте изчислили:


```javascript
function add(a, b) {
return a + b
}

const result = add(2, 3)
console.log(result)
```


Този печат:


```
5
```


След като бъде върната стойност, функцията се прекратява. Всичко след `return` в този блок не се случва.


```javascript
function saySomething() {

return "hi"

console.log("this never runs")

}

const message = saySomething()
console.log(message)
```


Отпечатва се само:


```
hi
```


защото се връща само `"hi"`. Редът `console.log("this never runs")` се пропуска.


Можете да мислите за функциите като за малки подпрограми. Всяка функция може да приеме някакъв вход, да го обработи и да ви върне някакъв изход.


Какво се случва, ако се опитаме да използваме върнатата стойност на функция, но тя не е върнала нищо?


```javascript
function doesNothing () {}

const x = doesNothing()

console.log(x)
```

Това ще изведе `неопределено`. Върнатата стойност на функция, която не е върнала нищо, е `undefined`.


## Обекти и класове

<chapterId>26689f25-8212-5057-8c21-3a05eee0ac75</chapterId>


JavaScript често се нарича обектно-ориентиран език.


Това означава, че той ви помага да организирате кода си, като групира стойности и функции в **обекти**.


### Какво е обект?


Обектът може да съдържа данни и функции, които оперират с тези данни. Когато една функция е вложена в обект, казваме, че тя е `метод`.


Първият обект, който видяхме, беше обектът `console`. Това е обект, който съдържа множество методи за отпечатване на нещо на екрана и за отстраняване на грешки в програмите ни.


Той дори може да се отпечатва сам; можете да направите


```javascript
console.log(console)
```


и ще отпечата списък с методите, които съдържа. Например, на моята машина той извежда


```javascript
Object [console] {
log: [Function: log],
warn: [Function: warn],
error: [Function: error],
dir: [Function: dir],
time: [Function: time],
timeEnd: [Function: timeEnd],
timeLog: [Function: timeLog],
trace: [Function: trace],
assert: [Function: assert],
clear: [Function: clear],
count: [Function: count],
countReset: [Function: countReset],
group: [Function: group],
groupEnd: [Function: groupEnd],
table: [Function: table],
debug: [Function: debug],
info: [Function: info],
dirxml: [Function: dirxml],
groupCollapsed: [Function: groupCollapsed],
Console: [Function: Console],
profile: [Function: profile],
profileEnd: [Function: profileEnd],
timeStamp: [Function: timeStamp],
context: [Function: context],
createTask: [Function: createTask]
}
```


Както можете да видите, той има много методи, които можете да използвате за отстраняване на грешки!


Javascript ни предоставя различни начини за създаване на нови обекти, които могат да правят каквото искаме.


### Създаване на обект


Най-лесният начин да създадете обект е като групирате данни и функции с помощта на **къдрави скоби** `{}`.


Така се създава така нареченият **анонимен обект**


```javascript
const cat = {
name: "Whiskers",
age: 3
}
```


Това създава обект и го съхранява в променлива, наречена `cat`.


Обектът има две **свойства**:



- `name` със стойност `"Whiskers"`
- `възраст` със стойност `3`


Нека го отпечатаме:


```javascript
console.log(cat)
```


Този печат:


```
{ name: 'Whiskers', age: 3 }
```


Можете да получите свойствата от обекта, като използвате точка, както в `objectName.propertyName`:


```javascript
console.log(cat.name)  // prints "Whiskers"
console.log(cat.age)   // prints 3
```


Можете също така да **промените** даден обект:


```javascript
cat.age = 4
console.log(cat.age)  // now it prints 4
```


Както виждате, дори ако даден обект е дефиниран като `const`, все още можете да променяте данните, които той съдържа.


В случай на обекти, `const` ще ви попречи да пренаредите целия обект:


```javascript
const cat = {
name: "Whiskers",
age: 3
}

cat.age = 5 // this works

cat = 5 // this throws an error, you're trying to reassign the whole object

```


Както вече споменахме, обектите могат да съдържат и **функции**, а когато една функция е част от обект, я наричаме **метод**.


Ето един пример:


```javascript
const cat = {
name: "Whiskers",
speak () {
console.log("Meow!")
}
}
```


Този обект има:



- Свойство `name`
- Метод `speak()`


Нека извикаме метода:


```javascript
cat.speak()
```


Той отпечатва:


```
Meow!
```


Методите могат да използват данните, които обектът съдържа, чрез ключовата дума `this`.

`this` се отнася за текущия обект. В този пример той ще бъде използван за отпечатване на името на котката:


```javascript
const cat = {
name: "Whiskers",
speak () {
console.log(`${this.name} says meow!`)
}
}

cat.speak()
```


Този печат:


```
Whiskers says meow!
```


Думата `this` означава "този обект"... в този случай обектът `cat`.



Този вид предмети са чудесни, когато искате нещо бързо и просто. Но ако трябва да създадете **много обекти** с една и съща структура, има по-добър начин и тук се появяват **класовете**.


### Класове и конструктори


**Класът** е като проект. Той указва на JavaScript как да създаде определен вид обект.


Дефинирането на клас се извършва с ключовата дума `class`, последвана от името на класа и от блок с къдрави скоби `{}`.


```javascript
class Dog {}
```


Класовете обикновено започват с главна буква по традиция.


Можете да създадете нов обект от даден клас с помощта на `new`:


```javascript
const hachiko = new Dog()
```


Опитайте се да отпечатате обекта:

```javascript
class Dog {}

const myDog = new Dog()

console.log(myDog)
```


Ще получите


```
Dog {}
```


Както виждате, класът Dog е празен, така че и обектът `myDog` е празен.


Можем да определим кои свойства трябва да съдържат обектите Dog, като добавим `конструктор`.


Конструкторът е специална функция, която се изпълнява при създаването (или "конструирането") на нов обект.


```javascript
class Dog {
constructor() { }
}
```


Искаме всяко куче да има име, затова добавяме параметър `име` към функцията:


```javascript
class Dog {
constructor(name) { }
}
```


И след това използваме `this`, за да декларираме, че `name` е `името` на обекта `Dog`, който изграждаме


```javascript
class Dog {
constructor(name) {
this.name = name
}}
```


Нека се опитаме да го използваме сега:


```javascript
class Dog {
constructor(name) {
this.name = name
}
}

const myDog = new Dog("hachiko")

console.log(myDog)
```


Отпечатва се нещо подобно на:


```
Dog { name: 'hachiko' }
```


Както виждате, методът `constructor` приема аргументите, които предавате на класа, когато правите `new Dog()`, и ги използва, за да изгради обекта.


Нека да го разделим:


`class Dog` дефинира класа Dog.


- `constructor(name)` задава обекта при създаването му.
- `this.name = name` съхранява стойността в новия обект.
- `new Dog("hachiko")` създава нов обект от класа със свойството `name`, зададено на `"hachiko"`.


Сега нека да добавим метод към нашия клас:


```javascript
class Dog {
constructor(name) {
this.name = name
}
speak () {
console.log(`${this.name} says barf!`)
}

}

const myDog = new Dog("hachiko")

myDog.speak()
```


Това ще отпечата


```javascript
hachiko says barf!
```


Ако направим същото за два различни екземпляра на Dog



```javascript
class Dog {
constructor(name) {
this.name = name
}
speak () {
console.log(`${this.name} says barf!`)
}

}

const myDog = new Dog("hachiko")

myDog.speak()

const yourDog = new Dog("bobby")

yourDog.speak()
```


получаваме


```
hachiko says barf!
bobby says barf!
```


методът `speak()` използва свойството `name` на кучето `Dog`, на което е извикан.


Това е основната причина за съществуването на класовете: те ни позволяват да дефинираме набор от методи, които оперират с данни, и след това да създаваме множество обекти, които споделят една и съща "форма" на данните.


Когато извикаме метод на някой от тези обекти, той ще работи с данните, които *съответният обект* притежава.


### Промяна на формата на обект


Обектите в JavaScript са гъвкави. Дори след като сте създали такъв обект, можете да добавяте нови свойства или да премахвате съществуващи.


Това е позволено, но трябва да се използва внимателно.


Нека да започнем с нашия прост клас `Dog`:


```javascript
class Dog {
constructor(name) {
this.name = name
}

speak() {
console.log(`${this.name} says barf!`)
}
}

const myDog = new Dog("Fido")
```


В този момент `myDog` има само едно свойство: `име`. Можем да добавяме нови свойства и след създаването му:


```javascript
myDog.age = 5

console.log(myDog.age) // prints 5
```


Можем да добавим и нов метод:


```javascript
myDog.jump = function () {
console.log(`${this.name} jumps!`)
}

myDog.jump() // Fido jumps!
```


Можем също така да премахваме свойства, като използваме ключовата дума `delete`.


```javascript
delete myDog.name

console.log(myDog.name) // prints 'undefined'
```


Това работи, но ето нещо важно, което трябва да знаете: Двигателите на JavaScript, като например V8 (използван в Node.js и в браузъра Chrome), работят по-бързо, когато обектите ви винаги поддържат едни и същи свойства. Ако добавяте или премахвате свойства след създаването на обекта, това може да забави работата.


В малките програми това няма голямо значение. Но в по-големи проекти (като игри) е по-добре да изброите всички свойства в конструктора от самото начало, дори и да не ги използвате веднага. Това поддържа формата на обекта стабилна и помага на кода ви да работи по-бързо.


Например, вместо това:


```javascript
class Dog {
constructor(name) {
this.name = name
}
}

const dog = new Dog("Rex")
dog.age = 4
dog.breed = "Labrador"
```


Можете да направите


```javascript
class Dog {
constructor(name, age, breed) {
this.name = name
this.age = age
this.breed = breed
}
}

const dog = new Dog("Rex", 4, "Labrador")
```


И двете версии работят, но втората е по-добра от гледна точка на производителността. Предварително казвате на енджина кои свойства ще има всеки обект и той може да оптимизира съответно.


JavaScript позволява свободно да променяте формата на обектите, но когато използвате класове, най-добре е предварително да планирате формата на обекта.



### Наследяване с extends и super()


Понякога искате да създадете клас, който е почти същият като друг клас, но с няколко допълнителни функции.


Вместо да променяте формата на обектите (което, както вече споменахме, не е оптимално за производителността) или да се налага да пишете нов клас от нулата, JavaScript ви позволява да използвате нещо, наречено **наследяване**.


Наследяването означава, че един клас може да **разширява** друг. Новият клас получава всички свойства и методи на стария, а вие можете да добавите още или да промените това, което ви е необходимо.


Да речем, че имаме базов клас, наречен `Vehicle`:


```javascript
class Vehicle {
constructor(brand) {
this.brand = brand
}

start() {
console.log(`${this.brand} vehicle is starting...`)
}
}
```


Сега искаме да създадем клас `Car`. Колата е вид превозно средство, но може да искаме да има някои допълнителни функции или различно съобщение, когато се запали. Вместо да пренаписваме всичко, можем да използваме `extends`:


```javascript
class Car extends Vehicle {
start() {
console.log(`${this.brand} car is ready to drive!`)
}
}
```


Класът `Car` вече **наследява** всичко от `Vehicle`. Той получава свойството `марка` и сме заменили метода `старт()` с наша собствена версия.


![](assets/en/004.webp)


Нека го изпробваме:


```javascript
const myCar = new Car("Toyota")
myCar.start()
```


Този печат:


```
Toyota car is ready to drive!
```


Въпреки че `Car` няма собствен конструктор, той все пак използва този от `Vehicle`. Но ако искаме да напишем собствен конструктор в `Car`, можем да го направим, просто трябва да включим извикване на конструктора на неговия родител, като използваме `super()`.


Ето как:


```javascript
class Vehicle {
constructor(brand) {
this.brand = brand
}

start() {
console.log(`${this.brand} vehicle is starting...`)
}

}

class Car extends Vehicle {
constructor(brand, model) {
super(brand) // call the parent constructor and passes the brand argument to it
this.model = model
}

start() {
console.log(`${this.brand} ${this.model} is ready to drive!`)
}
}

const myCar = new Car("Toyota", "Corolla")
myCar.start()
```


![](assets/en/005.webp)



Този печат:


```
Toyota Corolla is ready to drive!
```


И така, за да обобщим



- `Разширява` означава, че един клас се основава на друг.
- `super()` се използва за извикване на конструктора на класа, който разширявате.

*Новият клас получава всички свойства и методи на оригиналния клас.*


- Можете да **превземете** методи (като `start()`), за да ги накарате да правят нещо различно.


Това е полезно, когато имате няколко сходни неща (например автомобили, камиони и велосипеди) и искате те да споделят общ код, но да се държат по свой собствен начин.


### инстанция на


Ключовата дума `instanceof` проверява дали даден обект е създаден от определен клас.


Да речем, че имаме клас, наречен `User`:


```javascript
class User {
constructor(username) {
this.username = username
}
}

const regularUser = new User("julia123")

console.log(regularUser instanceof User)
```


Този печат:


```
true
```


Потвърждаване, че `regularUser` е `User`. Това е така, защото `regularUser` е създаден с помощта на класа `User`.


Работи и с **наследени** класове. Например, ето един клас `Admin`, който разширява класа `User`:


```javascript
class Admin extends User {}

const ourAdmin = new Admin("admin42")

console.log(ourAdmin instanceof Admin)   // true
console.log(ourAdmin instanceof User)    // true
```


И двата реда връщат `true`. Това е така, защото `Admin` е подклас на `User`, следователно `ourAdmin` е едновременно `Admin` и `User`

## Обработка на грешки

<chapterId>d0206bc5-d386-5e7f-9917-5803f392448c</chapterId>


Когато пишете по-сложни програми на JavaScript, ще се сблъскате с **грешки**. Това са неочаквани ситуации, в които нещо се обърква. Може би някоя променлива е `неопределена`, но вие се опитвате да я използвате, или някой код получава грешен тип входни данни.


Ако не се справим правилно с тези грешки, програмата ни може да се срине или да се държи по непредвидим начин. JavaScript предоставя инструменти за откриване и управление на тези грешки, така че да можем да се справяме с тях по-ефикасно.


### Често срещана грешка: достъп до стойност на недефинирана


Ето една често срещана ситуация, която води до грешка:


```javascript
const user = undefined
console.log(user.name)
```


Ако стартирате този код, ще получите грешка, която изглежда по следния начин:


```
TypeError: Cannot read properties of undefined (reading 'name')
```


Това е JavaScript, който ви казва: "Опитахте се да получите свойството `име` от нещо, което е `неопределено`, и това няма смисъл." И както виждате, когато се случи такъв вид грешка, програмата спира да работи, освен ако не сте написали специално код, който да я улови и обработи.


### хвърляне на грешка


Понякога искате ръчно да **извикате грешка** в кода си. В този случай използвайте ключовата дума `throw`.


```javascript
throw new Error("This is a custom error message")
```


Това незабавно спира програмата и отпечатва:


```
Uncaught Error: This is a custom error message
```


Можете да използвате `throw`, за да наложите правила в програмата си. Например:


```javascript
function divide(a, b) {
if (b === 0) {
throw new Error("You can't divide by zero")
}
return a / b
}

console.log(divide(10, 2))  // OK: prints 5
console.log(divide(10, 0))  // Error!
```


Второто извикване води до грешка, тъй като делението на нула не е разрешено в този пример.


### Улавяне на грешки с try...catch


Ако не искате програмата ви да се срине при възникване на грешка, можете да уловите грешката с помощта на блок `try...catch`. Това е полезно, когато искате програмата ви да продължи да работи**, дори ако нещо се провали.


```javascript
try {
const user = undefined
console.log(user.name)
console.log("End of the block") // this will never get printed
} catch (error) {
console.log("Oops! Something went wrong.")
}
```


Изход:


```
Oops! Something went wrong.
```


Ето как работи:



- Първо се опитва да се изпълни кодът в блока `try`.

*Ако възникне грешка, JavaScript **преминава към блока `catch`**, като прескача останалата част от блока `try`.*


- Блокът `catch` получава грешката, така че можете да я отпечатате или да я обработите по друг начин, като например


```javascript
try {
const user = undefined
console.log(user.name)
console.log("End of the block") // this will never get printed
} catch (error) {
console.log(`The message of the error was: "${error.message}"`)
}
```


Изход:


```
The message of the error was: "Cannot read properties of undefined (reading 'name')"
```


### Окончателният блок


Можете да добавите и блок `finally`. Това е код, който **винаги се изпълнява**, независимо дали е имало грешка или не.


```javascript
try {
console.log("Trying something risky...")
throw new Error("Uh oh!")
} catch (error) {
console.log("Caught the error:", error.message)
} finally {
console.log("This will run no matter what.")
}
```


Изход:


```
Trying something risky...
Caught the error: Uh oh!
This will run no matter what.
```


## Избягване на грешки

<chapterId>db12d9f6-5806-514c-998e-0ae24805104e</chapterId>


В тази глава са показани някои от най-често срещаните капани в JavaScript и как да ги избегнем.


### var и присвояване без декларация


В по-стария код на JavaScript променливите често се декларираха с ключовата дума `var`. За разлика от `let` и `const`, за които вече научихте, `var` може да се държи по объркващ начин.


Например:


```javascript
{
var message = "hello"
}
console.log(message)
```


Може да се очаква, че `message` ще съществува само вътре в блока, но това не е така. `var` пренебрегва обхвата на блока и прави променливата достъпна в цялата функция или файл.


Това може да доведе до неочаквано поведение, особено в по-големи програми. Поради тази причина в съвременния код на JavaScript винаги трябва да се използва `let` или `const` вместо `var`.


Още по-лошо, JavaScript ви позволява да присвоявате стойности на променливи **без да ги декларирате**:


```javascript
function greet() {
user = "Alice"
}
greet()
console.log(user) // prints "Alice"
```


Създава се нова глобална променлива `name` без декларация. Това може да се случи тихомълком и да доведе до грешки, които е трудно да бъдат открити, особено ако става дума за печатна грешка. Винаги декларирайте променливите с помощта на `let` или `const`.


### Слаба система от типове


JavaScript е слабо типизиран, което означава, че при необходимост автоматично преобразува стойности от един тип в друг. Това се нарича принудително задаване на тип и макар да е удобно, често води до объркващи резултати.


Например:


```javascript
console.log("5" + 1)    // "51"
console.log("5" - 1)    // 4
console.log(true + 1)   // 2
console.log(null + 1)   // 1
```


В тези примери JavaScript се опитва да отгатне какво сте имали предвид. Понякога той превръща низове в числа, булеви стойности в числа или низове в низове. Това може да накара кода ви да се държи по неочакван начин.


Важно е да сте наясно със слабата система за типизиране на JavaScript. Когато нещата започнат да се държат странно, това може да се дължи на неочаквана принуда на типа.


### "use strict"


Можете да активирате по-строг режим, който превръща някои негласни грешки в реални и ви пречи да използвате някои от най-опасните функции на езика.


За да активирате този по-строг режим, добавете този ред в началото на файла или функцията:


```javascript
"use strict"
```


Например:


```javascript
"use strict"
name = "Alice" // ReferenceError: name is not defined
```


Без строг режим JavaScript ще създаде безшумно глобална променлива, наречена `name`. Но със строгия режим това се превръща в истинска грешка, което ви помага да откривате грешки по-рано.


Стриктният режим също така деактивира някои остарели функции на JavaScript и улеснява оптимизирането и поддръжката на кода ви.


## Стойност срещу референция

<chapterId>bb898425-dc2f-5e5c-864b-0cb7a4a9aea9</chapterId>


JavaScript третира различни видове стойности по различен начин.


Някои стойности се **копират**, когато ги присвоявате на променлива.


Други са **споделени**, когато ги присвоите на нова променлива, така че ако промените едната, ще се промени и другата.


### Предаване по стойност


Когато дадена стойност се предава **по стойност**, JavaScript прави нейно **копие**.


Така че ако промените едното, това няма да се отрази на другото.


Това се случва с примитивни типове, като например:


*числа*

*низове*

*булеви стойности* (`true` и `false`)

`null`

`*неопределено*`


Нека разгледаме един пример:


```javascript
let a = 5
let b = a

b = 10

console.log(a) // 5
console.log(b) // 10
```


Дадохме на `b` стойността на `a`, но след това променихме `b` на `10`.


Тъй като числата се предават по стойност, JavaScript копира символа `5` в символа `b`. Числото `5` в `b` е независимо от оригиналното число `5` в `a`, така че промяната на стойността на `b` няма ефект върху `a`.


Нека опитаме с низ:


```javascript
let name1 = "Alice"
let name2 = name1

name2 = "Bob"

console.log(name1) // Alice
console.log(name2) // Bob
```


Отново, промяната на `име2` не се отрази на `име1`, тъй като низовете също се предават по стойност.


Същото се случва, когато предавате примитив на функция: той се копира. Така че функцията не може да промени оригинала.


```javascript
function plusOne(x) {
x = x + 1
}

let number = 5

plusOne(number)

console.log(number) // 5
```


Въпреки че вътре във функцията `1` е добавено към `x`, първоначалното `число` не се е променило.


Това е така, защото във функцията е предадено само **копие**.


### Предаване по референция


Обектите се предават **по препратка**.


Това означава, че вместо да ги копира, JavaScript просто предава **препратка** към тях и ако ги промените, всички други променливи, които сочат към тях, също ще видят промяната.


Например:


```javascript
const person1 = { name: "Alice" }
const person2 = person1

person2.name = "Bob"

console.log(person1.name) // Bob
console.log(person2.name) // Bob
```


И `person1`, и `person2` сочат към един и същ обект в паметта.


Така че, когато променихме `person2.name`, променихме и `person1.name`, тъй като и двете търсят едно и също нещо.


Масивите са обекти, така че нека опитаме същото с масив:


```javascript
const list1 = [1, 2, 3]
const list2 = list1

list2.push(4)

console.log(list1) // [1, 2, 3, 4]
console.log(list2) // [1, 2, 3, 4]
```


Преместихме `4` в `list2`, но това се отрази и на `list1`, тъй като и двата масива се отнасят към един и същ масив.


Нека да видим какво се случва, когато подадем обект на функция:


```javascript
function rename(user) {
user.name = "Charlie"
}

const person = { name: "Dana" }

rename(person)

console.log(person.name) // Charlie
```


Функцията променя обекта! Това е така, защото тя получи **препратка** към първоначалния обект `person`.


Той не получи копие. То получи достъп до оригиналния обект и с това получи възможност да го променя.


Важно е да запомним това разграничение, защото в противен случай кодът ни може да се държи различно от това, което очакваме. Например може да напишем функция с очакването, че няма да променя получените от нея аргументи, а по-късно да открием, че всъщност ги е променяла (защото са били обекти и са били предадени по референция).


## Работа с функции

<chapterId>e0d277a8-c642-5af7-9e53-dee27c811967</chapterId>


Вече научихте как да декларирате и използвате функции в JavaScript. Но JavaScript ви дава още инструменти за работа с функции по мощни начини.


### Функции на стрелките


Функциите стрелки са по-кратък начин за писане на функции. Вместо да използваме ключовата дума `function`, използваме стрелка (`=>`).


Ето една нормална функция:


```javascript
function greet(name) {
return `Hello, ${name}!`
}
```


Версията със стрелка изглежда по следния начин:


```javascript
const greet = (name) => {
return `Hello, ${name}!`
}
```


Ако функцията има **само един ред**, можете да пропуснете скобите и да `return`:


```javascript
const greet = (name) => `Hello, ${name}!`
```


Ако функцията има **само един параметър**, можете дори да пропуснете скобите около параметрите:


```javascript
const greet = name => `Hello, ${name}!`
```


Функциите стрелки са много разпространени в съвременния JavaScript, тъй като позволяват да се изразяват прости функции със значително по-малък брой шаблони.


### Параметри по подразбиране


Понякога искате функцията да има стойност по подразбиране**, ако не е подаден аргумент.


Можете да направите това по следния начин:


```javascript
function sayHello(name = "friend") {
console.log(`Hello, ${name}!`)
}

sayHello("Alice") // Hello, Alice!
sayHello()        // Hello, friend!
```


Стойността по подразбиране `"friend"` се използва, когато не е подадено нищо.


### Параметри на разпространението (...)


Какво да правите, ако функцията ви приема гъвкав брой аргументи?


Можете да използвате оператора **spread** (`...`), за да ги съберете в масив:


```javascript
function logAll(...items) {
console.log(items)
}

logAll(1, 2, 3) // [1, 2, 3]
logAll("a", "b") // ["a", "b"]
```


След това можете да използвате цикъл за обработка на всеки елемент:


```javascript
function logEach(...items) {
for (const item of items) {
console.log(item)
}
}
```


Операторът spread е полезен, когато не знаете колко аргумента ще бъдат подадени.


### Функции от по-висок ред


Функция от **висок ред** е функция, която:


*приема друга функция като вход*

*и/или* връща функция като изход


Ето един прост пример:


```javascript
function runTwice(action) {
action()
action()
}

function sayHello(name = "friend") {
console.log(`Hello, ${name}!`)
}

runTwice(sayHello)
```


Този печат:


```
Hello, friend!
Hello, friend!
```


Можем да му предадем функция със стрелка:


```javascript
runTwice(
() => console.log("Hello!")
)
```


Този печат:


```
Hello!
Hello!
```


Можете също така да пишете функции, които **връщат** други функции:


```javascript
function makeGreeter(name) {
return () => console.log(`Hi, ${name}`)
}

const greetAlice = makeGreeter("Alice")
const greetBob = makeGreeter("Bob")

greetAlice() // Hi, Alice
greetBob() // Hi, Bob
```


Функцията `makeGreeter` е функция, която изгражда други функции. Тя получава низ и връща нова функция, която използва този низ в извикването на `console.log`.


Този вид шаблони са много мощни, тъй като ви позволяват да оставяте "дупки" във функциите си, които можете да запълните по-късно с необходимото поведение.


### map(), filter(), reduce()


JavaScript ви предоставя някои полезни вградени методи за работа с масиви.


Тези методи приемат функции като аргументи, така че те също са функции от по-висок ред.


`map()` трансформира всеки елемент от масив в нещо друго.


Пример:


```javascript
const numbers = [1, 2, 3]
const doubled = numbers.map(x => x * 2)

console.log(doubled) // [2, 4, 6]
```


Всяко число се удвоява и резултатът е нов масив.


`filter()` премахва елементите от масива, ако те не преминат успешно даден тест.


Пример:


```javascript
const numbers = [1, 2, 3, 4, 5]
const numbersGreaterThanTwo = numbers.filter(x => x > 2)

console.log(numbersGreaterThanTwo) // [3, 4, 5]
```


Съхраняват се само записите в масива, за които условието `x > 2` връща `true`.


`reduce()` се използва за обединяване на всички елементи в масив в една стойност.


Пример:


```javascript
const numbers = [1, 2, 3, 4]
const total = numbers.reduce((current, next) => current + next)

console.log(total) // 10
```


`reduce` преминава през масива и в този пример прилага оператора `+` между `1` и `2`, след това между получените `3` и `3`, след това между получените `6` и `4`, докато получи сумата от всички записи в масива (която е 10).


Можете да използвате функцията `reduce()` за много неща, като например суми, средни стойности или построяване на нови стойности стъпка по стъпка.


Тези методи (`map`, `filter`, `reduce`) са мощни инструменти за обработка на данни, без да се налага да се пишат ръчни цикли.


Можете дори да ги комбинирате във верига от операции по следния начин:


```javascript
const numbers = [1, 2, 3, 4, 5]

const result = numbers
.map(n => n * 2)        // Double each entry, obtain [2, 4, 6, 8, 10]
.filter(n => n > 3) // Keep only the entries bigger than 3, so you get [4, 6, 8, 10]
.reduce((n1, n2) => n1 + n2) // Adds them: 4 + 6 + 8 + 10 = 28

console.log(result) // 28
```


## Работа с обекти

<chapterId>7842aada-f009-5518-b8e3-1104e166a035</chapterId>


В тази глава ще се запознаем с някои мощни и малко по-усъвършенствани инструменти за работа с обекти в JavaScript.


### Частни имоти


Понякога искаме да скрием дадено свойство на обект, така че то да не може да бъде променяно или достъпвано отвън. JavaScript ни дава възможност да направим това, като използваме `#` преди името на свойството. Това създава **частно** свойство, което е достъпно само от вътрешността на класа.


```javascript
class Person {
#age // this is a private property

constructor(name, age) {
this.name = name
this.#age = age
}

getAge() {
return this.#age
}
}

const alice = new Person("Alice", 30)
console.log(alice.name)      // Alice
console.log(alice.getAge())  // 30, the method can access the private property
console.log(alice.#age)      // ❌ Error! You can't access private properties directly
```


Частните свойства са полезни, когато искате да предотвратите случайни промени.


### статични свойства


Понякога искате дадено свойство да принадлежи на самия клас, а не на всеки обект, който създавате от този клас. За това е предназначен кодът `static`. Свойството `static` се съдържа в класа и всички обекти от този клас ще се позовават на него.


```javascript
class User {
static counter = 0 // this belongs to the class, not to instances. The same counter will be shared by all objects

constructor() {
User.counter++ // changes the static property every time an object of this class gets initiated
}
}

const a = new User() // the constructor will change the shared counter from 0 to 1
const b = new User() // the constructor will change the shared counter from 1 to 2

console.log(User.counter) //  prints 2
```


Това е полезно за съхраняване на споделени данни и методи, които се отнасят за цялата група обекти, а не само за един.


### получаване и задаване на


В JavaScript функциите `get` и `set` ви позволяват да създавате свойства, които изглеждат като обикновени променливи, но всъщност изпълняват специален код във фонов режим.


Методът `get`ter се изпълнява, когато се опитате да *прочетете* дадено свойство. Той се декларира, като се напише `get` пред метод с името на свойството.


```javascript
class User {
constructor(firstName, lastName) {
this.firstName = firstName
this.lastName = lastName
}

get fullName() {
return `${this.firstName} ${this.lastName}`
}
}

const user = new User("Jane", "Doe")
console.log(user.fullName) // Jane Doe
```


Въпреки че `fullName` не е обикновено свойство, можем да го използваме като такова и зад кулисите се изпълнява функцията `get`, за да се създаде пълното име.


Методът `set`ter се изпълнява, когато *придавате* стойност на свойство. Той ви позволява да контролирате какво ще се случи, когато някой се опита да промени тази стойност. Декларира се, като се напише `set` пред метод с името на свойството.


```javascript
class User {
constructor() {
this.firstName = "John"
this.lastName = "Doe"
}

get fullName() {
return `${this.firstName} ${this.lastName}`
}

set fullName(input) {            // gets the name that is passed
const parts = input.split(" ") // breaks it into parts
this.firstName = parts[0]      // uses the first part as first name
this.lastName = parts[1]       // uses the second part as last name
}
}

const user = new User()
user.fullName = "John Smith"

console.log(user.firstName) // John
console.log(user.lastName)  // Smith
```


Когато направим `user.fullName = "John Smith"`, се изпълнява методът `set` и се актуализират стойностите `firstName` и `lastName`.


Така че, въпреки че изглежда, че просто задаваме една променлива, всъщност задействаме логика, която актуализира други свойства.


## Ключове и стойности

<chapterId>01a397b8-c12a-5c39-82b3-6d9ebbb72a29</chapterId>


Всяко свойство в обект на JavaScript има **ключ** (наричан също име на свойство) и **стойност**.


Например:


```javascript
const user = {
name: "Alice",
age: 30
}
```


В този обект `"име"` и `"възраст"` са ключовете, а "Alice" и `30` са техните стойности.


### Динамичен достъп


Понякога не знаете предварително името на дадено свойство... може би го получавате от потребителския вход или го четете от променлива. Все пак можете да получите достъп до него, като използвате **запис в скоби**, например `myObject["keyName"]`.


```javascript
const user = {
name: "Alice",
age: 30
}

console.log(user["name"]) // Alice
```


Предадохме символа `име` на обекта, за да получим съответната стойност.


Можем да запишем ключ в променлива и да го използваме за достъп до съответната стойност по-късно, както


```javascript
const user = {
name: "Alice",
age: 30
}

const key = "name"

console.log(user[key]) // Alice
```


### Динамичен Assignment


Можете също така да създавате или актуализирате свойства на обекти, като използвате променливи като ключове.


```javascript
const settings = {}

const key = "theme"
settings[key] = "dark"

console.log(settings) // { theme: "dark" }
```


Това е полезно, когато искате да изградите обект стъпка по стъпка. Например:


```javascript
const user = {}

user["username"] = "bananaFan123"
user["email"] = "banana@fruit.com"

console.log(user)
// { username: "bananaFan123", email: "banana@fruit.com" }
```


Можете дори да използвате динамичен ключ *при създаването* на обекта, като използвате квадратни скоби:


```javascript
const key = "language"
const config = {
[key]: "JavaScript"
}

console.log(config.language) // JavaScript
```


Това се нарича **компютрирано свойство**. Стойността в квадратните скоби се оценява и резултатът се използва като ключ.


### Вид на символа


В допълнение към символните низове JavaScript ви позволява да използвате и специален тип, наречен `Symbol`, като ключ за обект.


Нека започнем с прост пример:


```javascript
const id = Symbol("userID")

const user = {
name: "Bob",
[id]: 12345
}

console.log(user[id]) // 12345
```


В този пример `id` е символ. Той не е низ, но все пак работи като ключ. Ако се опитате да запишете `user` в конзолата, ще видите следното:


```javascript
console.log(user) // { name: 'Bob', [Symbol(userID)]: 12345 }
```


Друго важно нещо: всеки символ, който създавате, е уникален, дори ако е създаден с един и същ низ.


```javascript
const a = Symbol("label")
const b = Symbol("label")

console.log(a === b) // false
```


Символите ви позволяват да дефинирате клавиши, които няма да влизат в конфликт с обикновените клавиши. Например, да кажем, че имате обект със свойство `име`, но обектът ще може да се персонализира от потребител в бъдеще по начин, който не можете да предвидите, и този потребител може да добави и свойство `име`. Ако първоначалното свойство `име` е дефинирано с низ, то ще бъде презаписано от новото, както е показано по-долу:


```javascript
const obj = {
name: "John"
}

obj.name = "Jimmy"

console.log(obj) // { name: 'Jimmy' }
```


Ако вместо това използваме Symbol:


```javascript
const name = Symbol("name")

const obj = {
[name]: "John"
}

obj.name = "Jimmy"

console.log(obj) // { name: 'Jimmy', [Symbol(name)]: 'John' }
```


Както виждате, по този начин се запазва оригиналното свойство `name`. Това може да е полезно в някои крайни случаи.

JavaScript ни дава някои полезни вградени обекти, които ни помагат да извършваме неща като отстраняване на грешки и математически операции.


### Други методи за конзола


Вече сте виждали файла `console.log`, който отпечатва стойности на екрана.


Обектът `console` разполага и с някои други полезни методи, които могат да ви помогнат да отстранявате грешки в програмите си.


#### `console.warn`


Отпечатва съобщение в жълт цвят (или с икона за предупреждение в някои среди):

```javascript
console.warn("This is just a warning.")
```

#### `console.error`


Отпечатва съобщение в червено, подобно на грешка:


```javascript
console.error("Something went wrong!")
```

#### `console.table`


Показва масив или обект като таблица:

```javascript
const users = [
```

```javascript
console.time("timer")
```

Отпечатва се нещо подобно на:


```
timer: 2.379ms
```


Полезни са за прости тестове на производителността.


### Обектът Math


JavaScript ви предоставя обект `Math` с полезни методи за извършване на изчисления.


#### `Math.random()`


Връща случайно число между 0 (включително) и 1 (изключително):


```javascript
const r = Math.random()
console.log(r)
```


Примерен изход:


```
0.4387429859
```


#### `Math.floor()` и `Math.ceil()`



- `Math.floor(n)` закръглява **надолу** до най-близкото цяло число
- `Math.ceil(n)` закръгля **нагоре** до най-близкото цяло число


```javascript
console.log(Math.floor(4.9)) // 4

```

Закръгляне до най-близкото цяло число:


```javascript
console.log(Math.round(4.4)) // 4
console.log(Math.round(4.6)) // 5
```


#### `Math.max()` и `Math.min()`


Връща най-голямата или най-малката стойност от списък с числа:


```javascript
console.log(Math.max(5, 9, 3)) // 9
console.log(Math.min(5, 9, 3)) // 3
```


#### `Math.pow()` и `Math.sqrt()`



- `Math.pow(a, b)` ви дава `a` на степента на `b`
- `Math.sqrt(n)` ви дава квадратния корен от `n`


```javascript
console.log(Math.pow(2, 3))   // 8
console.log(Math.sqrt(16))    // 4
```

## Други колекции

<chapterId>a9a70c6d-a343-5a46-a383-e288bc2700e3</chapterId>


JavaScript ни дава някои специални типове колекции, които надхвърлят обикновените масиви и обекти. Те включват `Map` и `Set`.


Те ви помагат да съхранявате и управлявате групи от стойности, но работят по различен начин от този, който видяхте досега.


`Мап` е колекция от двойки ключ-стойност**, също като обект. Но тя има някои важни разлики:



- Ключовете могат да бъдат **всякаква стойност**, а не само низове.

*Редът на елементите се запазва.*


- Той има вградени методи, които улесняват работата с него.


Създавате нова карта по следния начин:


```javascript
const myMap = new Map()
```


Това създава празна карта. За да добавите записи в нея, използвайте `myMap.set(key, value)`:


```javascript
myMap.set("name", "Alice")
```


Добавя се ключ `"name"` със стойност `"Alice"`.


Можете също така да използвате число като ключ:


```javascript
myMap.set(42, "The answer")
```


И дори обект като ключ:


```javascript
const objKey = { id: 1 }
myMap.set(objKey, "Object as key")
```


Това няма да работи с обикновени обекти, които позволяват само низови ключове.


За да **получите стойност**, използвайте `myMap.get(key)`:


```javascript
console.log(myMap.get("name"))     // Alice
console.log(myMap.get(42))         // The answer
console.log(myMap.get(objKey))     // Object as key
```


За да **проверите дали даден ключ съществува**, използвайте `myMap.has(key)`:


```javascript
console.log(myMap.has("name")) // true
```


За да **премахнете ключ**, използвайте `myMap.delete(key)`:


```javascript
myMap.delete("name")
```


За да **изчистите цялата карта**, използвайте `myMap.clear()`:


```javascript
myMap.clear()
```


Картите са чудесни за управление на големи колекции от стойности, тъй като достъпът до стойности в голяма карта обикновено е много по-ефективен, отколкото до голям обект.


### Задайте


`Set` е колекция от **само стойности** (без ключове), като всяка стойност трябва да е **уникална**. Това означава:


*Не можете да имате една и съща стойност два пъти*

*Стойностите се съхраняват в реда, в който сте ги добавили*


Създавате комплект по следния начин:


```javascript
const mySet = new Set()
```


За да **прибавите стойности**, използвайте `mySet.add(value)`:


```javascript
mySet.add(1)
mySet.add(2)
mySet.add(2) // duplicate, will be ignored
```


Въпреки че се опитахме да добавим `2` два пъти, множеството ще запази само едно копие.


За да **проверите дали дадена стойност е в набора**, използвайте `mySet.has(value)`:


```javascript
console.log(mySet.has(2)) // true
console.log(mySet.has(3)) // false
```


За да **изтриете стойност**, използвайте `mySet.delete(value)`:


```javascript
mySet.delete(2)
```


За да **изчистите всичко**, използвайте `mySet.clear()`:


```javascript
mySet.clear()
```


`Set` е полезен, когато искате да запазите колекция от уникални стойности, без да се налага да проверявате ръчно за дубликати:


```javascript
const numberArray = [1, 2, 2, 3, 4, 4, 4, 5]

const numberSet = new Set(numberArray)

console.log(numberSet) // Set(5) { 1, 2, 3, 4, 5 }
```


С помощта на `Set` се избягват дублиранията.


## Итератори

<chapterId>61d24e5e-b7e4-541a-8322-778f61f26a72</chapterId>


Повечето неща в JavaScript, над които можете да правите цикли (като масиви, низове, карти, множества), са **итерабилни**: те могат да предоставят итератори за съдържанието си.


**Итератор** е специален обект в JavaScript, който ви помага да преминавате през списък от елементи **един по един**.


### Итератори на обекти


За разлика от масивите или картите, обикновените обекти **не могат да се итерират** с `for...of`. Ако опитате това:


```javascript
const user = {
name: "Alice",
age: 30
}

for (const value of user) {
console.log(value)
}
```


Ще получите грешка:


```
TypeError: user is not iterable
```


Това е така, защото обикновените обекти нямат вграден итератор. Но JavaScript ви дава други инструменти за работа с тях.


#### `Object.keys()`


Можете да използвате `Object.keys(obj)`, за да получите масив от **ключовете** на обекта и след това да го прехвърлите в цикъл:


```javascript
const user = {
name: "Alice",
age: 30
}

const keys = Object.keys(user)

for (const key of keys) {
console.log(key)
}
```


Този печат:


```
name
age
```


#### `Object.values()`


За да прегледате в цикъл **стойностите**, използвайте `Object.values()`:


```javascript
const user = {
name: "Alice",
age: 30
}

const values = Object.values(user)

for (const value of values) {
console.log(value)
}
```


Този печат:


```
Alice
30
```


#### `Object.entries()`


Ако искате да получите **и ключа, и стойността**, използвайте `Object.entries()`:


```javascript
const user = {
name: "Alice",
age: 30
}

const entries = Object.entries(user)

for (const [key, value] of entries) {
console.log(`${key} is ${value}`)
}
```


Този печат:


```
name is Alice
age is 30
```


Въпреки че обектите не са директно итерабилни, тези методи ви дават пълен достъп до съдържанието им по начин, който работи добре с `for...of`.


Но как работят итераторите?


### Symbol.iterator


Тайната зад всички итератори е специален **символ**, наречен `Symbol.iterator`.


Този символ е вграден ключ, който указва на JavaScript: "Този обект може да бъде итериран."


Когато извикате `myIterable[Symbol.iterator]()`, JavaScript ви връща обект **iterator** с метод `.next()`.


Нека видим как изглежда това:


```javascript
const colors = ["red", "green", "blue"]

const iterator = colors[Symbol.iterator]()

console.log(iterator.next()) // { value: 'red', done: false }
```


Всяко извикване на `.next()` ви дава следващата стойност. Когато приключи, той се връща:


```javascript
{ value: undefined, done: true }
```


### следващ()


Методът `.next()` се използва за получаване на следващия елемент от последователността.


Всеки път, когато извикате `.next()`, получавате обект с два ключа:



- `value`: текущият елемент
- `done`: булева стойност, която показва дали итерацията е приключила


Нека направим пълен пример:


```javascript
const names = ["Lina", "Tom", "Eva"]      // declare an array
const iterator = names[Symbol.iterator]() // use the Symbol.iterator function to get an iterator for this array

let result = iterator.next()              // get the first element of the array

while (!result.done) {                    // repeat this loop until you reach the last element of the array, which is marked with { done: true }
console.log(result.value)               // print the value of each element
result = iterator.next()                // get the next element of the array
}
```


Този печат:


```
Lina
Tom
Eva
```


Ето как работи цикълът `for...of` под капака: той използва този модел с `.next()`.


Същият резултат ще получите с


```javascript
const names = ["Lina", "Tom", "Eva"]

for (const result of names) {
console.log(result)
}
```


### Създаване на итерабилен клас


Можете също така да дефинирате свой собствен клас **iterable**, като добавите метод `[Symbol.iterator]()`.


Да речем, че искаме клас, който да представя **обхват от числа**, например от 1 до 5.


```javascript
class Range {
constructor(start, end) {
this.start = start
this.end = end
}

[Symbol.iterator]() {
let current = this.start
const end = this.end

return {
next() {
if (current <= end) {
const result = { value: current, done: false }
current = current + 1
return result
} else {
return { done: true }
}
}
}
}
}

const myRange = new Range(1, 5)

for (const num of myRange) {
console.log(num)
}
```


Този печат:


```
1
2
3
4
5
```


Ето какво се случва:


**Дефинирахме клас `Range`**

*Вътре в класа имплементирахме `[Symbol.iterator]()`, за да може JavaScript да знае как да го итерира*

*Методът `next()` връща всяко число едно по едно*

Когато достигнем до `край`, той връща `{ done: true }`


Сега нашият клас `Range` работи като масив и можем да го използваме във всеки цикъл, който очаква итерабилна таблица.


### Функции на генератора и добив


За да улесни създаването на итератори, JavaScript ви дава **генериращи функции**, като използва ключовата дума `function*` (това е `function` с `*` в края) и ключовата дума `yield`.


Да опитаме:


```javascript
function* numberGenerator() {
yield 1
yield 2
yield 3
}

const iterator = numberGenerator()

console.log(iterator.next()) // { value: 1, done: false }
console.log(iterator.next()) // { value: 2, done: false }
console.log(iterator.next()) // { value: 3, done: false }
console.log(iterator.next()) // { value: undefined, done: true }
```


Всяко `yield` връща стойност и **спира** функцията до извикването на следващото `.next()`.


Можете също така да направите цикъл на генератор с помощта на `for...of`:


```javascript
for (const num of numberGenerator()) {
console.log(num)
}
```


Този печат:


```
1
2
3
```


## Съвместимост с обратни повиквания

<chapterId>f3fc76ca-b3ef-54eb-a06e-501007002054</chapterId>


Досега нашият код беше **синхронен**: изпълняваше се по един ред в последователност. Но някои неща в реалния свят отнемат време и ние не искаме цялата програма да спира, докато чака.


В тази глава ще представим нова концепция: **валута**. Тя ни позволява да манипулираме реда, по който се извършват нещата. Това е полезно при работа с неща като таймери, потребителски вход или четене на файлове от диска. JavaScript предлага различни инструменти за осъществяване на паралелност.


### setTimeout


Функцията `setTimeout` ви позволява да **изпълните дадена функция по-късно**, след като е минало известно време.


Пример:


```javascript
console.log("Start")

setTimeout(
() => console.log("This runs after 2 seconds"),
2000
)

console.log("End")
```


Този печат:


```
Start
End
This runs after 2 seconds
```


Въпреки че `setTimeout` се появява в средата на кода, той не блокира останалата част. Вместо това тя планира функцията да се изпълни **по-късно** и веднага продължава напред.


Числото `2000` означава 2000 милисекунди (което е 2 секунди).

Предлагаме ви по-обстоен и удобен за начинаещи препис на разделите **Callbacks** и **Promise**, в които се използват манипулации с данни и ясни анотации:


### Обратни извиквания


Обратната връзка** е просто функция, която даваме на друга функция, за да може тя да бъде **извикана по-късно**.


Нека разгледаме реален пример с числа. Представете си, че имаме списък с числа и искаме да удвоим всяко едно от тях, след което да приложим функция (обратна връзка) към получения "удвоен" масив, но искаме да го направим с малко закъснение, все едно чакаме нещо бавно (например зареждане на данни от интернет).


Ето една функция, която прави това с помощта на **обратна връзка**:


```javascript
function doubleNumbers(numbersArray, callback) {
// Pretend we're doing a slow operation using setTimeout
setTimeout(() => {
// Use the map method to create a new array where each number is doubled
const doubled = numbersArray.map(n => n * 2)

// When we're done, we call the callback function with the result
callback(doubled)
}, 1000) // Wait 1 second before running the code inside
}
```


Нека се опитаме да използваме тази функция:


```javascript
const input = [1, 2, 3]

doubleNumbers(input, function(result) {
console.log("Here is the doubled array:", result)
})
```


След 1 секунда това се отпечатва:


```
Here is the doubled array: [ 2, 4, 6 ]
```


**Какво става тук?**


1. Предаваме `input` като списък с числата, които искаме да удвоим.

2. Предаваме и функция за обратна връзка**, която указва на програмата какво да направи *след* удвояването.

3. Във вътрешността на `doubleNumbers` симулираме забавяне с помощта на `setTimeout`, след което извършваме удвояването.

4. След като това бъде направено, извикваме обратната връзка за получения "удвоен" масив.


Тази техника работи, но представете си, че искате да направите **повече стъпки** след това, например да филтрирате малките числа и след това да ги съберете. Ще трябва да **направите** повече подобни обратни извиквания:


```javascript
doubleNumbers(input, function(doubled) {
filterBigNumbers(doubled, function(filtered) {
sumNumbers(filtered, function(total) {
console.log("Final result:", total)
})
})
})
```


Трудно е да се чете и е разхвърляно. Този стил се нарича "ад на обратните връзки "** и е точно това, което `Promise` е създадено, за да поправи.


## Съвместимост с обещания

<chapterId>30fddaca-729f-5c8d-bf86-8dfc7b3c9800</chapterId>


Обещанието `Promise` е вграден обект на JavaScript, който представлява стойност, която ще бъде готова в бъдеще**.


Можем да създадем обещание по следния начин:


```javascript
const promise = new Promise((resolve, reject) => {
// Do something that takes time here...

resolve("It worked!") // This means everything went OK
})
```


Частта `new Promise()` създава обещанието.


Вътре в него му даваме функция с два параметъра:


`resolve` е функция, която извикваме, когато всичко е успешно

`reject` е функция, която извикваме, ако нещо се обърка


В примера по-горе просто го разрешаваме веднага със съобщението `"Работи!"`.


### .then()


За да направим нещо **след** изпълнението на обещанието, използваме `.then()`:


```javascript
const promise = new Promise((resolve, reject) => {
// Do something that takes time here...

resolve(100) // This means everything went OK
})

promise.then(result => {
console.log("The result is:", result)
})
```


Този печат:


```
The result is: 100
```


Стойността, която предадохме на `resolve()`, се изпраща на функцията вътре в `.then()` като `result`.


Нека симулираме задача, която отнема 2 секунди, като използваме `setTimeout`:


```javascript
const delayedPromise = new Promise(
(resolve, reject) => {
setTimeout(
() => resolve("Done waiting!"),
2000
)
})

delayedPromise.then(result => console.log(result))
```


Ще изчакате 2 секунди и след това ще отпечатате:


```
Done waiting!
```


### отхвърляне()


Нека създадем обещание, което **не се изпълнява**:


```javascript
const failingPromise = new Promise((resolve, reject) => {
reject("Something went wrong")
})
```


Ако сега използваме `.then()` за това, няма да се случи нищо, защото `.then()` обработва само успех.


За обработка на грешки използваме `.catch()`:


```javascript
const failingPromise = new Promise((resolve, reject) => {
reject("Something went wrong")
})

failingPromise
.then(
result => console.log("This will NOT run:", result)
)
.catch(
error => console.log("Caught an error:", error)
)
```


Това отпечатва само


```
Caught an error: Something went wrong
```


Стойността, предадена на `reject()`, се изпраща на функцията `.catch()`.


Нека създадем обещание, което **понякога работи, а понякога не успява**, въз основа на някакво условие.


```javascript
function checkNumber(n) {
return new Promise((resolve, reject) => {
if (n > 0) {
resolve("Positive number")
} else {
reject("Not a positive number")
}
})
}
```


Сега можем да извикаме това и да се справим и с двата случая:


```javascript
checkNumber(5)
.then(
msg => console.log("Success:", msg)
)
.catch(
err => console.log("Failure:", err)
)
```


Този печат:


```
Success: Positive number
```


И ако опитаме с друг номер:


```javascript
checkNumber(-1)
.then(
msg => console.log("Success:", msg)
)
.catch(
err => console.log("Failure:", err)
)
```


Той отпечатва:


```
Failure: Not a positive number
```


### Последователни операции с помощта на обещания



Можем да пренапишем предишния си пример, като използваме `Promise`, и той ще изглежда много по-чист.


Нека започнем с написването на нова версия на нашата функция за удвояване, но този път тя връща **обещание**:


```javascript
function doubleNumbers(numbers) {
return new Promise(resolve => {
// Wait 1 second before doing the operation
setTimeout(() => {
const doubled = numbers.map(n => n * 2)
resolve(doubled) // Return the result using resolve
}, 1000)
})
}
```


Сега можем да използваме `.then()`, за да кажем на JavaScript какво да направи с резултата:


```javascript
function doubleNumbers(numbers) {
return new Promise(resolve => {
// Wait 1 second before doing the operation
setTimeout(() => {
const doubled = numbers.map(n => n * 2)
resolve(doubled) // Return the result using resolve
}, 1000)
})
}

const input = [1, 2, 3]

doubleNumbers(input)
.then(
result => console.log("Doubled numbers:", result)
)
```


Този печат:


```
Doubled numbers: [ 2, 4, 6 ]
```


Досега това работи по същия начин като версията с обратна връзка, но сега кодът е по-лесен за разширяване и четене.


Да речем, че искаме да добавим още стъпки:


1. Първо, удвоете всички числа

2. След това премахнете числата, по-малки от 4

3. И накрая, добавете ги всички заедно


Можем да напишем по една функция за всяка стъпка, като всички те използват обещания:


```javascript
function doubleNumbers(numbers) {
return new Promise(resolve => {
// Wait 1 second before doing the operation
setTimeout(() => {
const doubled = numbers.map(n => n * 2)
resolve(doubled) // Return the result using resolve
}, 1000)
})
}

function filterBigNumbers(numbers) {
return new Promise(resolve => {
setTimeout(() => {
const filtered = numbers.filter(n => n > 3)
resolve(filtered)
}, 1000)
})
}

function sumNumbers(numbers) {
return new Promise(resolve => {
setTimeout(() => {
const total = numbers.reduce((acc, n) => acc + n, 0)
resolve(total)
}, 1000)
})
}
```


Сега можем да ги свържем заедно по следния начин:


```javascript
const input = [1, 2, 3]

doubleNumbers(input)
.then(filterBigNumbers)
.then(sumNumbers)
.then(
result => console.log("Final result after all steps:", result)
)
```


Този печат:


```
Final result after all steps: 10
```


Нека разгледаме какво прави това:


1. `doubleNumbers` удвоява масива: `[2, 4, 6]`

2. `filterBigNumbers` премахва всичко ≤ 3: `[4, 6]`

3. `sumNumbers` добавя останалите числа: `4 + 6 = 10`

4. Накрая отпечатваме резултата.


Всяка стъпка `.then()` изчаква завършването на предходната стъпка. Така можем да изградим **верига от действия** без влагане. Това прави кода по-четим и по-лесен за отстраняване на грешки.


## Съвместимост с async/await

<chapterId>6e93d29f-c8bf-5fd1-a9c9-4e794ee6cbd0</chapterId>


Видяхме как веригите `Promise` ни помагат да избегнем ада на обратните извиквания, но все пак те могат да станат малко трудни за четене, когато са включени много стъпки.


Тук влизат `async` и `await`. Те ни позволяват да пишем асинхронен код, **който изглежда като синхронен код**, което го прави по-лесен за разбиране.


### Какво е асинхронно?


Когато напишете ключовата дума `async` пред функция, JavaScript автоматично обвива връщаната стойност на функцията в обещание.


Нека видим един основен пример:


```javascript
async function greet() {
return "hello"
}
```


Ако извикате тази функция:


```javascript
const result = greet()
console.log(result)
```


Ще видите това:


```
Promise { 'hello' }
```


Въпреки че току-що сте върнали низ, JavaScript го превръща в обещание. Можете да получите действителната стойност, като използвате `.then()` по следния начин:


```javascript
greet().then( result => console.log(result) ) // prints "hello"
```


Или можете да използвате `await`...


### Какво е очакване?


Ключовата дума `await` казва на JavaScript: "изчакай, докато това обещание бъде изпълнено, и след това ми дай резултата."


Но можете да използвате `await` само **вътре в асинхронна функция**.


Нека пренапишем примера, като използваме `await`:


```javascript
async function greet() {
return "hello"
}

async function greetAndLog() {
const result = await greet()
console.log(result)
}

greetAndLog() // prints "hello"
```


Сега можем да използваме резултата, сякаш е обикновена стойност.


Нека сега направим нещо по-полезно.


### Симулиране на забавяне с await


Ще създадем проста функция `wait`, която приема като аргумент количество милисекунди и просто разрешава след този брой милисекунди, без да прави нищо друго:


```javascript
function wait(ms) {
return new Promise(resolve => {
setTimeout(resolve, ms)
})
}
```


Нека се опитаме да го използваме:


```javascript
async function test() {
console.log("waiting 2 seconds...")
await wait(2000)
console.log("done waiting")
}

test()
```


Този печат:


```
waiting 2 seconds...
done waiting
```


Можете да мислите за `await` като за "спрете тук, докато обещанието бъде изпълнено, след което продължете"


Това ви позволява да пишете код отгоре-надолу**, който се държи асинхронно, без да се налага верижно извикване на `.then()`.


### В очакване на данни


Нека повторим предишния пример, в който удвоявахме числата, след това филтрирахме, а след това сумирахме. Но този път ще използваме `async`/`await`.


Ще създадем 3 функции, които симулират чакане и връщат обещания:


```javascript
function doubleNumbers(numbers) {
return new Promise(resolve => {
setTimeout(() => {
const doubled = numbers.map(n => n * 2)
resolve(doubled)
}, 1000)
})
}

function filterBigNumbers(numbers) {
return new Promise(resolve => {
setTimeout(() => {
const filtered = numbers.filter(n => n > 3)
resolve(filtered)
}, 1000)
})
}

function sumNumbers(numbers) {
return new Promise(resolve => {
setTimeout(() => {
const total = numbers.reduce((acc, n) => acc + n, 0)
resolve(total)
}, 1000)
})
}
```


Сега нека напишем функция `async`, която да ги комбинира:


```javascript
async function process(numbers) {
const doubled = await doubleNumbers(numbers)
const filtered = await filterBigNumbers(doubled)
const total = await sumNumbers(filtered)

console.log("Final result:", total)
}

const input = [1, 2, 3]
process(input)
```


Този печат:


```
Final result: 10
```


Това е много по-лесно за четене, отколкото верижните `.then()` или вложените обратни извиквания.


Тя прилича на обикновена програма стъпка по стъпка, но все пак се държи асинхронно.


## Асинхронни итератори

<chapterId>438b037d-9931-56d7-9052-7b4470f3c75b</chapterId>


Вече научихте за **итераторите** и за това как можем да използваме `for...of`, за да прелистваме масиви и други итерабилни неща.


Но какво да правим, ако данните, които искаме да итерираме, пристигат за известно време?


Понякога искаме да прехвърлим в цикъл неща, които пристигат **асинхронно**, например съобщения от чат, редове от файл или числа от бавен източник.


За тази цел JavaScript ни предоставя **async итератори**.


### Функции на асинхронния генератор


Най-лесният начин за създаване на асинхронен итератор е да се използва **асинхронна генераторна функция**.


Пишем го така:


```javascript
async function* generateNumbers() {
yield 1
yield 2
yield 3
}
```


Това изглежда като обикновен генератор, но с `async` преди него.


Сега можем да използваме `for await...of`, за да консумираме стойностите:


```javascript
async function run() {
for await (const n of generateNumbers()) {
console.log("Got number:", n)
}

console.log("Done!")
}

run()
```


Това ще доведе до отпечатване на:


```
Got number: 1
Got number: 2
Got number: 3
Done!
```


Каква е разликата с обикновения генератор?


Разликата е, че вече можем да използваме `await` вътре в генератора.


Нека отново направим помощник за забавяне:


```javascript
function wait(ms) {
return new Promise(resolve => setTimeout(resolve, ms))
}
```


Сега нека да дадем числа **бавно**:


```javascript
async function* generateSlowNumbers() {
await wait(1000)
yield 1

await wait(1000)
yield 2

await wait(1000)
yield 3
}
```


Опитайте се да го използвате:


```javascript
async function run() {
for await (const n of generateSlowNumbers()) {
console.log("Got number:", n)
}

console.log("Done!")
}

run()
```


### Защо да използвате асинхронни итератори?


Асинхронните итератори са полезни, когато:


*Стойностите не пристигат наведнъж.*

*Искате да се справяте с тях един по един*, **когато се появят**.

**Работите с обещания и искате да правите цикли по чист начин.**


Например, ако искате да заредите съобщения от чат сървър едно по едно или да изтеглите голям файл на части, асинхронните итератори ви дават възможност да напишете цикъл `for`, който работи със забавени данни.


### Symbol.asyncIterator


Можем да използваме асинхронни итератори и в потребителски класове.


Ето един пример, който дава числа със закъснение:


```javascript
function wait(ms) {
return new Promise(resolve => setTimeout(resolve, ms))
}

class DelayedNumbers {
constructor(numbers) {
this.numbers = numbers
}

async *[Symbol.asyncIterator]() {
for (const n of this.numbers) {
await wait(1000)
yield n
}
}
}
```


Сега можем да използваме `for await...of` както преди:


```javascript
async function run() {
const source = new DelayedNumbers([10, 20, 30])

for await (const n of source) {
console.log("Received:", n)
}

console.log("All done!")
}

run()
```


Това ви позволява да създавате обекти, които могат да бъдат итерирани асинхронно


## Синтаксис Assignment захар

<chapterId>8b1ba7d8-ecfd-5470-b86e-73cb84ccc8b7</chapterId>


"Синтактична захар" означава да напишете нещо по-кратко или по-лесно, без да променяте действието му. Това е просто по-красив начин да се каже едно и също нещо.


JavaScript има някои вградени синтактични захари, които ни позволяват да пишем по-чисти и по-кратки декларации. В тази глава ще разгледаме как да присвояваме стойности въз основа на условие, да актуализираме променливи с математика, да извличаме стойности от масиви или обекти и да ги копираме или комбинираме с по-прост синтаксис.


### Тернарният оператор


В JavaScript можете да присвоите стойност въз основа на условие, като използвате оператора **ternary**, който представлява кратък начин за изписване на `if...else`.


Вместо да правите:


```javascript
let message

if (isMorning) {
message = "Good morning"
} else {
message = "Hello"
}
```


Можете да напишете:


```javascript
const message = isMorning ? "Good morning" : "Hello"
```


Това означава:



- Ако `isMorning` е true, използвайте `"Добро утро"`

*В противен случай използвайте `"Hello"`*


Общата форма е:


```javascript
condition ? valueIfTrue : valueIfFalse
```


Можете да го използвате и в други изрази:


```javascript
console.log(isSunny ? "Go outside" : "Stay in")
```


Само се уверете, че го използвате за **прости** решения. Ако логиката е сложна, използвайте `if...else`.


### Алтернативни оператори на Assignment


JavaScript разполага с **кратки оператори** за извършване на задания, комбинирани с операции.


Нека разгледаме нормалния начин:


```javascript
let counter = 1
counter = counter + 1
```


Това може да се съкрати до:


```javascript
let counter = 1
counter += 1 // same as counter = counter + 1
```


Ето най-често срещаните от тях:



| Оператор | Значение |
| -------- | ------------------- |
| `+=` | добавяне и присвояване |
| `-=` | изваждане и присвояване |
| `*=` | умножение и присвояване |
| `/=` | деление и присвояване |

Примери:


```javascript
let score = 10
score += 5  // now score is 15
score *= 2  // now score is 30
score -= 10 // now score is 20
```


Те са полезни, когато искате да актуализирате дадена променлива, като използвате нейната собствена стойност.


### Деструктуриране


**Деструктирането** ви позволява лесно да извличате стойности от масиви или обекти и да ги съхранявате в променливи.


#### Масиви


Да предположим, че имате:


```javascript
const colors = ["red", "green", "blue"]
```


Вместо да правите:


```javascript
const first = colors[0]
const second = colors[1]
```


Можете да направите:


```javascript
const [first, second] = colors
```


При това се присвояват:


`*първи*` за `"червен"`


- `втора` на `"зелена"`


Можете да пропуснете и стойности:


```javascript
const [,, third] = colors
console.log(third) // blue
```


#### Обекти


Можете да извличате стойности и от обекти:


```javascript
const user = { name: "Alice", age: 30 }

const { name, age } = user
console.log(name) // Alice
console.log(age)  // 30
```


Ако името на свойството е различно от това на желаната променлива, можете да го преименувате:


```javascript
const { name: username } = user
console.log(username) // Alice
```


Деструктурирането прави кода ви по-чист при работа с обекти и масиви.


### Синтаксис на Spread


Синтаксисът **spread** използва `...` за разопаковане или копиране на стойности.


#### Масиви


Можете да копирате или сливате масиви:


```javascript
const a = [1, 2]
const b = [3, 4]

const both = [...a, ...b]
console.log(both) // [1, 2, 3, 4]
```


Можете също така да клонирате масив:


```javascript
const original = [10, 20, 30]
const clone = [...original]
```


#### Обекти


Същото можете да направите и с обектите:


```javascript
const a = { x: 1 }
const b = { y: 2 }

const merged = { ...a, ...b }
console.log(merged) // { x: 1, y: 2 }
```


Можете също така да замените стойностите:


```javascript
const user = { name: "Alice", age: 30 }
const updated = { ...user, age: 31 }

console.log(updated) // { name: "Alice", age: 31 }
```


Това е много полезно, когато актуализирате обекти, без да променяте оригинала.

## Как стигнахме до Node

<chapterId>0da1d60c-06c9-54e6-a181-ae7dabf6e3b8</chapterId>


В тази глава ще научим малко исторически контекст за JavaScript и NodeJS.


Историческият контекст е много важен в областта на софтуера, тъй като често използваме инструменти, създадени от други хора, и следователно сме повлияни от решения, взети в миналото от тях.


Разбирането на причините за тези решения и на начина, по който инструментите, които използваме, са станали такива, каквито са, ще ни помогне да се чувстваме по-малко объркани в това, което правим.


### Произход на JavaScript


JavaScript започна като прост език, предназначен да направи уеб страниците интерактивни.


През 90-те години на миналия век в уеб браузъри като Netscape Navigator беше добавен JavaScript, така че разработчиците да могат да пишат код, който се изпълнява директно в браузъра.


Първоначалната идея беше Java да бъде основният език за създаване на уебсайтове (с т.нар. "Java аплети"), а JavaScript - само за второстепенни неща.


Основният дизайн е направен от Брендън Айх, който по това време е служител в Netscape, за по-малко от 2 седмици.


Но повечето хора предпочитаха да използват JavaScript пред Java, а и Java аплетите имаха някои проблеми със сигурността по онова време, така че в крайна сметка Java отпадна като опция и JavaScript се превърна в де факто стандарт за уеб разработване.


### Двигателят V8


JavaScript е интерпретиран език, за разлика от компилираните езици като C.


Кодът, написан на компилиран език, се превръща в двоичен код, който се подава директно към процесора на компютъра.


![](assets/en/006.webp)


От друга страна, интерпредните езици са по-удобни за ползване и са по-близки до начина, по който хората мислят ("високо ниво"), отколкото до начина, по който машините работят ("ниско ниво"); затова те обикновено имат изградена виртуална машина за изпълнение на техния код.


Виртуалната машина е специална програма, която се намира между написания от вас код и процесора и изпълнява вашия код (тъй като процесорът не може да го разбере).


Това ви позволява да програмирате, без да знаете твърде много за основната машина, но също така има цена по отношение на производителността, тъй като компютърът не работи само с вашата програма, а с програма (виртуална машина), която изпълнява вашата програма.


Тъй като уеб приложенията ставаха все по-сложни, възникна необходимост от подобряване на производителността на JavaScript. Двигателят V8 е интерпретаторът, който управлява JavaScript в Google Chrome. Той е разработен в Google и пуснат през 2008 г.


Докато по-старите енджини на JavaScript бяха предимно традиционни виртуални машини, енджинът на V8 е JIT (just-in-time) компилатор.


Кодът на JavaScript се подава към енджина V8 и той се опитва да компилира части от него като местни двоични файлове в движение. Това ви позволява да имате опита на език от високо ниво с производителност, която е малко по-близка до тази на езиците от ниско ниво. Това превърна JavaScript в най-бързия език от високо ниво в света - нещо като "най-доброто от двата свята".


### Изпълнението на NodeJS


Въпреки че е лесен за използване и доста бърз за изпълнение, след пускането на V8 JavaScript продължава да има огромно ограничение: може да се изпълнява само в браузър.


Защо това е проблем?


Тъй като браузърите изпълняват код от милиони различни източници в интернет, те лесно могат да се превърнат в зловреден софтуер, затова са "изолирани" от останалата част на операционната система.


![](assets/en/007.webp)


JavaScript не можеше да осъществява достъп до файловата система и други локални ресурси на компютъра (поне не така лесно, както другите езици), така че това беше значително ограничение за вида на приложенията, които можехте да създавате с него.


През 2009 г. Райън Дал публикува NodeJS, който представлява среда за изпълнение, която ви позволява да използвате енджина V8 извън браузъра, директно в собствената операционна система на вашия компютър. То също така добавя много функции, които са полезни за писане на програми от страна на сървъра и от команден ред. Например можете да използвате NodeJS, за да създадете уеб сървър, да четете и записвате файлове или да създавате инструменти, които автоматизират задачи.


![](assets/en/008.webp)


В този курс досега разгледахме функциите на JavaScript, които са налични както в браузъра, така и в NodeJS. Тези функции ни позволиха да дефинираме данни и да ги манипулираме по абстрактни начини. В следващите няколко урока ще разгледаме функциите, които са специфични за NodeJS и ни позволяват да взаимодействаме с операционната система.


## Аргументи на командния ред

<chapterId>960d20f3-c424-5d51-a041-ef17d2e94b6d</chapterId>


NodeJS ни позволява, наред с други неща, да създаваме CLI (Command Line Interfaces).


За тази цел ни е необходим начин за получаване на аргументи от командния ред, което в Node става с помощта на вградения обект `process`.


### процес


NodeJS предоставя специален обект, наречен `process`, който представлява текущата изпълнявана програма.


Можете да го използвате, за да проверите средата, текущата работна директория и дори да излезете от програмата, когато е необходимо.


Например:


```javascript
console.log(process.platform)
```


Тук се изписва платформата на операционната система, например `win32`, `linux` или `darwin` (Mac).


### process.argv


Когато стартирате програма NodeJS от терминала, можете да подадете допълнителни думи (аргументи) след името на скрипта. Те се съхраняват в `process.argv`.


Например, ако изпълните тази команда:


```
node my_script.js alpha beta
```


Можете да отпечатате аргументите по следния начин:


```javascript
console.log(process.argv)
```


Резултатът може да изглежда така:


```
[ '/path/to/node', '/path/to/my_script.js', 'alpha', 'beta' ]
```


Първите два елемента винаги са пътят до възела и пътят до скрипта. Всички допълнителни думи, които сте предали на скрипта, идват след това.


Масивът `process.argv` може да бъде нарязан като всеки друг масив с помощта на метода `.slice()`, така че за да получите само подадените аргументи, можете да направите следното


```javascript
const args = process.argv.slice(2)

console.log(args)
```


Достъпът до аргументите, които потребителят подава, е от основно значение за разработването на приложения за команден ред.


## Модули

<chapterId>4e1651a5-65fd-50bc-b22a-40313d5659ca</chapterId>


Изпълнителните среди на JavaScript като NodeJS обикновено разглеждат всеки файл на JavaScript като отделен модул.


Мислете за модула като за кутия. Кутията има свое собствено пространство, така че променливите и функциите, които сте декларирали в нея, не пречат на кода в други кутии. По принцип всеки модул има свой собствен обхват.


Един модул може да експортира част от съдържанието си, като го предоставя на други модули, и може да импортира съдържанието, което други модули са експортирали. JavaScript ви позволява да експортирате и импортирате съдържание между модули, да свързвате различни файлове.


Една програма на JavaScript често се състои от множество модули, които са свързани помежду си.


Защо да използвате модули? Като разделите кода си на модули, можете да организирате програмата си на по-малки, по-ясни и многократно използваеми части. Всеки модул може да се фокусира само върху един вид задача, например обработка на математически изчисления, работа с файлове или форматиране на текст.


### Модули CommonJS


В NodeJS най-разпространената система за управление на модули се нарича **CommonJS**.


В тази система можете да споделяте (експортирате) код от даден модул с помощта на `module.exports` и да го зареждате (импортирате) в друг файл с помощта на `require()`.


За да направите нещо достъпно извън даден модул, трябва да го присвоите на `module.exports`:


```javascript
const greeting = "Hello!"

module.exports = greeting
```


Тук низът `"Hello!"` е това, което този модул експортира.


За да използвате експортирания код от друг файл, използвайте функцията `require()` с пътя до този файл:


```javascript
const greeting = require("./greeting.js")

console.log(greeting)
```


Този печат:


```
Hello!
```


Можете да експортирате няколко неща, като ги обедините в анонимен обект, например


```javascript
const greeting1 = "hello"

const greeting2 = "hi"

module.exports = {
greeting1, greeting2
}
```


CommonJS е системата от модули, която първоначално е възприета от NodeJS. По-късно бяха добавени и модули ES.


### Модули ES


NodeJS поддържа и друг тип модули, наречени **ES Modules**, които са популярни в уеб разработката. Те използват ключовите думи `export` и `import`.


Модулите ES са разработени за браузъра и едва по-късно са добавени в Node. Ако искате да ги използвате, може да се наложи да използвате разширението на файла `.mjs` вместо `.js`, в зависимост от това коя версия на Node използвате.


В един файл, наречен `greeting.mjs`, пишем


```javascript
export const greeting = "Hello!"
```

Както можете да видите, експортираме константата директно там, където тя е дефинирана


В друг файл, наречен `index.mjs`, го импортираме:


```javascript
import { greeting } from "./greeting.mjs"

console.log(greeting)
```


Можете да експортирате различни декларации в различни части на файла, например


```javascript
export const num = 10

export function double (x) {
return x*2
}
```


### Стандартната библиотека на NodeJS


NodeJS включва и много вградени модули. Това са готови модули, предоставяни от NodeJS, които помагат за общи задачи като четене на файлове, работа с операционната система или свързване с мрежата.


Например модулът `os` дава информация за вашата операционна система:


```javascript
const os = require("os")

console.log(os.platform())
```


Не е необходимо да инсталирате тези вградени модули, те идват с NodeJS. Те формират "стандартната библиотека" на времето за изпълнение и се използват от повечето приложения на Node за извършване на дейности като четене на файлове или комуникация през интернет.


В следващите глави ще видите някои полезни примери за използването им.


## Модулът fs

<chapterId>911e953a-35ae-5ee7-bd74-372501c32e81</chapterId>


Модулът `fs` (съкращение от **файлова система**) е част от стандартната библиотека на NodeJS. Той ви позволява да работите с файлове и директории на вашия компютър: можете да четете файлове, да записвате файлове, да ги изтривате, да ги преименувате и др.


За да го използвате, първо трябва да го импортирате в горната част на файла:


```javascript
const fs = require("fs")
```


### Синхронизация API


Най-простият начин за използване на `fs` е чрез неговите методи **sync**.


Тези методи блокират програмата, докато не приключат (така че следващият ред код не се изпълнява, докато операцията не приключи).


Ето един пример за синхронно четене на файл:


```javascript
const fs = require("fs")

const data = fs.readFileSync("example.txt", "utf8")

console.log(data)
```


Ако в същата директория, в която се намира скриптът, има файл, наречен `example.txt`, ще отпечатате съдържанието му.


Можете също така да записвате в даден файл синхронно:


```javascript
const fs = require("fs")

fs.writeFileSync("output.txt", "Hello from Node!")

console.log("File written!")
```


Това създава (или презаписва) файл, наречен `output.txt`, с текста.


Ето някои обичайни операции, които можете да извършвате с този API:


```javascript
const fs = require("fs")

// List files and folders
const items = fs.readdirSync(".")
console.log("Items in current directory:", items)

// Create folder
fs.mkdirSync("my_folder")
console.log("Folder created")

// Delete folder
fs.rmdirSync("my_folder")
console.log("Folder deleted")

// Create & write file
fs.writeFileSync("my_file.txt", "Hello world")
console.log("File created & written")

// Read file
const content = fs.readFileSync("my_file.txt", "utf8")
console.log("File content:", content)

// Delete file
fs.unlinkSync("my_file.txt")
console.log("File deleted")
```


Синхронизаторът API е прост и добър за малки скриптове, но тъй като блокира програмата, докато тя не приключи, може да забави работата, ако работите с големи файлове или сървър.


### Асинхронно извикване на обратна връзка API


Обратната връзка API** е неблокираща: тя позволява на NodeJS да продължи да прави други неща, докато се извършва файловата операция.


Вместо да връща директно резултата, тя взема функция (**обратна връзка**), която се извиква, когато операцията приключи.


```javascript
const fs = require("fs")

fs.readFile("example.txt", "utf8", (err, data) => {
if (err) {
console.error("Error reading file:", err)
} else {
console.log(data)
}
})
```


Ето какво се случва:


`fs.readFile` започва да чете `example.txt`.

*NodeJS не чака, а продължава да изпълнява друг код, който може да сте написали.*

*Когато файлът приключи с четенето, се изпълнява обратната връзка:*



  - Ако е възникнала грешка, `err` съдържа грешката.
  - В противен случай `data` съдържа съдържанието.


Ето как се записва във файл:


```javascript
const fs = require("fs")

fs.writeFile("output.txt", "Hello async!", (err) => {
if (err) {
console.error("Error writing file:", err)
} else {
console.log("File written!")
}
})
```


Същата идея: програмата не спира, докато записва файла.


Няколко примера за неща, които можете да правите с този API:


```javascript
const fs = require("fs")

// List files and folders
fs.readdir(".", (err, items) => {
if (err) return console.error(err)
console.log("Items in current directory:", items)
})

// Create folder
fs.mkdir("my_folder", (err) => {
if (err) return console.error(err)
console.log("Folder created")
})

// Delete folder
fs.rmdir("my_folder", (err) => {
if (err) return console.error(err)
console.log("Folder deleted")
})

// Create & write file
fs.writeFile("my_file.txt", "Hello world", (err) => {
if (err) return console.error(err)
console.log("File created & written")
})

// Read file
fs.readFile("my_file.txt", "utf8", (err, content) => {
if (err) return console.error(err)
console.log("File content:", content)
})

// Delete file
fs.unlink("my_file.txt", (err) => {
if (err) return console.error(err)
console.log("File deleted")
})
```


Обратното извикване API е по-добро за сървъри и големи задачи, тъй като не блокира програмата, но вложените обратни извиквания могат да станат объркани, ако верижно изпълнявате много операции. Ето защо беше добавена асинхронна API, базирана на обещания.


### Обещание async API


Базираният на обещания API е модерен и работи чудесно с `.then()` и `async/await`. Той е достъпен като `fs.promises`.


Трябва да импортирате свойството `promises`:


```javascript
const fs = require("fs").promises
```


Използване на `.then()`:


```javascript
const fs = require("fs").promises

fs.readFile("example.txt", "utf8")
.then(data => {
console.log(data)
})
.catch(err => {
console.error("Error reading file:", err)
})
```


Или още по-добре - с помощта на `async/await`:


```javascript
const fs = require("fs").promises

async function readFile(fileName) {
try {
const data = await fs.readFile(fileName, "utf8")
console.log(data)
} catch (err) {
console.error("Error reading file:", err)
}
}

readFile("example.txt")
```


Записване във файл:


```javascript
const fs = require("fs").promises

async function writeFile(fileName, content) {
try {
await fs.writeFile(fileName, content)
console.log("File written!")
} catch (err) {
console.error("Error writing file:", err)
}
}

writeFile("output.txt", "Hello from promises!")
```


Обичайният списък с примери за API:


```javascript
const fs = require("fs").promises

// Use an async function to await operations
async function main() {
// List files and folders
const items = await fs.readdir(".")
console.log("Items in current directory:", items)

// Create folder
await fs.mkdir("my_folder")
console.log("Folder created")

// Delete folder
await fs.rmdir("my_folder")
console.log("Folder deleted")

// Create & write file
await fs.writeFile("my_file.txt", "Hello world")
console.log("File created & written")

// Read file
const content = await fs.readFile("my_file.txt", "utf8")
console.log("File content:", content)

// Delete file
await fs.unlink("my_file.txt")
console.log("File deleted")
}

main().catch(err => console.error(err))
```



## NPM

<chapterId>a91d9a75-55cc-51a3-a48f-0c0be6fe6e72</chapterId>


Когато пишете код , често ви се налага да използвате код, написан от други хора; например библиотеки, които ви помагат да работите с дати, цветове, сървъри или почти всичко друго.


Вместо да изтегляте и копирате файлове ръчно, можете да използвате **пакетния мениджър**.


Пакетният мениджър е инструмент, който:


*изтегляне на пакети*

*следи кои пакети са необходими на вашия проект*

**уверете се, че всички в екипа ви разполагат с еднакви версии на пакетите**


### Какво е NPM


В света на NodeJS най-популярният мениджър на пакети е **NPM**, което означава *Node Package Manager*.


Получавате NPM автоматично, когато инсталирате NodeJS.


Можете да проверите дали разполагате с NPM, като стартирате това в терминала:


```
npm -v
```


Това отпечатва версията на NPM, която имате, например:


```
10.2.1
```


### Създаване на пакет


В NodeJS **пакетът** е просто директория с файл `package.json` в нея.


Нека да създадем една стъпка по стъпка.


1. Създайте нова папка за вашия проект:


```
mkdir my_project
cd my_project
```


2. Изпълнете тази команда:


```
npm init
```


Започва интерактивна настройка, която ви задава въпроси като името на проекта, версията, описанието и т.н.


Ако не искате да отговаряте на всички въпроси, а просто да приемете настройките по подразбиране, можете да използвате:


```
npm init -y
```


След като го стартирате, в папката ви ще се появи нов файл, наречен:


```
package.json
```


### package.json


Файлът `package.json` е просто JSON файл, в който се съхраняват метаданни за вашия проект.


Ето един пример:


```json
{
"name": "my_project",
"version": "1.0.0",
"description": "",
"main": "index.js",
"scripts": {
"test": "echo \"Error: no test specified\" && exit 1"
},
"keywords": [],
"author": "",
"license": "ISC",
"type": "commonjs"
}

```


Няколко важни области:



- `name`: името на вашия пакет
- `version`: текущата версия
- `main`: входният файл (като `index.js`)
- `scripts`: команди, които можете да изпълнявате (като `npm start`)
- `зависимости`: изброява всички пакети, от които зависи проектът ви


### Инсталиране на пакет


Да речем, че искате да използвате определен пакет, наречен `picocolors`, за да добавите цветове към изхода на терминала.


Можете да го инсталирате, като стартирате:


```
npm install picocolors
```


Вече можете да го използвате в проекта си. Създайте файл `index.js` с


```javascript
const pico = require('picocolors')

console.log(
pico.green("This text is green!")
)
```


и се опитайте да го стартирате. Терминалът трябва да отпечата цветна версия на текста.


Какво направи НПМ?


*Той изтегли пакета и го съхрани в подпапка, наречена `node_modules/`*


- добави запис в `зависимости` във вашия `пакет.json`
- тя актуализира файла `package-lock.json`


Какво е `package-lock.json` ?


### package-lock.json


Този файл се създава автоматично от NPM.


Може би се чудите, че ако вече имаме `package.json`, защо ни е нужен още един файл?

Ето каква е причината:



- в `package.json` се посочва само от коя версия на пакета се нуждае проектът ви.

Пример:


```json
"dependencies": {
"picocolors": "^1.1.0"
}
```


`^1.1.0` означава "всяка версия, която е съвместима с 1.1.x", така че е гъвкава.


`package-lock.json` **замразява** точните версии на всеки отделен пакет и техните подзависимости, така че всеки, който инсталира проекта ви, да получи абсолютно същата работна конфигурация.


Защо това е важно?


Ако работите в екип, внедрявате проекта си на сървър или се връщате към него в бъдеще, искате да сте сигурни, че той продължава да работи по същия начин.

Ако пакетите са били актуализирани и преинсталирате без заключващ файл, може да получите малко по-различна версия, която се държи по различен начин.


Като запазите файла `package-lock.json` в проекта си, NPM винаги ще инсталира точно посочените в него версии, за да гарантира, че всички имат една и съща среда.


`package-lock.json` заключва всичко до много специфична версия, за да направи проекта по-възпроизводим на други машини.


Но ако вашият пакет трябва да се използва от много хора, може да изберете да не го предавате, така че NPM да намери само файла `package.json` и да му позволи да инсталира автоматично най-новите версии, които са разрешени в този файл.


Но това са неща, за които трябва да се притеснявате по-късно, когато започнете да публикувате собствения си код. Засега представихме основите на NPM само за да можете да намирате и използвате библиотеките, публикувани от други разработчици, в своите проекти.



## Работа в мрежа в NodeJS

<chapterId>f2cabd8b-754b-5c97-8d6a-8412a9a184c7</chapterId>


NodeJS често се използва като език за бекенд: можете да превърнете своя скрипт в сървър и да го използвате за подаване на заявки към други сървъри.


В тази глава ще ви представим някои основни мрежови функции, които ще ви позволят да направите това.


### извличане()


Ако искате програмата ви да изтегли данни от уебсайт или API, трябва да направите **HTTP заявка**.


В съвременните версии на NodeJS можете да използвате вградената функция `fetch()`.


Ето един пример за извършване на заявка GET към API:


```javascript
const response = await fetch("https://jsonplaceholder.typicode.com/posts/1")
const data = await response.text()

console.log(data)
```


Когато го стартирате, ще видите нещо подобно на:


```
{
"userId": 1,
"id": 1,
"title": "...",
"body": "..."
}
```


Ето какво се случва:


1. `fetch()` приема URL адрес и прави заявка.

2. Той връща **Победа**, която се преобразува в обект `Отговор`.

3. `response.text()` прочита тялото на отговора като низ.


Но низът, който получавате обратно, всъщност е JSON. Какво е това?


### JSON


Когато работите с уеб API, данните често се изпращат и получават като **JSON**, което означава JavaScript Object Notation.


JSON е просто текстов формат, който много прилича на обекти в JavaScript. Например:


```json
{
"name": "Alice",
"age": 30,
"likes": ["apples", "bananas"]
}
```


Обектът `JSON` е вградена помощна програма в JavaScript, която може да се използва за работа с този формат данни.


Можете да преобразувате обект на JavaScript в JSON низ с помощта на `JSON.stringify()`:


```javascript
const user = { name: "Alice", age: 30 }

const jsonString = JSON.stringify(user)

console.log(jsonString)
```


Този печат:


```
{"name":"Alice","age":30}
```


Можете също така да преобразувате JSON текст обратно в обект на JavaScript, като използвате `JSON.parse()`:


```javascript
const jsonText = '{"name":"Bob","age":25}'

const obj = JSON.parse(jsonText)

console.log(obj)
```


Този печат:


```
{ name: 'Bob', age: 25 }
```


Бъдете внимателни: `JSON.parse()` ще хвърли грешка, ако низът не е валиден JSON.


```javascript
JSON.parse("not json") // ❌ Error!
```


Затова се уверете, че низът е правилно форматиран.


### http сървър


NodeJS ви позволява да създадете уеб сървър, без да инсталирате нищо друго.


Можете да използвате вградения модул `http`, за да обработвате заявки от клиенти и да изпращате отговори.


Ето един много елементарен пример:


```javascript
const http = require("http")

const server = http.createServer((req, res) => {
res.statusCode = 200
res.end("Hello from NodeJS server!")
})

server.listen(3000, () => {
console.log("Server running at http://localhost:3000/")
})
```


Когато стартирате този скрипт и отворите `http://localhost:3000` в браузъра си, ще видите:


```
Hello from NodeJS server!
```


Ето какво се случва в кода:


1. Импортирали сте сървъра `http` от стандартната библиотека на Node.

2. `http.createServer()` създава сървър. Предадохте на `http.createServer()` обратна връзка `(req, res) => {...}`, която да се изпълнява всеки път, когато някой се свърже.

3. На отговора е зададен код на състоянието 200 (който показва успешна операция). Можете да прочетете за кодовете на състоянието на http [тук](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Status)

3. `res.end()` изпраща отговора и прекратява връзката.

4. `server.listen(3000)` стартира сървъра на порт 3000.


Можете също така да отбележите `req.url` и `req.method` в заявката, за да обработвате различни пътища или типове заявки.


Пример с маршрутизация:


```javascript
const server = http.createServer((req, res) => {
if (req.url === "/") { // handle requests for the root of the website
res.statusCode = 200
res.end("Home page")
} else if (req.url === "/about") { // handle requests for the about page
res.statusCode = 200
res.end("About page")
} else {
res.statusCode = 404 // we send a 404 status code to signal that the requested page is missing
res.end("Not Found")
}
})
```


Това са съвсем елементарни примери. За изграждането на по-усъвършенствани сървъри повечето разработчици вероятно ще изтеглят готова бекенд библиотека като [express](https://www.npmjs.com/package/express).


## Обработка на данни: буфери, събития, потоци

<chapterId>8c9623f0-a604-51a4-8fe4-871c849d3e3b</chapterId>


В тази глава ще представим основно три класа обекти:



- `Buffer`, който представлява малки части от двоични данни
- `EventEmitter`, който може да се използва за проследяване на асинхронен процес чрез излъчване на сигнали, наречени "събития"
- `Stream`, който ни позволява да обработваме голяма част от данните по един `Buffer` в даден момент и който проследява процеса чрез излъчване на събития


Те са изключително често срещани в професионалния код на NodeJS, така че дори и да не ги използвате в първите си проекти, е добре да придобиете основна представа за това кога ще ви се наложи да взаимодействате с тях


### Буфери


В NodeJS **буфер** е тип обект, използван за работа с двоични данни.


Можете да си представите буфера като контейнер с фиксиран размер за необработени байтове.


Ето как да създадете буфер от низ:


```javascript
const buf = Buffer.from("hello")
console.log(buf)
```


Отпечатва се нещо подобно на:


```
<Buffer 68 65 6c 6c 6f>
```


Тези цифри (`68`, `65`, `6c` и т.н.) са шестнадесетични представяния на буквите в `"hello"`.


Можете да го превърнете обратно в низ по следния начин:


```javascript
console.log(buf.toString())
```


Този печат:


```
hello
```


Можете също така да създадете буфер с определен размер, запълнен с нули:


```javascript
const buf = Buffer.alloc(10)
console.log(buf)
```


Отпечатва се нещо подобно на:


```
<Buffer 00 00 00 00 00 00 00 00 00 00>
```


Можете да записвате в буфера:


```javascript
buf.write("abc")
console.log(buf)
```


Можете да получите достъп до отделни байтове:


```javascript
console.log(buf[0]) // prints the ASCII number for 'a', which is 97
```


Буферите са особено полезни, когато трябва да манипулирате данни на много ниско ниво.


### Събития


В JavaScript **събитие** е нещо, което се случва във вашата програма и на което можете да реагирате.


Например:


*завършване на зареждането на даден файл*

*таймерът изключва*

*потребител щраква върху бутон*

*мрежова заявка връща данни*


Едно **събитие** е просто сигнал, че нещо се е случило, и можете да напишете код, който да слуша тези събития и да реагира на тях.


В NodeJS много обекти могат да излъчват събития. Тези обекти се наричат **EventEmitters**.


Ето един пример:


```javascript
const EventEmitter = require("events")

const emitter = new EventEmitter()

// Listen for an event
emitter.on("greet", () => {
console.log("Hello! An event happened.") // this will get printed when a "greet" event gets fired
})

// Emit the event
emitter.emit("greet")
```


Този печат:


```
Hello! An event happened.
```


Ето какво:


1. Създаваме обект `EventEmitter`.

2. Казваме му да изпълнява обратна връзка, когато се случи събитието `"greet"`, като използваме `.on("greet")`.

3. По-късно задействаме събитието `"greet"` с помощта на `.emit()`.

4. Изпълнява се нашето обратно извикване


Можете да предавате данни заедно със събитието:


```javascript
emitter.on("greet",
(name) => console.log(`Hello, ${name}!`)
)

emitter.emit("greet", "Alice") // first argument is the type of event, second argument is the data we pass with this event
```


Този печат:


```
Hello, Alice!
```


Можете да регистрирате слушатели и за други събития:


```javascript
emitter.on("goodbye", () => {
console.log("Goodbye!")
})

emitter.emit("goodbye")
```


Можете да прикачите колкото искате слушатели към даден тип събитие и да задействате много различни типове събития от един и същ излъчвател.


Много обекти в NodeJS излъчват събития, за да съобщят на останалата част от програмата, че нещо се случва.



### Какво представляват потоците?


Потоците комбинират буфери и събития, за да ни помогнат да обработваме данни.


Когато работим с файлове, данни от мрежата или дори с дълъг текст, невинаги е необходимо (или искаме) да зареждаме всичко в паметта наведнъж. Това може да бъде бавно или дори да доведе до срив на програмата, ако данните са твърде големи.


Вместо това можем да обработваме данните **постепенно**, докато те пристигат или се четат, подобно на пиене на вода през сламка, вместо да се опитваме да изпием цялата чаша наведнъж. Това се нарича **поток**.


В NodeJS потокът е обект, който ви позволява да четете данни от източник или да записвате данни към дестинация **поединично**.


NodeJS има четири основни типа потоци:



- Readable**: потоци, от които можете да четете данни (като четене на файл)

**Writable**: потоци, в които можете да записвате данни (като запис във файл)

**Дуплексни**: потоци, които могат да се четат и записват едновременно

**Трансформиране**: като двустранните потоци, но те могат да променят (трансформират) данните по време на движението им


### Потоци за четене


Да речем, че имате за обработка файл `bigfile.txt`. Можете да създадете четим поток с модула `fs`, за да прочетете файла парче по парче.


```javascript
const fs = require("fs")

const readableStream = fs.createReadStream(
"bigfile.txt"
)

readableStream.on("data", (chunk) => {
console.log("Received chunk:", chunk)
})

readableStream.on("end", () => {
console.log("Finished reading file.")
})

readableStream.on("error", (err) => {
console.error("Error reading file:", err)
})
```


Какво се случва тук?


1. `fs.createReadStream()` създава поток за четене.

2. Всеки път, когато част от файла е готова, потокът излъчва събитие `data` и ни предоставя "парче" от данни (`Buffer`). Отпечатваме парчето.

3. Когато целият файл е прочетен, се задейства събитието `end`.

4. Ако има грешка (например файлът не съществува), се задейства събитието `error`.


По този начин можете да четете гигантски файлове, без да ги зареждате всички наведнъж в паметта.


Ако искаме данните да пристигнат в разбираема за човека форма (вместо в двоичен вид), можем да посочим кодирането на потока:


```javascript
const fs = require("fs")

const readableStream = fs.createReadStream(
"bigfile.txt",
{ encoding: "utf8" } // we tell NodeJS that the file should be read as utf8
)

readableStream.on("data", (chunk) => {
console.log("Received chunk:", chunk)
})

readableStream.on("end", () => {
console.log("Finished reading file.")
})

readableStream.on("error", (err) => {
console.error("Error reading file:", err)
})
```


Сега кодът ще отпечатва файла във вид, удобен за четене от човека.


### Потоци, които могат да се записват


Записваемият поток ви позволява да изпращате данни някъде, парче по парче.


Ето един пример за запис във файл `target.txt` с помощта на поток:


```javascript
const fs = require("fs");

const stream = fs.createWriteStream("target.txt");

stream.on("error", (err) => {
  console.error("Error:", err);
});

stream.on("finish", () => {
  console.log("All data written.");
});

stream.write("First line\n");
stream.write("Second line\n");
stream.end("Finished writing\n");
```


Ето какво се случва:

1. `fs.createWriteStream()` създава поток за запис.
2. Регистрираме манипулатори за събитията `error` и `finish`.
3. Записваме текст в него чрез `.write()`.
4. Когато приключим, извикваме `.end()`, за да затворим потока.
5. След като всички буферирани данни бъдат изчистени и записани, се излъчва събитието `finish`. Ако нещо се обърка, се излъчва събитието `error`.


Подобно на потоците с възможност за четене, потоците с възможност за запис са подходящи за големи обеми от данни, тъй като не е необходимо да съхраняват всичко в паметта наведнъж.


### Тръбни потоци


Едно от най-готините неща при потоците е, че можете да ги свързвате по **проводи**: да свържете поток за четене директно към поток за запис.


```javascript
const fs = require("fs")

const readable = fs.createReadStream("bigfile.txt")
const writable = fs.createWriteStream("target.txt")

readable.pipe(writable)
```


Тук:


*Потокът за четене се чете от `bigfile.txt`.*

*Записваемият поток записва в `copy.txt`.*


- `.pipe()` изпраща данните директно от четящия към записващия поток.


### Двустранни потоци


Дуплексният поток може да се чете и записва. Пример за това е мрежовият сокет: можете да изпращате данни към него и да получавате данни от него.


Ето един много прост пример, в който се използва модулът `net`:


```javascript
const net = require("net")

const server = net.createServer((socket) => {
socket.write("Welcome!\n")

socket.on("data", (chunk) => {
console.log("Received:",
chunk.toString()  // we convert the chunk of data from Buffer to string
)
})
})

server.listen(3000, () => {
console.log("Server listening on port 3000")
})
```


В този пример:


*Обектът `socket` е дуплексен поток.*

*Можете да `записвате()` в него, както и да слушате за събития `данни` от него.*


### Трансформиране на потоци


Трансформиращият поток е двустранен поток, който също така модифицира данните, които преминават през него.


Например можете да използвате вградения модул `zlib`, за да компресирате или декомпресирате данни.


Ето как да компресирате файл с помощта на трансформиращ поток:


```javascript
const fs = require("fs")
const zlib = require("zlib")

const readable = fs.createReadStream("bigfile.txt")     // create a readable stream that reads from a file
const zip = zlib.createGzip()                           // create a transform stream that compresses data
const writable = fs.createWriteStream("bigfile.txt.gz") // create a writable stream that writes to a file

readable          // take the readable stream
.pipe(zip)      // pipe it into the transform stream to compress the data
.pipe(writable) // then pipe it into the writable stream that saves the data to a zipped file

writable.on("finish", () => {
console.log("File compressed.")
})
```


И да го декомпресирате обратно:


```javascript
const readable = fs.createReadStream("bigfile.txt.gz")
const unzip = zlib.createGunzip()
const writable = fs.createWriteStream("bigfile.txt")

readable.pipe(unzip).pipe(writable)

writable.on("finish", () => {
console.log("File decompressed.")
})
```


Трансформиращите потоци са много полезни за задачи като компресиране, криптиране или промяна на файловите формати по време на стрийминг.


### Противоналягане


Понякога записваемият поток е бавен при обработката на данни. Ако продължим да изпращаме данни към записваемия поток по-бързо, отколкото той може да обработи, може да се сблъскаме с проблеми. Това се нарича **напрежение**.


Ако извикате метода `.write()` на записваем поток, той връща булева стойност, която ви информира дали потокът се нуждае от пауза; може да се наложи да проверите върнатата стойност, както е показано тук:


```javascript
const fs = require("fs")

const readable = fs.createReadStream("example.txt")
const writable = fs.createWriteStream("copy.txt")
r
readable.on("data", chunk => {               // each chunk of data we read from the readable stream...

const canContinue = writable.write(chunk)  // ...we send it to the writable, which returns us a boolean to confirm we can continue

if (!canContinue) { readable.pause() }     // ...if we can't, we temporarily pause reading
})

writable.on("drain",                // the writable stream emits a "drain" event when the backpressure is gone

() => { readable.resume() }      // so we resume reading (and writing)

)
```


Това беше илюстративен пример за ръчно прехвърляне на данни от устройство за четене към устройство за писане и ръчно управление на обратното налягане.


Обикновено предаваме данните по тръба, като използваме метода `.pipe()`, който се справя автоматично с обратното налягане.


Така че трябва да се притеснявате за обратното налягане само когато по някаква причина ръчно извиквате `.write()`.
