# अपनी विकास यात्रा शुरू करें


जावास्क्रिप्ट और नोडजेएस पर इस पाठ्यक्रम में आपका स्वागत है।


जावास्क्रिप्ट दुनिया की सबसे लोकप्रिय प्रोग्रामिंग भाषा है: यह आधुनिक ब्राउज़रों की स्क्रिप्टिंग भाषा है, इसलिए *कुछ* जावास्क्रिप्ट लिखे बिना आधुनिक वेब एप्लिकेशन बनाना मूलतः असंभव है; और NodeJS रनटाइम के साथ इसका उपयोग ब्राउज़रों के बाहर भी किया जा सकता है, ताकि स्क्रिप्ट और एप्लिकेशन बनाए जा सकें जो सीधे आपके कंप्यूटर पर चलते हैं।


यह पाठ्यक्रम उन लोगों के लिए डिज़ाइन किया गया है जो प्रोग्रामिंग में बिल्कुल नए हैं, या जिन्होंने पहले अन्य भाषाओं का उपयोग किया है, लेकिन यह समझना चाहते हैं कि जावास्क्रिप्ट कैसे काम करता है, विशेष रूप से NodeJS के संदर्भ में।


पाठ्यक्रम के अंत तक, आप जावास्क्रिप्ट में अपने स्वयं के प्रोग्राम लिखने, नोडजेएस मानक लाइब्रेरी का उपयोग करने, तथा उपयोगी उपकरण बनाने के लिए तृतीय-पक्ष पैकेजों को स्थापित करने और उनका उपयोग करने में सक्षम हो जाएंगे।

# मूल जावास्क्रिप्ट

## स्थापित करना

<chapterId>ba05a290-1782-5268-87c9-62fd09590e05</chapterId>


इस अनुभाग में हम अपनी मशीन को अपना पहला जावास्क्रिप्ट प्रोग्राम लिखने और निष्पादित करने के लिए सेटअप करेंगे।


जावास्क्रिप्ट प्रोग्राम केवल (एक या अधिक) टेक्स्ट फाइलों का एक संग्रह है, जिसमें जावास्क्रिप्ट रनटाइम द्वारा निष्पादित किए जाने वाले कमांड होते हैं।


इन टेक्स्ट फ़ाइलों के नाम आमतौर पर `.js` फ़ाइल एक्सटेंशन के साथ समाप्त होते हैं, जैसे `my_script.js`, `my_program.js` आदि।


इनमें निहित आदेश जावास्क्रिप्ट प्रोग्रामिंग भाषा में लिखे गए हैं।


जावास्क्रिप्ट रनटाइम एक विशेष प्रोग्राम है जो इन फ़ाइलों को निष्पादित करता है।


![](assets/en/1.webp)


### NodeJS स्थापना


सबसे आम जावास्क्रिप्ट रनटाइम NodeJS है।


आप इसे [आधिकारिक निर्देशों](https://nodejs.org/en/download) का पालन करके स्थापित कर सकते हैं।


डाउनलोड पृष्ठ आपको तीनों प्रमुख ऑपरेटिंग सिस्टम (ऑपरेटिंग सिस्टम): विंडोज, लिनक्स और मैकओएस के लिए निर्देश प्रदान करेगा। यह मानकर चला जाता है कि आपको अपने ऑपरेटिंग सिस्टम में टर्मिनल खोलना आता है।


चूंकि NodeJS तीनों ऑपरेटिंग सिस्टम के लिए उपलब्ध है, इसलिए आपके द्वारा लिखे गए प्रोग्राम उन सभी पर निष्पादित किए जा सकेंगे (कुछ एज केस को छोड़कर)।


इसका मतलब यह है कि, उदाहरण के लिए, आप अपने विंडोज पीसी पर जावास्क्रिप्ट में एक सरल वीडियो गेम लिख सकते हैं और इसे अपने मित्र को उसके मैक पर चलाने के लिए दे सकते हैं।


![](assets/en/2.webp)


### पाठ संपादन


प्रोग्रामिंग के बारे में एक अच्छी बात यह है कि आप किसी भी टेक्स्ट एडिटर का उपयोग करके कोड लिख सकते हैं, यहां तक ​​कि अपने ऑपरेटिंग सिस्टम के डिफॉल्ट नोटपैड का भी उपयोग कर सकते हैं।


हालांकि कुछ टेक्स्ट एडिटर ऐसे हैं जो कोड लिखने के लिए विशेषीकृत हैं, कुछ निःशुल्क उपलब्ध हैं, जबकि अन्य के लिए आपको लाइसेंस के लिए भुगतान करना पड़ता है।


कोड एडिटर चुनना एक बहुत बड़ा सवाल है जो इस कोर्स के दायरे से बाहर है, इसलिए हम यहाँ इस पर बात नहीं करेंगे। अगर आपको नहीं पता कि क्या इस्तेमाल करना है, तो सबसे ज़्यादा इस्तेमाल किया जाने वाला मुफ़्त एडिटर [VSCode](https://code.visualstudio.com/) है।


इसका Interface थोड़ा बड़ा ज़रूर है, लेकिन इसमें वो सब कुछ है जो आपको चाहिए: एक फ़ाइल एडिटर, एक फ़ाइल एक्सप्लोरर (जिस डायरेक्टरी में आप काम कर रहे हैं, उसकी फ़ाइलों और सबडायरेक्टरीज़ को विज़ुअलाइज़ करने के लिए), और अपना कोड चलाने के लिए एक टर्मिनल। यह कई प्लगइन्स को भी सपोर्ट करता है, और इसमें डिफ़ॉल्ट रूप से जावास्क्रिप्ट सिंटैक्स हाइलाइटिंग भी है।


यदि आप थोड़ा अधिक Cypherpunk-y बनना चाहते हैं, तो आप इसके बजाय [VSCodium](https://vscodium.com/) का उपयोग कर सकते हैं।


### पहला प्रोग्राम (हैलो वर्ल्ड)


परंपरागत रूप से, जब कोई प्रोग्रामिंग भाषा सीखता है, तो पहला प्रोग्राम जो लिखा जाता है, वह कंसोल पर "hello world!" प्रिंट करना होता है।


`my_js_code/` नामक एक निर्देशिका बनाएं, जिसके अंदर `main.js` नामक एक फ़ाइल हो (ये नाम मनमाने हैं)।


VSCode के साथ निर्देशिका खोलें.


इस कोड को अपनी फ़ाइल में लिखें:


```javascript
console.log("hello world!")
```


टर्मिनल खोलें और प्रोग्राम चलाने के लिए इस कमांड को निष्पादित करें:


```
node main.js
```


परिणाम यह होना चाहिए


```
hello world!
```


### क्या हुआ


जावास्क्रिप्ट में, सब कुछ एक "ऑब्जेक्ट" है।


`कंसोल` एक ऑब्जेक्ट है, जिसका उपयोग प्रोग्राम को डीबग करने के लिए किया जाता है।


`console.log`, `console` का सबसे ज़्यादा इस्तेमाल किया जाने वाला तरीका है। यह बस वही तर्क प्रिंट करता है जो आप इसमें पास करते हैं।


आप `console.log` को गोल कोष्ठक `()` का उपयोग करके तर्क पास करते हैं।


उदाहरण के लिए, यदि आप संख्या `1000` प्रिंट करना चाहते हैं, तो आप बस लिखेंगे


```javascript
console.log(1000)
```


फिर इसे चलाकर निष्पादित करें


```
node main.js
```


आपके टर्मिनल में (अब से, यह पाठ्यक्रम यह मान लेगा कि आप जानते हैं कि इस तरह से आप प्रोग्राम को निष्पादित करते हैं)।


यह प्रिंट होना चाहिए


```
1000
```


आप कई चीजें पास कर सकते हैं, जैसे


```javascript
console.log(16, 8, 1993)
```


यह प्रिंट होगा


```
16 8 1993
```


## चर और टिप्पणियाँ

<chapterId>23050ab7-343b-5edf-9d37-e4e782e27ce0</chapterId>


प्रोग्राम आमतौर पर डेटा पर ऑपरेशन निष्पादित करते हैं।


वेरिएबल्स नामित बॉक्स की तरह होते हैं जिनका उपयोग हम डेटा संग्रहीत करने के लिए करते हैं। ये हमें डेटा के एक हिस्से को एक विशिष्ट नाम से जोड़ने की अनुमति देते हैं, ताकि हम बाद में उस नाम का उपयोग करके उसे पुनः प्राप्त कर सकें।


### `let` घोषणाएँ


जावास्क्रिप्ट में एक चर घोषित करने के लिए, हम `let` कीवर्ड का उपयोग कर सकते हैं।


`let` लिखने के बाद, हम वह नाम लिखते हैं जो हम चर को देना चाहते हैं, फिर `=` चिह्न, और फिर वह मान जिसे हम संग्रहीत करना चाहते हैं।


उदाहरण के लिए:


```javascript
let age = 25

console.log(age)
```


किसी चर का नाम (तकनीकी रूप से "पहचानकर्ता" कहा जाता है) में आमतौर पर अक्षर, अंडरस्कोर (`_`), डॉलर चिह्न (`$`) और संख्याएं शामिल हो सकती हैं, हालांकि पहला अक्षर संख्या नहीं हो सकता है।


उपरोक्त कोड में, हमने `age` नामक एक चर घोषित किया है और इसमें `25` मान संग्रहीत किया है।


फिर, हमने `console.log(age)` का उपयोग करके मान मुद्रित किया।


यदि आप इस कोड को `node main.js` के साथ चलाते हैं, तो आउटपुट होगा:


```
25
```


पहचानकर्ता केस-सेंसिटिव होते हैं, जिसका अर्थ है कि लोअर और अपर केस को पहचानकर्ताओं में अंतर के रूप में गिना जाता है, उदाहरण के लिए


```javascript
let age = 25

let Age = 20

console.log(age)
```


25 प्रिंट करेगा, क्योंकि उन्हें दो पूरी तरह से अलग चर माना जाता है!


आप स्ट्रिंग्स (टेक्स्ट) को एक वेरिएबल में भी संग्रहीत कर सकते हैं:


```javascript
let message = "hello again"

console.log(message)
```


यह प्रिंट करेगा:


```
hello again
```


पहले की तरह, हमने वेरिएबल में संग्रहीत मान को प्रिंट करने के लिए `console.log()` का उपयोग किया।


अब आइए दोनों काम एक साथ करें:


```javascript
let age = 25

let message = "hello again"

console.log(age)

console.log(message)
```


इसे चलाने पर प्रिंट होगा:


```
25
hello again
```

### रीअसाइनमेंट


`let` के साथ घोषित चरों को उनके निर्माण के बाद बदला जा सकता है।


इसे पुनःनिर्धारण कहा जाता है।


```javascript
let score = 10

console.log(score)

score = 15

console.log(score)
```


सबसे पहले, हम `score` को `10` असाइन करते हैं, फिर उसे प्रिंट करते हैं।


फिर हम `score` का मान `15` में बदलते हैं और इसे फिर से प्रिंट करते हैं।


आउटपुट होगा:


```
10
15
```


यह तब बहुत उपयोगी होता है जब समय के साथ मूल्य बदलता है, जैसे किसी खेल में जहां स्कोर बढ़ता है।


आइये इस मिश्रण में एक और चर जोड़ें:


```javascript
let score = 10
let player = "Alice"

console.log(score)
console.log(player)

score = 20
player = "Bob"

console.log(score)
console.log(player)
```


यह प्रिंट करेगा:


```
10
Alice
20
Bob
```


जैसा कि आप देख सकते हैं, `स्कोर` और `खिलाड़ी` दोनों बदल दिए गए हैं।


### `const` घोषणाएँ


हालाँकि, ज़्यादातर मामलों में हम नहीं चाहते कि कोई वेरिएबल बनने के बाद उसमें कोई बदलाव हो। इसके लिए हम `const` का इस्तेमाल करते हैं।


`const`, “constant” का संक्षिप्त रूप है। एक बार जब आप किसी `const` वेरिएबल को मान निर्दिष्ट कर देते हैं, तो आप उसे बदल नहीं सकते।


```javascript
const pi = 3.14
console.log(pi)
```


यह प्रिंट करता है:


```
3.14
```


लेकिन अगर आप ऐसा करने की कोशिश करते हैं:


```javascript
const pi = 3.14
console.log(pi)

pi = 99 // this line will cause an error
console.log(pi)
```


जावास्क्रिप्ट आपको इस प्रकार की त्रुटि देगा:


```
TypeError: Assignment to constant variable.
```


ऐसा इसलिए है क्योंकि `pi` को `const` का उपयोग करके घोषित किया गया था, और उसके बाद आप इसका मान नहीं बदल सकते। आप जावास्क्रिप्ट इंटरप्रेटर को बता रहे हैं कि आप उस वेरिएबल को नहीं बदलना चाहते।


यह उपयोगी है क्योंकि इससे गलती से उसमें बदलाव की संभावना कम हो जाती है। जब प्रोग्राम बहुत बड़े हो जाते हैं, हज़ारों पंक्तियों के कोड के साथ, तो एक साथ होने वाली हर चीज़ पर नज़र रखना असंभव हो जाता है (यही मुख्य कारण है कि हम कंप्यूटर का इस्तेमाल उन जटिल प्रक्रियाओं को अंजाम देने के लिए करते हैं जिनकी गणना हम अपने दिमाग से नहीं कर सकते), इसलिए इस तरह के प्रतिबंध लगाना उपयोगी हो जाता है, जो प्रोग्राम को ज़्यादा नियतिवादी बनाते हैं।


यह सर्वोत्तम अभ्यास माना जाता है कि हम अपने मानों को हमेशा `const` के रूप में घोषित करें, जब तक कि हम यह सुनिश्चित न कर लें कि हम उन्हें बाद में संशोधित करना चाहते हैं।


### जावास्क्रिप्ट में टिप्पणियाँ


कभी-कभी हम अपने कोड में ऐसे नोट्स लिखना चाहते हैं जो निष्पादित नहीं होते। इन्हें टिप्पणियाँ कहते हैं।


जब प्रोग्राम चलता है तो टिप्पणियों को नजरअंदाज कर दिया जाता है, लेकिन वे स्वयं को या अन्य लोगों को चीजें समझाने के लिए उपयोगी होती हैं।


एकल-पंक्ति टिप्पणी लिखने के लिए, `//` का प्रयोग करें


```javascript
// This is a comment
const x = 10 // This is also a comment
console.log(x)
```


यह अभी भी प्रिंट करेगा:


```
10
```


टिप्पणियाँ केवल मनुष्यों के पढ़ने के लिए हैं।


आप `/*` और `*/` का उपयोग करके बहु-पंक्ति टिप्पणियाँ भी लिख सकते हैं


```javascript
/*
This is a multi-line comment.
It can span several lines.
*/
const y = 20
console.log(y)
```


यह प्रिंट होगा


```
20
```


और टिप्पणी को नजरअंदाज कर दिया जाएगा.


आप अपने कोड में छोटे-छोटे एनोटेशन जोड़ने के लिए कमेंट्स का इस्तेमाल कर सकते हैं, ताकि आपको याद रहे कि यह क्या करता है और इसे एक खास तरीके से क्यों लिखा गया है। इससे दूसरे प्रोग्रामर्स को भी इसे समझने में मदद मिल सकती है।


## मूल प्रकार: संख्याएँ, स्ट्रिंग्स, बूलियन

<chapterId>cfdb04f6-21a8-5143-bbf9-7aaae04962f0</chapterId>


जावास्क्रिप्ट में, एक "प्रकार" आपको बताता है कि कोई मान किस प्रकार का डेटा है।


जावास्क्रिप्ट के कुछ बुनियादी प्रकार हैं, और इस अनुभाग में हम उनमें से कुछ का पता लगाएंगे।


### संख्याएँ और अंकगणितीय संक्रियाएँ


पहला प्रकार जिसका हम परिचय कराने जा रहे हैं वह है `संख्या`।


जावास्क्रिप्ट में संख्याएं पूर्णांक (जैसे `5`) या दशमलव (जैसे `3.14`) हो सकती हैं।


आप उनके साथ अंकगणित कर सकते हैं: जोड़, घटाव, गुणा और भाग।


यहाँ एक बुनियादी उदाहरण है:


```javascript
const a = 10
const b = 5

const sum = a + b
const difference = a - b
const product = a * b
const quotient = a / b

console.log(sum)
console.log(difference)
console.log(product)
console.log(quotient)
```


यह प्रिंट करेगा:


```
15
5
50
2
```


आप परिचालनों के क्रम को नियंत्रित करने के लिए कोष्ठक `()` का भी उपयोग कर सकते हैं:


```javascript
const result = (2 + 3) * 4
console.log(result)
```


यह प्रिंट करता है:


```
20
```


कोष्ठक के बिना, यह `2 + 3 * 4` होगा, जो है:


```javascript
const result = 2 + 3 * 4
console.log(result)
```


वह प्रिंट करेगा:


```
14
```


क्योंकि सामान्य गणित में, गुणा, जोड़ से पहले होता है।


### स्ट्रिंग्स और इंटरपोलेशन


दूसरा जावास्क्रिप्ट प्रकार जिसे हम प्रस्तुत करने जा रहे हैं वह है `स्ट्रिंग`।


स्ट्रिंग्स, टेक्स्ट के टुकड़े होते हैं। इन्हें बनाने के लिए आप सिंगल कोट्स `'...'` या डबल कोट्स `"..."` का इस्तेमाल कर सकते हैं।


```javascript
const greeting = "hello"
const name = 'Bob'
console.log(greeting)
console.log(name)
```


यह प्रिंट करता है:


```
hello
Bob
```


स्ट्रिंग्स को संयोजित करने के लिए, आप `+` ऑपरेटर का उपयोग कर सकते हैं:


```javascript
const greeting = "hello"
const space = " "
const name = "Bob"

const fullGreeting = greeting + space + name
console.log(fullGreeting)
```


यह प्रिंट करेगा:


```
hello Bob
```


लेकिन स्ट्रिंग्स को संयोजित करने का एक बेहतर तरीका है जिसे **स्ट्रिंग इंटरपोलेशन** कहते हैं। आप स्ट्रिंग `` `...` `` को घोषित करने के लिए बैकटिक्स का उपयोग करते हैं और स्ट्रिंग के अंदर `${...}` का उपयोग करके वेरिएबल लिखते हैं:


```javascript
const greeting = "hello"
const name = "Bob"

const fullGreeting = `${greeting} ${name}`
console.log(fullGreeting)
```


यह भी प्रिंट करता है:


```
hello Bob
```


आप `${...}` के अंदर कोई भी अभिव्यक्ति शामिल कर सकते हैं:


```javascript
const age = 30
console.log(`Next year, I will be ${age + 1} years old.`)
```


यह प्रिंट करता है:


```
Next year, I will be 31 years old.
```


आधुनिक जावास्क्रिप्ट में इंटरपोलेशन बहुत आम है।


### बूलियन, तुलना और तर्क संचालन


तीसरा प्रकार जिसका हम परिचय कराने जा रहे हैं, वह है `बूलियन`। इसका नाम गणितज्ञ जॉर्ज बूल के नाम पर रखा गया है, जिन्होंने बूलियन तर्क का आविष्कार किया था।


बूलियन सरल हैं: केवल दो संभावित मान, `सत्य` और `झूठ`।


आप इन्हें वेरिएबल्स में संग्रहीत कर सकते हैं:


```javascript
const theSkyIsBlue = true
const thisCourseIsBad = false

console.log(theSkyIsBlue)
console.log(thisCourseIsBad)
```


यह प्रिंट करता है:


```
true
false
```


आप तर्क ऑपरेटरों का उपयोग करके बूलियन को संयोजित कर सकते हैं:



- `&&` का अर्थ है “और”, और यह केवल तभी `true` लौटाएगा जब **दोनों** मान `true` हों, अन्यथा यह `false` लौटाएगा
- `||` का अर्थ “या” है, और यह `सत्य` लौटाएगा यदि **कम से कम एक** मान `सत्य` है, अन्यथा (यदि वे दोनों असत्य हैं) यह `असत्य` लौटाएगा
- `!` का अर्थ है “नहीं”, यह बूलियन से पहले लागू होता है, और यह इसे पलट देगा: यदि बूलियन `सत्य` है तो यह `झूठ` लौटाएगा, और इसके विपरीत।


![](assets/en/3.webp)


उदाहरण:


```javascript
const isSunny = true
const isWarm = true

console.log(isSunny && isWarm)  // true
console.log(isSunny || isWarm)  // true
console.log(!isSunny)           // false
```


आप जावास्क्रिप्ट में `>`, `<`, `===`, और `!==` जैसे ऑपरेटरों का उपयोग करके मानों की तुलना कर सकते हैं। इन तुलनाओं का परिणाम हमेशा बूलियन होता है।


```javascript
const first = 10
const second = 5

const firstIsGreater   = (a > b)
const secondIsGreater  = (a < b)
const theyAreEqual     = (a === b)
const theyAreDifferent = (a !== b)

console.log(firstIsGreater)   // true
console.log(secondIsGreater)   // false
console.log(theyAreEqual)  // false
console.log(theyAreDifferent)  // true
```


जावास्क्रिप्ट में भी `>=` का अर्थ "बड़ा या बराबर" तथा `<=` का अर्थ "छोटा या बराबर" होता है।


बूलियन, तुलना और तार्किक ऑपरेटरों को अक्सर प्रोग्रामों में जटिल स्थितियाँ घोषित करने के लिए संयोजित किया जाता है, जैसे यह सुनिश्चित करने के लिए कि "ईमेल आ गया है और उसमें वह छवि है जिसकी मुझे आवश्यकता है या ईमेल की लंबाई 10000 वर्णों से अधिक है"। आप बाद में पाएंगे कि ये प्रोग्राम के तर्क के निर्माण के लिए आवश्यक आधारशिलाएँ हैं।


## सारणी, शून्य, अपरिभाषित

<chapterId>7bf18183-5eae-53ed-83d2-b04982145d81</chapterId>


इस अनुभाग में, हम तीन और प्रकारों को कवर करेंगे जो जावास्क्रिप्ट प्रोग्रामों में बहुत आम हैं:



- सारणी**: मानों का अनुक्रम
- अपरिभाषित**: एक विशेष मान जिसका अर्थ है कि “कुछ भी निर्दिष्ट नहीं किया गया”
- शून्य**: एक और विशेष मान जिसका अर्थ है “जानबूझकर खाली”


### सारणी और अनुक्रमणिका पहुँच


**सरणी** एक प्रकार है जो सूची में एकाधिक मान रख सकता है।


आप वर्गाकार कोष्ठक `[]` का उपयोग करके तथा आइटमों को अल्पविराम से अलग करके एक सारणी बनाते हैं।


यहाँ एक बुनियादी उदाहरण है:


```javascript
const numbers = [10, 2, 88]
console.log(numbers)
```


यह प्रिंट करता है:


```
[ 10, 2, 88 ]
```


आप किसी भी चीज़ को सारणी में संग्रहीत कर सकते हैं, केवल संख्याएँ ही नहीं:


```javascript
const things = ["apple", 42, true]
console.log(things)
```


यह प्रिंट करता है:


```
[ 'apple', 42, true ]
```


ऐरे में किसी विशिष्ट आइटम तक पहुँचने के लिए, हम एक इंडेक्स का उपयोग करते हैं। इंडेक्स आइटम की स्थिति है, जो 0 से शुरू होती है।


तो इस सरणी में:


```javascript
const colors = ["red", "green", "blue"]
```



- `colors[0]` है `"लाल"`
- `colors[1]` `"Green"` है
- `colors[2]` `"नीला"` है


आओ कोशिश करते हैं:


```javascript
const colors = ["red", "green", "blue"]
console.log(colors[0])
console.log(colors[1])
console.log(colors[2])
```


यह प्रिंट करेगा:


```
red
green
blue
```


आप किसी सरणी के किसी विशिष्ट इंडेक्स को मान निर्दिष्ट कर सकते हैं


```javascript
const colors = ["red", "green", "blue"]

colors[1] = "yellow"

console.log(colors)
```


यह प्रिंट करेगा:


```
[ 'red', 'yellow', 'blue' ]
```


आप किसी भी प्राकृतिक संख्या को सूचकांक के रूप में उपयोग कर सकते हैं, यहां तक ​​कि किसी चर में संग्रहीत संख्या को भी


```javascript

const i = 1
const colors = ["red", "green", "blue"]
console.log(colors[i])
```


यह प्रिंट करेगा:


```
green
```


लेकिन यदि आप किसी ऐसे इंडेक्स तक पहुंचने का प्रयास करते हैं जो मौजूद नहीं है, तो आपको `undefined` मिलेगा:


```javascript
const colors = ["red", "green", "blue"]
console.log(colors[3])
```


यह प्रिंट करता है:


```
undefined
```


वह क्या है??


### `अपरिभाषित`


विशेष मान `undefined` का अर्थ है “कोई मान निर्दिष्ट नहीं किया गया था”।


यदि आप कोई चर बनाते हैं लेकिन उसे कोई मान नहीं देते हैं, तो वह `अपरिभाषित` होगा:


```javascript
const name
console.log(name)
```


यह प्रिंट करता है:


```
undefined
```


चूँकि हमने `name` को कुछ भी निर्दिष्ट नहीं किया है, इसलिए JavaScript इसे डिफ़ॉल्ट रूप से `undefined` पर सेट कर देता है।


जैसा कि पहले देखा गया है, जब आप किसी ऐसे ऐरे इंडेक्स तक पहुंचते हैं जो मौजूद नहीं है, तो आपको `undefined` भी मिल सकता है:


```javascript
const fruits = ["banana", "apple"]
console.log(fruits[2]) // There is no index 2
```


यह प्रिंट करता है:


```
undefined
```


### `null` और इसका इलाज कैसे करें


`null` भी एक विशेष मान है। इसका मतलब है, "यहाँ कुछ भी नहीं है, और मैंने ऐसा जानबूझकर किया है।"


`अनिर्धारित` के विपरीत, जो स्वचालित है, `शून्य` वह चीज़ है जिसे आप स्वयं सेट करते हैं।


उदाहरण के लिए:


```javascript
const currentUser = null
console.log(currentUser)
```


यह प्रिंट करता है:


```
null
```


`null` का इस्तेमाल क्यों करें? हो सकता है कि आपको बाद में कोई मान चाहिए हो, लेकिन वह अभी तैयार न हो:


```javascript
let winner = null

// Later in the program:
winner = "Alice"

console.log(winner)
```


यह प्रिंट करता है:


```
Alice
```


इसलिए `null` तब उपयोगी होता है जब आप कहना चाहते हैं, उदाहरण के लिए, “बाद में यहां कुछ होना चाहिए था, लेकिन अभी यह खाली है।”


## ब्लॉक और नियंत्रण प्रवाह

<chapterId>be985168-2636-5b0d-a48f-ac1bbfbff8a7</chapterId>


अब तक, हमने अधिकतर कोड की ऐसी पंक्तियाँ लिखी हैं जो एक के बाद एक चलती हैं।


लेकिन जब हम कोड लिखते हैं, तो हम उसके निष्पादन के क्रम को नियंत्रित कर सकते हैं।


इसे नियंत्रण प्रवाह कहा जाता है।


आइए ब्लॉक और स्कोप को समझने से शुरुआत करें।


### वैश्विक दायरा


हमारे द्वारा घोषित प्रत्येक वेरिएबल एक स्कोप में मौजूद होता है, जिसका अर्थ है कोड का वह क्षेत्र जहां वेरिएबल ज्ञात होता है।


यदि आप किसी ब्लॉक के बाहर कोई चर घोषित करते हैं, तो वह **वैश्विक दायरे** में मौजूद रहता है।


```javascript
const color = "blue"
console.log(color)
```


यह चर `color` वैश्विक दायरे में है, इसलिए इसे फ़ाइल में कहीं से भी एक्सेस किया जा सकता है।


यदि आप और पंक्तियाँ जोड़ते हैं:


```javascript
const color = "blue"
console.log(color)

const size = "large"
console.log(color)
console.log(size)
```


`color` और `size` दोनों वैश्विक चर हैं। वे फ़ाइल में हर जगह उपलब्ध हैं।


लेकिन एक ब्लॉक के अन्दर क्या होता है?


### ब्लॉक और स्थानीय दायरा


**ब्लॉक** कोड का एक टुकड़ा है जो घुंघराले ब्रेसिज़ `{}` से घिरा होता है।


किसी ब्लॉक के अंदर `let` या `const` के साथ घोषित चर केवल उसी ब्लॉक के अंदर ही मौजूद होते हैं।


```javascript
{
const message = "inside block"
console.log(message)
}
```


यह प्रिंट करता है:


```
inside block
```


लेकिन अगर आप यह प्रयास करें:


```javascript
{
const message = "inside block"
}
console.log(message) // Error!
```


जावास्क्रिप्ट आपको इस प्रकार की त्रुटि देगा:


```
ReferenceError: message is not defined
```


ऐसा इसलिए है क्योंकि `message` को ब्लॉक के अंदर घोषित किया गया था और यह इसके बाहर मौजूद नहीं है।


इसका मतलब यह है कि हम अपने कोड के कुछ हिस्सों को अलग करने के लिए ब्लॉक का उपयोग कर सकते हैं, और यह सुनिश्चित कर सकते हैं कि "ब्लॉक में जो कुछ होता है वह ब्लॉक में ही रहे" (कुछ-कुछ लास वेगास की तरह)।


हमारे कोड को ब्लॉकों में व्यवस्थित करने से हमें प्रोग्राम के निष्पादन को संरचित करने की भी अनुमति मिलती है, `if` जैसे नियंत्रण प्रवाह निर्माणों के साथ


### `अगर`, `अन्यथा`


कभी-कभी हम कोड तभी चलाना चाहते हैं जब कुछ सच हो। `if` स्टेटमेंट का यही उद्देश्य है।


```javascript
const myAge = 20

console.log("Am I an adult?")

if (myAge >= 18) {
console.log("Yes I am!")
}
```


यह प्रिंट करता है:


```
Am I an adult?
Yes I am!
```


जैसा कि आप देख सकते हैं, कोड `myAge` और `18` की तुलना करता है।

इस स्थिति में `>=` ऑपरेटर `true` लौटाता है, अतः ब्लॉक निष्पादित हो जाता है।

यदि शर्त `सत्य` नहीं है, तो ब्लॉक निष्पादित नहीं होता है।


```javascript
const myAge = 17

console.log("Am I an adult?")

if (myAge >= 18) {
console.log("Yes I am!")
}
```


यह प्रिंट करता है:


```
Am I an adult?
```


आप विपरीत स्थिति को संभालने के लिए `else` ब्लॉक जोड़ सकते हैं:


```javascript
const myAge = 17

console.log("Am I an adult?")

if (myAge >= 18) {
console.log("Yes I am!")
} else {
console.log("No, I am not.")
}
```


यह प्रिंट करता है:


```
Am I an adult?
No, I am not.
```


`if` और `else` दोनों ब्लॉक अभी भी ब्लॉक हैं - इसलिए उनके अंदर घोषित चर बाहर मौजूद नहीं हैं।


यदि हम यह सुनिश्चित करना चाहते हैं कि कोई बात सत्य नहीं है, तो हम क्या कर सकते हैं?


खैर, जैसा कि पहले चर्चा की गई थी, जावास्क्रिप्ट में एक "not" ऑपरेटर है, जो बूलियन को उलट देता है। तो हम कर सकते हैं


```javascript
const myAge = 17

const adult = myAge >= 18

console.log("Am I an adult?")

if (!adult) {
console.log("No, I am not.")
}
```

यह अभी भी प्रिंट करता है:


```
Am I an adult?
No, I am not.
```

क्योंकि हमने `adult` चर को उलटने के लिए `!` ऑपरेटर का उपयोग किया है।


`if (!adult) {...}` को "if not adult..." पढ़ा जाना चाहिए


ब्लॉक, तर्क और तुलना ऑपरेटरों का उपयोग करके, हम प्रोग्राम के निष्पादन को संरचित कर सकते हैं, इसके लिए उन चरों को परिभाषित करना होगा जो किसी चीज के घटित होने के लिए `सत्य` (या `असत्य`) होने चाहिए।


### `जबकि`, `ब्रेक`, `जारी`


`while` लूप कोड को तब तक दोहराता है जब तक कि शर्त सत्य है।


```javascript
let count = 0

while (count < 3) {
console.log("Count is", count)
count = count + 1
}
console.log("the loop is over!")
```


यह प्रिंट करता है:


```
Count is 0
Count is 1
Count is 2
the loop is over!
```


जब `count` 3 हो जाता है, तो लूप रुक जाता है।


आप `break` का उपयोग करके लूप को जल्दी रोक सकते हैं:


```javascript
let number = 1 // Start with number 1

while (true) { // This condition is always true, so this loop will run forever unless we stop it
console.log(number) // Print the current number
if (number === 3) { // If the number is 3, stop the loop
break
}
number = number + 1 // Add 1 to the number
}
```


यह प्रिंट करता है:


```
0
1
2
```


क्योंकि जब संख्या `3` हो जाती है, तो `if` ब्लॉक निष्पादित हो जाता है और यह लूप को रोक देता है।


आप `continue` का उपयोग करके लूप के शेष भाग को छोड़ सकते हैं:


```javascript
let number = 0 // Start with number 0

while (number < 5) { // Keep going while number is less than 5
number = number + 1 // Add 1 to the number

if (number === 3) { // If the number is 3
continue // Skip the rest of the block and go to the next iteration of the loop
}

console.log(number) // Print the number
}
```


यह प्रिंट करता है:


```
1
2
4
5
```


क्योंकि जब संख्या `3` थी, तो `continue` ने प्रोग्राम को संख्या प्रिंट करने वाली पंक्ति को छोड़ दिया।


### `के लिए ... का ...`


यदि आपके पास एक सारणी है, और आप इसमें प्रत्येक आइटम के साथ कुछ करना चाहते हैं, तो आप `for ... of ... {...}` का उपयोग कर सकते हैं।


```javascript
const fruits = ["apple", "banana", "cherry"]

for (const fruit of fruits) {
console.log(fruit)
}
```


यह प्रिंट करता है:


```
apple
banana
cherry
```

ब्लॉक को सरणी के प्रत्येक तत्व के लिए एक बार निष्पादित किया जाएगा।


यहां `fruit` एक नया चर है जो ब्लॉक के अंदर उस पर काम करने के लिए सरणी में प्रत्येक आइटम का मान लेता है।


### `के लिए ... में ...`


आप किसी सारणी की कुंजियों (इंडेक्स) पर लूप करने के लिए `for ... in` का उपयोग कर सकते हैं:


```javascript
const fruits = ["apple", "banana", "cherry"]

for (const index in fruits) {
console.log(index)
}
```


यह प्रिंट करता है:


```
0
1
2
```


आप मान प्राप्त करने के लिए सूचकांक का भी उपयोग कर सकते हैं:


```javascript
const fruits = ["apple", "banana", "cherry"]

for (const index in fruits) {
console.log(fruits[index])
}
```


यह `for ... of` के समान ही प्रिंट करता है:


```
apple
banana
cherry
```


व्यवहार में, सरणियों के लिए, आपको `for ... of` का उपयोग करना चाहिए, क्योंकि यह सरल और साफ है।


### सीमित लूप


कभी-कभी हम एक निश्चित संख्या में लूप करना चाहते हैं, या सामान्यतः कोड का एक टुकड़ा लिखना चाहते हैं जो किसी चीज़ का ट्रैक रखते हुए एक ब्लॉक को दोहराता है।

यही वह चीज है जिसके लिए एक बाउंडेड `for` लूप अच्छा है।

एक बाउंडेड लूप में आमतौर पर तीन शर्तें होती हैं, जिन्हें अर्धविराम `;` द्वारा अलग किया जाता है, जैसे कि `(... ; ... ; ....)`।


```javascript
for (let i = 0; i < 3; i = i + 1) {
console.log(i)
}
```


यह प्रिंट करता है:


```
0
1
2
```


आइये इसे समझाएं:



- `let i = 0`: ब्लॉक में उपयोग किए जाने वाले चर की घोषणा करता है (इस स्थिति में यह एक काउंटर है जो 0 से शुरू होता है)
- `i < 3`: ब्लॉक को निष्पादित करने के लिए शर्त को `सत्य` घोषित करता है (इस स्थिति में "जब तक `i` 3 से कम है तब तक दोहराएँ")
- `i = i + 1`: ब्लॉक के प्रत्येक निष्पादन के बाद चलाए जाने वाले कुछ कोड की घोषणा करें (इस स्थिति में "`i` को 1 से बढ़ाएँ")


जैसा कि आप देख सकते हैं, बाउंडेड लूप हमें कोड के एक भाग के बार-बार निष्पादन के लिए अधिक जटिल शर्तें घोषित करने के लिए प्रेरित करता है, लेकिन अधिकांश समय यह आवश्यक नहीं होता है।


### ब्लॉक लेबल


यदि आपको अधिक जटिल नियंत्रण प्रवाह लिखना है, तो जावास्क्रिप्ट आपको एक लेबल का उपयोग करके ब्लॉक का नाम देने की सुविधा देता है, जिसका उपयोग `ब्रेक` या `कंटिन्यू` द्वारा यह निर्दिष्ट करने के लिए किया जा सकता है कि *कहां* वापस लौटना है।


उदाहरण:


```javascript
outer: {
console.log("We're inside the outer scope.")

inner: {
console.log("We're inside the inner scope.")
break outer
}

console.log("This will not run")
}

console.log("Done")
```


यह प्रिंट करता है:


```
Inside outer block
Inside inner block
Done
```


हमने `outer` ब्लॉक से पूरी तरह बाहर निकलने के लिए `break outer` का उपयोग किया।


आप लूप्स को लेबल भी कर सकते हैं। आइए इस उदाहरण को लेते हैं:


```javascript
// Declare a variable to count the total number of days in a year
let totalDaysInOneYear = 0

// Declare one variable per month, with the number of the month
const january = 1
const february = 2
const march = 3
const april = 4
const may = 5
const june = 6
const july = 7
const august = 8
const september = 9
const october = 10
const november = 11
const december = 12

// Declare an array that holds the months that have 30 days
const monthsWith30Days = [
april, june, september, november
]

// Declare variables to keep track of the month and day we're in
let currentMonth = january
let currentDay = 1

monthsLoop: while (true) {  // Start a loop labeled "monthsLoop" to process each month

daysLoop: while (true) {  // Start a loop labeled "daysLoop" to process each day in the month
totalDaysInOneYear = totalDaysInOneYear + 1  // Increase the total number of days we counted by 1

if (                                                                   // We want to check if we're at the end of the month.
currentDay === 31                                                  // Check if the current day is 31 (for months with 31 days)...
|| currentDay === 30 && (monthsWith30Days.includes(currentMonth))  // ...or 30 if it's among the 30-days months...
|| currentDay === 28 && (currentMonth === february)                // ...or 28 if it's February. If it's any of these three, then:

){
currentMonth = currentMonth + 1  // Move to the next month
currentDay = 1                   // Reset the day to 1 for the new month
break daysLoop                   // Exit the inner loop (which tracks days) and go back to the outer loop (which tracks months)
}
else { currentDay = currentDay + 1 }                                   // Otherwise, we're not at the end of the month, and we just move to the next day

}
if (currentMonth > 12) {  // After processing a month, check if we've gone past December
break monthsLoop  // If so, break the outer loop and stop the day-counting process
}
}

console.log(totalDaysInOneYear)  // Print the total number of days in the year (should be 365)
```

यह एक बहुत ही उबाऊ उदाहरण था, लेकिन उम्मीद है कि इससे लेबल की (कभी-कभार) आवश्यकता स्पष्ट हो गई होगी।


## कार्यों का परिचय

<chapterId>cc324715-09c2-5cf7-9e6f-47a6f16bc04d</chapterId>


जैसे-जैसे आपके प्रोग्राम विकसित होते हैं, आप अक्सर कोड के टुकड़ों का पुनः उपयोग करना चाहेंगे।


**फ़ंक्शन** इसी के लिए हैं: वे आपको कुछ कोड को एक साथ समूहित करने, उसे एक नाम देने और जब चाहें उसे चलाने की सुविधा देते हैं।


### फ़ंक्शन घोषणा


किसी फ़ंक्शन को घोषित करने के लिए, हम `function` कीवर्ड का उपयोग कर सकते हैं। फिर हम उसे एक नाम, कोष्ठकों का एक युग्म `()` और उसके द्वारा ग्रहण किए जाने वाले तर्क, और निष्पादित किए जाने वाले कोड का एक ब्लॉक `{}` देते हैं। आइए एक ऐसे फ़ंक्शन से शुरुआत करें जो कोई तर्क नहीं लेता:


```javascript
function sayHello () {console.log(`Hello!`) }
```


यह कोड फ़ंक्शन की घोषणा तो करता है, लेकिन उसे अभी तक चलाता नहीं है।


### फ़ंक्शन कॉल


फ़ंक्शन को चलाने (या "कॉल") के लिए, आप उसका नाम कोष्ठक के बाद लिखते हैं:


```javascript
function sayHello () {console.log(`Hello!`) }
sayHello()
```


यह प्रिंट करता है:


```
Hello!
```


आप फ़ंक्शन को जितनी बार चाहें कॉल कर सकते हैं:


```javascript
function sayHello() {
console.log("Hello!")
}

sayHello()
sayHello()
```


यह प्रिंट करता है:


```
Hello!
Hello!
```


फ़ंक्शन के अंदर का कोड केवल तभी चलता है जब आप उसे कॉल करते हैं।


### फ़ंक्शन तर्क (फ़ंक्शन में इनपुट)


कभी-कभी, आप चाहते हैं कि कोई फ़ंक्शन कुछ इनपुट के साथ काम करे। आप कोष्ठकों के अंदर **आर्गुमेंट** जोड़कर ऐसा कर सकते हैं।


उदाहरण के लिए:


```javascript
function sayHelloTo (friend) {
console.log(`Hello ${friend}!`)
}
```


अब यह फ़ंक्शन `friend` नामक एक तर्क लेता है।


जब आप फ़ंक्शन को कॉल करते हैं, तो आप एक मान पास कर सकते हैं:


```javascript
sayHelloTo("Tommy")
```


यह प्रिंट करता है:


```
Hello Tommy!
```


आप फ़ंक्शन को पुनः किसी भिन्न नाम से कॉल कर सकते हैं:


```javascript
sayHelloTo("Sam")
```


यह प्रिंट करता है:


```
Hello Sam!
```


आपके द्वारा पास किया गया मान फ़ंक्शन के अंदर `friend` वेरिएबल को प्रतिस्थापित करता है।


आप एक से अधिक तर्क का भी उपयोग कर सकते हैं:


```javascript
function greetTwoPeople(person1, person2) {
console.log(`Hello ${person1} and ${person2}!`)
}

greetTwoPeople("Lina", "Marco")
```


यह प्रिंट करता है:


```
Hello Lina and Marco!
```


### `वापसी` (फ़ंक्शन से आउटपुट)


फ़ंक्शन मान भी लौटा सकते हैं। इसका मतलब है कि वे उस स्थान पर मान वापस भेजते हैं जहाँ फ़ंक्शन को कॉल किया गया था।


यहाँ एक सरल उदाहरण है:


```javascript
function getNumber() {
return 42
}

const result = getNumber()
console.log(result)
```


यह प्रिंट करता है:


```
42
```


फ़ंक्शन `getNumber()` `42` लौटाता है, और हम इसे `result` में संग्रहीत करते हैं, फिर इसे प्रिंट करते हैं।


आप अपनी गणना से कुछ भी लौटा सकते हैं:


```javascript
function add(a, b) {
return a + b
}

const result = add(2, 3)
console.log(result)
```


यह प्रिंट करता है:


```
5
```


एक बार जब कोई मान `return` हो जाता है, तो फ़ंक्शन रुक जाता है। उस ब्लॉक में `return` के बाद कुछ भी नहीं होता।


```javascript
function saySomething() {

return "hi"

console.log("this never runs")

}

const message = saySomething()
console.log(message)
```


यह केवल प्रिंट करता है:


```
hi
```


क्योंकि केवल `"hi"` लौटाया जाता है। `console.log("this never runs")` पंक्ति छोड़ दी जाती है।


आप फ़ंक्शन को छोटे उप-प्रोग्राम के रूप में सोच सकते हैं। प्रत्येक फ़ंक्शन कुछ इनपुट ले सकता है, उस पर काम कर सकता है, और आपको कुछ आउटपुट दे सकता है।


यदि हम किसी फ़ंक्शन के रिटर्न मान का उपयोग करने का प्रयास करते हैं, लेकिन वह फ़ंक्शन कुछ भी रिटर्न नहीं करता है, तो क्या होगा?


```javascript
function doesNothing () {}

const x = doesNothing()

console.log(x)
```

यह `undefined` प्रिंट करेगा। किसी फ़ंक्शन का रिटर्न मान जो कुछ भी रिटर्न नहीं करता, `undefined` होता है।


## ऑब्जेक्ट और क्लासेस

<chapterId>26689f25-8212-5057-8c21-3a05eee0ac75</chapterId>


जावास्क्रिप्ट को अक्सर ऑब्जेक्ट-ओरिएंटेड भाषा कहा जाता है।


इसका मतलब यह है कि यह मूल्यों और कार्यों को ऑब्जेक्ट्स में समूहित करके आपके कोड को व्यवस्थित करने में आपकी सहायता करता है।


### `ऑब्जेक्ट` क्या है?


एक ऑब्जेक्ट में डेटा और उस डेटा पर काम करने वाले फ़ंक्शन हो सकते हैं। जब किसी फ़ंक्शन को किसी ऑब्जेक्ट में डाला जाता है, तो हम उसे "मेथड" कहते हैं।


पहला ऑब्जेक्ट जो हमने देखा, वह था `कंसोल` ऑब्जेक्ट। यह एक ऐसा ऑब्जेक्ट है जिसमें स्क्रीन पर चीज़ें प्रिंट करने और हमारे प्रोग्राम्स को डीबग करने के कई तरीके होते हैं।


यह स्वयं भी प्रिंट कर सकता है; आप कर सकते हैं


```javascript
console.log(console)
```


और यह इसमें मौजूद विधियों की एक सूची प्रिंट करेगा। उदाहरण के लिए, मेरी मशीन पर यह प्रिंट हुआ


```javascript
Object [console] {
log: [Function: log],
warn: [Function: warn],
error: [Function: error],
dir: [Function: dir],
time: [Function: time],
timeEnd: [Function: timeEnd],
timeLog: [Function: timeLog],
trace: [Function: trace],
assert: [Function: assert],
clear: [Function: clear],
count: [Function: count],
countReset: [Function: countReset],
group: [Function: group],
groupEnd: [Function: groupEnd],
table: [Function: table],
debug: [Function: debug],
info: [Function: info],
dirxml: [Function: dirxml],
groupCollapsed: [Function: groupCollapsed],
Console: [Function: Console],
profile: [Function: profile],
profileEnd: [Function: profileEnd],
timeStamp: [Function: timeStamp],
context: [Function: context],
createTask: [Function: createTask]
}
```


जैसा कि आप देख सकते हैं, इसमें बहुत सारी विधियां हैं जिनका उपयोग आप डीबग करने के लिए कर सकते हैं!


जावास्क्रिप्ट हमें नई वस्तुएं बनाने के लिए अलग-अलग तरीके प्रदान करता है जो वे सब कुछ कर सकते हैं जो हम उनसे करवाना चाहते हैं।


### एक वस्तु बनाना


ऑब्जेक्ट बनाने का सबसे आसान तरीका **घुंघराले ब्रेसेज़** `{}` का उपयोग करके डेटा और फ़ंक्शन को समूहीकृत करना है।


इससे वह चीज़ बनती है जिसे हम **अनाम वस्तु** कहते हैं


```javascript
const cat = {
name: "Whiskers",
age: 3
}
```


यह एक ऑब्जेक्ट बनाता है और उसे `cat` नामक वेरिएबल में संग्रहीत करता है।


ऑब्जेक्ट के दो गुण हैं:



- `नाम` का मान `"Whiskers"` है
- `उम्र` का मान `3` है


आइये इसे प्रिंट करें:


```javascript
console.log(cat)
```


यह प्रिंट करता है:


```
{ name: 'Whiskers', age: 3 }
```


आप डॉट का उपयोग करके ऑब्जेक्ट से गुण प्राप्त कर सकते हैं, जैसे कि `objectName.propertyName` में:


```javascript
console.log(cat.name)  // prints "Whiskers"
console.log(cat.age)   // prints 3
```


आप संपत्ति भी बदल सकते हैं:


```javascript
cat.age = 4
console.log(cat.age)  // now it prints 4
```


जैसा कि आप देख सकते हैं, भले ही किसी ऑब्जेक्ट को `const` के रूप में परिभाषित किया गया हो, फिर भी आप उसमें मौजूद डेटा को संशोधित कर सकते हैं।


ऑब्जेक्ट के मामले में, `const` आपको केवल संपूर्ण ऑब्जेक्ट को ओवरराइड करने से रोकेगा:


```javascript
const cat = {
name: "Whiskers",
age: 3
}

cat.age = 5 // this works

cat = 5 // this throws an error, you're trying to reassign the whole object

```


जैसा कि पहले बताया गया है, ऑब्जेक्ट्स भी **फ़ंक्शन** रख सकते हैं, और जब कोई फ़ंक्शन किसी ऑब्जेक्ट का हिस्सा होता है, तो हम उसे **मेथड** कहते हैं।


यहाँ एक उदाहरण है:


```javascript
const cat = {
name: "Whiskers",
speak () {
console.log("Meow!")
}
}
```


इस वस्तु में है:



- एक `नाम` संपत्ति
- एक `speak()` विधि


आइए विधि को कॉल करें:


```javascript
cat.speak()
```


यह प्रिंट करता है:


```
Meow!
```


विधियाँ `this` कीवर्ड के माध्यम से ऑब्जेक्ट में मौजूद डेटा का उपयोग कर सकती हैं।

`this` वर्तमान ऑब्जेक्ट को संदर्भित करता है। इस उदाहरण में, इसका उपयोग बिल्ली का नाम प्रिंट करने के लिए किया जाएगा:


```javascript
const cat = {
name: "Whiskers",
speak () {
console.log(`${this.name} says meow!`)
}
}

cat.speak()
```


यह प्रिंट करता है:


```
Whiskers says meow!
```


शब्द `this` का अर्थ है "यह वस्तु"...इस मामले में, `cat` वस्तु।



जब आप बस कुछ जल्दी और सरल बनाना चाहते हैं, तो इस तरह के ऑब्जेक्ट बहुत अच्छे होते हैं। लेकिन अगर आपको एक ही संरचना वाले **कई ऑब्जेक्ट** बनाने हैं, तो एक बेहतर तरीका है, और यहीं पर **क्लास** काम आते हैं।


### कक्षाएं और कंस्ट्रक्टर


एक क्लास एक ब्लूप्रिंट की तरह होता है। यह जावास्क्रिप्ट को बताता है कि किसी खास तरह की ऑब्जेक्ट कैसे बनाई जाए।


आप `class` कीवर्ड का उपयोग करके एक क्लास को परिभाषित करते हैं, उसके बाद क्लास का नाम, और एक घुंघराले ब्रेसिज़ `{}` ब्लॉक का उपयोग करते हैं।


```javascript
class Dog {}
```


परंपरा के अनुसार कक्षाएं आमतौर पर बड़े अक्षर से शुरू होती हैं।


आप `new` का उपयोग करके किसी क्लास का नया ऑब्जेक्ट बना सकते हैं:


```javascript
const hachiko = new Dog()
```


ऑब्जेक्ट को प्रिंट करने का प्रयास करें:

```javascript
class Dog {}

const myDog = new Dog()

console.log(myDog)
```


आपको मिलेगा


```
Dog {}
```


जैसा कि आप देख सकते हैं कि Dog क्लास खाली है, इसलिए `myDog` ऑब्जेक्ट भी खाली है।


हम `कन्स्ट्रक्टर` जोड़कर परिभाषित कर सकते हैं कि डॉग ऑब्जेक्ट में कौन से गुण होने चाहिए।


कंस्ट्रक्टर एक विशेष फ़ंक्शन है जो तब चलता है जब आप कोई नया ऑब्जेक्ट बनाते हैं (या "निर्माण")।


```javascript
class Dog {
constructor() { }
}
```


हम चाहते हैं कि प्रत्येक कुत्ते का एक नाम हो, इसलिए हम फ़ंक्शन में एक `name` पैरामीटर जोड़ते हैं:


```javascript
class Dog {
constructor(name) { }
}
```


और फिर हम `this` का उपयोग यह घोषित करने के लिए करते हैं कि `name` उस `Dog` ऑब्जेक्ट का `name` है जिसे हम बना रहे हैं


```javascript
class Dog {
constructor(name) {
this.name = name
}}
```


आइये अब इसका उपयोग करने का प्रयास करें:


```javascript
class Dog {
constructor(name) {
this.name = name
}
}

const myDog = new Dog("hachiko")

console.log(myDog)
```


यह कुछ इस तरह प्रिंट होता है:


```
Dog { name: 'hachiko' }
```


जैसा कि आप देख सकते हैं, `constructor` विधि आपके द्वारा `new Dog()` करते समय क्लास में पास किए गए तर्कों को लेती है, और ऑब्जेक्ट बनाने के लिए इसका उपयोग करती है।


आइये इसे समझें:



- `class Dog` डॉग वर्ग को परिभाषित करता है।
- `constructor(name)` ऑब्जेक्ट को बनाते समय उसे सेट करता है।
- `this.name = name` नए ऑब्जेक्ट में मान संग्रहीत करता है।
- `new Dog("hachiko")` वर्ग से एक नया ऑब्जेक्ट बनाता है, जिसमें `name` गुण `"hachiko"` पर सेट होता है।


अब आइए अपनी क्लास में एक विधि जोड़ें:


```javascript
class Dog {
constructor(name) {
this.name = name
}
speak () {
console.log(`${this.name} says barf!`)
}

}

const myDog = new Dog("hachiko")

myDog.speak()
```


यह प्रिंट होगा


```javascript
hachiko says barf!
```


यदि हम कुत्ते के दो अलग-अलग उदाहरणों के लिए ऐसा ही करते हैं



```javascript
class Dog {
constructor(name) {
this.name = name
}
speak () {
console.log(`${this.name} says barf!`)
}

}

const myDog = new Dog("hachiko")

myDog.speak()

const yourDog = new Dog("bobby")

yourDog.speak()
```


हम पाते हैं


```
hachiko says barf!
bobby says barf!
```


`speak()` विधि उस `Dog` के `name` गुण का उपयोग करती है जिस पर उसे बुलाया जाता है।


कक्षाओं के अस्तित्व का यही मुख्य कारण है: वे हमें डेटा पर काम करने वाली विधियों का एक सेट परिभाषित करने की अनुमति देते हैं, और फिर एक ही डेटा "आकार" को साझा करने वाली कई ऑब्जेक्ट्स बनाते हैं।


जब हम इनमें से किसी ऑब्जेक्ट पर कोई विधि कॉल करते हैं, तो वह उस विशिष्ट ऑब्जेक्ट द्वारा रखे गए डेटा पर कार्य करेगी।


### किसी वस्तु का आकार बदलना


जावास्क्रिप्ट में ऑब्जेक्ट लचीले होते हैं। एक ऑब्जेक्ट बनाने के बाद भी, आप नए गुण जोड़ सकते हैं या मौजूदा गुण हटा सकते हैं।


इसकी अनुमति है, लेकिन आपको इसका उपयोग सावधानी से करना चाहिए।


आइए अपने सरल `Dog` वर्ग से शुरू करें:


```javascript
class Dog {
constructor(name) {
this.name = name
}

speak() {
console.log(`${this.name} says barf!`)
}
}

const myDog = new Dog("Fido")
```


इस समय, `myDog` में केवल एक ही प्रॉपर्टी है: `name`। इसके बनने के बाद भी हम नई प्रॉपर्टी जोड़ सकते हैं:


```javascript
myDog.age = 5

console.log(myDog.age) // prints 5
```


हम एक नई विधि भी जोड़ सकते हैं:


```javascript
myDog.jump = function () {
console.log(`${this.name} jumps!`)
}

myDog.jump() // Fido jumps!
```


और हम `delete` कीवर्ड का उपयोग करके गुण भी हटा सकते हैं।


```javascript
delete myDog.name

console.log(myDog.name) // prints 'undefined'
```


यह काम करता है, लेकिन यहाँ एक ज़रूरी बात जानना ज़रूरी है: V8 जैसे जावास्क्रिप्ट इंजन (Node.js और क्रोम ब्राउज़र में इस्तेमाल किए जाने वाले) तब तेज़ी से चलते हैं जब आपके ऑब्जेक्ट हमेशा एक जैसे गुण बनाए रखते हैं। अगर आप ऑब्जेक्ट बनने के बाद गुण जोड़ते या हटाते हैं, तो यह काम धीमा कर सकता है।


छोटे प्रोग्रामों में, यह ज़्यादा मायने नहीं रखता। लेकिन बड़े प्रोजेक्ट्स (जैसे गेम) में, कंस्ट्रक्टर में शुरू से ही सभी प्रॉपर्टीज़ को सूचीबद्ध करना बेहतर होता है, भले ही आप उनका तुरंत इस्तेमाल न करें। इससे ऑब्जेक्ट का आकार स्थिर रहता है और आपके कोड को तेज़ी से चलाने में मदद मिलती है।


उदाहरण के लिए, इसके स्थान पर:


```javascript
class Dog {
constructor(name) {
this.name = name
}
}

const dog = new Dog("Rex")
dog.age = 4
dog.breed = "Labrador"
```


तुम यह कर सकते थे


```javascript
class Dog {
constructor(name, age, breed) {
this.name = name
this.age = age
this.breed = breed
}
}

const dog = new Dog("Rex", 4, "Labrador")
```


दोनों संस्करण काम करते हैं, लेकिन प्रदर्शन के लिए दूसरा बेहतर है। आप इंजन को पहले ही बता देते हैं कि प्रत्येक ऑब्जेक्ट में कौन से गुण होंगे, और वह उसके अनुसार अनुकूलन कर सकता है।


जावास्क्रिप्ट आपको ऑब्जेक्ट्स को स्वतंत्र रूप से पुनः आकार देने की सुविधा देता है, लेकिन क्लासेस का उपयोग करते समय, अपने ऑब्जेक्ट के आकार की पहले से योजना बनाना सबसे अच्छा होता है।



### `extends` और `super()` के साथ वंशानुक्रम


कभी-कभी आप एक ऐसा वर्ग बनाना चाहते हैं जो *लगभग* किसी अन्य वर्ग के समान हो, लेकिन कुछ अतिरिक्त सुविधाओं के साथ।


ऑब्जेक्ट्स के आकार को संशोधित करने के बजाय (जैसा कि पहले बताया गया है कि यह प्रदर्शन के लिए इष्टतम नहीं है), या एक नई क्लास को नए सिरे से लिखने के बजाय, जावास्क्रिप्ट आपको **इनहेरिटेंस** नामक कुछ का उपयोग करने की सुविधा देता है।


वंशानुक्रम का अर्थ है कि एक वर्ग दूसरे वर्ग का विस्तार कर सकता है। नए वर्ग को पुराने वर्ग के सभी गुण और विधियाँ मिल जाती हैं, और आप अपनी ज़रूरत के अनुसार और गुण और विधियाँ जोड़ सकते हैं या बदल सकते हैं।


मान लीजिए हमारे पास `Vehicle` नामक एक आधार वर्ग है:


```javascript
class Vehicle {
constructor(brand) {
this.brand = brand
}

start() {
console.log(`${this.brand} vehicle is starting...`)
}
}
```


अब हम एक `Car` क्लास बनाना चाहते हैं। कार एक तरह का वाहन है, लेकिन हम चाहेंगे कि इसमें कुछ अतिरिक्त विशेषताएँ हों या शुरू होने पर कोई अलग संदेश हो। सब कुछ दोबारा लिखने के बजाय, हम `extends` का इस्तेमाल कर सकते हैं:


```javascript
class Car extends Vehicle {
start() {
console.log(`${this.brand} car is ready to drive!`)
}
}
```


`कार` क्लास अब `वाहन` से सब कुछ **विरासत में** लेती है। इसे `ब्रांड` प्रॉपर्टी मिलती है, और हमने `स्टार्ट()` मेथड को अपने संस्करण से बदल दिया है।


![](assets/en/4.webp)


आइये इसे आज़माएँ:


```javascript
const myCar = new Car("Toyota")
myCar.start()
```


यह प्रिंट करता है:


```
Toyota car is ready to drive!
```


हालाँकि `Car` का अपना कोई कंस्ट्रक्टर नहीं है, फिर भी यह `Vehicle` के कंस्ट्रक्टर का उपयोग करता है। लेकिन अगर हम `Car` में एक कस्टम कंस्ट्रक्टर लिखना चाहें, तो हम लिख सकते हैं, बस हमें `super()` का उपयोग करके इसके पैरेंट के कंस्ट्रक्टर को कॉल करना होगा।


ऐसे:


```javascript
class Vehicle {
constructor(brand) {
this.brand = brand
}

start() {
console.log(`${this.brand} vehicle is starting...`)
}

}

class Car extends Vehicle {
constructor(brand, model) {
super(brand) // call the parent constructor and passes the brand argument to it
this.model = model
}

start() {
console.log(`${this.brand} ${this.model} is ready to drive!`)
}
}

const myCar = new Car("Toyota", "Corolla")
myCar.start()
```


![](assets/en/5.webp)



यह प्रिंट करता है:


```
Toyota Corolla is ready to drive!
```


तो संक्षेप में



- `विस्तार` का अर्थ है कि एक वर्ग दूसरे पर आधारित है।
- `super()` का उपयोग उस क्लास के कन्स्ट्रक्टर को कॉल करने के लिए किया जाता है जिसे आप विस्तारित कर रहे हैं।
- नये वर्ग को मूल वर्ग के सभी गुण और विधियाँ प्राप्त हो जाती हैं।
- आप विधियों (जैसे `start()`) को **ओवरराइड** कर सकते हैं ताकि वे कुछ अलग कार्य कर सकें।


यह तब उपयोगी होता है जब आपके पास कई समान चीजें हों (जैसे कार, ट्रक और बाइक) और आप चाहते हैं कि वे कोड साझा करें लेकिन फिर भी अपने तरीके से व्यवहार करें।


### `उदाहरण`


`instanceof` कीवर्ड यह जांचता है कि क्या कोई ऑब्जेक्ट किसी निश्चित वर्ग से बनाया गया था।


मान लीजिए हमारे पास `User` नामक एक क्लास है:


```javascript
class User {
constructor(username) {
this.username = username
}
}

const regularUser = new User("julia123")

console.log(regularUser instanceof User)
```


यह प्रिंट करता है:


```
true
```


पुष्टि करें कि `regularUser` एक `User` है। ऐसा इसलिए है क्योंकि `regularUser` को `User` वर्ग का उपयोग करके बनाया गया था।


यह **विरासत में मिली** क्लासों के साथ भी काम करता है। उदाहरण के लिए, यहाँ एक `Admin` क्लास है जो `User` को एक्सटेंड करती है:


```javascript
class Admin extends User {}

const ourAdmin = new Admin("admin42")

console.log(ourAdmin instanceof Admin)   // true
console.log(ourAdmin instanceof User)    // true
```


दोनों पंक्तियाँ `true` लौटाती हैं। ऐसा इसलिए है क्योंकि `Admin`, `User` का एक उपवर्ग है, इसलिए `ourAdmin` एक `Admin` और एक `User` दोनों है।

## त्रुटि प्रबंधन

<chapterId>d0206bc5-d386-5e7f-9917-5803f392448c</chapterId>


जैसे-जैसे आप ज़्यादा जटिल JavaScript प्रोग्राम लिखते हैं, आपको **त्रुटियाँ** मिलेंगी। ये अप्रत्याशित स्थितियाँ होती हैं जहाँ कुछ गड़बड़ हो जाती है। हो सकता है कि कोई वेरिएबल `अपरिभाषित` हो, लेकिन आप उसका इस्तेमाल करने की कोशिश करते हैं, या किसी कोड को गलत प्रकार का इनपुट मिलता है।


अगर हम इन त्रुटियों को ठीक से नहीं संभालते, तो हमारा प्रोग्राम क्रैश हो सकता है या अप्रत्याशित तरीके से काम कर सकता है। जावास्क्रिप्ट इन त्रुटियों का पता लगाने और उन्हें प्रबंधित करने के लिए उपकरण प्रदान करता है ताकि हम उन्हें अधिक सुचारू रूप से संभाल सकें।


### सामान्य त्रुटि: `undefined` पर मान तक पहुँचना


यहाँ एक सामान्य स्थिति दी गई है जो त्रुटि का कारण बनती है:


```javascript
const user = undefined
console.log(user.name)
```


यदि आप यह कोड चलाएंगे, तो आपको एक त्रुटि मिलेगी जो इस प्रकार दिखाई देगी:


```
TypeError: Cannot read properties of undefined (reading 'name')
```


यह जावास्क्रिप्ट आपको बता रहा है: "अरे, आपने `name` प्रॉपर्टी को किसी ऐसी चीज़ से प्राप्त करने की कोशिश की जो `undefined` है, और इसका कोई मतलब नहीं है।" और जैसा कि आप देख सकते हैं, जब इस तरह की त्रुटि होती है, तो प्रोग्राम चलना बंद कर देता है जब तक कि आपने इसे पकड़ने और संभालने के लिए विशेष रूप से कोड नहीं लिखा हो।


### एक त्रुटि फेंकना


कभी-कभी आप अपने कोड में मैन्युअल रूप से **त्रुटि उत्पन्न करना** चाहते हैं। ऐसे में, आप `throw` कीवर्ड का उपयोग करते हैं।


```javascript
throw new Error("This is a custom error message")
```


इससे प्रोग्राम तुरन्त रुक जाता है और प्रिंट होता है:


```
Uncaught Error: This is a custom error message
```


आप अपने प्रोग्राम में नियम लागू करने के लिए `throw` का उपयोग कर सकते हैं। उदाहरण के लिए:


```javascript
function divide(a, b) {
if (b === 0) {
throw new Error("You can't divide by zero")
}
return a / b
}

console.log(divide(10, 2))  // OK: prints 5
console.log(divide(10, 0))  // Error!
```


दूसरी कॉल के कारण त्रुटि उत्पन्न होती है, क्योंकि इस उदाहरण में शून्य से भाग देने की अनुमति नहीं है।


### `try...catch` के साथ त्रुटियाँ पकड़ना


अगर आप नहीं चाहते कि कोई त्रुटि होने पर आपका प्रोग्राम क्रैश हो जाए, तो आप `try...catch` ब्लॉक का इस्तेमाल करके त्रुटि पकड़ सकते हैं। यह तब मददगार होता है जब आप चाहते हैं कि आपका प्रोग्राम किसी गड़बड़ी के बावजूद **चलता रहे**।


```javascript
try {
const user = undefined
console.log(user.name)
console.log("End of the block") // this will never get printed
} catch (error) {
console.log("Oops! Something went wrong.")
}
```


आउटपुट:


```
Oops! Something went wrong.
```


यह ऐसे काम करता है:



- `try` ब्लॉक के अंदर के कोड को पहले प्रयास किया जाता है।
- यदि कोई त्रुटि होती है, तो जावास्क्रिप्ट शेष `try` ब्लॉक को छोड़ते हुए `catch` ब्लॉक पर चला जाता है।
- `catch` ब्लॉक त्रुटि प्राप्त करता है, इसलिए आप इसे प्रिंट कर सकते हैं, या इसे किसी अन्य तरीके से संभाल सकते हैं, उदाहरण के लिए


```javascript
try {
const user = undefined
console.log(user.name)
console.log("End of the block") // this will never get printed
} catch (error) {
console.log(`The message of the error was: "${error.message}"`)
}
```


आउटपुट:


```
The message of the error was: "Cannot read properties of undefined (reading 'name')"
```


### `अंततः` ब्लॉक


आप एक `finally` ब्लॉक भी जोड़ सकते हैं। यह कोड हमेशा चलता रहता है, चाहे कोई त्रुटि हो या न हो।


```javascript
try {
console.log("Trying something risky...")
throw new Error("Uh oh!")
} catch (error) {
console.log("Caught the error:", error.message)
} finally {
console.log("This will run no matter what.")
}
```


आउटपुट:


```
Trying something risky...
Caught the error: Uh oh!
This will run no matter what.
```


## कीड़ों से बचना

<chapterId>db12d9f6-5806-514c-998e-0ae24805104e</chapterId>


यह अध्याय जावास्क्रिप्ट में कुछ सबसे आम कमियों को दर्शाता है, तथा उनसे बचने के तरीके भी बताता है।


### `var` और Assignment बिना घोषणा के


पुराने जावास्क्रिप्ट कोड में, वेरिएबल्स को अक्सर `var` कीवर्ड का उपयोग करके घोषित किया जाता था। `let` और `const` के विपरीत, जिनके बारे में आप पहले ही जान चुके हैं, `var` भ्रामक तरीके से व्यवहार कर सकता है।


उदाहरण के लिए:


```javascript
{
var message = "hello"
}
console.log(message)
```


आप उम्मीद कर सकते हैं कि `message` केवल ब्लॉक के अंदर ही मौजूद होगा, लेकिन ऐसा नहीं है। `var` ब्लॉक स्कोप को अनदेखा करता है और यह वेरिएबल को संपूर्ण फ़ंक्शन या फ़ाइल में उपलब्ध कराता है।


इससे अप्रत्याशित व्यवहार हो सकता है, खासकर बड़े प्रोग्रामों में। इस कारण से, आधुनिक जावास्क्रिप्ट कोड में हमेशा `var` के बजाय `let` या `const` का उपयोग किया जाना चाहिए।


इससे भी बदतर बात यह है कि जावास्क्रिप्ट आपको चरों को *बिना घोषित किए** मान निर्दिष्ट करने की सुविधा देता है:


```javascript
function greet() {
user = "Alice"
}
greet()
console.log(user) // prints "Alice"
```


इससे बिना किसी घोषणा के एक नया वैश्विक चर `name` बन जाता है। यह चुपचाप हो सकता है और बग्स उत्पन्न कर सकता है जिन्हें Hard द्वारा ट्रैक करना होगा, खासकर अगर यह केवल एक टाइपो हो। चरों को हमेशा `let` या `const` का उपयोग करके घोषित करें।


### कमजोर प्रकार की प्रणाली


जावास्क्रिप्ट कमज़ोर टाइपिंग वाली है, यानी ज़रूरत पड़ने पर यह स्वचालित रूप से एक प्रकार के मानों को दूसरे प्रकार में बदल देती है। इसे टाइप कॉर्शन कहते हैं, और हालाँकि यह सुविधाजनक हो सकता है, लेकिन इससे अक्सर भ्रामक परिणाम सामने आते हैं।


उदाहरण के लिए:


```javascript
console.log("5" + 1)    // "51"
console.log("5" - 1)    // 4
console.log(true + 1)   // 2
console.log(null + 1)   // 1
```


इन उदाहरणों में, जावास्क्रिप्ट यह अनुमान लगाने की कोशिश करता है कि आपका क्या मतलब था। कभी-कभी यह स्ट्रिंग्स को संख्याओं में, बूलियन्स को संख्याओं में, या स्ट्रिंग्स को स्ट्रिंग्स में बदल देता है। इससे आपका कोड अप्रत्याशित तरीके से व्यवहार कर सकता है।


जावास्क्रिप्ट की कमज़ोर टाइपिंग प्रणाली के बारे में जागरूक होना ज़रूरी है। जब चीज़ें अजीब तरह से काम करने लगें, तो यह अप्रत्याशित टाइप ज़बरदस्ती के कारण हो सकता है।


### `"use strict"`


आप एक सख्त मोड सक्षम कर सकते हैं जो कुछ मौन त्रुटियों को वास्तविक त्रुटियों में बदल देता है, और आपको भाषा की कुछ अधिक खतरनाक विशेषताओं का उपयोग करने से रोकता है।


इस सख्त मोड को सक्षम करने के लिए, अपनी फ़ाइल या फ़ंक्शन के शीर्ष पर यह पंक्ति जोड़ें:


```javascript
"use strict"
```


उदाहरण के लिए:


```javascript
"use strict"
name = "Alice" // ReferenceError: name is not defined
```


स्ट्रिक्ट मोड के बिना, जावास्क्रिप्ट चुपचाप `name` नामक एक ग्लोबल वैरिएबल बना देगा। लेकिन स्ट्रिक्ट मोड के साथ, यह एक वास्तविक त्रुटि बन जाती है, जिससे आपको बग्स को पहले ही पकड़ने में मदद मिलती है।


सख्त मोड जावास्क्रिप्ट की कुछ पुरानी सुविधाओं को भी निष्क्रिय कर देता है, और आपके कोड को अनुकूलित और रखरखाव करना आसान बनाता है।


## मूल्य बनाम संदर्भ

<chapterId>bb898425-dc2f-5e5c-864b-0cb7a4a9aea9</chapterId>


जावास्क्रिप्ट विभिन्न प्रकार के मानों को अलग-अलग तरीकों से व्यवहार करता है।


जब आप कुछ मानों को किसी चर में निर्दिष्ट करते हैं तो वे **कॉपी** हो जाते हैं।


जब आप उन्हें किसी नए चर में निर्दिष्ट करते हैं तो अन्य **साझा** हो जाते हैं, इसलिए यदि आप एक को बदलते हैं, तो दूसरा भी बदल जाता है।


### मान से पास करें


जब कोई मान **मूल्य द्वारा** पारित किया जाता है, तो जावास्क्रिप्ट उसकी **प्रतिलिपि** बना लेता है।


इसलिए यदि आप एक को बदलते हैं तो इसका दूसरे पर कोई प्रभाव नहीं पड़ता।


ऐसा आदिम प्रकारों के साथ होता है, जैसे:



- नंबर
- तार
- बूलियन (`सत्य` और `झूठ`)
- `शून्य`
- `अपरिभाषित`


आइये एक उदाहरण देखें:


```javascript
let a = 5
let b = a

b = 10

console.log(a) // 5
console.log(b) // 10
```


हमने `b` को `a` का मान दिया, लेकिन फिर हमने `b` को `10` में बदल दिया।


चूँकि संख्याएँ मान द्वारा प्रेषित होती हैं, इसलिए जावास्क्रिप्ट ने `5` को `b` में कॉपी कर लिया। `b` में `5`, `a` में मूल `5` से स्वतंत्र है, इसलिए `b` का मान बदलने से `a` पर कोई प्रभाव नहीं पड़ा।


आइये एक स्ट्रिंग के साथ प्रयास करें:


```javascript
let name1 = "Alice"
let name2 = name1

name2 = "Bob"

console.log(name1) // Alice
console.log(name2) // Bob
```


पुनः, `name2` को बदलने से `name1` प्रभावित नहीं हुआ, क्योंकि स्ट्रिंग्स भी मान द्वारा पास की जाती हैं।


जब आप किसी फ़ंक्शन में प्रिमिटिव पास करते हैं, तो भी यही होता है: वह कॉपी हो जाता है। इसलिए फ़ंक्शन मूल को बदल नहीं सकता।


```javascript
function plusOne(x) {
x = x + 1
}

let number = 5

plusOne(number)

console.log(number) // 5
```


यद्यपि फ़ंक्शन के अंदर `1` को `x` में जोड़ा गया था, फिर भी मूल `number` नहीं बदला।


ऐसा इसलिए है क्योंकि फ़ंक्शन में केवल एक प्रतिलिपि ही पास की गई थी।


### संदर्भ द्वारा पास


ऑब्जेक्ट्स को **संदर्भ द्वारा** पास किया जाता है।


इसका मतलब यह है कि उन्हें कॉपी करने के बजाय, जावास्क्रिप्ट बस उन्हें एक संदर्भ देता है, और यदि आप इसे संशोधित करते हैं, तो अन्य सभी चर जो इसे इंगित करते हैं, वे भी परिवर्तन देखेंगे।


उदाहरण के लिए:


```javascript
const person1 = { name: "Alice" }
const person2 = person1

person2.name = "Bob"

console.log(person1.name) // Bob
console.log(person2.name) // Bob
```


`person1` और `person2` दोनों मेमोरी में एक ही ऑब्जेक्ट की ओर इशारा करते हैं।


इसलिए जब हमने `person2.name` को बदला, तो हमने `person1.name` को भी बदल दिया, क्योंकि वे दोनों एक ही चीज़ को देख रहे हैं।


ऐरे ऑब्जेक्ट्स हैं, तो आइए एक ऐरे के साथ भी यही प्रयास करें:


```javascript
const list1 = [1, 2, 3]
const list2 = list1

list2.push(4)

console.log(list1) // [1, 2, 3, 4]
console.log(list2) // [1, 2, 3, 4]
```


हमने `4` को `list2` में धकेल दिया, लेकिन `list1` भी प्रभावित हुआ, क्योंकि वे दोनों एक ही सरणी को संदर्भित करते हैं।


आइए देखें कि जब हम किसी ऑब्जेक्ट को फ़ंक्शन में पास करते हैं तो क्या होता है:


```javascript
function rename(user) {
user.name = "Charlie"
}

const person = { name: "Dana" }

rename(person)

console.log(person.name) // Charlie
```


फ़ंक्शन ने ऑब्जेक्ट बदल दिया! ऐसा इसलिए क्योंकि उसे मूल `व्यक्ति` ऑब्जेक्ट का **संदर्भ** प्राप्त हुआ था।


उसे कोई प्रतिलिपि नहीं मिली। उसे मूल वस्तु तक पहुँच मिली, और इसके साथ ही उसे संशोधित करने की क्षमता भी मिली।


इस अंतर को याद रखना ज़रूरी है, क्योंकि अन्यथा हमारा कोड हमारी अपेक्षा से अलग व्यवहार कर सकता है। उदाहरण के लिए, हम एक फ़ंक्शन इस उम्मीद के साथ लिख सकते हैं कि वह प्राप्त होने वाले तर्कों को संशोधित नहीं करेगा, और बाद में पता चलता है कि वह वास्तव में उन्हें संशोधित कर रहा था (क्योंकि वे ऑब्जेक्ट थे, इसलिए उन्हें संदर्भ द्वारा पास किया गया था)।


## कार्यों के साथ कार्य करना

<chapterId>e0d277a8-c642-5af7-9e53-dee27c811967</chapterId>


आप पहले ही सीख चुके हैं कि जावास्क्रिप्ट में फ़ंक्शन कैसे घोषित और उपयोग किए जाते हैं। लेकिन जावास्क्रिप्ट आपको फ़ंक्शन के साथ प्रभावशाली तरीके से काम करने के लिए और भी टूल देता है।


### तीर फ़ंक्शन


एरो फ़ंक्शन, फ़ंक्शन लिखने का एक छोटा तरीका है। `function` कीवर्ड के बजाय, हम एरो (`=>`) का उपयोग करते हैं।


यहाँ एक सामान्य कार्य है:


```javascript
function greet(name) {
return `Hello, ${name}!`
}
```


तीर संस्करण इस तरह दिखता है:


```javascript
const greet = (name) => {
return `Hello, ${name}!`
}
```


यदि फ़ंक्शन में **केवल एक पंक्ति** है, तो आप ब्रेसेज़ को छोड़ सकते हैं और `return` कर सकते हैं:


```javascript
const greet = (name) => `Hello, ${name}!`
```


यदि फ़ंक्शन में **केवल एक पैरामीटर** है, तो आप पैरामीटर के चारों ओर कोष्ठक को छोड़ भी सकते हैं:


```javascript
const greet = name => `Hello, ${name}!`
```


आधुनिक जावास्क्रिप्ट में एरो फंक्शन बहुत आम हैं, क्योंकि वे सरल फंक्शन को काफी कम बॉयलरप्लेट के साथ व्यक्त करने की अनुमति देते हैं।


### डिफ़ॉल्ट पैरामीटर


कभी-कभी आप चाहते हैं कि यदि कोई तर्क पारित नहीं किया गया है तो फ़ंक्शन का एक **डिफ़ॉल्ट मान** हो।


आप ऐसा इस प्रकार कर सकते हैं:


```javascript
function sayHello(name = "friend") {
console.log(`Hello, ${name}!`)
}

sayHello("Alice") // Hello, Alice!
sayHello()        // Hello, friend!
```


जब कुछ भी पास नहीं किया जाता है तो डिफ़ॉल्ट मान `"friend"` का उपयोग किया जाता है।


### प्रसार पैरामीटर (`...`)


यदि आपका फ़ंक्शन लचीले तर्कों की संख्या लेता है तो क्या होगा?


आप उन्हें एक सरणी में इकट्ठा करने के लिए **स्प्रेड ऑपरेटर** (`...`) का उपयोग कर सकते हैं:


```javascript
function logAll(...items) {
console.log(items)
}

logAll(1, 2, 3) // [1, 2, 3]
logAll("a", "b") // ["a", "b"]
```


फिर आप प्रत्येक आइटम को संसाधित करने के लिए लूप का उपयोग कर सकते हैं:


```javascript
function logEach(...items) {
for (const item of items) {
console.log(item)
}
}
```


स्प्रेड ऑपरेटर तब उपयोगी होता है जब आपको पता नहीं होता कि कितने तर्क पारित किए जाएंगे।


### उच्च-क्रम फ़ंक्शन


**उच्च-क्रम फ़ंक्शन** एक फ़ंक्शन है जो:



- इनपुट के रूप में एक अन्य फ़ंक्शन लेता है
- और/या आउटपुट के रूप में एक फ़ंक्शन लौटाता है


यहाँ एक सरल उदाहरण है:


```javascript
function runTwice(action) {
action()
action()
}

function sayHello(name = "friend") {
console.log(`Hello, ${name}!`)
}

runTwice(sayHello)
```


यह प्रिंट करता है:


```
Hello, friend!
Hello, friend!
```


हम इसमें एक एरो फ़ंक्शन पास कर सकते हैं:


```javascript
runTwice(
() => console.log("Hello!")
)
```


यह प्रिंट करता है:


```
Hello!
Hello!
```


आप ऐसे फ़ंक्शन भी लिख सकते हैं जो अन्य फ़ंक्शन लौटाते हैं:


```javascript
function makeGreeter(name) {
return () => console.log(`Hi, ${name}`)
}

const greetAlice = makeGreeter("Alice")
const greetBob = makeGreeter("Bob")

greetAlice() // Hi, Alice
greetBob() // Hi, Bob
```


`makeGreeter` फ़ंक्शन एक ऐसा फ़ंक्शन है जो अन्य फ़ंक्शन बनाता है। यह एक स्ट्रिंग प्राप्त करता है और एक नया फ़ंक्शन लौटाता है जो उस स्ट्रिंग का उपयोग अपने `console.log` कॉल में करता है।


इस प्रकार का पैटर्न बहुत शक्तिशाली है, क्योंकि यह आपको अपने कार्यों में "छिद्र" छोड़ने की अनुमति देता है, जिसे आप बाद में अपनी आवश्यकता के अनुसार व्यवहार से भर सकते हैं।


### `मैप()`, `फ़िल्टर()`, `कम करें()`


जावास्क्रिप्ट आपको सरणियों के साथ उपयोग करने के लिए कुछ उपयोगी अंतर्निहित विधियाँ प्रदान करता है।


ये विधियाँ फ़ंक्शन को तर्क के रूप में लेती हैं, इसलिए वे उच्च-क्रम फ़ंक्शन भी हैं।


`map()` सरणी में प्रत्येक आइटम को किसी अन्य चीज़ में बदल देता है।


उदाहरण:


```javascript
const numbers = [1, 2, 3]
const doubled = numbers.map(x => x * 2)

console.log(doubled) // [2, 4, 6]
```


प्रत्येक संख्या को दोगुना किया जाता है, और परिणाम एक नई सरणी होती है।


यदि आइटम परीक्षण में पास नहीं होते हैं तो `filter()` उन्हें सरणी से हटा देता है।


उदाहरण:


```javascript
const numbers = [1, 2, 3, 4, 5]
const numbersGreaterThanTwo = numbers.filter(x => x > 2)

console.log(numbersGreaterThanTwo) // [3, 4, 5]
```


केवल वे सरणी प्रविष्टियाँ रखी जाती हैं जिनके लिए `x > 2` स्थिति `true` लौटाती है।


`reduce()` का उपयोग किसी सरणी में सभी आइटमों को एक एकल मान में संयोजित करने के लिए किया जाता है।


उदाहरण:


```javascript
const numbers = [1, 2, 3, 4]
const total = numbers.reduce((current, next) => current + next)

console.log(total) // 10
```


`reduce` सरणी से होकर गुजरता है और, इस उदाहरण में, `1` और `2` के बीच `+` ऑपरेटर लागू करता है, फिर परिणामी `3` और `3` के बीच, फिर परिणामी `6` और `4` के बीच, जब तक कि इसमें सभी सरणी प्रविष्टियों का योग नहीं हो जाता (जो कि 10 है)।


आप `reduce()` का उपयोग कई चीजों के लिए कर सकते हैं, जैसे कुल योग, औसत, या चरण दर चरण नए मान बनाना।


ये विधियाँ (`map`, `filter`, `reduce`) मैन्युअल लूप लिखे बिना डेटा को संसाधित करने के लिए शक्तिशाली उपकरण हैं।


आप इन्हें परिचालनों की एक श्रृंखला में भी संयोजित कर सकते हैं, जैसे:


```javascript
const numbers = [1, 2, 3, 4, 5]

const result = numbers
.map(n => n * 2)        // Double each entry, obtain [2, 4, 6, 8, 10]
.filter(n => n > 3) // Keep only the entries bigger than 3, so you get [4, 6, 8, 10]
.reduce((n1, n2) => n1 + n2) // Adds them: 4 + 6 + 8 + 10 = 28

console.log(result) // 28
```


## वस्तुओं के साथ कार्य करना

<chapterId>7842aada-f009-5518-b8e3-1104e166a035</chapterId>


इस अध्याय में, हम जावास्क्रिप्ट में ऑब्जेक्ट्स के साथ काम करने के लिए कुछ शक्तिशाली और थोड़े अधिक उन्नत टूल सीखेंगे।


### निजी संपत्तियां


कभी-कभी, हम किसी ऑब्जेक्ट के किसी गुण को छिपाना चाहते हैं ताकि उसे ऑब्जेक्ट के बाहर से बदला या एक्सेस न किया जा सके। जावास्क्रिप्ट हमें गुण के नाम से पहले `#` लगाकर ऐसा करने का एक तरीका देता है। इससे एक **निजी** गुण बनता है, जिसे केवल क्लास के अंदर से ही एक्सेस किया जा सकता है।


```javascript
class Person {
#age // this is a private property

constructor(name, age) {
this.name = name
this.#age = age
}

getAge() {
return this.#age
}
}

const alice = new Person("Alice", 30)
console.log(alice.name)      // Alice
console.log(alice.getAge())  // 30, the method can access the private property
console.log(alice.#age)      // ❌ Error! You can't access private properties directly
```


जब आप आकस्मिक परिवर्तनों को रोकना चाहते हैं तो निजी गुण उपयोगी होते हैं।


### `स्थैतिक` गुण


कभी-कभी, आप चाहते हैं कि कोई प्रॉपर्टी क्लास से ही संबंधित हो, न कि उस क्लास से बनाए गए हर ऑब्जेक्ट से। `static` इसी के लिए है। एक `static` प्रॉपर्टी क्लास में ही होती है और उस क्लास के सभी ऑब्जेक्ट उसे संदर्भित करेंगे।


```javascript
class User {
static counter = 0 // this belongs to the class, not to instances. The same counter will be shared by all objects

constructor() {
User.counter++ // changes the static property every time an object of this class gets initiated
}
}

const a = new User() // the constructor will change the shared counter from 0 to 1
const b = new User() // the constructor will change the shared counter from 1 to 2

console.log(User.counter) //  prints 2
```


यह साझा डेटा और विधियों को संग्रहीत करने के लिए उपयोगी है जो केवल एक ही नहीं बल्कि पूरे ऑब्जेक्ट समूह पर लागू होता है।


### `get` और `set`


जावास्क्रिप्ट में, `get` और `set` आपको ऐसे गुण बनाने देते हैं जो सामान्य चरों की तरह दिखते हैं, लेकिन वास्तव में पृष्ठभूमि में विशेष कोड चलाते हैं।


जब आप किसी प्रॉपर्टी को *पढ़ने* का प्रयास करते हैं, तो एक `get`ter विधि चलती है। इसे प्रॉपर्टी के नाम वाली विधि से पहले `get` लिखकर घोषित किया जाता है।


```javascript
class User {
constructor(firstName, lastName) {
this.firstName = firstName
this.lastName = lastName
}

get fullName() {
return `${this.firstName} ${this.lastName}`
}
}

const user = new User("Jane", "Doe")
console.log(user.fullName) // Jane Doe
```


यद्यपि `fullName` एक नियमित गुण नहीं है, फिर भी हम इसका उपयोग एक नियमित गुण की तरह कर सकते हैं, तथा पर्दे के पीछे यह पूर्ण नाम बनाने के लिए `get` फ़ंक्शन चलाता है।


जब आप किसी प्रॉपर्टी को कोई मान *निर्दिष्ट* करते हैं, तो एक `set`ter विधि चलती है। यह आपको यह नियंत्रित करने देती है कि जब कोई उस मान को बदलने का प्रयास करता है तो क्या होता है। इसे प्रॉपर्टी के नाम वाली विधि से पहले `set` लिखकर घोषित किया जाता है।


```javascript
class User {
constructor() {
this.firstName = "John"
this.lastName = "Doe"
}

get fullName() {
return `${this.firstName} ${this.lastName}`
}

set fullName(input) {            // gets the name that is passed
const parts = input.split(" ") // breaks it into parts
this.firstName = parts[0]      // uses the first part as first name
this.lastName = parts[1]       // uses the second part as last name
}
}

const user = new User()
user.fullName = "John Smith"

console.log(user.firstName) // John
console.log(user.lastName)  // Smith
```


जब हम `user.fullName = "John Smith"` करते हैं, तो यह `set` विधि चलाता है और `firstName` तथा `lastName` मानों को अद्यतन करता है।


इसलिए भले ही ऐसा लगता है कि हम सिर्फ एक साधारण चर सेट कर रहे हैं, हम वास्तव में तर्क को ट्रिगर कर रहे हैं जो अन्य गुणों को अपडेट करता है।


## कुंजियाँ और मान

<chapterId>01a397b8-c12a-5c39-82b3-6d9ebbb72a29</chapterId>


जावास्क्रिप्ट ऑब्जेक्ट में प्रत्येक प्रॉपर्टी की एक कुंजी (जिसे प्रॉपर्टी नाम भी कहा जाता है) और एक मान होता है।


उदाहरण के लिए:


```javascript
const user = {
name: "Alice",
age: 30
}
```


इस ऑब्जेक्ट में, `"name"` और `"age"` कुंजियाँ हैं, और "Alice" और `30` उनके मान हैं।


### गतिशील पहुँच


कभी-कभी, आपको किसी प्रॉपर्टी का नाम पहले से पता नहीं होता... हो सकता है कि आप उसे यूज़र इनपुट से प्राप्त कर रहे हों, या किसी वेरिएबल से पढ़ रहे हों। आप इसे **ब्रैकेट नोटेशन**, जैसे `myObject["keyName"]` का उपयोग करके भी एक्सेस कर सकते हैं।


```javascript
const user = {
name: "Alice",
age: 30
}

console.log(user["name"]) // Alice
```


हमने संबंधित मान प्राप्त करने के लिए स्ट्रिंग `name` को ऑब्जेक्ट में पास किया।


हम किसी चर में एक कुंजी सहेज सकते हैं और बाद में संबंधित मान तक पहुँचने के लिए इसका उपयोग कर सकते हैं, जैसे


```javascript
const user = {
name: "Alice",
age: 30
}

const key = "name"

console.log(user[key]) // Alice
```


### डायनेमिक Assignment


आप कुंजी के रूप में चर का उपयोग करके ऑब्जेक्ट गुण भी बना या अपडेट कर सकते हैं।


```javascript
const settings = {}

const key = "theme"
settings[key] = "dark"

console.log(settings) // { theme: "dark" }
```


यह तब उपयोगी होता है जब आप किसी ऑब्जेक्ट को चरणबद्ध तरीके से बनाना चाहते हैं। उदाहरण के लिए:


```javascript
const user = {}

user["username"] = "bananaFan123"
user["email"] = "banana@fruit.com"

console.log(user)
// { username: "bananaFan123", email: "banana@fruit.com" }
```


आप वर्गाकार कोष्ठकों का उपयोग करके ऑब्जेक्ट बनाते समय एक गतिशील कुंजी का भी उपयोग कर सकते हैं:


```javascript
const key = "language"
const config = {
[key]: "JavaScript"
}

console.log(config.language) // JavaScript
```


इसे **गणना गुण** कहा जाता है। वर्गाकार कोष्ठकों के अंदर दिए गए मान का मूल्यांकन किया जाता है, और परिणाम को कुंजी के रूप में उपयोग किया जाता है।


### `प्रतीक` प्रकार


स्ट्रिंग्स के अतिरिक्त, जावास्क्रिप्ट आपको ऑब्जेक्ट कुंजी के रूप में `Symbol` नामक एक विशेष प्रकार का उपयोग करने की भी सुविधा देता है।


आइये एक सरल उदाहरण से शुरू करें:


```javascript
const id = Symbol("userID")

const user = {
name: "Bob",
[id]: 12345
}

console.log(user[id]) // 12345
```


इस उदाहरण में, `id` एक प्रतीक है। यह एक स्ट्रिंग नहीं है, लेकिन फिर भी एक कुंजी के रूप में काम करता है। यदि आप `user` को कंसोल में लॉग इन करने का प्रयास करते हैं, तो आपको यह दिखाई देगा:


```javascript
console.log(user) // { name: 'Bob', [Symbol(userID)]: 12345 }
```


एक और महत्वपूर्ण बात: आपके द्वारा बनाया गया प्रत्येक प्रतीक अद्वितीय है, भले ही वे एक ही स्ट्रिंग का उपयोग करके बनाए गए हों।


```javascript
const a = Symbol("label")
const b = Symbol("label")

console.log(a === b) // false
```


प्रतीक आपको ऐसी कुंजियाँ परिभाषित करने की अनुमति देते हैं जो सामान्य कुंजियों से टकराएँगी नहीं। उदाहरण के लिए, मान लीजिए कि आपके पास `name` प्रॉपर्टी वाला एक ऑब्जेक्ट है, लेकिन भविष्य में कोई उपयोगकर्ता उस ऑब्जेक्ट को ऐसे तरीकों से अनुकूलित कर सकेगा जिनका आप अनुमान नहीं लगा सकते, और वह उपयोगकर्ता `name` प्रॉपर्टी भी जोड़ सकता है। यदि मूल `name` प्रॉपर्टी को एक स्ट्रिंग के साथ परिभाषित किया गया था, तो उसे नए द्वारा अधिलेखित कर दिया जाएगा, जैसे:


```javascript
const obj = {
name: "John"
}

obj.name = "Jimmy"

console.log(obj) // { name: 'Jimmy' }
```


यदि हम इसके स्थान पर प्रतीक का उपयोग करें:


```javascript
const name = Symbol("name")

const obj = {
[name]: "John"
}

obj.name = "Jimmy"

console.log(obj) // { name: 'Jimmy', [Symbol(name)]: 'John' }
```


जैसा कि आप देख सकते हैं, मूल `name` प्रॉपर्टी किसी तरह इसी तरह सुरक्षित रहती है। यह कुछ ख़ास मामलों में उपयोगी हो सकता है।


## उपयोगिता वस्तुएँ

<chapterId>516e74c8-2a11-545a-a4d1-c2cabb91a273</chapterId>


जावास्क्रिप्ट हमें कुछ उपयोगी अंतर्निहित ऑब्जेक्ट्स प्रदान करता है जो हमें डिबगिंग और गणितीय कार्यों जैसे कार्यों में मदद करते हैं।


### अन्य `कंसोल` विधियाँ


आपने पहले ही `console.log` देखा है, जो स्क्रीन पर मान प्रिंट करता है।


`कंसोल` ऑब्जेक्ट पर कुछ अन्य उपयोगी विधियां उपलब्ध हैं जो आपके प्रोग्राम को डीबग करने में आपकी सहायता कर सकती हैं।


#### `कंसोल.वार्न`


संदेश को पीले रंग में प्रिंट करता है (या कुछ वातावरणों में चेतावनी आइकन के साथ):


```javascript
console.warn("This is just a warning.")
```


#### `कंसोल.त्रुटि`


एक संदेश लाल रंग में प्रिंट करता है, जैसे कि एक त्रुटि:


```javascript
console.error("Something went wrong!")
```


#### `कंसोल.टेबल`


किसी सारणी या ऑब्जेक्ट को तालिका के रूप में प्रदर्शित करता है:


```javascript
const users = [
{ name: "Alice", age: 25 },
{ name: "Bob", age: 30 }
]

console.table(users)
```


यह इस प्रकार की तालिका प्रिंट करता है:


```
┌─────────┬────────┬─────┐
│ (index) │  name  │ age │
├─────────┼────────┼─────┤
│    0    │ 'Alice'│  25 │
│    1    │ 'Bob'  │  30 │
└─────────┴────────┴─────┘
```


यह संरचित डेटा को विज़ुअलाइज़ करने के लिए उपयोगी हो सकता है।


#### `console.time` और `console.timeEnd`


आप यह माप सकते हैं कि किसी चीज़ में कितना समय लगता है:


```javascript
console.time("timer")
for (let i = 0; i < 1000000; i++) {}
console.timeEnd("timer")
```


यह कुछ इस तरह प्रिंट होता है:


```
timer: 2.379ms
```


कुछ सरल प्रदर्शन परीक्षण के लिए उपयोगी.


### `गणित` वस्तु


जावास्क्रिप्ट आपको गणना करने के लिए उपयोगी विधियों के साथ एक `Math` ऑब्जेक्ट देता है।


#### `गणित.यादृच्छिक()`


0 (समावेशी) और 1 (अपवर्जित) के बीच एक यादृच्छिक संख्या लौटाता है:


```javascript
const r = Math.random()
console.log(r)
```


उदाहरण आउटपुट:


```
0.4387429859
```


#### `Math.floor()` और `Math.ceil()`



- `Math.floor(n)` निकटतम पूर्णांक तक पूर्णांकित करता है
- `Math.ceil(n)` निकटतम पूर्णांक तक पूर्णांकित करता है


```javascript
console.log(Math.floor(4.9)) // 4
console.log(Math.ceil(4.1))  // 5
```


#### `गणित.राउंड()`


निकटतम पूर्णांक तक पूर्णांकित करें:


```javascript
console.log(Math.round(4.4)) // 4
console.log(Math.round(4.6)) // 5
```


#### `Math.max()` और `Math.min()`


संख्याओं की सूची से सबसे बड़ा या सबसे छोटा मान लौटाता है:


```javascript
console.log(Math.max(5, 9, 3)) // 9
console.log(Math.min(5, 9, 3)) // 3
```


#### `Math.pow()` और `Math.sqrt()`



- `Math.pow(a, b)` आपको `a` की घात `b` देता है
- `Math.sqrt(n)` आपको `n` का वर्गमूल देता है


```javascript
console.log(Math.pow(2, 3))   // 8
console.log(Math.sqrt(16))    // 4
```


# उन्नत जावास्क्रिप्ट

## अन्य संग्रह

<chapterId>a9a70c6d-a343-5a46-a383-e288bc2700e3</chapterId>


जावास्क्रिप्ट हमें कुछ विशेष संग्रह प्रकार प्रदान करता है जो नियमित ऐरे और ऑब्जेक्ट से आगे जाते हैं। इनमें `मैप` और `सेट` शामिल हैं।


वे आपको मानों के समूहों को संग्रहीत और प्रबंधित करने में मदद करते हैं, लेकिन वे अब तक आपने जो देखा है उससे अलग तरीके से काम करते हैं।


एक `मैप` किसी ऑब्जेक्ट की तरह ही **कुंजी-मान युग्मों** का एक संग्रह होता है। लेकिन इसमें कुछ महत्वपूर्ण अंतर हैं:



- कुंजियाँ **कोई भी मान** हो सकती हैं, केवल स्ट्रिंग नहीं।
- वस्तुओं का क्रम संरक्षित रखा गया है।
- इसमें काम करना आसान बनाने के लिए अंतर्निहित विधियां हैं।


आप इस प्रकार एक नया मानचित्र बनाते हैं:


```javascript
const myMap = new Map()
```


इससे एक खाली मैप बनता है। इसमें प्रविष्टियाँ जोड़ने के लिए, `myMap.set(key, value)` का उपयोग करें:


```javascript
myMap.set("name", "Alice")
```


यह `"Alice"` मान के साथ एक कुंजी `"name"` जोड़ता है।


आप किसी संख्या को कुंजी के रूप में भी उपयोग कर सकते हैं:


```javascript
myMap.set(42, "The answer")
```


और यहां तक ​​कि एक वस्तु भी कुंजी के रूप में:


```javascript
const objKey = { id: 1 }
myMap.set(objKey, "Object as key")
```


यह नियमित ऑब्जेक्ट्स के साथ काम नहीं करेगा, जो केवल स्ट्रिंग कुंजियों की अनुमति देते हैं।


**मान प्राप्त करने के लिए**, `myMap.get(key)` का उपयोग करें:


```javascript
console.log(myMap.get("name"))     // Alice
console.log(myMap.get(42))         // The answer
console.log(myMap.get(objKey))     // Object as key
```


यह जांचने के लिए कि क्या कोई कुंजी मौजूद है, `myMap.has(key)` का उपयोग करें:


```javascript
console.log(myMap.has("name")) // true
```


**कुंजी हटाने के लिए**, `myMap.delete(key)` का उपयोग करें:


```javascript
myMap.delete("name")
```


**पूरा मानचित्र साफ़ करने के लिए**, `myMap.clear()` का उपयोग करें:


```javascript
myMap.clear()
```


मानचित्र, मानों के बड़े संग्रह को प्रबंधित करने के लिए बहुत अच्छे होते हैं, क्योंकि बड़े मानचित्र पर मानों तक पहुंचने से आमतौर पर बड़े ऑब्जेक्ट की तुलना में बेहतर प्रदर्शन मिलता है।


### `सेट`


एक `सेट` केवल मानों (बिना कुंजियों) का एक संग्रह होता है, जहाँ प्रत्येक मान अद्वितीय होना चाहिए। इसका अर्थ है:



- आपके पास एक ही मान दो बार नहीं हो सकता
- मान उसी क्रम में संग्रहीत किए जाते हैं जिस क्रम में आप उन्हें जोड़ते हैं


आप इस तरह एक सेट बनाते हैं:


```javascript
const mySet = new Set()
```


**मान जोड़ने** के लिए, `mySet.add(value)` का उपयोग करें:


```javascript
mySet.add(1)
mySet.add(2)
mySet.add(2) // duplicate, will be ignored
```


भले ही हमने `2` को दो बार जोड़ने का प्रयास किया, फिर भी सेट केवल एक ही प्रति रखेगा।


यह जांचने के लिए कि कोई मान सेट में है या नहीं, `mySet.has(value)` का उपयोग करें:


```javascript
console.log(mySet.has(2)) // true
console.log(mySet.has(3)) // false
```


**मान हटाने के लिए**, `mySet.delete(value)` का उपयोग करें:


```javascript
mySet.delete(2)
```


**सब कुछ साफ़ करने के लिए**, `mySet.clear()` का उपयोग करें:


```javascript
mySet.clear()
```


`सेट` तब उपयोगी होता है जब आप डुप्लिकेट की मैन्युअल जांच किए बिना अद्वितीय मानों का संग्रह रखना चाहते हैं:


```javascript
const numberArray = [1, 2, 2, 3, 4, 4, 4, 5]

const numberSet = new Set(numberArray)

console.log(numberSet) // Set(5) { 1, 2, 3, 4, 5 }
```


`सेट` आपके लिए डुप्लिकेट से बचता है।


## पुनरावर्तक

<chapterId>61d24e5e-b7e4-541a-8322-778f61f26a72</chapterId>


जावास्क्रिप्ट में अधिकांश चीजें जिन पर आप लूप कर सकते हैं (जैसे कि ऐरे, स्ट्रिंग्स, मैप्स, सेट) पुनरावृत्तीय हैं: वे अपनी सामग्री के लिए पुनरावृत्तक प्रदान कर सकते हैं।


**इटरेटर** जावास्क्रिप्ट में एक विशेष ऑब्जेक्ट है जो आपको **एक-एक करके** आइटमों की सूची से गुजरने में मदद करता है।


### `ऑब्जेक्ट` पुनरावर्तक


ऐरे या मैप्स के विपरीत, नियमित ऑब्जेक्ट्स `for...of` के साथ पुनरावृत्त नहीं होते। अगर आप यह कोशिश करें:


```javascript
const user = {
name: "Alice",
age: 30
}

for (const value of user) {
console.log(value)
}
```


आपको एक त्रुटि मिलेगी:


```
TypeError: user is not iterable
```


ऐसा इसलिए है क्योंकि साधारण ऑब्जेक्ट्स में कोई बिल्ट-इन इटरेटर नहीं होता। लेकिन जावास्क्रिप्ट आपको उन पर लूप करने के लिए दूसरे टूल देता है।


#### `ऑब्जेक्ट.कीज़()`


आप ऑब्जेक्ट की कुंजियों की एक सरणी प्राप्त करने के लिए `Object.keys(obj)` का उपयोग कर सकते हैं, और फिर उस पर लूप कर सकते हैं:


```javascript
const user = {
name: "Alice",
age: 30
}

const keys = Object.keys(user)

for (const key of keys) {
console.log(key)
}
```


यह प्रिंट करता है:


```
name
age
```


#### `ऑब्जेक्ट.वैल्यू()`


**मानों** पर लूप करने के लिए, `Object.values()` का उपयोग करें:


```javascript
const user = {
name: "Alice",
age: 30
}

const values = Object.values(user)

for (const value of values) {
console.log(value)
}
```


यह प्रिंट करता है:


```
Alice
30
```


#### `ऑब्जेक्ट.प्रविष्टियाँ()`


यदि आप **कुंजी और मान दोनों** चाहते हैं, तो `Object.entries()` का उपयोग करें:


```javascript
const user = {
name: "Alice",
age: 30
}

const entries = Object.entries(user)

for (const [key, value] of entries) {
console.log(`${key} is ${value}`)
}
```


यह प्रिंट करता है:


```
name is Alice
age is 30
```


यद्यपि ऑब्जेक्ट्स को सीधे पुनरावृत्त नहीं किया जा सकता, फिर भी ये विधियां आपको उनकी सामग्री तक पूर्ण पहुंच प्रदान करती हैं, जो `for...of` के साथ अच्छी तरह से काम करती है।


लेकिन इटरेटर्स कैसे काम करते हैं?


### `प्रतीक.इटरेटर`


सभी पुनरावृत्तीयों के पीछे का रहस्य एक विशेष प्रतीक है जिसे `Symbol.iterator` कहा जाता है।


यह प्रतीक एक अंतर्निहित कुंजी है जो जावास्क्रिप्ट को बताता है: "इस ऑब्जेक्ट को पुनरावृत्त किया जा सकता है।"


जब आप `myIterable[Symbol.iterator]()` को कॉल करते हैं, तो जावास्क्रिप्ट आपको `.next()` विधि के साथ एक **iterator ऑब्जेक्ट** देता है।


आइये देखें कि यह कैसा दिखता है:


```javascript
const colors = ["red", "green", "blue"]

const iterator = colors[Symbol.iterator]()

console.log(iterator.next()) // { value: 'red', done: false }
```


`.next()` को हर बार कॉल करने पर आपको अगला मान मिलता है। जब यह पूरा हो जाता है, तो यह रिटर्न देता है:


```javascript
{ value: undefined, done: true }
```


### `अगला()`


अनुक्रम से अगला आइटम प्राप्त करने के लिए `.next()` विधि का उपयोग किया जाता है।


हर बार जब आप `.next()` को कॉल करते हैं, तो आपको दो कुंजियों वाला एक ऑब्जेक्ट मिलता है:



- `value`: वर्तमान आइटम
- `done`: एक बूलियन जो आपको बताता है कि पुनरावृत्ति समाप्त हो गई है या नहीं


आइये एक पूर्ण उदाहरण देखें:


```javascript
const names = ["Lina", "Tom", "Eva"]      // declare an array
const iterator = names[Symbol.iterator]() // use the Symbol.iterator function to get an iterator for this array

let result = iterator.next()              // get the first element of the array

while (!result.done) {                    // repeat this loop until you reach the last element of the array, which is marked with { done: true }
console.log(result.value)               // print the value of each element
result = iterator.next()                // get the next element of the array
}
```


यह प्रिंट करता है:


```
Lina
Tom
Eva
```


`for...of` लूप इस प्रकार काम करता है: यह `.next()` के साथ इस पैटर्न का उपयोग करता है।


आपको वही परिणाम मिलेगा


```javascript
const names = ["Lina", "Tom", "Eva"]

for (const result of names) {
console.log(result)
}
```


### किसी वर्ग को पुनरावृत्तीय बनाना


आप `[Symbol.iterator]()` विधि जोड़कर अपना स्वयं का **iterable class** भी परिभाषित कर सकते हैं।


मान लीजिए कि हम एक ऐसा वर्ग चाहते हैं जो संख्याओं की एक श्रृंखला को दर्शाता है, जैसे 1 से 5 तक।


```javascript
class Range {
constructor(start, end) {
this.start = start
this.end = end
}

[Symbol.iterator]() {
let current = this.start
const end = this.end

return {
next() {
if (current <= end) {
const result = { value: current, done: false }
current = current + 1
return result
} else {
return { done: true }
}
}
}
}
}

const myRange = new Range(1, 5)

for (const num of myRange) {
console.log(num)
}
```


यह प्रिंट करता है:


```
1
2
3
4
5
```


यहाँ क्या हो रहा है:



- हमने एक वर्ग `Range` परिभाषित किया है
- क्लास के अंदर, हमने `[Symbol.iterator]()` को लागू किया है, इसलिए जावास्क्रिप्ट जानता है कि इसे कैसे पुनरावृत्त करना है
- `next()` विधि प्रत्येक संख्या को एक-एक करके वापस देती है
- जब हम `end` पर पहुँचते हैं, तो यह `{ done: true }` लौटाता है


अब हमारा `Range` वर्ग एक सारणी की तरह काम करता है, और हम इसका उपयोग किसी भी लूप में कर सकते हैं जो एक पुनरावृत्तीय की अपेक्षा करता है।


### जनरेटर फ़ंक्शन और `yield`


इटरेटर्स बनाना आसान बनाने के लिए, जावास्क्रिप्ट आपको `function*` कीवर्ड (यह `function` है जिसके अंत में `*` है) और `yield` कीवर्ड का उपयोग करके **जेनरेटर फ़ंक्शन** देता है।


आओ कोशिश करते हैं:


```javascript
function* numberGenerator() {
yield 1
yield 2
yield 3
}

const iterator = numberGenerator()

console.log(iterator.next()) // { value: 1, done: false }
console.log(iterator.next()) // { value: 2, done: false }
console.log(iterator.next()) // { value: 3, done: false }
console.log(iterator.next()) // { value: undefined, done: true }
```


प्रत्येक `yield` एक मान देता है, और अगले `.next()` के कॉल होने तक फ़ंक्शन को **रोक** देता है।


आप `for...of` के साथ जनरेटर पर भी लूप कर सकते हैं:


```javascript
for (const num of numberGenerator()) {
console.log(num)
}
```


यह प्रिंट करता है:


```
1
2
3
```


## कॉलबैक के साथ समवर्तीता

<chapterId>f3fc76ca-b3ef-54eb-a06e-501007002054</chapterId>


अब तक, हमारा कोड **सिंक्रोनस** रहा है: यह एक बार में एक पंक्ति, क्रम से चलता है। लेकिन वास्तविक दुनिया में कुछ चीज़ें समय लेती हैं, और हम नहीं चाहते कि पूरा प्रोग्राम प्रतीक्षा करते हुए रुक जाए।


इस अध्याय में हम एक नई अवधारणा पेश करने जा रहे हैं: **समवर्तीता**। यह हमें कार्यों के क्रम में बदलाव करने की अनुमति देती है। यह टाइमर, उपयोगकर्ता इनपुट, या डिस्क से फ़ाइलें पढ़ने जैसे कार्यों में उपयोगी है। जावास्क्रिप्ट समवर्तीता के लिए विभिन्न उपकरण प्रदान करता है।


### `सेटटाइमआउट`


फ़ंक्शन `setTimeout` आपको कुछ समय बीत जाने के बाद **फ़ंक्शन को बाद में चलाने** की सुविधा देता है।


उदाहरण:


```javascript
console.log("Start")

setTimeout(
() => console.log("This runs after 2 seconds"),
2000
)

console.log("End")
```


यह प्रिंट करता है:


```
Start
End
This runs after 2 seconds
```


भले ही `setTimeout` कोड के बीच में दिखाई देता है, लेकिन यह बाकी को ब्लॉक नहीं करता। इसके बजाय, यह फ़ंक्शन को बाद में चलाने के लिए शेड्यूल करता है और तुरंत आगे बढ़ जाता है।


`2000` का अर्थ है 2000 मिलीसेकंड (जो कि 2 सेकंड है)।

यहां **कॉलबैक** और **प्रॉमिस** अनुभागों का अधिक विस्तृत और शुरुआती-अनुकूल पुनर्लेखन दिया गया है, जिसमें संपूर्ण डेटा में हेरफेर और स्पष्ट एनोटेशन का उपयोग किया गया है:


### कॉलबैक


**कॉलबैक** एक फ़ंक्शन है जिसे हम किसी अन्य फ़ंक्शन को देते हैं ताकि उसे **बाद में कॉल किया जा सके**।


आइए संख्याओं का उपयोग करते हुए एक वास्तविक उदाहरण देखें। कल्पना कीजिए कि हमारे पास संख्याओं की एक सूची है, और हम उनमें से प्रत्येक को दोगुना करना चाहते हैं, और फिर परिणामी "दोगुनी" सरणी पर एक फ़ंक्शन (कॉलबैक) लागू करना चाहते हैं, लेकिन हम इसे थोड़ी देरी के बाद करना चाहते हैं, मानो हम किसी धीमी गति (जैसे इंटरनेट से डेटा लोड करना) का इंतज़ार कर रहे हों।


यहां एक फ़ंक्शन है जो **कॉलबैक** का उपयोग करके ऐसा करता है:


```javascript
function doubleNumbers(numbersArray, callback) {
// Pretend we're doing a slow operation using setTimeout
setTimeout(() => {
// Use the map method to create a new array where each number is doubled
const doubled = numbersArray.map(n => n * 2)

// When we're done, we call the callback function with the result
callback(doubled)
}, 1000) // Wait 1 second before running the code inside
}
```


आइये इस फ़ंक्शन का उपयोग करने का प्रयास करें:


```javascript
const input = [1, 2, 3]

doubleNumbers(input, function(result) {
console.log("Here is the doubled array:", result)
})
```


1 सेकंड के बाद, यह प्रिंट होता है:


```
Here is the doubled array: [ 2, 4, 6 ]
```


**यहाँ क्या हो रहा है?**


1. हम `input` को उन संख्याओं की सूची के रूप में पास करते हैं जिन्हें हम दोगुना करना चाहते हैं।

2. हम एक कॉलबैक फ़ंक्शन भी पास करते हैं जो प्रोग्राम को बताता है कि दोहरीकरण के बाद क्या करना है।

3. `doubleNumbers` के अंदर, हम `setTimeout` का उपयोग करके विलंब का अनुकरण करते हैं, फिर हम दोहरीकरण करते हैं।

4. एक बार ऐसा हो जाने पर, हम परिणामी "दोगुनी" सरणी पर कॉलबैक कॉल करते हैं।


यह तकनीक काम करती है, लेकिन कल्पना कीजिए कि आप इसके बाद **और कदम** उठाना चाहते हैं, जैसे छोटी संख्याओं को छांटना और फिर उन्हें जोड़ना। आपको इस तरह और कॉलबैक **नेस्ट** करने होंगे:


```javascript
doubleNumbers(input, function(doubled) {
filterBigNumbers(doubled, function(filtered) {
sumNumbers(filtered, function(total) {
console.log("Final result:", total)
})
})
})
```


यह Hard पढ़ने में बहुत ही गड़बड़ और अव्यवस्थित है। इस शैली को **कॉलबैक हेल** कहा जाता है, और `Promise` को ठीक इसी के लिए बनाया गया था।


## वादों के साथ समवर्तीता

<chapterId>30fddaca-729f-5c8d-bf86-8dfc7b3c9800</chapterId>


`वादा` एक अंतर्निहित जावास्क्रिप्ट ऑब्जेक्ट है जो एक ऐसे मान का प्रतिनिधित्व करता है जो **भविष्य में तैयार होगा**।


हम इस तरह एक वादा बना सकते हैं:


```javascript
const promise = new Promise((resolve, reject) => {
// Do something that takes time here...

resolve("It worked!") // This means everything went OK
})
```


`new Promise()` भाग वादा बनाता है।


इसके अंदर, हम इसे दो पैरामीटर वाला एक फ़ंक्शन देते हैं:



- `resolve`, एक फ़ंक्शन है जिसे हम तब कॉल करते हैं जब सब कुछ सफल हो जाता है
- `अस्वीकार`, एक फ़ंक्शन है जिसे हम तब कॉल करते हैं जब कुछ गलत हो जाता है


उपरोक्त उदाहरण में, हम इसे तुरंत ``यह काम कर गया!`` संदेश के साथ हल कर देते हैं।


### `.then()`


वादा पूरा होने के बाद कुछ करने के लिए, हम `.then()` का उपयोग करते हैं:


```javascript
const promise = new Promise((resolve, reject) => {
// Do something that takes time here...

resolve(100) // This means everything went OK
})

promise.then(result => {
console.log("The result is:", result)
})
```


यह प्रिंट करता है:


```
The result is: 100
```


`resolve()` को हमने जो मान दिया है, वह `.then()` के अंदर फ़ंक्शन को `result` के रूप में भेज दिया जाता है।


आइए `setTimeout` का उपयोग करके 2 सेकंड का समय लेने वाले कार्य का अनुकरण करें:


```javascript
const delayedPromise = new Promise(
(resolve, reject) => {
setTimeout(
() => resolve("Done waiting!"),
2000
)
})

delayedPromise.then(result => console.log(result))
```


यह 2 सेकंड प्रतीक्षा करेगा और फिर प्रिंट करेगा:


```
Done waiting!
```


### `अस्वीकार()`


आइये एक ऐसा वादा करें जो असफल हो:


```javascript
const failingPromise = new Promise((resolve, reject) => {
reject("Something went wrong")
})
```


अब यदि हम इस पर `.then()` का उपयोग करते हैं, तो कुछ नहीं होगा, क्योंकि `.then()` केवल सफलता को संभालता है।


त्रुटियों को संभालने के लिए, हम `.catch()` का उपयोग करते हैं:


```javascript
const failingPromise = new Promise((resolve, reject) => {
reject("Something went wrong")
})

failingPromise
.then(
result => console.log("This will NOT run:", result)
)
.catch(
error => console.log("Caught an error:", error)
)
```


यह केवल प्रिंट करता है


```
Caught an error: Something went wrong
```


`reject()` को दिया गया मान `.catch()` फ़ंक्शन को भेजा जाता है।


आइए एक ऐसा वादा बनाएं जो **कभी काम करता है और कभी विफल हो जाता है**, कुछ शर्तों के आधार पर।


```javascript
function checkNumber(n) {
return new Promise((resolve, reject) => {
if (n > 0) {
resolve("Positive number")
} else {
reject("Not a positive number")
}
})
}
```


अब हम इसे कॉल कर सकते हैं और दोनों मामलों को संभाल सकते हैं:


```javascript
checkNumber(5)
.then(
msg => console.log("Success:", msg)
)
.catch(
err => console.log("Failure:", err)
)
```


यह प्रिंट करता है:


```
Success: Positive number
```


और यदि हम किसी भिन्न संख्या के साथ प्रयास करें:


```javascript
checkNumber(-1)
.then(
msg => console.log("Success:", msg)
)
.catch(
err => console.log("Failure:", err)
)
```


यह प्रिंट करता है:


```
Failure: Not a positive number
```


### `Promise` का उपयोग करके श्रृंखलाबद्ध संचालन



हम `Promise` का उपयोग करके अपने पिछले उदाहरण को फिर से लिख सकते हैं, और यह अधिक साफ दिखेगा।


आइए अपने दोहरीकरण फ़ंक्शन का एक नया संस्करण लिखकर शुरुआत करें, लेकिन इस बार, यह एक वादा लौटाता है:


```javascript
function doubleNumbers(numbers) {
return new Promise(resolve => {
// Wait 1 second before doing the operation
setTimeout(() => {
const doubled = numbers.map(n => n * 2)
resolve(doubled) // Return the result using resolve
}, 1000)
})
}
```


अब हम `.then()` का उपयोग करके जावास्क्रिप्ट को बता सकते हैं कि परिणाम के साथ क्या करना है:


```javascript
function doubleNumbers(numbers) {
return new Promise(resolve => {
// Wait 1 second before doing the operation
setTimeout(() => {
const doubled = numbers.map(n => n * 2)
resolve(doubled) // Return the result using resolve
}, 1000)
})
}

const input = [1, 2, 3]

doubleNumbers(input)
.then(
result => console.log("Doubled numbers:", result)
)
```


यह प्रिंट करता है:


```
Doubled numbers: [ 2, 4, 6 ]
```


अब तक, यह कॉलबैक संस्करण के समान ही काम करता है, लेकिन अब कोड को विस्तारित करना और पढ़ना आसान है।


मान लीजिए कि हम और चरण जोड़ना चाहते हैं:


1. सबसे पहले, सभी संख्याओं को दोगुना करें

2. फिर, 4 से छोटी संख्याओं को हटा दें

3. अंत में, उन सभी को एक साथ जोड़ें


हम प्रत्येक चरण के लिए एक फ़ंक्शन लिख सकते हैं, सभी वादे का उपयोग करते हुए:


```javascript
function doubleNumbers(numbers) {
return new Promise(resolve => {
// Wait 1 second before doing the operation
setTimeout(() => {
const doubled = numbers.map(n => n * 2)
resolve(doubled) // Return the result using resolve
}, 1000)
})
}

function filterBigNumbers(numbers) {
return new Promise(resolve => {
setTimeout(() => {
const filtered = numbers.filter(n => n > 3)
resolve(filtered)
}, 1000)
})
}

function sumNumbers(numbers) {
return new Promise(resolve => {
setTimeout(() => {
const total = numbers.reduce((acc, n) => acc + n, 0)
resolve(total)
}, 1000)
})
}
```


अब हम उन्हें इस तरह एक साथ जोड़ सकते हैं:


```javascript
const input = [1, 2, 3]

doubleNumbers(input)
.then(filterBigNumbers)
.then(sumNumbers)
.then(
result => console.log("Final result after all steps:", result)
)
```


यह प्रिंट करता है:


```
Final result after all steps: 10
```


आइये देखें कि यह क्या करता है:


1. `doubleNumbers` सरणी को दोगुना करता है: `[2, 4, 6]`

2. `filterBigNumbers` 3 से कम वाली किसी भी चीज़ को हटा देता है: `[4, 6]`

3. `sumNumbers` शेष संख्याओं को जोड़ता है: `4 + 6 = 10`

4. अंत में, हम परिणाम प्रिंट करते हैं।


प्रत्येक `.then()` चरण के समाप्त होने तक प्रतीक्षा करता है। इस प्रकार हम बिना नेस्टिंग के क्रियाओं की एक श्रृंखला बना सकते हैं। इससे कोड अधिक पठनीय और डीबग करने में आसान हो जाता है।


## `async`/`await` के साथ समवर्तीता

<chapterId>6e93d29f-c8bf-5fd1-a9c9-4e794ee6cbd0</chapterId>


हमने देखा कि किस प्रकार `Promise` श्रृंखलाएं हमें कॉलबैक नरक से बचने में मदद करती हैं, लेकिन जब इसमें कई चरण शामिल होते हैं, तब भी उन्हें पढ़ने के लिए थोड़ा Hard की आवश्यकता हो सकती है।


यहीं पर `async` और `await` आते हैं। वे हमें एसिंक्रोनस कोड लिखने की सुविधा देते हैं **जो सिंक्रोनस कोड जैसा दिखता है**, जिससे इसे समझना आसान हो जाता है।


### `async` क्या है?


जब आप किसी फ़ंक्शन से पहले `async` कीवर्ड लिखते हैं, तो जावास्क्रिप्ट स्वचालित रूप से फ़ंक्शन के रिटर्न मान को एक प्रॉमिस में लपेट देता है।


आइये एक बुनियादी उदाहरण देखें:


```javascript
async function greet() {
return "hello"
}
```


यदि आप इस फ़ंक्शन को कॉल करते हैं:


```javascript
const result = greet()
console.log(result)
```


आप यह देखेंगे:


```
Promise { 'hello' }
```


भले ही आपने अभी-अभी एक स्ट्रिंग लौटाई हो, JavaScript उसे आपके लिए एक Promise में बदल देता है। आप `.then()` का उपयोग करके वास्तविक मान इस प्रकार प्राप्त कर सकते हैं:


```javascript
greet().then( result => console.log(result) ) // prints "hello"
```


या आप `await` का उपयोग कर सकते हैं...


### `प्रतीक्षा` क्या है?


कीवर्ड `await` जावास्क्रिप्ट को बताता है: “इस वादे के पूरा होने तक प्रतीक्षा करें, और फिर मुझे परिणाम दें।”


लेकिन आप `await` का उपयोग केवल async फ़ंक्शन के अंदर ही कर सकते हैं।


आइए `await` का उपयोग करके उदाहरण को पुनः लिखें:


```javascript
async function greet() {
return "hello"
}

async function greetAndLog() {
const result = await greet()
console.log(result)
}

greetAndLog() // prints "hello"
```


अब हम परिणाम का उपयोग इस प्रकार कर सकते हैं जैसे कि वह एक नियमित मान हो।


आइये अब कुछ अधिक उपयोगी कार्य करें।


### `await` के साथ विलंब का अनुकरण करना


हम एक सरल `wait` फ़ंक्शन बनाएंगे जो तर्क के रूप में कुछ मिलीसेकंड लेता है और कुछ और किए बिना, केवल कुछ मिलीसेकंड के बाद हल हो जाता है:


```javascript
function wait(ms) {
return new Promise(resolve => {
setTimeout(resolve, ms)
})
}
```


आइये इसका प्रयोग करके देखें:


```javascript
async function test() {
console.log("waiting 2 seconds...")
await wait(2000)
console.log("done waiting")
}

test()
```


यह प्रिंट करता है:


```
waiting 2 seconds...
done waiting
```


आप `प्रतीक्षा` को इस प्रकार समझ सकते हैं कि “यहाँ तब तक रुकें जब तक वादा पूरा न हो जाए, फिर आगे बढ़ें।”


यह आपको ऊपर से नीचे तक कोड लिखने की अनुमति देता है जो `.then()` कॉल को श्रृंखलाबद्ध किए बिना, अतुल्यकालिक रूप से व्यवहार करता है।


### डेटा की प्रतीक्षा


आइए अपने पिछले उदाहरण का फिर से इस्तेमाल करें, जहाँ हम संख्याओं को दोगुना करते हैं, फिर फ़िल्टर करते हैं, फिर योग करते हैं। लेकिन इस बार, हम `async`/`await` का इस्तेमाल करेंगे।


हम 3 फ़ंक्शन बनाएंगे जो प्रतीक्षा का अनुकरण करेंगे, और वादे लौटाएंगे:


```javascript
function doubleNumbers(numbers) {
return new Promise(resolve => {
setTimeout(() => {
const doubled = numbers.map(n => n * 2)
resolve(doubled)
}, 1000)
})
}

function filterBigNumbers(numbers) {
return new Promise(resolve => {
setTimeout(() => {
const filtered = numbers.filter(n => n > 3)
resolve(filtered)
}, 1000)
})
}

function sumNumbers(numbers) {
return new Promise(resolve => {
setTimeout(() => {
const total = numbers.reduce((acc, n) => acc + n, 0)
resolve(total)
}, 1000)
})
}
```


अब आइए इन्हें संयोजित करने के लिए एक `async` फ़ंक्शन लिखें:


```javascript
async function process(numbers) {
const doubled = await doubleNumbers(numbers)
const filtered = await filterBigNumbers(doubled)
const total = await sumNumbers(filtered)

console.log("Final result:", total)
}

const input = [1, 2, 3]
process(input)
```


यह प्रिंट करता है:


```
Final result: 10
```


यह `.then()` को श्रृंखलाबद्ध करने या कॉलबैक को नेस्ट करने की तुलना में पढ़ने में बहुत आसान है।


यह एक नियमित चरण-दर-चरण कार्यक्रम जैसा दिखता है, लेकिन यह अभी भी अतुल्यकालिक रूप से व्यवहार करता है।


## एसिंक्रोनस इटरेटर्स

<chapterId>438b037d-9931-56d7-9052-7b4470f3c75b</chapterId>


आप पहले ही **इटरेटर्स** के बारे में जान चुके हैं और यह भी कि हम `for...of` का उपयोग ऐरे और अन्य पुनरावृत्तीय चीजों पर लूप करने के लिए कैसे कर सकते हैं।


लेकिन क्या होगा यदि जिस डेटा पर हम पुनरावृति करना चाहते हैं, उसे आने में समय लगे?


कभी-कभी हम उन चीजों पर लूप करना चाहते हैं जो **अतुल्यकालिक रूप से** आती हैं, जैसे चैट से संदेश, फ़ाइल से लाइनें, या धीमे स्रोत से संख्याएं।


ऐसा करने के लिए, जावास्क्रिप्ट हमें **async iterators** देता है।


### एसिंक्रोनस जनरेटर फ़ंक्शन


एसिंक्रोनस इटरेटर बनाने का सबसे आसान तरीका एसिंक्रोनस जेनरेटर फंक्शन का उपयोग करना है।


हम इसे इस प्रकार लिखते हैं:


```javascript
async function* generateNumbers() {
yield 1
yield 2
yield 3
}
```


यह एक नियमित जनरेटर की तरह ही दिखता है, लेकिन इसके पहले `async` लगा होता है।


अब हम मानों का उपभोग करने के लिए `for await...of` का उपयोग कर सकते हैं:


```javascript
async function run() {
for await (const n of generateNumbers()) {
console.log("Got number:", n)
}

console.log("Done!")
}

run()
```


यह प्रिंट करेगा:


```
Got number: 1
Got number: 2
Got number: 3
Done!
```


तो फिर सामान्य जनरेटर से इसमें क्या अंतर है?


अंतर यह है: अब हम जनरेटर के अंदर `await` का उपयोग कर सकते हैं।


चलिए फिर से एक विलंब सहायक बनाते हैं:


```javascript
function wait(ms) {
return new Promise(resolve => setTimeout(resolve, ms))
}
```


अब आइये **धीरे-धीरे** संख्याएँ प्राप्त करें:


```javascript
async function* generateSlowNumbers() {
await wait(1000)
yield 1

await wait(1000)
yield 2

await wait(1000)
yield 3
}
```


इसका प्रयोग करके देखें:


```javascript
async function run() {
for await (const n of generateSlowNumbers()) {
console.log("Got number:", n)
}

console.log("Done!")
}

run()
```


### एसिंक्रोनस इटरेटर्स का उपयोग क्यों करें?


Async पुनरावर्तक तब उपयोगी होते हैं जब:



- सभी मूल्य एक साथ नहीं आते।
- आप उन्हें एक-एक करके, जैसे ही वे आएं, संभालना चाहते हैं।
- आप Promises के साथ काम कर रहे हैं, और एक साफ तरीके से लूप करना चाहते हैं।


उदाहरण के लिए, यदि आप चैट सर्वर से संदेशों को एक-एक करके लोड करना चाहते हैं, या किसी बड़ी फ़ाइल को टुकड़ों में डाउनलोड करना चाहते हैं, तो एसिंक्रोनस इटरेटर्स आपको `फॉर` लूप लिखने का तरीका देता है जो विलंबित डेटा के साथ काम करता है।


### `प्रतीक.asyncIterator`


हम कस्टम क्लासों में async iterators का भी उपयोग कर सकते हैं।


यहाँ एक उदाहरण दिया गया है जो विलंब से संख्याएँ उत्पन्न करता है:


```javascript
function wait(ms) {
return new Promise(resolve => setTimeout(resolve, ms))
}

class DelayedNumbers {
constructor(numbers) {
this.numbers = numbers
}

async *[Symbol.asyncIterator]() {
for (const n of this.numbers) {
await wait(1000)
yield n
}
}
}
```


अब हम पहले की तरह `for await...of` का उपयोग कर सकते हैं:


```javascript
async function run() {
const source = new DelayedNumbers([10, 20, 30])

for await (const n of source) {
console.log("Received:", n)
}

console.log("All done!")
}

run()
```


यह आपको ऐसी वस्तुएं बनाने की अनुमति देता है जिन्हें अतुल्यकालिक रूप से पुनरावृत्त किया जा सकता है


## Assignment सिंटैक्स शुगर

<chapterId>8b1ba7d8-ecfd-5470-b86e-73cb84ccc8b7</chapterId>


"सिंटेक्स शुगर" का मतलब है किसी चीज़ को उसके मूल रूप में बदलाव किए बिना, उसे छोटा या आसान तरीके से लिखना। यह बस वही बात कहने का एक बेहतर तरीका है।


जावास्क्रिप्ट में कुछ अंतर्निहित सिंटैक्स शुगर हैं जो हमें साफ़ और संक्षिप्त घोषणाएँ लिखने में मदद करते हैं। इस अध्याय में, हम देखेंगे कि किसी शर्त के आधार पर मान कैसे निर्दिष्ट करें, गणित के साथ चरों को कैसे अपडेट करें, ऐरे या ऑब्जेक्ट से मान कैसे निकालें, और उन्हें सरल सिंटैक्स के साथ कैसे कॉपी या संयोजित करें।


### टर्नरी ऑपरेटर


जावास्क्रिप्ट में, आप **टर्नरी ऑपरेटर** का उपयोग करके किसी शर्त के आधार पर मान निर्दिष्ट कर सकते हैं, जो `if...else` लिखने का एक संक्षिप्त तरीका है।


ऐसा करने के बजाय:


```javascript
let message

if (isMorning) {
message = "Good morning"
} else {
message = "Hello"
}
```


आप लिख सकते हो:


```javascript
const message = isMorning ? "Good morning" : "Hello"
```


इसका मतलब यह है:



- यदि `isMorning` सत्य है, तो `"Good morning"` का प्रयोग करें
- अन्यथा, `"Hello"` का उपयोग करें


सामान्य रूप है:


```javascript
condition ? valueIfTrue : valueIfFalse
```


आप इसे अन्य अभिव्यक्तियों के अंदर भी उपयोग कर सकते हैं:


```javascript
console.log(isSunny ? "Go outside" : "Stay in")
```


बस यह सुनिश्चित करें कि आप इसे **सरल** निर्णयों के लिए इस्तेमाल करें। अगर तर्क जटिल है, तो `if...else` का ही इस्तेमाल करें।


### वैकल्पिक Assignment ऑपरेटर


जावास्क्रिप्ट में ऑपरेशन के साथ संयुक्त असाइनमेंट करने के लिए शॉर्टकट ऑपरेटर होते हैं।


आइये सामान्य तरीके पर नजर डालें:


```javascript
let counter = 1
counter = counter + 1
```


इसे संक्षिप्त किया जा सकता है:


```javascript
let counter = 1
counter += 1 // same as counter = counter + 1
```


यहाँ सबसे आम हैं:


| Operator | Meaning             |
| -------- | ------------------- |
| `+=`     | add and assign      |
| `-=`     | subtract and assign |
| `*=`     | multiply and assign |
| `/=`     | divide and assign   |

उदाहरण:


```javascript
let score = 10
score += 5  // now score is 15
score *= 2  // now score is 30
score -= 10 // now score is 20
```


ये तब उपयोगी होते हैं जब आप किसी चर को उसके स्वयं के मान का उपयोग करके अद्यतन करना चाहते हैं।


### डिस्ट्रक्चरिंग


**डिस्ट्रक्चरिंग** आपको एरे या ऑब्जेक्ट से मान निकालने और उन्हें आसानी से वेरिएबल में संग्रहीत करने की सुविधा देता है।


#### सरणियों


मान लीजिए आपके पास:


```javascript
const colors = ["red", "green", "blue"]
```


ऐसा करने के बजाय:


```javascript
const first = colors[0]
const second = colors[1]
```


आप कर सकते हैं:


```javascript
const [first, second] = colors
```


यह निर्दिष्ट करता है:



- `पहले` से `"लाल"` तक
- `"Green"` से `दूसरा`


आप मानों को छोड़ भी सकते हैं:


```javascript
const [,, third] = colors
console.log(third) // blue
```


#### वस्तुओं


आप ऑब्जेक्ट्स से भी मान निकाल सकते हैं:


```javascript
const user = { name: "Alice", age: 30 }

const { name, age } = user
console.log(name) // Alice
console.log(age)  // 30
```


यदि प्रॉपर्टी का नाम आपके इच्छित वेरिएबल से भिन्न है, तो आप उसका नाम बदल सकते हैं:


```javascript
const { name: username } = user
console.log(username) // Alice
```


डिस्ट्रक्चरिंग ऑब्जेक्ट्स और ऐरे के साथ काम करते समय आपके कोड को अधिक साफ़ बनाती है।


### स्प्रेड सिंटैक्स


**स्प्रेड सिंटैक्स** मानों को अनपैक या कॉपी करने के लिए `...` का उपयोग करता है।


#### सरणियों


आप सरणियों की प्रतिलिपि बना सकते हैं या उन्हें मर्ज कर सकते हैं:


```javascript
const a = [1, 2]
const b = [3, 4]

const both = [...a, ...b]
console.log(both) // [1, 2, 3, 4]
```


आप किसी सरणी का क्लोन भी बना सकते हैं:


```javascript
const original = [10, 20, 30]
const clone = [...original]
```


#### वस्तुओं


आप वस्तुओं के साथ भी ऐसा ही कर सकते हैं:


```javascript
const a = { x: 1 }
const b = { y: 2 }

const merged = { ...a, ...b }
console.log(merged) // { x: 1, y: 2 }
```


आप मानों को भी ओवरराइड कर सकते हैं:


```javascript
const user = { name: "Alice", age: 30 }
const updated = { ...user, age: 31 }

console.log(updated) // { name: "Alice", age: 31 }
```


मूल वस्तु को बदले बिना उसे अद्यतन करते समय यह बहुत उपयोगी है।

इस अध्याय में हम जावास्क्रिप्ट और नोडजेएस के बारे में थोड़ा ऐतिहासिक संदर्भ जानेंगे।


सॉफ्टवेयर में ऐतिहासिक संदर्भ बहुत महत्वपूर्ण है, क्योंकि हम अक्सर अन्य लोगों द्वारा बनाए गए उपकरणों का उपयोग करते हैं, और इसलिए हम उनके द्वारा अतीत में लिए गए निर्णयों से प्रभावित होते हैं।


उन निर्णयों के कारण को समझना, तथा यह समझना कि हम जिन उपकरणों का उपयोग करते हैं, वे किस प्रकार इस प्रकार के बने, इससे हमें इस बात को लेकर थोड़ा कम भ्रमित होने में मदद मिलेगी कि हम क्या कर रहे हैं।


### जावास्क्रिप्ट की उत्पत्ति


जावास्क्रिप्ट की शुरुआत एक सरल भाषा के रूप में हुई थी जिसे वेब पेजों को इंटरैक्टिव बनाने के लिए डिज़ाइन किया गया था।


1990 के दशक में, नेटस्केप नेविगेटर जैसे वेब ब्राउज़रों में जावास्क्रिप्ट को जोड़ा गया ताकि डेवलपर्स ऐसा कोड लिख सकें जो सीधे ब्राउज़र में चलता हो।


मूल विचार यह था कि वेबसाइट बनाने के लिए जावा को मुख्य भाषा बनाया जाए (तथाकथित "जावा एप्लेट्स" के साथ) तथा जावास्क्रिप्ट को केवल छोटे-मोटे कामों के लिए उपयोग में लाया जाए।


इसका मुख्य डिजाइन ब्रेंडन आइच द्वारा किया गया था, जो उस समय नेटस्केप में एक कर्मचारी थे, तथा इसे 2 सप्ताह से भी कम समय में पूरा किया गया था।


लेकिन अधिकांश लोग जावा की अपेक्षा जावास्क्रिप्ट का उपयोग करना पसंद करते थे, और उस समय जावा एप्लेट्स में कुछ सुरक्षा संबंधी समस्याएं भी थीं, इसलिए अंततः जावा को विकल्प के रूप में छोड़ दिया गया और जावास्क्रिप्ट वेब विकास के लिए वास्तविक मानक बन गया।


### V8 इंजन


जावास्क्रिप्ट एक व्याख्याकृत भाषा है, जो सी जैसी संकलित भाषाओं के विपरीत है।


संकलित भाषा में लिखा गया कोड बाइनरी में बदल जाता है, और बाइनरी सीधे कंप्यूटर के सीपीयू में चला जाता है।


![](assets/en/6.webp)


दूसरी ओर, व्याख्या की गई भाषाएं अधिक उपयोगकर्ता-अनुकूल होती हैं, तथा मशीनों के काम करने के तरीके ("निम्न स्तर") की अपेक्षा मनुष्यों की सोच ("उच्च स्तर") के अधिक निकट होती हैं; इसलिए उनके कोड को चलाने के लिए आमतौर पर एक वर्चुअल मशीन बनी होती है।


वर्चुअल मशीन एक विशेष प्रोग्राम है जो आपके द्वारा लिखे गए कोड और सीपीयू के बीच स्थित होता है, और आपके कोड को निष्पादित करता है (क्योंकि सीपीयू इसे समझ नहीं सकता)।


इससे आप अंतर्निहित मशीन के बारे में बहुत अधिक जानकारी के बिना भी प्रोग्रामिंग कर सकते हैं, लेकिन इसके प्रदर्शन के संदर्भ में भी लागत आती है, क्योंकि कंप्यूटर केवल आपका प्रोग्राम नहीं चला रहा है; यह एक प्रोग्राम (वर्चुअल मशीन) चला रहा है जो आपके प्रोग्राम को चलाता है।


जैसे-जैसे वेब एप्लिकेशन अधिक जटिल होते गए, जावास्क्रिप्ट के प्रदर्शन में सुधार की माँग बढ़ती गई। V8 इंजन वह इंटरप्रेटर है जो गूगल क्रोम में जावास्क्रिप्ट को संचालित करता है। इसे गूगल द्वारा विकसित किया गया था और 2008 में जारी किया गया था।


जबकि पुराने जावास्क्रिप्ट इंजन ज्यादातर पारंपरिक वर्चुअल मशीन थे, V8 इंजन एक JIT (जस्ट-इन-टाइम) कंपाइलर है।


जावास्क्रिप्ट कोड V8 इंजन में डाला जाता है, और इंजन इसके कुछ हिस्सों को तुरंत मूल बाइनरी के रूप में संकलित करने का प्रयास करता है। इससे आपको एक उच्च-स्तरीय भाषा का अनुभव मिलता है, और इसका प्रदर्शन निम्न-स्तरीय भाषाओं के कुछ करीब होता है। इसने जावास्क्रिप्ट को दुनिया की सबसे तेज़ उच्च-स्तरीय भाषा बना दिया है, जो कि "दोनों दुनियाओं में सर्वश्रेष्ठ" होने जैसा है।


### NodeJS रनटाइम


उपयोग में आसान और काफी तेज निष्पादन के बावजूद, V8 के रिलीज के बाद जावास्क्रिप्ट में एक बड़ी सीमा बनी रही: यह केवल ब्राउज़र के अंदर ही चल सकता था।


वह एक समस्या क्यों है?


चूंकि ब्राउज़र इंटरनेट पर लाखों विभिन्न स्रोतों से प्राप्त कोड को निष्पादित करते हैं, इसलिए वे आसानी से मैलवेयर का शिकार हो सकते हैं, इसलिए उन्हें शेष ऑपरेटिंग सिस्टम से "सैंडबॉक्स" कर दिया जाता है।


![](assets/en/7.webp)


जावास्क्रिप्ट आपके कंप्यूटर पर फ़ाइल सिस्टम और अन्य स्थानीय संसाधनों तक नहीं पहुंच सकता था (कम से कम अन्य भाषाओं की तरह आसानी से नहीं), इसलिए यह एक महत्वपूर्ण सीमा थी कि आप इसके साथ किस प्रकार के अनुप्रयोग बना सकते थे।


2009 में, रयान डाहल ने NodeJS प्रकाशित किया, जो एक रनटाइम है जो आपको ब्राउज़र के बाहर, सीधे आपके कंप्यूटर के मूल ऑपरेटिंग सिस्टम पर V8 इंजन का उपयोग करने की अनुमति देता है। इसमें कई ऐसी सुविधाएँ भी शामिल हैं जो सर्वर-साइड और कमांड-लाइन प्रोग्राम लिखने के लिए उपयोगी हैं। उदाहरण के लिए, आप NodeJS का उपयोग वेब सर्वर बनाने, फ़ाइलें पढ़ने और लिखने, या कार्यों को स्वचालित करने वाले टूल बनाने के लिए कर सकते हैं।


![](assets/en/8.webp)


इस पाठ्यक्रम में अब तक, हमने ब्राउज़र और NodeJS, दोनों में मौजूद JavaScript सुविधाओं का अध्ययन किया है। इन सुविधाओं ने हमें डेटा को परिभाषित करने और उसे अमूर्त तरीकों से नियंत्रित करने में सक्षम बनाया है। अगले कुछ पाठों में, हम NodeJS की विशिष्ट सुविधाओं का अध्ययन करेंगे और हमें ऑपरेटिंग सिस्टम के साथ इंटरैक्ट करने में सक्षम बनाएंगे।


## कमांड लाइन तर्क

<chapterId>960d20f3-c424-5d51-a041-ef17d2e94b6d</chapterId>


NodeJS हमें अन्य चीजों के अलावा CLIs (कमांड लाइन इंटरफेस) बनाने की अनुमति देता है।


इसके लिए हमें कमांड लाइन तर्क प्राप्त करने का एक तरीका चाहिए, जो नोड में अंतर्निहित `प्रक्रिया` ऑब्जेक्ट का उपयोग करके किया जाता है।


### `प्रक्रिया`


NodeJS `process` नामक एक विशेष ऑब्जेक्ट प्रदान करता है जो वर्तमान चल रहे प्रोग्राम का प्रतिनिधित्व करता है।


आप इसका उपयोग पर्यावरण, वर्तमान कार्यशील निर्देशिका का निरीक्षण करने और आवश्यकता पड़ने पर प्रोग्राम से बाहर निकलने के लिए भी कर सकते हैं।


उदाहरण के लिए:


```javascript
console.log(process.platform)
```


यह ऑपरेटिंग सिस्टम प्लेटफॉर्म को प्रिंट करता है, जैसे `win32`, `linux`, या `darwin` (Mac).


### `प्रक्रिया.argv`


जब आप टर्मिनल से NodeJS प्रोग्राम चलाते हैं, तो आप स्क्रिप्ट नाम के बाद अतिरिक्त शब्द (आर्गुमेंट) पास कर सकते हैं। ये `process.argv` में संग्रहीत होते हैं।


उदाहरण के लिए, यदि आप यह आदेश चलाते हैं:


```
node my_script.js alpha beta
```


आप तर्कों को इस प्रकार मुद्रित कर सकते हैं:


```javascript
console.log(process.argv)
```


आउटपुट कुछ इस तरह दिख सकता है:


```
[ '/path/to/node', '/path/to/my_script.js', 'alpha', 'beta' ]
```


पहले दो आइटम हमेशा नोड पथ और आपकी स्क्रिप्ट पथ होते हैं। स्क्रिप्ट में आपके द्वारा दिए गए कोई भी अतिरिक्त शब्द उसके बाद आते हैं।


`process.argv` सरणी को `.slice()` विधि का उपयोग करके किसी भी अन्य सरणी की तरह काटा जा सकता है, इसलिए केवल पारित किए गए तर्कों को प्राप्त करने के लिए आप यह कर सकते हैं


```javascript
const args = process.argv.slice(2)

console.log(args)
```


उपयोगकर्ता द्वारा पास किए जा रहे तर्कों तक पहुंच होना कमांड-लाइन अनुप्रयोगों को विकसित करने के लिए मौलिक है।


## मॉड्यूल

<chapterId>4e1651a5-65fd-50bc-b22a-40313d5659ca</chapterId>


NodeJS जैसे जावास्क्रिप्ट रनटाइम आमतौर पर प्रत्येक जावास्क्रिप्ट फ़ाइल को एक अलग मॉड्यूल के रूप में मानते हैं।


मॉड्यूल को एक बॉक्स की तरह समझें। इस बॉक्स का अपना निजी स्पेस होता है, इसलिए इसमें आपके द्वारा घोषित वेरिएबल और फ़ंक्शन दूसरे बॉक्स के कोड में हस्तक्षेप नहीं करते। मूलतः, प्रत्येक मॉड्यूल का अपना स्कोप होता है।


एक मॉड्यूल अपनी कुछ सामग्री निर्यात कर सकता है, जिससे वह अन्य मॉड्यूल के लिए उपलब्ध हो जाती है, और वह उस सामग्री को आयात भी कर सकता है जिसे अन्य मॉड्यूल ने निर्यात किया है। जावास्क्रिप्ट आपको विभिन्न फ़ाइलों को जोड़ने के लिए मॉड्यूल के बीच सामग्री निर्यात और आयात करने की अनुमति देता है।


एक जावास्क्रिप्ट प्रोग्राम अक्सर कई मॉड्यूलों से बना होता है, जो एक दूसरे से जुड़े होते हैं।


मॉड्यूल का उपयोग क्यों करें? अपने कोड को मॉड्यूल में विभाजित करके, आप अपने प्रोग्राम को छोटे, स्पष्ट और पुन: प्रयोज्य भागों में व्यवस्थित कर सकते हैं। प्रत्येक मॉड्यूल केवल एक प्रकार के कार्य पर ध्यान केंद्रित कर सकता है, जैसे गणितीय गणनाएँ करना, फ़ाइलों के साथ काम करना, या टेक्स्ट को फ़ॉर्मेट करना।


### कॉमनजेएस मॉड्यूल


NodeJS में, मॉड्यूल प्रबंधित करने के लिए सबसे आम प्रणाली को **CommonJS** कहा जाता है।


इस प्रणाली में, आप `module.exports` का उपयोग करके किसी मॉड्यूल से कोड साझा (निर्यात) कर सकते हैं और `require()` का उपयोग करके इसे किसी अन्य फ़ाइल में लोड (आयात) कर सकते हैं।


किसी मॉड्यूल के बाहर कुछ उपलब्ध कराने के लिए, आप उसे `module.exports` को असाइन करते हैं:


```javascript
const greeting = "Hello!"

module.exports = greeting
```


यहां, स्ट्रिंग `"Hello!"` वह है जिसे यह मॉड्यूल निर्यात करता है।


किसी अन्य फ़ाइल से निर्यातित कोड का उपयोग करने के लिए, आप उस फ़ाइल के पथ के साथ `require()` फ़ंक्शन का उपयोग करते हैं:


```javascript
const greeting = require("./greeting.js")

console.log(greeting)
```


यह प्रिंट करता है:


```
Hello!
```


आप एक अनाम ऑब्जेक्ट में एक साथ बंडल करके कई चीजों को निर्यात कर सकते हैं, जैसे


```javascript
const greeting1 = "hello"

const greeting2 = "hi"

module.exports = {
greeting1, greeting2
}
```


कॉमनजेएस वह मॉड्यूल सिस्टम था जिसे शुरुआत में नोडजेएस द्वारा अपनाया गया था। बाद में ईएस मॉड्यूल भी जोड़े गए।


### ईएस मॉड्यूल


NodeJS एक अन्य प्रकार के मॉड्यूल का भी समर्थन करता है जिसे **ES मॉड्यूल** कहा जाता है, जो वेब डेवलपमेंट में लोकप्रिय हैं। ये `export` और `import` कीवर्ड का उपयोग करते हैं।


ES मॉड्यूल ब्राउज़र के लिए विकसित किए गए थे और बाद में नोड में जोड़े गए। अगर आप इनका इस्तेमाल करना चाहते हैं, तो आपको फ़ाइल एक्सटेंशन के तौर पर `.js` की बजाय `.mjs` का इस्तेमाल करना पड़ सकता है, यह इस बात पर निर्भर करता है कि आप नोड का कौन सा संस्करण इस्तेमाल कर रहे हैं।


`greeting.mjs` नामक एक फ़ाइल में हम लिखते हैं


```javascript
export const greeting = "Hello!"
```

जैसा कि आप देख सकते हैं, हम स्थिरांक को सीधे वहीं निर्यात कर रहे हैं जहाँ इसे परिभाषित किया गया है


`index.mjs` नामक एक अन्य फ़ाइल में, हम इसे आयात करते हैं:


```javascript
import { greeting } from "./greeting.mjs"

console.log(greeting)
```


आप फ़ाइल के विभिन्न भागों में विभिन्न घोषणाएँ निर्यात कर सकते हैं, जैसे


```javascript
export const num = 10

export function double (x) {
return x*2
}
```


### NodeJS मानक पुस्तकालय


NodeJS में कई बिल्ट-इन मॉड्यूल भी शामिल हैं। ये NodeJS द्वारा प्रदान किए गए पहले से तैयार मॉड्यूल हैं जो फ़ाइलों को पढ़ने, ऑपरेटिंग सिस्टम के साथ काम करने या नेटवर्क से कनेक्ट करने जैसे सामान्य कार्यों में मदद करते हैं।


उदाहरण के लिए, `os` मॉड्यूल आपको आपके ऑपरेटिंग सिस्टम के बारे में जानकारी देता है:


```javascript
const os = require("os")

console.log(os.platform())
```


आपको इन बिल्ट-इन मॉड्यूल्स को इंस्टॉल करने की ज़रूरत नहीं है, ये NodeJS के साथ आते हैं। ये रनटाइम की "मानक लाइब्रेरी" बनाते हैं और ज़्यादातर Node एप्लिकेशन फ़ाइलों को पढ़ने या इंटरनेट पर संचार करने जैसे कामों के लिए इनका इस्तेमाल करते हैं।


अगले अध्याय में आपको उनके उपयोग के कुछ उपयोगी उदाहरण दिखाए जाएंगे।


## `fs` मॉड्यूल

<chapterId>911e953a-35ae-5ee7-bd74-372501c32e81</chapterId>


`fs` मॉड्यूल (**फ़ाइल सिस्टम** का संक्षिप्त रूप) NodeJS मानक लाइब्रेरी का हिस्सा है। यह आपको अपने कंप्यूटर पर फ़ाइलों और निर्देशिकाओं के साथ काम करने की सुविधा देता है: आप फ़ाइलें पढ़ सकते हैं, लिख सकते हैं, उन्हें हटा सकते हैं, उनका नाम बदल सकते हैं, और भी बहुत कुछ कर सकते हैं।


इसका उपयोग करने के लिए, आपको सबसे पहले इसे अपनी फ़ाइल के शीर्ष पर आयात करना होगा:


```javascript
const fs = require("fs")
```


### सिंक एपीआई


`fs` का उपयोग करने का सबसे सरल तरीका इसकी **sync** विधियों के साथ है।


ये विधियां प्रोग्राम को तब तक ब्लॉक कर देती हैं जब तक कि वे समाप्त नहीं हो जातीं (इसलिए कोड की अगली पंक्ति तब तक नहीं चलती जब तक कि ऑपरेशन पूरा नहीं हो जाता)।


यहां एक फ़ाइल को समकालिक रूप से पढ़ने का एक उदाहरण दिया गया है:


```javascript
const fs = require("fs")

const data = fs.readFileSync("example.txt", "utf8")

console.log(data)
```


यदि आपकी स्क्रिप्ट के समान निर्देशिका में `example.txt` नामक कोई फ़ाइल है, तो यह उसकी सामग्री प्रिंट करेगा।


आप किसी फ़ाइल में समकालिक रूप से भी लिख सकते हैं:


```javascript
const fs = require("fs")

fs.writeFileSync("output.txt", "Hello from Node!")

console.log("File written!")
```


यह पाठ के साथ `output.txt` नामक फ़ाइल बनाता है (या अधिलेखित करता है)।


यहां कुछ सामान्य ऑपरेशन दिए गए हैं जो आप इस API के साथ कर सकते हैं:


```javascript
const fs = require("fs")

// List files and folders
const items = fs.readdirSync(".")
console.log("Items in current directory:", items)

// Create folder
fs.mkdirSync("my_folder")
console.log("Folder created")

// Delete folder
fs.rmdirSync("my_folder")
console.log("Folder deleted")

// Create & write file
fs.writeFileSync("my_file.txt", "Hello world")
console.log("File created & written")

// Read file
const content = fs.readFileSync("my_file.txt", "utf8")
console.log("File content:", content)

// Delete file
fs.unlinkSync("my_file.txt")
console.log("File deleted")
```


सिंक एपीआई सरल है और छोटी स्क्रिप्ट के लिए अच्छा है, लेकिन चूंकि यह प्रोग्राम को पूरा होने तक ब्लॉक कर देता है, इसलिए यदि आप बड़ी फ़ाइलों या सर्वर के साथ काम कर रहे हैं तो यह चीजों को धीमा कर सकता है।


### कॉलबैक एसिंक्रोनस API


**कॉलबैक एपीआई** नॉन-ब्लॉकिंग है: यह फ़ाइल ऑपरेशन होने के दौरान NodeJS को अन्य कार्य करने देता है।


परिणाम को सीधे लौटाने के बजाय, यह एक फ़ंक्शन (एक कॉलबैक) लेता है, जिसे ऑपरेशन पूरा होने पर कॉल किया जाता है।


```javascript
const fs = require("fs")

fs.readFile("example.txt", "utf8", (err, data) => {
if (err) {
console.error("Error reading file:", err)
} else {
console.log(data)
}
})
```


यहाँ क्या होता है:



- `fs.readFile` `example.txt` को पढ़ना शुरू करता है।
- NodeJS प्रतीक्षा नहीं करता, यह आपके द्वारा लिखे गए अन्य कोड को निष्पादित करने के लिए आगे बढ़ता है।
- जब फ़ाइल पढ़ना समाप्त हो जाता है, तो कॉलबैक चलता है:



  - यदि कोई त्रुटि थी, तो `err` में त्रुटि शामिल है।
  - अन्यथा, `data` में सामग्री शामिल होती है।


आप किसी फ़ाइल में इस प्रकार लिखते हैं:


```javascript
const fs = require("fs")

fs.writeFile("output.txt", "Hello async!", (err) => {
if (err) {
console.error("Error writing file:", err)
} else {
console.log("File written!")
}
})
```


वही विचार: फ़ाइल लिखते समय प्रोग्राम बंद नहीं होता।


इस API के साथ आप जो कुछ कर सकते हैं उसके कुछ उदाहरण:


```javascript
const fs = require("fs")

// List files and folders
fs.readdir(".", (err, items) => {
if (err) return console.error(err)
console.log("Items in current directory:", items)
})

// Create folder
fs.mkdir("my_folder", (err) => {
if (err) return console.error(err)
console.log("Folder created")
})

// Delete folder
fs.rmdir("my_folder", (err) => {
if (err) return console.error(err)
console.log("Folder deleted")
})

// Create & write file
fs.writeFile("my_file.txt", "Hello world", (err) => {
if (err) return console.error(err)
console.log("File created & written")
})

// Read file
fs.readFile("my_file.txt", "utf8", (err, content) => {
if (err) return console.error(err)
console.log("File content:", content)
})

// Delete file
fs.unlink("my_file.txt", (err) => {
if (err) return console.error(err)
console.log("File deleted")
})
```


कॉलबैक API सर्वर और बड़े कार्यों के लिए बेहतर है क्योंकि यह प्रोग्राम को ब्लॉक नहीं करता, लेकिन अगर आप कई ऑपरेशनों को एक साथ जोड़ते हैं, तो नेस्टेड कॉलबैक गड़बड़ हो सकते हैं। इसीलिए एक प्रॉमिस-आधारित एसिंक्रोनस API जोड़ा गया है।


### प्रॉमिस एसिंक्रोनस API


प्रॉमिस-आधारित API आधुनिक है और `.then()` और `async/await` के साथ बेहतरीन तरीके से काम करता है। यह `fs.promises` के रूप में उपलब्ध है।


आपको `promises` प्रॉपर्टी को आयात करने की आवश्यकता है:


```javascript
const fs = require("fs").promises
```


`.then()` का उपयोग करना:


```javascript
const fs = require("fs").promises

fs.readFile("example.txt", "utf8")
.then(data => {
console.log(data)
})
.catch(err => {
console.error("Error reading file:", err)
})
```


या इससे भी बेहतर, `async/await` का उपयोग करें:


```javascript
const fs = require("fs").promises

async function readFile(fileName) {
try {
const data = await fs.readFile(fileName, "utf8")
console.log(data)
} catch (err) {
console.error("Error reading file:", err)
}
}

readFile("example.txt")
```


फ़ाइल में लिखना:


```javascript
const fs = require("fs").promises

async function writeFile(fileName, content) {
try {
await fs.writeFile(fileName, content)
console.log("File written!")
} catch (err) {
console.error("Error writing file:", err)
}
}

writeFile("output.txt", "Hello from promises!")
```


API के लिए उदाहरणों की सामान्य सूची:


```javascript
const fs = require("fs").promises

// Use an async function to await operations
async function main() {
// List files and folders
const items = await fs.readdir(".")
console.log("Items in current directory:", items)

// Create folder
await fs.mkdir("my_folder")
console.log("Folder created")

// Delete folder
await fs.rmdir("my_folder")
console.log("Folder deleted")

// Create & write file
await fs.writeFile("my_file.txt", "Hello world")
console.log("File created & written")

// Read file
const content = await fs.readFile("my_file.txt", "utf8")
console.log("File content:", content)

// Delete file
await fs.unlink("my_file.txt")
console.log("File deleted")
}

main().catch(err => console.error(err))
```



## NPM

<chapterId>a91d9a75-55cc-51a3-a48f-0c0be6fe6e72</chapterId>


जब आप कोड लिखते हैं, तो आपको अक्सर अन्य लोगों द्वारा लिखे गए कोड का उपयोग करने की आवश्यकता होगी; उदाहरण के लिए, तारीखों, रंगों, सर्वरों या लगभग किसी भी अन्य चीज़ के साथ काम करने में आपकी मदद करने के लिए लाइब्रेरीज़।


फ़ाइलों को मैन्युअल रूप से डाउनलोड और कॉपी करने के बजाय, आप **पैकेज मैनेजर** का उपयोग कर सकते हैं।


पैकेज मैनेजर एक ऐसा उपकरण है जो:



- डाउनलोड पैकेज
- आपके प्रोजेक्ट को किन पैकेजों की आवश्यकता है, इसका ट्रैक रखता है
- यह सुनिश्चित करता है कि आपकी टीम के सभी सदस्यों के पास पैकेजों के समान संस्करण हों


### एनपीएम क्या है?


NodeJS की दुनिया में, सबसे लोकप्रिय पैकेज मैनेजर **NPM** है, जिसका अर्थ है *नोड पैकेज मैनेजर*।


जब आप NodeJS इंस्टॉल करते हैं तो आपको NPM स्वचालित रूप से प्राप्त हो जाता है।


आप अपने टर्मिनल में इसे चलाकर जांच सकते हैं कि आपके पास NPM है या नहीं:


```
npm -v
```


यह आपके पास मौजूद NPM का संस्करण प्रिंट करता है, जैसे:


```
10.2.1
```


### पैकेज बनाना


NodeJS में, एक पैकेज केवल एक निर्देशिका है जिसमें `package.json` फ़ाइल होती है।


आइये, एक-एक कदम आगे बढ़ें।


1. अपने प्रोजेक्ट के लिए एक नया फ़ोल्डर बनाएँ:


```
mkdir my_project
cd my_project
```


2. यह आदेश चलाएँ:


```
npm init
```


इससे एक इंटरैक्टिव सेटअप शुरू होता है, जिसमें आपसे आपके प्रोजेक्ट का नाम, संस्करण, विवरण आदि जैसे प्रश्न पूछे जाते हैं।


यदि आप सभी प्रश्नों का उत्तर नहीं देना चाहते और केवल डिफ़ॉल्ट को स्वीकार करना चाहते हैं, तो आप इसका उपयोग कर सकते हैं:


```
npm init -y
```


इसे चलाने के बाद, आपको अपने फ़ोल्डर में एक नई फ़ाइल दिखाई देगी जिसका नाम होगा:


```
package.json
```


### `पैकेज.json`


`package.json` फ़ाइल केवल एक JSON फ़ाइल है जो आपके प्रोजेक्ट के बारे में मेटाडेटा संग्रहीत करती है।


यहाँ एक उदाहरण है:


```json
{
"name": "my_project",
"version": "1.0.0",
"description": "",
"main": "index.js",
"scripts": {
"test": "echo \"Error: no test specified\" && exit 1"
},
"keywords": [],
"author": "",
"license": "ISC",
"type": "commonjs"
}

```


कुछ महत्वपूर्ण क्षेत्र:



- `name`: आपके पैकेज का नाम
- `संस्करण`: वर्तमान संस्करण
- `main`: प्रवेश बिंदु फ़ाइल (जैसे `index.js`)
- `स्क्रिप्ट`: कमांड जिन्हें आप चला सकते हैं (जैसे `npm start`)
- `निर्भरताएँ`: उन सभी पैकेजों को सूचीबद्ध करता है जिन पर आपका प्रोजेक्ट निर्भर करता है


### पैकेज स्थापित करना


मान लीजिए कि आप अपने टर्मिनल आउटपुट में रंग जोड़ने के लिए `picocolors` नामक एक निश्चित पैकेज का उपयोग करना चाहते हैं।


आप इसे चलाकर स्थापित कर सकते हैं:


```
npm install picocolors
```


अब आप इसे अपने प्रोजेक्ट में इस्तेमाल कर सकते हैं। `index.js` फ़ाइल बनाएँ


```javascript
const pico = require('picocolors')

console.log(
pico.green("This text is green!")
)
```


और इसे चलाकर देखें। टर्मिनल पर टेक्स्ट का रंगीन संस्करण प्रिंट होना चाहिए।


एनपीएम ने क्या किया?



- इसने पैकेज को डाउनलोड किया और इसे `node_modules/` नामक सबफ़ोल्डर में संग्रहीत किया
- इसने आपके `package.json` में `dependencies` के अंतर्गत एक प्रविष्टि जोड़ दी है
- इसने `package-lock.json` फ़ाइल को अपडेट किया


`package-lock.json` क्या है?


### `पैकेज-लॉक.json`


यह फ़ाइल NPM द्वारा स्वचालित रूप से बनाई जाती है।


आप सोच रहे होंगे कि यदि हमारे पास पहले से ही `package.json` है, तो हमें दूसरी फ़ाइल की क्या आवश्यकता है?

इसका कारण यह है:



- `package.json` केवल यह बताता है कि आपके प्रोजेक्ट को पैकेज के किस संस्करण **रेंज** की आवश्यकता है।

उदाहरण:


```json
"dependencies": {
"picocolors": "^1.1.0"
}
```


`^1.1.0` का अर्थ है “कोई भी संस्करण जो 1.1.x के साथ संगत है”, इसलिए यह लचीला है।



- `package-lock.json` प्रत्येक पैकेज और उनकी उप-निर्भरताओं के सटीक संस्करणों को स्थिर कर देता है, ताकि आपके प्रोजेक्ट को स्थापित करने वाले प्रत्येक व्यक्ति को बिल्कुल समान कार्यशील सेटअप प्राप्त हो।


यह महत्वपूर्ण क्यों है?


यदि आप किसी टीम में काम करते हैं, या आप अपनी परियोजना को किसी सर्वर पर तैनात करते हैं, या आप भविष्य में उस पर वापस आते हैं, तो आप यह सुनिश्चित करना चाहेंगे कि यह अभी भी उसी तरह काम करे।

यदि पैकेज अपडेट कर दिए गए हैं और आप लॉक फ़ाइल के बिना पुनः इंस्टॉल करते हैं, तो आपको थोड़ा अलग संस्करण मिल सकता है जो अलग तरह से व्यवहार करता है।


अपने प्रोजेक्ट में `package-lock.json` को रखकर, NPM हमेशा वहां सूचीबद्ध सटीक संस्करणों को स्थापित करेगा, जिससे यह सुनिश्चित होगा कि सभी के पास एक ही वातावरण है।


`package-lock.json` सब कुछ एक बहुत ही विशिष्ट संस्करण पर लॉक कर देता है, जिससे परियोजना को अन्य मशीनों पर अधिक पुनरुत्पादित किया जा सके।


लेकिन यदि आपके पैकेज को कई लोगों द्वारा उपयोग किए जाने की आवश्यकता है, तो आप इसे प्रतिबद्ध न करने का विकल्प चुन सकते हैं, ताकि NPM को केवल `package.json` फ़ाइल मिले और उसे उस फ़ाइल में अनुमत नवीनतम संस्करणों को स्वचालित रूप से स्थापित करने की अनुमति मिल जाए।


लेकिन ये ऐसी बातें हैं जिनकी आपको बाद में चिंता करनी चाहिए, जब आप अपना कोड प्रकाशित करना शुरू कर दें। अभी के लिए, हमने NPM की मूल बातें सिर्फ़ इसलिए बताई हैं ताकि आप अपने प्रोजेक्ट में दूसरे डेवलपर्स द्वारा प्रकाशित लाइब्रेरीज़ को ढूँढ़ सकें और उनका इस्तेमाल कर सकें।



## NodeJS में नेटवर्किंग

<chapterId>f2cabd8b-754b-5c97-8d6a-8412a9a184c7</chapterId>


नोडजेएस का उपयोग अक्सर बैकएंड के लिए एक भाषा के रूप में किया जाता है: आप अपनी स्क्रिप्ट को सर्वर में बदल सकते हैं, और इसका उपयोग अन्य सर्वरों से अनुरोध करने के लिए भी कर सकते हैं।


इस अध्याय में हम कुछ बुनियादी नेटवर्किंग सुविधाओं से परिचित कराएंगे जो आपको ऐसा करने में सक्षम बनाएंगी।


### `लाएँ()`


यदि आप चाहते हैं कि आपका प्रोग्राम किसी वेबसाइट या API से डेटा डाउनलोड करे, तो आपको एक **HTTP अनुरोध** करना होगा।


NodeJS के आधुनिक संस्करणों में, आप अंतर्निहित `fetch()` फ़ंक्शन का उपयोग कर सकते हैं।


यहां API पर GET अनुरोध करने का एक उदाहरण दिया गया है:


```javascript
const response = await fetch("https://jsonplaceholder.typicode.com/posts/1")
const data = await response.text()

console.log(data)
```


जब आप इसे चलाएंगे, तो आपको कुछ ऐसा दिखाई देगा:


```
{
"userId": 1,
"id": 1,
"title": "...",
"body": "..."
}
```


यहाँ बताया गया है कि क्या होता है:


1. `fetch()` एक URL लेता है और एक अनुरोध करता है।

2. यह एक वादा लौटाता है जो एक `Response` ऑब्जेक्ट में परिवर्तित हो जाता है।

3. `response.text()` प्रतिक्रिया बॉडी को एक स्ट्रिंग के रूप में पढ़ता है।


लेकिन आपको जो स्ट्रिंग वापस मिलती है, वह असल में JSON है। वह क्या है?


### जेएसओएन


वेब एपीआई के साथ काम करते समय, डेटा अक्सर JSON के रूप में भेजा और प्राप्त किया जाता है, जिसका अर्थ है जावास्क्रिप्ट ऑब्जेक्ट नोटेशन।


JSON बस एक टेक्स्ट फ़ॉर्मैट है जो काफ़ी हद तक JavaScript ऑब्जेक्ट जैसा दिखता है। उदाहरण के लिए:


```json
{
"name": "Alice",
"age": 30,
"likes": ["apples", "bananas"]
}
```


`JSON` ऑब्जेक्ट जावास्क्रिप्ट में एक अंतर्निहित उपयोगिता है जिसका उपयोग इस डेटा प्रारूप के साथ काम करने के लिए किया जा सकता है।


आप `JSON.stringify()` का उपयोग करके जावास्क्रिप्ट ऑब्जेक्ट को JSON स्ट्रिंग में परिवर्तित कर सकते हैं:


```javascript
const user = { name: "Alice", age: 30 }

const jsonString = JSON.stringify(user)

console.log(jsonString)
```


यह प्रिंट करता है:


```
{"name":"Alice","age":30}
```


आप `JSON.parse()` का उपयोग करके JSON टेक्स्ट को वापस जावास्क्रिप्ट ऑब्जेक्ट में भी परिवर्तित कर सकते हैं:


```javascript
const jsonText = '{"name":"Bob","age":25}'

const obj = JSON.parse(jsonText)

console.log(obj)
```


यह प्रिंट करता है:


```
{ name: 'Bob', age: 25 }
```


सावधान रहें: यदि स्ट्रिंग वैध JSON नहीं है तो `JSON.parse()` त्रुटि देगा।


```javascript
JSON.parse("not json") // ❌ Error!
```


इसलिए सुनिश्चित करें कि स्ट्रिंग उचित रूप से स्वरूपित है।


### `http` सर्वर


NodeJS आपको कुछ भी इंस्टॉल किए बिना वेब सर्वर बनाने की अनुमति देता है।


आप ग्राहकों से अनुरोधों को संभालने और प्रतिक्रियाएँ वापस भेजने के लिए अंतर्निहित `http` मॉड्यूल का उपयोग कर सकते हैं।


यहाँ एक बहुत ही बुनियादी उदाहरण है:


```javascript
const http = require("http")

const server = http.createServer((req, res) => {
res.statusCode = 200
res.end("Hello from NodeJS server!")
})

server.listen(3000, () => {
console.log("Server running at http://localhost:3000/")
})
```


जब आप इस स्क्रिप्ट को चलाएंगे और अपने ब्राउज़र में `http://localhost:3000` खोलेंगे, तो आप देखेंगे:


```
Hello from NodeJS server!
```


कोड में ऐसा हो रहा है:


1. आपने नोड मानक लाइब्रेरी से `http` सर्वर आयात किया है।

2. `http.createServer()` एक सर्वर बनाता है। आपने `http.createServer()` को एक कॉलबैक `(req, res) => {...}` पास किया है जो हर बार किसी के कनेक्ट होने पर निष्पादित होता है।

3. आपने प्रतिक्रिया को 200 का स्टेटस कोड दिया है (जो एक सफल ऑपरेशन का संकेत देता है)। आप http स्टेटस कोड के बारे में [यहाँ](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Status) पढ़ सकते हैं।

3. `res.end()` प्रतिक्रिया भेजता है और कनेक्शन समाप्त करता है।

4. `server.listen(3000)` पोर्ट 3000 पर सर्वर प्रारंभ करता है।


आप विभिन्न पथों या अनुरोध प्रकारों को संभालने के लिए अनुरोध में `req.url` और `req.method` की भी जांच कर सकते हैं।


रूटिंग के साथ उदाहरण:


```javascript
const server = http.createServer((req, res) => {
if (req.url === "/") { // handle requests for the root of the website
res.statusCode = 200
res.end("Home page")
} else if (req.url === "/about") { // handle requests for the about page
res.statusCode = 200
res.end("About page")
} else {
res.statusCode = 404 // we send a 404 status code to signal that the requested page is missing
res.end("Not Found")
}
})
```


ये बहुत ही बुनियादी उदाहरण हैं। ज़्यादा उन्नत सर्वर बनाने के लिए, ज़्यादातर डेवलपर्स शायद [express](https://www.npmjs.com/package/express) जैसी पहले से तैयार बैकएंड लाइब्रेरी डाउनलोड करेंगे।


## डेटा प्रसंस्करण: बफ़र्स, ईवेंट, स्ट्रीम

<chapterId>8c9623f0-a604-51a4-8fe4-871c849d3e3b</chapterId>


इस अध्याय में हम मुख्यतः तीन प्रकार की वस्तुओं का परिचय देंगे:



- `बफर`, जो बाइनरी डेटा के छोटे टुकड़ों का प्रतिनिधित्व करता है
- `EventEmitter`, जिसका उपयोग "ईवेंट" नामक सिग्नल उत्सर्जित करके अतुल्यकालिक प्रक्रिया द्वारा कुछ चरणों को ट्रैक करने के लिए किया जा सकता है
- `स्ट्रीम`, जो हमें एक समय में एक `बफर` द्वारा डेटा के बड़े हिस्से को संसाधित करने की अनुमति देता है, और जो घटनाओं को उत्सर्जित करके प्रक्रिया को ट्रैक करता है


ये पेशेवर NodeJS कोड में बेहद आम हैं, इसलिए भले ही आप उन्हें अपने पहले प्रोजेक्ट में उपयोग न करें, लेकिन जब आपको उनके साथ बातचीत करने की आवश्यकता होगी, तो बुनियादी समझ प्राप्त करना अच्छा होगा।


### बफ़र


NodeJS में, बफर एक प्रकार का ऑब्जेक्ट है जिसका उपयोग बाइनरी डेटा के साथ काम करने के लिए किया जाता है।


आप बफर को कच्चे बाइट्स के लिए एक निश्चित आकार के कंटेनर के रूप में सोच सकते हैं।


स्ट्रिंग से बफर बनाने का तरीका यहां दिया गया है:


```javascript
const buf = Buffer.from("hello")
console.log(buf)
```


यह कुछ इस तरह प्रिंट होता है:


```
<Buffer 68 65 6c 6c 6f>
```


वे संख्याएँ (`68`, `65`, `6c`, आदि) `"hello"` में अक्षरों के हेक्साडेसिमल निरूपण हैं।


आप इसे इस प्रकार स्ट्रिंग में परिवर्तित कर सकते हैं:


```javascript
console.log(buf.toString())
```


यह प्रिंट करता है:


```
hello
```


आप शून्य से भरे एक निश्चित आकार का बफर भी बना सकते हैं:


```javascript
const buf = Buffer.alloc(10)
console.log(buf)
```


यह कुछ इस तरह प्रिंट होता है:


```
<Buffer 00 00 00 00 00 00 00 00 00 00>
```


आप बफर में लिख सकते हैं:


```javascript
buf.write("abc")
console.log(buf)
```


और आप अलग-अलग बाइट्स तक पहुंच सकते हैं:


```javascript
console.log(buf[0]) // prints the ASCII number for 'a', which is 97
```


बफ़र्स विशेष रूप से तब उपयोगी होते हैं जब आपको बहुत निम्न स्तर पर डेटा में हेरफेर करने की आवश्यकता होती है।


### घटनाएँ


जावास्क्रिप्ट में, **ईवेंट** वह चीज है जो आपके प्रोग्राम में घटित होती है, जिस पर आप प्रतिक्रिया कर सकते हैं।


उदाहरण के लिए:



- फ़ाइल लोड होना समाप्त हो गया
- टाइमर बंद हो जाता है
- कोई उपयोगकर्ता किसी बटन पर क्लिक करता है
- नेटवर्क अनुरोध डेटा लौटाता है


एक घटना केवल एक संकेत है कि कुछ घटित हुआ है, और आप उन घटनाओं को सुनने और उन पर प्रतिक्रिया करने के लिए कोड लिख सकते हैं।


NodeJS में, कई ऑब्जेक्ट इवेंट उत्सर्जित कर सकते हैं। इन ऑब्जेक्ट्स को **EventEmitters** कहा जाता है।


यहाँ एक उदाहरण है:


```javascript
const EventEmitter = require("events")

const emitter = new EventEmitter()

// Listen for an event
emitter.on("greet", () => {
console.log("Hello! An event happened.") // this will get printed when a "greet" event gets fired
})

// Emit the event
emitter.emit("greet")
```


यह प्रिंट करता है:


```
Hello! An event happened.
```


यहाँ देखें:


1. हम एक `EventEmitter` ऑब्जेक्ट बनाते हैं।

2. हम इसे `.on("greet")` का उपयोग करके, जब भी `"greet"` घटना घटित हो, कॉलबैक चलाने के लिए कहते हैं।

3. बाद में, हम `.emit()` का उपयोग करके `"greet"` ईवेंट को ट्रिगर करते हैं।

4. हमारा कॉलबैक निष्पादित हो जाता है


आप इवेंट के साथ डेटा भी पास कर सकते हैं:


```javascript
emitter.on("greet",
(name) => console.log(`Hello, ${name}!`)
)

emitter.emit("greet", "Alice") // first argument is the type of event, second argument is the data we pass with this event
```


यह प्रिंट करता है:


```
Hello, Alice!
```


आप अन्य कार्यक्रमों के लिए भी श्रोताओं को पंजीकृत कर सकते हैं:


```javascript
emitter.on("goodbye", () => {
console.log("Goodbye!")
})

emitter.emit("goodbye")
```


आप किसी भी प्रकार की घटना से जितने चाहें उतने श्रोताओं को जोड़ सकते हैं, और आप एक ही उत्सर्जक से कई अलग-अलग प्रकार की घटनाएं चला सकते हैं।


NodeJS में कई ऑब्जेक्ट्स शेष प्रोग्राम को यह बताने के लिए ईवेंट उत्सर्जित करते हैं कि कुछ घटित हो रहा है।



### धाराएँ क्या हैं?


स्ट्रीम्स बफ़र्स और इवेंट्स को संयोजित करके डेटा को प्रोसेस करने में हमारी मदद करती हैं।


जब हम फ़ाइलों, नेटवर्क से डेटा, या यहाँ तक कि लंबे टेक्स्ट के साथ काम करते हैं, तो हमें हमेशा सब कुछ एक साथ मेमोरी में लोड करने की ज़रूरत नहीं होती (या हम ऐसा चाहते भी नहीं हैं)। अगर डेटा बहुत बड़ा है, तो यह धीमा हो सकता है, या प्रोग्राम क्रैश भी हो सकता है।


इसके बजाय, हम डेटा को **थोड़ा-थोड़ा करके** प्रोसेस कर सकते हैं, जैसे ही वह आता है या पढ़ा जाता है, मानो एक बार में पूरा गिलास पानी पीने की कोशिश करने के बजाय, स्ट्रॉ से पानी पी रहे हों। इसे **स्ट्रीम** कहते हैं।


NodeJS में, स्ट्रीम एक ऑब्जेक्ट है जो आपको किसी स्रोत से डेटा पढ़ने या गंतव्य पर डेटा लिखने की सुविधा देता है **एक समय में एक टुकड़ा**।


NodeJS में चार मुख्य प्रकार की स्ट्रीम हैं:



- पठनीय**: वे स्ट्रीम जिनसे आप डेटा पढ़ सकते हैं (जैसे फ़ाइल पढ़ना)
- लिखने योग्य**: वे स्ट्रीम जिनमें आप डेटा लिख ​​सकते हैं (जैसे किसी फ़ाइल में लिखना)
- डुप्लेक्स**: ऐसी स्ट्रीम जो पठनीय और लिखने योग्य दोनों हों
- रूपांतरण**: डुप्लेक्स स्ट्रीम की तरह, लेकिन वे प्रवाह के दौरान डेटा को बदल (रूपांतरित) सकते हैं


### पठनीय स्ट्रीम


मान लीजिए कि आपके पास प्रोसेस करने के लिए एक `bigfile.txt` है। आप फ़ाइल को टुकड़ों में पढ़ने के लिए `fs` मॉड्यूल की मदद से एक पठनीय स्ट्रीम बना सकते हैं।


```javascript
const fs = require("fs")

const readableStream = fs.createReadStream(
"bigfile.txt"
)

readableStream.on("data", (chunk) => {
console.log("Received chunk:", chunk)
})

readableStream.on("end", () => {
console.log("Finished reading file.")
})

readableStream.on("error", (err) => {
console.error("Error reading file:", err)
})
```


यहाँ क्या होता है?


1. `fs.createReadStream()` एक पठनीय स्ट्रीम बनाता है।

2. जब भी फ़ाइल का कोई भाग तैयार होता है, स्ट्रीम एक `data` इवेंट उत्सर्जित करती है और हमें डेटा का एक "खंड" (एक `Buffer`) देती है। हम इस खंड को प्रिंट करते हैं।

3. जब पूरी फ़ाइल पढ़ ली जाती है, तो `end` इवेंट ट्रिगर हो जाता है।

4. यदि कोई त्रुटि हो (जैसे कि फ़ाइल मौजूद नहीं है), तो `error` ईवेंट ट्रिगर हो जाता है।


इस तरह, आप बड़ी फ़ाइलों को एक साथ मेमोरी में लोड किए बिना पढ़ सकते हैं।


यदि हम चाहते हैं कि डेटा मानव-पठनीय रूप में आए (बाइनरी के बजाय), तो हम स्ट्रीम की एन्कोडिंग निर्दिष्ट कर सकते हैं:


```javascript
const fs = require("fs")

const readableStream = fs.createReadStream(
"bigfile.txt",
{ encoding: "utf8" } // we tell NodeJS that the file should be read as utf8
)

readableStream.on("data", (chunk) => {
console.log("Received chunk:", chunk)
})

readableStream.on("end", () => {
console.log("Finished reading file.")
})

readableStream.on("error", (err) => {
console.error("Error reading file:", err)
})
```


कोड अब फ़ाइल को मानव-पठनीय रूप में प्रिंट करेगा।


### लिखने योग्य स्ट्रीम


एक लिखने योग्य स्ट्रीम आपको डेटा को कहीं भी, खंड दर खंड भेजने की सुविधा देता है।


स्ट्रीम का उपयोग करके `target.txt` फ़ाइल में लिखने का एक उदाहरण यहां दिया गया है:


```javascript
const fs = require("fs")

const stream = fs.createWriteStream("target.txt")

stream.write("First line\n")
stream.write("Second line\n")
stream.end("Finished writing\n")

stream.on("finish", () => {
console.log("All data written.")
})

stream.on("error", err => {
console.error("Error:", err)
})
```


यहाँ क्या होता है:


1. `fs.createWriteStream()` एक लिखने योग्य स्ट्रीम बनाता है।

2. हम `.write()` का उपयोग करके इसमें कुछ पाठ लिखते हैं।

3. जब हमारा काम पूरा हो जाता है, तो हम स्ट्रीम को बंद करने के लिए `.end()` को कॉल करते हैं।

4. जब सारा डेटा लिख ​​लिया जाता है, तो `finish` इवेंट उत्सर्जित होता है।

5. यदि कुछ गलत हो जाता है, तो `error` घटना शुरू हो जाती है।


पठनीय स्ट्रीम की तरह, लेखनीय स्ट्रीम भी बड़े डेटा के लिए अच्छे होते हैं, क्योंकि उन्हें एक ही बार में सब कुछ मेमोरी में रखने की आवश्यकता नहीं होती।


### पाइपिंग धाराएँ


स्ट्रीम्स के बारे में सबसे अच्छी चीजों में से एक यह है कि आप उन्हें एक साथ जोड़ सकते हैं: एक पठनीय स्ट्रीम को सीधे एक लिखने योग्य स्ट्रीम से जोड़ सकते हैं।


```javascript
const fs = require("fs")

const readable = fs.createReadStream("bigfile.txt")
const writable = fs.createWriteStream("target.txt")

readable.pipe(writable)
```


यहाँ:



- पठनीय स्ट्रीम `bigfile.txt` से पढ़ी जाती है।
- लिखने योग्य स्ट्रीम `copy.txt` पर लिखती है।
- `.pipe()` डेटा को सीधे पठनीय से लेखनीय स्ट्रीम में भेजता है।


### डुप्लेक्स स्ट्रीम


एक डुप्लेक्स स्ट्रीम पठनीय और लेखनीय दोनों होती है। इसका एक उदाहरण नेटवर्क सॉकेट है: आप इसमें डेटा भेज सकते हैं और इससे डेटा प्राप्त कर सकते हैं।


यहां `net` मॉड्यूल का उपयोग करके एक बहुत ही सरल उदाहरण दिया गया है:


```javascript
const net = require("net")

const server = net.createServer((socket) => {
socket.write("Welcome!\n")

socket.on("data", (chunk) => {
console.log("Received:",
chunk.toString()  // we convert the chunk of data from Buffer to string
)
})
})

server.listen(3000, () => {
console.log("Server listening on port 3000")
})
```


इस उदाहरण में:



- `सॉकेट` ऑब्जेक्ट एक डुप्लेक्स स्ट्रीम है।
- आप इसमें `write()` कर सकते हैं और इससे `data` ईवेंट भी सुन सकते हैं।


### धाराओं को रूपांतरित करें


ट्रांसफॉर्म स्ट्रीम एक डुप्लेक्स स्ट्रीम है जो इससे गुजरने वाले डेटा को भी संशोधित करती है।


उदाहरण के लिए, आप डेटा को संपीड़ित या विघटित करने के लिए अंतर्निहित `zlib` मॉड्यूल का उपयोग कर सकते हैं।


ट्रांसफ़ॉर्म स्ट्रीम का उपयोग करके फ़ाइल को संपीड़ित करने का तरीका यहां दिया गया है:


```javascript
const fs = require("fs")
const zlib = require("zlib")

const readable = fs.createReadStream("bigfile.txt")     // create a readable stream that reads from a file
const zip = zlib.createGzip()                           // create a transform stream that compresses data
const writable = fs.createWriteStream("bigfile.txt.gz") // create a writable stream that writes to a file

readable          // take the readable stream
.pipe(zip)      // pipe it into the transform stream to compress the data
.pipe(writable) // then pipe it into the writable stream that saves the data to a zipped file

writable.on("finish", () => {
console.log("File compressed.")
})
```


और इसे वापस डिकम्प्रेस करने के लिए:


```javascript
const readable = fs.createReadStream("bigfile.txt.gz")
const unzip = zlib.createGunzip()
const writable = fs.createWriteStream("bigfile.txt")

readable.pipe(unzip).pipe(writable)

writable.on("finish", () => {
console.log("File decompressed.")
})
```


ट्रांसफ़ॉर्म स्ट्रीम संपीड़न, एन्क्रिप्शन या स्ट्रीमिंग के दौरान फ़ाइल स्वरूप बदलने जैसे कार्यों के लिए बहुत उपयोगी हैं।


### वापस दबाव


कभी-कभी एक राइटेबल स्ट्रीम डेटा को संभालने में धीमी होती है। अगर हम डेटा को राइटेबल पर उसकी क्षमता से ज़्यादा तेज़ी से डालते रहें, तो हमें समस्याएँ आ सकती हैं। इसे **बैकप्रेशर** कहते हैं।


यदि आप किसी लिखने योग्य स्ट्रीम पर `.write()` विधि को कॉल करते हैं, तो यह एक बूलियन लौटाता है जो आपको सूचित करता है कि क्या स्ट्रीम को विराम की आवश्यकता है; आपको इसके वापसी मान की जांच करनी पड़ सकती है, जैसे:


```javascript
const fs = require("fs")

const readable = fs.createReadStream("example.txt")
const writable = fs.createWriteStream("copy.txt")
r
readable.on("data", chunk => {               // each chunk of data we read from the readable stream...

const canContinue = writable.write(chunk)  // ...we send it to the writable, which returns us a boolean to confirm we can continue

if (!canContinue) { readable.pause() }     // ...if we can't, we temporarily pause reading
})

writable.on("drain",                // the writable stream emits a "drain" event when the backpressure is gone

() => { readable.resume() }      // so we resume reading (and writing)

)
```


यह डेटा को रीडेबल से राइटेबल में मैन्युअल रूप से पाइप करने तथा बैकप्रेशर को मैन्युअल रूप से प्रबंधित करने का एक उदाहरण था।


आमतौर पर हम `.pipe()` विधि का उपयोग करके डेटा पाइप करते हैं, जो बैकप्रेशर को स्वचालित रूप से संभालता है।


इसलिए आपको बैकप्रेशर के बारे में केवल तभी चिंता करने की आवश्यकता है जब किसी कारण से आप मैन्युअल रूप से `.write()` को कॉल कर रहे हों।
