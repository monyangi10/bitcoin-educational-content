## راه‌اندازی

<chapterId>ba05a290-1782-5268-87c9-62fd09590e05</chapterId>


در این بخش، ما قصد داریم ماشین خود را برای نوشتن و اجرای اولین برنامه جاوااسکریپت خود تنظیم کنیم.


یک برنامه جاوااسکریپت فقط مجموعه‌ای از (یک یا چند) فایل متنی است که حاوی دستورات برای اجرا توسط یک زمان‌اجرای جاوااسکریپت می‌باشد.


نام‌های این فایل‌های متنی معمولاً با پسوند فایل `.js` به پایان می‌رسند، مانند `my_script.js`، `my_program.js` و غیره.


دستورات موجود در آنها به زبان برنامه‌نویسی JavaScript نوشته شده‌اند.


زمان اجرای جاوااسکریپت یک برنامه ویژه است که این فایل‌ها را اجرا می‌کند.


![](assets/en/1.webp)


### نصب NodeJS


رایج‌ترین زمان اجرای JavaScript، NodeJS است.


می‌توانید آن را با دنبال کردن [دستورالعمل‌های رسمی](https://nodejs.org/en/download) نصب کنید.


صفحه دانلود دستورالعمل‌هایی برای هر سه سیستم‌عامل اصلی (Operating Systems) ارائه می‌دهد: ویندوز، لینوکس و مک‌اواس. فرض بر این است که شما می‌دانید چگونه یک ترمینال را در سیستم‌عامل خود باز کنید.


از آنجا که NodeJS برای هر سه سیستم‌عامل در دسترس است، برنامه‌هایی که می‌نویسید قادر خواهند بود بر روی همه آن‌ها اجرا شوند (به جز برخی موارد خاص).


این بدان معناست که شما می‌توانید، به عنوان مثال، یک بازی ویدئویی ساده را در جاوااسکریپت روی رایانه شخصی ویندوز خود بنویسید و آن را به دوستتان بدهید تا روی مک او اجرا کند.


![](assets/en/2.webp)


### ویرایش متن


یکی از ویژگی‌های جالب برنامه‌نویسی این است که می‌توانید با استفاده از هر ویرایشگر متنی، حتی نوت‌پد پیش‌فرض سیستم‌عامل خود، کد بنویسید.


برخی ویرایشگرهای متن وجود دارند که به‌طور خاص برای نوشتن کد طراحی شده‌اند، برخی از آن‌ها به‌صورت رایگان در دسترس هستند و برخی دیگر نیاز به خرید مجوز دارند.


انتخاب ویرایشگر کد یک موضوع پیچیده است که فراتر از محدوده این دوره می‌باشد، بنابراین در اینجا به آن نمی‌پردازیم. اگر نمی‌دانید از چه چیزی استفاده کنید، پرکاربردترین ویرایشگر رایگان [VSCode](https://code.visualstudio.com/) است.


Interface آن کمی حجیم است، اما آنچه را که نیاز دارید دارد: یک ویرایشگر فایل، یک مرورگر فایل (برای مشاهده فایل‌ها و زیرشاخه‌ها در دایرکتوری که روی آن کار می‌کنید)، و یک ترمینال برای اجرای کد شما. همچنین از بسیاری از افزونه‌ها پشتیبانی می‌کند و به طور پیش‌فرض با برجسته‌سازی نحو جاوااسکریپت ارائه می‌شود.


اگر می‌خواهید کمی بیشتر Cypherpunk-y باشید، می‌توانید به جای آن از [VSCodium](https://vscodium.com/) استفاده کنید.


### اولین برنامه (سلام دنیا)


به طور سنتی، هنگام یادگیری یک زبان برنامه‌نویسی، اولین برنامه‌ای که نوشته می‌شود شامل چاپ "hello world!" در کنسول است.


دایرکتوری‌ای به نام `my_js_code/` ایجاد کنید و درون آن فایلی به نام `main.js` قرار دهید (این نام‌ها دلخواه هستند).


دایرکتوری را با VSCode باز کنید.


این کد را در فایل خود بنویسید:


```javascript
console.log("hello world!")
```


یک ترمینال باز کنید و این دستور را برای اجرای برنامه اجرا کنید:


```
node main.js
```


نتیجه باید باشد


```
hello world!
```


### چه اتفاقی افتاد؟


در جاوااسکریپت، همه چیز یک "شیء" است.


`console` یک شیء است که برای اشکال‌زدایی برنامه استفاده می‌شود.


`console.log` پرکاربردترین متد `console` است. این متد هر آرگومانی که به آن بدهید را چاپ می‌کند.


شما آرگومان‌ها را با استفاده از پرانتزهای گرد `()` به `console.log` ارسال می‌کنید.


بنابراین، به عنوان مثال، اگر می‌خواستید عدد `1000` را چاپ کنید، فقط باید بنویسید


```javascript
console.log(1000)
```


سپس آن را با اجرای


```
node main.js
```


در ترمینال خود (از این به بعد، این دوره فرض می‌کند که شما می‌دانید این همان روشی است که یک برنامه را اجرا می‌کنید).


این باید چاپ شود


```
1000
```


شما می‌توانید چندین چیز را عبور دهید، مانند


```javascript
console.log(16, 8, 1993)
```


این چاپ خواهد شد


```
16 8 1993
```


## متغیرها و توضیحات

<chapterId>23050ab7-343b-5edf-9d37-e4e782e27ce0</chapterId>


برنامه‌ها معمولاً عملیات را بر روی داده‌ها اجرا می‌کنند.


متغیرها مانند جعبه‌های نام‌گذاری شده‌ای هستند که ما برای ذخیره داده‌ها از آن‌ها استفاده می‌کنیم. آن‌ها به ما اجازه می‌دهند تا یک قطعه داده را با یک نام خاص مرتبط کنیم، به طوری که بتوانیم بعداً با استفاده از آن نام، داده را بازیابی کنیم.


### اعلانات `let`


برای اعلام یک متغیر در جاوااسکریپت، می‌توانیم از کلمه کلیدی `let` استفاده کنیم.


پس از نوشتن `let`، نامی که می‌خواهیم به متغیر بدهیم را می‌نویسیم، سپس علامت `=` و سپس مقداری که می‌خواهیم ذخیره کنیم.


برای مثال:


```javascript
let age = 25

console.log(age)
```


نام یک متغیر (که به‌طور فنی "شناسه" نامیده می‌شود) معمولاً می‌تواند شامل حروف، زیرخط (`_`)، علامت دلار (`$`) و اعداد باشد، اگرچه اولین کاراکتر نمی‌تواند یک عدد باشد.


در کد بالا، یک متغیر به نام `age` تعریف کردیم و مقدار `25` را در آن ذخیره کردیم.


سپس، مقدار را با استفاده از `console.log(age)` چاپ کردیم.


اگر این کد را با `node main.js` اجرا کنید، خروجی به صورت زیر خواهد بود:


```
25
```


شناسه‌ها به حروف بزرگ و کوچک حساس هستند، به این معنی که حروف کوچک و بزرگ به عنوان تفاوت در شناسه‌ها محسوب می‌شوند، بنابراین به عنوان مثال


```javascript
let age = 25

let Age = 20

console.log(age)
```


25 را چاپ خواهد کرد، زیرا آن‌ها به‌عنوان دو متغیر کاملاً جداگانه در نظر گرفته می‌شوند!


شما همچنین می‌توانید رشته‌ها (متن) را در یک متغیر ذخیره کنید:


```javascript
let message = "hello again"

console.log(message)
```


این چاپ خواهد شد:


```
hello again
```


درست مانند قبل، ما از `console.log()` برای چاپ مقدار ذخیره شده در متغیر استفاده کردیم.


حالا بیایید هر دو را با هم انجام دهیم:


```javascript
let age = 25

let message = "hello again"

console.log(age)

console.log(message)
```


اجرای این دستور چاپ خواهد کرد:


```
25
hello again
```

### انتقال مجدد


متغیرهایی که با `let` اعلام می‌شوند، می‌توانند پس از ایجاد تغییر کنند.


این به عنوان تغییر وظیفه شناخته می‌شود.


```javascript
let score = 10

console.log(score)

score = 15

console.log(score)
```


ابتدا `10` را به `score` اختصاص می‌دهیم، سپس آن را چاپ می‌کنیم.


سپس مقدار `score` را به `15` تغییر می‌دهیم و دوباره آن را چاپ می‌کنیم.


خروجی خواهد بود:


```
10
15
```


این زمانی که مقدار در طول زمان تغییر می‌کند بسیار مفید است، مانند یک بازی که در آن امتیاز افزایش می‌یابد.


بیایید یک متغیر دیگر به ترکیب اضافه کنیم:


```javascript
let score = 10
let player = "Alice"

console.log(score)
console.log(player)

score = 20
player = "Bob"

console.log(score)
console.log(player)
```


این چاپ خواهد شد:


```
10
Alice
20
Bob
```


همان‌طور که می‌بینید، هم `score` و هم `player` تغییر کرده‌اند.


### اعلانات `const`


اغلب اوقات، ما نمی‌خواهیم یک متغیر پس از ایجاد تغییر کند. برای این منظور، از `const` استفاده می‌کنیم.


`const` مخفف "ثابت" است. هنگامی که مقداری را به یک متغیر `const` اختصاص می‌دهید، نمی‌توانید آن را تغییر دهید.


```javascript
const pi = 3.14
console.log(pi)
```


این چاپ می‌کند:


```
3.14
```


اما اگر سعی کنید این کار را انجام دهید:


```javascript
const pi = 3.14
console.log(pi)

pi = 99 // this line will cause an error
console.log(pi)
```


جاوااسکریپت به شما خطایی مانند زیر خواهد داد:


```
TypeError: Assignment to constant variable.
```


این به این دلیل است که `pi` با استفاده از `const` اعلام شده است و شما نمی‌توانید مقدار آن را پس از آن تغییر دهید. شما به مفسر جاوااسکریپت اعلام می‌کنید که نمی‌خواهید آن متغیر تغییر کند.


این مفید است زیرا احتمال تغییر آن به اشتباه را کاهش می‌دهد. وقتی برنامه‌ها بسیار بزرگ می‌شوند و شامل هزاران خط کد می‌شوند، پیگیری همه چیزهایی که به طور همزمان اتفاق می‌افتند غیرممکن است (این دلیل اصلی استفاده ما از کامپیوترها است، برای اجرای فرآیندهای پیچیده‌ای که نمی‌توانیم با مغزمان محاسبه کنیم)، بنابراین داشتن محدودیت‌هایی مانند این، که برنامه را بیشتر قطعی می‌کند، مفید می‌شود.


بهترین روش این است که همیشه مقادیر خود را به صورت `const` اعلام کنیم، مگر اینکه مطمئن باشیم می‌خواهیم آن‌ها را بعداً تغییر دهیم.


### نظرات در جاوااسکریپت


گاهی اوقات می‌خواهیم در کد خود یادداشت‌هایی بنویسیم که اجرا نمی‌شوند. به این‌ها نظرات یا کامنت‌ها گفته می‌شود.


نظرات توسط برنامه هنگام اجرا نادیده گرفته می‌شوند، اما برای توضیح دادن به خودمان یا دیگران مفید هستند.


برای نوشتن یک نظر تک‌خطی، از `//` استفاده کنید


```javascript
// This is a comment
const x = 10 // This is also a comment
console.log(x)
```


این هنوز چاپ خواهد شد:


```
10
```


نظرات فقط برای خواندن انسان‌ها وجود دارند.


شما همچنین می‌توانید از `/*` و `*/` برای نوشتن نظرات چند خطی استفاده کنید.


```javascript
/*
This is a multi-line comment.
It can span several lines.
*/
const y = 20
console.log(y)
```


این چاپ خواهد شد


```
20
```


و نظر داده نخواهد شد.


می‌توانید از نظرات برای افزودن توضیحات کوچک به کد خود استفاده کنید، تا به یاد بیاورید که کد چه کاری انجام می‌دهد و چرا به شیوه خاصی نوشته شده است. این کار همچنین می‌تواند به دیگر برنامه‌نویسان در درک کد کمک کند.


## انواع پایه: اعداد، رشته‌ها، بولین‌ها

<chapterId>cfdb04f6-21a8-5143-bbf9-7aaae04962f0</chapterId>


در جاوااسکریپت، "نوع" به شما می‌گوید که یک مقدار چه نوع داده‌ای است.


جاوااسکریپت چند نوع پایه دارد و در این بخش به بررسی برخی از آن‌ها خواهیم پرداخت.


### اعداد و عملیات حسابی


اولین نوعی که معرفی خواهیم کرد `number` است.


اعداد در جاوااسکریپت می‌توانند اعداد صحیح (مانند `5`) یا اعشاری (مانند `3.14`) باشند.


شما می‌توانید با آن‌ها محاسبات انجام دهید: جمع، تفریق، ضرب و تقسیم.


در اینجا یک مثال ساده آمده است:


```javascript
const a = 10
const b = 5

const sum = a + b
const difference = a - b
const product = a * b
const quotient = a / b

console.log(sum)
console.log(difference)
console.log(product)
console.log(quotient)
```


این چاپ خواهد شد:


```
15
5
50
2
```


شما همچنین می‌توانید از پرانتزها `()` برای کنترل ترتیب عملیات استفاده کنید:


```javascript
const result = (2 + 3) * 4
console.log(result)
```


این چاپ می‌شود:


```
20
```


بدون پرانتز، عبارت به صورت `2 + 3 * 4` خواهد بود، که:


```javascript
const result = 2 + 3 * 4
console.log(result)
```


این چاپ خواهد شد:


```
14
```


زیرا در ریاضیات معمولی، ضرب قبل از جمع انجام می‌شود.


### رشته‌ها و درون‌یابی


دومین نوع جاوااسکریپتی که معرفی خواهیم کرد `string` است.


رشته‌ها قطعاتی از متن هستند. شما می‌توانید از تک‌نقل‌قول‌ها `'...'` یا نقل‌قول‌های دوتایی `"..."` برای ایجاد آن‌ها استفاده کنید.


```javascript
const greeting = "hello"
const name = 'Bob'
console.log(greeting)
console.log(name)
```


این چاپ می‌شود:


```
hello
Bob
```


برای ترکیب رشته‌ها، می‌توانید از عملگر `+` استفاده کنید:


```javascript
const greeting = "hello"
const space = " "
const name = "Bob"

const fullGreeting = greeting + space + name
console.log(fullGreeting)
```


این چاپ خواهد شد:


```
hello Bob
```


اما روش بهتری برای ترکیب رشته‌ها وجود دارد که به آن **درج رشته‌ای** می‌گویند. شما از بک‌تیک‌ها برای اعلام رشته استفاده می‌کنید `` `...` `` و متغیرها را با `${...}` درون رشته می‌نویسید:


```javascript
const greeting = "hello"
const name = "Bob"

const fullGreeting = `${greeting} ${name}`
console.log(fullGreeting)
```


این نیز چاپ می‌کند:


```
hello Bob
```


می‌توانید هر عبارتی را داخل `${...}` قرار دهید:


```javascript
const age = 30
console.log(`Next year, I will be ${age + 1} years old.`)
```


این چاپ می‌شود:


```
Next year, I will be 31 years old.
```


درون‌یابی در جاوااسکریپت مدرن بسیار رایج است.


### بولین‌ها، عملیات مقایسه و منطق


سومین نوعی که معرفی خواهیم کرد `boolean` است. این نام به افتخار ریاضیدان جورج بول، که منطق بولی را ابداع کرد، نام‌گذاری شده است.


بولین‌ها ساده هستند: تنها دو مقدار ممکن، `true` و `false`.


می‌توانید آن‌ها را در متغیرها ذخیره کنید:


```javascript
const theSkyIsBlue = true
const thisCourseIsBad = false

console.log(theSkyIsBlue)
console.log(thisCourseIsBad)
```


این چاپ می‌شود:


```
true
false
```


می‌توانید بولین‌ها را با استفاده از عملگرهای منطقی ترکیب کنید:



- `&&` به معنای "و" است و تنها زمانی `true` را برمی‌گرداند که **هر دو** مقدار `true` باشند، در غیر این صورت `false` را برمی‌گرداند.
- `||` به معنای "یا" است و اگر **حداقل یکی** از مقادیر `true` باشد، مقدار `true` را برمی‌گرداند، در غیر این صورت (اگر هر دو مقدار `false` باشند) مقدار `false` را برمی‌گرداند.
- `!` به معنای "نه" است، قبل از یک بولین اعمال می‌شود و آن را برعکس می‌کند: اگر بولین `true` باشد، `false` برمی‌گرداند و بالعکس.


![](assets/en/3.webp)


مثال‌ها:


```javascript
const isSunny = true
const isWarm = true

console.log(isSunny && isWarm)  // true
console.log(isSunny || isWarm)  // true
console.log(!isSunny)           // false
```


می‌توانید مقادیر را در جاوااسکریپت با استفاده از عملگرهایی مانند `>`، `<`، `===` و `!==` مقایسه کنید. نتیجه این مقایسه‌ها همیشه یک مقدار بولی است.


```javascript
const first = 10
const second = 5

const firstIsGreater   = (a > b)
const secondIsGreater  = (a < b)
const theyAreEqual     = (a === b)
const theyAreDifferent = (a !== b)

console.log(firstIsGreater)   // true
console.log(secondIsGreater)   // false
console.log(theyAreEqual)  // false
console.log(theyAreDifferent)  // true
```


جاوااسکریپت همچنین از `>=` برای "بزرگتر یا مساوی" و `<=` برای "کوچکتر یا مساوی" استفاده می‌کند.


بولین‌ها، عملگرهای مقایسه‌ای و منطقی اغلب در برنامه‌ها برای اعلام شرایط پیچیده ترکیب می‌شوند، مانند اطمینان از اینکه "ایمیل رسیده است و حاوی تصویری است که نیاز دارم یا طول ایمیل بیشتر از 10000 کاراکتر است". بعداً خواهید دید که این‌ها اجزای اساسی برای ساخت منطق برنامه هستند.


## آرایه‌ها، null، undefined

<chapterId>7bf18183-5eae-53ed-83d2-b04982145d81</chapterId>


در این بخش، به سه نوع دیگر که در برنامه‌های جاوااسکریپت بسیار رایج هستند، خواهیم پرداخت:



- آرایه‌ها**: دنباله‌هایی از مقادیر
- تعریف‌نشده**: یک مقدار ویژه که به معنای "هیچ چیزی اختصاص داده نشده است" می‌باشد.
- null**: یک مقدار ویژه دیگر که به معنای "عمداً خالی" است


### آرایه‌ها و دسترسی به ایندکس‌ها


آرایه **نوعی** است که می‌تواند مقادیر متعددی را در یک لیست نگه دارد.


شما یک آرایه را با استفاده از کروشه‌ها `[]` و جدا کردن آیتم‌ها با کاما ایجاد می‌کنید.


در اینجا یک مثال ساده آمده است:


```javascript
const numbers = [10, 2, 88]
console.log(numbers)
```


این چاپ می‌کند:


```
[ 10, 2, 88 ]
```


شما می‌توانید هر چیزی را در یک آرایه ذخیره کنید، نه فقط اعداد:


```javascript
const things = ["apple", 42, true]
console.log(things)
```


این چاپ می‌کند:


```
[ 'apple', 42, true ]
```


برای دسترسی به یک آیتم خاص در آرایه، از یک **اندیس** استفاده می‌کنیم. اندیس موقعیت آیتم است که از **0** شروع می‌شود.


بنابراین در این آرایه:


```javascript
const colors = ["red", "green", "blue"]
```



- `colors[0]` برابر است با `"red"`
- `colors[1]` برابر است با `"Green"`
- `colors[2]` برابر است با `"blue"`


بیایید امتحان کنیم:


```javascript
const colors = ["red", "green", "blue"]
console.log(colors[0])
console.log(colors[1])
console.log(colors[2])
```


این چاپ خواهد شد:


```
red
green
blue
```


می‌توانید یک مقدار را به یک شاخص خاص از یک آرایه اختصاص دهید


```javascript
const colors = ["red", "green", "blue"]

colors[1] = "yellow"

console.log(colors)
```


این چاپ خواهد شد:


```
[ 'red', 'yellow', 'blue' ]
```


می‌توانید از هر عدد طبیعی به عنوان شاخص استفاده کنید، حتی عددی که در یک متغیر ذخیره شده است.


```javascript

const i = 1
const colors = ["red", "green", "blue"]
console.log(colors[i])
```


این چاپ خواهد شد:


```
green
```


اما اگر سعی کنید به ایندکسی که وجود ندارد دسترسی پیدا کنید، مقدار `undefined` را دریافت خواهید کرد:


```javascript
const colors = ["red", "green", "blue"]
console.log(colors[3])
```


این چاپ می‌کند:


```
undefined
```


اون چیه؟؟


### `تعریف‌نشده`


مقدار خاص `undefined` به معنای "هیچ مقداری اختصاص داده نشده است" می‌باشد.


اگر یک متغیر ایجاد کنید اما به آن مقداری ندهید، مقدار آن `undefined` خواهد بود:


```javascript
const name
console.log(name)
```


این چاپ می‌کند:


```
undefined
```


چون ما هیچ مقداری به `name` اختصاص ندادیم، جاوااسکریپت به طور پیش‌فرض آن را به `undefined` تنظیم می‌کند.


همانطور که قبلاً دیده شد، زمانی که به یک شاخص آرایه که وجود ندارد دسترسی پیدا می‌کنید، می‌توانید `undefined` نیز دریافت کنید:


```javascript
const fruits = ["banana", "apple"]
console.log(fruits[2]) // There is no index 2
```


این چاپ می‌کند:


```
undefined
```


### `null` و نحوه برخورد با آن


`null` همچنین یک مقدار ویژه است. به این معنی است که "اینجا چیزی نیست و من عمداً این کار را انجام دادم."


برخلاف `undefined` که به‌صورت خودکار است، `null` چیزی است که خودتان تنظیم می‌کنید.


برای مثال:


```javascript
const currentUser = null
console.log(currentUser)
```


این چاپ می‌شود:


```
null
```


چرا از `null` استفاده کنیم؟ شاید انتظار دارید که یک مقدار بعداً بیاید، اما هنوز آماده نیست:


```javascript
let winner = null

// Later in the program:
winner = "Alice"

console.log(winner)
```


این چاپ می‌کند:


```
Alice
```


بنابراین `null` زمانی مفید است که می‌خواهید بگویید، به عنوان مثال، "باید بعداً چیزی اینجا باشد، اما در حال حاضر خالی است."


## بلوک‌ها و جریان کنترل

<chapterId>be985168-2636-5b0d-a48f-ac1bbfbff8a7</chapterId>


تا کنون، ما بیشتر خطوط کدی نوشته‌ایم که یکی پس از دیگری اجرا می‌شوند.


اما وقتی کدنویسی می‌کنیم، می‌توانیم ترتیب اجرای آن را کنترل کنیم.


این به عنوان **کنترل جریان** شناخته می‌شود.


بیایید با درک بلوک‌ها و حوزه‌ها شروع کنیم.


### محدوده جهانی


هر متغیری که تعریف می‌کنیم در یک **حوزه** وجود دارد، که به معنای ناحیه‌ای از کد است که متغیر در آن شناخته شده است.


اگر یک متغیر را خارج از هر بلوکی اعلام کنید، در **حوزه‌ی سراسری** وجود دارد.


```javascript
const color = "blue"
console.log(color)
```


این متغیر `color` در حوزه‌ی سراسری قرار دارد، بنابراین می‌توان از هر جایی در فایل به آن دسترسی داشت.


اگر خطوط بیشتری اضافه کنید:


```javascript
const color = "blue"
console.log(color)

const size = "large"
console.log(color)
console.log(size)
```


هر دو متغیر `color` و `size` متغیرهای سراسری هستند. آن‌ها در همه جای فایل در دسترس هستند.


اما درون یک بلوک چه اتفاقی می‌افتد؟


### بلوک‌ها و حوزه محلی


یک **بلوک** قطعه‌ای از کد است که با آکولاد `{}` احاطه شده است.


متغیرهایی که با `let` یا `const` درون یک بلوک اعلام می‌شوند، **فقط** درون همان بلوک وجود دارند.


```javascript
{
const message = "inside block"
console.log(message)
}
```


این چاپ می‌کند:


```
inside block
```


اما اگر این را امتحان کنید:


```javascript
{
const message = "inside block"
}
console.log(message) // Error!
```


جاوااسکریپت به شما خطایی مانند زیر خواهد داد:


```
ReferenceError: message is not defined
```


این به این دلیل است که `message` در داخل بلوک اعلام شده و در خارج از آن وجود ندارد.


این بدان معناست که می‌توانیم از بلوک‌ها برای جداسازی بخش‌هایی از کد خود استفاده کنیم و مطمئن باشیم که "آنچه در بلوک اتفاق می‌افتد در بلوک می‌ماند" (کمی شبیه لاس وگاس).


سازماندهی کد ما در بلوک‌ها به ما اجازه می‌دهد تا اجرای برنامه را نیز با ساختارهای جریان کنترل مانند `if` ساختار دهیم.


### `if`، `else`


گاهی اوقات می‌خواهیم کد را **فقط اگر** چیزی درست باشد اجرا کنیم. این همان کاری است که دستور `if` انجام می‌دهد.


```javascript
const myAge = 20

console.log("Am I an adult?")

if (myAge >= 18) {
console.log("Yes I am!")
}
```


این چاپ می‌کند:


```
Am I an adult?
Yes I am!
```


همانطور که می‌بینید، کد `myAge` و `18` را مقایسه می‌کند.

در این مورد عملگر `>=` مقدار `true` را برمی‌گرداند، بنابراین بلوک اجرا می‌شود.

اگر شرط `true` نباشد، بلوک اجرا نمی‌شود.


```javascript
const myAge = 17

console.log("Am I an adult?")

if (myAge >= 18) {
console.log("Yes I am!")
}
```


این چاپ می‌کند:


```
Am I an adult?
```


می‌توانید یک بلوک `else` اضافه کنید تا حالت مخالف را مدیریت کند:


```javascript
const myAge = 17

console.log("Am I an adult?")

if (myAge >= 18) {
console.log("Yes I am!")
} else {
console.log("No, I am not.")
}
```


این چاپ می‌کند:


```
Am I an adult?
No, I am not.
```


هم بلوک‌های `if` و هم بلوک‌های `else` همچنان بلوک هستند - بنابراین متغیرهایی که درون آن‌ها اعلام می‌شوند، در خارج وجود ندارند.


اگر می‌خواهیم مطمئن شویم که چیزی **درست نیست**، چه کاری می‌توانیم انجام دهیم؟


خب، همان‌طور که قبلاً بحث شد، جاوااسکریپت یک عملگر "not" دارد که بولین‌ها را معکوس می‌کند. بنابراین می‌توانیم انجام دهیم


```javascript
const myAge = 17

const adult = myAge >= 18

console.log("Am I an adult?")

if (!adult) {
console.log("No, I am not.")
}
```

این هنوز چاپ می‌کند:


```
Am I an adult?
No, I am not.
```

زیرا ما از عملگر `!` برای معکوس کردن متغیر `adult` استفاده کردیم.


`if (!adult) {...}` باید به صورت "اگر بالغ نیست..." خوانده شود.


با استفاده از بلوک‌ها، منطق و عملگرهای مقایسه، می‌توانیم اجرای برنامه را ساختاردهی کنیم، با تعریف متغیرهایی که باید `true` (یا `false`) باشند تا چیزی اتفاق بیفتد.


### `while`، `break`، `continue`


یک حلقه `while` کد را *تا زمانی که* یک شرط درست باشد تکرار می‌کند.


```javascript
let count = 0

while (count < 3) {
console.log("Count is", count)
count = count + 1
}
console.log("the loop is over!")
```


این چاپ می‌کند:


```
Count is 0
Count is 1
Count is 2
the loop is over!
```


وقتی `count` به ۳ می‌رسد، حلقه متوقف می‌شود.


می‌توانید با استفاده از `break` یک حلقه را زودتر متوقف کنید:


```javascript
let number = 1 // Start with number 1

while (true) { // This condition is always true, so this loop will run forever unless we stop it
console.log(number) // Print the current number
if (number === 3) { // If the number is 3, stop the loop
break
}
number = number + 1 // Add 1 to the number
}
```


این چاپ می‌شود:


```
0
1
2
```


زیرا وقتی عدد به `3` می‌رسد، بلوک `if` اجرا می‌شود و حلقه متوقف می‌شود.


می‌توانید با استفاده از `continue` از ادامه یک حلقه صرف‌نظر کنید:


```javascript
let number = 0 // Start with number 0

while (number < 5) { // Keep going while number is less than 5
number = number + 1 // Add 1 to the number

if (number === 3) { // If the number is 3
continue // Skip the rest of the block and go to the next iteration of the loop
}

console.log(number) // Print the number
}
```


این چاپ می‌کند:


```
1
2
4
5
```


زیرا وقتی عدد `3` بود، `continue` باعث شد برنامه خطی که عدد را چاپ می‌کند را نادیده بگیرد.


### `for ... of ...`


اگر یک آرایه دارید و می‌خواهید کاری روی هر آیتم آن انجام دهید، می‌توانید از `for ... of ... {...}` استفاده کنید.


```javascript
const fruits = ["apple", "banana", "cherry"]

for (const fruit of fruits) {
console.log(fruit)
}
```


این چاپ می‌کند:


```
apple
banana
cherry
```

بلوک برای هر عنصر آرایه یک بار اجرا خواهد شد.


متغیر `fruit` در اینجا یک متغیر جدید است که مقدار هر آیتم در آرایه را می‌گیرد تا درون بلوک بر روی آن عمل کند.


### `برای ... در ...`


می‌توانید از `for ... in` برای پیمایش کلیدها (شاخص‌ها) یک آرایه استفاده کنید:


```javascript
const fruits = ["apple", "banana", "cherry"]

for (const index in fruits) {
console.log(index)
}
```


این چاپ می‌کند:


```
0
1
2
```


می‌توانید از شاخص برای دریافت مقدار نیز استفاده کنید:


```javascript
const fruits = ["apple", "banana", "cherry"]

for (const index in fruits) {
console.log(fruits[index])
}
```


این همان چیزی را چاپ می‌کند که `for ... of` چاپ می‌کند:


```
apple
banana
cherry
```


در عمل، برای آرایه‌ها، بهتر است از `for ... of` استفاده کنید، زیرا ساده‌تر و تمیزتر است.


### حلقه‌های محدود


گاهی اوقات می‌خواهیم یک حلقه را تعداد مشخصی بار اجرا کنیم، یا به طور کلی یک قطعه کد بنویسیم که یک بلوک را تکرار کند و در عین حال چیزی را پیگیری کند.

این همان چیزی است که یک حلقه `for` محدود برای آن خوب است.

یک حلقه محدود معمولاً سه شرط می‌گیرد که با یک نقطه‌ویرگول `;` جدا شده‌اند، مانند `(... ; ... ; ....)`.


```javascript
for (let i = 0; i < 3; i = i + 1) {
console.log(i)
}
```


این چاپ می‌کند:


```
0
1
2
```


بیایید آن را توضیح دهیم:



- `let i = 0`: یک متغیر را برای استفاده در بلوک اعلام می‌کند (در این مورد، یک شمارنده است که از 0 شروع می‌شود)
- `i < 3`: شرطی را اعلام می‌کند که برای اجرای بلوک باید `true` باشد (در این مورد "تکرار در حالی که `i` کمتر از 3 است")
- `i = i + 1`: اعلام کنید که کدی پس از هر اجرای بلوک اجرا شود (در این مورد "افزایش `i` به اندازه 1")


همان‌طور که می‌بینید، حلقه محدود به ما اجازه می‌دهد شرایط پیچیده‌تری را برای اجرای مکرر یک قطعه کد اعلام کنیم، اما در بیشتر مواقع این کار ضروری نیست.


### برچسب‌های بلوک


اگر نیاز دارید جریان کنترلی پیچیده‌تری بنویسید، جاوااسکریپت به شما اجازه می‌دهد با استفاده از یک **برچسب** یک بلوک را نام‌گذاری کنید که می‌تواند توسط `break` یا `continue` برای مشخص کردن *محل* بازگشت استفاده شود.


مثال:


```javascript
outer: {
console.log("We're inside the outer scope.")

inner: {
console.log("We're inside the inner scope.")
break outer
}

console.log("This will not run")
}

console.log("Done")
```


این چاپ می‌کند:


```
Inside outer block
Inside inner block
Done
```


ما از `break outer` برای خروج کامل از بلوک `outer` استفاده کردیم.


شما همچنین می‌توانید به حلقه‌ها برچسب بزنید. بیایید این مثال را بررسی کنیم:


```javascript
// Declare a variable to count the total number of days in a year
let totalDaysInOneYear = 0

// Declare one variable per month, with the number of the month
const january = 1
const february = 2
const march = 3
const april = 4
const may = 5
const june = 6
const july = 7
const august = 8
const september = 9
const october = 10
const november = 11
const december = 12

// Declare an array that holds the months that have 30 days
const monthsWith30Days = [
april, june, september, november
]

// Declare variables to keep track of the month and day we're in
let currentMonth = january
let currentDay = 1

monthsLoop: while (true) {  // Start a loop labeled "monthsLoop" to process each month

daysLoop: while (true) {  // Start a loop labeled "daysLoop" to process each day in the month
totalDaysInOneYear = totalDaysInOneYear + 1  // Increase the total number of days we counted by 1

if (                                                                   // We want to check if we're at the end of the month.
currentDay === 31                                                  // Check if the current day is 31 (for months with 31 days)...
|| currentDay === 30 && (monthsWith30Days.includes(currentMonth))  // ...or 30 if it's among the 30-days months...
|| currentDay === 28 && (currentMonth === february)                // ...or 28 if it's February. If it's any of these three, then:

){
currentMonth = currentMonth + 1  // Move to the next month
currentDay = 1                   // Reset the day to 1 for the new month
break daysLoop                   // Exit the inner loop (which tracks days) and go back to the outer loop (which tracks months)
}
else { currentDay = currentDay + 1 }                                   // Otherwise, we're not at the end of the month, and we just move to the next day

}
if (currentMonth > 12) {  // After processing a month, check if we've gone past December
break monthsLoop  // If so, break the outer loop and stop the day-counting process
}
}

console.log(totalDaysInOneYear)  // Print the total number of days in the year (should be 365)
```

این یک مثال بسیار خسته‌کننده بود اما امیدواریم که نیاز (گاه‌به‌گاه) به برچسب‌ها را روشن کرده باشد.


## معرفی توابع

<chapterId>cc324715-09c2-5cf7-9e6f-47a6f16bc04d</chapterId>


همانطور که برنامه‌های شما رشد می‌کنند، اغلب می‌خواهید قطعات کد را **مجدد استفاده** کنید.


این همان چیزی است که **توابع** برای آن هستند: آن‌ها به شما اجازه می‌دهند مقداری کد را با هم گروه‌بندی کنید، به آن یک نام بدهید و هر زمان که خواستید آن را اجرا کنید.


### اعلان تابع


برای تعریف یک تابع، می‌توانیم از کلمه کلیدی `function` استفاده کنیم. سپس به آن یک نام، یک جفت پرانتز `()` با آرگومان‌هایی که می‌گیرد، و یک بلوک کد `{}` برای اجرا می‌دهیم. بیایید با تابعی که هیچ آرگومانی نمی‌گیرد شروع کنیم:


```javascript
function sayHello () {console.log(`Hello!`) }
```


این کد **تابع را اعلام می‌کند**، اما هنوز آن را **اجرا نمی‌کند**.


### فراخوانی توابع


برای اجرای (یا "فراخوانی") تابع، نام آن را به‌دنبال پرانتزها می‌نویسید:


```javascript
function sayHello () {console.log(`Hello!`) }
sayHello()
```


این چاپ می‌شود:


```
Hello!
```


می‌توانید تابع را هر چند بار که می‌خواهید فراخوانی کنید:


```javascript
function sayHello() {
console.log("Hello!")
}

sayHello()
sayHello()
```


این چاپ می‌کند:


```
Hello!
Hello!
```


کد داخل تابع فقط زمانی اجرا می‌شود که آن را فراخوانی کنید.


### آرگومان‌های تابع (ورودی به توابع)


گاهی اوقات، می‌خواهید یک تابع با ورودی خاصی کار کند. می‌توانید این کار را با افزودن **آرگومان‌ها** داخل پرانتز انجام دهید.


برای مثال:


```javascript
function sayHelloTo (friend) {
console.log(`Hello ${friend}!`)
}
```


اکنون این تابع **یک آرگومان** به نام `friend` می‌گیرد.


وقتی تابع را فراخوانی می‌کنید، می‌توانید یک مقدار را ارسال کنید:


```javascript
sayHelloTo("Tommy")
```


این چاپ می‌کند:


```
Hello Tommy!
```


می‌توانید تابع را با نام دیگری دوباره فراخوانی کنید:


```javascript
sayHelloTo("Sam")
```


این چاپ می‌کند:


```
Hello Sam!
```


مقداری که وارد می‌کنید، متغیر `friend` را درون تابع جایگزین می‌کند.


شما همچنین می‌توانید از بیش از یک آرگومان استفاده کنید:


```javascript
function greetTwoPeople(person1, person2) {
console.log(`Hello ${person1} and ${person2}!`)
}

greetTwoPeople("Lina", "Marco")
```


این چاپ می‌شود:


```
Hello Lina and Marco!
```


### بازگرداندن (خروجی از توابع)


توابع می‌توانند مقادیر را نیز **برگردانند**. این بدان معناست که آن‌ها یک مقدار را به جایی که تابع فراخوانی شده بود، بازمی‌گردانند.


در اینجا یک مثال ساده آورده شده است:


```javascript
function getNumber() {
return 42
}

const result = getNumber()
console.log(result)
```


این چاپ می‌کند:


```
42
```


تابع `getNumber()` مقدار `42` را برمی‌گرداند و ما آن را در `result` ذخیره کرده و سپس چاپ می‌کنیم.


شما همچنین می‌توانید چیزی را که محاسبه می‌کنید، برگردانید:


```javascript
function add(a, b) {
return a + b
}

const result = add(2, 3)
console.log(result)
```


این چاپ می‌کند:


```
5
```


به محض اینکه یک مقدار `return` شود، تابع متوقف می‌شود. هر چیزی که بعد از `return` در آن بلوک باشد، اتفاق نمی‌افتد.


```javascript
function saySomething() {

return "hi"

console.log("this never runs")

}

const message = saySomething()
console.log(message)
```


این فقط چاپ می‌کند:


```
hi
```


زیرا فقط `"hi"` بازگردانده می‌شود. خط `console.log("this never runs")` نادیده گرفته می‌شود.


می‌توانید به توابع به‌عنوان زیر‌برنامه‌های کوچک فکر کنید. هر تابع می‌تواند مقداری ورودی بگیرد، روی آن کار کند و مقداری خروجی به شما بدهد.


اگر سعی کنیم از مقدار بازگشتی یک تابع استفاده کنیم، اما آن تابع چیزی برنگرداند، چه اتفاقی می‌افتد؟


```javascript
function doesNothing () {}

const x = doesNothing()

console.log(x)
```

این `undefined` را چاپ خواهد کرد. مقدار بازگشتی یک تابع که چیزی برنگردانده است `undefined` می‌باشد.


## اشیاء و کلاس‌ها

<chapterId>26689f25-8212-5057-8c21-3a05eee0ac75</chapterId>


جاوااسکریپت اغلب یک زبان شیء‌گرا نامیده می‌شود.


این بدان معناست که به شما کمک می‌کند تا کد خود را با گروه‌بندی مقادیر و توابع در کنار هم به صورت **اشیاء** سازماندهی کنید.


### یک `object` چیست؟


یک شیء می‌تواند شامل داده‌ها و توابعی باشد که بر روی آن داده‌ها عمل می‌کنند. وقتی یک تابع درون یک شیء قرار می‌گیرد، می‌گوییم که آن یک `متد` است.


اولین شیئی که دیده‌ایم، شیء `console` بود. این شیء شامل چندین متد است که برای چاپ چیزها روی صفحه و اشکال‌زدایی برنامه‌هایمان استفاده می‌شود.


حتی می‌تواند خودش را چاپ کند؛ شما می‌توانید


```javascript
console.log(console)
```


و لیستی از روش‌هایی که شامل می‌شود را چاپ خواهد کرد. برای مثال، در دستگاه من چاپ شد


```javascript
Object [console] {
log: [Function: log],
warn: [Function: warn],
error: [Function: error],
dir: [Function: dir],
time: [Function: time],
timeEnd: [Function: timeEnd],
timeLog: [Function: timeLog],
trace: [Function: trace],
assert: [Function: assert],
clear: [Function: clear],
count: [Function: count],
countReset: [Function: countReset],
group: [Function: group],
groupEnd: [Function: groupEnd],
table: [Function: table],
debug: [Function: debug],
info: [Function: info],
dirxml: [Function: dirxml],
groupCollapsed: [Function: groupCollapsed],
Console: [Function: Console],
profile: [Function: profile],
profileEnd: [Function: profileEnd],
timeStamp: [Function: timeStamp],
context: [Function: context],

```


همان‌طور که می‌بینید، روش‌های زیادی دارد که می‌توانید برای اشکال‌زدایی استفاده کنید!


جاوااسکریپت روش‌های مختلفی برای ایجاد اشیاء جدید به ما ارائه می‌دهد که می‌توانند هر کاری که می‌خواهیم انجام دهند.


### ایجاد یک شیء


آسان‌ترین راه برای ایجاد یک شیء، گروه‌بندی داده‌ها و توابع با استفاده از **آکولاد** `{}` است.


این چیزی را ایجاد می‌کند که ما آن را **شیء ناشناس** می‌نامیم.


```javascript
const cat = {
name: "Whiskers",
age: 3
}
```


این یک شیء ایجاد می‌کند و آن را در متغیری به نام `cat` ذخیره می‌کند.


شیء دارای دو **ویژگی** است:



- `name` با مقدار `"Whiskers"`
- `age` با مقدار `3`


بیایید آن را چاپ کنیم:


```javascript
console.log(cat)
```


این چاپ می‌کند:


```
{ name: 'Whiskers', age: 3 }
```


می‌توانید ویژگی‌ها را با استفاده از نقطه از شیء دریافت کنید، مانند `objectName.propertyName`:


```javascript
console.log(cat.name)  // prints "Whiskers"
console.log(cat.age)   // prints 3
```


شما همچنین می‌توانید یک ویژگی را **تغییر** دهید:


```javascript
cat.age = 4
console.log(cat.age)  // now it prints 4
```


همان‌طور که می‌بینید، حتی اگر یک شیء به‌عنوان `const` تعریف شده باشد، همچنان می‌توانید داده‌های موجود در آن را تغییر دهید.


در مورد اشیاء، `const` فقط شما را از بازنویسی کل شیء باز می‌دارد:


```javascript
const cat = {
name: "Whiskers",
age: 3
}

cat.age = 5 // this works

cat = 5 // this throws an error, you're trying to reassign the whole object

```


همان‌طور که قبلاً ذکر شد، اشیاء می‌توانند **توابع** را نیز نگه دارند و هنگامی که یک تابع بخشی از یک شیء باشد، آن را **متد** می‌نامیم.


در اینجا یک مثال است:


```javascript
const cat = {
name: "Whiskers",
speak () {
console.log("Meow!")
}
}
```


این شیء دارای:



- یک ویژگی `name`
- یک متد `speak()`


بیایید این روش را صدا بزنیم:


```javascript
cat.speak()
```


چاپ می‌کند:


```
Meow!
```


متدها می‌توانند از داده‌هایی که شیء شامل می‌شود از طریق کلمه کلیدی `this` استفاده کنند.

`this` به شیء جاری اشاره دارد. در این مثال، برای چاپ نام گربه استفاده خواهد شد:


```javascript
const cat = {
name: "Whiskers",
speak () {
console.log(`${this.name} says meow!`)
}
}

cat.speak()
```


این چاپ می‌کند:


```
Whiskers says meow!
```


کلمه `this` به معنای "این شیء" است...در این مورد، شیء `cat`.



این نوع اشیاء عالی هستند وقتی که فقط چیزی سریع و ساده می‌خواهید. اما اگر نیاز دارید **اشیاء زیادی** با ساختار یکسان ایجاد کنید، راه بهتری وجود دارد و اینجاست که **کلاس‌ها** وارد می‌شوند.


### کلاس‌ها و سازنده‌ها


یک **کلاس** مانند یک نقشه است. به جاوااسکریپت می‌گوید چگونه یک نوع خاص از شیء را ایجاد کند.


شما یک کلاس را با استفاده از کلمه کلیدی `class` تعریف می‌کنید، که به دنبال آن نام کلاس و یک بلوک آکولادی `{}` می‌آید.


```javascript
class Dog {}
```


کلاس‌ها معمولاً با یک حرف بزرگ شروع می‌شوند، طبق قرارداد.


می‌توانید یک شیء جدید از یک کلاس با استفاده از `new` ایجاد کنید:


```javascript
const hachiko = new Dog()
```


سعی کنید شیء را چاپ کنید:

```javascript
class Dog {}

const myDog = new Dog()

console.log(myDog)
```


شما دریافت خواهید کرد


```
Dog {}
```


همان‌طور که می‌بینید کلاس Dog خالی است، بنابراین شیء `myDog` نیز خالی است.


ما می‌توانیم با افزودن یک `constructor` تعیین کنیم که اشیاء Dog باید شامل چه ویژگی‌هایی باشند.


سازنده یک تابع ویژه است که زمانی که یک شیء جدید ایجاد (یا "ساخته") می‌شود، اجرا می‌شود.


```javascript
class Dog {
constructor() { }
}
```


ما می‌خواهیم هر سگ یک نام داشته باشد، بنابراین یک پارامتر `name` به تابع اضافه می‌کنیم:


```javascript
class Dog {
constructor(name) { }
}
```


و سپس از `this` استفاده می‌کنیم تا اعلام کنیم که `name` همان `name` شیء `Dog` است که در حال ساخت آن هستیم.


```javascript
class Dog {
constructor(name) {
this.name = name
}}
```


بیایید حالا سعی کنیم از آن استفاده کنیم:


```javascript
class Dog {
constructor(name) {
this.name = name
}
}

const myDog = new Dog("hachiko")

console.log(myDog)
```


این چیزی شبیه به این چاپ می‌کند:


```
Dog { name: 'hachiko' }
```


همان‌طور که می‌بینید، متد `constructor` آرگومان‌هایی را که هنگام ایجاد یک شیء با استفاده از `new Dog()` به کلاس می‌دهید، می‌گیرد و از آن برای ساختن شیء استفاده می‌کند.


بیایید آن را تجزیه کنیم:



- `class Dog` کلاس Dog را تعریف می‌کند.
- `constructor(name)` شیء را هنگام ایجاد تنظیم می‌کند.
- `this.name = name` مقدار را در شیء جدید ذخیره می‌کند.
- `new Dog("hachiko")` یک شیء جدید از کلاس ایجاد می‌کند که ویژگی `name` آن به `"hachiko"` تنظیم شده است.


حالا بیایید یک متد به کلاس خود اضافه کنیم:


```javascript
class Dog {
constructor(name) {
this.name = name
}
speak () {
console.log(`${this.name} says barf!`)
}

}

const myDog = new Dog("hachiko")

myDog.speak()
```


این چاپ خواهد شد


```javascript
hachiko says barf!
```


اگر همین کار را برای دو نمونه مختلف از سگ انجام دهیم



```javascript
class Dog {
constructor(name) {
this.name = name
}
speak () {
console.log(`${this.name} says barf!`)
}

}

const myDog = new Dog("hachiko")

myDog.speak()

const yourDog = new Dog("bobby")

yourDog.speak()
```


ما دریافت می‌کنیم


```
hachiko says barf!
bobby says barf!
```


متد `speak()` از ویژگی `name` شیء `Dog` که بر روی آن فراخوانی شده است، استفاده می‌کند.


این دلیل اصلی وجود کلاس‌ها است: آن‌ها به ما اجازه می‌دهند مجموعه‌ای از متدها را تعریف کنیم که بر روی داده‌ها عمل می‌کنند و سپس چندین شیء ایجاد کنیم که "شکل" داده‌ای یکسانی را به اشتراک می‌گذارند.


وقتی یک متد را روی یکی از این اشیاء فراخوانی می‌کنیم، بر روی داده‌هایی که *آن شیء خاص* نگه می‌دارد، عمل خواهد کرد.


### تغییر شکل یک شیء


اشیاء در جاوااسکریپت انعطاف‌پذیر هستند. حتی پس از ایجاد یک شیء، می‌توانید ویژگی‌های جدیدی اضافه کنید یا ویژگی‌های موجود را حذف کنید.


این مجاز است، اما چیزی است که باید با احتیاط از آن استفاده کنید.


بیایید با کلاس ساده `Dog` خود شروع کنیم:


```javascript
class Dog {
constructor(name) {
this.name = name
}

speak() {
console.log(`${this.name} says barf!`)
}
}

const myDog = new Dog("Fido")
```


در این مرحله، `myDog` تنها یک ویژگی دارد: `name`. ما همچنان می‌توانیم ویژگی‌های جدیدی پس از ایجاد آن اضافه کنیم:


```javascript
myDog.age = 5

console.log(myDog.age) // prints 5
```


ما می‌توانیم یک روش جدید نیز اضافه کنیم:


```javascript
myDog.jump = function () {
console.log(`${this.name} jumps!`)
}

myDog.jump() // Fido jumps!
```


و ما می‌توانیم ویژگی‌ها را نیز با استفاده از کلمه کلیدی `delete` حذف کنیم.


```javascript
delete myDog.name

console.log(myDog.name) // prints 'undefined'
```


این کار می‌کند، اما در اینجا چیزی مهم برای دانستن وجود دارد: موتورهای جاوااسکریپت مانند V8 (که در Node.js و مرورگر Chrome استفاده می‌شود) زمانی سریع‌تر اجرا می‌شوند که اشیاء شما همیشه ویژگی‌های یکسانی را حفظ کنند. اگر پس از ایجاد شیء، ویژگی‌ها را اضافه یا حذف کنید، می‌تواند باعث کندی شود.


در برنامه‌های کوچک، این موضوع اهمیت زیادی ندارد. اما در پروژه‌های بزرگ‌تر (مانند بازی‌ها)، بهتر است که از ابتدا تمام ویژگی‌ها را در سازنده فهرست کنید، حتی اگر بلافاصله از آن‌ها استفاده نکنید. این کار شکل شیء را پایدار نگه می‌دارد و به اجرای سریع‌تر کد شما کمک می‌کند.


به عنوان مثال، به جای این:


```javascript
class Dog {
constructor(name) {
this.name = name
}
}

const dog = new Dog("Rex")
dog.age = 4
dog.breed = "Labrador"
```


شما می‌توانید انجام دهید


```javascript
class Dog {
constructor(name, age, breed) {
this.name = name
this.age = age
this.breed = breed
}
}

const dog = new Dog("Rex", 4, "Labrador")
```


هر دو نسخه کار می‌کنند، اما نسخه دوم برای عملکرد بهتر است. شما از قبل به موتور می‌گویید که هر شیء چه ویژگی‌هایی خواهد داشت و می‌تواند بر این اساس بهینه‌سازی کند.


جاوااسکریپت به شما اجازه می‌دهد تا اشیاء را به‌طور آزاد تغییر شکل دهید، اما هنگام استفاده از کلاس‌ها، بهتر است شکل شیء خود را از قبل برنامه‌ریزی کنید.



### ارث با `extends` و `super()`


گاهی اوقات می‌خواهید کلاسی ایجاد کنید که *تقریباً* مشابه یک کلاس دیگر باشد، اما با چند ویژگی اضافی.


به جای تغییر شکل اشیاء (که همانطور که قبلاً ذکر شد برای عملکرد بهینه نیست)، یا نیاز به نوشتن یک کلاس جدید از ابتدا، جاوااسکریپت به شما اجازه می‌دهد از چیزی به نام **وراثت** استفاده کنید.


ارث به این معناست که یک کلاس می‌تواند **کلاس دیگری را گسترش دهد**. کلاس جدید تمام ویژگی‌ها و متدهای کلاس قدیمی را دریافت می‌کند و شما می‌توانید موارد بیشتری اضافه کنید یا آنچه را که نیاز دارید تغییر دهید.


بیایید بگوییم که یک کلاس پایه به نام `Vehicle` داریم:


```javascript
class Vehicle {
constructor(brand) {
this.brand = brand
}

start() {
console.log(`${this.brand} vehicle is starting...`)
}
}
```


اکنون می‌خواهیم یک کلاس `Car` بسازیم. یک خودرو نوعی وسیله نقلیه است، اما ممکن است بخواهیم ویژگی‌های اضافی داشته باشد یا پیام متفاوتی هنگام شروع نمایش دهد. به جای بازنویسی همه چیز، می‌توانیم از `extends` استفاده کنیم:


```javascript
class Car extends Vehicle {
start() {
console.log(`${this.brand} car is ready to drive!`)
}
}
```


کلاس `Car` اکنون **همه چیز** را از `Vehicle` به ارث می‌برد. این کلاس ویژگی `brand` را دریافت می‌کند و ما متد `start()` را با نسخه خودمان جایگزین کرده‌ایم.


![](assets/en/4.webp)


بیایید امتحان کنیم:


```javascript
const myCar = new Car("Toyota")
myCar.start()
```


این چاپ می‌شود:


```
Toyota car is ready to drive!
```


حتی اگر `Car` سازنده‌ی خود را نداشته باشد، همچنان از سازنده‌ی `Vehicle` استفاده می‌کند. اما اگر بخواهیم یک سازنده‌ی سفارشی در `Car` بنویسیم، می‌توانیم، فقط باید یک فراخوانی به سازنده‌ی والد آن با استفاده از `super()` اضافه کنیم.


در اینجا نحوه انجام آن آمده است:


```javascript
class Vehicle {
constructor(brand) {
this.brand = brand
}

start() {
console.log(`${this.brand} vehicle is starting...`)
}

}

class Car extends Vehicle {
constructor(brand, model) {
super(brand) // call the parent constructor and passes the brand argument to it
this.model = model
}

start() {
console.log(`${this.brand} ${this.model} is ready to drive!`)
}
}

const myCar = new Car("Toyota", "Corolla")
myCar.start()
```


![](assets/en/5.webp)



این چاپ می‌کند:


```
Toyota Corolla is ready to drive!
```


بنابراین برای خلاصه کردن



- `extends` به این معناست که یک کلاس بر اساس کلاس دیگری است.
- `super()` برای فراخوانی سازنده کلاسی که در حال گسترش آن هستید استفاده می‌شود.
- کلاس جدید تمام ویژگی‌ها و متدهای کلاس اصلی را دریافت می‌کند.
- می‌توانید متدها (مانند `start()`) را **بازنویسی** کنید تا کاری متفاوت انجام دهند.


این زمانی مفید است که چندین چیز مشابه دارید (مانند ماشین‌ها، کامیون‌ها و دوچرخه‌ها) و می‌خواهید آن‌ها کد را به اشتراک بگذارند اما همچنان به روش خودشان عمل کنند.


### `instanceof`


کلمه کلیدی `instanceof` بررسی می‌کند که آیا یک شیء از یک کلاس خاص ایجاد شده است یا خیر.


بیایید بگوییم کلاسی به نام `User` داریم:


```javascript
class User {
constructor(username) {
this.username = username
}
}

const regularUser = new User("julia123")

console.log(regularUser instanceof User)
```


این چاپ می‌کند:


```
true
```


تأیید اینکه `regularUser` یک `User` است. این به این دلیل است که `regularUser` با استفاده از کلاس `User` ایجاد شده است.


همچنین با کلاس‌های **ارث‌بری شده** کار می‌کند. به عنوان مثال، در اینجا یک کلاس `Admin` وجود دارد که از `User` ارث می‌برد:


```javascript
class Admin extends User {}

const ourAdmin = new Admin("admin42")

console.log(ourAdmin instanceof Admin)   // true
console.log(ourAdmin instanceof User)    // true
```


هر دو خط `true` را برمی‌گردانند. این به این دلیل است که `Admin` یک زیرکلاس از `User` است، بنابراین `ourAdmin` هم یک `Admin` و هم یک `User` است.

## مدیریت خطا

<chapterId>d0206bc5-d386-5e7f-9917-5803f392448c</chapterId>


همانطور که برنامه‌های پیچیده‌تری با جاوااسکریپت می‌نویسید، با **خطاها** مواجه خواهید شد. این‌ها موقعیت‌های غیرمنتظره‌ای هستند که در آن‌ها چیزی اشتباه پیش می‌رود. شاید یک متغیر `undefined` باشد اما شما سعی کنید از آن استفاده کنید، یا کدی ورودی با نوع اشتباه دریافت کند.


اگر این خطاها را به‌درستی مدیریت نکنیم، ممکن است برنامه ما خراب شود یا به‌طور غیرقابل پیش‌بینی رفتار کند. جاوااسکریپت ابزارهایی برای شناسایی و مدیریت این خطاها فراهم می‌کند تا بتوانیم آن‌ها را به‌صورت بهتری مدیریت کنیم.


### خطای رایج: دسترسی به یک مقدار در `undefined`


در اینجا یک وضعیت رایج که باعث خطا می‌شود:


```javascript
const user = undefined
console.log(user.name)
```


اگر این کد را اجرا کنید، با خطایی مواجه خواهید شد که به این شکل است:


```
TypeError: Cannot read properties of undefined (reading 'name')
```


این جاوااسکریپت به شما می‌گوید: «هی، شما سعی کردید ویژگی `name` را از چیزی که `undefined` است بگیرید، و این منطقی نیست.» و همان‌طور که می‌بینید، وقتی این نوع خطا رخ می‌دهد، برنامه متوقف می‌شود مگر اینکه به‌طور خاص کدی نوشته باشید که آن را بگیرد و مدیریت کند.


### `پرتاب` یک خطا


گاهی اوقات می‌خواهید به صورت دستی **یک خطا ایجاد کنید** در کد خود. در این صورت، از کلمه کلیدی `throw` استفاده می‌کنید.


```javascript
throw new Error("This is a custom error message")
```


این بلافاصله برنامه را متوقف کرده و چاپ می‌کند:


```
Uncaught Error: This is a custom error message
```


می‌توانید از `throw` برای اعمال قوانین در برنامه خود استفاده کنید. به عنوان مثال:


```javascript
function divide(a, b) {
if (b === 0) {
throw new Error("You can't divide by zero")
}
return a / b
}

console.log(divide(10, 2))  // OK: prints 5
console.log(divide(10, 0))  // Error!
```


تماس دوم باعث خطا می‌شود زیرا تقسیم بر صفر در این مثال مجاز نیست.


### گرفتن خطاها با `try...catch`


اگر نمی‌خواهید برنامه شما هنگام وقوع خطا متوقف شود، می‌توانید خطا را با استفاده از بلوک `try...catch` بگیرید. این کار زمانی مفید است که می‌خواهید برنامه شما **به کار خود ادامه دهد** حتی اگر چیزی خراب شود.


```javascript
try {
const user = undefined
console.log(user.name)
console.log("End of the block") // this will never get printed
} catch (error) {
console.log("Oops! Something went wrong.")
}
```


خروجی:


```
Oops! Something went wrong.
```


در اینجا نحوه کار آن آمده است:



- کد داخل بلوک `try` ابتدا امتحان می‌شود.
- اگر خطایی رخ دهد، جاوااسکریپت **به بلوک `catch` می‌پرد** و بقیه بلوک `try` را نادیده می‌گیرد.
- بلوک `catch` خطا را دریافت می‌کند، بنابراین می‌توانید آن را چاپ کنید یا به روش دیگری مانند مثال زیر مدیریت کنید:


```javascript
try {
const user = undefined
console.log(user.name)
console.log("End of the block") // this will never get printed
} catch (error) {
console.log(`The message of the error was: "${error.message}"`)
}
```


خروجی:


```
The message of the error was: "Cannot read properties of undefined (reading 'name')"
```


### بلوک `finally`


شما همچنین می‌توانید یک بلوک `finally` اضافه کنید. این کدی است که **همیشه اجرا می‌شود**، چه خطایی رخ داده باشد یا نه.


```javascript
try {
console.log("Trying something risky...")
throw new Error("Uh oh!")
} catch (error) {
console.log("Caught the error:", error.message)
} finally {
console.log("This will run no matter what.")
}
```


خروجی:


```
Trying something risky...
Caught the error: Uh oh!
This will run no matter what.
```


## اجتناب از باگ‌ها

<chapterId>db12d9f6-5806-514c-998e-0ae24805104e</chapterId>


این فصل برخی از رایج‌ترین مشکلات در جاوااسکریپت و نحوه اجتناب از آن‌ها را نشان می‌دهد.


### `var` و Assignment بدون اعلان


در کدهای قدیمی‌تر جاوااسکریپت، متغیرها اغلب با استفاده از کلمه کلیدی `var` اعلام می‌شدند. برخلاف `let` و `const` که قبلاً درباره آن‌ها یاد گرفته‌اید، `var` ممکن است به روش‌های گیج‌کننده‌ای رفتار کند.


برای مثال:


```javascript
{
var message = "hello"
}
console.log(message)
```


ممکن است انتظار داشته باشید که `message` فقط در داخل بلوک وجود داشته باشد، اما اینطور نیست. `var` به محدوده بلوک توجهی نمی‌کند و متغیر را در کل تابع یا فایل در دسترس قرار می‌دهد.


این می‌تواند به رفتار غیرمنتظره منجر شود، به‌ویژه در برنامه‌های بزرگ‌تر. به همین دلیل، کد مدرن جاوااسکریپت باید همیشه از `let` یا `const` به جای `var` استفاده کند.


حتی بدتر، جاوااسکریپت به شما اجازه می‌دهد مقادیر را به متغیرها **بدون اعلام آن‌ها به‌طور کامل** اختصاص دهید:


```javascript
function greet() {
user = "Alice"
}
greet()
console.log(user) // prints "Alice"
```


این یک متغیر سراسری جدید به نام `name` بدون هیچ اعلامی ایجاد می‌کند. این می‌تواند به صورت خاموش اتفاق بیفتد و منجر به اشکالاتی شود که پیگیری آن‌ها Hard است، به ویژه اگر فقط یک اشتباه تایپی باشد. همیشه متغیرها را با استفاده از `let` یا `const` اعلام کنید.


### سیستم نوع ضعیف


جاوااسکریپت به صورت ضعیف تایپ شده است، به این معنی که در صورت نیاز به‌طور خودکار مقادیر را از یک نوع به نوع دیگر تبدیل می‌کند. این فرآیند به عنوان اجبار نوع شناخته می‌شود و در حالی که می‌تواند مفید باشد، اغلب به نتایج گیج‌کننده‌ای منجر می‌شود.


برای مثال:


```javascript
console.log("5" + 1)    // "51"
console.log("5" - 1)    // 4
console.log(true + 1)   // 2
console.log(null + 1)   // 1
```


در این مثال‌ها، جاوااسکریپت سعی می‌کند حدس بزند که منظور شما چه بوده است. گاهی اوقات رشته‌ها را به اعداد تبدیل می‌کند، یا بولین‌ها را به اعداد، یا رشته‌ها را به رشته‌های دیگر. این می‌تواند باعث شود کد شما به روش‌های غیرمنتظره‌ای رفتار کند.


آگاهی از سیستم نوع‌دهی ضعیف جاوااسکریپت مهم است. وقتی چیزها شروع به رفتار عجیب می‌کنند، ممکن است به دلیل تبدیل نوع غیرمنتظره باشد.


### `"use strict"`


می‌توانید حالت سخت‌گیرانه‌تری را فعال کنید که برخی از خطاهای خاموش را به خطاهای واقعی تبدیل می‌کند و شما را از استفاده از برخی ویژگی‌های خطرناک‌تر زبان بازمی‌دارد.


برای فعال کردن این حالت سخت‌گیرانه‌تر، این خط را در بالای فایل یا تابع خود اضافه کنید:


```javascript
"use strict"
```


برای مثال:


```javascript
"use strict"
name = "Alice" // ReferenceError: name is not defined
```


بدون حالت سخت‌گیرانه، جاوااسکریپت به‌طور بی‌صدا یک متغیر سراسری به نام `name` ایجاد می‌کند. اما با حالت سخت‌گیرانه، این به یک خطای واقعی تبدیل می‌شود و به شما کمک می‌کند تا زودتر باگ‌ها را شناسایی کنید.


حالت سخت همچنین برخی از ویژگی‌های قدیمی جاوااسکریپت را غیرفعال می‌کند و کد شما را برای بهینه‌سازی و نگهداری آسان‌تر می‌سازد.


## مقدار در مقابل مرجع

<chapterId>bb898425-dc2f-5e5c-864b-0cb7a4a9aea9</chapterId>


جاوااسکریپت با انواع مختلف مقادیر به روش‌های متفاوتی برخورد می‌کند.


برخی مقادیر هنگام اختصاص به یک متغیر **کپی** می‌شوند.


دیگران زمانی **به اشتراک گذاشته می‌شوند** که آن‌ها را به یک متغیر جدید اختصاص دهید، بنابراین اگر یکی را تغییر دهید، دیگری نیز تغییر می‌کند.


### عبور با مقدار


وقتی یک مقدار **با مقدار** منتقل می‌شود، جاوااسکریپت یک **کپی** از آن می‌سازد.


بنابراین اگر یکی را تغییر دهید، بر دیگری تأثیری نمی‌گذارد.


این اتفاق با انواع اولیه می‌افتد، مانند:



- اعداد
- رشته‌ها
- بولین‌ها (`true` و `false`)
- `null`
- `تعریف‌نشده`


بیایید به یک مثال نگاه کنیم:


```javascript
let a = 5
let b = a

b = 10

console.log(a) // 5
console.log(b) // 10
```


ما به `b` مقدار `a` را دادیم، اما سپس `b` را به `10` تغییر دادیم.


از آنجا که اعداد به صورت مقداری منتقل می‌شوند، جاوااسکریپت عدد `5` را به `b` کپی کرد. عدد `5` در `b` مستقل از عدد اصلی `5` در `a` است، بنابراین تغییر مقدار `b` تأثیری بر `a` نداشت.


بیایید با یک رشته امتحان کنیم:


```javascript
let name1 = "Alice"
let name2 = name1

name2 = "Bob"

console.log(name1) // Alice
console.log(name2) // Bob
```


باز هم، تغییر `name2` بر `name1` تأثیری نداشت، زیرا رشته‌ها نیز با مقدار منتقل می‌شوند.


همین اتفاق زمانی می‌افتد که یک مقدار اولیه را به یک تابع ارسال می‌کنید: آن کپی می‌شود. بنابراین تابع نمی‌تواند اصل را تغییر دهد.


```javascript
function plusOne(x) {
x = x + 1
}

let number = 5

plusOne(number)

console.log(number) // 5
```


حتی با اینکه داخل تابع `1` به `x` اضافه شد، مقدار اصلی `number` تغییر نکرد.


این به این دلیل است که تنها یک **کپی** به تابع ارسال شد.


### ارجاع با مرجع


اشیاء **با ارجاع** منتقل می‌شوند.


این بدان معناست که به جای کپی کردن آن‌ها، جاوااسکریپت فقط یک **ارجاع** به آن می‌دهد و اگر آن را تغییر دهید، تمام متغیرهای دیگری که به آن اشاره می‌کنند نیز تغییر را مشاهده خواهند کرد.


برای مثال:


```javascript
const person1 = { name: "Alice" }
const person2 = person1

person2.name = "Bob"

console.log(person1.name) // Bob
console.log(person2.name) // Bob
```


هر دو `person1` و `person2` به یک شیء در حافظه اشاره می‌کنند.


بنابراین وقتی `person2.name` را تغییر دادیم، `person1.name` را نیز تغییر دادیم، زیرا هر دو به یک چیز نگاه می‌کنند.


آرایه‌ها اشیاء هستند، بنابراین بیایید همین کار را با یک آرایه امتحان کنیم:


```javascript
const list1 = [1, 2, 3]
const list2 = list1

list2.push(4)

console.log(list1) // [1, 2, 3, 4]
console.log(list2) // [1, 2, 3, 4]
```


ما `4` را به `list2` اضافه کردیم، اما `list1` نیز تحت تأثیر قرار گرفت، زیرا هر دو به یک آرایه اشاره می‌کنند.


بیایید ببینیم چه اتفاقی می‌افتد وقتی یک شیء را به یک تابع ارسال می‌کنیم:


```javascript
function rename(user) {
user.name = "Charlie"
}

const person = { name: "Dana" }

rename(person)

console.log(person.name) // Charlie
```


تابع شیء را تغییر داد! این به این دلیل است که یک **ارجاع** به شیء اصلی `person` دریافت کرد.


نسخه‌ای دریافت نکرد. به شیء اصلی دسترسی پیدا کرد و با آن توانایی تغییر آن را به دست آورد.


مهم است که این تمایز را به خاطر بسپاریم، زیرا در غیر این صورت کد ما ممکن است به شکلی متفاوت از آنچه انتظار داریم رفتار کند. به عنوان مثال، ممکن است تابعی بنویسیم با این انتظار که آرگومان‌هایی که دریافت می‌کند را تغییر ندهد، و بعداً متوجه شویم که در واقع آن‌ها را تغییر می‌داده است (زیرا آن‌ها اشیاء بودند و به صورت ارجاعی منتقل شده بودند).


## کار با توابع

<chapterId>e0d277a8-c642-5af7-9e53-dee27c811967</chapterId>


شما قبلاً یاد گرفته‌اید که چگونه توابع را در جاوااسکریپت اعلام و استفاده کنید. اما جاوااسکریپت ابزارهای بیشتری برای کار با توابع به روش‌های قدرتمند به شما می‌دهد.


### توابع پیکانی


توابع پیکانی راهی کوتاه‌تر برای نوشتن توابع هستند. به جای استفاده از کلمه کلیدی `function`، از یک پیکان (`=>`) استفاده می‌کنیم.


در اینجا یک تابع معمولی وجود دارد:


```javascript
function greet(name) {
return `Hello, ${name}!`
}
```


نسخه فلش به این شکل است:


```javascript
const greet = (name) => {
return `Hello, ${name}!`
}
```


اگر تابع **فقط یک خط** داشته باشد، می‌توانید آکولادها و `return` را حذف کنید:


```javascript
const greet = (name) => `Hello, ${name}!`
```


اگر تابع **فقط یک پارامتر** داشته باشد، حتی می‌توانید پرانتزهای اطراف پارامترها را حذف کنید:


```javascript
const greet = name => `Hello, ${name}!`
```


توابع پیکانی در جاوااسکریپت مدرن بسیار رایج هستند، زیرا امکان بیان توابع ساده با کد اضافی به‌طور قابل‌توجهی کمتر را فراهم می‌کنند.


### پارامترهای پیش‌فرض


گاهی اوقات می‌خواهید یک تابع دارای یک **مقدار پیش‌فرض** باشد اگر هیچ آرگومانی ارسال نشود.


شما می‌توانید این کار را به این صورت انجام دهید:


```javascript
function sayHello(name = "friend") {
console.log(`Hello, ${name}!`)
}

sayHello("Alice") // Hello, Alice!
sayHello()        // Hello, friend!
```


مقدار پیش‌فرض `"friend"` زمانی استفاده می‌شود که هیچ چیزی وارد نشود.


### پارامترهای گسترش (`...`)


اگر تابع شما تعداد متغیری از آرگومان‌ها را بپذیرد چه می‌شود؟


می‌توانید از **عملگر گسترش** (`...`) برای جمع‌آوری آن‌ها در یک آرایه استفاده کنید:


```javascript
function logAll(...items) {
console.log(items)
}

logAll(1, 2, 3) // [1, 2, 3]
logAll("a", "b") // ["a", "b"]
```


سپس می‌توانید از یک حلقه برای پردازش هر آیتم استفاده کنید:


```javascript
function logEach(...items) {
for (const item of items) {
console.log(item)
}
}
```


عملگر گسترش زمانی مفید است که نمی‌دانید چند آرگومان ارسال خواهد شد.


### توابع مرتبه بالاتر


یک **تابع مرتبه بالاتر** تابعی است که:



- یک تابع دیگر را به عنوان ورودی می‌گیرد
- و/یا یک تابع را به عنوان خروجی برمی‌گرداند


در اینجا یک مثال ساده آورده شده است:


```javascript
function runTwice(action) {
action()
action()
}

function sayHello(name = "friend") {
console.log(`Hello, ${name}!`)
}

runTwice(sayHello)
```


این چاپ می‌کند:


```
Hello, friend!
Hello, friend!
```


می‌توانیم یک تابع پیکان را به آن ارسال کنیم:


```javascript
runTwice(
() => console.log("Hello!")
)
```


این چاپ می‌کند:


```
Hello!
Hello!
```


شما همچنین می‌توانید توابعی بنویسید که **توابع دیگری را بازگردانند**:


```javascript
function makeGreeter(name) {
return () => console.log(`Hi, ${name}`)
}

const greetAlice = makeGreeter("Alice")
const greetBob = makeGreeter("Bob")

greetAlice() // Hi, Alice
greetBob() // Hi, Bob
```


تابع `makeGreeter` تابعی است که توابع دیگری را می‌سازد. این تابع یک رشته دریافت می‌کند و تابع جدیدی را برمی‌گرداند که از آن رشته در فراخوانی `console.log` خود استفاده می‌کند.


این نوع الگو بسیار قدرتمند است، زیرا به شما اجازه می‌دهد "حفره‌هایی" در توابع خود بگذارید که می‌توانید بعداً با رفتار مورد نیاز خود پر کنید.


### `map()`, `filter()`, `reduce()`


جاوااسکریپت برخی از متدهای داخلی مفید را برای استفاده با آرایه‌ها به شما ارائه می‌دهد.


این روش‌ها توابع را به عنوان آرگومان می‌پذیرند، بنابراین آن‌ها نیز توابع مرتبه بالاتر هستند.


`map()` هر آیتم در یک آرایه را به چیز دیگری تبدیل می‌کند.


مثال:


```javascript
const numbers = [1, 2, 3]
const doubled = numbers.map(x => x * 2)

console.log(doubled) // [2, 4, 6]
```


هر عدد دو برابر می‌شود و نتیجه یک آرایه جدید است.


`filter()` آیتم‌ها را از آرایه حذف می‌کند اگر آن‌ها از یک آزمون عبور نکنند.


مثال:


```javascript
const numbers = [1, 2, 3, 4, 5]
const numbersGreaterThanTwo = numbers.filter(x => x > 2)

console.log(numbersGreaterThanTwo) // [3, 4, 5]
```


فقط ورودی‌های آرایه که شرط `x > 2` برای آن‌ها `true` برمی‌گردد، نگه داشته می‌شوند.


تابع `reduce()` برای ترکیب تمام آیتم‌های یک آرایه به یک مقدار واحد استفاده می‌شود.


مثال:


```javascript
const numbers = [1, 2, 3, 4]
const total = numbers.reduce((current, next) => current + next)

console.log(total) // 10
```


`reduce` از طریق آرایه عبور می‌کند و در این مثال، عملگر `+` را بین `1` و `2` اعمال می‌کند، سپس بین نتیجه `3` و `3`، سپس بین نتیجه `6` و `4`، تا زمانی که مجموع تمام ورودی‌های آرایه (که 10 است) را به دست آورد.


می‌توانید از `reduce()` برای کارهای زیادی مانند محاسبه مجموع، میانگین یا ساختن مقادیر جدید به صورت مرحله به مرحله استفاده کنید.


این روش‌ها (`map`, `filter`, `reduce`) ابزارهای قدرتمندی برای پردازش داده‌ها بدون نوشتن حلقه‌های دستی هستند.


شما حتی می‌توانید آن‌ها را در یک زنجیره عملیات ترکیب کنید، به این صورت:


```javascript
const numbers = [1, 2, 3, 4, 5]

const result = numbers
.map(n => n * 2)        // Double each entry, obtain [2, 4, 6, 8, 10]
.filter(n => n > 3) // Keep only the entries bigger than 3, so you get [4, 6, 8, 10]
.reduce((n1, n2) => n1 + n2) // Adds them: 4 + 6 + 8 + 10 = 28

console.log(result) // 28
```


## کار با اشیاء

<chapterId>7842aada-f009-5518-b8e3-1104e166a035</chapterId>


در این فصل، ما با برخی از ابزارهای قدرتمند و کمی پیشرفته‌تر برای کار با اشیاء در جاوااسکریپت آشنا خواهیم شد.


### املاک خصوصی


گاهی اوقات، می‌خواهیم یک ویژگی از یک شیء را پنهان کنیم تا نتوان آن را از خارج از شیء تغییر داد یا به آن دسترسی پیدا کرد. جاوااسکریپت به ما روشی برای انجام این کار می‌دهد که با استفاده از `#` قبل از نام ویژگی است. این کار یک ویژگی **خصوصی** ایجاد می‌کند که فقط از داخل کلاس قابل دسترسی است.


```javascript
class Person {
#age // this is a private property

constructor(name, age) {
this.name = name
this.#age = age
}

getAge() {
return this.#age
}
}

const alice = new Person("Alice", 30)
console.log(alice.name)      // Alice
console.log(alice.getAge())  // 30, the method can access the private property
console.log(alice.#age)      // ❌ Error! You can't access private properties directly
```


خصوصیات خصوصی زمانی که می‌خواهید از تغییرات تصادفی جلوگیری کنید، مفید هستند.


### ویژگی‌های `static`


گاهی اوقات، می‌خواهید یک ویژگی به خود کلاس تعلق داشته باشد، نه به هر شیئی که از آن کلاس ایجاد می‌کنید. این همان چیزی است که `static` برای آن استفاده می‌شود. یک ویژگی `static` در کلاس قرار دارد و همه اشیاء آن کلاس به آن ارجاع خواهند داد.


```javascript
class User {
static counter = 0 // this belongs to the class, not to instances. The same counter will be shared by all objects

constructor() {
User.counter++ // changes the static property every time an object of this class gets initiated
}
}

const a = new User() // the constructor will change the shared counter from 0 to 1
const b = new User() // the constructor will change the shared counter from 1 to 2

console.log(User.counter) //  prints 2
```


این برای ذخیره داده‌ها و روش‌های مشترک که به کل گروهی از اشیاء اعمال می‌شود، نه فقط یکی، مفید است.


### `get` و `set`


در جاوااسکریپت، `get` و `set` به شما اجازه می‌دهند تا ویژگی‌هایی بسازید که *شبیه* متغیرهای معمولی به نظر می‌رسند، اما در واقع کد خاصی را در پس‌زمینه اجرا می‌کنند.


یک متد `get`ter زمانی اجرا می‌شود که شما سعی می‌کنید یک ویژگی را *بخوانید*. این متد با نوشتن `get` قبل از یک متد با نام ویژگی اعلام می‌شود.


```javascript
class User {
constructor(firstName, lastName) {
this.firstName = firstName
this.lastName = lastName
}

get fullName() {
return `${this.firstName} ${this.lastName}`
}
}

const user = new User("Jane", "Doe")
console.log(user.fullName) // Jane Doe
```


حتی با اینکه `fullName` یک ویژگی معمولی نیست، می‌توانیم از آن مانند یک ویژگی معمولی استفاده کنیم و در پشت صحنه، تابع `get` اجرا می‌شود تا نام کامل را بسازد.


یک متد `set`ter زمانی اجرا می‌شود که شما یک مقدار را به یک ویژگی *اختصاص* می‌دهید. این متد به شما اجازه می‌دهد کنترل کنید چه اتفاقی می‌افتد زمانی که کسی سعی می‌کند آن مقدار را تغییر دهد. این متد با نوشتن `set` قبل از یک متد با نام ویژگی اعلام می‌شود.


```javascript
class User {
constructor() {
this.firstName = "John"
this.lastName = "Doe"
}

get fullName() {
return `${this.firstName} ${this.lastName}`
}

set fullName(input) {            // gets the name that is passed
const parts = input.split(" ") // breaks it into parts
this.firstName = parts[0]      // uses the first part as first name
this.lastName = parts[1]       // uses the second part as last name
}
}

const user = new User()
user.fullName = "John Smith"

console.log(user.firstName) // John
console.log(user.lastName)  // Smith
```


وقتی که ما `user.fullName = "John Smith"` را انجام می‌دهیم، متد `set` اجرا می‌شود و مقادیر `firstName` و `lastName` را به‌روزرسانی می‌کند.


بنابراین حتی اگر احساس کنیم که فقط یک متغیر ساده را تنظیم می‌کنیم، در واقع منطقی را فعال می‌کنیم که ویژگی‌های دیگر را به‌روزرسانی می‌کند.


## کلیدها و مقادیر

<chapterId>01a397b8-c12a-5c39-82b3-6d9ebbb72a29</chapterId>


هر ویژگی در یک شیء جاوااسکریپت دارای یک **کلید** (که به آن نام ویژگی نیز گفته می‌شود) و یک **مقدار** است.


برای مثال:


```javascript
const user = {
name: "Alice",
age: 30
}
```


در این شیء، `"name"` و `"age"` کلیدها هستند، و "Alice" و `30` مقادیر آنها می‌باشند.


### دسترسی پویا


گاهی اوقات، شما نام یک ویژگی را از قبل نمی‌دانید... شاید آن را از ورودی کاربر دریافت می‌کنید، یا از یک متغیر می‌خوانید. شما همچنان می‌توانید با استفاده از **نشانه‌گذاری براکت** به آن دسترسی پیدا کنید، مانند `myObject["keyName"]`.


```javascript
const user = {
name: "Alice",
age: 30
}

console.log(user["name"]) // Alice
```


ما رشته `name` را به شیء منتقل کردیم تا مقدار مربوطه را دریافت کنیم.


ما می‌توانیم یک کلید را در یک متغیر ذخیره کنیم و از آن برای دسترسی به مقدار مربوطه در آینده استفاده کنیم، مانند


```javascript
const user = {
name: "Alice",
age: 30
}

const key = "name"

console.log(user[key]) // Alice
```


### داینامیک Assignment


شما همچنین می‌توانید با استفاده از متغیرها به عنوان کلید، ویژگی‌های شیء را ایجاد یا به‌روزرسانی کنید.


```javascript
const settings = {}

const key = "theme"
settings[key] = "dark"

console.log(settings) // { theme: "dark" }
```


این زمانی مفید است که می‌خواهید یک شیء را مرحله به مرحله بسازید. برای مثال:


```javascript
const user = {}

user["username"] = "bananaFan123"
user["email"] = "banana@fruit.com"

console.log(user)
// { username: "bananaFan123", email: "banana@fruit.com" }
```


شما حتی می‌توانید از یک کلید پویا *در حین ایجاد* شیء با استفاده از کروشه‌ها استفاده کنید:


```javascript
const key = "language"
const config = {
[key]: "JavaScript"
}

console.log(config.language) // JavaScript
```


این به عنوان یک **ویژگی محاسبه‌شده** شناخته می‌شود. مقدار داخل کروشه‌ها ارزیابی می‌شود و نتیجه به عنوان کلید استفاده می‌شود.


### نوع `Symbol`


علاوه بر رشته‌ها، جاوااسکریپت به شما اجازه می‌دهد از نوع خاصی به نام `Symbol` به عنوان کلید شیء استفاده کنید.


بیایید با یک مثال ساده شروع کنیم:


```javascript
const id = Symbol("userID")

const user = {
name: "Bob",
[id]: 12345
}

console.log(user[id]) // 12345
```


در این مثال، `id` یک Symbol است. این یک رشته نیست، اما همچنان به عنوان یک کلید عمل می‌کند. اگر سعی کنید `user` را در کنسول لاگ کنید، این را خواهید دید:


```javascript
console.log(user) // { name: 'Bob', [Symbol(userID)]: 12345 }
```


نکته مهم دیگر: هر نمادی که ایجاد می‌کنید منحصر به فرد است، حتی اگر با استفاده از رشته یکسانی ایجاد شده باشند.


```javascript
const a = Symbol("label")
const b = Symbol("label")

console.log(a === b) // false
```


نمادها به شما اجازه می‌دهند کلیدهایی تعریف کنید که با کلیدهای معمولی تداخل نداشته باشند. برای مثال، فرض کنید شما یک شیء با یک ویژگی `name` دارید، اما این شیء در آینده توسط یک کاربر به روش‌هایی که نمی‌توانید پیش‌بینی کنید، قابل سفارشی‌سازی خواهد بود و ممکن است آن کاربر نیز یک ویژگی `name` اضافه کند. اگر ویژگی اصلی `name` با یک رشته تعریف شده بود، توسط ویژگی جدید بازنویسی می‌شد، به این صورت:


```javascript
const obj = {
name: "John"
}

obj.name = "Jimmy"

console.log(obj) // { name: 'Jimmy' }
```


اگر به جای آن از یک نماد استفاده کنیم:


```javascript
const name = Symbol("name")

const obj = {
[name]: "John"
}

obj.name = "Jimmy"

console.log(obj) // { name: 'Jimmy', [Symbol(name)]: 'John' }
```


همان‌طور که می‌بینید، ویژگی اصلی `name` به نوعی به این شکل حفظ می‌شود. این می‌تواند در برخی موارد خاص مفید باشد.


## اشیاء کاربردی

<chapterId>516e74c8-2a11-545a-a4d1-c2cabb91a273</chapterId>


جاوااسکریپت به ما برخی از اشیاء داخلی مفید را می‌دهد که به ما در انجام کارهایی مانند اشکال‌زدایی و عملیات ریاضی کمک می‌کنند.


### سایر روش‌های `console`


شما قبلاً `console.log` را دیده‌اید که مقادیر را به صفحه نمایش چاپ می‌کند.


روش‌های مفید دیگری نیز در شیء `console` وجود دارند که می‌توانند به شما در اشکال‌زدایی برنامه‌هایتان کمک کنند.


#### `console.warn`


پیامی را به رنگ زرد چاپ می‌کند (یا با یک آیکون هشدار در برخی محیط‌ها):


```javascript
console.warn("This is just a warning.")
```


#### `console.error`


پیامی را به رنگ قرمز چاپ می‌کند، مانند یک خطا:


```javascript
console.error("Something went wrong!")
```


#### `console.table`


آرایه یا شیء را به صورت جدول نمایش می‌دهد:


```javascript
const users = [
{ name: "Alice", age: 25 },
{ name: "Bob", age: 30 }
]

console.table(users)
```


این یک جدول مانند زیر چاپ می‌کند:


```
┌─────────┬────────┬─────┐
│ (index) │  name  │ age │
├─────────┼────────┼─────┤
│    0    │ 'Alice'│  25 │
│    1    │ 'Bob'  │  30 │
└─────────┴────────┴─────┘
```


این می‌تواند برای تجسم داده‌های ساختاریافته مفید باشد.


#### `console.time` و `console.timeEnd`


می‌توانید مدت زمان انجام چیزی را اندازه‌گیری کنید:


```javascript
console.time("timer")
for (let i = 0; i < 1000000; i++) {}
console.timeEnd("timer")
```


این چیزی شبیه به این چاپ می‌کند:


```
timer: 2.379ms
```


مفید برای برخی آزمایش‌های ساده عملکرد.


### شیء `Math`


جاوااسکریپت یک شیء `Math` با روش‌های مفید برای انجام محاسبات به شما می‌دهد.


#### `Math.random()`


یک عدد تصادفی بین 0 (شامل) و 1 (غیرشامل) برمی‌گرداند:


```javascript
const r = Math.random()
console.log(r)
```


مثال خروجی:


```
0.4387429859
```


#### `Math.floor()` و `Math.ceil()`



- `Math.floor(n)` به نزدیک‌ترین عدد صحیح **پایین** گرد می‌کند.
- `Math.ceil(n)` به نزدیک‌ترین عدد صحیح **بالا** گرد می‌کند.


```javascript
console.log(Math.floor(4.9)) // 4
console.log(Math.ceil(4.1))  // 5
```


#### `Math.round()`


به نزدیک‌ترین عدد صحیح گرد کنید:


```javascript
console.log(Math.round(4.4)) // 4
console.log(Math.round(4.6)) // 5
```


#### `Math.max()` و `Math.min()`


بزرگترین یا کوچکترین مقدار را از یک لیست اعداد برمی‌گرداند:


```javascript
console.log(Math.max(5, 9, 3)) // 9
console.log(Math.min(5, 9, 3)) // 3
```


#### `Math.pow()` و `Math.sqrt()`



- `Math.pow(a, b)` به شما `a` به توان `b` را می‌دهد.
- `Math.sqrt(n)` به شما جذر `n` را می‌دهد


```javascript
console.log(Math.pow(2, 3))   // 8
console.log(Math.sqrt(16))    // 4
```


# جاوااسکریپت پیشرفته

## مجموعه‌های دیگر

<chapterId>a9a70c6d-a343-5a46-a383-e288bc2700e3</chapterId>


جاوااسکریپت به ما انواع مجموعه‌های خاصی را ارائه می‌دهد که فراتر از آرایه‌ها و اشیاء معمولی هستند. این‌ها شامل `Map` و `Set` می‌باشند.


آن‌ها به شما کمک می‌کنند تا گروه‌هایی از مقادیر را ذخیره و مدیریت کنید، اما به شکلی متفاوت از آنچه تاکنون دیده‌اید عمل می‌کنند.


یک `Map` مجموعه‌ای از **جفت‌های کلید-مقدار** است، درست مانند یک شیء. اما تفاوت‌های مهمی دارد:



- کلیدها می‌توانند **هر مقداری** باشند، نه فقط رشته‌ها.
- ترتیب اقلام حفظ می‌شود.
- این دارای روش‌های داخلی است تا کار با آن را آسان‌تر کند.


شما یک نقشه جدید به این صورت ایجاد می‌کنید:


```javascript
const myMap = new Map()
```


این یک نقشه خالی ایجاد می‌کند. برای اضافه کردن ورودی‌ها به آن، از `myMap.set(key, value)` استفاده کنید:


```javascript
myMap.set("name", "Alice")
```


این یک کلید `"name"` با مقدار `"Alice"` اضافه می‌کند.


شما همچنین می‌توانید از یک عدد به عنوان کلید استفاده کنید:


```javascript
myMap.set(42, "The answer")
```


و حتی یک شیء به عنوان کلید:


```javascript
const objKey = { id: 1 }
myMap.set(objKey, "Object as key")
```


این با اشیاء معمولی کار نمی‌کند، که فقط کلیدهای رشته‌ای را می‌پذیرند.


برای **دریافت یک مقدار**، از `myMap.get(key)` استفاده کنید:


```javascript
console.log(myMap.get("name"))     // Alice
console.log(myMap.get(42))         // The answer
console.log(myMap.get(objKey))     // Object as key
```


برای **بررسی وجود یک کلید**، از `myMap.has(key)` استفاده کنید:


```javascript
console.log(myMap.has("name")) // true
```


برای **حذف یک کلید**، از `myMap.delete(key)` استفاده کنید:


```javascript
myMap.delete("name")
```


برای **پاک کردن کل نقشه**، از `myMap.clear()` استفاده کنید:


```javascript
myMap.clear()
```


نقشه‌ها برای مدیریت مجموعه‌های بزرگ از مقادیر عالی هستند، زیرا دسترسی به مقادیر در یک نقشه بزرگ معمولاً عملکرد بسیار بهتری نسبت به یک شیء بزرگ ارائه می‌دهد.


### `تنظیم`


یک `Set` مجموعه‌ای از **فقط مقادیر** (بدون کلید) است، که هر مقدار باید **منحصر به فرد** باشد. این به این معنی است:



- نمی‌توانید یک مقدار را دو بار داشته باشید.
- مقادیر به ترتیبی که اضافه می‌کنید ذخیره می‌شوند.


شما مجموعه‌ای مانند این ایجاد می‌کنید:


```javascript
const mySet = new Set()
```


برای **افزودن مقادیر**، از `mySet.add(value)` استفاده کنید:


```javascript
mySet.add(1)
mySet.add(2)
mySet.add(2) // duplicate, will be ignored
```


حتی با اینکه تلاش کردیم `2` را دو بار اضافه کنیم، مجموعه فقط یک نسخه را نگه می‌دارد.


برای **بررسی اینکه آیا یک مقدار در مجموعه وجود دارد**، از `mySet.has(value)` استفاده کنید:


```javascript
console.log(mySet.has(2)) // true
console.log(mySet.has(3)) // false
```


برای **حذف یک مقدار**، از `mySet.delete(value)` استفاده کنید:


```javascript
mySet.delete(2)
```


برای **پاک کردن همه چیز** از `mySet.clear()` استفاده کنید:


```javascript
mySet.clear()
```


یک `Set` زمانی مفید است که می‌خواهید مجموعه‌ای از مقادیر یکتا را بدون نیاز به بررسی دستی تکراری‌ها نگه دارید:


```javascript
const numberArray = [1, 2, 2, 3, 4, 4, 4, 5]

const numberSet = new Set(numberArray)

console.log(numberSet) // Set(5) { 1, 2, 3, 4, 5 }
```


`Set` از تکراری‌ها جلوگیری می‌کند.


## تکرارگرها

<chapterId>61d24e5e-b7e4-541a-8322-778f61f26a72</chapterId>


اکثر چیزها در جاوااسکریپت که می‌توانید بر روی آن‌ها حلقه بزنید (مانند آرایه‌ها، رشته‌ها، نقشه‌ها، مجموعه‌ها) **قابل تکرار** هستند: آن‌ها می‌توانند برای محتوای خود تکرارگر ارائه دهند.


یک **iterator** یک شیء خاص در جاوااسکریپت است که به شما کمک می‌کند تا از طریق یک لیست از آیتم‌ها **یکی یکی** عبور کنید.


### `Object` تکرارکننده‌ها


برخلاف آرایه‌ها یا نقشه‌ها، اشیاء معمولی **قابل پیمایش** با `for...of` نیستند. اگر این را امتحان کنید:


```javascript
const user = {
name: "Alice",
age: 30
}

for (const value of user) {
console.log(value)
}
```


شما یک خطا دریافت خواهید کرد:


```
TypeError: user is not iterable
```


این به این دلیل است که اشیاء ساده یک iterator داخلی ندارند. اما جاوااسکریپت ابزارهای دیگری برای پیمایش بر روی آن‌ها به شما می‌دهد.


#### `Object.keys()`


می‌توانید از `Object.keys(obj)` برای دریافت یک آرایه از **کلیدهای** شیء استفاده کنید و سپس بر روی آن حلقه بزنید:


```javascript
const user = {
name: "Alice",
age: 30
}

const keys = Object.keys(user)

for (const key of keys) {
console.log(key)
}
```


این چاپ می‌کند:


```
name
age
```


#### `Object.values()`


برای پیمایش بر روی **مقادیر**، از `Object.values()` استفاده کنید:


```javascript
const user = {
name: "Alice",
age: 30
}

const values = Object.values(user)

for (const value of values) {
console.log(value)
}
```


این چاپ می‌کند:


```
Alice
30
```


#### `Object.entries()`


اگر می‌خواهید **هم کلید و هم مقدار** را داشته باشید، از `Object.entries()` استفاده کنید:


```javascript
const user = {
name: "Alice",
age: 30
}

const entries = Object.entries(user)

for (const [key, value] of entries) {
console.log(`${key} is ${value}`)
}
```


این چاپ می‌کند:


```
name is Alice
age is 30
```


حتی اگر اشیاء به‌طور مستقیم قابل پیمایش نباشند، این روش‌ها به شما دسترسی کامل به محتوای آن‌ها را به روشی که با `for...of` به خوبی کار می‌کند، می‌دهند.


اما تکرارگرها چگونه کار می‌کنند؟


### `Symbol.iterator`


راز تمام تکرارپذیرها یک **نماد** ویژه به نام `Symbol.iterator` است.


این نماد یک کلید داخلی است که به جاوااسکریپت می‌گوید: "این شیء قابل تکرار است."


وقتی که `myIterable[Symbol.iterator]()` را فراخوانی می‌کنید، جاوااسکریپت یک **شیء تکرارگر** با یک متد `.next()` به شما برمی‌گرداند.


بیایید ببینیم که این چگونه به نظر می‌رسد:


```javascript
const colors = ["red", "green", "blue"]

const iterator = colors[Symbol.iterator]()

console.log(iterator.next()) // { value: 'red', done: false }
```


هر بار که `.next()` فراخوانی می‌شود، مقدار بعدی را به شما می‌دهد. وقتی تمام شد، برمی‌گرداند:


```javascript
{ value: undefined, done: true }
```


### `بعدی()`


متد `.next()` برای دریافت آیتم بعدی از دنباله استفاده می‌شود.


هر بار که `.next()` را فراخوانی می‌کنید، یک شیء با دو کلید دریافت می‌کنید:



- `value`: آیتم فعلی
- `done`: یک بولین که به شما می‌گوید آیا تکرار به پایان رسیده است یا خیر


بیایید یک مثال کامل انجام دهیم:


```javascript
const names = ["Lina", "Tom", "Eva"]      // declare an array
const iterator = names[Symbol.iterator]() // use the Symbol.iterator function to get an iterator for this array

let result = iterator.next()              // get the first element of the array

while (!result.done) {                    // repeat this loop until you reach the last element of the array, which is marked with { done: true }
console.log(result.value)               // print the value of each element
result = iterator.next()                // get the next element of the array
}
```


این چاپ می‌کند:


```
Lina
Tom
Eva
```


این‌گونه است که یک حلقه `for...of` در پشت صحنه کار می‌کند: از این الگو با `.next()` استفاده می‌کند.


شما همان نتیجه را خواهید گرفت با


```javascript
const names = ["Lina", "Tom", "Eva"]

for (const result of names) {
console.log(result)
}
```


### قابل تکرار کردن یک کلاس


شما همچنین می‌توانید کلاس **iterable** خود را با اضافه کردن یک متد `[Symbol.iterator]()` تعریف کنید.


بیایید بگوییم که می‌خواهیم کلاسی داشته باشیم که **محدوده‌ای از اعداد** را نشان دهد، مانند ۱ تا ۵.


```javascript
class Range {
constructor(start, end) {
this.start = start
this.end = end
}

[Symbol.iterator]() {
let current = this.start
const end = this.end

return {
next() {
if (current <= end) {
const result = { value: current, done: false }
current = current + 1
return result
} else {
return { done: true }
}
}
}
}
}

const myRange = new Range(1, 5)

for (const num of myRange) {
console.log(num)
}
```


این چاپ می‌کند:


```
1
2
3
4
5
```


در اینجا آنچه در حال وقوع است:



- ما یک کلاس `Range` تعریف کردیم
- در داخل کلاس، ما `[Symbol.iterator]()` را پیاده‌سازی کردیم، بنابراین جاوااسکریپت می‌داند چگونه آن را پیمایش کند.
- متد `next()` هر عدد را یکی یکی برمی‌گرداند.
- وقتی به `پایان` می‌رسیم، مقدار `{ done: true }` را برمی‌گرداند.


اکنون کلاس `Range` ما مانند یک آرایه عمل می‌کند و می‌توانیم از آن در هر حلقه‌ای که انتظار یک iterable را دارد، استفاده کنیم.


### توابع مولد و `yield`


برای آسان‌تر کردن ایجاد iteratorها، جاوااسکریپت به شما **توابع مولد** می‌دهد، با استفاده از کلمه کلیدی `function*` (این `function` با یک `*` در انتها است) و کلمه کلیدی `yield`.


بیایید امتحان کنیم:


```javascript
function* numberGenerator() {
yield 1
yield 2
yield 3
}

const iterator = numberGenerator()

console.log(iterator.next()) // { value: 1, done: false }
console.log(iterator.next()) // { value: 2, done: false }
console.log(iterator.next()) // { value: 3, done: false }
console.log(iterator.next()) // { value: undefined, done: true }
```


هر `yield` یک مقدار را برمی‌گرداند و **اجرای** تابع را تا زمانی که `.next()` بعدی فراخوانی شود، متوقف می‌کند.


می‌توانید با استفاده از `for...of` بر روی یک تولیدکننده حلقه بزنید:


```javascript
for (const num of numberGenerator()) {
console.log(num)
}
```


این چاپ می‌شود:


```
1
2
3
```


## هم‌زمانی با بازخوانی‌ها

<chapterId>f3fc76ca-b3ef-54eb-a06e-501007002054</chapterId>


تا کنون، کد ما **هم‌زمان** بوده است: به ترتیب، خط به خط اجرا می‌شود. اما برخی چیزها در دنیای واقعی زمان‌بر هستند و نمی‌خواهیم کل برنامه در حین انتظار متوقف شود.


در این فصل ما قصد داریم یک مفهوم جدید را معرفی کنیم: **هم‌زمانی**. این مفهوم به ما اجازه می‌دهد تا ترتیب انجام کارها را تغییر دهیم. این موضوع زمانی که با مواردی مانند تایمرها، ورودی کاربر، یا خواندن فایل‌ها از دیسک سروکار داریم، مفید است. جاوااسکریپت ابزارهای مختلفی برای انجام هم‌زمانی ارائه می‌دهد.


### `setTimeout`


تابع `setTimeout` به شما اجازه می‌دهد تا **یک تابع را بعداً اجرا کنید**، پس از گذشت مدتی زمان.


مثال:


```javascript
console.log("Start")

setTimeout(
() => console.log("This runs after 2 seconds"),
2000
)

console.log("End")
```


این چاپ می‌کند:


```
Start
End
This runs after 2 seconds
```


حتی اگرچه `setTimeout` در وسط کد ظاهر می‌شود، بقیه را مسدود نمی‌کند. در عوض، تابع را برای اجرا در **آینده** زمان‌بندی می‌کند و بلافاصله ادامه می‌دهد.


`2000` به معنای 2000 میلی‌ثانیه است (که برابر با 2 ثانیه می‌باشد).

در اینجا بازنویسی مفصل‌تر و مناسب‌تر برای مبتدیان از بخش‌های **Callbacks** و **Promise** ارائه شده است، که از دستکاری داده‌ها و توضیحات واضح در سراسر متن استفاده می‌کند:


### بازگشت‌ها


یک **کالبک** فقط یک تابع است که ما به تابع دیگری می‌دهیم تا بتواند **بعداً فراخوانی شود**.


بیایید به یک مثال واقعی با استفاده از اعداد نگاه کنیم. تصور کنید که لیستی از اعداد داریم و می‌خواهیم هر یک از آن‌ها را دو برابر کنیم و سپس یک تابع (callback) را به آرایه "دو برابر شده" اعمال کنیم، اما می‌خواهیم این کار را پس از یک تأخیر کوچک انجام دهیم، گویی که منتظر چیزی کند هستیم (مانند بارگذاری داده‌ها از اینترنت).


در اینجا یک تابع وجود دارد که با استفاده از یک **callback** این کار را انجام می‌دهد:


```javascript
function doubleNumbers(numbersArray, callback) {
// Pretend we're doing a slow operation using setTimeout
setTimeout(() => {
// Use the map method to create a new array where each number is doubled
const doubled = numbersArray.map(n => n * 2)

// When we're done, we call the callback function with the result
callback(doubled)
}, 1000) // Wait 1 second before running the code inside
}
```


بیایید سعی کنیم از این تابع استفاده کنیم:


```javascript
const input = [1, 2, 3]

doubleNumbers(input, function(result) {
console.log("Here is the doubled array:", result)
})
```


پس از 1 ثانیه، این چاپ می‌شود:


```
Here is the doubled array: [ 2, 4, 6 ]
```


**اینجا چه خبر است؟**


1. ما `input` را به عنوان لیستی از اعدادی که می‌خواهیم دو برابر کنیم، ارسال می‌کنیم.

۲. ما همچنین یک **تابع بازگشتی** را ارسال می‌کنیم که به برنامه می‌گوید *بعد از* دو برابر کردن چه کاری انجام دهد.

۳. در `doubleNumbers`، ما با استفاده از `setTimeout` یک تأخیر شبیه‌سازی می‌کنیم، سپس دو برابر کردن را انجام می‌دهیم.

۴. پس از انجام این کار، ما callback را بر روی آرایه "دو برابر شده" نتیجه‌گیری شده فراخوانی می‌کنیم.


این تکنیک کار می‌کند، اما تصور کنید که می‌خواهید **گام‌های بیشتری** پس از آن انجام دهید، مانند فیلتر کردن اعداد کوچک و سپس جمع کردن آن‌ها. شما باید **بازگشت‌های بیشتری** مانند این را تو در تو کنید:


```javascript
doubleNumbers(input, function(doubled) {
filterBigNumbers(doubled, function(filtered) {
sumNumbers(filtered, function(total) {
console.log("Final result:", total)
})
})
})
```


این Hard برای خواندن و نامرتب است. این سبک **callback hell** نامیده می‌شود و دقیقاً برای رفع آن `Promise` ایجاد شد.


## هم‌زمانی با Promises

<chapterId>30fddaca-729f-5c8d-bf86-8dfc7b3c9800</chapterId>


`Promise` یک شیء داخلی جاوااسکریپت است که نشان‌دهنده مقداری است که **در آینده آماده خواهد شد**.


ما می‌توانیم یک Promise را به این صورت ایجاد کنیم:


```javascript
const promise = new Promise((resolve, reject) => {
// Do something that takes time here...

resolve("It worked!") // This means everything went OK
})
```


قسمت `new Promise()` وعده را ایجاد می‌کند.


درون آن، ما یک تابع با دو پارامتر به آن می‌دهیم:



- `resolve`، تابعی است که وقتی همه چیز با موفقیت انجام شد، آن را فراخوانی می‌کنیم.
- `reject`، تابعی است که فراخوانی می‌کنیم اگر مشکلی پیش بیاید.


در مثال بالا، ما بلافاصله آن را با پیام `"It worked!"` حل می‌کنیم.


### `.then()`


برای انجام کاری **بعد از** اتمام وعده، از `.then()` استفاده می‌کنیم:


```javascript
const promise = new Promise((resolve, reject) => {
// Do something that takes time here...

resolve(100) // This means everything went OK
})

promise.then(result => {
console.log("The result is:", result)
})
```


این چاپ می‌کند:


```
The result is: 100
```


مقداری که به `resolve()` ارسال کردیم به تابع داخل `.then()` به عنوان `result` ارسال می‌شود.


بیایید یک وظیفه را شبیه‌سازی کنیم که ۲ ثانیه طول می‌کشد با استفاده از `setTimeout`:


```javascript
const delayedPromise = new Promise(
(resolve, reject) => {
setTimeout(
() => resolve("Done waiting!"),
2000
)
})

delayedPromise.then(result => console.log(result))
```


این 2 ثانیه صبر می‌کند و سپس چاپ می‌کند:


```
Done waiting!
```


### `رد کردن()`


بیایید یک وعده ایجاد کنیم که **شکست می‌خورد**:


```javascript
const failingPromise = new Promise((resolve, reject) => {
reject("Something went wrong")
})
```


اکنون اگر از `.then()` روی این استفاده کنیم، هیچ اتفاقی نخواهد افتاد، زیرا `.then()` فقط موفقیت را مدیریت می‌کند.


برای مدیریت خطاها، از `.catch()` استفاده می‌کنیم:


```javascript
const failingPromise = new Promise((resolve, reject) => {
reject("Something went wrong")
})

failingPromise
.then(
result => console.log("This will NOT run:", result)
)
.catch(
error => console.log("Caught an error:", error)
)
```


این فقط چاپ می‌شود


```
Caught an error: Something went wrong
```


مقداری که به `reject()` ارسال می‌شود به تابع `.catch()` ارسال می‌گردد.


بیایید یک Promise بسازیم که **گاهی اوقات کار می‌کند و گاهی اوقات شکست می‌خورد**، بر اساس یک شرط.


```javascript
function checkNumber(n) {
return new Promise((resolve, reject) => {
if (n > 0) {
resolve("Positive number")
} else {
reject("Not a positive number")
}
})
}
```


اکنون می‌توانیم این را فراخوانی کرده و هر دو حالت را مدیریت کنیم:


```javascript
checkNumber(5)
.then(
msg => console.log("Success:", msg)
)
.catch(
err => console.log("Failure:", err)
)
```


این چاپ می‌شود:


```
Success: Positive number
```


و اگر با یک عدد متفاوت امتحان کنیم:


```javascript
checkNumber(-1)
.then(
msg => console.log("Success:", msg)
)
.catch(
err => console.log("Failure:", err)
)
```


چاپ می‌کند:


```
Failure: Not a positive number
```


### زنجیره‌سازی عملیات با استفاده از `Promise`ها



ما می‌توانیم مثال قبلی خود را با استفاده از `Promise` بازنویسی کنیم و بسیار تمیزتر به نظر خواهد رسید.


بیایید با نوشتن نسخه جدیدی از تابع دو برابر کردن خود شروع کنیم، اما این بار، آن یک **promise** برمی‌گرداند:


```javascript
function doubleNumbers(numbers) {
return new Promise(resolve => {
// Wait 1 second before doing the operation
setTimeout(() => {
const doubled = numbers.map(n => n * 2)
resolve(doubled) // Return the result using resolve
}, 1000)
})
}
```


اکنون می‌توانیم از `.then()` استفاده کنیم تا به جاوااسکریپت بگوییم با نتیجه چه کاری انجام دهد:


```javascript
function doubleNumbers(numbers) {
return new Promise(resolve => {
// Wait 1 second before doing the operation
setTimeout(() => {
const doubled = numbers.map(n => n * 2)
resolve(doubled) // Return the result using resolve
}, 1000)
})
}

const input = [1, 2, 3]

doubleNumbers(input)
.then(
result => console.log("Doubled numbers:", result)
)
```


این چاپ می‌شود:


```
Doubled numbers: [ 2, 4, 6 ]
```


تا اینجا، این به همان شیوه نسخه callback کار می‌کند، اما اکنون کد راحت‌تر قابل توسعه و خواندن است.


بیایید بگوییم می‌خواهیم مراحل بیشتری اضافه کنیم:


1. ابتدا، همه اعداد را دو برابر کنید

۲. سپس، اعداد کوچکتر از ۴ را حذف کنید

3. در نهایت، همه را با هم جمع کنید


ما می‌توانیم برای هر مرحله یک تابع بنویسیم، همه با استفاده از وعده‌ها:


```javascript
function doubleNumbers(numbers) {
return new Promise(resolve => {
// Wait 1 second before doing the operation
setTimeout(() => {
const doubled = numbers.map(n => n * 2)
resolve(doubled) // Return the result using resolve
}, 1000)
})
}

function filterBigNumbers(numbers) {
return new Promise(resolve => {
setTimeout(() => {
const filtered = numbers.filter(n => n > 3)
resolve(filtered)
}, 1000)
})
}

function sumNumbers(numbers) {
return new Promise(resolve => {
setTimeout(() => {
const total = numbers.reduce((acc, n) => acc + n, 0)
resolve(total)
}, 1000)
})
}
```


اکنون می‌توانیم آن‌ها را به این صورت **زنجیره** کنیم:


```javascript
const input = [1, 2, 3]

doubleNumbers(input)
.then(filterBigNumbers)
.then(sumNumbers)
.then(
result => console.log("Final result after all steps:", result)
)
```


این چاپ می‌کند:


```
Final result after all steps: 10
```


بیایید بررسی کنیم که این چه کاری انجام می‌دهد:


1. `doubleNumbers` آرایه را دو برابر می‌کند: `[2, 4, 6]`

2. `filterBigNumbers` هر چیزی که ≤ 3 باشد را حذف می‌کند: `[4, 6]`

3. `sumNumbers` اعداد باقی‌مانده را جمع می‌کند: `4 + 6 = 10`

4. در نهایت، نتیجه را چاپ می‌کنیم.


هر `.then()` منتظر می‌ماند تا مرحله قبل از آن به پایان برسد. بنابراین می‌توانیم یک **زنجیره‌ای از اقدامات** بدون تو در تو کردن بسازیم. این کار کد را خواناتر و اشکال‌زدایی آن را آسان‌تر می‌کند.


## هم‌زمانی با `async`/`await`

<chapterId>6e93d29f-c8bf-5fd1-a9c9-4e794ee6cbd0</chapterId>


ما دیدیم که زنجیره‌های `Promise` به ما کمک می‌کنند تا از جهنم بازگشت جلوگیری کنیم، اما وقتی مراحل زیادی درگیر هستند، هنوز هم می‌توانند کمی Hard برای خواندن باشند.


اینجاست که `async` و `await` وارد می‌شوند. آن‌ها به ما اجازه می‌دهند کدی غیرهمزمان بنویسیم **که شبیه کد همزمان به نظر می‌رسد**، که فهم آن را آسان‌تر می‌کند.


### `async` چیست؟


وقتی کلمه کلیدی `async` را قبل از یک تابع می‌نویسید، جاوااسکریپت به‌طور خودکار مقدار بازگشتی تابع را در یک Promise قرار می‌دهد.


بیایید یک مثال ساده ببینیم:


```javascript
async function greet() {
return "hello"
}
```


اگر این تابع را فراخوانی کنید:


```javascript
const result = greet()
console.log(result)
```


شما این را خواهید دید:


```
Promise { 'hello' }
```


حتی اگر شما فقط یک رشته را برگردانید، جاوااسکریپت آن را به یک Promise برای شما تبدیل می‌کند. شما می‌توانید مقدار واقعی را با استفاده از `.then()` به این صورت دریافت کنید:


```javascript
greet().then( result => console.log(result) ) // prints "hello"
```


یا می‌توانید از `await` استفاده کنید...


### `await` چیست؟


کلمه کلیدی `await` به جاوااسکریپت می‌گوید: «صبر کن تا این Promise تمام شود، و سپس نتیجه را به من بده.»


اما شما فقط می‌توانید از `await` **داخل یک تابع async** استفاده کنید.


بیایید مثال را با استفاده از `await` بازنویسی کنیم:


```javascript
async function greet() {
return "hello"
}

async function greetAndLog() {
const result = await greet()
console.log(result)
}

greetAndLog() // prints "hello"
```


اکنون می‌توانیم از نتیجه به‌عنوان یک مقدار معمولی استفاده کنیم.


بیایید حالا کاری کمی مفیدتر انجام دهیم.


### شبیه‌سازی تأخیر با `await`


ما یک تابع ساده `wait` ایجاد خواهیم کرد که یک مقدار میلی‌ثانیه به عنوان آرگومان می‌گیرد و فقط پس از آن تعداد میلی‌ثانیه حل می‌شود، بدون انجام هیچ کار دیگری:


```javascript
function wait(ms) {
return new Promise(resolve => {
setTimeout(resolve, ms)
})
}
```


بیایید از آن استفاده کنیم:


```javascript
async function test() {
console.log("waiting 2 seconds...")
await wait(2000)
console.log("done waiting")
}

test()
```


این چاپ می‌کند:


```
waiting 2 seconds...
done waiting
```


می‌توانید به `await` به‌عنوان "اینجا مکث کن تا وعده انجام شود، سپس ادامه بده." فکر کنید.


این به شما اجازه می‌دهد تا کد را به صورت **از بالا به پایین** بنویسید که به صورت ناهمگام عمل می‌کند، بدون نیاز به زنجیره کردن فراخوانی‌های `.then()`.


### در انتظار داده‌ها


بیایید از مثال قبلی خود استفاده کنیم، جایی که اعداد را دو برابر می‌کنیم، سپس فیلتر می‌کنیم، سپس جمع می‌کنیم. اما این بار از `async`/`await` استفاده خواهیم کرد.


ما ۳ تابع ایجاد خواهیم کرد که انتظار را شبیه‌سازی کرده و Promise‌ها را برمی‌گردانند:


```javascript
function doubleNumbers(numbers) {
return new Promise(resolve => {
setTimeout(() => {
const doubled = numbers.map(n => n * 2)
resolve(doubled)
}, 1000)
})
}

function filterBigNumbers(numbers) {
return new Promise(resolve => {
setTimeout(() => {
const filtered = numbers.filter(n => n > 3)
resolve(filtered)
}, 1000)
})
}

function sumNumbers(numbers) {
return new Promise(resolve => {
setTimeout(() => {
const total = numbers.reduce((acc, n) => acc + n, 0)
resolve(total)
}, 1000)
})
}
```


حالا بیایید یک تابع `async` بنویسیم تا آن‌ها را ترکیب کنیم:


```javascript
async function process(numbers) {
const doubled = await doubleNumbers(numbers)
const filtered = await filterBigNumbers(doubled)
const total = await sumNumbers(filtered)

console.log("Final result:", total)
}

const input = [1, 2, 3]
process(input)
```


این چاپ می‌کند:


```
Final result: 10
```


این بسیار آسان‌تر از زنجیره کردن `.then()` یا تو در تو کردن callbackها است.


به نظر می‌رسد یک برنامه گام‌به‌گام معمولی است، اما همچنان به صورت ناهمگام عمل می‌کند.


## تکرارگرهای ناهمگام

<chapterId>438b037d-9931-56d7-9052-7b4470f3c75b</chapterId>


شما قبلاً درباره **iterators** یاد گرفته‌اید و اینکه چگونه می‌توانیم از `for...of` برای پیمایش آرایه‌ها و دیگر چیزهای قابل پیمایش استفاده کنیم.


اما اگر داده‌ای که می‌خواهیم بر روی آن تکرار کنیم زمان ببرد تا برسد چه؟


گاهی اوقات می‌خواهیم بر روی چیزهایی که به صورت **غیرهمزمان** می‌رسند، مانند پیام‌ها از یک چت، خطوط از یک فایل، یا اعداد از یک منبع کند، حلقه بزنیم.


برای انجام این کار، جاوااسکریپت به ما **async iterators** می‌دهد.


### توابع مولد ناهمگام


آسان‌ترین راه برای ایجاد یک تکرارگر ناهمگام استفاده از یک **تابع مولد ناهمگام** است.


ما اینگونه می‌نویسیم:


```javascript
async function* generateNumbers() {
yield 1
yield 2
yield 3
}
```


این دقیقاً شبیه یک ژنراتور معمولی به نظر می‌رسد، اما با `async` قبل از آن.


اکنون می‌توانیم از `for await...of` برای مصرف مقادیر استفاده کنیم:


```javascript
async function run() {
for await (const n of generateNumbers()) {
console.log("Got number:", n)
}

console.log("Done!")
}

run()
```


این چاپ خواهد شد:


```
Got number: 1
Got number: 2
Got number: 3
Done!
```


تفاوت آن با یک ژنراتور معمولی چیست؟


تفاوت این است: اکنون می‌توانیم از `await` درون ژنراتور استفاده کنیم.


بیایید دوباره یک کمک‌کننده تأخیر بسازیم:


```javascript
function wait(ms) {
return new Promise(resolve => setTimeout(resolve, ms))
}
```


حالا بیایید اعداد را **به آرامی** تولید کنیم:


```javascript
async function* generateSlowNumbers() {
await wait(1000)
yield 1

await wait(1000)
yield 2

await wait(1000)
yield 3
}
```


سعی کنید از آن استفاده کنید:


```javascript
async function run() {
for await (const n of generateSlowNumbers()) {
console.log("Got number:", n)
}

console.log("Done!")
}

run()
```


### چرا از تکرارگرهای ناهمگام استفاده کنیم؟


تکرارگرهای ناهمگام زمانی مفید هستند که:



- مقادیر همه به یکباره نمی‌رسند.
- شما می‌خواهید آن‌ها را یکی یکی، **همان‌طور که می‌آیند**، مدیریت کنید.
- شما با Promises کار می‌کنید و می‌خواهید به روشی تمیز حلقه بزنید.


به عنوان مثال، اگر می‌خواهید پیام‌ها را از یک سرور چت یکی یکی بارگذاری کنید، یا یک فایل بزرگ را به صورت تکه تکه دانلود کنید، تکرارگرهای ناهمگام به شما روشی برای نوشتن یک حلقه `for` می‌دهند که با داده‌های تأخیری کار می‌کند.


### `Symbol.asyncIterator`


ما همچنین می‌توانیم از تکرارگرهای ناهمگام در کلاس‌های سفارشی استفاده کنیم.


در اینجا مثالی وجود دارد که اعداد را با تأخیر تولید می‌کند:


```javascript
function wait(ms) {
return new Promise(resolve => setTimeout(resolve, ms))
}

class DelayedNumbers {
constructor(numbers) {
this.numbers = numbers
}

async *[Symbol.asyncIterator]() {
for (const n of this.numbers) {
await wait(1000)
yield n
}
}
}
```


اکنون می‌توانیم از `for await...of` درست مانند قبل استفاده کنیم:


```javascript
async function run() {
const source = new DelayedNumbers([10, 20, 30])

for await (const n of source) {
console.log("Received:", n)
}

console.log("All done!")
}

run()
```


این به شما امکان می‌دهد اشیایی ایجاد کنید که می‌توان به صورت ناهمگام بر روی آن‌ها تکرار کرد.


## Assignment شکر نحوی

<chapterId>8b1ba7d8-ecfd-5470-b86e-73cb84ccc8b7</chapterId>


"Syntax sugar" به معنای نوشتن چیزی به صورت کوتاه‌تر یا آسان‌تر است، بدون اینکه عملکرد آن تغییر کند. این فقط یک روش زیباتر برای بیان همان چیز است.


جاوااسکریپت دارای برخی از شکرهای نحوی داخلی است که به ما اجازه می‌دهد تا اعلان‌های تمیزتر و کوتاه‌تری بنویسیم. در این فصل، به نحوه اختصاص مقادیر بر اساس یک شرط، به‌روزرسانی متغیرها با ریاضیات، استخراج مقادیر از آرایه‌ها یا اشیاء، و کپی یا ترکیب آن‌ها با نحو ساده‌تر خواهیم پرداخت.


### عملگر سه‌گانه


در جاوااسکریپت، می‌توانید با استفاده از **عملگر سه‌تایی**، که یک روش کوتاه برای نوشتن `if...else` است، یک مقدار را بر اساس یک شرط اختصاص دهید.


به جای انجام:


```javascript
let message

if (isMorning) {
message = "Good morning"
} else {
message = "Hello"
}
```


شما می‌توانید بنویسید:


```javascript
const message = isMorning ? "Good morning" : "Hello"
```


این به این معنی است:



- اگر `isMorning` درست است، از `"صبح بخیر"` استفاده کنید
- در غیر این صورت، از `"Hello"` استفاده کنید


فرم کلی به این صورت است:


```javascript
condition ? valueIfTrue : valueIfFalse
```


می‌توانید از آن در داخل عبارات دیگر نیز استفاده کنید:


```javascript
console.log(isSunny ? "Go outside" : "Stay in")
```


فقط مطمئن شوید که از آن برای تصمیمات **ساده** استفاده کنید. اگر منطق پیچیده است، از `if...else` استفاده کنید.


### اپراتورهای جایگزین Assignment


جاوااسکریپت دارای **عملگرهای میانبر** برای انجام انتساب‌ها همراه با عملیات است.


بیایید به روش معمول نگاه کنیم:


```javascript
let counter = 1
counter = counter + 1
```


این می‌تواند به این صورت خلاصه شود:


```javascript
let counter = 1
counter += 1 // same as counter = counter + 1
```


در اینجا رایج‌ترین‌ها آورده شده است:


| Operator | Meaning             |
| -------- | ------------------- |
| `+=`     | add and assign      |
| `-=`     | subtract and assign |
| `*=`     | multiply and assign |
| `/=`     | divide and assign   |

مثال‌ها:


```javascript
let score = 10
score += 5  // now score is 15
score *= 2  // now score is 30
score -= 10 // now score is 20
```


این‌ها زمانی مفید هستند که می‌خواهید یک متغیر را با استفاده از مقدار خودش به‌روزرسانی کنید.


### تخریب ساختار


**تخریب** به شما اجازه می‌دهد مقادیر را از آرایه‌ها یا اشیاء بگیرید و به راحتی آن‌ها را در متغیرها ذخیره کنید.


#### آرایه‌ها


فرض کنید شما دارید:


```javascript
const colors = ["red", "green", "blue"]
```


به جای انجام:


```javascript
const first = colors[0]
const second = colors[1]
```


می‌توانید انجام دهید:


```javascript
const [first, second] = colors
```


این اختصاص می‌دهد:



- `اول` به `"قرمز"`
- `second` به `"Green"`


می‌توانید مقادیر را نیز رد کنید:


```javascript
const [,, third] = colors
console.log(third) // blue
```


#### اشیاء


شما می‌توانید مقادیر را از اشیاء نیز استخراج کنید:


```javascript
const user = { name: "Alice", age: 30 }

const { name, age } = user
console.log(name) // Alice
console.log(age)  // 30
```


اگر ویژگی نامی متفاوت از متغیری که می‌خواهید دارد، می‌توانید آن را تغییر نام دهید:


```javascript
const { name: username } = user
console.log(username) // Alice
```


تخریب‌سازی کد شما را هنگام کار با اشیاء و آرایه‌ها تمیزتر می‌کند.


### نحو گسترش


**دستور گسترش** از `...` برای باز کردن یا کپی کردن مقادیر استفاده می‌کند.


#### آرایه‌ها


می‌توانید آرایه‌ها را کپی یا ادغام کنید:


```javascript
const a = [1, 2]
const b = [3, 4]

const both = [...a, ...b]
console.log(both) // [1, 2, 3, 4]
```


شما همچنین می‌توانید یک آرایه را کلون کنید:


```javascript
const original = [10, 20, 30]
const clone = [...original]
```


#### اشیاء


می‌توانید همین کار را با اشیاء انجام دهید:


```javascript
const a = { x: 1 }
const b = { y: 2 }

const merged = { ...a, ...b }
console.log(merged) // { x: 1, y: 2 }
```


همچنین می‌توانید مقادیر را نادیده بگیرید:


```javascript
const user = { name: "Alice", age: 30 }
const updated = { ...user, age: 31 }

console.log(updated) // { name: "Alice", age: 31 }
```


این زمانی که اشیاء را بدون تغییر اصل به‌روزرسانی می‌کنید بسیار مفید است.

## چگونه به Node رسیدیم؟

<chapterId>0da1d60c-06c9-54e6-a181-ae7dabf6e3b8</chapterId>


در این فصل، کمی درباره زمینه تاریخی جاوااسکریپت و NodeJS خواهیم آموخت.


زمینه تاریخی در نرم‌افزار بسیار مهم است، زیرا ما اغلب از ابزارهایی استفاده می‌کنیم که توسط افراد دیگر ساخته شده‌اند و بنابراین تحت تأثیر تصمیماتی که در گذشته توسط آن‌ها گرفته شده است، قرار می‌گیریم.


درک دلیل آن تصمیمات و چگونگی شکل‌گیری ابزارهایی که استفاده می‌کنیم، به ما کمک می‌کند تا کمی کمتر در مورد کاری که انجام می‌دهیم گیج شویم.


### 起源 جاوااسکریپت


جاوااسکریپت به عنوان یک زبان ساده طراحی شد تا صفحات وب را تعاملی کند.


در دهه ۱۹۹۰، مرورگرهای وب مانند Netscape Navigator جاوااسکریپت را اضافه کردند تا توسعه‌دهندگان بتوانند کدی بنویسند که مستقیماً در مرورگر اجرا شود.


ایده اصلی این بود که جاوا به عنوان زبان اصلی برای ساخت وب‌سایت‌ها (با اصطلاحاً "اپلت‌های جاوا") باشد و جاوااسکریپت فقط برای کارهای جزئی استفاده شود.


طراحی اصلی توسط برندان آیک، که در آن زمان کارمند نت‌اسکیپ بود، در کمتر از ۲ هفته انجام شد.


اما بیشتر مردم ترجیح می‌دادند از جاوااسکریپت به جای جاوا استفاده کنند و همچنین اپلت‌های جاوا در آن زمان مشکلات امنیتی داشتند، بنابراین در نهایت جاوا به عنوان یک گزینه کنار گذاشته شد و جاوااسکریپت به استاندارد د فاکتو برای توسعه وب تبدیل شد.


### موتور V8


جاوااسکریپت یک زبان تفسیری است، برخلاف زبان‌های کامپایل شده مانند C.


کدی که به زبان کامپایل شده نوشته می‌شود به یک باینری تبدیل می‌شود و این باینری مستقیماً به CPU کامپیوتر داده می‌شود.


![](assets/en/6.webp)


از سوی دیگر، زبان‌های تفسیری تمایل دارند که کاربرپسندتر باشند و به نحوه تفکر انسان‌ها ("سطح بالا") نزدیک‌تر باشند تا به نحوه کار ماشین‌ها ("سطح پایین")؛ بنابراین معمولاً یک ماشین مجازی برای اجرای کد آن‌ها ساخته شده است.


یک ماشین مجازی یک برنامه ویژه است که بین کدی که می‌نویسید و CPU قرار می‌گیرد و کد شما را اجرا می‌کند (زیرا CPU نمی‌تواند آن را بفهمد).


این به شما اجازه می‌دهد بدون دانستن زیاد درباره ماشین زیرین برنامه‌نویسی کنید، اما این کار هزینه‌ای از نظر عملکرد دارد، زیرا کامپیوتر فقط برنامه شما را اجرا نمی‌کند؛ بلکه برنامه‌ای (ماشین مجازی) را اجرا می‌کند که برنامه شما را اجرا می‌کند.


همان‌طور که برنامه‌های وب پیچیده‌تر شدند، تقاضا برای بهبود عملکرد جاوااسکریپت افزایش یافت. موتور V8 مفسری است که جاوااسکریپت را در گوگل کروم قدرت می‌بخشد. این موتور در گوگل توسعه یافت و در سال ۲۰۰۸ منتشر شد.


در حالی که موتورهای قدیمی‌تر جاوااسکریپت عمدتاً ماشین‌های مجازی سنتی بودند، موتور V8 یک کامپایلر JIT (just-in-time) است.


کد جاوااسکریپت به موتور V8 داده می‌شود و موتور سعی می‌کند بخش‌هایی از آن را به صورت باینری‌های بومی در لحظه کامپایل کند. این به شما این امکان را می‌دهد که تجربه یک زبان سطح بالا را با عملکردی که کمی به زبان‌های سطح پایین نزدیک‌تر است، داشته باشید. این امر جاوااسکریپت را به سریع‌ترین زبان سطح بالا در جهان تبدیل کرده است، چیزی شبیه به "بهترین‌های هر دو جهان".


### زمان اجرای NodeJS


در حالی که استفاده از آن آسان و اجرای آن بسیار سریع بود، پس از انتشار V8 جاوااسکریپت همچنان یک محدودیت بزرگ داشت: فقط می‌توانست داخل یک مرورگر اجرا شود.


چرا این یک مشکل است؟


خب، از آنجا که مرورگرها کدهایی را که از میلیون‌ها منبع مختلف در اینترنت دریافت می‌کنند اجرا می‌کنند، به راحتی می‌توانند به بدافزار آلوده شوند، بنابراین از بقیه سیستم‌عامل "sandboxed" می‌شوند.


![](assets/en/7.webp)


جاوااسکریپت نمی‌توانست به سیستم فایل و دیگر منابع محلی روی کامپیوتر شما دسترسی پیدا کند (حداقل نه به آسانی مانند زبان‌های دیگر)، بنابراین این یک محدودیت قابل توجه در نوع برنامه‌هایی بود که می‌توانستید با آن بسازید.


در سال 2009، رایان دال NodeJS را منتشر کرد، که یک زمان اجرا است که به شما اجازه می‌دهد از موتور V8 خارج از مرورگر، مستقیماً بر روی سیستم‌عامل بومی کامپیوتر خود استفاده کنید. همچنین ویژگی‌های زیادی را اضافه می‌کند که برای نوشتن برنامه‌های سمت سرور و خط فرمان مفید هستند. به عنوان مثال، می‌توانید از NodeJS برای ایجاد یک وب سرور، خواندن و نوشتن فایل‌ها، یا ساخت ابزارهایی که وظایف را خودکار می‌کنند، استفاده کنید.


![](assets/en/8.webp)


در این دوره تا کنون، ویژگی‌های جاوااسکریپت که هم در مرورگر و هم در NodeJS وجود دارند را بررسی کرده‌ایم. این ویژگی‌ها به ما اجازه دادند تا داده‌ها را تعریف کرده و به صورت انتزاعی آن‌ها را دستکاری کنیم. در چند درس بعدی، ویژگی‌هایی که مختص NodeJS هستند و به ما اجازه تعامل با سیستم‌عامل را می‌دهند، بررسی خواهیم کرد.


## آرگومان‌های خط فرمان

<chapterId>960d20f3-c424-5d51-a041-ef17d2e94b6d</chapterId>


نود‌جی‌اس به ما این امکان را می‌دهد که در کنار سایر قابلیت‌ها، رابط‌های خط فرمان (CLIs) بسازیم.


برای این کار ما نیاز به روشی برای دریافت آرگومان‌های خط فرمان داریم که در Node با استفاده از شیء داخلی `process` انجام می‌شود.


### `پردازش`


NodeJS یک شیء ویژه به نام `process` فراهم می‌کند که برنامه در حال اجرای فعلی را نشان می‌دهد.


می‌توانید از آن برای بررسی محیط، دایرکتوری کاری فعلی، و حتی خروج از برنامه در صورت نیاز استفاده کنید.


برای مثال:


```javascript
console.log(process.platform)
```


این سیستم‌عامل پلتفرم را چاپ می‌کند، مانند `win32`، `linux`، یا `darwin` (مک).


### `process.argv`


وقتی یک برنامه NodeJS را از ترمینال اجرا می‌کنید، می‌توانید کلمات اضافی (آرگومان‌ها) را بعد از نام اسکریپت وارد کنید. این‌ها در `process.argv` ذخیره می‌شوند.


به عنوان مثال، اگر این دستور را اجرا کنید:


```
node my_script.js alpha beta
```


می‌توانید آرگومان‌ها را به این صورت چاپ کنید:


```javascript
console.log(process.argv)
```


خروجی ممکن است به این شکل باشد:


```
[ '/path/to/node', '/path/to/my_script.js', 'alpha', 'beta' ]
```


اولین دو آیتم همیشه مسیر Node و مسیر اسکریپت شما هستند. هر کلمه اضافی که به اسکریپت منتقل کرده‌اید بعد از آن می‌آید.


آرایه `process.argv` می‌تواند مانند هر آرایه دیگری با استفاده از متد `.slice()` برش داده شود، بنابراین برای دریافت فقط آرگومان‌هایی که ارسال شده‌اند، می‌توانید انجام دهید


```javascript
const args = process.argv.slice(2)

console.log(args)
```


دسترسی به آرگومان‌هایی که کاربر ارائه می‌دهد برای توسعه برنامه‌های خط فرمان اساسی است.


## ماژول‌ها

<chapterId>4e1651a5-65fd-50bc-b22a-40313d5659ca</chapterId>


زمان‌های اجرای جاوااسکریپت مانند NodeJS معمولاً هر فایل جاوااسکریپت را به عنوان یک ماژول جداگانه در نظر می‌گیرند.


ماژول را به‌عنوان یک جعبه در نظر بگیرید. جعبه فضای خصوصی خود را دارد، بنابراین متغیرها و توابعی که در آن اعلام کرده‌اید با کدهای موجود در جعبه‌های دیگر تداخل ندارند. به‌طور اساسی، هر ماژول دامنه خاص خود را دارد.


یک ماژول می‌تواند بخشی از محتوای خود را صادر کند و آن را برای سایر ماژول‌ها در دسترس قرار دهد و می‌تواند محتوایی را که سایر ماژول‌ها صادر کرده‌اند، وارد کند. جاوااسکریپت به شما اجازه می‌دهد تا محتوا را بین ماژول‌ها صادر و وارد کنید تا فایل‌های مختلف را به هم متصل کنید.


یک برنامه جاوااسکریپت اغلب از چندین ماژول تشکیل شده است که با یکدیگر متصل هستند.


چرا از ماژول‌ها استفاده کنیم؟ با تقسیم کد خود به ماژول‌ها، می‌توانید برنامه خود را به بخش‌های کوچکتر، واضح‌تر و قابل استفاده مجدد سازماندهی کنید. هر ماژول می‌تواند فقط بر روی یک نوع وظیفه تمرکز کند، مانند انجام محاسبات ریاضی، کار با فایل‌ها، یا قالب‌بندی متن.


### ماژول‌های CommonJS


در NodeJS، رایج‌ترین سیستم برای مدیریت ماژول‌ها **CommonJS** نامیده می‌شود.


در این سیستم، می‌توانید کد را از یک ماژول با استفاده از `module.exports` به اشتراک بگذارید (صادر کنید) و آن را در فایل دیگری با استفاده از `require()` بارگذاری (وارد) کنید.


برای در دسترس قرار دادن چیزی خارج از یک ماژول، آن را به `module.exports` اختصاص می‌دهید:


```javascript
const greeting = "Hello!"

module.exports = greeting
```


در اینجا، رشته `"Hello!"` چیزی است که این ماژول صادر می‌کند.


برای استفاده از کد صادر شده از فایل دیگر، از تابع `require()` با مسیر آن فایل استفاده می‌کنید:


```javascript
const greeting = require("./greeting.js")

console.log(greeting)
```


این چاپ می‌کند:


```
Hello!
```


می‌توانید چندین چیز را با بسته‌بندی آن‌ها در یک شیء ناشناس صادر کنید، مانند


```javascript
const greeting1 = "hello"

const greeting2 = "hi"

module.exports = {
greeting1, greeting2
}
```


سیستم ماژول CommonJS ابتدا توسط NodeJS پذیرفته شد. بعدها ماژول‌های ES نیز اضافه شدند.


### ماژول‌های ES


NodeJS همچنین از نوع دیگری از ماژول به نام **ES Modules** پشتیبانی می‌کند که در توسعه وب محبوب هستند. آن‌ها از کلمات کلیدی `export` و `import` استفاده می‌کنند.


ماژول‌های ES برای مرورگر توسعه داده شدند و تنها بعداً به Node اضافه شدند. اگر می‌خواهید از آن‌ها استفاده کنید، ممکن است بسته به نسخه Node که استفاده می‌کنید، نیاز باشد از `.mjs` به عنوان پسوند فایل به جای `.js` استفاده کنید.


در یک فایل به نام `greeting.mjs` می‌نویسیم


```javascript
export const greeting = "Hello!"
```

همان‌طور که می‌بینید، ما ثابت را مستقیماً در جایی که تعریف می‌شود صادر می‌کنیم.


در فایل دیگری به نام `index.mjs`، آن را وارد می‌کنیم:


```javascript
import { greeting } from "./greeting.mjs"

console.log(greeting)
```


می‌توانید اعلان‌های مختلف را در بخش‌های مختلف فایل صادر کنید، مانند


```javascript
export const num = 10

export function double (x) {
return x*2
}
```


### کتابخانه استاندارد NodeJS


NodeJS همچنین شامل بسیاری از ماژول‌های داخلی است. این‌ها ماژول‌های آماده‌ای هستند که توسط NodeJS ارائه شده‌اند و به انجام وظایف رایج مانند خواندن فایل‌ها، کار با سیستم‌عامل، یا اتصال به شبکه کمک می‌کنند.


به عنوان مثال، ماژول `os` به شما اطلاعاتی درباره سیستم‌عاملتان می‌دهد:


```javascript
const os = require("os")

console.log(os.platform())
```


شما نیازی به نصب این ماژول‌های داخلی ندارید، آن‌ها همراه با NodeJS ارائه می‌شوند. آن‌ها "کتابخانه استاندارد" زمان اجرا را تشکیل می‌دهند و توسط اکثر برنامه‌های Node برای انجام کارهایی مانند خواندن فایل‌ها یا ارتباط از طریق اینترنت استفاده می‌شوند.


فصل‌های بعدی به شما چند مثال مفید از کاربرد آن‌ها را نشان خواهند داد.


## ماژول `fs`

<chapterId>911e953a-35ae-5ee7-bd74-372501c32e81</chapterId>


ماژول `fs` (مخفف **file system**) بخشی از کتابخانه استاندارد NodeJS است. این ماژول به شما امکان می‌دهد با فایل‌ها و دایرکتوری‌های روی کامپیوتر خود کار کنید: می‌توانید فایل‌ها را بخوانید، بنویسید، حذف کنید، تغییر نام دهید و کارهای بیشتری انجام دهید.


برای استفاده از آن، ابتدا باید آن را در بالای فایل خود وارد کنید:


```javascript
const fs = require("fs")
```


### همگام‌سازی API


ساده‌ترین راه برای استفاده از `fs` استفاده از متدهای **همگام** آن است.


این روش‌ها برنامه را تا زمانی که به پایان برسند مسدود می‌کنند (بنابراین خط بعدی کد تا زمانی که عملیات کامل نشود اجرا نمی‌شود).


در اینجا یک مثال از خواندن یک فایل به صورت همزمان آورده شده است:


```javascript
const fs = require("fs")

const data = fs.readFileSync("example.txt", "utf8")

console.log(data)
```


اگر فایلی به نام `example.txt` در همان دایرکتوری که اسکریپت شما قرار دارد وجود داشته باشد، این محتواهای آن را چاپ خواهد کرد.


شما همچنین می‌توانید به صورت همزمان در یک فایل بنویسید:


```javascript
const fs = require("fs")

fs.writeFileSync("output.txt", "Hello from Node!")

console.log("File written!")
```


این یک فایل به نام `output.txt` با متن ایجاد (یا بازنویسی) می‌کند.


در اینجا برخی از عملیات‌های رایج که می‌توانید با این API انجام دهید آورده شده است:


```javascript
const fs = require("fs")

// List files and folders
const items = fs.readdirSync(".")
console.log("Items in current directory:", items)

// Create folder
fs.mkdirSync("my_folder")
console.log("Folder created")

// Delete folder
fs.rmdirSync("my_folder")
console.log("Folder deleted")

// Create & write file
fs.writeFileSync("my_file.txt", "Hello world")
console.log("File created & written")

// Read file
const content = fs.readFileSync("my_file.txt", "utf8")
console.log("File content:", content)

// Delete file
fs.unlinkSync("my_file.txt")
console.log("File deleted")
```


API همگام ساده و برای اسکریپت‌های کوچک مناسب است، اما به دلیل اینکه برنامه را تا زمان اتمام مسدود می‌کند، می‌تواند در صورت کار با فایل‌های بزرگ یا سرور، باعث کندی شود.


### API ناهمگام Callback


**API فراخوانی** غیرمسدودکننده است: این به NodeJS اجازه می‌دهد تا در حالی که عملیات فایل انجام می‌شود، به انجام کارهای دیگر ادامه دهد.


به جای بازگرداندن نتیجه به‌طور مستقیم، یک تابع (**callback**) می‌گیرد که وقتی عملیات انجام شد، فراخوانی می‌شود.


```javascript
const fs = require("fs")

fs.readFile("example.txt", "utf8", (err, data) => {
if (err) {
console.error("Error reading file:", err)
} else {
console.log(data)
}
})
```


در اینجا اتفاقی که می‌افتد:



- `fs.readFile` شروع به خواندن `example.txt` می‌کند.
- NodeJS منتظر نمی‌ماند، بلکه به اجرای کدهای دیگری که ممکن است نوشته باشید، ادامه می‌دهد.
- وقتی خواندن فایل تمام شد، تابع بازگشتی اجرا می‌شود:



  - اگر خطایی وجود داشته باشد، `err` شامل خطا است.
  - در غیر این صورت، `data` حاوی محتویات است.


در اینجا نحوه نوشتن در یک فایل آمده است:


```javascript
const fs = require("fs")

fs.writeFile("output.txt", "Hello async!", (err) => {
if (err) {
console.error("Error writing file:", err)
} else {
console.log("File written!")
}
})
```


همان ایده: برنامه هنگام نوشتن فایل متوقف نمی‌شود.


برخی مثال‌ها از کارهایی که می‌توانید با این API انجام دهید:


```javascript
const fs = require("fs")

// List files and folders
fs.readdir(".", (err, items) => {
if (err) return console.error(err)
console.log("Items in current directory:", items)
})

// Create folder
fs.mkdir("my_folder", (err) => {
if (err) return console.error(err)
console.log("Folder created")
})

// Delete folder
fs.rmdir("my_folder", (err) => {
if (err) return console.error(err)
console.log("Folder deleted")
})

// Create & write file
fs.writeFile("my_file.txt", "Hello world", (err) => {
if (err) return console.error(err)
console.log("File created & written")
})

// Read file
fs.readFile("my_file.txt", "utf8", (err, content) => {
if (err) return console.error(err)
console.log("File content:", content)
})

// Delete file
fs.unlink("my_file.txt", (err) => {
if (err) return console.error(err)
console.log("File deleted")
})
```


API بازگشتی برای سرورها و وظایف بزرگ بهتر است زیرا برنامه را مسدود نمی‌کند، اما بازگشت‌های تو در تو می‌توانند پیچیده شوند اگر عملیات‌های زیادی را زنجیره کنید. به همین دلیل یک API ناهمگام مبتنی بر وعده اضافه شد.


### وعد API ناهمگام


API مبتنی بر Promise مدرن است و به‌خوبی با `.then()` و `async/await` کار می‌کند. این به‌صورت `fs.promises` در دسترس است.


باید ویژگی `promises` را وارد کنید:


```javascript
const fs = require("fs").promises
```


استفاده از `.then()`:


```javascript
const fs = require("fs").promises

fs.readFile("example.txt", "utf8")
.then(data => {
console.log(data)
})
.catch(err => {
console.error("Error reading file:", err)
})
```


یا حتی بهتر، با استفاده از `async/await`:


```javascript
const fs = require("fs").promises

async function readFile(fileName) {
try {
const data = await fs.readFile(fileName, "utf8")
console.log(data)
} catch (err) {
console.error("Error reading file:", err)
}
}

readFile("example.txt")
```


نوشتن در یک فایل:


```javascript
const fs = require("fs").promises

async function writeFile(fileName, content) {
try {
await fs.writeFile(fileName, content)
console.log("File written!")
} catch (err) {
console.error("Error writing file:", err)
}
}

writeFile("output.txt", "Hello from promises!")
```


فهرست معمولی از مثال‌ها برای API:


```javascript
const fs = require("fs").promises

// Use an async function to await operations
async function main() {
// List files and folders
const items = await fs.readdir(".")
console.log("Items in current directory:", items)

// Create folder
await fs.mkdir("my_folder")
console.log("Folder created")

// Delete folder
await fs.rmdir("my_folder")
console.log("Folder deleted")

// Create & write file
await fs.writeFile("my_file.txt", "Hello world")
console.log("File created & written")

// Read file
const content = await fs.readFile("my_file.txt", "utf8")
console.log("File content:", content)

// Delete file
await fs.unlink("my_file.txt")
console.log("File deleted")
}

main().catch(err => console.error(err))
```



## NPM

<chapterId>a91d9a75-55cc-51a3-a48f-0c0be6fe6e72</chapterId>


وقتی کد می‌نویسید، اغلب نیاز خواهید داشت از کدی که توسط افراد دیگر نوشته شده است استفاده کنید؛ برای مثال، کتابخانه‌هایی که به شما در کار با تاریخ‌ها، رنگ‌ها، سرورها، یا تقریباً هر چیز دیگری کمک می‌کنند.


به جای دانلود و کپی کردن فایل‌ها به صورت دستی، می‌توانید از یک **مدیر بسته** استفاده کنید.


یک مدیر بسته ابزاری است که:



- بسته‌های دانلود
- ردیابی می‌کند که پروژه شما به کدام بسته‌ها نیاز دارد
- مطمئن شوید که همه اعضای تیم شما نسخه‌های یکسانی از بسته‌ها دارند


### NPM چیست؟


در دنیای NodeJS، محبوب‌ترین مدیر بسته **NPM** است که مخفف *Node Package Manager* می‌باشد.


هنگام نصب NodeJS به طور خودکار NPM را دریافت می‌کنید.


می‌توانید با اجرای این دستور در ترمینال بررسی کنید که آیا NPM دارید یا خیر:


```
npm -v
```


این نسخه NPM شما را چاپ می‌کند، مانند:


```
10.2.1
```


### ایجاد یک بسته


در NodeJS، یک **پکیج** فقط یک دایرکتوری با یک فایل `package.json` در آن است.


بیایید مرحله به مرحله یکی بسازیم.


1. یک پوشه جدید برای پروژه خود بسازید:


```
mkdir my_project
cd my_project
```


۲. این دستور را اجرا کنید:


```
npm init
```


این یک راه‌اندازی تعاملی را شروع می‌کند و از شما سوالاتی مانند نام پروژه، نسخه، توضیحات و غیره می‌پرسد.


اگر نمی‌خواهید به همه چیز پاسخ دهید و فقط مقادیر پیش‌فرض را بپذیرید، می‌توانید از این استفاده کنید:


```
npm init -y
```


پس از اجرای آن، یک فایل جدید در پوشه شما ظاهر خواهد شد به نام:


```
package.json
```


### `package.json`


فایل `package.json` فقط یک فایل JSON است که اطلاعات متادیتا درباره پروژه شما را ذخیره می‌کند.


در اینجا یک مثال است:


```json
{
"name": "my_project",
"version": "1.0.0",
"description": "",
"main": "index.js",
"scripts": {
"test": "echo \"Error: no test specified\" && exit 1"
},
"keywords": [],
"author": "",
"license": "ISC",
"type": "commonjs"
}

```


چندین زمینه مهم:



- `name`: نام بسته شما
- `نسخه`: نسخه فعلی
- `main`: فایل نقطه ورود (مانند `index.js`)
- `scripts`: دستوراتی که می‌توانید اجرا کنید (مانند `npm start`)
- `dependencies`: تمام بسته‌هایی را که پروژه شما به آن‌ها وابسته است فهرست می‌کند


### نصب یک بسته


بیایید بگوییم که می‌خواهید از یک بسته خاص به نام `picocolors` برای افزودن رنگ به خروجی ترمینال خود استفاده کنید.


می‌توانید آن را با اجرای دستور زیر نصب کنید:


```
npm install picocolors
```


اکنون می‌توانید از آن در پروژه خود استفاده کنید. یک فایل `index.js` بسازید با


```javascript
const pico = require('picocolors')

console.log(
pico.green("This text is green!")
)
```


و سعی کنید آن را اجرا کنید. ترمینال باید نسخه‌ای رنگی از متن را چاپ کند.


NPM چه کاری انجام داد؟



- بسته را دانلود کرد و آن را در زیرپوشه‌ای به نام `node_modules/` ذخیره کرد.
- آن یک ورودی تحت `dependencies` در فایل `package.json` شما اضافه کرد.
- فایل `package-lock.json` را به‌روزرسانی کرد.


`package-lock.json` چیست؟


### `package-lock.json`


این فایل به‌طور خودکار توسط NPM ایجاد شده است.


ممکن است تعجب کنید، اگر ما قبلاً `package.json` را داریم، چرا به فایل دیگری نیاز داریم؟

در اینجا دلیل آن است:



- `package.json` فقط محدوده **نسخه**‌ای از یک بسته را که پروژه شما نیاز دارد، مشخص می‌کند.

مثال:


```json
"dependencies": {
"picocolors": "^1.1.0"
}
```


`^1.1.0` به معنای "هر نسخه‌ای که با 1.1.x سازگار است" می‌باشد، بنابراین انعطاف‌پذیر است.



- `package-lock.json` **نسخه‌های دقیق** هر بسته و زیر وابستگی‌های آن را فریز می‌کند، به طوری که هر کسی که پروژه شما را نصب می‌کند، دقیقاً همان تنظیمات کاری را دریافت می‌کند.


چرا این مهم است؟


اگر در یک تیم کار می‌کنید، یا پروژه خود را روی یک سرور مستقر می‌کنید، یا در آینده به آن بازمی‌گردید، می‌خواهید مطمئن شوید که همچنان به همان شکل کار می‌کند.

اگر بسته‌ها به‌روزرسانی شده باشند و شما بدون یک فایل قفل دوباره نصب کنید، ممکن است نسخه‌ای کمی متفاوت دریافت کنید که رفتار متفاوتی داشته باشد.


با نگه‌داشتن `package-lock.json` در پروژه خود، NPM همیشه نسخه‌های دقیقی که در آنجا فهرست شده‌اند را نصب می‌کند و اطمینان حاصل می‌کند که همه یک محیط یکسان دارند.


فایل `package-lock.json` همه چیز را به یک نسخه بسیار خاص قفل می‌کند تا پروژه در ماشین‌های دیگر قابل بازتولید باشد.


اما اگر بسته شما نیاز دارد که توسط افراد زیادی استفاده شود، ممکن است تصمیم بگیرید که آن را متعهد نکنید، به طوری که NPM فقط فایل `package.json` را پیدا کند و اجازه داشته باشد به صورت خودکار آخرین نسخه‌هایی که در آن فایل مجاز هستند را نصب کند.


اما این‌ها چیزهایی هستند که باید بعداً نگرانشان باشید، زمانی که شروع به انتشار کد خودتان می‌کنید. فعلاً، ما اصول اولیه NPM را معرفی کردیم تا به شما اجازه دهیم کتابخانه‌هایی که توسط توسعه‌دهندگان دیگر منتشر شده‌اند را در پروژه‌هایتان پیدا کرده و استفاده کنید.



## شبکه‌سازی در NodeJS

<chapterId>f2cabd8b-754b-5c97-8d6a-8412a9a184c7</chapterId>


NodeJS اغلب به عنوان یک زبان برای بک‌اند استفاده می‌شود: شما می‌توانید اسکریپت خود را به یک سرور تبدیل کنید و همچنین از آن برای ارسال درخواست به سرورهای دیگر استفاده کنید.


در این فصل، ما قصد داریم برخی از ویژگی‌های پایه شبکه را معرفی کنیم که به شما این امکان را می‌دهد.


### `fetch()`


اگر می‌خواهید برنامه شما داده‌ها را از یک وب‌سایت یا یک API دانلود کند، باید یک **درخواست HTTP** ارسال کنید.


در نسخه‌های مدرن NodeJS، می‌توانید از تابع داخلی `fetch()` استفاده کنید.


در اینجا یک مثال از انجام یک درخواست GET به یک API آمده است:


```javascript
const response = await fetch("https://jsonplaceholder.typicode.com/posts/1")
const data = await response.text()

console.log(data)
```


وقتی این را اجرا می‌کنید، چیزی شبیه به این خواهید دید:


```
{
"userId": 1,
"id": 1,
"title": "...",
"body": "..."
}
```


در اینجا اتفاق می‌افتد:


1. `fetch()` یک URL می‌گیرد و یک درخواست ارسال می‌کند.

۲. این یک **Promise** را برمی‌گرداند که به یک شیء `Response` حل می‌شود.

۳. `response.text()` بدنهٔ پاسخ را به‌صورت یک رشته می‌خواند.


اما رشته‌ای که دریافت می‌کنید در واقع JSON است. آن چیست؟


### JSON


هنگام کار با APIهای وب، داده‌ها اغلب به صورت **JSON** ارسال و دریافت می‌شوند که مخفف JavaScript Object Notation است.


JSON فقط یک فرمت متنی است که بسیار شبیه به اشیاء JavaScript به نظر می‌رسد. برای مثال:


```json
{
"name": "Alice",
"age": 30,
"likes": ["apples", "bananas"]
}
```


شیء `JSON` یک ابزار داخلی در جاوااسکریپت است که می‌تواند برای کار با این فرمت داده استفاده شود.


می‌توانید یک شیء جاوااسکریپت را با استفاده از `JSON.stringify()` به یک رشته JSON تبدیل کنید:


```javascript
const user = { name: "Alice", age: 30 }

const jsonString = JSON.stringify(user)

console.log(jsonString)
```


این چاپ می‌کند:


```
{"name":"Alice","age":30}
```


می‌توانید متن JSON را با استفاده از `JSON.parse()` به یک شیء جاوااسکریپت تبدیل کنید:


```javascript
const jsonText = '{"name":"Bob","age":25}'

const obj = JSON.parse(jsonText)

console.log(obj)
```


این چاپ می‌کند:


```
{ name: 'Bob', age: 25 }
```


مراقب باشید: `JSON.parse()` خطا می‌دهد اگر رشته یک JSON معتبر نباشد.


```javascript
JSON.parse("not json") // ❌ Error!
```


بنابراین مطمئن شوید که رشته به درستی قالب‌بندی شده است.


### `http` سرور


NodeJS به شما امکان می‌دهد یک وب سرور ایجاد کنید بدون اینکه چیز دیگری نصب کنید.


می‌توانید از ماژول داخلی `http` برای مدیریت درخواست‌ها از مشتریان و ارسال پاسخ‌ها استفاده کنید.


در اینجا یک مثال بسیار ساده است:


```javascript
const http = require("http")

const server = http.createServer((req, res) => {
res.statusCode = 200
res.end("Hello from NodeJS server!")
})

server.listen(3000, () => {
console.log("Server running at http://localhost:3000/")
})
```


وقتی این اسکریپت را اجرا می‌کنید و `http://localhost:3000` را در مرورگر خود باز می‌کنید، خواهید دید:


```
Hello from NodeJS server!
```


این چیزی است که در کد اتفاق می‌افتد:


1. شما سرور `http` را از کتابخانه استاندارد Node وارد کردید.

۲. `http.createServer()` یک سرور ایجاد می‌کند. شما به `http.createServer()` یک تابع بازگشتی `(req, res) => {...}` دادید تا هر بار که کسی متصل می‌شود اجرا شود.

۳. شما یک کد وضعیت ۲۰۰ (که نشان‌دهنده یک عملیات موفق است) به پاسخ اختصاص دادید. می‌توانید درباره کدهای وضعیت HTTP [اینجا](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Status) بخوانید.

۳. `res.end()` پاسخ را ارسال کرده و اتصال را خاتمه می‌دهد.

۴. `server.listen(3000)` سرور را روی پورت ۳۰۰۰ راه‌اندازی می‌کند.


شما همچنین می‌توانید `req.url` و `req.method` را در درخواست بررسی کنید تا مسیرها یا انواع درخواست‌های مختلف را مدیریت کنید.


مثال با مسیریابی:


```javascript
const server = http.createServer((req, res) => {
if (req.url === "/") { // handle requests for the root of the website
res.statusCode = 200
res.end("Home page")
} else if (req.url === "/about") { // handle requests for the about page
res.statusCode = 200
res.end("About page")
} else {
res.statusCode = 404 // we send a 404 status code to signal that the requested page is missing
res.end("Not Found")
}
})
```


این‌ها مثال‌های بسیار پایه‌ای هستند. برای ساخت سرورهای پیشرفته‌تر، اکثر توسعه‌دهندگان احتمالاً یک کتابخانه آماده بک‌اند مانند [express](https://www.npmjs.com/package/express) را دانلود می‌کنند.


## پردازش داده‌ها: بافرها، رویدادها، جریان‌ها

<chapterId>8c9623f0-a604-51a4-8fe4-871c849d3e3b</chapterId>


در این فصل، ما عمدتاً سه دسته از اشیاء را معرفی خواهیم کرد:



- `بافر`، که نشان‌دهنده تکه‌های کوچک داده‌های باینری است
- `EventEmitter`، که می‌تواند برای ردیابی برخی مراحل توسط فرآیندهای غیرهمزمان با ارسال سیگنال‌هایی به نام "رویدادها" استفاده شود.
- `Stream`، که به ما اجازه می‌دهد بخش بزرگی از داده‌ها را یک `Buffer` در هر زمان پردازش کنیم، و که فرآیند را با انتشار رویدادها پیگیری می‌کند


این‌ها در کدهای حرفه‌ای NodeJS بسیار رایج هستند، بنابراین حتی اگر ممکن است در پروژه‌های اولیه خود از آن‌ها استفاده نکنید، خوب است که یک درک پایه‌ای داشته باشید تا زمانی که نیاز به تعامل با آن‌ها پیدا کنید.


### بافرها


در NodeJS، **بافر** نوعی شیء است که برای کار با داده‌های باینری استفاده می‌شود.


می‌توانید به یک بافر به عنوان یک ظرف با اندازه ثابت برای بایت‌های خام فکر کنید.


در اینجا نحوه ایجاد یک بافر از یک رشته آمده است:


```javascript
const buf = Buffer.from("hello")
console.log(buf)
```


این چیزی شبیه به این چاپ می‌کند:


```
<Buffer 68 65 6c 6c 6f>
```


آن اعداد (`68`, `65`, `6c`, etc.) نمایش‌های هگزادسیمال حروف در `"hello"` هستند.


می‌توانید آن را به یک رشته به این صورت تبدیل کنید:


```javascript
console.log(buf.toString())
```


این چاپ می‌کند:


```
hello
```


شما همچنین می‌توانید یک بافر با اندازه مشخص پر شده با صفرها ایجاد کنید:


```javascript
const buf = Buffer.alloc(10)
console.log(buf)
```


این چیزی شبیه به این چاپ می‌کند:


```
<Buffer 00 00 00 00 00 00 00 00 00 00>
```


می‌توانید در بافر بنویسید:


```javascript
buf.write("abc")
console.log(buf)
```


و می‌توانید به بایت‌های جداگانه دسترسی داشته باشید:


```javascript
console.log(buf[0]) // prints the ASCII number for 'a', which is 97
```


بافرها به‌ویژه زمانی مفید هستند که نیاز به دستکاری داده‌ها در سطح بسیار پایین دارید.


### رویدادها


در جاوااسکریپت، **رویداد** چیزی است که در برنامه شما اتفاق می‌افتد و می‌توانید به آن واکنش نشان دهید.


برای مثال:



- یک فایل بارگذاری را به پایان می‌رساند
- یک تایمر خاموش می‌شود
- یک کاربر روی یک دکمه کلیک می‌کند
- یک درخواست شبکه داده‌ها را برمی‌گرداند.


یک **رویداد** فقط یک سیگنال است که نشان می‌دهد چیزی اتفاق افتاده است، و شما می‌توانید کدی بنویسید که به این رویدادها گوش دهد و به آن‌ها واکنش نشان دهد.


در NodeJS، بسیاری از اشیاء می‌توانند رویدادها را منتشر کنند. این اشیاء **EventEmitters** نامیده می‌شوند.


در اینجا یک مثال است:


```javascript
const EventEmitter = require("events")

const emitter = new EventEmitter()

// Listen for an event
emitter.on("greet", () => {
console.log("Hello! An event happened.") // this will get printed when a "greet" event gets fired
})

// Emit the event
emitter.emit("greet")
```


این چاپ می‌کند:


```
Hello! An event happened.
```


اینجا این است که:


1. ما یک شیء `EventEmitter` ایجاد می‌کنیم.

۲. ما به آن می‌گوییم که هر زمان رویداد `"greet"` اتفاق افتاد، یک callback اجرا کند، با استفاده از `.on("greet")`.

۳. بعداً، ما رویداد `"greet"` را با استفاده از `.emit()` فعال می‌کنیم.

۴. فراخوان ما اجرا می‌شود


می‌توانید داده‌ها را همراه با رویداد ارسال کنید:


```javascript
emitter.on("greet",
(name) => console.log(`Hello, ${name}!`)
)

emitter.emit("greet", "Alice") // first argument is the type of event, second argument is the data we pass with this event
```


این چاپ می‌کند:


```
Hello, Alice!
```


می‌توانید شنوندگان را برای رویدادهای دیگر نیز ثبت کنید:


```javascript
emitter.on("goodbye", () => {
console.log("Goodbye!")
})

emitter.emit("goodbye")
```


می‌توانید به هر تعداد که می‌خواهید شنونده به یک نوع رویداد متصل کنید و می‌توانید انواع مختلفی از رویدادها را از همان منتشرکننده اجرا کنید.


بسیاری از اشیاء در NodeJS رویدادهایی را منتشر می‌کنند تا به بقیه برنامه بگویند که چیزی در حال وقوع است.



### جریان‌ها چیستند؟


جریان‌ها بافرها و رویدادها را ترکیب می‌کنند تا به ما در پردازش داده‌ها کمک کنند.


وقتی با فایل‌ها، داده‌ها از شبکه، یا حتی متن‌های طولانی کار می‌کنیم، همیشه نیازی نیست (یا نمی‌خواهیم) همه چیز را به‌طور همزمان در حافظه بارگذاری کنیم. این کار می‌تواند کند باشد، یا حتی اگر داده‌ها خیلی بزرگ باشند، برنامه را خراب کند.


در عوض، می‌توانیم داده‌ها را **کم‌کم** پردازش کنیم، همان‌طور که می‌رسند یا خوانده می‌شوند، شبیه به نوشیدن آب از طریق نی به جای تلاش برای نوشیدن کل لیوان به یک‌باره. این به عنوان یک **جریان** شناخته می‌شود.


در NodeJS، یک جریان شیئی است که به شما اجازه می‌دهد داده‌ها را از یک منبع بخوانید یا داده‌ها را به یک مقصد **یک تکه در یک زمان** بنویسید.


NodeJS چهار نوع اصلی از جریان‌ها دارد:



- قابل‌خواندن**: جریان‌هایی که می‌توانید داده‌ها را از آن‌ها بخوانید (مانند خواندن یک فایل)
- قابل نوشتن**: جریاناتی که می‌توانید داده‌ها را به آن‌ها بنویسید (مانند نوشتن در یک فایل)
- دوپلکس**: جریان‌هایی که هم قابل خواندن و هم قابل نوشتن هستند
- تبدیل**: مانند جریان‌های دوطرفه، اما می‌توانند داده‌ها را هنگام جریان تغییر دهند (تبدیل کنند)


### جریان‌های قابل خواندن


فرض کنید یک `bigfile.txt` برای پردازش دارید. می‌توانید با استفاده از ماژول `fs` یک جریان خواندنی ایجاد کنید تا فایل را تکه تکه بخوانید.


```javascript
const fs = require("fs")

const readableStream = fs.createReadStream(
"bigfile.txt"
)

readableStream.on("data", (chunk) => {
console.log("Received chunk:", chunk)
})

readableStream.on("end", () => {
console.log("Finished reading file.")
})

readableStream.on("error", (err) => {
console.error("Error reading file:", err)
})
```


اینجا چه اتفاقی می‌افتد؟


1. `fs.createReadStream()` یک جریان خواندنی ایجاد می‌کند.

۲. هر زمان که بخشی از فایل آماده شود، استریم یک رویداد `data` منتشر می‌کند و یک "تکه" از داده‌ها (یک `Buffer`) به ما می‌دهد. ما این تکه را چاپ می‌کنیم.

۳. زمانی که کل فایل خوانده شد، رویداد `end` فعال می‌شود.

۴. اگر خطایی رخ دهد (مانند عدم وجود فایل)، رویداد `error` فعال می‌شود.


به این ترتیب، می‌توانید فایل‌های بزرگ را بدون بارگذاری کامل آن‌ها در حافظه بخوانید.


اگر بخواهیم داده‌ها به صورت قابل خواندن برای انسان (به جای باینری) برسند، می‌توانیم کدگذاری جریان را مشخص کنیم:


```javascript
const fs = require("fs")

const readableStream = fs.createReadStream(
"bigfile.txt",
{ encoding: "utf8" } // we tell NodeJS that the file should be read as utf8
)

readableStream.on("data", (chunk) => {
console.log("Received chunk:", chunk)
})

readableStream.on("end", () => {
console.log("Finished reading file.")
})

readableStream.on("error", (err) => {
console.error("Error reading file:", err)
})
```


کد اکنون فایل را به صورت قابل خواندن برای انسان چاپ خواهد کرد.


### جریان‌های قابل نوشتن


یک جریان قابل نوشتن به شما اجازه می‌دهد داده‌ها را به جایی ارسال کنید، تکه به تکه.


در اینجا مثالی از نوشتن به یک فایل `target.txt` با استفاده از یک جریان آورده شده است:


```javascript
const fs = require("fs")

const stream = fs.createWriteStream("target.txt")

stream.write("First line\n")
stream.write("Second line\n")
stream.end("Finished writing\n")

stream.on("finish", () => {
console.log("All data written.")
})

stream.on("error", err => {
console.error("Error:", err)
})
```


در اینجا اتفاقی که می‌افتد:


1. `fs.createWriteStream()` یک جریان نوشتنی ایجاد می‌کند.

۲. با استفاده از `.write()` مقداری متن به آن می‌نویسیم.

۳. وقتی کارمان تمام شد، با فراخوانی `.end()` جریان را می‌بندیم.

۴. زمانی که تمام داده‌ها نوشته شد، رویداد `finish` منتشر می‌شود.

۵. اگر مشکلی پیش بیاید، رویداد `error` فعال می‌شود.


دقیقاً مانند جریان‌های قابل خواندن، جریان‌های قابل نوشتن برای داده‌های بزرگ مناسب هستند زیرا نیازی به نگه‌داشتن همه چیز در حافظه به‌طور همزمان ندارند.


### جریان‌های لوله‌کشی


یکی از جالب‌ترین چیزها درباره استریم‌ها این است که می‌توانید آن‌ها را به هم **متصل** کنید: یک استریم خواندنی را مستقیماً به یک استریم نوشتنی وصل کنید.


```javascript
const fs = require("fs")

const readable = fs.createReadStream("bigfile.txt")
const writable = fs.createWriteStream("target.txt")

readable.pipe(writable)
```


اینجا:



- جریان قابل خواندن از `bigfile.txt` می‌خواند.
- جریان قابل نوشتن به `copy.txt` می‌نویسد.
- `.pipe()` داده‌ها را مستقیماً از جریان خواندنی به جریان نوشتنی ارسال می‌کند.


### جریان‌های دوطرفه


یک جریان دوطرفه هم قابل خواندن و هم قابل نوشتن است. یک مثال از آن یک سوکت شبکه است: شما می‌توانید داده‌ها را به آن ارسال کنید و داده‌ها را از آن دریافت کنید.


در اینجا یک مثال بسیار ساده با استفاده از ماژول `net` آورده شده است:


```javascript
const net = require("net")

const server = net.createServer((socket) => {
socket.write("Welcome!\n")

socket.on("data", (chunk) => {
console.log("Received:",
chunk.toString()  // we convert the chunk of data from Buffer to string
)
})
})

server.listen(3000, () => {
console.log("Server listening on port 3000")
})
```


در این مثال:



- شیء `socket` یک جریان دوطرفه است.
- می‌توانید با استفاده از `write()` به آن بنویسید و همچنین به رویدادهای `data` از آن گوش دهید.


### جریان‌های تحول‌یافته


یک جریان تبدیل یک جریان دوپلکس است که داده‌هایی را که از آن عبور می‌کنند نیز تغییر می‌دهد.


به عنوان مثال، می‌توانید از ماژول داخلی `zlib` برای فشرده‌سازی یا از فشرده‌سازی خارج کردن داده‌ها استفاده کنید.


در اینجا نحوه فشرده‌سازی یک فایل با استفاده از یک جریان تبدیل آمده است:


```javascript
const fs = require("fs")
const zlib = require("zlib")

const readable = fs.createReadStream("bigfile.txt")     // create a readable stream that reads from a file
const zip = zlib.createGzip()                           // create a transform stream that compresses data
const writable = fs.createWriteStream("bigfile.txt.gz") // create a writable stream that writes to a file

readable          // take the readable stream
.pipe(zip)      // pipe it into the transform stream to compress the data
.pipe(writable) // then pipe it into the writable stream that saves the data to a zipped file

writable.on("finish", () => {
console.log("File compressed.")
})
```


و برای بازگرداندن آن به حالت فشرده نشده:


```javascript
const readable = fs.createReadStream("bigfile.txt.gz")
const unzip = zlib.createGunzip()
const writable = fs.createWriteStream("bigfile.txt")

readable.pipe(unzip).pipe(writable)

writable.on("finish", () => {
console.log("File decompressed.")
})
```


جریان‌های تبدیل برای وظایفی مانند فشرده‌سازی، رمزگذاری، یا تغییر فرمت فایل در حین پخش بسیار مفید هستند.


### فشار معکوس


گاهی اوقات یک جریان نوشتنی در پردازش داده‌ها کند است. اگر داده‌ها را سریع‌تر از توانایی پردازش به یک جریان نوشتنی ارسال کنیم، ممکن است با مشکلاتی مواجه شویم. به این وضعیت **فشار معکوس** گفته می‌شود.


اگر متد `.write()` را بر روی یک جریان قابل نوشتن فراخوانی کنید، یک مقدار بولی برمی‌گرداند که به شما اطلاع می‌دهد آیا جریان نیاز به توقف دارد یا خیر؛ ممکن است لازم باشد مقدار بازگشتی آن را به این صورت بررسی کنید:


```javascript
const fs = require("fs")

const readable = fs.createReadStream("example.txt")
const writable = fs.createWriteStream("copy.txt")
r
readable.on("data", chunk => {               // each chunk of data we read from the readable stream...

const canContinue = writable.write(chunk)  // ...we send it to the writable, which returns us a boolean to confirm we can continue

if (!canContinue) { readable.pause() }     // ...if we can't, we temporarily pause reading
})

writable.on("drain",                // the writable stream emits a "drain" event when the backpressure is gone

() => { readable.resume() }      // so we resume reading (and writing)

)
```


این یک مثال تصویری از انتقال دستی داده‌ها از یک Readable به یک Writable بود و مدیریت فشار برگشتی به صورت دستی.


معمولاً ما داده‌ها را با استفاده از متد `.pipe()` انتقال می‌دهیم که به‌طور خودکار فشار برگشتی را مدیریت می‌کند.


بنابراین فقط زمانی نیاز به نگرانی در مورد فشار معکوس دارید که به دلایلی به صورت دستی `.write()` را فراخوانی می‌کنید.
